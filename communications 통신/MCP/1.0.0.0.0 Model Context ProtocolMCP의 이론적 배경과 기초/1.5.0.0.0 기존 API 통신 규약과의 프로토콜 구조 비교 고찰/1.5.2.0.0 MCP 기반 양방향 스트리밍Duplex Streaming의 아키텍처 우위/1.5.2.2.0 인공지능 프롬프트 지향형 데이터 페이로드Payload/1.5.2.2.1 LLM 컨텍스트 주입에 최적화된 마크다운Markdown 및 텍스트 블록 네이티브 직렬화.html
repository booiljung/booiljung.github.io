1.5.2.2.1. LLM 컨텍스트 주입에 최적화된 마크다운(Markdown) 및 텍스트 블록 네이티브 직렬화

기존 애플리케이션 프로그래밍 인터페이스(API)의 페이로드(Payload) 표준은 의심할 여지 없이 JSON(JavaScript Object Notation)이었다. 클라이언트 애플리케이션의 프론트엔드 코드(JavaScript나 앱의 뷰 렌더러)가 이 중첩된 JSON 트리의 객체 키(Key) 값을 파싱(Parsing)하여 사용자 화면(UI)의 적절한 위치에 데이터를 조립하여 그리는 것이 일반적인 수순이다.

그러나 이 데이터를 최종적으로 수신하는 주체가 프론트엔드 모듈이 아니라 거대한 텍스트의 확률적 흐름을 읽어내는 **대규모 언어 모델(LLM)**이라면 상황은 완전히 달라진다.

0.1 구조적 JSON 파싱의 컨텍스트 낭비(Token Waste) 현상

LLM 에이전트가 데이터베이스에서 사용자 정보를 조회하기 위해 구형 REST API를 호출했다고 가정해 보자. 서버가 응답으로 다음과 같이 10줄짜리 JSON 데이터를 배출(Emit)한다면,
{"users": [{"id": 1, "metadata": {"name": "Alice", "role": "admin"}}, {"id": 2, ...}]}

LLM은 이 응답을 온전히 이해하기 위해 의미론적으로 전혀 쓸모없는 수많은 중괄호({, }), 대괄호([, ]), 쌍따옴표("), 그리고 콜론(:) 텍스트 조각들마저 소중한 **토큰(Token)**으로 할당하고 해독 연산(Inference)을 거쳐야 한다. 만약 전송해야 하는 데이터가 배열 1만 줄이 넘어가는 거대한 로그 파일이라면, 이 쓸모없는 기호(Syntax) 토큰들이 LLM의 제한된 컨텍스트 윈도우(Context Window)를 무의미하게 잠식해버리는 ’토큰 낭비 현상’이 극심해진다.

0.2 렌더링된 텍스트 블록(Text Block)의 강력함

Model Context Protocol(MCP)은 기계 친화적(Machine-centric)인 구조화 데이터 대신, LLM이 네이티브하게 소비(Consume)하기 가장 최적화된 형태인 자연어 텍스트(Text)와 마크다운(Markdown) 블록 단위의 직렬화(Serialization)를 페이로드 표준 스펙의 기반으로 채택하였다.

MCP 서버는 클라이언트가 특정 테이블 조회를 요청(resources/read)할 때, 단순히 JSON 덩어리를 뱉어내는 것이 아니라 서버 측에서 미리 데이터들을 사람이 바로 읽을 수 있는 마크다운 형식의 표(Table)나 포맷팅된 설명문 블록으로 렌더링(Rendering)하여 내려보낸다.

에이전트 인프라(클라이언트)는 이 반환된 데이터(텍스트, 이미지 블록 등)를 파싱할 필요 없이, 곧바로 LLM의 현재 세션 채팅 프롬프트(Prompt) 제일 밑단에 “붙여넣기(Append)“해 버리면 그만이다.

이러한 프롬프트 네이티브(Prompt-native) 페이로드 전략은 세 가지의 연산적 이득을 안겨준다.

  1. 토큰 다이어트(Token Diet): 불필요한 괄호 문법 등을 소거하고 정보의 알맹이만 마크다운 표 혹은 리스트 형태로 주입하여 컨텍스트 윈도우의 효율을 극대화한다.
  2. 의미론적 보장(Semantic Guarantee): JSON 속성 이름(Key)만 던져주며 LLM이 그 의미를 알아서 유추하게 방치하는 대신, ## 에러 로그 분석 결과\n - 유저 ID: 123 처럼 서버가 명확하게 맥락이 포함된 자연어로 헤더(Header) 제목과 설명을 감싸서 내려주기 때문에 LLM의 기계적 오독 현상(Hallucination)이 급감한다.
  3. UI/UX와 에이전트 간의 관점 통합(View Homomorphism): 사용자가 보는 채팅창에 이 마크다운 데이터를 그대로 렌더링해 줌으로써, AI 인공지능이 보는 시야와 사람이 보는 시야의 괴리를 근본적으로 일치시킨다.

결과적으로 MCP의 데이터 페이로드는 “시스템이 데이터를 교환한다“는 기존의 기계 부품적 관점을 부수고, 외부 도구(Tool) 서버가 에이전트 채팅 대화방에 참여한 ’또 한 명의 인간 전문가’처럼 깔끔하고 정돈된 보고서를 건네주는 문명화된 교환 방식으로 통신 스펙을 끌어올렸다.
