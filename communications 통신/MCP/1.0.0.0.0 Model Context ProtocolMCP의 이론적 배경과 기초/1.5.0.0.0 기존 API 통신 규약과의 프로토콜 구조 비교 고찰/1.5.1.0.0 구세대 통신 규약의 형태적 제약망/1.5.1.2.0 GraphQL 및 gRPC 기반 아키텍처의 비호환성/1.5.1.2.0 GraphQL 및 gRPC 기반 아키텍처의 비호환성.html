1.5.1.2. GraphQL 및 gRPC 기반 아키텍처의 비호환성

REST API의 한계를 극복하기 위해 등장한 GraphQL과 gRPC는 각각 뛰어난 기술적 성취를 이루어냈다. GraphQL은 단일 엔드포인트(Endpoint)를 통해 클라이언트가 원하는 데이터 필드만을 정확하게 선언하여 가져올 수 있게 함으로써 오버페칭(Over-fetching) 문제를 해결했고, 구글(Google)이 주도한 gRPC는 HTTP/2와 프로토콜 버퍼(Protobuf)를 결합하여 마이크로서비스 간의 통신 지연(Latency)을 이진(Binary) 레벨에서 압도적으로 줄여냈다.

그러나 인간의 프론트엔드 최적화와 서버 간의 성능 극대화를 위해 정교하게 조율된 이 최신 아키텍처들 역시, 대규모 언어 모델(LLM)을 기반으로 자율적인 추론을 전개하는 에이전트 인터페이스(Agentic Interface) 생태계에 편입되는 순간 전혀 예상치 못한 비호환성(Incompatibility)과 아키텍처적 불협화음을 발생시킨다.

0.1 고도로 전문화된 추론의 진입 장벽

LLM 에이전트가 외부 시스템과 원활하게 소통하기 위해서는 통신 규약 자체가 자연어의 연장선상에 있거나 직관적인 JSON 텍스트 모델에 기대고 있어야 한다. 하지만 GraphQL과 gRPC는 통신 효율을 끌어올리기 위해 시스템 고유의 복잡한 문법 구조, 즉 도메인 특화 언어(DSL, Domain-Specific Language)와 엄격한 이진 스키마(Binary Schema)를 에이전트에게 강제한다.

  • GraphQL의 경우: 에이전트는 깊은 중첩 구조를 지닌 Query나 Mutation의 문자열(String) 형식을 스스로 합성(Synthesizing)해 내야 한다. 하지만 LLM은 그래프 스키마의 깊이(Depth)나 관계성(Edges and Nodes)을 사전에 완벽히 인지하지 못하므로, 빈번한 문법 오류(Syntax Error)를 일으키며 쿼리 작성에 실패한다.
  • gRPC의 경우: 에이전트의 주된 입출력 포맷은 자연어 토큰(Token)의 집합인 텍스트 스트림이지만, gRPC의 프로토콜 버퍼(Protocol Buffers)는 기계계층의 이진 직렬화(Binary Serialization)를 채택하고 있다. 이는 에이전트가 그 자체로는 해독할 수 없는 거대한 암호문(Cipher)과 같으므로, 반드시 어딘가에 이진 형식을 JSON으로 번역해 주는 추가적인 트랜스코더(Transcoder) 프록시를 배치해야 하는 번거로움을 낳는다.

0.2 유연한 도구 발견(Tool Discovery)의 상실

MCP(Model Context Protocol)가 추구하는 에이전트 생태계의 핵심은 런타임에 서버가 주도하여 자신의 능력을 클라이언트에게 동적으로 던져주는 “자가 선언적(Self-describing)” 생태계에 있다.

하지만 GraphQL과 gRPC는 클라이언트 측에서 미리 서버의 인터페이스 정의 언어(IDL, Interface Definition Language) 파일(예: .graphql 폴리시 구조나 .proto 파일)을 컴파일(Compile) 시점에 정적으로 내려받아 코드(Stub)를 생성해 두어야만 정상적으로 작동하는 강결합(Tight Coupling) 모델을 원칙으로 삼고 있다.

이러한 강결합 구조는 에이전트가 동적으로 세상을 탐색하고 순간순간 적절한 자원(Resource)과 도구(Tool)를 불러와 유기적인 대화 맥락을 이어나가는 과정에 심각한 족쇄(Shackle)가 된다. 서버의 스키마가 한 줄이라도 변경되면, 클라이언트 에이전트의 워크플로우를 모두 일시 정지하고 IDL을 새롭게 컴파일하여 주입해야 하는 것이다.

결론적으로, GraphQL과 gRPC는 각각 데이터 응집도(Cohesion)와 직렬화 속도(Serialization Speed) 면에서 과거 통신망의 한계를 돌파했지만, “자율성을 지닌 확률 모델“과 통신한다는 관점에서는 인간 중심의, 혹은 기계 중심의 경직된 철학에서 한 발자국도 벗어나지 못했다. 다음 세부 절들에서는 이 훌륭한 두 기술이 어째서 에이전트 친화적(Agent-friendly) 워크플로우와 치명적으로 어긋나는지를 각각의 설계 철학 관점에서 낱낱이 해부해 볼 것이다.
