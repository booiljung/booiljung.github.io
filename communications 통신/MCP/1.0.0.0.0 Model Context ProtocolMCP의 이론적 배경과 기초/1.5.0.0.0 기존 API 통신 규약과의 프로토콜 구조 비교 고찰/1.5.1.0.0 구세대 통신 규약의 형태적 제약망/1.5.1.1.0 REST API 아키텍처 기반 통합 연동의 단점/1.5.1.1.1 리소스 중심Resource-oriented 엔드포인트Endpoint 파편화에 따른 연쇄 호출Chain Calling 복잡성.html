1.5.1.1.1. 리소스 중심(Resource-oriented) 엔드포인트(Endpoint) 파편화에 따른 연쇄 호출(Chain Calling) 복잡성

REST(Representational State Transfer) 아키텍처가 지배해 온 패러다임의 기저에는 ’세상의 모든 상태와 행동을 명사(Noun), 즉 리소스(Resource) 단위로 쪼개어 엔드포인트(Endpoint)로 매핑한다’는 굳건한 철학이 자리 잡고 있다. 예를 들어 사용자, 결제, 상품이라는 모델이 있다면, 이들은 각각 /users, /payments, /products라는 엄격하게 파편화된(Fragmented) 경로를 할당받는다. 인간이 작성한 프론트엔드 애플리케이션은 이 쪼개진 URI들을 논리적으로 엮어내는 접착제(Glue) 체계를 코드 레벨에서 사전에 통제할 수 있지만, 인공지능 에이전트의 워크플로우에서는 이 리소스 중심(Resource-oriented) 설계가 치명적인 병목 현상(Bottleneck)으로 둔갑한다.

0.1 복합 추론(Complex Reasoning) 과정에서의 API 연쇄 호출 마찰

대규모 언어 모델(LLM)이 수행하는 임무는 대부분 단일 리소스의 단순 조회가 아닌, 여러 데이터 도메인에 걸친 다중 논리 추론(Multi-step Logic Reasoning)이다.

가령 사용자가 인공지능 에이전트에게 “가장 최근에 가입한 활성 사용자(Active User)가 어제 구매한 상품의 배송 상태를 확인해 줘“라고 자연어로 지시했다고 가정해 보자. 이 단순해 보이는 요청을 처리하기 위해, 엄격한 REST API 기반의 환경에서 에이전트는 다음과 같은 강제적인 연쇄 호출(Chain Calling) 늪에 빠지게 된다.

  1. 에이전트는 먼저 /users?status=active&sort=created_at_desc&limit=1을 호출하여 user_id를 가져온다.
  2. 이후 파싱된 user_id를 파라미터로 삼아 /users/{user_id}/orders?date=yesterday를 다시 반환받아 도출된 order_id를 추출한다.
  3. 마지막으로 /orders/{order_id}/deliveries를 요청하여 최종 배송 상태를 확정한다.

이러한 단계적 프로시저(Procedure) 전개는 매 호출마다 전체 네트워크 레이턴시(Network Latency)를 누적시킬 뿐만 아니라, 이전 단계의 JSON 응답 결과를 LLM이 다시 문자열로 파싱하여 다음 도구 호출(Tool Calling)의 JSON 파라미터를 조립해야 하는 심각한 추론 소모를 강요한다. 에이전트는 한 번의 종합적인 조망(Holistic view)으로 끝낼 수 있는 통찰 기반의 사고를, REST 엔드포인트의 구조적 조각 맞추기에 낭비하게 된다.

0.2 컨텍스트 이탈(Context Loss)과 API 에러 전파(Error Propagation)의 치명성

무엇보다 연쇄 호출(Chain Calling)이 늘어날수록 에이전트가 통제해야 하는 실패 지점(Single Point of Failure)이 기하급수적으로 늘어난다.

만약 두 번째 체인인 /users/{user_id}/orders 요청 단계에서 서버가 500 에러를 뱉거나 403 권한 없음을 반환한다면, 에이전트의 추론 트리는 중간에 끊어지게 된다.人間은 직관적으로 원인을 파악하여 다른 페이지를 탐색하겠지만, LLM은 종종 이전의 성공했던 1단계 상태(user_id를 획득한 사실)조차 망각하고 환각(Hallucination)에 빠져 무의미한 에러 복구 프롬프트를 남발하거나 엉뚱한 파라미터로 무한 루프(Infinite Loop)에 빠지는 현상을 일으킨다.

다음의 시퀀스 다이어그램은 REST의 파편화된 엔드포인트가 에이전트의 행동을 어떻게 파생시키고, 이것이 어떠한 형태적 복잡성(Morphological Complexity)을 야기하는지 시각화한 것이다.

결론적으로 REST API의 명사(Noun) 중심, 자원 분할적 철학은 인간의 브라우저가 화면을 부분 유닛으로 렌더링하기 위해 고안된 패러다임이다. 이를 동적인 상황 판단과 거시적 데이터(Macro-data) 종합이 필요한 에이전트 레이어에 그대로 강제하는 것은, 인공지능에게 자원의 실체가 아닌 흩어진 퍼즐 조각의 URI 지도만을 쥐여주며 미로를 탐색하게 하는 가혹한 제약망인 것이다. Model Context Protocol(MCP)은 이 파편화된 리소스 호출을 추상화된 단일 도구(Tool) 프레임워크 안에 캡슐화(Encapsulation)함으로써 이러한 체인 호출의 비효율성을 무효화한다.
