1.1.1.2.1 확률 기반 다음 토큰(Next Token) 예측 알고리즘의 본질적 정보 왜곡 한계

대규모 언어 모델(LLM)이 그럴듯한 허위 정보(Hallucination)를 생성해내는 가장 근본적인 수학적 이유는, 트랜스포머(Transformer) 디코더(Decoder) 아키텍처의 핵심 작동 원리인 ‘다음 토큰 예측(Next Token Prediction)’ 알고리즘에 숨어 있다. 언어 모델은 근본적으로 진실(Truth)이나 사실 관계(Fact)를 이해하는 데이터베이스가 아니다. 이들은 철저히 이전 문맥(Context)이 주어졌을 때, 학습 코퍼스(Corpus)의 통계적 분포를 바탕으로 가장 높은 확률을 가진 문자의 조각(Token)을 순차적으로 생성해 나가는 확률론적 텍스트 생성기(Probabilistic Text Generator)이다.

이 알고리즘 체계 내에서 언어 모델의 유일한 목적 함수(Objective Function)는 생성된 문장의 어법학적(Syntactic) 자연스러움과 통계적 유창성(Fluency)을 극대화하는 것이다. 여기에는 명제적 참거짓이나 사실의 연역적 타당성을 판별하는 로직이 완전히 배제되어 있다. 즉, 모델은 자신이 출력하는 토큰의 조합이 현실 세계에서 어떠한 물리적 법칙이나 역사적 사실과 위배되는지에는 전혀 관여하지 않으며, 오직 “이 단어 다음에는 이 단어가 오는 것이 통계적으로 가장 자연스럽다“는 확률적 매핑(Mapping) 연산만을 수행한다.

이러한 특성은 치명적인 정보 왜곡 한계를 노출한다. 예를 들어, 전문적인 의학 용어나 특정 국가의 생소한 법령과 같이 코퍼스(Corpus) 내에서의 출현 빈도가 낮아 확률 분포가 희소한(Sparse) 도메인에 직면했을 때, 모델은 지식 공간의 빈틈을 “모른다“고 답하는 대신 가장 그럴듯해 보이는 보편적 패턴의 단어들로 결합하여 매끄러운 거짓 문장을 직조해낸다. 그 논리와 문법이 너무나도 인간의 언어와 유사하고 완벽에 가깝기 때문에 사용자와 시스템은 이를 사실로 오인하게 되는 것이다.

더욱 심각한 것은 이러한 확률적 생성 방식이 자가 증폭 효과(Self-amplification Effect)를 가진다는 점이다. 다음 토큰을 예측하는 과정에서 초기에 아주 미세하게 왜곡된 사실이 포함된 토큰이 한 번 출력되면, 이는 자가 회귀(Auto-regressive) 아키텍처의 특성에 따라 이후 문맥의 입력값으로 다시 피드백된다. 모델은 자신의 거짓말을 문맥적 사실로 수용하여 그 위에 더 정교한 논리적 거짓을 쌓아 올리는 눈덩이(Snowball) 오류 사이클을 형성하게 된다.

결론적으로, 다음 토큰 예측이라는 알고리즘적 유전자는 언어 모델을 창의력을 요하는 문장 생성(Generation)에는 탁월하게 만들지만, 결정론적(Deterministic) 정답과 팩트 체크(Fact Checking)가 요구되는 시스템 통합에서는 제어 불가능한 변수 덩어리로 전락시킨다. 이는 정보 검색 증강 생성(RAG, Retrieval-Augmented Generation)이나 Model Context Protocol (MCP)와 같이, 생성 이전 단계에서 외부의 절대적이고 확고한 프롬프트(Prompt) 지식 컨텍스트를 강제로 주입하여 확률 벡터의 방향을 ‘진실’ 쪽으로 고정시키는 아키텍처의 필요성을 강력하게 방증한다.
