1.1.1.3 파라미터 내부 기억 공간의 비효율적 할당 구조 원리

대규모 언어 모델(LLM)이 당면한 또 다른 심각한 아키텍처적 딜레마는, 모델이 세상의 방대한 지식(Knowledge)을 기억(Memorization)하고 저장하는 방식 그 자체의 비효율성에 있다. 언어 모델을 단순한 함수(Function)나 추론 엔진(Reasoning Engine)이 아니라, 인류의 모든 지배적 지식을 담아내는 ’거대한 데이터베이스(Database)’로 취급하려는 패러다임은 모델의 파라미터(Parameter) 확장 경쟁을 강제해왔다. 그러나 트랜스포머(Transformer) 아키텍처의 내부 기억 공간 할당 방식은 전통적인 정보 저장 시스템과 비교할 때 극도로 비효율적이며 구조적인 낭비를 수반한다.

데이터베이스 시스템(RDBMS)이나 검색 인덱스(Search Index) 구조는 정보의 저장과 색인(Indexing), 추출(Retrieval)이 명확한 주소(Address)와 스키마(Schema)에 의해 결정론적으로 관리된다. 반면, 언어 모델은 특정 사실(Fact)을 특정 뉴런(Neuron) 위치에 할당하지 않는다. 지식은 수천억 개의 파라미터 계층을 오가는 다차원 벡터(Vector) 공간 위에 넓고 얕게, 그리고 암묵적(Implicit)으로 분산되어 저장된다.

이러한 암묵적이고 분산된 지식의 압축 방식은 문맥의 뉘앙스를 파악하고 유창한 텍스트를 생성하는 데에는 탁월한 이점을 제공하지만, “지식의 정확한 보존과 관리“라는 측면에서는 다음과 같은 세 가지 치명적인 구조적 제약을 발현시킨다.

첫째, 모델 내부에 융해된 지식을 사용자가 원할 때 왜곡 없이 온전히 꺼내어 쓰는 거대 매개변수에 분산된 팩트(Fact) 정보의 명시적 추출 제약성이다.
둘째, 새로운 지식을 주입하거나 기존 지식을 교정하기 위해 모델의 가중치를 미세 조정(Fine-Tuning)할 때 필연적으로 동반되는 치명적 망각(Catastrophic Forgetting) 현상이다.
셋째, 학습 코퍼스(Corpus)에 드물게 등장하는 지식을 모델이 기억하지 못하고 증발시켜버리는 저빈도(Low-frequency) 코퍼스의 기억 실패율과 파라미터 점유 비효율성 간의 상관관계이다.

결론적으로, 언어 모델의 가중치 네트워크를 사실(Fact)의 저장소로 활용하려는 접근은 연산 자원의 극심한 낭비를 초래할 뿐만 아니라, 지식의 무결성을 보장하지도 못한다. 이는 모델의 파라미터는 논리적 추론(Reasoning)과 언어적 표현 능력(Linguistic Representation)을 향상시키는 데에만 집중적으로 할당하고, 실질적인 사실 정보(Fact)의 보존 및 추출은 외부의 전용 저장 플랫폼에 위임해야 한다는 인프라 설계의 대원칙을 성립시킨다. 향후 전개될 하위 절에서는 이러한 파라미터 기억 공간의 비효율성을 야기하는 세부적인 구조 원리를 논증한다.
