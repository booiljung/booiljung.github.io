1.2.1.3.3 관련성이 낮은 문서 다수 포함 시 발생하는 주의력 분산(Attention Dilution) 증상

다중 문서 탐색 기반의 대규모 언어 모델(LLM) 환경에서 가장 빈번하게 발생하는 인지적 장벽 중 하나는 ‘주의력 분산(Attention Dilution)’ 증상이다. 모델의 어텐션(Attention) 자원은 무한하지 않고, 한정된 가중치(Weight) 확률의 총합 1(100\%) 내에서 자원을 배분해야만 하는 제로섬(Zero-Sum) 형태의 배타적 게임 구조를 띤다. 따라서 정보의 질적 정합성이 확보되지 않은 맹목적인 복수 문서(Multi-Document) 주입은 타겟 문서에 집중되어야 할 주의력을 부수적인 텍스트들로 소산시켜 시스템 추론력을 마비시킨다.

0.1 매트릭스 차원에서의 활성화(Activation) 희석과 수학적 증명

언어 모델의 트랜스포머 디코딩(Transformer Decoding) 블록 내부에서 관련성이 떨어지는(Irrelevant) 다수의 문서를 억지로 이어 붙여 컨텍스트 윈도우를 수집한다고 가정하자. 어텐션 스코어의 총합 제약성으로 인해 타겟 토큰이 가져가야 할 스코어의 임계 수치는 다음과 같이 하향 조정된다.

단일 타겟 문서만 주어졌을 때의 특정 핵심 토큰 T_{target}이 획득하는 어텐션 스코어를 A(T_{target})이라 하고, K개의 무관한 노이즈 문서가 동시 병합 삽입되었을 때 변동된 스코어를 A'(T_{target})이라 하면 그 역학은 분산 방정식으로 설명된다.

A'(T_{target}) = \frac{\exp(e_{target})}{\sum_{i \in \text{Target}} \exp(e_i) + \sum_{j \in \text{Noise}} \exp(e_j)}

위 수식에서 방대한 무관 문서군(\text{Noise})의 삽입으로 인해 분모 항목이 거대하게 증식한다. 비록 각 노이즈 토큰이 갖는 내적 스코어(e_j) 값 자체는 작더라도, 수만 개에 이르는 군집적 수량이 모이면 분모를 압도적으로 팽창시키고, 결과적으로 타겟 토큰이 차지하는 정규화 확률인 A'(T_{target})은 한 자릿수 퍼센티지 미만으로 추락하는 가중치 붕괴(Weight Collapse) 현상에 직면한다.

결과적으로, 다수의 무관 문서가 산만하게 주입되었을 때, 인간은 특정 문서 하나를 골라내어 읽을 수 있는 선별 능력이 있으나 언어 모델은 수학적 평활화로 인하여 모든 텍스트를 “똑같이 흐릿하게 중요한” 것으로 간주해버리는 주의력 분열증을 겪게 된다.

2. 검색 증강 생성(RAG) 및 MCP 연동 시의 치명적 설계 오류

이 모델 역학은 검색 증강 생성(RAG, Retrieval-Augmented Generation) 파이프라인과 Model Context Protocol(MCP) 클라이언트 아키텍처에 경종을 울린다. 흔히 검색 엔진이나 벡터 데이터베이스를 활용하여 유사도 상위 Top\text{-}K 개의 문헌을 모델에 반환할 때, K 값을 무리하게 20 혹은 50 이상으로 높여 반환(Recall) 가능성을 극대화하려는 전략을 빈번히 사용한다.

위의 위상 공간 도식에서 증명하듯이, 관련도가 희박한 문서를 다수 포함시키는 아키텍처는 데이터 보존이라는 측면에서는 방어적인 스텐스일 수 있으나 LLM 연산 측면에서는 가장 공격적인 추론 교란 기술에 해당한다. 연관성이 희소(Sparse)한 45개의 문서들이 지닌 자질구레한 토큰 매핑 엣지(Edge)들이 거미줄처럼 엮여서, 정작 중요한 의미망 연산의 자원을 흡수해버리는 이른바 ’블랙홀 효과’를 낳는다.

3. 구조적 해법 제언: 적응형 컨텍스트 배제 모델링

주의력 분산 현상을 막보자 한다면, 시스템 아키텍처 단에서 무조건적인 컨텍스트 허용(Context Allowance) 정책을 지양하고 엄격한 게이팅 메커니즘(Gating Mechanism)을 도입해야만 한다. MCP를 매개로 하는 정보망에서는 자원 도구(Tool)가 문서를 단순히 복사-전달하는 중계기 이상의 역할을 수행해야 한다.

클라이언트 미들웨어에서 먼저 검색된 문서들의 연관성 밀도(Relevance Density)를 계산하여, 특정 임계선(Threshold Borderline) 이하의 유사성 지표를 나타내는 문서는 모델 내부로 아예 진입조차 하지 못하도록 차단하는 적응형 컨텍스트 배제(Adaptive Context Exclusion) 필터링 망을 필수적으로 설계해야만, 분산 위기에 처한 모델의 잔존 인지 능력을 보전할 수 있다.
