1.2.2.1.3 다단계 도약 검색(Multi-hop Retrieval) 시 도출되는 노이즈 증폭 및 추론 실패 역설

단일 레퍼런스로 대답할 수 없는 고차원적인 복합 질의(Complex Query)를 해결하기 위해, 검색 증강 생성(RAG) 프레임워크는 여러 문서들을 징검다리 밟듯 순차적 혹은 병렬적으로 탐색하는 ’다단계 도약 검색(Multi-hop Retrieval)’을 도입하였다. “A 기업의 자회사 B가 작년에 출시한 C 제품의 영업 이익률과, 그 경쟁사 중심 제품과의 점유율 차이를 계산하라“와 같은 질의는 필연적으로 B의 정체 파악(1-Hop), C 제품 검색(2-Hop), 재무 제표 탐색(3-Hop), 경쟁사 탐색 및 수치 계산(4-Hop) 이라는 복합 체인을 요구한다. 그러나 이 연쇄적 패러다임은 단계가 깊어질수록 노이즈(Noise)가 지수 함수적으로 증폭되는 치명적인 역설을 내포한다.

0.1 연쇄적 에러 전파(Cascading Error Propagation) 매커니즘

다단계 도약 질의의 본질적 위험성은 이전 단계 검색의 정확도가 다음 단계 검색의 입력(Input) 품질을 완벽히 결정짓는 종속 구조(Dependent Structure)에 있다. 즉, 1-Hop에서 90\%의 정확도로 ’자회사 B’를 특정했고, 2-Hop에서 다시 90\% 확률로 올바른 ’C 제품’을 찾더라도, 4-Hop까지 이어지게 되면 최종적인 정보 색인 정확도는 0.9^4 \approx 65\% 수준으로 급감하게 된다.

가장 큰 문제는 중간 단계의 10\% 실패가 일으키는 연쇄 작용이다. 만약 1-Hop 쿼리에서 ’자회사 B’가 아닌 철자가 비슷한 ’자회사 D’의 문서가 섞여 들어왔다면, 이후의 플로우는 ’D 기업’을 파고드는 가짜 정보 체인(Fake Information Chain)으로 변질된다. 대규모 언어 모델(LLM)은 이를 의심 없이 사실로 받아들여 최종적으로 “D 기업의 E 제품이 훌륭하다“는 완전히 왜곡된 망상(Hallucination)의 결과를 태연하게 도출한다.

0.2 다중 컨텍스트 병합에 따른 주의력 분산(Attention Distraction) 현상

Multi-hop 시스템은 여러 도약을 거쳐 획득한 다수의 문헌 조각들을 하나의 거대한 프롬프트에 병합(Concatenation)하여 LLM에 밀어 넣는다. 이때, 1-Hop, 2-Hop, 3-Hop에서 각각 불러온 문서들은 본래 서로 다른 목적으로 작성된 이질적인 맥락(Heterogeneous Contexts)을 지니고 있다.

이들이 무분별하게 하나의 선형 컨텍스트로 결합될 때, 트랜스포머(Transformer)의 어텐션 메커니즘은 핵심 정보에 집중(Focus)하지 못하고 수많은 무관한 문장에 할당된 어텐션 가중치(Attention Weight)에 에너지를 분산시킨다. 결과적으로 잡음 대비 신호 비율(SNR, Signal-to-Noise Ratio)이 극도로 저하되며, 정답을 생성하는 데 필요한 1%의 핵심 문장이 나머지 99%의 배경 설명이나 관계없는 지표들에 파묻혀 ’주의력 희석(Attention Dilution)’을 일으키고 결국 모델이 추론(Reasoning)을 포기하거나 엉뚱한 결론을 내리게 만드는 추론 실패(Reasoning Failure)를 유발한다.

0.3 Model Context Protocol(MCP)를 통한 절차적 통제 지향성

위와 같은 Multi-hop 검색의 자가당착을 극복하기 위해서는, 수동적으로 외부 데이터를 쓸어 담아 한 번에 컨텍스트로 밀어 넣는 벡터 검색 방식만으로는 어림도 없다.

Model Context Protocol(MCP)이 제안하는 돌파구는 언어 모델이 ‘스스로’ 도구를 탐색하고, 첫 단계가 완료된 이후 자체적으로 중간 결과를 검증하여 다음 단계 리소스의 호출 여부를 결정하는 ‘동적이고 절차적인 통제(Procedural Routing)’ 방식에 있다. MCP는 LLM에게 단순 텍스트 프래그먼트를 던져주는 것에 머물지 않고, 함수 호출(Function Calling) 체계를 통한 프로토콜 규약을 제시함으로써 노이즈가 증폭되기 전 단계별로 맥락을 정리하고 필터링할 수 있는 독립적인 사고 사슬(Chain of Thought) 공간을 확보해 준다. 이는 곧 Multi-hop의 비극을 끝내고 진정한 자율 제어형 인공지능 앱(Autonomous Agent) 패러다임으로 넘어가기 위한 필수적 통로이다.
