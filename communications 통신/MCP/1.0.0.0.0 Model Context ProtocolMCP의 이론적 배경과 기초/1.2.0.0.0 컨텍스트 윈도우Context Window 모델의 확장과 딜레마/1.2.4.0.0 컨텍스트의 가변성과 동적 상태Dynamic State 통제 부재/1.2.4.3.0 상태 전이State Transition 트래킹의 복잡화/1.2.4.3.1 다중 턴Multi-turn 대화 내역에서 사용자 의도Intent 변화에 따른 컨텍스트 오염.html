1.2.4.3.1 다중 턴(Multi-turn) 대화 내역에서 사용자 의도(Intent) 변화에 따른 컨텍스트 오염

대규모 언어 모델(LLM)을 활용한 상호작용의 가장 강력한 무기 중 하나는 과거의 발화를 기억하여 대화를 이어가는 다중 턴(Multi-turn) 기능이다. 하지만 사용자의 의도(Intent)가 특정 대화의 궤적 안에서 급격하게 전환(Shift)될 때, 이 편리한 다중 턴 메모리 구조는 과거의 잔재가 현재의 추론을 집어삼키는 심각한 ’컨텍스트 오염(Context Contamination)’의 늪으로 전락하게 된다.

0.1 누적 컨텍스트 윈도우의 ‘고립된 관성력’ 현상

사용자는 대화 중 언제나 마음을 바꾸거나 새로운 주제로 도약할 권리를 가진다.
예를 들어, 사용자가 챗봇 에이전트와 “올해 최고의 로맨스 영화를 추천해 줘. 분위기는 아련한 느낌으로“라며 10턴에 걸쳐 감성적인 영화 추천 대화를 이어갔다고 가정하자. 이 과정에서 컨텍스트 윈도우 내부에는 ‘감성’, ‘로맨스’, ‘사랑’, ‘눈물’ 등 특정 도메인에 쏠린 토큰의 가중치(Attention Weight)가 두터운 퇴적암처럼 쌓이게 된다.

그런데 11번째 턴에서 사용자가 돌연 “아, 영화 이야기는 일단 접고. 내일 제출할 AI 반도체 시장 수익성 분석 보고서 초안 좀 짜봐“라고 의도(Intent)를 급격히 틀어버린다.
이상적인 시스템이라면 앞선 로맨스 영화의 대화 내역(Context) 패턴을 일거에 삭제하고 보고서 작성이라는 새로운 인지적 평면에서 추론을 시작해야 한다. 하지만 기존의 슬라이딩 윈도우(Sliding Window) 체계는 이전 10턴 간의 달콤한(?) 영화 추천 로그를 그대로 짊어진 채 11번째 답변 생성에 돌입한다.
결과적으로 모델은 잔여 어텐션(Residual Attention)의 강한 관성력을 이기지 못하고 “AI 반도체 시장의 수익성은 마치 가을밤의 아련한 첫사랑처럼 불안정하지만 매혹적인 상승 곡선을 그리고 있습니다“와 같은, 맥락이 오염된 기괴하고 광기 어린 혼종(Chimera) 답변을 도출해 내고 만다.

0.2 하드 리셋(Hard Reset)의 딜레마와 맥락 전환 판독 불가

시스템 개발자들은 이러한 오염을 방지하기 위해 “사용자가 주제를 바꾸면 컨텍스트를 지워라(Clear History)“라는 단순 무식한 버튼 하나를 UI에 얹어두곤 한다. 하지만 이는 근본적인 해결책이 아니다. 사용자가 맥락을 ‘완전히’ 바꾼 것인지, 아니면 ‘살짝 결부시켜’ 확장한 것인지를 판단하기 매우 어렵기 때문이다.

“영화 이야기는 접고, AI 반도체 수익 보고서를 쓰자. 단, 아까 그 로맨스 영화의 배급사가 투자한 반도체 기업 위주로“라고 질문한다면 어떨까?
이때는 사용자의 의도가 급변했음에도 불구하고, 앞선 대화에 존재했던 ’영화 배급사’의 정보와 엔티티(Entity) 데이터가 반드시 유지되어야 한다. 기존의 인프라는 현재 들어온 프롬프트 하나만 보고서는, 과거 컨텍스트를 ‘전부 폐기(Hard Reset)할지’ 아니면 ’전부 유지(Accumulate)할지’를 식별해 낼 수 있는 이진(Binary) 분류의 잣대가 결여되어 있다. 이로 인해 모델은 쓰레기 데이터 속에 유용한 다이아몬드가 파묻혀 있거나, 반대로 유용한 정보창고에 쓰레기 토큰이 범람하는 극단적인 컨텍스트 오염망의 노예가 된다.

0.3 Model Context Protocol(MCP)의 의도 기반 의미적 분리(Semantic Isolation) 체계

MCP (Model Context Protocol)는 인간처럼 맥락의 급변동을 포착하고 대화 내역의 가중치를 동적으로 격리, 결합시키는 ‘의미론적 세션 분리(Semantic Session Isolation)’ 능력을 시스템에 투여한다.

    A[사용자 신규 턴 발화] --> B{MCP Intent Detection Middleware}
    
    B -->|분석 1: 과거 대화와의 유사도 계측| C[Semantic Similarity Score]
    
    C -->|Score < 0.2: 완전한 의도 Shift| D{결정: 격리 Isolation}
    D --> E[기존 대화 Context 임시 동결 Freeze]
    D --> F[새로운 Workspace Context 할당 및 주입]
    
    C -->|Score > 0.7: 기존 주제의 연장| G{결정: 지속 Continuation}
    G --> H[기존 대화 Context 유지 및 Tool Call 연동 이어가기]
    
    C -->|Score 0.4: 하이브리드 의도| I{결정: 선택적 추출 Extraction}
    I --> J[기존 대화에서 핵심 Entity만 뽑아냄]
    J --> K[신규 Context에 Entity만 조립하여 주입]
    
    F & H & K --> L[오염 0%, 최적 가중치의 LLM 추론 달성]

MCP 환경 하의 LLM 에이전트는 사용자의 새로운 프롬프트가 들어올 때마다, 이를 맹목적으로 이전 꼬리에 이어 붙이지 않는다. MCP 라우터에 탑재된 의도 탐지 모듈(Intent Detection Module)이 이전 대화 로그와의 메타-거리(Meta-distance)를 즉각 계측한다.

만일 사용자의 의도가 완전히 다른 도메인으로 튀어버렸다고 감지(Score < 0.2)하면, MCP는 즉시 기존의 컨텍스트 블록(영화 추천 내역)에 열쇠를 채워 동결(Freeze)시켜 버린다. 이 정보는 삭제되지 않고 메모리에 남아있지만, 모델의 현재 어텐션 망에는 절대로 주입되지 않는다(격리, Isolation). 반면 하이브리드 한 의도가 감지되면 앞선 대화 블록에서 ’영화 배급사’라는 특정 엔티티 조각만을 발췌해 새 책상 위에 올려주는 치밀함을 발휘한다.
결과적으로 Model Context Protocol은 LLM의 뇌리에 차곡차곡 쌓이던 대화의 카르마(Karma)를 끊어내고, 사용자의 변덕스러운 의도 전이(Intent Shift)에도 불구하고 언제나 가장 정갈하고 무결점의 도화지(Clean State)에서 모델이 연산을 시작할 수 있도록 보위하는 완벽한 인지 방역 시스템의 역할을 해낸다.
