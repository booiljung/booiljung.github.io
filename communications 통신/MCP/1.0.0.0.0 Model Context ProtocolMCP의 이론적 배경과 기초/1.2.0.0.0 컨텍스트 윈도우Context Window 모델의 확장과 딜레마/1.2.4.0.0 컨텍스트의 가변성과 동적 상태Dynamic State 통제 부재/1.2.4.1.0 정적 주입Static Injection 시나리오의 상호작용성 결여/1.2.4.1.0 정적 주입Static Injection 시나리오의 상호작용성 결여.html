1.2.4.1 정적 주입(Static Injection) 시나리오의 상호작용성 결여

대규모 언어 모델(LLM)을 사내 정보망이나 외부 지식 데이터베이스에 연결할 때, 대다수의 초급 시스템 아키텍처는 이른바 ’정적 주입(Static Injection)’이라는 지극히 단순하고 평면적인 설계 패턴을 답습하게 된다. 이는 사용자의 질문이 발화되는 그 찰나의 순간(Point-in-time)에만 모든 검색을 종료하고, 그 결괏값을 프롬프트의 앞단에 포장(Packing)하여 모델에게 던져주는 방식이다. 하지만 이 견고해 보이는 밀어넣기식 아키텍처는 조금만 복잡한 추론망을 그려야 하는 자율형 과업(Autonomous Tasks)에 들어서면, 근본적인 ’상호작용성(Interactivity)의 결여’라는 심각한 기능적 동맥경화를 노출한다.

0.1 단일 패스(Single-pass) 검색과 맹목적 추론의 한계

정적 주입 시나리오는 파이프라인의 구조가 직렬적이고 파편화되어 있다는 치명적 단점을 안고 있다.
예를 들어, 사용자가 “회사의 A 프로젝트가 실패한 원인을 B 프로젝트의 재무 지표와 비교해서 설명해 줘“라고 질문했다고 가정해 보자. 기존 시스템은 즉각 벡터 DB(Vector DB)를 가동해 ‘A 프로젝트 실패’, ’B 프로젝트 재무’라는 키워드 위주로 의미망 검색(Semantic Search)을 실행한다. 이 1회성 서치(One-shot Search)를 통해 얻어진 상위 10개의 문서 청크(Chunk)가 모델의 컨텍스트 윈도우에 ‘정적으로’ 주입된다.

LLM은 이 주입된 데이터의 한계 안에서만 사고해야(Confined reasoning) 한다. 만약 LLM이 문서를 분석하던 도중 “A 프로젝트가 실패한 핵심 원인은 C 하청 업체의 부도 때문이구나! 그럼 C 하청 업체의 재무 기록을 추가로 봐야겠어“라고 강력한 2차 추론 가능성을 포착하더라도, 시스템은 여기서 멈춘다. 정적 주입 환경에서는 모델이 백엔드 DB에게 “C 하청 업체 데이터를 추가로 가져와 달라“고 역(Reverse)으로 요청할 수 있는 네트워크 통신 소켓 자체가 아예 닫혀 있기 때문이다. 결국 모델은 주어진 빈약한 1차 데이터만으로 ‘장님 코끼리 만지기’ 식의 결론을 지어내거나(Hallucination), “제공된 문서에는 관련 내용이 없습니다“라는 무책임한 방어벽 뒤로 숨어버린다.

0.2 초기 척후병(Scout) 부재가 초래하는 컨텍스트 윈도우의 낭비

이러한 상호작용성(Interactivity)의 극단적 부재는 역설적으로 ’과탐색(Over-retrieval)’과 API 과금을 유발하는 악순환의 고리가 된다.
시스템 설계자는 모델이 중간에 추가 자료를 요청할 수 없다는 설계적 결함을 보완하기 위해, 초기 검색 단계부터 “필요할지도 모르는” 모든 서류, 즉 A부터 Z까지의 관련된 문서를 무작위로 쓸어 담아 컨텍스트 윈도우 한도 끝까지 꽉꽉 채워 넣는 이른바 ’투망식 사냥’에 몰두하게 된다.

실시간 핑퐁(Ping-pong)을 통해 정답을 좁혀나가는 양방향의 ‘동적 주입’ 체계였다면, 수백 토큰의 스무고개만으로 정확한 진단에 닿을 수 있었을 문제를, 정적 주입형 코드는 소통의 부재를 거대한 토큰의 양(Volume)으로 때우려는 미개한 자본 집약적 아키텍처로 전락시키는 것이다. 모델이 지식을 깨우치는 런타임(Runtime) 궤적과 인프라의 데이터 서빙 궤적이 평행선을 그리며 전혀 교차하지 못하는 현상은 LLM 상용화의 가장 가슴 아픈 구조적 모순이다.

0.3 Model Context Protocol(MCP)의 대화형(Conversational) 자원 획득망

이 꽉 막힌 정적 주입의 동맥경화를 시원하게 파쇄하는 혁명이 바로 Model Context Protocol(MCP)이 제창하는 ‘도구 호출(Tool Calling)’ 및 ’자원(Resource) 조회’의 실시간 양방향 프로토콜 규격이다.

시퀀스 다이어그램에서 펼쳐지는 광경은 과거의 죽어있던 정적(Static) RAG 시스템과는 차원이 다르다. MCP 환경 하에서 대형 언어 모델은 스스로가 능동적인 수사관(Investigator)의 지위로 격상된다.
모델의 두뇌(Inference Engine)가 추론을 이어가다 막히는 부분이 생기면, 즉시 추론을 홀드(Hold)한 채로 MCP 서버에게 표준 지정된 포맷의 JSON-RPC 요청을 보낸다. MCP 미들웨어는 이 지시를 받아 물리적인 외부 세계의 DB나 API를 실시간으로 흔들어 깨우고, 그 결괏값만 도출해 내어 LLM의 컨텍스트 윈도우 꼬리 공간에 조심스럽게 봉합(Suture)해 준다. 주입(Injection)이 더 이상 개발자가 던져주는 1회성의 적재물이 아니라, 에이전트 스스로 끊임없이 자신의 지식 상태(State)를 보정(Correction)하며 앞으로 나아가는 고도의 상호작용적(Interactive) 유기체로 탈바꿈하는 순간이다.
