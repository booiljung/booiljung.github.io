1.3.1.4.3 비의도적 자원 과다 소모(Resource Exhaustion)를 방어하는 샌드박스 격리망의 필요성

대규모 언어 모델(LLM)이 주도하는 자율형 에이전트(Autonomous Agent)는 데이터를 악의적으로 파괴하지 않더라도 시스템의 근간을 비참하게 붕괴시킬 수 있다. 그것은 바로 통제받지 않은 무한한 긍정주의와 확률적 집착이 빚어내는 ‘비의도적 자원 과다 소모(Resource Exhaustion)’ 현상이다. 에이전트는 결코 지치지 않으며, 네트워크 레이턴시나 과금의 개념을 인지하지 못한다. 이 무한한 동력을 가진 기계가 외부 API 호출 루프에 갇히게 될 때, 인프라는 순식간에 Denial of Service(DoS) 상태로 추락하게 되며, 이를 방어하기 위한 군사적 수준의 샌드박스(Sandbox) 격리망이 현대 시스템 아키텍처의 필수 불가결한 요건으로 대두되었다.

0.1 무한 긍정과 강박적 재시도가 부르는 Rate Limit 철퇴

에이전트가 “사용자 수 증가 추이를 엑셀 파일로 리포팅 하라“는 지시를 받았다고 가정하자.
에이전트는 1차적으로 통계 서버 API를 호출하지만, 하필 그 시간에 백엔드 DB가 무거워져 ‘503 Service Unavailable’ 에러가 반환되었다. 인간 엔지니어라면 당연히 ’아, 서버가 터졌네. 10분쯤 뒤에 다시 해봐야지’하고 물러날 것이다.

하지만 언어 모델은 임무를 완수하고 싶은 강렬한 목적 함수에 지배당하고 있다. 에이전트는 즉각 “서버가 일시적인 오류를 뱉었다. 내 프롬프트 인자가 틀렸나?“라며 날짜 파라미터를 1일씩 바꿔가며 1초에 50번씩 API를 미친 듯이 때리기 시작한다. 이는 해커가 가하는 DDoS 공격 패턴과 완벽하게 동일하다.
결국 자사 혹은 타사의 외부 연동 서비스는 이 비정상적인 트래픽 폭주를 감지하고 즉각 차단(Rate Limit) 장치를 가동해 에이전트의 IP 자체를 블랙리스트에 등재해 버린다. 이는 에이전트의 악의가 아닌, 순수한 임무 달성의 강박과 무지(Ignorance)가 초래한 전형적인 비의도적 인프라 파괴 사고다.

0.2 비용 모방(Cost Mocking)과 토큰/스토리지 증발 사태

자원 소모의 재앙은 네트워크 트래픽에만 국한되지 않는다.
에이전트에게 “웹사이트를 스크래핑해 오라“는 권한을 개방해 주었더니, 에이전트가 위키백과의 ’메인 페이지’뿐만 아니라 연관된 모든 하이퍼링크 1,000개를 순차적으로 긁어와 컨텍스트 윈도우(Context Window) 안에 전부 쌓아버리려 하는 경우가 흔히 발생한다.

결과적으로 단 1번의 사용자 프롬프트가 수십만 토큰(Token)의 API 호출 비용을 단숨에 증발시켜 버리는 천문학적인 클라우드 과금 버그(Billing Bug)를 발생시킨다. 또한 에이전트가 로컬 파일 생성을 허가받은 경우, 임시 결과값을 저장한답시고 수 기가바이트(GB)의 무의미한 JSON 쓰레기 더미 파일들을 로컬 스토리지에 무한정 쏟아부어 디스크 풀(Disk Full) 장애를 야기하는 일도 비일비재하다.
기계의 시야(Context)에는 ’디스크 잔여 용량’이나 ’과금 영수증’이 입력되지 않기 때문에 벌어지는 비극이다.

0.3 Model Context Protocol(MCP)을 통한 격리된 샌드박스(Sandbox) 배관계의 구축

자원의 물리적 한계를 인지하지 못하는 에이전트의 폭주로부터 시스템의 지갑과 디스크를 방어하기 위해, Model Context Protocol(MCP)은 에이전트의 구동 환경과 실물 인프라(Host System) 사이의 연결 지점을 ‘샌드박스(Sandbox)’ 형태로 완전히 격리하는 방어 철학을 구현했다.

MCP의 설계 사상에서, 에이전트는 결코 어플리케이션이 구동되는 네이티브 호스트(Host)의 자원을 마음대로 끌어다 쓸 수 없다.
모든 파일 입출력 및 데이터 조회 도구(Tool)들은 MCP 라우터에 의해 샌드박스화(Sandboxing)되어 있어, 에이전트가 파일을 읽고 쓰려 해도 오직 런타임이 허가한 /tmp/mcp_agent_sandbox/ 와 같은 철저히 격리된 가상 공간 안에서만 조작이 허용된다.

그뿐만 아니라, MCP 서버는 에이전트의 도구 호출 빈도를 통제하는 강력한 스로틀링(Throttling) 및 할당량(Quota) 제어 기능을 탑재하고 있다. 에이전트가 초당 10회가 넘는 과도한 API 연쇄 호출을 시도하면, MCP 서버 단에서 자체적인 서킷 브레이커(Circuit Breaker)를 발동해 백엔드로의 요청을 끊어버리고, 클라이언트에게 "너무 많은 요청입니다. 10분간 대기하십시오"라는 상태 에러를 정제하여 반환한다.

이러한 MCP의 엄격한 자원 격리망은 에이전트를 모래알 같은 무한한 긍정의 폭주로부터 강제로 건져내어, 제한된 예산과 규격 내에서만 영리하게 춤추도록 강제하는 현대 시스템 인프라의 최고 보안관 역할을 수행하게 되었다.
