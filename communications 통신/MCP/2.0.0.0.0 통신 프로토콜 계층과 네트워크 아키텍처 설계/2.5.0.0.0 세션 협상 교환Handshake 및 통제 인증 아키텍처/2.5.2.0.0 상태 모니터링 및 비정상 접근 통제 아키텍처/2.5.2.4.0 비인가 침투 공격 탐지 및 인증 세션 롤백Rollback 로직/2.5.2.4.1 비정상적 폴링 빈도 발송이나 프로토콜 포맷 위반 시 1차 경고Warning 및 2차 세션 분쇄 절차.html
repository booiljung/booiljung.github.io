2.5.2.4.1. 비정상적 폴링 빈도 발송이나 프로토콜 포맷 위반 시 1차 경고(Warning) 및 2차 세션 분쇄 절차

악성 클라이언트는 정상적인 에이전트인 척 세션을 맺은 후, 서버의 자원을 말려 죽이기 위해 초당 수만 번의 tools/call을 날리거나(Rate Limit 우회 시도), JSON-RPC의 본문 크기를 1GB쯤으로 부풀리는 프로토콜 포맷 위반(Protocol Violation) 공격을 시도한다.

이에 대응하기 위해 라우팅 미들웨어(Middleware)는 엄격한 점진적 격퇴 프로시저를 가동한다.

  1. 쓰로틀링(Throttling) 및 1차 경고: 정상 범주를 약간 넘어서는 빈도의 요청이 들어오면 서버는 연결을 끊기 전, HTTP 429 Too Many Requests 상태 코드나 RateLimited JSON-RPC 예외와 함께 Retry-After 헤더를 반환하여 클라이언트에게 공격 중단을 지시하는 1차 경고(Warning)를 보낸다.
  2. 서킷 브레이커(Circuit Breaker) 가동: 경고를 무시하고 1초 이내에 다시 페이로드를 쏘거나, 의도적으로 파싱 불가능한 깨진 JSON을 발송(Parse Error)하는 경우 서버는 오작동하는 클라이언트로 낙인찍는다.
  3. 2차 세션 분쇄 절차: 즉시 소켓을 강제 차단(TCP RST)하며 해당 커넥션에 얽힌 세션 ID를 파기하고 메모리를 파쇄(2.5.2.2.3)한다. 동시에 방화벽(WAF) 단에 해당 트래픽이 출발한 IP 주소를 넘겨주어 네트워크 계층에서 접속을 원천 분쇄해버린다.
