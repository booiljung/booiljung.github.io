2.5.2.2. 컨텍스트 이력 파기 정책 및 캐시 무효화(Cache Invalidation) 제어망

언어 모델 에이전트는 프롬프트 컨텍스트를 유지하기 위해 막대한 양의 토큰 히스토리와 임시 상태(State) 데이터를 백엔드 서버의 인메모리(API 캐시 등)나 디스크 공간에 켜켜이 쌓아둔다. 정상적인 통신 환경에서 이 데이터는 ’기억의 연속성’을 담당하는 핵심 보물이다.

그러나 세션이 비정상적으로 끊어지거나, 클라이언트가 명시적인 해지(Termination)를 선언했을 때에도 이 거대한 데이터 덩어리를 고스란히 끌어안고 있다면, 서버는 필연적으로 메모리 고갈(OOM, Out of Memory) 사태를 맞이하게 된다. 나아가, 다른 에이전트 세션으로 파편화된 컨텍스트가 유출될 가능성마저 존재한다.

따라서 MCP는 생성의 연속성만큼이나, 파기의 단절성을 강조하는 파멸(Destruction) 메커니즘을 가동하여 컨텍스트 이력 파기 및 주기적 캐시 무효화 제어망을 구성한다. 본 챕터에서는 이를 구체적으로 파고든다.
