2.5.2.2.1. 호스트 소프트웨어 비정상 종료 시 수집된 임시 메모리 누수 방지용 가비지 소거 규정

클라이언트 애플리케이션(예: 터미널 에이전트)이 운영체제의 커널 레벨 패닉이나 강제 종료(예: SIGKILL)를 당하게 되면, “나는 접속을 끊습니다” 라는 예의 바른 TCP 종료 메세지를 보낼 틈조차 없이 증발해 버린다.

MCP 백엔드 서버의 세션 관리자는 이렇게 이탈해 버린 좀비 세션의 할당된 자원을 강제로 압류(Confiscate)하는 가비지 소거(Garbage Collection) 규정을 발동해야 한다.

  1. 타임아웃 감지에 이은 고아 자원(Orphaned Resources) 마킹: 앞선 2.5.2.1 절의 핑-퐁 감시망이 세션을 Dead로 판정하면, 가비지 컬렉터(Garbage Collector) 큐에 해당 세션 ID가 ’고아 자원’으로 즉각 등록된다.
  2. 미결 RPC 요청의 스태시 파쇄: 진행 중이던 비동기 도구 실행(tools/call) 결과물의 임시 버퍼 영역들을 물리적으로 지운다. 기다리는 주인이 사라진 약속된 반환 객체(Promise)들을 강제로 Reject 처리하여 묶여 있던 스레드를 해방시킨다.
  3. 리소스 구독자(Subscribers) 목록의 능동 이탈: 리소스 변경 시마다 notifications/resources/updated 알림을 받기로 한 구독 목록 체인에서 해당 죽은 세션의 콜백 주소(Callback Address)를 pop() 시켜버림으로써, 끊어진 소켓에 메세지를 끝없이 발사하는 무한 대기 오작동을 차단한다.
