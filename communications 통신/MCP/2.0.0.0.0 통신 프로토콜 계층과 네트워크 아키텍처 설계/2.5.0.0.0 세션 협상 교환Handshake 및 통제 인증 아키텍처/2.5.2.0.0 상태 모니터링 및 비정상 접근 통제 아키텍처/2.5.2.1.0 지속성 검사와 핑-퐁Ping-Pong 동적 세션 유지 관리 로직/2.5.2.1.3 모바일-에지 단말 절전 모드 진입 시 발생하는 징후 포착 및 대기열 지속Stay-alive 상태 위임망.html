2.5.2.1.3. 모바일/에지 단말 절전 모드 진입 시 발생하는 징후 포착 및 대기열 지속(Stay-alive) 상태 위임망

모바일 환경(예: iOS/Android 기반 에이전트 사용자)이나 전력 제어가 엄격한 노트북 환경에서는 OS 차원에서 애플리케이션을 백그라운드로 밀어 넣고 네트워크 소켓을 강제 동결(Sleep/Doze Mode)시켜 버린다.

이 경우 서버는 무의미한 ping을 날리며 데드라인을 소모하다가 세션을 파괴하게 되는데, 이는 사용자가 5분 뒤 앱을 다시 켰을 때 초기화된 컨텍스트에 분노하게 만드는 원인이 된다.

  1. 수면 징후 알림(Sleep Intent Notification): 최적화된 에이전트 클라이언트는 OS의 백그라운드 전환 이벤트를 인터셉트(Intercept)하자마자, 소켓이 물리적으로 얼어붙기 전 서버를 향해 notifications/system/sleep_intent 구조의 단방향 알림을 급파한다.
  2. 세션 동결 및 컨텍스트 위임(Stay-alive): 서버는 이 신호를 포착하면 즉각 ping 발사를 중단하고, 해당 세션 구조체를 디스크나 외부 캐시(Redis) 스토리지에 ’직렬화(Serialize)’하여 안전하게 대피시킨다.
  3. 무결점 재단장(Seamless Wake-up): 단말기가 깨어나 새로운 IP 환경에서 다시 연결을 시도하면, 클라이언트는 이전 세션 ID를 제시한다. 서버는 위임해 두었던 상태망을 인메모리로 다시 퍼 올리며, 마치 한숨 자고 일어난 것처럼 아무런 문맥 손실 없이 통신을 이어나가는 고차원의 상태 위임망이 완성된다.
