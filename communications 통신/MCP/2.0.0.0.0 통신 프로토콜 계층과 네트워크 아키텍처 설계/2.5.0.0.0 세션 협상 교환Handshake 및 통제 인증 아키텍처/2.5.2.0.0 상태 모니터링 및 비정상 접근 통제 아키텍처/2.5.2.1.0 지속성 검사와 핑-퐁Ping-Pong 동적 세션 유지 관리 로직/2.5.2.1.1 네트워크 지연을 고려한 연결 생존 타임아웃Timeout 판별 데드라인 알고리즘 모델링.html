2.5.2.1.1. 네트워크 지연을 고려한 연결 생존 타임아웃(Timeout) 판별 데드라인 알고리즘 모델링

단순히 30초마다 ping을 보내고 대답이 없으면 연결을 끊는 자비 없는 고정 타임아웃(Hard Timeout) 방식은 대륙 간 통신이나 불안정한 모바일망에서는 심각한 오작동(False Positive)을 낳는다. 클라이언트는 살아있지만 거대한 AI 추론 결과 페이로드에 네트워크 대역폭이 밀려 pong 응답이 지연될 뿐일 수 있기 때문이다.

이를 해결하기 위해 MCP 모니터링 데몬은 동적 데드라인(Dynamic Deadline) 알고리즘을 모델링한다.

  1. 지터(Jitter) 기반 기준선 추적: 서버는 클라이언트와의 최근 5회 통신 RTT(Round Trip Time)를 측정하여 네트워크 지연의 평균값과 분산을 계산한다.
  2. 소프트 및 하드 데드라인 분리: 평균 RTT + (분산 * 가중치)를 계산하여 1차 경계선인 Soft Deadline을 설정한다. 이 선을 넘기면 서버는 연결 해제가 아닌 “핑 재전송(재촉)” 모드에 돌입한다.
  3. 최종 사형 선고: 소프트 데드라인을 넘긴 후, 통상적인 네트워크 마비 시간(예: 15초)을 더한 절대 한계선인 Hard Deadline 구간마저 돌파한다면, 서버는 비로소 이 세션을 “회생 불가 상태(Dead)“로 판별하고 소켓 파이프라인의 물리적 절단 절차에 돌입한다.
