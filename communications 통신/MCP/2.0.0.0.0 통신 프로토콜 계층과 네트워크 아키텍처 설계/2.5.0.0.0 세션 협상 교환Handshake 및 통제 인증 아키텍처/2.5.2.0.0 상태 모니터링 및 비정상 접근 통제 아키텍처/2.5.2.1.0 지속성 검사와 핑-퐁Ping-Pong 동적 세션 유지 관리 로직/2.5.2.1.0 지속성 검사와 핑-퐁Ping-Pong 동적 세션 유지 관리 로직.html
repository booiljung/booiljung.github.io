2.5.2.1. 지속성 검사와 핑-퐁(Ping-Pong) 동적 세션 유지 관리 로직

네트워크 소켓(TCP/WebSocket/SSE)은 근본적으로 불안정하다. 클라이언트 쪽 랜선이 뽑히거나, 프로세스가 SIGKILL로 강제 종료될 경우, 운영체제의 TCP FIN/RST 패킷이 서버에 도달하지 않는 ‘하프 클로즈(Half-Closed)’ 상태의 유령 세션이 발생할 확률이 매우 높다.

MCP 백엔드 서버는 이러한 유령 세션들이 메모리와 파일 핸들을 갉아먹는 것을 막기 위해 애플리케이션 계층에서의 능동적인 지속성 검사(Keep-Alive/Heartbeat) 체계를 가동해야 한다.

JSON-RPC 스키마에 내장된 ping 메소드를 주기적으로 교환하며(Ping-Pong), 양측의 의식이 뚜렷하게 살아있는지를 검증하는 동적 세션 유지 관리의 알고리즘적 토대와 네트워크 지연 보정 전략을 이 절에서 서술한다.
