2.5.1. 연결 초기화 단계의 프로토콜 버전 및 능력값(Capabilities) 검증

전송 채널 소켓이 연결된 직후, 클라이언트 에이전트는 즉시 서버를 향해 자신을 알리는 initialize JSON-RPC 요청을 발사한다. 이것이 MCP 통신의 첫 번째 생명 반응이자 논리적 핸드셰이크(Handshake)의 시작점이다.

이 초기화 페이로드 내부에는 클라이언트가 지원하는 프로토콜 버전 명세와 그들이 활용하고자 하는 세부적인 기능 파편(Capability Flags)들이 탑재되어 있다. 서버는 이를 넘겨받아 자신의 버전 상태 머신 및 모듈 활성화 조건과 치열하게 대조하는 버전 합의 및 능력 검증을 수행한다. 이 검문 절차를 통해 최신 명세와 레거시 환경 간의 충돌을 방지하고 상호 호환되는(Interoperable) 가장 안정적인 통신 규격을 확정 짓는 핵심 과정을 본 절에서 논의한다.
