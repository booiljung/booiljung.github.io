2.5.1.4.1. 프로세스 중단 없이 서버 측 능력이 변경될 시 핑거프린트를 실시간 재전송하는 이벤트 통지 루프

로컬 파일럿 개발 환경에서는 개발자가 서버 측의 도구(Tool) 소스코드를 수정하고 저장(Save)하는 순간, 백엔드 프레임워크는 프로세스를 죽이지 않은 채 도구 모듈만 메모리상에서 스왑(Swap)한다. 이때, 서버의 진화된 능력은 클라이언트에게 실시간으로 통보되어야 한다.

0.1 ListChanged 알림 시스템

  1. 변동 이벤트 발사: 서버는 도구나 자원의 명세가 변경되었음을 감지하면, 즉각적으로 연결된 모든 에이전트에게 단방향 알림 패킷인 notifications/tools/list_changed (또는 resources/list_changed)를 쏜다.
  2. 에이전트의 재조회 트리거: 이 알림은 “내가 가진 도구가 바뀌었어“라는 단순한 신호(Ping)일 뿐, 무거운 전체 목록을 강제로 우겨넣지 않는다. 이 알림을 수신한 클라이언트는 자신이 한가해진 스레드 틈을 타 서버 측에 tools/list 요청을 새롭게 날려, 변경된 최신 도구의 핑거프린트 목록을 우아하게 동기화(Sync)한다.
  3. UI 백그라운드 갱신: 이를 통해 IDE는 사용자 코딩 창을 얼리거나 프로세스를 재시작하지 않고도, ’사용 가능 도구 목록 패널’에 새로운 도구 버튼을 소리 없이 렌더링(Silently Rendering)하며 실시간 확장성을 확보한다.
