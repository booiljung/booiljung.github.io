2.5.1.4. 무정지 핫 리로딩(Hot-Reloading) 상태 협상 갱신 프로토콜

전통적인 프로토콜 아키텍처에서는 서버 측의 능력이나 제공 가능한 도구 목록(Tools List)이 확장되거나 축소되면, 기존 연결을 무참히 끊어버리고 클라이언트가 다시 재접속(Re-connect)하며 새로운 핸드셰이크를 맺도록 강제했다. 하지만 컨텍스트를 길게 끌고 가는 LLM 에이전트의 특성상, 세션 단절은 곧 심각한 “기억 상실“을 의미한다.

MCP는 이러한 태생적 한계를 극복하기 위해, 기반 소켓이 살아서 펄떡이는 그 시점에도 절대 연결을 끊지 않고 양측의 계약 상태(협상 결과)만을 유연하게 바꿔치기하는 핫 리로딩(Hot-Reloading) 상태 협상 갱신망을 내장하였다. 본 절에서는 파일 변경 감지부터 동결 메커니즘에 이르는 무정지 런타임 갱신 프로토콜을 파헤친다.
