2.5.1.2.1. 접속 주체(IDE, CLI, Agent)의 이름(Name) 및 버전 명시화 검증 구조

클라이언트가 핸드셰이크의 포문을 여는 initialize 요청 스키마 내에는 clientInfo라는 필수(Required) 객체가 존재한다. 이 곳이 바로 클라이언트가 스스로의 정체성을 서버에 통보하는 명함과도 같은 위치다.

  1. name 및 version의 절대적 강제화: 통상적인 웹 통신의 User-Agent가 선택적이고 무의미한 길이의 문자열인 것과 달리, MCP에서는 클라이언트 애플리케이션의 고유 이름(예: claude-desktop, zed-editor)과 시맨틱 버전(예: 1.2.4) 명시를 매우 강력하게 강제한다. 만약 이 필드가 누락되어 있거나 형식이 어긋난 채로 진입하면, 서버는 예외를 던지고 즉각 소켓을 파괴(Abort)해 버린다.
  2. 벤더별 맞춤형(Custom) 워크플로우 분기: 서버는 이 명시적인 clientInfo를 파싱하여 능동적인 가지치기(Branching)를 수행한다. 예를 들어 접속한 에이전트가 name: cursor라면 코드 컨텍스트 주입 도구를 우선적으로 활성화하고, name: claude-desktop이라면 운영체제 제어 도구를 추가로 열어주는 식별 기반의 맞춤형 서비스 프로비저닝(Provisioning)이 이 검증 구조에 의해 달성된다.
