2.5.1.1.1. 최신 프로토콜 명세와 레거시 호환성을 동시 유지하기 위한 폴백(Fallback) 하위호환 규칙

클라이언트 에이전트가 첫 initialize 요청을 보낼 때, 페이로드에는 반드시 protocolVersion 속성이 명시되어야 한다 (예: 2024-11-05).

서버는 이 날짜 기반(Date-based)의 버전을 확인하고 자신이 지원하는 최신 명세와 대치시킨다. 만약 클라이언트가 서버 내장 엔진의 버전보다 오래된 레거시(Legacy) 명세를 요구한다면, 서버는 즉시 응답 구조를 강등(Downgrade)시키는 폴백(Fallback) 하위호환 규칙을 발동해야 한다.

  • 응답 스키마 다운그레이닝(Downgrading): 서버가 최신 v2 스키마(예: 다중 미디어 타입 지원)를 사용할 수 있더라도, 클라이언트의 버전을 고려해 구형 v1 스키마(순수 텍스트 기반 반환)로 결과값을 변환하여 내려준다.
  • 최신 필드의 묵음 처리: 레거시 클라이언트 JSON 파서가 알지 못하는 새로운 속성(Property)들(예: 최근 추가된 annotations 필드)은 아예 페이로드에서 탈락(Omit)시켜 전송한다. 이를 통해 구형 파서의 Strict schema validation 에러 발생 공간을 원천 소거한다.
