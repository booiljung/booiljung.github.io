2.5.1.1. 상호 호환성(Interoperability)을 보장하는 버전 합의 메커니즘(Negotiation)

에이전트 생태계가 팽창하면서 구형 버전의 IDE(예: 구형 Cursor 에디터)와 최신 버전의 백엔드 MCP 서버가 조우하는 파편화된 환경은 필연적으로 발생한다. 구형 클라이언트가 알지 못하는 새로운 스키마를 최신 서버가 강제로 밀어 넣는다면, JSON 파싱 에러(Parsing Error)와 함께 양쪽 모두 크래시(Crash)되는 파국을 맞이한다.

따라서 MCP는 양 끝단의 통신 모듈 간에 이질적인 버전 파편화를 봉합하고 공통의 통신 규격을 타결(Negotiate)하는 버전 합의 메커니즘을 핸드셰이크의 척추로 삼는다.

클라이언트가 제안한 버전을 서버가 어떻게 해석하고, 폴백(Fallback) 하위 호환을 적용할지, 그리고 도저히 합의 불가능할 때 어떻게 연결을 우아하게 끊어낼지(Reject)에 대한 런타임 폴리시(Policy) 역학이 여기서부터 전개된다.
