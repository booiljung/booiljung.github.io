2.5.1.1.3. 메이저/마이너 버전 불일치 시 연결을 차단(Reject)하거나 격리하는 폴리시(Policy) 모델

서버가 아무리 폴백(Fallback) 방어막을 쳐둔다 하더라도, 프로토콜 구조 자체가 완전히 뜯어고쳐진 파괴적(Breaking) 메이저 버전 차이 앞에서는 하위 호환 구조가 작동할 수 없다. (예: 클라이언트는 v2를 요청하는데, 서버는 철 지난 v1 전용 구식 파서만 돌리고 있는 경우)

이러한 돌이킬 수 없는 버전 불일치를 감지했을 때, 서버 엔진은 비정상적인 상태로 연결을 이어가 폭주하는 대신 차가운 거부 및 격리(Rejection & Isolation) 폴리시 모델을 전개한다.

  1. 명시적 Connection Rejected 예외 투척: 서버는 클라이언트의 소켓 연결을 단순히 끊어버리지 않고, 표준화된 JSON-RPC 예외(예: ProtocolVersionNotSupported)를 에러 페이로드로 발사하여 “너의 클라이언트를 최신 버전으로 업데이트하라“는 명확한 조치 가능 행동(Actionable Insight)을 UI에 띄우도록 유도한다.
  2. 기능 제한적(Degraded) 샌드박스 격리: 만약 치명적인 메이저 버전 불일치가 아닌, 마이너 속성의 충돌이라면 서버는 예외를 던지는 대신 클라이언트를 경고 샌드박스 모드로 격리시킨다. 도구 실행(Tool Execution) 같은 민감한 쓰기 액션은 막아버리고, 리소스 읽기(Read-only)라는 가장 기초적인 기능만 활성화시켜 최소한의 데이터 생존권만 허용하는 고도화된 폴리시다.
