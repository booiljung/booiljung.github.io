2.5. 세션 협상 교환(Handshake) 및 통제 인증 아키텍처

로컬 프로세스(Stdio) 혹은 원격 네트워크(SSE/HTTP)를 통해 물리적 혹은 전송 계층의 파이프라인이 성공적으로 연결되었다면, 클라이언트 에이전트와 백엔드 서버는 본격적인 데이터를 주고받기 전 서로의 정체성과 능력을 확인하는 치밀한 준비 의식을 치러야 한다. 이를 **세션 협상 교환(Handshake)**이라 칭한다.

이는 단순한 인사치레가 아니다. 서로가 구사하는 프로토콜의 버전이 맞는지, 서버가 제공할 수 있는 자원(Resource)과 도구(Tool)를 클라이언트가 소화할 수 있는지, 그리고 접속을 시도한 에이전트가 변조된 악성 프로그램이 아닌지를 판별하는 최초이자 최후의 검문소다. 본 장에서는 연결을 확정 짓는 버전 및 능력값(Capabilities) 검증 절차부터, 상태 모니터링을 통한 비정상 접근 통제망에 이르는 전방위적 런타임 인증 아키텍처를 세밀하게 분해한다.
