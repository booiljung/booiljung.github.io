2.1.2.4.1. 사용 가능한 주변 로컬 MCP 서버 목록을 런타임에 검색하여 자동 바인딩하는 DNS-SD 프로토콜 포팅

로컬 PC, 사내 워크스테이션, 혹은 가정용 와이파이(Wi-Fi) 공유기로 묶인 폐쇄망 환경에서 수많은 개별 MCP 에지 노드(로컬 파일 인덱서, 개인용 IoT 제어 서버 등)들이 구동될 때, 클라이언트 시스템이 이들의 IP 주소와 포트(Port) 포인터를 일일이 환경 설정 파일에 하드 리스트업(Hard-list-up)하는 것은 형편없는 사용자 경험(UX)을 유발한다.

이러한 로컬 인트라넷 환경에서의 제로 설정(Zero-Configuration) 생태계를 구축하기 위해 MCP는 DNS-SD(DNS Service Discovery, mDNS 기반) 프로토콜 기반의 브로드캐스트 포팅 기법을 전격 수용하였다.

0.1 로컬 서브넷 멀티캐스트(Multicast) 브로드캐스팅

로컬 백그라운드 환경에서 새로운 MCP 서버(예: mcp-local-db) 데몬이 런칭되면, 서버 노드는 중앙 관리자에게 소켓을 여는 대신 224.0.0.251:5353과 같은 멀티캐스트 IP 주소를 향해 UDP 패킷을 허공으로 방사(Broadcasting)한다.
이 패킷에는 “나의 서비스 명칭은 _mcp._tcp.local이며, 현재 192.168.1.15:8080 포트에서 JSON-RPC 요청을 받을 준비가 되어 있다“라는 메타데이터 텍스트 레코드(TXT Record)가 각인되어 있다.

0.2 클라이언트 측 런타임 탐지망과 자동 바인딩

에이전트 클라이언트 앱(호스트)이 기동됨과 동시에 백그라운드에서는 이 _mcp._tcp.local 시그니처를 지닌 멀티캐스트 패킷들을 항시 청취(Listening)하는 리스너(Listener) 스레드가 활성화된다.
클라이언트 스레드가 주변 서버의 메타데이터를 캐치하면, 어떠한 수동 구성(Manual Configuration)도 대입할 필요 없이 즉각 런타임 상에서 소켓을 열고 핸드셰이크 프로토콜을 수행하는 **자동 바인딩(Auto-Binding)**을 실현한다. 사용자는 그저 에이전트를 켜기만 해도, 에이전트가 “로컬 파일 제어 도구를 발견했습니다“라고 보고하는 마법 같은 동적 네트워크 접합(Splicing) 결합망을 완성하게 되는 것이다.
