2.1.2.4. 동적 디스커버리(Service Discovery) 알고리즘 적용망

Model Context Protocol(MCP)이 지향하는 거대한 분산 데이터 에지(Edge) 네트워크가 생명력을 얻기 위해서는 클라이언트가 연결할 백엔드 서버의 주소가 영구 불변하게 고정(Static Configuration)되어 있어서는 안 된다. 오늘 작동하던 내 로컬망의 데이터베이스 서버가 내일은 주소가 변경될 수 있으며, 퍼블릭 클라우드에 띄워둔 람다(Lambda) 서버는 트래픽 폭증에 따라 수 분 만에 백 개의 인스턴스로 분할 생성되었다가 사라지기를 반복한다.

이러한 고도의 가변적 인프라 환경에서 클라이언트 혹은 프록시 계층이 “어느 서버에 접속해야 도구(Tool)를 쓸 수 있는가?“라는 근원적인 물음을 런타임에 실시간으로 해결해 주는 아키텍처가 바로 동적 디스커버리(Dynamic Service Discovery) 알고리즘 적용망이다.

기존의 마이크로서비스 아키텍처(MSA)에서는 서비스 레지스트리(Service Registry)라는 중앙 명부(예: Eureka, Consul)를 두고 서버들이 자신의 생존을 기록하게 하는 방식을 주로 채택했다.
그러나 MCP는 이보다 한발 더 나아가, 이질적으로 구성된 양극단 환경(로컬 폐쇄망 및 클라우드 K8s 클러스터) 모두에 대응할 수 있도록 디스커버리 방식을 듀얼 프로토콜로 세분화하여 설계하였다.

이어지는 절에서는 복잡한 중앙 명부 없이도 같은 스위치 장비에 물린 주변 장치를 찾아내는 DNS 기반 서비스 검색(DNS-SD) 메커니즘, 그리고 웅장한 클라우드 오케스트레이터 기반에서 신규 엔드포인트를 실시간으로 동기화하는 기법, 나아가 붕괴된 비정상 노드를 재빠르게 끊어내는 자동 퇴출(Eviction) 알고리즘을 심층적으로 해부한다.
