2.2.1.3. 스트리밍(Streaming) 페이로드 기반 대용량 전송 프로토콜 확장

거대 언어 모델(LLM)이 텍스트를 한 글자씩 뱉어내며 생성하는 원리(Autoregressive Generation)와 마찬가지로, 현대의 에이전트 시스템이 다루어야 하는 외부 자원들 역시 그 덩치가 기하급수적으로 팽창하고 있다. 단 한 번의 호출로 500MB에 달하는 서버 로그 파일을 읽어오거나, 수백 개의 페이지로 이루어진 기업용 PDF 문서 데이터를 컨텍스트 안에 우겨넣어야 하는 상황이 빈번하게 발생한다.

이러한 대용량 페이로드(Payload)를 단일 JSON 문자열로 직렬화(Serialize)하여 메모리에 적재한 뒤 한 번에 전송하려고 시도하면, JSON을 파싱하는 V8 자바스크립트 엔진 메모리가 한도(Max Heap Size)를 초과하여 치명적인 Out-Of-Memory(OOM) 크래시를 발생시킨다.

Model Context Protocol(MCP)은 이러한 메모리 한계를 우회하고 초기 지연 시간(Time-To-First-Byte, TTFB)을 극적으로 단축시키기 위해 스트리밍(Streaming) 페이로드 전송이라는 핵심 확장 규격을 프로토콜 내재적으로 차용하였다.

단일 트랜잭션의 한계를 넘어서는 거대 파일과 이진 데이터가 어떻게 조각나고 조립되는지, 그리고 무수히 쪼개어진 청크(Chunk)들이 혼란스러운 네트워크 환경을 뚫고 지나가면서도 어떻게 그 앞뒤 순서를 완벽하게 보장받을 수 있는지 다음 절들을 통해 통신 시스템 메커니즘을 심도 있게 분석한다.
