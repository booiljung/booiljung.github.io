2.2.2.2. 파이프라이닝(Pipelining) 기반 대용량 병렬 쿼리 지원

거대 언어 모델(LLM)이 단일 사고 과정(Chain of Thought)을 수행할 때, 종종 독립적인 복수의 데이터를 동시에 요구하는 상황이 빈번히 발생한다. “로컬의 A 파일과 B 폴더의 내용, 그리고 내부망의 사내 위키 C 페이지를 모두 읽고 이를 기반으로 취약점을 분석하라“는 복합적인 명령이 주어졌을 때, 에이전트가 이를 직렬적(Sequential)으로 하나씩 요청하고 응답받기를 반복한다면 지연 시간(Latency)은 덧셈이 아닌 곱셈 단위로 치솟게 된다.

이러한 직렬 병목(Head-of-Line Blocking)을 타파하기 위해 MCP 전송 계층은 단일 물리적 커넥션 위에서 끊임없이 메시지를 겹쳐 밀어 넣을 수 있는 파이프라이닝(Pipelining) 기술을 채용했다.

이는 클라이언트가 첫 번째 패킷의 응답을 기다리지 않고도, 고유한 id가 부여된 수십 개의 독립된 요청을 하나의 네트워크 소다관(Pipe) 속에 연이어 쏟아붓는(Push) 구조적 허용을 의미한다.
이어지는 섹션들을 통해 병렬로 쏟아진 요청들을 어떻게 처리하고, 거대한 크기의 파일들은 어떻게 청크(Chunk) 단위로 쪼개어 실시간으로 스트리밍하는지, 그리고 이 역동적인 난장판 속에서 수신 측의 메모리가 터지지 않도록 지켜주는 방어벽(Backpressure)과, 흩어진 퍼즐들을 다시 모아 완성하는 데이터 합의 지점(Join-Point)의 세부 아키텍처를 심도 있게 해부한다.
