2.2. 전송 계층(Transport Layer) 기반 규격 모델링: JSON-RPC 2.0

Model Context Protocol(MCP)의 물리적 위상 기하학이 클라이언트와 분산된 서버 간의 추상적인 공간적 연결망을 규정했다면, 이 공간을 실제로 가로지르는 언어의 문법과 대화의 규칙을 정의하는 것은 전송 계층(Transport Layer) 모델링의 영역이다.

MCP는 이 핵심 메시징 레이어로 독자적인 이진(Binary) 프로토콜이나 무거운 SOAP, 전통적인 REST API를 변형하여 사용하지 않고, 범용적이고 극도로 경량화된 JSON-RPC 2.0 규격을 전격 채택하였다. 인간이 읽을 수 있는(Human-readable) JSON의 특성을 띠면서도, 완벽한 원격 프로시저 호출(RPC)의 비동기적 강점을 살린 이 표준은 LLM의 자율적 에이전트 환경에 유례없는 유연성을 부여한다.

JSON-RPC 2.0은 기본적으로 HTTP와 같은 특정 하위 프로토콜에 종속되지 않는 전송 독립적(Transport-agnostic)인 성향을 지닌다. 따라서 로컬 파일 디스크립터(Stdio)를 통과하든, 네트워크의 웹소켓(WebSocket) 훅을 거치든 동일한 메시지 구문 분석기(Parser) 논리를 재활용할 수 있게 만든다.

더욱이 ‘어떤 메소드(Method)를 호명할 것인가’, ‘어떤 인자(Params)를 넘길 것인가’, 그리고 ’호명한 요청에 대한 식별자(ID)는 무엇인가’를 완벽하게 분리한 이 규격의 철학은 여러 개의 서버 인스턴스와 동시다발적으로 교신해야 하는 에이전트의 다중-비동기 환경을 구현하는 가장 단단한 주춧돌이 된다.

본 장에서는 왜 수많은 스키마 통신법 중에서 하필 JSON-RPC 2.0 규격이 채택되었는지 그 당위성을 짚어보고, 이를 기반으로 파생된 상태 보전 파이프라인(Stateful Pipeline), 비동기 메시지 송수신 패턴, 그리고 통신 중 발생하는 예외 패킷들의 라우팅 복구력 최적화 전략을 심층적으로 살펴본다.
