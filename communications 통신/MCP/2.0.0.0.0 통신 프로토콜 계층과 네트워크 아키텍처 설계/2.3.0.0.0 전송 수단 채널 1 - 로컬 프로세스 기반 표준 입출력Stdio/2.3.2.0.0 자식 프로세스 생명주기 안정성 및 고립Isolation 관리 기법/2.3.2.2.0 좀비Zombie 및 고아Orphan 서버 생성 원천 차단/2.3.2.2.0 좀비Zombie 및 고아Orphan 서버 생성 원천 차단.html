2.3.2.2. 좀비(Zombie) 및 고아(Orphan) 서버 생성 원천 차단

로컬 컨텍스트를 다루는 에이전트 아키텍처에서 가장 두려운 시나리오는, 사용자는 이미 클라이언트 UI 창을 닫고 퇴근했건만 백그라운드의 무수한 MCP 서버 프로세스들은 그 사실을 모른 채 시스템 자원을 맹렬히 집어삼키는 기생충 무리로 전락하는 상황이다.

클라이언트 애플리케이션(부모 프로세스) 자체에 메모리 누수(OOM), 치명적인 세그멘테이션 폴트(Segmentation Fault) 혹은 사용자의 작업 관리자 강제 종료(Task Kill)와 같은 예기치 못한 크래시 현상이 발생해 돌연사할 경우, 자식 프로세스를 향해 평화로운 SIGTERM 신호를 보낼 수 있는 유예 시간 따위는 존재하지 않는다.

부모와의 대화 채널이 기습적으로 증발해버린 하위 서버들은 통제 불능의 **좀비(Zombie)**나 목적을 잃고 떠도는 고아(Orphan) 프로세스가 되어 개발 환경의 NVMe 디스크 I/O와 CPU 코어를 잠식하게 된다. 이어지는 챕터에서는 클라이언트가 갑작스럽게 사망하더라도 이 하위 인스턴스들이 어떻게 스스로 무덤을 파고 순장(殉葬)되는지, 호스트의 자원을 절대적으로 덜어내는 기계적 차단 논리를 다룬다.
