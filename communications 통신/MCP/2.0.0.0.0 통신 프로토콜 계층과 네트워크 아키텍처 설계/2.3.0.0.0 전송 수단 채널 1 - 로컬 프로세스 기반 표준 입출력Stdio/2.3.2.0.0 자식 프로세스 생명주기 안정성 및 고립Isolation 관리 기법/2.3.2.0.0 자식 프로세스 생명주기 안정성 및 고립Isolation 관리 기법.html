2.3.2. 자식 프로세스 생명주기 안정성 및 고립(Isolation) 관리 기법

Stdio 채널을 통해 에이전트와 서버를 단독 호스트(Single Host) 기계 내부의 부모-자식 트리 노드로 결합(Coupling)시키는 구조는, 네트워크 지연을 없애는 대신 **‘운영체제 레벨의 생명주기(Lifecycle) 동기화’**라는 극히 까다로운 엔지니어링 과제를 잉태한다.

클라이언트 앱(예: VSCode 확장)이 닫힐 때 하위에 매달린 무거운 코드 분석 서버 프로세스들도 같이 죽어줄 것인가, 아니면 백그라운드에서 메모리를 갉아먹는 고아 탈출범(Orphan Escapee)이 될 것인가? 외부 서버가 무한 루프에 빠져 CPU 100%를 독점할 때, 부모인 클라이언트 프로세스는 이를 어떻게 탐지하고 자식의 숨통을 정확하게 끊어낼(Terminate) 수 있을까?

원격 네트워크(HTTP/SSE) 채널에서는 단순히 통신 라우터의 소켓을 끊어버리면(Drop) 그만이지만, Stdio 모델 하에서는 클라이언트가 곧 서버를 육체적(Physical Memory)으로 통제하는 커널의 통치자(Governor)가 되어야 한다.
본 섹션에서는 Windows, macOS, Linux 간 파편화된 프로세스 시그널링 체계를 한 데로 감싸안고, 좀비(Zombie) 서버의 양산을 원천 차단하며, IPC 보안망을 통해 악의적인 커널 탈취 시도를 무력화하는 최고 수준의 생명주기 고립 및 안정성(Lifecycle Isolation & Stability) 기법들을 파헤친다.
