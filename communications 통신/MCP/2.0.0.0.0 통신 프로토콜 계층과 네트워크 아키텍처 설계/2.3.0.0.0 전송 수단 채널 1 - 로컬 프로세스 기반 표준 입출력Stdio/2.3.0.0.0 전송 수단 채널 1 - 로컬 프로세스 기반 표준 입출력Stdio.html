2.3. 전송 수단 채널 1: 로컬 프로세스 기반 표준 입출력(Stdio)

Model Context Protocol(MCP)이 제안하는 아키텍처는 클라이언트 에이전트와 서버가 반드시 물리적으로 분리된 원격 네트워크 상에 존재해야만 작동하는 것이 아니다. 오히려 개발자의 데스크톱이나 로컬 CI/CD 파이프라인과 같은 근거리 환경에서는, 복잡한 TCP/IP 네트워크 스택을 타는 것 자체가 막대한 자원 낭비이자 보안 취약점을 야기하는 불필요한 오버헤드(Overhead)로 작용한다.

이러한 로컬 최적화 요구를 만족시키기 위해 MCP는 전송 계층의 첫 번째 공식 채널로 로컬 프로세스 기반 표준 입출력(Standard Input/Output, Stdio) 통신 방식을 채택했다.
이는 클라이언트 호스트 애플리케이션(예: VSCode, 로컬 에이전트 터미널)이 MCP 서버를 독립적인 자식 프로세스(Child Process)로 직접 파생(Fork)시키고, 운영체제가 제공하는 가장 기초적이고 빠른 통신 매체인 Stdio 파이프(Pipe)를 통해 JSON-RPC 패킷을 직거래하는 아키텍처이다.

본 장에서는 이 Stdio 채널이 어떻게 네트워크 계층을 완전히 생략하면서 제로 지연(Zero-latency)에 가까운 초고속 컨텍스트 교환을 이뤄내는지, 그리고 프로세스 격리(Isolation)와 샌드박싱(Sandboxing)을 결합하여 로컬 OS의 자원을 어떻게 안전하게 통제하는지 그 기저 설계 원리를 심층적으로 해부한다.
