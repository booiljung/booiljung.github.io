이 수십 개의 프로세스들이 과거의 영세한 RPC 프레임워크처럼 전부 TCP 소켓을 점유하는 빙식으로 구동된다면, 엔지니어는 시스템 전역에서 발생하는 유니크한 할당 포트 번호(5000, 5001, 5002…) 충돌 전쟁(Port Collision)에 휩싸이게 된다. OS가 가용할 포트는 제한되어 있으며, 무분별하게 떨어지는 REST API 단기 클라이언트 연결(Time-Wait 찌꺼기 스레드)들은 금세 타임아웃을 유발하는 포트 고갈(Port Exhaustion) 사태를 몰고 온다.

0.1 유니버셜 FD(File Descriptor) 라우팅 체계

Stdio 전송망은 OS 커널이 제공하는 범용 포인터인 파일 디스크립터(File Descriptor) 매핑으로 이 재앙을 극복한다.
클라이언트 메인 프로세스는 백그라운드 서버 A를 띄울 때 fd 4, 5를, 서버 B를 띄울 때 fd 6, 7을 할당받는 식으로, OS 커널 내부적인 테이블 매핑만으로 수만 개의 자식 프로세스 라우팅 세션을 철저히 관리한다.

네트워크 포트라는 극도로 귀중한 전역 자원에 일절 손대지 않고도, 클라이언트 프로세스 고유의 FD 테이블 공간 안에서 무한에 가까운 서버 인스턴스를 격리하고 다중 연결할 수 있는 것이다. 이러한 커널-레벨의 직접 연결망 덕분에, LLM 에이전트는 마치 운영체제가 무한대의 플러그인 동적 링크 라이브러리(DLL/SO)를 자유자재로 로드하고 내리듯이, 단 한 번의 포트 충돌 공포 없이 무수한 외부 도구 세계의 빗장을 열어젖힐 수 있게 된다.
