2.3.1.2.1. 로컬 저장소 메모리 맵핑(mmap) 캐싱 활용 기반 거대 코드베이스 컨텍스트 초고속 전송

에이전트 클라이언트가 로컬 프로그래밍 환경에 투입될 경우, “이 100만 줄짜리 모노레포(Monorepo) 전체를 스캔하여 의존성 그래픽을 그려라“와 같은 폭력적인 도구 호출이 수시로 발생한다. 이때 서버 프로세스가 파일의 모든 내용을 읽어 Stdio 파이프로 욱여넣고 클라이언트가 이를 다시 파싱하여 들고 있는 것은 엄청난 데이터 이중 복사(Double-Copy) 낭비를 초래한다.

이러한 로컬 컨텍스트의 병목을 우회하기 위해, MCP Stdio 전송망은 동일한 호스트 OS 환경이라는 이점을 극대화한 운영체제 메모리 맵핑(mmap) 캐싱 기법을 창의적으로 결합할 수 있는 토대를 제공한다.

0.1 파일 시스템과 IPC의 역할 분리

클라이언트와 сервера가 단일 호스트를 물리적으로 공유하고 있다면, 서버는 거대한 응답 데이터 본문 전체를 파이프에 밀어 넣는 우를 범하지 않는다.
대신 서버는 1GB의 프로젝트 AST(Abstract Syntax Tree) 덤프 결과를 호스트의 로컬 NVMe 캐시 디렉터리(~/.mcp/cache/) 특정 임시 파일에 고속으로 기록(mmap)해 둔다.
그리고 JSON-RPC 통신 파이프라인의 result 페이로드에는 { "data_pointer": "file:///home/user/.mcp/cache/ast_req_99.bin" } 이라는 가느다란 추상화 포인터(Location URI) 텍스트 1줄만을 실어서 에이전트에게 전송한다.

에이전트 클라이언트 역시 동일한 파일 시스템에 결속되어 있으므로(Zero-Tier Host OS), Stdio 파이프로 5ms 만에 이 포인터를 수신한 뒤, 커널의 메모리 맵핑을 통해 해당 위치의 이진 파일을 자신 측 메모리로 곧장 제로 카피(Zero-Copy) 로딩해 버린다.
이 기법은 네트워크 대역폭 전송이라는 족쇄에서 벗어나, 상위 계층의 어플리케이션이 사실상 물리적 버스(BUS) 속도로 초거대 코드베이스를 교환하게 만드는 강력한 통신 패턴이다.
