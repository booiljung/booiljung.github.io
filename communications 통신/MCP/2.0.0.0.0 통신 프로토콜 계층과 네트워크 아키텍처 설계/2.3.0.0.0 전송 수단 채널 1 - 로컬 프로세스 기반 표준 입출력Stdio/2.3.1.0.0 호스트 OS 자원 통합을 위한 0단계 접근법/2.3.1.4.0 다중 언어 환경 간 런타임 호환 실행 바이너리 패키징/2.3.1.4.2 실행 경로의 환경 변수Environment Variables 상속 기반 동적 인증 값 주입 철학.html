2.3.1.4.2. 실행 경로의 환경 변수(Environment Variables) 상속 기반 동적 인증 값 주입 철학

로컬 프로세스로 서버를 띄울 때 가장 민감한 문제는 GitHub PAT(Personal Access Token), 특정 사내망 VPN 접속 키와 같은 시크릿 인증(Secret Authentication) 값들을 자식 프로세스에 어떻게 전달할 것인가이다.
이러한 민감 정보를 Stdio 파이프라인을 타고 넘어가는 평문 JSON-RPC 초기화(initialize) 메시지에 담아 보내는 것은 프로토콜 차원에서는 편리할지 모르나 보안 감사를 통과하기 어려운 위험한 안티 패턴(Anti-pattern)이다.

0.1 OS 수준의 투명한 시크릿 주입(Transparent Secret Injection)

서버 데몬이 안전하게 민감 키를 획득하도록, 클라이언트 에이전트는 OS의 환경 변수 상속(Environment Variable Inheritance) 메커니즘을 적극적으로 활용(Leverage)한다.

자식 프로세스는 태생적으로 부모 프로세스의 환경 공간을 복사하여 상속받는다. 따라서 클라이언트 애플리케이션(예: 에이전트 CLI)은 자식 통신망을 개설하는 spawn() 시스템 콜을 호출하기 직전에, env 딕셔너리에 사용자가 보관하고 있는 시크릿 키(예: export GITHUB_TOKEN=abc...)를 세팅하여 넘겨준다.

이를 통해 MCP 서버 개발자는 로컬 파일 시스템의 .env 파일을 찾거나 클라이언트의 JSON 메시지를 뒤적거릴 필요 없이, 단지 언어 스펙에 내장된 process.env.GITHUB_TOKEN (Node.js) 혹은 os.environ.get('GITHUB_TOKEN') (Python) 명령어 단 하나로 즉각 인증 컨텍스트를 확보하게 된다. 이는 자격 증명(Credential) 처리를 프로토콜의 영역에서 커널과 OS의 영역으로 완전히 분리해 내어, 클라이언트의 메모리와 서버의 메모리 사이에서만 비밀 텍스트가 조용히 교환되도록 만드는 강력한 동적 주입(Dynamic Injection) 철학이다.
