2.3.1. 호스트 OS 자원 통합을 위한 0단계 접근법

에이전트가 수행하는 가장 원초적이고 핵심적인 임무 중 하나는, 현재 작업 중인 컴퓨터 호스트(Host)의 내밀한 환경을 정밀하게 탐색하고 제어하는 것이다. 로컬 파일 시스템을 읽고, Git 브랜치를 조작하며, SQLite 데이터베이스를 직접 쿼리하는 등의 로컬 리소스 접근은 보안과 속도 측면에서 원격 API를 호출하는 것과는 궤를 달리한다.

이러한 **호스트 OS 자원 통합(Host OS Resource Integration)**의 본질적 요구를 해결하기 위해 MCP는 Stdio 채널을 이른바 **‘0단계 접근법(Zero-Tier Approach)’**으로 명명하고 최우선 권장 규격으로 제시한다.

’0단계’라는 명칭이 시사하듯, 이는 중간 에이전트나 HTTP 게이트웨이, 가상화된 네트워크 어댑터 등 어떠한 미들웨어(Middleware)도 개입시키지 않고, 커널(Kernel)이 프로세스 간(Inter-Process)에 직접 터널을 뚫어주는 가장 날도의(Raw) 전송 방식이다. 클라이언트는 자신이 구동 중인 데스크톱 환경과 동일한 권한 컨텍스트(Context)를 MCP 서버 프로세스에 위임할 수 있으며, 이로써 에이전트는 운영체제의 권한 경계 내에서 마치 자신이 사용자인 것처럼 네이티브(Native) 자원들을 마음껏 주물럭거릴 수 있는 강력한 통제권을 확보하게 된다.
