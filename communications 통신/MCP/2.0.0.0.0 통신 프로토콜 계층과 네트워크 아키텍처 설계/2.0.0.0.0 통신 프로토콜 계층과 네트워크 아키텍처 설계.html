Chapter 2. 통신 프로토콜 계층과 네트워크 아키텍처 설계

Model Context Protocol(MCP)은 대규모 언어 모델(LLM)과 파편화된 외부 데이터 소스 간의 간극을 봉합하기 위해 탄생하였다. 그러나 이 거대한 목표를 달성하기 위해 단지 ‘데이터 교환 규격(Format)’ 하나만을 정의하는 것으로는 불충분했다. 보안이 극도로 엄격한 로컬 샌드박스부터 지연 시간(Latency)이 치명적인 클라우드 원격 서버 환경에 이르기까지, 모든 다형성(Polymorphism)을 단일 시스템으로 포용하려면 가장 밑바닥의 통신 프로토콜 계층과 네트워크 아키텍처의 근본적인 재설계가 필수불가결하였다.

본 장(Chapter 2)에서는 MCP가 그 기초부터 어떻게 설계되어 네트워크의 복잡성을 통제하고 있는지, 그리고 그 이면에 깔린 시스템 아키텍처의 철학적, 기술적 기반을 심도 있게 파헤친다. 단순히 “데이터를 주고받는다“는 행위를 넘어, “어떻게 안전하고(Secure), 어떤 계층을 거치며(Layered), 어떻게 결합도를 낮출 것인가(Decoupled)“에 대한 공학적 해답을 제시한다.

구체적으로 본 장은 다음과 같은 세부적인 네트워크 위상(Topology)과 프로토콜 설계 기조를 심층적으로 전개한다.

  1. 초연결 시대의 3계층 아키텍처: 과거의 단순한 2계층(Client-Server) 모델을 부수고, 호스트(Host), 클라이언트(Client), 그리고 서버(Server)라는 정교한 3계층 분리 모델을 통해 책임과 권한을 어떻게 분할했는지 살펴본다.
  2. 프로토콜 계층의 표준 스펙, JSON-RPC 2.0: 대용량 파일 스트리밍부터 무거운 장기 실행 백그라운드 작업, 역방향 알림 푸시(Push Notification)에 이르기까지, 상태 보전(Stateful) 기반의 양방향 통신을 구현하기 위한 내부 메시징 표준 체계를 분석한다.
  3. 다중 전송 채널망의 추상화 구현: 인프라 조건에 따라 로컬 환경의 보안성을 극대화하는 표준 입출력(Stdio) 통신망과 거리적 한계를 없애는 **원격 통신망(SSE 및 HTTP)**이 어떻게 하나의 코어로 제어 및 투명 분리(Transparent Routing) 되는지 그 실체를 해부한다.
  4. 세션 제어 및 보안 검증 통로: 클라이언트와 서버가 만나 능력을 협상(Handshake)하고, 서로를 검증(Authentication)하며, 인가되지 않은 비정상 접근을 통제하는 인증 아키텍처(Identity Architecture)를 규명한다.

결과적으로, MCP는 단순한 API 스펙 문서가 아니라 전송 계층(Transport), 세션 계층(Session), 그리고 응용 계층(Application) 전체를 관통하는 차세대 분산 시스템(Distributed System)의 설계도라 할 수 있다. 이제 그 기저를 이루는 위상 기하학적 토폴로지(Topology)의 세계로 파고들어 보자.
