2.4.2.1. 비대칭적 양방향 이벤트 라우터 설계 모델

Stdio 파이프가 동일한 파일 디스크립터(FD)를 통해 읽기와 쓰기를 동시에 수행하는 대칭적 모델이라면, HTTP/SSE 결합 모델은 통로의 형태와 규격이 완전히 다른 비대칭적(Asymmetric) 모델이다.
클라이언트는 메시지를 보낼 때는 단발성 HTTP Request/Response 트랜잭션(POST)을 수행하고, 서버로부터 이벤트를 받을 때는 종료되지 않는 길쭉한 스트림(GET)을 감시해야 한다.

이 비대칭 구조의 가장 큰 아키텍처적 난제는 “내가 방금 보낸 POST 요청에 대한 서버의 결과 응답 파일이 타겟팅된 POST 응답 바디(Body)로 돌아올 것인가, 아니면 길게 열려있는 SSE 스트림을 통해 비동기 이벤트로 돌아올 것인가?“를 철저히 분류하고 라우팅하는 것이다. 이를 해결하기 위해 백엔드는 독립적인 엔드포인트와 세션 바인딩(Session Binding) 절차를 융합하는 정교한 이벤트 라우터를 가동해야만 한다.
