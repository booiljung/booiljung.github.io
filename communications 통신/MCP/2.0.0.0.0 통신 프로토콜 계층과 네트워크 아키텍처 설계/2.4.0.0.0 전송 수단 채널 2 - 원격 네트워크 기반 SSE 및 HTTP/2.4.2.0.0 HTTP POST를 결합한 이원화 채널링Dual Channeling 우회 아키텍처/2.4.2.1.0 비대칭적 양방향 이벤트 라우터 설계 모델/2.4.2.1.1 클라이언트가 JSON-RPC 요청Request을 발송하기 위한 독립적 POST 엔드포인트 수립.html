2.4.2.1.1. 클라이언트가 JSON-RPC 요청(Request)을 발송하기 위한 독립적 POST 엔드포인트 수립

이원화 채널 아키텍처의 첫 번째 단계는 클라이언트가 능동적 액션을 취할 수 있는 진입점, 즉 상향(Upstream) 트래픽 전용의 독립적인 HTTP POST 엔드포인트를 여는 것이다.

클라이언트 에이전트가 “데이터베이스 조회 도구를 실행하라(CallTool)“는 명령을 내리려 할 때, 이 명령 페이로드는 스트림(SSE) 창구(/mcp/events)로 들어설 수 없다. 대신 사전에 약속된 전용 URL, 예를 들어 /mcp/messages 엔드포인트를 향해 application/json 콘텐츠 타입으로 포장된 단발성 HTTP POST 요청을 쏘아야 한다.

0.1 무상태(Stateless) 명령어 전달의 장점

이 POST 엔드포인트는 원칙적으로 HTTP의 가장 순수한 기능인 무상태성(Statelessness)을 따른다.
클라이언트는 TCP 소켓을 오랫동안 붙잡고 있을 필요 없이, 명령을 던지고(POST) ‘202 Accepted’ 서명만 받으면 해당 HTTP 연결을 쿨하게 닫아버려도 무방하다. 덕분에 서버 입장에서는 동시 접속자가 10만 명으로 폭증하더라도 10만 개의 양방향 소켓을 열어두는 대신, 오직 그 순간 명령을 쏘고 있는 수백 개의 활성 POST 요청만 RAM에서 신속히 처리(Parse & Push to Worker)하고 내려놓는 극도의 메모리 효율을 달성할 수 있다.
