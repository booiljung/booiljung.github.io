2.4.2. HTTP POST를 결합한 이원화 채널링(Dual Channeling) 우회 아키텍처

단방향 하향 스트림인 SSE(Server-Sent Events)만으로는 클라이언트 에이전트가 원격 서버에게 능동적인 명령(도구 실행, 리소스 요청 등)을 내릴 수 없다. 양방향 프로토콜의 완전성을 성취하기 위해, MCP 원격 채널은 클라이언트에서 서버로 올라가는 상향(Upstream) 트래픽을 전담할 별도의 고속도로를 설계해야 한다.

이러한 상향 명령의 전달 창구로 표준 HTTP POST 방식이 결합된다. 결국 원격 MCP 아키텍처는 하향망(SSE)과 상향망(POST)이라는 완전히 분리된 두 개의 물리적 파이프를 논리적으로 하나처럼 맞물리게 하는 이원화 채널링(Dual Channeling) 모델로 완성된다. 본 장에서는 비대칭적 양방향 라우터의 세션 동기화 원리부터 서버리스(Serverless) 인프라와의 연계성, 그리고 CORS에 이르는 방어 설계망을 세밀하게 해체한다.
