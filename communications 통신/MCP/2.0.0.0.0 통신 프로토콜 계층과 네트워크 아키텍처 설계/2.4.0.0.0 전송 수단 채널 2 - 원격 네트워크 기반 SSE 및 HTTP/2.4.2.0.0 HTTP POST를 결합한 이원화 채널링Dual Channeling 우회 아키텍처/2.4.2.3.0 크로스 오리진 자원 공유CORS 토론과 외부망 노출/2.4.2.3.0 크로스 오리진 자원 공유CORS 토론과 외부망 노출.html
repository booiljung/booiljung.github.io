2.4.2.3. 크로스 오리진 자원 공유(CORS) 토론과 외부망 노출

현대 인터넷 생태계에서 백엔드 서버를 외부로 개방(Public Exposure)한다는 것은 곧 전 세계의 모든 웹 브라우저와 익명 악의적 스크립트의 공격 표면(Attack Surface)이 됨을 의미한다. MCP 프로토콜이 웹 기반(Web-based) 에이전트를 지원하기 시작하면서, 필연적으로 서로 다른 도메인 간의 자원 호출을 규제하는 브라우저의 강력한 보안 정책인 크로스 오리진 자원 공유(Cross-Origin Resource Sharing, CORS) 장벽과 정면으로 충돌하게 되었다.

본 절에서는 MCP 백엔드가 안전하게 브라우저 에이전트를 수용하기 위한 화이트리스트(Whitelist) 기반의 오리진(Origin) 검증 룰부터, 프리플라이트(Preflight) 핑퐁으로 낭비되는 인프라 지연을 깎아내는 최적화 기법, 그리고 서드파티 도메인(Third-party) 통신 간 Auth 토큰 탈취를 격멸하는 SameSite 쿠키 보안망을 집중적으로 조명한다.
