2.4.2.4.2. 세션 동기화 오작동을 능동 파악하여 타임아웃 만료 클라이언트를 쳐내는 라이프사이클 데몬

순수 폴링(Polling) 기반의 폴백(Fallback) 라우팅을 가동할 때 백엔드 서버가 짊어지게 되는 가장 큰 위협은 바로 좀비 세션(Zombie Session)의 누적이다.
SSE 환경에서는 클라이언트 네트워크가 끊어지면 소켓 에러가 터지므로 즉시 해당 세션을 폐기할 수 있다. 하지만 폴링 환경에서는 클라이언트가 잠시 쉬고 있는 것인지, 앱이 강제 종료되어 영영 돌아오지 못하는 것인지 판별할 물리적인 소켓 신호망이 없다.

이 상태로 돌아오지 않는 클라이언트를 위해 서버가 무한정 자원 갱신 이벤트를 큐(Queue)에 쌓아 둔다면, OOM(Out of Memory) 크래시는 시간문제다.

0.1 데드 세션(Dead Session) 축출을 위한 라이프사이클 데몬

이 파국을 막기 위해 서버 백엔드에는 1분에 한 번씩 메모리를 빗자루질하는 잔혹한 **라이프사이클 데몬스레드(Lifecycle Daemon Thread)**가 배회한다.

  1. 최종 접근 스탬핑(Last-Access Stamping): 폴백 환경의 클라이언트가 GET 폴링을 한 번 찔러 넣을 때마다, 서버는 해당 세션 객체의 메타데이터에 초 단위 정밀도의 타임스탬프(last_seen_at)를 갱신해 둔다.
  2. 유예 기간(Grace Period) 초과 검열: 백그라운드 리퍼(Reaper) 데몬은 주기적으로 전체 활성 세션 맵을 스캔(Scan)한다. 만약 어떤 세션의 last_seen_at 지표가 설정된 임계치(예: 3분 혹은 5분)를 초과하도록 멈춰있다면, 시스템은 이를 “세션 동기화 오작동에 의한 영구 단절“로 능동 판결(Conviction) 내린다.
  3. 무자비한 자원 강제 해제(Eviction): 선고 즉시, 해당 세션에 매달려 있던 모든 임시 큐, 버퍼, 그리고 백그라운드 연산을 강제로 소거(Destroy)하여 메모리 누수를 원천 절단한다. 이 데몬의 존재 없이는 스케일 아웃된 폴백 폴링 서버 클러스터는 한 시간도 버티지 못하고 무너지고 만다.
