2.4.2.4. 양방향 RESTful 폴백(Fallback) 라우팅 프로시저

이원화 채널링(SSE + HTTP POST) 아키텍처는 대부분의 현대적 클라우드 인프라에서 완벽히 구동되지만, 기업망 내부의 지독한 레거시 방화벽(L7 DPI)이나 HTTP/1.1 스트리밍의 응답 헤더조차 통과시키지 않는 극단적인 클라우드 벤더 제약을 맞닥뜨리는 순간 속절없이 무너진다. SSE 파이프 자체가 시작조차 되지 않거나(Timeout), 연결되자마자 강제로 닫혀버리는(RST 패킷) 것이다.

단 하나의 에이전트라도 연결 끊김으로 방치할 수 없는 견고한 엔터프라이즈 MCP 런타임은 가장 원시적인 형태, 즉 단발성 HTTP Request/Response 트랜잭션만을 활용하는 양방향 RESTful 폴백(Fallback) 채널로 생존 투쟁을 벌여야 한다. 본 절에서는 SSE 포기 선언점부터, 순수 폴링(Polling) 우회 전략과 가변적 적응형 주기를 통제하는 폴백 라우팅의 전면전을 기술한다.
