2.4.1. 서버 센트 이벤트(Server-Sent Events) 단방향 채널 특성 파악

네트워크를 가로지르는 통신에서 서버가 클라이언트에게 비동기적으로(Asynchronously) 메시지를 밀어 넣기(Push) 위해 개발된 SSE(Server-Sent Events)는, 클라이언트가 먼저 요청(Request)을 보내야만 응답(Response)을 받을 수 있는 전통적 HTTP 트랜잭션의 장벽을 우아하게 허문다.

SSE 채널은 본질적으로 오직 서버에서 클라이언트로 흘러가는 단방향(Unidirectional) 하향 스트림이다. 클라이언트가 특정 엔드포인트(예: /mcp/events)로 최초의 GET 요청을 보내면, 서버는 응답을 즉시 종료하는 대신 TCP 연결을 살려둔 채 text/event-stream 포맷으로 끝없는 스트리밍을 시작한다. 서버는 이 영구적 파이프를 통해 시스템 알림, 자원 상태의 변경, 또는 장기 실행 작업의 진행 상태 퍼센티지(Progress)를 페이로드에 담아 클라이언트로 무한정 투척할 수 있게 된다.

이러한 설계 방식이 왜 복잡한 Webhook이나 양방향 WebSocket보다 엔터프라이즈 환경에서 더 견고한 MCP 이벤트 수신 채널 구축망으로 선호되는지 다음 절에서 그 기술적 특성을 조명한다.
