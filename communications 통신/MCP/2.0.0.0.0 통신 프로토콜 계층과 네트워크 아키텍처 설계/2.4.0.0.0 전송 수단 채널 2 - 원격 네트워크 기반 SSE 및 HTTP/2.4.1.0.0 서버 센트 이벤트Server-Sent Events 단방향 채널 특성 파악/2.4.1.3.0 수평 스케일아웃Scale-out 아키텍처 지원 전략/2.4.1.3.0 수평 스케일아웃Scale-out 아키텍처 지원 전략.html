2.4.1.3. 수평 스케일아웃(Scale-out) 아키텍처 지원 전략

1대의 로컬 머신에서 구동되던 서버 애플리케이션이 클라우드(Cloud) 환경으로 진출하여 수천, 수만 대의 클라이언트 에이전트와 맞닥뜨리는 순간, 단일 프로세스 아키텍처(Scale-up)는 필연적으로 한계에 봉착한다. 특히나 연결을 맺고 끊기를 반복하는 REST API와 달리, 장기 지속 연결(Long-lived Connection)을 전제로 하는 SSE 이벤트 스트림 모델은 서버의 소켓과 메모리를 점유하며 거대한 트래픽 부하를 누적시킨다.

따라서 글로벌 스케일의 MCP 원격 네트워크 채널은 다수의 서버 인스턴스(Node)를 병렬로 배치하는 수평 스케일아웃(Scale-out) 아키텍처를 전격적으로 수용해야만 한다. 본 절에서는 L4/L7 로드 밸런싱 환경에서 SSE 세션을 끈질기게 유지하는 기법부터, 클러스터링 기반 다중 커넥션 현상을 돌파하는 브로드캐스팅과 트래픽 셰이핑 메커니즘을 상술한다.
