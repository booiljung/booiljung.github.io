2.4.1.2. 알림 스트림(Notification Stream) 및 이벤트 수신 채널 구축 메커니즘

MCP 프로토콜에서 원격 서버가 클라이언트에게 일방적인 통보를 날려야 하는 시나리오는 매우 빈번하다.
예를 들어, 로컬 파일 시스템을 감시하던 시스템 도구가 파일 변경됨 이벤트를 감지했거나, 백그라운드 빅데이터 쿼리 서버가 30분 만에 분석 완료 상태로 전이했을 때, 서버는 클라이언트가 물어보기도 전에 이 사실을 즉시 알려야(Push) 한다.

이러한 수동적 폴링(Polling)의 비효율성을 타파하기 위해, MCP 원격 채널 아키텍처는 앞서 논의한 SSE 연결망을 전용 **알림 스트림(Notification Stream)**으로 정의하고, 이를 통해 클라이언트의 상태 엔진을 실시간으로 동기화시킨다. 본 절에서는 단순한 텍스트 파이프를 넘어, 비동기 상태 갱신, 유실된 이벤트의 재연결(Reconnection) 및 데드레터 큐(DLQ) 복구 로직 등을 아우르는 강건한 이벤트 수신 채널의 핵심 구축 메커니즘을 해부한다.
