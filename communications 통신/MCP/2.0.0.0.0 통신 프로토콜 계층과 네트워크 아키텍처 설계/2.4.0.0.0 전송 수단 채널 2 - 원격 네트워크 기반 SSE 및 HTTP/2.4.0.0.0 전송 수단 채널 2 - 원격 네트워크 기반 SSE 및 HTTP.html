2.4. 전송 수단 채널 2: 원격 네트워크 기반 SSE 및 HTTP

단일 물리 머신 안에서 프로세스 간 파이프라인(Stdio)을 통해 데이터를 교환하는 로컬 방식은 극단적인 저지연성을 보장하지만, 분산 시스템(Distributed System) 환경이나 클라우드 자원을 활용해야 하는 모델에서는 치명적인 한계를 드러낸다. MCP 에이전트가 AWS 등 원격 서버에 격리된 거대한 데이터베이스나 API 게이트웨이와 대화하기 위해서는 물리적 거리를 횡단할 수 있는 새로운 전송 계층이 필수 불가결하다.

이러한 수평적 확장을 위해 MCP는 범용 네트워크 통신 스펙인 **SSE(Server-Sent Events)**와 **HTTP(Hypertext Transfer Protocol)**의 결합(Dual-Channel)을 원격 전송 채널의 표준 모델로 규정한다. 본 장에서는 양방향 WebSocket 대신 단방향 SSE를 심장으로 채택한 아키텍처적 당위성부터, 일시적 네트워크 단절과 프록시(Proxy) 방화벽을 우회하는 실전 통신 안정성 원리까지 원격 네트워크 채널의 모든 구조를 심층적으로 분해한다.
