버그를 추적하려면 과거의 코드를 봐야 한다.

  1. URI 기반 해시 포인팅: MCP는 git://repo_name/commit/8f2a1b 형태의 독자적인 URI 리소스를 노출시킨다. 에이전트가 이 리소스를 호출하면, MCP 내부의 워커가 로컬 머신에서 git checkout을 직접 치지 않고, 순수하게 git show 8f2a1b:main.py 명령어를 내부(in-memory)에서 실행한다.
  2. 트리 뷰(Tree view) 스냅샷: 특정 시점의 폴더 구조 전체를 요구할 경우, 서버는 이를 JSON 형태의 마크다운 디렉터리 트리로 렌더링하여 건네주어 워킹 디렉터리(Working Directory)를 오염시키지 않고 과거의 환영(Snapshot)을 안전하게 읽게 한다.
