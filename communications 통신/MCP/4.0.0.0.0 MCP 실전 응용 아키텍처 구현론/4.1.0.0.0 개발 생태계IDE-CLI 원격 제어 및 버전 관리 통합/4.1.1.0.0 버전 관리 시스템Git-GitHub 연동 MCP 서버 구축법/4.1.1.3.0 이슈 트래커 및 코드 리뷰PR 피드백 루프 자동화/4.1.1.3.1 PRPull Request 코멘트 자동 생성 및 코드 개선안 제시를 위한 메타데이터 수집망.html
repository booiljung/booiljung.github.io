4.1.1.3.1. PR(Pull Request) 코멘트 자동 생성 및 코드 개선안 제시를 위한 메타데이터 수집망

“이 PR 스크리닝 좀 해줄래?“라는 사용자의 지시에 대응하기 위해, 어떤 도구들이 작동해야 할까.

  1. 복합 엔드포인트 파싱(Multi-endpoint Parsing): 에이전트는 get_pr_details 도구를 호출할 뿐이지만, 내부 브로커는 GitHub API를 연쇄 호출하여 “PR 본문”, “변경된 파일 목록”, “각 파일의 Diff 텍스트“를 싹 다 끌어모아 기가바이트급의 거대한 컨텍스트 창을 구성한다.
  2. Reviewer 페르소나 주입: 동시에 서버 위임형 프롬프트 엔진이 작동하여 "너는 이제 10년 차 시니어 엔지니어다. 보안 취약점과 성능 최적화 위주로만 리뷰하라"는 페르소나 지시문을 머릿속에 강제로 주입한다.
  3. 인라인 리뷰(Inline Review) 작성기: 분석을 마친 에이전트는 create_pr_comment 도구를 호출하면서 {"file": "login.py", "line": 42, "body": "여기에 SQL Injection 취약점이 있습니다."} 라는 JSON을 던진다. 브로커는 이를 GitHub API 스펙으로 매핑하여 인간 동료가 쓴 것과 똑같은 인라인 리뷰 코멘트를 깃허브에 등재시킨다.
