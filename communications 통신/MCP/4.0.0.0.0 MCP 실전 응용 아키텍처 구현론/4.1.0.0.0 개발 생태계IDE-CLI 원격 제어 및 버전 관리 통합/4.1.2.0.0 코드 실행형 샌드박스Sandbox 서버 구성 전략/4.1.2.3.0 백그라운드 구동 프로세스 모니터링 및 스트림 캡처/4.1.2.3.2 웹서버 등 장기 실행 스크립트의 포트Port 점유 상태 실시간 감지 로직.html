4.1.2.3.2. 웹서버 등 장기 실행 스크립트의 포트(Port) 점유 상태 실시간 감지 로직

“npm run dev“를 치고 서버가 켜질 때, 에이전트는 언제 브라우저 도구를 켜서 localhost:3000에 접속해야 할까? 무작정 5초를 쉴 수는 없다.

  1. 소켓 바인딩 후킹(Socket Binding Hooking): JIT 컴파일러 단락에 내장된 감시자는 자식 프로세스가 OS 레벨에서 bind() 시스템 콜을 호출하여 특정 포트를 리스닝(Listening)하기 시작하는 순간을 ptrace나 커널 이벤트 단위로 미세하게 스니핑한다.
  2. Ready 이벤트 발산: 포트 점유가 확인되는 정확한 1ms의 순간, 혹은 표준 출력 스트림에서 [Server started on port 3000] 정규식이 캡처되는 순간, 백그라운드 관리자는 이 상태를 Ready로 마킹한다.
  3. 블로킹 해제(Unblocking) 메타 응답: 에이전트는 start_server 도구를 호출할 때 await_ready=true 플래그를 보낸다. 서버가 완전히 구동되었다는 이벤트가 발산될 때 비로소 {"status": "ready", "port": 3000} JSON 응답이 എ이전트로 리턴되며, 이를 바탕으로 에이전트는 한 치의 타이밍 오차도 없이 다음 디버깅 턴을 이어간다.
