4.1.2.2.2. SSRF 방어를 위한 내부망 접근 원천 차단 룰셋 및 화이트리스트 패키지 매니저(npm/pip) 프록시

컨테이너를 격리해도 네트워크가 뚫려 있으면 문제가 된다. 에이전트가 코드로 requests.get("http://169.254.169.254") 서버 메타데이터를 긁거나(SSRF 공격), 채굴 스크립트를 다운받으면 대형 보안 사고다.

  1. 아웃바운드 엄격 제한(Outbound Egress Rule): 샌드박스는 사내망 대역(10.0.0.0/8 등)과 AWS 메타데이터 엔드포인트를 라우팅 테이블 레벨에서 블랙홀(Blackhole) 처리하여 무단 스캔을 원천 봉쇄한다.
  2. 패키지 매니저 화이트리스트 프록시: 에이전트가 코딩을 위해 pip install pandas를 요구할 때 인터넷 전체를 열어줄 수는 없다. MCP 환경망은 PyPI나 NPM 레지스트리로 향하는 트래픽만 허락하는 아웃바운드 프록시(Squid 등)를 통과하게 한다.
  3. 더 나아가, 사내 패키지 미러: 극도의 맹독성 샌드박스에서는 화이트리스트 프록시조차 막고, 사내망에 미리 다운로드받아 보안 검증을 마친 아티팩토리(Artifactory) 로컬 레지스트리로 DNS를 강제 우회시킴으로써 가장 신뢰할 수 있는 개발 의존성(Dependencies)만을 에이전트에게 공급한다.
