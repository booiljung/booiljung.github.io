4.1.2.2.1. 도커(Docker) 컨테이너 기반 일회성(Ephemeral) 코드 평가 환경 생명주기

“가장 완벽한 보안은 코드가 끝난 뒤 모든 흔적을 함께 태워버리는 것이다.”

  1. JIT 스포닝(Just-in-Time Spawning): MCP 클라이언트 세션이 연결되는 즉시, 브로커는 미리 얼려두었던 초경량 리눅스 이미지(예: Alpine)를 기반으로 도커 컨테이너를 론칭한다. 부팅 시간은 밀리초에 불과하다.
  2. 에이전트 전용 감방(Jail): 앞서 설명한 모든 REPL 환경과 파이썬 커널은 이 컨테이너 격벽 안에서만 실행된다. LLM이 우회 해킹으로 드롭퍼(Dropper) 디스크렉토리를 탈환한다 한들, 그것은 컨테이너 내부의 / 루프일 뿐, 진짜 호스트 서버 파일에는 생채기 하나 내지 못한다.
  3. 휘발성 생명주기(Ephemeral Lifecycle): 에이전트의 워크플로우 턴이 완전히 종료되거나, 클라이언트 앱이 꺼지면, 관리 감시 데몬은 그 즉시 docker rm -f를 트리거하여 세션 메모리와 파일 찌꺼기 등을 우주의 먼지로 되돌린다(Tear-down).
