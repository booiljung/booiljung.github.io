4.1.2.1.1. 파이썬(Python jupyter) 커널 및 JS 동적 코드 실행 런타임 제어망 구축

상태가 유지되는 코드 실행 환경을 구축하기 위해 널리 사용되는 최적의 구조는 Jupyter 커널 아키텍처를 브릿징하는 것이다.

  1. 지속성 커널 할당(Persistent Kernel Allocation): MCP 에이전트가 데이터 분석 세션을 요구하면, 브로커는 백그라운드에서 ipykernel 혹은 Node.js REPL 인스턴스를 하나 띄운다. 이 커널은 세션이 끝날 때까지 메모리 공간과 전역 변수 트리를 보존한다.
  2. 비동기 블록 실행(Asynchronous Block Execution): 에이전트는 작성한 코드를 통째로 보내지 않고 {"code": "import pandas as pd; df = pd.read_csv('data.csv')"} 형태로 잘게 청크(Chunk)하여 전송한다. 커널은 이를 평가(Eval)하고 결과를 MCP 브로커에게 반환한다.
  3. 가변 런타임 스위칭: 파이썬 코드를 실험하던 에이전트가 웹 크롤링을 위해 갑자기 Puppeteer가 필요해지면, 동일한 세션 내에서 Node 커널을 병렬로 띄우고 두 언어 간의 JSON 출력을 인터포트(Inter-port) 파이핑하여 매끄럽게 통신하게 돕는 제어망이 구축된다.
