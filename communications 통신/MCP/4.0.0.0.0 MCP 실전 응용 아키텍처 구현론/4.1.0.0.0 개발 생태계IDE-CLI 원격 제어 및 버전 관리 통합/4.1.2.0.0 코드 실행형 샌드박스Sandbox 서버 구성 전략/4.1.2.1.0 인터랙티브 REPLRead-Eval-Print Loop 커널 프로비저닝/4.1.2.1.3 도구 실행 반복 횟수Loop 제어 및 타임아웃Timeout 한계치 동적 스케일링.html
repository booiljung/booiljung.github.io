4.1.2.1.3. 도구 실행 반복 횟수(Loop) 제어 및 타임아웃(Timeout) 한계치 동적 스케일링

“1부터 1억까지 소수(Prime number)를 찾는 코드를 짜서 실행시켜 줘.” 이 무심한 코드는 샌드박스 커널을 수 시간 동안 먹통으로 만들어버릴 수 있다. 무한 루프나 과도한 연산은 시스템 리소스를 말라붙게 한다.

  1. 타임아웃 동적 쿼터제: 기본적으로 도구의 실행 시간(Timeout)은 3초 내외로 묶여 있다. 하지만 데이터셋 훈련처럼 정당한 시간이 필요한 경우, 에이전트가 execute_code(..., timeout=30) 명시적으로 메타데이터를 넘겨 시간 한계치를 동적 스케일링할 수 있다.
  2. 시그널 킬(SIGKILL) 강제 중단: 허용된 타임아웃 쿼터를 초과하는 순간 프로세스 감시자가 커널에게 SIGKILL 시그널을 보내 강제로 루프 연산을 날려버린다(Abort).
  3. 루프백 호출 방지(Anti-recursion): LLM이 동일한 에러를 고치지 못하고 실행 -> 에러 -> 조금 수정하여 실행 -> 에러 무한 반복에 빠지는 현상 역시 모니터링된다. 도구 호출이 5회를 넘어가는 시점에 브로커가 개입하여, “사람의 도움을 요청하라“는 강제 지시문으로 트랜지션을 발생시킨다.
