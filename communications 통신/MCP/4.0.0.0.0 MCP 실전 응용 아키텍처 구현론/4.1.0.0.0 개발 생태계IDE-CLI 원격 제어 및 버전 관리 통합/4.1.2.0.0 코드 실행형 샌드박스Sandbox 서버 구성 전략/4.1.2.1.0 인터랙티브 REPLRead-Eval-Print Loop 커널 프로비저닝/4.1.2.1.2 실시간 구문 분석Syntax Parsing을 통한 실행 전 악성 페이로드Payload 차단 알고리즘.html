4.1.2.1.2. 실시간 구문 분석(Syntax Parsing)을 통한 실행 전 악성 페이로드(Payload) 차단 알고리즘

에이전트(LLM)는 환각에 의해, 혹은 외부 악성 프롬프트 인젝션에 의해 os.system("rm -rf /") 같은 자멸적인 코드를 REPL에 써넣을 위험이 존재한다.

  1. AST 선제적 검사(Pre-flight AST Inspection): 커널이 코드를 수신하더라도 즉시 eval 하지 않는다. 브로커 단에 내장된 정적 분석 엔진이 파이썬 스크립트를 추상 구문 트리(AST) 수준으로 분해한다.
  2. 금지된 함수 트리거: AST 트리거 노드 속에 os.system, subprocess, fs.unlinkSync 처럼 OS 레벨에 치명적인 영향을 주는 시스템 콜(System call) 명령어 호출이 탐지되면 차단 모듈이 켜진다.
  3. 거부 인터럽트 및 에이전트 반성 루프: 차단된 코드는 커널에 도달하지 못하며, 대신 에이전트에게 ["Security Violation: Use of 'os.system' is strictly forbidden in this environment"]라는 응답이 즉각 전송된다. 에이전트는 이 예외를 받아들여 안전한 파이썬 내장 라이브러리로 코드를 우회 수정하는 반성(Reflection) 과정을 거친다.
