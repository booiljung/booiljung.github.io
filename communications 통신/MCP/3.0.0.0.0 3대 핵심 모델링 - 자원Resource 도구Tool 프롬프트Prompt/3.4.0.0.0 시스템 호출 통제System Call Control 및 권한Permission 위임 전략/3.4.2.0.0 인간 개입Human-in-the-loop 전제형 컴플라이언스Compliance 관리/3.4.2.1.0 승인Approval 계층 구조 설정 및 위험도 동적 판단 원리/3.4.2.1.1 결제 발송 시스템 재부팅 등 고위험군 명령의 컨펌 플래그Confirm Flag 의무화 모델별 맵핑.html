3.4.2.1.1. 결제 발송, 시스템 재부팅 등 고위험군 명령의 컨펌 플래그(Confirm Flag) 의무화 모델별 맵핑

MCP 서버의 툴 스키마 명세서에는 파라미터뿐만 아니라, 이 도구를 실행하기 위해 인간의 컨펌이 필요한지를 나타내는 절대적 플래그가 존재한다.

  1. 사용자 확인 필수(userConfirmationRequired): 도구 정의 메타데이터 내에 명시적으로 requiresConfirmation: true 값이 설정된 도구들(예: git_push, drop_database, execute_payment)은 에이전트의 의지만으로는 절대 서버 백엔드 게이트를 통과할 수 없다. 라우터에서 완벽히 블로킹(Blocking)된다.
  2. 조직 맵핑 정책(Organizational Policy Mapping): 이 플래그는 단일 서버에 하드코딩되는 것이 아니라, 중앙화된 엔터프라이즈 컴플라이언스 서버와 연동되어 사원 직급(주니어 vs 시니어)이나 프로젝트 환경(Dev vs Prod)에 따라 “개발 서버는 자율 패스, 상용 서버는 컨펌 필수“로 동적 맵핑(Dynamic Mapping) 전환 구조를 이룬다.
  3. 단위 도구의 멱등적 보장: 한 번 플래그가 수립된 도구는 LLM이 아무리 우회 프롬프트 명령어("사용자가 허락했으니 그냥 넘어가.")를 던진다 하더라도, 이 플래그는 LLM 모델 관점이 아니라 서버 자체의 하드웨어 브릿지단에 설정된 규약이므로 뚫고 들어올 방법이 전무하다.
