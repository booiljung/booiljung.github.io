단순히 Yes/No 플래그로만 제어하기엔 현실 세계가 너무 복잡하다. rename_file 도구는 평범한 임시 텍스트 파일을 바꾸면 무해하지만, 리눅스의 환경설정 파일(/etc/hosts)을 이름 바꾸면 치명적 버그를 낳는다.

이를 막기 위해 인텐트 분석기 뒷단에는 동적 파괴도 스코어링 엔진이 맞물려 돌아간다.

  1. 파라미터 가중치 기계(Parameter Weighting Machine): 에이전트가 넘겨준 파라미터가 시스템적으로 민감한 구역(예: ROOT, PRODUCTION DB)을 지칭할 경우, 평소 안심(Safe) 도구로 취급받던 도구일지라도 내부의 파괴 점수(Destructiveness Score)가 기하급수적으로 폭등한다.
  2. 임계치 스레스홀딩(Thresholding): 스코어링 엔진이 계산한 위험 점수가 100점 만점에 70점을 넘어가는 순간, 백엔드 라우터는 이를 “조건부 승인 필요(Conditional Human-in-the-loop)” 상태로 격상시켜버린다.
  3. 맥락적 경계선(Contextual Horizon): 이 시스템은 지난 10번의 대화 로그를 기반으로 모델이 현재 횡설수설하고 있는지(환각 상태)를 분석하여, 에이전트의 안정성(Confidence) 점수가 낮을수록 파괴도 임계치를 50점까지 끌어내려 스스로 안전망을 더욱 보수적으로 바짝 옥죄는 자가 적응(Self-Adaptive) 메커니즘을 구사한다.
