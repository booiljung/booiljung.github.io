3.4.2.3.2. 작업 승인 중단 큐(Pending Queue) 시스템의 지연 대기에 따른 타임아웃 자동 취소 정책

클라이언트 UI가 서버 리소스에 결제 승인을 띄워놓고 1시간 동안 응답을 하지 않으면 어떻게 되는가?

  1. 상태 레지스트리 펜딩 노드: 도구 호출은 즉각 실행 파이프라인 대신, 메모리 효율이 높은 “보류 큐(Pending Queue)” 트랙에 안착하여 슬립(Sleep)에 들어간다.
  2. TTL(Time To Live) 스위프(Sweep) 데몬: 보통 인간 펜딩 작업의 TTL은 서버 단에서 5분~15분으로 꽤 길게 세팅된다. 15분이 지날 때까지 사용자의 수락/거절(Accept/Reject) 응답이 안 오면, 가비지 컬렉터(Garbage Collector)가 이 펜딩 객체를 자동으로 거절(Auto-Reject) 처리해 버린다.
  3. 컨텍스트 증발 통지: 타임아웃으로 취소되면, 콜백 채널을 통해 LLM에게 “Timeout: User did not respond in 15m. Operation aborted.“라고 무미건조한 시스템 텍스트를 흘려보내어, 에이전트가 “사용자가 대답하지 않아 작업이 취소되었으니 정중하게 다시 물어봐야겠다“라고 컨텍스트를 회복하도록 돕는다.
