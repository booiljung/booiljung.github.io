3.4.2.2.1. 파일 덮어쓰기(WriteFile) 도구 실행 전 기존 상태 대비 변경본 텍스트 비교 선행 노출기(Diff)

에이전트가 “A 파일 수정할까요?“라고 물었을 때, “네“라고 답했다가 멀쩡한 코드가 통째로 날아가는 낭패를 막으려면, 무엇이 어떻게 바뀌는지 정확하게 보여주는 Diff 프리뷰(Preview) 기능이 핵심이다.

  1. 상태 프리페치(Pre-fetch): 에이전트가 write_file(A, new_content)를 호출형 대기 큐에 올리면, 브로커는 서버에서 원본 파일 A의 내용을 먼저 긁어온다.
  2. 시각적 Diff 생성: 클라이언트 앱 내에 내장된 Diff 엔진이 원본 텍스트와 에이전트가 제안한 new_content 사이의 차이점(레드-마이너스, 그린-플러스)을 직관적인 깃허브(GitHub) 스타일의 Split 뷰나 Unified 뷰로 렌더링 하여 화면 정중앙에 플로팅 시킨다.
  3. 토큰 레벨(Token-level) 정밀 하이라이팅: 단순히 줄(Line) 단위 비교를 넘어, var = 1이 let = 1로 바뀐 단어(Token) 수준의 정밀한 하이라이팅을 제공하여 찰나의 순간에도 사용자의 인지 능력을 극대화한다.
