3.4.2.2.3. 확인, 거절, 수정(Edit args) 옵션을 제공하는 사용자 터미널 다이얼로그 프로토콜

위의 시각적인 GUI 방식 외에도 순수 터미널(CLI) 환경에서 작동하는 CLI 에이전트 도구들은 터미널 다이얼로그 기반의 **인터랙티브 프롬프트(Interactive Prompt)**를 채택한다.

  1. 삼지선다(Yes/No/Edit) 옵션 제안: 에이전트가 실행 전 브레이크포인트(Breakpoint)를 걸면 터미널에 [Y]es, run it / [N]o, cancel / [E]dit arguments?의 선택지가 제시된다.
  2. 동적 파라미터 수정(In-place Argument Patching): 사용자가 E를 누르면 vim 이나 nano 같은 로컬 편집기가 즉석에서 열리며 방금 에이전트가 조립한 JSON 파라미터(예: {"query": "DROP TABLE..."})가 텍스트로 나타난다. 사용자가 이를 직접 안전하게 고치고 저장하면, 그 수정본(Patched Arguments)이 다시 서버로 날아간다.
  3. LLM 지식 격리: 특이한 점은, 사용자가 이렇게 중간에 변수를 고집스럽게 바꿔서 서버로 쏘아 도구를 실행시키면, “최종적으로 서버에 실행된 변수“의 결과가 LLM 컨텍스트에도 다시 덮어씌워져 기록된다. 인지 부조화(Cognitive Dissonance)를 방지하는 것이다.
