3.4.2.2.2. 코드-마크다운 렌더링 결과물 프리뷰(Preview) 구성을 통한 인간 작업자 피드백 획득 과정

쉘 스크립트 실행이나 외부 API(예: 결제 송금) 호출의 경우 Diff 차이점이 존재할 수 없다. 이 경우에는 **구조화된 의도 프리뷰(Intent Preview)**가 UI에 뜬다.

  1. 렌더링된 메타데이터 패널: “목적지: api.stripe.com/payout”, “금액: $500”, “대상: Alice” 와 같은 핵심 파라미터들이 난잡한 JSON형태가 아니라, 디자이너가 의도한 마크다운 테이블이나 UI 컴포넌트로 예쁘게 파싱되어 노출된다.
  2. 사이드 이펙트 경고문(Disclaimer): UI 렌더러는 서버 스펙에 기재된 destructive: true 메타데이터를 인지하고 화면 테두리를 붉게 물들이며 “경고: 이 작업은 되돌릴 수 없습니다“라는 팻말을 붙인다.
  3. 피드백 사이클의 루프백(Loopback): 단순히 수락/거절을 넘어서, 만약 사용자가 “500달러 말고 300달러로 바꿔서 다시 제안해“라고 피드백(텍스트)을 입력하면 이 자연어 문장이 다시 LLM 컨텍스트로 흘러들어가 에이전트의 자기 보정(Self-Correction)을 통한 재수정(Re-drafting) 제출 루프를 발생시킨다.
