3.4.2.4.1. 인간 검열관 개입에 따른 거절(Reject) 및 부분 수정 승인(Partial Approval)에 대한 무결성 체인 보증

인간이 에이전트의 제안을 있는 그대로 100% 수용하기보단, 파라미터를 살짝 고쳐서 승인하는 일(Edit args)이 빈번하게 일어난다. 이때 “AI가 제안한 것“과 “인간이 최종 수정한 것“이 뒤섞이면 감사(Audit)는 파탄 난다.

  1. 상태 천이 불변성(Immutable State Transition): 도구 스키마는 최초 LLM이 생성한 파라미터(Proposed Payload)를 절대로 덮어씌워 지우지(Overwrite) 않는다. 이를 원본 상태(State 0)로 보존한다.
  2. 패치 체이닝(Patch Chaining): 사용자가 파라미터를 수정하면, 시스템은 원본 스키마 위에 사용자의 입력 차이점(Diff Patch)만을 State 1으로 추가 기록한다.
  3. 최종 서명(Final Signature): 이 두 개의 묶음에 현재 로그인된 인간의 세션 ID, IP 주소, 그리고 승인 버튼을 누른 밀리초 단위의 타임스탬프를 묶어 해시(Hash) 서명함으로써, “AI는 A를 제안했으나 인간이 B로 고쳐서 승인했다“는 완벽한 무결성 체인(Integrity Chain)을 완성한다.
