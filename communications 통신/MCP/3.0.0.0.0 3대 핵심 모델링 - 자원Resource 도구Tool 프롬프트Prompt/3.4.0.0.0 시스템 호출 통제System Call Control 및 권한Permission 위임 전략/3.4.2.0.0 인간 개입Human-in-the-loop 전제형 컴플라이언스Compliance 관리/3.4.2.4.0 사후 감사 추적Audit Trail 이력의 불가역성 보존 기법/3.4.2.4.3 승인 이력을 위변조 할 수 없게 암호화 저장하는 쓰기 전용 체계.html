3.4.2.4.3. 승인 이력을 위변조 할 수 없게 암호화 저장하는 쓰기 전용 체계

사내의 악의적인 시스템 관리자가 도구를 사용해 횡령을 저지른 뒤, 자신의 ‘승인(Approval) 내역’ 레코드를 데이터베이스에서 직접 지워버리면 완전 범죄가 성립된다. 이를 막는 것이 쓰기 전용(Append-only) 암호화 저장 체계다.

  1. 웜(WORM: Write Once Read Many) 스토리지: 컴플라이언스 로그가 적재되는 데이터베이스 시스템은 태생적으로 UPDATE나 DELETE 쿼리가 아예 존재하지 않는 구조(예: AWS QLDB)로 구축된다.
  2. 해시 체인(Hash Chain): 이전 블록(방금 전 승인 내역)의 해시값이 다음 블록(현재 승인 내역)의 암호화 헤더에 포함되는 블록체인 래저(Ledger) 기술을 차용한다. 만약 관리자가 DB를 해킹해 이전 기록을 날려버리면 그 이후의 모든 해시 체인이 와해되어 즉각적인 알람(Tamper Alert)이 쏟아져 나온다.
  3. 비대칭 키 서명(Asymmetric Key Signing): 로그를 작성할 때는 오직 서버의 Private Key만 개입하며, 감사관(Auditor)은 퍼블릭 키(Public Key)를 통해 이 기록들이 단 한순간도 위변조되지 않았음을 수학적으로 확증(Verify)할 수 있는 완벽한 증명 체계를 갖게 된다.
