3.4.1.3.1. 파일시스템 감옥(Chroot Jail) 도입을 통한 컨테이너형 디렉터리 생성 및 하드웨어 통제 우회 차단

단순 스크립트 도구를 실행하기 위해 매번 수 기가바이트짜리 가상 머신을 띄울 수는 없다. 도구 실행 엔진은 OS 레벨의 가벼운 격리 기능인 파일시스템 감옥(Chroot Jail) 패턴을 차용한다.

  1. 전용 워크스페이스(Dedicated Workspace) 그루핑: MCP 서버는 도구를 위임받는 순간 /tmp/mcp_workspace_sandbox/job_1a2b/ 라는 일회용 빈 껍데기 폴더를 생성하고, 프로세스의 루트 디렉터리(/)를 이 임시 폴더로 묶어버린다(Chroot).
  2. 상위 디렉터리 탈출 금지: 에이전트가 무슨 짓을 해서 명령줄에 cd ../../../../etc 를 욱여넣든 간에 샌드박스 내부의 프로세스에게는 자신이 갇힌 임시 폴더가 이 세계의 끝(Root)이다. 결코 호스트의 민감한 시스템 파일에 접근할 수 없다.
  3. 하드웨어 및 네트워크 네임스페이스 격리: Linux의 cgroups와 namespaces 기술을 연계하여, 이 도구가 비트코인 채굴 같은 엉뚱한 짓을 막으려 CPU 10%, RAM 512MB 이상을 먹지 못하도록 하드웨어 자원을 틀어막고(Cap), 불법 패킷을 밖으로 쏘지 못하게 네트워크 망마저 가상화(Virtualize)시켜버린다.
