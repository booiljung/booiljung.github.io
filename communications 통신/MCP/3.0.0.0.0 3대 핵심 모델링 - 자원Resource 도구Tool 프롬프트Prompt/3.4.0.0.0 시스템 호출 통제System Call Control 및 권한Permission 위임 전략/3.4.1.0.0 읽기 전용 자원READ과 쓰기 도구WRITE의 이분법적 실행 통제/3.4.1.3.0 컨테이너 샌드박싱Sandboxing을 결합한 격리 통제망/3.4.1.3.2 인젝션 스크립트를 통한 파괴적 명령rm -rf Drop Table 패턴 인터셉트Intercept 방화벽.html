3.4.1.3.2. 인젝션 스크립트를 통한 파괴적 명령(rm -rf, Drop Table) 패턴 인터셉트(Intercept) 방화벽

에이전트에게 샌드박스를 제공하더라도, 정상적인 워크스페이스 로직 안에서 그들이 생성한 데이터를 모두 삭제하여 프로젝트의 근간을 흔들 여지는 남아있다.

이를 막기 위해 OS 쉘로 명령이 내려가거나 RDBMS로 데이터 쿼리가 날아가기 직전, 거대한 텍스트 필터망인 명령어 패턴 인터셉터가 작동한다.

  1. 정규 구문 블랙리스트(Blacklist Syntax): 쉘 스크립트 실행 도구의 경우, 실행 스트림 텍스트 안에 rm -rf /, mkfs, sudo 와 같은 시스템 파괴형 명령어 클레임이 단 하나라도 매칭되면 즉시 실행을 중단(Abort)한다.
  2. SQL 문법 방어막: 데이터베이스 도구 어댑터 내부에는 단순 문자열 검사가 아니라 SQL 파서 단축키가 박혀있어 DROP TABLE, TRUNCATE, ALTER TABLE 과 같은 파괴적 DDL이 감지되는 순간 쿼리를 데이터베이스로 쏘기도 전에 쓰레기통에 던져버린다.
  3. 엄격 모드 런타임: NODE_ENV나 언어별 런타임에서 특정 시스템 모듈(예: Node.js의 fs.unlinkSync 혹은 child_process)에 대한 심볼릭 링크를 인터셉트하여, 안전하다고 판단된 모듈 호출 이외의 스크립팅(Scripting) 자체를 내부적으로 무력화시키는 함수 스와핑(Function Swapping) 흑마법이 적용되기도 한다.
