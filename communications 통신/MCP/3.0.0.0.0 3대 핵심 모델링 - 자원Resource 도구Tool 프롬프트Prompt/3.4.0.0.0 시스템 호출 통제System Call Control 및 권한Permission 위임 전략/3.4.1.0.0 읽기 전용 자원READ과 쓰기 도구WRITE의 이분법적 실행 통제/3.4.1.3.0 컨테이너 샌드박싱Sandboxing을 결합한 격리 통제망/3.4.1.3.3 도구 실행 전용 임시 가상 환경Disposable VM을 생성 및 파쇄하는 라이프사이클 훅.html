3.4.1.3.3. 도구 실행 전용 임시 가상 환경(Disposable VM)을 생성 및 파쇄하는 라이프사이클 훅

위의 모든 방어막을 겹겹이 쳐놓아도, 무언가 오염된 상태가 영구적으로 남아 다음 에이전트의 다른 요청을 더럽힌다면(State Pollution) 진정한 보안 격리라 부를 수 없다. 궁극의 해결책은 **‘사용 후 즉각 파쇄’**다.

  1. 에페메랄(Ephemeral) 라이프사이클: MCP 서버는 도구가 트리거 될 때 도커(Docker) 컨테이너나 경량 마이크로 VM(Firecracker)을 동적으로 올려 순결한(Pristine) 가상 환경(Disposable VM)을 만든다.
  2. 도구 런타임 후크 연결(Hook Binding): 도구가 실행되면서 패키지를 다운로드하거나, 난잡하게 테스트 파일을 생산해 내는 행위는 전부 가상화된 메모리나 OverlayFS 공간 위에서만 이루어진다.
  3. 즉각적 자원 소강(Teardown): 에이전트의 도구 호출이 끝나고 "성공/실패" 결과값 텍스트만 뽑아내어 MCP 호스트로 반환하고 나면, 멈칫할 틈도 없이 전체 가상 환경의 프로세스를 SIGKILL로 파괴해 버린다. 아무런 잔존 위협 지식도, 바이러스의 확산 경로도 완벽하게 단절되는 무결성 파이프라인의 완성이다.
