3.4.1.4.1. 클라이언트-서버 간 모든 데이터 흐름의 원천 송출 방향(Directionality) 기록 시스템

“누가, 언제, 어디서, 무엇을, 어떻게 했는가?” 이 육하원칙이 단 1byte의 누락도 없이 기록되어야만 에이전트 시스템을 엔터프라이즈 환경에 도입할 수 있다.

  1. 페이로드 양방향 미러링(Bidirectional Mirroring): 에이전트가 쏘아 보낸 JSON 요청(Request)과 워커가 반환한 JSON 응답(Response)은 단지 통신을 맺고 끝나는 것이 아니다. 이 통신 규격의 복사본이 실시간 섀도우 로거(Shadow Logger)를 타고 엘라스틱 서치(Elasticsearch)나 AWS CloudWatch로 영구 적재된다.
  2. 인과율 트레이싱(Causal Tracing) ID: 단순한 시간순 정렬로는 여러 도구가 병렬로 실행될 때 문맥을 파악하기 힘들다. 모든 패킷에는 시스템 부팅 시부터 할당된 부모-자식 관계의 Trace ID가 박혀, “에이전트가 A를 읽었기 때문에(Cause), B를 덮어썼다(Effect)“라는 서사적 인과관계를 그래프 형태로 그려낼 수 있게 한다.
  3. 은닉 채널 차단(Covert Channel Block): 도구가 콘솔(stdout, stderr) 출력으로 내뱉은 임의의 로그까지 모두 이 기록 시스템으로 파이프라이닝 되어, 에이전트가 몰래 시스템 환경 변수를 커맨드라인에 덤프(Dump)해 빼돌리려는 시도조차 시스템 눈을 피하지 못하게 만든다.
