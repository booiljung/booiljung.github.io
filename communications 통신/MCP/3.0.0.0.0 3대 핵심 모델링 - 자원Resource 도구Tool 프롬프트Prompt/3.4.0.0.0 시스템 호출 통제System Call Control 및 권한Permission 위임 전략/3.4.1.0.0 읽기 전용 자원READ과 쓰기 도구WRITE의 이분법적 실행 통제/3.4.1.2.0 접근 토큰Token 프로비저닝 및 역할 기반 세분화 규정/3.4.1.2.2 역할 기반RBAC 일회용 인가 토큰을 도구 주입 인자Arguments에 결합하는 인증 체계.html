3.4.1.2.2. 역할 기반(RBAC) 일회용 인가 토큰을 도구 주입 인자(Arguments)에 결합하는 인증 체계

일부 중요 도구, 예컨대 commit_and_push_code 같은 도구는 단순히 세션이 유효하다는 것만으로는 부족하다. 해당 행위를 뒷받침하는 Git 계정 권한이 필요하다.

  1. 플러그인 크레덴셜 매핑: 클라이언트 에이전트(예: Claude)가 도구를 호출할 때 자신의 순수 파라미터(예: message="bug fix")만 보내지 않는다. MCP 클라이언트는 자신의 환경변수에 구비된 일회용 인가 토큰(예: GitHub PAT, AWS 임시 토큰)을 해당 도구 페이로드의 암시적 헤더(Implicit Header)에 결합하여 함께 서버로 쏜다.
  2. 서버 측 롤백(RBAC) 크로스 체크: 도구가 실행되기 직전, 어댑터 계층은 도구 자체의 파라미터 무결성과 더불어 첨부된 토큰의 유효성을 서드파티에 한 번 더 Ping-Pong 쳐서 크로스 체크한다.
  3. 토큰 소진 및 스코프 제한: 이 일회용 토큰은 오직 그 특정 도구 런타임 수명기 동안만 사용되고 메모리에서 소멸되며, “이 레포지토리에 푸시만 가능“과 같이 도달 범위(Scope) 클레임이 철저히 제한된 상태로만 도구에 던져진다.
