3.4.1.2.3. 권한 이탈 요청 발생 시 즉각적인 403 Forbidden 응답 반환 설계

권한 시스템은 거절(Denial)할 때가 가장 중요하다. 만약 에이전트가 “운영 DB 삭제 통로“를 찔렀는데 아무런 응답을 주지 않으면 타임아웃까지 무의미한 지연이 생기며, 모호하게 “오류가 발생했습니다“라고 주면 에이전트는 다른 방식으로 DB 삭제를 집요하게 재시도할지 모른다.

  1. 즉각적이고 차가운 거절(Fast Reject): 권한 검증 벽에서 막히면 도구 실행 엔진은 단 1ms도 지연하지 않고 403 Forbidden JSON-RPC 응답을 즉각 클라이언트로 내려꽂는다.
  2. 명확한 인지 차단 메시지: 반환되는 에러의 본문은 대단히 명시적인 가이드라인을 띄고 있다. "error": "You do not have permission to execute 'delete_table'. Your role (GUEST) is restricted to READ-only operations."
  3. 에이전트 컨텍스트 변형 유도: 이 명확한 거절 메시지는 LLM의 다음 턴 사유(Reasoning) 과정에서 거대한 벽돌로 작용하여 “아, 나는 권한이 없으므로 이 작업을 삭제 방식이 아니라 보고서를 작성해 사용자에게 넘기는 방식으로 우회해야겠다“라고 즉각적인 플랜 B를 도출하도록 유도한다.
