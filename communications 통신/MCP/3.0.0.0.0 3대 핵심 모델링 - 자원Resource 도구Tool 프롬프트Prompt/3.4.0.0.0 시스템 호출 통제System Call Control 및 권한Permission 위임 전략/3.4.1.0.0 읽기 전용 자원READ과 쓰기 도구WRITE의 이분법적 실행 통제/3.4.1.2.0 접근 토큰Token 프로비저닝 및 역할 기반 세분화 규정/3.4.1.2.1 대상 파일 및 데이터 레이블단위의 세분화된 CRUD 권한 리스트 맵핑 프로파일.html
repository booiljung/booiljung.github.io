3.4.1.2.1. 대상 파일 및 데이터 레이블단위의 세분화된 CRUD 권한 리스트 맵핑 프로파일

대충 “파일 시스템 도구 접근 권한 허용“이라는 뭉뚱그려진 토큰은 MCP 세계에 존재하지 않는다. 이는 보안 구멍(Security Hole) 그 자체이기 때문이다.

  1. 마이크로 권한(Micro-Permissions): 초기 인증 시 클라이언트는 단순한 JWT가 아니라, 어떤 경로의 어떤 명령어 집합에 접근할 수 있는지가 매우 잘게 쪼개진(Fine-grained) 프로파일(Profile)을 서버와 합의한다.
  2. 대상 세분화(Target Granularity): “에이전트 A는 /tmp/workspace/* 에 대해서는 CRUD 권한을 모두 갖지만, /home/user/* 경로에 대해서는 Read 권한만 갖고, 나머지 /etc/ 는 접근 자체(Visibility)가 금지된다“는 식의 구체적인 JSON 매핑 구조체(Mappping Struct)가 세션에 결합된다.
  3. 토큰 내장 인가(Embedded Authorization): 이 프로파일 정보는 서버의 메모리에만 있는 것이 아니라, JWT 페이로드의 클레임(Claim) 혹은 세션 메타데이터에 직렬화되어 박히므로 수천 개의 툴 접근 분기점이 이 프로파일 단 하나에 의해 O(1)의 속도로 통제된다.
