3.4.1.1.2. 데이터베이스 변경(Update) 및 인프라 조작 도구 대상의 명시적 세션 권한 분류

반면, 요청이 tools/call 엔드포인트를 때렸다면 인텐트 분석기는 긴급 경보(Red Flag)를 발동한다. 도구는 필연적으로 비가역적인 **상태 변이(Mutation)**를 낳기 때문이다.

  1. 상태 변경(State-Mutation) 전제: 아무리 무해해 보이는 도구("도메인 핑 던지기")일지라도 tools 범주에 속해있다면 시스템은 이를 인프라 상태를 변화시킬 수 있는 임계 호출로 취급하여 특별한 격리 큐(Isolation Queue)로 넘긴다.
  2. 보안 세션 토큰 결속: 이 파이프라인은 진입 즉시 현재 MCP 연결 세션에 바인딩 된 클라이언트의 권한(Role) 수준을 검증한다. 만일 외부 게스트용 클라이언트로 접속된 세션이라면, 디비 삭제를 시도하는 도구는 즉각 401 Unauthorized 차단 벽에 부딪히게 된다.
  3. 명시적 권한 모델(Explicit Privilege): 도구를 실행하기 위해선 암묵적인 허용(Implicit Policy)이 적용되지 않는다. 화이트리스트(Whitelist) 맵핑 프로토콜에 따라 해당 도구와 해당 클라이언트 간의 명시적인 매핑(Mapping)이 증명되어야만 비로소 엔진이 방아쇠를 당긴다.
