Chapter 3. 3대 핵심 모델링: 자원(Resource), 도구(Tool), 프롬프트(Prompt)

Model Context Protocol(MCP)의 물리적 네트워크 전송 계층(Chapter 2)이 튼튼하게 구축되었다면, 그 인프라 위를 실질적으로 달리는 데이터의 형이상학적 본질을 규명해야 한다.
에이전트(클라이언트)가 호스트(서버) 환경과 유의미하게 상호작용하기 위해, MCP는 세상의 모든 디지털 컨텍스트를 자원(Resource), 도구(Tool), **프롬프트(Prompt)**라는 3대 핵심 원형(Archetype)으로 철저히 추상화 및 규격화하였다.

본 장에서는 이 3대 모델의 아키텍처적 본질과 역할, 그리고 이들이 어떻게 조합되어 거대한 에이전틱(Agentic) 생태계를 구축하는지 각 컴포넌트의 내부 구조를 치밀하게 해부한다.
