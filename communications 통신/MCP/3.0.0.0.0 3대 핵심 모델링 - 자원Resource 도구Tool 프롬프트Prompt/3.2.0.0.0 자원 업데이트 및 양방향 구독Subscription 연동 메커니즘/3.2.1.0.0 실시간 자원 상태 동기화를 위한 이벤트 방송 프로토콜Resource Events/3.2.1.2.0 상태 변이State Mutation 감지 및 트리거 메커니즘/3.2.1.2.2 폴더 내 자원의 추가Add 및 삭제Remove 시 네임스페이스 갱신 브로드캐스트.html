3.2.1.2.2. 폴더 내 자원의 추가(Add) 및 삭제(Remove) 시 네임스페이스 갱신 브로드캐스트

단일 파일의 내용이 바뀌는 것보다 더 포괄적인 변화는 프로젝트 내에 새로운 파일이 생성되거나 기존 파일이 삭제되는 등 형태(Topology) 자체가 바뀌는 것이다. 에이전트가 특정 작업 경로(/src/)를 통째로 구독하고 있다면, 이 거대한 트리 구조망의 변이 역시 정교하게 방송되어야 한다.

  1. Tree Hierarchy 변경 감지: 폴더 감시자가 FILE_CREATE 또는 FILE_DELETE 인터럽트를 받으면, 서버는 해당 이벤트가 일어난 노드가 구독된 네임스페이스 트리의 어디쯤 속하는지 상대 경로 맵핑을 수행한다.
  2. 명시적 목록 갱신(List Updated) 신호: 파일의 추가/삭제는 개별 자원의 내용 변화(ResourceUpdated)가 아니라 자원 목록의 갱신에 해당한다. 서버는 notifications/resources/list_changed 이벤트 규격을 통해 “이 프로젝트의 자원 지도가 바뀌었으니, resources/list를 다시 호출해 최신 맵을 로드하라“는 브로드캐스트 알림을 뿌린다.
  3. 가시성 필터 재적용: 파일이 추가되었다고 바로 알리지 않는다. 앞서 3.1.1.4 절에서 다룬 배제 필터(gitignore 등)를 먼저 통과시켜, 새로 생겼지만 에이전트에겐 보여주지 말아야 할 파일(.env.local 등)에 대해서는 의도적으로 알림을 묵살(Silence)한다.
