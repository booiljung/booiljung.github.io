3.2.1.2.1. 운영체제 레벨 inotify 파일 감시자 래핑을 통한 핫 트리거 수립

서버가 파일의 실시간 변경 사항을 자원 소모 없이 알 수 있는 유일한 방법은, 파일의 주인이자 관리자인 운영체제(OS)에게 알림을 위임하는 것이다.

MCP 백엔드 애플리케이션은 호스트 플랫폼별로 특화된 저수준 제어 API들을 추상화하여, inotify 래핑 시스템 트리거를 수립한다.

  1. 커널 레벨의 이벤트 인터럽트(Interrupt): 리눅스 계열의 경우 inotify(inode notify), 윈도우 환경은 ReadDirectoryChangesW, macOS는 FSEvents 등 커널 수준의 서브시스템 API를 호출한다. 이는 서버 프로세스가 바이트 스트림을 계속 감시하는 것이 아니라, 해당 파일 아이노드(inode) 블록에 변화가 일어날 때마다 커널이 프로세스에게 비동기 하드웨어 인터럽트 신호를 날려주는 방식이다.
  2. 이벤트 타입 분기: 운영체제가 던져주는 이벤트(WRITE, ATTRIB, DELETE)를 가로챈 서버는, 이를 MCP 규격에 맞게 분류한다. 파일 내용 수정 시에는 ResourceUpdatedEvent 트리거가 당겨지고, 파일 접근 권한(chmod)만 변경되었다면 컨텍스트엔 영향이 없으므로 무시(Discard)하는 스마트 필터링 단계를 거친다.
  3. 네트워크 푸시 전환: 감지된 변경 이벤트 파동은 즉각 클라이언트 세션 구독자 명부로 전달되며, 일치하는 에이전트가 존재할 경우 notifications/resources/updated 원격 알림 패킷을 조립하여 전송 소켓에 태워 보낸다.
