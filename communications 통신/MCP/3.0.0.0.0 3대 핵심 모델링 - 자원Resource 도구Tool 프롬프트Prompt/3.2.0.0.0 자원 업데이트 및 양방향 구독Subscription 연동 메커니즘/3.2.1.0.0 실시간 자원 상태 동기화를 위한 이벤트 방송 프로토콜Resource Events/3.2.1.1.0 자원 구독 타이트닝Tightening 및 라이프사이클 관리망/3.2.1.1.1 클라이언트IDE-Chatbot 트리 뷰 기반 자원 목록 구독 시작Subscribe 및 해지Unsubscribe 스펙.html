3.2.1.1.1. 클라이언트(IDE/Chatbot) 트리 뷰 기반 자원 목록 구독 시작(Subscribe) 및 해지(Unsubscribe) 스펙

VS Code나 Claude Desktop 같은 클라이언트는 보통 좌측에 프로젝트 리소스 트리를 렌더링하고, 현재 열려있는 탭 주변의 상태 변화에 민감하게 반응한다. MCP는 이를 지원하기 위해 명시적인 제어 엔드포인트를 제공한다.

  1. 옵트인(Opt-in) 기반 구독 명령: 에이전트가 README.md 파일을 읽기 전, 혹은 읽음과 동시에 resources/subscribe, uri="mcp://project/README.md" 메시지를 전송한다. 서버는 이 URI를 구독 수신자(Subscriber) 명단에 등록한다.
  2. 트리 노드(Tree Node) 확장을 통한 일괄 구독: 단일 파일뿐 아니라, mcp://project/src/ 처럼 특정 부모 디렉터리 자체를 구독하여 하위 요소를 모조리 감시망에 편입시키는 것 또한 가능하다.
  3. 가비지 컬렉션을 위한 구독 해지(Unsubscribe): 에이전트 컨트롤러가 해당 파일을 닫거나 다른 프로젝트로 전환하면, 반드시 resources/unsubscribe를 발송하여 서버 측 이벤트 발생기(Emitter)가 더 이상 무의미한 CPU 사이클을 낭비하지 않도록 구독 관계를 끊어낸다.
