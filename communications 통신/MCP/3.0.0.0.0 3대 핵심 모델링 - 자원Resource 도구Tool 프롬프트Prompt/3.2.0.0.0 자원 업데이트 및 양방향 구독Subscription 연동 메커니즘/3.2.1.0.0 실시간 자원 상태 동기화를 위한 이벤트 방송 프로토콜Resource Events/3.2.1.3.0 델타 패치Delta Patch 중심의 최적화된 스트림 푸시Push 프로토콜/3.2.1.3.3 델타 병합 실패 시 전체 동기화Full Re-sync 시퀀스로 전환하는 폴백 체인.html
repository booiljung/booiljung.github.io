3.2.1.3.3. 델타 병합 실패 시 전체 동기화(Full Re-sync) 시퀀스로 전환하는 폴백 체인

만일 네트워크 큐 오버플로우나 에이전트의 일시 정지로 인해 중간의 델타 패치가 누락되었다고 가정하자. 클라이언트가 패치를 적용하고 해시를 돌려봤으나, 서버가 명시한 tag와 해시가 불일치하는 무결성 붕괴(Integrity Breach) 사태가 터진다.

오염된 컨텍스트를 가지고 LLM이 추론을 강행하는 것은 끔찍한 오답을 낳으므로, 시스템은 즉시 전체 동기화(Full Re-sync) 폴백(Fallback) 시퀀스로 돌입한다.

  1. 오류 통지 및 캐시 폐기: 클라이언트 로직은 해시 불일치를 감지하는 즉시, 이 자원에 대해 메모리에 들고 있던 모든 캐시 데이터를 ‘오염됨(Corrupted)’ 상태로 마킹하고 폐기처분(Drop)한다.
  2. 명시적 풀 리딩(Full Read): 그 직후 에이전트는 일반적인 resources/read, uri="mcp://..." 호출을 서버로 날려서 전체 데이터(Full Payload)를 밑바닥부터 다시 퍼올린다.
  3. 태그 동기화 및 델타 수신 재개: 새로 받은 데이터 뭉치의 태그를 기준점(Baseline)으로 삼고, 이후부터 쏟아져 들어오는 델타 패치들을 다시 순차적으로 쌓아나간다. 즉, 델타 기반의 ’효율망’이 무너졌을 때는 가차 없이 ’안전망(Full-sync)’으로 떨어지는 고가용성 구조가 달성된다.
