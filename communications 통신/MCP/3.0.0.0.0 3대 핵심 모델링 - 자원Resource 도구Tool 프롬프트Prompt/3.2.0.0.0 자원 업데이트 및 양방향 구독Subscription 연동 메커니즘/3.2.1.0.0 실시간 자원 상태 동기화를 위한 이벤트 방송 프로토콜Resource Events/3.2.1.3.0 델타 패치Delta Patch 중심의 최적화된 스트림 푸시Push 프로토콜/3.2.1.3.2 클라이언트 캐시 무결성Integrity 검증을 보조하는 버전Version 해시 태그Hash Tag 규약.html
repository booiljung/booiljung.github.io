3.2.1.3.2. 클라이언트 캐시 무결성(Integrity) 검증을 보조하는 버전(Version) 해시 태그(Hash Tag) 규약

델타 캐스팅(Delta Casting)은 네트워크 비용을 비약적으로 줄여주지만, 패킷이 순서가 뒤집히거나 단발성 드롭으로 인해 단 하나의 델타 패치라도 유실되면, 클라이언트의 상태 트리는 영원히 서버와 엇갈려버리는(Desync) 돌이킬 수 없는 치명적인 약점을 지닌다.

이를 제어하기 위해 모든 자원 업데이트 패킷에는 버전 해시 태그(Version Hash Tag) 규약이 엄격하게 동봉된다.

  1. 상태 직렬화 해시(State SHA-256): 서버는 이벤트 푸시를 생성할 때, 단순히 델타 패치 기록만 넘기는 것이 아니라 수정된 이후의 ’완벽한 전체 파일’에 대한 무결성 해시값(Hash Tag)을 함께 계산하여 부착한다. (예: tag: "v-8a9b2c")
  2. 클라이언트 측 적용 후 자체 검증: 클라이언트 에이전트는 수신된 델타 패치를 자신의 캐시에 덮어씌운(Apply) 직후, 자신이 가진 전체 문자열 덩어리를 다시 해시 함수로 돌려 tag 값과 비교한다.
  3. 무결성 사슬 확립: 만약 계산된 해시가 서버가 보내준 해시와 정확히 일치한다면, 클라이언트는 “내부 캐시가 서버의 디스크 완전한 거울상이다“라는 100%의 수학적 확신(Mathematical Assurance)을 가지고 다음 추론 작업을 이어나갈 수 있다.
