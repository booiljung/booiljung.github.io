3.2.1.3.1. 전체 데이터 스냅샷 재송신 대신 파일의 변경점(Diff)만 압축 전송하는 델타 캐스팅

MCP의 고급 동기화 모드는 텍스트 데이터를 통째로 교환하는 대신, Git이 코드를 관리하듯 데이터의 차이점(Differential)만을 뽑아내어 **델타 캐스팅(Delta Casting)**을 진행한다.

  1. 마이어스 차이 알고리즘(Myers Diff Algorithm) 등재: 서버는 구독 중인 자원이 변경되면, 이전 상태의 스냅샷 캐시와 현재의 디스크 상태를 고속의 Diff 엔진(마이어스 알고리즘 등)에 태워 삭제된 행(-), 추가된 행(+)의 바이트 인덱스와 페이로드를 구한다.
  2. JSON 패치(JSON Patch / RFC 6902) 포맷 준수: 이 차이점은 단순히 두루뭉술한 지시가 아니라 "op": "replace", "path": "/content/15", "value": "updated_string"처럼 엄격하게 표준화된 JSON Patch 오퍼레이션 규격으로 압축 인코딩된다.
  3. 클라이언트 측 적용(Apply): 알림을 받은 클라이언트는 자신의 복사본 메모리 모델 위에서 이 패치 스크립트를 재현(Apply)한다. 네트워크를 타고 온 것은 불과 수십 바이트지만, 클라이언트의 상태는 1GB 단위의 서버 상태와 완벽한 거울상(Reflection)을 이루게 되는 마법이 성립된다.
