3.2. 자원 업데이트 및 양방향 구독(Subscription) 연동 메커니즘

MCP 환경에서 자원(Resource)은 결코 한 번 읽고 버려지는 화석이 아니다. 개발자가 코드를 수정하거나, 데이터베이스의 값이 백그라운드 스크립트에 의해 변경되는 등 현실 세계의 상태는 끊임없이 요동친다. 에이전트가 단발성으로 읽어들인(Pull) 컨텍스트만 믿고 추론을 장시간 이어가다가는, 오염된 과거 데이터에 기반해 환각(Hallucination)적인 답변을 쏟아내는 결과를 초래한다.

이러한 **컨텍스트 부패(Context Decay)**를 방지하기 위해, MCP는 서버가 능동적으로 상태 변화를 클라이언트 측에 밀어주는(Push) 양방향 스트리밍 동기화 체계, 즉 구독(Subscription) 연동 메커니즘을 프로토콜의 핵심 기둥으로 채택하였다.
