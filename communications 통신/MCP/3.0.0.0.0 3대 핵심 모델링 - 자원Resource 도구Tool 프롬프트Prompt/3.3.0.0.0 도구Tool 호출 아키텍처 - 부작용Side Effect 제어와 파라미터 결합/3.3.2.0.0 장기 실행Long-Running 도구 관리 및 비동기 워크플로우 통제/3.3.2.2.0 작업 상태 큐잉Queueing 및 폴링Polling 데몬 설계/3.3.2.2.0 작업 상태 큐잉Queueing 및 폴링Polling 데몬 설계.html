3.3.2.2. 작업 상태 큐잉(Queueing) 및 폴링(Polling) 데몬 설계

장기 실행 작업을 백그라운드로 밀어 넣었다면, 서버는 이를 안정적으로 수행할 체력이 필요하고, 클라이언트는 그 작업이 어디까지 왔는지 알아낼 권리가 있다.

이를 통제하는 것이 서버 내부의 **비동기 작업 큐잉 머신(Job Queueing Machine)**과 에이전트가 주기적으로 두드려볼 수 있는 정형화된 상태 폴링(Polling) 데몬 도구이다. 본 절에서는 내부 큐잉 구조와 외부 연계 도구의 페어링 설계를 기술한다.
