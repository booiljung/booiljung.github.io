3.3.2.2.1. 서버 측 백그라운드 워커에 적재된 작업 큐(Job Queue) 상태 머신

서버가 요청을 수락(Accepted)한 뒤 수행하는 백그라운드 작업은 불투명한 블랙박스여서는 안 된다. 작업의 한 걸음 한 걸음은 명확한 상태 전이(State Transition) 규칙을 따르는 유한 상태 기계(FSM) 위에서 굴러간다.

  1. 상태 노드(State Nodes): PENDING(대기 중) -> RUNNING(실행 중) -> RETRYING(일시적 오류로 재시도 중) -> COMPLETED(완료) 혹은 FAILED(영구 실패)의 5단계 기본 궤도를 갖는다.
  2. 전이 이벤트 브로드캐스팅: 워커 스레드가 상태를 변경할 때마다 그 결과값은 내부 데이터베이스나 Redis에 원자적(Atomic)으로 쓰인다.
  3. 영속성(Persistence): 백그라운드 작업 도중 서버가 셧다운되더라도, 이 작업 큐는 DB에 기록되어 있으므로 재부팅 시 워커가 고아 상태(Orphaned)의 RUNNING 작업들을 긁어모아 RETRYING으로 넘기며 불사신(Resilient)처럼 복구해 내는 설계가 반영된다.
