3.3.2.2.3. 백그라운드 작업 완료 상태 조회를 돕는 폴링 전용 도구(Polling Tool) 페어링 설계

작업을 백그라운드로 밀어 넣었다면, 에이전트는 어떻게 그 작업이 끝났는지 알 수 있을까? 물론 나중에 배울 ’능동적 알림(Push)’도 있지만, 가장 기본적인 접근은 에이전트가 생각날 때마다 물어볼 수 있도록 **폴링 툴(Polling Tool)**을 원-플러스-원(1+1)으로 제공하는 것이다.

  1. 페어링(Pairing)된 도구 셋 제공: 서버는 무거운 도구(예: train_model)를 제공할 때, 반드시 짝꿍이 되는 check_job_status 도구를 함께 스키마 명단에 묶어 제공한다.
  2. ID 기반 단일 질의: 에이전트는 여유가 될 때 (또는 사용자에게 진행률을 보고할 타임이 될 때) check_job_status(tracking_id: "job-1a2b")를 호출한다. 단 10ms 만에 큐잉 머신의 최신 State가 JSON으로 응답된다.
  3. 완료 후 결과물 발라내기(Unwrap): 만약 응답 코드가 COMPLETED라면, 이 폴링 도구는 단순히 문자열 상태만 알리는 것이 아니라 그 내부에 도구의 본래 결과물인 result_payload (예: “모델 학습 완료. 정확도 98%”)를 함께 동봉하여, 에이전트가 임무 완수에 필요한 지식 퍼즐 조각을 최종적으로 회수(Gather)하게끔 돕는다.
