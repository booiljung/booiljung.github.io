3.3.2.2.2. 외부 서비스 지연(API Rate Limit) 시 발생하는 쓰로틀링(Throttling) 패널티 유예 로직

에이전트가 요청한 장기 작업이 외부 클라우드의 무거운 API를 만 번 호출하는 것이라면, 외부 서버의 API Rate Limit(예: 429 Too Many Requests)에 걸릴 확률이 100%다.

백그라운드 큐는 이 경우 작업을 즉시 파기하지 않고, 지능적인 지수 백오프(Exponential Backoff) 유예 로직을 가동한다.

  1. Rate Limit 인터셉트: 워커 스레드가 외부 통신 중 429 에러나 503 Service Unavailable을 맞닥뜨리면, 진행 상황을 일시 정지(Freeze)하고 상태를 RETRYING으로 선언한다.
  2. 지수 백오프 휴면(Sleep): 워커는 쓰레드 폭주를 막기 위해 2초, 4초, 8초, 16초 단위로 점진적으로 증가하는 슬립 모드에 돌입한다. 이때 할당된 스레드는 점유를 풀고 풀(Pool)로 돌아가 다른 에이전트의 작업을 처리하는 논블로킹 전환을 시연한다.
  3. 에이전트 안심 전파: 이 유예 기간 중 에이전트가 상태를 물어보면 결코 Error를 던지지 않고 "Status: RETRYING. Rate limit hit. Will resume in 8 seconds."라는 섬세한 안내를 내려보내어, 에이전트가 불안감에 중복 호출(Panic Retry)을 일으키는 것을 달랜다.
