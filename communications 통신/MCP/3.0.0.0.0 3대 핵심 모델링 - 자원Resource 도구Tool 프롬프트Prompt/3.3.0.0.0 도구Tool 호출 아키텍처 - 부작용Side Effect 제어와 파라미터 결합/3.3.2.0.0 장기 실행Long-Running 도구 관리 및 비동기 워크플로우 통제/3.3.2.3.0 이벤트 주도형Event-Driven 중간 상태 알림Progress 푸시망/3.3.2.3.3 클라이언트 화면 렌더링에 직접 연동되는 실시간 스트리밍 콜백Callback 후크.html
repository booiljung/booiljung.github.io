3.3.2.3.3. 클라이언트 화면 렌더링에 직접 연동되는 실시간 스트리밍 콜백(Callback) 후크

수신된 진행률(Progress) 이벤트는 LLM의 컨텍스트 창(Context Window)을 낭비하지 않는 편이 좋다. “45% 진행됨”, “46% 진행됨” 과 같은 텍스트 조각을 컨텍스트에 꾸역꾸역 집어넣는 순간 LLM의 프롬프트 한도가 초과되어 버리기 때문이다.

  1. 프롬프트 우회 바이패스(Bypass): MCP 클라이언트 프레임워크(예: Claude Desktop)는 job_progress 알림을 수신하면 이를 LLM에게 넘기지 않고(Hide from Agent), 곧바로 GUI 컨트롤러 계층으로 우회시킨다.
  2. 리액티브 프론트엔드 연동: 프론트엔드의 React 훅(Hook)이나 네이티브 UI 스레드는 이 값을 구독받아, 대화창 컴포넌트 바로 밑에 시각적인 프로그레스 바(Progress Bar, ▓▓▓▓▓░░░░░)나 스피너(Spinner)를 실시간으로 다시 그린다(Re-render).
  3. 투명한 인지 분리(Cognitive Separation): 이 완벽한 콜백 후크 아키텍처 덕분에, LLM은 “기울어진 시스템 상태“를 글로 읽느라 뇌(토큰)를 낭비하지 않고, 사용자(Human User)는 직관적인 GUI를 통해 서버 저편에서 수행되는 물리적 작업 진행도를 시각적으로 관음(Observe)할 수 있는 완벽한 인지적 분리 상태를 이루게 된다.
