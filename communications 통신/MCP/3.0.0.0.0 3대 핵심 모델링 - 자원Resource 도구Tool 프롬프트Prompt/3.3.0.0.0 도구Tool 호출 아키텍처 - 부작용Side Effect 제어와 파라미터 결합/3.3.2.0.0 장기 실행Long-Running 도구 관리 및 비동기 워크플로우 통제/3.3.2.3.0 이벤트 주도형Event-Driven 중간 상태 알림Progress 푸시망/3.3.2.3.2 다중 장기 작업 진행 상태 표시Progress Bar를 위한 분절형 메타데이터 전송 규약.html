3.3.2.3.2. 다중 장기 작업 진행 상태 표시(Progress Bar)를 위한 분절형 메타데이터 전송 규약

사용자(Human)는 터미널에 에이전트가 1시간 동안 아무런 로그도 남기지 않으면 “이게 죽은 건가 아니면 워커가 돌아가고 있는 건가?“라는 극심한 불안감에 시달린다.

이를 해소하기 위해, 서버 워커는 단순히 시작과 끝만 알리는 것이 아니라 **중간 진행률(Progress)**을 분절하여 실시간으로 보고해야 한다.

  1. 진행률 토큰 생성: 100만 건의 DB 레코드를 마이그레이션 하는 도구라면, 내부 카운터가 1만 건을 넘을 때마다 notifications/tools/job_progress 이벤트를 발송한다.
  2. 메타데이터 스키마: 이 이벤트 패킷은 단순히 텍스트만 갖는 것이 아니라, {"job_id": "...", "progress_percent": 45, "current_step": "Migrating Table Users", "estimated_time_remaining": "10m" } 라는 정형화된 JSON 메타데이터 덩어리를 싣고 나른다.
  3. 네트워크 최적화: 0.1% 오를 때마다 푸시를 쏘면 네트워크가 타버리므로, 보통 1% 단위 혹은 2초 단위의 디바운싱(Debouncing) 간격을 두어 진행 알림을 분절 송출(Chunked Emit)한다.
