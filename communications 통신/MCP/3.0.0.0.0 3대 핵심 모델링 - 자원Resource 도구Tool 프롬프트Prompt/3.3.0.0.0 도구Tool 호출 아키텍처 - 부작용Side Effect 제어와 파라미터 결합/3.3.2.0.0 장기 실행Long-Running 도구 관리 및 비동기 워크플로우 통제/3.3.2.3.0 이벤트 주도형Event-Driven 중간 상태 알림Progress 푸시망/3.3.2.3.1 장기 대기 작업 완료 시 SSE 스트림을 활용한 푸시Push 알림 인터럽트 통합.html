3.3.2.3.1. 장기 대기 작업 완료 시 SSE 스트림을 활용한 푸시(Push) 알림 인터럽트 통합

MCP는 기본적으로 양방향 연결(JSON-RPC 기반 SSE나 Stdio) 뼈대를 가지고 있으므로, 굳이 에이전트가 물어볼 때까지 기다릴 필요가 없다.

서버 측 워커 스레드가 작업을 최종적으로 마쳤거나 영구적으로 실패했을 때, 감시자(Watcher) 데몬은 즉시 notifications/tools/job_completed 라는 특수한 원격 프로시저를 에이전트를 향해 쏘아 보낸다.

  1. 상태 전환 후크(Post-Transition Hook): 큐잉 엔진의 FSM이 COMPLETED 노드에 도달하면, 이를 낚아채는 아웃바운드 푸시 브로커가 작동한다.
  2. 비동기 개입(Asynchronous Interruption): 에이전트가 다른 일을 추론하며 텍스트를 생성하고 있는 와중이라도, 클라이언트 사이드 스트림 제어기는 들어오는 이 완료 이벤트를 중간(Interrupt)에 끼워 넣거나 메인 컨텍스트 윈도우 하단에 강제 주입(Inject)시킨다.
  3. 루프 부활(Re-awakening): “당신이 맡겼던 ID: job-1a2b의 결과값을 받아왔다“는 사실을 깨달은 LLM은 진행 중이던 사고를 잠시 멈추고 (또는 마무리 지은 뒤), 그 결과 데이터를 토대로 본래 의도했던 메인 태스크(예: 최종 보고서 작성)를 막힘없이 이어나간다.
