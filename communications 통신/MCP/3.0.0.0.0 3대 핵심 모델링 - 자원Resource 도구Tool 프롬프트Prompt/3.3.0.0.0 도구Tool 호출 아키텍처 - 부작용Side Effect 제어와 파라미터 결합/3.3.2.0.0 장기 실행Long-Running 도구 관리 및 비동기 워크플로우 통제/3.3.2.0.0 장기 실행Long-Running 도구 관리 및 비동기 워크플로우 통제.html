3.3.2. 장기 실행(Long-Running) 도구 관리 및 비동기 워크플로우 통제

대규모 데이터베이스 마이그레이션 도구나, 기계학습 모델 학습 파이프라인처럼 완료까지 수 분에서 수 시간이 걸릴 수 있는 도구를 동기(Synchronous) 방식으로 호출하게 되면 네트워크 커넥션은 타임아웃(Timeout)을 뱉고, LLM 컨텍스트는 추론을 멈춘 채 좀비(Zombie) 상태에 빠져버린다.

이를 막기 위해 MCP는 무거운 작업을 호출 즉시 백그라운드 큐(Queue)로 던져버리고 비동기(Asynchronous) 식별자만 교환한 뒤, 향후 상태를 역추적하는 장기 실행 워크플로우(Long-Running Workflow) 패턴을 필수 아키텍처로 내재화했다. 본 섹션에서는 이 비동기 처리가 가져다주는 아키텍처적 우위성과 그 동작 원리를 살펴본다.
