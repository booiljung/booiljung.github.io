3.3.1.1.1. 데이터 타입(Type) 엄격 강제 및 필수 인자(Required) 식별 기반 할루시네이션(Hallucination) 원천 방어

대규모 언어 모델(LLM) 기반 에이전트는 창의적인 사고방식을 가졌기에, 명확한 제약이 없다면 인자값을 임의로 생략하거나 형태를 비틀어버리는 치명적 단점을 지닌다. “문서 아이디“를 달라고 했더니 "id": "문서 번호는 3번입니다"라고 응답하는 식이다.

서버는 이 환각을 방어하기 위해 JSON Schema의 규격을 극한으로 끌어올린다.

  1. 원시 데이터 타입(Primitive Type) 바인딩: 각 매개변수 속성(Property)에 대해 type: "string", type: "integer", type: "boolean" 등으로 엄격하게 벽을 세운다. LLM은 이 스키마 텍스트를 컨텍스트 윈도우로 읽어 들인 순간부터 내부적인 문법 제약(Grammar Constrain) 추론 프로세스를 가동하여 결과물 포맷을 스스로 억제한다.
  2. 필수(Required) 필드 강제화: ["filePath", "content"]와 같이 이 도구를 실행하기 위해 절대적으로 필요한 필드를 required 배열에 명시한다. 이를 본 에이전트는 모르는 값이 있더라도 함부로 실행을 누르지 않고, 오히려 사용자(Human)에게 “작업을 위해 content 내용이 필요합니다“라고 역명오를 내리게 된다.
  3. 가드레일 열거형(Enum Pattern): 더욱 제한이 필요한 파라미터에는 아예 선택지 목록(["CREATE", "UPDATE", "DELETE"])을 하드코딩해 던져주어, 벗어난 문자열이 토큰화되는 것을 원천적으로 가로막는다.
