3.3.1.1.2. 도구 설명(Description) 문맥 정밀화를 통한 LLM 관점의 의도 일치(Intent Matching) 척도 향상

기존의 API 설계에서 Description 필드는 ’인간 개발자’를 위한 주석에 불과했다. 하지만 MCP 아키텍처에서 Tool.description은 에이전트의 신경망 속에서 **“이 상황에 이 도구를 써도 되는가?”**를 결정짓는 핵심 가중치(Routing Weight)로 작용한다.

  1. 시맨틱 라우팅(Semantic Routing): LLM은 사용자 프롬프트와 여러 도구의 Description 간의 벡터 유사도(Vector Similarity)와 맥락적 함의를 분석해 도구를 선택한다. 따라서 Description이 “사용자를 생성합니다“처럼 파편적이면 오작동 확률이 높아진다.
  2. 정밀한 컨텍스트 인젝션: MCP 도구 선언망에는 “사용자 정보를 시스템 관리자 권한으로 추가합니다. 단, 게스트 계정 생성에는 사용하지 마십시오.“와 같이 제한 사항(Constraints)과 용법(Usage)을 매우 촘촘한 자연어로 서술하는 패턴이 정착되어 있다.
  3. 각 파라미터별 독립적 설명 부여: 최상위 도구의 설명뿐만 아니라, JSON Schema 내의 개별 필드(예: timeout_seconds)에도 description: "기본값은 30초입니다. 대용량 다운로드 시에는 120초 이상을 권장합니다"를 덧붙여 LLM이 엣지 케이스(Edge Case)를 스스로 피해 나가도록 정밀히 유도한다.
