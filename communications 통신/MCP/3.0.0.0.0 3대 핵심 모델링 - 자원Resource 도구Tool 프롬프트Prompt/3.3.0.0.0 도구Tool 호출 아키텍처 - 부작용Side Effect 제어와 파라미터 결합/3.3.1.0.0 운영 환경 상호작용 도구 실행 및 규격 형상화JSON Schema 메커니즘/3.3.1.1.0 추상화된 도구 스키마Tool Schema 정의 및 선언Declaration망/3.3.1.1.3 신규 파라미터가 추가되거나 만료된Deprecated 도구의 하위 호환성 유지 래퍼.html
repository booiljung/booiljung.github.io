3.3.1.1.3. 신규 파라미터가 추가되거나 만료된(Deprecated) 도구의 하위 호환성 유지 래퍼

서버가 지속적으로 진화함에 따라 도구의 스펙도 바뀐다. 기존에 createTask(name)만 받던 도구가 갑자기 createTask(name, deadline)으로 바뀌었다면, 길게는 며칠 전에 열려있던 세션의 에이전트는 바뀐 스펙을 모른 채 과거의 형태로 호출을 시도하여 Missing Required Parameter 에러를 터트리게 될 위험이 있다.

  1. 더미 파라미터 주입(Dummy Parameter Injection): 파라미터가 추가될 때 초기에는 이를 required 목록에 넣지 않고 선택적(Optional)으로 둔 뒤 파싱 로직에서 기본값(Default fallback)을 할당한다.
  2. 데프리케이션(Deprecation) 플래그 알림: 기존 도구가 만료(Deprecated) 예정일 경우, 도구의 메타데이터나 Description 꼬리말에 [DEPRECATED: Use 'create_task_v2' instead] 문구를 동적으로 밀어 넣어 LLM이 다음 호출부터 자발적으로 신규 버전을 라우팅하도록 소프트 브레이킹(Soft Breaking)을 유도한다.
  3. 스키마 매핑 래퍼(Wrapper): 불가피한 브레이킹 체인지(Breaking Change)가 있는 경우, 내부적으로 v1 어댑터를 살려두어 에이전트가 예전 포맷으로 요청했을 때 서버 측 컴파일러 패턴 파서가 현대적 스펙으로 호환성 역-치환(Backporting)을 단행해 버리는 투명망을 도입한다.
