3.3.1.2.1. LLM이 생성한 JSON 인자값에 대한 스키마 적합성 선행(Pre-execution) 밸리데이션

에이전트로부터 전달받은 tools/call의 페이로드가 곧바로 비즈니스 컨트롤러로 넘어가는 일은 결코 일어나지 않는다. 가장 앞단에 위치한 **JSON Schema Validator (예: Ajv, Zod)**가 길목을 지키고 선다.

  1. 구조적 토폴로지 매칭: 전송된 매개변수 맵(Map)이 프로토콜상 정의된 스키마 트리의 윤곽(Shape)과 정확히 맞물려 들어가는지 확인한다. 요구되지 않은 추가 필드(additionalProperties)가 딸려온 경우, 이를 무자비하게 제거(Strip)하여 쓸데없는 사이드 이펙트를 밀봉한다.
  2. 정규형 속성 정밀 횡단(Traversal): 문자열의 maxLength: 255, 숫자의 minimum: 0과 같이 디테일하게 정의된 바운더리를 확인한다. 만일 LLM이 환각에 취해 나이에 -10을 할당했다면 프로그램 크래시를 기다릴 것도 없이 여기서 즉각 처단된다.
  3. 신속한 피드백 루프 트리거링: 선행 밸리데이션에서 실패할 경우 실행 로직을 0.1ms도 낭비하지 않은 상태에서 에이전트에게 InvalidParams 예외를 JSON 포맷으로 패킹하여 송출한다. 이를 받은 에이전트는 내부적으로 “아, 내가 정규식을 위반했구나“를 스스로 판단하고 수정된 페이로드를 조립해 즉시 **재귀 호출(Recursive Call)**을 시도하는 자기 치유(Self-healing) 프로세스가 이루어진다.
