3.3.1.2. 런타임 매개변수(Parameter) 유효성 검사 및 정제(Sanitization)

아무리 정교하게 JSON Schema를 선언해 두었다 한들, LLM 모델 단에서 생성(Generate)되는 토큰 스트림을 100% 통제할 수는 없다. 에이전트 모델의 온도가 조금만 높아져도 name: "Alica" 대신 name: "<script>alert(1)</script>"나 {"cmd": "rm -rf /"} 처럼 파멸적인 명령을 텍스트 안에 섞어 보낼 위험을 내포하고 있다.

따라서 MCP 서버 모듈이 그 인자를 넘겨받아 ‘방아쇠를 당기기 직전’, 반드시 중간 게이트웨이에서 파라미터 **정제(Sanitization)**와 **초엄격 유효성 검사(Validation)**를 런타임에 동적으로 수행해야만 한다.
