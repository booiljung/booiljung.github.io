3.3.1.2.2. 악의적 문자열 포함 및 포맷 위반 시 초기 탈락(Fast-Fail)을 유도하는 타입 캐스팅 월(Cast Wall)

단순히 스키마의 “구조“만 검사하는 것으로는 보안이 확보되지 않는다. type: "string"이라고 명시해 두었더니 에이전트가 <script>alert("hacked")</script>를 보내거나 파일 경로에 ../../../etc/passwd라는 무시무시한 디렉터리 탐색 문자열을 던질 수 있다.

  1. 정규 표현식 방어막(Pattern Constraints): filePath와 같은 치명적인 매개변수 스키마에는 pattern: "^[a-zA-Z0-9_\\-\\./]+$" 와 같이 정규식 제약 조건을 아주 촘촘하게 추가하여, 시스템 예약어나 디렉터리 우회 문자열(..)의 주입을 입구컷(Fast-Fail)시킨다.
  2. 이스케이프(Escaping) 및 새니타이즈(Sanitizing): 밸리데이터를 통과했더라도 쉘이나 SQL로 내려가기 전, 서버의 타입 캐스팅 월(Type Cast Wall)은 rm -rf 같은 명령어가 섞여 있을지 모를 문자열을 특수문자 이스케이프 함수에 통과시켜 안전한 원시(Raw) 리터럴로 치환해 버린다.
  3. 도메인 특화 타입 캐스팅(Domain Type Casting): "false"라는 문자열을 boolean 타입으로 억지로 파싱해 내는 관대한 방식 대신, 엄격한 언어 체계(Rust, Go)로 짜인 백엔드 환경에서는 입력 형 변환 불가 시 프로그램 내부 깊숙이 침투하기 전에 경계선(Boundary)에서 즉시 요청을 포기(Abort)토록 설계된다.
