3.3.1.3.2. 실행 의존성을 갖는 도구 묶음의 순차적(DAG) 트랜잭션 라우팅

모든 도구가 항상 독립적일 수는 없다. “파일을 생성하고(A) -> 빈 파일에 텍스트를 쓴다(B)“라는 질의를 에이전트가 동시에 병렬 토큰으로 던졌을 때, 서버가 B를 먼저 실행해 버리면 FileNotFound 에러가 발생한다.

따라서 고급 MCP 브로커는 단순한 Scatter가 아니라 DAG(Directed Acyclic Graph) 기반 트랜잭션 라우팅을 기저에 둔다.

  1. 의존도 자가 진단(Self-Dependency Check): 에이전트가 여러 도구를 동시에 던질 때, 파라미터 내부에 _depends_on: ["call_A_id"] 형태의 백도어 메타데이터를 끼워 넣을 수 있도록 프로토콜을 확장한다.
  2. 위상 정렬(Topological Sort) 큐잉: 라우터는 요청된 M개의 툴을 무식하게 스레드 풀에 던지는 대신, 의존성 그래프를 그려 위상 정렬을 수행한다. 선행 파일 생성이 끝나기 전까지 쓰기 작업 스레드는 락(Lock)을 걸고 대기(Await)한다.
  3. 부분 실패(Partial Failure) 전파망: 만일 선행 노드(A)가 권한 부족으로 실패했다면, 후행 노드(B)는 굳이 I/O 자원을 소모하며 실행할 필요가 없다. 라우터는 B를 즉각적으로 Cascading Failure 처리하고 에이전트에게 트랜잭션의 어떤 포인트에서 연결 고리가 끊어졌는지 낱낱이 보고한다.
