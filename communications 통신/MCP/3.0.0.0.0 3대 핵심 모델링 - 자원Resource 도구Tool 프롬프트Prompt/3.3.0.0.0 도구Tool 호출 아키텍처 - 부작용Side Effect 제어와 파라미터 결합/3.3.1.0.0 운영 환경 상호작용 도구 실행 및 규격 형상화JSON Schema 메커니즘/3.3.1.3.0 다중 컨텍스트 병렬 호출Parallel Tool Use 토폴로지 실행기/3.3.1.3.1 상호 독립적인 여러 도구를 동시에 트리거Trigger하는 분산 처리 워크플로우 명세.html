3.3.1.3.1. 상호 독립적인 여러 도구를 동시에 트리거(Trigger)하는 분산 처리 워크플로우 명세

LLM은 사용자 질의를 종합적으로 파악한 후, “이 정보와 저 정보가 동시에 필요하다“고 판단하여 단발성 JSON 배열 형태([ { call A }, { call B }, { call C } ])로 MCP 서버에 폭격하듯 도구 토큰을 반환한다.

  1. Scatter & Gather 토폴로지: MCP 클라이언트 런타임은 이 도구 배열을 받는 즉시 비동기 코루틴(Async Coroutines)이나 스레드 풀(Thread Pool)을 열어 각 도구를 동시 출발점(Scatter)에서 격발시킨다.
  2. 독립된 샌드박스 실행: 함수 A(날씨 API)와 B(주가 DB 조회)는 서로 간섭 없이 완전히 독립적인 I/O 파이프라인과 트랜잭션 수명 주기를 배정받아 실행된다.
  3. 결과 집계(Aggregation): 모든 도구의 실행이 완료되거나 타임아웃이 나면, 다시 하나의 큰 call_results 배열 텍스트 블록으로 모여져서(Gather), 오직 한 번의 단일 피드백 메시지로 에이전트의 컨텍스트 창에 복원 제출된다. 이는 전체 추론 루프(Reasoning Loop) 턴을 극적으로 절약한다.
