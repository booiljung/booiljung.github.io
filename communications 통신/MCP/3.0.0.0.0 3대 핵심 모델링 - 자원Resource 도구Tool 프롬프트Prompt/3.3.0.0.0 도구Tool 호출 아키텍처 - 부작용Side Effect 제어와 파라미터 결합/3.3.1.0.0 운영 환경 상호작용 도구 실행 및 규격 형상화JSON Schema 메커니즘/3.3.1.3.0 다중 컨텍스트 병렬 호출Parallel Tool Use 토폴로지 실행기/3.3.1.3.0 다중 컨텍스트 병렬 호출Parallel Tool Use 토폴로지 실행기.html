3.3.1.3. 다중 컨텍스트 병렬 호출(Parallel Tool Use) 토폴로지 실행기

LLM은 한 번 추론을 할 때 시간(Thinking Time)이 꽤 걸리며, 토큰당 과금이 매겨진다. 만약 사용자 프로필을 조회하고(A), 날씨를 확인하고(B), 주가를 검색하는(C) 세 가지 툴이 필요할 때 이를 A 호출 -> 대기 -> B 호출 -> 대기 -> C 호출 식으로 직렬 수행하면 사용자 응답 시간은 영원히 지연된다.

MCP와 현대 LLM(Claude 3.5 Sonnet 등)은 이 병목을 분쇄하기 위해 **병렬 도구 사용(Parallel Tool Use)**을 네이티브로 지원한다. 본 챕터에서는 에이전트가 단일 턴(Turn) 안에 쏟아내는 동시다발적 도구 호출 요청을 시스템이 어떻게 엮어내는지 그 토폴로지와 스레드 격리망을 추적한다.
