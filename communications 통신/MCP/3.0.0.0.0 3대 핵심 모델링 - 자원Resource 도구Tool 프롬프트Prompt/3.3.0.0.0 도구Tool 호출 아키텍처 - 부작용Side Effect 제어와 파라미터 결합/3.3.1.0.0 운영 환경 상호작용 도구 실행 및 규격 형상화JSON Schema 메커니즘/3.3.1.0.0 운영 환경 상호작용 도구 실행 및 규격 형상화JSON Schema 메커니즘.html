3.3.1. 운영 환경 상호작용 도구 실행 및 규격 형상화(JSON Schema) 메커니즘

에이전트가 도구를 마음대로 호출하게 내버려 두면, 그들은 종종 요구하지 않은 불필요한 인자를 던지거나, 숫자를 요구하는 자리에 자연어 문자열을 집어넣는 환각(Hallucination) 현상을 일으킨다.

이를 막기 위해, MCP 시스템은 에이전트와 도구 사이에 강력한 계약서인 제이슨 스키마(JSON Schema) 형상화(Formulation) 메커니즘을 둔다. 에이전트가 도구를 호출하기 전부터, 서버는 도구의 모양과 규칙을 매우 세밀한 계약서 형태로 선언하여 에이전트의 뇌리(Context)에 박아 넣는다.
