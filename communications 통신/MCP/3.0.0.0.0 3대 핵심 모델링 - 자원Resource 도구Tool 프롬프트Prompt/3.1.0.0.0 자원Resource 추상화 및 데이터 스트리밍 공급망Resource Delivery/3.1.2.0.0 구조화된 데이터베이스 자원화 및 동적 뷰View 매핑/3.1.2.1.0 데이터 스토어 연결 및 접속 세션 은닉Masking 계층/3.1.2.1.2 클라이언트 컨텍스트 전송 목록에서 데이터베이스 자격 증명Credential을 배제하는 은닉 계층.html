3.1.2.1.2. 클라이언트 컨텍스트 전송 목록에서 데이터베이스 자격 증명(Credential)을 배제하는 은닉 계층

에이전트가 resources/list 같은 탐색 명령을 내리면, 서버는 현재 연결 가능한 자원 공급원(예: 접속된 데이터베이스 목록)의 메타데이터를 내려준다. 이때 개발자의 부주의나 프레임워크의 직렬화(Serialization) 오류로 인해, DSN(Data Source Name)이 통째로 JSON 필드에 말려 들어가는 치명적 사고가 발생할 수 있다.

이를 방지하기 위해 직렬화 파이프라인의 가장 아랫단에는 강제적 자격 증명 배제(Credential Redaction) 필터가 적용된다.

  1. 기밀 변수 블랙리스트(Blacklist) 선언: 서버의 환경 변수로 설정된 POSTGRES_PASSWORD나 AWS_ACCESS_KEY 등의 문자열 값 자체를 정규식 블랙리스트 버퍼에 미리 로드(Load)한다.
  2. 반환 페이로드 스캐닝: 에이전트로 향하는 모든 resources/list 반환 JSON의 값(Value)들을 순회하며, 블랙리스트에 등록된 문자열과 정확히 일치하거나 비슷한 포맷의 정규식 구문이 존재하는지 능동적으로 스캔한다.
  3. 자동 마스킹(Auto-masking): 만약 실수로 포함된 자격 증명이 발견되면 즉시 [REDACTED_BY_MCP] 혹은 ******** 라는 더미 텍스트로 치환해 버린다. 이는 단순히 화면에 보이지 않게 하는 것이 아니라, 전송되는 네트워크 바이트 단위에서 정보를 삭제하여 패킷 스니핑마저 무력화시킨다.
