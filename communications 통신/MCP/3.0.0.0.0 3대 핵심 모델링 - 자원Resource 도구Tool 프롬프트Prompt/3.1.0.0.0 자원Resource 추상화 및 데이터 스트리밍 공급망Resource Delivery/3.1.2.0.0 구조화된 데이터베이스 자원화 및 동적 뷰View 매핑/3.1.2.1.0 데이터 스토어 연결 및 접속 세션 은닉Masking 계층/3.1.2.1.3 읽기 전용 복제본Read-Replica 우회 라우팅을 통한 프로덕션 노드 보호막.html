3.1.2.1.3. 읽기 전용 복제본(Read-Replica) 우회 라우팅을 통한 프로덕션 노드 보호막

AI 에이전트의 특징 중 하나는 “가끔 통제 불능 상태로 폭주하여 생각지도 못한 방대한 범위의 데이터 쿼리를 요청할 수 있다“는 점이다. 이러한 폭주 쿼리가 실제 고객들이 결제를 진행하고 있는 메인 데이터베이스(Primary Node)로 직접 꽂힌다면, 전체 프로덕션 서비스가 정지되는 참사가 발생한다.

MCP 서버의 데이터베이스 연결 계층은 이러한 “에이전트발 인프라 사고“를 막기 위해 지능형 **복제본 우회 라우터(Replica Bypass Router)**를 가동한다.

  1. 물리적 엔드포인트 격리: MCP 애플리케이션의 DB 연결 문자열(DSN)은 설계 시점부터 프라이머리(Primary) 쓰기 노드가 아닌, 10초 정도의 복제 시차(Replication Lag)를 가진 읽기 전용(Read-only) 노드를 기본 엔드포인트로 설정하도록 강제된다.
  2. 트래픽 분산과 방탄막: 에이전트가 미쳐 날뛰어 수만 건의 대규모 Select 쿼리(예: “모든 사용자의 5년 치 구매 이력 분석”)를 요청하더라도, 그 부하는 오로지 읽기 전용 복제본의 CPU/메모리 만을 소진시킨다.
  3. 읽기 지연(Staleness) 수용성 모델: 비록 에이전트가 “가장 최신의 1초 전 데이터“를 보지 못할지라도, 코드 분석이나 로그 탐색 같은 일반적인 인공지능 추론 임무에서는 몇 초의 Lag이 치명적인 오류를 낳지 않는다. 즉, 극미한 데이터 지연을 희생하여 전체 프로덕션 시스템의 절대적 가용성(Availability)을 수호하는 트레이드오프(Trade-off) 철학이 배어있다.
