3.1.2.3.1. SQL 주입(SQL Injection) 차단을 위한 매개변수화(Parameterized) 쿼리 강제 검증망

AI 에이전트가 직접 완전한 조작 능력을 갖춘 날것의 SQL(Raw SQL) 쿼리 문자열 전체를 작성해서 넘기도록 허용하는 것은, 보안과 시스템 안정성을 통째로 포기하는 것과 같다. 이 경우 전통적인 웹 공격인 SQL Injection의 타겟이 되기 쉽다.

따라서 MCP의 쿼리 래퍼 도구(Query Wrapper Tool)는 문자열 결합 필터링을 포기하고, 철저한 매개변수화 쿼리(Parameterized Query) 강제화 정책을 채택한다.

  1. 사전 정의된 쿼리 스켈레톤(Skeleton): 서버는 에이전트에게 “사용자 검색 도구“를 제공할 뿐이다. 내부적으로는 SELECT * FROM users WHERE name = ? 라는 스켈레톤 구문을 미리 컴파일(Pre-compiled) 상태로 쥐고 있다.
  2. 파라미터 분리 파싱: 에이전트는 이 도구를 호출할 때, { "name": "Alice'; DROP TABLE users; --" } 라는 JSON 파라미터 객체만 인계한다. 도달한 파라미터는 쿼리 코드로 해석되지 않고 순수한 문자열 상수(Literal)로만 평가된다.
  3. AST(Abstract Syntax Tree) 검증: 그럼에도 불구하고 동적 SQL 생성 도구를 부득이하게 열어준 경우, 서버는 에이전트가 전송한 SQL 텍스트를 즉시 실행하지 않고, 데이터베이스 파서 블록을 활용해 AST를 추출한다. 트리 내부에서 DELETE, DROP, ALTER 같은 변이(Mutation) 노드가 발견되면 즉시 해당 쿼리를 소각(Drop)하고 QueryExecutionError를 에이전트에 통보하여 학습시킨다.
