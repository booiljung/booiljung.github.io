3.1.2.2.2. 그래프 DB(Node-Edge) 및 NoSQL 비정형 구조의 평탄화(Flattening) 해석기

구조가 명확한 RDBMS와 달리 MongoDB나 Neo4j 같은 NoSQL/Graph DB 시스템은 스키마가 없거나(Schemaless), 노드 간의 엣지(Edge) 연결이 다차원적(Multi-dimensional) 복잡계 네트워크를 이룬다.

에이전트에게 이런 다차원 데이터를 3차원 상태로 제공하면 토큰 공간이 비효율적으로 낭비되므로, 이를 **평탄화(Flattening)**하는 기법이 동원된다.

  1. 샘플링 및 덕 타이핑(Duck Typing): 스키마가 없는 NoSQL의 경우 서버는 최근 100개의 문서를 랜덤 샘플링하여, 빈번하게 등장하는 키(Key) 값들을 수집하는 덕 타이핑 기법으로 가상의 스키마 구조를 유추해 낸다.
  2. 비정형 객체의 깊이 평탄화(Depth Flattening): JSON 문서 내부에 객체가 5-Depth까지 깊숙하게 엮여있다면, 서버의 해석기는 이를 user.address.street.name과 같은 점 표기법(Dot-notation) 기반 1-Depth 평탄화 배열로 풀어헤친다.
  3. 엣지 관계의 표 형식 치환: 그래프 DB의 릴레이션(Relation)은 [Node A] -> [Edge Action] -> [Node B] 형태의 트리플릿(Triplet) 레코드로 치환되어, 에이전트가 “출발지, 연관성, 목적지“라는 마치 RDBMS의 조인(Join) 테이블을 보듯 직관적으로 인식하도록 시각적 기만을 유도한다.
