3.1.2.2.3. 추출된 스키마 메타데이터의 LLM 친화적 의존성 그래프 마크다운 생성법

서버가 훌륭하게 메타데이터를 정규화했더라도, 이를 JSON 그 자체로 LLM 에이전트에게 건네주면 토큰 인코딩 비용이 폭증하고 컨텍스트 리딩 성능이 급감한다. 대규모 언어 모델은 괄호({, })나 따옴표가 난무하는 JSON 구조보다, 자연어로 기술된 설명과 리스트 기반의 마크다운(Markdown) 문서 형태에 훨씬 더 뛰어난 추론 성능을(Reasoning Edge) 보장하기 때문이다.

따라서 MCP 자원 포매터(Formatter)는 LLM 친화적 마크다운 직렬화를 수행한다.

  1. 테이블 스펙의 마크다운 표 치환: 각 테이블은 마크다운 헤더(# 테이블 이름)로, 컬럼들은 각주와 타입이 명시된 마크다운 테이블 형식으로 치환된다.
    (예: | 컬럼명 | 타입 | PK/FK | 설명 |)
  2. Mermaid 기반 릴레이션 다이어그램 생성: 복잡한 외래 키(FK) 연결 관계는 단순히 텍스트로 풀어쓰는 것을 넘어, 마크다운 내에 삽입 가능한 mermaid 형식의 ER-Diagram 코드로 역-렌더링된다. 에이전트는 이를 보고 테이블 간의 계층적 연관성을 거시적인 관점에서 도출해 낼 수 있다.
  3. 컨텍스트 청킹 스키마 공급: 만약 테이블이 수백 개 단위라면 이 마크다운 문서마저도 거대해지므로, 앞서 언급된 자원 청킹(Chunking) 기법(3.1.1.3)을 적용해 주제별로 쪼개진 모듈형태 스키마 문서들로 잘라서 배급한다.
