3.1.2.4.1. 사용자 역할(RBAC) 태그 기반 행(Row)-열(Column) 레벨 마스킹 처리 인터페이스

에이전트에게 할당된 세션 토큰은 철저한 역할 기반 접근 통제(Role-Based Access Control, RBAC) 속성을 지닌다. “단순 코드 분석기” 에이전트와 “총무/재무 결산 보조” 에이전트는 동일한 users 테이블을 조회하더라도 반환받는 데이터 스트림의 형태가 확연히 달라야 한다.

이를 가능케 하는 것은 다차원 레코드 마스킹(Multi-dimensional Record Masking) 인터페이스다.

  1. 컬럼 레벨 권한 검증(Column-Level Security): 쿼리 실행 직전, 추출기(Extractor)는 에이전트의 권한 태그 딕셔너리를 검사한다. ‘재무’ 권한이 없는 에이전트의 쿼리 결과셋에서 credit_card_hash 및 ssn 컬럼은 서버가 메모리에 로드하기도 전에 데이터베이스 드라이버 레벨에서 투사(Projection) 조건이 필터링되어 삭제(Drop)된다.
  2. 로우 레벨 지오-펜싱(Row-Level Security): 데이터 행위를 제한하는 태그(예: tenant_id=5)가 에이전트에 부여되어 있다면, 서버는 백그라운드 쿼리 조작기(Query Mutator)를 통해 WHERE tenant_id = 5 조건을 강제로 주입한다. 이로써 타 조직의 지식 데이터 레코드는 추출 원천 단계에서 원천 봉쇄된다.
  3. 가변적 동적 치환(Dynamic Substitution): 아예 지워버리면 컨텍스트 흐름상 어색할 경우, ssn 컬럼의 값을 [RESTRICTED_BY_SERVER] 등의 경고 메시지로 치환하여, LLM에게 “이 데이터는 존재하나 당신의 권한으로 읽을 수 없다“는 명백한 추론 근거를 제공한다.
