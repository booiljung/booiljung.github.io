3.1. 자원(Resource) 추상화 및 데이터 스트리밍 공급망(Resource Delivery)

에이전트가 현실 세계를 이해하기 위한 첫 번째 수단은 ’읽기(Read)’다. 파일 시스템, 데이터베이스, API 응답 등 세상에 존재하는 읽기 가능한 모든 정적/동적 데이터는 MCP 아키텍처 내에서 **자원(Resource)**이라는 단일한 개념으로 결속(Binding)된다.

자원 모델링의 본질은 클라이언트가 서버의 복잡한 로컬 환경(예: 어떤 드라이브인지, 어떤 DB 벤더인지)을 전혀 알 필요 없이, 마치 웹 브라우저가 URI를 치고 들어가 문서를 열람하듯 데이터를 스트리밍받을 수 있게 하는 **단방향 데이터 공급망(Delivery Pipeline)**을 구축하는 데 있다.

이 절에서는 로컬 파일이 가상 파일 시스템(VFS)으로 편입되고, 고유한 URI를 통해 에이전트의 컨텍스트 창(Context Window)으로 안전하게 청킹(Chunking) 스트리밍되는 추상화 기법들을 다룬다.
