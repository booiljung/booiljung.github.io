3.1.1.2. 데이터 포맷 자가 진단 및 메타데이터 추론(Inference) 체계

MCP의 자원(Resource) 모델이 단순히 “여기 파일이 있습니다“라는 원시 바이트(Raw Byte) 덩어리를 에이전트에게 내던지는 수준에 머무른다면, 에이전트는 이 덩어리가 자신이 읽을 수 있는 텍스트 도메인인지, 아니면 파싱 불가능한 바이너리 실행 파일인지 알 길이 없다. 컨텍스트 창에 무턱대고 바이너리를 쑤셔 넣는 순간 LLM의 추론은 완전히 붕괴된다.

따라서 MCP 서버는 파일의 내용을 스트리밍하기 이전에, 해당 리소스의 DNA를 기계적으로 진단하여 클라이언트에게 친절한 메타데이터(MIME-Type, Encoding 등)를 함께 첨부해 주는 데이터 포맷 자가 진단 및 추론 체계를 필수적으로 가동한다. 본 절에서는 파일의 본성에 확정적 레이블을 부여하는 추론 모델을 심층 탐구한다.
