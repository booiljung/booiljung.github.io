3.1.1.2.1. 매직 넘버(Magic Number) 및 확장자 조합 기반 MIME-Type 런타임 추론 모델

사용자가 파일 확장자를 의도적이거나 실수로 누락하더라도, MCP 서버는 이를 알아서 텍스트(text/plain)나 이미지(image/png) 등으로 분석해 낼 수 있는 강건한 추론 능력이 필요하다.

단순한 문자열 확장자 비교를 넘어선 하이브리드 MIME-Type 추론 모델이 런타임에 구동된다.

  1. 고속 확장자 딕셔너리 매칭: 일차적으로 .json, .md, .py 등의 명시적인 확장자가 존재하면 서버에 내장된 고속 해시 맵(Hash Map)을 통해 O(1)의 속도로 application/json, text/markdown, text/x-python 등의 기본 MIME-Type을 추출한다.
  2. 매직 넘버(Magic Number) 딥 스니핑: 확장자가 없거나 불분명할 경우, 서버는 해당 파일의 첫 수십 바이트(Header) 읽기를 시도한다. 89 50 4E 47로 시작한다면 image/png로, 25 50 44 46으로 시작한다면 application/pdf로 확정 짓는 파일 시그니처 분석(Magic Number Sniffing)을 수행한다.
  3. MIME-Type 내장 페이로드 결합: 추론이 완료된 포맷 문자열은 resources/read 응답 객체의 mimeType 필드에 명시적으로 주입되어, LLM 에이전트가 “아, 이 문자열은 Python 코드 문법으로 렌더링해서 읽으면 되겠군“이라 판단할 수 있게 강력한 힌트를 제공한다.
