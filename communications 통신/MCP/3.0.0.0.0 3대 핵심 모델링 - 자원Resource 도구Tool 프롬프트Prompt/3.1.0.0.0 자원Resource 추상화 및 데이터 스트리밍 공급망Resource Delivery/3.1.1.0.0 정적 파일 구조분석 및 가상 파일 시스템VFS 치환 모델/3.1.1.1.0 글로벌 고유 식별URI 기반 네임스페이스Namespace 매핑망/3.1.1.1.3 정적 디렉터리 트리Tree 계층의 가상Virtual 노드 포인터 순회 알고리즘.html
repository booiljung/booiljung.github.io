3.1.1.1.3. 정적 디렉터리 트리(Tree) 계층의 가상(Virtual) 노드 포인터 순회 알고리즘

서버에 단일 파일이 아니라 방대한 소스코드 폴더 구조가 존재할 때, 클라이언트 에이전트가 전체 구조를 파악하려면 한 번에 10만 개의 리소스 목록을 수신해야 하며, 이는 심각한 대역폭 과부하를 초래한다.

MCP는 디렉터리 역시 하나의 ‘자원 템플릿(Resource Template)’ 또는 ’계층적 폴더’로 가상화(Virtualization)하여, 에이전트가 디렉터리를 트리(Tree) 구조로 탐색할 수 있도록 돕는 가상 노드 포인터(Virtual Node Pointer) 순회 알고리즘을 지원한다.

  1. 템플릿 치환 검색(Template Substitutions): 클라이언트는 resources/templates/list를 통해 서버 내에 어떤 형태의 자원 꼴(예: mcp://src/{filename}.js)이 존재하는지 파악한다. 디렉터리에 해당하는 노드들은 직접적인 바이트 스트림을 주지 않고, 자기 밑에 존재하는 하위 요소들의 ‘형식’ 또는 ’하위 URI 목록’만을 반환한다.
  2. 가상 가지치기(Virtual Pruning): 에이전트가 코드를 탐색하다 tests 폴더를 건너뛰고 싶다고 판단하면, 해당 계층을 더 이상 쿼리(Query)하지 않고 순회(Traversal)를 멈춘다(가지치기).
  3. 지연 로딩(Lazy Loading) 및 인덱싱: 서버는 클라이언트가 실제로 물리적인 하위 노드 접근을 요구(Call)하기 전까지는 전체 하위 트리의 내용물(예: 10GB 분량의 로그)을 메모리에 올려두지 않고, 오직 요청받은 현재 노드의 포인터 인덱싱 테이블만을 유지함으로써 극단적인 메모리 최적화를 이뤄낸다.
