3.1.1.1.1. 로컬 물리 경로(Path)를 추상화(Abstraction)하는 커스텀 스키마(mcp://) 변환 메커니즘

서버가 클라이언트에게 리소스 목록을 제공할 때, 내부 보안 규정상 물리적인 디렉터리 경로(예: /var/www/html/secret.txt)를 그대로 평문으로 유출하는 것은 시스템 침투의 빌미를 제공하는 행위다.

이를 봉쇄하기 위해 애플리케이션 계층에서는 mcp:// 커스텀 스키마 변환 메커니즘이 적용된다.

  1. 내부 해시(Hash) 라우팅 테이블 생성: 물리 경로 /var/www/html/을 서버의 내부 메모리에서 기준 식별자 project-root와 매핑시킨다.
  2. URI 포맷 조립: 대상 파일 secret.txt에 접근하기 위해 mcp://project-root/secret.txt 라는 가상의 URI(Uniform Resource Identifier)를 생성하여 클라이언트 측에 제공한다(resources/list 반환).
  3. 디코딩(Decoding) 및 역추적: 클라이언트가 훗날 resources/read로 해당 URI 문자열을 요청하면, 서버 라우터는 접두어 mcp://를 파싱해 내고 내부 라우팅 테이블을 역조회(Reverse Lookup)하여 실제 커널의 파일 디스크립터(FD)에 연결한다. 이 과정에서 클라이언트는 자신이 접근하는 바이트 덩어리가 C 드라이브에 있는지 네트워크 나스(NAS)에 있는지 전혀 알 수 없는 완벽한 추상화 벽이 완성된다.
