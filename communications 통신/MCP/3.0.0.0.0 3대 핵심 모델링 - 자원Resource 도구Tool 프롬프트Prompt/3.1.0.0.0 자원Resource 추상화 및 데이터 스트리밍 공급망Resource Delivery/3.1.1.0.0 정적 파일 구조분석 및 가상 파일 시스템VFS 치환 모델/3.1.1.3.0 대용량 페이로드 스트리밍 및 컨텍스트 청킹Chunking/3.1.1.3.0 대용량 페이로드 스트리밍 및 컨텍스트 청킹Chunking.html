3.1.1.3. 대용량 페이로드 스트리밍 및 컨텍스트 청킹(Chunking)

에이전트가 resources/read를 호출하여 10GB 크기의 데이터베이스 덤프(Dump) 파일이나 무한정 길어지는 서버 로그를 단일 JSON 문자열로 받으려 시도한다면, JSON 파서는 물론이고 시스템의 가용 메모리 전체가 붕괴되는 치명적인 장애가 발생한다.

또한, 아무리 강력한 대규모 언어 모델(LLM)이라도 한 번에 소화할 수 있는 토큰의 양(Context Window)에는 엄격한 한계가 존재한다.

따라서 MCP 자원 공급망의 척추에는 거대한 데이터를 잘게 부수고(Chunking) 스트리밍하는 세밀한 데이터 분절 및 연속성 보장 메커니즘이 내제되어 있다. 본 챕터에서는 서버와 클라이언트가 대용량 데이터를 안전하게 퍼나르는 윈도우 슬라이딩 및 오프셋 매핑 기법을 다룬다.
