3.1.1.4.1. 보안 정책 스펙(e.g., .gitignore, .mcpignore) 기반의 노출 금지 파일 동적 배제

개발자들은 이미 .gitignore 나 .dockerignore 등의 표준화된 문법을 통해 수많은 시크릿 파일과 런타임 결과물(예: node_modules, build/)을 레포지토리에서 배제하는 데 익숙하다.

MCP 서버는 이 철학을 그대로 계승하여, 에이전트에게 제공할 자원 트리를 빌드할 때 동적 배제(Dynamic Exclusion) 엔진을 가동한다.

  1. 로컬 무시 규칙 탐색 및 파싱: VFS 트리 탐색기(3.1.1.1.3)가 디렉터리를 순회하다가 .mcpignore 또는 .gitignore 파일을 조우하면, 해당 파일의 정규식 패턴(Glob Pattern / Regex)들을 즉시 인메모리 필터 딕셔너리에 추가한다.
  2. 패턴 매칭을 통한 목록 도려내기: 이후 발견되는 모든 하위 자원들은 이 패턴 검증기를 의무적으로 거쳐야 한다. 만약 *.pem 이나 .env* 패턴에 걸리는 파일이 발견되면, 이 파일은 resources/list 응답 배열에 아예 포함되지 않고 유령 파일(Ghost File) 처리된다.
  3. 직접 접근(Direct Access) 차단: 에이전트가 우연히, 혹은 악의적인 해킹 목적으로 파일 구조를 유추하여 resources/read, uri="mcp://src/.env"를 직접 찔러보더라도, 이 배제 엔진이 경로 라우터 앞단에서 요청을 가로채 Resource Not Found(404) 에러를 반환함으로써 그 존재 자체를 부정한다.
