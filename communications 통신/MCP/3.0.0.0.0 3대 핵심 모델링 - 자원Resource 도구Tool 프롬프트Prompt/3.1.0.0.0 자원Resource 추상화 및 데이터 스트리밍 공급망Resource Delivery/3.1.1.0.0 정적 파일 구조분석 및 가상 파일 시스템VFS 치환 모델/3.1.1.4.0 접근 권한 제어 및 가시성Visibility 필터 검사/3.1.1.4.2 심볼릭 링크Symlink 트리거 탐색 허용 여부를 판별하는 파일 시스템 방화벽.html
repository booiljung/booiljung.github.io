3.1.1.4.2. 심볼릭 링크(Symlink) 트리거 탐색 허용 여부를 판별하는 파일 시스템 방화벽

리눅스/유닉스 환경에서 에이전트에게 할당된 작업(Workspace) 디렉터리는 /home/user/project로 제한되어 있을지라도, 그 프로젝트 내부에 /etc/passwd나 시스템 루트 폴더(/)를 향해 열려있는 심볼릭 링크(Symbolic Link)가 섞여 있을 수 있다.

에이전트가 VFS를 탐색하다 이 링크를 타고 넘어가 서버 전체의 보안을 무너뜨리는 현상을 디렉터리 트래버설(Directory Traversal) 해킹이라 부른다. 이를 봉쇄하기 위한 심볼릭 링크 탐색 방화벽이 작용해야 한다.

  1. 링크 경계선(Boundary) 역추적: 서버는 자원의 절대 경로(Absolute Path)를 결정할 때, OS 커널의 realpath() 또는 readlink() 시스템 콜을 강제하여 최종 목적지의 물리적 실체 경로를 구한다.
  2. 탈출(Escape) 징후 판별 알고리즘: 계산된 최종 목적지 경로 문자열이 사전에 선언된 Workspace Root 경로의 문자열 포함(Starts-with) 범위 내에 들어오는지 문자열 비교(String Comparison) 알고리즘을 수행한다.
  3. 가시성 단절: 만약 심볼릭 링크가 가리키는 곳이 샌드박스 허용 범위 바깥(Out of Boundary)이라면, 방화벽은 해당 경로를 무효화하고 클라이언트의 트리 검색망에서 해당 심볼릭 링크 노드를 가차 없이 삭제(Drop)해버린다.
