3.5.2.4.1. 단일 마스터 프롬프트 대신 작은 조각(Snippet) 프롬프트 여러 개를 런타임에 직렬 결합하는 스케줄러

전사를 관통하는 거대한 수만 줄짜리 ’마스터 프롬프트’를 하나 만들어두는 것은 끔찍한 모놀리식(Monolithic) 안티패턴이다.

  1. 지시문 마이크로서비스(Micro-Prompts): “보안 가이드라인(100줄)”, “사내 윤리 규범(50줄)”, “파이썬 컨벤션(200줄)” 등 각각의 프롬프트 조각(Snippet)들은 독립적인 엔터티로 쪼개져 저장 공간에 산재해 있다.
  2. 매니페스트(Manifest) 파일 조립기: 클라이언트가 write_code를 요청하면, 서버의 프롬프트 스케줄러는 미리 정의된 매니페스트 [security_v1, ethics_v2, python_v3]를 룩업(Lookup)한다.
  3. 런타임 직렬 결합(Sequential Concat): 스케줄러는 그 즉시 3개의 스니펫을 DB에서 병렬로 끌어온 뒤 순서대로 결합(Concat)하여 하나의 완전한 템플릿(Composite Template)으로 만들어 낸다. 이를 통해 부서별로 자신들의 프롬프트 스니펫만 독립적으로 유지보수(Decoupling)할 수 있게 된다.
