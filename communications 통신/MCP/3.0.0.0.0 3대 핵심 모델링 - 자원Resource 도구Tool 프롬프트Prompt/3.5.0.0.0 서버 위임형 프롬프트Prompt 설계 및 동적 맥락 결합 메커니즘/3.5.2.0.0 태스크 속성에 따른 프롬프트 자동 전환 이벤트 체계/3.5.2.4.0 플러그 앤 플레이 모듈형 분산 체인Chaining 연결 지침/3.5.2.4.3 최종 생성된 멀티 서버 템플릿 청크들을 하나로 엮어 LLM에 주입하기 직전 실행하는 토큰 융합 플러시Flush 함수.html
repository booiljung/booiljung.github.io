3.5.2.4.3. 최종 생성된 멀티 서버 템플릿 청크들을 하나로 엮어 LLM에 주입하기 직전 실행하는 토큰 융합 플러시(Flush) 함수

여러 템플릿이 무작위로 합쳐지다 보면 System:, System: 태그가 중복 생성되거나 내용 모순이 발생할 수 있다.

  1. 의미론적 중복 제거(Semantic Deduplication): 조립의 마지막 관문에서 렌더러는 합쳐진 여러 청크들 사이의 문법적 중복 태그 마진(Margin)을 지워버리고(Collapse), 충돌하는 지시문 중 더 상위 레벨(우선순위)의 보안 지시문이 이길 수 있도록 오버라이딩 병합 룰(Overriding Merge Rules)을 가동한다.
  2. JIT 토큰 정제(Sanitization): 또한 합쳐진 템플릿 전체를 한 번 더 훑어보며 미처 치환되지 못한 {{ ... }} 구멍이나 불법적인 인젝션 시도가 있는지 최종 샌드박스 검수(Sanity Check)를 시행한다.
  3. 아웃바운드 플러시(Outbound Flush): 모든 결합과 검수가 끝난 완벽한 무결성의 프롬프트 객체는, 마침내 버퍼 메인 파이프포트를 열고 클라이언트를 향해(혹은 클라이언트를 패스스루하여 LLM을 향해) HTTP 응답으로 폭포수처럼 쏟아져 나간다(Flushed).

이 일련의 모듈형 동적 융합 과정을 통해, MCP는 죽어있던 정적 텍스트 프롬프트를 살아 움직이는 유기체적 오퍼레이팅 스크립트로 격상시킨다.
