3.5.2.4.2. 다수의 이기종 서버 상에 존재하는 서로 상관된 템플릿 간 교차 연결(Cross-Reference)을 위한 중계 구조

이 분산 체이닝의 진가는 서버 대 서버(Server-to-Server) 통신에서 발휘된다.

  1. 원격 프롬프트 레퍼런스(Remote Prompt Ref): 프롬프트 본문 안에 자신의 로컬 파일뿐만 아니라, 네트워크 너머에 있는 다른 MCP 서버의 프롬프트를 포인팅하는 {{ remote_prompt(server="auth_server", name="get_rbac_rules") }} 구문이 허용된다.
  2. 분산 위임 쿼리(Distributed Delegated Query): 이 구문을 만난 스케줄러 브로커는 파이프라인 컴파일을 일시 중단하고, auth_server 측으로 권한 관련 템플릿의 서브-렌더링(Sub-rendering)을 요청한다. 보안 서버는 자신의 로컬 환경 변수에 맞게 렌더링 된 “최종 결과 문자열“만을 브로커에게 반환한다.
  3. 토폴로지 독립적 융합: 브로커는 자신이 모르는 타 서버의 비즈니스 로직(예: 권한 판별 방식)을 알 필요 없이 오직 결과 텍스트만 넘겨받아 체인에 용접 결합한다. 완벽한 MSA(Micro-service Architecture) 철학이 프롬프트 레벨까지 이식된 것이다.
