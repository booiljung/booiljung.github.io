3.5.2.1.2. 실시간 모드 전환 버튼 클릭 이벤트(Client Event)를 수신해 서버 템플릿 목록(List)을 동기 갱신하는 브릿지

때로는 자동 분류가 아니라, 인간 사용자가 명시적으로 “이제부터는 코드 작성 말고 문서만 만들어“라고 모드를 전환(Mode Switch)할 때가 있다 (예: 클라이언트 UI의 툴바 버튼 클릭).

  1. 클라이언트 이벤트 버스: 사용자가 GUI 상에서 Mode: Document Writer를 클릭하는 순간, 클라이언트는 서버 측으로 notifications/client_state_changed 이벤트를 쏜다.
  2. 동적 목록(List) 갱신: 서버는 이 이벤트를 받자마자, 해당 세션에 대해 prompts/list 엔드포인트가 반환해 주던 프롬프트 목록 자체를 통째로 갈아 끼운다(동기 갱신). 이전까지 보이던 create_component 프롬프트는 숨겨지고 write_api_spec 같은 문서용 프롬프트들만 노출된다.
  3. 가이드레일(Guardrail)의 즉각 수립: 이를 통해 프롬프트 자체가 하나의 UI 컨트롤러처럼 작동하여, 문서를 써야 할 에이전트가 엉뚱하게 코드 작성 도구를 가져다 쓸 여지를 원천적으로 봉쇄하는 소프트웨어적 가이드레일이 수립된다.
