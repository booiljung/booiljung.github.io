3.5.2.1.1. 코드 생산(Creator), 코드 리뷰(Reviewer), 디버깅(Debugger) 등 작업 목적에 맞춘 지시문 분기 라우팅

MCP 서버의 프롬프트 관리자는 다양한 목적으로 직조된 독립적 지시문 트리(Tree)를 보유하고 있다.

  1. 태스크 분류기(Task Classifier): 클라이언트가 “오류가 발생했어요“라고 말하면 정규식 혹은 경량 분류 모델이 이를 DEBUG 클래스로 분류한다. 반면 “로그인 화면 만들어줘“는 CREATE 클래스에 배정된다.
  2. 페르소나 맵핑 다이렉티브: CREATE 시에는 "창의적이고 최신 트렌드를 반영한 컴포넌트를 작성하라"는 System Prompt 계통으로 라우팅 된다. 반면 DEBUG 전환 시 "절대 새로운 기능을 추가하지 말고 원인만 분석하여 최소한의 수정을 가하라"라는 극도로 보수적인 페르소나 지시문이 LLM 창으로 흘러간다.
  3. 오염 없는 재시작(Clean Slate Restart): 다른 페르소나로 넘어갈 때, LLM의 기존 대화 문맥(Context)에 묻어있던 이전 페르소나의 유산이 혼란을 주지 않도록, 보이지 않는 곳에서 강제로 system 메시지를 새로운 것으로 덮어씌워 완전히 새로운 뇌로 재조립(Reassemble)시킨다.
