3.5.2.1.3. 에이전트의 현재 추론 단계(Reasoning Stage)에 따라 점진적으로 세밀(Detailed)해지는 모달 변화 로직

큰 프로젝트는 단 한 번의 턴(Turn)으로 끝나지 않는다. 계획 -> 실행 -> 검토라는 추론의 단계(Reasoning Stage)를 거치게 되는데, 이 스테이지마다 요구되는 프롬프트의 관점(Perspective)도 달라야 한다.

  1. 추론 상태 기계(Stage FSM): 서버 어댑터는 하나의 세션 내에서 에이전특가 몇 번째 턴업을 수행 중인지 추적한다.
  2. 거시에서 미시로(Macro to Micro): 작업 초반(턴 1~2) 발급되는 분석용 프롬프트는 아키텍처 다이어그램이나 도메인 지식처럼 시스템 전체를 조망하는 거시적(High-level) 컨텍스트 템플릿 위주로 결합된다.
  3. 점진적 포커싱(Progressive Focusing): 작업이 구체화되어 세부 구현 파트로 진입(턴 3~)하면, 서버는 거시적 문서를 프롬프트에서 조용히 내리고(Dropped), 대신 현재 수정 중인 특정 함수의 유닛 테스트나 타입 정의(Interface) 등 극도로 미시적(Low-level)이고 세밀한 컨텍스트만을 보간하여 토큰 한도를 아낌과 동시에 LLM의 집중력을 한 곳으로 철저히 모아준다.
