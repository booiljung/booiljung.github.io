3.5.2.2.3. 접속 사용자의 세션 토큰 정보를 통한 부서별 개인화(Personalized) 어조 반영기

동일한 사내 어시스턴트 봇이라도, 마케팅 부서 직원이 부를 때와 시니어 백엔드 엔지니어가 부를 때 템플릿의 분위기는 달라야 한다.

  1. 권한 기반 프로파일 읽기: 클라이언트가 prompts/get을 날릴 때 동봉된 JWT 토큰 안의 클레임({"role": "marketing_staff", "name": "Alice"})을 서버가 정밀하게 분해한다.
  2. 어조 폴리모피즘(Polymorphism of Tone): 마케팅 직원의 세션이라면 답변 프롬프트에 [지시: 기술적 용어를 최대한 배제하고 비즈니스 임팩트 위주로 부드럽게 대답하라]가 합성되고, 시니어 엔지니어 세션이라면 [지시: 인사말은 생략하고 코드와 터미널 커맨드 위주로 간결하게 답변하라] 형태의 어조 변환(Tone-switching) 블록이 삽입된다.
  3. 개인화된 사용자 경험(UX): 이러한 동적 컴파일 파이프라인 덕분에, 회사는 단 한 벌의 프롬프트 세트를 유지보수하면서도 수만 명의 사용자에게 완벽하게 초개인화(Hyper-Personalized)된 톤 앤 매너(Tone & Manner)를 일관되게 제공하는 비용 효율적 마법을 시연한다.
