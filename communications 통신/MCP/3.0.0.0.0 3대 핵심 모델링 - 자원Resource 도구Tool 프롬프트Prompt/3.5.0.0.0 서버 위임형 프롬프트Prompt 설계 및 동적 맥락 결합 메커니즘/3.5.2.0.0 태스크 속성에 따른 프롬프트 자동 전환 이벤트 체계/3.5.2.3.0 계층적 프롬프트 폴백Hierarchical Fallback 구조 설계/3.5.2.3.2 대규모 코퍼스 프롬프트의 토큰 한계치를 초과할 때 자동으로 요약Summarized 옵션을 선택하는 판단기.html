3.5.2.3.2. 대규모 코퍼스 프롬프트의 토큰 한계치를 초과할 때 자동으로 요약(Summarized) 옵션을 선택하는 판단기

자원 통합 엔진이 파일 시스템에서 로그를 100MB어치 긁어모아 프롬프트에 병합(Merge)하려 하면, 128K 토큰 제한을 가진 LLM은 즉각 TokenLimitExceeded 에러를 토해낸다.

  1. 토큰 계측기(Token Estimator): JIT 컴파일러 단락에는 틱토큰(Tiktoken)과 같은 경량 토큰 계산기가 내장되어 있어, 문자열 조립이 최종 완료되기 직전에 예상 토큰 볼륨을 게이징(Gauging)한다.
  2. 트리머(Trimmer) 개입: 만약 토큰 한계치의 90%를 초과할 위기라면, 시스템은 Fallback Level 2 플래그를 켠다. 이때 100MB의 로그는 그대로 들어가지 않고, 꼬리표 1MB만 남기거나 혹은 또 다른 경량 로컬 LLM에게 summarize()를 시킨 짧은 요약본 텍스트로 치환된다.
  3. 자동 페이징(Auto-Paging): 부득이하게 전체 맥락이 필요한 경우, 프롬프트를 강제로 여러 조각으로 쪼개어 “1편 읽어”, “2편 읽어“라는 형태로 에이전트의 워크플로우를 다중 턴(Multi-turn)으로 강제 변환시키는 전략적 폴백(Strategic Fallback)을 구사한다.
