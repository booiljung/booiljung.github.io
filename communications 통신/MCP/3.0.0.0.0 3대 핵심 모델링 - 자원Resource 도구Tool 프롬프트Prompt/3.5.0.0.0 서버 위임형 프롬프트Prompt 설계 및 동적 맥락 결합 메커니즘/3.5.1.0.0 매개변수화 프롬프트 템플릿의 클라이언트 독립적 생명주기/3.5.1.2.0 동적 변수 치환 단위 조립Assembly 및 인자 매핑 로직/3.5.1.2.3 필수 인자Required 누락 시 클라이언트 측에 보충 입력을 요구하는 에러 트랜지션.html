3.5.1.2.3. 필수 인자(Required) 누락 시 클라이언트 측에 보충 입력을 요구하는 에러 트랜지션

LLM 에이전트가 “에러 원인 분석해 줘“라고 템플릿을 호출했는데, 결정적으로 error_log 파라미터를 넘기지 않았다면 서버는 빈칸인 템플릿을 응답하면 안 된다. 환각을 부추길 뿐이다.

  1. 스키마 기반 런타임 밸리데이션: 템플릿 엔진 앞단에는 JSON Schema 검증기가 서 있다. 클라이언트의 요청이 required: ["error_log"] 필드를 누락했는지 밀리초(ms) 단위로 스캔한다.
  2. Missing Arg 인터럽트 생성: 누락이 발견되면 서버는 템플릿 합성을 즉각 중단(Halt)하고, 특별한 400 Bad Request 에러와 함께 {"missing_arguments": ["error_log"], "message": "Please provide the exact terminal output."} 이라는 구조적 피드백을 내던진다.
  3. LLM의 자동 보충 프롬프팅: 이 에러를 수신한 클라이언트(에이전트)는 당황하여 종료되지 않고, 스스로 “아, 내가 로그 복사를 깜빡했네“라고 인지하여 다시금 터미널 도구를 호출해 로그를 긁어온 뒤, 파라미터를 채워 재귀 호출(Retry)하는 자가 치유(Self-Healing) 사이클을 완성한다.
