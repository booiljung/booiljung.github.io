3.5.1.2.1. 템플릿 내 빈 슬롯 구간(Arguments)에 실시간 상황 데이터를 주입하는 인터페이스

서버가 선언하는 프롬프트 스키마는 도구(Tool) 스키마의 쌍둥이 동생과 같다. 명확한 인자 명세서(Arguments Schema)를 들고 있다.

  1. JSON Schema 규약 명세: explain_error라는 프롬프트는 내부적으로 description 뿐만 아니라, arguments 블록을 통해 "error_code", "stack_trace"라는 빈 슬롯(Slots)을 에이전트에게 내민다.
  2. 런타임 컨텍스트 페칭(Fetching): 에이전트는 터미널에서 터진 빨간 줄(에러)을 즉각 긁어모은 뒤, {"error_code": "E110", "stack_trace": "Line 52..."}라는 JSON 페이로드를 만들어 서버로 보낸다.
  3. Mustache/Jinja 기반 보간(Interpolation): 서버의 프롬프트 팩토리는 클라이언트가 보내준 이 살점들을 전달받아, 미리 준비된 거대한 마크다운 템플릿 내부의 {{error_code}} 빈 슬롯 구간에 오차 없이 즉시 끼워 넣어 매끈한 단일 지시 문안을 완성시킨다.
