3.5.1.1.2. LLM 메타 프롬프트 업데이트 비용 최적화를 담보하는 엣지(Edge) 배포 아키텍처

글로벌로 분산된 만 명의 개발자가 쏘는 code_review 프롬프트 요청을 단일 중앙 서버가 매번 문자열 결합 연산을 치르며 응답하면 CPU 병목이 발생한다.

  1. 글로벌 CDN 및 엣지 캐싱(Edge Caching): MCP 프롬프트 라우터는 Vercel Edge나 Cloudflare Workers 같은 엣지 노드에 정적으로 렌더링된 메타 프롬프트 청크들을 배포해 둔다.
  2. 해시 기반 퍼지(Cache Purge): 사내 깃허브에서 표준 개발 인스트럭션 문서가 커밋(Commit)되면, CI/CD 파이프라인이 엣지에 퍼져있는 프롬프트 캐시의 해시 체인을 퍼지(Purge)하여 즉각 무효화시킨다.
  3. 가변 인자(Variable Args)의 최후 결합 지연: 서버(단일 노드)는 고정된 시스템 지시문 본체(수천 토큰)의 결합을 엣지에 떠넘기고, 클라이언트가 던진 미세한 변수({"targetFunc": "login"}) 치환 로직만 워커 스레드에서 마지막(Late Binding)에 엮어주는 비용 최적화(Cost Optimization)를 달성한다.
