3.5.1.3.3. 복합 참조 과정에서 무한 순환 호출(Circulation)에 빠지지 않게 막아주는 구조적 뎁스(Depth) 커트

“A 파일이 B를 참조하고, B 파일이 다시 A를 참조한다면?” 템플릿 엔진이 이를 무식하게 계속 결합하면, 메모리가 터질 때까지(OOM) 무한 루프에 빠지게 된다.

  1. 호출 스택 프레임(Call Stack Frame): 자원 관리자는 결합을 시작할 때 루트 프롬프트부터 Call Depth = 0이라는 고유 식별자를 부여하며 파고들어 간다.
  2. 환형 의존성(Circular Dependency) 탐지기: 뎁스가 하나씩 내려갈 때마다 현재 방문 중인 URI 스택 트리(A -> B -> C -> A)를 점검한다. 만약 이미 부모 트리에 있는 URI를 또 만나면, 컴파일러는 이 재귀를 즉시 끊고 <자원 참조 순환 감지됨: 생략>이라는 짤막한 메타 텍스트만 남긴 채 탈출(Bail out)한다.
  3. 최대 깊이 임계치(Max Depth Throttle): 무한 순환이 아니더라도, 엄청난 양의 마트료시카형 조립이 일어나면 프롬프트 컨텍스트 창(Context Window)이 넘쳐버린다. 보통 Max Depth = 3 수준에서 하드 커팅 룰(Hard Cutting Rule)을 두어 구조적 폭주를 방어한다.
