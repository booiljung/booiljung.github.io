3.5.1.3.2. 서버 메모리 상태 스냅샷 다이내믹 렌더링을 통한 마크다운 기반 상황 정보 블록화

프롬프트 안에 결합되어야 할 것은 정적인 파일만이 아니다. 현재 서버가 얼마나 메모리를 쓰고 있는지, 워커 큐가 몇 개나 비었는지 등의 살아 숨 쉬는 런타임 지표가 필요할 때가 있다.

  1. 동적 지표 매크로: 프롬프트 안에 {{ system_metrics() }} 라는 특수 렌더링 훅(Hook)이 삽입되어 있으면, 이 훅은 파일 시스템이 아니라 서버의 OS 프로세스나 Redis 통계 엔진을 호출한다.
  2. 구조화된 마크다운 덤프: 단순히 MEM=80% 이렇게 던지는 것이 아니라, LLM이 파싱하기 쉽게 | 지표 | 상태 | \n | CPU | 45% |와 같이 아름답게 렌더링된 마크다운 표(Table) 형태로 덤프시켜 프롬프트 상단 부록(Appendix) 칸에 박아버린다.
  3. 상황 인지(Context-Awared) 지능 부여: 에이전트는 이 동적 스냅샷 블록을 읽음으로써 “아, 지금 서버 CPU 부하가 99%이므로 컴파일 도구를 돌리면 죽겠구나. 일단 기다리자“라고 판단하는 이른바 시스템 기계-감각(Machine Sense)을 획득하게 된다.
