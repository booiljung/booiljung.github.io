3.5.1.4.2. 데이터 타입 일치성 검사를 통해 오염된 문자열을 정제(Sanitize)하는 문법 수호기

JSON 스키마에서 어떠한 파라미터가 integer나 boolean 형이어야 한다고 정의되었음에도, 클라이언트(또는 악의적인 해커)가 교묘한 주입 문자가 적힌 긴 텍스트 String을 욱여넣으면 템플릿 결합기가 파탄 난다.

  1. 엄격 모드 타입 캐스팅(Strict Type Casting): 렌더러가 작동하기 직전 파라미터 객체 트리를 순회하는 검수 데몬이 깨어난다. 만약 retry_count로 정의된 필드에 "10; Drop Server" 따위의 문자열이 온다면 즉각 typeof 검사를 통과하지 못해 트랜잭션이 기각된다.
  2. 화이트리스트 열거형(Enum) 통제: "lang": "python" 처럼 문자열이지만 극도로 제한된 상태를 요하는 값들은, 오픈형 스팬 대신 사전에 선언된 열거형(Enum) 화이트리스트 배열 안에 일치하는지 M:1 맵핑 룩업(Lookup)을 거친다.
  3. 자동 가지치기(Auto-Pruning): 스키마 명세에 없는 잉여 파라미터 변수 키(Key)값이 슬쩍 끼어들어올 경우, 이를 버그로 여기고 기각하는 대신, 조용히 잉여 가지를 잘라내어(Prune) 오직 허락된 규격의 인터페이스 살점만 남기는 면역 작용이 일어난다.
