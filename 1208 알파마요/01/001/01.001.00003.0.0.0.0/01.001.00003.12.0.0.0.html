3.12 구성 요소 간 데이터 흐름과 시스템 통합

1. 개요

자율주행 시스템은 다수의 센서, 소프트웨어 모듈, 액추에이터가 실시간으로 데이터를 교환하며 협력하는 복합 시스템이다. 본 절에서는 구성 요소 간의 데이터 흐름 구조, 데이터 대역폭 요구 사항, 그리고 시스템 통합의 핵심 과제를 기술한다.

2. 데이터 흐름 구조

자율주행 시스템의 데이터 흐름은 센서에서 액추에이터까지의 주요 경로(Main Data Path)와 보조 데이터 경로(Auxiliary Data Path)로 구분된다.

2.1 주요 데이터 경로

\text{센서} \xrightarrow{\text{원시 데이터}} \text{인지} \xrightarrow{\text{구조화 표현}} \text{예측} \xrightarrow{\text{미래 상태}} \text{판단} \xrightarrow{\text{궤적}} \text{제어} \xrightarrow{\text{명령}} \text{액추에이터}

이 경로에서 각 단계의 데이터 형태와 대역폭은 다음과 같다.

단계데이터 형태대역폭 (일반적)
카메라 → 인지원시 영상 (YUV/RGB)카메라당 ~500 Mbps (8MP, 30fps)
라이다 → 인지포인트 클라우드 (x,y,z,i)~100 Mbps
레이더 → 인지객체 목록 또는 포인트 클라우드~10 Mbps
인지 → 예측객체 목록 (위치, 속도, 클래스)~1 Mbps
예측 → 판단예측 궤적 집합~1 Mbps
판단 → 제어목표 궤적~0.1 Mbps
제어 → 액추에이터조향각, 가속도, 제동~0.01 Mbps

데이터 대역폭은 센서에서 인지 모듈까지의 구간에서 가장 높으며, 후속 단계로 갈수록 데이터가 추상화되어 대역폭이 감소한다.

2.2 보조 데이터 경로

주요 데이터 경로 외에 다음의 보조 데이터 흐름이 존재한다.

  • 위치 추정 → 인지, 판단: 차량의 현재 위치와 자세 정보를 인지 및 판단 모듈에 제공한다.
  • HD Map → 인지, 위치 추정, 판단: 사전 구축된 지도 정보를 다수 모듈에 제공한다.
  • V2X → 인지, 판단: 외부 통신으로 수신된 교통 정보를 인지 및 판단 모듈에 제공한다.
  • 차량 상태 → 제어, 판단: 차량의 현재 속도, 가속도, 조향각 등을 제어 및 판단 모듈에 피드백한다.

3. 시스템 통합의 핵심 과제

3.1 실시간 제약 (Real-Time Constraints)

자율주행 시스템은 센서 데이터 수집에서 제어 명령 출력까지의 전체 지연(End-to-End Latency)이 안전한 주행에 필요한 시간 제약 내에 완료되어야 한다. 일반적으로 100~200ms 이내의 종단간 지연이 요구되며, 고속 주행 환경에서는 더 엄격한 제약이 적용된다. 각 소프트웨어 모듈의 실행 시간 예산(Time Budget)이 사전에 할당되어야 하며, 최악 실행 시간(Worst-Case Execution Time, WCET)의 분석이 필요하다.

3.2 센서 동기화 (Sensor Synchronization)

이종 센서의 데이터는 동일 시점의 환경 상태를 반영하도록 시간적으로 정합되어야 한다. 카메라의 노출 시점, 라이다의 스캔 시작 시점, IMU의 샘플링 시점 등이 정밀하게 동기화되어야 하며, 하드웨어 트리거(Hardware Trigger)와 타임스탬프(Timestamp) 기반의 소프트웨어 보간(Interpolation)이 활용된다.

3.3 좌표계 변환 (Coordinate Transformation)

각 센서는 자체 좌표계(Sensor Frame)에서 데이터를 출력하며, 이를 공통 좌표계(Vehicle Frame 또는 World Frame)로 변환하여야 한다. 센서의 외부 파라미터(Extrinsic Parameters)는 캘리브레이션(Calibration) 과정을 통해 결정되며, 정확한 캘리브레이션은 센서 융합과 3D 인지의 정확도에 직접적으로 영향을 미친다.

3.4 데이터 무결성과 진단 (Data Integrity and Diagnostics)

센서 데이터의 무결성을 실시간으로 검증하고, 센서 고장이나 데이터 이상을 감지하는 진단 기능이 필요하다. 센서 오염(렌즈 오염, 라이다 커버 오염), 센서 차단(눈·얼음 적재), 전자기 간섭 등의 이상 상태를 감지하여 인지 모듈에 센서 신뢰도 정보를 제공하여야 한다.

3.5 소프트웨어 버전 관리와 OTA 갱신

자율주행 소프트웨어는 차량 출하 이후에도 지속적으로 갱신되어야 한다. OTA(Over-the-Air) 갱신을 통해 인지 알고리즘의 개선, 새로운 ODD 지원, 안전 패치 등이 배포된다. 소프트웨어 갱신 과정에서의 안전성 보장, 갱신 실패 시의 복구(Rollback) 메커니즘, 갱신 전후의 호환성 검증이 핵심 과제이다.
