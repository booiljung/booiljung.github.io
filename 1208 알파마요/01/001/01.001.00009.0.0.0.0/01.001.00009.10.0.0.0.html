9.10 차량 간 통신(V2X) 요구사항

차량 간 통신(Vehicle-to-Everything, V2X)은 차량이 다른 차량, 기반 시설, 보행자, 그리고 통신 망과 정보를 주고 받는 무선 통신 기술의 총칭이다. Level 4 자율주행 시스템에서 V2X는 시계 외부 또는 폐색 영역의 정보 획득, 협력적 인지(cooperative perception), 협력적 운전 조작(cooperative manoeuvre), 안전 관련 경고의 광역 전파를 가능하게 한다. 이 절에서는 V2X의 분류, 통신 기술, 메시지 표준, 보안, 성능 요구사항을 학습 순서에 따라 기술한다.

1. V2X의 학술적 정의와 분류

V2X는 통신 상대에 따라 다음과 같이 세분된다.

  • 차량 대 차량(Vehicle-to-Vehicle, V2V): 인접한 차량 간의 직접 통신
  • 차량 대 기반 시설(Vehicle-to-Infrastructure, V2I): 신호등, 노변 장치(Road-Side Unit, RSU), 가변형 표지 등과의 통신
  • 차량 대 보행자(Vehicle-to-Pedestrian, V2P): 보행자가 휴대하는 단말기와의 통신
  • 차량 대 망(Vehicle-to-Network, V2N): 이동 통신 망 또는 클라우드 서비스와의 통신
  • 차량 대 그리드(Vehicle-to-Grid, V2G): 전력 그리드와의 통신(주로 전력 거래와 충전 제어 목적)

이 분류는 ETSI TR 102 638 V1.1.1 Vehicular Communications; Basic Set of Applications; Definitions가 제시한 분류 체계에 기초한다.

2. 무선 통신 기술의 분류

V2X에 사용되는 무선 통신 기술은 다음의 두 계열로 분류된다.

2.1 IEEE 802.11p 계열

IEEE 802.11p-2010 Wireless Access in Vehicular Environments는 5.9 GHz 대역의 면허 면제 채널에서 동작하는 단거리 무선 통신 표준이다. 미국에서는 이 표준을 기반으로 하는 Dedicated Short-Range Communications (DSRC)가 사용되어 왔으며, 유럽에서는 ITS-G5라는 명칭의 프로파일이 ETSI EN 302 663 V1.3.1에 의하여 정의되어 있다. 후속 표준인 IEEE 802.11bd는 처리량과 신뢰성을 향상시켰다.

2.2 셀룰러 V2X(C-V2X)

3GPP가 표준화한 셀룰러 V2X는 LTE-V2X와 NR-V2X의 두 세대로 구분된다. LTE-V2X는 3GPP Release 14 (2017)에서 도입되었으며, 직접 통신을 위한 PC5 인터페이스와 망 기반 통신을 위한 Uu 인터페이스를 동시에 지원한다. NR-V2X는 3GPP Release 16 (2020)에서 도입되어 5G NR 무선 접속을 PC5 인터페이스에 적용하였으며, 광대역, 저지연, 고신뢰의 특성을 통하여 협력적 운전 조작에 요구되는 성능을 제공한다.

3. 메시지 집합 표준

V2X 응용은 사전 정의된 메시지 형식을 통하여 정보를 교환한다. 주요 메시지 표준은 다음과 같다.

  • ETSI EN 302 637-2 V1.4.1, Cooperative Awareness Basic Service (CA Basic Service): 협력적 인식 메시지(Cooperative Awareness Message, CAM)를 정의하며, 차량의 위치, 속도, 가속도, 헤딩 등을 주기적으로 방송한다.
  • ETSI EN 302 637-3 V1.3.1, Decentralized Environmental Notification Basic Service (DEN Basic Service): 분산형 환경 통지 메시지(Decentralized Environmental Notification Message, DENM)를 정의하며, 사고, 노면 위험, 공사 구역과 같은 사건을 전파한다.
  • SAE J2735_202309, V2X Communications Message Set Dictionary: 북미 환경에서 사용되는 BSM (Basic Safety Message), MAP, SPaT (Signal Phase and Timing), TIM (Traveler Information Message) 등의 메시지 집합을 정의한다.
  • ETSI TS 103 324 V2.1.1 Cooperative Perception Service (CPS): 협력적 인지 메시지(Collective Perception Message, CPM)를 정의하며, 차량이 자체 인지 결과를 다른 차량에 공유하여 시계 외부 객체에 대한 인지를 가능하게 한다.
  • ETSI TS 103 561 V2.1.1 Maneuver Coordination Service (MCS): 협력적 운전 조작 메시지(Maneuver Coordination Message, MCM)를 정의하며, 차량 간의 의도와 궤적을 사전에 협상한다.

4. V2X 응용

V2X 응용은 안전, 효율, 편의의 세 범주로 분류된다. Level 4 자율주행과 직접적으로 관련되는 안전 응용은 다음과 같다.

  • 교차로 충돌 회피 보조: 신호등 SPaT 정보와 인접 차량의 BSM/CAM을 결합하여 교차로 충돌을 회피한다.
  • 사각 지대 경고: 후속 차량의 위치를 V2V로 공유하여 차로 변경 시의 사각 지대 충돌을 예방한다.
  • 비신호 교차로 협상: MCM을 통하여 우선권을 협상한다.
  • 협력적 인지: 자체 인지가 폐색에 의하여 제한될 때 다른 차량과 RSU의 인지 결과를 결합한다.
  • 비상 차량 우선권 통지: 응급 차량의 접근을 사전에 통지한다.
  • 위험 사건 전파: 노면 결빙, 사고, 정지 차량 등을 광역으로 전파한다.

5. 보안 요구사항

V2X 메시지는 위변조와 사칭에 의한 위험을 방지하기 위하여 강력한 인증·무결성 보호 절차를 요구한다. 주요 보안 표준은 다음과 같다.

  • IEEE Std 1609.2-2022 Wireless Access in Vehicular Environments — Security Services for Applications and Management Messages: 메시지 서명, 인증서 형식, 인증서 폐기, 익명성 보호를 위한 가명 인증서(pseudonym certificate)를 정의한다.
  • ETSI TS 103 097 V2.1.1, Intelligent Transport Systems (ITS); Security; Security header and certificate formats: ETSI 환경에서의 보안 헤더와 인증서 형식을 정의한다.
  • ETSI TS 102 941, Intelligent Transport Systems (ITS); Security; Trust and Privacy Management: 신뢰 모형과 익명성 관리 절차를 정의한다.
  • ISO/SAE 21434:2021, Road vehicles — Cybersecurity engineering: 차량 사이버 보안의 일반 원칙을 정의하며, V2X 통신을 차량 보안 경계의 일부로 다룬다.

V2X 응용에서는 메시지의 실시간성을 보장하기 위하여 서명·검증의 처리 지연을 최소화해야 하며, 가명 인증서의 갱신 주기와 익명성 사이의 균형이 학술적으로 분석된다.

6. 성능 요구사항

V2X 통신의 성능 요구사항은 응용에 따라 차등적으로 부여된다. ETSI TR 102 638과 3GPP TS 22.186 V17.0.0 Service requirements for enhanced V2X scenarios는 다음의 핵심 지표를 정의한다.

  • 단대단 지연(end-to-end latency): 안전 응용의 경우 일반적으로 100 ms 이내가 요구되며, 협력적 운전 조작의 경우 3 ms에서 10 ms 수준의 초저지연이 요구된다.
  • 메시지 신뢰도(packet reception ratio): 안전 응용에서 95% 이상의 수신율이 요구된다.
  • 통신 거리: 도시 환경에서 최소 150 m, 고속 도로에서 300 m 이상의 통신 거리가 요구된다.
  • 메시지 갱신 주기: CAM/BSM은 1 Hz에서 10 Hz의 가변 주기로 송신되며, 차량의 동적 상태 변화에 따라 적응적으로 결정된다.
  • 채널 부하: 분산 혼잡 제어(Decentralized Congestion Control, DCC)에 의하여 채널 점유율이 사전에 정의된 한계 미만으로 유지되어야 한다.

7. V2X와 자율주행 인지의 결합

V2X는 차량의 자체 센서가 가지는 시야의 한계를 보완한다. 협력적 인지 메시지(CPM)는 차량과 RSU가 자체 인지 결과를 다른 차량과 공유하도록 함으로써 가려진 객체에 대한 인지가 가능하도록 한다. 이를 위하여 다음의 학술적 과제가 식별되어 있다.

  • 좌표계의 정합: 송신자와 수신자의 좌표계와 시간 동기화의 일치
  • 인지 결과의 신뢰도 표기: 객체별 분류 점수, 위치 분산, 양상 정보의 표준화
  • 중복 객체의 결합: 다른 송신자의 인지 결과와 자체 인지 결과를 결합하는 융합 절차
  • 잘못된 인지의 전파 방지: 신뢰도 가중과 다중 출처 검증

이러한 과제는 ETSI TS 103 324와 ISO 23150:2021 Road vehicles — Data communication between sensors and data fusion unit for automated driving functions — Logical interface의 결합 적용을 통하여 다루어진다.
