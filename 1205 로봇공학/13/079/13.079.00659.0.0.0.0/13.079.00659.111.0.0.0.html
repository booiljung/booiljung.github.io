659.111 변환 발행 주파수 최적화

1. 개요

TF2 변환의 발행 주파수는 변환의 시간적 해상도, 네트워크 대역폭 소모, CPU 부하에 직접적으로 영향을 미친다. 과도한 발행 주파수는 시스템 자원을 낭비하고, 부족한 발행 주파수는 변환 외삽(extrapolation) 오류를 야기한다. 본 절에서는 최적의 발행 주파수를 결정하는 원칙과 방법을 기술한다.

2. 발행 주파수 결정 원칙

2.1 나이퀴스트 기준 적용

변환 소비자(consumer)가 주파수 f_c로 변환을 조회하면, 변환 발행 주파수 f_p는 최소한 f_c 이상이어야 한다. 보간(interpolation)의 정확도를 보장하려면:

f_p \geq 2 f_c

2.2 변환 변화율 고려

변환이 빠르게 변하는 관절(예: 고속 매니퓰레이터)은 높은 발행 주파수가 필요하고, 느리게 변하는 변환(예: map → odom)은 낮은 주파수로 충분하다.

3. 권장 발행 주파수

변환 유형변화 특성권장 주파수
센서 고정 변환불변정적 변환 (/tf_static)

바퀴 관절연속 회전20~50 Hz

4. 발행 주파수 측정과 조절

# 현재 발행 주파수 확인
ros2 run tf2_ros tf2_monitor

# robot_state_publisher의 발행 주파수 변경
ros2 param set /robot_state_publisher publish_frequency 50.0

5. 요약

변환 발행 주파수는 변환의 변화율과 소비자의 조회 빈도에 따라 결정되며, 정적 변환에는 /tf_static을, 동적 변환에는 변환 변화율에 비례하는 적절한 주파수를 설정하여야 한다.
