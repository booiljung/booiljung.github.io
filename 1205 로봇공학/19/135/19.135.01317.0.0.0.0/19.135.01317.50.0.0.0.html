1317.50 PlanSys2 아키텍처의 향후 발전 방향

1. 개요

PlanSys2(Planning System 2)는 ROS2 기반의 PDDL 태스크 플래닝 프레임워크로서, 로봇 시스템의 자율적 임무 계획 및 실행을 가능하게 한다. 현재까지 PlanSys2는 도메인 전문가, 문제 전문가, 플래너, 실행기의 4대 핵심 노드 아키텍처와 POPF 기반 플래닝 엔진을 중심으로 발전해 왔다. 그러나 자율 로봇 시스템의 복잡성이 증대되고, AI 플래닝 분야의 연구가 진전됨에 따라, PlanSys2 아키텍처의 지속적인 확장과 개선이 요구된다. 본 절에서는 PlanSys2의 향후 발전 방향으로 예상되는 기술적 과제와 연구 동향을 학술적 관점에서 분석한다.

2. PDDL 표현력의 확장

2.1 고급 PDDL 기능 지원

현재 PlanSys2는 PDDL 2.1 수준의 기능을 주로 지원한다. 향후 발전 방향으로는 PDDL의 더 높은 수준(level)의 표현력을 수용하는 것이 핵심 과제이다.

  • PDDL 2.2의 파생 술어(derived predicates): 복잡한 전제 조건을 계층적으로 정의할 수 있는 파생 술어의 완전한 지원이 필요하다. 파생 술어를 통해 도메인 모델의 표현력이 향상되며, 로봇의 상태를 보다 추상적 수준에서 기술할 수 있다.
  • PDDL 3.0의 제약 조건(constraints)과 선호도(preferences): 계획의 품질을 평가하기 위한 소프트 제약 조건 및 선호도 명세의 지원이 요구된다. 이를 통해 단순히 목표 달성 여부만이 아닌, 계획의 최적성을 정량적으로 평가할 수 있다.
  • PDDL+의 연속적 변화(continuous change): 하이브리드 시스템(hybrid system)에서 이산적 액션과 연속적 물리 과정이 상호작용하는 도메인을 모델링하기 위한 PDDL+ 확장이 고려된다. 이는 특히 이동 로봇의 에너지 소모, 센서 드리프트 등 연속적 물리량의 변화를 계획에 반영할 때 유용하다.

2.2 확률적 플래닝 지원

고전적 PDDL은 결정론적(deterministic) 가정에 기반한다. 그러나 실세계의 로봇 환경은 본질적으로 불확실성을 내포한다. 향후 PlanSys2는 다음과 같은 비결정론적 플래닝 형식(formalism)의 통합을 고려할 수 있다.

  • PPDDL(Probabilistic PDDL): 액션의 효과를 확률 분포로 표현하여, 확률적 상태 전이를 모델링한다.
  • RDDL(Relational Dynamic Influence Diagram Language): 결정 다이어그램(influence diagram)에 기반한 순차적 의사결정 문제의 기술이 가능하다.
  • FOND(Fully Observable Non-Deterministic) 플래닝: 비결정론적 액션 효과 하에서 모든 가능한 결과에 대해 목표를 달성하는 강건한 정책(policy)을 도출한다.

3. 플래너 엔진의 다양화

3.1 현대적 플래너의 통합

PlanSys2의 기본 플래너인 POPF(Partial Order Planning Forward) 외에, 국제 플래닝 대회(International Planning Competition, IPC)에서 우수한 성능을 보인 현대적 플래너들의 통합이 필요하다.

  • Fast Downward: 휴리스틱 탐색(heuristic search)에 기반한 고성능 플래너로서, SAS+ 형식(formalism)을 활용한다. Landmark 기반 휴리스틱, merge-and-shrink 추상화 등 다양한 휴리스틱 함수를 지원한다.
  • LAMA: Landmark 기반 휴리스틱과 비용 기반 탐색을 결합한 플래너로서, IPC 2008에서 최우수 성적을 기록하였다.
  • OPTIC: 시간적 플래닝(temporal planning)과 수치적 조건(numeric conditions)을 지원하는 최적 시간적 플래너이다.
  • ENHSP(Expressive Numeric Heuristic Search Planner): 수치적 플래닝(numeric planning)에 특화된 플래너로서, 연속적 수치 변수의 처리에 강점을 갖는다.

PlanSys2의 플래너 플러그인 아키텍처는 이러한 다양한 플래너의 교체 가능한(pluggable) 통합을 지원하도록 설계되어 있으나, 각 플래너의 입출력 형식, PDDL 지원 수준, 실행 환경 요구사항 등의 차이를 흡수하는 어댑터 계층(adapter layer)의 고도화가 필요하다.

3.2 온라인 플래닝과 지속적 플래닝

전통적인 오프라인 플래닝(offline planning)은 계획을 사전에 완전히 수립한 후 실행하는 방식이다. 그러나 동적 환경에서 운용되는 로봇은 실행 중에 환경 변화에 실시간으로 대응해야 하므로, 다음과 같은 접근법의 통합이 요구된다.

  • 온라인 플래닝(online planning): 실행과 계획을 교대로 수행하며, 각 단계에서 현재 상태에 기반한 부분 계획(partial plan)을 도출한다.
  • 지속적 플래닝(continual planning): 계획의 실행 중에도 계속해서 계획을 갱신하며, 새로운 목표의 추가, 기존 목표의 변경, 예측하지 못한 상황의 발생 등에 유연하게 대응한다.
  • 반응적 플래닝(reactive planning): 사전 계획 없이 현재 상태에 대한 즉각적인 반응 규칙에 따라 행동을 선택한다. 행동 트리(behavior tree)와의 통합이 이 범주에 해당한다.

4. 계층적 태스크 플래닝과의 융합

4.1 HTN 플래닝 통합

계층적 태스크 네트워크(Hierarchical Task Network, HTN) 플래닝은 복잡한 태스크를 하위 태스크(subtask)로 분해(decomposition)하여 해결하는 접근법이다. PDDL 기반의 고전적 플래닝과 HTN 플래닝은 상호 보완적인 관계를 갖는다.

  • 고전적 플래닝은 목표 상태(goal state)를 명시하고 플래너가 자율적으로 계획을 도출하는 선언적(declarative) 접근법이다.
  • HTN 플래닝은 도메인 전문가가 태스크 분해 방법(method)을 명시적으로 제공하는 절차적(procedural) 접근법이다.

PlanSys2에 HTN 플래닝을 통합함으로써, 도메인 전문 지식을 활용한 효율적인 탐색 공간 축소와 보다 직관적인 태스크 명세가 가능해진다. SHOP2, JSHOP2, Panda 등의 HTN 플래너를 플러그인으로 통합하는 방안이 고려될 수 있다.

4.2 HDDL(Hierarchical Domain Definition Language) 지원

HDDL은 HTN 플래닝을 위한 표준화된 도메인 기술 언어로서, PDDL의 확장으로 설계되었다. HDDL은 IPC 2020에서 HTN 트랙의 공식 입력 언어로 채택되었으며, 향후 PlanSys2가 HDDL 파서와 HTN 플래너를 통합할 경우, 계층적 태스크 분해와 고전적 플래닝을 동일한 프레임워크 내에서 혼합 운용할 수 있게 된다.

5. 다중 로봇 플래닝 아키텍처

5.1 분산 플래닝

현재 PlanSys2는 단일 로봇 시스템을 주 대상으로 설계되어 있다. 다중 로봇 시스템(multi-robot system)에서의 태스크 플래닝은 다음과 같은 추가적인 과제를 수반한다.

  • 태스크 할당(task allocation): 복수의 로봇에게 태스크를 최적으로 분배하는 문제이다. 옥션(auction) 기반 할당, 마켓 기반 접근법(market-based approach), 게임 이론적 방법 등이 연구되어 왔다.
  • 조율(coordination): 복수의 로봇이 상호 간의 충돌을 회피하고, 자원을 공유하며, 동기화된 행동을 수행하기 위한 메커니즘이다.
  • 통신 제약(communication constraints): 분산 환경에서 로봇 간 통신 대역폭의 제한, 통신 지연, 통신 단절 등의 제약 조건을 고려한 계획 수립이 필요하다.

PlanSys2의 향후 발전 방향은 ROS2의 멀티 로봇 지원 기능(네임스페이스 분리, DDS 도메인 분리, 분산 실행기 등)을 활용한 분산 플래닝 아키텍처의 구축을 포함한다.

5.2 이기종 로봇 팀 지원

이기종(heterogeneous) 로봇 팀은 서로 다른 능력(capability)을 가진 로봇들로 구성된다. 각 로봇의 PDDL 도메인이 상이할 수 있으므로, 도메인 전문가 노드가 복수의 도메인 모델을 관리하고, 플래너가 로봇별 능력 차이를 고려하여 계획을 수립하는 확장이 필요하다.

6. 학습 기반 플래닝과의 통합

6.1 강화학습과 태스크 플래닝의 결합

강화학습(Reinforcement Learning, RL)과 고전적 태스크 플래닝의 결합은 최근 로봇 공학 연구에서 활발히 탐구되는 주제이다. 두 접근법의 결합 방식은 다음과 같이 분류된다.

  • RL 정책을 플래닝 액션의 실행기로 사용: 고수준의 태스크 계획은 PDDL 기반 플래너가 수립하고, 각 액션의 저수준 실행은 RL로 학습된 정책이 담당한다.
  • 학습된 휴리스틱을 플래너에 주입: 심층 신경망(deep neural network)을 통해 학습된 휴리스틱 함수를 플래닝 탐색 과정에 활용하여 탐색 효율을 향상시킨다.
  • 도메인 모델의 자동 학습: 로봇의 실제 운용 데이터로부터 PDDL 도메인 모델(액션의 전제 조건과 효과)을 자동으로 학습한다. 이를 도메인 학습(domain learning) 또는 액션 모델 학습(action model learning)이라 한다.

6.2 대규모 언어 모델(LLM)과의 연계

대규모 언어 모델(Large Language Model, LLM)은 자연어 이해 및 생성 능력을 통해 로봇 태스크 플래닝의 새로운 가능성을 제시한다. 향후 PlanSys2와 LLM의 연계 방향은 다음과 같다.

  • 자연어 목표 명세의 PDDL 변환: 사용자가 자연어로 기술한 임무 목표를 LLM을 통해 PDDL 문제 정의로 자동 변환한다.
  • 도메인 모델 자동 생성: LLM을 활용하여 비정형 문서(매뉴얼, 사양서 등)로부터 PDDL 도메인 모델의 초안을 자동 생성한다.
  • 계획 설명(plan explanation): 도출된 계획의 의미와 근거를 자연어로 설명하여, 비전문가 사용자의 이해를 돕는다.

다만, LLM 기반 접근법은 환각(hallucination) 현상으로 인한 부정확한 도메인 모델 생성, 형식적 정확성(formal correctness)의 보장 어려움 등의 한계를 내포하고 있으므로, 검증(verification) 단계를 반드시 수반해야 한다.

7. 실시간성 및 안전성 강화

7.1 실시간 플래닝 보장

안전 필수(safety-critical) 로봇 시스템에서는 플래닝 결과의 적시 도출이 보장되어야 한다. 이를 위해 다음과 같은 방향이 고려된다.

  • 제한 시간 플래닝(bounded-time planning): 플래너의 탐색 시간에 상한을 설정하고, 제한 시간 내에 발견된 최선의 계획을 반환하는 전략이다. 임의 시점 알고리즘(anytime algorithm)의 활용이 이에 해당한다.
  • 사전 계산된 계획 라이브러리(plan library): 빈번하게 사용되는 계획을 사전에 계산하여 저장하고, 실행 시 라이브러리에서 가장 적합한 계획을 검색·적용한다.
  • 점증적 플래닝(incremental planning): 이전 계획의 결과를 재활용하여, 환경 변화에 대응하는 새로운 계획을 신속하게 도출한다.

7.2 계획 검증과 안전 제약

자율 로봇의 계획이 안전 요구사항을 위반하지 않도록 보장하는 것은 중요한 과제이다. 향후 PlanSys2에는 다음과 같은 검증 메커니즘의 도입이 기대된다.

  • 형식적 검증(formal verification): 모델 검증(model checking) 기법을 활용하여 계획이 안전 속성(safety property)을 만족하는지 자동으로 검증한다.
  • 안전 제약의 PDDL 통합: 안전 관련 제약 조건을 PDDL 도메인의 불변 조건(invariant) 또는 상태 제약(state constraint)으로 명시하여, 플래너가 이를 위반하는 계획을 원천적으로 배제하도록 한다.
  • 런타임 모니터(runtime monitor): 계획의 실행 중에 안전 제약의 위반 여부를 실시간으로 감시하며, 위반이 탐지되면 비상 정지(emergency stop) 또는 안전 모드 전환을 수행한다.

8. ROS2 생태계와의 심화 통합

8.1 Nav2, MoveIt2, ros2_control과의 통합 심화

PlanSys2의 향후 발전은 ROS2 생태계의 핵심 프레임워크들과의 더욱 긴밀한 통합을 포함한다.

  • Nav2 통합 심화: 내비게이션 계층에서의 동적 장애물 회피, 다중 층(multi-floor) 내비게이션, 실외 환경 내비게이션 등의 고급 시나리오를 태스크 플래닝과 연계한다.
  • MoveIt2 통합: 매니퓰레이션(manipulation) 태스크의 계획과 실행을 PlanSys2의 태스크 계층과 연동하여, 이동-조작 복합 태스크(mobile manipulation)를 통합적으로 계획한다.
  • ros2_control 통합: 하드웨어 추상화 계층과의 직접적인 연동을 통해, 액추에이터 수준의 제어 명령을 태스크 계획에서 직접 참조하거나 모니터링할 수 있는 인터페이스를 구축한다.

8.2 클라우드 로보틱스와의 연계

클라우드 로보틱스(cloud robotics)는 플래닝 연산을 클라우드 서버에 위임하고, 로봇은 계획의 실행에 집중하는 아키텍처를 의미한다. ROS2의 DDS 기반 통신 아키텍처는 WAN(Wide Area Network) 환경에서의 노드 간 통신을 지원하며, 이를 활용한 클라우드 기반 PlanSys2 배치(deployment)가 가능하다. 계산 집약적인 플래닝 연산을 클라우드에서 수행하고, 자원 제약적인 엣지 로봇에서는 계획의 실행과 센서 데이터 처리에 집중하는 분리 아키텍처가 향후 발전 방향으로 고려된다.

9. 요약

PlanSys2 아키텍처의 향후 발전 방향은 PDDL 표현력의 확장, 현대적 플래너 엔진의 다양화, 계층적 태스크 플래닝(HTN)과의 융합, 다중 로봇 분산 플래닝, 학습 기반 플래닝과의 통합, 실시간성 및 안전성 강화, 그리고 ROS2 생태계와의 심화 통합 등 광범위한 영역을 포괄한다. 이러한 발전은 단순한 기능 추가를 넘어, 자율 로봇 시스템의 복잡성 증가와 실세계 배치 요구에 체계적으로 대응하기 위한 아키텍처적 진화를 의미한다.
