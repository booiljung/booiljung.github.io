FF 플래너의 구조와 알고리즘 (Structure and Algorithm of the FF Planner)

1. 개요

FF(Fast Forward)는 Hoffmann과 Nebel(2001)이 개발한 순방향 탐색 기반 계획기로, h_{\text{FF}} 휴리스틱과 도움 행동(helpful actions) 가지치기를 결합하여 빠른 계획 생성을 달성한다. IPC 2000에서 최고 성능을 달성하며, 현대 계획기의 기반이 되었다.

2. 핵심 구성 요소

2.1 h_{\text{FF}} 휴리스틱

삭제 완화된 문제의 해를 다항 시간에 구하여 휴리스틱으로 사용한다.

2.2 도움 행동 (Helpful Actions)

h_{\text{FF}} 계산 과정에서 완화된 계획의 첫 번째 행동들을 도움 행동으로 식별하여, 이 행동들만을 탐색 후보로 제한한다. 이에 의해 분기 계수가 현저히 감소한다.

2.3 실행 유지 탐색 (Enforced Hill Climbing)

GBFS(탐욕 최선 우선 탐색)를 기본 전략으로 사용하되, 정체(plateau) 상태에 빠지면 BFS로 전환하여 탈출한다.

function EHC(s₀):
    s = s₀
    while G ⊄ s:
        found = false
        for each helpful action a of s:
            s' = γ(s, a)
            if h_FF(s') < h_FF(s):
                s = s'; found = true; break
        if not found:

3. FF의 전체 알고리즘

  1. 초기 상태 s_0 설정
  2. h_{\text{FF}}(s_0) 계산 및 도움 행동 식별
  3. EHC로 계획 탐색
  4. EHC 실패 시 완전한 GBFS로 전환

5. PlanSys2에서의 활용

PlanSys2에서 FF는 플래너 플러그인으로 사용 가능하다. POPF(FF의 시간적 확장)가 PlanSys2의 기본 계획기이다.
