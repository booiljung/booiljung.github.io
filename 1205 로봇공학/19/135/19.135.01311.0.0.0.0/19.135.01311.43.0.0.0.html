안전 필수 시스템에서의 패러다임 선택 고려 사항 (Paradigm Selection Considerations for Safety-Critical Systems)

1. 개요

안전 필수(safety-critical) 로봇 시스템에서는 행동의 예측 가능성, 형식 검증 가능성, 실시간 보장이 최우선이다. 이러한 요구는 하드코딩 기반 접근을 강하게 선호하며, 자율 계획은 보조적 역할에 한정된다.

2. 안전 관련 고려 사항

요구 사항하드코딩자율 계획
예측 가능성완전 보장제한적
형식 검증가능 (FSM, BT)어려움
실시간 보장가능어려움
안전 인증용이 (IEC 61508, ISO 13849)매우 어려움
최악 실행 시간분석 가능예측 불가

3. 안전 필수 하이브리드 설계

  • 안전 관련 행동 (비상 정지, 충돌 회피, 과부하 보호): 하드코딩 필수
  • 비안전 임무 계획 (배달 순서, 탐사 경로): 자율 계획 허용
  • 안전 가드: 자율 계획이 생성한 행동이 안전 제약을 위반하지 않는지 행동 트리로 감시

4. 참고 문헌

  • IEC 61508: Functional Safety of Electrical/Electronic/Programmable Electronic Safety-Related Systems.
  • ISO 13849-1: Safety of machinery — Safety-related parts of control systems.
