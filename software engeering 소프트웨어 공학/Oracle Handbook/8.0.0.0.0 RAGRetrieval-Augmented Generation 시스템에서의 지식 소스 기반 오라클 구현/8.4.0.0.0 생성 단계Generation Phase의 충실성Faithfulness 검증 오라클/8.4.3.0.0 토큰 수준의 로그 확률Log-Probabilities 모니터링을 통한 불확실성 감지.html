8.4.3 토큰 수준의 로그 확률(Log-Probabilities) 모니터링을 통한 불확실성 감지

가장 저렴하면서 백엔드 언더후드(Under-the-hood) 수준에 맞닿아 있는 검증 방법론이다. LLM(예: OpenAI API)이 텍스트를 생성할 때 반환하는 logprobs 값을 오라클로 활용한다.

  • 모델이 “오펜하이머는…” 이라는 토큰 다음에 “1904년(확률 99%)“을 붙인다면 이는 근거가 확실한 정답이다.
  • 그런데 연도 토큰에서 {"1904": 20%, "1905": 19%, "1899": 15%} 와 같이 확률 분포가 박빙(Perplexity가 높음)이라면, 모델이 모르는 것을 ‘찍어 맞추고’ 있다는 뜻이다.
  • 오라클은 연속된 3 토큰 이상의 분포가 Top 1 Logprob < 50%인 구간이 발생하면 즉시 생성 중단을 걸고(Abort), 해당 답변을 Low Confidence로 스위칭해야 한다.
