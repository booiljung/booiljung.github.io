8.4.7 수치 데이터 및 고유 명사(Named Entity)의 정확한 인용 검증

텍스트의 분위기가 맞아도 숫자가 틀리면 비즈니스에서는 100% 오답이다. 일반 LLM 심판은 스토리가 그럴듯하면 “수익률 3.5%“를 “수익률 5.3%“라고 적은 치명적 숫자 도치 현상을 놓치기 십상이다.

이를 방어하는 “이중 포위망” 오라클:

  1. spaCy 등의 NER(Named Entity Recognition) 도구로 원본과 생성본에서 [사람, 날짜, 금액, 퍼센트]를 모조리 추출한다.
  2. 생성본에 포함된 집합이 원본의 집합에 100% 포함(Subset)되지 않는다면, 즉시 수치 불일치 에러(Numeric Hallucination Error)를 raise 시킨다.
