8.10.2 검증 비용(Token Cost) 절감을 위한 작은 모델(SLM) 활용 전략

모든 사용자의 쿼리마다 GPT-4 수준의 심판관을 소환하면 회사는 파산한다. 오라클 시스템 역시 경제성 원칙을 따라야 한다.

  • 오라클의 지식 증류(Distillation): 거대 모델이 판별한 (Context, Answer, Pass/Fail) 쌍 데이터 10만 개를 모은다. 이를 8B 이하의 소형 오픈소스 모델(Llama 3 8B, Mistral 등)에 미세 조정(Fine-tuning)하여, “오직 특정 도메인의 환각 여부 판별기“로만 깎아낸다.
  • 이 작고 날카로운 SLM(Small Language Model) 심판관을 로컬 컨테이너에 올려두면, 검증 비용은 GPT-4 대비 1/50로 떨어뜨리면서도 오라클 기능은 완벽히 유지할 수 있다.
