LLM-as-a-Judge를 프로덕션에 태우면 봇 생성 3초 + 판단 3초 = 6초라는 치명적 지연이 발생한다.

  • 병렬 검사기(Parallel Evaluators): 답변이 다 만들어질 때까지 오라클이 기다리지 않는다. 스트리밍(Streaming) 환경에서 생성 모델이 1차 문맥 패시지를 뱉어내는 즉시 이 비동기 조각들을 오라클 API 3개가(관련성, 충실성 판독) 동시에 덮쳐서 분석한다.
  • Fail-fast 단락 회로(Short-circuit): 무거운 의미론적 평가(LLM) 전에, 가변운 구문론 평가(정규식, 토큰 오버랩)를 전방에 배치하여 쓰레기 답변은 0.01초 안에 먼저 기각해 버린다.
