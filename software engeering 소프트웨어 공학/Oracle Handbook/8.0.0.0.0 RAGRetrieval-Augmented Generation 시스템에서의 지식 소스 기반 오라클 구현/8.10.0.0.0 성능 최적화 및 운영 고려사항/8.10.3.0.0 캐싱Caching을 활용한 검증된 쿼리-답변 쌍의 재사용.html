8.10.3 캐싱(Caching)을 활용한 검증된 쿼리-답변 쌍의 재사용

사내 임직원들이 묻는 질문의 70%는 대개 “회사 와이파이 비번”, “연말 정산 기한” 등 파레토 법칙(Pareto Principle)을 따르는 뻔한 질문이다.

  • Semantic Cache Oracle: 사용자의 쿼리가 들어왔을 때 무작정 리트리버를 켤 것이 아니라, Redis 등 캐시 DB를 먼저 뒤진다.
  • 이때 이전 임직원이 물어봐서 이미 오라클의 엄격한 1.0(Pass) 검증을 뚫고 저장 완료된 쿼리-답변 튜플이 존재하고, 입력된 질문과 캐시된 검색 쿼리의 의미론적 유사도(Semantic Similarity)가 95% 이상이라면, 파이프라인 전체를 바이패스(Bypass)하고 그 검증된 답변을 0.05초 만에 뿌린다.
