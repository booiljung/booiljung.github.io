8.2.2 청킹(Chunking) 전략이 오라클 정확도에 미치는 영향

원문을 잘게 쪼개는 청킹(Chunking)은 검색 모델의 임베딩 성능은 물론 시스템 전체의 평가 분해능(Resolution)을 지배한다.

청크가 너무 거대할 경우(10,000 토큰), 오라클이 Context 안에서 진짜 필요한 단 한 줄을 찾지 못해 “관련성이 떨어진다“고 판단하는 오탐(False Positive)의 늪에 빠진다. 역으로 지나치게 작으면 문맥 단절 탓에 대명사가 지칭하는 본질이 사라져(예: “그것은 3%의 이자율을 가진다”), 오라클이 사실 확인(Fact-check)을 포기하고 오류를 뱉게 된다.
