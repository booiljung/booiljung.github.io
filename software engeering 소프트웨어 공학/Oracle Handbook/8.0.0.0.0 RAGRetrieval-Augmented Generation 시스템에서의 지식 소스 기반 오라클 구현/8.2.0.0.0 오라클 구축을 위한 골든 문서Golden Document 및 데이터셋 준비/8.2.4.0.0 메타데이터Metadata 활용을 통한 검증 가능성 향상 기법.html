8.2.4 메타데이터(Metadata) 활용을 통한 검증 가능성 향상 기법

RAG 오라클 시스템은 순수한 텍스트만 검사하는 것이 아니다. 데이터베이스에 청크를 적재할 때 반드시 꼬리표(Metadata)를 달아 두어야 한다.

{
  "chunk_text": "2025년 기준 반차 휴가 부여 일수는 15일이다.",
  "metadata": {"doc_id": "HR_2025_001", "effective_date": "2025-01-01", "department": "HR"}
}

이 메타데이터는 오라클이 ‘시점’ 과 ‘권한’ 을 다차원적으로 평가하게 해 준다. 사용자가 “작년 기준 내 휴가 일수는?“이라고 물었을 때, 검색 모델이 2025년 청크를 가져왔다면 오라클은 텍스트의 유사성을 무시하고 effective_date 불일치를 근거로 리트리버를 강력하게 기각(Reject)할 수 있다.
