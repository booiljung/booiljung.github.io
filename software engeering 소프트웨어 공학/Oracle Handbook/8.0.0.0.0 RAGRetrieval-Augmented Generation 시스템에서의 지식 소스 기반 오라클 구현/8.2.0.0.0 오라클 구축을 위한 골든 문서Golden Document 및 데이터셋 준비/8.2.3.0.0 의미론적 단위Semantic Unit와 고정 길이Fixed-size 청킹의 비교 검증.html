8.2.3 의미론적 단위(Semantic Unit)와 고정 길이(Fixed-size) 청킹의 비교 검증

청크를 자르는 방식은 단순한 ’길이 제한’이 아니다. 문맥을 담아내는 그릇(Vessel)의 형태를 결정하는 것이다.

  • Fixed-size 청킹: 무식하게 512 Token 단위로 자르는 방식이다. 문장의 허리가 끊겨 “Apple이 발표한 새로운 기기는…“에서 끝나버리면, 오라클 시스템은 “Apple이 무엇을 발표했는지“에 대한 사실 확인 판검사 로직에서 에러를 뿜는다.
  • Semantic 청킹: 마크다운의 ## Header나 문단 교체(\n\n) 등 의미론적 단위를 기준으로 자른다. 개발 비용은 비싸지만, 이 청크 덩어리는 완결성을 가지기 때문에 오라클 LLM이 읽어들일 때 Context Relevance 점수가 극명하게 상향 안정화된다.
