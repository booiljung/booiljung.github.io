Chapter 8. RAG(Retrieval-Augmented Generation) 시스템에서의 지식 소스 기반 오라클 구현

단순한 프롬프트 주입(Zero-shot)을 넘어, 기업 내부의 파편화된 지식 베이스를 연동한 RAG(Retrieval-Augmented Generation) 시스템은 엔터프라이즈 AI의 표준이 되었다. 그러나 RAG의 복잡한 파이프라인은 환각(Hallucination)이라는 치명적인 리스크를 검색(Retrieval)과 생성(Generation)이라는 두 가지 차원에서 증폭시킨다.

오라클이 존재하지 않는 RAG 시스템은, 눈구멍 뚫린 백과사전을 들고 있는 달변가와 같다. 제 아무리 말을 번지르르하게 해도 그 근거를 확증할 수 없다면 비즈니스에 쓰일 수 없다. 본 장에서는 주어진 문서 자원(Knowledge Source)만을 절대적인 진리로 상정하고, 모델의 응답이 1비트도 이 진리를 벗어나지 않았음을 기계적으로 증명하는 ’지식 기반 오라클(Knowledge-Based Oracle)’의 구축 기법을 심층적으로 해부한다.
