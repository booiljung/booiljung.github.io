8.1.4 RAG 파이프라인 내 오라클의 위치: 검색(Retrieval) 단계와 생성(Generation) 단계

RAG는 모델이 응답을 한 번만 반환하고 끝나는 구조가 아니므로, 오라클 역시 파이프라인 길목 길목마다 포스(Post)를 세워야 한다.

flowchart LR
    Q[User Query] --> R(Retrieval Oracle)
    R -->|Fail| E1[Rewrite / Reject]
    R -->|Pass Context| G(Generation Oracle)
    G -->|Fail| E2[Regenerate / Warning]
    G -->|Pass| A[Final Output]
  1. 검색 오라클 (Retrieval Oracle): 사용자의 질문을 해결할 만한 문서(Context)를 DB에서 제대로 건져 올렸는가?
  2. 생성 오라클 (Generation Oracle): 건져 올린 문서만을 바탕으로 어긋남 없이 최종 답변을 요약 및 합성해 내었는가?
    이 두 지점을 분리 타격하지 않고 최종 결과에만 통합 점수를 매기면, 문제가 검색기의 리콜(Recall) 실패인지 모델의 멍청함인지 식별할 수 없다.
