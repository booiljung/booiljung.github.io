8.11.1 근사 최근접 이웃(ANN) 알고리즘이 야기하는 검색 결과의 순위 변동 문제

대부분의 벡터 DB는 연산 속도를 위해 정확한 K-NN(K-Nearest Neighbors) 대신 HNSW 기법 등 ANN(Approximate Nearest Neighbors) 방식을 쓴다.

  • 문제점: 똑같은 질문을 똑같은 DB에 던져도, 어제는 A문서가 1등으로 오고 오늘은 B문서가 1등으로 오는 순위 변동 현상(Rank Volatility)이 발생한다.
  • 오라클 해결책: 오라클 테스트 시 벡터 검색 노드에서 무작위성 시드(Seed)를 고정하거나, 평가용 골든 셋트를 돌릴 때만은 시스템 파라미터를 강제 변경하여 느리지만 100% 동일한 결과값을 내뱉는 완전 탐색 모드(Exact KNN 검색) 로 전환해야 한다. 이 통제 조건 없이는 오라클의 평가 점수조차 매일 널뛰기하게 된다.
