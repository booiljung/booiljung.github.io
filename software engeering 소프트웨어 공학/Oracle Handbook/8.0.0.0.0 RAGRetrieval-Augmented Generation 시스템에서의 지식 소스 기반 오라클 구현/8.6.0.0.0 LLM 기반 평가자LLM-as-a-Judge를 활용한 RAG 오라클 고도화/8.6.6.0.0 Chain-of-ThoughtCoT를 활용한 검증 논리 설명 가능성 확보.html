8.6.6 Chain-of-Thought(CoT)를 활용한 검증 논리 설명 가능성 확보

“이 RAG 답변은 쓸모없음. Fail.” 이라고 오라클이 스위치를 내리고 죽어버리면 개발자는 무엇을 고쳐야 할지 모른다. 시스템이 “왜 Fail인지“를 설명(XAI)해야 한다.

  • 프롬프트에 [Reasoning Step] 필드를 만들어, 최종 Pass/Fail 점수를 출력하기 직전에 심판 모델이 자신의 사고(Chain of Thought)를 먼저 타자 치듯 써 내려가게 강제하라.
  • (예: “검색 문맥의 3번째 줄에 명시된 환불 기간은 14일인데, 생성 봇의 답변은 30일이라고 서술하였다. 따라서 이 답변의 충실성은 완벽히 파괴되었다. score: 0”)
    이 CoT 과정은 디버깅 로그로 남을 뿐 아니라, 심판관 LLM 자체가 점수를 매기기 전 스스로의 인지적 오류를 교정(Self-correction)하도록 돕는 마법의 열쇠다.
