8.6.5 평가용 LLM의 편향(Bias) 제어 및 일관성 확보 방안

판사가 뇌물을 먹거나 오락가락한다면 오라클이라 부를 수 없다. LLM 심판은 텍스트의 순서나 장황함(Verbosity)에 쉽게 속아 넘어간다.

  • 위치 편향(Position Bias) 회피: (문맥, 봇답변, 정답지)를 줄 때 배열 순서를 섞어 2번 검증(Swap-test)하여 모두 Pass일 때만 통과시킨다.
  • 길이 편향 회피: 심판관 프롬프트에 [경고: 문장이 길거나 수사가 화려하다고 해서 높은 점수를 주면 안 되며, 오직 건조한 사실의 일치성만 평가할 것]이라는 시스템 인스트럭션을 대문자로 강제 삽입하여, 답변의 미사여구가 점수를 부풀리는 환각을 잘라낸다.
