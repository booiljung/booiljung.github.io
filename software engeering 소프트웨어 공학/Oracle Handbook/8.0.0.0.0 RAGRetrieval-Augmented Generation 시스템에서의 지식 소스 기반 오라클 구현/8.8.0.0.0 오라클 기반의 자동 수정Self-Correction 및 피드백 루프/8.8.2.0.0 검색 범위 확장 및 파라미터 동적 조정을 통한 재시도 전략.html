8.8.2 검색 범위 확장 및 파라미터 동적 조정을 통한 재시도 전략

2번째 시도에서도 오라클이 여전히 불충분함을 지적한다면 무식하게 똑같이 쿼리만 던져서는 안 된다.

검색 범위를 동적으로 이완(Relaxation)시키는 아키텍처.

  • 첫 시도: 밀집 벡터 탐색(Dense)으로 Top-K=3, Threshold 0.8의 깐깐한 망을 던짐.
  • 실패 후 두 번째 시도: RAG 컨트롤러가 키워드(BM25) 위주의 하이브리드 검색 모드로 스위칭 한 뒤 Top-K=10으로 광망을 펼쳐 문서량을 늘림.
    결국 이 늘어난 컨텍스트 덩어리를 다시금 메인 모델에 주입하여, 까탈스러운 오라클의 Faithfulness 채점기를 만족시킬 수 있는지 기계적 확률을 높인다.
