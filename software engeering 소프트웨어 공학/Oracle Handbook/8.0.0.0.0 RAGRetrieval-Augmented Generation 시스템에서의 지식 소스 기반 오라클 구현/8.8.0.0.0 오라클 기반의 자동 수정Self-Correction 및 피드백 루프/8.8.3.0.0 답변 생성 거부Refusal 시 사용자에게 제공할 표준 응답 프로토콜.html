8.8.3 답변 생성 거부(Refusal) 시 사용자에게 제공할 표준 응답 프로토콜

자가치유 루프(Max Retry)를 다 돌았음에도 오라클이 결코 초록불을 켜주지 않을 때는 장렬히 전사해야 한다. 여기서 “어버버” 하면서 횡설수설하면 오라클의 가치가 완전히 날아간다.

  • 오라클 기각 이벤트 발생 시, 프론트엔드는 통제되지 않은 그 어떤 토큰도 화면에 렌더링해서는 안 된다.
  • 사전에 결정론적으로 박아둔(Hard-coded) 완충 언어(Fallback String)로 리다이렉션 한다. 예: [사내 지식 베이스 검색 결과, 해당 질문에 대한 명확한 회사 규정을 찾을 수 없습니다. (검색 키워드: X, Y, Z)]
    이는 사용자 신뢰도를 잃지 않으면서도 기업 레벨에서 가장 안전하게 모델의 약점을 감싸는 최후의 UX 보루다.
