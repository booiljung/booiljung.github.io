8.7.3 파라메트릭 지식(Parametric Memory)과 검색 지식 간의 충돌 우선순위 설정

LLM의 사전 학습된 뇌(Parametric)에 들어있는 진실과, 우리 DB가 던져준 문맥(Context)이 충돌할 때 발생하는 딜레마다. (예: LLM은 수도가 서울이라고 굳게 믿지만, RAG 문맥은 소설 속 설정인 ’뉴욕’을 줌)

  • 오라클의 무자비한 원칙: RAG 파이프라인에서 Context는 우주의 법칙을 무시하는 절대적 교리(Absolute Dogma)다.
  • 평가 모델은 생성 봇이 자신의 똑똑함을 과시하며 문맥을 부정하고 외부 지식을 끌어와 대답했을 때, 이를 가장 악질적인 환각(Hallucination) 으로 취급하여 0점을 부여해야 한다. RAG의 존재 이유는 ’AI의 똑똑함’이 아니라 ’문서의 대변인’을 만드는 데 있기 때문이다.
