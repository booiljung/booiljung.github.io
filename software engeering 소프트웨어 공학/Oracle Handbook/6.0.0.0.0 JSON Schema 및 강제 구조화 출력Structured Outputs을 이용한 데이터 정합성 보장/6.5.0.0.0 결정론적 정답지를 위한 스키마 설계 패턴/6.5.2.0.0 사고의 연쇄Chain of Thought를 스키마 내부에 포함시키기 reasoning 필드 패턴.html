6.5.2 사고의 연쇄(Chain of Thought)를 스키마 내부에 포함시키기 (reasoning 필드 패턴)

LLM에게 다짜고짜 정답 JSON만 뱉으라고 하면, 모델은 생각할 시간(연산 토큰 공간)이 없어 바보 같은 결론을 낸다. 이를 해결하기 위해 답변 필드 “앞단“에 사고 과정을 의무적으로 서술하도록 스키마를 배치하는 패턴이다.

{
  "reasoning_steps": [
    "1. 사용자 불만에서 환불 요구 확인",
    "2. 규정상 15일 이내인지 날짜 대조",
    "3. 이에 따라 환불 거절이 맞음"
  ],
  "final_decision": "REJECT_REFUND"
}

반드시 final_decision 보다 먼저 reasoning_steps를 정의해야 한다. 그래야 모델이 추론 로그 토큰을 선형적으로 쭉 써 내려가며 문맥(Context)을 이해한 뒤 결론에 도달할 수 있다. 이 reasoning 필드는 나중에 개발자가 오라클 디버깅(Debugging) 용도로 활용할 때 완벽한 화이트박스 단서가 된다.
