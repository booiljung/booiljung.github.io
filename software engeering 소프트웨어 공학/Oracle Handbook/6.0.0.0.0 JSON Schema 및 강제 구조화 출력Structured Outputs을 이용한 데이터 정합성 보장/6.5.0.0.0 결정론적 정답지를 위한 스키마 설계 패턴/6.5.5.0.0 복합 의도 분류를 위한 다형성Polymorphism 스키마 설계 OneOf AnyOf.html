6.5.5 복합 의도 분류를 위한 다형성(Polymorphism) 스키마 설계 (OneOf, AnyOf)

한 가지 API 엔드포인트가 상황에 따라 여러 개의 다른 스키마 형태를 뱉어야 할 때가 있다 (예: 사용자 질문이 ’배송 조회’일 때와 ’환불 요청’일 때 필요한 파라미터가 다름).

JSON Schema의 oneOf 또는 anyOf 키워드를 활용하면 다형성 모델을 짤 수 있다.

"oneOf": [
  { "$ref": "#/definitions/TrackingRequest" },
  { "$ref": "#/definitions/RefundRequest" }
]

Pydantic에서는 Union[TrackingRequest, RefundRequest] (파이썬 3.10 이상에서는 TrackingRequest | RefundRequest) 타입으로 선언된다.
오라클은 이 다형성 객체를 전달받은 즉시 최상위의 판별자(Discriminator) 필드(예: intent)를 기준 삼아, 하위 프로퍼티들이 올바르게 타이핑(Typing) 되었는지 정밀 타격한다.
