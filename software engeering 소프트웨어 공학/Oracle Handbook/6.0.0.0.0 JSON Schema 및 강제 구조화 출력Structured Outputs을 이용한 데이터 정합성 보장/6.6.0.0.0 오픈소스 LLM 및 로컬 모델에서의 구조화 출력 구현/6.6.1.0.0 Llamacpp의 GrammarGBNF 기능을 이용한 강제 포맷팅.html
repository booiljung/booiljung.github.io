6.6.1 Llama.cpp의 Grammar(GBNF) 기능을 이용한 강제 포맷팅

Llama.cpp 모듈은 CPU와 경량화 환경을 파고든 오픈소스 추론 엔진의 핵심이다.
이 툴의 가장 파괴적인 기능 중 하나는 GBNF(GGML BNF) 문법 지원이다.

GBNF는 정규표현식(Regex)과 유사하지만 텍스트 추출이 아닌 토큰 생성기 강제용으로 쓰인다.

root ::= "{" ws "\"name\"" ws ":" ws string "," ws "\"age\"" ws ":" ws number "}"
string ::= "\"" [a-zA-Z]+ "\""
number ::= [0-9]+
ws ::= [ \t\n]*

Llama.cpp에 이 체계를 주입하면 모델이 GBNF 규칙 밖의 토큰을 고르는 확률을 수학적으로 0으로 고정시킨다. JSON뿐만 아니라 YAML 탭(Tab) 들여쓰기나 독자적 사내 템플릿 언어까지 100% 보장하는 결정론적 오라클을 구축할 때 쓰이는 극저단(Lowest-level) 무기다.
