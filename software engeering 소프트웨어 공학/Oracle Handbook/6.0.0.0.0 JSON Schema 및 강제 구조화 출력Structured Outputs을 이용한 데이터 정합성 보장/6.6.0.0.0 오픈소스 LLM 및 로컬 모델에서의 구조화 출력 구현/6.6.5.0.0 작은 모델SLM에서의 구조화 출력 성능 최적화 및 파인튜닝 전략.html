6.6.5 작은 모델(SLM)에서의 구조화 출력 성능 최적화 및 파인튜닝 전략

7B 매개변수 이하의 작은 모델(SLM)들은 Attention 폭이 좁아 아무리 JSON Schema를 강제해도 “중첩 딕셔너리의 브라켓(})을 닫지 않거나” 필드 매핑 순서를 까먹는 치명적인 약점을 갖는다.

오라클의 신뢰도를 높이기 위한 SLM 전용 튜닝 전략:

  1. No-nested Structure: SLM용 스키마는 절대 깊이를 1로 유지해야 한다. 복잡한 계층은 컴포넌트 호출 파이프라인상에서 각각 단일 필드 추출 API로 여러 번 분해해서 부르는 것이 안전하다.
  2. LoRA Fine-Tuning: 스키마 구조 변경 없이 수개월간 동일하게 유지된다면 파인튜닝이 구조화 성능 보장의 치트키가 된다.
    오라클을 설계할 땐 엔진 볼륨(SLM vs LLM)에 따라 인내할 수 있는 스키마의 깊이와 복잡도를 먼저 계측해야 한다.
