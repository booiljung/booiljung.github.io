6.7.2.2 자동 수정 과정에서 발생하는 데이터 의미 훼손(Semantic Loss) 예방

정규식 기반 자동 수정(Auto-fixing)이 늘 능사인 것은 아니다.
JSON 파서가 고장난 문자열을 강제로 딕셔너리로 마사지하는 과정에서, 쉼표의 위치가 어긋나거나 큰따옴표의 쌍이 잘못 매칭되어 원래 “10,000“이었던 금액이 “10“으로 짤려 파싱되는 ’의미 훼손(Semantic Loss)’이 발생할 수 있다.

오라클의 관점에서 가장 위험한 상황은 ’명백한 실패’가 아니라 ’조용히 변형된 성공’이다. 파서가 자동 수정을 가동했다면 이 성공은 반드시 Warning(경고) 레벨 로깅을 남기거나 confidence_score 패널티를 부여해야 하며, 금융같이 극도의 보수성을 요구하는 도메인에서는 자동 수정 옵션 자체를 끄고(disable) 차라리 LLM 재호출(Retry)을 일으키는 것이 윤리적이다.
