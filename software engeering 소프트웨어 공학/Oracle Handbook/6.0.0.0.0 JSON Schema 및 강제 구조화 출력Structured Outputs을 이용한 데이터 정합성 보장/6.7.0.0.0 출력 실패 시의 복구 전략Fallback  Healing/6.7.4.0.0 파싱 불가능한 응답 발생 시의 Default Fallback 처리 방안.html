6.7.4 파싱 불가능한 응답 발생 시의 Default Fallback 처리 방안

3번의 피드백 루프조차 통과하지 못했다면 시스템은 고장난 프롬프트를 포기하고 파이프라인의 다음 노드(Node)에 ‘통제된 실패(Controlled Failure)’ 상태를 넘겨야 한다.

try:
    # LLM 호출 및 검증 로직 실행
    structured_data = chain.invoke(...)
except ParseError:
    # 파싱 불가능 시 Default 폴백 발동
    structured_data = {
      "status": "FAIL",
      "extracted_data": null,
      "fallback_reason": "MAX_RETRY_EXCEEDED"
    }

결정론적 오라클의 철칙은 “예외(Exception)를 밖으로 던져 서버를 죽이지 말고(Crash), 실패 상태를 명시한 기본(Default) 구조체를 반환해라“는 것이다. 이를 통해 인간 검수자(HITL) 라우팅 등의 다음 비즈니스 프로세스로 안전하게 연결할 수 있다.
