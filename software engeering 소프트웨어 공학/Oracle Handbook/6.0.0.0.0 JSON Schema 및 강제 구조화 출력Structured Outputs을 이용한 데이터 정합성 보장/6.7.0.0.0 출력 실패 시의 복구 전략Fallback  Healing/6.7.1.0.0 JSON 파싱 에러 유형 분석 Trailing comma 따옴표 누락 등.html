6.7.1 JSON 파싱 에러 유형 분석 (Trailing comma, 따옴표 누락 등)

오라클이 받아든 텍스트가 파싱에 실패했다면, 무작정 재시도(Retry)를 때릴 것이 아니라 오류의 원인을 디버깅해야 한다.

  • Trailing Comma: {"name": "A", "age": 20,} 처럼 마지막 프로퍼티 뒤에 쉼표가 붙는 현상. LLM이 속성을 이어 쓰려다 멈출 때 자주 발생한다.
  • Unescaped Quotes: 원문 텍스트 내부에 이미 큰따옴표가 있는데, 이스케이프(\") 처리를 하지 않고 JSON 내부에 주입할 때 발생한다. (예: {"review": "He said "Great""})
  • Markdown Wrappers: json ... 같은 불필요한 마크다운 블록이 텍스트의 처음과 끝을 감싸는 현상.

단순한 오라클이라면 이를 빌드 실패 처리하겠지만, 견고한 파이프라인이라면 이 정도의 경미한 문법 에러는 정규표현식 클렌저(Cleanser)를 통해 1차 치유(Heal)를 시도해야 한다.
