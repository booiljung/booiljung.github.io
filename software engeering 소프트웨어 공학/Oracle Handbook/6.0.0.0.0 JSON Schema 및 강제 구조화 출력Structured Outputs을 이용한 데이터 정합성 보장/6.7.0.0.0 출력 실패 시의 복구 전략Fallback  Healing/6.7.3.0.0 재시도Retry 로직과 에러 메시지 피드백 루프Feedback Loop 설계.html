6.7.3 재시도(Retry) 로직과 에러 메시지 피드백 루프(Feedback Loop) 설계

LLM은 자신이 실수한 것을 곧바로 지적해 주면(Self-Correction) 두 번째 시도에서는 거의 100% 확률로 포맷을 준수하는 특성이 있다.
에러 피드백의 필수 요소:

  1. 기존 응답 삽입: 네가 지난번에 뱉은 텍스트는 이러했다 (Previous Output).
  2. 에러 원문 삽입: 파이썬 인터프리터가 던진 날것의 밸리데이션 에러 코드를 그대로 보여준다 (Pydantic ValidationError: 1 validation error for DeliveryInfo...).
  3. 재지시: “위 에러를 참고하여 다음 JSON Schema에 완벽히 부합하도록 다시 작성해라.”

이 재시도 트랜잭션(Transaction)은 프롬프트 체인 상에서 최대 3회(Max Retries = 3)까지만 허용하도록 제한(Rate Limit)을 묶어야 비용 폭발을 막을 수 있다.
