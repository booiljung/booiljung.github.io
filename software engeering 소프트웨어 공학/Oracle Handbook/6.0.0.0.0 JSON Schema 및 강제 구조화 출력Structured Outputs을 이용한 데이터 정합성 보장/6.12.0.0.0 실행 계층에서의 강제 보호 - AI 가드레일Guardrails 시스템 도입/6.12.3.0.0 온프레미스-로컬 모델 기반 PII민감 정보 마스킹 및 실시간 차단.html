6.12.3 온프레미스/로컬 모델 기반 PII(민감 정보) 마스킹 및 실시간 차단

LLM 파이프라인에서 보안 사고는 곧 회사의 존폐와 직결된다. 특히 OpenAI 등 외부 API를 탈 때 가장 우려되는 것은 개인 민감 정보(PII; Personal Identifiable Information)의 유출이다.

로컬 기반 가드레일(Local Guardrail)의 역할:

  1. 아웃바운드 필터링(Outbound Filtering): 유저 프롬프트가 외부 망으로 나가기 전, 로컬에 구축된 Presidio 같은 PII 인식 엔진이 먼저 주민등록번호, 신용카드 번호, 여권 번호를 정규식 및 허깅페이스 토크나이저를 통해 잡아낸다.
  2. 마스킹 대치: [SSN-MASKED] 형태로 익명화 처리한 뒤 안전한 텍스트만 외부 오픈 API로 발송한다.
  3. 오라클의 기준선: “이 시스템은 어떠한 상황에서도 평문 PII를 허용하지 않는다“는 가드레일 정책 자체가 보안 규정 준수(Compliance) 오라클의 가장 강력한 합격/불합격 판정 기준이 된다.
