6.12.1 구조화 출력을 보완하는 런타임 제약 조건 엔진 (Guardrails AI, NeMo Guardrails)

JSON Schema가 ’형태’를 강제한다면, 전문 가드레일 엔진은 ’내용과 컨텍스트’를 강제한다.
대표적인 프레임워크인 Guardrails AI 또는 NVIDIA의 NeMo Guardrails는 프록시(Proxy) 형태로 LLM과 애플리케이션 사이에 위치한다.

  • 작동 방식: 애플리케이션이 LLM에 던지는 인풋(Input), 그리고 LLM이 반환하는 아웃풋(Output)의 양방향 패킷을 인터셉트(Intercept)한다.
  • 검증 엔진: 사전에 정의된 *.co 파일(Colang)이나 XML 파일에 기반해 텍스트의 유해성 검증, 경쟁사 언급 여부, 지시 위반 여부 등을 작고 빠른 별도의 평가 모델이나 휴리스틱 룰 엔진으로 0.1초 만에 검문한다.
    통과하지 못한 응답은 즉시 오라클 ‘실패’ 처리되며, 사용자에게는 [Blocked by Guardrail 정책]이라는 명시적인 예외를 던진다.
