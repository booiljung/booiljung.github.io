6.4.4 유효성 검사 로직(Validator) 심기: LLM 생성 시점의 제약 조건

구조화된 출력 기능이 타입(Type)을 강제해 주지만, 도메인 특화(Domain-specific) 로직까지 알아서 계산해 주진 않는다.
LLM이 타입은 맞췄으나 내용은 틀린 데이터(age: -5)를 보낼 때, Pydantic의 @field_validator는 이를 런타임에서 방어하는 방파제가 된다.

from pydantic import BaseModel, field_validator

class DeliveryInfo(BaseModel):
    zip_code: str
    
    @field_validator('zip_code')
    def must_be_valid_kr_zip(cls, v):
        if len(v) != 5 or not v.isdigit():
            raise ValueError(f"우편번호는 5자리 숫자여야 합니다. (입력값: {v})")
        return v

여기서 발생한 ValueError는 단순한 크래시(Crash)로 끝나지 않고, 이후 6.7절에서 다룰 재시도(Retry) 기법의 에러 메시지로 재주입(Feed-forward)되어 LLM이 자기 수정(Self-correction)을 하도록 돕는 결정적 단서가 된다.
