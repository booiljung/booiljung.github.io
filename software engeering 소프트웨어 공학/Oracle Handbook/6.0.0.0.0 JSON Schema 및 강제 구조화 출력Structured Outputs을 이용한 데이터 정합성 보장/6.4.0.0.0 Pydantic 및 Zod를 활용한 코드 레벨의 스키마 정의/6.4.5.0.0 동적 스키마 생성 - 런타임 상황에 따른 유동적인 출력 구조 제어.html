6.4.5 동적 스키마 생성: 런타임 상황에 따른 유동적인 출력 구조 제어

비즈니스의 요구 조건은 사용자(User)의 상태(State)에 따라 시시각각 변한다.
VIP 회원의 정보를 추출할 때와 비회원의 정보를 추출할 때 필요한 필수 필드의 개수가 다를 수 있다.

이때는 클래스를 하드코딩하지 않고 함수 코드를 통해 런타임에 동적으로(Dynamic) Pydantic 모델 클래스를 팩토리(Factory) 형태로 생성해낸다.

from pydantic import create_model

def get_dynamic_schema(is_vip: bool):
    fields = {'name': (str, ...), 'age': (int, ...)}
    if is_vip:
        fields['vip_code'] = (str, ...) # VIP일 때만 이 필드가 필수가 됨
    
    return create_model('DynamicProfile', **fields)

이러한 동적 프로그래밍이야말로 고정된 JSON 스크립트 파일이 아닌 오직 애플리케이션 코드로 스키마를 렌더링해야 하는 강력한 엔지니어링적 명분이다.
