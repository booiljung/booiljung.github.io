6.9.4 스키마 캐싱(Schema Caching)을 통한 요청 처리 속도 개선

구조화 추출 엔진의 문법 컴파일(Cold Start) 지연을 지우는 백엔드 기술이다.
OpenAI API는 사용자가 제출한 JSON Schema 규격이 이전 요청과 완벽히 바이트 단위로 동일할 경우, 두 번째 호출부터는 생성된 CFG 오토마타를 캐시에서 재사용(Zero-latency overhead)한다.

  • 체크포인트: 스키마 객체 내에 동적으로 변하는 키값(Timestamp 등)이 섞여 들어가면 캐시가 무효화(Cache Miss)되어 매번 수백 밀리초의 오버헤드를 얻어맞게 된다. 스키마 자체는 100% 정적으로 캐시에 얼려 두고, 변동값은 시스템 프롬프트(System Prompt)로 분리해서 주입하는 것이 자원 최적화의 기본 조건이다.
