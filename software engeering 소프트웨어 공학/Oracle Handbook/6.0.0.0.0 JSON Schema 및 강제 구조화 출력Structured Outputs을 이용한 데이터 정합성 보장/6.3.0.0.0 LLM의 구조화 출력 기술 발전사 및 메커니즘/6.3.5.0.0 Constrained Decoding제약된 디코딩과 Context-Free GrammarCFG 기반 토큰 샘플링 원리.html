6.3.5 Constrained Decoding(제약된 디코딩)과 Context-Free Grammar(CFG) 기반 토큰 샘플링 원리

현존하는 구조적 통제 기술 중 가장 로우 레벨(Low-level)이면서도 완벽에 가까운 기술이다.

LLM이 다음 토큰을 뱉을 때 확률 분포를 계산하는데, Constrained Decoding 엔진은 이 과정에 개입하여 “스키마(혹은 Regex)에 위배되는 토큰“의 생성 확률을 0으로(Logit Penalty = -inf) 강제 차단한다.
예를 들어 {"age": 뒤에 올 토큰은 오직 0~9 또는 공백이어야 한다. 알파벳 토큰들이 샘플링 계산에서 원천 차단되기 때문에, 모델이 아무리 알파벳을 뱉어내고 싶어 미치려 해도 시스템은 숫자로 된 트리 노드만을 강제 탐색하게 만든다. CFG(Context-Free Grammar)는 이러한 규칙 트리를 정의하는 수학적 명세다. 이것이 적용된 상태에서 스키마 이탈은 수학적으로 불가능하다.
