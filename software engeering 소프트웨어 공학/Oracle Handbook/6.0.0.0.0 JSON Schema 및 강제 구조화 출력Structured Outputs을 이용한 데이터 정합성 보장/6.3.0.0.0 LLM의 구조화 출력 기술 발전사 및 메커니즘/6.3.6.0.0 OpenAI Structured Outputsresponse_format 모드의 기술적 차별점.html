6.3.6 OpenAI Structured Outputs(response_format) 모드의 기술적 차별점

2024년 8월, OpenAI는 단순히 프롬프트를 조작하는 것을 넘어선 Structured Outputs 모드를 발표했다.
개발자가 response_format 파라미터 안쪽 객체에 strict: true 옵션을 켜서 JSON Schema를 전달하면, OpenAI 서버 측 엔진이 위에서 설명한 Constrained Decoding을 네이티브로 수행한다.

  • 100% Reliability: 타당한 스키마가 주어지는 한 모델은 스키마의 키값 누락, 타입 불일치 위반을 단 한 글자도 범하지 못한다.
  • Cold Start Latency: 최초에 주어진 스키마를 CFG 오토마타 단위로 파싱하는데 미세한 딜레이가 걸릴 수 있으나, 스키마 캐싱(Schema Caching) 최적화를 통해 후속 요청을 비약적으로 가속시킨다.
    결정론적 오라클을 구축하는 파이프라인에서 이 기능은 더 이상 선택이 아닌 필수 표준(De-facto Standard)이 되었다.
