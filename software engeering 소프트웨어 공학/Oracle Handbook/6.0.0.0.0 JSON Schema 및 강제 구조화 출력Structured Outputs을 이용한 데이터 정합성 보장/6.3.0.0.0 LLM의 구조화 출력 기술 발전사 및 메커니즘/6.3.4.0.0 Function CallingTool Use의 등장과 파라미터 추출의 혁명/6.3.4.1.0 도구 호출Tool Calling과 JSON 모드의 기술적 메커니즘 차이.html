6.3.4.1 도구 호출(Tool Calling)과 JSON 모드의 기술적 메커니즘 차이

LLM API에는 response_format: {"type": "json_object"} 포맷 지정 기능과, tools: [...] 파라미터가 동시에 존재한다.

  • JSON 모드: 모델이 마크다운 래퍼 없이 철저히 유효한 파서블(Parseable) JSON 생성을 책임지도록 유도한다. 단, 이 모드는 “어떤 스키마(Schema)를 지켜야 하는가“를 완전히 보장해 주지는 않는다. 필드 누락이나 키 이름의 변형이 발생할 수 있다.
  • 도구 호출 (Tool Calling): 내부적으로 JSON Schema를 주입하여 모델이 “이 함수를 호출하려면 어떤 인자가 반드시 필요한지“를 인지하게 만든다. 토큰 예측 과정이 스키마의 제약을 직접 받으므로, 훨씬 더 견고한 오라클 결과를 도출한다. 의도 추출기나 데이터 파서의 목적으로는 JSON 모드보다 Tool Calling이 강력히 권장된다.
