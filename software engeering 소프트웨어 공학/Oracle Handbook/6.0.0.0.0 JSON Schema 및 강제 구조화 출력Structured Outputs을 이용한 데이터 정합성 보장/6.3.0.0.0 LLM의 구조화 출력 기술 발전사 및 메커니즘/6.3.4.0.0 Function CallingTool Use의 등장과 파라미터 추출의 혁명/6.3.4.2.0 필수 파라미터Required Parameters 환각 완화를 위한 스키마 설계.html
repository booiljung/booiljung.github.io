6.3.4.2 필수 파라미터(Required Parameters) 환각 완화를 위한 스키마 설계

자연어 지시문인 유저 프롬프트와 기계적 규격인 Tool Schema 간의 괴리가 환각(Hallucination)을 낳는다. “이메일 안의 전화번호를 뽑아줘“라고 했는데 원문에 전화번호가 없으면, 모델은 스키마의 required 조건을 맞추기 위해 가짜 번호(010-0000-0000)를 지어낸다.

이를 방어하는 최적의 오라클 스키마 설계:

  1. 해당 필드의 type을 ["string", "null"] 형태로 Nullable하게 선언한다.
  2. 필드의 description에 반드시 원문에 이 정보가 없다면 절대 유추하지 말고 null을 반환하라(Do not guess, return null if not found in context) 라는 시스템 지시를 삽입한다.
    이 두 가지 조합만이 LLM의 ’지어내는 습성’을 합법적인 ’결측치(Null) 처리’로 강제 우회시킬 수 있다.
