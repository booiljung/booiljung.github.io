6.8.3 비즈니스 로직 제약 조건(Constraints) 검증 자동화

JSON Schema를 넘어 애플리케이션 계층(Pydantic/Zod)과 연동된 구조화 출력은 그 자체로 도메인 지식을 시험하는 2차 오라클로 동작한다.

  • check_in_date가 check_out_date보다 과거여야 한다는 런타임 검증 로직을 클래스에 심어둔다.
  • 테스트 파이프라인이 수십 개의 상황극(Role-play) 프롬프트를 발송한다.
  • LLM이 날짜의 선후 관계를 유추하지 못하고 모순된 데이터를 반환하면 클래스 인스턴스화 과정에서 ValidationError가 터진다.
    이 에러 리포트 자체가 AI 모델이 ’시간적 인과관계’라는 비즈니스 상식을 얼마나 잘 이해하고 있는지를 측정하는 채점표(Grader)가 된다.
