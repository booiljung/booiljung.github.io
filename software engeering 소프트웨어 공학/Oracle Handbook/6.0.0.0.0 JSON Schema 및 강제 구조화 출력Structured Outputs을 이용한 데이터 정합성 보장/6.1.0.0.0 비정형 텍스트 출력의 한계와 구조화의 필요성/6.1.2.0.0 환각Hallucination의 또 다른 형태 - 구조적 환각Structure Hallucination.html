6.1.2 환각(Hallucination)의 또 다른 형태: 구조적 환각(Structure Hallucination)

환각(Hallucination)은 내용적 측면에만 국한되지 않는다.
LLM은 때로 지식은 정확하게 인출하면서도 개발자가 요구한 출력 ‘구조’ 자체를 임의로 왜곡하는 구조적 환각(Structure Hallucination) 을 일으킨다.

  • 예: 파라미터로 명시적으로 규정한 user_name 키를 자의적으로 UserName이나 이름으로 바꿔서 반환하는 행위.
  • 예: 숫자가 들어가야 할 나이(age) 속성에 "Thirty"라는 문자열(String)을 쑤셔 넣는 행위.

결정론적 오라클의 입장에서, 구조적 환각은 정보의 오답과 동일한, 빌드를 실패(Fail)시켜야 할 치명적인 버그(Bug)다. 데이터 계약(Data Contract)이 지켜지지 못했다면 정보의 사실 여부를 평가하는 2차 파이프라인으로 넘어갈 자격조차 박탈해야 한다.
