6.1.4 정규 표현식(Regex) 기반 추출의 한계와 위험성

프롬프트 엔지니어가 구조화를 포기하고 가장 쉽게 타협하는 방식이 결괏값을 정규표현식으로 긁어내는 것이다.
하지만 정규식은 자연어의 창의성을 담아낼 수 없는 완고한 족쇄다.

  • 예상 밖의 포맷: “금액을 적어줘“라고 했을 때 정규식 \d+를 쓰면, 달러화 기호($)나 유럽식 쉼표 표기(1.000,00)가 들어온 순간 코드는 즉시 다운된다.
  • 멀티라인 처리 한계: 여러 항목이 줄바꿈으로 나열된 텍스트에서 정규표현식은 백트래킹(Backtracking) 폭발을 일으키거나, 엉뚱한 맥락에 섞인 비슷한 단어를 오탐지(False Positive)하게 된다.

정규표현식을 오라클로 쓰는 것은 단일 마커(Marker) 검증에 한해서만 허용되어야 하며, 데이터의 계층적 트리를 통째로 파싱하겠다는 시도는 기술 부채의 지름길이다.
