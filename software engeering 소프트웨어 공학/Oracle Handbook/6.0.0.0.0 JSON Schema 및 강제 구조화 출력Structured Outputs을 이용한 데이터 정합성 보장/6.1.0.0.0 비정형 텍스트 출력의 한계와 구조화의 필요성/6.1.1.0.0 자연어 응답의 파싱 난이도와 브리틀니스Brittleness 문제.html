6.1.1 자연어 응답의 파싱 난이도와 브리틀니스(Brittleness) 문제

자연어는 그 자체로 모호성(Ambiguity)의 결정체다.
프롬프트에서 아무리 “목록 형태로 결과를 나열해 줘“라고 강제하더라도, LLM은 첫인사(“네, 알겠습니다!”), 끝맺음말(“도움이 되셨기를 바랍니다.”), 혹은 불규칙한 앞 공백 등을 기분 내키는 대로 추가한다.

이러한 사소한 텍스트 기만은 전통적인 파서(예: JSON.parse())를 즉시 붕괴시킨다. 시스템은 본질적으로 깨지기 쉬운 취성(Brittleness)을 갖게 되며, 개발자는 언제 터질지 모르는 파싱 에러를 잡기 위해 try-catch 블록과 너덜너덜한 정규표현식 보수 공사에 매일 밤을 허비하게 된다. 이 악순환의 고리를 끊기 위해선 파싱 로직의 책임 자체를 LLM의 토큰 생성 과정(Generation Phase)으로 이전(Shift-left)해야 한다.
