11.8.4 지속적인 학습을 위한 데이터 파이프라인 자동화

오라클 불일치 에러가 뜬 모든 세션 데이터는 암호화되어 Data Lake에 낙하(Drop)해야 한다.

  • 밤마다 Airflow 기반의 배치 배치가 돌아가 이를 파싱하고, MLOps 담당자의 휴먼 리뷰 대시보드(Human-in-the-loop)로 푸시한다.
  • 여기서 승인된 수정안(Corrected Pair)만이 다음 날 아침 빌드의 CI/CD 파이프라인을 타고 골든 데이터셋(v1.6 -> v1.7)으로 재탄생하는 멈추지 않는(Continuous) 학습 사이클 아키텍처 다이내믹스가 완성된다.
