11.10.3 도구 실행 결과를 반영한 챗봇의 최종 응답 비교 검증 로직

도구(Tool) 호출이 성공하고 오라클 API가 {"status": "success", "ticket_number": "T-999"}를 리턴했을 때, AI가 유저에게 마지막 메시지를 던지는 단계다.

  • 이 단계에서도 AI가 환각을 일으켜 “예약 번호는 T-123입니다“라며 숫자를 오염시킬 수 있다.
  • 포스트 오라클 검증(Post-validation): 최종으로 출력될 문장 스트링을 가로채어 그 안에 T-999 라는 도구 리턴값이 정확한 정규식으로 박혀 있는지(Existence Check) 동적으로 검증한 후에야 유저 프론트 화면으로 렌더링을 허용한다.
    이는 도구의 시작부터 끝까지 무지성 AI의 폭주를 막는 폐쇄회로(Closed-Loop) 오라클 설계의 백미다.
