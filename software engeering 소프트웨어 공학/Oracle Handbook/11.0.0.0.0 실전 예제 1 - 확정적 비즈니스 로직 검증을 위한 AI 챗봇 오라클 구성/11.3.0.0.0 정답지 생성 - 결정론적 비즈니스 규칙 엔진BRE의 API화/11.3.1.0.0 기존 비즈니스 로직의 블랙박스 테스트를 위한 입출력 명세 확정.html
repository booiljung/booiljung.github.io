11.3.1 기존 비즈니스 로직의 블랙박스 테스트를 위한 입출력 명세 확정

오라클은 본질적으로 그 내부를 알 필요가 없는 블랙박스(Black-box) 함수여야 한다. f(x) = y 에서 x 와 y 의 스키마(Schema)를 정의하라.

  • Input (x): {"age": int, "has_accident": boolean}
  • Output (y): {"is_approved": boolean, "premium": int, "reason": string}
    기존 레거시 개발팀과 협의하여 이 입출력 규격을 고정(Freeze)하는 것이 첫 번째 미션이다. 이 스키마가 흔들리면 AI 파이프라인 전체의 테스트 신뢰도가 붕괴된다.
