11.3.3 오라클의 출력 정규화: JSON 포맷을 통한 검증 가능 데이터 구조 정의

오라클의 리턴 값은 자바(Java) 객체나 파이썬(Python)의 튜플 같은 언어 종속적 형태여서는 안 된다. 오라클 시스템은 궁극적으로 다국어 마이크로서비스로 통신해야 하므로 반드시 JSON 표준 포맷으로 직렬화(Serialization) 되어야 한다.

{
  "status": "SUCCESS",
  "data": {
    "is_approved": true,
    "premium": 60000
  },
  "oracle_version": "1.0.4"
}

여기서 oracle_version 메타데이터 태깅은 필수다. 향후 정책이 바뀌었을 때 “왜 이 테스트가 실패하는가?“를 디버깅하려면 구형 오라클 버전으로 채점했는지, 신형 버전으로 채점했는지의 출처(Provenance) 보장이 필요하기 때문이다.
