11.5.3.1 상태 불일치(State Mismatch) 해결을 위한 양방향 동기화(Two-way Sync) 구조

테스트 환경에서 오라클의 응답과 AI의 응답이 불일치하는 흔한 이유는, 테스트 도중 오라클 데이터베이스의 기준 정보가 바뀌었기 때문이다(예: 기본 요금 5만 원 -> 6만 원).

  • 테스트 런타임 진입 시 데이터베이스 모킹(Mocking) 인터페이스를 통해 챗봇 세션과 검증 오라클 세션에 동일한 “Time-freezed State“를 강제(Injection)해야 양방향 동기화가 유지된다.
