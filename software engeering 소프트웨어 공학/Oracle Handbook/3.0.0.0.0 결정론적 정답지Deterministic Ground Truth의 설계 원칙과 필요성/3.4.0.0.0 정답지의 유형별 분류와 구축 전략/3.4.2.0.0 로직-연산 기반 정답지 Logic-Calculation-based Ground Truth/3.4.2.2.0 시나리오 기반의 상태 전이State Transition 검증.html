3.4.2.2 시나리오 기반의 상태 전이(State Transition) 검증

챗봇이나 자율 에이전트(Autonomous Agent)는 단 한 번의 질의응답으로 끝나지 않는다.
“[상품조회] -> [장바구니 담기] -> [결제] -> [환불]” 이라는 사용자 발화의 지그재그식 연속 시나리오 안에서, AI가 시스템의 상태계를 망가뜨리지 않고 올바르게 전이(Transition) 했는지 검증할 때 상태 전이 정답지가 요구된다.

1. 상태 전이 정답지의 구성

이 정답지는 단일 토큰 매치가 아닌 일련의 플로우 차트를 검사한다.

  1. Step 1: 사용자가 결제를 요청함. (State: Payment_Ready)
  2. Step 2: AI가 신용카드 번호 입력을 요청함. (State Transition: Waiting_For_Card_Info)
  3. Step 3: 사용자가 엉뚱한 욕설을 입력함.

이때 정답지 내부의 오라클은 “욕설 입력 시에는 강제로 State를 Error_Handling 으로 전이시킨다“는 메타 룰셋을 들고 있다. 만약 AI가 욕설을 받고도 결제를 승인해버려 상태계가 Payment_Success로 넘어갔다면 오라클은 시스템 크래시(Crash) 방어를 위해 이 시나리오 테스트를 붉은 불로 점등시킨다.
