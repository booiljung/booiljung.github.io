3.4.2.1 수학적 연산 결과 및 코드 실행 결과의 일치 여부

“1년 차 사원의 연봉이 3,000만 원일 때, 10% 상승한 2년 차 연봉과 보너스 500만 원의 합계를 계산해 주시오.”

이런 복합 수학적 프롬프트에 대해 우리는 AI가 “합계는 3,800만 원입니다“라고 찍어 맞추는 결과를 정답으로 맹신할 수 없다. 연산 기반 정답지를 갖춘 오라클은 AI의 출력값을 문자열이 아닌 실행 가능한 코드(Executable Code)나 수학적 다항식으로 강제 추출하여, 격리된 샌드박스(Sandbox) 안에서 실제로 돌려본 결괏값을 정답지와 매치시킨다.

  • 절차적 오라클: AI 파이프라인(Agent)이 이 문제를 해결하기 위해 백그라운드에 생성한 파이썬 연산식(3000 * 1.1 + 500)을 AST(추상 구문 트리)로 가로채 추출한다.
  • 그리고 이 함수를 돌린 결과가, 정답지에 하드코딩해놓은 Integer 값 3800에 떨어지는지 결정론적으로 판정한다.
    결과뿐만 아니라 도출되는 프로그램(Code)의 런타임 값까지 정답지로 통제하는 이 방식이 Text-to-SQL이나 코드 코파일럿 검증의 기본 폼이다.
