3.4.3.1 JSON Schema, Pydantic 모델을 이용한 필드 유효성 검사

“이력서 PDF를 읽고 주요 항목을 뽑아주세요“라는 요청에 AI가 아무리 기깔나는 문장으로 분석 리포트를 썼다 한들, 사내 HR 데이터베이스에 저장할 수가 없으면 무용지물이다.
이때 정답지는 파이썬(Python)의 Pydantic 클래스나 JSON Schema 정의 파일 그 자체다.

1. 필드 검증 오라클 (Field Validator Oracle)

테스트 파이프라인에서 AI가 뱉어낸 출력을 즉시 이 정답 스키마에 load 시도한다.

# 구조 기반 정답지 (Pydantic 모델)
class ResumeOutput(BaseModel):
    name: str
    experience_years: int
    matched_skills: list[str]

AI 출력이 경력: 5년 같은 줄글로 튀어나오면 experience_years 에 매핑되지 않아 ValidationError가 터지며 즉시 오라클 Fail이 격발된다. 이 구조 기반 정답지는 환각 확률 0%로 모든 기형적 토큰을 API 서버 앞에서 요격하는 최고의 가드레일이다.
