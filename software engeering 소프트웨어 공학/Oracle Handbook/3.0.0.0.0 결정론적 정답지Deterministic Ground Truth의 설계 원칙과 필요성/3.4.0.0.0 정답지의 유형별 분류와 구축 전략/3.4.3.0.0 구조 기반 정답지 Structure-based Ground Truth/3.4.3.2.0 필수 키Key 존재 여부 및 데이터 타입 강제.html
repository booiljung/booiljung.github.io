3.4.3.2 필수 키(Key) 존재 여부 및 데이터 타입 강제

구조 기반 검증의 다음 단계는 단순히 JSON이 정상적으로 구문 분석(Parsing) 되는가를 넘어, 그 내부의 세밀한 자료형 생태계를 박제시키는 것이다.

  1. 타입(DataType) 강제: AI 모델이 숫자를 쓸 때 자꾸 따옴표를 붙여 문자열(String) "100" 으로 넘기는 오염을 잡아내기 위해, 오라클의 정답지는 Expected Type: Integer(100) 임을 명시하고 타입 캐스팅 실패 시 엄격히 차단한다.
  2. 필수 키(Required Key) 룰셋: 고객 불만 챗봇이 출력할 결과에 complaint_level(불만 강도) 필드는 데이터가 없더라도 결코 테이블에서 누락되어서는 안 된다고 정답지에 선언(required: true)해 둔다. 만약 모델이 마음대로 키값을 빼버리고 User_Name만 리턴한다면, 이 트랜잭션은 Fail이다.

이러한 규격화 검증은 소프트웨어 애플리케이션의 타입 세이프티(Type Safety)를 AI 출력 구간으로 확장하는 핵심 기법이다.
