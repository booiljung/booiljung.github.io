3.4.4.1 임베딩 벡터 유사도(Cosine Similarity)의 임계값(Threshold) 설정

“사랑해“와 “정말 좋아해“는 철자가 1바이트도 겹치지 않지만 의미적(Semantics)으로는 95% 이상 일치하는 등가 언어다. 이를 기계 오라클이 이해하게 만드는 기술이 임베딩 벡터(Embedding Vector)다.

1. 임계값을 통한 부분적 결정론 획득

준결정론적 정답지 파이프라인은 다음과 같이 작동한다.

  1. 인간이 작성해 둔 기준 정답지 G(“시스템에 로그인해 주세요”)를 임베딩 모델(예: text-embedding-ada-002)을 통해 다차원 수(Numeric Vector)로 인코딩한다.
  2. AI가 구동되어 A(“접속을 위해서 로그인을 부탁드립니다”)라는 문자열을 내뱉는다. 이 역시 수열 벡터로 인코딩한다.
  3. 오라클은 두 벡터 간의 코사인 유사도(Cosine Similarity) 를 수리적으로 측정한다. (이때 점수는 0.91로 계산된다).

여기까지는 확률의 영역이다. 하지만 이 파이프라인을 ’결정론적’으로 탈바꿈시키는 것은 미리 설계해 둔 하드 임계값(Hard Threshold) 이다.
정답지 메타데이터에 Threshold >= 0.85; Verdict=Pass 라고 하드코딩해놓는 순간, 이 평가는 더 이상 주관의 영역이 아니라 0.91 > 0.85 == True 라는 완벽한 결정론적 논리식(Boolean)으로 승화되어 CI/CD 파이프라인의 다음 관문을 열어젖히게 된다.
