3.7.1 과적합(Overfitting) 유발: 테스트 데이터가 프롬프트에 유출(Leakage)되는 경우

전통적 머신러닝에서 가장 경계하는 데이터 유출(Data Leakage) 현상은 프롬프트 엔지니어링에서도 부지불식간에 일어난다.

  • 안티 패턴: QA 팀이 100개의 골든 정답지를 만들었다. 개발팀은 이 100개의 질문과 정답지를 통과하기 위해서만 시스템 프롬프트(System Prompt)를 Few-shot 기법으로 우겨 넣는다.
  • 결과: 모델은 100개의 테스트 셋에서 100점을 맞지만, 프로덕션의 101번째 생경한 질문형식 앞에서는 즉각 환각(Hallucination)을 내뿜는다. 완벽한 프롬프트 과적합(Prompt Overfitting) 의 도래다.

결정론적 정답지는 철저하게 Train Set과 Test(Oracle) Set으로 완벽히 에어갭(Air-gap) 분리되어야 하며, 개발팀은 오라클이 쥐고 있는 정답지 목록에 결코 접근하지 못하게(Blind Testing) 블라인드 처리해 두어야 한다.
