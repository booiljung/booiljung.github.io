3.7.3 문맥(Context)을 무시한 단편적 사실 검증

정답지가 너무 원자(Atomic) 단위로 잘게 쪼개져 팩트의 유무만 검사하다 보면, 전체 문맥이 모순되는(Contradictory) 코미디 같은 출력이 통과되는 사고가 터진다.

  • 안티 패턴:
  • 질문: “iPhone 15의 가격과 출시 여부는?”
  • 정답지: ["120만 원", "출시됨"]
  • AI의 출력: “120만 원이면 좋겠지만 아직 출시되지 않았고 향후 출시될지도 미지수입니다.”
  • 오류 발생: AI의 문장 속에 정답지가 기대했던 120만 원과 출시라는 단어가 모두 들어있기 때문에 토큰 검색 오라클은 이를 합격(Pass) 판정한다. 전혀 반대의 대답을 내놓았음에도 불구하고.

이 함정을 우회하기 위해 문맥 검증은 단순히 참/거짓 단어의 유무를 찾는 단계에서 한 단계 격상되어 자연어 추론(NLI, Natural Language Inference) 검증기 등을 사용해 정답 전제 조건이 답변에 내포(Entailment)되어 있는지 문맥적으로 확인해야 한다.
