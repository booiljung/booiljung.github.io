3.7.5 평가 모델(Judge Model) 편향성: 특정 LLM이 생성한 정답을 선호하는 현상

이 현상은 LLM-as-a-Judge(LLM을 심판관으로 쓰는 오라클 체계)를 결정론적 정답지와 무리하게 혼용할 때 생기는 최악의 안티 패턴인 자기 강박적 수용(Self-Enhancement Bias) 이다.

  • 안티 패턴: 골든 데이터셋의 정답을 생성할 때 GPT-4를 이용해 대량의 정답지를 뽑아냈다. 그리고 시스템에 묻혀질 ‘비교 심판관(Judge Model)’ 인프라 역시 GPT-4 백엔드를 사용했다.
  • 오류 발생: 새로운 모델 옵션인 Claude3나 Gemini 파이프라인으로 시스템을 변경했을 때, 테스트를 돌려보니 모두 최악의 점수를 받고 Fail 해버린다.
  • GPT-4 심판관은 ’자신(GPT) 특유의 어투’로 쓰여 있는 정답지 문자열의 뉘앙스를, 비록 의미상 똑같더라도 타사 모델(Claude)의 정제된 글쓰기 방식보다 더 선호하도록 잠재적으로 편향(Bias)되어 있기 때문이다.

이러한 편향된 정답계에 종속되면, 회사는 영구적으로 단일 벤더(Vendor Lock-in)에 갇혀 벗어날 수 없게 된다. 정답지는 어떤 LLM 생태계의 특성도 반영하지 않는 순수한 메타데이터와 결정론적 팩트 조각만으로 세절(Shredding) 되어 있어야 한다.
