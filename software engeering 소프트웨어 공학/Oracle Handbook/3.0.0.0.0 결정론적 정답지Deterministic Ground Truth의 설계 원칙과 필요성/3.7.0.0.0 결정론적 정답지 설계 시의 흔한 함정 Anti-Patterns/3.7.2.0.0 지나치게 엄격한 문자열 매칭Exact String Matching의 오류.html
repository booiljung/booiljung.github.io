3.7.2 지나치게 엄격한 문자열 매칭(Exact String Matching)의 오류

앞선 ‘포맷 불가지론(Format Agnosticism)’ 원칙에서도 경고한 바 있지만, Exact Match(완전 일치) 방식을 AI의 줄글에 적용하려는 것은 가장 순진하고도 치명적인 실수다.

  • 안티 패턴: 질문: “이 영화의 이름은?” / 정답지: Assert(Output == "스파이더맨")
  • 오류 발생: AI가 "스파이더맨 (Spider-Man)" 이라고 더 친절하게 대답하거나 "영화의 이름은 스파이더맨입니다." 라고 완벽한 정답을 내도 파이프라인은 0점 Fail 처리를 날린다.

결정론적 정답지를 활용할 때는 결코 문자열 완전 일치를 남발해선 안 된다. 이는 결함 없는 시스템을 ‘거짓 실패(False Negative)’ 늪으로 몰아넣어 개발팀이 오라클을 끄고 싶도록 만드는 최대 원흉이다. 정답지는 반드시 하드코딩된 문자열이 아니라, ‘포함 여부(Contains)’, ‘의미 매칭 임계값’, ‘정규식 파싱 후 비교’ 레이어를 거치도록 설계해야 한다.
