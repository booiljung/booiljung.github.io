3.9.2 정답지 메타데이터(Metadata)와 시계열성 타임스탬프(Temporal Stamp) 결합 방법론

“어제의 정답은 어제 시점에서는 100점이어야 한다.”
QA 팀이 지난달 발생했던 모델의 사고를 추적(Tracing)하기 위해 서버 로그를 뜯어볼 때, 현재의 5만 원 무료배송 정답지로 지난달 3만 원 시절의 시스템 상태를 들이대고 평가하는 것은 완벽한 시계열 붕괴 모순을 낳는다.

1. 차원 정답지 아키텍처 (Temporal Metadata)

이를 방어하기 위해 모든 결정론적 정답지 객체(JSON) 내부에는 Temporal Stamp(시계열 타임스탬프) 메타데이터가 영구 문신처럼 각인되어야 한다.

{
  "test_id": "SP-001",
  "prompt": "무료 배송 정책을 알려줘",
  "expected_truth": {
    "value": "50,000",
    "operator": ">="
  },
  "metadata": {
    "valid_from": "2024-05-01T00:00:00Z",
    "valid_until": "9999-12-31T23:59:59Z",  // 최신 정답지
    "replaced_version_id": "SP-000-Old"
  }
}

이 메타데이터 구조를 띠고 있다면 오라클은 평가 트랜잭션이 일어났던 그 당시의 시점 타임스탬프를 기준으로, 유효했던 유일한 정답지 버전만을 불러와 매칭한다.
이는 복잡한 엔터프라이즈 환경에서 시간성(Temporality)이라는 가변 요소를 완벽히 제어하여, 언제 어느 시점으로 돌아가는 회귀 테스트(Regression Test) 타임머신을 돌리든 시스템의 항상성(Homeostasis)을 수호하는 마법의 키워드다.
