3.2.1.1 사실 관계 오류(Factual Error)와 논리적 오류(Logical Fallacy)의 식별

AI 시스템이 범하는 오류는 크게 ’사실 관계의 결함’과 ’논리 추론의 붕괴’로 나뉜다. 이 두 가지를 스캐닝(Scanning)하기 위해서는 확률적 접근이 아닌 결정론적인 진리 테이블(Truth Table)이 필수적이다.

  1. 사실 관계 오류 (Factual Error)
  • 문제: 모델 자체가 잘못된 백과사전적 지식을 내뱉거나, 사내 데이터베이스(DB) 검색 결과(RAG)를 무시하고 변형된 숫자를 생성하는 현상.
  • 정답지의 제어: 정답지에 [매출액: 5,000만 달러, 핵심 파트너: A사]라는 팩트 키워드를 하드코딩해 두어, AI의 답변에 이 숫자가 완벽히 포함(Exact Match)되어 있는지를 강제로 비교한다.
  1. 논리적 오류 (Logical Fallacy)
  • 문제: “A는 B보다 비싸다. C는 A보다 비싸다. 따라서 B가 가장 비싸다“와 같이 어텐션(Attention)이 붕괴되어 삼단논법에 위배되는 답변을 도출하는 현상.
  • 정답지의 제어: 출력물의 ‘형태’ 자체보다는 전제와 결론 간의 모순을 잡아내기 위해, 결정론적 정답지는 논리적 도출물(Expected Deduction) 을 미리 할당해 놓고 조건문 단언(Assertion)을 통해 시스템을 파괴(Fail)시킨다.
