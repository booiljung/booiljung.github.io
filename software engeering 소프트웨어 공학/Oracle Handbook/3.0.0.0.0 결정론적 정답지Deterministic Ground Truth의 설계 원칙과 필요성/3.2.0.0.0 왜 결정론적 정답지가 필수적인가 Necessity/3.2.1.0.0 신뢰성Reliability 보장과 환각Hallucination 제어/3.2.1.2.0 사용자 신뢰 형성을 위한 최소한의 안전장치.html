3.2.1.2 사용자 신뢰 형성을 위한 최소한의 안전장치

B2C 애플리케이션에서 소프트웨어 시스템이 제공할 수 있는 최고의 미덕은 극단적으로 빠른 응답성(Latency)이나 화려한 문체가 아니라, “내가 언제 어떤 상황에 이 서비스를 사용하더라도 위험한 모험을 강요받지 않는다” 는 확신이다.

결정론적 정답지는 이 신뢰가 파괴될 수 있는 선을 긋는 세이프가드(Safeguard) 다.

  • 차단(Block)으로서의 정답지: 극단적으로 폭력적이거나, 경쟁사를 비하하거나, 민감한 개인정보를 탈취하려는 프롬프트가 주입되었을 때, AI가 유려하게 회피 기동을 할지라도 이는 위험하다. 정답지(예: 금칙어 DB나 정규화 패턴)는 AI의 응답을 가차 없이 끊어내버리고 안전한 통조림 메시지(Canned Response)로 교체한다.
  • 운영 중단 제어 (Fail-Safe): 이 강력한 정답지가 인프라의 뒷단을 지키고 있음을 사용자에게 증명(SLA 보장 등)할 수 있을 때, 기업은 비로소 법적 리스크의 공포 없이 모델 개발에 전속력을 다할 수 있다.
