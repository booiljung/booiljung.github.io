3.2.2 회귀 테스트(Regression Testing)의 기준점 확보

현대 소프트웨어 아키텍처는 매일 코드가 커밋(Commit)되고 통합되는 애자일(Agile) 환경 속에서 굴러간다.
한 버전에 존재하던 버그를 해결했다고 생각했는데 뜻박의 곳에서 어제까지 잘 되던 코드가 깨져버리는 현상, 이 끔찍한 파괴의 사이클을 막는 기술이 회귀 테스트(Regression Testing) 이다. 그리고 회귀 테스트가 존립하기 위한 절대 전제가 바로 결정론적 정답지(Deterministic Ground Truth)다.
