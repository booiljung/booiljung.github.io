3.3.1.1 복합 질문의 분해와 개별 검증 가능한 단위 설정

사용자는 AI에게 단답형으로 묻지 않는다. “아이폰 15 Pro의 가격과 무게, 그리고 A17 칩이 언제 출시되었는지 알려줘“와 같은 다중 의도(Multi-intent) 프롬프트가 기본이다.

1. 분해 불가능한 정답지의 비극

이때 정답지(Ground Truth) 란에 “가격은 120만 원, 무게는 221g, 칩은 23년에 나옴“이라는 거대한 한 문장을 통째로 집어넣어선 안 된다. 만약 모델이 무게와 칩셋은 정확히 맞췄는데 가격만 100만 원이라고 오답을 냈을 경우, 거대한 덩어리의 정답지를 든 오라클은 그냥 ‘Fail(0점)’ 처리를 해버리고 끝난다. 무엇이 맞았고 무엇이 틀렸는지 세밀한 척도 분석이 박살 난 것이다.

2. 원자적 단위(Atomic Unit)의 분해

따라서 정답지는 검증 가능한 최소의 단위 테이블로 강제 슬라이싱(Slicing) 되어 메타데이터화 되어야 한다.

  • [Assertion 1] Price == "120만원"
  • [Assertion 2] Weight == "221g"
  • [Assertion 3] Chip_Release_Year == "2023"

이렇게 쪼개진 원자적 정답지는 오라클이 스코어링(Scoring)을 할 때 “3개 중 2개 검증 통과(부분 점수 부여 가능)“라는 정밀한 진단과 국소적인 디버깅을 기능케 한다.
