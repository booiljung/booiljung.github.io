3.3.1.2 테스트 케이스의 독립성 보장

단일 테스트 케이스의 정답지는 다른 테스트 케이스의 결과값에 절대적으로 의존(Coupling)해서는 안 된다.

  • 상태 의존성 오염: 테스트 B의 정답지가 테스트 A에서 AI가 내뱉은 변수에 기반하여 스위칭되도록 구현하게 되면, 테스트 A가 우연한 환각(Hallucination)으로 실패했을 때 뒤에 멀쩡하게 대기하던 테스트 B와 C마저 연쇄적으로 붕괴해 버린다.

각 프롬프트와 정답지 세트는 언제나 백지상태(Stateless)에서 무작위 순서로 100번을 섞어서 실행하더라도 동일한 개별 Pass/Fail 결과를 도출해야 한다. CI/CD 파이프라인에서 수천 개의 테스트를 병렬 가동(Parallel Execution)하기 위한 필수 전제 조건이기 때문이다.
