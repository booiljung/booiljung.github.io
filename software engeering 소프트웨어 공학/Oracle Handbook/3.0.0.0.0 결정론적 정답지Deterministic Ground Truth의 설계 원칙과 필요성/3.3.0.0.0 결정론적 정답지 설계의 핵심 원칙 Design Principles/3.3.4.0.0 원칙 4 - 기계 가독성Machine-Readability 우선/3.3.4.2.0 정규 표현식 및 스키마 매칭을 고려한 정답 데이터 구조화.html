3.3.4.2 정규 표현식 및 스키마 매칭을 고려한 정답 데이터 구조화

보다 심화한 기계 가독성은 단순한 키워드 유무를 떠나 ’문자열의 구조적 정합성’을 강제하는 단계로 나아간다.

  1. 정규표현식(Regex) 매핑:
    AI가 이메일 주소를 추천해 줄 때, 정답지는 특정 이메일 이름이 아니라 expected_pattern: "^[a-zA-Z0-9+_.-]+@[a-zA-Z0-9.-]+$" 구조 자체를 정답지로 품고 오라클에게 룰셋을 투척한다.
  2. 스키마 매칭 (Schema Matching):
    API에 연동하기 위한 백엔드 AI 페르소나가 출력하는 JSON은 키(Key) 값의 오동작이나 누락이 시스템 크래시(Crash)를 일으킨다. 이를 방어하는 정답지는 통상 Pydantic과 같은 검증 라이브러리의 클래스 정의 형태 그 자체를 빌려 작성된다.

극단적으로 말해, 정답지는 철학이 아니라 코드(Code) 여야만 무결점의 자동화를 쟁취할 수 있다.
