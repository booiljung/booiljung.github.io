3.3.4.1 인간의 개입 없이 자동화된 스크립트로 비교 가능한 구조

CI/CD 파이프라인이 하루에도 수십 번씩 빌드되며 테스트 스위트(Test Suite)를 가동할 때, 누군가 옆에 앉아 문맥을 해석해 줄 수는 없다.

정답지 파일(Ground Truth)은 엑셀 파일이 아니라, 저장소의 .json 내지 .yaml 형태의 퓨어 데이터 에셋(Data Asset)이어야 하며 파이썬 스크립트와 같은 기계 로직이 곧바로 open() 해 읽을 수 있도록 배열화되어 있어야 한다.

  • 기계가 읽을 수 없는 정답: 환불 기한인 7일을 언급해 줄 것
  • 기계 가독성 완벽 정답:
{
  "test_id": "TC-REFUND-001",
  "expected_keywords": ["7일", "일주일"],
  "logical_operator": "OR"
}

이러한 설계가 안착되면, 어떠한 LLM 심판관 없이 re.search() 같은 기초적인 파이썬 코드만으로도 밀도 높은 자동화 오라클 비교 엔진을 하루 스크립트 1만 번 단위로 거뜬히 실행시킬 수 있게 된다.
