3.3 결정론적 정답지 설계의 핵심 원칙 (Design Principles)

현업에서 결정론적 정답지를 구축하라고 지시하면, 보통 엑셀을 열고 왼쪽에는 질문, 오른쪽에는 “아주 긴 서술형 정답“을 써 내려가는 것으로 착각하곤 한다. 이렇게 만들어진 정답지는 자동화 파이프라인에서 단 1%의 유효성도 발휘하지 못한다.

신뢰할 수 있고 기계가 파싱(Parsing)할 수 있는 오라클 인프라를 구축하기 위해, 정답지 데이터 셋(DataSet)은 반드시 현대 소프트웨어 공학의 객체지향 설계 원칙(SOLID)에 버금가는 엄격한 설계 아키텍처 원칙에 따라 튜닝되어야 한다. 본 절에서는 성공적인 결정론적 정답지 설계를 관통하는 네 가지 핵심 기둥(원자성, 포맷 불가지론, 경계 조건 명시, 기계 가독성)을 상술한다.
