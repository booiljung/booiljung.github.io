3.1.2 정답지(Ground Truth) vs. 오라클(Oracle) vs. 평가 지표(Metric)의 개념 구분

AI 테스팅 방법론을 설계하다 보면, 개발자와 QA 엔지니어 사이에서 용어의 혼선이 빈번하게 발생한다. 특히 AI 실험실의 과학자 출신들과 서버 아키텍처를 짜온 백엔드 언지니어들이 만날 때 대화가 산으로 간다. 이 세 가지 용어는 촘촘하게 맞물려 있지만 본질적인 역할과 책임(R&R)은 완벽히 분리된다.

1. 엄밀한 정의와 상호작용

  1. 정답지 (Ground Truth):
  • 정의: 비즈니스가 보장하는 ‘영원불변하는 모범 답안’. 객관적 사실이자 절대적인 기준점이다.
  • 비유: 100M 단위 측정의 척도가 되는 파리 보관소의 ‘미터원기(Standard Metre)’.
  • 예시: 데이터베이스 내 고객 정보 표(Table), 혹은 기획부서에서 직접 넘겨준 “회원 가입 불가 사유 항목” 리스트.
  1. 평가 지표 (Metric):
  • 정의: 시스템의 실행 결과물(Output)이 저기 있는 ’정답지’와 비교해서 얼마나 가깝거나 유사한지 수학적으로 재어주는 ‘척도 자(Ruler)’.
  • 비유: 미터원기와 비교하기 위해 사용되는 정밀한 ‘레이저 눈금 측정기’.
  • 예시: 코사인 유사도(Cosine Similarity) = 0.87, 정밀도(Precision) 스코어 = 95점.
  1. 테스트 오라클 (Test Oracle):
  • 정의: 평가 지표의 스코어(Metric)를 들이밀었을 때, 그래서 이 코드를 세상 밖 프로덕션(Production)으로 통과시킬지(Pass) 쓰레기통으로 던져버릴지(Fail)를 결정하고 심판을 내리는 ‘알고리즘적 재판관’.
  • 비유: 레이저 눈금 측정 결과를 보고 “공차 0.05mm를 초과했으니 불합격시켜!“라고 도장을 찍는 불량 검수원.
  • 예시: if (Cosine_Similarity > 0.90) return Pass; else return Fail;

이 세 가지 기어(Gear)가 순서에 맞게 구동되어야 한다. 그중에서도 정답지(Ground Truth) 가 썩어버렸거나 기준을 잡지 못하면, 그 뒤의 지표가 아무리 정교하고 오라클의 임계값(Threshold) 판정이 무자비하다 한들 전체 QA 시스템은 폐기 처분된 파이프라인에 불과하다.
