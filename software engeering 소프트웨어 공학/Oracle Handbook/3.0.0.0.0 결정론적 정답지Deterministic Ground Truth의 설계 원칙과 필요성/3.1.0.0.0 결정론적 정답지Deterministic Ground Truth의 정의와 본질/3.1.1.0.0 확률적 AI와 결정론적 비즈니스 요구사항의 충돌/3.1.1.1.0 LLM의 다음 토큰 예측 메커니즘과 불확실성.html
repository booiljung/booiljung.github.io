3.1.1.1 LLM의 ‘다음 토큰 예측(Next Token Prediction)’ 메커니즘과 불확실성

AI의 본질적인 구조적 한계는 태생부터가 불확실성(Uncertainty)을 연료 삼아 움직인다는 데 있다. 최근 열풍을 몰고 온 대규모 언어 모델(LLM)들은 그 속내를 들여다보면 단지 어마어마한 크기의 통계적 추론 엔진에 불과하다.

1. 토큰 샘플링의 나비효과

언어 모델은 문장 전체를 한 번에 이해하고 내뱉는 것이 아니다. 오직 지금까지 생성된 문맥을 보고 “그다음 올 단어(Next Token)로 어떤 것이 가장 확률적으로 적절한가?” 만을 계산하여 하나씩 이어 붙인다(Autoregressive Generation).

graph LR
    Input["Prompt: 나는 밥을..."] --> Model{LLM (Token Predict Matrix)}
    
    Model -- 80% 확률 --> Token_A[먹었다]
    Model -- 15% 확률 --> Token_B[지었다]
    Model -- 5% 확률 --> Token_C[남겼다]
    
    Token_A --> Temperature{Temperature 파라미터 개입}
    Token_B --> Temperature
    Token_C --> Temperature
    
    Temperature --> |"Random Sampling"| Output["선택된 토큰은 매번 다를 수 있음"]

위의 머메이드 구조도처럼, 온도(Temperature)나 탑-피(Top-P) 같은 튜닝 파라미터가 개입되면서 모델은 가장 정답에 가까운 ’먹었다’를 고르지 않고, 가끔 창의성을 발휘해 ’남겼다’를 토큰으로 던져버린다.
이처럼 기반 구조 자체가 ’주사위 던지기(Stochastic Sampling)’로 이루어져 있기 때문에, AI에게 “2+2는 무엇인가?“와 같은 완벽한 팩트를 물어도, 통계적 운의 장난으로 “5“라는 다음 토큰이 튀어나올 가능성을 완벽히 0%로 만들 수단이 존재하지 않는다. 이것이 LLM에 내재된 영구적인 불확실성의 근본 원인이다.
