3.6.3.2 가중치 기반의 정답 유사도 평가 모델

부분 점수를 계산할 때, 단순히 맞춘 개수 / 전체 정답수 로 기계적 나눗셈을 하는 것은 비즈니스의 경중을 무시하는 행위다.

1. 정답 항목(Assertion) 간의 가중치 할당

  • 상품 이름 일치 (weight: 10)
  • 배송일 일치 (weight: 8)
  • 상품 가격 일치 (weight: 100 - 치명적 요소)
  • 인사말 포함 (weight: 1)

이런 식별 라벨을 정답 딕셔너리에 매핑해 놓고 가중 평균 합(Weighted Sum)을 계산한다.
AI가 다른 건 다 맞았어도 가격 한 가지를 틀렸다면 환각 스코어가 폭락하여 Fail에 도달하고, 인사말 스코어를 틀렸을 경우에는 99/100 점을 획득하여 배포 파이프라인(Pass)을 통과한다. 모호성을 수치화(Indexation) 함으로써 기계적 타당성을 획득하는 순간이다.
