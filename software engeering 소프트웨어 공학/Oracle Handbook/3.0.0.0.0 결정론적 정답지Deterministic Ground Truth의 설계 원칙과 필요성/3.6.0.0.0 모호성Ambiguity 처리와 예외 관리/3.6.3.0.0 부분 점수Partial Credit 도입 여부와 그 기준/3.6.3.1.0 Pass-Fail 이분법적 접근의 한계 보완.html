3.6.3.1 Pass/Fail 이분법적 접근의 한계 보완

Boolean(0 또는 1) 기반의 절대적 판정 방식은 단순 명료하지만, 개발팀의 사기를 꺾고 오버피팅(Overfitting)을 유도하는 단점이 있다.

개발팀은 0점을 피하기 위해 일반화된 문장 대신 오직 오라클의 하드코딩된 특정 단어에만 집착하도록 프롬프트를 기형적으로 비틀게 된다. (Prompt Overfitting). 이를 타개하기 위해 정답지에 ’점수제(Score)’의 개념이 도입된다.

1. 부분 점수 체계 단위 리스트

  • 0점 (Critical Fail): 금칙어 사용, 보안 위반, 환각적 사실의 서술 등 결정적 타격.
  • 0.5점 (Partial Pass): 요구한 포맷(JSON 등)은 지켰으나, 부가적인 설명(예: 고객 응대 문구)이 정답지 레이블과 누락된 경우.
  • 1점 (Perfect Pass): 문장 형식과 핵심 키워드가 완벽하게 오라클 스키마를 100% 만족하는 경우.

이 부분 점수 시스템은 모델이 극단적인 오류만 회피한다면(0점 방어), 부드러운 전진을 할 수 있도록 개발팀에 여유를 제공한다.
