3.6.2.1 톤앤매너(Tone & Manner) 가이드라인의 정량화 시도

“친근하게 대답해 달라“는 톤앤매너(Tone & Manner) 요구는 오라클이 스니핑(Sniffing)할 수 없는 무형의 감각이다. 이를 결정론적 오라클 시스템에 올리기 위해서는 가이드라인을 강제로 정량화(Quantification)하는 메타데이터 정답지가 구축되어야 한다.

  • 정답지 변환 전: “응답이 너무 딱딱하면 Fail 시킨다.” (사용 불가)
  • 정답지 변환 후 (규칙 기반화):
  1. Assertion 1: 문장의 마무리가 다/나/까의 격식체가 아니라 해요/죠/네요 등의 종결어미 배열(Friendly_Suffix_List)로 끝날 것.
  2. Assertion 2: 텍스트 전체의 Gunning Fog Index(가독성 난이도 지수) 등 수식 연산 결과가 8.0 이하일 것.
  3. Assertion 3: 이메일 본문 길이(Length())가 500바이트를 넘지 않을 것.

창의성은 풀어주되, 이 세 개의 단단한 제약(Constraint) 벽을 하나라도 건드리면 즉각 결함 처리함으로써, 무한한 창의가 허용된 좁은 울타리를 건설하는 것이 핵심이다.
