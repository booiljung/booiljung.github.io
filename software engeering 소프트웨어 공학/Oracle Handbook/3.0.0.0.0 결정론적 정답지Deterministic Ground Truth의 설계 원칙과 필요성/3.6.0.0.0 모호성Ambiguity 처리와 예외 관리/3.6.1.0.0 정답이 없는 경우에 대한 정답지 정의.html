3.6.1 정답이 없는 경우에 대한 정답지 정의

대부분의 주니어 개발자들은 “태양은 어떤 색인가?“라는 질문에 ["노란색", "흰색"]이라는 정답 쌍을 빚어내는 데 혈안이 되어 있다. 하지만 엔터프라이즈 환경에서 더 중요한 것은 “당신 회사의 주가가 내일 오를까요?“라는 질문에 대한 답변이다.

이 질문은 본질적으로 정답이 존재하지 않는다(Unanswerable Query).

1. 모름(Unknown)의 결정론화

정답이 없는 것에 대해 모델이 억지로 꾸며낸(Hallucinated) 대답은 치명적인 법적 리스크를 낳는다. 따라서 오라클의 정답지 데이터셋에는 의도적으로 Unanswerable 태그가 붙은 프롬프트 세트를 대량으로 주입해야 한다.

  • 기대 출력 (Expected Output): “제가 답변할 수 없는 영역입니다.” (Polite Refusal)
  • 오라클의 판정 로직: 모델이 해당 질문에 대해 특정한 사실(Fact)이나 숫자, 구체적 주장을 단 한 번이라도 내뱉는 순간을 파싱하여 가차 없이 Fail 처리한다. 모르는 것은 모른다고 대답하는 안전성(Safety) 자체가 훌륭한 ’정답’으로 박제되는 것이다.
