3.5.2.1 라벨러 간 일치도(Inter-Annotator Agreement) 관리

1,000개의 프롬프트-응답 세트를 10명의 라벨러에게 던져주고 “이 결정론적 정답의 참 거짓 여부를 기입하시오” 라고 지시했을 때, 사람마다 생각이 다르면 그 정답지는 파이프라인의 기준으로 기능할 자격을 즉시 상실한다. 정답지는 모호해서는 안 되기 때문이다.

1. 합의의 수학적 증명 (IAA)

이때 QA 공학이 요구하는 도구는 라벨러 간 일치도(IAA, Inter-Annotator Agreement) 이다.
대표적으로 코헨의 카파(Cohen’s Kappa) 나 플라이스의 카파(Fleiss’ Kappa) 같은 수학적 계수를 사용한다.

  • 만약 3명의 작업자가 똑같은 프롬프트 출력을 보고 한 명은 Pass, 두 명은 Fail이라고 판정한다면 카파 점수는 0점에 수렴한다.
  • 이는 라벨러가 멍청해서가 아니라, 우리가 부여한 ’정답지 구축 기준 명세서’의 언어가 너무나 모호하거나 엣지 케이스 통제력이 부족하다는 시스템의 적신호다.
  • 통상적으로 카파 계수가 0.8 이상(강한 일치)을 획득하지 못한 정답지 슬롯은 전부 폐기되거나 혹은 팀 토론을 거친 후 완벽한 만장일치를 얻어낼 때까지 재조정되어야 한다.
