3.5.1.1 실제 운영 로그(Production Log) 기반의 데이터 추출

가장 훌륭한 초기 테스트 정답지의 원천은 이미 살아 움직이고 있는 레거시(Legacy) 시스템의 운영 로그(Production Log)다.

1. 로그 파밍(Log Farming) 기법

  • 기존 시스템에서 고객이 실제로 묻고 상담원이 답변했던 1년 치 콜센터 텍스트 로그 혹은 검색 질의어 로그 트랜잭션을 추출한다.
  • 이 원시 데이터들을 클렌징(Cleaning)하여 오타, 불완전 문장, 문맥이 잘린 질문들을 있는 그대로 유지한 채 [프롬프트 원본]으로 확보한다. 깨끗한 척하는 테스트 세트보다, 이렇게 더러운(Noisy) 데이터셋이 AI의 견고성(Robustness)을 테스트하기에 백 배는 더 훌륭하다.
  • 과거 상담원이나 시스템이 이 더러운 질문에 대해 내놓았던 결괏값 중, 평점 5점을 받았거나 Success 코드를 가진 트랜잭션만을 [초기 정답지 세트]로 간주하여 결합한다.
