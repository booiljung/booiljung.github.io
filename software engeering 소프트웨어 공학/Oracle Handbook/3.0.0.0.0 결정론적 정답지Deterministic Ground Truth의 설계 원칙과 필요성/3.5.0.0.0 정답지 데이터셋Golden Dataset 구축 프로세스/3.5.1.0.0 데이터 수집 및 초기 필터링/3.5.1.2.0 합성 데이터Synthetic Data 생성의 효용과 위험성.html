3.5.1.2 합성 데이터(Synthetic Data) 생성의 효용과 위험성

초기 배포라서 수집된 운영 로그가 전무(Empty)하거나, 극비 보안이 요구되는 영역의 경우 개발팀은 대규모 언어 모델(LLM)을 이용해 가짜 공격 데이터와 가짜 정답지를 수만 개 찍어내는 합성 데이터(Synthetic Data) 생성 기법을 흔히 채용한다. (“GPT-4를 이용해 GPT-3.5를 테스트할 정답지 1만 개를 생성하라”)

1. 양날의 검, 합성 데이터

효용 (Pros):
사람이 하루 종일 타이핑해야 만들 수 있는 엣지 케이스(Edge Case) 정답지를 불과 10분 만에 만 단위로 뻥튀기(Augmentation)할 수 있다. 이는 극한의 비용 절감을 가져온다.

위험성 (Cons):
합성 데이터를 만들어낸 마더 LLM 본연의 환각(Hallucination)과 편향이 정답지에 고스란히 묻어 들어간다.

  • LLM은 인간 사용자만큼 멍청하거나 창의적이지 않다. 그들이 생성한 질문들은 보통 판에 박힌 듯 정형화된다.
  • 만약 이 합성 데이터 더미를 교차 검증하지 않고 파이프라인의 오라클 진리로 섬기게 되면, 결국 AI가 AI의 실수를 묵인하는 악순환(Model Collapse)의 나락에 빠지게 된다. 따라서 합성 데이터는 반드시 통계적 필터링을 거쳐 독소 조항을 걸러내야만 정답지로 승격될 수 있다.
