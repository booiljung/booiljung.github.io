3.5 정답지 데이터셋(Golden Dataset) 구축 프로세스

어떤 완벽한 정규표현식(Regex)과 임베딩 모델(Embedding Model)을 오라클 인프라로 채택하더라도, 결국 그 오라클이 스캔해야 할 대조군은 ‘인간이 만들어 둔’ 데이터의 묶음, 즉 골든 데이터셋(Golden Dataset) 이다. 이 데이터셋의 품질이 소프트웨어 AI의 품질(Quality)을 직간접적으로 100% 결정한다.

“모든 것을 AI가 검증하게 하라“는 허상에서 벗어나, 이 골든 데이터셋만큼은 극도로 보수적인 공학 프로세스로 구축하고 관리해야 한다. 본 절에서는 원시 데이터를 수집하는 과정부터, 사람의 손을 거쳐 라벨링(Labeling)되고, 마지막으로 Git 파이프라인에 등재되어 생명주기(Lifecycle)를 관리받는 전이 과정을 상세하게 해부한다.
