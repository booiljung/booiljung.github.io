3.5.3.2 Git 기반의 데이터셋 형상 관리 전략(DVC 등 활용)

코드가 Git 브랜치(Branch)를 타듯이 데이터셋도 버전으로 분기되어야 한다. “작년 12월 크리스마스 프로모션 당시의 시스템이 맞게 동작했었나?“를 회귀 테스트(Regression Test)하려면, 반드시 코드 버전뿐 아니라 그 당시의 정답지 데이터 셋도 동일한 스냅샷(Snapshot)으로 불러와야 하기 때문이다.

1. DVC (Data Version Control) 연동

용량이 큰 JSON 포맷의 정답지 묶음을 .git 파일 자체에 꾸역꾸역 집어넣는 것은 멍청한 짓이다.
대신 프로덕션 레벨에서는 DVC (Data Version Control) 구조를 활용한다. S3 버킷 등에 실제 데이터셋 바이너리를 보관하고, 이 S3 파일의 Hash MD5 주소값만을 가벼운 텍스트로 Git에 올려 코드 버전과 데이터 버전을 톱니바퀴처럼 1:1로 매핑시키는 것이다.

이 아키텍처 위에서는 누군가 정답지의 토큰 하나를 마음대로 실수령으로 변경하더라도, Git 해시가 어긋나는 순간 오라클 파이프라인이 즉각 다운되므로 완벽한 데이터 위변조 거버넌스가 확립된다.
