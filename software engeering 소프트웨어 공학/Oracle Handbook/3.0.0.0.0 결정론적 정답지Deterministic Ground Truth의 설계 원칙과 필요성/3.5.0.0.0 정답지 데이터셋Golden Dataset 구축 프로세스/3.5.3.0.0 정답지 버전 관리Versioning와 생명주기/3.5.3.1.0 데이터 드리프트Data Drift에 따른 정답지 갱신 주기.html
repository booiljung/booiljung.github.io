3.5.3.1 데이터 드리프트(Data Drift)에 따른 정답지 갱신 주기

데이터 드리프트(Data Drift) 란 현실 세계의 사실관계와 사용자 언어 패턴이 서서히 변해가면서, 기존에 학습된 AI 모델과 현재 보유한 결정론적 정답지가 실제 비즈니스 흐름과 동떨어지게 되는 현상을 의미한다.

1. 유통기한(Expiration Date) 설정 체계

이러한 부패 현상을 막기 위해, 골든 데이터셋 안의 개별 정답지 트랜잭션에는 반드시 expires_at (만료 기한) 속성을 강제 할당해야 한다.

  • 예: 금융 상품 금리와 관련된 정답지 세트는 만료 기한을 배포 후 30일로 설정한다.
  • CI/CD 파이프라인에서 테스트가 실행될 때, 이 만료 기한이 지나버린 정답지는 오라클 비교 엔진에서 자동 배제되고 “정답지 노후화 경고(Stale Ground Truth Warning)” 알람을 데이터 파이프라인 팀에 발송한다.

주기적으로 낡은 정답지를 버리고 새로운 운영 로그 기반의 프롬프트-응답 쌍을 채워 넣는 갱신(Renewal) 작업만이 오라클의 영속성을 보장한다.
