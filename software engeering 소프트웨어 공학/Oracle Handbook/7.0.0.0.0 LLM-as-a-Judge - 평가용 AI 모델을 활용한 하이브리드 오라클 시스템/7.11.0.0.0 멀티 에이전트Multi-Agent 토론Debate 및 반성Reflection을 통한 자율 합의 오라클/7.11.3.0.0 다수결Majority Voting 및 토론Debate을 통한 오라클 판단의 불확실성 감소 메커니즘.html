7.11.3 다수결(Majority Voting) 및 토론(Debate)을 통한 오라클 판단의 불확실성 감소 메커니즘

서로 다른 모델 3~4개를 심사위원으로 앉혀 놓으면 의견이 엇갈리기 마련이다. 단순히 다수결(Majority Voting)로 가장 많이 나온 숫자를 고르는 방법도 있지만, 더 영리한 토론(Debate) 프레임워크가 존재한다.

  • 방법론: 각 모델(예: Claude, GPT, Gemini)이 1차 점수안과 근거(Reasoning)를 낸다.
  • 이 3개의 보고서를 다시 수합하여 각 참가자에게 배포하고, “타인의 채점평을 읽어본 뒤, 논리 오류를 수긍한다면 네 점수를 수정하라“고 지시한다.
    보통 2번의 라운드 루프를 돌면 모델들은 서로의 맹점을 깨우치며 하나의 진리점(Consensus)으로 스코어를 수렴시킨다. 인간 재판부의 합의 과정과 소름 돋게 일치하며 높은 재현성(Reproducibility)을 담보하는 최고급 오라클 기법이다.
