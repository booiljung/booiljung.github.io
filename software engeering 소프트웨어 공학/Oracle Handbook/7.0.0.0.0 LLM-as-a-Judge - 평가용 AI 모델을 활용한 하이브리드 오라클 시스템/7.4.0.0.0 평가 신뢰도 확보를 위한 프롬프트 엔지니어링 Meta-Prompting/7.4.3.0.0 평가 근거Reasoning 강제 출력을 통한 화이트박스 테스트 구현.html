7.4.3 평가 근거(Reasoning) 강제 출력을 통한 화이트박스 테스트 구현

심판이 점수 달랑 하나만 반환하는 블랙박스(Black-box) 오라클은 엔터프라이즈 환경에서 쓸 수 없다. 왜 5점이 아니라 4점인지에 대한 ’채점평’이 있어야만 디버깅과 메타-평가(Meta-Evaluation)가 가능하다.

{
  "reasoning": "사용자의 질문은 '리눅스 크론탭 설정법'이나, 답변은 '윈도우 작업 스케줄러'를 설명하고 있으므로 관련성(Relevance) 항목에서 치명적으로 위배됨.",
  "score": 1
}

JSON Schema 강제 출력을 통해 reasoning 필드를 score보다 반드시 먼저 오도록 배열하라. LLM은 자기가 먼저 내뱉은 문자들을 기반으로 논리를 이어나가기 때문에, 텍스트로 합당한 비판을 쓰게 강제하면 후속된 점수 추론이 훨씬 날카로워진다.
