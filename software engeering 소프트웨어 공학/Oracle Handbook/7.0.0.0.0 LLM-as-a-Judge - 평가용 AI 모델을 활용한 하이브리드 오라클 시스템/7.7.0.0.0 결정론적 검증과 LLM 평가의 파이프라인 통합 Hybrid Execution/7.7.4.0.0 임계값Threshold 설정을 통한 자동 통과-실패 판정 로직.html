7.7.4 임계값(Threshold) 설정을 통한 자동 통과/실패 판정 로직

LLM 심판이 매긴 점수 구조체( JSON {"score": 4.5})를 받았을 때, 이를 CI/CD 도구가 인식할 수 있는 Exit Code(0 또는 1)로 치환하는 문지기 로직이다.

예를 들어 번역 파이프라인의 임계값을 아래와 같이 설정한다.

  • Score >= 4.0: sys.exit(0) -> GitHub Actions 초록불, 자동 PR 병합 허용.
  • 3.0 <= Score < 4.0: Target Warning -> 경고 플래그만 달고 보류.
  • Score < 3.0: sys.exit(1) -> 빌드 즉각 중단, 개발자에게 슬랙(Slack) 알림 발송.
    임계값의 미세 조정(Tuning)은 오라클 시스템을 너무 뻑뻑하게(모든 빌드 실패) 만들지도, 너무 물렁하게(오탐 허용) 만들지도 않는 운영 파이프라인의 생명선이다.
