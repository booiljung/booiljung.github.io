7.5.5 무작위 셔플링(Shuffling)과 블라인드 테스트 기법 적용

LLM 심판은 평가 선택지의 인덱스 번호([1], [2], [A], [B]) 자체에도 반응할 수 있다. A번이 정답일 확률이 높다는 데이터셋의 우연적 편향을 학습했을 수 있기 때문이다.

  • 방어 수칙: 평가 항목 매개변수나, 객관식 문항의 순서를 난수(Random ID)에 기반하여 100% 셔플링(Shuffling)한 뒤 프롬프트에 주입하라. 심판에게 반환받을 답변 형태를 A, B가 아니라 UUID 문자열값 자체가 되게 하면 모든 기계적 편향의 뿌리를 뽑을 수 있다.
