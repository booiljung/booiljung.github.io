7.8.5 토큰 사용량 절감을 위한 평가 프롬프트 압축 기법

LLM-as-a-Judge에 사용되는 루브릭(평가 기준표)은 길면 수천 자에 달한다. 매 요청마다 이 수천 자가 시스템 프롬프트(토큰)로 소비되며 과금된다.

  • 프롬프트 캐싱(Prompt Caching): 서빙되는 LLM 엔진이나 API 플랫폼(예: Anthropic Claude, OpenAI Cache)에서 제공하는 기능으로, 변하지 않는 방대한 평가 지침(System Prompt)을 노드 메모리에 고정시켜 토큰 과금을 면제받는 기법이다.
  • 언어 압축(LLMLingua 등): 평가 프롬프트 중 문법 조사나 관사 등 심판 모델이 의미를 파악하는 데 불필요한 단어를 자체 압축기로 가지치기(Pruning)하여 길이를 절반으로 깎아 던지는 하드코어 최적화 기법도 존재한다.
