7.8.3 평가 결과의 캐싱(Caching) 및 재사용 전략

AI가 AI를 검증하는 작업은 근본적으로 엄청난 중복 연산(Redundancy)을 내포한다. 모델 V1과 모델 V2의 A/B 테스트를 진행할 때, 생성물이 글자 하나 틀리지 않고 똑같은 답변을 내놓았다면 어떨까?

  • 레디스(Redis) 같은 인메모리 스토리지에 [Context의 해시값 + 응답 텍스트 해시값 = 이전 오라클 평가 점수] 형태로 룩업 테이블(Look-up Table)을 방어막으로 하나 둘러친다.
  • 테스트 워커(Worker)가 동일한 텍스트에 직면하면 LLM API 호출을 스킵(Cache Hit)하고 저장된 점수를 리턴한다. 회귀 테스트 러닝 시간을 획기적으로 줄여준다.
