7.8.4 배치 처리(Batch Processing)를 통한 평가 API 호출 최적화

API 요금표를 파훼하는 고도의 엔지니어링 팁이다. 100개의 리뷰를 100번 호출하여 점수를 물어보는 것보다, 한 번의 거대한 프롬프트에 10개의 리뷰를 배열로 묶어 밀어 넣는 것(Batching)이 HTTP 네트워크 오버헤드와 지연 시간 측면에서 압도적으로 유리하다.

  • OpenAI의 Batch API 등을 적극 사용하여 “24시간 내 채점만 끝나면 되는” 성격의 오프라인 벤치마킹 작업은 50%의 토큰 할인율을 적용받도록 한다.
  • 다만 프롬프트가 과도하게 길어지면 심판관 LLM의 어텐션 상실(Lost in the middle)로 인해 평가의 정확도가 급락할 수 있으므로, 배열 청크(Chunk Size)를 적절히 10~20건 단위로 컨트롤해야 한다.
