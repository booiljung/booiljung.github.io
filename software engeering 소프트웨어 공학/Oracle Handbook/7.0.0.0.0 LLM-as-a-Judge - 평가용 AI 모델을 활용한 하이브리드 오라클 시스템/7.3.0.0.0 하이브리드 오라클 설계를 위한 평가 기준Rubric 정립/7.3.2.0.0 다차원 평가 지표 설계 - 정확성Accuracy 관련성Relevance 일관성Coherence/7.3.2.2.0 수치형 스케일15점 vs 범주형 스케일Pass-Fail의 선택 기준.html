7.3.2.2 수치형 스케일(1~5점) vs 범주형 스케일(Pass/Fail)의 선택 기준

LLM에게 오라클의 임무를 맡길 때, 결과값의 데이터 타입을 어떻게 정의할 것인가는 파이프라인의 성격을 좌우한다.

  • 범주형 스케일 (Pass/Fail, Boolean):
  • 목적: 자동화된 CI/CD 배포 파이프라인의 수문장(Gatekeeper).
  • 특징: “할루시네이션이 1문장이라도 있는가?“처럼 임계값이 명확할 때 사용한다. 인간의 해석 개입 여지를 차단한다.
  • 수치형 스케일 (1~5점, 0~100점):
  • 목적: 모델 벤치마킹 장표(Leaderboard) 작성 및 A/B 테스트의 우위 판별.
  • 특징: 4점과 5점 사이의 모호함을 허용하기 때문에, 점수 자체가 파이프라인을 멈추기보다는 리포팅(Reporting) 목적으로 사용된다. 자동화 오라클에서는 수치형 평가를 1차로 받은 뒤 Score >= 4를 Pass로 강제 형변환(Casting)하는 랙(Lag) 구조를 취해야 한다.
