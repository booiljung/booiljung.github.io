7.2.2 동일 모델 간의 교차 검증(Cross-Validation)과 자기 비평(Self-Correction)

특수한 사내 보안망 등 강력한 외부 API를 쓸 수 없는 에어갭(Air-gapped) 환경에서는 수행 모델과 심판 모델이 완전히 일치하는 상황이 발생한다(예: Llama 70B가 생성하고, Llama 70B가 평가).

이 경우 발생할 확증 편향을 예방하기 위해 ’생성용 파이프라인’과 ’비평용 파이프라인’의 온도를 조절한다.

  • 생성 시점: Temperature 0.7, 창의적이고 다양한 각도의 답변 도출.
  • 평가 시점: Temperature 0.0, 역할 부여(Role-playing) 프롬프트에 엄격한 감사자(Auditor) 페르소나를 극도로 강제.
    동일 모델이라도 페르소나 컨텍스트(Context)를 분리함으로써 제한적이나마 자가 수정(Self-correction) 능력을 이끌어낼 수 있다.
