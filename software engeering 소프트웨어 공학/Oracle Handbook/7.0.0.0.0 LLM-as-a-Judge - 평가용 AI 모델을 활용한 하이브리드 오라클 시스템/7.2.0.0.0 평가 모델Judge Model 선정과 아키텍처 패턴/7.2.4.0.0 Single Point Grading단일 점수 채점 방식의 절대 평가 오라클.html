7.2.4 Single Point Grading(단일 점수 채점) 방식의 절대 평가 오라클

결과물 단 하나만을 보고 점수나 Pass/Fail을 도출하는 평가 오라클 시스템이다. 주로 “이 생성 문서가 우리 회사의 가이드라인 체계를 100% 준수하고 있는가?“와 같이 명확한 루브릭(Rubric)이 존재하는 상황에 쓰인다.

단점은 모델이 중간 점수(예: 3점)를 남발하는 평가적 편향(Central Tendency Bias)을 갖는다는 것이다. 이를 완화하기 위해 평가 스케일에 소수점 단위를 허용하거나, 아예 “1점, 3점, 5점“처럼 간격을 극단적으로 넓혀 명확한 근거 없이는 5점을 주지 못하도록 제동을 건다. 성능은 조금 떨어지지만 데이터 파이프라인(배치 처리) 구조가 가장 단순해진다는 강력한 장점이 있다.
