7.2.6 도메인 특화 소형 언어 모델(sLLM)을 활용한 평가 전용 미세조정(Fine-tuning)

거대 상용 모델(LLM)을 심판으로 매일 수십만 건의 회귀 테스트 루프를 돌리는 것은 클라우드 뱅크럽트(Cloud Bankrupt)를 초래한다.

비용 효율적 평가 아키텍처의 종착지는 ‘평가 전용 sLLM’ 구축이다.

  1. GPT-4o를 이용해 “정확한 평가 근거와 1~5점의 점수“가 포함된 고품질(QA) 데이터셋 1만 건을 생성한다 (지식 증류, Knowledge Distillation).
  2. 이 데이터를 Llama-3 8B나 Mistral 같은 소형 모델에 LoRA 방식으로 미세조정(Fine-tuning)한다.
  3. 이렇게 탄생한 sLLM은 글짓기 능력은 상실했으나, 오직 해당 도메인 문서에 한정해서는 GPT-4o와 95% 이상 일치하는 눈썰미(Judging Capability)를 지니게 된다. 이 모델을 로컬 런타임에 올리면 사실상 오라클 운영 비용이 0에 수렴한다.
