7.6.4 오라클의 오탐(False Positive)과 미탐(False Negative) 모니터링

AI 심판관의 판단 오류는 두 가지 방향으로 발생하며 파급 효과(Impact)가 다르다.

  • 미탐 (False Negative): 심각한 환각이 포함된 모델의 텍스트를 “완벽하다“고 5점을 주고 배포 파이프라인을 통과시켜 버린 경우. (가장 치명적임)
  • 오탐 (False Positive): 잘 대답한 모델의 출력을 알 수 없는 이유로 감점하여 CI/CD 파이프라인 빌드를 멈춰 세운 경우. (가용성 감소)
    심판관의 루브릭 설계는 False Negative를 0으로 맞추는 데 극단적으로 치우쳐야 한다(Recall 지향). 즉, 오라클은 의심스럽다면 일단 Fail을 외쳐 시스템을 멈추고 인간 관리자를 호출하도록 극도의 보수성을 지녀야 한다.
