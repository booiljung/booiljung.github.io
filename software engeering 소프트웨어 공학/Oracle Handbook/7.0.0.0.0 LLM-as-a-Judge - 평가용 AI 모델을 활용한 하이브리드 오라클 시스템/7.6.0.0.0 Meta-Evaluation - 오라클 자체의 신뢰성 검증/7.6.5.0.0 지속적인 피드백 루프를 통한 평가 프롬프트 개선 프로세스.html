7.6.5 지속적인 피드백 루프를 통한 평가 프롬프트 개선 프로세스

단 한 번 작성한 오라클 프롬프트가 영원히 동작할 수는 없다. 사용자 인풋 데이터 분포(Data Drift)가 바뀌면 평가 잣대도 진화해야 한다.

  1. 로그 기반 랜덤 샘플링: 프로덕션에서 평점이 매겨진 대량의 로그 중 수치 편차가 심하거나(예: 다른 모델과 일치도가 낮음), 점수 분포가 갑자기 쏠린 케이스 10선을 매주 샘플링한다.
  2. 에러 원인 분석: 오라클 모델이 reasoning 필드에 적어놓은 채점 평을 인간이 읽으며 “프롬프트 내에 어떤 엣지 케이스를 놓쳤는지” 분석한다.
  3. 루브릭 업데이트: 평가 프롬프트에 [예외 예시]를 추가하고 버전을 업그레이드하여 모델 심판의 기준점을 교정(Calibration)한다.
