7.1.2 확률적 모델을 검증 도구로 사용할 때의 역설과 해결 방안

“AI의 결과를 AI로 검증해도 되는가? 심판(Judge)마저 환각(Hallucination)에 빠지면 어떻게 하는가?”
이것은 LLM-as-a-Judge를 도입할 때 마주하는 가장 근본적인 역설(Paradox)이다.

이 역설을 통제하는 엔지니어링적 접근법은 다음과 같다.

  1. 역할의 비대칭성: 문서를 요약하거나 소설을 쓰는 모델(Temperature > 0.7)과 달리, 심판 역할을 하는 모델은 철저하게 낮은 온도(Temperature = 0)와 고정된 시드(Seed) 값으로 묶여야 한다.
  2. 구조화된 의사결정: 심판관은 자유로운 사설을 늘어놓아서는 안 된다. 직전 장(Chapter 6)에서 학습한 JSON Schema 강제 출력 기능을 접목하여 “평가점수 1~5점” 사이의 정수(Integer)만을 뱉어내도록 구조적으로 제압해야 한다.
