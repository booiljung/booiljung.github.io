10.7.2 모델 변경(Model Shift) 시 오라클 기준 재조정을 위한 베이스라인(Baseline) 재설정

모델 아키텍처나 파운데이션 모델 공급자(OpenAI -> Anthropic)가 바뀔 때, 기존의 평가 베이스라인 점수 그래프는 사실상 무의미해진다.

  • 쉐도우 테스팅(Shadow Testing): 신규 모델을 프로덕션망에서 복제(Mirrored)된 트래픽으로 1주일간 가동한다.
  • 구형 모델과 신형 모델의 응답에 대해 기존 오라클을 병렬(Parallel)로 실행한다. 이 1주일 간의 데이터가 새로운 “Baseline Score“로 확정된다. 기존에 95점이 나오던 테스트셋이 신형 모델에서 88점이 나온다면, 88점을 새로운 0점으로 세팅하고 거기서부터 테스트 후퇴 여부를 모니터링해야 오라클이 늑대 소년(Crying Wolf)이 되는 것을 막을 수 있다.
