10.7.3 레거시 프롬프트와 신규 모델 간의 호환성 검증을 위한 데이터셋 마이그레이션

구현 방식(Implementation) 변경 없이 언어 모델 버전을 올렸더니, 구형 프롬프트를 신형 모델이 완전히 잘못 이해하는 “프롬프트 호환성 파괴” 현상이 발생한다.

  • 구 모델(프롬프트를 대충 써도 알아듣는 경향)과 신형 모델(가이드라인에 극도로 예민하게 반응하는 경향) 간의 성격 차이가 원인이다.
  • 골든 데이터셋을 이용한 마이그레이션 전략: 기존 데이터셋을 “LLM 교정기(Prompt Optimizer)“에 장착한다. 최적화기(Optimizer)는 내부 루프를 돌며 신규 모델이 기존 골든 정답을 완벽하게 맞출 때까지 수천 번 프롬프트를 미세 조정(Tweaking)해 새로운 버전의 시스템 프롬프트를 자동으로 찾아내는 일(Auto-Prompting)을 수행한다.
