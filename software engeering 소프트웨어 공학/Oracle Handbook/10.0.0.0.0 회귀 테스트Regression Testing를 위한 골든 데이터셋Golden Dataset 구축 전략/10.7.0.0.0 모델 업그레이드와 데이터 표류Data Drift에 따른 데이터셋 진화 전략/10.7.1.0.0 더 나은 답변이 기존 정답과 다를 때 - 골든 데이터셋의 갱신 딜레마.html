10.7.1 ’더 나은 답변’이 ’기존 정답’과 다를 때: 골든 데이터셋의 갱신 딜레마

GPT-3.5에서 GPT-4o로 엔진을 통째로 교체했다고 가정하자. 테스트를 돌렸더니 오라클이 수만 개의 Error를 뿜어낸다. 모델이 바보가 된 걸까?

  • 정답의 상향 평준화: 아니다. 오히려 새 모델의 답변이 더 유창하고 친절(더 나은 답변)해졌기 때문에, 구형 봇의 퉁명스러운 단답형을 ’정답’으로 박아둔 기존 골든 데이터와 충돌(Mismatch)한 것이다.
  • 해결 방안: 이 딜레마를 해결하기 위해, 메이저 버전 교체 시에는 기존 오라클의 similarity_threshold를 강제로 대폭 낮추고, 실패한 케이스들을 A/B 뷰(기존 정답 vs 새 답변) 데스크톱 툴에 띄워 인간 검증자가 “이 실패는 퇴행(Regression)이 아니라 진화(Evolution)다“라고 확인한 후 새로운 답변을 Ground Truth로 덮어쓰는 동의 절차(Approval Workflow)를 밟아야 한다.
