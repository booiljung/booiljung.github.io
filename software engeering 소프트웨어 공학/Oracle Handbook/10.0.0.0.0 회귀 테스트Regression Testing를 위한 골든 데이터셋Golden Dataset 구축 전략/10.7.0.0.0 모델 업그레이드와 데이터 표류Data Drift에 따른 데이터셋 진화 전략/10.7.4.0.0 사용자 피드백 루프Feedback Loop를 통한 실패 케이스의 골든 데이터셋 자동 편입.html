10.7.4 사용자 피드백 루프(Feedback Loop)를 통한 실패 케이스의 골든 데이터셋 자동 편입

오라클 환경(실험실)에서는 모두 성공했는데 상용 환경에서 유저가 “불만족(Thumbs Down)” 버튼을 눌렀다면, 이는 오라클의 사각지대(Blind Spot)다.

  • 자동화된 수집 파이프라인: 유저가 부정적 피드백(별점 1점 등)을 남긴 세션 턴 전체를 즉각 [Review Required] 상태로 대기열(Queue)에 넣는다.
  • 인간 개발자가 이 이슈를 디버깅하여 “이런 상황에서는 이렇게 대답해야 했다“는 올바른 궤적을 작성하고 버그를 클로즈(Close)하는 순간, 그 수정된 데이터 페어는 다음번 빌드의 Test Case #10,001 번으로 자동 편입되어 영원히 망각(Regression)을 차단해야 한다.
