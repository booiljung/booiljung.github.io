10.6.2 전체 기능 검증을 위한 풀(Full) 회귀 테스트셋 구성

스모크 테스트를 통과한 패치는 상용계(Production) 배포 직전, Nightly Build 나 주말 정기 릴리즈 잡(Job)에서 Full Dataset을 정면으로 마주해야 한다.

  • 수만 건의 전체 인텐트 및 보안 엣지 케이스를 비동기 병렬 처리(Async Worker Pool)로 호출한다.
  • 여기서 도출된 오라클의 종합 보고서가 커버리지 98% 이상을 찍어야만 최종 릴리스 버튼(Release Gate)의 데프콘 락아웃(Lock-out)이 풀린다. 이 데이터셋은 비용은 비싸지만 “AI가 사고 치지 않았다“는 법적/사업적 보험금에 다름없다.
