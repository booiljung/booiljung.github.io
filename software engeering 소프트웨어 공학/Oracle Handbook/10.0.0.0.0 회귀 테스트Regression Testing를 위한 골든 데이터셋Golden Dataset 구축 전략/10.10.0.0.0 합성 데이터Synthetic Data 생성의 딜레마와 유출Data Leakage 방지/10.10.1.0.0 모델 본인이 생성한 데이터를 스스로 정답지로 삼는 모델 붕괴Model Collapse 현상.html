10.10.1 모델 본인이 생성한 데이터를 스스로 정답지로 삼는 모델 붕괴(Model Collapse) 현상

동일한 모델 A(GPT-4 등)를 통해 질문을 생성하고 다시 답변 템플릿까지 작성하여 10만 건의 데이터셋을 만들었다. 그다음 모델 A를 채점관(Judge)으로 앉혀 테스트한다. 완벽한 100점이 도출된다!

  • 순환 논리의 오류(Circular Reasoning): 이는 평가가 아니다. 모델 A가 자신이 가장 선호하는 단어 패턴으로 문제를 만들고 자기가 채점했으니, 사실상 자아도취(Echo Chamber)에 빠진 것에 불과하다. 이 데이터셋에 모델 B(Claude)를 가져가면 “말투가 다르다“는 이유로 오라클이 실패를 주는 오류를 범한다.
  • 이를 AI 학계에서는 모델 붕괴(Model Collapse) 또는 합성 데이터 수렴 현상이라 부른다. 따라서 데이터를 합성해 낼 때는 반드시 교차 모델 생성(Cross-model Synthesis) 원칙(문제는 Claude가 내고, 검수는 Gemini가 하고, 채점관은 GPT가 맡는 식)을 강제해야 한다.
