10.10.3 합성 데이터 주입 후 인간 검증자(HITL)의 어노테이션 교차 일치도 추적

기계가 무한정 뱉어낸 합성(Synthetic) 데이터는 겉으로 보기에 완벽한 포맷(JSON)을 갖추지만, 비즈니스의 ’맥락’이 거세(Decapitated)된 경우가 많다.

  • LLM 파이프라인이 1만 건의 양질 코드를 생산했다 해도, 최종적으로 골든셋으로 편입(Commit)되기 직전 스테이징(Staging) 구역에서 인간 도메인 전문가(SME)가 랜덤하게 5%를 뽑아 블라인드 테스트(Blind review)를 진행해야 한다.
  • 인간이 보기에 이 기계 생성 데이터의 논리가 빈약하거나 비즈니스 규정을 곡해했다면(일치도가 하락한다면), 해당 합성 배치(Batch) 레이블 전체를 트래시 캔(Trash)으로 폐쇄하고 생성기(Generator System Prompt) 모델의 온도를 낮춘 후 재가동하는 엄격한 수율(Yield Rate) 관리 프로세스가 확립되어야 한다.
