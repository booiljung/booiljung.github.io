10.9.3 지식 베이스(Knowledge Base) 업데이트가 반영되지 않은 구형 골든 데이터의 오류

전자상거래 기업에서 배송비 정책이 무료에서 3,000원으로 변경된 후, RAG 시스템 DB 업데이트를 마친 후 파이프라인 CI 빌드가 연쇄적으로 터졌다(Fail).

  • 원인: 수천 개의 골든 데이터 안에는 여전히 expected_output_core: "배송비는 무료입니다"가 하드코딩되어 있었다. 제대로 정책을 반영하여 똑똑해진 AI 모델이, 멍청하고 낙후된 과거의 오라클 정답지에 가로막혀 배포 승인(Release Gate)을 받지 못하는 아이러니한 상황이다.
  • 교훈: 실시간 지식(RAG)에 의존하는 질문-답변의 경우, 위 10.4 챕터에서 강조했듯, 정답을 고정하지 말고 “주어진 스냅샷 컨텍스트 문서의 비용 정책과 모델의 파싱 비용 숫자가 일치하는지“를 판별하는 로직 테스트(Dynamic Field Extraction) 방식으로 데이터셋의 채점 구조를 추상화(Abstraction) 시켜 놓았어야 한다.
