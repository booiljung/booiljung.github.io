10.9.1 과적합(Overfitting)된 프롬프트 예제 데이터가 범용성을 해친 사례

한 의료 챗봇 스타트업은 의사가 직접 작성한 매우 학술적이고 상세한 50개의 문장만을 골든 데이터 100%로 삼았다. 오라클의 채점을 뚫기 위해 개발팀은 시스템 프롬프트에 “모든 답변은 의학 논문 형식을 따를 것“을 강제했다.

  • 결과: 테스트를 100점으로 통과한 모델이 실 환경(Production)에 나가자, “배가 아파요“라는 일반 환자의 단문 질문에도 3장의 장황한 생물학 논문을 뱉어내며 UX를 완전히 파괴해 버렸다.
  • 교훈: 오직 좁은 범주(High-quality)의 데이터에만 오라클을 맞추면, 그 프롬프트는 엣지 케이스만 잘 맞히고 보편적인 상식 처리 능력(Generalization)을 잃어버리는 **평가 과적합(Eval-Overfitting)**에 빠진다.
