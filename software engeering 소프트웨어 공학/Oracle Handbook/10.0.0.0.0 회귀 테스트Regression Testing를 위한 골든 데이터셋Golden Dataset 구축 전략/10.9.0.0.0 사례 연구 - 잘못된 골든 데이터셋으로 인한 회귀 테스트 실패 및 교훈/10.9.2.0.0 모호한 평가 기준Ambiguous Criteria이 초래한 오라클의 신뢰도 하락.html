10.9.2 모호한 평가 기준(Ambiguous Criteria)이 초래한 오라클의 신뢰도 하락

“친절하게 응답할 것” 이라는 인간도 이해하기 모호한 문장을 수천 개의 expected_output 평가 기준(Criteria) 필드에 적어 놓았다.

  • 결과: LLM-as-a-Judge로 돌아가는 오라클은 같은 모델의 같은 답변을 두고도, 아침에는 Pass를 주고 저녁에는 Fail을 주는 심각한 비결정성(Nondeterminism)을 보였다. 팀은 에러 로그를 믿지 않게 됐고 마침내 테스트 스크립트를 꺼버렸다.
  • 교훈: 오라클은 무당이 아니고 심판이다. 채점 기준은 “반드시 상대방의 이름을 언급하며 시작할 것”, “환불 시 수수료 1,000원을 공제한다고 숫자를 명시할 것” 처럼 완전히 기계적이고(Deterministic) 바이너리(Binary)한 논리 검증식으로만 쪼개져야 한다.
