10.5.3 인간 전문가와 AI 판사(AI Judge) 간의 채점 불일치 데이터 역분석

이론적으로 완벽한 테스트 파이프라인이라면 인간과 AI 오라클의 채점 결과가 같아야 한다. 무작위 샘플 확보를 통해 이 둘을 충돌시켜야 한다.

  • 인간은 Pass를 줬는데 AI Judge는 Fail을 선언한 경우: AI가 너무 융통성(Tolerance) 없이 깐깐하게 채점 중이거나, 모델의 시스템 프롬프트가 과적합되어 있다. 오라클의 임계값(Threshold) 하향 조절 기준이 된다.
  • 인간은 Fail을 줬는데 AI Judge는 Pass를 선언한 경우: 치명적이다. AI 판사가 모델의 교묘한 환각(Hallucination)에 속아 넘어간 상태(Sycophancy)다. 오라클 고도화 팀은 이 불일치(Disagreement) 데이터만을 핀포인트로 추출해 AI 판사의 판단 근거(CoT)를 강제로 교정해야 한다.
