10.4.4 참조 지식(Reference Context) 스냅샷 포함: RAG 시스템의 검색 결과 고정(Freezing)

RAG 환경에서 가장 무서운 변수는 “내일 DB 문서가 바뀌면 어제 지식으로 채점하는 테스트가다 깨진다“는 점이다.

  • 스냅샷 강제화: 골든 데이터셋의 JSON 하위 필드에 reference_context_snapshot을 배열 형태로 봉인해 넣는다.
  • 테스트 런타임 시, 오라클은 현재 DB를 검색(Retrieval)하는 대신 이 스냅샷 데이터를 컨텍스트로 Mocking하여 주입한다. 이를 통해 “검색기(Retriever)의 성능 변화“와 “생성기(Generator)의 모델 변화“라는 두 가지 변수 중 하나를 완벽히 통제(Control)하며, 오직 단일 변인 테스트(Single-Variable Testing)를 성립시킨다.
