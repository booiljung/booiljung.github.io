10.8.2 데이터 저장소의 접근 제어 및 감사 로그(Audit Log)

골든 데이터는 모델의 성능을 속일 수 있는 ’시험지 원본’이다. 누군가 악의를 품고 정답셋을 위변조하면 오라클 시스템 전체를 바보로 만들 수 있다.

  • RBAC(Role-Based Access Control): 일반 개발자는 CI에서 오라클을 구동시켜 결과물을 ’읽기(Read)’만 열려 있어야 하며, 골든 데이터셋을 추가/수정/삭제하는 쓰기 권한은 오직 Lead QA Engineer 레벨로 통제해야 한다.
  • 모든 데이터셋의 조작 내역(누가 언제 어떤 정답을 바꿨는지)은 블록체인형 혹은 위변조 방지 AWS CloudTrail 같은 WORM(Write Once Read Many) 스토리지에 감사 로그(Audit Log)로 타임스탬프와 함께 기록되어야 한다.
