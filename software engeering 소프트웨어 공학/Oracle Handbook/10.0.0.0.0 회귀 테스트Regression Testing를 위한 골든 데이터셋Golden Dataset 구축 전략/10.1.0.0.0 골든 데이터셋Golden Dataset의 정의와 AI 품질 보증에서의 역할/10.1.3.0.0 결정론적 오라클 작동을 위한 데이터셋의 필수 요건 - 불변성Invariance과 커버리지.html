10.1.3 결정론적 오라클 작동을 위한 데이터셋의 필수 요건: 불변성(Invariance)과 커버리지

오라클이 제대로 동작하려면, 그 판별의 기준이 되는 판례(골든 데이터)가 상황에 따라 바뀌어서는 안 된다.

  • 불변성(Invariance): 외부 지식(RAG DB 업데이트, 최신 뉴스)이 업데이트되어도 과거 로직을 검증할 수 있도록, 골든 데이터 내에는 당시 주입했던 문맥(Context Snapshot)이 함께 봉인(Freeze)되어 있어야 한다.
  • 커버리지(Coverage): 코드 커버리지가 아니라 **“비즈니스 의사결정 커버리지”**다. 정상 처리 시나리오(Happy Path) 20%와, 모델이 오작동하기 쉬운 엣지 케이스(Edge Case) 및 보안 위반 시나리오 80%로 데이터의 비율을 찌그러뜨려 놓아야 강력한 보호막을 형성할 수 있다.
