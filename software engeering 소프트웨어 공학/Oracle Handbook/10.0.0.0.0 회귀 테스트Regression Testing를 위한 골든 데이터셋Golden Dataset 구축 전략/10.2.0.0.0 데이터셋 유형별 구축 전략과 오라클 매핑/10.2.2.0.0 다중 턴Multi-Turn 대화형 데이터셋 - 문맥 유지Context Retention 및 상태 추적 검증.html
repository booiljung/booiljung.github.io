10.2.2 다중 턴(Multi-Turn) 대화형 데이터셋: 문맥 유지(Context Retention) 및 상태 추적 검증

AI 챗봇이나 에이전트의 가장 흔한 런타임 에러는 대화가 길어질 때 앞의 기억을 까먹는 현상(Amnesia/Catastrophic Forgetting)이다.

  • 데이터 구조: 입력이 단일 문장이 아니라 메시지 배열(Array of Messages)의 History 형태를 차용한다. 
    {
  "turns": [
    {"user": "내 이름은 앨리스야.", "bot": "기억했습니다."},
    {"user": "피자 주문해 줘.", "bot": "어디로 배달할까요?"},
    {"user": "내 이름으로 예약해.", "expected_bot_action": "예약자: '앨리스'로 컨텍스트 채움"}
  ]
}

- **오라클 매핑**: 상태 추적(State-tracking) 오라클을 활용하여, 마지막 턴(Turn)의 응답에서 초기 컨텍스트(`앨리스`)가 정확히 렌더링되었는지를 의미론적(LLM-as-a-Judge)으로 검사한다.
