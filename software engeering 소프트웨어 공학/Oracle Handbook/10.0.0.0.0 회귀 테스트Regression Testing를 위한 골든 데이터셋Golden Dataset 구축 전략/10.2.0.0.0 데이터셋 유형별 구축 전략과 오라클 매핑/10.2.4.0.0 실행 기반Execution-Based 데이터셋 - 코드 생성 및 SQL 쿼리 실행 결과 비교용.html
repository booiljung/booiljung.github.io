10.2.4 실행 기반(Execution-Based) 데이터셋: 코드 생성 및 SQL 쿼리 실행 결과 비교용

코딩이나 SQL을 생성하는 AI를 텍스트 유사도로 채점하는 것은 의미가 없다. (알고리즘의 구현체는 무한대에 가깝기 때문이다.)

  • 데이터 기반 아키텍처:
  • prompt: “유저 테이블에서 가입일자가 가장 오래된 사람의 이름을 찾아라.”
  • hidden_db_state: {"users": [{"name":"A", "date":"2020"}, ... ]} (데이터베이스 스냅샷)
  • expected_execution_result: "A"
  • 오라클 매핑: AI가 만든 SQL이나 Python 코드를, 격리된 샌드박스 망(Docker) 안에서 hidden_db_state 데이터를 넣고 실제 실행(Execution)한다. 그 후 메모리나 DB에서 반환된 최종 아웃풋이 expected_execution_result와 동일한지 비교하는 ’블랙박스(Black-box) 동적 오라클’을 가동한다.
