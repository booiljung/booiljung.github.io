2.3.3 오라클 비용(Oracle Cost): 정답 생성 및 비교 비용이 테스트 비용을 초과하는 딜레마

엔터프라이즈 환경에서 기술적인 가능 여부보다 먼저 고려되어야 하는 것은 경제성, 즉 ’비용(Cost)’이다. 오라클 문제의 가장 치명적인 함정이 바로 ‘오라클 비용의 역전 위상’ 현상이다.

시스템 SUT를 실행하여 1,000만 개의 결과를 도출하는 데 10달러의 컴퓨팅 자원과 1시간의 러닝 타임만이 소비된다고 가정하자. 만약 이 결과값이 맞는지 검사하기 위한 오라클 정답 세트를 생성하고 비교 연산을 수행하는 데 1,000달러와 일주일의 시간이 낭비된다면, 이것은 공학적으로 완벽히 실패한 설계 구조다.

1. 오라클 비용의 3가지 구성 요소

오라클 구축에 소모되는 비용(Oracle Cost)은 크게 세 가지 축으로 구성된다.

  1. 정답지 생성 비용 (Generation Cost): 완벽한 정답 데이터(Ground Truth)를 계산하거나 수집하는 비용. 외부 데이터베이스 쿼리를 날리거나, 사람(전문가)이 직접 개입하여 데이터를 라벨링하는 인건비가 여기에 집중된다.
  2. 비교 연산 비용 (Comparison Cost): 덤프 된 실제 값과 생성된 기대 값을 퍼즐처럼 맞춰보는 비용이다. 값어치가 싼 스칼라(Scalar) 비교라면 다행이지만, 거대한 영상 벡터(Vector) 두 개의 픽셀 오차율이나 AI가 생성한 두 논문의 문맥적 유사도(Cosine Similarity)를 비교할 때는 고가의 GPU 서버 인스턴스가 쉴 새 없이 돌아가야 한다.
  3. 유지보수 비용 (Maintenance Cost): 비즈니스 정책이나 코드 리팩토링으로 시스템 로직이 아주 미세하게 변경되었을 때, 오라클 정답지 1,000만 개를 다시 처음부터 전부 폐기하고 재생성(Regeneration)해야 하는 생애주기 리스크 자금이다.
