2.3 오라클 문제(The Oracle Problem)와 자동화의 한계

소프트웨어 시스템이 복잡해질수록 테스트를 수행하고 검증하는 메커니즘 역학도 비례하여 거대해진다. 앞 절에서 우리는 참 오라클, 휴리스틱 오라클 등 다양한 방식으로 정답(Ground Truth)의 기준을 세워보려 노력했다. 그러나 이러한 모든 노력에도 불구하고, 도저히 기계적인 알고리즘이나 수학적 명세서로 ’완벽한 정답’을 도출할 수 없는 절대적인 벽에 부딪히는 현상이 발생한다.

수십 년간 소프트웨어 공학계의 가장 거대한 난제로 꼽혀 온 이 한계 상황을 오라클 문제(The Oracle Problem) 라고 부른다. 본 절에서는 오라클 문제의 학술적 정의를 명확히 하고, 테스트 자체가 불가능한 프로그램들의 구조적 특징을 살펴본다. 이를 통해 어째서 완벽한 테스트 자동화 파이프라인(Test Automation Pipeline)이 현실 세계의 특정 도메인 앞에서는 번번이 좌절하는지 증명한다.
