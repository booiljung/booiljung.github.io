2.7.1 골든 데이터셋(Golden Dataset)의 정의와 AI 품질 보증의 기준점

자동화된 오라클 파이프라인(Oracle Pipeline)을 구축할 때 가장 최상단의 권위를 지니는 정보의 원천은 다름 아닌 골든 데이터셋(Golden Dataset) 이다. 이는 인간 도메인 전문가들이 직접 교차 검증하고 승인하여 오차율이 0%에 수렴한다고 합의된 완벽한 참조 데이터(Reference data)의 결합체다.

1. 품질 보증(QA)의 북극성

AI 모델, 특히 RAG(검색 증강 생성) 시스템을 QA 파이프라인에 올렸을 때, 골든 데이터셋은 망망대해에서 배가 길을 잃지 않게 해주는 북극성의 역할을 수행한다.

  • 구성 요소: [명확한 사용자 프롬프트] + [시스템이 꺼내 와야 할 문서 원본] + [결정론적으로 산출되어야 할 완벽한 정답 출력물]
  • 활용법: 만약 파운데이션 모델(Foundation Model)을 GPT-4에서 Claude-3.5로 마이그레이션(Migration)해야 하는 중대한 아키텍처 결정을 내린다고 가정해 보자. 경영진은 두 모델 중 누가 자사 서비스에 더 뛰어난지 감으로 결정할 수 없다. 이때 1,000개의 골든 데이터셋을 두 모델에 동시 주입 백투백(Back-to-back) 테스팅을 돌림으로써, 기존 정답 체계와 더 높은 일치율(Exact Match)을 보이는 모델을 수학적이고 객관적으로 증명할 수 있다.

골든 데이터셋이 비어있는 상태에서 프롬프트 엔지니어링 튜닝을 진행하는 것은, 나침반 없이 눈을 감고 자동차 핸들을 마구잡이로 돌리는 것과 같은 최악의 재앙이다.
