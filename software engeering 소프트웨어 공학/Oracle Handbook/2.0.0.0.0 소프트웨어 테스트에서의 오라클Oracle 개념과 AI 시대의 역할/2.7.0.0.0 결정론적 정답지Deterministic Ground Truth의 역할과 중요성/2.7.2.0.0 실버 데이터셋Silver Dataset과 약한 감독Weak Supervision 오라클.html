2.7.2 실버 데이터셋(Silver Dataset)과 약한 감독(Weak Supervision) 오라클

인간 전문가가 하나하나 장인정신으로 검수한 골든 데이터셋(Golden Dataset)은 품질은 최고지만 가격이 상상을 초월한다(Oracle Cost Explosion). 이를 극복하기 위해 산업계에서 빈번하게 채용하는 차선책 오라클 전략이 바로 실버 데이터셋(Silver Dataset) 과 이를 기반으로 한 약한 감독(Weak Supervision) 메커니즘이다.

1. 실버 데이터셋의 파이프라인 생성

실버 데이터셋은 인간이 아닌, 또 다른 강력한 기계 시스템이나 휴리스틱 스크립트를 통해 자동으로 대량 생성(Auto-generated) 된 정답지 뭉치를 뜻한다.

  • 빠르지만 성능이 아쉬운 오픈소스 로컬 모델(Llama-3 8B) 튜닝을 목표로 한다.
  • 데이터 정답지를 만들기 위해 사람을 고용하는 대신, 프롬프트 10만 개를 GPT-4o나 Claude-3.5-Opus 같은 당대 최고 모델(SOTA) API에 쏘아서 정답지를 생성한다.
  • 이 정답지를 ’사실상 진리(Pseudo Ground Truth)’로 간주하여 로컬 모델의 출력을 채점하는 오라클 잣대로 사용한다.
flowchart LR
    InputDB[(무작위 사용자 질문 10만 개)] --> SOTA[초거대 AI API\n(예: GPT-4o)]
    InputDB --> SUT[자사 경량화 로컬 AI\n(예: Llama-3 8B)]
    
    SOTA --> |비싼 비용 발생| Silver[(실버 데이터셋\n자동 생성된 정답지)]
    SUT --> |저렴한 로컬 연산| Output[(실제 출력 결괏값)]
    
    Output --> Comparator{오라클 비교기}
    Silver --> Comparator
    
    Comparator -- 일정 유사도 이상 --> Pass[Weak Supervision 통과]

2. 약한 감독(Weak Supervision)의 한계 타협

실버 데이터셋에 포함된 정답은 GPT-4o 본인이 뱉어버린 무작위 환각(Hallucination)이 1~2% 정도 섞여 있을 수 있으므로 완벽한 진실이 아닐 수 있다(노이즈 정답).

이 오라클 체재를 약한 감독 시스템이라고 부른다. 그럼에도 불구하고, 수동으로 1,000건을 테스트하는 것보다 약간의 노이즈가 섞였더라도 100만 건을 통계적으로 테스트하는 쪽이 엔지니어링 관점에서 회귀 결함을 막아내는 데 훨씬 경제적이고 안전하다는 트레이드오프(Trade-off) 전략인 셈이다.
