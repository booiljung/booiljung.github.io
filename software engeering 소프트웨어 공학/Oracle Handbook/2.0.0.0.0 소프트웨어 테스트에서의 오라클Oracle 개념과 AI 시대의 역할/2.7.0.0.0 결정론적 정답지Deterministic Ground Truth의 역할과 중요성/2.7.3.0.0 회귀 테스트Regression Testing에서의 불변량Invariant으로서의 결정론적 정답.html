2.7.3 회귀 테스트(Regression Testing)에서의 불변량(Invariant)으로서의 결정론적 정답

LLM의 무작위성 때문에 테스트가 실패를 반복하는 ’Flaky Test 현상’을 잡기 위해서, 오라클의 기준은 반드시 ’텍스트 표면의 유사성’을 버리고 ’논리적 불변량(Invariant)’으로 내려가야 한다.

1. 불변량(Invariant)이란 무엇인가?

결정론적 정답지가 요구하는 ’불변량’이란, 시스템의 함수가 어떤 난리 법석을 떨든, 프롬프트를 어떻게 바꾸든 간에 시작부터 끝날 때까지 절대 변하지 말아야 할 단단한 객체의 성질이나 조건을 말한다. 이는 AI가 다채로운 답변을 내놓더라도 비즈니스 로직을 사수하는 회귀 테스트(Regression Testing)의 철골 구조가 된다.

  1. 상태 불변량 (State Invariant): 보안 AI봇이 코드를 생성했을 때 그 코드의 들여쓰기나 변수명은 매번 다르더라도(비결정성), 이 코드가 실행될 때 chmod 777과 같은 권한 부여 코드가 존재해서는 안 된다는 성질.
  2. 사전 조건/사후 조건 (Pre/Post-conditions): PDF 문서를 기반으로 금융 지표를 뽑아내는 테이블 추출 AI의 경우. 테이블 칼럼명이나 텍스트 길이는 AI 맘대로 생성하게 두더라도, “추출된 자산 총계 숫자(Y)는, 단기 자산(a)과 장기 자산(b)의 합과 수리적으로 정확히 일치해야만 한다(Y == a + b)“는 조건식.

이처럼 AI의 난해하고 비결정적인 표면(Surface) 문자열들을 모두 걷어내고, 그 안에 남겨진 데이터들의 수학적/논리적 교집합만을 추출하여 어설트(Assert) 구문에 묶어버리는 결정론적 오라클만이 Flaky Test를 0으로 만들고 진정한 회귀 방어 기능을 수행하게 만든다.
