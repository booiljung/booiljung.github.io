2.9.2 지속적 학습(Continuous Learning) 환경에서의 동적 오라클(Dynamic Oracle) 업데이트

기계 학습 모델 파이프라인이 MLOps 위에 안착하게 되면, 사용자들의 피드백(Thumbs-up 등)과 새로운 데이터 스트리밍이 유입될 때마다 백그라운드에서 모델이 스스로 파인튜닝(Fine-tuning)을 거듭하는 지속적 학습(Continuous Learning) 궤도에 진입한다. 모델이 살아 움직이는데 오라클만 돌밭에 고정되어 있다면 아무런 의미가 없다.

1. 정적 오라클에서 동적 오라클로의 전환

기존의 정적 오라클(Static Oracle)은 개발자가 하드코딩한 CSV 파일에 [질문, 정답]을 10만 개 채워두고 영원히 테스트 매치만 반복했다. 반면, MLOps에 결합된 동적 오라클(Dynamic Oracle) 시스템은 다음과 같이 유기적으로 정답지를 갱신한다.

  1. 지식 베이스(Knowledge Base) 연동: 오라클이 정답 텍스트를 로컬 변수에 캐싱하는 대신, 테스트 런타임에 직접 사내 실시간 컨플루언스(Confluence)나 노션(Notion) 데이터베이스의 API를 호출하여 최신의 Ground Truth를 Fetch해 온 뒤 평가 기조로 삼는다.
  2. 피드백 루프 증강: 프로덕션에서 인간 QA(은연중에 고객들이 수행하는 어뷰징 신고, 별점 평가 등)에 의해 새롭게 입증된 뛰어난 프롬프트-응답 쌍을 즉각 추출하여 골든 데이터셋 오라클 바구니에 끊임없이 집어넣는다.

이러한 동적 오라클 파이프라인만이 끊임없이 팽창하는 생성형 AI의 지식 생태계를 통제할 수 있는 유일한 모니터링 수단이다.
