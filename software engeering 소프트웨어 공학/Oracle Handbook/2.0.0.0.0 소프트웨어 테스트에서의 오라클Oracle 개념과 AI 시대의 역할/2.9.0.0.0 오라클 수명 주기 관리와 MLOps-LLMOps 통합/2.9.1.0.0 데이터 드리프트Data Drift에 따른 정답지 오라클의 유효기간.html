2.9.1 데이터 드리프트(Data Drift)에 따른 정답지 오라클의 유효기간

통합 테스트(Integration Test)를 위해 assert 구문 안에 하드코딩 해 둔 상수는 영원불변할까? LLMOps 생태계에서 오라클의 정답 세트는 외부 현실 세계의 변화에 의해 가만히 놔두어도 서서히 부패해 가는 유기체와 같다. 이 현상을 데이터 드리프트(Data Drift) 라고 부른다.

1. 진리의 부패 현상

  1. 시간적 드리프트 (Time Drift): “미국의 현재 대통령은 누구인가?” 라는 질문에 대한 오라클의 정답(Expected Value)은 선거철이 지나는 순간 통째로 오답으로 전락한다. AI 모델은 RAG(Retrieval-Augmented Generation)를 통해 최신 뉴스 문서로 알맞은 정답(현재 대통령)을 뱉어주었으나, 과거의 정답을 포개고 누워있는 게으른 오라클이 오히려 “너는 틀렸다!” 며 무고한 모델 생성을 Fail시켜 버리는 이질적 붕괴가 일어난다.
  2. 비즈니스 정책 변동: 금융 상품의 이자율이 3.5\% 에서 이사회 결정으로 하룻밤 새 2.8\% 로 변경되었다고 가정하자. 만약 파이프라인에서 수천 개의 테스트 스크립트 오라클을 즉시 2.8\% 모델로 마이그레이션하지 못하면 파이프라인 전체 트러스트가 날아간다.

따라서 AI 오라클 아키텍처는 반드시 정답 데이터베이스(Ground Truth DB) 컬럼 속에 “이 정답의 유통기한(Expiration Date)은 언제까지인가?” 혹은 **“이 정답지 산출의 근원이 되는 사내 API 통신의 버전 해시는 무엇인가?”**를 추적하는 TTL(Time-To-Live) 메타데이터를 필수적으로 내장해야만 한다.
