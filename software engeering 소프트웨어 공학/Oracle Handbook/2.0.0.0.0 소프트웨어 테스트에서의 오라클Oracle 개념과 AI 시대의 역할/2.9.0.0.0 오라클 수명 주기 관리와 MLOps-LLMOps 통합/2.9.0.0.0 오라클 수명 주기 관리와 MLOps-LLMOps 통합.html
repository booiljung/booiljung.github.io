2.9 오라클 수명 주기 관리와 MLOps/LLMOps 통합

“오라클은 한 번 지어두고 평생 우려먹을 수 있는 성벽이 아니다.”
소프트웨어 1.0 시대에는 요구사항 명세서(Requirements Document)가 변하지 않는 한 테스트 케이스의 정답(Ground Truth)도 절대 변하지 않았다. 그러나 거대 모델과 실시간 데이터망이 엮인 LLM 기반 소프트웨어 애플리케이션 세계에서, 세상의 진리는 매초마다 변동한다.

본 절에서는 변화무쌍한 실제 비즈니스 컨텍스트 속에서 테스트 정답지, 즉 오라클의 ’유효기간’을 어떻게 선언할 것인가 논의한다. 데이터 드리프트(Data Drift) 현상이 파괴하는 오라클의 영속성부터, 버전 관리(Version Control) 시스템과의 결합, 그리고 CI/CD 및 MLOps 파이프라인에서 낡은 오라클을 유기적으로 폐기하고 갱신하는 동적(Dynamic) 수명 주기 관리 철학을 명세한다.
