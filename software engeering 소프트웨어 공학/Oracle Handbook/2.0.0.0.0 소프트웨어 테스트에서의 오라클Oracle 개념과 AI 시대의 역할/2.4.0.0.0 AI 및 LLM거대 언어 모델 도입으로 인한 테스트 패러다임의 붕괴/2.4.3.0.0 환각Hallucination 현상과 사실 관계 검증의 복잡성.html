2.4.3 환각(Hallucination) 현상과 사실 관계 검증의 복잡성

LLM이 일으키는 오류 중 가장 대중적으로 치명타를 입힌 것은 다름 아닌 ‘환각(Hallucination)’ 현상이다. 환각은 단순히 모르는 것을 침묵하는 것이 아니라, 문맥의 통계적 확률망을 타고 그럴듯한 거짓말을 창조해 내는 적극적인 기망 체계이다.

1. 논리적 모순과 외부 사실의 왜곡

환각은 두 가지 차원에서 소프트웨어 퀄리티를 폭파한다.

  1. 내부 논리 구조 붕괴: “A는 B보다 크고, B는 C보다 크다“고 앞서 말해놓고, 두 문단 아래에서 “따라서 C가 가장 큰 값이다“라고 결론을 내버리는 기하학적 어텐션(Attention) 망실 현상이다.
  2. 외부 팩트의 위조: “1994년 월드컵 우승팀을 알려줘“에 대해 “잉글랜드가 우승했습니다“라고 완전히 다른 세계선의 지식을 당당하게 내뱉는다.

이 환각이 테스트에서 그토록 치명적인 이유는 환각 산출물이 문법적, 구조적으로는 완벽한 정답의 형식을 띠고 있기 때문이다. 즉, 에러 코드 500을 뱉거나 문장이 끊기는 등 암시적 오라클(Implicit Oracle)에 걸릴 만한 그 어떤 구조적 결함도 보이지 않는다.

이 그럴싸한 거짓말을 색출하기 위해서는, AI가 뱉은 고유 명사나 숫자 데이터를 실제 엔터프라이즈의 백엔드 데이터베이스(DB)나 외부 검색 엔진 결과와 AND 조건으로 대조하는 무거운 사실 관계 검증(Fact-checking) 파이프라인이 오라클 내부에 강제로 탑재되어야만 한다.
