2.4.2.2 주관적 품질(가독성, 톤앤매너)에 대한 객관적(Objective) 지표화의 난제

B2B 혹은 B2C 플랫폼에서 AI가 생성한 문장을 고객에게 전송한다고 할 때, 비즈니스는 문맥의 ’어투(Tone and Manner)’나 ’정중함(Politeness)’을 통제하고 싶어 한다. 문제는 이러한 속성이 지극히 주관적(Subjective)이라는 데 있다.

1. 객관화의 수리적 한계

“AI의 답변이 고객을 기분 나쁘게 하지 않는 부드러운 말투인가?“라는 제약 조건(Constraints)을 오라클로 검증한다고 가정해 보자.

  • “죄송하지만, 규정상 어렵습니다.”
  • “안타깝게도 해당 요청은 지원하지 않습니다.”
  • “원하시는 바를 이뤄드리지 못해 죄송한 마음입니다.”

이 답변 중 어떤 것이 ‘부드럽고 훌륭한’ 가독성 스코어를 지녔는지 기계적 알고리즘(예: 길이, 쉼표 개수, 어휘 빈도 등)으로 수치화(Objective quantification)하는 것은 불가능에 가깝다. BLEU나 ROUGE 같은 전통적인 NLP 채점 알고리즘은 단어 일치율만을 따질 뿐 인간이 느끼는 뉘앙스의 결을 잡아내지 못한다.

결국 “고객 지향적 어투인가?” 같은 소프트 스펙(Soft Specification)은 필연적으로 인간 오라클(Human Oracle)을 호출하거나, 어투 자체를 매우 보수적인 단답형(JSON 구조 제출 등)으로 강제 변환시켜 버리는 ’아키텍처적 거세’를 병행해야만 통제가 가능해진다.
