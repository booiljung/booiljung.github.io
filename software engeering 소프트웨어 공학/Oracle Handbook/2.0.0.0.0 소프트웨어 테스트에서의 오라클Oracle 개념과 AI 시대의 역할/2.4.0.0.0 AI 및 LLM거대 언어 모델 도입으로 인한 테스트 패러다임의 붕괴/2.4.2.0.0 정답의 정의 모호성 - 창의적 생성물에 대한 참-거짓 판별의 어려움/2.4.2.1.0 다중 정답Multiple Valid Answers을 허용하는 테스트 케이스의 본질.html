2.4.2.1 다중 정답(Multiple Valid Answers)을 허용하는 테스트 케이스의 본질

“N개의 원소를 정렬하라“는 요구사항에 대해 소프트웨어가 내놓을 수 있는 결과물 집합(Result Set)은 완벽히 원소가 오름차순으로 정리된 ’단 하나의 배열’뿐이다. 그러나 생성형 모델의 생태계는 완전히 다른 차원의 차수(Dimension)를 갖는다.

1. 폭발하는 정답의 궤적

AI에게 “주어진 CSV 파일에서 유저 번호와 이메일만 추출하는 정규표현식(Regex)을 짜줘“라고 요청했을 때, 통과될 수 있는 정답 코드는 수백 가지에 달한다.

  1. re.findall(r'(\d+),([\w\.-]+@[\w\.-]+)', text)
  2. re.finditer(...) 등 다른 내장 함수 사용
  3. Pandas 데이터프레임 구조를 차용한 df['email'].extract() 방식

위 세 가지 답변 모두 실행 시 완벽한 결과를 내놓는 참(True)이지만 문자열 그 자체는 완벽하게 다르다.

flowchart TD
    Input[프롬프트 입력] --> AI{거대 언어 모델}
    
    AI --> Ans1[유효한 정답 A]
    AI --> Ans2[유효한 정답 B]
    AI --> Ans3[유효한 정답 C]
    
    Ans1 --> Test[(정답 데이터베이스)]
    Ans2 -. Test_Fail .-x Test
    Ans3 -. Test_Fail .-x Test
    
    Note right of Test: 만약 오라클이 A만을<br/>정답으로 하드코딩해 두었다면,<br/>완벽한 B와 C를 내다 버리는<br/>거절의 오류가 발생!

전통적인 테스트 프레임워크는 유용한 정답이 한 개(Single Valid Answer)일 때만 작동하도록 설계되어 있다. 이처럼 무한대에 가까운 다중 정답 영역(Multiple Valid Answers)을 허용해야만 하는 상황은 기존의 assertEquals 같은 1차원적 오라클의 기능을 완전히 정지시킨다.
