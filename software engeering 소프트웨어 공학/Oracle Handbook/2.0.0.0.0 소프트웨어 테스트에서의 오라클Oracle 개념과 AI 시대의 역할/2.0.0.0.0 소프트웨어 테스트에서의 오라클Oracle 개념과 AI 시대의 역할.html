Chapter 2. 소프트웨어 테스트에서의 오라클(Oracle) 개념과 AI 시대의 역할

인공지능(AI)과 대규모 언어 모델(LLM)이 엔터프라이즈 소프트웨어 시스템의 핵으로 자리 잡으면서, 과거 수십 년간 당연하게 여겨졌던 검증과 테스트의 패러다임이 송두리째 흔들리고 있다. 입력을 넣으면 항상 동일한 예상을 반환하던 프로그램의 시대(Software 1.0)가 저물고, 확률과 통계에 기반하여 매칭되는 결과를 생성하는 비결정론적 소프트웨어의 시대(Software 2.0)가 도래했기 때문이다.

이러한 불확실성의 격랑 속에서 시스템의 무결성을 지탱할 단 하나의 동앗줄은 바로 ’테스트 오라클(Test Oracle)’이다. 오라클은 시스템의 결과가 참인지 거짓인지 판별하는 절대적인 심판이자 정답지이다. 하지만 AI 시대에 이르러 이 오라클의 구축마저 중대한 도전에 직면했다.

본 장에서는 소프트웨어 테스팅의 근간을 이루는 전통적 오라클의 이론과 분류 체계를 재조명하고, 이른바 ’오라클 문제(The Oracle Problem)’가 거대 언어 모델 환경에서 어떻게 극한으로 치닫는지 분석한다. 나아가, 불확실성의 심연 속에서 시스템의 신뢰성을 보장하기 위해 확률적 결과(Stochastic Output)를 결정론적 잣대(Deterministic Ground Truth)로 제어하려는 현대적 오라클의 진화와 역할을 심도 있게 탐구한다.
