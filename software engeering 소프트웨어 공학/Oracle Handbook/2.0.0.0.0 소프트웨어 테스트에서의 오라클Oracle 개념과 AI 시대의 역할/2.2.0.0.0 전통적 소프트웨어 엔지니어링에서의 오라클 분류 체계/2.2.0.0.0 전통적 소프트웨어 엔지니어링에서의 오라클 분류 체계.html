2.2 전통적 소프트웨어 엔지니어링에서의 오라클 분류 체계

소프트웨어 시스템은 각기 다른 목적과 예산을 가지고 설계되며, 그에 따라 모든 소프트웨어 파이프라인(Pipeline)이 동일한 수준의 정밀한 오라클을 가질 수는 없다. 학계와 산업계는 비용과 정확성의 트레이드오프(Trade-off)를 극복하기 위해 오라클을 여러 가지로 쪼개고 분류해 왔다.

완벽하게 정답을 알려줄 수 있는 모델부터, 그저 에러가 나지 않는 것만을 감사히 여기는 수동적 모델까지, 오라클의 분류 체계는 매우 다양하다. 이 장에서는 B.B. Weyuker (1982) 등의 고전적인 테스팅 문헌에서 정의된 오라클의 종류를 6가지 체계(참 오라클, 휴리스틱 오라클, 일관성 오라클, 모델 기반 오라클, 파생 오라클, 암시적 오라클)로 세분화하여 각 개념의 본질과 장단점을 탐구한다.

이러한 고전적 분류법을 이해하는 것은, 훗날 등장할 방종하고 거친 AI 모델의 출력을 제어하기 위해 어떠한 조합의 오라클을 차용하여 인프라를 짜야 할지 결정하는 강력한 레퍼런스가 될 것이다.
