2.2.1 참 오라클(True Oracle): 모든 입력에 대해 완벽한 정답을 제공하는 이상적 모델

소프트웨어 테스팅에서 ’참 오라클(True Oracle)’이란, 시스템에 가능한 한 모든 형태의 무한한 입력(Inputs)을 던져넣더라도 그에 대응하는 오류 제로(0%)의 절대적이고 완벽한 결괏값(Ground Truth)을 제공할 수 있는 궁극의 엔진을 뜻한다.

1. 참 오라클의 구조와 특징

참 오라클은 계산적 버그가 원천적으로 불가능한 이상 세계의 개념에 가깝다.

  • 수학적 완결성: 만약 테스트 대상 시스템(SUT)이 로켓의 궤도를 계산하는 함수를 짰다면, 참 오라클은 실제 뉴턴 물리학 및 아인슈타인의 상대성 이론을 오차 없이 연산하는 완벽한 수리적 모델이어야 한다.
  • 전수 테스팅(Exhaustive Testing) 파트너: 참 오라클이 존재한다면 우리는 모든 경계값이나 엣지 케이스를 무한대로 넣어도 절대 멈추지 않는 테스팅 루프를 돌릴 수 있다. SUT가 도출한 계산값과 참 오라클이 도출한 계산값을 단순한 동등 연산자(==)로 비교하면 그만이기 때문이다.

2. 현실에서의 비용 제약

하지만 “참 오라클은 유니콘(Unicorn)과 같다“는 업계의 뼈 있는 농담처럼, 현실에서 이것을 구축하기란 사실상 불가능에 가깝다.

  1. 복잡도 패러독스(Complexity Paradox): 완벽한 정답을 산출하는 참 오라클을 프로그래밍할 수 있다면, 애초에 그 오라클 자체를 프로덕션 라이브용 소스 코드로 쓰면 된다. 굳이 완벽한 연산기를 두고 별도의 SUT를 만들 이유가 없다.
  2. 블랙 스완 현상: 단순 산술이나 환율 변환(예: 1 USD = 1,400 KRW) 등 극히 제한된 도메인에서는 참 오라클 테이블 구축이 가능하지만, 기상 예측, 자율 주행, 사람의 언어 요약 등과 같은 열린 세계(Open-world) 도메인에서는 무한한 입력 차원(Input space)을 감당할 수학적 정답지 자체가 존재하지 않는다.
