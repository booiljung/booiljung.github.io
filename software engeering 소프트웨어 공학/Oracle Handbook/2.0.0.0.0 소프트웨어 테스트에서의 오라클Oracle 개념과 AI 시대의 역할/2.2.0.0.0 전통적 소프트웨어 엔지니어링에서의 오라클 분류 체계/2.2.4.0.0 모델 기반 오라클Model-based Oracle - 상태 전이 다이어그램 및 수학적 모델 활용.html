2.2.4 모델 기반 오라클(Model-based Oracle): 상태 전이 다이어그램 및 수학적 모델 활용

복잡한 상태(State)를 가진 시스템, 예컨대 임베디드 장비, 항공 관제 시스템, 비디오 게임의 코어 로직 등을 테스트할 때는 단편적인 입력 -> 출력 1회성 검증만으로는 무결성을 보장할 수 없다. 시간의 흐름과 연속된 행동들에 의해 내부 상태가 변화하기 때문이다. 이처럼 시스템의 동작 메커니즘을 외부로 추상화하여 오라클로 활용하는 기법을 모델 기반 오라클(Model-based Oracle)이라 부른다.

1. 상태 전이(State Transition)와 유한 상태 기계(FSM)

모델 기반 오라클은 테스터나 설계 엔지니어가 시스템이 가질 수 있는 모든 상태와 그 변화를 유한 상태 기계(Finite State Machine, FSM) 로직이나 수학적 다이어그램으로 미리 구축해 두는 방식이다.

  • 온라인 쇼핑몰 카트 로직 예시:
  • 상태 A: 빈 장바구니
  • 상태 B: 상품이 있는 장바구니
  • 상태 C: 결제 중
  • 상태 D: 결제 완료

이 경우 시스템은 반드시 A -> B -> C -> D 의 궤적을 거쳐야 한다. 만약 A (빈 장바구니) 상태에서 C (결제 중) 상태로 곧바로 점프하는 이벤트가 감지된다면, 이는 모델 기반 오라클의 추상 모델 규칙을 정면으로 위반한 거대한 결함으로 포착된다.

2. AI 시스템을 향한 모델의 이식성

이 다이어그램들은 시스템 전반의 불변 규칙, 즉 모델 인과성(Model Causality) 을 제어한다. AI 에이전트(Agent)가 자율적으로 마우스를 제어하고 폼을 제출하는 자율 행동(Autonomous Action) 파이프라인에서, 에이전트가 어떤 페이지로 튀어 나갈지 텍스트 봇처럼 예측이 불가능하다.

하지만 모델 기반 오라클을 AI 출력 망 바깥에 둘러싸면 무적의 쉴드를 펼칠 수 있다. “AI가 어떤 확률적 생성 코드를 작성하든 자유지만, 결제 과정에서 C 상태(결제 대기)를 누락하고 바로 D 장부(구매 확정)에 커밋을 시도한다면 무조건 차단한다“는 강력한 구조적 가드레일 역할을 성공적으로 해낼 수 있기 때문이다.
