2.5.2.1 비즈니스 리스크 기반의 허용 오차(Tolerance Margin) 할당

어떤 모델의 성능 스코어나 코사인 유사도(Cosine Similarity) 수치가 곧장 그 자체로 Pass/Fail을 의미하지 거지는 않는다. 동일한 AI 챗봇 알고리즘이라 하더라도, 이 시스템이 적용되는 ’비즈니스 도메인의 리스크 수준’에 따라 오라클의 허용 오차(Tolerance Margin)는 극단적으로 달라져야 한다.

1. 상황별 오라클 임계값 동적 할당

  1. 저위험군 (Low Risk): 블로그 본문 요약 AI
  • 요구사항: 오타나, 맥락이 약간 어색한 문장이 있어도 생명이나 자산에 영향을 미치지 않는다.
  • 허용 오차 할당: 의미론적 임베딩 유사도 기준 Threshold = 0.65 이상이면 Pass 처리. 빠르고 저렴하게 많은 데이터를 통과시킨다.
  1. 고위험군 (High Risk): 의료 약물 처방 조언 혹은 금융 대출 평가 AI
  • 요구사항: 환각이 일어나는 순간 소송이나 인명 피해로 직결된다. 절대적인 수리적·논리적 합격을 보장해야 한다.
  • 허용 오차 할당: 유사도 스코어링 같은 애매한 통계 척도를 버린다. 무조건 정규표현식 매칭, 사내 약물 금기 DB와의 교차 교집합 사이즈 0(Zero) 등 강성 규칙 기반의 오라클을 병행하고, 모든 평가 지표의 Threshold = 0.99 이상일 때만 통과시킨다.

이처럼 리스크 기반 테스팅(Risk-based Testing) 룰에 의해, 엔진은 같지만 비즈니스의 생사여탈 여부에 따라 오라클의 Threshold를 유동적으로 할당하는 것이 확률적 출력을 결정론적으로 묶어두는 훌륭한 타협점이다.
