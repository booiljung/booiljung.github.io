2.5.1 AI 오라클의 새로운 요구사항: 유창성(Fluency), 사실성(Factuality), 일관성(Consistency)

전통적인 시스템 테스팅의 유일한 목표가 Actual == Expected의 논리적 합치 여부였다면, 생성형 AI(Generative AI)의 오라클은 인간과 유사한 언어의 모호성을 검증해야 하므로 평가 지표가 다원화된다.

AI 오라클은 더 이상 “정답이 맞는가?“라는 단일 차원의 판정기(Verdict)로 기능할 수 없으며, 출력 데이터가 지닌 다면적인 속성을 개별적으로 채점하는 종합 평가 엔진(Evaluation Engine)으로 진화해야 한다. 그 3대 핵심 요구사항은 다음과 같다.

1. 사실성 (Factuality)

가장 강력하고 필수적인 검증 요건으로, 환각(Hallucination) 여부를 판정한다. 생성된 텍스트나 코드가 현실 세계의 진리(Ground Truth) 혹사 사내 데이터베이스의 정보와 부합하는지를 가린다.

  • 오라클의 역할: 응답 문장 내에서 중요한 명사, 숫자, 고유 명사를 추출(Information Extraction)한 뒤, 그것이 RAG(검색 증강 생성) 파이프라인의 원본 문서에 존재하는 내용인지 역추적(Traceability)하는 역할을 한다. 조금이라도 위조된 정보가 섞였다면 Fail 처리한다.

2. 일관성 (Consistency)

모델이 동일한 질문이나 문맥상 유사한 질문에 대해 얼마나 일관된 논리 스탠스(Stance)를 유지하는가에 대한 지표이다.

  • 오라클의 역할: 모델이 세션 초반에 “A 알고리즘이 낫다“라고 대답해놓고, 나중에 프롬프트를 약간 변형해서 다시 물어보았을 때 “B 알고리즘이 낫다“라고 말을 바꾸는지 감시한다. (예: Self-Consistency 검증 알고리즘 탑재).

3. 유창성 (Fluency) 및 윤리성 (Safety)

아무리 사실에 입각한 내용이라 하더라도 문법적으로 파열음이 심하거나 너무 어색한 문장 구조를 지녔는지, 나아가 편향적이고 공격적인 언어(Toxicity)를 포함하고 있지는 않은지 검증한다.

  • 오라클의 역할: 이는 전통적인 단언문(Assertion)으로는 절대 해결 불가한 영역이다. 따라서 유창성 및 안전성 오라클은 필연적으로 LLM-as-a-Judge 기법을 동원한 별도의 ‘평가 심판 AI’ 계층을 탑재하여 스코어링(Scoring)을 매기는 방향론으로 나아가게 된다.

새로운 시대의 오라클은 위 세 가지 속성의 채점 결과가 비즈니스 도메인의 요구 수준(Threshold)을 만족시킬 때 비로소 초록 불(Pass)을 밝힌다.
