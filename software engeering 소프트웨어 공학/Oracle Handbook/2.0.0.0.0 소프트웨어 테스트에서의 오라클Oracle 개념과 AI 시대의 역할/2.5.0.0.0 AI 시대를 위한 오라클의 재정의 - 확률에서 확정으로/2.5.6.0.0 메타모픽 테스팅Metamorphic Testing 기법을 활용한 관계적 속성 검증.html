2.5.6 메타모픽 테스팅(Metamorphic Testing) 기법을 활용한 관계적 속성 검증

복잡계나 AI 기반 애플리케이션 등 “완벽한 정답 출력값(Ground Truth Output)을 사전에 알 수 없는” 극단적 상황에서 소프트웨어 엔지니어링 업계가 고안해 낸 가장 우아하고 강력한 테스트 프레임워크가 바로 **메타모픽 테스팅(Metamorphic Testing)**이다.

이 테스트 기법의 핵심 철학은 “개별 출력값이 정답인지는 몰라도, 입력값을 특정 방식으로 변형(Transform)했을 때 출력값들도 반드시 어떠한 ’관계적 수식’을 따라 변형되어야만 한다“는 불변의 관계성(Metamorphic Relation)을 오라클로 활용하는 데 있다.

1. 비결정성에 대항하는 변형 관계(Metamorphic Relations)

AI 모델 테스트에 메타모픽 테스팅이 어떻게 오라클 역할을 수행하는지 살펴보자.

  1. 감성 분석(Sentiment Analysis) 모델 예시
  • 소스 입력(x): “이 영화의 스토리는 평범했다.” -> AI 판정 결과(f(x))는 ‘중립(Neutral)’.
  • (참 오라클 부재: ’중립’이 완벽한 정답인지 모름)
  • 변형 룰 개발(Metamorphic Relation): 문미에 “하지만 결말이 너무 끔찍하다.” 라는 명백히 부정적인 구문을 덧붙이면, 전체 감성 점수는 **“반드시 이전보다 더 하락하거나 적어도 동등해야 한다”**는 절대 규칙을 잡는다.
  • 후속 입력(x') 주입: “이 영화의 스토리는 평범했다. 하지만 결말이 너무 끔찍하다.” -> AI 판정 결과(f(x'))가 ’긍정(Positive)’으로 나온다면?
  • 오라클의 일격: 우리는 f(x)의 정답을 모름에도 불구하고, 메타모픽 규칙인 Score(x) \ge Score(x') 를 위반했다는 단초 하나만으로 즉각 AI 모델의 핵심 논리적 버그(Fail)를 확정적으로 잡아낼 수 있다.
