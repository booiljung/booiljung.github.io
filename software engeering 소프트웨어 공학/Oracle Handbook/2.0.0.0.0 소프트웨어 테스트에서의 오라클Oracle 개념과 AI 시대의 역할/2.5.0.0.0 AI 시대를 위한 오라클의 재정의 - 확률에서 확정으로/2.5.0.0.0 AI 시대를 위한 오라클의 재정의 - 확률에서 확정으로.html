2.5 AI 시대를 위한 오라클의 재정의: 확률에서 확정으로

“AI 시스템이 반환한 데이터가 완벽한가?”
앞선 2.4장에서 무한대의 경우의 수와 환각(Hallucination)의 위협 속에서 이 질문에 대답하는 것이 불가능에 가깝다는 점을 확인했다. 하지만 엔터프라이즈 환경에서 “이 서비스는 AI 기반이라 테스트가 불가능하니 그냥 배포합시다“라는 핑계는 결코 통용될 수 없다. 어떻게든 시스템은 검증되어야만 한다.

본 장에서는 옛 시대의 화석처럼 여겨지던 오라클의 개념을 AI 시대에 맞게 재정의(Redefinition)한다. 확률적으로 요동치는 생성형 결괏값을 어떻게든 ‘결정론적인 안전망’ 내부로 가두기 위해 통제 가능한 평가 척도를 재설계하는 과정이다.
유창성과 사실성이라는 새로운 요구사항의 대두부터, JSON과 같은 구조적 제약을 거는 구조적 오라클(Structural Oracle), 메타모픽 테스팅(Metamorphic Testing)을 차용한 논리 검증까지, 새로운 시대에 딥러닝 애플리케이션의 멱살을 잡고 신뢰성을 견인할 현대적 오라클의 아키텍처적 구성을 살펴본다.
