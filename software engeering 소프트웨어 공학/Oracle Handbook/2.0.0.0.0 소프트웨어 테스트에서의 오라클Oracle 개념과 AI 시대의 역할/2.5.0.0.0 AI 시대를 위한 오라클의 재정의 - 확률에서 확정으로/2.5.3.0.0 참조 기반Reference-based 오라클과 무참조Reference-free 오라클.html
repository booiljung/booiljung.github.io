2.5.3 참조 기반(Reference-based) 오라클과 무참조(Reference-free) 오라클

AI 모델의 출력 결과가 문맥상 유요한지 판별하는 현대적 테스트 프레임워크는, 검증의 잣대로 사용할 기준점(Reference)의 유무에 따라 아키텍처가 둘로 쪼개진다. 이는 정답지(Ground Truth) 확보의 비용 문제를 완화하기 위해 고안된 체계다.

1. 참조 기반 오라클 (Reference-based Oracle)

테스트 케이스마다 사람이 잘 짜놓은 “모범 정답(Gold Reference)“이 데이터베이스에 선존칭하는 방식이다.

  • AI가 뱉어낸 응답과 이 모범 정답 간의 일치도(Rouge, BLEU 혹은 코사인 임베딩 스코어)를 계산하여 거리가 가까우면 합격시킨다.
  • 한계: 비즈니스 정책이 변하면 수만 개의 모범 정답 DB 라벨을 기획자가 다시 써넣어야 하는 엄청난 오라클 유지보수 비용(Maintenance Cost)이 발생한다.

2. 무참조 오라클 (Reference-free Oracle)

혁신적인 방식으로, 아예 비교해야 할 ‘모범 정답’ 자체를 두지 않고 출력 결과물 그 자체가 내포한 성질만을 가지고 평가한다.

  • 방법론: LLM이 작성해 낸 긴 문단이 들어오면, 이 오라클은 “정답 텍스트와 비교“하는 대신 출력물 안에 모순되는 논리가 없는지, 혹은 시스템에 알려진 금지어구(Blocklist) 데이터베이스 룰에 위배되지 않는지만을 독립적으로 스캐닝한다.
  • LLM-as-a-Judge 기법과도 궁합이 잘 맞는다. “정답은 안 줄 테니, 네가 AI의 출력을 읽어보고 문법이 파괴되었거나 회사의 윤리 강령을 어겼는지 평가만 해줘“라고 또 다른 AI 구조에 던지는 식이다.

이러한 무참조 오라클로의 이전은 정답 데이터셋 구축 비용(Oracle Cost)을 획기적으로 감축시켜, 기업이 애자일(Agile)하게 AI 앱을 테스트하고 배포할 수 있는 환경을 열어준다.
