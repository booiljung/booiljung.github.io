2.1 테스트 오라클(Test Oracle)의 기초 이론 및 정의

소프트웨어가 올바르게 작동하는지 확인하는 과정은 본질적으로 “입력(Input) 대비 출력(Output)의 정합성 검증“이다. 특정 입력값을 소프트웨어라는 상자(Box) 안에 밀어 넣으면 로직이 작동하여 어떠한 상태 변이나 메시지를 반환한다. 이때 반환된 그 결과값이 ’우리가 의도했던 정답’인지 판별하기 위해서는 비교 대상이 되는 명백한 기준점이 필요한데, 이를 소프트웨어 공학에서는 테스트 오라클(Test Oracle) 이라고 부른다.

본 절에서는 소프트웨어 테스팅의 뼈대를 이루는 오라클의 기초 이론을 확립한다. 오라클이 지니는 본질적인 의미부터, 어떠한 요소들로 오라클이 구성되며, 어떤 원천(Source)으로부터 그 기준을 세우는지 살펴본다. 이 기초 이론은 향후 AI의 비결정론적 출력물을 평가할 때 왜 전통적인 오라클이 붕괴하고, 새로운 오프라인 평가 파이프라인(Evaluating Pipeline)이 필요한지 이해하기 위한 첫 번째 나침반이 될 것이다.
