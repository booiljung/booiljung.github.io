2.8.4 AI 오라클의 신뢰성 문제: 편향(Bias)의 전이와 자기 강화(Self-Reinforcement) 위험

AI-as-a-Judge 패러다임이 선사하는 편의성의 이면에는, 소프트웨어 공학 역사상 유례를 찾아볼 수 없는 ’가장 소리 없는 결함(Silent Bug)’의 그림자가 드리워져 있다. LLM을 오라클(정답의 판별자)로 세웠을 때 발생하는 존재론적인 신뢰성 붕괴 사이클에 관한 문제다.

1. 편향의 메아리 방 (Echo Chamber of Bias)

심판관 위치에 있는 AI 역시 어차피 방대한 인간의 데이터베이스로 프리트레이닝(Pre-training) 된 통계적 앵무새에 불과하다. 심판관 자체가 젠더 편향, 인종 편향, 혹은 특정 정치적 스탠스를 내면화하고 있다면, 아무리 객관적인 SUT의 출력이더라도 심판관이 “이 대답은 나의 편향과 맞지 않는다“며 가차 없이 오류(Fail)로 찍어 누르는 사태가 발생한다.

결국 테스트 파이프라인 전체가 심판관 모델의 그릇된 편향성에 맞춰 아키텍처 구조를 강제로 왜곡시키는 방향으로 수렴하게 된다.

2. 자기 거장의 폐해 (Self-Reinforcement)

더 끔찍한 것은 SUT를 개발한 회사와 심판관 오라클을 제공하는 벤더가 같을 때(예: GPT-3.5의 미디어 출력을 GPT-4로 테스트할 때) 일어나는 자기 선호도(Self-preference) 편향이다.

  • 심판관 GPT-4는 타사 모델(Claude)의 미려한 문장보다 자사 모델(GPT-3.5) 특유의 딱딱한 문장 구조를 ’더 완벽한 정답’이라고 후하게 채점하는 경향성을 띤다.
  • 이 오라클의 잣대 안에서 모델 체계는 영원히 개선 교차점을 잃어버리고 자가 당착에 빠진 채 발전(Evolution)을 빙자한 자기 강화(Self-Reinforcement) 사이클로 추락하고 만다.

따라서 AI 기반 오라클은 궁극의 정답지가 아니라 어디까지나 휴리스틱의 확장판으로 간주해야 하며, 결국 인프라 최후의 브레이크로서 결정론적 제약(Deterministic Constraints) 구조가 등 뒤에 버티고 있어야만 한다.
