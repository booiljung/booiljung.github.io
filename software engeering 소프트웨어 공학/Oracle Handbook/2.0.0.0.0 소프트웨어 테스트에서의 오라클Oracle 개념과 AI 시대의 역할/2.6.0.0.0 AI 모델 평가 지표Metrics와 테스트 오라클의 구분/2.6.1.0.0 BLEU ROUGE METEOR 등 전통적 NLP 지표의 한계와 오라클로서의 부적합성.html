2.6.1 BLEU, ROUGE, METEOR 등 전통적 NLP 지표의 한계와 오라클로서의 부적합성

전통적 자연어 처리(NLP) 분야에서는 기계 번역이나 문서 요약 결과물을 평가하기 위해 인간이 작성한 모범 답안과 얼마나 많은 단어가 겹치는지 측정했다. N-그램(N-gram) 기반의 교집합 비율을 계산하는 BLEU, ROUGE, METEOR 지표들이 그것이다. 그러나 이들을 소프트웨어 릴리즈 파이프라인의 오라클로 임명하는 것은 명백한 설계 결함이다.

1. 의미를 버리고 형태만 쫓는 맹점

다음 두 문장을 비교해 보자.

  • Reference (정답지): “환불은 전액 불가능합니다.”
  • AI Output (실제 출력): “환불은 무조건 가능합니다.”

이 경우 어절 수준에서 “환불은“과 “가능합니다” 형태소가 완벽하게 겹치므로 ROUGE 점수는 상당히 높게 치솟는다. 하지만 소프트웨어 정책상 이 AI의 출력은 회사의 재무 기둥을 뽑아버리는 치명타(Fail)이다. 반대로 의미가 완벽히 동일하더라도 AI가 유의어(Synonyms)를 사용하면 이 바보 같은 지표들은 점수를 0점으로 깎아버린다.

    Output[AI 생성 발화] --> Match
    Match -- "단어 많이 겹침!" --> HighScore[Score: 80점]
    end
    
    HighScore -. "하지만 핵심 논리가\n정반대라면?" .-> Disaster((비즈니스 참사))

절대 훼손되어서는 안 되는 엔터프라이즈의 ’금지 규칙’이나 ’무결성 전제 조건’을 방어하는 데 있어, 전통적인 NLP의 통계적 단어 일치율 지표는 아무런 방어막을 쳐주지 못하는 허수아비에 불과함을 명심해야 한다.
