2.6.3 정량적 점수(Score)를 이진 판정(Pass/Fail) 오라클로 변환하는 기준

모델 지표(Metrics)들의 한계를 인지했다고 하더라도, 모델을 제어하려면 결국 그 메트릭들을 재료로 써야만 한다. 지표는 연속적인 소수점(0.0 \sim 1.0)의 나열이다. 소프트웨어 엔지니어링의 오라클 파이프라인은 CI/CD 배포를 위해 무조건 **Yes(Pass) or No(Fail)**라는 이분법적 결론을 강제한다. 따라서 연속된 점수를 이진 시스템으로 끊어버리는 변환 공정이 필요하다.

1. 단두대(Threshold)의 설정 조건

스코어를 이진 판정기로 바꾸기 위해서는 도메인 로직에 기반한 복합 임계값(Composite Threshold) 필터를 설계해야 한다.

  • 예시 전략 1: 절대적 상한선(Hard Boundary): API 구조를 만들어내는 코드에서는 문법이나 스키마 일치율이 100% 미만이면 가차 없이 Fail시킨다. if (Schema_Match_Score < 100) return FAIL;
  • 예시 전략 2: 유동적 허용선(Soft Boundary): 인간의 감정을 분석하거나 위로하는 심리 상담 AI 챗봇의 경우, “어조의 공감성(Empathy) 스코어“가 70점만 넘겨도 폭넓게 Pass 처리를 내어준다.
# 다차원 지표를 이진 오라클로 변환하는 파이프라인 로직 (예시 구조)
def evaluator_oracle(ai_output, expected_ground_truth):
    
    # 지표 1: 형식 검사 (100% 일치해야 함)
    if not validate_json_schema(ai_output):
        return "FAIL: 구조 파괴"
        
    # 지표 2: 환각 사실 검사 (99% 이상 신뢰도)
    fact_score = check_factuality_score(ai_output, db_context)
    if fact_score < 0.99:
        return f"FAIL: 환각 의심 (Score: {fact_score})"
        
    # 지표 3: 코사인 유사도 (부분적 허용)
    sim_score = compute_cosine_similarity(ai_output, expected_ground_truth)
    if sim_score < 0.70:
        return f"FAIL: 원래 기획 의도와 다름 (Score: {sim_score})"
        
    # 모든 허들을 통과했을 때 비로소 초록 불(Pass) 
    return "PASS"

위 로직처럼 연속적인 퍼지(Fuzzy) 확률 데이터들을, 명확한 임계값 브레이커(Breaker)를 통해 불린(Boolean) 형태의 테스트 오라클로 형변환(Type-casting) 시키는 일련의 과정이야말로 AI 엔지니어링의 중추를 구성한다.
