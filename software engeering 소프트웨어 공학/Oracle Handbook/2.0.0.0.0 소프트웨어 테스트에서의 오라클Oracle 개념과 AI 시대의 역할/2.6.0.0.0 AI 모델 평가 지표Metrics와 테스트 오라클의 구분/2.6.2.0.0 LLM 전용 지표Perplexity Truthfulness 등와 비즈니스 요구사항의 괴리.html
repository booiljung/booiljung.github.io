2.6.2 LLM 전용 지표(Perplexity, Truthfulness 등)와 비즈니스 요구사항의 괴리

앞선 NLP 지표의 한계를 통감한 과학자들은 LLM을 위해 더 발전된 통계 지표를 발명해 냈다.
가장 대표적인 것이 텍스트의 유창성과 자연스러움을 재는 **퍼플렉서티(Perplexity)**나, 거짓말 확률을 잡는 트루스풀니스(Truthfulness) 등이다. 하지만 이 최첨단 지표들조차도 소프트웨어 엔지니어 생태계에 곧장 적용하기에는 결정적 괴리가 존재한다.

1. 아카데믹 스코어 vs 도메인 계약 지표

퍼플렉서티(Perplexity)가 낮게 나왔다는 것은 다음 단어 예측이 매우 매끄러워서 마치 인간이 쓴 글처럼 자연스럽다는 의미일 뿐이다. 진실성과는 하등 연관이 없다. 극단적으로 말해, LLM이 놀라울 정도로 부드럽고 유창한 말솜씨로 희대의 사기성 거짓 안내문을 줄줄이 적어 내려갈 때, 모델의 퍼플렉서티 지표는 ’최고 등급’을 가리킨다.

또한 글로벌 데이터셋으로 학습된 Truthfulness 평가 지표는 “지구가 둥글다“는 상식의 진위를 판별할 수는 있어도, **“우리 회사 VIP 고객에게 적용되는 마일리지 환산 비율이 1:1.5가 맞느냐?”**라는 극도로 폐쇄적인 엔터프라이즈 사양(Specification)의 진위는 절대 알 수 없다.

소프트웨어 시스템은 논문의 리더보드 점수나 일반 상식으로 구동되지 않는다. 시스템을 굴러가게 하는 것은 철저한 ’도메인의 비즈니스 규칙(Domain Business Rule)’이다. 따라서 아카데믹스(Academics)에서 칭송받는 평가 지표들을 맹신하여 사내 QA 파이프라인의 오라클로 대충 치환해 버리는 무지는 곧바로 상업적(Commercial) 배포 사고로 이어진다.
