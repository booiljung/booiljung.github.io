2.6.4 인간 평가(Human Evaluation)와 자동화된 오라클 간의 상관관계 분석

오라클이 아무리 높은 잣대와 엄밀한 임계값(Threshold)을 구비했다 한들, 그 오라클이 정말로 실효성이 있는지 입증하는 궁극의 심판은 결국 ’인간(사내 도메인 전문가)’이다. 따라서 이상적인 오라클 파이프라인을 구축하려면 개발된 ’스코어 변환 오라클’과 ‘실제 인간 전문가의 채점 결과’ 사이의 상관관계(Correlation)가 기하급수적으로 일치해야만 한다.

1. 인간과의 정렬(Alignment) 확인

  1. 자동화 체계가 Pass를 선언한 산출물 1,000건과 Fail을 선언한 산출물 1,000건을 랜덤 샘플링(Random sampling)한다.
  2. 이 데이터를 오라클 채점 점수 스티커를 가린 채 인간 도메인 변호사/의사(Human Oracle)에게 넘겨 평가시킨다.
  3. 자동화 오라클은 “최고의 답변“이라며 99점을 주었으나 인간이 “논리적 헛소리“라며 0점을 폭격한 케이스가 양산된다면, 해당 지표는 당장 파기되어야 할 쓰레기 지표다.

2. 스피어만 상관계수(Spearman Correlation) 기반 방어

일반적으로 잘 설계된 하이브리드 자동화 오라클은 인간 도메인 지식과의 스피어만 상관계수 산출 결과가 0.8 혹은 그 이상으로 강하게 유착되어 있다. 즉, 기계가 Pass시킨 것은 사람 눈에도 훌륭해야 하고, 기계가 차단시킨 것은 사람 눈에도 형편없어야 한다.

만약 이 상관관계가 성립되지 않은 허술한 코사인 유사도 척도 메트릭을 CI/CD의 통과 팩터로 꽂아둔다면, 경영진 눈에는 매일매일 “테스트 성공률 100%“의 아름다운 대시보드가 찍히겠지만, 시장의 유저들은 서비스가 쏟아내는 이상한 답변에 분노하며 이탈할 것이다. 자동화된 오라클의 권위는 오직 인간과의 극한적 평가 교집합성(Intersection rate)에서만 발생한다.
