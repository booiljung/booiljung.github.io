2.10 요약 및 3장 연결: 왜 우리는 다시 결정론적 정답지를 갈구하는가

소프트웨어 2.0 시대를 맞이하며 가장 환호했던 기술은 모든 것을 기계에 일임하는 딥러닝(Deep Learning)의 블랙박스 구조였다. 개발자들은 코드를 작성하는 대신 데이터를 쏟아붓고 결과가 무작위로 좋게 튀어나오길 기도했다.

그러나 이 환희는 길지 않았다. 엔터프라이즈 환경이라는 거친 야생에 이 ’통제되지 않는 지능’을 방사하자, 시스템은 확률(Probabilities)의 수렁에 빠져버렸고 테스트 오라클(Test Oracle)은 정답을 찾지 못해 비명을 질렀다.

우리는 2장을 통해 AI 시대의 역설을 목도했다. 거대 언어 모델이 아무리 화려한 자유도를 뽐내더라도, 결국 비즈니스의 심장을 꿰뚫는 핵심 파이프라인은 1과 0, 단 하나의 정답만을 허용하는 ’결정론적 정답지(Deterministic Ground Truth)’의 엄격한 규율계 내부로 포섭되어야 한다는 사실이다.
