2.10.1 통제 불가능한 AI를 통제 가능한 시스템으로 편입시키기 위한 필수 조건

마이클 조던(Michael I. Jordan) 교수와 같은 선구자들이 경고했듯, 기계 학습은 그 자체로 마법의 블랙홀이 되어선 안 되며 기존의 엔지니어링 생태계, 즉 ’설계-구현-테스트-배포’라는 예측 가능한 공학 기계 내부의 부품으로 안착해야만 한다(Engineering System Integration).

1. 불변의 브레이크 장치

통제 불가한 확률 변수를 통제 가능계로 끌어내리기 위해서는 다음과 같은 강력한 징검다리가 요구된다.

  1. 명세(Specification)의 제약: “아무 대답이나 멋지게 해줘“가 아니라, JSON 스키마 구조라는 굴레를 강제로 씌움으로써 출력을 규격화시킨다. 구조적 오라클(Structural Oracle)의 역할이다.
  2. 관계성(Metamorphic Relation)의 확보: 정답을 모른다 한들, 입력 A에 특정 연산을 가해 조작한 A가 들어갔을 때, 응답 B와 B 사이의 관계를 수학적 대수 식에 욱여넣음으로써 논리를 박제한다.
  3. 규제 앵커(Compliance Anchor) 타설: 그 어떤 감성적 칭찬 앞에서도 멈춰야 할 땐 멈춰 세울 수 있는 강성 오라클(Hard Oracle)이 자동차 최후의 유압식 브레이크처럼 작동해야 한다.

AI는 창의력을 발휘하는 엔진이고, 이러한 결정론적 규약 세트들은 자동차가 절벽으로 굴러떨어지지 않게 하는 타이어와 조향 시스템이다. 이 두 가지가 완전히 결합할 때 비로소 상용화 가능한 통제 시스템이 완성된다.
