5.5.4 무작위 입력 생성(Fuzzing)을 통한 프롬프트 인젝션 방어력 테스트

오라클은 공격받는다. 일반 사용자의 오타부터, 의도적으로 시스템의 제어권을 탈취하려는 치명적 프롬프트 인젝션(Prompt Injection)까지.
입력 경계의 무결성을 시험하기 위해 랜덤 데이터를 쏟아붓는 기법을 퍼징(Fuzzing) 이라 한다.

  • 변이 퍼징 (Mutation Fuzzing): 정상적인 텍스트 중간에 SQL 인젝션 공격 쿼리나, “Ignore above instructions” 따위의 악의적 우회 명령어를 랜덤하게 섞는다.
  • 오라클의 역할: 이러한 쓰레기 데이터 폭격 속에서도 우리의 AI 에이전트가 절대로 비밀 키(Secret)를 토해내지 않고, {"status": "error", "reason": "unauthorized"} 라는 무표정한 JSON 포맷만을 견고하게 반환하는지 감시한다.

퍼징 기반의 속성 테스트는 단순한 QA를 넘어서, 엔터프라이즈 AI 서비스의 보안과 규정 준수를 증명하는 자동화된 감사관(Auditor)이 된다.
