5.5.1 Hypothesis 등 프레임워크를 활용한 입력 불변식(Invariant) 검증

Hypothesis(Python) 같은 속성 기반 테스트 프레임워크는 개발자가 데이터 생성 전략(Strategy)을 선언하면, 프레임워크가 알아서 수천 개의 더미 데이터를 쏟아부어 코드를 고문한다.

오라클 시스템에 이를 적용할 때의 핵심은 불변식(Invariant) 의 정의다.

  • 생성 규칙: 무작위 길이의 영문자, 특수기호 혼합 문자열 주입
  • 오라클 불변식 (Invariant): “사용자 입력이 얼마나 길든, AI 번역 모듈의 응답은 항상 언어 감지 API에서 ’Korean’으로 식별되어야 한다.”

이 테스트는 assertEquals("안녕하세요", response)를 넘어서, 입력이 쓰레기(Garbage)든 극도로 복잡한 소설이든 간에 우리의 프롬프트 엔지니어링이 “출력을 한국어로 강제한다” 는 시스템적 속성을 수학적으로 입증해 준다.
