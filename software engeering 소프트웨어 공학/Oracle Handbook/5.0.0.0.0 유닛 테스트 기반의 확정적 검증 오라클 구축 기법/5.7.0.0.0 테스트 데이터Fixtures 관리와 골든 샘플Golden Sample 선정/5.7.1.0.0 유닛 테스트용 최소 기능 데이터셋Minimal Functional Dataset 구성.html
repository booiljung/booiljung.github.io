5.7.1 유닛 테스트용 최소 기능 데이터셋(Minimal Functional Dataset) 구성

유닛 테스트 환경에서 “1MB짜리 로그 파일“을 통째로 쑤셔 넣어 프롬프트 응답을 판단하는 것은 시간과 비용의 낭비다.
테스트 엔지니어는 방대한 원본 로그에서 오직 ’버그를 촉발(Trigger)시키는 최소 단위의 문맥(Minimal Functional Dataset)’만을 추출해 Fixture로 등록해야 한다.

Fixture 작성 원칙:

  • 5줄 이하로 압축하라. 불필요한 배경 설명(Noise)을 덜어내면 토큰 소모가 줄어들고 테스트 속도가 폭발적으로 올라간다.
  • 50개의 정예 골든 샘플(Golden Sample)을 만든다. 이는 ‘반드시 특정 결괏값이 도출되어야 하는’ 고정된 정답지 역할을 하며, 프롬프트 엔지니어가 템플릿의 변수를 수정할 때마다 무결성을 증명하는 기준점이 된다.
