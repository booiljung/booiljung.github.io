5.7.2 엣지 케이스(Edge Case) 및 코너 케이스(Corner Case) 발굴 전략

일반적인 해피 패스(Happy Path) 데이터만으로는 AI의 환각을 진단할 수 없다. 오라클이 가장 격렬하게 비틀거리는 지점인 경계선 변형 데이터를 전략적으로 투입해야 한다.

  • 언어 혼합 (Code-switching): 영어로 질문하다가 갑자기 접속사만 한국어로 바꾸는 등의 입력.
  • 포맷 교란: 줄바꿈(\n)이 의도적으로 누락되었거나, 이중 탭(\t\t)이 섞여 있는 입력 지문.
  • 불안정한 지시문 (Adversarial Prompt): 시스템 지시문을 거스르게 유도하는 트릭이 숨겨진 사용자 문장.

이와 같은 코너 케이스들을 test_corner_cases.json 에 분리 수집하고, 정규식 기반의 오라클이 이런 변칙 공격 속에서도 FAIL 이나 UNKNOWN 등의 폴백(Fallback) 라우팅을 정상 수행하는지 관측하는 것이 테스트 데이터 관리의 핵심이다.
