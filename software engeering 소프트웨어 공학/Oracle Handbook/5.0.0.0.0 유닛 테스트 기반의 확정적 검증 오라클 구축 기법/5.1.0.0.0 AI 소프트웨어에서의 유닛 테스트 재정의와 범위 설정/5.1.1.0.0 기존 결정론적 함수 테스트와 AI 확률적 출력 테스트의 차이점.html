5.1.1 기존 결정론적 함수 테스트와 AI 확률적 출력 테스트의 차이점

전통적인 유닛 테스트는 함수의 기계적 투명성(Mechanical Transparency)에 의존한다.
calculateDiscount(1000) 함수는 백만 번을 실행해도 정확히 900을 반환해야만 CI 버퍼를 통과(PASS)할 수 있다.

반면, summarizeText("...문서...") 라는 AI 함수를 전통적 assertEquals 방식으로 단위 테스트하려 들면, 문자열이 “이 문서는…” 에서 “해당 문서는…“으로 단 한 글자만 바뀌어도 즉각적으로 단위 테스트가 깨져버리는(Break) 대참사가 발생한다. 이를 취약성(Brittleness) 이라 부른다.

AI 소프트웨어의 확률적 출력을 다룰 때, 우리의 오라클은 ’절대적인 형태(Exact Form)’를 검증하는 사냥꾼에서 ’비즈니스 룰을 지켰는가(Adherence to Rules)’를 판독하는 논리적 심판관으로 진화해야만 한다. 즉, 값이 어떤 형태를 띠든 “의도된 상태 공간(State Space)” 내에 머물러 있다면 PASS를 던질 수 있는 관용적이지만 엄격한 조건식 작성이 AI 유닛 테스트의 첫걸음이다.
