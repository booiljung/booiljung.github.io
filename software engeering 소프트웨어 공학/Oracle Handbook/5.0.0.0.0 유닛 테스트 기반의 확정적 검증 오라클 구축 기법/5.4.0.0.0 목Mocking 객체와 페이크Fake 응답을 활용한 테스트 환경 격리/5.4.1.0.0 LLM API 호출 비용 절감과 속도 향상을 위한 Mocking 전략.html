5.4.1 LLM API 호출 비용 절감과 속도 향상을 위한 Mocking 전략

유닛 테스트(Unit Test)의 정의는 ’네트워크와 DB 등 외부 의존성이 100% 끊어진 빠르고 독립적인 검증’이다.
AI 기반의 오라클 시스템이라도 이 원칙에서 예외가 될 수는 없다.

프롬프트 빌더(Builder)와 응답 파서(Parser)의 로직 결함을 잡기 위한 일일(Daily) 수백 번의 코드 커밋 과정에서 매번 진짜 OpenAI 클라이언트에 HTTPS 요청을 날리면 회사의 통장은 텅 비게 되며 개발자의 집중력은 흩어진다.

Python의 unittest.mock.patch나 Jest의 jest.mock 등을 이용해 API 호출 어댑터 함수(예: call_gpt())를 최상단에서 완전히 낚아채어 가로막는다.
진짜 모델이 생각하는 시간 없이, 대신 하드코딩된 ’미리 정의된 테스트용 응답(Fake Text)’을 즉각 반환하도록 덮어 씌움으로써, 로직상 존재하는 95%의 구문/파싱 오류를 비용 0원으로 검수해 낸다.
