5.4.4 API 에러, 지연(Latency), 토큰 초과 등 예외 상황 시뮬레이션 테스트

결정론적 오라클은 텍스트의 파싱 결과뿐만 아니라, 인프라 단계의 장애(Fault Tolerance) 도 결정론적으로 통제하고 재시도(Retry)할 수 있어야 한다.

  • 모델 API 레이어를 철저히 모킹(Mocking)하여, 임의로 HTTP 상태 코드 429 (Too Many Requests) 나 503 (Service Unavailable) 에러를 발생시킨다. 이때 우리의 코드베이스가 허둥대지 않고 Exponential Backoff 알고리즘을 태워 n초 뒤 우아하게 재시도를 날리는지 단위 테스트한다.
  • 강제 타임아웃 지연 시뮬레이션 레이턴시를 60초 이상 발생시키고 시스템이 TimeoutException을 내며 UNKNOWN 처리나 Fallback 로직으로 격하되는지 관측한다.
  • Context_Length_Exceeded 모킹 에러를 발생시켜 오라클 시스템이 긴 로그 데이터를 절반으로 자르거나 요약 압축하여 재송신하는 등의 방어 코드를 수행하는지 확인한다.

NFR(망 절단, 지연, 비용 한도 초과 등)이 몰아치는 전쟁통 속에서도, 우리의 테스트 오라클이 절대 죽지 않고(Always-Alive) 기계적인 리포트를 생산해 냄을 증명하는 궁극의 테스트 과정이다.
