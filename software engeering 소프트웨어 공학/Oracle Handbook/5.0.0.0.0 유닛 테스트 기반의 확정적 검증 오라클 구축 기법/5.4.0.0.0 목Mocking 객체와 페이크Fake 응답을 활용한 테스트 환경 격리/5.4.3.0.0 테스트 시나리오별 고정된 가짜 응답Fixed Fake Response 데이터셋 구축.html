5.4.3 테스트 시나리오별 고정된 가짜 응답(Fixed Fake Response) 데이터셋 구축

로컬 파서가 제대로 훈련받기 위해서는 오직 ’아름답게 떨어지는 성공 응답’만 던져주어선 안 된다. 오히려 모델이 미쳐 날뛰는(Hallucinate) 상황을 강제 모의관측(Simulate)해야 한다.

이를 위해 테스트 코드 내에 고정된 가짜 응답 데이터셋(Fake Response Fixtures) 을 시나리오별로 세팅한다.

  • fixture_success_json: 완벽한 마크다운 블록과 JSON 키를 가진 이상적인 덱(Deck).
  • fixture_missing_brace: 마지막 괄호 } 하나가 고의로 잘려나간 깨진 JSON 덱 (파서의 Exception 복구 능력을 검증).
  • fixture_chatty_AI: “물론입니다! 이 코드를 분석해 보았습니다. 결론은 아래와 같습니다…” 등 불필요한 서술형 인트로가 섞인 덱 (파서 정규식이 본문만 깔끔히 추출해 내는지 검증).

이렇게 AI가 저지를 수 있는 악의적 행동 양식을 수동으로 박제해 조립해 두면, 실제 모델의 컨디션 여부와 관계없이 오라클 검사 파이프라인의 내구도를 완벽히 증명할 수 있다.
