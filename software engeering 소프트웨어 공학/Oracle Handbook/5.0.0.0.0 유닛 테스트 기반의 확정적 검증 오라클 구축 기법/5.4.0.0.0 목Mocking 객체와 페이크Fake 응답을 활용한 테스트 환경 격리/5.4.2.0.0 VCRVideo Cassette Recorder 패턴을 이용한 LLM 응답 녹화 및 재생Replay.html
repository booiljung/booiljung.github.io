5.4.2 VCR(Video Cassette Recorder) 패턴을 이용한 LLM 응답 녹화 및 재생(Replay)

일일이 Fake 응답 객체를 만들기란 번거롭다. 테스트 엔지니어링의 마법 중 하나인 VCR 패턴을 AI에 도입하면 이 고통에서 해방될 수 있다.

Ruby의 VCR에서 시작되어 Python의 vcrpy, JavaScript의 nock 등으로 구현된 이 매커니즘은 마치 카세트테이프처럼 작동한다.

  1. 처음 실행 시 (Record 모드): 설정된 실제 엔드포인트(Real LLM API)에 붙어 요청과 응답을 수행하고, 그 교환된 HTTP 페이로드(Payload)를 JSON 형태의 .yaml이나 .json 파일 단위의 카세트(Cassette)로 하드디스크에 저장한다.
  2. 이후 실행 시 (Replay 모드): 두 번째 테스트 런타임부터는, 네트워크 바깥으로 나가지 않고 로컬에 저장된 카세트 파일 데이터를 읽어서 프록시 라이브러리가 대신 1초 만에 응답을 서빙한다.

이 방식을 이용하면 실제 LLM이 내뱉는 날것의 마크다운 포맷이나 구조적 결함을 100% 캡처하여 오프라인 환경에서도 무한하게 재련하고 파서 로직을 정밀 타격할 수 있다.
