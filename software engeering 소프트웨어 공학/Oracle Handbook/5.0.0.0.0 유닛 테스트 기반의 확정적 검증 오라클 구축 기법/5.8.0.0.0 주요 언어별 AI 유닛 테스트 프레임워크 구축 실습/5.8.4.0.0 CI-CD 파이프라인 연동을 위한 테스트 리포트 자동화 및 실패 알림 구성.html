5.8.4 CI/CD 파이프라인 연동을 위한 테스트 리포트 자동화 및 실패 알림 구성

로컬에서 혼자 돌려보는 오라클은 무의미하다. 진정한 ’가드레일(Guardrail)’은 GitHub Actions 든 GitLab CI 든 팀의 메인 브랜치 병합 전에 물리적으로 작동해야 한다.

  1. 빌드 블로커 (Build Blocker): Pull Request 이벤트 발생 시, pytest의 메타모픽/속성 기반 테스트 스위트가 실행된다. 실패(FAIL)할 경우 리뷰어의 승인이 있어도 Merge 버튼이 비활성화된다.
  2. 테스트 리포트 생성 (Allure, JUnit XML): AI의 응답 텍스트가 깨져서 파싱 실패가 났을 경우, CI 콘솔에 “Expected: JSON 형태, Actual: 마크다운이 누락된 텍스트“라는 명백한 Diff를 리포트에 출력한다.
  3. 슬랙(Slack) 알림: “프롬프트 v2.1 업데이트로 인해 요약 오라클 테스트 15개가 깨졌습니다.” 라는 알림이 즉각 채널에 발송되며, 파라미터 회귀(Regression)를 팀 전체에 고발한다.
