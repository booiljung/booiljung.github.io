5.8.2 JavaScript/TypeScript 환경: Jest/Vitest를 이용한 비동기 체인 테스트

Vercel의 AI SDK나 LangChain.js를 다루는 구역이다. JS 생태계에서는 Jest 또는 더 빠르고 최신인 Vitest가 주도권을 쥔다. Node.js 특성상 모든 LLM 호출은 철저한 비동기 체인(Async/Await)을 갖는다.

  • nock 라이브러리 활용: OpenAI 서버(api.openai.com)로 날아가는 HTTP 패킷을 중간에서 완전히 차단(Intercept)하고 하드코딩된 더미 JSON을 던져준다.
import nock from 'nock';
import { summarizeEmail } from './aiService';

test('이메일 요약 파이프라인에서 500 에러 발생 시 Fallback 체인 작동 여부', async () => {
    // OpenAI API에 장애가 난 상황을 모방 (Mocking)
    nock('https://api.openai.com')
        .post('/v1/chat/completions')
        .reply(500, { error: 'Internal Server Error' });

    const result = await summarizeEmail("긴 이메일 내용...");
    // 오라클: 시스템이 뻗지 않고 기본 안내문을 반환하는가?
    expect(result.summary).toBe("현재 요약 서버에 장애가 있습니다. 다시 시도해주세요.");
});

이러한 모킹과 expect() 체이닝을 통해 V8 엔진 위에서 초당 수천 개의 프롬프트 경계조건(Boundary Condition)을 검수할 수 있다.
