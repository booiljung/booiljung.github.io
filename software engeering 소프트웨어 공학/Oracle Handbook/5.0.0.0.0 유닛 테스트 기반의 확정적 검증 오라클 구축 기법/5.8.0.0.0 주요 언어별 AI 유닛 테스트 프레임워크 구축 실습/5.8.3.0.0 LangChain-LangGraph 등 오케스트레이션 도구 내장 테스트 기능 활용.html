5.8.3 LangChain/LangGraph 등 오케스트레이션 도구 내장 테스트 기능 활용

복잡한 RAG(Retrieval-Augmented Generation) 봇이나 다중 에이전트를 LangChain/LangGraph로 얽어 놓으면, 단일 함수의 단위 테스트가 사실상 불가능해진다. 노드 간에 텐서(Tensor)처럼 돌아다니는 상태(State)를 디버깅해야 하기 때문이다.

이때는 프레임워크가 제공하는 트레이서(Tracer) 와 가짜(Fake) LLM 클래스를 오라클로 편입시킨다.

  • FakeListLLM: 랭체인에서 기본 제공하는 클래스로 정해진 문자열 리스트를 순차적으로 뱉어낸다. 에이전트 노드가 이 가짜 답변을 받고 다음 노드로 정상 라우팅(Routing) 되는지 컴포넌트 간 연결망만을 테스트한다.
  • InMemoryStore: 값비싼 Pinecone, Milvus 등 벡터 DB 대신 메모리 기반의 유닛 테스트용 단발성 벡터 검색기를 주입하여, “검색 과정” 자체의 정합성을 측정한다.
