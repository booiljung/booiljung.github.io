5.9.3 감성 분석기: 특정 단어 리스트 기반의 긍정/부정 분류 정확도 검증 매커니즘

문장이 ’긍정적(Positive)’인지 판별하는 기능은 주관성이 매우 개입된다. 이를 단위 테스트로 잡기 위해서는 메타모픽 속성과 확률적 관용을 결합한 오라클이 필요하다.

오라클 테스트 로직:

  1. 노골적인 긍정어 풀(Pool)과 부정어 풀을 선언한다. (예: ["환상적이다", "훌륭하다"], ["끔찍하다", "최악이다"])
  2. 테스트 스크립트가 해당 풀에서 단어를 무작위로 뽑아(Fuzzing) “이 서비스는 정말 [단어] 하네요.“라는 문장 100개를 만들어 AI에게 차례로 던진다.
  3. 무조건 JSON {"sentiment": "POSITIVE" | "NEGATIVE"} 포맷으로 떨어지도록 설정한 후, 긍정어 풀에서 조합된 문장은 POSITIVE로 일관되게 뱉는지 판독한다. 단 하나의 오차라도 발견되면 프롬프트의 톤 파악 능력이 회귀(Regress)한 것으로 간주하여 빌드를 실패 처리한다.
