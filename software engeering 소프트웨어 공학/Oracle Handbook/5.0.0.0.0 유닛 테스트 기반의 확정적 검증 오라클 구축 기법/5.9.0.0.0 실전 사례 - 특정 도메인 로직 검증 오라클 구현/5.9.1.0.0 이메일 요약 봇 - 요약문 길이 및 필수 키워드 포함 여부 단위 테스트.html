5.9.1 이메일 요약 봇: 요약문 길이 및 필수 키워드 포함 여부 단위 테스트

사용자의 긴 불만 접수 이메일을 CS 팀을 위해 3줄로 요약하는 AI 봇이 있다고 가정하자.

오라클 테스트 로직:

  1. LLM API를 호출하여 요약문을 받아온다 (개발 환경에선 모킹(Mocking) 사용 권장).
  2. 요약문 문자열을 파싱한다.
  • NFR 검증: assert len(summary.split('\n')) <= 3 (정확히 3줄 이하인가?)
  • Inclusion 검증: assert any(keyword in summary for keyword in ["환불", "불만", "취소"]) (원문에 있던 핵심 비즈니스 명사가 누락되지 않았는가?)

단어와 구두점이 어떻게 바뀌든, 길이를 지키고 핵심 단어를 보존했다면 이 테스트는 PASS를 반환하는 관용적인 오라클이다.
