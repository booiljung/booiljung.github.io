5.9.2 코드 생성기: 생성된 코드의 구문 오류(Syntax Error) 및 컴파일 가능성 검증

AI가 개발자의 코드를 리뷰하거나 대신 짜주는 툴(Copilot)을 만들 때, AI가 내뱉은 텍스트가 ‘실제로 동작하는 코드인지’ 판단하는 유일한 오라클은 파라미터가 아니라 외부 컴파일러다.

오라클 테스트 로직:

  1. AI가 반환한 마크다운 블록에서 pyscript_code = re.search(r'```python(.*?)```', response, re.DOTALL).group(1) 정규식으로 순수 Python 코드만 덜어낸다.
  2. 파이썬 내장 모듈인 ast (Abstract Syntax Tree)를 사용하여 파싱을 시도한다.
import ast
try:
ast.parse(pyscript_code)
except SyntaxError:
pytest.fail("AI가 문법이 깨진 파이썬 코드를 생성했습니다.")

이 테스트는 로직이 맞는지(x=1)를 떠나, 괄호가 누락되거나 들여쓰기가 망가진 치명적인 Syntax 결함을 즉시 색출해 낸다.
