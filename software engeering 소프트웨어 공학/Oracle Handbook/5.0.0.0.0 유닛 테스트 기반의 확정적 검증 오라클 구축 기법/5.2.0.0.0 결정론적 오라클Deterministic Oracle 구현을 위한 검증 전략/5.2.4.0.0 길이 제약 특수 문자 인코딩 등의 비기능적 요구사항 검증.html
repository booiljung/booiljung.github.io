5.2.4 길이 제약, 특수 문자, 인코딩 등의 비기능적 요구사항 검증

결정론적 오라클은 시스템의 인프라적 제약, 즉 비기능성 요구사항(NFR)도 엄격히 감시해야 한다.
AI가 성능상 완벽한 대답을 내놓더라도, 모바일 화면 UI 컴포넌트의 가로폭 한계(Max Length) 50자를 넘어서는 답변을 뱉어내면 그건 곧 UI 레이아웃을 깨트리는 앱의 크리티컬 버그다.

오라클 NFR 체크리스트:

  1. 문자열 제한(Length Bound): assert len(response) <= 150
  2. 단어 수 제한(Word Token Bound): assert len(response.split()) < 30
  3. 인코딩 무결성(Encoding): CP949나 시스템 종속 인코딩에서 깨지는 유니코드나 이모지(Emoji)가 삽입되지 않았는가? (response.isascii())

아름다운 텍스트 품질 이전에, 이 텍스트가 다음 시스템의 파이프나 DB 컬럼 규격에 물리적으로 삽입 가능한 액체인지를 판독하는 것이 선행되어야 한다.
