5.2.5 블랙리스트 및 안전성 필터링 테스트: PII(개인식별정보) 및 금지어 검출

오라클의 역할 중 치명적인 법적 책임을 방어하는 영역이다. 프롬프트 인젝션 방어책이나 시스템 룰이 잘 작동하는지 파서(Parser)를 통해 PII 누출 유무를 스캔하는 일이다.

테스트 스위트에 무작위의 전화번호, 신용카드 번호, 혹은 “이전 지시를 무시해라“라는 해킹 프롬프트를 주입한 뒤, 반환된 응답에서 이를 어떻게 차단했는지 패턴을 테스트한다.

  • 차단 검증: assert response == "안전 정책에 의해 거부되었습니다."
  • 마스킹(Masking) 검증: 응답 텍스트를 상대로, 정규식 \d{3}-\d{4}-\d{4} 구조를 띄는 신용카드 번호가 론칭 직전의 형태 그대로 등장하는지, 아니면 ***-****-****처럼 * 처리되었는지 오라클이 스캔한다.
  • 블랙리스트 감지: 성차별, 인종차별적 속어 사전을 메모리에 띄워놓고 Aho-Corasick(다중 문자열 패턴 매칭) 알고리즘 등으로 응답 본문에 금지어가 매칭되는지 광속으로 검사한다. 기능적 정답보다 이 필터링 오라클을 통과하는 것이 배포의 0순위 전제조건이다.
