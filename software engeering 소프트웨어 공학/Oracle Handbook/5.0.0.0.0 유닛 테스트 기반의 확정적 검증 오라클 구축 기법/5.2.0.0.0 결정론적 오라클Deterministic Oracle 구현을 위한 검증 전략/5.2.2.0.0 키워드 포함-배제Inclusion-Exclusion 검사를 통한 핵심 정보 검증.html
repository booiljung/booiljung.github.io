5.2.2 키워드 포함/배제(Inclusion/Exclusion) 검사를 통한 핵심 정보 검증

구조화되지 않은 내러티브 형태의 텍스트(예: 요약기능, 리뷰 생성)를 검사할 때는 정규식조차 짜기 어렵다. 이때 동원되는 것이 논리 게이트 기반의 집합 연산(Set Operations) 이다.

오라클은 해당 응답이 가져야 할 필수 키워드 배열(Inclusion) 과, 절대 나와서는 안될 금지 키워드 배열(Exclusion) 을 대조한다.

def test_summary_oracle(response_text):
    must_includes = {"핵심", "요약", "매출"}
    must_excludes = {"추측하건대", "아마도", "불확실"}
    
    assert all(k in response_text for k in must_includes), "필수 키워드 누락"
    assert not any(k in response_text for k in must_excludes), "망상형 금지어 감지"

이 테스트는 AI가 소설을 쓰든 딱딱하게 답하든 상관없이, 도메인 비즈니스적 뼈대(Entity)가 살아있는지, 혹은 치명적인 법적 위반 단어(금지어)가 없는지 즉각적인 방화벽 역할을 수행한다.
