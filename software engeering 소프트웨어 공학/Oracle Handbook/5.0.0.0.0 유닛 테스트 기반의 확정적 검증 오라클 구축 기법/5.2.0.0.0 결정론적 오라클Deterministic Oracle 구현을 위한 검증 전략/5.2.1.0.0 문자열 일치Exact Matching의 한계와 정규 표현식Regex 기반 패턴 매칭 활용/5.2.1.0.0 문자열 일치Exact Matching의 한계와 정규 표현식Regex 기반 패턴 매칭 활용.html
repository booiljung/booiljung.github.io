5.2.1 문자열 일치(Exact Matching)의 한계와 정규 표현식(Regex) 기반 패턴 매칭 활용

LLM의 산출물을 상대로 고전적인 구문 assert expected == actual 을 시도하는 것은 자살 행위와 같다.
AI는 동일한 의미를 전달할 때 “성공!”, “네, 성공입니다.”, “결과: 성공” 등 무한한 문자열 공간을 유영한다.

따라서 오라클 검증의 뼈대는 Exact Matching이 아닌 정규 표현식(Regex) 기반의 패턴 매칭으로 전면 교체해야 한다.
예컨대 런타임 로그에서 에러 코드를 뽑아내는 테스트라면, 단순히 “Error 404가 포함되어 있는가?“가 아니라 정규식을 통해 r"Error\s*(?:Code)?\s*:\s*(404|500)" 패턴에 매칭되는지 질의해야만 주변 수식어에 흔들리지 않는 굳건한 오라클을 세울 수 있다.
