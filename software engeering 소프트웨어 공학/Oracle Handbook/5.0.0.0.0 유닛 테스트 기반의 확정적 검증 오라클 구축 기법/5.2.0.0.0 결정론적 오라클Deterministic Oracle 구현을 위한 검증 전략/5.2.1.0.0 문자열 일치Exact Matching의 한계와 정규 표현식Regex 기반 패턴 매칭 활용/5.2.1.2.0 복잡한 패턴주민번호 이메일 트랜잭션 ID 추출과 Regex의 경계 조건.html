5.2.1.2 복잡한 패턴(주민번호, 이메일, 트랜잭션 ID) 추출과 Regex의 경계 조건

결정론적 오라클이 정규 표현식을 사용할 때 가장 잦은 결함이 일어나는 지점은 ’경계 조건(Boundary Condition)’이다.

예를 들어 주민등록번호를 추출하는 \d{6}-\d{7}이라는 정규식을 돌렸다고 가정하자. 자연어 텍스트 특성상 모델이 실수로 “123456-12345678”(뒤가 8자리)을 뱉었음에도, 위 정규식은 앞의 7자리만 매칭되어 통과(PASS) 판정을 내리는 ‘거짓 양성(False Positive)’ 오류를 범하게 된다.

오라클을 위한 경계 통제
추출 검증 시 반드시 단어 경계(\b)를 설정하라.

  • ❌ 취약한 오라클: assert re.search(r'\d{6}-\d{7}', response)
  • ✅ 안전한 오라클: assert re.search(r'\b\d{6}-\d{7}\b', response)

오라클의 Regex는 타겟팅한 문자 이외에는 앞뒤로 어떤 부속물도 더 붙어있지 않음을 단호하게 증명해야 한다.
