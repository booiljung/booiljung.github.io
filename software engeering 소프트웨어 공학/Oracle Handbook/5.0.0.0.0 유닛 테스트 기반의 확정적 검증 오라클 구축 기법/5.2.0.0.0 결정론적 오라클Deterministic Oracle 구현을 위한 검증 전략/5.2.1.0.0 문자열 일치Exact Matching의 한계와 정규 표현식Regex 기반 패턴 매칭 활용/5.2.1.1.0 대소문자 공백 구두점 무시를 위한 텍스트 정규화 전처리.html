5.2.1.1 대소문자, 공백, 구두점 무시를 위한 텍스트 정규화 전처리

정규식을 돌리기 직전, 먼저 거쳐야 할 관문이 텍스트의 정규화(Normalization)다.
AI는 마침표(.)나 개행문자(\n)를 무의식적으로 덧붙이거나 대소문자를 섞는 경향이 짙다.

이를 오라클 파이프라인에서 처리하는 ’전처리 통과 3원칙’은 다음과 같다.

  1. 대소문자 평탄화: response.lower() 를 통해 문자열 통일.
  2. 화이트스페이스 정제: re.sub(r'\s+', ' ', response).strip() 으로 다중 공백과 탭 문자를 단일 공백으로 치환.
  3. 구두점 필터링: re.sub(r'[^\w\s]', '', response)를 통해 느낌표나 콤마 등을 제거(비즈니스 룰상 허용될 때만).

이 한 줄의 전처리 훅(Hook)을 Assertion 이전에 주입하는 것만으로 단위 테스트의 억지 실패(Flaky Fail) 비율을 80% 이상 감소시킬 수 있다.
