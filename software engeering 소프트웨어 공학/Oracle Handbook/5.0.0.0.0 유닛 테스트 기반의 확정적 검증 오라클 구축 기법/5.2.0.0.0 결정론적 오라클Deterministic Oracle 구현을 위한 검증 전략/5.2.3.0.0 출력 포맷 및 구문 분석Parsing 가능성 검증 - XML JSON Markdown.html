5.2.3 출력 포맷 및 구문 분석(Parsing) 가능성 검증: XML, JSON, Markdown

현대 엔터프라이즈 AI 테스트의 성패는 “LLM이 파싱 가능한 머신 리더블(Machine-readable) 객체를 반환했는가“에 달려있다.
가장 직관적이고 강력한 유닛 테스트는 파서(Parser)의 try-catch 구문을 곧바로 오라클로 사용하는 것이다.

def test_json_structure_oracle(ai_response):
    try:
        # 1. 포맷 검증: 완벽한 JSON으로 역직렬화(Deserialize) 가능한지 판독
        data = json.loads(ai_response)
        
        # 2. 스키마 검증: 필수 키가 존재하는지 판독
        assert "status" in data
        assert "confidence_score" in data
        assert isinstance(data["confidence_score"], float)
        
    except json.JSONDecodeError:
        pytest.fail("AI가 결함 있는 JSON 구문을 생성했습니다.")

Markdown 블록이나 XML 구조에서도 동일하게 작동한다. 이 검증 기법은 자연어의 애매함을 배제하고, AI를 일종의 불안정한 ’구조체 송신 API’로 대우함으로써 단위 테스트의 견고함을 얻는다.
