5.3.4 테스트 가능한 프롬프트 템플릿 구조와 버전 관리 전략

프롬프트를 파이썬 코드 안에 f""" ... """ 형태로 하드코딩하면, 로직(Logic)과 뷰(View)가 결합되어 단위 테스트가 최악으로 까다로워진다.

모든 프롬프트는 .py 파일의 밖으로 빼내어 prompt_v1.2.txt 혹은 sql_oracle.yaml 같은 독립된 에셋(Asset) 파일로 구조를 분리(Decoupling) 해야 한다.

  • 테스트 용이성 확보: 테스트 스크립트가 로직을 건드리지 않고, 파일 시스템에서 오직 prompt_vX 파일만을 메모리로 로드해 정규식 검사나 린트(Lint)를 수행할 수 있게 된다.
  • A/B 롤아웃과 회귀 테스트 보장: 신규 프롬프트 패치(v2)가 생성되면, CI/CD 스위트는 기존 버전을 보존한 채 새 버전을 섀도우 컨테이너에 주입하여 테스트 매트릭스를 돌린다. 여기서 구 버전보다 결과 일관성이 밀린다면 즉시 배포를 롤백할 수 있는 기반 암이 된다.
