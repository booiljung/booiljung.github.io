5.3.2 동적 변수 주입(Variable Injection) 시나리오별 경계값 분석(Boundary Value Analysis)

시스템 프롬프트가 정적 텍스트라면 에러가 나지 않는다. 버그의 99%는 런타임에 동적으로 주입되는 변수(예: {{ user_code }}, {{ today_date }}) 공간에서 발생한다.

파서와 템플릿 엔진(예: Jinja2)이 결합된 프롬프트를 유닛 테스트할 때는 AI 모델 호출을 배제하고 오직 “완성된 텍스트 렌더링 결과물“만을 상대로 경계값 분석(BVA) 을 수행해야 한다.

  • 변수 결측값(Null/Empty): {{ user_log }} 가 공백이거나 None으로 주입되었을 때, 프롬프트 인젝션 방어 장치가 작동하지 않거나 포맷이 깨지지 않는지 프롬프트 생성 빌더(Builder) 레이어를 검증한다.
  • 초과 길이(Max Token Exceed): 사용자가 고의로 1만 줄짜리 텍스트를 주입했을 때, 템플릿 엔진 렌더러가 문자열 길이를 잘라(Truncate) 프롬프트 윈도우 에러(HTTP 400 Bad Request)를 사전에 방지하는지 어써션한다.

프롬프트를 보내기 전에 문자열 빌드 단계에서부터 펜스(Fence)를 치는 것이 가장 경제적인 버그 예방책이다.
