5.6.3 수치 계산 및 데이터 변환 로직의 외부 코드 실행(Code Execution) 검증

LLM의 절대적인 약점을 꼽으라면 정밀한 수학 연산이다.
수만 개의 임베딩 벡터 공간을 곱하고 더하는 트랜스포머 아키텍처 특성상, “12345 * 6789 = ?“와 같은 확정적 산술 연산을 물어보면 그럴싸하게 틀린 답(답변이 틀린 Halucination)을 내뱉기 일쑤다.

따라서 AI 오라클은 수치 연산을 절대 모델에게 위임해선 안 된다.
대신 플러그인 연동 및 함수 호출(Function Calling) 기법을 사용해 파이프라인 외부로 연산을 분리해야 한다.

  • AI의 역할: 식을 세우고, 데이터를 파싱해 매개변수(Params) 딕셔너리를 만든다.
  • 오라클 유닛 검수 로직: AI가 내뱉은 인자(Arguments)를 바탕으로 파이썬 내부의 eval() 이나 sandbox_calc()를 돌린다.
  • 테스트 프레임워크는 AI가 만들어낸 연산식 스키마가 외부 컴파일러를 터뜨리지 않고 올바른 타입(Integer/Float)으로 도출되었는지만 기계적으로 Assert 한다.
