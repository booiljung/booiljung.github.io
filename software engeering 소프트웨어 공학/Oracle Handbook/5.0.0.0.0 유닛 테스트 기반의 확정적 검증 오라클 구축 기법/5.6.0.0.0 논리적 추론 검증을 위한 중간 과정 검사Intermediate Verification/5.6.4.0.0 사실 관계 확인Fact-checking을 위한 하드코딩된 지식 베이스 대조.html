5.6.4 사실 관계 확인(Fact-checking)을 위한 하드코딩된 지식 베이스 대조

파라메트릭 인지 공간(가중치 속에 구워진 지식)은 과거 모델 트레이닝 시점의 편향된 찌꺼기를 안고 있다.
“테스트 환경의 기본 포트를 말하라“고 했을 때, 모델 혼자 과거 지식으로 8080을 내뿜고 이게 오라클 최종 판정을 엎어버릴 수 있다.

AI 오라클의 신뢰도는 최종 답변을 도출하기 위한 근거(Grounding Document) 와 대조하여 증명해야 한다.

  • 유닛 테스트 환경에 사내 마이크로서비스 포트 정보가 담긴 config_mock.json 과 같은 작은 데이터베이스(Knowledge Base)를 물려 둔다.
  • 모델의 추론 thought 내역에서 추출된 포트 번호 값이, 이 config_mock.json 원본 데이터 풀 내에 정확히 위치(Exist) 하는지 판독한다.
    이는 RAG(Retrieval-Augmented Generation) 시스템의 답변 신뢰도(Faithfulness)를 검증하는 것과 똑같은 이치다. 존재하지 않는 사실을 허구적으로 지어냈는지를 외부 데이터의 하드코어한 조회(Lookup)로 심판한다.
