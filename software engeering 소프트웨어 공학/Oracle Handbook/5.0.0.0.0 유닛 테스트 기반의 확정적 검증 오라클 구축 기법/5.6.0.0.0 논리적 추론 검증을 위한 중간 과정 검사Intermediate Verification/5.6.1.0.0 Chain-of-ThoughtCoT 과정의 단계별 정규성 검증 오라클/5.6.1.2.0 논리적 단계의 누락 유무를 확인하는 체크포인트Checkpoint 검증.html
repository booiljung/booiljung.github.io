5.6.1.2 논리적 단계의 누락 유무를 확인하는 체크포인트(Checkpoint) 검증

테스트 명세에 “A 확인, B 확인 후 결론 도출“이라고 정의했다면, AI의 산출물에도 이 A와 B를 처리했다는 명백한 체크포인트 마크가 관측되어야 한다.

CoT 템플릿에 다음과 같이 스키마를 강제해 두었다고 가정하자.
"Step 1: DB 연결 분석", "Step 2: 타임아웃 분석", "최종: 결론"

유닛 테스트 오라클은 응답 문자열을 파싱할 때 Step 1과 Step 2라는 리터럴 접두사(Literal Prefix)가 순서에 맞게 정확히 출현했는지를 카운팅하여 검증해야 한다. 만약 Step 2를 건너뛰고 바로 최종이 출력되었다면 이는 추론 과정의 누락이므로, 결괏값의 정합성과 관계없이 즉각 프로세스를 중단(Fail-fast) 시켜야 한다.
