5.11.1 LangSmith, Phoenix 등을 활용한 프롬프트 체인 추적

기존의 print() 문이나 파일 로깅으로는 체인 구조로 얽힌 에이전트 다이내믹스를 추적할 수 없다.
LangSmith나 Arize Phoenix와 같은 LLM 가시성 솔루션은 네트워크 페이로드를 가로채(Intercept) 다음과 같은 추적(Trace) 트리를 웹 UI에 제공한다.

  1. 시간 지연 추적: 총 10초의 응답 시간 중 검색(Vector DB)에 2초, LLM 1차 요약에 5초, 오라클 검수 로직이 3초 걸렸음을 병목(Bottleneck) 그래프로 보여준다.
  2. 입출력 페이로드 보존: 1차 실패 시 재시도(Retry)된 두 번째 프롬프트에 정확히 무슨 파라미터(Temperature, Logit_bias)가 얹어져서 전송되었는지 날것의 JSON 요청을 남김없이 보존한다.
  3. 이 트레이스 ID 시스템과 테스트 프레임워크를 연동시켜라. FAIL 난 테스트는 assert 로그와 함께 즉시 LangSmith의 해당 트레이스 URL 링크를 리포트 하단에 꽂아 넣어, 엔지니어가 단건 클릭만으로 문제의 블랙박스 심장부로 파고들 수 있게 해야 한다.
