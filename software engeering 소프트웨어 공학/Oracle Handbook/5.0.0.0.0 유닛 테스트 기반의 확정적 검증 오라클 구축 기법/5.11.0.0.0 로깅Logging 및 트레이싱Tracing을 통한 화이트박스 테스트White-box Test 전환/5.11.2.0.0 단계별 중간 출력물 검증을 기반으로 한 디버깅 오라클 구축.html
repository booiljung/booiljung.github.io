5.11.2 단계별 중간 출력물 검증을 기반으로 한 디버깅 오라클 구축

화이트박스 오라클은 “입력과 최종 출력“의 끝단만 비교하지 않는다. 체인 마디마다 검문소를 두는 것이 디버깅의 정수다.

  • 노드 A: 문서 검색
  • 노드 B: 임시 매핑(Mapping)
  • 노드 C: 최종 요약(Summarize)

트레이서 시스템과 연동된 오라클은 파이프라인 전체가 끝날 때까지 기다리지 않는다.
만약 노드 A의 중간 검색 리스트 개수가 0개였다면, 노드 B와 C의 LLM 연산에 돈을 태우지 않고 그 자리에서 AssertionError: Context Missing을 일으키며 파이프라인을 격추(Fail-Fast) 시킨다.
이렇게 체인(Chain)의 모든 허리춤에 단위 조건절 오라클을 심어두면, 밤새 원인을 찾지 못해 헤매던 프롬프트 디버깅 시간이 기계적 오류 스냅샷 덕분에 ‘분’ 단위로 단축될 수 있다.
