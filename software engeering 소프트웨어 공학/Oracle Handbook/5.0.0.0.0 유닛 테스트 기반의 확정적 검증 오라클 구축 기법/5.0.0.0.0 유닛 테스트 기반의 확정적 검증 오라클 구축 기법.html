Chapter 5. 유닛 테스트 기반의 확정적 검증 오라클 구축 기법

결정론적 소프트웨어 공학의 심장은 유닛 테스트(Unit Test)다. 그러나 AI가 도입되자 개발자들은 “말을 할 때마다 달라지는데 어떻게 테스트 코드를 짤 수 있느냐“며 회귀 검증(Regression Testing)을 포기하는 경향이 생겼다.

오라클 시스템 아키텍처에서 가장 위험한 발상이다. AI가 확률적으로 행동한다면, 인간이 그 확률을 가두고 통제하는 ‘확정적 검증 오라클(Deterministic Verification Oracle)’ 을 짜면 된다.

본 장에서는 AI 컴포넌트를 기존의 소프트웨어 개발 생명주기(SDLC)에 완벽하게 결합시키기 위해, CI/CD 파이프라인 위에서 수만 번 반복 실행되어도 깨지지 않는(Flaky하지 않은) 강건한 단위 테스트 프레임워크를 어떻게 설계하고 구현해야 하는지 그 최전선의 기법들을 다룬다.
