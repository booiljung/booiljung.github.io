1.3.2 확률적 AI 출력의 본질적 결함: 환각(Hallucination)과 창의성 사이의 모호한 경계

대규모 언어 모델(LLM)이 극찬을 받는 가장 큰 이유 중 하나는 마치 인간처럼 맥락을 파악하고 기발한 아이디어를 도출해 정교한 시(Poetry)나 소설을 창작할 수 있다는 점이다. 이러한 창의성(Creativity)은 온도(Temperature) 등의 샘플링 파라미터를 조절하여 모델이 덜 그럴듯한(low-probability) 토큰 조합을 결합하도록 유도함으로써 발생한다. 그러나 엔지니어링 관점에서 보았을 때, 이 위대한 ’창의성’은 통계적 조작이 만들어낸 ‘사실의 왜곡’, 즉 환각(Hallucination)과 수학적으로 완벽하게 동일한 기전을 지닌다.

1. 환각과 창의성의 동전 양면성

일반 사용자가 챗봇에게 “가상의 마법 세계관의 동물 이름 3개를 만들어줘“라고 지시했을 때 얻어지는 결과는 ’창의성’이다. 모델이 학습 데이터에 없는 새로운 단어의 파편들을 교묘하게 조합해 냈기 때문이다.

하지만 개발자가 시스템 내부에서 “주어진 로그 데이터를 바탕으로 시스템 다운타임(Downtime) 원인을 분석하라“라고 지시했을 때, 모델이 코퍼스(Corpus)에 있던 다른 회사의 장애 사례나 가상의 에러 코드를 짜깁기하여 매끄러운 보고서를 만들어낸다면 그것은 치명적인 ’환각’이 된다.

2. 모호한 경계가 불러오는 엔지니어링 딜레마

모델은 자신이 진실(Ground Truth)을 말하고 있는지, 아니면 유창하게 거짓말(Fluent Bullshit)을 지어내고 있는지 스스로 자각(Self-Awareness)하지 못한다. LLM은 오직 단어 간의 통계적 상관관계(Statistical Correlation) 최적화에만 몰두할 뿐이다.

개발자는 이 모호한 경계를 통제하기 위해 프롬프트 상단에 "너는 엄격한 데이터 분석가다. 없는 사실을 지어내지 마라(Do not hallucinate)."라는 강력한 제약(Constraint)을 주입하지만, 이는 결코 컴파일러(Compiler) 수준의 결정론적 구속력을 가지지 못한다. 결국 확률적 시스템은 코딩이나 데이터 변환과 같이 문법(Syntax)과 사실(Fact)이 100% 일치해야 하는 엔지니어링 태스크 안에서도 언제 어떻게 창의성을 발현(?)하여 코드를 오염시킬지 예측할 수 없는 잠재적 시한폭탄과 같다.
