1.5.2 사용자 경험(UX)의 일관성 저해: 같은 질문에 대해 매번 다른 어조와 포맷으로 응답

엔터프라이즈 벤더나 글로벌 B2C(Business-to-Consumer) 서비스가 제공하는 애플리케이션에서 브랜드 보이스(Brand Voice)와 사용자 경험(User Experience, UX)의 핵심은 일관성(Consistency)이다. 고객은 페이지를 새로고침하거나 동일한 기능을 두 번 호출했을 때 100% 예측 가능한 디자인과 동일한 정보의 배열을 기대한다. 그러나 인터페이스의 출력을 대규모 언어 모델(LLM)의 생성 텍스트에 직결시키는 순간, UX의 일관성은 철저히 파괴된다.

1. 어조와 포맷의 양극단

개발자가 프롬프트 상단에 시스템 메시지(System Message)로 당신은 친절한 회사의 AI 비서입니다.라고 지정한다 하더라도, 확률적 샘플링(Temperature, Top-p)에 의해 모델의 응답은 극과 극을 오간다.

  • 조회 1회 차: “안녕하세요! 문의하신 계좌의 남은 잔액은 5,000원입니다. 더 도와드릴 일이 있으신가요? 😊”
  • 조회 2회 차 (새로고침): “잔액: 5000원.”
  • 조회 3회 차 (시간 경과 후): “현재 잔액은 5,000 KRW입니다.”

이러한 어조(Tone)의 일관성 부재는 단순한 감정적 문제를 넘어 시스템 자체에 대한 신뢰도(Trust)를 하락시킨다. 사용자는 시스템이 어딘가 고장 났거나 해킹당했다는 의구심을 가질 수밖에 없다.

2. 구조화된 포맷팅의 붕괴

더욱 뼈아픈 타격은 프론트엔드 리액트(React)나 플러터(Flutter) 컴포넌트로 데이터를 바인딩(Binding)하기 위해 JSON 혹은 특정 마크다운 구조를 반환할 때 발생한다.

프론트엔드 개발자는 일반적으로 result.items와 같은 정형화된 배열 구조를 기대하고 DOM 컴포넌트를 랜더링한다. 그러나 AI가 “항목을 제공해 드릴게요“라는 미세한 서론(Introduction)을 텍스트 배열 맨 앞에 추가해버리면 컴포넌트 전체가 파싱 에러를 뱉으며 클라이언트 브라우저가 하얀 화면(White Screen of Death)으로 죽어버린다.

이처럼 AI 출력의 비결정성은 단순히 글이 달라지는 수준을 넘어서 프론트엔드 애플리케이션의 뼈대와 규칙을 무너뜨리며, 한 치의 오차도 허용되지 않는 모던 웹/앱 UX 환경에서 심각한 시각적, 경험적 위협으로 다가온다.
