1.5 비결정성이 비즈니스와 보안에 미치는 구체적 위협

인공지능의 비결정성(Nondeterminism)이 야기하는 문제들은 단순한 코드 작성 단계의 불편함이나 품질 검증 과정에서의 디버깅(Debugging) 지연에 국한되지 않는다. 일단 이러한 확률적 특성이 내재된 애플리케이션이 실제 운영 환경(Production Environment)에 배포되면, 시스템의 통제 불능성은 기업의 비즈니스 연속성과 법적 책임, 더 나아가 고객의 신뢰와 시스템 보안 전반에 걸쳐 심각하고 구체적인 위협(Threats)으로 발현된다.

본 장에서는 대규모 언어 모델(LLM) 기반의 기능들이 엔터프라이즈 환경에서 어떻게 취약점으로 작용하는지 면밀히 파악한다. 법적 규제 준수(Compliance)의 실패부터 프롬프트 주입 공격(Prompt Injection)에 이르기까지, 불확실성이 내포하는 다차원적 리스크들을 세부적으로 살펴본다. 이는 단순한 기술적 오류를 넘어 왜 오라클(Oracle)을 통한 엄격한 정답 판별 체계가 비즈니스 전략의 핵심 방어 기제로 도입되어야 하는지에 대한 필연적 근거를 제시할 것이다.
