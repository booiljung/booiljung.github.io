1.8.3 사례 3: 의료 AI의 환각으로 인한 처방 지침 위반 리스크

의료 도메인(Medical Domain)에서 시스템의 출력 오류는 기업의 재무적 손실을 넘어 인간의 생명이라는 가장 좁고 치명적인 임계점(Critical Threshold)을 건드린다. 환자의 전자의무기록(EMR)을 기반으로 한 챗GPT 기반 진단 보조 시스템들이 파일럿(Pilot) 단계에서부터 즉각적인 서비스 중단 철퇴를 맞았던 사례들이 존재한다.

1. 문맥 소실과 잘못된 상관관계 매핑

대규모 언어 모델은 확률적인 추측 엔진이므로, 방대한 환자의 가족력, 과거 알레르기 반응 등의 차트 정보가 입력 컨텍스트(Context)로 쏟아져 들어올 때 특정 키워드를 다른 키워드로 교차 결합해 버리는 치명적인 ’어텐션의 혼선(Attention Confusion)’을 겪는다.

  • 실제 유입 데이터: “환자는 페니실린 알레르기가 있음. 현재 심한 호흡기 질환 앓음.”
  • AI의 추론/출력: “해당 호흡기 질환 해결에 가장 널리 쓰이는 항생제인 아목시실린(페니실린 계열)을 우선 투여할 것을 권고함.”

모델은 글로벌 지식 베이스에서 “호흡기 질환 == 아목시실린 처방“이라는 높은 확률의 연관성을 뽑아냈으나, 문서 초반에 명시되었던 한 줄의 결정적 제약 조건(“페니실린 알레르기”)을 가중치 계산에서 미끄러뜨리거나 무시(Lost in the middle)해 버린 것이다.

2. 도메인 규칙 절대성의 붕괴

의료 지침이나 로직 플로우는 엄밀한 순서도를 가진 결정 모델(Decision Tree)이다. A를 무시하면 즉사하는 환경에서, 99.9% 잘 요약해 내는 언어 모델의 확률적 스코어링(Scoring)은 이 0.1%의 ‘오접합’ 앞에서 살인 병기로 둔갑할 수 있다. 이는 도메인 특화 지식(Domain Knowledge Base)을 프롬프트 안에 넣어서 예방하는 수동적인 방식만으로는 환각을 도저히 피할 수 없으며, 출력된 결과물 내에 포함된 처방 코드가 환자의 기존 EMR 금기(Contraindication) 리스트와 결정론적 교집합(Intersection)을 가지는지 검사하는 필터형 오라클 체계만이 윤리적 대참사를 막을 수 있음을 시사한다.
