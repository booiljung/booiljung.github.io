1.2.1.1 자기회귀(Autoregressive) 생성의 특징과 누적 오차(Accumulated Error)

대규모 언어 모델이 시퀀스(Sequence)를 생성하는 기본 방식인 자기회귀(Autoregressive) 모델링은 본질적인 취약성을 가지고 있다. 자기회귀란 이전 단계(Step)에서 모델 자신이 예측하여 출력한 결과(Output)가 다음 단계의 입력(Input)으로 다시 편입되는 순환적 피드백 루프(Recursive Feedback Loop)를 의미한다. 이러한 구조는 긴 문장이나 코드를 매끄럽게 이어나가도록 돕지만, 역설적으로 오류의 눈덩이 효과(Snowball Effect)를 조장한다.

1. 누적 오차(Accumulated Error) 현상

수학적으로, k개의 토큰으로 이루어진 시퀀스가 완벽하게 올바르게 생성될 확률은 각 단계의 조건부 확률의 곱으로 표현된다.

P(w_1, w_2, ..., w_k) = \prod_{i=1}^{k} P(w_i \vert w_1, ..., w_{i-1})

만약 모델이 100줄짜리 파이썬 코드를 생성하는 과정에서, 단 1개의 토큰(예: 변수명을 count 대신 counter로 잘못 예측)을 잘못 선택했다고 가정해보라. 이 ’오류 토큰’은 즉각적으로 모델의 컨텍스트(Context Workflow)에 통합된다. 다음 토큰을 예측할 때 모델은 자신이 방금 내뱉은 잘못된 변수명 counter가 정답이라고 간주한 채 연산을 수행하므로, 그 이후의 함수 호출과 로직 전개 전체가 counter를 기준으로 재조정되면서 원래 의도했던 로직에서 완전히 탈선(Derailment)하게 된다.

긴 컨텍스트 윈도우(Long Context Window)의 양면성

최근 모델들이 수십만 토큰의 컨텍스트 윈도우를 지원하면서 이러한 누적 오차 문제는 겉보기에 완화된 것처럼 보이지만, 실상은 더욱 복잡해졌다. 모델이 참조할 수 있는 과거 상태(Past State)가 너무 길어지면 주의 집중 공간(Attention Space) 내에서 노이즈(Noise)가 축적된다.

결과적으로 시스템은 길고 안정적인 출력을 내는 것처럼 보이다가도, 초반부에 미세하게 어긋난 한 토큰의 확률적 선택으로 인해 후반부 수천 줄의 출력이 ’우아하게 실패(Gracefully Fail)’해버리는 모순을 발생시킨다. 이는 결정론적 컴파일러(Compiler) 체계에서는 한 글자의 오타도 즉각적인 문법 오류(Syntax Error)로 튕겨내던 것과 완벽히 상반되는 양상이다.
