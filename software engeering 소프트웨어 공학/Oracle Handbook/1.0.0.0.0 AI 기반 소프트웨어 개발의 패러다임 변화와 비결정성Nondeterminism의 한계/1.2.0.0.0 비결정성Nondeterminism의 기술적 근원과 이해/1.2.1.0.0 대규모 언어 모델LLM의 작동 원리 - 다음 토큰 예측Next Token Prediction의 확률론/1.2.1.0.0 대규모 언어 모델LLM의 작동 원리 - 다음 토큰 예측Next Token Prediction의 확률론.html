1.2.1 대규모 언어 모델(LLM)의 작동 원리: 다음 토큰 예측(Next Token Prediction)의 확률론

인공지능, 특히 트랜스포머(Transformer) 아키텍처를 기반으로 하는 대규모 언어 모델(LLM)이 인간의 언어를 능숙하게 구사하는 것처럼 보일지라도, 그 기저에서 구동되는 핵심 연산 원리는 극히 단순한 통계적 메커니즘에 기반한다. 그것은 바로 ’주어진 이전 토큰(Token)들의 시퀀스(Sequence)를 바탕으로, 그 다음에 올 가능성이 가장 높은 토큰이 무엇인지 예측하는 것’이다. 이를 다음 토큰 예측(Next Token Prediction)이라고 부른다.

1. 언어의 수학자적 치환: 토큰(Token)

컴퓨터는 텍스트를 인간처럼 문맥과 의미론(Semantics)을 직관적으로 이해하여 처리하지 못한다. 대신 문자열을 모델이 처리할 수 있는 가장 작은 의미 단위인 ’토큰(Token)’으로 쪼개고, 각 토큰을 고차원 벡터(Vector)로 매핑(Mapping)한다. 토큰은 언어마다, 모델마다(BPE, WordPiece 등) 분할 기준이 다르지만 통상적으로 단어이거나 단어의 일부(Subword) 형태를 띤다.

2. 확률적 패턴 매칭으로서의 생성

LLM이 “하늘은 매우“라는 입력 시퀀스를 받았을 때, 모델은 인터넷에 존재하는 수백만 건의 코퍼스(Corpus)에서 학습한 가중치(Weights)를 바탕으로 다음 단어를 확률적으로 추론한다.

  1. “푸르다” - 85%
  2. “맑다” - 10%
  3. “흐리다” - 4%
  4. “아름답다” - 1%

모델은 이처럼 사전(Vocabulary)에 등록된 수만 개의 토큰 각각에 대해 다음에 등장할 확률(Probability) 값을 부여한다. 여기서 모델이 단어를 생성하는 행위 자체는, 본질적으로 결정론적 탐색(Deterministic Search)이 아니라 확률 공간에서의 샘플링(Sampling)이다.

        Model-->>User: 토큰 출력(예: "fox")
    end

3. 비결정성의 첫 번째 맹점

다음 토큰 예측이라는 모델의 기저 원리가 시스템에 비결정성(Nondeterminism)을 도입하는 근본적인 이유는, 모델이 가장 확률이 높은 1순위 단어(Greedy Search)만을 맹목적으로 선택하지 않는다는 데 있다. 항상 가장 높은 확률의 분포만을 따라간다면 모델의 응답은 평이하고 뻔한 말들로만 구성되어 마치 로봇 같은(Robotic) 언어를 구사하게 될 것이다. 이를 극복하기 위해 의도적으로 차순위 확률을 가진 단어가 선택될 여지를 남겨두는 확률적 샘플링(Stochastic Sampling) 기법이 결합된다.

즉, 동일하게 “하늘은 매우“라는 입력이 주어졌을 때, 85%의 확률을 뚫고 10%의 확률을 가진 “맑다“가 선택될 수 있으며, 이것이 바로 같은 질문에 응답할 때마다 결과가 달라지게 만드는 가장 원초적인 기술적 근원이다.
