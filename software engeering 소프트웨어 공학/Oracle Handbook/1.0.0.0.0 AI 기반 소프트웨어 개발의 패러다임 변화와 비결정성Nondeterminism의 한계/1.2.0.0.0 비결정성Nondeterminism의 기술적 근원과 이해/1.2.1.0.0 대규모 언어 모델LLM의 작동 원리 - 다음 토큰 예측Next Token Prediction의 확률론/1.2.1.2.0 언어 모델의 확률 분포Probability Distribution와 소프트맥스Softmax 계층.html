1.2.1.2 언어 모델의 확률 분포(Probability Distribution)와 소프트맥스(Softmax) 계층

언어 모델의 마지막 계층(Layer)에서 출력되는 값은 단어가 아니라, 모델의 사전에 포함된 모든 어휘(Vocabulary)에 대한 실수 벡터인 로짓(Logit) 스코어다. 이 정제되지 않은 스코어(Raw Score)를 확률값으로 치환하여 최종적으로 어떤 토큰을 선택할지 결정하게 하는 수학적 장치가 바로 소프트맥스(Softmax) 함수이다.

1. 소프트맥스(Softmax) 함수의 수학적 이해

소프트맥스 함수는 N차원의 벡터 z를 입력받아, 각 요소를 0과 1 사이의 값으로 정규화(Normalization)하고 모든 요소의 합이 1이 되도록 변환한다. 수식은 다음과 같다.

P(y_i \vert x) = \frac{e^{z_i/T}}{\sum_{j=1}^{N} e^{z_j/T}}

여기서 z_i는 i번째 단어의 로짓(Logit) 스코어이며, T는 온도(Temperature) 파라미터이다. 소프트맥스를 통과한 후의 분포는 마치 룰렛(Roulette) 판과 같다. 모델이 가장 그럴듯하다고 평가한 단어는 룰렛에서 아주 넓은 영역을 차지하고, 터무니없는 단어는 바늘 하나 꽂을 정도로 미세한 영역을 차지한다.

비결정성의 통계적 기원

이 수학적 계층이 의미하는 바는 명확하다. LLM은 결코 100% 확신을 가지고 말하지 않는다. 위 다이어그램에서 “blue“가 70%로 모델이 생각하는 ’가장 논리적인(Greedy) 정답’일지라도, 시스템은 구동될 때마다 확률 분포 위에서 무작위 샘플링을 진행한다.

어느 날 모델이 20%의 “cloudy“를 픽업하고, 또 어떤 날은 1%의 “falling“을 픽업하여 “하늘이 무너진다(The sky is falling)“라는 완전히 결이 다른 문장을 창조해낼 수 있다. 이러한 확률 분포를 엔지니어가 어떻게 조율하느냐에 따라 환각(Hallucination)의 발생 빈도와 시스템의 창의성이 극명하게 갈리는 것이 소프트웨어 2.0 운영의 딜레마다.
