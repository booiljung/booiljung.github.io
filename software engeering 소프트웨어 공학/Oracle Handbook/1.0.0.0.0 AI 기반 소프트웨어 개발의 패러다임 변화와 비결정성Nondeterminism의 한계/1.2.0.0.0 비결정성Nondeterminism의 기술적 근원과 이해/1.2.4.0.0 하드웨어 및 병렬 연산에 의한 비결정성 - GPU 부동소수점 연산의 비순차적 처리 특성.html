1.2.4 하드웨어 및 병렬 연산에 의한 비결정성: GPU 부동소수점 연산의 비순차적 처리 특성

소프트웨어와 알고리즘의 모든 파라미터가 완벽하게 통제되었을지라도, 대규모 병렬 연산을 수행하는 하드웨어의 물리적 아키텍처 자체가 내재적인 비결정성(Hardware-induced Nondeterminism)을 유발한다. 가장 주요한 원인은 그래픽 처리 장치(GPU) 또는 텐서 처리 장치(TPU) 내에서 이루어지는 부동소수점(Floating-Point) 연산의 한계에서 기인한다.

1. 부동소수점 연산의 결합 법칙 위배

컴퓨터 과학에서 실수(Real Numbers)를 표현하는 IEEE 754 부동소수점 표준은 제한된 비트 수(예: 16-bit, 32-bit, 최근에는 하드웨어 최적화를 거친 bfloat16 등)에 무한한 실수를 구겨 넣기 때문에 소수점 이하의 낮은 자릿수에서 반올림 오차(Rounding Error)가 필연적으로 발생한다.

이러한 반올림 오차로 인해 부동소수점 덧셈은 수학의 기본 원리인 결합 법칙(Associative Property)을 만족하지 못하게 된다.
즉, (A + B) + C \neq A + (B + C)가 성립하는 상황이 발생한다.

2. GPU 스레드의 비결정적 스케줄링

LLM은 수백억 개의 매개변수 행렬 연산(Matrix Multiplication)을 처리할 때, GPU에 존재하는 수천 개의 병렬 코어(CUDA Cores)를 활용하여 연산을 잘게 쪼개어(Reduction) 동시에 수행한다. 문제는 이 병렬 연산들이 완료되고 결과를 하나로 합산(Summation)하는 순서가 하드웨어의 클럭 스케줄링(Clock Scheduling), 메모리 지연(Memory Latency), 열 스로틀링(Thermal Throttling) 등에 의해 매 실행마다 달라진다는 점이다.

행렬 연산 시 수백, 수천 개의 숫자를 더하는 순서가 뒤바뀌면서, 아주 미미한 반올림 오차가 쌓이게 된다. 신경망의 깊이가 수십 개 층(Layers)에 달할 경우, 앞서 발생한 10^{-4} 수준의 미세 오차가 비선형 활성화 함수(Non-linear Activation Functions)와 누적 과정을 거치며 눈덩이처럼 부풀어 오른다.

결국 하드웨어 단에서의 미세한 비결정성(Nondeterminism)이 네트워크의 최종 로짓(Logit) 값을 흔들어놓고, 이는 가장 높은 확률을 가지는 토큰 자체를 엎어버리는 상황으로 이어질 수 있다. 따라서 부동소수점을 기반으로 하는 거대 하드웨어 스택을 포기하지 않는 한, 완벽한 수준의 모델 출력 일관성을 담보하는 것은 입증 불가능한 난제와 같다.
