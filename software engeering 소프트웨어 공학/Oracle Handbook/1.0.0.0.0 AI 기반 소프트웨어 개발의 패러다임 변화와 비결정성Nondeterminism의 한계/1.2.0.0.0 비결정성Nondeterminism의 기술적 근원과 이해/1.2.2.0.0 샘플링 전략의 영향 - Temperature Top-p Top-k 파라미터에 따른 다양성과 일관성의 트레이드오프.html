1.2.2 샘플링 전략의 영향: Temperature, Top-p, Top-k 파라미터에 따른 다양성과 일관성의 트레이드오프

대규모 언어 모델이 출력할 확률 분포(Probability Distribution)를 형성했다 하더라도, 최종적으로 어떠한 토큰을 채택할 것인가의 문제는 전적으로 샘플링(Sampling) 기법에 달려 있다. 개발자는 시스템 프롬프팅 시 몇 가지 파라미터(Hyperparameters)를 조작하여 언어 모델의 출력 특성을 결정론에 가깝게 만들 것인지, 혹은 고도의 비결정성을 허용할 것인지 타협점(Trade-off)을 찾게 된다.

1. Temperature (온도)

Temperature 파라미터 T는 소프트맥스(Softmax) 계층의 로짓(Logit) 값의 척도를 재조정(Rescaling)하는 역할을 한다. 수식적으로 볼 때, T 값이 낮아지면(예: T \rightarrow 0), 로짓 값의 차이가 기하급수적으로 부각되어 가장 확률이 높은 단어 하나가 사실상 100%에 가까운 파이를 차지하게 된다. 이를 보통 탐욕적 디코딩(Greedy Decoding)이라 부른다.

반대로 T 값이 높아지면(예: T = 1.0), 확률 분포가 평탄해지며(Flatten) 덜 그럴듯한(Low-probability) 단어들이 채택될 확률이 유의미하게 상승한다. 이는 시나리오 작가나 시인 봇(Bot)에게는 유용할 수 있으나, 엄밀한 JSON 구조나 SQL 쿼리를 생성해야 하는 공학적 파이프라인에서는 런타임 오류로 직결된다.

2. Top-k 샘플링

Top-k 샘플링은 로짓 기반으로 확률을 내림차순 정렬한 뒤, 상위 K개의 후보 단어 그룹 내에서만 샘플링을 수행하는 기법이다.
예를 들어 K=50으로 설정하면, 51번째부터 사전에 등록된 수만 개의 하위 꼬리(Tail) 단어들은 아예 고려 대상에서 배제(Truncated)된다. 이는 문법이 완전히 무너지는 횡설수설(Gibberish) 출력을 방어하기 위한 강력한 통제 수단이 된다.

3. Top-p (Nucleus) 샘플링

Top-p 샘플링은 고정된 개수(K)를 자르는 대신, 누적 확률(Cumulative Probability) 질량이 특정 임계값 p (예: p=0.9)에 도달할 때까지만 상위 후보군을 포함시킨다.

Top-p 방식은 컨텍스트가 매우 명확하여 정답 단어 1~2개가 확률의 90% 이상을 차지할 경우에는 후보군을 좁혀주고, 반대로 예측이 평탄하여 모호한 경우에는 수십 개의 후보군을 열어두는 동적인 특징을 갖는다.

결론적으로, 이 세 가지 파라미터를 아무리 철저히 튜닝(Tuning)하더라도 Temperature=0을 제외하면 본질적으로 모델은 무작위성(Randomness)을 내포하게 되며, 이는 완벽한 재현성(Reproducibility)을 저해하는 기폭제로 작용한다.
