1.2 비결정성(Nondeterminism)의 기술적 근원과 이해

소프트웨어 시스템에서 비결정성(Nondeterminism)이란 동일한 초기 상태(Initial State)와 동일한 입력(Input)이 주어졌음에도 불구하고 시스템이 실행될 때마다 서로 다른 출력(Output)이나 상태 변화를 유발하는 성질을 의미한다. 현대의 인프라 스택이나 운영체제의 스레드 스케줄링(Thread Scheduling)에서 발생하는 미세한 수준의 비결정성도 존재하지만, 대규모 언어 모델(Large Language Model, LLM)을 기반으로 하는 소프트웨어 2.0(Software 2.0) 시대의 비결정성은 차원이 다른 불확실성을 시스템에 부여한다.

본 장에서는 AI 모델이 본질적으로 비결정적일 수밖에 없는 기술적 근원을 상세히 파헤친다. 대규모 언어 모델이 단어를 생성하는 가장 기저의 원리인 다음 토큰 예측(Next Token Prediction) 메커니즘을 시작으로, 확률 분포와 샘플링(Sampling) 기법이 조장하는 무작위성(Randomness)을 분석한다. 나아가 소프트웨어를 넘어서서 GPU 부동소수점 연산의 비순차적 처리와 같은 하드웨어 종속적인 문제, 그리고 API 제공자의 잠수함 패치(Silent Patch)로 인해 발생하는 버전 드리프트(Model Drift)까지 포괄적으로 다룬다.

이러한 기술적 근원에 대한 깊이 있는 이해는, 향후 개발자가 완전히 통제 불가능한 이 비결정성을 어떻게 엔지니어링적으로 관리하고 ’오라클(Oracle)’을 통해 결정론적(Deterministic) 강건함을 확보할 것인지 설계하는 토대가 될 것이다.
