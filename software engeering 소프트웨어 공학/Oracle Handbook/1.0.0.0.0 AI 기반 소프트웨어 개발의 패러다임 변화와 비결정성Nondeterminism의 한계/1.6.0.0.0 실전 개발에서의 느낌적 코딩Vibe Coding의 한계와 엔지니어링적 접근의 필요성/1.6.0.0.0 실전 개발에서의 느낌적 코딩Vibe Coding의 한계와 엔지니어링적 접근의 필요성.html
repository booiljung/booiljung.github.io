1.6 실전 개발에서의 ’느낌적 코딩(Vibe Coding)’의 한계와 엔지니어링적 접근의 필요성

최근 생성형 AI와 챗봇 인터페이스의 눈부신 발전으로 인해, 비개발자 혹은 주니어 엔지니어들조차 자연어를 통해 손쉽게 코드의 초안을 작성하고 실행할 수 있는 시대가 열렸다. 개발자가 머릿속에 떠오르는 아이디어를 엄밀한 문법 검증 없이 자연어로 던지면(Prompting), AI가 그 느낌을 파악해 코드로 구현해주는 이른바 ‘느낌적 코딩(Vibe Coding)’ 현상이 만연하고 있다.

하지만 데모 시연장(Demospace)에서는 마법처럼 작동하던 이 방식이, 실제 무결성이 보장되어야 하는 프로덕션(Production) 환경에서는 처참하게 붕괴하는 모습을 자주 목격하게 된다. 본 장에서는 프롬프트에만 의존하는 개발론이 지니는 구조적 맹점을 파헤친다.

단순히 프롬프트 엔지니어링을 고도화하거나 사람의 눈으로 수동 검토하는 방식(Human-in-the-loop)이 왜 근본적 한계에 봉착할 수밖에 없는지를 짚어내고, 결국 이를 해결하기 위해서는 체계적인 평가 파이프라인과 결정론적 오라클(Deterministic Oracle)을 필두로 한 하드코어 엔지니어링적 접근이 필수적임을 논의한다.
