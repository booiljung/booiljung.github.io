1.7 오라클(Oracle) 문제: AI 출력을 검증할 기준의 부재

소프트웨어 시스템이 뱉어내는 수많은 난해한 비결정성과 결함들을 관통하는 가장 근본적이고 철학적인 질문이 하나 남는다. “결정론적인 절대 정답(Ground Truth)조차 존재하지 않거나, 존재하더라도 비용이 너무 비싸서 구할 수 없다면, 우리는 AI가 도출한 결과가 맞는지 틀린 지 대체 어떻게 평가할 것인가?”

컴퓨터 과학과 소프트웨어 테스팅 분야에서 이 난제는 오랜 기간 동안 ‘테스트 오라클 문제(The Test Oracle Problem)’ 로 명명되어 왔다. 본 장에서는 오라클 문제의 핵심 정의를 AI의 맥락에서 재정립하고, 기존 소프트웨어 공학에서 사용되던 다양한 오라클의 유형들을 살펴본다. 나아가, 향후 본 서적에서 중점적으로 다루게 될 ’단순한 경험적 평가(Vibe Assessment)’를 넘어선 체계적이고 확정적인 ’평가 엔진(Evaluating Engine)’으로서의 새로운 오라클 설계 방향을 논의한다.
