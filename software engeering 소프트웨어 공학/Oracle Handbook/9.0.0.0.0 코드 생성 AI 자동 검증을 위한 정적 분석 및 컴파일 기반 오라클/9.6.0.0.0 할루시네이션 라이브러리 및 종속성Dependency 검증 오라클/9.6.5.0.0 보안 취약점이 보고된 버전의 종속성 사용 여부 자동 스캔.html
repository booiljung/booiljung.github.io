9.6.5 보안 취약점이 보고된 버전의 종속성 사용 여부 자동 스캔

AI가 아무리 좋은 코드를 생성했더라도 10년 전 버그 투성이의 패키지 Log4J 2.14를 임포트하면 시스템은 그 즉시 뚫린다.

  • AI 파이프라인의 오라클은 CI/CD 도구에 통합된 npm audit, pip-audit, 또는 Snyk 엔진에 방금 생성된 가상의 package.json 명세서를 던진다.
  • 즉석에서 CVE(Common Vulnerabilities and Exposures) 저장소와 대조하여 치명도(Critical, High) 등급의 보안 경고가 나오면, 오라클은 AI에게 “메이저 버전을 올리거나, 취약점이 패치된 다른 버전을 사용해 다시 작성하라“고 빠꾸를 놓는다.
