9.7.3 안전하지 않은 함수(Unsafe Functions) 사용에 대한 즉각적인 거부(Reject) 로직

C/C++의 메모리 파괴는 대부분 strcpy나 sprintf 같은 구시대 유물 함수에서 튀어나온다. AI 모델은 낡은 오픈소스를 너무 많이 읽었기에 이들을 무심코 부활시킨다.

  • 차단(Reject) 오라클 세팅: 오라클 설정 파일에 철저한 블랙리스트 맵핑을 하드코딩한다 {"strcpy": "strncpy", "gets": "fgets"}.
  • AST에서 저주받은 함수 호출(Call Node)이 식별되면 모델 전체 파이프라인이 정지된다. 더 무서운 점은 오라클이 단순 거부로 끝나는 게 아니라 강압적인 교정 프롬프트(Correction Prompt)를 쏴서 모델이 최신 표준에 맞게 코드를 강제 리팩토링하도록 채찍질한다는 것이다.
