9.11.3 분석 도구의 오탐(False Positive) 처리 및 예외 허용(Whitelisting) 전략

도구가 오작동을 일으켜 훌륭한 AI 코드를 죽일 때도 있다. (예: 메타프로그래밍이나 리플렉션을 사용한 기발한 코드를 스파게티 코드로 오탐함).

  • 예외 허용 우회로: 시스템에 “개발자 리뷰 요청(Flag for human review)” 이라는 브랜치를 설계한다.
  • 린터가 막아 세운 코드라도 AI 모델이 내부 논리망(CoT)에서 이 코드의 구조적 필요성 = 95% 라고 확신한다면, 코드를 킬하지 않고 // @ts-ignore 나 # noqa 같은 예외 처리 주석을 스스로 달도록 권한을 주고, 단 이를 머지(Merge)하기 전 반드시 시니어 엔지니어의 승인(Approve)을 얻게 하는 화이트리스트 루틴을 뚫어준다.
