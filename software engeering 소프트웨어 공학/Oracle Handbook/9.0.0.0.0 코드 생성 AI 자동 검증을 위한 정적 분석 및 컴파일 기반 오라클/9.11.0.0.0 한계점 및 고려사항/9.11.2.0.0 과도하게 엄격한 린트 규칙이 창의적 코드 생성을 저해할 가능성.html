9.11.2 과도하게 엄격한 린트 규칙이 창의적 코드 생성을 저해할 가능성

AI에게 완벽을 요구하며 ESLint의 모든 깐깐한 플래그(예: no-magic-numbers, max-statements)를 몽땅 켜두면, 코드 생성 파이프라인 자체가 숨을 쉬지 못한다.

  • 재시도 지옥(Retry Hell): 에러가 너무 자주 발생하면 AI는 무한히 자가 수정을 돌다 임계값을 넘고 뻗어버린다.
  • 해결책: AI 파이프라인용 린터 프로필(Profile)은 인간 개발자용보다 Warning 수준을 낮춘(Relaxed) 별도의 ai-relaxed.eslintrc를 운용해야 한다. 오직 크리티컬한 구문 오류와 타입 미스매치만 차단하고, 스페이스바 1개 같은 스타일 이슈는 사후 포매터(Prettier)가 기계적으로 교정하게 넘겨주는 것이 현명하다.
