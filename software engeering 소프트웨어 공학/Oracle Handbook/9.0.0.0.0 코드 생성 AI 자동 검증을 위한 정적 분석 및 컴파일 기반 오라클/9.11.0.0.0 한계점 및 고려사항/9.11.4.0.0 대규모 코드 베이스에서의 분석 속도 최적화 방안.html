9.11.4 대규모 코드 베이스에서의 분석 속도 최적화 방안

거대한 프로젝트(모노레포, 수백만 줄의 자바 소스)에서 AI가 코드 10줄을 생성할 때마다 전체 린터와 검사기를 돌리면 한 번 검증에 5분이 걸린다.

  • 극단적 시프트 레프트(Shift-Left) 캐싱: 변경되지 않은 수많은 의존성 모듈의 AST 트리 상태를 디스크나 브랜치 레벨에 해시 단위로 캐싱(Caching)해 둔다.
  • AI가 건드린 바로 그 특정 파일의 함수 스코프만을 고립시켜 증분 정적 분석(Incremental Static Analysis)을 돌려야, 에이전트가 생각하고-검증하고-다시 수정하는 턴어라운드(Turnaround) 비용이 5초 이내로 돌아 AI의 생산 속도를 보장할 수 있다.
