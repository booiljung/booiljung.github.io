9.11.1 정적 분석으로 탐지할 수 없는 런타임 논리 오류의 한계

정적 오라클의 가장 큰 구멍은 “잘못된 수학 연산도 문법은 맞다“는 점이다.

  • 예시: AI가 덧셈 함수 내에서 return a - b; 라고 뺄셈을 해놓아도, 컴파일러나 타입 체커는 두 변수가 정수라면 기쁘게 Pass 통과 도장을 찍는다.
  • 결론: 정적 오라클은 무의미한 코드를 걸러내는 ’거버넌스 필터’이지, 로직 검증의 끝이 아니다. 여기서 통과된 코드는 반드시 다음 챕터(Chapter 10)에서 다룰 동적 테스트 정답지(Golden Dataset)의 도마에 다시 올라 실효성을 검증받아야 한다.
