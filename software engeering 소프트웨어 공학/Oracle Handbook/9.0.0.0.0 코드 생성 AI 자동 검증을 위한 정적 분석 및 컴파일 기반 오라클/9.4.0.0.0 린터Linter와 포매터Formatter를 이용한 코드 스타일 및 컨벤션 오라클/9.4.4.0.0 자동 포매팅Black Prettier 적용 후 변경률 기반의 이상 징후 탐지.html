9.4.4 자동 포매팅(Black, Prettier) 적용 후 변경률 기반의 이상 징후 탐지

가장 교묘한 형태의 오라클은 포매터 자체를 역이용하는 것이다.

  • 방법론: 생성된 코드 원본(A)을 저장한다. 그 위에 Black(Python)이나 Prettier(JS)를 가장 공격적으로 실행한 포맷 후 코드(B)를 얻는다.
  • Diff 분석: A와 B의 글자 단위(Levenshtein Distance) 차이를 구한다. 이 Diff 비율이 30%를 넘는다면, 이는 AI가 언어의 근본적 관용구(Idiom) 자체를 무시한 엉터리 코드를 썼다는 증명이다. 이 경우 수정 코드(B)를 채택하는 것이 아니라 생성물 전체(A)를 이상 징후로 판별해 폐기한다.
