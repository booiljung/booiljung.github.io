9.4.3 AI 생성 코드의 가독성(Readability) 지표 측정 및 임계값 설정

AI는 인간보다 타이핑 속도가 무한에 가까우므로, 가끔 불필요하게 코드를 길고 복잡하게(Boilerplate) 나열하는 경향이 있다.

  • 오라클 메트릭: 할스테드 복잡도(Halstead Complexity)나 코드의 응집도 지표를 계산한다.
  • 린터 기반 측정: max-lines-per-function 룰을 50줄로 강제한다. AI가 하나의 함수에 60줄짜리 스파게티 로직을 짜냈다면 오라클은 이를 기각하고 “보조 함수(Helper Functions) 2개로 분할 선언하라“는 프롬프트를 쏴서 자가 수정(Self-correction) 리팩토링을 강제한다. 인간이 유지보수할 수 없는 AI 코드는 악성코드와 같다.
