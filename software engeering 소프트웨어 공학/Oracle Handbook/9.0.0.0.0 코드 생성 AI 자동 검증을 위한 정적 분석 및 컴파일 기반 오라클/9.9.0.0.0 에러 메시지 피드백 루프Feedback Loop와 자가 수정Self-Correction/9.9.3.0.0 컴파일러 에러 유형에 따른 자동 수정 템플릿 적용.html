9.9.3 컴파일러 에러 유형에 따른 자동 수정 템플릿 적용

에러의 본질(Syntax vs Type vs Lint)에 따라 AI를 다루는 온도(Temperature)와 파라미터가 달라져야 한다.

  • Syntax Error의 경우 모델의 추론(Reasoning) 능력보다 타이핑 실수가 원인이므로, 모델의 Temperature를 0.0으로 낮추고 철저한 결정론적 수정을 지시한다.
  • 반면 Type Error나 Unused Variable, Undefined Reference 같은 경우, 이는 모델의 논리 도메인 이해 부족에서 기인하므로, 약간의 Temperature 여유(0.3)를 주거나 해당 라이브러리의 공식 인터페이스 문서를 RAG로 추가 검색해서 피드백 컨텍스트에 덧대어(Augment) 주어야 한다.
