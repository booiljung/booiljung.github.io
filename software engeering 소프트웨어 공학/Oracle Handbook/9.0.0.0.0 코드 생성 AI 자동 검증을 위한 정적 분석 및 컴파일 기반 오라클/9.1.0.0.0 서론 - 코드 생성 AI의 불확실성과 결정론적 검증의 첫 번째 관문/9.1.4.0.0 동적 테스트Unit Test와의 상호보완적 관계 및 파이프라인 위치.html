9.1.4 동적 테스트(Unit Test)와의 상호보완적 관계 및 파이프라인 위치

오라클 시스템 아키텍처에서 이 두 가지 검증 툴은 병렬이 아닌 완벽한 직렬 계층구조(Cascading Hierarchy) 로 배치되어야 한다.

flowchart LR
    A[Generative AI] --> B(Static Oracle)
    B -->|Syntax Error| A
    B -->|Pass| C(Dynamic Oracle / Unit Test)
    C -->|Logic Error| A
    C -->|Pass| D[Merge / Deploy]
  • 정적 오라클은 코드가 ‘적법한 문서’ 인지를 심사한다. (형식적 합치성)
  • 동적 오라클은 코드가 ’원하는 일’을 잘 수행하는지를 심사한다. (의미론적 합치성)
    이 두 개의 깔때기가 통합(Hybrid Verification Pipeline)되어야만 AI가 작성한 악성코드나 런타임 오류가 메인 리포지토리에 스며드는 것을 원천 차단할 수 있다.
