9.1.2 실행 전 검증(Pre-execution Verification)의 중요성과 비용 절감 효과

동적 테스트(Unit Test, Integration Test)는 컨테이너를 스핀업(Spin-up)하고, DB를 연결하고, 환경 변수를 세팅해야 하므로 수 초에서 수십 분이 걸리는 값비싼 검증이다.

  • 비용의 비대칭성: 괄호를 닫지 않은 코드나 임포트(Import)가 누락된 코드를 컨테이너에 밀어넣는 것은 극심한 서버 자원의 낭비다.
  • Shift-Left 패러다임: 실패(Fail)는 가장 왼쪽 프로세스에서 터질수록 저렴하다. 정적 분석 오라클은 텍스트 파싱과 트리 연산만으로 작동하므로 밀리초(ms) 단위로 수행된다. 여기서 1차로 필터링된 “형식적으로 완벽한 코드“만이 비싼 동적 테스트 오라클로 넘어가야 컴퓨팅 리소스(FinOps)가 최적화된다.
