9.5.1 컴파일 언어(C++, Go, Rust, Java)에서의 빌드 성공 여부를 통한 오라클 구현

정적 분석기와 린터가 ’권고’라면 컴파일 오류는 ’법의 집행’이다. AI가 생성한 Go나 Rust 코드가 빌드 명령어(go build, cargo build)를 통과하지 못하면 모든 프로세스는 즉시 얼어붙는다.

  • Sandbox 컴파일러 훅(Hook): AI의 아웃풋 스트림이 완전히 종료되면, CI는 즉시 임시 컨테이너에 코드를 붓고 g++ 이나 javac를 타격한다.
  • 바이너리 생성 검증: 경고 하나 없이 바이트코드(.class)나 바이너리가 디스크에 무사히 아웃풋(Output)되어야만 오라클은 Build_Success = True 신호를 쏜다. 단 1바이트라도 메모리 소유권(Rust Borrow Checker) 에러가 나면 판정은 가차 없이 Fail이다.
