9.3.5 제네릭(Generics) 및 복잡한 타입 정의의 유효성 검사

AI에게 제네릭 리스트 박싱(List<T>)이나 상속 체인 구현을 맡기면 타입 소실(Type Erasure)의 늪에 빠져 Any나 Object 남발이라는 편법을 쓴다.

  • Any/Unknown 방어선: 오라클의 mypy 설정 창에서 disallow_any_generics 및 disallow_untyped_defs 플래그를 True로 선언한다.
  • AI가 제네릭 구조체에 들어갈 명확한 데이터 파생 타입을 유추하지 못하고 List[Any]를 내뱉는 순간, 이는 “나는 코드의 미래를 확정할 수 없다“는 항복 선언이므로 오라클은 이를 맹렬히 타격하여 TypedDict 나 명시적 클래스 모델링을 요구해야 한다.
