9.8.1 순환 복잡도(Cyclomatic Complexity) 측정 및 임계값 제한

AI가 조건문(If-Else)을 난사하면 소스 코드는 수십 개의 분기점(Branch)을 가진 괴물이 된다.

  • 맥케이브(McCabe) 복잡도 산정: 오라클은 Radon 이나 ESLint (complexity) 룰을 주입하여 생성된 함수의 순환 복잡도를 측정한다.
  • 임계값 설정: 함수의 복잡도 점수가 10을 초과하는 순간(분기점이 10개를 넘으면) 무조건 Fail 처리한다. AI에게 “하나의 함수가 책임지는 분기(If/Switch)가 너무 많으니, strategy pattern이나 다형성(Polymorphism)을 활용하여 클래스로 분리하라“는 리팩토링 명령을 하달한다.
