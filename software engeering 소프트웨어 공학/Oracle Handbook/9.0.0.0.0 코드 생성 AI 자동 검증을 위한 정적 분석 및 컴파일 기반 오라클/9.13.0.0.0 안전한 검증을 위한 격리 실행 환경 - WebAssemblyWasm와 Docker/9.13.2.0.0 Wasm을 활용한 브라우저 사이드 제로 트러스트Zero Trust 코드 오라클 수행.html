9.13.2 Wasm을 활용한 브라우저 사이드 제로 트러스트(Zero Trust) 코드 오라클 수행

도커를 띄우는 수백 밀리초의 지연마저 아까울 때 가장 현대적 대안은 WebAssembly(Wasm) 기술이다.

  • Wasm 오라클: Python 코드를 검증해야 한다면 서버의 파이썬 인터프리터를 호출하지 않고, 검증 워커(Node.js 등) 내부의 메모리 샌드박스에 컴파일된 Pyodide(Python in Wasm)를 주입한다.
  • AI가 만든 코드는 호스트 OS와는 완전히 격리된 이 샌드박스의 논리적 가상 CPU 위에서만 AST 파싱과 린터(Pylint in Wasm)를 거친다. 파일 시스템(I/O) 자체가 스푸핑 방지되어 있으므로 궁극의 제로 트러스트(Zero-trust) 검증 망이 구축된다.
