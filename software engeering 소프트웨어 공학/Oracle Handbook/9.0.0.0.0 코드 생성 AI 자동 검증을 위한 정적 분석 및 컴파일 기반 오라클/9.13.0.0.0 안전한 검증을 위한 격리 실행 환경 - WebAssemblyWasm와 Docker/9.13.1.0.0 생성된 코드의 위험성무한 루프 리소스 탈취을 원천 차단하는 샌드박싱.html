9.13.1 생성된 코드의 위험성(무한 루프, 리소스 탈취)을 원천 차단하는 샌드박싱

AI가 while(true) { allocate_memory(); } 라는 기괴한 코드를 짰다고 해보자. 이를 도커(Docker) 기반 샌드박스에서 돌릴 때 자원 제약이 없다면 검증용 서버 자체가 터져버린다.

  • 격리 오라클의 자원 제한(Resource Quotas): 정적 검증과 임시 컴파일을 수행하는 Docker 컨테이너에 극단적인 제한을 건다. Memory Limit = 128MB, CPU = 0.5 Core, 플랫폼 타임아웃(Timeout) = 5초.
  • 이 시간 안에 파싱과 컴파일이 끝나지 않거나, 서버의 디렉토리를 읽으려는 권한 밖의 시스템 콜(OS Syscall)이 모니터링 모듈에 잡히면, 오라클은 컨테이너 머리를 날려버리고(Kill) AI 코드 생성 태스크를 “치명적 악성코드(Malicious Code) 등급“으로 영구 밴(Ban)한다.
