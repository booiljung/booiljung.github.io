4.5.2 결정론적 오라클을 위한 ‘골든 예제(Golden Examples)’ 선정 기준

단순히 예제를 많이 준다고 좋은 것이 아니다. 잘못된 예제는 모델의 편향을 왜곡한다. 예측 가능성을 극대화하기 위해 삽입할 예제들은 다음의 ’골든 스탠다드’를 충족해야 한다.

  1. 포맷의 완전한 무결성:
    만약 JSON으로 출력하길 원한다면, 예제 속 답변은 띄어쓰기 한 칸과 줄바꿈(Newline)까지도 최종적으로 기대하는 시스템 에코(Echo) 포맷과 100% 일치해야 한다.
  2. 다차원적 커버리지(MCE, Mutually Collectively Exhaustive):
    긍정 사례(PASS)만 잔뜩 줘서는 안 된다. 오라클이 판별해야 할 핵심 클래스가 4개(PASS, FAIL_LOGIC, FAIL_SYNTAX, UNKNOWN)라면, 이 4가지 클래스가 모두 한 개 이상 포함된 균형 잡힌 예제 묶음을 제공해야 한다.
  3. 간결성(Conciseness):
    예제 문제의 난이도가 너무 길고 복잡하면, 정작 평가해야 할 본 문제의 컨텍스트 비중이 희석되어 버린다. 본질적 패턴만을 가장 날카롭게 보여주는 최소 단위의 텍스트가 골든 예제다.
