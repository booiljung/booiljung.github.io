4.5.5 동적 퓨샷 프롬프팅(Dynamic Few-Shot Prompting): 입력값 유사도 기반 예제 교체

단일 프롬프트를 사용하는 고정형 오라클(Static Oracle)은, 검증할 데이터의 도메인이 방대할 경우 컨텍스트 윈도우 한계에 부딪혀 효율성이 떨어진다. 이를 극복하는 엔터프라이즈 패턴이 동적 퓨샷(Dynamic Few-Shot) 파이프라인이다.

1. K-NN 기반의 동적 예제 검색 아키텍처

  1. 예제 풀(Pool) 구축:
    오라클의 판단 기준이 되는 수백 개의 골든 예제(질문-정답 쌍)를 미리 작성하고, 각 질문을 임베딩(Embedding) 벡터로 변환하여 벡터 데이터베이스(Vector DB)에 저장한다.
  2. 실시간 검색(Retrieve):
    사용자의 실제 테스트 데이터(Query)가 들어오면, 벡터 DB에서 해당 Query와 코사인 유사도(Cosine Similarity)가 가장 높은 상위 3개의 K-NN 예제를 즉석에서 검색한다.
  3. 프롬프트 합성(Injection):
    시스템 프롬프트의 퓨샷(Few-shot) 자리표시자(Placeholder)에 방금 검색해 온 3가지 예제를 런타임에 동적으로 주입하여 LLM을 호출한다.

이 방식을 사용하면 모델에게 “지금 네가 마주한 문제와 수학적으로 가장 비슷한 과거의 기출문제“를 실시간 족집게로 제공하는 격이 되어, 그 어떤 복잡한 비즈니스 룰 모델이더라도 답변의 정확성과 결정론을 99% 수준으로 끌어올릴 수 있다.
