4.5.3 예제 순서(Order)가 모델의 편향과 응답 일관성에 미치는 영향

흥미롭게도 LLM은 프롬프트에 주입된 예제들의 내용뿐만 아니라 그 순서(Sequence) 에 극도로 예민하게 반응한다. 이를 근접성 편향(Recency Bias 혹은 Order Bias) 이라 부른다.

어텐션 레이어는 가장 최근에(가장 마지막에) 읽은 예제의 패턴에 가장 무거운 가중치를 부여한다.

  • 문제 상황: 프롬프트의 예제 3개가 [PASS, PASS, FAIL] 순서로 제시되어 있다면, 모델은 무의식적으로 “아, 마지막 아웃풋 확률 공간은 FAIL로 끝나는 패턴이구나“를 학습하여 실제 문제를 풀 때 과도하게 FAIL을 내뱉는 ’보수적 편향’에 빠질 수 있다. 반대로 [FAIL, FAIL, PASS]의 배열이면 오탐(False Positive)의 확률이 올라간다.
  • 방어 기법:
    가급적 라벨(Label)의 균형을 맞출 뿐만 아니라, 특정 라벨이 마지막을 장식하지 않도록 섞거나, 동일한 프롬프트라도 예제의 순서를 변경해 가며 검증(Majority Voting)하는 방식이 고급 오라클 파이프라인에서 채택된다.
