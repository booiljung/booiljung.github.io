4.5.1 제로샷(Zero-Shot)의 불확실성과 퓨샷(Few-Shot)의 지향성

아무런 예제 없이 지시문만 던지는 제로샷(Zero-Shot) 상태에서 LLM은 자신이 읽어들인 방대한 인류의 코퍼스 중 어느 영역의 ’페르소나’를 활성화해야 할지 넓게 탐색한다. 이 과정에서 필연적으로 ’다양성’이 개입하며, 동일한 룰을 주어도 번역가 모드, 요약가 모드, 친절한 비서 모드를 오가며 매번 다른 형태로 출력할 확률이 높다.

반면, 퓨샷(Few-Shot) 예제를 삽입하는 것은 모델 내부적으로 ‘패턴 매칭(Pattern Matching)’ 알고리즘을 강하게 트리거하는 행위다. 어텐션(Attention) 메커니즘은 새롭게 들어온 사용자의 질문을 이전 예제의 Input들과 비교하고, 그 Output의 문법적 구조를 완벽하게 복제하려고 시도한다.

즉, 퓨샷은 지시문을 이해시키는 행위가 아니라, 형식을 모방하게 만드는 조건반사(Conditioning) 장치이며, 출력의 외형적 일관성을 확보하는 가장 원초적이고 확실한 기술이다.
