4.5.4 다양한 엣지 케이스(Edge Case)를 포함한 예제 구성 전략

오라클이 무너지는 지점은 전형적이고 명백한 데이터가 들어올 때가 아니다. 오라클의 엔진이 다운되는 것은, 코드에 고의적인 문법 에러가 섞이거나, 프롬프트 인젝션 방어 규칙을 교묘하게 비켜나간 ‘회색 지대(Edge Case)’ 데이터가 들어올 때다.

따라서 퓨샷에 포함될 3~5개의 예제 중 최소 1개는 반드시 이 Edge Case에 대한 모범 답안이어야 한다.

1. 필수 포함 Edge Case 예시

  1. 불충분한 컨텍스트(Insufficient Context):
    정보가 부족할 때 AI가 상상(Hallucination)하지 않고 단호히 “UNKNOWN“이나 “예외 처리“를 내뿜는 예제.
  2. 모호한 경계(Boundary Condition):
    정책상 허용 한도치의 딱 턱걸이에 걸린 데이터를 어떻게 파싱하는가에 대한 엄격한 해석 예제.

엣지 케이스를 예로 보여주면, 모델은 자신의 판단에 불확실성이 감지될 때 자의적인 추론을 멈추고 예제에 명시된 ’안전한 예외 코드(Fallback code)’로 비상 탈출하는 법을 학습한다.
