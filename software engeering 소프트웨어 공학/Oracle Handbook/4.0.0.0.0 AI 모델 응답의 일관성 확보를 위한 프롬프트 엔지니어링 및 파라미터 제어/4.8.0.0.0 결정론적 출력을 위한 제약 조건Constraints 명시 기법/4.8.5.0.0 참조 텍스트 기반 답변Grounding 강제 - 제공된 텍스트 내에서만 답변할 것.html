4.8.5 참조 텍스트 기반 답변(Grounding) 강제: “제공된 텍스트 내에서만 답변할 것”

대형 언어 모델은 이미 학습된 사전 지식(Parametric Knowledge)과 프롬프트를 통해 외부에서 주입된 문맥(Contextual Knowledge) 사이에서 잦은 충돌을 겪는다.

사내 라이브러리 MyUtil이 업데이트되어서 “B 함수“를 쓰도록 바뀌었는데, 모델의 사전 지식(1년 전 오픈소스 짬뽕)에는 “A 함수가 정답“이라고 박혀 있다면, 오라클은 테스트 코드가 정상임에도 FAIL 처리를 할 것이다. 이런 현상을 억제하는 기술이 바로 그라운딩(Grounding) 통제다.

Grounding 프롬프트 지시문:

“당신의 본래 지식 기반은 무시하고, 오로지 이 프롬프트의 <doc> 태그 안에 제공된 사내 규정 텍스트만을 진실로 간주(Grounding)하여 평가하라. 제시된 텍스트 밖에 있는 정보로 추론을 전개하면 즉각 결함으로 간주하겠다.”

이 선언 하나로 오라클 파이프라인은 환각의 안개 속을 벗어나 ’주어진 문서 기반의 기계적 판독기’로 회귀하게 된다.
