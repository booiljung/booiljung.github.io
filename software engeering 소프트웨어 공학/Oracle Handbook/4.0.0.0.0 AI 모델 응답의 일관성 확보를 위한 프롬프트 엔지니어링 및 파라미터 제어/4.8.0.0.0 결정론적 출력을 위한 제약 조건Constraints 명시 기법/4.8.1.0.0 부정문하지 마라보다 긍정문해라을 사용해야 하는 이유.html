4.8.1 부정문(“~하지 마라”)보다 긍정문(“~해라”)을 사용해야 하는 이유

LLM의 내부 어텐션 연산은 단어 간의 연관성에 집중할 뿐, 논리학의 ‘Not(부정)’ 연산자에 태생적으로 취약하다.

“사과에 대해 말하지 마라“라고 썼을 때, 인간은 ’사과’를 머릿속에서 즉각 지워버리지만, 모델은 ’사과’라는 단어 자체에 어텐션이 쏠리며 사과와 관련된 단어(과일, 빨갛다)를 끌고 들어올 확률이 높아진다. 이를 부정문의 역설(Negative Prompt Paradox) 이라고 한다.

결정론적 결과를 원한다면 코딩의 화이트리스트(Whitelist) 룰처럼 긍정문으로 제한하라.

  • ❌ 오답: “결과를 출력할 때 설명이나 인사를 포함하지 말고 JSON 이외의 글을 쓰지 마라.”
  • ✅ 정답: “결과는 오직 중괄호 {}로 시작하고 끝나는 JSON 포맷 하나만 단독으로 반환하라.”

명령을 해야지, 변명의 여지를 주어서는 안 된다.
