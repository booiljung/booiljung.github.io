4.8.4 선택지(Option) 기반의 분류 문제로 변환하여 주관식 배제하기

폐쇄형 질문의 궁극적 진화 형태가 바로 ’다중 객관식(Multiple Choice) 라우팅’이다. 오라클 시스템을 설계할 때 생성(Generation) 태스크를 모두 분류(Classification) 태스크로 다운그레이드 시키는 것이 가장 안전한 프롬프트 패턴이다.

선택지 프롬프트 패턴 예시:

다음 오류 로그를 분석하고, 근본 원인을 아래 4가지 클래스(A, B, C, D) 중 정확히 하나로만 매핑하라. 절대 다른 텍스트는 작성하지 마라.

A) NETWORK_TIMEOUT
B) AUTH_TOKEN_EXPIRED
C) SYNTAX_ERROR
D) UNKNOWN_ANOMALY

결괏값 포맷: <class>선택된 알파벳 단 하나</class>

이 기법을 사용하면 오버피팅 된 머신러닝의 텍스트 분류기(Text Classifier) 수준에 버금가는 99% 이상의 동일 답변 재현성(Reproducibility)을 손쉽게 얻어낼 수 있다.
