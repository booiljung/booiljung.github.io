4.8.2 출력 포맷 강제를 위한 마크다운(Markdown) 및 태그 활용 지침

자연어로 “결과를 이쁘게 표 형식으로 작성해 줘“라고 말하면 모델은 가끔 마크다운으로, 가끔은 일반 띄어쓰기로, 가끔은 HTML로 결과물을 반환할 것이다. 텍스트 생성의 일관성을 확보하려면 표준 포맷팅 언어의 뼈대를 프롬프트에 주입하여 구조적 탈선을 막아야 한다.

오라클에 가장 많이 쓰이는 두 가지 포맷 통제법은 다음과 같다.

  1. 마크다운 코드 블록 통제: 
    답변은 반드시 아래와 같이 마크다운 json 블록으로 감싸야 한다:

{ ... }

   이렇게 강제하면 후속 스크립트에서 정규식 ` ```json(.*?)``` ` 로 100% 안전하게 본문을 파싱해낼 수 있다.
2. **XML/HTML 커스텀 태그 통제**:
   JSON이 아니라 부분 문자열을 검증해야 한다면 `<answer>FAIL</answer>` 처럼 가상의 태그 문법을 강제하라. LLM은 코딩 데이터셋으로 대량 학습되었기 때문에 여는 태그를 작성하면 닫는 태그를 기계적으로 생성하려는 강력한 코딩 본능이 발동하여 완벽한 구조적 출력을 담보한다.
