4.12.2 DSPy 개요: 시그니처(Signature)와 모듈(Module) 기반의 로직 분리

DSPy(Demonstrate-Search-Predict) 라이브러리는 프로그램 로직에서 프롬프트의 텍스트를 완벽하게 분리해 낸다. 핵심 요소는 모듈과 시그니처다.

  1. 시그니처 (Signature):
    함수의 타입 힌트와 유사하게 입출력 유형만 간결하게 선언한다.
    "code_snippet, policy_text -> evaluation_result"
    여기에는 구차한 시스템 메시지나 “안녕 나는 어시스턴트야” 따위의 글이 존재하지 않는다. 오직 스펙 명세만 남는다.
  2. 모듈 (Module):
    신경망을 연결하는 블록 단위다. 예를 들어 dspy.ChainOfThought 모듈을 선언하면, 내부적으로 Let's think step by step 논리나 <thought> 태그 관련 지시사항을 엔진이 알아서 적절한 위치에 주입해 준다.
import dspy

# 1. 시그니처(입출력 명세) 정의만으로 오라클 로직을 작성
class OracleValidator(dspy.Signature):
    """주어진 정책(policy)을 바탕으로 코드(code)의 결함을 판단하라."""
    policy = dspy.InputField()
    code = dspy.InputField()
    verdict = dspy.OutputField(desc="PASS, FAIL, UNKNOWN 중 하나")

# 2. CoT 모듈과 결합
oracle_pipeline = dspy.ChainOfThought(OracleValidator)

이 코드는 놀랍게도 그 어떤 마크다운 지시문이나 감정 배제 프롬프트 하드코딩 없이도, 내부적으로 수백 자에 달하는 최적의 오라클 시스템 프롬프트로 알아서 번역되어 모델에 전달된다.
