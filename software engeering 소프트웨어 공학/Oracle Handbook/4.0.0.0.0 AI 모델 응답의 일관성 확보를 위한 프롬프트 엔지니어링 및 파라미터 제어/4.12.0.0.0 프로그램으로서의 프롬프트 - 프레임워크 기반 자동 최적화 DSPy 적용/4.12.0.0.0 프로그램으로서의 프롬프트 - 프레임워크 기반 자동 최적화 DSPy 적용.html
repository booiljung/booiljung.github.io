지금까지 배운 퓨샷(Few-shot) 템플릿과 CoT 구성 기술은 모두 인간 언어(자연어)에 기반한 하드코딩(Hard-coding) 엔지니어링이다. 그러나 모델 버전이 바뀔 때마다 사람이 일일이 문맥을 수정하고 띄어쓰기를 실험하며 A/B 테스트를 하는 것은 지속 가능하지 않다.

본 절에서는 스탠퍼드 연구진이 제안한 혁명적인 패러다임 전환이자, 프롬프트를 컴파일(Compile) 가능한 소프트웨어 객체로 승격시킨 프레임워크인 DSPy(Declarative Self-Improving Language Programs) 가 어떻게 오라클 세계관의 정점에 안착하는지 논의한다.
