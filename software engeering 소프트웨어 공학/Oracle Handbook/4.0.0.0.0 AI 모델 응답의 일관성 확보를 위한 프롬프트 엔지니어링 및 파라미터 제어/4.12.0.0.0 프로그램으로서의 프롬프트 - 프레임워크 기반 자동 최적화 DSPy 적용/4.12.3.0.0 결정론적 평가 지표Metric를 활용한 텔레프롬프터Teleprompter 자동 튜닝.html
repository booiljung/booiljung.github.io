4.12.3 결정론적 평가 지표(Metric)를 활용한 텔레프롬프터(Teleprompter) 자동 튜닝

DSPy의 진정한 마법은 텔레프롬프터(Teleprompter) 라는 이름의 옵티마이저(Optimizer) 기능에서 발휘된다.

개발자가 해야 할 일은 백일장처럼 프롬프트를 깎는 것이 아니라, 오직 다음과 같은 요소만 준비하는 것이다.

  • 정답지 스펙(Golden Dataset): (Input Code, Output=FAIL) 세트 50개.
  • 평가 함수(Metric): AI의 결괏값이 내가 정의한 정답(FAIL)과 정확히 일치하는지(ExactMatch) 평가하는 파이썬 함수 하나.

이 두 가지를 텔레프롬프터인 BootstrapFewShot 엔진에 집어넣고 ‘컴파일(Compile)’ 버튼을 누르면, DSPy 엔진이 스스로 자신이 설정한 프롬프트를 조금씩 바꾸거나, 50개의 데이터셋 중 모델의 어텐션에 가장 깊게 반응하는 엣지 케이스 3개를 ’퓨샷 예제(Few-shot examples)’로 동적 선발하고 조합하여 수만 번의 시뮬레이션을 돌린다.

결과적으로 목표한 MetricScore(정답 일치율률 100%)를 달성하는 최적의 프롬프트와 퓨샷 예제 조합을 기계가 자동으로 찾아내어 내부에 캐싱한다. 인간은 더 이상 예제 순서가 미치는 영향을 고민할 필요가 없어지는 것이다.
