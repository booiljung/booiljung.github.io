4.1 확률적 AI와 결정론적 소프트웨어의 충돌

소프트웨어 엔지니어링의 근간은 ’제어 가능성(Controllability)’과 ’예측 가능성(Predictability)’에 있다. 코드는 개발자가 작성한 규칙 체계 내에서만 동작해야 하며, 입력이 주어지면 항상 동일한 출력을 반환해야 한다는 대전제를 바탕으로 수십 년간 진화해 왔다.

그러나 생성형 AI(Generative AI)의 도입은 이러한 전통적인 공학적 토대를 송두리째 흔들어 놓았다. AI 모델은 고정된 규칙이 아닌 확률 분포(Probability Distribution)를 기반으로 작동하기 때문이다.

본 4.1절에서는 태생적으로 비결정적(Nondeterministic)인 AI 모델이 왜 기존의 멱등성(Idempotency)을 요구하는 IT 인프라와 근본적인 충돌을 일으키는지 그 기저 원인을 심층 분석한다. 나아가, 이러한 불확실성 속에서도 어떻게 ’공학적 통제권’을 되찾아 올 수 있을지에 대한 거시적인 해결 전략을 조망할 것이다.
