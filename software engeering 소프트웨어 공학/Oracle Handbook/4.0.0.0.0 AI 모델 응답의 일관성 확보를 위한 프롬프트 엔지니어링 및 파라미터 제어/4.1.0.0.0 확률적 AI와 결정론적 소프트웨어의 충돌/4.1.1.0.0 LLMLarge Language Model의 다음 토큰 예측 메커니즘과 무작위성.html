4.1.1 LLM(Large Language Model)의 다음 토큰 예측 메커니즘과 무작위성

거대 언어 모델(LLM)이 인간처럼 유려한 글을 써 내려가는 마법의 본질은 사실 극도로 단순한 통계적 연산이다. LLM은 주어진 컨텍스트 x_{1}, x_{2}, \dots, x_{n}이 주어졌을 때, 그다음에 올 토큰 x_{n+1}의 확률 분포 P(x_{n+1} \mid x_{1}, \dots, x_{n})를 계산하는 ’다음 토큰 예측기(Next Token Predictor)’일 뿐이다.

1. 무작위성(Randomness)의 개입 지점

문제는 확률 분포가 계산된 직후, 모델이 가장 확률이 높은(정답에 가까운) 토큰 하나만을 기계적으로 취하지 않는다는 점이다.

  1. 소프트맥스(Softmax)와 확률 분포:
    모델의 마지막 층을 통과한 로짓(Logit) 값들은 Softmax 함수를 거쳐 모든 어휘(Vocabulary)에 대한 0부터 1 사이의 확률값으로 변환된다. 예를 들어, “The quick brown” 다음에 “fox“가 올 확률이 0.8, “dog“가 올 확률이 0.2로 계산된다.
  2. 샘플링 프로세스와 주사위 굴리기:
    LLM은 항상 “fox“를 선택하지 않는다. 내부의 샘플링 알고리즘(Sampling Algorithm)은 이 확률표를 가지고 일종의 ‘주사위를 굴려’ 토큰을 뽑는다. 20%의 확률로 “dog“가 선택될 여지가 의도적으로 열려 있는 것이다.

이러한 무작위성은 AI가 매번 똑같은 기계적 답변 대신, 시 쓰는 시인처럼 다양하고 창의적인 문장을 구사하게 만드는 핵심 원동력이다. 하지만 코드를 검토하고 비즈니스 룰을 판별하는 ’오라클’의 세계에서는, 이 찬란한 창의성이 곧 시스템의 근간을 파괴하는 치명적인 버그(Bug)가 된다. 일관성이 결여된 무작위성은 검증과 신뢰를 원천적으로 불가능하게 만든다.
