4.4.1 시스템 메시지 vs 사용자 메시지의 가중치 차이 이해

왜 모든 지시사항을 User 메시지에 때려 넣지 않고 System 메시지로 분리해야 하는가? 이는 모델 아키텍처 내부의 어텐션(Attention) 가중치와 맞닿아 있다.

  • User Message: LLM은 이를 ’해석하고 반응해야 할 일회성 대상’으로 취급한다. 대화가 길어지거나 복잡한 텍스트가 입력되면, User 메시지 초반에 적어둔 규칙(“짧게 대답해”)은 컨텍스트 윈도우(Context Window) 속에서 서서히 잊혀지거나(Attention Decay) 사용자 측의 새로운 요구사항에 의해 덮어씌워진다.
  • System Message: 모델의 베이스라인 행동 지침이다. 최신 모델들은 시스템 프롬프트 영역의 지시사항이 이후 제기되는 어떠한 사용자 메시지에도 절대 양보되지 않도록(System Prompt Steering) 매우 높은 어텐션 가중치를 부여받도록 튜닝되어 있다.

따라서 변하지 않는 오라클의 평가 규칙(출력 스키마, 감정 배제, 전문 검수자 역할 등)은 무조건 System 메시지 블록 상단에 콘크리트처럼 타설해 두어야 한다.
