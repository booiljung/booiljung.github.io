4.4.2.2 감정적 어조(Emotional Tone) 배제와 사실 기반(Fact-based) 지시문 작성법

기본적으로 LLM(특히 ChatGPT 기반 모델들)은 사용자를 즐겁게 하기 위해 훈련(RLHF)되어 있으므로 “네, 맞습니다!”, “도움이 되어 기쁩니다” 같은 감정적 꼬리표를 붙이는 습성이 있다. 이는 JSON 파서나 Assert 함수에 치명타를 입힌다.

이를 제동 걸기 위해서는 감정적 발화를 원천 금지하는 문장을 박아 넣어야 한다.

감정 배제 안티-패턴 vs 베스트 프랙티스:

  • ❌ 안티-패턴: “필요 없는 말은 하지 말고 결과만 말해줘.” (모델은 ’필요 없는 말’의 기준을 주관적으로 해석한다.)
  • ✅ 베스트 프랙티스: “어떠한 인사말, 사과, 부연 설명, 감정적 표현도 출력하지 마라. 오직 제공된 JSON 스키마 구조의 문자열만 단독으로 반환하라.”

이처럼 금지할 구체적 행위(인사, 설명, 사과)를 나열함으로써 사실(Fact)로만 이루어진 기계적 답변을 이끌어 낼 수 있다.
