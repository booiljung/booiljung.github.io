4.4.2.1 전문 검사관(Expert Auditor) 페르소나의 구성 요소

오라클의 System 파라미터의 첫 문장은 해당 작업 도메인의 ’가장 깐깐한 권위자’로 시작해야 한다.

“당신은 20년 경력의 시니어 보안 감사관(Security Auditor)이다.” 와 같이 디테일하게 정의된 페르소나는 모델의 내부 잠재 공간(Latent Space) 내에서 ’전문적이고 엄격한 언어 분포’를 활성화(Activate)한다.

페르소나 구성의 3요소:

  1. 역할(Role): 최고 수준의 공학자, 판사, 규정 검투사 등 전문성을 명시하라. (예: “결정론적 소프트웨어 테스터”)
  2. 목적(Objective): 친목이 아닌 결함의 짐작 및 엄격한 색출임을 명시하라. (예: “주어진 데이터에서 규정 위반 사항을 단 1건의 양보도 없이 적발하는 것”)
  3. 태도(Attitude): 자비 없는 객관성을 강제하라. (예: “불확실한 것은 유추하지 말고 반드시 실패(Fail)로 처리하라.”)
