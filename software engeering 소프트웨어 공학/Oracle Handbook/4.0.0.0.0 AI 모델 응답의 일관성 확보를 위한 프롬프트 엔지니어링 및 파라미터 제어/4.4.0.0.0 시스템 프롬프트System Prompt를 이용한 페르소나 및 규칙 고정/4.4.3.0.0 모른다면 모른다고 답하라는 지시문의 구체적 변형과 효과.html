4.4.3 “모른다면 모른다고 답하라“는 지시문의 구체적 변형과 효과

AI의 가장 무서운 적은 오답이 아니라, 확신에 찬 거짓말인 ’환각(Hallucination)’이다. 오라클 시스템이 테스트 데이터를 평가하다가 문맥이 모호하여 결과를 판단할 수 없을 때, 무리하게 ’PASS’를 유추해서는 안 된다.

이를 방어하기 위한 방어적 프롬프팅 요령은 다음과 같다.

  1. 단순한 방어 (Basic Guardrail):
    “제시된 원문만으로 사실 관계 검증이 불가능한 경우 절대 지어내지 말고 ’UNKNOWN’을 반환하라.”
  2. 패널티 기반 방어 (Penalty-driven):
    “근거가 명확하지 않은 추론을 진실로 보고하는 것은 가장 치명적인 실패(Critical Failure)입니다. 정보가 누락된 경우 즉각 ‘INSUFFICIENT_DATA’ 예외 단어를 출력하십시오.”
  3. 인용구 강제 (Citation Forcing):
    “판단의 근거가 된 텍스트를 반드시 <quote> 태그 안에 원문 그대로 인용하라. 인용할 수 없다면 ’NULL’을 반환하라.”

이 구문들은 모델이 스스로의 확률적 불확실성을 감지했을 때, 억지로 소설을 쓰는 대신 당당하게 ‘판독 불가’ 경로로 스레드를 빠져나오게(Exit) 하는 우회로(Escape Hatch) 역할을 한다.
