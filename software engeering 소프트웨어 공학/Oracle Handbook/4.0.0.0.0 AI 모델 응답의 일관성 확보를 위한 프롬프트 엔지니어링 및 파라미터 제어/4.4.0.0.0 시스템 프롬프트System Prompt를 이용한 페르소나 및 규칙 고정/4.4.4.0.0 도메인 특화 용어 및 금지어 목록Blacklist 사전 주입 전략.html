4.4.4 도메인 특화 용어 및 금지어 목록(Blacklist) 사전 주입 전략

개발 조직 내부에서만 사용하는 특수한 약어나 도메인 용어(Domain Specific Language)는 일반 코퍼스로 학습된 LLM에게는 노이즈(Noise)다. 테스트 오라클이 내부망의 전문 지식을 다룬다면, 시스템 프롬프트에 간단한 ’사전(Glossary)’과 ’블랙리스트’를 미리 박아두어 컨텍스트 혼동을 막아야 한다.

1. Glossary & Blacklist 프롬프트 예시

# 용어사전 (Glossary)
- "V1_App": 모바일 클라이언트 애플리케이션의 레거시 버전을 뜻한다.
- "G-Tier": 무료 사용자 등급을 의미한다.

# 금지어 (Blacklist)
다음 단어들은 출력에 절대 포함시키지 마라:
- "어쩌면", "아마도", "가능성이 있습니다" (불확실성을 나타내는 언어)
- "Client" (대신 반드시 "Customer"를 사용할 것)

이 규칙은 파서(Parser)가 Client라는 단어를 정규식으로 잡지 못해 테스트가 미탐/오탐되는 결함을 어휘 생성 단계에서 미리 잘라내는 효과적인 제어망이다.
