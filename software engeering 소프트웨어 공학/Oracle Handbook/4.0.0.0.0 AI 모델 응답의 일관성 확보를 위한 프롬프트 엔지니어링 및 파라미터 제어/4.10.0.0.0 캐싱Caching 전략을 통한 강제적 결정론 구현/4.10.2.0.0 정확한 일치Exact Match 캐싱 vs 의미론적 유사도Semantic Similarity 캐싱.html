4.10.2 정확한 일치(Exact Match) 캐싱 vs 의미론적 유사도(Semantic Similarity) 캐싱

캐시키(Cache Key)를 무엇으로 잡을 것인가에 따라 시스템의 방어력과 유연성이 크게 갈린다.

  1. 정확한 일치 (Exact Match) 기반 해싱:
    프롬프트 내용 전체를 SHA-256 등으로 해싱(Hashing)하여 저장한다. 단어 하나, 줄바꿈 하나만 달라도 다른 캐시로 인식한다 (Cache Miss).
  • 장점: 100% 안전. 절대적인 기계적 멱등성 보장. 회귀 테스트의 오라클 캐싱에 가장 적합.
  • 단점: 사용자가 로그 맨 뒤에 타임스탬프를 1초 다르게 주입하면 매번 과금이 발생함.
  1. 의미론적 유사도 (Semantic Similarity) 기반 벡터 검색:
    프롬프트를 임베딩 벡터로 변환하여 Vector DB(예: Pinecone, Milvus)에 저장한 뒤, 유클리드 거리가 0.98 이상(즉 뜻이 매우 유사함)이면 같은 질문으로 간주하고 캐시를 반환한다.
  • 장점: “코드가 이상해“와 “코드 문제점 찾아줘“를 동일 문제로 간주하여 엄청난 비용 절감 가능.
  • 단점: 엄격한 오라클 판정에서 미세한 엣지 케이스를 놓칠 위험이 있어 ‘자동화 검증’ 파이프라인에서는 지양해야 함.
