4.10.3 캐시 만료 정책과 모델 재검증 주기 설정

오라클 환경에서 캐시를 너무 오랫동안 쥐고 있으면 이 또한 버그가 된다. “V1 보안 정책” 하에서 PASS로 판정받고 캐싱된 결과가, “V2 보안 정책“이 공표된 6개월 후에도 프록시 계층에서 PASS로 자동 반환되면 최악의 취약점을 양산한다.

오라클 캐시 TTL (Time-To-Live) 모범 사례:

  • 시스템 프롬프트(오라클 룰)의 버전 해시값이 변경되는 즉시 관련 캐시 공간 전체를 Flush(초기화) 해야 한다.
  • 하루 1회 배치(Batch)로 TTL을 만료시켜, 캐시 오염을 막고 매일 새롭게 변화하는 인프라 룰에 적응할 수 있는 재검증(Re-validation) 창구를 열어주어야 한다.
