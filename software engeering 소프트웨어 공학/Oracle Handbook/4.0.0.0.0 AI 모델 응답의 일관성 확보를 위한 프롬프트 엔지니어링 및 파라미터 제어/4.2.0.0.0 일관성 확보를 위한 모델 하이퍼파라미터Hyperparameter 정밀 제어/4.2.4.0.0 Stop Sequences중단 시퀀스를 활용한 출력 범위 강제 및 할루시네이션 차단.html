4.2.4 Stop Sequences(중단 시퀀스)를 활용한 출력 범위 강제 및 할루시네이션 차단

Stop Sequences는 LLM 출력 스트림에 끼어드는 가장 폭력적이고 확실한 개입 수단이다. 개발자가 지정한 특정 문자열 토큰 리스트(예: ["\n\n", "User:", "END_OF_JSON"]) 배열이 등장하는 즉시 무조건 모델의 펜을 강제로 부수고 생성을 중단시킨다.

1. 결정론적 테스팅에서의 Stop Sequence 무기화

오라클 환경에서는 모델이 묻지 않은 말에 장황하게 대답하는 것(Chatty) 자체가 파서를 망가뜨리는 치명적인 결함이다. 이를 제어하기 위해 Stop Sequences를 치밀하게 세팅해야 한다.

  1. 마크다운 탈주 방지:
    예를 들어 <result>PASS</result> 포맷으로만 정답을 추출하고 싶다면, </result> 기운을 풍기는 즉시 답변 생성을 끊어버려라. 뒤에 이어질지 모르는 “이 테스트를 통과한 이유는 다음과 같습니다…” 식의 과설명(Over-explanation) 및 환각(Hallucination) 리스크를 원천 봉쇄할 수 있다.
  2. 반복 출력 및 무한 루프 절단:
    잘못된 프롬프트나 온도 0의 부작용으로 인해 {"status":"OK"}를 수만 자에 걸쳐 무한 루프하며 내뱉는 장애가 존재한다. 이때 JSON 닫기 마크 }나 마크다운 코드 블록 닫기인 ```를 Stop Sequence로 등록하면 API 호출의 데드락을 방어할 수 있다.
