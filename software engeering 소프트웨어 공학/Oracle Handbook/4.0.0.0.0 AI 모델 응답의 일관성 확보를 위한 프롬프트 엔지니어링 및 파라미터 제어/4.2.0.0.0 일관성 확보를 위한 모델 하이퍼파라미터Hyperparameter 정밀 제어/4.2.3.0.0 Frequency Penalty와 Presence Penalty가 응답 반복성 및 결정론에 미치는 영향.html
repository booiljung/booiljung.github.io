4.2.3 Frequency Penalty와 Presence Penalty가 응답 반복성 및 결정론에 미치는 영향

이 두 파라미터는 이미 모델이 내뱉은 텍스트(History)에 페널티를 부여하여 ’어휘의 다양성’을 자극하는 장치다.

  • Frequency Penalty: 빈도 페널티. 특정 토큰이 10번 등장했다면 1번 등장한 경우보다 더 무거운 페널티를 매긴다. (동일 단어의 지나친 반복 사용 억제)
  • Presence Penalty: 존재 페널티. 횟수와 상관없이 특정 토큰이 한 번이라도 등장했다면 일정량의 페널티를 매겨 완전히 새로운 화두를 던지도록 유도한다.

창의적 글쓰기나 챗봇 대화에서는 이러한 페널티 값이 긍정적이지만, 정밀한 JSON 생성이나 테스트 결과 검증 오라클에서는 독약과 다름없다.

결정론적 오라클은 “성공”, “실패”, “값”, “필드” 같은 특정 예약어나 포맷 문자가 수백 번 반복되어 포맷팅되는 것을 두려워하지 않아야 한다. 만약 Frequency Penalty가 양수로 들어가 있다면, JSON 파싱을 위해 20번 등장한 "result": 라는 키(Key)가 갑자기 "outcome": 혹은 "status": 로 무작위 변경될 확률이 폭증한다.

결론: 완벽한 스키마 일관성과 기계적인 정답지 출력을 강제하려면, 두 페널티 값을 언제나 0.0 (Neutral) 으로 끈질기게 유지해야 한다.
