4.2.6 Logit Bias 조작: 특정 토큰의 생성 확률 강제 배제 및 유도

만약 오라클 개발자에게 주어질 수 있는 가장 잔인하고 극단적인 무기 하나를 꼽으라면, 그것은 바로 Logit Bias 맵(Map) 조작이다.

이는 단순히 자연어로 지시하는 것을 넘어, 언어 모델의 뇌 속에 핀셋을 찔러 넣어 특정 단어 자체를 기억상실(Amnesia) 상태로 만들거나, 강박증 수준으로 집착하게 만드는 하드코어 조작 기법이다.

1. 편향 맵(Bias Map)의 마법

API 요청 시 {"logit_bias": {"50256": -100}} 처럼 토큰 ID와 가중치값(-100 ~ 100) 쌍을 주입한다.

  • 절대 불가(Ban) 설정: 페널티를 -100 (Max Negative)에 가깝게 부여하면, 이 토큰의 로짓 확률은 마이너스 무한대 극한으로 떨어져, 모델이 이 단어를 뱉어낼 확률은 수학적으로 0%가 됨을 의미한다. 만약 챗봇이 “ 죄송하지만“ 이라는 거절 문구를 남발하는 것이 지긋지긋하다면 죄, 송 이라는 토큰 자체를 밴(Ban) 해버리는 식이다.
  • 강제 발화 유도: 반대로 +100 페널티를 주면 그 단어만 무한 반복하는 기괴망측한 앵무새가 되어버리므로 조심해야 하지만, +5에서 +10 사이의 적절한 양수 바이어스를 채워주면 모델이 JSON 키워드("result", "score")나 승인 문구(PASS)를 우선순위의 윗선으로 치켜세워 구문 오류 빈도를 획기적으로 줄여준다.

다만 BPE(Byte-Pair Encoding) 토크나이저 체계에서 띄어쓰기(예: PASS 와 PASS)에 따라 ID가 다르게 파싱되므로, 치밀하게 사전 토큰 ID 조사를 거친 후에 주입해야 오작동을 막을 수 있다.
