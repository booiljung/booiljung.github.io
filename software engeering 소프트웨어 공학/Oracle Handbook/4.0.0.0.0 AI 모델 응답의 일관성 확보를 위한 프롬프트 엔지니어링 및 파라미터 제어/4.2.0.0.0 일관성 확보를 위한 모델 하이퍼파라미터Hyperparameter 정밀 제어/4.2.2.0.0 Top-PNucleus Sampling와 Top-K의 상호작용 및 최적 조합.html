4.2.2 Top-P(Nucleus Sampling)와 Top-K의 상호작용 및 최적 조합

Temperature가 후보 토큰 간의 ’확률 격차(간격)’를 벌려 놓는다면, Top-P와 Top-K는 아예 후보 토큰들을 ‘잘라내는(Cutting)’ 역할을 수행한다.

  1. Top-K (랭킹 기반 컷오프):
    가장 원시적인 필터링이다. 확률에 상관없이 Top-K=10으로 설정하면, 로짓 상위 1위부터 10위까지만 남기고 11위 이하는 후보군에서 배제한다. 후보가 균등하게 경쟁하든, 1등이 압도적이든 무조건 10개만 남긴다.
  2. Top-P (누적 확률 기반 컷오프):
    이른바 코어 샘플링(Nucleus Sampling)이라 불리며, 더 지능적인 접근법이다. Top-P=0.9로 설정하면 상위 1등부터 누적 확률 합이 90%가 될 때까지만 토큰 후보를 남긴다. 만약 1등 혼자 90% 확률을 차지한다면 1개만 남고, 반대로 고만고만한 토큰들이 분산되어 있다면 수십 개가 살아남는다. (맥락에 따른 동적인 컷오프)

1. 오라클을 위한 파라미터 조합 (Best Practice)

결정론적 단일 답을 요구하는 오라클 시스템을 구축할 때는 가변성을 극도로 옥죄어야 한다. 보편적인 권장 조합은 다음과 같다.

  • Temperature=0을 사용할 경우, 로직상 가장 1등만 선택하므로 Top-P와 Top-K의 수치는 무의미해진다(Ignored).
  • 그러나 만약 추론의 ’스텝 논리성’이 온도 0에서 심하게 저하되어 불가피하게 Temperature=0.1~0.2로 살려두기로 했다면, 반드시 Top-P=0.1 같이 극단적인 누적 컷오프를 동시에 걸어줘야 한다. 이는 상위 소수의 엘리트 집단 내에서만 주사위를 굴리도록 통제하여 답변의 궤적 이탈을 물리적으로 방지한다.
