4.2.1.1 Temperature 0이 완전한 결정론(Determinism)을 보장하지 못하는 이유

초보 엔지니어들은 API 파라미터에 {"temperature": 0.0}을 넣고 나서 모든 것이 해결되었다며 안심한다. 그러나 현실에서는 동일한 온도 0의 조건에서도 간헐적으로 다른 답변이 반환된다. 이것은 버그가 아니라 최신 LLM 아키텍처의 한계다.

  1. 병렬 연산과 하드웨어 노이즈:
    A100, H100과 같은 거대 GPU 클러스터 내부에서 CUDA 커널의 병렬 실행 순서에 미세한 차이가 발생하고, 부동소수점 오차(Floating Point Error)가 누적되며 로짓(Logit) 계산의 끝자리 값이 흔들린다.
  2. MoE(Mixture of Experts) 라우팅 변동성:
    최첨단 모델(예: GPT-4)은 수십 개의 서브 모델(Expert) 중 입력 토큰을 어느 쪽으로 라우팅할지 실시간으로 결정한다. 네트워크 상태, 서버 로드, 배치(Batch) 사이징에 따라 이 경로가 미세하게 달라질 수 있으며, 로짓 1등과 2등의 격차가 소수점 단위로 박빙일 때 선택이 뒤집히는 결과를 낳는다.

그렇기에 Temperature 0은 결코 ’절대 반지’가 아니다. 이것을 출발선으로 놓고 추가적인 파라미터 잠금과 프롬프트 규제를 겹겹이 두르는 방어망 설계가 뒤따라야만 한다.
