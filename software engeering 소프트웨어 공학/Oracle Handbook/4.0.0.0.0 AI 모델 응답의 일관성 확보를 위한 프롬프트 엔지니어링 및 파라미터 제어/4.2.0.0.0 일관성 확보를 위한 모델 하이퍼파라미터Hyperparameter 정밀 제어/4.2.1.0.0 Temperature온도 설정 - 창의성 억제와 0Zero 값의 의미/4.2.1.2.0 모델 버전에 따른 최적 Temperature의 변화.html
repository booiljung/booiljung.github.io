4.2.1.2 모델 버전에 따른 최적 Temperature의 변화

“GPT-3.5 에서는 Temperature=0이 최선이었는데, 왜 Llama 3이나 Claude 3.5 Sonnet에서는 오히려 빈약한 응답이 나오는가?”

이러한 현상은 각 벤더별 모델의 얼라인먼트(Alignment) 방식, 특히 RLHF(Reinforcement Learning from Human Feedback)의 패널티 적용 강도가 다르기 때문이다.

어떤 하위 오픈소스 모델들의 경우, Temperature=0을 주입하면 가장 흔하고 진부한 ’안전 편향(Safe Bias)’에 갇혀버린다. 예를 들어 “내부 오류가 발생했습니다“와 같은 지배적인 1순위 토큰에서 빠져나오지 못하고 무한 루프에 빠지는 현상이다.

반면 최신 메이저 모델일수록 Temperature=0.0 (또는 심지어 0.1)이 추론 능력 저하 없이 정답성을 높이도록 잘 튜닝되어 있다.
결론적으로 오라클 구축을 위해 무지성으로 온도 0을 때려 넣기보다는, 모델의 버전을 고정(Pinning)한 채 T={0.0, 0.1, 0.2} 사이에서 자동화 벤치마크를 돌려 해당 버전에 맞는 ‘건조함(Dryness)’ 스윗스팟을 찾아내야 한다.
