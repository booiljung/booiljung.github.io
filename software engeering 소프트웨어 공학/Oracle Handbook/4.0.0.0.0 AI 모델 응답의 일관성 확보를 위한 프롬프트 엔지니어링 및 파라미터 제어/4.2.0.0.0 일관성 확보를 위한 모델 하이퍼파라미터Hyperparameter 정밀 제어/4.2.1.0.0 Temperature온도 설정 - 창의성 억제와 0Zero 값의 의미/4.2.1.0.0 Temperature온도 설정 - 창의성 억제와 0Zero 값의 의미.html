4.2.1 Temperature(온도) 설정: 창의성 억제와 0(Zero) 값의 의미

Temperature는 소프트맥스(Softmax) 함수의 출력에서 각 토큰 확률의 ’격차’를 조절하는 역할을 한다. 수식이 p_i = \frac{exp(x_i / T)}{\sum exp(x_j / T)} 로 정의되기 때문에, 온도 T가 낮아질수록 확률 격차는 극단적으로 벌어진다.

  • Temperature = 1 (Default): 모델이 계산한 본래의 확률 분포를 그대로 보존한다.
  • Temperature > 1 (예: 1.5): 확률 분포를 평탄하게 깎아내어(Flatten), 본래라면 1%도 선택되지 못할 희귀 토큰들의 등장 확률을 끌어올린다. 소설을 쓰거나 브레인스토밍을 할 때 유용하다.
  • Temperature = 0: 이론적으로 Argmax 함수처럼 동작한다. 즉, 주관적으로 단 0.01%라도 확률이 더 높은 1등 토큰이 존재한다면, 나머지 2등부터 10만 등까지의 구토큰들을 무조건 배제하고 100% 선택하게 된다.

결정론적 오라클 시스템을 구축할 때 Temperature = 0은 선택이 아닌 필수사항이다. 창의성은 테스트 케이스에서 ’가장 배척해야 할 버그’이기 때문이다.
