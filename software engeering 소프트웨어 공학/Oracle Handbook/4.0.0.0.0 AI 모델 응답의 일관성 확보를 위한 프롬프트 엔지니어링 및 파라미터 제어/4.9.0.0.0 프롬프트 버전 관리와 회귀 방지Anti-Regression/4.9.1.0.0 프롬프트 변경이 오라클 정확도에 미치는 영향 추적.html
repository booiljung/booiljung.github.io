4.9.1 프롬프트 변경이 오라클 정확도에 미치는 영향 추적

“더 똑똑하게 평가해줘“라고 시스템 프롬프트를 튜닝(Tuning) 했더니, 그동안 문제없이 통과(PASS)하던 3,000개의 레거시 코드가 갑자기 FAIL로 뒤집히는 이른바 프롬프트 회귀(Prompt Regression) 는 AI 파이프라인의 일상적 재앙이다.

오라클의 정확도 수렴이 무너지지 않았는지 모니터링하기 위해서는, 프롬프트를 한 줄 수정한 뒤 이전 커밋(Commit)에서 사용했던 ’골든 테스트 세트(Golden Test Set, 고정된 입력과 기대되는 정답쌍 100개)’를 통과시켜야만 배포를 허용하는 Prompt CI 체계가 필요하다.
기존 정답률이 95%였는데, 프롬프트를 건드린 뒤 정답률이 80%로 깨진다면 그 변경은 치명적인 부작용(Side Effect)을 품고 있는 것이다.
