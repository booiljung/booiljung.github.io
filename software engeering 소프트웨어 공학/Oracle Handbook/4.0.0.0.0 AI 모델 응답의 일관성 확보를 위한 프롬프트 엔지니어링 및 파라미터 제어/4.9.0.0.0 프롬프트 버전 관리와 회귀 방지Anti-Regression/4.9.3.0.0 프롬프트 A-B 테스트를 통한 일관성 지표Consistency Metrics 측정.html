4.9.3 프롬프트 A/B 테스트를 통한 일관성 지표(Consistency Metrics) 측정

버전 1.0의 프롬프트와 버전 2.0 중 어느 것이 더 뛰어난 결정론적 방어선을 구축했는가를 어떻게 수치로 증명할 것인가? 감에 의존한 프롬프트 튜닝을 수학의 영역으로 끌고 들어와야 한다.

A/B 테스트를 수행할 때는 동일 조건 하에 100건의 복잡한 입력 데이터를 10번씩 던져본다(총 1,000회).

  • V1 (구 버전 프롬프트): 10번 중 7번은 PASS, 3번은 FAIL로, 답변 변동률(Variance)이 30%를 기록. (즉, 운이 나빠서 FAIL이 떨어진 플래키 테스트 유발자)
  • V2 (신 버전 프롬프트 + 제약조건 강화): 10번 모두 일관되게 FAIL 반환. 답변 변동률 0%.

이처럼 100%의 정답률 이전에, 먼저 0%의 답변 변동률(100% Consistency Metric) 을 달성하는 프롬프트를 찾아내는 것이 오라클 구축 A/B 테스트의 제일 목적이 되어야 한다.
