4.9.2 Git을 활용한 프롬프트 템플릿 버전 관리(Prompt-as-Code)

프롬프트는 본질적으로 ’AI 컴파일러를 위한 소스코드’다. 따라서 SaaS 툴의 UI 창에 메모해 두는 방식은 최악의 보안 및 정합성 리스크를 낳는다.

프롬프트 엔지니어링의 정석은 Prompt-as-Code (PaC) 파라다임이다.

  • 프롬프트의 지시 문자열 자체를 prompt_security_v1.0.txt 와 같이 Git Repository에 체크인(Check-in)한다.
  • 런타임에 사용하는 언어(예: Python, Go) 코드에서 파일 시스템이나 S3 스토리지에서 버전에 맞는 프롬프트 파일을 읽어 들여와(Load) 변수를 바인딩(Binding) 한다.
  • 누군가 프롬프트의 “설명을 제거해라” 지시문을 수정했다면, 반드시 PR(Pull Request) 리뷰를 거치고 CI 서버에서 골든 티켓 회귀 검증 스크립트가 돌아가도록 격리해야 한다.
