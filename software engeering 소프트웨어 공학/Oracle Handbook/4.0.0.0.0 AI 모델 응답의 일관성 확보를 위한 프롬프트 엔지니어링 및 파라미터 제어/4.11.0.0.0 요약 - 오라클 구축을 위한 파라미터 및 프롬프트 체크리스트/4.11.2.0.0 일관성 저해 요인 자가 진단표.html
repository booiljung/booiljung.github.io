4.11.2 일관성 저해 요인 자가 진단표

프롬프트 엔지니어링 검토(Code Review) 시 다음 사항들을 점검하라.

  1. 지시문의 모호성 (Ambiguity): “유도리 있게 검사해라“라는 주관적 언어가 포함되어 있는가? (-> “100ms를 초과하면 무조건 에러 발생으로 처리하라” 로 수정)
  2. 부전제어 패러독스 (Negative Constraint): “~하지 마라“로 규정된 룰이 있는가? (-> 오직 “~로만 답해라” 로 수정)
  3. 포맷의 누락 (Format Escape): 퓨샷 예제(Few-shot examples)의 출력 포맷이 본 마크다운 구조와 띄어쓰기 1칸까지 완벽하게 동일한가?
  4. CoT 부재 (No Reasoning): 수백 줄의 코드를 보고 단 1단어(PASS) 로만 답하게 되어 있어 직관적 비약이 심한가? (-> 분석 과정을 1줄 요약하게 강제)
