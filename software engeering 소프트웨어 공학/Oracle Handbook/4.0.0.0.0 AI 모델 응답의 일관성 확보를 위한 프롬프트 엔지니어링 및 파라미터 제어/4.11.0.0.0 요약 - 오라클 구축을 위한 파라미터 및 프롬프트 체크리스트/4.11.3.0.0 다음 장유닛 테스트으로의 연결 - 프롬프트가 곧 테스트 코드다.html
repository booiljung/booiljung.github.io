4.11.3 다음 장(유닛 테스트)으로의 연결: 프롬프트가 곧 테스트 코드다

지금까지 우리는 프롬프트를 ‘챗봇 조련용’ 언어가 아니라, 소프트웨어 컴파일러에게 던져주는 명세서(Specification) 그 자체로 취급해야 한다는 사실을 배웠다.

파라미터가 꽉 조여진 결정론적 모델과 정규화가 이루어진 프롬프트 템플릿의 결합은, 마침내 ’동작하는 오라클 컴포넌트(Working Oracle Component)’의 탄생을 의미한다. 이어지는 5장에서는 이렇게 기계화된 AI 오라클을 CI/CD 파이프라인의 핵심인 단위 테스트(Unit Test) 프레임워크(JUnit, PyTest)에 캡슐화하여, 사람의 개입 없이 매일 밤 수천 번씩 스스로 코드를 검증하는 실전 통합 시나리오로 넘어간다.
