4.7.2 입력 텍스트의 정규화(Normalization)를 통한 프롬프트 캐싱 적중률 향상

최근 Anthropic의 Claude 3.5나 OpenAI의 최신 API 아키텍처는 프롬프트 캐싱(Prompt Caching) 기능을 지원한다. 수천 자의 컨텍스트를 보낼 때, 기존에 캐싱된 토큰 접두사(Prefix)와 정확히 일치하면 모델 추론 비용을 90% 아끼고 응답 속도를 극대화할 수 있다.

그러나 캐싱의 기본 전조 조건은 ‘완벽한 문자열 일치(Exact Token Match)’ 다.

사용자가 시스템 프롬프트 중간의 주석 기호를 # 에서 // 로 살짝 바꿨을 뿐이거나, 들여쓰기(Indentation) 공간 4칸을 2칸으로 바꾼 것만으로도 토큰 시퀀스는 어긋나고 캐싱은 즉각 증발(Cache Miss)한다.
따라서, 평가하고자 하는 본문 데이터뿐만 아니라 이를 감싸고 있는 전체 프롬프트 템플릿의 문단 간격부터 줄바꿈 형식까지 린터(Linter) 수준으로 엄격하게 정규화(Normalization)하여 버전을 관리해야 한다.
