4.7.3 긴 문맥(Long Context)에서의 “Lost in the Middle” 현상과 중요 지시문의 위치 전략

LLM은 수만 토큰의 긴 문서를 읽을 때 사람과 똑같은 행동을 보인다. 즉, “시작 부분(Primacy Effect)“과 “끝 부분(Recency Effect)“은 잘 기억하지만, 중간에 끼어있는 정보는 망각해버린다. 이를 ‘Lost in the Middle’ 현상이라고 한다.

테스트 오라클로서의 가장 중요한 판단 기준(예: “경고 메시지가 나오면 무조건 FAIL 처리할 것”)을, 방대한 로그 데이터의 정중앙에 끼워 넣으면 모델은 높은 확률로 이 무거운 제약조건을 무시하고 환각을 생성한다.

배치 전략 (Sandwich Prompting):

  1. 상단 (Top): 페르소나, 출력 포맷 스키마, 가장 중요한 절대 규칙 1번을 배치한다.
  2. 중단 (Middle): 분석해야 할 대용량의 덤프 파일, 코드 조각, 방대한 테스트 로그 원문을 배치한다.
  3. 하단 (Bottom): 상단에 적었던 중요 규칙을 다른 단어로 짧게 다시 한번 요약(Reminder)하고, “자, 이제 위의 데이터를 바탕으로 답하라“는 최종 구문을 배치하여 Recency Bias를 극대화한다.
