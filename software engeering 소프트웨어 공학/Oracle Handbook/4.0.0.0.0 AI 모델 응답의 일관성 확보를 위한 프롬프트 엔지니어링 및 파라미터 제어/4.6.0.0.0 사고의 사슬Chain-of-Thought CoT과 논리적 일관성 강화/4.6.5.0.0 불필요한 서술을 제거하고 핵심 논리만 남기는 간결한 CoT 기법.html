4.6.5 불필요한 서술을 제거하고 핵심 논리만 남기는 간결한 CoT 기법

사고의 사슬(Chain-of-Thought)은 필연적으로 토큰(Token) 소모량을 급증시키며, 모델의 생성 속도(Latency)를 저하시킨다. 하루에 수만 건의 자동화 테스트 돌려야 하는 CI 파이프라인에서 지나치게 구어체적인 CoT 연산은 비용과 시간의 극심한 폭발을 부른다.

이를 제어하기 위한 대안이 Concise CoT (간결성 압축 추론) 이다.

비효율적 CoT 예시:

“먼저 사용자 코드를 살펴보겠습니다. 14번째 줄에서 변수를 확인합니다. 이 변수는 NULL일 가능성이 크군요. 에러가 발생하겠네요. 따라서 결론은 FAIL입니다.”

압축된 Concise CoT 예시:

“분석: L14 변수 선언 -> NULL 가능성 탐지 -> 런타임 에러 추론. 결과: FAIL.”

시스템 프롬프트 단계에서 “추론 과정은 완전한 문장(Full sentence)이 아닌, 간결한 명사구와 화살표(->) 기호만을 사용하여 짧게 요약하라.” 라고 선언함으로써, API 비용과 지연시간(Latency)을 절반 이상 단축하며 논리적 사슬의 효과는 그대로 유지할 수 있다.
