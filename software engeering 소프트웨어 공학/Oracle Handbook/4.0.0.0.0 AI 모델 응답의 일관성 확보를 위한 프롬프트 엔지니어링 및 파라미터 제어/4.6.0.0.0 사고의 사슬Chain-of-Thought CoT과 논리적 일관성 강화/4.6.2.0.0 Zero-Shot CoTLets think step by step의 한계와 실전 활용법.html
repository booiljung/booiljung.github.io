4.6.2 Zero-Shot CoT(“Let’s think step by step”)의 한계와 실전 활용법

코지마(Kojima) 외 연구진이 발견한 마법의 문구 “Let’s think step by step(단계별로 생각해 보자)” 은 제로샷(Zero-shot) 환경에서도 모델의 추론 성능을 비약적으로 끌어올리는 것으로 유명하다.

하지만 엔터프라이즈 오라클 환경에서 단순히 프롬프트 끝에 저 문장 하나를 붙이는 것은 위험하다.

1. Zero-shot CoT의 한계

모델이 자체적으로 ’단계별 계획’을 수립하게 방치하면, 그 단계의 구조와 포맷 자체가 비결정적(Nondeterministic)이 된다. A라는 테스트에는 3단계로 분석하고 JSON을 뱉어내다가, B 테스트에는 5단계로 분석한 뒤 마크다운(Markdown) 테이블을 그리는 등 결과 파싱을 불가능하게 만든다.

2. 실전 활용법: 제약 조건과의 결합

Zero-shot CoT를 사용할 경우, 반드시 중간 사고 과정의 위치와 포맷을 강제해야 한다.

분석을 시작하기 전, 반드시 <thought> 태그 열고 닫기 내부에 단계별 추론 과정을 작성하라. 
추론이 완료된 후에만 <result> 태그 안에 최종 값을 반환하라.

이러한 <thought> 숨김 기법은 후술할 데이터 캐싱 파싱 단계에서 정규식으로 논리 블록을 깔끔하게 발라내고 오직 <result> 블록만 취하게 하여 CoT의 이점과 파서의 안정성을 동시에 취할 수 있게 한다.
