4.3.2 OpenAI 및 주요 LLM API에서의 seed 파라미터 적용 방법

최근 OpenAI를 비롯한 메이저 API 제공자들은 이러한 결정론적 니즈를 수용하여 API에 명시적인 seed 매개변수를 도입했다.

다음은 OpenAI Chat Completions API에서 재현성을 확보하기 위한 대표적인 페이로드(Payload) 예시다.

{
  "model": "gpt-4-turbo",
  "messages": [{"role": "user", "content": "이 코드의 시간 복잡도는?"}],
  "temperature": 0.5,
  "seed": 12345
}

이와 같이 호출하면, 비록 Temperature가 0.5로 설정되어 일부 무작위성이 허용된 상태라 하더라도, seed: 12345가 내부 난수 발생기를 통제하기 때문에 이론적으로 몇 번을 호출하든 똑같은 응답이 반환된다. 창의적인 작풍(Tone)을 일정 수준 유지하면서도, 그 작풍 자체를 “고정(Freeze)“시켜서 유닛 테스트에서 Assertion 할 수 있게 만드는 강력한 무기다.
