4.3.4 Seed 설정에도 불구하고 발생하는 미세한 비결정성(Floating Point Non-determinism)의 한계

OpenAI는 공식 문서를 통해 “Seed 파라미터를 고정하더라도 100% 완벽하게 결정론적인 결과를 반환한다는 것을 보장하지 않는다(beta feature)“라고 경고한다. 이는 하드웨어 레벨의 병렬 연산 특성 때문이다.

GPU 내부의 수천 개 코어가 행렬 곱셈을 수행하면서 부분합(Partial Sum)을 더해갈 때, 도착 순서가 어긋나는 경우가 빈번하다. 부동 소수점(Floating Point) 덧셈은 교환법칙 (A+B = B+A)은 성립하지만 결합법칙 ((A+B)+C \neq A+(B+C))은 부동 소수점 반올림 오차로 인해 성립하지 않는다.

수백 레이어와 수백억 개의 파라미터를 통과하면서 이 미세한 오차가 누적되면, 최종 로짓(Logit) 스코어의 끝자리 숫자가 변하게 된다. 로짓 1등과 2등의 점수 차이가 극도로 박빙인 분기점에서 이 오차가 1등을 뒤집어버리면, 동일한 Seed라도 전혀 다른 문장 경로로 빠지게 되는 ’비결정성의 나비효과’가 발생한다.
