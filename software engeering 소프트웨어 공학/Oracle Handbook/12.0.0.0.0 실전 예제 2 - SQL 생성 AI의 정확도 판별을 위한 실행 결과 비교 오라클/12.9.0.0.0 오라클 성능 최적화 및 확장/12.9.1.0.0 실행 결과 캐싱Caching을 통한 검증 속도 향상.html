12.9.1 실행 결과 캐싱(Caching)을 통한 검증 속도 향상

기준 DB의 데이터 구조가 고정(Freeze)되어 있다면, 구문이 완전히 같은 Golden SQL을 매번 실행할 필요가 없다.

  • 파이썬 단위에서 SQL Hash -> Result Set Array Hash 를 Redis나 로컬 캐시 딕셔너리에 담아둔다.
  • 테스트를 런칭할 때 똑같은 정답 쿼리가 들어오면, DB까지 내려가지 않고 즉각적으로 In-memory 캐시에서 Golden Result를 꺼내온다. 이는 전체 CI 파이프라인의 속도를 30% 이상 끌어올리는 극적인 효과를 가져온다.
