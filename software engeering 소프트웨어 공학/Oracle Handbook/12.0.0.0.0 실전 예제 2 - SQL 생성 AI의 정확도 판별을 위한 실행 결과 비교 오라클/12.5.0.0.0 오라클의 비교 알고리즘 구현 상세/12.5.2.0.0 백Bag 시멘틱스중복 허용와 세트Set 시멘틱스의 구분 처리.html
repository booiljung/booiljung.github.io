12.5.2 백(Bag) 시멘틱스(중복 허용)와 세트(Set) 시멘틱스의 구분 처리

SQL의 본질적 난제 중 하나다.

  • SELECT DISTINCT 가 붙은 골든 쿼리는 Set(중복 불허)을 뜻하고, 명시되지 않으면 DB는 Bag(중복 허용) 시멘틱스를 따른다.
  • 만약 AI 커스텀 쿼리가 DISTINCT를 빠뜨려서 중복 데이터가 반환되었을 때 이를 틀렸다고 할 것인가?
  • 평가 정책: 보통은 엄격성(Strict Mode)을 디폴트로 하여, 중복된 행의 출현 횟수(Frequency)까지 완벽히 오라클과 일치(Bag Equality)해야 PASS로 판정하는 것이 금융권 및 엔터프라이즈의 표준이다.
