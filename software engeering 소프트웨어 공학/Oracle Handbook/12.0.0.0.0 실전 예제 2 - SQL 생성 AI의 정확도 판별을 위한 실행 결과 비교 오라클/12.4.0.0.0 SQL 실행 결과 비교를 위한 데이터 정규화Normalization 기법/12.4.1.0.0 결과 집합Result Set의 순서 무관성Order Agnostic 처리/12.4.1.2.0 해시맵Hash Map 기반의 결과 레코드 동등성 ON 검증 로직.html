12.4.1.2 해시맵(Hash Map) 기반의 결과 레코드 동등성 O(N) 검증 로직

거대한 테이블을 정렬(Sort)하는 비용은 O(N log N)이다. 속도가 중요한 CI에서는 이를 백 시맨틱스(Bag Semantics) 즉 다중 집합(Multiset)의 카운팅으로 처리해야 한다.

  • 각 로우(Row) 데이터를 통째로 직렬화하여 해시(Hash)값으로 만든다 (hash(str(row))).
  • 결과셋 A의 아이템별 출현 빈도를 카운팅한 해시 맵(Dict[Hash, int])을 만들고, 결과셋 B의 해시 맵과 딕셔너리 동등성 비교로 빠르고 정확하게 순서 무관성(Order Agnostic) 검증을 마무리한다. Time Complexity를 O(N)으로 방어하는 핵심 기술이다.
