12.4.1.1 ORDER BY 절 부재 시 발생하는 판정 모순 해결 방법

SQL 스펙상 ORDER BY가 명시되지 않은 쿼리는 DB 엔진에 따라 반환 순서를 보장하지 않는다.

  • 모순 발생: 두 쿼리가 동일한 레코드 3개 [A, B, C]를 반환했지만 런타임 A는 [A, B, C], 런타임 B는 [B, C, A]를 던질 경우 리스트 튜플로 매칭하면 FAIL이 뜬다.
  • 오라클 로직 방어:
  • AST(추상 구문 트리) 파서를 돌려서 원본 프롬프트나 Golden SQL에 ORDER BY 절이 명시되어 있는지 확인(Flag)한다.
  • 명시되어 있지 않다면, 두 파이썬 DataFrame (결과셋)을 불러들여 내부적으로 프라이머리 키(Primary Key)나 컬럼 전체를 기준으로 수동 정렬(df.sort_values())을 수행한 뒤 비교 연산으로 넘긴다.
