12.3.3 다중 정답(Multiple Valid Queries) 허용 문제와 실행 기반 검증의 이점

골든 SQL이 1개라고 해서 걱정할 필요는 없다.

  • 사용자 의도: “전체 직원 수를 알려줘.”
  • Golden SQL: SELECT COUNT(*) FROM employees;
  • AI SQL: SELECT COUNT(id) FROM employees;

두 쿼리는 다르지만 도출되는 스칼라(Scalar) 결괏값은 동일하게 150이다. 이것이 바로 실행 기반(Execution Engine) 오라클 설계가 갖는 거대한 축복이다. 인간이 수작업으로 모든 구문의 예외를 등록하지 않아도, 데이터 결과 테이블이 같다면 오라클은 기계적으로 PASS를 판정한다.
