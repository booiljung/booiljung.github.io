12.3.4 Spider, WikiSQL 등 오픈 벤치마크 데이터셋의 활용과 커스텀 데이터셋 구축 전략

학계에서 사용하는 데이터셋은 모델 학습용일 뿐 기업의 비즈니스 시맨틱이 녹아있지 않다.

  • Transfer Strategy: Spider 벤치마크의 난이도 체계(Easy, Medium, Hard, Extra Hard - 복잡한 JOIN과 중첩 서브쿼리 기준)를 차용하여 커스텀 사내 데이터셋을 구축한다.
  • 10개의 핵심 도메인 테이블별로 난이도를 섞어 총 500개의 사내 Golden Query 리포지토리를 만들고, 이 위에서 매일 밤 회귀 테스트(Regression Test)가 돌아가도록 하는 것이 엔터프라이즈의 표준이다.
