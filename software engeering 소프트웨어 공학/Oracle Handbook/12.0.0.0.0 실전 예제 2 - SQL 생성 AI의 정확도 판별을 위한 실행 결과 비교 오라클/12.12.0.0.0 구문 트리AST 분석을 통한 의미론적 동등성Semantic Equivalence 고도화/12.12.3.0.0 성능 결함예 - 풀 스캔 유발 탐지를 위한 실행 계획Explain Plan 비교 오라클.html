12.12.3 성능 결함(예: 풀 스캔 유발) 탐지를 위한 실행 계획(Explain Plan) 비교 오라클

결과가 동일하더라도 악성 쿼리(Malicious Performance Query)를 스코어링 테이블에서 깎아내기 위한 정밀 오라클이다.

  • AI가 짠 쿼리에 EXPLAIN (혹은 EXPLAIN ANALYZE)을 덧대어 가상의 런타임에 던진 후 실행 계획(Execution Plan) JSON을 뽑아온다.
  • 추출된 JSON 내에서 cost 값이나 seq_scan(풀 스캔 플래그)을 스니핑하여, 골든 쿼리의 실행 계획 대비 I/O 비용 수치가 임계치 바깥으로 치솟는다면 논리적으로 맞았더라도 PERFORMANCE_FAIL 처리를 때려 운영 서버를 보호한다.
