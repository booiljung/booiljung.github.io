12.6.1 읽기 전용(Read-Only) 사용자 권한 및 트랜잭션 롤백(Rollback) 전략

도커 샌드박스를 띄웠더라도 계정 세팅이 가장 중요하다.

  • 오라클 실행기(Executor)가 접속하는 DB 커넥션(Connection)은 반드시 SELECT 권한만을 가진 읽기 전용 유저(Read-Only User)로 매핑되어야 한다.
  • 만약 모종의 이유로 DML(Data Manipulation Language) 테스트가 필요한 극단적인 경우라면, 각 검증 세션을 트랜잭션 빈 블록(BEGIN TRAN)으로 감싸고 쿼리 평가가 끝나는 즉시 ROLLBACK을 호출하는 Context Manager 패턴을 구현해야 한다.
