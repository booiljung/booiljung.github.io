아래 내용은 2025년 시점을 고려하여 최신 LTS 버전의 Node.js(현재 시점 기준 Node.js 20 LTS 가정)를 사용하여 React + TypeScript + (선택사항) Electron 환경을 구성하는 예시입니다. Linux(Ubuntu 등 Debian 계열)를 기준으로 작성했지만, 다른 Linux 배포판에서도 큰 차이 없이 적용 가능합니다.

1. Node.js LTS 설치¶

1-1. NVM(Node Version Manager) 사용 권장¶

NVM을 사용하면 여러 버전의 Node.js를 쉽게 관리할 수 있습니다. Linux 환경에서 설치하는 방법은 아래와 같습니다.

# NVM 설치 (공식 GitHub의 install script 활용)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.3/install.sh | bash

# 또는 wget 사용 시
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.3/install.sh | bash

- 위 스크립트 버전(v0.39.3)은 예시입니다. nvm 공식 저장소에서 최신 버전을 확인 후 적용하세요.

설치가 끝나면 .bashrc 또는 .zshrc를 다시 로드하거나 터미널을 재시작한 뒤, nvm 명령어가 인식되는지 확인합니다.

nvm --version

1-2. Node.js 최신 LTS 버전 설치¶

Node.js 20 LTS(가정)를 설치해 봅시다:

nvm install 20
nvm use 20

설치가 완료되면 Node.js와 npm 버전을 확인해봅니다.

node -v     # v20.x.x
npm -v      # 버전 확인

2. React + TypeScript 프로젝트 구성¶

2-1. Vite를 사용한 React + TS 템플릿 예시¶

CRA(Create React App)도 많이 사용하지만, 최근에는 Vite가 더 빠르고 설정이 간편해 널리 쓰입니다. Vite 공식 템플릿을 활용해 React + TS 프로젝트를 빠르게 생성할 수 있습니다.

# npm 6.x 이상부터 create-vite 방식을 지원
npm create vite@latest

# 프로젝트 이름을 입력(예: my-react-app), 프롬프트에 따라 템플릿으로 React + TypeScript 선택
# 또는 CLI 옵션 사용:
npm create vite@latest my-react-app -- --template react-ts

cd my-react-app
npm install

설치가 끝나면 아래 명령어로 개발 서버를 실행하고, http://localhost:5173 (기본 포트) 등에서 프로젝트가 동작하는지 확인합니다.

npm run dev

2-2. 기본 React + TypeScript 구조¶

Vite로 생성된 폴더 구조는 대략 다음과 같습니다.

my-react-app/
├─ index.html
├─ package.json
├─ tsconfig.json
├─ vite.config.ts
├─ src/
│  ├─ App.tsx
│  ├─ main.tsx
│  └─ ... (기타 파일)
└─ ...
  • tsconfig.json: TypeScript 설정파일
  • vite.config.ts: Vite 관련 설정
  • src/*: 소스 코드. React 컴포넌트, 라우팅, 스타일 등을 여기서 개발

3. Electron (데스크톱 앱)까지 사용할 경우¶

React 웹 앱만으로도 <video> 태그와 File API를 이용해 사용자가 선택한 로컬 파일을 재생할 수 있지만, 자동으로 로컬 경로를 열거나 시스템 리소스에 자유롭게 접근하려면 Electron 환경이 편리합니다.

3-1. Electron을 추가 설치하기¶

이미 만들어둔 Vite + React + TS 프로젝트 폴더에서 다음과 같이 Electron을 설치합니다.

npm install --save-dev electron

Electron 기본 구조를 설정하기 위해 Main 프로세스용 파일(electron-main.js 혹은 main.ts)과 preload.ts 등을 만들어야 합니다.

3-2. 스크립트 및 폴더 구조 예시¶

아래는 예시적인 폴더 구조입니다.

my-electron-app/
├─ package.json
├─ tsconfig.json
├─ vite.config.ts
├─ electron/
│  ├─ main.ts        # Electron Main 프로세스 실행 파일
│  ├─ preload.ts     # Preload 스크립트 (IPC 통신용)
│  └─ ...
├─ src/
│  ├─ App.tsx        # React 메인 소스
│  └─ ...
├─ index.html
└─ ...

main.ts (Electron Main 프로세스 예시)¶

import { app, BrowserWindow, ipcMain, dialog } from 'electron';
import path from 'path';

let mainWindow: BrowserWindow | null = null;

function createWindow() {
  mainWindow = new BrowserWindow({
    width: 800,
    height: 600,
    // Preload 스크립트 연결 (contextIsolation 확인)
    webPreferences: {
      preload: path.join(__dirname, 'preload.js'),
      nodeIntegration: false,
      contextIsolation: true,
    },
  });

  // React 앱 (Vite dev server) 혹은 빌드 산출물 로드
  // 개발 모드 예: http://localhost:5173
  // 배포 모드 예: file://.../dist/index.html
  if (process.env.VITE_DEV_SERVER_URL) {
    mainWindow.loadURL(process.env.VITE_DEV_SERVER_URL);
  } else {
    mainWindow.loadFile(path.join(__dirname, '../dist/index.html'));
  }

  mainWindow.on('closed', () => {
    mainWindow = null;
  });
}

app.whenReady().then(createWindow);

ipcMain.handle('open-file-dialog', async () => {
  const result = await dialog.showOpenDialog({
    filters: [{ name: 'Video', extensions: ['mp4', 'mov', 'avi'] }],
    properties: ['openFile']
  });
  return result.canceled ? null : result.filePaths[0];
});

preload.ts (Preload 스크립트 예시)¶

import { contextBridge, ipcRenderer } from 'electron';

contextBridge.exposeInMainWorld('electronAPI', {
  openFileDialog: () => ipcRenderer.invoke('open-file-dialog'),
});

빌드 시에는 preload.ts와 main.ts를 각각 번들하거나 트랜스파일해 .js로 만들어야 합니다.

- Vite만으로는 Electron 메인 프로세스를 번들하기엔 추가 설정이 필요할 수 있으며, electron-builder나 vite-plugin-electron 등의 플러그인을 활용하는 방법이 있습니다.

React 측(App.tsx 등)에서 IPC 호출¶

declare global {
  interface Window {
    electronAPI?: {
      openFileDialog: () => Promise<string | null>;
    };
  }
}

function App() {
  const [videoPath, setVideoPath] = React.useState("");

  const handleOpenDialog = async () => {
    if (!window.electronAPI) return;
    const path = await window.electronAPI.openFileDialog();
    if (path) {
      setVideoPath(path);
    }
  };

  return (
    <div>
      <button onClick={handleOpenDialog}>비디오 파일 열기</button>
      {videoPath && (
        <video src={`file://${videoPath}`} controls />
      )}
    </div>
  );
}

export default App;

4. 패키지 스크립트 예시¶

4-1. 개발(로컬) 실행¶

  1. Vite dev server로 React를 실행
  2. Electron을 띄워서 Main 프로세스와 연동

package.json 예시:

{
  "name": "my-electron-app",
  "version": "0.0.1",
  "scripts": {
    "dev": "concurrently \"npm:dev:react\" \"npm:dev:electron\"",
    "dev:react": "vite",
    "dev:electron": "cross-env VITE_DEV_SERVER_URL=http://localhost:5173 electron ./electron/main.ts",
    "build": "vite build && tsc -p electron/tsconfig.json",
    "start": "electron ./dist/electron/main.js"
  },
  "devDependencies": {
    "electron": "^26.0.0",
    "typescript": "^5.0.0",
    "vite": "^4.0.0",
    "concurrently": "^7.0.0",
    "cross-env": "^7.0.3"
  }
}

(버전은 예시로 기재했으며, 실제 최신 LTS 시점에 맞춰 업데이트하세요.)

  • npm run dev: React(Vite) 개발 서버와 Electron을 동시에 실행
  • npm run build: React를 빌드하고, Electron main/preload도 트랜스파일
  • npm start: 빌드된 결과물을 이용해 Electron 앱 실행

4-2. 배포(빌드) 패키징¶

Electron 기반 데스크톱 앱을 패키징하려면 electron-builder 등을 이용할 수 있습니다.

  • npm install --save-dev electron-builder 후 package.json에 빌드 설정을 추가하면, Windows(.exe), macOS(.app), Linux(.AppImage / .deb / .snap) 등 플랫폼별 실행 파일을 생성할 수 있습니다.

5. 정리¶

  1. Node.js 최신 LTS(예: Node 20)로 개발 환경을 꾸리려면, NVM을 통해 관리하는 방법이 편리합니다.

  2. React + TypeScript 프로젝트는 npm create vite@latest -- --template react-ts 등을 통해 생성하면 빠르고 가벼운 개발 환경을 구축할 수 있습니다.

  3. 로컬 비디오 파일 재생은 두 가지로 구분해 볼 수 있습니다.

  4. (웹 전용) 브라우저

    <video>

    태그 +

    File

    API

    • 사용자가 직접 파일을 업로드/선택

  5. (데스크톱 전용) Electron + Node.js 파일 시스템 접근

    • dialog.showOpenDialog로 파일 선택, file:// 프로토콜로 <video> 재생

  6. 배포 형태가 웹이라면 Vite 빌드 결과물을 호스팅하면 되고, 데스크톱이라면 Electron + electron-builder로 패키징하여 배포합니다.

이 과정을 차근차근 진행하면 Linux 환경(또는 다른 OS)에서도 Node 최신 LTS 버전으로 React + TypeScript 기반 프로젝트(필요 시 Electron까지) 개발이 가능합니다.

만약 프로젝트 규모가 크고 배포가 복잡해진다면, Vite + Electron + Builder를 통합해주는 여러 템플릿/스타터(예: vite-plugin-electron, electron-vite-boilerplate 등)를 사용하면 설정이 한결 편해질 수 있습니다.

이상으로, 최신 LTS 버전(Node.js 20 가정)을 기반으로 한 React + TypeScript + Electron 개발 환경 구성 가이드였습니다. 필요한 범위에 맞추어 골라서 적용해 보세요!
