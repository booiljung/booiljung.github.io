project home page와 email 등을 package.json에 기술 하라는 오류가 발생하였다

Electron Builder로 앱을 패키징하려 할 때, 프로젝트의 메타데이터(홈페이지, 이메일 등)가 부족하면 “project home page와 email 등을 package.json에 기술하라”는 식의 오류나 경고가 발생할 수 있습니다. 이는 package.json 파일 내에 저자(author) 정보나 프로젝트 정보가 제대로 작성되지 않았기 때문입니다.

아래를 참고하여 package.json에 필요한 항목을 추가하면 해결할 수 있습니다.

1. author, homepage, repository 등 필드 설정하기¶

Electron Builder를 비롯해 npm 생태계 전반에서는 저자(author), 라이선스(license), 홈페이지(homepage) 등을 중요한 메타데이터로 취급합니다. package.json에 다음 예시처럼 작성해 주세요.

{
  "name": "my-electron-app",
  "version": "1.0.0",
  // author 객체 형태나 문자열 형태 모두 가능
  "author": {
    "name": "Your Name",
    "email": "[email protected]",
    "url": "https://your-website.com"
  },
  // 또는 간단히 문자열로
  // "author": "Your Name <[email protected]> (https://your-website.com)",

  // homepage(프로젝트 웹사이트 or GitHub Pages 등)
  "homepage": "https://github.com/your-username/my-electron-app",

  // repository 정보 (Git 저장소 위치)
  "repository": {
    "type": "git",
    "url": "https://github.com/your-username/my-electron-app.git"
  },

  // license(프로젝트 라이선스)
  "license": "MIT",

  "build": {
    "appId": "com.example.my-electron-app",
    "productName": "MyElectronApp",
    ...
  },
  ...
}

1-1. author 필드¶

  • 문자열 형태: "author": "홍길동 <[email protected]> (https://my-homepage.com)"

  • 객체

형태:

jsonc "author": { "name": "홍길동", "email": "[email protected]", "url": "https://my-homepage.com" }

  • npm 공식 문서상 문자열이 전통적으로 많이 사용되어 왔으나, 객체도 잘 인식됩니다.

1-2. homepage 필드¶

  • 프로젝트의 공식 홈페이지, 또는 GitHub 레포지토리 주소를 기재.
  • "homepage": "https://github.com/your-username/my-electron-app" 식으로 작성.

1-3. repository 필드¶

  • 프로젝트의 소스 코드 저장소(Git 등) 정보를 명시하는 공식 필드.

  • 예:

jsonc "repository": { "type": "git", "url": "https://github.com/your-username/my-electron-app.git" }

이 밖에 bugs, contributors 등 여러 npm 표준 필드가 있으니, 필요하면 함께 기재할 수 있습니다.

2. 왜 필요한가?¶

Electron Builder(또는 npm)에서 앱 메타데이터를 생성하거나, 설치 패키지(.AppImage, .deb, .exe 등)에 정보를 포함할 때 author, homepage, repository 등을 참조하는 경우가 많습니다. 예를 들어:

  • 앱의 “제작자 정보”나 업데이트 시 홈페이지 링크를 표시할 때 필요
  • 스캐폴딩 및 자동 업데이트 기능에서 관련 데이터가 쓰임

메타데이터가 누락되면 패키징 과정에서 경고나 오류가 뜰 수 있으므로, 적절한 정보를 기재해 주는 것이 좋습니다.

3. 정리¶

  1. package.json에 저자(author), 홈페이지(homepage), 레포(repository) 등을 추가한다.
  2. 문자열 또는 객체 형태 모두 가능하며, 일반적으로 "author": "Name <[email protected]> (URL)" 형식을 많이 씁니다.
  3. Electron Builder는 해당 필드를 이용해 설치 파일 메타데이터 등을 구성하므로, 누락 시 오류가 발생할 수 있습니다.

위 방법대로 package.json을 보완하면, “프로젝트 home page와 email 등을 작성하라”는 오류나 경고가 사라지고 정상적으로 빌드가 진행될 것입니다.
