PCL 바이너리 패키지 설치 방법¶

PCL(Point Cloud Library)을 빠르게 시작하려면 공식적으로 제공되는 바이너리 패키지를 활용하는 방법이 편리하다. 바이너리 패키지를 사용하면 소스 코드를 직접 빌드하지 않고도 PCL이 제공하는 주요 기능을 손쉽게 테스트할 수 있다. 여기서는 운영체제별로 제공되는 바이너리 패키지 설치 과정을 간략히 살펴보고, 설치 과정에서 주의해야 할 점과 필수 설정 항목들을 정리한다. 이후 PCL 활용을 위한 추가 설정 방법이나 간단한 테스트 방법도 함께 다룬다.

Windows 환경에서의 바이너리 설치¶

Windows 환경에서는 PCL 공식 웹사이트에서 제공하는 Windows용 인스톨러를 받아 바로 실행하는 방법이 널리 쓰인다. 보통 Visual Studio를 기반으로 빌드된 PCL 패키지를 제공하므로, 미리 Visual Studio가 준비되어 있거나 최소한 Visual C++ 재배포 가능 패키지 등이 함께 설치되어 있어야 한다. 일반적으로 인스톨러를 실행하면 기본 경로에 PCL 헤더, 라이브러리, 실행 파일 등이 자동으로 배치된다. 설치가 완료된 뒤에는 PCL을 사용하려는 프로젝트의 C/C++ 컴파일러 설정에서 PCL 헤더 디렉터리와 라이브러리 디렉터리를 연결해야 한다. 예를 들어 Visual Studio를 이용할 경우 속성 창에서 Include 디렉터리와 Library 디렉터리를 설치 위치에 맞추어 지정하고, 링커 옵션에 PCL 라이브러리를 추가하면 된다.

공식 웹사이트에서 배포되는 Windows 인스톨러를 사용하지 않고, 다른 패키지 매니저를 활용할 수도 있다. 대표적으로 vcpkg를 사용할 수 있는데, vcpkg를 통해 PCL을 설치하면 자동으로 의존 라이브러리들도 함께 설치되므로 편리하다. vcpkg를 사용하려면 먼저 vcpkg 저장소를 클론하고, Visual Studio 환경에서 bootstrap-vcpkg.bat 파일을 실행해 vcpkg 실행 파일을 생성해야 한다. 생성된 vcpkg.exe를 통해 PCL을 설치하면 되는데, 다음과 같이 명령을 입력할 수 있다.

vcpkg install pcl

설치가 완료된 후에는 Visual Studio 프로젝트에서 vcpkg integrate install 명령을 이용해 vcpkg가 설치한 라이브러리를 자동으로 연결하도록 설정할 수 있다. 이렇게 설정하면 매번 프로젝트 속성에서 PCL 관련 헤더 경로와 라이브러리 경로를 직접 입력할 필요가 없다. 다만, vcpkg를 이용해 설치한 PCL 버전이 공식 배포 버전보다 최신이거나 구 버전일 수 있으므로, 꼭 필요한 버전이 있다면 버전 태그를 확인하여 원하는 시점의 커밋이나 릴리스를 사용하도록 주의해야 한다.

PCL 인스톨러나 vcpkg 모두 내부적으로 다양한 의존 라이브러리를 설치한다. 예를 들어, Boost, Eigen, FLANN, VTK 등은 PCL에서 중요한 역할을 하는데, 이러한 라이브러리들이 정상적으로 설치되지 않으면 PCL 기능 일부가 동작하지 않을 수 있다. 설치 도중 문제가 발생했을 때는 의존 라이브러리들의 설치 로그를 확인하거나 Visual Studio에서 빌드 시 출력되는 에러 메시지를 면밀히 살피는 것이 중요하다. Windows 환경은 특정 라이브러리 버전 간의 호환성 문제나 런타임 DLL 충돌이 발생하기도 하므로, 만약 문제가 생겼다면 다른 버전의 PCL 바이너리나 재배포 가능 패키지를 시험해 보는 것이 좋다.

Linux 환경에서의 바이너리 설치¶

Linux 배포판마다 지원하는 패키지 매니저가 다르지만, 대체로 공식 저장소에 PCL 바이너리가 포함되어 있어 명령어만으로 간단히 설치할 수 있다. 예를 들어 Debian 계열(Ubuntu 등) 환경에서는 apt-get을 이용하여 PCL을 설치할 수 있다. 다음과 같이 명령을 입력하면 필요한 의존 라이브러리들과 함께 PCL 패키지가 자동으로 설치된다.

sudo apt-get update
sudo apt-get install libpcl-dev

이 과정을 거치면 /usr/include/pcl, /usr/lib 혹은 /usr/lib/x86_64-linux-gnu 등의 위치에 PCL 관련 헤더와 라이브러리가 배치된다. 설치가 완료된 후에는 CMake를 이용하는 경우 find_package(PCL REQUIRED)를 통해 필요한 헤더 경로와 라이브러리 경로를 자동으로 불러올 수 있다. 프로젝트의 CMakeLists.txt 파일에 다음과 같은 코드를 추가해 사용할 수 있다.

find_package(PCL REQUIRED)
include_directories(${PCL_INCLUDE_DIRS})
add_definitions(${PCL_DEFINITIONS})
link_directories(${PCL_LIBRARY_DIRS})
...
target_link_libraries(내_프로젝트_이름 ${PCL_LIBRARIES})

만약 CMake 없이 직접 g++를 사용한다면 다음과 같이 -I와 -L 옵션을 통해 헤더와 라이브러리 경로를 직접 지정하고, 필요한 라이브러리를 링크해 주어야 한다.

g++ -std=c++11 -I/usr/include/pcl ...
    -L/usr/lib/x86_64-linux-gnu -lpcl_common -lpcl_io ...
    -o 내_프로그램

배포판에 따라 PCL이 포함된 저장소가 업데이트되지 않아 최신 버전이 아닌 경우도 있다. 이럴 때는 PPA(Personal Package Archive) 등 별도 저장소를 추가하거나, PCL 공식 사이트 혹은 깃허브에서 제공하는 바이너리 패키지를 직접 내려받아 설치하는 방법을 고려할 수 있다. 다만, 수동으로 바이너리를 내려받아 /usr/local 등에 배치할 경우, 의존 라이브러리가 제대로 연결되지 않을 수 있으므로 설치 경로나 라이브러리 경로 설정을 주의 깊게 확인해야 한다.

오래된 리눅스 배포판을 사용하고 있거나 최신 기능이 필요한 경우에는 패키지 매니저가 제공하는 표준 패키지가 아닌, 소스 빌드를 통해 설치해야 할 수도 있다. 그러나 간단한 테스트나 PCL 입문을 위해서는 패키지 매니저를 통한 바이너리 설치가 훨씬 빠르고 편리하다. 의존 라이브러리(Boost, Eigen, VTK 등)가 자동으로 함께 설치되므로, 버전 호환성 문제도 크게 줄어든다.

macOS 환경에서의 바이너리 설치¶

macOS에서는 Homebrew를 이용하여 손쉽게 PCL을 설치할 수 있다. Homebrew가 설치되어 있지 않다면 먼저 다음 명령을 통해 설치해야 한다.

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install.sh)"

Homebrew 설치 후에는 다음과 같이 PCL을 설치할 수 있다.

brew update
brew install pcl

이 과정을 거치면 /usr/local/Cellar/pcl/버전/include/pcl과 /usr/local/Cellar/pcl/버전/lib 경로 등에 PCL 헤더와 라이브러리가 배치된다(설치 경로는 Homebrew 설정에 따라 달라질 수 있음). 기본적으로 Homebrew는 설치되는 라이브러리를 /usr/local 혹은 /opt/homebrew 경로 아래에 배치하므로, PCL을 이용하는 프로젝트에서 이 경로를 자동으로 인식할 수 있도록 설정을 확인해야 한다.

CMake를 사용하는 경우, find_package(PCL REQUIRED)로 필요한 정보를 자동으로 가져올 수 있다. 설치가 완료된 후에 CMakeLists.txt 파일에 PCL 설정 코드를 추가하고 빌드를 진행하면 PCL을 정상적으로 링크할 수 있다. 만약 CMake를 사용하지 않고 직접 컴파일한다면 -I, -L, 그리고 필요한 라이브러리를 링크하는 옵션을 명시해야 한다. 다음은 간단한 예시이다.

g++ -std=c++11 -I/opt/homebrew/include/pcl-1.XX ...
    -L/opt/homebrew/lib -lpcl_common -lpcl_io ...
    -o 내_프로그램

버전 호환성이나 추가적인 옵션 설정이 필요한 경우가 있으므로, Homebrew를 이용하는 프로젝트에서 종종 brew info pcl 명령을 통해 의존 라이브러리 정보와 설치 옵션을 확인하는 습관을 들이는 것이 좋다. 간혹 macOS의 SDK 버전에 따라 컴파일 옵션 충돌이 발생할 수 있는데, 이때는 Homebrew가 설치한 Eigen, VTK, Boost 등과의 버전 충돌 여부를 확인해야 한다.
