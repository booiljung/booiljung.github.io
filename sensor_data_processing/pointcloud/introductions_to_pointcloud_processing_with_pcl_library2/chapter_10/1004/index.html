포인트클라우드 데이터의 시각화에서 사용자와의 상호작용은 데이터 분석 및 이해를 돕는 중요한 요소이다. 확대, 회전, 픽킹과 같은 상호작용 이벤트를 효과적으로 처리함으로써 사용자는 포인트클라우드를 다양한 각도에서 탐색하고, 관심 있는 지점을 선택할 수 있다. 이 절에서는 PCL 라이브러리를 활용하여 이러한 상호작용 이벤트를 처리하는 방법에 대해 자세히 다룬다.

PCLVisualizer와 상호작용 이벤트¶

PCL에서는 pcl::visualization::PCLVisualizer 클래스를 사용하여 포인트클라우드를 시각화한다. 이 클래스는 다양한 상호작용 이벤트를 지원하며, 사용자 정의 콜백 함수를 통해 이벤트를 처리할 수 있다. 기본적인 확대, 회전 기능은 PCLVisualizer에 내장되어 있으나, 추가적인 사용자 정의 상호작용을 위해 콜백 함수를 등록할 수 있다.

확대 이벤트 처리¶

확대는 사용자가 시각화된 포인트클라우드를 더 가까이 보거나, 전체적인 구조를 파악하기 위해 사용하는 기본적인 상호작용이다. PCLVisualizer에서는 마우스 휠을 이용한 확대/축소 기능을 기본적으로 제공한다. 그러나, 필요에 따라 사용자 정의 확대 기능을 구현할 수도 있다.

기본 확대 기능¶

기본적으로 PCLVisualizer는 카메라의 시야(Field of View, FOV)를 조절하여 확대/축소를 구현한다. 사용자가 마우스 휠을 굴리면 카메라의 FOV가 변경되어 시점이 확대되거나 축소된다.

viewer->registerMouseCallback(&mouseEventOccurred, (void*)&viewer);

위의 코드에서 registerMouseCallback 함수를 사용하여 마우스 이벤트를 처리하는 콜백 함수를 등록할 수 있다.

사용자 정의 확대 기능¶

사용자 정의 확대 기능을 구현하기 위해서는 카메라의 위치와 시점을 조절해야 한다. 카메라의 위치는 시점에서의 거리와 방향을 나타내는 벡터로 표현되며, 이를 조절함으로써 확대/축소를 구현할 수 있다.

\[ \mathbf{C} = \mathbf{E} + d \cdot \mathbf{D} \]

여기서 \(\mathbf{C}\)는 카메라의 위치, \(\mathbf{E}\)는 시점의 중심, \(d\)는 시점과의 거리, \(\mathbf{D}\)는 카메라의 방향 벡터이다.

void zoomCallback(const pcl::visualization::KeyboardEvent& event, void* viewer_void)
{
    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    double distance;
    viewer->getCameraParameters()->distance;
    if (event.getKeySym() == "Up")
    {
        distance *= 0.9;
    }
    else if (event.getKeySym() == "Down")
    {
        distance *= 1.1;
    }
    viewer->setCameraPosition(
        viewer->getCameraPosition().x,
        viewer->getCameraPosition().y,
        distance,
        viewer->getCameraViewPoint().x,
        viewer->getCameraViewPoint().y,
        viewer->getCameraViewPoint().z,
        0, 0, 1);
}

위의 예제에서는 키보드 이벤트를 통해 카메라의 거리를 조절하여 확대/축소를 구현하였다.

회전 이벤트 처리¶

포인트클라우드를 다양한 각도에서 관찰하기 위해 회전 기능은 필수적이다. PCLVisualizer는 기본적으로 마우스 드래그를 통해 회전을 지원하지만, 사용자 정의 회전 이벤트를 구현하여 더욱 세밀한 제어가 가능하다.

기본 회전 기능¶

기본 회전 기능은 마우스 드래그를 통해 구현된다. 사용자가 마우스를 드래그하면 카메라의 시점이 해당 방향으로 회전하게 된다.

사용자 정의 회전 기능¶

사용자 정의 회전 기능을 구현하기 위해서는 회전 행렬을 이용하여 카메라의 위치와 방향을 업데이트해야 한다. 회전은 주로 두 가지 방식으로 구현된다: 월드 좌표계를 기준으로 회전하거나, 카메라의 현재 방향을 기준으로 회전하는 방법이다.

월드 좌표계를 기준으로 회전할 경우, 다음과 같은 회전 행렬을 사용할 수 있다.

\[ \mathbf{R} = \begin{bmatrix} \cos \theta & -\sin \theta & 0 \\ \sin \theta & \cos \theta & 0 \\ 0 & 0 & 1 \end{bmatrix} \]
void rotateCallback(double angle, const std::string& axis, pcl::visualization::PCLVisualizer* viewer)
{
    Eigen::Matrix3f rotation;
    if(axis == "x")
    {
        rotation = Eigen::AngleAxisf(angle, Eigen::Vector3f::UnitX()).toRotationMatrix();
    }
    else if(axis == "y")
    {
        rotation = Eigen::AngleAxisf(angle, Eigen::Vector3f::UnitY()).toRotationMatrix();
    }
    else if(axis == "z")
    {
        rotation = Eigen::AngleAxisf(angle, Eigen::Vector3f::UnitZ()).toRotationMatrix();
    }
    Eigen::Vector3f cam_pos(viewer->getCameraPosition().x,
                            viewer->getCameraPosition().y,
                            viewer->getCameraPosition().z);
    cam_pos = rotation * cam_pos;
    viewer->setCameraPosition(cam_pos.x(), cam_pos.y(), cam_pos.z(),
                              viewer->getCameraLookAt().x(),
                              viewer->getCameraLookAt().y(),
                              viewer->getCameraLookAt().z(),
                              0, 0, 1);
}

위의 함수는 특정 축을 기준으로 카메라의 위치를 회전시킨다. Eigen 라이브러리를 사용하여 회전 행렬을 생성하고, 이를 카메라의 위치에 적용한다.

픽킹 이벤트 처리¶

픽킹은 포인트클라우드 내의 특정 지점을 선택하거나, 선택한 지점의 속성을 확인하는 기능이다. 이를 통해 사용자는 관심 있는 지점의 좌표나 색상 정보를 얻을 수 있다.

픽킹 이벤트의 개념¶

픽킹 이벤트는 사용자가 시각화된 포인트클라우드에서 특정 지점을 선택할 때 발생한다. 선택된 지점의 좌표, 색상, 인덱스 등의 정보를 제공함으로써 데이터 분석에 도움을 준다.

픽킹 이벤트 구현¶

PCLVisualizer에서는 마우스 클릭 이벤트를 통해 픽킹을 구현할 수 있다. 사용자가 클릭한 지점의 픽셀 좌표를 포인트클라우드의 3D 좌표로 변환하여 해당 지점을 선택한다.

void pickPointCallback(const pcl::visualization::AreaPickingEvent& event, void* viewer_void)
{
    if(event.getPointsIndices().size() == 0)
        return;

    int idx;
    event.getPointIndex(idx);
    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    pcl::PointXYZ picked_point;
    viewer->getPointCloud()->points[idx].getVector3fMap().copyTo(picked_point.getVector3fMap());

    std::cout << "Picked point: (" << picked_point.x << ", " 
              << picked_point.y << ", " << picked_point.z << ")" << std::endl;
}

위의 예제에서는 영역 선택 이벤트(AreaPickingEvent)를 처리하여 선택된 포인트의 인덱스를 얻고, 해당 포인트의 좌표를 출력한다.

이벤트 처리의 통합¶

확대, 회전, 픽킹 이벤트는 각각 독립적으로 구현될 수 있으나, 사용자 경험을 향상시키기 위해서는 이들 이벤트를 통합적으로 관리하는 것이 중요하다. 이를 위해 이벤트 핸들러를 중앙에서 관리하고, 각 이벤트 간의 충돌을 방지하는 구조를 설계할 필요가 있다.

int main(int argc, char** argv)
{
    pcl::visualization::PCLVisualizer::Ptr viewer(new pcl::visualization::PCLVisualizer("3D Viewer"));
    // 포인트클라우드 로드 및 추가
    viewer->addPointCloud<pcl::PointXYZ>(cloud, "sample cloud");

    // 확대 이벤트 콜백 등록
    viewer->registerKeyboardCallback(zoomCallback, (void*)&viewer);

    // 픽킹 이벤트 콜백 등록
    viewer->registerAreaPickingCallback(pickPointCallback, (void*)&viewer);

    while(!viewer->wasStopped())
    {
        viewer->spinOnce(100);
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }

    return 0;
}

위의 메인 함수 예제에서는 확대와 픽킹 이벤트 콜백을 등록하고, 뷰어의 루프를 돌면서 이벤트를 처리한다. 이를 통해 사용자는 마우스와 키보드를 이용하여 포인트클라우드를 확대, 회전, 픽킹할 수 있다.

고급 픽킹 기능¶

기본 픽킹 기능 외에도, 사용자의 요구에 따라 더욱 정교한 픽킹 기능을 구현할 수 있다. 예를 들어, 다중 포인트 선택, 영역 내 포인트 하이라이팅, 선택된 포인트의 속성 편집 등이 있다. 이러한 고급 기능을 구현하기 위해서는 픽킹 이벤트 처리 로직을 확장하고, 추가적인 데이터 구조 및 사용자 인터페이스 요소를 도입해야 한다.

다중 포인트 선택¶

다중 포인트 선택 기능은 사용자가 한 번의 상호작용으로 여러 포인트를 선택할 수 있게 한다. 이를 통해 사용자는 관심 영역 내의 여러 포인트를 동시에 분석할 수 있다. 다중 선택을 구현하기 위해서는 AreaPickingEvent를 활용하여 선택된 모든 포인트의 인덱스를 가져오고, 이를 처리하는 로직을 추가해야 한다.

void multiPickPointCallback(const pcl::visualization::AreaPickingEvent& event, void* viewer_void)
{
    std::vector<int> indices;
    if(event.getPointsIndices(indices) == 0)
        return;

    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    std::cout << "Picked points:" << std::endl;
    for(auto idx : indices)
    {
        pcl::PointXYZ picked_point;
        viewer->getPointCloud()->points[idx].getVector3fMap().copyTo(picked_point.getVector3fMap());
        std::cout << "(" << picked_point.x << ", " 
                  << picked_point.y << ", " << picked_point.z << ")" << std::endl;
    }
}

위의 예제에서는 사용자가 선택한 모든 포인트의 인덱스를 indices 벡터에 저장한 후, 각 포인트의 좌표를 출력한다. 이를 통해 다중 포인트 선택을 구현할 수 있다.

포인트 하이라이팅¶

선택된 포인트를 시각적으로 강조하여 사용자가 쉽게 식별할 수 있도록 하는 기능이다. 포인트 하이라이팅을 구현하기 위해서는 선택된 포인트에 대해 별도의 렌더링 속성을 적용하거나, 별도의 포인트 클라우드로 분리하여 다른 색상이나 크기로 표시할 수 있다.

void highlightPickedPoints(const std::vector<int>& indices, pcl::visualization::PCLVisualizer* viewer)
{
    pcl::PointCloud<pcl::PointXYZ>::Ptr picked_cloud(new pcl::PointCloud<pcl::PointXYZ>());
    for(auto idx : indices)
    {
        picked_cloud->points.push_back(viewer->getPointCloud()->points[idx]);
    }
    viewer->addPointCloud<pcl::PointXYZ>(picked_cloud, "picked points");
    viewer->setPointCloudRenderingProperties(pcl::visualization::PCL_VISUALIZER_COLOR, 1.0, 0.0, 0.0, "picked points");
    viewer->setPointCloudRenderingProperties(pcl::visualization::PCL_VISUALIZER_POINT_SIZE, 5, "picked points");
}

위의 함수는 선택된 포인트들을 새로운 포인트 클라우드로 생성한 후, 빨간색과 큰 크기로 하이라이팅한다. 이를 통해 사용자는 선택된 포인트를 쉽게 식별할 수 있다.

선택된 포인트의 속성 편집¶

선택된 포인트의 속성(예: 색상, 반사도 등)을 수정하여 데이터 분석을 용이하게 할 수 있다. 이를 위해서는 선택된 포인트의 속성을 변경하는 함수와, 변경된 속성을 시각화에 반영하는 로직이 필요하다.

void editPickedPointProperties(const std::vector<int>& indices, pcl::visualization::PCLVisualizer* viewer)
{
    for(auto idx : indices)
    {
        pcl::PointXYZRGB point = viewer->getPointCloud<pcl::PointXYZRGB>("sample cloud")->points[idx];
        // 예를 들어, 포인트의 색상을 녹색으로 변경
        point.r = 0;
        point.g = 255;
        point.b = 0;
        viewer->updatePointCloud<pcl::PointXYZRGB>(viewer->getPointCloud<pcl::PointXYZRGB>("sample cloud"), "sample cloud");
    }
}

위의 예제는 선택된 포인트의 색상을 녹색으로 변경하는 기능을 구현한 것이다. 이를 통해 사용자는 선택된 포인트의 속성을 직관적으로 확인할 수 있다.

이벤트 핸들러의 최적화¶

상호작용 이벤트 처리 시 성능 최적화는 중요한 고려 사항이다. 특히, 대규모 포인트클라우드 데이터에서는 이벤트 처리 로직이 실시간으로 동작해야 하므로, 효율적인 알고리즘과 데이터 구조의 사용이 필수적이다.

이벤트 디바운싱¶

사용자가 빠르게 여러 이벤트를 발생시킬 경우, 이벤트 핸들러가 과부하되지 않도록 디바운싱 기법을 적용할 수 있다. 디바운싱은 일정 시간 동안 발생한 연속적인 이벤트를 하나로 묶어 처리하는 방법이다.

#include <chrono>
#include <mutex>

std::mutex event_mutex;
std::chrono::steady_clock::time_point last_event_time = std::chrono::steady_clock::now();
const std::chrono::milliseconds debounce_duration(50);

void debouncedCallback(const pcl::visualization::KeyboardEvent& event, void* viewer_void)
{
    std::lock_guard<std::mutex> lock(event_mutex);
    auto current_time = std::chrono::steady_clock::now();
    if(current_time - last_event_time < debounce_duration)
        return;
    last_event_time = current_time;

    // 이벤트 처리 로직
    zoomCallback(event, viewer_void);
}

위의 예제는 키보드 이벤트 콜백에 디바운싱을 적용한 것이다. 이벤트가 연속적으로 발생할 경우, 일정 시간 간격을 두고 처리함으로써 핸들러의 과부하를 방지한다.

효율적인 데이터 접근¶

포인트클라우드 데이터에 대한 빈번한 접근은 성능 저하의 원인이 될 수 있다. 이를 방지하기 위해, 데이터 접근을 최적화하고 캐싱 기법을 사용할 수 있다.

class PointCloudCache
{
public:
    PointCloudCache(pcl::PointCloud<pcl::PointXYZ>::Ptr cloud) : cloud_(cloud) {}

    pcl::PointXYZ getPoint(int idx)
    {
        std::lock_guard<std::mutex> lock(cache_mutex_);
        if(cache_.find(idx) == cache_.end())
        {
            cache_[idx] = cloud_->points[idx];
        }
        return cache_[idx];
    }

private:
    pcl::PointCloud<pcl::PointXYZ>::Ptr cloud_;
    std::unordered_map<int, pcl::PointXYZ> cache_;
    std::mutex cache_mutex_;
};

위의 클래스는 포인트클라우드 데이터에 대한 접근을 캐싱하여, 반복적인 데이터 접근 시 성능을 향상시킨다. 이를 통해 대규모 데이터에서도 효율적인 이벤트 처리가 가능하다.

사용자 인터페이스 개선¶

상호작용 이벤트 처리의 효율성과 직관성을 높이기 위해, 사용자 인터페이스(UI)를 개선하는 것이 중요하다. UI 개선을 통해 사용자는 보다 쉽게 포인트클라우드를 탐색하고, 필요한 작업을 수행할 수 있다.

툴바와 버튼 추가¶

확대, 회전, 픽킹 등의 기능을 직관적으로 사용할 수 있도록 툴바와 버튼을 추가할 수 있다. PCLVisualizer는 기본적으로 간단한 UI 요소를 제공하지 않지만, 외부 라이브러리와 연동하여 UI를 확장할 수 있다.

#include <QtWidgets/QApplication>
#include <QtWidgets/QPushButton>
#include <pcl/visualization/pcl_visualizer.h>

int main(int argc, char** argv)
{
    QApplication app(argc, argv);
    QWidget window;

위의 예제는 Qt 라이브러리를 사용하여 간단한 버튼을 추가한 것이다. 버튼 클릭 시 확대 및 축소 로직을 연결하여 사용자 인터페이스를 향상시킬 수 있다.

키보드 단축키 설정¶

자주 사용하는 기능에 대해 키보드 단축키를 설정하면, 사용자가 보다 빠르게 작업을 수행할 수 있다. PCLVisualizer에서는 키보드 이벤트를 등록하여 단축키를 구현할 수 있다.

void keyboardCallback(const pcl::visualization::KeyboardEvent& event, void* viewer_void)
{
    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    if(event.keyDown())
    {
        if(event.getKeySym() == "w")
        {
            // 카메라 위로 이동
        }
        else if(event.getKeySym() == "s")
        {
            // 카메라 아래로 이동
        }
        // 추가 단축키 구현
    }
}

int main(int argc, char** argv)
{
    pcl::visualization::PCLVisualizer::Ptr viewer(new pcl::visualization::PCLVisualizer("3D Viewer"));
    // 포인트클라우드 로드 및 추가
    viewer->addPointCloud<pcl::PointXYZ>(cloud, "sample cloud");

    // 키보드 이벤트 콜백 등록
    viewer->registerKeyboardCallback(keyboardCallback, (void*)&viewer);

    while(!viewer->wasStopped())
    {
        viewer->spinOnce(100);
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
    }

    return 0;
}

위의 예제는 'w'와 's' 키에 대한 단축키를 등록한 것이다. 이를 통해 사용자는 키보드로 카메라를 위아래로 이동시킬 수 있다.

상호작용 이벤트의 테스트 및 디버깅¶

상호작용 이벤트를 구현한 후에는 이를 철저히 테스트하고 디버깅하여 안정성을 확보해야 한다. 특히, 다양한 사용자 입력과 예상치 못한 상호작용 상황에 대비하는 것이 중요하다.

유닛 테스트¶

이벤트 처리 로직에 대한 유닛 테스트를 작성하여, 각 기능이 올바르게 동작하는지 검증할 수 있다. 예를 들어, 확대 기능이 올바르게 작동하는지, 픽킹 이벤트가 정확한 포인트를 선택하는지 등을 테스트할 수 있다.

#include <gtest/gtest.h>

TEST(ZoomCallbackTest, ZoomIn)
{
    pcl::visualization::PCLVisualizer viewer("Test Viewer");
    double initial_distance = viewer.getCameraParameters()->distance;
    // 확대 이벤트 시뮬레이션
    pcl::visualization::KeyboardEvent event;
    // 이벤트 설정 로직
    zoomCallback(event, &viewer);
    double new_distance = viewer.getCameraParameters()->distance;
    EXPECT_LT(new_distance, initial_distance);
}

int main(int argc, char** argv)
{
    testing::InitGoogleTest(&argc, argv);
    return RUN_ALL_TESTS();
}

위의 예제는 Google Test 프레임워크를 사용하여 확대 기능을 테스트한 것이다. 이를 통해 기능의 정확성을 검증할 수 있다.

디버깅 도구 활용¶

디버깅 도구를 활용하여 이벤트 처리 로직의 문제를 신속하게 발견하고 해결할 수 있다. 예를 들어, 브레이크포인트를 설정하여 이벤트 발생 시의 상태를 확인하거나, 로그를 출력하여 이벤트 흐름을 추적할 수 있다.

void zoomCallback(const pcl::visualization::KeyboardEvent& event, void* viewer_void)
{
    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    double distance = viewer->getCameraParameters()->distance;
    std::cout << "Initial distance: " << distance << std::endl;
    if (event.getKeySym() == "Up")
    {
        distance *= 0.9;
        std::cout << "Zooming in. New distance: " << distance << std::endl;
    }
    else if (event.getKeySym() == "Down")
    {
        distance *= 1.1;
        std::cout << "Zooming out. New distance: " << distance << std::endl;
    }
    viewer->setCameraPosition(
        viewer->getCameraPosition().x,
        viewer->getCameraPosition().y,
        distance,
        viewer->getCameraLookAt().x(),
        viewer->getCameraLookAt().y(),
        viewer->getCameraLookAt().z(),
        0, 0, 1);
}

위의 코드에서는 확대/축소 이벤트 발생 시 현재 거리와 새로운 거리를 로그로 출력하여 디버깅을 용이하게 한다.

상호작용 이벤트의 확장¶

기본적인 확대, 회전, 픽킹 외에도, 포인트클라우드 시각화의 상호작용성을 향상시키기 위해 다양한 기능을 추가할 수 있다. 예를 들어, 드래그 앤 드롭을 통한 객체 이동, 슬라이더를 이용한 속성 조절, 실시간 데이터 필터링 등이 있다.

드래그 앤 드롭을 통한 객체 이동¶

포인트클라우드 내 특정 객체를 선택하여 이동시키는 기능은 데이터 분석 및 편집에 유용하다. 이를 구현하기 위해서는 선택된 객체의 위치를 실시간으로 업데이트하는 로직이 필요하다.

bool is_dragging = false;
int dragged_point_idx = -1;

void mouseDragCallback(const pcl::visualization::MouseEvent& event, void* viewer_void)
{
    pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
    if(event.getButton() == pcl::visualization::MouseEvent::LeftButton && event.isDrag())
    {
        if(!is_dragging)
        {
            // 드래그 시작 시 포인트 선택
            dragged_point_idx = /* 포인트 인덱스 가져오는 로직 */;
            is_dragging = true;
        }
        if(dragged_point_idx != -1)
        {
            // 마우스 위치에 따라 포인트 이동
            pcl::PointXYZ new_position = /* 마우스 위치에 따른 새로운 좌표 계산 */;
            viewer->getPointCloud()->points[dragged_point_idx] = new_position;
            viewer->updatePointCloud<pcl::PointXYZ>(viewer->getPointCloud(), "sample cloud");
        }
    }
    else
    {
        is_dragging = false;
        dragged_point_idx = -1;
    }
}

위의 예제는 마우스 드래그 이벤트를 통해 특정 포인트를 이동시키는 기능을 구현한 것이다. 드래그 시작 시 포인트를 선택하고, 마우스 이동에 따라 포인트의 위치를 업데이트한다.

슬라이더를 이용한 속성 조절¶

포인트클라우드의 속성을 실시간으로 조절할 수 있는 슬라이더를 추가하면, 사용자가 데이터 시각화를 더욱 정밀하게 조정할 수 있다. 예를 들어, 색상 매핑의 임계값을 슬라이더로 조절하거나, 포인트의 투명도를 변경할 수 있다.

#include <QtWidgets/QSlider>

int main(int argc, char** argv)
{
    QApplication app(argc, argv);
    QWidget window;
    QSlider slider(Qt::Horizontal, &window);
    slider.setGeometry(150, 10, 200, 30);

    pcl::visualization::PCLVisualizer::Ptr viewer(new pcl::visualization::PCLVisualizer("3D Viewer"));
    // 포인트클라우드 로드 및 추가
    viewer->addPointCloud<pcl::PointXYZRGB>(cloud, "sample cloud");

    QObject::connect(&slider, &QSlider::valueChanged, [&](int value) {
        double alpha = value / 100.0;
        viewer->setPointCloudRenderingProperties(pcl::visualization::PCL_VISUALIZER_OPACITY, alpha, "sample cloud");
    });

    window.show();

    while(!viewer->wasStopped())
    {
        viewer->spinOnce(100);
        app.processEvents();
    }

    return 0;
}

위의 예제는 Qt 슬라이더를 사용하여 포인트클라우드의 투명도를 조절하는 기능을 구현한 것이다. 슬라이더 값을 실시간으로 읽어와 투명도 속성에 반영한다.

실시간 데이터 필터링¶

사용자가 특정 조건에 따라 포인트클라우드를 실시간으로 필터링할 수 있는 기능을 제공하면, 대규모 데이터에서도 필요한 정보를 신속하게 추출할 수 있다. 예를 들어, 높이 기준으로 포인트를 필터링하거나, 특정 색상 범위 내의 포인트만 표시할 수 있다.

#include <pcl/filters/passthrough.h>

void filterPointsCallback(const pcl::visualization::KeyboardEvent& event, void* viewer_void)
{
    if(event.getKeySym() == "f" && event.keyDown())
    {
        pcl::visualization::PCLVisualizer* viewer = static_cast<pcl::visualization::PCLVisualizer*>(viewer_void);
        pcl::PointCloud<pcl::PointXYZ>::Ptr cloud_filtered(new pcl::PointCloud<pcl::PointXYZ>());
        pcl::PassThrough<pcl::PointXYZ> pass;
        pass.setInputCloud(viewer->getPointCloud<pcl::PointXYZ>("sample cloud"));
        pass.setFilterFieldName("z");
        pass.setFilterLimits(0.0, 1.0);
        pass.filter(*cloud_filtered);
        viewer->updatePointCloud<pcl::PointXYZ>(cloud_filtered, "sample cloud");
    }
}

위의 예제는 'f' 키를 눌렀을 때 z축 기준으로 포인트를 필터링하는 기능을 구현한 것이다. 이를 통해 사용자는 특정 범위 내의 포인트만을 실시간으로 볼 수 있다.
