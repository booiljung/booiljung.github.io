54.2 동기식(Synchronous) 대 비동기식(Asynchronous) 헤테로지니어스 매핑 아키텍처 간 런타임 연산 부하 모델링

1. 개요

ROS 2(Robot Operating System 2) 환경에서 동작하는 slam_toolbox의 매핑 아키텍처는 처리의 완결성과 연산 부하 간의 상관관계에 따라 동기식(Synchronous) 처리 모드와 비동기식(Asynchronous) 처리 모드로 분류된다. 이 두 아키텍처는 센서 오도메트리(Odometry) 입력 및 라이다(LiDAR) 스캔 데이터의 스레드 처리 방식에서 극명한 차이를 보이며, 이는 자율 드론의 탑재 컴퓨터와 같은 제한된 다중 코어 환경에서 런타임 연산 부하(Runtime Computation Load)를 분배하는 지배적인 요소로 작용한다. 헤테로지니어스 매핑 아키텍처 내에서 두 방식의 알고리즘적 특성과 CPU 코어 점유 시간, 메모리 병목 현상 등을 수학적 연산 부하 모델로 분석하여 시스템의 목적에 부합하는 SLAM(Simultaneous Localization and Mapping) 구성 전략을 정립한다.

2. 동기식(Synchronous) 매핑 아키텍처의 연산 부하 특성

동기식 모드(Synchronous Mode)에서는 새로운 레이저 스캔 메시지가 수신될 때마다 스캔 매칭(Scan Matching) 및 포즈 그래프 추가, 하부 그래프(Sub-graph) 생성을 단일 루프 내에서 순차적으로 처리한다.

2.1 단일 스레드 병목 및 큐(Queue) 대기 모델

모든 데이터 파이프라인이 직렬화되어 실행되므로, 센서 주기(Sensor Rate) f_s보다 SLAM 알고리즘 내부 처리 시간 T_{proc}이 클 경우 즉각적인 메시지 폐기(Message Drop) 혹은 큐 지연(Queue Delay)이 발생한다.
동기식 시스템에서의 연산 지연 대기 모델 D_{sync}는 다음과 같다.
D_{sync} = \max(0, T_{proc} - \frac{1}{f_s}) + \epsilon
여기서 \epsilon은 ROS 2 미들웨어(DDS) 통신 레이어에서 발생하는 메시지 전송 오버헤드 상수이다. 동기화 루프를 강제하기 때문에 스캔 프레임이 누락되지 않는 한 가장 높은 정합도를 가지는 밀집 맵(Dense Map)을 생성할 수 있다. 그러나 비행 속도가 빠르거나 센서 버퍼의 처리 스루풋이 하드웨어 스펙을 초과하는 경우 CPU 스레드의 사용률(Utilization) U가 100% 한계치에 도달하여 치명적인 연산 스파이크(Computation Spike)를 유발한다.

2.2 포즈 그래프 최적화(Pose Graph Optimization) 수반 블로킹

Karto SLAM 알고리즘 기반의 백엔드(Backend) 포즈 그래프 최적화 과정이 동작하는 순간, 포즈 그래프 내 노드 개수 N에 대하여 \mathcal{O}(N^3) 수준의 행렬 연산 연산량이 동기식 프론트엔드의 메인 스레드를 블로킹(Blocking)한다. 이는 자율 드론의 실시간 회피 및 내비게이션 노드에 연쇄적인 지연을 초래하므로 광역 스케일 매핑(Large-Scale Mapping) 환경에서는 통제 불가능한 적체 부하를 발생시킨다.

3. 비동기식(Asynchronous) 매핑 아키텍처의 연산 부하 특성

slam_toolbox의 비동기식 모드(Asynchronous Mode)는 메인 루프에서 센서 수신을 전담하는 스레드 영역과, 행렬 계산을 수행하는 최적화 스레드 영역을 분리하여 스레드 풀(Thread Pool)을 활용한 비선형 솔버 처리를 백그라운드에서 병행한다.

3.1 처리 자원 한계 기반의 프레임 취사선택 로직

수신되는 스캔 데이터의 스트림 중, 현재 시스템 프로세서가 할당받아 처리 가능한 메시지만을 취사선택하여 최적화를 수행(Process)하고, 메모리 처리 한계 도달 시 잔여 메시지는 큐에서 강제 폐기(Drop)함으로써 로컬라이제이션(Localization)의 최신성(Recency) 파라미터를 우선 확보한다. 연산 부하 및 처리 스루풋 분배 모델은 다음과 같이 정의된다.
T_{async\_proc} \le \frac{1}{f_s}
위 등식을 초과하는 프론트엔드 연산 도중 큐에 도착한 k개의 잔여 센서 프레임은 그래프 삽입 단계를 생략한다. 이를 통해 CPU 부하는 개발자가 설정 가능한 최대 사용 시간폭 미만으로 제한되며, 이를 활용하여 가용 하드웨어의 CPU 할당량을 항법 제어(Flight Controller) 로직에 우선적으로 고정 배분하여 보호할 수 있다.

2.2 백그라운드 지연 연산 및 분산 최적화 모델

비동기 백그라운드 스레드에서 전역 포즈 그래프 최적화(Global Pose Graph Optimization) 공간 변환이 연산되는 동안, 로컬 프론트엔드(Frontend)는 새로운 맵 타일을 구축하고 스캔 매칭을 독립적으로 이어나간다.
구조적 분리에 의해 발생하는 백엔드 연산 부하 L_{backend}는 메인 맵핑 연산 스레드에서 비동기적으로 스케줄링되므로, 전체 프로세서 코어의 병렬 처리 구조 하에서 시스템 자원 경합(Resource Contention) 감소 모델로 산술된다. 비동기 환경에서의 프론트엔드 유효 지연시간 D_{async}는 상수 딜레이 모델 혹은 \mathcal{O}(1) 수렴 시간복잡도를 달성한다. 단, 비동기 처리는 스캔 프레임을 강제로 생략하므로 동기식 맵 최적화에 비하여 최종 레이어의 픽셀 해상도 및 포즈 그래프 토폴로지의 정밀도가 미세하게 저하되는 트레이드오프(Trade-off) 한계를 수반한다.

3. 런타임 환경 목적 기반 매핑 아키텍처 구성 전략

자율 조종 드론의 제한적 컴퓨팅 아키텍처 공간 내에서, 단일 혹은 다중 스레드 할당 모델의 설계는 에이전트의 운용 시나리오에 종속적으로 판단해야 한다.

  1. 정밀 오프라인 매핑 (Offline/Precision Mapping): 백 팩(Backpack) 형태의 수동 조작 비행이나 최저속 비행을 통해 철저한 공간 데이터를 수집해야 할 경우, 메인스레드의 블로킹 및 연산 부하를 감수하더라도 동기식(Synchronous) 모드를 채택하여 고분해 공간 해상력(Spatial Resolution) 데이터를 담보한다.
  2. 고속 자율 비행 로컬라이제이션 (High-Speed Autonomous Navigation): 폐쇄 루프 기반의 실시간 응답 제어가 필수적인 드론의 특성 상, 자율 에이전트 내비게이션 파이프라인은 비동기식(Asynchronous) 상태 추정 모드를 구축하여 SLAM 최대 연산 처리 부하의 마진을 명확히 규제해야 한다. 이 경우 런타임 부하가 과적될 경우를 대비하여 max_laser_range, resolution, 최적화 반복 임계치(Threshold) 등의 매개변수를 하향 하드코딩하여 연산 스파이크를 방어하는 기법이 동원된다.

두 연산 부하 아키텍처는 단일 시스템 내에서 ROS 2 플랫폼의 런치(Launch) 파라미터 제어를 통해 스위칭될 수 있으며, 시스템의 실시간성(Real-time Constraints) 준수 기준선에 맞추어 런타임 환경의 연산 파이프라인 자원을 탄력적으로 보장하는 중대한 설계 기준표 역할을 담당한다.
