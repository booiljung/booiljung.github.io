56.1 Nav2 플래너(nav2_planner) 스코어링 아키텍처 및 전역 경로 탐색(A*, Dijkstra) 휴리스틱(Heuristic) 최적화 방법론

1. 개요

ROS 2(Robot Operating System 2)의 자율 주행 프레임워크인 Nav2(Navigation 2)에서 nav2_planner 노드는 에이전트의 현재 전역 상태(Global State)로부터 최종 목표 목적지(Goal Pose)까지 이르는 전역 경로(Global Path)를 연산하는 중추 모듈이다. 이 노드는 글로벌 비용 맵(Global Costmap) 상에서 수학적 궤적 공간 탐색 알고리즘을 수행하며, 지도의 정적 점유 그리드 데이터뿐만 아니라 인플레이션(Inflation) 반경 등이 혼합된 비용 스칼라를 입력 인자로 채택한다. 본 절에서는 nav2_planner의 스코어링 아키텍처 특성과 함께, 기반 그래프 탐색 알고리즘인 다익스트라(Dijkstra)와 A* 알고리즘의 최적화 휴리스틱(Heuristic) 모델링에 대해 학술적으로 분석한다. (Nav2 v1.1 기준 참조)

2. 전역 경로 스코어링(Scoring) 아키텍처

nav2_planner를 구동하는 핵심 알고리즘 플러그인(예: NavFn, Smac Planner)은 기본적으로 2D 래스터(Raster) 기반 그리드를 이산적 위상 공간의 노드 표상으로 변환한다. 스코어링 아키텍처의 목적은 무한에 가까운 후보 궤적 벡터 집단 속에서 장애물과 에이전트 외면 사이의 상대적 안전 이격도를 보장하면서, 가장 낮은 누적 비용을 소모하는 연속된 공간 좌표 배열을 도출하는 데 있다.

이 때 그리드 지도 상의 한 노드 v_i와 인접 노드 v_j 사이의 단일 전이 비용 C(v_i, v_j)는 매개변수화된 수식에 의해 결정되며, 통상적 형태의 비용 함수 \mathcal{J}는 다음과 같이 정의된다.

\mathcal{J}(P) = \sum_{k=0}^{N-1} \left( \alpha \cdot \text{dist}(v_k, v_{k+1}) + \beta \cdot \text{costmap}(v_{k+1}) \right)

  • \text{dist}(v_k, v_{k+1}) : 정점 간 물리적 유클리드 이동 한계 길이이다.
  • \text{costmap}(v_{k+1}) : 해당 지점에 대한 로컬 텐서 스칼라(장애물 근접에 기반한 인플레이션 페널티)이다.
  • \alpha, \beta : 환경 목표 특성 및 요구사항에 따른 하이퍼파라미터 가중치이다.

결과적으로 최적 전역 경로 탐색은 유효 정점 열 P^* = \operatorname{argmin}_P \mathcal{J}(P) 를 연산해내는 수학적 최소 비용 도출 알고리즘으로 치환된다.

3. 다익스트라(Dijkstra) 알고리즘 확장 및 기저 복잡도

오리지널 NavFn 플러그인의 근본 전역 탐색 엔진인 다익스트라(Dijkstra) 알고리즘은 단일 출발 정점에서부터 연결된 모든 정점으로의 최소 접근 경로를 탐색하는 등방성 파동 확산(Wavefront Expansion) 체계를 채택한다. 알고리즘 설계 사상 측면에서, 목표 지점에 대한 사전 위치 방향 지식 없이 우선순위 자료 구조(Priority Queue)의 최상층에서 최소 경로 평가 값을 지닌 정점 객체만을 맹목적으로 채택하여 확장한다.

해당 그래프 전수 조사 방식은 코스트맵의 그리드 배열 구조 하에서 목표 객체에 도달할 때까지 탐색 트리의 물리적 반경이 전방위로 비대화된다. 따라서 최단 경로의 이론적 완전성(Completeness)과 객관적 최적성(Optimality)을 견고하게 수호하나, 노드의 수 V와 간선의 수 E의 증가에 비례하여 전역 계산 복잡도가 \mathcal{O}(\vert V \vert \log \vert V \vert + \vert E \vert) 단위로 팽창되어 고면적(High-Resolution) 코스트맵 기반 글로벌 경로 리엔지니어링 상황에서는 연산 지연 시간(Latency)의 급증 요인으로 작용한다.

4. A* 알고리즘 기저 휴리스틱(Heuristic) 함수 최적화 방법론

다익스트라 알고리즘에 필연적으로 동반되는 무지향적 탐색 메모리 및 연산 오버헤드를 억제하기 위해, Smac Planner 등 최신 플러그인 프레임워크는 A* 탐색 로직을 전역 수렴 아키텍처로 도입하고 있다. 명시적 A* 탐색 아키텍처는 목표 좌표 방향으로의 우선 탐색 편향성을 주입할 목적으로, 시작 정점으로부터 현재 임의 조사 정점 n까지의 실 가산 누적 비용 g(n)과 평가 정점 n에서 목표 목적 좌표 위치까지의 추정된 잔여 비용 지수인 휴리스틱 함수 h(n)을 논리 연산 결합하여 평가 함수 f(n)을 스코어링의 절대적 기준으로 삼는다.

f(n) = g(n) + h(n)

설계된 후보 휴리스틱 h(n)이 실제 잔류 궤도 경로의 하한 궤적(Lower bound) 길이를 통계 추론적으로 초과하지 않는다는 허용성(Admissible) 제약 논리를 만족할 시, A* 알고리즘 렌더링 결과 역시 완벽한 전역 최적 경로 도출을 수학적으로 보장한다. nav2_planner의 휴리스틱 최적화 연산망은 주로 다음과 같은 기하 분포 메트릭 연산을 변수 모델로 선택한다.

휴리스틱 타입 (Heuristic Type)수식 모델링 (Formula Modeling)탐색 최적화 적용 및 구조적 특징 (Optimization & Structural Features)
맨해튼 거리 (Manhattan Distance)h(n)= \vert x_g - x_n \vert + \vert y_g - y_n \vert이동 모델이 4방향 폰 노이만(Von Neumann) 근방 이산 이동만을 수행하는 환경 모델에서 탐색 지연 연산 비용을 극소화한다.
유클리드 거리 (Euclidean Distance)h(n)= \sqrt{(x_g - x_n)^2 + (y_g - y_n)^2}대각선 직선 이동을 전면 허용하는 무방향 연속 벡터 공간상에서 현실적인 최단 방향 궤도 하한선 값을 보장하여 제공한다.
체비쇼프 거리 (Chebyshev Distance)h(n)= \max(\vert x_g - x_n \vert, \vert y_g - y_n \vert)8방향 무어(Moore) 이웃 체계를 사용하는 비격자 그리드 확장에서 대각선 최단 노드 이격 스칼라를 보수적으로 측정 및 예측한다.

SmacPlanner 아키텍처 내 2D A* 구현체는 원시 기하 중심적 다항 측정에만 머물지 않고, 환경 코스트맵의 직접적 점유 위험 가중치를 h(n) 내부 방정식에 동적 혼합(Dynamic Blending)하는 진보된 환경 기반 휴리스틱(Enhanced Environment Heuristic) 모델을 내재화하여 투사한다. 또한 상수 계수 \omega(>1) 크기의 가중 배분을 강제로 인가한 \hat{f}(n) = g(n) + \omega \cdot h(n) 연산 형태의 차등 결합 A*(Weighted A*) 궤도 기법을 보조 사용하면, 다소간의 기하 궤도 스칼라 최적성을 희생량으로 삼고 그 반대급부로 광야(Open Space) 형태의 코스트맵 내 파생 평가 정점 객체 생성 개체 수를 대수적(Logarithmic) 혹은 지수함수적으로 획기적으로 절단해내 물리적 최단 수 밀리초 대역(ms)의 경로 반환 가속 속도를 구현할 수 있다.
