52.7 C++ 기반 bt_navigator 커스텀 트리 노드(Custom Tree Node) 작성 가이드 및 플러그인 레지스트리 통합

1. 기반 가상 인터페이스 상속(Inheritance) 및 노드 타입 선언 아키텍처

Nav2의 행동 트리(BT) 프레임워크는 사실상 어떠한 사이버 물리적 요구사항이라도 수용할 수 있는 무한한 확장성을 플러그인 체계를 통해 보장한다. 자율 에이전트 드론 전용의 특수 동작(예: ‘초음파 센서 기반 초근접 착륙’ 혹은 ‘짐벌 이드 추적’)을 XML 트리에 편입시키기 위해서는, 개발자는 C++ 환경에서 Nav2가 제공하는 코어 가상 계층(Virtual Layer) 클래스를 상속받아 커스텀 노드를 구현해야 한다.

동기식 단순 검증 모델이 필요하다면 BT::ConditionNode 또는 BT::SyncActionNode를 상속받는다. 반면 수 초에서 수 분이 소요되는 장동(Long-running) 비동기 통신 모델, 특히 하부 액션 서버(Action Server)에 골(Goal)을 전송하는 로직이라면 Nav2 래퍼 계층인 nav2_behavior_tree::BtActionNode<T> 인터페이스를 상속받는 것이 정규적이다. 템플릿 파라미터 <T>에는 ROS 2 액션 메시지 타입(예: nav2_msgs::action::NavigateToPose)이 명시적으로 주입되며, 이를 통해 기저 엔진이 통신 디스패치와 피드백 런타임을 자동으로 은닉 및 대리(Proxy) 수행하게 된다.

2. providedPorts() 함수를 통한 정적 블랙보드 포트(Port) 바인딩 명세

커스텀 노드가 BT 스케줄러 내에서 유효한 블랙보드 객체 참조 권한을 취득하기 위해서는 자신의 데이터 기저면(Data Boundary)을 포트로 정적 명세(Static Specification)해야 한다.

C++ 코드 내부에 반드시 오버라이드(Override)되어야 하는 providedPorts() 정적 멤버 함수는 BT::InputPort 및 BT::OutputPort 타입의 키(Key) 딕셔너리를 반환하도록 스크립팅된다. 예컨대 BT::InputPort<double>("target_altitude")로 포트가 스폰(Spawn)되면, 이후 XML 런타임 파일에서 target_altitude="{global_alt}" 와 같은 매핑 구문을 통해 블랙보드 메모리상의 전역 실수 값 포인터가 노드 객체 내부의 안전한 추상화 참조 공간으로 바인딩된다. 이 규격화는 노드 생성 및 트리 파싱 시점에 C++ RTTI(Run-Time Type Information) 캐스팅 유효성 검증을 수반하므로, 타입 매칭 예외를 런타임 도중이 아닌 스택업 단계에서 완벽히 색출하는 기틀을 마련한다.

3. tick(), on_tick(), halt() 콜백 내의 비동기 생명주기(Lifecycle) 통제 체계

커스텀 노드의 물리적 행동 제어는 프레임워크가 주입하는 생명주기 가상 콜백(Virtual Callback) 함수군에 의해 통제된다. BtActionNode를 상속받는 경우, 순수 BT의 원시 tick() 오퍼레이터는 래핑되어 숨겨지고, 대신 개발자는 on_tick() 메소드를 구현하여 최초 타겟 액션 골(Action Goal)의 셋업 및 발송 절차만을 정의하게 된다.

노드가 트리거되어 RUNNING 상태를 점유하는 동안 ROS 2 미들웨어 피드백이 도달하면 on_wait_for_result()가 백그라운드 태스크로 연동되며, 목표를 완수한 순간에는 on_success() 또는 on_aborted()가 호출되어 제어 루프를 정상적으로 종결시킨다. 치명적인 중요성은 halt() 메소드의 오버라이드에 존재한다. 다른 반응형 브랜치의 전단(Pre-emption)으로 인해 트리가 취소될 때 프레임워크는 즉시 halt()를 발송(Interrupt)한다. 커스텀 노드는 이 신호 수신 시 자신이 점유 중인 드론 하드웨어 스레드 자원을 해제하고 ROS 통신 큐를 플러시(Flush)하여, 데드락(Deadlock) 없는 완벽한 취소 기동(Cancel Maneuver) 무결성을 달성해야 한다.

4. C++ Pluginlib 매크로 레지스트리(Registry) 통합과 XML 동적 로딩 역학

컴파일된 C++ 공유 라이브러리(.so) 바이너리 객체가 Nav2 런타임 생태계의 bt_navigator 프로세스에서 가시성을 확보하려면, 소스 파일 종단 점에 반드시 ROS pluginlib 익스포트 매크로가 바인딩되어야 한다. BT_REGISTER_NODES(factory) 메이커 매크로를 통해 커스텀 노드 클래스 명칭(예: DroneTakeoffAction)을 BT 팩토리(Factory)의 플러그인 딕셔너리에 문자열 ID와 함께 등록한다.

이 등록 절차가 완수되면, 시스템 설계자는 nav2_params.yaml의 bt_navigator.plugin_lib_names 리스트 배열에 해당 라이브러리를 추가 기입하는 행위만으로 통합 관제를 달성하게 된다. 결과적으로 전체 시스템을 전면 재빌드할 필요 없이, bt_navigator는 기동 중에 ament_index를 스캔하여 커스텀 플러그인 바이너리를 동적 로딩(Dynamic Loading)하고, XML 파일 내 정의된 커스텀 태그 <DroneTakeoffAction /> 구조를 인메모리 트리 힙 공간으로 즉각 객체화(Instantiation)하는 가공할 모듈식 유연성(Modular Flexibility)을 완성시킨다.
