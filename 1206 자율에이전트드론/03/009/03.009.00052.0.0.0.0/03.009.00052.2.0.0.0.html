52.2 행동 트리(Behavior Tree, BT)의 수학적 모델링 및 유한 상태 기계(FSM) 대비 제어 유연성(Flexibility) 분석

1. 유한 상태 기계(FSM)의 전이 행렬 폭발(Transition Matrix Explosion) 문제

과거 전통적인 로봇 제어 아키텍처는 유한 상태 기계(FSM; Finite State Machine)를 기초로 설계되었다. FSM은 수학적으로 상태 집합 S, 입력 알파벳 \Sigma, 전이 함수 \delta: S \times \Sigma \rightarrow S, 초기 상태 s_0, 최종 상태 집합 F 구성을 갖는 M = (S, \Sigma, \delta, s_0, F)의 튜플로 정의된다.

이 FSM 모델을 자율 에이전트 드론에 적용할 경우, 기본 비행(FLY), 장애물 감지(AVOID), 배터리 경고(LOW_BATT)의 세 가지 상태만 존재하더라도 \delta 테이블 맵핑에 3 \times 3 = 9개의 전이(Transition) 화살표를 프로그래밍해야 한다. 만약 시스템에 통신 단절, 페이로드 오류, 강풍 감지 등 N개의 추가 상태가 도입될 경우, 시스템의 잠재적 전이 복잡도는 \mathcal{O}(N^2)의 지수 팽창 스케일 폭발(State Explosion) 현상에 직면하게 되며, 이는 제어 로직 코드 내부의 광범위한 goto 문이나 난해한 중첩 switch-case 구문을 양산하여 시스템 유지보수와 디버깅을 파국으로 이끈다.

2. 행동 트리(Behavior Tree, BT)의 수학적 구조와 그래프 모델링

이러한 전이 맹점을 타파하기 위해 Nav2 메타 시스템은 행동 트리(Behavior Tree, BT) 구조를 전격 채용하였다. BT는 수학적으로 하나 유일한 루트(Root) 노드를 가지며 사이클(Cycle)이 생성되지 않는 방향성 비순환 그래프(DAG; Directed Acyclic Graph)의 특수한 트리 다이내믹스 스펙을 따른다. BT의 그래프 노드는 집합 V = V_{control} \cup V_{execution} 로 이분되며, 부모-자식 간의 간선(Edge) 집합 E 로 결합된다.

BT의 최상단에서 출발한 제어 틱(Tick) 시그널은 Sequence나 Selector(Fallback) 같은 제어 노드(V_{control})를 거쳐 단말 리프(Leaf)인 실행 노드(V_{execution})로 전파된다. 이때 각 노드가 틱의 실행 결과로서 부모에게 반환(Return)하는 상태값 모델은 \mathbb{R} \in \{SUCCESS, FAILURE, RUNNING\} 의 3-State 열거형(Enum)으로 엄격하게 제한된다. 이 3-State 피드백 루프는 FSM이 가진 고정된 전이 행렬 없이도, 각 노드가 독립된 스레드에서 블랙박스 형태로 평가된 후 트리 그래프의 위계를 따라 자율적인 다음 연산 스케줄(Next Compute Schedule) 큐로 라우팅되게 하는 강력한 구동 엔진의 기반이다.

3. 무상태성(Statelessness) 기반 제어 논리와 RUNNING 상태 변수

설계 관점에서 FSM 대비 BT가 갖는 가장 극명한 제어 논리의 우위는 단일 노드의 원천적인 무상태성(Statelessness)에 있다. FSM에서는 특정 동작을 실행하기 위해 반드시 시스템 전역의 현재 상태(s_c \in S)가 사전에 검증되어야 하지만, BT의 개별 Action Node는 전체 시스템이 지금 비행 중인지 착륙 중인지의 컨텍스트(Context)를 알 필요 없이 오로지 할당된 단일 로직(예: 모터 회전)만 수행한 후 SUCCESS나 FAILURE를 반환한다.

유일한 예외는 비동기 장기 실행(Async Long-running) 프로세스의 생명 연장인 RUNNING 상태값이다. 목표 지점으로 주행하는 NavigateToPose 액션 노드는 단일 로직 사이클(수 밀리초) 안에 종료되지 않는다. 이 경우 리프 노드는 부모에게 RUNNING 상태를 연달아 피드백하며, BT 스케줄러는 트리의 루트부터 순회하는 다음 틱(Tick) 트리거 주기에 강제로 다른 브랜치를 스킵하고 바로 앞선 RUNNING 노드로 제어 스킵핑(State Regression Avoidance) 점프를 수행한다. 이 메커니즘은 다중 장동(Long-action) 코루틴(Coroutine)의 일시 정지와 재개를 FSM의 락(Lock) 없이 완벽히 직렬화(Serialize) 해낸다.

4. 복구(Recovery) 논리 모델의 결합성(Composability)과 확장 위상

FSM 하에서는 비상 정지(E-Stop) 후 회피 궤도 기동이라는 구조(Recovery) 시나리오를 주입하기 위해 기존의 FLY \rightarrow IDLE 화살표를 모두 찢어발기고 FLY \rightarrow ESTOP \rightarrow RECOVERY 행렬 차원에 역으로 패치(Patch) 해야 한다. 이는 레거시 알고리즘과의 치명적 충돌 파편화(Fragmentation)를 유발한다.

반면에 BT는 트리 계층 간의 구조적 결합성(Composability)을 지니고 있어, 기존 메인 트리 구조를 손끝 하나 건드리지 않고도 오직 Selector 부모 노드 하나를 기존 Action 노드 바로 상단에 래핑(Wrapping) 인서트하여 서브 트리(Sub-tree) 브랜치 확장을 완료할 수 있다. 만약 Drive 리프가 장애물로 인해 FAILURE를 던지더라도, 래핑된 Selector 노드는 시스템 플로우를 바로 우측의 Spin(제자리 돌기) 노드로 전이시켜 회복을 시도한다. 이러한 모듈식 조립 특성(Modular Assemble Property)은 개발자가 C++ 소스 코드를 재수정하지 않고도 XML로 규격화된 트리 스키마 파일을 동적 로드(Dynamic Load)하는 것만으로 에이전트 드론의 반응형 계층 지능(Reactive Logic)을 무한히 증강시킬 수 있는 플루이드 유연성(Fluid Flexibility)을 성립시킨다.
