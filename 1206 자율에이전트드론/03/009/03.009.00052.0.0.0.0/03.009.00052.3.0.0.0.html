52.3 Nav2 행동 트리 노드(Action, Condition, Control, Decorator) 요소의 계층적 동작 명세

1. 제어 흐름 분기를 관장하는 제어 노드(Control Node) 위상

행동 트리(Behavior Tree; BT)의 내골격을 형성하는 제어 노드(Control Node)는 틱(Tick) 시그널 모델이 자식 노드 배열로 어떻게 전파(Propagation)될지를 결정하는 계층적 분기점(Branching Point)이다.

제어 노드의 가장 핵심적인 유형인 Sequence 노드는 모든 자식 노드가 순차적으로 SUCCESS를 반환해야만 자신도 부모에게 SUCCESS를 보고하는 엄격한 AND 논리 연산자 형상을 띤다. 단 하나의 자식이라도 FAILURE를 던지면 연쇄를 중단하고 실패를 반환한다.
반면 Fallback(또는 Selector) 노드는 OR 논리 연산자와 대응된다. 한 자식이 FAILURE를 반환하더라도 이를 에러로 간주하지 않고 즉시 우측의 다음 자식에게 틱을 전달하며, 단 하나의 자식이라도 SUCCESS를 쟁취하면 즉시 부모에게 성공 상태를 전이한다. 이는 일차적인 목표 도달 시도가 좌절되었을 때 대안적인 회피 기동(Recovery)이나 차선책 경로 탐색을 렌더링하는 본질적인 결함 허용(Fault Tolerance) 모델을 작동시킨다. 또한 Parallel 노드는 동시 다발적인 센서 읽기와 궤적 출력을 위해 다수의 자식을 동시에 틱트(Ticked)하게 만들어 비동기 동시성(Asynchronous Concurrency)을 창출한다.

2. 실질적 상태 전이 및 비동기 런타임을 수행하는 실행 노드(Action Node)

트리의 말단 리프(Leaf) 계층에 위치하는 Action Node는 ROS 2 미들웨어 환경에서 실제 시스템의 사이버 물리적 상태 변경(State Alternation)을 관할하는 실체이다. Nav2의 NavigateToPose, ComputePathToPose, Spin, Wait 등은 이 클래스를 상속받는다.

이 노드들은 단순히 함수 블록 단위의 즉각적인 응답(Synchronous Return)만을 의미하지 않는다. 에이전트 드론이 수 초에서 수십 분간 비행을 전개해야 하는 특성을 고려하여, Action 노드는 nav2_cmds 또는 커스텀 액션 서버로 골(Target Goal) 패킷을 RPC 호출 형태로 디스패치(Dispatch)하고 자신은 지속적으로 RUNNING 상태를 트리 스케줄러에 반환한다. 이러한 비동기(Asynchronous) 설계 지향성은, Action 노드가 무거운 궤적 연산이나 물리 모터를 직접 점유하지 않고 그저 백그라운드 태스크의 경과를 폴링(Polling)만 하도록 분리시냄으로써 트리 전체 엔진(bt_navigator)의 평가 주파수(Tick Frequency)가 프레임 드롭 없이 유지되게 보장한다.

3. 전역 상태 진단 및 게이트웨이(Gateway)로서의 조건 노드(Condition Node)

Condition Node 역시 BT의 말단(Leaf)에 부착되지만, 물리적 환경 상태를 절대 변경하지 않는 읽기 전용(Read-Only) 진단 모듈이라는 점에서 Action Node와 엄격하게 구별된다.

이 노드의 책무는 ‘목표 궤적과 현재 에이전트 포즈 간의 잔여 편차가 오차 허용 범위(Tolerance) 이내인가?’(GoalReached), 혹은 ‘배터리 전압이 랜드 임계치 이하로 추락했는가?’(isBatteryLow) 와 같은 이진(Boolean) 질의를 평가하여 시그널을 즉각적으로 환송(SUCCESS 혹은 FAILURE)하는 것이다. Condition 노드는 RUNNING 상태를 절대 반환하지 않는다. 이러한 특성 덕분에 Condition 노드는 주로 Sequence 노드의 최좌측에 배치되어, 무거운 Action Node 가 불필요하게 기동되는 것을 사전에 차단하는 방화벽(Gateway) 및 사전 조건(Pre-condition Guard) 역할을 완벽히 수행한다.

4. 틱 주파수 변조 및 로직 래핑(Wrapping)을 위한 데코레이터 노드(Decorator Node)

제어 논리의 정밀한 시간-위상 조작(Time-Phase Manipulation) 수단으로서 채택된 Decorator Node는 무조건 단 하나 유일한 자식(Single Child)만을 보유해야 하는 제약을 가진 특수 계층이다. 이 노드는 내재된 서브 트리의 틱 전파 주기를 변형하거나 그 자식이 반환한 원래의 상태 팩터를 역전(Invert)시킨다.

대표적으로 RateController 데코레이터는 하위 플랜 생성 서브 트리(예: ComputePathToPose)를 감싸 틱의 진입을 강제로 통제한다. bt_navigator 글로벌 틱이 100Hz 로 몰아치더라도 RateController가 1Hz 로 설정되어 있다면, 이 서브 폴더 안으로는 1초에 단 한 번만 틱이 허용된다. 이는 고비용 A^* 경로 탐색 연산이 쓸데없이 남용되어 CPU L3 캐시를 마비시키는 것을 방어하는 핵심 보호 기법이다. 또한 Inverter 노드는 하위 상태를 역전시켜, FAILURE를 SUCCESS로 위장(Spoofing)함으로써 시스템 설계자가 복잡한 NOT 게이트 회로 논리를 무상태성 패러다임 하에서 손쉽게 전개하도록 지원한다.
