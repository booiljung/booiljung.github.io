52.6 복합 장애 상황의 이탈을 위한 반응형 시퀀스(Reactive Sequence) 및 복구(Recovery) 논리 전이 역학

1. 비동기 환경 변수 개입과 일반 제어 노드(Control Node)의 맹점

전통적인 행동 트리(BT)의 제어 노드인 Sequence와 Fallback은 틱(Tick)이 자식 노드들에게 순차적으로 전파되며, 특정 자식이 장기 실행 태스크로 인해 RUNNING 상태를 피드백할 경우, 다음번 글로벌 틱 루프 타임에서는 이전에 SUCCESS를 반환했던 앞선 자식 노드들을 건너뛰고(Skipping) 곧바로 해당 RUNNING 노드로 직행하는 성질을 띤다.

그러나 무인 에이전트 환경에서는 앞선 계층에서의 사전 조건(Pre-condition)이 런타임 중에 무효화되는 상황이 빈번하게 발생한다. 예를 들어 드론이 “1. [조건] 배터리가 충분한가? (SUCCESS) \rightarrow 2. [액션] 대상 지점으로 비행하라 (RUNNING 중)“의 Sequence를 밟고 있을 때, 비행 도중 배터리 잔량이 임계치 밑으로 추락하는 물리적 인터럽트(Interrupt)가 발생할 수 있다. 일반적인 Sequence는 이미 1번 조건 노드가 SUCCESS로 종결되었다고 캐싱(Caching)해 두었으므로, 뒤늦게 발생한 배터리 고갈 사태를 다음 틱에서 재검사하지 않고 비행을 강행하다 추락하는 논리적 맹점(Blind Spot)을 노출하게 된다.

2. 반응형(Reactive) 제어 노드의 사전 조건 재평가(Re-evaluation) 메커니즘

이러한 치명적 무시 현상을 타개하기 위해 Nav2 메타 아키텍처는 ReactiveSequence 및 ReactiveFallback 이라는 강화된 제어 노드 계층을 도입하였다. 반응형(Reactive) 노드들은 내부 자식이 RUNNING 상태를 반환하여 스케줄 통제권을 래치(Latch)하고 있더라도, 다음번 글로벌 틱이 인입되었을 때 앞서 SUCCESS로 판정된 이전 자식 노드(조건 노드 등)들을 절대로 스킵하지 않고 강압적으로 처음부터 다시 틱(Tick)하여 재평가(Re-evaluation)를 수행한다.

위의 배터리 시나리오에서 루트가 ReactiveSequence로 설정되어 있다면, 비행 궤적(RUNNING)을 진행함과 동시에 매 틱(100Hz)마다 배터리 검사 조건 노드가 다시 평가된다. 만약 배터리 전압 드롭 이벤트가 포착되어 1번 조건 노드가 FAILURE를 반환하는 순간, ReactiveSequence는 당면한 2번 파이프라인(RUNNING 중인 비행 노드)의 실행을 즉각 파기하고 부모 계층에 치명적 중단 코드를 환송하여, 즉각적으로 시스템을 Fallback 구역의 자율 귀환(Return-To-Home, RTH) 브랜치로 긴급 이탈시킨다.

3. 예외 도래 시의 노드 강제 중지(Halt) 및 자원 회수(Cleanup) 전이

반응형 재평가 논리에 의해 흐름이 끊기거나, 상위 시스템으로부터 선점적 임무 취소 명령인 Cancel이 접수될 경우, BT 프레임워크는 현재 RUNNING 상태에 깊게 계류 중인 모든 자식 및 손자 노드들에게 하향식으로 강제 중지 시그널인 Halt() 메소드를 직렬 통지(Broadcast)한다.

이 Halt() 인터럽트를 수신한 액션 노드(예: FollowPath)는 즉각적으로 자신이 바인딩하고 있던 ROS 2 Action Client의 cancel_goal()을 통신 미들웨어 계층으로 발행하여 백엔드 제어기(Controller Server)의 모터 점유를 물리적으로 절단해야 한다. 더 나아가 할당했던 힙 메모리를 해제(Cleanup)하고 락(Mutex)을 반환함으로써 데드락(Deadlock)과 좀비 스레드를 예방한다. 이 무결한 Halt 체인 아키텍처야말로 복합 장애 시나리오 내에서 BT가 FSM 대비 유연하면서도 메모리 안전(Memory-safe)을 보장하는 핵심 역학 체계론이다.

4. 복구 서버(Recovery Server)와 결합된 계층적 토폴로지 구성

경로 추종 실패, 라이다 센서 사각지대에 의한 코스트맵 단절 등 FollowPath 액션 노드가 FAILURE를 반환할 때, 시스템은 즉각 포기하지 않고 RecoveryNode(종종 RoundRobin이나 Fallback 래퍼로 묶인 특수 형태)로 흐름을 전이시킨다.

Nav2의 Recovery 서브 트리는 가벼운 장애부터 중증 장애까지 계층적으로 발현되는 토폴로지를 구성한다. 일차적으로 제자리 회전(Spin)이나 일정 거리 후진(BackUp)과 같은 단일 ROS 2 액션(Action) 콜을 통해 장애물과의 로컬 탈출(Local Escape)을 모색한다. 일차 복구가 성공하여 SUCCESS가 반환되면 ReactiveFallback 브랜치에 묶인 본래의 글로벌 ComputePathToPose 노드가 다시 틱을 인가받아 경로를 재생성한다. 그러나 수차례의 가벼운 복구 트리 순회에도 불구하고 FAILURE가 잔존할 경우, 시스템은 보다 파괴적인 글로벌 비용 맵 클리어(ClearEntireCostmap) 노드를 트리거하거나, 최종단 플래그 이벤트를 융성하여 지상 통제소(GCS)로 구호 시그널 체인을 상향 전파하는 강건한(Robust) 방어 논리 스택을 완성한다.
