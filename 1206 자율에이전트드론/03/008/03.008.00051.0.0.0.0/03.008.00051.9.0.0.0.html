51.9 고해상도 이미지 및 다중 채널 LiDAR 스트림 병합 스토리지 최적화를 위한 데이터 압축(Compression) 알고리즘 체계

1. 광대역 텔레메트리 누적에 따른 저장 공간의 물리적 한계점

자율 에이전트 드론이 V-SLAM(Visual-Simultaneous Localization and Mapping) 또는 강화학습(RL) 오프라인 데이터 수집을 목적으로 센서 스트림을 기록할 때, 가장 큰 용량을 점유하는 페이로드는 무손실 원시 이미지(.bmp 또는 비압축 sensor_msgs/Image)와 3차원 LiDAR 벡터 배열이다. 통상적으로 60fps 4K 스테레오 카메라 스트림과 32채널 로터리 LiDAR 스캔을 동시 기록할 경우 초당 시스템 버스에는 수백 메가바이트(MB/s)의 트래픽이 쏟아지며, 이는 고작 1시간의 비행 체공 시간만으로도 임베디드 보드에 이식된 1TB NVMe 저장 매체를 포화(Saturation)시키는 물리적 한계를 노출한다.

디스크 공간 고갈 현상은 단순히 수집 시간의 단축을 넘어 OS의 가상 메모리 공간(Swap Area)까지 침범하여 ROS 2 런타임 자체를 붕괴시킬 수 있다. 이러한 빅데이터 텔레메트리 병합(Merging) 기록의 물리적 한계를 극복하기 위해 rosbag2 파이프라인은 미들웨어 차원의 스토리지 데이터 압축(Compression) 알고리즘 체계를 플러그인(Plugin) 인터페이스로 지원한다.

2. 지원 압축 포맷(Zstd, LZ4 등)과 시스템의 압축률 역학

rosbag2의 압축 서브시스템에 동적 연동이 가능한 알고리즘은 압축률(Compression Ratio)과 압축 속도(Speed), 그리고 복원(Decompression) 대역폭 간의 상충 논리(Trade-off)를 가진다.

ROS 2 백엔드에서 널리 채택되는 기본 포맷은 Zstandard(Zstd) 알고리즘이다. Zstd는 BWT(Burrows-Wheeler Transform)의 변형과 엔트로피 코딩을 혼합하여 매우 뛰어난 압축률을 달성하면서도 비교적 일관된 O(N)의 해제(Decompression) 속도를 보장한다. Zstd를 통해 무손실 원시 흑백/RGB 이미지나 중복된 거리 값을 다수 포함하는 3D PointCloud2 배열을 기록할 경우 원본 레코딩 체적의 대략 20 \sim 30\% 이하 수준으로 수축시키는 강력한 정보 엔트로피 압축을 증명한다. 초고속 기록(Record)이 최우선이고 압축률을 일부 양보할 수 있다면 LZ4 포맷으로 전환하여 스레드 인코딩 레이턴시를 최소화하는 것도 가능하다. --compression-format zstd --compression-mode file과 같이 직관적인 매개 변수를 인가함으로써 저장 매체의 논리적 확장을 도모한다.

3. 기록 포맷에서의 메시지 단위(Message) vs 파일/블록 단위(File) 압축 전략

압축 알고리즘을 데이터를 패키징하는 어느 위상에 인가할 것인가를 결정하는 --compression-mode 매개변수는 시스템 재생 및 오류 내성에 직접적인 역학을 미친다.

메시지 단위(message) 압축 모드는 수신 파이프라인이 큐에서 건져 올린 개별 SerializedMessage마다 독립적인 헤더와 압축 해제 딕셔너리를 부여하는 방식이다. 압축 효율 자체는 코덱(Codec) 엔진이 참조할 컨텍스트 윈도우(Context Window)가 협소하여 블록 단위에 비해 다소 떨어진다. 반면, 재생 시 단일 메시지 바이트가 파손되더라도 인접한 직렬 스트림의 무결성을 해치지 않으며 무작위 오프셋 탐색 재생 시간 복잡도를 낮추는 강점을 지닌다.

파일 단위(또는 내부 블록(file/block)) 모드는 다수의 메시지 이벤트를 하나의 커다란 덩어리로 묶은 뒤 광범위한 딕셔너리를 투사하여 엔트로피 코딩을 수행한다. 특히 중복된 환경(예: 드론이 정지 비행 중 주변의 동일한 점군을 반복 수집할 때)에서 데이터 체적을 급격히 붕괴시키는 압도적인 압축률을 달성한다. 단, 이 모드 하에서는 백 파일 시스템의 EOF나 특정 청크 인덱스가 유실되었을 때 단일 바이트의 밀림 현상만으로 해당 블록 전체가 파손 페이로드로 확산(Cascade)되는 높은 오류 취약성을 동반한다.

4. 임베디드 환경에서의 CPU 오버헤드와 I/O 병목 상쇄 효과 정량 분석

엣지 컴퓨터 환경에서 실시간 압축 알고리즘을 구동하는 것은 필연적으로 기록(Writer) 데몬에 바인딩된 백그라운드 스레드의 CPU 유효 연산(Instruction Clock) 사이클을 점유한다. Zstd 압축 레벨을 상향시킬수록 연산 복잡도가 증가하여, 데이터 인입 주파수가 특정 한계 임계값을 초과할 때 CPU 유휴 자원 고갈에 의한 버퍼 오버플로우(Buffer Overflow)가 발생하게 된다.

그러나 실험적 모델링을 통해 \lambda_{compress} (압축 후 디스크로 플러시되는 바이트 총량)와 \mu_{io\_bandwidth} (디스크 하드웨어 자체 쓰기 한계)의 역학 관계를 조명하면 고도화된 타협점 도출이 가능하다. 원시 스트림이 SD 카드나 eMMC의 쓰기 허용 대역폭인 50MB/s를 돌파하여 I/O 병목(Disc Stall)이 발생 중일 때, CPU 자원의 10 \sim 15\%를 지불하여 Zstd 레벨 1 압축을 투입하면 출력 데이터 대역폭(\lambda_{compress})이 15MB/s 수준으로 하강한다.

결과적으로 압축 프로세싱의 CPU 오버헤드 비용이 물리 디스크 I/O 기계적 대기 시간에 의한 로우 큐(Raw Queue) 누적 페널티 비용을 완벽히 상쇄(Offset) 및 역전시켜, 전체 기록 파이프라인의 실시간성을 확립하는 구조적 돌파구(Breakthrough)로 작동하게 된다.
