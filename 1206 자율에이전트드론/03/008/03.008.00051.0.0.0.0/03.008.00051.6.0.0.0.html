51.6 재생(Playback) 제어 모듈의 가변 속도(Time Scaling) 제어 기능 및 타임스탬프 스푸핑(Spoofing) 메커니즘

1. 시간 도메인의 분리 및 가상 시계(Virtual Clock) 아키텍처

저장 매체에 직렬화된 rosbag2 데이터를 미들웨어 생태계로 다시 펌핑(Pumping)하는 재생(Playback) 과정은 단순한 I/O 스트리밍이 아니다. 기록 당시의 물리적 절대 시간(Wall Time)과 재생 런타임의 시스템 시간이 충돌할 경우, 제어 및 상태 추정 알고리즘들(예: EKF, Nav2 Controller)은 과거 데이터의 \Delta t를 거대한 미래 지연(Delay)으로 역산하여 시스템 에러를 트리거한다.

이를 방지하기 위해 rosbag2 재생 모듈은 호스트 OS의 RTC(Real-Time Clock)와 독립된 가상 시계(Virtual Clock) 아키텍처를 가동한다. 재생이 시작되면 플레이어 노드는 백(Bag) 파일에 명시된 메시지의 원본 타임스탬프 t_0 를 새로운 재생 좌표계의 영점(Epoch)으로 매핑하며, --clock 파라미터를 통해 ROS 2 네트워크 상의 /clock 토픽으로 이 가상 시간을 브로드캐스팅한다. 모든 청취 노드(Listener Nodes)는 use_sim_time 파라미터를 true로 설정함으로써 내부 오실레이터의 참조점(Reference Point)을 시스템 클럭에서 이 가상 통제 클럭으로 전환하게 된다.

2. 알고리즘 검증을 위한 가속/감속(Time Scaling) 역학

오프라인 알고리즘 디버깅이나 머신러닝 모델의 평가(Validation) 과정에서는 1배속(Real-time) 재생의 한계를 뛰어넘는 비선형적 런타임 조작이 요구된다. rosbag2 플레이어는 --rate 매개변수로 정의되는 가변 속도(Time Scaling) 제어 메커니즘을 지원한다.

가변 속도 계수가 R = 2.0 (2배속 가속)으로 주어질 경우, 내부 스케줄러는 인접한 두 메시지의 레코드 시간차 \Delta t_{recorded} 를 물리적으로 대기하는 수면 함수(Sleep Function) 체공 시간을 sleep\_duration = \Delta t_{recorded} / R 로 축소 연산하여 디스패치(Dispatch)한다. 반대로 R = 0.5 (절반 속도 슬로우 모션)일 경우에는 수면 시간을 팽창시킨다. 이 스케일링 벡터 추종을 통해 자율 주행의 장거리 경로 플래닝 시나리오를 수 분 내에 고속 검증하거나, 라이다 포인트 클라우드가 일그러지는 엣지 케이스(Edge Case)를 프레임 단위로 지연 분석할 수 있다.

3. 타임스탬프 스푸핑(Spoofing)을 통한 실시간성(Real-time) 변조

과거에 로깅된 센서 데이터를 실시간 스트림으로 조작(Spoofing)해야 하는 HIL(Hardware-In-the-Loop) 시뮬레이션 환경이 존재한다. 이때는 단순히 가상 시계(/clock)를 퍼블리시하는 것에 그치지 않고, 메시지 헤더(std_msgs/Header) 계층에 각인된 원본 타임스탬프 자릿수 자체를 재생 시점의 호스트 OS 현재 시간으로 전면 개서(Rewrite)해야 한다.

rosbag2는 이 목적을 달성하기 위해 메시지 전송 직전에 헤더 객체의 stamp 필드를 시스템의 현재 에포크(Epoch) 시간으로 치환하여 재구성하는 타임스탬프 오버라이드(Override) 기능을 프록시(Proxy) 단에서 제공할 수 있다. 이러한 조작 인젝션은 수신기가 데이터의 출처가 과거의 백 파일인지, 아니면 현재 직접 연결된 물리적 디바이스인지 분간할 수 없게 함으로써 투명한 인젝션 테스팅(Transparent Injection Testing)을 가능하게 한다.

4. 가변 재생 시뮬레이션의 QoS 및 버퍼 오버런(Overrun) 한계

디스크 I/O 스루풋이 방대한 텍스트, 이미지 묶음에 대해 R = 5.0 과 같은 고도 가압(High-Rate Scaling)을 수행할 때는 치명적인 재생 부작용(Side Effects)이 발생한다. 메시지를 미들웨어로 내보내는 빈도가 DDS 전송 레이어의 QoS(Quality of Service) 버퍼 용량 한계나 네트워크 대역폭(예: Loopback Interface 10Gbps 리밋)을 정복하게 되면, 브로드캐스트 자체가 블로킹(Blocking)되거나 대규모 패킷 폐기(Packet Drop)가 일어난다.

이러한 네트워크 및 버퍼 오버런(Buffer Overrun) 상태에서는 기록 당시에 완벽히 보존되었던 다중 센서 간의 타임스탬프 위상(Phase) 일치성이 랜덤하게 붕괴된다. 카메라 프레임은 버려지고 간헐적인 IMU 패킷만 생존하여 로컬 플래너로 수신될 경우, V-SLAM이나 EKF 기반 오도메트리 솔버는 발산(Divergence) 오류를 방출하며 셧다운된다. 고배속 재생 환경에서는 대상 노드들의 구독자 QoS 프로파일 허용치를 우회하거나 최적화하는 IPC 제로 카피 링킹 조치가 동반되어야만 타임 스케일링 모델의 무결성을 획득할 수 있다.
