51.7 시간 도메인 클럭 지속성(Clock Continuity) 유틸리티를 위한 시뮬레이션 환경 내 /clock 토픽 동기 규격

1. 하드웨어 클럭(Hardware Clock)과 재생 시간의 위상 충돌 매커니즘

운영체제(OS)를 기반으로 하는 모든 컴퓨터 리소스는 내부 크리스탈 오실레이터(Crystal Oscillator)에 종속된 RTC(Real-Time Clock)와 System Clock을 참조하여 런타임 수명 주기를 결정한다. 자율 드론의 내비게이션 스택(Nav2)이나 EKF 알고리즘 모듈 역시 기본적으로 이 호스트 시스템 클럭을 기반으로 dt (Delta Time)를 산출하여 상태 추정 및 속도 투영에 응용한다.

그러나 과거의 비행 환경에서 추출된 rosbag2 페이로드를 현재의 시스템 아키텍처 위에서 평가용으로 재생(Playback)할 때, 드론 통제기는 호스트의 ’현재 시각’과 유입되는 센서 데이터 헤더에 기록된 ‘수 시간 전 혹은 수 일 전 시각’ 사이에서 거대한 타임 점프(Time Jump) 즉, 위상 충돌 현상을 인지하게 된다. 이 충돌은 PID 제어기의 적분/미분 텀(Term) 발산과 TF 브로드캐스터의 Extrapolation 지연 예외(Exception)를 트리거하여 전체 로직을 즉각적으로 셧다운시킨다. 이러한 타임라인 왜곡에 대응하기 위해 ROS 2 미들웨어는 추상화된 클럭 객체를 도입하여 시간 인지 주체를 동적으로 교체하는 아키텍처를 강제한다.

2. use_sim_time 파라미터 매니페스트 및 노드 클럭 래치(Latching) 체계

시간 참조의 충돌을 방어하는 ROS 2 핵심 규격의 중심에는 use_sim_time 파라미터가 자리잡고 있다. 자율 비행 시스템 노드(Node) 생성 시, 해당 파라미터를 명시적으로 활성화(true)하면 객체 내부에 바인딩된 시간 팩토리(Time Factory) 속성이 RCL_SYSTEM_TIME에서 RCL_ROS_TIME 참조 모델로 전환된다.

이 상태로 상전이된 노드들은 내부 로직에서 호스트 OS의 시스템 콜백 타이머를 차단(Latching)하고, 오로지 ROS 2 네트워크의 글로벌 네임스페이스인 /clock 토픽에 발행(Publish)되는 가상의 타임스탬프 메세지(rosgraph_msgs/Clock)만을 시스템 현재 시간으로 인가(Acknowledge)한다. 이러한 래칭 구조는 수십 개의 독립된 서브 프로세스들이 완벽하게 단일화된 모의 시간(Simulated Time) 도메인 아래에서 논리적 시계열을 전개하도록 보장하는 소프트웨어 스위치이다.

3. rosbag2 브로드캐스트와 외부 시뮬레이터 간의 클럭 주도권(Mastery) 모델링

시뮬레이션 재생 단계에서 가상 시간 /clock 토픽을 주도(Mastery)하는 객체는 크게 두 가지 환경으로 분리된다. 첫째는 순수한 rosbag2 재생 유틸리티이다. 레코더 플레이어가 --clock 옵션으로 실행될 때, 플레이어 스레드 자체가 과거의 물리 데이터와 맞물려 동작하는 가상 클럭 생성자가 되어 1Hz ~ 1000Hz 단위로 토픽을 갱신한다. 이는 로봇 하드웨어나 HIL(Hardware-In-the-Loop) 테스팅 데스크에 센서 입력을 단독 투입할 때 활용된다.

둘째는 물리 엔진 기반 시뮬레이터(Gazebo Classic / Ignition) 와의 공존 상태이다. 시뮬레이터는 중력 가속도 연산 및 링크(Link) 간 충돌 처리를 위해 자신만의 동역학 기반 가상 시계를 /clock에 강력하게 통제 송신한다. 만약 이 환경에서 rosbag2가 브로드캐스트하는 타임스탬프를 덮어씌울 경우, 두 시계 생성자 간의 경합(Clock Contention)에 의해 TF 타이밍 에러가 범람하게 된다. 따라서 SITL(Software-In-The-Loop) 모델 융합 환경에서는 rosbag2의 클럭 권한을 정지시키고 시뮬레이터가 조율하는 물리 시간에 맞추어 수동적으로 레코드된 토픽들을 주입하는(Injected) 종속 재생 규격 설계가 필수적이다.

4. 시계열 일관성(Clock Continuity)를 위한 재생 단절 복구 아키텍처

루프 재생(Loop Playback)은 과거 데이터를 시뮬레이터 환경에서 무한 회귀 테스트하기 위한 필수 요건이나, 백 파일의 종점 t_{end} 에서 시작점 t_{start} 로 데이터가 회귀하는 지점에서 심각한 시계열 일관성 단절이 유발된다. 시간 위상이 과거로 수직 낙하(Jump Backward)하는 이 현상은 Nav2 코스트맵의 낡은 장애물 지우기 스레드나 EKF(Extended Kalman Filter) 행렬 계산식에 발산 지향적 크리티컬 에러를 주입한다.

이를 무효화하기 위해 시간 도메인 클럭 지속성(Clock Continuity) 유틸리티는 반복 재생 시 \Delta t를 재초기화(Reset)하여 이전 주기의 t_{end} 이후부터 연속적인 양의 상수를 가산하는 논리 추상화를 적용하거나, /clock 토픽이 뒤로 도약하는 이벤트를 감지할 때마다 로컬 제어기 파이크라인의 내부 타이머 상태 변수 및 에러 누적 버퍼를 프로그램적으로 일괄 소거(Flush)하는 역학적 복원 구조(Reset Workaround)를 동원해야 한다.
