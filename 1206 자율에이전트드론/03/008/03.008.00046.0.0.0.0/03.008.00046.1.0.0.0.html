46.1 sensor_msgs 통합 데이터 모델의 물리적 차원(Dimensionality) 및 SI 국제단위계(SI Unit) 표준 명세

1. 개요

자율 에이전트 체계에서 복잡한 센서 네트워크가 원활하게 상호작용하기 위해서는 각각의 센서 하드웨어가 생성하는 물리적 계측치(Physical Measurements)가 통일된 데이터 모델로 캡슐화(Encapsulation)되어야 한다. ROS2의 sensor_msgs 패키지는 이러한 이기종 데이터를 일관성 있게 다루기 위한 통합 데이터 모델을 제공하며, 특히 물리적 차원(Dimensionality)의 엄밀한 정의와 SI 국제단위계(International System of Units)의 일괄 적용을 원칙으로 삼고 있다.

본 장에서는 sensor_msgs 패키지를 구성하는 핵심 데이터 모델들이 어떻게 물리적 계측 환경의 차원을 추상화하고 있는지, 그리고 상태 추정, 제어 이론, 동기화 아키텍처에서 발생할 수 있는 스케일 모순(Scale Contradiction)을 방지하기 위해 SI 단위계가 시스템 전반에 걸쳐 어떻게 규격화되어 수치적 안전성(Numerical Stability)을 확보하는지 학술적 관점에서 분석한다.

2. 물리적 차원(Dimensionality)의 추상화 모델

sensor_msgs 내의 메시지 구조체들은 측정하는 물리 현상의 차원에 따라 그 자료구조가 설계된다.

2.1 1차원 시계열 데이터(1D Time-Series Data)

IMU(가속도, 각속도), 배터리 상태표시, 온도계 등 특정 시간 t에 대한 단일 스칼라 혹은 정해진 축(Axis)에 대한 벡터 출력을 제공하는 하드웨어는 선형적 시계열 모델로 추상화된다. sensor_msgs/msg/Imu나 sensor_msgs/msg/BatteryState와 같은 모델은 \mathbb{R} 혹은 \mathbb{R}^3 공간의 고정 크기 벡터(Fixed-size Vector)로 정의되어, 직렬화(Serialization) 시 페이로드 오버헤드를 최소화하고 시스템 콜(System Call) 주기를 최단으로 유지할 수 있는 자료 구조를 취한다.

2.2 공간 텐서 데이터(N-Dimensional Spatial Tensor)

카메라 이미지나 LiDAR 포인트 클라우드는 다차원 공간에 분산된 매트릭스 혹은 텐서 형태로 수집된다. sensor_msgs/msg/Image 및 sensor_msgs/msg/PointCloud2 데이터 모델은 N차원의 데이터를 1차원 바이트 배열(std::vector<uint8_t>)로 직렬화하여 송신하고, 폭(Width), 높이(Height), 스텝(Step/Row_step)과 같은 메타데이터 필드를 부여하여 수신 노드가 해독 시 O(1) 복잡도로 임의의 공간 좌표에 해당하는 데이터 포인트로 재구성(Reconstruction)할 수 있도록 차원 정보를 명시적으로 제공한다. 이는 메모리 단편화(Fragmentation)를 막고 DDS 계층 전송 효율을 극대화하는 컴파일 타임(Compile-time) 최적화의 산물이다.

3. SI 국제단위계(SI Unit) 표준 준수 규격

개별 하드웨어 제조사가 제공하는 SDK(Software Development Kit)는 독자적인 단위계를 채택하는 경우가 잦다(예: 가속도의 단위로 g-force 혹은 mg, 각속도의 단위로 RPM 혹은 Degree/sec 사용). 로봇 소프트웨어 공학에서 이러한 이질성은 곧 상태 피드백 루프의 치명적 발산(Divergence)으로 이어진다.

ROS2는 REP-0103(ROS Enhancement Proposal 103) 표준 명세에 의거하여 sensor_msgs 패키지를 거치는 모든 물리량에 대해 SI Base Units 및 Derived Units의 엄격한 사용을 강제한다.

3.1 동역학 파라미터 표준 단위원

드론의 비행 제어기(Flight Controller) 파이프라인으로 유입되는 sensor_msgs/msg/Imu의 경우, 선형 가속도는 미터 매초 제곱(m/s^2), 각속도는 라디안 매초(rad/s) 단위로 변환되어 퍼블리시(Publish)되어야 한다.
자기장 데이터(sensor_msgs/msg/MagneticField)는 마이크로테슬라(\mu T)나 가우스(Gauss)가 아닌 테슬라(T)를 기본 단위로 규정하여, 별도의 단위 환산 함수 블록 없이 EKF(Extended Kalman Filter) 행렬 연산 모듈에 직접 투입될 수 있는 선형성(Linearity)을 확보한다.

3.2 공간 및 광학(Optical) 파라미터 척도

레이저 스캔 데이터(sensor_msgs/msg/LaserScan)의 거리(Range) 단위는 미터(m)로, 각도(Angle)의 증분 등은 모두 라디안(rad) 단위로 통일된다. 또한 온도(sensor_msgs/msg/Temperature)나 조도(sensor_msgs/msg/Illuminance) 데이터 역시 각각 섭씨(^\circ C) 스케일을 허용하지 않고 오직 단위 켈빈(K)과 럭스(lx)를 규준으로써 엄격하게 정립한다.

4. 단위 변환 로직 누락에 따른 시스템 무결성 훼손 메커니즘

sensor_msgs 표준 명세를 따르지 않고 원시(Raw) 데이터를 그대로 직렬화하는 드라이버 노드가 시스템에 병합될 경우, PID 제어기의 비례 이득(Proportional Gain)과 적분 이득(Integral Gain)이 예상된 추력 한계선(Thrust Baseline)을 초과하게 만드는 제어 불능 현상을 유발한다.
이는 관측-제어 결합망(Observation-Control Coupled Network) 내에서 제어 편차(Error)의 차원 단위계의 오염으로 인해 역학 모델의 수학적 형태가 붕괴되는 원인이 되므로, 각 센서의 브릿지 드라이버 모듈 레이어에서 반드시 SI 표준 변환을 강제 연산 처리하는 전처리 계층(Preprocessing Layer) 도입이 학술적으로 엄격하게 권고된다.
