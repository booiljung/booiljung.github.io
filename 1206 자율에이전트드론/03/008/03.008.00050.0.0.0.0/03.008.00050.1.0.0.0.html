50.1 pointcloud_to_laserscan 차원 축소(Dimensionality Reduction) 아키텍처 및 ROS2 기반 시스템 연산량 제약 한계치 모델링

1. 차원 축소 아키텍처의 학술적 개요

자율 에이전트 드론의 인식(Perception) 시스템에서 3D LiDAR 센서는 주변 환경의 고해상도 공간 정보를 제공하지만, 처리해야 할 데이터 포인트의 수가 기하급수적으로 증가하는 차원의 저주(Curse of Dimensionality) 문제를 수반한다. pointcloud_to_laserscan 패키지는 이러한 3차원 포인트 클라우드 공간(\mathbb{R}^3) 내의 데이터를 특정 고도와 각도 조건에 따라 여과한 후 2차원 평면(\mathbb{R}^2) 상의 거리 프로파일 구조체인 투영 스캔(Projected Scan)으로 변환하는 차원 축소(Dimensionality Reduction) 아키텍처를 구현한다. 이 아키텍처의 주된 학술적 목표는 드론의 순항(Cruising) 혹은 국지적 장애물 회피 시 과도한 공간 지각 연산 부하를 최소화하고, 제어 루프(Control Loop)의 주기를 단축하는 데 있다.

2. pointcloud_to_laserscan 패키지의 아키텍처 구조

pointcloud_to_laserscan 시스템 아키텍처는 데이터 파이프라인 관점에서 수신(Subscription), 필터링 및 변환(Filtering & Transformation), 정규화(Normalization), 그리고 발행(Publishing) 레이어로 설계되어 있다.

  1. 동적 공간 필터링 레이어: sensor_msgs/msg/PointCloud2 메시지를 수신한 후, 드론의 로컬 좌표축(일반적으로 Base Link 좌표계)을 기준으로 Z축의 임계 영역(Z_{min} \le Z_i \le Z_{max})을 설정하여 관심 영역(Region of Interest, ROI) 밖의 포인트를 우선적으로 소거한다. 이는 전체 시스템 연산의 오버헤드를 줄이는 선제적 조건부 분기(Conditional Branching)이다.
  2. 평면 사상 및 이산화(Discretization) 레이어: 남은 거시적 3D 포인트 집합을 물리적 기하학에 기반해 2D 방사형 격자(Radial Grid) 상에 사상한다. 유클리드 거리 좌표 (x_i, y_i, z_i)는 극좌표 (r_i, \theta_i) 구조로 변환되며, 여기서 \theta_i = \arctan2(y_i, x_i)이다.
  3. 최적 대표값 선택 레이어: 이산화된 하나의 각 해상도 \Delta\theta(Angle Increment) 구간 내에 복수의 포인트 포인트가 투영될 경우 최단 거리 속성을 갖는 r_{min}을 해당 각도의 대표 방사형 거리로 결정한다.

3. ROS 2 환경에서의 시간 복잡도 모델링

포인트 클라우드의 초기 N 포인트 크기가 \sim \mathcal{O}(10^5)에서 \mathcal{O}(10^6) 단위라고 가정할 때, 3D Voxel Grid 필터나 복잡한 3D 표면 법선 추정 알고리즘을 수행할 경우 연산 비용은 드론의 내장 시스템에서 병목(Bottleneck) 현상을 유발한다. 반면, pointcloud_to_laserscan 노드로 압축을 실행할 경우의 알고리즘 시간 복잡도(Time Complexity)는 원시 배열 포인터에 대한 1회 단일 순회를 허용하므로 \mathcal{O}(N) 선형 시간(Linear Time)에 변환 작업이 종결된다.

이후 변환된 1차원 배열(1D Array Format)인 sensor_msgs/msg/LaserScan의 인덱스 크기를 M이라고 정의할 경우, M = \lfloor (\theta_{max} - \theta_{min}) / \Delta\theta \rfloor 에 의해 일반적으로 \sim \mathcal{O}(10^3) 내외의 공간 상수로 축소되며, 이를 수신하는 하위 Nav2 모듈(예: Local Costmap Generator)은 \mathcal{O}(N)이 아닌 축소된 \mathcal{O}(M) 기반의 평면 스위프(Plane Sweep) 처리를 지속하게 되어 전체적인 제어 시스템 연산율이 하향 안정화된다.

4. 시스템 연산량 제약 한계치 모델링

연산 리소스가 제한된 엣지 인공지능 통제기 내에서, 차원 축소 전후의 시스템 프로세서 부하(System Processor Payload Constraint) 한계치 모델링은 연산 허용 시간 T_{limit} 및 시스템 명령어 처리율 I로 공식화할 수 있다.

원시 3D 처리 시 필요한 추정 연산 주기 T_{3D}가 제어 루프 제한 시간 T_{limit}을 초과하는 조건, 즉 T_{3D} > T_{limit}일 경우 제어의 불안전성이 촉발된다. 이때 pointcloud_to_laserscan를 통과한 스위프 모델의 연산 주기 T_{2D}는 다음과 같은 한계치 관계식을 성립시켜야 한다.

T_{2D} = T_{transform}(N) + T_{planner}(M) < T_{limit}

여기서 T_{transform}(N)은 N개의 포인트 클라우드를 M 바구니(Bins)로 사상하는 데 드는 소요 시간이며 하드웨어의 클럭에 선형 비례한다. 결과적으로 3차원 포인트 클라우드 자체의 기하학적 체적(Volume) 평가를 희생하는 대신, T_{2D}가 자율 드론의 비행 제어 상한 주기(일반적으로 10Hz~50Hz) 이내에 수렴하도록 보장하는 것이 본 차원 축소 아키텍처를 도입하는 절대적 제약 한계치 모델링의 본질이다.
