50.3 기하학적 z축 높이 투영(Height Projection) 범위를 이산화하는 임계값(min_height, max_height) 동적 필터링 방정식

1. 고도 투영(Height Projection)의 기하학적 필요성

자율 드론의 비행 환경은 실내 지면효과(Ground Effect) 구간부터 협소한 통로 구조물에 이르기까지 3차원적 변위가 연속적으로 일어나는 공간이다. 3D LiDAR가 획득한 방대한 PointCloud2 데이터 전체를 2D 평면 스캔으로 변환할 경우, 천장의 조명 기구, 비행 고도보다 한참 아래에 있는 바닥의 패턴 등 드론의 충돌 위험 반경과 무관한 환경 변수들까지 2D 스캔 데이터의 장애물로 매핑(Mapping)되는 치명적인 왜곡 현상(Distortion)이 발생한다.

이러한 수직 공간의 불필요한 노이즈를 억제하고 로봇의 입체적 물리 규격(Kinematic Footprint)에 기반한 유효 충돌 회피 영역(Effective Collision Avoidance Zone)만을 2D 평면으로 이산화(Discretization)하기 위해서는, 기하학적 Z축 투영 범위를 수학적으로 제한하는 고도 필터링(Height Filtering) 과정이 선행되어야 한다.

2. 정적 임계값 기반의 공간 이산화 방정식

pointcloud_to_laserscan 패키지의 파라미터로 제공되는 min_height (z_{min}) 와 max_height (z_{max})는 센서 좌표계 기준에서 유의미한 장애물 층(Layer)을 추출하기 위한 수직적 임계값 함수이다. i번째 3D 포인트의 높이를 z_i라고 할 때, 투영에 참여하는 필터링된 유효 포인트 집합 P_{valid}는 다음과 같은 조건부 대수 방정식으로 정의된다.

P_{valid} = \{ p_i(x_i, y_i, z_i) \in P \mid z_{min} \le z_i \le z_{max} \}

이 방정식은 유클리드 공간 \mathbb{R}^3을 두 개의 평행한 무한 평면 Z = z_{min}과 Z = z_{max}로 절단(Clipping)하여 닫힌 슬라이스 형상의 부분공간(Subspace)을 생성하는 수학적 이산화 행위에 해당한다. 센서가 드론 정중앙에 부착되어 있고 드론의 수직 높이 제원이 H_{drone}일 때, 가장 보수적인 안전율을 고려한 정적 투영 범위는 센서 장착점 기준 z_{min} \approx -(H_{drone}/2 + \delta), z_{max} \approx +(H_{drone}/2 + \delta) 로 설정될 수 있다 (여기서 \delta는 제어 여유 마진 상수).

3. 동역학적 섭동을 반영한 동적 임계값 필터링

드론은 지상 이동 로봇(UGV)과 달리 이동 중 피치(Pitch)와 롤(Roll) 각도 변화가 극심하게 발생한다. 비행 중 피치 각 \theta_{pitch}가 양(Positive)의 방향으로 틸팅(Tilting)될 경우, 센서의 전방 관측 시야는 위를 향하게 되고 하단 관측 시야는 바닥을 향하게 된다. 이때 정적으로 고정된 z_{min}, z_{max} 임계값을 유지할 경우, 바닥 지형이 z_{min} 값의 경계를 침범하여 가상의 충돌체로 잘못 투영되는 고스팅 오차(Ghosting Error)가 필연적으로 동반된다.

따라서 6자유도(6-DoF) 기반 비행체 환경에서는 고정 필터링을 넘어 드론의 실시간 IMU(Inertial Measurement Unit) 기울기 피드백에 비례하여 투영 범위를 동기화하는 동적 필터링(Dynamic Filtering) 방정식이 도입되어야 한다. 드론의 기준 프레임(Base Footprint) 대비 수평면(World Frame) 간의 자세 변화행렬을 \mathbf{R}(\phi, \theta, \psi) 라고 할 때, z_{min} 과 z_{max} 는 다음과 같은 동적 변화(Dynamic Thresholding) 모델의 영향을 받는다.

z_{min}'(t) = f(\mathbf{R}(t), z_{min}, \text{LaserScan Range})
z_{max}'(t) = f(\mathbf{R}(t), z_{max}, \text{LaserScan Range})

실제 구현상에서는 ROS 2의 TF2 (Transform Library) 엔진 내에서 센서 프레임을 로컬 수평면 베이스 프레임(base_footprint 또는 안정화된 수평 기준 좌표계)으로 미리 역투영 행렬 연산(Inverse Projection)을 거친 후에 pointcloud_to_laserscan 노드로 전달하여 필터링하는 파이프라인이 일반적이다.

4. 필터링 최적화를 통한 제어 안정성 확보

z_{min}과 z_{max}를 좁은 간격으로 이산화할 경우(협대역 필터), 처리할 포인트 수가 급감하여 연산 부하는 비례적으로 감소하나 상하로 입체적인 형상의 나뭇가지나 구조물의 돌출부를 인지하지 못하는 오탐지율(False Negative)이 증가한다. 반대로 설정 범위를 과도하게 넓힐 경우(광대역 필터), 바닥 정적 노이즈가 스캔 반경으로 혼입되어 드론이 정방향을 불필요한 장애물로 인지하고 회피를 멈추는 데드록(Deadlock)에 빠질 위험이 있다.

그러므로 min_height와 max_height의 간격 절충(Trade-off) 전략은 단순히 연산량 제어 차원의 파라미터 튜닝이 아니라, 자율 드론의 3차원 체적 방어선(Volume Defense Line)을 어떻게 2차원 공간 단면 계층으로 모델링할 것인가를 결정하는 중대한 설계 요소이다. 주행 환경의 수직적 복잡도 수치와 드론 기체의 최대 피치-롤 제어 한계 등 다면적 특성을 정량화하여 투영 윈도우 슬라이스 모델을 산출해야 한다.
