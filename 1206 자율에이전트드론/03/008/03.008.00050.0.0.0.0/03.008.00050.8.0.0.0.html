50.8 실내 구조물 주행 중 피치/롤 섭동에 의한 고도 변위(Altitude Shift)를 보상하는 슬라이스(Slice) 윈도우 동적 보정

1. 비행 자세 섭동(Attitude Perturbation)과 인식 윈도우의 기하학적 왜곡

평탄한 노면을 주행하는 자율 주행 자동차(UGV)와 달리, 자율 에이전트 드론(UAV)은 전진 및 측방향 이동을 위해 필연적으로 기체를 기울이는 역학적 특성을 지닌다. 특히 실내 비행 환경이나 협소한 벽면 및 천장 사이의 회랑(Corridor)을 비행할 때, 드론은 공기역학적 난기류(Turbulence)와 자세 제어기의 보상 작용으로 인해 지속적인 피치(Pitch) 및 롤(Roll) 각도 섭동(Perturbation)을 겪는다.

만약 3D 포인트 클라우드를 2D 레이저 스캔으로 투영(Projection)하는 시스템이 드론 기체(Base Link) 프레임에 견고하게 고정(Rigidly Tied)된 상태로 작동한다면, 드론이 전진을 위해 피치 각도를 낮추는 순간 센서의 인식 윈도우(Viewing Frustum) 역시 아래를 향하게 된다. 이는 정상적인 수평 비행 조건에서 필터링되도록 맞춰진 min_height와 max_height 기반의 투영 슬라이스(Slice) 윈도우가 바닥 면적을 장애물로 포착하게 만드는 기하학적 왜곡(Geometric Distortion) 현상을 발생시킨다.

2. 고도 변위(Altitude Shift)에 따른 코스트맵(Costmap) 오차 심화

자세 섭동으로 인한 슬라이스 윈도우의 이탈 현상은 드론 시스템 내에서 고도 변위(Altitude Shift)라는 형태의 데이터 오염을 초래한다. 예를 들어, min_height를 0.1m로 설정하여 드론 하단부의 근접 물체만을 걸러내려 했으나, 10^\circ의 피치 섭동이 발생하는 순간 기체 전방 1.5m 거리에 있는 안전한 바닥면의 포인트 클라우드 레이어들이 0.1m 투영 슬라이스 범위 안으로 밀려 올라오게 된다.

이러한 고역학적 비행 자세 변화 속에서 파손된 스캔 정보가 Nav2 스택(Stack)의 로컬 코스트맵(Local Costmap) 레이어에 지속적으로 누적될 경우, 드론은 전방에 투명한 벽(Virtual Wall)이 생성된 것으로 오판속(Misjudgment)하여 전진 유효 속도(Velocity Command)를 급감시키거나 후진 및 회피 기동을 반복하는 스위칭(Switching) 또는 진동(Oscillation) 모드에 빠지게 된다.

3. 좌표계 변환 행렬(Transformation Matrix) 기반 동적 보정 방정식

이러한 문제를 극복하기 위하여 pointcloud_to_laserscan 노드는 단순히 센서 프레임의 원시 포인트를 수치적으로 절삭하는 것이 아니라, 중력 방향을 참조하여 항상 수평을 유지하는 관성 기준 프레임(Inertial Reference Frame, 일반적으로 base_footprint 또는 odom)으로 데이터를 역투영사(Inverse Projection)한 뒤 슬라이스 윈도우를 적용하는 동적 보정 체계를 요구한다.

센서 프레임 S에 측정된 3차원 포인트 행렬을 \mathbf{P}_S 라 하고, IMU(Inertial Measurement Unit)를 통해 추정된 로봇 기체 프레임 B의 수평면 위상 변환 행렬을 \mathbf{T}_S^B 라고 정의할 때, 수평이 보정된 3차원 클라우드 행렬 \mathbf{P}_{aligned} 는 다음과 같이 산출된다.

\mathbf{P}_{aligned} = \mathbf{T}_S^B \cdot \mathbf{P}_S

위 변환을 통해 포인트들의 Z 축 값은 로봇의 틸팅 각도가 아닌 완전한 지면 추정축(Gravity-aligned Z-axis) 방향으로 기하학적 정렬(Alignment)이 완료된다. 이후 동적으로 보정된 이 \mathbf{P}_{aligned} 좌표계 위에서 정적 파라미터인 z_{min} 과 z_{max} 필터를 투과시킴으로써, 기체의 피치/롤 섭동에 관계없이 항상 지면과 완전 평행하게 설계된 가상의 수평 절단면(Horizontal Slice Window)을 취득할 수 있다.

4. 실내 협소 공간에서의 IMU-LiDAR 연동 제어 안정성

pointcloud_to_laserscan 노드 이전 또는 외부 프로세스(대표적으로 tf2 패키지)를 경유하여 확립된 동적 보정 슬라이스 체계는, 자율 드론이 좁은 터널, 기둥이 빽빽한 실내 창고, 복잡한 파이프 랙(Pipe Rack) 구조물 사이를 관통할 때 시스템적 강건성(Robustness)을 보장한다.

고주파 대역으로 샘플링되는 IMU 자세 벡터를 3D 고밀도 포인트 클라우드의 프레임마다 정확히 동기화(Time Synchronization)시켜 보상 행렬 \mathbf{T}_S^B 를 추출하는 과정은 시스템의 계산 자원을 추가적으로 소요한다. 그럼에도 불구하고, 3D 가속도계와 자이로스코프의 융합(Sensor Fusion)을 거친 자세 보정 스캔 데이터를 통해 생성된 안정적인 2D 단면 코스트맵(Costmap2D)은 엣지 보드에서의 자율 회피 반응성을 높이며 고속 주행 시 가짜 장애물(False Positive)의 생성을 체계적으로 차단한다.
