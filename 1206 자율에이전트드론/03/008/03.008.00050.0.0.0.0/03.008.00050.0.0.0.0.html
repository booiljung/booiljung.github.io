Chapter 50. pointcloud_to_laserscan 패키지를 활용한 3D 라이다 제어 연산량 하향 변환

1. 개요 및 학술적 목표

포인트 클라우드(Point Cloud) 데이터는 3차원 공간의 기하학적 정보를 고해상도로 표현하는 데 필수적이나, 자율 에이전트 드론의 실시간 제어 루프에 직접 통합될 경우 과도한 연산량(Computational Payload)을 유발할 수 있다. pointcloud_to_laserscan 패키지는 3차원 포인트 클라우드 데이터를 2차원 평면의 레이저 스캔(Laser Scan) 데이터로 투영(Projection)하여 차원을 축소하는 ROS 2 핵심 유틸리티이다. 본 장의 학술적 목표는 3D 포인트 클라우드 데이터 공간을 2D 레이저 스캔 제어 평면으로 투영하는 변환 알고리즘의 최적화 원리를 이해하고, 이를 통해 공간 제약이 있는 엣지 컴퓨팅 환경에서 인지 및 장애물 회피 시스템의 연산 주기를 하향 변환(Down-conversion)하는 기술적 기반을 확립하는 데 있다.

2. 3D 점군 데이터의 투영 원리 및 수학적 모델

pointcloud_to_laserscan 패키지의 핵심 알고리즘은 3차원 유클리드 공간 \mathbb{R}^3에 존재하는 포인트 클라우드 집합 P = \{p_i \vert p_i = (x_i, y_i, z_i), i = 1, \dots, N\}의 특정 고도 구간 내의 객체를 2차원 극좌표계(Polar Coordinate System) \mathbb{R}^2 상의 레이저 스캔 집합 S = \{(r_j, \theta_j)\}로 사상(Mapping)하는 과정이다.

투영 과정은 우선 min_height와 max_height 파라미터로 정의되는 z축 방향의 구간(Clipping Range) 내에 존재하는 유효한 포인트 집합 P_{valid}를 필터링하는 단계로 시작한다.
P_{valid} = \{p_i \in P \vert z_{min} \le z_i \le z_{max}\}

이후, 필터링된 3차원 좌표 (x_i, y_i, z_i)를 x-y 평면으로 정사영(Orthogonal Projection)하여 각도 \theta_i와 거리 r_i를 산출한다.
\theta_i = \arctan2(y_i, x_i)
r_i = \sqrt{x_i^2 + y_i^2}

로봇의 구조적 제약조건을 반영하여 range_min 및 range_max, 그리고 센서의 측정 각도 범위인 angle_min 및 angle_max 구간을 초과하는 데이터는 무효화 처리(NaN 또는 Inf 할당)된다.

3. pointcloud_to_laserscan 노드의 내부 처리 파이프라인

ROS 2 PointCloud2 메시지 프레임워크 기반의 pointcloud_to_laserscan 노드 처리 파이프라인은 다음과 같은 순차적 연산으로 구성된다.

  1. 메시지 수신 및 검증: sensor_msgs/msg/PointCloud2 포맷의 메시지를 수신하고, 메시지 헤더의 타임스탬프와 프레임 ID(Frame ID)의 동기화 상태를 검사한다.
  2. 포인트 순회 및 인덱스 할당: 수신된 포인트 클라우드 배열을 순회하며, 각 포인트의 좌표를 기반으로 결과 레이저 스캔 배열의 해당 각도 해상도 단위(Angular Resolution Bin) 인덱스를 계산한다. 레이저 스캔 버퍼는 사전에 angle_increment에 기반하여 배열로 할당된다.
  3. 최솟값 갱신(Minimum Range Selection): 다수의 3D 포인트가 동일한 2D 스캔 인덱스 위치로 투영될 경우, 즉 동일한 방향성 레이(Ray)에 여러 객체가 투영될 경우, 가장 짧은 거리를 가진 포인트의 r 값을 해당 인덱스의 대표값으로 기록한다. 이는 충돌 회피 알고리즘에서 가장 근접한 장애물을 우선적으로 인지하기 위한 필수 안전 로직이다.
  4. 메시지 발행: 구성이 완료된 1차원 거리 배열 버퍼를 sensor_msgs/msg/LaserScan 포맷으로 패키징하여 지정된 토픽으로 발행(Publish)한다.

4. 차원 축소에 따른 연산량 하향 변환 효과 분석

N개의 3D 포인트를 처리하여 M개의 2D 스캔 데이터를 생성하는 변환 모델에서 출력 해상도 M은 다음과 같이 정의된다.
M = \lfloor \frac{\text{angle\_max} - \text{angle\_min}}{\text{angle\_increment}} \rfloor + 1

원시 3D 포인트 클라우드는 일반적으로 수십만 개(N \sim 10^5)의 요소로 구성되는 반면, 투영된 레이저 스캔 메시지의 스펙트럼 배열 크기 M은 수백에서 수천 단위(M \sim 10^3)로 대폭 감소한다. 이러한 비선형 차원 축소는 하위 단계의 로컬 플래너(Local Planner), 척력 기반 가상 포텐셜 필드(Artificial Potential Field), 또는 2D 격자 맵(Occupancy Grid Map) 갱신 알고리즘의 시간 복잡도(Time Complexity)를 O(N) 단위에서 O(M) 단위로 낮추는 결과적 연산량 하향 변환을 가져온다.

5. 자율 드론 제어 시스템 응용 및 한계점

자율 에이전트 드론 내비게이션 아키텍처 내에서 본 패키지는 고비용의 3D 공간 연산을 우회하는 지름길(Shortcut)로 활용된다. 드론의 순항 고도가 비교적 일정하고 환경 구조물이 수직적 일관성을 가질 때, 제한된 min_height와 max_height 대역에서의 추출은 2D 평면 공간의 LiDAR 역할을 완전하게 대체할 수 있다.

그러나 3차원 투영 방식의 근본적 한계점으로 입체적 지형(돌출된 나뭇가지, 공중에 떠 있는 와이어 등)이 투영 평면에 반영될 경우, 객체의 3차원적 형상과 체적 공간 정보가 완전히 소실된다. 그러므로 Pointcloud_to_laserscan 기반의 장애물 회피 제어는 주로 단일 레이어의 국지적 공간 충돌 방지(Local Collision Avoidance) 체계에 국한되어야 하며, 드론의 고도 변경 및 3D 동적 회피 기동을 위해서는 복합적인 3D 공간 추정 시스템의 보조적 활용이 수반되어야 한다.
