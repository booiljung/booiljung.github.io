50.2 범용 다채널 LiDAR 원시 포인트 클라우드(sensor_msgs/PointCloud2) 구조와 2D 레이저 스캔(sensor_msgs/LaserScan) 데이터 변환 정합성

1. 다채널 LiDAR 포인트 클라우드(sensor_msgs/PointCloud2)의 메모리 아키텍처

범용 다채널(Multi-Channel) 3D LiDAR 센서(예: Ouster, Velodyne 등)에서 출력되는 원시 데이터는 ROS 2 생태계에서 sensor_msgs/msg/PointCloud2 형태의 바이트 스트림 자료구조로 인스턴스화된다. 이 메시지 포맷은 유연성을 극대화하기 위해 직렬화된(Serialized) 가변 바이너리 데이터(Byte Array) 형태로 메모리에 적재되며, 각 포인트(Point)의 속성을 정의하는 fields 구조 배열을 동반한다.

fields 배열은 개별 포인트가 가지는 X, Y, Z 공간 좌표뿐만 아니라 역반사 강도(Intensity), 링(Ring) 번호, 개별 반사파 타임스탬프 등 추가 메타 데이터를 바이트 오프셋(Byte Offset)과 데이터 타입(Datatype) 단위로 명세한다. 이러한 비정형 다형성(Polymorphism) 구조는 고밀도 공간 정보의 유실 없는 전송을 보장하지만, 제어 알고리즘 루틴 내에서 데이터를 파싱(Parsing)하고 역직렬화(Deserialization)하는 과정에 높은 메모리 대역폭과 디코딩 연산 오버헤드를 발생시킨다.

2. 2D 레이저 스캔(sensor_msgs/LaserScan) 메시지의 구조적 위상

sensor_msgs/msg/LaserScan 인터페이스는 2차원 극좌표 중심의 1차원 정적 벡터(Static Vector) 모델을 차용한다. 이 메시지는 중심 원점(Origin)으로부터 부채꼴 형태로 펼쳐지는 스와스(Swath) 내의 계측 거리를 일정한 각도 분해능(angle_increment)에 따라 1차원 해시 배열 구조인 ranges 버퍼에 연속적으로 정렬한다.

각 배열 인덱스 i는 센서의 전방 기조 축(Forward Axis)을 영점으로 하는 측정 방위각 \theta_i에 대해 \theta_i = \text{angle\_min} + i \times \text{angle\_increment} 의 선형적 상관관계를 형성한다. 이는 데이터를 순회하기 위한 별도의 메모리 역직렬화 메커니즘 없이 물리적 각도 변위를 배열의 인덱스 산술 연산만으로 매핑한다는 점에서 제어 평면에서의 참조 지역성(Locality of Reference)을 극대화한다.

3. 이종 메시지 간의 데이터 변환 정합성(Transformation Consistency)

pointcloud_to_laserscan 시스템에 의한 데이터 전이(Data Transition) 관점에서, 3D 포인트 구조체에서 2D 1차원 배열로의 변환은 비가역적(Irreversible) 차원 축소 연산이므로 물리적 정합성(Physical Consistency)을 보장하는 기준이 필수적이다.

변환 정합성의 첫째 조건은 기하학적 투영 동기화이다. 3D 포인트 클라우드 좌표계의 프레임(Frame ID)과 추출 시점의 타임스탬프는 LaserScan 헤더로 정확히 인계(Inheritance)되어야 한다. 이를 위해 변환 노드는 포인트 클라우드의 TF(Transform Tree) 위상을 검증하고 변환 전후의 기하학적 기준점(Origin Point)이 관성 좌표계 상에서 동일한 원점을 점유하도록 좌표계 변위를 통제한다.

둘째 조건은 거리 대표성(Range Representativeness)의 무결성이다. PointCloud2의 지정된 z축 투영 고도 대역(z \in [h_{min}, h_{max}]) 내에 복수의 반사 지점 P_{k}가 존재하여 이들이 모두 단일 LaserScan 각도 인덱스 i에 중첩 사상될 경우, 그 중 최단 유클리드 거리 반경을 가진 점 P_{min} 의 r 값만을 투영 정합성의 표본으로 채택한다. 이는 장애물 회피 목적성에 기초하여 최근접 객체를 보존하는 최악조건(Worst-Case) 모델링을 준수하는 것이다.

4. 반사 강도(Intensity) 필드의 결속 및 정규화 메커니즘

PointCloud2 내의 다수 포인트가 하나의 2D 인덱스 i 로 병합될 때, 공간 좌표의 정합성뿐만 아니라 센서 반사가인 Intensity 데이터의 정합성 역시 요구된다. pointcloud_to_laserscan 런타임은 r_{min} 이 확정된 해당 반사 지점포인트의 원시 Intensity 속성을 추적하여 LaserScan 의 intensities 1차원 버퍼의 동일 인덱스 i 위치로 병렬 본딩(Binding) 처리한다.

그러나 3D LiDAR 제조사별로 원시 포인트의 Intensity 비트 스케일(8-bit, 16-bit, 또는 Float Float 연속 범위)이 불규칙하므로, ROS 2 데이터 규격 준수를 위해서는 변환 시 fields 내의 Intensity 데이터 유형 명세를 검사하고 이를 표준 단정밀도 부동 소수점(Single-precision Floating-point, float32) 배열로 형 변환(Type Casting)하는 정규화(Normalization) 과정이 필수적으로 수반되어야 한다. 이를 통해 다채널 라이다의 제조사 종속적인 원시 데이터 구조를 하위 제어 어플리케이션이 공통적으로 인지할 수 있는 균일화된 센서 추상화 데이터 정합성에 도달하게 된다.
