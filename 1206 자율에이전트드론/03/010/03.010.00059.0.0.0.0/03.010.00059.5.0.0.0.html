59.5 기구학적 궤적 추종 오차(Tolerance) 파라미터 명세 및 Goal/Path 공차(Tolerance) 위반 예외 처리기

1. 궤적 추종 오차(Tolerance) 감시망의 동역학적 중요성

ROS2 분산 제어 스택에서 구동되는 JointTrajectoryController(JTC)는 자체 생성한 다항식 보간 곡선(Interpolated Reference Curve) 지시값과, 저수준 하드웨어 액추에이터 계층의 엔코더(Encoder) 등으로부터 주기적으로 실시간 피드백(Feedback)되어 올라오는 실제 관절 물리 상태(Actual State) 간의 엄격한 상호 일치성을 보장하기 위해, 추종 오차(Tracking Error Residue)에 대한 다중 공차(Tolerance) 모니터링 방어 체계를 런타임에 상시 가동한다.

자율 에이전트 무인 비행체에 장착된 다축 비전 짐벌(Gimbal)이나 중량물 이송 로봇 암(Manipulator)이 복합적인 물리 환경에서 구동될 때, 로봇 조인트 내부의 기계적 동마찰(Dynamic Friction), 외부 환경의 돌풍(Wind Gust Disturbance), 혹은 액추에이터 모터의 토크 포화 한계(Torque Saturation Limit)로 인해 물리 엔진이 이론적인 수학적 다항식 궤적을 0%의 오차율로 완벽히 추종해 내는 것은 동역학적으로 불가능에 가깝다.

따라서 지능형 제어기는 정상적인 물리적 통신 지연이나 관성에 의한 허용 가능 잔차(Acceptable Residual) 수준과, 시스템 물리적 고장 및 궤도 이탈을 직접적으로 의미하는 치명적 이탈 에러 오차(Critical Deviation Error)를 선형적으로 구분 및 판별해야 할 의무가 있다. 이를 수행하기 위해 JTC 아키텍처는 controller.yaml 파라미터 서버 명세를 통해 두 가지 차원의 제어 계층 공차 파라미터, 즉 Path Tolerance(경로 공차)와 Goal Tolerance(목표점 공차) 제한 블록을 정의하고, 실시간 제어 루프(Control Loop) 내부에서 이 두 공차 영역의 위반 여부를 각각 독립적인 스레드와 조건문으로 병렬 평가하는 실시간 감시 상태 머신(Monitoring State Machine)을 운용한다.

2. Path Tolerance와 Goal Tolerance의 파라미터 명세 및 물리적 임계점

오차 감시 루틴(Error Monitoring Routine) 내에서 매 틱(Tick)마다 수학적으로 평가되는 두 가지 공차 기준 영역은 그 역할과 제약 평가 범위가 완전히 상이하다.

  1. Path Tolerance (경로 주행 공차 허용 한계치):
    모터 조인트가 시작점과 도착 목표 웨이포인트(Waypoint) 사이에 형성된 연속적인 다항식 시간 보간 궤적 구간을 수학적으로 횡단 주행하는 과정 전체 동안 감독하는 파라미터이다. 매 update() 제어 주기마다 JTC 루프는 실시간으로 엔코더가 피드백하는 모터 물리 위치(또는 1차 미분 속도 변수)와 앞서 계산된 이론적 다항식 지령 파라미터 간의 유클리드 잔차 절대값 e(t) = | \theta_{cmd}(t) - \theta_{act}(t) | 가 사전에 임계치로 세팅 허용된 path_tolerance 파라미터 차원 범위를 위험 수준으로 이탈 및 초과했는지 정밀 검사한다. 이 감시망은 동역학계 시스템이 연산 궤적을 잃고 발산(Diverging) 현상에 빠졌거나 외부의 예측 불허 장애물과 충돌하여 물리적으로 멈춰버렸는지(Stall)를 제 즉시 최소 시간 단위로 감지하는 가장 선제적인 런타임 방어선 코어이다. 파라미터 명세 상 해당 기본 제한 수치 단위는 조인트 유형에 따라 라디안(rad) 회전각 혹은 미터(m) 병진 거리 단위로 할당되며, state_publish_rate의 속도 주파수 주기로 즉각 샘플링(Sampling) 감시된다.

  2. Goal Tolerance (최종 목표점 안착 공차 허용 한계치):
    전체 혹은 국지 궤적 수행의 최종 명시 시간인 time_from_start 시점 무렵, 혹은 이동 궤적 내 각 노드 도착 예정 시점에 해당 구동 조인트가 의심 없는 시스템 멈춤 및 목표점 정착 상태(Settled Condition) 영역에 안정적으로 진입 도달했는지를 최종 판정하는 승인 지표이다. 측정된 실제 위치 추적 오차 e(t_f) = | \theta_{target} - \theta_{act}(t_f) | 가 파라미터로 설정된 goal_tolerance 공간 임계 허용값을 물리적으로 만족해야 한다. 이와 동시에 잔존 진동 여부 검사를 위해 목표 속도 잔차 한계 \dot{e}(t_f) \approx 0 (정지에 가까운 상태) 조건을 충족하는 정착 시점이, 허용된 시간 오차 대역 파라미터인 goal_time_tolerance 제한 기한 이내에 모터에서 성립 완료되어야 비로소 상위 모션 제어기로 성공적인 궤적 도달 판정(Success Declare) 인터럽트를 발생시키게 된다. 이는 폐루프 서보 제어 시스템(Servo System) 고유의 정상 상태 오차(Steady-State Error) 소멸도를 최종 평가하는 합격 척도 기준이 된다.

3. 공차 위반(Tolerance Violation) 이벤트에 따른 시스템 예외 처리기(Exception Handler)

실시간 피드백 모니터링 방어 체계가 런타임 진행 스텝에서 상기 명시된 공차 임계값의 명백한 위반 사항을 확정 판독하게 되면, JTC 스케줄러는 즉각적으로 방어적 예외 처리기(Defensive Exception Handler) 루틴을 제 1순위로 강제 가동시켜 시스템 전반의 추가적인 물리적 연쇄 손상을 차단하는 조치에 돌입한다.

만약 감시 루프에서 Path Tolerance 이탈 위반 이벤트가 솟아오르면, 제어기 내부 상태 머신은 즉각적으로 해당 진행 구간의 궤적 추종 시도가 완전히 물리적으로 실패(Tracking Failed)했음을 선언하고, 진행 연산 중이던 모든 보간 버퍼 처리 연산을 그 즉시 강제로 낙선 및 파기(Abort) 처리한다. 곧이어 해당 임무를 지시했던 상위 Action 서버(FollowJointTrajectory Action)의 외부 클라이언트 측 노드에 묵시적인 강제 실패 인터럽트인 PATH_TOLERANCE_VIOLATED 에러 코드를 포함한 실패 결과 스코어(Abort Result)를 통지한다. 이와 동시에 하위 서보 제어 루프에는 어긋난 궤적 유지를 위해 높은 토크를 쏟아붓는 추가적인 무리한 제어 신호 송출을 원천 중단시키고, 그 위반 시점의 현재 조인트 상태 각도를 고정 유지(Hold Present Position)하라는 안전 임시 명령어 영(0) 속도 패킷을 즉각 인가함으로써 하드웨어 간섭 물리적 충돌에 의한 백드롭 액추에이터의 전류 소손 발화를 미연에 집중 소거시킨다.

반대로 Goal Tolerance 획득 실패 위반 이벤트의 발생은, 모터 액추에이터가 사전 설정된 시간 유예 파라미터인 goal_time_tolerance 데드라인 제한 시간이 만료되었음에도 불구하고 아직 구조적 마찰이나 모델링 게인 오차 문제로 인해 목적지의 정상 상태 오차(Steady-state Error) 경계 범위 내로 완전히 정착 수렴하지 못했거나 해당 지점을 기점으로 헌팅 진동(Hunting Oscillation) 현상을 일으키고 있는 미해결 역학 상태를 상징한다. 이 경우 JTC 예외 처리기는 GOAL_TOLERANCE_VIOLATED 실패 규격 코드를 상방향으로 발송함과 동시에, 조인트의 부족 감쇠(Underdamped) 역학 상태나 외란 억제 실패를 교정하기 위해 제어기 PID 튜닝 파라미터를 점검하라는 치명적 시스템 경고(Warning Trigger)를 진단 파이프라인에 발생시킨다. 이러한 일련의 단계적 공차 위반 방어 메커니즘은 제어 불능 발산 상태에 빠질 위기에 처한 로봇 에이전트를 물리 엔진 단에서 안전하게 조기 정지 및 무력화시키기 위한 무인 이동체 고신뢰 안전 공학(Failsafe Engineering)의 필수 핵심 가드레일(Guardrail) 로직 아키텍처로 작용한다.
