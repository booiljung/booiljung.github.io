Chapter 59. ros2_controllers 내 JointTrajectoryController 위치/속도/노력(Effort) 보간 로직

로봇 공학과 무인 비행체(Unmanned Aerial Vehicle) 시스템에서 다관절 매니퓰레이터(Manipulator) 혹은 복합 기구학 체인(Kinematic Chain)을 제어할 때, 목표 웨이포인트(Waypoint) 간의 매끄러운 궤적 생성을 담당하는 컴포넌트는 필수적인 역학 구동 요건을 이룬다. ROS2 환경의 분산 제어 시스템 표준을 확립하는 ros2_control 아키텍처는 이를 위해 ros2_controllers 메타 패키지 내부에 JointTrajectoryController(이하 JTC) 모듈을 기본 탑재하고 있다. JTC는 상위 모션 플래닝 라이브러리(예: MoveIt 2)로부터 전달받은 trajectory_msgs/msg/JointTrajectory 양식의 희소 웨이포인트(Sparse Waypoints) 데이터를 단위 스텝의 실시간 하드웨어 제어 주기에 맞추어 고밀도로 보간(Interpolation)하는 중추적인 역할을 수행한다.

본 장에서는 JTC가 채택하고 있는 다항식 기반의 궤적 보간(Trajectory Interpolation) 이론적 배경을 탐구한다. 모션 프로파일(Motion Profile)의 연속성을 보장하기 위한 스플라인(Spline) 수학 모델의 원리부터 시작하여 위치(Position), 속도(Velocity), 그리고 역학적 토크 출력을 나타내는 노력(Effort)과 연계된 다중 제어 계층 로직까지 상세히 규명한다.

1. 조인트 궤적 보간의 기구학적 요구사항

자율 에이전트 드론이 공중에서 다관절 매니퓰레이터를 정밀하게 전개하거나 복합적인 비전 센서 탑재 짐벌(Gimbal)을 고속으로 회전시킬 때, 궤적을 단순히 시작점(\theta_{start})과 종료점(\theta_{end}) 사이의 1차 함수로 연결하는 선형적 이동 모델은 제어 동역학 측면에서 채택될 수 없는 취약한 접근 방식이다. 수학적 선형 보간(Linear Interpolation)은 각 제어 구간의 경계 지점에서 속도(Velocity)의 불연속성을 유발하며, 이는 필연적으로 무한대에 수렴하는 가속도(Acceleration) 임펄스(Impulse)를 미분 연산 상에서 발생시켜 액추에이터 기어박스에 파괴적인 기계적 충격량이나 전류 피크(Current Peak)에 의한 권선 소손 오차를 야기한다.

따라서 궤적 생성 알고리즘은 최소한 기구학적 1차 도함수인 속도와 2차 도함수인 가속도의 프로파일 분포가 경과 시간에 대하여 부드럽고 연속적인 제어 곡선(Smooth Curve)을 형성해야 한다는 역학 설계의 학술적 요건을 엄격히 만족시켜야 한다. JTC 아키텍처는 이를 확보하기 위하여 로봇 조인트의 목표 상태 제약 조건(다변수 위치, 속도 한계치 등)이 얼마나 구체적으로 할당되었느냐에 따라 동적으로 최적화된 다항식 스플라인(Spline) 보간 루틴을 자동 선택하도록 프로그래밍되어 있다.

2. JointTrajectoryController의 다항식 보간(Polynomial Interpolation) 메커니즘

상위 제어 노드에서 발행되는 trajectory_msgs/msg/JointTrajectoryPoint 메시지는 특정 절대 경과 시간(t_i)에 수렴해야 할 조인트의 기준 상태점(Reference Point) 구조체를 배열 형태로 통지한다. JTC 내부의 보간 엔진(Interpolation Engine)은 인접한 두 웨이포인트 시간 범위(t_{i-1}, t_i) 내에서 통신 지연 혹은 샘플링 시간 분해능 변수 t 에 대응하는 목표 제어량을 실시간 도출하기 위해 유한 차수 다항식을 수립한다.

  1. 선형 보간 (Linear Interpolation):
    궤적 메시지상에 모터 위치(Position) 정보 좌표만 명시되어 있고 속도, 가속도의 미분 제약 조건 데이터가 누락되어 전달된 예외적 상황일 경우, 가장 기초적인 1차 방정식 기반 선형 보간을 폴백(Fallback) 수단으로 단행한다. 이는 하드웨어 충격 감쇄(Shock Absorption) 측면에서 지양되어야 하는 한계 기법이나, 시스템 중단을 방어하기 위한 최소한의 데이터 연결성(Connectivity)을 제공하는 원시적 목적으로 잔존한다.
  2. 3차 다항식 스플라인 (Cubic Spline Interpolation):
    웨이포인트 메시지 구조체에 시작 및 도달 위치와 더불어 목표 속도(Velocity) 제약 조건 지표까지 명시되었을 경우 주로 적용되는 방식이다. 두 시점 구간 내의 매개변수 곡선인 위치 궤적 함수 \theta(t) 를 3차 지수 다항식(a_0 + a_1 t + a_2 t^2 + a_3 t^3)의 형태로 수립하고, 경계 구간 양 끝단의 위치 및 속도 수식을 연립 방정식으로 구성하여 4개의 스칼라 계수를 역산출해 낸다. 3차 스플라인을 적용할 경우 전체 시간 축에 대한 위치와 속도의 궤적 곡선 연속성(C1 Continuous)은 기술적으로 보증할 수 있다. 그러나 가속도 도함수 프로파일이 각 웨이포인트의 경계 병합 지점에서 계단 함수(Step Function) 형태로 불연속적으로 전환될 제약이 수반되어, 결과적으로 유한하지만 변동 폭이 단기적으로 급격한 가속도 제어 충격(Jerk)을 드론 기체 프레임에 전가할 여지가 남는다.
  3. 5차 다항식 스플라인 (Quintic Spline Interpolation):
    궤적 제어 메시지의 상태 파라미터 구조 상에 위치, 국소 속도, 그리고 순간 가속도(Acceleration) 제약 변수가 모두 누락 구간 없이 완벽히 정의되었을 경우 JTC 보간 시스템이 최우선적인 적합성으로 판단하여 수행하는 최고 수준 알고리즘 규격이다. 구간별 위치 제어 궤적의 모델링을 5차 다항식 방정식으로 상향 대치하여, 총 6차원에 달하는 경계 제약 조건(시작점과 종료점 각각에 대한 위치, 속도, 가속도 수치)을 오차 0 수준으로 일치시키는 스플라인(Spline)을 구성하게 된다. 5차 다항식 보간법이 성공적으로 궤적 생성기에 통과될 경우, 해당 조인트 기구학 도함수는 가속도 단계(2차 미분)까지 수학적으로 철저한 상시 곡선 연속성(C2 Continuous) 등급을 확보하므로 모터 토크 변동의 척도인 저크(Jerk, 가속도의 시간에 대한 도함수) 충격량을 가장 이상적이고 신뢰성 있게 감쇄시키는 최고 품질의 부드러운 유체 역학적 궤적(Fluidic Kinematic Trajectory)을 엔드 이펙터 체인 끝단에서 획득할 수 있다.

3. 제어 출력과 Effort(노력) 매핑 계층

JointTrajectoryController 컴포넌트 스택은 기하학 전 영역에서 도출된 시간 영역 궤적 스플라인을 보간 처리하는 단일 수학 연산 절차를 넘어, 연결된 hardware_interface(하드웨어 로우 레벨 인터페이스) 단의 수용 가능 명령어 물리 유형(Command Interface Type Specification) 규칙에 부합하도록 방대한 연산의 궤적점 제어값 배열(\theta_{ref}(t), \dot{\theta}_{ref}(t))을 실제 서보 드라이버가 인식 가능한 액추에이터 인가 제어 데이터 단일 패킷으로 상호 환산시키는 폐루프(Closed Servo Loop) 관리 프로세스를 필수적으로 탑재하여 연동한다.

만약 ROS2 노드 최하위를 관장하는 하드웨어 통신 드라이버가 엔코더를 기반으로 하는 위치 지령 인터페이스(position)나 내부 RPM 정속을 요하는 속도 인터페이스(velocity) 명령 타입 식별 체계를 자체 제어기 내부 MCU 로직에 의해 물리적으로 직접 수용한다면, JTC는 역학 보간 연산으로 얻어진 결과값을 일체의 변형 없이 직렬 패스스루(Passthrough) 방식으로 전달한다.

그러나 직구동 다이렉트 드라이브 통제 환경과 같이 하드웨어 모터 제어기가 위치 제어기 루프를 내장하고 있지 않아 직류 도선의 인가 전류 전압치나 샤프트 측 토크 단위, 혹은 역학적 힘과 비례하는 노력 지시 펄스 폭(effort) 정보 패킷만을 배타적 입력으로 수용하는 종속적 구조일 경우에는 상황이 상이하다. 이러한 조건에서는 보간된 궤적 벡터들이 JTC 소프트웨어 내부에 컴포넌트 속성 파라미터로 할당되어 탑재된 독립적인 이산 위치/속도 PID (Proportional-Integral-Derivative) 피드백 제어기를 차례로 거치게 된다. 조인트의 목표(Target) 위치 파라미터 및 속도 파라미터와, 모터의 절대 엔코더 및 속도 센서 계통에서 주기적으로 피드백(Feedback)되어 역류하는 현재 관측 하드웨어 상태(State_Interfaces 참조 배열) 간에 발생하는 동역학적 수학적 잔차(시스템 추적 누적 오차, System Tracking Error Matrix) 변수를 연속 연산 회로에서 처리한다. 최종적으로 이 잔차 연산 결과물에 대한 PID 제어 출력의 조합을 전송 패킷에 포함시켜 시리얼 토크 지령(Effort Command) 명령 형태로 변환 반환하는 폐쇄형 자기 교정 루프 제어(Closed-loop Self-Correction Control) 아키텍처 방식을 강건하게 유지한다.
