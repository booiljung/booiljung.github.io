58.7 제어 관리자(Controller Manager) 제어 주기(Update Rate) 동기화 결함 시 발생하는 타이밍 지터(Jitter) 오차 분석

1. 개요 및 타이밍 지터(Timing Jitter)의 이론적 정의

이산 시간(Discrete-Time) 제어 시스템에서 제어기의 성능은 샘플링 주기(Sampling Period)의 일정성에 크게 의존한다. ROS2 ros2_control 아키텍처 내에서 제어 관리자(Controller Manager)는 설정된 제어 주기(Update Rate)에 따라 주기적으로 제어 루프를 스케줄링하여 센서 데이터를 읽고(Read), 제어기를 갱신(Update)하며, 계산된 명령을 액추에이터에 인가(Write)한다. 이때 각 제어 루프의 실제 실행 간격이 목표 주기에서 벗어나는 시간적 변동성을 타이밍 지터(Timing Jitter)라고 정의한다.

타이밍 지터는 주로 운영체제의 스레드 스케줄링 지연, 하드웨어 통신 버스의 병목 현상, 제어 알고리즘의 비결정적 연산 시간 등에 의해 발생한다. 제어 주기가 수학적으로 완전히 엄밀하게 동기화되지 못할 경우, 제어 시스템의 위상 여유(Phase Margin)가 감소하고 고주파 대역에서 비선형 오차가 누적되는 역학적 결함이 나타난다.

2. 제어 주기 동기화 결함의 발생 원인

ROS2 시스템에서 제어 관리자의 주기 동기화가 실패하는 주요 원인은 다음과 같다.

첫째, 비실시간 커널의 스케줄링 간섭이다. 표준 리눅스(Linux) 환경에서는 프로세스 선점(Preemption) 과정에서 커널이 다른 고우선순위 인터럽트를 처리하기 위해 제어 타이머를 지연시킬 수 있다.
둘째, 이기종 디바이스 간의 클록 드리프트(Clock Drift)이다. 센서에서 데이터를 발생시키는 하드웨어 클록과 제어 관리자가 스레드를 구동하는 CPU의 시스템 클록 간에 동기화 오차가 누적되면, 데이터를 읽어 들이는 주기에 위상 천이(Phase Shift)가 발생한다.
셋째, 메모리 페이지 폴트(Page Fault)로 인한 지연이다. 제어 스레드가 동적 메모리 할당을 수행하거나 스왑 영역에 접근하게 되면 일시적인 멈춤 현상이 나타나 극심한 스파이크 형태의 지터를 유발한다.

3. 지터 오차의 수학적 역학 변이

주기 T_0로 구동되도록 설계된 선형 시불변(Linear Time-Invariant) 이산 시스템의 상태 방정식은 다음과 같이 정의된다.

x[k+1] = A_d x[k] + B_d u[k]

지터가 발생할 경우, 실제 k번째 샘플링 주기는 가변 주기인 T_k = T_0 + \delta_k가 된다. 여기서 \delta_k는 지터 변위 벡터이다. 가변 주기에 의해 이산화된 시스템 행렬 A_d(T_k)와 연속 행렬 계수 e^{A T_k} 간의 테일러 급수 전개를 적용하면 식은 다음과 같이 확장된다.

A_d(T_k) = e^{A(T_0 + \delta_k)} \approx e^{A T_0}(I + A \delta_k)

결과적으로 지터 항 A \delta_k가 상태 전이 행렬에 결합되어 고유값(Eigenvalue) 위치를 단위원(Unit Circle)의 외곽으로 이동시킬 수 있는 시변 비선형 교란(Perturbation)으로 작용한다. 이러한 역학적 교란은 비례 적분 미분 제어 알고리즘의 미분 항에서 노이즈 증폭을 극대화시키며, 적분 항에서는 오차의 누적 시점을 왜곡하여 최종적으로 한계 사이클(Limit Cycle) 혹은 발산을 초래한다.

4. 지터 오차 측정 및 통계적 평가 기법

제어 관리자의 지터 오차를 정량화하기 위해서는 제어 루프 내에서 고해상도 타임스탬프(High-Resolution Timestamp)를 이용하여 이전 주기 기록 시점과 현재 갱신 시점 간의 차이 \Delta t_k를 추출해야 한다. 추출된 데이터 군집을 기반으로 통계 모델을 도출한다.

  • 통계적 대푯값 정량화: 평균 \mu = E[\Delta t_k]와 목표 주기 T_0 간의 차이로 모델링된다.
  • 확률 밀도 함수(Probability Density Function): 지터 분포의 분산 \sigma^2을 확인한다. 정규 분포를 벗어나는 긴 꼬리(Long-tail) 형태는 제어 시스템의 결정성(Determinism) 붕괴를 의미한다.
  • 전력 스펙트럼 밀도(Power Spectral Density): \delta_k 데이터 시계열의 주파수 응답을 분석하여 특정 통신 주기 혹은 펌웨어 인터럽트에 의한 공진 주파수 영향 파악을 수행한다.

5. 지터 확산 방지를 위한 프레임워크 튜닝 정책

ros2_control 프레임워크는 지터 발생 억제를 위해 실시간 운영체제(RTOS) 기준을 만족하는 스레드 스케줄링 튜닝을 요구한다. PREEMPT_RT 패치가 적용된 커널 구동 환경에서, 제어 관리자 스레드의 정책은 완전 선점형으로 설계되어야 하며 SCHED_FIFO와 같이 높은 실시간 우선순위 구성을 명시적으로 할당받아야 한다.

또한 제어 시스템 프로세스의 생명주기(Lifecycle) 설정 시, 메모리 공간 확보를 위한 mlockall 시스템 호출 플래그를 실행하여 프로세스의 영역이 디스크 스왑 구조로 진입하는 것을 커널 메모리 할당 계층에서 강제 차단하는 조치가 필수적이다. 하드웨어 인터페이스(Hardware Interface)의 함수 규격 내부에서는 또 다른 시스템 콜이나 비동기적인 동적 메모리 모델이 배제되어 구동 환경 내의 주기 변동 폭을 제한된 하한계 아래로 격리시킨다.
