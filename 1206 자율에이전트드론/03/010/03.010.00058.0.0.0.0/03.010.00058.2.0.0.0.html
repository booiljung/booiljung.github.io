58.2 System, Sensor, Actuator 이기종 하드웨어 컴포넌트 통합 및 관리형 노드 생명주기(Lifecycle) 상태 속성

ros2_control 프레임워크는 드론 및 로보틱스 플랫폼 내에 존재하는 다양한 전기·기계적 장치들을 논리적으로 추상화하기 위해 세 가지 명시적인 하드웨어 컴포넌트 클래스를 제공한다(사용 버전: ROS 2 Humble/Iron 기준). 이기종(Heterogeneous) 하드웨어 생태계를 동기화된 일원적 제어망 내에 통합하고, 이들의 구동 이력을 관리형 노드(Managed Node)의 유한 상태 기계(Finite State Machine)로 제어하는 것은 분산 시스템의 강건성을 유지하기 위한 필수적 아키텍처이다.

1. 이기종 하드웨어 컴포넌트: System, Sensor, Actuator

ros2_control 아키텍처에서 하드웨어 인터페이스는 기능과 복잡도에 따라 System, Sensor, Actuator의 세 가지 주요 컴포넌트 타입으로 분류된다.

  1. System 컴포넌트 (hardware_interface::SystemInterface):
    다수의 관절(Joint)과 센서를 포함하는 복합 하드웨어 조립체를 추상화한다. 예를 들어 다관절 매니퓰레이터나 드론의 쿼드 로터 베이스 통신을 전담하는 메인 MCU 보드와의 단일 직렬 통신 버스는 System으로 통합된다. 읽기(Read)와 쓰기(Write) 동작이 모두 가능하며, 복합적인 상호 의존성을 갖는 컴포넌트를 단일 플러그인에서 처리하기에 용이하다.
  2. Sensor 컴포넌트 (hardware_interface::SensorInterface):
    주변 환경이나 로봇 자체의 물리적 상태를 관측하여 값으로 반환하지만, 물리적 외력이나 변위를 발생시키지 않는 수동적 독립 컴포넌트이다. IMU(관성 측정 장치), 카메라 트리거 데이터, 혹은 1D 라이다 등의 인터페이스에 활용된다. 이 컴포넌트는 제어 루프 내에서 읽기(Read) 전용으로 설계되어 쓰기 인터페이스를 노출하지 않는다.
  3. Actuator 컴포넌트 (hardware_interface::ActuatorInterface):
    기본적으로 1자유도(1 DOF) 기반의 단일 구동 모터를 추상화한다. 개별 스텝 모터 또는 서보 앰프를 통해 제어되는 독립된 축 하나를 구동할 때 사용되며, 읽기 및 쓰기 접근이 모두 허용된다.

이러한 세 종류의 컴포넌트 플러그인은 구동 루프(controller_manager) 내의 리소스 관리자(Resource Manager)에 의해 단일 메모리 공간으로 매핑되어 단일화된 통합 제어 프로세스를 구축한다.

2. 관리형 노드 생명주기(Lifecycle) 상태 속성 및 제어 역학

ros2_control에 등록되는 각 하드웨어 컴포넌트와 제어기(Controller)는 ROS 2 생명주기(Lifecycle) 노드의 규격을 엄격히 따른다. 컴포넌트의 무질서한 하드웨어 접근을 막고 예측 가능한 응답을 도출하기 위해 상태 전이(State Transition) 메커니즘이 강제된다. 주요 주기 상태 및 전이 함수는 다음과 같다.

  1. Unconfigured (미구성 상태):
    시스템 런치의 초기 단계로 리소스 관리자가 플러그인을 동적으로 로드한 직후의 상태이다. 어떠한 하드웨어 통신 채널도 개방되지 않으며, 제어 명령은 철저히 무시된다.
  2. Configuring \rightarrow Inactive (구성 중 전이 및 비활성 상태 전환):
    on_configure() 콜백이 호출되면, URDF 파라미터를 파싱하고 필수 메모리(상태 및 지령 버퍼)를 정적으로 사전 할당(Pre-allocation)한다. 이 과정은 실시간(Real-Time) 제어 루프 밖에서 수행되며, 직렬 포트나 소켓 캐시 핸들을 획득하여 Inactive 상태로 진입한다. Inactive 상태에서는 상태 값을 주기적으로 읽을(read()) 수는 있으나 제어값 인가(write())는 차단된다.
  3. Activating \rightarrow Active (활성화 중 전이 및 활성 상태 전환):
    하위 하드웨어가 올바르게 응답하면 on_activate() 콜백을 거쳐 최종적으로 Active 상태에 진입한다. Active 상태는 실제 제어 명령을 하드웨어 모터에 승인하는 운영 단계이며, 컨트롤러 매니저의 write() 함수가 액추에이터 및 시스템 컴포넌트로 지령 값을 플러싱(Flushing)한다. 이 단계에서 모터 앰프의 활성화 핀(Enable Pin) 전기 신호가 켜지게 된다.
  4. Deactivating / CleaningUp (비활성화 및 초기화 복귀):
    안전 정지 상황이나 프로세스 소멸 시 on_deactivate()가 발생하여 모터를 제동하고 안전하게 Inactive로 복귀시킨다. on_cleanup() 호줄 시 할당된 메모리와 통신 핸들을 해제하여 다시 Unconfigured로 되돌린다.

이기종 로보틱스 컴포넌트는 이러한 Lifecycle 상태 검증망을 내부적으로 거침으로써, 런타임 단계에서 발생하는 통신 장애 시의 치명적인 물리적 오작동 파급력을 하드웨어 추상 계층에서 자체적으로 방어하는 성질을 띠게 된다.
