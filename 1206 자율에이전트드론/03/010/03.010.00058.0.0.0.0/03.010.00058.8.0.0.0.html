58.8 Joint State Broadcaster 체인을 이용한 서보 모터(Servo Motor) 엔코더 센서 피드백 상태 전파(Propagation) 원리

서보 모터(Servo Motor)를 비롯한 동력학 구동기는 로봇 시스템의 움직임을 제어하는 핵심 요소이며, 이러한 구동기의 현재 물리적 상태를 정확히 파악하는 것은 폐루프 제어(Closed-loop Control) 및 자율 주행의 필수 전제 조건이다. ros2_control 프레임워크는 하드웨어 추상화 계층(Hardware Abstraction Layer, HAL)을 통해 이기종 하드웨어의 상태 정보를 일관된 인터페이스로 수집한다. 수집된 엔코더(Encoder) 센서의 피드백 데이터는 Joint State Broadcaster를 거쳐 ROS 2 토픽 생태계 전반으로 전파(Propagation)된다. 본 절에서는 Joint State Broadcaster의 내부 메커니즘과 서보 모터 엔코더 센서 값의 전파 원리를 학술적으로 규명한다.

1. Joint State Broadcaster의 아키텍처 상 역할

Joint State Broadcaster는 ros2_controllers 패키지 그룹에 포함된 특수한 형태의 컨트롤러이다. 일반적인 컨트롤러가 하드웨어에 명령(Command)을 내리는 반면, Joint State Broadcaster는 오직 하드웨어 인터페이스(Hardware Interface)로부터 상태(State) 데이터를 읽어 ROS 2 통신망으로 방송(Broadcasting)하는 단방향 데이터 흐름을 담당한다.

이 컨트롤러는 서보 모터의 엔코더가 측정하는 위치(Position), 속도(Velocity), 노력(Effort, 토크 또는 전류) 값을 주기적으로 읽어 sensor_msgs/msg/JointState 메시지 타입으로 변환한다. 변환된 메시지는 /joint_states 토픽으로 발행(Publish)되며, 시스템 내의 다른 노드들이 해당 하드웨어의 최신 관절 상태를 실시간으로 구독(Subscribe)할 수 있도록 한다.

2. 엔코더 센서 데이터의 하드웨어 추상화 계층 통과

서보 모터에 부착된 엔코더는 관절의 회전 각도나 병진 거리를 디지털 신호로 변환하여 제어기에 전달한다. ros2_control의 시스템 인터페이스(System Interface) 계층은 read() 메서드를 호출하여 주기적으로 서보 모터의 엔코더 레지스터 값을 폴링(Polling)하거나 인터럽트 수신을 통해 획득한다.

획득된 원시 데이터(Raw Data)는 하드웨어 인터페이스 플러그인 내에서 SI 단위계(위치의 경우 라디안(Radian) 또는 미터(Meter), 속도의 경우 라디안/초(Rad/s) 또는 미터/초(m/s))로 스케일링(Scaling)된다. 스케일링된 값은 URDF(Unified Robot Description Format)에 정의된 관절(Joint) 이름과 매핑되어 제어 관리자(Controller Manager)가 관리하는 리소스 관리자의 상태 인터페이스 풀(State Interface Pool)에 등록된다. Joint State Broadcaster는 초기화 과정에서 이 상태 인터페이스들에 대한 참조(Reference)를 획득하여 실시간 메모리 접근 권한을 확보한다.

3. 실시간 제어 루프 내에서의 비동기 퍼블리싱 메커니즘

ros2_control 프레임워크는 엄격한 실시간(Real-Time) 제약 조건을 요구한다. 제어 주기를 담당하는 메인 스레드는 데드라인 미스(Deadline Miss)를 방지하기 위해 메모리 할당(Allocation)이나 시스템 콜(System Call), 그리고 스레드 블로킹(Blocking)을 발생시키는 작업을 수행해서는 안 된다.

ROS 2의 기본 퍼블리셔(Publisher)는 네트워크 I/O 스케줄링 및 큐 관리에 의해 블로킹이 발생할 가능성이 존재한다. 이를 방지하기 위해 Joint State Broadcaster는 realtime_tools::RealtimePublisher 템플릿 클래스를 사용한다. 이 클래스는 락 프리가 보장되는 링 버퍼(Lock-free Ring Buffer)를 내부적으로 유지한다.

실시간 메인 스레드는 update() 주기마다 상태 인터페이스의 값을 링 버퍼에 비동기적으로 기록(Write)하며, 별도로 분리된 비실시간 스레드가 버퍼에서 메시지를 꺼내어 ROS 2 미들웨어(DDS) 계층으로 전달한다. 이러한 스레드 분리 설계는 통신 지연(Latency)이나 네트워크 병목이 실시간 제어 커널의 연산 주기를 방해하지 않도록 원천적으로 격리한다.

4. 로봇 상태 퍼블리셔(Robot State Publisher)와의 운동학적 체인 연계

Joint State Broadcaster가 발행하는 /joint_states 메시지 자체는 각 관절의 독립적인 스칼라 값만을 포함하며, 로봇 전체의 공간적 자세(Pose) 정보를 직접적으로 나타내지는 않는다. 이를 전역적인 기하학적 정보로 변환하기 위해 로봇 상태 퍼블리셔(Robot State Publisher) 노드가 사용된다.

Robot State Publisher는 부팅 시 로봇의 URDF 모델을 로드하여 기구학적 트리(Kinematic Tree)를 구성한다. 이후 /joint_states 토픽을 지속적으로 구독하면서, 새로운 관절 상태 데이터가 수신될 때마다 순운동학(Forward Kinematics) 연산을 수행한다. 연산 결과는 로봇의 베이스 링크(Base Link)를 기준으로 각 링크(Link)의 위치와 방향을 계산한 3차원 변환 행렬(Transformation Matrix)의 집합으로 도출된다.

계산된 변환 행렬은 tf2_ros::TransformBroadcaster를 통해 TF 트리(TF Tree)에 게시(Publish)된다. 이 과정을 마침으로써 1차원적인 서보 모터 엔코더 값은 로봇의 전역 좌표계 상의 공간적 기하 정보로 확장(Propagation)되며, 내비게이션(Nav2), 모션 플래닝(MoveIt 2) 등의 상위 수준 자율 알고리즘 모듈들이 로봇의 현재 상태를 기반으로 경로를 연산할 수 있는 기초 데이터를 얻게 된다.

5. 분산 시스템에서의 상태 동기화 및 타임스탬프 무결성

다중 프로세스 혹은 분산 호스트 환경에서 엔코더 피드백 상태 전파의 핵심은 시공간적 동기화(Spatiotemporal Synchronization)이다. Joint State Broadcaster가 생성하는 메시지의 헤더(std_msgs/msg/Header)에는 센서 데이터를 획득한 시점의 타임스탬프(Timestamp)가 정밀하게 기록되어야 한다.

제어 관리자의 update() 루프가 시작되는 시점에 획득된 단일 시간(Time Source)이 전체 상태 인터페이스 샘플링 시점으로 일괄 적용되어야 한다. 네트워크 전송 과정에서 발생하는 지터(Jitter)로 인해 메시지 도착 시간이 변동되더라도, ROS 2의 TF 시스템과 상위 알고리즘은 메시지 헤더에 기록된 원시 타임스탬프를 기준으로 데이터를 정렬하고 보간(Interpolation)한다. 따라서 상태 전파 체인을 구성할 때는 하드웨어 클럭(Hardware Clock)과 ROS 2 시스템 타임(System Time) 간의 오차를 최소화하고 균일한 샘플링 레이트(Sampling Rate)를 유지하는 것이 중요하다. 오차가 누적될 경우, 제어기와 모션 플래너 간의 위상 차이(Phase Lag)가 발생하여 제어 불안정성 또는 위치 인식의 발산 현상을 초래할 수 있다.
