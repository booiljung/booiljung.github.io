68.2 폴리곤 카운트(Polygon Count) 데시메이션(Decimation) 기반 메시 토폴로지(Topology) 최적화 및 렌더링 부하(Overhead) 경감 기법

자율 에이전트 드론을 위한 고정밀 CAD(Computer-Aided Design) 원본 모델은 제조를 목적으로 수백만 개 이상의 폴리곤(Polygon)으로 구성되는 경우가 빈번하다. 이러한 “무거운(Heavy)” 기하학 데이터를 시뮬레이션 환경에 원형 그대로 이식할 경우, GPU 렌더링 파이프라인과 물리 엔진의 충돌 연산(Collision Checking) 과정에서 심각한 병목 현상(Bottleneck)이 유발된다. 본 절에서는 시각적 형태를 최대한 보존하면서도 폴리곤 카운트를 드라마틱하게 축소시키는 데시메이션(Decimation) 알고리즘과, 이를 통한 시뮬레이션 시스템의 연산 부하(Overhead) 경감 기법을 다룬다.

1. 데시메이션(Decimation) 알고리즘의 위상학적 원리

메시 데시메이션은 주어진 3D 폴리곤 메시의 정점(Vertex)과 면(Face)의 수를 능동적으로 줄이는 위상 및 기하학적 단순화 공정이다. 이 과정의 핵심 목표는 정점을 무작위로 삭제하는 것이 아니라, 원본 기하학의 곡률(Curvature)과 기하학적 특징(Feature)을 보존하는 최적의 단순화 모델을 탐색하는 데 있다.

대표적인 수학적 기법으로는 2차 오차 메트릭 기반의 모서리 붕괴(Quadric Error Metrics based Edge Collapse) 알고리즘이 널리 활용된다. 이 알고리즘의 동작 기전은 다음과 같다.

  1. 모서리 붕괴(Edge Collapse): 연결된 두 정점 v_1과 v_2를 단일 정점 \bar{v}로 병합하여 하나의 모서리를 제거한다. 이 과정에서 주변의 인접한 삼각형 두 개가 소멸된다.
  2. 비용 함수(Cost Function) 평가: 각 모서리가 붕괴될 때 발생하는 기하학적 형태학 변화량(오차)을 2차 형식(Quadratic Form)으로 수치화한다. 평면(Planar) 영역에서의 붕괴 비용은 매우 낮게 분포하며, 예리한 엣지(Sharp Edge)나 모서리(Corner) 영역에서의 비용은 높게 산출된다.
  3. 그리디 최적화(Greedy Optimization): 모든 모서리에 대한 붕괴 비용을 평가하여 우선순위 큐(Priority Queue)에 정렬한 후, 최소 비용을 갖는 모서리부터 순차적으로 병합을 수행하여 지정된 목표 폴리곤 수(Target Polygon Count)에 도달할 때까지 반복한다.

\Delta(\bar{v}) = \bar{v}^T Q \bar{v}

여기서 Q는 정점 \bar{v}에서 원래의 표면 평면들까지의 제곱 거리 합을 나타내는 4 \times 4 대칭 2차 오차 행렬(Symmetric Quadric Error Matrix)이다. 이를 최소화하는 \bar{v}의 좌표를 탐색하여 최적의 병합 지점을 결정한다.

2. 렌더링 파이프라인에서의 부하(Overhead) 경감 기전

데시메이션이 렌더링 및 물리 시스템 성능 향상에 기여하는 바는 다음과 같이 요약할 수 있다.

  1. 정점 버퍼(Vertex Buffer)의 축소:
    수백만 개의 정점 데이터를 저장하기 위한 비디오 메모리(VRAM) 대역폭의 한계를 해소한다. 정점의 수가 감소하면 GPU의 Vertex Shader 호출 횟수가 선형적으로 감소하며, 이는 즉각적인 프레임 레이트(Frame Rate) 향상으로 이어진다.
  2. 메모리 캐시 히트율(Cache Hit Ratio) 향상:
    간결해진 메시 토폴로지는 GPU 렌더링 파이프라인의 캐시 공간 내에 더 많은 삼각형 인덱스를 상주하게 만들어 메모리 접근 지연(Latency)을 단축시킨다.
  3. Draw Call 병목 해소:
    다중 에이전트(Multi-agent) 시뮬레이션 환경, 즉 여러 대의 군집 드론을 동시에 로딩할 경우 CPU에서 GPU로 명령을 하달하는 Draw Call 부하가 기하급수적으로 증가한다. 데시메이션을 통한 에셋 규격화는 인스턴싱(Instancing) 렌더링 기법의 효율을 극대화하여 CPU 오버헤드를 경감시킨다.

3. SDF 구조 내 LOD (Level of Detail) 체계 적용

로봇 시뮬레이터 실무에서는 단일 단계의 데시메이션뿐만 아니라, 카메라 거리에 따라 다수의 메시 모델을 동적으로 교체하는 LOD(Level of Detail) 기법을 SDF 구조에 접목해야 한다.

  • LOD 0: 높은 해상도의 원본에 근접하는 메시. 드론에 장착된 RGB 카메라와 같은 로컬 센서가 자신의 기체를 근접 관찰할 때 렌더링된다.
  • LOD 1~2: 중간 해상도의 데시메이트된(Decimated) 메시. 관찰 카메라가 적당한 거리에 위치할 때 스왑(Swap)된다.
  • LOD 3: 극도로 단순화시킨 디스플레이용 저해상도 메시. 수백 미터 이상 떨어진 거리에서 시야각(FOV) 내의 극히 적은 픽셀만을 점유할 때 구동되어 불필요한 GPU 파이프라인 구동을 원천 차단한다.
