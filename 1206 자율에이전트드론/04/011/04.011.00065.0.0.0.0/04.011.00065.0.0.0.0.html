Chapter 65. 로봇 모델링 언어 URDF(Unified Robot Description Format) 링크/조인트 문법 기초

URDF(Unified Robot Description Format)는 ROS(Robot Operating System) 생태계에서 로봇의 기구학적(Kinematics), 동역학적(Dynamics), 시각적(Visual), 그리고 물리적 충돌(Collision) 속성을 XML(eXtensible Markup Language) 구조로 정형화하여 기술하는 표준 모델링 언어이다. 로봇을 구성하는 각 강체 부품은 링크(Link)로, 부품 간의 연결 및 상대적 운동은 조인트(Joint)로 정의되며, 이를 통해 트리(Tree) 구조의 로봇 모델이 구성된다. URDF 모델은 Rviz를 통한 시각화, TF(Transform) 트리를 이용한 좌표계 연산, 그리고 Gazebo와 같은 물리 엔진 시뮬레이터에서 핵심적인 환경 파일로 다루어진다. 이 장에서는 URDF의 뼈대를 이루는 <link>와 <joint> 요소의 문법 구조를 학술적으로 명확히 분석한다.

1. URDF의 기본 XML 구조 및 형식

URDF 파일은 기본적으로 <robot> 요소(Element)를 최상위 루트 단위로 삼아 작성된다. <robot> 하위에는 다수의 <link>와 <joint> 요소들이 위치하며, 하나의 URDF 문서 내에서 로봇의 전체 형상과 물리적 성질이 계층화되어 서술된다.

<?xml version="1.0"?>
<robot name="autonomous_drone">
  <!-- Link 정의부 -->
  <link name="base_link"> ... </link>
  <link name="rotor_1"> ... </link>
  
  <!-- Joint 정의부 -->
  <joint name="base_to_rotor_1" type="continuous"> ... </joint>
</robot>

XML의 헤더는 문서의 버전을 나타내며, <robot name="명칭"> 속성을 통해 해당 모델의 고유 식별자를 선언한다. 이 트리 구조 안에서는 임의의 닫힌 루프(Closed-loop) 기구학을 직접 표현할 수 없는 제약이 있으며, 오직 부모-자식 관계의 트리 형태로만 위계성을 확립해야 한다.

2. 링크(Link) 요소의 기하학적 및 동역학적 문법

URDF에서 <link> 태그는 로봇을 구성하는 각 독립된 강체(Rigid body)를 지칭한다. 링크의 명칭은 고유해야 하며, 내부적으로는 관성(inertial), 시각(visual), 충돌(collision)의 세 가지 핵심 하위 속성으로 구성된다.

2.1 시각적 속성 (Visual Property)

<visual> 태그는 Rviz 또는 시뮬레이션 환경 내에서 로봇이 인간의 눈에 보이게 될 시각적 외형(Geometry)과 재질(Material), 그리고 원점 좌표계(Origin)를 서술한다.

  • <origin>: 링크 좌표계에서 시각 모델의 기준점(xyz 위치 및 rpy 오일러 각)을 설정한다.
  • <geometry>: 박스(box), 원통(cylinder), 구(sphere) 형태의 원시 도형(Primitive) 또는 외부의 폴리곤 메시 파일(예: .dae, .stl)를 지정한다.
  • <material>: 색상(color RGBA)이나 텍스처(texture)를 지정하여 렌더링 파이프라인에서 처리할 질감 속성을 부여한다.

2.2 물리적 충돌 속성 (Collision Property)

<collision> 태그는 물리 엔진 내부에서 충돌 검사(Collision detection)를 계산하기 위한 기하학적 형태를 정의한다.

  • 일반적으로 시각적(Visual) 모델과 동일한 형상을 사용할 수 있으나, 연산 부하를 최소화하고자 복잡한 메시 파일 대신 단순화된 원시 도형(박스, 원통 등)으로 충돌 영역을 근사화(Approximation) 하는 것이 학술적 성능 최적화의 기본 원칙이다.
  • <visual>과 마찬가지로 이 역시 독립적인 <origin>과 <geometry>를 하위 태그로 지닌다.

2.3 동역학적 관성 속성 (Inertial Property)

<inertial> 태그는 시뮬레이터(예: Gazebo) 내에서 정확한 물리적 거동과 동역학 해석을 수행하기 위해 강체의 질량(Mass) 및 관성 텐서(Inertia Tensor)를 정의한다.

  • <mass>: 강체의 전체 질량을 킬로그램(kg) 단위로 value 속성에 담아 표현한다.
  • <inertia>: 3x3 대칭 관성 행렬(Symmetric Inertia Matrix)의 요소인 ixx, ixy, ixz, iyy, iyz, izz 값을 직접 설정한다. 이 값은 강체의 질량 중심에 대하여 산출된 값이어야 모션 방정식 적분 시 정확한 토크 계산이 가능하다.
  • <origin>: 링크 기준 좌표계에서 강체의 질량 중심(Center of Mass)의 위치를 기술한다.

3. 조인트(Joint) 요소의 위상학적 및 모션 문법

<joint> 태그는 두 개의 독립된 링크 간의 기하학적 구속(Constraint)을 형성하여 로봇의 자유도(Degree of Freedom)와 움직이는 방식을 명시한다. 조인트의 type 속성에 따라 해당 구속의 성질이 결정된다.

3.1 조인트의 작동 타입(Type)

제공되는 조인트의 타입 속성은 로봇의 운동학적 목적에 따라 다음 범주로 분류된다.

  • fixed: 두 링크가 어떠한 상대적 움직임 없이 완전히 결합된 상태의 강건한 고정을 뜻한다. (자유도 0)
  • revolute: 지정된 단일 축을 기준으로 회전 운동하며, 상한과 하한 각도 제한이 존재하는 관절이다.
  • continuous: 회전축을 기준으로 각도 제한 없이 무한히 회전할 수 있는 조인트 유형으로, 드론의 로터(Rotor)나 자동차의 바퀴 등에 사용된다.
  • prismatic: 지정된 단일 축을 따라 선형적인 병진 운동(Translation)을 수행하며 위치 상하한이 존재한다.
  • planar: 기준 평면 내에서의 운동을 허용한다.
  • floating: 특정 기준 없이 모든 6자유도 축(병진 3, 회전 3)에 대한 공간 내의 상대적 모션을 허용한다.

3.2 조인트 위상학 및 변환 구조

조인트는 트리 구조에서 링크의 상하 위계를 결정하는 기준이 된다.

  • <parent link="부모_링크_이름"/>: 해당 관절의 기준이 되는 상위 프레임을 지정한다.
  • <child link="자식_링크_이름"/>: 해당 관절을 통해 부모 링크에 구속되어 상대적 운동을 하게 될 하위 링크를 연결한다.
  • <origin>: 부모 링크 기준 좌표계에서 자식 링크 기준 좌표계가 연결되는 기구학적 기점을 명시한다.

3.3 조인트 축 설정 및 물리 모델링 파라미터

로봇 기구학의 올바른 투영을 위해서는 회전이나 이동이 일어날 축의 정의와 이에 수반되는 한계 및 제어 파라미터의 정밀한 선언이 요구된다.

  • <axis>: revolute, continuous, prismatic 등의 조인트에서 운동의 기준이 되는 1차원 벡터 (x, y, z) 방향을 지정한다.
  • <limit>: 관절의 물리적 혹은 제어 사양적 한계치를 정의한다. 하한/상한 위치(lower, upper), 최대 한계 속력(velocity), 관절이 견디거나 가할 수 있는 최대 힘/토크(effort)를 반드시 포함해야 시뮬레이터에서 예외 거동이 차단된다.
  • <dynamics>: 관절의 점성 마찰 계수(damping)와 정지 마찰력(friction)을 명기함으로써, PID 제어기나 임피던스 제어 상에서 보다 현실과 유사한 비선형적 마찰 저항 보상 알고리즘을 테스트할 수 있도록 근거를 제공한다.
