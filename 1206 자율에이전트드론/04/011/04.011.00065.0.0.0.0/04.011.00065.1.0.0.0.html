65.1 로봇 기술 언어(URDF)의 XML 스키마(Schema) 구조 및 다관절 로봇 위상 기하학적(Topological) 모델링

1. URDF의 XML 스키마(Schema) 및 파싱 메커니즘

URDF(Unified Robot Description Format)는 ROS 계열 소프트웨어 생태계에서 다관절 로봇 및 드론 등의 기구학적 모델을 명세하기 위해 고안된 XML(eXtensible Markup Language) 기반의 표준 스키마이다. 해당 스키마는 기계가독성(Machine-readability)과 인간가독성(Human-readability)을 동시에 충족하기 위해 트리(Tree) 계층 구조에 따른 엄격한 부모-자식 노드 문법을 채택하고 있다.

최상위 계층에는 <robot> 태그가 위치하며, 그 하위에 병렬적으로 다수의 <link> 태그와 <joint> 태그가 분기된다. XML 파서(Parser)인 urdf_parser_plugin 등은 이 스키마를 읽어들여 메모리 상에 객체(Object)화된 기구학적 그래프 구조를 생성한다. 이를 통해 TF(Transform) 브로드캐스터, Rviz와 같은 시각화 툴, 그리고 Robot State Publisher가 로봇 각 링크의 3차원 공간상 절대 위상을 계산할 수 있는 기반 데이터 구조를 확보하게 된다.

2. 다관절 로봇의 위상 기하학적(Topological) 모델링 원칙

위상 기하학적 측면에서 URDF는 다관절을 포함하는 로봇 체계를 반드시 **방향성 비순환 그래프(Directed Acyclic Graph, DAG)**의 일종인 루트가 있는 트리(Rooted Tree) 구조로만 기술하도록 강제한다. 이는 로봇 베이스(Base Link)를 유일한 루트 노드(Root Node)로 설정하고, 조인트 컴포넌트들을 간선(Edge) 삼아 각 링크 컴포넌트를 단일 경로로 순회할 수 있도록 하기 위함이다.

2.1 폐구조 기구학(Closed-Loop Kinematics)의 표현 제약

URDF 스키마의 근본적인 한계점이자 수학적 특성은 닫힌 루프(Closed-loop), 즉 병렬 링크나 폐쇄된 기구학적 체인을 네이티브로 표현할 수 없다는 것이다. 트리 위상 구조는 행렬 연산과 순방향 기구학(Forward Kinematics) 풀이에 있어서 계산 복잡도를 O(N)으로 낮출 수 있는 커다란 이점을 제공하지만, 4절 링크 기구(Four-bar linkage)와 같은 구조에서는 필연적으로 모순을 야기한다. 따라서 순수 URDF만으로는 이러한 폐쇄 루프를 직접 명시할 수 없으며, 물리 엔진과 결합되었을 때에는 추가적인 가상 구속(Virtual Constraint) 혹은 SDF(Simulation Description Format)로의 변환을 통해서만 해결해야 한다.

2.2 좌표계 변환 트리의 연쇄(Concatenation) 속성

각 <joint> 요소는 하위 <link>가 기준이 되는 상위 <link>로부터 어떻게 위치 변환(Translation) 및 회전 변환(Rotation)되었는가를 위상학적으로 설명한다. 조인트 j에 의해 연결된 부모 링크 L_{i-1}과 자식 링크 L_i 사이의 상대적 동차 변환 행렬(Homogeneous Transformation Matrix) T_{i-1}^{i}는 조인트의 origin 태그 및 조인트의 구동 값 q_i에 의해 동적으로 결정된다.

루트 링크 L_0부터 특정한 말단 장치(End-Effector) L_n에 이르기까지의 전역 변환 행렬 T_{0}^{n}은 다음과 같이 트리 경로를 따르는 동차 행렬들의 연속적인 곱으로 계산된다.

T_{0}^{n}(q) = \prod_{k=1}^{n} T_{k-1}^{k}(q_k)

URDF의 트리 스키마 설계는 위 수식을 Robot State Publisher가 최적화된 형태로 연산하여 실시간으로 ROS 네트워크 상의 /tf 토픽에 게시할 수 있도록 보장하는 가장 필수적인 위상학적 명세서 역할을 수행한다.
