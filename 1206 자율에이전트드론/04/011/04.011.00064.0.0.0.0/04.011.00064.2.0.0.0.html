64.2 시뮬레이션 상태 관리용 엔티티-컴포넌트-시스템(Entity-Component-System, ECS) 분산 아키텍처의 수학적 배경

1. 객체 지향 프로그래밍(OOP) 패러다임의 메모리 조각화 문제

고전적인 로보틱스 시뮬레이터 소프트웨어는 클래스 계층 내 상속(Inheritance)과 가상 함수 기반 다형성(Polymorphism)에 기초한 객체 지향 프로그래밍(OOP, Object-Oriented Programming) 패러다임으로 설계되었다. OOP 생태계 내에서 가상 드론 에이전트는 기체의 동역학적 질량 속성, 3차원 충돌 기하학 메타데이터, 시각적 렌더링 텍스처 등 모든 상태값을 단일 내포하는 거대한 객체(Object) 인스턴스로 정의되며, 이는 운영체제의 메모리 힙(Heap) 공간 상에 비연속적으로 동적 할당된다. 시뮬레이션의 스텝 주기가 도래할 때마다 중앙 처리 장치(CPU)는 각 객체의 파편화된 메모리 주소를 포인터로 개별 순회하며 행위 함수를 호출해야 하므로, 캐시 메모리 적중 실패(Cache Miss) 현상이 필연적으로 증폭된다. 이러한 ‘배열 구조의 배열(Array of Structures, AoS)’ 형태의 데이터 레이아웃은 수천 단위의 에이전트 인스턴스와 수만 단위의 센서 레이캐스트(Raycast)가 동시 다발적으로 연동되는 군집 시스템 구동 시 비선형적인 연산 병목점(Bottleneck)으로 귀결된다.

2. 데이터 지향 설계(DOD) 기반의 수리적 컴포넌트 분해

Gazebo Sim(Harmonic 버전) 플랫폼은 상기한 문제를 물리 엔진 수준에서 제거하고자 데이터 지향 설계(Data-Oriented Design, DOD) 패러다임을 도입하였으며, 전체 시뮬레이션 상태를 수학적 집합론 관점에서 완전히 해체 및 연산하는 Entity-Component-System(ECS) 아키텍처를 채택하였다.

  • 엔티티(Entity, E): 어떠한 종속적 메모리 데이터 블록이나 내부 상태 함수도 갖지 않으며, 오직 가상 공간 상의 논리성만을 표방하는 64비트 정수형(Integer) 식별자 e \in E (단, E는 전체 식별자의 유한 집합)로 규정된다.
  • 컴포넌트(Component, C): 엔티티가 소유하는 객체 속성의 수리적 벡터 구조이다. 강체의 질량 벡터 \mathbf{m}, 쿼터니언 회전 포즈 벡터 \mathbf{p}, 관성 텐서(Inertia tensor) 행렬 \mathbf{I} 와 같은 단일 특성들은 C = \{c_1, c_2, \dots, c_n\} 집합 공간상에서 종속성이 전혀 없는 순수상태 데이터(Plain Old Data, POD) 구조로 분리되어 1차원 선형 메모리 배열에 연속적으로 적재된다.

이러한 ‘구조체의 배열(Structure of Arrays, SoA)’ 기반 데이터 포맷을 통해 특정 속성 c_i 범주에 동위 속하는 에이전트들의 모든 속성 데이터는 물리 메모리에 연속된 파이프라인 청크(Chunk) 공간으로 나열된다. 결과적으로 궤적 추정, 충돌계수 연산과 같은 단일 수학 연산이 요청될 때 CPU는 선입선출 기반 단일 캐시 루프만으로 연산을 완료(Cache Locality 극대화)할 수 있어, 하드웨어 계층의 SIMD(Single Instruction Multiple Data) 병렬 가속 연산을 알고리즘 수준에서 구조적으로 보장하게 된다.

3. 시스템(System) 연산 함수의 직교성과 시계열 상태 전이 메커니즘

ECS 아키텍처 체계 내에서 시스템(System, S) 모듈은 자체적인 메모리 상태 스토리지를 은닉 보유하지 않으며 오직 시간 변위에 따른 특정 컴포넌트 조합 배열을 일괄 판독하여, t+\Delta t 시점의 컴포넌트로 비동기 전이시키는 상태 변환 사상(Mapping) 함수 S: C(t) \to C(t + \Delta t) 단위로만 동작한다.
일례로 강체 동역학을 주관하는 역학 시스템(Physics System)은 전체 엔티티 집합 E 중에서 공간 포즈 배열 집합 구조 \mathbf{P} 와 관성 텐서 집합 구조 \mathbf{M} 컴포넌트를 동시에 부착 보유한 조건을 만족하는 서브셋 E_{dyn} \subseteq E 영역만을 수리적으로 필터링 추출해 낸다. 시스템은 외부 힘(Wrench) 행렬을 인가하여 대상 서브셋의 가속도를 적분하는 뉴턴-오일러 수치해석 연산만을 단일 프로세스 루프 안에서 완수한다. 이와 같이 연산 요소와 데이터 구조 컴포넌트 간의 철저한 상호 직교성(Orthogonality) 분리 독립은 시뮬레이션 노드 프로세스 전반에 걸친 런타임 메모리 데드락(Deadlock)과 쓰레드 경쟁 상태(Race Condition)를 수학적 메카니즘을 통해 원천적으로 배제한다. 결과적으로 다중 센서 토픽 수집 시스템, 네비게이션 그래픽 출력 등 복합 모듈 파이프라인들이 타 모듈에 대한 데이터 간섭 없이 각자 할당 메모리에만 자원을 투입함으로써 대규모 분산 엣지 환경 대응의 무결성을 시스템 공학적으로 확립한다.
