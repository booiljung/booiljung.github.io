37.7 온디바이스 연산 중 동적 매개변수 리로드에 따른 프로세스 뮤텍스 교착 상태 회피 체계

자율 에이전트 노드는 본질적으로 멀티 스레딩(Multi-threading) 환경에서 동작한다. 센서 데이터를 수집하는 인터럽트 스레드, 제어 스펙트럼을 계산하는 메인 루프 스레드, 그리고 외부로부터의 ~/set_parameters RPC 요청을 수신하는 미들웨어 데몬 스레드가 단일 프로세스 메모리 공간을 공유한다. 이러한 동시성(Concurrency) 모델 하에서, 메인 루프가 특정 파라미터 변수를 읽어 최적화 방정식(Optimization Equation)을 연산하고 있는 도중에 미들웨어 데몬이 해당 변수에 대한 쓰기(Write) 연산을 강행한다면, 메모리 경합(Data Race)으로 인한 심각한 산술 오염이 발생한다. 본 절에서는 이러한 동시성 결함을 방어하기 위해 도입되는 뮤텍스(Mutex) 락킹 메커니즘과, 이 과정에서 태생적으로 파생되는 교착 상태(Deadlock)를 회피하기 위한 C++ 시스템 프로그래밍 기법을 분석한다.

1. 파라미터 메모리 경합(Data Race)의 파괴적 파급 효과

드론의 경로 계획(Path Planning) 노드가 3차원 장애물 맵을 스캔하기 위해 search_radius 파라미터를 사용한다고 가정하자.

1.1 비결정론적 상태의 런타임 발현

\mathcal{O}(N^3)의 시간 복잡도를 가지는 A* 알고리즘이 search_radius 변수를 읽어와 k-번째 노드를 탐색하고 있는 수백 밀리초(ms)의 연산 도중, 사용자가 ros2 param set 커맨드를 통해 이 반경 값을 5.0\text{m}에서 2.0\text{m}로 축소시켰다. 만약 아무런 동기화 방어막 없이 이 값이 즉시 메모리에 덮어씌워진다면, 알고리즘은 전반부에서는 반경 5.0을 기준으로, 후반부에서는 반경 2.0을 기준으로 연산을 수행하는 기형적 상태 분리(State Bifurcation)를 겪게 된다. 이는 힙(Heap) 메모리 참조 범위의 위반을 초래하여 세그멘테이션 결함(Segmentation Fault)으로 직결된다.

2. 상호 배제(Mutual Exclusion) 구현과 교착 상태(Deadlock)의 위협

데이터 레이스를 방어하는 표준적인 시스템 콜은 std::mutex를 통한 상호 배제(Mutual Exclusion)이다. 파라미터를 읽어 알고리즘을 수행하는 메인 영역과 파라미터를 쓰는 콜백 영역 모두에 락(Lock)을 걸어 배타적 접근 권한을 획득하게 한다.

2.1 락 순서 역전(Lock Ordering Inversion)과 교착 모델

그러나 ROS2의 콜백 그룹(Callback Group) 아키텍처 상에서 부주의한 뮤텍스 사용은 즉각적인 스레드 결빙(Thread Freezing), 즉 교착 상태(Deadlock)를 초래한다.
제어 루프(Thread A)가 파라미터 P_1을 읽기 위해 mutex_A를 점유한 채로 내부 서비스 호출 대기 상태에 들어가고, 동시에 파라미터 이벤트 콜백(Thread B)이 파라미터 P_1을 갱신하기 위해 mutex_A 획득을 시도하며 대기(Block) 상태로 빠져든다면, 시스템 스케줄러(Scheduler)는 영원히 해소될 수 없는 환형 대기(Circular Wait) 사이클에 진입한다. 교착 상태에 빠진 노드는 운영체제로 에러조차 보고하지 않은 채 침묵하며 시스템 리소스를 점유하므로, 비행 에이전트 측면에서는 명백한 소프트웨어 충돌(Crash)보다도 훨씬 발견 및 복구가 어려운 치명적 결함으로 귀결된다.

3. 락-프리(Lock-Free) 원자적 객체와 섀도우 버퍼(Shadow Buffer) 캐싱 전략

실시간 제어 시스템의 엄격한 타이밍 마진(Timing Margin)을 충족하면서 데이터 레이스를 방어하기 위해, 최신 자율 에이전트 노드 설계는 블로킹 락(Blocking Lock)을 지양하고 원자적 변수(Atomic Variables) 및 메시지 캐싱(Caching) 패턴을 채택한다.

  1. std::atomic 기반의 플래그 래핑(Wrapping): 구조체가 아닌 단일 스칼라 수치형(예: bool, int, double 등) 파라미터의 경우 C++11 규격의 std::atomic<T> 템플릿으로 변수를 선언하여 OS 레벨의 락 없이도 CPU 인스트럭션 차원에서의 읽기/쓰기 원자성(Atomicity)을 보장받는다.
  2. 섀도우 더블 버퍼링(Double Buffering): 다차원 배열 파라미터나 복합 트리 구조 파라미터의 경우, 파라미터 콜백 함수는 현재 제어 루프가 읽고 있는(Reading) 메모리 A 블록을 직접 갱신하지 않는다. 대신 콜백 스레드는 새롭게 복제된 메모리 B 블록(Shadow Buffer)에 파라미터를 조립해둔다. 메인 제어 루프는 연산 루프클럭(t_0 \sim t_1) 중간에는 절대 B 블록을 바라보지 않으며, 연산이 완전히 종료되어 다음 루틴을 준비하는 휴지기(Idle Phase) 에 다다라서야 포인터 스와핑(std::swap) 연산 \mathcal{O}(1) 만으로 최신 파라미터 B 블록을 실환경 컨텍스트로 교체(Reload)한다.

이러한 수리적 타이밍 분리(Timing Isolation) 설계는 뮤텍스 블로킹으로 인한 제어 타이밍 손실(Latency Jitter)을 0에 수렴시키며, 연산 도중의 상태 불일치(State Inconsistency) 구조를 아키텍처 레벨에서 근절한다.
