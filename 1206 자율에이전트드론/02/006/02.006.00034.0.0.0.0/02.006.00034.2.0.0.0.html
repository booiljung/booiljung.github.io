34.2 Python 기반 Launch Description 구조 및 선언적 프로세스 실행 흐름 제어

ROS1의 roslaunch가 XML의 마크업(Markup) 문법에 종속되어 정적인 환경 설정에 머물렀다면, ROS2는 강력한 튜링 완전(Turing-Complete) 언어인 Python(파이썬)을 전면 도입하여 동적인 시스템 부트스트래핑(Bootstrapping)의 새 지평을 열었다. 파이썬 기반의 런치 시스템은 단순히 스크립트가 위에서 아래로 실행되는 절차적 모형이 아니라, 향후 런타임 엔진에 의해 평가(Evaluation)될 객체들의 계층적 선언(Declarative Declaration)을 뼈대로 한다. 본 절에서는 이러한 객체 지향적(Object-Oriented) 런치 구조의 핵심인 LaunchDescription 클래스의 내부 토폴로지와, 이를 기반으로 한 선언적 프로세스 실행 흐름의 제어 원리를 분석한다.

1. LaunchDescription 객체의 위상 수학적 구조(Topological Structure)

모든 파이썬 런치 파일은 generate_launch_description()이라는 단일 진입점(Entry Point) 함수를 의무적으로 반환(Return)해야 한다. 이 함수가 생성하는 LaunchDescription 인스턴스는 트리(Tree) 자료구조의 루트(Root) 역을 담당하며, 그 하위의 모든 노드 실행 명세와 조건문들을 포괄하는 거대한 컨테이너(Container)이다.

1.1 액션(Action) 객체의 지연 평가(Lazy Evaluation) 모델

LaunchDescription 배열 내부에 추가(Add)되는 원소들은 즉각적으로 운영체제의 exec() 시스템 콜을 유발하지 않는다. 대신 이들은 launch.actions.Action을 상속받은 메타데이터 객체들로서, 런치 데몬이 파싱(Parsing)을 완전히 끝내고 이벤트 루프(Event Loop)를 가동할 때 비로소 상태 전이 루틴에 편입되는 ‘지연 평가’ 모델을 따른다.

  • Node 액션: ROS2 런타임 환경 위에서 특정 패키지의 실행 파일(Executable)을 구동하기 위한 명세이다. 패키지명, 실행 파일명, 런타임 네임스페이스(Namespace), 주입할 추상 파라미터 값들이 매개변수로 캡슐화되어 담긴다.
  • ExecuteProcess 액션: ROS 미들웨어 외부의 순수 리눅스 데몬이나 셸(Shell) 스크립트를 ROS의 생명주기 제어망 안으로 끌어들이는 포크(Fork) 인터페이스이다.

2. 런타임 치환(Substitution) 및 문맥(Context) 주입

환경의 동적 변화에 대응하기 위해 파이썬 런치 프레임워크는 변수 값이 런치 파일 로딩 시점이 아닌, 실제 액션 실행 직전에 문맥(Context) 체인을 통해 해석되는 치환(Substitution) 설계 패턴을 채택하였다.

2.1 추상화된 변수 토큰 클래스

  • LaunchConfiguration: CLI 명령줄에서 사용자가 입력한 인자(Argument) 값을 런타임에 액션 내부로 매핑하는 토큰이다.
  • FindPackageShare: 로컬 컴퓨터의 설치 환경마다 달라지는 패키지의 절대 디렉토리 경로(Ament Index 구조)를 빌드 타임이 아닌 런타임에 동적으로 역추적(Backtrace)하여 치환한다.

이러한 치환 클래스 배열들은 Python 내부적으로 perform(context: LaunchContext) -> str 메서드 시그니처를 구현하여, 최종적으로 문자열 형태로 환원(Resolution)된다. 이로써 단일 파이썬 스크립트가 드론 시뮬레이터 랩 환경과 극한의 실비행 임베디드 보드 사이를 수정 없이 교차 횡단할 수 있는 강건성(Robustness)의 기반이 된다.

3. 조건부(Conditional) 실행과 위상 정렬(Topological Sort) 모델

자율 무인기 제어망은 특정 하드웨어의 부재나 에러 핸들링 전략에 따라 프로세스 그룹의 전원 투입(Power-on) 논리가 분기되어야 한다. 이를 위해 선언적 흐름 제어가 개입한다.

3.1 IfCondition 및 UnlessCondition 분기

Node 인스턴스의 condition 매개변수에 IfCondition(LaunchConfiguration('use_lidar')) 형태의 검증 객체를 삽입함으로써, 런치 데몬은 해당 문맥이 True로 환원될 때만 시스템 포크를 수행한다. 파일 내에 지루한 if-else 파이썬 구문을 남발하는 대신, 액션 단위의 선언적 메타 속성으로 분기를 통제하는 보다 정형화된 아키텍처이다.

결론적으로, Python 기반 ROS2 런치 스크립트는 단순한 터미널 실행의 나열(Sequence of Commands)을 의미하지 않는다. 이는 이벤트 루프와 비동기 콜백 체계를 밑바탕으로 깔고, 치환 변수와 조건 제어 블록이 상호 얽혀 있는 고도화된 선언적 노드 그래프(Declarative Node Graph) 프로비저닝 엔진이다. 이 진보된 파이프라인 덕분에 개발자는 수백 개의 마이크로서비스로 분할된 로보틱스 생태계를 단 하나의 응집된 진입점(Entry Point)으로 안전하게 조율(Orchestration)할 수 있다.
