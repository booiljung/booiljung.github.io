34.7 실행 조건(Condition) 분기 처리를 통한 하드웨어 종속적 분산 파이프라인 모델링

자율 에이전트 소프트웨어 아키텍처의 이상적인 형태는 하드웨어 추상화(Hardware Abstraction)를 통해 물리적 기체의 이기종성(Heterogeneity)과 무관하게 단일화된 코드베이스를 유지하는 것이다. 그러나 현실의 로보틱스 시스템 개발은 데스크탑 시뮬레이션(SITL), 제한된 컴퓨팅 파워의 엣지 보드(HIL), 그리고 완제품 드론 하드웨어에 이르기까지 그 배포 환경이 극단적으로 파편화되어 있다. 이러한 하드웨어 종속성(Hardware Constraint)을 극복하고 단일 런치 스크립트(Launch Script)로 모든 배포 시나리오를 포용하기 위해, ROS2 런치 프레임워크는 실행 조건문(Condition Clauses)을 활용한 동적 토폴로지 분기(Dynamic Topology Branching) 기법을 제공한다. 본 절에서는 Condition 객체의 논리적 평가 원리와 하드웨어 특정적(Hardware-Specific) 파이프라인 모델링 패턴을 상술한다.

1. 런치 조건문(Condition) 객체의 논리 연산 모델

런치 파이프라인 상의 모든 Action 객체는 condition이라는 메타 매개변수를 수용할 수 있다. 이 속성에 할당되는 launch.conditions.Condition 베이스 클래스 파생 객체들은 런치 이벤트 루프가 해당 액션을 포크(Fork)할지 여부를 판독하는 게이트키퍼(Gatekeeper) 역할을 수행한다.

1.1 IfCondition 및 UnlessCondition의 평가(Evaluation) 메커니즘

가장 범용적인 두 파생 객체인 IfCondition과 UnlessCondition은 내부적으로 런타임에 치환되는 문자열 블록을 단일 논리값(Boolean)으로 환원(Resolution)하는 파서 엔진을 거친다.
예를 들어 IfCondition(LaunchConfiguration('sim_mode'))로 선언된 조건문은, 시스템 구동 시 커맨드라인에서 맵핑된 sim_mode 값이 'true', '1', 't', 'y' 등의 긍정 문자열 해시풀(Hash Pool)에 일치할 경우에만 해당 액션 가지(Branch)를 실행 상태로 스케줄링한다. 반대의 구문들은 부정 문자열을 파싱한다.

이러한 지연 평가(Late Evaluation) 덕분에, 코드베이스 내에 “시뮬레이션 전용 파일“과 “실기체 전용 파일“을 분리하여 스파게티 코드(Spaghetti Code)를 양산하는 안티패턴(Anti-Pattern)을 방지할 수 있다.

2. 하드웨어 및 페이로드 상태 분기(Branching) 아키텍처

다수의 센서와 혼합 컴퓨팅 유닛(CPU + GPU 가속기)을 탑재한 시스템에서는, 특정 하드웨어가 물리적으로 부재(Absent)할 때 에러를 발생시키는 대신 대체 모듈을 우회(Bypass) 실행하는 포용적 아키텍처가 필수적이다.

2.1 텐서 가속기 유무에 따른 추론 노드(Inference Node) 이원화

드론에 고성능 NPU(Neural Processing Unit)가 탑재된 기체와, 전력 제한으로 일반 CPU만을 탑재한 기체가 혼재하는 군집 비행 모델을 상정할 수 있다. 이 경우, 단일 런치 파일 내에서 --ros-args -p use_npu:=true/false 환경 변수에 따라 노크 그래프가 분기되도록 설계한다.

  1. IfCondition(use_npu): 고해상도 이미지 처리용 PyTorch 최적화 가속 노드 포크.
  2. UnlessCondition(use_npu): 동일한 메시지 서명(Topic Signature)을 가지지만 연산량이 경량화된(Lightweight) CPU 기반의 OpenCV 폭포수 모델(Cascade) 노드로 대체 포크.

결과적으로 미들웨어를 구독하는 후행 피드백 제어기 입장에서는, 하부의 추론 처리 노드가 NPU 기반인지 CPU 기반인지 그 내부 복잡도를 인지할 필요 없이 투명하게 데이터 스트림을 공급받는 완벽한 다형성(Polymorphism)이 구현된다.

2.2 조건부 포함(Conditional Include)을 통한 지연 로딩 오버헤드 삭감

이러한 Condition 논리식은 단일 Node 액션을 넘어, 거대한 서브 파이프라인을 병합하는 IncludeLaunchDescription 계층에도 동일하게 적용 가능하다. 즉, 특정 하드웨어나 모드에 진입하지 않는 이상 무거운 서브 시스템의 XML/Python DOM 트리를 해석조차 하지 않게 됨으로써 런치 데몬의 초기 메모리 풋프린트(Footprint)와 런타임 지연을 획기적으로 삭감시킨다.

ROS2의 실행 조건 처리는 런치 시스템을 단순한 부트로더(Bootloader)에서 환경-반응형(Environment-Reactive) 오케스트레이션 엔진으로 격상시키는 강력한 언어적 장치이다.
