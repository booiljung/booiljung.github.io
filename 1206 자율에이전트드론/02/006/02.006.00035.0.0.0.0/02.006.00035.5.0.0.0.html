35.5 시스템 런치 레벨 로드 컴포저블 노드(LoadComposableNodes) 지시자 응용 및 런타임 스레드 할당

컴포지션(Composition) 아키텍처의 이론적 이점을 실제 구동 가능한 로보틱스 애플리케이션으로 전환하기 위해서는, 개별 컴포넌트(.so 파일)들을 어느 컴포넌트 관리자(Component Manager) 시스템 컨테이너에 적재할 것인지를 결정하고 그에 따른 CPU 스레드(Thread) 자원을 조율해야 한다. ROS2 런치 프레임워크는 이러한 동적 맵핑을 파이썬 스크립트 상에서 추상화된 모델로 제어할 수 있도록 LoadComposableNodes 런치 액션을 제공한다. 본 절에서는 이 런치 액션의 작동 원리를 탐구하고, 고밀도 멀티 컴포넌트 환경에서 단일 프로세스 내의 이그제큐터(Executor) 스레드를 어떻게 효과적으로 할당하고 런타임 콜백 처리에 스케줄링하는지를 명세한다.

1. LoadComposableNodes 액션을 통한 동적 프로세스 프로비저닝

ROS1의 모놀리식 .launch 파일 구조와 구분되는 ROS2 런치 메타-모델의 핵심은 프로세스(컨테이너)의 기동 행위와 라이브러리(노드)의 적재 행위가 논리적으로 완전히 격리된다는 점이다.

1.1 지연-적재(Late-Loading) 패턴과 컨테이너 타겟팅

통상적으로 런치 파일 내에서 빈 컨테이너 프로세스(ComposableNodeContainer)가 포크(Fork)된 후, 독립적인 액션인 LoadComposableNodes 명령이 뒤따른다. 이 런치 액션은 필수적으로 두 가지 메타 파라미터를 요구한다.

  1. target_container: 적재의 목적지가 될 호스트 컨테이너 프로세스의 이름표(식별자).
  2. composable_node_descriptions: 해당 목적지로 전송될 하나 이상의 ComposableNode 명세 객체(패키지명, 플러그인 클래스 이름, 파라미터, 네임스페이스 매핑 등)의 배열.

이 분리된 지시 모델은 다중 드론 편대의 런치 환경에서 강력한 유연성을 제공한다. 예를 들어, 중앙에서 공통 컨테이너를 하나 띄워두고 개별 센서나 알고리즘 런치 스크립트들이 LoadComposableNodes 액션만을 호출하여 각자의 플러그인을 그 빈 컨테이너 “속으로” 끼워넣는 형태의 모듈식 조립성(Modular Composability)이 성립된다.

2. 런타임 스레드 할당(Thread Allocation) 및 이그제큐터 바인딩

다수의 노드 클래스를 단일 프로세스 컨테이너 안으로 로드한 직후 당면하는 시스템 공학적 과제는 스케줄링(Scheduling)이다. 수십 개의 토픽 콜백(Callback)과 타이머가 하나의 OS 프로세스 내부에 공존할 때, 이들의 동시성(Concurrency)을 어떻게 담보할 것인가 하는 문제다.

2.1 싱글(Single) vs 멀티(Multi) 스레드 이그제큐터 파라미터화

컴포지션 프레임워크는 개별 노드가 자체적인 백그라운드 구동 스레드를 소유하는 것을 지양하고, 컨테이너가 제공하는 중앙집중식 스레드 풀(Thread Pool)인 Executor에게 콜백 처리를 위임한다. 이 이그제큐터의 모드는 런치 시 ComposableNode 인스턴스의 파라미터를 통해 명시적으로 스위칭할 수 있다.

  • SingleThreadedExecutor (기본값): 컨테이너 내부의 모든 컴포넌트가 단 1개의 물리적 CPU 스레드 위에서 순차적으로(Sequential) 콜백을 처리한다. 동기화 뮤텍스(Mutex) 오버헤드가 배제되어 결정론적 제어가 필수적인 저수준(Low-Level) 파이프라인에서 유리하지만, 코어 활용 밀도가 떨어진다.
  • MultiThreadedExecutor: 컨테이너가 가용한 복수의 CPU 워커 스레드(Worker Thread)를 할당받아, 각 컴포넌트의 가용 콜백들을 락-프리(Lock-free) 환경 내에서 병렬 해소한다.

2.2 노드별 논리적 스레드 그룹 격리 (Callback Group)

하나의 거대 컨테이너 안에서도 특정 비전 딥러닝 컴포넌트가 전체 스레드 코어를 장기 점유(Starvation)하는 상황을 회피하기 위해, 런치 파라미터를 통해 특정 ComposableNode들을 상호 배타적인(Mutually Exclusive) CallbackGroup 토폴로지로 묶을 수 있다. 이는 단일 OS 프로세스 내부에서도 중요도가 높은 텔레메트리 컴포넌트가 최우선 순위 스레드 블록을 점유하여 시분할(Time-slicing) 인터럽트를 강제하는 고도화된 실시간 런타임 자원 스케줄링(Runtime Resource Scheduling) 패턴을 완성한다.
