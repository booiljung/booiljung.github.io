35.8 실시간 인터럽트 루프 제어 스택 내 컴포지션 연산 아키텍처를 통한 시스템 큐잉 지연 감소율

드론의 하위 비행 제어기(Flight Controller, FC)와 교신하는 고빈도(High-Frequency) 실시간 인터럽트 제어 루프(Real-time Interrupt Control Loop)에서는 단 1\text{ms}의 시스템 큐잉 지연(Queueing Delay) 누적조차도 전체 제어 시스템의 위상 여유(Phase Margin)를 감소시켜 치명적인 자율비행 불안정성을 야기한다. ROS2 미들웨어 스택에서 데이터를 교환할 때 발생하는 엔드투엔드(End-to-End) 레이턴시는 순수 연산 시간뿐만 아니라, 이그제큐터(Executor) 큐에서의 대기 시간과 OS 커널 레벨의 컨텍스트 스위칭(Context Switching) 비용을 포괄한다. 본 절에서는 컴포지션(Composition) 연산 아키텍처가 실시간 제어 스택의 큐잉 지연(Queueing Delay)을 수학적으로 얼마나 감축시킬 수 있는가에 대한 모델링 및 한계 감소율을 실증적으로 고찰한다.

1. 다중 프로세스 토폴로지에서의 큐잉 릴레이(Queueing Relay) 누적 모델

모터 제어 출력을 생성하기 위해 인식(Perception) -> 상태 추정(Estimation) -> 궤적 계획(Planning) -> 제어(Control)로 이어지는 전형적인 직렬 파이프라인(Serial Pipeline)을 상정할 때, 각 단계가 독립된 OS 프로세스(Standalone Node)로 구동될 경우 지연 시간은 선형적 가산 모델을 넘어선다.

1.1 버퍼링 지연(Buffering Delay)과 OS 스케줄링 간섭

전체 제어 주기 T_{control\_loop} 내에서 발생하는 큐잉 지연 모델은 다음과 같다.

T_{queue\_total} = \sum_{i=1}^{K} \left( D_{DDS\_tx}^{(i)} + W_{OS\_sched}^{(i)} + D_{exec\_rx}^{(i)} \right)

여기서 K는 파이프라인의 노드 개수, D_{DDS\_tx}는 데이터 그램이 네트워크 소켓 버퍼로 들어가기 위해 대기하는 송신 큐 지연, W_{OS\_sched}는 다음 노드가 OS 스케줄러에 의해 CPU 코어를 선점(Preemption)받기까지의 대기 시간 유격(Jitter), D_{exec\_rx}는 수신된 메시지가 해당 노드의 SingleThreadedExecutor 이벤트 큐에 적재된 후 콜백 함수가 호출되기까지의 대기 시간이다.
특히 W_{OS\_sched}는 비결정론적(Non-deterministic) 스케줄러 환경 하에서 극심한 테일 레이턴시(Tail Latency) 스파이크를 발생시키는 주범이 된다.

35.8.2 컴포지션 통합에 따른 콜백 큐잉 평탄화(Flattening) 역학

전체 제어 스택을 단일 컴포넌트 컨테이너(Component Container)로 병합(Compositioning)할 경우, 통신 파이프라인은 앞서 서술된 OS 커널 레벨의 중개자를 거치지 않는다.

스레드 내부 이벤트 릴레이(In-Thread Event Relay)

컴포지션 아키텍처 하에서는 데이터가 송신 큐(D_{DDS\_tx})나 수신 네트워크 버퍼 시스템을 통과할 필요 없이, 발행(Publish) 즉시 대상 서브스크라이버의 이그제큐터 레디 큐(Ready Queue)로 스마트 포인터가 다이렉트로 삽입된다.

그 결과, 큐잉 지연 함수는 컨테이너 내에서 구동되는 멀티스레드 이그제큐터의 스레드 대기 상태만으로 극단적으로 단순화된다.

T_{queue\_comp} = W_{exec\_wait} + \epsilon

여기서 W_{exec\_wait}는 시스템의 워커 스레드(Worker Thread)가 앞선 다른 콜백의 처리를 마칠 때까지 발생하는 컨테이너 내부 큐잉 지연이며, OS 스케줄링 간계로 인한 레이턴시 점프(W_{OS\_sched})는 수학적으로 완전히 소거(\rightarrow 0)된다. 또한, D_{DDS\_tx}와 연관된 커널 인터럽트 비용도 제거되어 지연 감소율(Delay Reduction Rate)은 스택의 깊이 K가 깊어질수록 지수적으로 퍼포먼스 우위를 확보한다.

2. 실시간성(Real-time) 한계와 PBO(Priority-Based Orchestration)의 필요성

컴포지션 적용 시 \mathcal{O}(K)에 달하던 릴레이 오버헤드는 붕괴하지만, W_{exec\_wait} 자체의 최적화는 여전히 컴포지션 스택의 잔여 과제로 남는다.

단일 컨테이너 내부에서 라이다 필터(LiDAR Filter)와 같이 연산 밀도가 압도적으로 높은 콜백 함수가 특정 워커 스레드를 장시간 독점할 경우(Head-of-Line Blocking), 실시간성이 보장되어야 할 1000\text{Hz} PID 제어기 콜백이 큐(W_{exec\_wait})에서 무기한 체류할 위험성이 상존한다.
따라서 진정한 의미의 실시간 인터럽트 루프 제어를 달성하기 위해서는, 컴포지션을 통해 메모리 바운더리(Memory Boundary)를 허무는 것에 그쳐서는 안 된다. rclcpp 이그제큐터 레벨에서 하드 리얼타임(Hard Real-time) 운영체제의 PREEMPT_RT 패치와 연동하여, 고우선순위(High-priority) 마이크로 제어 콜백이 무거운 비전 콜백을 즉각적으로 인터럽트(Interrupt)하고 자신을 큐의 최상단으로 격상시킬 수 있는 우선순위 기반 오케스트레이션(Priority-Based Orchestration, PBO)의 구조적 보강이 필수적으로 병행되어야만 임무 수행 무결성이 입증된다.
