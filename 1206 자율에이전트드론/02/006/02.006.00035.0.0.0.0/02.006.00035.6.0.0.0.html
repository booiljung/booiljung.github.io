35.6 멀티스레드 이그제큐터 결합을 통한 병렬 이벤트 콜백 처리량 대역폭 확장 모델

단일 프로세스 컨테이너(Single Process Container) 내부에 수십 개의 컴포지션(Composition) 노드를 밀집시킬 경우 필연적으로 수반되는 시스템 병목(Bottleneck)은 콜백(Callback) 연산의 직렬화(Serialization) 현상이다. ROS2의 기본 스케줄러인 SingleThreadedExecutor는 큐(Queue)에 적재된 이벤트를 순차 해소하므로, 가장 연산량이 무거운 비전 딥러닝 루틴이 실행되는 동안 가벼운 관성 센서(IMU) 타이머 콜백마저 무기한 대기(Starvation)하는 치명적인 지연 페널티를 유발한다. 본 절에서는 운영체제(OS)의 가용 커널 스레드를 컨테이너 내부로 바인딩하여 동시성 레이턴시(Concurrency Latency)를 극복하는 MultiThreadedExecutor의 동적 커플링 구조와 처리량 대역폭(Throughput Bandwidth) 확장 모델을 분석한다.

1. 멀티스레드 이그제큐터(MultiThreadedExecutor)의 코어 할당 기제

MultiThreadedExecutor는 단일 rclcpp 문맥(Context) 하에서 호스트 머신의 멀티코어 역량을 컴포지션 프레임워크에 매핑하는 표준 엔진이다.

1.1 워커 스레드(Worker Thread) 계층망 구성

이그제큐터가 발진(Spin)할 때, 개발자는 생성자 인자로 활성화할 \text{Thread\_Count}(N)을 명시할 수 있다(미지정 시 시스템의 논리 코어 수만큼 자동 생성). 이 워커 스레드들은 전역 이벤트 큐(Global Event Queue)를 지속적으로 폴링(Polling)한다.
큐에 새 메시지가 도달하면 유후(Idle) 상태의 워커 스레드가 즉각적으로 콜백 함수 포인터를 탈취하여 병렬 연산에 돌입한다. 수학적으로 시스템 내 총 콜백 누적 처리 시간 T_{total}은 워커 스레드 수 N에 선형적으로 반비례하여 T_{total} \approx \frac{1}{N} \sum t_{callback} 로 단축되는 이상적인 대역폭 확장을 달성한다.

2. 콜백 그룹(Callback Group) 기반의 상호 배제형 락(Lock) 토폴로지

무분별한 멀티스레딩은 C++ 힙(Heap) 메모리에 대한 동시 접근(Concurrent Access)을 유발하여 시스템 상태 변수를 오염(Data Corruption)시키는 안티패턴(Anti-Pattern)을 낳는다. 이를 수학적으로 방어하기 위해 컴포지션 노드는 모든 구독자(Subscriber)와 타이머를 CallbackGroup이라는 논리적 락-바운더리(Lock-Boundary)로 묶어 이그제큐터에 종속시켜야 한다.

  • MutuallyExclusive (상호 배제 그룹): 같은 그룹 소속인 콜백 C_1, C_2는 서로 다른 스레드에 할당되더라도 어떠한 경우에도 물리적 시간축에서 겹쳐서 실행되지 않는다(Non-overlapping). 이는 단일 데이터 구조체에 대한 안전한 동기(Synchronous) 업데이트를 보장한다.
  • Reentrant (재진입 가능 그룹): 같은 그룹 소속의 콜백들이 여러 스레드에 의해 완전히 병렬적으로(Parallel) 난입 실행될 수 있음을 허용한다. 엄격한 스레드 세이프(Thread-safe) 코드로 작성된 순수 함수형 필터 등 대역폭 극대화가 필요한 연산에 할당된다.

3. 병렬 처리 최적화 컴포지션 설계 실증

가장 빈도(Frequency)가 높은 IMU 피드백 센서(1kHz)와 상대적으로 처리 시간이 긴 RGB 카메라 객체 추적기(30Hz)가 동일 컨테이너에 공존하는 아키텍처를 상정해보자.

단일 스레드 구조에서는 33ms가 소요되는 한 번의 이미지 콜백 동안 33개의 IMU 콜백이 차단되어 제어망의 발산(Divergence)을 초래한다. 그러나 MultiThreadedExecutor 환경 하에서, IMU 서브스크라이버를 별도의 MutuallyExclusive 콜백 그룹으로 분리하고 높은 시스템 우선순위(Priority)를 부여하면, OS 스케줄러는 코어 1번에 카메라 딥러닝 스레드를 할당하여 장기 연산을 지속하는 동시에 코어 2번을 통해 IMU 콜백을 1ms마다 전혀 멈춤 없이(Non-blocking) 비동기 처리해낸다.

결론적으로 멀티스레드 이그제큐터의 결합은 단일 프로세스 컨테이너 안에서도 Node 간 엄격한 시간적 분산(Temporal Distribution)을 강제해, 시스템 전체의 데이터 처리량 한계선(Throughput Ceiling)을 호스트의 물리적 코어 한계치까지 확장하는 결정적 아키텍처이다.
