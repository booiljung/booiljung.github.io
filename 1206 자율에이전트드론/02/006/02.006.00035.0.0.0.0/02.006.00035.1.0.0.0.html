35.1 독립 실행형 노드(Standalone Node)와 컴포지트 노드(Composable Node) 메모리 결합 구조 비교

자율주행 드론 군집 시스템과 같이 센서 데이터의 대역폭 집중도(Bandwidth Congestion)가 극심한 환경에서, 로보틱스 미들웨어의 토폴로지(Topology) 설계는 전체 제어 루프 레이턴시(Control Loop Latency)를 좌우하는 핵심 인자다. ROS2는 구동 바이너리의 링킹(Linking) 아키텍처 관점에서, 운영체제(OS) 프로세스로 직결되는 ’독립 실행형 노드(Standalone Node)’와 런타임에 단일 프로세스 내로 적재되는 ’컴포지트 노드(Composable Node)’라는 두 가지 이원화된 메모리 결합 구조를 제공한다. 본 절에서는 두 토폴로지의 객체 생명주기와 메모리 참조 구역(Memory Reference Boundary) 측면에서의 아키텍처적 차이를 수학적 시스템 모델 관점에서 비교 분석한다.

1. 독립 실행형 노드(Standalone Node)의 프로세스 격리 모델

독립 실행형 노드는 전통적인 POSIX 커널의 강건성(Robustness) 원칙을 그대로 계승한 구조이다. C++ 소스 코드 레벨에서 명시적인 main() 엔트리 포인트(Entry Point)를 소유하며, 컴파일러를 거쳐 독자적인 실행 바이너리(Executable Binary)를 생성한다.

1.1 메모리 고립(Memory Isolation)과 페이로드 복사 오버헤드

이 모델에서 N개의 노드가 구동되면 OS 상에는 완전히 분리된 N개의 가상 메모리 공간(Virtual Memory Space)을 할당받은 프로세스가 발진한다.

  • 장점(결함 격리): 노드 A(비전 알고리즘)에서 널 포인터 역참조(Null Pointer Dereference)와 같은 치명적인 세그멘테이션 오류(Segmentation Fault)가 발생하더라도, 이 버그가 격리된 메모리 장벽을 넘어 노드 B(모터 제어기)의 프로세스를 파괴할 수 없다.
  • 단점(IPC 병목): 센서-타겟 메시지 전달 시 퍼블리셔 프로세스는 메시지 구조체를 평탄화(Flattening)하여 커널 메모리 혹은 네트워크 스택으로 복사(Copy)해야 하고, 수신 프로세스는 이를 다시 객체로 조립해야 한다. 고해상도 스테레오 카메라 데이터의 경우 이 메모리 복사 연산 비용 \mathcal{O}(m) (여기서 m은 페이로드의 크기)이 실시간 제약(Real-Time Constraint)을 훼손할 수 있다.

2. 컴포지트 노드(Composable Node)의 공유 주소 공간 통합

ROS2 아키텍처의 혁신적 중추인 컴포지트 노드는 노드 객체를 독립적인 프로세스가 아닌 동적 공유 라이브러리(Dynamic Shared Library, .so) 형태로 캡슐화한다. main() 함수가 거세되고, 대신 플러그인 레지스트리에 자신의 클래스 생성자를 등록하는 매크로(RCLCPP_COMPONENTS_REGISTER_NODE)가 그 자리를 대체한다.

2.1 컨테이너(Container) 프로세스와 단일 힙(Single Heap) 메모리 동거

런타임에 시스템 오퍼레이터가 컴포넌트 컨테이너(Component Container) 프로세스를 포크시키면, 이 단일 호스트 프로세스는 여러 컴포지트 노드의 .so 라이브러리들을 dlopen() 콜을 통해 자신의 가상 메모리 공간 내부로 로드(Load)한다.
즉, N개의 논리적 노드가 1개의 물리적 OS 프로세스 하부에 종속되는 것이다. 다차원 배열 데이터가 노드 간 통신을 위해 rclcpp::Publisher::publish(std::unique_ptr<Msg>) 인터페이스를 거칠 때, 데이터 자체가 직렬화되어 커널 소켓을 타는 것이 아니라 메모리 힙(Heap) 영역 내에 위치한 데이터의 포인터 주소(Pointer Address) 자격증명만이 \mathcal{O}(1)의 상수 시간으로 교환된다.

3. 시스템 토폴로지 결정의 트레이드오프(Trade-off)

독립 실행형과 컴포지트 모델 중 어느 것을 채택할 것인가는 신뢰성(Reliability) 과 처리량(Throughput) 간의 비선형적 트레이드오프 곡선 상에서 결정된다.

자율 드론 시스템 엔지니어링에서는 실험적인 AI 정책 모델이나 잦은 재시작이 요구되는 비핵심 서브루틴은 철저히 독립 프로세스로 박스(Boxing) 처리하여 전체 시스템의 가용성을 보호한다. 반면에, 라이다(LiDAR) 포인트클라우드 필터링 체인 제어나 VIO(Visual-Inertial Odometry) 상태 추정 파이프라인처럼 테라바이트급의 누적 대역폭을 요구하며 지연 스파이크(Latency Spike)가 치명적인 임무 흐름은, 단일 컴포넌트 컨테이너로 강하게 묶어(Tightly-coupled) 메모리 공유 이점을 극대화하는 하이브리드(Hybrid) 아키텍처를 취하는 것이 가장 정석적인 디자인 패턴(Design Pattern)으로 간주된다.
