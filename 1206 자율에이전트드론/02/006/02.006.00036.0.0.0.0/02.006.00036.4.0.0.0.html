36.4 매개변수 선언 지시자(DeclareLaunchArgument) 구조 및 런타임 바인딩 제어 메커니즘

ROS2 기반 자율 에이전트 소프트웨어에서 시스템의 유연성을 극대화하기 위해서는 특정 패키지의 핵심 노드를 구동할 때, 외부 터미널 CLI 사용자나 상위 부모 런치 파일로부터 런타임 변수(Runtime Variable)를 동적으로 전달받을 수 있는 인터페이스 창구가 필요하다. C++ 컴파일 기반 로직에서 main(int argc, char **argv)를 통해 커맨드 라인 인자를 수신하듯, 파이썬 기반의 런치 프레임워크는 DeclareLaunchArgument라는 액션 지시자를 통해 시스템 외부의 변수와 런치 내부의 컨텍스트를 동기화하는 바인딩(Binding) 메커니즘을 지원한다. 본 절에서는 런치 인자(Launch Argument) 선언의 구조적 명세와, 이를 실제 노드 파라미터나 구동 경로로 전달하는 통제 원리를 분석한다.

1. DeclareLaunchArgument 액션의 구조적 명세 체계

런치 파이프라인(Launch Pipeline)에서 변수를 외부로부터 수신하기 위해서는 반드시 DeclareLaunchArgument 객체가 LaunchDescription 트리의 최상단 부근에 선언되어야 한다. 이 행위는 일종의 “이 런치 파일은 해당 이름의 외부 인자를 허용한다“는 문법적 계약(Contract)을 정의하는 것이다.

1.1 인터페이스 매개변수화 및 메타데이터 선언

이 액션 지시자는 세 가지 핵심 파라미터를 갖는다.

  1. name: 시스템 내외부에서 해당 변수를 참조하기 위한 고유한 키(Key) 식별자 문자열이다. (예: 'use_sim_time')
  2. default_value (선택): 외부로부터 명시적인 값 주입이 누락되었을 때, 시스템이 안전하게 후퇴(Fallback)할 수 있는 안전 기본값이다.
  3. description (선택): 타 개발자나 CLI 엔진이 ros2 launch package script.launch.py --show-args 명령어를 입력했을 때 출력되는 문서를 구성하는 메타 문자열이다.

이러한 선언부(Declaration)는 실제 변수 값을 포함하는 것이 아니며, 단순히 시스템의 런치 런타임 로더(Loader)에게 “이름이 name인 변수 포인터를 런치 스코프 내에 예약(Reserve)하라“고 지시하는 추상적 메모리 바운딩 행위에 해당한다.

2. CLI 및 부모 런치 계층으로부터의 런타임 밸류 주입

선언된 인자는 시스템 구동의 두 가지 경로를 통해 동적인 값이 채워진다.

  1. CLI(터미널)에서의 동적 주입:
    개발자가 커맨드라인에서 런치 파일을 실행할 때 <key>:=<value> 포맷을 추가적으로 덧붙이면, 런치 파서(Parser)가 이를 포획하여 DeclareLaunchArgument 스코프에 주입한다.
    예: ros2 launch drone_core bringup.launch.py map_yaml:=/tmp/map1.yaml
  2. 부모-자식(상속) 런치 체인 주입:
    단일 드론의 bringup.launch.py가 여러 개의 하위 모듈(예: sensor.launch.py, control.launch.py)을 IncludeLaunchDescription 액션으로 편입시킬 때, launch_arguments 딕셔너리를 하위로 패싱(Passing)함으로써 객체 지향 언어의 생성자 매개변수 주입과 동형의 계층적 하향 전파(Top-down Propagation) 토폴로지를 달성한다.

3. LaunchConfiguration 기반의 런타임 평가 변수 추출

바인딩이 완료된 시점에서, 파이썬 런치 스크립트의 하위 로직(예: Node 액션 블록)이 해당 인자 값을 추출하여 노드의 실제 parameters로 전환하기 위해서는 반드시 LaunchConfiguration 객체를 통한 문맥 평가(Context Evaluation) 단계를 거쳐야 한다.

map_yaml_config = LaunchConfiguration('map_yaml')

# ... 중략 ...
Node(
    package='nav2_map_server',
    executable='map_server',
    parameters=[{'yaml_filename': map_yaml_config}]
)

여기서 map_yaml_config은 파이썬 시점(Script Parsing Phase)에서는 값이 비어있는 대리자(Proxy) 객체일 뿐이다. 실제 값이 문자열로 평가(Evaluation)되어 C++ 시스템의 메모리로 넘어가는 시점은, OS 레벨에서 노드 서브 프로세스가 포크(Fork)되는 런타임 페이즈(Runtime Phase)까지 최대한 평가를 뒤로 늦추는 지연 평가(Late Evaluation)의 원리가 철저히 적용된다. 이 강력한 제어 메커니즘 덕분에 ROS2 런치 아키텍처는 단일 파이썬 스크립트 실행 환경의 제약을 벗어나, 비결정론적으로 다가오는 외부 환경 파라미터 변화에 유연하게 대응하는 런타임-안전(Runtime-safe) 분산 궤도 시스템으로 격상될 수 있는 것이다.
