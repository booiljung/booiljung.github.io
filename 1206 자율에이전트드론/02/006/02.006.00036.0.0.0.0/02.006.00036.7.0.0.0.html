36.7 리스트(List) 및 바이트 배열 복합 데이터의 YAML 역직렬화 무결성 검증

로보틱스 튜닝 파라미터는 단일 스칼라(Scalar) 값에 머무르지 않는다. 상태 추정 필터(EKF)의 방대한 15 \times 15 공분산 행렬(Covariance Matrix) 초기값이나, 다관절 로봇의 각 조인트(Joint) 각도 한계치 배열, 혹은 시리얼 통신의 초기화 바이트 페이로드(Byte Payload) 등은 본질적으로 복합 데이터 타입(Composite Data Type)을 요구한다. ROS2 파라미터 백엔드는 이러한 1차원 리스트(List)와 바이트 배열(Byte Array)을 지원하나, C++의 강타입(Strongly Typed) 환경으로 인입되는 과정에서 발생할 수 있는 자료형 오염과 역직렬화(Deserialization) 결함을 방어해야만 한다. 본 절에서는 YAML 구성 파일 내 배열 데이터가 ROS2 메모리로 사상(Mapping)되는 역직렬화 구조와 그 무결성 검증 메커니즘을 상세히 다룬다.

1. 혼합 타입(Mixed Type) 리스트의 묵시적 캐스팅오류 차단

YAML 스펙 자체는 하나의 리스트 안에 정수, 실수, 문자열이 혼재된 이종(Heterogeneous) 배열을 허용하지만, ROS2 파라미터 시스템의 원시 모델인 rcl_interfaces::msg::ParameterValue는 이를 수학적으로 철저히 배척한다.

1.1 동질성(Homogeneous) 검증 알고리즘

YAML 파서가 covariance_matrix: [1.0, 0.5, 0] 와 같은 선언을 읽어들일 때, 파서는 리스트 내의 모든 원소가 정확히 동일한 기본 타입(Base Type)을 공유하는지 선형 시간 \mathcal{O}(N) 으로 스캔한다.
위 예시에서 1.0과 0.5는 PARAMETER_DOUBLE로 해석되나, 마지막 원소인 0은 PARAMETER_INTEGER로 해석될 여지가 있다. 초기 ROS2 릴리즈에서는 이러한 미묘한 혼합이 런타임 에러를 유발했으나, 현재의 파서 백엔드는 소수점이 생략된 정수 표현이라도 리스트의 지배적 타입이 double일 경우 이를 부동소수점 0.0으로 묵시적 상향 변환(Implicit Upcasting)하는 톨러런스(Tolerance)를 제공한다.
그러나 [1.0, "high_gain", true]와 같이 병합이 수리적으로 불가능한 리스트가 주입될 경우, 런치 시스템은 예외를 던지고 파이프라인 부팅을 즉각 중단(Hard Abort)하여 타입 오염이 제어 루프로 전파되는 것을 원천 차단한다.

2. 바이트 배열(Byte Array)의 직렬화 페이로드 최적화

일반적인 숫자 배열(DoubleArray, IntegerArray)과 달리, 하드웨어 장치 드라이버(I2C, SPI, CAN)를 초기화하기 위한 원시 이진 데이터(Raw Binary Data)는 ByteArray 타입으로 선언되어야 한다.

2.1 바이너리 스트림의 인코딩과 메모리 할당

일반적인 YAML 숫자 배열로 [255, 0, 128]과 같이 바이트를 선언하면 메모리 상에서 이는 N \times 64\text{-bit} 크기의 IntegerArray로 취급되어 심각한 메모리 오버헤드와 캐시 미스(Cache Miss)를 유발할 수 있다.
이를 방지하기 위해 ROS2 YAML 파서는 이진 데이터를 bytes 타입으로 단일화할 수 있는 바이트 스트림(Byte Stream) 역직렬화 패스를 제공한다. 노드 내부의 C++ 런타임에서는 이것이 std::vector<uint8_t> 객체로 매핑되며, 커널 공간과 유저 공간 사이에서 최적화된 블록 복사(memcpy) 대상이 되어 직렬화 및 I/O 버퍼 전송의 대역폭 한계를 확장시킨다.

3. 다차원 텐서(Tensor) 파라미터의 단일 차원 수렴성

자율 비행의 고도화된 스택은 때때로 N \times M 차원의 행렬을 요구하지만, ROS2 파라미터 스펙은 오직 1차원(1D) 배열만을 공식 서포트한다. 이는 시스템 간 통신 IDL(Interface Definition Language)의 단순성을 극대화하기 위한 의도적인 구조적 억제(Structural Constraint)이다.

따라서 2차원 이상의 행렬 파라미터([ [1, 0], [0, 1] ] 등)가 담긴 YAML을 런치 파이어(Launch Parser)에 주입하면 해석 오류가 발생한다. 로보틱스 엔지니어는 반드시 수학적 열 우선(Column-major) 또는 행 우선(Row-major) 규칙을 자율적으로 적용하여 이러한 텐서를 1 \times (N \times M) 크기의 단일 차원(Flattened) 리스트로 평탄화(Flattening)하여 YAML에 기재해야 한다. C++ 노드는 자신의 초기화 코드 내부에서 this->get_parameter로 확보한 1차원 std::vector를 다시 Eigen::Matrix 등의 수리 라이브러리 타입으로 리셰이핑(Reshaping)하는 매퍼(Mapper) 로직을 필연적으로 구현하게 되며, 이는 설정 구성의 추상성과 행렬 연산의 무결성 간의 타협점(Trade-off)을 보여주는 대표적 패턴이다.
