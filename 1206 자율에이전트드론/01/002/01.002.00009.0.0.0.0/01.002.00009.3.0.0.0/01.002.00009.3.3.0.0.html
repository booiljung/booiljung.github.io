9.3.3 다중 기종 연동을 위한 메시징 추상화

1. 다중 기종 메시징 추상화의 필요성

이기종 오토파일럿(ArduPilot, PX4 등)은 제각기 다른 내부 통신 프로토콜과 메시지 정의를 사용한다. ArduPilot은 내부적으로 데이터 공유를 위해 AP_HAL 및 객체 참조를 활용하며, 외부 통신은 MAVLink 프로토콜을 표준으로 채택하고 있다. ROS2 기반의 상위 제어 시스템이 비행 제어기의 기종에 종속되지 않고 동일한 제어 알고리즘을 수행하기 위해서는, MAVLink 메시지 또는 내부 데이터 버스의 구조를 ROS2의 DDS(Data Distribution Service) 메시지 인터페이스로 통합하는 메시징 추상화(Messaging Abstraction) 계층이 필수적이다.

2. 메시지 추상화 계층의 구조적 아키텍처

다중 기종 연동을 위한 메시징 추상화 계층은 크게 기종 특화 메시지 직렬화(Serialization) 계층, 공통 의미론 매핑(Semantic Mapping) 계층, 그리고 ROS2 퍼블리셔/서브스크라이버 어댑터(Adapter) 계층으로 구성된다.
ArduPilot의 경우, 상위 컴패니언 컴퓨터(Companion Computer)와의 연동 시 MAVLink 시스템의 특정 방언(Dialect, 예: ardupilotmega.xml)에 종속된 메시지가 생성된다. 추상화 컴포넌트는 수신된 기종 종속적인 페이로드(Payload)를 표준화된 ROS2 데이터타입(예: sensor_msgs/NavSatFix, geometry_msgs/Twist)으로 매핑하여 퍼블리시한다. 반대로 상위 제어 명령은 공통 메시지 포맷으로 서브스크라이브된 후, ArduPilot이 해석할 수 있는 하위 MAVLink 제어 명령어(예: SET_POSITION_TARGET_LOCAL_NED) 또는 특화된 직렬화 포맷으로 변환되어 하드웨어로 전달된다.

3. 공통 트랜슬레이션 노드 패턴

효과적인 메시징 추상화를 위해 브리지 시스템 상단에 트랜슬레이션 노드(Translation Node) 구성을 적용한다. 트랜슬레이션 노드는 기종간 차이를 은닉하기 위해 다음과 같은 데이터 변환 명세를 수행한다.

  • 좌표계 변환(Coordinate Frame Conversion): 대상 비행 제어기가 사용하는 NED(North-East-Down) 좌표계를 ROS2에서 주로 사용하는 국제 표준 표기인 ENU(East-North-Up) 좌표계로 일괄 변환한다.
  • 단위 및 스케일 동기화(Unit and Scale Synchronization): ArduPilot에서 정수 기반으로 전송되는 센서 값(예: 10^7 스케일의 위경도 데이터) 또는 원시 밀리초(ms) 단위 데이터를 국제 단위계(SI)의 부동소수점 데이터 구조 매핑을 통해 정밀도를 일치시킨다.
  • 상태 머신 매핑(State Machine Mapping): 비행 모드(Flight Mode) 지시자의 추상화를 수행한다. 예를 들어 ArduPilot의 GUIDED 모드와 PX4의 OFFBOARD 모드는 외부 제어를 인가한다는 측면에서 동일한 역할을 수행하므로, 상위 인터페이스에서는 AUTONOMOUS_CONTROL과 같은 추상화된 상태 열거형(Enum)으로 통합 매핑하여 제공한다.

4. QoS 제어 및 대역폭 비대칭성 보정

다중 기종 비행 제어기 연동 시, 각 하드웨어의 연산 성능 및 텔레메트리 채널(UART, USB, CAN)의 대역폭 한계로 인해 메시지 발행 주기에 기종 간 편차가 발생한다. 메시징 추상화 계층은 ROS2 품질 서비스(QoS: Quality of Service) 프로파일을 동적으로 설정하여 이러한 비대칭성을 해결한다.
자주 업데이트되는 자세 추정 데이터(Attitude/IMU)의 경우 BEST_EFFORT 신뢰성과 VOLATILE 내구성(Durability)을 적용하여 네트워크 오버헤드와 통신 지연을 최소화한다. 반면, 홈 위치(Home Position) 갱신이나 경로점(Waypoint) 데이터 등 상태 변화에 필수적인 메시지는 상태 손실 방지를 위해 RELIABLE 신뢰성 및 TRANSIENT_LOCAL 내구성을 부여하여 메시지 무결성을 보장한다. 이 아키텍처를 통해 상위 ROS2 노드는 하위 텔레메트리 연결의 물리적 및 프로토콜적 제약과 무관하게 일관된 메시지 수신 및 발신 성능을 유지할 수 있다.
