10.1 MAVROS 개요 및 등장 배경

1. 개요

MAVROS(MAVLink to ROS Gateway)는 자율 에이전트 드론의 하위 비행 제어기(Flight Controller)와 상위 임무 제어를 담당하는 컴패니언 컴퓨터(Companion Computer) 간의 통신을 물리적·논리적으로 매개하기 위해 고안된 ROS 1 및 초기 ROS 2 환경의 핵심 브리지(Bridge) 패키지이다. 자율 드론 연구의 초창기에는 비행 제어기가 사용하는 독자적인 MAVLink 프로토콜 생태계와 고수준의 컴퓨터 비전 및 경로 탐색 알고리즘이 동작하는 ROS 생태계가 완전히 배타적으로 분리되어 존재했다. MAVROS는 이 이질적인 두 통신 규격을 중재하는 미들웨어 게이트웨이(Middleware Gateway) 역할을 수행함으로써, 로보틱스 연구자들이 드론의 저수준 물리 계층을 직접 제어하는 펌웨어 수정 없이도 고수준의 자율 에이전트 인공지능(AI)과 행동 정책 모델을 즉각적으로 통합할 수 있게 한 역사적 교두보이다.

2. 이기종 생태계의 분리와 통신 인터페이스 통합 요구

자율 비행 시스템은 본질적으로 다양한 타이밍 제약을 지닌 다중 처리 체계(Multiphasic Processing System)를 요구한다.

  • 운영체제 및 컴퓨팅 생태계의 단절: 시스템 하단에서는 고정된 시간 내에 반드시 연산을 마쳐야 하는 실시간 운영체제(RTOS) 기반의 PX4 및 ArduPilot 펌웨어가 대역폭 1KB 미만의 극도 경량화된 MAVLink 직렬 메시지를 사용하여 모터 출력과 로우 레벨 센서를 제어한다. 반면 상위 시스템에서는 Ubuntu 환경을 기반으로 동작하는 ROS 프레임워크 패키지가 기가바이트(GB) 단위의 라이다(LiDAR) 포인트 클라우드 및 다차원 텐서(Tensor) 배열을 비동기적으로 다룬다.
  • 프로토콜 간극의 구조적 존재: MAVLink는 시리얼 통신(Serial Communication) 인터페이스의 좁은 대역폭 한계를 극복하기 위해 설계된 매우 조밀한 이진(Binary) 구조 인코딩을 지니지만, 상위의 ROS 미들웨어(ROS 1의 TCPROS 또는 ROS 2의 DDS 규격)는 대용량 로컬 네트워크 분산 컴퓨팅 환경에서의 유연한 퍼블리셔-서브스크라이버(Pub/Sub) 기능에 최적화되어 있다. 이 생태학적 언어를 상호 번역(Translation)할 수 있는 통합 아키텍처의 부재는 무인기 통제권 확장의 가장 큰 장애물이었다.

3. MAVROS 브리지의 아키텍처 등장 배경

MAVROS는 앞서 언급한 두 프레임워크 간의 하드웨어 트랜스포트 계층과 직렬화 프로토콜 간극을 메우기 위한 브리지 게이트웨이로 설계도면에 등장했다.

  1. 표준 ROS 로봇 메시지로의 델타 변환: 로보틱스 커뮤니티는 내비게이션 프레임워크(Nav2)나 로봇 팔 모션 플래닝(MoveIt)과 같은 성숙한 ROS 표준 알고리즘 파이프라인을 드론에 그대로 적용하기 원했다. 이를 위해 MAVROS는 비행 제어기단에서 직접 올라오는 MAVLink 규격의 ATTITUDE 메시지나 GLOBAL_POSITION_INT 상태 메시지를 ROS 운영체제의 표준형 지오메트리 패키지인 geometry_msgs/PoseStamped 또는 역학 구조 패키지인 sensor_msgs/NavSatFix 데이터로 래핑하여 규격화하는 변환 렌더링을 제공했다.
  2. 다중 비행 제어 플랫폼 하드웨어 호환(Hardware-Agnosticism): 통제 스크립트가 특정한 비행 하드웨어나 펌웨어 제조사에만 종속되지 않는 범용 연구 환경을 구성하기 위하여, 컴패니언 시스템 설계는 특정 보드의 파라미터 방언(Dialect)에만 의존하지 않고 PX4, ArduPilot, 커스텀 펌웨어 환경 전반에 걸쳐 적용 가능한 통합된 MAVLink 코어 규격을 ROS 생태계 내로 범상하게 포섭할 본질적 목적이 강하게 작용했다.
  3. 오프보드 제어(Offboard Control) 아키텍처의 확립 구도: 드론이 자체 연산력으로 웨이포인트(Waypoint)를 수동 추종하는 방식을 뛰어넘어, 상위 컴패니언 컴퓨터 상의 고도화된 SOTA 궤적 추종 루프 알고리즘이 비행 제어기에게 다이렉트 명령으로 실시간 타겟 위치, 가속도 벡터, 그리고 추력 벡터 세트를 밀어 넣어주는 오프보드 제어(Offboard Control) 패러다임이 업계에 대중에 확산됨에 따라, 이를 고빈도로 안정적으로 중계 및 수신할 데몬 채널(Daemon Channel)로서 MAVROS 아키텍처가 전격 도입되었다.

4. 로보틱스 생태계 융합으로의 기여 의의

자율 드론 시스템의 파생기를 지배했던 MAVROS 프로세스는 단순한 패킷 브리지 프로토콜 치환의 역할을 넘어, 독립적으로 성장한 상업 및 오픈소스 드론 비행 기술이 거대한 로봇공학계의 데이터 생태계 안으로 구조적 융합되도록 만든 핵심 결합체(Glue Layer Middleware)였다고 학문적으로 평가된다.

  • 알고리즘 제어 개발자들은 MAVROS라는 범용 통제 인터페이스 단일 계층망을 통해, 로우 레벨 펌웨어 수준의 임베디드 C/C++ 하드웨어 코어 로직을 역설계하며 깊게 파고들지 않더라도, 주류 파이썬(Python) 환경이나 C++ 응용 환경에서 ROS 토픽 콜백 연산만으로 로터리 크래프트를 정밀하게 통제하고 자율 알고리즘을 즉각 테스트할 수 있는 개발 생명주기(Development Lifecycle)의 극적인 비례적 단축 효과 시스템을 이뤄냈다.
