10.3.4 오프보드 제어 모드 활성화 및 모드 전환 프로토콜

오프보드 제어(Offboard Control) 모드는 비행 제어기(Flight Controller, FCU)의 내부 자율 비행 로직을 우회하여, 보조 컴퓨터(Companion Computer)에서 산출된 외부 제어 명령을 직접 수용하는 작동 상태를 의미한다. MAVROS 아키텍처 내에서 오프보드 제어 모드의 활성화 및 비행 모드 전환은 mavros_msgs/srv/SetMode 서비스 인터페이스와 MAVLink의 MAV_CMD_DO_SET_MODE 프로토콜을 기반으로 수행된다.

1. 오프보드 모드 활성화의 전제 조건 및 안전 메커니즘

PX4 오토파일럿 생태계에서 오프보드 모드로 진입하기 위해서는 안전성 확보를 위한 엄격한 전제 조건이 요구된다. 보조 컴퓨터는 오프보드 모드를 활성화하기 이전에 반드시 목표 위치, 속도 또는 자세 명령(Setpoint)을 일정 주기(일반적으로 2Hz 이상)로 FCU에 지속 스트리밍해야 한다.
명령 스트리밍이 누락되거나 지연된 상태에서 오프보드 모드 전환을 요청할 경우, FCU는 이를 반려(Reject)하며 시스템 안전을 위해 기존 모드를 유지한다. 이러한 메커니즘은 통신 단절이나 보조 컴퓨터의 연산 지연 시 발생할 수 있는 제어 상실(Loss of Control)을 방지하기 위한 필수적인 아키텍처 설계이다.

2. MAVROS SetMode 서비스 인터페이스 구조

MAVROS는 ROS(Robot Operating System) 생태계의 서비스 호출 방식을 통해 비행 모드 전환 기능을 노출한다. mavros_msgs/srv/SetMode 서비스는 두 가지 주요 필드로 구성된다. 첫째, custom_mode 필드는 텍스트 문자열(예: “OFFBOARD”, “AUTO.RTL”, “MANUAL”)을 통해 펌웨어의 고유 비행 모드를 지정한다. 둘째, base_mode 필드는 MAVLink 표준 베이스 모드를 비트마스크 형태로 설정할 때 활용된다.

ROS 노드에서 서비스 전환을 요청하면, MAVROS 내부의 sys_status 플러그인은 해당 요청을 파싱하고 FCU의 펌웨어(PX4 등)에 적합한 MAVLink 메시지로 변환한다. 이 과정에서 펌웨어 종속적인 모드 문자열이 해석되어 MAVLink 통신 계층의 규격화된 명령으로 매핑된다.

3. MAVLink 명령 프로토콜 및 비행 제어기 처리 로직

MAVROS 내부에서 식별된 비행 모드 전환 요청은 MAVLink의 COMMAND_LONG 메시지로 패키징되며, 이때 명령 ID MAV_CMD_DO_SET_MODE가 할당된다. 패키징된 패킷은 직렬 통신(UART) 또는 이더넷 브리지를 통해 FCU로 전송된다.
비행 제어기의 커맨더(Commander) 서브시스템은 수신된 MAV_CMD_DO_SET_MODE 패킷을 디코딩하고, 현재 기체의 상태(시동 여부, 센서 상태, 전원 신뢰성 등)와 전제 조건 충족 여부를 엄격하게 평가한다.
평가 결과에 따라 FCU는 내부 상태 머신(State Machine) 논리를 적용하여 모드를 전환하며, 처리 결과를 COMMAND_ACK 패킷을 통해 성공 또는 거부 사유 코드와 함께 MAVROS로 회신한다. MAVROS는 이 응답을 역직렬화하여 ROS 서비스의 성공 여부(Boolean 플래그)로 매핑함으로써 폐루프(Closed-loop) 상태 통제의 완결성을 보장한다.

4. 모드 전환 시의 통신 연속성 유지 메커니즘

오프보드 모드 활성화 이후에도 보조 컴퓨터는 제어 명령 스트리밍 통신 주기를 엄격히 유지해야 한다. 비행 제어기 내부에는 오프보드 손실 방지 메커니즘(Offboard Loss Failsafe)이 구현되어 있으며, 지정된 타임아웃 시간(예: 0.5초) 내에 새로운 Setpoint 업데이트가 수집되지 않으면 예외 처리를 트리거한다.
타임아웃 발생 시, 시스템은 오프보드 모드를 강제 해제하고 위치 유지(Position/Loiter) 또는 복귀(RTL) 모드로 자동 전환하여 표류를 방지한다. 따라서 MAVROS를 이용한 오프보드 제어 아키텍처 구축 시, 실시간성이 보장되는 제어 루프 설계와 시스템 하트비트(Heartbeat)의 지속적인 교환이 전체 시스템의 주권 통제를 보장하는 핵심 원리로 작용한다.
