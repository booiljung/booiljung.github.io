11.2.2 비행 제어기 내 클라이언트 구동 메커니즘

비행 제어기(FCU)의 초소형 실시간 운영체제(RTOS, 예: NuttX) 펌웨어 커널 내부에서 네이티브로 컴파일되어 동작하는 Micro RTPS 클라이언트(micrortps_client)는, 가용 자원이 극도로 제한된 마이크로컨트롤러(MCU) 연산 환경 속에서 최소한의 메모리 접근 지터(Jitter)로 고주파 데이터의 시리얼 송·수신 입출력을 통제 방어하는 핵심 임베디드 모듈이다. 해당 클라이언트는 통신망의 상태를 관리하지 않고 철저하게 기계적인 I/O 직렬화(Serialization) 및 역직렬화(Deserialization) 버퍼 펌프 엔진으로서만 기능하며, 내부의 마이크로 객체 브로커(uORB) 생태계 메모리 트리와 외부 물리 통신 버스 장치(UART 등) 사이의 원천 데이터 게이트웨이 역할을 전담 수행한다.

1. uORB 버스 데이터 구독 및 송신 파이프라인 (Tx)

송신(Transmit) 메커니즘 전개는 하위 비행 제어기 센서 계층 및 자체 상태 추정 융합 알고리즘에서 산출 갱신된 원시 텔레메트리 데이터를 외부 엣지 동반 컴퓨터로 밀어내는 일방향 스트리밍 과정을 의미한다. 클라이언트는 펌웨어 초기 부팅 시 시스템의 독립적인 런타임 스레드(Thread) 혹은 고속 작업 큐(Work Queue)에 스케줄링 할당되어 백그라운드 루프에서 동작을 적극 개시한다.

  1. 상태 폴링(Polling) 및 접근 잠금(Lock): 클라이언트 스레드는 송신 프로파일 모델로 사전에 규정 설정된 특정 uORB 토픽들의 가용 갱신 상태(Updated State)를 인터럽트 혹은 고속 타이머로 폴링한다. 공간 내 새로운 데이터 인스턴스가 식별 판단되면 해당 uORB 메시지 메모리 블록에 즉각적인 읽기 잠금(Read Lock) 뮤텍스를 걸어 레이스 컨디션을 방지하고 대상 데이터를 안전하게 임시 지역 버퍼 큐로 복사 이관한다.
  2. MicroCDR 고속 직렬화 연산: 복사 적재된 다차원 데이터 구조체(C/C++ Struct)는 통신 파이프라인에 이식된 eProsima의 MicroCDR 마이크로 라이브러리 함수를 물리적으로 통과하며, 잉여 빈 공간이 제거 평탄화된(Flat) 압축 1차원 바이트 배열(Byte Array)로 기계적 직렬화 변환된다.
  3. 프레이밍(Framing) 패키징 및 버스 전송: 생성 추출된 바이트 배열 페이로드의 맨 선두에, 수신부가 어떤 종류의 객체 토픽인지 즉시 판별 식별하기 위한 고유 해시 토픽 ID 및 내부 페이로드 데이터의 패킷 길이를 명시하는 초경량 프레임 헤더(Framing Header) 바이트가 삽입 추가된다. 최종 결합 완성된 이 직렬 패킷 스트림은 MCU 하드웨어의 UART 시리얼 포트 혹은 이더넷 UDP 소켓 하위 드라이버 버퍼를 통해 물리적으로 강제 밀어내기(Push) 고속 송출된다.

2. 수신 파이프라인 및 내부 uORB 토픽 발행 (Rx)

수신(Receive) 역방향 메커니즘은 외부 동반 컴퓨터의 거대 ROS2 지능 연산망으로부터 인가 하달된 초고주파 제어 명령이나 다이나믹 궤적 경로점(Setpoint) 텐서 데이터를 하위 비행 제어기 내부 실행 코어로 안전하게 주입하는 역방향 디코딩 과정을 전담 처리한다.

  1. 시리얼 스트림 바이트 파싱: 클라이언트 컴포넌트의 수신 전담 스레드는 구성된 물리적 시리얼 포트 드라이버를 상시 지속적으로 인터럽트 모니터링하며 외부 유입되는 파편화된 바이트 스트림을 조심스럽게 청취한다. 불규칙한 스트림 패턴의 늪에서 약속된 프레임 헤더를 정확히 포착 인식하면, 시스템은 대상 토픽 ID와 순차적으로 읽어 들일 바이트 크기 길이를 즉시 추출하여 수신 임시 링 버퍼(Ring Buffer)에 온전히 수집한다.
  2. MicroCDR 역직렬화 복원 공간: 유효한 온전한 프레임 길이만큼의 페이로드가 버퍼에 성공적으로 수집 안착되면, 클라이언트 시스템 로직은 추출해 낸 토픽 ID 표에 1:1로 사전 정적 매핑된 특정 역직렬화 함수 코드를 직접 콜백 호출하여, 선형 1차원 바이트 배열을 다시 본래 펌웨어가 사용하는 uORB C/C++ 구조체 메모리 레이아웃 격자에 맞게 팽창 변환 기계 복원한다.
  3. uORB 버스 내부 발행 인가: 디코딩 및 정합성 검증이 최종 완료된 해당 메시지 객체는 지체 없이 최종적으로 비행 제어기의 uORB 내부 트리의 버스 채널 공간에 직접 네이티브 발행(Publish) 적용된다. 이를 통해 비행 제어기의 핵심 코어인 메인 역학 제어 루프 컨드롤러나 모터 펄스 믹서(Mixer) 구동 모듈이, 마치 자체 내부 센서가 직접 데이터를 측위 생성한 것처럼 완벽한 환영 속에서 상위 ROS2 네트워크 망의 인공지능 제어 명령을 즉각적으로 전달받아 오차 없이 소비 집행하게 된다.

3. 코드 자동 생성(CodeGen) 기반의 정적 펌웨어 통합 무결성

마이크로 클라이언트 모듈의 하드웨어 타겟 특이성(Target-specific) 중 시스템 공학적으로 가장 본질적이고 중요한 설계 안전 요소는, 펌웨어 송수신 연산에 핵심 개입되는 모든 부피가 큰 직렬화 C++ 함수와 토픽 헤더 ID 레지스터 표가 런타임이 아닌 커널 컴파일 타임(Compile-time)에 빌드 시스템에 의해 기계적으로 일괄 자동 생성 주입된다는 점이다.

개발 시스템 아키텍트가 비행 제어기 코어 파일 트리의 .msg 메시지 명세 텍스트 파일을 추가하거나 수정 정의하고 상위 빌드 시스템 환경(CMake)을 가동 실행하면, fastrtpsgen 스크립트 파서(Parser) 툴 체인이 이 명세 파일들을 모조리 순회 전수 스캐닝하여 대상 클라이언트 노드용 C/C++ 소스 코드를 기계적으로 동적 렌더링(Rendering) 생성해 펌웨어 작업 디렉토리에 강제 삽입한다. 이러한 런타임 이전(Pre-runtime) 국면에서의 선제적 자동 코드 생성 및 주입 방식은 펌웨어 작동 중의 동적 메모리 할당(Dynamic Memory Allocation) 행위를 시스템적으로 원천 차단 배제해야 하는 RTOS 환경 특유의 극심한 펌웨어 메모리 안정성 엄격성을 강력히 지켜 보장해 주며, 런타임 문자열 파싱 시도에 의한 치명적인 메모리 누수 및 단편화(Fragmentation) 장애 결함이나 변수 타입 캐스팅(Type Casting) 오버헤드 연산 부하를 시스템 엔지니어링 단위에서 아예 원천 차단 박멸하는 가장 결정적인 공학적 방어 수단으로 작용한다.
