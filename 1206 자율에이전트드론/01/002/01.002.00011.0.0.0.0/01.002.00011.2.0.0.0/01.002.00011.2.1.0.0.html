11.2.1 클라이언트-에이전트 비대칭 구조 모델 분석

Micro RTPS 브릿지의 근간을 이루는 클라이언트-에이전트(Client-Agent) 아키텍처는 전형적인 글로벌 통신 미들웨어가 지향하는 대칭적(Symmetric) 피어-투-피어(Peer-to-Peer) 모델을 의도적으로 탈피하여 철저한 기능적 비대칭성(Asymmetry)을 기반으로 설계되었다. 이러한 비대칭 구조는 연산 자원과 물리적 메모리가 극도로 제약된 마이크로컨트롤러(MCU)와, 상대적으로 풍부한 범용 연산 능력을 갖춘 동반 컴퓨터(Companion Computer) 간의 물리적 체급 차이를 소프트웨어 공학적 관점에서 철저히 수용 타협한 필연적인 결과물이다.

1. 연산 자원 불균형(Resource Constraint) 기반 역할 분배

ROS2가 채택한 DDS(Data Distribution Service) 표준 네트워크 규격은 본래 리눅스(Linux) 환경과 같은 컴퓨팅 성능이 보장된 범용 운영체제(OS)를 구동하는 기기 간의 고성능 텐서 통신을 우선 가정한다. 어떠한 DDS 노드가 전역 네트워크에 편입되어 정상적으로 동작하기 위해서는 엔드포인트 동적 발견(Endpoint Discovery), 참여자 생존 하트비트(Heartbeat) 모니터링, 메시지 전송 이력(History) 버퍼 캐싱, 그리고 동적 서비스 품질(QoS) 상태 협상 매핑과 같은 무거운 백그라운드 관리 연산이 상시 수반되어야 한다.

만일 물리적 비행 제어기 펌웨어 코어 내부에 완전한 형태의 대칭적 DDS 인터페이스 통신 규약을 그대로 구동할 경우, 위와 같은 무거운 미들웨어 백그라운드 관리 로직이 마이크로컨트롤러의 주성능 메모리와 CPU 연산 주기를 심각하게 과도 점유하게 된다. 이는 기체 자세 제어 및 모터 PWM 구동 신호 동기화와 같은 수 마이크로초(\mus) 단위의 실시간 기한(Hard Deadline)이 엄격하게 요구되는 치명적 미션 프로세스의 타이밍 지터(Jitter) 혹은 스레드 블로킹(Thread Blocking)을 유발하여 치명적인 물리적 추락을 야기할 수 있다. 따라서 Micro RTPS 시스템은 비행 제어기 측의 클라이언트를 가장 무기력하고(Stateless) 가벼운 백엔드 형태로 영속 유지하는 비대칭 역할 분배 오프로딩 모델을 채택 도입하였다.

2. 무상태(Stateless) 클라이언트와 상태유지(Stateful) 에이전트 결합

논리적 구조적 비대칭성의 가장 근본적인 핵심 차이점은 네트워크 계층의 상태 유지 여부(Statefulness) 척도에 수렴한다.

  • FCU 클라이언트(상태 비저장): 클라이언트 펌웨어는 외부의 다른 네트워크 노드가 현재 망에 참여 존재해 있는지, 혹은 방금 전송 송출된 데이터 패킷이 목적지에 단 하나라도 정상 도달했는지를 스스로 능동 추적하거나 큐 상태를 관리하지 않는 철저한 무상태(Stateless) 직렬 엔진이다. 단순히 설정된 타임 주기에 맞추어 인터럽트로 uORB 버퍼의 가용 데이터를 읽어 들여 C/C++ 바이트 배열로 직렬화(Serialization)한 후 지정 매핑된 물리적 직렬 포트(UART)로 맹목적으로 밀어내거나(Push), 반대로 포트에서 수신 폴링한 바이트를 단지 역직렬화 파싱하여 uORB 트리 버스에 기계적으로 통보 발행하는 1차원적 선형 I/O 연산만을 가장 신속히 무한 반복한다.
  • CC 에이전트(상태 유지): 동반 컴퓨터 백그라운드 계층에 위치한 에이전트 데몬은 상위의 무상태 클라이언트가 발생시킨 이 맹목적인 기계적 바이트 스트림 파편을 즉시 인계받아 전역 분산망의 완전한 논리를 강력히 부여하는 상태 유지(Stateful) 통합 브로커이다. 에이전트는 ROS2 전역 네트워크 단위망을 향해 자신을 정식 DDS 참여자(Participant)로서 그 존재 위상을 방송 개방하고, 외부 구독자 노드와의 데이터 핸드셰이크(Handshake) 협상, 누락된 손실 데이터의 재전송 큐(Queue) 요청 처리 응답 등 RTPS 표준 규약이 요구 명시하는 대단히 복잡 다단한 상태 천이 모델(State Machine) 연산을 엣지 컴퓨터 자원을 소모하여 전적으로 방어 대행한다.

3. 토폴로지 위상 변환: 단일 직렬망에서 광역 P2P 분산망으로의 계층 확장

이 클라이언트-에이전트 결합 비대칭 모델 아키텍처는 전체 물리 시스템 토폴로지(Topology) 관점에서 종단 직렬 통신의 한계선을 초월 극복하는 지수적 변곡점 역할을 훌륭히 수행한다. 물리 보드인 비행 제어기와 에이전트 호스트 브릿지 데몬 간의 직접적인 물리 하드웨어 통신 공간은 철저하게 일대일(Point-to-Point) 닫힌 단일 폐쇄 직렬 파이프라인으로 구성 존재하지만, 해당 데이터가 브릿지 큐 버퍼 레이어를 넘어서는 바로 그 결정적 찰나의 순간 전역적인 다-대-다(N-to-N) P2P 분산 유향 그래프 데이터 망으로 그 구조적 통신 위상 가치가 다이나믹 증폭 확장된다.

즉, 이러한 의도된 비대칭 공학 설계는 자원적으로 가장 취약한 최하위 임베디드 오프보드 송수신 구간을 구조 프로토콜의 극한 경량화로 안전하게 방어 수행하면서도, 데이터가 무사히 도달한 ROS2 최상단 네이티브(Native) 메모리 공간에서는 어떠한 동시성 트래픽 제약 없이 수십 개의 다중 엣지 딥러닝 컴포넌트나 비선형 궤적 생성기 멀티 모듈들이 비행 제어기의 데이터 소스를 각자의 세부 QoS 방어 정책에 맞게 평등히 자유롭게 점유 통제하고 안정적으로 다중 구독할 수 있도록 돕는 빛나는 아키텍처적 완성도를 역사적으로 구현해 낸 것이다.
