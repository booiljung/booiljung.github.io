11.2 Micro RTPS 브릿지 시스템 아키텍처 구조

전체적인 Micro RTPS 브릿지 시스템의 아키텍처 파이프라인은 비행 제어기(FCU) 측의 내장 펌웨어 최하단에 위치 직접 컴파일되는 ‘마이크로 클라이언트(Micro Client)’ 모듈과, 방대한 연산 자원이 풍부하게 확보된 엣지 동반 컴퓨터(Companion Computer) 측의 범용 운영체제(OS) 백그라운드 상에서 구동되는 ‘호스트 에이전트(Host Agent)’ 데몬이라는 두 개의 대칭적인 논리 계층 모듈로 엄격하게 이원화되어 설계되었다. 이러한 논리적-물리적 이원화 구조는 가혹하고 엄격한 실시간성(Real-time) 분산 구동 제약 하에 동작 작동하는 임베디드 코어 환경과 동적인 비결정론적 네트워크망을 상시 운용하는 상위 미들웨어 인프라 간의 물리적 통신 장벽을 해소 연동하기 위한 가장 핵심적인 프레임워크 단계를 구성한다.

1. FCU 펌웨어 임베디드 계층: Micro RTPS 클라이언트(Client)

물리적 비행 제어기의 코어 실시간 운영체제(예: NuttX OS) 커널 내부에서 직접 링크 구동되는 클라이언트(micrortps_client) 컴포넌트는 하드웨어 인터럽트 스레드 레벨에서 계측 획득된 원시 센서 페이로드 데이터와 비행 상태 역학 정보를 시스템 외부 채널로 안전하고 신속하게 추출 포워딩하는 최전방 엔드포인트(Endpoint) 게이트웨이이다.

  • uORB 네이티브 인터페이스 직접 결합: 클라이언트 코드는 비행 제어기 펌웨어 내부의 객체 지향적 고속 메시징 버스인 uORB(Micro Object Request Broker) 트리 공간과 다이렉트로 상호작용한다. 물리 버스 송신 시에는 사전에 결정 정의된 특정 uORB 센서 텔레메트리 토픽들을 직접 구독(Subscribe)하고, 채널 수신 시에는 외부 엣지단에서 유입된 비전 기반 제어 명령 토픽 구조를 메모리에 언팩 식별하여 타겟 uORB 버스 상에 무결하게 즉각 발행(Publish)한다.
  • 초경량 무상태 직렬화(Serialization) 고속 엔진: 하드웨어에 종속된 클라이언트 펌웨어는 고성능 범용 DDS 시스템이 광범위하게 필연 포함하는 무거운 네트워크 세션 유지 관리 로직이나 부가적인 프로토콜 헤더 트래킹 매핑 체계를 제한된 내장형 RAM 메모리에 일절 포함 잔류시키지 않는다. 대신 순수 C/C++ 기반의 초경량 직렬화 전담 라이브러리인 MicroCDR 모듈만을 코어에 탑재 결합하여, 내부 메모리 상의 객체 데이터 구조체를 최적 평탄화된 플랫(Flat) 바이트 배열(Byte Array) 스트림으로 직렬화/역직렬화 연속 압축 변환하는 선형 연산만을 최소 최단 시간 지연 바운드 내에 기계적으로 반복 수행한다.

2. 컴퓨팅 동반 컴퓨터 계층: Micro RTPS 에이전트(Agent)

로보틱스 미들웨어 글로벌 환경이 고도 구축된 동반 컴퓨터 생태계 내부에서 백그라운드 리스너 데몬으로 상시 구동 영속되는 에이전트(micrortps_agent) 런타임 모듈은, 거친 물리 통신 링크 포트에서 수신 폴링(Polling)한 원시 파편화 바이트 스트림을 역조립하여 정식 글로벌 분산 네트워크상의 정합 자산으로 안전하게 격상 관문 역할을 수행하는 핵심 통합 브로커이다.

  • Fast RTPS 라이브러리 스택 연동 망: 해당 에이전트는 미들웨어 핵심 표준인 eProsima의 Fast RTPS(현 Fast DDS) 통신 라이브러리 프레임워크 스택 위층에서 인스턴스화되어 구현되어 있다. 브로커는 물리적인 저수준 직렬 텔레메트리 링크(UART 등)를 통해 파편화 지연되어 도달한 바이트 페이로드(Payload) 시그널을 신속하게 포착 버퍼링하고, 이를 내부 알고리즘으로 정밀하게 구조 디코딩하여 ROS2 워크스페이스 호환 RTPS 글로벌 네트워크 규격 패킷 덩어리로 무결점 재조립 묶음 처리한다.
  • DDS 글로벌 데이터 스페이스로의 전역 라우팅: 시스템 메모리 상 재조립 정합이 검증 완료된 패킷 데이터 구조체는 ROS2 워크스페이스 런타임 내에서 rmw_fastrtps 규격 인터페이스를 투명하게 거쳐 망내 글로벌 토픽 공간(Global Topic Space) 전역으로 최종 즉시 퍼블리시 확산된다. 완벽히 반대 전개 방향의 역방향 제어 명령 흐름에 있어서도, 에이전트는 ROS2 글로벌 망 내의 비행 궤적 제어 메시지를 청취 구독하여 상층 패킷을 디코딩 한 후 하부 클라이언트를 향해 직렬 패킷 바이트 묶음으로 압축 역송출 브로드캐스팅 포워딩한다.

3. 직교적 메타 구조적 매핑(Mapping)과 공통 사전(Common Dictionary) 체제

이원화된 시스템의 두 양극단 엔드포인트 계층이 비결정론적 외란 속에서 데이터 구조의 손실 유실이나 위상 왜곡을 전혀 허용 범람하지 않고 쌍방향 초고속 통신 루프를 안정적으로 성립 보장시키기 위해서는, 전체 시스템 컴파일 링킹 타임에 물리 메모리적으로 확정 고정된 ‘공통 데이터 구조 사전(Common Dictionary)’ 룰베이스에 어긋남 없이 철저히 의존 종속되어야만 한다.

아키텍처의 프레임워크 설계 구조상, 최하단 비행 제어 펌웨어 내부에 존치된 추상 .msg 메타데이터 명세 파일과 에이전트 컴포넌트를 상단 구동 렌더링하는 환경의 최상위 ROS2 IDL(Interface Definition Language) 파일 소스 포맷 체계는 그 본질적 형태 구조적으로 반드시 단일성(Singularity) 보존을 전제 유지한다. 전체 브릿지 스크립트 빌드 소요 시점에 fastrtpsgen과 통합된 템플릿 코드 자동 전사 생성기(Code Generator) 도구가 직접 시스템에 개입 침투하여 클라이언트와 에이전트 양측 타겟 보드 디렉토리에 완벽히 수학적으로 동일한 메모리 정합성(Memory Alignment) 패딩을 갖춘 C/C++ 직렬화/역직렬화 전용 헤더 코드 체계를 강제 주입 삽입한다.

결과적으로, 이 완성된 견고한 아키텍처망은 하위 비행 제어 펌웨어 코드가 외부 동반 로컬 컴퓨터 네트워크의 주소 논리를 전혀 직접 인지 추적 관리할 필요가 없고, 상위의 ROS2 지능 제어 노드 또한 하위 임베디드 MCU의 내부 원시 메모리 데이터 버스 물리 사양 주소를 알 필요가 구조상 전혀 없는 완벽한 상호 블랙박스(Black-box) 지향적 독립 추상화(Abstraction) 분리 구조를 학술적으로 최종 모델링 완성한다.
