11.5.2 직렬 버스 대역폭 최적화를 위한 메시지 필터링 및 주파수 제어

1. 이기종 브리지 환경에서의 대역폭 포화(Saturation) 한계

마이크로 RTPS(Micro RTPS) 브리지는 그 아키텍처적 특성 상, 단일 보드 마이크로컨트롤러(MCU) 기반의 비행 제어기와 고성능 오프보드(Offboard) 리눅스 컴퓨터를 물리적인 직렬(Serial/UART) 버스로 연결하는 사례가 빈번하다. 일반적인 UART 통신의 보드레이트(Baudrate)는 921600 bps(실효 대역폭 약 92 KB/s) 내외로 설정되며, 이는 초당 수 메가바이트(MB) 데이터를 넘나드는 PX4 내부의 전체 uORB(Micro Object Request Broker) 트래픽을 처리하기에 절대적으로 부족한 물리적 한계 대역폭이다. 만일 비행 제어기(FCU) 펌웨어 계층에서 생성되는 모든 고빈도 센서 데이터와 상태 추정(State Estimation) 메시지를 여과 없이 브리지로 덤프(Dump)할 경우, 물리 계층 송신 버퍼의 역방향 압력(Backpressure)과 오버플로우(Overflow)가 필연적으로 발생한다. 종국에는 브리지 통신 스레드의 블로킹이 발생하여 시스템 전체의 통제 불능(Lock-up)을 초래할 수 있다. 따라서 마이크로 RTPS 브리지 설계에 있어 정적 메시지 필터링(Message Filtering)과 정밀한 주파수 제어(Frequency Control)는 시스템 아키텍처의 생존을 결정짓는 필수적인 망 최적화 기제이다.

2. YAML 기반 선택적 메시지 필터링 아키텍처

대역폭 포화를 방지하기 위한 1차 방어선은 브리지를 통과할 수 있는 uORB 토픽만을 화이트리스트(Whitelist) 방식으로 선별하는 정적 통제 아키텍처이다.

  1. 송수신 방향성 기반 필터 매니페스트: 마이크로 RTPS 브리지는 소스 코드 컴파일 및 런타임 초기화 시점에 직렬화 명세서(.yaml 설정 파일)를 참조하여 브리징의 대상을 엄격하게 제한한다. 이 설정 구조체에는 에이전트로 송신(Publish)될 토픽과 네트워크로부터 수신(Subscribe)받을 토픽의 리스트 구분자와 타입 명세가 하드코딩된다.
  2. 어플리케이션 계층에서의 원천 차단: 브리지 클라이언트는 매니페스트에 등록되지 않은 임의의 토픽에 대해서는 uORB 구독 요청 자체를 수행하지 않으며, 메모리 복사 및 매핑 연산을 선제적으로 생략한다. 자율 비행 역학 루프에 직접 관여하지 않는 디버깅용 파라미터나 고밀도 원시 로깅(Raw Logging) 페이로드는 직렬 버스에 진입하기 전단계인 uORB 층위에서 차단(Drop)된다. 이를 통해 UART 구간 대역폭의 불필요한 고갈을 방지함과 동시에 FCU 칩셋의 CPU 사이클 낭비를 억제한다.

3. 서브샘플링(Subsampling) 및 주파수 제어 로직

특정 토픽이 필터링 계층을 통과하여 브리징 대상으로 선정되었다 할지라도, 대상 센서의 물리적 갱신 빈도(Update Rate)가 직렬 대역폭을 여전히 초과하는 병목이 발생할 수 있다. 이를 제어하기 위해 2차 방어선인 동적 주파수 제어 기법이 동원된다.

  1. 구독 간격 하향 조절(Downsampling): PX4 커넥터 체계의 uORB API는 특정 구독자(Subscriber)가 특정 토픽에 대해 수신을 원하는 최소 업데이트 간격(Interval)을 밀리초(ms) 단위로 제한할 수 있는 인터페이스를 제공한다. 마이크로 RTPS 클라이언트는 이 API 구조를 활용하여, 내부적으로 초당 250Hz 이상의 주기로 발행되는 고속 SensorCombined 토픽을 브리지 파이프라인 상으로는 50Hz 수준으로만 폴링(Polling)되도록 하위 계층에서 원천적으로 서브샘플링한다.
  2. 최신 데이터 페치(Fetch)와 덮어쓰기 타당성: 하향 조절된 인터벌 주기로 클라이언트가 큐에서 데이터를 페치할 때, 통과하지 못한 과거의 중간 데이터는 덮어씌워지며, 가장 최신의 상태 시퀀스(Last Sequence) 페이로드만이 직렬화 연산을 통과해 UART 버스에 배차된다. 이는 QoS 정책에서의 Best-Effort 모델과 결합하여 송신 주파수는 낮추되 시간적 실시간 참값(Ground Truth)의 무결성을 보장하는 거시적 최적화 효과를 유도한다.

4. 대역폭 최적화의 제어 공학적 의의

이러한 메시지 필터링 및 서브샘플링 파이프라인의 구조화는 인터페이스 하드웨어 리소스의 극단적인 제약 조건 하에서 통신의 결정론성(Determinism)을 확보하기 위한 핵심 엔지니어링 설계이다. 전체 직렬 버스 대역폭의 70% 이하를 적정 부하(Optimal Load) 한계선으로 유지함으로써, 저 우선순위의 텔레메트리 큐잉이 최상위 우선순위를 갖는 치명적 제어 명령(Vehicle Command)의 전파 시간을 지연시키는 전송 전이 역전(Priority Inversion) 현상을 상쇄할 수 있다. 학술적으로도 이 아키텍처는 클라우드 기반 연동 제어나 다중 에이전트 VLA(Vision-Language-Action) 모델이 발산하는 거대 지능 트래픽이, 하위 비행 물리 계층의 실시간 안전 주파수 대역(Safety Envelope)을 침식하지 못하도록 방어하는 격벽(Bulkhead) 통신망으로서의 중대한 가치를 지닌다.
