11.5.1 Best-Effort 및 Reliable 전송 모델의 로직 차이 분석

1. 퍼블리시-서브스크라이브 모델의 투트랙(Two-Track) 전송 전략

자율 에이전트 드론의 시스템 아키텍처에서 마이크로 RTPS(Micro RTPS) 브리지는 ROS2 기반의 고성능 컴패니언 컴퓨터 노드와 단일 보드 마이크로컨트롤러(MCU) 기반의 비행 제어기(FCU) 소스 간의 통신 결합을 매개한다. 이 간극을 넘나드는 데이터 트래픽의 스펙트럼은 크게 고빈도로 갱신되는 센서 텔레메트리(Telemetry)와 간헐적이고 시스템 치명적인 제어 명령으로 대별된다. 이질적인 두 요구사항을 동시에 충족시키기 위해, RTPS 규약은 데이터 배포 서비스(DDS) 표준에 입각하여 Best-Effort(최선 노력) 모델과 Reliable(신뢰성 보장) 모델이라는 투트랙 전송 정책을 제공하며, 각 모델은 프로토콜의 상태 머신 설계 및 큐잉(Queueing) 계층에서 극명한 처리 로직의 차이를 나타낸다.

2. Best-Effort 전송 모델의 비동기적 발송 및 폐기 로직

Best-Effort 전송 모델의 근본적인 목적은 네트워크 통신 오버헤드를 물리적 한계점까지 극소화하고 메시지 파이프라인의 이송 지연 지터(Jitter)를 억제하는 데 있다.

  1. 단방향 Fire-and-Forget 메커니즘: 퍼블리셔(Publisher) 에이전트는 직렬화된 데이터를 브리지 캐시에서 물리 전송 매체(Serial/UDP)로 밀어낸 직후, 해당 데이터 패킷의 생명주기 통제를 즉각 종료한다. 수신 노드로부터의 패킷 확인 응답(Acknowledge; ACK)을 기다리거나 검증하지 않으므로, 네트워크 상의 양방향 패킷 왕복 타임(Round Trip Time)이 통신 대역폭을 점유하는 현상이 원천적으로 차단된다.
  2. 논리적 재전송 큐의 부재: 노이즈 개입, 패킷 분할(Fragmentation)에 의한 훼손, 체크섬 불일치 등으로 데이터가 수신측에서 유실되더라도, 브리지 프로토콜 스택은 이를 감지할 규약적 책임이 존재하지 않으며 어떠한 재전송 시도도 하지 않는다.
  3. 학술적 적용 타당성: 자율 드론의 IMU(관성 측정 장치), 광학 흐름(Optical Flow 센서), EKF(확장 칼만 필터) 기반의 오도메트리(Odometry) 등은 초당 수백 헤르츠(Hz) 이상의 매우 촘촘한 주기로 갱신된다. 만일 1회의 패킷 유실을 복원하기 위해 수 밀리초(ms)가 구역 내에서 지연(Delay)될 경우, 수신측 제어기는 시간적 참값(Ground Truth)의 유효성을 잃은 낡은 데이터를 주입받게 되며, 이는 비행 상태 추정(State Estimation) 발산을 초래할 수 있다. 따라서 시계열적 연속성이 무결성보다 우선시되는 텔레메트리 스트림은 철저히 비동기적 Best-Effort 모델로 구조화되어야 한다.

3. Reliable 전송 모델의 피드백 제어 및 윈도우 추적 로직

Reliable 전송 모델은 전자기적 간섭이나 일시적인 네트워크 병목 상황에서도 메시지의 100% 논리적 도달을 보장해야 하는 결정론적(Deterministic) 제어 모델을 지향한다.

  1. 상태 기반 ACKNACK 핸드쉐이크 프로토콜: 퍼블리셔는 메시지를 발송할 때마다 내부 시리얼라이즈 버퍼퍼(Serialize Buffer)에 패킷의 고유 시퀀스와 복제본을 보류(Hold)시킨다. 수신 노드는 데이터를 파싱한 후 ACK 제어 패킷을 반환하며, 패킷 시퀀스 상의 이빨 빠짐(Gap)을 파악할 경우 NACK(Negative Acknowledge) 신호에 결락된 시퀀스 메타데이터를 담아 명시적인 핀포인트 재전송을 요구한다.
  2. 재전송 슬라이딩 윈도우(Sliding Window) 큐잉: 퍼블리셔 통신 스레드는 구독자 노드로부터 최종 수신 증명(ACK)을 수령할 때까지 송신 버퍼 메모리의 블록(Block)을 해제하지 않고 잠금(Lock) 상태로 유지한다. 마이크로 RTPS 브리지가 감당할 수 있는 송신 큐의 페이로드는 제한적이므로, 미확인(Unacknowledged) 메시지 총량이 윈도우 임계치를 초과할 경우, 응용 계층의 publish() 시스템 콜 자체가 병목 모드(Blocking View)로 전환되거나 버퍼 오버플로우 예외(Exception)를 반환하는 구조를 취한다.
  3. 학술적 적용 타당성: 기체 암(Arm) 상태의 변경, 비행 모드 토글(예: 수동 위치 제어에서 오프보드 자율 제어로의 전환), 정밀 웨이포인트(Waypoint) 좌표 배열 업로드 등과 같은 토픽 데이터는 단 한 프레임의 유실조차 자율 에이전트의 제어 불능(Runaway)이나 추락을 야기하는 치명적(Critical) 결함으로 직결된다. 이러한 데이터 세트는 도달 시간이 지연되어 제어 루프 레이턴시 측면에서 다소 패널티를 안게 되더라도, 프로토콜 계층이 반드시 자가 복구를 수행하여 비행의 임무 무결성을 관철시키는 Reliable 모델로 분리 매핑되어 전송된다.

4. 논리적 추상화의 분리와 시스템 아키텍처적 결론

마이크로 RTPS 브리지 소스코드 레벨에서 Best-Effort와 Reliable 처리를 전담하는 송수신 시스템 스레드(Thread)는 물리적으로 동일한 UART 채널이나 UDP 소켓 포트를 공유함으로써 병렬로 멀티플렉싱(Multiplexing)되지만, 논리적인 큐 스케줄러 계층에서는 서로 간의 간섭을 차단하는 모듈형 박스로 분리 구동된다. 이러한 데이터 품질 요건 기반의 로직 차별화 아키텍처는, “파이프라인 지연 최소화(역학적 안정성 수호)“와 “명령 트랜잭션의 완벽한 보장(논리적 시스템 통제성 확립)“이라는 근원적으로 상충하는 두 소프트웨어 공학적 딜레마를 ROS2라는 광범위한 통신 구조체 내에서 타협 없이 공존시키는 기반 학술 모델로 평가된다.
