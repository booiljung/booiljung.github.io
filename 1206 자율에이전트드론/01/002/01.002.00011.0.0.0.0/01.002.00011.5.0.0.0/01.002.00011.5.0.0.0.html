11.5 QoS 통신 정책 적용 및 구조화

1. RTPS 기반 서비스 품질(QoS) 개요

마이크로 RTPS(Micro RTPS) 브리지 아키텍처 모델에서 데이터의 무결성과 실시간성을 보장하는 핵심 기제는 서비스 품질(Quality of Service, QoS) 정책이다. ROS2의 기저를 이루는 DDS(Data Distribution Service) 미들웨어는 노드 간의 데이터 교환 방식과 신뢰성 요구 수준을 세밀하게 제어하기 위해 다양한 QoS 프로파일을 제공한다. PX4의 내부 메시징 버스인 uORB(Micro Object Request Broker)는 근본적으로 단일 통제 메모리 공간 내에서의 동기식/비동기식 출판-구독(Publish-Subscribe) 모델을 사용하므로, 프로토콜 자체적으로는 네트워크 통신 지연이나 패킷 유실을 깊이 고려하지 않는다. 그러나 마이크로 RTPS 브리지를 통해 uORB 데이터가 통신 매체를 타고 외부의 분산 네트워크 층위(ROS2 생태계)로 전송될 때, 이러한 비결정적(Non-deterministic) 통신 환경을 제어하고 예측 가능성을 확보하기 위해 RTPS 표준에 입각한 통신 정책의 구조화가 필수적으로 요구된다.

2. 드론 시스템의 핵심 QoS 파라미터 구조화

자율 드론 시스템의 비행 동역학적 특성과 임무 요구사항의 치명도에 따라, 마이크로 RTPS 브리지는 다음과 같은 핵심 QoS 파라미터들을 매핑 및 구조화하여 적용한다.

2.1 신뢰성(Reliability) 정책

신뢰성 정책은 네트워크 상에서 데이터 전송의 성공 보장 및 유실 시의 처리 방식을 정의한다.

  • Best Effort(최선 노력): 패킷 유실을 허용하며 전송 계층에서의 재전송 매커니즘을 작동시키지 않는다. IMU(Inertial Measurement Unit) 측정값이나 GPS 좌표와 같이 높은 주파수 대역으로 갱신(Update)되는 고주기 센서 스트림에 주로 적용된다. 이는 과거 데이터의 유실 복구를 위해 파이프라인 지연을 초래하기보다는 최신 데이터를 즉각 수신하는 역학 제어적 이점이 크기 때문이다.
  • Reliable(신뢰성 보장): 패킷이 유실될 경우 RTPS 계층에서 확인 응답(ACK/NACK)을 교환하여 재전송을 수행, 반드시 도달함을 보장한다. 드론의 비행 모드 전환 명령(Vehicle Command)이나 웨이포인트(Waypoint) 미션 업로드와 같은 일회성 제어 메시지 전송에 주로 구조화된다.

2.2 이력(History) 및 내구성(Durability) 정책

이 두 정책은 노드가 네트워크에 늦게 참여(Late Joiner)했을 때, 브리지가 과거 발행된 데이터를 어떻게 보관하고 처리할 것인지 규정한다.

  • History(Keep Last vs. Keep All): 보관할 누적 메시지의 개수를 정의한다. 수신 노드의 큐(Queue) 메모리 오버플로우를 제한하기 위해 고속 드론 상태 데이터는 대개 ‘Keep Last’ 속성과 얕은 큐 사이즈(예: 1 또는 5)로 구조화된다.
  • Durability(Volatile vs. Transient Local): 퍼블리셔가 메시지를 발행한 후, 해당 메시지의 물리적 생명주기를 정의한다. 일반적인 센서 데이터는 ‘Volatile’(퍼블리셔 또는 브리지 메모리에서 휘발성으로 취급)로 설정되며, 시스템 파라미터 설정값과 같이 변동이 적고 영구적인 유지보수가 필요한 데이터는 늦게 연결된 서브스크라이버도 조회할 수 있도록 ‘Transient Local’ 상태로 유지 구조화된다.

2.3 타이밍 기반(Timing-based) QoS 제어

  • Deadline(마감 시간): 퍼블리셔 시스템이 특정 시간 주기 이내에 반드시 갱신된 데이터를 발행할 것을 논리적으로 보장하는 정책이다. 수신측(ROS2 노드)은 해당 주기 내에 기체 제어 상태 메시지(예: VehicleLocalPosition)가 브리지를 통과해 도착하지 않을 경우, 알람 이벤트를 발생시켜 페일세이프(Fail-Safe) 안전 알고리즘을 즉각 트리거링(Triggering)할 수 있다.
  • Liveliness(생존성): 시스템에 참여하는 브리지 에이전트와 서브스크라이버 노드 간의 상호 연결 무결성이 정상적으로 유지되고 있는가를 구조적으로 감시하고 심장박동(Heartbeat) 신호로 증명한다.

3. uORB-RTPS 간 QoS 매핑 아키텍처

마이크로 RTPS 브리지는 에이전트(Agent)와 클라이언트(Client) 간 통신을 수립할 때, *.msg IDL(Interface Definition Language) 명세서 분석 결과와 구조화된 설정 파일을 기준으로 uORB 토픽별 맞춤형 QoS 프로파일을 매핑한다. 런타임 시작 시 에이전트 인스턴스는 이 매핑 규칙을 파싱하여, ROS2 영역의 DDS 데이터 리더/라이터(Data Reader/Writer) 엔티티(Entity)를 생성할 때 해당 QoS 프로파일 인자를 정밀하게 인젝션(Injection)한다.

예를 들어, SensorCombined uORB 토픽 스트림이 브리지를 관통할 때, 클라이언트는 직렬 버스상에 페이로드를 지연 없이 밀어내며, ROS2 측의 마이크로 RTPS 에이전트는 DDS 도메인에 해당 토픽을 출판할 때 ’Best Effort’와 ‘Volatile’ QoS 조합을 선언하여 네트워크 오버헤드를 구조적으로 최소화한다. 반면, 오프보드(Offboard) 노드에서 드론으로 전달되는 VehicleCommand 메시지는 재전송 큐를 활성화하는 ‘Reliable’ 모드로 매핑·브리징되어, 제어 신호 패킷 유실로 인해 발생할 수 있는 드론의 치명적인 제어상실(Loss of Control) 현상을 아키텍처 관점에서 절대적으로 차단한다.

4. QoS 정책 구조화의 학술적 의의

이러한 세밀한 QoS 통신 정책의 입체적 구조화는, 비행 제어기(FCU)라는 단일 보드 마이크로컨트롤러(MCU) 상의 수치 해석적 결정론적(Deterministic) 제어 루프와 다중 리눅스 프로세스가 혼재하는 비결정적 컴패니언 컴퓨터(Companion Computer) 간의 시간적·공간적 추상화 결합도를 조율하는 핵심 메커니즘으로 자리매김한다. 마이크로 RTPS 브리지 내부에서의 철저한 QoS 정책 적용 및 튜닝은 불필요한 네트워크 대역폭 병목을 억제하고, 프로토콜 지연 시간(Latency)의 상한계(Upper Bound) 산출에 대한 수리적인 기초 자료를 제공함으로써 자율 에이전트 드론 연구의 학술적 제어 신뢰성을 공고히 보장한다.
