11.6.3 시리얼 버스 평문 통신의 보안 취약점 식별

1. 아키텍처 내 보안 인식 부재와 평문 전송의 특성

자율 에이전트 드론의 내부 통신망 구조에서, 과도기적 브리지 역할을 수행한 마이크로 RTPS(Micro RTPS) 아키텍처는 제한된 컴퓨팅 자원 내에서 처리량(Throughput) 확보와 지연 시간(Latency) 최소화에 모든 공학적 설계 초점이 맞추어져 발전해 왔다. 이러한 성능 지향적 아키텍처 모델 철학은 필연적으로 통신 스트림의 암호화(Encryption), 송수신 물리 노드의 상호 인증(Authentication), 그리고 데이터 페이로드 위변조 방지를 위한 암호학적 캡슐화와 같은 소프트웨어적 보안 오버헤드(Security Overhead)를 철저히 배제하는 결과를 낳았다.

결과적으로 비행 제어기(FCU)의 엣지 칩셋과 고성능 컴패니언 컴퓨터(Companion Computer) 간을 직결하는 UART(Universal Asynchronous Receiver-Transmitter) 기반의 직렬 버스는, 복잡한 ROS2 DDS 토픽 데이터를 0과 1비트의 육안 및 파서 식별이 명확하게 가능한 바이트 배열 형태로 전송하는 완전한 평문 통신(Plaintext Communication, Cleartext) 체계로 운용된다.

2. 물리적 계위에서의 해킹 공격 벡터(Attack Vector) 식별

마이크로 RTPS 평문 통신 아키텍처는 드론의 전장(Avionics) 하드웨어 토폴로지 상, 악의적인 공격자가 물리적 회선에 직접 접근이 가능한 시나리오에서 시스템 근간을 뒤흔드는 치명적인 보안 결함을 노출한다.

2.1 패킷 스니핑(Packet Sniffing)을 통한 기밀 텔레메트리 유출

드론 내부의 직렬 케이블 베어본(TX/RX 핀) 라인에 공격자가 초소형 무선 브리지 회로나 고속 로직 애널라이저(Logic Analyzer) 프로브를 병렬 분기(Tap/Splice) 연결할 경우, 브리지 계층을 넘나드는 모든 uORB 및 DDS 페이로드를 일말의 해독 브루트포싱(Bruteforcing) 과정 없이 즉각적으로 스니핑(Sniffing)하여 복제할 수 있다. 이는 기체가 취득한 고정밀 RTK-GPS 좌표 궤적, 지형 매핑 슬램(SLAM) 옥트리 데이터, 타겟 인식 비전 센서의 위치 메타데이터 등 군사적·산업적 임무 기밀이 외부 망으로 고스란히 유출되는 1차원적인 기밀성(Confidentiality) 침해 사고를 야기한다.

2.2 데이터 인젝션(Injection) 및 제어권 탈취(Spoofing) 공격 역학

보다 심각한 시스템적 위협은 수동적인 스니핑을 넘어, 공격자가 평문 프레이밍 구조를 역설계하여 RX 라인으로 거짓 데이터를 강제 주입(Injection)하는 동적 스푸핑(Spoofing) 공격 방어 로직이 존재하지 않는다는 점이다. 패킷 동기화 헤더(Sync Header) 규격과 단순 다항식 CRC(순환 중복 검사) 알고리즘만 파악된다면, 공격자는 자신이 정상적인 ROS2 컴패니언 컴퓨터인 것처럼 완벽하게 위장하여 FCU 데몬 측으로 논리 조작된 VehicleCommand (예: 비행 중 강제 모터 정지, 암 해제) 또는 TrajectorySetpoint (자율 비행 경로 탈취 조작) 패킷을 밀어 넣을 수 있다.
비행 제어기 내부의 마이크로 RTPS 클라이언트 코어는 수신된 메시지의 디지털 서명이나 송신 주체의 정당성을 암호학적으로 교차 검증하는 인가(Authorization) 로직 자체가 아키텍처 상에 전무하므로, 이러한 주입식 조작 제어 패킷을 맹목적인 참값 제어 명령으로 신뢰하여 실행해 버리며, 이는 궁극적인 기체의 완전한 납치(Hijacking)나 지면 추락 사태를 초래한다.

3. 차세대 네트워크 아키텍처의 암호학적 당위성 정립

마이크로 RTPS 브리지는 단일 보드 마이크로컨트롤러 상에 고립되어 있던 펌웨어의 제어 지형을 ROS2 기반 거대 다중 에이전트 드론 영역으로 비약적으로 확장시키는 데 결정적인 공학적 기여를 달성하였다. 그러나 “외부망과 단절된 기체 내부의 오프보드 컴퓨터 및 케이블 시스템은 언제나 무결하게 신뢰할 수 있다“는 폐쇄망 우선주의적 가정을 기반으로 설계된 탓에, 물리적 하드웨어 침투 해킹 공격 벡터에는 항복 선언을 할 수밖에 극도로 취약한 소프트웨어 공학적 모델이라는 냉혹한 학술적 결론에 도달하였다.

이러한 내부 직렬 버스 모델링의 평문 통신이 지닌 태생적 취약점 식별은, 궁극적으로 보안 통신 계층 보호막 자체를 아키텍처의 부가물이 아닌 필수 근간으로 삼는 방향으로의 진화를 요구하였다. 이는 통신 프로토콜의 표준 보안 확장 규격인 DDS-Security 플러그인(엄격한 퍼블리셔/서브스크라이버 접근 제어 및 데이터 페이로드 암호화) 체계를 인프라 레벨에서 네이티브로 수용하고, 처리 오버헤드를 상쇄하기 위해 마이크로컨트롤러 코어의 하드웨어 암호화 가속 장치(Hardware Crypto Accelerator)를 능동적으로 활용하는 차세대 XRCE-DDS 및 SROS2(Secure ROS2) 연동 아키텍처로의 자율 통신망 전면 재확립이라는 획기적인 기술적 토대를 마련하는 뼈아픈 교훈이 되었다.
