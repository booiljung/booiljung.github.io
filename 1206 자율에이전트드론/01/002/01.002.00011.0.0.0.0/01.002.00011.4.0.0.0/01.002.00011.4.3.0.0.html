11.4.3 이기종 전송 매체별 패킷 프레이밍 및 CRC 무결성 검증

1. 이기종 매체 통신에서의 패킷 프레이밍 구조

마이크로 RTPS(Micro RTPS) 브리지 아키텍처는 직렬(Serial/UART) 포트와 UDP/IP 네트워크라는 이기종(Heterogeneous) 전송 매체를 모두 지원하도록 설계되었다. 두 전송 매체는 물리적 특성과 데이터 전송 방식이 상이하므로, RTPS(Real-Time Publish-Subscribe) 메시지를 안정적으로 교환하기 위해서는 각 매체의 특성에 부합하는 패킷 프레이밍(Packet Framing) 기법이 필수적으로 요구된다. 패킷 프레이밍은 연속적인 바이트 스트림(Byte Stream) 또는 네트워크 페이로드 상에서 개별 메시지의 시작과 끝을 식별하고, 수신 노드가 이를 성공적으로 추출할 수 있도록 데이터 구조를 캡슐화하는 과정을 의미한다.

2. 직렬 매체 기반 스트림 프레이밍 규격

UART와 같은 직렬 통신 매체는 데이터를 바이트 단위의 연속적인 스트림 형태로 전송한다. 직렬 버스 상에는 메시지의 경계를 나타내는 하드웨어적인 식별 매커니즘이 존재하지 않으므로, 소프트웨어 계층에서 논리적인 프레임 경계를 정의해야 한다. 마이크로 RTPS 브리지는 직렬 전송 시, 데이터 페이로드 스트림 내에서 개별 패킷을 분리하고 탐지하기 위한 정형화된 헤더 파싱 기법을 도입한다.

프레임 헤더에는 일반적으로 동기화 바이트(Sync Byte), 페이로드의 길이 정보, 그리고 토픽 식별을 위한 메타데이터가 포함된다. 수신측의 프레이밍 알고리즘은 버퍼에 누적되는 바이트 스트림에서 동기화 바이트를 탐색하고, 뒤따르는 길이 필드를 파싱하여 다음 패킷의 경계를 계산한다. 이 과정에서 물리적인 노이즈로 인해 중간 바이트가 유실되거나 변조될 경우 프레임 경계 정렬(Frame Alignment)이 붕괴될 위험이 존재하므로, 지속적인 바이트 검사 및 재동기화(Resynchronization) 매커니즘이 수반되어야 한다.

3. UDP/IP 기반 데이터그램 프레이밍

UDP(User Datagram Protocol)를 매체로 사용하는 경우, 전송 계층 프로토콜 자체가 데이터그램(Datagram) 단위의 메시지 전송을 보장한다. 이더넷 또는 Wi-Fi 기반 통신에서는 운영체제의 네트워크 스택이 프레임의 시작과 끝, 길이를 기반 계층에서 처리하여 응용 계층으로 전달한다. 직렬 통신에서 요구되던 스트림 내 동기화 바이트 탐색 구조는 근본적으로 불필요해지며, 마이크로 RTPS 브리지는 수신된 각 UDP 데이터그램의 페이로드 전체를 단일 마이크로 RTPS 메시지로 매핑한다.

하위 네트워크 스택에 의한 IP 단편화(Fragmentation)가 발생할 수 있으나, 일반적으로 마이크로 RTPS 메시지의 크기가 네트워크의 최대 전송 단위(MTU; Maximum Transmission Unit) 이하를 유지하므로 단일 프레임 내에 캡슐화된다. UDP 기반 프레이밍은 운영체제의 소켓 버퍼 수신 단위와 브리지 시스템의 응용 메시지 단위를 일치시킬 수 있어 프로토콜 처리 효율성이 향상된다.

4. CRC 연산 기반 데이터 무결성 검증 아키텍처

이기종 전송 매체를 관통하는 데이터의 논리적 무결성(Integrity)을 보장하기 위해 프레이밍 페이로드의 후미에는 순환 중복 검사(Cyclic Redundancy Check, CRC) 코드가 추가된다. CRC 연산은 다항식 나눗셈을 기반으로 송신측 페이로드 전체 비트 배열에 대한 연산 값을 생성하여 프레임 꼬리(Tail) 영역에 첨부하는 방식이다.

  1. 송신측 캡슐화 연산: 직렬화된 uORB 메시지 페이로드 데이터에 특정 분산 다항식(Polynomial)을 적용시켜 체크섬 해시(Checksum Hash)를 도출한 후, 데이터 프레임에 부착한다.
  2. 수신측 파싱 및 검증: 수신 노드는 물리 매체에서 읽어들여 추출한 메시지 페이로드에 대해, 송신측과 동일한 패킷 프레이밍 조건 내에서 독자적으로 CRC 연산을 수행한다.
  3. 무결성 판정 로직: 수신측 연산으로 도출된 CRC 결괏값과 전송된 프레임에서 파싱한 원본 CRC 값을 논리적으로 비교한다. 두 값이 불일치할 경우, 해당 통신 프레임을 비트 에러(Bit Error) 또는 변조가 개입된 손상 패킷으로 간주하고 시스템에서 즉각 폐기(Drop)한다.

CRC 무결성 검증은 직렬 통신 라인의 전자기적 간섭(EMI)으로 유발되는 비트 플립(Bit Flip)이나, UDP 스택의 오차 범위를 넘어서는 잔여 오류를 어플리케이션 계층에서 최종적으로 선별하는 기제로 동작한다. 이를 통해 ROS2 생태계의 비행 제어 알고리즘으로 오염된 상태(State) 데이터가 유입되는 현상을 차단하고, 자율 드론의 행동 및 제어 무결성을 유지한다.
