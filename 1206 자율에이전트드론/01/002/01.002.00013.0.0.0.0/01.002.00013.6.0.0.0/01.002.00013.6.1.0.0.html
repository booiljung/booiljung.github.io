13.6.1 엔드투엔드 암호화 및 접근 제어 토큰 기반 기체 인증 체계

멀티 에이전트 드론 시스템이 무인기 교통 관리(UTM) 프레임워크나 다중 벤더의 클라우드 관제 플랫폼과 융합됨에 따라, 네트워크를 경유하는 제어 명령(Command and Control, C2) 및 텔레메트리 파이프라인의 기밀성(Confidentiality)과 무결성(Integrity) 보장은 안전 필수(Safety-Critical) 요건으로 대두되었다. 특히 광역 통신망(WAN)은 사이버 공격의 주요 표적이 되며, 중간자 공격(Man-In-The-Middle, MITM)이나 스푸핑(Spoofing)으로 기체의 제어권이 탈취될 경우 물리적 재난으로 직결된다. 이러한 위협을 원천적으로 차단하기 위해 텔레메트리 송출단(드론)부터 수신단(클라우드 서버 또는 지상 관제소)에 이르기까지 어떠한 중계 노드도 원문을 해독할 수 없도록 보장하는 엔드투엔드(End-to-End, E2E) 암호화 아키텍처와, 기체의 고유 식별 및 권한을 증명하는 접근 제어 토큰(Access Control Token) 기반 인증 체계가 통합 설계되어야 한다.

1. 엔드투엔드 기밀성 및 하드웨어 기반 난수 인증

엔드투엔드 암호화의 핵심은 데이터의 생성 주체와 최종 소비 주체만이 암호화 키를 소유하는 것이다. ROS2 생태계의 SROS2(Secure ROS 2) 및 범용 IoT 프로토콜 아키텍처 내에서 이는 비대칭키 기반의 상호 인증(Mutual Authentication)과 대칭키 교환 암호계를 통해 구현된다.

  1. 하드웨어 신뢰 부리(Root of Trust): 엣지 컴퓨터에 내장된 신뢰할 수 있는 플랫폼 모듈(TPM, Trusted Platform Module) 또는 하드웨어 보안 모듈(HSM) 내부에 기체 고유의 개인키(Private Key)를 안전하게 격리한다. 암호화 연산 자체를 하드웨어 모듈 내부에서 수행함으로써, 기체의 운영 체제가 해킹되더라도 개인키의 유출을 수학적, 물리적으로 방지한다.
  2. 세션 키(Session Key) 기반 고속 E2E 암호화: 비행을 시작하기 전, 디피-헬만(Diffie-Hellman, ECDHE) 키 교환 알고리즘을 통해 드론과 클라우드 서버는 전방 향 안전성(Forward Secrecy)이 보장된 일회성 세션 키를 생성한다. 이후 초당 수백 회 발생하는 고빈도 텔레메트리 스트리밍은 획득된 세션 키와 AES-GCM(Galois/Counter Mode) 알고리즘을 결합하여 지연 시간을 최소화하면서 지속적으로 암호화된다.

이러한 암호화 메커니즘을 통과한 데이터 파이프라인은 중간의 릴레이 서버나 5G 기지국 등 모든 브리지(Bridge) 인프라에서 단순한 비트열로만 관측되므로, 페이로드의 완전한 엔드투엔드 기밀성이 보호된다.

2. 다이내믹 권한 증명: 접근 제어 토큰 (Access Control Token) 메커니즘

수많은 이종(Heterogeneous) 비행체와 다수의 운영자가 동시에 접속하는 거대 클라우드 관제망에서는 단순히 통신 채널을 암호화하는 것을 넘어, “이 패킷을 전송하는 개체가 정당한 권한을 보유하고 있는가“를 매 순간 확인하는 인가(Authorization) 메커니즘이 수반되어야 한다.

이를 위해 무인기 아키텍처에서는 JSON Web Token(JWT) 기반의 상태 비저장(Stateless) 접근 제어 체계가 널리 사용된다.

  • 토큰 발급과 서명(Signing): 비행 준비 단계에서 기체는 TPM 기반 자격 증명을 클라우드의 인증 서버(Identity Provider)로 제출하고, 승인을 통해 짧은 유효 기간(Time-To-Live, TTL)을 가지는 토큰을 발급받는다. 이 토큰에는 기체의 고유 식별자(UUID), 비행 승인 공역 크기 유효 기간, 허용된 제어 토픽(예: /fmu/in/vehicle_command) 접근 권한과 같은 메타데이터가 포함되며 인증 서버의 개인키로 전자서명된다.
  • 무상태성 무결성 검증(Stateless Verification): 비행 중 드론이 텔레메트리 파이프라인이나 MAVLink/XRCE-DDS 페이로드의 헤더에 토큰을 동봉하여 전송하면, 수신 노드(클라우드 분산 워커 노드)는 통신 지연을 유발하는 중앙 서버로의 세션 확인 요청 없이 즉각적으로 공개키를 통해 서명의 조작 여부와 토큰의 만료를 독립적으로 검증한다.
  • 세분화된 역할 기반 접근 제어(RBAC): 토큰 내부에 명시된 스코프(Scope)를 기반으로 하여, 특정 드론이 클라우드의 글로벌 맵 최적화 서비스에는 데이터를 게시(Publish)할 수 있지만, 동일 망 내에 있는 다른 방제 드론의 동력 시스템 제어 권한은 원천적으로 격리되는 다중 테넌트(Multi-tenant) 제어 계층을 구축한다.

결과적으로 엔드투엔드 연속 암호화와 동적으로 갱신되는 접근 제어 토큰의 결합은 기체의 데이터 스트리밍 경로와 권한 실행 파이프라인을 다중으로 방어하며, Zero-Trust 아키텍처에 부합하는 가장 강력한 형태의 클라우드 연동형 드론 보안 체계를 완성한다.
