13.5.1 단대단 지연율 기반 동적 연산 분배 알고리즘

무인항공기(UAV)의 임무가 고도화됨에 따라 비전 처리, 경로 계획 및 딥러닝 추론과 같은 연산 집약적 작업(Computation-intensive Tasks)의 요구량이 급증하고 있다. 그러나 탑재된 엣지 컴퓨팅 하드웨어의 전력 및 자원 제약으로 인해 모든 연산을 기체 내에서 실시간으로 처리하는 것은 물리적 한계를 지닌다. 이를 극복하기 위해 클라우드 또는 모바일 엣지 컴퓨팅(MEC, Mobile Edge Computing) 환경으로 연산을 위임하는 클라우드 오프로딩(Cloud Offloading) 아키텍처가 필수적으로 요구된다. 특히, 비행 제어 및 충돌 회피와 같은 안전 필수(Safety-critical) 작업은 지연에 매우 민감하므로, 네트워크 상태와 연산 부하를 실시간으로 인지하여 단대단 지연율(End-to-End Latency)을 최소화하는 동적 연산 분배 알고리즘(Dynamic Computation Offloading Algorithm)의 설계가 아키텍처의 핵심을 이룬다.

1. 단대단 지연율 모델링 (End-to-End Latency Modeling)

연산 분배 과정에서의 단대단 지연율은 기체에서 클라우드로 데이터를 전송하는 업링크 지연(Uplink Delay), 클라우드 서버에서의 연산 처리 지연(Computation Delay), 그리고 처리된 결과 데이터를 기체로 회신하는 다운링크 지연(Downlink Delay)의 총합으로 정의된다. 시스템의 상태를 이산 시간 환경으로 가정할 때, 타임 슬롯 t에서의 단대단 지연 지표 T_{total}(t)는 다음 수식으로 모델링된다.

T_{total}(t) = T_{up}(t) + T_{comp}(t) + T_{down}(t)

1. 업링크 및 다운링크 통신 지연 (Communication Delay)

통신 지연은 무선 채널의 상태와 대역폭에 직접적인 영향을 받는다. 입력 데이터 크기 D_{in}(t)를 클라우드로 전송할 때 발생하는 업링크 지연 T_{up}(t)는 섀넌-하틀리 정리(Shannon-Hartley Theorem)에 기반한 업링크 전송 속도 R_{up}(t)에 의해 결정된다.

T_{up}(t) = \frac{D_{in}(t)}{R_{up}(t)}

마찬가지로 결과 데이터 크기 D_{out}(t)와 다운링크 전송 속도 R_{down}(t)에 의해 다운링크 지연 T_{down}(t)이 산출된다. 동적 셀룰러 네트워크나 위성 통신 환경에서는 페이딩(Fading) 및 간섭(Interference)으로 인해 통신 속도의 시변성(Time-varying)이 강하게 나타나므로, 통신 지연의 불확실성을 최소화하기 위한 추정(Estimation) 과정이 동반되어야 한다.

1.1 클라우드 연산 처리 지연 (Computation Delay)

클라우드 환경에서의 연산 처리 시간 T_{comp}(t)는 작업에 필요한 총 CPU 사이클 수 W_{total}(t)와 클라우드 서버에 할당된 연산 주파수(가상 머신 또는 컨테이너의 처리 성능) f_{cloud}(t)에 비례하여 결정된다.

T_{comp}(t) = \frac{W_{total}(t)}{f_{cloud}(t)}

대기 큐(Queue) 이론에 기반하여 클라우드 서버의 부하율이 높은 경우 프로세스 대기 시간(Queuing Delay)이 추가적으로 발생할 수 있다.

동적 분배(Offloading) 결정 정책

단대단 지연율 기반 동적 연산 분배의 목적은 기체 내부 프로세싱(Local Processing)과 클라우드 오프로딩(Cloud Offloading) 중 지연 시간과 에너지 소모를 최적화하는 분배 비율(Offloading Ratio) \alpha \in [0, 1]를 결정하는 최적화 문제(Optimization Problem)를 푸는 것이다. \alpha = 0은 완전한 로컬 처리, \alpha = 1은 완전한 클라우드 오프로딩을 의미하며, 0 < \alpha < 1은 부분 분배(Partial Offloading)를 나타낸다.

이를 해결하기 위해 다양한 수학적 및 기계학습적 접근법이 적용된다.

리아프노프 최적화 (Lyapunov Optimization)

제약 조건(예: 기체의 배터리 잔량, 큐 안정성) 하에서 장래의 시간 평균 지연율을 최소화하기 위해 리아프노프 최적화 기법이 널리 활용된다. 큐의 상태를 기반으로 리아프노프 드리프트(Lyapunov Drift)를 정의하고, 에너지 소비와 지연율 간의 트레이드오프(Trade-off)를 제어 변수 V를 통해 조절한다. 즉, 매 타임 슬롯마다 드리프트 플러스 페널티(Drift-plus-Penalty) 식을 최소화하는 하위 문제를 독립적으로 풀어 결정론적인 오프로딩 정책을 산출한다. 이는 사전 통계적 정보(A priori statistical information) 없이도 시스템의 안정성을 보증할 수 있는 강력한 수학적 도구이다.

심층 강화학습 기반 연속 공간 제어 (Deep Reinforcement Learning)

매우 빠르게 변화하는 비행 환경과 비선형적인 통신/연산 시스템 모델에서는 심층 강화학습(DRL, Deep Reinforcement Learning)이 효과적인 해결책을 제공한다. 에이전트(UAV)는 상태(State, 가용 대역폭, 로컬 부하, 데이터 크기)를 관측하고, 행동(Action, 오프로딩 분배율)을 수행한 후, 보상(Reward, 단대단 지연 시간의 역수 또는 에너지 패널티)을 받는다. DDPG(Deep Deterministic Policy Gradient)나 PPO(Proximal Policy Optimization)와 같은 연속 공간 알고리즘을 활용하여 동적 분배 정책을 학습시킴으로써 확률적 네트워크 변동성에 강건한 모델을 구현할 수 있다.

ROS2 시스템 구현 고려사항

ROS2 프레임워크 기반의 자율 드론 환경에서는 동적 연산 분배 알고리즘을 구현하기 위해 미들웨어의 QoS(Quality of Service) 정책과 오프로딩 스케줄러 노드의 통합이 필수적이다.

  1. 지연율 관측 노드 구성: sensor_msgs 또는 사용자 정의 메시지를 처리하는 파이프라인에서 타임스탬프를 추출하여 로컬 처리 지연율과 원격 콜백 지연율을 주기적으로 측정하는 옵저버(Observer) 생태계를 구축해야 한다.
  2. QoS 오프셋 프로파일링: 클라우드 오프로딩 시 통신 채널의 상태에 따라 ROS2 DDS/XRCE-DDS의 RELIABILITY 및 DEADLINE QoS 설정을 동적으로 스위칭해야 한다. 오프로딩 파이프라인은 통신 지연이 한계치를 초과할 경우 즉각 로컬 처리 파이프라인 회복(Fallback)으로 전환되도록 설계되어야 무인기의 안전성이 보장된다.
  3. 분산 Action Server 구성: 대규모 연산 분배 시 ROS2 Action 인터페이스를 활용하여 연산 진행 상태에 대한 지속적인 피드백(Feedback)을 수신해야 하며, 타임아웃 발생 시 즉각 임무 재계획 알고리즘에 인터럽트를 발생시켜야 한다.

안전 필수적인 시스템 아키텍처의 특성상, 단대단 지연율 기반 동적 연산 분배 알고리즘은 단일 실패점(SPOF, Single Point of Failure)을 극복하고 제어 주기의 무결성을 수호하기 위한 최상위 스케줄러로 작동해야만 한다.
