14.2.4 대역폭 포화 상태에서의 전송 누락 및 지연 증가 패턴

비행 제어기(Flight Controller)와 컴패니언 컴퓨터(Companion Computer)를 연결하는 물리적 인터페이스는 주로 비동기 직렬 버스(UART)에 의존한다. 최신 시스템에서 보드레이트(Baudrate)를 921,600 bps나 그 이상인 2~3 Mbps로 설정하더라도, 고해상도 오도메트리(Odometry)와 수백 Hz의 IMU 데이터, 그리고 다중 액추에이터 제어 명령이 교차하는 환경에서는 채널 대역폭 한계(Bandwidth Limit)에 도달하는 포화 상태(Saturation)가 간헐적으로 발생한다. 브릿지 통신 계층의 물리적 대역폭이 포화되었을 때 시스템이 응답하는 양태는 단순한 ’지연’이 아니라 제어 역학을 붕괴시키는 특이적인 패턴(Pattern)으로 나타나며, 이를 이해하는 것은 실시간 통신망을 방어하는 기초가 된다.

1. 대역폭 포화 시의 선형적 지연 증가 곡선

데이터 발생률(Data Generation Rate)이 버스의 물리적 전송률(Transmission Rate)을 초과하는 순간부터, 브릿지 에이전트 내부의 송신 버퍼(Transmit Buffer)에는 데이터가 잉여적으로 누적되기 시작한다.

  • 버퍼블로트(Bufferbloat) 지연: ROS2 QoS나 브릿지의 내부 큐(Queue)가 데이터를 즉시 폐기하지 않고 버퍼에 저장(Queueing)하도록 설정되어 있을 경우, 새롭게 발행된 최신 제어 데이터는 과거의 낡은 데이터들이 모두 전송될 때까지 대기해야 한다. 무한정 깊은 큐(Infinite Queue Depth)를 가질 경우 지연 시간은 대기열의 길이에 비례하여 O(N)으로 선형 증가(Linear Increase)하게 되며, 이는 수백 밀리초(ms)에 달하는 치명적인 위상 지연(Phase Lag)의 직접적인 원인이 된다.
  • 군집 전송(Bursty Transmission) 패턴: 커널 스케줄러나 버스 인터럽트의 간섭으로 인해 데이터가 일정한 간격으로 나가지 못하고 병목에 막혀 있다가, 일시에 대량으로 수신단에 쏟아지는 군집 전송 현상이 발생한다. 비행 제어기 입장에서는 데이터 수신 주기가 무작위로 축소되었다가 팽창하는 극단적인 지터(Jitter)로 관측된다.

2. 전송 누락(Packet Loss)의 비대칭적 양태

큐의 깊이(Depth)가 얕을 경우(예: KEEP_LAST, Depth=1), 대역폭 포화 시 오래된 데이터는 무작위로 유실(Drop)된다. 이러한 전송 누락은 데이터의 특성에 따라 기체에 비대칭적인 치명상을 입힌다.

  1. 상태 추정 연속성의 파괴: 고속으로 기동하는 드론의 EKF(Extended Kalman Filter)는 일정한 주기의 예측-보정(Predict-Update) 단계를 밟는다. 오프보드 비전 시스템에서 계산된 시각적 주행 거리 측정(VIO) 페이로드가 빈번히 유실되어 EKF 업데이트 주기를 무작위로 건너뛰게 되면, 필터의 공분산(Covariance) 상수가 폭주하여 기체의 공간 좌표가 튀는 공간적 글리치(Spatial Glitch)를 유발한다.
  2. 연속된 제어 명령의 손실과 페일세이프 트리거: 브릿지가 대역폭을 선점한 거대한 텔레메트리 데이터 덩어리를 직렬화하여 송신하는 동안 버스는 점유(Busy state)된다. 이때 발행된 민첩한 충돌 회피용 TrajectorySetpoint 메시지가 큐 오버플로우로 연속 폐기될 경우, 비행 제어기 내부의 오프보드 만료 타이머(Offboard Timeout)가 작동하여 지상망의 통제가 상실된 것으로 판별하고, 의도치 않은 자체 RTL(Return-To-Launch) 페일세이프(Fail-Safe) 기동에 돌입하는 설계상 모순이 발생한다.

결국 대역폭 포화 상태 하에서 관측되는 지연 증가와 패킷 무작위 누락 패턴은 단순한 하드웨어의 성능 부족이 아니며, 응용 계층의 트래픽을 식별하고 제어 명령에 절대적 우선권(Absolute Priority)을 부여하지 않은 미들웨어 스케줄링 설계의 실패에서 기인한다 볼 수 있다.
