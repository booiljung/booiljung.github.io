14.2.2 범용 OS 커널의 스레드 선점 및 컨텍스트 스위칭 오버헤드

ROS2 생태계를 호스팅하는 오프보드 임무 컴퓨터(예: 라즈베리 파이, 엔비디아 젯슨 등)는 대다수 리눅스(Linux) 기반의 범용 운영체제를 채택하고 있다. 브릿지 애플리케이션(Micro XRCE-DDS Agent 또는 MAVROS)은 이 범용 커널 위에서 하나의 유저 공간(User Space) 프로세스 및 스레드로 인스턴스화되어 구동된다. 범용 OS 커널은 기본적으로 ‘완전 공정 스케줄러(CFS, Completely Fair Scheduler)’ 철학을 바탕으로 다수의 작업이 CPU 자원을 균등하게 시분할(Time-sharing)하도록 설계되어 있으므로, 하드웨어 타이밍 보장이 필수적인 사이버-물리 시스템(CPS)의 통신 관점에서는 치명적인 스레드 선점(Preemption) 지연 및 컨텍스트 스위칭(Context Switching) 오버헤드를 발생시킨다.

1. 컨텍스트 스위칭에 따른 하드웨어 아키텍처 페널티

브릿지 스레드가 직렬 포트(UART)로부터 수신된 바이트 스트림을 역직렬화하는 도중 커널의 스케줄러에 의해 CPU 점유권을 박탈당하고 다른 프로세스(예: 비전 처리 노드)로 제어권이 넘어가는 과정을 컨텍스트 스위칭이라 한다. 이 과정은 단순한 소프트웨어적 상태 저장을 넘어 막대한 하드웨어 페널티를 동반한다.

  • 파이프라인 및 레지스터 플러시: CPU 파이프라인의 명령어 흐름이 끊기며, 현재 실행 중이던 브릿지 스레드의 프로그램 카운터(PC)와 범용 레지스터 상태가 커널 메모리(Task Control Block)로 복사된다.
  • 번역 색인 버퍼(TLB) 무효화 및 캐시 미스(Cache Miss): 다른 프로세스로 전환될 때 가상 메모리의 페이지 테이블 기준 주소가 변경되므로, 기존 메모리 주소를 매핑하고 있던 TLB가 무효화(Flush)된다.
    이후 브릿지 스레드가 다시 CPU를 할당받아(Wake-up) 실행을 재개할 때, 필수적인 데이터(예: ROS2 메시지 구조체)가 CPU의 L1/L2 캐시에서 축출(Eviction)된 상태이므로 주 메모리(RAM)까지 접근해야 하는 거대한 캐시 미스 페널티가 발생하여 실행 시간이 밀리초 단위로 요동치게 된다.

2. 스레드 선점 지연(Preemption Latency)과 시스템 콜 블로킹

브릿지 통신 계층의 지터를 더욱 악화시키는 요인은 커널 자체가 유발하는 선점 지연이다.

  1. 비선점형 임계 영역(Non-preemptible Critical Section): 표준 리눅스 커널에서는 시스템 콜(System Call)을 수행하거나 특정 커널 락(Spinlock)을 획득하여 커널 공간에서 코드가 실행 중일 때, 더 높은 우선순위의 하드웨어 인터럽트나 스레드가 활성화되더라도 즉시 선점하지 못하는 구간이 존재한다. 브릿지 노드가 read()나 write() 시스템 콜을 통해 통신할 때 다른 커널 작업이 스케줄러를 블로킹하고 있다면, 통신 패킷의 처리 데드라인이 무작위로 지연된다.
  2. 우선순위 역전(Priority Inversion) 현상: 브릿지 스레드에 실시간 우선순위(RT Priority)를 부여하더라도, 하위 우선순위를 가진 프로세스가 커널 내부의 공유 자원(예: 네트워크 소켓 버퍼 락)을 먼저 선점하고 있을 경우, 브릿지 스레드는 하위 스레드의 작업이 완료될 때까지 대기(Block)해야 하는 설계상 모순이 발생한다.

3. 하부 인터럽트 처리(Bottom Half)의 비결정성

하드웨어 통신 컨트롤러(예: 이더넷 MAC, UART 컨트롤러)에 데이터가 도착하여 하드웨어 인터럽트(Top Half)가 트리거된 후, 실제 데이터를 커널 버퍼로 복사하고 브릿지 스레드를 깨우는 연기된 작업(Bottom Half, SoftIRQ)의 스케줄링 타이밍 또한 비결정적이다. 커널 부하가 심각할 경우 SoftIRQ 데몬(ksoftirqd)의 실행이 후순위로 밀리게 되며, 이는 직렬 버스를 통과한 제어 데이터가 소프트웨어 계층에 도달하기 직전에 알 수 없는 큐잉 지연(Queuing Delay)을 겪게 됨을 의미한다.

결과적으로 범용 OS 커널에서의 통신 스레드 운영은 커널의 공정성 철학과 하드웨어 캐시 아키텍처의 부작용이 맞물려 필연적인 백색 잡음(White Noise) 형태의 지터를 양산하며, 이는 비행 제어기 측의 역학 모델 수렴 속도에 직접적인 외란(Disturbance)으로 작용한다.
