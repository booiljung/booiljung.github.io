14.2.1 직렬화-역직렬화 수행 시 가변 지연 시간 측정

ROS2와 PX4 오토파일럿을 연결하는 브릿지 아키텍처(예: Micro XRCE-DDS)에서 가장 빈번하게 발생하는 연산 오버헤드는 이기종 시스템 간의 데이터 규격을 일치시키기 위한 직렬화(Serialization) 및 역직렬화(Deserialization) 과정이다. 메모리에 상주하는 다차원 구조체(Struct)나 객체(Object)를 직렬 통신(UART)이나 네트워크 소켓으로 전송하기 위해서는 바이트 단위의 1차원 배열(Byte Stream)로 패킹(Packing)하는 직렬화가 필수적이며, 수신단에서는 이를 다시 본래의 변수 타입으로 언패킹(Unpacking)해야 한다. 이 과정은 하드웨어 종속적인 메모리 정렬(Memory Alignment) 규칙과 데이터 타입의 복잡성에 따라 매회 다른 처리 시간을 요구하므로, 시스템의 타이밍 변동성을 증폭시키는 주된 가변 지연(Variable Latency) 요인으로 작용한다.

1. 데이터 구조 결합도에 따른 직렬화 복잡도 차이

직렬화 알고리즘의 소요 시간은 페이로드(Payload)의 데이터 타입(Data Type) 구조에 상응하여 비선형적인 지터를 발생시킨다.

  • 기본 자료형과 정적 배열(Static Array): sensor_msgs/Imu나 단위 쿼터니언(Quaternion)과 같은 고정 해상도의 스칼라(Scalar) 값 집합은 메모리 상에 연속적으로 배치되므로, memcpy와 같은 저수준 메모리 복사 명령을 통해 상수 시간(O(1)) 내에 직렬화가 완료된다. 이러한 메시지는 지터 발생률이 극히 낮다.
  • 가변 길이 배열(Variable-Length Array) 및 동적 할당: 반면, sensor_msgs/PointCloud2나 가변적인 경로 웨이포인트(Waypoint) 배열과 같이 실행 도중(Run-time) 크기가 변하는 구조체는 심각한 지연을 초래한다. 직렬화 과정에서 힙(Heap) 메모리의 동적 할당(Dynamic Allocation)이 발생하며, 배열의 길이에 비례하는 O(N)의 선형적 인코딩 시간이 요구된다. 특히 리눅스 범용 커널에서는 동적 메모리 할당 시 커널의 락(Lock) 경합 상태에 따라 밀리초(ms) 단위의 예측 불가능한 스톨(Stall)이 발생할 수 있다.

2. 엔디안(Endianness) 변환 및 메모리 패딩(Padding) 정렬 비용

이기종 하드웨어 간의 아키텍처 차이는 단순한 복사를 넘어 부가적인 CPU 연산을 강제한다.

  1. 바이트 순서(Byte Order) 스와핑: 컴패니언 컴퓨터(예: ARM64 아키텍처)와 특정 비행 제어기의 마이크로컨트롤러 간에 메모리를 읽는 방향(Little-Endian vs Big-Endian)이 다를 경우, 브릿지의 직렬화 엔진(예: FastCDR)은 송신 시점이나 수신 시점에 모든 변수의 바이트 순서를 역전(Swapping)시키는 연산을 수행해야 한다.
  2. 단편화와 패딩 정렬 비용: 데이터 버스의 접근 효율을 높이기 위해 컴파일러가 임의로 삽입하는 패딩(Padding) 바이트는 네트워크 대역폭 낭비를 막기 위해 직렬화 시 제거(Stripping)되어야 한다. 구조체 내부의 데이터 타입이 혼재(예: 8-bit 정수와 64-bit 부동소수점의 교차 배치)되어 있을수록 이 패딩 제거 연산을 위한 조건 분기(Branching)가 증가하여, CPU 파이프라인의 분기 예측(Branch Prediction) 실패로 인한 연산 지터가 누적된다.

3. 정밀 프로파일링을 통한 가변 지연도 계측

이러한 직렬화-역직렬화 오버헤드를 정량적으로 측정하고 통제 범위 내에 두기 위해서는 다음과 같은 계측 기법이 동원되어야 한다.

  • 브릿지 노드의 퍼블리시(Publish) 함수가 호출되기 직전과 직렬화 함수(serialize())가 반환되는 시점에 C++ std::chrono::high_resolution_clock을 이용해 타임스탬프를 획득하여 나노초(ns) 해상도의 순수 직렬화 델타(Delta) \Delta T_{ser}를 도출한다.
  • 획득한 \Delta T_{ser} 데이터세트를 바탕으로 확률밀도함수(PDF)를 플로팅(Plotting)하고, 99번째 백분위수(99th Percentile) 최악 실행 시간(WCET)이 하드 실시간 제어 루프의 데드라인 마진(Margin)을 침범하지 않도록 데이터 구조(IDL) 최적화를 수행한다.

가변적인 직렬화 시간 측정은 메시지 체계의 물리적 무게를 다는 과정과 같으며, 데이터 타입의 평탄화(Flattening)와 고정 크기 배치를 통해 지터를 상수화하는 것이 실시간 브릿지 통신망 설계의 핵심 원칙이다.
