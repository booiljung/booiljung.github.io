19.2.2 이기종 콜백(고주파 IMU 대 저주파 경로 계획) 간 스케줄링 간섭

자율 드론의 비행 제어 시스템은 본질적으로 다중 주기(Multi-rate) 특성을 갖는 이기종(Heterogeneous) 프로세스들의 집합체이다. ROS2(Robot Operating System 2) 환경에서 이러한 이기종 노드들이 동일한 Executor나 상호 간섭이 가능한 스레드 풀을 공유할 때, 치명적인 스케줄링 간섭(Scheduling Interference)이 유발될 수 있다. 특히 100Hz에서 500Hz 이상의 고주파 주기를 요구하는 관성측정장치(IMU: Inertial Measurement Unit) 제어 루프와 1Hz에서 10Hz 미만의 저주파 주기를 가지며 대규모 연산을 수반하는 전역 경로 계획(Global Path Planning) 알고리즘 간의 충돌은 대표적인 간섭 모델이다.

1. 이기종 콜백의 실행 시간(Execution Time) 비대칭성

관성측정장치 제어 루프는 드론의 자세 제어(Attitude Control)와 직접적으로 연관되며, 매우 짧은 지연 시간(Low Latency)과 엄격한 데드라인을 요구한다. IMU 콜백은 메시지 크기가 작고 연산 복잡도가 극히 낮아 단일 콜백 실행 시간 단위가 마이크로초(\mu s) 단위에 불과하다.

반면, 경로 계획 콜백(예: RRT*, A* 알고리즘 기반 궤적 생성)은 복잡한 공간 탐색과 충돌 회피 연산을 수반하기 때문에 실행 시간이 수십 밀리초(ms)에서 수백 밀리초에 달한다. Multi-Threaded Executor가 이 두 가지 상이한 성격의 콜백을 동일한 스레드 풀 환경에서 라운드 로빈(Round-Robin) 방식으로 디스패칭할 경우, 실행 기간이 긴 경로 계획 콜백이 워커 스레드를 장시간 점유하게 된다.

2. 선점 불가(Non-Preemptable) 특성으로 인한 간섭 기작

ROS2 콜백 디스패칭은 운영체제의 타이머 인터럽트와 달리, 한 번 스레드에 할당된 콜백이 스스로 종료되어 반환될 때까지 다른 콜백이 해당 스레드를 선점(Preemption)할 수 없는 비선점형 방식이다. 따라서 다음과 같은 스케줄링 간섭 체인이 형성된다.

  1. 스레드 블로킹(Thread Blocking): 저주파의 경로 계획 콜백이 스레드 풀의 가용 스레드들을 점유하여 장기 연산에 돌입한다.
  2. 고주파 큐(Queue) 팽창: 연산이 진행되는 동안, 수백 헤르츠(Hz)로 유입되는 IMU 데이터 메시지들이 Ready Set의 구독 큐(Subscription Queue)에 급격하게 누적된다.
  3. 지터(Jitter) 발생 및 신호 지연: 즉각적으로 처리되어야 할 IMU 데이터가 제때 스케줄링되지 못함에 따라, 센서 업데이트 주기 사이에 심각한 지터가 발생하며, 칼만 필터(Kalman Filter)와 같은 상태 추정기(State Estimator)로 전달되는 상태 변수들에 위상 지연(Phase Lag)이 유도된다.

3. 제어 공학적 시스템 붕괴 원인

이기종 콜백 간의 간섭으로 인해 유발되는 고주파 제어 루프의 지터와 위상 지연은 단순한 소프트웨어 비효율성을 넘어, 드론의 동역학적 안정성(Dynamic Stability)을 붕괴시킨다. 피드백 제어 시스템(Feedback Control System) 관점에서 센서 관측 지연(Observation Delay)은 폐루프(Closed-loop) 시스템의 위상 여유(Phase Margin)를 급격히 감소시킨다. 위상 여유가 확보되지 않은 상태에서의 PID 제어기 작동은 발산하는 형태의 진동(Oscillation)을 야기하며, 최종적으로 드론의 추락으로 귀결될 수 있다.

4. 학술적 시사점

초기 ROS2 아키텍처(예: Galactic, Humble 버전에 적용된 기본 rclcpp Executor) 기반의 단일 런타임 내에서 CPU 연산 집중적(CPU-bound) 작업과 I/O 처리 집중적(I/O-bound) 실시간 작업을 단순히 동기화 프리미티브나 멀티 스레드로 혼재시키는 것은 구조적 한계가 명확하다. 따라서 고주파 하드웨어 타이머 속성에 종속적인 센서 처리망과 비동기식 탐색 알고리즘망 사이의 간섭을 원천적으로 차단하기 위해, 각각 독립적인 Callback Group으로 분리하고 물리적으로 격리된 단일 스레드 구조(Single-Threaded Executor)를 복수로 구성하여 운영체제 수준의 프로세스 스케줄러(Process Scheduler) 정책에 위임하는 설계 패턴이 필수적으로 요구된다.
