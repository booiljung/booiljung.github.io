19.1.3 자율 비행 코어 컴포넌트 간 콜백 처리의 결정론적 요구사항

자율 에이전트 드론을 구성하는 코어 컴포넌트들, 특히 자세 추정(Attitude Estimation)과 모터 믹싱(Motor Mixing)을 담당하는 하위 비행 제어 루프(Flight Control Loop)는 수학적으로 엄격한 실시간성(Real-Time)을 요구한다. 이 문맥에서 실시간성이란 단순히 “빠른 연산“을 의미하는 것이 아니라, 콜백 함수의 실행 지연(Latency)과 주기의 지터(Jitter)가 사전에 계산된 시간적 상한선(Upper Bound)을 절대 초과하지 않는 결정론(Determinism)을 의미한다. 만약 400Hz(2.5ms 주기)로 동작해야 하는 다이내믹스 제어 콜백이 이그제큐터의 스케줄링 변동성으로 인해 1ms의 지연을 겪는다면, 이는 위상 여유(Phase Margin)를 감소시켜 드론의 공기역학적 발산을 초래할 수 있다.

1. 연산 지연의 상한성(Bounded Latency) 엄수

자율 비행 코어 컴포넌트가 결정론을 확보하기 위해서는 우선 개별 콜백 자체의 연산 주기가 철저하게 상한성(Bounded)을 가져야 한다. ROS2 이그제큐터는 콜백을 선점(Preemption)할 수 없는 런-투-컴플리션(Run-to-Completion) 구조이므로, 하나의 콜백 내부에 무한 루프나 동적 락(Dynamic Lock)으로 인한 멈춤 현상이 발생하면 스레드 전체의 결정론이 파괴된다.

따라서 비행 제어 코드를 작성할 때는 O(1) 또는 예측 가능한 O(N)의 시간 복잡도를 가지는 알고리즘만을 사용해야 하며, 블로킹 I/O 호출 및 malloc과 같은 런타임 동적 메모리 할당을 철저히 배제해야 한다. 메모리 풀(Memory Pool) 모델을 기반으로 WaitSet 및 메시지 버퍼를 정적(Static)으로 사전 할당(Pre-allocation)하는 것은 이그제큐터 레벨의 결정론을 사수하기 위한 첫 번째 규칙이다.

2. 코어 별 콜백 스레드의 토폴로지 격리와 실시간 선점 제어

소프트웨어적인 상한성이 보장되더라도, 다중 스레드(Multi-Thread) 이그제큐터가 운영체제의 기본 시간 분할(Time-Sharing) 스케줄러(예: Linux의 SCHED_OTHER) 위에서 동작한다면 커널에 의한 문맥 교환(Context Switching) 지터에 노출된다. 이 한계를 극복하기 위해, 비행 제어 및 SLAM 코어와 같이 결정론적 디스패칭이 필수적인 콜백 그룹(Callback Group)은 전용(Dedicated) 단일 스레드 이그제큐터로 격리되어야 한다.

이렇게 격리된 이그제큐터 스레드의 가상 메모리는 커널 메모리 락(mlockall)을 통해 페이징(Paging) 스와프 아웃을 방지하며, 실시간 운영체제(RTOS) 확장 패치 공간 내에서 POSIX 실시간 스케줄링 정책인 SCHED_FIFO나 SCHED_RR과 결합되어 가장 높은 정적 우선순위(Static Priority)를 부여받아야 한다. 이를 통해 백그라운드의 로그 저장 스레드나 비디오 스트리밍 콜백이 CPU 자원을 점유하고 있더라도, 센서 인터럽트가 발생하는 즉시 커널 스케줄러가 비행 제어 이그제큐터 코어를 즉각적으로 선점(Preempt)하여 실행을 보장할 수 있다.

3. 엔드투엔드(End-to-End) 결정론적 파이프라인 형성

결과적으로, 드론 자율 비행의 결정론적 스케줄링은 콜백 계층 단일의 최적화가 아닌 ’센서(Sensor) \rightarrow RMW 계층(DDS QoS) \rightarrow 이그제큐터(Executor) \rightarrow 알고리즘(Algorithm) \rightarrow 액추에이터(Actuator)’로 이어지는 엔드투엔드(End-to-End) 파이프라인 전 구간의 일관된 실시간 제어로 완성된다.

비행 제어 소프트웨어 설계자는 이그제큐터 큐 크기를 유한하게 고정하고, Deadline 기반의 DDS 서비스 품질(QoS) 프로파일을 적용하여 메시지 드롭(Drop) 현상을 제어해야 하며, 지연 시간을 모니터링하기 위한 텔레메트리(Telemetry) 구조를 마련하여 시스템 운영 중 결정론의 훼손 여부를 실시간으로 입증해야 한다.
