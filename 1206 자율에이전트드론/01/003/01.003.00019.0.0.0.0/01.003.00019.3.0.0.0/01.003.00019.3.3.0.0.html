19.3.3 독립 이그제큐터 다중 인스턴스 기반 서브시스템 스레드 격리 정책

ROS2(Robot Operating System 2) 응용 프로그램 개발 시 가장 빈번하게 범하는 구조적 오류는 단일 프로세스 내의 모든 노드와 콜백(Callback)을 rclcpp::spin()을 통해 암시적으로 생성된 단일 전역 이그제큐터(Global Executor)에 종속시키는 것이다. 자율 드론과 같이 실시간성 요구 사항이 극명하게 갈리는 복합 시스템에서는 이러한 설계가 치명적인 병목 현상을 유발하므로, 서브시스템(Subsystem)의 도메인 특성에 맞추어 독립적인 이그제큐터 다중 인스턴스(Multiple Executor Instances)를 명시적으로 인스턴스화하고 스레드를 격리하는 정책이 요구된다.

1. 다중 인스턴스 이그제큐터 토폴로지 구조

다중 인스턴스 기반 스레드 격리 정책은 단일 프로세스 메모리 공간 내부를 논리적, 물리적으로 파티셔닝(Partitioning)하는 작업이다. 이 토폴로지는 드론의 핵심 기능을 인지(Perception), 계획(Planning), 제어(Control)의 3단계로 분리하고 각 단계의 워크로드 특성에 최적화된 독립된 이그제큐터를 매핑한다.

  1. 제어 서브시스템 이그제큐터 (Control Subsystem Executor): 비행 동역학의 기저를 담당하는 자세 제어 및 모터 PWM 신호 할당 계층은 극도의 결정론(Determinism)을 요구한다. 따라서 rclcpp::executors::SingleThreadedExecutor의 독립 인스턴스를 부여하여, 해당 스레드가 어떠한 상황에서도 다른 고부하 연산에 의해 지연되거나 락(Lock) 대기에 빠지지 않도록 완벽히 격리해야 한다.
  2. 인지 서브시스템 이그제큐터 (Perception Subsystem Executor): 스테레오 카메라 라우팅, 라이다(LiDAR) 포인트 클라우드 처리 등 병렬 처리가 필수적인 계층에는 코어 수에 맞추어 스레드 풀 크기가 제한된 rclcpp::executors::MultiThreadedExecutor를 할당한다. 이미지 단위의 막대한 데이터 대역폭 처리가 비행 제어 스레드의 캐시를 오염시키지 않도록 분리하는 것이 핵심이다.
  3. 계획 서브시스템 이그제큐터 (Planning Subsystem Executor): 다익스트라(Dijkstra)나 A* 등 장기 연산이 수반되는 궤적 최적화 계열은 별도의 이그제큐터 인스턴스 하위에서 동작하게 함으로써, 인지 서브시스템의 스레드를 고갈(Starvation)시키지 않게 격리한다.

2. Callback Group을 이용한 명시적 노드 할당 (Explicit Node Assignment)

이그제큐터 다중 인스턴스 정책을 코드 레벨에서 구현하기 위해서는 노드에서 생성되는 타이머(Timer)와 구독(Subscription) 객체가 기본(Default) 그룹에 바인딩되는 관행을 철저히 배제해야 한다.

개발자는 각 노드의 생성자 단계에서 create_callback_group API를 호출하여 도메인 목적에 맞는 콜백 그룹(Callback Group) 인스턴스를 복수 개 정의해야 한다. 이후 NodeOptions를 통하거나 콜백 생성 함수의 매개변수로 명시적인 콜백 그룹을 지정하고, 메인(Main) 프로세스 스핀 루프에서 각 이그제큐터 인스턴스의 add_callback_group 메서드를 통해 해당 그룹만을 선택적으로 스케줄러에 등록하는 전략 방식을 취한다.

// 개념적 구조 예시
auto control_executor = std::make_shared<rclcpp::executors::SingleThreadedExecutor>();
auto perception_executor = std::make_shared<rclcpp::executors::MultiThreadedExecutor>(...);

control_executor->add_callback_group(node->get_control_callback_group(), node->get_node_base_interface());
perception_executor->add_callback_group(node->get_perception_callback_group(), node->get_node_base_interface());

3. 스레드 격리 정책의 실시간 제어적 가치

이와 같은 다중 이그제큐터 인스턴싱과 명시적인 콜백 그룹 라우팅을 통해, 단일 프로세스 아키텍처는 마치 격리된 마이크로서비스(Microservices)의 조합처럼 작동하게 된다. 다중 스레드 환경에서 흔히 발생하는 우선순위 역전(Priority Inversion)과 교착 상태(Deadlock)의 발생 표면적 구조를 수학적으로 최소화할 수 있으며, 이그제큐터가 구동되는 표준 C++ 스레드 인스턴스(std::thread) 각각에 별도의 자원 제한(Resource Limits)과 CPU 선호도(CPU Affinity)를 독립적으로 부여할 수 있는 제어 프레임워크가 완성된다.
