19.5.3 NIC 인터럽트 처리 코어와 노드 콜백 스레드의 충돌 회피

자율 에이전트 드론 아키텍처는 군집 비행(Swarm Flight)을 위한 기체 간(V2V) 통신이나 지상 통제소(GCS)로의 고해상도 영상 스트리밍을 위해 막대한 네트워크 트래픽을 송수신한다. 이 과정에서 네트워크 인터페이스 컨트롤러(NIC: Network Interface Controller)가 발생시키는 하드웨어 인터럽트(Hardware Interrupt)와 운영체제 수준의 후속 처리 로직은, 극단적인 실시간성을 요구하는 ROS2(Robot Operating System 2) 비행 제어 이그제큐터(Executor) 스레드의 실행을 치명적으로 방해하는 숨겨진 성능 병목(Bottleneck) 요인으로 작용한다.

1. 하드웨어 인터럽트와 SoftIRQ 선점 현상

이더넷(Ethernet)이나 고속 Wi-Fi 통신망을 통해 UDP 기반의 DDS(Data Distribution Service) 패킷이 NIC에 도달하면, NIC는 즉각적으로 CPU 코어에 하드웨어 인터럽트(Hard IRQ)를 발생시킨다. 인터럽트를 수신한 CPU 코어는 현재 실행 중인 사용자 공간(User Space)의 스레드(예: ROS2 비행 제어 노드)를 강제로 일시 중지(Preemption)시키고 커널 공간(Kernel Space)의 인터럽트 서비스 루틴(ISR)을 실행한다.

이후 하드웨어 인터럽트는 처리를 신속하게 종료하기 위해 실제 네트워크 패킷의 프로토콜 스택(TCP/IP 또는 UDP 소켓 버퍼) 처리를 운영체제의 ksoftirqd 데몬이 관장하는 소프트 인터럽트(SoftIRQ, NET_RX_SOFTIRQ)로 지연(Defer)시킨다. 문제는 이 SoftIRQ 처리 과정이 엄청난 CPU 사이클을 소모한다는 점이다. 만약 NIC의 하드웨어 인터럽트가 비행 제어가 구동 중인 코어에 인가될 경우, 실시간 스케줄러(SCHED_FIFO)를 적용한 ROS2 스레드조차도 커널 레벨의 하드웨어/소프트웨어 인터럽트보다 낮은 우선순위를 가지므로 코어 제어권을 수 밀리초(ms) 이상 강제로 박탈당하게 된다.

2. IRQ 스티어링(Steering)과 smp_affinity 튜닝

이러한 커널 레벨의 선점 폭격을 방지하기 위해서는 NIC 인터럽트 처리 경로와 센서 제어콜백 처리 경로를 하드웨어 토폴로지상에서 완벽하게 분할해야 한다. 리눅스(Linux) 환경에서는 대칭형 다중 처리(SMP: Symmetric Multi-Processing) 친화도 설정을 통해 인터럽트의 목적지 코어를 명시적으로 조정(Steering)할 수 있다.

개발자는 먼저 시스템 코어 격리 정책(isolcpus)을 통해 제어 전담 코어(예: Core 1)를 사용자 공간에서 분리한 후, /proc/irq/{인터럽트_번호}/smp_affinity 인터페이스를 직접 수정하여 해당 NIC와 연관된 송수신 큐 인터럽트(RX/TX Queue IRQ)가 백그라운드 코어(예: Core 0)로만 전달되도록 비트마스크(Bitmask)를 설정해야 한다. 일부 고성능 NIC 장치에서 동작하는 irqbalance 데몬이 인터럽트를 시스템 전역으로 무작위 분산시키는 로직 또한 비행 제어 시스템의 결정론(Determinism)을 파괴하므로 반드시 비활성화되어야 한다.

3. 미들웨어 부하로부터의 제어기 고립 효과

NIC 인터럽트 처리 코어를 특정 유휴 코어나 비핵심 노드(텔레메트리, 로깅)가 할당된 코어로만 스티어링하는 설계는 제어 공학 측면에서 압도적인 가치를 지닌다. 군집 드론 네트워크에서 순간적으로 브로드캐스트 패킷 폭풍(Broadcast Storm)이 발생하거나, DDS의 다이나믹 디스커버리(Dynamic Discovery) 멀티캐스트(Multicast) 트래픽이 폭증하더라도, 해당 패킷을 해체하고 메모리에 복사하는 커널의 모든 페널티는 “Core 0“에 국한된다.

따라서 물리적으로 분할된 “Core 1“에서 독립적으로 구동 중인 ROS2 비행 제어 커스텀 이그제큐터는, 통신 채널의 외부적 혼잡 상황과 완벽하게 무관하게 자신의 관성측정장치(IMU) 제어 루프를 마이크로초(\mu s) 단위의 편차 없이 결정론적으로 유지할 수 있게 된다. 이는 통신과 제어라는 이기종 도메인의 물리적 충돌 관계를 완전히 해소하는 핵심 아키텍처이다.
