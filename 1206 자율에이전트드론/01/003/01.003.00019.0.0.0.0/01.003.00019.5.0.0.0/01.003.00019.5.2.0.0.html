19.5.2 머신비전 노드와 실시간 이그제큐터 간 물리 코어 분할

현대의 자율 에이전트 드론 아키텍처는 고전적인 동역학 제어뿐만 아니라 비주얼 오도메트리(VIO), 딥러닝 기반 객체 인식(Object Detection), 3차원 장애물 맵핑(Mapping)과 같은 머신비전(Machine Vision) 파이프라인을 필수적으로 포함한다. 이러한 비전 처리 노드와 고주파 비행 제어 루프를 하나의 공유된 처리 공간에 혼재시킬 경우 발생하는 하드웨어 자원 충돌은 물리적 코어 분할(Physical Core Partitioning) 설계를 통해서만 근본적으로 해결될 수 있다.

1. 머신비전 워크로드의 CPU 및 버스 포화 한계

카메라 노드에서 수집되는 초당 60프레임 이상의 고해상도(4K/1080p) 이미지는 프로세서 내부에 막대한 데이터 유입(Data Ingress) 트래픽을 유발한다. OpenCV 기반의 행렬 연산이나 실시간 이미지 디포매팅(De-formatting) 작업은 본질적으로 메모리 대역폭 집약적(Memory-Bandwidth Intensive)이며 CPU 내의 연산 유닛(ALU)을 포화(Saturation)상태로 내몰게 된다.

만약 머신비전 노드가 포함된 이그제큐터(Executor)와 실시간 제어(Attitude Control) 계층이 동일한 물리 코어(Physical Core) 또는 동일한 L2/L3 캐시(Cache) 바운더리를 공유하게 된다면, 비전 노드의 순차적 메모리 스캔은 제어 스레드의 중요 데이터를 캐시에서 지속적으로 축출(Eviction)한다. 더불어 주 메모리(RAM)와 연결된 시스템 버스(System Bus)의 혼잡으로 인해 제어 루프의 센서 데이터 페치(Fetch)가 지연되며, 최악 실행 시간(WCET: Worst-Case Execution Time)이 수십 밀리초(ms) 단위로 급등하는 제어 역학적 대참사가 발생한다.

2. 물리 코어 분할 모델 (Physical Core Partitioning Model)

이러한 상호 간섭을 차단하기 위해, 드론 임베디드 컴퓨터(예: 쿼드 코어 기반의 ARM Cortex 프로세서)에서의 토폴로지는 엄격한 하드웨어 파티셔닝 정책을 따라야 한다. 일반적인 분할 기준은 다음과 같이 공식화된다.

  • Core 0 (System & Network Plane): 운영체제의 백그라운드 데몬, 하드웨어 인터럽트 핸들러, 그리고 텔레메트리 송수신과 같은 DDS 계층의 일반적인 네트워크 소켓 I/O 처리에 전담된다.
  • Core 1 (Real-time Control Plane): 1000Hz 이상의 관성측정장치(IMU) 데이터 융합, 모터 믹싱, PID/MPC 자세 제어를 관장하는 커스텀 또는 단일 스레드 이그제큐터(Single-Threaded Executor) 전용 코어. 운영체제 레벨에서 isolcpus로 격리하여 비전 스레드의 이주(Migration)를 하드웨어 레벨에서 봉쇄한다.
  • Core 2 & 3 (Vision & Planning Plane): 이미지 콜백 처리, 객체 인식 추론(Inference), 궤적 최적화 연산이 수행되는 다중 스레드 이그제큐터(Multi-Threaded Executor) 공간.

3. 하드웨어 가속기(NPU/GPU) 오프로딩과 메모리 격리

비전 워크로드의 CPU 코어 할당 제한 정책을 보완하기 위해서는 영상 처리의 주요 연산을 하드웨어 가속기 구조로 전이시켜야 한다. 합성곱 신경망(CNN)이나 특징점 추출(Feature Extraction) 알고리즘은 NPU 또는 GPU 단으로 오프로드(Offloading)되어야 한다.

이때 ROS2 비전 노드는 원시 데이터가 메인 CPU의 L1/L2 캐시를 오염시키지 않도록, DMA(Direct Memory Access)를 활용하여 카메라 센서 모듈에서 하드웨어 가속기의 독립된 메모리 영역(Shared Memory Pool)으로 제로 카피(Zero-Copy) 버퍼를 전송하는 V4L2 기반의 파이프라인을 구축해야 한다. 이러한 복합적인 분할 전략만이 머신비전 파이프라인이 유도하는 시스템 부하로부터 물리적인 비행 제어 연산을 완벽하게 샌드박싱(Sandboxing)하는 표준 설계 방법론으로 규정된다.
