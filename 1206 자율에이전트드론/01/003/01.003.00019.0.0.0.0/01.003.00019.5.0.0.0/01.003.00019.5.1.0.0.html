19.5.1 CPU Affinity 속성을 통한 캐시 적중률 극대화

자율 에이전트 드론에 탑재되는 현대의 임베디드 멀티코어 프로세서(예: ARM Cortex-A 시리즈 기반 SoC)는 계층적인 메모리 구조를 지닌다. 프로세서 코어와 주 메모리(RAM) 사이에 존재하는 L1, L2, L3 캐시(Cache)는 CPU의 연산 속도와 메인 메모리의 접근 속도 차이를 보간하는 핵심 버퍼 역할을 수행한다. ROS2(Robot Operating System 2)의 고주파 비행 제어 루프를 최적화함에 있어, 운영체제의 스레드 마이그레이션(Thread Migration)을 통제하고 캐시 적중률(Cache Hit Ratio)을 극대화하기 위한 CPU 친화도(CPU Affinity) 속성의 할당은 필수적인 시스템 튜닝 기법이다.

1. 스케줄러 마이그레이션과 캐시 미스(Cache Miss) 페널티

리눅스(Linux)를 포함한 다중 작업(Multi-tasking) 운영체제의 기본 스케줄러는 시스템의 전체적인 부하 분산(Load Balancing)을 달성하기 위해, 실행 중인 스레드를 유휴 상태의 다른 물리적 코어로 빈번하게 이주시킨다(Migration).

어떤 스레드가 ’Core 0’에서 연산을 수행하는 동안, 해당 스레드의 명령어(Instruction) 및 데이터 컨텍스트(자세 제어 행렬, 칼만 필터 예측 변수 등)는 ’Core 0’에 종속된 L1/L2 캐시에 적재(Warm-up)된다. 그러나 스케줄러에 의해 이 스레드가 ’Core 1’로 이주하게 되면, ’Core 1’의 로컬 캐시에는 해당 스레드의 데이터가 존재하지 않는 콜드 캐시(Cold Cache) 상태에 직면하게 된다.

결과적으로 스레드는 필요한 연산 데이터를 느린 메인 메모리 리소스나 상위 공유 캐시(L3)로부터 다시 페치(Fetch)해야 하며, 이 과정에서 발생하는 극심한 역동적 캐시 미스(Dynamic Cache Miss)는 실시간성을 파괴하는 수십에서 수백 마이크로초(\mu s)의 지터(Jitter)로 누적된다.

2. CPU Affinity 적용과 메모리 지역성(Memory Locality) 보존

CPU Affinity(또는 Processor Affinity)는 특정 스레드나 프로세스가 실행될 수 있는 논리적 CPU 코어의 집합을 운영체제 레벨에서 강제로 제한하는 속성이다. ROS2 개발자는 C++ 기반의 POSIX 스레드 API인 pthread_setaffinity_np를 호출하거나, 실행(Launch) 단계에서 taskset 유틸리티를 적용하여 이그제큐터(Executor) 스레드의 마스크(Mask)를 특정 단일 코어로 핀(Pinning)할 수 있다.

스레드 피닝(Thread Pinning)이 타겟 코어에 확립되면, 스케줄러의 부하 분산 로직이 해당 스레드에 대해서 무효화된다. ROS2의 비행 제어 이그제큐터가 단일 코어에 고정됨에 따라 다음과 같은 성능적 이득률이 발생한다.

  1. 시간적 지역성(Temporal Locality)의 극대화: 500Hz 이상의 짧은 간격으로 지속적으로 호출되는 PID 제어 콜백의 알고리즘 인스트럭션과 센서 버퍼 주소들이 L1 및 L2 캐시 라인(Cache Line)에서 축출(Eviction)되지 않고 영구적으로 유지될 확률이 비약적으로 상승한다.
  2. 데이터 플로우파이프라인 이점: 동일 코어 내에서의 연속적 실행은 분기 예측기(Branch Predictor)의 훈련 상태를 보존하고 파이프라인 스톨(Pipeline Stall) 현상을 선제적으로 억제한다.

3. 시스템 아키텍처 관점의 도메인 매핑

드론 시스템의 컴퓨팅 효율을 지배하는 규칙은 “올바른 태스크를 올바른 코어에 배치하는 것“이다. 이는 단순히 캐시 적중률을 높이는 것을 넘어 이기종 코어의 HMP(Heterogeneous Multi-Processing) 하드웨어적 물성, 즉 big.LITTLE 아키텍처 특성에 완벽히 부합해야 한다. 제어 스레드는 마이크로아키텍처 관점에서 클럭 속도와 캐시 용량이 가장 큰 성능 코어(P-Core, Performance Core)에 Affinity가 부여되어야 하며, 로깅 체계나 텔레메트리 에이전트는 전력 효율 코어(E-Core)로 분리되어야 컴퓨팅 파워와 데이터 버스의 병목 없이 궁극적인 캐시 효율화 토폴로지가 완성된다.
