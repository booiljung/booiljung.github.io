19.4.3 커스텀 우선순위 스케줄러 구현 및 목표 지연 시간 모델링

비행 제어 시스템의 극단적인 실시간성(Hard Real-time) 요구사항을 충족하기 위해서는, ROS2(Robot Operating System 2)의 단순한 이벤트 트리거 및 폴링(Polling) 방식을 넘어, 제어 공학적 지연 시간 모델링(Latency Modeling)에 수학적으로 기반한 커스텀 우선순위 스케줄러(Custom Priority Scheduler)의 구현이 필수적이다. 이는 시스템 내의 모든 태스크(Task)가 마감 시간(Deadline) 이내에 반드시 실행을 완료함을 보장하는 근본적인 소프트웨어 아키텍처이다.

1. 커스텀 우선순위 스케줄러(Custom Priority Scheduler) 구현 전략

일반적인 FIFO(First-In-First-Out) 디스패칭의 비선점형 역전 현상을 해결하기 위해, 커스텀 이그제큐터는 콜백(Callback)의 의미론적(Semantic) 특성에 따른 계층화된 실행 체계(Hierarchical Execution Architecture)를 설계해야 한다.

  1. 우선순위 큐(Priority Queue) 기반 구조체 도입: 이그제큐터 루프 내부의 대기열(WaitSet 혹은 Event Queue)을 단일 큐가 아닌 다중 우선순위 큐 배열로 분리한다. 콜백이 활성화되면 스케줄러는 해당 콜백에 매핑된 CallbackGroup의 정적 우선순위(Static Priority)에 맞추어 특정 계층의 큐에 푸시(Push)한다.
  2. 비율 단조 스케줄링(RMS: Rate-Monotonic Scheduling) 알고리즘 매핑: 비행 제어 워크로드의 특성상 정기적으로 반복되는 주기적(Periodic) 콜백이 주를 이룬다. 따라서 실행 주기(Period)가 짧은 고주파 센서 콜백(예: 1000Hz IMU 제어)에 정적 최상위 우선순위를 부여하고, 주기가 긴 콜백(예: 10Hz 상태 퍼블리셔)에 낮은 우선순위를 부여하는 RMS 알고리즘을 이그제큐터 디스패치 레벨에 소프트웨어적으로 매핑한다.

2. 목표 지연 시간 모델링 (Target Latency Modeling)

커스텀 스케줄러의 타당성을 입증하고 안전 여유(Safety Margin)를 증명하기 위해서는, 운영체제(OS) 수준의 선점 모델과 단일 스레드 이그제큐터 내부의 비선점(Non-preemptive) 실행 모델의 특성을 결합한 수학적 응답 시간 모델링이 뒤따라야 한다.

어떤 태스크 i (특정 콜백)의 최악 응답 시간(WCET: Worst-Case Response Time) R_i 모델링은 다음 수식을 기저로 한다.

R_i = C_i + B_i + I_i

  • C_i (순수 실행 시간, Execution Time): 콜백 알고리즘(예: 자세 제어 PID 연산) 자체가 CPU 코어에서 실행되는 고유 시간이다.
  • B_i (블로킹 지연 시간, Blocking Time): 기본 ROS2 모델에서는 다른 하위 우선순위 콜백이 실행 중일 경우 선점 불가 특성으로 인해 대기해야 하는 최대 시간이다. 커스텀 스케줄러에서는 이를 방지하기 위해 이그제큐터 분리 정책을 차용하므로, B_i는 Mutex 락이나 하부 미들웨어(DDS) 인터럽트 딜레이로 한정하여 상한치(Upper Bound)를 계산한다.
  • I_i (간섭 대기 시간, Interference Time): 상위 우선순위를 갖는 태스크들에 의해 선점되어 연산이 지연되는 시간의 총합이다. \sum \lceil \frac{R_i}{T_j} \rceil C_j (여기서 T_j는 상위 태스크 j의 주기) 형태로 산출 전개된다.

실시간 제어 스케줄러의 무결성 검증

비행 제어기의 스케줄러 설계는 주어진 하드웨어 타겟(Target Hardware)에서 모델링된 모든 콜백 태스크 i에 대하여, 최악 응답 시간 R_i가 허용된 데드라인 D_i 안에 존재하는지(R_i \le D_i)를 정량적으로 증명하는 과정을 수반한다.

목표 지연 시간 모델링을 통해 스레드 오버슈트(Thread Overshoot)와 콜백 기아(Starvation) 현상이 수학적으로 배제된다면, 해당 커스텀 스케줄러는 자율 드론의 HIL(Hardware-in-the-Loop) 시뮬레이션 및 실기체(Real Flight) 환경에서 극심한 통신 병목이나 연산 과부하 상황에서도 자세 제어 루프의 결정론성(Determinism)을 유지할 수 있는 견고한 시스템 코어로 작용할 수 있다.
