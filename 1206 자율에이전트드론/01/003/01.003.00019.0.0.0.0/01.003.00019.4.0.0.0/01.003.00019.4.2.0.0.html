19.4.2 EventsExecutor 트리거 기반 인터페이스 성능 벤치마킹

ROS2(Robot Operating System 2)의 실시간성 한계를 극복하기 위해 제안된 대표적인 대안 아키텍처 중 하나는 이벤트 구동(Event-Driven) 방식의 EventsExecutor이다. 네이티브 이그제큐터가 O(N)의 시간 복잡도를 갖는 폴링(Polling) 방식의 WaitSet에 의존하는 것과 달리, EventsExecutor는 통신 미들웨어가 생성하는 트리거(Trigger)를 능동적으로 수신하는 구조를 채택하여 스케줄링 오버헤드를 근본적으로 재설계하였다.

1. 트리거 기반(Trigger-based) 푸시(Push) 아키텍처

EventsExecutor의 핵심 메커니즘은 인터럽트 방식과 유사한 콜백 트리거링(Callback Triggering)이다. DDS(Data Distribution Service) 계층에서 네트워크 패킷이 수신되어 처리가 완료되면, 미들웨어는 이그제큐터가 폴링할 때까지 대기하지 않고 즉각적으로 이그제큐터 내부의 스레드 안전한(Thread-safe) 이벤트 큐(Event Queue)에 알림(Notification)을 푸시(Push)한다.

이러한 수동적 폴링(Passive Polling)에서 능동적 푸시(Active Push)로의 패러다임 전환은 이그제큐터의 스케줄링 복잡도를 전체 엔티티 총량인 N이 아닌, 실제 활성화된 이벤트의 수인 E에 비례하게 만든다. 즉, 스케줄링 시간 복잡도가 O(N)에서 O(E)로 극적으로 감소하며, 매 사이클마다 WaitSet을 동적으로 파괴하고 재구성하는 데 소모되던 메모리 재할당 비용을 배열 스택 연산으로 대체한다.

2. 자율 드론 환경 가상의 벤치마킹 분석 모델

자율 에이전트 드론과 같이 노드 팽창이 극심한 시스템(엔티티 N > 200, 동시 다발적 활성화 이벤트 E \ll N)에서의 벤치마킹 모델은 EventsExecutor의 도입 효과를 명확하게 지시한다.

  1. CPU 점유율(CPU Utilization) 지표: 네이티브 SingleThreadedExecutor는 1000Hz 제어 루프 환경에서 빈번한 WaitSet 갱신으로 인해 단일 코어 점유율이 40% 이상으로 치솟을 수 있다. 반면 EventsExecutor는 대기 큐에서 이벤트가 꺼내어질(Pop) 때만 CPU 사이클을 소비하므로 스케줄링 자체에 소진되는 베이스라인 점유율을 5% 미만으로 억제한다.
  2. 응답 지연 시간(Response Latency) 지표: 500Hz의 관성측정장치(IMU) 토픽과 다수의 고용량 이미지 메시지가 혼재된 상황에서, 기존 이그제큐터의 꼬리 지연(Tail Latency, 99번째 백분위수)은 WaitSet 스캔 지연에 의해 수 ms 단위로 요동친다. 반면 트리거 인터페이스하에서는 이벤트가 큐에 삽입되는 순서대로 즉각적인 문맥 전환(Context Switching)이 가능해져 지연 시간의 상한선(Bounded Latency) 내 분포가 개선된다.

3. 실시간 제어 모델 적용을 위한 학술적 시사점

성능 벤치마킹 결과는 EventsExecutor가 WaitSet의 폴링 병목을 성공적으로 제거하여 평균적인 시스템 응답성을 크게 향상시킴을 입증한다. 그러나 자율 드론의 비행 제어 논리(Flight Control Logic) 관점에서 볼 때, 이벤트 구동 방식 자체만으로는 완벽한 실시간성을 담보할 수 없다는 한계점 또한 내포하고 있다.

EventsExecutor의 이벤트 큐 역시 기본적으로는 선입선출(FIFO) 방식의 큐잉 모델을 따르기 때문이다. 즉, 이벤트 발생 빈도가 폭증하는 혼잡(Congestion) 상태에서는 먼저 도착한 저우선순위 이미지 데이터 이벤트가 나중에 도착한 고우선순위 자세 제어 이벤트를 차단(Blocking)하는 현상이 여전히 유효하다. 따라서 진정한 의미의 실시간 제어 루프를 완성하기 위해서는, EventsExecutor의 효율적인 O(E) 트리거 인프라 위에 이벤트의 의미론적 중요도(Semantic Criticality)에 기반한 엄격한 우선순위 스케줄링(Priority Scheduling) 알고리즘을 융합하는 커스텀 아키텍처 확장이 필수적으로 요구된다.
