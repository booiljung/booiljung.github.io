16.5.2 활성 목표와 신규 목표 간 우선순위 스케줄링 전략

1. 개요

자율 에이전트 드론이 다중 임무를 수행하는 환경에서는 현재 실행 중인 활성 목표(Active Goal)와 이벤트 기반으로 새로 발행된 신규 목표(New Goal) 간의 자원 경합이 발생한다. ROS2 액션 서페이스(Action Surface)는 클라이언트의 비동기적 요청을 다루기 위한 유연한 통신 패턴을 제공하지만, 내부적으로 어떤 목표를 먼저 처리할 것인지 결정하는 엄격한 스케줄링 전략이 요구된다. 본 절에서는 드론의 시스템 안전성과 임무 달성 효율성을 동시에 만족하기 위한 목표 우선순위 스케줄링(Priority Scheduling) 아키텍처와 결정론적 평가 메커니즘을 서술한다.

2. 목표 큐(Goal Queue) 기반 스케줄링 메커니즘

액션 서버는 다수의 클라이언트로부터 수신되는 목표 요청을 관리하기 위해 내부적으로 목표 큐(Goal Queue)를 유지한다. 단순한 선입선출(FIFO) 방식은 드론의 실시간 제어 요구사항을 충족시키지 못하므로, 각 목표 요청 구조체에 명시된 우선순위 메타데이터(Priority Metadata)를 기반으로 작동하는 우선순위 큐(Priority Queue)가 도입된다.

큐 관리자는 새로운 목표가 수신될 때마다 현재 활성 상태인 목표의 우선순위 값과 신규 목표의 우선순위 값을 비교한다. 이 비교 연산은 단순한 상수 비교가 아니라, 임무의 잔여 시간, 목표 위치까지의 거리, 드론의 배터리 상태 등 동적 변수들을 포함하는 다차원 가중치 함수로 구현된다.

3. 선점형(Preemptive) 및 비선점형(Non-preemptive) 스케줄링

활성 목표와 신규 목표 간의 스케줄링 전략은 크게 선점형 정책과 비선점형 정책으로 분류된다.

  • 선점형(Preemptive) 스케줄링: 신규 목표의 우선순위가 현재 활성 목표보다 절대적으로 높을 경우(예: 충돌 회피, 배터리 부족으로 인한 긴급 귀환), 액션 서버는 즉시 현재 목표를 취소(Cancel) 상태로 전환하고 신규 목표에 자원을 할당한다. 이 방식은 반응성(Responsiveness)이 매우 뛰어나 안전 필수(Safety-Critical) 상황에 적합하다.
  • 비선점형(Non-preemptive) 스케줄링: 신규 목표의 우선순위가 활성 목표보다 높더라도 특정 임계 조건을 만족하지 못하는 경우(예: 현재 목표 달성까지 1초 미만 등), 큐 관리자는 현재 목표가 완료(Succeed)되거나 중단(Aborted)될 때까지 신규 목표를 큐에서 대기(Pending) 상태로 유지한다. 이는 불필요한 문맥 교환으로 인한 제어 오버헤드를 막고 제어 궤적의 안정성을 보장한다.

4. 문맥 교환(Context Switch) 비용과 평가 함수

목표가 전환될 때는 물리적인 제어 지연과 컴퓨팅 자원 소모를 포함하는 문맥 교환 비용(Context Switch Cost)이 발생한다. 스케줄링 전략은 이 비용을 모델링하여 평가 함수(Evaluation Function)에 반영해야 한다.

선점 조건은 수학적으로 다음 부등식을 만족할 때 발동된다.

P_{new} > P_{active} + C_{switch}

여기서 P_{active}는 활성 목표의 최적화된 우선순위, P_{new}는 신규 목표의 최적화된 우선순위, C_{switch}는 문맥 교환 및 제어 재설정에 필요한 오버헤드 비용이다. 이러한 정량적 평가를 통해 잦은 우선순위 변동으로 인해 아무런 목표도 달성하지 못하고 시스템이 진동하는 상태 편향성(Thrashing)을 방지할 수 있다.
