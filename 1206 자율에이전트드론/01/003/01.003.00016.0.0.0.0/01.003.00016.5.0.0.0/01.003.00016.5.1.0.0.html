16.5.1 장애물 감지 인터럽트 시 능동적 액션 선점 모델

1. 개요

자율 에이전트 드론의 비행 중 갑작스러운 장애물의 출현은 시스템의 생존성과 직결되는 중대한 환경 변화이다. 이러한 비정형적이고 동적인 기하학적 장애물에 대응하기 위해서는 단순한 궤적 수정이 아닌, 현재 수행 중인 모든 임무 모듈을 중단시키고 최우선순위의 회피 기동을 즉각적으로 실행하는 능동적 액션 선점(Active Action Preemption) 모델이 요구된다. 본 절에서는 센서 데이터 기반의 장애물 인터럽트가 발생했을 때, ROS2 액션 서버 내에서 기존 목표를 어떻게 선점하고 새로운 회피 목표로 제어권을 전이하는지에 대한 구조적 모델을 분석한다.

2. 장애물 감지 인터럽트(Interrupt) 구조

장애물 감지는 주로 라이다(LiDAR), 심도 카메라(Depth Camera), 그리고 레이더(Radar) 등의 다중 센서 융합을 통해 이루어진다. 시스템은 센서 데이터를 고주파수로 샘플링하며, 설정된 안전 반경(Safety Radius) 내에 미확인 객체가 진입할 경우 즉각적인 하드웨어 인터럽트를 발생시키거나 최우선순위의 메시지를 발행(Publish)한다.

이때 발생하는 인터럽트는 ROS2의 이벤트 루프 내에서 가장 높은 QOS(Quality of Service) 정책이 적용된 토픽으로 전달된다. 이를 수신한 임무 제어 노드는 지연 시간(Latency)을 최소화하기 위해 전용 이그제큐터(Executor)와 콜백 그룹(Callback Group)을 통해 즉시 선점 요청 절차에 돌입한다.

3. 능동적 액션 선점 아키텍처

기존의 수동적 선점 방식이 주기적인 상태 피드백 루프에서 목표 변경을 확인하는 구조라면, 능동적 선점 모델은 이벤트 구동형(Event-driven) 방식으로 설계된다.

  • 즉각적 목표 취소(Immediate Goal Cancellation): 장애물 감지 알림이 임무 스케줄러(Action Client)에 도달하면, 스케줄러는 현재 활성화된 목표(Active Goal)를 취소하는 CancelGoal 비동기 서비스 요청을 액션 서버로 전송한다.
  • 회피 목표 주입(Evasion Goal Injection): 취소 요청과 거의 동시에, 국지 경로 탐색기(Local Path Planner)가 생성한 긴급 회피 궤적을 새로운 목표로 하는 SendGoal 요청이 전송된다. 액션 서버는 두 요청을 연속적으로 수신하며, 정의된 우선순위 규칙에 따라 기존 목표를 강제 종료하고 회피 목표를 수락한다.

4. 동적 제어 전이(Control Handoff) 과정

능동적 선점 모델이 성공적으로 작동하기 위해서는 모터 제어 명령의 연속성을 보장해야 한다. 급격한 제어권 변화는 드론의 동역학적 불안정성을 유발할 수 있으므로, 상태 머신 내에서 부드러운 전이(Smooth Transition)를 위한 완충 로직이 포함된다.

액션 서버가 기존 목표를 ‘Canceled’ 상태로 변경함과 동시에, 현재의 속도 및 가속도 벡터를 초기 조건으로 삼아 새로운 회피 궤적 제어 루프를 시작한다. 이 과정에서 PID 제어기나 MPC(Model Predictive Control) 제어기의 적분항 제한(Anti-windup) 및 초기화 기법이 적용되어 제어 신호의 튀는(Spike) 현상을 억제한다.
