16.5.3 부분 임무 완료 상태 허용 및 복귀 분기점 정책 평가

1. 개요

자율 에이전트 드론의 장기 실행 임무 중 목표 선점이나 시스템 인터럽트가 발생할 경우, 기존 임무 전체를 실패(Aborted)로 간주하는 것은 임무 효율성과 자원 활용 측면에서 비합리적이다. ROS2 액션 서버 생태계에서는 목표가 끝까지 수행되지 않았더라도, 현재까지 달성한 작업의 유효성을 인정하는 부분 임무 완료(Partial Mission Completion) 상태를 허용하는 구조가 요구된다. 본 절에서는 부분 완료 상태의 정의와 함께, 중단된 위치에서 임무를 재개할 것인지 혹은 기지(Base)로 귀환할 것인지를 결정하는 복귀 분기점(Bifurcation Point) 정책의 평가 모델을 서술한다.

2. 부분 완료(Partial Completion) 상태의 정의 및 필요성

일반적인 ROS2 액션 인터페이스는 Succeeded, Canceled, Aborted의 세 가지 최종 상태(Terminal State)를 갖는다. 그러나 광활한 지역의 매핑이나 복수 지점의 화물 배송과 같은 다단계 임무에서는 Canceled 상태가 반드시 작업의 무효화를 의미하지 않는다.

부분 임무 완료 상태 허용 아키텍처는 액션 서버가 선점 요청을 수락하여 목표를 취소할 때, 최종 피드백(Feedback) 및 결과(Result) 메시지에 ‘보존된 작업 비율(Preserved Work Ratio)’ 또는 ’완료된 체크포인트 식별자(Checkpoint ID)’를 포함시킨다. 이를 통해 임무 관리 스케줄러(Action Client)는 잔여 임무량을 정확히 파악하고, 재실행 시 오버헤드를 최소화할 수 있다.

3. 복귀 분기점(Return-to-Base Bifurcation Point) 정책 모델

부분 완료 상태로 임무가 중단된 이후, 드론은 신규 목표(예: 배터리 충전, 긴급 회피)를 수행한다. 이 신규 목표가 달성된 직후, 에이전트는 기지로 복귀할 것인지 아니면 중단되었던 부분 완료 지점으로 돌아가 잔여 임무를 수행할 것인지를 결정해야 한다. 이를 복귀 분기점 정책이라고 한다.

복귀 분기점의 결정은 주로 비용 함수(Cost Function)에 기반하여 평가된다.

  • 잔여 에너지(Residual Energy): 중단 지점까지의 이동, 잔여 임무 수행, 그리고 최종 귀환에 필요한 예상 소모 에너지가 드론의 배터리 잔량보다 낮아야 한다.
  • 임무 시간 제약(Mission Time Constraint): 잔여 임무의 유효 시간(Deadline) 내에 작업을 재개하고 완료할 수 있는지 평가한다.
  • 환경 상태 지수(Environmental State Index): 기상 조건이나 통신 링크 품질 등의 외부 환경 변수가 목표 재개 지점에서 안전한 비행을 보장하는지 판단한다.

4. 상태 전이 및 임무 재개(Mission Resumption) 평가 로직

복귀 분기점 평가 척도를 통과하여 임무 재개가 결정되면, 임무 스케줄러는 이전에 수신된 부분 완료 Result 데이터를 기반으로 새로운 Goal 메시지를 재구성한다. 재개되는 목표는 데이터셋의 첫 부분부터 추종하는 것이 아니라, 저장된 추정 상태 벡터(Estimated State Vector)를 기준으로 실행되는 재진입(Re-entry) 모델을 따른다.

이러한 상태 전이 로직은 추후 행동 트리(Behavior Tree) 아키텍처와 연계되어 복구 동작(Recovery Behavior) 및 조건 분기 수단으로 활용되며, 시스템의 운영 중단 시간을 최소화하고 임무 달성률을 극대화하는 핵심적인 역할을 수행한다.
