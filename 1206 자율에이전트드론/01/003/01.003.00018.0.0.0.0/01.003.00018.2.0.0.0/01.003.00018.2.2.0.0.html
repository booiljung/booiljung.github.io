18.2.2 읽기 전용 매개변수의 보안 분류 기준 및 계층형 락 체계

안전 필수(Safety-Critical) 제어 시스템에서 모든 파라미터가 동일한 민감도를 지니는 것은 아니며, 시스템 파괴 잠재력에 따라 엄격한 보안 등급으로 구획화되어야 한다. 자율 에이전트 드론의 런타임 아키텍처는 데이터의 불변성(Immutability) 요구 수준에 기반하여 파라미터를 정적 상수(Static Constant), 환경 의존적 설정(Environment-dependent Configuration), 런타임 가변 변수(Runtime Variable)로 분류하는 다층적 보안 체계를 요구한다. 특히 기체의 역학적 본질을 규정하는 최상위 민감도 데이터에 대해 읽기 전용(Read-Only) 제약을 강제하기 위한 핵심 기법으로 계층형 락(Hierarchical Lock) 체계가 적용된다.

1. 기계 물리적 제원의 정적 상수 맵핑

드론의 섀시 구조, 이륙 중량(Takeoff Weight)의 범위를 결정하는 로터의 기하학적 배치(Rotor Geometry), 모터 암(Motor Arm)의 길이, 관성 측정 장치(IMU)의 물리적 장착 각도 등은 런타임에 동적으로 변경될 수 없는 절대적인 기계-물리적 상수이다. 이러한 데이터 그룹은 제어 동역학 행렬(Dynamics Matrix) 계산의 근원 텐서(Source Tensor)로 사용되므로, 런타임 오염 시 즉각적인 센서 퓨전의 발산을 초래한다. 따라서 아키텍처 설계 단계에서 이 파라미터들은 최상위 등급의 보안 분류 객체로 지정되어, 노드(Node) 초기화 이후에는 그 어떠한 문맥(Context)에서도 수정이 불가한 읽기 전용(Read-Only) 메모리 세그먼트에 격리되어야 한다.

2. 초기화 권한의 자진 반납과 계층형 락 모델

읽기 전용 속성을 런타임에 보장하기 위해, ROS2 기반 파라미터 프레임워크는 초기화 권한과 런타임 인터페이스 권한을 분리하는 계층형 락(Hierarchical Lock) 모델을 도입한다. 시스템 부팅 단계에서 고도의 인증(Authentication)을 거친 최상위 관리자(Lifecycle Manager) 스레드만이 물리 상수를 파라미터 서버에 1회 적재(Load)할 수 있는 루트 권한(Root Privilege)을 획득한다. 초기화(Initialization)가 성공적으로 완료되어 노드가 활성(Active) 상태로 천이(Transition)하는 즉시, 이 관리자 스레드는 쓰기 권한을 자진 반납(Drop Privilege)한다. 이 지점을 기점으로 파라미터 딕셔너리에는 하드웨어 수준의 뮤텍스 락(Mutex Lock)이 걸리게 되며, 제어기를 포함한 시스템 내의 모든 구성 요소는 해당 파라미터 평면에 대해 오직 읽기(Read) 인터페이스만을 노출받게 된다.

3. 권한 상승 공격 억제 및 무결성 증명

이러한 권한 구획화(Privilege Compartmentalization) 구조는 사이버-물리 시스템(CPS) 관점에서 외부의 악의적 조작 및 내부 소프트웨어 버그에 대한 궁극적인 방어 기폭제가 된다. 만약 손상된 지상국 애플리케이션이나 논리적 오류에 빠진 내비게이션 노드가 읽기 전용 파라미터 인터페이스로 쓰기 요청(Write Request)을 전달하더라도, 권한이 이미 소멸된 파라미터 서버는 즉시 접근 거부(Access Denied) 예외를 발생시키고 해당 요청 큐를 폐기한다. 이 방어 메커니즘은 매개변수 주입을 활용한 권한 상승 유도나 버퍼 오버플로우 시도를 원천 차단하며, 드론의 역학 모델이 비행 종료 시점까지 수학적 무결성(Mathematical Integrity)을 완벽히 유지하고 있음을 아키텍처 레벨에서 증명한다.
