18.2.1 사전 선언 강제 정책 기반 런타임 메모리 안전성

ROS2를 비롯한 현대의 고신뢰(High-Reliability) 미들웨어 프레임워크는 소프트웨어의 런타임 궤적이 비결정론적으로 확장되는 것을 극도로 경계한다. 특히 파라미터는 외부 입력에 의해 그 자료구조의 형태나 개수가 언제든 변할 수 있는 본질적 가변성을 내제하고 있어, 비행 제어기와 같은 안전 필수(Safety-Critical) 계층에서 가장 위험한 메모리 불안정 요소로 분류된다. 이러한 불안정성을 원천 차단하기 위해 도입된 핵심 아키텍처가 “사전 선언 강제 정책(Declaration-enforced Policy)“이며, 이는 런타임 메모리 안전성(Runtime Memory Safety)을 보장하는 구조적 근간으로 기능한다.

1. 동적 메모리 할당의 폐해와 초기화 시점의 정적 고정

자유로운 런타임 매개변수 생성 환경에서는 새로운 변수가 추가되거나 타입이 변경될 때마다 힙(Heap) 영역에서 동적 메모리 할당(Dynamic Memory Allocation)과 해제가 반복적으로 발생한다. 실시간 운영체제(RTOS) 환경에서 이러한 비결정론적 메모리 연산은 가용 메모리의 단편화(Fragmentation)를 유발하고 예측 불가능한 처리 시간 지연순간(Spike)을 야기하여, 고주파수의 제어 루프를 마비시킬 수 있다. 반면 사전 선언 강제 정책 하에서는 노드(Node) 초기화 과정에서 시스템 내 존재하는 모든 파라미터의 타입, 식별자(이름), 저장 공간 크기가 확정(Commit)된다. 이로써 시스템 부팅이 완료되고 실제 비행 임무(Active State)가 시작된 이후에는 파라미터 처리에 있어 메모리 재할당 오버헤드가 단 1바이트도 발생하지 않는 완벽한 런타임 정적(Static) 환경이 구축된다.

2. 미선언 매개변수 주입 시도에 대한 예외 트랩(Exception Trap) 방어

사전 선언 정책은 논리적인 방화벽(Logical Firewall) 역할을 수행하여 외부의 비인가 데이터 주입을 물리적으로 거부한다. 지상국(GCS) 애플리케이션의 설정 오류나 악의적인 네트워크 간섭으로 인해 노드 내부에 선언되지 않은 임의의 파라미터(Undeclared Parameter)를 강제로 푸시(Push)하려는 시도가 인입될 수 있다. 이 경우, 파라미터 서버는 해당 변수를 무시하거나 메모리의 여유 공간에 임의로 적재하지 않고, 즉시 ’선언되지 않은 매개변수 접근 예외(Undeclared Parameter Access Exception)’를 발생시켜 해당 트랜잭션을 하드 로드 시점에 기각(Drop)한다. 이 메커니즘은 유령(Ghost) 변수에 의한 소프트웨어 교착 상태나 예상치 못한 분기(Branch) 로직의 활성화를 원천적으로 차단한다.

3. 지연 시간 제한(Bounded Latency) 및 결정론적 자원 증명

또한, 런타임 도중 파라미터 해시 맵(Hash Map)이나 레지스트리의 크기가 절대 증가하지 않는다는 불변의 제약은 파라미터 조회(Lookup) 알고리즘의 최악 사례 지연 시간(Worst-Case Execution Time, WCET)을 기하급수적으로 낮춘다. 파라미터 트리의 깊이와 총 노드 수가 선언 시점에 고정되므로, 제어기가 수시로 파라미터를 호출하더라도 그 조회 주기는 항상 상한선이 있는 O(1) 혹은 O(\log n) 시간 복잡도를 엄격히 유지하게 된다. 즉, 사전 선언 강제 정책은 런타임 시스템이 요구하는 메모리 풋프린트 스페이스(Memory Footprint Space)와 검색 시간 지연(Search Time Latency)의 최대 한계치를 시스템 부팅 전에 수학적으로 증명(Deterministic Allocation Proof)할 수 있게 만드는 필수적인 학술적 전제 조건이다.
