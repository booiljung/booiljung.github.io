18.1.2 ROS1 전역 파라미터 서버의 동시성 제어 한계 및 상태 불일치

장기간 로봇 공학의 표준으로 자리 잡았던 ROS1 아키텍처는 노드 간의 느슨한 결합(Loose Coupling)을 지향하지만, 파라미터 관리 체계만큼은 마스터(Master) 노드에 의존하는 강력한 중앙 집중형 구조를 채택하였다. ROS1의 전역 파라미터 서버(Global Parameter Server)는 네트워크 상의 모든 노드가 접근 가능한 전역 딕셔너리(Global Dictionary)의 형태로 XMLRPC 프로토콜을 기반으로 구동된다. 이는 개발의 편의성과 시스템의 전역적 가시성을 도모한다는 장점이 있었으나, 복잡한 비행 동역학을 다루는 자율 에이전트 드론의 안전 필수(Safety-Critical) 환경에서는 치명적인 아키텍처적 결함을 내재하고 있었다.

1. 비동기 환경 하의 동시성 제어 부재 및 경합 조건

ROS1 전역 파라미터 서버의 가장 결정적인 한계는 멀티스레드 멀티프로세스 환경을 위한 엄격한 동시성 제어(Concurrency Control) 메커니즘의 부재에서 기인한다. 복수의 노드—예를 들어 궤적 생성 노드와 오프보드 제어(Offboard Control) 노드—가 동일한 비행 고도 제한 파라미터를 동시에 갱신하려 시도할 경우, 트랜잭션을 보장하는 락킹(Locking) 기법이나 원자적(Atomic) 연산이 지원되지 않는다. 이로 인해 시스템은 필연적으로 경합 조건(Race Condition)에 노출되며, 한 노드의 업데이트가 다른 노드의 쓰기 작업에 의해 묵시적으로 덮어씌워지거나, 다차원 매개변수 배열 중 일부만 갱신되는 부분 갱신(Partial Update) 오류가 발생한다. 결과적으로 시스템 전반에 걸쳐 파라미터의 정보 비대칭성이 심화되고, 제어 루프는 손상된 입력 벡터를 바탕으로 제어 명령을 출력하게 된다.

2. 로컬-전역 상태 불일치와 스플릿 브레인(Split-Brain) 현상

전역 파라미터 서버 방식은 네트워크 지연이나 토폴로지 단절 상황에서 상태 불일치(State Inconsistency)를 기하급수적으로 악화시킨다. ROS1 노드들은 성능 개선을 위해 전역 파라미터를 로컬 메모리에 캐싱(Caching)하는 기법을 종종 활용하는데, 중앙 마스터에서 특정 파라미터가 변경되더라도 이 변경 사항이 즉각적이고 확정적으로 모든 로컬 캐시에 동기화된다는 보장이 없다. 무선 통신의 핑 파이크(Ping Spike)나 일시적 패킷 손실이 개입될 경우, 비행 제어 프로세스는 갱신 전의 레거시(Legacy) 파라미터를 참조하는 반면 충돌 회피 모듈은 갱신된 파라미터를 참조하는 소위 ‘스플릿 브레인(Split-Brain)’ 현상이 런타임에 발현된다. 두 주요 모듈이 서로 상충되는 물리적 기준을 가지고 동작함으로써 드론은 심각한 역학적 충돌을 자초하게 된다.

3. 단일 장애점(Single Point of Failure)으로서의 마스터 아키텍처

안전 필수 시스템 공학 관점에서 ROS1 마스터 노드에 종속된 파라미터 서버는 가장 전형적이고 위협적인 단일 장애점(Single Point of Failure, SPOF)이다. 마스터 노드 프로세스 자체가 다운되거나 통신 대역폭이 포화될 경우, 전체 제어망의 파라미터 질의 및 갱신 프로세스는 즉각적으로 교착 상태(Deadlock)에 빠지게 된다. 이는 시스템 리커버리나 파라미터 비상 롤백을 수행해야 할 최악 사례(Worst-Case) 상황에서, 정작 롤백 메커니즘을 관장해야 할 구성 평면 자체가 응답 불능(Unresponsive) 상태에 빠진다는 모순을 낳는다. 이러한 구조적 취약성은 ROS1이 현대의 고신뢰 군집 비행이나 도심항공교통(UAM) 실시간 미들웨어로 존속할 수 없는 근본적 이유를 명백히 입증한다.
