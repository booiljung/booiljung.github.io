15.3.3 액추에이터 계층: 모터 아밍(Arming) 안전 검증과 상태 전이 결합

자율 에이전트 드론에서 모터와 전자 변속기(ESC)를 포함하는 액추에이터 계층은 소프트웨어의 출력 산출물이 물리적인 기체의 추력과 직접적으로 직결되는 최말단 시스템이다. 액추에이터 제어 명령을 담당하는 ROS2 컴포넌트가 Active 상태로 전이되어 제어 신호 퍼블리시를 시작하는 것은 곧 모터의 회전을 의미하므로, 이 계층의 생명주기 관리에는 엄격한 하드웨어 안전 잠금 해제, 즉 아밍(Arming) 절차가 결합되어야 한다. 본 절에서는 LifecycleNode의 상태 전이 메커니즘과 하위 비행 제어기(Flight Controller)의 아밍 프로세스를 논리적, 시퀀스적으로 결합하는 방안을 기술한다.

1. 액추에이터 노드의 암묵적/명시적 상태 매핑

드론의 물리적 구동을 관장하는 제어 인터페이스 노드(예: MAVROS 커맨드 퍼블리셔, CAN 믹서 노드)의 ROS2 Lifecycle 상태는 비행 제어기의 하드웨어적인 안전 상태(Safety State)와 수동적으로 동기화된다.

  • Unconfigured 단계: 시스템 부팅 직후, 액추에이터 노드가 메모리에 적재되었으나 펌웨어와의 통신 링크 설정이나 포트 매핑 자체를 규정하지 못한 상태이다.
  • Inactive (Disarmed 대기) 단계: on_configure 전이 콜백을 통과한 후, 노드는 비행 제어기와 데이터 송수신 채널을 확립한다. 그러나 노드가 Inactive 기본 상태에 머무는 동안은 로터에 제어 설정값(Setpoint)을 인가하는 퍼블리셔 인터페이스가 프레임워크 수준에서 원천 차단(Deactivated)된다. 이 상태는 소프트웨어는 대기 중이나, 물리적인 비행 제어기는 여전히 안전 모드인 디스암(Disarmed) 상태를 유지하는 것과 등가(Equivalent)로 설계된다.
  • Active (Armed 활성화) 단계: Activating 전이 단계의 on_activate 콜백이 성공적으로 완료되면 궤적 추적기(Trajectory Tracker) 및 기하학적 제어기(Geometric Controller)로부터 내려온 제어 설정값이 미들웨어를 거쳐 직접 하드웨어로 전달된다. 노드의 Active 진입은 드론 파워트레인의 물리적 활성화(Arming)를 공식적으로 허가하는 선언이다.

2. 상태 전이(Transition)와 시스템 안전 잠금(Interlock)의 결합 로직

액추에이터 계층의 컴포넌트가 Activating 전이 상태를 거칠 때, 시스템 소프트웨어 아키텍처는 단순한 포인터 체인의 개방뿐만 아니라 드론 인프라 전체의 안전성 검증(Safety Interlock Pre-condition Check)을 원자적으로 수행해야 한다.

  • 전제 조건 점검 (Pre-Arm System Checks): on_activate 콜백 내부에서 최우선적으로 실행되는 로직은 선행 의존 노드들의 상태 모니터링이다. 글로벌 Lifecycle 관리자 노드의 API를 경유하여 IMU, GPS, 상태 추정기 등 비행에 핵심적인 필수 노드들이 모두 결함 없이 Active 상태에 도달했는지 교차 점검한다. 단 하나의 구성요소라도 장애가 발생하여 Inactive 상태로 강등되어 있다면, 아밍 전이 요청은 즉시 FAILURE를 반환하고 철회된다.
  • 물리적 아밍 커맨드 송신 및 확증(Acknowledgment): 상기 소프트웨어적 및 네트워크 레이턴시 검증이 모두 통과하면, on_activate 콜백 루틴의 마지막 시퀀스에서 비행 제어기 펌웨어 하드웨어 인터페이스 측으로 물리적 아밍 커맨드(예: MAV_CMD_COMPONENT_ARM_DISARM)를 시리얼 버스 등을 통해 송신한다. 이후 비행 제어기 보드로부터 모터 아밍 성공 루틴에 대한 명백한 ACK 응답(Response) 패킷을 확인한 직후에만 콜백 반환값 SUCCESS를 전파시켜 ROS2 계층의 상태를 Active로 승격시킨다.

3. 예외 상황 처리 및 비상 정지(Disarming) 컴플라이언스

안전 관리가 필수적인 시스템(Safety-Critical System)에서, 정상적인 Active 상태로 가동되던 액추에이터 노드가 하드웨어 펄스 결함이나 배터리 컷오프 등의 치명적인 위험성을 감지했을 때 발생하는 Deactivating 전이는 즉각적이고 결정론적(Deterministic)이어야 한다.

  • on_deactivate를 결합한 비상 하드웨어 제어: 네트워크 분산 환경에서 지상 관제소 컴포넌트나 충돌 예방 노드 등으로부터 페일 세이프(Fail-Safe) 이벤트 플래그를 수신하거나, 이그제큐터(Executor) 그룹의 워치독 타임아웃이 발생하면, 액추에이터 노드는 무조건적인 최우선 순위로 Deactivating 상태 진입 강제 명령을 수신한다. 이 구간 내에서는 모든 평시 퍼블리셔의 권한을 즉시 차단함과 동시에, 비행 제어기로 디스암(Disarm) 커맨드를 하드웨어 인터럽트로 발신하여 물리적인 추력 생성을 차단한다.
  • 단방향 비가역적 전이 설계(Irreversible Transition Design): 비상 위험 상황으로 인해 Deactivating 단계를 거쳐 Inactive로 복원된 시스템 구동계는, 문제의 근본적인 원인이 하드웨어 리셋 및 오프보드(Off-board) 진단기 레벨에서 클리어되어 전체 글로벌 오케스트레이션 관리자에 의해 명시적 락-해제 이벤트가 인가될 때까지 소프트웨어적인 재아밍(Activating) 전이가 거부되도록 하드 클로징(Hard-closing) 기법을 설계하여 2차 사고를 방지한다.
