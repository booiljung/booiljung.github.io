15.4.1 서브시스템 의존성 기반 순차적 기동 그래프 설계

자율 에이전트 드론 시스템은 다수의 독립적인 서브시스템(Subsystem)이 결합된 형태를 띠며, 각 서브시스템은 데이터 흐름과 제어 로직에 따라 명확한 선후행 의존성을 지닌다. 안정적인 비행 시스템의 기동(Startup)을 위해서는 이러한 의존성을 철저히 보장하면서 ROS2 노드들의 생명주기(Lifecycle) 상태 전이를 체계적으로 스케줄링하는 순차적 기동 그래프(Sequential Startup Graph) 설계가 필수적이다.

1. 서브시스템 간 의존성 모델링

비행 제어 아키텍처 내에서 각 서브시스템은 독립된 데이터 퍼블리셔(Publisher)와 서브스크라이버(Subscriber) 구조를 통해 상호작용한다. 하위 서브시스템의 데이터 출력이 상위 서브시스템의 정상적인 제어기 연산을 위한 필수 입력으로 작용하므로, 이들 간의 관계는 방향성 비순환 그래프(Directed Acyclic Graph, DAG) 형태로 추상화 및 모델링되어야 한다.

자율 드론의 핵심 서브시스템 의존성은 일반적으로 다음과 같은 상향식 데이터 흐름 계층으로 구성된다.

  1. 하드웨어 및 센서 인터페이스 계층(Hardware & Sensor Layer): IMU, GPS, LiDAR, 비전 센서 구동 노드들이 포함되며, 외부 환경 및 기체의 원시 상태 데이터를 수집하여 상위로 전달한다. 타 서브시스템에 대한 실행 의존성을 갖지 않는 최하위 노드 그룹이다.
  2. 상태 추정 계층(State Estimation Layer): 복수의 센서 데이터를 융합 필터링하여 드론의 3차원 위치, 자세, 속도 등을 정밀 추정(예: EKF, VIO 파이프라인)한다. 센서 계층의 노드들이 정상적으로 데이터를 발행하고 있어야만 오차 발산 없는 유효한 연산이 가능하다.
  3. 경로 계획 및 임무 계층(Path Planning & Mission Layer): 추정된 상태 데이터를 바탕으로 목표 지점까지의 안정적인 궤적을 생성하고 충돌 회피 알고리즘을 수행한다. 상태 추정 계층의 데이터 제공 지연이나 데이터의 비일관성은 치명적 경로 오류를 유발하므로 강한 종속성을 갖는다.
  4. 역학 및 비행 제어 계층(Dynamics & Control Layer): 경로 계획 데이터와 현재 추정 상태를 비교하여 최종적인 액추에이터 제어 명령(Control Effort)을 생성한다. 하위 계층별 데이터 스트림이 완전히 안정화된 이후 활성화되어야만, 모터 제어 신호의 불연속성에 의한 기체 추락 등 물리적인 비행 불안정성을 방지할 수 있다.

2. 순차적 기동 그래프의 구조적 설계

순차적 기동 그래프는 시스템을 구성하는 개별 노드를 정점(Vertex)으로, 노드 간의 데이터 종속성을 간선(Edge)으로 정의하는 DAG 행렬로 표현된다. 글로벌 Lifecycle 관리자 노드는 부팅 과정에서 이 그래프 정보를 기반으로 위상 정렬(Topological Sorting)을 수행하여 전체 상태 전이 시퀀스를 결정론적으로 도출한다.

  • 위상 정렬 기반의 실행 시퀀스 확립: 그래프 내의 어떤 정점도 자신을 순환적으로 향하는 간선의 시작점이 되지 않도록 엄격히 정렬해야 한다. 이는 데이터 의존성이 충족되지 않은 노드가 하위 노드보다 앞서 활성화되는 논리적 역전 오류를 시스템 기동 이전 단계에 원천적으로 차단한다.
  • 독립 컴포넌트의 병렬적 기동 탐색: 그래프 분석 시 동일한 진입 차수(In-degree)를 가지며 상호 간선이 존재하지 않는 동위 노드 그룹(예: 서로 무관한 다수의 센서 드라이버 노드)은 병렬적인(Parallel) 상태 전이 호출이 가능하다. 설계 단계에서 이러한 독립적 집합을 묶어 병렬 기동 그룹으로 설정하면, 전체 시스템의 부팅 시간(Boot-up Time)을 효과적으로 단축하고 초기 리소스 병목 지연을 완화할 수 있다.

3. 의존성 기반 상태 전이(State Transition) 동기화 전략

완성된 순차적 기동 그래프를 바탕으로 각 노드의 상태를 Unconfigured에서 Inactive, 최종적으로 Active로 전이하는 구동 제어 과정 프로세스는 다음과 같이 단계별로 동기화되어 수행된다.

  • 1단계: 전역 시스템 구성(Global Configuration): 트리 상의 모든 노드를 대상으로 on_configure 전이를 순회 트리거한다. 서브시스템들은 내부 파라미터를 초기화하고 메모리를 정적 할당하며, 퍼블리셔/서브스크라이버 인터페이스를 생성한다. 이 단계에서는 실제 토픽 발행이 발생하지 않으므로 순서에 크게 구애받지 않는 병렬 호출 지시가 가능하다.
  • 2단계: 종속성 제어 활성화(Sequential Dependency Activation): 생성된 DAG의 최하단 레이어부터 on_activate 상태 전이를 순차 실행한다. Lifecycle 관리자는 하위 제어 노드가 Active 상태에 완전 도달하고 유효한 형태의 데이터 청크를 생산하기 시작했음을 이벤트 토픽(Event Topic) 메커니즘을 통해 폴링하여 확인한다. 하위 노드의 데이터 무결성이 보장된 이후에만 종속된 직상위 계층 노드의 활성화를 트리거한다. 이러한 동기적 로드락(Load-lock) 방식의 전이는 상위 제어 구동기가 초기 쓰레기 값(Garbage Data)이나 버퍼 공백(Buffer Empty)에 의해 예기치 못하게 포화되거나 오동작하는 제반 위험을 소거한다.
