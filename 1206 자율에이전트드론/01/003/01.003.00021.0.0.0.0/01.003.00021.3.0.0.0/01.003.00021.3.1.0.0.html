21.3.1 분산 노드 환경 하 rclcpp::TimerBase 클래스의 스케줄 대기 시간 초과(Overshoot) 발생 패턴

자율 에이전트 드론을 구성하는 다수의 센서와 액추에이터는 ROS2 생태계 내에서 개별적인 노드(Node) 구조를 띠며 통신을 수행한다. 이처럼 복잡도가 높은 분산 노드 통신 환경에서는 고정밀 제어를 위해 반복 주기적(Periodic) 콜백을 주관하는 요소로서 rclcpp::TimerBase 클래스가 광범위하게 활용된다. 그러나 고주파수(High-Frequency) 제어 도메인을 다루는 최근의 전문 문헌들은 해당 클래스의 본질적인 스케줄링 오버슈트(Scheduling Overshoot) 패턴을 시스템적 한계점으로 명확히 지목하고 있다. 본 절에서는 rclcpp::TimerBase 객체가 초래하는 스케줄 대기 시간 초과 현상의 기전과 그 학술적 병목 패턴을 분석한다.

1. rclcpp::TimerBase 클래스의 비동기 스케줄링 메커니즘

rclcpp::TimerBase는 노드 생명주기 내에서 std::chrono 기반의 고정밀 시간 객체를 참조하여 사전에 설정된 주기(Period) 경과 여부를 판별하는 역할을 수행한다. 그러나 타이머 객체 자체는 독자적인 실행 스레드를 소유하지 않으며, 단지 콜백 큐를 관리하는 이그제큐터(Executor) 계층으로 하여금 자신이 만료되었음(Expired)을 상태 플래그로서 통지할 뿐이다. 타이머에 바인딩된 콜백의 실질적 실행 권한은 이그제큐터의 감시 스레드와 기저 운영체제(OS) 스케줄러가 상호 작용하는 문맥 내에서만 부여된다. 이러한 다단계의 간접적 통지 및 실행 승인 메커니즘이 시간 지연을 필연적으로 누적하는 근원으로 작용한다.

2. 대기 시간 초과(Overshoot) 발생 기전 및 학술적 분류

타이머 객체의 논리적 주기가 만료되었음에도 불구하고, 지정된 콜백 함수가 즉각적으로 호출되지 못하고 실제 물리적 대기 시간이 지연되는 현상을 스케줄 대기 시간 초과(Overshoot) 패턴이라고 정의한다. 제어 공학 분야의 다수 검증 실험에서 규명된 대기 시간 초과의 핵심 요인은 다음과 같이 세밀하게 분류된다.

  1. 상태 머신 스핀(Spin) 지연 편차 현상: 이그제큐터 내에서 rclcpp::spin() 함수에 의해 구동되는 wait_set 루프는 다른 이벤트(예: 데이터 토픽 수신, 서비스 호출, 액션 상태 갱신 등) 처리가 동반될 때 수행 완료 정착 시간이 일정하지 않은 특성을 보인다. 동시 다발적인 타 이벤트 처리에 소요된 시간 단면적만큼 rclcpp::TimerBase의 만료 확인 시점이 뒤로 밀리는 구조적 오버슈트가 상시 대기하고 있다.
  2. 시스템 타이머 큐(System Timer Queue) 경합 구조: Linux 커널 등 범용 운영체제의 타이머 서브시스템은 다수의 프로세스가 경쟁적으로 등록한 인터럽트 타이머 이벤트를 선형적, 혹은 트리 구조의 큐(Queue) 형태로 병합 관리한다. 드론 내 수십 개의 ROS2 기반 분산 노드가 개별 주기 타이머를 과다 등록할 경우, 시스템 레벨의 우선순위 스케줄러 역전 현상과 누적된 처리 대기열의 병목으로 인해 타이머 콜백 반환 지연이 기하급수적으로 폭증한다.
  3. 타이밍 왜곡 렌즈(Timing Distortion Lens) 현상: rclcpp::TimerBase는 시스템 클럭(Wall-Clock Timer)이나 단조 클럭(Steady-Clock Timer) 기반 모드를 지원한다. 이상적인 주기를 유지하려는 단조 모드 설정 하에서는 지속되는 오버슈트를 보상하기 위해 다음 타이머 주기를 인위적 연산으로 단축시키려는 동작을 수행한다. 이는 제어 루프의 샘플링 주기를 들쭉날쭉하게 만들어 극단적인 비선형성을 야기하는 타이밍 왜곡 현상을 낳는다.

3. 분산 임무 통제 아키텍처 비선형성에 대한 고찰

이러한 스케줄 타이머 오버슈트 패턴은 제어 주파수가 1kHz를 상회해야 하는 모터 제어 명령기(Motor Command Dispatcher) 모듈이나 정밀 IMU 센서 데이터 샘플링 루프에서 치명적인 시스템 교란 요소로 직결된다. 타임 스탬프 지연이 누적되어 데이터 샘플링의 위상이 변형되면 자율 비행 제어기(Flight Controller)의 이산 시간 보상기(Discrete-Time Compensator)는 잘못된 선형 방정식을 추정하게 된다. 이는 자율 드론의 폐루프(Closed-Loop) 시스템 응답 특성 구배를 왜곡하고, 제어 시스템 이득 여유(Gain Margin)를 파괴하여 궁극적으로 드론이 추락에 이르는 진동(Oscillation)을 유발하는 중대 결함이다. 선행 학술 연구들은 이를 근거로 분산 환경의 네이티브 타이머 객체 남용에 대해 강력히 경고하고 있다.
