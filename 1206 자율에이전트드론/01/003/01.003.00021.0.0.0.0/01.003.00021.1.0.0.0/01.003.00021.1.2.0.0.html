21.1.2 기본 ROS2 소프트웨어 타이머(Timer)가 가지는 밀리초(ms) 단위 해상도 물리적 한계점

자율 에이전트 드론의 비행 제어기(Flight Controller)를 ROS2 환경 위에 네이티브(Native)하게 구축하려는 시도에서 가장 먼저 직면하게 되는 치명적인 기술적 장벽은 소프트웨어 타이머(Software Timer)의 물리적 해상도(Resolution) 문제이다. 고성능 드론의 조종을 위해서는 이너 루프(Inner loop, 각속도 제어)가 최소 400Hz(2.5ms 주기)에서 많게는 1,000Hz(1ms 주기)의 정시성을 확보한 채 구동되어야 한다. 그러나 ROS2 프레임워크가 제공하는 기본 패러다임(rclcpp::TimerBase 등)은 태생적으로 비실시간성(Non-RTOS) 범용 운영체제 기반의 비동기 분산 시스템에 맞춰 설계되었기에, 밀리초(ms) 단위의 마이크로 역학적 실시간 제어를 보장하는 데 근본적인 한계를 지닌다.

1. ROS2 소프트웨어 타이머의 구조적 메커니즘

rclcpp API에서 개발자가 생성하는 타이머는 하드웨어 인터럽트(Hardware Interrupt)와 직접적으로 매핑되는 구조가 아니다. ROS2의 소프트웨어 타이머는 이그제큐터(Executor) 패러다임 내에서 관리되는 논리적 이벤트 객체일 뿐이다. 이그제큐터는 rcl_wait() 시스템 호출을 통해 특정 파일 디스크립터(File Descriptor)들의 상태 변화 유무와 타이머 트리(Timer Tree)에 등록된 만료 시간(Expiration time)을 폴링(Polling) 방식으로 검사한다.

루프 실행 시간이 지정된 타이머 만료 시간에 도달하게 되면, 타이머 관리자는 해당 타이머와 연결된 콜백 함수(Callback Function)를 이그제큐터의 준비 큐(Ready Queue)에 적재(Enqueue)한다. 이후 이그제큐터는 큐를 순회하며 콜백 함수를 꺼내어 쓰레드(Thread) 상에 인보크(Invoke)한다. 이 일련의 논리적 소프트웨어 파이프라인은 유연한 비동기 프로그래밍을 가능하게 하지만, 각 전달 단계마다 예측할 수 없는 시간 지연(Latency)이 누적되는 구조적 결함을 내포하고 있다.

2. 범용 운영체제 커널 계층의 시스템 틱(System Tick) 제약

ROS2 타이머가 구동되는 밑단에는 리눅스(Linux)를 필두로 한 운영체제의 커널(Kernel) 스케줄러가 자리 잡고 있다. 커널 내에서 시간의 흐름을 계측하고 쓰레드의 선점형(Preemptive) 스위칭을 관장하는 가장 기본적인 단위는 시스템 틱(System Tick, 또는 Jiffy)이다. 이 시스템 틱의 주파수는 커널 컴파일 시 CONFIG_HZ 파라미터로 결정되며, 현대의 데스크톱/서버용 Ubuntu 등 범용 리눅스는 타이머 인터럽트 비율을 250Hz(4ms) 또는 최대 1,000Hz(1ms)로 설정하여 배포된다.

이러한 커널 구조가 시사하는 막대한 한계점은 명백하다. 만약 기본 CONFIG_HZ가 250인 시스템에서 구동되는 비행 제어 컴포넌트가 rclcpp::create_timer(1ms)를 기동하더라도, 운영체제가 시간을 깨닫는 최소 단위 자체가 4ms에 머물러 있으므로 콜백의 호출 빈도는 물리적으로 250Hz를 초과할 수 없다. 설령 CONFIG_HZ=1000으로 컴파일된 커널을 차용하더라도, 운영체제가 보장할 수 있는 최고의 이론적 시간 해상도는 1ms로 제한된다. 즉, 400Hz(2.5ms) 루프를 구현하려 할 경우 2ms에서 이벤트가 누락되고 3ms 시점으로 밀리는 등의 심각한 에일리어싱(Aliasing)을 피할 수 없다.

3. 이그제큐터 오버헤드와 지터(Jitter)로 인한 제어 대역폭 제약

타이머 해상도 한계의 또 다른 축은 이그제큐터 내부 구조에 기인한 지터(Jitter)이다. 시스템 틱이 해당 쓰레드에 할당된 1ms를 정확히 지시하더라도, 컨텍스트 스위칭(Context Switching), C++의 다형성 추상화 과정, 자원 잠금(Mutex Locking), 메모리 할당 등의 오버헤드(Overhead)로 인해 실제 콜백이 연산을 시작하기까지 파편화된 지연이 발생한다.

지터는 설정된 타이머 주기 대비 실제 콜백이 인용되는 시간 간격의 표준 편차(Standard Deviation)를 의미한다. 만약 1,000Hz(1ms) 제어 루프를 가정할 때, ±0.5ms 수준의 거대한 지터가 개입된다면 주기적인 제어 모델은 무너지고 ZOH(Zero-Order Hold) 모델에서 가정한 \Delta t 파라미터가 난수(Random Number)처럼 요동치게 됨을 뜻한다.

결국 ROS2 기본 소프트웨어 타이머가 가지는 밀리초 수준의 해상도 한계와 이그제큐터의 비결정론적 오버헤드 구조는 드론의 이너 제어 루프가 요구하는 위상 여유(Phase Margin)를 거칠게 파괴하며, 이는 곧장 쿼드로터 동력 모델의 적분 발산(Integral Windup)과 기체 추락이라는 역학적 실패로 직결됨을 학술적으로 시사한다.
