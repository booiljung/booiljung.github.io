21.2.3 슬립 상태(Sleep State, C-State) 전이 과정이 타이머 지터(Jitter) 생성에 미치는 근원적 영향

틱리스(Tickless) 커널 정책의 근간을 이루는 하드웨어 레벨의 기전은 고급 전원 관리 인터페이스(ACPI) 표준에 명시된 프로세서 유휴 상태, 즉 C-상태(C-State)이다. 현대 마이크로프로세서는 연산 부하가 없을 때 시스템 버스 및 논리 코어의 클럭을 차단하고 전압을 강하하는 다단계의 슬립 상태(C1, C2, C3, C6 등)로 빠르게 전이(Transition)한다. 그러나 고성능 비행 제어를 위한 1,000Hz 수준의 주기 연산 환경에서는 이러한 하드웨어 C-State의 전이 메커니즘이, \Delta t의 등시성(Ischronism)을 파괴하고 치명적인 타이머 지터(Jitter)를 폭발시키는 핵심 하드웨어적 원흉으로 지목된다.

1. C-State의 단계별 전력 차단과 캐시 플러시(Cache Flush)

CPU가 연산을 수행하는 런타임 상태는 C0(Active)이다. ROS2 비행 제어 루프가 \Delta t 연산을 마치고 sleep() 또는 rclcpp::Rate::sleep() 단계에 진입하여 런큐를 비우게 되면, 커널의 cpuidle 거버너(Governor)는 즉각적으로 프로세서를 얕은 슬립(C1) 혹은 깊은 슬립(C3, C6)으로 전환시킬 것을 하드웨어에 명령한다.

각 C-State 단계가 깊어질수록 전력 소모는 획기적으로 줄어들지만, 잃어버리는 하드웨어 상태(Hardware Context)는 기하급수적으로 커진다.

  • C1 State: 단순히 HLT 명령어에 의해 프로세서의 주 클럭(Core Clock) 트리 작동만 일시 정지된다. 복구 레이턴시는 비교적 짧다.
  • C3 State (Sleep): CPU의 코어 파이프라인 클럭이 멈출 뿐만 아니라, L1/L2 캐시에 저장된 컨텍스트가 외부의 공유 캐시(L3)나 메인 메모리로 쫓겨나는 캐시 플러시(Cache Flush)가 동반된다.
  • C6 State (Deep Sleep): 코어에 공급되는 전압(Vcc) 자체가 차단되며 시스템 클럭 제너레이터(PLL)의 동작마저 멈춘다. 해당 코어의 아키텍처 상태는 모두 무효화된다.

2. 복구 레이턴시(Wake-up Latency)와 클럭 재동기화의 비선형성

치명적인 타이머 지터의 발현은 슬립 상태에 빠져있던 프로세서가 hrtimer의 만료 신호를 받고 다시 C0 상태로 깨어나는(Wake-up) 복구 구간에서 형성된다. C3 이상의 깊은 수면 상태에서 C0로 진입하기 위해서는 하드웨어 전압이 다시 인가(Power-gating 해제)되어야 하고, 멈춰있던 위상 동기 루프(PLL)가 목표 주파수까지 록킹(Locking)되는 아날로그적 대기 시간이 소모된다.

이후 플러시되었던 L1 및 L2 캐시에 명령어와 데이터(IMU 측정치, 제어 매트릭스 등)를 다시 페치(Fetch)해야 하므로 막대한 캐시 미스 가중치가 발생한다. 이러한 C-State 복원 과정에서 소요되는 시간 레이턴시(Exit Latency)는 수소(Silicon) 물리적 특성 및 메모리 컨트롤러 부하에 따라 매번 달라지는 비정형(Non-deterministic) 스펙트럼을 가진다. C6 상태에서 깨어나 제어 소스 코드를 실행하기까지 걸리는 시간은 통상 30\mu\text{s}에서 길게는 200\mu\text{s}에 달한다.

3. C-State 억제(C-State Isolation)와 비행 역학적 당위성

이러한 수십에서 수백 마이크로초의 복구 레이턴시는 그 자체로 1ms(1,000Hz) 제어 루프에서 볼 때 10~20%에 달하는 물리적 위상 깎임(Phase Erosion)을 유발한다. 나아가 매 틱마다 슬립의 깊이가 달라진다면 \Delta t 파라미터는 완전히 비선형적인 파형을 그리게 되며, 이는 드론의 미들/이너 제어 루프에서 나이퀴스트 점을 초과하는 영차 홀드 적분 발산을 유도한다.

따라서 항공 우주 및 자율 드론 분야의 실시간 시스템 튜닝에서는 운영체제의 커널 부트 파라미터(예: intel_idle.max_cstate=0, processor.max_cstate=0 혹은 idle=poll)를 강제 변경하여, 하드웨어 전력 관리 거버너의 C-State 전이 자체를 하드웨어 단에서 봉인하는 정책을 우선순위로 채택한다. 이는 비행 제어 스레드가 배정된 CPU 코어가 항상 최대 전압과 클럭(C0)을 유지하게 함으로써, 타이머 콜백 인터럽트 수신 시 지체 없는 인스트럭션 플로(Instruction Flow)를 물리적으로 보장하기 위한 타협 불가능한 실시간성 확보 아키텍처의 필수적 단계로 규정된다.
