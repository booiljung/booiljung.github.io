17.3.3 다중 센서 융합 스레드의 데드락 및 기아 현상 분석

실시간 자율 에이전트 드론의 상태 추정(State Estimation) 파이프라인은 관성 측정 장치(IMU), 위성 항법 장치(GNSS), 레이저 스캐너(LiDAR), 그리고 비전 센서 등 다양한 주기를 갖는 이종 센서 데이터를 융합(Sensor Fusion)하여 동작한다. 이 과정에서 확장 칼만 필터(EKF) 또는 비선형 최적화(Nonlinear Optimization)를 수행하는 융합 스레드는 다수의 센서 콜백(Callback) 스레드와 데이터 버퍼 및 상태 변수 등 공유 자원을 빈번하게 교환한다. 본 절에서는 이러한 다중 활성 스레드 환경에서 상호 배제(Mutex) 메커니즘 오용으로 인해 발생하는 교착 상태(Deadlock)와 기아 현상(Starvation)의 구조적 원인을 학술적으로 분석한다.

1. 다중 자원 확보 과정에서의 교착 상태(Deadlock) 분석

교착 상태는 두 개 이상의 스레드가 서로 상대방이 점유한 자원의 해제를 무한정 대기하는 교착 상태 조건을 의미하며, 주로 상호 배제, 점유 대기(Hold and Wait), 비선점(No Preemption), 그리고 환형 대기(Circular Wait)의 네 가지 구조적 조건이 동시에 충족될 때 발생한다.

다중 센서 융합 아키텍처에서, 특정 시점의 드론 상태를 업데이트하기 위해 센서 A(예: IMU)의 데이터 큐 잠금인 Lock_{A}와 센서 B(예: 비전 오도메트리)의 데이터 큐 잠금인 Lock_{B}를 동시에 획득해야 하는 스레드 T_{Fusion}이 존재한다고 가정한다. 동시에, 센서 데이터의 일관성을 검증하기 위해 하위 진단 스레드 T_{Diag}가 이 두 자원을 역순으로 요청할 수 있다.

  1. 시점 t_{1}: T_{Fusion}이 Lock_{A}를 획득하고, T_{Diag}가 Lock_{B}를 획득한다.
  2. 시점 t_{2}: T_{Fusion}은 동기화된 상태 추론을 위해 Lock_{B}를 추가로 요청하며, 동시에 T_{Diag}는 Lock_{A}를 요청한다.
    이러한 자원 획득 순서의 비대칭성은 전형적인 환형 대기 상태를 유발하며, 시스템 스케줄러 계층에서 외부적 개입이나 자원 선점이 허용되지 않는 한 전체 융합 프로세스의 영구적인 정지(Halt)를 초래한다.

2. 고빈도 센서에 의한 기아 현상(Starvation) 메커니즘

기아 현상은 특정 태스크나 스레드가 시스템의 스케줄링 자원이나 공유 자원을 지속적으로 할당받지 못하여 논리적 진행이 차단되는 상태를 의미한다. 이는 교착 상태와 달리 시스템 전체가 정지하는 것은 아니나, 저우선순위 스레드의 마감 시간(Deadline) 이탈을 유도하는 심각한 결함이다.

자율 드론 시스템에서는 IMU와 같이 매우 높은 샘플링 주파수(예: 400Hz~1kHz)로 동작하는 센서 스레드 T_{High}와, GNSS와 같이 상대적으로 낮은 업데이트 주기(예: 5Hz~10Hz)를 갖는 스레드 T_{Low}가 공존한다. 만약 T_{High}가 공유 데이터 큐의 Mutex를 빈번하게 요청하고 해제하는 과정에서, RTOS의 스케줄러나 Mutex 대기 큐 처리 정책이 엄격한 선입선출(FIFO)이 아닌 우선순위 기반(Priority-based)으로만 동작한다면, T_{High}의 연속적인 잠금 획득 시도가 T_{Low}의 접근 기회를 구조적으로 탈취하게 된다. 결과적으로 T_{Low}는 자신의 업데이트 주기를 놓치게 되며, 이는 확장 칼만 필터의 위치 보정 지연 및 발산(Divergence) 현상으로 직결된다.

3. 구조적 방어 및 회피 기법 제언

이러한 동시성 제어 결함을 방어하기 위해 시스템 설계 관점에서 몇 가지 구조적 완화 기법이 적용되어야 한다. 교착 상태의 예방을 위해서는 모든 스레드가 다중 Mutex를 획득할 때 반드시 전역적으로 합의된 동일한 순서(Global Lock Ordering)를 따르도록 강제하는 정책이 요구된다. 또한, 기아 현상 완화를 위해서는 블로킹 대기가 수반되는 Mutex 락 획득(Lock) 방식 대신, 타임아웃(Timeout) 파라미터를 갖는 락 시도(Try-Lock) 연산을 적용하여 지정된 시간 내에 자원 획득 실패 시 스레드가 제어권을 반환하도록 설계해야 한다. 더 나아가, 읽기-쓰기 이중 버퍼링(Double-buffering) 또는 잠금 없는(Lock-free) 환형 큐(Circular Queue) 자료구조를 채택하여 물리적인 스레드 간 블로킹 접점을 논리적으로 완전히 분리하는 비동기적 아키텍처의 도입이 필수적이다.
