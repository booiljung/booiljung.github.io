17.3.2 물리 데이터 버스(I2C, SPI, CAN) 선점 대응 우선순위 상한 모델

자율 에이전트 드론의 하드웨어 아키텍처에서 관성 측정 장치(IMU), 전자 변속기(ESC), 그리고 기타 주변 장치들은 주로 I2C, SPI, CAN 방식의 물리 데이터 버스를 통해 통신한다. 실시간 운영체제(RTOS) 환경에서 이러한 직렬 통신 버스는 논리적 스레드 레벨의 객체가 아닌 물리적 하드웨어 자원이므로, 단일 트랜잭션(Transaction)이 진행되는 도중에는 데이터 무결성을 위해 임의적인 선점(Preemption)이 엄격히 금지된다. 이와 같은 하드웨어적 비선점 특성은 시스템 내에서 스케줄링 간섭을 유발하며, 우선순위 상속 프로토콜(PIP)만으로는 통제 불가능한 교착 상태(Deadlock) 및 연쇄 블로킹(Chained Blocking) 현상을 야기할 수 있다. 본 절에서는 물리 데이터 버스의 선점 한계를 극복하기 위한 우선순위 상한 프로토콜(Priority Ceiling Protocol, PCP)의 적용 모델을 분석한다.

1. 물리 데이터 버스의 비선점 제약과 블로킹 모델

SPI 통신을 통해 고주파수로 데이터를 샘플링하는 상태 추정 태스크(T_{H})와 저수준 I2C 통신을 통해 배터리 상태를 모니터링하는 태스크(T_{L})가 공존하는 스케줄링 모델을 고려한다. 데이터 버스에 대한 접근은 RTOS 수준에서 상호 배제(Mutex) 또는 세마포어(Semaphore)로 제어되지만, 통신 컨트롤러 레지스터에 데이터가 쓰여지고 클럭(Clock)이 인가되는 시점부터 트랜잭션 완료 시점까지의 구간은 비선점형 임계 영역(Non-preemptive Critical Section)으로 작동한다.

만약 T_{L}이 I2C 버스를 점유하여 트랜잭션을 수행하는 과정에서, 비동기 인터럽트(Interrupt)에 의해 T_{H}가 활성화되어 SPI 버스를 요청하더라도, 두 버스가 내부 DMA(Direct Memory Access) 컨트롤러나 인터럽트 핸들러를 공유하는 구조일 경우 T_{H}는 대기 상태에 이르게 된다. 다중 버스 및 다중 자원이 얽힌 복합적인 구조에서는 태스크 간의 환형 대기(Circular Wait)가 발생하여 시스템 전체의 교착 상태로 이어질 위험이 존재한다.

2. 우선순위 상한 프로토콜(Priority Ceiling Protocol, PCP)의 구조적 접근

우선순위 상한 프로토콜(PCP)은 각 공유 자원에 대해 미리 정의된 상한(Ceiling) 우선순위를 부여함으로써, 동적 상속에 따르는 교착 상태와 연쇄 블로킹을 사전 차단하는 정적 관리 기법이다.

  1. 우선순위 상한(Priority Ceiling)의 정의: 시스템 내의 모든 공유 하드웨어 자원(I2C, SPI, CAN 버스 등) R_{i}에 대하여, 해당 자원에 접근할 수 있는 모든 태스크 중 가장 높은 우선순위를 자원 상한 우선순위 C(R_{i})로 정적으로 할당한다. 즉, C(R_{i}) = \max \{ P(T_{j}) \mid T_{j} \text{는 } R_{i} \text{를 요청할 수 있음} \}
  2. 자원 접근 허가 조건: 임의의 태스크 T_{k}가 새로운 공유 자원 R_{req}를 요청할 때, T_{k}의 현재 우선순위 P(T_{k})가 시스템 내에서 현재 다른 태스크들에 의해 점유된 자원들의 상한 우선순위 중 최댓값보다 반드시 높아야만 자원 획득이 허용된다. 즉, P(T_{k}) > \max \{ C(R_{m}) \mid R_{m}\text{은 현재 점유된 자원} \} 조건을 만족해야 한다.
  3. 상한 기반의 우선순위 상속: 만약 T_{k}의 요청이 위 조건에 의해 거부되어 블로킹될 경우, 블로킹의 원인이 된 자원(최고 상한 우선순위 자원)을 점유하고 있는 하위 태스크는 일시적으로 T_{k}의 우선순위를 상속받아 임계 영역을 신속히 이탈하도록 강제된다.

3. 최악례 차단 시간(Worst-Case Blocking Time, WCBT)의 최적화 보장

PCP를 자율 드론의 비행 제어기 버스 관리에 적용할 경우, 두 가지 핵심적인 이론적 보장이 성립한다. 첫째, 환형 대기 조건의 제거로 인해 어떠한 태스크 조합에서도 교착 상태가 독립적으로 발생하지 않는다. 둘째, 연쇄 블로킹이 원천적으로 차단되어, 임의의 태스크가 하위 우선순위 태스크에 의해 지연되는 횟수는 시스템 전체의 수행 주기 내에서 단 1회의 임계 영역 수행 시간(최대치)으로 제한된다.

이러한 특성은 비행 제어 소프트웨어의 스케줄가능성 분석(Schedulability Analysis)에서 최악례 차단 시간(WCBT)을 수학적으로 엄밀하게 한정할 수 있는 기반을 제공한다. 결과적으로 고주파수 제어 루프를 담당하는 태스크의 타이밍 데드라인 준수율을 극대화하고, 물리 데이터 버스 충돌에 따른 지터(Jitter) 현상을 시스템 설계 단계에서 통제 상태 범위 내부로 제한하는 학술적 토대가 된다.
