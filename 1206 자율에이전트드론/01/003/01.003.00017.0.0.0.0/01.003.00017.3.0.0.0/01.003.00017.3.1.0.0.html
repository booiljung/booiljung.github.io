17.3.1 Mutex 환경에서의 우선순위 상속 프로토콜 적용

실시간 운영체제(RTOS)를 기반으로 동작하는 자율 에이전트 드론의 비행 제어 소프트웨어 아키텍처에서는 다중 스레드 간의 자원 공유가 빈번하게 발생한다. 특히, 상호 배제(Mutual Exclusion, Mutex) 메커니즘을 통해 공유 자원의 무결성을 보장하는 과정에서, 태스크 간의 우선순위 역전(Priority Inversion) 현상은 시스템의 실시간성을 심각하게 훼손하는 주요 요인으로 작용한다. 본 절에서는 우선순위 역전 현상의 수학적 발생 원인을 분석하고, 이를 방어하기 위한 우선순위 상속 프로토콜(Priority Inheritance Protocol, PIP)의 이론적 배경 및 적용 방안을 다룬다.

1. 우선순위 역전(Priority Inversion) 현상의 분석

우선순위 역전은 우선순위가 높은 프로세스나 스레드가 실행 대기 상태에 머무는 동안, 그보다 낮은 우선순위를 가진 태스크가 스케줄러에 의해 지속적으로 CPU 자원을 점유하는 현상이다. 이 현상은 RTOS의 선점형 스케줄링(Preemptive Scheduling) 환경에서 자원 접근이 동기화될 때 주로 발생한다.

태스크 집합 T = \{T_{H}, T_{M}, T_{L}\}이 존재하고, 각 태스크의 우선순위가 P(T_{H}) > P(T_{M}) > P(T_{L})이라 가정한다. 태스크 T_{L}이 공유 자원 R에 접근하기 위해 Mutex M을 획득(Lock)한 상태에서 선점되어, 태스크 T_{H}가 실행을 시작한다. 이후 T_{H} 역시 자원 R에 접근을 시도하여 Mutex M의 대기 큐(Wait Queue)에 진입할 경우, T_{H}는 블로킹(Blocking) 상태로 전환된다. 이 시점에서 T_{H}와 T_{L} 사이의 우선순위를 갖는 태스크 T_{M}이 활성화되면, T_{M}은 공유 자원 R을 필요로 하지 않음에도 불구하고 T_{L}을 선점하여 실행된다. 결과적으로 T_{H}는 T_{M}의 실행 시간 전체에 걸쳐 지연을 겪게 되며, 이는 무한 대기 현상 또는 최악례 실행 시간(Worst-Case Execution Time, WCET)의 예측 불가능성을 야기한다.

2. 우선순위 상속 프로토콜(Priority Inheritance Protocol, PIP)의 구조

우선순위 상속 프로토콜(PIP)은 앞서 서술한 간접적인 지연 문제를 해결하기 위한 동적 스케줄링 정책이다. 이 정책의 핵심 원리는 하위 우선순위 태스크가 점유한 리소스를 상위 우선순위 태스크가 요청할 경우, 하위 태스크의 논리적 실행 우선순위를 상위 태스크의 우선순위 수준으로 임시 상향 조정(Inherit)하는 것이다.

  1. 상속 조건의 성립: 공유 자원 R을 소유한 태스크 T_{L}이 존재할 때, 이보다 높은 베이스 우선순위(Base Priority)를 갖는 태스크 T_{H}가 R에 접근하기 위해 대기 상태로 전환되면 우선순위 상속이 트리거된다.
  2. 동적 우선순위 할당: 태스크 T_{L}의 활성 우선순위(Active Priority)는 다음 관계식에 따라 일시적으로 재할당된다.
    P_{active}(T_{L}) = \max(P_{base}(T_{L}), P_{base}(T_{H}))
  3. 상속의 반환(Reversion): 태스크 T_{L}이 임계 영역(Critical Section)에서의 연산을 완료하고 Mutex M을 해제(Unlock)하는 즉시, T_{L}의 활성 우선순위는 원래의 베이스 우선순위로 복원된다.

3. PIP 적용 시의 한계점과 제약

우선순위 상속 프로토콜 적용은 중간 우선순위 태스크(T_{M})에 의한 무제한적인 블로킹 시나리오를 제거할 수 있다는 장점이 있으나, 다중 Mutex 환경에서는 구조적인 취약점을 내포하고 있다. 대표적인 한계점은 교착 상태(Deadlock)의 발생 가능성이다. 두 개 이상의 태스크가 서로 다른 공유 자원을 순환적 비대칭 형태로 대기할 때, PIP의 상속 알고리즘 단독으로는 이를 예방할 수 없다.

또한, 연쇄 블로킹(Chained Blocking) 현상 역시 발생 가능하다. 태스크 T_{H}가 공유 자원을 획득하기 위해 다수의 하위 태스크가 자원을 해제하기를 순차적으로 대기해야 하는 상황에서, 각 상속 과정이 누적되어 스케줄링 오버헤드를 유발하고 응답 시간(Response Time)의 지연을 초래한다. 자율 드론의 고주파수 제어 루프나 안전 필수(Safety-Critical) 임베디드 통제 환경에서는 이러한 연쇄 지연 시간이 스케줄링 불일치를 발생시킬 수 있다. 따라서 제한된 환경적 특성을 고려하여, 시스템 설계 단계에서 독립성 제고 및 콜백 모델의 최적화가 필수적으로 요구된다.
