17.6.2 워치독 타이머 인터럽트에 의한 시스템 락업 판정

실시간 운영체제(RTOS) 기반의 자율 에이전트 드론 시스템에서 제어 루프의 무한 루프, 교착 상태(Deadlock), 인터럽트 폭주 등으로 인해 시스템 자원이 포화될 경우, 시스템 락업(System Lockup) 상태가 발생한다. 이러한 치명적인 결함을 독립적인 하드웨어 측면에서 감지하고 복구하기 위한 핵심 방어 메커니즘으로 워치독 타이머(Watchdog Timer, WDT) 인터럽트가 활용된다. 워치독 타이머 인터럽트는 정상적인 소프트웨어 실행 흐름이 중단되었을 때 이를 하드웨어 타이머 만료를 통해 판정하고 적절한 대응 절차를 호출하는 감시 역할을 수행한다.

1. 워치독 타이머의 동작 원리와 시스템 락업 판정 기준

워치독 타이머는 사전에 설정된 시간 주기로 카운터를 감소시키거나 증가시키는 독립적인 타이머 회로이다. 정상 상태의 운영체제 및 응용 소프트웨어는 타이머가 만료(Timeout)되기 전에 주기적으로 카운터를 초기화(Clear 또는 Kick)하는 루틴을 실행해야 한다. 만약 시스템 자원의 포화나 커널 패닉(Kernel Panic)으로 인해 카운터를 초기화하는 명령어 실행이 지연되거나 누락될 경우, 타이머는 설정된 임계값에 도달하여 인터럽트를 발생시키거나 시스템을 강제로 하드웨어 리셋(Hardware Reset)한다. 락업 판정은 이 카운터 만료 시점을 기준으로 결정되며, 이는 소프트웨어 계층의 응답 불가 상태를 물리적으로 증명하는 명확한 기준표가 된다.

2. 하드웨어 및 소프트웨어 계층의 다중 워치독 아키텍처

자율 비행 제어기 구조에서는 단일 장애점(Single Point of Failure)을 극복하기 위해 하드웨어 기반 워치독 타이머와 소프트웨어 기반 워치독(Task Watchdog)을 결합한 다중 계층 아키텍처를 채택한다.
하드웨어 워치독 타이머는 마이크로컨트롤러 유닛(MCU) 내부에 실장된 독립된 클럭 소스를 사용하므로 메인 시스템 버스나 CPU 클럭의 교란으로부터 자유롭다. 반면, 소프트웨어 기반 태스크 워치독은 RTOS 수준에서 개별 스레드나 태스크의 실행 주기를 감시한다. 특정 콜백(Callback)이나 주기적 제어 태스크가 지정된 시간 내에 완료되지 않으면 소프트웨어 워치독 인터럽트가 발생하여, 어느 컨텍스트에서 제어 흐름이 차단되었는지를 세밀하게 식별한다.

3. 인터럽트 서비스 루틴(ISR) 기반 락업 대응 메커니즘

워치독 타이머가 만료되어 인터럽트가 발생하면, 시스템은 즉시 최고 우선순위를 지닌 워치독 인터럽트 서비스 루틴(Watchdog Interrupt Service Routine)으로 진입한다. 해당 ISR은 시스템이 하드웨어 리셋으로 넘어가기 전 코어 레지스터의 상태, 스택 포인터, 현재 스레드 정보 등의 메타 데이터를 비휘발성 메모리에 덤프(Dump)하여 사후 분석을 위한 텔레메트리 에비던스를 확보한다. 이후, 현재 구동 중인 모터 출력을 정지 또는 안전 모드(Safe Mode)로 절체하는 등 즉각적인 물리적 제동을 수행한 뒤 시스템 리셋을 트리거한다.

4. 자율 비행 제어 루프에서의 방어 로직 설계

자율 에이전트 드론에서는 워치독 타이머의 임계값을 비행 동역학의 최소 유효 제어 주기와 연동하여 엄격하게 설정한다. 일반적으로 내부 모터 제어 루프의 주파수에 기반하여 마이크로초(Microsecond)에서 밀리초(Millisecond) 단위의 마진을 두고 초기화 주기를 결정한다. CPU 사이클 부족이나 로직 결함으로 인해 해당 주기를 놓치게 되면 워치독 메커니즘이 강제적으로 락업을 선언함으로써 비행체 추락이나 예상치 못한 모터 폭주 등으로 발생할 수 있는 2차적인 하드웨어 손상을 방지한다. 워치독 인터럽트는 외부 감시 회로와 함께 물리적 안전을 보장하기 위한 종단 방어선 역할을 수행하며, 시스템의 결정론적(Deterministic) 실패 동작을 유도한다.
