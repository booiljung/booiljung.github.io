17.5.1 Micro XRCE-DDS 클라이언트의 인터럽트 주도형 패킷 수신 아키텍처

자율 비행 에이전트의 분산 시스템 아키텍처에서 임베디드 비행 제어 블록은 최상위 ROS2 통신망과 미들웨어 소프트웨어 프레임워크를 매개로 연결된다. 이때 기본 네트워킹 전송 계층 추상화를 담당하는 Micro XRCE-DDS(eXtremely Resource Constrained Environments DDS) 클라이언트는 메인 연산을 수행하는 컴패니언 컴퓨터(Companion Computer) 또는 외부 원격 지상 통제소(GCS)로부터 유입되는 제어 궤적 명령 및 시스템 파라미터 업데이트 패킷을 엄밀하게 실시간으로 수신 검증해야 한다. 본 절에서는 이러한 통신 미들웨어의 패킷 수신 처리 과정에서 발생하는 스케줄링 태스크 간섭 지연을 구조적으로 배제하고 시스템의 즉각적인 응답 반응성을 확보하기 위한 인터럽트 주도형(Interrupt-driven) 패킷 수신 인터페이스 아키텍처를 학술적으로 분석한다.

1. 폴링(Polling) 수신 스케줄링의 지연 한계

소프트웨어 시스템 구현에서 전통적인 방식의 통신 수신 확인 루틴은 특정 태스크 스레드가 고정된 시계열적 주기마다 직렬 인터페이스 포트(예: 고속 UART, SPI) 또는 이더넷 컨트롤러 하드웨어의 자체 수신 버퍼 레지스터 큐 상태를 순차 조회(Polling)하는 형태로 구성되어 적용된다. 이러한 소프트웨어 동기식 폴링 구조 접근법은 물리적 전송선을 통한 데이터 패킷의 하드웨어 유입 완료 시점과 통신 수신 스레드 계층의 실제 연산 활성화 시점 사이에 필연적으로 낭비되는 비동기성 스케줄링 오프셋(Asynchronous Offset) 지연을 지닌다.

특히 선점형 RTOS 커널 환경에서는 통신 수신 대기 태스크의 재활성화 허용 주기가 스케줄러 계층의 시스템 하드웨어 틱(System Tick) 해상도 주파수 리미트에 종속되므로, 외부 패킷이 완전히 도착하더라도 다음 스레드 스케줄링 슬롯(Scheduling Slot) 권한이 재할당될 때까지 처리되지 못해 최소 수 밀리초(ms) 규모의 물리적인 맹목적 대기 시간인 데이터 수신 지연(Polling Latency)이 구조적으로 야기된다. 이는 고주파수 폐루프(Closed-loop) 제어로 피드백 구성을 형성하는 자율 드론 비행의 단대단(End-to-End) 물리 제어 응답 시간 모델 지표 내에서 매우 치명적인 동역학적 제어 위상 지연(Phase Lag)의 직접적 원인 요소로 작용하게 된다.

2. 외부 하드웨어 인터럽트 주도형(Interrupt-driven) 수신 설계

이러한 물리적 레이턴시 병목 한계를 응용 아키텍처 수준에서 해결하기 위해, 비행 제어 플랫폼 시스템에 탑재되는 Micro XRCE-DDS 클라이언트의 하드웨어 포팅 추상화 계층 통로 설계는 하드웨어 시리얼 컨트롤러 자체에 내장된 전용 수신 라인 신호 인터럽트(RX Interrupt) 라인 또는 범용 외부 인터럽트 핀(EXTI)을 직접 활용 연결하는 인터럽트 주도형 전면 비동기 설계로 변환 채택되어야 한다.

이 수정된 아키텍처 설계 구조에서는 시스템 마이크로컨트롤러 통신 트랜시버(Transceiver) 칩셋이 새로운 직렬 통신 프레임의 헤더 스타트 바이트를 인식하여 자기 RX 레지스터에 기록 적재하는 완료 즉시, 물리적인 하드웨어 시스템 버스 인터럽트 신호 로직이 메인 CPU 계층으로 즉각 인가 송출된다. 이는 RTOS 스케줄러의 타임 슬라이스 배정 상태나 흐름과 무관하게 현재 실행 중인 모든 비임계 하위 애플리케이션 코드를 강제로 선점(Preemption) 단절시키고, CPU의 메모리 연산 제어 권한을 사전 구속된 데이터 수집 전담 인터럽트 서비스 루틴(ISR, Interrupt Service Routine)으로 극도로 신속하게 양도 토글시킨다. 이 체계적인 메커니즘을 구동함으로써 시스템은 데이터 패킷의 물리적 수신 시점과 소프트웨어적 처리 시작 시점 사이의 간극 시간을 이론적인 하드웨어 클럭 사이클 트랜지션 지연 지표 레벨 수준으로 극도로 단축 정밀화할 수 있다.

3. 하부 반반(Bottom-Half) 디코딩 구조와 버퍼 오버플로우 통제 최적화

인터럽트 신호 트리거 주도형 수신 체계 아키텍처는 고속 통신 환경에서의 실시간 패킷 손실(Drop) 현상을 방어해 주지만, ROS2 체계를 계승한 광역 DDS 미들웨어 통신 패킷 구조 자체는 헤더 파싱(Parsing), 데이터 역직렬화(Deserialization) 분해, 그리고 암호화된 페이로드 체크섬(Checksum) 무결성 검증 등의 상당히 연산 집약적이고 무거운 디코딩 뎁스(Depth) 하위 연산을 필연적으로 수반한다. 이러한 수십 마이크로초 이상의 복합 데이터 연산을 문맥 교환이 차단된 ISR 내부 좁은 프레임에서 모두 직접 수행 처리할 경우, ISR의 실행 점유 시간이 비정상적으로 연장 초과되어 시스템 제어 타이머나 모터 펄스 인터럽트 등 핵심 비행 제어 필수 하드웨어 인터럽트 계층의 정시 처리 기회를 배타적으로 역탈취하는 중대한 데드락(Deadlock) 멈춤 시스템 제어 상실 위험을 초래하게 된다.

따라서 최적화 검증 배포되는 Micro XRCE-DDS 하드웨어 수신 계층 포팅 모델은 수신 ISR 루틴 내에서 락 프리 링 버퍼(Ring Buffer) 형태를 띄는 전역 수신 직접 메모리 접근(DMA) 버퍼 메모리 풀(Memory Pool)에 인입된 원시 바이트 스트림 프레임을 단순 전송 메모리 복사(Copy)하는 기계적 수준의 최소한의 연산만을 수행 점유해야 한다. 이 메모리 복사가 완료되고 인터럽트 플래그가 클리어된 직후, 커널 운영체제의 지연된 프로시저 호출(Deferred Procedure Call) 지시자 세팅이나 단일 세마포어(Semaphore) 카운트 통지 로직을 통해 커널 백그라운드 사용자 공간 도메인에 수면 대기 중인 독립적인 XRCE-DDS 통신 에이전트 해석 스레드를 평가 상태로 승격(Wake-up)시킨다. 이와 극단적으로 분리된 형태의 하부 반반(Bottom-Half) 인터럽트 분리 처리 및 비동기 워커 위임 체계 디자인은 분산 ROS2 통신 미들웨어망의 안정적인 완전한 패킷 처리 보장성 요건과 임베디드 코어 비행 제어 시스템의 견고한 고주파수 실시간 제어 방어 체제 요건 양단을 하드웨어 상호 간섭 훼손 없이 독립적으로 영구 유지하는 임베디드 소프트웨어 인터페이스 아키텍처의 공학적 핵심 해법을 제공한다.
