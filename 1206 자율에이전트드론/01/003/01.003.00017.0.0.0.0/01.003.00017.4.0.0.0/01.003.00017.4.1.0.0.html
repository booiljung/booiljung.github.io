17.4.1 하드웨어 타이머 기반 고주파 제어 입력 폴링 루틴

자율 에이전트 드론의 비행 제어 시스템은 외부 조종기(Radio Control) 신호, 컴패니언 컴퓨터(Companion Computer)의 자율 비행 궤적 명령, 그리고 텔레메트리(Telemetry) 링크를 통한 통신 패킷을 지속적으로 수신해야 한다. 이러한 제어 입력은 드론의 역학적 안정성을 유지하는 핵심 상태 변수이므로, 데이터 버스로부터 신호를 읽어들이는 폴링(Polling) 루틴은 매우 높은 주파수와 엄격한 시계열적 결정성(Determinism)을 요구한다. 본 절에서는 소프트웨어 기반 타이머의 한계를 극복하기 위해 설계된 하드웨어 타이머 기반의 고주파 제어 입력 폴링 루틴의 체계적 아키텍처를 학술적으로 분석한다.

1. 소프트웨어 타이머 스케줄링의 지터(Jitter) 한계

일반적인 실시간 운영체제(RTOS) 환경에서 스레드 휴면(Sleep) 함수나 논리적 소프트웨어 타이머를 이용하여 폴링 주기를 설정할 경우, 시스템의 시스템 틱(System Tick) 해상도에 의해 최소 대기 시간의 하한이 결정된다. 통상적인 임베디드 RTOS의 시스템 틱 주기는 1ms(1000Hz) 수준이나, 운영체제의 스케줄러 계층에서 발생하는 문맥 교환(Context Switching) 오버헤드, 상위 우선순위 태스크에 의한 선점(Preemption), 그리고 커널 잠금(Kernel Lock) 상태로 인해 스레드가 활성화되는 시점에는 필연적으로 스케줄링 지터(Jitter)가 발생한다.

고기동성 자율 비행이나 다중 로터의 마이크로초 단위 모터 제어가 요구되는 환경에서, 제어 입력 스레드의 활성화 시점이 1~2ms 지연되는 현상은 칼만 필터의 시스템 모델 예측 주기를 왜곡시키며 전반적인 제어 성능의 치명적인 저하를 초래한다. 따라서 고주파 제어 제어기 환경에서 제어 루프를 스케줄러 기반의 소프트웨어 타이머에 전적으로 의존하는 것은 구조적인 취약성을 내포한다.

2. 하드웨어 카운터 기반 인터럽트 서비스 루틴(ISR)의 적용

이러한 지터 제약을 회피하기 위하여, 하드웨어 타이머 인터럽트를 활용한 물리 계층의 폴링 스케줄링 아키텍처가 적용된다. 마이크로컨트롤러(MCU)에 내장된 독립적인 하드웨어 카운터(Hardware Counter)는 CPU의 주 연산 흐름과 완전히 분리되어 작동하며, 물리적 클럭 주파수를 기반으로 설정된 프리스케일러(Prescaler)와 오토 리로드 레지스터(Auto-Reload Register) 값을 통해 매우 정확한 마이크로초 단위의 시계열 주기를 생성한다.

타이머 카운트가 목표치에 도달하면 하드웨어 코어는 즉각적으로 인터럽트(Interrupt)를 발생시켜 현재 진행 중인 모든 비임계 태스크의 실행을 중단시키고 연산 제어 권한을 인터럽트 서비스 루틴(Interrupt Service Routine, ISR)으로 강제 이전시킨다. 이 ISR 컨텍스트 내에서 통신 직렬 데이터 버스(예: UART, SPI 인터페이스)의 수신 레지스터 상태를 직접 읽어들이는 하드웨어 폴링을 수행함으로써 운영체제 스케줄러 계층을 우회하는 최고 수준의 실시간성을 보장받을 수 있다.

3. 하드웨어 폴링 환경의 제약과 차상위 처리(Bottom-Half) 통지

하드웨어 타이머 인터럽트를 통한 폴링 구조는 시간적 정밀도를 극대화하지만, ISR 내부 구조에서는 운영체제의 블로킹 API(예: Mutex 대기, 동기식 메시지 큐 등)를 원천적으로 사용할 수 없다는 중대한 시스템 제약 조건을 수반한다. 좁은 문맥 계층을 지닌 ISR 루틴의 과도한 실행 시간 연장은 타 하드웨어 인터럽트의 처리 손실을 유발하여 시스템의 전체적인 데드락(Deadlock) 현상이나 와치독(Watchdog) 초기화를 유발할 위험이 있다.

이를 방어하기 위해 하드웨어 폴링 루틴 구조는 엄격한 분업 원칙을 채택한다. 수신된 원시 제어 데이터는 락 프리(Lock-Free) 링 버퍼 등 비블로킹 메모리 영역으로 즉각 복사되며, 이후 RTOS의 비동기 통지 매커니즘인 경량 세마포어(Semaphore) 상태 변경, 이벤트 플래그(Event Flag) 전송, 또는 차상위 인터럽트 처리(Bottom-Half Processing, 지연된 프로시저 호출) 기법을 활용하여 백그라운드의 연산 스레드를 간접 활성화한다. 이러한 상호작용 아키텍처는 고주파 폴링의 물리적 시계열 정확성과 어플리케이션 계층 데이터 연산의 논리적 유연성을 동시에 충족시키는 최적화된 설계 표준이다.
