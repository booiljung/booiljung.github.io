17.4.3 락 프리 자료구조 및 링 버퍼 기반 스레드 블로킹 억제

실시간 운영체제(RTOS) 환경에서의 프로세스 간 통신(IPC) 및 다중 스레드 동기화 기법은 상호 배제(Mutual Exclusion)를 통해 공유 메모리의 무결성을 보장하지만, 근본적으로 운영체제 스케줄링 계층의 문맥 교환(Context Switch) 오버헤드와 비결정적인 스레드 지연(Blocking) 현상을 수반한다. 고기동성 자율 비행 객체의 제어 소프트웨어 아키텍처에서는 밀리초 미만의 미세한 지연조차 제어 루프의 역학적 불안정성(Instability)을 초래하므로, 스케줄링 대기를 원천적으로 배제하는 비블로킹(Non-blocking) 기술의 도입이 필수적이다. 본 절에서는 스레드 간 데이터 교환 시 병목 현상을 억제하는 락 프리 자료구조(Lock-Free Data Structure) 및 링 버퍼(Ring Buffer)의 학술적 원리와 적용 방안을 다룬다.

1. 락 프리 구조의 원자적(Atomic) 연산 기반

락 프리 자료구조는 상호 배제 객체(Mutex, Semaphore 등)에 의존하지 않고 다중 스레드의 동시성 접근을 허용하는 메모리 설계 기법이다. 이는 운영체제의 논리적 동기화 계층 대신 마이크로프로세서(MCU) 하드웨어 아키텍처 차원에서 물리적으로 지원하는 원자적 명령어(Atomic Instruction)를 기반으로 성립된다.

대표적인 접근법인 비교환(Compare-And-Swap, CAS) 연산과 패치 앤 애드(Fetch-And-Add) 등의 범용 기계어 명령어는 단일 클럭 사이클 내에서, 혹은 다중 코어 시스템 버스 잠금(Bus Lock)을 통해 물리적 독립성을 확보한 채 메모리 포인터의 값을 검증하고 갱신한다. 이러한 원자적 연산을 메모리 배열 인덱스 갱신에 적용함으로써, 하나의 스레드가 메모리 기록 연산을 지연시키거나 상위 프로세스에 의해 선점(Preempted)되더라도 다른 스레드의 데이터 접근 통로가 연쇄적으로 블로킹되는 교착 상태(Deadlock) 및 간접적 우선순위 역전(Priority Inversion) 현상을 시스템 아키텍처 수준에서 예방할 수 있다.

2. 링 버퍼(Ring Buffer) 메모리 아키텍처 최적화

비행 제어 시스템의 이종 센서 데이터 융합 및 비동기 이벤트 퍼블리싱 체계에서 가장 광범위하게 활용되는 락 프리 자료구조 구현체는 링 버퍼(Ring Buffer, 환형 큐 Circular Queue)이다. 링 버퍼는 고정된 크기의 연속된 배열 메모리 형태를 지니며, 새로운 데이터가 삽입되는 오프셋을 지시하는 꼬리(Tail) 포인터와, 누적된 데이터가 추출되는 시작 지점을 가리키는 머리(Head) 포인터 변수의 독립적인 상태 갱신만으로 전체 가용 상태가 정의된다.

  1. 단일 생산자-단일 소비자(SPSC) 최적화 패턴: 고주파수 하드웨어 타이머에 종속된 센서 폴링 콜백 함수(생산자)와 메인 상태 추정 확장 칼만 필터 스레드(소비자) 간의 정보 교환 등 SPSC 패턴에서는 두 포인터의 변경 소유권이 물리적으로 완전히 분리된다. 생산자는 오직 Tail 포인터의 상태만을 단방향으로 전진시키며, 소비자는 Head 포인터만을 독점적으로 갱신한다.
  2. 비블로킹 데이터 트랜잭션의 완결성: 생산자가 배열 버퍼 내부에 데이터를 순차적으로 기록한 후 Tail 포인터를 원자적으로 갱신하는 시점에 메모리 배리어(Memory Barrier)를 적용하면 분산 코어 간 메모리 가시성(Memory Visibility) 매칭 문제가 구조적으로 해결된다. 소비자는 시스템 락(Lock) 획득 대기열에 진입하지 않고 Head와 Tail 변수의 위치 차이만을 논리적으로 비교 검산하여 지연 없이 즉각적으로 새로운 데이터를 수집한다.

3. 시스템 실행 컨텍스트의 최악 시간 결정성 확보

자율 미션 통제 장치 및 다중 분산 제어 멀티프로세싱 환경에 링 버퍼와 결합된 락 프리 자료구조 설계 패턴을 적용할 경우, 생산자와 소비자 양측 스레드의 실행 파이프라인 시간 편차(Jitter)는 메모리 복사 연산에 소요되는 순수한 CPU 명령어 사이클 최소치 수준으로 극적 수렴하게 된다.

특히 외부 텔레메트리 데이터 스트리밍 전송과 같은 저우선순위의 대용량 통신 스레드가 혼잡한 네트워크 환경 구축으로 인한 예측 불가능한 딜레이를 겪더라도, 고주파수의 시스템 코어 제어 스레드는 병목 대기 없이 단순히 링 버퍼의 Tail 포인터만을 갱신시키며 기존의 통제 진행을 무정지 기반으로 지속할 수 있다. 결론적으로, 통신 블로킹을 배제하는 이러한 인터페이스 아키텍처는 고가용성 컴퓨팅 자원이나 전력이 한정적인 임베디드 드론 컴패니언 하드웨어 환경에서 엄격하게 제한된 시간 결정성(Time Determinism)을 수호하는 핵심적인 소프트웨어 설계 기술이다.
