17.1.2 실시간 태스크 스케줄링의 결정론적 제약

1. 개요

자율 비행 제어기와 같은 안전 필수(Safety-Critical) 임베디드 시스템 설계에서 가장 핵심적인 학술적 논점은 연산의 평균 처리량(Average Throughput)을 상향시키는 것이 아니라, 예측 불가능한 런타임 환경 속에서도 모든 태스크(Task)가 정해진 마감 시간(Deadline) 이내에 완수됨을 수학적으로 보증하는 데 있다. 이를 실시간 태스크 스케줄링의 ’결정론적 제약(Deterministic Constraint)’이라 명명한다. 본 절에서는 운영체제 스케줄러가 부과하는 시간적, 구조적 제약의 본질을 구명하고, 이를 준수하기 위한 선점 체계와 분석 이론을 서술한다.

2. 결정론적 제약(Deterministic Constraint)의 정의

실시간 운영체제(RTOS) 환경 하에서 시스템의 결정론성이란, 동일한 하드웨어 상태와 시간적 입력 벡터가 주어졌을 때 특정한 태스크의 실행 궤적(Execution Path)과 결과 반환 시점이 시간 축 위에서 항상 사전에 정의된 오차(Tolerance Margin) 이내로 수렴하는 확고한 물리적 성질을 의미한다.

자율 에이전트 드론에서는 센서 퓨전(Sensor Fusion), 기하학적 궤적 제어기(Trajectory Controller), 그리고 이산 모터 믹서(Motor Mixer) 등의 코어 모듈들이 강제적인 주기적 태스크(Periodic Task)로 동작한다. 이들 파이프라인 중 어느 하나라도 스케줄링 지연으로 인해 연산 마감 시간을 초과(Deadline Miss)할 경우, PID 역학 제어 루프의 위상 지연(Phase Lag)이 지수적으로 누적되어 즉각적인 기체 붕괴를 야기한다. 따라서 운영체제 프레임워크는 태스크 스레드의 생성, 블로킹(Blocking), 그리고 상태 해제(Unblocking) 메커니즘 전반에 걸쳐 최악 실행 시간(WCET: Worst-Case Execution Time)의 예측 가능성을 보장하기 위한 엄격한 메모리 및 타이밍 제약을 부과한다.

3. 선점형 스케줄링(Preemptive Scheduling)과 분기 지연

고주파수 비행 제어 루프가 로깅(Logging)이나 통신(Comm) 태스크에 의해 시간적으로 침범받지 않도록 보장하기 위해, RTOS는 고정 우선순위 선점형 스케줄링(Fixed-Priority Preemptive Scheduling) 정책을 아키텍처의 근간으로 채택한다.

저우선순위 태스크가 공유 리소스를 무한정 락(Lock)하는 우선순위 역전(Priority Inversion)과 같은 비결정론적 블로킹을 방어하는 것도 제약의 일환이지만, 물리적인 장치 사이에서 발생하는 문맥 교환 지연(Context Switch Latency) 자체가 제약의 하한선을 정의한다. 현재 실행 중인 태스크의 레지스터 세트를 스택(Stack) 영역에 푸시(Push)하고 다음 차례 태스크의 컨텍스트를 복원하는 과정은 수 마이크로초(µs) 사이클 내에 완전히 마무리되어야 한다. 이 때문에 태스크 풀(Task Pool)의 크기와 스레드별 스택 프레임의 크기를 런타임 중 동적 할당하지 않고, 빌드 및 초기화 시점에 정적으로 제약(Static Constraining)하는 구조가 동반된다.

4. 스케줄링 분석 모델: RMS와 DMS 이론

실제 자율 비행 제어기 펌웨어의 아키텍처를 설계할 때, 할당된 특정 ’태스크 셋(Task Set)’이 데드라인 내에 항상 스케줄링 가능한지(Schedulability)를 사전에 증명하기 위해 고전적인 이산 수학 스케줄링 이론이 제약 조건 평가에 도입된다.

  • 비율 단조 스케줄링(RMS, Rate Monotonic Scheduling): 태스크가 반복되는 주기가 짧을수록 더 높은 정적 우선순위를 부여하는 전략이다. 관성 센서(IMU) 폴링(예: 1ms 단위)은 가장 높은 우선순위를 점유하며, 텔레메트리 송수신(예: 100ms 단위)은 하위 우선순위를 할당받는 형태이다.
  • 마감 시간 단조 스케줄링(DMS, Deadline Monotonic Scheduling): 호출 주기와는 관계없이, 구동 개시 시점으로부터 산정된 목표 마감 시간(Relative Deadline)이 짧은 태스크에 가장 높은 우선순위를 할당하여, 이벤트 구동형(Event-driven) 하드웨어 인터럽트 처리의 생존성을 극대화하는 분석 모델이다.

비행 제어기 소프트웨어 플랫폼 계층에서는 이러한 두 이론적 한계 평가식을 기준으로 CPU의 허용 점유율(Utilization Bound) 임계치를 검증하고 애플리케이션 프레임워크를 규격화한다.
