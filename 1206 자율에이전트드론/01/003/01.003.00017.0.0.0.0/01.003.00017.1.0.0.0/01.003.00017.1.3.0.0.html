17.1.3 ROS2 컴패니언 컴퓨터와 RTOS 비행 제어기 간 실행 컨텍스트 비대칭성

1. 개요

최신 자율 에이전트 드론 아키텍처는 고도의 인지 및 전역 경로 탐색 연산을 수행하는 컴패니언 컴퓨터(Companion Computer, Ubuntu/ROS2 등 범용 운영체제 기반)와 1KHz 이상의 고주파수 기구학적 모터 추종 제어를 전담하는 임베디드 비행 제어기(Flight Controller, RTOS 기반)로 분리된 이기종 하드웨어 이원화 체계를 채택한다. 본 절에서는 이 두 상이한 컴퓨팅 플랫폼이 상호 연동될 때 물리적/논리적으로 직면하는 ’실행 컨텍스트(Execution Context)의 구조적 비대칭성(Asymmetry)’을 학술적으로 분석하고, 시스템의 파국을 유발할 수 있는 타이밍 미스매치(Mismatch) 및 그 브릿징(Bridging) 해소 방안을 서술한다.

2. 범용 OS와 RTOS의 스케줄링 패러다임 간극

컴패니언 컴퓨터는 통상 Linux 계열 운영체제 상에서 구동되며, 완전 공정성 기반의 시분할 스케줄링(Fair Time-sharing Scheduling)을 준수한다. 이 환경에서는 OS 커널의 백그라운드 I/O 작업이나 순간적인 로드 스파이크(Load Spike)로 인해 ROS2 노드 스레드가 밀리초(ms) 단위의 비결정론적 지터(Jitter)를 겪게 된다.

반면, 임베디드 측 비행 제어기의 실시간 운영체제(RTOS)는 마이크로초(µs) 단위의 엄밀한 마감 시간(Hard Deadline) 보장을 목표로 고정 우선순위 선점형 스케줄링(Fixed Priority Preemptive Scheduling)을 강제한다. 이러한 철학적 간극은 미들웨어 통신 버스를 통해 두 시스템이 결합될 때, 타이밍 모델과 자원 접근 속도의 극단적인 불일치를 낳는다.

3. 실행 컨텍스트(Execution Context) 비대칭성의 구조적 측면

실행 컨텍스트 비대칭성이란, 이벤트 구동(Event-driven)과 비동기적 대용량 데이터 전송에 최적화된 ROS2의 다중 스레드 풀(Thread Pool) 논리와, 폴링(Polling) 금지, 고정 크기 스택, 그리고 최소 인터럽트 지연(Minimum Interrupt Latency) 확보에 종속된 RTOS 로직 간의 충돌 상태를 의미한다.

  • 스레드 생명주기 및 메모리 비대칭: ROS2의 다중 스레드 이그제큐터(Multi-threaded Executor)는 통신 부하에 따라 유연하게 힙(Heap) 메모리를 동적 전개하지만, RTOS는 런타임 이전(Pre-runtime)에 태스크 객체 풀(Task Pool)과 스택 크기를 정적 사이징(Static Sizing)한다. 따라서 ROS2 계층에서 쏟아지는 이벤트 스톰(Event Storm)은 RTOS 수신부의 정적 컨텍스트를 즉각적으로 오버플로우시킬 위험을 내포한다.
  • 페이로드(Payload) 연산과 선점 비대칭: 컴패니언 컴퓨터는 방대한 용량의 시각적 측정치나 궤적 배열을 하나의 토픽으로 발행하지만, 이를 수신 파싱(Parsing)하는 RTOS 태스크 프레임워크는 해당 통신 인터럽트 루틴으로 인해 코어 역학 제어 루프의 실행 스케줄이 선점당하고 블로킹(Blocking)되는 최악 사례(Worst Case)에 직면할 수 있다.

4. 이기종 컨텍스트 브릿징(Heterogeneous Context Bridging) 메커니즘

이러한 실행 컨텍스트의 비대칭성으로 인한 타이밍 붕괴와 동기화 소실을 차단하기 위해, 시스템 아키텍트는 분리된 시리얼(UART) 또는 이더넷(Ethernet) 링크 상에 마이크로-ROS(micro-ROS) 에이전트-클라이언트 브릿지 혹은 MAVLink 프로토콜 역직렬화 전용 미들웨어를 배치한다.

이 브릿지 컴포넌트는 컴패니언 컴퓨터로부터 하달되는 대단위 비동기 콜백 데이터를 수신하여 우선순위 기반 원형 버퍼(Ring Buffer)에 안전하게 큐잉(Queueing)한다. 이후 DMA(Direct Memory Access) 컨트롤러 또는 최하위 태스크 인터럽트를 경유하여 RTOS의 강고한 사이클 주기에 맞춰 데이터를 점진적으로 주입(Incremental Injection)한다. 이 아키텍처는 고주파수 메인 제어 루프의 간섭을 회피하면서도 외부 지령의 데이터 일관성을 무결하게 연결한다.
