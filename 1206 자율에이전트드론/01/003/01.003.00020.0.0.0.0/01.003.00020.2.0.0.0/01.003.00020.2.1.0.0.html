## 0.1 데이터 경합 방어를 위한 콜백 스레드 직렬화 기법

멀티코어 마이크로프로세서를 기반으로 구동되는 현대의 ROS 2 비행 제어기(Flight Controller) 아키텍처 환경에서, 다중 통신 채널을 경유하여 비동기적으로 수신되는 고주파수 센서 데이터 콜백과 임무 지시 타이머 콜백은 동일한 노드 주소 공간의 메모리 객체 통제권에 대하여 필연적으로 빈번한 경쟁 상태를 촉발한다. 다수의 병렬 프로세서 스레드(Parallel Threads)들이 임의의 시스템 상태 클래스(예: 위치 공간 행렬 포인터, 내부 역학 변수 필드, 플라이트 모드 상태 트리)에 동시다발적인 쓰기(Write) 연산을 인가할 경우, 인스트럭션 연산의 비결정론적 교차로 인하여 메모리 버퍼가 기형적으로 오염되는 데이터 경합(Data Race) 오류가 발현된다. 런타임 중에 발동하는 시스템 수준의 치명적인 데이터 파손을 가장 견고하게 방어하기 위한 소프트웨어 공학적 해법이 바로 ’콜백 그룹 단위의 강제 직렬화(Serialization) 기법’이다.

ROS 2의 상호 배타적 콜백 그룹(Mutually Exclusive Callback Group)은 프레임워크 층위에서 이 무결성 직렬화 설계를 구현하는 강력하고 표준적인 스케줄링 모델이다. 이를 활용한 시스템 단위 직렬화 방어 기법은 다음과 같은 논리 절차로 아키텍처 위에 전개된다.

첫째, 단일 노드 내부에서 공통 공유 객체의 메모리 주소 영역(Shared Memory Object Pointer)을 상호 참조하거나 교차 조작해야 하는 연관 콜백 묶음(Bundle)을 도출한다. 이들을 단순히 분산된 개별 통신 객체가 아니라 논리적인 단일 상호 배타적 그룹으로 분류하여 엄격하게 캡슐화(Encapsulation)한다. 예를 들면, 200Hz의 빈도로 외부 환경 객체를 탐지하는 비전 콜백 블록과, 도출된 추종 궤적을 제어기에 인가하는 50Hz 주기의 비선형 칼만 필터(Kalman Filter) 콜백이 하나의 동일한 ’전역 환경 상태 행렬’을 참조 수정한다고 가정하자. 이 경우 설계자는 동시 접근에 의한 벡터 포인터 혼돈을 원천 차단하기 위해 반드시 아키텍처 레벨에서 두 함수를 유일한 단일 상호 배타적 블록으로 결속하여 디폴트 멀티스레드 이그제큐터(Multi-Threaded Executor) 풀에 전달해야만 한다.

둘째, 런타임 환경에 진입하면 이그제큐터 내부 스핀(Spin) 구동 메커니즘은 이 콜백 그룹 객체에 내장된 고유 락 부울린(Lock Boolean) 변수 상태 공간을 기반으로 경합 방어 파이프라인을 상시 가동시킨다. 스레드 풀 내부에 포진된 어느 하나의 물리 코어 스레드가 200Hz 빈도의 비전 데이터를 대기열에서 픽업(Pick-up)하여 시스템 연산 블록 안으로 전진시켰다면, 콜백 로직이 시작됨과 동시에 프레임워크는 해당 그룹 도메인 전체의 락 플래그 상태를 참(true) 논리로 스위칭한다. 이렇게 전이된 락 블록 구조는 시스템 버스 내의 수많은 다중 코어들이 완전한 유휴 상태(Idle)로 대기 중이라 할지라도, 대기열 구조체(Wait Set)에 선착된 위치 궤적 필터링 콜백을 결코 강제로 인취(Fetch)하지 못하도록 엄격히 구속(Bind)한다.

셋째, 직렬화의 한계선과 병목 시스템 스루풋 저항성이다. 프레임워크 기반으로 강제된 콜백 스레딩 직렬화 기법은, 애플리케이션 개발자가 수작업으로 시스템 콜(System Call)을 삽입하여 std::mutex 잠금 객체를 할당하고 해제 타이밍을 디자인하는 아키텍처링 리스크—구체적으로 락 해제 누락(Missing Release)으로 인한 노드 교착 상태(Deadlock)나 세그멘테이션 폴트(Segmentation Fault)의 발생—를 사실상 완벽하게 면제하는 학술적 강점을 보유한다.
다만 역설적으로 데이터의 연관성이 완전히 단절된 독립적 통신 콜백 파이프라인 영역까지 상호 배타적 구조를 관성적으로 남용하게 될 경우, 코어 자원의 막대한 휴면 상태 지속과 전체 드론 제어 노드의 스루풋(Throughput, 처리량) 급감이라는 치명적인 타이밍 트레이드오프(Trade-off)가 수반된다.

따라서 데이터 경합 인자를 제거하는 파이프라인 아키텍처의 완성도는 맹목적인 단일화가 아니다. 고도로 연산 집중적인 비행 제어 모듈 내에서 서브시스템 간 스레드 상호 의존성(Thread Inter-Dependency) 데이터 그래프를 도출해 낸 후, 오직 동일 메모리 블록을 필연적으로 시간 분할 점유해야만 하는 최소 필수 기능(Minimum Viable Routines)들만을 정밀하게 선별해 폐쇄형 상호 락 루프에 묶어 두는 최적화 튜닝 역량에 좌우된다고 학술적으로 묘사할 수 있다.
