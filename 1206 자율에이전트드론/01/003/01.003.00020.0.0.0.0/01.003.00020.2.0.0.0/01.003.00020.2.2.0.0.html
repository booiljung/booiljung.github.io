## 0.1 제어 루프와 센서 융합 루틴 간 엄격한 실행 순서 보장

자율 에이전트 드론의 이중 루프 코어 소프트웨어 아키텍처 내에서 비행 제어기 스택(Flight Controller Stack)과 센서 융합 루틴(Sensor Fusion Routine)은 상호 불가분하며 인과적인 데이터 의존성을 지닌다. 비선형 역학 모델을 추종하는 모터 추력 연산(Control Loop) 로직은 직전에 즉각적으로 최신화된 기체의 3차원 공간 상태 추정치(State Estimate)를 온전한 입력값으로 받아들여야만 한다. 반대로 센서 융합 모델인 초정밀 확장형 칼만 필터(EKF)는 제어기가 송출한 지시값 행렬을 다음 시간 스텝의 역학 예측(Predict Phase) 단계 입력으로 반드시 귀환(Feedback)받아야 한다. 이러한 논리적이고 수학적인 선후 관계를 추호의 오차 없이 훼손하지 않는 실행 순서 보장 메커니즘은 비행 안정성 확보를 위한 절대적인 전제 조건이다.

동기식(Synchronous) 단일 프로세서 환경과 달리 ROS 2의 분산 기반 메시지 패싱 컴퓨팅 환경에서는, 이종 주파수를 지닌 다발성 비동기 센서 이벤트들이 하위 네트워크 미들웨어(Middleware)를 거쳐 이그제큐터의 대기열(Wait Set)에 동시다발적으로 진입하게 된다. 이로 인해 메시지 도달 시점 및 코어 할당 경쟁에 따른 비결정론적 콜백 스케줄 역전(Inversion) 및 실행 순서 교차가 항시 잔존한다. 이를 프레임워크 층위에서 물리적으로 통제하고 엄격한 수학적 인과율을 보장하기 위하여 고안된 방법론이 바로 상호 배타적 콜백 그룹(Mutually Exclusive Callback Group)을 차용한 강제 동기화 아키텍처이다.

실시간성이 극대화된 진보적 형태의 실행 순서 보장 파이프라인 설계는 다음과 같은 전개 구성을 취한다.
첫째, 최저 실행 지연(Minimum Latency)이 요구되는 초당 400Hz 호출 주기의 기하학적 제어 루프 컴포넌트와, 고빈도(예: 100Hz에서 500Hz 사이)로 분출되는 관성 측정 장치(IMU) 기반 센서 융합 콜백을 여타의 백그라운드 프로세스로부터 완전히 이격시킨다. 이 두 핵심 함수 포인터들을 단 하나의 유일한 상호 배타적 콜백 그룹 공간(Namespace) 내에 철저히 응집시킨다. 이 그룹은 외부적으로 멀티스레드 이그제큐터(Multi-Threaded Executor) 풀에 매핑되어 다중 코어 자원의 수용 범위에 놓이더라도, 프레임워크 자체의 엄격한 상호 잠금(Mutual Lock) 변수에 의해 그 내부 메커니즘은 강제로 싱글 스레드화되어 보호받는 직렬 실행(Serial Execution)의 견고한 울타리를 형성한다.

둘째, ROS 2 콜백 스케줄러의 무작위성을 차폐하기 위하여, 상호 배타적 락 파이프라인 내부에 종속적 순차 호출(Dependent Chain Invocation) 기법을 설계한다. 이를테면 최우선 순위의 센서 융합 인터럽트가 도달하여 기체의 상태 추정 변수를 최신화하는 칼만 필터(Kalman Filter) 콜백 계층이 이그제큐터의 락을 획득하고 연산을 완료하는 즉시, 해당 콜백의 함수 말단(End-of-Function)에서 소프트웨어적인 컨디션 변수(Condition Variable)를 포지티브 전환(Trigger)하거나, 메모리 내부 큐를 매개체로 하여 체인(Chain)으로 결합된 후단 자세 제어 루프 연산 함수를 다이렉트로 인보크(Invoke)하는 로직 구조이다. 이 연결 과정에서 상호 배타적 콜백 그룹의 락 플래그는 외부 시스템에 여전히 차단 중(Locked)인 상태를 유지한다. 종국적으로 외부의 비결정적 저주파수 텔레메트리 정보나 사용자 서비스 호출이 두 핵심 인과 기능—상태 추정과 추력 역학 제어—실행 순간 사이에 난입(Interrupt)하는 참사를 프레임워크 락을 통해 물리적으로 배제시키는 확정성을 지닌다.

셋째, 데이터 벡터의 시간적 일관성(Temporal Consistency) 유효성 검증 계층이 병합 도입되어야 한다. 상호 배타적 그룹의 보호 아래 직렬화되어 무사히 실행되는 두 루틴이라 할지라도, 네트워크 지연 등의 외부 요인으로 제어기가 최종적으로 읽어 들인 센서 융합 결과물이 드론의 고유 물리적 제어 주기 임계치(예: 타겟 주파수 역산 2.5ms)를 초과하여 경과한 상태라면, 이른바 낡은(Stale) 데이터가 제어기에 삽입되어 앨리어싱(Aliasing) 및 발산 오차를 급증시킨다. 따라서 센서 융합 콜백에서 갱신 및 배포되는 상태 벡터 구조체는 언제나 하드웨어 동기화 타임스탬프 기반의 데이터 타당성 식별자(Timestamp Spinlock Identifier)를 필수 내장해야만 한다. 궁극적으로 제어 루프를 개시하는 도입부(Entry Point)에서 이 타임스탬프 슬롯의 데드라인 유효 여부를 검증(Verify)하는 방어 로직이, 상호 배타적 제어 락 영역 내에 반드시 파이프라인으로 보호되어야 전체 비행 스택의 완전한 인과적 체인 실행 순서가 입증된다.
