## 0.1 콜백 큐 블로킹이 실시간 제어 지터에 미치는 인과 관계

자율 비행 드론 제어 노드에서 상호 배타적 콜백 그룹(Mutually Exclusive Callback Group)을 차용한 이그제큐터 직렬화(Serialization) 설계는 메모리 자원에 대한 데이터 경합(Data Race)을 완벽하게 방어하는 뛰어난 역량을 발휘한다. 그러나 런타임 스케줄링의 이면을 분석하면, 이 보수적인 동기화 아키텍처는 이그제큐터(Executor) 계층 내부의 ’콜백 큐 블로킹(Callback Queue Blocking)’이라는 구조적 페널티(Penalty)를 시스템 파이프라인 내에 잉태한다. 이러한 큐 레벨의 블로킹 현상은 하위 실시간 커널(RTOS)의 인터럽트 해상도가 아무리 정밀하게 보장되더라도, 실제 애플리케이션 계층 내 비행 제어 연산 공간으로의 진입 시점을 가변적으로 지연시킴으로써 종국적으로 시스템 전반의 실시간 제어 지터(Jitter)로 파급되는 명백하고도 치명적인 물리적 인과 관계를 형성한다.

ROS 2 프레임워크(Humble/Iron 기준)의 이그제큐터 동작 모델에서 대기열(Wait Set) 내에 적재된 어느 특정 콜백 인스턴스가 호출되어 스레드 풀의 연산 제어권을 배타적으로 장악하게 되면, 동일한 상호 배타적 단위로 편성된 다른 모든 보류 및 대기 콜백들은 실행 페이즈(Execute Phase)로의 진입이 원천 거부된 채 메모리 큐에 적체된다. 이때, 제어권을 선점한 호출 콜백(Preempting Callback) 내부의 연산 소요 시간인 최악 실행 시간(WCET, Worst-Case Execution Time)이, 비행 제어기 모델이 강제하는 최소 타겟 주기(Target Period, 예: 400Hz 시스템에서의 2.5ms)의 여유 마진을 초과하게 될 경우, 후속으로 대기 중이던 긴급 제어 콜백들의 디스패칭 시점은 블로킹 시간에 비례하여 누적 연기된다.
이러한 블로킹 오버헤드는 고정된 주기 간격으로 칼같이 만료되고 실행 구문으로 전이되어야 할 핵심 타이머 콜백의 실제 기동 시간(Actual Wake-up Time)을 원래 배정된 타이밍 위상(Timing Phase)의 우측으로 강제 이탈시킨다. 시간 축 위에서 발현되는 이 디스패칭 오차의 통계학적 분산(Variance)이 바로 시스템 제어 이론 역학에서 규정하는 ‘지터(Jitter)’ 현상의 본질이다.

소프트웨어 스케줄링 결함으로 잉태된 이 제어 지터가 드론의 물리적 피드백 루프에 미치는 역학적 인과성은 시스템 파괴적(Destructive)이다.
초정밀 제어 주기의 위상이 불규칙하게 진동(Fluctuation)할 경우, PID 계열 혹은 LQR(Linear Quadratic Regulator) 기반의 제어기 내부 시간 적분(Integration) 및 미분(Derivation) 항의 기반 전제가 되는 \Delta t(시간 변화량) 값이 논리적으로 고정된 전제치(예: 0.0025s)와 실제 마이크로커널의 물리적 경과치 사이에서 심각하게 괴리되는 현상을 유발한다. 이 미스매치(Mismatch)는 즉각적으로 모터 인버터 믹서(Mixer)에 인가되는 PWM 출력 계단 파형의 비연속성과 제어의 기하학적 발산(Divergence)을 초래한다. 가령 초고속으로 동작하는 코어 자세 제어기의 런타임 지터 폭이 임계 한계인 1ms를 넘어서 진동하게 되면, 드론 편대는 수직 하강 풍(Downwash)이나 돌풍(Gust)에 대한 역학적 복원력(Resilience)을 순간적으로 상실하며 나이퀴스트-섀넌(Nyquist-Shannon) 샘플링 정리의 위반 한계 영역을 타격, 결과적으로 추력 제어 불능에 따른 동역학적 실속(Stall) 및 군집 추락 사태에 직면하게 된다.

결론적으로 콜백 큐 블로킹과 제어 루프 지터 간의 함수적 인과 관계는, 컴포넌트의 단순한 기능적 연관성에만 얽매여 이질적인 체류 시간을 지닌 여러 콜백들을 맹목적으로 하나의 상호 배타적 그룹에 압축시키는 안일한 스레딩 아키텍처가 실시간 비행 제어 시스템에서 얼마나 재앙적인 병목(Bottleneck) 인자가 될 수 있는지를 학술적으로 엄중히 입증한다. 시스템 통합 설계자는 큐 블로킹의 종속 고리를 단절하기 위하여 WCET가 극단적으로 짧고 데드라인 준수가 핵심인 루프 콜백들만을 정밀하게 선별 이격하여, OS 레벨의 전용 실시간 마이크로 이그제큐터(Micro-Executor) 스레드 위에서 생존시키는 극단적인 타이밍 토폴로지 분리 원칙을 필히 고수해야 한다.
