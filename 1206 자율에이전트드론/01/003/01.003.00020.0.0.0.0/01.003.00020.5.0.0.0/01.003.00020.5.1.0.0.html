## 0.1 이기종 콜백 토폴로지의 메모리 접근 위반 차단

다수의 코어 스레드를 관장하는 멀티스레드 이그제큐터(Multi-Threaded Executor) 위상 환경이나, 복수의 이질적인 상호 배타적/재진입(Reentrant) 콜백 그룹 집합체들이 하나의 물리 메모리 체계에 거대하게 융합된 복합 프레임워크 시스템 내에서, 성격과 주파수가 근본적으로 상이한 이기종 콜백 스레드 연산들이 ROS 2 기체의 전역 데이터 모델(Global Data Model)을 향해 비동기적으로 동시 교섭할 때 가장 빈번하게 발생하는 시스템 레벨의 파괴적 결함은 바로 메모리 접근 위반(Memory Access Violation, 즉 세그멘테이션 폴트) 파손이다.

현상학적으로 비전 카메라 인터럽트 콜백 스레드가 고해상도의 이미지 매트릭스를 전역 힙(Heap) 메모리 공간에 포인터 형태로 할당하여 막대한 연속 쓰기(Write) 인스트럭션을 집행하는 찰나의 시간 분할 윈도우 안에, OS 스케줄러에 의해 선점된 네비게이션 경로 탐색(Global Planner) 콜백 스레드가 해당 메모리의 완전성이 검증되지 않은 덜 완성된 절반 영역을 불쑥 읽어(Read) 추종 연산을 강행하게 되면, 배열 인덱스 어보트 결함 등 단순한 프레임워크 예외 던지기 수준을 넘어 해당 제어기 프로세스 자체가 그 즉시 즉사(Killed)하여 드론이 추락하는 치명적인 상태로 귀결된다. 컴퓨터 공학에서는 이를 수학적으로 통제 불가능한 레이스 컨디션(Race Condition) 혹은 데이터 경합(Data Contention)이라 명명하며, 단 한 번의 예외도 허용치 않는 하드 실시간(Hard Real-time) 비행 생존성이 뒷받침되어야 하는 자율 제어 노드 구획 내에서는 소프트웨어 설계적으로 전면 원천 차단 통제해야만 한다.

거대한 이기종 콜백 토폴로지 공간 상에서 이와 같은 파멸적 메모리 접근 위반 충돌을 완전 차폐하는 시스템 아키텍처 측면의 핵심 기법은 로컬 스냅샷(Local Snapshot) 패턴 메커니즘과 더블 버퍼링에 입각한 쓰기 시 복사(Copy-on-Write, COW) 전략의 철저한 도입이다.
첫 번째 원칙으로, 고주파 상호 배타적 제어 동역학 루프를 1순위로 관장하는 프라이머리 콜백(Primary Thread Callback) 구문은, 비행에 중대한 영향을 미치지 않는 여타의 모든 백그라운드 잉여 콜백들(예: 디스크 데이터 로거, 대용량 텔레메트리 스트리밍 송신 루틴)과 그 어떠한 형태의 직접적인 C++ 원시 포인터(Raw Pointer)도 단일하게 공유해서는 절대 안 된다. 범용 메모리 구조에서는 OS나 이그제큐터 레벨의 비동기적 컨텍스트 스위칭(Context Switching) 칼날이 언제 어느 행렬 번지를 파고들어 분할할지 수학적으로 확정할 수 없기 때문이다.
그 안전한 구조적 대안으로서 설계자는 기체 상태 객체를 철저히 std::shared_ptr와 불변 객체(Immutable Object Design) 모델링 복제본 클래스로 캡슐화하여 묶어두어야만 한다. 하위 백그라운드 콜백이 인출(Fetch) 연산을 시작하려 할 경우, 이그제큐터는 락 충돌 방지를 위하여 원본 공유 메모리 징벌 스핀 대신, 당시 트리거 시점에서의 완전한 상태 행렬 인스턴스를 통째로 해당 백그라운드 스레드의 로컬 메모리 스코프(Local Memory Stack/Scope)에 즉시 깊은 복사(Deep Copy) 파생시키거나 소유권 한정 양도 스왑(Swap)을 걸어주는 스냅샷 아키텍처 방어망을 운용해야만 메모리 겹침 오염을 근절할 수 있다.

두 번째 원칙은 이질적 그룹 간 메모리 락 획득 순서 체계의 엄격한 유향 그래프(DAG) 모델링이다. 각기 다른 트리거 생명주기를 지닌 이기종 콜백 인스턴스 인프라 간에 불가해하게 복수의 블로킹 락 포인터인 std::mutex를 서로 엇갈려 역방향 소유하려 시도하다가 영구 대기 데드락(Deadlock)에 빠지는 시스템 붕괴 현상을 방패로 막기 위하여, 비행 노드 컴포넌트 내부에 생성된 모든 자원 잠금(Lock)의 종속 위상을 단방향 비순환 그래프(Directed Acyclic Graph) 역학 형태로 수학적으로 단일 배치한다.
즉, 콜백 스레드 하나가 복합 연산을 위해 상이한 복수의 메모리 자원에 연쇄 진입하고자 할 때는 반드시 정해진 일방통행의 상위 계층 락 클래스 번지점부터 시작하여 종단 하위 락 순서 벡터축 방향으로만 진입 포인터를 얻어내도록 강제하는 ‘토폴로지 락 계층화(Topology Lock Hierarchy)’ 방어 명세를 엄중 확립하여야 한다. 이 원칙을 컴파일 및 정적 소스 코드 통합 레벨에서부터 사전 인가하여야만 비로소 복잡다단한 멀티 스레드 노드 내부망에서 이기종 포인터 오염과 메모리 교착 파손 증명에 의한 비정상 비행 불능 상태를 프레임워크에서 물리적으로 온전히 근절 선언할 수 있다.
