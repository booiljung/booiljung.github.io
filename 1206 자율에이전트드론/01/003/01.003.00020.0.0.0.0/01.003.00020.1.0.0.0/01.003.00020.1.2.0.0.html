## 0.1 Mutually Exclusive 및 Reentrant 콜백 그룹의 동기화 차이

ROS 2의 분산 비행 제어기 노드 아키텍처 내에서 단일 프로세스의 동시성 제약을 극복하고 콜백 실행의 타이밍 결정론성을 정밀하게 설계하기 위해서는, rclcpp 클라이언트 라이브러리가 제공하는 두 가지 핵심 콜백 그룹 인터페이스의 동기화(Synchronization) 메커니즘 차이를 명확하게 규명해야 한다. 객체 지향적으로 선언된 콜백 그룹의 타입(Type)은, 프레임워크 내부의 스핀(Spin) 스케줄러가 이벤트 대기열(Wait Set)에 진입한 콜백을 추출하여 다중 스레드 풀(Thread Pool)에 디스패치(Dispatch)할 때, 락(Lock) 체결 여부와 재진입 허용 권한을 강제하는 절대적인 지표가 된다.

첫째, 상호 배타적 콜백 그룹(Mutually Exclusive Callback Group)이다.
이 콜백 그룹은 편입된 모든 콜백 인스턴스에 대하여 엄격한 ‘단일 스레드 점유’ 원칙을 강제하는 단일화된 뮤텍스(Mutex) 동기화 메커니즘을 적용받는다. 이그제큐터(Executor)가 대기열을 폴링(Polling)하여 해당 그룹 내의 특정 콜백 A를 실행 계층(Execute Phase)으로 전환하고 물리적 스레드에 할당하면, 프레임워크 내부의 그룹 논리 플래그(Flag)는 즉시 차단(Locked) 상태로 전이된다. 이로 말미암아 콜백 A가 완전히 연산을 종료하고 제어권을 반환(Return)하기 전까지는, 논리적으로 동일한 상호 배타적 그룹으로 결속된 서브스크립션 콜백 B나 타이머 콜백 C에 새로운 하드웨어 수신 이벤트가 도달하여 준비 상태(Ready)가 되더라도 절대 병렬 코어로 디스패치되지 않는다.
이러한 강고한 동기화 차단 특성은 전역 변수 풀이나 드론의 메모리 임계 구역(Critical Section)을 제어할 때 큰 이점이 있다. 소프트웨어 엔지니어가 C++ 레벨의 자체적인 동기화 객체(예: std::mutex 혹은 std::shared_mutex)를 인위적으로 작성하지 않더라도 메모리 경쟁 현상인 데이터 레이스(Data Race)를 프레임워크 차원에서 봉쇄해 준다는 강력한 무결성 보장을 제공한다. 반면, 그룹 내부의 어느 한 콜백 함수 로직에 지연 시간이 긴 연산 병목 구조(예: 대용량 이미지 직렬화)가 존재할 경우, 그에 종속된 여타 고주파 제어 콜백들이 연쇄적으로 대기하는 후위 블로킹(Trailing Blocking) 현상을 필연적으로 양산하게 된다.

둘째, 재진입 가능 콜백 그룹(Reentrant Callback Group)이다.
이 구조 모델은 앞서 서술한 전역 락 메커니즘을 사실상 해제하여, 해당 그룹에 등록된 복수의 이종 콜백들이 가용한 멀티스레드 이그제큐터(Multi-Threaded Executor) 환경 내에서 어떠한 프레임워크적 차단 제약 없이 자유롭게 병렬(Parallel) 진입할 수 있도록 설계된 개방형 위상 구조이다. 이를 통해 이벤트 큐 구조체에 도달한 다수의 콜백 이벤트는 스레드 풀 자원이 허용하는 한 즉각적이고 독립적으로 물리 프로세서 코어들에 분산 할당되어 동시 연산에 돌입한다.
특히 제어 모듈이 참조 투명성(Referential Transparency)을 가진 순수 함수(Pure Function)로 구축되어 있거나, 내부에서 소비되는 노드 메시지 간의 상호 의존성이 전혀 없는 고정밀 수치 해석(예: 외부 환경 벡터의 비가역적 변환)일 경우 이 동기화 모델은 노드의 전체 스루풋(Throughput) 역량을 극대화하는 가장 이상적인 성능 지표를 산출한다.
그러나 이 구조가 지니는 가장 큰 학술적 취약점은 바로 메모리 자원 접근에 대한 동시성 통제와 스레드 안전성(Thread Safety) 보장 책임이 ROS 2 프레임워크가 아닌 온전히 애플리케이션 개발자 측으로 위임된다는 사실이다. 다중 코어가 동일한 인스턴스의 컴포넌트 내부 변수에 동시 쓰기(Concurrent Write) 연산을 인가할 경우 비결정론적 상태 파손 내지 치명적인 세그멘테이션 폴트(Segmentation Fault)가 초래되며, 비행 제어 노드 전체가 즉사(Crash)하는 심각한 폴트 라인으로 작용한다. 따라서 내부 자료구조 접근 구간에는 엄격한 락 프리(Lock-Free) 링 버퍼나 원자적(Atomic) 변수 업데이트 기술이 요구된다.

요약하자면 프레임워크 수준의 상호 배та적 구조는 상호 락 메커니즘을 매개로 직렬화(Serialization)를 통한 결정적 안전성을 보장하되 코어의 공간적 병렬성을 대폭 상실하는 반면, 재진입 모델은 병렬 실행 성능을 극대화하는 대신 노드 소프트웨어 내부 자원에 대한 자체적인 동시성 차폐 로직 설계를 전제로 한다는 극명한 시스템 프로그래밍 패러다임의 양극화(Polarization)를 시사한다.
