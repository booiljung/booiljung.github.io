## 0.1 단일 노드 내 다중 콜백 스레딩 및 스케줄링 메커니즘

비행 제어 소프트웨어 아키텍처상 단일한 자율 드론 제어 노드(Node) 객체 내부에서 여러 다중 콜백 구조체들이 프로세서 자원을 점유하고 실행을 완수하는 일련의 전체 생애주기(Lifecycle) 메커니즘은, 프레임워크 최상위의 이그제큐터(Executor) 계층과 하위 C 라이브러리인 RCL(ROS Client Library)의 대기열(Wait Set) 갱신 로직 간의 상호작용에 의해 완전하게 통제된다. 단일 노드 내 스케줄링 메커니즘의 구조적 근원은 폴링(Polling) 기반의 단일 지향 큐 컴파일 스레딩에 있으며, 이는 복수의 다양한 이벤트가 노드에 도착했을 때 이그제큐터가 어떠한 규칙과 정책을 매개로 이벤트를 대기열에서 추출(De-queue)하는지를 결정짓는 핵심 아키텍처이다.

ROS 2 생태계 런타임의 디폴트 스케줄링 메커니즘은 기본적으로 이벤트를 폴링 상태로 기다리는 ’대기 페이즈(Wait Phase)’와 충족된 조건 연산을 실제로 호출하는 ’디스패칭 페이즈(Dispatching Phase)’의 강제 순환 루프로 구성된다.
하위 미들웨어 네트워크 레이어(예: Fast DDS 혹은 Cyclone DDS)를 경유하여 새로운 토픽 메시지가 도달하거나, RTOS(Real-Time Operating System) 커널 시계를 참조한 타이머(Timer)의 만료 주기가 도래하였을 때, 수면(Sleep) 상태에 있던 이그제큐터는 내부의 하드웨어 인터럽트를 통해 즉각 대기열을 해제(Wake-up)한다. 이 과정 직후 이그제큐터는 RCL 추상화 계층에 메모리 컨텍스트를 질의하여, 현재 논리적으로 호출 가능한 모든 타이머, 서브스크립션, 서비스 서버, 그리고 클라이언트 액션 인스턴스들의 객체 포인터 목록 체계를 수집 구축한다.
이 지점에서 소프트웨어 공학적으로 특기할 만한 사실은 기본 이그제큐터 구조가 철저한 라운드 로빈(Round-Robin) 기법에 기초한 비선점형(Non-preemptive) 고정 우선순위 스케줄러 방식을 채택하고 있다는 것이다. 다시 말해, 맵 데이터 파싱과 같은 지연 연산 성격이나 비행 자세 변환이라는 초임계 데드라인의 긴급도와 무관하게, 항상 “타이머 \rightarrow 서브스크립션 \rightarrow 서비스 \rightarrow 클라이언트“의 아키텍처로 하드 코딩(Hard-coding)된 정적 카테고리(Static Category) 순열에 따라 대기열의 콜백 객체가 차례로 인출된다.

이러한 정적 스케줄링 체계 하에서 다중 코어 자원을 상정한 멀티스레드 이그제큐터(Multi-Threaded Executor)가 단일 노드 객체에 주입되었을 때, 다중 콜백의 스레딩 연산 과정은 복합적으로 전개된다.
사전 할당된 스레드 풀(Thread Pool) 내의 스레드들은 이그제큐터의 글로벌 상태 머신인 전역 뮤텍스(Global Mutex)를 매개로 wait_for_work() 구문 루프에서 치열하게 경쟁 대기한다. 대기열에 진입한 이벤트 구조체에 대하여 최선착 락(Lock)을 획득한 특정 유휴 스레드(Idle Thread)가 해당 콜백의 포인터를 낚아채어 자체 실행 힙(Heap) 공간으로 격리시킨다. 이때 만약 호출이 승인된 콜백이 런타임 생성 시점에 상호 배타적 콜백 그룹(Mutually Exclusive Callback Group) 소속으로 명시되었다면, 스케줄러는 해당 콜백 그룹 전체 주소 공간의 플래그를 일시적 ‘차단’ 상태로 전이시킨다. 이를 통해 풀 내에 대기 중인 다른 잉여 유휴 스레드들이 동일 그룹 소속의 또 다른 대기 콜백을 절대 인취(Fetch)하지 못하도록 엄격히 차폐한다. 반면, 재진입 가능(Reentrant) 그룹 속성일 경우 이러한 상태 플래그 차단 동기화 과정 없이 자유롭고 동시다발적인 원시 인취를 지속 허용함으로써 코어 스레드의 공간적 동시성을 강제로 극대화한다.

단일 노드 아키텍처 내부에서 이처럼 스크린 된(Screened) 이그제큐터 레벨의 스케줄링 및 락 메커니즘을 투명하게 분석하는 절차는, 체계 통합 설계자로 하여금 고주파 고정밀 비행 제어 루프의 실행 위상이 어떠한 병목 층위에서 훼손될 수 있는지를 입체적으로 입증하게 한다. 고해상도 하드웨어 타이머 콜백과 대용량의 잦은 저주파수 메시지 수신 연산이 락 메커니즘의 보호막 없이 단일한 이그제큐터의 라운드 로빈 큐를 통제 없이 공유하고 있을 경우, 멀티스레딩이라는 표상적인 병렬 컴퓨팅 구조 뒤편에 은닉된 대기열 갱신 병목 비용(Wait Set Overhead)과 고정 카테고리 우선 처리 로직에 의하여 치명적인 시스템 디스패칭 지터(Dispatching Jitter)를 필연적으로 유발하게 된다. 이는 역설적으로 분산 실시간 처리 시스템에서 노드의 물리적 통폐합보다 콜백 그룹의 논리적 단일화 및 이그제큐터 격리 패러다임이 우선적으로 강제되어야만 하는 확고한 학술적 당위성으로 작용한다.
