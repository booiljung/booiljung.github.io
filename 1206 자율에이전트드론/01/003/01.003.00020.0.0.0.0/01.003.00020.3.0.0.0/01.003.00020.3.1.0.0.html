## 0.1 I/O 바운드 및 CPU 바운드 콜백 간 비동기 오버랩 처리

자율 에이전트 드론 제어 노드를 구성하는 이질적이고 다종다양한 콜백 구조체들은 프로세서 자원 점유 특성과 병목(Bottleneck) 기원에 따라 크게 I/O 바운드(I/O-Bound) 콜백과 CPU 바운드(CPU-Bound) 콜백이라는 두 가지 상반된 컴퓨팅 프로파일(Computing Profile)로 대별된다.
원격 지상 통제소(GCS)로 전송할 고해상도 비전 데이터를 압축 직렬화(Serialization)하여 네트워크 소켓에 밀어 넣거나, 하드웨어 저수준 인터페이스(예: UART, I2C, SPI) 버스로부터 외부 계측기의 응답 페이로드를 읽어 들이기 위해 대기하는 인터럽트 대기 작업은 시스템 버스 및 외부 통신 지연에 전적으로 종속되는 전형적인 I/O 바운드 연산에 속한다. 반대로, 수집 방출된 라이다(LiDAR) 3차원 포인트 클라우드에 기반하여 실시간 유클리디안 경로 탐색 알고리즘(Euclidean Path Planning)을 연산하거나, 기체 6축 역학 상태 행렬의 비선형 예측 모델링(Non-linear Estimation)을 수행하는 종단 연산은 시스템 내 방대한 부동소수점 연산기(FPU) 대역폭과 ALU 레지스터를 극심하게 소모하는 전형적인 CPU 바운드 작업에 해당한다.

통상적인 ROS 2(Humble/Iron) 기본 설계인 단일 상호 배타적 콜백 그룹 체제에서는 콜백의 하드웨어적 본질적 성격과 무관하게, 어느 하나의 I/O 바운드 콜백이 하드웨어 패킷 응답을 기다리며 실행 제어권을 파지한 채 커널 블로킹(Blocking) 상태에 돌입해 버리면, 물리적 CPU 연산 코어는 연산을 멈춘 유휴 상태(Idle)임에도 이그제큐터 그룹 락(Lock) 정책에 의해 전체 노드의 수학적 계산 활동이 완전히 마비된다. 이러한 치명적이고 비효율적인 코어 자원 낭비 타임라인을 타파하고, 전체 서브시스템의 컴퓨팅 스루풋(Throughput, 처리량)을 나이퀴스트 물리 한계치까지 극대화하기 위한 시스템 레벨의 설계 최적화 공법이 곧 다중 재진입(Reentrant) 콜백 그룹을 필두로 한 학술적 ‘비동기 오버랩(Asynchronous Overlap)’ 처리 아키텍처이다.

프레임워크 층위에서의 비동기 오버랩 연산 설계 원리는, 리눅스 운영체제 계층이 제공하는 스레드 간 컨텍스트 시분할(Time-Sharing Context) 격리 특성을 ROS 2 런타임 추상화 위로 강제 끌어올려 융합하는 데 기반한다.
그 구현 절차로서 첫째, 코어 연산이 지배적인 CPU 바운드 콜백들과 외부 하드웨어 인가 통신 위주의 I/O 바운드 컴포넌트들을 단일한 형태의 “재진입 가능 콜백 그룹(Reentrant Callback Group)“으로 느슨하게 종속 편입시킨다. 이를 통해 이그제큐터(Executor) 객체는 이벤트 대기열(Wait Set)에서 통신 개시 이벤트나 플래닝 타이머가 추출되는 즉시, 멀티 스레드 풀(Thread Pool) 유휴 코어들에 해당 인스트럭션 블록들을 교차 배제(Mutual Exclusion) 단계 없이 선형 동시 매핑한다.
둘째, 이 매핑된 물리 코어 위에서 I/O 바운드 콜백 스레드가 하드웨어 응답 지연이나 네트워크 소켓 타임아웃(Socket Timeout) 대기로 인하여 select 내지 sleep과 같은 커널 레벨의 블로킹(I/O Wait Period) 인터럽트에 빠지는 찰나의 순간, OS 스케줄러 계층과 재진입 설계 위상을 거쳐 자유로워진 여분의 멀티 코어 유휴 슬롯 혹은 컨텍스트 스위칭(Context Switching)된 기존 코어의 빈 공간에 인과성이 없는 CPU 바운드 연산 콜백 스레드가 즉각적으로 파고들어(Overlap) 삽입 연산된다. 컴파일러는 이 여유 공간 안에서 남은 프로세서 명령어 사이클을 빈틈없이 타이트하게 소모 연산한다.

이러한 극한의 오버랩 스케줄링 구조가 오류 없이 올바르게 관철되기 위해서는, 컴포넌트 연산 로직 내부에서 객체 간의 상태 의존성(State Dependency)을 완벽하게 차단 분리해야 하는 전제가 따른다. 데이터 로거 I/O 콜백이 디스크 메모리에 로깅 버퍼 블록을 플러싱(Flushing) 하느라 스레드를 연장 체류시키는 동안, 병렬 오버랩으로 진입한 CPU 바운드 콜백이 하필 그 동일한 로깅 버퍼 주소 포인터를 조작하려 들면 즉각 메모리 세그멘테이션 오류 파손(Segmentation Fault)이 야기되어 전체 프로세스가 즉사하기 때문이다.
따라서 설계자는 I/O 송출 지연 콜백이 네트워크나 디스크로 데이터를 덤프 전송하기 직전 단계에서, 원본 데이터에 대한 독립적인 로컬 복사본이나 더블 버퍼링(Double Buffering) 아키텍처를 기반으로 변조 불가한 데이터 스냅샷(Snapshot) 메모리를 사전에 안전하게 분리 이격 확보하여야 한다. 이 방어선이 구축된 후에서야 해당 콜백은 그룹 락(Lock) 없이 독립된 재진입 가능 병렬 스레드 위에 오버랩 도출될 수 있다.
결과론적으로 재진입 위상 그룹 안에서의 이질적인 CPU/IO 콜백 간 오버랩 아키텍처 설계는, 물리적 드론 엣지 보드에 탑재된 제한된 MPU 연산 자원의 시간 분할 공백 시퀀스를 완벽하게 메워 압축함으로써, 대규모 데이터 처리 하중에서도 비행 제어 반응성의 획기적인 지연 감축(Latency Minimization)을 이론적으로 성취한다.
