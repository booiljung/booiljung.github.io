## 0.1 다중 콜백 그룹 간 부하 분산 매핑을 통한 병목 해소

거대한 단일 자율 드론 제어 노드 내에서 기능적 역할과 위상별로 철저히 분리 구획된 다중 콜백 그룹(Callback Group)들—초고주파 타이머, 텔레메트리 통신, 대용량 센서 인터페이스 등—이 각자의 데드라인(Deadline) 내에서 안정적인 처리율(Throughput)을 영구 유지하기 위해서는, 분리된 이 개별 콜백 그룹들을 다중 코어 자원을 추상화하여 관리하는 최상위 운영 주체인 이그제큐터(Executor)에 매핑(Mapping)함에 있어 산술적인 부하 분산(Load Balancing) 토폴로지 최적화 설계가 필수적으로 강제된다.

본래 ROS 2 프레임워크의 기본 클래스 아키텍처에서는 노드(Node) 초기화 단계 시 프로그래머가 코드 상 명시적인 매핑 규칙 포인터를 부여하지 않을 경우, 선언된 모든 이종 콜백 그룹 구조체들이 단일한 범용 이그제큐터의 대기열(Wait Set) 큐에 암시적이고 무작위적으로 병합(Merge)되어 버리는 구조적 한계를 띠고 있다. 이는 결과집행적으로 OS 스케줄러 층위에서 특정 빈 물리 코어로 유휴 작업이 편중되거나 반대로 한 코어의 스핀 사이클 안에 헤비급 비전 연산 콜백들이 중첩 동시 할당되어 치명적인 런타임 제어 병목(Bottleneck)이 촉발되는 예측 불가능한 스케줄링 볼륨 불균형을 야기한다.

이러한 무작위적 간섭의 스레딩 한계를 타개하고 비전과 동역학이라는 상극의 요소를 원활히 공전시키기 위한 시스템 부하 분산 매핑 기술은 크게 두 가지 융합 아키텍처 트랙으로 전개된다.

첫째, 특수 목적 이그제큐터 분할 물리 할당(Dedicated Executor Partitioning) 기법의 도입이다. 전체 거대 비행 드론 컴포넌트 내부의 콜백 연산들을 주기성(Periodicity)과 하드웨어 점유 성격에 따라 세분화한 뒤, 메인 노드 런타임 컨텍스트 스위처(Context Switcher) 계층에서 가용할 독립 이그제큐터의 인스턴스(Instance)를 논리적으로 복수 생성(Instantiation)한다.
예를 들어 고화질 전방향 비전 카메라 수신과 압축을 전담하는 I/O 인터럽트 위주의 재진입(Reentrant) 그룹 뭉치는 스레드 풀(Thread Pool) 크기가 가용 코어 수(예: 4~6코어) 수준으로 확장된 ‘커스텀 멀티스레드 이그제큐터(Multi-Threaded Executor)’ 객체 E_A에 executor.add_callback_group() 시스템 호출로 직접 분배 등록한다. 반대로 추락 방어 시스템과 정밀 동역학 제어 모멘트 배분을 전담하는 최상위 등급의 상호 배타적(Mutually Exclusive) 제어 타이머 그룹은, 코어 스핀이 단독으로 매핑 채번된 ‘단일 스레드 이그제큐터(Single-Threaded Executor)’ 객체 E_B에 수동 포인터 방식으로 독립 연결 배분한다.
이러한 아키텍처 분배망 구축을 통하여, 예측 불가한 네트워크 및 디스크 통신 지연에 종속된 I/O 인스트럭션이 마스터 동역학 코어의 실행 스핀 클럭을 갉아먹는 메모리 침범 및 간섭 현상이 프레임워크 상단 층위에서부터 물리적으로 원천 차단 분산된다.

둘째, 스레드 친화도(Thread Affinity) 기반의 OS 커널 하드웨어 매핑 최적화 설계이다. 리눅스 기반 실시간 커널(예: Ubuntu PREEMPT_RT 패치) 위에서 동작하는 ROS 2 멀티 이그제큐터 모델의 진정한 의미의 부하 분산은 바로 OS 계열 코어 친화도 명령(sched_setaffinity 등)과의 시스템 레벨 접목을 통해서만 비로소 완성된다. 고빈도 상호 배타적 콜백 제어 그룹을 독점 관장하는 E_B 단일 스레드 이그제큐터의 스레드 ID 식별자를 획득하여 마이크로프로세서의 가장 고립되고 여유로운 단일 코어(예: Core 1) 공간에 영구적으로 바인딩(Binding)시킨다. 이후 OS 커널의 글로벌 스케줄러 계층을 조작하여 여타의 범용 백그라운드 프로세스가 이 전용 제어 코어(Core 1)로 컨텍스트 스위칭하여 밀고 들어오지 못하도록 스케줄 차폐(Shielding) 및 격리(Isolation) 세팅을 단행한다. 반면, 단순히 해방된 스루풋 처리량 극대화가 유일한 목적인 E_A 범용 이그제큐터는 나머지 대칭형 멀티프로세싱(SMP) 코어군에 C-Group 폴리시를 적용하여 유동적으로 자율 배분 순환시킨다.

전술한 이러한 다층 물리-논리 분산 부하 구조는, 종래의 단일 이그제큐터 내부 공간이 짊어지던 단일 큐 중심의 폴링 루프 한계, 즉 비대한 라운드 로빈(Round-Robin) 대기열 갱신 오버헤드(Wait Set Update Overhead)의 체류 기간을 파생된 이그제큐터 개체 수만큼 쪼개어 거시적으로 잘게 분쇄 소거해 내는 기계적 이점을 갖춘다. 비행 제어 스택에서 콜백 그룹의 소프트웨어 무결성을 정의하는 것이 상호 배제 이론의 영역이라면, 실제 마이크로초(\mu s) 단위의 타이머 마감 기한(Deadline) 준수를 직접적으로 물리 차원에서 결정짓는 실행 환경에서의 인과론은 바로 이 이그제큐터 부하 분산 매핑론에 입각한 능동적 병목 해소 모델이라 학술적으로 정의 요약할 수 있다.
