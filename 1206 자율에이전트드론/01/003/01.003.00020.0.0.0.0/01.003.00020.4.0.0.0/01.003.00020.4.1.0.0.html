## 0.1 실시간 타이머 콜백 방어를 위한 독립형 Mutually Exclusive 컨테이너

자율 비행 드론의 핵심 동력학 제어 루프를 관장하는 하드-실시간 타이머(Hard Real-time Timer) 콜백은 6자유도(6-DoF) 비행 시스템 전체의 인과율을 통제하는 마스터 심박(Master Heartbeat) 함수와 같다. 이 타이머 기반 제어 루프가 나이퀴스트-섀넌(Nyquist-Shannon) 샘플링 정리를 만족하는 고주파수 제어 임계값(예: 400Hz 내지 1000Hz) 영역에서 기하학적 추력(Thrust) 및 모멘트 분배를 한 치의 오차 없이 정밀 연산하기 위해서는, 해당 프로세서가 그 어떠한 외부 통신 인터럽트 블로킹이나 운영체제 수준의 스케줄링 간섭으로부터도 철저하게 배제되고 방어되어야 한다. 이를 소프트웨어 미들웨어 층위에서 아키텍처적으로 완성하기 위한 고도화된 스케줄 패턴이 바로 ‘제어 루프 타이머 콜백 방어 전용의 독립형 상호 배타적 컨테이너(Standalone Mutually Exclusive Container)’ 통합 기법이다.

ROS 2(Humble/Iron) 기본 노드 생성자(Constructor)의 디폴트 초기화 환경 하에서는, 명시적인 스케줄러 위상 분리가 선행되지 않는 한 개발자가 선언한 타겟 타이머 개체와 수많은 대용량 서브스크립션 메시지큐, 그리고 무거운 서비스 서버 인스턴스들이 모두 단일한 기본 콜백 그룹(Default Callback Group)의 메모리 공간 안에 암시적으로 범용 통합 할당되어 버린다. 만일 멀티스레드 이그제큐터(Multi-Threaded Executor) 위에서 이러한 안일한 디폴트 구성이 스핀(Spin) 구동될 경우, 방대한 대역폭의 텐서 연산을 요하는 비전 처리 서브스크립션 스레드나, 경로 탐색을 맵핑하는 장기 체류 실행(Long-running) 콜백이 가용 스레드 풀(Thread Pool)의 물리 코어 전체를 장기 선점(Preemption)해 버리는 참사가 발생한다. 그 결과, 전체 시스템 안에서 타이밍 통제 데드라인이 가장 시급한 실시간 코어 제어 타이머 콜백이 정작 실행 슬롯 코어를 제때 할당받지 못하여 대기열에서 굶주리게 되는 치명적인 ’우선순위 역전(Priority Inversion)’과 ‘큐 블로킹(Queue Blocking)’ 현상이 복합적으로 발현된다.

위와 같은 파괴적인 스케줄링 런타임 변동성을 통제하고 제어 루프 지연(Latency)을 원천 봉쇄하기 위해서는, 핵심 제어 루프용 타이머 콜백 개체를 여타의 모든 I/O 및 통신 연산 콜백들로부터 메모리 락 단계에서부터 강제로 격리 차단하는 독립형 컨테이너 토폴로지를 선제 구축해야만 한다.
이 결함 내성 방어 설계 절차는 다음과 같이 전개된다. 첫째, 비행 제어 노드 객체가 초기화될 때 메인 제어 루프를 호출하는 타이머를 선언하는 과정에서, ROS 2 프레임워크가 제공하는 별도 전용의 상호 배타적 콜백 그룹(rclcpp::CallbackGroupType::MutuallyExclusive) 변수를 강제로 생성하여 단독 매핑(Mapping)한다.
둘째, 이 독립 생성된 논리적 컨테이너 객체 내부 공간에는 오직 단 하나의 핵심 하드-실시간 타이머 콜백 연산 함수 포인터만이 유일하게 편입 존재해야 하며, 어떠한 파생 구독 통신 콜백도 이 공간에 난입시켜서는 안 된다. 나아가 극단적인 실시간 마진 확보를 위해, 해당 타이머 전용 컨테이너를 범용 멀티스레드 이그제큐터의 관리망에서 완전히 도려내어 분리하고, 스레드 호출 우선순위(Thread Priority) 권한이 RTOS 커널(예: PREEMPT_RT, Xenomai) 정책 수준에서 최상단으로 강제 보장 세팅된 별도의 ‘단일 스레드 이그제큐터(Dedicated Single-Threaded Executor)’ 위로 격리 스핀 시킬 것이 학술적으로 권장된다.

이러한 철저히 고립된 독립형 상호 배타적 타이머 컨테이너 설계 구조는, 제어 루프 연산이 이그제큐터 내부의 라운드 로빈(Round-Robin) 대기열에서 통신 및 서비스 같은 가변적인 비동기 파편 이벤트들과 무의미하게 경합하는 딜레이 오차 요인을 하드웨어 런타임 단계에서부터 근본적으로 배제 차폐한다. 코어 제어 타이머는 오직 클럭(Clock) 하드웨어의 시계열 만료 인터럽트에 의해서만 단독 최우선적으로 락(Lock)을 획득하고 연산 페이즈를 전진 전개하므로, 타 스크립트 스레드의 예외적 폭주나 실행 지연에 절대 구속받지 않는 수학적으로 완벽하게 확정적인 타이밍 마진(Deterministic Timing Margin)을 확보하게 된다. 궁극적으로 이 방어 아키텍처는 외부 네트워크나 이기종 연산 요인으로 인한 제어 신호 지터(Jitter)의 유입 파동을 사실상 제로 극한치에 가깝게 수렴 방어해 내는 가장 결함 내성적이고 방어적인 실시간 제어 스케줄링 모델로서 드론 공학에서 평가받는다.
