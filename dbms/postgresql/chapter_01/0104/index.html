PostgreSQL는 1980년대 중반 UC Berkeley에서 시작된 POSTGRES 프로젝트를 기원으로 하여 오랜 시간에 걸쳐 발전해 왔다. 초기에는 Postgres95라는 이름으로 불렸으나 1996년 이후부터는 PostgreSQL로 재명명되었고, 데이터베이스 이론과 실무 양쪽을 모두 충실히 반영하며 꾸준히 성장하고 있다. 아래에 제시한 연혁은 PostgreSQL가 어떤 경로로 기능을 확장하고 안정성을 확보해 왔는지를 보여준다. 버전마다 대표적인 특징과 개선 사항이 존재하는데, 이를 통해 PostgreSQL가 오픈소스 DBMS로서 현재의 위상을 확보하게 된 과정을 살펴볼 수 있다.

Postgres95¶

  • 1994년경부터 'Postgres95'라는 명칭으로 불렸다.
  • Postquel이 아닌 SQL을 사용하는 최초의 실험적 버전이었다.
  • 기존 POSTGRES 코드 기반을 유지하면서 성능 개선과 SQL 인터페이스의 추가가 이루어졌다.
  • 이 시기를 거치며 PostgreSQL로 공식 명칭을 변경하기 위한 준비 과정을 진행하였다.

PostgreSQL 6.0 (1996년 ~ 1997년)¶

  • Postgres95에서 PostgreSQL로 명칭이 변경된 첫 정식 릴리스다.
  • 오픈소스 DBMS로서 공식적으로 인식받기 시작했고, 커뮤니티 주도의 발전 체계를 확립하였다.
  • MVCC(Multiversion Concurrency Control)의 안정화에 기여하였고, SQL 표준 호환성을 높이는 작업이 진행되었다.

PostgreSQL 7.x (2000년대 초반)¶

  • WAL(Write-Ahead Logging) 메커니즘이 도입되어 장애 복구 기능이 한층 강화되었다.
  • 대규모 테이블을 다루는 데 필요한 기능 및 쿼리 최적화가 개선되면서 산업 현장에 널리 채택되기 시작하였다.
  • 특정 OS에 한정되지 않고, 다양한 플랫폼을 정식 지원하기 위한 포팅 작업이 활발히 이루어졌다.

PostgreSQL 8.x (2005년 ~ 2010년대 초반)¶

  • Windows 플랫폼을 기본 지원하면서 사용자 저변이 크게 확대되었다.
  • 테이블스페이스, PITR(Point-In-Time Recovery) 등 기업 환경에서 요구되는 고급 기능들이 추가되었다.
  • 외부 데이터 소스와의 연동, 복제 솔루션 개선, 성능 튜닝 관련 옵션이 점진적으로 향상되었다.

PostgreSQL 9.x (2010년대 초중반)¶

  • 스트리밍 복제(Streaming Replication)와 핫 스탠바이(Hot Standby)가 도입되어 고가용성(High Availability) 환경 구성에 적합해졌다.
  • 9.1 버전에서 동기식 복제(Synchronous Replication)가 추가되어, 중요한 금융·거래 시스템에서도 본격적으로 활용할 수 있게 되었다.
  • 9.4 버전부터 JSONB 자료형이 도입되어 문서 지향 DBMS 기능을 어느 정도 흡수하였으며, 9.5에서는 UPSERT(ON CONFLICT 구문)가 추가되는 등 SQL 언어적 유연성이 크게 향상되었다.
  • 9.6 버전에서 병렬 쿼리(Parallel Query)가 도입되어 대규모 데이터 처리를 위한 성능이 강화되었다.

PostgreSQL 10 (2017년)¶

  • 버전 표기 체계를 단순화하여 메이저 버전을 두 자리에서 한 자리로 변경하였다(예: 10.0이 아닌 PostgreSQL 10).
  • 논리 복제(Logical Replication) 기능이 추가되어, 테이블 단위의 정교한 복제와 외부 시스템과의 실시간 데이터 교환이 가능해졌다.
  • 파티셔닝(Declarative Partitioning), 쿼리 병렬화 등 엔터프라이즈급 확장성에 핵심적인 기능들의 완성도가 높아졌다.

PostgreSQL 11 (2018년)¶

  • 성능 최적화를 위해 JIT(Just-In-Time) 컴파일을 지원하기 시작하여, 데이터 처리 속도 및 CPU 활용도가 크게 개선되었다.
  • 파티션 관리 기능이 더욱 편리해지고, SQL 표준 준수 범위가 확장되었다.
  • 인덱스 기법 개선과 저장 방식 최적화를 통해 대규모 OLTP, OLAP 환경 모두에 적합한 기반을 다졌다.

PostgreSQL 12 (2019년)¶

  • 파티션 테이블 관련 기능이 대폭 강화되었고, 여러 종류의 쿼리 최적화 개선 사항이 추가되었다.
  • 윈도 함수와 공통 테이블 표현(CTE) 처리 효율성도 향상되어 복잡한 분석 작업에 대한 지원이 더욱 강력해졌다.
  • REINDEX CONCURRENTLY 등 시스템 운영 중에도 인덱스 재작성이 가능해져 가용성이 높아졌다.

PostgreSQL 13 (2020년)¶

  • B-트리 인덱스 중복 키 처리 최적화(덮어쓰기 방식)가 이루어져 인덱스 효율이 향상되었다.
  • 파티션 유지보수가 용이해졌으며, 복제 슬롯 확장 기능을 통해 관리 편의성이 높아졌다.
  • 다양한 통계 및 모니터링 지표가 개선되어 운영자가 시스템 상태를 더 잘 파악할 수 있게 되었다.

PostgreSQL 14 (2021년)¶

  • JSON 기능이 한층 보강되어 문서 기반 작업 시 유연성과 성능이 향상되었다.
  • 지속적인 병렬 처리 엔진 최적화로 대규모 데이터 질의 처리 성능이 더욱 강화되었다.
  • 다중 접속 환경에서의 동시성 제어, 잠금 해제 시점 등이 개선되어 고성능 트랜잭션 처리에 유리해졌다.

PostgreSQL 15 (2022년)¶

  • MERGE 구문 지원으로 다양한 ETL 작업 및 데이터 동기화가 단순화되었다.
  • 논리 복제 구성 시 성능 및 관리 측면에서 편의 기능이 추가되었다.
  • 파티션 관리자, 확장성 라이브러리 등 대규모 분산 환경을 운영하는 데 필요한 구성 요소들이 강화되었다.

PostgreSQL 16 (2023년)¶

  • 고성능 워크로드 환경에서 효과를 발휘하는 개선들이 다수 포함되었다. SQL/JSON 관련 기능 및 쿼리 최적화가 추가되었다.
  • 확장된 통계 수집 기능을 바탕으로 모니터링 및 운영 효율을 높일 수 있게 되었으며, 부분적인 병렬 처리 알고리즘이 개선되었다.
  • 다양한 호환성과 보안 패치가 적용되어 엔터프라이즈급 데이터베이스로서의 안정성과 신뢰도가 더욱 높아졌다.

이처럼 PostgreSQL는 학계에서 출발한 연구용 DBMS를 산업계에서도 적극 활용 가능한 범용 DBMS로 발전시켜 왔다. 오픈소스 개발 모델 특성상 주요 기능은 실제로 대규모 서비스를 운영하는 사용자 및 기업들의 요구 사항을 신속히 반영해 왔고, 이를 통해 PostgreSQL는 기능·성능·안정성에서 지속적인 진화를 거듭하고 있다. 특히 9.x 버전대에서 복제 기술과 JSON을 비롯한 문서 지향 기능이 급격히 향상되었고, 10 버전부터는 메이저 버전 번호 전략을 바꾸어 완전히 새로운 기능들을 적극 도입하고 있다. 최근 15와 16 버전에서는 SQL 표준 기능을 넓게 포용하는 동시에, 실시간 분석·분산 처리·운영 편의성 등을 대폭 강화하여 현대적인 데이터베이스 환경을 만족시키고 있다.

PostgreSQL 16 시점에서 되돌아보면, PostgreSQL의 진화 과정은 단순한 기능 추가를 넘어 안정적인 트랜잭션 처리, 포괄적인 확장성, 오픈소스 커뮤니티의 폭넓은 기여를 통해 서로 다른 요구 사항을 아우르는 방향으로 진행되어 왔다. 이는 PostgreSQL가 오픈소스 영역에서의 대표적인 관계형 데이터베이스로 자리 잡는 데 결정적인 역할을 했다. 이런 역사를 이해하면 PostgreSQL 16이 왜 현대적인 요구 사항을 충실히 반영하고 있는지, 그리고 앞으로 어떤 방향으로 발전할지를 가늠하는 데도 큰 도움이 될 것이다.
