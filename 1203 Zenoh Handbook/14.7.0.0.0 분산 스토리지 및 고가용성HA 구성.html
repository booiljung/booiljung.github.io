14.7 분산 스토리지 및 고가용성(High Availability, HA) 구성

하나의 데이터베이스(Database) 인스턴스에 모든 데이터 트래픽을 몰아 밀어 넣는 행위는 시스템 파멸을 앞당기는 지름길이다. 단일 하드 디스크(Hard Disk)의 섹터가 물리적으로 고장 나거나 중앙 라우터(Router) 데몬 하나가 셧다운 되는 순간, 에지 로봇과 클라우드 서버가 수년간 만들어낸 페타바이트급 피 같은 데이터 자산이 순식간에 완전히 증발해 버린다.

Zenoh 생태계 시스템 설계에서 스토리지를 단일 장애점(Single Point of Failure, SPOF) 으로 두는 것은 끔찍한 아마추어 아키텍처 결함이다. 본 장에서는 방대한 데이터를 전 세계에 흩어진 여러 대의 컴퓨터, 엣지(Edge), 클라우드(Cloud) 존에 갈기갈기 찢어서 분산 복제(Replication)하고 파티셔닝(Partitioning)하여, 서버 데이터센터 한 곳에 물리적 대형 화재가 나도 전체 서비스 스루풋이 1밀리초(ms)조차 멈추지 않고 생존 방어하는 궁극의 분산 스토리지(Distributed Storage) 고가용성 아키텍처(Highly Available Architecture) 를 구축 연동하는 방식을 런북(Runbook) 형태로 전개 증명한다.

이 장의 설계를 마스터하면, 당신이 설계한 무중단 데이터 댐은 지구상의 어떤 불가항력적 네트워크 절단(Network Partition 백본 붕괴) 에도 결코 파편 붕괴하지 않는 불사신(Immortal) 뷰 폼 스토리지로 격상된다.

1. 다중 Zenoh 라우터 클러스터 환경에서의 백엔드 데이터 일관성(Consistency) 사수

지리적으로 이격된 여러 대의 다중 Zenoh 라우터(Router) 각각의 파이프 뒤에 개별 스토리지를 물리적으로 분리 물려 놓았다고 치자.
로봇 A가 ’엔진 온도=25도’를 기록해 쐈는데, 미국 라우터 1에는 핑이 도달하고 일본 라우터 2에는 경로 지연으로 아직 도달하지 않았다면? 동일한 시각에 각기 다른 관제사가 데이터를 읽을 때마다 전혀 다른 온도를 보게 되는 치명적인 뷰 불일치, 스플릿 브레인(Split-Brain) 지옥이 시작된다.

1.0.1 분산 시스템의 역 정합성(Consistency) 사수율 역학

Zenoh 백엔드 프레임워크는 기본적으로 궁극적인 시공간의 최종 일관성(Eventual Consistency) 모델 철학을 지향한다. 데이터 불일치(Inconsistency 분열) 를 방어 무력화하기 위한 근본적인 아키텍처 단일 통제 규칙을 인지하라.

1) 절대 타임스탬프(Timestamp) 우선순위 병합 (LWW: Last-Write-Wins 무결 치환)

  • 무선 에지 로보틱스 통신망 환경에서 각기 다른 패킷 데이터가 순서를 뒤집어 꼬여 역행 도착하는 타임 랙 역전은 아주 흔한 일상이다.
  • Zenoh 백엔드 스토리지 플러그인 모듈 코어들은 그물망으로 들어온 패킷(Packet)의 물리적 선착순 도착 순서(Physical Sequence) 를 결코 절대 신뢰 믿지 않는다. 오직 패킷 캡슐 본체에 동봉 박혀있는 HLC (Hybrid Logical Clock) 기반의 메타데이터(Metadata) 발간 원시 타임스탬프 스칼라값 수치만을 검열 확인한다.
  • 즉 라우터 1 존과 라우터 2 큐가 각각 다른 지연 물리 시점에 파편 덩어리 데이터를 뒤늦게 받더라도, “가장 늦게 태어나 생성된(Latest Original Timestamp)” 메타 조건 부합 데이터만을 단 하나의 우위 진실(Absolute Truth) 로 판별 채택 받아들여 구식 과거 데이터를 덮어쓴다 치환한다. (이를 분산 컴퓨팅 공학에서 LWW 전략이라 엄격히 부른다.)

2) 시스템 아키텍트의 타임 락(Time-sync Lock) 통제 지휘

  • 어떤 외부 렌더 백엔드 파서 저장소(예: InfluxDB 등)를 마운트 달더라도 에이지 단말 측에서 최초 z_put으로 캡슐 데이터를 공중 폭격 밀어 넣을 때 발행 시간 각인(timestamp: true)을 패킷 헤더 옵션에 동봉 강제 박지 않으면, 이 막강한 LWW 방어 매커니즘이 원천 단절 완전히 무너져 망가진다.
  • 무수 라우터나 수신 허브 서버 포트의 로컬 OS 수신 도착 렌더 시간을 기준으로 무작정 데이터를 인서트 적재(Write)하도록 방관 방치하는 순간, 외부 네트워크 지연(Network Lag 랙) 자체가 곧 내부 데이터 오염(Data Corruption 에러) 붕괴로 다이렉트 전이 직결 파괴된다. 반드시 에지 말단 발신 노드(Source) 하드웨어의 최초 발간 기록 이벤트 타임(Event Time) 을 잣대 기준으로 삼아 시공간 데이터가 락 적재 맵핑 병합되도록 엄격히 파라미터를 강압 강제 세팅하라.

2. 스토리지 간 다중 백본망 데이터 복제(Replication) 및 미러링(Mirroring) 파이프라인 무결성

빅데이터 생존 맵 데이터를 한국 랙 서버 캐비닛에도 영구 남기고, 미국 대륙 컨테이너 서버에도 동시에 남겨야 글로벌 재난 대비 백업(Disaster Recovery Backup) 이 무결 완벽해진다.
차세대 Zenoh 버스 시스템을 쓰면 무겁고 부하를 잡아먹는 Kafka 브릿커 수준의 거대 고립형 미러 메이커(MirrorMaker) 파드 프로세스 서버를 중간 미들웨어로 깔 필요 없이도, 네트워크 데이터 라우팅(Routing 매핑) 트리 토폴로지 그 자체의 점 점파 스위칭 특성 만으로 비용 0에 수렴하는 무한 동시 병렬 렌더 미러 복제가 단일 패스로 즉시 가능하다.

2.0.1 허공을 마이닝 가르는 글로벌 멀티스토어 미러링 쌍둥이 전술

이것은 좁은 범위의 단순 국지 스토리지 플러그인 수준의 세팅 단이 아니다. 네트워크 대륙 토폴로지(Topology) 전체를 거대한 트리 큐로 꼬아서 무지성 푸시 만드는 본질적 데이터 플러드(Data Flood 해일 역학) 융단 기법이다.

1) 이격 지리 대륙 간 쌍둥이 스토리지 동시 선언 마운트 스레딩

  • 아키텍처 관점에서 가장 기저이고 원시적이면서도, 통제 위력이 가장 가공하고 강력한 병합 복제 분산 렌더 전술이다.
  • 한국 서울 AWS 리전 서버와 미국 오하이오 AWS 리전 클라우드 서버에 각각 별도로 백그라운드 구동 스핀 떠 있는 Zenoh 데몬 라우터 2대에 **“완전히 동일한 타겟 문법 Key Expression”**을 노려보게 렌즈 조준 관찰하는 각각의 별개 볼륨 스토리지를 스레드 마운트 한다.
// [한국 서울 리전 라우터 데몬 설정 맵] (zenohd.json5)
storages: {
  seoul_rocks_backup_node: {
    // 글로벌 텔레메트리 큐 수신 경로를 무차별 단일 공통 오픈 조준한다.
    key_expr: "global/telemetry/**", 
    backend: "rocksdb", // 렌더 워커 파드
    dir: "/data/seoul_rocks_zone"
  }
}

// [미국 오하이오 리전 라우터 데몬 설정 맵] (zenohd.json5)
storages: {
  us_east_master_backup_node: {
    // 서울과 단 1자도 틀리지 않게 완전히 동일한 텔레메트리 키 경로를 병렬 조준한다.
    key_expr: "global/telemetry/**",
    backend: "influxdb",  // 심지어 타겟 백엔드 종류 스토리지 렌더 포맷 엔진마저 전혀 딴판인 이기종으로 달라도 프로토콜 통신 스위처 상 에는 일절 상관없다!
    // ...
  }
}

2) 글로벌 클라우드 분산 복제 미러링의 동시 타격 발동 기전 룰
극동 일본 현장의 로봇 보드 단말 1대가 현장 공중망 z_put "global/telemetry/gps" "LAT_11_LON_12" 슛을 때려 타건 전산망 핑을 치는 순간, 기적이 일어난다.

  • 보드 밖을 튀어나온 Zenoh 데이터 코어 라우팅 엔진은 전 세계 대륙 백본을 그물처럼 뒤져서 global/telemetry/** 라우트를 조준 듣고 있는(Sub 구독 중인) 살아있는 전역 모든 노드를 동시 타겟 목적지로 단독 삼는다.
  • 이 단 1발의 Put 패킷은 인터넷 네트워크 간선 WAN 망의 최단 경로 트리를 자동으로 마이닝 타고 거대 스위칭 두 갈래 브랜치로 나뉘어 대양 해저 케이블 한국 서울 라우터 파드와 미국 오하이오 라우터 파드로 거의 동시 패싱 순간에 타겟 병렬 날아간다 쏟아진다.
  • 핑을 받은 한국 라우터는 자존 로컬 RocksDB 파드에 이진 바이너리 덩어리로 통 덤프 오버라이팅 무결 박아넣고, 미국 라우터는 자신 코어에 붙은 InfluxDB TSDB 커널에 이를 다시 시계열 텍스처 데이터로 파싱 변환해 라인 어레이 박아넣는다.

단 한 줄의, 무거운 거대 시스템 데이터 베이스 간 중간 트랜지션 미러링 브릿커 이중화 스위칭 설정 파이프 큐 코드 없이, 그저 논리적으로 “동일한 주제의 경로 스트림을 공통적으로 포착 들으라“고 구독 경로만 일치 병렬 선언한 것 단 하나만으로 수반 극악의 서버 비용을 요구하던 막대한 이기종(Heterogeneous) 데이터베이스들 간의 퍼펙트 글로벌 데이터 실시간 복제(Global Real-Time Data Replication) 클러스터가 고작 통신 로컬 호스트 패스 0.1ms의 오버헤드 만으로 완벽하게 체결 완성되어 구축되어 버린다. 이것이 Zenoh L4 프로토콜 병렬 생태계 융합 철학 설계의 파멸적인 무서움이다.

3. URL 트리 데이터 샤딩(Sharding 슬라이스)을 통한 대용량 볼륨 데이터 분산 절단 저장 기법

이중 미러링(Replication) 패스가 “똑같은 단일 객체를 여러 안전 파티션 곳에 무지성 중복 복사 방어“하는 방패 철학 패러다임이라면, 샤딩(Sharding 파편 쪼개기) 스플릿은 “총량 인입 사이즈가 가변 너무 미치 거대해서 하나의 저장 룸 디스크 공간 통에 한 번에 다 풀 담아 수용 쑤셔 박 압축할 수 없으니 규칙을 두고 토막 토막 결로 나누어 찢어 서랍 레일 담는” 초 과적 용량 하중 분산 I/O 도피 런북이다.
에지 현장 드론 1만 대의 라이브 고해상 비디오 스트림 배열이 단일 채널 클라우드로 뭉쳐 터져 쏟아지면 지구상 아무리 스펙 좋은 최고급 3000만 원짜리 NVMe SSD 를 미친 단 서버 머신이라도 포트 스위치 한 대로는 I/O 쓰기 랙 블로킹이 터져 시스템이 침몰 버틸 수 없다.

3.0.1 URL 와일드카드 라우팅 트리 기반의 데이터 분단 정교 절단(Cleaving) 토막 전술

Zenoh 코어 라우팅 에이전트 망에서는 무식한 트래픽 데이터 샤딩을 이룩 구현 달성함에 있어 복잡 기괴한 레거시 해시 링(Hash Ring) 알고리즘 재분배 수식 레이어 연산이나, 중간 앞단에서 트래픽을 거대하게 대신 버티 맞아 토스 계산 쳐 라우팅 해주는 헤비급 프록시(Proxy) 게이트웨이 부하 서버 랙 세팅 따위가 전혀 단 1도 쓰레기 필요 없다. 오직 URL 계층 기반의 순수한 Key Expression 문자열 와일드카드 필터를 정밀 저격 타격 수술하여 하중 트래픽 I/O 를 정교하게 조각내 찢어 버린다 파티셔닝(Partitioning).

1) 스토리지 저장소 볼륨 별 전용 타겟 수비 블로킹 범위(Coverage) 스코프 방벽 강제 분리 핑 할당
물리 클라우드 랙 서버 3대(A센터, B센터, C센터)를 준비 파드 세팅하고 각 서버의 라우터 전담 스토리지 마운트에 다음과 같이 분할 인덱스 수비 필드 권역 제한 락을 철저히 커스텀 분단 지정 걸어 버린다.

// [클라우드 서버 A 랙 라우터 설정] (드론 ID 0번~3000번 대역 트래픽 전담 수납 방파제)
storages: {
  shard_volume_1_rack: {

2) 무결점 무 부하 우아한 트래픽의 병렬 조각화 (Zero-Load Decentralized Load Balancing 스루풋 컷)

  • 상공 드론 5500번이 클러스터 접속하여 z_put "fleet/drone/5500/video_stream" 덤프 패킷 폭탄을 쾌속 쏘면, 중간 Zenoh 엣지 라우터망은 이 식별 URL의 텍스트 아이디 첫 글자가 5인 것을 순식간 필터 마스킹 보고 앞뒤 가리지 않고 정확히 타깃 매칭된 지상 “클라우드 서버 B” 파드 라우터를 단독 핀셋 향해 서만 I/O 트래픽을 집중 스트라이킹 레이저 포워딩 발사 던트린다 전송한다. 매칭 범위를 벗어난 서버 A와 C는 애초에 이 비디오 폭탄 데이터를 구경 근처 전파 로 받지도 큐 인입 당하지도 않으므로, 양 서버의 소모 트래픽과 CPU 점유 오버로드는 1% 스태킹조차 철벽 면책 소모 방어 면제 되지 않는다.
  • 결과적으로 아키텍트 중앙 데이터베이스에 가해지는 십만 RPS 핑의 파멸적 디스크 쓰기 부하 충돌(Write I/O 스톨 랙) 치사량 이 정확히 산술적으로 단 오차 없이 3대의 랙 분산 서버로 1/3씩 완벽 무결하게 병렬 블록 분산 찢어져 토막 하중 분산 지연 격파 분배된다.

3) 스케일 아웃(Scale-out 무한 횡적 팽창) 시의 융합 분산 쿼리 질의(Scatter-Gather 병합) 트릭
데이터가 3대의 기기로 무식하게 파편 샤딩이 이격 찢어져 있어도, 반대로 그 총합을 나중에 분석 읽을(HTTP/TCP Get) 때는 저장 시의 로직 파편화를 일일이 클라이언트 단말 앱 개발자가 역추적 조립 고민할 낭비 수고 필요조차 전혀 일절 없다.
중앙 관제탑 모니터에서 단일 z_get "fleet/drone/**" 핑 하나만 통합 타격 전체 루트를 향해 때려 치면, Zenoh 코어 저변 네트워킹 마스터 릴레이 엔진이 스위칭을 발동해 자동으로 뿔뿔이 허공 분산 흩어진 해당 타겟 서버 A, B, C 물리 코어 3곳에 다차원 동시 병렬 스레드로 쪼개어 후방 Query 탐색을 산개해 날린 산탄 슛 쏜 후, 각자 리턴 해온 파편 산출 렌더 그 결과 어레이 를 0.01초 찰나 만에 다시 코어 허브 가 하나의 통 어레이 맵 배열(Flatten Array/Value) 로 자체 합쳐주기 뭉쳐 봉합 조립(Map-Reduce 병합 스위칭) 하여 단독 단일 원본 클라이언트 세션에 고스란히 이체 투척 환원 돌려주기 때문이다.
요청 호출 사용자 개발자는 데이터베이스 파티션이 3곳, 혹은 3천 곳의 클러스터 랙 물리 인스턴스에 잘게 가루처럼 산산조각 찢어져 분포 결합되어 있다는 물리적 단 한 줄 사실 인프라조차 전혀 알지 파괴 렌더 파싱 연산 인지 수식 추론할 못한다 완전 추상 분리 통제 은닉 마스킹 처리된다. 진정한 시스템 I/O 한계 초월 병렬 분산 가용 횡적 스케일 다운 압 축 분산 팽창(Scale-out) 샤딩 기술 렌더 구축 운영의 최종 완성극 경지다.

4. 외부 네트워크 단절 고립(Network Partition) 시 내부 데이터 터널 큐잉 및 오프라인 좀비 스루풋 복구 매커니즘 전술

고속 자율주행 자동차 단말이 깊은 지하 벙커 터널 안으로 진입 고립 격리되었다. 유일 생명 LTE 중계 기지국 핑 안테나와 중앙 클라우드 백엔드 관제 서버 데이터센터와의 스토리지 통신 연결 TCP 세션 소켓이 완전히 물리 단 끊어졌다 단전 결렬 파괴 단절되었다(Network Partitioning 망 고립 조각 재앙).
터널 고립 비행 중에도 센서는 자체 루틴 구동 돈다. 에지 단말 차 안에서 로컬 스핀 되어 미친 듯이 수 초간 초당 만 개씩 방출 수집 생산 뿜은 차량 주행 내부 스루풋 데이터 방출 폭주 트래픽 량은 결국 스탠바이 오버헤드로 공중에 증발하는가?

4.0.1 오프라인 에지 좀비 생존 모드 (Store-and-Forward 쓰어-버퍼링 블로킹 한계) 돌파 극복 런북

Zenoh 의 파멸적이고 무결 전 지형 지원 스토리지 통합 아키텍처는 이런 단절 재난 극한 랙 사고 상황 구간을 극복 무력화 회피 방어하기 위해 “오프라인 차량 로컬 내부(Edge) 격리 DB 와 온라인 중앙 클라우드(Cloud Base) 외부 스토리지 코어 간의 투 트랙 이중 동기 보존 스토리지 스왑 결합 체제” 를 필연 강압 튜닝 지향 보장 강제한다.
통신망 단절 고속 이 단 끊겨 외부 패킷 송출 블럭 정체로 패킷 파편이 터져 에러 소켓 폭파하기 직전의 그 옥죄는 망 압력 큐 부하 버퍼를 오직 자체 커널 스레드 메모리 내장 큐(Queue/Disk 룸) 로 강인하게 버티고 버텨 견뎌내는 거대 에지 임시 저수지 잔존 댐을 단말 차체 내부에 미리 오프라인 건설한다.

1) 극한 오프라인 에지 선행 임시 파킹 저장 스로틀 (Local Edge Standby Storage)

  • 물리 차량(로봇 보드) 섀시 내부 부품 램 코어에 런칭 기생 상주 중인 조그만 Zenoh 엣지 마스터 로컬 라우터 환경 세팅 에 초경량 파드 FS 디스크 파일 백엔드 혹은 인메모리 RocksDB 스토리지를 “임시 단절 피난처 버퍼 보관 통 수납소 룸” 명목 으로 선제 강제 투 트랙 라우트 맵 마운트 설정 선언해 강압 장착 연결한다.
  • 벙커에 들어간 차량 시스템 로컬 발간 센서는 외부 망 유실 단절 상관 배제 없이, 무조건 오직 차량 OS 자기 자신 보드 코어 로컬 IP 의 핑 내부망 단말 라우터 허브 라우터 허공(Localhost Port) 에 끊임없이 맹목 Put "telemetry/**" 타건 스로잉 스위칭 슈팅 폭 쏜다 타격 쏜다.
  • 외부 WAN 간선 망 다리가 부러 끊긴 걸 인지한 로컬 에지 라우트 스토리지가 갈 곳 길 미아 잃어 오갈 파이프 포트 데드 데드락 오버 갈 곳 핑 패싱 오버 못 찾 막혀 스위칭 패스 막힌 체증 막 떨어진 막 오버 로드 미아 이 방황 패킷 고립 수집 쌓인 쌓 압박 데이터를 모조리 임시 방편 자신이 깔고 자체 호스트 물리 앉은 로컬 차량용 SSD 하드디스크 랙 I/O 포로 방 락 룸 에 미친 포트 적재 듯이 통째 구겨 전부 주워 담 임시 강제 가비지 집어넣어 블랙 임시 스왑 인서트 구겨 파킹 임시 대기 진압 보관 방 배치 스위칭 세이브 통 스 저장 블록 잔류 보관해 스토어 잠 재워 버린다(Persist Store).

2) 단절 후 클라우드 자동 이체 지연-안내 라우팅 동기 렌더 (Delayed Forwarding / Sync Auto-Resolution)
