51.4.3.2 이탈량의 이동 평균(Moving Average)이 공간적 임계치(Spatial Threshold)를 초과할 경우 GCS 측에 발행되는 비동기 이벤트 알람(Asynchronous Event Alarm) 인터럽트 발생 체계

1. 개요

무인항공기(UAV)가 사전에 프로그래밍된 궤적을 추종하거나 복귀(Return) 모드 등으로 비행할 때, 돌풍(Wind Gust)이나 일시적인 구동기 결함으로 인해 기하학적 궤적에서 벗어나는 궤적 이탈(Trajectory Deviation)이 발생할 수 있다. 이 과정에서 단순히 순간적인 위치 오차를 기준으로 알람을 발생시키면 센서 노이즈 등에 의한 오발람(False Alarm)이 빈번하게 발생한다.
PX4-Autopilot은 이러한 현상을 방지하기 위해 이탈량에 대한 이동 평균(Moving Average) 필터를 거치며, 그 결과값이 사전에 정의된 공간적 임계치(Spatial Threshold) 초과 유무를 판단하여, 초과 시 기체에서 지상 관제 시스템(GCS, QGroundControl) 측으로 비동기 이벤트 알람(Asynchronous Event Alarm) 인터럽트(Interrupt)를 발생시키는 강건한 모니터링 시스템을 갖추고 있다.

2. 궤적 이탈량 계산 및 이동 평균(Moving Average) 필터 적용

위치 제어기(Position Controller)는 현재 기체의 위치 상태 좌표(Current Position) \mathbf{p}_{curr}와 제어 목표 지점(Setpoint) 좌표 \mathbf{p}_{sp} 사이의 3차원 유클리드 거리 오차인 이탈 거리(Error Distance) 집합을 산출한다.

순간적인 이탈량 e_{dev}[k] 데이터는 고주파 노이즈 성분이 존재하므로, N 샘플 윈도우(Window) 크기를 갖는 이동 평균 필터(Moving Average Filter) 또는 1차 저역통과필터(Low Pass Filter)를 거치게 된다. 이동 평균 처리된 이탈량 \bar{e}_{dev}[k]는 다음과 같이 표현된다.

\bar{e}_{dev}[k] = \frac{1}{N} \sum_{i=0}^{N-1} e_{dev}[k-i]

또는 지수 이동 평균(Exponential Moving Average) 방식을 적용할 경우 다음과 같은 차분 방정식으로 계산된다.

\bar{e}_{dev}[k] = \alpha \cdot e_{dev}[k] + (1 - \alpha) \cdot \bar{e}_{dev}[k-1]

여기서 \alpha는 필터의 차단 주파수(Cutoff Frequency)에 의해 결정되는 평활화 계수(Smoothing Factor)이다. 필터링 과정을 거치면서 우발적인 센서 튀김 현상은 배제되고, 기체가 지속적으로 궤적을 벗어나고 있는 물리적 경향성만을 분리할 수 있다.

3. 공간적 임계치(Spatial Threshold) 초과 검증

필터링된 이탈량 \bar{e}_{dev}[k]는 매 순환 주기(Control Loop)마다 공간적 임계치 D_{thresh}와 비교된다. 이 임계값은 비행 모드의 허용 위치 오차 파라미터(예: NAV_ACC_RAD 혹은 특정 모드 전용의 이탈 허용 반경)를 기반으로 설정된다.

  • 안전 조건(Safe Condition): \bar{e}_{dev}[k] \leq D_{thresh}
  • 이탈 조건(Deviation Condition): \bar{e}_{dev}[k] > D_{thresh}

이탈 조건이 성립(True)되는 즉시, 시스템은 폴링(Polling) 대기 방식이 아닌 인터럽트 트리거(Interrupt Trigger) 방식으로 알람 프로세스를 호출한다.

4. 비동기 이벤트 알람(Asynchronous Event Alarm) 인터럽트 발생 체계

이탈 조건이 충족되면 PX4 시스템 내에서는 단순히 상태 변수를 업데이트하고 넘어가는 것이 아니라, uORB 토픽(Topic) 연동 기반의 이벤트(Event)를 즉각 발행(Publish)한다.

최신 PX4 버전(v1.14 이상)에서는 MAVLink의 보다 정형화된 EVENT 프로토콜을 적용하여 인터럽트를 GCS에 전달하거나, 혹은 전통적 메커니즘의 STATUSTEXT (메시지 ID #253) 프로토콜을 이용한다.

이러한 비동기 메커니즘의 장점은 다음과 같다:

  1. 관제 측의 부하 감소: GCS (QGroundControl)가 주기적으로 텔레메트리(Telemetry) 트래픽을 스스로 분석하여 알람 여부를 판단하는 책임을 지지 않아도 된다.
  2. 즉각적 반응성: 궤적 이탈 인터럽트가 발생하자마자 GCS UI에 크리티컬 경고 팝업이 전시되거나 오디오 알림(Audio Alert)이 즉시 구동된다.
  3. 대역폭 효율성: 데이터 링크 대역폭(Bandwidth) 낭비를 최소화하면서도, 위급 상황 정보만을 선별 전송한다. Ardupilot 시스템이 일부 알람을 GCS 측의 상태 폴링 루프(Polling Loop)에 의존하는 경향이 있는 반면, PX4 시스템의 인터럽트 아키텍처는 에지단(Edge) 드론 내부의 분산 처리 효율성을 증진시킨다.

6. 결론

순간 궤적 오차에 대한 이동 평균 필터링 수학적 모델과 인터럽트 기반의 알람 발생 아키텍처는, PX4 기반 무인항공기가 외부 환경(돌풍, 강풍유도 저항, 드라이브-트레인 결함 등)에 의해 점진적으로 표류(Drifting)하거나 경로 추종 능력을 상실하는 치명적인 상황을 오작동 없이 정확히 포착한다. 이는 GCS 운용 관제사가 기체의 이상 유무를 신속하게 인지하고 수동개입(Manual Override) 혹은 비행 종료(Flight Termination)를 지시하도록 구성된 핵심 안전망(Safety Net) 기능을 수행한다.
