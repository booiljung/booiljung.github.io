51.1.2 위상 공간(Topological Space) 기반의 목적지 탐색 알고리즘과 그래프 이론(Graph Theory) 적용

본 절에서는 단순한 유클리드 기하학(Euclidean Geometry)적 공간 개념을 넘어, 비행 가능 영역을 위상 공간(Topological Space)으로 추상화하고 그래프 이론(Graph Theory)을 적용하여 안전한 목적지를 탐색하는 무인항공기의 경로 계획(Path Planning) 알고리즘을 분석한다. (기준 펌웨어: PX4-Autopilot v1.14 이상, 관제 소프트웨어: QGroundControl v4.3 이상)

1. 위상 공간(Topological Space)과 미터릭 맵(Metric Map)의 차이

대부분의 소형 무인기(sUAV)는 3차원 미터릭 맵(지리 좌표계 상의 위도, 경도, 고도로 구성된 연속 공간) 내에서 최단 거리를 비행하도록 설계된다. 그러나 도심지나 산악 지형, 또는 복잡한 장애물이 존재하는 실내외 교차 환경에서는 목적지까지 직접 날아가는 선형 경로(Linear Path)가 지형지물에 의해 차단될 확률이 높다.

위상 공간(Topological Space) 기반 맵핑은 비행 영역을 노드(Node, 의미 있는 위치: 이착륙장, 웨이포인트, 교차로 등)와 에지(Edge, 노드 간의 안전한 비행 가능 경로)로 구성된 이산적인 네트워크(Discrete Network)로 추상화한다.

  • 장점: 3D 공간을 셀룰러 격자(Grid) 단위로 나누는 수백만 번의 연산량이 기하급수적으로 감소하며, 장애물 회피를 위한 복잡한 벡터 연산 대신 그래프 탐색 최적화에만 컴퓨팅 리소스를 집중할 수 있다.
  • 적용 영역: PX4에서는 자율 복귀(RTL) 시 다중 랠리 포인트(Multi-Rally Point) 간의 귀환 경로 재구성이나, ROS2 에이전트를 통한 고차원 오프보드(Offboard) 전역 경로 계획(Global Path Planning)에서 주로 이 개념이 채택된다.

2. 그래프 이론(Graph Theory)의 경로 계획 적용

무인기의 위상 공간 기반 경로 탐색은 유향 그래프 G = (V, E) 구조로 수학적 모델링이 이루어진다.

  • V (Vertices): 웨이포인트(Mission Items), 랠리 포인트(Rally Points), 시동 좌표(Home Position) 등의 이산 좌표점 집합
  • E (Edges): 두 좌표점 간 장애물이 없는 가시선(Line of Sight, LoS)이 확보되어 물리적 비행이 허용되는 유효 경로망 집합. 에지의 가중치(Weight)는 이동 거리, 배터리 소모량 역학 방정식, 비행 제한 구역(Geofence) 근접도에 따른 패널티를 결합한 비용 함수(Cost Function)로 산정된다.

PX4의 내비게이터(Navigator) 인스턴스는 복잡한 임무 수행 전이나 긴급 복귀(Return) 모드 트리거 시, 이러한 노드와 에지들의 가중치를 바탕으로 안전한 경로가 연속적으로 연결되어 있는지를 내부 상태 머신 내에서 검증한다.

3. 목적지 탐색 알고리즘 (Dijkstra vs A* 휴리스틱)

위상 공간에서 최종 목적지를 도달하기 위해 비용 함수를 최소화하는 Dijkstra 및 A*(A-Star) 알고리즘이 널리 구현된다.

  • Dijkstra 다익스트라 탐색: 시작점에서 모든 도달 가능한 노드에 이르는 최소 누적 비용 경로를 찾는다. 장애물의 위치나 랠리 포인트의 위치가 동적으로 변하지 않는 정적 위상 공간(Static Topology)에서 탐색 노드 정점의 확실한 해를 보장한다.
  • A 휴리스틱(Heuristic) 탐색*: 목적지 방향으로의 잔여 거리 추정치(Euclidean Distance 또는 Manhattan Distance)를 휴리스틱 가중치로 활용하여, 불필요한 이면 방향 탐색 반경을 절단(Pruning)하고 탐색 속도를 기하급수적으로 향상시킨다. 자원 대비 연산 효율이 높아 배터리 제약이 심한 현장 투입용 멀티로터의 실시간 경로 재설정에 최적화되어 있다.

4. ArduPilot 대비 관제 및 연산 아키텍처 차이

  • PX4-Autopilot: 비행 제어기(FCU) 내장 메모리 제약으로 인해, 수백 개의 노드를 수반하는 복잡한 3D 그래프 탐색 및 위상 공간 연산은 자체 펌웨어가 아닌 동반 컴퓨터(Companion Computer, 예: Raspberry Pi, Jetson Nano 등)의 ROS2/Micro XRCE-DDS 노드에 이관(Offloading)하는 오프보드(Offboard) 지향적 아키텍처 철학을 가진다.
  • ArduPilot: ArduPilot 생태계는 BendyRuler 알고리즘이나 OA(Obstacle Avoidance) Dijkstra 탐색 로직 등 상당한 수준의 공간 회피 연산을 비행 제어기 공간 내부에 모놀리식(Monolithic)으로 탑재하고 있다. 외부 컴퓨터 의존도가 상대적으로 낮아 하드웨어 단일 장애점(SPOF)에 강하지만, 복잡한 위상 그래프 확장 시 제어기 내부 MCU의 연산 부하 마진을 유의해야 한다.

5. QGroundControl(QGC)에서의 관제 연산 적용

QGroundControl에서 다각형 펜스(Polygon Fences)나 랠리 포인트 리스트(Rally Points)를 등록하면, GCS 관제 엔진 내부에서 일종의 암묵적인 위상 네트워크를 형성한다.

  1. 조종자가 새로운 웨이포인트를 추가할 때마다 QGC의 C++ 백엔드(Mission Controller)는 해당 웨이포인트를 향하는 선형 에지(Edge)가 펜스 경계면(Boundary)과 교차하는지 선분 교차 검증(Line Intersection Check) 알고리즘을 수행한다.
  2. 교차(Intersection)가 감지되는 즉시, QML 프론트엔드 UI를 통해 ‘Flight Plan Constraint Warning’ 시각적 경고를 발송하여 경로 이탈의 잠재 요소를 차단한다.

추가적인 맞춤형 GCS(Custom GCS Application)를 설계할 경우, 개발자는 위상 그래프 정보를 JSON 포맷이나 XML 트리로 추상화하여 관리해야 한다. 비행 중 MAVLink 통신망에서 음영이 생길 경우, GCS 내부 백그라운드 스레드의 A* 그래프 알고리즘이 오프라인 백업 경로(Fallback Return Route)를 생성하여 선제적으로 MAVLink MISSION_ITEM_INT 배치 리스트를 기체에 갱신/주입하도록 설계하는 것이 선진 드론 관제 기술의 모범 사례이다.
