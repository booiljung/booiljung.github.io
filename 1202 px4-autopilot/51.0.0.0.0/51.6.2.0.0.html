51.6.2 배터리 등가 회로 모델링(Equivalent Circuit Modeling) 기반의 극한 잔여 체공 시간 예측

드론 운용 시 배터리 고갈로 인한 치명적인 추락을 방지하기 위해서는 정확한 잔여 체공 시간(Remaining Flight Time) 예측이 필수적이다. 멀티로터나 고정익 드론은 돌풍(Gust)에 의한 자세 제어나 고속 비행 시 모터의 급격한 전류 소모를 동반하며, 이때 배터리의 순간 전압 강하(Voltage Drop) 현상이 발생한다. 단순한 개방 회로 전압(Open Circuit Voltage, OCV) 측정 방식으로는 이러한 동적 전압 변동을 정확히 보상할 수 없으므로, 배터리 등가 회로 모델링(Equivalent Circuit Modeling, ECM) 기반의 상태 추정 기법이 요구된다.

1. 아키텍처 및 등가 회로 모델의 이해

배터리의 전기화학적 특성을 수학적으로 모사하기 위해 PX4 시스템은 저항(Resistor)과 커패시터(Capacitor)를 조합한 등가 회로 모델을 참조한다. 가장 대중적으로 사용되는 1차 또는 2차 테브닌 모델(Thevenin Model)은 다음과 같은 구성 요소를 가진다.

  • V_{ocv} : 배터리의 개방 회로 전압 (무부하 상태의 전압)
  • R_0 : 배터리의 직렬 내부 저항(Internal Resistance)으로, 순간적인 전류 변화에 따른 즉각적인 전압 강하(Ohmic Drop) 요소
  • R_1, C_1 : 분극 저항 및 커패시터로, 배터리의 전기화학적 반응에 의한 지연된 전압 변화(Transient Response)를 모사하는 RC 네트워크
  • I_L : 부하 전류(Load Current)

실제 비행 중 드론의 센서에서 측정된 단자 전압(Terminal Voltage, V_t)은 다음과 같이 표현할 수 있다.
V_t = V_{ocv} - I_L \cdot R_0 - V_p
여기서 V_p는 분극 RC 네트워크에 걸리는 전압이다. PX4는 연산 부하를 최소화하기 위해 기본적으로 내부 저항(R_0, 파라미터 BAT1_R_INTERNAL)만을 고려하는 형태의 간소화된 0차 ECM 모델을 적용하여 OCV를 역산한다. 역산된 OCV를 방전 곡선(Discharge Curve) 테이블 룩업(Table Lookup)에 사용하여 왜곡 없는 실제 충전 상태(State of Charge, SoC)를 추정한다.

2. 잔여 체공 시간(Remaining Flight Time) 예측 메커니즘

잔여 체공 시간은 역산된 SoC와 현재 시점의 전력 소모량을 기반으로 산출된다.

  1. 전력 필터링: 모터의 순간 구동으로 인한 전류의 노이즈를 제거하기 위해 저역통과필터(Low Pass Filter)를 사용해 평균 소모 전류량(I_{avg})을 계산한다.
  2. 잔여 용량 연산: 초기 설정된 배터리 총 용량(BAT1_CAPACITY)에 현재 SoC 비율을 곱하여 잔여 시간 동안 사용할 수 있는 전류량(mAh)을 구한다.
  3. 시간 예측: 잔여 용량을 평균 소모 전류량으로 나누어 잔여 체공 시간을 예측한다. 예측된 시간은 순간 전압 강하를 내부 저항 기반으로 보상했기 때문에, 강한 기동 시에도 배터리 잔량 표기가 오르락내리락하는 현상(Spring Effect)을 크게 줄일 수 있다.

3. 센서 이해 및 통신

PX4는 두 가지 방식으로 배터리 데이터를 수집한다.

  1. 아날로그 전력 모듈(Analog Power Module): 비행 제어기(FC)의 ADC(Analog-to-Digital Converter) 포트로 들어오는 전압 배열을 읽어들인다. 이 경우 소프트웨어(PX4 battery_status 모듈) 내부에서 수식 기반의 모델링 예측이 수행된다.
  2. 스마트 배터리(Smart Battery): SMBus(I2C) 시스템 또는 UAVCAN 프로토콜이 탑재된 배터리 관리 시스템(BMS) 자체 패키징 모듈이다. 이 경우 배터리 제조사가 직접 튜닝한 고차원의 ECM과 쿨롱 카운팅(Coulomb Counting) 융합 알고리즘이 적용된 계산값을 FC가 그대로 수신하므로 예측의 정밀도가 극한으로 향상된다.

4. 소스 코드 분해 (Source Code Analysis)

주요 로직은 src/modules/battery_status/의 Battery 계층 클래스들에 구현되어 있다.

// src/modules/battery_status/Battery.cpp 핵심 요약 분석
void Battery::updateBatteryStatus()
{
    // 필터링된 배터리 전압 및 전류 계산
    float filtered_voltage = _voltage_filter.update(measured_voltage);
    float filtered_current = _current_filter.update(measured_current);

    // 1. 내부 저항(Internal Resistance)에 의한 전압 보상 적용 (ECM 기반 OCV 추정)
    // 보상된 전압 = 측정 전압 + (측정 전류 * 내부 저항)
    float voltage_resting = filtered_voltage + (filtered_current * _params.r_internal);

    // 2. Load Drop 보상 (배터리 방전 곡선의 셀프 튜닝 기반)
    // 방전 곡선 룩업 테이블을 사용하여 Resting Voltage에 해당하는 SoC(액체 잔량 비율) 확인
    float remaining_capacity_pct = estimateStateOfCharge(voltage_resting);

    // 잔존 상태(SoC)에 따른 Scale 변환 및 uORB 퍼블리싱 (battery_status)
    _battery_status.remaining = remaining_capacity_pct;
    _battery_status.voltage_filtered_v = filtered_voltage;
    // ...
}

위와 같이 단순 측정 전압(filtered_voltage)으로 SoC를 판단하지 않고, 전류량과 r_internal 파라미터를 곱해 내부 스텝 전압 강하분을 보상한 voltage_resting을 기준으로 SoC를 판단하여 복귀 판별 로직(Return To Launch RTL)의 신뢰성을 극대화한다.

5. 파라미터(Parameter) 설정 및 구성

안전한 극한 체공 시간 예측을 위해서는 배터리의 실측치에 기반한 QGroundControl 파라미터 튜닝이 필수적이다.

  • BAT1_V_EMPTY: 배터리 셀당 소진 상태로 간주할 최저 전압 (일반적인 LiPo는 3.5V ~ 3.3V)
  • BAT1_V_CHARGED: 배터리 셀당 완충 상태 전압 (LiPo 기준 4.2V)
  • BAT1_CAPACITY: 배터리의 명목 용량 (예: 5000 mAh)
  • BAT1_R_INTERNAL: (핵심) 배터리의 내부 저항. 사용에 따라 배터리 수명(State of Health, SoH)이 저하될수록 이 수치가 증가한다. 정확한 값을 입력해야 부하 순간 전압 강하가 SoC 예측에 미치는 악영향을 상쇄할 수 있다. 기본값은 -1(자동 판단 혹은 미사용)이나 0.005 Ohm 등으로 실측 후 입력해야 한다.

6. MAVLink 및 GCS와의 통신 (QGroundControl)

PX4 내부에서 산출된 예측값은 BATTERY_STATUS MAVLink 메시지로 GCS에 초당 수회(Telemetry Rate) 전송된다.
메시지의 주요 필드:

  • voltages[10] : 각 셀의 전압(mV)
  • current_battery : 배터리 총 전류 소모(10*mA)
  • battery_remaining : 보상된 잔류 용량 비율(1~100%)
  • time_remaining : 현재 전력 소모 기반의 (초 단위) 잔여 비행 시간. (PX4는 이 필드에 필터링 계산된 값을 채워 전송한다.)

QGroundControl(GCS)은 이 MAVLink 메시지를 수신하여 상단 툴바에 직관적인 배터리 잔량 아이콘과 시간으로 표시한다. 만약 잔여 시간이 RTL 수행에 필요한 예상 소요 시간보다 적어지면 시각적/음성 경고가 GCS에서 1차 발동된다.

7. Ardupilot 대비 차이 (Differences with Ardupilot)

  • 배터리 필터링 구조: PX4와 Ardupilot 모두 전압 강하를 추정하기 위한 내부 저항 모델을 사용하지만(AP_BattMonitor 라이브러리의 BATT_CRT_VOLT 등의 저항 옵션), PX4는 스마트 배터리 프로토콜 호환성을 uORB 계층 내 드라이버 단에서 추상화하여, 아날로그 센서인지 하드웨어 BMS인지 상관없이 동일한 예측 파이프라인(EKF 및 Commander Failsafe 트리거)으로 일관되게 연결하는 구조가 매우 강조되어 있다.
  • 자동 내부 저항 측정: Ardupilot의 일부 버전에서는 비행 데이터 및 로그를 기준으로 배터리의 내부 전압 저항을 사후 분석하여 자동으로 갱신(Auto-learn)하는 기능에 다소 중점을 두는 반면, PX4는 기본적으로 파라미터로 명시적 선언을 하거나 스마트 배터리를 통한 정보 전송을 신뢰하는 설계 방향을 많이 띈다.
