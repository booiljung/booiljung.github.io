51.4.1 프레네-세레 틀(Frenet-Serret Frame) 기반의 공간 오차 분해 및 벡터 투영(Vector Projection)

1. 개요

무인항공기가 비상 복귀(Return/RTL) 궤적이나 자율 임무 궤적(Mission Trajectory)을 추종(Tracking)할 때, 제어 루프(Control Loop)의 품질을 결정하는 가장 핵심적인 요소는 ’기체의 현재 위치가 이상적인 선형(또는 곡선형) 경로로부터 수학적으로 얼마나, 그리고 어느 방향으로 이탈했는가’를 정확하게 정량화하는 것이다.

PX4-Autopilot은 단순히 목표 지점 방향(Bearing)만 모만 바라보고 직행하는 원시적인 제어를 탈피하여, 기체의 비행 경로 곡선을 중심축으로 삼는 움직이는 지역 좌표계인 **프레네-세레 틀(Frenet-Serret Frame)**을 통해 3차원 전역 좌표 오차를 직관적이고 독립적인 제어 성분으로 분해(Decomposition)한다. 본 절에서는 이러한 기하학적 오차를 구하기 위한 내부 벡터 투영(Vector Projection) 메커니즘 식과 알고리즘 아키텍처를 상세히 기술한다.

2. 프레네-세레 틀(Frenet-Serret Frame)의 수학적 구성

수학의 미분 기하학(Differential Geometry)에서 임의의 부드러운 3차원 공간 곡선(비행 궤적)을 파라미터 s(호의 길이, Arc Length)로 정의할 때, 프레네-세레 틀은 곡선 상의 특정 수반점(이동하는 기체의 위치)에서 다음 세 가지 직교하는 단위 벡터(Unit Vector)로 동적 로컬 좌표계를 구축한다.

  1. 접선 벡터 (Tangent Vector, \vec{T}): 곡선의 진행 방향과 일치하는 벡터로, 수학적으로 곡선 궤적의 1차 미분에 해당한다. PX4의 NPF(Nonlinear Path Following) 비행 태스크 환경에서는 이전 웨이포인트(Previous Waypoint)에서 목표 웨이포인트(Target Waypoint)를 가리키는 정규화된(Normalized) 직선 방향 벡터 단위로 근사하여 산출된다.
  2. 법선 벡터 (Normal Vector, \vec{N}): 곡선의 휨(곡률, Curvature) 중심을 향하는 방향으로, 접선 벡터 \vec{T}의 변화율 방향과 일치하며 \vec{T}에 직교한다. 측면 방향의 오차를 계산하는 축이 된다.
  3. 종법선 벡터 (Binormal Vector, \vec{B}): 접선과 법선 벡터의 외적(\vec{T} \times \vec{N})으로 도출되며, 이 둘이 구성하는 곡선 평면 전체에 수직인 방향을 가리킨다. (보통 3D 비행 오차에서 고도 이탈 성분으로 투영됨)

3. 벡터 투영(Vector Projection)을 통한 공간 오차 분해 메커니즘

PX4 내의 제어 라이브러리는 기체의 현재 측정 좌표 \vec{P}_{actual}가 궤적 선분(이전 웨이포인트 \vec{P}_{prev}에서 타겟 \vec{P}_{target}) 위에서 기하학적으로 어떻게 분포되어 있는지를 분해한다.

기준 시작점에서 현재 기체까지의 물리적 오차 벡터를 \vec{E}라 하자.
\vec{E} = \vec{P}_{actual} - \vec{P}_{prev}

이 원시 벡터 \vec{E}는 궤적의 방향을 정의하는 접선 벡터 \vec{T}를 기준으로 벡터 투영(Vector Projection) 과정을 거치며 두 가지 독립된 제어축 성분으로 나뉜다.

3.1 진행 방향(Along-Track) 및 투영 거리 스칼라 (Projection Scalar)

기체가 기준 경로 위에서 ‘앞으로 얼마나 전진했는가’ 또는 ’진행이 얼마나 지연되었는가’를 나타내는 스칼라(Scalar) 값이다. 오차 벡터 \vec{E}와 접선 단위 벡터 \vec{T}의 내적(Dot Product)을 통해 투영(Project)된다.
E_{along} = \vec{E} \cdot \vec{T}
이 성분은 궤적 생성기(Trajectory Generator)가 예상한 가상 타겟 포인트의 속도 프로파일(Speed Profile)과 실제 비행 속도의 차이를 보상하는 병진 제어(Longitudinal Control) 피드백에 사용된다.

3.2 직교 교차점 오차 벡터 (Cross-Track Error Vector)

기체가 기준 궤적 선상에서 좌우 또는 상하(측면)로 ’얼마나 이탈했는지’를 수직 교차거리로 명시하는 핵심 벡터다. 원시 오차 벡터 \vec{E}에서 위에서 구한 직진 투영 성분 벡터를 소거(Subtraction)하여 도출한다.
\vec{E}_{cross} = \vec{E} - (\vec{E} \cdot \vec{T})\vec{T}
위 식을 통해 도출된 벡터의 크기인 \vert \vec{E}_{cross} \vert 값이 곧 프레네-세레 틀의 법선 방향(\vec{N}) 기준 이탈 거리가 된다. PX4 멀티로터의 위치 제어기(Position Controller)나 고정익의 L1 가이던스(L1 Guidance) 로직은 이 교차 오차 거리를 0으로 최소화시킬 수 있는 측면 역동학적 요구 가속도 수치(예: Roll Setpoint)를 산출해낸다.

4. 아키텍처 구조의 공학적 이점 (vs. 원시 각도 제어)

해당 궤적 분해 모델은, 타겟까지의 목표 방위각(Bearing to Target)과 현재 기체의 방위(Heading) 차이 각도(Angle Error)만을 비례제어(P-Control)로 틀어버리는 원시적인(레거시 시대의 Ardupilot 코드가 일부 채택했던) 요(Yaw) 제어기 대비 탁월한 이점을 선사한다.

  • 디커플링(Decoupling) 모델 강건성: 복잡한 타원체상의 3차원 동역학 제어 문제를 진행 방향(속도 제어)과 측면 직교 방향(조향 및 교차 거리 제어)이라는 서로 간섭(Coupling)이 없는 선형 1차원 PID의 결합으로 완벽하게 분할시켜 시스템의 수학적 안정성 여유도(Margin)를 극적으로 높인다.
  • 바람 벡터(Wind Vector) 보상: 강한 측풍(Crosswind) 조건 하에서, 직교 교차 오차 축(\vec{E}_{cross} 위주) 쪽에만 풍력 추정기(Wind Estimator)가 제공하는 측면 편이(Drift) 오프셋 속도를 더 쉽게 주입시킬 수 있어 크랩 비행(Crab Flight, 기수를 비튼 채 직선 궤적을 유지하는 현상)을 더 정교하게 연출할 수 있다.
