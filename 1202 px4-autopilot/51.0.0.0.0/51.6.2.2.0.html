51.6.2.2 요구 체공 에너지 밀도와 잔여 가용 에너지의 대소 비교 연산을 통한 경로 중간 강제 착륙(Mid-path Forced Landing) 비상 트리거 발생 및 GCS 오버라이드

드론이 자율 비행 임무(Mission)를 수행하거나 복귀(Return) 모드에 진입했을 때, 이륙 지점(Home)까지 온전히 도달할 수 있는지 판단하는 것은 기체 생존과 직결된다. PX4-Autopilot은 복귀 궤적 추종에 요구되는 **체공 에너지 밀도(유지 전력 \times 예상 도달 시간)**와 배터리의 **잔여 가용 에너지(SoC 유효분)**의 대소 비교 연산을 내부적으로 수행하여, 비행이 불가능하다고 판단되면 이륙 지점까지 가지 않고 현재 위치 혹은 현재 경로 중간에 즉시 내려앉는 경로 중간 강제 착륙(Mid-path Forced Landing) 비상 트리거를 작동시킨다.

1. 잔여 가용 에너지와 계층적 예외 처리 시스템

PX4 내부의 Commander 모듈은 잔여 가용 에너지를 기반으로 배터리 경고 임계값을 세 가지 계층(Low, Critical, Emergency)으로 분류하여 상태 머신(State Machine) 예외 처리를 수행한다.

  • Low (경고 단계, BAT_LOW_THR): 배터리가 일정 수준(예: 15%) 이하로 떨어지면 GCS에 단순히 알람만 전송한다. (과거에는 저전압복귀(Low Battery RTL)를 발동시키기도 했으나, 최신 펌웨어 아키텍처는 Critical 단계에서 RTL을 권장한다.)
  • Critical (복귀/착륙 단계, BAT_CRIT_THR): 시스템이 계산한 잔여 에너지 기준치(예: 7%) 도달 시 발동한다. 파라미터 COM_BAT_ACT 에 설정된 값에 따라 Return(복귀) 또는 Land(착륙)를 강압적으로 트리거한다.
  • Emergency (비상 착륙 단계, BAT_EMERGEN_THR): 에너지가 철저히 고갈(예: 5% 이하)되어 기체가 추락하기 직전의 상태이다. 이때 목표 지점이 얼마나 남았는지 상관없이 즉시 제어 방향을 수직 하강(Land)으로 꺾어버린다. 이를 **경로 중간 강제 착륙(Mid-path Forced Landing)**이라고 한다.

2. 경로 중간 강제 착륙(Mid-path Forced Landing) 로직의 아키텍처 이해

기체가 Return 모드로 전환되어 Home으로 날아오던 중이더라도 강한 역풍(Headwind)을 만나면 예상보다 요구 체공 에너지는 급증(배터리 광탈)하고 잔여 가용 에너지는 빠르게 줄어든다.

요구 에너지 (Return 소모량) > 잔여 가용 에너지 (배터리 SoC)

위 대소 비교 연산의 결과가 ’참(True)’이 되는 순간(즉, Home 도착 이전 공중에서 전원이 꺼질 확률이 높은 순간), PX4 Commander의 State Machine 구조는 Return 궤적 추종을 즉시 중단시킨다. 시스템은 강압적으로 내부 비행 모드를 AUTO.LAND로 변경하고 현재 위치의 지표면으로 곧장 하강시켜 최소한의 기체 전원이라도 유지한 채 지면에 닿도록 유도한다. 이것은 허공에서 프로펠러가 멈춰 자유낙하(Freefall)하여 기체가 완파되거나 지상의 인명피해를 유발하는 것을 막기 위한 가장 확실한 조치이다.

3. 소스 코드 분석 (Commander)

이러한 예외 처리는 주로 src/modules/commander/ 내부의 배터리 상태 감시 타이머 루틴에서 실행된다.

// src/modules/commander/StateMachines/StateMachines.cpp 관련 개념 코드
void updateBatteryFailsafe(battery_status_s &battery)
{
    // 배터리 상태가 Emergency (비상) 상태인지 평가
    if (battery.warning == battery_status_s::BATTERY_WARNING_EMERGENCY) {
        
        // 현재 Failsafe Action이 착륙(Land)으로 구속되지 않았다면 Land 트리거
        if (internal_state.main_state != commander_state_s::MAIN_STATE_DEFAULT_LAND) {
            
            // 콘솔 및 GCS 알림 전송
            mavlink_log_critical(&mavlink_log_pub, "Failsafe: Battery empty. Forced landing.");
            
            // 모든 수동 및 자율 궤적 명령을 중단하고 수직 강하(Land) 모드로 상태 강제 전환
            main_state_transition(internal_state, commander_state_s::MAIN_STATE_DEFAULT_LAND, ...);
        }
    }
}

4. GCS 오버라이드(Override) 메커니즘 및 관제 기술

비상 착륙(Forced Landing)이 항상 최선의 결과는 아닐 수 있다. 예를 들어 드론의 현재 위치 하단이 낭떠러지, 고속도로, 물가(호수) 등 물리적 회수가 불가능하거나 위험한 지형일 경우, 관제사(Operator)는 기체가 망가질지언정 안전한 구역까지 억지로 비행을 연장하도록 시스템을 덮어써야(Override) 할 때가 있다.

QGroundControl과 PX4는 이러한 휴먼 인 더 루프(Human-in-the-loop) 개입을 위해 GCS 오버라이드를 지원한다.

  1. 모드 강제 전환: 비상 착륙이 트리거된 직후라도 관제사는 조종기를 조작하여 식별하기 쉬운 수동 모드(Stabilized 또는 Position)로 스위치를 넘기거나, QGroundControl의 화면에서 MAVLink MAV_CMD_DO_SET_MODE 명령을 수동으로 전송해 비상 착륙 프로세스를 강제로 취소(Cancel)시킬 수 있다.
  2. 배터리 Failsafe 무시 설정: COM_BAT_ACT 등의 파라미터를 GCS에서 비행 전 0 (None/Warning Only)로 꺼버리면, 배터리가 0%를 가리키더라도 비상 액션을 취하지 않고 조종자의 명령만 끝까지 맹종하게 만들 수 있다. 다만 이 경우 ESC가 차단(Cut-off)될 때까지 날다가 자유낙하 할 위험을 인간이 전적으로 떠안게 된다.

5. Ardupilot 대비 관제 차이점

  • Ardupilot: BATT_FS_VOLT_ACT 및 BATT_FS_CRT_ACT 파라미터를 통해 각각의 배터리 단계별로 촘촘하고 세분화된 액션(RTL, SmartRTL, Land, Terminate)을 지정할 수 있으며, 배터리 용량(mAh)보다는 전압 베이스의 Failsafe 발동이 조금 더 직관적이고 기본적으로 세팅되어 있다.
  • PX4-Autopilot: 최신 펌웨어는 용량과 전압을 결합 추정한 상태(Warning Level) 자체를 기준으로 판단하는 것에 치중되어 있다. COM_BAT_ACT 파라미터 하나로 전반적인 Failsafe 동작을 통합 설정하는 경향이 있으며, Emergency 단계에 진입했을 때는 시스템이 사용자의 의도를 묻지 않고 일단 살리기 위해 무조건 Land로 찍고 내리는 매우 방어적이고 결정론적인 생존 설계 철학을 가진다.
