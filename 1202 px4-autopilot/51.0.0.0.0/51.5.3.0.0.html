51.5.3 제어기 포화(Controller Saturation) 상태 기반의 최종 지면 접촉 및 시동 해제 로직

무인항공기가 공역 비행을 마치고 지면에 안착할 때, 외부 계측기인 센서 기반의 운동학적 융합 판정 외에 프로세서 내부 제어 루프(Control Loop)의 소프트웨어적 반응 자체를 훌륭한 착륙 지지 지표로 삼는 방법이 존재한다. 제어기 포화(Controller Saturation) 상태 변수 지표는 물리적으로 기체가 더 이상 목표 지시 공간 궤적으로 진행할 수 없는 지표면 한계 부피 장벽(Boundary)에 도달했음을 나타내는 아주 강력한 제어 공학적 증거이다.
PX4-Autopilot은 이 점에 착안하여 내부 위치 제어 루프(Position Control Loop)와 자세 제어 루프(Attitude Control Loop) 내에서 역학적으로 발생하는 구동기 출력 명령의 단절 포화 현상을 기반 삼아 최종 지면 접촉(Landed Touchdown)을 확정 짓고 시스템 시동 차단 해제(Disarming)를 유도해낸다. 본 절에서는 이러한 폐루프 내부 제어기의 한계 포화 상태를 감지하여 물리적 착륙을 결정짓고 시동을 종료시키는 일련의 알고리즘 논리를 서술한다.

1. 위치/자세 제어기 포화(Controller Saturation) 방어 시스템 발생 원리

드론 기체가 수직 고도 하강을 진행하다 최종 지표면에 도달하여 가로막히게 되면, 목표 하강 감속도(Target Descent Rate)를 강제 유지하기 위해 조작을 가하는 Z축(상하) 위치 제어기와, 흩어지려는 수평 호버링 균형 밸런스를 상시 맞추려는 롤(Roll)/피치(Pitch) 자세 제어기가 견고한 바닥 저항이라는 막강한 외부 저항력과 충돌 반발하게 된다.

기체 프레임이 바닥을 뚫고 더 강하할 수 없으므로, 시스템 내에 포진한 PID 제어기(Proportional-Integral-Derivative Controller) 로직 중 특히 적분기(Integrator) 파트는 지속적으로 축적되는 예측 불가 원치 않는 고도 잔여 오차(Altitude Error)를 억지로 해소하기 위하여 모터 스로틀 구동 출력을 지속적으로 끝없이 줄이도록(혹은 강제 보상하도록) 명령을 내린다.
제어기 내의 이러한 피드백 맹목성 결과로, Z축 수직 추력 명령(Vertical Thrust Command)은 기체가 낼 수 있는 허용 단위 최소 출력(Minimum Thrust Limit)에 강제로 도달하여 더 이상 시스템 값(PWM 등)을 끌어내릴 수 없는 바닥 하단 포화(Lower Bounds Saturation) 한계 상태에 빠지게 된다.
동시에 기체 랜딩 휠이나 프레임 다리가 바닥 노면과 온전히 맞닿아 지지 기반이 생기면서 상대적인 X-Y축 자세 보정 복원력을 위한 미세 추력 델타 차이가 소멸되므로, 자세 제어 역시 의미 있는 모터를 통제할 조작 한계점에 봉착하거나 정지 불감대 구역(Dead-band)으로 자연스럽게 수렴한다.

2. PX4 코어의 지면 접촉 및 자율 시동 해제(Auto-Disarm) 판별 감지 로직

PX4 프레임워크의 플라이트 스택에 내장된 센서 관제 모듈인 land_detector (주로 멀티로터 전용 MulticopterLandDetector 서브 모듈) 내부 소스 코드는, 앞서 선행 언급되었던 센서 진동 속도 미분 교차 데이터 및 레이저 고도계 ToF 정적 수렴 정보와 함께 이 추력 ’제어기 포화 상태 플래그’를 결정적인 가중치로 퓨전(Fusion) 평가해 낸다.

포화 판별 관문에서 시스템은 다음 두 가지 주요 변수를 검사한다.

  1. 스로틀 하한 포화 강제 검사: 수직 Z축 제어 추력 내부 명령(thrust_setpoint)이 시스템 파라미터로 지정된 극단적 최소 스로틀 한계치(MPC_THR_MIN) 수치 바운더리에 일정 샘플 틱(Tick) 연속적으로 머무는 상태인지 여부를 상시 추적한다.
  2. 구동기 제어 포화 플래그(Saturation Flag) 검출 조명: 믹서(Mixer) 모듈로 하달되는 내부적인 ActuatorControls uORB 메시지 메타 데이터 토픽 상에서, 제어기 프로세스가 자가 진단한 논리 구동 한계 봉착 상태 깃발(Saturation Flag)이 ‘True(활성화)’ 되었는지 Boolean 평가를 수행한다.

이러한 제어 소프트웨어 논리적 포화 조건이 앞서 분석한 기계/광학적 타 센서 지표와 결합 융합되어 내부 임계 설정 시간 이상 동시 유지 공명될 경우, 상태 머신은 최종 착륙 단계인 Landed 상태로 전이 승격을 일으키며, 내부 최상위 코어 시스템인 커맨더(Commander) 모듈로 즉각 시동 해제(Disarm) 하달 명령 메커니즘을 트리거(Trigger) 시킨다.

3. Ardupilot과 대비되는 시동 해제 관제 특성 및 MAVLink 응용

Ardupilot 오픈소스 기반의 비행 플랫폼은 지면 안착 감지 판단이 서고 난 후에도 상당히 보수적인 모터 스로틀 컷오프 최소화 공회전 대기 상태를 장시간 유지하며, 주로 사람인 조종조종사가 RC 조종기 매뉴얼 스틱(예: Yaw 스틱 좌하단 유지 입력 유도 등) 기동을 이용해 수동 개입으로 물리적 디스암(Manual Disarm)을 지시하는 방식을 혼용하는 기조가 강하게 남아 있다.
반면, PX4-Autopilot은 멀티로터나 VTOL 항공 자율 임무의 ‘완전한 무인 자동화(Full Autonomy)’ 생태계 달성을 설계 일념 목표로 삼아 제작되어, 착륙 이벤트가 확정되고 내부 Auto-Disarm 타이머(통상 2.0초 디폴트) 경과 시 즉시 시스템 커맨더 메인 모듈 자체 논리 로직에 의거해 구동 액추에이터 출력 포트를 물리적으로 완전히 절연 차단하는 제어권 자율 환수를 매우 신뢰도 있고 과감하게 수행해 낸다.

4. 관제 구성 가이드 요소 및 QGroundControl 연동 시그널링 모니터링

안전하고 유기적인 최적의 포화 기반 디스암 제어를 현장에서 올바르게 구성 구동시키기 위해 개발자는 다음과 같은 PX4 튜닝 환경 설정(Parameter)을 완비해야 한다.

  • COM_DISARM_LAND: 무인 기체가 Landed (최종 착륙 확정) 상태로 내부 인식된 시점부터 전체 시스템 모터를 완전히 셧다운(Disarm) 시킬 때까지 카운팅하며 대기 초과하는 초(Second) 단위 지연 딜레이 타임아웃 지표. 이 파라미터가 너무 짧으면 공중 난기류 센서 교란 시 거짓(False) 안착 판정에 의한 허공 추락의 주된 원인이 되며, 반대로 너무 길면 로터가 노면 그라운드 면에서 요동치며 헛돌아 파손/과열될 기계적 화재 위험이 유발된다.
  • MPC_THR_MIN: 위치 제어기가 하강 포화를 판정 짓는 데드라인 최하단 트러스트 추력 바운더리를 규정한다. 탑재한 멀티로터의 적재 중량 무게 대 추력비(Thrust-to-Weight Ratio) 스펙 곡선을 신중히 고려하여, 기체가 떠오르지 않는 정확하고 안전한 저공회전(Idle Bounds) 모터 역치 임계값 수준으로 공학적으로 맞춰야 한다.

해당 하드웨어적 디스암 로직 절차의 수행이 완전히 완료되면, MAVLink 네트워크 프로토콜 상에서 원격지 생사 확인을 전담하는 HEARTBEAT 패킷 프레임 내의 베이스 상태 모드(Base Mode) 속성에서, 그간 켜져 있던 MAV_MODE_FLAG_SAFETY_ARMED 활성화 깃발 논리 비트가 즉시 0으로 강제 해제 철회된다.
이를 무선 수신한 QGroundControl 관제 모니터 툴 컴포넌트 시스템은 조종 비행 대시보드 화면 중앙의 프로미넌트(Prominent) 암(Arm) 상태 위젯을 즉각 녹색(안전 상태 표상)의 ‘Disarmed’ 표기 상태로 업데이트 점등 인가하여, 설계된 비행 임무의 완벽한 물리적 종료 사실을 지상 파일럿의 시각 인지 구조에 최종 각인시키는 것으로 전체 비행 라이프사이클 궤적의 여정을 마무리 짓는다.
