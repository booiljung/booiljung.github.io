51.4.1.2 기체 위치 벡터의 목표 선분 위로의 스칼라 정사영(Scalar Projection) 연산을 통한 진행 방향 오차(Along-Track Error) 정량화

본 절에서는 무인항공기(UAV)가 비행 중 사전에 정의된 두 경유지(Waypoint)를 잇는 목표 선분(Target Line Segment)을 추종할 때, 위치 벡터의 스칼라 정사영(Scalar Projection)을 이용하여 기체의 진행 방향 오차(Along-Track Error)를 정량화하는 기법을 상세히 논의한다. 해당 내용은 최신 PX4-Autopilot(v1.14 기준)의 비행 제어 및 관제 스택 내 위치 제어 알고리즘과 수학적 아키텍처를 기반으로 전개된다.

1. 아키텍처 및 제어 기술 이해: 스칼라 정사영을 통한 진행 방향 거리 산출

비행 제어 루프(Flight Control Loop) 내에서 기체가 이상적인 선상에 배열되어 있는지 확인하기 위해 교차 오차(Cross-Track Error)와 함께 반드시 진행 방향 오차 및 도달 거리를 계산해야 한다. 진행 방향 거리(Along-Track Distance)란 현재 기체의 3차원 위치 벡터를 이전 웨이포인트(Previous Waypoint)와 다음 웨이포인트(Next Waypoint)를 잇는 궤적 선분에 직교 투영(Orthogonal Projection)하여 얻어진 스칼라 길이를 의미한다.

1.1 수학적 원리와 수식 전개

지구 고정 좌표계(Earth-Fixed Coordinate System, NED 프레임)에서 이전 웨이포인트의 위치를 벡터 A, 다음 웨이포인트의 위치를 벡터 B, 그리고 센서 모델(GPS/RTK-GPS 및 IMU 퓨전)을 통해 추정된 기체의 현재 위치를 벡터 P라 정의한다.

목표 선분의 방향 벡터 \vec{AB}는 다음과 같이 정의된다.
\vec{AB} = B - A

이전 웨이포인트 A에서 현재 기체 위치 P를 향하는 상대 위치 벡터 \vec{AP}는 다음과 같다.
\vec{AP} = P - A

벡터 \vec{AP}를 벡터 \vec{AB} 위로 투영한 스칼라 정사영의 길이 d_{along} (진행 방향 거리)은 내적(Dot Product)을 이용하여 도출한다.
d_{along} = \frac{\vec{AP} \cdot \vec{AB}}{\vert \vec{AB} \vert}
여기서 기호 \vert 대신 \vert를 사용하여 벡터의 크기(Norm) \vert \vec{AB} \vert를 명시하였다. 이렇게 구한 d_{along} 값을 전체 선분 길이 \vert \vec{AB} \vert로 나누면 목표 선분 상에서의 궤적 진행률(Progress Ratio)을 정량화할 수 있다.

1.2 기하학적 형태의 가시화

진행 방향 오차와 직교 투영의 기하학적 형상은 다음 다이어그램과 같이 도식화할 수 있다.

graph TD
    subgraph "Vector Projection Frame"
        A[Prev Waypoint A] -->|Target Line Vector AB| B[Next Waypoint B]
        A -->|Current Position Vector AP| P[Current UAV Position P]
        P -.->|Orthogonal Projection| P_proj[Projected Point P']
        A -->|Along-Track Distance| P_proj
        P_proj -->|Distance to go| B
        P -.->|Cross-Track Error| P_proj
    end

도출된 투영점 P'는 A + d_{along} \frac{\vec{AB}}{\vert \vec{AB} \vert}로 결정되며, 해당 점의 위치 정보를 토대로 MAVLink 관제와 제어기에서 잔여 거리(Distance to Go)가 산출된다.

2. PX4-Autopilot의 소스 코드 구현 및 Ardupilot 대비 제어 관점 차이

2.1 PX4-Autopilot 위치 제어 모듈 분석

PX4-Autopilot 환경에서는 비행 모드(Flight Mode) 중 Mission 모드 또는 Return 모드 실행 시 PositionControl (위치 제어기) 알고리즘 스택에서 위와 같은 수식을 활용한다. 특히 NPFG(Nonlinear Position Feedback Guidance)와 로컬 궤적 목표물(Local Trajectory Target) 생성 로직이 이 스칼라 투영 연산을 수행하기 위해 C++ 코드로 작성되어 있다. FlightTaskAutoMapper와 npfg 모듈이 벡터 사이의 스칼라 정사영을 기반으로 현재의 진행률(Progression)을 실시간 갱신한다.

2.2 Ardupilot의 L1 Controller와의 철학적 차이

Ardupilot은 고정익(Fixed-wing)과 멀티로터의 제어 아키텍처에서 주로 L1 Navigation Controller 기준의 가상 점 추종(Virtual Point Tracking) 방식을 차용하는 편이다. 이와 달리 PX4-Autopilot의 최신 아키텍처(버전 1.14 이후)는 NPFG 유도 법칙(“A New Extension of Nonlinear Guidance Logic for Trajectory Tracking”)을 적용하거나 기구학적 한계를 고려한 Jerk 제한형 궤적 탐색(Jerk-limited Trajectory Tracking) 방식을 융합하여 사용한다. 두 펌웨어 간의 주요 차이는 경로 추종 오차를 정량화하고 이를 자세 제어(Attitude Control) 명령으로 환산하는 수식 구조에서 발생한다. PX4는 Along-Track의 비율을 바탕으로 타겟 감속점과 교차 오차 복구 지점을 매우 밀접한 수학 행렬로 동시에 계산하는 데 유리한 구조를 갖는다.

3. GCS(QGroundControl)와의 MAVLink 상호작용 및 ROS2 활용

3.1 QGroundControl 관제 시점에서의 활용

산출된 스칼라 정사영 비율 및 진행 방향 오차는 GCS(지상 관제 시스템)인 QGroundControl(v4.3.0 이상 권장)에서 시각화된다. QGroundControl 내비게이션 관리 매니저는 Autopilot으로부터 수신한 NAV_CONTROLLER_OUTPUT 메시지의 wp_dist 필드를 이용해 기체가 다음 Waypoint에 성공적으로 도달했는지를 판단한다. 도달 판별 알고리즘은 벡터의 크기 연산 결과(\vert B - P \vert 혹은 목표선상 잔여 거리)가 수용 반경(Acceptance Radius) 이내에 편입되었는지를 기준으로 동작한다.

3.2 ROS2 및 Micro XRCE-DDS 에이전트를 통한 외부 자율 판단 전략

심화된 자율 에이전트(Autonomous Agent) 연구 환경에서는 ROS2 미들웨어를 PX4와 연동(uXRCE-DDS)하여 외부 관제를 시도한다. 외부 컴퓨팅 보드에 탑재된 ROS2 노드는 기체의 센서(VIO, LiDAR, RTK GPS 등)를 융합하여 얻은 고도화된 위치 토픽(VehicleOdometry)과 목표 궤적 정보를 구독(Subscriber)한다. 이후 ROS2 노드 측에서 자체적으로 С++ 또는 Python을 통해 앞선 1.1절의 스칼라 정사영 모델링을 재수행하여 진행 방향 오차(Along-track divergence)를 실시간 판단하고, 임계 범위 이탈 시 Offboard 제어 모드로 회피 명령 개입을 인가하는 방식으로 고도화된 관제 시스템을 설계할 수 있다. 이는 단순한 Ardupilot 대비 한층 유연성을 제공하는 차별화된 오프보드 연동 형태라 할 수 있다.
