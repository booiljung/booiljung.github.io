51.5.3.2 체공 판단(Land Detector) 모듈의 상태 천이가 완료된 후 발행되는 자동 시동 해제(Auto-Disarm) 트리거 및 PWM 출력 차단 시퀀스

1. 개요 및 역학적 배경

무인항공기(UAV)가 복귀(Return) 또는 자동 착륙(Auto Land) 임무를 수행하여 지면에 안전하게 접촉(Touchdown)한 이후에는 기체의 모터 회전을 신속하고 안전하게 정지시켜야 한다. 기체가 접지한 상태에서 프로펠러가 계속 회전하면 지면 요철에 의한 기체 전복 혹은 인명 피해를 유발할 수 있다. PX4-Autopilot은 land_detector(체공 판단) 모듈이 기체의 완전히 착륙한 상태를 인지하고 상태 천이를 완료한 후, 일정 지연 시간(Timeout)을 거쳐 commander(커맨더) 모듈을 통해 **자동 시동 해제(Auto-Disarm)**를 트리거한다. 이어 시스템은 물리적인 구동부로 향하는 PWM 또는 DShot 제어 신호를 차단(PWM Kill Sequence)하여 기체를 최종 대기 상태로 전환한다.

2. 자동 시동 해제(Auto-Disarm) 트리거 로직

PX4-Autopilot 내에서의 자동 시동 해제는 기체의 비행 모드를 관리하는 핵심 모듈인 commander에 의해 관장된다.

2.1 상태 천이 및 지연 시간 타이머

land_detector 부속 모듈(예: MulticopterLandDetector)은 기체의 접지 상태를 판별하여 vehicle_land_detected uORB 메시지의 landed 플래그를 true로 설정한다. commander는 해당 메시지를 구독(Subscribe)하며, landed 상태가 지속되는 시간을 모니터링한다.
관련 파라미터는 COM_DISARM_LAND로, 기체가 착륙한 상태에서 자동으로 시동 시스템을 차단하기까지의 대기 시간(단위: 초)을 정의한다.

  1. vehicle_land_detected.landed == true 획득
  2. 내부 타이머(Timer) 시작
  3. 경과 시간이 COM_DISARM_LAND를 초과할 때까지 기체 상태가 다시 이륙(In-air)으로 변환되지 않음 확인
  4. 시스템 시동 해제 루틴 호출

2.2 시동 해제 처리 및 메시지 발행

조건이 충족되면 commander는 actuator_armed uORB 메시지의 armed 플래그를 false로 갱신하여 시스템 전역에 시동 해제 상태를 브로드캐스트한다. 동시에 MAVLink의 HEARTBEAT 시스템 상태(System Status) 필드를 MAV_STATE_STANDBY로 천이시켜 지상 관제 시스템(GCS)에 통보한다.

3. PWM 출력 차단 시퀀스 (PWM Kill Sequence)

시동 해제 트리거가 발생하면, 시스템 코어에서 액추에이터(모터 및 서보)로 전달되는 실제 제어 신호를 전기적으로 차단하는 백엔드(Back-end) 로직이 수행된다.

3.1 액추에이터 믹서 및 출력 드라이버

actuator_armed 상태가 업데이트되면 믹서(Mixer) 또는 control_allocator는 더 이상 제어 입력(actuator_controls_0)을 모터 회전 명령으로 매핑하지 않는다. 대신, 각 출력 핀 버스(PWM_OUT, DShot 등)를 관리하는 하드웨어 드라이버 모듈(예: fmu 또는 px4io)이 다음과 같은 동작을 수행한다.

  1. PWM 프로토콜의 경우: 출력 신호를 미리 정의된 PWM_DISARMED 파라미터 값(대략 900µs 이하 혹은 0µs)으로 낮추거나 펄스(Pulse) 출력을 중단시켜 원시 ESC(Electronic Speed Controller)가 구동되지 않게 강제한다.
  2. DShot 마이크로코드의 경우: 디지털 모터 정지(텔레메트리 0) 패킷 비트를 전송하여 즉각적인 모터 전자 제어를 중단시킨다.

4. Ardupilot 대비 아키텍처 및 파라미터 차이

자동 착륙 후의 시동 해제 로직에 있어 두 오픈소스 펌웨어의 아키텍처는 유사하나 명칭과 모듈 결집도에 차이가 있다.

  • PX4-Autopilot: COM_DISARM_LAND라는 커맨더(Commander) 종속의 범용 파라미터를 사용하여 시동 해제를 트리거한다. 체공 판단 모듈(land_detector), 비행 제어 및 모드 관리 모듈(commander), 액추에이터 관리 로직(control_allocator 및 I/O 드라이버)이 uORB 브로커를 통해 철저하게 비동기(Asynchronous) 이벤트 방식으로 분리되어 있다.
  • Ardupilot: DISARM_DELAY라는 파라미터로 시동 해제 딜레이를 설정한다. Copter 코드 뼈대에서 Copter::land_detector가 착륙을 확정한 후, 차량의 상태 전역 객체(AP_Vehicle) 내에서 모터 객체(AP_Motors)의 armed 함수를 직접 해제(False)해버리는 함수 호출 구조가 강하다. 따라서 스레드 간의 통신 지연 비용이 적은 반면 모듈화에 있어서는 구조적 결합도가 비교적 높다.

5. 모듈 간 연동 시퀀스 다이어그램

접지 판단 직후부터 실제 하드웨어 신호 차단까지의 내부 메시징 흐름은 다음과 같다.

sequenceDiagram
    participant LD as 체공 판단 (land_detector)
    participant CMD as 커맨더 (commander)
    participant CA as 제어 할당기 (control_allocator)
    participant DRV as 출력 드라이버 (fmu / px4io)
    participant ESC as 전자변속기 (ESC)

    LD->>CMD: 접지 상태 발행 (vehicle_land_detected: landed=true)
    CMD->>CMD: COM_DISARM_LAND 타이머 시작
    Note over CMD: 지정된 시간 동안 상태 모니터링
    CMD->>CA: 시동 해제 명령 발행 (actuator_armed: armed=false)
    CA->>CA: 제어 입력 무시 및 모터 목표치 초기화
    CA->>DRV: 디스암 출력 명령 전달 (Zero/Disarm PWM)
    DRV->>ESC: 펄스 하향 또는 차단 (PWM_DISARMED)
    Note over ESC: 모터 회전 전기적 정지

6. QGroundControl (GCS) 측면의 상태 추적

기체가 자동 시동 해제 시퀀스를 통과하면 PX4는 MAVLink의 HEARTBEAT 메시지의 base_mode 마스킹에서 MAV_MODE_FLAG_SAFETY_ARMED 비트를 해제한다(0으로 변경). QGroundControl(QGC)은 백엔드 Vehicle 객체를 통해 이를 식별하여 상태 머신을 업데이트한다. 동시에 사용자에게 오디오 알람(“Disarmed”)을 송출하고 UI의 상단 기체 상태 바퀴를 초록색 “Ready” 혹은 “Disarmed” 상태로 전환한다. GCS 개발자는 이 비트를 모니터링하여 자율 비행의 단일 임무 흐름이 완전히 종료되었음을 확인하는 체크포인트로 삼고, 로그 다운로드(Log Download) 같은 유휴 시간 백그라운드 작업을 시작하도록 시스템을 설계할 수 있다.
