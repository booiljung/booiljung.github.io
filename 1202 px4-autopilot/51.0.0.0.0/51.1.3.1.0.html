51.1.3.1 마르코프 결정 과정(Markov Decision Process) 관점에서의 비행 단계 전이 확률(Transition Probability) 모델링 및 안전성 평가

1. 개요 및 모드(Mode)의 이해

PX4-Autopilot(펌웨어 v1.14 기준)의 복귀(Return/RTL) 모드는 본질적으로 확정적(Deterministic) 유한 상태 머신(Finite State Machine, FSM) 규칙에 기반하여 구현되어 있으나, 이를 학술적 관점의 마르코프 결정 과정(Markov Decision Process, MDP)으로 추상화하면 고도화된 안전성(Safety) 평가와 자율 에이전트 기반 의사 결정을 실험할 수 있다. 실제 비행 환경은 센서 잡음(Sensor Noise), 돌풍 교란(Wind Disturbance), 그리고 하드웨어의 미세한 지연 시간으로 인해 완벽히 결정론적이지 않다. 따라서 MDP를 도입하여 각 비행 상태 간의 전이 확률(Transition Probability)을 모델링함으로써, 예상치 못한 결함이나 극한 환경 하에서의 생존 여부를 확률론적으로 검증하고 보완하는 것은 매우 중요한 공학적 접근이다.

2. 모형의 정의 (아키텍처 및 제어 기술 이해)

MDP 관점에서 복귀 모드의 아키텍처는 튜플 (S, A, P, R, \gamma)로 재정의된다.

  • 상태 환경(State Space, S): 서브 모드 집합 S = \{\text{Climb}, \text{Transit}, \text{Loiter}, \text{Descend}, \text{Land}, \text{Failsafe}\}
  • 행동 공간(Action Space, A): 비행 제어기(Flight Controller)의 이산적 결정 공간 A = \{a_{\text{maintain}}, a_{\text{transition\_next}}, a_{\text{abort}}\}
  • 전이 확률(Transition Probability, P): P(s_{t+1}|s_t, a_t)로 정의되며, 행동 a_t를 취했을 때 다음 상태 s_{t+1}로 전이할 확률이다. 돌풍이나 GPS 오차 시 이 확률이 1.0(결정론적)을 벗어나 분포를 갖게 된다.
  • 보상 함수(Reward Function, R) 및 할인 인자(\gamma): RTL 목적지 도달 시 양의 보상, Failsafe 추락 시 음의 보상.

2.1 마르코프 전이 그래프 (Transition Graph) 도식화

다음은 상태 천이의 불확실성을 표현한 마르코프 모델 전이 곡선이다.

stateDiagram-v2
    [*] --> Climb : Return 발동
    Climb --> Climb : P=0.2 (고도 확보중, 유지)
    Climb --> Transit : P=0.75 (고도 도달성공)
    Climb --> Failsafe : P=0.05 (센서 에러)
    
    Transit --> Transit : P=0.1 (경로 주행중)
    Transit --> Loiter : P=0.8 (반경 도착)
    Transit --> Failsafe : P=0.1 (배터리 전압 급감)
    
    Loiter --> Descend : P=0.95 (시간 도과)
    Descend --> Land : P=0.99 (하강 완료)
    Land --> [*]

위 그래프에서 전이 확률 P 수치 값은 예제 형식이며, 현실에서는 EKF2 필터 혁신(Innovation) 분산량에 의해 비선형적으로 계산된다.

3. 센서 이해 (Sensor Understanding)와 확률론적 교란

센서의 무결성은 전이 확률 P(s_{t+1}|s_t, a_t)을 동적으로 변동시키는 가장 큰 교란 변수이다.

  • EKF2 관점: 기압계(Barometer) 기준 상대 고도(Relative Altitude) 측정치의 공분산(Covariance) 허용치가 EKF2_HGT_NOISE를 상회하게 되면, 상태가 s_t = \text{Climb}에서 s_{t+1} = \text{Transit}으로 넘어가야 할 “목표 고도 달성 확률“이 유의미하게 저하된다. 상태 전이 결정 임계값이 확률론적으로 흐려지면서 FSM은 다음 단계 진입을 주저하게 되는데, 이는 과도기 상태(Transient State)의 체류 시간 연장을 발생시킨다.
  • GNSS 의존성: GPS의 수평 정밀도 저하(HDOP 증가)는 \text{Transit}에서 \text{Loiter}로의 전이 조건을 흔들며, \vert \Delta x \vert < \text{RTL\_ACCEPTED\_RAD} 조건의 판별 과정에 마르코프 잡음(Markovian Noise)을 더하여 오차 확률을 증가시킨다.

4. 소스 코드 이해 (Source Code Understanding)에 반영된 확률적 평가

실제 PX4 Navigator 모듈(src/modules/navigator/rtl.cpp)은 확률 함수보다는 엄격한 부동소수점 검증 및 허용 이산시간 타이머를 통해 이를 보완한다. MDP 이론의 관찰 시퀀스를 코드로 반영하면 다음과 같다.

// MDP 기반 전이 확률 보정을 위한 추상화 코드의 형태 (개념적 예시)
bool RTL::check_transit_to_loiter_probability(const float vehicle_pos[2], const float target_pos[2]) {
    float pos_error = get_distance_to_next_waypoint(vehicle_pos[0], vehicle_pos[1]);
    
    // 센서 분산을 고려한 마르코프 전이 가치 평가
    float ekf2_pos_variance = _estimator_status_sub.get().pos_horiz_accuracy;
    
    // 확정적 임계값(RTL_ACCEPTED_RAD)에 분산을 더해, 
    // 전이 오판별 방지를 위한 조건 검사(결정론적 코드를 MDP 관점으로 투영)
    if (pos_error < (_param_rtl_accepted_rad.get() + ekf2_pos_variance)) {
        // 전이 확률 곡선상 95% 이상 신뢰 구간 달성 간주
        return true; 
    }
    return false;
}

5. Ardupilot 모델링과의 차이 (Comparison with Ardupilot)

MDP 관점에서 두 프레임워크의 차이는 전이 제어의 아키텍처 결합도에서 비롯된다.

  • PX4-Autopilot: 상태(State)가 클래스의 독립적 스코프로 묶여있어, 학술적으로 \text{Transition Function } P를 노드 간 마르코프 지향적 추정기(Estimator)로 이분법적 교체가 용이하다. 즉, 특정 조건에 도달한 return true/false의 이산 로직을 퍼지 제어(Fuzzy Control) 혹은 연속적인 확률 함수로 치환하는 실험에 유리하다.
  • Ardupilot: 단일 파일(mode_rtl.cpp) 내에 상승, 비행, 강하 분기가 연속되어 있어, 전이 확률(Transition Probability) 행렬 계수들을 추출하고 수치 해석적(Numerical Analytical) 모듈화 분석을 진행하기보다 제어 루프를 통한 몬테카를로(Monte Carlo) 시뮬레이션에 전체를 통째로 의존하는 방식이 주로 쓰인다.

6. MAVLink 및 ROS2 연동 이해 (MAVLink and ROS2 Integration)

복귀 궤적의 안정성을 높이기 위해 외부 연산 자원을 결합할 경우, MDP의 정책(Policy \pi) 최적화를 GCS 혹은 Companion Computer에게 양도할 수 있다.

  • MAVLink 이해: 상태 S와 현재 보상(Reward) 척도를 알기 위해 QGroundControl(v4.3)은 HIGH_LATENCY2 패킷 혹은 SYS_STATUS 내 각종 센서 에러 카운터를 파싱한다. 이를 통해 연결된 지상국에서 비행 환경 전이 확률의 변동성(풍속 증가, 위성 수 감소)을 인지하여 운용자에게 주의를 주거나 강제 제어권 이양을 요구한다.
  • ROS2 연동 이해: uXRCE-DDS 미들웨어를 통해 발신되는 기체 오도메트리(VehicleOdometry) 및 상태 벡터(VehicleStatus) 토픽을 ROS2 노드가 구독한다. 이 데이터를 바탕으로, 강화 학습 알고리즘은 MDP 관점에서 “현재 기상 상황 및 센서 열화 시, 단순히 순항(Transit)을 유지하는 행동(Maintain)보다 비상 착륙(Descend Failsafe) 행동의 기대 반환값(Expected Return)이 크다“고 도출하면 VehicleCommand를 역발행(Publish)하여 궤도를 임의 수정한다. 이러한 형태의 외부 심층 강화 학습 관리는 자율 에이전트 관제의 핵심 기술이 된다.

7. 설정 및 구성 (Configuration and Setup)

안전성을 위협하는 불안정 상태로의 전이 확률(Transition Probability to Failsafe)을 최소화하기 위한 구성 파라미터는 다음과 같다.

  • RTL_ACCEPTED_RAD: 순항에서 체공으로 넘어가는 확률 밀도 함수의 분산을 허용하는 이격 반경. 수치를 크게 하면 확률적으로 도달 판단이 성급해지고 궤적이 매끄러우나 정확도가 떨어진다.
  • EKF2_GPS_CHECK: GPS 품질 검증 허용치. MDP 관점에서 위치 상태 관측 행렬(Observation Matrix, 부분 관찰자 모델인 POMDP로 취급 시)의 무결성을 보장하는 중요한 노브(Knob)이다.
  • COM_RC_IN_MODE: 원격 조종기(RC) 명령의 마진 허용. 데이터 링크 유실 시 RTL 발동 확률을 규정한다.
