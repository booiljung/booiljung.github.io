51.1. 복귀(Return/RTL) 모드의 범용 시스템 아키텍처 및 유한 상태 머신(FSM) 심층 설계

복귀(Return/RTL) 모드는 예상치 못한 비상 상황이나 사용자의 명시적 명령에 의해 기체를 안전한 위치로 귀환시키는 필수적인 기능이다. PX4-Autopilot(v1.14 기준)에서 복귀 모드는 단순한 경유지(Waypoint) 이동이 아닌, 고도의 내결함성(Fault Tolerance)을 갖춘 독립적인 시스템 아키텍처와 유한 상태 머신(Finite State Machine, FSM)을 기반으로 동작한다. 본 절에서는 복귀 모드의 아키텍처 구조와 제어 흐름을 심층적으로 분석한다.

1. 범용 시스템 아키텍처(Universal System Architecture) 개요

PX4-Autopilot의 아키텍처는 기능별로 철저하게 모듈화되어 있다. 복귀 모드는 제어기(Controller) 레벨이 아닌, 임무 및 궤적 계획을 담당하는 내비게이터(Navigator) 모듈에서 처리된다. GCS(QGroundControl v4.3 기준) 또는 송신기(RC)로부터 복귀 명령이 인가되면, 시스템은 다음과 같은 데이터 흐름을 거친다.

  1. Commander 모듈: MAVLink를 통해 수신된 MAV_CMD_NAV_RETURN_TO_LAUNCH 메시지를 파싱하고, 현재 기체의 상태(시동 여부, GPS 수신 상태 등)를 검증하여 모드 전환이 가능한지 판별한다.
  2. Navigator 모듈: Commander로부터 모드 전환 승인을 받으면, 내부적으로 navigator_mode를 Return으로 변경한다. 내비게이터는 vtol_att_control, fw_pos_control_l1, 혹은 mc_pos_control과 같은 하위 제어기(Position Controller)로 궤적 셋포인트(trajectory_setpoint)를 uORB 메시지로 퍼블리시(Publish)한다.
  3. Position Controller: 수신된 셋포인트를 기반으로 추력(Thrust)과 자세(Attitude) 셋포인트를 산출하여 자세 제어기(Attitude Controller)로 전달한다.

이러한 수직적 분리 아키텍처는 Ardupilot이 특정 비행 모드의 로직을 메인 제어 루프에 다소 강하게 결합시키는 방식과 대조적으로, PX4가 코드를 유지보수하고 새로운 비행 모드를 확장하는 데 매우 유리한 구조를 제공한다.

2. 복귀 모드의 유한 상태 머신(FSM) 알고리즘

복귀 모드의 내부는 기체가 위치한 고도 및 거리에 따라 상태를 분기하는 유한 상태 머신으로 설계되어 있다. 시스템은 C++의 클래스 다형성(Polymorphism)을 활용하여 상태 객체를 관리하며, 주요 상태는 다음과 같이 정의된다.

stateDiagram-v2
    [*] --> STATE_CLIMB : 복귀 명령 수신
    STATE_CLIMB --> STATE_RETURN : 설정된 복귀 고도 도달
    STATE_RETURN --> STATE_DESCEND : 홈(Home) 또는 랠리 포인트 도달
    STATE_DESCEND --> STATE_LAND : 하강 고도(RTL_DESCEND_ALT) 도달
    STATE_LAND --> [*] : 지면 접촉 및 시동 해제(Disarmed)

    state STATE_CLIMB {
        [*] --> CheckCurrentAlt
        CheckCurrentAlt --> ClimbToRTLAlt : 현재 고도 < RTL_RETURN_ALT
    }

2.1 STATE_CLIMB (상승 상태)

복귀 명령이 활성화되면 기체는 가장 먼저 장애물을 회피하기 위해 안전 고도로 상승한다. 파라미터 RTL_RETURN_ALT에 설정된 고도와 기체의 현재 고도를 비교하여, 이미 설정 고도보다 높을 경우 상승 단계는 즉각 생략(Skip)되고 다음 상태로 전이된다.

2.2 STATE_RETURN (순항 상태)

설정된 복귀 고도에 도달하면, 기체는 목표 지점(홈 위치 또는 랠리 포인트)을 향해 2D 평면상에서의 순항 비행(Cruise Flight)을 시작한다. 이 과정에서 MPC_XY_CRUISE 또는 RTL_SPEED 등에 정의된 속도 제한 파라미터가 적용되며, 로컬 포지션 에러(Local Position Error)를 최소화하도록 위치 제어기가 동작한다.

2.3 STATE_DESCEND (하강 상태)

목표 지점의 반경(Acceptance Radius) 내에 진입하면 기체는 하강을 시작한다. 지표면 근처에서의 정밀한 제어를 위해 RTL_DESCEND_ALT 고도까지는 비교적 빠른 속도로 하강하고, 이 고도를 기점으로 더 느린 착륙 속도로 전환한다.

2.4 STATE_LAND (착륙 및 종료 상태)

지면과 접촉할 때까지 하강 속도(MPC_LAND_SPEED)를 유지한다. 착륙 감지기(Land Detector)가 가속도계 데이터와 모터 출력을 분석하여 지면 접촉(Landed)을 판독하면, 일정 시간(RTL_LAND_DELAY) 대기 후 시동 메커니즘을 해제(Disarm)함으로써 FSM 루프가 완전히 종료된다.

3. 코드 레벨의 객체 지향적 접근

소스 코드 측면에서 PX4 펌웨어의 src/modules/navigator/return.cpp 파일을 살펴보면, Return 클래스는 MissionBlock 클래스를 상속받아 구현된다. 부모 클래스가 제공하는 범용적인 웨이포인트(Waypoint) 생성 루틴을 재사용하면서, 각 상태(STATE_CLIMB, STATE_RETURN 등)마다 set_return_item() 함수를 오버라이딩(Overriding)하여 궤적을 동적으로 생성한다. 이러한 추상화(Abstraction)는 자율 에이전트 개발자가 향후 새로운 유형의 복귀 궤적(예: 곡선형 복귀 경로 등)을 설계할 때 기존 아키텍처를 손상시키지 않고 확장할 수 있는 기반이 된다.
