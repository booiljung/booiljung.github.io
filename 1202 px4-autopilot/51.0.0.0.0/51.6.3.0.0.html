51.6.3. 관제사 개입에 의한 휴먼-인-더-루프(Human-in-the-loop) 제어권 회수 및 동역학 스무딩

무인항공기(UAS)의 자율 비행, 특히 복귀(Return/RTL) 절차 중에 발생하는 예기치 못한 장애물 출현이나 비정상적인 비행 궤적 이탈 상황에서는 지상 관제 시스템(GCS) 오퍼레이터 또는 원격 조종자(Remote Pilot) 구출을 위한 즉각적인 제어권 개입이 요구될 수 있다. 휴먼-인-더-루프(Human-in-the-loop, HITL) 제어권 회수란, 시스템 주도(System-driven)의 자동 비행 모드에서 인간 주도(Human-driven)의 수동 또는 혼합 제어 모드로 주도권이 전이(Transition)되는 일련의 계층적 예외 처리 기법을 의미한다. 본 절에서는 제어권 회수 시퀀스의 작동 원리와, 오버라이드(Override) 과정에서 필연적으로 발생하는 목표 지령치(Setpoint)의 불연속성을 완화하기 위한 동역학 스무딩(Kinematic Smoothing) 기법에 대하여 상세히 분석한다. (본 분석은 주로 PX4-Autopilot v1.14 및 QGroundControl v4.3.0 이상을 기준으로 한다.)

1. 휴먼-인-더-루프 기반 제어권 회수 아키텍처

PX4-Autopilot의 커맨더(Commander) 모듈은 자동 비행(Auto Mode) 작동 중, 관제사 또는 조종사로부터 유효한 수동 조작 입력이 감지될 때 지정된 로직에 따라 즉각적으로 모드 전환(Mode Transition)을 수행한다. 조종기(RC/Joystick)의 스틱 변위가 사전에 정의된 임계값(Deadband Threshold)을 초과할 경우, 기존에 실행 중이던 복귀 비행(Flight Task Return) 상태 머신을 선점(Preemption)하여 위치 제어(Position Control) 모드 등으로 롤백(Rollback)하는 과정이다.

1.1 스틱 오버라이드(Stick Override) 검출 메커니즘

수동 비행 체계에서 스틱 오버라이드 여부를 확인하기 위해, 시스템은 rc_channels 또는 manual_control_setpoint uORB 토픽을 지속적으로 모니터링한다.

sequenceDiagram
    participant Operator as GCS / RC Operator
    participant Commander as Commander Module
    participant RTL as Flight Task Return
    participant Position as Flight Task Position
    participant MPC as Multicopter Position Control

    Operator->>Commander: Stick Deflection (Manual Setpoint > Threshold)
    activate Commander
    Commander->>Commander: Evaluate Override Logic
    Note right of Commander: COM_RC_OVERRIDE Parameter Check
    Commander->>RTL: Preempt RTL State
    deactivate RTL
    Commander->>Position: Switch Mode to Position / Altitude
    activate Position
    Position->>MPC: Update Trajectory Setpoint
    deactivate Commander

커맨더 모듈 내 제어권 회수 판별 로직의 핵심 파라미터는 COM_RC_OVERRIDE이다. 해당 파라미터를 활성화할 경우, 드론이 자동 모드 비행 중에 스틱이 치우침(Deflection)을 감지하면 즉각적으로 Position 모드 등의 수동 베이스 모드로 변경하여 조종자에게 기동성을 반환한다. ArduPilot의 경우 비슷한 기능을 RTL_CLIMB_MIN 등과 결합하여 스틱 오버라이드를 수행하지만, 시스템 철학의 차이 상 PX4는 비행 작업(Flight Task) 아키텍처를 스위칭하는 보다 명시적인 상태 머신 전이를 사용한다.

2. 동역학 스무딩(Kinematic Smoothing)의 필요성

자동 비행과 수동 비행 간의 전환 시나리오에서는 필연적으로 지령치 오차(Setpoint Transient) 가 발생하게 된다. 예를 들어, 자동 복귀 모드에서 드론이 10m/s로 지정된 웨이포인트를 향해 하강 비행을 하고 있을 때, 갑작스럽게 제어권이 수동으로 회수된다고 가정해보자. 조종자의 스틱이 중앙(Neutral)에 위치해 있다면 속도 명령은 즉시 0m/s 수렴을 요구하게 된다. 제어기의 목표 지령치가 스텝 형태(Step Function)로 변함에 따라 가속도 한계를 넘는 과도한 토크(Torque)가 유발되어 기체 자세의 순간적인 흔들림(Jerk)이나 심각한 고도 강하를 초래할 수 있다.

이러한 기구학적 임펄스 영향을 방지하기 위해 PX4-Autopilot의 궤적 생성기(Trajectory Generator)는 동역학 스무딩(Kinematic Smoothing) 필털르 적용한다.

2.1 저크 제한(Jerk-limited) 궤적 생성

PX4에서는 위치 및 속도 제어에서 지령치 변화율을 다항식 형태로 제약하여 부드러운 전이를 돕는다. FlightTaskManualPosition 및 관련 제어 클래스에서는 목표 속도(v_{sp})와 가속도(a_{sp})를 생성할 때 다음 원칙을 준수한다.

  • 가속도 한계(Acceleration Constraint): 기체가 도달할 수 있는 최대 가속도 제한 (MPC_ACC_HOR_MAX 등)
  • 저크 한계(Jerk Constraint): 기체의 가속도가 변화하는 비율(Jerk)의 제한 (MPC_JERK_MAX 등)

전환 순간 시점 t_0에서 자동 모드의 속도 벡터 V_{auto}(t_0)를 초기 조건으로 채택하며, 전환 후 사용자의 수동 입력 벡터 V_{manual}(t)를 최종 지향점으로 삼아 저크 제한 궤적(Jerk-Limited Trajectory)을 형성한다. 이를 통해 PID 제어단의 에러가 폭증하는 것을 차단하고 안정적인 자세 보정을 유도한다.

2.2 ArduPilot 대비 제어 스무딩 철학의 차이

PX4-Autopilot은 상태 궤적(State Trajectory)을 전방위 모드에서 스무딩하는 데 초점을 둔 설계를 채택하고 있다. 이는 FlightTask 및 SlewRate 컴포넌트들을 사용하여 독립 모듈이 일관되게 동역학 제약 조건 궤적을 뿜어내도록 하는 구조(Architecture)로 구성되어 있다.
반면, ArduPilot의 경우 L1 Controller 기반의 방식이나 서브 모드 내에서 속도 지령치에 대한 저역 통과 필터(Low Pass Filter) 또는 S-Curve 가속도 프로파일을 개별 제어기에 이식하는 방향성을 주로 취하고 있다. PX4 방식은 모터 출력 제어 전 단계에서 일관된 궤적 생성기가 전념하므로 상위(Higher-Level) 소프트웨어 전환에서 더 유연한 장점을 지닌다.

3. 오버라이드 동작 시 시스템 동작 시퀀스

아래는 조종사의 관제 개입에 의해 자동 실행 모드에서 수동 오버라이드로 동작이 전이되는 세부 절차이다.

  1. 상태 천이 조건 발동: 조종자가 GCS의 가상 조이스틱을 움직이거나 MAVLink MANUAL_CONTROL 패킷을 전송.
  2. uORB 메시지 업데이트: manual_control_setpoint 토픽이 발행되며, 이 메시지의 타임스탬프와 값이 최신 상태로 갱신.
  3. 오버라이드 마스킹 파악: Commander는 현재 비행 모드(Nav State)가 RTL 등 Auto 모드일 때 스틱 값이 오버라이드 데드존(Parameter: COM_RC_STICK_OV)을 넘어선 상태인지 판별.
  4. 모드 해제 및 롤백: 로직이 참(True)이면 즉시 기존 작업(return task)을 삭제하고 Position 제어 작업(Position Task)을 할당한다. GCS상에는 “Taking over manual control” 메시지와 함께 모드 변경 ACK 전송.
  5. 초기화 및 스무딩: Position 제어기는 전환된 직후 앞선 추정 상태(Estimated State)를 현재의 지령치 초기값으로 설정하여 유한 스텝 변화를 최소화하고 매끄러운 S-Curve 과도 응답 특성을 보이도록 궤적을 계획한다.

이러한 체계적인 제어권 회수 절차 및 스무딩 과정은 휴먼 관심사(Human Attention)가 집중되는 위급 상황 시에 기동 충격을 방지함으로써, 복귀 실패라는 사고 시나리오로부터 치명적인 추락을 효과적으로 예방하는 시스템 신뢰성(Reliability)의 핵심이다.
