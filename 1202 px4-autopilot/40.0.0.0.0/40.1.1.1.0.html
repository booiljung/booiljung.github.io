40.1.1.1. 레일리 페이딩(Rayleigh Fading) 환경의 비결정론적 패킷 유실률(PLR) 추정

PX4-Autopilot과 QGroundControl 간의 MAVLink 통신은 텔레메트리 라디오나 컴패니언 컴퓨터(Companion Computer)의 Wi-Fi, LTE 모듈을 통해 이루어진다. 앞선 논의에서 다루었듯이 무인기가 도심지의 빌딩 숲이나 험준한 산악 지형에서 비행할 때, 뚜렷한 가시거리(LOS, Line-of-Sight)가 확보되지 않는 환경에 처하게 되며 통신 채널은 레일리 페이딩(Rayleigh Fading) 채널로 근사화된다. 이 경우 수신단(예: GCS의 텔레메트리 모듈)에 도달하는 다중 경로 신호들의 간섭으로 인해 신호 대 잡음비(SNR, Signal-to-Noise Ratio)가 임계점 아래로 급격히 떨어지는 딥 페이드(Deep Fade) 현상이 발생한다. 본 절에서는 이러한 레일리 페이딩 채널에서 MAVLink 패킷 유실률(PLR, Packet Loss Rate)을 통계적, 비결정론적(Nondeterministic) 관점에서 추정하는 방법을 다룬다.

1. MAVLink 패킷 유실률(PLR)의 통계적 정의

디지털 통신에서 비트 에러율(BER, Bit Error Rate)은 SNR의 함수로 결정된다. MAVLink 패킷 하나는 헤더, 페이로드(명령 데이터), 그리고 CRC(Cyclic Redundancy Check)를 포함하여 일반적으로 수십에서 수백 바이트 크기를 갖는다. 패킷 안의 비트 중 단 하나라도 오류가 발생할 경우, CRC 검증을 통과하지 못해 해당 패킷은 전체가 폐기(Drop)된다.

따라서 단일 비트 처리 에러율을 P_b라고 하고, MAVLink 패킷의 길이를 L (단위: bit)이라고 할 때, 비트 에러가 서로 독립적이라는 가정하에 패킷 유실률(PLR)은 아래와 같이 계산된다.

PLR = 1 - (1 - P_b)^L

그러나 레일리 페이딩 환경에서는 채널의 이득(Gain)이 확률적으로 끊임없이 변동하므로, 순간적인 비트 에러율 P_b 역시 상수가 아니라 확률 변수가 된다. GCS 제어 명령 설계자는 비결정론적으로 요동치는 이 PLR에 대응하기 위해 타임아웃(Timeout) 및 재전송(Retry) 횟수를 산정해야 한다.

2. 레일리 페이딩 환경에서의 평균 비트 에러율(Average BER) 유도

단순화를 위해 MAVLink 데이터 링크가 널리 쓰이는 FSK(Frequency Shift Keying) 혹은 BPSK(Binary Phase Shift Keying) 모듈레이션을 사용한다고 가정한다.

AWGN(Additive White Gaussian Noise) 채널에서 순간 SNR \gamma 에 대한 순간 비트 에러율 P_b(\gamma)가 특정한 비선형 감소 함수(예: Q(\sqrt{2\gamma}) 등)를 그린다고 할 때, 레일리 페이딩 채널을 통과하는 전체 평균 비트 에러율 \bar{P}_b는 다음과 같이 순간 SNR \gamma 의 확률 밀도 함수(PDF)인 p(\gamma) 를 이용하여 기대값(Expectation)으로 추정된다.

\bar{P}_b = \int_{0}^{\infty} P_b(\gamma) \cdot p(\gamma) d\gamma

레일리 페이딩 채널에서 순간 SNR \gamma 는 지수 분포(Exponential Distribution)를 따른다는 수학적 특성이 있다. 이 때 평균 SNR을 \bar{\gamma} 라 하면 PDF p(\gamma) 는 다음과 같다.

p(\gamma) = \frac{1}{\bar{\gamma}} \exp\left(-\frac{\gamma}{\bar{\gamma}}\right), \quad \gamma \geq 0

이를 기대값 식에 대입하여 적분하면 변조 방식에 따른 레일리 환경의 평균 에러율의 닫힌 형태(Closed-form) 해를 얻을 수 있다. 예를 들어 일관성 없는(Non-coherent) FSK의 경우, 페이딩 환경 하에서의 평균 비트 에러율 \bar{P}_b는 \frac{1}{2 + \bar{\gamma}} 로 수렴하게 되며, 이는 AWGN의 지수적 감소(\sim e^{-\gamma / 2})와 달리 페이딩 채널에서는 에러율이 SNR의 역수로 완만히 감소한다는 치명적인 사실을 시사한다.

즉, 송신 전력을 조금 높인다고 해서 깊은 페이딩 계곡에 빠진 패킷을 구출할 수 없음을 증명하는 것이다.

3. 타임아웃 튜닝을 위한 비결정론적 PLR 계산과 시사점

앞서 구한 \bar{P}_b를 PLR 수식에 대입하면, GCS 관점에서 체감하는 MAVLink 명령어 연속 유실 확률을 예측할 수 있다.

    F --> G[Retry 로직 가동]

3.1 버스트 에러(Burst Error)와 패킷 손실 상관성

실제 환경에서는 비트 에러가 독립적으로 발생하기보다는 페이딩 주기에 따라 집중적으로 나타나는 버스트 에러(Burst Error) 특성을 지닌다. 기체가 특정한 이동 속도로 비행할 때 발생하는 도플러 주파수(Doppler Frequency)는 페이딩의 지속 시간에 지대한 영향을 미친다. 이 때문에 단순히 \bar{P}_b로 계산한 PLR보다 실제 체감하는 명령 지연과 유실이 덩어리(Burst) 단위로 발생한다.

3.2 PX4와 GCS 커맨드 큐에 대한 공학적 적용

이러한 비결정론적인 PLR의 특성으로 인해 GCS 개발 시 타임아웃과 횟수 한계(Retry Limit) 설계에 근본적인 제약이 생긴다.
만약 일시적인 버스트 손실 구간이 QGroundControl에 설정된 Response Timeout * Max Retries 시간보다 길다면 연결 상실(Link Loss) 선언 트리거가 당겨진다.

따라서 PX4-Autopilot를 포함한 제어 시스템 설계 시:

  1. 명령어 파편화 조절: MAVLink 페이로드 길이 L을 적절하게 유지하여 단일 비트 에러가 훼손하는 정보를 최소화한다.
  2. 동적 파라미터 적용 제안: GCS는 MAVLink RADIO_STATUS 등의 메시지에서 RSSI(Received Signal Strength Indicator)를 모니터링하여 평균 \bar{\gamma} 의 추세를 확인하고, SNR이 레일리 특성의 척박한 상태에 빠졌다면 타임아웃 대기 틱(Tick)을 소프트웨어적으로 연장시켜 재전송 패킷의 혼잡(Congestion)을 피하는 유연함을 갖추는 것이 강력히 권장된다.

결론적으로 레일리 페이딩 하의 PLR 추정은 통신 에러가 하드웨어 한계를 넘어서는 비결정론적 영역에 존재함을 밝혀주며, PX4를 뒷받침하는 관제 소프트웨어 계층(Application Layer)에서의 정교한 Timeout-Retry 메커니즘을 당위적으로 요구한다.
