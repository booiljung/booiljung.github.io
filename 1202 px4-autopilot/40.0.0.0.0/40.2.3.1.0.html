40.2.3.1. MAV_RESULT 열거형 변수의 마르코프 결정 과정(Markov Decision Process) 기반 전이도

1. 개요(Overview)

지상 관제 시스템(GCS)에서 무인항공기 기체(PX4-Autopilot)로 전송된 특정 COMMAND_LONG 명령어는 항상 즉각적이고 확정적인 성공을 보장하지 않는다. 기체의 센서 교정 상태, 현재 GPS 위성 수신 품질(DOP), 배터리 방전율 등은 명령 수락 여부에 간섭하는 비결정론적 유동 변수이다.
MAVLink 프로토콜은 이러한 명령에 대한 처리 결과를 COMMAND_ACK 패킷 내의 MAV_RESULT 열거형 변수를 통해 GCS로 피드백한다.
본 절에서는 관제 시스템이 MAV_RESULT를 단순한 리턴 코드로 바라보는 대신, 불확실성이 내재된 비행 환경에서의 행동 지침을 모델링하기 위해 수학적인 **마르코프 결정 과정(Markov Decision Process, MDP)**으로 해석하고, QGroundControl(v4.3)과 PX4(v1.14.0) 간의 상태 전이 함수로 매핑하는 구조를 깊이 있게 분석한다.

2. MAV_RESULT 열거형의 핵심 속성과 역할

MAVLink 표준 규격에 명시된 MAV_RESULT는 단순히 성공/실패의 이분법 구조를 뛰어넘는 강력한 문맥적 의미를 지니고 있다.

  • MAV_RESULT_ACCEPTED (0): 명령이 유효하며 시스템에 의해 정상 수락되어 실행 절차에 돌입함.
  • MAV_RESULT_TEMPORARILY_REJECTED (1): 명령의 포맷 자체는 정당하나, 기체의 현재 비행 모드나 물리적 한계치로 인해 지금 당장은 실행 불가함. (예: 공중 비행 상태에서 센서 캘리브레이션 명령 수신).
  • MAV_RESULT_DENIED (2): 명령이 유효하며 기체 상태도 해당 명령 처리에 적합하나 논리적으로 무력화되거나 보안상의 이유로 수락 거절됨. (예: 시동 전제 조건 실패로 인한 ARM 거부).
  • MAV_RESULT_UNSUPPORTED (3): 알 수 없거나 기체 펌웨어가 지원하지 않는 명령.
  • MAV_RESULT_FAILED (4): 명령 실행 도중 치명적인 오류 발생.
  • MAV_RESULT_IN_PROGRESS (5): 명령이 장기적인 시간을 요구하여 수락한 뒤 아직 수행 중인 상태. (진행 상태 % 포함 피드백).

3. 마르코프 결정 과정(MDP) 기반 제어 상태 모델링

자율 에이전트 통제 아키텍처에서 GCS 액션에 따른 결과 편차가 크기 때문에, 기체 제어 모듈을 5 튜플 MDP = (S, A, P, R, \gamma) 위상으로 맵핑한다.

  1. 상태(State, S): 기체의 현재 FSM 위상 정보(Standby, Armed, Mission 등).
  2. 행동(Action, A): GCS 단에서 발생하는 제어 트리거(이륙 커맨드, 모드 변경 커맨드 a_t).
  3. 전이 확률 모델기(Transition Model, P): P(s_{t+1} \vert s_t, a_t) 는 명령 a_t 인가 후 비결정론적으로 기체가 상태 s_{t+1}로 넘어갈 확률이다.
  4. 보상 피드백(Reward, R): 기체로부터 수신되는 MAV_RESULT가 곧 액션에 대한 직접적인 모델 보상치이다.
  • R \leftarrow ACCEPTED: 양의 보상 극대화, 해당 행동 종료.
  • R \leftarrow TEMPORARILY_REJECTED: 타임아웃 대기 후 재전송 시퀀스 재진입 유도.
  1. 감가율(Discount Factor, \gamma): 시간 지연에 따른 명령어의 효용 감소 폭.

3.1 GCS 내부 상태 결정을 위한 정책(Policy) 트리거

QGroundControl(v4.3)의 내부 코어를 살펴보면 Vehicle::sendMavCommandWithHandler C++ 소스 코드 영역에서 응답 메커니즘을 다룬다.
이 메서드는 반환받은 MAV_RESULT 응답값을 기준으로 타임아웃 타이머 스레드를 리셋할지, 아니면 사용자 UI 레이어 화면에 팝업 메시지를 표출하고 명령 재시도(Retry) 로직을 파기(Abort)할지 결정짓는 GCS 측 핵심 정책(Policy) 평가 구간이다. Ardupilot 전용 GCS인 Mission Planner 또한 동일한 IN_PROGRESS 피드백을 핸들링하지만 재시도 카운트 파라미터나 상태 위상 추정치 업데이트 간격 세분화 로직에서 차이를 보인다.

4. PX4 소스 코드 내 MAV_RESULT 생성기(Generator) 분석

위상 공간에서 MDP 피드백을 생성하는 원천지(Source)는 PX4 내 mavlink_receiver.cpp 파일의 Command Long 수신 블록이다.
명령이 들어오면 이는 vehicle_command uORB 토픽으로 변환되어 로컬 파이프라인을 타고 Commander 인스턴스에 도달한다. Commander는 현재 기체의 센서 및 제어기 헬스 상태 벡터를 바탕으로 TransitionResult 결괏값을 연산하며, 이 반환형은 결과적으로 시스템의 응답 코드 매퍼를 거쳐 다음과 같은 분기 트리를 탄다.

// PX4-Autopilot Source Logic 맵핑 의사코드 예시
if (result == TransitionResult::ACCEPTED) {
    send_command_ack(command, MAV_RESULT_ACCEPTED);
} else if (result == TransitionResult::DENIED_TIMEOUT) {
    send_command_ack(command, MAV_RESULT_TEMPORARILY_REJECTED);
} else {
    // 보안, 하드웨어 불만족, Pre-flight 체크 실패 등
    send_command_ack(command, MAV_RESULT_DENIED);
}

6. 결론 및 실무 제언

독자적인 GCS 플랫폼이나 자동 비행 에이전트 스크립트(MAVSDK, ROS2)를 설계하는 엔지니어는 단순한 MAVLink 송수신 튜토리얼 수준에 머물러서는 곤란하다.
GCS의 비동기 제어 시스템 아키텍처 내에서 MAV_RESULT 피드백은 기체의 상태(State)와 환경(Environment)의 변동성으로 인한 추계학적 결과물이라는 점을 명확히 인식해야 한다.
따라서, GCS의 코어를 작성할 때는 마르코프 결정 과정(MDP)의 피드백 보상 원리를 참고하여, 어떠한 오류 반환값이 돌아오더라도 그 즉시 응용 애플리케이션의 동작(Action) 스레드가 고착(Deadlock)되거나 UI가 먹통이 되지 않도록 유한 상태 기반의 촘촘한 예비 복구 정책(Fallback Policy)을 마련해야만 산업 현장의 돌발 변수를 견뎌내는 신뢰성 높은 관제 소프트웨어로 거듭날 수 있다.
