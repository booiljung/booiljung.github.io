40.3.3. 다목적 최적화(Multi-Objective Optimization) 기반 휴리스틱 타임아웃 가중치 할당

지상 관제 시스템(GCS)과 PX4-Autopilot 간의 데이터 링크는 단일 목적이 아닌, 종종 상충(Trade-off)하는 여러 가지 요구사항을 동시에 만족시켜야 한다. 예를 들어, 제어 지연(Control Latency) 최소화와 대역폭(Bandwidth) 낭비 최소화, 이 두 가지는 서로 상충되는 목표다. 텔레메트리 타임아웃을 짧게 가져가면 지연을 줄일 수 있지만 과도한 재전송으로 인해 대역폭이 소진되고, 반대로 타임아웃을 길게 유지하면 무선 채널 점유율은 낮출 수 있으나 명령어 도달 지연이 증가하여 PX4 제어기의 응답성이 무너진다.

이러한 모순적인 상황에서 단일 해답을 시계열 필터로만 찾는 것에는 한계가 존재한다. 본 절에서는 다목적 최적화(Multi-Objective Optimization) 프레임워크와 휴리스틱(Heuristic) 알고리즘을 결합하여, QGroundControl(QGC) 내부의 MAVLink 재전송 타이머에 어떻게 동적 가중치(Weight)를 할당할 수 있는지에 대한 공학적 기법을 기술한다.

1. 다목적 최적화(MOO) 문제의 정의

다목적 최적화란 주어진 제약 조건 하에서 두 개 이상의 목적 함수(Objective Function)를 동시에 최적화하는 과정이다. GCS 타임아웃 조율 메커니즘에서는 다음의 주요 목적 함수 세 가지를 수립할 수 있다.

  1. F_1(t_{rto}): 총 전송 지연시간(Total Transmission Delay) 최소화
    MAVLink COMMAND_LONG이 발송된 순간부터 드론의 COMMAND_ACK 수신까지 걸리는 시간(Delay)을 최소화한다. 타임아웃 t_{rto}가 작을수록 지연 복구 속도가 빠르다.
  2. F_2(t_{rto}): 채널 점유율(Network Utilization) 및 패킷 재전송 비용 최소화
    가짜 타임아웃(Spurious Timeout)으로 발생하는 불필요한 패킷 중복 오버헤드를 줄인다. 즉 t_{rto}가 무선 채널의 최대 Jitter보다 길어야 달성할 수 있다.
  3. F_3(t_{rto}): 비행체 제어 안정성(Vehicle Control Stability) 최대화
    센서 정보(GLOBAL_POSITION_INT 등)의 수신 일관성을 유지하고, 잦은 오프보드 제어(Offboard Control) 단절(Fail-safe) 모드 진입 빈도를 낮춘다.

이 세 가지 함수는 서로 트레이드오프 관계를 형성하며, 절대적으로 우월한 단일 해(Unique Solution)가 존재할 수 없다. 이를 수학적으로 표현하면 파레토 최적(Pareto Optimality) 공간 상에서의 절충안(Trade-off Solution)을 찾아야 함을 의미한다.

2. 파레토 프론트(Pareto Front)와 휴리스틱 가중치 스케줄링

엄밀한 다목적 최적화 해를 해석적으로 도출하는 방법은 비행 중 발생하는 실시간 오버헤드가 너무 크므로, GCS 애플리케이션 내부에서는 빠르고 실용적인 연산이 가능한 휴리스틱(Heuristic) 결합 스칼라화(Scalarization) 기법을 적용한다.

여러 목적 함수를 단일 적합도 함수(Fitness Function) J(t_{rto})로 묶기 위해 가중합(Weighted Sum) 방식을 사용한다.

J(t_{rto}) = w_1 F_1(t_{rto}) + w_2 F_2(t_{rto}) + w_3 F_3(t_{rto})

핵심은 비행 모드(Flight Mode)나 현재 통신 채널 메트릭(RADIO_STATUS)에 따라 가중치 세트 W = \{w_1, w_2, w_3\} 를 실시간 동적으로 변환하는 것이다. \vert 기호를 포함한 상태 조건별 휴리스틱 평가 지표는 다음과 같이 구성된다.

  • 미션(Mission/Auto) 모드: 경로점 위주로 자동 비행하므로 응답성보다는 데이터 통신의 무결성(대역폭 확보)이 더 중요하다. 가중치는 \vert w_2 \vert > \vert w_1 \vert 로 배정되어 타임아웃 마진을 넓게 가져간다.
  • 오프보드(Offboard) 모드: ROS2 Companion Computer나 QGC의 가상 조이스틱으로 초당 수 회 이상의 Setpoint를 꽂아넣어야 하므로 찰나의 지연도 용납되지 않는다. 가중치는 \vert w_1 \vert \gg \vert w_2 \vert 로 조정되어 즉각적인 재전송 체제로 돌입한다.

3. 휴리스틱 퍼지 추론(Fuzzy Inference) 기반 구조 설계

수학적 가중치 결정을 위해 QGC의 Vehicle 클래스 백엔드 스레드는 규칙 기반의 휴리스틱 매니저(Heuristic Manager)를 운영할 수 있다. 복잡한 신경망보다 예측 가능성이 높은 규칙 기반 퍼지 로직(Fuzzy Logic) 도입이 유리하다.

    D --> E[최적 RTO 산출 및<br/>VehicleCommand 전송 큐 반영]

3.1 퍼지 규칙의 예시

  1. IF (Link_RSSI is LOW) AND (Mode is OFFBOARD)
    • THEN : w_1을 대폭 확대하여 공격적인 재전송 채택. 하지만 채널 포화 스레스홀드 한계에 도달하면 즉각 백오프(Back-off).
  2. IF (Link_Buffer is FULL) AND (Mode is MISSION)
    • THEN : 통신 채널에 심각한 병목 현상이 발생했으므로 w_2를 높여 불필요한 COMMAND_LONG 재전송을 중단하고 수 초 이상의 보수적인 RTO 적용.

3.2 Ardupilot 대비 PX4 특화 설계 이점

Ardupilot 환경의 기본 Mission Planner 로직은 보통 시리얼 포트 객체의 단순 읽기 타임아웃(Read Timeout)에 의존하여 단편적인 재전송 틱(Tick) 스케줄러를 사용하는 경향이 있다.
반면 PX4 펌웨어의 아키텍처 사상과 맞물린 MAVSDK나 QGroundControl 관제 아키텍처 내부에서는 CommandManager가 개별 패킷 단위의 ACK 상태 체인과 비행 모드에 직접 연동될 수 있는 유연한 미들웨어를 지원하므로, 이와 같은 휴리스틱 MOO(다목적 최적화) 가중치 할당 메커니즘을 C++ 계층에 깔끔하게 추상화하여 구현할 수 있다.

4. 요약 및 실무 적용 가이드

통신 에러 환경에서 MAVLink 타임아웃을 하드코딩된 상수 값(예: 500ms 반복)으로 두는 것은 다양한 목적 함수—빠른 반응성과 낮은 대역폭 소모—중 어느 하나도 완벽히 만족시키지 못하는 서브옵티멀(Sub-optimal) 상태를 만든다.

다목적 최적화(Multi-Objective Optimization) 프레임워크는 RTO(Retransmission Timeout)를 결정하기 위해 지연(Delay), 채널 부하(Congestion), 조종 자유도(Controllability)의 상충 관계를 수식화하고 파레토 균형점을 찾도록 돕는다. 그 실행 과정에서는 QGroundControl 내부의 비행 상태(Flight Mode, RADIO_STATUS) 정보를 휴리스틱 가중치로 치환하여 단일 적합도 함수 기반의 타임아웃 산출 방식으로 매핑한다.

결과적으로 관제자는 고속 회피 기동이나 섬세한 VIO(Visual Inertial Odometry) 퓨전 시에는 매우 공격적인 통신을, 장거리 임무 비행 시에는 보수적인 통신을 자동으로 수행하도록 하는 “지능형 텔레메트리 관제 시스템“을 확립할 수 있다.
