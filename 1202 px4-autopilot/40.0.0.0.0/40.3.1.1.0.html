40.3.1.1 지수 가중 이동 평균(EWMA) 평활화 필터의 극점-영점(Pole-Zero) 해석

본 문서에 사용된 기준 버전은 PX4-Autopilot v1.14.0 및 QGroundControl v4.3.0이다.

무인기 지상 관제 시스템(GCS)에서 MAVLink 프로토콜의 적응형 재전송 타임아웃(Adaptive Retransmission Timeout) 기한을 산출할 때, 기저 알고리즘으로 가장 널리 쓰이는 시계열 평활화 필터가 지수 가중 이동 평균(Exponentially Weighted Moving Average, EWMA)이다. 타임아웃 제어 알고리즘의 민감도 튜닝(Tuning)을 단순 직관이나 소모적인 시행착오(Trial-and-Error) 체계에 전적으로 의존하는 대신, 이를 이산 시간(Discrete-Time) 선형 시불변(LTI) 제어 시스템 기법으로 수식화하고 Z-변환(Z-Transform) 상의 극점(Pole)과 영점(Zero) 분포를 정밀 분석하면, 통신 노이즈 환경 파동에 따른 필터의 응답 특성(Response Characteristics)을 수학적으로 완벽히 지배하고 통제할 수 있다.

1. EWMA 필터의 시간 영역 차분 방정식(Difference Equation) 모델

QGroundControl(QGC) 등에서 텔레메트리 모뎀을 통해 TIMESYNC 핑(Ping) 왕복 지연 시간(RTT)을 지속 획득할 때, 측정된 이산 샘플 원본을 x[n], 평활화 처리된 지연 추정 결과값을 y[n]이라 정의하면 기본 1차 차분 방정식 모델은 다음과 같이 전개된다.

y[n] = (1 - \alpha) y[n-1] + \alpha x[n]

위 수식에서 \alpha 패러미터는 0 < \alpha \le 1 구간의 값으로 정의되는 평활화 계수(Smoothing Factor)이다.

  • \alpha 팩터가 1에 근접하는 경우: 필터가 과거 데이터 이력 수치에 크게 연연하지 않고 최신 센서 계측값 x[n] 의 돌출 변화에 거의 즉각적으로 동기 반응(Fast Tracking)한다. 이는 무선 링크가 자주 끊겼다 붙었다를 반복하는 극심한 간섭 도심 구역에서 기민한 타임아웃 변경에 유리하다.
  • \alpha 팩터가 0에 강력히 수렴하는 경우: 매우 긴 시간의 관성 평균값을 고수하며, 억제력(Strong Smoothing)을 발휘하여 한두 번 튀는 순간적인 RTT 스파이크(Spike) 변동 노이즈에 의연하게 버틴다. Ardupilot과 대비하여 장거리 고정익 비행 시 우발적인 전파 교란을 걸러내는 메타 제어 로직에 매우 유리하다.

2. Z-변환(Z-Transform) 및 극점-영점(Pole-Zero) 궤적 해석

EWMA 필터 루프의 구조적 동적 특성(Structural Dynamics)을 주파수 영역에서 분간하기 위해 위 차분 방정식의 양변 집합을 라플라스 이산 매핑인 Z-변환하여 시스템의 고유 전달 함수(Transfer Function) H(z)를 다음과 같이 도출할 수 있다.

Y(z) = (1 - \alpha) z^{-1} Y(z) + \alpha X(z)
H(z) = \frac{Y(z)}{X(z)} = \frac{\alpha}{1 - (1 - \alpha) z^{-1}} = \frac{\alpha z}{z - (1 - \alpha)}

해당 영점이 복소평면상 정확히 정중앙 원점에 뿌리내리고 있으므로, 이 필터의 개입은 입력 타이밍 신호의 위상 딜레이(Phase Delay)에 본질적인 지연 지표(Characteristic Retardation)를 필연적으로 부여하지만, 반면 제어 시스템 메인 루프 안정성에는 하등 파괴적인 직교 간섭 요인으로 작용하지는 않는다.

2.2 극점(Pole)의 바운더리 검토 및 제어 안정성(Stability Margin)

분모 다항식 배열의 해(Characteristic Root)를 구하면 시스템 메인 극점 궤적은 다음과 같이 정의된다.

z_{pole} = 1 - \alpha

여기서 도출된 극점-영점 배치도(Pole-Zero Plot)를 MAVLink 재전송 타임아웃 제어 공학 스펙트럼 관점에서 해석하면 다음과 같은 실무적 핵심 인사이트(Insight)를 거머쥘 수 있다.

  1. 절대 안정도 판별(BIBO Stability Limit): 폐루프 제어 시스템의 극점은 항상 1을 한계 반경으로 갖는 단위원(Unit Circle)의 내부에 온전히 위치해야 궤적이 발산 파괴되지 않는다. \alpha의 한계 조건이 0 < \alpha \le 1 이므로, 도출된 극점 z_{pole}은 복소평면 안의 실수축 궤적 (0, \ 1) 내에만 놓이게 된다. 따라서 이 필터식은 수학적으로 어떠한 지연 스파이크를 때리더라도 구조적으로 절대안정(Bounded-Input Bounded-Output Stable)함을 보장받는다.
  2. 동적 과도 응답(Transient Response)의 수렴 속도성: 극점의 위치 1-\alpha가 원점(0) 공간에 육박하여 가까울수록(즉, \alpha값이 지표 1을 향해 클수록) 필터의 에너지 감쇠 시간 소모치가 극적으로 짧아져, 백엔드 시스템이 텔레메트리 지연율 악화 지표 변화에 예리하게 대처하게 된다. 이와 정반대로 극점의 위치가 단위원 외곽 바운더리 경계인 1점의 벽에 바싹 달라붙으면(즉, \alpha \to 0 극한 도달 시), 아주 예전에 누적된 RTT 지표의 “관성 찌꺼기“가 너무 질기게 남아서 현 텔레메트리 환경의 급격한 악화 상황 타개 국면을 재빨리 반영하지 못하고 무의미하게 재전송 패킷 낭비(Timeout Congestion Error)를 초래하게 된다.

2. QGC 실무 백엔드 적용: 대역폭(Bandwidth) 제한 및 필터 컷오프 특성 설계

주파수 주파 영역 스펙트럼에서 EWMA 필터는 본질적으로 1단계 저역 통과 필터(1st-order Low-Pass Filter, LPF)로 동작한다. 통과를 차단하는 임계 차단 주파수(Cut-off Frequency) \omega_c 는 샘플링 획득 주기 T_s 간격에 대하여 수학적으로 근사적으로 이와 같은 밀접 관계를 가진다.

\omega_c \approx \frac{\alpha}{(1-\alpha) T_s}

데스크톱 관제 소프트웨어 툴체인(QGC 등)이나 ROS2 MAVROS 인터페이스를 통해 복수의 군집 비행(Swarm Control Agent)을 수행할 때는 시시각각 공중망에 난입하는 고주파 통신 잡음(Wi-Fi 채널 컨제스천, 지향성 안테나 방위에 따른 노이즈 등)을 필터링해 내야 하므로 이러한 LPF 스탠스 유지가 필수적이다. 만약 PX4 내부 파라미터 트리 구조체 데이터베이스 내부나 GCS 소프트웨어 고급 설정(Advanced Settings) UI 창 내부에서 사용자가 \alpha 필터 제어값을 슬라이더 게이지 따위로 능동 조작할 수 있게 파라미터를 엑스포트(Export)하여 노출시킨다면, **“극점(Pole)을 원점에서 점진적으로 멀어지게 세팅할 수 있는 옵션(즉, \alpha 슬라이스 낮춤)”**을 지원함으로써 비행 시퀀스 시스템을 훨씬 강건하고 다치지 않게 방어할 수 있다.

4. 커스텀 지상 관제망(Custom GCS) 구축 설계 시 소프트웨어 아키텍처 구현 고려사항

  • 적응형 상황 인지 기반 \alpha 튜닝 로직 차용: PX4 비행체와 텔레메트리 통신하는 커스텀 GCS C++ 어플리케이션을 새로이 작성할 때, 평활화 계수 alpha 상수 인자 값을 한 가지로 고착시켜 하드코딩(Hard-coding)하지 마라. GCS 시스템 반경과 비행체 사이의 이격 거리가 상대적으로 가까워 RSSI(수신 전파 신호 강도, Radio Signal Strength Indicator) 지표가 아주 뛰어날 때는 고정된 \alpha=0.33 수준 정도로 크게 세팅하여 지연에 기민하게 반응하고, 반대로 거리가 수 킬로미터(km) 바깥으로 벌어져 무선 채널 노이즈 저변 레벨이 높아질 때 \alpha=0.03 근처까지 대폭 스스로 강등시키는 동적 극점 궤적 강제 조정(Dynamic Pole Placement Control) 기술 기법을 시스템 로직 분기에 적극 적용하면 실로 이상적이고 파괴되지 않는 최고 무결성의 관제 통과 성능을 담보할 수 있다.
  • 부동소수점 오버헤드 부하 절감을 위한 최적화: 본 이산 극점-영점 필터 수식을 C/C++로 구현하여 마이크로컨트롤러(NuttX RTOS) 임베디드 칩(예: STM32 레벨 프로세서군) 사이드 내부의 uORB 워크 큐에 집적 이식한다면, 한 틱 한 틱 쏟아지는 부동소수점 마이크로 연산을 과감하게 회피하고 ALU 코어를 보존하기 위해 하드웨어 친화형 고정소수점 비트 단위 논리 시프트 연산(\alpha = \frac{1}{2^k}) 트리로 근사 수치화하여 치환하는 백엔드 컴파일러 스텝 치트키 최적화 기법을 결코 망각하지 말라.
