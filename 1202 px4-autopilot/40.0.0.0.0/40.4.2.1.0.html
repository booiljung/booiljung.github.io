40.4.2.1 절삭형 백오프(Truncated Backoff) 함수의 점근 표기법(Asymptotic Notation) 기반 상한 제어

무인기 관제 네트워크에서 이진 지수 백오프(Binary Exponential Backoff, BEB) 알고리즘은 혼잡을 피하는 데 탁월하지만, 재전송 횟수 n이 증가함에 따라 대기 시간 윈도우가 기하급수적으로 무한정 넓어지는 기근(Starvation) 문제를 내포하고 있다. 특히 PX4-Autopilot과 QGroundControl 간의 MAVLink 제어 루프는 실시간성(Real-time)이 생명이므로,명령이 수십 초 혹은 수 분 동안 대기 상태에 빠지는 것을 결코 허용해서는 안 된다. 따라서 대기 시간 확장에 절대적인 한계(Ceiling)를 두는 절삭형 이진 지수 백오프(Truncated Binary Exponential Backoff) 메커니즘을 적용하고, 이를 점근 표기법(Asymptotic Notation)을 통해 제어 루프의 시간 복잡도 상한(Upper Bound)을 고정(Bounding)하는 설계가 필수적이다.

1. 절삭형 백오프(Truncated Backoff) 함수의 수학적 모델링

절삭형 백오프 함수는 연속된 충돌이나 타임아웃이 일정 임계치(Threshold) N_{max}에 도달하면, 그 이후부터는 백오프 윈도우 크기를 더 이상 늘리지 않고 최댓값으로 고정시킨다.

재전송 횟수를 n이라 할 때, 백오프 윈도우의 최대 슬롯 배수 상한을 CW_{max}(Contention Window Max)라고 하면, 재전송 딜레이를 결정하기 위해 선택되는 난수 k의 범위는 다음과 같이 점근적(Asymptotic)으로 절삭(Truncated)된다.

k \sim U(0, \min(2^n - 1, CW_{max}))
T_{backoff}(n) = k \times T_{slot}

이를 시간 복잡도를 나타내는 점근 표기법(Big-\mathcal{O} Notation)으로 해석해 보자.

  • 초기 지수적 증가 구간 (n \le \log_2(CW_{max} + 1)):
    최대 대기 시간 T_{max}는 \mathcal{O}(2^n)의 복잡도로 기하급수적으로 확장된다.
  • 절삭 이후 구간 (n > \log_2(CW_{max} + 1)):
    최대 대기 시간은 \mathcal{O}(1) (상수 시간 복잡도)로 Bounding 되어, CW_{max} \times T_{slot}을 초과하지 않는다.

이러한 점근적 상한(Upper Bounds constraint) 적용은 네트워크의 처리량(Throughput)이 0에 수렴하는 현상을 막고, 제한 시간 내에 GCS가 최종적으로 연결 단절(Data Link Loss)에 따른 Failsafe 모드 전환을 결단할 수 있는 타이밍 기준선(Baseline)을 제공한다.

2. PX4 아키텍처와 QGroundControl(QGC)에서의 적용 차이

2.1 QGroundControl (C++) 기반 타임아웃 루프 상한

QGroundControl 내 핵심 통신 담당 모듈들(예: VehicleCommandManager)은 COMMAND_LONG 응답 대기 시 무한정 연결을 잡고 있지 않다. BEB 알고리즘을 소스 코드 단에 확장 구현할 때, 최댓값 매크로 상수(예: #define MAX_BACKOFF_MSEC 2000)를 설정하여 타이머(QTimer::setInterval()) 로직에 주입한다. 만약 재전송 횟수 변수 _retryCount가 상승하여 연산된 대기 인터벌이 MAX_BACKOFF_MSEC를 초과하면 곧바로 절삭(qMin(computed_backoff, MAX_BACKOFF_MSEC)) 연산으로 덮어쓴다. 이후 상한 고정 상태에서의 재전송마저 허용 횟수를 초과해 실패하면, 더 이상 기다리지 않고 즉각 예외 처리(Failsafe 또는 Command NACK) 이벤트로 상향 전파(Bubble Up)시킨다.

2.2 Ardupilot 대비 관제 한계(Limit) 처리 철학의 관점

Ardupilot 생태계(Mission Planner 등)는 종종 드론의 응답성에 대해 다소 여유로운 타임아웃 마진을 가져가는 편이지만, PX4와 QGC는 더 민첩하고 보수적인 통신 단절 감지(Data Link Loss Detection) 철학을 따른다. PX4 시스템은 실시간 ROS2 로컬 경로 계획(Local Path Planning)이나 uXRCE-DDS 브릿지를 통한 고빈도(High-frequency) Offboard 제어 명령이 주를 이룰 수 있기 때문이다. 따라서 타임아웃의 점근적 상한 \mathcal{O}(1) 도달 시간을 짧게(Short Truncation) 설정하여 시스템상의 교착 상태(Deadlock)를 조기 차단하는 것이 적합하다.

3. 절삭형 백오프와 임계치 도달의 상태 다이어그램 (Mermaid)

다음은 GCS가 절삭형 백오프 기반으로 재전송 간격을 조정하다가, 점근적 상한치(CW_{max}) 곡선에 닿은 후 일정한 최대 딜레이로 재전송을 시도하고 결국 최종 방기(Drop) 상태에 이르는 알고리즘의 제어 흐름도이다.

stateDiagram-v2
    [*] --> Init
    Init --> Backoff_O_2n : Retry (n <= 임계치)
    
    Backoff_O_2n --> Backoff_O_2n : Timeout 추가 발생<br>(n += 1, 지수적 인터벌 2^n)
    
    Backoff_O_2n --> Truncated_O_1 : n > log2(CW_max) 도달
    
    Truncated_O_1 --> Truncated_O_1 : Timeout 연속 발생<br>(Delay를 O(1) 상한값 고정)
    
    Truncated_O_1 --> Failsafe : Max Retries 총합 초과<br>(Data Link Loss 판정)
    Backoff_O_2n --> Success : 응답 수신 (ACK) 완료
    Truncated_O_1 --> Success : 응답 수신 (ACK) 완료
    
    Success --> [*]
    Failsafe --> [*]

위의 다이어그램에서 보듯, Backoff_O_2n 전이 단계에서는 매회 지수적으로 시간이 증가하지만, Truncated_O_1 상태 노드로 진입한 후에는 딜레이 증가세가 완전히 평탄해진다. 이는 무한루프 대기를 방지하는 데드라인 파라미터(Deadline Parameter)의 전형적 거동이다.

4. 결론 및 공학적 시사점

동적 통신망 환경에서 기하급수적인 무한한 대기는 곧 무인기의 완전한 제어 상실(Loss of Control)과 직결된다. 이진 지수 백오프 알고리즘에 점근적 상한(Asymptotic Bounding)을 적용한 절삭형 백오프 체제는, 일시적인 RF 간섭 트래픽을 회피하면서도 GCS 시스템을 영원한 대기큐의 수렁에서 건져내는 핵심적인 제어 공학적 호흡 장치이다.
Autopilot 및 QGC 연동 시스템을 재작성하거나 확장 설계할 때, 실시간 제어명령(Offboard)과 일반 웨이포인트(Mission) 등 각 트래픽 고유의 성격에 따라 Truncation의 \mathcal{O}(1) 임계치를 다르게 부여하라. 이를 통해 통신 링크의 가용 대역폭을 영리하게 보존하고, 극단적으로 열악한 비가시권(BVLOS) 환경의 장거리 비행에서도 즉각적인 복원력(Resilience)을 발휘하는 PX4 시스템을 달성할 수 있을 것이다.
