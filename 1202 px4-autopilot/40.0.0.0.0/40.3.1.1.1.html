40.3.1.1.1. 저역 통과 필터(Low-Pass Filter) 관점의 임펄스 응답(Impulse Response) 특성

지상 관제 시스템(GCS)과 무인항공기 간의 통신 네트워크는 고유의 지연(Delay)과 지터(Jitter)를 수반한다. 이러한 비결정론적 특성을 제어하고 MAVLink 명령어 전송의 신뢰성을 확보하기 위해 PX4-Autopilot과 QGroundControl 시스템은 필터링 기법을 적용한다. 특히 지수 가중 이동 평균(Exponentially Weighted Moving Average, EWMA) 필터는 구조가 단순하면서도 효과적인 무한 임펄스 응답(Infinite Impulse Response, IIR) 저역 통과 필터(Low-Pass Filter, LPF)로 널리 사용된다.

본 절에서는 저역 통과 필터 관점에서 EWMA 필터의 임펄스 응답 특성을 수학적으로 분석하고, 이러한 필터가 통신 딜레이 및 타임아웃 제어에서 어떤 역할을 수행하는지 깊이 있게 살펴본다. (기준 펌웨어: PX4-Autopilot v1.14 이상, QGroundControl v4.3 이상)

1. 지수 가중 이동 평균(EWMA) 필터의 IIR 특성

EWMA 필터는 이전 상태의 추정치와 현재 측정된 측정치의 가중합을 통하여 새로운 추정치를 계산한다. 이를 이산 시간(Discrete-time) 차분 방정식(Difference Equation)으로 표현하면 다음과 같다.

y[n] = (1 - \alpha) y[n-1] + \alpha x[n]

  • y[n]: n 시점에서의 필터링된 출력 (예: 예측된 네트워크 지연 시간)
  • x[n]: n 시점에서의 새로운 입력 (예: 새롭게 측정된 MAVLink 패킷 레이턴시)
  • \alpha: 평활화 계수(Smoothing Factor)로 0 < \alpha < 1 의 범위를 가진다.

이 방정식은 현재의 출력을 계산하기 위해 이전의 출력 y[n-1]을 사용하므로 재귀적(Recursive) 특성을 지닌다. 이는 곧 유한한 입력이 주어졌을 때 출력이 무한히 지속될 수 있음을 의미하며, 이 때문에 IIR 필터로 분류된다. LPF 관점에서 \alpha 값이 작을수록 이전 상태 비율이 높아져 차단 주파수(Cut-off Frequency)가 낮아지고 노이즈 제거 효과가 커지지만, 시스템의 응답 속도는 느려지게 된다.

2. LPF의 임펄스 응답(Impulse Response) 도출

어떤 선형 시불변(Linear Time-Invariant, LTI) 시스템의 특성을 파악하는 가장 기본적인 방법은 단위 임펄스(Unit Impulse) 연산자인 크로네커 델타(Kronecker Delta) \delta[n]을 입력으로 가했을 때의 출력, 즉 임펄스 응답 h[n]을 확인하는 것이다.

입력 x[n] = \delta[n]를 위의 차분 방정식에 대입하여 임펄스 응답 h[n]을 구해보자. 시스템이 초기 상태 0에서 시작한다고 가정한다 (h[-1] = 0).

  • n = 0 일 때: h[0] = (1 - \alpha)h[-1] + \alpha \delta[0] = \alpha
  • n = 1 일 때: h[1] = (1 - \alpha)h[0] + \alpha \delta[1] = \alpha(1 - \alpha)
  • n = 2 일 때: h[2] = (1 - \alpha)h[1] + \alpha \delta[2] = \alpha(1 - \alpha)^2
  • 일반화하면: h[n] = \alpha(1 - \alpha)^n \quad (n \ge 0)

이 결과를 통해 임펄스 응답이 지수적으로 감소(Exponentially Decaying)하는 형태임을 알 수 있다. 임펄스 응답의 신호 길이(Signal Duration)가 무한대(Infinite)이기 때문에 IIR 시스템에 속하며, 이러한 점진적 감소 형태는 저주파 성분을 통과시키고 고주파 노이즈를 억제하는 전형적인 1차 저역 통과 필터(1st-order Low-Pass Filter)의 시간 영역 거동 특성과 정확히 부합한다.

3. 네트워크 지연 예측 모델로서의 의미

MAVLink 프로토콜상에서 타임아웃과 재전송을 결정할 때 예측된 RTT(Round-Trip Time)는 제어의 안정성을 좌우하는 매우 중요한 요소이다. 만약 측정된 네트워크 RTT(입력 x[n])에 극단적인 스파이크 노이즈(임펄스 신호)가 발생했다고 가정해 보자.

이때 EWMA 필터를 통과한 임펄스 응답 h[n]은 즉시 \alpha만큼만 치솟은 이후, (1-\alpha) 비율을 매 단계마다 곱해가며 서서히 원래 상태(안정적인 추정치)로 수렴하게 된다. 이를 통해 일시적인 통신 지연 스파이크(Delay Spike)가 QGroundControl 측의 MAVLink 재전송(Retransmission) 로직을 불필요하게 트리거하는 오작동을 방지할 수 있다.

4. Ardupilot 대비 PX4-Autopilot 관제 환경의 LPF 활용 분석

PX4-Autopilot과 Ardupilot 모두 텔레메트리 스트림 수신이나 내부 Estimator(EKF2 등)에서 LPF 구조를 다수 적용한다. 통신 타임아웃 및 패킷 전송 신뢰성 측면에서, Ardupilot 생태계(Mission Planner 주 사용)는 상대적으로 하드코딩된 정적 딜레이 임계치(Static Delay Threshold)에 의존하여 타임아웃을 강제로 리셋하는 루틴이 잦은 경향이 있다.

반면, PX4-Autopilot 환경과 주로 통합되는 QGroundControl(QGC)은 통신 채널 링크 상태나 다중 기체 군집 통신(Swarm Communication) 시 토폴로지 변화에 강건하게 대응하도록 동적 딜레이 계산 로직(Dynamic Delay Calculation) 적용을 지향하는 쪽으로 진화하고 있다. QGC 백엔드(C++)의 Vehicle 및 LinkManager 클래스 내부에서 RTT Ping 테스트 결과를 반복 계산할 때, 이 EWMA 임펄스 특성이 부드러운 저역 통과 필터로 작용하여 MAVLink의 상태 머신(Wait State) 타임아웃 주기를 유동적으로 확장하거나 축소하는 기반 알고리즘 역할을 수행한다.

5. 전달 함수(Transfer Function)와 극점(Pole) 분석

Z-변환(Z-Transform)을 통하여 EWMA 필터의 전달 함수 H(z)를 수식적으로 구획해 보자. 차분 방정식의 양변을 Z-변환하면 다음과 같다.

Y(z) = (1 - \alpha)z^{-1}Y(z) + \alpha X(z)

Y(z) \left( 1 - (1 - \alpha)z^{-1} \right) = \alpha X(z)

H(z) = \frac{Y(z)}{X(z)} = \frac{\alpha}{1 - (1 - \alpha)z^{-1}} = \frac{\alpha z}{z - (1 - \alpha)}

위 식에서 시스템의 극점(Pole)은 분모가 0이 되는 지점인 z = 1 - \alpha 부분이다. 평활화 계수 \alpha가 0 < \alpha < 1 이므로, 극점 1 - \alpha은 복소 평면(Complex Plane) 위 단위원(Unit Circle)의 내부에 안정적으로 궤적을 그리며 존재하게 된다.

극점이 단위원 내부에 위치한다는 것은 시스템이 이산 리아프노프 안정성(Discrete Lyapunov Stability)을 확실히 만족함을 수학적으로 보증하는 것이다. 임펄스 응답 반경 관점에서 고찰하면, 분모의 극점이 실수 축의 양(Positive)의 방향, 다시 말해 단위원 경계 1에 수렴할수록(\alpha \to 0), 임펄스 신호 감소율이 현저하게 느려져 필터가 과거 데이터 표본에 더 지속적으로 의존하게 되며 응답 시상수(Time Constant)가 장기화된다.
반대로 \alpha가 1에 가까워질수록 극점은 원점 영역에 근접하여, 임펄스의 발산이 급격히 제동되어 짧은 소멸 주기를 갖게 된다.

이러한 극점-영점(Pole-Zero) 특성을 면밀히 분석함으로써 관제 시스템 엔지니어는 네트워크 지연 가변성(Jitter)이 심대한 열악한 연결 체계에서는 \alpha 값을 축소하여 강한 LPF 댐핑(Damping)을 부여하고, 데이터 링크가 쾌적하여 상태 갱신 민감도를 확보하고자 할 때는 \alpha를 상승 조정하는 체계적 파라미터 튜닝을 도모해야 한다.

2. 소스 코드 구현 예시 및 응용 메커니즘 (C++)

PX4-Autopilot의 펌웨어나 GCS 애플리케이션 프레임워크 내 C++ 소스 코드에서 이러한 임펄스 응답 특성을 갖는 저역 통과 필터는 공통된 객체 지향 형태로 구현된다. 타임아웃 모듈 레벨에서 수신 지연 주기를 보정하는 EWMA 구조체는 다음과 같이 추상화하여 구현할 수 있으며, 이 메커니즘을 ROS2 Offboard 제어나 데이터 로깅 파이프라인 지터 상쇄에도 동일하게 확장 적용할 수 있다.

#include <iostream>

class EWMALowPassFilter {
private:
    float _alpha;
    float _y_prev;
    bool _is_initialized;

public:
    EWMALowPassFilter(float alpha) : _alpha(alpha), _y_prev(0.0f), _is_initialized(false) {}

    // 필터 연산 적용
    float apply(float x_curr) {
        if (!_is_initialized) {
            _y_prev = x_curr; // 초기 상태 바인딩
            _is_initialized = true;
            return _y_prev;
        }
        // 차분 방정식: y[n] = (1 - alpha)*y[n-1] + alpha*x[n]
        _y_prev = (1.0f - _alpha) * _y_prev + _alpha * x_curr;
        return _y_prev;
    }
    
    // 임펄스 응답의 지수적 감쇠 도출 시뮬레이션 메서드
    void simulateImpulseResponse(int steps) {
        float impulse = 1.0f; // 초기 t=0에서 Dirac delta 입력 모사
        for (int n = 0; n < steps; ++n) {
            float out = apply(impulse);
            impulse = 0.0f; // n=0일 때만 1.0, 그 이후로 계속 0.0 유지
            printf("Step %d - Impulse Response h[%d]: %f\n", n, n, out);
        }
    }
};

위의 시뮬레이션을 실행하면, 생성되어 반환되는 out 값 시퀀스가 앞서 2장에서 이끌어낸 이론적 방정식 h[n] = \alpha(1 - \alpha)^n의 도출값을 오차 없이 완전하게 재생함을 알 수 있다.

3. 결론 사항

GCS 비동기 I/O 커널 구조 설계나, 여러 기체를 운용하는 자율 스웜(Swarm) 에이전트 다중 링크 체계 구축 시, 본 문서와 같은 EWMA 저역 통과 필터 기반의 임펄스 응답 역학을 철저히 규명하고 적용해야 한다. 올바르게 조율된 저역 통과 필터는 일회성 임펄스 교란이나 통신 딜레이 돌발 요소가 전체 무인항공기 상태 동기화를 오염시키지 못하도록 막강한 필터링 장벽 역할을 수행한다. 개발 엔지니어는 이러한 특성을 내재화하여, PX4-Autopilot으로 발송되는 핵심 MAVLink 제어 구문(예: COMMAND_LONG)들의 타임아웃 파라미터를 실시간 네트워크 특질에 부합하도록 적응 제어(Adaptive Control)하라.
