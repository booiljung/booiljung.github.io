40.3.2.1. 제이콥슨/카렐 알고리즘(Jacobson/Karels Algorithm)의 타이머 보정 분산(Variance) 추정

지상 관제 시스템(GCS)과 PX4-Autopilot 간의 신뢰성 있는 제어 명령(예: MAVLink COMMAND_LONG) 전송을 보장하기 위해서는, 통신 네트워크의 동적인 지연 변화(Latency Jitter)를 정확하게 예측하고 타임아웃(Timeout) 주기를 유연하게 조절해야 한다. 이를 위해 현대의 데이터 비동기 네트워크 프로토콜에서 널리 차용되는 기법이 바로 TCP 혼잡 제어의 근간이 된 제이콥슨/카렐 알고리즘(Jacobson/Karels Algorithm)이다.

이 알고리즘은 단순한 평균 왕복 지연 시간(SRTT, Smoothed RTT)뿐만 아니라, 네트워크 지연의 절대적 분산(Variance) 치를 함께 산출하여 재전송 타임아웃(RTO, Retransmission Timeout) 임계치에 엄격하게 반영하는 것이 최우선 핵심이다. 본 절에서는 이 알고리즘이 무인항공기 관제 네트워크에서 어떻게 작용하는지 상세하게 전개한다. (기준 펌웨어: PX4-Autopilot v1.14 이상, QGroundControl v4.3 이상)

1. 지연 분산(Variance) 추정의 필요성

전파 통신(RF, LTE/5G, 위성)을 원거리 기반으로 하는 무인기 운용 환경은 유선 네트워크 대비 외부 간섭, 기체의 비행 자세(Attitude) 변화에 따른 안테나 음영(Shadowing) 현상 등으로 인해 패킷 도달 시간의 변동폭(Jitter) 상하 편차가 매우 광범위하다. 만약 GCS가 산출한 타임아웃 주기가 RTT의 단순 평균값에만 의지한다면, 평균 대역에서 조금만 벗어나는 정상적인 지연 스파이크(Delay Spike)조차도 통신 단절(Timeout)로 시스템이 오판하게 만들어 불필요한 명령 재전송 스톰(Retransmission Storm) 트래픽을 폭증시킨다.

따라서 타임아웃 설계는 평균 RTT 파라미터 값에 네트워크의 “불안정성(Variance)” 정도를 비례하여 곱한 무결성 안전 여유분(Safety Margin)을 더하여 동적으로 보정 계산되어야 한다. 이를 연산 효율적으로 해결할 수 있게 도출된 수리적 계통 모델이 제이콥슨/카렐 알고리즘 체계이다.

2. 알고리즘의 핵심 수학적 모델링 메커니즘

알고리즘 재계산 연산은 크게 세 단계로 이루어지며, GCS가 PX4-Autopilot으로부터 MAVLink 수신 응답(ACK)을 인계 받을 때마다 실시간 백그라운드로 갱신된다.

2.1 Step 1: 평활화된 왕복 시간(SRTT) 갱신

가장 최근 도달하여 산술 측정된 RTT_{new}를 바탕으로, 지수 가중 이동 평균(EWMA)을 사용하여 이전에 추정된 부드러운 RTT 연속 확률값(SRTT)을 즉각적으로 갱신 적용한다.

SRTT_{n} = (1 - \alpha) \cdot SRTT_{n-1} + \alpha \cdot RTT_{new}

일반적으로 임베디드 통신 GCS 환경에서 가중 \alpha(알파) 값은 0.125(1/8)를 고정적으로 사용하여, 마이크로프로세서가 비트 논리 시프트 연산(>> 3)으로 최적화 속도를 달성할 수 있도록 치밀하게 설계한다.

Step 2: RTT 지연 오차 매칭 분산(RTTVAR) 추정

그 다음 파이프라인으로, 새로 측정된 RTT_{new}와 기존 평면 평균 SRTT_{n-1} 간의 절대 오차를 구하고, 이를 제2의 이동 평균 궤적으로 평활화하여 순수 RTT 분산치(RTTVAR)를 계산한다. (테이블 셀이나 행렬식 내부 기호가 아닌 절대 오차 연산을 표기할 때는 수학적 \vert 로 명시적 표기한다)

RTTVAR_{n} = (1 - \beta) \cdot RTTVAR_{n-1} + \beta \cdot \vert RTT_{new} - SRTT_{n-1} \vert

분산을 추정하는 매개변수 \beta(베타) 값은 분산 증감 편차에 루프가 훨씬 더 민감하게 즉각 반응하도록 \alpha 대비 소폭 큰 비율인 0.25(1/4)를 보편적으로 차용한다. 이 식은 RF 네트워크가 극심하게 요동쳐 절대 오차가 순간 커지면 RTTVAR 수치가 딜레이 없이 즉각적으로 증폭되도록 강제하는 수학적 효과를 만든다.

2.2 Step 3: 최종 재전송 타임아웃(RTO) 산출 한계 도출

전체 통신 데이터 제어 루프의 재전송 대기 한계(Timeout threshold) 주기 RTO_{n}은 최종 단에서 다음과 같이 합산 도출된다.

RTO_{n} = SRTT_{n} + K \cdot RTTVAR_{n}

여기서 K는 분산 증류 가중치로서 일반적으로 4로 정의되어 설정된다. 지연 분산 편차 폭이 커질수록 4배의 가중치로 타임아웃을 대폭 안전하게 늘림으로써, 네트워크가 일시적인 간섭으로 흔들리는 상황에서 섣부른 재전송 트래픽 유발로 인한 대역폭(Bandwidth) 포화를 원천적으로 차단한다.

3. Ardupilot과 PX4/QGC 생태계 관제 구조 측면의 응용 및 차이점 분석

  • Ardupilot (Mission Planner 주력): MAVLink 명령 송신 간 전송 실패 및 지연 시 동적 보정 피드백 로직을 구획하여 갖추기보다는, 타임아웃 한계치(Timeout Limit)를 사전에 충분히 넓은 여유 있게(보통 1~2초 이상 고정) 데스크탑 애플리케이션에 하드코딩(Hard-coding)된 변수로 정적(Static)으로 설정하는 관성적 경향이 강하다. 이런 형태는 일시적 지연 분산 요동 변화에 시스템이 둔감하여 연결 자체는 시각적으로 잘 유지되는 것처럼 보이나, 빠른 비상 정지(Kill Switch) 제어 개입 요소가 필요할 때 시스템의 동기 반응이 한 박자씩 늦어지는 맹점을 품고 있다.
  • PX4-Autopilot & QGroundControl: 현대적 ROS2 스웜(Swarm) 멀티 비행체 다중 통신 관리 체계와 고주파수(High-frequency) Offboard 연속 제어를 아키텍처 중심부로 구성 및 고려하여 설계된 PX4 코어 생태계는, 상호 오가는 MAVLink 패킷 큐(Queue) 주기가 훨씬 촘촘하고 복잡 다단하다. 따라서 QGroundControl 내부 코어의 Vehicle 커맨드 응답 매니저 파이프라인에서 패킷 핑(Ping) 결과를 바탕으로 제이콥슨/카렐 모델 분산 변동 설계 철학 등에서 영감을 받은 분산 기반의 논블로킹(Non-blocking) 비동기 대기 및 동적 타임아웃 스케일링(Scaling) 모듈 체계를 구비해 둔 형태를 지향한다. 이는 네트워크 상태가 간섭 없이 쾌적할 때는 타임아웃 한계 변수를 촘촘히 조여 즉각 반응형으로 구축하고, 열악할 때는 분산 보정치 오차 방어에 의해 자동으로 타임아웃을 확장해 안전 우선시 동작(Failsafe approach)을 취하는 형태로 귀결된다.

5. 소스 코드 구현체 예시 해석 (C++)

GCS 및 엣지 컴패니언 컴퓨터(Edge Companion Computer)에서 동적 MAVLink 발송 타임아웃을 능동적으로 관리하는 고수준 추상화된 단일 C++ 클래스는 다음과 같은 형태로 알고리즘을 소화하여 구성될 수 있다. 오차 기반 연산 수행 시 CPU의 코프로세서 간접 부하를 줄이기 위해 절대값 편차 공식을 단순 정수(Integer) 시프트 연산이나 경량 부동 소수점(Floating-Point) 캐스팅만으로 컴파일 처리하여 연산을 경량화 구성한다.

#include <cmath>
#include <algorithm>

// 제이콥슨/카렐 알고리즘 기반 타임아웃 산출 클래스
class JacobsonKarelsTimeout {
private:
    float _srtt;     // Smoothed RTT 예측치
    float _rttvar;   // RTT 분산 편차량
    float _rto;      // 산출 도출된 Retransmission Timeout 
    
    // 알고리즘 기본 규약 매개변수 파라미터 세트
    const float ALPHA = 0.125f; // 지수 이동가중 알파 (1/8)
    const float BETA  = 0.25f;  // 편차 가중 반영률 베타 (1/4)
    const float K     = 4.0f;   // 분산 확대 가중 마진
    
    // 절대 하한 타임아웃 (0.5초 등 시스템 물리적 리밋 보장 설계)
    const float MIN_RTO = 0.5f; 

    bool _initialized; // 초기화 스위치 플래그

public:
    JacobsonKarelsTimeout() : _initialized(false) {}

    // 새로운 MAVLink ACK 수신 RTT 도달 시간 기반 타임아웃 루틴 업데이트 함수
    float update_timeout(float current_rtt) {
        if (!_initialized) {
            // First run initial 바인딩 초기화 셋업
            _srtt = current_rtt;
            _rttvar = current_rtt / 2.0f;
            _rto = _srtt + std::max(0.1f, K * _rttvar);
            _initialized = true;
            return _rto;
        }

        // 1. 편차 오차 절대 연산 (절대값 치환) 계산
        float error = current_rtt - _srtt;
        
        // 2. 분산(RTTVAR) 상태 벡터성 업데이트
        _rttvar = (1.0f - BETA) * _rttvar + BETA * std::abs(error);
        
        // 3. 평균(SRTT) 선형적 업데이트
        _srtt = (1.0f - ALPHA) * _srtt + ALPHA * current_rtt;
        
        // 4. 추정식 조합에 따른 최종 통신 타임아웃(RTO) 임계치 도출 
        _rto = _srtt + (K * _rttvar);

        // 하한선 필터링 타임아웃 강제 보정 기작 (지나친 단축 방지)
        if (_rto < MIN_RTO) {
            _rto = MIN_RTO;
        }

        return _rto;
    }
};

알고리즘 시스템의 내부 시뮬레이션 동작 특성에 따라, 통신 환경 모델망에 의도적인 백색 노이즈를 다수 주입하여 측정 입력 변수인 current_rtt 값이 무작위로 심하게 요동치는 충격파를 주더라도, _rttvar 알고리즘 내부 변수가 반동 편차로 상승함에 따라 전반적인 RTO 방어 임계치가 즉각 안전하게 우상향 포물선을 그리며 마진 영역을 벌리는 방어 형태를 시뮬레이션으로 명확히 확인 관찰할 수 있다.

6. 결론 사항 정리

모든 MAVLink 패킷 시스템 개발에서 단순히 비례 계수만 할당 고정하여 GCS와 PX4-Autopilot 제어 보드 간 통신 타임아웃을 상수로 예측 평가하는 행위는 무선 RF 통신 채널망의 치명적인 비정형 요동성(Unpredictable Variation)을 간과한 반쪽짜리 파편적 아키텍처 접근이다.

제이콥슨/카렐 분산 알고리즘 계통의 구조와 같이 실질 통신망의 왕복 시간의 동적 에러 분산(Variance) 수치를 실시간으로 맹렬히 추적·반영하는 수학적 방어 모듈 체계를 반드시 코어에 도입함으로써, 드론 기체가 통신 교착 상태인 데드락(Dead lock) 상태에 빠진 채 멈춰 있거나 GCS 비디오 스트리밍 UI 화면이 프리징(Freezing)되는 사태를 근본 아키텍처단부터 물리적으로 차단할 수 있다.

특히 향후 군집 주행 등 다중 기체의 자율 복합 미션 운용 간에는 대량의 데이터 요청으로 인한 MAVLink 병목 통신폭주(Congestion) 누수 현상이 가장 빈번하고 발생률이 높은 아키텍처 결함 오류로 대두되므로, 본 절에서 제안하고 분석한 동적 분산 추정식 기반의 RTO 설계 철학을 반드시 구조적으로 이해하고 응용 GCS 시스템 소스 컨트롤러 코드에 탑재하도록 구성 설계하라.
