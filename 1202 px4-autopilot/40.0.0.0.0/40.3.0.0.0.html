40.3. 적응형 타임아웃(Adaptive Timeout)의 시계열 제어 이론 적용

지상 관제 시스템(GCS)과 무인항공기(UAV) 간의 무선 통신 환경은 다중 경로 페이딩(Multipath Fading), 간섭(Interference), 기체 자세 변화에 따른 안테나 방사 패턴의 제약 등으로 인해 지연 시간(Latency)이 동적으로 변화한다. 기존의 고정된 타임아웃(Static Timeout) 방식은 이러한 네크워크 특성을 반영하지 못해 불필요한 재전송(Retransmission)을 유발하거나, 심각한 데이터 손실을 감지하는 속도를 지연시킨다. 본 절에서는 QGroundControl(설명 기준: v4.3 이상)과 PX4-Autopilot(설명 기준: v1.14 이상)의 MAVLink 통신 스택에 시계열 제어 이론(Time-Series Control Theory)을 적용한 적응형 타임아웃(Adaptive Timeout) 모델을 살펴본다.

1. 적응형 타임아웃(Adaptive Timeout)의 개념 및 필요성

적응형 타임아웃은 이전 송수신 트랜잭션에서 측정된 패킷 왕복 시간(RTT: Round-Trip Time)의 통계적 특성을 기반으로, 다음 트랜잭션의 타임아웃 주기를 동적으로 조절하는 기법이다. 통신이 원활할 때는 타임아웃을 짧게 유지하여 패킷 유실을 빠르게 감지하고 대응하며, 네트워크가 혼잡하여 지연 시간이 길어질 때는 타임아웃을 연장하여 불필요한 중복 재전송을 방지한다.

  • 고정 타임아웃의 한계: 설정된 타임아웃 T가 실제 RTT보다 짧으면(Early Timeout) 불필요한 재전송 폭풍(Retransmission Storm)이 발생하고, 실제 RTT보다 길면 통신 장애 감지 및 Failsafe 발동이 늦어져 기체 제어권을 잃을 위험이 증가한다.
  • 적응형 타임아웃의 이점: 네트워크 지연의 변동성(Jitter)을 동적으로 학습하여 시스템 대역폭을 보존하고 관제 신뢰성을 높인다.

2. 시계열(Time-Series) 필터링 모델 적용

적응형 타임아웃을 추정하기 위해서 가장 널리 쓰이는 시계열 제어 이론은 지수 가중 이동 평균(EWMA: Exponentially Weighted Moving Average)과 분산 추정을 혼합한 방식이다. 이는 TCP의 패킷 재전송 타임아웃 산정에 사용되는 Jacobson/Karels 알고리즘과 수학적으로 동일한 궤를 가진다.

2.1 RTT(Round-Trip Time) 평활화 (Smoothed RTT)

이전 RTT 샘플 R이 새로 측정되었을 때, 평활화된 통신 지연시간 SRTT는 이전의 SRTT와 현재 R의 가중합으로 계산한다.
SRTT_{new} = (1 - \alpha) \cdot SRTT_{old} + \alpha \cdot R
여기서 \alpha는 평활화 인자(Smoothing Factor)로 보통 0.125를 사용한다.

2.2. 편차 추정 (RTT Variation)

네트워크의 지터(Jitter) 혹은 불안정성을 나타내는 편차 RTTVAR은 다음과 같이 갱신된다.
RTTVAR_{new} = (1 - \beta) \cdot RTTVAR_{old} + \beta \cdot \vert SRTT_{old} - R \vert
여기서 \beta는 분산 평활화 인자로 보통 0.25를 사용하여 변동성에 보다 민감하게 반응하도록 설계한다.

2.2 적응형 재전송 타임아웃(RTO: Retransmission Timeout) 산정

최종적으로 다음 MAVLink 트랜잭션의 타임아웃 RTO는 다음과 같이 설정된다.
RTO = SRTT_{new} + \max(G, 4 \cdot RTTVAR_{new})

  • G: 시스템 타이머의 최소 분해능(Granularity)
  • 분산(RTTVAR)에 4배의 가중치를 두어 급격한 네트워크 변화에도 안전 마진을 확보한다.

3. QGroundControl 및 PX4 시스템 적용 아키텍처

QGroundControl에서의 동적 파라미터 백업(Parameter Protocol)이나 FTP 전송 로직, 혹은 무리 비행(Swarm) 제어 시, 이러한 적응형 타임아웃을 활용할 수 있다.

4. 실무 적용 및 주의사항

  • Karn’s Algorithm 적용: 재전송된 패킷에 대한 ACK는 원래 데이터와 매핑될 수 없고 RTT를 왜곡시키므로, 재전송된 패킷의 RTT 측정값은 EWMA 필터에 입력으로 포함하지 마라.
  • 비정상적인 스파이크 필터링: 일시적으로 통신이 끊어지거나 노이즈로 인해 극단적으로 큰 R 값이 산출되었을 때, 이를 그대로 시계열 필터에 적용하면 타임아웃이 과도하게 커지는 문제가 발생한다. 따라서 SRTT 상한선(Max Cap)을 하드코딩된 값(예: 3초)으로 제한하라.
  • QGroundControl의 Vehicle 커맨드 매니저 모듈을 재작성하여 고정값 기반의 타이머(QTimer) 대신, 상기 공식을 동적으로 계산하여 타임아웃 주기를 설정하도록 오버라이딩(Overriding)하라.

적응형 타임아웃은 단순한 MAVLink 송수신 관리를 넘어서, 자율 회피 기동이나 드론 군집 통신의 안정성을 확보하기 위한 핵심 시계열 제어 기반 인프라이다.
