40.3.2.1.1. 리아프노프 안정성(Lyapunov Stability) 판별법을 적용한 타임아웃 임계치 발산 억제

무인기 관제 시스템(GCS; Ground Control Station)에서 MAVLink 프로토콜을 이용해 비행 제어기(PX4-Autopilot)로 전송하는 오프보드(Offboard) 제어 명령 경로는 비선형적(Nonlinear)이고 동적(Dynamic)인 네트워크 상태의 지배를 받는다. 지연 시간(Latency) 추정 오차로 인해 발생한 부적절한 타임아웃(Timeout) 설정은 GCS의 큐(Queue)에 대기 중인 패킷 수와 재전송 횟수를 급격히 증가시켜 결국 통신 채널을 마비시키는 발산(Divergence) 현상을 일으킨다.

이와 같은 비선형 피드백 제어 시스템의 안정성을 해석하기 위해, 러시아 수학자 알렉산드르 리아프노프(Aleksandr Lyapunov) 가 고안한 **리아프노프 안정성 판별법(Lyapunov Stability Criterion)**을 MAVLink 타임아웃 제어 알고리즘에 적용할 수 있다. 본 절에서는 네트워크 혼잡 상태를 상태 공간(State Space)으로 정의하고, 타임아웃 임계치가 시스템 에너지를 무한정 팽창시키지 않도록 임계치 상하한 여유(Margin)를 설계하는 해석 기법에 대해 철저하게 파악한다.

1. MAVLink 통신 큐 상태 공간(State Space) 정의

리아프노프 안정성을 논의하기 위해 가장 먼저 시스템의 동적 특성을 대변하는 상태 변수(State Variable) 집합을 정의해야 한다. QGroundControl(이하 QGC)을 포함한 전형적인 GCS 타임아웃 제어기에서 가장 중요한 지표는 전송을 대기중이거나 재전송을 반복하고 있는 패킷의 누적량이다.

이산 시간(Discrete Time) k에서의 시스템 상태 벡터를 \mathbf{x}[k] = [q[k], \tau[k]]^T 로 정의한다.

  • q[k]: 현재 송신 버퍼 및 큐에 누적된 활성 COMMAND_LONG 패킷의 개수.
  • \tau[k]: 현재 GCS가 평가하고 있는 평균 대기열 지연 시간.

이 상태 변수는 다음과 같은 상태 천이 방정식(State-Transition Equation)을 갖는다.

\mathbf{x}[k+1] = f(\mathbf{x}[k], RTO[k], D[k])

여기서 RTO[k]는 적응형 재전송 타임아웃(Retransmission Timeout)이며, D[k]는 외부 교란(Disturbance, 예를 들어 무선 채널의 간섭 심화로 패킷이 드롭되는 비율)을 나타낸다.

2. 리아프노프 함수(Lyapunov Function)의 구축

리아프노프 제2방법(직접법)은 시스템 방정식 f(\mathbf{x})를 선형적으로 풀지 않고도, **스칼라 에너지 함수(Energy Function) V(\mathbf{x})**의 거동만으로 평형점(Equilibrium Point)의 안정성을 판별한다. MAVLink 큐 네트워크에서는 “누적된 패킷의 양과 지연의 크기“를 가상의 포텐셜 에너지로 상정한다.

다음과 같은 2차 형식(Quadratic Form)의 리아프노프 후보 함수를 고려한다.

V(\mathbf{x}[k]) = \mathbf{x}[k]^T P \mathbf{x}[k] = p_{11}q[k]^2 + 2p_{12}q[k]\tau[k] + p_{22}\tau[k]^2

여기서 체계(System)가 안정성을 보장하기 위해서는 행렬 P가 양의 정부호(Positive Definite, P > 0)여야 한다. 이것은 어떠한 네트워크 상태 변화가 있더라도 에너지가 항상 관찰 가능한 양수임을 의미한다. 즉 V(\mathbf{x}) > 0 이고 V(0) = 0 조건을 만족한다.

2. 안정성 판별과 타임아웃 임계치 발산 억제 원리

시스템이 점근적으로 안정(Asymptotically Stable)하기 위한 핵심 필요충분조건은 상태가 진전될수록 리아프노프 함수(시스템 에너지)의 시간 변화율(차분)이 반드시 음수(Negative)가 되어야 한다는 것이다.

\Delta V(\mathbf{x}[k]) = V(\mathbf{x}[k+1]) - V(\mathbf{x}[k]) < 0

3.1 발산하는 재전송 타임아웃(RTO) 설계의 함정

만일 QGC의 내부 타임아웃 파라미터를 통신 대역폭 대비 지나치게 짧게(예: RTO < 100\text{ms}) 강제 설정했다면, 실제 물리 채널이 감당할 수 없는 수준의 재전송 명령이 쏟아져 들어온다. 이 경우 시간 경과에 따라 큐잉 지연 \tau[k]와 버퍼 데이터 q[k]가 동시에 팽창하므로 다음과 같은 현상이 발생한다.

\Delta V(\mathbf{x}[k]) > 0

에너지가 계속 증가한다는 것은 평형점으로 수렴하지 못하고 시스템이 마비(Congestion Collapse)된다는 것을 뜻한다. 결과적으로 COMMAND_ACK를 받지 못한 MAVLink 스레드는 락아웃(Lock-out) 상황에 빠져 치명적 장애를 초래한다.

2.1 발산 억제를 위한 임계치 제어기의 설계 목표

리아프노프 판별법에 기초한 타임아웃 제어기의 목표는 어떤 통신 지터와 외란(D[k]) 속에서도 \Delta V(\mathbf{x}[k]) \leq 0 을 유지하도록 RTO[k] 를 계산해 내는 것이다. 이를 달성하기 위해 제이콥슨/카렐 분산(Variance) 추정치 등을 가미할 때, RTO의 하한 임계치(Lower-Bound Threshold) RTO_{min}을 강제하여 비선형성이 에너지를 증가시키지 않도록 억제(Bounding)하는 필터를 구현할 수 있다.

3. GCS 개발 응용 시사점

자체 GCS 애플리케이션 프레임워크나 MAVSDK를 통한 응용 소프트웨어를 개발할 때, 위에서 규명한 리아프노프 안정성 모델을 아키텍처 내부에 “혼잡 회피 보호벽(Congestion Avoidance Guard)” 패턴으로 이식할 수 있다.

  • PX4 비행 제어기로 전송되는 MAVLink 흐름에 있어 텔레메트리 백프레셔(Backpressure)를 모니터링한다. (예: SERIAL_CONTROL 또는 메시지 큐 지표 감지)
  • 큐의 크기가 임계치를 초과함으로써 \Delta V 관점의 양수(+) 편향이 관찰될 경우, 즉시 타임아웃 로직의 백오프 계수(Backoff Factor, TCP의 경우 보통 2배수 곱셈)를 증가시켜 시스템 에너지를 강제로 감소시킨다.
  • 이로 인해 Ardupilot의 단일 스레드 기반 직렬 통신 구조에서 발생하던 관제 소프트웨어 멈춤(Freezing) 현상을 멀티스레드 기반의 비동기 PX4 대응 GCS에서 완벽하게 예방할 수 있다.

4. 요약

오프보드 제어(Offboard Control)나 군집 비행(Swarm) 환경에서 다량의 제어 지령을 발송할 때 타임아웃과 재전송 주기를 맹목적으로 줄이는 것은 제어 시스템 관점에서 볼 때 발산을 초래하는 행위와 다를 바 없다. **리아프노프 안정성 판별법(Lyapunov Stability Criterion)**은 복잡하고 불확실한 통신 채널 환경에서 MAVLink 버퍼 메모리의 점증 현상을 하나의 에너지 함수 팽창으로 모델링한다. 이를 통해 엔지니어는 MAVLink 백엔드 큐 엔진을 설계할 때, 에너지 소산(Dissipation)을 보장하는 안전한 재전송 타임아웃 임계치의 하한선을 수학적, 알고리즘적으로 확립할 수 있다.

이 설계 방식은 예측하기 어려운 다양한 외란 요인(바람, 무선 주파수 스위핑, EMI 노이즈)에 둘러싸인 무인기 통신의 강건성(Robustness)을 극한까지 견인하는 역할을 한다.
