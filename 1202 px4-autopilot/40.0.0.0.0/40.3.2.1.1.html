40.3.2.1.1. 리아프노프 안정성(Lyapunov Stability) 판별법을 적용한 타임아웃 임계치 발산 억제

지상 관제 시스템(GCS)에서 PX4-Autopilot으로 발송되는 제어 명령의 타임아웃(Timeout) 주기를 동적으로 산출하는 알고리즘은 본질적으로 현재의 네트워크 상태 변수(상태 공간)에 의존하는 피드백(Feedback) 동역학 시스템으로 모델링된다. 예측 불허의 극한 통신 환경 속에서, 적응형 타임아웃 수치가 적절한 구간 내에서 평형을 이루지 못하고 양(+)의 무한대로 발산(Divergence)하거나 지속적인 진동(Oscillation)을 겪는다면 기체와의 통신 링크는 심각한 단절 위기에 직면한다.

본 절에서는 타임아웃 산출 알고리즘의 상태 평형점(Equilibrium Point)을 해석하기 위해 제어 이론의 근간 매커니즘 중 하나인 리아프노프 안정성(Lyapunov Stability) 판별법을 수학적으로 적용해 보고, 이를 통해 타임아웃 임계치의 발산을 어떻게 소프트웨어적으로 억제하는지에 관해 심층 분석한다. (기준 펌웨어: PX4-Autopilot v1.14 이상, QGroundControl v4.3 이상)

1. 리아프노프 안정성(Lyapunov Stability)의 수학적 기초 공간

위상 공간(Phase Space) 및 상태 공간(State Space)을 다루는 제어 공학에서 특정 시스템의 상태 벡터 \mathbf{x}(t)가 초기 상태에 관계 없이 어떤 에너지 관점의 평형점 \mathbf{x_e} 주위에 머물거나 그 점으로 서서히 수렴한다면, 그 시스템은 구획적으로 ’안정(Stable)’하다고 판별한다. 러시아 수학자 알렉산드르 리아프노프(Aleksandr Lyapunov)가 고안한 이 직접 판별법(Direct Method)의 핵심 골자는, 시스템의 상태 방정식(미분 방정식) 해를 직접 풀이하지 않고도 스칼라(Scalar) 에너지 함수인 리아프노프 함수(Lyapunov Function) V(\mathbf{x})를 구성 및 분석하여 시스템 궤도를 증명하는 것이다.

어떤 평형점 주변 영역에서 시스템이 점근적으로 안정(Asymptotically Stable)하기 위한 두 가지 필요충분 조건은 다음과 같다.

  1. 연속 공간상에서 V(\mathbf{x})는 양의 정부호(Positive Definite) 행렬 형태를 띠어야 한다. 즉, 평형점이 아닌 \mathbf{x} \neq \mathbf{0} 일 때 항상 V(\mathbf{x}) > 0 이고, 오로지 평형점 \mathbf{x} = \mathbf{0} 에서만 V(\mathbf{0}) = 0 기저를 갖는다.
  2. 시간에 대한 함수 V(\mathbf{x})의 전미분 계수(Time Derivative) \dot{V}(\mathbf{x})는 음의 정부호(Negative Definite) 방향성을 띠어야 한다. 즉, 모든 상태 궤적을 훑으며 에너지가 소산(Dissipation)되어 \dot{V}(\mathbf{x}) < 0 형태로 수렴해야만 한다.

2. 타임아웃 제어 루프의 에너지 함수(Lyapunov Function) 모델링

앞 절(40.3.2.1)에서 소개한 편차 오차 기반의 타임아웃 연산 알고리즘 계통을 단일 상태 공간 이산 모델체계로 환원해 보자. MAVLink 명령어 송신 지연과 관련된 상태 변수 x_1을 “타임아웃 산출치와 이상적 목표 임계치 변수 간의 오차(Error)” 라 설정하고, x_2를 “오차의 변화율(Derivative of Error)“이라 통칭한다. 상태 벡터는 \mathbf{x} = [x_1, x_2]^T 로 전개된다.

이 알고리즘 시스템 내의 리아프노프 에너지 후보 함수 V(\mathbf{x})를 2차(Quadratic) 행렬식 형태 형상으로 정의해 볼 수 있다.

V(\mathbf{x}) = \mathbf{x}^T \mathbf{P} \mathbf{x} = p_{11} x_1^2 + 2p_{12} x_1 x_2 + p_{22} x_2^2

이때 행렬 \mathbf{P}가 양의 정부호 대칭 행렬(Symmetric Positive Definite Matrix)일 때 V(\mathbf{x})는 항상 양수이다 (V(\mathbf{x}) > 0).
네트워크 루프 체계에서 V는 타임아웃 펄스 “에너지(Energy)” 또는 “불안정성 척도“를 상징한다. 타임아웃 변화의 미분 \dot{V}(\mathbf{x})를 구했을 때, 통신 지연 편차가 줄어들어 타임아웃 산출 로직이 특정 주파수 변동폭 사이에서 평형을 찾게 하려면 반드시 \dot{V}(\mathbf{x}) < 0 을 만족하도록 각 이득 파라미터(Gain Parameter \alpha, \beta 등)의 극점을 이산 수치 공간 안으로 끌어들여 강제 구속해야(Constraining) 한다.

만약 극심한 간섭 환경 때문에 알고리즘상 \dot{V}(\mathbf{x}) > 0 이 되어 에너지가 점진적으로 축적(즉, 산출 타임아웃 주기가 무한대로 연이어 커짐)되는 징후가 이 판별 도메인에서 도출된다면, 해당 타임아웃 제어 알고리즘은 발산 궤도에 돌입한 것으로 이론적 확정(Determinism)을 짓는다.

3. 타임아웃 임계치 발산(Divergence)의 위험성 및 Anti-Windup 억제 체계

비행 제어체계 관점에서 MAVLink 타임아웃 임계치가 위 발산 궤도 영역에 들어서 무한대에 수렴하게 되면 심각한 문제를 초래한다. 예를 들어 “Waypoint 도달 명령“이나 “비상 정지(Kill Switch) 명령“의 타임아웃 변수가 발산하여 수십 초, 수 분으로 늘어나게 되면, 기체 연결이 일시 끊긴 상태에서 GCS는 재전송 시스템을 발동시키지 않고 오직 타임아웃이 종료되기만을 하염없이 대기하는 교착 상태(Deadlock)의 데스 록 트랩에 빠지게 된다.

RIA(Robustness Investigation Analysis) 관점에서 이러한 리아프노프 불안정을 방지하기 위해 GCS 엔진 아키텍처는 소프트웨어적으로 Anti-Windup(오차 누적 방지) 경계면 로직 억제제를 물리적으로 추가 설계한다.

  • Saturation Limit 방벽: 타임아웃 계산 함수 출력단에 비선형 포화 함수(Saturation Function) 블록 단위계를 결합한다. 어떠한 내부 증폭 게인이 연산을 튀게 만들더라도 최종 도출 타임아웃 변수가 시스템에 설정된 물리적 최상단 허용치(Threshold Max Limit, 예: 3.0초) 및 허용 하한치(Threshold Min Limit, 예: 0.5초)를 초과하지 못하도록 강력하게 억제 가위질(Clipping)을 수행한다.
  • 적분 누적(Integral Windup) 삭감 분리: 분산 오차를 스윕 누적하는 메모리 버퍼 단락에서 상한 리밋에 부딪혔을 때는 즉시 더 이상의 오차 축적이 진행되지 않도록 수리적 루프 이음세를 단절(Decoupling)하여 \dot{V}(\mathbf{x}) 가 양수 영역으로 돌파하지 않도록 수리적 에너지 댐핑을 시도한다.

4. Ardupilot과 PX4-Autopilot 환경에서의 타임아웃 발산 억제 철학 비교

  • Ardupilot (Mission Planner 주력 운용): 앞선 절에서 지적한 바와 같이 범용 Mission Planner는 통신 환경 측정에 따른 적응형의 비선형 가변 함수 체계를 기피하고 상수 타임아웃 고정폭 파라미터를 사용하므로, 초인적인 리아프노프 발산 에너지 자체가 시스템 논리상 거의 축적될 우려 구간이 없다. 이는 역설적으로 매우 직관적이고 안정된(보장된 평형점 V=0) 로직처럼 체감될 수 있으나, 다양한 딥-러닝(Deep-learning) 데이터 등 광대역의 비정형 페이로드 전송 딜레이 동기화가 필요한 첨단 ROS 복합 제어 시나리오에서는 유연성이 극히 부족한 한계를 보인다.
  • PX4-Autopilot 및 QGC/MAVSDK 환경: PX4 생태계는 동적인 변동 타임아웃 알고리즘을 소급 전제한다. 따라서 백엔드 통신 코어(LinkManager 또는 MAVSDK 내부 비동기 콜백 로직 등) 내부 깊숙이 비선형 Anti-Windup 클리핑(Clipping) 리미터 보호기가 다중 첩첩이 설치된 아키텍처 양상을 띤다. 만에 하나 통신이 급격이 단절되어 측정 RTT 오차가 무한 급수로 증가하려 해도 보호 코어 단말에서 리아프노프 안정성을 강제로 보증 받아, 예측 타임아웃은 수학적 발산을 멈추고 안전한 Ceiling(천장) 수치에 부드럽게 고정 상륙(Soft Landing)하도록 구성되었다.

6. 소스 코드 관점의 안정화 블록 처리 예시 (C++)

QGroundControl 아키텍처나 자체 커스텀 지상 관제 시스템(Custom GCS)을 구성할 때, 이러한 리아프노프 안정성을 보증하기 위한 C++ 소스 수준 추상화 코드는 아래의 방식을 채택하여 명시적 분리선(Decoupling plane) 방어막을 구축토록 지침된다.

#include <algorithm>

class LyapunovStableTimeoutController {
private:
    float _current_timeout;
    float _error_integral;

    // 절대 상하한 Safety Saturation Constants (단위: 초)
    const float MAX_SAFE_RTO = 3.0f;
    const float MIN_SAFE_RTO = 0.5f;

public:
    LyapunovStableTimeoutController(float initial_timeout) 
        : _current_timeout(initial_timeout), _error_integral(0.0f) {}

    // 피드백 오차가 주어지고 타임아웃을 업데이트하는 코어 제어 루틴
    float update_timeout_bounded(float rtt_error, float ki_gain, float dt) {
        
        // 보충하려는 오차 누적분 가상 적분 실행량 계산
        float attempted_integral = _error_integral + (rtt_error * dt);
        
        // 동적 타임아웃 재연산 (상수 Kp 분의 영향 생략 등가화)
        float attempted_timeout = _current_timeout + (ki_gain * attempted_integral);

        // [핵심] 리아프노프 상한 안정성 판별 방어벽 가동 (Anti-Windup)
        // 1. 발산(Divergence) 상단 클리핑
        if (attempted_timeout > MAX_SAFE_RTO) {
            _current_timeout = MAX_SAFE_RTO;
            // 리아프노프 에너지 발산을 막기 위해 적분기 분리 멈춤
            // _error_integral 갱신을 생략(Clamp)하여 dV/dt가 양수가 됨을 저술
        } 
        // 2. 수축(Shrink) 하단 클리핑
        else if (attempted_timeout < MIN_SAFE_RTO) {
            _current_timeout = MIN_SAFE_RTO;
            // 적분기가 음수 무한대로 이탈하는 것을 보호 거부
        } 
        // 3. 안정 평형 궤도 영역 (Lyapunov Safe Region)
        else {
            _current_timeout = attempted_timeout;
            _error_integral = attempted_integral; // 상태 변수 허가 궤적
        }

        return _current_timeout;
    }
};

상단 로직의 주요 핵심은, 시도되는 타임아웃 변화폭(attempted_timeout)이 사전 지정된 임계 방어선 규격(MAX_SAFE_RTO)을 상회하며 폭주하려 들 때, 가차 없이 그 수치를 한계점 스티로폼(Ceiling)에 맞대어 잘라버림(Clamp)과 동시에, 에러 누적 변수(_error_integral)의 메모리 업데이트 진행 전개 자체를 차단(Decoupling block)해버린다는 점에 있다. 이를 통해 적분기가 양의 무한대로 감겨 들어가는 윈드업(Windup)을 방지하고 상태 미분값이 점근적으로 음수로 유지되게 하여 수리적 시스템 안정성을 이끌어 낸다.

7. 결론 사항 정리

무인항공기 제어체계 설계자들은 단순히 패킷이 도달하지 않으면 명령을 한 번 더 보내는 1차원적인 논리에서 근본적으로 탈피해야 한다. PX4-Autopilot과 같은 현대의 고도화된 실시간 제어 스택에서, 통신 타임아웃이란 네트워크 지연 오차(Error)를 입력받아 제어 반응(Reaction Timeout Threshold)을 수시 재도출 하는 명백한 하나의 “비선형 폐루프 동역학 시스템(Nonlinear Closed-loop Dynamics System)“이다.

따라서 알고리즘 설계자는 이 논리 시스템이 임의의 노이즈 폭발 외란에도 스스로 평형점을 잃고 발산 분해되어 버리는 끔찍한 현상을 방어하기 위해, 수학적 리아프노프 안정성(Lyapunov Stability) 보증 경계 제한법(Bounding Logic Method)을 철학적 뼈대로 도입 증명하고, 코드 레벨에서 이를 철통같이 클램핑(Clamping)하여 어떠한 환경에서도 시스템 멈춤 상태(Deadlock)에 직면하지 않는 견고한 GCS 통신 아키텍처 환경을 사전에 정초(Foundation) 마련하라.
