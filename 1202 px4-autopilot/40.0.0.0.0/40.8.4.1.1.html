40.8.4.1.1 통신 주기 공진(Resonance)에 의한 응답 지연 이상치(Outlier) 필터링 임계값 설정

1. 개요

일반적으로 GCS(지상 관제 시스템)인 QGroundControl(v4.3)과 비행 제어기 PX4-Autopilot(v1.14) 간의 통신은 각자의 고유한 운영 주기(Operating Frequency)를 따른다. 이 과정에서 QGroundControl 명령의 주기적인 송출 패턴과 PX4-Autopilot의 내부 uORB 메시지 처리(혹은 NuttX RTOS 스케줄러) 주기가 특정한 정수비 형성을 통해 간섭할 경우, 기계적 공명과 같은 통신 주기 공진(Communication Resonance) 현상이 발생할 수 있다.

이 공진 현상의 대표적인 결과물은 주기적으로 통신 버퍼나 큐(Queue)에 트래픽이 집중되면서 생겨나는 응답 지연의 극단적 이상치(Latency Outlier) 이다. 이러한 이상치를 일반적인 네트워크 단절이나 하드웨어 결함으로 오인하여 Failsafe(안전 복구 시스템)를 가동하게 되면 불필요한 비행 중단을 초래할 수 있다. 따라서 공진에 의한 지연 스파이크(Spike) 현상의 통계적 특성을 이해하고, 이를 여과(Filtering)할 수 있는 수학적인 임계값 설정 기법을 면밀히 다루어야 한다.

2. 공진 현상에 의한 이상치(Outlier) 발생 메커니즘

통신 주기 공진은 물리학에서의 ‘맥노리(Beat)’ 현상과 매우 유사한 수학적 기반 위에서 동작한다. GCS에서 MAVLink 텔레메트리 요청을 보내는 주기를 f_{\text{GCS}}, PX4-Autopilot 보드 내에서의 특정 인터페이스(예: 직렬 포트 DMA 인터럽트 또는 uORB 토픽 폴링)의 운영 주기를 f_{\text{PX4}}라 하자.

두 주기의 위상(Phase)이 시간이 지남에 따라 교차하게 되면, 패킷 처리 오버헤드가 겹치는 지점에서 병목이 발생하여 지연 시간이 순간적으로 증폭되는 \Delta f = \vert f_{\text{GCS}} - f_{\text{PX4}} \vert 의 율동을 띠게 된다.

  • Ardupilot 대비 차이점: Ardupilot은 특정 주기로 실행되는 스케줄링 루프 테이블 내에서 동작하므로, I/O 스레드의 지연이 메인 제어 루프를 밀어내어 비교적 직관적인 주기 지연으로 발현된다.
  • PX4-Autopilot 아키텍처의 한계: RTOS 기반으로 I/O 처리와 고수준 비행 제어가 서로 다른 비동기 스레드에서 돌아가므로, 특정 백그라운드 태스크(예: SDCARD 로깅 부하)와 MAVLink 라우팅 모듈 사이에 위상 간섭이 생겼을 때 매우 극단적이고 비정형적인 지터(Jitter) 스파이크(이상치)를 내뿜게 된다.

3. 이상치(Outlier) 필터링을 위한 수학적 접근: 햄펄 필터(Hampel Filter)

공진에 의해 주기적으로 발생하는 극단적인 스파이크 노이즈를 식별하고 걸러내려면 고전적인 3-시그마(3\sigma) 규칙을 적용하기 어렵다. 이상치의 크기가 너무 크면 표본 평균과 표준편차 자체를 심각하게 왜곡하기 때문이다. 이럴 때는 강건한(Robust) 통계 기법인 중앙값 절대 편차(Median Absolute Deviation, MAD) 기반의 햄펄 필터 알고리즘을 사용하라.

n 개의 MAVLink 응답 지연 표본 집합 X = \{X_1, X_2, \ldots, X_n\} 에 대해 MAD는 다음과 같이 정의된다.

\text{MAD} = \operatorname{median}(\vert X_i - \operatorname{median}(X) \vert)

이 MAD 값을 바탕으로 추정된 표준편차 산포도는 \sigma_{\text{est}} \approx 1.4826 \times \text{MAD} 이다.
특정 응답 지연 패킷 X_i가 통신 공진에 의한 이상치로 판별되기 위한 필터링 임계 조건은 아래와 같다.

\vert X_i - \operatorname{median}(X) \vert > k \cdot \text{MAD}

일반적으로 k의 값은 3 (약 3\sigma에 해당) 또는 그 이상으로 설정하여, 그 구간을 넘어서는 지연은 “물리적 단절이 아니라 내부 프로세스 공진에 의한 일시적 이상치“로 간주하고 무시(Bypass)하거나 재시도 로직을 보류하도록 GCS를 설계하라.

5. 지연 시간 필터링 알고리즘 예제 (Python 기반)

QGroundControl 내부의 백엔드(C++) 모듈을 개조하기 전에, 오프라인 시뮬레이션 데이터를 바탕으로 햄펄 필터(Hampel Filter) 기반 임계값을 설정하는 프로토타이핑을 실시하라.

import numpy as np
import matplotlib.pyplot as plt

def hampel_filter_outliers(data, window_size=50, k=3.0):
    """
    MAD 기반 Hampel Filter 알고리즘 구현
    """
    n = len(data)
    outlier_indices = []
    threshold_upper = []
    
    for i in range(n):
        # 윈도우 범위 설정 (경계값 처리 포함)
        start_idx = max(0, i - window_size)
        end_idx = min(n, i + window_size)
        window_data = data[start_idx:end_idx]
        
        # 중앙값 및 MAD 계산
        median_val = np.median(window_data)
        mad = np.median(np.abs(window_data - median_val))
        
        # 통계적 임계값 상한선 계산
        upper_limit = median_val + (k * mad)
        threshold_upper.append(upper_limit)
        
        # 이상치 검출
        if data[i] > upper_limit:
            outlier_indices.append(i)
            
    return outlier_indices, threshold_upper

# 1. 시뮬레이션 데이터: 정상 지연(약 20ms)에 공진 이상치(150+ ms) 혼합
np.random.seed(42)
time_series_delay = np.random.normal(loc=20.0, scale=2.0, size=500)
# 공진 스파이크 주입
resonance_spikes = [50, 150, 250, 350, 450]
time_series_delay[resonance_spikes] = [180, 210, 160, 240, 190]

# 2. Hampel 필터 적용 (파라미터: Window 20, k=4)
outliers, upper_thresholds = hampel_filter_outliers(time_series_delay, window_size=20, k=4.0)

# 3. 결과 시각화
plt.figure(figsize=(12, 5))
plt.plot(time_series_delay, label='Raw Latency Data', color='gray', alpha=0.7)
plt.plot(upper_thresholds, label='Hampel Threshold (Upper)', color='red', linestyle='dashed')
plt.scatter(outliers, time_series_delay[outliers], color='blue', label='Resonance Outliers', zorder=5)

plt.title("MAVLink Response Latency Filtering using Hampel Filter")
plt.xlabel("Packet Sequence")
plt.ylabel("Latency (ms)")
plt.legend()
plt.grid(True)
plt.show()

6. 결론

공진 현상에 의한 극단적인 통신 지연 스파이크는 물리적 단절이 아님에도 불구하고 관제 소프트웨어가 통신 실패로 오판하게 만드는 가장 큰 주범이다. 중앙값 절대 편차(MAD)를 이용한 유연하고 강건한 통계적 임계값 설정을 QGroundControl의 타임아웃 처리 로직에 적용하면, 기계적 공명으로 인한 가짜 실패(False Positive) 신호를 지능적으로 무시할 수 있으며 전체 무인항공기 관제 시스템의 안정성을 극대화할 수 있다.
