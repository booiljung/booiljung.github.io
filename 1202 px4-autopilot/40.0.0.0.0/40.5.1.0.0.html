40.5.1. 비대칭 라우팅(Asymmetric Routing) 하의 비잔틴 장군 문제(Byzantine Generals Problem) 완화

현대의 무인기 군집 제어(Swarm Control) 망 구조는 다수의 지상 관제 시스템(GCS), 중계 장비, 그리고 통신 노드로 참여하는 여러 드론들이 메쉬(Mesh) 형태로 연결된 복합 아키텍처를 이룬다. 이러한 환경에서는 송신 경로와 수신 경로가 달라지는 비대칭 라우팅(Asymmetric Routing)이 빈번하게 발생하며, 일부 노드가 물리적/소프트웨어적 결함으로 인해 왜곡된(Corrupted) MAVLink 제어 패킷을 전달할 가능성이 존재한다. 이처럼 시스템 내 일부 노드가 배반하거나 오동작하는 상황에서 올바른 합의에 도달해야 하는 난제를 분산 컴퓨팅의 고전인 비잔틴 장군 문제(Byzantine Generals Problem)로 정의한다. 본 절에서는 PX4-Autopilot 기반의 군집 관제 네트워크에서 이 난제를 어떻게 완화(Mitigation)하는지 통신 프로토콜 수준에서 정밀하게 분석한다.

1. MAVLink 데이터 링크에서의 비잔틴 오작동 환경 분석

군집 비행 중인 무인기 네트워크(예: ROS2 uXRCE-DDS가 탑재된 다기종 드론망)에서 특정 중계 드론이 전파 간섭, 우주 방사선(Single-Event Upset), 또는 해킹으로 인해 내부 로직이 훼손되었다고 가정하자.

정상적인 GCS는 “모든 드론은 현재 고도를 유지(Hold)하라“고 COMMAND_LONG 메시지를 브로드캐스트했다. 그러나 결함이 발생한 비잔틴 중계 노드는 이 메시지 내용을 악의적으로 조작하거나 라우팅 지연으로 인해 과거의 “귀환(RTL)” 명령으로 바꿔치기하여 전파한다.
심지어 무선 채널의 페이딩(Fading)으로 송신 링크와 수신 링크의 특성이 일치하지 않는 비대칭 라우팅 구조 하에서는, 특정 드론이 정상 GCS의 메시지는 수신하지 못하고 비잔틴 노드가 뿌리는 오염된 명령만 편파적으로 수신할 위험이 커진다.

아무런 방어 기재가 없는 초창기 드론 시스템이라면, 일부 기체가 대형(Formation)을 이탈하여 귀환을 시도함으로써 상호 공중 충돌(Mid-air Collision)이라는 재앙적인 결과를 초래하게 된다.

2. PX4-Autopilot의 암호학적 서명(Cryptographic Signature) 방어

이러한 악의적(또는 오류가 있는) 노드의 메시지 변조 공격(Byzantine Attack)에 대한 1선 방어는 통신 페이로드의 무결성(Integrity) 및 인증(Authentication)을 확보하는 것이다. PX4-Autopilot은 MAVLink v2 규격부터 도입된 메시지 서명(Message Signature) 기능을 지원한다.

MAVLink v2 서명 패킷은 다음과 같이 구성된다.

  • Timestamp (48-bit): 재생 공격(Replay Attack) 방지를 위한 절대적 시간 인덱스.
  • Link ID (8-bit): 메시지가 전송되는 물리적 채널 식별자.
  • Signature (48-bit): 메시지 페이로드 전체와 비밀키(Secret Key), Timestamp, Link ID를 조합하여 SHA-256 해시 알고리즘을 수행한 후 앞부분 6바이트를 잘라낸 암호학적 서명.

PX4 기체가 특정 COMMAND_LONG 메시지를 수신했을 때, 내부의 mavlink_receiver 컴포넌트는 해당 서명을 검증한다. 만약 비잔틴 노드가 페이로드의 모드(Mode) 필드를 조작하여 라우팅(Routing)했다 하더라도, 비밀키를 알지 못하는 이상 유효한 서명을 재생성할 수 없으므로 해당 메시지는 즉각 폐기된다. 이는 수학적 증명을 통해 “거짓 정보를 전파하는 장군“을 시스템 상태 변경에서 배제하는 직접적이고 강력한 방법이다.

3. Quorum(정족수) 기반 다중 링크 투표 알고리즘

서명 계층이 패킷 훼손은 막아주지만, 비잔틴 노드가 단지 메시지를 고의로 누락(Drop)하거나 무작위로 지연(Delay)시켜 시스템을 마비시키는 서비스 거부(Denial of Service) 형태의 배반은 막기 어렵다. 특히 가용 채널에 편향이 발생하는 비대칭 라우팅 모델에서는, 기체가 단일 통신 링크(가령, 900MHz Telemetry 1개)에만 의존해서는 진실을 판별할 수 없다.

따라서 고도의 생존성이 보장되어야 하는 관제 아키텍처에서는 다중 링크(Redundant Links)를 채택한다 (예: Microhard 라디오 1기, 4G/LTE 모뎀 1기, SATCOM 1기 병렬 구성).

비잔틴 실패 한계점(Byzantine Fault Tolerance) 규칙에 따라, 시스템 내에 f개의 결함 노드가 존재할 때 올바른 합의에 도달하기 위해서는 최소 3f + 1개의 독립적인 정보 경로가 요구된다. PX4의 커스텀 통합 모듈이나 컴패니언 컴퓨터 상의 관제 브로커는 수신된 동일 명령(Sequence ID 및 Timestamp 기준)을 수 초간 결함 허용 버퍼(Fault-Tolerance Buffer)에 저장한 뒤, 다수결(Majority Vote) 절차로 최종 모드 변경을 승인한다. 위 다이어그램처럼 지상 통신망(LTE, RF)에서 모두 Hold를 지시한다면, 메쉬 망 하나가 RTL을 외치더라도 이를 비잔틴 찌꺼기로 취급하여 필터링한다.

4. QGroundControl 측의 비잔틴 완화 전략

GCS인 QGroundControl(v4.3 기준) 측에서도 여러 기체가 보내오는 HEARTBEAT 및 시스템 상태(SYS_STATUS) 메시지가 얽힐 때 비일관성을 억제해야 한다. 비대칭 라우팅 환경으로 인해 특정 드론의 상태 메시지는 수신(Rx)되지만 해당 드론으로의 송신(Tx)은 불가능한 상황이 생길 수 있다.

이때 QGroundControl은 해당 기체와의 통신 상태를 Link Active에서 Half-Duplex Degradation 또는 간헐적 연결(Intermittent Connection) 상태로 강등시킨다. 즉, 수신되는 정보만으로 상황판(UI Map)은 업데이트하되, 해당 노드를 통해 다른 드론으로 메시지를 라우팅하는 오토 라우팅(Auto-routing) 테이블에서는 즉각 제외시킨다. 이는 신뢰할 수 없는 “망가진 장군“을 정보 릴레이 체인에서 신속히 격리(Isolation)시켜 전체 네트워크의 혼돈 확산을 막는 예방 조치이다.

결론적으로 분산 드론망에서의 비잔틴 오작동은 단순히 하드웨어의 결함(Fault)을 넘어 전체 군집의 자멸을 초래할 수 있는 심각한 위험 인자이다. 개발자는 PX4의 MAVLink 서명 기능과 다중 링크 쿼럼(Quorum), 그리고 ROS2 수준에서의 비동기 검증 아키텍처를 결합하여 이 난제를 제어 공학의 범주 내에 통제 가능한(Mitigation) 상태로 묶어두어야 한다.
