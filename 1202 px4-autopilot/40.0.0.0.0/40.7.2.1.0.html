40.7.2.1. 마이클-스콧 큐(Michael-Scott Queue) 구조의 Compare-And-Swap (CAS) 원자적 연산 무결성 증명

고주파수 텔레메트리(Telemetry)를 실시간으로 브로드캐스팅하는 PX4-Autopilot의 uORB 아키텍처나 지상 관제 시스템(GCS)의 비동기 네트워크 수신단에서는 다중 생산자-다중 소비자(MPMC, Multi-Producer Multi-Consumer) 환경의 큐잉 모델이 빈번하게 요구된다. 본 절에서는 1996년 Maged M. Michael과 Michael L. Scott이 제안하여 현재까지도 락-프리(Lock-Free) 동시성 제어의 산업 표준으로 채택되고 있는 **마이클-스콧 큐(Michael-Scott Queue)**의 구조와, 그 핵심인 Compare-And-Swap(CAS) 명령을 활용한 원자적 연산 무결성 증명(Atomic Muti-thread Integrity Proof) 과정을 수학적이고 체계적으로 분석한다.

1. 다중 생산자-다중 소비자(MPMC) 환경의 큐잉 과제

PX4 통신 모듈(예: mavlink_receiver.cpp)이 다수의 시리얼 포트(UART, USB, UDP)로부터 MAVLink 패킷을 동시에 수신하여 전역 로깅 큐(Global Logging Queue)에 삽입(Enqueue)하려는 상황을 가정하라.

  • 문제점: 동시에 여러 스레드가 연결 리스트(Linked List) 기반 큐의 Tail 포인터를 수정할 경우, 노드가 유실되거나 무한 푸프에 빠질 위험이 존재한다.
  • 블로킹 큐의 한계: std::mutex 기반의 블로킹 큐를 사용하면 락 경합(Lock Contention)으로 인해 패킷 파싱 라우틴이 지연(Blocked)되어 드론의 관제 실시간성이 파괴될 수 있다.

이에 따라, 하드웨어에서 지원하는 CAS 연산을 두 번 수행(Two-step CAS)함으로써 락킹 과정 없이 데이터 일관성을 무결하게 유지하는 마이클-스콧 큐 아키텍처가 필수적으로 적용된다.

2. 마이클-스콧 큐(Michael-Scott Queue)의 구조 및 알고리즘

이 자료구조는 단방향 연결 리스트(Singly Linked List)를 기반으로 하며, 항상 큐의 맨 앞과 맨 뒤를 가리키는 두 개의 노드 포인터(Head와 Tail)를 관리한다. 초기화 시 빈 공간을 나타내는 더미 노드(Dummy Node) 하나를 생성하여 Head와 Tail이 동시에 이를 가리키게 함으로써, 삽입과 삭제 시 발생할 수 있는 널 포인터(Null Pointer) 예외 처리 로직을 우아하게 제거한다.

2.1 Enqueue (삽입 연산)의 CAS 무결성 논리 구조

새로운 노드 N_{new}를 큐의 꼬리(Tail)에 삽입하기 위해, 마이클-스콧 큐는 다음 두 단계의 CAS 연산을 순차적으로 수행한다. 이 투스텝(Two-Step) 메커니즘이 무결성의 핵심이다.

  1. 링크 연결 (Link CAS): 현재 Tail이 가리키는 노드의 next 포인터가 null인지 확인하고, 맞다면 이를 N_{new}를 가리키게 CAS를 이용해 원자적으로 변경한다. 만약 실패한다면 다른 스레드가 먼저 삽입했으므로 재시도한다.
  2. 꼬리 이동 (Tail CAS): 성공 시 큐의 전역 Tail 포인터가 방금 연결한 N_{new}를 가리키도록 또 한 번의 CAS 연산을 통해 업데이트한다.
graph TD
    subgraph State 1: Before Enqueue
        H1[Head] --> D1((Dummy))
        T1[Tail] --> N1((Node A))
        D1 --> N1
        N1 --> Null1[Null]
    end
    
    subgraph State 2: After Link CAS Success
        H2[Head] --> D2((Dummy))
        T2[Tail] --> N2((Node A))
        D2 --> N2
        N2 -->|CAS Success| N3((Node B: New))
        N3 --> Null2[Null]
    end
    
    subgraph State 3: After Tail CAS Success
        H3[Head] --> D3((Dummy))
        D3 --> N4((Node A))
        N4 --> N5((Node B))
        T3[Tail] -.->|Wait/In Progress| N5
        T3 -->|Tail CAS Success| N5
        N5 --> Null3[Null]
    end

만약 스레드 T1이 Link CAS를 성공한 직후 스케줄러에 의해 선점되어 중단(Preempted)되더라도, 스레드 T2가 삽입을 시도할 때 Tail->next가 널이 아님을 감지하고, 대신 앞선 스레드 T1이 미처 수행하지 못한 Tail CAS(꼬리 이동) 작업을 도와준(Help) 후 자신의 삽입을 재시도한다. 이것이 장애 면역(Obstruction-free)을 넘어선 진정한 **락-프리(Lock-Free)**의 진척(Progress) 보장성이다.

3. 원자적 연산의 수학적 무결성 증명

이 알고리즘이 큐의 불변성(Invariants)을 위반하지 않음을 수학적 논리로 증명한다. 큐의 상태 벡터 \mathbf{S} 가 무결성을 띠기 위해서는 다음 명제들이 성립해야 한다.

  1. 명제 1: 큐 노드는 항상 연결 리스트 형태를 유지하며 중간에 단절(Disconnection)이 일어나지 않는다.
  2. 명제 2: Head 포인터는 항상 살아있는 첫 번째 원소(또는 더미)를 가리킨다.
  3. 명제 3: Tail 포인터는 항상 마지막 노드 혹은 마지막에서 바로 직전 노드(최대 하나 차이)를 가리킨다.

명제 3의 증명:
임의의 시간 t에 스레드 T가 Enqueue를 시도한다고 하자.
Tail이 배열의 참된 최하단 노드 N_{last}를 가리킨다고 가정하면, N_{last}.\text{next} == \text{null} 이다. 이때 T_1이 CAS(&N_{last}.next, null, N_{new})에 성공한다면, 이제 Tail 포인터는 마지막에서 두 번째 노드를 가리키는 상태가 된다. 즉 거리가 정확히 1칸 떨어진다.
그 직후 어떠한 스레드(자신 포함)든 Tail의 위치와 Tail.next의 널 여부를 검사하게 되며, 널이 아닌 것을 감지한 즉시 CAS(&Tail, N_{last}, N_{new})를 수행하여 Tail을 다시 마지막 위치로 복구시킨다. 따라서 분산 시스템 내 모든 지연 시간에 대해 \text{Dist}(\text{Tail}, \text{Actual Last Node}) \le 1 이 수학적으로 보장된다.

4. QGroundControl 및 PX4 관제 아키텍처 실무 설계 가이드

ROS2 미들웨어나 QGroundControl 백엔드 구조에서 이런 다중 컨슈머 통신을 락-프리로 설계할 경우, 마이클-스콧 큐는 안정성의 바이블로 취급된다. 그러나 실제 하드웨어 아키텍처로 컴파일할 땐 아래 C++11/14 메모리 모델 기반 최적화를 고려해야 한다.

  • std::memory_order_acquire 및 release: x86 계열은 강한 메모리 순서(Strong Memory Ordering)를 가지나 PX4가 구동되는 ARM Cortex-M 계열은 약한 메모리 순서(Weak Memory Ordering) 모델이다. CAS 루프 내부에 명시적인 메모리 장벽(Memory Barrier, release/acquire 시맨틱)을 둬 무결성 캐시 동기화를 유도하라.
  • 이 구조체는 메모리 누수를 극복하기 위해 본질적으로 ABA 문제(ABA Problem)라는 함정을 내포하고 있다. 이를 완벽하게 무결점으로 보완하기 위해서는 다음 절에서 서술할 해저드 포인터(Hazard Pointer) 시스템 등 생명주기 제어 기법과의 통합 기획이 절대적으로 수반되어야 한다.
