40.2.3.1.1. 거절(DENIED) 및 보류(TEMPORARILY_REJECTED) 상태의 흡수 마르코프 연쇄(Absorbing Markov Chain) 모델링

비행 제어 시스템(Flight Control System)에서 지상 관제 시스템(GCS)이 하달한 명령어(예: 이륙, 임무 시작, 모드 변경)가 항상 성공적으로 수용되는 것은 아니다. 특히 수많은 센서의 정렬 상태와 GPS 3D Fix 여부, 배터리 임계치 등을 동시다발적으로 판단해야 하는 PX4-Autopilot의 커맨더(Commander) 아키텍처는 조건이 엄격하게 충족되지 않을 때 MAV_RESULT_DENIED 또는 MAV_RESULT_TEMPORARILY_REJECTED를 반환한다. 본 절에서는 이 두 가지 예외 응답 상태를 **흡수 마르코프 연쇄(Absorbing Markov Chain)**로 수학적 모델링하여, 시스템 백엔드가 무한 재시도(Infinite Retry) 오류에 빠지지 않고 확정적으로 루프를 탈출(Escape)하는 메커니즘을 증명한다.

1. 흡수 마르코프 연쇄(Absorbing Markov Chain)의 수학적 기초

마르코프 연쇄(Markov Chain)에서 특정 상태 S_i에 진입한 후 다른 어떤 상태로도 전이(Transition)할 수 없는 확률이 1인 경우(P_{ii} = 1), 이를 **흡수 상태(Absorbing State)**라고 부른다. 하나 이상의 흡수 상태를 가지며, 모든 비흡수 상태(Transient State)에서 최종적으로 어떤 흡수 상태로든 도달할 가능성이 존재하는 체인을 흡수 마르코프 연쇄라 한다.

QGroundControl과 같은 GCS의 제어 명령 재시도(Retry) 로직은 바로 이 흡수 마르코프 연쇄의 정규형 행렬(Canonical Form Matrix)로 표현된다.
전이 확률 행렬 P를 블록 행렬(Block Matrix) 형태로 구성하면 다음과 같다.

P = \begin{pmatrix} Q & R \\ 0 & I \end{pmatrix}

  • Q: 비흡수 상태(예: CMD_SEND, TEMPORARILY_REJECTED, TIMEOUT) 간의 전이 확률 행렬
  • R: 비흡수 상태에서 흡수 상태로 전이될 확률 행렬
  • I: 흡수 상태(예: ACCEPTED, DENIED, FAILED)의 단위 행렬
  • 0: 영행렬(Zero Matrix)

수학적으로, 기본 행렬(Fundamental Matrix) N = (I - Q)^{-1}의 각 원소 n_{ij}는 시스템이 흡수 상태로 블랙홀처럼 빨려 들어가기 전까지 비흡수 상태 j에 머무는 평균 기대 횟수(Expected Number of Visits)를 의미한다. 즉, GCS가 비행 제어기로 핑을 보내고 거절당하는 평균 재시도 회수 최적치가 이 역행렬에서 도출된다.

2. 거절(DENIED) 상태의 영구적 흡수 모델링

MAV_RESULT_DENIED (2) 응답은 커맨더 로직에서 물리적이거나 논리적인 강한 제약(Hard Constraint)이 위배되었을 때 배출된다.
예를 들어 기체가 이미 공중에 떠 있는 상황(vehicle_status.nav_state == NAVIGATION_STATE_AUTO_MISSION)에서 시동(Arming) 명령을 역으로 인가할 경우, 혹은 수동 모드 권한이 없는 비인가 RC 조종기가 오버라이드(Override)를 요청할 경우 PX4는 즉시 DENIED를 반환한다.

이 상태는 **완전한 흡수 상태(Strict Absorbing State)**이다.

  • QGC 백엔드 처리 원칙: DENIED를 수신한 GCS 스레드는 어떠한 경우에도 지수 백오프(Exponential Backoff) 기반의 재시도(Retry)를 수행해서는 안 된다. 즉시 I 행렬 궤도에 진입하여 시스템 트랜잭션을 강제 종료(Terminate)하고, 사용자(Operator) UI에 붉은색 경고 알림(Critical Alarm) 현시한 뒤 사용자 개입을 요구해야 한다.

3. 보류(TEMPORARILY_REJECTED) 상태의 회전 궤도 모델링

MAV_RESULT_TEMPORARILY_REJECTED (1) 응답은 연판정(Soft Constraint) 위배 시 발생한다.
가장 대표적인 PX4의 예시로, EKF2(Extended Kalman Filter) 추정기가 이륙 전 자기장(Magnetometer) 정렬 수렴을 완료하지 않았거나, home_position이 아직 확보되지 않은 찰나의 순간에 MAV_CMD_NAV_TAKEOFF 명령이 들어오면 커맨더는 이 보류 상태를 내뱉는다.

이 상태는 비흡수 상태(Transient State)의 자기 순환(Self-looping) 궤도로 모델링된다. 모델 상에서 이 상태는 흡수 상태가 아니며 행렬 Q 파트에 머물면서 타임아웃 타이머에 의존하게 된다.

3.1 PX4-Autopilot과 Ardupilot의 보류 상태 전이 차이

  1. PX4-Autopilot의 FSM 폴링(Polling) 정책: PX4는 GCS로 일단 TEMPORARILY_REJECTED를 쏜 뒤 해당 명령을 내부 큐에서 즉시 파기한다(Stateless). 따라서 상태를 복구하려면 **QGroundControl이 능동적으로 주기를 두고 재요청(Re-send)**을 해야만 한다.
  2. Ardupilot의 지연 수용(Deferred Acceptance) 정책: Ardupilot은 특정 MAVLink 명령어에 대해 스스로 타임아웃 버퍼 블록을 두고 잠시 기다린다. 즉, TEMPORARILY_REJECTED를 보내는 대신 1~2초간 메인 루프에서 조건을 재평가한 뒤 조건이 만족되면 늦게나마 ACCEPTED를 반환하는 IN_PROGRESS 지향적 패턴을 일부 혼용한다.

4. QGroundControl 흡수 위상 제어 흐름 (Mermaid)

다음은 GCS 백엔드의 Vehicle Command Manager 루프가 두 예외 상태를 어떻게 위상학적으로 상반되게 흡수/회전시키는지를 나타내는 상태도이다.

stateDiagram-v2
    direction LR

    state "비흡수 파트 (Transient Matrix Q)" as Transient {
        state "명령 하달 (Command Send)" as Send
        state "지수 백오프 대기 (Wait)" as Backoff
        
        Send --> Backoff : TEMPORARILY_REJECTED 수신
        Backoff --> Send : 백오프 타이머 만료
    }
    
    state "흡수 파트 (Absorbing Matrix I)" as Absorbing {
        state "명령 완전 거절 (DENIED)" as Denied
        state "타임아웃 한계 도달 (Timeout Death)" as Timeout
        state "명령 승인 (ACCEPTED)" as Accepted
    }

    Send --> Denied : DENIED 수신 (즉각 흡수)
    Send --> Accepted : ACCEPTED 수신 (정상 흡수)
    
    Backoff --> Timeout : Maximum Retry Count 초과 시 강제 흡수 전환
    
    Denied --> [*] : 사용자에게 경고 (UI Event)
    Timeout --> [*] : 연결 불안정 알람
    Accepted --> [*] : 제어 시퀀스 다음 단계 진행

도식에서 증명되듯, 비흡수 궤도에 존재하는 TEMPORARILY_REJECTED조차도 무한히 행렬 Q 내벽을 맴돌게 내버려두면 안 된다. GCS 아키텍처는 반드시 재시도 횟수 한계(Maximum Retry Limit)라는 탈출 간선(Escape Edge)을 두어, 일정한 사이클이 초과하면 강제로 Timeout Death라는 인공적 흡수 상태로 시스템을 멱살 잡아 끌어내려야 스레드 행(Thread Hang) 및 데드락(Deadlock)을 방지할 수 있다.

5. 결론

무인 항공기 관제 애플리케이션 개발에 있어 MAVLink 명령 전송 시스템을 단순한 절차적 프로그래밍(Procedural Programming) 시각으로 접근하는 것은 지양해야 한다. DENIED와 TEMPORARILY_REJECTED는 본질적으로 수학적 거동이 완전히 다른 상태공간(State Space)의 텐서(Tensor)들이다.
PX4 소프트웨어 및 QGroundControl 연동 구조를 튜닝할 때는, 이 두 상태를 흡수 마르코프 연쇄의 기본 행렬(Fundamental Matrix) 개념으로 엄격히 분리하여 모델링하라. DENIED에는 자비 없는 루프의 완전 종결을, TEMPORARILY_REJECTED에는 수학적 수렴 기대치 기반의 유한한 지수 백오프 회전을 부여함으로써, 드론 관제 컴퓨터의 I/O 비동기 메인스레드가 런타임 함정에 빠지지 않도록 견고하게 설계하여야 한다.
