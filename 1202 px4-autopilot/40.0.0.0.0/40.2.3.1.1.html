40.2.3.1.1. 거절(DENIED) 및 보류(TEMPORARILY_REJECTED) 상태의 흡수 마르코프 연쇄(Absorbing Markov Chain) 모델링

1. 개요(Overview)

자율 무인항공기 관제 환경에서 지상 관제 시스템(GCS)이 PX4-Autopilot으로 지속적인 제어 명령을 하달할 때, 기체의 상태는 앞서 분석한 마르코프 결정 과정(MDP)에 따라 변동한다. 그중에서도 특정 응답 상태(State)에 한 번 진입하면 외부의 강력한 개입(조종자의 강제 오버라이드나 하드웨어 리부팅 등) 없이는 다른 상태로 스스로 벗어날 수 없는 수학적 위상을 **흡수 상태(Absorbing State)**라고 한다.
MAVLink 통신의 응답 코드 체계에서 **MAV_RESULT_DENIED**와 **MAV_RESULT_TEMPORARILY_REJECTED**는 각각 완전한 흡수 상태와 일시적(조건부) 흡수 상태의 성질을 갖는다. 본 문서에서는 이 두 가지 응답 상태가 GCS와 기체 시스템을 어떻게 루프에 가두고 붕괴시키는지 흡수 마르코프 연쇄(Absorbing Markov Chain) 이론을 빌려 소스코드 레벨에서 분석한다.

2. 흡수 마르코프 연쇄의 수학적 정의 및 적용

마르코프 연쇄 모형에서 다음의 두 가지 조건을 만족하면 흡수 마르코프 연쇄라 부른다.

  1. 적어도 하나 이상의 흡수 상태(P_{ii} = 1, 자기 자신으로 전이할 확률이 100%)가 존재한다.
  2. 모든 비흡수 상태(일시적 상태, Transient State)에서 적어도 하나의 흡수 상태로 도달할 수 있는 경로가 존재한다.

PX4와 QGroundControl 간의 상태 통신을 표준형 전이 확률 행렬 P로 모델링하면 다음과 같은 구조를 띈다.

P = \begin{pmatrix} Q & R \\ 0 & I \end{pmatrix}

  • Q: IN_PROGRESS, ACCEPTED 등 명령이 수행 중이거나 재전송을 요하는 일시적(Transient) 상태 간의 전이 행렬
  • R: 일시적 상태에서 흡수 상태(DENIED)로 빠지는 전이 확률
  • 0: 흡수 상태에서 일시적 상태로 빠져나올 수 없음을 나타내는 제로 행렬
  • I: 흡수 상태가 스스로 루프를 도는 항등 행렬 단위 (Self-loop)

3. 거절(DENIED) 상태: 강한 흡수성(Strong Absorbing State)

MAV_RESULT_DENIED 피드백이 떨어지는 순간은 시스템이 **완전 흡수 연쇄(Perfectly Absorbing Chain)**에 도달했음을 의미한다.

3.1. PX4 내부 발생 메커니즘

PX4(v1.14.0)의 src/modules/commander/Commander.cpp 로직에서 시동(Arm) 명령이 인가되었을 때, Pre-flight Check (IMU 캘리브레이션 불량, GPS 수신 불가 등)에서 하드 록(Hard Lock)이 걸린 경우가 이에 해당한다.
기체는 DENIED를 반환하고 자체 FSM의 현재 위치에 멈춰버리며, 더 이상의 상태 전이를 거부한다.

3.2. QGroundControl 백엔드의 반응

QGroundControl(v4.3)의 C++ 백엔드(차량 매니저 스레드) 역시 DENIED를 수신하면 명령 대기 큐(Queue)를 강제로 클리어(Flush)하고 관련된 백그라운드 재시도 로직을 모두 파기(Abort) 처리한다. UI 상에는 적색 에러 팝업을 발생시키며, 조종자가 원천적인 물리적 에러를 수정하기 전까지 이 자동화 루프는 철저히 정지된 채 항등 행렬 I 수준의 자가 루핑 모드로 돌입하게 된다.

4. 보류(TEMPORARILY_REJECTED) 상태: 약한 흡수성 및 시한부 전이

반면 MAV_RESULT_TEMPORARILY_REJECTED는 수학적으로 준 흡수 상태(Semi-absorbing State) 또는 특정 횟수 제한 안에서만 도는 워시아웃 루프(Washout Loop)로 모델링된다.

4.1. 발생 및 메커니즘 차이

기체가 ACCEL/GYRO 캘리브레이션을 진행 중이거나 플래시 메모리에 데이터를 기록하는 중(I/O Blocking)에 조종자가 비행 모드 변경을 요청할 때 발생한다.
명령 자체의 논리적 모순은 없으므로 PX4 측은 “잠시 후 다시 시도하라“는 의미의 거절을 반환한다. Ardupilot의 단일 제어 루프 아키텍처는 내부 지연 요소를 짧게 가져가려는 성향이 짙어 일시적 지연을 최소화하지만, PX4는 uORB 워크큐(Work Queue) 스케줄링 특성 우선도에 따라 동적으로 일시 거절 응답이 도출될 확률 공간이 넓다.

4.2. GCS의 재시도(Retry) 행렬 모델

QGC는 이 코드를 수신할 경우 흡수 상태로 인식하지 않고 Q 행렬 내의 일시적 상태 변이로 인식하여 일정 타임아웃 딜레이 후 재전송을 실시한다.
하지만 재시도 횟수(최대 3~5회)를 초과할 때까지 시스템이 병목 현상을 해소하지 못한다면, GCS 측에서 자체적으로 **임의 타임아웃 흡수 상태(Time-out Induced Absorbing State)**로 강제 붕괴시켜 DENIED와 동일한 레벨로 처리 파이프라인을 종료시킨다. 즉, 무한정 루프를 도는 것을 방지하기 위해 R 공간으로 시스템을 던져버리는 것이다.

5. 흡수 마르코프 연쇄 기반 전이도(Mermaid)

다음은 GCS 명령어 전송 시 일어나는 과도적 상태와 최종 흡수 상태로의 전이 경로를 나타낸 위상 다이어그램이다.

stateDiagram-v2
    direction LR
    
    state "명령 발송\n(Action)" as Cmd
    
    state "일시적 궤적 공간 (Transient Space - Q)" as Transient {
        state "IN_PROGRESS" as Inp
        state "TEMP_REJECTED" as T_Rej
    }
    
    state "흡수 공간 (Absorbing Space - I)" as Absorbing {
        state "DENIED" as Den
        state "ACCEPTED" as Acc
    }
    
    Cmd --> Transient : 통신 시도 및 응답 대기
    Cmd --> Absorbing : 1차 적인 판단 결과 즉시 
    
    T_Rej --> Cmd : 타임아웃 이내 논리적 재전송 루프
    T_Rej --> Den : 재전송 한계 초과 시 붕괴 (Sink)
    
    Inp --> Inp : 진척률(Progress) 지속 모니터링
    Inp --> Acc : 수행 성공 완료 시 위상 정착
    Inp --> Den : 수행 중 하드웨어 결함 발생 시 추락/거절
    
    Den --> Den : 무한 자가 궤적 (조종자 개입 필수)
    Acc --> Acc : 후속 명령 대기 상태 정착

6. 결론 및 실무 제연: 기본 행렬(Fundamental Matrix) 스크립팅

임무 자율성이 고도화된 스웜(Swarm) 환경이나 독자적인 GCS 환경을 구축할 때는, 이처럼 시스템이 DENIED 상태로 흡수될 때까지 거쳐가는 “예상 소요 시간” 및 “재전송 점유율“을 예측하는 것이 매우 중요하다.
흡수 마르코프 연쇄의 기본 행렬(Fundamental Matrix) N = (I - Q)^{-1}의 각 원소는 특정 일시적 상태(예: TEMP_REJECTED)에서 출발해 최종 흡수 상태(ACCEPTED 혹은 DENIED)에 도달할 때까지 소모되는 총합 트랜잭션의 횟수를 제공한다.
따라서 GCS 소프트웨어 엔지니어는 MAVLink 백엔드 루프 최적화 단계에서 이 행렬 특성을 역순으로 설계하여, 특정 모드 변경 시나리오에서 시스템 정지가 유발되기까지 소모되는 최대 리트라이(Retry) 임계 구간을 실시간 알고리즘 스레드로 산출해내는 고스펙의 타임아웃 핸들러를 장착해야만 현장의 혹독한 조건을 이겨낼 수 있다.
