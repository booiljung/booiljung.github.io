40.1. 통신 채널의 추계학적(Stochastic) 모델링 및 정보 이론적 한계

1. 개요

무인항공기(UAV)와 지상 관제 시스템(GCS)을 연결하는 무선 데이터 링크는 대기 상태, 기체의 이동성, 지형지물의 장애물 스캐터링(Scattering) 등에 의해 시시각각 변화하는 비결정론적 특성을 지닌다. PX4-Autopilot 기반의 드론 제어 환경에서 실시간 텔레메트리(Telemetry)를 수신하고, 중요 제어 명령(Control Command)을 신뢰도 있게 기체로 전달하기 위해서는 통신 링크에 존재하는 **불확실성(Uncertainty)**을 정량화하고 극복해야 한다.
이를 위해 본 장에서는 추계학적(Stochastic) 모델링을 통하여 무선 채널의 통계적 특성을 파악하고, 클로드 섀넌(Claude Shannon)의 정보 이론(Information Theory)에 기초하여 송수신 가능한 데이터율(Data Rate)의 이론적 한계를 고찰한다.

2. 정보 이론적 한계 (Information-Theoretic Limits)

모든 디지털 무선 통신 시스템은 본질적으로 채널 대역폭(Bandwidth)과 신호 출력(Signal Power)이라는 물리적 자원 제약을 받는다.

2.1 섀넌-하틀리 정리 (Shannon-Hartley Theorem)

채널에 가우스 잡음(Additive White Gaussian Noise, AWGN)이 존재할 때, 전송 오류 없이 보낼 수 있는 최대 데이터 전송 속도인 채널 용량(Channel Capacity, C)은 다음과 같이 수식화된다.

C = B \log_2\left(1 + \frac{S}{N}\right) \quad (\text{bps})

이때 각 파라미터는 다음을 의미한다.

  • C: 달성 가능한 최대 채널 용량
  • B: 채널의 주파수 대역폭 (Hz)
  • S: 수신 신호의 전력 (Signal Power)
  • N: 수신 심볼에 더해지는 가우스 백색 잡음 전력 (Noise Power)
  • S/N: 신호 대 잡음비 (Signal-to-Noise Ratio, SNR)

2.2. PX4 MAVLink 텔레메트리 환경에의 적용

UAV 운영 시 흔히 사용되는 915MHz 또는 433MHz 대역의 SiK Telemetry Radio를 살펴보면, 가용 대역폭이 좁으며 전파 간섭이 심하다. 일반적으로 소프트웨어적으로 57600 bps 수준의 직렬(Serial) 보드 레이트(Baud Rate) 설정으로 묶여 있지만, 거리가 멀어짐에 따라 SNR이 낮아지면 샤논의 한계점 이하로 채널 용량이 저하되어, 결과적으로 패킷 드롭(Packet Drop) 및 지연 시간(Latency) 증가로 표출된다. 따라서 GCS 단에서는 이러한 SNR 마진 축소를 감지하고, MAVLink 스트림 전송률(Stream Rate)을 동적으로 낮추어 채널 용량 초과로 인한 혼잡 에러를 피해야 한다.

3. 통신 채널의 추계학적(Stochastic) 모델링

실제 UAV 비행 환경에서는 전파의 회절 및 반사에 의해 다중 경로(Multipath)가 발생하므로 단순히 AWGN 모델을 넘어서는 비정상(Non-Stationary) 추계학 모델이 요구된다.

3.1. 무선 채널 상태 전이 모델 (Gilbert-Elliott Model)

드론 비행 시 패킷 유실은 단발적으로 발생하기보다 특정 순간(장애물 뒤로 숨는 등)에 연속적으로 버스트(Burst) 에러 형태로 나타나는 경향이 강하다. 이를 수학적으로 은닉 마르코프 모델(Hidden Markov Model)의 일종인 **길버트-엘리엇 모델(Gilbert-Elliott Model)**로 묘사할 수 있다.

  • 상태 0 (Good State): 패킷 유실률이 극미한 안정적 상태 (예: LOS 환경)
  • 상태 1 (Bad State): 패킷 유실이 잦거나 통신 단절이 빈번한 상태 (예: 장애물 간섭 환경)

모델은 각 상태간 확률적 전이(Transition Probability) 매트릭스를 구성하여, 향후 N초 직후에 텔레메트리 수신이 가능할지 실패할지에 대한 기댓값을 제공한다.

4. QGroundControl 및 제어 시스템에서의 실무적 시사점

추계학적 채널 모델링과 정보 이론적 한계를 고려할 때, QGroundControl(이하 QGC) 등 GCS 애플리케이션 개발자는 다음과 같은 통신 최적화 전략을 취해야 한다.

  1. 동적 대역폭 할당(Dynamic Bandwidth Allocation): QGC는 MAVLINK_MSG_ID_RADIO_STATUS 메시지를 통해 실시간 노이즈 수준(N)과 신호 강도(S)를 평가한다. SNR이 급감하는 배드 스테이트(Bad State) 국면에서는, 파라미터 동기화(Parameter Sync) 등 비순차적이고 트래픽 양이 큰 백그라운드 요청을 일시적으로 중단하고, 고도/위치 등 핵심 제어 텔레메트리만 송신하여 채널 용량(C) 병목 현상을 타개하라.
  2. 타임아웃(Timeout) 한계 예측: 추계학적 모델링에 의해 1초 이상의 버스트 유실 확률이 통계적으로 높다면, 어설픈 잦은 재전송(Retry)은 오히려 버퍼 오버플로우와 스케줄링 지연을 야기한다. 이런 경우 지수 백오프(Exponential Backoff)를 강하게 설정해 핑거프린트 유실율을 낮추어라.
  3. 오류 정정 부호(Forward Error Correction, FEC): 샤논 한계에 근접하기 위해 링크 레이어에서 FEC를 탑재한 통신 모듈(Microhard, RFD900 등)을 도입하여, MAVLink 데이터 무결성을 하드웨어 수준에서 강화하라.
