40.1. 통신 채널의 추계학적(Stochastic) 모델링 및 정보 이론적 한계

PX4-Autopilot 기반 무인기 제어 시스템에서 지상 관제 시스템(GCS)과 기체 간의 통신망은 무선 환경 특성상 확률적(Stochastic)인 잡음(Noise)과 페이딩(Fading) 환경에 노출된다. 특히 MAVLink 프로토콜을 이용한 제어 명령과 텔레메트리 데이터 전송에 있어 채널의 불확실성을 수학적으로 모델링하여 이해하는 것은 GCS의 통신 파라미터(타임아웃, 재전송 횟수 등)를 설계하기 위한 이론적인 기초가 된다. 본 절에서는 무선 통신 채널의 추계학적 모델링 기법과 정보 이론(Information Theory)에서 제시하는 통신 성능의 한계를 다방면에서 심도 있게 검토한다.

1. MAVLink 데이터 링크의 확률적 특성 (Stochastic Properties)

드론과 GCS를 잇는 통신 링크는 시간에 따라 대역폭과 신호 대 잡음비(SNR, Signal-to-Noise Ratio)가 연속적으로 진동하는 시변 시스템(Time-varying System)이다. PX4 관점에서 텔레메트리 모듈이나 Wi-Fi/LTE 링크를 고려할 때, 채널 상태는 일반적으로 마르코프 연쇄(Markov Chain)를 기반으로 한 길버트-엘리엇 모델(Gilbert-Elliott Model)로 추상화된다.

1.1 길버트-엘리엇 모델 (Gilbert-Elliott Model)

길버트-엘리엇 모델은 통신 채널을 두 개의 상태인 ‘Good(G)’ 상태와 ‘Bad(B)’ 상태로 구분한다.

  • Good 상태: 에러 발생 확률이 매우 낮아 패킷 유실률이 임계값 이하인 상태
  • Bad 상태: 일시적 음영 구역(Shadowing)이나 간섭으로 인해 패킷 유실 확률이 매우 높은 상태

이러한 두 상태 간의 전이는 마르코프 프로세스를 따른다.

stateDiagram-v2
    direction LR
    Good: Good Channel State (Low PLR)
    Bad: Bad Channel State (High PLR)

    Good --> Good : 1 - p
    Good --> Bad : p
    Bad --> Bad : 1 - r
    Bad --> Good : r

위의 다이어그램에서 전이 확률 p와 r을 측정함으로써 통신 단절(Link Loss)의 지속 시간을 예측할 수 있다. QGroundControl은 내부 통신 관리자(Vehicle Link Manager)를 통해 연속적인 패킷 유실을 감지하고, 이 확률 모델의 파라미터에 기반하여 동적 재전송 간격을 조정하는 개념적 접근 방식을 사용하기도 한다.

2. 채널 용량(Channel Capacity)과 섀넌 한계 (Shannon Limit)

통신 채널의 정보 이론적 한계는 클로드 섀넌(Claude Shannon)의 논문인 “A Mathematical Theory of Communication“에 기반한다. AWGN(Additive White Gaussian Noise) 채널 환경을 가정할 때, 채널이 신뢰성 있게 전송할 수 있는 최대 데이터 전송률(Data Rate)인 채널 용량 C는 다음 식에 의해 정의된다.

C = B \log_2 \left( 1 + \frac{S}{N} \right)

여기서:

  • C: 채널 용량 (bits/second)
  • B: 통신 대역폭 (Hz)
  • S: 평균 수신 신호 전력 (Watts)
  • N: AWGN 잡음 전력 (Watts)

PX4 시스템의 MAVLink 스트리밍 시, 고대역폭 데이터(예: VIO용 이미지 프레임 스트림 등)를 전송하게 될 경우 데이터 생성률이 채널 용량 C에 근접하거나 이를 초과하게 된다.

2.1 섀넌 한계를 넘어서는 상황의 여파

정보 비율 R (설정된 전송률)이 용량 C에 도달하거나 초과하게 되면, 수신단인 QGC가 디코딩 과정에서 극심한 패킷 유실(Packet Loss)을 경험한다. 이는 MAVLink 패킷(특히 COMMAND_LONG 등 중요 명령) 전송을 보장하기 위한 GCS 재전송 버퍼 큐(Queue)의 포화를 유발한다.

3. 정보 이론 기반 PX4-Autopilot 통신 설계 고려사항

3.1 에러 정정 및 검출 한계 분석

MAVLink 2.0 프로토콜은 패킷 오염을 감지하기 위한 CRC(Cyclic Redundancy Check)를 사용하지만, 이는 오류를 검출(Detection)할 뿐, 정방향 오류 수정(FEC, Forward Error Correction) 기능을 갖추고 있지 않다. 따라서 정보 한계에 다다른 링크 상태에서는 패킷 유실 시 무조건적인 재접수(Retransmission)에 의존해야 한다.

GCS에서 재전송 횟수 최대치를 무작정 늘린다면, 오버헤드가 누적되어 텔레메트리 큐 자체가 블록(Block)될 수 있다. 이는 채널의 용량이 고정되어 있는 상태 시스템에서의 전형적인 큐잉 이론상의 꼬리 법칙 한계와 일치한다. 결과적으로 QGroundControl 및 PX4 측 mavlink_receiver의 세심한 타임아웃 튜닝이 필수적이다.

3.2 Ardupilot의 접근과의 상호 비교

Ardupilot 환경의 대표적 GCS인 Mission Planner 통신 아키텍처에서는 무선 링크(예: Sik Radio) 내부 프레임의 에러 교정 메커니즘을 보다 적극적으로 연동시킨다. 반면, PX4-Autopilot과 QGroundControl 조합은 응용 계층의 강건성(Application-level Robustness)에 보다 중점을 두는 설계 철학을 채택하고 있다. 이는 하부 링크 계층에 관계없이, 상위 MAVLink 계층의 타임아웃 타이머에서 정보 이론상의 에러를 포괄적으로 보정하겠다는 의미이다.

4. 요약

무인기 통신 환경 모델링에 도입되는 길버트-엘리엇과 같은 마르코프 연쇄 채널 모델과 섀넌 한계는 무선 데이터 링크를 통한 기체 제어 시 왜 필수불가결하게 ’타임아웃’과 ‘재전송’ 정책을 사용해야 하는지를 과학적으로 입증한다. GCS 제어기 설계자는 채널의 물리적 잡음(Noise) 특성이 야기하는 무작위성과 정보 이론적 용량 한계를 충분히 모델링하고, 이를 바탕으로 최적의 재전송 타임아웃 임계치와 MAVLink 스트리밍 대역폭 파라미터(예: PX4의 SER_TEL1_BAUD)를 설정해야 한다.
