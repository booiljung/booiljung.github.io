40.1.3.1 길버트-엘리엇(Gilbert-Elliott) 2상태(Two-State) 버스트 에러 모델의 점근적 특성

무인항공기 시스템(UAS)의 성공적인 자율 비행과 원격 관제를 위해서는 기체와 지상 관제 시스템(GCS)을 연결하는 무선 데이터 링크의 신뢰성이 절대적으로 보장되어야 한다. 그러나 실제 대기권 환경에서 이루어지는 RF 텔레메트리(Telemetry) 통신, 제어용 Wi-Fi 혹은 LTE/5G 네트워크는 다중 경로 페이딩(Multipath Fading), 도플러 효과, 전자기적 간섭(EMI) 등 다양한 외부 요인의 영향을 받는다. 이러한 요인들은 단일 패킷의 무작위적 손실보다는 특정한 시간 동안 오류가 연속으로 발생하는 버스트 에러(Burst Error) 현상을 일으킨다. 이를 추계학적(Stochastic)으로 정교하게 모델링하기 위해 길버트-엘리엇(Gilbert-Elliott) 2상태(Two-State) 버스트 에러 모델이 널리 도입된다.

본 절에서는 길버트-엘리엇 모델의 수학적 정의와 상태 간 전이의 점근적 특성(Asymptotic Properties)을 분석하고, 이 모델이 PX4-Autopilot 기반 통신망 설계와 페일세이프(Failsafe) 파라미터 튜닝에 어떠한 이론적 토대를 제공하는지 기술한다. (참고: MAVLink 및 제어 로직 설명에 있어서는 PX4 펌웨어 v1.14 및 QGroundControl v4.3.x 이상 환경을 기준으로 서술한다.)

1. 길버트-엘리엇 모델의 이론적 배경

길버트-엘리엇 모델은 에드거 넬슨 길버트(Edgar Nelson Gilbert, “Capacity of a Burst-Noise Channel”, 1960)와 에너지 오 앨리엇(E. O. Elliott, “Estimates of error rates for codes on burst-noise channels”, 1963)이 제안한 고전 무선 채널 모델이다. 은닉 마르코프 모델(Hidden Markov Model, HMM)의 변형으로, 데이터 링크의 품질을 ‘좋음(Good, G)’ 상태와 ‘나쁨(Bad, B)’ 상태라는 두 개의 이산적 계층(Discrete State)으로 추상화한다.

  • 좋음 상태(G, Good State): 외부 간섭이 적거나 다중 경로 페이딩의 영향에서 벗어나 신호 대 잡음비(SNR)가 충분히 확보된 상태이다. 해당 상태에서는 비트 에러율(BER)이 극도로 낮아 패킷 전송 성공 확률이 거의 1에 수렴한다.
  • 나쁨 상태(B, Bad State): 물리적 간섭이나 딥 페이딩(Deep Fading) 상황에 진입한 구간이다. 해당 상태에서는 패킷 유실률(Packet Loss Rate)이 극단적으로 높아지며, 사실상 통신 마비 구간으로 간주된다.

PX4-Autopilot의 MAVLink 로그를 분석해 보면 패킷 손실이 무작위 독립 사건(Independent Random Event)으로 발생하는 것이 아니라, 기체 회전 시 안테나 가림 현상이나 지형 장애물 통과 시 일시적으로 뭉쳐서 나타남을 확인할 수 있다. 길버트-엘리엇 모델은 이를 상태 머신(State Machine)으로 정확히 기술한다.

다음은 두 상태 간의 전이를 나타내는 마르코프 연쇄 도표를 Mermaid 다이어그램으로 도식화한 것이다.

stateDiagram-v2
    direction LR
    G: Good State (G)
    B: Bad State (B)

    G --> G : 1 - p
    G --> B : p
    B --> G : r
    B --> B : 1 - r

위 도표에서 사용된 메트릭은 다음의 통계적 전이 확률(Transition Probability)을 의미한다.

  • p: 현재 ‘좋음(G)’ 상태에서 다음 패킷 구간에 ‘나쁨(B)’ 상태로 악화될 확률
  • r: 현재 ‘나쁨(B)’ 상태에서 다음 패킷 구간에 ‘좋음(G)’ 상태로 회복될 확률
  • 1-p: ‘좋음(G)’ 상태가 지속될 확률
  • 1-r: ‘나쁨(B)’ 상태가 지속될 확률

2. 전이 확률 행렬과 점근적 상태 확률(Asymptotic State Probability)

채널이 어느 상태에 놓일 것인가에 대한 예측은 전이 확률 행렬(Transition Probability Matrix) P를 통해 전개된다.

P = \begin{bmatrix} 1 - p & p \\ r & 1 - r \end{bmatrix}

비행 시간이 장기화되어 시간 t가 무한대로 접근함에 따라, 무선 링크가 G 또는 B 상태에 놓일 정상 상태 확률(Steady-State Probability) 즉 점근적 확률 \pi = [\pi_G, \pi_B]에 수렴하게 된다. 이 벡터 \pi는 \pi = \pi P 및 \pi_G + \pi_B = 1 의 선형 대수 방정식을 풀이하여 다음 해를 갖는다.

  • 점근적으로 채널이 좋음 상태일 확률: \pi_G = \frac{r}{p + r}
  • 점근적으로 채널이 나쁨 상태일 확률: \pi_B = \frac{p}{p + r}

이러한 점근적 특성 지표 \pi_B는 GCS 운영 과정에서 데이터 링크 무효화 비율의 평균값을 근사하는 핵심 척도이다. 예를 들어, 통신 장애 회복률(r)이 감소하고 발생률(p)이 증가하면 전체 비행 시간 중 B 상태에 머무는 점근적 비중 \pi_B가 커지며, 이는 MAVLink 대역폭의 실질적 감소와 커맨드 타임아웃을 유발하는 원인으로 작용한다.

3. 연쇄 유실 구간(Burst Length)의 고찰과 PX4 파라미터의 상관관계

에러 상태 B가 계속해서 유지되는 횟수를 일종의 연속된 무전송 구간(Outage Duration)이라고 칭한다. 나쁨 상태가 k번 연속으로 발생할 확률 P(L_B = k)는 기하 분포(Geometric Distribution) 모형에 따라 산출된다.

P(L_B = k) = (1 - r)^{k-1} r

버스트 에러 지속 시간의 기댓값, 즉 평균 버스트 길이(Average Burst Length) E[L_B]는 다음과 같다.

E[L_B] = \frac{1}{r}

PX4-Autopilot 제어 시스템 적용 예시:
PX4 내의 커맨더(Commander) 모듈의 경우 COM_DL_LOSS_T 파라미터를 통해 데이터 링크 단절(Data Link Loss) 판단 기준 시간을 조정한다. 비행 현장의 RF 환경에서 측정된 E[L_B]가 2초라면, 단절 기준(Timeout) 시간을 2초와 근접하게 설정할 경우 통신이 일시적으로 끊어진 것뿐임에도 불구하고 잦은 Failsafe(예: Return 비행 모드 즉각 전환)가 발동되는 오작동이 유발될 수 있다.
따라서 버스트 길이의 점근적 최대 편차를 수용할 수 있도록 COM_DL_LOSS_T 값을 기댓값 E[L_B]보다 여유 있게 산정해야 통신의 탄력성과 시스템의 안전성을 동시에 확보할 수 있다.

4. GCS(QGroundControl) 및 Firmware 레벨 관제 철학 비교

길버트-엘리엇 모델 상에서 점근적 성능 저하가 징후로 나타날 때, 각 기종과 시스템이 취하는 대응 기조에는 철학적 차이가 존재한다.

  • PX4-Autopilot 의 엄격한 텔레메트리 관제:
    PX4는 자율 에이전트의 비행 안전을 최우선으로 취급한다. 따라서 \pi_B가 특정 궤적을 이탈하거나 E[L_B]를 초과하는 장기 유실 상태에 진입하면 보수적으로 데이터 링크가 소실된 것으로 간주한다. 시스템은 MAVLink 데이터 메시지의 seq 필드를 확인해 \frac{1}{r} 이상의 패킷이 연쇄 유실(Burst)됨을 탐지한 후, 곧바로 Offboard 제어 명령 수용을 중단하고 보류(Hold)나 귀환(Return to Launch) 로직으로 진입하게 된다.

  • Ardupilot 대비 상대적 관제 차이:
    Ardupilot 환경에서는 버스트 에러로 인한 일시적 통신 지연에 대하여 다소 유연한 마진을 허용하는 필터링 정책을 주로 사용한다. 즉 \pi_B가 단기적으로 급증하더라도 Failsafe로 넘어가는 타임루프 필터 측면에서 완만하여, 단기 딥 페이딩(Deep Fading) 시 즉각적인 RTL(Return To Launch) 트리거보다는 기존 모드를 유지하면서 패킷 재요청(Retry)이나 텔레메트리 재결합을 인내심 있게 시도하는 방식에 강점을 둔다.

  • QGroundControl 구현 측면:
    QGC(v4.3 이상)에서는 백엔드에 위치한 Vehicle 및 MAVLinkProtocol 관리 객체가 텔레메트리 스트림의 패킷 손실률(Loss Rate)을 실시간으로 추적 연산한다. 이 손실 메트릭이 길버트-엘리엇 모델의 점근적 한계치를 초과할 시, 프론트엔드 UI를 통해 시각적인 알람(링크 아이콘 적색 표기 및 오디오 경고)을 발생시켜 관제사가 통신 불안정 상태를 조기에 시각적으로 인지할 수 있게 지원한다.

결론적으로 길버트-엘리엇 모델은 PX4가 변화무쌍하고 동적인 무선 환경에서 얼마나 공격적이고 예측 가능한 비행 Failsafe 설계를 취해야 하는지에 대한 중요하고 정량적인 가이드라인을 제공하며, 이를 통해 공학도는 무인기 관제 아키텍처의 이론적 기반을 견고히 확립할 수 있다.
