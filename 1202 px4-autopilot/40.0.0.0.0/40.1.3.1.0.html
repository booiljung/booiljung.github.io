40.1.3.1 길버트-엘리엇(Gilbert-Elliott) 2상태(Two-State) 버스트 에러 모델의 점근적 특성

1. 개요 및 이론적 배경

지상 관제 시스템(QGroundControl 등)과 무인항공기(UAS) 간의 RF 텔레메트리 채널은 드론의 기동성, 차폐물 개입, 안테나 편파 손실 등의 이유로, 패킷 에러가 시간적으로 상호 독립적으로 발생하는 무기억(Memoryless) 특성보다는 한 번 에러가 생기면 연쇄적으로 발생하는 버스트(Burst) 특성을 강하게 띤다.

통신 채널을 묘사하는 이산(Discrete) 시간 마르코프 체인 모델 중 가장 널리 분석되고 통신 공학 실무에 채틱되는 모델이 바로 길버트-엘리엇(Gilbert-Elliott) 2상태 모델이다. 본 절에서는 이 모델을 PX4-Autopilot 및 GCS 관제 통신망의 통계적 채널 모델로 차용하여 그 점근적 특성(Asymptotic Characteristics)을 수학적으로 분석하고, 이를 관제 시스템 스케줄링 및 Failsafe 파라미터 튜닝에 접목시키는 설계 기법을 상세히 다룬다.

2. 길버트-엘리엇(Gilbert-Elliott) 2상태 모델의 구조

초기 길버트(Gilbert) 모델을 일반화한 엘리엇(Elliott) 모델은 채널의 상태를 양호 상태(Good State, G)와 불량 혹은 버스트 상태(Bad/Burst State, B) 두 개의 이산 확률 상태로 재정의한다. 단순한 독립 베르누이 시행(Independent Bernoulli Trials)과 달리, 모델 내의 각 상태는 고유의 지속성(시간적 상관성)을 가지며 이를 나타내는 전이 확률(Transition Probability) 행렬이 존재하여 연속적인 에러 유실을 현실적으로 묘사한다.

2.1 상태 전이 파라미터 및 방출 확률

  • P_{GG}: 통신이 한 번 성공했을 때, 다음 타임스텝에서도 양호 상태(G)에 머무를 확률.
  • P_{GB}: 양호 상태(G)에서 급격한 장애물이나 페이딩으로 인해 불량 상태(B)로 전이할 확률. (P_{GG} = 1 - P_{GB})
  • P_{BB}: 불량 상태(B)에 진입한 후 계속해서 불량 상태에 체류할 확률. 관제 불능 상태(Outage)가 지속되는 시간의 기대치(Bursty Nature)를 결정하는 가장 핵심이 되는 파라미터이다.
  • P_{BG}: 불량 상태(B)에서 무선 환경이 복구되어 다시 양호 상태(G)로 전이할 확률. (P_{BB} = 1 - P_{BG})

게다가 엘리엇 확장에 따라, 각 상태 체류 시 무조건적인 성공/실패가 아닌 특정한 비트 에러 발생 확률(Error Probability)인 e_G, e_B (e_G \ll e_B) 가 방출(Emission)된다. 이로써 에러 분포의 점근적 거동을 예측할 수 있다.

3. 모델의 점근적(Asymptotic) 극한 수학적 분석

무인 비행 환경에서 통신 링크 세션이 무한히 지속되는(또는 장기BV 임무를 수행하는) 시계열의 점근적 극한(Asymptotic Limit)에서, 시스템이 G 또는 B 상태에 놓일 정상 상태 확률(Steady-State Probability) \pi_G와 \pi_B는 마르코프 연쇄의 특성방정식을 통해 다음과 같이 도출된다.

\pi_G = \frac{P_{BG}}{P_{GB} + P_{BG}} \\ \pi_B = \frac{P_{GB}}{P_{GB} + P_{BG}}
이 점근적 닫힌 형태(Closed-form) 극한 공식은 드론 텔레메트리의 전체 세션 동안 패킷이 유실되는 평균 비율, 즉 장기간의 채널 평균 에러율(Average Error Rate, AER)을 다음 공식으로 직관적으로 수렴시킨다.
\ AER = \pi_G e_G + \pi_B e_B
PX4 통신 시스템 디자이너는 새로운 기체를 대상 타겟 고도 및 지형에 비행시키며 얻은 ULog 혹은 tlog 데이터 스트림 시계열을 분석하여 해당 환경 전용의 P_{GB} 행렬과 P_{BG} 행렬 값을 추정(Training)해냄으로써, 향후 비행에서의 장기적인 텔레메트리 손실 기댓값을 고도화하여 도출할 수 있다.

4. QGroundControl 관제 아키텍처 적용 전략 (Ardupilot 대비)

이러한 점근적 특성 분석의 궁극적인 임베디드 공학적 목적은 ’현재 단일 에러가 발생했을 때 연이어 다음 패킷 역시 손실될 확률’의 시간적 관성을 QGC의 재전송 대기 큐(Retransmission Wait Queue) 최적화에 연동하는 것이다.

  1. 상태 체류성(Memory)을 고려한 타임아웃 지수 할당: 고전적인 Ardupilot MAVLink 계층이나 Mission Planner 구조에서는 주로 어플리케이션단에서 고정된 파라미터(SRn_ 시리즈 및 TIMEOUT 상수)를 사용하여 재전송을 폴링(Polling)한다. 반면, 길버트 모델 철학이 적용되는 진보된 GCS 모듈에서는 현재 모델링 관측 결과가 불량(B) 상태에 연속 진입했다고 평가되는 순간 (즉, NACK 혹은 수신 타임아웃이 도래하여 P_{BB} 지배 구간에 놓임), 즉각적이고 맹목적인 재전송 낭비로 대역폭을 낭비하는 대신, P_{BB} 확률값에 비례하는 지수 백오프(Exponential Backoff)를 MAVLink 재송신 파이프라인에 주입해 트래픽을 지연시켜야 한다.
  2. 평균 버스트 유실 길이에 기반한 Failsafe 튜닝: 버스트 에러 상태(B)의 평균 체류 길이(Average Burst Length) 기댓값은 점근적으로 수학적 역수인 L_B = \frac{1}{P_{BG}} 로 주어진다. 이는 텔레메트리 락 다운(Lock-down) 현상이 일어났을 때 평균적으로 얼마의 MAVLink 프레임 동안 회복불능 상태가 지속되는지를 보여준다. PX4 개발팀은 드론의 통신 링크 차단 트리거(예: COM_DL_LOSS_T 파라미터 혹은 GCS의 Vehicle::ConnectionLost 임계 타이머 변수)를 L_B 기댓값에 분산 마진을 더한 수치로 튜닝 설계해야만, 물리 채널의 일시적 페이딩을 오해하여 야기되는 치명적인 거짓 데이터 링크 손실(False Failsafe Trigger / RTL 발동)을 완벽하게 방어할 수 있다.

5. 결론 및 요약

드론의 BVLOS 및 Swarm 환경에서 발생하는 텔레메트리 채널의 제어 데이터 오류는 단순한 일회적 스파이크(Spike) 현상이 아니며, 길버트-엘리엇 모델이 묘사하는 거시적인 2상태(Two-State) 점근 분포 연쇄 특성을 따른다. 최첨단 관제 애플리케이션을 설계하고 연동하는 딥테크 엔지니어는 경험적인 패킷 손실률 수치에 막연히 의존하는 타성을 벗어나야 한다. 상태 전이 확률 파라미터(P_{BB}, P_{BG})의 체득 및 정상 상태 확률의 점근적 극한을 시스템 분석에 포괄하여 인용하고, 버스트 한계를 수학적으로 포용할 수 있는 동적 넌블로킹 재전송 알고리즘 파이프라인을 구축해 내야만 한다.
