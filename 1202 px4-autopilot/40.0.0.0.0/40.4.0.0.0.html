40.4 피드백 제어 시스템(Feedback Control System) 관점의 재전송 알고리즘

이 절에서는 지상 관제 시스템(GCS, QGroundControl v4.3 기준)과 드론(PX4-Autopilot v1.14 기준) 간의 통신 재전송(Retransmission) 메커니즘을 단순한 IT 네트워크 프로토콜이 아닌 피드백 제어 시스템(Feedback Control System) 관점에서 수학적, 공학적으로 재해석하여 기술한다. 통신 오류 및 패킷 유실 상황에서 기체의 상태를 안전하고 결정적으로(Deterministically) 천이시키기 위한 폐루프 제어(Closed-loop Control) 아키텍처를 상세히 확인하라.

1. MAVLink 명령어 전송을 위한 폐루프(Closed-loop) 제어 모델링

QGroundControl에서 특정한 제어 명령(예: COMMAND_LONG을 통한 비행 모드 변경 또는 이륙 지시)을 송출하고, 기체로부터 COMMAND_ACK 응답을 수신하는 과정은 본질적으로 클래식한 피드백 제어 루프와 완벽히 동일한 위상을 갖는다.

이를 제어공학의 블록 다이어그램 요소로 치환하여 모델링하라.

  • 참조 입력(Reference Input, r(t)): GCS가 달성하고자 하는 목표 기체 상태 (Target State, 예: ‘임무 비행 모드 진입’).
  • 플랜트(Plant, G(s)): MAVLink 송수신 채널과 PX4-Autopilot의 내부 상태 공간 모델(State Space Model).
  • 제어 입력(Control Input, u(t)): GCS에서 발송하는 제어 패킷(예: COMMAND_LONG).
  • 출력(Output, y(t)): PX4 기체의 실제 제어 상태(Actual State).
  • 피드백(Feedback, h(t)): 기체가 GCS로 회신하는 응답 패킷(COMMAND_ACK) 및 HEARTBEAT 기반 시스템 상태 텔레메트리.
  • 제어기(Controller, K(s)): QGroundControl 내부에 구현된 상태 감시 타이머 및 재전송 제어 로직(Retry Logic).

이 제어 루프에서 제어 오차 e(t) = r(t) - h(t) 가 완벽한 0(오차 없음)으로 수렴할 때까지 제어기(QGroundControl)는 적절한 제어 입력(MAVLink 커맨드 재전송)을 지속적으로 생성하는 알고리즘 루틴을 보장해야 한다.

2. 재전송 제어기(Retransmission Controller)의 수학적 거동 특성

재전송이라는 행위는 제어 공학의 임펄스 함수(Impulse Function, \delta(t)) 응답 제어로 도식화될 수 있다. 오차 e(t) \neq 0 인 상태에서 특정 타임아웃 주기 T_{retry}마다 제어 입력을 재인가하는 펄스 진폭 변조(PAM)와 유사한 형태를 띤다. 제어기 출력 u(t)는 단위 계단 함수의 지연 결합 수식으로 근사하여 표현할 수 있다.

u(t) = \sum_{n=0}^{N_{max}} u_0 \cdot \delta(t - n \cdot T_{retry}) \cdot \begin{cases} 1, & \text{if } e(t-n \cdot T_{retry}) \neq 0 \\ 0, & \text{otherwise} \end{cases}

여기서 N_{max}는 재전송 상한 횟수(Max Retries) 매개변수이다. 간섭이 심한 RF 채널 등 통신이 극도로 불안정한 환경에서 맹목적으로 무한 재전송 루프를 반복할 경우, 채널 포화(Channel Saturation) 및 제어 발산(Control Divergence) 현상을 유발할 수 있으므로 N_{max} 설정과 T_{retry}의 지수 백오프(Exponential Backoff) 채택 여부가 전체 관제 시스템의 안정도(Stability)를 좌우한다.

3. 명령의 멱등성(Idempotency)과 상태 발산 방지

피드백 제어 시스템에서 동일한 제어 입력(u(t))이 플랜트 단에 중복으로 투입될 때, 플랜트가 예측 불가능한 비선형 응답을 보이거나 토글형 거동을 보이면 제어 시스템은 통제 불능에 빠진다. 따라서 재전송 기반 제어에서는 플랜트(PX4-Autopilot)의 커맨드 수용 처리 계층이 반드시 수학적 **멱등성(Idempotency)**을 보장하도록 설계되어야 한다.

  • PX4 (멱등성 보장 설계): PX4 내부의 Commander 모듈은 이미 달성된 상태 공간에 대해 동일한 식별자의 모드 변경 커맨드가 중복 수신될 경우, 시스템의 물리적 비행 궤적에 악영향을 주지 않고 즉시 MAV_RESULT_ACCEPTED ACK 만을 재생산하여 안전하게 반환한다. 이는 제어 루프를 안정적으로 닫는(Close) 훌륭한 신뢰성 플랜트 설계이다.
  • Ardupilot 대비 차이점: Ardupilot 또한 비행 모드 변경 등의 하이레벨 제어에서 상당 부분 멱등성을 지니나, 일부 구형 하위 하드웨어 릴레이 제어 명령(페이로드 도어 개폐, 카메라 트리거 루프 등)에서는 연속 수신 시 상태 토글(Toggle)이나 중복 액션 실행이 일어날 가능성이 존재한다. 따라서 Ardupilot 관제 통합 시에는 GCS에서 MAVLink 시퀀스 번호 추적을 매우 엄격하게 수행해야 한다. 반면 PX4는 Commander 상태 머신 레벨에서 강력한 필터링 및 방어를 수행하므로 GCS 측 제어기 구현의 복잡성 부담이 다소 경감된다.

4. QGroundControl (v4.3) 기반 피드백 모니터링 아키텍처 구조

GCS 애플리케이션 개발 시 피드백 루프를 구현할 때, 단순히 MAVLink COMMAND_ACK 수신에만 전적으로 의존해서는 안 된다. 패킷 채널 유실은 상향 링크(Uplink)뿐만 아니라 하향 링크(Downlink)에서도 빈번하므로, 통신 구조 특성상 기체가 명령을 성공적으로 수신하여 상태를 이행했음에도 불구하고 GCS로 반환되는 ACK 패킷만이 단독 유실될 수 있다.

이러한 사각지대를 극복하기 위해 QGroundControl 내부에 적용해야 할 강건한 관제 아키텍처 다이어그램을 다음과 같이 구성하라.
