40.4.1 베르누이 시행(Bernoulli Trial) 누적 이항 분포를 활용한 재전송 상한치 최적화

이 절에서는 MAVLink 패킷 손실 확률 모델에 기반하여 QGroundControl(GCS, v4.3 기준)과 PX4-Autopilot(Firmware 버전: v1.14 기준) 간의 통신에서 ’재전송 횟수의 상한치(Maximum Retries, N_{max})’를 통계학적으로 최적화하는 기법을 구조화한다. 베르누이 시행(Bernoulli Trial)과 누적 이항 분포(Cumulative Binomial Distribution) 모델을 도입하여 무분별한 재전송으로 인한 대역폭 낭비를 막고, 수학적 보증 하에 신뢰도 목표를 달성하는 알고리즘을 설계하라.

1. MAVLink 패킷 전송의 베르누이 시행(Bernoulli Trial) 모델링

단일 MAVLink 커맨드(예: COMMAND_LONG) 패킷이 무선 주파수(RF) 채널을 통해 GCS에서 기체로 날아가고, 기체에서 GCS로 COMMAND_ACK가 회신되는 전체 왕복 과정(Round-Trip)을 하나의 확률적 실험으로 간주할 수 있다.

이 실험은 오직 두 가지 결과만을 가지므로, 통계학에서의 ‘베르누이 시행(Bernoulli Trial)’ 요건을 완벽히 충족한다.

  • 성공(Success, S): 명령 패킷이 기체에 도달하여 수락되고, ACK 패킷이 GCS로 온전히 수신됨.
  • 실패(Failure, F): 명령 패킷 또는 ACK 패킷 중 하나라도 유실되거나 훼손되어 지정된 타임아웃 내에 응답을 받지 못함.

이때 단일 시행에서의 패킷 유실률(Packet Loss Rate) 역방향 확률을 p (0 \le p \le 1), 단일 전송 성공 확률을 q = 1 - p라 정의한다. 실제 비행 환경에서 이 p 값은 안테나 이득, 다중 경로 페이딩(Multipath Fading), 간섭 등에 의해 비선형적으로 동맥 변동하는 매개변수이다.

2. 누적 이항 분포(Cumulative Binomial Distribution) 기반 재전송 수학적 도출

1회의 전송(최초 전송)과 n회의 재전송(Retransmission)을 포함하여, 총 N = n + 1 번의 전송 시도를 수행한다고 가정하자. N번의 베르누이 독립 시행 중 적어도 1번 이상 전송에 성공할 확률은 누적 이항 분포의 공식을 통해 도출할 수 있다.

전송이 완전히 실패할 확률(즉, N번 모두 패킷 유실이 발생할 확률)은 다음과 같다.

P(X = 0) = \binom{N}{0} q^0 p^N = p^N

따라서 N번 이내에 적어도 한 번 통신에 성공할 누적 확률(목표 신뢰도), 즉 관제 제어 도달 확률 R은 여사건의 확률로 다음과 같이 산출된다.

R = 1 - p^N

제어 시스템 엔지니어링에서는 시스템이 요구하는 최소 요구 신뢰도(Required Reliability) R_{target} (예: 99.9% = 0.999)가 사전에 주어지므로, 이 부등식 1 - p^N \geq R_{target}을 만족하는 최소한의 상한 횟수 N_{max}를 역산하여 도출하는 알고리즘 루틴이 QGroundControl 내부에 탑재되어야 한다.

p^{N_{max}} \leq 1 - R_{target}
N_{max} \geq \frac{\ln(1 - R_{target})}{\ln(p)}

예컨대 실시간 패킷 드롭률 p = 0.3 (30% 유실)이고, 요구 신뢰도가 R_{target} = 0.999일 때, N_{max} \ge \ln(0.001) / \ln(0.3) \approx 5.72 가 되므로 총 6회의 통신 시도(최초 1회 + 재전송 5회)가 최적 상한치라는 수학적 명제에 도달하게 된다.

3. PX4와 Ardupilot에서의 재전송 부하(Overhead) 처리 차이

  • PX4-Autopilot (동적 재전송 상한에 유리):
    PX4의 GCS/MAVLink 처리 모듈은 앞서 설명한 커맨드의 멱등성(Idempotency)이 잘 갖춰져 있으므로, 위 공식에 의해 역산 도출된 상한치 N_{max}가 다소 크더라도(예: 7~8회) 플랜트에 가해지는 물리적 부작용(토글링 등)이 없다. 따라서 QGroundControl은 PX4 기체 연동 시 매우 높은 R_{target}을 상정하고 재전송 로직을 공격적으로 설계할 수 있다.
  • Ardupilot (정적/보수적 상한 선호):
    Ardupilot은 일부 비-멱등적(Non-idempotent) 명령 계층 구역에 대해 과도한 재전송이 이중 처리를 촉발할 잠재적 위험성을 내포할 수 있다. 따라서 MAVLink 패킷 드롭률 p가 극단적으로 높을 때는 N_{max} 상한치를 강제 제한(Clamping)하고, 대신 사용자(조종자)의 GCS UI에 경보(Failsafe Alert)를 발송하여 수동 개입 단계를 유도하는 관제 철학을 적용해야 실무적으로 안전하다.

5. 실무적 한계 및 방어 코드(Defensive Coding) 제언

위와 같은 베르누이 시행 확률론은 각 패킷 유실 이벤트가 ’서로 완벽하게 독립적(Independent)’이라는 통계적 가정을 전제로 성립한다. 그러나 실제 이동체 지상-공중 RF 링크 환경에서는 건물 뒤편 교모, 통신 음영 지대 진입 등으로 인하여 버스트 에러(Burst Error, 연속적이고 종속적인 패킷 손실 현상)가 주로 발생하므로 유실 간 독립성이 보장되지 않는다.

따라서 GCS 관제망 실무 개발자는 산출된 N_{max}를 맹신하여 고정 간격으로 들이부을 경우 이른바 재전송 폭풍(Retransmission Storm)이 발생하여, 버스트 블록 구간의 유일한 전송 틈새 슬롯마저 소모해버릴 수 있음을 경계하라. 이를 완화 방어(Defense)하기 위해 N_{max} 최적화 값 허용 구간과 함께, 재전송 간 타임아웃 대기 시간을 지수 함수적으로 배가시키는 이진 지수 백오프(Binary Exponential Backoff) 제어 이론 알고리즘을 믹스하여 관제 모듈(Command Manager)을 최종 구현해야 한다.
