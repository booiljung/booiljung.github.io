63.6.5. 가속도 시간 변화율(Jerk) 벡터 제한 및 3차 스플라인(Cubic Spline) 궤적 프로파일링을 통한 불연속적 자세 제어 명령의 저대역 필터링(Low-pass Filtering) 기법

1. 개요: 불연속적 제어 명령의 한계와 스무딩(Smoothing)의 필요성

외부 지상 관제 시스템(GCS)이나 ROS 2 오프보드(Offboard) 노드에서 비행 제어기로 전달되는 목표 위치나 자세 명령(Setpoint)은 종종 계단 파형(Step Function)과 같은 불연속성을 지닌다. 멀티로터의 자세 제어기(Attitude Controller)에 불연속적인 목표 자세각이 즉각적으로 입력될 경우, 미분(Derivative) 제어항에 의해 각속도(Angular Rate) 목표치가 과도하게 튀는 현상(Spike)이 발생하며, 이는 모터 출력의 급격한 변화를 초래하여 구동기(Actuator)의 수명을 단축시키고 기체의 기구학적 진동을 유발한다.

이를 방지하기 위해 PX4-Autopilot은 가속도의 시간 변화율인 저크(Jerk) 벡터를 제한하고, 3차 스플라인(Cubic Spline)과 같은 보간(Interpolation) 기술을 결합하여 궤적을 부드럽게 프로파일링하는 기법을 내부적으로 채택하고 있다. 더불어, 고주파수의 노이즈 및 급격한 명령 변동을 억제하기 위한 저대역 필터링(Low-pass Filtering)을 단계적으로 적용한다.

2. 가속도 시간 변화율(Jerk) 벡터의 한계 설정 원리

저크(Jerk, j)는 가속도(a)의 미분이며 위치(p)의 3차 미분이다 (j = \frac{da}{dt} = \frac{d^3 p}{dt^3}). 저크를 제한한다는 것은 모터가 발휘해야 하는 추력의 변화율을 기체 동역학이 허용하는 한계 내로 묶어둔다는 것을 의미한다.

PX4에서는 MPC_JERK_MAX, MPC_JERK_AUTO 등과 같은 파라미터를 통해 궤적 생성기(Trajectory Generator) 내에서 최대 저크 한계치를 설정한다. 특히, 자세 제어 관점에서는 기체의 최대 각가속도 단위의 저크 성분을 제한함으로써 다음과 같은 물리적 이점을 얻는다.

  1. 기계적 피로도 감소: 붐(Boom)대와 모터 마운트에 가해지는 순간적인 토크 충격을 최소화한다.
  2. 공기역학적 효율 향상: 프로펠러의 급격한 RPM 변화로 인한 난류 발생 및 양력(Lift) 손실을 방지한다.
  3. 센서 노이즈 유입 억제: 불필요한 기체 진동을 줄여 관성 측정 장치(IMU)에 유입되는 구조적 고주파 공진 노이즈를 근본적으로 차단한다.

3. 3차 스플라인(Cubic Spline) 궤적 프로파일링 모델링

목표 궤적(Target Trajectory)의 시작점과 끝점 사이를 연결할 때 선형 보간을 사용하면 가속도가 불연속적으로 변하는 문제가 속출한다. 이를 극복하기 위해 PX4 내부의 비행 작업(FlightTask) 및 궤적 매니저 알고리즘은 2계 도함수(가속도)까지 연속성을 보장하는 다항식, 주로 3차 스플라인 함수를 사용하여 궤적을 프로파일링(Profiling)한다.

3.1 C2 연속성(C2 Continuity) 조건의 수학적 표현

시간 t \in [t_0, t_1] 구간에서 궤적 \mathbf{p}(t)를 3차 다항식으로 표현하면 다음과 같다.

\mathbf{p}(t) = \mathbf{a}_0 + \mathbf{a}_1 t + \mathbf{a}_2 t^2 + \mathbf{a}_3 t^3

이때, 계수 집합 \{\mathbf{a}_0, \mathbf{a}_1, \mathbf{a}_2, \mathbf{a}_3\}은 다음의 경계 조건(Boundary Conditions)을 만족하도록 계산된다.

  1. 경유점에서의 위치 일치 (C0 연속성)
  2. 경유점에서의 속도 일치 (C1 연속성)
  3. 경유점에서의 가속도 일치 연속 (C2 연속성 보장)

이러한 수학적 성질 덕분에 \mathbf{p}(t)의 가속도 성분인 2\mathbf{a}_2 + 6\mathbf{a}_3 t는 시간에 대해 선형적인 연속 함수 형태를 띠며, 결과적으로 저크는 유한한 상수(Constant) 영역에 머무르게 되어 제어 시스템이 발산하지 않고 안정성을 유지할 수 있도록 보장한다.

4. 불연속적 자세 제어 명령의 저대역 필터링(Low-pass Filtering) 기법

스플라인 기반의 궤적 프로파일링이 위치 및 속도 명령을 스무딩하는데 탁월하다면, 자세 제어기(Attitude Controller) 단에서는 외부 시스템(예: ROS 2 오프보드 제어기)에서 직접적으로 주입되는 VehicleAttitudeSetpoint 내의 쿼터니언(Quaternion)이나 오일러 각의 단계적 응답(Step Response)을 제어하기 위해 2차적인 저대역 필터링 기법이 병행된다.

4.1 1차 저대역 필터 (PT1 Filter) 적용 로직

PX4 소스코드의 mc_att_control (멀티로터 자세 제어 모듈) 내부를 살펴보면, 외부로부터 유입된 기준 목표 자세(Reference Setpoint)와 실제 자세(Actual Attitude) 간의 오차를 계산하기 전, 또는 계산 후에 생성된 각속도 명령(Rate Command)에 대해 다음과 같은 형태의 이산 시간 기반 1차 저대역 필터가 적용된다.

y[k] = \alpha x[k] + (1 - \alpha) y[k-1]

여기서 \alpha = \frac{\Delta t}{\tau + \Delta t} 이며, \tau는 필터의 시정수(Time Constant), \Delta t는 제어 루프의 샘플링 주기(Sampling Period)이다.

  • 제어 파라미터(Parameter): MC_ROLL_P, MC_PITCH_P 계열과 연계하여 자세 명령 자체를 필터링하거나, 각속도 제어 이전에 MC_D_TERM_CUTOFF를 통해 노이즈의 대역을 컷오프(Cut-off)한다.
  • 구형파 타겟 자세 커맨드가 들어올 경우, 이 필터를 통과한 명령값은 지수 함수 형태의 완만한 곡선을 그리게 되어 기체의 각운동(Angular Motion)이 Jolt(급격한 움직임) 없이 추종된다.

5. 기법 적용을 고려한 ROS 2 연동 모범 사례 (Best Practice)

ROS 2(uXRCE-DDS) 환경에서 에이전트 노드를 개발하여 궤적을 제어할 때, PX4 내부의 저대역 필터와 스플라인 매니저가 어떻게 동작하는지 정확히 이해하고 퍼블리싱(Publishing) 레이트를 설계하는 것이 중요하다.

  1. 목표치가 멀리 떨어져 있고 간헐적으로 전송되는 궤적 지점(Sparse Waypoints)인 경우: 위치/속도 커맨드를 보내어 PX4의 궤적 생성기(Trajectory Generator)가 중간 과정을 스플라인 곡선으로 부드럽게 프로파일링하도록 위임하라.
  2. 실시간 100Hz로 자세(Attitude)를 직접 전송하는 VehicleAttitudeSetpoint 활용의 경우: 보내는 시점부터 ROS 2 노드 자체에서 Jerk 제한 및 필터링을 거친 매끄러운 커맨드 프로파일을 연속 생성하여 전송하라. PX4 내부 필터에만 온전히 의존할 경우, 네트워크 지연(Latency) 변동성에 의해 필터 입력에 무작위한 계단 현상이 반복될 수 있다.
