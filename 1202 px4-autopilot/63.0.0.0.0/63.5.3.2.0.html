63.5.3.2. 다항식 궤적 생성기(Polynomial Trajectory Generator)의 피드포워드(Feed-forward) 제어 입력을 통한 정적 위치 상태 오차(Steady-state Error) 점근적 최소화 기법

오프보드(Offboard) 모드에서 단순한 위치 제어만을 수행할 경우, 기체는 목표 위치에 도달하는 과정에서 관성이나 공기 저항, 기타 외란 등에 의해 제어 시스템 내 정적 위치 상태 오차(Steady-state Error)가 발생할 수 있다. 특히 고속 비행이나 정밀한 궤적 추종이 요구되는 상황에서 이러한 오차는 자율 비행 시스템의 안정성과 정확도를 크게 저하시키는 요인이 된다. 이를 근본적으로 해결하고 오차를 점근적(Asymptotically)으로 최소화하기 위해, **다항식 궤적 생성기(Polynomial Trajectory Generator)**를 설계하고 피드포워드(Feed-forward) 제어 입력을 시스템에 인가하는 방식이 적용된다.

이 절에서는 PX4-Autopilot 환경과 ROS2 미들웨어를 연동하여, 목표 궤적에 대한 다항식 기반의 피드포워드 입력을 설계하고 이를 통해 제어 오차를 줄이는 기법을 심도 있게 분석한다.

1. 정적 위치 상태 오차와 피드포워드 제어의 필요성

일반적인 비례-적분-미분(PID) 위치 제어루프(Position Control Loop)는 현재 위치와 목표 위치 간의 차이(Error)에만 의존하여 피드백(Feedback) 제어 입력을 생성한다. 피드백 루프는 외란을 상쇄하고 목표 지점으로 시스템을 이끄는 핵심적인 역할을 하지만, 다음과 같은 한계가 존재한다.

  1. 지연 시간(Latency) 문제: 오차가 발생한 이후에야 제어 입력이 생성되므로, 동적인 궤적을 추종할 때 근본적인 추종 지연(Tracking Lag)을 피할 수 없다.
  2. 정적 상태 오차(Steady-state Error): 경유지 사이를 선형적 혹은 비선형적으로 이동하는 환경에서, 속도나 가속도 명령 없이 단순 위치 명령만 주어지면 영속적인 추종 오차가 남게 된다.

이러한 한계를 극복하기 위해서는 시스템의 동역학적 특성을 사전에 반영한 피드포워드(Feed-forward) 입력을 결합해야 한다. 최적의 피드포워드 입력은 목표 궤적이 요구하는 이상적인 속도(Velocity) 및 가속도(Acceleration)를 궤적 생성기 단계에서 도출하고, 이를 TrajectorySetpoint의 추가적인 필드(예: velocity, acceleration)로 컨트롤러에 미리 전달함으로써 구현된다. 결과적으로 피드백 루프는 사전에 계획된 궤적에서 벗어난 선형적인 미세 오차만을 보상하면 되므로 제어 효율성과 정확도가 비약적으로 향상된다.

2. 다항식 궤적 생성기(Polynomial Trajectory Generator) 아키텍처

오프보드 경로 제어에서 부드럽고 율동적인 비행을 달성하기 위해 3차(Cubic), 5차(Quintic), 7차(Septic) 등의 다항식(Polynomial) 곡선 궤적 기법이 널리 사용된다. 특히 **승차감(Jerk)**을 최소화하는 최소 저크 궤적(Minimum Jerk Trajectory) 생성 기법은 학술적으로나 실무적으로 가장 많이 활용되는 방법이다.

다항식의 차수에 따라 제어 가능한 미분 요소가 결정된다.

  • 3차 다항식(Cubic): 위치와 속도의 경계 조건을 만족시킨다. 단점으로는 가속도의 불연속성이 발생하여 기체에 기계적 무리가 갈 수 있다.
  • 5차 다항식(Quintic): 위치, 속도, 가속도의 경계 조건을 완벽하게 스무딩(Smoothing) 처리 가능하여 연속적인 제어 입력을 생성한다.
  • 7차 다항식(Septic): 저크(Jerk)의 미분인 스냅(Snap)까지 연속 구조로 만들 수 있어 쿼드로터 중심의 고기동성 제어에 매우 적합하다. 논문 Minimum Snap Trajectory Generation and Control for Quadrotors에서도 멀티로터의 동역학 특성에 최적화된 최소 스냅 곡선을 제안하고 있다.

2.1 차 다항식 모델 설계 수식

1차원 공간(예: X 축) 상에서 시간 t에 따른 위치 x(t)를 5차 다항식으로 표현하면 다음과 같다.

x(t) = c_0 + c_1 t + c_2 t^2 + c_3 t^3 + c_4 t^4 + c_5 t^5

이를 시간에 대해 미분하여 선속도 \dot{x}(t)와 선가속도 \ddot{x}(t)를 유도한다.

\dot{x}(t) = c_1 + 2 c_2 t + 3 c_3 t^2 + 4 c_4 t^3 + 5 c_5 t^4
\ddot{x}(t) = 2 c_2 + 6 c_3 t + 12 c_4 t^2 + 20 c_5 t^3

시작 시간 t_0 = 0 및 도착 시간 t_f에서의 경계 조건(Boundary Conditions)은 위치, 속도, 가속도 모두 0에서 시작해 목표치에 부드럽게 도달하도록 설정된다. 이를 만족하는 계수 c_0, c_1, \ldots, c_5를 구하는 역행렬 계산 연산을 통해 유일한 궤적 식별이 가능하다. 모든 축(X, Y, Z, Yaw)에 대해 독립적으로 다항식 매개변수를 구하면 부드러운 3차원 공간 궤적과 속도, 가속도 벡터를 산출할 수 있다.

3. 피드포워드 입력을 활용한 제어 오차 점근적 최소화 구조

궤적 생성기에서 산출된 x(t) (위치), \dot{x}(t) (선속도), \ddot{x}(t) (선가속도)는 ROS2 노드 내부에서 샘플링 시간 \Delta t 단위로 이산화되어 TrajectorySetpoint 메시지로 조립된다. PX4의 비행 제어기(Flight Controller) 스택, 특히 위치 컨트롤러 모듈(Position Controller Module)에서는 이 정보를 수신하여 구조적 피드백-피드포워드 합성 제어를 수행한다.

graph TD
    A[Polynomial Trajectory Generator] -->|Position Setpoint| B(Position Error Block)
    A -->|Velocity Setpoint - Feed-forward| C(Velocity Controller)
    A -->|Acceleration Setpoint - Feed-forward| D(Acceleration / Thrust Allocator)
    
    B -->|Feedback Velocity Command| C
    C -->|Feedback Acceleration Command| D
    D -->|Target Attitude & Thrust| E[Attitude & Rate Controller]
    E --> F[Actuator Outputs]
    
    F -->|Vehicle Dynamics| G[Real Vehicle State]
    G -->|Estimator VIO/EKF2| B
    G -->|Estimator VIO/EKF2| C

위의 제어 흐름도에서 알 수 있듯이, 궤적 생성기(A)가 목표 위치뿐만 아니라 피드포워드 방식의 목표 선속도 및 목표 체공 선가속도를 동시에 주입한다.
선속도 성분은 위치 컨트롤러의 미분 요소 부담을 경감시키고, 선가속도 성분은 기체의 질량-관성 역학을 극복하는 즉각적인 추력 변환을 관장한다. 피드포워드 루프의 개입으로 인해 제어기는 상태 지연이 유발하는 정적 오차를 이론적 제로(0)에 수렴하도록 설계되어 점근적 안정성(Asymptotic Stability)을 보장한다.

4. 실습: ROS2 기반 Oﬀboard 다항식 피드포워드 궤적 제어 노드 작성

이번 실습에서는 C++와 ROS2 rclcpp를 활용하여 시작점과 목표점 사이를 5차 다항식을 적용하여 매끄럽게 연결하고, 위치와 속도 피드포워드를 TrajectorySetpoint 메시지로 퍼블리시하는 제어 노드를 구현한다.

4.1. 5차 다항식 계수 계산 함수 구현

먼저, 주어진 경계 조건(위치/시간)에서 계수를 도출하고, 특정 시간 t에서의 위치와 속도를 동적으로 반환하는 클래스를 정의한다. 단순화를 위해 시작 및 도착 지점에서의 속도와 가속도를 0(v=0, a=0)으로 설정된 상황을 가정한다.

#include <cmath>
#include <vector>

class QuinticPolynomial {
private:
    double c0, c1, c2, c3, c4, c5;

public:
    QuinticPolynomial(double start_pos, double end_pos, double duration_sec) {
        c0 = start_pos;
        c1 = 0;
        c2 = 0;
        c3 = 10 * (end_pos - start_pos) / std::pow(duration_sec, 3);
        c4 = -15 * (end_pos - start_pos) / std::pow(duration_sec, 4);
        c5 = 6 * (end_pos - start_pos) / std::pow(duration_sec, 5);
    }

    // 시간 t에 대한 위치(Position) 반환
    double get_position(double t) const {
        return c0 + c1*t + c2*std::pow(t,2) + c3*std::pow(t,3) + c4*std::pow(t,4) + c5*std::pow(t,5);
    }

    // 시간 t에 대한 속도(Velocity) 반환 (피드포워드 성분)
    double get_velocity(double t) const {
        return c1 + 2*c2*t + 3*c3*std::pow(t,2) + 4*c4*std::pow(t,3) + 5*c5*std::pow(t,4);
    }
};

4.2. ROS2 노드 내부의 TrajectorySetpoint 동기화

각 축(X, Y, Z)에 대하여 독립적으로 5차 다항식을 계산하고, 이를 TrajectorySetpoint의 position 배열과 velocity 배열에 동기화시켜 전송한다. 이 방식을 통해 기체는 목표 위치만 전송받을 때 발생하는 초기 추종 둔감도(Sluggish Response)와 초과 응답(Overshoot)을 현저히 줄이고, 정적 오차 없이 매우 정확한 비행 곡선을 그리게 된다.

// ROS2 Timer Callback 내에서의 제어 루프 예시
void TrajectoryNode::timer_callback() {
    auto current_time = this->get_clock()->now();
    double t = (current_time - start_time_).seconds();

    // 목표 도착 시간(duration) 초과 시 t를 목표 시간으로 고정 (포화 상태)
    if (t > duration_) t = duration_;

    px4_msgs::msg::TrajectorySetpoint setpoint_msg;
    setpoint_msg.timestamp = this->get_clock()->now().nanoseconds() / 1000;

    // X, Y, Z 축에 대한 피드포워드 위치 및 속도 매핑
    setpoint_msg.position[0] = poly_x_->get_position(t);
    setpoint_msg.velocity[0] = poly_x_->get_velocity(t);
    
    setpoint_msg.position[1] = poly_y_->get_position(t);
    setpoint_msg.velocity[1] = poly_y_->get_velocity(t);
    
    // Z축은 NED 좌표계(음수가 상승)이므로 - 부호 적용에 주의
    setpoint_msg.position[2] = poly_z_->get_position(t);
    setpoint_msg.velocity[2] = poly_z_->get_velocity(t);

    setpoint_msg.yaw = 0.0f;          // Yaw 각도 고정

    trajectory_setpoint_publisher_->publish(setpoint_msg);
}

이와 같이 피드포워드 된 속도 변수들(velocity[0], velocity[1], velocity[2])는 PX4 내부의 다단 PID 스택(Cascaded PID Controller)에서 위치 오차의 미분항보다 한 단계 먼저 예측값으로 결합되어, 공력학적 지연 시간을 돌파하는 마진(Margin)을 확보한다. 이는 정적 오류가 축적되는 적분(Integral) 현상을 방지하여, 실질적인 상태 오차의 점근적 소멸(Asymptotic Evanescence)을 이끌어 낸다.

5. 요약 및 고찰

오프보드 제어 단에서 시스템의 위치 상태 변수만을 제어 명령으로 하달하는 계단 함수(Step Function) 형태의 접근은 정적 위치 오차를 유발할 위험이 높다. 이를 방지하기 위해 5차 다항식 기반의 곡선형 궤적 계획과 더불어, 위치의 미분값인 선속도를 적극적으로 피드포워드 입력으로 활용하는 구조를 도입해야 한다. 피드포워드 제어 기법은 PX4-Autopilot 기반의 고속 자율 에이전트 개발이나 군집 비행(Swarm) 환경에서 에이전트 간의 동적 오차를 줄이고 정밀성을 담보하기 위한 초석이 된다.
