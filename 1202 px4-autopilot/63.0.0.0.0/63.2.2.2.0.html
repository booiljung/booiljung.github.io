63.2.2.2. 궤적 설정점 토픽(TrajectorySetpoint)의 다차원 운동학적 상태 변수(공간적 위치, 선속도, 선가속도 벡터) 텐서(Tensor) 구성 요소

고도화된 자율 비행 임무를 수행하는 PX4-Autopilot 기반 무인항공기 기체 제어기(Flight Controller)에 있어, 궤적 추종(Trajectory Tracking) 알고리즘의 기준 신호(Reference Signal)를 전달하는 가장 기초적이면서도 방대한 토픽이 바로 TrajectorySetpoint이다. 지상 관제 시스템이나 동반 컴퓨터(Companion Computer)에서 생성된 오프보드(Offboard) 제어 명령은 ROS2 uXRCE-DDS 또는 MAVLink를 경유하여 기체로 전달되며, 위치 분산 제어 루프(Position Cascaded Control Loop)의 다차원 운동학적 상태 변수(Kinematic State Variables)를 직접 또는 간접적으로 구속(Constraint)한다. 본 절에서는 해당 토픽을 구성하는 상태 변수 벡터의 수학적 정의와 텐서(Tensor) 구조 특성을 면밀히 분석한다.

1. TrajectorySetpoint 토픽의 설계 사상과 텐서(Tensor)적 특성

Ardupilot 환경에서는 지상 관제 시스템이 좌표 이동을 지시할 때 MAVLink의 SET_POSITION_TARGET_LOCAL_NED 메시지에 위치, 속도, 가속도 등을 포함해 한 번에 송신하며 개별 필드들의 무시(Ignore) 처리를 비트마스크(Bitmask)에 의존한다. 반면, PX4-Autopilot(v1.14 이상)의 TrajectorySetpoint 구조체는 이와 본질적으로 유사한 다차원 변수군을 내포하고 있으나, uORB 아키텍처 상위의 독립된 토픽으로서 제어 목표(Setpoint) 데이터만을 순수하게 분리 모델링하였다.

이는 경로 계획(Path Planning) 알고리즘이 발생시키는 복합적인 1차(속도), 2차(가속도), 3차(저크) 동역학적 파생 변수들을 텐서 형태로 수용할 수 있게 고안된 결과이다. 각 변수는 독립된 3차원 벡터 어레이 형식을 취하며 NaN(Not a Number) 처리를 통해 제어기 진입 인터페이스(Interface)에서 제어 권한을 암묵적으로 포기하는(Yield) 설계 논리를 갖춘다.

1.1 구조체를 구성하는 다차원 운동학적 벡터 요소

TrajectorySetpoint는 기체 운동학 수식 기반의 모델 예측 제어(MPC: Model Predictive Control)나 다항식 궤적 생성 시 필수적인 다차원 요소들을 제공한다. 주요 구성 요소는 다음과 같다.

  • 공간적 위치 벡터 (Position Vector):
    \mathbf{p} = [p_x, p_y, p_z]^\mathsf{T} (단위: m)
    NED(North-East-Down) 로컬 좌표계 기준의 3차원 절대 좌표 지령치이다.
  • 선속도 벡터 (Linear Velocity Vector):
    \mathbf{v} = [v_x, v_y, v_z]^\mathsf{T} (단위: m/s)
    위치 벡터의 1차 시간에 대한 미분항(d\mathbf{p}/dt)으로, 속도 추종 제어 루프의 기준 신호로 작용한다.
  • 선가속도 벡터 (Linear Acceleration Vector):
    \mathbf{a} = [a_x, a_y, a_z]^\mathsf{T} (단위: m/s^2)
    기체의 관성력 극복 및 동적 외란(Dynamic Disturbance)을 상쇄하기 위한 피드포워드(Feedforward) 응답성 제고에 쓰이는 2차 미분항(d^2\mathbf{p}/dt^2)이다.
  • 저크 벡터 (Jerk Vector):
    \mathbf{j} = [j_x, j_y, j_z]^\mathsf{T} (단위: m/s^3)
    가속도의 변화량(3차 미분항)으로, 탑승 가능한 UAM이나 정밀 카메라 짐벌을 탑재한 기체의 스냅(Snap)을 줄이고 궤적을 평활화(Smooth)하는 데 투입된다.
  • 요(Yaw) 방위 및 각속도:
    \psi (단위: rad) 및 \dot{\psi} (단위: rad/s)
    동체 좌표계(Body-frame)의 1차원 회전 운동학 변수로 궤적 진행 방향(Heading)을 구속한다.

2. 동역학적 상태 변수의 제어기 인가(Injection) 메커니즘

2.1 NaN 제어 마스킹(NaN Masking) 기법

TrajectorySetpoint의 가장 큰 구조적 특징은 제어 목표값 할당 시 부동소수점의 IEEE 754 표준에 정의된 ’NaN(Not A Number)’을 교묘하게 마스킹 도구로 전용한 점이다.
예를 들어, 다중 에이전트 환경에서 순수하게 상위 노드가 산출한 최적 회피 위치로 비행하려 할 시 속도(\mathbf{v})나 가속도(\mathbf{a}) 항에 수치 해석 오류를 뜻하는 NaN을 삽입하여 전송한다. PX4의 PositionControl 모듈 로직은 수신된 상태 변수 성분을 검사하여, std::isnan(velocity(0)) 연산이 참(True)으로 반환될 경우 해당 PID 캐스케이드(Cascade)의 하위 속도 추종 연산을 우회하거나 기여도 게인(Gain)을 0으로 처리한다.

2.2 부드러운 궤적 추종(Smooth Trajectory Tracking) 응용 체계

복잡한 실내 환경이나 군집(Swarm) 비행 시 ROS2의 Nav2 또는 독자적인 경로 계획기(예: 미니멈 스냅 궤적 생성기)가 존재할 경우, 단순한 위치(Pos) 지령의 주기적 전송만으로는 기체 거동이 거칠어진다.
따라서 현대 제어 공학의 기조(Trend)는 동반 컴퓨터 측에서 시스템의 다이나믹스 한계를 고려해 \mathbf{p}, \mathbf{v}, \mathbf{a} 세 요소를 동시해 연산하고 미분 연속성(Differentiable Continuity)을 띤 궤적을 텐서화해 TrajectorySetpoint로 밀어 넣는 방식을 따른다. PX4 내부 제어기는 수신된 \mathbf{v}와 \mathbf{a}를 바탕으로 속도 제어기의 피드포워드 게인 블록(Feedforward Gain Block)에 이를 직접 인가함으로써 궤적 지연 오차율(Lag Error Rate)을 대폭 절감한다.

결론적으로 TrajectorySetpoint 토픽은 위치 비례 제어를 뛰어넘어 완전한 고차원 비행 역학(Flight Mechanics) 구동의 밑단 칩(Micro-chip) 역할을 수행하며, Ardupilot 대비 우수한 PX4 특유의 유연성을 대변하는 데이터 구조체라 할 수 있다.
