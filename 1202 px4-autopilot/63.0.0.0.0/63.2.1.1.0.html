63.2.1.1 마이크로-ROS(Micro-ROS) 에이전트와 ROS2 클라이언트 간의 개방형 시스템 간 상호 접속(OSI) 모델 전송 계층 세션 라우팅 매핑

1. 개요

PX4-Autopilot의 오프보드(Offboard) 제어 환경에서 비행 제어기(FC) 내부의 uXRCE-DDS 클라이언트(Client)와 임무 컴퓨터(Companion Computer)의 Micro-XRCE-DDS 에이전트(Agent)는 끊임없이 방대한 데이터를 교환한다. 이러한 이기종 간의 미들웨어 통신은 단순한 바이트 스트림의 나열이 아니며, 데이터의 송수신 신뢰성과 구조적 안정성을 담보하기 위해 개방형 시스템 간 상호 접속(OSI, Open Systems Interconnection) 모델의 전송 계층(Transport Layer, L4) 및 세션 계층(Session Layer, L5)의 개념을 차용한 정교한 세션 라우팅(Session Routing) 체계를 바탕으로 한다. 본 절에서는 PX4 v1.14 및 ROS 2 Humble 플랫폼을 기준으로 어떠한 방식으로 전송 매체가 추상화되고 세션이 매핑되는지 심층적으로 기술한다.

2. 전송 계층(Transport Layer, L4) 논리 매핑 및 차상위 구조

PX4와 ROS 2 간의 물리적 브릿징은 하드웨어 단자 여건에 따라 직렬 통신(UART) 또는 이더넷(UDP/TCP) 등 상이한 매체 위에서 이루어지지만, uXRCE-DDS는 이를 동일한 OSI 전송 계층 논리로 추상화(Abstraction)한다.

  • UDP 전송 매핑 (이더넷 보드 적용 시):
    비행 제어기가 고속의 이더넷 포트(예: Pixhawk 6X)를 지원할 때 주로 채택된다. UDP 원형 데이터그램은 비연결성(Connectionless)을 지녀 도달을 보장하지 않으므로, Micro-XRCE-DDS는 UDP 페이로드 헤더 위에 자체적인 순서 인덱싱(Sequence Indexing) 및 신뢰성 프레임 규격을 덮어씌워 OSI 모델의 신뢰성 높은 전송 계층 한계를 극복한다.
  • 직렬 통신 전송 매핑 (UART 적용 시):
    가장 대중적이고 범용적인 텔레메트리 연결망이다. UART는 OSI 모델상 단순한 버스 기반의 링크 물리 매체에 불과하여 데이터의 경계를 알 수 없다. 따라서 에이전트와 클라이언트는 독자적인 프레이밍(Framing) 규격(HDLC 형식의 플래그 바이트와 구조체 경계 식별자 적용)을 차용하여 논리적인 전송 계층 엔드포인트(Endpoint)를 성공적으로 에뮬레이션(Emulation)한다.

3. 세션 계층(Session Layer, L5) 기반의 라우팅 체계 및 동기화 루틴

안정적인 전송망이 확립되면 양측은 실시간 오프보드 제어를 위한 논리적 세션(Session)을 초기화한다.

3.1 논리적 세션 확립 프로세스

  1. 클라이언트 초기화 (Client Handshake):
    비행 제어기의 내장 모듈인 uXRCE-DDS Client가 구동될 때 생성된 고유의 클라이언트 식별자(Client Key)와 세션 난수(Session ID)를 바탕으로 에이전트 측에 통신 생성을 요청(Create Client Payload)한다.
  2. 세션 라우팅 테이블(Routing Table) 편성:
    임무 컴퓨터에서 대기 중인 Micro-XRCE-DDS Agent는 요청받은 클라이언트 식별자와 물리적 스트림 소스 자원(예: /dev/ttyTHS1 또는 특정 UDP 포트)을 논리적으로 상호 맵핑하는 라우팅 테이블 레코드를 내부 인프라에 할당한다.
  3. 토픽 엔티티(Entity) 생성 및 라우팅 바인딩:
    이후 클라이언트 측에서 VehicleLocalPosition과 같은 토픽의 발행(Publish)을 요구하면, 에이전트는 ROS 2 DDS 공간 내에 해당하는 진짜 퍼블리셔 시스템 엔티티를 생성하고, 해당 엔티티 ID를 앞서 확립한 고유 Session ID의 테이블과 단단하게 종속(Binding)시킨다. 이렇게 구성된 통로가 세션 기반 다중화(Multiplexing)이다.

4. Ardupilot 모델 대비 패킷 라우팅 세션 관리의 우수성

Ardupilot을 필두로 한 MAVLink 기반의 기존 브릿지 아키텍처(예: MAVROS 구성)에서는, OSI 모델에서 명시하는 접속 지향 세션(Connection-oriented Session) 개념보다는 MAVLink 특유의 System ID 및 Component ID에 의지하는 무상태(Stateless) 기반의 브로드캐스트 라우팅(Broadcast Routing) 개념에 의존하는 경향이 크다.

이에 반해 최신 PX4의 Micro-ROS 아키텍처는 개방형 시스템 모델의 원형을 철저히 따라 명시적인 **세션 핸드셰이크(Session Handshake)**를 강제한다. 이러한 구조적 원칙은 임무 컴퓨터(Agent) 시스템이 도중에 재부팅되거나 케이블 간섭으로 연결이 단절되었을 때 그 진가를 발휘한다. 연결 단절 시 상호 시스템은 기존 세션의 만료를 완벽히 인지하고, 통신망에 걸려 있던 좀비 형식의 이전 토픽 엔티티 묶음들을 완전히 폐기 분해한 다음 클린 상태(Clean State)에서 새로운 세션 기반 라우팅을 자동으로 신속히 재구축하는 강력한 상태 인식(State-aware) 내결함성 관리를 성취한다.

5. 결론

마이크로-ROS 에이전트와 ROS 2 클라이언트 사이에 성립된 OSI 모델 전송/세션 기반 브릿징 설계는, 단순 무지향성 바이트 통신망의 한계를 뛰어넘어 완벽하게 동기화된 시스템 간 세션 라우팅의 교과서적인 체계를 확립해 준다. 이 같은 네트워크 아키텍처 진보 덕분에, 향후 비전 AI나 수많은 군집 조종 토픽 및 서비스의 동시다발적 호출이 단 하나의 제한된 물리 포트 위에서 엉킴 없이 멀티플렉싱(Multiplexing) 전송될 수 있으며 초경량 플랫폼에서도 자율 비행 인프라의 견고성을 이끌어 낼 수 있다.
