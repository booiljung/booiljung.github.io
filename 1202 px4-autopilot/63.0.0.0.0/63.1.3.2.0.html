63.1.3.2. 실시간 통신 발행 주기(Publishing Rate)에 따른 나이퀴스트-섀넌 표본화 정리(Nyquist-Shannon Sampling Theorem) 기반 데이터 전송 지연 시간(Latency) 및 채널 대역폭 활용 효율성 분석

최신 PX4-Autopilot 기반의 비행 제어 시스템에서 외부 임무 컴퓨터(Companion Computer)나 지상 관제 시스템(GCS)이 기체를 제어할 때, 센서 데이터의 수신 및 제어 명령(Setpoint)의 송신 주기는 비행 안정성에 절대적인 영향을 미친다. 통신 미들웨어(MAVLink 또는 ROS2 uXRCE-DDS) 상에서의 실시간 데이터 발행 주기(Publishing Rate) 설정은 단순히 ’빠를수록 좋다’는 직관적 접근을 넘어서 정보 이론 및 신호 처리의 기본 원리인 **나이퀴스트-섀넌 표본화 정리(Nyquist-Shannon Sampling Theorem)**에 기반한 엄밀한 수학적 튜닝을 요구한다.

본 절에서는 통신 스트리밍 주파수에 따른 앨리어싱(Aliasing) 회피와 네트워크 대역폭 한계로 인한 지연 시간(Latency)의 상관관계를 분석한다.

1. 나이퀴스트-섀넌 표본화 정리와 비행 동역학 데이터의 표본화

연속적인 환경에서 다이나믹하게 움직이는 무인항공기(UAV)의 물리적 선형 및 각운동 상태 신호 x(t)를 이산 시간 네트워크 패킷 시스템으로 복원 및 제어하기 위해서는 표본화 정리를 충족해야 한다.
나이퀴스트-섀넌 제2정리에 따르면, 신호의 최대 유효 주파수 대역이 f_{max} 일 때, 이를 정보 손실 없이 복원하기 위한 최소 표본화 주파수(발행 주기) f_s는 다음과 같이 정의된다.

f_s > 2 f_{max}

1.1 역학계 주파수 응답 대역폭 (Dynamic System Bandwidth)

일반적인 소형 멀티로터의 물리적 기계역학계 최대 응답 대역폭은 프로펠러 관성 및 프레임 특성에 따라 10 Hz ~ 30 Hz 내외에 형성된다. 따라서 상태 감지 및 제어 루프를 위한 외부 시스템과의 통신 발행 주기는 나이퀴스트 주파수의 최소 2배인 20 Hz ~ 60 Hz를 절대적인 하한선(Lower Bound)으로 간주해야 한다.

  • 만약 GCS나 ROS2 Offboard 노드가 f_s < 2 f_{max} 의 낮은 주기로 상태 토픽을 구독(Subscribe)한다면, 고주파수 진동이나 돌풍에 의한 외란이 저주파로 왜곡되어 측정되는 **앨리어싱 현상(Aliasing)**이 발생한다.
  • 이는 PID 제어기의 미분(Derivative) 항을 극도로 불안정하게 만들며 위상 여유(Phase Margin)를 감소시킨다.

2. 데이터 전송 지연 시간(Latency) 및 큐잉(Queueing) 효과 분석

나이퀴스트 정리를 충족시키기 위해 발행 주기를 무한정 높이는 접근은 현실 네트워크 모델에서는 역효과를 초래한다. 실제 통신망에서의 총 지연 시간 L_{total}은 다음 수식으로 근사할 수 있다.

L_{total} = L_{prop}(전파 지연) + L_{proc}(처리 지연) + L_{queue}(큐 대기 지연)

1.1 퍼블리싱 레이트 초과에 따른 고착 현상(Buffer Bloat)

마이크로컨트롤러 내부의 NuttX RTOS 기반 uORB 토픽 처리 한계 속도나 물리적 직렬 포트(UART)의 보드레이트(Baudrate) 한계를 극복할 정도로 비정상적으로 높은 주파수(예: 500 Hz 이상)로 데이터를 스트리밍하게 되면, 하드웨어 계층의 버퍼 공간에 전송되지 못한 패킷이 적체된다.
이는 결과적으로 L_{queue}를 급격히 증가시켜 **가변적 핑 오버헤드(Jitter)**와 **버퍼 블로트(Buffer Bloat)**에 의한 대규모 지연을 낳으며, 신호의 실시간성을 치명적으로 파괴한다. 즉, ’최신 정보’를 전달하기 위해 발행 주기를 높였으나 오히려 ’오래된 과거의 정보’가 지연 전달되는 패러독스가 발생한다.

2. 섀넌-하틀리(Shannon-Hartley) 대역폭 활용 효율성 분석

통신 물리 매체의 신뢰성은 채널 용량(Channel Capacity)에 종속된다. 섀넌-하틀리 정리에 따르면 오류 없는 최대 데이터 전송 속도 C (bps)는 채널 대역폭 B와 신호 대 잡음비 S/N에 의해 결정된다(C = B \log_2(1 + S/N)).

2.1 MAVLink 텔레메트리 환경 (저대역폭 통신)

전통적인 SiK 라디오 텔레메트리망(예: 57,600 bps)은 대역폭(B)이 매우 좁다. MAVLink v2 기반 고해상도 IMU 데이터(HIGHRES_IMU)를 50 Hz로만 퍼블리싱해도 채널 포화율(Saturation Rate)이 80%에 육박하게 되며, 패킷 크래시 및 재전송이 빈번해져 채널 대역폭 효율성이 급감한다. 이 경우 나이퀴스트 기준점 바로 위인 10 Hz ~ 20 Hz 영역에서 타협을 보아야 한다.

2.2 ROS2 uXRCE-DDS 임무 컴퓨터 환경 (대역폭 확장)

대조적으로, 이더넷 기반 로컬 네트워크로 PX4와 연결되는 ROS2 임무 컴퓨터의 통신망은 기가비트(Gbps) 급 대역폭을 제공하여 섀넌 한계 제약이 사실상 사라진다. 이 경우 수백 Hz의 VIO(Visual Inertial Odometry) 메시지와 VehicleOdometry 토픽을 제약 없이 주고받을 수 있어 나이퀴스트 마진을 10배 이상(Oversampling) 확보할 수 있다.

4. 실무적 결론 : 최적화 튜닝 가이드

결론적으로, PX4-Autopilot의 MAVLink(MAV_1_RATE 시스템 파라미터 등) 설정이나 ROS2 DDS 설정 시 다음 수칙을 준수해야 한다.

  1. 나이퀴스트 관점: 기체 물리 특성 주파수의 최소 2~3배를 타깃 퍼블리싱 주기로 선정. (일반적으로 Offboard 제어 요구치 10 Hz ~ 50 Hz)
  2. 대역폭 관점: 텔레메트리 채널의 Baudrate가 감당할 수 있는 페이로드 사이즈를 넘지 않도록, 주요 토픽들을 시분할 다중화 방식으로 배분하여 큐 지연(Queue Delay)을 원천 차단. QGroundControl v4.3 이상의 Inspector 기능을 통해 Stream 누락(Drop) 율을 상시 모니터링한다.
