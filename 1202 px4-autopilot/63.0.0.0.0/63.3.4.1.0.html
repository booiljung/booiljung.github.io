63.3.4.1. 임무 컴퓨터 측 제어 노드 교착 상태(Deadlock) 발생 시 비행 제어기 펌웨어가 독립적으로 수행하는 리아프노프 안정성(Lyapunov Stability) 기반 비상 안전(Failsafe) 이산 상태 전이 로직

동반 컴퓨터(Companion Computer)에서 구동되는 고수준(High-level) 자율 비행 알고리즘(예: SLAM, 객체 인식, VIO 퓨전 등)은 비전 처리 부하나 메모리 누수(Memory Leak) 등으로 인해 예기치 않은 스레드 교착 상태(Deadlock)나 운영체제 패닉(Kernel Panic)에 빠질 확률을 늘 안고 있다. 이러한 상위 인지 체계의 붕괴가 즉각적인 기체 추락으로 이어지지 않게 막는 최후의 보루는 비행 제어기(FC: Flight Controller) 펌웨어 자체에 내장된 강건(Robust) 제어 로직이다. 본 절에서는 PX4-Autopilot(v1.14 이상)이 외부 제어기의 생명줄이 끊어졌을 때 찰나의 순간에 개입하여, 발산하려는 기체의 동역학을 리아프노프 안정성(Lyapunov Stability) 궤도 안으로 끌어내리는 이산 상태 전이(Discrete State Transition) 과정을 다룬다.

1. 리아프노프 안정성(Lyapunov Stability)과 페일세이프(Failsafe)의 동역학적 의미

제어 시스템이 교착 상태에 빠지면, 가장 최근에 내려진 위치/속도/자세 설정점(Setpoint)이 영차 파지(ZOH)에 의해 무한정 홀드(Hold)된다. 만약 그 마지막 명령이 “초속 5\text{m/s}로 전진하라“였다면, 기체는 통신이 끊긴 채 그 속도로 장애물에 돌진하는 런어웨이(Runaway) 발산 상태로 돌입한다. 이는 수학적으로 폐루프 시스템의 고유 성질이 리아프노프 불안정(Lyapunov Instability) 영역으로 넘어감을 뜻한다.

Ardupilot 통제 시스템은 이럴 경우 RTL(Return to Launch)이나 Land 모드로 신속히 전환하는 보편적 룰을 따르지만, PX4의 오프보드 대응 체계는 현재의 동역학적 관성 스페이스(Inertial Space)를 점진적으로 감쇠(Damping)시키는 다단 브레이크(Braking) 메커니즘을 보다 세밀하게 모델링한다.

1.1 하이패스 동역학의 제어권 탈취 (Control Authority Override)

COM_OF_LOSS_T 파라미터(기본 0.5\text{초}) 타이머가 만료되어 교착 상태가 확정된 순간, 비행 제어기 내부의 커맨더(Commander) 모듈은 외부에서 입력된 모든 궤적 설정점을 즉시 폐기(Drop)한다.
대신, 기체의 현재 센서 데이터(관성 항법 시스템 출력)를 초기 조건으로 삼아 위치 제어 루프의 내부 리퍼런스(Internal Reference) 생성기를 독자적으로 가동한다. 이는 다음과 같은 점진적 운동 에너지 소산(Energy Dissipation) 과정을 거쳐 기체를 리아프노프 안정한 원점(Origin) 평형 상태로 유도한다.

V(x) = \frac{1}{2} x^T P x > 0 \quad (\text{단}, x \neq 0)
\dot{V}(x) < 0

여기서 V(x)는 기체의 위치 오차 에너지 함수이며, PX4는 가속도 제어 브레이크를 걸어 \dot{V}(x)를 음수로 만듦으로써 기체를 강제 수렴시킨다.

2. 비상 안전(Failsafe) 이산 상태 기계(Discrete State Machine) 전이 로직

이러한 물리적 감쇠 과정은 소프트웨어 아키텍처 상에서 엄격한 이산 사건(Discrete Event) 기반의 상태 기계로 구현된다.

2.1 3단계 다중 모드 전이 체계 (Triple-phase State Transition)

  1. Active Braking (제동 감쇠):
    통신 유실 직전의 선속도(Linear Velocity) 벡터를 역으로 상쇄하기 위해 Hold 모드로 강제 스위칭되며, 최대 각가속도와 가속도를 제한하는 저크 제한(Jerk-limited) 브레이킹 궤적을 펌웨어가 자체 생성한다. 이를 통해 급정거 시 발생할 수 있는 피치(Pitch)/롤(Roll) 플립(Flip) 현상을 막고 운동 에너지를 공기 저항으로 소산시킨다.
  2. Position Hold (위치 고정 대기):
    속도가 영(0)에 수렴하면(리아프노프 안정점 도달), 기체는 현재의 X, Y, Z 좌표를 락(Lock) 상태로 유지한다. 이 상태에서 파라미터 COM_OBL_RC_ACT 에 따라 조종자가 수동 조종기(Radio Control)로 즉각적인 개입(Takeover)을 할 수 있도록 스루풋(Throughput)을 열어둔다.
  3. Automated Return / Land (자동 복귀 및 착륙):
    Hold 상태에서 일정 시간(예: 데이터 링크 유실 타임아웃 NAV_DLL_ACT)이 경과해도 동반 컴퓨터가 리부트(Reboot) 후 복구되지 않거나 복귀 명령이 없다면, 기체는 최종 페일세이프 상태인 RTL(복귀)이나 제자리 헬기 착륙을 감행하여 대기 공간에서의 에너지를 지면으로 완전히 소거시킨다.

결론적으로, PX4의 오프보드 교착 상태 대처 로직은 단순히 소프트웨어 프로세스 간의 예외 처리(Exception Handling)를 넘어선다. 그것은 중력과 관성이 지배하는 물리 공간에서 폭주하려는 강체의 에너지를 리아프노프 에너지 튜브(Energy Tube) 안으로 욱여넣어 질서 있는 정지 상태로 강제 회귀시키는 폐루프 동역학(Closed-loop Dynamics)의 최종 안전망이다.
