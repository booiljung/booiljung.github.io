63.7.1. ActuatorMotors 및 ActuatorServos 메시지를 통한 제어 할당 행렬(Control Allocation Matrix) 수반 행렬 역산 및 믹서(Mixer) 동역학 모델 우회 개념

1. 개요: 비행 제어기 내부의 제어 할당(Allocation) 파이프라인 우회

PX4-Autopilot과 같은 현대적인 비행 제어 소프트웨어는 기체의 하드웨어적 형상(멀티로터, 고정익, VTOL)에 상관없이 제어 알고리즘을 범용적으로 적용하기 위해 파이프라인을 다층적으로 추상화한다. 자세 제어기(Attitude Controller)와 각속도 제어기(Rate Controller)는 가상의 제어 모멘트(Virtual Control Moments: 롤, 피치, 요 토크 및 수직 추력)를 산출하며, 이 가상 제어량들은 “제어 할당기(Control Allocator, 또는 Mixer)“를 거쳐 실제 구성된 모터와 서보의 기계적 신호로 매핑된다.

오프보드(Offboard) 모드에서 ActuatorMotors 및 ActuatorServos 메시지를 직접 퍼블리싱(Publishing)한다는 것은, 동반 컴퓨터(Companion Computer) 내부의 자체 ROS 2 제어기가 이 모든 비행 역학(Flight Dynamics) 제어 레이어와 제어 할당 행렬 인버전 연산을 우회하여 궁극의 저수준(Low-level) 하드웨어 지시를 내림을 의미한다.

2. 제어 할당 행렬(Control Allocation Matrix) 모델링 차원

멀티로터의 제어 할당기 구동 원리를 수학적으로 이해해 보자. 가상의 제어 모멘트 벡터 \mathbf{v} = [T_x, T_y, T_z, F_z]^\top 가 주어졌을 때 (여기서 T는 토크, F_z는 총 수직 추력), 실제 n개의 모터가 발휘해야 하는 추력 벡터 \mathbf{u} = [u_1, u_2, \dots, u_n]^\top 와의 관계는 다항식과 기하학적 형상(Geometry)의 성질을 갖는 제어 할당 행렬 \mathbf{P} \in \mathbb{R}^{4 \times n} 로 표현된다.

\mathbf{v} = \mathbf{P} \mathbf{u}

  • \mathbf{P} 행렬의 각 열(Column)은 개별 모터의 설치 위치(무게중심으로부터의 x, y 거리)와 모터가 회전할 때 발생하는 반작용 토크 방향(시계방향/반시계방향 Constant)을 나타낸다.

3. 수반 행렬(Pseudo-inverse) 역산과 오버액추에이티드(Over-actuated) 구조

쿼드코프터(n=4)의 경우 \mathbf{P} 행렬은 4 \times 4 정방 행렬(Square Matrix)이 되며 역행렬이 존재할 가능성이 높지만, 헥사콥터나 옥토콥터 등 구동기가 4개보다 많은 오버액추에이티드(Over-actuated) 플랫폼의 경우 직사각형 행렬(4 \times 6, 4 \times 8) 형태가 된다.

이 경우 주어진 가상 모멘트 \mathbf{v}를 만족하는 모터 추력 \mathbf{u}의 해가 무수히 많게 되므로, PX4 내부 제어 할당기는 모터의 총 에너지 소모(Norm)를 최소화하기 위해 무어-펜로즈 유사 역행렬(Moore-Penrose Pseudo-inverse, \mathbf{P}^+)을 사용하여 최적해를 도출한다.

\mathbf{u} = \mathbf{P}^+ \mathbf{v} = \mathbf{P}^\top (\mathbf{P} \mathbf{P}^\top)^{-1} \mathbf{v}

2.1 믹서 동역학 우회의 의미

위와 같은 수반 행렬 방식은 에너지 최적성과 추력의 가용 한계를 동시에 달성하기 위한 복잡한 연산 과정을 거치며, 모터 한계 도달 시 우선 순위(Desaturation) 재분배 알고리즘 파이프라인을 수반한다.

ActuatorMotors 메시지를 사용하는 것은 PX4 내부의 이러한 기하학적 \mathbf{P} 형상 및 유사 역행렬 연산을 원천적으로 무효화(Bypass) 시키는 행위이다. 즉, 새로운 틸트로터나 특이한 형상의 드론을 제작하는 연구자의 경우, 복잡한 PX4 펌웨어 소스 코드를 빌드하고 Custom Mixer Matrix를 주입할 필요 없이, 동반 컴퓨터의 ROS 2 노드 자체 C++ 코드 내에서 형상 역역학(Inverse Geometry Dynamics)을 풀고 직접 \mathbf{u} 벡터를 송신하면 된다.

3. 아키텍처적 우회(Bypass) 매커니즘 및 메시지 분석

OffboardControlMode 메시지에서 direct_actuator=true 로스팅 플래그가 전산망에 퍼블리시되면, PX4의 최상단 커맨더 모듈은 내부 PID 제어기(Attitude/Rate)와 제어 할당기 모듈 간의 연결 스위치를 단절성 상태(Disconnected State)로 격리한다.

    A[ROS 2 Node] -->|ActuatorMotors| B(uORB: actuator_motors)
    A -->|OffboardControlMode| C(Commander)
    C -->|Bypass Signal| D[Control Allocator]
    E[PX4 Rate Controller] -.->|Ignored| D
    B --> F[Output Module PWM/DShot]
    D -.->|Silenced| F
  1. 이 단계에서 PX4의 상태 추정기(EKF2)나 자이로스코프(Gyroscope) 데이터 등은 정상 작동하지만, 그 결과가 믹서 출력에 관여하지 않게 된다.
  2. ActuatorMotors 토픽 값은 Output Module (예: fmu, pwm_out)로 직결되어, 설정된 파라미터(예: MOT_SLEW_MAX, PWM_MAIN_MIN / MAX)에 의존하는 최종 펄스 폭 매핑(Pulse Width Mapping) 및 슬루율 제한 단계 이외의 어떠한 소프트웨어적 간섭도 받지 않게 된다.

4. 기존 믹서 기반 방식 대 오프보드 직접 제어 방식 요약 비교

Ardupilot이 오랜 시간 커스텀 모터 할당을 위해 LUA 스크립트 기반 혼합(Mixing)을 제공하거나 백엔드 코드를 재빌드하는 방식을 거쳐야 했던 것과 달리, 현대 PX4는 uXRCE-DDS 레이어에 구동기(Actuator) 최말단 추상화를 직접 노출하여 높은 접근성을 달성했다.
