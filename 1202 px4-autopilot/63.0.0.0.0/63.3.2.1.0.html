63.3.2.1. 시동 전제 조건 검사(Pre-arm Check) 통과 여부의 논리곱(Logical AND) 연산 기반 상태 기계 진입 폴링(Polling) 알고리즘

자율 에이전트(Autonomous Agent)가 무장(Arming) 명령을 내리기 전, 기체의 총체적인 건전성(Health)을 확인하는 과정은 안전 비행의 절대적 필요 조건이다. PX4-Autopilot(v1.14 이상)은 센서들의 캘리브레이션 상태, EKF2의 오차 공분산, 그리고 전원 시스템의 안정성 등을 개별적으로 평가하는 방대한 ‘시동 전제 조건 검사(Pre-arm Check 또는 Preflight Check)’ 룰셋(Ruleset)을 운용한다. 본 절에서는 오프보드(Offboard) 노드가 이러한 이질적인 검사 항목들을 논리곱(Logical AND)으로 묶어 단 하나의 무결성 플래그로 치환하고, 이를 기반으로 상태 기계(State Machine) 진입을 폴링(Polling)하는 알고리즘의 제어 공학적 구현 방안을 논의한다.

1. 이산적(Discrete) 안전 지표의 논리곱(AND) 게이팅(Gating) 아키텍처

Ardupilot 통제 환경(Mission Planner 등)에서는 프리암(Pre-arm) 검사 실패 시 화면에 붉은색 텍스트(예: “PreArm: Compass not calibrated”)로 그 사유를 명시적으로 띠운다. 이를 MAVLink 코드로 파싱(Parsing)하여 에이전트가 예외 처리를 할 수는 있으나, 그 문자열 매칭(String Matching) 방식은 실시간성 제어 시스템에서 매우 불안정(Brittle)하다.

반면, PX4-ROS2 연동 체계에서는 기체의 안전 상태를 수학적인 불리언(Boolean) 배열로 추상화하여 제공한다. 특히 VehicleStatus 토픽 내의 플래그들과, 비행 제어기 내부의 system_power, sensor_gps 등의 하위 토픽 건전성을 종합하여 다음과 같은 형태의 논리 함수 F_{arm}(t)를 구축하는 것이 오프보드 제어기의 핵심 논리 회로(Logic Circuit)이다.

F_{arm}(t) = \bigwedge_{i=1}^{N} C_i(t) = C_{EKF\_pos} \wedge C_{EKF\_vel} \wedge C_{sensor} \wedge C_{battery} \wedge C_{datalink}

이 수식은 모든 개별 조건 C_i(t)가 동시에 True(1)를 만족해야만 최종적으로 무장 명령(VehicleCommand_ARM) 호출 권한이 스케줄러(Scheduler)로 넘어감을 의미한다. 단 하나의 센서라도 결함 플래그를 띄울 경우 F_{arm}(t)는 False(0)로 붕괴된다(단락 평가, Short-circuit Evaluation).

2. 블로킹 없는(Non-blocking) 폴링(Polling) 알고리즘 체계 설계

동반 컴퓨터에서 이러한 논리곱 게이팅을 구현할 때 가장 주의해야 할 소프트웨어 공학적 함정은 스레드 블로킹(Thread Blocking)을 유발하는 무한 대기 루프(예: while(!is_ready) { sleep(1); })의 사용이다.

ROS2(rclcpp/rclpy) 환경은 앞서 언급했듯 지속적인 콜백 처리가 생명인 비동기 기반(Asynchronous-driven) 시스템이다. 블로킹 루프를 사용하면 다른 센서 데이터(TrajectorySetpoint 스트리밍 등) 구독이 모두 정지되어 타임아웃(Timeout)에 걸리고 오히려 프리암 검사를 영원히 통과할 수 없는 데드락(Deadlock)에 빠지게 된다.

2.1 상태 기계 진입 폴링(State Machine Entry Polling) 로직

따라서 자율 이륙 어플리케이션은 내부적으로 자체적인 유한 상태 기계(FSM)를 시계 틱(Tick) 단위로 전진시키는 상태 기계 진입 폴링 기법으로 설계되어야 한다.

  1. 상태 정의: 노드는 INIT, WAIT_FOR_PREARM, ARMING, TAKEOFF 등의 내부 상태 열거형(Enum)을 가진다.
  2. 비동기 평가: 10\text{Hz} 주기의 타이머 콜백 내에서 현재 상태가 WAIT_FOR_PREARM일 경우 앞서 정의한 F_{arm}(t) 논리곱 연산을 1회 평가(Evaluate)한다.
  3. 천이 발생: 결과가 True이면 즉시 상태를 ARMING으로 전이하고, VehicleCommand를 비동기 송출한다. False라면 아무 행동도 하지 않고 다음 타이머 주기로 제어권을 즉각 운영체제(OS)에 반환(Yield)한다.

    G --> H[Publish VehicleCommand: ARM]

2.2 하위 노이즈 필터링: 디바운스(Debounce) 로직의 결합

실제 비행 환경에서는 GNSS 위성 개수의 변동이나 자기장 간섭으로 인해 특정 플래그 C_i(t)가 밀리초 단위로 0과 1을 오가는 플래핑(Flapping) 현상이 빈번하다.
제어 공학에서는 이를 해결하기 위해 디지털 필터의 일종인 디바운스(Debounce) 로직을 논리곱 연산기 이전에 배치한다. 즉, 특정 조건이 True로 판별되었다 하더라도 그것이 최소 T_{hold} 시간(예: 2\text{초}) 이상 연속 유지되었을 때만 최종 C_i(t)=1로 인정하는 슈미트 트리거(Schmitt Trigger)와 유사한 이력(Hysteresis) 현상을 인위적으로 부여하는 것이다.

결합하자면, 오프보드에서의 무장(Arming) 시퀀스는 단순한 명령어 발송 루틴이 아니라, 기류와 전파가 어지럽게 뒤섞인 환경 속에서 기체의 절대적인 건전성이 **확정적으로 보장된 시간적 창(Time Window)**을 찾아내기 위한 비동기 논리곱 연산 및 능동적 시계열 필터의 집약체 결실이다.
