63.1.3.1. 패킷 지향적 점대점(Point-to-Point) 통신과 데이터 중심의 발행-구독(Publish-Subscribe) 모델 간의 방향성 비순환 그래프(Directed Acyclic Graph) 라우팅 비교

최신 자율 비행 시스템(PX4 Autopilot v1.14 이상 탑재 기체)의 통신망 설계에 있어서, 제어 정보의 전달 방식은 단순한 데이터의 이동을 넘어 전체 시스템의 반응성과 확장성을 좌우하는 아키텍처의 근간이다. 무인항공기(UAV)와 지상 관제 시스템(GCS), 그리고 임무 컴퓨터(Companion Computer) 간의 네트워크는 크게 두 가지 패러다임으로 대비된다. 하나는 기존 MAVLink 프로토콜의 기반이 되는 패킷 지향적 점대점(Packet-Oriented Point-to-Point) 통신이며, 다른 하나는 ROS2와 uXRCE-DDS(Micro XRCE-DDS) 미들웨어가 채택하고 있는 데이터 중심의 발행-구독(Data-Centric Publish-Subscribe) 모델이다.

본 절에서는 이 두 가지 데이터 라우팅 방식의 차이를 방향성 비순환 그래프(DAG, Directed Acyclic Graph) 이론을 도입하여 네트워크 토폴로지 관점에서 엄밀하게 비교하고 분석한다.

1. MAVLink의 패킷 지향적 점대점 라우팅 구조

MAVLink 프로토콜은 본질적으로 특정 주소(System ID 및 Component ID)를 명시하여 패킷을 전송하는 어드레싱(Addressing) 기반의 점대점(Point-to-Point) 통신 모델을 취한다.

1.1 정적 라우팅 트리(Static Routing Tree)

이 방식에서는 메시지의 발신자(Sender)와 수신자(Receiver) 간의 경로가 명시적이다. 정보 전파 과정은 트리(Tree) 또는 스타(Star) 토폴로지에 가까운 제한적인 방향성 그래프를 형성한다. QGroundControl(v4.3 이상)과 같은 단일 마스터 노드가 여러 노드 컴포넌트에게 제어 명령(Command)을 하달할 때, 패킷은 지정된 경로(직렬 포트, 특정한 UDP 포트 등)만을 거치게 된다.

1.2 그래프 이론적 제약 요인

  • 강한 결합도(Tight Coupling): 송신 노드와 수신 노드 사이에 직접적인 엣지(Edge)가 형성되어야만 한다.
  • 라우팅 병목(Routing Bottleneck): 시스템 엔티티가 늘어날 경우, MAVLink 라우터 모듈에 모든 트래픽이 집중되며, 다중 홉(Multi-hop) 라우팅 구성 시 그래프 상의 특정 노드가 단일 장애점(SPOF, Single Point of Failure)이 될 확률이 비약적으로 상승한다.

2. ROS2 uXRCE-DDS의 데이터 중심 발행-구독 라우팅 구조

반면, PX4의 최신 Offboard 제어에서 널리 활용되는 ROS2 uXRCE-DDS 미들웨어는 **데이터 중심(Data-Centric)**의 라우팅 아키텍처를 제공한다. 여기서는 ‘누구에게’ 데이터를 보낼지가 아니라, ’어떠한 주제(Topic)’의 데이터를 다루는지가 네트워크 라우팅의 핵심 기준이 된다.

2.1 분산 글로벌 데이터 공간(Global Data Space)

발행-구독 모델은 발행자(Publisher)와 구독자(Subscriber) 사이의 직접적인 연결을 요구하지 않으며(Decoupling), DDS 표준의 글로벌 데이터 공간 개념을 통해 각 참여자가 관심 있는 토픽만 선별적으로 수신한다.

2.2 방향성 비순환 그래프(DAG) 형성의 동적 특성

이 네트워크에서 노드(ROS2 Node 또는 PX4 내부 uORB 클라이언트)를 정점(Vertex)으로, 토픽의 흐름을 방향성 간선(Directed Edge)으로 정의할 때, 런타임 중에 노드의 디스커버리 매커니즘(Discovery Mechanism)에 의해 매우 거대하고 다차원적인 **방향성 비순환 그래프(DAG)**가 동적으로 구성된다.

  • 시스템은 피드백 루프(Feedback Loop)에 의한 무한 순환(Infinite Cycle)을 방지하도록 DDS 차원 혹은 uORB 매커니즘 차원에서 위상 정렬(Topological Sorting)이 보장된 방향성 비순환 정보망을 유지한다.
  • 하나의 발행자에서 여러 구독자로 이어지는 정보 다중화(Fan-out)와 여러 센서 데이터가 하나의 제어 노드로 집중되는 정보 융합(Fan-in)이 네트워크 스택 병목 없이 유연하게 그려진다.

4. 로보틱스 정보 이론 관점에서의 결론 및 시사점

두 라우팅 패러다임은 각각 최적의 응용 분야가 구분된다.

  1. 상태 관리 및 강건 결합 필요성: 기본 비행 제어 및 파라미터 캘리브레이션 등 엄격한 순서 보장과 단순한 에러 복구(ACK/NACK)가 요구될 경우 MAVLink의 점대점 구조가 안전망(Safety Net) 측면에서 유리하다.
  2. 다차원 센서 퓨전 및 자율 비행 확장성: 분산 시스템 환경에서 VIO 연동, 라이다 추적, 경로 계획 등 데이터 중심의 워크로드가 폭증할 때, 시스템은 ROS2 uXRCE-DDS의 DAG 기반 발행-구독 모델 없이는 통신망 병목을 회피할 수 없다. 동적인 토픽 그루핑 메커니즘은 노드의 추가/제거에도 아키텍처의 재설계를 요구하지 않는 극한의 유연성(Flexibility)을 부여한다.

최종적으로, 현대의 복합 PX4-Autopilot 기반 무인기 플랫폼은 두 가지 라우팅 그래프 네트워크를 병렬 체계로 구성함으로써, MAVLink의 강건성과 DDS 아키텍처의 데이터 확장성이라는 두 마리 토끼를 동시에 취하고 있다.
