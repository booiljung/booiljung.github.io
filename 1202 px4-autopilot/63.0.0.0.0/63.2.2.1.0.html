63.2.2.1. 기체 제어 모드 설정 토픽(OffboardControlMode)의 다중 제어 루프 스위칭을 위한 불리언(Boolean) 비트마스크(Bitmask) 체계 분석

자율 비행 기반의 무인항공기 관제에 있어 다중 제어 루프(Multi-loop Control)의 동적 할당과 권한 위임은 시스템의 안정성을 책임지는 가장 핵심적인 메커니즘이다. PX4-Autopilot 환경 하에서 이러한 비행 제어기(FC: Flight Controller) 내부의 캐스케이드(Cascade) 제어 루프들을 외부 클라이언트(예: 지상 관제 시스템 및 ROS2 기반 동반 컴퓨터)가 활성화 또는 비활성화할 수 있도록 매개하는 중추적 토픽이 바로 OffboardControlMode (오프보드 제어 모드)이다. 본 절에서는 해당 토픽의 불리언(Boolean) 비트마스크 구조 및 모드 스위칭 시 발생하는 내부 알고리즘의 제어 공학적 특성을 분석한다.

1. OffboardControlMode 토픽의 설계 철학과 구조적 의의

Ardupilot은 MAVLink 프로토콜의 SET_POSITION_TARGET_LOCAL_NED 메시지 내부에 포함된 type_mask 필드를 사용하여 제어하고자 하는 목표 변수를 비트 단위로 마스킹(Masking)하는 방식을 취한다. 그러나 PX4-Autopilot은 이와 달리 유연하고 확장성 있는 구조를 갖추기 위하여, 제어 모드를 설정하는 논리적 스위치(OffboardControlMode)와 실제 물리적 제어 설정값(TrajectorySetpoint, VehicleRatesSetpoint 등)을 구조적으로 철저히 분리(Decoupling)하였다.

이러한 설계 철학은 ROS2 uXRCE-DDS 퍼블리셔(Publisher)가 단순히 수치 데이터를 전송하기 전 해당 데이터가 어느 제어 계층 구조(Control Hierarchy)를 자극할지 사전에 비트마스크 형태로 명시함으로써 제어기 런타임 중에 예측 불가능한 스위칭 지연을 방지하기 위함이다.

1.1 불리언 비트마스크의 세부 필드 및 논리 구조

PX4 v1.14 이상의 펌웨어를 기준으로 OffboardControlMode 구조체는 아래와 같이 상호 배타적이면서도 보완적인 불리언 필드들을 제공한다. 이 필드들은 uORB 메시지 포맷 내에서 1바이트(Byte) 이하의 가벼운 비트 연산 기반의 마스크로 작용한다.

  • position (위치 제어 루프 활성화): 3차원 공간상의 위치(Position) 오차를 기반으로 속도 지령을 생성하는 최상위 외곽 루프(Outer Loop)를 활성화한다.
  • velocity (선속도 제어 루프 활성화): 선속도(Linear Velocity) 오차를 보상하여 가속도 지령을 산출한다.
  • acceleration (선가속도 제어 루프 활성화): 제어기의 피드포워드(Feedforward) 응답성을 높이거나 저크(Jerk) 제한 궤적 생성 시 사용된다.
  • attitude (자세 제어 루프 활성화): 회전 동역학에 의한 롤, 피치, 요(Roll, Pitch, Yaw)의 쿼터니언(Quaternion) 방위 각도를 제어한다.
  • body_rate (내부 각속도 제어 루프 활성화): 기체 좌표계(Body Frame) 기준의 최내곽(Inner) 각속도 제어 루프를 활성화하며 빠른 응답성을 요구하는 회피 동역학 등에 적합하다.
  • actuator / direct_actuator (구동기 직접 제어): 캐스케이드 제어기를 완전히 우회하여 ESC 및 서보 모터(Servo Motor)에 원시(Raw) PWM/DShot 신호 배분에 개입한다.

2. 제어 공학적 스위칭 알고리즘 분석

2.1 다중 루프 스위칭과 범프리스 전송(Bumpless Transfer) 문제

지상 관제 시스템(QGroundControl 인터페이스) 또는 자율 에이전트가 예기치 않게 제어 권한을 획득하거나 반납할 때 가장 큰 문제는 ’스위칭 충격(Switching Transient)’이다. 위치 제어(Position)에서 각속도 제어(Body Rate)로 비트마스크가 변경되는 찰나의 순간, 기존 제어기의 적분기(Integrator)에 누적되어 있던 오차 값인 와인드업(Windup) 현상이 새로운 제어 루프의 초기 상태에 영향을 주어 급격한 제어 입력을 인가하는 현상이 발생할 수 있다.

이를 방지하기 위해 PX4의 내부 다중 루프 스위칭 로직은 OffboardControlMode가 갱신될 때마다 비활성화되는 루프의 적분기 항을 초기화(Reset)하거나, 이전 상태의 지령치를 새로운 제어기의 초기 조건(Initial Condition)으로 정합(Matching)시키는 알고리즘을 수행한다.

2.2 폴링(Polling) 및 하트비트(Heartbeat) 요구사항

OffboardControlMode는 1회성(Single-shot) 설정이 불가능하다. PX4-Autopilot의 Failsafe 철학에 입각하여, 오프보드 제어를 인가하기 위해서는 해당 토픽과 함께 목표 설정값 토픽(예: TrajectorySetpoint)이 최소 2Hz(권장 50~100Hz) 이상의 주기로 지속 발행(Publish)되어야 한다.

만일 OFFBOARD_TIMEOUT(기본값 0.5초) 이상 해당 비트마스크 신호의 갱신이 누락될 경우, 비행 제어기의 유한 상태 기계(FSM: Finite State Machine)는 즉각적으로 해당 권한을 박탈하고 복귀(RTL)나 제자리에 머물기(Hold) 상태로 하향 천이(Fallback)한다. 이는 ROS2 미들웨어의 QoS(Quality of Service) 설정 중 DEADLINE 폴리시와 연계되어 실시간 제어기의 생존력을 체계적으로 보장하는 논리 기반이 된다.

요약하자면, OffboardControlMode 구조체는 기계적인 데이터 패킷의 나열을 넘어, 복잡한 캐스케이드 이중 제어기(Dual-controller)의 각 분기를 정밀히 단속하는 매우 중추적인 스택 스위치 네트워크로 이해되어야 한다.
