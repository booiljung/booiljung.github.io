63.5.3.1 현재 기체 중심점 벡터와 목표 공간 벡터 간의 3차원 유클리드 노름(Euclidean Norm) 거리 오차 산출 기반 경유지 수렴 경계 조건(Acceptance Radius) 판별 로직

자율 에이전트가 오프보드(Offboard) 모드에서 다수의 3차원 웨이포인트를 순차적으로 비행할 때, 수학적으로 기체가 정확히 부피가 없는 한 ’점(Point)’인 목표 좌표계에 완벽히 일치하는 것은 공기 역학적 교란(Aerodynamic Disturbance)과 센서 노이즈로 인해 물리적으로 불가능하다. 따라서 제어 공학에서는 기체가 목표 지점 반경 내에 진입했을 때 도달(Reached)한 것으로 간주하고 다음 목표로 궤적을 전환하는 수렴 경계 조건(Acceptance Radius) 판별 로직을 필수적으로 적용해야 한다.

1. 3차원 유클리드 노름(Euclidean Norm) 및 거리 오차 산출

어떤 위치를 표현하는 두 개의 3차원 유클리드 공간 벡터 \mathbf{p}_{current} = [x_c, y_c, z_c]^T (현재 기체 중심점 벡터)와 \mathbf{p}_{target} = [x_t, y_t, z_t]^T (현재 목표 공간 벡터)가 주어졌을 때, 두 벡터 간의 직선거리는 L^2 노름(Euclidean Norm) 오차 방정식으로 표현된다.

거리 d는 다음과 같이 정의된다:

d = \Vert \mathbf{p}_{target} - \mathbf{p}_{current} \Vert_2 = \sqrt{(x_t - x_c)^2 + (y_t - y_c)^2 + (z_t - z_c)^2}

이 d 값이 개발자가 사전 정의한 수렴 경계 임계값(Threshold)인 수용 반경(Acceptance Radius, R_a)보다 작거나 같아지는 순간(d \le R_a), 기체는 해당 웨이포인트를 성공적으로 추종한 것으로 간주된다.

2. ROS2 C++ 노드 체계에서의 상태 판별 로직 구현

PX4-Autopilot 생태계(v1.14 기준)와 ROS2 데이터 브릿징 환경에서 기체의 현재 위치 피드백은 주로 px4_msgs::msg::VehicleOdometry (또는 VehicleLocalPosition) 토픽을 통해 고주파수로 수신된다. 이 위치 데이터는 PX4 로컬 프레임(NED)을 따르므로, 타겟 역학 프레임 역시 NED로 통일시킨 후 연산하는 것이 계산 비용 측면에서 유리하다.

C++ Eigen 라이브러리를 활용한 판별 구현 예시:

#include <Eigen/Dense>

// 사전 정의된 수용 반경 (단위: 미터)
const double ACCEPTANCE_RADIUS = 0.5;

/**
 * @brief 현재 기체 측정 위치와 타겟 간의 수렴 여부 판별
 * @param current_pos_ned VehicleOdometry에서 수신한 NED 위치 벡터
 * @param target_pos_ned 현재 제어 목표인 NED 공간 벡터
 * @return true 목표 도달, false 도달 중
 */
bool check_waypoint_reached(const Eigen::Vector3d& current_pos_ned, 
                            const Eigen::Vector3d& target_pos_ned) 
{
    // Eigen 라이브러리의 norm() 메서드를 사용하여 L2 노름(3차원 유클리드 거리) 계산
    double distance_error = (target_pos_ned - current_pos_ned).norm();
    
    // 수렴 경계 조건 평가
    if (distance_error <= ACCEPTANCE_RADIUS) {
        // [디버그/로그] "Waypoint reached! Distance error: " + distance_error
        return true; 
    }
    return false;
}

제어 타이머 콜백 내에서는 위 check_waypoint_reached 함수를 매 루프마다 호출하여 상태를 감시한다.

// ROS2 제어 루프 내부 발췌
if (check_waypoint_reached(current_position_, target_waypoints_[current_wp_index_])) {
    current_wp_index_++; // 다음 목표로 타겟 전환
    if (current_wp_index_ >= target_waypoints_.size()) {
        current_state_ = State::LAND; // 모든 임무 완료 시 착륙으로 상태 전이
    }
}

3. 동적 수용 반경(Dynamic Acceptance Radius) 튜닝 파라미터 최적화

수용 반경 R_a의 크기 설정은 비행체 제어 성능과 타협점(Trade-off)을 갖는다.

  • R_a가 너무 클 경우: 기체가 웨이포인트에 근접하기도 전에 다음 목표로 선회하는 코너 컷팅(Corner Cutting) 현상이 심해져 정밀한 궤적 추종이 불가능해진다.
  • R_a가 너무 작을 경우: 시스템의 위치 오차가 R_a 보다 크게 유지되면(정상 상태 오차, Steady-state Error), 기체가 웨이포인트 주위를 맴도며 끝없이 진동하는 토일렛 볼 이펙트(Toilet Bowl Effect) 또는 고착 상태(Stuck)에 빠질 수 있다.

따라서 고속 비행 시에는 기체의 관성력과 제동 거리를 반영하여 선속도(Linear Velocity)에 비례하게 수용 반경을 확장하는 동적(Dynamic) 수용 반경 알고리즘을 적용하는 것이 고급 ROS2 Offboard 제어 기술의 모범 사례(Best Practice)이다.
