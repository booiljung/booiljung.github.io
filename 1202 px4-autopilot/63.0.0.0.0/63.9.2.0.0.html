63.9.2 임무 컴퓨터와 비행 기체 제어기 간의 범용 비동기 송수신기(UART) 직렬 버스 상실(Data Link Loss) 이벤트 트리거 시 PX4 내부의 자동 비상 복구(출발점 귀환, 제자리 체공, 비상 하강) 유한 상태 전이 논리 회로 설계

다중 시스템이 결합된 상업용 및 연구용 자율 비행 드론의 경우, 실시간 영상 처리나 복잡한 임무 계산을 수행하는 둔중한 임무 컴퓨터(Mission Computer, 예: Nvidia Jetson, Raspberry Pi)와 비행 동역학의 낮은 지연 시간(Low Latency) 제어를 전담하는 PX4-Autopilot 비행 제어기(FC, Flight Controller) 간의 결합이 필수적이다. 두 장치 간에는 대부분 데이터 직렬화가 용이한 범용 비동기 송수신기(UART, Universal Asynchronous Receiver-Transmitter) 기반의 MAVLink 또는 마이크로-ROS(uXRCE-DDS) 통신 버스가 활용된다.

그러나 비행 중 겪게 되는 극심한 고주파 진동, 케이블 핀의 기계적 열화, 전자기 간섭(EMI, Electromagnetic Interference), 내부 버스 포화 현상(Bus Saturation) 등으로 인해 물리적 결선이 끊어지거나 데이터 교환이 일시적으로 정지되는 데이터 링크 상실(Data Link Loss) 현상이 언제든지 일어날 수 있다. 따라서 PX4 내부의 내결함성(Fault Tolerance) 모듈은 이러한 이벤트를 즉각 감지하고, 기체 및 지상 환경의 안전을 도모할 수 있는 견고한 유한 상태 전이(FSM, Finite State Machine) 회로를 내장하고 있다.

본 절에서는 PX4-Autopilot의 커맨더(Commander) 내부 모듈이 구현하는 오프보드(Offboard) 데이터 링크 상실 이벤트 트리거 메커니즘과 각 비상 복구 행위에 대한 논리 전이 구조(PX4 v1.14 펌웨어 기준)를 심층 분석한다.

1. 데이터 링크 상실 감지 임계점(Threshold) 및 아키텍처

오프보드 모드는 외부의 임무 컴퓨터가 PX4의 내부 위치 및 자세 제어 루프를 완전히 장악하는 모드이므로, 통신 상실은 곧 “기체의 뇌사 상태“를 의미할 수 있다. PX4의 메인 상태 결정자 스레드인 commander_main 로직은 주기적으로 uORB 메시지의 최신성을 검증한다.

임무 컴퓨터가 PX4에 보내는 offboard_control_mode 또는 제어 위상 목표치(TrajectorySetpoint, VehicleRatesSetpoint 등) 메시지의 타임스탬프(Timestamp)가 직전 수신 시간 대비 파라미터에 정의된 최장 허용 유예 기간을 넘길 경우, 시스템은 결함(Fault) 상태를 활성화한다.

  • COM_OF_LOSS_T: Offboard 데이터 통신 상실로 판단하기까지의 타임아웃 지연 시간 파라미터이다 (단위: 초, 기본값: 1.0s ~ 2.0s 내외).
  • 타임아웃이 초과되는 순간 PX4 커맨더는 내부 메시지 버스에 이벤트 트리거를 전파하고, 즉각 현재 항법 상태(Navigation State)를 “안전 강제 대기(Offboard Loss Failsafe)” 절차로 우회(Override)시킨다.

2. 오프보드 상실 시 자동 비상 복구 행동 인자 파라미터 분석

데이터 링크 상실로 전환될 때의 행동 양식은 현재 조종사의 수동 무선 조종기(RC, Radio Controller) 연결 유무와 기체의 지역 좌표 위치 측정(Local Position Estimate) 가용 상태에 따라 여러 갈래로 분기된다.

2.1 RC 조종기가 연결되어 있지 않은 순수 무인 자율 비행의 경우 (COM_OBL_ACT)

인간 조종사의 직접 개입이 불가능한 환경에서는 파라미터 COM_OBL_ACT(Offboard Loss Action)에 의해 완전 자율적인 비상 보호 행동이 선택된다.

  1. **0 (Return Mode - 출발점 귀환) **: 가장 많이 쓰이는 권장 팩토리 기본값(Default)이다. 기체가 즉시 RTL(Return to Launch) 유한 상태 전이로 진입한다. 안전 고도 파라미터(RTL_RETURN_ALT)까지 수직 상승한 후, 시동을 인가했던 출발 위치 혹은 정해진 홈(Home), 랠리 포인트(Rally Point) 좌표로 자율 귀환하여 자동 착륙(RTL_DESCEND_ALT)한다.
  2. **1 (Land Mode - 비상 착륙) **: 현재 머무르고 있는 2차원 평면 좌표(X, Y)를 그대로 유지한 채 즉각적인 수직 비상 하강(Descend and Land)를 수행한다. 배터리 소진이 임박했거나 장애물 회피를 보장할 수 없고, 지정된 궤도 밖으로 나가는 것을 극도로 기피하는 환경(예: 실내 비행망 내부)에 적합하다.
  3. **2 (Hold Mode - 제자리 체공) **: 현재 3차원 위치(Longitude, Latitude, Altitude)를 동기화하여 공중에 고정된 채 임무를 유보(Hovering)하는 모드이다. 링크 회복이나 2차 네트워크의 개입 전까지 대기하는 시나리오에 널리 활용된다.

2.2 RC 조종기가 예비(Backup)로 연결된 경우 (COM_OBL_RC_ACT)

조종사가 기체를 육안으로 모니터링하며 수동 스틱 제어를 할 준비가 된 상황에서 적용되는 파라미터이다.

  • 0 (Position Control로 권한 이양): 오프보드 모듈이 멈추면 기체의 제어 권한을 부드럽게(Bump-less) 조종기의 RC 스틱 매핑으로 넘기며 “Position 모드(위치 제어)” 혹은 “Altitude 모드“로 안전하게 강등(Fallback)시킨다. 조종사가 즉각 스틱을 쳐서 회피 기동을 할 수 있다.
  • 1 (오프보드 규칙 우선 추종): 조종사가 연결되어 있더라도 COM_OBL_ACT 파라미터의 명령(RTL, Land, Hold)을 우선적으로 추종하여 기계 로직이 직접 처결한다.

3. 유한 상태 전이(Finite State Machine) 회로 모식도

이러한 복합적인 상호작용과 파라미터 개입은 아래의 상태 전이 설계 논리 회로도로 명확히 표현할 수 있다.

stateDiagram-v2
    [*] --> Offboard_Control_Active : 임무 컴퓨터 제어 (Normal)
    
    state Offboard_Control_Active {
        direction LR
        Valid_Heartbeat --> Valid_Heartbeat : t < COM_OF_LOSS_T
    }
    
    Offboard_Control_Active --> Data_Link_Loss_Detected : t >= COM_OF_LOSS_T 발생
    
    state Data_Link_Loss_Detected {
        state "RC Available Check" as RC_Check
        RC_Check --> RC_Present : RC Alive
        RC_Check --> RC_Absent : No RC/Lost
        
        RC_Present --> Act_RC_Param : COM_OBL_RC_ACT 평가
        Act_RC_Param --> Fallback_Position : Set to 0 (Fallback to PosCTL)
        Act_RC_Param --> Check_Global_Param : Set to 1 (Obey OBL_ACT)
        
        RC_Absent --> Check_Global_Param : COM_OBL_ACT 평가
        
        state "Evaluate COM_OBL_ACT" as Evaluate_ACT
        Check_Global_Param --> Evaluate_ACT
        
        Evaluate_ACT --> State_Return : 0 (RTL/Return)
        Evaluate_ACT --> State_Land : 1 (Land)
        Evaluate_ACT --> State_Hold : 2 (Hold/Loiter)
    }
    
    Fallback_Position --> Position_Mode_Active : 인간 조종 개입
    State_Return --> RTL_Sequence_Active : 상승/이동/착륙 자동 전이
    State_Land --> Land_Sequence_Active : 제자리 수직 하강
    State_Hold --> Loiter_Sequence_Active : 현 위치 고도 유지

4. 데이터 링크 회복(Link Recovery) 시 재개(Resume) 로직 설계 시 유의점

임무 컴퓨터 제어 노드의 프로세스 재시작(Restart)이나 케이블 통신 노이즈 해소로 인해 UART 직렬 버스 데이터 플로우가 다시 복구되었다고 가정해보자. PX4는 복구되는 즉시 기체의 통제권을 즉각 이전의 임무 컴퓨터에게 기계적으로 반환하지 않는다. 이는 이전 궤적이 만료(Stale) 되었을 수 있고, 예기치 않은 급기동으로 인한 공기 역학적 스트레스를 예방하기 위함이다.

  • 이미 RTL이나 Land 상태 전이로 빠진 시스템은 해당 “Failsafe Sequence“를 기본적으로 끝까지 지속한다.
  • 개발자는 임무 컴퓨터의 ROS2 코드 안착 시점에 통상적인 오프보드 모드 요청 프로토콜(예: VehicleCommand::VEHICLE_CMD_DO_SET_MODE 발송)을 발동하여, PX4가 명시적(Explicit) 승인 플래그를 올려 다시 제어 모드(Mode Switch)를 Offboard 로 재전환 하도록 설계해야 한다. 이것은 기체 이착륙의 전체 안전을 담보하는 필수적인 페일-세이프 아키텍처(Failsafe Architecture) 설계 기법이다.
