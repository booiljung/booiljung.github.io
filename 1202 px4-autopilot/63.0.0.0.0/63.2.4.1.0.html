63.2.4.1. 데이터 전송 신뢰성 정책: 무결성 검증 생략(Best Effort) 방식과 재전송 기반 신뢰성(Reliable) 보장 방식이 제어 시스템의 극점(Pole) 배치 및 댐핑 계수(Damping Ratio)에 미치는 영향

ROS2 아키텍처를 도입한 PX4-Autopilot(v1.14 이상)에서 비행 제어 루프를 오프보드(Offboard) 노드로 확장할 때, 가장 빈번하게 발생하는 동역학적 불안정성(Instability)의 원인은 미들웨어의 ‘신뢰성(Reliability)’ QoS 프로파일 오설정에 기인한다. 본 절에서는 데이터 송수신 파이프라인에서 무결성 검증을 생략하는 Best Effort 방식과, 패킷의 도달을 엄격히 추적하여 재전송하는 Reliable 방식이 기체의 위치 및 자세 제어기의 폐루프 극점(Closed-loop Pole) 위치와 댐핑 계수(Damping Ratio, \zeta)에 어떠한 수학적 파급을 미치는지 제어 공학의 패러다임으로 분석한다.

1. 전송 신뢰성 정책과 제어 루프의 지연 모델링 (Delay Modeling)

MAVLink 기반 시스템(예: Ardupilot)은 본질적으로 비연결성 UDP(User Datagram Protocol) 소켓에 의존하므로 텔레메트리나 제어 목표값이 단발성(Best Effort) 성격을 지닌다. 반면 ROS2 uXRCE-DDS 브릿지는 TCP 메커니즘과 유사한 Reliable 정책을 옵션으로 제공한다. 문제는 이 ’재전송 메커니즘’이 제어 루프 체인에 포함될 경우 발생한다.

수학적으로, 샘플링 시간(Sampling Time) T_s를 갖는 이산 시간 제어 시스템에서 통신 지연(Delay) \tau는 플랜트 모델 G(s) 앞에 e^{-\tau s}의 순수 시간 지연 요소로 합성된다.

  • Best Effort 방식: 네트워크 혼잡 시 패킷이 완전히 유실(Loss)된다. 비행 제어기의 영차 파지(ZOH: Zero-Order Hold) 회로가 이전 프레임의 제어값을 그대로 유지하므로 지연 변동폭(Jitter)이 닫힌 구간 내에 머문다.
  • Reliable 방식: 수신 확인(ACK)이 없으면 패킷을 대기열에 쌓아두고 재전송한다. 이 과정에서 \tau는 예측 불가능한 비선형적 램프 함수(Ramp Function) 형태로 폭증하며, ’블로킹(Blocking) 현상’을 유발하여 \tau > T_s를 아득히 초과하게 된다.

1.1 극점(Pole) 변위와 주파수 응답 공간의 붕괴

선형 제어 이론의 근궤적법(Root Locus) 측면에서, 시간 지연 \tau의 증가는 개루프 전달 함수의 위상 그래프를 가파르게 하강시킨다.
\angle G(j\omega) e^{-j \tau \omega} = \angle G(j\omega) - \tau \omega

Reliable 정책으로 인해 연속적인 재전송이 발생하면 -\tau \omega 패널티가 기하급수적으로 커져 시스템의 위상 여유(Phase Margin)가 0도를 돌파(Negative Phase Margin)하게 된다. 이렇게 되면 복소 평면(Complex Plane) 상 좌반평면(LHP)에 위치하던 안정적인 지배 극점(Dominant Poles)들이 이매지너리(Imaginary) 축을 넘어 우반평면(RHP)으로 이동하며 댐핑 계수 \zeta가 음수(-ve)가 되는 발산(Divergence) 혹은 강한 발진(Oscillation)을 촉발한다.

2. 정책 선택에 따른 제어 강건성(Robustness) 설계 지침

따라서 PX4-Autopilot의 FSM 로직과 다차원 병진 동역학 루프는 각 토픽의 제어 주기율표(Rate Table)에 맞게 철저히 ’QoS 분리 설계(Decoupled Design)’를 적용하도록 권장한다.

2.1 고속 제어기(TrajectorySetpoint, VehicleRatesSetpoint)

50\text{Hz} 이상의 주파수로 스트리밍되는 오프보드 궤적 추종 루프는 반드시 Best Effort 정책을 채택해야 한다. 통신 교란 시 한두 개의 패킷을 버림으로써 순수 시간 지연 e^{-\tau s} 항의 비대화를 차단하고, EKF2의 상태 추정치 외삽법(Extrapolation)을 활용하는 것이 댐핑 계수 유지에 절대적으로 유리하다. 이를 위반할 시 기체는 이른바 “변기통 현상(Toilet-bowling)“이라 불리는 위상 지연성 궤적 발산을 일으킨다.

2.2 이산 모드 전환 스위치(VehicleCommand, OffboardControlMode)

초당 1\sim2\text{Hz} 주기로 발행되며 기체의 전체 생명주기(Lifecycle) 및 무장 상태를 통제하는 이산 모델 토픽들은 Reliable 정책이 강제된다. 이는 제어 루프의 극점 배치에는 관여하지 않으나(외곽 루프 밖의 분기(Branch) 로직이므로), 명령의 도달 무결성(Integrity) 자체가 하향 링크 타임아웃(Timeout) 방어 기제의 근간이 되기 때문이다.

결론적으로 ROS2 아키텍처 환경에서는 단순한 기능 구현(Implementation)을 넘어 데이터 수명(Durability)과 지연 공간(Delay Margin)을 제어 모델링(Control Modeling)의 1급 객체(First-class Citizen)로 격상시키는 시스템 엔지니어링적 통찰이 요구된다.
