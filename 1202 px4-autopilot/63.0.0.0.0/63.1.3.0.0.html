63.1.3. 직렬 통신(MAVLink) 기반 제어와 미들웨어(ROS2 uXRCE-DDS) 기반 제어의 정보 이론적 아키텍처 비교

무인항공기(UAV, Unmanned Aerial Vehicle) 시스템을 위한 PX4-Autopilot의 발전 과정에서, 외부 시스템(지상 관제 시스템 및 임무 컴퓨터)과의 통신 아키텍처는 기술적 전환기를 맞이하고 있다. 전통적으로 PX4-Autopilot은 경량화된 직렬 통신 프로토콜인 **MAVLink(Micro Air Vehicle Link)**를 표준 미들웨어로 채택해왔으나, 분산 제어 환경과 높은 확장성을 요구하는 최신 자율비행 시스템에서는 ROS2 uXRCE-DDS(Micro eXtremely Resource Constrained Environments - Data Distribution Service) 미들웨어로 무게중심이 이동하고 있다. PX4 펌웨어 버전 v1.14 이상에서 공식 편입된 uXRCE-DDS 연동 체계는 기존 MAVLink 중심의 시스템 아키텍처 패러다임을 혁신적으로 재구성한다.

본 절에서는 MAVLink 기반 제어 모델과 ROS2 uXRCE-DDS 기반 제어 모델을 정보 이론(Information Theory) 및 네트워크 토폴로지(Network Topology) 관점에서 엄밀하게 비교하고 분석한다.

1. MAVLink 기반 제어 아키텍처의 정보 이론적 특성

MAVLink 프로토콜은 매우 제한된 대역폭(Bandwidth) 하에서도 높은 신뢰성을 담보하기 위해 고안된 **패킷 중심 마샬링 프로토콜(Packet-centric Marshalling Protocol)**이다. 이는 GCS(Ground Control Station) 환경(예: QGroundControl v4.3 이상) 및 제한적인 텔레메트리 네트워크에서 효율적인 제어를 가능케 한다.

1.1 점대점(Point-to-Point) 및 스타 토폴로지 기반 데이터 흐름

MAVLink는 근본적으로 직렬 통신(UART 기반)의 마스터-슬레이브(Master-Slave) 혹은 라우팅 트리를 통한 점대점 아키텍처를 따른다. 제어 명령(Command)과 텔레메트리(Telemetry) 상태 정보는 고정된 시스템 ID(System ID)와 컴포넌트 ID(Component ID)를 기반으로 단일 경로를 통해 전송된다. 정보 이론 관점에서 이러한 구조는 채널 인코딩의 단순성을 보장하며, 잡음이 많은 무선 연결 환경(Noisy Channel)에서 섀넌 대역폭 한계(Shannon-Hartley Theorem) 내에서의 패킷 손실 확률을 낮춘다.

1.2 모놀리식(Monolithic) 명령 처리 구조

MAVLink 통신망에서 Offboard 시스템의 제어 명령은 COMMAND_LONG, SET_POSITION_TARGET_LOCAL_NED 등의 전용 메시지 포맷으로 한정되어 전송된다. PX4 내부에서는 의존성을 가진 MAVLink 모듈(MAVLink Receiver Module)이 이 직렬화된 바이트 스트림을 해석하여 동기적(Synchronous)으로 시스템 커맨더(Commander) 또는 내부의 uORB(Micro Object Request Broker) 메시지로 변환한다. 이 과정은 패킷 해석에 의한 지연(Parsing Overhead)과 직렬화 체인의 결속성을 높이는 특징이 있다.

2. ROS2 uXRCE-DDS 기반 제어 아키텍처의 정보 이론적 접근

DDS 기반 미들웨어 연동 체계는 자원 제약형 마이크로컨트롤러(NuttX RTOS)와 범용 운영체제 기반의 고성능 임무 컴퓨터(Companion Computer) 사이를 매끄럽게 결합한다.

2.1 퍼블리시-서브스크라이브(Publish-Subscribe) 네트워크 브릿지

uXRCE-DDS의 가장 두드러진 아키텍처적 차별성은 비동기적 퍼블리시-서브스크라이브(Asynchronous Publish-Subscribe) 매커니즘이다. 정보의 생성자(Publisher)와 소비자(Subscriber)가 상호 간의 물리적 주소를 인지하지 않고 데이터 토픽(Topic)을 매개로 분리(Decoupling)된다. 정보 흐름은 다대다(N:M) 비순환 그래프(Directed Acyclic Graph) 구조를 띠며, 이는 공간적 분리(Spatial Decoupling)와 시간적 분리(Temporal Decoupling)를 동시에 달성하게 해 준다.

2.2 uORB 토픽의 원형 보존 및 직접 매핑 체계

MAVLink 관제 구조에서는 uORB 메시지가 MAVLink 프로토콜 구조에 억지로 끼워 맞춰 변환되는 오버헤드가 발생하지만, uXRCE-DDS 아키텍처에서는 PX4 내부의 uORB 메시지 형태가 RTPS(Real-Time Publish Subscribe) 패킷으로 **투명성(Transparency)**을 유지한 채 원형 그대로 1:1 매핑된다.
CDR(Common Data Representation) 포맷을 사용한 이 직렬화(Serialization) 방식은 직렬화 과정에서의 정보 엔트로피 손실을 방지하고 코드 제너레이터(Code Generator)에 의한 컴파일 타임 최적화를 실현하여 직렬화 지연 시간(Serialization Latency)을 극도로 압축한다.

4. 제어 지연성 및 확장성(Scalability) 분석 결론

통신 채널에서 발생하는 제어 응답의 나이퀴스트-섀넌 제어 주기 최적화 및 지연 시간(Latency) 확보 관점에서 다음과 같은 결론을 도출할 수 있다. 관련하여 “Performance Evaluation of micro-ROS for Distributed Robotics (2020)” 논문 등의 선행 연구에서도 DDS 브릿징 모델이 분산 제어 환경에서 뛰어난 응답성을 제공함이 정량적으로 검증되어 있다.

  1. 지연 시간 확정성 보장(Determinism):
  • MAVLink는 단일 큐(Queue) 통신 병목의 확률이 높다. 반면 ROS2 DDS 환경에서는 QoS(Quality of Service) 정책을 활용하여 필수 제어 토픽에 대해 우선순위 할당이 가능하며, 이는 제어 레이턴시의 스파이크(Latency Spike)를 최소화한다.
  1. 정보 처리 대역폭(Bandwidth)의 상대적 효율성:
  • MAVLink 프레임 모델은 헤더 및 체크섬 구조가 매우 타이트하여 극도로 낮은 RF 원격 통신(예: 433MHz / 915MHz 텔레메트리 모듈) 환경에 최적화되어 있다. 반면 uXRCE-DDS 모델은 고속의 로컬 통신(이더넷, 고속 시리얼) 기반에서 대용량 데이터(비전, 스웜 정보)를 원형 손실 없이 처리할 때 오버헤드 비율이 현격하게 감소하는 정보 이론적 이득을 취한다.

PX4 펌웨어를 기반으로 정교한 Offboard 제어 알고리즘 및 자율 비행(Autonomous Flight) 소프트웨어를 설계할 때, 시스템 통합의 위치(원격 관제 vs 기체 내부 이기종 컴퓨터 구성)에 따라 위 아키텍처적 특성을 토대로 MAVLink 또는 ROS2 체계를 명확히 취사선택해야 한다.
