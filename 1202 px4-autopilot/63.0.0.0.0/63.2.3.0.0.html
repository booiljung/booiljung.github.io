63.2.3. 상향 링크(Uplink) 상태 피드백 토픽의 세부 논리 구조 및 상태 관측기(State Observer) 변수 분석

오프보드(Offboard) 기반의 폐루프 제어(Closed-loop Control) 환경에서 지상 관제 시스템(GCS) 또는 동반 컴퓨터(Companion Computer)가 정확한 전역 궤적 제어 입력을 산출하기 위해서는 비행 제어기(FC) 내부의 현재 동역학적 상태를 정밀하게 파악해야 한다. 무인항공기에서 비행 제어기로부터 관제 노드 측으로 전달되는 정보의 흐름을 상향 링크(Uplink) 또는 텔레메트리(Telemetry) 라우팅이라 칭한다. 본 절에서는 최신 PX4-Autopilot(v1.14 이상) 아키텍처 상향 링크를 구성하는 핵심 피드백 토픽들의 세부 논리 구조와 상태 관측기(State Observer) 기반 동작 원리를 제어 공학의 관점에서 서술한다.

1. 상향 링크 피드백 체계의 미들웨어 아키텍처

Ardupilot은 전통적으로 MAVLink의 주파수 대기열(SRx_ 파라미터 계열)을 통해 위치, 자세, 시스템 상태 메시지를 지정된 헤르츠(Hz)로 브로드캐스트하는 방식을 선호한다. 이와 대조적으로 PX4-Autopilot은 uORB 기반의 출판-구독(Publish-Subscribe) 모델을 ROS2 uXRCE-DDS 에이전트와 직접 결합시킴으로써, 상향 링크 피드백 토픽들을 오직 클라이언트 노드가 요구(Subscribe)할 때만 선택적으로 직렬화(Serialization)하여 대역폭(Bandwidth)을 비약적으로 보존하는 아키텍처를 취한다.

이는 900MHz 대역의 저토크 텔레메트리 라디오 환경이나 다중 에이전트(Swarm) 통신 하에서 네트워크 붕괴를 예방하는 구조적 이점으로 작용한다.

2. 상태 관측기(State Observer)와 핵심 피드백 변수 분석

실제 비행 중인 기체의 위치와 속도는 센서 노이즈가 없는 완벽한 형태로 측정될 수 없으므로, 모든 상향 링크 데이터는 EKF2(Extended Kalman Filter) 확장 칼만 필터 모듈이라는 상태 관측기(State Observer)를 통과한 ’추정(Estimation)’의 결과물이다.

동반 컴퓨터에서 수신하는 핵심 상향 링크 토픽은 제어기에 피드백 루프(Feedback Loop) 폐합을 위한 오차 산출용으로 직접 투입된다. 대표적인 토픽들은 다음과 같다.

2.1 VehicleLocalPosition 토픽 (위치 및 속도 관측치)

동역학 모델의 병진 운동(Translational Motion) 상태 벡터를 규정하는 체계이다.

  • 관측 변수:
  • x, y, z (NED 좌표계 기준 위치, 단정밀도 32비트 Float 변수)
  • vx, vy, vz (NED 좌표계 기준 선속도)
  • 제어 공학적 의미: 동반 컴퓨터의 경로 생성기(Trajectory Generator)는 목표 경로점(Waypoint)과 해당 토픽으로 수신된 기체의 현재 위치 변수(\mathbf{p}_{est}, \mathbf{v}_{est}) 간의 리아프노프(Lyapunov) 오차를 산출하여 다음 단계의 하향 링크 명령(TrajectorySetpoint)을 재생산한다.
  • 불확실성 지표: eph와 epv, evh, evv와 같은 오차 공분산(Error Covariance) 유도 필드를 포함하여, 관측치가 포함하고 있는 분산(Variance) 수준을 GCS 사용자에게 제공한다.

2.2 VehicleAttitude 토픽 (자세 관측치)

회전 동역학(Rotational Dynamics)의 현재 해(Solution)를 제공하는 토픽이다.

  • 관측 변수: q[4] (방위를 나타내는 쿼터니언(Quaternion) 벡터)
  • 설계 기조: 짐벌 록(Gimbal Lock) 방지를 위해 오일러 각(Euler Angle) 대신 쿼터니언을 표준 송신 체계로 차용하였다.
  • 비행 제어기 하위 루프(Rate Controller)에서 직접 소비되므로 상향 링크 빈도가 가장 높게(예: 50~100Hz) 요구되는 변수 그룹이기도 하다.

2.3 VehicleOdometry 토픽 (외부 시각 주행 기록계 상호교환)

단순 피드백을 넘어, VIO(Visual Inertial Odometry)를 구동하는 ROS2 동반 컴퓨터가 자신의 관측 결과를 비행 제어기로 쏴주고(Uplink-injection),FC의 EKF2가 이를 다시 융합하여 상향 링크(Uplink-feedback)로 돌려주는 양방향성을 띤 특수 토픽이다.
PX4의 설계상 기체-동반 컴퓨터 간의 단일 시간 축(Single Time Axis) 동기화가 이루어지도록 timestamp 헤더가 핵심 요소로 다루어진다.

3. 상향 링크 폴링 지연(Polling Latency)과 시스템 강건성(Robustness) 고려사항

네트워크 기반 제어 시스템(NCS: Networked Control System) 문헌(“Evaluation of ROS2 Communication for Micro Air Vehicles”, 2022) 등에서 지적되듯, 상향 링크 통신 채널 상의 가변 시간 지연(Time-varying Delay)은 모델 예측 제어(MPC)나 고이득 궤환(High-gain Feedback) 제어기의 폐루프 극점(Pole)을 우반평면(Right-half Plane)으로 진동시켜 비행 발산(Divergence)을 초래할 수 있다.

따라서 PX4-Autopilotv v1.14 연동 시, ROS2 수신 노드는 토픽 갱신 주파수를 모니터링해야 한다. 일정 시간계수 동안 VehicleLocalPosition의 갱신이 누락될 시, 제어 모델은 이전 속도에 의존한 선형 외삽법(Linear Extrapolation)으로 추정 관측치를 메우거나(Dead Reckoning), 즉각적으로 안전 상태(Failsafe) 플래그를 하향 링크로 내려보내는 방어적 조치를 이행해야 한다.

결과적으로 PX4 상향 링크 토픽은 단순한 모니터링 정보를 넘어, 분산된 프로세서들 간 폐루프 상태를 결정짓고 수학적 필터 공간을 연결하는 관측기 인터페이스 그 자체로서 매우 엄정한 분석이 요구된다.
