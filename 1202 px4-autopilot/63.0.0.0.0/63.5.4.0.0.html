63.5.4. [실습] 동적 이동 객체 추적을 위한 실시간 선속도 벡터(Linear Velocity Vector) 기반 다중 루프(Cascaded) 비례-적분-미분(PID) 제어기 튜닝 및 차분 방정식(Difference Equation) 구현

오프보드(Offboard) 환경에서 동적으로 이동하는 객체(예: 지상의 이동 차량, 다른 드론 등)를 정밀하게 추적하기 위해서는 단순히 정지된 경유지를 향해 비행하는 것을 넘어, 객체의 실시간 움직임을 반영한 동적인 속도 제어가 필수적이다. 이를 위해 ROS2 미들웨어를 통해 연속적인 **실시간 선속도 벡터(Linear Velocity Vector)**를 생성하고 전달해야 한다.

PX4-Autopilot은 위치와 속도를 처리하기 위해 다중 루프(Cascaded) PID 제어기 아키텍처를 채택하고 있다. 외측 루프(Outer Loop)는 위치 오차를 기반으로 목표 속도를 생성하고, 내측 루프(Inner Loop)는 속도 오차를 기반으로 목표 가속도 및 추력을 생성한다. 이러한 다단 제어 구조는 비선형성이 강한 멀티로터 시스템에서 추종 성능과 안정성을 극대화하기 위해 설계되었다.

이 절에서는 C++ 및 ROS2를 활용하여 영상 인식이나 외부 센서 시스템에서 획득한 타겟의 실시간 위치 오차 데이터를 받아, 이산화된 **차분 방정식(Difference Equation)**을 통해 커스텀 외측 통신 루프 속도 제어기를 구현하는 방법과 튜닝 실무를 알아본다.

1. PX4-Autopilot 다중 루프(Cascaded) PID 제어 아키텍처 이해

멀티로터의 동력학(Dynamics)은 위치, 속도, 자세, 각속도 간의 강한 결합(Coupling)을 지닌다. PX4는 이러한 복잡한 동특성을 제어하기 위해 직렬로 연결된 다단 제어루프(Cascaded Control Loops)를 사용한다.

graph TD
    A[Position Error: X, Y, Z] -->|Outer Loop: P Controller| B(Velocity Setpoint)
    B --> C[Velocity Error: Vx, Vy, Vz]
    C -->|Inner Loop: PID Controller| D(Acceleration / Thrust Setpoint)
    D --> E[Attitude Error: Roll, Pitch, Yaw]
    E -->|P Controller| F(Angular Rate Setpoint)
    F -->|PID Controller| G(Actuator / Motor PWM)

추적을 위한 Offboard 외부 루프(ROS2 단)에서 직접 TrajectorySetpoint의 속도 벡터(velocity) 필드를 지속적으로 주입하면, PX4의 자체 위치 제어기(P 제어)를 단락시키고 즉각적으로 PX4의 내부 속도 제어기(PID 제어)와 직결되는 효과를 얻는다. 객체를 추적할 때 외부 노드(ROS2)에서 P 또는 PI 가미형 위치 기반 속도 도출을 수행하면 지연 속도가 크게 단축된다.

2. 동적 추적을 위한 차분 방정식(Difference Equation) 기반 제어기 설계

연속 시간 시스템(Continuous-time System)의 수학적 모델인 미분 방정식은 컴퓨터 시스템이나 마이크로컨트롤러에서 다루기 위해 일정한 샘플링 주기로 이산화(Discretization)되어야 한다. 이 과정에서 이산 시간(Discrete-time) 모델인 차분 방정식(Difference Equation) 형태로 변환된다.

동적 이동 타겟에 대한 추적을 수행할 때, 가장 일반적인 형태의 비례-적분-미분 제어의 이산 시간 근사는 다음과 같다.

시간 k에서의 위치 오차를 e[k]라고 할 때, 제어 입력(선속도 벡터, \mathbf{v}[k])은 다음 수식으로 표현된다.

\mathbf{v}[k] = K_p \cdot e[k] + K_i \cdot \sum_{j=0}^{k} e[j] \Delta t + K_d \cdot \frac{e[k] - e[k-1]}{\Delta t}

  • \mathbf{v}[k]: 현재 주기의 명령 속도 (Linear Velocity Command)
  • K_p, K_i, K_d: 비례, 적분, 미분 게인(Gain)
  • \Delta t: 샘플링 주기 (ROS2 노드의 타이머 주기, 예: 0.02s = 50Hz)
  • e[k], e[k-1]: 현재 주기 및 이전 주기의 벡터 오차

2.1 시각적 유도 방식과의 융합

영상 기반 추적 알고리즘(예: YOLO v8 객체 인식 기반 픽셀 오차)의 출력이 픽셀 거리로 주어지면, VIO나 짐벌 측위학(Kinematics)을 거쳐 3차원 유클리드 공간 공간 벡터 오차로 변환된 뒤 위 차분 방정식의 입력 e[k]로 전달된다. 계산된 \mathbf{v}[k]는 ROS2 px4_msgs::msg::TrajectorySetpoint의 velocity 배열(크기 3)에 삽입되어 발행된다.

3. [실습] ROS2 기반 선속도 벡터 추출 제어기(C++) 템플릿 작성

다음 실습은 ROS2 노드 내부에서 이동 객체의 목표 위치와 현재 기체 위치의 오차를 기반으로, 자체적인 이산 PID 계산(차분 방정식 적용)을 통해 3차원 선속도 제어 인자를 50Hz 단위로 퍼블리시하는 노드이다. 코드는 NED 로컬 좌표계를 기준으로 동작하며, 노이즈 억제 및 안정성 개선을 위해 미분항에 저대역 통과 필터(Low-pass Filter)를 추가 적용하는 것이 실무에서 권장된다.

3.1 C++ Tracker Node 구현

#include <rclcpp/rclcpp.hpp>
#include <px4_msgs/msg/trajectory_setpoint.hpp>
#include <px4_msgs/msg/vehicle_local_position.hpp>
#include <Eigen/Dense>

using namespace std::chrono_literals;

class TargetTrackerNode : public rclcpp::Node {
public:
    TargetTrackerNode() : Node("target_tracker_node") {
        publisher_ = this->create_publisher<px4_msgs::msg::TrajectorySetpoint>(
            "/fmu/in/trajectory_setpoint", 10);
        
        position_subscription_ = this->create_subscription<px4_msgs::msg::VehicleLocalPosition>(
            "/fmu/out/vehicle_local_position", rclcpp::SensorDataQoS(),
            std::bind(&TargetTrackerNode::position_callback, this, std::placeholders::_1));

        timer_ = this->create_wall_timer(
            20ms, std::bind(&TargetTrackerNode::control_loop, this)); // 50 Hz 제어 주기

        // PID 상태 변수 초기화
        error_integral_ = Eigen::Vector3d::Zero();
        previous_error_ = Eigen::Vector3d::Zero();
    }

private:
    rclcpp::Publisher<px4_msgs::msg::TrajectorySetpoint>::SharedPtr publisher_;
    rclcpp::Subscription<px4_msgs::msg::VehicleLocalPosition>::SharedPtr position_subscription_;
    rclcpp::TimerBase::SharedPtr timer_;

    // 현재 기체 위치 (NED 좌표계)
    Eigen::Vector3d current_position_;
    // 가상의 동적 타겟 위치 (추후 비전 센서 토픽 등으로 업데이트 됨)
    Eigen::Vector3d target_position_ = Eigen::Vector3d(10.0, 5.0, -3.0); 

    // 제어기 게인 (튜닝 필요)
    const double kp_ = 1.2;
    const double ki_ = 0.05;
    const double kd_ = 0.1;
    const double dt_ = 0.02; // 20ms

    Eigen::Vector3d error_integral_;
    Eigen::Vector3d previous_error_;

    void position_callback(const px4_msgs::msg::VehicleLocalPosition::SharedPtr msg) {
        current_position_ << msg->x, msg->y, msg->z;
    }

    void control_loop() {
        // 1. 오차 벡터 계산: e[k] = Target - Current
        Eigen::Vector3d error = target_position_ - current_position_;

        // 2. 적분항 업데이트: 누적 오차 (바람 등 지속적인 외란 상쇄)
        // Anti-windup 로직 구현 필요 (본 예제에서는 생략)
        error_integral_ += error * dt_;

        // 3. 미분항 업데이트: 오차 변화율 (제동 및 초과 응답 억제)
        Eigen::Vector3d error_derivative = (error - previous_error_) / dt_;
        previous_error_ = error;

        // 4. 차분 방정식 기반 제어 입력 (선속도) 도출
        Eigen::Vector3d velocity_command = (kp_ * error) + (ki_ * error_integral_) + (kd_ * error_derivative);

        // 5. 최대 속도 제한 (Saturation) - 안전 비행을 위함
        double max_velocity = 5.0; // 5 m/s 제한
        if (velocity_command.norm() > max_velocity) {
            velocity_command = velocity_command.normalized() * max_velocity;
        }

        // 6. TrajectorySetpoint 메시지에 매핑 및 전송
        px4_msgs::msg::TrajectorySetpoint setpoint;
        setpoint.timestamp = this->get_clock()->now().nanoseconds() / 1000;
        
        // 위치는 추종 목적이므로 NaN으로 설정하고 오직 속도만 제어에 이용
        setpoint.position = {NAN, NAN, NAN};
        setpoint.velocity = {static_cast<float>(velocity_command.x()), 
                             static_cast<float>(velocity_command.y()), 
                             static_cast<float>(velocity_command.z())};
        setpoint.yaw = NAN; 

        publisher_->publish(setpoint);
    }
};

int main(int argc, char * argv[]) {
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<TargetTrackerNode>());
    rclcpp::shutdown();
    return 0;
}

3.2 실습 분석 및 유의 사항

  • 위의 코드에서는 TrajectorySetpoint 메시지의 position과 yaw 필드를 NAN으로 명시하여, PX4 내부 제어기가 위치 제어기 단계를 바이패스(Bypass)하고 즉시 속도 설정점을 받도록 유도하였다.
  • 안티 와인드업(Anti-windup) 체계: 이동 표적을 추적할 때는 오차가 급격하게 변할 수 있어 적분항(error_integral_) 누적이 포화 상태(Saturation)를 유발할 수 있다. 조건부 적분 컷오프(Clamping) 처리를 실무 단위에서 추가하도록 한다.
  • 위치 토픽 /fmu/out/vehicle_local_position 수신 QoS를 SensorDataQoS (Best Effort) 로 두어 UDP 통신의 빠른 처리를 유도하였다.

4. 제어기 튜닝 가이드라인 (Tuning Guidelines)

동적 이동체를 쫓는 무인 비행체의 외부 제어 게인(K_p, K_i, K_d)을 튜닝하는 것은 이론적 튜닝 기법(예: Ziegler-Nichols 방법)보다는 관측 기반의 경험적 튜닝(Heuristic Tuning) 기법이 많이 활용된다.

  1. 비례 게인 (K_p) 초기 튜닝
    먼저 K_i와 K_d를 모두 0으로 둔다. 타겟이 일정한 속도로 움직일 때 기체가 반응하는 민첩성을 테스트한다. 이동하는 상대 쪽으로 충분히 가깝게 따라가되, 멈출 때 목표 타겟 주위로 격하게 진동(Oscillation)하지 않도록 가장 높은 임계값을 찾아 0.6을 곱하여 초기 K_p 값으로 지정한다.
  2. 미분 게인 (K_d) 추가를 통한 댐핑(Damping)
    동적 타겟 추종에서는 타겟 위치가 수시로 바뀌므로 빠른 선형 돌진이 필요하지만, 그만큼 오버슈트(Overshoot) 위험성이 높다. K_d를 단계적으로 올려, 목적지 근방에 다다를 때 제동을 걸어주어 부드럽게 정지하도록 설계방향을 맞춘다. 미분항은 카메라 비전 인식 노이즈에 매우 큰 반응을 보이므로, 센서 융합 시스템에서 노이즈가 강하다면 K_d값을 무조건 줄이고 저대역 패스 필터를 전처리 과정에 결합해야 한다.
  3. 적분 게인 (K_i) 미세 추가
    추종 과정에서 강한 맞바람이 불거나, 객체의 등속 지속 이동으로 인해 일정한 유지 오차(Tracking Lag)가 발생할 때 K_i를 매우 작은 단위(예: 0.01)로 올린다. 다중 시스템에서는 보통 K_i는 외부 ROS2 통신보다는 PX4 내부 제어기(Inner Loop)의 몫에 맡기는 편이 더 안정적이라 외측 루프(Outer Loop)에서는 최소화하는 경향이 있다.

5. 요약

이번 장에서는 정지된 포인트-투-포인트 방식의 웨이포인트 이동이 아닌, 가변 상태의 동적 객체를 추적하기 위한 외측 차분 방정식 제어 아키텍처를 구현하였다. 선속도 벡터를 산출하여 TrajectorySetpoint를 통해 직접 주입할 경우 멀티로터의 반응성이 극대화되며, 다중 루프(Cascaded) 제어기의 속성 상 외곽에서는 위치-속도 P/PID를 구성하고 내부에서 PX4가 각운동학(Angular kinematics)과 전개 추력을 할당하는 매끄러운 역할 분배가 가능해진다.
