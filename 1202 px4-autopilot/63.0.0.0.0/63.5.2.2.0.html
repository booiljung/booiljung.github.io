63.5.2.2 좌표축 위상 반전 및 방향 코사인 행렬(Direction Cosine Matrix)을 이용한 3차원 유클리드 공간 벡터의 동기적 매핑 및 선형 변환(Linear Transformation) 구현

이전 절에서 논의한 SO(3) 변환 행렬을 바탕으로, 본 절에서는 수식적 이론을 넘어 실제 ROS2 기반의 Offboard 제어 노드(C++) 환경에서 3차원 유클리드 공간 벡터를 동기적(Synchronous)으로 변환하는 파이프라인 구현 기법을 살펴본다. 특히 고도 및 하강 제어 시 가장 잦은 오류를 유발하는 Z축의 위상 반전(Phase Inversion) 현상을 명확하게 제어하는 방향 코사인 행렬(Direction Cosine Matrix, DCM) 기반 선형 변환(Linear Transformation)의 소프트웨어적 접근법을 다룬다.

1. 방향 코사인 행렬(DCM)과 좌표축 위상 반전의 의미

방향 코사인 행렬(DCM)은 한 좌표계의 기저 벡터(Basis Vector)를 다른 좌표계의 기저 벡터로 투영(Projection)한 코사인 값들로 이루어진 3 \times 3 행렬이다. ROS2의 ENU 좌표계를 PX4의 NED 좌표계로 변환하는 DCM R_{ENU}^{NED}은 이전 절에서 다음과 같이 도출되었다.

R_{ENU}^{NED} = \begin{bmatrix} 0 & 1 & 0 \\ 1 & 0 & 0 \\ 0 & 0 & -1 \end{bmatrix}

이 행렬을 임의의 ENU 기반 위치 또는 속도 공간 벡터 \mathbf{v}_{ENU}에 곱하는 선형 변환 \mathbf{v}_{NED} = R_{ENU}^{NED} \mathbf{v}_{ENU} 연산을 수행하면 다음과 같은 현상이 발생한다.

  • 교환(Swapping): 행렬의 [0,1]과 [1,0] 성분으로 인해 X축과 Y축 데이터가 서로 교환된다.
  • 위상 반전(Phase Inversion): 행렬의 최하단 우측 [2,2] 성분인 -1로 인해 Z축 벡터 데이터의 부호가 강제로 역전된다. 즉, 상승(Up) 의도를 가진 양수 벡터가 하강(Down) 의도를 가지는 음수 벡터로 투영된다.

비행 제어 소프트웨어 설계 시 이 위상 반전을 망각할 경우, 에이전트가 이륙하지 않고 바닥으로 전력을 다해 곤두박질치는 치명적 제어 역전(Control Reversal) 오류가 발생하므로, 제어 루프 내에 명시적인 동기적 매핑(Synchronous Mapping) 함수를 두어야 한다.

2. 동기적 매핑(Synchronous Mapping) 파이프라인 설계

선형 변환은 제어기 내부의 타이머 콜백(Timer Callback) 스레드에서 생성되는 궤적 명령(예: 50Hz 제어 루프) 주기에 맞춰 **동기적(Synchronous)**으로, 그리고 지연 속도 변동(Jitter) 없이 즉각적으로 실행되어야 한다. 이를 위해 객체지향적 C++ 환경에서는 템플릿 또는 인라인(Inline) 함수를 활용한 오버헤드 없는 매핑 계층을 구성한다.

3. 선형 변환의 C++ 코드 구현 (Eigen 라이브러리 활용)

로보틱스 분야의 표준 선형대수 라이브러리인 선언적(Declarative) 템플릿 기반 Eigen3 라이브러리를 활용하면, 반복적인 행렬 곱을 높은 성능으로 처리할 수 있다.

아래는 ROS2 제어기 노드 내에서 ENU 형태의 3차원 유클리드 공간 벡터를 NED 형태의 px4_msgs::msg::TrajectorySetpoint 타겟에 쓰기 위해 선형 변환 행렬을 동기적으로 곱하는 인터페이스 구현 예시이다.

#include <Eigen/Dense>
#include <px4_msgs/msg/trajectory_setpoint.hpp>

// 1. 방향 코사인 행렬(DCM) 상수 초기화
const Eigen::Matrix3d R_ENU_TO_NED = (Eigen::Matrix3d() << 
    0, 1, 0,
    1, 0, 0,
    0, 0, -1).finished();

/**
 * @brief ENU 관성 좌표계 벡터를 NED 좌표계 벡터로 동기적 변환 (Linear Transformation)
 * @param v_enu 변환할 ENU 기준 3차원 공간 벡터 (위치 또는 속도)
 * @return Eigen::Vector3d 변환된 NED 기준 공간 벡터
 */
inline Eigen::Vector3d apply_dcm_mapping(const Eigen::Vector3d& v_enu) {
    // DCM 행렬 곱 연산 수행
    return R_ENU_TO_NED * v_enu;
}

// [ROS2 타이머 루프 내부]
void offboard_control_timer_callback() {
    // 목표 위치: East 5.0m, North 2.0m, Up 10.0m
    Eigen::Vector3d target_position_enu(5.0, 2.0, 10.0);
    
    // 선형 변환 적용 (Z축 10.0이 -10.0으로 위상 반전 매핑됨)
    Eigen::Vector3d target_position_ned = apply_dcm_mapping(target_position_enu);
    
    // TrajectorySetpoint 메시지 패킹 (동기화 블록)
    px4_msgs::msg::TrajectorySetpoint setpoint_msg;
    setpoint_msg.position[0] = target_position_ned.x(); // 2.0
    setpoint_msg.position[1] = target_position_ned.y(); // 5.0
    setpoint_msg.position[2] = target_position_ned.z(); // -10.0
    
    // 이외 속도/가속도 벡터도 동일한 apply_dcm_mapping 함수 사용 가능
    // ...
    // setpoint_publisher_->publish(setpoint_msg);
}

위와 같이 Eigen::Matrix3d를 결합하여 설계된 오퍼레이터를 사용하면, 복잡한 비행 궤적(Spline, Polynomial Trajectory 등)에서 산출되는 무수히 많은 지점(Waypoints) 데이터를 제어 스레드 루프 안에서 지연 없이 동기적으로 매핑할 수 있다.

4. 요약

공간 데이터의 무결성을 위해, 오프보드 제어 애플리케이션에서는 반드시 방향 코사인 행렬을 기반으로 한 엄밀한 선형 변환 계층을 적용해야 한다. 특히 Z축 위상 반전에 관련된 부분은 비행체의 안전과 직결되므로, 동기적 루프 내에서 하드코딩된 인덱스 변환보다 Eigen과 같은 검증된 선형 변환 대수 라이브러리를 통해 명시적으로 변환하는 것이 예기치 못한 비행 제어 역전 현상을 예방하는 모범 사례(Best Practice)에 해당한다.
