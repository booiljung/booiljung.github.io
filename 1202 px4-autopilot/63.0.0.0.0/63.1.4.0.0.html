63.1.4. Offboard 모드 진입 및 유지를 위한 시스템 상태의 필수 전제 조건

PX4-Autopilot 아키텍처에서 **Offboard 모드(Offboard Mode)**는 기체 내부의 자율 비행 로직을 우회하고, 외부의 컴패니언 컴퓨터(Companion Computer, 예: Raspberry Pi, NVIDIA Jetson)나 지상 관제 시스템(GCS)에게 제어 권한(Control Authority)을 독점적으로 위임하는 고급 비행 모드이다. 외부 시스템의 제어 알고리즘(예: ROS2 기반 비전 회피 알고리즘, 심층 강화 학습 제어기 등)이 직접 생성한 제어 설정점(Setpoint)을 실시간으로 기체에 주입하는 형태이므로, 비행 안전성(Flight Safety) 확보를 위한 다중의 **필수 전제 조건(Prerequisites)**과 유지 로직(Failsafe)이 PX4 커맨더(Commander) 모듈 내부에 매우 엄격하게 하드코딩되어 있다.

본 절에서는 특정 펌웨어(PX4 v1.14 이상)를 기준으로 Offboard 모드에 진입(Transition)하고 이를 안정적으로 유지하기 위해 시스템적으로 충족되어야 하는 상태 머신(State Machine)의 조건들을 심층적으로 살펴본다.

1. Offboard 모드 진입 전제 조건 (Pre-conditions for Mode Switch)

임무 컴퓨터가 Offboard 모드 전환을 요청(명령)하기 전에 PX4의 상태 머신은 다음과 같은 필수적인 전제 조건들이 모두 만족되어 있는지 검사한다. 이 중 단 하나라도 누락되면 커맨더는 전환 요청을 거부(Deny)하고 NACK(Negative Acknowledgment)를 반환한다.

1.1 하드웨어 및 Estimator의 건강 상태 (Health & EKF2 Status)

외부 시스템이 ’위치(Position)’나 ‘속도(Velocity)’ 기반의 제어 설정점을 주입하려는 경우, 기체 내부의 상태 추정기(Estimator, 주로 EKF2)가 유효한 로컬 위치 또는 글로벌 위치를 추정하고 있어야 한다.

  • 국소 위치 유효성(Local Position Validity): 비전 측위(VIO) 또는 GPS를 통해 융합된 상태 추정 토픽 VehicleLocalPosition 메시지의 xy, z 추정치(Estimate)가 유효(Valid)해야 한다.
  • 센서 캘리브레이션이 완료되지 않았거나 센서(IMU, Mag 등)에 하드웨어적 결함 플래그가 발생한 경우 모드 진입은 원천적으로 차단된다.

1.2 선제적 제어 설정점(Setpoint) 스트리밍 개시 (데드맨 스위치)

Offboard 모드 진입 시 가장 빈번하게 발생하는 실패 원인은 제어 데이터의 스트리밍 순서이다. PX4는 외부 제어기가 활성화되어 정상 작동 중인지 확인하기 위해, 모드 전환 명령을 내리기 전에 이미 제어 설정점(Setpoint)이 수신되고 있어야 함을 시스템적으로 강제한다.

  • 임무 컴퓨터는 오프보드 전환 플래그(VehicleCommand::VEHICLE_CMD_DO_SET_MODE)를 전송하기 직전 최소 수 밀리초에서 수백 밀리초 전부터 지속적으로 타깃 토픽(예: TrajectorySetpoint 또는 MAVLink SET_POSITION_TARGET_LOCAL_NED)을 기체로 발행(Publishing)해야 한다.

2. Offboard 모드 유지 조건 및 락-스텝(Lock-step) 타임아웃 메커니즘

모드 진입에 성공하여 기체가 외부 명령을 수용하더라도, 연결성이나 임무 컴퓨터 운영체제의 결함에 의한 통신 두절은 추락으로 직결될 수 있다. 이를 방지하기 위해 PX4는 소프트웨어적인 데드맨 스위치(Dead-man’s Switch) 타임아웃 로직을 가동한다.

2.1 요구되는 최소 표본화(스트리밍) 주파수

  • PX4 v1.14 최신 펌웨어 기준, Offboard 설정점은 최소 2 Hz (0.5초 주기) 이상의 빈도로 지속 수신되어야 한다. 안정적인 궤적 추종을 위해서는 실무적으로 20 Hz ~ 100 Hz 사이의 연속적인 스트리밍이 요구된다.
  • 만약 설정점 메시지가 정해진 타임아웃 임계값(기본 0.5초 파라미터 COM_OF_LOSS_T) 이상 도착하지 않으면, 커맨더는 오프보드 제어 링크가 유실된 것으로 간주한다.

2.2 연결 유실 시 페일세이프(Failsafe) 이행

설정점 타임아웃이 발생하면 PX4는 즉각적으로 비상 복구 절차에 돌입한다. 페일세이프 기본 설정(COM_OBL_ACT 파라미터)에 따라 다음과 같이 상태가 자동 전환된다.

  1. Hold (Loiter) 모드 전환: 위치 추정이 유효한 경우, 현재 위치에서 호버링(Hovering)하며 외부 제어기의 명령이 재수신되기를 대기한다.
  2. Land 모드 전환: 일정 시간 동안 통신이 복구되지 않거나 배터리가 부족한 경우 즉시 착륙을 시도한다.
  3. Return (RTL) 모드 전환: 설정에 따라 안전한 홈 위치나 이륙 지점으로 복귀한다.

3. Offboard 상태 전환 및 페일세이프 상태 머신 (Mermaid 도표)

임무 컴퓨터와 통신하는 PX4 내부 커맨더(Commander)의 Offboard 모드 승인 및 추방(Eviction)에 관한 상태 머신(State Machine) 흐름은 다음과 같이 도식화할 수 있다.

stateDiagram-v2
    [*] --> Initialization
    Initialization --> Ready: Boot & Sensor Calibration OK
    
    state "Manual / Auto(Mission) Mode" as Standby
    Ready --> Standby

    Standby --> CheckingPreconditions: Request Offboard Mode
    
    state CheckingPreconditions {
        Direction TB
        check1: Estimator Local Position Valid?
        check2: Setpoint Stream Active (> 2Hz)?
        check1 --> check2
    }
    
    CheckingPreconditions --> OffboardMode: All Valid (Accept)
    CheckingPreconditions --> Standby: Invalid (Reject/NACK)
    
    state "Offboard Mode (Active)" as OffboardMode
    OffboardMode --> FailsafeTrigger: Setpoint Timeout (> 0.5s)
    
    state "Failsafe Handling" as FailsafeTrigger {
        state "Hold (Loiter)" as Hold
        state "Land" as Land
        state "Return to Launch" as RTL
    }
    
    FailsafeTrigger --> OffboardMode: Stream Restored (If applicable)

4. 모드 제어를 위한 통신 미들웨어별 구현 인터페이스 요약

외부 시스템 개발자는 환경에 맞춰 다음의 수단을 통해 모드 제어 명령을 하달해야 한다.

  • MAVLink (QGroundControl / MAVSDK):
  • 제어 시스템은 COMMAND_LONG 메시지를 통해 MAV_CMD_DO_SET_MODE 커맨드를 송신하며, Base Mode 플래그와 Custom Mode 에 PX4_CUSTOM_MAIN_MODE_OFFBOARD를 할당한다. 통신망 지연으로 인한 NACK 응답에 대비해 재시도(Retry) 로직이 필요하다.
  • ROS2 uXRCE-DDS:
  • VehicleCommand uORB 토픽으로 매핑되는 메시지를 발행(Publishing)한다. command 필드에 VehicleCommand::VEHICLE_CMD_DO_SET_MODE 값을 입력하고 파라미터 매핑을 맞춘다.

Offboard 제어기 응용 프로그램은 위와 같은 엄격한 시스템 제약 사항과 타임아웃 메커니즘을 소프트웨어 구조(예: 별도의 퍼블리싱 스레드 구성, 독립된 타이머 루프 적용)에 철저하게 반영해야만 자율 비행 시스템의 내결함성(Fault Tolerance)을 달성할 수 있다.
