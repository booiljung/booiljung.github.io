63.6.4. [실습] 기압 고도 유지 제어 폐루프와 결합된 직접적 자세 제어를 활용한 비선형 공간 곡예 비행(Acrobatic Maneuver) 한계 주기 궤도(Limit Cycle Orbit) 패턴 구현

1. 개요 및 수학적 배경

다차원 공간에서 무인기(UAV)의 곡예 비행이나 지속적인 정찰 임무를 수행할 때, 특정 궤적을 무한히 반복하는 한계 주기 궤도(Limit Cycle Orbit, LCO) 비행 패턴이 자주 요구된다. 오프보드(Offboard) 모드에서 이러한 궤도를 생성하는 가장 직관적인 방법은 위치나 속도를 직접 제어하는 것이지만, 더욱 역동적이고 기민한 기동(Acrobatic Maneuver)을 구현하기 위해서는 직접적인 자세 제어(Direct Attitude Control) 기법을 활용하는 것이 유리하다.

본 실습에서는 동반 컴퓨터(Companion Computer)의 ROS 2 노드에서 수행되는 기압 고도(Barometric Altitude) 유지 제어 루프와 PX4-Autopilot의 내부 자세 제어기(Inner Attitude Controller)를 결합하여 비선형적인 8자 비행(Figure-8) 모양의 한계 주기 궤도를 구현한다. (PX4 v1.14.0 이상, ROS 2 Humble/Iron 환경 기준)

1.1 직접 자세 제어와 고도 제어의 결합 원리

단일 오프보드 노드 내에서 횡방향(Lateral) 및 종방향(Longitudinal) 이동은 오일러 각(Euler Angles: Roll, Pitch)을 주기적으로 변화시켜 생성하고, 이와 동시에 수직방향(Z축)은 기압계(Barometer) 또는 지역 포지셔닝(Local Position) 피드백을 통해 계산된 수직 추력(Vertical Thrust)으로 보상한다.

1.2 비선형 한계 주기 궤도 (LCO) 수학적 모델링

대표적인 8자 궤적인 리사주 도형(Lissajous Figure) 형태를 자세 제어 명령 위상 공간(Phase Space)에서 유도하기 위해 다음과 같은 단순화된 사인파 조합 모델을 사용할 수 있다.

  • \phi_{cmd}(t) = \Phi_{max} \sin(\omega t) (Roll 커맨드)
  • \theta_{cmd}(t) = \Theta_{max} \cos(2\omega t) (Pitch 커맨드)

여기서 \Phi_{max}와 \Theta_{max}는 각각 롤 및 피치 축의 최대 기울기 각도(Amplitude)이며, \omega는 궤적의 순환 주파수(Angular Frequency)이다. 요(Yaw) 각도 \psi는 비행 경로의 접선(Tangent) 방향을 바라보게 하거나 일정한 상수로 유지할 수 있다.

2. 시스템 제어 루프 아키텍처 (System Control Loop Architecture)

ROS 2 노드는 고도 제어를 위한 외곽 폐루프(Outer Closed-loop)로 동작하며, PX4 시스템은 자세 제어를 위한 내부 폐루프(Inner Closed-loop)로 동작한다.

위 구조에서 고도 오차(Altitude Error) e_z = Z_{ref} - Z_{current}에 비례, 적분, 미분(PID) 이득을 곱하여 추력 명령(Thrust Command)을 산출하며, 이를 무차원 추력(0.0 ~ 1.0) 단위로 정규화하여 사용한다.

3. 오프보드 제어 노드(C++) 구현 실습

기압 고도 기반 위치 정보를 읽어들이기 위해 VehicleLocalPosition을 구독(Subscribe)하고, 제어 명령을 위해 VehicleAttitudeSetpoint와 OffboardControlMode를 발행(Publish)하는 ROS 2 C++ 노드 예제를 구현한다.

3.1 노드 구성 및 의존성

C++ POSIX 스레드나 타이머 인터페이스(Timer Interface)를 이용해 100Hz(10ms) 주기의 연속적인 제어 명령을 퍼블리시한다. 직접 자세 제어를 활성화하기 위해 OffboardControlMode 메시지 내의 attitude 필드를 true로 설정한다.

3.2 C++ 실습 코드 핵심 구현

#include <rclcpp/rclcpp.hpp>
#include <px4_msgs/msg/offboard_control_mode.hpp>
#include <px4_msgs/msg/vehicle_attitude_setpoint.hpp>
#include <px4_msgs/msg/vehicle_local_position.hpp>
#include <px4_msgs/msg/vehicle_command.hpp>

#include <cmath>
#include <chrono>

using namespace std::chrono_literals;

class AcrobaticLcoNode : public rclcpp::Node
{
public:
    AcrobaticLcoNode() : Node("acrobatic_lco_node"),
                         t_start_(this->now()),
                         base_thrust_(0.5f), // 호버링(Hovering) 기본 추력 기준
                         target_altitude_(-10.0f) // NED 좌표계 기준 고도 10m
    {
        offboard_control_mode_publisher_ = 
            this->create_publisher<px4_msgs::msg::OffboardControlMode>("/fmu/in/offboard_control_mode", 10);
        attitude_setpoint_publisher_ = 
            this->create_publisher<px4_msgs::msg::VehicleAttitudeSetpoint>("/fmu/in/vehicle_attitude_setpoint", 10);
        vehicle_command_publisher_ = 
            this->create_publisher<px4_msgs::msg::VehicleCommand>("/fmu/in/vehicle_command", 10);

        local_position_subscriber_ = this->create_subscription<px4_msgs::msg::VehicleLocalPosition>(
            "/fmu/out/vehicle_local_position", rclcpp::SensorDataQoS(),
            std::bind(&AcrobaticLcoNode::position_callback, this, std::placeholders::_1));

        // 100Hz 루프
        timer_ = this->create_wall_timer(10ms, std::bind(&AcrobaticLcoNode::timer_callback, this));
    }

private:
    void position_callback(const px4_msgs::msg::VehicleLocalPosition::SharedPtr msg) {
        current_altitude_ = msg->z; // NED 좌표계: 위로 올라갈수록 음(Negative)의 방향
        
        // P 제어기 기반의 단순화된 고도 추력 보상 알고리즘
        float alt_error = target_altitude_ - current_altitude_;
        float kp_z = 0.05f; // P-Gain
        
        // 자세 기울어짐(Tilt)에 의한 수직 성분 감소 보상 수식을 추가할 수 있다.
        current_thrust_ = base_thrust_ + (kp_z * alt_error);
        
        // 한계치 클리핑(Clipping)
        if(current_thrust_ > 0.9f) current_thrust_ = 0.9f;
        if(current_thrust_ < 0.1f) current_thrust_ = 0.1f;
    }

    void timer_callback() {
        publish_offboard_control_mode();
        publish_attitude_setpoint();
    }

    void publish_offboard_control_mode() const {
        px4_msgs::msg::OffboardControlMode msg{};
        msg.timestamp = this->get_clock()->now().nanoseconds() / 1000ULL;
        msg.position = false;
        msg.velocity = false;
        msg.acceleration = false;
        msg.attitude = true; // 자세 기반 제어 활성화
        msg.body_rate = false;
        offboard_control_mode_publisher_->publish(msg);
    }

    void publish_attitude_setpoint() {
        auto now = this->now();
        double t = (now - t_start_).seconds();

        // 비선형 동적 LCO 모델 (Figure-8 패턴)
        // Roll과 Pitch 각도를 시간에 따라 변화 (라디안 단위)
        float max_angle = 30.0f * (M_PI / 180.0f); // 30도 최대 한계 각도
        float omega = 0.5f; // 주파수
        
        float roll  = max_angle * std::sin(omega * t);
        float pitch = max_angle * std::cos(2.0f * omega * t);
        float yaw   = 0.0f; // 고정된 기수상태 유지

        px4_msgs::msg::VehicleAttitudeSetpoint msg{};
        msg.timestamp = this->get_clock()->now().nanoseconds() / 1000ULL;

        // 오일러 각(Euler Angles)에서 쿼터니언(Quaternion) 변환 수행
        // 회전 순서는 Z-Y-X (Yaw-Pitch-Roll)
        double cr = std::cos(roll * 0.5);
        double sr = std::sin(roll * 0.5);
        double cp = std::cos(pitch * 0.5);
        double sp = std::sin(pitch * 0.5);
        double cy = std::cos(yaw * 0.5);
        double sy = std::sin(yaw * 0.5);

        msg.q_d[0] = cr * cp * cy + sr * sp * sy; // w
        msg.q_d[1] = sr * cp * cy - cr * sp * sy; // x
        msg.q_d[2] = cr * sp * cy + sr * cp * sy; // y
        msg.q_d[3] = cr * cp * sy - sr * sp * cy; // z

        // Z축 수직 추력 매핑. 
        // NED 좌표계에서 Z축은 아랫방향이므로 음수를 취급해야 하나 추력(Thrust) 명령은 항상 양수 (0~1)를 갖는다.
        msg.thrust_body[0] = 0.0;
        msg.thrust_body[1] = 0.0;
        msg.thrust_body[2] = -current_thrust_; // 추력 벡터의 Z 성분 (기체 상단으로 향함)

        attitude_setpoint_publisher_->publish(msg);
    }

    rclcpp::Time t_start_;
    float base_thrust_;
    float target_altitude_;
    float current_altitude_{0.0f};
    float current_thrust_{0.5f};

    rclcpp::Publisher<px4_msgs::msg::OffboardControlMode>::SharedPtr offboard_control_mode_publisher_;
    rclcpp::Publisher<px4_msgs::msg::VehicleAttitudeSetpoint>::SharedPtr attitude_setpoint_publisher_;
    rclcpp::Publisher<px4_msgs::msg::VehicleCommand>::SharedPtr vehicle_command_publisher_;
    rclcpp::Subscription<px4_msgs::msg::VehicleLocalPosition>::SharedPtr local_position_subscriber_;
    rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char **argv)
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<AcrobaticLcoNode>());
    rclcpp::shutdown();
    return 0;
}

4. QGroundControl 및 SITL 환경에서의 검증

게이지보(Gazebo) 기반 SITL(Software In The Loop) 터미널을 실행하고(예: make px4_sitl gz_x500), uXRCE-DDS 에이전트를 구동한 상태에서 상기 ROS 2 노드를 실행하라. 해당 노드는 독립된 프로세스로서 동반 컴퓨터의 부담을 시뮬레이션한다.

4.1 QGroundControl을 이용한 모니터링

  1. 지상 관제 검증 (GCS Monitoring): QGroundControl v4.3 이상의 GCS 인터페이스에 연결하여 Analyze Tools -> MAVLink Inspector 항목에서 ATTITUDE_TARGET 메시지와 LOCAL_POSITION_NED 메시지의 변화 추이를 모니터링하라.
  2. 트래킹 플롯 분석 (Tracking Plot Analysis): Analyze Tools 내의 실시간 플로터를 활용하여 Z 고도 오차가 수 십 센티미터(Centimeters) 이내로 수렴하는 지 확인하면서, Roll과 Pitch 제어 입력이 온전한 리사주 곡선(Lissajous Curve)을 형성하는 것을 검증한다. 목표 자세(Desired Attitude)와 추정 자세(Estimated Attitude) 사이의 위상 지연(Phase Delay)을 분석함으로써 내부 제어기(Inner Loop)의 대역폭 한계(Bandwidth Limitation)를 유추할 수 있다.
  3. 자세와 추력의 이중성 확인: 기체가 가파른 자세를 취할 때는 양력(Lift)의 수직 성분이 감소하므로(수직 성분 비율은 \cos(\phi)\cos(\theta)에 비례), 상단 ROS 2 코드의 PID 제어기가 즉각적으로 추력을 비례적으로 증가시켜 고도 강하(Altitude Drop) 현상을 억제하는 과정을 데이터를 통해 정량적으로 확인할 수 있다. 고급 구현에서는 1 / (\cos(\phi)\cos(\theta)) 와 같은 피드포워드(Feed-forward) 보상 항을 더하여 반응 시간을 단축시킬 수 있다.
