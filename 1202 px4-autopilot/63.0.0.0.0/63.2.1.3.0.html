63.2.1.3. 직렬화 과정에서의 바이트 정렬(Endianness) 불일치 해소 및 패딩(Padding) 메모리 단편화 최소화 기법

현대의 무인항공기 시스템(UAS) 제어 아키텍처는 단일 마이크로컨트롤러(MCU)에서 연산을 독점하던 과거를 벗어나, 강력한 연산 능력을 지닌 컴패니언 컴퓨터(Companion Computer)와 비행 제어기(Flight Controller)가 실시간으로 통신하는 오프보드(Offboard) 형태로 진화하였다. 오픈소스 생태계를 선도하는 PX4-Autopilot(v1.14.0 기준)과 ROS2(vHumble 기준) 간의 통신은 마이크로 에이전트(uXRCE-DDS) 브릿지를 통해 이루어지며, 이때 서로 다른 이종 하드웨어 간 데이터 교환을 위해 직렬화(Serialization) 및 역직렬화(Deserialization)가 필수적으로 개입된다.

본 절에서는 데이터 직렬화 과정에서 발생하는 시스템 간 바이트 정렬(Endianness) 불일치 문제를 해결하는 메커니즘과, C/C++ 구조체 메모리 정렬에서 유발되는 패딩(Padding)으로 인한 단편화(Fragmentation)를 최소화하는 아키텍처 수준의 기법들에 대해 상세히 분석한다.

1. 이종 시스템 간 바이트 정렬(Endianness) 불일치 문제

데이터를 메모리에 연속적으로 저장할 때, 최상위 바이트(MSB)를 먼저 저장하는 방식을 빅 엔디안(Big-Endian)이라 하고, 최하위 바이트(LSB)를 먼저 저장하는 방식을 리틀 엔디안(Little-Endian)이라 한다. 일반적으로 Pixhawk 시리즈에 내장된 ARM Cortex-M 마이크로컨트롤러는 리틀 엔디안 아키텍처를 기본으로 채택하고 있으나, 특정 외부 센서(Sensor) 장비나 특수한 네트워크 프로토콜 칩셋은 빅 엔디안을 사용할 수 있다.

1.1 CDR(Common Data Representation) 명세의 엔디안 플래그 활용

단일 데이터 구조(예: MAVLink)만을 사용하여 내부 패킷을 오프셋(Offset)으로 관리하는 고전적인 Ardupilot 프레임워크와 달리, PX4-Autopilot과 ROS2의 미들웨어 환경은 OMG(Object Management Group)에서 제정한 CDR(Common Data Representation) 표준을 준수한다.

CDR 직렬화 기법은 패킷 헤더(Header)의 두 번째 바이트 위치에 현재 페이로드(Payload)가 어떠한 엔디안 방식으로 직렬화되었는지 알려주는 플래그(Flag)를 삽입한다.
수신측인 에이전트 브릿지는 패킷을 수신한 후 헤더 플래그를 읽어 자기가 구동 중인 시스템의 네이티브(Native) 엔디안과 비교한다. 만일 구조적 불일치가 확인되면, 런타임 역직렬화 메커니즘이 강제적으로 바이트 스왑(Byte Swapping) 연산을 수행하여 원본 수치를 정상적으로 복원해 낸다.

2. 메모리 패딩(Padding) 발생 원리와 단편화 최소화 기법

펌웨어 소스 코드를 빌드할 때, C/C++ 컴파일러는 중앙 처리 장치(CPU)가 메모리에 배열된 변수 값들에 가장 빠르게 접근할 수 있도록 일정한 바이트 경계(Byte Boundary)에 맞춰 변수를 배치한다. 이를 정렬(Alignment)이라고 부르며, 크기가 맞지 않은 변수들 사이에는 빈 바이트 레이어인 패딩(Padding) 공간이 강제적으로 삽입된다.

2.1 uORB 및 ROS2 IDL(Interface Definition Language) 메시지의 구조 정렬 원칙

패딩 바이트가 메시지 내부에 무분별하게 삽입될 경우, 실제로는 데이터가 없으면서 대역폭만 낭비하는 오버헤드(Overhead)가 심각하게 누적된다. 또한 동적 메모리의 힙(Heap) 영역에 객체를 할당/해제하는 과정에서 공간들이 조각나는 메모리 단편화(Memory Fragmentation) 현상이 발생하여, 임베디드 장치에 치명적인 커널 아웃오브메모리(OOM)가 야기될 수 있다.

PX4 개발팀은 이를 원천적으로 방지하기 위해 uORB 메시지 구조체(.msg 파일)를 설계할 때 **‘모든 변수를 바이트 크기 기준 내림차순(Descending)으로 정렬한다’**는 엄격한 소스 코드 컨벤션을 강제한다.

자료형 (Data Type)자료형 크기 (Bytes)메모리 정렬 오프셋 할당(Alignment) 조건
uint64_t timestamp8N \pmod 8 = 0 조건을 충족하는 시작 주소
float32[3] position12 (4바이트 배열)N \pmod 4 = 0 조건을 충족하는 시작 주소
uint16_t error_count2N \pmod 2 = 0 조건을 충족하는 시작 주소

참고로, 위 표의 정렬 수식 논리에서 모듈러 기호의 절대값 평가 등에 사용되는 기호는 \vert x \vert의 형태로 표시할 수 있다.

내림차순 정렬 규칙을 준수하면, 크기가 홀수이거나 배수가 일치하지 않아 발생하던 구조체 내 패딩 블록의 발생률이 수학적으로 최소화된다. 컴팩트(Compact)해진 페이로드 덕분에 QGroundControl(v4.3.0) 지상 관제 시스템으로 전송되는 텔레메트리 대역폭 또한 획기적으로 개선된다.
