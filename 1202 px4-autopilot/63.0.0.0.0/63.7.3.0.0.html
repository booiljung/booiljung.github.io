63.7.3. [실습] 수직 이착륙기(VTOL) 및 고정익 공기역학 모델의 사용자 정의 틸트 로터 천이(Transition) 위상각 제어를 위한 맥라린 급수(Maclaurin Series) 근사화 동기화 제어 구현

이 절에서는 수직 이착륙기(VTOL, Vertical Take-Off and Landing)의 핵심 기동인 틸트 로터(Tiltrotor) 천이(Transition) 과정에서 발생하는 공기역학적 비선형성(Non-linearity)을 제어하기 위한 사용자 정의(Custom) 위상각(Phase Angle) 제어 기술을 다룬다. 특히, 로터의 기울기 각도에 따른 추력 분배 방정식의 연산 부하를 시스템 단에서 최소화하기 위해 수학적 기법인 맥라린 급수(Maclaurin Series) 근사화 알고리즘을 도입하고, ROS 2 Offboard 모드와 PX4-Autopilot 제어 할당기(Control Allocator)를 연동하여 서보 모터 위상각을 직접 제어하는 개루프(Open-loop) 동기화 실습을 진행한다. (본 실습은 PX4-Autopilot v1.14.x 및 QGroundControl v4.3.x, ROS 2 Humble 버전 환경을 기준으로 서술한다.)

1. 틸트 로터 비행체의 천이 동역학 및 제어적 한계의 이해

VTOL 비행체는 이착륙 시 호버링(Hovering)을 위한 수직 양력(Lift)을 생성하고, 순항 비행(Cruise Flight) 모드에서는 전진 프롭(Propeller)을 통한 수평 추력을 발생시킨다. 틸트 로터는 이 두 비행 위상 사이를 잇는 과도기(Transient State)인 ’천이 과정’을 거치게 되며, 이 과정에서 틸트 서보(Servo) 모터는 추력 벡터(Thrust Vector)의 방향 공간을 0도(완전 수직)에서 90도(완전 수평)로 연속적으로 변환한다.

천이 상태에서 기체는 멀티로터 동역학과 고정익 공기역학 모델의 영향을 동시에 받게 되므로, 급격한 받음각(Angle of Attack, AoA)의 변화나 외부 풍향의 교란(Disturbance)에 매우 취약하다. 이를 해결하기 위해 기본 제공되는 PX4-Autopilot 믹서(Mixer) 또는 제어 할당 행렬(Control Allocation Matrix)은 표준화된 비행 프로파일(Flight Profile)에 기반하여 틸팅(Tilting) 동작을 수행한다. 그러나 새로운 형상의 사용자 정의 공기역학 모델, 혹은 세차 운동(Gyroscopic Precession)이 강력하게 발생하는 거대 질량의 커스텀(Custom) 로터를 장착한 시스템의 경우 기본 천이 속도 및 궤적만으로는 시스템 불안정성이 발생할 수 있다. 따라서 개발자는 ActuatorServos uORB 메시지를 활용하여 특정 위상각 제어 알고리즘을 믹서를 거치지 않고 직접 인가하는 방식을 설계해야 한다.

2. 맥라린 급수(Maclaurin Series) 기반 비선형 삼각함수 근사화 이론

틸트 서보 모터의 위상각 \theta (라디안) 변화에 따른 추력기(Thruster)의 Z축(수직) 및 X축(수평) 벡터는 다음과 같은 삼각함수 기저로 이루어진다.

F_z = F_{total} \cdot \cos(\theta)
F_x = F_{total} \cdot \sin(\theta)

일반적으로 고성능 모델 예측 제어 로직(Model Predictive Control, MPC)을 실시간으로 컴패니언 컴퓨터(Companion Computer)나 비행 제어기(FC, Flight Controller)의 마이크로컨트롤러 유닛(MCU)에서 실행할 때, 제어 주기마다 호출되는 고정밀 비선형 삼각함수는 부동 소수점 처리(Floating-point Processing) 연산에 상당한 자원을 소모한다. 즉, 제어 주파수 대역폭(Bandwidth)을 500Hz ~ 1kHz 수준으로 끌어올리기 위해서는 계산 효율(Computational Efficiency)을 극대화해야 한다.

이러한 제한을 극복하기 위해 테일러 급수(Taylor Series) 전개를 기준점 x = 0 (여기서는 \theta = 0, 즉 다중 로터 모드) 주변에서 수행한 매클로린 급수(Maclaurin Series)를 활용하여 함수를 다항식으로 선형 및 부분 비선형 근사화를 진행할 수 있다.

사인 함수(\sin(\theta))의 맥라린 급수 전개:
\sin(\theta) = \theta - \frac{\theta^3}{3!} + \frac{\theta^5}{5!} - \dots

코사인 함수(\cos(\theta))의 맥라린 급수 전개:
\cos(\theta) = 1 - \frac{\theta^2}{2!} + \frac{\theta^4}{4!} - \dots

천이 과정에서 연산 부하를 획기적으로 줄이기 위해 상위 고차항을 절삭(Truncated)하고 3차 이하의 급수항만을 취하여 근사 방정식을 유도한다.

F_x \approx F_{total} \cdot \left( \theta - \frac{\theta^3}{6} \right)
F_z \approx F_{total} \cdot \left( 1 - \frac{\theta^2}{2} \right)

위 수식은 제어 프로그래밍 상에서 곱셈과 덧셈만으로 \theta에 대한 최적화를 가능케 함으로써, 서보 제어 동기화 시 연산 지연 없는 실시간 위상각 결정을 제공한다.

3. ROS 2 및 PX4-Autopilot의 서보 동기화 통신 아키텍처

근사화된 위상각을 토대로, ROS 2의 Offboard 노드는 틸트 로터를 전진 혹은 수직 상태로 변경하기 위한 서보 명령 값을 지속적으로 발행(Publishing)한다. 이 명령 열(Sequence)은 제어 주기(예: 100Hz)마다 ActuatorServos 메시지를 통해 PX4-Autopilot의 Micro XRCE-DDS 에이전트 측으로 전달된다.

아래 다이어그램은 ROS 2 노드에서 맥라린 급수를 계산하고 PX4로 직접 서보 위상각을 전송하는 전반적인 아키텍처를 나타낸다.

4. [실습] C++ 기반 맥라린 급수 동기화 제어 노드 작성

다음은 ROS 2 Humble 환경을 이용하여 VehicleStatus 메시지로 천이 모드 전개를 모니터링하면서, 천이 조건 돌입 시 Maclaurin 근사화 방정식의 해를 구해 ActuatorServos로 제어값을 인가하는 핵심 실습 코드이다.

#include <rclcpp/rclcpp.hpp>
#include <px4_msgs/msg/actuator_servos.hpp>
#include <px4_msgs/msg/vehicle_status.hpp>
#include <cmath>

using namespace std::chrono_literals;

class TiltrotorTransitionController : public rclcpp::Node
{
public:
    TiltrotorTransitionController() : Node("tiltrotor_transition_controller"), current_phase_angle_(0.0)
    {
        // ActuatorServos 퍼블리셔 선언: 틸트 서보를 직접 구동하기 위한 제어 토픽
        servo_publisher_ = this->create_publisher<px4_msgs::msg::ActuatorServos>(
            "/fmu/in/actuator_servos", 10);

        // VehicleStatus 서브스크라이버 선언: 천이 진행률 및 비행 모드 상태 확인
        status_subscriber_ = this->create_subscription<px4_msgs::msg::VehicleStatus>(
            "/fmu/out/vehicle_status", 10,
            std::bind(&TiltrotorTransitionController::status_callback, this, std::placeholders::_1));

        // 50Hz (20ms) 주기의 제어 루프 타이머
        timer_ = this->create_wall_timer(
            20ms, std::bind(&TiltrotorTransitionController::control_loop, this));
            
        RCLCPP_INFO(this->get_logger(), "Tiltrotor Transition Node Initialized.");
    }

private:
    rclcpp::Publisher<px4_msgs::msg::ActuatorServos>::SharedPtr servo_publisher_;
    rclcpp::Subscription<px4_msgs::msg::VehicleStatus>::SharedPtr status_subscriber_;
    rclcpp::TimerBase::SharedPtr timer_;

    bool is_transitioning_ = false;
    double current_phase_angle_;   // 라디안 단위의 현재 틸트 각도
    const double TARGET_PHASE_ANGLE = M_PI / 2.0; // 90도 (고정익 수평 순항 모드)

    // 맥라린 급수 (Maclaurin Series) 근사화 함수: 연산 지연 최소화를 위한 고차항 절삭
    double calculate_maclaurin_sin(double theta)
    {
        // sin(θ) ≈ θ - (θ^3)/6
        return theta - (std::pow(theta, 3) / 6.0);
    }

    void status_callback(const px4_msgs::msg::VehicleStatus::SharedPtr msg)
    {
        // PX4의 자체 Transition State를 평가 (예: 천이 시작 시 트리거 활성화)
        if (msg->vehicle_type == px4_msgs::msg::VehicleStatus::VEHICLE_TYPE_ROTARY_WING && msg->in_transition_mode) {
            is_transitioning_ = true;
        }
    }

    void control_loop()
    {
        if (!is_transitioning_) {
            return;
        }

        // 간단한 1차 지연계 궤적 생성을 통한 위상각 증가 (천이 가속 프로파일링)
        if (current_phase_angle_ < TARGET_PHASE_ANGLE) {
            current_phase_angle_ += 0.01; // 루프당 약 0.57도 증가
        }

        // 맥라린 급수 함수를 호출하여 투영된 수평 추력 계수 근사값 도출 (추가적인 역학 보상에 사용 가능)
        double thrust_coeff_x = calculate_maclaurin_sin(current_phase_angle_);
        
        // 서보 모터 위상각을 PX4 Actuator 메시지 체계에 매핑 (통상적으로 -1.0 ~ 1.0)
        // 여기서는 단방향 틸팅 (0.0 ~ 1.0)으로 가정
        px4_msgs::msg::ActuatorServos servo_msg;
        servo_msg.timestamp = this->get_clock()->now().nanoseconds() / 1000ULL;
        
        // Tilt Servo가 0 채널 (servo_msg.control[0])에 할당되어 있다고 가정
        float mapped_actuation = static_cast<float>(current_phase_angle_ / TARGET_PHASE_ANGLE);
        
        // 위상 및 근사 모델에 기반하여 서보 제어값 인가 (추력 계수 보상을 추가 결합 가능)
        servo_msg.control[0] = mapped_actuation; 
        
        // 나머지 서보 채널은 기본값(NaN) 유지 (PX4 내 할당기 정책에 위임)
        for(int i=1; i<8; i++) {
            servo_msg.control[i] = NAN;
        }

        servo_publisher_->publish(servo_msg);

        // 결과 로깅
        RCLCPP_DEBUG(this->get_logger(), "Publishing Servo Phase | Phase: %.2f rad | Control: %.2f | Approx Tx Coefficient: %.3f", 
                     current_phase_angle_, mapped_actuation, thrust_coeff_x);
    }
};

int main(int argc, char **argv)
{
    rclcpp::init(argc, argv);
    rclcpp::spin(std::make_shared<TiltrotorTransitionController>());
    rclcpp::shutdown();
    return 0;
}

제시된 코드는 맥라린 급수를 통해 T_x의 추력 비율 변화 계수를 고속으로 산출하며(thrust_coeff_x), 이를 토대로 사용자 정의된 천이 궤적(current_phase_angle_의 증분)을 ActuatorServos의 제어 값으로 매핑(Mapping)한다. 이러한 설계는 PX4 시스템 내부의 믹서 동역학을 완전히 우회하고, ROS 2의 제어 노드가 타겟 틸트 위상각을 절대 권한으로 주입하는 개루프(Open-loop) 제어로 작동하게 한다.

5. ArduPilot과의 튜닝 및 제어 차별성 고찰

VTOL 기체 제어에 있어서 ArduPilot은 Q_TILT_MASK, Q_TILT_RATE_UP, Q_TILT_RATE_DN 파라미터 등을 통해 QuadPlane의 틸트 서보 동작률(Rate)을 선형적으로 제어하는 풍부하고 완성도 높은 편의성을 지원한다. 또한 Lua 스트립트를 통해 천이 과정을 부분적으로 유도할 수 있다.

그러나 특정한 비공식 비행 역학(Non-standard Flight Dynamics) 논문에 부합하는 제어나, 고속 실시간 삼각함수 근사화, 혹은 딥러닝 기반의 천이 예측 강화학습 궤적이 필요할 경우, 정해진 매개변수에 의존하는 ArduPilot의 설계는 수학적 이식 측면에서 유연성이 부족할 수 있다. 반면, PX4-Autopilot의 ROS 2 uORB 브릿지(XRCE-DDS) 기반 Control Allocation 추상화를 이용할 경우, C++ 레벨에서 서보의 위상각과 각속도의 관계를 재구성하는 “사용자 정의 천이 동기화 제어기“를 물리적인 하드웨어 추상화 계층(Hardware Abstraction Layer) 밖으로 완벽하게 분리하여 구현할 수 있다는 점에서 강력한 이점을 제공한다.

6. 결론

본 절에서는 틸트 로터 VTOL 공기역학 모델링 시 발생하는 각운동학적 연산 복잡도를 완화하기 위한 맥라린 급수의 활용 원리를 소개하였다. 이를 통해 사용자 정의된 ROS 2 기반 서보 모터 직접 제어가 가능하며, PX4-Autopilot 아키텍처가 지향하는 제어기와 액추에이터의 논리적 결합 해제(Decoupling) 모델이 얼마나 실무적인 연구 확장성을 가지고 있는지 실습을 통해 증명하였다. 이러한 고속 수학 연산 근사 기법은 향후 임베디드(Embedded) 시스템 최적화를 위한 핵심 설계 지침으로 작용하게 된다.
