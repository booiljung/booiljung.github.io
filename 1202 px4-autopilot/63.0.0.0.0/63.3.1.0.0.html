63.3.1. 디지털 이산 시간(Discrete-time) 제어 루프의 생명주기 및 설정점 전송 주기화 알고리즘

현대 무인항공기 탑재 컴퓨터(Companion Computer)가 수행하는 자율 비행 알고리즘은 본질적으로 연속 시간(Continuous-time)의 물리적 우주를 디지털 마이크로프로세서의 이산 시간(Discrete-time) 도메인으로 샘플링하여 흉내 내는 과정이다. PX4-Autopilot(v1.14 이상)의 오프보드(Offboard) 모드 제어 아키텍처에서 가장 핵심적인 제약 사항은 이 이산화된 제어 루프가 비행 제어기(FC)의 내부 스케줄러가 납득할 수 있는 ’생명주기(Lifecycle)’와 ’정주기성(Periodicity)’을 엄격히 준수해야 한다는 점이다. 본 절에서는 설정점(Setpoint) 토픽의 주기화 알고리즘이 ROS2 제어 노드의 생명주기에 어떻게 통합되는지 제어 공학의 샘플링 이론(Sampling Theory)을 바탕으로 구명한다.

1. 이산 시간 제어 시스템에서의 생명주기(Lifecycle) 모델링

Ardupilot은 MAVLink를 통한 제어 입력을 폴링(Polling) 방식으로 처리하며, 입력 주기의 미세한 흔들림(Jitter)에 비교적 관대하다. 그러나 PX4의 오프보드 제어 체계는 유한 상태 기계(FSM) 안전망과 맞물려 있어, 제어 노드의 생명주기를 ‘초기화(Initialization)’, ‘예열(Warm-up)’, ‘활성(Active Control)’, ’종료(Teardown)’의 4단계로 엄격히 구분하여 관리할 것을 강제한다.

1.1 스트리밍 예열(Warm-up) 주기

가장 빈번하게 발생하는 오프보드 진입 실패 원인은 제어 토픽(예: TrajectorySetpoint) 발행(Publish)과 모드 전환 명령(VehicleCommand_DO_SET_MODE) 발행 체인의 순서 역전 현상이다.

이산 시간 제어 관점에서 컨트롤러가 닫힌 루프(Closed-loop)를 형성하기 위해서는 t=0 시점에 이미 과거 데이터에 의존하지 않는 유효한 타겟(Valid Target) 세트가 메모리에 존재해야 한다.
따라서 오프보드 제어 노드는 런타임 진입 직후, 모드 전환 명령을 내리기 최소 0.5초에서 1.0초 전부터 TrajectorySetpoint를 퍼블리싱하는 예열 주기를 가져야만 커맨더(Commander)의 진입 심사를 통과할 수 있다.

2. 설정점 전송 주기화(Periodicity) 알고리즘과 샘플링 간격 제어

2.1 주기율표(Rate Table)와 월-클럭(Wall-clock) 동기화

ROS2 환경에서 제어 노드를 작성할 때, rclcpp::WallTimer 또는 rclpy.create_timer를 사용하여 제어 루프를 구성하는 것이 일반적이다. 이 타이머 기반 콜백(Timer Callback) 알고리즘은 내부 OS 스케줄러의 타임 슬라이스(Time Slice)에 의존하므로, 순수한 이산 제어 수식 u[k] = -K x[k] 에서 가정한 이상적인 델타 시간 T_s와 실제 경과 시간 \Delta t 간에 위상 잡음(Phase Noise)을 파생시킨다.

PX4 오프보드 제어권 유지를 위한 최소 요구 주파수는 2\text{Hz}(500\text{ms} 타임아웃)이나, 50\sim100\text{Hz}(T_s = 0.02\sim0.01\text{sec}) 주기로 TrajectorySetpoint와 OffboardControlMode를 병렬 퍼블리싱하는 것이 동역학 제어의 나이퀴스트-섀넌 샘플링 정리(Nyquist–Shannon Sampling Theorem)를 안정적으로 충족하는 실무적 권장안이다.

2.2 클럭 슬립(Clock Slip) 보상 알고리즘

리눅스(Linux)와 같은 범용 시분할 운영체제(Time-sharing OS)에 얹혀 구동되는 ROS2 노드는 커널 스레딩 부하로 인해 간헐적으로 타이머 주기를 건너뛰는 클럭 슬립(Clock Slip) 현상을 겪는다.

동반 컴퓨터의 오프보드 제어 노드는 타겟 주기 T_s를 보장하기 위해, 이전 제어 루프의 타임스탬프 t_{k-1}과 현재 타임스탬프 t_k의 차이를 실시간으로 계측하여 궤적 생성기의 보간(Interpolation) 입력값으로 삽입하는 적응형(Adaptive) 주기화 알고리즘을 구비해야 한다. 이는 단순한 sleep(Ts) 함수의 무분별한 호출을 배제하고, 절대 시간 기반의 누적 오차 보정 기법을 적용함으로써 구현되며, 기체의 궤적이 부자연스럽게 점프하는 외란(Disturbance)을 원천 차단하는 가장 핵심적인 소프트웨어 아키텍처 패턴이다.
