37.3.2.1. 상호 배제적(Mutually Exclusive) 제어 입력의 상태 종속적(State-dependent) 마스킹(Masking) 처리

지상 관제 시스템(GCS) 또는 조종기(Radio Control)의 조작계는 다수의 스위치와 조이스틱(Joystick)으로 구성되며, 조작자의 실수로 인해 논리적으로 동시에 성립될 수 없는 상호 배제적(Mutually Exclusive) 명령이 오토파일럿으로 유입될 가능성이 항상 존재한다. PX4-Autopilot은 이러한 모순점(Contradiction)을 해결하기 위해 유한 상태 기계(FSM, Finite State Machine)의 현재 상태(State)에 종속적으로 기각 또는 무시 정책을 적용하는 상태 종속적 마스킹(State-dependent Masking) 아키텍처를 도입하고 있다.

1. 상호 배제적 제어 입력의 본질적 위험성

통신 채널(MAVLink 또는 RC SBUS)을 통해 오토파일럿으로 수신되는 명령 중에는, 물리적 혹은 제어 논리적으로 양립할 수 없는 커맨드 시퀀스(Command Sequence)가 존재한다.

대표적인 상호 배제적 이벤트의 예시는 다음과 같다.

  • Arm vs Disarm: 시동(Arm) 스위치와 시동 해제(Disarm) 채널이 물리적 배선 단락 또는 MAVLink 메시지 큐의 비정상적 지연(Lag) 교차로 인해 동시에 인가되는 경우.
  • Mission vs RTL(Return To Launch): 웨이포인트(Waypoint)를 추종하는 자동 임무(Mission)를 지시하는 동시에 복귀(RTL) 명령이 하달된 경우.
  • Acro vs Position Hold: 기압계 및 GPS 센서를 융합하여 위치를 잡는 완전히 안정적인 Hold 모드 토글 스위치와, 모든 자세 제어 센서를 무시하고 각속도(Rate)만 추종하는 완전 수동 Acro 모드 토글 스위치가 다중 스위치 설정 오류로 동시에 ON 상태로 매핑된 경우.

이러한 모순이 발생할 때, 코어 제어 루프가 우선순위를 명확히 결단(Arbitration)하지 못하면 기체는 제어 모드가 초당 수십 번씩 스위칭 현상(Chattering)을 일으켜 추락에 이르게 된다.

2. 상태 종속적(State-dependent) 마스킹(Masking) 아키텍처

PX4 커맨더(Commander) 내부의 상태 평가자(State Evaluator)는 입력 신호망의 “마스킹(Masking) 처리“를 통해, 현재 기체의 유한 상태 노드 결정을 방어한다. 이는 입력 포트 자체를 논리적으로 차폐(Filter-out)시키는 기법이다.

2.1 마운팅 우선순위(Priority Rules) 기반 트리거 마스킹

상호 배제적인 명령이 동시 송출된 상황에서 PX4는 정적인 하드코드 트리(Hard-coded Tree) 우선순위를 갖는다.

  • Failsafe 계열 제어 (최상위): RTL, Land, Failsafe
  • Auto 계열 제어 (차상위): Mission, Hold, Takeoff
  • Manual 계열 제어 (최하위): Altitude, Stabilized, Manual, Acro

조종자가 Mission 스위치와 RTL 스위치를 동시에 올렸을 경우, Commander는 하위 우선순위인 Mission 스위치에서 들어오는 이산 계단 신호(Discrete step signal)를 마스킹 처리(무시)하고 즉각 RTL 모드의 전환 조건으로 병합한다.

2.2 FSM 맥락 기반(Context-aware) 동적 마스킹

오로지 우선순위 트리에만 의존하는 것은 한계가 있다. 특정 상태에서는 상위 계열 명령이라 할지라도 기각되어야 하는 ‘상태 종속(State-dependent)’ 특징이 부여되어야 한다.

  • 착륙 상태(Landed State)에서의 Kill Switch 마스킹: 기체가 지면에 닿아있는 상태(Landed == True)라면, 조종자의 Kill Switch 입력 또는 Disarm 명령의 우선순위는 극대화된다. 하지만 반대로, 기체가 공중(In-air == True)이라면, 설정된 Circuit Breaker 파라미터(예: CBRK_FLIGHTTERM)에 의해 조종자의 Kill Switch 명령이 강제로 마스킹(차폐) 당할 수 있다. (실수로 공중에서 모터를 끄는 행위를 방지)
  • 물론 파라미터를 통해 “공중이라도 무조건 끈다“로 설정할 수 있으나, 기본값(Default)은 이산 사건의 현재 상태 노드(In-air)를 신뢰하고 조종자의 치명적 오입력을 필터링하는 데 초점을 둔다.

3. Ardupilot의 제어 스위칭 로직과의 패러다임 차이

  • Ardupilot (Flight Mode 1~6 PWM 믹싱): Ardupilot의 전통적인 RC 제어 방식은 하나의 RC 채널(일반적으로 채널 5)을 6개의 PWM 밴드(Band)로 쪼개어 비행 모드를 할당한다. 이는 구조적으로 하드웨어 라인에서 2개의 모드가 동시에 눌리는 상호 배제적 오입력 사태를 원천 차단(SBUS 상에서 1개의 PWM 값만 존재하므로)하는 아키텍처적 장점이 있다.
  • PX4 (Multi-Channel Boolean Mapping): 반면 PX4는 설계 철학상 다채널 버튼 기반 매핑(예: Joypad의 A, B, X, Y 버튼 각각에 모드 부여)을 적극 수용한다. 이는 GCS나 ROS 2 에이전트로부터 비동기 MAVLink 메시지가 난입하기 쉬움을 시사한다. 따라서 PX4는 MAVLink의 백엔드 단에서 vehicle_command 구조체 내의 이산 상태 조합을 Commander의 방어막 메커니즘인 ‘우선순위 스케줄러 및 상태 종속 마스킹’ 알고리즘으로 엄밀하게 걸러내도록 고도의 소프트웨어 설계가 집중되어 있다.

4. 제어 공학적 결론

상호 배제적이고 모순된 제어 입력이 주어졌을 때, 시스템이 패닉(Panic) 상태에 빠지지 않고 정형적 우선순위와 현재 기계 상태의 맥락(Context)을 파악하여 하위 논리를 의도적으로 거부(Masking)하는 것은 고도로 발전된 로보틱스 관제 미들웨어의 특징이다. 이러한 동적 입력 차폐 로직은 결함 주입 테스트(Fault Injection Test)나 조종 미숙으로 유발되는 극단적인 멀티-오더 결함을 방어하는 핵심 자산이다.
