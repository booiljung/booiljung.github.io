37.1.1 무선 텔레메트리 채널의 확률론적 특성과 섀넌-하틀리 정리(Shannon-Hartley Theorem) 기반 대역폭 한계 분석

현대의 무인항공기(UAV) 지상 관제 시스템(GCS)은 유선망이 아닌 무선 텔레메트리(Wireless Telemetry) 송수신기를 통해 외부와 연결된다. 대표적으로 PX4-Autopilot은 SiK Radio(433MHz 또는 915MHz 대역)나 Wi-Fi(2.4GHz/5GHz), 간혹 위성 통신(SATCOM) 모듈을 MAVLink 프로토콜의 전달 매체로 활용한다. 이러한 무선 채널은 근본적으로 불안정하며 확률론적 성질을 가지므로, 관제 소프트웨어 개발자는 통신 대역폭의 한계와 그 기저에 있는 정보 이론(Information Theory)을 명확히 이해해야 한다. 본 절에서는 섀넌-하틀리 정리(Shannon-Hartley Theorem)를 바탕으로 무선 텔레메트리의 한계를 정량적으로 분석하고, 이를 극복하기 위한 PX4의 통신 아키텍처를 살펴본다.

1. 무선 텔레메트리 채널의 확률론적 특성과 열화 요인

무선 채널을 통한 로버스트(Robust)한 관제를 달성하기 위해서는 먼저 공기 공간(Air Interface)을 통해 전파되는 신호가 어떠한 불확실성을 겪는지 파악해야 한다. 무선 통신 환경에서의 신호 열화(Signal Degradation)는 결정론적(Deterministic) 감쇠와 확률론적(Probabilistic) 잡음의 복합적인 결과물이다.

1.1 다중 경로 페이딩(Multipath Fading) 및 도플러 변이(Doppler Shift)

기체가 비행하는 환경(특히 도심지나 산악 지형)에서는 GCS의 안테나에서 방사된 전파가 건물이나 지표면에 반사, 굴절, 산란되어 수신기에 도달한다. 여러 경로를 거친 전파들은 도달 시간(Delay Spread)과 위상(Phase)이 달라 서로 간섭(Interference)을 일으키며, 이를 다중 경로 페이딩(Multipath Fading)이라고 한다. 또한 비행체 특유의 빠른 이동 속도로 인해 발생하는 도플러 변이(Doppler Shift)는 반송파(Carrier) 주파수를 편이시켜 수신 측의 복조(Demodulation) 실패 확률을 높인다.

1.2 신호 대 잡음비(SNR)의 가변성과 비트 오류율(BER)

위와 같은 환경적 요인으로 인하여 무선 텔레메트리 수신 모듈이 감지하는 신호의 세기는 확률 변수(Random Variable)로 취급된다. 레일리 페이딩(Rayleigh Fading) 또는 라이시안 페이딩(Rician Fading) 모델을 사용하여 이러한 신호 대 잡음비(Signal-to-Noise Ratio, S/N)의 변화를 근사할 수 있으며, 궁극적으로 이는 패킷 데이터의 비트 오류율(Bit Error Rate, BER)을 크게 변동시키는 원인이 된다. GCS 제어 로직은 언제든 BER이 임계치를 초과하여 통신 단절(Data Link Loss) 상태가 될 수 있음을 가정하고 설계되어야 한다.

2. 섀넌-하틀리 정리를 통한 대역폭(Bandwidth) 한계 분석

통신 채널의 한계를 설명하는 가장 근본적인 물리/수학적 이론은 클로드 섀넌(Claude Shannon)이 정립한 채널 용량(Channel Capacity) 이론이다. PX4 관제에서 텔레메트리 채널의 처리량 상한선을 파악할 때 섀넌-하틀리 정리가 필수적으로 인용된다.

2.1 섀넌-하틀리 정리의 정의

가우시안 백색 잡음(AWGN, Additive White Gaussian Noise)이 존재하는 통신 채널에서, 오류 없이 전송할 수 있는 최대 정보 전송률(Maximum Data Rate)인 채널 용량 C(bps)는 다음 수식으로 정의된다.

C = B \cdot \log_{2}\left(1 + \frac{S}{N}\right)

여기서 각 변수의 의미는 다음과 같다.

  • C: 채널 용량 (Channel Capacity, 단위를 초당 비트 수 bps로 표현)
  • B: 채널의 대역폭 (Bandwidth, Hz)
  • S: 평균 수신 신호 전력 (Average Received Signal Power, Watt)
  • N: 평균 잡음 전력 (Average Noise Power, Watt)

2.2 PX4-Autopilot 텔레메트리 환경에의 적용 추론

일반적으로 PX4를 구동하는 기체의 주 통신 수단인 915MHz SiK 텔레메트리는 매우 좁은 주파수 대역폭(Narrowband, B가 극히 작음)을 사용하며, 송신 출력(Transmit Power) 역시 통신법 규제로 인하여 100mW(20dBm) 수준으로 제한된다.
거리가 멀어질수록 역제곱 법칙(Inverse-square Law)에 따라 수신 신호 전력 S는 급격히 감소하므로, \frac{S}{N} 비율이 낮아진다. 정리 공식에 따르면 B가 고정되어 있고 S/N이 0에 수렴할수록 로그 항이 작아지므로 채널 용량 C는 기하급수적으로 하락하게 됨을 알 수 있다.
이로 인해 통상적인 SiK Radio의 실효 전송률은 57,600bps (약 57kbps) 이하 수준으로 제한되며, 이는 수십 메가비트를 상회하는 Wi-Fi 대역통신과 대비되는 치명적인 제약(Bottleneck) 요소이다.

3. 대역폭 한계를 극복하기 위한 PX4의 미들웨어 아키텍처

섀넌-하틀리 정리에 의해 증명된 좁은 무선 대역폭의 물리적 한계를 극복하기 위해, PX4 및 QGroundControl은 고도의 트래픽 관리(Traffic Management)와 직렬화(Serialization) 최적화 기술을 통신 스택에 적용하고 있다.

3.1 MAVLink 데이터 직렬화 및 스트림 압축 전략

XML 기반의 무거운 데이터 형식이 아닌 커스텀 바이너리 패킷 포맷으로 헤더 오버헤드를 줄인 MAVLink v2 프로토콜은 패킷 압축 및 널 바이트(Null-byte) 페이로드 생략(Zero-trimming) 기능을 제공한다. 이는 앞서 산출된 용량 C 내에 최대한 많은 관제 의미론(Semantics)을 구겨 넣는(Packing) 응용 계층의 대응책이다.

3.2 메시지 스트림 퍼블리싱(Stream Publishing)의 우선순위 제어

제한된 초당 전송량 하에서 GCS-UAV 통신이 정체되는 것을 막기 위해, PX4 펌웨어의 mavlink_messages.cpp 핸들러는 메시지들의 주기를 동적으로 조절한다.
필수 안전 진단 정보(예: HEARTBEAT, SYS_STATUS, ATTITUDE)는 주파수를 높게 할당하고, 상대적으로 덜 중요한 정보(VFR_HUD 등)는 QGC에서 요청한 스트림 레이트(Stream Rate)를 기체의 하위 MAVLink 제어기(MAVLink Receiver/Transmitter 노드)가 동적으로 삭감(Throttling)하여 전송한다.

4. 결론: 확률적 무선 환경과 QGC의 대응 논리

결론적으로, GCS(특히 QGroundControl) 소프트웨어를 설계할 때 무선 통신 링크는 ’항상 빠르고 유실이 없는 파이프라인’이라 가정해서는 안 된다. 섀넌-하틀리 정리가 증명하는 바와 같이 텔레메트리 데이터 링크는 거리에 따라 용량이 요동치는(Fluctuating) 불확실성의 공간이며, 패킷 손실률(BER) 상승은 필연적이다. 따라서 GCS의 제어 시스템은 통신 용량의 부족과 드롭 아웃(Drop-out) 상황에서도 데이터 연결 상실(Data Link Loss) 예외를 우아하게 처리(Graceful Degradation)하고 파라미터 재동기화를 실시하는 다중 강건화 회로(Robust Mechanism)를 내장해야만 한다.
