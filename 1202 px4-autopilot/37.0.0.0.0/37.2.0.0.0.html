37.2 PX4 오토파일럿의 방향성 비순환 그래프(DAG) 기반 유한 상태 기계(FSM) 토폴로지

무인기 비행 제어 시스템의 코어 로직은 복잡한 환경 변수와 사용자 명령을 안전하게 처리하기 위해 고도로 정형화된 유한 상태 기계(Finite State Machine, FSM)로 모델링된다. PX4-Autopilot은 상태 간의 전이(State Transition)가 결코 무한 루프나 교착 상태(Deadlock)에 빠지지 않도록 유도하기 위해 방향성 비순환 그래프(Directed Acyclic Graph, DAG) 토폴로지의 특성을 차용하여 커맨더(Commander) FSM을 설계하였다. 본 절에서는 PX4 상태 머신의 구조적 특징을 분석하고, 이를 통해 안전성(Safety)과 결정론성(Determinism)을 획득하는 메커니즘을 상세히 다룬다.

1. 비행 상태 제어에서의 안전 토폴로지(Topology) 요구사항

항공 비행 시스템에서는 단일 오프셋 오류(Offset Error)나 상태 전이 조건의 모순이 곧바로 기체의 추락이나 유령 제어(Ghost Control)로 이어질 수 있다. 따라서 상태의 흐름은 일방향적이고 명확한 위계(Hierarchy)를 지녀야 한다.

1.1 순환성(Cyclicity) 배제의 원칙

순환 그래프(Cyclic Graph) 구조의 상태 기계는 특정 외란이나 모순된 명령 입력 아래에서 S_1 \rightarrow S_2 \rightarrow S_3 \rightarrow S_1과 같이 목적 없는 무한 상태 천이(Infinite State Chattering)를 반복할 위험이 있다. 이러한 상태 채터링은 CPU 스레드를 점유할 뿐만 아니라 액추에이터에 상충되는 PWM 신호를 고주파로 전달하여 모터와 ESC 파괴를 유발할 수 있다.

1.2 DAG 기반 아키텍처의 채택

이를 방지하기 위해 PX4는 상태의 격상(Escalation) 단계—예를 들어 Init \rightarrow Pre-flight \rightarrow Armed \rightarrow Takeoff \rightarrow Mission—에 엄격한 방향성(Directedness)을 부여한다. 역방향으로의 전이(예: Takeoff에서 공중 Pre-flight로의 회귀)는 물리적으로 불가능하거나, 오직 비상 종료(Failsafe/Disarm)라는 명시적이고 제한된 엣지(Edge)를 통해서만 허용된다. 이러한 비순환적(Acyclic) 구조는 시스템의 도달 가능성(Reachability)을 수학적으로 완벽히 검증 가능하게 만든다.

2. PX4-Autopilot 커맨더(Commander) FSM 분석

PX4의 상태 머신 로직은 주로 src/modules/commander/Commander.cpp 내부의 State Machine 스레드에 의해 관리된다. 이 모듈은 uORB를 통해 수집된 센서 데이터, 배터리 상태, MAVLink 명령 등을 종합하여 DAG 기반의 다음 상태를 결정한다.

2.1 주 상태(Main State)와 내부 내비게이션 상태(Nav State)의 분리

PX4의 상태 공간은 크게 시스템 무장 상태(Arming State)와 내비게이션 상태(Navigation State, 비행 모드)의 2차원 직교 공간으로 분리되어 계층적으로 운영된다.

  • Arming State: INIT, STANDBY, ARMED, SHUTDOWN 등 물리적 모터의 활성화 여부를 결정하는 최상위 노드.
  • Nav State: MANUAL, ALTCTL, POSCTL, AUTO_MISSION, AUTO_RTL 등 커맨더가 하위 제어기(Position/Attitude Controller)로 전달하는 궤적 생성의 논리 모드.

2.2 상태 전이의 토폴로지적 특징

  • 비역진성(Irreversibility): 이륙(Takeoff) 상태에서 지상 대기(Standby) 상태로 돌아가기 위해서는 반드시 착륙(Landed) 상태를 거쳐 모터가 정지(Disarmed)되는 경로를 통과해야만 한다. 그래프 이론 관점에서 이는 상태 전이 행렬상에 지름길(Short-cut)이 존재하지 않는 강한 비순환성을 의미한다.
  • 단일 Sink 노드 구조: 비상 상황(Data Link Loss, Low Battery, Engine Failure 등)이 발생했을 때, 그래프 상의 모든 비행 상태 노드는 Failsafe 혹은 RTL(Return to Launch) 노드 방향으로 직결되는 단방향 엣지를 보유한다. 이는 시스템 설계상 예외 처리가 항상 종착점(Sink Node)으로 수렴(Convergence)함을 보증한다.

3. PX4와 Ardupilot의 상태 머신 토폴로지 비교

두 오픈소스 비행 제어 소프트웨어는 상태 천이를 다루는 기본 철학에서 뚜렷한 차이를 보인다.

  • Ardupilot: 실용주의적 관점에서 조종자의 개입 유연성을 중시한다. 상태 기계의 그래프 구조에 다양한 역방향 엣지가 허용되며, 조건 분기문(If-else) 위주의 절차적(Procedural) 모드 전환 방식을 채택하고 있어 상대적으로 상태 전이 그래프가 유연하고 상호 연결성(Inter-connectivity)이 높다. 즉, 엄격한 DAG 형식이라기보다는 일반 다중 그래프(Multi-graph) 성격을 띈다.
  • PX4-Autopilot: 상태 기계를 수학적으로 정형화하는 것을 선호한다. 모드 전환은 독립된 FSM 객체 내에서 엄밀한 전제 조건(Pre-condition) 검사를 통과해야만 이루어지며, 상태 간의 단절성(Decoupling)이 명확하다. 이러한 DAG 기반의 보수적 상태 설계는 자율 비행(Offboard, ROS2 연동) 에이전트를 개발할 때 예외 상황 테스트 범위(Test Coverage)를 크게 줄여주는 아키텍처적 이점을 제공한다.

다이어그램에서 확인되듯 주요 임무 전개 프로세스는 하향(Forward) 방향성을 가지며, 시스템 복구를 위한 역방향 화살표는 오직 안전한 절차(착륙 및 셧다운)를 통해서만 스탠바이(Standby) 상태로 되돌아가게 통제되어 있다. AUTO MISSION과 MANUAL 간의 전환은 이 모델에서 예외적으로 양방향 엣지가 허용되는 사용자 인더루프(Human-in-the-loop) 인터벌에 해당한다.

5. 결론

PX4-Autopilot의 커맨더 구조는 무분별한 상태 전이를 원천적으로 차단하기 위해 방향성 비순환 그래프(DAG) 형태의 토폴로지를 적극 도입하였다. 이 아키텍처는 유한 상태 단계 간의 논리적 모순과 무한 루프 붕괴 리스크를 공학적으로 소거(Eliminate)한다. 개발자는 PX4의 새로운 커스텀 비행 모드를 MAVSDK나 ROS2(uXRCE-DDS) 환경에서 연동할 때, 이 기존의 비순환적, 결정론적 위계도를 정확히 파악하고 그 위상을 거스르지 않도록 통제 인터페이스를 설계해야만 오토파일럿의 상태 안전망(Safety Interlock)과 상충(Collision)하지 않는 안정적 제어를 달성할 수 있다.
