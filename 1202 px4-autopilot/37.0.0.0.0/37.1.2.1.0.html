37.1.2.1 제어 입력에 대한 기체 동역학적 반응의 부분 관측 마르코프 결정 과정(POMDP) 특성

지상 관제 시스템(GCS)이나 경로 계획 에이전트가 무인항공기(UAV)에 하달하는 제어 입력(Control Input)은 기체의 동역학적 반응(Dynamic Response)을 즉각적으로 유발하지만, 관제 주체는 기체의 정확한 진성 상태(True State)를 실시간으로 완벽히 파악할 수 없다. 이러한 불완전한 상태 인지 환경 하에서의 순차적 의사결정 문제는 확률론적 제어 이론의 핵심인 부분 관측 마르코프 결정 과정(Partially Observable Markov Decision Process, POMDP)으로 명확히 모델링될 수 있다. 본 절에서는 GCS와 PX4-Autopilot 간의 상호작용을 POMDP 프레임워크로 해석하고, 이것이 관제 시스템 설계에 주는 시사점을 논의한다.

1. 마르코프 결정 과정(MDP)에서 부분 관측성(Partial Observability)의 개입

일반적인 마르코프 결정 과정(MDP)은 현재 상태 s \in S를 관측하고 행동 a \in A를 취했을 때, 상태 전이 확률 T(s' | s, a)에 따라 다음 상태 s'로 전이되며 보상 R을 얻는 수학적 모델이다. 그러나 GCS 환경에서는 센서 노이즈와 텔레메트리 대역폭의 한계로 인해 기체의 실제 동역학적 상태 s를 직접적으로 관측할 수 없다는 치명적 제약이 따른다.

1.1 POMDP의 구성 요소 수식화

GCS 통제 시스템을 POMDP로 정의하기 위한 튜플 (S, A, T, R, \Omega, O)의 구성 요소는 다음과 같다.

  • 상태 공간 (S): 기체의 진성 동역학 상태(위치, 속도, 자세, 바람 외란 등) 및 시스템 내부 논리 상태(Arming, Flight Mode).
  • 행동 공간 (A): GCS가 PX4에 발송할 수 있는 MAVLink 제어 명령어(COMMAND_LONG, SET_POSITION_TARGET 등).
  • 전이 모델 (T): GCS의 명령 a 인가 후, PX4 커맨더 및 내부 물리 엔진에 의한 실제 상태 s에서 s'로의 확률적 전이 T(s'|s,a).
  • 관측 공간 (\Omega): GCS가 텔레메트리를 통해 수신할 수 있는 제한된 메시지 공간(HEARTBEAT, LOCAL_POSITION_NED, ATTITUDE 등).
  • 관측 확률 모델 (O): 실제 상태가 s'일 때, 기체의 센서 및 통신 채널을 거쳐 GCS가 특정 관측 o \in \Omega를 수신할 확률 O(o | s', a). 통신 지연이나 노이즈가 클수록 이 확률 분포는 넓게 퍼지게 된다(High Uncertainty).

2. GCS의 신뢰 상태(Belief State) 갱신 메커니즘

POMDP 환경 하에서, GCS는 기체의 정확한 상태 s 대신, 기체가 특정 상태 s에 있을 확률 분포인 **신뢰 상태(Belief State, b)**를 유지하고 갱신해야 한다. b(s)는 기체가 상태 s에 있을 확률을 뜻하며, \sum_{s \in S} b(s) = 1을 만족한다.

2.1 베이즈 정리(Bayes’ Theorem)에 기반한 Belief 갱신

GCS가 이전 신뢰 상태 b에서 명령 행동 a를 취하고, 새로운 MAVLink 텔레메트리 관측치 o를 수신했을 때, 새로운 신뢰 상태 b'(s')는 베이즈 정리에 의해 다음과 같이 갱신된다.

b'(s') = \eta \cdot O(o | s', a) \sum_{s \in S} T(s' | s, a) b(s)

(단, \eta는 확률의 총합을 1로 만들기 위한 정규화 상수이다.)

PX4 환경의 QGroundControl(QGC) 내부 ‘Vehicle’ 객체 모델은 엄밀한 수학적 확률을 실시간으로 연산하지는 않지만, 논리적으로 위와 합치되는 휴리스틱 기반의 신뢰 상태 추정기(Belief Estimator)를 운용하고 평활화(Smoothing)한다. GCS는 수신되는 STATUSTEXT나 HEARTBEAT을 o로 취급하여, 내부 GUI와 상태 변수 b를 지속적으로 동기화한다.

3. 기체 동역학의 지연 반응과 POMDP 제어 한계

로터의 관성(Inertia)과 기체 무게 등 물리적 동역학으로 인하여 제어 명령 a가 즉각적인 상태 s'의 도달을 보장하지 않는다.

3.1 과도 응답(Transient Response) 관측의 지연

예를 들어, GCS에서 Takeoff 명령을 하달했을 때 기체의 추력(Thrust)은 설정된 고도까지 목표 궤적을 그리며 서서히 증가한다. 이때 단기적으로 수신되는 관측 o (고도 변화량)는 매우 미미할 수 있다. POMDP 모델에서 이는 관측 확률 O(o | s', a)이 행동 a 성공 여부를 확정하기에 정보량이 부족함(Low Information Gain)을 의미한다. 이를 보완하기 위해 PX4는 물리적 고도 도달 전에 논리적 COMMAND_ACK를 즉시 회신함으로써, GCS의 확신(Belief Confidence)을 임의로 높여준다.

3.2 상태 발산 방어를 위한 안전 보수주의(Safety Conservatism)

GCS 또는 상위 ROS2 관제 에이전트를 설계할 때, 연속된 관측 o의 누락(Telemetry Loss)이 발생하면 식의 업데이트가 중단되고 신뢰 상태 b의 불확실성(Entropy)이 급증한다. POMDP 최적 정책(Optimal Policy)에 따르면 불확실성이 특정 임계치를 초과할 때 가장 보수적이고 안전한 행동(예: Hovering 명령 대기, RTL 모드 강제 전환)을 선택해야 한다. PX4-Autopilot의 Failsafe 철학은 철저히 이러한 POMDP 기반 위험도 최소화 수학 모델과 일치한다.
