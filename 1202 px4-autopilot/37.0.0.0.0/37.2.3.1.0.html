37.2.3.1 인접 행렬(Adjacency Matrix)을 활용한 상태 간 도달 가능성(Reachability) 평가

복잡한 비행 제어 소프트웨어에서 기체가 겪을 수 있는 모든 상태를 노드(Node, V)로, 허용되는 상태 전이를 유향 엣지(Directed Edge, E)로 간주할 경우, 전체 시스템의 유한 상태 기계(FSM)는 하나의 거대한 수학적 그래프(Graph, G = (V, E))로 모델링된다. PX4-Autopilot의 커맨더(Commander) 아키텍처는 임의의 상태에서 다른 특정 상태로의 직접적인 이동이 유효한지(Valid), 아니면 거부되어야 하는지(Invalid)를 판별하기 위해, 그래프 이론의 인접 행렬(Adjacency Matrix) 기반 도달 가능성(Reachability) 평가 메커니즘을 이면에 내포하고 있다.

1. 인접 행렬(Adjacency Matrix) 모델의 수학적 정의

PX4 시스템에 존재하는 n개의 상태 집합을 S = \{s_1, s_2, \dots, s_n\} 이라 할 때, 시스템의 상태 전이 유효성을 n \times n 차원의 인접 행렬 A로 기술할 수 있다.

행렬의 각 원소 a_{ij} (여기서 i는 현재 상태 노드의 인덱스, j는 타겟 상태 노드의 인덱스)는 다음과 같은 불리언(Boolean) 값을 갖는다:

a_{ij} = \begin{cases} 1, & s_i \text{에서 } s_j \text{로의 직행 전이 규칙이 소스 코드 내에 허용된 경우} \\ 0, & s_i \text{에서 } s_j \text{로의 직행 전이가 금지된 경우 (도달 불가능)} \end{cases}

1.1 희소 행렬(Sparse Matrix)로서의 제어 토폴로지

항공 제어 시스템의 안전성을 담보하기 위해 행렬 A는 대부분의 원소가 0인 몹시 성긴 희소 행렬(Sparse Matrix) 양상을 띈다. 이는 ’아무 상태에서나 아무 상태를 즉시 호출할 수 없다’는 강한 제약 조건을 시사한다. 예를 들어 기체가 INIT(부팅 중) 상태에서 단계를 뛰어넘어 직접 AUTO_MISSION(자율 임무) 상태로 전환되는 전이 a_{INIT, MISSION} = 0 이며, 시스템은 이러한 전이 시도를 소스 코드 레벨에서 즉시 무시한다.

2. PX4-Autopilot의 상태 도달 가능성(Reachability) 제어 구조

소스 코드 내부(src/modules/commander/)에서, 이 인접 행렬 로직은 거대한 2차원 배열 데이터가 아니라 main_state_transition 함수 내부의 엄밀한 switch-case와 조건 배제 논리로 구현되어 있다.

2.1 방향성 비순환 그래프(DAG) 특성과 k-Step 도달 가능성

특정 상태 s_i에서 s_j로 진입하기 위해서는 반드시 중간 상태를 거쳐야 하는 경우가 있다.
그래프 이론에 따르면 행렬 A를 k번 곱한 A^k의 i행 j열 원소가 0보다 크다면, 상태 s_i에서 k번의 전이 스텝을 통해 s_j에 논리적으로 도달할 수 있음을 의미한다.

  • 예시 시나리오: 기체가 STANDBY(s_1) 상태에 있을 때 조종자가 GCS에서 곧바로 TAKEOFF(s_3) 버튼을 누르면, \text{Adjacency}(s_1, s_3)는 0이므로 직접 전이는 블로킹된다. 그러나 커맨더 논리상 STANDBY \rightarrow ARMED \rightarrow TAKEOFF라는 2-Step(A^2) 경로는 열려 있으므로, GCS나 에이전트 소프트웨어는 명령어 전송의 중간 단계(MAV_CMD_COMPONENT_ARM_DISARM)를 반드시 먼저 끼워넣어 순차적 도달 가능성을 해소해야만 목적을 달성할 수 있다.

2.2 예외적 전연결(Fully-Connected) 엣지의 허용

보수적이고 성긴 인접 행렬 토폴로지 속에서도, 기압계 고장, 배터리 임계치 도달, 또는 강제 비상 정지(Kill Switch)와 같은 안전 관련 Failsafe 상태 노드에 대해서는 기체의 생존을 위해 예외적인 규칙이 부여된다.
즉, 모든 비행 모드 상태 노드 s_{active}에 대하여, \text{Failsafe} 노드로의 전이 값 a_{active, failsafe} 는 항상 1로 강제(Force) 오픈된다.

3. PX4와 Ardupilot의 인접 행렬 토폴로지 지향점 차이

  • Ardupilot: 실용적 비행을 중시하여 상태 간의 연결 엣지가 PX4보다 조밀(Dense)하게 얽혀 있다. 특정 상태에서 다른 모드로 진입 가능한 경우의 수(행렬 원소 중 1의 비율)가 많아 조종 편의성은 높지만, 수학적 모델링 관점에서는 상태의 전개 향방을 수학적으로 증명하기가 까다로워진다.
  • PX4: 도달 가능성의 패스(Path)를 지극히 단조롭고 엄격하게 축소한 희소 행렬 철학을 갖는다. 이는 논리적 버그나 무한 루프 상태를 예방하는 가장 훌륭한 정형 기법(Formal Method)이며, ROS2(uXRCE-DDS) 등 이기종 컴퓨터와의 상태 로직 연동 시 개발자가 FSM 스펙 문서를 직관적으로 검증할 수 있는 신뢰성의 척도가 된다.

4. 커맨더의 인접 행렬과 가상의 상태 전이 매트릭스 도식화

PX4의 핵심 상태들의 인접 관계를 도식화하면, 허용된 1-Step 도달 가능성(행 번호 -> 열 번호 전이)을 표 현하는 가상의 불리언 행렬 구조를 다음과 같이 확인할 수 있다.

        S1 -.->|a_13 = 0 (도달 불가)| S3:::Invalid
        S1 -.->|a_14 = 0 (도달 불가)| S4:::Invalid

    end

도해에서 보듯, STANDBY에서 MISSION 체공으로 직행하려는 시도는 a_{14} = 0 이라는 인접 행렬의 룰에 의해 커맨더 단에서 가차 없이 폐기된다. 오토파일럿 오퍼레이터는 이 징검다리를 정석적으로 통과(Traverse)하는 커맨드 시퀀스를 수립해야 한다.

5. 결론

비행 제어 소프트웨어의 생명력은 외적인 연산 능력의 속도가 아니라 런타임 제어권의 수학적 통제 및 분리 가능성에 기인한다. PX4-Autopilot은 인접 행렬 관점의 도달 가능성(Reachability) 평가를 상태 천이 논리망의 뼈대로 차용함으로써, 시스템이 예기치 못한 유령 모드(Phantom Mode)로 전이되는 리스크를 행렬 대수학적으로 원천 봉쇄하였다. 자율 비행 알고리즘을 설계하는 현대의 GCS 및 Companion Computer 임베디드 개발자는 대상 기체의 인접 행렬 맵을 철저히 독해하여, MAVLink 요청 시퀀스가 유효한 도달 경로 스텝(A^k)을 완벽히 만족하도록 트랜잭션 패킷을 재설계하는 수학적 통제 마인드를 갖추어야 한다.
