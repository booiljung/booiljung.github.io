37.3.1.1 현재 상태 벡터(Current State Vector)와 목표 상태 벡터(Target State Vector) 간의 위상학적 거리(Topological Distance) 계산

무인항공기(UAV)의 제어는 본질적으로 기체가 현재 위치하고 있는 물리적 및 논리적 상태에서, GCS(Ground Control Station)나 자율 관제 에이전트가 요구하는 새로운 목표 상태로 전이(Transition)하는 과정이다. 이 전이의 타당성과 물리적 가용성을 평가하기 위해 PX4-Autopilot은 **현재 상태 벡터(Current State Vector)**와 목표 상태 벡터(Target State Vector) 간의 **위상학적 거리(Topological Distance)**를 산출하여 동적 명령어 검증에 활용한다. 본 절에서는 이러한 수학적 거리가 제어 명령의 수용 및 기각 논리에 어떻게 개입하는지 분석한다.

1. 상태 벡터(State Vector)의 정의

비행 시스템에서 상태 벡터는 단순히 3차원 공간 좌표(Position)만을 의미하지 않는다. 상태 공간 모델링(State Space Modeling) 관점에서, 무인기의 상태는 물리적 기구학(Kinematics) 상태와 오토파일럿의 논리적 내부 상태(Logical State)를 모두 포괄하는 다차원 구조체이다.

  • 현재 상태 벡터 (\mathbf{S}_{curr})
  • \mathbf{S}_{curr} = [x, y, z, v_x, v_y, v_z, \phi, \theta, \psi, \omega_x, \omega_y, \omega_z, M_{curr}, A_{curr}, I_{curr}]^T
  • 여기서 x, y, z는 NED(North-East-Down) 좌표계에서의 위치, v는 속도, \phi, \theta, \psi는 오일러 각(Euler Angles) 기반의 자세, \omega는 각속도를 나타낸다.
  • M_{curr}는 현재 비행 모드(예: Position, Offboard), A_{curr}는 시동 상태(Armed/Disarmed), I_{curr}는 하드웨어 인터락 잠금 상태를 뜻한다.
  • 목표 상태 벡터 (\mathbf{S}_{target})
  • GCS나 Companion Computer(ROS 2)가 하달하는 MAVLink 명령어(예: SET_POSITION_TARGET_LOCAL_NED)를 매핑형 함수를 통해 목표 벡터로 정규화한 형태이다.

2. 위상학적 거리(Topological Distance)의 산출

PX4의 Commander 및 FlightTask 모듈 아키텍처는 목표로 하는 상태(예: 새로운 웨이포인트 위치 또는 새로운 비행 모드)가 현재 상태에서 유효하게, 그리고 안전하게 도달 가능한지 계산한다. 이를 위상학적 거리(Topological Distance) 함수 D(\mathbf{S}_{curr}, \mathbf{S}_{target}) 로 정의할 수 있다.

이 거리는 단순한 유클리드 거리(Euclidean Distance)가 아니며, 상태 기계(State Machine)의 전이 가중치 역학을 포함한다.

D(\mathbf{S}_{curr}, \mathbf{S}_{target}) = \alpha D_{kinematic}(\mathbf{S}_{curr}, \mathbf{S}_{target}) + \beta D_{logical}(\mathbf{S}_{curr}, \mathbf{S}_{target})

2.1 기구학적 거리 (D_{kinematic}) 평가

물리적인 거리의 제어 수용성(Acceptability)을 평가한다. GCS에서 현재 기체 위치에서 100km 떨어진 좌표로 이동하라는 Goto 명령이 하달되었다면, PX4는 현재 남은 배터리 잔량의 예상 비행 거리 파라미터나 멀티로터의 최대 이동반경(Geofence 제한) 제약 조건 내에 있는지 연산한다.
만약 목표 상태와의 물리적 거리가 기구학적 제약을 초과한다면, 이는 명령어 변조 현상이나 자율 에이전트의 경로 계획 알고리즘 오류로 간주하여 수용을 거부(Reject)하거나 안전한 중간 웨이포인트(Safe target)로 값을 클리핑(Clipping) 처리한다. (MPC 트래젝터리의 연산 오류 사전 방지)

2.2 논리적 상태 거리 (D_{logical}) 평가

상태 천이 그래프(State Transition Graph)에서 현재 노드에서 목표 노드까지 도달하기 위한 최소 간선 가중치(Edge Wait) 또는 필수 선행 조건의 불일치 정도를 측정한다.

  • 예시: 시동이 꺼져 있는(Disarmed) 상태에서 이륙(Takeoff) 상태로 직접 전이하려는 명령이 수신된 경우.
  • 논리적 거리 D_{logical} 은 극환적으로 크게(무한대 \infty) 산출된다. 정상적인 전이는 Disarmed \rightarrow Armed \rightarrow Takeoff 의 단계를 거쳐야 하기 때문이다.
  • 논리적 거리가 임계치 이상일 시, Commander는 해당 명령을 즉각 기각하며 MAV_RESULT_DENIED를 GCS에 응답한다.

3. Ardupilot과 PX4의 위상 거리 계산 패러다임 차이

명령이 기체의 현재 상태에서 무리한(Unreachable) 목표를 요청할 때 두 플랫폼의 대응 구조에 차이가 존재한다.

  • PX4의 접근 방식: 엄격한 이산 사건 시스템(Discrete Event System) 분석에 가깝다. PX4는 FlightTask 서브모듈 계층 구조를 통해 명령된 위치나 속도가 물리적으로 추종 가능한지 지속적으로 평가하고, 한계에 부딪히면 오프보드(Offboard)와 같은 독립적 제어 루프를 아예 강제 오버라이드하여 Hold 모드로 천이시킨다. 목표 자체를 기체가 달성할 수 없는 경우 위상 공간 상 경로 자체를 끊어버림으로써 시스템이 예기치 않게 튀는(Runaway) 현상을 막는다.
  • Ardupilot의 접근 방식: 상대적으로 유연한 경로 유지 및 비례-미분(PID) 포화(Saturation) 방지 구조에 더 의존한다. 불가능한 목표가 주어지면, Ardupilot은 우선 도달 가능한 방향으로 내비게이션 컨트롤러(L1 Controller 등)를 최대한 구동시키며, 지속적인 에러 누적에 대한 의존도를 높게 가져가면서 GCS 관제사의 연속 개입(User override)을 더 친화적으로 허용하는 경향이 있다.

4. ROS 2 오프보드 제어 환경의 위상학적 스무딩(Smoothing) 시사점

ROS 2(Micro XRCE-DDS) 브리지를 활용한 자율 비행 개발자는 이러한 위상학적 거리 개념을 경로 계획(Path Planning) 노드에 내재화시켜야 한다.
오토파일럿 내부의 위치/속도 제어 루프 주파수(약 50~250Hz)로 고속 명령을 스트리밍할 때, 이전 목표 벡터와 새로운 목표 벡터 간의 유클리드 거리가 비현실적으로 크게 튀는 이른바 “저크 스파이크(Jerk Spike)“가 발생하면 무결성 검증 필터에 걸려버릴 확률이 있다.

도표와 같이, GCS 또는 자율 에이전트 측은 목표 상태를 하달하기 전 자체적인 경로 스무딩(Smoothing) 궤적을 통해 위상 거리를 좁혀(Threshold 이내) PX4에 순차적으로 주입해야만 제어 명령 기각과 비정상적인 Fail-safe 발동을 예방할 수 있다. 이를 통해 비행 모드 스위칭과 물리적 운동 방정식의 일치성을 동기화하는 것이 오토파일럿 기반 자율 이동 시스템 설계의 척도이다.
