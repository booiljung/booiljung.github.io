37.2.1 오토마타 이론(Automata Theory)에 입각한 PX4 시스템 상태 공간(System State Space) 정의

현대의 복잡한 무인항공기(UAV) 비행 제어 소프트웨어, 특히 PX4-Autopilot은 단순히 절차적(Procedural) 제어 명령의 나열로 동작하지 않으며, 수학적으로 엄밀하게 정의된 유한 오토마타(Finite Automata)의 원리에 기반하여 구동된다. 이는 기체가 수많은 센서 입력과 조종자 명령 속에서도 항상 예측 가능하고 안전한(Safe and Deterministic) 행동 양식을 유지하도록 강제하는 토대이다. 본 절에서는 오토마타 이론의 관점에서 PX4 시스템이 가지는 상태 공간(System State Space)을 어떻게 정형화(Formalize)하여 정의하는지 학문적, 구조적으로 분석한다.

1. 유한 오토마타(Finite Automata)로서의 PX4 제어기 모델링

오토마타 이론에서 결정론적 유한 오토마타(Deterministic Finite Automaton, DFA)는 다음과 같은 5-튜플(Tuple) M = (Q, \Sigma, \delta, q_0, F)로 정의된다. PX4의 커맨더(Commander) 모듈을 이 프레임워크에 대입하여 해석할 수 있다.

1.1 PX4 상태 기계의 5-튜플(Tuple) 매핑

  • 상태의 유한 집합 (Q): PX4 비행 제어 시스템이 동시에 가질 수 있는 모든 상호 배타적(Mutually Exclusive)인 시스템 상태들의 집합. 예를 들어 INIT, STANDBY, ARMED, TAKEOFF, MISSION, LANDING, FAILSAFE 등이 노드로 존재한다.
  • 입력 알파벳 (\Sigma): 기제의 상태를 변이시키는 모든 유효한 외부 및 내부 이벤트의 집합. GCS로부터 수신된 MAVLink 명령(COMMAND_LONG), RC 조종기의 스틱 스위치 조작, 센서 데이터 한계 초과 알림 (예: 배터리 부족 이벤트), uORB를 통한 EKF2(Estimator)의 위치 불량 플래그 등이 이에 해당한다.
  • 전이 함수 (\delta : Q \times \Sigma \rightarrow Q): 현재 상태 q \in Q에서 특정 이벤트 \sigma \in \Sigma가 발생했을 때, 시스템이 합법적으로 진입해야 할 다음 상태 q'를 지시하는 매핑 규칙이다. (예: $\delta(STANDBY, ARM\_COMMAND) = ARMED$)
  • 초기 상태 (q_0): 펌웨어 부팅 후 NuttX RTOS가 커맨더 스레드를 시작할 때 진입하는 최초 시스템 상태. 보통 BOOT 또는 INIT 상태로 설정된다.
  • 최종 상태 또는 수락 상태 집합 (F): 비행 임무를 무결하게 마치고 시스템이 전원 차단을 허용할 수 있는 종료 상태 (예: 착륙 후 모터 회전이 완전히 정지된 DISARMED & SHUTDOWN_READY 상태).

2. PX4 상태 공간(State Space)의 다차원적 분리 (Dimensional Separation)

항공 제어 소프트웨어에서 상태 공간 Q를 단일 차원으로 평면화(Flattening)할 경우 상태 폭발(State Explosion) 문제가 발생한다. PX4는 이를 방지하기 위해 상태 공간을 직교하는 벡터 부분 공간(Orthogonal Subspaces)으로 격리하여 다차원 FSM으로 운용한다.

2.1 주 상태(Main State) 벡터와 보조 상태(Sub State) 벡터

PX4의 커맨더 로직 내 상태 변동은 주로 vehicle_status.msg uORB 토픽으로 브로드캐스팅된다. 이 상태 공간은 크게 세 가지 하위 상태 공간으로 세분화된다.

  1. 무장 상태 공간 (Arming State Space): 물리적 모터 구동 및 동력 계통의 활성화 여부를 다루는 가장 중요한 물리 통제 계층. (INIT, STANDBY, ARMED, SHUTDOWN)
  2. 비행 모드 상태 공간 (Nav State Space / Flight Mode): 제어기가 궤적(Trajectory)을 생성하는 논리를 결정하는 공간. 조종자 입력의 개입 정도와 자율성 수준에 따라 결정된다. (MANUAL, ALTCTL, POSCTL, AUTO_MISSION, AUTO_RTL, OFFBOARD 등)
  3. 안전 장치 상태 공간 (HIL / Failsafe State Space): 하드웨어 고장, 배터리 소진, 데이터 링크 소실(Data Link Loss) 등에 의해 조건부로 트리거되어 비행 모드에 글로벌 오버라이드(Global Override) 특성을 행사하는 안전 상태 체계.

이러한 분리 설계 덕분에 PX4 FSM은 “기체가 Armed 상태이면서 동시에 Mission 비행 모드 구조를 가지는” 형태의 상태 데카르트 곱(Cartesian Product) 인식이 가능해져, 코드를 구조적/논리적으로 엄밀하게 유지할 수 있다.

3. PX4와 Ardupilot 상태 공간 정의의 이기종 패러다임

유한 오토마타의 전이 함수 \delta를 소스 코드 레벨에서 어떻게 정의하고 통제하느냐에 있어 PX4와 Ardupilot은 현격한 차이를 보인다.

  • Ardupilot의 암묵적(Implicit) 상태 공간:
    Ardupilot (예: ArduCopter)의 경우 모드 전환 시나리오가 각 비행 모드 클래스(C++ Class) 내부의 init() 및 run() 메서드 안에 분산되어 조건문(if-else) 형태로 존재한다. 따라서 중앙 집중화된 상태 전이 맵(State Transition Map)이 명시적으로 존재하지 않으며, 오토마타 측면에서 볼 때 \Sigma에 따른 비결정성(Non-determinism)이 발생할 여지가 비교적 더 크다. 사용자의 유연성은 뛰어나나, 상태 공간의 이론적 정형화는 부족한 ’실용주의 아키텍처’이다.

  • PX4의 명시적(Explicit) 상태 공간:
    PX4는 Commander라는 단일 모듈에 FSM을 철저하게 중앙 집중화(Centralization)한다. 전이 함수 \delta는 커맨더 내의 명시적인 상태 전이 행렬 함수(set_main_state(), arming_state_transition())들을 통과해야만 실행되며, 이를 우회한 모드 변경은 원천적으로 불가능하다. 오토마타 이론의 본질에 매우 충실하게 설계된 PX4는, 예상치 못한 외란 입력 \Sigma_{unexpected}이 들어오더라도 전이 함수에 사전 정의되지 않았다면 거부(Reject)함으로써 ’강력한 결정론적 유한 오토마타’의 이점을 살린다.

4. PX4 상태 공간 부분 공간 모델의 분리 다이어그램

PX4 상태 머신의 다차원적 분리 구조를 오토마타 관점에서 도식화하면 다음과 같다.

graph TD
    subgraph S_Alpha [입력 알파벳 Sigma]
        I1[RC Switch Toggle] 
        I2[MAVLink COMMAND] 
        I3[Sensor Failsafe Event]
    end

    subgraph Q_Total [전체 상태 공간 Q Cartesian Product]
        direction LR
        
        subgraph Q_Arm [Arming 상태 공간]
            A1[STANDBY] --> A2[ARMED]
            A2 --> A1
        end

        subgraph Q_Nav [비행 모드 상태 공간]
            N1[POSCTL] <--> N2[AUTO_MISSION]
            N2 --> N3[OFFBOARD]
        end
        
        subgraph Q_Fail [예외/보안 상태 공간]
            F1[NORMAL] --> F2[FAILSAFE_RTL]
            F2 --> F3[TERMINATION]
        end
    end

    I1 -->|delta 전이 함수| Q_Nav
    I2 -->|delta 전이 함수| Q_Arm
    I3 -->|우선순위 오버라이드| Q_Fail
    
    %% Q_Total represents the cross product of these sub-spaces

이처럼 PX4는 다차원의 부분 공간 Q_{Arm} \times Q_{Nav} \times Q_{Fail} 구조를 운용하여, 단일 차원 배열 시 폭발적으로 증가할 수 있는 노드 개수를 효과적으로 억제한다.

5. 결론

무인항공기를 하나의 유한 오토마타 시스템으로 해석하는 관점은 PX4-Autopilot 아키텍처의 철학적 통찰을 제공한다. PX4는 전이 함수 \delta와 상태 공간 Q를 고도로 중앙 집중화하고 직교 공간으로 분할(Orthogonal Partitioning)함으로써, 런타임(Runtime) 시에 발현될 수 있는 스레드 데드락(Deadlock)이나 상태 채터링(Chattering) 현상을 원천 방어한다. 항공 소프트웨어 개발자 및 자율 비행 로직 설계자는 단순한 MAVLink 통신 API 호출 레벨에 머무를 것이 아니라, 이러한 오토마타 이론적 구조를 완벽하게 숙지한 상태에서 SET_POSITION_TARGET 등의 제어 입력 알파벳 \Sigma를 적시에 주입해야만 GCS-기체 간의 상충 없는 부드러운 상태 전이를 유도할 수 있다.
