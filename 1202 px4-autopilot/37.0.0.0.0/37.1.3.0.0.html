37.1.3 상태 비저장(Stateless) 아키텍처와 상태 기반(Stateful) 아키텍처의 정보 엔트로피(Information Entropy) 비교

복잡한 네트워크 제어망에 놓인 무인항공기(UAV) 지상 관제 시스템(GCS)은 주어진 채널 용량 한계 내에서 최대한의 지휘 통제력(Command and Control Capability)을 발휘해야 한다. 이때 관제 시스템의 설계가 상태 비저장(Stateless) 아키텍처이냐 상태 기반(Stateful) 아키텍처이냐에 따라 교환해야 하는 통신 트래픽의 형태와 제어 불확실성, 즉 정보 엔트로피(Information Entropy)의 크기가 극명하게 달라진다. 본 절에서는 통신 링크 상에서 교환되는 메시지 정보량의 차이를 섀넌 엔트로피(Shannon Entropy) 관점에서 정량적으로 해석하고, PX4 시스템이 왜 상태 기반 아키텍처를 강제하는지 학술적으로 고찰한다.

1. 제어 아키텍처의 정보 엔트로피 해석 모델

정보 이론(Information Theory)에서 섀넌 엔트로피 H(X)는 확률 변수 X가 가지는 평균 정보량(불확실성)을 의미하며, 다음과 같이 정의된다.

H(X) = - \sum_{x \in X} p(x) \log_{2} p(x) \quad \text{(bits)}

GCS와 오토파일럿 간의 통신에서 ’정보’란, 수신자가 기존에 알지 못했던 새로운 상태 변동(State Transition)이나 외란(Disturbance)에 관한 지식이다. 따라서 설계된 아키텍처가 통신 채널의 엔트로피를 얼마나 효율적으로 관리하느냐가 시스템 최적화의 척도가 된다.

2. 상태 비저장(Stateless) 아키텍처

상태 비저장 관제란, GCS가 기체의 현재 상태를 기억(Memory)하지 않거나 신뢰하지 않은 채, 매 제어 주기마다 원하는 전체 목적 상태(Goal State) 전체를 반복적으로 구겨 넣어 발송하는 패러다임이다.

2.1 높은 메시지 리던던시(Redundancy)와 비효율성

이 방식에서 GCS는 통신 단절로 인한 명령 누락을 방비하기 위해, 변경되지 않은 파라미터나 상태 지시까지 포함된 무거운(Heavy-weight) 패킷을 높은 주파수(예: 50Hz)로 연속 전송한다.
이는 송신되는 메시지 세트 X_{msg}에 실질적인 새로운 정보(상태 변화 엔트로피)가 거의 포함되어 있지 않음에도 불구하고 물리적 대역폭만을 가혹하게 점유하는 현상을 낳는다. 수식적으로, 상호 정보량(Mutual Information) I(X; Y) 대비 중복된 비트의 비율이 지나치게 커 통신 효율이 급감한다.

2.2 제어 멱등성(Idempotency)의 결여

상태 비저장 아키텍처의 더 큰 문제는명령 중복 처리이다. ’이륙(Takeoff)’이나 ’웨이포인트 진행(Next Waypoint)’과 같은 비가역적(Irreversible) 명령 스트림이 맹목적으로 반복 수신될 경우, 오토파일럿 내부에서는 동일한 명령을 여러 번 수행하려다 로직 충돌을 빚거나 Failsafe를 촉발하는 중복 제어(Over-control) 오류가 발생한다. 상태를 기억하지 않기 때문에 제어 멱등성을 보장하기 힘들다.

3. 상태 기반(Stateful) 아키텍처

PX4와 QGroundControl(QGC)이 채택하고 있는 상태 기반 관제는, 통신 양단(UAV와 GCS)에 기체 상태를 모사하는 동기화된 프록시(Proxy) 유한 상태 기계(Finite State Machine, FSM)를 구비하는 방식이다.

3.1 델타 인코딩(Delta-encoding)과 엔트로피 압축 효과

상태 기반 아키텍처 하에서 GCS는 오직 평형(Equilibrium)을 깨뜨리는 **상태 천이 이벤트(State Transition Event)**만을 발송한다. 예를 들어 “현재 Armed 상태이니, Takeoff 모드로 전환하라“는 단일 MAVLink 커맨드 패킷만 전송하며, 이후에는 기체가 발송하는 HEARTBEAT 및 COMMAND_ACK를 통해 프록시 상태를 동기화한다.
이 방식은 기존에 동기화된 상태 공간 정보(기지의 정보)를 송수신 트래픽에서 완전히 배제(델타 압축)하므로, 통신 채널 위를 흘러가는 메시지가 지닌 순수 논리적 엔트로피(단위 비트당 실질적 정보량)를 극대화한다. 즉, 협소한 대역폭의 텔레메트리 채널(SiK Radio 등) 효율을 섀넌 한계(Shannon Limit) 근방까지 극한으로 끌어올릴 수 있다.

3.2 상태 전이 모델 일치성(Consistency)의 조건

정보 압축의 이점을 누리기 위해서, 상태 기반 아키텍처는 양단(GCS와 PX4 Commander) 간의 상태 전이 모델 정합성이 완벽해야 한다는 엄격한 한계 조건을 가진다. 만약 GCS 논리에는 ’센서 고장 시 A 모드로 전이’하는 룰이 누락되어 있고 PX4 내부에는 해당 룰이 내장되어 있다면, 센서 고장이 발생한 순간 양 노드의 프록시 상태는 비가역적으로 엇갈리게 되며 시스템은 통제 불능(Out-of-Sync) 상태로 빠진다.

5. 결론

제어 관점에서 보았을 때, 상태 비저장식 아키텍처는 정보 엔트로피 측면에서 대단히 낭비적이며 멱등성 결여로 인해 자율 비행 로직 붕괴의 태생적 리스크를 안고 있다. 반대로 PX4가 지향하는 상태 기반 아키텍처는 GCS 내부의 거대한 상태 추적기(Proxy FSM) 유지를 요구하는 컴퓨팅 오버헤드가 단점이나, 통신 페이로드의 정보 엔트로피를 응축시켜 병목 현상을 해소하고 강력한 제어 무결성(Integrity)을 보증한다. 향후 다중 무인기 관제(Swarm Control) 및 클라우드(Cloud) 미들웨어 설계에 있어서 이러한 엔트로피 최적화 기반 상태 동기화 기법은 필연적으로 요구되는 아키텍처 표준이다.
