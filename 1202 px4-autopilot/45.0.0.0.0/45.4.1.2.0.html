45.4.1.2. 칼만 필터 혁신 과정(Innovation Sequence)의 백색성(Whiteness) 평가 및 마할라노비스 거리(Mahalanobis Distance) 기반 카이제곱 검정(Chi-square Test) 혁신 게이팅(Innovation Gating) 이탈 감지

1. 개요 및 이론적 배경

PX4-Autopilot(펌웨어 버전 1.14+ 기준)의 핵심 상태 추정 모듈인 EKF2(Extended Kalman Filter 2)는 다수의 센서(IMU, GNSS, 마그네토미터, 기압계 등) 데이터를 융합하여 무인항공기의 위치, 속도, 자세(State)를 정밀하게 추정한다. 이 융합 과정에서 결함이 있거나 과도한 노이즈가 섞인 비정상적인 관측치(Outlier)가 상태 추정에 악영향을 미치는 것을 막기 위해 수학적 검증 로직이 요구된다. 모델 예측과 실측치 사이의 오차를 다루는 혁신(Innovation) 시퀀스의 통계적 특성을 분석하고, 이를 바탕으로 관측치의 신뢰성을 평가하여 수용(Accept) 여부를 결정하는 기법이 바로 **혁신 게이팅(Innovation Gating)**이다.

본 절에서는 칼만 필터의 혁신 과정이 지녀야 하는 백색성(Whiteness) 요건과, 마할라노비스 거리(Mahalanobis Distance) 및 카이제곱 검정(Chi-square Test)을 기반으로 구현된 PX4-Autopilot 내부의 이상 관측치 이탈 감지(Outlier Rejection) 아키텍처를 상세히 분석한다.

2. 혁신 시퀀스(Innovation Sequence)와 백색성(Whiteness) 평가

2.1 혁신(Innovation)의 정의

확장 칼만 필터(EKF) 구조에서 혁신 y_k는 시점 k에서 센서로부터 들어온 실제 관측치 z_k와, 현재 상태 추정치를 바탕으로 예측한 기댓값 h(\hat{x}_{k \vert k-1})의 차이로 정의된다.

y_k = z_k - h(\hat{x}_{k \vert k-1})

혁신은 필터가 관측치로부터 얻은 “새로운 정보(New Information)“의 양을 의미한다.

2.2 백색성(Whiteness)의 수학적 속성

올바르게 설계되고 수렴된 칼만 필터 내에서, 이상적인 혁신 시퀀스 y_1, y_2, \dots, y_k는 평균이 0인 통계적 백색 잡음(Zero-mean White Noise) 특성을 지녀야 한다. 즉, 다음 두 가지 통계적 척도를 만족해야 한다.

  1. 무상관성(Uncorrelatedness): 서로 다른 시점의 혁신 값 간에는 상관관계가 없어야 한다(E[y_k y_j^T] = 0, \text{ for } k \neq j).
  2. 공분산 일치성: 혁신의 자체 공분산 행렬(Innovation Covariance) S_k는 모델 예측 공분산 P_{k \vert k-1}과 관측 노이즈 행렬 R_k의 합인 S_k = H_k P_{k \vert k-1} H_k^T + R_k에 이론적으로 일치해야 한다.

실제 무인항공기의 비행 중 센서 고장, 진동, 혹은 비정상적인 전자기적 간섭(마그네토미터 교란 등)이 발생하면 혁신 시퀀스가 특정 방향으로 편향(Bias)되거나 분산이 급격히 증가하며 백색성을 상실하게 된다. PX4-Autopilot은 이러한 특성 변화를 감지하여 상태 추정의 붕괴(Divergence)를 방지한다.

3. 마할라노비스 거리(Mahalanobis Distance) 기반 혁신 게이팅

단일 센서 데이터가 통계적 정상 범주에 속하는지 판단하기 위해서 유클리디안 거리(Euclidean Distance)가 아닌 **마할라노비스 거리(Mahalanobis Distance)**를 사용한다. 이는 각 변수의 분산과 공분산을 고려하여 오차를 정규화(Normalization)하는 강력한 방법이다.

3.1 마할라노비스 거리 계산식

관측 차원이 m인 혁신 벡터 y_k와 혁신 공분산 S_k에 대한 제곱 마할라노비스 거리 D_M^2는 다음과 같이 계산된다.

D_M^2 = y_k^T S_k^{-1} y_k

PX4 EKF2의 소스 코드에서는 계산 부하를 줄이기 위해 행렬 역연산 보다는 각 상태계 변수에 독립적으로 할당된 혁신 분산(Innovation Variance) 대비 혁신(Innovation) 제곱을 비교하는 비율로 변환하여 처리한다.

2.2 카이제곱 검정(Chi-square Test)에 따른 게이트(Gate) 설정

정상 상태에서 백색성(Whiteness)을 띄는 혁신 시퀀스의 D_M^2는 자유도가 m(관측 차원)인 카이제곱 분포(Chi-square Distribution) \chi^2(m)를 따른다. 이를 근거로, PX4는 센서 측정이 정상일 확률 한계선을 나타내는 임계값을 \gamma로 설정한다.

  • Accept (수용): D_M^2 < \gamma (관측치가 칼만 필터 업데이트에 사용됨)
  • Reject (배제): D_M^2 \ge \gamma (관측치를 이탈치로 판단하여 차단함 = Innovation Gating)

3. PX4-Autopilot 파라미터 구성 및 소스코드 분석

PX4는 비행 환경 특성에 맞게 게이팅 임계값을 직접 파라미터화하여 관리한다. EKF2_*_GATE 형태의 파라미터들이 마할라노비스 거리의 최대 임계 한도 역할을 수행하며, 이 한도를 초과할 확률은 \chi^2 분포 검정을 기초로 한다.

3.1 핵심 게이팅 파라미터

  • EKF2_GPS_V_GATE: GPS 속도 관측에 대한 혁신 게이트 크기 (보통 5.0 SD 수준)
  • EKF2_GPS_P_GATE: GPS 수평/수직 위치 관측에 대한 혁신 게이트 크기
  • EKF2_MAG_GATE: 자기장 3축 관측에 대한 혁신 게이트 크기 (일반적으로 3.0 SD)
  • EKF2_BARO_GATE: 기압계 고도 관측에 대한 혁신 게이트 크기

이 파라미터 수치들은 표준편차(Standard Deviation, SD)의 배수를 의미하며, 파라미터 값이 커질수록 칼만 필터는 오차를 더 관대하게 받아들인다.

3.2 소스코드 아키텍처 (C++)

게이팅 로직은 PX4 펌웨어 내 src/modules/ekf2/EKF/ 디렉터리에 집중되어 구현되어 있다. 측정치 테스트 비(Test Ratio)를 계산하는 과정은 스칼라 분산(innov_var)과 혁신 값(innov)에 기반한다.

// PX4 EKF2 유사 코드의 혁신 검증 체계
float test_ratio = sq(innov) / (innov_var * sq(gate_size));

if (test_ratio < 1.0f) {
    // Chi-square 범위 이내: 관측치 수용
    fuse_measurement();
} else {
    // 범위를 초과: 혁신 게이팅(거부) 
    reject_measurement();
}

테스트 비(Test Ratio)가 1.0 미만이면 센서 데이터를 여전히 신뢰할 수 있음을 의미한다. 1.0 이상인 채로 지속되면 시스템 파괴로 판단하여 커맨더 모듈 측으로 Fallback(타 센서로 의존성 전이)을 지시하게 된다.

4. Ardupilot (AP_NavEKF3) 대비 아키텍처 차이

Ardupilot의 EKF3(AP_NavEKF3 모듈) 또한 혁신 게이트 검정 아키텍처를 취하지만 구체적인 파라미터화와 오차 복원법에서의 철학이 약간 상이하다.

  • 파라미터 세분화 방식: PX4-Autopilot은 EKF2_GPS_P_GATE, EKF2_OFLOW_GATE 등 각 하위 채널과 상태계별로 개별 제어가 가능하도록 극도로 나뉘어져 있다. Ardupilot은 복합 센서의 포괄적 튜닝을 선호하는 경향이 있어 EKF 계층 단위에서 더 묶어서 처리되는 로직이 일부 파라미터에 혼재한다.
  • 이탈치 대응 로직(Lane Switching): PX4는 거절(Rejection)이 연속될 경우 Local Position Invalid 플래그를 세워 커맨더가 Fail-safe를 발동하게 유도하는 반면, Ardupilot은 EKF 코어 내에 존재하는 여러 레인의 필터(Lane Switching)를 자체적으로 절체(Switching)하여 자구책 마련을 내부에서 빠르게 수행하려는 경향이 더 강하다.

5. MAVLink 상태 관제 및 QGroundControl 모니터링 적용

기체가 안정적인 자율 비행 상태에 있는지 모니터링하기 위해, 지상 관제 시스템(GCS)에서는 혁신 게이팅 통과 지표를 실시간으로 살핀다. QGroundControl(v4.3+ 환경)의 MAVLink Inspector 기능은 이 분석에 탁월하다.

PX4는 주기적으로 MAVLink의 ESTIMATOR_STATUS (ID: 230) 메시지를 전송한다. 해당 메시지 구조에는 vel_ratio, pos_horiz_ratio, pos_vert_ratio, mag_ratio 필드가 존재한다.

  • 각 ratio 필드는 실제 소스 상의 Test Ratio 값을 의미한다.
  • 이 값이 1.0 미만인 것은 필터가 현재 환경 추정치에 만족하고 있음을 나타낸다.
  • 반대로, 비행 중 mag_ratio 등이 일시적으로 치솟다가 1.0에 포화(Saturation)되는 것은 즉각적인 강한 자장 교란을 겪었거나 자력계가 Gating 당하여 신뢰도 대열에서 이탈했음을 방증한다.

6. ROS2 uXRCE-DDS 및 자율 관제 기술 연동

ROS2 프레임워크 기반의 Companion Computer(자율 주행 에이전트)는 ROS2 브리지를 경유하여 uORB의 estimator_status 토픽을 주기적으로 구독(Subscribe)하고 지능형 예외 처리를 수행할 수 있다.

# ROS2 Python 센서 혁신 상태 모니터링 노드 예시
from px4_msgs.msg import EstimatorStatus

def estimator_status_callback(self, msg):
    # 혁신 테스트 비가 여유 마진을 넘는지 모니터링
    if msg.mag_test_ratio > 0.8:
        self.get_logger().warn("Warning: 마그네토미터 혁신 마진 감소, 자장 교란 점검 필 요!")
        # 1.0을 넘으면 VIO(Visual Inertial Odometry) 노드로만 위치 추정을 전적으로 의존하는 백업 컨트롤러 가동 등 조치 수행

이와 같은 혁신 비율 상태 모니터링 데이터들은 군집 비행(Swarm) 환경이나 고정밀 실내외 측위 등에서 특정 기체의 궤도 이탈을 선제적으로 예방하고, MAVLink Setpoint 명령을 안전한 모드로 전환하도록 지시하는 데 핵심 기준으로 사용된다.
