45.1.3. 무선 데이터 링크의 시변 지연(Time-Varying Delay) 특성 및 통계적 분석

비행 제어 시스템(Flight Control System)과 지상 관제 시스템(Ground Control Station, GCS)을 잇는 무선 데이터 링크는 무인 항공기(UAV)의 ‘신경망’ 역할을 담당한다. 그러나 대기를 매질로 삼는 무선 주파수(Radio Frequency, RF) 통신은 유선 네트워크와 달리 신호의 전파 지연, 패킷의 소실, 대기열(Queue) 정체 등 복합적인 요인에 의해 그 지연 시간이 고정되지 않고 시시각각 변한다. 본 절에서는 이러한 가변적인 통신 지연을 시변 지연(Time-Varying Delay)으로 정의하고, PX4-Autopilot과 QGroundControl 간 제어 신뢰성을 확보하기 위해 이 지연 특성을 어떻게 통계적으로 통찰하고 분석해야 하는지 심도 있게 논의한다.

1. 무선 데이터 링크 지연의 계층적 구조

MAVLink 메시지가 GCS(QGroundControl)에서 출발하여 PX4 내부의 커맨더(Commander) 모듈에 도달하기까지의 전체 지연(Total Latency) \tau(t) 는 여러 하위 지연 요소들의 합으로 이루어진다.

\tau(t) = \tau_{proc\_tx}(t) + \tau_{queue}(t) + \tau_{mac}(t) + \tau_{prop} + \tau_{proc\_rx}(t)

  • \tau_{proc\_tx}(t) : GCS 측 응용 계층에서의 MAVLink 직렬화(Serialization) 및 OS 버퍼링 처리 소요 시간.
  • \tau_{queue}(t) : RF 모뎀 펌웨어 내 송신 버퍼(Tx Buffer)에서 패킷이 전송을 대기하는 시간. 데이터 트래픽이 몰릴 때 기하급수적으로 증가한다.
  • \tau_{mac}(t) : 매체 접근 제어(Medium Access Control, MAC) 계층 지연. CSMA/CA 메커니즘을 사용하는 Wi-Fi 기반 링크나 특정 시분할 다중 접속(TDMA) 기반 텔레메트리 모뎀에서, 채널 점유를 위해 대기하거나 충돌 후 재전송을 수행하며 발생하는 지연이다. 이것이 시변 지연의 가장 지배적인 요인이다.
  • \tau_{prop} : 전자기파의 절대적 전파 지연(Propagation Delay). 광속에 의존하므로 가시권 비행에서는 수 마이크로초(us)에 불과하여 보통 상수로 간주된다.
  • \tau_{proc\_rx}(t) : 수신 측(PX4)의 UART 인터페이스 DMA 인터럽트 발생 및 NuttX RTOS 상에서의 mavlink_receiver 스레드 스케줄링 대기 시간. uORB 메시지로 변환되어 최종 구독자(Subscriber) 모듈에 전달되기까지의 시간이다.

이들 요소들의 합산적 작용으로 인해 총 지연 시간 \tau(t) 는 상수가 아닌 확률 변수(Random Variable)로 군림하게 된다.

2. 시변 지연의 통계적 분포 특성 (Statistical Distribution)

일반적인 고전 제어 이론에서는 통신 지연을 고정 상수 \tau 혹은 정규 분포(Gaussian Distribution)를 갖는 노이즈로 근사화하여 설계하는 경향이 있다. 그러나 무선 텔레메트리 데이터 속도 분석 결과를 통계학적으로 엄밀히 바라보면, 지연 데이터는 비대칭적(Asymmetric) 구조의 단방향 꼬리가 긴 분포(Heavy-tailed Distribution)를 띤다.

무선 데이터 패킷 대부분은 장애물이 없는 깨끗한 채널에서 최소 물리적 지연(Minimum Delay Floor) 근처에 밀집하여 도착한다. 하지만 페이딩(Fading), 전파 간섭(Interference), 재전송(Retransmissions) 현상을 겪은 극소수의 패킷들은 평균 지연 대비 수십 배 긴 시간 후에 도달하게 된다. 이 현상을 적절히 묘사하기 위해 감마 분포(Gamma Distribution), 와이불 분포(Weibull Distribution), 혹은 대수 정규 분포(Lognormal Distribution) 모델이 주로 연구에 채택된다.

3. 통계적 특성을 반영한 관제 소프트웨어 파라미터 설계

Ardupilot 대비 PX4는 비행 시스템의 안전 모드(Failsafe) 진입 인계점(Threshold)이 정교하고 보수적으로 파라미터화되어 있다. 데이터 링크 손실률(Data Link Loss) 판별이나 QGC의 MAVLink 명령 송출 시의 타임아웃(Timeout) 결정에 있어, 개발자는 단순한 평균(Mean, \mu)을 기준으로 삼아서는 안 된다.

  • 명령 재전송(Retry) 타임아웃: 만약 GCS의 재전송 타임아웃 변수를 지연 분포의 평균인 30ms 수치로 짧게 설정한다면, 롱테일 영역에 있는 패킷(예: 80ms)들이 아직 정상 전송 중임에도 불구하고 QGC는 응답 실패(ACK Loss)로 단정짓고 명령을 재발견(Duplicate Transmission)하게 된다. 이는 대역폭의 불필요한 고갈과 관제망 폭주 부하를 초래한다.
  • 백분위수(Percentile) 차용: 강건한 시스템 설계를 위해서는 관측된 실 데이터를 기반으로 95번째 백분위수(P_{95}) 또는 99번째 백분위수(P_{99})를 도출해야 한다. 99%의 패킷이 수신 가능한 한계 지연 시간(예: P_{99} = 120ms)의 약 1.5배~2배를 GCS의 트랜잭션 타임아웃 상한(UpperBound Configuration)으로 적용하는 것이 학술적/실무적 모범 사례(Best Practice)이다.
  • 데이터 링크 단절(COM_DL_LOSS_T) 진단: PX4-Autopilot 내부 파라미터 COM_DL_LOSS_T (데이터 링크 상실 타임아웃)는 일회성 패킷 지연이 아닌 통신 흐름의 영구적 차단을 판별한다. 시변 지연 곡선에서 측정된 최대 재전송 회복 지연의 꼬리 끝점을 넘어가는 값(보통 10초 내외)으로 설정하여, 안테나를 일시적으로 돌릴 때나 장애물 그림자(Shadowing)를 통과할 때 불필요한 귀환(Return To Launch, RTL) 발동을 억제한다.

4. 제어 공학적 지연 보상 아키텍처 (Delay Compensation)

오프보드 제어(Offboard Control) 모드에서 ROS2 기반 미들웨어(Micro XRCE-DDS 등)를 통해 기체의 위치나 속도를 폐루프(Closed-loop) 기반으로 실시간 제어할 경우, 통계적으로 분석된 시변 지연 \tau(t) 모델은 칼만 필터(Kalman Filter)나 마르코프 점프 선형 시스템(Markov Jump Linear System) 설계의 입력 변수가 된다.

네트워크 모델링 기법을 적용하여 관제단에서 지연량을 실시간 추정하는 네트워크 지연 관측기(Network Delay Observer)를 구축하면, 현재 입력하려는 제어 명령 u(t) 가 기체에 도달하는 시점 t+\tau(t) 의 기동 동역학을 선반영할 수 있다. 이는 고주파 진동성 불안정(High-frequency Oscillatory Instability)을 방지하고 제어 대역을 확장하는 필수 메커니즘이다.

5. 결론 및 종합 분석

현대 드론 통신 시스템에서 무선 링크 채널의 상태는 결코 정적(Static)이지 않으며 고무줄처럼 늘고 줄어드는 시변 지연 특성을 가진다. 지상 관제 시스템 운영자와 시스템 설계 공학자는 핑(ping) 테스트로 얻게되는 단일 평균값의 함정에 빠지지 말고, 확률 분포의 롱테일 메커니즘을 명백히 파악해야 한다. 무작위성과 비대칭성이 공존하는 통계적 속성을 올바르게 인지하고 대응 파라미터를 조율할 때 비로소 GCS와 FC 간의 자율 통신 체계가 이상적인 무결점(Zero Defect) 동기화 성능에 도달할 수 있다.
