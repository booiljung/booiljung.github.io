45.8.2.1 웨이포인트(Waypoint) 좌표계의 기하학적 위상(Topological Phase) 유효성 판별 (볼록 다각형 내포 여부 및 지오펜스 교차 검증)

임무(Mission) 자율 비행 모드에서 웨이포인트(Waypoint)는 단순한 3차원 공간상의 점(Point) 이상을 의미한다. 기체가 주어진 궤적을 자율적으로 추종하기 위해서는 각 웨이포인트가 물리적으로 도달 가능하고, 설정된 안전 구역 내에 존재하는지를 수학적으로 엄밀히 검증하는 기하학적 위상(Topological Phase) 유효성 판별 과정이 필수적이다. PX4-Autopilot(Firmware v1.14 이상 기준)은 비행 전 또는 임무 업로드 시점에 이러한 검증 파이프라인을 동작시켜, 유효하지 않은 임무 계획이 실행되는 것을 원천적으로 차단한다.

특히 지오펜스(Geofence) 시스템과의 교차 검증은 기체의 생존성과 직결되는 핵심 안전 로직이다. 본 절에서는 웨이포인트가 볼록 다각형(Convex Polygon)으로 정의된 지오펜스 내부에 속해 있는지(내포 여부)를 판별하는 알고리즘과, 웨이포인트 간의 경로 선분이 지오펜스의 경계를 교차하는지 판별하는 아키텍처 및 소스 코드 레벨의 구현을 상세히 분석한다.

1. 기하학적 위상 유효성 판별의 수학적 배경

웨이포인트 유효성 검사는 주로 2차원 평면(위도 및 경도 기반의 로컬 투영 평면) 상에서 기하학적 알고리즘을 통해 이루어진다. 지오펜스는 일반적으로 다각형(Polygon) 형태로 묘사되며, PX4 시스템에서는 컴퓨팅 자원의 효율성을 위해 볼록 다각형(Convex Polygon) 기반의 판별 로직을 선호하지만, 오목 다각형(Concave Polygon)에 대해서도 유효성을 검증할 수 있는 광선 교차법(Ray Casting Algorithm)을 지원한다.

1.1 점과 다각형의 내포(Point-in-Polygon) 검증 알고리즘

주어진 웨이포인트 P(x, y)가 N개의 꼭짓점(Vertex)으로 이루어진 지오펜스 다각형 G 내부에 속해 있는지를 판별하기 위해, PX4 내부에서는 레이 캐스팅 알고리즘(Ray-Casting Algorithm) 또는 윈딩 넘버(Winding Number) 알고리즘을 사용한다. 대표적인 판단 기준은 다음과 같다.

  • 광선 교차법 (Ray Casting Algorithm): 점 P에서 임의의 방향(일반적으로 양의 x축 방향)으로 무한히 뻗어나가는 반직선(Ray)을 그어, 지오펜스 다각형의 변(Edge)과 교차하는 횟수를 센다. 교차 횟수가 홀수이면 점은 다각형 내부에 존재하고, 짝수이면 외부에 존재한다.

계산 복잡도는 다각형의 꼭짓점 개수 N에 대해 O(N)의 시간 복잡도를 갖는다.

1.2 경로 선분 교차(Line Segment Intersection) 검증

웨이포인트 A와 다음 웨이포인트 B를 잇는 비행 경로(선분 \overline{AB})가 지오펜스 경계를 통과하여 외부로 나갔다가 다시 들어오는지 확인해야 한다. 비록 두 웨이포인트가 모두 지오펜스 내부에 있더라도 오목 다각형(Concave Polygon)인 지오펜스에서는 경로 선분이 외부를 교차할 수 있기 때문이다.

두 선분 \overline{AB}와 \overline{CD}의 교차 여부를 판별하기 위해 외적(Cross Product) 연산에 기반한 CCW(Counter-Clockwise) 알고리즘을 주로 사용한다.

2. PX4-Autopilot의 지오펜스 검증 아키텍처 및 소스 코드 분석

PX4 의 지오펜스 처리 및 웨이포인트 검증은 주로 FlightSector, Geofence, Navigator 모듈 내부에서 처리된다. QGroundControl(QGC, v4.3.0 이상)에서 전송된 임무 및 지오펜스 정보는 MAVLink 프로토콜(MISSION_ITEM_INT, FENCE_POINT 메시지)을 통해 비행 제어기(Flight Controller)에 저장된다.

2.2 Data Manager (dataman) 와의 연동

웨이포인트와 폴리곤 꼭짓점 정보는 플래시 메모리 절약과 무결성 확보를 위해 dataman이라는 데이터 관리를 위한 내부 서브시스템을 통해 직렬화(Serialize)되어 읽고 쓰인다. 지오펜스 모델을 실시간으로 메모리에 모두 상주시키기보다는, 필요 시점에 dataman에서 버퍼로 읽어와 기하학적 연산을 수행하는 구조로 되어 있다.

2.3 Geofence 클래스의 논리적 구현

PX4 소스 코드의 src/modules/navigator/geofence.cpp 에는 웨이포인트 검증 알고리즘이 구현되어 있다. (소스 코드의 세부 디렉토리와 구조는 버전에 따라 상이할 수 있으나 논리적 과정은 유사하다.)

// PX4 소스코드 내부에서 지오펜스 내포 여부를 판별하는 기능의 논리적 예제

bool Geofence::insidePolygon(const vehicle_global_position_s &pos, const GeofencePolygons &polygons) 
{
    // 광선 교차법 (Ray Casting Algorithm) 구현
    bool is_inside = false;
    
    for (int i = 0, j = polygons.num_vertices - 1; i < polygons.num_vertices; j = i++) {
        const float vertex_i_lat = polygons.vertices[i].lat;
        const float vertex_i_lon = polygons.vertices[i].lon;
        const float vertex_j_lat = polygons.vertices[j].lat;
        const float vertex_j_lon = polygons.vertices[j].lon;
        
        // 논리적 판단: 레이가 다각형의 엣지(Edge)와 교차하는가?
        if (((vertex_i_lat > pos.lat) != (vertex_j_lat > pos.lat)) &&
            (pos.lon < (vertex_j_lon - vertex_i_lon) * (pos.lat - vertex_i_lat) / 
            (vertex_j_lat - vertex_i_lat) + vertex_i_lon)) {
            is_inside = !is_inside; // 교차 횟수 반전 (홀/짝 상태 반영)
        }
    }
    return is_inside;
}

이 검사를 통해 임의의 웨이포인트(Target Point) 가 지오펜스의 Inclusing 다각형(접근 허용 구역) 내부에 있는지 확인한다. 여러 개의 다각형(허용 다각형 및 제한 다각형 등)이 존재할 경우, Exclusion 설정이 된 지역을 교집합으로 제외하는 단계도 거친다.

3. Ardupilot과의 비교: 차원(Dimensionality) 및 연산 스케줄링 차이

3.1 3차원적 검사와 고도 기반 회피 한계

Ardupilot(버전 4.x 이상)의 경우 지오펜스 검증을 구면 좌표계(Spherical Coordinate) 연산을 보다 적극적으로 사용하여 AP_Limit_Geofence 라이브러리를 통해 수행한다. 원통형(Cylindrical), 다각형(Polygon), 혹은 포함/제외(Inclusion/Exclusion) 볼륨 등의 조합을 지원한다.

PX4-Autopilot은 단순하고 안정적인 로컬 평면 투영(Local Plane Projection) 방식 기반의 Ray-casting 알고리즘을 사용한다. PX4는 복잡한 오목 다각형 위상의 검사 시간을 최소화하기 위해 연산 범위를 제한하는 스로틀링(Throttling) 및 스케줄링 정책 구조를 가지고 있으며, QGC에서 전송된 MAVLink 패킷 수신과 검증 단계를 분리하여 시스템에 무리를 주지 않도록 관리한다.

결과적으로, 두 시스템 모두 볼록/오목 다각형 지오펜스 위상을 수학적으로 유효성 판별하지만, 그 판별을 언제(When) 얼마나 많은 CPU 할당량(Quota) 내에서(How much) 처리하느냐에 있어 스케줄링 아키텍처에 차이가 있다.

4. 임상 시나리오(Clinical Scenario) 예시: 오목 다각형 진입 시의 동작

사각 지대(Blind Spot)나 건물이 포함된 제한 구역(Exclusion Zone)을 고려하여 U자 형태의 오목한 지오펜스를 구획했다고 가정하자. 이 상황에서 QGroundControl에서 U자의 한 끝에서 다른 끝으로 직선 웨이포인트를 생성할 경우, 두 웨이포인트 자체 상의 기하학적 위상 유효성 판별 검사는 ‘통과(Valid)’ 할지라도 선분 교차(Line Intersection) 알고리즘에 의해 경로 상 안전 구역 이탈이 감지된다.

이러한 위상 모델 판별 실패 시 PX4의 Navigator 모듈은 다음 동작을 수행한다.

  1. 임무 대기열(Mission Queue)에서 대상 웨이포인트 접근 불허 판정.
  2. 기체를 현재 위치에서 Loiter(호버링 대기) 상태로 전환 (또는 설정된 RTL Action 수행).
  3. MAVLink STATUSTEXT 메시지를 통해 GCS(QGC) 화면에 “Geofence Violation Restricted” 경고 구문 점등 및 운영자 개입 유도.
