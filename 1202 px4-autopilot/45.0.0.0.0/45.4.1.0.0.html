45.4.1 확장 칼만 필터(EKF2) 상태 추정치(State Estimate)의 통계적 유효성(Statistical Validity) 종속성 검사 체계

PX4-Autopilot의 핵심 항법 시스템은 원시(Raw) 센서 데이터를 직접 제어루프에 사용하지 않고, 반드시 확장 칼만 필터(EKF2, Extended Kalman Filter)를 거쳐 융합된 상태 추정치(State Estimate)에 의존한다. 지상 관제 시스템(GCS)이 모드 변경(예: Position 모드 전환)을 요구하거나 비행 제어기(Flight Controller)가 자율 제어를 지속할 때, EKF2가 산출한 추정치가 수리 통계적으로 얼마나 진실에 가까운지, 즉 통계적 유효성(Statistical Validity)을 판단하는 로직은 기체의 통제 불능(Fly-away)을 막는 궁극적 안전장치이다. 본 절에서는 EKF2의 추정치 유효성 검증 수학적 원리와, 이를 비행 모드 전환에 종속시키는 시스템 아키텍처를 상세히 다룬다.

1. 통계적 유효성 판별 수학적 배경: 혁신율(Innovation)과 검정

확장 칼만 필터는 운동학적 예측 모델(Prediction Step)과 실제 센서 관측치(Update Step)를 결합하여 상태를 추정한다. 이 두 값의 차이를 혁신(Innovation) 혹은 잔차(Residual)라고 부른다. 단순히 오차의 절대적인 크기만으로는 이 데이터가 유효한지 판단할 수 없으므로, EKF2는 **혁신 검정비(Innovation Test Ratio)**라는 통계적 지표를 활용한다.

혁신 검정비는 센서의 관측 오차 공분산(Observation Error Covariance)과 예측 공분산(Prediction Covariance)의 합으로 혁신 행렬을 정규화(Normalization)하여 얻는다. 실무 레벨에서는 주로 카이제곱 검정(Chi-square Test) 수준의 통계적 확률 한계치를 설정한다.

  • 혁신 비(Innovation Ratio) < 1.0: 관측된 센서 데이터가 수학적 예측 범위(신뢰 구간) 내에 잘 들어맞음을 의미하며, 융합에 정상적으로 반영된다.
  • 혁신 비(Innovation Ratio) >= 1.0: GPS 다중 경로(Multipath), 급격한 자기장 교란, 센서 하드웨어 글리치 등으로 인해 시스템 모델의 허용 범위를 벗어난 이상치(Outlier)로 통계적 판별(Rejection)을 내리고, 상태 추정 벡터 계산에서 일시적으로 배제한다.

2. EKF2 상태 유효성(State Validity) 플래그와 종속성(Dependency) 매핑

이노베이션 비가 지속적으로 허용치(Threshold)를 초과하거나, 특정 센서의 타임아웃이 발생하면 EKF2 모듈은 더 이상 해당 자유도(DOF, Degree of Freedom)에 대한 위치나 속도를 보장할 수 없다고 스스로 판별한다. 결과적으로 EKF2는 vehicle_local_position.msg 및 vehicle_global_position.msg uORB 토픽에 다음과 같은 상태 유효성 플래그를 실시간 브로드캐스팅(Broadcasting)한다.

  • xy_valid: 2차원 수평 로컬/글로벌 위치 추정치의 통계적 신뢰성
  • z_valid: 기압계 또는 거리 측정 센서(LiDAR) 융합 기반 고도 추정치 신뢰성
  • v_xy_valid / v_z_valid: 3차원 속도 벡터 추정치 신뢰성
  • heading_good_for_control: 지자기 센서 또는 VIO, 듀얼 안테나 RTK 기반 요(Yaw) 절대 각도 추정치 편차 안정성

커맨더(Commander.cpp) 모듈의 SetNavState 파이프라인 및 mode_requirements 테이블은 대상 비행 모드가 요구하는 속성(Attributes)과 위 플래그들을 **하드 록(Hard Lock) 형태의 종속성(Dependency)**으로 묶어 놓았다.

예를 들어, GCS에서 MAVLink MAV_CMD_DO_SET_MODE를 통해 Position 모드로의 전환을 구동하면, Commander는 비트 마스킹(Bit Masking)을 통해 xy_valid와 z_valid가 모두 True 상태인지를 직관적으로 확인하며, 하나라도 False인 상태라면 상태 천이를 반려(MAV_RESULT_DENIED)한다.

3. Ardupilot 대비 PX4의 종속성 체계 강건성(Robustness) 철학

PX4와 Ardupilot(EKF3 기반) 모두 칼만 필터 잔차를 모니터링하지만, 이 오차를 상위 비행 모드에 강제 종속시키는 철학에서 거대한 아키텍처적 차이가 두드러진다.

  • Ardupilot: Ardupilot은 역사적으로 공기역학적 복원력이 강한 고정익 비행이나 장거리 비행의 생존성을 극대화하기 위하여, GPS 등 단일 센서의 혁신율이 튀더라도 관성항법(INS) 데드 레코닝(Dead Reckoning) 여유 범위를 비교적 넓게 가져간다. 비상 시 조종자(Human Pilot)가 즉각 수동 조작으로 보정할 수 있다는 자유도를 상당 부분 위임하는 사상이 녹아있다.
  • PX4-Autopilot: PX4는 상태 공간 제어(State Space Control)의 “관측 불가한 상태에서는 제어하지 않는다“는 원론적 통제주의에 극히 충실하다. EKF2의 각 차원별 유효성 플래그가 하나라도 깨진다면(예: xy_valid = False), 커맨더 모듈은 모드 전환을 원천 거부하는 것 뿐만 아니라, 이미 Position이나 Mission 자동 비행 중이더라도 통계적 유효성 상실을 즉시 감지하여 안전 모드인 Altitude 모드나 Hold 모드로 **자동 강제 천이(Fallback Transition)**를 비가역적으로 단행한다.

4. GCS (QGroundControl / MAVSDK) 관제망 설계 가이드

이러한 PX4의 절대적인 통계적 종속성 기반 파이프라인을 다루기 위해, ROS2(uXRCE-DDS) 자율 비행 노드 개발자나 독자 엔진의 GCS 애플리케이션 아키텍트는 방어적 상태 동기화 기법을 도입해야 한다.

  1. 상태 폴링(Polling) 선행: 비행 모드 변경(특히 이륙, 미션 시작, 복귀 등) 명령 송신 전에, MAVLink의 ESTIMATOR_STATUS 속 통계적 상태 플래그(flags) 패킷을 먼저 수신 및 파싱하여 융합 유효성을 논리 게이트 수준에서 확인해야 한다.
  2. 트랜잭션 피드백루프: GCS 사용자 인터페이스에 ‘GPS 3D Fix’ 같은 단순 하드웨어 정보를 넘어, “EKF 수평 위치 신뢰성 판정 완료” 형태의 내부 소프트웨어적 필터 상태 표시등(Indicator)을 구현해야 한다.
  3. 거부 방어전략(Rejection Handling): 명령 송신 후 COMMAND_ACK에서 MAV_RESULT_TEMPORARILY_REJECTED나 MAV_RESULT_DENIED가 반환될 경우, 이것이 통신 오류가 아닌 “PX4 커맨더가 EKF2의 통계값이 안정화될 때까지 스스로를 보호하는 정상적인 강건 검사 행위“임을 인지하고, 재전송 타이머(Backoff)나 상태 진단 로직으로 진입하도록 FSM(Finite State Machine)을 설계해야 한다.
