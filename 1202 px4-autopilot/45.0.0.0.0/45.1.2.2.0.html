45.1.2.2. 통신 채널의 비동기성(Asynchrony)에 따른 지연-미분 방정식(Delay-Differential Equation)적 상태 불일치(State Mismatch) 발산 해석

무인 항공기(UAV) 운영 체계에서 기체 내부의 비행 제어기(Flight Controller, FC)와 외부의 지상 관제 시스템(Ground Control Station, GCS) 사이에는 무선 데이터 링크를 통한 비동기적(Asynchronous) 통신 채널이 필연적으로 존재한다. 통신 매체의 대역폭(Bandwidth) 한계와 물리적 매질에 의한 전송 지연(Latency)은 GCS가 인식하는 기체의 모델 상태와 실제 기체의 상태 간에 불일치(State Mismatch)를 유발한다. 본 절에서는 이러한 비동기적 시변 지연(Time-Varying Delay)이 관제 시스템의 안정성에 어떻게 악영향을 미치며 오차를 발산(Divergence)시키는지 수학적인 지연-미분 방정식(Delay-Differential Equation, DDE) 모델을 통해 엄밀히 해석한다.

1. 비동기 통신 채널과 상태 오차의 정의

관제 시스템(QGroundControl)의 역할 중 하나는 MAVLink 텔레메트리(예: GLOBAL_POSITION_INT, ATTITUDE, LOCAL_POSITION_NED)를 수신하여 디스플레이 및 고위도(High-level) 자율 제어 모듈에 현재 기체의 상(Image)을 실시간으로 갱신하는 것이다.

t 시점에서 실제 PX4 배장 시스템(EKF2 추정기 기반)이 유지하고 있는 실제 상태 벡터를 x_{FC}(t) 라 하고, 통신 지연 \tau(t) 후에 GCS에 도달하여 QGC 내부 객체(Vehicle Object)에 업데이트된 인식 상태를 x_{GCS}(t) 라 하자. 통신 채널의 본질적 비동기성으로 인해 GCS가 보유한 상태는 과거 기체의 상태가 된다.
x_{GCS}(t) = x_{FC}(t - \tau(t))

상태 불일치 오차(State Mismatch Error) e(t) 는 다음과 같이 정의된다.
e(t) = x_{FC}(t) - x_{GCS}(t) = x_{FC}(t) - x_{FC}(t - \tau(t))

2. 지연-미분 방정식(DDE)적 오차 동역학 발산 해석

드론의 평행 이동과 같은 연속적인 변화를 선형 상미분방정식(Ordinary Differential Equation, ODE)로 단순화할 때, 기체의 동역학은 \dot{x}_{FC}(t) = A x_{FC}(t) + B u(t) 로 표현된다. 여기서 제어 입력 u(t)를 GCS가 생성하는 원격 명령(예: Offboard 모드의 속도 제어) 기반의 페루프(Closed-loop) 제어로 상정해보자.
GCS는 자신의 낡은 인지 상태 x_{GCS}(t) 에 기반하여 제어 오차를 계산하고 제어 입력 u_{GCS}(t) = -K x_{GCS}(t) 를 생성하여 송출한다. 이 제어 명령 또한 \tau_2(t) 의 지연을 겪고 FC에 도달한다. 총 지연 \tau_{total} = \tau + \tau_2 로 가정할 때 기체의 동역학은 전형적인 **지연-미분 방정식(DDE)**의 형태를 취하게 된다.

\dot{x}_{FC}(t) = A x_{FC}(t) - B K x_{FC}(t - \tau_{total})

일반적인 ODE가 초기 상태(Initial State)에 의존하는 반면, DDE 시스템 행태는 구간 초기 함수(Initial History Function)에 의존하며 차수가 무한차원에 달하게 된다.

이 DDE 식의 특성 방정식(Characteristic Equation)은 지연 상수 \tau 가 결합된 초월 방정식(Transcendental Equation)인 \det(sI - A + B K e^{-\tau s}) = 0 으로 도출된다.
\tau_{total}이 특정 임계값(Critical Delay Margin)을 초과하게 되면, 폐루프 시스템의 특성근(Pole) 중 일부가 복소 평면(Complex Plane)의 우반면(Right-Half Plane)으로 이동하게 된다. 이 시점이 지나면서 시스템의 오차 e(t) 는 **진동하며 지수적으로 발산(Oscillatory Exponential Divergence)**하는 거동을 보이게 된다. 사용자는 이 단계에서 QGC UI 상 기체의 좌표가 앞뒤로 널뛰기 하거나 방향성을 상실하는 ‘고무줄 효과(Rubber-banding)’ 현상을 목격하게 된다.

3. PX4 및 QGC의 통신 비동기성에 대한 대처 현황

Ardupilot이 비교적 중앙집중형 루프(Main Loop)에 가깝게 하드웨어를 다루었던 과거 아키텍처와 대조적으로, PX4는 uORB(Micro Object Request Broker) 기반의 극단적인 비동기적 발간/구독(Publish/Subscribe) 미들웨어 패턴으로 작성되었다. 따라서 내부 지연은 마이크로초(us) 단위로 관리된다.

하지만 이는 기체 내부의 문제일 뿐, 외부(GCS, 보조 컴퓨터 등)와의 소통 창구인 mavlink_receiver 및 mavlink_messages 객체의 송출 주기는 UART 또는 UDP의 대역폭에 강제적으로 제한받는다. 네트워크 대역폭 포화(Congestion)가 발생했을 때 MAVLink 노드는 중요도가 낮은 텔레메트리 패킷을 폐기(Drop)하거나 지연 전송을 유발하여 \tau(t) 가 불규칙한 시변 지연(Time-Varying Delay) 특성을 지니게 만든다.

이러한 \tau(t)의 변화율 \dot{\tau}(t) 이 1에 가까워지면(패킷이 도착하지 않고 밀리는 현상 등) DDE 모델 상 국소적 불안정성은 극대화되고 제어 루프의 발산을 치유 불능 상태로 몰아넣는다.

5. 지연 보상(Delay Compensation) 및 해결 방법론

GCS 아키텍처나 자율 군집 관제 프로그램을 설계할 때, DDE적 발산을 차단하기 위해 원격 관제 소프트웨어 층에 지연 보상(Delay Compensation) 기법을 반드시 도입해야 한다.

  1. GCS 내장형 스미스 예측기(Smith Predictor):
    통신 지연 \tau 의 모델을 GCS 내부의 예측 루프에 포함시킨다. GCS는 FC가 보내오는 과거의 상태 x_{FC}(t-\tau) 와 GCS가 \tau 이전에 내린 제어 명령의 예상 결과를 비교하는 형태로 폐루프 분모의 e^{-\tau s} 항을 제거하여 안정성을 확보한다.
  2. 동적 예측 필터 및 외삽 파생(Extrapolation):
    QGroundControl은 지도가 끊기지 않도록 기체의 속도 벡터(vx, vy, vz)를 함께 전달받는다. 수신 지연이 발생한 동안 (마지막 텔레메트리 타임스탬프 기준) 이 속도 백터를 적분하여 GCS 클라이언트 상에서 기체의 현재 체공 지점을 근사 추정(Dead Reckoning)하는 상태 관측기 보완 로직이 포함되어야 한다. 이를 통해 UI 상의 불일치 시각적 저하 방지와 더불어 외부 ROS2 자율 제어 노드의 데이터 튀어오름 현상(Jitter)을 방지할 수 있다.

6. 결론

PX4와 GCS 간의 관제 링크에서 필연적인 통신 데이터의 불연속성과 지연은 곧 시스템의 상태를 지연-미분 방정식(DDE)의 지배를 받는 거동으로 재편한다. 이를 무시한 순진한 피드백 루프는 불안정성에 의한 발산과 사고(Crash)를 초래한다. 현대의 드론 관제 인터페이스 설계자는 텔레메트리 지연의 수학적 리스크를 명확히 인지하고, 소프트웨어 단의 예측/보간 필터링 기법들을 적극 적용해 안정 마진(Stability Margin)을 확보해야 한다.
