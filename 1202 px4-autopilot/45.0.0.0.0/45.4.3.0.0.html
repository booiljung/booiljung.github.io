45.4.3. 다중화(Redundancy)된 외부 제어 인터페이스(RC, 텔레메트리) 간의 제어 권한 중재(Arbitration) 및 선점형 스케줄링(Preemptive Scheduling) 우선순위 로직

1. 개요 및 이론적 배경

안전한 무인항공기 운용을 위해서는 물리적 조종기(RC, Radio Controller)와 지상 관제 시스템(GCS) 또는 자율 에이전트 간의 텔레메트리(Telemetry, MAVLink/ROS2)라는 다중화(Redundancy)된 제어 인터페이스 구성이 필수적이다. 만약 조종사와 컴퓨터 시스템이 동시에 상반된 기동 명령을 내린다면, 기체 내부에서는 둘 중 누구의 명령을 채택할지 결정하는 규칙이 필요하다.

이러한 충돌 문제를 해결하기 위해 PX4-Autopilot의 커맨더(Commander) 아키텍처는 제어 권한 중재(Arbitration) 로직과 특정한 긴급 이벤트가 기존의 임무 수행을 즉시 중단시키고 제어권을 빼앗아오는 선점형 스케줄링(Preemptive Scheduling) 기법을 채택하고 있다. 본 절에서는 이 권한 계층 구조의 원리와 설계 구현을 소스 코드 수준에서 상세히 고찰한다.

2. 인터페이스 데이터 스트림 구조

PX4는 외부의 제어 입력 소스를 크게 물리적 조종기 기반의 manual_control_setpoint 토픽과 MAVLink를 통해 수신되는 vehicle_command 및 경로/자세 타겟 토픽으로 분리하여 처리한다.

  1. RC 인터페이스: 수신기(SBUS, CRSF 등)를 통해 하드웨어 드라이버 단에서 디코딩(Decoding)된 후, 일관된 스케일을 거쳐 -1.0 ~ 1.0 정규화 값을 지니는 수동 제어값으로 미들웨어에 퍼블리싱된다.
  2. 텔레메트리 제어 명령 (MAVLink): GCS의 자동 비행 지시 혹은 모드 전환 명령 등은 직렬 통신망을 지나 미들웨어 상의 명령 큐(Command Queue)에 들어간다.
  3. 오프보드 제어 (ROS2): Companion Computer에서 발생시키는 실시간 제어명령(Setpoint)은 50Hz 이상의 빈도로 시스템에 직접 퍼블리싱된다.

이 서로 다른 비동기적 스트림은 모두 제어 모듈 계층으로 모이며, “누가 현재 기체를 스티어링(Steering) 할 것인가“를 선택하는 멀티플렉서(Multiplexer) 단계를 거친다.

3. 제어 권한 중재(Arbitration) 우선순위 아키텍처

PX4 커맨더 내부 로직은 명확하게 지정된 우선순위 트리(Tree)에 따라 활성화된 제어 권한을 판별한다. 시스템의 근본적인 명령 수용 척도는 다음과 같은 하드 리얼타임 선점형(Preemptive) 계층을 지닌다.

  1. 최우선: 하드웨어 Failsafe 및 시스템 비상 트리거 (배터리 고갈, 센서 발산 등으로 인한 강제 RTL, Land 또는 Flight Termination)
  2. 2순위: 사용자 RC 오버라이드(RC Override) (조종자에 의한 물리적 스틱 개입)
  3. 3순위: GCS MAVLink 명령 (명시적 자동 비행 전환, Return 명령 등)
  4. 4순위: ROS2/MAVSDK 오프보드 제어(Offboard Control) (가장 하위에 속하여 GCS/RC가 언제든 권한을 회수할 수 있음)

이와 같은 설계는 자율 에이전트에 제어 결함이 발생했을 때 인간 조종사가 텔레메트리 명령을 무시하고 기체를 즉시 수동으로 포획(Take-over)할 수 있도록 물리적 킬 스위치(Kill Switch)의 역할을 보장하기 위함이다.

4. 선점형 스케줄링 및 RC 오버라이드(Override) 메커니즘

모드 천이 과정에서 선점형 시스템 동작을 지배하는 가장 핵심적인 파라미터는 COM_RC_OVERRIDE이다.
기체가 Mission(자동 임무 비행)이나 Offboard(ROS2 자율 비행) 모드에서 항로를 추종 중일 때, 조종사가 RC 스틱을 특정 임계치 이상으로 튕기게 되면 커맨더 모듈 내의 오버라이드 탐지 쓰레드가 깨어나 액추에이터 제어 권한을 즉시 선점한다.

4.1 선점(Preemption) 발생 조건 추적

// PX4 Commander (State Machine) 동작 유사 코드
bool is_stick_deflected_beyond_threshold(manual_control_setpoint_s &rc) {
    float threshold = get_parameter("COM_RC_STICK_OV");
    return (fabsf(rc.y) > threshold || fabsf(rc.x) > threshold || fabsf(rc.z - 0.5f) > threshold);
}

if (current_mode == MISSION || current_mode == OFFBOARD) {
    if (get_parameter("COM_RC_OVERRIDE") == ENABLED && is_stick_deflected_beyond_threshold(rc_input)) {
        // 기존 자동 비행 상태 스케줄러 선점 및 강제 폐기
        abort_current_mode();
        // 위치 제어(Position) 또는 고도 제어(Altitude) 모드로 긴급 천이
        request_mode_switch(POSITION_MODE);
        notify_gcs_of_override();
    }
}

위 로직에 따라, 오프보드나 MAVLink 웨이포인트 알고리즘이 아무리 높은 빈도로 연산을 수행하더라도, 커맨더 계층에서 이들의 명령 패킷은 버려지고 RC 스틱 매핑 테이블로 즉각 권한이 중재(Arbitration) 전이된다.

5. 핵심 파라미터 구성

  • COM_RC_IN_MODE: RC 수신기를 필수로 사용할지 혹은 조이스틱이나 텔레메트리만으로 비행할지 정의한다. 완전 자율 기체에서는 이 값을 1(RC Optional) 또는 2(Joystick/No RC)로 설정하여 예외적인 Failsafe를 우회할 수 있다.
  • COM_RC_OVERRIDE: (0: 비활성, 1: 활성) 활성화될 경우 자동 비행 중 스틱 개입 시 모드가 Position 모드로 떨어진다.
  • COM_RC_STICK_OV: RC 오버라이드가 반응할 데드존(비율값) 임계치. 스틱을 기본값인 10% 이상 움직여야만 선점 모듈이 발동하도록 하여, 미세한 노이즈로 인한 자율 비행 해제를 방지한다.

6. Ardupilot 대비 관제 및 권한 중재 차이

Ardupilot은 전통적으로 MAVLink의 RC_OVERRIDE_CHANNELS 메시지를 매우 빈번히 활용하도록 설계되었다. 즉, GCS 또는 Companion Computer가 조종기 스틱 신호를 흉내 내어 MAVLink로 시뮬레이션 매핑(Virtual RC)하는 방식이 뿌리 깊게 자리 잡고 있다. 이 때문에 진짜 물리 RC와의 충돌 중재를 위해 MAVLink 텔레메트리 측에 타임아웃 팩터(Timeout Factor)를 복잡하게 설정해야 했다.

반면, PX4-Autopilot은 아키텍처 철학상 미들웨어에서 manual_control_setpoint와 자율 비행 제어기(trajectory_setpoint)의 논리적 책임을 명확하게 분리한다. PX4는 “GCS나 ROS2가 기체를 제어하려면 가상 RC 프로토콜을 쓰지 말고 명시적인 Offboard 모드나 MAVSDK Position 제어 명령을 사용하라“는 스탠스를 취한다. 그 결과 권한 중재기가 물리적(RC) / 논리적(MAVLink) 소스를 확실하게 판별할 수 있어, 상태 천이 모델(State Transition Model)의 투명성과 안전성이 한층 강화된 형태를 띤다.

7. ROS2 와의 연동 및 타임아웃(Timeout) 관제

드론과 ROS2가 통신하는 환경에서 에이전트 측의 텔레메트리 링크 단절 혹은 프로세스 정지는 기체 관점에선 심각한 위협이다.

ROS2(또는 MAVSDK) 기반 오프보드 제어 시, 노드는 반드시 OffboardControlMode 메시지와 TrajectorySetpoint를 초당 2회(2Hz) 이상의 빈도로 핑(Ping)과 함께 전송해야 한다.
만약 에이전트의 연산 과부하 스케줄링으로 인해 이 주기(Heartbeat)가 중단되면, 커맨더 측의 타임아웃(Timeout) 트래킹 구조체가 이를 즉각 감지하고, MAVLink 우선순위 리스트에서 ROS2 컨트롤러의 권한을 즉결 폐기(Preemptive Kill)한다. 그리고 사전 정의된 COM_OBL_RC_ACT 파라미터에 따라, RC 생존 시 Position 모드로 권한을 위임하거나, 그렇지 않다면 Failsafe(RTL/Land)를 강제 발동시켜 오작동 중인 관제 에이전트로부터 물리 계층을 원천 분리한다.
