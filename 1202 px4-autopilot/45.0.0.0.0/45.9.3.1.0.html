45.9.3.1. 연판정(Soft-decision) 및 경판정(Hard-decision) 로직 융합을 통한 고신뢰성 모드 전환 전이 스크립팅 설계 패턴(Design Pattern)

PX4-Autopilot 기반의 지상 관제 시스템(GCS) 또는 자율 에이전트를 개발할 때, 비행 모드(Flight Mode)의 전환은 비행 제어기 내부의 상태 머신 로직과 제어 권한을 송두리째 뒤바꾸는 매우 중요한 행위이다. 특히, 위치 제어(Position Control) 기반의 수동 비행에서 오프보드(Offboard)와 같은 완전 자율 비행 모드로 전이(Transition)할 때, 단순한 양방향 통신 응답에만 의존하는 것은 극심한 무선 간섭이나 센서 외란 환경에서 치명적인 시스템 오작동을 초래할 수 있다.

본 절에서는 통신 공학과 제어 이론에서 차용한 **경판정(Hard-decision)**과 연판정(Soft-decision) 개념을 GCS 모드 전환 관제 스크립팅에 융합하여, 불확실성이 내재된 운영 환경에서도 극한의 신뢰성을 이끌어내는 고도화된 설계 패턴(Design Pattern)을 심도 있게 분석한다.

1. 경판정(Hard-decision) 로직: MAVLink 명시적 응답 기반 검증

경판정(Hard-decision)은 통신 수신기가 신호를 복조할 때 0 또는 1이라는 확정적(Deterministic) 결론을 내리는 방식을 뜻한다. 관제 소프트웨어 파이프라인에서 경판정 논리란, MAVLink 프로토콜의 명시적 응답 패킷만을 스위치(Boolean)의 기준으로 삼는 검증 로직을 말한다.

  • 작동 원리: GCS가 모드 전환 명령어(MAV_CMD_DO_SET_MODE)를 발행하면, 드론의 커맨더(Commander) 모듈은 전환 가능 여부를 판단한 뒤 COMMAND_ACK 패킷을 반환한다. GCS는 반환된 MAV_RESULT 값이 수락(ACCEPTED)인지 거부(DENIED)인지에 따라 전이 성공 여부를 단정 짓는다.
  • 장점과 한계: 직관적이고 구현이 단순하며, 소프트웨어 아키텍처 상의 상태 천이를 즉각적으로 동기화할 수 있다. 그러나 타임아웃이나 패킷 손실이 발생하면 거짓 음성(False Negative)이 도출될 수 있으며, 반대로 기체가 명령을 수락했더라도 진동(Vibration)이나 외풍으로 인해 추정기(EKF2)의 특정 상태가 찰나의 불안정한 구간에 놓여있는지를 GCS가 인지하지 못하는 거짓 양성(False Positive)의 맹점을 가진다.

2. 연판정(Soft-decision) 로직: 다중 텔레메트리 융합 및 신뢰도(Confidence) 산출

연판정(Soft-decision)은 0과 1 사이의 연속적인 확률 분포나 신뢰도(Confidence Level) 구간을 산출하여 보다 유연하고 종합적인 판단을 내리는 로직이다. 모드 전이 파이프라인에 이를 대입하면, 단순 ACK 판별을 넘어 기체가 쏘아 보내는 다차원의 텔레메트리 스트림의 시계열 추세를 합의(Consensus)하는 과정이 된다.

  • 작동 원리: 모드 전환 전후의 윈도우(Time Window) 동안 VIO/GPS 이노베이션(Innovation), 칼만 필터(EKF2)의 분산(Covariance), 옵티컬 플로우(Optical Flow)의 퀄리티(Quality) 등 다수의 상태 토픽을 종합하여 비행 안정성 스코어(Stability Score)를 산출한다.
  • 예시: 특정 센서(예: GPS HDOP) 값이 임계치를 미세하게 넘어 경판정 기준에서는 ’경고’를 발생시킬 수 있지만, 연판정 로직에서는 하방 비전 센서나 Lidar 고도 데이터의 강력한 신뢰도를 가중 융합(Weighted Fusion)함으로써, 안전 반경 이내라면 오프보드(Offboard) 모드 진입을 ’연성 승인(Soft Approve)’하는 체계를 설계할 수 있다.

3. 로직 융합형 고신뢰성 모드 전환 전이 아키텍처

경판정과 연판정을 융합(Fusion)하면 상호 보완적인 고신뢰성(Robust) 스크립팅 파이프라인을 구축할 수 있다. 아래 Mermaid 다이어그램은 두 로직이 결합된 이중 검증 전이 설계 패턴(Dual-Verification Transition Design Pattern)을 묘사한다.

stateDiagram-v2
    [*] --> Request_Mode_Change
    
    state Request_Mode_Change {
        [*] --> Send_SetMode_Command
    }
    Request_Mode_Change --> Hard_Decision_Evaluation
    
    state Hard_Decision_Evaluation {
        [*] --> Await_COMMAND_ACK
        Await_COMMAND_ACK --> ACK_Received : MAV_RESULT_ACCEPTED
        Await_COMMAND_ACK --> Timeout : Retries Exceeded
    }
    
    Hard_Decision_Evaluation --> Fallback_Recovery : Timeout / DENIED
    Hard_Decision_Evaluation --> Soft_Decision_Evaluation : ACK_Received
    
    state Soft_Decision_Evaluation {
        [*] --> Observe_Telemetry_Window
        note left of Observe_Telemetry_Window
            Collect EKF Status, Position 
            Error, Vibration Metrics 
            for T seconds.
        end note
        Observe_Telemetry_Window --> Calculate_Confidence_Score
        Calculate_Confidence_Score --> Converged : Score > Threshold
        Calculate_Confidence_Score --> Diverged : Trend unstable
    }
    
    Soft_Decision_Evaluation --> Fallback_Recovery : Diverged (Abort Transition)
    Soft_Decision_Evaluation --> Transition_Complete : Converged
    Transition_Complete --> [*]
  1. GCS의 비동기 워커가 전환 명령을 전송한다.
  2. 1차 관문으로 경판정 평가 엔진이 동작하여 COMMAND_ACK 및 타임아웃을 판별한다.
  3. ACK를 받았더라도 즉시 성공을 선언하지 않고, 연판정 평가 엔진으로 진입하여 약 0.5초 ~ 1.5초 간의 Observation Window 동안 위치 오차 및 속도 제어 추종성(Tracking Performance) 텔레메트리가 수렴(Converge)하는지 관측한다.
  4. 모든 점수가 임계치 이상일 때 최종적으로 Transition_Complete 상태를 선언한다.

4. 실무 적용: PX4 Offboard 진입 시의 로직 융합 응용

수동 제어 모드에서 자율 에이전트에 의한 오프보드(Offboard) 모드로 전환할 때는 사전에 주파수 2Hz 이상의 기하학적 목표점(Setpoint)이 스트리밍되고 있어야만 PX4 커맨더가 진입을 허용한다.

  • MAKVSDK(C++)를 활용할 때, 단순히 offboard.start()를 날려 Future로 대기하는 것은 전형적인 경판정(Hard-decision) 로직이다.
  • 고신뢰성 GCS에서는 offboard.start() 성공 반환 이후에도, 텔레메트리의 position_velocity_ned 토픽 가속도 변화율(Jerk)이나 수평 위치 예측 오차를 분석하여 기준선을 이탈하는 스파이크 지표가 발견될 시 즉시 action.hold()를 트리거하여 호버링(Hovering) 모드로 롤백하는 연판정 기반 강건(Robust) 예외 처리 로직을 함께 스크립팅한다.

5. ArduPilot과의 관제 구조 설계 차이 비교

ArduPilot 관제 생태계에서는 모드 속성(Guided Mode 등)을 변경할 때 GCS가 MAVLink SET_MODE 패킷을 보내고, 드론킷(DroneKit)은 단순히 해당 속성(Attribute) 값이 변경될 때까지 루프를 도는 단편적인 경판정 접근 방식을 주로 취해 왔다.

반면, PX4 시스템은 uORB 미들웨어를 거치는 내부 상태 머신 구성이 매우 엄격하며 멀티콥터 센서 융합 최적화가 강력하게 결합되어 있다. 따라서 PX4용 최신 GCS 아키텍처(MAVSDK 기반)는 GCS로 하여금 수동적인 단편 대기 메커니즘을 탈피해 능동적으로 다차원 결측치(Outlier)를 검열하는 연판정 필터 관측기(Soft-decision Observer)를 애플리케이션 계층(Application Layer)에 직접 구축하도록 유도하는 철학적 차이를 보인다.

6. 결론

“경판정 및 연판정 융합 파이프라인 패턴“은 단일 응답의 속도와 다중 텔레메트리 관측의 안정성을 유기적으로 결합한 관제 소프트웨어 아키텍처 설계의 정수(精髓)다. 이러한 복합 판별 설계 패턴을 접목함으로써 개발자는 극심한 센서 노이즈와 네트워크의 지연 속에서도 무인기에 치명적인 이상 동작(Erratic behavior)을 유발하지 않는, 방산업계나 도심항공교통(UAM) 수준의 고신뢰성 모드 전환 전이 스크립트를 구현할 수 있다.
