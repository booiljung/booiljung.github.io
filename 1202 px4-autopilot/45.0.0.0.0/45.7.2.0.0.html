45.7.2 패킷 유실(Packet Drop) 확률 모델(Gilbert-Elliott Model)에 기반한 트랜스포트 계층(Transport Layer) 혼잡 제어(Congestion Control)

본 절에서는 무인항공기(UAV)와 지상 관제 시스템(GCS, Ground Control Station) 간의 불안정한 무선 통신 환경에서 신뢰성을 확보하기 위해 적용되는 패킷 유실 확률 모델과 이를 기반으로 한 트랜스포트 계층의 혼잡 제어(Congestion Control) 최적화 방안에 대해 서술한다. 구체적으로는 버스트(Burst) 형태의 에러를 효과적으로 모델링하는 길버트-엘리엇 모델(Gilbert-Elliott Model)을 바탕으로, PX4-Autopilot(펌웨어 버전 v1.14 기준) 및 QGroundControl(버전 v4.3 기준)의 네트워크 신뢰성 관리 기법과, Ardupilot과의 설계 철학 차이를 비교 분석한다.

1. 개요

드론의 텔레메트리(Telemetry) 무선 링크는 기상 변화, 구조물에 의한 다중 경로 페이딩(Multipath Fading), 전자기 간섭(EMI, Electromagnetic Interference) 등 다양한 요인에 의해 지속적으로 변화한다. 이러한 환경에서는 패킷 유실이 독립적으로 발생하기보다는 군집하여(Bursty) 나타나는 경향이 있다.

이러한 군집 에러 특성을 수학적으로 설명하기 위해 널리 사용되는 기법이 마르코프 체인(Markov Chain) 기반의 **길버트-엘리엇 모델(Gilbert-Elliott Model)**이다. 이 모델을 통해 통신 상태가 ’좋은 상태(Good State)’와 ’나쁜 상태(Bad State)’를 오가는 확률 추이를 계산할 수 있으며, PX4 및 QGC의 MAVLink 통신 스택(Stack)은 이러한 통계적 특성을 간접적으로 반영하여 재전송 타임아웃(RTO, Retransmission Timeout)과 전송률(Transmission Rate)을 동적으로 조절한다.

2. Gilbert-Elliott 모델 기반 통신 링크 상태 추정

길버트-엘리엇 모델은 통신 채널을 두 가지 지연 상태(State)로 단순화하여 채널의 오류율을 예측한다.

  • Good State (G): 비트 에러율(BER, Bit Error Rate)이 매우 낮거나 0에 수렴하여 패킷 유실 확률이 극히 희박한 상태.
  • Bad State (B): 신호 대 잡음비(SNR, Signal-to-Noise Ratio)가 급격히 저하되어 높은 에러율을 보이며, 연속적인 패킷 유실이 발생하는 상태.

위의 상태천이도(State Transition Diagram)에서 p는 G 상태에서 B 상태로 전이될 확률(에러 발생 진입)을 의미하며, r은 B 상태에서 G 상태로 회복될 확률을 나타낸다. 트랜스포트 계층에서는 수신 상태의 MAVLink 패킷 시퀀스 번호(Sequence Number)를 통해 현재의 채널 상태 체류 시간을 추정하고, 나쁜 상태가 지속될 것으로 예측될 경우 선제적으로 송신량을 줄임으로써 혼잡(Congestion) 상태의 악화를 방지한다. 참고 논문인 “A Markov Model for Block Errors on Fading Channels“에 이러한 모델의 근본적인 수학적 증명이 잘 전개되어 있다.

3. PX4-Autopilot의 MAVLink 기반 트랜스포트 혼잡 제어

PX4-Autopilot은 원격 제어 및 텔레메트리 데이터 전송에 주로 UDP 통신과 MAVLink 프로토콜을 사용한다. 기본적으로 UDP 자체는 신뢰성을 보장하지 않으므로, MAVLink 프로토콜의 라우팅 레이어와 커맨드 재전송 규칙을 통해 트랜스포트 계층의 신뢰성을 확보한다.

  1. 대역폭 할당량(Bandwidth Allocation) 동적 조절
    PX4의 텔레메트리 모듈(mavlink 모듈)은 현재 설정된 직렬 포트(Serial Port)의 데이터 전송률(Baudrate)과 MAVLink 패킷 큐(Queue)의 포화 상태를 모니터링한다. 무선 링크의 패킷 유실로 인해 GCS로부터 송신 성공 응답인 ACK가 지연되면(Gilbert-Elliott 모델의 Bad State 진입), PX4의 MAVLink 스트리머는 하위 우선순위 토픽(예: 배터리 상태, 추가 센서 데이터 등)의 전송 주기를 일시적으로 줄인다. 반면 자세 제어 및 위치 정보와 같은 고우선순위 MAVLink 메시지(예: ATTITUDE, GLOBAL_POSITION_INT)의 전송에 대역폭을 우선 할당하는 혼잡 제어(Congestion Control) 셰이핑(Shaping)을 수행한다.

  2. 명령어 스트림의 버퍼링 및 드롭 처리
    무선 링크의 병목(Bottleneck) 현상으로 버퍼가 가득 찰 경우, 버퍼 블로트(Bufferbloat) 현상을 억제하기 위해 가장 오래된 데이터를 버리는 꼬리 버림(Tail Drop) 방식이 적용된다. 단, 반드시 도착해야 하는 제어 명령어(COMMAND_LONG 등)의 경우 큐에서 소실되지 않도록 별도의 트랜잭션 버퍼를 활용한다.

4. QGroundControl (GCS) 측의 재전송 제어 전략

QGroundControl(QGC)은 GCS 관점에서 기체로 명령(Command)을 전송할 때 발생할 수 있는 패킷 유실에 대비한다.

  • 동적 재전송 타이머(Dynamic Retransmission Timer): QGC는 COMMAND_LONG 혹은 파라미터 요청(Parameter Request) 메시지를 보낸 후 기체로부터 COMMAND_ACK를 기다린다. Gilbert-Elliott 모델에서 B 상태의 평균 체류 길이(Average Burst Length)는 1/r로 수렴한다. QGC의 차량 통신 관리 모듈(Vehicle 클래스 산하)은 일정한 고정 시간(예: 500ms)이 경과하면 재전송을 시행하나, 지속적으로 패킷이 유실되는 구간에서는 과도한 재전송이 도리어 채널 혼잡을 가중할 수 있으므로 재전송 횟수에 임계치(예: 최대 3~5회)를 설정하여 무분별한 요청 폭주를 차단한다.
  • 혼잡 시 사용자 인터페이스(UI) 피드백: QGC는 통신 유실(Comm Loss) 상태를 감지하면 UI 상에 경고 창을 띄워 운영자가 인지하고 대비할 수 있도록 돕는다. 이는 단순한 프로토콜 오류 제어를 넘어 운영자의 대처 시간을 벌어주는 인간-기계 상호작용(HMI) 관점에서의 혼잡 제어이기도 하다.

5. Ardupilot과 PX4-Autopilot의 혼잡 제어 철학 비교

PX4-Autopilot과 Ardupilot은 동일하게 MAVLink를 채택하고 있으나, 패킷 유실과 네트워크 혼잡을 대하는 철학적 차이가 존재한다.

PX4는 실시간성과 ROS 2 등 오프보드(Offboard) 보조 컴퓨터와의 지연 없는 연동을 중시하기 때문에, 무선 구간이 Bad State에 진입하여 혼잡에 빠졌을 때 과거의 상태 정보보다 지연 없는 ’현재 상태’의 최신화(Freshness)를 가장 최우선으로 확보한다.

6. 결론

무인항공기의 제어 링크 환경에서 Gilbert-Elliott 모델 기반의 이론적 이해는 실무적으로 통신 지연 및 패킷 드롭을 극복하는 강건한 시스템을 구축하는 데 필수적이다. PX4-Autopilot과 QGroundControl은 재전송 임계치 설정과 트래픽 우선순위 조절이라는 형태의 트랜스포트 계층 혼잡 제어 기법을 통해, 일시적인 무선 간섭 상황에서도 기체를 안전하게 제어하고 상태를 복원할 수 있는 신뢰성 있는 아키텍처를 구현하고 있다. 향후 네트워크 대역폭 제한이 큰 군집 비행(Swarm Flight) 환경 등에서는 이러한 채널 추정 기반의 적응형 혼잡 제어 알고리즘의 중요성이 더욱 부각될 것이다.
