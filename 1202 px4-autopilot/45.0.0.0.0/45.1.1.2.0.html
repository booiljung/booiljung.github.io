45.1.1.2. 비선형 동역학(Nonlinear Dynamics) 시스템에서의 제어 권한 위임(Handover) 시 리아프노프 안정성(Lyapunov Stability) 보장 조건 및 흡수 영역(Region of Attraction) 해석

무인항공기(UAV)는 로터의 공기역학적 특성과 강체(Rigid Body) 동역학이 결합된 고도의 비선형 동역학(Nonlinear Dynamics) 시스템이다. 서로 다른 제어 모드 간에 권한이 매끄럽게 넘어가는 제어 권한 위임(Handover) 과정은 이론적으로 두 개의 각기 다른 비선형 제어기(Controller) 구성을 교차 전환하는 것과 같다. 이 과정에서 기체가 진동하거나 오버슈트 상태로 빠져 정상 범주로부터 발산(Divergence)하는 것을 방지하기 위해 PX4-Autopilot의 내부 로직은 리아프노프 안정성(Lyapunov Stability) 이론을 만족시키도록 치밀하게 설계되어 있다.

1. 비선형 상태 공간 모델과 제어 권한 위임(Handover)

PX4가 담당하는 쿼드콥터 혹은 고정익 드론의 비선형 상태 공간 방정식은 시간 t에 대하여 일반적으로 \dot{x} = f(x, u)로 표현된다.
기체가 특정 비행 모드 M_1(예: 다이렉트 자세 제어를 수행하는 Stabilized)에서 M_2(예: 폐루프 위치 제어를 수행하는 Position)로 전환할 때, 제어 입력 u(t)를 산출하는 피드백 지배 방정식이 u = k_1(x)에서 u = k_2(x)로 순간 교체된다. 이 절환 제어 논리를 갖는 하이브리드(Hybrid) 모델은 비선형 스위칭 시스템(Switched System)으로 해석되며, 수학적으로 상태 전환 지점 t_s에서 시스템이 일시적 불안정성으로 발산할 치명적인 위험을 내재하고 있다.

제어 권한 위임이 성공적이고 안전하게 이뤄지기 위해서는, 새로운 제어기 k_2(x)가 시스템 모델을 목표 평형 상태(Target Equilibrium State) x^*로 점근적으로 안정하게 이끌어갈 수 있는 위상 영역 내부에서만 전환이 허가되어야 한다. 이 이론적 영역을 흡수 영역(Region of Attraction, ROA)이라고 정의한다.

2. 모드 전환 시 리아프노프 안정성(Lyapunov Stability) 보장 조건

PX4-Autopilot은 GCS로부터 모드 전환(Mode Switch) 명령이 인가되었을 때, 전환 이후 제어 모델의 안정성을 담보하기 위해 커맨더(Commander) 및 내부 Estimator에서 여러 동역학 파라미터를 교차 판별한다. 이 소프트웨어 로직들은 제어 이론상 리아프노프 안정성을 입증하기 위한 경험적이고 해석적인 조건과 밀접히 상응한다.

2.1 다중 리아프노프 함수(Multiple Lyapunov Functions) 접근법

연속적인 두 모드 시스템 M_1과 M_2에 대하여 궤적의 안정성을 분석하는 각각의 리아프노프 후보 에너지 함수(Lyapunov Candidate Function) V_1(x)와 V_2(x)가 존재한다고 가정하자. 비행 안정성(Flight Stability)을 보장하는 주요 수학적 전제는 다음과 같다.

  1. \dot{V}_1(x) < 0 및 \dot{V}_2(x) < 0 이라는 음의 정부호(Negative Definite) 조건을 목표 평형 지점 근방의 궤적에서 공통적으로 만족하여야 한다.
  2. 제어 권한이 M_1에서 M_2로 이양되는 전환 시각 t_s 관점에서 에너지 시스템 함숫값은 증폭되어서는 안 된다. 즉, V_2(x(t_s)) \le V_1(x(t_s))를 성립시켜야만 전환 후에도 제어 텐션(Spike)에너지가 돌연 급증하지 않고 점근적 궤적 수렴 및 흡수를 지속할 수 있다.

실제 오픈소스 코드단에서 PX4는 확장이칼만필터(EKF2) 관측기의 위치/속도 공분산(Covariance) 및 이노베이션(Innovation) 테스트 지표가 이 에너지 함수의 감시 역할을 대신 수행하도록 설계되었다. EKF2의 상태 불확실성이 지정된 임계값(Threshold) 미만 수준으로써 리아프노프 에너지 발산이 여전히 제한 가능 영역에 있다고 확인될 때만, M_2(예: Auto 형 모드)로의 권한 위임(Handover)이 승인된다.

2.2 흡수 영역(Region of Attraction, ROA) 평가 알고리즘

흡수 영역(ROA)은 초기 상태 변수 집합 x(t_s)에서 출발한 기체의 동역학적 궤적이 시간 t \rightarrow \infty를 거치며 목표 상태 x^*로 안착하고 이탈하지 않는(Asymptotically Convergent) 위상 공간들의 부분집합 영역을 말한다.

PX4-Autopilot에서는 기체가 너무 가파른 에폭(High Roll/Pitch Angle)으로 회전해 있거나, 과도한 침하 속도(Overspeed Descent)로 추락하고 있는 경우 상위 Auto 제어 모드로의 진입 이벤트가 Commander 검증 단계에 의해 즉각 거부된다. 이는 직관적으로 현재의 동역학적 상태 x(t)가 **신규 모드 제어기(예: Position Controller 계층)의 흡수 영역 밖(Outside of ROA)**에 놓여 있음을 명시하기 때문이다.
따라서, Commander 데몬 아키텍처는 다음을 수리적으로 먼저 판별한다.

  • 추정된 현재 수직 속도 v_z 가 절대 하강 한계치 \text{Maximum Descent Rate} 조건을 만족하는가.
  • 기체의 자세(Attitude) 각도 경사도가 신규 M_2 제어기 기반의 선형화 가정(Linearization Assumption)을 위배할 정도로 과도하게 틀어지지 않았는가.

4. 결론

무인항공기의 강체 동역학 모델처럼 3차원 공간에서 고도의 비선형성(Nonlinearity)을 내포한 시스템에서 모드 간 제어 권한 위임(Handover)은 심각한 불안정과 추락의 기폭제가 될 수 있다. 논문 *“Region of Attraction Analysis for Unmanned Aerial Vehicles during Mode Transitions”*와 같은 현대 비행 비선형 제어 이론의 잣대에서 볼 때, PX4-Autopilot의 FSM 철학은 단순한 플래그(Flag) 교환 루틴을 넘어서, 리아프노프 안정성을 유지하고 흡수 영역 내부에서의 전환이라는 엄밀한 수학 조건을 보장하기 위한 수리 생태학적 안전 스위치 그 자체이다. 관제 앱 및 ROS 2 미들웨어 체계를 구현하는 시스템 개발자는 이러한 통제 아키텍처의 엄격함을 이해하고, 무리한 권한 위임 오버라이드 명령을 소프트웨어 트랜잭션단에서 억지로 강제하지 않도록 안전하게 설계해야 한다.
