45.6.2.3. 무선 통신 지터(Jitter) 역학을 고려한 지수 가중 이동 평균(Exponentially Weighted Moving Average, EWMA) 기반 동적 검증 윈도우(Verification Window) 할당 기법

1. 개요

무인항공기(UAV)와 지상 관제 시스템(GCS) 사이의 MAVLink 통신망(Telemetry)은 이상적인 유선 네트워크 환경과 달리, 외부 전파 간섭, 안테나 지향성 손실, 그리고 기체와 GCS 간 이격 거리에 따른 다중 경로 페이딩(Multipath Fading) 등에 의해 필연적으로 지터(Jitter) 현상을 수반한다. 지터란 주기적으로 송수신되어야 하는 패킷의 도착 시간 편차(Variation in packet delay)를 의미하며, 이는 기체의 비행 모드나 상태가 정상적으로 인가되었는지 판독하는 타임아웃(Timeout) 판정에 매우 치명적인 노이즈로 작용한다.

만약 QGroundControl(이하 QGC)과 같은 GCS가 비행 모드 전환 확인을 위해 정적(Static)이고 고정된 검증 윈도우(예: 항상 2초 대기)만 사용한다면, 네트워크 상태가 일시적으로 악화되었을 때 명령이 실패한 것으로 오인하는 오탐지(False Positive)가 급증하게 된다. 결과적으로 불필요한 명령 재전송(Retransmission) 스톰(Storm)이 발생하여 제한된 무선 대역폭이 낭비되고, 시스템의 제어 무결성에 치명타를 입힐 수 있다. 본 절에서는 지수 가중 이동 평균(Exponentially Weighted Moving Average, EWMA) 알고리즘을 도입하여, 실시간 무선 지터 역학(Dynamics)을 학습하고 상태 동기화를 위한 **동적 검증 윈도우(Dynamic Verification Window)**를 유연하게 할당하는 최첨단 제어 기법을 살펴본다.

2. 무선 통신 지터(Jitter)의 수학적 모델링

기체(PX4-Autopilot 시스템)는 HEARTBEAT 메시지나 고주파 상태 스트림을 \Delta t_{expected} (예: 1초, 1Hz) 간격으로 송신하도록 스케줄링되어 있으나, GCS 수신단에 도달하는 실제 간격 \Delta t_i는 지연 시간의 연속적인 요동에 지배받는다. 특정 송신 시점 i에서의 지터 단위 성분 J_i는 다음과 같이 정의된다.

J_i = \vert \Delta t_i - \Delta t_{expected} \vert

참고: 편차의 크기를 산정하기 위해 절대값(\vert)이 사용되었다.

PX4 내부의 통신 큐(uORB 기반 MAVLink 송신기) 스케줄링 지연과 공중 물리 매체(Air interface) 지연이 합산되어 발현되며, 이 거시적인 역학계를 안정화시키지 못하면 GCS의 상태 머신 추종기(State Machine Follower)는 심각한 오작동을 유발할 수 있다.

3. EWMA 기반 동적 검증 윈도우 할당 알고리즘

네트워크 혼잡도 예측 모델 구축에 고전적이고 확실한 신뢰성을 지니는 EWMA 기법은 최신 측정치에 통계적 가중치를 높게 부여하는 1차 저주파 통과 필터(IIR Filter)의 형태를 지닌다. GCS는 비행 모드 전환 인가를 대기하는 윈도우 시간(Verification Timeout Window, W)을 고정값으로 두지 않고, 매 MAVLink 패킷 수신 때마다 아래의 점화식을 통해 윈도우의 기준값을 갱신한다.

W_{new} = \max(\alpha \cdot J_{current} + (1 - \alpha) \cdot W_{old}, \,\, W_{min})

  • W_{new}: 새롭게 산출된 동적 검증 윈도우의 타임아웃 경계값
  • W_{old}: 바로 직전까지 사용되었던 윈도우 크기
  • J_{current}: 가장 최근에 감측된 패킷 지터 폭
  • \alpha: 평활 계수(Smoothing Factor)로 0 < \alpha < 1 의 범위이다. 망의 변동성에 얼마나 민감하게 반응할지를 결정한다 (보통 0.125나 0.25 계열 사용).
  • W_{min}: 제어 루프가 최소한으로 유지해야 하는 프로토콜 베이스라인 응답 시간(Baseline RTT).

이 수식을 통해 네트워크의 편차가 커질 경우 W_{new}가 자연스럽게 확장되어, 섣부른 타임아웃 판정을 유예하는 “관용(Tolerance)” 모델을 형성하게 된다. 반대로 무선 링크 품질이 회복되면 윈도우는 신속하게 W_{min}에 수렴하여 빠른 반응성을 되찾는다.

3. 아키텍처 연동 및 Ardupilot 대비 PX4 관제망의 차별성

이러한 동적 검증 윈도우 할당 기법은 특히 MAVSDK나 ROS2(uXRCE-DDS) 미들웨어를 사용하는 PX4 군집 비행(Swarm) 관제 모델에서 큰 차이를 만들어낸다.

  • Ardupilot 및 기존 Mission Planner 모델: 대부분 고정 윈도우 타임아웃(ex: Retry Timer 3000ms 고정) 및 무조건적인 브루트 포스(Brute-force) 형태의 재전송 처리 방식을 선호한다. 이는 근거리 조종에는 안정적이나 LTE, 5G망 등 넓은 지터 분포를 지니는 초장거리 BVLOS(비가시권) 비행 시에는 통신 과부하를 초래한다.
  • PX4 및 고도화된 GCS 통신망 체계: 다중 노드 간의 PING, TIMESYNC 메시지 및 상태 교차 검증 파이프라인과 이 EWMA 윈도우 이론을 결합하여, 채널의 대역폭과 지연 현황을 스스로 학습하고 적응하는 자가 적응형 스로틀링(Self-adaptive Throttling)을 구현하는 데 탁월한 기반을 제공한다.

4. EWMA 변동성 적응 흐름 (Mermaid 도표)

아래 도표는 주기적으로 들어오는 텔레메트리를 통해 GCS가 어떻게 윈도우를 스스로 재평가하고, 모드 변경 시 해당 윈도우 갭을 방어선으로 사용하는지 나타낸다.

graph TD
    A[Telemetry 수신 스레드] --> B[수신 타이밍 측정 및 J_current 산출]
    B --> C{J_current 계산 완료}
    C --> D[EWMA 필터 적용: W_new = α * J + 1-α * W_old]
    D --> E[전역 상태: 동적 검증 윈도우 크기 갱신]
    
    F[사용자: 비행 모드 변경 명령] --> G[명령 전송: MAV_CMD_DO_SET_MODE]
    G --> H[검증 타이머 구동 시작]
    E -.-> |W_new 참조| H
    H --> I{W_new 타임아웃 도달 전 응답(또는 모드 변경 확인) 획득 여부}
    I -- 성공 --> J[명령 인가 완료 통보]
    I -- 시간 초과 --> K[명령 실패 판정 및 재전송 큐 삽입]

타이머(Timer)는 고정 하드코딩된 값이 아니라 백그라운드 스레드에서 끊임없이 산출되는 W_{new} 값을 구독(Subscribe)하므로, 그 어떤 네트워크 환경의 등락에도 제어 유연성을 잃지 않게 설계된다.

5. 결언 및 학술적 평가 방안

무선 통신 지터 역학과 결합된 EWMA 기반 윈도우 스케줄링은 “State Estimation and Adaptive Timeout Models in Unmanned Aerial Systems Networks” 등 다수의 최신 논문에서도 주요하게 언급되는 차세대 신뢰성 보장 모델이다. GCS 개발자는 QGroundControl 소스 코드 내에서 Vehicle::setFlightMode 처리 루틴이나 MAVLink Protocol 의 재전송 타이머 블록에 이와 같은 동적 평가 알고리즘을 추가 구현함으로써, PX4-Autopilot의 구조적 강건성을 가장 이상적인 형태로 격상시킬 수 있을 것이다.
