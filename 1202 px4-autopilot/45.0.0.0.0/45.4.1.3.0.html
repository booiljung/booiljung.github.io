45.4.1.3. 고도 제어(Altitude) 모드 진입: 기압계 및 거리 측정기 관측 행렬(Observation Matrix)의 가시성(Observability) 확보 유무 판별을 위한 그람 행렬(Gramian Matrix) 분석

1. 개요 및 이론적 배경

무인항공기의 자율 비행에서 가장 기초가 되는 자동 제어 모드는 고도 제어(Altitude Control) 모드이다. PX4-Autopilot에서 고도를 안정적으로 제어하려면 외부 센서(기압계, 거리 측정기, GNSS 등)로부터 수직 위치(Z축)와 수직 속도(Z축 속도)를 지속적이고 정확하게 추정해야 한다.

수학적 제어 이론 관점에서 특정 상태 변수(수직 위치 및 속도)를 측정치로부터 역추산할 수 있는 성질을 **가시성(Observability)**이라 한다. 본 절에서는 현대 제어 이론의 척도인 관측 행렬(Observation Matrix)과 그람 행렬(Gramian Matrix)의 개념을 빌려, PX4의 EKF2(Extended Kalman Filter 2) 알고리즘이 기압계(Barometer)와 거리 측정기(Rangefinder) 데이터를 이용해 고도 상태 변수의 가시성을 어떻게 확보하고, 커맨더(Commander) 스레드가 고도 모드 진입을 허가하는 로직에 대해 상세히 분석한다.

2. 상태 고도 추정과 관측 행렬(Observation Matrix)

선형화된 제어 시스템(혹은 EKF의 오차 상태 공간 모델)은 아래와 같이 정의된다.

\dot{x}(t) = A x(t) + B u(t)
y(t) = H x(t) + v(t)

여기서 H는 관측 행렬(Observation Matrix)이며, 상태 변수 x (고도, 수직 속도 등)가 센서 측정치 y로 어떻게 매핑되는지를 나타낸다. 고도 추정을 위한 일차적 센서는 **기압계(Baro)**와 **거리 측정기(Rangefinder/LiDAR)**이다.

  • 기압계 측정 모델: y_{\text{baro}} = -x_z + v_{\text{baro}} (고도는 대기압에 반비례하며, 아래 방향을 양수(NED 좌표계)로 취하므로 음수 기호가 붙는다)
  • 거리 측정기 모델: y_{\text{rng}} = \frac{-x_z}{\cos(\theta)\cos(\phi)} + v_{\text{rng}} (자세의 롤 \phi, 피치 \theta에 의한 보상 적용)

각 센서가 활성화될 때 칼만 필터의 관측 행렬 H는 독립적인 행을 구성하며, 특정 시간 동안 시스템이 고도라는 상태를 온전히 추정할 수 있는지 확인하기 위해 가시성 분석이 수행된다.

3. 관측성 그람 행렬(Observability Gramian Matrix) 분석

수학적으로 시스템이 구간 [0, t]에서 완전 가시성(Fully Observable)을 갖기 위한 필요충분조건은 관측성 그람 행렬(Observability Gramian Matrix) W_o가 정칙(Non-singular) 행렬이 되며 정정부호(Positive Definite) 특성을 갖는 것이다.

W_o(0, t) = \int_{0}^{t} \Phi^T(\tau, 0) H^T H \Phi(\tau, 0) d\tau

  • \Phi(\tau, 0)는 시스템의 상태 천이 행렬(State Transition Matrix)이다.
  • W_o의 계수(Rank)가 시스템 차원과 일치하면 센서 H를 통해 고도 상태 축을 온전하게 분리해낼 수 있음을 의미한다.

PX4 EKF2의 내부 코어 구현에서는 이 그람 행렬을 직접 매 틱마다 역전환(Inverse)하여 계산하기엔 연산량 부하가 크므로, 대신 오차 공분산 행렬(Error Covariance Matrix) P의 수직 위치/속도에 해당하는 대각항(Diagonal Elements) 수렴도와 **혁신 분산(Innovation Variance)**을 모니터링하여 가시성이 실효적으로 달성되었는지를 근사적으로 판별하는 방식을 취한다.

4. PX4-Autopilot의 고도 모드 진입 전제 조건 아키텍처

PX4가 고도 제어(Altitude) 모드로의 천이(Transition) 명령을 수신하면, 커맨더 파트(commander.cpp)와 상태 머신 로직은 센서 융합 코어가 출력하는 플래그(Flag)들을 토대로 진입 전제 조건(Pre-conditions)을 평가한다.

4.1 EKF2 센서 소스 선택 파라미터 구성

PX4는 고도의 주 기준 센서를 파라미터로 사전에 정의한다.

  • EKF2_HGT_REF: 고도 기준을 결정한다 (0: Barometric Pressure, 1: GNSS, 2: Range sensor, 3: Vision).
    기압계(0) 혹은 거리 측정기(2)를 선택할 경우, 필터는 해당 센서 관측치에 대하여 H 행렬을 구성하여 업데이트 프로세스를 활성화한다.

4.2 Local Position 유효성 검사 로직

커맨더는 vehicle_local_position 토픽 내의 유효성 검증 플래그를 확인한다.

// PX4 Commander 모듈에서 Altitude Mode 진입 요구 조건 판별 로직 요약
bool alt_control_allowed = local_pos.z_valid && local_pos.v_z_valid;

if (!alt_control_allowed) {
    // EKF가 고도 상태 가시성을 확보하지 못한 상태
    transition_to_altitude_mode_denied();
} else {
    // 그람 행렬적으로 관측 가능성이 입증되어 covariance가 수렴된 상태
    accept_mode_switch();
}

여기서 z_valid 및 v_z_valid 플래그는 EKF2 코어 모듈이 수직 상태계의 칼만 오차 공분산 P 값이 임계치 이하로 수렴하고 지속적인 센서 융합(Fusion)이 이루어져 가시성(Observability)이 충족되었을 때만 true로 설정된다.

5. Ardupilot (AP_NavEKF3) 대비 차이점

Ardupilot의 EKF3 설계 역시 기본적으로 기압계 역학 모델과 거리 측정기 모델을 활용해 고도 칼만 업데이트를 수행한다. 그러나 기준 센서 전환 및 가시성 상실 대처 방식에 뚜렷한 차이가 존재한다.

  • 거리 측정기 폴백(Fallback) 구조:
    PX4는 EKF2_RNG_CTRL 등을 통해 일정 고도 이하에서만 Rangefinder를 사용하고 그 이상에서는 Baro/GNSS로 H 행렬 소스를 동적으로 블렌딩(Blending)하거나 교체하는 유연성을 제공한다.
    Ardupilot은 EK3_SRC1_POSZ, EK3_SRC2_POSZ 등을 통해 1차, 2차, 3차 백업 소스를 하드코딩 형식에 가깝게 우선순위화한다. 특정 센서의 혁신 테스트가 카이제곱 범위를 이탈할 시, 시스템 차원에서 두 번째 소스로 완전히 전환(Lane Switch 또는 Source Switch)하는 강경한 방식을 취한다.
  • 가시성(Observability) 기준의 스케줄링: PX4는 고도 상태가 발산 중일 경우 플래그(z_valid)를 즉시 내리고 수동 제어(Manual Mode)로 추락을 감수하더라도 관성 이탈을 막는 보호 성향이 강하다. 반면, Ardupilot은 가능한 모든 백업 조합을 평가하여 비행 상태를(Position Hold나 Altitude Hold) 유지시키려 시도하는 복원 중시형 튜닝을 지향한다.

6. MAVLink을 통한 상태 전달 및 관찰

지상 관제 시스템(QGroundControl)은 EKF의 가시성 획득 여부를 알기 위해 ESTIMATOR_STATUS (ID: 230) 및 LOCAL_POSITION_NED (ID: 32) 메시지를 관찰한다.

  • MAVLink가 전달하는 LOCAL_POSITION_NED 시퀀스 안에서 수직 좌표(Z축) 정보가 누락되거나 매우 큰 편차 노이즈를 띠면, GCS 측에서는 EKF 내부의 가시성이 저해되었다고 판단할 수 있다.
  • 상태 기반 제어 앱 개발자는 명령 전송 전, MAVSDK를 통해 Telemetry::health().local_position_ok 플래그를 폴링(Polling)하여 고도 모드로의 전환 명령(COMMAND_LONG의 모드 변경) 전송 가능 시점을 논리적으로 스케줄링할 수 있다.

7. ROS2 연동 기반 예외 처리

ROS2 에코시스템 내 자율 관제 에이전트는 기압계 및 거리 측정기 가시성 저하 스파이크 현상에 대응하기 위해 uORB 브리지를 통과한 토픽을 구독한다.

# ROS2 Python 고도 가시성 모니터링 및 Failsafe
from px4_msgs.msg import VehicleLocalPosition

def local_pos_callback(self, msg):
    # z_valid (가시성을 통한 추정 무결성 플래그)가 False가 되면
    if not msg.z_valid:
        self.get_logger().error("수직 상태계 가시성(Observability) 상실!")
        # 자동 하강 스크립트 강제 발동 또는 조종자 긴급 알림 전송 (ROS2 Node Action)
        self.trigger_emergency_descent()

위와 같이 ROS2 레이어는 기체 하부의 관측 행렬 특성 및 그람 행렬 안정성 붕괴로 야기된 상태 불확실성(Covariance Growth)을 z_valid 불리언 값으로 파악하고, 상위 계층의 행동 결정(Behavior Tree)을 통해 기체를 비상 대피시키거나 임무를 일시 중단하는 고도화된 관제를 구현할 수 있다.
