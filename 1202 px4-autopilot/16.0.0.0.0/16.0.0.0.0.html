Chapter 16. PX4 펌웨어 소스 코드 구조 및 디렉토리 분석

PX4-Autopilot의 소스 코드는 수십만 줄의 방대한 C++ 코드베이스(Codebase)와 Python 메타 스크립트, 그리고 NuttX RTOS 서브모듈(Submodule)이 거미줄처럼 얽혀 있는 거대한 유기체이다. 이 거대한 지식의 숲에서 길을 잃지 않고 특정 비행 제어 알고리즘이나 센서 드라이버가 묻혀 있는 진원지를 단숨에 짚어내기 위해서는, 최상위 루트(Root) 디렉토리부터 하위 src/ 트리의 끝단까지 이어지는 구조적 철학과 명명 규칙(Naming Convention)을 완벽하게 숙지해야만 한다.

이 장에서는 단순히 폴더 목록을 나열하는 것을 넘어, 빌드 시스템(CMake)이 소스 파일을 어떻게 빨아들이는지, 하드웨어 추상화 계층(HAL)이 물리적 보드 구조와 어떻게 매핑되는지, 그리고 수백 개의 모듈(Modules)들이 어떤 디렉토리 그룹에 편제되어 있는지를 해부구조학적 관점에서 철저하게 뜯어본다.

이 장을 관통하는 핵심 개념은 다음과 같다.

  • 빌드와 소스의 분리: 플랫폼 무결성을 유지하기 위한 boards/, platforms/ 아키텍처
  • 도메인 격리: src/drivers/ 와 src/modules/ 의 계층적 단절 원리
  • 중앙 통신망: msg/ 폴더를 통한 uORB 인터페이스 자동 생성 메커니즘
  • 외부 종속성: Kconfig와 src/lib/ 가 결합하여 펌웨어 크기를 최적화하는 원리

이를 통해 엔지니어는 단순한 코드 리딩(Reading)을 넘어, 새로운 센서를 추가하거나 독자적인 비행 제어 앱을 이식할 때 정확히 어느 디렉토리 분기점에 칼을 대야 하는지 구조적 직관을 얻게 될 것이다.
