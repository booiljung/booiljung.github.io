35.4.4.2. 확장 칼만 필터(Extended Kalman Filter) 내부 상태 혁신(Innovation) 분산 값 모니터링을 통한 이종 센서 간 발산(Divergence) 감지

자율 비행의 신뢰성은 관성 측정 장치(IMU), 위성 항법 시스템(GNSS), 지자기 센서, 기압계 등 이기종(Heterogeneous) 센서 결합의 정밀도에 달려있다. 하지만 실세계 환경에서는 고층 빌딩 숲에서의 GPS 다중 경로 오차(Multipath Error), 금속 구조물 근처에서의 자기장 간섭(Magnetic Interference) 처럼 국소적이고 치명적인 센서 기만이 시시각각 발생한다.

수동적인 Ardupilot 과거 버전이 센서의 물리적 단절(Disconnect)만을 고장으로 판별했던 것과 달리, 최신 PX4-Autopilot은 알고리즘적으로 엮여 있는 EKF2(Extended Kalman Filter 2)의 내부 수학적 스트레스를 모니터링하여 센서가 물리적으로 연결되어 있으나 거짓말을 하고 있는 상태를 동적으로 찾아낸다. 본 절에서는 내부 상태 혁신(Innovation) 분산 값의 모니터링 체계와 센서 발산 방어 로직을 분석한다.

1. 혁신(Innovation) 지표와 테스트 비율(Test Ratio)의 수학적 정의

칼만 필터 아키텍처에서 ’혁신(Innovation)’이란, 필터가 과거의 상태(IMU 적분)를 기반으로 예측(Prediction)한 현재 기체의 상태 수치와, 방금 들어온 외부 센서(GPS, 나침반 등)의 실제 측정값(Measurement) 사이의 ’오차’를 뜻한다.

센서가 정상적이라면 이 혁신 값은 영(Zero)을 중심으로 한 좁은 가우시안 얀 분산(Gaussian Variance) 내에 머물러야 한다. 센서 진단 모니터링(Health Monitoring) 체계는 이 혁신 오버헤드를 추적하기 위해 estimator_status 및 estimator_innovations 토픽을 통해 다음의 수학적 지표를 발행한다.

\text{Test Ratio} = \frac{\text{Innovation}^2}{\text{Innovation Variance}}

  • 테스트 비율 (Test Ratio): 특정 센서 데이터가 EKF 수학 모델 파라미터(게이트 임계값, Gate Threshold)가 허용하는 최대 공차의 몇 퍼센트(%)를 차지하고 있는지를 0.0 ~ 1.0 사이의 정규화된 스칼라로 표현한 지표이다.
  • GPS 속도, GPS 위치, 기압계 고도, 자력계 방위각 등에 대해 각각 독립적인 테스트 비율이 매 프레임마다 산출된다.

2. 혁신 분산 모니터링 및 센서 발산(Divergence) 판단 메커니즘

commander (기체 생명주기 관리자) 스레드와 ekf2 모듈 자체의 헬스 진단기는 이 테스트 비율을 비동기적으로 폴링(Polling)하며 이종 센서 간의 발산(Divergence) 징후를 감시한다.

  1. 지속적 발산 감지: 기체가 도심 항공(Urban Area)에 진입하여 GPS 신호가 반사파에 섞이기 시작하면, EKF2가 예측한 위치와 GPS가 보고하는 위치 간의 괴리가 커진다. 이때 GPS 위치 혁신(Position Innovation) 테스트 비율이 급격히 상승하여 임계치(1.0)를 돌파하게 된다.
  2. 센서 거부(Sensor Rejection): 테스트 비율이 1.0을 초과하는 상태가 특정 윈도우 시간(예: EKF2_REQ_VDRIFT 연관 지연 시간) 이상 지속되면, EKF2는 해당 센서(위 예시의 GPS)가 기만당했다고 판독한다. 내부 융합 스케줄러는 즉각 해당 센서의 가중치 행렬을 0으로 만들어버림으로써, 고장 난 데이터가 전체 공간 추정(Spatial Estimation) 풀을 오염시키는 것을 하드웨어적으로 잘라낸다(Reject).
  3. 텔레메트리 경보 스트리밍: 이 순간 estimator_status 토픽의 비트마스크(Bitmask) 깃발(Flag) 중 해당 센서의 health 비트가 0(False)으로 전환되며, 이는 즉시 MAVLink SYS_STATUS 와 EXTENDED_SYS_STATE 로 맵핑되어 GCS(지상 관제소) 화면에 “GPS Fusion Rejected“라는 치명적 붉은 경고 체계를 점등시킨다.

3. 다중 센서 생존 논리(Fallback)로의 계층적 전이

센서가 잘려나간(Rejected) 후, PX4 시스템 구동계는 멍청하게 자리를 지키지 않는다. 구독 시스템을 통해 EKF2의 혁신 발산을 보고받은 FSM(유한 상태 머신) 내비게이터(Navigator)는 즉각 전술적 폴백(Tactical Fallback) 절차를 시동한다.

  • 위치 제어(Position Control) 모드 상실: GPS와 옵티컬 플로우(Optical Flow) 등 횡방향 위치 추정(XY-Position Estimation) 혁신 값이 모두 한계치를 넘어 폐기될 경우, FSM은 위치를 스스로 보존할 수 없다고 판단한다.
  • 고도 제어(Altitude) 또는 수동(Manual) 폴백: 기체는 스스로의 의지에 따라 GPS 기반의 완전 자동 임무(Mission)나 위치 고정(Position) 모드를 강제 해제하고, 오직 기압계 고도 혁신(Baro Innovation)만이 살아남아 있는 고도 고정(Altitude) 모드, 혹은 순수 IMU 자세각에 의지하는 안정화(Stabilized) 모드로 제어 계층을 한 단계 격하시킨다.

결론적으로 PX4의 확장 칼만 필터 혁신(Innovation) 구독 시스템은, 단순히 센서가 “켜져 있는지 꺼져 있는지“를 묻는 단편적 핑(Ping) 테스트를 초월한다. 이는 각 센서가 인접 센서들과 물리적 우주의 법칙(Kinematics) 안에서 얼마나 일치하게 노래하고 있는지의 화음(Harmony) 붕괴를 수학적으로 포착해내는 고도화된 생존 이력 감시기이다.
