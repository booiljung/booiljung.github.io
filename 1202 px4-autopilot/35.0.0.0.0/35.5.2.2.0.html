35.5.2.2. 지상 접촉(Landed State) 감지 알고리즘 연동을 통한 비행 중 동력 차단 방지 소프트웨어 안전 기계 장치(Safety Interlock) 구현

고도 100미터 상공에서 자율 비행 중인 드론에, 오작동한 지상 관제 스크립트나 인간 조종사의 치명적 실수가 맹목적인 action.disarm() 명령을 전송한다면 시스템은 어떻게 반응해야 할까? 만약 비행 제어기(Flight Controller)가 이 명령을 그대로 수용하여 모터 전원을 즉각 차단한다면, 기체는 수 초 내에 지상과 충돌하여 수천 달러의 장비와 페이로드가 완전히 파괴될 것이다.

PX4-Autopilot은 이러한 치명적인 인위적 하달 오류(Command Error)를 방어하기 위해, 기체가 물리적으로 단단한 지상에 닿아 있는지 수리적으로 판별하는 Landed Detector(지상 접촉 감지기) 모듈을 유한 상태 머신(FSM)의 핵심 인터락(Interlock)으로 깊숙이 연동시켜 놓았다. 본 절에서는 이 극한의 예외 처리 아키텍처를 해부한다.

1. 다중 센서 융합 기반 지상 접촉(Landed State) 추론 알고리즘

기체가 땅에 닿았는지 공중에 떠 있는지를 아는 것은 인간에게는 시각적으로 직관적이나, 눈이 없는 비행 제어기에게는 고도의 수학적 추론 과정을 요구한다. PX4의 land_detector 라이브러리는 단순히 기압 고도계가 0미터를 가리키는지를 맹신하지 않는다(기압계는 대기압과 온도 변화에 따라 영점이 실시간으로 표류하기 때문이다). 대신 다음과 같은 동역학적 파라미터들의 논리적 교집합을 평가한다.

  1. 상하(Z축) 속도 및 가속도 수렴: EKF2 추정기를 통해 계산된 기체의 절대 수직 하강 속도(V_z)가 LNDMC_Z_VEL_MAX 파라미터(예: 0.25 \text{m/s}) 미만으로 떨어졌으며, IMU 정규화 캘리브레이션 가속도가 정확히 지구 중력 가속도(1\text{G}) 근처에서 노이즈 없이 안정화되었는가?
  2. 수평(XY축) 횡방향 운동 상실: 기체의 수평 이동 속도가 고해상도 GPS 및 지자기 센서 추정 데이터 상으로 특정 임계치 이하로 떨어져 물리적인 고착 상태에 돌입했는가?
  3. 추력(Thrust) 출력 한계 도달: 내부의 위치 제어기(Position Controller)가 현재 고도를 유지하기 위해 모터 회전 속도 조절기(ESC)에 명백하게 호버링 추력 최소치(MPC_THR_MIN)보다 한참 낮은 빈약한 펄스 폭 변조(PWM) 신호를 강제하고 있음에도 불구하고, 기체가 더 이상 하강하지 않는가? (바닥이 기체를 밀어 올려 지지하고 있음을 시사)

위 세 가지 동역학적 조건이 모두 일정 윈도우 시간(LNDMC_TRIG_TIME, 통상 0.3~1초) 동안 미분 요동을 일으키지 않고 유지될 때, 비로소 vehicle_land_detected uORB 토픽의 landed 부울(Boolean) 플래그가 참(True)으로 변환(Assertion)된다.

2. 소프트웨어 안전 기계 장치(Safety Interlock)의 록다운(Lockdown) 메커니즘

commander 모듈의 중앙 상태 머신(FSM)은 이 Landed State 플래그를 절대적인 물리적 진실로 받아들이며, 외부에서 밀려 들어오는 모든 동력 차단(Disarm) 요청을 검열하는 수문장(Gatekeeper)으로 삼는다.

  • 체공 중(In-air) 차단 권한 무효화: 기체가 비행 중(landed == false)일 때 조종기 스틱을 좌하단으로 짓누르는 아밍 락(Arming Lock) 제스처를 수 초간 취하거나, MAVSDK 스크립트 기반 터미널에서 버그로 인해 action.disarm() API가 빗발치듯 호출되더라도, FSM 로직은 이를 인간 또는 스크립트의 ’치명적 단일 실수(Single Point of Error)’로 치부하고 완전히 묵살(Ignore)해 버린다. 모터는 공기 역학적 자세 제어력을 파괴하지 않기 위해 ‘에어 모드(Air Mode)’ 아이들(Idle) RPM을 상공에서 굳건히 사수한다.
  • 착륙 후 자동 차단(Auto-disarm) 연동: 반대로 기체가 완전히 지상 표면에 착지하여 landed == true 상태가 굳어지면, FSM은 내부 카운트다운 타이머를 가동한다. COM_DISARM_LAND 매개변수로 명시된 시간(보통 2~5초)이 경과하면, 지상 관제 시스템(GCS)의 명시적인 명령이 도달하기도 전에 FSM이 스스로 시스템 인터럽트를 걸어 액추에이터 출력을 물리 계층에서 끊어버린다. 이는 착륙 직후 지면과 기체 간섭으로 인해 드론이 통통 튀어 오르다 스스로를 뒤집어엎는 치명적인 그라운드 레조넌스(Ground Resonance) 현상을 프로그래밍 단에서 차단하기 위함이다.

3. 딜레마 역전: 예외적 강제 개입(Kill Switch)의 절대적 보장

지상 접촉 감지 모듈을 생명 유지의 최상위 방어막으로 설정함에 따라 필연적으로 발생하는 논리적 모순, 즉 통제 권한의 기계 쏠림 현상을 파훼하기 위해 PX4 구조는 오직 단 하나의 비대칭 백도어(Backdoor)를 개방해 두고 있다.

만약 비행 중 돌발적으로 프로펠러 한 짝이 파손되어 기체가 이미 미친 듯이 제자리 스핀하며 추락 중이라면, IMU 잔진동과 EKF2 궤적 추정기의 엉망진창인 쓰레기 값들로 인해 land_detector 알고리즘은 평생 기체가 땅에 얌전히 닿았다는 수리적 결론을 내리지 못할 것이다. 이때 조종자나 관제사는 action.kill() (MAVLink MAV_CMD_DO_FLIGHTTERMINATION) 비상 탈출 명령을 발동해야만 한다. 이 Kill 명령은 Landed State 검증 단계를 전면 바이패스(Bypass)하여 즉각 하드웨어 타이머 레지스터를 영점 처리함으로써, 통제 불능의 탄환이 된 드론 모터에 전력을 차단해 2차 대형 참사를 강제로 중단시킨다.

결론적으로 PX4의 지상 접촉 감지 연동 보안 인터락 아키텍처는 유한 상태 제어 권한을 영리하게 양분한다. 일상적인(Nominal) 비행 생명주기는 고도화된 내부 추정 알고리즘적 확신(Landed State) 에 전적으로 기대어 관제 오작동을 두텁게 필터링하고, 극한의 재난 상황에서는 비대칭 비상 선언(Kill Switch) 패러다임을 통해 하드웨어 제어권을 인간에게 무조건적으로 반환하는, 완벽에 가까운 강건한 산업 시스템 생태계를 완성해 낸다.
