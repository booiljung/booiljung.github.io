35.7.1.1. 제어 목표치(Set-point) 스트리밍의 최소 요구 주파수 마진(Frequency Margin) 계산 및 나이퀴스트 주파수(Nyquist Frequency) 충족 조건 분석

PX4의 오프보드(Offboard) 모드에서 외부 동반 컴퓨터가 시리얼 버스를 관통하여 비행 제어기(FCU)의 하단 제어기로 쏟아내는 제어 목표치(Setpoint) 스트림 변수는, 단순한 통신 데이터 패킷이 아니라 기체의 아날로그적인 물리적 동역학을 완전히 지배하는 연속 시간 비선형 제어(Continuous-time Nonlinear Control) 신호의 이산화(Discretization)된 표본(Sample) 상수이다.

외부 제어용 C++ 응용 데몬이 이 셋포인트를 퍼블리시(Publish)하는 업데이트 빈도(Frequency)를 대체 어느 정도의 주파수 헤르츠(Hz)로 설계하고 설정해야만, 이더넷 버스의 통신 대역폭을 불필요하게 낭비하지 않으면서도 기체 강체의 물리적 제어 대역폭(Control Bandwidth) 추종성을 침해하지 않을 수 있는가? 이는 단순히 프로그래머 감각의 영역이 아니며, 센서 데이터 융합과 신호 처리 공학의 절대적 빙벽인 나이퀴스트-섀넌 표본화 정리(Nyquist-Shannon Sampling Theorem) 에 입각한 극단적인 주파수 여유 마진(Frequency Margin) 컷오프 계산 수식을 철저히 요구한다.

1. 기체 비행 동역학 대역폭(Kinematic Bandwidth) 한계와 나이퀴스트 주파수 절대 조건

수 킬로그램의 멀티로터 무인항공기는 롤(Roll), 피치(Pitch) 회전 기동에 대해 헬리콥터와 맞먹는 매우 폭발적이고 높은 동적 응답성(High Dynamic Responsiveness)을 지닌다. 오프보드 제어를 통해 어떤 계층 깊이(Position 공간, Velocity 벡터, Attitude 쿼터니언)의 제어 루프를 다이렉트로 타격하느냐에 따라 셋포인트 스트리밍의 필요 최소 요구 주파수 대역이 수학적으로 거칠게 달라진다.

  1. 내부 제어 루프 레이어의 시스템 고유 주파수(System Natural Bandwidth):
  • PX4 추종 뼈대의 최하단 브러시리스 각속도(Angular Rate) PID 루프는 250\text{Hz} \sim 800\text{Hz} 의 파멸적인 나노초 속도로 회전하며, 자세(Attitude) 추종 루프는 약 50\text{Hz}, 최상단 최외곽에 위치한 위치/속도(Position/Velocity) 공간 제어 루프는 약 10\text{Hz} \sim 20\text{Hz} 의 물리적 대역폭 컷오프(Cut-off)로 모델링된다.
  • 외부 제어기가 궤적(Trajectory)을 허공에 그리며 MAVLink 셋포인트를 하사할 때, 이 입력 신호의 전송 주파수가, 모델이 그 궤적을 쫓기 위해 기동해야 할 물리적 최고 진동수(최대 기동 대역폭 f_{max})의 디테일을 온전히 담아내지 못하면, 기체의 움직임은 정보를 심각하게 상실하고 끔찍한 앨리어싱(Aliasing) 정보 왜곡(Distortion) 현상에 시달려 궤적을 벗어나게 된다.
  1. 나이퀴스트-섀넌(Nyquist-Shannon) 충족 수학적 제약(Criterion):
  • 섀넌의 정보 정리에 따라, 목적 궤적의 가장 빠르고 날카로운 움직임(예: 급격한 방위각 코너링 턴) 주파수 성분이 f_{max} 일 때, 이를 아날로그의 손실 없이 완벽히 디지털 이산화 셋포인트로 보내기 위한 절대적 최소 스트리밍 샘플링 주파수 f_s 는 영구적으로 f_s > 2 \cdot f_{max} (나이퀴스트 속도 Limit)를 관통하여 넘어서야만 한다.
  • 예를 들어, 장애물 회피를 위해 기체가 5\text{Hz} 주기의 S자 급격한 스플라인 곡선 궤적 진동(f_{max}=5\text{Hz})을 극단적으로 추종해야 한다면, 외부 제어기는 최소 10\text{Hz} 이상의 주기로 치열하게 셋포인트를 퍼블리시 쏘아보내지 않으면, 그 S자 궤적의 뾰족한 봉우리(Peak) 진폭 좌표점이 통신 패킷 사이의 텅 빈 시간 틈(Time Gap)의 지옥으로 듬성듬성 증발해 버려 기체가 코너를 돌지 못하고 허공 밖 경로를 무참히 이탈(Overshoot/Flyaway)하게 되어 버린다.

2. 위상 지연(Phase Lag) 방어와 텔레메트리 레이턴시 극복을 위한 실무적 주파수 마진(Frequency Margin) 오버샘플링 계산

이론상 나이퀴스트 주파수 극한 컷오프인 정확히 2배수로만 스트리밍 루프를 아슬아슬하게 돌리는 것은, OS 스케줄러의 컨텍스트 스위칭 지연과 텔레메트리 파이프라인의 예측 불가한 핑 지터(Jitter)가 난무하는 실제 무인기 하드웨어 보드 환경에서는 자살 행위에 가깝다. 끔찍한 위상 지연(Phase Lag) 진동을 막고 시스템 안전망(Failsafe Timeout) 퓨즈를 여유롭게 빗겨가기 위해 엄청나게 과장된 여유 주파수 마진(Margin)을 보수적으로 더해 연산해야 한다.

  • 위상 지연 보상 억제 마진 (Phase Margin Lag Compensation):
  • 셋포인트 펌핑 주파수가 기체의 동작 주파수 최대치와 2배수로 아슬아슬하게 근접할수록, 제어 영-차 유지기(Zero-Order Hold, ZOH) 로직에 의한 이산화 시점의 계단형 지연 타임 에러가 급증하여 시스템 위상 여유(Phase Margin)를 거세게 갉아먹고 결국 기체를 불안정하게 발산(Divergent Oscillation)시켜 떨어뜨린다.
  • 따라서 실무 제어 공학 설계에서는, 2배수를 아득히 무시하고 나이퀴스트 주파수 하한의 최소 5배에서 최대 10배(10 \cdot f_{max}) 까지의 과도한 오버샘플링(Oversampling) 대역폭 마진을 무식하게 부여한다. 즉, 위치/속도 공간 셋포인트 스트리밍 플러그인은 20\text{Hz} \sim 50\text{Hz} 의 넉넉한 수준을, 자세 각/수동 각속도(Attitude/Rate) 뼈대 스트리밍은 최소 100\text{Hz} \sim 250\text{Hz} 의 맹렬한 대역을 유지해야만 비로소 기체의 파이프라인에서 떨림 없는 부드럽고 잔인한 궤적 추종이 그래픽처럼 달성된다.
  • 오프보드 안전망 퓨즈 주파수 절대 하한선 (Failsafe Baseline Threshold Limit):
  • 나이퀴스트 조건의 피 말리는 계산 이전에, PX4의 Commander 생명 파이프라인은 COM_OF_LOSS_T 파라미터 퓨즈(기본 0.5초)에 의해 최소 초당 2\text{Hz} 라는 어떠한 목에 칼이 들어와도 거절할 수 없는 하드웨어 하한 통신 주파수 방어선을 강제로 그어둔다.
  • 외부 리눅스 컴퓨터의 CPU 스로틀링(Throttling) 연산 과부하 등으로 인해 오프보드 스트리밍 퍼블리시 데몬의 심박수 주기가 단 한 번이라도 2\text{Hz} 밑으로 헐떡거리며 떨어지는 찰나의 순간, 즉시 나이퀴스트 계산이고 뭐고 모든 데이터 링크가 악의적으로 끊어진 공격으로 간주되어 권한 사살(Revoke) 및 RTL/Loiter 강제 폴백 처분을 끔찍하게 당하게 됨을 뼛속 깊이 명심해야 한다.

결론적으로 외부 오프보드 플러그인 연동 시스템에서, C++ 제어 데몬의 셋포인트 퍼블리시 주파수 f_s 렌더링 값의 설계 결정은, 무선 대역폭 자원을 가난하게 절약하려는 소모적이고 소극적인 네트워크 태도와, 기체 동역학 대역폭(f_{max})의 영혼을 온전히 담아내려는 나이퀴스트 정리의 절대적 한계선 스펙트럼 사이에서 맹렬하게 대립 연산하며, 궁극적으로 지터(Jitter) 방어 오버샘플링 마진과 타임아웃(Timeout) 한계선의 안전 범위 교집합을 치열하게 산출해 내어 하드코딩(Hard-coding) 되어야 하는 폐루프 정밀 외부 제어 아키텍처의 필수 불가결한 공학 수학 방정식이다.
