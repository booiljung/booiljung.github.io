35.4. 센서 융합(Sensor Fusion) 기반 원격 측정(Telemetry) 데이터 비동기 파이프라인 설계

현대의 무인항공기(UAS)는 관성 측정 장치(IMU), 위성 항법 시스템(GNSS), 지자기 센서(Magnetometer), 기압계(Barometer) 등 수많은 이종(Heterogeneous) 센서를 탑재한다. 각 센서는 고유의 샘플링 주파수(Sampling Frequency)와 노이즈(Noise) 특성을 가지며, 이를 바탕으로 기체의 현재 위치와 자세를 정확하게 추정하는 센서 융합(Sensor Fusion) 과정은 비행 제어의 가장 무거운 연산 부하를 차지한다.

지상 관제 시스템(GCS) 입장에서 기체가 현재 “어디에 있고, 어떤 자세를 취하고 있는가“를 실시간으로 모니터링하기 위해서는, 혼재된 센서 데이터와 확장 칼만 필터(EKF)의 융합 결과물을 병목 현상 없이 비동기적(Asynchronous)으로 추출하여 통신 채널로 내보내는 원격 측정(Telemetry) 데이터 파이프라인 설계가 필수적이다. 본 절에서는 PX4-Autopilot의 내부 uORB 메시징 아키텍처부터 MAVLink 스트리밍까지 이어지는 비동기 데이터 흐름을 구명한다.

1. PX4 센서 융합 및 텔레메트리 아키텍처 개요

단일 메인 루프 기반의 공유 메모리 변수 직접 접근 방식을 주로 선호하는 전통적인 Ardupilot 아키텍처와 구별되게, PX4-Autopilot은 단일 책임 원칙(Single Responsibility Principle)을 지향하는 독립된 데몬(Daemon) 스레드들의 집합으로 동작한다.

  1. 센서 드라이버 계층: IMU(예: 1000Hz), 나침반(예: 50Hz), GPS(예: 5Hz) 드라이버 스레드는 하드웨어 버스(SPI/I2C/UART)로부터 원시 데이터를 읽어 즉시 sensor_accel, sensor_gps 등의 uORB 토픽(Topic)으로 퍼블리싱(Publish)한다.
  2. 센서 융합 계층 (Estimator): EKF2(Extended Kalman Filter 2) 모듈은 이러한 센서 uORB 토픽들을 구독(Subscribe)한다. EKF2는 고주파수의 IMU를 예측(Prediction) 단계로 삼고, 저주파수의 GPS나 기압계를 교정(Correction) 단계로 융합하여 vehicle_local_position, vehicle_attitude 등의 정제된 융합 상태(Fused State) uORB 토픽을 발행한다.
  3. 텔레메트리 송출 계층 (MAVLink 데몬): mavlink 모듈은 GCS로 데이터를 보내기 위해 융합된 uORB 토픽들을 비동기적으로 폴링(Polling)하거나 구독 콜백(Callback)을 통해 읽은 뒤, 이를 ATTITUDE, LOCAL_POSITION_NED, GLOBAL_POSITION_INT 등의 MAVLink 메시지 포맷으로 직렬화(Serialization)하여 통신 포트(UART 또는 UDP)로 밀어낸다.

2. uORB 기반 비동기 스트리밍 (Asynchronous Streaming) 파이프라인

센서 융합 파트에서 출력되는 uORB 발행 주기와 MAVLink 모듈이 GCS로 쏘아보내는 전송 주기는 서로 비동기적으로 분리(Decoupling)되어 있다. EKF2 모듈이 기체 자세를 400Hz로 추정하더라도, 대역폭 한계가 있는 무선 링크를 통해 관제소에 400Hz로 계속 보낼 수는 없다.

PX4의 mavlink 모듈은 내부적으로 각 메시지 타입에 대한 스트리밍 노드(Stream Node) 클래스를 선언하고, 각 노드가 자신만의 타이머를 기반으로 데이터를 비동기 샘플링하는 구조를 갖는다.

2.1 스트리밍 대역폭 제어 및 폴링(Polling) 최적화

MAVLink 데몬은 메인 루프에서 다중화된 파일 디스크립터(File Descriptor)를 통해 알림 이벤트를 감지하는 px4_poll() (POSIX poll 함수의 래퍼) 메커니즘을 사용한다.

예를 들어, GCS 운영자가 기체 자세(Attitude) 데이터를 50Hz로 내려받기를 요망(MAV_CMD_SET_MESSAGE_INTERVAL 커맨드 인가)했다면, 파이프라인은 다음과 같이 동작한다.

  • MAVLink 자세 스트림 노드의 업데이트 타이머는 20ms(50Hz)로 설정된다.
  • 매 20ms 시점마다 노드는 uORB vehicle_attitude 토픽의 최신 인스턴스를 확인(orb_copy)한다.
  • 만약 EKF2가 그사이 새로운 자세 데이터를 16번(400Hz 기준) 융합하여 퍼블리싱 했더라도, 텔레메트리 파이프라인은 앞선 15개의 중간 과정 데이터 접근을 스킵하고 가장 마지막 최신 데이터 단 하나만을 취득(Drop-sampling)하여 MAVLink로 전송한다.

이러한 비동기 폴링 기법은 무거운 락(Lock) 메커니즘에 의한 메인 추정기(Estimator) 스레드의 지연(Blocking)을 방지하는 탁월한 스레드 안전성(Thread-safety)을 제공한다.

3. 다중 데이터 링크 채널(Data Link Channel) 분리 아키텍처

최신 군집 UAS나 기업형(Enterprise) 드론 운용 환경에서는 물리적 통신 채널이 단일 RF 모뎀에 국한되지 않는다. 1차 포트는 900MHz RF 텔레메트리(느림, 저대역폭), 2차 포트는 LTE 보조 컴퓨터 연결(빠름, 임의 대역폭), 3차 포트는 카메라 짐벌 MAVLink 인터페이스일 수 있다.

파이프라인 설계 관점에서 각 텔레메트리 포트는 독립적인 mavlink 모듈 인스턴스로 구동되며, 각자 uORB 토픽을 독립적으로 구독한다.

  • 채널 1 (RF 모뎀, 대역폭 부족): 자세(Attitude) 5Hz, GPS 1Hz 축소 전송.
  • 채널 2 (보조 컴퓨터 UDP, 대역폭 충분): 자세(Attitude) 100Hz, 고해상도 IMU 원시 데이터 200Hz 덤프.

이와 같이 융합 데이터를 생성하는 EKF 관점에서는 데이터를 통신망에 어떻게 내보낼지 고려할 필요 없이 단일 uORB 파이프망에 데이터를 엎어놓기만 하면, 서로 다른 전송 대역의 복수 텔레메트리 객체들이 각자의 비동기 타이머 병행성(Concurrency) 모델에 맞춰 데이터를 퍼올리는(Pumping) 탈중앙화된 파이프라인 철학을 완성하게 된다.
