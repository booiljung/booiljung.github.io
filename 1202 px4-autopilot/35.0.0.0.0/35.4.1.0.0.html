35.4.1. 전역 좌표계(Global Coordinate System) 및 위치 추정 상태 데이터 비동기 추출

무인항공기의 자율 비행(Autonomous Flight)과 지상 관제 시스템(GCS)에서의 궤적 추적은 기체의 현재 위치를 지구 표면의 특정 지점으로 매핑하는 전역 좌표계(Global Coordinate System) 위에서 이루어진다. 하지만 단순 원시 GPS 신호만으로는 비행에 요구되는 고속의 제어 대역폭(Control Bandwidth)과 정밀도를 충족시킬 수 없다. 따라서 PX4-Autopilot은 확장 칼만 필터(EKF2)를 통해 관성 측정 장치(IMU)의 고주파 관성 데이터와 GPS의 저주파 절대 위치 데이터를 융합(Fusion)하여 최적의 위치 추정치 상태(Estimated State) 데이터를 산출한다.

본 절에서는 융합이 완료된 전역 좌표계 기반의 위치 데이터가 PX4의 uORB 메시징 버스를 거쳐 MAVLink 프로토콜로 비동기적(Asynchronous)으로 추출되는 일련의 소프트웨어 파이프라인 구조를 심도 있게 구명한다.

1. 전역 위치 융합(Global Position Fusion) 아키텍처

PX4의 센서 융합 모듈인 ekf2 트리는 내부적으로 북-동-하(NED, North-East-Down) 로컬 평면 좌표계에서 물리적 관성 역학 법칙을 누적 연산한다. 이후, 비행 시작 시점에 획득한 기준점(Global Origin, 보통 시동(Arming) 시점의 GPS 좌표)을 바탕으로 로컬 좌표계의 (x, y, z) 미소 변위를 위도, 경도, 고도로 재투영(Reprojection)하는 과정을 거친다.

이 수학적 변환의 결과물은 NuttX RTOS 내부의 독립적인 퍼블리싱 스레드를 통해 vehicle_global_position 이라는 최상위 uORB 토픽으로 끊임없이 전파된다.

  • 위도/경도: 배정밀도 부동소수점(double) 포맷으로 임시 연산된 뒤, MAVLink 패킷 손실을 최소화하고 직렬화(Serialization) 오버헤드를 줄이기 위해 IEEE 754 형식 대신 스케일링된 정수형(int32_t, 10^7 곱셈)으로 다운캐스팅된다.
  • 고도: MAVLink 스펙상 평균 해수면 고도(AMSL, Above Mean Sea Level)와 타원체 하단 지면으로부터의 상대 고도(Relative Altitude)가 분리되어 관리되며, 밀리미터 단위 정수(int32_t) 포맷으로 텔레메트리 스레드 쪽에 전달된다.

2. 텔레메트리 데이터 비동기 동시성(Concurrency) 추출

Ardupilot은 전통적으로 스케줄러 내의 메인 프레임워크 훅(Hook)에서 전역 위치 변수 묶음을 직렬로 읽어 들이는 절차적(Procedural) 폴링 구조를 상대적으로 더 많이 의존해왔다. 반면 현대적 비행 스택인 PX4는 원격 측정(Telemetry) 스레드 패밀리가 EKF 스레드와 완전히 분리되어 작동하는 비동기 데이터 펌핑(Pumping) 패턴을 지향한다.

2.1 MAVLink 데몬의 독립적 구독 메커니즘

GCS 측에서 요구하는 글로벌 위치 업데이트 주파수가 10Hz라고 가정하자.
기체에 탑재된 MAVLink 직렬(Serial) 송출 데몬은 메인 루프에서 poll() 함수를 활용하여 vehicle_global_position uORB 토픽의 변경 사항 이벤트를 감시한다.

  1. 이벤트 타이머 만료: 데몬에 할당된 100ms(10Hz 주기) 타이머가 만료되면 스레드가 활성화(Wake-up)된다.
  2. 비동기 페치(Asynchronous Fetch): orb_copy(ORB_ID(vehicle_global_position), ...) 시스템 콜을 호출하여 uORB 버퍼의 가장 최신 사본을 자기 스레드의 로컬 메모리 공간으로 복제(Copy)한다. 이 과정에서 뮤텍스(Mutex) 잠금 시간은 나노초(ns) 단위로 극도로 짧게 유지되어 EKF 모듈의 본 연산을 늦추지 않는다.
  3. 직렬화 및 송출: 취득한 C 구조체를 바탕으로 MAVLink 규격인 GLOBAL_POSITION_INT 구조체(ID: 33) 패킷으로 바이트 직렬화하여 하드웨어 UART 또는 UDP 스택 FIFO 큐에 비동기적으로 밀어 넣는다(Push).

2.2 주파수 앨리어싱(Aliasing) 및 타임스탬프 동기화 방어

EKF2 모듈이 100Hz로 전역 좌표를 산출하지만 MAVLink 텔레메트리는 무선 대역폭 한계로 5Hz 빈도로 데이터를 추출할 경우, 95개의 융합 데이터는 중간에 버려지게 된다(Drop Target). 이때 추출되는 데이터가 무선 전송 지연(Latency) 상에서 언제 생성된 것인지 정확하게 역추산할 수 있도록, PX4는 GLOBAL_POSITION_INT 구조체 헤더 요소 공간에 time_boot_ms (부팅 후 시스템 밀리초 타임스탬프)를 반드시 캡슐화한다.

덕분에 GCS 쪽의 QGroundControl 백엔드나 ROS2(Mavros/uXRCE-DDS) 위치 구독자는 수신된 시점이 아닌 발신된 시점의 원시 타임스탬프를 바탕으로, 비동기 파이프라인 상의 통신 지연에 관계없이 과거 궤적 보간(Interpolation) 연산 및 지연 보상 제어를 매끄럽게 수행할 수 있다.

이와 같이 EKF 연산 블록과 통신 블록 간의 느슨한 결합(Loose Coupling)을 유도하는 uORB 비동기 메시징은 통신망이 과부하에 걸려 병목 현상을 겪더라도 자율 비행 핵심 코일(Core Thread)의 제어 지터 조차 발생시키지 않는, 매우 결함 감내성(Fault-tolerance)이 돋보이는 모던 아키텍처라 볼 수 있다.
