35.3.3. 네트워크 세션 추적(Session Tracking) 및 통신 단절(Link Loss) 진단 유한 상태 머신(Finite State Machine)

무인항공기(UAS) 제어 스택에서 지상 관제 시스템(GCS)과의 통신 채널은 기체의 생명선과도 같다. 특히 비가시권(BVLOS) 비행이나 다중 에이전트(Swarm) 운용 중 통신망이 단절될 경우, 기체는 즉각적으로 상황을 인지하고 자율적인 안전 복귀(Return To Launch, RTL)나 비상 착륙 상태로 전환해야 한다. 이를 규제하는 핵심 논리가 바로 네트워크 세션 추적(Session Tracking)과 통신 단절(Link Loss) 진단 유한 상태 머신(Finite State Machine, FSM)이다.

본 절에서는 PX4-Autopilot의 내부 아키텍처 및 MAVSDK 환경에서 통신 세션의 생존성을 어떻게 판독하고 관제하는지, 그 수학적 상태 전이(State Transition) 모델을 깊이 있게 다룬다.

1. MAVLink 심장박동(Heartbeat) 메커니즘과 세션 추적

PX4와 GCS(혹은 보조 컴퓨터) 간의 MAVLink 통신 모델에서 각 노드는 자신의 생존 상태(Liveness)를 알리기 위해 1Hz(초당 1회) 주기로 HEARTBEAT 메시지를 브로드캐스트(Broadcast)한다. 이 패킷은 단순한 생존 신호를 넘어, 현재 기체의 비행 모드(Custom Mode), 무장 상태(Armed/Disarmed), 그리고 시스템의 전반적인 상태(System Status)를 1바이트 비트마스크 형태로 압축하여 전달한다.

PX4 펌웨어 내부의 commander 모듈(설계상 시스템 상태를 관장하는 최고 책임자)은 수신되는 데이터 링크 트래픽(주로 텔레메트리 포트로 들어오는 MAVLink 메시지)의 타임스탬프(Timestamp)를 지속적으로 추적한다.

1.1 데이터 링크 단절(Datalink Loss) 판별 시간 임계값

세션이 안전하게 유지되고 있는지, 아니면 단절되었는지를 판별하는 기준은 COM_DL_LOSS_T 파라미터로 정의된다.

  • COM_DL_LOSS_T: 데이터 링크가 끊어졌다고 판단하기까지의 타임아웃(Timeout) 시간이다 (기본값: 10초). 10초 이상 유효한 MAVLink 패킷이 수신되지 않으면 PX4 FSM은 통신 단절(Link Loss) 예외 이벤트를 트리거(Trigger)한다.

Ardupilot의 경우 FS_GCS_ENABL 파라미터를 통해 GCS 안전 장치(Failsafe)를 켜고 끄며 타이머를 별도로 두지만, PX4는 commander 모듈 트리에 데이터 링크 트래커 클래스를 독립적인 FSM 노드로 구성하여 보다 모듈화된 객체 지향적 폴백(Fallback) 전이를 수행한다.

2. 통신 단절 진단 유한 상태 머신(FSM) 아키텍처

PX4 및 MAVSDK 애플리케이션 내부에 구현된 세션 추적 FSM은 이산적인 상태(Discrete States)의 집합과, 시간(Time) 및 이벤트(Event)에 의한 전이(Transitions) 규칙으로 엄격하게 정의된다.

다음은 통신 세션 트래커의 전형적인 상태 다이어그램이다.

stateDiagram-v2
    [*] --> INIT: 시스템 부팅 및 인터페이스 초기화
    
    INIT --> CONNECTED: 첫 MAVLink HEARTBEAT 수신
    
    CONNECTED --> CONNECTED: 지속적인 타임스탬프 갱신
    CONNECTED --> LINK_LOST: [Timer > COM_DL_LOSS_T]
    
    LINK_LOST --> RECOVERING: HEARTBEAT 재수신
    LINK_LOST --> TERMINATED: 장기 단절 임계값 초과 (강제 종료)
    
    RECOVERING --> CONNECTED: 연속 패킷 수신 및 무결성 확인
    RECOVERING --> LINK_LOST: 간헐적 재단절 (Ping-pong)
  1. INIT (초기화 단계): 시스템 전원이 인가되었으나 아직 GCS로부터 어떤 트래픽도 받지 못한 상태. 이 상태에서는 통신 단절 예외 메커니즘(RTL 등)이 억제된다. GCS가 아예 연결되지 않은 채 수동 조종기(RC)만으로 비행하는 환경을 허용하기 위함이다.
  2. CONNECTED (정상 연결 상태): 원활한 양방향 통신이 이루어지고 있는 상태. 수신 패킷의 타임스탬프가 로컬 시스템 클럭(ex. hrt_absolute_time()) 기준으로 일정하게 최신화된다.
  3. LINK_LOST (단절 진단 상태): 최신 타임스탬프 갱신 시점으로부터 현재 시점까지의 차이(Delta)가 COM_DL_LOSS_T 파라미터 임계값을 초과한 즉시 시스템이 진입하는 예외 상태이다. 이 상태로 전이되면 PX4는 자율적으로 비행 포락선(Flight Envelope)을 제한하고 지정된 NAV_DLL_ACT (Datalink Loss Action 파라미터, 예: RTL, Hold, Land 등) 대응 절차를 시작한다.
  4. RECOVERING (복구 상태): 단절 상태에서 패킷이 기적적으로 1~2개 도달했을 때의 과도기(Transient)이다. 일시적 수신에 속아 FSM 제어권을 GCS로 성급히 넘길 경우 핑퐁(Ping-pong) 현상이 발생하여 기체 제어가 심하게 흔들릴 수 있다. 따라서 일정 시간 이상 안정적 통신이 입증되어야만 다시 CONNECTED로 진입한다.

3. 통신망 상태 모니터링의 소프트웨어적 접근 (QGroundControl 연동)

GCS 쪽 소프트웨어인 QGroundControl(QGC) 내부의 C++ 백엔드 아키텍처(구체적으로 VehicleLinkManager 클래스) 역시 PX4 기체의 심장박동을 감시하는 거울(Mirror) 역할을 수행한다.

QGC는 단순히 타임아웃만 재는 것이 아니라, 수신되는 RADIO_STATUS 메시지(하드웨어 모뎀이 보고하는 통신망 품질 데이터)를 융합하여 통신 단절을 선제적으로 진단한다.

  • **수신 신호 강도 지수(RSSI, Received Signal Strength Indicator)**를 모니터링하여 임계값 이하로 떨어지는 하강 기울기(Gradient)를 감지하면 조종사 화면에 UI 팝업(텔레메트리 연결 불량 임박)을 경고한다.
  • 노이즈 스펙트럼(Noise Spectrum) 및 버퍼 오버런(Buffer Overrun) 수치가 치솟을 경우 타임아웃(COM_DL_LOSS_T)에 도달하지 않더라도 데이터 링크 손상으로 간주하고, 운영자에게 “Telemtry Link Degraded” 시각적 피드백을 즉각 제공한다.

4. 임무(Mission) 지속성 논리와 자율화

통신 단절 FSM의 복잡성은 “기체 무장(Arming) 여부“와 “자율 임무(Mission) 수행 여부“라는 상태 공간 확장에 의해 배가된다.

예를 들어, 조종기가 없는 완전 자율화 엣지 컴퓨팅(Edge Computing)이나 ROS2 기반 오프보드(Offboard) 환경에서는 “GCS 세션이 끊어지면 무조건 귀환한다“라는 규칙이 독이 될 수 있다. 기체가 깊은 산악 지역에 진입하여 일시적으로 LTE나 RF 망이 끊기더라도 지속적으로 지형 탐사를 수행(임무 연속성 보장)해야 한다면, GCS와의 MAVLink Session FSM의 출력을 NAV_DLL_ACT = 0 (Disabled)로 우회(Bypass)하도록 파라미터를 동적 변경하거나 MAVSDK에서 제어 권한을 로컬 루프로 유지하는 코어 설계가 수반되어야 한다.

결론적으로 네트워크 세션 추적을 위한 심장박동 타이머 기반 유한 상태 머신은 비행 자율성과 안전 장치 사이의 팽팽한 균형을 유지하는 핵심 관제 백엔드 알고리즘이다.
