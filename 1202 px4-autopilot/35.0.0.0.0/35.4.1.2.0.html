35.4.1.2. 위성 항법 시스템(GNSS)의 수평/수직 정밀도 저하율(DOP) 수신 및 칼만 필터(Kalman Filter) 위치 공분산(Covariance) 행렬 갱신

위성 항법 시스템(GNSS) 수신기는 단순히 기체의 위도, 경도, 고도만을 출력하는 센서가 아니다. 수신기가 포착한 위성들의 상호 간 기하학적 배치(Geometry)가 얼마나 이상적인지, 혹은 얼마나 한쪽 하늘에 치우쳐 있어 위치 결정의 오차 증폭률이 높은지를 정량화한 지표인 정밀도 저하율(DOP, Dilution of Precision)을 함께 제공한다.

PX4-Autopilot의 확장 칼만 필터(EKF2)는 이 DOP 데이터를 수동적인 모니터링 수치로만 취급하지 않고, 자율 비행 상태 추정의 신뢰도를 실시간으로 동적 조율하는 수학적 공분산(Covariance) 행렬의 핵심 입력 파라미터로 사용한다. 본 절에서는 GNSS DOP가 수신되어 EKF2의 측정 노이즈(Measurement Noise) 공분산 행렬을 갱신하기까지의 데이터 파이프라인과 그 제어 공학적 의미를 분석한다.

1. 정밀도 저하율(DOP)과 예상 위치 오차(EPE)의 상관관계

수평 정밀도 저하율(HDOP)과 수직 정밀도 저하율(VDOP)은 무차원(Dimensionless) 상수로서 그 자체로는 물리적 거리 오차를 대변하지 않는다. 따라서 PX4의 gps 드라이버는 제조사별(e.g., u-blox, Septentrio) 고유의 오차 모델 수식을 적용하여, 이 무차원 DOP에 위성 신호의 기본 측정 오차율(UERE, User Equivalent Range Error)을 곱해 실제 미터(m) 단위의 예상 위치 오차(EPE, Expected Position Error)로 변환한다.

이 변환된 데이터는 PX4 내부적으로 eph(Expected Position error Horizontal)와 epv(Expected Position error Vertical)라는 변수명으로 sensor_gps uORB 토픽에 적재된다.

일반적으로 탁 트인 상공에서 HDOP < 1.0 일 때 eph는 0.5m~1.0m 내외로 산출되며, 도심지 빌딩 숲에서 HDOP > 3.0 으로 치솟으면 eph는 5m~10m 이상으로 급팽창하게 된다.

2. 확장 칼만 필터(EKF2)의 위치 공분산 행렬 동적 갱신

비행 제어의 코어 모듈인 EKF2는 센서 데이터를 맹목적으로 추종하지 않는다. 매 시퀀스마다 자체적인 IMU 관성 적분을 통한 ’추측(Prediction)’과, 외부 센서(GPS, 비전 등)를 통한 ‘교정(Correction/Update)’ 사이에서 가중치를 산출하는 칼만 이득(Kalman Gain, K) 방정식을 푼다.

여기서 GPS의 교정 가중치를 결정짓는 가장 중요한 인자가 바로 측정 노이즈 공분산 행렬 R 이다.

R_{gps} = \begin{bmatrix} \sigma_{eph}^2 & 0 & 0 \\ 0 & \sigma_{eph}^2 & 0 \\ 0 & 0 & \sigma_{epv}^2 \end{bmatrix}

  1. 상태 머신 데이터 수신: EKF2 노드는 sensor_gps 토픽에서 수신된 eph와 epv 값을 읽어들여 자신의 관측 오차 분산(Variance, \sigma^2)으로 실시간 맵핑한다.
  2. 칼만 이득 조율(Tuning): 만약 기체가 빌딩 근처를 비행하며 GPS 다중 경로(Multipath) 반사파로 인해 순간적으로 HDOP가 폭등하여 eph가 1m에서 15m로 급격히 증가했다고 가정하자. EKF2는 행렬 R_{gps} 의 요소 크기가 팽창함에 따라 칼만 이득 K 연산 시 GPS 정보가 덜 반영되도록 가중치를 자동 억제(De-weighting)한다.
  3. IMU 편향성(Bias) 의존성 전이: GPS의 신뢰도가 떨어지는 순간, EKF2는 단기 정밀도가 높은 자체 IMU 적분 궤적에 더 크게 의존하게 됨으로써 기체가 오차 15m짜리 GPS 좌표점을 따라 갑작스럽게 스월빙(Swerving, 급선회) 하거나 런어웨이(Runaway)하는 현상을 텔레메트리 백엔드 레벨에서 차단(Rejection)한다.

3. 원격 관제소(GCS)로의 비동기 피드백 파이프라인

조종자나 자동화된 관제 인프라는 비행 중인 기체의 현재 인공위성 신호 무결성(Integrity) 상태를 지속적으로 모니터링해야만 안전한 미션 진입 시점을 판별할 수 있다.

PX4 텔레메트리 파이프라인은 앞서 uORB에 변환 적재된 eph와 epv를 비동기 폴링(Polling) 방식으로 추출하여, MAVLink 표준 프로토콜 인코더를 통해 GPS_RAW_INT 패킷의 해당 필드에 밀어 넣는다 (보통 5Hz 주기). 또한 GPS 신호의 교란이 너무 심해 EKF2가 GPS 요소 자체를 일시적으로 거부(Reject)하거나 타임아웃(Timeout)시킨 경우, 텔레메트리는 ESTIMATOR_STATUS MAVLink 메시지의 flags 비트마스크를 통해 “GPS 위치 측정 실패(GPS Position Glitch)” 상태를 GCS 프론트엔드인 QGroundControl로 비동기 송출한다.

결론적으로 PX4의 DOP 런타임 수신 체계는 불확실성이 내재된 외부 RF(무선 주파수) 환경 팩터를 수학적인 칼만 확률론(Probabilistic) 모델 안으로 끌어들여, 동적으로 자율 반응하는 유기적인 비행 포락선 생존망(Survival Net)을 구축하는 핵심이라 할 수 있다.
