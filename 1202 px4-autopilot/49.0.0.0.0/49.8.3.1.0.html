49.8.3.1 임펄스 외란(Impulse Disturbance) 및 스텝 입력(Step Input) 주입을 통한 제어기 과도 응답(Transient Response) 성능 검증

안정적인 무인항공기 비행 제어를 보장하기 위해서는 정상 상태(Steady State)에서의 오차 분석뿐만 아니라, 시스템이 새로운 목표치(Setpoint)로 전환되거나 외부 환경으로부터 급격한 충격을 받았을 때의 동적 반응을 정량적으로 평가해야 한다. 본 절에서는 PX4-Autopilot 제어 시스템(기준 최신 v1.14 펌웨어)의 위치 제어기(Position Controller)와 자세 제어기(Attitude Controller)를 대상으로, **임펄스 외란(Impulse Disturbance)**과 **스텝 입력(Step Input)**을 활용하여 제어기의 과도 응답(Transient Response) 성능을 엄밀하게 검증하는 방법론을 다룬다.

1. 스텝 입력(Step Input)과 과도 응답 지표(Transient Response Metrics)의 이해

고전 제어 공학에서 특정 시스템의 동역학적 특성을 파악하기 위해 널리 사용되는 기법이 바로 스텝 입력 테스트(Step Response Test)이다. 비행 제어에서 스텝 입력이란 기체의 목표 궤도(Trajectory)나 속도, 혹은 위치 지령(Position Setpoint)을 수학적인 계단 함수(Heaviside Step Function) 형태로 순식간에 변화시키는 것을 의미한다.

Offboard 모드를 통한 MAVLink SET_POSITION_TARGET_LOCAL_NED 메시지를 활용하면 GCS나 동반 컴퓨터(Companion Computer)에서 인위적인 스텝 변화를 쉽게 주입할 수 있다. 예를 들어, 현재 위치가 X = 0\text{m}에서 호버링(Hovering) 중인 기체에 t=t_0 시점에서 즉각적으로 X = 10\text{m}의 지령을 하달하는 방식이다.

이러한 스텝 입력이 인가되었을 때, 폐루프 제어계(Closed-loop Control System)가 목표에 도달하기까지 보여주는 과적 궤적을 과도 응답이라 하며, 이를 벤치마킹하기 위해 다음의 3대 핵심 시간 도메인 지표를 필히 측정해야 한다.

  1. 상승 시간(Rise Time, T_r): 기체의 실제 응답이 최종 목표 값의 10%에서 90%에 이르는 데 걸리는 시간. 시스템의 반응 속도(대역폭, Bandwidth)를 대변한다.
  2. 퍼센트 오버슈트(Percent Overshoot, \%OS): 기체가 목표치를 순간적으로 초과하여 상승한 최대 오차 비율. P 게인(Proportional Gain)이 과도하거나 D 게인(Derivative Gain)이 부족할 때 크게 발생한다.
  3. 정착 시간(Settling Time, T_s): 응답이 목표치의 허용 오차폭(예: \pm 2\%) 내부로 진입하여 유지되기 시작하는 시간.

2. 임펄스 외란(Impulse Disturbance)과 외란 제거 성능(Disturbance Rejection)

스텝 입력이 목표 지령(Setpoint)의 변화를 테스트하는 기준이라면, 임펄스 외란은 외부 환경 변화에 대한 강건성(Robustness)을 평가하는 수단이다. 임펄스 외란은 매우 짧은 델타 시간(Delta Time) 동안 기체에 가해지는 극단적인 힘(Force)이나 토크(Torque)의 주입을 모사한다. 디랙 델타 함수(Dirac Delta Function)와 유사한 형태를 띤다.

실제 환경에서는 강한 돌풍(Wind Gust), 특정 로터(Rotor)의 일시적 추력 상실, 혹은 페이로드(Payload)의 급격한 이동 등이 임펄스 성격의 외란에 해당한다.
시뮬레이션 인 더 루프(SITL) 환경에서는 Gazebo 물리 엔진의 커스텀 플러그인(Wind Plugin)을 조작하여, 기체의 물리 바디(Rigid Body) 모델에 동역학적인 충격을 순간 병진력(Translational Force)이나 모멘트(Moment) 형태로 인가할 수 있다.

이때 PX4의 제어기는 이 외란을 최소화하기 위해 자세 제어 루프 내부의 PID 제어기 피드백 루틴을 초고속(수백 Hz 이상)으로 가동한다. 훌륭히 튜닝된 제어기는 임펄스 입력 직후 즉각적으로 자세를 반대 방향으로 기울여(Roll/Pitch 변환) 위치 이탈(Position Drift)을 억제하고 신속히 원래의 호버링 좌표로 회복(Recovery)해야 한다.

3. 평가를 위한 수학적 비용 함수(Cost Function)

연구 및 개발 목적에서 다중 에이전트나 새로운 제어 알고리즘을 도입했을 때, 그 성능을 단순히 육안으로 튜닝하는 것은 한계가 있다. 센서 융합 시스템 내부의 ULog를 통한 원격 측정 데이터(Telemetry)를 파싱(Parsing)한 후, 시간 적분 기반의 제어 성능 지수(Performance Index) 수식을 사용하여 객관적인 점수를 도출하라. 가장 기본적이며 강력한 지표는 **적분 제곱 오차(ISE, Integral Square Error)**이다.

\text{ISE} = \int_{0}^{t} {\vert e(\tau) \vert}^2 d\tau

(단, 오차 e(t) = r(t) - y(t) 이며, r(t)는 목표 궤적 지령, y(t)는 EKF가 추정한 기체의 실제 상태(State)이다.)

스텝 파라미터 튜닝 시 이 ISE 값이 최소가 되는 방향으로 P, I, D 게인 조합(예: MC_ROLL_P, MC_PITCH_P, MPC_XY_P)을 최적화(Optimization)해야만 한다.

5. 비행 제어기 튜닝 시 모범 사례(Best Practices)

스텝 응답 곡선을 분석할 때, 빠른 반응 속도(T_r 감소)와 진동의 억제(\%OS 감소) 간의 트레이드오프(Trade-off)를 극복하기 위해 제어 공학의 이론적 배경에 근거하여 파라미터를 수정하라.

  1. 반응이 지나치게 느린 경우 지연 제어 문제(Sluggish Response): 비례 게인(MPC_XY_P, MC_ROLL_P)을 점진적으로 증가시키며, 상승 시간을 모니터링한다. 시스템이 과소제동(Under-damped) 상태로 전환되기 전까지 최적화할 수 있다.
  2. 오버슈트 및 진동(Ringing)이 과도한 경우: 미분 게인(MC_ROLL_D)을 추가하여 시스템 내 감쇠비(Damping Ratio)를 높여라. 진동을 물리적으로 억제하는 브레이크 역할을 수행한다. 그러나 D 게인의 과도한 증가는 센서 노이즈를 증폭시켜 모터 과열을 초래할 수 있음에 극히 주의해야 한다.
  3. 임펄스 주입 시 영구적 오프셋 잔존: 외부 바람과 같은 지속적 교란(Disturbance) 요소가 제거되었는데도 기체가 목표 위치에 복귀하지 못하고 영구적인 정상 상태 오차(Steady-State Error)를 남긴다면, 적분 게인(MC_ROLL_I, MPC_XY_I)을 투입하여 오차를 시간 누적 계산으로 보상해야 한다.

요약하건대, PX4-Autopilot를 사용하는 동역학 제어 설계자는 반드시 통제된 방식의 임펄스 외란 및 스텝 입력 실험을 통해 시스템의 폐루프 제어 특성을 근본적으로 이해하고, 안정도 한계를 평가하는 정량적인 소프트웨어 검증 체계를 확립해야만 한다. 이는 관련된 유명 학술 문헌 “Robust Flight Control System Design for Unmanned Aerial Vehicles“의 이론적 근거와 궤를 같이 한다.
