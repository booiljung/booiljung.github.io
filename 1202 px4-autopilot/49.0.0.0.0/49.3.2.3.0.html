49.3.2.3 구면 선형 보간(Slerp)을 이용한 궤적점 사이의 부드러운 자세 천이

1. 개요 및 필요성

PX4-Autopilot(자율주행 펌웨어, 기준 버전: v1.14 이상)에서 무인 항공기(UAV)가 궤적점(Waypoint) 사이를 비행할 때, 기체의 자세(Attitude)를 부드럽게 천이하는 것은 제어의 안정성과 비행 효율을 결정짓는 핵심 요소이다. 기체가 한 궤적점에서 다른 궤적점으로 이동할 때 단순히 Euler 각(Euler Angle)을 선형 보간(LERP, Linear Interpolation)할 경우, 회전 속도가 일정하지 않거나 짐벌 락(Gimbal Lock) 현상에 빠져 기체가 비정상적인 거동을 보일 위험이 존재한다.

이를 해결하기 위해 단위 쿼터니언(Unit Quaternion) 대수 체계를 활용한 구면 선형 보간(Slerp: Spherical Linear Interpolation) 기법이 적용된다. Slerp는 두 쿼터니언 사이를 사원수 공간의 단위 구면(Unit Sphere)을 따라 최단 경로로 보간하며, 각속도(Angular Velocity)를 일정하게 유지할 수 있다는 수학적 장점을 제공한다. 이는 ArduPilot의 자세 제어 아키텍처에서도 유사하게 다루어지나, PX4의 경우 모듈화된 행렬 연산 라이브러리(lib/matrix)를 통해 직관적이고 일관된 Slerp 연산을 수행하는 특징을 지닌다.

2. 구면 선형 보간(Slerp)의 수학적 원리

Slerp는 1985년 Ken Shoemake의 논문 “Animating Rotation with Quaternion Curves“에서 컴퓨터 그래픽스 및 애니메이션 분야의 회전 보간법으로 널리 알려졌으며, 현재 항공우주 및 로보틱스 분야의 자세 제어에 필수적으로 사용된다.

두 단위 쿼터니언 q_1과 q_2가 주어졌을 때, 보간 매개변수 t \in [0, 1]에 대한 Slerp 함수는 다음과 같이 정의된다.

\text{Slerp}(q_1, q_2, t) = \frac{\sin((1 - t)\Omega)}{\sin(\Omega)} q_1 + \frac{\sin(t\Omega)}{\sin(\Omega)} q_2

여기서 \Omega는 q_1과 q_2 사이의 각도를 의미하며, 이는 두 쿼터니언의 내적(Dot Product)을 통해 구한다.
\cos(\Omega) = q_1 \cdot q_2

만약 두 쿼터니언의 내적이 음수(q_1 \cdot q_2 < 0)인 경우, 보간 경로가 구면 상에서 더 긴 경로(Longest Path)를 택하게 되므로, 이를 방지하기 위해 q_2의 부호를 반전(-q_2)시켜 최단 경로(Shortest Path) 보간을 수행해야 한다. PX4-Autopilot의 내부 알고리즘은 이러한 예외 처리를 엄격히 수행한다.

3. PX4-Autopilot 아키텍처 내 Slerp 구현 및 활용

PX4 시스템에서 Slerp는 기체의 자세 지령(Attitude Setpoint)을 생성하고 추종하는 제어 로직 전반에 걸쳐 사용된다. 특히 실시간 제어(Offboard 모드)나 자율 비행(Mission 모드) 중 웨이포인트(Waypoint) 간의 자세 천이를 계산할 때 핵심적인 역할을 담당한다.

graph TD;
    A[이전 궤적점 자세 쿼터니언 q1] --> C{내적 연산 및 경로 최적화}
    B[목표 궤적점 자세 쿼터니언 q2] --> C
    C --> |q1 * q2 < 0 이면 부호 반전| D[Slerp 수학적 연산 수행]
    D --> |t 파라미터 업데이트| E[보간된 실시간 자세 쿼터니언 q_t 생성]
    E --> F[Attitude Controller 지령 전달]
    F --> |PID 제어 루프| G[Rate Controller 및 Actuator 출력]

소스 코드 관점에서, PX4는 자체 의존성을 갖춘 외부 라이브러리인 Matrix(src/lib/matrix/matrix/Quaternion.hpp) 내에 Slerp 함수를 정의하여 범용적으로 사용한다.

예를 들어, 경로 추종(Path Following) 및 궤적 생성 모듈인 FlightTask 및 PositionControl 아키텍처에서는 다음과 같은 형태로 Slerp를 호출하여 지령(Setpoint)을 부드럽게 생성한다.

// PX4 Matrix 라이브러리의 Slerp 알고리즘 개념적 예시
matrix::Quatf q_current(current_attitude);
matrix::Quatf q_target(target_attitude);

// t는 0.0 에서 1.0 사이의 보간 계수 (시간 경과에 비례)
float t = compute_interpolation_weight(dt);

// Slerp 연산을 통한 목표 자세 쿼터니언 계산
matrix::Quatf q_setpoint = q_current.slerp(t, q_target);

이 연산을 통해 도출된 q_setpoint는 vehicle_attitude_setpoint uORB 토픽(Topic)으로 발행(Publish)되며, 자세 제어기(Attitude Controller)에서 즉각적으로 추종하게 된다.

4. 특이점 해소와 제어의 안정성 (Ardupilot 대비 시사점)

Euler 각을 사용할 경우 Pitch 각도가 ±90도에 도달했을 때 Roll과 Yaw의 회전축이 겹치는 짐벌 락(Gimbal Lock) 현상이 발생한다. 이는 고기동(Acrobatic) 비행이나 급격한 궤적 변경 시 제어기 발산을 초래할 수 있다. PX4-Autopilot은 제어계의 기준 좌표를 단위 쿼터니언으로 일원화하고, 모든 자세 천이에 Slerp 연산을 적용함으로써 이러한 특이점(Singularity)을 원천적으로 차단한다.

Ardupilot 또한 내부적으로 쿼터니언 기반의 제어를 수행하지만, PX4는 FlightTask 설계 철학을 통해 Offboard 궤적 추종 단계부터 철저히 Slerp 궤적 생성(Trajectory Generation)을 독립 모듈화한 점에서 차별화된 안정성을 제공한다. 결과적으로 궤적 전이(Transition) 구간에서 모터(Actuator)로 전달되는 비연속적인 각속도 스파이크를 효과적으로 감쇠시켜 시스템의 기계적 무리를 줄이고 전력 효율성을 향상시킨다.

5. 요약 및 시사점

구면 선형 보간(Slerp)은 PX4-Autopilot이 고도의 비행 안정성을 달성하기 위한 수학적 토대 중 하나이다. 공학도로서 ROS2 미들웨어나 MAVLink를 통해 직접 계산된 자세 지령(Attitude Setpoint)을 Offboard로 인가할 기회가 있다면, 외부 애플리케이션(GCS 또는 Companion Computer) 수준에서도 단순히 값의 점프를 인가하는 대신 Slerp된 부드러운 쿼터니언 곡선을 기체에 제공해야 최적의 제어 응답성을 확보할 수 있다. 이는 복잡한 로보틱스 시스템 제어 설계에서 일관된 일정한 각속도 곡선을 담보하는 가장 신뢰할 만한 방론이다.
