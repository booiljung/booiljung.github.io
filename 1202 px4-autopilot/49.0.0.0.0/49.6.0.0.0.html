49.6 실시간 궤적 생성(Trajectory Generation) 및 최적 제어(Optimal Control) 알고리즘 적용

멀티로터 비행체의 자율 비행(Autonomous Flight)과 Offboard 모드 기반 병렬 제어 환경에 있어서, 지상 통제소(GCS) 또는 온보드 컴패니언 컴퓨터(Companion Computer, 예: ROS2 노드)가 단순히 단일 ’목표 좌표(Setpoint)’를 즉각적으로 하달하는 방식만으로는 매끄럽고 물리적으로 추종 가능한 비행을 보장할 수 없다. 비행체는 관성(Inertia)과 공기역학적 항력을 지닌 물리적 역학 시스템이므로, 급격한 지령의 계단 함수적 변화(Step Input)는 심각한 오버슈트(Overshoot), 기체 진동, 그리고 모터 액추에이터 포화(Actuator Saturation)를 유발한다. 따라서 출발점에서 목적지까지 이동할 때, 기체의 물리적 가속/감속 한계 내에서 실시간으로 연속적인 곡선 궤적(Continuous Curve Trajectory)을 생성하고, 이를 정교하게 추종하기 위한 궤적 제어(Trajectory Control) 기법을 설계하는 과정이 필수적이다. 이 절에서는 PX4-Autopilot의 실시간 궤적 생성 프레임워크와 최적 제어 기반의 확장 가능성에 대하여 심층적으로 분석한다.

1. 궤적 생성 알고리즘의 필요성과 동역학적 수학 모델링

로봇 및 항법 제어 공학에서 궤적 통제는 단순한 경로(Path, 기하학적 3차원 공간상의 연결 지점)를 넘어서, 시간 t에 따라 변위계 변수인 위치, 속도, 가속도, 그리고 저크(Jerk, 가속도의 시간에 대한 1계 미분)를 동기화하여 연속적인 다항식 함수(Polynomial Function)들을 정의하는 과정이다.

PX4-Autopilot의 위치 제어기 아키텍처는 내부적으로 위치, 속도, 가속도를 캐스케이드(Cascade) 형태로 제어한다. GCS에서 웨이포인트를 전송하거나 ROS2 노드에서 목적지를 MAVLink SET_POSITION_TARGET_LOCAL_NED 메시지로 10Hz 이상의 주기로 지속 전송할 때, PX4 내부에 상주하는 FlightTask 모듈은 이를 즉각적인 도착지로 직접 수용하지 않고, 운동학적 제약을 만족하는 스무딩(Smoothing)된 중간 궤적으로 분해한다.

이를 수학적인 최적화 문제로 도식화하면, 부드러운 승차감(카메라 영상 흔들림 최소화 등)을 제공하기 위한 기준 추종 궤적 r(t)는 대부분 최소 저크 궤적(Minimum Jerk Trajectory) 또는 최소 스냅 궤적(Minimum Snap Trajectory) 최적화로 정의된다.

\min \int_{t_0}^{t_f} \left\Vert \frac{d^k r(t)}{dt^k} \right\Vert^2 dt

일반적으로 멀티로터의 제어 미분 차수 k를 3(저크 최소화) 또는 4(스냅 최소화)로 설정하여 목적 함수를 구성한다. 최신의 PX4 펌웨어(v1.14 기준)에서는 탑재된 플라이트 컨트롤러(Flight Controller)의 연산 오버헤드를 방지하면서도 실시간성을 극대화하기 위해, 순수 다항식 행렬 연산의 역행렬을 계산하는 대신 운동학적 필터와 Slew-rate 기반의 효율적인 궤적 생성기를 운용한다.

2. PX4-Autopilot의 실시간 궤적 생성기 아키텍처 구조

PX4 펌웨어의 제어 소스 코드에서는 FlightTaskAlgorithm이라는 최상위 C++ 추상화 가상 클래스를 상속받아, 비행 환경 및 모드에 최적화된 다양한 독립적 비행 테스크(예: FlightTaskAuto, FlightTaskOffboard, FlightTaskManualPosition)가 각자의 궤적 생성을 관장하도록 모듈화되어 있다.

작업 공간 내에서 자동 비행(Mission)이나 컴퓨터 비전 기반 Offboard 회피 기동 시 적용되는 실시간 궤적 스무딩 기술은 크게 다음의 물리적 클리핑과 필터 체인으로 달성된다.

2.1. 저크-제한 궤적 생성 로직 (Jerk-limited Trajectory)

운용자가 QGroundControl 등의 GCS를 통해 파라미터 MPC_JERK_MAX, MPC_JERK_AUTO 등을 통해 기체가 허용할 수 있는 최대 저크 한계값을 지정하면, PX4는 내부적으로 위치-속도-가속도-저크로 이어지는 3차 역학 프로파일을 계산한다. 이러한 제한은 이륙 직후나 웨이포인트 통과 시 방향 전환 시에 발생하는 피치/롤 회전 각도의 급격한 출렁임을 강력히 억제하여 탑재 페이로드(예: 레이저 스캐너, 짐벌 카메라)의 계측 정숙성을 대폭 향상시킨다.

2.2. 변화율(Slew-rate) 한계 필터를 통한 외부 스트림 스무딩

ROS2의 uXRCE-DDS 에이전트 브리지를 거쳐 외부에서 매 50Hz 이상의 고주파수 속도 지령 제어 명령(Velocity Setpoint)이 입력되는 Offboard 제어 상황의 경우, 네트워크 통신 지연 지터(Jitter)나 연산 노드의 주기 불균일성에 의해 지령 페이로드 중간중간에 수학적 노이즈나 스파이크 신호가 유입되기 마련이다. 이를 방어하기 위해 PX4는 소프트웨어 내부적으로 SlewRate 제약 클래스를 사용하며, 가속도 한계(MPC_ACC_HOR, MPC_ACC_UP 등) 물리량 내에서 속도 지령의 순간 델타 변화율을 한계치 상한으로 동적 클리핑(Dynamic Clipping)한다.

3. 선형 이차 레귤레이터(LQR) 및 최적 제어기 도입 패러다임

현재 PX4-Autopilot 공식 배포 펌웨어의 코어 멀티로터 항법 제어 루트는 피드포워드(Feedforward) 이득을 포함시킨 발전된 형태의 비례-적분-미분(PID) 위치/속도 제어 모델을 채택하고 있다. 그러나 군집 비행, 곡예 비행, 장애물 밀집 지역에서의 드론 레이싱 등 임무 난이도가 고도화됨에 따라 학계와 산업계에서는 PX4의 코어 아키텍처 위에 최적 제어 기반의 선형 이차 레귤레이터(LQR, Linear Quadratic Regulator) 위치 제어나, 예측 윈도우 기반 모델 예측 제어(MPC, Model Predictive Control)를 이식하여 결합하려는 응용 사례가 쇄도하고 있다.

LQR 제어 공법은 시스템 내 에너지 소비와 목표 위치 추종 오차 사이의 모순되는 상충 관계(Trade-off)를, 다음과 같은 비용 함수(Cost Function) J를 누적 최소화하는 상태 공간 피드백 이득 행렬 K를 찾는 대수학적 과정으로 귀결시킨다.

J = \int_{0}^{\infty} \left( x^T Q x + u^T R u \right) dt

이때 시스템 행렬 공간 내 상태 벡터 x는 모델화된 위치와 속도의 궤적 오차이며, 제어 입력 변수 u는 가속도 혹은 물리적 추력 지령 벡터이다. 상태 가중 행렬 Q 행렬과 입력 가중 행렬 R 행렬의 계수를 튜닝함으로써, ’배터리를 최대한 아끼는 경제적 비행 곡선’을 생성할 것인지 혹은 ’액추에이터를 혹사하더라도 오차를 제로(Zero)에 가깝게 유지하는 공격적 비행’을 할 것인지를 설계자 의도에 맞게 수리적으로 세팅할 수 있다.

뛰어난 개방성을 지닌 PX4의 소프트웨어 소스코드 계층 구조는 위치 제어 파이프라인(PositionControl.cpp) 인터페이스 클래스를 교체하는 것만으로 손쉽게 외부 리서치 제어기를 주입할 수 있도록 고도의 모듈화를 지원한다. 관련 저명 논문인 “Aggressive Flight of Quadrotors using Model Predictive Control“의 연구사례와 같이, 무거운 최적 설계 연산은 컴패니언 컴퓨터의 노드 상에서 계산하고 산출된 제어 입력 궤적(Optimal Control Variables)들만을 uXRCE-DDS 프로토콜로 직접 스트리밍 및 주입하는 Offboard 분산 처리 아키텍처도 대단히 실용적으로 응용된다.

2. Ardupilot Copter 궤적 제어 모델과의 철학적 차분

Ardupilot은 전통적이고 검증된 항법 분야에 뿌리를 두고 있는 만큼, S-Curve 프로파일(Kinematic SCurve Path Planner)과 내비게이션 컨트롤러 모듈을 장기간에 걸쳐 신뢰성 위주로 고도화시켜 왔다. 특히 항공 매핑, 산업용 자동 방제선 등 안정성에 초점을 맞춘 시장에 타겟팅하여 웨이포인트들 사이를 이동하거나 출발, 제동할 시의 가감속 여유 곡선을 미려하게 설계하여 영상 탑재체(Gimbal) 흔들림 없는 미드-엔드(Mid-end)급 상업 시장의 표준으로 자리매김했다.

반면 PX4-Autopilot은 이보다 초기 시스템 도약 시점부터 ETH Zurich의 학술적 백그라운드를 흡수하면서, ’고정익 경로 추종’을 넘어서 ’멀티 모터 동역학의 직접 제어’를 핵심 철학으로 삼아 진화해 왔다. 위치 지령 단계부터 저크 클리핑 제한을 상태 머신 수학 모델에 체계적으로 매핑하였고, 특히 ROS/ROS2 및 최신 외부 센서 퓨전(SLAM/VIO) 환경의 로봇 운영 체제 생태계로 통합되는 과정을 우선시하여 개발되었다. 예측할 수 없는 궤적을 뿜어내는 ’고도 개방형 자율 에이전트 로봇(Autonomous Agent Robot)’으로서의 극도의 실시간 지령 수용성에 초점을 맞춤으로써, 외부 비전 기반 회피 시스템이나 외부 MPC 제어기의 지휘를 보다 지연 없이 동적으로 받아들이는 인터페이스 수용 구조가 돋보인다.
