49.3.2 비특이 자세 표현법(Non-singular Attitude Representation) 적용

무인항공기(UAV)의 자세(Attitude)를 표현하는 방법은 비행 제어 및 동역학 해석에서 핵심적인 요소이다. 일반적으로 직관적인 이해를 돕기 위해 롤(Roll), 피치(Pitch), 요(Yaw)로 구성된 오일러 각(Euler Angle)이 널리 쓰이지만, 특정 자세에서는 짐벌 락(Gimbal Lock)이라는 수학적 특이점(Singularity)이 발생하여 제어의 연속성을 잃게 된다. 이를 극복하고 전 방위각에서 안정적인 제어를 수행하기 위해 PX4-Autopilot(주로 v1.14 최신 펌웨어 기준)에서는 단위 쿼터니언(Unit Quaternion)을 활용한 비특이 자세 표현법(Non-singular Attitude Representation)을 내부 제어 및 추정의 근간으로 적용하고 있다.

1. 특이점(Singularity)과 짐벌 락(Gimbal Lock)의 이해

오일러 각을 사용하여 3차원 회전을 매개변수화(Parameterization)할 때, 피치 각이 \pm 90^\circ 에 근접하면 롤 축과 요 축이 기구학적으로 겹쳐 구분이 불가능해지는 현상이 발생한다. 이를 짐벌 락이라고 한다.

예를 들어 Z-Y-X 오일러 동역학 회전 순서를 따를 때 피치(Y축 회전)가 정확히 90^\circ가 되면, 첫 번째 회전축(Z축)과 세 번째 회전축(X축)의 방향 벡터가 평행선상에 놓이게 된다. 이는 시스템의 자유도(Degree of Freedom) 하나가 상실됨을 의미하며, 제어기(Controller)의 자코비안(Jacobian) 행렬이 비가역적(Non-invertible) 특성을 띠어 역기구학이나 미분 제어 행렬 계산 시 발산(Divergence)을 초래한다. 고기동 멀티로터나 고정익 기체에서 이러한 특이점 제어 불능은 기체 추락과 같은 치명적인 상태를 유발할 수 있다.

2. 단위 쿼터니언(Unit Quaternion) 기반의 비특이 표현법

PX4-Autopilot은 이와 같은 근본적인 한계를 해결하고자 내부적으로 모든 3차원 회전 군(SO(3), Special Orthogonal Group) 연산에 단위 쿼터니언을 적용한다.

단위 쿼터니언은 4개의 매개변수 \mathbf{q} = [q_0, q_1, q_2, q_3]^T 로 구성되며, \|\mathbf{q}\| = 1 의 구속조건(Constraint)을 갖는다. 이는 3차원 자세 공간의 비특이 전역 매개변수화(Global Non-singular Parameterization)를 가능하게 하며, 다음과 같은 장점을 지닌다.

  1. 특이점 완전 제거: 오일러 각에서 나타나는 짐벌 락 좌표계 특이점이 존재하지 않는다. 아크로바틱(Acrobatic) 비행이나 큰 롤, 피치 기동을 수행할 때 연속적이고 안정적인 제어가 보장된다.
  2. 연산 효율성 향상: 회전 행렬(Rotation Matrix)은 9개의 원소를 가지며 행렬 곱을 요구하지만, 쿼터니언은 4개의 요소만을 사용하며 삼각함수 연산을 완전히 배제하므로 임베디드 환경(NuttX RTOS 상의 마이크로컨트롤러)에서 연산 부하가 대폭 감소한다.
  3. 자세 오차(Attitude Error) 궤적 최적화: 구면 선형 보간(Slerp, Spherical Linear Interpolation)을 통해 최단 경로의 회전 각 분산을 유도하므로 부드러운 자세 천이 역학 특성을 이끌어낸다.

3. PX4-Autopilot 소스 코드 내 자세 제어 적용 사례

PX4의 소스 코드 구조를 살펴보면, 내부 행렬 라이브러리(px4/Matrix)에서 쿼터니언 자료구조와 연산을 일관되게 제공한다. 멀티콥터 자세 제어기 모듈인 mc_att_control (MulteCopter Attitude Control) 등에서 이 비특이 자세 표현법이 직접적으로 활용된다.

PX4의 제어기는 원하는 지령 자세(Target Attitude Setpoint) \mathbf{q}_{sp} 와 현재 추정된 자세 \mathbf{q} 사이의 오차 성분을 새로운 쿼터니언 \mathbf{q}_{error}로 산출하여 최적의 각속도 지령을 생성해 낸다. 오차 쿼터니언은 아래와 같은 전역 켤레곱(Conjugate Multiplication)으로 계산된다.

\mathbf{q}_{error} = \mathbf{q}^{-1} \otimes \mathbf{q}_{sp}

이때 발생한 오차 쿼터니언 \mathbf{q}_{error}의 벡터부(Vector Part)를 추출하면, 각 축(Roll, Pitch, Yaw)에 대한 최단 거리 각도 피드백 신호로 사용할 수 있어 직관적인 P-PD 제어루프 구성을 돕는다. 선형대수 및 동역학 이론을 기반으로 한 이러한 접근 방식은 논문 “A Quaternion-based Feedback Control for UAV” 구조적 아이디어와 등가성을 지닌다.

4. 아키텍처 및 시스템 상호 작용

PX4의 모듈 간 통신 시스템인 uORB 메시지 처리나, Ardupilot과 비교되는 관제 인터페이스에서의 자세 처리도 단위 쿼터니언을 주로 채택한다. 특히 vehicle_attitude와 vehicle_attitude_setpoint 토픽(Topic) 페이로드는 쿼터니언 배열 값을 그대로 전달한다.

    A[GCS QGroundControl / MAVLink] -->|MAVLink ATTITUDE_TARGET<br>Quaternion & Thrust| B(MAVLink Receiver<br>PX4 모듈)
    B -->|vehicle_attitude_setpoint<br>q_sp 발행| C[mc_att_control<br>자세 제어기]
    D[EKF2 Estimator<br>확장 칼만 필터] -->|vehicle_attitude<br>q 발행| C
    C -->|q_error 계산 및 각속도 P 제어| E(vehicle_rates_setpoint)
    E --> F[mc_rate_control<br>각속도 PID]

위의 Mermaid 아키텍처 다이어그램에서 볼 수 있듯, GCS인 QGroundControl(버전 4.3 이상) 측에서 사용자 편의를 위해 오일러 각으로 지령을 시각화하고 사용자 입력을 받더라도, MAVLink 통신 과정에서는 ATTITUDE_TARGET 메시지에 쿼터니언 배열로 변환되어 전달된다. PX4 펌웨어는 수신 즉시 역변환 없이 쿼터니언을 uORB 토픽으로 배포(Publishing)하고 자세 제어기(Attitude Controller) 및 이중 루프(Double Loop)의 각속도 제어기로 피드백을 구성한다.

Ardupilot과의 철학 차이

Ardupilot 역시 동일하게 내부 추정기에 EKF 라이브러리를 사용하며 기초 모델링에 단위 쿼터니언을 도입하고 있다. 하지만 센서 계측치 갱신이나 비행 모드에 따른 회전 처리 과정에서 코드 베이스 상 오일러 회전 변환 유틸리티를 병용하는 빈도가 상대적으로 높은 편이다. 반면 PX4는 최신 Matrix 라이브러리로 전면 통폐합을 진행하면서 시스템 아키텍처 하부 전반에서 쿼터니언 기반 추정 및 제어를 획일화하였으며, 이는 수리적 무결성(Mathematical Integrity)과 코드 일관성 측면에서 모던 소프트웨어 공학의 기법을 보다 적극적으로 수용한 결과로 평가할 수 있다.

5. 결론

무인 기체의 동역학을 오차 없이 제어하기 위해 PX4-Autopilot은 짐벌 락 현상을 원천 차단하는 비특이 자세 표현법인 단위 쿼터니언을 필수적으로 적용한다. 이는 단순한 이론적 대수에 그치지 않고, EKF2 기반의 혼합 시스템(Sensor Fusion), 내부 uORB 통신 데이터 구조, 역학 제어기, 그리고 QGroundControl과의 MAVLink 프로토콜 교환 등 시스템 아키텍처 전체에 걸쳐 긴밀하게 스며들어 있다. 이와 관하여 실무 소프트웨어 엔지니어나 연구자는 기체의 비행 양상을 면밀히 분석하고 확장 애플리케이션을 개발할 때 오일러 각 중심 사고방식에서 벗어나 기구학적 특이점이 배제된 SO(3) 쿼터니언 공간 매핑을 명확히 이해해야만 한다.
