49.2.3 비연속적 제어 지령 입력 시의 과도 상태(Transient State) 감쇠 및 범플리스 트랜스퍼(Bumpless Transfer)

1. 개요 및 제어 공학적 위험성

무인항공기(UAV)의 제어 과정, 특히 Offboard 자율 비행 모드나 관제 시스템(GCS)으로부터 명령을 수신하는 상황에서는 제어 지령(Setpoint)이 연속적이지 않은 계단형 입력(Step Input)으로 주어지는 경우가 잦다. 예를 들어, 기체가 현재 (0, 0, 0)에 호버링하고 있는데 지상 관제소가 목적지를 갑자기 (100, 100, -50)으로 하달한 경우가 그러하다.

드론과 같은 고기동성 동적 시스템(Dynamical System) 비행 제어기(PID) 내부로 이러한 불연속적(Discrete)인 에러(Error)가 여과 없이 유입되면,

  1. 제어기 포화(Control Saturation): 액추에이터(모터)에 한계치 이상의 출력이 요구되어 제어력을 상실한다.
  2. 미분 폭주(Derivative Kick): 오차의 급격한 변화율에 의해 PID의 미분(D)항 파라미터가 비정상적인 스파이크(Spike)를 일으켜 기체가 극심하게 진동한다.
  3. 과도 상태(Transient State): 최종 정상 상태(Steady State)에 도달하기 전 기동이 불안정해지고, 예측 불가능한 경로 이탈 현상을 초래한다.

PX4-Autopilot은 이러한 파멸적인 물리 엔진의 붕괴를 막기 위해 궤적 평활화(Trajectory Smoothing) 및 범플리스 트랜스퍼(Bumpless Transfer) 알고리즘을 시스템 기저(Architecture)에 핵심적으로 탑재하고 있다.

2. 궤적 평활화 및 저크 제한(Jerk Limiting) 아키텍처

외부에서 불연속적인 목적지나 속도 지령이 하달되면, PX4의 비행 제어 스택(특히 FlightTask 모듈 내부)은 이 목표 지점을 즉각적인 위치 에러(Position Error)로 사용하지 않는다. 대신, 목표점까지 부드럽게 이어지는 ’가상의 마이크로 세트포인트(Micro-setpoints)’를 실시간으로 파생시켜 제어기로 전달하는 경로 생성(Trajectory Generation) 단계를 반드시 거친다.

2.1 승차감과 기계 피로도 조절 (Motion Constraints)

이를 구현하기 위해 사용되는 것이 기구학적 제약 조건(Kinematic Constraints)이다.
PX4는 파라미터를 통해 다음과 같은 물리적 한계를 부여한다.

  • 최대 속도 (MPC_XY_CRUISE, MPC_Z_VEL_MAX_UP 등)
  • 최대 가속도 (MPC_ACC_HOR_MAX, MPC_ACC_UP_MAX 등)
  • 최대 저크 (MPC_JERK_MAX, MPC_JERK_AUTO 등): 가속도의 변화율(Jerk)을 제한함으로써 기체가 덜컹거리는 현상(Surge)을 억제한다.

계단형 세트포인트가 주입되면, 내부 궤적 생성기는 위 제약 조건을 기반으로 속도 사다리꼴(Trapezoidal) 또는 S-Curve(Jerk-limited) 형태의 부드러운 가상 궤적을 250Hz의 제어 루프 주기에 맞추어 촘촘하게 쪼개어(Interpolation) 하위 제어기(Position Controller)에 공급한다.

3. 범플리스 트랜스퍼 (Bumpless Transfer) 메커니즘

제어 지령의 급변은 ‘비행 모드 변경(Mode Transition)’ 시에도 빈번하게 일어난다. 고속 전진 비행 중 Manual(수동) 모드에서 Position(위치 고수) 모드로 전환하거나, Failsafe 발동으로 인해 Offboard 모드가 RTL 모드로 덮어씌워질 때가 대표적이다.

**범플리스 트랜스퍼(Bumpless Transfer)**란, 제어기의 전환 또는 세트포인트의 스위칭 발생 시 출력단(예: 롤/피치 자세 지령 및 모터 추력)에 충격(Bump)이 발생하지 않도록 제어기의 내부 상태(Internal States)를 조화시키는(Aligning) 제어 공학의 기법이다. 논문 “Bumpless Transfer Control for Unmanned Aerial Vehicles” 시스템 분과에서 이 방식의 유효성이 흔히 검증되곤 한다.

3.1 PX4의 실제 구현 및 적분항 재초기화

모드 전환이 발생하는 순간, PX4 시스템 내부에서는 다음과 같은 동기화 작업이 일어난다.

  1. 적분기 조정(Integrator Alignment): 이전 제어 모드에서 쌓여 있던 PID 적분기(I-term, 주로 바람에 대항하던 트림 값)가 스위칭 시 그대로 유지되거나, 새로운 모드의 출력 계산식이 이전 비행 모드의 찰나의 순간과 동일한 결과값을 내도록 적분기 초기값을 역산하여 덮어쓴다. (이를 통해 적분기 와인드업 풀림 현상으로 인한 기체의 뒤틀림을 최소화)
  2. 현재 상태의 상속(State Inheritance): FlightTask 객체가 소멸하고 새로운 객체가 생성(전환)될 때, EKF2로부터 산출된 현재 기체의 실제 속도(Actual Velocity)와 가속도를 새로운 궤적의 ‘초기 속도’ 조건으로 그대로 상속받는다. 따라서 급발진이나 급정거를 막을 수 있다.

4. Ardupilot 대비 아키텍처적 우월성

Ardupilot은 C++ 구조상 레거시가 긴 절차적(Procedural) 제어 분기가 혼재된 경향이 있으나, PX4의 제어 계층은 보다 객체 지향적(Object-Oriented)이다. PX4의 src/modules/flight_mode_manager/tasks/FlightTask.cpp 기반 상속 구조는, 어떤 커스텀 비행 모드가 추가되더라도 모든 모드 클래스가 최상위 부모 클래스인 FlightTask를 상속하게 함으로써 강제적인 상태 동기화(State Synchronization) 규약을 제공한다.
즉, 개발자가 의식하지 못하더라도 PX4 프레임워크 자체가 과도 상태 감쇠를 위한 초기화 인터페이스(activate(), update())를 필수적으로 거치게 하므로 안전성이 구조적으로 담보된다.

5. 결론 및 GCS 관제 실무에서의 응용

GCS 개발자나 자율 임무 설계자가 MAVSDK/ROS2를 통해 기체를 제어할 때 명심해야 할 점은, 굳이 자신들의 관제 소프트웨어 내부에서 복잡한 Spline 곡선 보간법(Interpolation)을 구현하여 기체로 조밀하게 쏴주기 위해 CPU 자원을 낭비할 필요가 “상대적으로” 적다는 것이다.

PX4-Autopilot의 범플리스 트랜스퍼 및 저크 제한 궤적 생성기는 매우 강력하다. 단순히 공간상 동떨어진 타겟 좌표(Target Point)만 던져주더라도(Sparse Setpoint), 비행 제어기가 스스로 현재 동역학 한계(MPC 파라미터)를 계산하여 덜컹임 없이 매끄러운 상승 및 가속을 알아서 처리하기 때문이다. 이는 중앙 집중 관제 위주의 Ardupilot(Mission Planner) 체계에 비해, 분산 에이전트 연산의 짐을 덜 수 있는 PX4만의 뛰어난 비행 제어 계층의 장점이다.
