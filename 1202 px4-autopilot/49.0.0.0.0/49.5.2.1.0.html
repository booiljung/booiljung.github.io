49.5.2.1 위치, 속도, 가속도 벡터의 상호 배타적 선택 및 결합(Blending) 필터 구조

비행 제어 컴퓨터(FCC)가 GCS나 동반 컴퓨터(Companion Computer)로부터 전달받은 SET_POSITION_TARGET_LOCAL_NED 메시지의 다중 목표 변수들은, 앞서 살펴본 비트마스크 필터를 거쳐 trajectory_setpoint uORB 토픽으로 변환된다. 이렇게 정제된 3차원 위치(Position), 속도(Velocity), 가속도(Acceleration) 벡터 세트는 PX4-Autopilot의 위치 제어 모듈(Position Controller)인 FlightTaskOffboard 클래스에 유입된다. 본 절에서는 이러한 다중 제어 벡터들이 독립적으로 어떻게 배타적(Mutually Exclusive)으로 선택되거나, 상호 유기적으로 결합(Blending)되어 최종 자세 제어(Attitude Control) 루프로 하달되는지 그 핵심 필터 구조와 동역학적 메커니즘을 상세하게 분석한다.

1. 3차원 벡터 지령의 상호 배타적 선택 (Mutually Exclusive Selection)

가장 기본적인 Offboard 제어 전략은 한 번에 단일 계층의 제어 변수만 제어 루프에 관여하도록 하는 것이다. PX4-Autopilot의 캐스케이드(Cascade) 제어 구조 아래에서 위치, 속도, 가속도의 상호 배타적 제어 분기는 부동소수점 예외 문자인 무효값 NAN의 스위칭 메커니즘을 통해 구현된다.

flowchart TD
    A[uORB Topic: trajectory_setpoint 수신] --> B{NAN 판별 로직 적용}
    B -- 위치 정상, 속도/가속도 NAN --> C[위치 제어 모드 활성화<br>Position P-Controller 작동]
    B -- 위치 NAN, 속도 정상, 가속도 NAN --> D[속도 제어 모드 활성화<br>외곽 루프 우회하여 Velocity Loop 직접 진입]
    B -- 위치/속도 NAN, 가속도 정상 --> E[가속도 제어 모드 활성화<br>동역학 변환 후 근접 자세 추정 반영]

    F --> D
    D --> G[목표 가속도 산출]
    G --> E
    E --> H[자세 및 추력 지령<br>Vehicle Attitude Setpoint 발행]

  • 순수 위치 제어: 위치 백본(Backbone) 외의 속도/가속도를 NAN으로 치환해 무시하면, PX4 위치 제어기(Position P-Controller)가 현재 위치와의 위치 오차(Position Error)에 비례하는 목표 속도를 독자적으로 산출하여 하위 계층에 전달한다.
  • 순수 속도 제어: 위치 제어가 NAN 스위치를 통해 비활성화되면(Bypass 적용), 제어 스택은 위치 오차 계산을 생략한다. 전달된 조종 속도가 즉각적으로 기구학 속도 제어 루프(Velocity PID Loop)로 집접 주입(Direct Injection)되어 빠른 반응성을 확보한다.
  • 순수 가속도 제어 혹은 힘(Force) 제어: 위치와 속도 루프 전체가 무시(Bypass)되며, 제공된 가속도 벡터는 기체의 질량(Mass)과 중력을 보상하는 최종 추력(Thrust) 및 목표 자세(Attitude Setpoint)로 동역학적 변환 과정을 거쳐 최하단 루프에 직결된다. 이 방식은 드론 무리 시스템에서의 충돌 임펄스 회피나 민첩한 곡예비행 제어 논문 등에서 자주 논의되는 제어 기법이다.

2. 제어 벡터의 상호 결합(Blending) 및 피드포워드(Feedforward) 융합 구조

상호 배타적 선택만 이용할 경우, 급격한 지점 이동(Step Input) 명령이 들어왔을 때 드론의 위치 제어 시스템은 커다란 위상 지연(Phase Lag)이나 진동(Oscillation)을 수반하게 된다. 학술적으로 증명된 궤적 추종(Trajectory Tracking) 최적화 로직은 목표 위치와 해당 위치에서의 방향 지시 벡터인 유도 속도를 동시에 MAVLink로 전송하는 하이브리드 결합 방식(Blending)을 권장하고 있다.

PX4-Autopilot의 로직 체계는 위치-속도 P-PID 캐스케이드 루프 앞단에 “피드포워드 결합 필터” 구조를 내재화하고 있다. 이를 수식으로 나타내면 다음과 같다.

최종 목표 속도 \vec{V}_{target}은 위치 오차에 의한 비례항과, 외부 지령으로 수신된 피드포워드 목표 속도 \vec{V}_{ff} 벡터의 합으로 산출된다.
\vec{V}_{target} = K_p \times (\vec{P}_{sp} - \vec{P}_{curr}) \vert + \vec{V}_{ff}

만약 제어명령에 위치 벡터 \vec{P}_{sp} 와 속도 벡터 \vec{V}_{ff} 가 한꺼번에 유효한 값으로 할당되어 들어오게 되면, 위치 에러를 없애고자 하는 피드백 성분과 동반 컴퓨터가 계산한 미래 예상 궤도 속도가 블렌딩된다. 이러한 결합 기법을 통해 제어 정상 상태 오차(Steady-State Error)를 없애고 외란에 대한 추적 능력이 수직 상승한다.

결합 필터에서의 저크(Jerk) 제한 및 프로파일링

단순 선형 덧셈으로 결합될 경우, 지령 벡터의 급등(Step Jump)에 의해 기체가 과도한 피치/롤 앵글을 가질 위험성이 여전히 존재한다. 이 부작용을 억제하기 위해 PX4는 블렌딩되어 산출된 속도 벡터 \vec{V}_{target} 에 대해 속도 한계 제약 필터 및 저크 동역학 클리핑(Dynamic Clipping) 회로를 거친다. 대표적으로 MPC_JERK_MAX 파라미터와 최고 허용 경사각을 연산하여 목표 가속도를 제한하는 S-곡선(S-Curve) 속도 프로파일 스무딩 처리가 적용된 후에 하위 속도/자세 제어기로 통제권이 넘어간다.

3. Ardupilot 대비 결합 메커니즘 아키텍처적 대비

동일한 목표 궤적 추종 알고리즘을 지향하지만, 데이터 결합의 구현 체계 관점에서 두 대표적인 펌웨어는 상이한 접근 철학을 따른다.

  • Ardupilot (Copter PosControl 라이브러리): 외부 속도와 위치가 함께 주입될 때, Ardupilot은 현재 궤적이 갖는 가상의 점 타겟(Leash)을 미래 위치 지령 방향으로 선형 이동시키는 이른바 ’끈 방식의 동역학 모델(Leash-based kinematics)’을 주요 필터로 운용한다. 속도 인자가 타겟 생성기의 보간 제어에 개입하는 이 구조는, 유연한 순항에서 특화되어 있으나 초고주파 업데이트를 요구하는 동적 환경에서는 결합 레이턴시에 다소 불리할 수 있다.
  • PX4 (FlightTask Offboard 로직): 입력 변수가 NAN이 아닌 경우 그 즉시 각 단의 피드포워드 채널(Feedforward Channel) 스킵 분기를 열어 직관적이고 매우 수학적으로 공격적(Aggressive) 덧셈 블렌딩 연산을 강제한다. 이는 드론 레이싱 경기나 장애물 회피 같이 미세 단위 초의 제어 결합 요구 환경에서 한계를 확장할 유리한 설계 철학으로 분석된다.

4. 로봇 운영 체제(ROS2)를 활용한 궤적 결합 시스템 구현 예시

ROS2 기반 uXRCE-DDS 퍼블리셔 환경에서는 TrajectorySetpoint 토픽을 배포하여 위치와 속도의 이중 블렌딩 융합 성능을 이끌어낼 수 있다. 다음 파이썬 API 활용 코드는 위치 목표점 좌표와 이상적인 이동 궤적을 제시하는 사전 계산(Pre-calculated) 피드포워드 벡터를 결합 전송하는 예시이다.

from px4_msgs.msg import TrajectorySetpoint
import rclpy

def publish_hybrid_blending_setpoint(publisher, target_x, target_y, target_z, vel_x, vel_y):
    msg = TrajectorySetpoint()
    # 통신 지연시간 분석을 위한 엄격한 타임스탬프 동기화
    msg.timestamp = int(node.get_clock().now().nanoseconds / 1000)
    
    # 1단계: 위치 추종(Feedback)을 위한 좌표 할당 (Local NED계열)
    msg.position = [target_x, target_y, target_z]
    
    # 2단계: 최적 제어기 블렌딩(Feedforward) 작동을 위한 의도된 속도 할당
    msg.velocity = [vel_x, vel_y, 0.0]
    
    # 가속도는 사용하지 않기 위해 부동소수점 무효값(NAN) 설정으로 제어 분기 스킵 명령
    msg.acceleration = [float('nan'), float('nan'), float('nan')]
    
    # 요 각도는 0도로 고정하여 주행 시야 방향성 확보, 각속도는 우회 차단 처리
    msg.yaw = 0.0
    msg.yawspeed = float('nan')
    
    publisher.publish(msg)

위와 같은 복합 지령을 일정 헤르츠(Hz, 예: 50Hz) 단위로 인가시키면 기체 내 탑재 컴퓨터는 수신된 위치 오차를 해결하고자 하는 노력치에 사전 계산된 기준 속도를 합산 필터링하며 비약적으로 우수한 궤도 추종(Tracking) 성취도를 발휘한다.

5. 관제 프로토콜 설계 시 고려 및 주의 사항

이렇듯 다차원 피드포워드 벡터의 상호 배타적 분기 작동과 선형적 블렌딩 체계는 드론 펌웨어 안정성에 근간을 이루는 로직이다. 따라서 GCS 백엔드의 프로토콜 수립 시, 제어기 앞단 부에서 서로 상이한 목적의 상쇄 모순 방정식(예를 들면, 우측 방향의 위치 지령과 반대 방향인 좌측으로 치우친 속도 결합 동시 인가)이 데이터 전송에 섞이게 될 우려를 근절하여야 한다.
오류 데이터가 중첩되거나 논리 결함이 존재하는 비트마스크 필터링이 제어단에 전해졌을 경우, 블렌딩 로직에서 이상 피드백 증폭(Feedback Amplification)이 이뤄지며 비행체는 예측할 수 없는 동역학 발산 상태를 나타낼 것이다. 결과적으로, 동반 센서나 GCS 내부의 비행 승인 아키텍처 단에서 다변수 지령 스위칭의 결함과 벡터 적합성을 사전에 차단 검증하는 페일세이프(Failsafe) 논리가 비행 자율성을 수호하는 방파제라는 사실을 필히 유념해야 한다.
