49.7.3.1 목표 상태 공간(Target State Space)과 추정 상태 공간(Estimated State Space) 간의 L2 노름(L2 Norm) 오차 계산

기체가 3차원 공간 상에서 오프보드(Offboard) 궤적 지령이나 임무 웨이포인트(Waypoint)를 추종할 때, 목표 상태 공간(Target State Space)과 현재 기체의 추정 상태 공간(Estimated State Space) 간의 편차를 직관적이고 공학적인 단일 스칼라(Scalar) 수치로 요약하는 것은 지상 관제 시스템(GCS) 모니터링의 기본이다. 이를 위해 대다수의 비행 제어 스택 및 GCS(QGroundControl 등) 백엔드에서는 L2 노름(L2 Norm) 오차 혹은 유클리드 거리(Euclidean Distance)를 핵심 성능 지표로 채택한다. 본 절에서는 L2 노름 오차의 수학적 정의와 로직, 그리고 PX4-Autopilot 환경에서의 페일세이프(Failsafe) 연계 방식에 대해 상세히 다룬다.

1. 노름(Norm)의 기하학적 의미 및 수학적 정의

제어 공학 및 선형 대수학에서 노름(Norm)은 벡터 공간에서 벡터의 크기나 길이를 측정하는 함수이다. 그중에서도 L2 노름은 우리가 직관적으로 이해하는 물리적 3차원 공간에서의 ’직선 거리’를 의미한다.

1.1 3차원 위치(Position)의 추종 오차 모델링

PX4-Autopilot의 NED(North-East-Down) 직교 좌표계를 기준으로, 시간 t에서의 기체 목표 좌표(Target)를 벡터 \mathbf{P}_{tgt}(t) = [x_{tgt}, y_{tgt}, z_{tgt}]^T, 그리고 EKF2 추정기를 통해 도출된 현재 기체 좌표(Estimated)를 벡터 \mathbf{P}_{est}(t) = [\hat{x}_{est}, \hat{y}_{est}, \hat{z}_{est}]^T라 정의하자.

이때, 3차원 공간상의 위치 잔차 벡터 \mathbf{e}(t)는 다음과 같다.
\mathbf{e}(t) = \mathbf{P}_{tgt}(t) - \mathbf{P}_{est}(t)

이 잔차 벡터의 L2 노름(L2 Norm), 즉 스칼라 형태의 유클리드 오차 D(t)는 벡터의 각 성분의 제곱합에 제곱근을 취한 값으로 정의된다.
D(t) = \Vert \mathbf{e}(t) \Vert_2 = \sqrt{(x_{tgt} - \hat{x}_{est})^2 + (y_{tgt} - \hat{y}_{est})^2 + (z_{tgt} - \hat{z}_{est})^2}

동일한 메커니즘을 3차원 속도 벡터 공간(Velocity State Space)에도 병렬적으로 적용하여 목표 속도와 현재 기체의 속도 추종 오차의 L2 노름을 계산할 수 있다.

2. PX4 제어 루프와 QGroundControl(GCS)에서의 L2 노름 연산

2.1 펌웨어 내부 위치 제어 루프에서의 활용

PX4의 멀티로터 위치 제어 모듈(mc_pos_control) 내부에서는 매 제어 주기마다 위상별 편차를 개별적으로 피드백하는 동시에, 이 L2 노름 스칼라 오차 거리를 사용하여 이륙 지점 복귀나 목표 도달 여부(Acceptance Radius 파라미터 영역 내 진입 여부)를 판별한다. 파라미터 튜닝 시 이 오차 거리가 일관되게 작게 유지된다면 위치 제어기(PID)의 추종 성능(Tracking Performance)이 높은 것으로 평가된다.

2.2 QGroundControl에서의 MAVLink 기반 실시간 시각화

QGroundControl(v4.3 기준)의 C++ 백엔드 로직에서는 기체로부터 MAVLink 네트워크를 통해 초당 수십 회 전달되는 LOCAL_POSITION_NED 메시지와 POSITION_TARGET_LOCAL_NED 메시지를 캡처한다. 백엔드의 다중 스레드 연산부는 이 두 메시지의 데이터들을 앞선 L2 노름 삼각 함수 수식에 대입하여 실시간 추종 오차 D(t)(단위: 미터)를 산출해 낸다. 이후 QML 기반의 프론트엔드 UI를 통해 시계열 꺾은선 그래프 플로팅(Plotting)이나 경고등 형태의 인디케이터로 조종자에게 디스플레이한다.

3. L2 노름 오차 한계(Bounds)기반의 페일세이프(Failsafe) 트리거

무인기 시스템 관제 철학에 있어 L2 노름 오차는 단순히 모니터링 수치를 넘어, 시스템 복구 절차 돌입을 위한 척도(Metric)로 활용된다.

강풍(Wind Gust) 등 불가항력적인 외부 외란이나 모터 추진력의 고장 등으로 인해 기체가 목표 상태를 추종하지 못하고 표류(Drifting)하게 될 경우, 오차 지표 D(t)는 지속적으로 누적 증가하게 된다.

미리 정의된 임계 반경(Threshold Radius)을 D_{max}, 허용 시간을 \Delta T_{allow}라 할 때, 시스템 설계자는 다음과 같은 조건문을 구성할 수 있다.
\text{If } D(t) > D_{max} \text{ for } t_{duration} \ge \Delta T_{allow} \implies \text{Trigger Position Failsafe}

QGroundControl 단에서는 이 조건에 부합할 경우 “Position Tracking Diverged” 경고를 시각적, 청각적으로 송출하며, 기체 측에서는 오프보드(Offboard) 모드에서 Position 모드나 보류(Hold) 임무로 즉각 천이점(Transition)을 발동하여 충돌의 위험성을 낮춘다.

4. PX4와 ArduPilot의 추종 검증 철학 비교

  1. PX4-Autopilot의 궤적 추종 독립성:
    PX4는 오프보드 및 위치 제어 루프 내부에서 제어기 주도의 추종 오차 검증에 초점을 맞춘다. L2 노름 오차 한계를 위치 제어(mc_pos_control) 혹은 임무 제어 아키텍처 상의 파라미터로 두어, 비행 제어 지령 발산에 대한 방어벽을 독립적인 계층으로 운영한다.
  2. ArduPilot의 상태 변수 분산도 중심:
    ArduPilot은 독립적인 추종 오차 검증보다는 FS_EKF_THRESH 파라미터 등을 기반으로 하는 EKF 예측/관측 혁신(Innovation)과 센서 비정상 상태 테스트(EKF Variance)에 크게 의존하는 경향이 있다. 즉, 제어 지령을 얼마나 잘 따라가느냐보다, 내 항법 시스템(Nav Filter)이 지수적으로 붕괴되고 있는가를 확인하여 그 자체를 페일세이프로 묶는 방식의 모놀리식(Monolithic) 성향이 짙다.

5. 결론

실시간으로 연산되는 L2 노름(L2 Norm) 오차 지표는 기체의 목표 추종 능력을 유클리드 3차원 상의 물리적 거리라는 단일 스칼라 수치로 단순하고 명료하게 변환해 주는 공학적 장치이다. 이 지표를 QGroundControl의 다중 스레드 백엔드 단에서 병렬 산출하여 모니터링함으로써, 조종자는 현재 비행 제어 시스템의 물리적 한계점 돌파나 외란으로 인한 발산 현상을 선제적으로 탐지하여 신뢰성 있는 드론 관제 환경을 영위할 수 있다.
