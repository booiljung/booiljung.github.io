49.6.1.1 비례-적분-미분(PID) 제어의 적분기 와인드업(Integrator Windup) 방지를 위한 안티-와인드업(Anti-windup) 동적 클리핑

멀티로터의 속도 제어 루프(Velocity Control Loop) 및 자세 제어 루프(Attitude Control Loop)에 필수적으로 적용되는 비례-적분-미분(PID) 제어기에서, 가장 다루기 까다롭고 비행 안정성을 심각하게 위협하는 비선형 현상 중 하나가 바로 **‘적분기 와인드업(Integrator Windup)’**이다. 특히 지상 통제소(GCS)나 ROS2 기반 컴퓨터 비전 시스템이 연동된 Offboard 모드 비행 중에 강한 돌풍을 맞이하거나, 장애물 회피를 위해 극한의 회피 기동 지령 좌표가 순간적으로 갱신 주입될 때 발생 확률이 높아진다. 이때 탑재 액추에이터(BLDC 모터)가 이미 하드웨어적 출력 포화(Saturation) 한계에 도달하여 더 이상의 힘을 낼 수 없는데도 불구하고, 제어기의 수학적 적분항이 오차를 계속 누적하여 시스템을 ‘브레이크 불능’ 상태로 빠뜨리게 된다. 이 절에서는 적분기 와인드업의 발생 메커니즘과, PX4-Autopilot이 제어 상실(Control Loss)을 방어하기 위해 소스 코드 단에 구현된 안티-와인드업(Anti-windup) 동적 클리핑(Dynamic Clipping) 기법을 상세히 분석한다.

1. 적분기 와인드업(Integrator Windup) 현상의 제어 공학적 역학 원리

PID 제어기, 그중에서도 관성을 통제하는 속도 제어기의 적분항(I-term)은 지속적으로 좁혀지지 않는 정상 상태 오차(Steady-state Error)를 제거하기 위해 고안되었으며, 시간에 따른 이격 오차 e(t)를 연속적으로 누적 합산한다.

u_i(t) = K_i \int_{0}^{t} e(\tau) d\tau

멀티로터가 강한 역풍(Headwind)을 거슬러 전진 비행해야 하거나, 무거운 페이로드 부착으로 인해 비례 제어(P-term)만으로는 목표 고도에 결코 도달할 수 없는 상황이 벌어진다. 이처럼 지속적인 환경적 외란이 작용할 때, 적분항은 목표 도달 시까지 제어 출력 명령 u(t)의 크기를 끈질기게 증가시키는 중추적인 임무를 맡아 기체를 밀어붙인다.

하지만 물리적 실체인 전자변속기(ESC)와 모터 프로펠러 조합은 유한한 회전수(RPM)와 최대 발생 양력을 지닌다(예: 100% PWM Duty Cycle). 만약 제어기가 산출해 낸 요구 제어량 u(t)가 시스템의 물리적 최대 한계치 u_{max}를 초과해 버리면, 기체는 더 이상 추력을 더 낼 수 없다. 그럼에도 불구하고 마이크로컨트롤러 내부의 수학적 적분기는 오차가 줄어들지 않고 있음을 확인하고 수치적으로 수십, 수백 배의 가상 오차를 계속 누적(Windup)하여 눈덩이처럼 팽창시킨다.

시간이 지나 돌풍이 멈추거나 기체가 마침내 목표 지점 상공에 도달하여 오차의 부호가 정상적으로 반전(e(t) < 0, 제동 요구)되더라도, 제어기 메모리 내에는 이미 비대한 양수로 누적된 적분항이 잔존하고 있다. 이를 다시 음수로 깎아 내려 방전(Unwinding)시키기 전까지는 제어기는 계속해서 최대 출력 전진을 지시하게 되며, 결과적으로 기체가 웨이포인트를 크게 이탈해 멀리 미끄러지는 거대한 오버슈트(Huge Overshoot)를 일으킨다. 조향 각도가 심각할 경우 롤오버(Rollover)를 동반한 추락 사고로 직결된다.

2. PX4-Autopilot의 안티-와인드업(Anti-windup) 아키텍처 구현 전략

PX4-Autopilot(v1.14 기준)은 이러한 비선형 스톨(Stall) 현상을 사전에 강력히 억제하기 위해, PID 코어 라이브러리(PID.cpp) 및 속도 제어(VelocityController) 모듈 내에 견고한 이중 안티-와인드업 제한기(Double-layer Limiter) 로직을 구축하였다.

2.1 1차 방어: 정적 적분 제한 (Static Integral Limit)

가장 원시적이면서도 철저한 하드 컷오프(Hard Cut-off) 방어체계로, 파라미터 튜닝을 통해 적분 변수 자체가 유지할 수 있는 절대 매그니튜드(Magnitude)의 한계선을 강제한다.
예를 들어, 수평(XY축) 속도 제어 루프의 적분기 상한선은 MPC_XY_VEL_I_MAX 파라미터로 매핑되며, 수직(Z축) 고도 제어의 경우 MPC_Z_VEL_I_MAX로 지정된다. 이는 기체가 감당할 수 없을 비현실적인 가상의 적분량 크기(예: 10 \text{m/s}^2 이상의 적분 보상 등)가 메모리 변수에 애초에 누적되지 못하도록 고정된 임계값 아래로 동결 클리핑(Clipping)하는 역할을 수행한다.

2.2 2차 방어: 동적 조건부 제어 정지 (Dynamic Conditional Integration Freeze)

현대적인 PX4 제어기의 차별점인 동적 안티-와인드업 기법은, 현재 액추에이터의 실시간 포화 상태(Actuator Saturation Status)와 현재 입력되는 오차(Error)의 부호를 알고리즘적으로 교차 검증하여, 적분 누적을 동적으로 일시 유보(Freeze)시키는 똑똑한 기법이다. 매 제어 루프 사이클(예: 250Hz)마다 아래의 논리합 연산을 즉시 평가한다.

  1. 포화 상태 플래그 판별: 하위 제어 할당기(Control Allocator) 모듈로부터 전달받은 상태 변수를 통해, 현재 기체의 모터가 최대 출력(Max Thrust) 또는 최대 기울기 허용각(MPC_TILTMAX_AIR) 한계에 100% 충돌하였는지 판별한다.
  2. 부호 정합성(Sign Matching) 판별: 한계에 도달한 최상위 출력이 요구하는 증감 방향(+ 또는 - 부호)과, 지금 적분하려고 대기 중인 현재 속도 오차 e(t)의 부호가 동일한 방향인지 검사한다.
  3. 적분 컷오프(Integration Cut-off): 위 두 교집합 조건이 모두 논리값 ’참(True)’이라면, 즉 “모터가 100% 스로틀로 회전하여 한계치까지 밀어붙였음에도 불구하고 오차가 줄어들지 않아 제어기가 더 세게 밀어붙이라고 요구하는 수치적 착각 상황“이라면, 해당 제어 주기에서는 I-term 코드를 건너뛰어(Skip) 오차 누적을 원천 봉쇄한다. 반대로 기체가 웨이포인트를 초과하여 역방향으로 브레이크를 걸어야 하는 방향(오차 부호 반전)일 경우에는 포화 상태에 관계없이 적분 누적을 즉시 허용하여, 비대해진 적분기가 초고속으로 방전(Unwinding)되도록 경로를 열어둔다.

3. Ardupilot Copter 체계의 안티-와인드업 철학과 설계의 상대성

Ardupilot 역시 동일한 관성 제어 역학 문제를 극복하기 위하여 코어 S/W C++ 모듈인 AC_PID 라이브러리 내부에 훌륭하고 오랜 기간 검증된 안티-와인드업 구조체를 내장하고 있다. 그러나 파라미터 설정 및 컴포넌트 간 통신 철학에서 양대 오픈소스 진영 간의 극명한 취향 차이가 존재한다.

  • Ardupilot의 I_MAX 단일화 제어 철학:
    Ardupilot의 Mission Planner 설정 환경에서는 비례 제어(P) 게인이 출력할 수 있는 물리적 한계 절대값을 기준으로 삼아 누적량을 조절하는 PID_IMAX 파라미터가 명시적으로 노출되어 있다. 이는 사용자 접근성이 뛰어나며, 내부적으로 누수 적분기(Leaky Integrator: 시간이 경과함에 따라 누적 값이 마치 구멍 난 독의 물처럼 스스로 아주 조금씩 방전 상쇄되는 기법)라는 보조 안전장치를 고유하게 조합하여, 오래 지속된 무의미한 환경 오차 적분 값이 은연중에 자연 소멸하도록 설계되었다.
  • PX4-Autopilot의 역방향 연동(Back-propagation) 철학:
    PX4-Autopilot은 이기종 멀티로터, VTOL(수직 이착륙기) 및 로버 제어 아키텍처의 범용 통합 모듈성을 중시하여 개발되었다. 이에 따라 MPC_*_I_MAX 단위 변수를 활용한 1차적 정적 클리핑 외에도, 최하단 믹서(Mixer) 또는 제어 할당 코어의 하드웨어 포화(Saturation) 감지 Boolean 플래그를 상위 계층인 속도 제어 루프나 자세 제어 루프로 역방향 전송하는 타이트한 상태 피드백 시스템 스택을 지향한다. 이는 Ardupilot 구조 대비 객체 간 S/W 인터페이스의 복잡도를 가중시키지만, 단일 모터 파손/출력 저하 등 비대칭적인 로터 포화가 전체 기체의 자세 제어에 불러오는 와인드업 재앙 연쇄 반응을 끊어내는 데 있어 극도로 민첩한 반응성(Agility)을 발휘하는 이점을 제공한다.

4. QGroundControl (QGC)를 통한 실 비행 환경에서의 튜닝 실무 가이드

특히 MAVSDK나 ROS2 Offboard를 통해 다이내믹한 자율 비행 알고리즘을 설계하는 인공지능/제어 엔지니어는 QGroundControl의 Parameter 인터페이스를 통해 기체의 이륙 중량 증가나 외장형 라이더(LiDAR) 탑재로 인한 공기 역학적 표면적 저항을 고려하여 I_{MAX} 값을 필히 최적화해야 한다.

  1. 와인드업 발현 징후의 시각적 관측:
    알고리즘 연산에 의한 장애물 회피(Obstacle Avoidance) 루틴이 갑작스럽게 기체 진행 방향의 180도 반대편으로 단기 웨이포인트를 재갱신했을 때, 기체가 즉각 제동하지 않고 한참을 원래 방향으로 미끄러지며 관성 비행을 하다가 아주 뒤늦게 기수를 극단적으로 쳐들면서 역방향 오버슈팅 한다면 전형적인 적분기 와인드업 스톨 현상에 노출된 것이다.
  2. Flight Review 로그 데이터 시계열 매칭:
    기체 착륙 후 ULog를 확보하여 vehicle_local_position_setpoint 지점 갱신 데이터와 하위 객체인 actuator_controls_0 내의 스로틀 및 피치 축 인가 값을 대조하라. 목표 방향 좌표가 변경되었음에도 불구하고 실제 가속도 인가값 변경의 위상 지연(Phase Lag)이 1초 이상 지속된다면, 이는 PID 적분량이 범람하여 연산 병목 및 방전 불가 상태를 겪고 있다는 방증이다.
  3. 적분기 제한 파라미터의 타이트닝(Tightening) 조치:
    QGC를 통해 속도 누적 탄력인 MPC_XY_VEL_I 값 자체는 유지하되, 제한 파라미터인 MPC_XY_VEL_I_MAX 한계 파라미터를 현재 값의 20~30% 수치로 하향 조정하여, 적분기가 지나치게 커지지 못하도록 방파제(Barrier)의 높이를 강제적으로 낮추라. 단, 주의할 점으로 I_{MAX}를 아주 극단적인 수치(0 부근)로 봉쇄해 버리면, 기체가 초속 10m/s 이상의 지속적인 강풍(Tailwind)에 노출되어 떠밀려 갈 때 영영 목표 지점 위치 반환 복귀에 실패하는 무한 표류(Continuous Drift)에 빠질 수 있으므로 반드시 탁 트인 실 환경에서의 극한 바람 대응 테스트가 병행 인가되어야 한다.
