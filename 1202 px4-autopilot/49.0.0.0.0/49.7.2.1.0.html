49.7.2.1 센서 노이즈 필터링을 위한 중앙값 필터(Median Filter) 및 이동 평균 필터(Moving Average Filter) 설계

지상 관제 시스템(GCS, Ground Control Station)인 QGroundControl이나 조종기(RC)에서 입력되는 조이스틱(Joystick)의 아날로그 신호는 필연적으로 물리적, 전기적 센서 노이즈를 포함한다. 이러한 노이즈는 기체의 예기치 않은 움직임을 유발하거나, 제어 불안정성을 초래할 수 있으므로 제어 지령(Setpoint)으로 변환되기 전에 적절한 신호 처리 과정을 거쳐야 한다. 본 절에서는 원시 신호(Raw Signal)에서 스파이크 형태의 이상치(Outlier)를 제거하는 중앙값 필터(Median Filter)와 고주파 잡음을 평활화(Smoothing)하는 이동 평균 필터(Moving Average Filter)의 설계 원리와 PX4-Autopilot 및 QGroundControl(v4.3 기준)에서의 적용 방식에 대해 깊이 있게 다룬다.

1. 노이즈의 특성과 필터링의 필요성

조이스틱 내부에 장착된 가변 저항(Potentiometer)이나 홀 센서(Hall Effect Sensor)에서 측정된 아날로그 전압 값은 아날로그-디지털 변환기(ADC)를 거쳐 이산 시간(Discrete-time) 신호로 샘플링된다. 변환된 신호에는 주로 다음과 같은 두 가지 형태의 노이즈가 혼입된다.

  1. 임펄스 노이즈(Impulse Noise) 및 스파이크(Spike): 전자기적 간섭(EMI)이나 하드웨어 접점의 일시적인 불량으로 인해 튀는 비정상적인 값이다.
  2. 백색 잡음(White Noise): 열잡음 등에 의해 발생하는 연속적이고 상대적으로 진폭이 작은 고주파 평탄 잡음이다.

조종자의 의도(Operator Intent)를 정확하게 모델링하고 제어 지연(Control Delay)을 최소화하기 위해서는 위의 두 노이즈를 각각 효과적으로 제거하는 다단계 필터링(Multi-stage Filtering) 파이프라인 설계가 필수적이다.

2. 중앙값 필터(Median Filter) 설계

중앙값 필터는 비선형(Non-linear) 디지털 필터로, 지정된 윈도우(Window) 크기 내의 데이터들을 크기순으로 정렬한 뒤 그중 중앙값을 출력으로 취하는 방식이다.

2.1 수학적 모델링

이산 시간 인덱스 k에서의 입력 신호를 x[k], 윈도우의 크기를 N(일반적으로 홀수)이라 할 때, 중앙값 필터의 출력 y_{med}[k]는 다음과 같이 정의된다.

y_{med}[k] = \text{median}(x[k], x[k-1], \dots, x[k-N+1])

2.2. 특징 및 응용

중앙값 필터의 가장 큰 장점은 조종 신호의 급격한 계단형 변화(Step Change)인 에지(Edge) 인과성을 선명하게 유지하면서도 임펄스 노이즈를 극도로 잘 제거한다는 점이다. 선형 저역통과 필터(Low-Pass Filter)를 사용하여 스파이크를 제거할 경우 필터 고유의 지연(Delay) 현상으로 인해 빠른 조종 입력이 둔화될 우려가 있다. 하지만 중앙값 필터는 조작 신호의 반응성을 유지하면서 튀는 값을 배제할 수 있으므로, QGroundControl 내부의 JoystickManager나 원시 RC 입력 처리단에서 1차적인 이상치 소거 필터로 매우 빈번하게 사용된다.

3. 이동 평균 필터(Moving Average Filter) 설계

중앙값 필터를 통해 치명적인 임펄스 노이즈가 제거된 후에도 남아있는 미세한 고주파 백색 잡음을 제거하기 위해, 선형 필터인 이동 평균 필터나 지수 가중 이동 평균 필터(Exponential Moving Average, EMA)가 적용된다.

3.1. 단순 이동 평균(SMA, Simple Moving Average)

신호의 최근 M개 샘플의 산술 평균을 취하는 저역통과 필터(LPF) 방식이다. 입력이 x_{med}[k]라 할 때 다음과 같다.

y_{sma}[k] = \frac{1}{M} \sum_{i=0}^{M-1} x_{med}[k-i]

2.2 지수 가중 이동 평균(EMA) 필터의 소프트웨어 탑재

소프트웨어 펌웨어 관점에서 단순 이동 평균은 M개의 과거 데이터를 버퍼링해야 하므로 메모리와 연산 오버헤드가 지속 발생한다. 따라서 QGroundControl과 PX4-Autopilot은 메모리에 상태 하나만 유지하면 되는 1차 지수 가중 이동 평균 필터(EMA)를 주로 채택한다.

평활화 계수(Smoothing Factor)를 \alpha (0 < \alpha \le 1)라고 할 때 EMA 필터 결과는 다음과 같다.

y_{ema}[k] = \alpha \cdot x[k] + (1 - \alpha) \cdot y_{ema}[k-1]

  • \alpha 값이 1에 가까울수록 원시 신호를 그대로 수용하여 시스템 응답성은 빨라지나 노이즈 소거는 약화된다.
  • \alpha 값이 0에 가까울수록 과거 누적 값에 의존하게 되어 평활화 효과는 극대화되나 위상 지연(Phase Delay)이 커진다.

PX4(v1.14 기준) 펌웨어 내부에서는 RC_FLT_SMP라는 파라미터를 통해 RC 채널 신호의 차단 주파수(Cut-off Frequency)를 설정하며, 알고리즘 내부적으로 \alpha 팩터를 동적 계산하여 이산 시퀀스 영역에서 제어 율을 부드럽게 유지한다.

5. PX4-Autopilot과 ArduPilot의 신호 필터링 철학 비교

운영체제 수준의 디바이스 드라이버에서 센서 노이즈를 다루는 아키텍처는 PX4와 ArduPilot 오픈소스 생태계 간에 의미 있는 구조적 기조가 상이하다.

  1. PX4-Autopilot의 마이크로 모듈 구조:
    PX4의 센서 및 입력 처리 모듈(sensors 또는 rc_update 앱 체인)은 모듈형 컴포넌트 철학을 따른다. uORB 구독(Subscriber) 파이프라인 내에서 독립적으로 필터 클래스를 구동시켜 다중 필터 인스턴스 배치가 매우 용이하다.
  2. ArduPilot의 모놀리식 제어 쉐이핑:
    ArduPilot 플랫폼은 모놀리식(Monolithic) 메인 폴링 루프 내에서 센서 처리가 순차적으로 이어진다. 단순히 RC_FEEL_RP (Roll/Pitch 조작감 파라미터)와 같이 튜닝 변수를 통해, 노이즈 필터링의 목적을 넘어서 조종자에게 전달되는 ‘스틱 텐션과 응답 속도의 조화(Feel)’ 자체를 수학적으로 프로파일링 하는 것에 강점이 있다.

6. 설계 시 고려사항: 평활화와 제어 지연의 상충 관계

조종 신호를 안정화하기 위해 중앙값 필터의 윈도우 크기(N)와 평활화 계수(\alpha)를 가혹하게 설정하면 통계적으로 노이즈는 완벽에 가깝게 소거되지만, 제어 시스템 공학 관점에서 위상 지연 기조가 치명적으로 증가한다. 이것은 조종자가 조이스틱을 입력한 시점과 기체가 반응하는 시점 반사 속도의 이격을 벌려 예측을 어렵게 만들고, 종국에는 ’조종자 유발 진동(PIO, Pilot-Induced Oscillation)’이라는 불안정을 발생시킨다.

따라서 최적의 시스템 설계를 위해서는 중앙값 필터의 깊이를 최소한(예: 3 샘플 버퍼)으로 유지하고, 이동 평균 필터 단에서는 섀넌의 표본화 정리(Shannon Sampling Theorem)와 멀티로터 기체의 액추에이터 대역폭(Bandwidth) 특성을 고려한 차단 주파수 제어가 이뤄져야 한다. 이를 바탕으로 PX4의 오프보드(Offboard)와 매뉴얼(Manual) 임무 모드에서 GCS 백엔드의 MAVLink 폴링(Polling) 환경에 최적화된 저지연 비동기 전송 아키텍처를 유도할 수 있다.
