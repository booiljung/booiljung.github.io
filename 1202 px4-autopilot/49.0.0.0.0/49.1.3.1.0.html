49.1.3.1 비전 기반(Vision-based) 객체 추적(Object Tracking) 및 시각 서보 보어(Visual Servoing) 루프의 폐루프(Closed-loop) 구성

자율 에이전트 분산 토폴로지에서 가장 실용적이면서도 난도가 높은 애플리케이션 중 하나가 바로 **비전 기반 객체 추적(Vision-based Object Tracking)**이다. 드론이 목표물을 카메라 중심에 지속적으로 가두면서(Lock-on) 따라가기 위해서는 임무 컴퓨터의 인지(Perception) 모듈과 PX4-Autopilot의 하위 비행 제어기가 유기적인 폐루프(Closed-loop) 시스템으로 묶여야 한다. 본 섹션에서는 객체의 화소(Pixel) 오차를 물리적인 비행 지령으로 변환하는 시각 서보 보어(Visual Servoing) 루프의 아키텍처와 PX4 오프보드 연동 기법을 분석한다.

1. 시각 서보(Visual Servoing)의 두 가지 패러다임

시각 서보란 카메라에서 획득된 비전 정보를 기반으로 로봇의 모션을 제어하는 기법을 일컫는다. 제어 피드백의 기준 좌표계에 따라 크게 두 가지로 나뉜다.

  1. 위치 기반 시각 서보 (Position-Based Visual Servoing, PBVS)
  • 목표물의 3차원 위치(Cartesian 3D) 좌표를 먼저 추정(예: Depth 카메라 활용이나 객체의 크기 정보 역산)한 뒤, 드론이 해당 위치로 이동하도록 3D 공간상의 위치 제어(Position Control) 지령을 내리는 방식이다.
  • 3D 복원을 위한 카메라 캘리브레이션과 PnP(Perspective-n-Point) 알고리즘 정확도에 크게 의존한다.
  1. 이미지 기반 시각 서보 (Image-Based Visual Servoing, IBVS)
  • 목표물의 3차원 위치를 계산하지 않고, 카메리로 들어오는 2차원 화소(Pixel) 평면상에서의 오차(Image Feature Error)를 역학적인 카메라 속도 제어(Velocity Control) 입력으로 직접 사상(Mapping)하는 방식이다.
  • 모델 정보가 불명확한 환경에서 드론을 추적 모드로 구현할 때 연산량이 적고 오차가 누적되지 않아 실시간 객체 추적(Real-time Object Tracking)에 압도적으로 많이 쓰인다.

PX4와 ROS2를 활용한 현대적 드론 시스템 구성에서는 객체 추적 로직에 주로 **IBVS 기법과 속도 제어 기반 오프보드 루프(Velocity-based Offboard Loop)**를 혼합하여 설계한다.

2. 폐루프(Closed-loop) 통제 루프 아키텍처

객체 추적을 위한 IBVS 폐루프의 구성은 일반적으로 영상 인지 \rightarrow 에러 산출 \rightarrow 제어기(PID) 연산 \rightarrow 속도 설정값(Velocity Setpoint) \rightarrow PX4 동적 응답의 순환 구조를 가진다.

    B -->|Bounding Box| C[IBVS Controller Node]

    J -->|New Frame| A

3. 화소 에러(Pixel Error)와 운동학적 매핑(Kinematic Mapping)

임무 컴퓨터에서 객체를 인식(예: YOLO, 선형 트래커(KCF), 컬러 트래커 등 사용)하면, 화면 상의 바운딩 박스(Bounding Box)의 중심점 좌표 (u, v)를 얻게 된다.
이미지 평면의 정중앙 좌표인 (u_c, v_c)와의 차이가 바로 시각 에러 \mathbf{e} 이다.

  • 수평 위치 오차: e_u = u_c - u
  • 수직 위치 오차: e_v = v_c - v
  • 객체 크기 오차(거리 근사): e_A = A_{target} - A_{box} (영역 크기 A)

이 시각 에러를 영(Zero)으로 만들기 위해 ROS2 노드 내부의 독립적인 시각 제어용 PID 컨트롤러가 동작한다. 이 컨트롤러의 출력은 위치 좌표가 아니라, 오프보드로 전송될 기체의 **목표 속도(Velocity Setpoint) 및 요-레이트(Yaw Rate)**이다.

  • 목표 요-레이트 (Yaw Rate): e_u에 비례 (기체를 좌우로 회전하여 객체를 중앙에 맞춤)
  • 목표 고도 속도 (Vertical Velocity): e_v에 비례 (기체를 위아래로 움직여 객체를 중앙에 맞춤)
  • 목표 전진 속도 (Forward Velocity): e_A에 비례 (객체가 작아지면 다가가고 커지면 멀어짐)

4. 프레임 변환(Frame Transformation) 시의 고려사항

기체 좌표계(Body Frame)와 카메라 좌표계(Camera Frame)의 정렬(Alignment)은 객체 추적의 성공 여부를 결정하는 핵심 요소이다.
PX4의 오프보드 TrajectorySetpoint에서 속도 제어 지령을 Body 프레임으로 보낼 것인지, NED(North-East-Down) 글로벌 프레임으로 보낼 것인지 결정해야 한다.

  • 일반적으로 카메라가 완전히 기체 전방(Forward)에 고정된 FPV 드론이라면, Body Frame(Velocity_frame = Velocity_frame_body_ned)으로 앞/뒤 전진 속도 레이트를 주고 요-레이트를 제어하는 것이 수월하다.
  • 짐벌(Gimbal) 시스템이 장착된 기체라면 드론의 기구학적 한계보다 짐벌 모터가 객체를 추적하고, 드론은 짐벌의 인코더 오차(Gimbal Yaw/Pitch Error)를 해소하는 방향으로 천천히 따라가게끔 모델링을 하는 것이 더 안정적이다. (이를 Gimbal-slaved Tracking이라 한다.)

5. 지연 시간(Latency) 및 노이즈에 대처하는 루프 설계 원칙

이미지 처리에는 필연적으로 수십 밀리초(ms) 이상의 연산 딜레이가 발생한다.
비전 노드의 인지 속도가 10Hz라고 가정해 보자. 앞선 49.1.3 단원의 “Offboard Failsafe Timeout” 정책에 따르면, PX4는 최소 2Hz 이상(안정적으로는 50Hz) 제어 지령을 받아야 유지된다.

여기서 발생하는 데이터 주기의 비대칭(Asymmetry) 문제를 해결하기 위해, ROS2 임무 컴퓨터 측에서는 폐루프를 이중화(Dual-loop)하여 토폴로지를 구성해야 한다.

  1. Vision Loop (10~30Hz): 카메라 이미지를 분석하고 화소 오차 기반으로 목표 속도만 갱신한다. 이 속도는 버퍼 메모리에만 기록된다.
  2. PX4 Streaming Loop (50Hz 이상): 별도의 스레드(Thread)나 타이머 콜백을 두어, 비전 루프가 갱신한 최신 목표 속도 데이터를 50Hz 간격으로 꾸준히 TrajectorySetpoint 메시지로 펌핑(Publish)하여 PX4의 오프보드 타임아웃을 방지한다. 만약 객체를 일시적으로 잃어버렸다면(Tracking Lost), 이 루프는 속도를 0으로 하여 제동(Braking/Hovering) 지령을 밀어 넣도록 안전판 역할을 해야 한다.

6. Ardupilot의 Guided 속도 제어와의 차이 (Ardupilot 대비 차이)

두 비행 기체 모두 객체 추적을 구현할 수 있으나 속도 루프의 동특성이 다르다.

  • PX4-Autopilot: 속도 제어(Velocity Control)를 오프보드로 던졌을 때, MPC_ACC_HOR / MPC_JERK_MAX 등의 파라미터가 네이티브하게 작동하여 지령을 갑자기 바꾸더라도 기체가 알아서 부드러운 스무딩(Smoothing) 처리를 해준다. 따라서 비전에서 나오는 다소 노이즈 높은 출력값을 필터링 없이 넘기더라도 시스템이 자체적으로 가속도를 제한해주어 부드러운 카메라 무빙(Cinematic Tracking)을 지원하는 이점이 크다.
  • ArduPilot: SET_POSITION_TARGET_LOCAL_NED 메시지의 Velocity 필드를 활용하지만, 관성 및 스무딩 처리가 기체 파라미터 구성에 따라 다소 거칠게 나타날 수 있으며, PID 오버슈트(Overshoot)를 억제하기 위해 GCS(외부 컴퓨터) 쪽에서 선형 필터율을 추가로 고려해야 하는 경우가 많다.

7. 결론

비전 기반 객체 추적과 IBVS 제어는 임무 컴퓨터와 PX4 간의 유기적 폐루프 융합을 상징하는 기술이다. 화소 이동량을 기구학적 속도로 매핑하는 정교한 PID 설계는 물론이거니와, 오프보드 제어 토폴로지의 생리학적 특성(타임아웃 방어 스레드, 제어 주기 펌핑 50Hz 유지 등)을 모두 이해하고 시스템을 구성해야만 흔들림 없는 타겟 록온과 고성능 자율 추적 루프를 완성할 수 있다.
