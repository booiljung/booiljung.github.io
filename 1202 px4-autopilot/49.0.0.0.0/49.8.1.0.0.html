49.8.1. ULog 원격 측정(Telemetry) 데이터의 시계열(Time Series) 분석 방법론

PX4-Autopilot의 로깅 시스템은 시스템의 모든 내부 상태 변수를 uORB 메시지 형태로 기록하는 ULog 포맷을 채택하고 있다. 이는 단순한 이벤트 기록을 넘어서 제어 공학적 관점에서의 정밀한 모델링과 응답성 평가를 가능하게 한다. 특히, 실시간 Offboard 제어 명령이 인가되는 환경에서는 지상 관제 시스템(GCS)이나 보조 컴퓨터(Companion Computer)에서 전송된 지령(Setpoint)과 실제 기체의 물리적 반응(State)을 시계열(Time Series) 영역에서 대조하는 분석 기법이 필수적이다. 본 절에서는 ULog 데이터를 추출하여 수학적이고 통계적인 시계열 분석을 수행하는 구체적인 방법론을 다룬다. (참고 기준: PX4-Autopilot 펌웨어 v1.14 이상, Python pyulog 라이브러리)

1. 데이터 추출 파이프라인(Data Extraction Pipeline)

SD 카드 또는 MAVLink 로깅을 통해 수집된 .ulg 파일은 이진 바이너리 형식으로 인코딩되어 있으므로, 이를 분석 가능한 배열 형태의 시계열 데이터로 가공하기 위한 파이프라인의 구축이 필요하다. 분석 자동화를 위해 Python 기반의 pyulog 라이브러리와 pandas 데이터프레임 구조를 결합하는 것이 표준적인 접근법이다.

다음은 ULog 파일에서 시계열 데이터를 추출하고 전처리하는 일반적인 데이터 파이프라인의 아키텍처이다.

flowchart TD
    A[비행 완료 후 .ulg 파일 획득] --> B[pyulog 라이브러리로 바이너리 파싱]
    B --> C{필요한 uORB 메시지 추출}
    
    C -->|지령 타겟| D[vehicle_local_position_setpoint]
    C -->|기체 상태| E[vehicle_local_position]
    C -->|가속도/각속도| F[sensor_combined / vehicle_angular_velocity]
    C -->|액추에이터 출력| G[actuator_controls / actuator_outputs]
    
    D --> H[메시지 타임스탬프 동기화<br/>(Time Alignment & Interpolation)]
    E --> H
    F --> H
    G --> H
    
    H --> I[Pandas DataFrame 변환]
    I --> J[추종 오차 산출, 주파수 응답 분석, 시각화]

2. 타임스탬프 동기화(Time Alignment) 및 보간(Interpolation)

uORB 메시지는 각 모듈(예: 센서 드라이버, EKF2, Position Controller) 특성에 따라 서로 다른 주파수(예: 50Hz, 100Hz, 250Hz)로 비동기적으로(Asynchronously) 발행(Publishing)된다. 즉, vehicle_local_position_setpoint 메시지와 vehicle_local_position 메시지는 기록된 타임스탬프(timestamp 필드, 마이크로초 단위)가 정확히 일치하지 않는다.

따라서 시계열 분석을 수행하기 전에는 동일한 시간축(Time Base)을 기준으로 두 신호를 이산화(Discretization)하고 보간하는 과정이 선행되어야 한다. 일반적으로 다음과 같은 방법을 적용한다.

  1. 공통 시간 벡터(Common Time Vector) 정의: 분석하고자 하는 주파수(예: 100Hz)를 설정하여 일정한 간격의 새로운 시간 배열을 생성한다.
  2. 선형 보간(Linear Interpolation): 비동기 데이터 포인트들을 공통 시간 벡터의 시점에 맞추어 선형 보간한다. 급격히 변동하는 센서 데이터의 경우 스플라인 보간(Spline Interpolation)이 유리할 수 있으나, 지령과 위치 데이터의 일반적 분석에는 1차 선형 보간으로 족하다.
  3. 영차 유지(Zero-Order Hold, ZOH): Offboard에서 인가되는 단계 지령(Step Input)과 같은 불연속 데이터의 경우, 값이 갱신되기 전까지 이전 값을 유지하는 ZOH 방식을 적용해야 센서의 지연 시간(Latency)을 왜곡 없이 캡처할 수 있다.

3. 기본 시계열 분석 지표 확립

동일한 시간축으로 동기화된 데이터프레임이 준비되면, 다음과 같은 제어 성능 지표를 수치적으로 연산할 수 있다.

  • 최대 오버슈트(Maximum Overshoot): 오프보드 경로점(Waypoint) 갱신 시, 기체가 목표 위치를 넘어서는 최대 거리 백분율.
  • 정상 상태 오차(Steady-State Error): 목표 속도 지령(vz) 인가 시, 일정한 속도에 도달한 후 지령값과 실제 속도(vehicle_local_position.vz) 간의 편차(Bias).
  • 상승 시간(Rise Time) 및 정착 시간(Settling Time): 지령의 10%에서 90%까지 도달하는 시간과, 오차 대역(일반적으로 2% 내지 5%) 내로 진입하여 유지되는 시간.

3.1 Python 분석 코드 구현 예시

pyulog를 이용하여 두 메시지를 추출하고 오차(Error)를 계산하는 가장 핵심적인 파이썬 스크립트 뼈대는 다음과 같이 작성된다.

from pyulog import ULog
import pandas as pd
import numpy as np

# ULog 파일 로드
ulog = ULog('flight_log.ulg')

# setpoint와 실제 위치 데이터 추출
setpoint_data = ulog.get_dataset('vehicle_local_position_setpoint')
state_data = ulog.get_dataset('vehicle_local_position')

# Pandas DataFrame으로 변환
df_sp = pd.DataFrame(setpoint_data.data)
df_st = pd.DataFrame(state_data.data)

# 시간축 기준(최소 타임스탬프를 0으로 정규화)
t0 = min(df_sp['timestamp'][0], df_st['timestamp'][0])
df_sp['time_s'] = (df_sp['timestamp'] - t0) / 1e6
df_st['time_s'] = (df_st['timestamp'] - t0) / 1e6

# 공통 시간대(예: 100Hz, 10ms 간격)로 보간 (Interpolation)
common_time = np.arange(0, df_st['time_s'].iloc[-1], 0.01)
x_setpoint = np.interp(common_time, df_sp['time_s'], df_sp['x'])
x_state = np.interp(common_time, df_st['time_s'], df_st['x'])

# 추종 오차(Tracking Error) 시계열 배열 생성
tracking_error_x = x_setpoint - x_state

이와 같이 추출된 tracking_error_x 배열은 후속 분석에서 평균 제곱 오차(RMSE, Root Mean Square Error)나 분산(Variance)을 도출하는 핵심 데이터로 활용된다. PX4-Autopilot의 고정밀 타임스탬프 시스템은 이와 같이 시스템 식별(System Identification) 및 성능 프로파일링을 위한 정밀한 시계열 도구를 구성하는 토대가 된다.
