49.8.1.2. 보데 선도(Bode Plot) 주파수 응답 기법을 활용한 외부 루프(Outer Loop) 이득 여유(Gain Margin) 및 위상 여유(Phase Margin) 도출

오프보드(Offboard) 모드에서 인가되는 위치 및 속도 지령은 멀티로터 제어 아키텍처의 가장 바깥쪽에 위치하는 외부 루프(Outer Loop)를 구동한다. 위치 제어기(Position Controller, 통상적으로 P 제어)와 속도 제어기(Velocity Controller, 통상적으로 PID 제어)를 포함하는 외부 루프의 안정성을 수학적으로 보증하기 위해서는 주파수 영역(Frequency Domain)에서의 해석이 필수적이다. 본 절에서는 ULog 데이터 혹은 실시간 텔레메트리(Telemetry)를 바탕으로 주파수 스윕(Frequency Sweep) 주입 비행을 수행하고, 이를 보데 선도(Bode Plot)로 시각화하여 이득 여유(Gain Margin)와 위상 여유(Phase Margin)를 도출하는 기법에 대해 상세히 다룬다.

1. 외부 루프 주파수 응답(Frequency Response) 해석의 기저

PX4-Autopilot의 기본 위치 제어 아키텍처에서 외부 루프는 저궤적(Low-frequency) 동역학을 지배하며, 내부 루프(Inner Loop)인 자세 및 각속도 제어 루프의 대역폭(Bandwidth)에 크게 의존한다. 시스템 식별(System Identification, SYSID) 이론에 따르면, 폐루프(Closed-loop) 제어기의 개루프 전달함수(Open-loop Transfer Function) L(j\omega)를 근사하기 위해서는 기체에 지속적으로 변하는 주파수의 사인파(Chirp Signal) 지령을 인가해야 한다.

r(t) = A \sin(2\pi f(t) t)

이때 f(t)는 시간이 지남에 따라 점진적으로 증가하는 주파수이다. 이 지령에 대한 시스템의 실제 위치 혹은 속도 궤적 y(t)를 ULog 포맷에서 추출한 뒤, 지표를 산출한다.

제어 안정성의 핵심 지표

  • 이득 여유(Gain Margin, GM): 위상이 -180도일 때(위상 교차 주파수 \omega_{pc}) 이득이 0dB(크기 비 1)에 도달하기까지의 여유 공간. 통상적으로 학술적 및 실무적 기준에서 GM > 6\text{dB}를 권장한다.
  • 위상 여유(Phase Margin, PM): 이득이 0dB일 때(이득 교차 주파수 \omega_{gc}) 위상이 -180도에 도달하기까지의 여유 공간. 무인항공기의 외부 루프 제어에서는 외란에 대한 로버스트니스(Robustness)를 보장하기 위해 PM > 45^\circ \sim 60^\circ를 권장한다.

2. 주파수 스윕 비행 및 데이터 추출 파이프라인

주파수 스윕 비행은 보조 컴퓨터의 ROS 2 (uXRCE-DDS) 혹은 MAVSDK 스크립트를 통해 자동으로 수행된다.

3. 보데 선도 산출을 위한 Python 구현 예시

추출된 이산 시계열 데이터인 지령 벡터 r(k)와 출력 벡터 y(k)를 사용하여 시스템의 주파수 응답을 도출하기 위해서는 파이썬의 scipy.signal 모듈에 내장된 스펙트럼 밀도 함수와 교차 스펙트럼(Cross-Spectrum) 밀도 함수 (e.g., Welch’s Method)를 활용할 수 있다.

경험적 전달함수(Empirical Transfer Function Estimate, ETFE) G(j\omega)는 다음과 같이 개략적으로 산출된다.

import numpy as np
import scipy.signal as signal
import matplotlib.pyplot as plt

def generate_bode_plot(x_setpoint, x_state, sample_rate_hz):
    """
    Welch Method를 이용한 전력 스펙트럼 밀도(PSD) 기반 경험적 전달함수 추정
    """
    # 입력(지령)과 출력(상태) 간의 교차 자가 스펙트럼(Cross Spectral Density)
    f, Pxy = signal.csd(x_setpoint, x_state, fs=sample_rate_hz, nperseg=1024)
    # 입력(지령)의 전력 스펙트럼 밀도(Power Spectral Density)
    f, Pxx = signal.welch(x_setpoint, fs=sample_rate_hz, nperseg=1024)
    
    # 전달함수(Transfer Function) 근사
    H = Pxy / Pxx
    
    # 이득(Magnitude) 및 위상(Phase) 연산
    magnitude_dB = 20 * np.log10(np.abs(H))
    phase_deg = np.angle(H, deg=True)
    
    # Bode Plot 시각화 (코드 단순화)
    fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(8, 6))
    ax1.semilogx(f, magnitude_dB)
    ax1.set_ylabel('Magnitude (dB)')
    ax1.grid(True, which="both", ls="-")
    
    ax2.semilogx(f, phase_deg)
    ax2.set_ylabel('Phase (deg)')
    ax2.set_xlabel('Frequency (Hz)')
    ax2.grid(True, which="both", ls="-")
    
    plt.show()

# 사용 예: 100Hz로 샘플링된 시계열 데이터
# generate_bode_plot(df_aligned['x_setpoint'], df_aligned['x_state'], 100.0)

위의 다이어그램과 코드를 통하여 도출된 보데 선도는 외부 제어 루프가 몇 Hz 대역까지 입력에 대해 0dB(추종률 100%)를 유지하는지 정량적으로 보여준다.

4. 진단 및 최적화 전략

도출된 주파수 응답 곡선 상에서 이득 여유 혹은 위상 여유가 부족하다면 다음과 같은 디버깅 전략을 수립해야 한다.

  1. 내루프(Inner Loop) 간섭 확인: 만약 외부 루프의 대역폭(Bandwidth)이 너무 높게 설정되어 자세 제어기의 대역폭과 충돌하는 경우(대역폭 분리 원칙 위배), 위상 지연이 급격히 발생하여 발진(Oscillation)으로 이어진다. 외부 루프(MPC_XY_P 등)의 게인을 낮추어 이득 교차 주파수를 좌측으로 이동시켜야 한다.
  2. 필터 설정 검토: GCS나 보조 컴퓨터에서 생성된 경로의 불요한 고주파 잡음이 trajectory_setpoint로 인가될 때, PX4 파라미터 내의 소프트웨어 저역통과필터(Low-pass filter, 예: MPC_XY_VEL_D_LP) 설정이 부적절하여 추가적인 위상 지연(Phase Lag)을 유도할 수 있다.

이와 같이 주파수 응답을 활용하는 방법론은 PX4-Autopilot의 오프보드 제어 성능을 블랙박스가 아닌 역설계(Reverse Engineering) 관점에서 완전히 해부할 수 있도록 해준다.
