49.4.1.2 네트워크 지터(Jitter)로 인한 샘플링 주기 불균일성이 이산 제어기(Discrete Controller) 안정성에 미치는 영향

1. 개요(Introduction)

지상 관제 시스템(GCS, Ground Control Station)에서 PX4-Autopilot으로 실시간 제어 지령(Setpoints)을 전송하는 Offboard 모드 운용에 있어 통신 네트워크의 지연(Latency)과 지터(Jitter)는 필연적으로 발생한다. 특히 지터(Jitter)는 패킷의 도착 시간 간격(Inter-arrival Time)이 일정하지 않고 변동하는 현상으로, 컨트롤러의 관점에서 샘플링 주기(Sampling Period)의 불균일성을 초래한다.

비행 제어 시스템의 이산 제어기(Discrete Controller)는 설계 시 고정된 샘플링 시간(Fixed Sampling Time, T_s)을 가정하지만, 네트워크 지터로 인해 실제 데이터 갱신 주기(\Delta t)가 매번 달라지게 되면 제어 시스템의 성능 저하 및 안정성(Stability) 훼손을 유발할 수 있다. PX4(v1.14 기준)의 Offboard 모드 구조 하에서는 주기적인 MAVLink 메시지 수신 빈도가 보장되어야 하며, 이를 위반할 경우 Failsafe 메커니즘이 트리거되어 제어권을 상실할 수 있다.

2. 이산 제어기(Discrete Controller)에서의 샘플링 주기와 안정성(Stability)의 관계

대부분의 비행 제어 알고리즘은 연속 시간(Continuous Time) 영역에서 설계된 후 이산화(Discretization) 과정을 거쳐 오토파일럿 보드에 탑재된다. 제어기의 이산화 과정(예: Forward Euler, Tustin/Bilinear transformation 등)은 일정한 샘플링 주기 T_s를 기반으로 한다.

정규 이산 시간 상태 방정식(State-Space Equation)은 다음과 같이 표현된다.
x[k+1] = A_d x[k] + B_d u[k]

여기서 A_d = e^{A T_s} 이며, T_s가 상수가 아닌 시변 매개변수 T_s(k) = T_0 + \delta(k) 로 대체되는 경우, 시스템 모델은 시변 시스템(Time-Varying System)으로 변모하게 된다. 지터로 인해 변동폭 \delta(k)가 지나치게 커지면 제어기의 고유 루프 극점(Closed-loop Poles)이 단위원(Unit Circle) 밖으로 이동할 위험이 발생하여 시스템의 점근적 안정성(Asymptotic Stability)이 영구적으로 보장되지 않는다.

3. 네트워크 지터(Jitter)가 유발하는 주요 현상과 제어적 영향

3.1 위상 여유(Phase Margin)의 감소

시스템의 수학적 위상 지연은 e^{-j\omega T_s} 로 나타나며, 샘플링 시간 변동은 모델링되지 않은 가변적인 위상 지연(Time-varying Phase Delay)을 추가로 발생시킨다. 이러한 불필요한 위상 지연은 제어 루프의 위상 여유를 갉아먹으며(Phase Margin Degradation), 이로 인해 기체의 제어 응답성에 심각한 오버슈트(Overshoot)와 진동(Oscillation) 현상을 유발할 수 있다.

3.2 속도 및 위치 추정의 오차 누적

Mavlink를 통한 유도 제어(Guided) 위치/속도 지령이 일정한 간격으로 들어오지 않는 경우, PX4 내부의 위치 제어기(Position Controller)는 지령 사이의 불연속적인 보간(Interpolation) 또는 외삽(Extrapolation)을 지속하게 된다.
특히 **QGroundControl(v4.3 기준)**과 정상적인 네트워크 동기화가 이루어지더라도, 중간 경로의 Wi-Fi 혹은 LTE망의 비동기 링크 스케줄링 간섭 등으로 인해 물리적 패킷 전송 간격 자체가 불규칙해지는 현상이 빈번하게 확인된다.

3.3 나이퀴스트-섀넌 샘플링 정리(Nyquist-Shannon Sampling Theorem) 위반 위험

시스템의 대역폭(Bandwidth) 내에 존재하는 동력학적 비선형 특징을 완전히 제어하기 위해서는 충분히 짧고 규칙적인 이산화 샘플링 주기가 요구된다. 네트워크 통신의 고질적 지터로 인해 순간적인 샘플링 간격이 제어 대역폭의 한계(Shannon Limit) 이상으로 벌어질 경우, 앨리어싱(Aliasing) 효과 등 비선형적인 파괴적 제어 불안정성이 대두된다.

5. 지터 보상 및 완화 설계 기법 (Mitigation Techniques)

안정성에 미치는 치명적인 영향을 제어하기 위해서 제어 공학 분야에서는 다양한 보상 기법이 연구되고 있으며, PX4 및 MAVLink 통신 체계 내에서 다음과 같은 기법들이 부분적으로 도입 혹은 검토될 수 있다.

  1. 지터 버퍼(Jitter Buffer) 또는 플레이아웃 버퍼(Playout Buffer)의 도입:
    도착한 제어 패킷을 즉시 반영하지 않고 일정 시간 버퍼 블록에 대기시킨 후 고정된 주기로 내부 제어기(uORB 토픽 등)에 정주기 발행(Publish)하는 기법이다. 이는 지터를 예측 가능한 고정 지연(Latency)으로 치환함으로써, 일관된 이산 제어기 모델(T_s 상수 유지 모델)을 적용할 수 있게 해준다. Ardupilot 대비 PX4의 uORB 메시징 구조상 버퍼 구현 프레임워크를 브릿지로 추가하기 용이하나, 시스템의 전체 통신 지연이 절대적으로 증가하는 명확한 상충 관계(Trade-off)가 발생하므로 극히 주의 깊은 튜닝이 요구된다.

  2. 시변 이산 제어(Time-varying Discrete Control) 미세 튜닝 및 게인 스케줄링(Gain Scheduling):
    수신된 데이터의 실제 타임스탬프(Timestamp) 오프셋을 역산하여, 변동된 샘플링 주기 \Delta t를 제어기 이득(Gain) 계산식 연산에 실시간으로 반영하는 동적 보상 형태이다. 제안 논문 “Stability Analysis of Networked Control Systems with Jitter” 등에서 제시하는 이 접근법은 GCS 통신망 환경 변화에 대한 강건 제어(Robust Control) 범주에 속한다.

  3. 관측기(Observer) 기반 상태 외삽 예측(Extrapolation Prediction):
    지연과 지터로 인해 누락되었거나 도착 주기가 과도하게 지연된 샘플 인스턴스를 기체 자체의 동력학 모델(Kinematics Model) 필터를 통해 수학적으로 추정해낸다. PX4 내부의 정밀한 Estimator(ECL/EKF2) 정보를 활용하여 외부 지령 제어기의 공백 상태를 연속적으로 보상해주는 방식이다.

6. 결론

네트워크 환경의 지터는 불균일한 샘플링 주기를 유발하여, 이산화 시스템으로 엄격하게 설계된 PX4-Autopilot의 Offboard 제어 성능과 위상 안정성 마진을 심각하게 감소시키는 핵심 요인이다. 따라서 임무(Mission) GCS(QGroundControl 등) 백엔드 단에서 실시간 커널을 활용한 지터 최소화 전송 설계는 기본이며, 기체 측 MAVLink 수신 처리 모듈 및 비행 제어 알고리즘 파이프라인 전반에 걸친 지터 극복 강건 설계(버퍼링 혹은 타임스탬프 기반 동적 보상)가 현장 운용 차원에서 반드시 병행되어야 한다.
