49.4.1 사용자 데이터그램 프로토콜(UDP) 기반 비동기 통신 채널의 확률론적 모델링

1. 개요: UAV 비동기 제어 통신에서 UDP의 당위성

자율 비행 에이전트(Autonomous Agent) 운용 시 지상 관제 시스템(GCS) 또는 동반 컴퓨터(Companion Computer)에서 PX4-Autopilot으로 전달되는 Offboard 제어 지령 데이터는 고도의 실시간성을 요구한다. 이러한 환경에서는 데이터의 절대적인 도착(Reliability)보다 적시에 최신 정보가 도착하는 것(Timeliness)이 제어 성능에 더 결정적인 영향을 미친다.

TCP(전송 제어 프로토콜)는 패킷 손실 발생 시 손실된 패킷이 재전송될 때까지 이후의 모든 패킷 수신을 블로킹(Head-of-Line Blocking)하므로, 매우 심각한 지연 스파이크(Latency Spike)를 유발하여 비행 제어 시스템을 발산시킬 위험이 있다. 따라서 실시간 드론 통신 프로토콜인 MAVLink는 물론, PX4 v1.14 이후 도입된 ROS2 uXRCE-DDS 미들웨어 통신 브리지 역시 사용자 데이터그램 프로토콜(UDP: User Datagram Protocol) 을 기본 전송 계층(Transport Layer)으로 채택한다. UDP 기반 채널은 비연결형(Connectionless) 장점이 있으나, 무선 환경(Wi-Fi, RF, LTE) 특성상 발생하는 지터(Jitter)와 트래픽 손실을 제어 시스템 설계 단계에서 확률론적으로 모델링해야 하는 숙제를 동반한다.

2. 확률론적 모델링 접근법

비동기 통신 채널을 수학적으로 분석하기 위해, 통신 지연 분포와 패킷 손실 확률 모델이라는 두 가지 주요 차원으로 접근한다.

2.1 라우팅 지연 시간(Jitter)의 확률 밀도 함수(PDF)

무선 네트워크 위를 이동하는 지령 패킷의 생존 지연 시간(End-to-end Delay) T_{delay}은 고정된 상수가 아니라, 확률 밀도 함수(PDF, Probability Density Function)를 갖는 확률 변수로 모델링된다. 무선 멀티홉 네트워크나 LTE 간접 라우팅 환경에서는 지연 시간이 정규 분포(Gaussian Distribution)보다는 긴 꼬리(Long-tail)를 가지는 감마 분포(Gamma Distribution) 또는 대수-정규 분포(Log-Normal Distribution) 에 더 가깝게 나타난다.

드론의 제어 루프 관점에서는 이 지연 시간 모델을 기반으로 폐루프 제어의 안정성 기준을 초과하는 ’늦게 도착한 지령’을 버리거나 혹은 앞선 추정치로 보간(Interpolation)하는 칼만 필터 예측 메커니즘을 적용하게 된다.

2.2 패킷 손실의 길버트-엘리엇(Gilbert-Elliott) 마르코프 모델

단순히 “평균 패킷 손실률이 5%이다“라는 베르누이(Bernoulli) 모델은 실제 무선 페이딩(Fading) 환경을 정확히 모사하지 못한다. 무선 전파 간섭이 발생하면 패킷은 독립적으로 손실되는 것이 아니라 연쇄적(Bursty)으로 누락되는 경향이 강하다.

이를 확률론적으로 모델링할 때 가장 널리 쓰이는 방법론이 2-상태 마르코프 체인(Two-state Markov Chain) 인 길버트-엘리엇 모델(Gilbert-Elliott Model) 이다. 채널은 다음과 같이 두 가지 상태 중 하나에 놓이게 된다.

stateDiagram-v2
    direction LR
    Good: Good State (G)
    Bad: Bad State (B)
    
    Good --> Good : 1 - p
    Good --> Bad : p
    Bad --> Bad : 1 - r
    Bad --> Good : r
  • Good State (G): 채널 상태가 양호하여 패킷 손실 확률이 0이거나 매우 낮은 상태.
  • Bad State (B): 채널 페이딩, 간섭, 혹은 장애물에 의해 패킷이 연달아 손실될 확률이 1이거나 매우 높은 상태.
  • p: Good 상태에서 Bad 상태로 전이할 확률 (에러 뱅크 진입 확률).
  • r: Bad 상태에서 Good 상태로 복구될 확률.

이 모델을 통해 연쇄적 패킷 손실(Burst Loss)의 지속 시간(Expected Burst Length)을 1/r 로 정량화할 수 있다. PX4 Offboard 제어 설계 시, 제어기가 버틸 수 있는 최대 시간 지연(Maximum Tolerable Delay)은 이 평균 연쇄 손실 지속 시간보다 반드시 커야만 안전 비행이 보장된다.

3. PX4 및 Ardupilot 시스템 연계성

  • PX4-Autopilot: commander 모듈의 파라미터 COM_OF_LOSS_T (Offboard Loss Timeout)는 위와 같이 모델링된 연쇄 손실(Burst Loss) 지속 시간을 수용할 수 있을 만큼 길게, 반면 기체가 물리적으로 위험한 한계 각도나 속도에 도달하기 전에는 Failsafe가 발동되도록 짧게 설계되어야 한다. 실험적 시스템 식별(System Identification) 및 마르코프 모델 분석을 통해 도출된 채널의 특성이 QGroundControl의 RC/GCS 타임아웃 튜닝에 직접적으로 반영될 수 있다.
  • Ardupilot: Ardupilot은 TCP보다 UDP 기반의 MAVLink 연결을 권장하지만, 시리얼 브리징 등 혼합 매체를 사용하는 경우 모델링 변수가 더욱 복잡해진다. FS_GCS_ENABL 및 관련 타임아웃 세팅 값 역시 위 확률론적 버스트 에러 예측 길이를 토대로 제어 시스템 내성을 확보하는 방식으로 최적화되어야 한다.

4. 시사점 및 학술적 평가

단순한 지상 관찰 앱개발을 넘어서, 진정한 공학적 GCS 및 Companion Computer 제어 시스템을 설계하려면 단순한 Socket 연동에 그치지 않고 통신 채널에 대한 수학적 통찰력을 지녀야 한다. 학술 및 필드 시스템에서는 상술한 마르코프 체인 손실 모델을 로컬(Local)의 모델 예측 제어기(MPC)나 확장 칼만 필터(EKF)의 공분산 행렬 업데이트 오프셋으로 주입하여, 네트워크가 ‘Bad’ 상태에 빠진 구간에서도 앞선 상태값을 외삽(Extrapolation)하여 비행 안정성을 능동적으로 보호하는 스키마가 개발 및 응용되고 있다.
