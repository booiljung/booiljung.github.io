49.1.2.1 확장 칼만 필터(Extended Kalman Filter, EKF2)의 국소 위치(Local Position) 및 전역 위치(Global Position) 혁신(Innovation) 일관성 검증

PX4-Autopilot의 상태 추정기(State Estimator)인 EKF2(Extended Kalman Filter)는 무인항공기(UAV)의 안정적인 자율 비행을 보장하는 핵심 모듈이다. 오프보드(Offboard) 제어 모드나 정밀한 웨이포인트(Waypoint) 비행을 수행하기 위해서는 기체의 현재 위치에 대한 신뢰성 높은 데이터가 필수적이다. 본 섹션에서는 EKF2가 생성하는 국소 위치(Local Position) 및 전역 위치(Global Position) 추정치의 무결성을 검증하는 핵심 메커니즘인 혁신(Innovation) 일관성 테스트에 대해 상세히 분석한다.

1. EKF2 예측 및 업데이트 사이클 개요

확장 칼만 필터(EKF2)는 기본적으로 두 가지 단계, 즉 **예측(Prediction)**과 업데이트(Update) 단계를 반복하며 기체의 6자유도(6-DoF) 상태 변수(위치, 속도, 자세 등)를 추정한다.

  1. 예측(Prediction): IMU(Inertial Measurement Unit)의 고주파수(통상 250Hz 이상) 가속도 및 각속도 데이터를 적분하여 다음 동적 상태를 선제적으로 계산한다.
  2. 업데이트(Update/Correction): GPS(GNSS), 기압계(Barometer), VIO(Visual Inertial Odometry), 거리 측정기(Rangefinder) 등에서 비교적 저주파수로 들어오는 외부 관측 데이터(Measurement)를 이용하여 예측된 상태를 교정(Correction)한다.

이 교정 단계에서 가장 중요한 수학적 지표가 바로 **혁신(Innovation)**이다.

2. 혁신(Innovation) 메커니즘의 수학적 정의

혁신(Innovation), 또는 잔차(Residual)란 현재 센서가 측정한 실제 관측값과 EKF2의 수학적 모델이 예측한 관측값 간의 차이를 뜻한다.

수식으로 표현하면 다음과 같다.

y_k = z_k - h(x_{k|k-1})

  • y_k: 시간 k에서의 혁신 행렬 (Innovation)
  • z_k: 실제 센서 측정 벡터 (Measurement)
  • h(x_{k|k-1}): 현재 상태 예측치 x_{k|k-1}를 바탕으로 한 관측 모델 모델의 예측값

만약 기계적 진동, 센서 고장, 지자기 교란(Magnetic Anomaly), GPS 다중 경로(Multipath) 등으로 인해 엉뚱한 측정값이 들어오게 되면, 혁신 y_k의 값은 급격하게 증가한다. 혁신 값이 비정상적으로 크다는 것은 시스템 모델과 실제 관측치 사이에 심각한 모순이 발생했음을 나타낸다.

3. 국소(Local) 및 전역(Global) 위치 혁신 일관성 검증 (Innovation Consistency Test)

EKF2는 비정상적인 센서 관측치가 기체의 위치 추정 상태를 훼손하는 것을 막기 위해 **혁신 일관성 검증(Innovation Consistency / Gate Test)**을 수행한다.

이는 혁신 y_k가 예상되는 오차 공분산 범위를 벗어나는지 확인하는 통계적 테스트다.

3.1 혁신 공분산 (Innovation Covariance)

혁신 \mathbf{S}의 공분산 행렬은 다음과 같이 계산된다.

\mathbf{S}_k = \mathbf{H}_k \mathbf{P}_{k|k-1} \mathbf{H}_k^T + \mathbf{R}_k

  • \mathbf{H}_k: 관측 행렬의 야코비안(Jacobian)
  • \mathbf{P}_{k|k-1}: 시스템 오차 공분산 (Error Covariance)
  • \mathbf{R}_k: 관측 노이즈 공분산 (Measurement Noise Covariance)

3.2 정규화된 혁신 제곱 (Normalized Innovation Squared, NIS)

관측의 유효성을 판별하기 위해, 정규화된 혁신 제곱값인 검정 통계량을 구한다. 이는 마할라노비스 거리(Mahalanobis Distance)의 제곱과 같다.

\text{Test Ratio} = y_k^T \mathbf{S}_k^{-1} y_k

이 Test Ratio가 EKF2 시스템 파라미터(예: EKF2_GPS_V_GATE, EKF2_GPS_P_GATE)로 설정된 임계값(Threshold, 통상 카이제곱 분포(Chi-squared distribution)를 따름)을 초과할 경우, EKF2는 해당 관측 데이터를 즉각 거부(Reject)한다.

3.3 위치 혁신 일관성에 따른 플래그 제어

PX4의 EKF2 로직에서는 위치(Position) 혁신 검증 결과를 기반으로 내부 상태 플래그를 실시간으로 제어한다.

  • 국소 위치 유효(Local Position Valid): 주로 IMU와 VIO, Optical Flow 등의 융합 결과로 초기화 지점(Home) 대비 상대 경로가 추적될 때 활성화된다.
  • 전역 위치 유효(Global Position Valid): GPS(GNSS) 신호가 안정적으로 수신되고, GPS 위치 혁신값(Test Ratio)이 일정 시간 동안 게이트 프레임을 연속으로 통과(Accept)할 때 비로소 true로 설정된다.

이 플래그들은 vehicle_local_position 및 vehicle_global_position uORB 토픽 내의 xyz_valid 필드 등에 반영되어 PX4의 핵심 제어기(Position Controller)와 커맨더(Commander)로 전파된다. 만약 GPS 글리치(Glitch)가 발생하여 혁신 일관성 테스트를 지속해서 통과하지 못하면, EKF2는 해당 스팬 동안 전역 위치 유효 플래그를 false로 강제로 내린다.

4. 커맨더(Commander)의 모드 천이(Mode Transition) 거부 매커니즘

이렇게 판별된 EKF2의 국소 및 전역 위치 유효성 플래그는 PX4의 비행 모드 변경 인가 로직에 가장 중요한 기준표가 된다.

GCS(QGroundControl)나 MAVSDK를 통해 Position Mode, Mission Mode, 또는 Offboard Mode로 변경을 지시하면 커맨더 모듈은 아래의 내부 검증을 거친다.

graph TD
    A[MAVLink SET_MODE 명령 수신: Position/Offboard] --> B{vehicle_local_position.xyz_valid == true ?}
    B -- Yes --> C{vehicle_global_position ? (Global 필요시)}
    B -- No --> D[Rejection: EKF Local Position Error]
    C -- Yes --> E[모드 변경 승인 (Accepted)]
    C -- No --> F[Rejection: EKF Global Position Error]
    
    D --> G[GCS에 MAV_RESULT_TEMPORARILY_REJECTED 전송]
    F --> G

만약 EKF2 내에서 혁신 일관성 실패로 인해 유효성 플래그가 소실된 상태라면, 커맨더는 외부의 제어 모드 진입 요청을 즉시 **거부(Reject)**하고 시스템 메시지로 “Reject Position Control” 등을 출력한다. 이는 센서 데이터 융합이 불완전하여 기체가 제어 루프를 정상적으로 추종할 수 없기 때문에 발생하는 치명적인 사고를 미연에 방지하기 위한 페일세이프(Failsafe) 아키텍처의 핵심 기능이다.

5. ArduPilot과의 차이: 혁신 통제 철학 (Ardupilot 대비 차이)

혁신 일관성을 대하는 철학에서 PX4와 ArduPilot(EKF3)은 약간의 차이를 보인다.

  • PX4-Autopilot: 다소 보수적(Conservative)이고 엄격(Strict)하다. 센서 융합 중 혁신 테스트 비율(Innovation Test Ratio)이 설정된 게이트를 초과하면 해당 센서를 즉각 차단하고, 그 시간이 길어질 시 즉각적으로 위치 제반 모드를 정지 및 강등(예: Altitude -> Stabilized 모드 강등 등) 시키는 안전 우선주의 철학을 따른다.
  • ArduPilot: EKF3의 EKF_Failsafe 로직은 센서 글리치가 감지되었을 때, 다중 IMU와 컴패스를 이용한 레인(Lane) 전환(Lane Switching) 기능이 매우 공격적으로 설계되어 있다. 또한 특정 상황에서는 관성 항법(Dead Reckoning)을 보다 오랫동안 신뢰하여 모드 강등을 최대한 지연(Delay)시키며 비행을 어떻게든 유지하려는 특성을 띤다.

6. 결론

EKF2의 국소 위치 및 전역 위치 혁신(Innovation) 일관성 검증 메커니즘은, 부정확한 측정치가 무인항공기의 내부 상태 모델을 교란하는 것을 막아주는 ‘방화벽(Firewall)’ 역할을 한다. GCS 개발이나 실시간 자율 에이전트를 위한 제어 애플리케이션 개발 시, 개발자는 외부 제어 루프를 코딩하는 것에 그치지 않고, EKF2의 혁신 상태(예: estimator_status의 test ratios 필드)를 지속적으로 감시하여 기체가 언제 외부 제어 권한을 잃고 Failsafe 로직으로 빠질지를 선제적으로 예측하는 아키텍처를 반드시 고려해야 한다.
