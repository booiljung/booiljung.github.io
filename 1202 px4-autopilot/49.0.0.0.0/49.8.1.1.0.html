49.8.1.1. 국소 위치(Local Position) 및 궤적 지령(Trajectory Setpoint) 메시지 비교를 통한 추종 오차(Tracking Error) 분산 계산

자율 주행 에이전트의 오프보드(Offboard) 제어 성능을 정량적으로 입증하는 가장 기본적인 방법은 지령값(Setpoint)과 실제 상태값(State) 간의 오차를 확률 및 통계적 기법으로 분석하는 것이다. PX4-Autopilot 환경에서는 이러한 분석이 MAVLink를 통해 인가되는 trajectory_setpoint uORB 메시지와 기체의 EKF(Extended Kalman Filter)가 추정하는 vehicle_local_position uORB 메시지의 직접적인 비교를 통해 수행된다. 본 절에서는 추종 오차(Tracking Error)의 수학적 모델링과 그 분산(Variance)을 도출하는 상세한 과정을 다룬다.

1. 통계적 추종 오차(Tracking Error) 모델링

이산 시간(Discrete Time) 시스템으로 기록되는 ULog 데이터를 기반으로 할 때, 시점 k에서의 추종 오차 벡터 \mathbf{e}(k)는 3차원 공간(x, y, z) 좌표계에 대하여 다음과 같이 정의된다.

\mathbf{e}(k) = \mathbf{r}(k) - \mathbf{y}(k)

여기서 \mathbf{r}(k)는 trajectory_setpoint 메시지의 위치 지령값 행렬이며, \mathbf{y}(k)는 vehicle_local_position 메시지에서 계측된 기체의 실제 위치 행렬이다. 통상적으로 PX4-Autopilot은 NED(North-East-Down) 원형 좌표계를 사용하므로, 양의 z 방향은 지표면 하단을 향한다는 점에 각별히 유의해야 한다.

오차의 기댓값과 분산 산출

추종 오차에 기인한 시스템의 불안정성 빈도를 분석하기 위해 확률론적 접근을 수행한다. 측정된 전체 비행 세션 차원수를 N이라 할 때, 각 축(예: x축)에 대한 오차의 평균(Mean, 기댓값 \mu_{e_x})과 분산(Variance, \sigma_{e_x}^2)은 아래와 같이 계산된다.

\mu_{e_x} = \frac{1}{N} \sum_{k=1}^{N} e_x(k)
\sigma_{e_x}^2 = \frac{1}{N-1} \sum_{k=1}^{N} (e_x(k) - \mu_{e_x})^2

여기서 연산된 평균 오차 \mu_{e_x}가 0에서 크게 벗어난다면, 이는 PID 제어기 관점에서 적분기 누적 오차(Windup) 문제이거나 기체의 무게 중심(Center of Gravity) 불균형 등에 기인한 정상 상태 편차(Steady-State Bias)가 존재함을 의미한다. 반대로 분산 \sigma_{e_x}^2 (혹은 표준 편차 \sigma_{e_x})가 크다는 것은 외부 외란(풍압 등)에 취약하거나 비례 제어(Proportional) 게인이 부족하여 목표 지점 부근에서 넓게 진동하고 있음을 시사한다. 이러한 경우 MPC_XY_P (위치 비례 게인) 및 MPC_XY_VEL_P (속도 비례 게인) 파라미터의 조정이 요구된다.

2. 분산 연산을 위한 Python 구현

앞서 언급한 pyulog 및 pandas 파이프라인(49.8.1 참조)을 기반으로 하여, 타임스탬프 동기화(Time Alignment) 처리가 완료된 상태에서 분산을 계산하는 알고리즘은 다음과 같이 파이썬 스크립트로 구현할 수 있다.

import numpy as np
import pandas as pd

def calculate_tracking_error_statistics(x_setpoint_aligned, x_state_aligned):
    """
    동기화된 지령값 및 상태값을 바탕으로 오차의 통계 지표를 계산한다.
    
    Parameters:
        x_setpoint_aligned (np.ndarray): 보간된 X축 위치 지령
        x_state_aligned (np.ndarray): 보간된 X축 실제 위치
        
    Returns:
        dict: 오차 평균, 분산, RMSE
    """
    # 추종 오차(Tracking Error) 계산
    error_x = x_setpoint_aligned - x_state_aligned
    
    # 통계적 수치 도출
    mean_error = np.mean(error_x)
    variance_error = np.var(error_x, ddof=1) # 표본 분산 (N-1)
    std_dev_error = np.std(error_x, ddof=1)
    rmse = np.sqrt(np.mean(error_x**2))
    
    return {
        "Mean Error (m)": mean_error,
        "Variance (m^2)": variance_error,
        "Standard Deviation (m)": std_dev_error,
        "RMSE (m)": rmse
    }

3. 오차 분산 분석을 통한 디버깅 시나리오

  1. 위치 제어기 다이나믹스 평가: 만약 \sigma^2가 0.05 m^2 내외로 1SD(표준 편차)가 약 0.22m에 분포한다면, 일반적인 멀티로터의 GPS 기반 자율 주행에서는 허용되는 수치이다. 그러나 실내 VIO(Visual Inertial Odometry)나 정밀 RTK 모듈을 사용함에도 분산이 높다면 위치 제어기(Position Controller) 튜닝 실패를 확신할 수 있다.
  2. MAVLink 데이터 병목 감지: 의도치 않은 통신 병목으로 인하여 GCS 명령어 전송 주기(예: 30Hz)가 불규칙해지는 경우, 궤적 지령 데이터의 파이프라인에서 계단 현상(Step)이 길게 유지되고 오차 분산이 폭증하게 된다. 이를 방지하기 위해서는 QGroundControl의 C++ 백엔드 로직 스레드 비동기성을 재점검하고 MAVLink 네트워크 포트의 다중화 병렬 구조를 도입해야 한다.

통계적 분산 분석은 이처럼 ULog 데이터만으로 기체 제어기와 지상 관제 시스템 양측의 오류를 가리키는 강력한 진단 지표가 된다. 차후 주파수 응답 기법과의 교차 검증을 통해 시스템의 동역학적 대역폭을 심층 분석하게 된다.
