49.2.3.1 모드 전환 시점의 운동학적 상태 변수(Kinematic State Variable) 초기화 및 평활화(Smoothing) 알고리즘

무인항공기(UAV)의 비행 중 제어 모드(Flight Mode)를 전환하는 과정은 기체의 안정성과 임무 연속성을 보장하기 위해 매우 정교한 제어 로직을 요구한다. 특히 위치 제어(Position Control) 모드에서 실시간 외부 명령에 의해 제어되는 오프보드(Offboard) 모드 또는 유도(Guided) 모드로 전환될 때, 혹은 그 반대의 상황에서 제어 지령(Setpoint)의 급격한 불연속성이 발생할 수 있다. PX4-Autopilot(기준 펌웨어 버전: v1.14 및 v1.15)은 이러한 비약적 변화(Jump)를 방지하고 부드러운 모드 천이를 달성하기 위해 운동학적 상태 변수(Kinematic State Variable)의 초기화(Initialization) 및 평활화(Smoothing) 알고리즘을 내부적으로 수행한다. 본 절에서는 해당 알고리즘의 수학적, 구조적 메커니즘과 구현 방식에 대해 심층적으로 다룬다.

1. 운동학적 상태 변수와 모드 전환의 문제점

비행 기구학(Flight Kinematics)에서 상태 변수(State Variable)는 기체의 3차원 위치(Position), 속도(Velocity), 가속도(Acceleration), 자세(Attitude) 및 각속도(Angular Rate) 등을 포함한다. 조종사가 수동 조종(Manual Control) 모드에서 비행하다가 자율 비행(Autonomous Flight) 모드로 전환하는 순간, 이전 모드에서 생성된 목표 궤적 지령과 새로운 모드에서 인가되는 목표 궤적 지령 간에는 필연적으로 오차가 존재한다.

만약 모드 전환 시점에 새로운 경로 지령(Target Setpoint)을 아무런 전처리 없이 제어기(Controller)에 직접 인가하면, 위치 오차(Position Error)나 속도 오차(Velocity Error)가 순간적으로 급증하여 비례-적분-미분(PID) 제어기의 미분기(Differentiator)와 비례기(Proportional)에서 과도한 제어 입력(Control Effort)을 생성하게 된다. 이는 모터의 포화(Actuator Saturation) 현상을 유발하고, 극단적인 경우 기체의 피치(Pitch)나 롤(Roll) 각도를 한계치까지 흔드는 위험한 기동을 초래할 수 있다. 이처럼 제어 지령의 급격한 변화를 방지하고 안정적으로 제어권을 이양하는 과정을 제어 공학에서는 범플리스 트랜스퍼(Bumpless Transfer)라고 명명한다.

2. 상태 변수 초기화(State Initialization) 알고리즘

PX4-Autopilot은 모드가 전환되는 트리거(Trigger) 시점에 현재 기체의 추정 상태(Estimated State)를 기반으로 제어기의 운동학적 상태 변수를 동기화(Synchronization)하는 초기화 과정을 수행한다. 이를 통해 목표 지령과 현재 상태 간의 초기 오차를 0으로 만들거나 최소화한다.

  1. 상태 추정기(Estimator) 데이터 로드: EKF2(Extended Kalman Filter 2) 모듈에서 추정된 현재의 글로벌/로컬 위치(로컬 NED 좌표계 기준), 속도, 요(Yaw) 각도 정보를 읽어온다(uORB 토픽: vehicle_local_position).
  2. 세트포인트(Setpoint) 리셋: 비행 관리자(Flight Mode Manager) 산하의 비행 태스크(Flight Task)가 활성화되는 순간, 궤적 생성기(Trajectory Generator)의 초기 시작점을 현재 기체의 실제 상태 값으로 강제 할당한다.
  3. 가속도 및 저크(Jerk) 제로화: 급격한 과도 응답을 막기 위해 궤적 생성의 초기 가속도 지령과 저크 지령을 0으로 둔고, 스팬(Span)을 설정해 나간다.

이러한 로직은 PX4-Autopilot 소스 트리 내부의 src/modules/flight_mode_manager/tasks/FlightTask 클래스를 상속받는 하위 태스크들(예: FlightTaskOffboard, FlightTaskManualPosition 등)의 인스턴스가 활성화(Activate)될 때 호출되는 activate() 함수 내에 구현되어 있다.

3. 궤적 평활화(Trajectory Smoothing) 및 클리핑(Clipping) 메커니즘

초기 상태가 기체의 현재 상태와 동기화되었다 하더라도, 사용자가 GCS(Ground Control Station)나 MAVROS, MAVSDK 등을 통해 전달한 최종 목표 지령(Target Setpoint)까지 도달하기 위한 경로는 여전히 평활화 대상이다.

PX4에서는 이러한 평활화를 위해 크게 두 가지 기법을 복합적으로 사용한다.

  • 슬루 레이트 리미터(Slew Rate Limiter): 제어 변수의 시간에 대한 변화율(미분값)에 상한선을 두는 기법이다. 예를 들어, 오프보드 모드에서 하드웨어 스펙을 초과하는 급격한 속도 지령이 MAVLink 메시지(SET_POSITION_TARGET_LOCAL_NED)를 통해 들어올 경우, 위치 제어기(FlightTask)는 해당 속도 지령의 변화율, 즉 가속도가 설정된 파라미터(예: MPC_ACC_HOR_MAX, MPC_ACC_UP_MAX 등)를 초과하지 않도록 선형적으로 제한한다.
  • 저주파 통과 필터(Low-pass Filter): 불연속적인 단계 입력(Step Input)을 부드러운 곡선 형태로 변환하기 위해 1차 또는 2차 저주파 통과 필터를 적용한다. 지령값이 변경될 단, 지수적(Exponential) 곡선이나 S 커브(S-curve) 형태의 감쇠를 유도하여, 드론의 기구학적 동적 특성(Dynamic Characteristics)에 부드럽게 융화되도록 한다.

아래 수식은 1차 저주파 통과 필터를 이산 시간(Discrete Time) 상에서 구현한 대표적인 예시다.
y_k = \alpha x_k + (1 - \alpha) y_{k-1}
여기서 x_k는 새로운 목표 지령(Target Setpoint), y_k는 평활화되어 제어기에 인가되는 최종 지령(Smoothed Setpoint), y_{k-1}은 이전 스텝의 지령, \alpha는 샘플링 시간과 시정수(Time Constant)에 의해 결정되는 평활화 계수(Smoothing Factor)이다. 필터가 강하게 걸릴수록 지령 추종의 지연(Latency)은 길어지지만 모션은 훨씬 부드러워진다.

결론

선형 및 비선형 제어에 있어서 초기 조건의 불일치는 시스템 불안정성을 야기하는 핵심 원인이다. PX4-Autopilot은 FlightModeManager를 중심으로 모드 간 전환 시 추정 상태와 지령 간의 오차를 0으로 강제화하는 동기화 메커니즘을 적용하며, 전환 이후에도 가속도 제한, 저크 제한, 저주파 통과 필터링을 포괄하는 궤적 평활화(Trajectory Smoothing) 로직을 구동한다. 이러한 일련의 과정은 MAVLink 프로토콜을 통해 빈번하게 발생하는 외부 제어 개입과 자율 비행 알고리즘 간의 상호작용 속에서도 드론이 물리적 한계를 초과하지 않고 부드럽고 일관된 거동을 유지하게 하는 필수 공학 기반을 제공한다.
