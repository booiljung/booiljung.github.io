49.6.3.1 유한 시간 구간(Finite Time Horizon) 내에서의 시스템 제어 입력(Control Input) 최적화 및 페널티 함수(Penalty Function) 설계

1. 개요

모델 예측 제어(MPC)가 최고의 성능을 발휘하기 위해서는 주어진 수학적 비용 함수(Cost Function)를 얼마나 정교하게 설계하느냐가 관건이다. 유한 시간 구간(Finite Time Horizon) 내에서 드론의 미래 움직임을 예측하고, 해당 예측 구간 안에서 발생하는 참조 궤적과의 위치/속도 오차와 이를 교정하기 위해 소모되는 제어 에너지의 크기를 일종의 벌점인 ’페널티 함수(Penalty Function)’로 산정한다. 이 페널티 총합을 최소화하는 해(Solution)가 곧 현재 스텝의 최적 제어 입력(Optimal Control Input)이 된다.

2. 상태 공간 평방정식과 유한 예측 구간(Finite Horizon)

무인항공기(UAV)의 비선형 동역학은 선형화 및 이산화(Discretization)를 거쳐 x_{k+1} = A x_k + B u_k 형태의 상태 공간 모델로 변환된다. 여기서 x_k는 상태 벡터(위치, 속도, 자세 등)이며, u_k는 제어 입력 벡터(추력, 각속도 지령 등)이다.

무한한 미래를 모두 계산하는 것은 실종된 컴퓨팅 파워로는 불가능하므로, 시스템은 현재 시각부터 정해진 스텝 N(예: 0.05초 간격으로 20스텝 = 1초)까지만 앞을 내다보는 유한 시간 구간(Finite Time Horizon) 을 설정한다. 이 N 값(Prediction Horizon)이 너무 짧으면 제어기가 궤적 변화를 인지하지 못하고 오버슈트가 발생하며, 너무 길면 PX4나 Companion PC(ROS 2) 프로세서의 연산 부하가 치솟게 된다.

3. 페널티 함수(Penalty Function)의 설계와 이차 구조

가장 널리 쓰이는 MPC의 형태인 이차 계획법(Quadratic Programming)에서 페널티 함수인 목적 함수(Objective Function) J는 다음과 같이 정의된다.

J = \sum_{k=0}^{N-1} \left[ (x_k - x_{ref, k})^T Q (x_k - x_{ref, k}) + u_k^T R u_k \right] + (x_N - x_{ref, N})^T P (x_N - x_{ref, N})

위 식에서 핵심이 되는 설계 파라미터는 Q, R, P의 세 가지 페널티 행렬(가중치 행렬)이다.

  • 상태 오차 페널티 행렬 (State Error Penalty Matrix, Q): 궤적을 얼마나 정확하게 따라갈 것인지를 결정한다. 대각 행렬(Diagonal Matrix)의 형태로 설계되며, Q의 위치 성분 값을 크게 할수록 시스템은 오차를 조금이라도 허용하지 않으려 매우 공격적으로 반응한다.
  • 제어 입력 페널티 행렬 (Control Effort Penalty Matrix, R): 제어 명령을 내는 행위(모터 회전속도 변경 등) 자체에 부여하는 벌점이다. 배터리를 보존하고 부드러운 움직임(기계적 스트레스 저감)을 원할 경우 R 성분 값을 증가시킨다.
  • 종단 상태 페널티 행렬 (Terminal Cost Matrix, P): 예측 구간의 가장 마지막 지점 N에서의 오차를 특별히 강하게 처벌하여, 유한 시간 최적화의 수학적 한계로 인한 폐루프 안정성(Closed-loop Stability) 저하 구역에 빠지는 것을 막는다.

4. 제어 입력 최적화와 드론의 ‘성향(Personality)’ 결정

위의 Q와 R 행렬을 어떻게 튜닝하느냐에 따라 PX4 기반 드론의 비행 성향이 완전히 뒤바뀐다.

  1. 공격적인 추종 (Aggressive Tracking; Q >> R): 위치 오차 페널티 Q가 제어 페널티 R보다 압도적으로 크면, 기체는 배터리를 과도하게 소모하거나 모터에 큰 부하(Jerk)를 주면서라도 오차를 없애려 맹렬하게 기동한다. 자율 드론 레이싱(Autonomous Drone Racing) 환경에서 흔히 채택되는 설정이다.
  2. 부드러운 비행 (Smooth Flight; R 비율 상향): 촬영용 드론(Cinematography)의 경우, 약간의 궤적 오차(Track Error)가 발생하더라도 짐벌 카메라에 전달되는 각속도 변화를 줄여야 한다. 따라서 R 값을 높게 설정하여 궤적을 둥글게 깎으면서 도는(Cutting Corners) 매우 온화한 비행을 연출할 수 있다.

5. 결론 및 실무적 고려사항

유한 시간 모델 예측 제어에서 페널티 함수를 수학적으로 올바르게 튜닝하는 것은 곧 개발자가 기체의 물리 시스템에 요구하는 트레이드오프(정밀도 vs 부드러움/에너지)를 정의하는 과정이다. ROS 2와 같이 분산된 구조로 구동되는 GCS/Companion PC 백엔드 환경에서, 동적 상황에 맞게 비행 중 실시간으로 이 Q, R 행렬 값을 스케줄링(Gain Scheduling)하는 고급 관제 기술은 기체의 성능을 한 차원 더 높은 자율성의 경지로 이끌어 준다.
