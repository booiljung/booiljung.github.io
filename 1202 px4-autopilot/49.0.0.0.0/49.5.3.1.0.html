49.5.3.1. 오차 쿼터니언(Error Quaternion) 지령에 기반한 3축 각속도(Angular Rate) 피드포워드(Feedforward) 생성

PX4-Autopilot의 역학 제어 시스템 내부에서 고수준(High-level)의 자세 명령이 어떻게 저수준(Low-level)의 모터 제어 명령으로 환원되는지를 이해하기 위해서는 쿼터니언(Quaternion) 수학과 피드포워드(Feedforward) 루프의 대수 결합을 파악해야 한다. 통신망을 통해 SET_ATTITUDE_TARGET 메시지로 하달된 외부 쿼터니언 지령 \mathbf{q}_{sp}(Setpoint Quaternion)는 기체의 현재 자세 추정치 \mathbf{q} 와 비교 연산되어 **오차 쿼터니언(Error Quaternion, \mathbf{q}_{error})**을 생성한다. 이렇게 내부 모듈에서 산출된 오차 행렬은 3축 각속도 피드백 루프의 기준 1차 입력(Reference Input)으로 작용함과 동시에, MAVLink 페이로드로 수신된 외부 3축 각속도(Angular Rate) 값과 결합하여 동역학적 전향 보상(Feedforward) 텀을 형성한다.

본 절에서는 쿼터니언 다차원 공간에서의 자세 오차 산출 원리와, 펌웨어 내부 자세 제어기(mc_att_control)가 이를 어떻게 3차원 각속도 목표치로 환산하고 결합(Blending)하여 최종 비행 성능의 공격성(Aggressiveness)을 끌어올리는지 심층적으로 다룬다.

1. 다차원 공간의 자세 오차: 오차 쿼터니언 수학 연산

유러 각(Euler Angle)과 달리 쿼터니언은 축(Axis)과 회전각(Angle)의 4차원 스칼라비 벡터 조합으로 공간 정지 특이점(Gimbal Lock) 없이 유려한 회전을 행렬식으로 연산할 수 있는 강력한 수학적 기법이다. PX4 내부 제어 스레드에서는 vehicle_attitude(ECL/EKF2가 융합해 낸 현재 기체 상태) 토픽과 vehicle_attitude_setpoint(오프보드로 전달받은 목표 상태) 토픽을 획득한 후, 가장 먼저 \mathbf{q}_{error} 보정량을 산출한다.

오차 쿼터니언은 수학적으로 “현재 자세 프레임으로부터 목표 자세 프레임으로 도달하기 위해 필요한 나머지 절대적 회전 변환“으로 정의되며, 쿼터니언 외적 곱셈 기호 \otimes와 켤레(Conjugate) 행렬 거듭제곱 연산을 통해 다음과 같이 유도 도출된다.

\mathbf{q}_{error} = \mathbf{q}_{sp} \otimes \mathbf{q}^{-1}
(단, 비행 제어기 내부의 \mathbf{q}는 정규화(Normalized)된 유닛 쿼터니언이므로 공간적 켤레 \mathbf{q}^* 는 역원 행렬 \mathbf{q}^{-1} 과 동일한 속성을 지닌다.)

이 방정식의 결과로 도출된 \mathbf{q}_{error} = [w_e, x_e, y_e, z_e]^T 는 각 축(Roll, Pitch, Yaw) 단위의 3차원 공간 내 가장 짧은 회전 경로(Shortest Path) 오차각 스칼라 성분을 내포하고 있다.

2. P 제어망을 통과한 각속도 피드백(Feedback) 도출 기법

PX4의 mc_att_control (멀티로터 자세 제어 C++ 클래스)는 오차 쿼터니언의 허수부(Vector Part: x_e, y_e, z_e)를 직접 추출하여 오차 각도를 도출하는 테일러 급수 근사화(Approximation) 수학 기법을 사용한다. 작은 각도 변위의 경우 사인(Sine) 함수 근사가 대수적으로 가능하므로, 이를 벡터 연산으로 추출해 낸 오차 행렬 \vec{e}_{rate\_fb}은 PX4 파라미터 시스템에 설정된 각 축 단위 독립 증폭 상수인 P 이득(Proportional Gain, K_p)과 복소 곱셈되어 기본 피드백 기반 목표 각속도를 산출한다.

\vec{V}_{rate\_fb} = 2 \times K_p \times \text{sign}(w_e) \times [x_e, y_e, z_e]^T

이 순수한 피드백 루프 제어만으로도 기체를 목표 자세로 회전 방사시킬 수는 있으나, 거대한 각도 변화 지령(Step Input)이 인가되었을 경우 모터의 공기 역학적 포화(Saturation)가 역산 발생하거나, P 이득의 하드 한계에 부딪혀 상승 시간(Rise Time) 지연 및 오버슈트(Overshoot)를 회피하기가 대단히 어렵다. 이를 극복하기 위해 소프트웨어적인 피드포워드 설계망이 접목된다.

2. 각속도 피드포워드(Feedforward) 주입 논리와 3축 연산 믹싱 블렌딩

외부 고성능 컴퓨터 시스템(예: ROS2 Agent)에서 SET_ATTITUDE_TARGET 메시지를 송신할 때, 비트마스크(Type Mask)의 IGNORE_RATE 관련 허가 비트들을 모두 0(유효)으로 설정하고 목표 롤/피치/요 각속도 행렬(\vec{V}_{rate\_ff})을 쿼터니언과 함께 동시 인가했다고 가정하자.

기체 내부의 자세 제어망 알고리즘은 피드백 각속도 \vec{V}_{rate\_fb} 와 MAVLink 통신망으로부터 직접 꽂힌 수동 외부 각속도 \vec{V}_{rate\_ff} 를 선형결합 연산하여, 하위계층 PID 각속도 제어 루프(mc_rate_control)로 넘길 궁극적인 절대 지시 각속도 토픽 패키지 vehicle_rates_setpoint 를 패키징한다.

\vec{\mathbf{\Omega}}_{target} = \vec{V}_{rate\_fb} + \vec{V}_{rate\_ff}

  • 피드포워드 주입의 물리적 의미: 기존의 순수 P 루프가 오차를 인지하고 공기 저항력을 추정하여 반응하기 이전에, 외부 컴퓨터가 수치해석학적으로 미리 연산 계산해 놓은 예정된 회전 속도 장력을 펌웨어의 각 모터 PWM 신호망에 먼저 실어주는 선제적 강제 타격(Pre-emptive Strike) 제어 기법이다.
  • 스텝 응답 비교 차질: 피드포워드 값이 0으로 누락 무시된 시스템은 목표 자세 도달 전 점진적 감소 속도를 갖는 S 커브 마감 성향을 갖지만, 피드포워드가 명시적으로 반영된 융합 제어 루프는 최대 상승 한계선(Rate Limit)까지 치솟아 즉각적으로 최단 시간 내에 해당 각도로 회전 이동한 후, 기계적 공력 역추진 브레이킹을 거는 가혹한 스포츠카와 같은 동역학 곡률 반응을 갖게 된다.

4. Ardupilot의 sqrt_controller와의 아키텍처적 비교 논점

쿼터니언 오차를 각속도 보정량으로 변환하는 소프트웨어적 수학 처리 과정에서 PX4-Autopilot과 Ardupilot 모두 선형 단위의 P-이득 캐스케이드(Cascade) 시스템을 차용하지만 한계 상황을 마주하는 비상 필터링 모듈에서 특성적 차이가 관찰된다.

  • Ardupilot (AC_AttitudeControl 알고리즘): 자세 제어 오차가 각속도로 변환되는 찰나의 과정에서 이른바 sqrt_controller 라 불리는 특유의 제곱근(Square Root) 기반 비선형 이득 알고리즘 서브루틴을 사용한다. 각도 오차가 작을 때는 선형 P 반응 모델링을 따르되, 오차 각도가 임계점 넘게 거대해지면 감쇄율 곡선(Square Root Profile)을 강제 적용하여 기체 물리적 한계 각속도 파라미터를 넘지 않도록 브레이킹 감속 존을 부드럽게 소프트웨어 자가 계산한다.
  • PX4-Autopilot (mc_att_control): 현대 제어 논문에 입각하여 오차 쿼터니언 추출 로직의 아키텍처 자체가 극단적인 순수 선형성(Linear) 지향을 추구한다. 선형 블렌딩 후 도출된 \vec{\mathbf{\Omega}}_{target} 자체를 MC_ROLLRATE_MAX 파라미터 등을 활용하여 하드 클리핑(Hard Clipping) 제약시켜 버리고 최후방의 방어를 Slew-rate(저크 동역학 제한) 로우-패스 필터망으로 후가공하는 구조를 띤다. 피드포워드 변수의 병렬 수용성에 있어 외부 알고리즘에 보다 맹목적이고 적극적인 모터 관제를 이뤄내는 모던 제어 이론 팩토리(Modern Control Theory Factory) 철학을 강하게 유지하고 있다.

5. 결론: 학술 논문 및 실무 적용 시의 고찰

오차 쿼터니언 기반의 피드백 연산에 3축 각속도 피드포워드 스칼라 텐서를 합산하는 SET_ATTITUDE_TARGET 제어 이중망 구조는, 펌웨어가 가진 근원적인 PID 하드웨어적 관성 딜레이를 소프트웨어적으로 극복하기 위한 GCS 연구자들의 궁극적인 우회 로직 설계라 할 수 있다.

시각 기반 유도(Visual-Inertial Odometry Guidance)나 강화학습 신경망 딥러닝 로직을 회피 비행 제어에 응용하는 데이터 파이프라인 소프트웨어 개발자는 펌웨어 내부의 P 이득과 외부 망의 피드포워드 게인(Gain) 간의 스칼라 의존성을 논문 구상 단계에서부터 세심하게 고려해야만 한다. 시스템이 도출한 목표 자세 곡선의 자코비안 미분값(Derivative)을 고스란히 피드포워드 각속도 배열망으로 매핑하여 전달해주면, 수신 드론 객체는 마치 메인 컴퓨터와 신경망이 하나로 융합 연결된 사이보그처럼 제어 이질감 없는 무반동 회전 곡예 모션을 보여줄 것이다.
반면, 비트마스크 네거티브 논리 설정 누락 등으로 각속도 항이 예외 NAN 처리되거나 어설프게 0.0 으로 결속(Bound)되어 하달될 경우, 맹렬하게 기동해야할 드론 시스템은 여전히 펌웨어 내부의 P 이득 상수의 물리적 한계 의존성에 갇혀, 학술 논문에서 선전하고자 기획된 ’제로 레이턴시(Zero Latency) 극한 스텔시(Stealthy) 고속 비행 목표’를 전혀 체감하지 못하는 물리적 추종 병목 현상의 늪에 빠지게 될 것임을 각별히 경계해야 한다.
