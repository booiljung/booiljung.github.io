42.3.6. 다중 중복 센서 퓨전 아키텍처 결함 격리를 위한 투표(Voting) 알고리즘

무인기 시스템(UAS)의 생존성과 신뢰성을 보장하기 위해, 현대의 비행 제어기(예: Pixhawk 4, 6X 등)는 관성측정장치(IMU), 지자기 센서, 기압계 등 주요 센서를 2개에서 3개 이상 중첩하여 탑재하는 다중 중복 센서 아키텍처(Multi-Redundant Sensor Architecture)를 채택한다. 본 절에서는 여러 센서에서 들어오는 측정치 중 정상 센서 데이터를 선별하고 결함이 발생한 센서를 격리(Fault Isolation)하기 위해 PX4-Autopilot(Firmware v1.14 이상 기준) 내부의 센서 허브 및 확장 칼만 필터(EKF2) 단위에서 수행되는 투표 알고리즘(Voting Algorithm) 로직을 심층적으로 분석하라.

1. 센서 데이터 취합 및 uORB 메시지 처리 구조

PX4 하드웨어 계층에서는 SPI 또는 I2C 버스를 통해 각각 독립적으로 다중 센서 데이터를 수집한다. 수집된 원본(Raw) 데이터는 드라이버(Driver) 레벨에서 기본적인 무결성 검사를 거친 후, uORB(Micro Object Request Broker) 버블리셔에 의해 개별 토픽으로 발행된다.

  • 개별 센서 모듈(Sensor Module): 물리적으로 여러 개인 센서는 sensor_accel, sensor_gyro 등 동일한 토픽명을 가지지만, sensor_accel_0, sensor_accel_1, sensor_accel_2와 같이 고유한 인스턴스로 분리되어 관리된다.
  • 센서 데몬(Sensors Daemon): 개별 인스턴스들을 모두 구독(Subscribe)한 후, 비행 제어 로직(Flight Control Logic)과 EKF2 추정기에 제공하기 위해 단일화된 vehicle_imu 토픽으로 융합하는 투표 파이프라인을 거친다.

2. 센서 투표 알고리즘의 작동 원리

PX4의 결함 격리 로직은 이노베이션(Innovation) 기반의 다수결 투표(Majority Voting) 방식에 우선순위 가중치 기법을 더하여 동작한다. 소스 코드 상에서는 src/modules/sensors/voted_sensors_update 계층 및 멀티 EKF 아키텍처 내부에서 핵심적인 역할을 수행한다.

알고리즘 구성요소기능적 역할
주 센서 선정(Primary Sensor Selection)시스템 부팅 및 초기화 단계에서 측정 노이즈 모델이 가장 작고 캘리브레이션 무결성이 높은 센서를 주(Primary) 센서로 자동 지정한다.
이노베이션 잔차(Innovation Residual) 분석주 센서의 관측치를 기준으로 보조(Secondary) 센서 간의 상태 오차 거리를 산출한다. 잔차 수식 예시: \Delta_i = \vert Z_i - \hat{Z}_{primary} \vert
결함 임계치 누적(Fault Threshold Accumulation)각 센서의 \Delta_i가 정해진 편차 임계치(\sigma)를 연속해서 초과하거나 하드웨어 타임아웃이 발생할 경우, 해당 센서에 페널티 점수(Penalty Score) 에러 카운터를 증가시킨다.
페일오버 및 결함 격리(Failover & Isolation)주 센서의 페널티 점수가 치명적 한계 도달 시, 해당 센서의 토픽 수신을 차단(격리)하고, 잔차가 가장 낮고 안정적인 차순위 센서를 즉각적으로 주 센서로 전환한다.

이러한 로직은 하드웨어 단선, 극심한 기계적 진동으로 인한 에일리어싱(Aliasing) 현상 또는 버스 에러 스파이크를 효과적으로 우회하여, 오염된 센서 데이터가 제어기 루프 내로 유입되는 것을 방지한다.

3. Ardupilot의 결함 허용(Fault Tolerance)과 PX4 간의 비교

  • Ardupilot: 전통적으로 여러 개의 독립적인 EKF 구조(EKF3 코어)를 생성하여, 각각의 코어가 서로 다른 센서 조합을 기반으로 병렬 상태 추정을 수행한다. 그리고 도출된 최종 상태 추정치(Final Output) 간에 투표 프로세스(Lane Switch)를 진행하여 최종 자세와 위치를 결정하는 구조에 가깝다.
  • PX4-Autopilot: 병렬 EKF 코어(Multi-EKF) 토폴로지를 지원함과 동시에, 그보다 앞단인 센서 데몬(sensors 모듈) 계층에서 원시 데이터를 직접 투표(Sensor Level Voting)하여 가장 신뢰할 만한 단일 센서 스트림으로 전처리하는 파이프라인이 발달되어 있다. 이는 EKF 코어로 유입되는 원천적인 노이즈를 감소시켜, 제한된 시스템 자원에서 효율성을 극대화하는 아키텍처적 특성이다.

5. QGroundControl 연동과 관리자의 실시간 모니터링

관제자는 QGroundControl(QGC) 인터페이스 및 파라미터 설정을 통해 알고리즘 개입 강도를 능동적으로 제어할 수 있다.

  • 통제 파라미터: SENS_IMU_MODE, EKF2_MULTI_IMU 등 파라미터를 통해 다중 EKF 운영 모드 및 센서 다수결 투표의 적용 여부를 설정할 수 있다.
  • 관제 피드백: 비행 중 주 센서의 페일오버(Failover)가 발동할 경우, SYS_STATUS 메시지 내 시스템 정상 비트가 변경되어 QGroundControl에 알림을 전송한다. 화면에는 “Sensor Fusion Degraded” 또는 “Primary IMU switched“와 같은 위험 메시지가 시현되며, 지상 관제자는 이를 인지하고 즉각 비상 복귀(RTL) 또는 착륙명령을 인가하는 오퍼레이션 절차를 수행해야 한다.

6. 결론

다중 중복 센서 투표 알고리즘은 결함 인지 및 격리를 실시간으로 수행함으로써, 단일 센서 하드웨어 고장에서 비롯된 치명적 추락 사고를 방어하는 가장 근본적이고 수학적인 논리 장치이다. PX4의 전처리 센서 투표 파이프라인은 정밀도 높은 이노베이션 매트릭스를 기반으로 구축되어 학술적인 신뢰성을 보장한다.
