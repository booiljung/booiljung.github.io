42.5.1.1. 쿨롱 계수(Coulomb Counting) 적분 오차 보정을 위한 개방 회로 전압(Open Circuit Voltage) 비선형 매핑

1. 쿨롱 계수(Coulomb Counting)의 원리와 적분 오차(Integration Drift)의 근본적 한계

무인항공기(UAV)의 안전한 비행 및 상태 관제를 위해서는 배터리의 실시간 잔존 용량(State of Charge, SoC)을 정확하게 추정하는 것이 필수적이다. PX4-Autopilot은 이를 위해 일차적으로 쿨롱 계수(Coulomb Counting) 방식을 사용한다. 쿨롱 계수는 배터리로 유출입되는 전류를 시간에 대해 적분하여 현재의 SoC를 계산하는 방식이다. 수식으로 표현하면 다음과 같다.

SoC(t) = SoC(t_0) - \frac{1}{C_{nom}} \int_{t_0}^{t} I(\tau) d\tau

여기서 SoC(t_0)는 초기 배터리 잔량, C_{nom}은 배터리의 정격 용량(Nominal Capacity), I(\tau)는 방전 전류이다.

이 방식은 단기적인 배터리 소모량을 매우 정밀하게 추적할 수 있다는 장점이 있으나, 시간 단위가 길어질수록 적분 오차가 누적되는 치명적인 단점을 지닌다. 전류 센서의 계측 노이즈(Measurement Noise), 온도 변화에 따른 오프셋(Offset) 드리프트, ADC 변환 오차 등이 포함된 측정 전류를 단위 시간마다 계속 누적하기 때문이다. 특히, 비행 시간이 길어지는 고정익(Fixed-wing)이나 VTOL(Vertical Take-Off and Landing) 기체의 경우 누적된 오차로 인해 실제 배터리는 고갈되었음에도 GCS(지상 관제 시스템)에는 여유가 있는 것으로 잘못 보고될 수 있으며, 이는 심각한 추락 사고로 직결된다.

2. 개방 회로 전압(Open Circuit Voltage) 비선형 매핑을 통한 적분 오차 보정 설계

이러한 쿨롱 계수의 적분 오차를 해결하기 위해, PX4-Autopilot은 개방 회로 전압(Open Circuit Voltage, OCV)과 SoC 간의 비선형적인 관계를 활용하여 장기적인 관점에서 오차를 보정(Correction)한다. OCV는 배터리에 부하가 연결되지 않은 상태(전류가 0인 상태)에서의 순수 전압을 의미한다. 리튬 폴리머(LiPo)나 리튬 이온(Li-ion) 배터리의 화학적 특성상, OCV와 SoC는 고유의 비선형 곡선 형태를 그린다.

PX4-Autopilot은 부하가 걸린 상태에서 측정된 단자 전압(Terminal Voltage)과 내부 저항(Internal Resistance)에 의한 전압 강하(Voltage Drop)를 계산하여, 간접적으로 현재의 OCV를 역추산한다.

V_{OCV} = V_{terminal} + I \cdot R_{internal}

역추산된 V_{OCV}는 PX4 펌웨어 내부에 하드코딩된 셀 모델 혹은 사용자가 정의한 파라미터 기반의 배터리 방전 곡선(Discharge Curve) 매핑 테이블에 대입되어 독립적인 전압 기반 추정 SoC(SoC_{OCV})를 도출한다. 최종적으로 쿨롱 계수를 통해 얻은 SoC_{CC}와 전압을 통해 매핑된 SoC_{OCV}를 상호 보완적으로 융합하는 필터링 루틴(예: 상보 필터 혹은 저주파 통과 필터 역산 적용)을 통해 배터리의 최종 SoC를 결정한다.

1.1 Ardupilot 대비 PX4-Autopilot의 배터리 상태 추정 및 관제 차이

PX4-Autopilot과 Ardupilot 모두 쿨롱 계수와 실시간 전압 측정 기반의 융합 방식을 사용하지만, 내부 구조와 보정 민감도에서 뚜렷한 차이를 보인다.

Ardupilot은 전통적으로 전압 기반 추정과 전류 가스 게이지(Gas Gauge) 기반 추정을 사용자가 설정한 가중치 파라미터(예: BATT_CAPACITY, BATT_VOLT_MULT 등)에 의존하여 단순 비례 혼합(Proportional Blending)하는 방식의 색채가 강하다. 만약 사용자 설정이 정밀하지 않으면 배터리 잔량 퍼센티지가 갑자기 널뛰는(Jumping) 현상이 종종 발생한다.
반면, PX4-Autopilot의 배터리 유틸리티(Battery Status 모듈) 모델은 배터리의 동적 임피던스를 모델링하는데 보다 진보된 방식을 제공한다. PX4는 BAT_V_EMPTY (완전 방전 상태에서의 셀 전압), BAT_V_CHARGED (완충 상태에서의 셀 전압), BAT_N_CELLS 파라미터 등을 복합적으로 계산하고 스로틀 레벨(Throttle Level)에 따른 부하(Load) 변화율에 따라 OCV 가중치를 비선형적으로 슬루 레이트(Slew Rate) 제한을 걸어 부드럽게 보정한다. 이는 비행 중 GCS인 QGroundControl에 급격한 잔존 용량 변화를 보여주어 조종자에게 혼란을 주는 것을 방지하고, 통신 링크상의 상태 보고 신뢰성을 높여주는 역할을 한다.

2. PX4-Autopilot 배터리 추정 아키텍처 및 소스 코드 심층 이해

PX4 소스 코드 내에서 배터리 상태 처리는 주로 src/modules/battery_status/ 경로의 아키텍처에 구현되어 있다. 특히 Battery.cpp 클래스에서 이러한 비선형 매핑과 융합 로직을 확인할 수 있다. ADC 드라이버로부터 얻은 전압, 전류 원시 데이터는 uORB 메시지인 sensor_battery 또는 adc_report를 통해 퍼블리시(Publish)된다. 이를 배터리 상태 모듈이 서브스크라이빙(Subscribe)하여 연산을 수행한 뒤, 최종 산출물인 battery_status 토픽을 재발행한다.

2.1 전압 기반 SoC 매핑 알고리즘 코드 구조 예시

배터리 모듈 내부에서는 전류 적분을 지속적으로 갱신하되, 일정 주파수 간격 혹은 배터리 부하가 최소인 구간에서 전압 기반 SoC를 계산하여 적분기를 업데이트한다.

// 펌웨어 내부 배터리 전압-SoC 비선형 매핑 모사 로직 (개념적 예시)
float Battery::estimate_soc_from_voltage(float filtered_vol, float current, float internal_res) {
    // 1. 내부 저항과 실시간 전류를 이용한 OCV 역산
    float estimated_ocv = filtered_vol + (current * internal_res);
    
    // 2. 1셀 당 OCV로 변환
    float cell_ocv = estimated_ocv / _params.n_cells;
    
    // 3. 비선형 OCV 구간 테이블 매칭 및 선형 보간 (Linear Interpolation)
    float soc_voltage_based = 0.0f;
    if (cell_ocv >= _params.v_charged) {
        soc_voltage_based = 1.0f;
    } else if (cell_ocv <= _params.v_empty) {
        soc_voltage_based = 0.0f;
    } else {
        // 배터리 방전 특성 곡선 LUT(Look-Up Table)를 참조하여 SoC 추출
        soc_voltage_based = lookup_soc_table(cell_ocv);
    }
    
    return soc_voltage_based;
}

위와 같이 battery_status 노드에서 계산된 정보는 QGroundControl과 같은 GCS에 MAVLink BATTERY_STATUS (#147) 패킷으로 변환되어 주기적으로 전송된다.

4. 결론 및 GCS 관제(Pre-flight Check)와의 연관성

PX4-Autopilot의 정교한 개방 회로 전압(OCV) 기반 쿨롱 계수 보정 메커니즘은 단순히 화면상에 잔량을 표기하는 것을 넘어 무인기 시스템 공간 안전성 검증의 핵심이 된다. QGroundControl은 MAVLink를 통해 받은 정밀한 SoC를 바탕으로, 기체의 잔여 비행 가능 시간을 역산(Time-to-Empty)하고, 임무(Mission) 시작 전 Pre-flight Check 단계에서 현재 등록된 웨이포인트(Waypoint)를 모두 순회하고 복귀할 배터리 에너지가 충분한지 사전 승인 단계에서 엄격하게 판별한다. 나아가 데이터 링크 오류(Data Link Loss)에 따른 GCS와의 통신 단절 상황에서 잔존 용량을 바탕으로 바로 착륙할 것인지 복귀(RTL)할 것인지 독립적인 의사 결정을 하도록 비상 복구 시스템(Failsafe Mechanism)의 기준 척도를 제공하는 중추적인 역할을 담당한다.
