42.3.6.1.1. 공분산 교차(Covariance Intersection) 기법을 활용한 상호 의존적 다중 센서 노이즈 대역의 통계적 독립성 검증

무인기 비행 제어 시스템의 상태 추정(State Estimation) 과정에서 두 개 이상의 분산된 센서가 물리적, 환경적으로 동일한 영향(예: 동일한 방진 댐퍼 공유, 동일한 마운트 내 전자기 간섭)을 받을 때, 비록 독립적인 회로라 할지라도 각 측정 노이즈는 심각한 상호 의존성(Cross-correlation)과 대역 증폭 현상을 겪게 된다. 확장 칼만 필터(EKF2)는 이러한 다중 측정의 노이즈가 통계적으로 독립적(Statistically Independent)이라고 가정하므로, 종속성 조건에서는 필터가 공분산을 과도하게 축소(Overconfidence)시켜 발산(Divergence)을 초래한다. 본 절에서는 이러한 다중 센서 간의 미확인 상호 의존성 한계를 극복하기 위해 통계적 독립성 검증 및 PX4-Autopilot 아키텍처에 적용 가능한 ‘공분산 교차(Covariance Intersection, CI)’ 기법을 심층적으로 분석하라.

1. 통계적 독립성 위배와 기존 상태 추정기의 한계

PX4-Autopilot의 확장 칼만 필터(EKF2)는 다중 IMU(Multi-IMU) 또는 중복 지자기 센서 데이터를 처리할 때 각 센서의 관측 노이즈 공분산 R_1, R_2가 서로 교차되지 않는 독립적 상수(R_{12} = 0) 행렬이라고 전제한다.

  • 종속성 발생 원인: 좁은 폼팩터(Form Factor) 내에 근접하게 장착된 두 개의 내부 IMU(예: BMI088, ICM42688)는 모터 및 로터 회전(Rotor Vibration)에 의해 동일한 위상의 강인한 공진 노이즈 스파이크를 시계열적으로 공유하게 된다.
  • EKF의 치명적 오류 (Overconfidence Error): 해당 노이즈 대역이 독립적이라는 가정 하에 융합(Fusion) 연산만 진행하면, 필터는 “서로 다른 독립 변별력을 가진 두 센서가 완전히 동일한 오차 방향을 가리킨다“고 착각하게 된다. 결과적으로 모델에 대한 신뢰도를 실제보다 훨씬 높게 평가하여 상태 공분산 행렬 P를 비정상적으로 작게 축소시킨다. 이는 이후 속도나 위치 이종 센서(예: RTK GPS, Optical Flow)와의 칼만 게인 융합 시 외부 센서 가중치를 지나치게 억누르는 결과로 이어져 기체 궤적 발산을 초래한다.

2. 공분산 교차(Covariance Intersection) 기법의 수학적 접근

공분산 교차(CI) 기법은 상호 상관관계 공분산 트레이니 행렬 P_{12}를 명확히 모델링할 수 없을 때(Unknown Cross-covariance), 두 추정치가 구성하는 다차원 공분산 타원(Covariance Ellipsoids)의 교집합 전체를 포괄(Bounding)하는 가장 보수적인 융합 경계면을 산출하는 필터링 기술이다 (관련 문헌: Uhlmann, “Covariance Intersection”, 1995 참조).

항목수식
CI 융합 상태 공분산 (Bounding Space)P_{CI}^{-1} = \omega P_1^{-1} + (1 - \omega) P_2^{-1}
CI 융합 상태 추정 벡터\hat{x}_{CI} = P_{CI} \left[ \omega P_1^{-1} \hat{x}_1 + (1 - \omega) P_2^{-1} \hat{x}_2 \right]
최적 최적화 가중치 \omega 도출\min_{\omega \in [0,1]} \text{det}(P_{CI}(\omega)) 또는 \text{tr}(P_{CI}(\omega))

위의 비선형 가중치 비율식 통계 융합을 통해 도출된 \hat{x}_{CI}는 센서 네트워크 간의 미지(Unknown)의 주파수 노이즈 종속성에 상관없이 항구적인 필터 일관성(Consistency)을 유지한다. 즉, 부정확성에 대해 타협 가능한 가장 보수적인 안전망을 제공함으로써 EKF 과신 에러를 배제한다.

3. PX4 아키텍처 및 ROS2 환경 내 적용 방법론

현재 메인라인 PX4 Firmware(v1.14 등)의 임베디드 EKF2 코어는 비행 파이프라인 연산량 튜닝 목적으로 엄밀한 수치적 CI 알고리즘을 모든 루프에 내장하기보다는, 멀티 EKF 투표(Multi-EKF Voting) 구조나 인플레이션 스케일링으로 종속성을 분리 우회한다.
그러나 자율 비행(Autonomous Flight) 소프트웨어 및 커스텀 분산 관제 구조 설계 시 다음과 같은 형태로 완벽히 병합 가능하다.

  • uORB 미들웨어 가로채기(Interception): PX4의 sensors 데몬에서 생성된 멀티 센서 원시 데이터(sensor_accel_x)를 독립적으로 구독(Subscribe)하여 CI 융합 알고리즘 C++ 래퍼를 거치게 한 뒤, 통합 계층 토픽인 vehicle_imu로 발행하도록 사용자 정의(Custom) 모듈을 작성한다. Ardupilot의 강성 모놀리식 구조(Monolithic Structure) 대비 모듈 이식성이 뛰어난 PX4만의 철학을 활용하는 방식이다.
  • ROS2/uXRCE-DDS 오프보드 연동 분산 필터: 비행 제어기(Pixhawk) 내부 연산 자원이 부족한 경우, 각 센서의 로우데이터와 각각의 EKF 인스턴스 출력물을 uXRCE-DDS를 통해 컴패니언 컴퓨터(Companion Computer, ROS2 노드)로 퍼블리시한다. 고성능 병렬 프로세싱 환경에서 CI 기반 최적 게인 연산을 집중적으로 수행하고 융합된 절대 상태 예측 변수를 VehicleOdometry 토픽 형태로 비행 제어기에 피드백(Publish)한다.

5. 결론

공분산 교차 기법 기반의 데이터 융합 기술은 대역폭 노이즈가 구조적으로 중첩되는 고진동 멀티로터나 엔진 구동식 기체 환경에서 다수의 동일 폼팩터 센서를 배열할 때 발생하는 통계학적 맹점을 근본적으로 격파하는 진보된 매트릭스 알고리즘이다. PX4의 uORB 메시징 구조와 ROS2 연동 분산 처리 유연성은 이 보수성 높은 무결성 검증 아키텍처를 외부 컴패니언 노드에서 오프보드 제어기로 파견(Dispatch)하기에 최적화된 토폴로지를 제공한다.
