42.3.1.2 잔차 제곱합(Sum of Squared Residuals) 기반 캘리브레이션 행렬 조건수(Condition Number) 평가

관성측정장치(IMU) 및 지자기 센서(Magnetometer) 하드웨어에서 출력되는 로우(Raw) 데이터는 센서 칩셋 자체의 제조 공차, 기판 실장 시의 비틀림, 그리고 센서 고유의 비선형성으로 인해 완벽한 직교성 단위 벡터를 출력하지 못한다. 이를 수학적으로 보상하기 위해 PX4-Autopilot은 기체를 전·후·좌·우·상·하의 6면(6-Sided) 방향으로 지향하게 한 뒤 데이터를 샘플링하여 캘리브레이션(Calibration)을 수행한다. 이 과정의 최종 목적은 센서의 스케일 팩터(Scale Factor) 보정 행렬과 영점 편향(Offset/Bias) 벡터를 도출하는 것이다.

성공적인 캘리브레이션을 보증하기 위해, PX4의 센서 캘리브레이션 서브시스템은 수집된 실측 데이터 샘플의 잔차 제곱합(SSR, Sum of Squared Residuals)을 최소화하는 피팅(Fitting) 알고리즘을 수행한다. 더 나아가, 도출된 추정 기하 행렬의 조건수(Condition Number)를 냉혹하게 평가하여 데이터 세트의 물리적 수집 유효성을 검증한다. 본 절에서는 이러한 수학적 검증 아키텍처를 심층 분석한다.

1. 잔차 제곱합(SSR) 최소화와 추정 모델(Fitting) 로직

이상적으로 캘리브레이션이 완료된 기체의 가속도계 출력 벡터 \mathbf{a} 와 지자기 출력 벡터 \mathbf{m} 는 기체를 3차원 공간상에서 무작위로 회전시킬 때 완벽한 구면(Sphere) 궤적 상의 점들을 구성해야 한다. 그러나 캘리브레이션 이전의 미가공 데이터 타점들은 센서 왜곡으로 인해 타원체(Ellipsoid) 궤적을 그리게 된다. PX4 캘리브레이션 루틴은 이 찌그러진 타원체를 다시 완벽한 구면 쉘(Shell)로 매핑(Mapping)하는 교정 계수를 찾는다.

\mathbf{y}_{calibrated} = \mathbf{S} (\mathbf{y}_{raw} - \mathbf{b})

위 식에서 \mathbf{S}는 스케일 및 3축 간의 직교성 보상(Cross-axis Sensitivity)을 처리하는 3 \times 3 행렬이며, \mathbf{b}는 영점 오프셋 벡터이다.

이때 무작위 방향에서 수집된 N개의 샘플 포인트들에 대하여, 수리적 최적화를 통해 계산된 구면 궤적 표면과 실제 측정 포인트 간의 기하학적 거리 오차 벡터 \mathbf{r}_i 들의 유클리디안 패널티(길이 공간 제곱합)를 바로 통계적 **잔차 제곱합(SSR)**이라 정의한다.

\text{SSR} = \sum_{i=1}^{N} \vert\mathbf{r}_i\vert^2

PX4는 비선형 최소 자승법(Least Squares Method, 주로 Gauss-Newton이나 Levenberg-Marquardt 변형)을 통해 이 SSR 비용 함수(Cost Function)를 최소로 만드는 최적의 \mathbf{S}와 \mathbf{b}를 찾아낸다. 캘리브레이션 연산이 완료되더라도, 최종 수렴된 SSR 절대값이 사전에 정의된 임계값(Threshold)을 초과한다면 센서에 심각한 고주파 진동이 가해졌거나 외부 물리적 간섭(지자기의 경우 쇠붙이 등)이 컸던 것으로 간주된다. 이 경우 Commander 모듈은 즉시 QGroundControl로 “Calibration Failed: High variance” 상태 메시지를 송신하며 파라미터 저장을 단호히 거부한다.

2. 캘리브레이션 디자인 행렬의 조건수(Condition Number, \kappa) 평가 지표

SSR이 아무리 0에 가깝게 최소화되었더라도, 이는 “수집된 샘플들이 모델에 잘 맞는다“는 뜻일 뿐, “공간 전체를 충분히 대변하는 교정 인가“를 의미하지는 않는다. 예컨대 사용자가 기체의 Pitch 축으로만 열심히 돌리고 Roll 축이나 Yaw 축 방면의 극단 데이터 포인트를 충분히 수집하지 않았다면, 교정 방정식은 수학적으로 불량 조건(Ill-conditioned) 상태에 빠진다.

이를 수치해석적으로 방어하고 관측 가능성(Observability)을 판단하기 위해 도입된 핵심 지표가 조건수(Condition Number)이다.
수집된 데이터 공분산 행렬 또는 캘리브레이션 디자인 행렬(Design Matrix) \mathbf{X} 에 대하여, 행렬의 최대 특잇값(Singular Value) \sigma_{max} 과 최소 특잇값 \sigma_{min} 의 비율을 역행렬 연산의 불안정 지표인 조건수 \kappa(\mathbf{X})라 정의한다.

\kappa(\mathbf{X}) = \frac{\sigma_{max}}{\sigma_{min}}

  • 조건수가 1에 가까운 경우 (안정적): 사용자가 기체를 3차원 공간 상의 모든 축 벡터로 균일하게 회전시키며 골고루 데이터를 샘플링했음을 증명한다. 도출된 스케일 팩터 파라미터 집합은 수치적으로 매우 안정적이며 신뢰할 수 있다.
  • 조건수가 과도하게 큰 경우 (불안정적): 특정 축 방향의 데이터 샘플링이 결여되어 추정 행렬이 특이 근접 상태(Near-Singularity)로 무너졌음을 의미한다.

PX4의 내부 C++ 소스 코드(src/modules/commander/calibration_routines.cpp) 모듈에서는, 적합 행렬 타원체 알고리즘(Ellipsoid Fit) 진입 전후로 특잇값 분해(SVD)나 고윳값(Eigenvalue) 스프레드 검사를 통해 이 조건수를 산출한다. 조건수가 임계값을 초과하면, PX4는 “데이터 포인트 스팬(Data Point Span)이 불충분하다“고 판다내려 해당 면의 교정 데이터를 무효화한다.

3. PX4 관제 상태 머신 및 QGroundControl 연동 아키텍처

사용자가 지상 관제소 즉, QGroundControl(QGC) 버튼을 조작하여 MAVLink의 MAV_CMD_PREFLIGHT_CALIBRATION 명령을 Autopilot 측으로 송출하게 되면, PX4 내부에서는 다음과 같은 검증 상태 머신 파이프라인(State Machine Pipeline) 스레드가 기동된다.

sequenceDiagram
    participant QGC as QGroundControl (GCS)
    participant CMD as Commander (PX4)
    participant CAL as Calibration Routine Worker
    participant SD as Sensor Driver
    
    QGC->>CMD: MAVLink: MAV_CMD_PREFLIGHT_CALIBRATION 송신
    CMD->>CAL: Thread: Start Calibration Worker
    loop 6-Sided Static Detection
        SD-->>CAL: Fetch Raw Sensor Data 
        CAL->>CAL: Extract features & Buffer Valid Samples (Static 판별)
        CAL-->>QGC: MAV_CMD_ACK (Progress % 피드백)
    end
    CAL->>CAL: Least Squares Ellipsoid Fitting 수행
    CAL->>CAL: Calculate Target SSR (Noise/Variance Threshold)
    CAL->>CAL: Compute Design Matrix $\kappa$ (Condition Number 스팬 검증)
    alt $\kappa$ < Threshold & SSR < Threshold 만족
        CAL->>CMD: Calibration SUCCESS 반환
        CMD->>SD: Commit CAL_ACCx_ID, CAL_ACCx_XSCALE/OFFSET 파라미터 저장
        CMD-->>QGC: STATUSTEXT "Calibration successful" 전달
    else $\kappa$ 혹은 SSR 검증 Failed 
        CAL->>CMD: Calibration FAILED 반환
        CMD-->>QGC: STATUSTEXT "Failed: Poor orientation span or high noise" 전달 및 드롭
    end

3.1 Ardupilot 관제 철학과의 설계 비교

  • PX4-Autopilot: 다소 보수적 철학을 따른다. 6면 캘리브레이션(6-Point Static Calibration)을 엄격하게 통제한다. QGC 프론트엔드 UI 역시 상태-기반 피드백 뷰 컨트롤(View Control)을 통해 각 면을 지정하는 위치에 정확하게 거치하고 물리적으로 “가만히 고정(Static)” 하도록 강제한다. 즉, 철저한 정적(Static) 데이터를 수급한 환경 아래서만 유효 샘플로 버퍼에 쌓아, 조건수를 가장 이상적으로 확보하려는 수리학적 결벽성을 가진다.
  • Ardupilot: Mission Planner를 사용한 가속도계 교정 역시 위상학적으로 동일한 수식을 거치나, 조금 더 환경에 유연하다. 특히 지자기(Mag) 센서의 경우, ‘Onboard Mag Calibration’ 방식(일명 “기체 춤추기(Dance) 모드”)을 허용한다. 이는 QGC처럼 면마다 정지 상태를 강제하지 않고, 허공에서 사용자가 지속적으로 무작위로 기체를 회전시키며 난해한 궤적에 대해서 데이터 포인트를 폭넓게 끌어모으는 방식을 채택한다. 데이터에 진동 노이즈가 다소 끼어 SSR이 상대적으로 높더라도, 피팅 이터레이션 반복(Iteration)을 통해 일정 적합도(Fitness Ratio)에 다다르면 패스(Pass)시켜주는 상대적 관용성(Tolerance Architecture)을 보여준다.

4. 커스텀 GCS 제작 팁 및 실무 제언

독자적인 GCS 애플리케이션 프론트엔드를 설계 및 구현할 때는, QGroundControl이 채택하고 있는 정교한 캘리브레이션 모니터링 로직을 그대로 이식하는 것에 그치지 마라. 개발자는 MAV_CMD_ACK에 내재된 진행률 페이로드뿐만 아니라, STATUSTEXT로 파싱되는 실패 원인을 분류하여 조종자 경고 UI에 표출하도록 해야 한다. 실패의 본질이 “기체를 모든 축으로 충분히 회전시키지 않은 조건수(Condition Number) 부족” 때문인지, 아니면 캘리브레이션 테이블 위의 미세 진동으로 인한 “잔차 제곱합(SSR Variance) 규격 초과” 때문인지를 명시적으로 사용자에게 가이드해 주어라.

더불어, 최후에 플래시 메모리에 박힌 CAL_ACC0_XSCALE, CAL_ACC0_YSCALE 파라미터의 산출 계수가 1.0 (디폴트) 기준으로 5% 범위를 현격히 벗어난다면, 그것은 당신이 납땜하거나 구매한 센서 보드 자체가 회로적 제조 결함에 빠져있을 확률이 압도적이므로 즉시 하드웨어를 교체할 것을 강력히 권고한다.
