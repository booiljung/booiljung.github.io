42.4.1 EKF2 상태 추정기의 이노베이션 분산(Innovation Variance) 분석

대상 펌웨어 및 관제 소프트웨어 버전

  • PX4-Autopilot: 안정화 버전(Stable Release) v1.14.x 이상 기준
  • QGroundControl (QGC): v4.3.x 이상 기준

무인항공기(UAV)의 강건한 비행 제어를 위해서는 정확한 상태 추정(State Estimation)이 필수적이다. PX4-Autopilot은 상태 추정을 위해 주로 확장 칼만 필터(Extended Kalman Filter, EKF) 알고리즘을 최적화한 모듈인 EKF2를 기본 추정기(Estimator)로 채택하고 있다. 반면 ArduPilot은 버전에 따라 EKF2 혹은 EKF3를 사용하여 자체적인 파라미터 튜닝 시스템과 다중 센서 융합 아키텍처를 구성한다. 두 비행 제어기 모두 수학적 토대는 동일하나, PX4-Autopilot은 비행 전 점검(Pre-flight Check) 단계 및 비행 과정에서 이노베이션(Innovation, 측정 잔차)과 이노베이션 분산(Innovation Variance)을 모니터링하여 센서의 고장이나 물리적 이상을 감지하는 구조를 한층 엄격하게 적용한다. 본 절에서는 이러한 EKF2 상태 추정기 내부에서 연산되는 이노베이션 분산의 본질적 의미와 분석 기법을 심도 있게 다룬다.

1. 이노베이션(Innovation) 및 이노베이션 분산(Innovation Variance)의 수학적 개념

기본적인 칼만 필터(Kalman Filter) 이론에서 이노베이션은 관측 기기(센서)로부터 들어온 실제 측정값과 시스템의 동역학 모델(Kinematic Model)을 통해 자체적으로 예측한 측정값 사이의 차이를 의미한다. 측정 업데이트(Measurement Update) 단계에서 EKF는 이 이노베이션을 사용하여 현재 상태 예측치를 교정(Correction)한다.

  • 예측된 관측치(Predicted Measurement): \hat{z}_{k|k-1}
  • 실제 센서 관측치(Actual Measurement): z_k
  • 이노베이션(Innovation): \tilde{y}_k = z_k - \hat{z}_{k|k-1}

여기서 이노베이션 분산(Innovation Variance) 또는 공분산(Innovation Covariance) 행렬 S_k는 다음과 같이 정의된다.

S_k = H_k P_{k|k-1} H_k^T + R_k

테이블을 통해 수식 내 변수의 역할을 상세히 나열한다.

변수의미역할
H_k관측 행렬(Observation Matrix)상태 변수와 측정치 간의 관계 도출 (비선형 모델의 경우 야코비안 행렬 역산 반영)
P_{k\vert k-1}사전 예측된 상태 공분산 행렬(A Priori State Covariance Matrix)예측된 상태의 불확실성 수준
R_k관측 잡음 공분산 행렬(Measurement Noise Covariance Matrix)센서 자체 노이즈의 분산 크기를 결정

PX4-Autopilot의 EKF2 모듈은 각종 센서(GPS, IMU, 자력계(Magnetometer), 기압계(Barometer) 등)의 데이터를 융합할 때, 각 센서 채널별로 생성되는 이노베이션 \tilde{y}_k가 이론적인 공분산 S_k에 비례하여 적정한 허용 범위 내에 있는지 평가하는 이노베이션 테스트(Innovation Test)를 수행한다. 이노베이션이 이노베이션 분산이 허용하는 신뢰 구간(Confidence Interval)을 초과할 경우, EKF2는 해당 센서 데이터에 심각한 오류가 포함된 것으로 간주하고 측정 업데이트 단계에서 이를 기각(Reject)한다.

2. PX4-Autopilot과 ArduPilot의 이노베이션 기반 관측치 거절(Rejection) 비교

PX4-Autopilot과 ArduPilot 모두 칼만 필터를 사용하기 때문에, 이노베이션 분산을 기반으로 이상치를 거절(Outlier Rejection)하는 메커니즘을 내장하고 있다.

  • PX4-Autopilot (EKF2): 게이트(Gate) 크기를 결정하는 파라미터 세트를 제공한다(예: EKF2_GPS_V_GATE, EKF2_MAG_GATE 등). 특정 센서의 이노베이션 수치가 파라미터로 설정된 표준 편차(Standard Deviation)의 배수 범위를 지속적으로 초과하면, 관측치 거절 상태로 전환되고 상태 플래그(estimator_status 메시지의 깃발 데이터)를 통해 QGroundControl에 즉시 경고를 발생시킨다. 특히 비행 전 점검(Pre-flight Check) 단계에서는 이러한 분산 통계가 아직 안정되지 않았거나 이노베이션이 큰 경우 시동(Arming)이 원천적으로 차단된다.
  • ArduPilot (EKF3): 마찬가지로 이노베이션 일관성 검사(Innovation Consistency Check)를 사용하지만, 다중 EKF(Multiple EKF Lanes) 구조를 적극 활용하여 하나 이상의 EKF 레인이 특정 센서의 허용 오차를 넘으면 즉각적으로 코어 스위칭(Core Switching)을 수행하는 데 초점이 맞추어져 있다. PX4는 단일 EKF 코어 내의 분산 허용도 평가를 더 민감하게 설정하여 개별 센서 융합 관리에 비중을 둔 설계를 띠는 경향이 있다.

3. QGroundControl(QGC)을 통한 모니터링 체계

QGC에서는 MAVLink 메시지인 ESTIMATOR_STATUS 내부의 요소들을 해석하여 화면상의 게이지와 경고 알림으로 표시한다. 이 중 혁신 분산과 직접적으로 연관된 필드들은 다음과 같다:

  • pos_horiz_accuracy, pos_vert_accuracy: 위치 관련 공분산 결과
  • 센서의 융합 상태를 알리는 이노베이션 테스트 통과 여부(Test Ratio)

Test Ratio는 다음과 같이 정의된다.

\text{Test Ratio} = \frac{\tilde{y}_k^2}{(\text{Gate Size})^2 \cdot S_k}

이 값이 1.0을 초과하면 해당 관측치는 거절됨을 의미한다. 사용자(조종자 또는 개발자)는 QGC 시스템 모니터(System Monitor) 창을 통해 특정 센서의 Test Ratio 값이 급증하는 현상을 모니터링하여, 자력계 간섭(Magnetic Interference)이나 GPS 다중 경로 오류(Multipath Error) 여부를 판별한다.

3.1 이노베이션 분산 평가 알고리즘 로직 흐름

다음은 PX4 EKF2 모듈 내부에서 수행되는 이노베이션 분산 평가와 상태 업데이트 여부를 판별하는 단위 과정을 나타낸다.

    A[센서 관측치 수신: $z_k$] --> B[관측 모델 예측: $\hat{z}_{k|k-1}$]
    B --> C[이노베이션 계산: $\tilde{y}_k = z_k - \hat{z}_{k|k-1}$]
    B --> D[이노베이션 분산 연산: $S_k = H_k P_{k|k-1} H_k^T + R_k$]
    C --> E[Test Ratio 산출: $\tilde{y}_k^2 / (\text{Gate}^2 \cdot S_k)$]
    D --> E
    E --> F{Test Ratio $\le$ 1.0?}
    F -- Yes (통과) --> G[칼만 이득 산출 및 상태(State) 업데이트 수행]

4. 소스 코드(Source Code) 레벨에서의 이해

PX4-Autopilot의 EKF2 소스 코드(src/modules/ekf2/EKF)를 살펴보면 관측 융합(Fusion)을 관장하는 개별 메서드들이 존재한다.
예를 들어, GPS 측정치를 융합하는 함수 내에서는 항상 이노베이션과 공분산을 산출한다.

  • R_k 역할: EKF2_GPS_P_NOISE, EKF2_GPS_V_NOISE 등의 파라미터가 관측 잡음(Measurement Noise)으로 입력되어 동적으로 반영된다.
  • 혁신 분산 검증 실패 시: 코드 내의 fusion_failing 부울(boolean) 변수가 true로 설정되며, 일정 횟수 이상 누적되면 제어권이 다른 센서 또는 강제 착륙(Land) 등 보호 모드로 넘어가도록 유도한다. 이 메커니즘을 통해 잘못된 관측치가 상태 추정기를 오염시키는 것을 사전에 차단한다.

5. 결론 및 실무적 고려사항

공학적 관점에서 비행 제어기와 센서의 신뢰성을 판단할 때 최우선으로 점검해야 하는 지표 중 하나가 바로 EKF2의 이노베이션 분산이다.

  1. 캘리브레이션 불량: 특히 자력계의 경우 하드웨어 철 구조물 주위에서 캘리브레이션(Calibration)을 잘못 수행하면 이노베이션 값이 튀는 현상(Innovation Spike)이 급증한다. 이를 방지하기 위해 비행 전 올바른 환경에서의 정교한 캘리브레이션이 매우 중요하다.
  2. 진동(Vibration)의 영향: 기체의 진동이 커질 경우 IMU 입력의 노이즈가 증대되어 P_{k|k-1} 성분이 발산할 위험이 있다. 방진재(Vibration Damping Pad)를 삽입하여 근본적인 진동 저감 설계를 강구하라.
  3. 관련 연구 동향을 살피면, 논문 “A New Extended Kalman Filter for UAV Attitude Estimation” 등에서 언급하듯 정밀한 R_k(관측 잡음 모델링) 기반의 지능형 분산 조정 기법 구조들이 꾸준히 제시되고 있다. PX4의 소스코드 개선 과정 또한 이러한 동적 이노베이션 게이팅(Dynamic Innovation Gating) 기준을 고도화하는 방향으로 발전 중이다.

비행 안정성 오류에 봉착했다면, 비행 로그(ULog format) 파일에서 estimator_innovations 메시지의 항목들을 관제 도구를 통해 분석하여, 어떠한 시스템 센서 데이터가 이노베이션 신뢰 한계치를 이탈했는지 철저히 규명하고 튜닝하라.
