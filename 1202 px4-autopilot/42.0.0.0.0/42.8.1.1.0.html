42.8.1.1 매개변수 마스킹(Parameter 마스킹)에 의한 안전 제약 조건의 국소적 비활성화 위험도 모델링

1. 개요 및 위험도 모델링의 필요성

PX4-Autopilot의 서킷 브레이커 파라미터(CBRK_*) 조작은 본질적으로 특정 시스템 고장 플래그 처리 과정을 마스킹(Masking)하는 기법이다. 이러한 안전 제약 조건의 국소적 비활성화(Local Deactivation)는 하드웨어 인테그레이션(Integration)과 기능 개발 시의 테스트 편의성을 극대화한다. 그러나 전체 무인기 시스템(UAS) 아키텍처 관점에서는 연쇄적인 결함(Cascading Faults)을 유발할 수 있는 치명적 위험(Critical Risk)을 동반한다.

따라서 단순히 파라미터를 변경하는 것에 그치지 않고, 국소적 마스킹이 비행 제어기(FC)의 커맨더(Commander) 상태 머신(State Machine) 및 추정기(Estimator) 전반에 어떠한 파급 효력을 미치는지 논리적으로 분석하는 위험도 모델링(Risk Modeling) 기법이 요구된다.

2. 매개변수 마스킹 메커니즘의 아키텍처적 진단

매개변수 마스킹이란 센서 처리 계층(Sensor Processing Layer)이나 자체 오류 감지(Error Detection) 알고리즘이 생산하는 경보 플래그(Warning Flags) 자체를 시스템에서 소거하는 것이 아니다. 센서의 드라이버나 uORB 토픽 발행부(Publisher)는 여전히 에러 데이터를 카운팅(Counting)하지만, 시동(Arming) 가능 여부나 비상 모드(Failsafe) 진입을 총괄하는 커맨더 모듈 내에서 해당 플래그를 읽고도 상위 로직으로의 전이(State Transition)를 고의로 무시하게 만드는 것을 의미한다.

  • 데이터 계층 보존: 센서는 정상적으로 오작동 피드백을 생산한다.
  • 제어 계층 맹목화(Blindness): 커맨더는 오류 상태임을 전달받았음에도 CBRK_* 파라미터의 매직 넘버(Magic Number) 조건을 충족하면, 해당 위험 요소를 정상적인 궤도 내 통제 가능한 변수로 오인(False Validation)하고 다음 단계 제어를 속행한다.

3. 국소적 비활성화 위험도 모델 (Risk Model)

매개변수 마스킹의 위험성을 평가하기 위해 결함 트리 분석(Fault Tree Analysis, FTA) 기법을 기반으로, 마스킹된 파라미터가 초래할 수 있는 연쇄 반응을 모델링한다.

3.1 감지 상실 (Loss of Detection) 효과

예를 들어, CBRK_GPSFAIL 파라미터를 활성화하여 GPS 신호 불량에 대한 비행 전 점검을 우회했다고 가정하라. 실제 GPS에 치명적인 글리치(Glitch)가 발생하더라도, 위치 제어 모드(Position Control Mode)는 강제로 해제되지 않는다. 이는 EKF2 추정기로 하여금 신뢰할 수 없는 위성 좌표 데이터를 참조하게 만들며 결과적으로 추정값 발산(Divergence) 현상 및 기체의 통제 불능 가속(Fly-away)을 유발하게 된다.

3.2 제어 무결성 (Control Integrity) 훼손

마스킹이 적용된 하위 모듈(Sub-module)의 오류 데이터를 제어기(Controller)가 그대로 수용하게 됨으로써, 제어 지령(Control Command) 연산의 무결성이 붕괴된다. 액추에이터(Actuator)로 무효한 제어값이 출력되며 모터의 비정상 회전 등 직관적인 하드웨어 파손으로 연결된다.

4. QGroundControl 및 MAVLink 기반의 알람 보존 규칙 (Alarm Retention Rule)

위험도 모델링을 설계 및 분석하는 과정에서 도입해야 할 핵심 보안 철학은 알람의 가시성(Visibility) 보존이다. PX4는 특정 안전 제약 조건이 마스킹되어도, GCS(QGroundControl)로 향하는 MAVLink STATUSTEXT 메시지에 포함된 에러 로그의 전파 경로까지 마스킹하지는 않아야 한다.

  • 내부적으로 제어 로직은 마스킹을 통해 우회되더라도, 관제 화면 상에는 “Pre-flight Check failed but overridden by CBRK_” 와 같은 시스템 상태 메시지가 가시적으로 전시되어야 한다.
  • 관제사(Operator)는 기체가 불완전한 상태로 우회 기동 중임을 지속적으로 인지함으로써 추가적인 사고를 사전에 통제할 수 있다.

5. 결론 및 실무 제언

개발자나 연구자는 CBRK_* 파라미터를 통한 국소적 비활성화가 단순하게 ‘성가신 경고를 끄는 옵션’이 아니라는 점을 분명히 해야 한다. 위험도 모델 관점에서 보았을 때 제약 조건 마스킹 행위는 기체의 오차 감내(Fault Tolerance) 아키텍처와 잔존 안전 기능을 단숨에 무너뜨리는 행위이다. 이를 활용해야 하는 개발 단계에서는 물리적인 킬 스위치(Kill Switch) 릴레이 탑재나, 테더링(Tethering)된 실험 환경 구축과 같은 철저한 백업 통제망 속에서 설계 검증을 속행하라.
