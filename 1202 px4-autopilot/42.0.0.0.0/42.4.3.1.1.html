42.4.3.1.1. 카이제곱(Chi-square) 분포 검정을 통한 센서 측정 공간의 혁신 시퀀스(Innovation Sequence) 백색성(Whiteness) 증명

PX4-Autopilot(펌웨어 v1.14.0 기준)의 상태 추정기인 EKF2(Extended Kalman Filter) 운영 환경에서 센서 측정의 결함을 조기에 판단하는 것은 지극히 중요하다. 본 절에서는 카이제곱(Chi-square, \chi^2) 분포 검정을 통해 혁신 시퀀스(Innovation Sequence)의 백색성(Whiteness)을 통계적으로 증명하고, 이를 통해 센서 데이터의 일관성을 평가하는 알고리즘 중심의 아키텍처를 고찰하라.

1. 혁신 시퀀스(Innovation Sequence)와 백색성(Whiteness)의 수학적 기초

칼만 필터 아키텍처에서 혁신(Innovation, 잔차)은 실제 센서로부터의 측정 벡터와 필터 모델에 의해 예측된 측정 벡터 간의 차이를 식별하는 핵심 지표이다. 최적의 튜닝 상태를 갖는 EKF 환경에서 이 혁신 시퀀스는 시간에 대한 상호 상관성(Cross-correlation)이 상쇄된, 평균이 0인 백색 잡음(White Noise) 특성을 유지해야 한다. 이를 ’백색성(Whiteness)’이라 정의한다.

혁신 시퀀스 벡터 \tilde{y}_k는 다음의 수식으로 모델링하라.

\tilde{y}_k = z_k - h(\hat{x}_{k \vert k-1})

여기서:

  • z_k: k 시점에서의 실제 센서 측정 벡터 (예: GNSS 위치, IMU, 기압계 등)
  • h(\cdot): 상태 공간에서 측정 공간으로 매핑하는 비선형 관측 모델 함수
  • \hat{x}_{k \vert k-1}: 사전 상태 추정치(A priori state estimate)

이때 혁신 벡터의 잔차 공분산(Residual Covariance) 행렬 S_k는 다음과 같이 정의된다.

S_k = H_k P_{k \vert k-1} H_k^T + R_k

센서 노이즈가 가우시안 확률 분포(Gaussian distribution)를 따른다고 가정할 때, 측정 공간의 차원(Dimension) m에 대해 **정규화된 잔차 제곱합(Normalized Innovation Squared, NIS)**은 다음과 같이 산출하라.

\epsilon_k = \tilde{y}_k^T S_k^{-1} \tilde{y}_k

연산된 NIS 값 \epsilon_k는 자유도 m인 **카이제곱 분포(\chi^2_m)**를 따른다. 이 통계적 특성을 근거로 새로운 측정치의 유효성을 판별하는 체계가 바로 ‘카이제곱 검정’ 기반 센서 이상 감지(Sensor Fault Detection) 기법이다.

2. PX4-Autopilot EKF2 아키텍처 내 측정치 퓨전(Fusion) 및 거부(Rejection) 로직 분석

PX4의 EKF2 엔진(src/modules/ekf2)과 그 기반이 되는 ECL(Estimation and Control Library)에서는 모든 외부 관측 데이터를 필터 상태에 융합(Integrate)하기 직전에 이노베이션의 일관성을 검증한다.
카이제곱 검정에 실패하는 측정값은 상태 업데이트를 오염시킬 수 있는 ‘이상점(Outlier)’ 혹은 ’결함(Fault)’으로 취급되어 최종 융합 단계에서 기각(Rejected)된다.

2.2 카이제곱 임계값(Threshold) 처리 및 파라미터 적용 기법

이론적인 완전한 행렬 연산 \epsilon_k = \tilde{y}_k^T S_k^{-1} \tilde{y}_k은 컴퓨팅 로드 억제를 위해 PX4 소스 코드 상에서 종종 차원에 따라 1차원적 상한선(Gate) 단위로 단순화하여 평가된다.

QGroundControl(QGC v4.3.0 기준)에서 관리되는 게이트 튜닝 파라미터를 통해 이 \chi^2 검정의 엄격도를 조절하라.

  • EKF2_GPS_GATE: GPS 위치(Position) 및 속도(Velocity) 데이터에 대한 이노베이션 일관성 검사에 작용하는 게이트(Gate) 비율. 표준 편차 단위이며 일반적인 디폴트는 5.0이다.
  • EKF2_BARO_GATE: 기압계 기반 고도 센서 데이터의 유효성을 통제한다.
  • EKF2_MAG_GATE: 자기 나침반 관측치의 퓨전을 통제하며, 주변 자기장 간섭 발생 시 이 파라 파라미터를 통해 순간적인 기각 빈도를 조절할 수 있다.

특정 센서 퓨전이 타임아웃(Timeout) 및 거부 처리를 반복할 경우, 맹목적으로 게이트 크기를 넓히는 튜닝은 비행 발산(Divergence)의 위험을 높인다. 원인 불명의 \chi^2 게이트 초과 시에는 기체의 진동, 배선 간섭, 또는 해당 센서의 물리적 파손 여부를 선형 분석하라.

3. Ardupilot (EKF3) 대비 PX4의 다중 관측치 평가 철학 비교

EKF 알고리즘을 최적화하는 데 있어 PX4와 Ardupilot(Mission Planner 활용)은 서로 다른 내결함성 처리 구조를 시현(Demonstrate)한다.

아키텍처 항목PX4-Autopilot (EKF2)Ardupilot (EKF3)
카이제곱 검정 결속 계층ECL 내의 독립적인 퓨전 객체 단위로 검증되며, 모듈러(Modular) 구조로 분리되어 철저한 단일 추정기 파이프라인의 백색성을 방어한다.핵심 AP_NavEKF3 클래스 깊은 곳에서 다차원 레인(Multiplex Lane) 체계를 보조하는 트리거 인자로 활용된다.
연속 기각(Rejection) 발생 시 대응\chi^2 검정 실패 시 측정치를 즉시 제거(Ignore)하고, EKF 상태가 무효화되면 Pre-flight 단계 진입 전 Arming 거부로 이어진다.특정 센서 데이터 기각 폭주로 해당 Lane의 상태가 불량 판정될 경우, 백그라운드의 양성(Healthy) Lane으로의 스위칭(Lane Switch) 메커니즘을 전개한다.
MAVLink 기반 GCS 상태 관제이노베이션 혁신 추정치의 분산 상한 초과 에러(Estimator Fault)를 정교하게 ESTIMATOR_STATUS MAVLink 메시지로 보고하여 QGC에 실시간 상태를 띄운다.Mission Planner의 EKF Status 모니터를 통해 개별 센서 잔차 분산치(Variance)를 막대 그래프(Bar graph) 형태로 종합 관제한다.

PX4의 EKF2는 파라미터를 통해 각 센서 게이트의 통계적 \chi^2 바운더리를 독립적으로 엄밀하게 규정하고 측정치의 불균형 유입을 차단하는 데 중점을 두는 반면, Ardupilot의 EKF3는 잉여(Redundant) Lane을 구비하여 궤도를 분리하는 시스템 공학적 유연성을 적용하는 차이가 있음을 분석하라.

4. 결론

무인 시스템 모델에서 카이제곱(Chi-square) 알고리즘을 응용한 측정 벡터 공간의 백색성(Whiteness) 증명은 외부 관측치의 결함을 통계적 정밀도로 걸러내는 핵심 방어 메커니즘이다. 엔지니어는 혁신 시퀀스의 물리적 의미와 잔차 공분산의 스케일 지표(NIS)를 심도 있게 이해해야 한다. 이를 기반으로 PX4 소스 리포지토리(ecl/EKF)의 측정치 갱신(Measurement update) 로직과 비행 데이터(ULog) 상의 이노베이션 분산 추이를 교차 검증하여 안정적인 자율 에이전트 환경 설계를 이룩하라.
