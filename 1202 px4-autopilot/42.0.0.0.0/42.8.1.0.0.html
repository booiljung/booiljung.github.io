42.8.1 서킷 브레이커 파라미터(CBRK_*) 조작을 통한 바이패스(Bypass) 기법

1. 개요 및 설계 철학

PX4-Autopilot(v1.14 이상 기준)은 비행 전 점검(Pre-flight Check) 과정에서 기체의 센서, 전원, 통신 등의 상태를 엄격하게 검증하여 비정상적인 이륙을 단절하는 방어적 아키텍처를 가지고 있다. 그러나 특수한 실내 자율 주행 연구, 센서 퓨전(Sensor Fusion) 오차 모델링, 하드웨어 테스트 등에서는 특정 점검 항목을 의도적으로 우회(Bypass)해야 할 필요가 생긴다. 이를 위해 설계된 것이 서킷 브레이커(Circuit Breaker, CBRK_*) 파라미터 시스템이다.

서킷 브레이커를 활성화하면 특정 안전 점검 요건을 무시하고 강제로 시동(Arming)을 걸거나 다음 제어 상태(Control State)로 전이를 강제할 수 있다. 이는 고장 주입 검증(Fault Injection Testing)이나 극한 환경 테스트 비행을 수행할 때 필수적으로 활용되는 연구 및 개발형 기법이다.

2. 매직 넘버(Magic Number) 기반 검증 구조

서킷 브레이커 시스템의 가장 큰 특징은 단순한 활성화/비활성화 플래그(예: 1 또는 true)를 사용하지 않는다는 점이다. 대신, 의도하지 않은 파라미터 변경이나 사용자 인터페이스 상의 실수로 인한 우회를 방지하기 위해 매직 넘버(Magic Number) 원칙을 채택하였다. 각 서킷 브레이커 파라미터에는 고유한 특정 정수값이 사전 할당되어 있으며, 해당 값을 오차 없이 입력하였을 때만 점검 기능이 바이패스된다.

2.1 주요 CBRK 매직 넘버 (PX4-Autopilot 기준)

  • CBRK_SUPPLY_CHK (894281): 전원 공급 및 배터리 전압, 직렬 구성(Cell count) 이상 여부 확인 루틴을 무시한다.
  • CBRK_USB_CHK (197848): USB 케이블이 비행 제어기(Flight Controller)에 연결되어 있을 때 시동을 거부하는 로직을 무시한다. 벤치 테스트(Bench Test)에서 USB가 연결된 상태로 모터의 동작을 확인할 때 주로 사용한다.
  • CBRK_IO_SAFETY (22027): 하드웨어 안전 스위치(Safety Switch)의 인터락(Interlock) 입력을 우회하여, 버튼을 누르지 않아도 즉각적인 반응이 가능케 한다.
  • CBRK_BBU_CHK (280718): 배터리 백업 전원 측정 상태를 검사하지 않는다.
  • CBRK_GPSFAIL (240024): GPS 신호 단절, 위치 추정 불가(Position Invalid) 상태에서 일어나는 강제 안전 모드(Failsafe) 진입을 무시한다. 이는 VIO(Visual Inertial Odometry)나 옵티컬 플로우(Optical Flow)만을 의존한 실내 자율 비행 프로젝트 등에서 주로 무시된다.
  • CBRK_FLIGHTTERM (121212): 치명적인 비행 종료(Flight Termination) 조작을 명령 체계단에서 우회한다. 기체의 추락이나 극단적인 상황에서도 모터 정지가 일어나는 제약 방지를 위해 실험 목적으로만 사용한다.

3. Ardupilot의 ARMING_CHECK와의 아키텍처적 비교

Ardupilot 운영체제 체계는 ARMING_CHECK라는 16비트 단일 마스크(Bitmask) 파라미터를 사용하여 안전 조건 점검을 통합 제어한다. 즉, 특정 비트를 끄거나 ARMING_CHECK = 0으로 설정하면 각각의 센서 검사 생략이 가능하다.

반면, PX4의 CBRK_* 아키텍처 방식은 독립된 파라미터 체계(Independent Parameter Architecture) 철학을 반영했다.

  1. 명시적 가독성 향상: QGroundControl 파라미터 뷰에서 특정 우회 설정에 대한 변수명 자체가 가독성을 지니므로 직관적으로 현황을 파악할 수 있다.
  2. 휴먼 에러(Human Error) 최소화: 복잡한 16진수 비트 연산을 통해 안전망을 조정할 필요 없이, 지정된 매직 넘버 입력을 통해 접근 제어를 한 단계 강화하여 현장에서 우발적으로 해제되는 오류를 방지한다.
  3. 분산형 컴포넌트 검증: 통신, 관제 모듈이 여러 기능을 수행할 때, 특정 Sub-component 에러는 해당 CBRK_* 값의 uORB 토픽만을 조회하여 검증 판단을 독립적으로 내리게 해 확장성(Scalability)이 우수하다.

4. QGroundControl(QGC)을 활용한 설정 및 운용 예시

실제 서킷 브레이커 파라미터를 통해 우회 제어가 필요하다면, QGroundControl 기반 파라미터 검색기능(Parameters > CBRK 부문)을 활용하라.

  • 예를 들어, USB를 통한 실시간 디버깅 셸(NuttShell) 접속 상태에서 모터와 ESC의 미세 튜닝을 해야 한다면, CBRK_USB_CHK 파라미터 검색 후 해당 텍스트 상자에 197848 값을 입력하고 비행 제어기(FC)를 재부팅하여 변경을 확정(Commit)하라.

주의 사항(Caution)
서킷 브레이커 조작은 시스템이 설계상 보유한 생존 본능(Survival Instinct)을 인위적으로 거세하는 강력한 양날의 검이다. 활성화는 반드시 실내 철망, 혹은 구속 장치가 설치된 테스트베드의 엄격한 환경 하에서만 수동 점검과 함께 수행해라. 실제 비행 궤도 투입 전, 해당 파라미터들의 값이 ’0’인지 필터링 및 복구 절차를 거치는 것을 비행 작전 절차(SOP) 내에 표준화 시켜야 한다. 매직 넘버 방식의 도입 이유가 바로 “의도치 않은 사고적 안전 해제 방지“에 있다는 점을 상기해야 한다.
