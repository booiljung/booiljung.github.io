42.3.5.1.1. 저역 통과 무한 임펄스 응답(IIR) 필터 위상 지연(Phase Delay)에 따른 영점 이탈(Zero-drift) 동적 감지

무인항공기(UAS)의 대기속도 센서(Airspeed Sensor)는 난기류 및 기계적 진동으로 인해 필연적으로 고주파 환경 노이즈를 수반한다. PX4-Autopilot은 이러한 노이즈를 효과적으로 억제하기 위해 센서 파이프라인에서 저역 통과 무한 임펄스 응답(Low-pass Infinite Impulse Response, IIR) 필터를 거친다. 그러나 IIR 필터 적용 시 컷오프 주파수(Cut-off Frequency) 설정에 따라 위상 지연(Phase Delay) 현상이 발생하며, 이 과정에서 센서의 실제 출력이 0임에도 동압 추정치가 잔류하는 영점 이탈(Zero-drift) 문제가 동적으로 유발될 수 있다. 본 절에서는 이러한 영점 이탈 현상을 PX4 아키텍처 내에서 어떻게 감지하고 보정하는지 심층적으로 분석하라.

1. IIR 필터 적용과 위상 지연 메커니즘

PX4 소스코드 트리 내 src/modules/sensors/vehicle_air_data/VehicleAirData.cpp 등의 센서 드라이버 파이프라인은 베르누이의 정리(Bernoulli’s Principle)를 기반으로 측정된 동압(Dynamic Pressure) 원본 데이터(Raw Data)에 1차 또는 2차 IIR 필터를 적용한다.

  • IIR 저역 통과 필터: 현재의 상태값과 이전의 상태값이 피드백 구조로 결합되어 노이즈 스펙트럼 밀도를 크게 낮춘다. 구조가 비교적 간단하여 NuttX RTOS 기반의 제한된 연산 환경을 가지는 Pixhawk 하드웨어에서 효율적으로 동작한다.
  • 위상 지연(Phase Delay): 시스템이 높은 주파수의 노이즈를 억제하는 대신, 동압 변화가 급격히 발생하는 동적 비행 조건에서 필터링된 신호는 시간축에서 뒤쳐지는 위상 지연 특성을 가진다.
  • 영점 이탈(Zero-drift) 결함: 기체가 정지 상태에 도달하거나 급격한 감속을 수행할 때, 위상 지연으로 인해 필터 내부 버퍼에 남은 누적된 압력값이 서서히 감소한다. 이때, 물리적인 대기속도는 0에 근접하지만 소프트웨어적인 대기속도 측정치는 특정 임계값을 초과하는 영점 이탈(Zero-drift) 상태를 동적으로 형성하게 된다.

2. 영점 이탈 동적 감지 알고리즘

PX4-Autopilot은 단순한 통계적 임계치 컷오프를 넘어, 비행 모드와 제어 기술(위치 제어 및 자세 제어 알고리즘) 등 기체의 동역학적 상태를 융합하여 센서 유효성을 평가한다.

2.1 잔차 유의성 및 동적 팩터 평가

EKF2(확장 칼만 필터) 기반의 상태 추정기 구조에서, 필터링 이전의 동압 P_{raw}와 IIR 필터 적용 후의 동압 P_{iir} 간의 잔차(Residual) R_k를 실시간으로 모니터링한다. 영점 이탈 동적 감지 평가식은 다음과 같이 정의될 수 있다.

항목수식
잔차 계산R_k = \vert P_{raw,k} - P_{iir,k} \vert
동적 영점 이탈 조건R_k > \theta_{drift} 및 V_{gnd} \approx 0

여기서 V_{gnd}는 위성항법장치(GNSS) 등으로부터 측정된 지상 속도이며, \theta_{drift}는 영점 드리프트 임계치이다. 특정 시간 창(Time Window) 동안 해당 조건이 만족되면, PX4 커맨더(Commander) 모듈은 이를 센서 결함 혹은 이탈로 간주한다.

3. Ardupilot 대비 PX4 관제 구조의 차이점

Ardupilot은 전통적으로 EKF(Extended Kalman Filter) 이노베이션(Innovation) 상태 모니터링에 고도로 의존하여 대기속도 센서의 이상 유무를 판단하는 아키텍처를 따른다.
반면 PX4 시스템은 모듈화된 uORB(Micro Object Request Broker) 퍼블리싱-서브스크라이빙 아키텍처를 적극 활용한다. IIR 필터 적용과 이로 인한 위상 지연 및 드리프트 평가는 EKF2 모듈 외부의 전처리 센서 모듈 계층(예: sensors 데몬)에서 1차적으로 수행된다. 이로 인해 센서 데이터가 EKF 추정기에 도달하기 전에 동형 결함을 빠르게 격리(Fault Isolation)할 수 있어, 제어기(Position/Attitude Controller)에 잘못된 피드백 오차가 반영되는 것을 조기에 차단한다.

4. QGroundControl (QGC) 관제 연동 전략

동적 제로 드리프트 현상이 감지되면 PX4는 MAVLink 백엔드를 통해 QGroundControl로 상태 정보를 피드백한다.

  • 경고 메커니즘: QGC(버전 4.0 이상)에서는 상태 모니터링 모듈이 “Airspeed Sensor Error” 또는 “High Zero-drift Detected” 등의 경고를 발생시킨다.
  • 재교정 명령: GCS 소프트웨어는 사용자에게 MAV_CMD_PREFLIGHT_CALIBRATION 명령어를 전송할 것을 권장하며, 조종사는 대기속도 센서의 피토관을 차폐한 상태에서 영점 교정을 다시 수행하여 동적 드리프트 오프셋을 재설정해야 한다.

이와 같이 PX4-Autopilot은 단순한 신호 처리를 넘어, 위상 지연으로 파생되는 비선형적인 물리적 오류를 비행 전(Pre-flight Check)과 실시간 관제(In-flight Monitoring) 양면에서 체계적으로 관리하는 견고한 소프트웨어 철학을 구현하고 있다.
