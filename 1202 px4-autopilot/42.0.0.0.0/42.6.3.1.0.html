42.6.3.1. 제어 입력 신호의 나이퀴스트 주파수(Nyquist Frequency) 만족 여부 및 에일리어싱(Aliasing) 방지 점검

무인항공기(UAV)의 비행 제어 시스템은 본질적으로 연속 시간(Continuous-time)의 물리적 역학을 제어하는 이산 시간(Discrete-time) 기반의 디지털 제어기이다. 따라서 조종기의 RC 입력이나 지상 관제 시스템(GCS), 반려 컴퓨터(Companion Computer)로부터 전달되는 제어 입력 신호는 디지털 샘플링 과정을 거친다. 이때 제어 입력 신호의 주파수 대역폭과 제어 시스템의 샘플링 속도 간의 수학적 관계, 특히 나이퀴스트 주파수(Nyquist Frequency) 제약 조건을 명확히 해석하고 검증하는 것은 시스템의 응답 안정성을 확보하는 데 필수적이다. 만약 이 조건이 충족되지 않으면, 고주파 제어 명령이 저주파로 왜곡되는 에일리어싱(Aliasing) 현상이 발생하여 기체에 예기치 않은 진동이나 발산을 유발할 수 있다.

1. 나이퀴스트 주파수와 제어 신호 에일리어싱의 이론적 배경

단일 채널 연속 시간 제어 입력 u(t)를 주파수 f_s(Sampling Rate)로 샘플링하여 이산 신호 u[k] = u(k T_s)를 생성한다고 가정하자. (단, T_s = 1/f_s)

나이퀴스트-섀넌 샘플링 정리(Nyquist-Shannon Sampling Theorem)에 따르면, 원본 신호 u(t)를 정보의 손실이나 왜곡 없이 완벽하게 복원(또는 유효하게 제어기로 맵핑)하기 위해서는 신호의 최대 대역폭 주파수 f_{max}가 다음의 나이퀴스트 조건(Nyquist Condition)을 만족해야 한다.
f_s \ge 2 f_{max}
여기서 나이퀴스트 주파수 f_N은 샘플링 주파수의 절반(f_N = f_s / 2)으로 정의된다.

제어 입력(예: 조종사의 급격한 스틱 조작, 혹은 ROS 2 Offboard 노드에서 계산된 고주파 궤적 명령)이 나이퀴스트 주파수 f_N을 초과하는 대역폭 에너지를 포함할 경우, f_N을 넘는 주파수 성분은 제어기 내부에서 낮은 주파수 대역으로 ‘접혀서(Folded)’ 나타난다. 이를 에일리어싱(Aliasing)이라 부르며, 제어기는 이를 실제 저주파 제어 명령으로 착각하여 추종하게 되므로 기체의 심각한 궤적 오차나 위상 지연(Phase Lag) 타격을 초래한다.

2. PX4-Autopilot의 입력 신호 처리 및 에일리어싱 방지 메커니즘

PX4 시스템은 비행 제어 로직에 입력되는 신호의 급격한 변동(Step Input)이나 잡음 섞인 샘플링을 매끄럽게 처리하기 위해 소프트웨어 기반의 디지털 저역통과필터(Low-Pass Filter, LPF) 및 궤적 생성기(Trajectory Generator)를 적용한다.

2.1. RC 입력 필터링 (RC Input Filtering)

조종사가 조작하는 수동 조종 방식에서는, 수신기(Receiver)에서 들어오는 SBUS/CRSF 프로토콜의 패킷 주기가 제한적이다(보통 50Hz ~ 150Hz).

  • RC_FLT_SMP_RATE: RC 신호의 샘플링 속도를 정의한다.
  • RC_FLT_CUTOFF: RC 입력 신호의 필터 차단 주파수(Cut-off Frequency)를 설정한다.
    이러한 필터를 통해 나이퀴스트 주파수 언저리에서 발생할 수 있는 고주파 스파이크성 입력(예: 스틱의 떨림)을 감쇠시켜 에일리어싱을 선제적으로 억제한다.

2.2. 오프보드 및 자동 비행에서의 세트포인트 평활화 (Setpoint Smoothing)

반려 컴퓨터에서 유입되는 MAVLink 메시지나 uXRCE-DDS 퍼블리싱(Publishing) 데이터(예: VehicleLocalPositionSetpoint) 역시 디지털 샘플링 계통을 탄다. PX4 내부 모듈인 FlightTask 및 제어 계통(Position Controller)은 저크 제한 궤적(Jerk-limited Trajectory) 알고리즘을 사용하여 입력된 위치/속도 지시값을 부드럽게 필터링한다.
이는 수학적으로 고주파 스텝 명령에 대해 에너지 대역폭을 저주파 대역으로 강제 스퀴즈(Squeeze)하는 역할을 하여 필연적인 에일리어싱 부작용을 원천봉쇄한다. 파라미터 MPC_XY_CRUISE 및 MPC_JERK_MAX 등이 이 궤적의 형태(주파수 성분 비율)를 결정한다.

3. Ardupilot 대비 PX4의 입력 필터링 설계 철학 차이

PX4-Autopilot과 Ardupilot 모두 입력 에일리어싱과 필터링 시스템을 갖추고 있으나, 세트포인트의 평활화(Smoothing) 관점에서 설계 철학의 차이를 보인다.

  • PX4-Autopilot: 다단 필터 구조 보다는 역학적 궤적 제한(Kinematic Trajectory Limiter)(Jerk, Acceleration 한계값)을 통해 위치/속도 입력망 자체를 연속적인 상태 곡선 형태로 재생산하는 방식을 선호한다. 이로 인해 이산적으로 튀는 Setpoint가 들어와도 비행 동역학 모델 궤적 내에서 에일리어싱이 소멸된다.
  • Ardupilot: 멀티로터의 제어 아키텍처에서 ATC_INPUT_TC (Attitude Control Input Time Constant)와 같은 1차(1st-order) 저역 통과 필터(로우패스 필터)를 적극적으로 활용한다. 조종사의 입력값을 시스템 동역학에 매칭시키기 위해 시간 상수(Time Constant)를 통해 명시적으로 입력 대역폭을 잘라내는 형태를 주로 취한다.

4. 실제 GCS 및 파라미터 튜닝 시 주의 사항

시스템 최적화 및 시험 비행 전, 제어 입력 신호의 건전성을 확보하기 위해 QGroundControl(QGC) 내에서 다음과 같은 점검 사항을 검증해야 정밀 제어가 가능하다.

  1. 에비오닉스(Avionics) 루프 레이트 호환성 점검: 센서 데이터를 읽고 제어기를 연산하는 루프(Loop) 속도(일반적으로 250Hz ~ 400Hz) 대비 제어 명령 퍼블리싱 주기가 적절한지 평가하라. GCS를 통한 조이스틱(Joystick) 제어나 ROS 2 Offboard 노드 개발 시, 퍼블리시 주파수(예: 30Hz)가 루프 샘플링 주파수와 배수 관계 불일치로 인한 채터링(Chattering)이 생기지 않도록 타이머 콜백을 엄밀히 조정하라.
  2. 로그 리뷰(Flight Review) 분석: 비행 테스트 완료 후 ULog 파일 분석(Flight Review 웹 도구 등 활용) 시, vehicle_local_position_setpoint 데이터의 지터(Jitter) 비율을 살펴보라. 입력 주파수가 과도하게 높아 나이퀴스트 허용치를 위협한다면 계단 현상(Staircase effect)과 함께 에일리어싱 왜곡이 제어기 응답 곡선에 반영됨을 확인할 수 있다. 이를 해소하기 위해 오프보드 명령 송신기의 주파수를 낮추거나 내부 궤적 필터를 보강하라.

결론적으로, PX4-Autopilot 기반 개발자는 제어 입력 파이프라인의 **나이퀴스트-섀넌 제약(Nyquist-Shannon Constraint)**을 단순히 이론적 영역으로 취급하지 말고, 로직 내부의 필터 파라미터(RC_FLT_CUTOFF, MPC_JERK_MAX 등) 튜닝을 통해 적극적인 에일리어싱 억제 메커니즘을 시스템에 내재화해야 한다.
