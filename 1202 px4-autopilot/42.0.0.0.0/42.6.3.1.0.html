42.6.3.1 제어 입력 신호의 나이퀴스트 주파수(Nyquist Frequency) 만족 여부 및 에일리어싱(Aliasing) 방지 점검

무인항공기(UAV)의 비행 제어 시스템에서 오프보드(Offboard) 제어기나 지상 관제 시스템(GCS)으로부터 전달되는 제어 입력 신호는 이산 시간(Discrete-time) 기반으로 샘플링되어 처리된다. 이때, 샘플링 정리에 어긋나는 신호가 유입되면 제어 루프의 안정성이 심각하게 훼손될 수 있다. 본 절에서는 제어 입력 신호가 나이퀴스트 주파수(Nyquist Frequency) 조건을 만족하는지 점검하고, 에일리어싱(Aliasing) 현상을 방지하기 위한 PX4-Autopilot의 아키텍처와 검증 메커니즘을 심도 있게 분석한다. (기준 버전: PX4-Autopilot v1.14.0)

1. 나이퀴스트-섀넌 샘플링 정리(Nyquist-Shannon Sampling Theorem)와 제어 신호

아날로그 연속 시간 신호를 이산 시간 신호로 변환할 때, 원래 신호를 왜곡 없이 복원하기 위해서는 샘플링 주파수(f_s)가 원신호(Original Signal)의 최대 주파수 성분(f_{max})보다 최소 2배 이상 커야 한다는 원리이다. 즉, 다음의 조건을 만족해야 한다.

f_s \ge 2 f_{max}

여기서 나이퀴스트 주파수(f_N)는 샘플링 주파수의 절반(f_s / 2)으로 정의되며, 시스템이 표현할 수 있는 최대 주파수를 의미한다. 만약 입력 신호에 나이퀴스트 주파수를 초과하는 고주파 성분이 포함되어 있다면, 해당 성분은 저주파 영역으로 스펙트럼이 접혀서(Folded) 나타나게 되는데, 이를 에일리어싱(Aliasing)이라고 한다. 제어 시스템에서 에일리어싱이 발생하면 저주파의 거짓 신호(False Signal)가 제어기에 유입되어 원치 않는 발진(Oscillation)이나 제어 불능 상태를 유발할 수 있다.

2. 오프보드 제어 및 MAVLink 통신에서의 문제 정의

PX4-Autopilot은 이중 제어 루프 아키텍처(내부 자세 제어 루프 및 외부 위치/속도 제어 루프)를 채택하고 있다. 오프보드 모드(Offboard Mode)에서 외부 컴퓨터(예: ROS2를 구동하는 Companion Computer)가 생성한 제어 명령(위치, 속도, 가속도, 또는 자세 설정값 등)은 MAVLink 프로토콜 또는 uXRCE-DDS 퍼블리싱을 통해 PX4로 전달된다.

일반적으로 오프보드 제어기의 제어 주기는 50Hz에서 100Hz 수준이다. 이는 샘플링 주파수 f_s가 50Hz ~ 100Hz 임을 의미하며, 따라서 나이퀴스트 주파수 f_N은 25Hz ~ 50Hz가 된다. 그러나 네트워크 지터(Jitter), 패킷 손실, 혹은 외부 제어기의 비정상적인 연산 폭주로 인해 입력 신호에 유효 주파수 대역폭을 벗어나는 고주파 노이즈나 급격한 비연속적 계단파(Step-like) 신호가 섞일 경우 문제가 발생한다.

2.1 Ardupilot 대비 PX4의 구조적 차이

Ardupilot은 전통적으로 Guided 모드를 통해 GCS나 외부 API 통신 기반 제어를 수행하며, 오프보드 제어 흐름에 있어 비교적 유연하고 느슨하게 튜닝된 입력 평활화(Smoothing) 로직을 갖는다. 반면, PX4-Autopilot은 Offboard 모드를 통해 매우 엄격한 주기반복성(Periodicity)을 요구한다. MAVLink의 SET_POSITION_TARGET_LOCAL_NED 메시지는 일정한 타임아웃 주기를 가지고 있으며, 입력 데이터의 급격한 변동을 억제하기 위해 내부적으로 더 강건한 저주파 통과 필터(LPF: Low Pass Filter) 및 슬루 레이트 제한(Slew Rate Limiter) 아키텍처를 도입하고 있다.

3. 에일리어싱 방지 통제 메커니즘 (Anti-Aliasing Control Mechanism)

PX4 내부에서는 오프보드 제어 입력으로 인한 에일리어싱 및 고주파 잡음 유입을 차단하기 위해 다양한 단계의 필터링 로직을 수행한다.

3.1 슬루 레이트 제한기(Slew Rate Limiter) 및 가속도 제한

유효 대역폭을 넘어가는 비정상적인 입력 신호 중 가장 대표적인 형태는 비연속적인 급변(Step Change) 신호이다. 이는 주파수 도메인에서 넓은 대역(Broadband)에 걸쳐 주파수 성분을 가지므로 필연적으로 나이퀴스트 주파수 이상의 성분을 포함하여 에일리어싱을 일으킨다.

PX4에서는 이러한 현상을 방지하기 위해 위치/속도 설정값(Setpoint) 경로 계획(Path Planning) 및 생성 단계에 슬루 레이트 제한기(Slew Rate Limiter)를 파라미터로 두고 있다.

  • MPC_ACC_UP_MAX, MPC_ACC_DOWN_MAX: Z축 (수직) 조종 시 최대 허용 가속도
  • MPC_ACC_HOR: 수평 (X, Y축) 조종 시 최대 허용 가속도
  • MPC_JERK_MAX: 최대 허용 저크(Jerk)

이 파라미터들을 통해 계단형 신호를 램프형(Ramp) 혹은 스플라인(Spline) 궤적으로 스무딩(Smoothing) 변환하여 고주파 성분을 강제로 억제(Cut-off)한다. 이는 시간 도메인 필터링의 일환으로 동작하여, 제어기에 입력되는 신호 대역이 나이퀴스트 주파수 한계를 넘지 못하도록 물리적으로 제약한다.

3.2 2차 저주파 통과 필터(2nd Order Low Pass Filter)

입력 신호 외에도 센서 피드백에 의해 발생하는 에일리어싱을 방지하는 것이 필수적이다. 관성 측정 장치(IMU: Inertial Measurement Unit)의 자이로스코프(Gyroscope) 데이터는 진동체 고유 주파수 및 모터 회전에 따른 고주파 잡음을 수반한다.

PX4의 센서 파이프라인(Sensor Pipeline)은 앤티에일리어싱(Anti-aliasing) 목적의 아날로그 및 하드웨어 디지털 필터 외에도 소프트웨어 단계에서 강력한 2차(Second-order) 저주파 통과 필터를 구현한다. (예: IMU_GYRO_CUTOFF 파라미터). 이 구현체를 이해하기 위해 아래와 같은 다이어그램 구조를 참조할 수 있다.

flowchart LR
    A[오프보드 제어기 / GCS] -->|이산 시간 샘플링 제어 명령| B(uORB 토픽 : vehicle_local_position_setpoint)
    B --> C{슬루 레이트 제한기 Slew Rate Limiter & Jerk Limit}
    C -->|대역 제한된 설정값| D[위치/속도 제어기 Position/Velocity Controller]
    
    E[IMU 센서 자이로/가도계] -->|Raw 데이터| F[하드웨어 Anti-aliasing Filter]
    F -->|샘플링| G[디지털 2차 LPF 및 Notch Filter]
    G -->|고주파 억제된 피드백| D

    H --> I[모터 / 액추에이터]

4. 나이퀴스트-에일리어싱 검증 모니터링 체계

PX4 시스템 운영 중 제어기가 나이퀴스트 주파수 제약 조건을 위반하고 있는지 여부를 모니터링하는 현장 검증 방법은 uLog 데이터 분석을 사용하는 것이다. Flight Review 툴을 사용하면 오프보드 제어 입력(Setpoint)과 실제 응답(Estimated State) 간의 주파수 응답을 스펙트로그램(Spectrogram)이나 퓨리에 변환(FFT) 그래프로 표현할 수 있다.

  1. 로그 분석: ULog 파일에서 vehicle_local_position_setpoint 및 vehicle_attitude_setpoint 토픽을 추출하여 해당 신호의 FFT 분석을 수행한다.
  2. 나이퀴스트 대역폭 확인: 해당 설정값의 갱신 주기(예: 50Hz 발행 시 f_N = 25Hz)를 확인하고, 파워 스펙트럼 밀도(PSD) 상에서 f_N 부근의 주파수 성분이 -20dB 이상 남아 있는지 점검한다.
  3. 이상 징후 탐지: 만약 주파수 한계를 초과하는 경우, GCS 연결 불안정에 따른 패킷 병합 현상이나 오프보드 제어기의 불량한 신호 처리 알고리즘을 의심하고, 퍼블리싱(Publishing) 주기 안정화와 외부 컴퓨터 측의 통신 버퍼 튜닝이 요구된다.

이러한 모니터링과 튜닝 프로세스는 무인항공기가 고주파 외란에 의해 구조적 피로(Structural Fatigue) 또는 발열(Motor Overheating)을 일으키는 것을 사전에 방지하는 필수적인 설계 점검 단계이다.
