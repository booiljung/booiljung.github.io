42.5.3 전원 이중화(Redundant Power) 시스템의 교차 점검

상업용 드론 및 대형 무인기(UAV) 운영에서 전원 공급 단절은 기체의 즉각적인 추락으로 이어지는 가장 치명적인 결함(Critical Failure)이다. 특히 가시권 밖(BVLOS) 비행이나 도심 항공 모빌리티(UAM/eVTOL) 환경에서는 단일 점 고장(Single Point of Failure, SPOF)을 방지하기 위해 전원의 이중화(Redundancy) 설계가 필수 요구사항이다.

본 절에서는 하드웨어 계층의 전원 이중화 구조가 PX4-Autopilot 소프트웨어와 어떻게 연동되는지, 비행 전 점검(Pre-flight Check) 단계에서 이중화된 전원의 무결성을 어떻게 교차 점검(Cross-check)하는지 상세히 분석한다.

1. 전원 이중화 하드웨어 아키텍처

Pixhawk 시리즈(예: Pixhawk 4, 5X, 6X 등)를 비롯한 최신 비행 제어기(FC)는 기본적으로 두 개 이상의 전원 입력 포트(POWER1, POWER2)를 제공한다.

하드웨어 레벨에서는 주로 아이디얼 다이오드(Ideal Diode)나 MOSFET 기반의 전원 선택 회로(Power Selection Module)가 동작한다. 이 회로는 두 개의 배터리 혹은 전원 모듈(Power Module)에서 공급되는 전압을 실시간으로 비교하여, 전압이 조금이라도 더 높은 쪽을 주 전원(Primary Power)으로 자동 선택한다. 만약 주 배터리에 셀 단락(Short)이 발생하거나 커넥터가 분리되어 전압이 급속히 떨어지면, 하드웨어 회로는 수 밀리초(ms) 이내에 보조 전원(Secondary Power)으로 스위칭(Switch-over)하여 FMU(Flight Management Unit)의 재부팅을 방지한다.

2. PX4-Autopilot 다중 배터리 소프트웨어 아키텍처

하드웨어의 무중단 스위칭만으로는 비행을 지속할지 Failsafe를 발동할지 결정할 수 없다. PX4는 여러 개의 배터리를 개별적으로 모니터링하고 가시화하는 논리적 계층을 포함한다.

flowchart TD
    A[Power Module 1<br/>V1, I1] -->|ADC / I2C| B(Battery 1 Driver)
    C[Power Module 2<br/>V2, I2] -->|ADC / I2C| D(Battery 2 Driver)
    B --> E[battery_status_0 발행]
    D --> F[battery_status_1 발행]
    E --> G{System Power 모듈<br/>배터리 관리자}
    F --> G
    G --> H[전체 시스템 통합 SoC 및<br/>전압 상태 계산]
    H --> I[Commander / Failsafe 발동 여부 판별]

비행 전 점검(Pre-flight Check) 시, PX4의 Commander 로직은 사용자 파라미터(예: BAT_N_CELLS, CBRK_SUPPLY_CHK)를 참조하여 두 전원 모듈의 교차 점검(Cross-check)을 수행한다.

  • 연결 확인: 두 번째 배터리가 필수로 지정되어 있을 경우, battery_status_1 메시지가 정상적으로 들어오는지 확인한다.
  • 전압 편차 검사: 두 배터리를 병렬로 묶어 사용할 때, 두 배터리의 전압 차이(\vertV_1 - V_2\vert)가 임계값을 초과하면 큰 순환 전류(Inrush/Circulating Current)가 발생하여 화재의 위험이 있으므로 시동(Arming)을 거부(Deny)한다.

3. Ardupilot 대비 아키텍처의 차이

Ardupilot의 경우 BATT_MONITOR 파라미터를 2개 이상 설정하여 다중 배터리 상태를 모니터링할 수 있다. 기능적인 로직은 유사하나, PX4-Autopilot은 상태 머신(State Machine) 구조가 각 센서 및 전원 모듈 메시지를 구독(Subscribe)하는 미들웨어(uORB) 중심적이기 때문에, 특정 전원 드라이버가 오작동하더라도 코어 비행 로직과는 완벽히 격리된 상태에서 결함 인지 및 처리가 가능하다.

또한 PX4는 BAT_N_CELLS와 같은 파라미터를 배터리 인스턴스별로 명확히 분리(BAT1_N_CELLS, BAT2_N_CELLS 등)하여 설정 및 관리하는 아키텍처가 매우 파편화/모듈화되어 있어, 서로 용량이나 스펙이 다른 배터리(비대칭 이중화)를 사용할 때의 모니터링 관리가 직관적이다.

4. QGroundControl 관제 UI 연동

MAVLink 프로토콜은 BATTERY_STATUS 메시지 내에 id 필드를 포함하고 있어, 여러 개의 배터리를 독립적으로 GCS에 전송할 수 있다.

  • QGroundControl(QGC)은 이 id를 기반으로 두 개 이상의 배터리를 툴바(Toolbar)에 “Bat 1”, “Bat 2” 아이콘으로 분리하여 렌더링한다.
  • 교차 점검 알람: 비행 전(Pre-flight) 단계에서 QGC는 PX4로부터 수신되는 SYS_STATUS의 에러 텍스트 메시지(“Preflight Fail: Battery 2 voltage mismatch”)를 화면 중앙에 경고 패널로 띄우며, 작업자가 두 번째 전원의 결선을 물리적으로 확인하도록 유도한다.
  • QGC 내부 소스 코드상에서는 배터리 팩트(Fact) 그룹을 동적으로 생성하여, 기체(Vehicle)가 3개 이상의 배터리(id > 1)를 보유하더라도 자동으로 리스트업하고 개별 셀 전압(Cell Voltage)까지 확장해서 모니터링할 수 있도록 설계되어 있다.

5. 자율 에이전트(Autonomous Agent) 관점 관제 시사점

완전 자율 에이전트 비행에서는 사람이 직접 배터리 커넥터의 체결 상태나 이상 유무를 육안으로 점검할 수 없다. (예: 드론 스테이션에서의 자동 충전 및 이륙)
에이전트에 탑재된 ROS2 관제 노드(Monitor Node)는 uXRCE-DDS를 통해 모든 battery_status_X 토픽을 실시간으로 넘겨받아야 한다. 이중화된 전원의 내부 저항 차이, 방전 속도 편차 등을 머신러닝이나 통계적 기법으로 장기(Long-term) 교차 점검함으로써, 겉으로는 두 개의 전원이 모두 정상처럼 보이더라도 “배터리 2번의 급격한 성능 저하 징후“를 사전 도출해내어 자율적으로 정비(Maintenance) 모드로 전환하는 고도화된 스케줄링 기술이 적용되어야 한다.
