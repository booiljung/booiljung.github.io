42.6.4 다각형(Polygon) 지오펜스 경계 검사 알고리즘을 통한 위치 확인

무인기 비행 관제에서 지오펜스(Geofence)는 가상의 3차원 울타리를 설정하여 기체가 특정 구역을 이탈하거나 진입하는 것을 방지하는 핵심 안전 장치이다. 초기 드론 시스템들은 비행 반경을 원형으로 제한하는 단순한 실린더(Cylinder) 형태의 지오펜스를 사용했으나, 현대의 복잡한 비행 환경 파악 및 도심항공교통(UAM) 규제를 준수하기 위해서는 임의의 형태를 가지는 다각형(Polygon) 지오펜스의 적용이 필수적이다. 본 절에서는 PX4-Autopilot 소스 코드 내에 구현된 다각형 지오펜스의 경계 검사 알고리즘 원리와 수학적 위치 확인(Position Verification) 아키텍처를 상세히 분석한다. (기준 버전: PX4-Autopilot v1.14.0)

1. 다각형 지오펜스의 필요성 및 아키텍처 개요

도심 지역 비행, 시설물 점검, 농업용 방제 등 다양한 임무 환경에서는 비행 인가 구역이 비정형적인 다각형 형태를 띠는 경우가 대부분이다. 또한, 비행 금지 구역(No-Fly Zone, NFZ)을 회피하기 위해 여러 개의 배타적 지오펜스를 동시에 활성화해야 할 때도 있다.

PX4에서는 지오펜스 경계 관리를 위해 navigator 모듈 내에 Geofence 클래스를 구현하여 운용한다. 지상 관제 시스템(GCS)인 QGroundControl은 MAVLink 프로토콜의 FENCE_POINT 혹은 MISSION_ITEM_INT (지오펜스 확장) 메시지를 통해 사용자가 지도상에 지정한 다각형의 꼭짓점 위치(Latitude, Longitude) 리스트를 비행 제어 컴퓨터(Flight Controller, FC)로 전송한다. PX4는 이 포인트 리스트를 내부 메모리(Data Logger 및 RAM)에 저장하고, 실시간 비행 중 추정된 기체의 GPS 위치와 지속적으로 대조(Polled Inspection)하는 구조를 갖는다.

2. 레이 캐스팅 알고리즘(Ray Casting Algorithm)의 원리

다각형 내에 현재 점(비행기의 위치)이 포함되어 있는지 판별하는 수학적 문제를 Point-in-Polygon (PIP) 문제라고 한다. PX4-Autopilot은 이 문제를 해결하기 위해 연산 효율성이 극도로 높은 **레이 캐스팅 알고리즘(Ray Casting Algorithm)**을 응용한다. 이 알고리즘은 ‘점 교차 이론(Crossing Number Method)’ 또는 ‘Even-Odd Rule’ 명칭으로도 불린다.

2.1 수학적 구현 원리

검증하고자 하는 기체 위치 점 P(x, y)에서 시작하여, 2차원 평면상 어느 한 방향(일반적으로 X축 양의 방향, 즉 \vec{r} = (1, 0))으로 무한히 직진하는 가상의 반직선(Ray)을 쏜다고 가정한다.

이 광선이 다각형을 이루는 폐곡선(Closed Boundary)의 각 선분(Edge)과 교차하는 횟수를 카운트한다.

  • 교차 횟수가 **홀수(Odd)**번이면: 기체는 지오펜스 **내부(Inside)**에 위치한다.
  • 교차 횟수가 **짝수(Even)**번이면 (또는 0번이면): 기체는 지오펜스 **외부(Outside)**에 위치한다.

    B --> C{내비게이터 루프 Navigator Loop}
    C --> |기체 현재 위치 좌표 변환| D[Local NED 혹은 구면 좌표 변환]
    D --> E[각 선분마다 교차점 Crossing Number 확인]
    E --> F{Ray 교차 횟수가 홀수인가?}
    F -->|Yes| G[비행 인가 구역 내부 확인 Safe]
    F -->|No| H[지오펜스 이탈 감지 Breach Detected]

3. PX4 소스 코드 내 Geofence 검증 아키텍처 구조

PX4 펌웨어 내 src/modules/navigator/geofence.cpp 에는 실시간으로 PIP 문제를 풀기 위한 메커니즘이 C++로 최적화되어 있다.

3.1 좌표 변환의 중요성

GCS에서 전송된 다각형 꼭짓점 데이터는 일반적으로 WGS84 좌표계인 위도와 경도(Global Coordinates) 값이다. 그러나 레이 캐스팅 알고리즘 등 공간 기하학 연산을 구면 좌표계에서 직접 수행하면 곡률(Curvature)에 의한 오차와 극지방에서의 특이점 문제로 인해 복잡한 삼각함수 연산 부하가 심각해진다.

이를 해결하기 위해 PX4는 기체가 홈 포지션(Home Position)을 기준으로 생성한 **Local NED(North-East-Down) 직교 좌표계(Cartesian Coordinates)**를 활용한다. 다각형 꼭짓점 정보를 수신하면 로컬 맵 투영 방법(Map Projection Method)을 이용하여 2D 미터(meter) 단위의 평면 좌표계로 일괄 변환한다. 이렇게 함으로써 교차 검증은 단순한 직선의 방정식과 대수 연산 수준으로 복잡도가 대폭 낮아져 제어기의 CPU 연산량을 절약할 수 있다.

3.2 경계 검사 최적화 (Bounding Box Pre-check)

임무 중 다수의 다각형과 수백 개의 꼭짓점이 설정되었을 때 매 프레임마다 모든 선분에 대해 레이 캐스팅을 수행하는 것은 오버헤드가 크다. PX4는 AABB(Axis-Aligned Bounding Box) 검사 기법을 선행 전처리 과정으로 수행한다.

  1. 다각형 구역을 완전히 감싸는 평행한 최소 직사각형(Min/Max 래핑 박스)을 미리 계산한다.
  2. 기체 위치 좌표가 이 직사각형 범위(Bounding Box) 바깥에 있다면, 굳이 각 선분과의 레이 캐스팅 연산을 돌리지 않고 바로 바깥쪽(Outside)으로 판단한다.
  3. 기체가 Bounding Box 내부에 진입했을 때만 레이 캐스팅 알고리즘을 활성화하여 정확한 IN/OUT 판별 연산을 수행한다.

위와 같은 구조는 유한한 자원을 가진 임베디드 실시간 운영체제(NuttX RTOS) 환경에서 지연 속도를 거의 0에 수렴시키는 핵심 소프트웨어 디자인 패턴 중 하나이다.

4. Ardupilot 대비 지오펜스 아키텍처 차이 결론

Ardupilot의 경우 지오펜스 관리를 위한 모듈이 센서의 종류에 구애받지 않는 최상위 레벨 안전 관리자(Safety Manager)인 AP_Limit 클래스 패밀리를 통해 좀 더 범용적으로 실행된다. GCS의 접근 차단 메시지 체계 또한 원통형 회피(Cylindrical Avoidance)나 배제 구역(Exclusion Zone)에 집중되어 설계된 시기가 있었다.

반면, PX4-Autopilot은 navigator라는 목적 함수 달성 모듈에 강력히 종속되어 있다. MAVLink 데이터 전송 지연 시간이나 센서 모듈의 오차가 있더라도, Commander에 의해 조율된 단일 비행 상태(State Machine) 내에서 지오펜스 데이터가 일관되게 평가되도록 모듈화의 경계를 확실히 하였다. 이는 복잡한 로티드 다각형(Complex Rotated Polygon) 구역 비행 검사에 더 일관되고 지연이 적은 이점을 제공한다. 향후 자율 에이전트 다수 비행체 제어 시 이러한 PX4의 선형 검증 모델이 비행 허가 충돌 검출 성능에서 크게 유리하게 작용할 것이다.
