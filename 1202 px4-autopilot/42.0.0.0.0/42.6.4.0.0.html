42.6.4. 다각형(Polygon) 지오펜스 경계 검사 알고리즘을 통한 위치 확인

무인항공기(UAV)의 작전 반경이 넓어지고 도심지 내 비행이나 혼잡 공역 비행이 증가함에 따라, 기체가 허가되지 않은 영역으로 이탈하는 것을 방지하는 비행 금지 구역 설정이 필수적이다. 이를 소프트웨어적으로 구현한 것을 지오펜스(Geofence, 지리적 울타리)라 부른다. PX4-Autopilot은 단순한 원형(Circular) 지오펜스 뿐만 아니라, 임무 지역의 복잡한 지형지물이나 사유지 경계를 정밀하게 반영할 수 있는 다각형(Polygon) 형태의 지오펜스 경계 모델을 지원한다. 기체의 비행 전 점검(Pre-flight Check) 단계에서부터 지오펜스의 설정 유효성을 검사하며, 이륙 후에는 실시간으로 다각형 내부(또는 외부) 점유 여부를 검증한다. 본 절에서는 PX4 아키텍처 내에서 다각형 지오펜스를 구성하고 경계를 검사하는 전반적인 과정을 살펴본다. (기준 버전: PX4-Autopilot v1.14 이상, QGroundControl v4.3 이상)

1. 다각형 지오펜스(Polygon Geofence) 구조의 이해

다각형 지오펜스는 GPS(GNSS) 기반의 2D 평면 좌표계(위도, 경도) 혹은 3D 공간(고도 포함) 상에 정의된 꼭짓점(Vertex)들의 집합으로 구성된다. PX4에서는 이러한 다각형 영역을 두 가지 논리적 형태로 구분한다.

  • 포함 구역(Inclusion Zone): 기체가 반드시 이 다각형 영역 ’내부’에 머물러야 하는 펜스이다. 경계를 넘어설 경우 지오펜스 위반(Breach)으로 간주된다.
  • 제외 구역(Exclusion Zone): 기체가 절대 진입해서는 안 되는 영역이다. (예: 공항 주변, 군사 시설, 비행 금지 사유지 등)

이러한 지오펜스 정보는 기체의 비휘발성 메모리에 저장되며, commander 모듈의 내부 지오펜스 라이브러리를 통해 초당 수십 회 이상 기체의 현재 로컬/글로벌 위치와 대조된다.

2. PX4-Autopilot의 지오펜스 경계 검증 파이프라인

PX4의 지오펜스 검증 로직은 센서로부터 EKF2를 거쳐 추정된 실시간 기체 위치(Global Position, 위도/경도)를 다각형 알고리즘에 대입하여 이루어진다.

graph TD
    A[EKF2 상태 추정기<br/>Global Position 갱신] --> B[Navigator 모듈<br/>Geofence Checker]
    C[지오펜스 꼭짓점 데이터베이스<br/>Georeferenced Polygons] --> B
    B --> D{경계 검사 알고리즘<br/>Polygon Inclusion/Exclusion Check}
    D -->|내부 정상 비행| E[명령 수행 지속]
    D -->|지오펜스 위반 감지| F[안전 장치 트리거<br/>Failsafe Action]
    F --> G[RTL / Land / Hold / Terminate]

2.1 파라미터(Parameter) 기반 설정

  • GF_ACTION: 지오펜스를 위반했을 때 수행할 안전 장치 조치를 정의한다.
  • 0: None (경고창만 발생, 조치 없음)
  • 1: Warning (로깅 및 QGC 알림)
  • 2: Hold mode (진행을 멈추고 현 위치에서 호버링)
  • 3: Return mode (RTL, 홈 위치로 복귀)
  • 4: Land mode (즉시 착륙)
  • 5: Flight Termination (비행 종료, 모터 정지 - 가장 치명적일 때 사용)
  • GF_MAX_HOR_DIST / GF_MAX_VER_DIST: 다각형 지오펜스 이외에도 보조적으로 페일세이프용 원형/원기둥형 최대 비행 반경과 고도를 지정하는 파라 파라미터이다.
  • GF_PREDICT: 센서 지연 등을 감안하여, 기체의 속도 벡터를 기반으로 몇 초 후의 위치가 펜스를 넘을 것인지 선제적으로 예측(Prediction)하여 조치할지를 결정한다. 고속으로 이동하는 고정익 기체에서 필수적이다.

3. QGroundControl (QGC)을 통한 지오펜스 MAVLink 연동

지오펜스는 QGroundControl에서 시각적으로 손쉽게 디자인할 수 있다. QGC 플랜(Plan) 뷰의 폴리곤 도구를 사용해 지도 위에 점들을 찍어 포함 구역과 제외 구역을 분리하여 그린다. 사용자가 설정을 기체로 업로드할 때, QGC는 MAVLink의 마이크로 서비스인 펜스 프로토콜(Geofence Protocol)을 사용하여 데이터를 송신한다.

다각형의 각 꼭짓점은 MISSION_ITEM_INT (혹은 최신 프로토콜의 FENCE_POINT) 패킷 형태로 기체로 전송되며, 여기에는 해당 꼭짓점이 속한 폴리곤 인덱스와 위도/경도 좌표(x, y)가 정밀하게 포함된다. PX4 비행 제어기는 전체 꼭짓점 전송이 완료되면 FENCE_STATUS를 통해 정상적으로 수신되었음을 QGC에 응답한다. 비행 전 점검(Pre-flight Check) 단계에서 commander 모듈은 이 폴리곤의 면적이 정상적으로 단일 폐쇄 루프(Closed Loop)를 이루고 있는지, 선분 간 자가 교차(Self-intersection)는 없는지 기본 검수 절차를 거친다.

4. Ardupilot 대비 지오펜스 아키텍처의 차이점

다각형 지오펜스의 구성 방식에서 PX4-Autopilot과 Ardupilot은 MAVLink를 매개로 한다는 공통점이 있으나, 처리 방식과 내부 검증 철학에 미세한 차이가 존재한다.

  • PX4-Autopilot: 지오펜스를 철저히 네비게이터(navigator)의 서브 모듈 수준에서 실시간 플래그로 관리한다. 따라서 어떤 비행 모드(수동 혹은 자동)에 진입해 있느냐에 관계없이 독립적인 데몬(Daemon)처럼 백그라운드에서 동작하여 GF_PREDICT 기능과 융합된 선제적 궤적 제한 시스템(Trajectory Limiter) 역할을 강력하게 수행한다.
  • Ardupilot: 폴리곤 외에도 Geo-fence(원형), 틴(Tin) 형태 등 다양한 지리적 폼팩터를 광범위하게 제공한다. 지오펜스 활성화 시, Ardupilot은 물리적 경계 도달 전 ’경계 범퍼(Fence Margin 방어선)’를 생성하여, 조종사의 스틱 입력 중 펜스를 향하는 성분 자체를 감쇄시켜 마치 기체가 유리벽에 기댄 것처럼 미끄러지게 만드는(Rally 기능 포함) 로봇 제어(Repulsion) 기반의 회피 기능이 상대적으로 더 부각되어 있다.

5. 알고리즘적 도전: 다각형 내 포함(Inclusion) 여부

다각형 지오펜스의 핵심은 “현재 점 P(x,y)가 주어진 폴리곤 내부 공간에 속하는가?“를 판단하는 기하학적 검증이다. 단순한 직사각형 영역이라면 x 좌표와 y 좌표의 최소/최대 범위만 비교하는 O(1) 연산으로 족하지만, 현실의 지오펜스는 오목한 부분(Concave)이 존재하는 비볼록 다각형(Non-convex Polygon) 형태를 띤다.

이를 위해 PX4는 광선 투사 알고리즘(Ray-casting Algorithm)이나 자취 검사 등 위상 수학적 기법을 비행 제어기 보드의 제한적 메모리에서도 연산 가능하도록 최적화하여 펌웨어에 이식하였다. 이어지는 내용에서는 이러한 비볼록 다각형 경계의 심화된 검증 알고리즘 매커니즘을 상세하게 다룬다.
