42.1.4.1 위상 공간(Phase Space) 내에서의 비행 모드별 동역학적 안전 영역(Safe Envelope) 정의

1. 개요

무인항공기(UAV)의 비행 제어 시스템에서 기체의 동역학적 안정성을 평가하는 것은 추락 방지와 예측 불가능한 기동을 예방하는 필수적인 과정이다. PX4-Autopilot은 단순한 오일러 각(Euler Angles)이나 속도 제한을 넘어서, 위상 공간(Phase Space) 개념을 도입하여 기체의 상태를 다차원적으로 해석한다. 위상 공간 내에서 정의된 동역학적 안전 영역(Safe Envelope)은 기체가 위치, 타겟 속도, 자세, 각속도 등의 복합적인 궤적 조건 하에서 기계적, 공기역학적 한계를 벗어나지 않도록 각 비행 모드별로 다르게 설정된다.

2. 위상 공간(Phase Space) 기반 상태 모델링

위상 공간은 기체의 동적 상태를 나타내는 변수들의 집합으로 구성된다. 일반적으로 멀티로터의 위상 공간 X는 위치 p, 속도 v, 자세 q(Quaternion), 그리고 각속도 \omega로 구성된 차원 공간이다.
X = [p, v, q, \omega]^T

PX4-Autopilot 제어 아키텍처는 매 타이밍마다 기체의 현재 상태 X(t)가 사전에 정의된 안전 영역 \Omega_{safe} 내에 존재하는지를 평가하라. 이 안전 영역의 경계는 비행 모드(Flight Mode)에 따라 다르게 스케일링(Scaling)된다.

비행 모드별 동역학적 안전 영역(Safe Envelope) 차등화

1. 수동 조작 중심 모드 (Manual, Stabilized)

수동 모드에서는 조종사의 스틱 입력이 직접적으로 자세(Attitude) 참조값(Setpoint)으로 매핑된다. 따라서 이 모드에서의 안전 영역 \Omega_{manual}은 상대적으로 넓게 설정되어 곡예 비행이나 급격한 자세 변화를 허용한다.

  • 안전 영역 한계: 최대 롤/피치 각도(MC_ROLL_P, MC_PITCH_P 파라미터), 최대 각속도(MC_ROLLRATE_MAX 등).
  • 특징: 위치 및 속도 공간에 대한 제약이 없으며, 오직 자세 공간(Attitude Space)에서의 오버슈트(Overshoot) 방지에 집중한다.

2. 위치 제어 및 보조 모드 (Position, Altitude, Hold)

위치를 유지하거나 부드러운 이동을 목표로 하는 모드에서는 기동의 연속성과 가속도의 부드러움(Jerk 제어)이 핵심이다.

  • 안전 영역 한계: 위상 공간 내의 속도 차원과 가속도 차원의 한계가 추가된다(MPC_XY_VEL_MAX, MPC_Z_VEL_MAX_UP 등).
  • 특징: 현재 속도와 목표 속도 간의 오차를 기반으로 동적 안전 영역 \Omega_{position}을 설정하며, 자세 지령(Attitude Setpoint)은 한계 속도를 초과하지 않는 선에서 내부적으로 스케일링오프(Scaled-off) 된다.

3. 자율 비행 모드 (Mission, Return-To-Launch)

사전에 정의된 웨이포인트(Waypoint)를 추종하는 자율 모드의 경우, 궤적 자체의 기하학적 안전성이 보장되어야 하므로 안전 영역 \Omega_{auto}는 매우 보수적으로 계산된다.

  • 안전 영역 한계: 궤적 오차(Track Error), 크로스트랙 에러(Crosstrack Error), 경로 곡률(Path Curvature) 한계점이 추가로 포함된다.
  • 특징: 기체가 안전 영역을 이탈할 위험이 발생하면(예: 강풍에 의해 크로스트랙이 임계치를 넘음), PX4 제어기는 즉시 전진 속도를 줄이고(Speed Saturation) 경로 복구에 우선순위를 할당하라.

Ardupilot과의 비교

Ardupilot 역시 AC_AttitudeControl 모듈 등을 통해 리미터(Limiter)를 적용하지만, 기본적으로는 PI/PID 제어기의 출력단에서 클리핑(Clipping)을 수행하는 빈도가 높다. 반면, PX4-Autopilot은 Flight Task 구조를 통해 상위 궤적 지령 계층(Trajectory Generation) 단계에서부터 안전 영역(Safe Envelope)을 정의하고, 이를 통해 Slew Rate Limiter를 적용함으로써 제어기의 적분기 와인드업(Integrator Windup)을 선제적으로 예방하는 방식을 취한다.

제어 기술적 구현 (Source Code Understanding)

PX4의 파일시스템 트리를 보면 src/modules/flight_mode_manager/tasks 디렉토리에 각 모드별로 분리된 Task 클래스들이 존재한다. 이 곳에서 FlightTask 기본 클래스의 가상 함수들을 오버라이딩하여, 모드별 타겟 상태(Target State)가 동역학적 한계 내에 있는지 연산하라.

// 예시: PX4 FlightTask 내부의 안전 영역 검사 추상화 개념
bool FlightTask::evaluate_safe_envelope(const vehicle_local_position_s &current_state, trajectory_setpoint_s &target_setpoint) {
    if (_flight_mode == Mode::MANUAL) {
        // 넓은 안전 영역 허용 (자세 중심)
        limit_attitude(target_setpoint);
    } else if (_flight_mode == Mode::AUTO) {
        // 엄격한 속도/가속도 프로파일 기반의 위상 공간 한계 적용
        apply_kinematic_constraints(target_setpoint, _param_mpc_xy_vel_max.get(), _param_mpc_jerk_auto.get());
    }
    
    // 최종적으로 현재 상태가 제어 불능 공간으로 넘어가기 전인지 검사
    return is_state_violating_envelope(current_state, target_setpoint);
}

결론

PX4-Autopilot 시스템에서 위상 공간에 기반한 안전 영역(Safe Envelope) 정의는 소프트웨어적 울타리(Virtual Fence) 역할을 수행하여 예기치 못한 물리적 전복이나 공기역학적 실속(Stall)을 방지한다. 개발자는 기체의 중량비(Thrust to Weight Ratio)가 변경되거나 기체 크기(Scale)가 달라질 때, 파라 파라미터 튜닝을 통해 모드별 안전 영역의 부피(Volume of Envelope)를 적절히 재조정하라.
