42.7.1 SYS_STATUS 메시지를 활용한 비트마스크(Health Bitmask) 인코딩 메커니즘

지상 관제 시스템(Ground Control Station, GCS)과 무인항공기(UAV) 간의 데이터 텔레메트리(Telemetry) 통신에서 가장 핵심적인 기체 상태 정보는 MAVLink 프로토콜의 SYS_STATUS 메시지(Message ID: 1)를 통해 주기적으로 전달된다. 이 메시지는 시스템의 전반적인 상태, 배터리 전압, 센서 헬스(Health), 통신 오류율을 포함하며, 탑재된 다양한 센서 및 제어 서브시스템의 물리적 존재(Presence), 활성화(Enabled), 그리고 정상(Health) 여부를 32비트 비트마스크(Bitmask) 형태로 엄격하게 인코딩하여 전송한다. 이러한 비트마스크 기반의 인코딩 기법은 최소한의 페이로드(Payload) 크기로 수십 개의 서브시스템 상태를 다중화(Multiplexing)할 수 있는 고효율 데이터 직렬화(Serialization) 메커니즘을 제공한다. 본 문서에서는 PX4-Autopilot 펌웨어(v1.14 이상)와 QGroundControl(v4.3 이상)을 기준으로 비트마스크 시스템의 세부적인 작동 구조를 심층적으로 분석한다.

1. 비트마스크(Health Bitmask) 구조 및 MAV_SYS_STATUS_SENSOR 매핑 메커니즘

SYS_STATUS 메시지는 데이터 효율성을 극대화하기 위하여 세 가지 주요 32비트 무부호 정수(unsigned 32-bit integer, uint32_t) 필드를 정의한다. 각 비트(0~31)는 사전에 정의된 특정 서브시스템에 1:1로 매핑된다.

  1. onboard_control_sensors_present: 하드웨어나 서브시스템이 물리적으로 장착되어 보드 상에 인식됨을 나타낸다. (예: I2C 버스에 자력계가 성공적으로 프로빙된 상태)
  2. onboard_control_sensors_enabled: 해당 서브시스템이 현재 드라이버 레벨에서 활성화(Enabled)되어 비행 제어 루프나 추정기 구동에 실질적으로 참여하고 있음을 의미한다.
  3. onboard_control_sensors_health: 각 서브시스템의 무결성 및 정상 동작 여부(Health Ok)를 지시한다. 비트가 1이면 정상 스펙 범위 내에서 데이터가 들어오고 있음을 의미하고, 0이면 데이터 시간 초과(Timeout) 등 고장(Failure) 또는 예외 상태를 대변한다.

각 32비트 변수의 특정 비트는 MAV_SYS_STATUS_SENSOR 열거형(Enum)에 정의된 표준 매핑 룰을 따른다. MAVLink 표준에 기반하여 주로 활용되는 센서 및 서브시스템의 비트 매핑은 다음과 같다.

비트 플래그 (Hex)비트 위치제어 및 센서 서브시스템 (MAV_SYS_STATUS_SENSOR 기준)
0x00000001Bit 03D 자이로스코프(3D Gyroscope)
0x00000002Bit 13D 가속도계(3D Accelerometer)
0x00000004Bit 23D 자력계(3D Magnetometer / Compass)
0x00000008Bit 3절대 압력 센서(Absolute Pressure / Barometer)
0x00000010Bit 4차압 센서(Differential Pressure / Airspeed)
0x00000020Bit 5위성항법장치(GNSS / GPS)
0x00000040Bit 6광학 흐름 센서(Optical Flow)
0x00000080Bit 7컴퓨터 비전 기반 위치 데이터(Computer Vision Position)
0x00000100Bit 8레이저 거리 측정기(Laser Rangefinder)
0x02000000Bit 25AHRS 서브시스템 (자세 추정기 무결성)
0x08000000Bit 27지형 인식(Terrain) 서브시스템

이러한 개별 비트들은 논리합(Bitwise OR) 연산으로 병합되어 32비트 변수 내에 다수의 센서 상태를 고도로 압축된 형태로 나타내게 된다.

2. PX4-Autopilot의 비트마스크 생성 로직 및 아키텍처 (소스 코드 계층)

PX4-Autopilot의 내부 소프트웨어 아키텍처에서는 uORB (Micro Object Request Broker) 메시징 시스템을 통해 수많은 센서 인스턴스와 확장 칼만 필터(EKF2) 추정기의 상태를 실시간으로 퍼블리시한다. 중앙 집중식 통합 통신 모듈인 mavlink 모듈(설계상 mavlink_messages.cpp 컴포넌트 내부의 MavlinkStreamSysStatus 스트리밍 클래스)은 주기적으로 1Hz 전후의 빈도로 vehicle_status 및 내부 상태 집합(예: sensor_combined 토픽)을 구독하여 위 3가지 32비트 마스크 필드를 갱신한다.

2.1 인코딩을 위한 비트 논리 연산(Bitwise Operation) 프로세스

내부 스트리머 모듈 및 commander 데몬에서는 아래와 같이 비트 마스킹(Bit Masking)과 비트 반전(Bit Inversion) 연산을 사용하여 시스템 결함 여부를 동적으로 헬스 맵에 투영한다. 단 한 차례의 클록 사이클에 다수의 상태를 일괄 판별하기 위한 비트 연산 기법이 집중적으로 활용된다.

graph TD;
    A[uORB 센서 데이터 수신 및 상태 머신 평가] --> B{서브시스템 평가 분기점};
    B -->|자이로스코프 정상| C1[sys_health |= MAV_SYS_STATUS_SENSOR_3D_GYRO\n'Bit 0 On'];
    B -->|자이로스코프 고장 감지| C2[sys_health &= ~MAV_SYS_STATUS_SENSOR_3D_GYRO\n'Bit 0 Off'];
    B -->|위성항법장치 정상 작동| D1[sys_health |= MAV_SYS_STATUS_SENSOR_GPS\n'Bit 5 On'];
    B -->|GPS 타임아웃 감지| D2[sys_health &= ~MAV_SYS_STATUS_SENSOR_GPS\n'Bit 5 Off'];
    B -->|기타 하드웨어 모듈 검사| E[압력 센서, 비전, 거리 센서 누적 집계];
    C1 --> F[최종 집계: 32-bit sys_health_mask 변수];
    C2 --> F;
    D1 --> F;
    D2 --> F;
    E --> F;
    F --> G[SYS_STATUS 메시지에 패키징하여 UART/UDP 통해 전송];
    G --> H[QGroundControl 관제 모듈에서 Decode 및 UI 전시];

이 연산 구조에서 비트 논리합 할당 연산자(|=)는 특정 센서의 정상 컨디션을 1로 활성화할 때 이용된다. 반대로 임의의 센서 시간 초과(Timeout)가 발생하거나 비정상적인 데이터 스파이크 등으로 무결성 검사(Sanity Check)에 실패하게 되면, 비트 반전 직후의 마스킹 연산(&= ~마스크상수)을 통해 해당 센서의 비트 필드를 정밀하게 0으로 소거하여 고장(Failure)임을 즉각적으로 반영한다.

3. PX4-Autopilot과 Ardupilot의 비트마스크 아키텍처 차이점 분석

PX4와 Ardupilot 모두 오픈소스 MAVLink 스펙이라는 공통 언어를 기반으로 하지만 텔레메트리 내부 구조에서의 추상화(Abstraction) 전략은 현격한 차이점을 지닌다.

  1. PX4-Autopilot의 uORB 중앙 상태 관리 패러다임
    다중 센서 인스턴스(Multi-sensor redundancy, 예: 3개의 자이로 센서 탑재) 환경에서 PX4 내부 투표기(Voter, 센서 셀렉터 컴포넌트) 메커니즘과 상태 추정기(EKF2)가 어떤 센서가 대표적인 “건강한 상태“인지 사전에 평가 및 채택한다. 즉, 하위 드라이버에서 취합된 정보를 sensor_combined 레벨에서 융합한 뒤 비트마스크로 요약하여 보고하므로 SYS_STATUS가 차량 수준(Vehicle-level)의 대표 헬스를 대변하게 되는 경향이 강하다. 이는 컴포넌트간 결합도가 낮아 이식성이 뛰어난 장점이 있다.
  2. Ardupilot의 디바이스 드라이버 상태 전파 모델
    Ardupilot 프론트엔드 계층 구조 내에서는 개별 센서 객체 및 AP_NavEKF 코어 라이브러리 차원에서부터 독립적인 헬스 비트 플래그를 관리한다. 장치 드라이버에서 고장이 발생하면 전역 배열 변수를 직접 갱신하여 상위 MAVLink 스트리머 객체의 버퍼로 고주파수(High frequency)로 융합되는 바운더리 업데이트(Boundary update) 패턴이 주를 이룬다.

4. 모니터링 및 실무 적용 (QGroundControl 연동 메커니즘)

QGroundControl (이하 QGC)의 Vehicle 관리자 백엔드 객체(Vehicle.cc)는 C++ 기반 컴포넌트로 동작하며, 지속 수신되는 SYS_STATUS 메시지를 파싱하여 내장된 onboard_control_sensors_health 비트를 해석한다.
특정 비트가 present 및 enabled 상태임에도 불구하고 health 비트가 0으로 전환되는 경우, QGC는 이를 즉각적인 하드웨어 오류로 판단한다. 그 즉시 UI 상단 툴바의 인디케이터(Indicator) 색상을 적색으로 점멸시키고 내부 로깅(App Log) 시스템에 오류를 기록하며, 심각도(Severity)에 따른 보이스 피드백(Voice warning) 시스템을 함께 호출한다. 이는 조종자가 공중에서 발생한 센서 오작동을 즉시 파악하고, 페일세이프(Failsafe) 매뉴얼 대응을 할 수 있게 돕는 관제(GCS) 소프트웨어 아키텍처의 필수적인 구현 요소로 자리잡고 있다.

따라서 비트마스크 인코딩 방법론은 비단 통신 대역폭 절감 효과뿐만 아니라, 무인기의 자율 비행 신뢰성과 지상 관제의 기민한 판단을 결속하는 고도로 최적화된 저지연(Low-latency) 시스템 안전성 아키텍처라 정의할 수 있다.
