42.7.1.1 비트 논리 연산(Bitwise Operation)을 통한 서브시스템 상태 벡터의 직렬화(Serialization) 기법

1. 개요

PX4-Autopilot과 같은 현대적인 비행 제어 소프트웨어는 수많은 이기종 센서(자이로스코프, 자력계, GPS 등)와 상태 추정 서브시스템(EKF2 등)을 동시에 제어하고 운영한다. 하드웨어 자원의 제약이 존재하는 임베디드 실시간 운영체제(RTOS) 환경과 대역폭이 한정적인 무선 데이터 링크(Telemetry) 구간에서, 수십 가지에 이르는 서브시스템 헬스 상태(Health States)를 지상 관제 시스템(GCS)으로 다중화하여 보고하는 가장 핵심적인 기반 기술은 비트 논리 연산(Bitwise Operation)을 활용한 상태 벡터의 직렬화(Serialization) 기법이다. 이 기법은 개별 서브시스템의 불리언(Boolean) 상태들을 32비트(uint32_t)와 같은 단일 정수형 공간 내에 개별 비트로 매핑시킴으로써, 메모리 및 네트워크 페이로드 부하를 기하급수적으로 낮출 수 있는 최적의 아키텍처적 장점을 제공한다.

2. 상태 벡터(State Vector)의 직렬화 원리와 비트 매핑

상태 벡터 직렬화는 통상적인 배열 구조나 개별 변수들을 논리곱(AND), 논리합(OR), 논리 부정(NOT), 비트 시프트(Bit Shift) 연산을 사용하여 하나의 레지스터 공간(예: 32-bit Integer variable)으로 압축 패키징하는 프로세스이다. PX4-Autopilot의 텔레메트리 스트리머 컴포넌트는 uORB 메시지 구조체(주로 vehicle_status_s)에서 하위 센서 모델 및 구동 모듈의 헬스 상태를 읽어들인 뒤, MAVLink 프로토콜의 표준 시스템 상태 메시지인 SYS_STATUS의 비트마스크 필드에 즉각 매핑한다.

2.1 주요 비트 논리 연산 기초 기법

C++ 환경의 펌웨어 소스 레벨에서 상태를 인코딩하기 위해 주로 다음 3가지 비트 연산 기법을 채택한다.

  1. 비트 셋(Bit Set) - 논리합(OR): 특정 서브시스템이 정상이거나 활성화되었을 때, 기존 비트들은 보존한 채 목적하는 위치의 비트만을 1로 치환하기 위해 사용한다. 
    // 3번 비트 위치에 매핑된 자력계(Magnetometer)를 정상(Health)으로 설정
uint32_t _health_mask |= (1 << 3); 

2.  **비트 클리어(Bit Clear) - 단항 부정(NOT) 및 논리곱(AND)**: 특정 프로세스나 자력계 버스에 하드웨어 I2C 타임아웃이나 오류 스파이크가 발생하여 비정상(Unhealthy)으로 판정될 경우, 다른 정상 상태 센서의 데이터 비트는 훼손하지 않고 해당 비트만을 `0`으로 소거한다.
    ```cpp
    // 3번 비트 위치에 매핑된 자력계 상태만을 초기화(0)하여 고장 상태로 마킹
    uint32_t _health_mask &= ~(1 << 3);
  1. 비트 체크(Bit Check) - 논리곱(AND): 직렬화된 시스템 벡터 데이터를 QGroundControl(QGC)와 같은 관제 시스템에서 온전히 수신한 후, 이를 직렬화 해제(Deserialization)하고 UI에 파싱하여 시각화할 때 활용된다. 
    // 3번 비트 위치에 매핑된 모듈이 살아있는지 검증
bool is_mag_healthy = (_health_mask & (1 << 3)) != 0;


## 3.  PX4-Autopilot 내부 직렬화 아키텍처 및 구현 흐름


PX4-Autopilot (v1.14 펌웨어 기준) 소스 코드 내부의 MAVLink 스트리밍 엔진 모듈(`src/modules/mavlink/mavlink_messages.cpp`의 `MavlinkStreamSysStatus` 구현체 등)에서는 서브시스템 상태를 단일 프레임으로 직렬화하기 위해 명확하게 계층화된 데이터 플로우 흐름을 거친다.

```mermaid
sequenceDiagram
    participant S as 센서 드라이버 (Sensor Driver)
    participant U as uORB 버스 (sensor_combined 등)
    participant C as Commander (안전성 상태 머신)
    participant M as MAVLink Streamer
    participant G as GCS (QGroundControl)

    S->>U: 센서 로(Raw) 데이터 및 타임스탬프 퍼블리시
    U->>C: 상태 이탈 또는 데이터 갱신 누락 감지 스레드 동작
    C->>C: 센서 무결성 알고리즘 평가 (Timeouts, Spikes)
    C->>U: 글로벌 vehicle_status_s 구조체 갱신 발행
    M->>U: 약 1Hz 속도로 상태 토픽 구독 (Polling)
    M->>M: 단일 uint32 변수에 비트 논리 연산 직렬화 개시
    Note right of M: sys_health_mask |= MAV_SYS_STATUS_SENSOR_GPS
    M->>G: 직렬화 완료 후 MAVLink SYS_STATUS 인코딩 전송
    G->>G: 패킷 수신 후 직렬화 해제(Deserialization) 및 UI 투영

위 시퀀스 다이어그램에서 나타나듯, Commander 컴포넌트 내부의 상태 머신 평가 루프는 최대 디폴트 루프 타임인 수백 헤르츠의 고속 클록 주기로 동작하며 하부 센서 객체 및 EKF2의 추정 건전성을 면밀히 감시한다. 그러나 무선 통신 레이어를 전담하는 MAVLink 스트리머는 상대적으로 저속(보통 1Hz 내외 빈도)으로 폴링 엔진을 돌면서 uORB의 집약 데이터를 비트 벡터(Bit Vector)로 압축 및 직렬화한다. 이러한 레이어 분리 설계는 고빈도로 변화하는 내부 상태 노이즈나 일시적 지터(Jitter)를 안정적으로 필터링하고 직렬화 과정에서 유발될 수 있는 CPU 오버헤드를 극소화하는 공학적 이점이 돋보이는 구조이다.

4. 직렬화 메커니즘을 접근하는 PX4와 Ardupilot 간 아키텍처 철학 대칭

비트 연산을 수단으로 한 헬스 데이터 직렬화 논리 구조물은 고도로 최적화된 C++ 객체 지향 환경을 채택한 PX4-Autopilot과 Ardupilot 진영 모두 펌웨어 최하단까지 광범위하게 채용하고 있다. 그러나 해당 비트가 매핑되고 패키징되는 데이터의 취합 궤적은 양 진영 간 구조적 차이가 대조적이다.

  • PX4-Autopilot (Publisher/Subscriber 모델의 중앙 집중 직렬화): PX4는 철저히 uORB 토픽 지향 설계를 취하며, 각 하위 디바이스 드라이버가 MAVLink 메시지를 직접 참조하고 알지 못하도록 완전히 격리되었다. 즉, 드라이버 모듈 전용의 독립적 불리언 플래그가 글로벌 vehicle_status 구조체로 도달하며 한 곳에 모인 뒤, MAVLink 모듈 계층(Data Link Layer)의 단일 접점에서 비트 연산을 통하여 논리적 일괄 압축이 달성된다. 이에 따라 관심사의 분리(Separation of Concerns) 아키텍처가 매우 선명하게 도출된다.
  • Ardupilot (공유 객체 기반의 분산 투영): Ardupilot의 코어 백엔드 아키텍처는 거대한 전역 싱글톤(Singleton) 객체 레퍼런스 모델을 근간으로 삼고 있다. 각각의 모듈 센서 프론트엔드 모듈 객체가 자체적인 포인팅 레퍼런스를 참조하며 자신의 하드웨어 헬스 상태 배열 플래그 변수를 직접 다이렉트 변조한다. 통신 매니저인 GCS_MAVLink 블록이 인터럽트 루프 상에서 이러한 분산된 프론트엔드 상태들을 취합해 즉시 마스킹 논리합 연산을 처리하므로, 커플링 레이어(Coupling layer)가 보다 조밀하고 깊숙하게 엮여 있는 편이다. 다이렉트 캐싱(Caching) 관점에서의 메모리 속성 접근은 미세하게 고속일수 있으나 소스 코드의 컴포넌트 논리 분할 관점의 유지보수 난이도는 증가하는 양상을 띤다.

5. 요약 및 결론

비트 논리 연산을 핵심 동력으로 설계된 서브시스템 상태 벡터 직렬화 기술은, PX4-Autopilot 내 단일 32비트 워드(Word) 메모리 공간에 수십 개의 분별적 안전 센서 상태(Health States) 체인을 패키징하는 초고효율 소프트웨어 아키텍처를 실현했다. 이 기법은 시리얼(Serial) 버스와 무선 공중 텔레메트리 관제망 등 제한된 대역폭 하중 조건 속에서, 단 한 바이트의 패킷 낭비 없이 무인기 관제(UAV)의 다중 하드웨어 중복성(Redundancy) 건전도를 지연 없이 통제할 수 있게 돕는 시스템 공학적 최신 설계의 표준으로 자리매김하였다.
