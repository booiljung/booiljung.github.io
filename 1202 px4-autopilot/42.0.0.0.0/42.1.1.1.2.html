42.1.1.1.2 비잔틴 장애 허용(Byzantine Fault Tolerance) 관점에서의 센서 데이터 무결성 합의 기준

1. 개요

현대의 무인항공기(UAV) 비행 제어 시스템이 직면하는 가장 치명적인 고장 시나리오는 센서가 단순히 신호 전송을 멈추는 페일 스톱(Fail-Stop) 형태가 아니다. 센서가 물리적으로는 작동하지만 자성 왜곡, 열화 노이즈, 혹은 소프트웨어 버그로 인해 “거짓이거나 무작위적인 데이터“를 마치 정상적인 데이터인 것처럼 계속해서 비행 제어 컴퓨터(FCC)에 송출하는 상황이 훨씬 더 위협적이다. 이를 분산 컴퓨팅 이론에서는 비잔틴 장애(Byzantine Fault) 라 일컫는다.

PX4-Autopilot(v1.14 기준)의 다중화(Redundancy) 센서 시스템에서 가장 중요한 철학은, 단순히 여러 센서의 평균값을 취하는 것이 아니라, 악의적이거나 모순된(Malicious or Conflicting) 데이터를 내뿜는 ’비잔틴 센서 노드’를 어떻게 수학적으로 격리하고 나머지 정상 노드들 사이에서 무결성 합의(Integrity Consensus)를 도출해 낼 것인가에 있다.

2. 무인기 시스템에서의 비잔틴 장애 허용(BFT) 조건

전통적인 비잔틴 장군 문제(Byzantine Generals Problem) 솔루션에 따르면, f개의 비잔틴 결함 노드를 수용하고 올바른 합의에 도달하기 위해서는 전체 노드 수 N이 N \ge 3f + 1을 만족시켜야 한다.
따라서 무인기 환경에서 단 1개의 센서가 거짓 정보를 내뿜는 비잔틴 장애(f=1)를 완벽하게 극복하고 어느 센서가 정상인지 다수결(Majority Voting)로 합의하기 위해서는 최소 4개의 센서가 필요하게 된다.

그러나 상용 Pixhawk 시스템(예: Pixhawk 6X)은 대개 3중 다중화(Triple Modular Redundancy, TMR) 센서를 채택한다. 즉, N=3이므로 엄밀한 수학적 BFT 조건(N \ge 4)을 충족하지 못하는 시스템적 한계를 지닌다. PX4-Autopilot은 이 한계를 극복하기 위해 단순 다수결 합의가 아닌, 내부 추정기(Estimator)의 동역학적 예측(Kinematic Prediction) 모델을 합의 과정에 개입시키는 우회 전략을 구사한다.

3. PX4-Autopilot의 센서 무결성 합의 아키텍처 (EKF2 기반)

센서가 3개(N=3)인 상황에서 1개의 비잔틴 장애가 발생할 경우, 2개의 센서는 서로 다른 두 값을 내뿜게 되며 단순 보팅 알고리즘으로는 교착 상태(Deadlock)에 빠질 수 있다. PX4는 커맨더(Commander) 내부의 Sensor Selection 모듈과 확장 칼만 필터(EKF2)를 결합하여 데이터 무결성을 검증한다.

동작 원리는 다음과 같다: EKF2는 과거의 상태량(State)과 물리 법칙(가속도-속도-위치 적분)을 바탕으로 ’현재 나와야만 하는 이상적인 추정치’를 가지고 있다. 만약 3개의 센서 데이터가 서로 맞지 않을 때, 이 ’물리적 추정치’와의 오차 파생분(Innovation)이 허용 범위 내에 들어오는 센서들만이 합의 그룹에 편입되며, 벗어나는 센서 데이터는 거짓 정보로 규정되어 가중치가 0으로 격하된다.

4. Ardupilot 대비 무결성 합의 기준의 차별성

PX4와 Ardupilot 모두 3중 센서를 운용하지만, 비잔틴 장애에 대한 대응 로직에서 구현의 방향성이 다르다.

  • Ardupilot의 레인(Lane) 기반 합의: Ardupilot은 독립적인 EKF3 코어를 3개의 ’레인(Lane)’으로 병렬 구동하여, 각 레인이 각자의 센서를 물고 3개의 독립적인 시스템 상태 추정치를 생산해 낸다. 이후 최상위 계층에서 3개의 레인 헬스치(Health Score)를 비교하여 가장 건강한 레인으로 통째로 전환(Lane Switch)하는 통나무형(Monolithic) 합의 방식을 애용한다.
  • PX4-Autopilot의 개별 컴포넌트 합의: 반면 PX4는 EKF 연산에 들어가기 직전, sensors 모듈에서 3개의 IMU 측정치를 먼저 모아 놓고 (EKF 외부에서) 가중 평균(Weighted Average)이나 단일 선별(Selection)을 해버리기 때문에 단일 EKF 코어만이 구동된다. 이는 연산 자원의 절약과 비잔틴 데이터의 조기 격리(Early Isolation)에 탁월하나, 알고리즘 튜닝이 어긋날 경우 한 번 버려진 센서가 다시 합의에 참여하기 어렵다는 특이점을 지닌다. 따라서 QGroundControl(v4.3)의 비행 전 점검(Pre-flight Check) 단계에서 센서 간 일치성을 매우 타이트하게 강제하는 것이다.

5. 실무적 관점: 커스텀 페일세이프 룰셋 설계 가이드

비잔틴 장애 허용 논술은 지상 관제망 개발자들에게 중요한 교훈을 준다. QGroundControl이나 커스텀 GCS를 구성할 때 단순히 텔레메트리 값을 화면에 표시하는 것에 그쳐서는 안 되며, 특정 센서의 혁신 오차 비율(Innovation Test Ratio)이 치솟는 상황을 GUI 상에 경보(Warning)로 가시화해야 한다.

“비잔틴 장애를 허용한다(Fault Tolerant)“는 것은 결함이 무시되어도 좋다는 뜻이 아니라, 결함이 시스템을 추락시키지 않게 막아둔 임시 보호 상태에 불과함을 직시하라. 페일세이프 룰셋은 BFT 메커니즘이 발동한 순간 즉각 기체를 지상으로 복귀(RTL)시키도록 설계되어야 한다.

6. 결론

무인항공기의 3중 다중화(Redundancy) 센서 네트워크 환경에서, 센서 모듈이 내뿜는 소리 없는 거짓말(비잔틴 장애)은 전체 제어 인프라를 전복시킬 수 있는 최대의 위협이다. PX4-Autopilot은 N=3이라는 분산 합의 이론의 구조적 한계를 EKF2라는 동역학적 예언자(Predictor)를 개입시킴으로써 우회하는 우아한 알고리즘을 선보인다. 이 BFT 관점의 센서 무결성 메커니즘을 온전히 이해해야만, 예외적 환경에서도 결코 붕괴되지 않는 자율 비행 시스템의 안전 보장에 한 걸음 더 다가설 수 있다.
