42.2.4. 히스테리시스(Hysteresis) 필터링 기법을 적용한 센서 노이즈 기반 오탐지(False Positive) 방지

무인기 시스템의 비행 전 점검(Pre-flight Check) 단계는 기체의 안전한 비행을 보장하기 위한 가장 필수적인 검증 과정이다. 이 과정에서 관성측정장치(IMU), 지자기 센서(Magnetometer), 기압계(Barometer) 등 다양한 센서(Sensor)로부터 수집되는 원시 데이터(Raw Data)는 필연적으로 기계적 진동이나 전기적 특성에 의해 노이즈(Noise)를 동반한다.

센서 데이터의 유효성을 판단할 때, 단순한 임계값(Threshold) 기반의 상태 판별 기준을 적용하면 찰나의 노이즈 스파이크나 고주파 진동으로 인해 센서 상태가 정상(True)에서 비정상(False)으로 급격히 흔들리는 상태 전이 진동 현상, 이른바 **채터링(Chattering)**이 발생할 수 있다. 이러한 채터링은 기체가 실제로는 정상임에도 불구하고 센서 에러로 오보하는 **오탐지(False Positive)**를 유발하여 불필요하게 이륙 승인(Arming)을 거부(Deny)하는 결과를 초래한다.

따라서 PX4-Autopilot(예: 펌웨어 버전 v1.14 이상 기준)은 단순한 임계값 판단 방식을 넘어서, 시간 기반 및 상태 의존적인 히스테리시스(Hysteresis) 필터링 기법을 적용하여 센서 노이즈에 강건한(Robust) 상태 검증 로직을 구현하고 있다. 본 절에서는 이런 히스테리시스 처리 기법의 이론적 배경과 PX4 소스 코드 내의 실제 구현 방식을 상세히 분석하라.

1. 히스테리시스(Hysteresis) 필터링 기법의 개요

1.1 단순 임계값 로직의 한계점 (Limitations of Simple Threshold Logic)

단일 임계값(Single Threshold) 방식에서는 측정값이 한계치를 초과하는 즉시 ‘Failed’ 상태로 진입한다. 만약 측정값이 임계값 주변에 걸쳐 미세하게 진동(Fluctuation)한다면, 시스템의 상태는 수 밀리초(ms) 단위로 Valid와 Invalid를 왕복하게 된다. 이는 상태 머신(State Machine)에 심각한 부하를 유발하고 예측 불투명성을 높인다.

1.2 시간 기반 히스테리시스 (Time-based Hysteresis)

PX4-Autopilot은 상태 전환을 결정하기 전에 새로운 상태가 일정 시간 동안 유지되는지 검사하는 시간 지연(Transition Delay) 기반의 히스테리시스 평가를 채택했다. 이는 일시적인 노이즈 임펄스는 무시하고, 실제 물리적 고장이나 편향(Bias)이 지속될 때만 상태를 전환하도록 허용한다.

2. PX4 소스 코드 아키텍처 내 히스테리시스 구현 분석 (Hysteresis Implementation in PX4 Source Code)

PX4 펌웨어에서 히스테리시스 관련 클래스는 주로 src/lib/hysteresis/Hysteresis.h 와 Hysteresis.cpp를 중심으로 구현되어 있다.

2.1 Hysteresis C++ 클래스 구조의 이해

히스테리시스 객체는 변경 지연 시간(Time Delay) 파라미터를 갖는 독립적인 필터링 인터페이스를 제공한다.

  • set_hysteresis_time_from(const bool state, const hrt_abstime time_us)
  • 초기 기준에서 대상 상태(True 또는 False)로 전환될 때 요구되는 대기 시간 설정. 즉, False(에러) 상태에서 True(정상) 상태로 복구되거나, 반대로 넘어갈 때의 지연시간을 비대칭적으로(Asymmetric) 설정할 수 있다.
  • set_state_and_update(const bool state_update, const hrt_abstime now)
  • 센서의 즉각적인 논리 평가 결과를 입력받아, 내장된 타이머 로직과 비교하여 필터링된 최종 상태를 갱신하라.
  • get_state(): 필터링이 완료되어 공식적으로 승인된 안정된(Stable) 상태값을 반환하라.

2.2 Pre-flight Check에서의 활동 시퀀스 메커니즘

다음 다이어그램을 통하여 센서 노이즈가 발생했을 때 Hysteresis 모듈이 이를 어떻게 억제하고 안정화하는지 확인하라.

sequenceDiagram
    participant Sensor as IMU/Mag Sensor
    participant Check as Preflight Check Module
    participant Hyst as Hysteresis Class
    participant State as System State Machine

    Sensor->>Check: Raw Data (Includes Spike)
    Check->>Check: Raw Evaluation > Threshold? (Yes)
    Check->>Hyst: set_state_and_update(false, now)
    note right of Hyst: 상태 변경 임시 기록 (타이머 시작)
    Hyst-->>Check: get_state() -> returns true (Still Valid)
    Check->>State: Report Sensor: OK
    
    Sensor->>Check: Raw Data (Back to normal)
    Check->>Hyst: set_state_and_update(true, now)
    note right of Hyst: 이전 타이머 리셋 및 정상 유지
    Hyst-->>Check: get_state() -> returns true

위 사례에서 센서는 일시적으로 임계값을 벗어났으나, 히스테리시스 타이머가 초과하기 전에 다시 정상 궤도에 진입했으므로 상태 머신은 에러로 전환되지 않았다. 반면, 노이즈가 아닌 진짜 고장일 경우는 타이머 만료 시점에 get_state()에서 에러 상태(false)를 반환하여 기체를 보호하라.

3. 센서 검증에 사용되는 구체적 파라미터(Parameter) 연동

PX4는 GCS인 QGroundControl(QGC)을 통하여 관리할 수 있는 파라미터 시스템에 히스테리시스 기준을 연동시킬 수 있다.
예를 들어, 확장 칼만 필터(EKF2) 기반의 상태 추정기(Estimator) 검증 로직에서는 각 센서의 혁신(Innovation)이 특정 임계값 수준(COM_ARM_IMU_ACC, COM_ARM_MAG_ANG 등)을 넘을 때, 단순 비교가 아니라 내부적으로 히스테리시스 객체를 거친 뒤 최종 Pre-flight 승인을 판단한다.

  • COM_ARM_EKF_POS (Position Innovation 검증): GPS나 비전 센서로부터 유도된 위치가 튀는 현상(Multipath interference 등)에 대비해 Hysteresis 시간을 부여하여 순간적인 신호 불안정에 대한 이륙(Arming) 거부를 회피하라.

4. Ardupilot 대비 아키텍처(Architecture) 및 제어 철학의 차이

Ardupilot의 Pre-flight 평가(예: AP_Arming.cpp) 방식은 각종 서브클래스에서 개별적인 has_error() 형태로 센서 상태 플래그(Flag)나 필터된 에러 카운터를 폴링(Polling)해 온다. 일부 센서 캘리브레이션 모듈은 선형 저역통과 필터(Low Pass Filter) 형태를 응용하지만, 상태 전이의 타이머 딜레이를 객체화된 히스테리시스 공통 라이브러리(Common Library)로 분리해 쓰는 PX4의 접근 방식이 소프트웨어 공학적으로 더욱 구조적이다.

PX4에서는 Hysteresis 객체를 미들웨어 도구로써 모듈 전반에 상속 또는 포함시켜, 센서 에러 판정 외에도 배터리 전압 부족 경고(Low Battery Warning)나 통신 링크 단절(Data Link Loss) 판별 등 여러 시스템 전반의 채터링을 손쉽게 제거할 수 있다는 설계 철학적 강점이 있다.

5. 설계 시 고려점 및 학술적 시사점

Hysteresis의 시간 지연 값을 지나치게 길게(Long Delay) 설정하면 오탐지율(False Positive Rate)은 줄어들지만, 진짜 에러 상태를 감지하고 대처하는 데 걸리는 반응 시간(Reaction Time)이 늦어지게 되며, 이는 **이상행동 수용률(False Negative)**이 반대로 솟아오르는 절충점(Trade-off)을 만든다.

이러한 문제에 관해 《Detection and Isolation of Sensor Faults in Unmanned Aerial Vehicles》 등 다양한 논문(Paper)에서도 센서의 단선/잡음/오프셋(Offset) 등의 에러 특성에 따라 최적의 Hysteresis 윈도우 크기를 실험적으로 도출할 것을 강조하고 있다.

결론적으로, PX4-Autopilot의 Pre-flight Check 시스템에 적용된 히스테리시스 필터링 기법은 자율 비행 로봇 시스템이 실외의 거칠고 예측 불가능한 환경(Harsh Environment)에서 신뢰성 있게 이륙 및 상태 전이를 판단하기 위한 가장 단순하면서도 치명적으로 중요한 1차 안전 메커니즘을 구사하고 있다. 이를 명확하게 연구하고 실제 비행 데이터 로그에 빗대어 평가하라.
