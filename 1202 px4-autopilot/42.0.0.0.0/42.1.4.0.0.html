42.1.4 비행 모드별(수동, 자율, 오프보드) Pre-flight Check 요구 수준의 차등화 및 스케일링(Scaling) 원리

1. 개요

무인항공기 시스템(UAS)의 안전한 시동(Arming)과 비행을 보장하기 위해 수행되는 비행 전 점검(Pre-flight Check)은 고정된 일률적인 기준을 적용하지 않는다. PX4-Autopilot(펌웨어 버전 v1.14 기준)은 비행 모드(Flight Mode)의 종류(수동, 자율, 오프보드)에 따라 시스템이 요구하는 안전 무결성(Safety Integrity)의 수준이 다르다는 점에 착안하여, 센서 유효성(Sensor Validity) 및 상태 추정(State Estimation) 요구 수준을 동적으로 스케일링(Scaling)하는 아키텍처를 채택하고 있다. 이는 제한된 센서 조건에서도 조종사의 책임 하에 비행을 허용하거나, 고도의 자율성이 요구되는 상황에서는 엄격한 하드웨어 검증을 강제하는 유연성을 제공한다.

2. 비행 모드에 따른 Pre-flight Check의 차등화 아키텍처

PX4-Autopilot의 커맨더(Commander) 모듈은 사용자가 요청한 비행 모드에 따라 기체가 만족해야 하는 논리적, 물리적 조건을 다르게 정의한다. QGroundControl(QGC)을 통해 비행 전 점검 결과를 모니터링할 때, 수동 모드에서는 통과되었던 시스템이 자율 모드로의 전환 시에는 에러를 발생시키는 경우가 바로 이 스케일링 원리 때문이다.

3. 수동 모드(Manual Mode)에서의 스케일링 완화

수동 제어 기반의 모드(Stabilized, Acro 등)는 인간 조종사의 개입이 지속적으로 이루어진다. 따라서 PX4-Autopilot은 GNSS의 HDOP(Horizontal Dilution of Precision)나 확장 칼만 필터(EKF2)의 위치 추정 분산(Position Variance)이 일시적으로 허용치(Threshold)를 초과하더라도 시동을 허용한다.
이는 실내 비행이나 GPS 수신 불가 지역에서의 비행을 가능하게 하는 스케일링의 대표적인 예이다. 그러나 기본 자세 추정에 필수적인 자이로스코프(Gyroscope)와 가속도계(Accelerometer)의 캘리브레이션 오차가 임계값을 넘으면 절대 시동을 허용하지 않는다.

4. 자율/오프보드 모드(Autonomous/Offboard Mode)에서의 엄격한 스케일링

Mission(임무) 모드나 Offboard 모드는 조종사의 직접적인 조작 없이 기체가 스스로 경로 계획(Path Planning)을 수행하거나, 외부 컴퓨터(ROS2 Companion Computer 등)의 제어 지시를 받는다.

  1. 내비게이터(Navigator) 인지와 점검: 자율 모드에서는 커맨더 모듈이 내비게이터 모듈의 상태를 추가적으로 검증하라.
  2. 글로벌 포지셔닝 검증: GNSS 3D Fix 판정뿐만 아니라, 위성 개수, 위치 정밀도, EKF2의 속도 및 위치 혁신(Innovation) 수식 \vert y_k - h(\hat{x}_k) \vert 값이 일관된 허용 오차 내에 수렴해야만 Pre-flight Check를 통과한다.
  3. 데이터 링크 무결성: Offboard 모드의 경우 uXRCE-DDS 또는 MAVLink를 통해 들어오는 외부 제어 명령의 주기(Rate)가 일정 시간(일반적으로 0.5초) 이상 단절되지 않았는지를 강력하게 검증하라.

5. Ardupilot과의 아키텍처 비교

Ardupilot의 경우 ARMING_CHECK 파라미터를 비트마스크(Bitmask) 형태로 사용자에게 제공하여, 점검 항목을 정적으로 켜고 끄는 방식이 주를 이룬다. (예: GPS 점검 해제, 나침반 점검 해제 등). 즉, 비행 모드에 따른 동적 차등화보다는 정적인 파라미터 셋핑에 가깝다.
반면 PX4-Autopilot은 COM_ARM_ 계열 파라미터를 통해 기본 기준을 설정하되, 시동을 차단하는 로직 자체를 현재 커맨더(Commander) FSM(Finite State Machine)의 목표 모드에 따라 내부적으로 스케일링(Scaling) 처리하라. 이는 비정상 상황에서 모드 스위치를 변경하는 행위만으로도 시동 가능 여부가 동적으로 계산되는 철학을 바탕으로 한다.

6. 학술적 배경 및 연구 참조

모드별로 안전 무결성 단계를 차등화하는 설계는 “A Hierarchical Mode-Dependent Safety Architecture for Autonomous Unmanned Aerial Vehicles” 와 같은 문헌에서 제안하는 상태 공간 보호(State Space Protection) 개념에 부합한다. 특정 모드에서 제어기가 요구하는 센서의 신뢰도 수준이 다르기 때문에, 관측기(Observer)의 분산값을 모드별로 동적으로 매핑하는 방식은 고신뢰성 시스템의 표준적인 접근이다.

7. 결론

PX4-Autopilot의 Pre-flight Check 요구 수준 스케일링 메커니즘을 이해하라. 드론 운용 중 QGroundControl에서 “Pre-flight Fail“이 발생했다면, 무작정 센서를 재캘리브레이션하기보다 현재 선택된 비행 모드가 요구하는 센서의 엄격한 상한선이 무엇인지 먼저 분석하는 것이 필수적이다. 필요하다면 수동 모드로 변경하여 시동 후 위성 신호가 안정될 때 자율 모드로 전환하는 유연한 운용 시나리오를 구성하라.
