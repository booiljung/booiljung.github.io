42.6.3.1.1. 영차 파지(Zero-Order Hold) 샘플링 주기에 따른 폐루프 제어기(Closed-loop Controller) 위상 여유(Phase Margin) 훼손 검사

디지털 비행 제어 시스템인 PX4-Autopilot은 근본적으로 이산 시간(Discrete-time) 제어기다. 연속 시간(Continuous-time)의 동역학을 가진 무인기 기체를 제어하기 위해, 디지털 시스템은 내부 연산 결과를 액추에이터(모터, 서보 등)로 보낼 때 다음 샘플링 주기 전까지 그 제어값을 일정하게 유지(Hold)해야 한다. 이 과정을 수학적으로 **영차 파지(Zero-Order Hold, ZOH)**라 부른다. ZOH는 이상적인 임펄스 샘플링을 물리적 연속 신호로 재구성하는 필수 장치이지만, 필연적으로 시간 지연(Time Delay)을 유발하여 폐루프(Closed-loop) 제어기의 위상 여유(Phase Margin)를 갉아먹는 주요 원인이 된다. 본 절에서는 ZOH 샘플링 주기가 위상 여유에 미치는 악영향을 파악하고, PX4 아키텍처 관점에서의 대응 전략을 살펴본다. (기준 버전: PX4-Autopilot v1.14 이상, QGroundControl v4.3 이상)

1. 영차 파지(Zero-Order Hold, ZOH)의 수학적 모델링

이산 시간 제어기에서 연산된 제어 신호 u[k]는 ZOH 펄스 전달 함수(Pulse Transfer Function)를 거쳐 연속 신호 u(t)로 변환되어 플랜트(Plant, 무인기 동역학)에 인가된다. ZOH의 전달 함수 G_{ZOH}(s)는 라플라스 변환(Laplace Transform) 영역에서 다음과 같이 표현된다.

G_{ZOH}(s) = \frac{1 - e^{-T_s s}}{s}

여기서 T_s는 제어기의 샘플링 주기(Sampling Period)이다. 주파수 영역(s = j\omega)에서 ZOH의 주파수 응답을 분석하면, 크기(Magnitude)와 위상(Phase)은 다음과 같이 도출된다.

  1. 크기 응답: |G_{ZOH}(j\omega)| = T_s \left| \frac{\sin(\omega T_s / 2)}{\omega T_s / 2} \right|
  2. 위상 응답: \angle G_{ZOH}(j\omega) = -\frac{\omega T_s}{2} \quad \text{(단, } \omega < 2\pi/T_s \text{ 조건)}

위상 응답 수식에서 명확히 드러나듯, ZOH는 주파수 \omega에 비례하여 정확히 **-T_s / 2의 시간 지연(Time Delay)**과 동등한 수준의 순수 위상 지연(Phase Lag)을 생성한다.

2. 위상 여유(Phase Margin) 훼손의 역학

폐루프 제어(Closed-loop Control) 이론에서 위상 여유(Phase Margin)는 시스템이 불안정해지기 전까지 감내할 수 있는 여분의 위상 각도를 의미한다. 멀티로터의 자세 제어기(Attitude Controller)와 같이 고응답속도를 요구하는 루프에서는 약 45^\circ \sim 60^\circ 수준의 위상 여유 확보가 권장된다.

그러나 앞서 도출된 위상 지연 수식 \angle G_{ZOH}(j\omega) = -\frac{\omega T_s}{2} 에 의해, 샘플링 주기 T_s가 길어질수록(즉, 루프 속도가 느려질수록) 시스템의 대역폭(Bandwidth) 주변에서 위상 여유가 급격하게 침식(Erosion)된다. 위상 여유가 줄어들면 기체 응답이 과도하게 오버슈트(Overshoot)하거나, 심하면 발진(Oscillation)하여 제어 불능 상태에 빠지게 된다.

3. PX4-Autopilot의 폐루프 최적화 및 ZOH 대응

PX4는 광범위한 하드웨어 호환성(Pixhawk 표준 등) 위에서 ZOH로 인한 위상 마진 훼손을 최소화하기 위해 루프 레이트(Loop Rate)를 탄력적으로 운영한다.

  1. D-Shot 및 고속 프로토콜 활용: ZOH 시간 T_s를 줄이기 위해, PX4는 최신 ESC와의 통신에서 D-Shot Protocol을 지원한다. 기존 PWM 통신(약 400Hz) 대비 D-Shot 600(30kHz 이상 패킷 전송, 실질 제어 1~8kHz 연동)을 사용하면 모터 커맨드의 홀딩 타임(Holding Time)이 비약적으로 단축되어 위상 지연이 대폭 감소한다.
  2. 자세 제어(Attitude Loop)의 고속화: 외부 위치 제어(Position Control)는 50Hz 단위로 느리게 동작해도 무방하지만, 내부 자세 루프(Inner Rate Loop)는 IMU 동기화 주기에 맞물려 400Hz ~ 800Hz(IMU_GYRO_RATEMAX 파라미터 조율)로 펌프질한다. 이를 통해 \omega_{co} (교차 주파수) 대역에서의 ZOH 위상 지연을 1^\circ \sim 2^\circ 이내로 억제한다.

4. Ardupilot 대비 관제 및 제어 구조적 차이

두 오픈소스 시스템 모두 기본적으로 PID를 활용한 직렬 루프(Cascaded Loop) 제어 아키텍처를 가지지만, ZOH 지연 처리 및 프로세스 스케줄링에서 차별점을 지닌다.

  • PX4-Autopilot: NuttX RTOS 기반의 이벤트 드리븐(Event-driven) Publish/Subscribe(uORB) 아키텍처를 채택하였다. IMU 센서 데이터가 수신되는 즉시(Event) 최단 경로로 Rate 제어 모듈이 활성화되므로 프로세스 간 대기 시간(Jitter)이 적다. 따라서 ZOH 본연의 지연 T_s / 2 이외의 불필요한 시스템 지연(System Latency) 추가분이 상대적으로 억제되어 위상 여유 방어에 유리하다.
  • Ardupilot: ChibiOS 기반의 고정 타임 슬라이스(Fixed Time Slice) 및 스레드 기반 스케줄링을 운영한다. 루프(예: Fast Loop 400Hz)가 엄격히 고정된 타이밍에 실행되므로, ZOH 주기 T_s가 철저하게 일정하게 유지된다. 이는 제어 주파수의 결정론적(Deterministic) 보장에는 우위를 주지만, 센서 샘플링과 루프 타이밍 간의 미세한 위상차(Phase Shift) 처리는 필터 로직으로 보상해야 한다.

5. 실무 적용: QGroundControl을 이용한 위상 여유 확보 가이드

GCS(QGroundControl)를 통해 새로운 기체의 PID 튜닝을 수행할 때, ZOH로 인한 위상 마진 축소가 현상으로 나타나면(예: D-Gain 증가 시 급격한 진동) 다음 절차를 검토하라.

  1. 대역폭(Bandwidth) 타협: 기체가 무겁거가 프레임 유연성이 커 위상 곡선이 빠르게 하락한다면, 비현실적인 고속 응답을 요구하기보다 P-Gain(MC_ROLLRATE_P, MC_PITCHRATE_P)을 줄여 교차 주파수(Crossover Frequency)를 낮추어라. 교차 주파수가 낮아질수록 ZOH로 인한 위상 지연 각도(\omega T_s / 2)가 작아져 위상 여유가 확보된다.
  2. 액추에이터 출력 속도 최적화: ESC가 지원한다면 PWM 대신 DShot 혹은 OneShot125 프로토콜로 PWM_RATE (또는 DShot 파라미터 DSHOT_CONFIG)를 최대 설정값으로 상향 조정하라. 제어기 연산 주기가 400Hz이더라도, 비동기식 PWM 출력 환경에서는 ZOH가 엇박자로 늘어나 위상을 갉아먹는다. 디지털 프로토콜 도입은 하드웨어 단의 ZOH 주기를 소거하는 가장 직접적인 방어 수단이다.

결과적으로, ZOH에 의한 위상 지연은 디지털 비행 제어기 설계의 피할 수 없는 ’숨은 그림자’이나, PX4 내부의 실시간 운영 체제(RTOS) 최적화와 ESC 프로토콜 선진화를 통해 시스템 한계를 한 차원 높게 끌어올릴 수 있다.
