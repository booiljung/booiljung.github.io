42.6.3.1.1 영차 파지(Zero-Order Hold) 샘플링 주기에 따른 폐루프 제어기(Closed-loop Controller) 위상 여유(Phase Margin) 훼손 검사

디지털 제어 시스템인 PX4-Autopilot에서 연속 시간(Continuous-time) 제어 플랜트는 마이크로컨트롤러(MCU) 내부에서 이산 시간(Discrete-time) 제어기로 구현된다. 이 과정에서 아날로그 신호를 샘플링하고 일정 시간 유지하는 영차 파지(Zero-Order Hold, ZOH) 요소가 필연적으로 개입된다. 본 절에서는 ZOH 샘플링 주기가 폐루프 제어기의 위상 여유(Phase Margin)에 미치는 악영향을 분석하고, 이것이 드론의 비행 안정성에 어떤 영향을 미치는지, 그리고 PX4에서는 이를 어떻게 검증하고 보상하는지 상세히 다룬다. (기준 버전: PX4-Autopilot v1.14.0)

1. 영차 파지(Zero-Order Hold)의 수학적 모델링과 위상 지연(Phase Delay)

영차 파지는 이산 샘플링된 제어 신호를 다음 샘플링 순간까지 일정한 값으로 유지(Hold)하여 계단형 연속 신호로 변환하는 장치이다. 이산 시간 시스템과 연속 시간 플랜트(예: 멀티로터의 동역학 모델)를 연결하는 핵심적인 인터페이스 역할을 수행한다.

수학적으로 ZOH의 전달 함수 G_{ZOH}(s)는 다음과 같이 정의된다.

G_{ZOH}(s) = \frac{1 - e^{-T_s s}}{s}

여기서 T_s는 제어기의 샘플링 주기(Sampling Period)이다. 주파수 영역(s = j\omega)에서 ZOH의 주파수 응답을 분석하면 매그니튜드(Magnitude) 특성과 위상(Phase) 특성을 도출할 수 있다. 해석을 단순화하기 위해 위상각(Phase Angle) \angle G_{ZOH}(j\omega)을 살펴보면 다음과 같이 근사된다.

\angle G_{ZOH}(j\omega) \approx -\frac{\omega T_s}{2}

이 수식은 ZOH 블록이 제어 루프 내에서 주파수 \omega에 비례하여 시스템에 순수 시간 지연(Pure Time Delay) 효과를 부여함을 의미한다. 지연되는 시간의 크기는 샘플링 주기의 절반(T_s / 2)에 해당한다.

2. 위상 여유(Phase Margin)의 훼손 및 비행 제어기 불안정성

위상 여유(Phase Margin, PM)는 개루프(Open-loop) 주파수 응답이 0dB 교차점(Gain Crossover Frequency)에서 발생하는 위상이 -180도보다 얼마나 떨어져 있는지를 나타내는 지표로, 폐루프 시스템의 상대적 안정성(Relative Stability)을 결정짓는 핵심 요소이다. 일반적으로 PX4와 같은 고기동 드론을 제어할 때는 최소 45도에서 60도 수준의 견고한 위상 여유를 확보하도록 PID 이득(Gain)을 튜닝한다.

그러나 앞서 수학 모델에서 보았듯, ZOH로 인한 위상 지연 효과는 고주파수 영역으로 갈수록 극심해진다. 만약 내부 제어 루프(Inner Loop: 각속도 제어 등)의 대역폭(Bandwidth)이 넓어져 교차 주파수가 높아지거나, 샘플링 주기 T_s가 길어지면 다음과 같은 문제가 연속적으로 연쇄 작용을 일으킨다.

  1. 위상 하강(Phase Drop): ZOH로 인해 개루프 응답의 전체적인 위상 곡선이 아래로 이동.
  2. 위상 여유 감소: 목표로 했던 45~60도의 위상 여유가 확보되지 못하고 감소 (예: 20도 이하).
  3. 오버슈트(Overshoot) 및 진동 증폭: 계단 응답(Step Response)에서 오버슈트가 현저하게 발생하며 감쇠(Damping)가 부족해짐.
  4. 발산(Divergence): 위상 여유가 0도에 도달하거나 음수가 되면 제어 시스템이 발산하여 멀티로터가 추락함.

3. PX4-Autopilot 아키텍처 내 ZOH 영향 관리 기법

PX4는 비행 역학(Flight Dynamics)의 높은 주파수 특성으로 인해 발생하는 ZOH에 의한 위상 지연을 최소화하기 위해 다중 루프 제어 주기 분리 및 실시간 스케줄링 기법을 적용한다.

3.1 Rate Controller의 고주파 스케줄링

ZOH에 의한 위상 지연 -\frac{\omega T_s}{2}를 줄이기 위한 가장 직관적이고 물리적인 접근법은 샘플링 주기 T_s를 극한으로 감소시키는 것이다.

  • Rate Control (각속도 제어 루프): PX4의 mc_rate_control 모듈은 내부 체계를 IMU 센서 데이터가 수신되는 즉시 동작하도록 구성하여, 하드웨어가 지원하는 가장 높은 주파수(루프 타임: 400Hz ~ 1000Hz, 즉 T_s = 2.5ms \sim 1ms)에서 작동한다. 이로 인해 ZOH 시간 지연은 1ms 이하로 제한되어 Rate 루프 대역폭 내에서의 위상 여유 훼손을 최소화할 수 있다.
  • Attitude & Position Control (자세 및 위치 제어 루프): 기구학적 응답이 느린 위치(Position)의 경우, 교차 주파수 \omega가 상대적으로 낮으므로 ZOH의 위상 지연 효과가 크지 않다. 따라서 이들 루프는 50Hz ~ 100Hz 수준에서 운영하여 CPU 리소스를 절약하는 구조를 갖는다.

3.2 제어기 이득 연산(Tuning) 시의 여유 각 보수적 확보

Ardupilot은 PID 튜닝 파라미터를 제공할 때 필터 이득과 Rate 제어기 이득을 다소 보수적으로 잡는 경향이 있어 ZOH로 인한 지연 마진 폭이 넓다. 반대로 PX4는 극한의 응답성을 중시하는 아키텍처 특성 상, Auto-tuning (System Identification 기능) 모듈 사용 시 ZOH 효과와 소프트웨어 저주파 통과 필터(LPF)에 의한 위상 지연까지 총체적인 지연 모델(Total Time Delay T_d)에 포함시켜 계산한다.

T_d = T_{sensor} + T_{filter} + \frac{T_s}{2} + T_{actuator}

튜닝 알고리즘은 총 루프 딜레이 T_d를 기반으로 식별된 드론의 플랜트 모델 상에서 안정성을 확보할 수 있도록 비례 제어(P) 및 미분 제어(D) 이득 설정 상한을 결정한다.

4. GCS를 통한 사전 및 사후 검증(Monitoring) 체계

QGroundControl(QGC) 또는 MAVLink 프로토콜을 통과하는 시스템은 ZOH 관련 지표를 명확히 보여주는 GUI 필드를 제공하지 않지만, 시스템 관리자는 Flight Review 등을 통해 간접적으로 그 악영향을 모니터링한다.

  • 액추에이터 활성도 모니터링: 만약 프로세서(MCU)의 부하율이 상승하여 RTOS 기반 주기 Task 스케줄링 딜레이가 발생하면, 실효 샘플링 주기(T_s)가 늘어나는 스파이크 현상(Jitter)이 발생한다.
  • 이때 위상 여유가 수시로 훼손되어 모터 출력인 actuator_outputs 토픽에 미세한 채터링(Chattering)과 진동이 기록된다. PX4 개발자나 GCS 운용자는 비행 전 CPU 로드율(CPU Load)과 vehicle_rates_setpoint 대비 vehicle_angular_velocity의 오버슈트량을 점검함으로써, ZOH로 유발되는 잠재적 위상 불안정성을 사전에 파악할 수 있다.
