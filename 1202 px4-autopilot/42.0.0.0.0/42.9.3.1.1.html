42.9.3.1.1 텔레메트리 병목 현상(Bottleneck) 환경에서의 상태 업데이트 주파수 분해능(Frequency Resolution) 손실 보상 기법 비교

무인항공기(UAV) 운영 환경에서 원거리 비행이나 전파 간섭이 심한 구역에서는 지상 관제 시스템(GCS)과 자동비행장치(Autopilot)를 잇는 텔레메트리(예: 433MHz/915MHz SiK Radio)의 대역폭 한계로 인해 데이터 병목 현상(Data Bottleneck)이 잦게 발생한다. 이로 인한 MAVLink 패킷 드롭(Drop) 또는 지연은 기체 상태 업데이트의 주파수 분해능(Frequency Resolution) 저하를 초래한다. 본 절에서는 대역폭 포화 상태에서 PX4-Autopilot(기준: PX4 v1.14)과 Ardupilot이 텔레메트리 스트림 손실을 방어하고 상태 동기화를 보상하는 기법의 차이를 분석한다.

1. 텔레메트리 병목 현상과 주파수 분해능 손실의 이해

GCS(예: QGroundControl) 화면에 렌더링되는 기체의 3D 자세나 위치, 속도 정보는 일반적으로 10\text{Hz}에서 50\text{Hz} 수준의 MAVLink 스트림(ATTITUDE, GLOBAL_POSITION_INT 등)에 의존한다. 데이터 링크가 포화(Saturated)되면 비행 제어기 측 MAVLink 송신 버퍼가 꽉 차게 되어 패킷 드롭이 발생하며, 이는 GCS 관점에서 상태 업데이트 주파수 분해능이 2\text{Hz} 이하로 뚝 떨어지는 현상으로 이어진다. 이로 인해 관제자는 기체가 끊기면서 이동(Stuttering)하는 것으로 인식하게 된다.

2. PX4-Autopilot의 동적 대역폭 할당 및 이벤트 기반 보상 기법

PX4의 MAVLink 모듈(src/modules/mavlink)은 고정된 송신 주기를 맹목적으로 고집하지 않고, 통신 채널의 상태를 능동적으로 파악하는 적응형 레이트 컨트롤(Adaptive Rate Control)을 적용한다.

2.1 토큰 버킷(Token Bucket) 기반 대역폭 스로틀링(Throttling)

PX4는 각 MAVLink 채널에 할당된 최대 대역폭(MAV_0_RATE 파라미터 등)을 기준으로 토큰 버킷 알고리즘을 사용한다.

  • 하드웨어 시리얼 버퍼(Serial Buffer)가 일정 수준 이상 차오르면, PX4는 하위 우선순위 메시지(예: 디버깅 텍스트, 사용자 지정 센서 값)의 전송을 즉각적으로 생략(Skip/Drop)한다.
  • 생략된 대역폭은 HEARTBEAT, SYS_STATUS, ATTITUDE와 같이 관제에 치명적인(Critical) 고우선순위 메시지에 몰아주어 상태 분해능의 핵심 뼈대를 유지한다.

2.2 비동기 이벤트(Events) 인터페이스 기반 주요 상태 보장

기존의 지속적 스트리밍 방식과 별개로, PX4는 비행 모드 변경이나 치명적 에러 발생 등의 극단적인 상태 변경 시 uORB 이벤트 토픽을 통해 최우선순위 MAVLink 메세지로 즉시 밀어넣는다(Push). 이는 스트리밍 분해능이 떨어지더라도 State Machine 자체의 동기화 오버헤드는 0(Zero)에 가깝게 보장하려는 목적이다.

3. Ardupilot의 트래픽 쉐이핑(Traffic Shaping) 및 Stream-Rate 기반 접근

Ardupilot 역시 텔레메트리 병목을 막기 위한 정교한 버퍼링 스케줄러를 보유하고 있으나, 스트림 제어의 아키텍처적 관점이 다소 다르다.

3.1 SRn_* 파라미터 기반의 정적 할당 및 라운드 로빈(Round Robin)

Ardupilot은 SR1_EXT_STAT, SR1_EXTRA1 (예: 자세 데이터)과 같이 메시지 그룹을 세분화하여 각 채널별 타겟 주파수를 정의한다.

  • 데이터 링크가 포화될 경우, 스케줄러는 단순히 큐(Queue)에 자리가 날 때까지 메세지 송신을 지연시키며, 틱(Tick)마다 그룹별로 라운드 로빈 방식으로 전송 기회를 부여한다.
  • 이 방식은 특정 메시지만 극단적으로 누락되는 것을 방지하여 ’데이터의 균형(Balance)’을 유지하지만, 반대로 전체적인 주파수 분해능이 고르게 하락하는(Uniform Degradation) 부작용을 낳기도 한다.

4. GCS (QGroundControl) 측면의 분해능 보상 메커니즘

기체에서 송신하는 패킷이 줄어들더라도, GCS 뷰(View)의 부자연스러움을 해소하기 위해 QGroundControl은 자체적인 보상 기법을 사용한다.

  • 추측 항법(Dead Reckoning) 렌더링: 수신된 속도 벡터(GLOBAL_POSITION_INT의 V_x, V_y, V_z)와 자이로(Gyro) 정보가 끊길 경우, 이전 수신 데이터를 기반으로 현재의 위치와 자세를 선형 보간(Linear Interpolation) 또는 외삽(Extrapolation)하여 UI 렌더링 주사율(프레임 레이트, 60fps)을 부드럽게 유지한다.
  • 실제 MAVLink 업데이트 틱과의 매칭: 기체의 상태 데이터가 너무 오래 지연(전형적으로 1.5\text{초} 이상)되면, 외삽을 중단하고 즉시 통신 불량(Communication Lost) 타임아웃 처리를 발동한다.
