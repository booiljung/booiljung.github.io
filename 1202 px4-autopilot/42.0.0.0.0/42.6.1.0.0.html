42.6.1. 무선 조종(RC) 신호의 수신 품질 및 패킷 손실률(Packet Loss Rate) 평가

무인기 시스템에 있어 최후의 보루인 수동 개입(Manual Override)을 보장하기 위해서는, 조종기(Radio Control, RC) 송신기와 기체 수신기(Receiver) 사이의 무선 물리 계층(Physical Layer)이 안정적으로 유지되어야 한다. 비행 전 점검(Pre-flight Check) 단계 및 실시간 운용 중 PX4-Autopilot 아키텍처는 단순한 장치 연결(Connected) 유무를 판단하는 수준을 넘어, 다중 경로 페이딩(Multipath Fading)이나 전자기 간섭(EMI) 환경에서도 정상적인 스틱 제어가 가능한 수준인지를 통계적으로 진단한다. 본 절에서는 수신 신호 강도만을 평가하는 기존의 한계를 보완하기 위해 도입된 패킷 손실률(Packet Loss Rate) 평가 알고리즘과 관련 시스템 소프트웨어 아키텍처를 심층 분석한다.

1. 수신 신호 강도 지시자(RSSI)의 한계와 평가 패러다임 전환

전통적으로 항공 통신 링크 상태는 수신 신호 강도 지시자(RSSI, Received Signal Strength Indicator)를 기반으로 평가되었다. 이는 수신 안테나 디바이스에 유도되는 RF 신호의 아날로그 전압 레벨을 데시벨 밀리와트(dBm) 단위나 0~100% 비율로 정규화한 값이다. 그러나 아날로그적 척도인 RSSI 수치는 도심 환경, 폐쇄 공간 구역 등과 같이 신호 반사 및 굴절이 심한 공간에서는 간섭 노이즈 파워(Noise Power)까지 합산되어 수신 강도가 높게 표출되는 가짜 강건성(False Robustness) 현상을 초래하는 근본적 허점을 지닌다.

따라서 현대의 PX4-Autopilot 아키텍처는 RSSI와 더불어, 디지털 프레임 기반의 ‘패킷 손실률(Packet Loss Rate, PLR)’ 또는 ‘링크 품질 지수(Link Quality Indicator, LQI)’ 메트릭을 교차 검증하는 하이브리드(Hybrid) 진단 패러다임을 도입하고 있다.

2. 프로토콜별 통신 품질 메트릭 추출 구조 메커니즘

PX4 운영체제 하단 시스템은 MCU의 직렬 통신(UART/SPI) 버스 하드웨어 드라이버 모듈을 통해 조종기 수신기의 프레임을 실시간 디코딩한다. 지원하는 수신기 텔레메트리 프로토콜 스펙에 따라 링크 품질 정보를 획득하는 소스 코드 라우팅 체계가 구별되도록 설계되었다.

  1. SBUS (Futaba/FrSky 계열): 프로토콜 프레임 헤더 내에 명시적인 통신 품질(RSSI 등) 필드가 할당되지 않은 경우가 많다. 이 경우 프레임 디코더 내부에 위치한 페일세이프 비트(Failsafe Bit) 구조 및 프레임 손실 비트(Frame Lost Bit)를 기반으로 PX4 소프트웨어 단에서 통계적 PLR을 역산으로 판별해낸다. 일부 상용 수신기는 특정 가상 채널(예: Channel 16)에 PWM 출력 형태로 RSSI를 담아 보내는데, 이 경우 PX4 파라미터 인터페이스 RC_RSSI_PWM_CHAN을 통해 소프트웨어적으로 매핑할 수 있다.
  2. CRSF (TBS Crossfire / ExpressLRS 계열): 양방향 초저지연 프레임워크인 CRSF 체계는 명시적으로 Link Statistics 패킷 데이터(0x14 프레임 ID)를 네이티브 수준에서 지원한다. 여기에는 SNR(신호 대 잡음비), 활성 안테나 인덱스 상태(Antenna Diversity), 그리고 업링크 통신 품질 지수(Uplink Link Quality)가 정밀하게 포함된다. PX4 커스텀 드라이버는 이를 직접 바이너리 파싱하여 코어 input_rc uORB 토픽 버스에 고해상도로 갱신한다.

3. 패킷 손실률(Packet Loss Rate) 도출 알고리즘 및 구현 분석

명시적인 자체 LQI 패킷을 송출하지 못하는 범용 저가형 수신기를 모두 고려하여, PX4 디바이스 디코더 드라이버 계층은 일정 시간 윈도우(Time Window) 블록 동안 주기적으로 정상 디코딩된 프레임 빈도 수와 타이머 기반 예상 프레임 수를 비교하여 자체적인 패킷 손실률 모델을 도출해낸다. 다음은 관련 데이터 처리 로직의 아키텍처 실행 흐름이다.

// src/drivers/rc_input/RCInput.cpp 의 로직 의사 코드(Pseudo-code) 구조 분석
void RCInput::calculate_rc_link_quality() {
    const hrt_abstime now = hrt_absolute_time();
    
    // 타임스탬프 분석을 통한 프레임 주기 시간 간격(Delta t) 계산
    float dt = (now - _last_rc_timestamp) / 1e6f; 
    
    // 예상 프레임 통신 빈도 대비 실제 통신 디코딩 완료 프레임 카운트 산정
    uint32_t expected_frames = expected_hz * time_window;
    uint32_t lost_frames = expected_frames - _valid_frames_count;

    // 순간적인 지터(Jitter) 혹은 드롭으로 인한 수치 급격한 뜀(Spike)을 방지하기 위한 
    // 저역 통과 필터(Low-pass Filter) 연산 적용
    float instantaneous_loss = (float)lost_frames / expected_frames;
    _rc_packet_loss_rate = (0.9f * _rc_packet_loss_rate) + (0.1f * instantaneous_loss);
    
    // 집계 윈도우 카운터 초기화
    _valid_frames_count = 0;
}

제시된 통계적 알고리즘을 통해 계산된 _rc_packet_loss_rate (수학적 0.0 ~ 1.0 비율 범위) 구조체는 RSSI 파생 정보와 병합 파싱되어 input_rc 메시지 페이로드에 담기며, 최상위 Commander의 시스템 상태 검사 파이프라인 연산기 요소로 넘겨진다.

4. Ardupilot 대비 통신 아키텍처 및 철학적 비중 차이

다중 채널 생존성 측면에서 PX4-Autopilot은 이중화 RC 다이버시티(Receiver Diversity, 수신기 2개 하드웨어 동시 연결) 설정 시, 두 수신기 포트 버스에서 병렬적으로 올라오는 스트림 중 패킷 손실률(PLR) 및 타임스탬프 최신성을 비교 연산하여 가장 양호한 라우팅 데이터 채널을 실시간 우선 스위칭하는 동적 지능(Dynamic Switching) 메커니즘을 내장하고 있다. 이는 통신 자원의 유연성과 플랫폼 범용성을 중시하는 PX4의 핵심 특성이다.

반면, Ardupilot 체계 시스템은 무인기 제어의 전통적으로 물리 라인 기반의 다중 수신기 인터페이스 입력에 대한 강력한 하드웨어 페일오버(Failover) 스위치 철학을 고수하며, RC 직렬 수신 프로세스(AP_RCProtocol 모듈 객체) 단에서 더욱 보수적인 디바운싱(Debouncing) 타임을 할당한다. 산출된 수신 품질 데이터를 활용하는 정책 결정을 내릴 때, PX4는 commander 단의 고수준 상태 추론기(High-Level Estimator)에서 이를 통계적으로 부드럽게 필터링하여 판단 모체의 재료로 쓰는 반면, Ardupilot은 디바이스 드라이버 최하단 계층이 즉각적으로 Muxing을 통제 스위칭하고 상위로 제어 우선 권한의 결과를 통보하여 이양하는 직접적인 하드웨어 인터럽트 성격의 비중이 상대적으로 훨씬 높고 직렬적이다.

5. 지상 관제망(QGroundControl) 모니터링 체계 및 Failsafe 연동 동작

비행 전 상태 인지 단계에서 도출된 패킷 손실률 수치가 통상 허용 임계 밴드인 일정 수준(일반적으로 커스텀 모델별 15%~30% 범위로 상이함)을 초과할 경우, 시스템은 조종사 조타 입력 지연의 불확실성 위험 인가 진입을 방지하기 위해 Pre-flight 기본 요건 불충족 판단 및 이륙(Arming)을 원천적으로 거부한다.

GCS(QGroundControl) 애플리케이션의 메인 HUD 툴바 상단 패널에는 RC 수신 상태 표시 아이콘이 실시간으로 전시되는데, 이는 단순 RSSI가 아닌 패킷 손실률과 안테나 정보가 정규식 기반으로 결합된 신뢰도 매트릭스를 시각화하여 운영자에게 전달한다.

자율 비행 혹은 수동 계기 비행 운용 중 해당 손실률이 급주행하여 유효 프레임이 파라미터 COM_RC_LOSS_T (시스템 기본값 0.5초 대기) 윈도우 타임 동안 완전히 도달하지 않는다면, FC는 시스템 제어기를 즉각 RC Loss Failure 위력 상태 공간으로 천이시킨다. 이후 시스템 제어권은 사전에 할당된 NAV_RCL_ACT 파라미터 정책 프로토콜에 의거하여 다음과 같이 결단 수행 체인에 진입한다.

  • RTL (Return to Launch): 기체가 비행을 자체 중단 후 홈 회귀, 고도 하강 및 자동 착륙 단계 진입. (대부분의 실외 공간 비행 시스템 기본 권장값 설정)
  • Hold / Loiter: 제자리 공중 체공(Hovering)하며 일시적인 물리적 전파 음영 지대 환경의 이탈을 대기.
  • Terminate/Land: 통제 불능 위험 구역 진입 조짐으로 판단 시 대지 즉각 자동 착륙 단계를 밟거나 극단적일 경우 모터 구동 모듈 정지 명령 투사.

학술 시스템 설계 관점에서, 이러한 패킷 손실률 기반 수신 채널 품질(Channel Quality) 진단 평가는 무전통신 시절의 전통적인 지속파(Continuous Wave) 아날로그 수신 강도 측정 단일 기법의 한계를 일탈하여, 고속 디지털 비트 에러 정정 및 타이밍 동기화 체결 확률을 종합 반영하므로 차세대 자율 비행 안전 공간 추정의 수학적 시스템 신뢰도를 획기적으로 향상시키는 공학적 기여를 지닌다.
