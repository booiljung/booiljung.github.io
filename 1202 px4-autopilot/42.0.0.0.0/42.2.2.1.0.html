42.2.2.1 마르코프 결정 과정(Markov Decision Process) 관점에서의 상태 전이 확률 제어

1. 개요

복잡한 비행 환경에서 기체의 센서들은 완벽한 정보를 제공하지 못한다. 센서 노이즈, 기상 악화, 일시적인 데이터 링크 단절 등 불확실성(Uncertainty)이 만연한 상황에서, 단순히 참(True)과 거짓(False)의 결정론적 로직(Deterministic Logic)만으로 유한 상태 머신(FSM)의 전이를 통제하는 것은 비합리적일 수 있다. 본 절에서는 PX4-Autopilot의 FSM 내부 평가 로직을 **마르코프 결정 과정(Markov Decision Process, MDP)**이라는 학술적 관점에서 재해석하고, 상태 전이 과정에서 센서의 불확실성이 어떻게 확률적으로 제어되는지 심층적으로 분석한다.

2. 마르코프 결정 과정(MDP) 개요와 PX4 매핑

MDP는 의사결정을 모델링하는 수학적 틀로, 튜플 (S, A, P_a, R_a)로 정의된다.

  • S (상태 공간): PX4의 Navigation State 및 Arming State (예: AUTO_MISSION, RTL, FAILSAFE)
  • A (행동 집합): 전환 시도, 에러 무시, 페일세이프 트리거 등 커맨더(Commander)의 행동
  • P_a(s, s') (전이 확률): 현재 상태 s에서 행동 a를 취했을 때 s'로 전이될 확률
  • R_a(s, s') (보상 함수): 안전 비행 지속 시 (+) 보상, 추락 시 핵심적인 (-) 보상

PX4 커맨더는 명시적으로 “확률 계산“을 코드로 수행하지는 않는다. 그러나 확장 칼만 필터(EKF2)를 통해 센서 오차 공분산(Covariance) 및 추정 분산치(Variance)를 산출하고, 이 신뢰도(Confidence) 지표가 임계값(Threshold)을 넘는가에 따라 상태 전환 조건을 허가한다. 이는 곧 MDP에서의 상태 전이 확률의 조건부 필터링과 수학적으로 동일한 맥락이다.

3. 상태 전이 확률 제어 아키텍처

기체가 LOITER 모드에서 AUTO_RTL(복귀) 모드로 전이된다고 가정하자. 이 전이는 GPS 위치 데이터의 신뢰도에 극도로 의존한다.

  1. 상태 관측 (Observation): EKF2는 GPS, IMU, 기압계의 센서 퓨전을 통해 현재 궤적(State Estimate)과 함께 위치 혁신 분산(Position Innovation Variance)을 출력한다.
  2. 확률적 전이 조건 (Probabilistic Guarding): 커맨더 모듈 내의 FSM 검증 루프는 EKF가 출력하는 ekf2_innovations uORB 토픽을 수신하라. 만약 테스트율(Test Ratio, 설계 오차 대비 실제 혁신의 비율)이 1.0을 초과하면, 이는 위치 측정값이 오염되었을 확률이 매우 높음을 의미한다.
  3. 전이 제어 (Transition Control): 에러 확률이 임계값을 상회하면, PX4는 AUTO_RTL로의 상태 전이를 거부하고, ALT_HOLD나 가장 최선의 페일세이프(Failsafe) 상태를 보상 함수 상 최적의 결정으로 간주하여 대체 전이(Alternative Transition)를 강제한다.
graph TD
    A[현재 상태 S: LOITER] --> B{외부 명령: RTL 요청}
    
    B --> C[EKF2 상태 추정치 및 Test Ratio 수신]
    
    C --> D{P(위치 신뢰도 > 임계값)}
    
    D -- 높은 확률 (Ratio < 1.0) --> E[RTL 상태 전이 승인 S']
    D -- 낮은 확률 (Ratio >= 1.0) --> F[MDP 최적 정책: 궤적 포기 및 생존 우선]
    
    F --> G[상태 전이 거부 및 Altitude Hold로 강제 전이 S'']

4. 소스 코드 분석 예시 (혁신 기반 전이 통제)

EKF2가 제공하는 추정 상태의 신뢰성을 상태 전이 조건에 연동시키는 로직은 PX4의 안전 설계 핵심이다.

// 예시: PX4 Commander의 EKF 상태 연동 상태 전이 검증 (추상화)
bool is_transition_safe(const vehicle_status_s &status, const estimator_status_s &ekf_status)
{
    // EKF 혁신(Innovation) 테스트 비율 검사 (마르코프 관점의 확률적 임계치)
    float pos_test_ratio = ekf_status.vel_pos_innov_var[3]; // 위치 혁신 비율

    // 확정적이지 않은 상태에 대한 확률 가드 (Guard)
    if (pos_test_ratio > 1.0f) {
        // 위치 추정의 신뢰 확률이 낮아 자율 비행 모드로의 전이를 불허함
        return false; 
    }

    return true;
}

5. Ardupilot과 확률 제어 철학의 차이

Ardupilot은 EKF3를 기본으로 사용하며, 분산이 커질 경우 DCM(Direction Cosine Matrix) 백업이나 Lane Switch(중복 필터 교체) 로직을 통해 상태(State) 추정기를 통째로 스왑하는 강력한 생존 전환 체계를 갖추고 있다. 즉, Ardupilot은 “에러 매트릭스 발생 시 코어 필터 변경“이라는 확률적 선택 정책(Policy)을 중시한다.

반면, PX4-Autopilot 시스템은 필터 스위칭을 최후의 수단으로 간주하며, 커맨더(FSM) 레이어에서 EKF의 테스트 비율(Test Ratio) 파라미터(EKF2_REQ_EPH, EKF2_REQ_EPV 등)를 통해 상위 비행 모드 전이를 선제적으로 금지하는 안전 스태킹(Safety Stacking) 방식을 택한다. 이는 상태 공간 내에서의 전이 통제를 통해 기동 한계를 제어하는 전형적인 MDP 최적화 관점의 설계이다.

6. 결론

PX4의 상태 전이 루프는 절대적인 불리언(Boolean) 검사에 그치지 않고, EKF2 데이터가 내포하는 분산(Variance)이라는 통계적 확률 지표를 상태 전환의 통과 기준으로 삼고 있다. 설계 과정에서 특수 페이로드를 장착해 기체 동역학이 어긋나게 되었을 때, 개발자는 MDP 관점에서 센서 노이즈가 상태 전이에 미치는 영향을 평가하라. 신뢰도 임계값 파라미터를 동적으로 스케일링하여, 특정 잡음 환경하에서도 가장 안전한 상태(State)로 기체가 전이될 수 있는 최적의 정책(Policy)을 유도하는 것이 매우 중요하다.
