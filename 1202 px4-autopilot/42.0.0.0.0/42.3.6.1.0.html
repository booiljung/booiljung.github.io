42.3.6.1. 마할라노비스 거리(Mahalanobis Distance)를 활용한 센서 측정치 이상치(Outlier) 배제 논리

무인기 시스템(UAS)의 상태 추정기(State Estimator)는 외부 환경의 급격한 변화, 전자기 간섭, 또는 센서의 특정 하드웨어적 결함으로 인해 비정상적인 스파이크를 동반하는 관측 노이즈(Observation Noise)를 수신할 가능성이 있다. PX4-Autopilot의 확장 칼만 필터(EKF2)는 이러한 이상치(Outlier) 데이터가 코어 필터의 상태 갱신(State Update) 루프에 치명적으로 반영되는 것을 방어하기 위해 ‘마할라노비스 거리(Mahalanobis Distance)’ 검정 기법을 적용한다. 본 절에서는 단순한 유클리디안 거리(Euclidean Distance) 대신 분산(Variance)과 공분산(Covariance) 정보를 통합 수용하는 마할라노비스 거리 수학적 모델이 PX4의 EKF2 아키텍처 내부에서 어떻게 이상치를 판별하고 배제하는지 상세히 분석하라.

1. 유클리디안 거리와 마할라노비스 거리의 차이점

센서 데이터의 이상치를 검증할 때, 상태 추정치와 실제 관측치 변수들 간의 단순 거리인 유클리디안 거리를 산출하게 되면 각 축(예: X, Y, Z축 가속도)이 가지는 상태 분산과 변수 간의 상관관계(Correlation)를 무시하게 되는 한계가 존재한다.
반면 **마할라노비스 거리(Mahalanobis Distance)**는 관측 데이터 집합의 분산-공분산 행렬(Variance-Covariance Matrix)을 스케일링 필터로 반영한다. 이는 EKF 내부에서 각 상태 변수의 고유한 신뢰도를 정량적인 척도로 사용하는 특성과 직결되며, 불확실성(Uncertainty)이 큰 방향의 오차에는 상대적으로 너그러운 가중치를 주고, 불확실성이 작고 확고한 방향의 오차는 매우 엄격하게 평가하여 즉각적인 배제를 수행한다.

2. EKF2 모듈 내부의 이상치 판별 수학적 모델

PX4 소스코드(예: src/modules/ekf2/EKF/estimator_interface.cpp 및 관련된 서브모듈)의 혁신(Innovation) 기반 관측 배제 프레임워크(Rejection Framework)에서 마할라노비스 거리 검정(D_M)은 다음과 같은 행렬 수식을 통해 도출된다.

항목수식
이노베이션 벡터(Innovation Vector)y = z - h(\hat{x})
z: 실제 센서 관측치, h(\hat{x}): 예측된 관측모델
이노베이션 공분산(Innovation Covariance)S = H P H^T + R
H: 관측 행렬, P: EKF 상태 공분산, R: 관측 노이즈 공분산
마할라노비스 거리 통계량D_M^2 = y^T S^{-1} y
이상치 배제 조건(Outlier Rejection Gate)D_M^2 > \chi^2_{\alpha, p}
혹은 D_M^2 > K_{gate}^2

위의 수학적 정의에서 볼 수 있듯이, 관측된 이노베이션 잔차 벡터 y를 이노베이션 공분산 행렬의 역행렬 S^{-1}을 통해 정규화하여 거리의 제곱 D_M^2를 구한다. 이 스칼라 값이 카이제곱 분포의 임계치 층위 혹은 파라미터로 지정된 게이트 배수(K_{gate})를 초과할 경우, 데이터는 통계적으로 유의미한 외란 범위를 이탈한 것으로 판단된다. 즉, 시스템 공분산을 초과하는 비정상 데이터이므로 PX4 EKF 시스템은 해당 프레임의 관측 데이터를 칼만 게인(Kalman Gain) 연산에서 즉시 폐기(Reject)한다.

3. 타 센서 파이프라인으로의 적용 범례

  • 위성항법장치(GNSS): 난반사(Multipath Error)가 극심하게 나타나는 도심지나 송전탑 등 특정 지형지물 환경에서 GPS의 속도 및 위치 좌표는 일시적으로 수십 미터 단위의 오티 튀어오름(Jump) 현상을 발생시킨다. 이때 속도 성분 V_x, V_y 또는 위상 성분 P_x, P_y의 이노베이션은 즉각 마할라노비스 게이트 한계치를 돌파한다. EKF2는 파라미터를 참조하여 이 데이터를 수 초간 강제 거부(GPS Reject)함으로써, 기체의 극단적인 다이빙이나 자세 서지(Surge)를 성공적으로 차단한다.
  • 광학 흐름 센서(Optical Flow): 바닥면 텍스처(Texture)가 희박하고 조도 편차가 심한 상황에서, 잘못 매칭된 영상 프레임 연산 결과로 인해 비정상적인 광학 흐름 속도 벡터가 추출된다. 이 경우에도 EKF2는 지정된 EKF2_OF_GATE 제한을 이용해 마할라노비스 거리의 범주에 부합하지 않는 비이상 속도 데이터를 제거하여 안정성을 확보한다.

5. QGroundControl (QGC) 연동과 관제 설계

비행 전후의 상태 통제 및 데이터 디버깅을 수행하는 GCS(QGroundControl)에서는 이 고차원적인 통계 배제 로직을 사용자가 세밀하게 파라미터로 튜닝하여 제어할 수 있다.

  • 표준 편차 게이트(Gate) 튜닝: 시스템은 각 센서의 물리적 특성별로 마할라노비스 게이트 스케일러 파라미터를 노출한다. 대표적으로 EKF2_GPS_V_GATE (GPS 속도 이노베이션 게이트 배수), EKF2_MAG_GATE (지자기 이노베이션 게이트 크기) 값이 존재한다. 숫자를 높이면 통계적 포용 오차가 넓어져 더 많은 데이터를 승인하지만 외란 방어력이 떨어지고, 낮추면 조금의 오차만 있어도 배제(Reject)하여 안전성을 극대화하나 GPS Denied 상황을 너무 쉽게 유발할 수 있다.
  • Flight Review 통합 분석: 비행 시험 종료 후 PX4의 비행 기록장치인 ULog 데이터 파일을 웹 매트릭스 분석 도구(Flight Review)로 업로드하면, estimator_status 토픽 내부의 비트마스크 배열 중 innovation_rejected 플래그를 정밀 분석할 수 있다. 관제 엔지니어는 그래프 분석을 통해 마할라노비스 거리 임계에 부딪혀 특정 센서가 억압받았던 정확한 시간 구간(timestamp)을 도출하고, 그에 따른 튜닝 및 하드웨어 점검 계획을 수립한다.

6. 결론

마할라노비스 거리를 활용한 데이터 배제 모델은 시스템 운영 중에 센서 데이터의 절대적인 거리 크기뿐만 아니라, 시스템이 스스로 인식하는 분산-공분산 모델의 불확실성을 기하학적으로 연동시키는 가장 진보된 다차원적 통계 배제 메커니즘이다. PX4는 Ardupilot과의 아키텍처 비교 우위를 논할 수 있는 한 축으로써, 이러한 높은 학술적 타당성에 내재된 방어 기술을 EKF2 하위 클래스들 전반에 모듈식으로 견고하게 배치함으로써, 극한의 비행 임무 환경 조건에서도 신뢰성 있는 자세와 위치 도출을 지속한다.
