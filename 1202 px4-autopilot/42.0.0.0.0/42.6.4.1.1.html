42.6.4.1.1 빈센티 공식(Vincenty’s Formulae)을 적용한 지구 타원체 상의 측지선(Geodesic Line) 거리 및 교차점 연산 정밀도 검증

장거리 비행을 수행하는 고정익(Fixed-wing) 무인기나 VTOL(Vertical Take-Off and Landing) 기체의 경우, 지오펜스(Geofence)의 반경이 수십 킬로미터(km)에 달할 수 있다. 이처럼 광범위한 영역에서 지오펜스 다각형의 선분(Edge)과 기체의 교차(Intersection) 여부를 판별할 때, 지구를 단순한 평면이나 완전한 구(Sphere)로 가정하는 유클리드 기하학(Euclidean Geometry)이나 하버사인 공식(Haversine Formula)을 사용하면 치명적인 오차가 발생한다. 본 절에서는 PX4-Autopilot이 WGS84 타원체(Ellipsoid) 모델 기반에서 정밀한 측지선(Geodesic Line) 거리와 방위각을 연산하기 위해 적용하는 빈센티 공식(Vincenty’s Formulae)의 수학적 원리와 그 검증 메커니즘을 심도 있게 다룬다. (기준 버전: PX4-Autopilot v1.14.0)

1. 지구 타원체 모델과 하버사인 공식의 한계

GPS 센서로부터 획득하는 위도(Latitude)와 경도(Longitude) 데이터는 WGS84 좌표계에 기반한다. WGS84는 지구를 적도 반지름이 극 반지름보다 약간 긴 ’회전 타원체(Oblate Spheroid)’로 정의한다.

  • 하버사인 공식(Haversine Formula): 지구를 완전한 구로 가정하고 두 지점 간의 대원(Great Circle) 거리를 구하는 방식이다. 계산 주기가 빠르지만, 적도 부근과 극지방 부근에서의 곡률 차이를 반영하지 못해 지구 전체적으로 최대 약 0.5% (수십 킬로미터 거리에서 수백 미터)의 오차가 발생할 수 있다.
  • 이 정도의 오차는 좁은 반경을 비행하는 멀티로터에는 무방하나, 100km 이상을 횡단하는 배달용 드론이나 정찰용 무인기의 지오펜스 경계선 도달 여부를 판별할 때는 치명적인 ‘거짓 이탈(False Breach)’ 혹은 ’미감지(Missed Detection)’를 유발한다.

2. 빈센티 공식(Vincenty’s Formulae)의 수학적 구조

타데우스 빈센티(Thaddeus Vincenty, 1975)가 발표한 이 공식은 회전 타원체 표면에서 두 지점 간의 최단 거리인 측지선(Geodesic Line)의 길이와 방위각을 반복 연산(Iterative Method)을 통해 0.5mm 수준의 극초정밀도로 산출하는 수학적 기법이다. 계산은 크게 두 가지 문제로 나뉜다.

2.1 역문제(Inverse Problem)

역문제는 두 지점의 위도와 경도 (\phi_1, \lambda_1)와 (\phi_2, \lambda_2)가 주어졌을 때, 두 지점 사이의 표면 거리 s와 서로를 바라보는 초기/최종 방위각 \alpha_1, \alpha_2를 구하는 알고리즘이다.

PX4의 지오펜스 및 궤적 추종(Trajectory Tracking) 연산에서는 이 역문제가 주로 사용된다. 지오펜스를 구성하는 다각형의 꼭짓점 V_1과 V_2 사이의 정확한 선분 길이와 방향을 도출하기 위해, WGS84 타원체의 장반경 a, 단반경 b, 편평률 f = (a-b)/a를 매개변수로 하여 다음의 반복 연산을 수행한다.

  1. 계수 축소 위도(Reduced Latitude) \beta_1, \beta_2를 계산한다: \tan\beta = (1-f)\tan\phi
  2. 적도 상의 경도 차이 \lambda = \lambda_2 - \lambda_1 를 기준으로 구면 상의 경도 차이 L을 가정하고, 수렴 조건이 달성될 때까지 아래 식을 반복한다.
    \sin\sigma = \sqrt{(\cos\beta_2\sin\lambda)^2 + (\cos\beta_1\sin\beta_2 - \sin\beta_1\cos\beta_2\cos\lambda)^2}
    \cos\sigma = \sin\beta_1\sin\beta_2 + \cos\beta_1\cos\beta_2\cos\lambda
  3. L의 값이 이전 단계의 값과 비교하여 오차 허용 범위(10^{-12} 수준) 내로 수렴하면 반복을 종료하고 타원체 상의 거리 s를 계산한다.

2.2 순문제(Direct Problem)

순문제는 시작점 (\phi_1, \lambda_1)에서 특정 방위각 \alpha_1을 향해 거리 s만큼 이동했을 때의 도착점 위치 (\phi_2, \lambda_2)를 구하는 것이다. PX4 내부에서는 특정 웨이포인트(Waypoint)를 기준으로 반경(Radius)을 설정하는 원형 지오펜스의 경계 꼭짓점을 생성할 때 이 공식이 제한적으로 활용된다.

3. PX4 소스 코드 내 빈센티 공식 적용 및 최적화

초정밀도를 자랑하는 빈센티 공식이지만, 알고리즘 내부에 삼각함수와 반복 루프(Iteration)가 포함되어 있어 컴퓨팅 부하(Overhead)가 상대적으로 높다. 따라서 PX4 개발팀은 연산 자원을 최적화하기 위해 다음과 같은 전략적 타협(Trade-off) 아키텍처를 구현하였다.

3.1 geo.cpp 유틸리티 모듈의 계층적 적용

PX4 소스 트리 내 src/lib/geo/geo.cpp 라이브러리에는 두 지점 간의 거리를 구하는 전역 함수들이 정의되어 있다.
시스템은 요구되는 정밀도와 지점 간의 거리에 따라 연산 방식을 동적으로 전환한다.

일반적인 쿼드콥터 비행 모듈에서는 C 혹은 D 경로가 주로 실행되지만, navigator 모듈 내에서 **글로벌 다각형 지오펜스(Global Polygon Geofence)**의 대규모 선분을 Local NED 평면으로 최초 1회 투영(Projection)하여 기준점을 매핑(Mapping)하는 핵심 초기화 과정에는 빈센티 공식 수준의 정밀 연산이 간접적으로 활용되거나, 기준점과의 편차를 정확히 반영하는 투영 알고리즘이 적용된다.

3.2 투영 맵(Projection Map)과 교차점 연산의 결합

비행 중 기체가 매 루프(약 50Hz)마다 빈센티 공식을 사용하여 수십 개의 꼭짓점을 가진 비볼록 다각형 경계와의 교차점을 직접 구하는 것은 불가능에 가깝다.

따라서 PX4-Autopilot은 이중화 검증 전략을 취한다.

  1. 사전 연산(Pre-computation): 지오펜스 데이터가 업로드될 때, 혹은 홈 포지션(Home Position)이 갱신될 때 단 1회 타원체 변환(빈센티 유사 정밀도)을 거쳐 모든 MAVLink 글로벌 다각형 꼭짓점들을 기체 기준의 정확한 Local NED 2D 직교 좌표계로 변환하여 메모리에 고정한다.
  2. 실시간 교차 연산(Real-time Intersection): 비행 중에는 2D 직교 좌표계상에서 단순화된 “광선 투사 알고리즘(Ray-casting)“과 1차 선형 방정식만을 사용하여 교차점 연산을 수행한다.

이러한 설계 패턴(Design Pattern)은 지구 타원체 상의 거리 왜곡 문제를 해결함과 동시에 RTOS 환경의 혹독한 실시간 처리 제약(Real-time Constraints)을 완벽하게 만족시키는 공학적 타협의 정수라 할 수 있다.

4. 측지선 정밀도 검증의 결론

결론적으로 PX4-Autopilot은 단순히 알고리즘을 수학적으로 완벽하게 구현하는 것에 그치지 않고, “정밀도가 필요한 시점(초기 맵 투영)“과 “속도가 필요한 시점(실시간 광선 투사 교차 검사)“을 분리하는 소프트웨어 아키텍처를 취하고 있다. GCS 개발자는 장거리 비행 임무 설계 시, 기체가 내부적으로 수행하는 이 타원체 측지선 보정 로직을 신뢰하고 MAVLink MISSION_ITEM_INT (1e7 스케일의 고정밀 좌표) 포맷을 적극 활용하여 다각형 꼭짓점 정밀도를 보장해 주어야 한다.
