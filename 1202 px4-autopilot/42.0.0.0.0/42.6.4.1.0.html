42.6.4.1 광선 투사 알고리즘(Ray-casting Algorithm)을 응용한 비볼록(Non-convex) 다각형 내 위상 수학적 포함(Inclusion) 여부 검증

무인기 비행 안전을 보장하는 지오펜스(Geofence) 시스템은 복잡한 도심지나 특정 시설물 주변을 회피하기 위해 종종 요철이 있는 비볼록(Non-convex) 다각형 형태로 설정된다. 일반적인 볼록(Convex) 다각형과 달리, 비볼록 다각형은 내부 통과 구역을 수학적으로 판별하기 까다롭기 때문에 특별한 평가 알고리즘이 요구된다. 본 절에서는 PX4-Autopilot이 비볼록 다각형 지오펜스 내 기체의 포함(Inclusion) 여부를 실시간으로 검증하기 위해 채택한 광선 투사 알고리즘(Ray-casting Algorithm)의 기하학적, 수학적 원리를 심층적으로 분석한다. (기준 버전: PX4-Autopilot v1.14.0)

1. 비볼록(Non-convex) 다각형의 기하학적 특성과 문제 제기

다각형은 모든 내각이 180도 미만이고, 임의의 두 내점을 잇는 선분이 항상 다각형 내부에 존재하는 볼록 다각형(Convex Polygon)과, 하나 이상의 내각이 180도를 초과하여 이 성질을 만족하지 못하는 비볼록(Non-convex) 혹은 오목(Concave) 다각형으로 분류된다.

  • 볼록 다각형의 검사: 모든 선분에 대해 반평면 검사(Half-plane Test)를 수행하여, 점이 모든 선분의 안쪽(동일한 방향)에 있으면 내부에 있다고 쉽게 판별할 수 있다. (예: 외적을 이용한 CCW 알고리즘 상수항 검사)
  • 비볼록 다각형의 문제: 장애물을 세밀하게 감싸는 지오펜스는 필연적으로 비볼록 형태를 띤다. 비볼록 다각형에서는 반평면 검사가 더 이상 유효하지 않으므로, 수학적으로 위상 기하학(Topology)에 기반한 더 범용적인 Point-in-Polygon (PIP) 해법이 필요하다.

이러한 범용적인 해법 중 하나가 조르당 곡선 정리(Jordan Curve Theorem)에 기반한 광선 투사 알고리즘이다.

2. 광선 투사 알고리즘(Ray-casting Algorithm)의 위상 수학적 원리

조르당 곡선 정리에 따르면, 연속적인 단순 폐곡선(Simple Closed Curve)은 2차원 평면을 명확히 하나의 내부(Interior)와 하나의 외부(Exterior) 구역으로 분할한다.

이를 이산적인 다각형 선분에 적용한 것이 교차점 카운팅(Crossing Number) 메서드 혹은 **홀짝 규칙(Even-Odd Rule)**이다. 임의의 위치 P(x_p, y_p)에 기체가 존재할 때 포함 여부 검증 원리는 다음과 같다.

  1. 점 P에서 시작하여 한쪽 방향(일반적으로 X축 양의 방향)으로 발사되는 무한한 반직선(Ray) R을 정의한다. 수식으로는 y = y_p, x \ge x_p의 경로를 따른다.
  2. 다각형을 구성하는 선분 집합 E_i = (\overline{V_i V_{i+1}})에 대해 반직선 R이 선분 E_i와 교차하는지 검사한다.
  3. 교차하는 횟수(Crossing Number, C)를 합산한다.
  4. C \pmod 2 \equiv 1 이면 점 P는 다각형 내부에 존재하며 (위상 수학적 포함 상태), C \pmod 2 \equiv 0이면 외부에 존재한다.

3. PX4 소스 코드 내 광선 투사 알고리즘 최적화 구현 전략

PX4-Autopilot은 navigator 모듈의 지오펜스 루틴에서 이 알고리즘을 C++ 로 구현하고 있다. 컴퓨팅 리소스가 제한적인 임베디드 실시간 운영체제(RTOS) 상에서 레이 캐스팅 루틴은 매우 빈번하게(수십~수백 Hz) 실행되므로 연산 최적화가 필수적이다.

3.1 분기 예측 최적화(Branch Prediction Optimization)와 예외 처리

임의로 발사한 X축 평행 광선이 선분과 교차하는 조건을 수식으로 풀기 전에, PX4는 불필요한 부동 소수점(Floating-point) 연산을 회피하기 위한 신속한 거부(Fast-Rejection) 조건을 확인한다.

임의의 선분 양 끝점 V_1(x_1, y_1)과 V_2(x_2, y_2)가 있을 때, 광선 y = y_p, x \ge x_p와의 교차를 판별하기 위해 다음의 예외 규칙들을 고속으로 연산한다.

  1. Y축 범위 검사 (Y-bound Check): 두 점 V_1, V_2의 y 좌표가 모두 y_p보다 작거나 모두 y_p보다 큰 경우, 해당 선분은 광선과 절대 만날 수 없다.
  • 조건: (y_1 > y_p) \land (y_2 > y_p) 또는 (y_1 \le y_p) \land (y_2 \le y_p)
  1. X축 방향성 검사 (X-bound Fast Reject): V_1, V_2의 x 좌표가 모두 x_p보다 작다면 광선이 뻗어나가는 방향(X 양수 방향)의 뒤편에 존재하므로 교차하지 않는다.
  • 조건: \max(x_1, x_2) < x_p

위의 고속 거절 조건을 통과한 선분에 대해서만 실제 교차점의 x좌표(x_{cross})를 비례식으로 연산한다.

x_{cross} = x_1 + \frac{(y_p - y_1)(x_2 - x_1)}{y_2 - y_1}

검사점 P의 x좌표 점 x_p보다 x_{cross}가 크다면(x_p < x_{cross}) 광선은 다각형의 선분을 확정적으로 관통한 것이며, 교차 카운트 C를 1 증가시킨다.

3.2 꼭짓점 통과 예외 처리(Vertex Intersection Edge Case)

레이 캐스팅에서 가장 까다로운 예외 상황은 광선이 다각형의 꼭짓점을 정확히 관통할 때이다. 꼭짓점을 지나는 광선은 인접한 두 선분에 의해 이중으로 교차 카운트될 위험이 있다.

PX4-Autopilot 및 표준 컴퓨터 그래픽스 알고리즘에서는 이를 해결하기 위해 반개구간(Half-open Interval) 개념을 채택한다. 즉, 각 선분의 교차 범위에서 한쪽 끝점은 포함하고 반대쪽 끝점은 배제하는 방식(예: y_1 \le y_p < y_2)을 적용하여, 꼭짓점 관통 시 1회 교차로만 인식되거나 아예 무시되도록 논리적 무결성을 유지한다.

4. GCS 연동 및 관제 시스템의 검증

QGroundControl(QGC) 내의 Mission Planner 모듈 역동학(Dynamics) 아키텍처는 PX4 펌웨어와 상호 대조 확인을 수행한다. GCS에서 사용자가 복잡한 형태의 폴리곤 지오펜스를 마우스 클릭으로 생성할 경우, QGC 측에서도 위와 동일한 형태의 교차 검사 알고리즘을 백그라운드로 실행하여 사용자가 드론을 펜스 밖으로 비행시키려는 임무(Waypoint)를 설계하는 것을 실시간 경고(Warning) 팝업으로 차단한다.

이를 통해 “비정형 비볼록 장애물 회피 시스템“은 기체 내부의 실시간 감시(Active Fencing) 메커니즘과 지상국 관제의 계획 단계(Pre-flight Mission Verification) 양측에서 높은 신뢰도를 갖추게 된다.
