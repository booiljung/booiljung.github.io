43.3.6 무선 주파수(RF) 링크의 패킷 수신율(PRR) 및 수신 신호 강도 지시자(RSSI)의 이동 평균 필터링 기반 연결 무결성 평가

지상 관제 시스템(GCS) 또는 조종기(Radio Control)와 기체 간의 안정적인 무선 제어(Radio Frequency, RF) 채널은 무인항공기의 생명줄과 같다. PX4-Autopilot은 조종기로부터 수신되는 RC 신호(SBUS, PPM, CRSF 등) 및 고수준의 MAVLink 데이터 링크(텔레메트리) 단절 시 기체를 안전하게 귀환(Return)시키거나 강제 착륙(Land)시키는 강력한 페일세이프(Failsafe) 메커니즘을 내장하고 있다.

단순히 “신호가 들어온다 / 안 들어온다“의 이분법적인 판단을 넘어, 시스템 엔지니어링 측면에서 PX4는 조종기 수신기(RC Receiver)로부터 수신되는 **수신 신호 강도 지시자(RSSI)**와 디지털 데이터 전송의 신뢰도를 대변하는 **패킷 수신율(PRR)**을 종합적으로 모니터링한다. 순간적인 전파 간섭이나 프레임 손실(Frame Loss)로 인해 기체가 갑작스럽게 수동 제어 불능 상태(Manual Control Loss)로 판치지 않도록, 다단 이동 평균 필터링(Moving Average Filtering) 을 적용하여 연결 무결성(Connection Integrity)을 엄격하게 평가한다.

1. 수신 신호 강도 지시자 (RSSI: Received Signal Strength Indicator)

RSSI는 기체의 수신 안테나에 도달한 전파의 물리적인 에너지 강도를 나타낸다. (일반적으로 -100dBm ~ 0dBm 또는 0% ~ 100% 범위로 스케일링되어 보고된다.)

조종기 칩셋(예: ELRS, Crossfire, FrSky)은 현재 잡히는 전파의 절대적인 에너지 세기를 RC 프로토콜의 특정 채널(예: SBUS의 16번 채널 등)에 실어 비행 제어기(FC)로 넘기거나, 전용 텔레메트리 프레임(예: CRSF 링크 통계 프레임)을 통해 PX4의 rc_input 모듈로 직접 주입한다.

그러나 RSSI는 안테나의 지향성 각도 변화, 기체의 일시적인 뱅크(Bank) 기동에 따른 동체 가림(Body Masking), 지면 반사파 등 물리적 외란에 의해 매우 심하게 출렁(Fluctuation)거린다. 따라서 순간적으로 RSSI가 0%로 찍히더라도 실제 조종 명령 패킷은 간신히 1~2개씩 들어오고 있을 확률이 크다.

2. 패킷 수신율 (PRR: Packet Reception Rate / Link Quality)

RSSI가 “전파의 세기“라면, PRR(흔히 Link Quality, LQ로도 불림)은 “그래서 실제 해독에 성공한 데이터 패킷이 몇 개나 되는가?“를 따지는 통신 프로토콜 계층의 논리적 지표이다.

  • 100Hz로 전송되는 RC 시스템에서 1초 동안 90개의 패킷을 정상적으로 파싱(Parsing)하고, 10개의 패킷을 CRC(순환 중복 검사) 에러나 프레임 유실로 버렸다면 PRR은 90%가 된다.
  • 디지털 프로토콜(예: TBS Crossfire)은 RSSI가 낮더라도 강력한 오류 정정 코드(FEC)의 이점을 받아 PRR을 100% 가깝게 유지하는 특징이 있다.

PX4의 rc_update 모듈과 커맨더(Commander) 내부 로직은 단일 RSSI 데이터보다 **단위 시간당 성공한 프레임 갱신율(Frame Update Rate)**인 이 PRR을 텔레메트리 단절의 훨씬 더 확정적인 트리거(Trigger) 변수로 신뢰한다.

3. 이동 평균 필터링 (Moving Average Filtering) 기반 무결성 평가 모형

가까운 거리에서도 송신기와 수신기 사이에 갑작스럽게 사람이나 물체가 지나가면 패킷 수신율(PRR)이나 RSSI가 1~2 프레임 동안 0으로 폭락하는 현상이 발생할 수 있다. 이때 즉각적으로 Failsafe(RTL) 모드로 전환되면 비행의 부드러움이 산산조각 날 것이다.

이를 방지하기 위해 PX4는 이동 평균 필터(Moving Average Filter)와 RC Loss Timeout 계수를 결합하여 신호를 디바운싱(Debouncing)한다. 즉, 특정 시간(Window Size) 동안의 평균적인 신호 강도와 수신 빈도수를 평가하여 참된 단절(True Loss) 상태인지를 판결한다.

3.1 필수 설정 파라미터

  • COM_RC_LOSS_T 파라미터: 수신된 유효 패킷이 완전히 끊기거나, 혹은 이동 평균 필터를 통과한 PRR이 기능적 불능 상태로 떨어졌을 때, 시스템이 RC 손실(RC Control Loss)을 선언하기까지 기다려주는 대기 시간(기본값 통상 0.5초)이다. 이 지연 시간이 지나치게 짧으면 오작동이 잦고, 너무 길면 통제 불능 기체가 수십 미터를 그대로 날아가 사고를 친다.
  • RC_RSSI_PWM_MIN 및 RC_RSSI_PWM_MAX: 일부 아날로그 수신기의 경우 별도의 PWM 채널 채널 한 개를 희생하여 RSSI의 세기를 펄스폭(Duty Cycle)으로 넘겨주는데, 이때 PX4가 읽어들이는 펄스의 상하한 범위를 매핑(Mapping)하는 파라미터들이다.

이러한 필터링 과정을 거쳐 검증된 무결성 지표는 QGroundControl의 상단 안테나 아이콘 게이지 바(Gauge Bar)의 칸 수를 결정짓는 근거 데이터로 지속 발행(input_rc 내의 rssi 변수)된다.

4. Ardupilot 대비 차이 (Architectural Differences)

RC 연결 무결성을 측정하고 Failsafe를 발동시키는 기저 철학은 PX4와 Ardupilot이 매우 흡사하나, 처리하는 필터링의 계층과 유저 경험(UX) 측면에서는 다른 방향성을 지닌다.

  • RSSI vs Link Quality (LQ) 우선주의: PX4는 비교적 최신 디지털 프로토콜(CRSF, MAVLink RC 등)의 보급에 따라 프레임 유실률 방어, 즉 LQ 중심의 RC 무결성 판독에 아키텍처 비중이 많이 옮겨져 있다. 반대로 Ardupilot은 유서 깊은 오픈 소스답게 전통적인 아날로그 핀 아웃(Analog Pin-out RSSI) 수신기와 수많은 Legacy 프로토콜 포맷 지원에 더 많은 하드웨어 포팅 가이드가 엮여 있어 세밀한 필터링 계수 변수를 지원한다.
  • FS 발동 지연 시간 (Timeout Layering): 두 시스템 모두 타이머 초과 기반의 FS 트리거(COM_RC_LOSS_T vs FS_THR_VALUE / FS_SHORT_ACTN)를 지니지만, Ardupilot은 Short Failsafe(잠깐 끊기면 Circle/Hold 돌기)와 Long Failsafe(심각하게 끊기면 RTL 하기)로 대응 시간을 이원화(Two-tier)하여 구성하는 경향이 크다. 반면, PX4 커맨더 로직은 시스템 전반의 무효화(Invalidation) 단계를 상태 트리거로 명확히 단일화하여, 연결이 죽었다고 판정난 시점에서 바로 최우선 대응 모드(Primary Failsafe Action)로 천이한다.

결과적으로 PX4의 전파 링크 무결성 평가는, 단순히 RSSI의 크기를 추적하는 것이 아닌 비행 제어 입력을 실질적으로 책임지는 패킷 도달률(PRR)의 연속성을 통계학적으로 필터링하여 기체의 시스템적 고아가 발생하지 않도록 감시하는 통신 최후의 보루 역할을 수행하고 있다.
