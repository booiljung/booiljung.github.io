43.6.4 지수적 백오프(Exponential Backoff) 계수를 적용한 MAVLink 제어 패킷의 혼잡 제어(Congestion Control) 윈도우 스케일링 및 재전송 큐(Queue) 관리

무인항공기(UAV) 시스템의 통신 단말인 기체(PX4-Autopilot)와 지상 관제 시스템(QGroundControl) 간의 무선 데이터 링크는 본질적으로 대역폭이 제한적이며 패킷 손실률이 높은 특성을 지닌다. 특히 제어 명령(COMMAND_LONG 등)과 같이 시스템의 안전과 직결되는 핵심적인 MAVLink(Micro Air Vehicle Link) 패킷은 어떠한 네트워크 장애 조건에서도 신뢰성 있는 전달이 보장되어야 한다. 이를 달성하기 위해 PX4-Autopilot의 통신 스택은 단순한 재전송을 넘어, 네트워크 혼잡도에 적응적으로 반응하는 혼잡 제어(Congestion Control) 윈도우 스케일링 기법과 지수적 백오프(Exponential Backoff) 기반의 큐(Queue) 관리 메커니즘을 적용하여 시스템의 강건성을 극대화한다.

1. 무선 텔레메트리 환경과 혼잡 제어의 필요성

일반적인 텔레메트리(Telemetry) 라디오 모듈(예: SiK Radio, 900MHz/2.4GHz 대역)은 반이중(Half-Duplex) 통신 방식을 채택하고 있어 송신과 수신이 동시에 이루어질 수 없다. 이러한 환경에서 시스템이 지수적 백오프 없이 고정된 타임아웃(Timeout) 주기로 응답받지 못한 제어 명령을 반복해서 재전송할 경우, 대역폭 포화(Bandwidth Saturation)를 유발하여 오히려 다른 중요 패킷(예: 하트비트, 자세 데이터)의 송신 기회마저 박탈하는 이른바 재전송 폭풍(Broadcast Storm) 현상에 직면하게 된다.

이를 방지하기 위해 PX4-Autopilot과 QGroundControl(설명 기준: QGroundControl v4.3.x 이상, PX4 Firmware v1.14.x 이상)은 혼잡 상태를 능동적으로 파악하고 송신 간격을 동적으로 확장하는 지수적 백오프 전략을 활용하여 네트워크 효율성과 제어 명령의 도달률 간의 최적의 균형을 맞춘다.

2. 지수적 백오프(Exponential Backoff) 알고리즘의 공학적 모델

지수적 백오프는 충돌 감지 매체 접근 제어(CSMA/CD) 방식 등 네트워크 공학의 기초에서 널리 사용되는 체증 회피 알고리즘이다. MAVLink 제어 패킷의 재전송에서, 시간 지연 T_{wait}는 재전송 시도 횟수 n이 증가함에 따라 기하급수적으로 증가하도록 설계된다.

T_{wait}(n) = T_{base} \times k^n

  • T_{base} : 최초 재전송을 대기하는 기준 시간 (Base Timeout)
  • k : 백오프 계수 (보통 2를 적용하는 이진 지수적 백오프, Binary Exponential Backoff 활용)
  • n : 현재까지의 재전송 실패 횟수 (예: 1, 2, 3…)

재전송이 실패할 때마다 대기 시간 윈도우를 2배씩 스케일링함으로써 현재 채널의 심각한 패킷 로스 상황을 회피하고 송신 대역을 양보(Yield)하는 효과를 낳는다. 시스템은 무한정 대기할 수 없으므로, 재전송 최대 횟수(Max Retries) 인계점 혹은 최대 대기 시간(Max Timeout Limit) 전제 조건을 설정하여, 한계 도달 시 링크 단절(Data Link Loss)에 따른 Failsafe 루틴으로 전환한다.

3. 재전송 큐(Retransmission Queue) 및 버퍼 생명주기 관리

지상 관제 시스템이 기체로 제어 패킷을 송신하면, 해당 패킷은 즉시 소거되지 않고 대기 큐(Pending Queue) 내부에 객체 상태로 안전하게 보관된다. MAVLink 명령어(mavlink_command_long_t)에 대한 ACK(COMMAND_ACK)를 수신했을 때 비로소 큐에서 제거(Dequeue)되며, 메모리 자원 누수를 철저히 통제하기 위해 엄격한 생명주기(Lifecycle) 관리가 적용된다.

Ardupilot 대비 PX4-Autopilot 큐 관리의 설계 철학

Ardupilot은 내부 큐 관리에서 종종 고정된 정적 배열 크기를 선호하고 순환 버퍼(Circular Buffer) 형태를 띠는 편견향이 있는 반면, PX4-Autopilot은 uORB 이벤트 기반 처리와 결합된 동적 리스트 형태의 큐 설계를 통해 동시 다발적인 복수 명령들에 대한 개별 타이머 구조를 좀 더 독립적으로 운용한다. 하지만 C/C++ 기반의 임베디드 특성상 파편화를 막기 위해 Intrusive List 형태의 메모리 풀(Memory Pool) 모델을 활용해 오버헤드를 극소화하는 공학적 접근을 취한다.

4. 혼잡 제어 윈도우 스케일링 메커니즘 (Mermaid 모델링)

아래의 상태 머신 모델은 제어 명령이 GCS 혹은 기체의 MAVLink 큐에 진입한 이후, 어떻게 윈도우 크기를 스케일링하고 재전송을 판별하는지를 나타낸다.

stateDiagram-v2
    [*] --> Idle: 시스템 대기
    Idle --> Command_Enqueue: 신규 명령 송신 요청
    
    state "재전송 큐(Retransmission Queue)" as Queue {
        Command_Enqueue --> Await_ACK: 패킷 발송 및 T_base 설정
        Await_ACK --> Await_ACK: ACK 미수신 상태로 시간 경과
        Await_ACK --> Scaling_Window: T_wait 초과 (Timeout 발생)
        
        Scaling_Window --> Retry_Check
        Retry_Check --> Re_Transmit: 실패 횟수 n < Max
        Re_Transmit --> Await_ACK: T_wait = T_base * 2^n 확장
        
        Retry_Check --> Time_Out_Failure: 실패 횟수 n >= Max
    }
    
    Await_ACK --> Success_Dequeue: COMMAND_ACK 수신 성공
    Success_Dequeue --> Idle: 큐 항목 해제 (메모리 해제)
    Time_Out_Failure --> Failsafe_Trigger: 명령 실패 처리 및 Failsafe
  1. 송신 및 대기(Transmitting & Awaiting) : GCS 프론트엔드에서 생성된 명령은 MAVLink 송신 버퍼를 타며 메인 대기열에 진입(Enqueue)한다.
  2. 윈도우 스케일링(Scaling Window) : 설정된 Base 타임아웃 내에 응답이 없으면, 시스템 혼잡이 발생한 것으로 인지하고 T_{wait}를 지수 단위로 팽창(Scale-up)시킨 후 재전송한다.
  3. 성공 및 해제(Success & Dequeue) : 유효한 시퀀스 번호(Sequence Number)를 포함한 ACK가 도착하면 타임머를 즉각 중단하고 메모리를 반환한다.
  4. 한계 도달(Time-out Failure) : 설정된 재시도 횟수 한계점(예: 3회 ~ 5회)을 초과하면 해당 명령 프레임은 버려지고 상위 통신 단절 처리 모듈로 심각한 수준의 에러 플래그(Error Flag)가 보고된다.

5. 소스 코드 관점의 트러블슈팅 분석 역량

차세대 비행 제어 기술 구현에서 엔지니어가 MAVLink 의 타임아웃/재전송 루틴을 분석해야 하는 경우는 대개 GCS상의 명령이 “반응 없음(No Response)” 상태에 빠졌을 때이다. 이때 시스템 로깅 파일(ULog)을 통해 mavlink_messages를 확인하거나, GCS 내의 디버깅 통계를 추출해야 한다.

  • command_ack 메시지의 누락 원인이 대역폭 제한인지 기체 내부의 처리 지연(CPU Load)인지 구별하는 것이 필수적이다.
  • GCS 소스코드 수준에서는 LinkManager 와 Vehicle 객체단에서 RTT(Round-Trip Time)를 추정하여 T_{base} 자체를 동적으로 조절하는 향상된 윈도우 스케일링이 지원된다. RTT가 길어지면 시스템은 지수적 백오프의 기준점 자체를 이동시켜, 불필요한 재전송을 사전에 차단한다.

결론적으로 PX4-Autopilot과 QGroundControl 생태계가 채택한 지수적 백오프 기반 혼잡 제어 및 큐 관리는, 불안정한 무선 링크의 물리적 한계를 소프트웨어적 지능으로 극복하는 공학 설계의 훌륭한 사례이며, 임무의 결정성에 치명적인 통신 병목을 예방하는 가장 강력한 수단 중 하나이다.
