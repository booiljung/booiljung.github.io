43.7.1.1 지자기 센서의 비직교성(Non-orthogonality) 및 연자성/경자성(Soft/Hard Iron) 왜곡 계수에 따른 3차원 투영 벡터 성분 불일치 오차의 수학적 해소 기법

무인항공기(UAV) 시스템의 안정적인 비행을 위해서는 정확한 방위각(Heading) 추정이 필수적이다. 이를 위해 PX4-Autopilot(v1.14 이상 기준)은 다중 지자기 센서(Magnetometer)를 활용하여 지구 자기장을 측정한다. 그러나 기체에 장착된 지자기 센서는 주변 환경 및 센서 고유의 시스템적 결함으로 인해 왜곡(Distortion)을 경험한다. 본 절에서는 지자기 센서 측정값의 수학적 왜곡 모델을 정의하고, 연자성(Soft Iron) 및 경자성(Hard Iron) 왜곡, 그리고 센서 축 간의 비직교성(Non-orthogonality)에 따른 3차원 투영 벡터 성분 불일치 오차를 해소하기 위한 과정과 이를 PX4 아키텍처 및 시스템 설정(QGroundControl v4.3.x 이상 기준)에 적용하는 방법에 대해 서술한다.

1. 지자기 센서 왜곡의 수학적 모델링 (Mathematical Modeling of Magnetometer Distortion)

지자기 센서에서 측정된 자기장 벡터 \vec{m}_m은 지구의 순수 자기장 벡터 \vec{m}_t와 기체 및 센서 자체의 전자기적 간섭에 의해 수학적으로 다음의 선형 변환식으로 표현된다.

\vec{m}_m = W \cdot S \cdot N \cdot R \cdot \vec{m}_t + \vec{b} + \vec{\epsilon}

여기서 각 변수는 다음과 같은 의미를 가진다.

  • \vec{m}_m: 측정된자기장 벡터 (Measured Magnetic Field Vector)
  • \vec{m}_t: 참 지구 자기장 벡터 (True Earth Magnetic Field Vector)
  • W: 연자성 왜곡 행렬 (Soft Iron Distortion Matrix, 3 \times 3 대칭 행렬)
  • S: 축 비율 불일치 대각 행렬 (Scale Factor Matrix)
  • N: 비직교성 행렬 (Non-orthogonality Matrix, 3 \times 3 상삼각 행렬)
  • R: 기체 좌표계 대비 센서의 오리엔테이션 행렬 (Orientation Matrix)
  • \vec{b}: 경자성 왜곡 벡터 (Hard Iron Offset Vector, 3 \times 1)
  • \vec{\epsilon}: 센서 측정 노이즈 벡터 (Measurement Noise Vector)

1.1 경자성 왜곡(Hard Iron Distortion)

경자성 왜곡 \vec{b}는 기체 내의 영구 자석이나 모터, 전원 케이블의 직류 전류와 같은 고정된 자기장 소스에 의해 발생한다. 측정된 벡터의 원점을 이동시키므로 구형(Sphere)의 중심 오프셋(Offset)으로 나타난다.

1.2 연자성 왜곡(Soft Iron Distortion) 및 비직교성(Non-orthogonality)

연자성 왜곡 W는 기체 프레임, 배터리 등 주변의 자화 가능한 물질에 의해 외부 자기장의 선속(Magnetic Flux)이 굴절되면서 발생한다. 이는 구형(Sphere)의 측정 궤적을 타원체(Ellipsoid)로 왜곡시킨다. 또한, 센서 제조 공정의 한계로 인해 X, Y, Z 축이 완벽한 90도를 이루지 못하여 발생하는 비직교성 N, 그리고 축 간의 감도 차이 S는 최종적으로 보정 매트릭스 \mathcal{S}_{3 \times 3}에 결합되어 수학적 해소가 이루어진다.

종합적인 보정식은 계측된 값을 역산하여 다음과 같이 도출된다.
\vec{m}_{calibrated} = \mathcal{S}^{-1} (\vec{m}_m - \vec{b})

2. PX4-Autopilot의 지자기 센서 캘리브레이션 알고리즘 아키텍처

PX4-Autopilot은 자력계 보정을 위하여 commander 모듈 하위에 캘리브레이션 루틴을 구현하며, 확장 칼만 필터(EKF2, Extended Kalman Filter 2)를 사용하여 비행 중 동적 보정(In-flight Calibration) 또한 수행한다. 반면, Ardupilot은 EKF3 아키텍처를 기반으로 나침반 동기화 및 학습 기반의 컴파스 모터 캘리브레이션(Compass Motor Calibration)을 더 적극적으로 노출하는 차이가 있다.

2.1 Ardupilot 대비 PX4-Autopilot 보정 메커니즘의 차이

PX4-Autopilot은 다중 센서를 uORB(Micro Object Request Broker) 퍼블리셔로 추상화한다. 센서 파이프라인(sensor_mag)에서 각 기기별 CAL_MAGx_ID 매개변수에 바인딩된 오프셋 및 스케일 팩터를 즉각적으로 적용한다. Ardupilot은 MAVLink를 통한 실시간 동적 자기장 학습(Learn Inflight)의 범주가 넓은 반면, PX4-Autopilot은 사전에 철저한 오프라인 캘리브레이션을 요구하며 EKF2 내부 상태 벡터 관리를 통해 드리프트를 상쇄하는 보수적 접근을 추구한다.

2.2 소스 코드 레벨 분석

PX4 펌웨어 소스(src/modules/commander/MagCalibration.cpp)에서는 QGroundControl(이하 QGC)의 MAVLink 명령어 MAV_CMD_PREFLIGHT_CALIBRATION를 수신했을 때 자기장 센서 보정 시퀀스를 개시한다. 다수의 측정 데이터 포인트를 수집한 뒤, 최소 제곱법(Least Squares Method) 또는 구면 피팅 알고리즘(Ellipsoid Fitting Algorithm)을 통하여 오프셋 파라미터(CAL_MAGx_XOFF, YOFF, ZOFF) 및 대각 스케일 팩터(CAL_MAGx_XSCALE, YSCALE, ZSCALE)를 계산한다.

또한 비직교성을 포함한 완전한 타원체 보정을 위해 비대각 요소(Off-diagonal elements)를 켤 수 있도록 CAL_MAG_COMP_TYP 설정이 존재한다.

// src/modules/sensors/vehicle_magnetometer/VehicleMagnetometer.cpp 의 일부 (PX4 아키텍처 예시)
void VehicleMagnetometer::Run()
{
    // 1. 원시 자기장 uORB 토픽 수신
    sensor_mag_s sensor_mag;
    if (_sensor_mag_sub.update(&sensor_mag)) {
        
        // 2. 경자성(Hard Iron) 오프셋 제거
        Vector3f mag_raw{sensor_mag.x, sensor_mag.y, sensor_mag.z};
        mag_raw -= _offset; 

        // 3. 연자성(Soft Iron) 및 비직교성 보정 변환 행렬 적용
        Vector3f mag_calibrated = _scale_matrix * mag_raw;

        // 4. 보정된 자기장 벡터 uORB 토픽 퍼블리싱 (차후 EKF2 전달용)
        vehicle_magnetometer_s vehicle_mag{};
        vehicle_mag.magnetometer_ga[0] = mag_calibrated(0);
        // ... (생략)
        _vehicle_magnetometer_pub.publish(vehicle_mag);
    }
}

3. 센서 데이터 처리 및 ROS2/MAVLink 연동 아키텍처

지자기 센서에서 측정되어 보정 매트릭스를 거친 신호는 uORB 토픽 vehicle_magnetometer로 발행된다. 이 토픽은 내부 Estimator(EKF2)로 전달됨과 동시에 uXRCE-DDS 클라이언트를 통해 ROS2(Robot Operating System 2) 환경으로 브릿징된다.

  • MAVLink 제어 및 로깅: MAVLink 프로토콜의 SCALED_IMU 패킷 시스템이나 HIGHRES_IMU 메시지의 xmag, ymag, zmag(단위: Gauss 또는 Tesla) 데이터로 변환되어 지상 관제 시스템(GCS)으로 무선 송신된다.
  • ROS2 연동(uXRCE-DDS): ROS2의 센서 메시지 타입 sensor_msgs/msg/MagneticField로 변환된다. 자율 비행 에이전트 개발 시, 보정된 지자기 데이터를 Subscribing하여 SLAM (Simultaneous Localization and Mapping) 또는 VIO (Visual Inertial Odometry)의 헤딩 보정 트리거로 응용할 수 있다.

아래는 센서 데이터가 수집되어 보정을 거치고, 최종 클라이언트(QGC 및 ROS2)까지 도달하는 데이터 흐름도이다.

4. QGroundControl (GCS) 설정 및 구성 (Configuration)

QGroundControl(v4.3.x) 환경에서 지자기 센서의 비직교성 오차 및 왜곡을 수학적으로 보상하기 위해 다음 단계가 수행된다.

  1. Sensors Setup Menu 진입: 기체를 PC와 USB 또는 텔레메트리 기반으로 연결한 뒤 “Compass” 탭에 진입한다.
  2. 6축 컴파스 캘리브레이션(Six-Axis Compass Calibration): 기체를 X, Y, Z, -X, -Y, -Z 축 방향으로 천천히 회전시킨다. 이 때, 화면에 나타난 구(Sphere) 주변으로 초록색 포인트들이 일정 밀도로 기록된다.
  3. 데이터 피팅 및 파라미터 적용: 수집이 완료되면 내부 알고리즘이 앞서 설명한 타원체 피팅 알고리즘(Ellipsoid Fitting)을 수행하여 경자성/연자성 왜곡 계수를 산출하고 재시동한다.
  4. 관련 파라미터(Parameters):
  • CAL_MAG0_ID: 첫 번째 지자기 센서의 고유 ID (보정 무결성 검증 용도)
  • CAL_MAG0_ROT: 기체 대비 센서의 각도 오리엔테이션 행렬(R 행렬 설정 매개변수)
  • CAL_MAG0_XOFF, YOFF, ZOFF: 산출된 Hard Iron 보정 값
  • CAL_MAG0_XSCALE, YSCALE, ZSCALE: 산출된 Soft Iron 보정 값

5. 결론 및 공학적 시사점

결과적으로 지자기 센서의 비직교성과 자성 왜곡 계수를 정확한 수학적 모델 기반으로 해소하지 않으면, 투영 벡터의 성분 불일치로 인하여 방위각 계산에서 큰 오차가 발생하게 된다. 이는 다중 센서 융합 시스템(EKF2)의 mag_innovation (자력계 예측치와 실제치 사이의 오차) 수치를 급증시키며, 심각한 경우 시동 거부(Arming Denial) 또는 비행 중 기체 요동(Toilet Bowl Effect) 현상으로 귀결될 수 있다.

관련 분야의 학술적 연구인 “A geometric approach to strapdown magnetometer calibration in sensor frame” 논문 등에서도 제시하듯, 고정 자성 노이즈와 프레임의 왜곡 행렬을 분리하여 추정하는 메커니즘은 신뢰성 있는 비행 및 군집(Swarm) 제어 임무의 핵심 전제조건(Pre-condition)이다.
