43.5.2. 자동 시동 해제(Auto-Disarm)를 위한 커널 타이머(Kernel Timer) 콜백 메커니즘 및 시간 지연 상수에 따른 상태 천이 방정식

무인항공기(UAV)의 시동(Arming) 상태는 프로펠러가 회전할 수 있는 매우 위험한 상태를 의미한다. 조종자의 실수나 예기치 못한 통신 단절 등으로 인해 비행 종료 후에도 기체가 계속 시동 상태를 유지한다면 심각한 안전사고로 이어질 수 있다. 이를 방지하기 위해 PX4-Autopilot은 착륙 직후 또는 이륙 전 장기 대기 상태에서 시스템을 자동으로 안전 상태로 되돌리는 자동 시동 해제(Auto-Disarm) 메커니즘을 내장하고 있다.

본 절에서는 자동 시동 해제가 활성화되는 PX4 운영체제(NuttX RTOS) 수준의 커널 타이머(Kernel Timer) 콜백(Callback) 구동 원리와, 시간 지연 상수(Time Delay Constant)에 따른 상태 천이(State Transition) 방정식을 분석한다.

1. 커널 타이머(Kernel Timer) 콜백 메커니즘 및 HRT 아키텍처

PX4-Autopilot 시스템은 펌웨어 내부에서 고해상도 타이머(High-Resolution Timer, HRT)를 사용하여 정밀한 스케줄링 및 이벤트 알림을 처리한다.

자동 시동 해제 트리거는 Commander 모듈에서 관리된다. 백그라운드 태스크나 커널 타이머 인터럽트를 통해 지속적으로 “시동 이후 이륙하지 않은 시간” 또는 “착륙 후 경과된 시간“을 카운트하며, 임계 시간에 도달하면 콜백(Callback) 함수를 호출하여 즉각적으로 시동 해제를 명령한다.

NuttX RTOS 기반에서 이러한 타이머는 hrt_call 인프라스트럭처나 Work Queue 메커니즘을 통해 구현된다. 상태 감지 로직(예: Land Detector)이 ’착륙(Landed)’을 선언하면 타이머 레지스터가 시작되고, 일정 틱(Tick)이 지나면 비동기적으로(Asynchronously) disarm() 로직이 포함된 콜백 함수가 실행된다. 이러한 커널 레벨 디자인은 다른 사용자 레벨 애플리케이션의 부하(Load)와 무관하게 안전 관련 기능이 확정적인(Deterministic) 시간 안에 실행됨을 보장한다.

2. 시간 지연 상수(Time Delay Constant) 및 관련 파라미터

자동 시동 해제 로직은 기체의 운용 시나리오에 따라 두 가지 시간 지연 상수를 파라미터로 나누어 관리한다. QGroundControl(QGC)에서 사용자는 이 상수들을 튜닝하여 비행 특성에 맞게 자동 시동 해제 시점을 조절할 수 있다.

  1. 착륙 후 자동 시동 해제 지연 상수 (COM_DISARM_LAND)
  • 설명: 기체가 지면에 착륙(Landed)한 상태로 판별된 이후, 완전히 시동을 끄기까지 대기하는 시간이다.
  • 기본값: 일반적으로 2초 (2.0s) 내외이다.
  • 역할: 착륙 직후 기체가 지면에서 튕기는 현상(Bouncing)이나 재이륙(Touch and Go)을 수행할 여지를 확보하기 위한 버퍼(Buffer) 시간을 제공한다.
  1. 비행 전 대기 자동 시동 해제 지연 상수 (COM_DISARM_PRFLT)
  • 설명: 시동(Arm) 명령이 인가되었으나, 스로틀(Throttle)이 올라가지 않아 이륙하지 않은 상태로 대기할 수 있는 최대 시간이다.
  • 기본값: 일반적으로 10초 (10.0s) 내외이다.
  • 역할: 조종자가 시동을 걸었으나 이륙을 주저하거나 조작을 잊었을 때, 기체가 스스로 시동 해제를 수행하여 안전을 확보한다.

3. 상태 천이 방정식(State Transition Equation) 및 조건 모델링

자동 시동 해제 메커니즘을 이산 사건 시스템(Discrete Event System)으로 모델링할 때, 상태 천이는 지시 함수(Indicator Function)와 타이머 적분값의 관계로 정의될 수 있다.

기체의 시동 상태를 S(t) \in \{0, 1\} (0: Disarmed, 1: Armed)라 하자.
비행 상태 플래그를 비행 중 F(t)=1, 지상 대기(착륙) F(t)=0으로 정의할 때, 착륙 감지 이후의 경과 시간 \tau_{land}와 이륙 전 대기 시간 \tau_{pre}는 다음과 같이 적분된다.

\tau_{land}(t) = \int_{t_{land}}^t (1 - F(u)) \cdot S(u) \, du
\tau_{pre}(t) = \int_{t_{arm}}^t (1 - F(u)) \cdot S(u) \, du

자동 시동 해제를 발생시키는 델타 변이(Transition) 함수는 다음 조건을 만족할 때 S(t) \rightarrow 0 으로 천이한다.

S(t+\Delta t) = \begin{cases} 0 & \text{if } (\tau_{land}(t) > T_{COM\_DISARM\_LAND}) \lor (\tau_{pre}(t) > T_{COM\_DISARM\_PRFLT}) \\ S(t) & \text{otherwise} \end{cases}

즉, 착륙 상태이거나 시동 후 비행 전 상태일 때, 타이머 카운터가 지정된 파라미터 상수 임계점을 초과하는 즉시 커널 콜백이 Trigger되어 S(t)를 0으로 덮어쓴다.

4. 자동 시동 해제 콜백 상태 머신 다이어그램

아래 다이어그램은 PX4 Commander 모듈 내에서 타이머 콜백을 기반으로 자동 시동 해제를 수반하는 시스템의 상태 머신(State Machine) 흐름을 나타낸다.

stateDiagram-v2
    [*] --> Disarmed : 전원 인가 및 초기화
    
    Disarmed --> Armed : 시동 명령 (Arm Command)\n/ Start Pre-flight Timer
    
    state Armed {
        [*] --> Pre_Flight_Wait : 이륙 대기
        
        Pre_Flight_Wait --> Flying : 스로틀 증가 및 이륙\n/ Stop Pre-flight Timer
        Pre_Flight_Wait --> Auto_Disarm_Trigger : Timer > COM_DISARM_PRFLT
        
        Flying --> Landed_Wait : 지면 접촉 (Landed 감지)\n/ Start Land Timer
        
        Landed_Wait --> Flying : 재이륙 (Takeoff)\n/ Stop Land Timer
        Landed_Wait --> Auto_Disarm_Trigger : Timer > COM_DISARM_LAND
    }
    
    Auto_Disarm_Trigger --> Disarmed : 커널 타이머 콜백(HRT Call) 실행

5. 아키텍처 구현 및 예외 처리

QGroundControl(QGC)과 같은 관제 소프트웨어가 MAVLink 패킷을 통해 시스템을 제어할 때, 자동 시동 해제로 인해 기체가 안전 상태로 돌아가면 PX4는 HEARTBEAT 메시지의 base_mode 플래그에서 MAV_MODE_FLAG_SAFETY_ARMED 비트를 해제하여 시스템의 상태 변화를 GCS에 즉각 보고한다.

GCS 개발자는 예기치 않은 시스템 Auto-Disarm이 발생했을 때 (예: 자동 임무 수행 중 장기 대기로 인한 시동 꺼짐 등) 이 상태 천이를 빠르게 감지하고 사용자에게 경고(UI Alert)를 표출하거나 후속 대응 명령을 재전송하도록 관제 통신 로직을 설계해야 한다.
