43.3.2. 이종(Heterogeneous) 다중 센서 다중화(Redundancy) 시스템의 비동기 샘플링 일치성 판별 및 베이즈 확률(Bayesian Probability) 기반 가중치 투표(Weighted Voting) 메커니즘

최신 픽스호크(Pixhawk) 기반의 비행 제어기(Flight Controller) 하드웨어는 신뢰성 및 안전성 확보를 위해 서로 다른 제조사와 모델의 센서를 혼합 구성하는 이종(Heterogeneous) 다중 센서 다중화(Redundancy) 아키텍처를 채택하고 있다(예: InvenSense, Bosch, ADI 등의 IMU 혼용). PX4-Autopilot(기준 펌웨어 버전: v1.14.x)의 Sensors 모듈과 EKF2는 시동(Arming) 전후에 이 다중 센서 시스템으로부터 들어오는 비동기적(Asynchronous) 샘플링 데이터를 수합하여 일관성을 판별하고, 어떤 센서를 주(Primary) 센서로 선택할지 동적으로 결정하는 메커니즘을 내장하고 있다.

1. 이종 센서의 비동기 샘플링 일치성 판별 로직

서로 다른 칩셋(Chipset)은 각기 다른 내부 클럭을 가지고 있으므로, uORB(Micro Object Request Broker) 통신망으로 퍼블리시(Publish)되는 센서 데이터의 타임스탬프(Timestamp)와 샘플링 레이트(Sampling Rate)는 일치하지 않는 비동기적 특성을 보인다. PX4는 시동(Arming) 전 전제 조건 검사 시 이러한 이종 센서 간 데이터 일치성을 판별한다.

1.1 시간 동기화 및 보간(Interpolation)

EKF2는 관측 데이터(Observation Data)의 지연 시간(Time Delay)을 보상하기 위해 원형 버퍼(Ring Buffer)를 유지한다. 비동기 샘플링된 이종 IMU 데이터가 들어오면, 시간 동기화를 위해 버퍼 내에서 타임스탬프 기반의 선형 보간법(Linear Interpolation)을 수행하여 동일한 가상 시간 평면상에서 센서값을 배열한다.

1.2 센서 간 분산(Variance) 및 이탈도(Inconsistency) 측정

동일한 타임라인에 정렬된 다수의 센서 데이터(예: 3개의 가속도계)에 대해 상호 간 오차 벡터 평음을 산출한다. PX4의 Sensors 스레드 내 Voter 로직은 각 센서의 측정값 x_i와 전체 센서의 가중 평균치 \bar{x} 사이의 유클리드 거리(Euclidean Distance)를 지속해서 계산한다. 이 편차가 시스템 파라미터로 설정된 임계치(예: COM_ARM_IMU_ACC 또는 SENS_IMU_ACC_DIF)를 일정 시간 이상 초과할 경우 “Inconsistent Accelerometers” 경고를 송출하며 시동을 차단한다.

2. 베이즈 확률(Bayesian Probability) 기반 센서 가중치 투표(Weighted Voting) 메커니즘

센서 간 불일치가 발생하거나 특정 센서에 백색 잡음(White Noise) 혹은 진동이 과도하게 인입되었을 때, 어떤 센서를 신뢰할 것인지 혹은 어느 비율로 융합할 것인지 결정하는 메커니즘이 필요하다.

2.1 EKF2 다중 인스턴스 구조와 베이즈 확률 가중치

PX4 v1.13.x 이상부터 EKF2는 사용 가능한 IMU 수만큼 여러 개의 독립적인 추정기 인스턴스(Multi-instance)를 동시에 실행한다. 각 인스턴스는 자신의 이노베이션(Innovation) 공분산 \mathbf{S}를 산출한다.
베이즈 확률(Bayesian Probability) 정리에 입각하여 센서 예측치와 실제 측정치가 얼마나 잘 맞아떨어지는지를 바탕으로 각 EKF2 인스턴스의 상대적 신뢰도 척도인 테스트 비율(Test Ratio)을 계산하게 된다.

2.2 가중치 기반 블렌딩(Weighted Blending) 및 원활한 전환(Seamless Switch)

각 EKF 인스턴스에서 출력되는 위치, 속도, 자세 상태 벡터 추정치 \mathbf{\hat{x}}_i는 해당 인스턴스의 신뢰도에 비례하는 가중치 w_i를 곱하여 다음과 같이 메인 상태 벡터로 블렌딩된다.

\mathbf{\hat{x}}_{blended} = \sum_{i=1}^{N} w_i \mathbf{\hat{x}}_i

만약 주(Primary) EKF 인스턴스의 센서가 고장(Fault) 혹은 극심한 하드웨어 클리핑(Clipping)으로 인해 확률적 신뢰도가 급감하면 w_i가 0에 수렴하게 되며, 가장 신뢰도 높은 대체 인스턴스의 가중치가 1로 부상하게 된다. 커맨더(Commander) 모듈은 이러한 EKF 인스턴스 스위칭이 단기간 내 다수 발생하면 시스템의 심각한 센서 문제를 알리고 시동을 억제한다.

4. Ardupilot 모델비교 : 차단(Exclusion) 방식 대 확률적 블렌딩(Blending) 방식

PX4와 Ardupilot 모두 다중 센서와 다중 EKF를 지원하지만 접근 모델에서 차이를 보인다.

  • PX4-Autopilot: 다수 EKF 인스턴스의 출력을 동적 가중치(Dynamic Weighting)를 사용하여 실시간으로 섞는 블렌딩(Blending) 방식을 활용한다. 이를 통해 에러 발생 시 상태량이 튀어버리는(Jump) 현상을 최소화한다.
  • Ardupilot: 기본적으로 각 센서별 EKF 코어(EKF3) 중 주(Primary) 코어 하나만을 제어 기준으로 사용하고, 해당 코어의 혁신율(Innovation Ratio)이 파라미터 임계치를 넘어가 복구 불가능으로 간주될 경우 다음 코어로 전환(Lane Switch 혹은 Fallback)을 시도하는 배타적 선택(Exclusive Selection) 방식을 주로 채택한다.
