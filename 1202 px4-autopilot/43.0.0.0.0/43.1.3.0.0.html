43.1.3 펄스 폭 변조(PWM) 듀티 사이클 및 DShot 프로토콜의 양자화(Quantization)된 유휴 대역폭과 제어 활성화 상태의 신호 스펙트럼 전이 특성

무인항공기 시스템(UAS)의 제어 명령은 비행 제어기(Flight Controller)에서 생성되어 모터 변속기(ESC, Electronic Speed Controller) 또는 서보 모터로 전달된다. 이 과정에서 시동(Arming) 전후의 상태 전환은 하드웨어 신호 레벨에서 극적인 스펙트럼 전이(Spectrum Transition)를 수반한다. 본 장에서는 전통적인 펄스 폭 변조(PWM)와 최신 디지털 통신 프로토콜인 DShot의 신호 양자화 특성을 분석하고, 제어 활성화(시동) 시 발생하는 유휴 대역폭(Idle Bandwidth)의 물리적 변화를 설명한다.

1. 펄스 폭 변조(PWM) 기반 제어 신호의 아날로그적 한계

PWM은 일정한 주기(Period, 통상 400Hz 또는 50Hz) 내에서 High 상태가 유지되는 시간(Duty Cycle)을 조절하여 제어량을 전달하는 방식이다.

1.1 시동 및 시동 해제 시의 PWM 스펙트럼 특성

  • 시동 해제(Disarmed): PX4에서 시동이 해제된 상태의 기본 PWM 출력은 일반적으로 900µs (파라미터 PWM_MAIN_DISARM)로 설정된다. 이 신호는 ESC에 “어떠한 추력도 발생시키지 마라“는 명시적인 휴지(Dormant) 명령으로 작용하며, ESC는 이 신호를 인식하여 모터의 3상 전력을 완전히 차단한다.
  • 시동(Armed) 및 유휴(Idle) 상태: 기체에 시동이 걸리면 출력은 즉시 1000µs (또는 PWM_MAIN_MIN 파라미터 값) 부근으로 도약(Transition)한다. 멀티로터의 경우 공중에서의 기체 제어 상실(Desync)을 방지하기 위해 프로펠러가 아주 느리게 회전하는 아이들링(Idling) 상태를 유지한다.

이러한 PWM 방식은 마이크로컨트롤러 타이머의 해상도에 따라 연속적인 아날로그적 성격을 띠지만, **신호 모호성(Signal Ambiguity)**과 **지연(Latency)**이라는 치명적인 단점을 지닌다. 특히 오실로스코프로 관측 시 시동 상태로 넘어가는 전이 구간에서 듀티 사이클이 펄스 폭의 지터(Jitter)에 취약해지는 아날로그 대역폭의 한계를 노출한다.

2. DShot 프로토콜: 양자화(Quantization)된 디지털 제어

PWM의 아날로그적 한계를 극복하기 위해 도입된 DShot(Digital Shot) 프로토콜은 제어 신호를 완전히 양자화된(Quantized) 디지털 비트 스트림으로 변환한다. DShot은 초당 150kbit(DShot150)에서 1.2Mbit(DShot1200)의 전송 속도를 가지며, 각 패킷은 16비트(11비트 스로틀 데이터 + 1비트 텔레메트리 요청 + 4비트 CRC)로 구성된다.

2.1 디지털 양자화와 유휴 대역폭(Idle Bandwidth)

DShot 프로토콜 체계에서 제어 값은 0부터 2047까지의 이산적(Discrete)인 정수로 완벽히 양자화된다.

  • 값 0: 시동 해제(Disarmed) 또는 모터 정지.
  • 값 48 ~ 2047: 실질적인 스로틀 제어 범위. (1~47은 특수 명령어로 예약됨)

DShot의 가장 큰 특징은 유휴 대역폭의 결정론성이다. 시동이 해제된 상태(Disarmed)에서는 단순히 0이라는 디지털 패킷이 일정한 주파수로 반복 전송된다. 노이즈가 유입되더라도 CRC(순환 중복 검사)를 통해 유효하지 않은 패킷은 엄격히 폐기되므로, 모터가 오작동으로 회전할 확률이 0에 가깝다.

3. 제어 활성화(Arming) 시의 신호 스펙트럼 전이(Transition) 특성

시동 해제 상태에서 시동(Arming) 상태로 전환될 때, PWM과 DShot은 각기 다른 통신망 스펙트럼 전이 양상을 보인다.

sequenceDiagram
    participant FC as Flight Controller (PX4)
    participant ESC as Motor ESC
    
    Note over FC,ESC: [Disarmed State]
    FC->>ESC: PWM: 900µs 펄스 전송 / DShot: '0' 밸류 패킷 연속 전송
    
    Note over FC: User triggers Arming Command
    FC->>FC: State Machine: Transition to Armed
    
    Note over FC,ESC: [Spectrum Transition Phase]
    FC->>ESC: PWM: 타이머 레지스터 변경 (900µs -> 1000µs)
    FC->>ESC: DShot: 디지털 워드 변경 ('0' -> '48' 이상 Idle RPM)
    
    Note over FC,ESC: [Armed State (Idle)]
    FC->>ESC: 명령된 유휴 속도(Idle Speed) 유지

3.1 DShot 제어 활성화에 따른 대역폭 활용(Bidirectional DShot)

PX4-Autopilot에서 DShot을 사용할 경우, 시동(Arm) 시 단순히 일방적인 출력 전이만 발생하는 것이 아니다. 양방향 DShot(Bidirectional DShot) 기능이 활성화된 상태라면, 제어기가 ESC 코일의 역기전력(Back-EMF)을 기반으로 측정한 모터 RPM(회전수) 데이터를 텔레메트리 대역폭을 통해 반환받기 시작한다.

즉, 시동(Arm) 이벤트는 제어 루프의 대역폭을 수직적으로 확장시키는 기폭제가 된다. 제어 활성화 상태로 전이됨과 동시에 ESC로부터 실시간 ERPM(Electrical RPM) 피드백이 들어오며, PX4 내부의 동적 노치 필터(Dynamic Notch Filter)는 이 주파수 스펙트럼 데이터를 기반으로 모터의 고조파 진동(Harmonic Vibration)을 실시간으로 추적 및 상쇄하는 고도의 폐루프(Closed-loop) 제어로 전환된다.

4. 결론

무인항공기 제어에서 시동(Arming) 행위는 단순히 상태 지시등을 바꾸는 것이 아니다. 펄스 폭 변조(PWM) 체계에서는 연속 시간 대역에서의 듀티 사이클의 거시적인 점프(Jump)로 나타나며, DShot 환경에서는 양자화된 디지털 공간에서의 결정론적 값의 이동 및 양방향 텔레메트리 대역폭의 개방으로 이어진다. Ardupilot과 PX4 공히 최신 기체 설정에서는 아날로그 지터와 노이즈에 취약한 PWM보다 확실한 신호 스펙트럼 전이와 CRC 무결성 검증을 제공하는 DShot 프로토콜 채택이 필수로 자리잡고 있다.
