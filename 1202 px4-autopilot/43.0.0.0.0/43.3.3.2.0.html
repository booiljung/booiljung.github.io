43.3.3.2 혁신량(Innovation) 벡터 기반 마할라노비스 거리(Mahalanobis Distance) 연산을 통한 센서 이상치(Outlier) 배제 및 상태 융합 거부 로직

PX4-Autopilot의 상태 추정기(Estimator)인 EKF2(Extended Kalman Filter 2)는 다수의 센서 데이터를 융합(Fusion)하여 기체의 현재 상태(자세, 속도, 위치 등)를 추정한다. 이 과정에서 센서의 고장, 일시적인 노이즈 결함, 환경적 요인(예: 자기장 간섭 또는 GPS 다중 경로 오차)으로 인해 물리적으로 타당하지 않은 비정상 데이터(Outlier)가 유입될 수 있다.

이러한 이상치가 칼만 필터의 상태 업데이트에 반영되면 추정 결과가 발산(Divergence)하여 비행 중 치명적인 사고로 이어질 수 있다. 이를 방지하기 위해 PX4 EKF2는 혁신량(Innovation) 벡터와 마할라노비스 거리(Mahalanobis Distance) 기반의 통계적 검정(Statistical Gating) 기법을 사용하여, 유효한 센서 데이터만을 융합하고 비정상 데이터는 융합을 거부(Rejection)하는 메커니즘을 적용한다. (참고: PX4 v1.14 및 최신 메인 브랜치 기준)

1. 혁신량(Innovation)과 혁신 공분산(Innovation Covariance)의 역학적 의미

칼만 필터의 상태 추정 과정은 예측(Prediction)과 보정(Correction/Update)의 두 단계로 나뉜다. 혁신량(\nu)은 센서로부터 관측된 **실제 측정값(z)**과 현재까지의 추정 상태를 바탕으로 예측한 기대 측정값(\hat{z}) 간의 차이로 정의된다.

\nu = z - \hat{z}

  • 혁신량(Innovation) 벡터 (\nu): 현재 센서 관측치가 기존의 시스템 모델 추정치를 얼마나 ‘새롭게’ 갱신할 것인지를 나타내는 벡터이다.
  • 혁신 공분산(Innovation Covariance) (S): 혁신량의 통계적 불확실성을 나타내는 행렬로, 예측 상태 공분산(P), 관측 행렬(H), 그리고 센서 노이즈 공분산(R)의 조합으로 계산된다 (S = HPH^T + R).

만약 특정 센서에서 일시적인 스파이크 노이즈가 발생했다면, 해당 센서 데이터로 구한 혁신량 벡터의 크기는 비정상적으로 커지게 된다. 그러나 혁신량의 절대적인 크기만으로는 해당 센서가 실제로 고장인지 판별하기 어렵다. 왜냐하면 각 센서마다 가진 고유의 측정 단위와 분산(Variance) 특성이 다르기 때문이다.

2. 마할라노비스 거리(Mahalanobis Distance)를 이용한 통계적 일치성 검정

이러한 이종 센서(IMU, GPS, 기압계 등) 간의 스케일 차이 및 불확실성 편차를 정규화(Normalization)하여 통계적 거리를 측정하는 수학적 도구가 바로 **마할라노비스 거리(Mahalanobis Distance)**이다. 단일 차원에서는 단순한 분산 분의 1로 스케일링 된 편차의 제곱이지만, 다차원 벡터의 경우 다음과 같이 정의된다.

D_M^2 = \nu^T S^{-1} \nu

여기서:

  • D_M^2 : 마할라노비스 거리의 제곱
  • \nu : 혁신량 행렬(벡터)
  • S^{-1} : 혁신 공분산의 역행렬

PX4 EKF2 소스 코드 내부에서는 이 값을 직접 검정 통계량(Test Statistic)으로 사용하여 카이제곱 분포(Chi-squared distribution) 기반의 가설 검정(Hypothesis Testing)을 수행한다. 즉, 혁신량이 기설정된 수학적 신뢰 구간 내에 들어오는지 판단하는 것이다.

1.1 PX4 소스 코드 내 Test Ratio 연산 구조

PX4에서는 계산상의 효율을 위해 마할라노비스 거리 제곱값을 사전에 정의된 임계값(Gate Size, 보통 \sigma 단위로 설정됨)의 제곱으로 나눈 값인 **테스트 비율(Test Ratio)**을 활용한다.

\text{Test Ratio} = \frac{\nu^2}{S \times (\text{Gate Size})^2}
(계산 편의상 1차원 업데이트로 분리될 때의 스칼라 수식 예시)

  • Test Ratio \le 1.0 : 측정값이 유효하다고 판단하여 EKF 상태 보정(Fusion)에 반영함 (승인).
  • Test Ratio > 1.0 : 측정값이 통계적 허용 범위를 벗어난 이상치(Outlier)로 판단하여 상태 보정에 사용하지 않음 (거부/Rejection).

이 과정은 src/modules/ekf2/EKF/ 내부의 각 센서 융합 메서드(예: fuseVelPosHeight, fuseMag 등) 내에서 혁신량 일관성 검사(Innovation Consistency Check)라는 이름으로 수행된다.

3. 센서 게이트(Gate) 파라미터 및 튜닝 전략

PX4 파라미터(Parameter) 시스템은 센서 데이터 융합의 허용 오차폭을 조정하기 위해 게이트 사이즈 파라미터를 제공한다. (파라미터 접두사: EKF2_)

  • EKF2_GPS_V_GATE: GPS 속도 측정 혁신량의 게이트. (기본값: 보통 5.0)
  • EKF2_GPS_P_GATE: GPS 위치 측정 혁신량의 게이트. (기본값: 보통 5.0)
  • EKF2_BARO_GATE: 기압계 고도 측정 혁신량의 게이트.
  • EKF2_MAG_GATE: 지자기 센서 혁신량의 게이트.

게이트 사이즈가 크면 허용 오차 범위가 넓어져 센서 노이즈나 일시적 외란에도 융합이 잘 깨지지 않지만, 반대로 센서 결함이 EKF에 침투할 위험이 커진다. 반면 게이트 사이즈가 작으면 안정성은 높아질 수 있으나, 정상적인 동적 기동 상태(예: 급격한 선회)에서 센서 측정을 이상치로 오판하여 융합이 자주 끊기는 거절(Rejection) 현상이 발생할 수 있다.

따라서 비행 로그(Log) 분석 프로그램(예: Flight Review)을 통해 각 센서별 test_ratio 변수(예: estimator_innovations uORB 메시지 내 mag_innov_test_ratio, vel_pos_innov_test_ratio 등)를 확인하여, 정상 비행 시 test_ratio가 1.0을 넘지 않도록 파라미터를 맞춤 튜닝하는 것이 공학적으로 매우 중요하다.

4. 상태 융합 거부(Fusion Rejection) 및 타임아웃(Timeout) 로직

마할라노비스 거리 검정을 통과하지 못한 센서 데이터는 즉각적으로 융합 거부(Fusion Rejection) 처리가 되어, 칼만 필터의 상태 업데이트 수식에 반영되지 않는다.

하지만, 이러한 거부 상태가 장기화될 경우에는 문제가 복잡해진다. 만약 IMU 예측값이 실제 위치(진값)에서 점점 멀어지는 편향을 가져서 실제 올바른 GPS 값이 지속적으로 거부(Reject)되는 상황이라면 어떨까?

PX4-Autopilot은 이와 같은 극단적인 교착 상태(Deadlock)를 방해하기 위해 타임아웃 메커니즘을 적용한다.

  1. Rejection 카운팅: 센서 융합이 거부될 때마다 내부 상태 플래그(예: innov_check_fail_status)를 활성화하고 시간을 측정한다.
  2. 센서 타임아웃(Timeout): 거부 상태가 특정 시간(EKF2_REQ_X_TO 등의 파라미터로 정의) 이상 지속되면, EKF는 해당 측정치를 더 이상 단순 결함으로 보지 않고, 내부 모델(IMU)에 심각한 오류가 있거나 외부 물리적 제약상 상태 공간이 크게 어긋났다고 간주한다.
  3. 상태 초기화 및 리셋: 이 경우 EKF는 거부되었던 측정값(예: 최근 GPS 위치값)을 강제로 사용하여 EKF의 현재 상태 변수(State)와 혁신 공분산 행렬을 강제 초기화(Reset) 시킨다.

5. Ardupilot 대비 차이 (Architectural Differences)

Ardupilot 측정 이상치 처리 방식 또한 마할라노비스 거리 측정 및 이노베이션 게이팅(Innovation Gating)을 차용한 EKF3 아키텍처를 지니고 있다. 기본적인 선형 대수학적 검증 메커니즘은 궤를 같이 하지만 디테일에서 차이가 있다.

  • 상태 추정기 우선순위 레이어: Ardupilot은 PX4의 단일 코어 기반 EKF2 멀티 인스턴스(Multi-instance) 병렬화 정책(EKF2_MULTI_IMU/MAG)에 비견되는 다중 EKF 레인(Lanes) 시스템을 사용한다. 하나의 센서 스트림에서 지속적으로 Gating 실패 현상이 발생할 경우, 자체적인 Affinity Score(정확도 평가 점수)를 통해 가장 건강한 센서 데이터를 소비하는 다른 EKF 레인으로 빠르게 Lane Switch를 단행하는 로직이 PX4보다 더 전위적으로 강조된다.
  • 파라미터 제원 구조화: 두 플랫폼 모두 Gating Limit 값을 설정할 수 있지만, PX4가 EKF2_MAG_GATE 같은 직관적인 단일 Gate 분산을 파라미터화 해둔 반면에 Ardupilot의 EK2_ 또는 EK3_ 게이트 파라미터 셋은 각각의 센서 종류별로 보다 복잡한 노이즈 모델 파라미터 간 상대적인 비율값에 의존하여 Rejection 타임아웃 계수를 세밀하게 조율하는 경향이 크다.

결론적으로 PX4의 마할라노비스 거리 기반 이상치 배제 로직은 단순한 임계값 초과 검색(Threshold Checking)이 아닌, 확률 통계적 관점에서 각 센서의 불확실성을 가로지르는 최적의 신뢰성 검증 필터라 할 수 있다.
