43.6.1.1 수락(ACCEPTED), 일시적 거부, 영구 거부의 논리 분기 트리에 따른 GCS 백엔드 콜백(Callback) 인스턴스 할당 구조

본 절에서는 QGroundControl(QGC, v4.3.0 기준)과 PX4-Autopilot(v1.14.0 기준) 통신 환경에서, COMMAND_ACK 응답의 세 가지 주요 결과(수락, 일시적 거부, 영구 거부)에 대응하기 위한 GCS 백엔드의 콜백(Callback) 인스턴스 할당 구조와 논리 분기 트리 아키텍처를 분석한다.

1. 통신 비동기성과 콜백(Callback) 인스턴스 할당

지상 관제 시스템(GCS)이 PX4 기체로 제어 명령을 전송할 때, QGC 백엔드의 Vehicle 클래스는 비동기 메서드(예: sendMavlinkCommand)를 호출한다. 이때 응답을 처리하기 위해 C++ 람다(Lambda) 함수나 함수 포인터 형태의 콜백 인스턴스를 메모리에 힙(Heap) 할당하고 응답 대기 큐(Queue)에 등록한다. 이 인스턴스는 COMMAND_ACK 패킷이 수신될 때까지 명령의 파라미터 컨텍스트(Context)와 타임아웃(Timeout) 타이머를 캡처하여 유지한다.

2. 논리 분기 트리(Logical Branching Tree)의 구조

기체로부터 COMMAND_ACK 프로토콜의 결과(Result) 멤버를 파싱하여 수신하면, 등록된 콜백 함수가 실행되며 다음과 같은 3가지 주요 분기 트리를 거친다.

  1. 수락 (MAV_RESULT_ACCEPTED)
  • 기체가 스케줄러 내에서 명령을 성공적으로 실행하였음을 의미한다.
  • 콜백 로직: 콜백 인스턴스는 성공 시그널(Signal)을 방출(Emit)하여 QML 프론트엔드의 사용자 인터페이스(UI)를 비동기 갱신한다(예: ‘비행 중’ 상태 표시). 이후 해당 콜백 인스턴스는 메모리에서 즉시 해제(Deallocation)되며, 해당 명령에 할당되었던 타임아웃 타이머도 정지된다.
  1. 일시적 거부 (MAV_RESULT_TEMPORARILY_REJECTED)
  • 센서 초기화 중이거나 선행 로직(예: EKF2 얼라인먼트)이 완료되지 않아 현재는 명령을 수행할 수 없으나, 추후 성공할 상태 공간의 여지가 있는 경우이다.
  • 콜백 로직: 콜백 인스턴스는 즉시 소멸되지 않고, 내부의 재전송 횟수 카운터(Retry Counter)를 증가시킨다. 지정된 지연 시간(Exponential Backoff) 이후에 명령을 텔레메트리로 재전송하며, 펌웨어에 정의된 최대 재시도 횟수를 초과할 경우에만 영구 거부로 상태를 강제 전환한다.
  1. 영구 거부 (MAV_RESULT_DENIED / MAV_RESULT_UNSUPPORTED)
  • 하드웨어 체결 불량, 비행 전 점검(Pre-flight Check) 실패, 혹은 현재 비행 모드에서 원천적으로 지원하지 않는 명령으로 인해 명령을 절대 수행할 수 없는 상태이다.
  • 콜백 로직: 콜백 인스턴스는 즉시 실패(Fault) 처리를 수행하고, 사용자에게 명시적인 에러 메시지(추가 수신되는 STATUSTEXT 파싱 결과 연동)를 렌더링하도록 프론트엔드 모듈에 뷰 모델(ViewModel) 이벤트를 전달한다. 처리가 끝난 인스턴스는 콜백 대기열 큐에서 소멸된다.

3. GCS 백엔드 콜백 분기 처리 다이어그램

다음은 람다 콜백 컨텍스트 안에서 이루어지는 논리 분기 트리를 나타낸 순서도(Flowchart)이다.

4. 아키텍처 구현 상의 특성 (Ardupilot과의 비교)

과거 Ardupilot의 일부 오픈소스 GCS 구현(예: 구버전 Mission Planner 모듈)에서는 개루프(Open-loop) 방식으로 단순히 명령을 던지고(Fire-and-forget) 폴링(Polling) 타이머를 통해 기체의 시스템 상태(System Status) 변화를 수동 감지하는 방식을 혼용하는 경우가 있었다.

반면, 1세대 시스템을 극복하고 최신 PX4 아키텍처의 철학을 완벽히 활용하는 QGC 백엔드 및 MAVSDK는 개별 명령 단위로 고유 식별자(Command ID)와 비동기 콜백을 엄격히 1대1 매핑하는 C++ 기반의 프로미스/퓨처(Promise/Future) 패턴 지향적 설계를 갖추고 있다. 이를 통해 자율 에이전트(Autonomous Agent)가 다중 명령을 쉴새없이 병렬로 전송하더라도, 각 콜백 인스턴스가 독립적인 생명주기(Lifecycle)를 가지며 상태 전이를 스레드로부터 안전(Thread-safe)하게 관리하는 완성된 폐루프(Closed-loop) 제어망을 형성한다.
