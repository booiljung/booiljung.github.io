43.3.4.1 반송파 위상(Carrier Phase) 고정 해(Fix Solution)의 정수 모호성(Integer Ambiguity) 해결 상태 역추적

무인항공기(UAS)의 정밀 농업, 측량, 군집 비행 등 산업용 애플리케이션에서는 미터(m) 급의 오차를 갖는 일반적인 단일 GNSS(코드 위상 기반) 연산만으로는 요구 정밀도를 충족할 수 없다. 센티미터(cm) 단위의 초정밀 측위를 위해 등장한 RTK(Real-Time Kinematic) GPS 시스템은, 위성 신호의 코드가 아닌 반송파 위상(Carrier Phase) 자체를 추적하여 거리를 산출하는 기법을 사용한다.

PX4-Autopilot의 확장 칼만 필터(EKF2)는 UBlox F9P 모듈과 같은 RTK 지원 하드웨어로부터 데이터를 수신할 때, 시스템이 진정한 RTK 수준의 정밀도에 도달했는지를 판단하기 위해 정수 모호성(Integer Ambiguity) 해결 상태를 내부적으로 역추적(Back-tracking)하고 모니터링한다.

1. 정수 모호성(Integer Ambiguity)이란?

위성이 송신하는 GNSS 신호는 사인파 형태의 반송파(파장 길이 약 19cm~24cm)에 실려 전송된다. 수신기는 파장의 한 주기 내에서의 미세한 위상 차이(소수점 단위 파장 측정)는 매우 정밀하게 측정할 수 있지만, 위성에서 수신기까지 뻗어 있는 **전체 파장의 개수(정수형 파장 횟수)**가 몇 개인지는 관측만으로는 즉성에서 알 수 없다. 전체 파장의 정확한 개수(정수)를 알아내지 못하면 반송파 위상을 이용한 절대 거리를 계산할 수 없는데, 이 잃어버린 ’정수값 파장 개수’를 **정수 모호성(Integer Ambiguity)**이라고 부른다.

2. RTK GPS 위치 해(Solution) 상태

정수 모호성을 해결해 나가는 컴퓨터 수치 해석적 과정에 따라 RTK 측위 품질은 크게 세 단계로 분류된다.

  • RTK Float (부동소수점 해): 베이스 스테이션(Base Station)으로부터 보정 신호(RTCM) 데이터는 수신받고 있지만, 아직 수학적 최적화 알고리즘이 완벽한 ‘정수(Integer)’ 파장 개수를 도출하지 못하고 부동소수점 수준의 근사치를 헤매고 있는 상태이다. 단위 오차율은 대략 수십 cm ~ 1 m 안팎에 머문다.
  • RTK Fix (고정 해): 수신기 칩셋 내부 알고리즘이 결국 올바른 정수 개수를 찾아내어(Fix), 위성-수신기 간 거리를 파장의 단위까지 확정 지은 상태이다. 이때의 정밀도는 1~2cm 급으로 도약하며, 우리는 이것을 **반송파 위상 고정 해(Carrier Phase Fix Solution)**라 호칭한다.

3. PX4 내부의 정수 모호성 해결 상태 모니터링 로직

PX4 비행 제어기 내부의 EKF2 로직은 센서 구동 드라이버(src/drivers/gps/) 단에서부터 RTK 상태 변수를 읽어오며 추적한다. uORB 메시지인 sensor_gps.msg 구조체를 살펴보면, fix_type 필드 변수가 존재한다.

  • 0: 픽스 없음 (No Fix)
  • 3: 단일 독립 3D 픽스 (일반 GPS)
  • 5: RTK Float (정수 모호성 해결 실패/진행 중)
  • 6: RTK Fix (정수 모호성 해결 완료)

PX4 EKF2 모듈은 단순히 fix_type == 6임을 관찰하는 데 그치지 않고, EKF 노이즈 공분산(Covariance)과 혁신량(Innovation)을 함께 모니터링하는 **상태 역추적(Back-tracking)**을 수행한다.

커맨더(Commander) 또는 지상 관제 시스템(GCS) 레벨에서 정밀한 지오펜스(Geofence) 돌파 여부나 정밀 착륙을 결정할 때, RTK Fix 상태가 일시적으로 Float으로 떨어지는 롤백(Roll-back) 현상을 탐지하면 커맨더 시스템은 즉각적으로 GPS 융합에 대한 임계값 신뢰도를 강등시킨다.

    E --> F{fix_type 추적 판단}
    F -->|Fix 6->5 상태 강등 인지| G[RTK Float 역추적 트리거]

만약 나무 밑이나 건물 계곡(Urban Canyon) 사이를 지날 때 정수 모호성 해결 알고리즘이 전파 난반사로 인하여 붕괴(Fix -> Float 강등)되면, EKF2는 과거의 Fix 상태에서 누적해 놓았던 위치 바이어스(Position Bias) 보상치와 혁신 변수를 보존한 채로 관측 의존도를 낮춘다. 이것이 PX4가 자랑하는 RTK 상태 역추적(Back-tracking) 기반의 부드러운 위치 감쇠(Graceful Degradation) 로직이다.

4. 관련 설정 파라미터 (Parameters)

PX4 사용자는 임무의 중요도에 따라 RTK 정수 모호성 미해결 상태(Float)에서의 동작 한계를 파라미터로 설정할 수 있다.

  • EKF2_GPS_CTRL: 이 파라미터 비트마스크를 통해 RTK 듀얼 안테나를 사용할지, 어떤 블렌딩 기법을 적용할 지 결정한다.
  • COM_ARM_MAG_STR: 지자기 센서 대신 듀얼 안테나 RTK의 Heading Phase 측정치에만 의존하여 이륙할 경우 통과 조건 등을 정의할 때 연계된다.
  • Arming Check 분기: 일부 정밀도가 필수인 임무에서는 fix_type == 6 (RTK Fix)이 달성되기 전까지는 시동(Arm) 자체를 완전히 무시하게끔 안전 전제 조건을 강화할 수 있다.

5. Ardupilot 대비 차이 (Architectural Differences)

Ardupilot 환경에서도 RTK Fix/Float 상태는 GPS_STATUS 레벨에서 관리되며 EKF3의 Position Innovation 노이즈 스케일링에 즉각 개입한다. 가장 두드러진 차이는 모드 전환 메커니즘에 있다.

  • 포지션 스위칭 철학: Ardupilot은 다수의 멀티 EKF Core 체계를 차용하고 있기에, 만약 GPS 1번이 RTK Fix이고 GPS 2번이 일반 3D Fix일 때, 1번이 어느 순간 Float으로 강등되어 혁신 오차의 공분산이 일반 GPS(Fix)보다 뒤쳐지는 역전 구간이 발생하면 과감하게 코어를 스위칭해버리는 하드-액션(Hard-Action) 레인 전환을 강조한다.
  • 역추적 보상 필터링: PX4 메인 브랜치의 EKF2는 RTK 상태가 Fix에서 Float으로 강제 이탈할 때 단일 코어 내의 노이즈 공분산(R)을 다이나믹하게 조정함으로써, 일시적 정수 모호성 소실이 위치 계산상의 ’점프(Position Jump/Glitch)’로 이어지지 않게끔 필터 내부에서 유연한 클리핑(Clipping)을 수행하는데 더욱 특화되어 설계되었다.

결과적으로 PX4에서 RTK 고정 해의 정수 모호성 상태 추적은 단순한 “상태 표시용 디스플레이 변수” 역할이 아닌, 실제 칼만 필터의 위치 상태 방정식 상의 신뢰 가중치를 결정하는 핵심 동적 트리거(Trigger) 메커니즘이다.
