43.7.1 방향성 비순환 그래프(Directed Acyclic 정Graph, DAG) 기반 결함 수 분석(Fault Tree Analysis)을 활용한 시동 거부(Arming Denial) 연쇄 인과성 추적 모델

무인항공기가 물리적으로 지면을 떠나기 위한 최초의 능동적 행위인 **시동(Arming)**은 PX4-Autopilot의 상태 전이 아키텍처 중에서 파국적 사고(Catastrophic Failure)를 예방하기 위해 가장 보수적이고 엄격하게 통제되는 임계점(Critical Point)이다.

비행 전 점검(Pre-flight Checks) 단계에서 특정 조건이 충족되지 않아 발생하는 **시동 거부(Arming Denial)**는 단순한 단일 센서의 오작동에서 비롯될 수도 있지만, 대부분의 복잡한 항공 전자 시스템에서는 여러 모듈의 상태 추정치가 결합된 연쇄적 실패의 결과로 나타난다. 따라서 이러한 시동 거부 원인을 단편적인 텍스트 메시지가 아닌, 결함 수 분석(Fault Tree Analysis, FTA) 공간에서의 방향성 비순환 그래프(Directed Acyclic Graph, DAG) 모델로 접근하여 인과성을 추적하는 공학적 분석 기법이 필수적이다.

1. 시동 전제 조건의 방향성 비순환 그래프(DAG) 구조 모델링

PX4 내부의 HealthAndArmingCheck 모듈은 단순히 수십 가지의 독립된 센서 상태를 일차원적으로 나열하여 점검하지 않는다. 각 시스템 컴포넌트(센서, 액추에이터, 배터리, 통신 링크 등)와 하이레벨 상태 추정기(EKF2)는 상호 의존성을 가지는 그래프 구조 상의 노드(Node)로 편재된다.

이는 사이클(Cycle)이 없는 상향식 트리 구조, 즉 **DAG(Directed Acyclic Graph)**로 모델링할 수 있다. 하위 컴포넌트 노드에서 발생한 건전성 상실(Health Loss) 이벤트는 방향성을 띠고 상위 추정기 노드로 전파되며, 최상위 루트 노드인 ‘시동 권한(Arming Authorization)’ 노드의 상태를 거부(Denied)로 평가(Evaluate)하게 만든다.

이러한 결함 전파 과정을 시각화한 예시 모델은 다음과 같다.

graph TD
    A[Arming Authorization: Denied] --> B[EKF2 State Estimate: Invalid]
    A --> C[Battery Status: Valid]
    B --> D[Position Estimate: Failed]
    B --> E[Attitude Estimate: Valid]
    D --> F(GPS 3D Fix: Lost)
    D --> G(Mag Sensor: High Interference)
    E --> H(IMU Gyro: Valid)

    style B fill:#ff9999,stroke:#333,stroke-width:2px

위 DAG 계층도에서 나침반 센서(Mag Sensor, 노드 G) 근처에 강한 자기장 간섭이 발생하면, 하위 노드 결함이 EKF2의 추정(노드 B)을 신뢰할 수 없는 상태로 격하(Degrade)시킨다. 이는 곧 최종 시동 결정(노드 A)에 치명적 플래그로 전가되어 연쇄적인 인과 관계를 형성한다.

2. Ardupilot 대비 PX4-Autopilot의 시동 점검 차별성

이러한 DAG 차원의 연쇄 평가 아키텍처는 Ardupilot의 고전적인 Pre-Arm Checks 설계와 뚜렷하게 대비된다.

  • Ardupilot의 비트마스크(Bitmask) 기반 순차 검사:
    Ardupilot의 경우 ARMING_CHECK 파라미터가 32비트 마스크 형태로 존재하여, IMU 점검, GPS 점검, Compass 점검 등을 순차적으로 실행(Sequential Check)한다. 결과적으로 상호 의존적인 모듈 간의 결함이 발생했을 경우 연쇄 원인보다 표면적인 첫 번째 탈락 조건을 단순 리포팅하는 나열식(Flat) 구조에 가깝다.
  • PX4-Autopilot의 이벤트 및 리소스 기반 컴포넌트 설계:
    PX4는 개별 백그라운드 태스크나 센서 드라이버 모듈이 자신의 구동 가능 여부를 uORB 기반의 event 메시지 큐를 통해 Commander 에 능동적으로 보고한다. 이는 객체 지향적인(Sub-system Evaluation) 방식이며, 각 서브 시스템이 트리의 말단에서부터 합산된 자신의 ‘Health’ 상태를 상위로 전송하는 진보된 구조이다.

3. QGroundControl (GCS) 내 시동 거부 구문 분석 메커니즘

GCS 사용자는 비행 준비 단계에서 “Arming Denied“라는 시각적 / 음성적 피드백을 받게 된다. QGroundControl은 (PX4 1.14 최신 기준) 기체로부터 수신되는 단순히 평문(Plaintext) 형태의 STATUSTEXT MAVLink 메시지를 맹목적으로 출력하지 않는다.

대신 기체의 진단 모듈이 발생시킨 다차원적 결함 원리 텍스트(예: “Preflight Fail: Compass 0 uncalibrated”)를 내부 정규 표현식(Regular Expression) 파서(Parser)를 통해 분석한다. 이를 통해 GCS는 DAG 트리의 어느 하위 노드에서 병목이 생겼는지를 정확히 추적(Trace)하고, 단순히 “시동 불가“를 띄우는 것이 아니라 조종사에게 “나침반 센서 1번의 캘리브레이션을 다시 수행하라“는 직접적인 해결책(Actionable Resolution) 프롬프트 팝업으로 유도할 수 있다.

4. 트러블슈팅 분석 공학적 활용

개발 및 트러블슈팅 단계에서 엔지니어는 시동이 지속적으로 거부되는 현상(특히 간헐적으로 발생하여 디버깅이 어려운 Heisenbug의 경우)을 추적하기 위해 ULog 내의 이벤트를 결함 수 분석(FTA) 이론을 적용해 해석해야 한다.

  • AND 게이트(AND Gate) 및 OR 게이트(OR Gate) 적용: 시스템 내에 2개의 GPS 노드가 장착된 듀얼 포지셔닝(Dual Positioning) 상태라면 2개의 수신기 결함은 시동 거부로 이어지는 논리적 AND 게이트를 형성한다. 어느 한도까지 Fail-over가 가능한 조건을 설계에 반영했는지 ULog의 sensor_gps 토픽의 Innovation 값을 통해 시계열 역추적을 수행해야만 한다.

결론적으로, PX4의 시동 거부 매커니즘은 단순 조건문의 결속체(Collection)가 아니라 인과적으로 얽혀있는 수학적 DAG 모델 추론 프로세스의 결과이다. 이를 이해하고 트러블슈팅에 임하는 엔지니어는 1차원적인 에러 로그의 맹신을 벗어나 시스템 설계자의 관점에서 복합 고장(Complex Failure)을 분해하고 해석하는 능력을 갖출 수 있다.
