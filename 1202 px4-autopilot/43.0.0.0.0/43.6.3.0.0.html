43.6.3 마르코프 체인(Markov Chain) 기반 네트워크 패킷 손실 확률 모델 및 왕복 지연 시간(RTT) 추정에 따른 동적 제어 타임아웃(Timeout) 한계 결정 알고리즘

본 절에서는 텔레메트리(Telemetry) 무선 네트워크의 상태를 추정하고 제어 명령 전송 시 타임아웃(Timeout) 한계를 가변적으로 결정하기 위한 마르코프 체인(Markov Chain) 확률 모델 기반의 GCS 백엔드 알고리즘을 분석한다.

1. 무선 텔레메트리 환경의 패킷 손실 특성과 한계

드론 관제 환경에서 915MHz나 2.4GHz ISM 대역을 사용하는 텔레메트리는 도심지 전파 간섭(Interference), 기체 자세 변화에 따른 안테나 그림자 현상(Shadowing), 그리고 다중 경로 페이딩(Multipath Fading) 등에 의해 네트워크 상태가 급변한다.

전통적인 GCS(예: 초기 Ardupilot 기반 시스템)에서는 이러한 변동성을 무시하고 MAVLink 제어 패킷의 대기 시간을 정적(Static) 상수(예: 500ms)로 하드코딩(Hard-coding)하는 방식이 흔했다. 그러나 정적 타임아웃은 링크 품질이 좋을 때는 불필요한 재전송 딜레이를 유발하고, 품질이 악화될 때는 재전송 큐(Queue)가 조기에 고갈되는 극단적인 비효율성을 초래한다.

2. 2-State (Gilbert-Elliott) 패킷 손실 마르코프 체인 모델

현대적인 GCS 및 자율 에이전트(Autonomous Agent) 관제 소프트웨어는 패킷 손실이 단발성이 아니라 연쇄적으로 발생(Burst Loss)하는 무선 네트워크의 실제 특성을 반영하기 위해, Gilbert-Elliott 확률 모델로 알려진 2-State 마르코프 체인을 적용한다.

채널의 상태는 크게 Good State (S_G) 와 Bad State (S_B) 로 정의된다.

  • S_G (Good): 채널 상태가 양호하며 패킷 에러 확률이 낮음(p < \epsilon)
  • S_B (Bad): 채널 간섭이나 페이딩으로 인해 패킷 손실률이 급증함(p \to 1)

상태 전이 확률(Transition Probabilities) 행렬 P는 아래와 같이 구성되며 주기적인 PING 및 HEARTBEAT 패킷의 누락 벡터를 통해 GCS 백엔드에서 실시간 갱신된다.

P = \begin{bmatrix} P_{GG} & P_{GB} \\ P_{BG} & P_{BB} \end{bmatrix}

(예: 직전 패킷이 손실되었을 때 다음 패킷도 손실될 확률은 P_{BB}로 계산됨)

43.6.3.3 동적 왕복 지연 시간(RTT) 추정 및 타임아웃(Timeout) 한계 결정 알고리즘

Gilbert-Elliott 모델을 통해 현재 채널 상태를 추론함과 동시에, GCS는 TCP의 Jacobson/Karels 알고리즘과 원리가 유사한 SRTT (Smoothed Round-Trip Time) 및 RTTVAR (Round-Trip Time Variation) 값을 측정한다. MAVLink 스트림에서는 TIMESYNC 메시지를 활용하여 기체와 GCS 간 네트워크 지연을 실시간 계측한다.

RTT 평활화 수식 (지수 가중 이동 평균, EWMA):
SRTT_n = (1 - \alpha) \cdot SRTT_{n-1} + \alpha \cdot RTT_{new}
RTTVAR_n = (1 - \beta) \cdot RTTVAR_{n-1} + \beta \cdot |SRTT_{n-1} - RTT_{new}|

(일반적으로 QGC/MAVSDK C++ 구현 시 \alpha=0.125, \beta=0.25 사용)

동적 타임아웃 윈도우 계산 로직:
최종적으로 콜백(Callback) 관리를 위한 동적 타임아웃 윈도우 T_{RTO} (Retransmission Timeout)는 아래와 같이 채널 손실 마르코프 모델의 가중치 변수(W_{Markov})를 융합하여 결정된다.

T_{RTO} = SRTT_n + (K \cdot RTTVAR_n) + W_{Markov}

  • 채널 양호(Good) (P_{GG} 지배적): W_{Markov}가 0으로 수렴, 네트워크 대기시간을 짧게 책정하여 빠른 재전송 도모.
  • 채널 침체(Bad) (P_{BB} 지배적): W_{Markov} 극대화 적용. 패킷이 연속으로 유실될 것이 예상되므로 공격적인 재전송을 회피하고 텔레메트리 큐의 병목을 막기 위해 타임아웃 상한을 길게 늦춤.

3. 폐루프 제어 아키텍처 상의 기대 효과

동적 RTT 추정치와 마르코프 체인 확률 모델을 결합한 스마트 타임아웃 로직은 비동기 제어 환경에서 기체와의 텔레메트리 링크 대역폭(Bandwidth) 낭비를 극적으로 줄인다.

이를 통해 ROS2 기반 군집 에이전트 다중 관제 상황이나 낮은 보드레이트(Baudrate)의 장거리 시리얼 라디오 환경에서도, QGroundControl은 ’적절한 시점, 횟수’로 제어 명령을 푸시하여 COMMAND_ACK 응집성을 높이고 GCS 프론트엔드 UI의 일시적인 멈춤 현상(Freezing)이나 불필요한 다중 경고창 팝업을 방지하는 안정적 관제 철학을 완성한다.
