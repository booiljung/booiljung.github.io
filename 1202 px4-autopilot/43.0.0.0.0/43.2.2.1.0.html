43.2.2.1 Parameter 1: 시동/해제 상태를 결정하는 이진(Binary) 상태 제어 플래그의 스텝 함수(Step Function) 분기 논리

MAVLink 프로토콜의 MAV_CMD_COMPONENT_ARM_DISARM (Command ID: 400) 명령에서 기체의 시동(Arm) 및 시동 해제(Disarm) 여부를 결정짓는 핵심 인자는 Parameter 1이다. 지상 관제 시스템(GCS) 사용자 인터페이스의 단순한 슬라이더 조작은 네트워크를 거쳐 비행 제어기(Flight Controller)에 도달할 때, 연속적인 실수형(Floating-Point) 도메인에서 엄밀한 이진(Binary) 스텝 함수(Step Function) 논리로 매핑되어 시동 상태 머신의 분기를 결정한다. 이 절에서는 Parameter 1이 평가되는 알고리즘적 분기 논리와 그 이면에 자리한 시스템 안정성 확보 매커니즘을 상세히 분석한다.

1. 이진 상태 제어 플래그의 실수형(Float) 매핑

앞선 절에서 설명한 바와 같이 COMMAND_LONG 메시지는 7개의 단정밀도 부동소수점 파라미터를 갖는다. 시동/해제라는 논리적(Boolean) 이진 상태는 MAVLink 규약에 따라 Parameter 1 공간에 다음과 같이 강제 매핑된다.

  • 시동(Arming) 인가: param1 = 1.0f
  • 시동 해제(Disarming) 인가: param1 = 0.0f

제어 공학적 관점에서 이는 단위 계단 함수(Unit Step Function, 또는 Heaviside Step Function) H(x) 로 모델링할 수 있다. 입력 x 가 param1의 값일 때, 시스템의 상태 변수 S는 다음과 같이 정의된다.

S(x) = \begin{cases} 0 & (\text{Disarmed, if } x < 0.5) \\ 1 & (\text{Armed, if } x \ge 0.5) \end{cases}

2. 분기 논리(Branching Logic) 및 비행 전 점검(Pre-flight Check)의 결합

PX4-Autopilot(v1.14.0 기준)의 commander 모듈은 이 스텝 함수의 결과를 맹목적으로 기체 구동부(Actuator)에 직결하지 않는다. Parameter 1이 1.0f 로 입력되어 S(x) = 1 조건이 성립하더라도, 이는 ’사용자의 시동 의도(Intent)’를 나타낼 뿐 즉각적인 인터럽트 권한을 가지지는 않는다.

시동 의도 플래그는 시스템 내부의 비행 전 점검(Pre-flight Check) 논리와 논리곱(AND, \land) 연산으로 평가된다.

    B -- "param1 >= 0.5 (Arm)" --> C[시동 의도 판별]

    C --> E{비행 전 점검 통과?}

    D --> H{비행 중인가?}
    H -- Yes --> I{강제 해제(param2) 조건 평가}
    H -- No --> J[정상 시동 해제 승인]
    
    F --> K((상태 머신 전이: ARMED))
    J --> L((상태 머신 전이: DISARMED))

2.1 아키텍처적 안정성 검증 기제

  • 오차 내성(Fault Tolerance): 통신 상의 노이즈나 QGroundControl(GCS) 내부의 자바스크립트 부동소수점 처리 결함으로 인해 param1이 0.9999f 나 1.0001f 로 수신되더라도 분기문은 if (param1 > 0.5f) 와 같이 임계치(Threshold)를 기준으로 설계되어 있어 시스템 오작동을 방지한다.
  • 상태 전이 록(Lock): param1 = 1.0f 이더라도 센서 캘리브레이션 누락, EKF 혁신(Innovation) 오차 초과, 또는 킬 스위치(Kill Switch)가 활성화되어 있다면 Commander는 분기 논리에서 이를 MAV_RESULT_DENIED 로 응답하고 상태 전이를 거부한다.

3. 소스 코드 분석: Commander 모듈의 스텝 함수 구현

PX4 소스 트리 내 src/modules/commander/Commander.cpp 에서는 MAVLink 객체에서 추출된 명령이 vehicle_command_s 형태로 변환된 후 다음과 같은 로직 흐름을 거친다.

// PX4 Commander 핸들러 의사 코드
bool Commander::handle_command(vehicle_command_s *cmd)
{
    if (cmd->command == vehicle_command_s::VEHICLE_CMD_COMPONENT_ARM_DISARM) {
        
        // 1단계: param1을 이용한 스텝 함수 기반 의도 파악
        bool arm_request = (cmd->param1 > 0.5f);
        
        if (arm_request) {
            // 2단계: 시동 전제 조건(Pre-condition) 확인
            if (preflight_check_passed() && !safety_button_pressed()) {
                transition_to_armed_state();
                answer_command(cmd, vehicle_command_ack_s::VEHICLE_CMD_RESULT_ACCEPTED);
            } else {
                answer_command(cmd, vehicle_command_ack_s::VEHICLE_CMD_RESULT_DENIED);
            }
        } else {
            // 시동 해제 분기
            if (is_landed() || is_force_disarm(cmd->param2)) {
                transition_to_disarmed_state();
                answer_command(cmd, vehicle_command_ack_s::VEHICLE_CMD_RESULT_ACCEPTED);
            } else {
                // 비행 중 정상 시동 해제 거부
                answer_command(cmd, vehicle_command_ack_s::VEHICLE_CMD_RESULT_DENIED);
            }
        }
        return true;
    }
    return false;
}

이 로직에서 두드러지는 특징은 시동 해제(param1 < 0.5f) 분기 로직이다. 기체가 비행 중(In-air)인 상태에서 조종자의 실수나 GCS 버그로 인해 param1 = 0.0f 명령이 인입되었을 때, 단순 스텝 함수 분기만 따른다면 공중에서 모터가 꺼지는 치명상(Crash)을 입을 것이다. 따라서 PX4 동역학 모델은 기체의 접지 여부(is_landed())를 필수 교차 가드(Guard) 조건으로 강제한다.

4. Ardupilot과의 분기 논리 차별성

PX4는 비행 전 점검(Pre-flight Check)과 Parameter 1의 스텝 함수를 커맨더(Commander)라는 최상위 상태 머신에서 중앙집중적으로(Centrally) 결합한다. 반면, Ardupilot은 MAVLink 수신 루틴인 GCS_MAVLink::handle_command_long_arm_disarm() 내부에서 즉시 AP_Arming::arm() 이나 disarm() 을 호출하며 분기한다.

Ardupilot의 AP_Arming 클래스는 arm() 함수 수행 시 다시 객체 지향적으로 내장된 센서 서브 클래스들의 상태를 순회(Iteration)하며 검사하는 ‘폴링 역전(Polling Inversion)’ 구조를 가진다. 두 시스템 모두 안전성 측면에서는 높은 신뢰도를 갖지만, PX4의 중앙집중형 스텝 논리가 ROS2 기반의 다중 에이전트 개발이나 외부 컴퓨터(Companion Computer)로부터의 명령 충돌을 디버깅하기에 상태 추적망이 보다 직관적이고 시각화하기에 유리한 아키텍처적 이점을 제공한다.

요약하자면, MAV_CMD_COMPONENT_ARM_DISARM 명령의 Parameter 1은 단순한 부동소수점 데이터가 아니라, 무인항공기의 내부 복합 안전 가드와 연계되어 시동 상태 머신의 분기를 결정짓는 치명적인 이산 이벤트 트리거(Discrete Event Trigger)이자 단위 계단 함수이다.
