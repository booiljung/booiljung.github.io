43.1.4 결정론적 유한 상태 기계(Deterministic Finite Automaton) 모델을 적용한 PX4 시스템 상태 공간(State Space) 정의 및 무결성 보장 경계

유인 항공기와 마찬가지로 무인항공기 시스템(UAS) 제어 소프트웨어에서 가장 경계해야 할 것은 ’정의되지 않은 상태(Undefined State)’로 빠지는 것이다. 시스템이 어느 상태에 있는지, 그리고 다음 상태로 어떻게 전이되어야 하는지 명확하지 않으면 치명적인 사고로 직결된다. 이를 방지하기 위해 PX4-Autopilot은 결정론적 유한 상태 기계(DFA, Deterministic Finite Automaton) 아키텍처를 도입하여 전체 시스템의 상태 공간(State Space)을 엄격하게 정의하고 제어 무결성을 보장한다.

1. 결정론적 유한 상태 기계(DFA) 모델의 공학적 정의

DFA는 시스템이 가질 수 있는 유한한 개수의 상태(States)를 정의하고, 입력(Input)에 따라 오직 하나의 다음 상태(Next State)로만 결정론적으로 전이(Transition)하는 수학적 모델이다.

PX4 시스템은 이를 vehicle_status uORB 메시지 내의 nav_state (Navigation State) 및 arming_state 변수로 구체화한다. 조종기의 스위치 조작이나 지상 관제 시스템(QGroundControl)의 MAVLink 제어 명령은 단순한 ‘실행’ 명령이 아니라, 상태 기계의 **전이 조건(Guard Condition)**을 트리거하는 이벤트로 취급된다.

2. PX4 시스템의 내비게이션 상태 공간(Navigation State Space)

PX4의 내비게이션 상태 공간은 연속적인 비행 동역학(Dynamics) 위에 이산적인(Discrete) 논리 구조를 덧씌운 것이다. 대표적인 상태 공간의 원소들은 다음과 같다.

  • NAVIGATION_STATE_MANUAL: 모든 자동 제어가 꺼지고 조종자의 스틱 입력이 직접 혼합(Mixing) 로직으로 전달되는 순수 수동 모드.
  • NAVIGATION_STATE_ALTCTL: 기압계 등 논리적 고도 센서가 유효할 때 진입 가능한 고도 제어 모드.
  • NAVIGATION_STATE_POSCTL: 유효한 글로벌 위치 데이터(GPS 등)와 자세 데이터 모두가 수학적으로 무결성 범위를 충족할 때만 진입 가능한 위치 제어 모드.
  • NAVIGATION_STATE_AUTO_MISSION: 사전 입력된 웨이포인트를 자율적으로 비행하는 모드.
  • NAVIGATION_STATE_AUTO_RTL: 복귀(Return To Launch) 상태.

3. DFA 전이 규칙과 무결성 보장 경계(Integrity Boundary)

DFA 모델에서 각 상태로의 전이는 무결성 보장 경계라는 뚜렷한 조건(Pre-condition)을 통과해야만 이루어진다. PX4의 commander 태스크는 이러한 전이 조건을 수학적으로 평가하는 관문(Gateway) 역할을 한다.

3.1 상태 전이 평가 매커니즘

만약 기체가 ALTCTL 모드로 비행 중이며 조종자가 POSCTL (위치 보류 모드)로 스위치를 넘기려 한다고 가정하자.

  1. 입력 발생: 조종기 스위치 변경으로 상태 전이 이벤트 트리거.
  2. 조건 평가 (Guard Evaluation): 커맨더는 EKF2 추정기의 local_position_valid 및 global_position_valid 플래그를 검사한다.
  3. 분기 처리:
  • GPS 수신이 양호하여 플래그가 참(True)이면, 상태는 POSCTL로 즉각 전이된다.
  • GPS 수신 불량으로 위치 불확실성 경계(Position Uncertainty Boundary)를 초과하여 플래그가 거짓(False)이면, 전이는 거부(Denied)되고 시스템은 현재 상태(ALTCTL)를 유지한다. 사용자에게는 경고(예: “Reject Position Control”)가 출력된다.

이 매커니즘은 시스템이 외부 강압에 의해 정의되지 않은 혹은 신뢰할 수 없는 센서 데이터에 기반한 제어 모드로 넘어가는 것을 설계 단계에서 차단한다.

3.2 시각적 상태 전이 예시

4. Failsafe와 무결성 경계의 상호작용

무결성 보장 경계는 사용자 명령 시에만 작동하는 것이 아니다. 비행 중 지속적으로 센서의 혁신 오차(Innovation Error)나 분산(Variance)이 특정 임계치를 초과하면, PX4는 스스로 상태 전이를 강제 발동한다. 이를 상태 강등(State Fallback) 혹은 Failsafe 로직이라 부른다.

예를 들어 POSCTL 모드 비행 중에 심각한 전파 간섭으로 인해 EKF2 추정기의 예측 모델(Prediction)과 관측 모델(Observation)간의 오차가 시스템 내부 설계 마진을 넘어서면, 기체는 즉각 GPS 의존성을 버리고 위치 제어 능력을 포기한 상태(ALTCTL 등)로 강등(Fallback)된다. 이 또한 DFA의 결정론적 전이 엣지(Edge) 중 하나이다.

5. 결론

PX4-Autopilot의 핵심은 정교한 제어 알고리즘 그 자체보다, 어떠한 외부적 노이즈나 잘못된 조작에도 시스템이 논리적으로 정의된 상태 공간 내에서만 궤적을 그리도록 억제하는 데 있다. 결정론적 유한 상태 기계(DFA)에 기반한 상태 전이 매커니즘은 이런 무결성 역학을 유지하는 근간이며, 자율 비행 객체가 수학적 안전 경계를 벗어나 폭주하지 않도록 강력한 소프트웨어적 봉인(Seal)을 제공한다.
