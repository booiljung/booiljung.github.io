52.1.2.1 비상 정지(Kill Switch)의 가역적 제어 차단 특성 및 일시적 외란(Disturbance) 인가 상태의 논리적 정의

본 절에서는 무인항공기 비상 제어 시스템 중 가장 빈번하게 활용되는 비상 정지(Kill Switch) 기능의 가역적 제어 차단(Reversible Control Inhibition) 특성과, 유한 상태 기계(Finite State Machine)의 관점에서 이를 일시적 외란(Transient Disturbance) 인가 상태로 해석하는 논리적 원리를 심도 있게 다룬다. 본 절의 분석은 PX4 펌웨어 v1.14.0 및 QGroundControl v4.3.0을 기준으로 진행된다.

1. 가역적 제어 차단(Reversible Control Inhibition)의 공학적 의미

비상 정지, 즉 킬 스위치(Kill Switch)는 작동 즉시 비행 제어기(Flight Controller)에서 각 액추에이터(Actuator)로 전달되는 구동 신호(PWM, DShot, UAVCAN 등)를 시스템 최소 구동값(Disarmed Value)으로 강제 초기화하는 안전 기능이다. 여기서 매우 중요한 특성은 **가역성(Reversibility)**이다. 킬 스위치가 스위치 토글이나 MAVLink 명령을 통해 입력되었다 하더라도, 기체의 파괴나 심각한 고장이 발생하지 않았다면 사용자나 관제 시스템의 의도에 따라 다시 킬 스위치를 해제하여 시스템을 즉시 제어 가능한 상태(Controllable State)로 되돌릴 수 있어야 한다.

이는 시스템의 영구적인 비가역적 정지를 유발하는 비행 종료(Flight Termination)와 명확히 구분되는 공학적 특성이다. 킬 스위치가 인가되는 동안 센서 데이터 융합(Data Fusion), 자세 추정(Attitude Estimation, EKF2), 그리고 MAVLink 통신 등 제어 스택(Control Stack)의 핵심 백그라운드 프로세스는 중단되지 않고 지속된다. 즉, 오직 제어기(Controller)의 최종 출력 단(Output Stage)에서만 논리적 차단이 이루어진다.

2. 일시적 외란(Transient Disturbance) 상태의 논리적 정의

제어 이론(Control Theory)의 관점에서 비상 정지 상태는 시스템이 제어 불능에 빠져 최종 상태(Terminal State)로 흡수된 것이 아니라, 시스템에 의도적으로 인가된 매우 거대한 크기의 **일시적 외란(Transient Disturbance)**으로 간주할 수 있다.

PX4의 커맨더(Commander) 모듈은 유한 상태 기계(FSM)를 기반으로 기체의 상태를 관리한다. 킬 스위치가 인가될 때 기체의 무장(Armed) 상태 자체는 해제(Disarm)되지 않을 수 있으나, 출력이 억제됨에 따라 기체의 추력(Thrust)은 즉시 사라진다. 이 상태에서 제어기 내부의 적분기(Integrator)에 오차(Error)가 누적되어 스위치 해제 시 갑작스러운 제어 입력 변동(Windup 현상)이 발생하는 것을 방지하기 위해, PX4는 내부적으로 제어 루프의 초기화 및 적분기 리셋 메커니즘을 구동한다.

2.1 상태 천이 메커니즘 (State Transition Mechanism)

킬 스위치가 활성화 및 비활성화되는 상태 천이 과정을 Mermaid 다이어그램으로 나타내면 다음과 같다.

stateDiagram-v2
    [*] --> Disarmed
    Disarmed --> Armed : Arming Command
    
    state Armed_State {
        direction LR
        Normal_Flight --> Kill_Switch_Engaged : Kill Switch ON
        Kill_Switch_Engaged --> Normal_Flight : Kill Switch OFF (Reversible)
    }
    
    Armed --> Armed_State
    Armed_State --> Disarmed : Disarm Command / Touchdown

위 다이어그램에서 보듯, Kill_Switch_Engaged 상태는 Armed_State 내에 존재하는 하위 상태(Sub-state)이다. 이 상태를 벗어나 Normal_Flight로 다시 전이하는 경로가 존재하므로 이는 흡수 상태(Absorbing State)가 아니다.

3. PX4 소스 코드 수준의 아키텍처 분석

PX4 펌웨어에서는 킬 스위치 관련 로직이 주로 src/modules/commander/Commander.cpp 와 src/modules/rc_update/rc_update.cpp 에 구현되어 있다. 킬 스위치는 전용 조종기 채널(RC_MAP_KILL_SW)이나 MAVLink 메시지(MAV_CMD_DO_FLIGHTTERMINATION에서 파라미터 1을 0.0으로 사용할 경우)를 통해 인가된다.

킬 스위치가 인가되면 커맨더 데몬은 다음과 같은 일련의 가역적 억제 절차를 실행한다.

  1. 플래그 설정: 시스템 상태 구조체인 vehicle_status_s 내의 kill_switch_action 플래그를 true로 설정한다.
  2. 액추에이터 출력 제한: 제어 할당 코어(Control Allocation Core)에서 스러스터 추력(Thruster Output)을 0으로 강제하는 록다운(Lockdown) 메커니즘이 활성화된다. 이때, GCS에는 MAVLink의 HEARTBEAT 및 SYS_STATUS 메시지를 통해 현재 기체가 킬 스위 상태에 진입했음을 알린다.
  3. 적분기 리셋(Anti-Windup): 위치 제어(Position Control) 및 자세 제어(Attitude Control) 모듈이 현재 추력이 발생하지 않고 있음을 유효하게 전달받아 비례 적분 미분 제어(PID Control) 내부의 I-term 누적을 중지시킨다.

4. Ardupilot 통제 시스템과의 아키텍처 및 철학 비교

PX4의 비상 정지 메커니즘을 Ardupilot 메커니즘과 대비하면 그 설계 철학의 차이를 명확히 이해할 수 있다.

  • Ardupilot (Motor Interlock 및 E-Stop): Ardupilot은 역사적으로 헬리콥터 인터록(Interlock) 채널에 기원하여 모터 비상 정지 기능을 구현했다. 근래에는 E-Stop(Emergency Stop) 기능을 도입하였는데, Ardupilot의 E-Stop은 환경 변수에 따라 가역적일 수도 있고, 설정에 따라 비행 종료(Crash)로 직결되도록 유도하기도 한다. Mission Planner 관제 화면에서는 E-Stop 기능을 독립된 긴급 버튼으로 전면에 배치하는 경향이 있다.
  • PX4-Autopilot (Kill Switch): 제어의 연속성과 가역성에 강한 주안점을 둔다. PX4는 상태 머신 기반의 엄격한 모듈성을 지향하기 때문에, 킬 스위치는 ’출력 차단’이라는 본연의 기능에만 집중하며 내비게이션 상태(Navigation State)를 파괴하지 않는다. QGroundControl(QGC)에서는 Vehicle 매니저 클래스 산하에서 킬 스위치 이벤트를 시그널(Signal)로 처리하며, 킬 스위치 활성화 시 UI 상단에 눈에 띄는 빨간색 배너와 경고음을 통해 시각적, 청각적 피드백을 실시간으로 제공한다(QGCApplication.cpp 및 FlightDisplayView.qml 영역 참조).

5. MAVLink 및 ROS2 기반 제어 (Micro XRCE-DDS)

외부 에이전트(Companion Computer)에서 ROS2 기반 자율주행 알고리즘 구동 시, 돌발 상황에서 킬 스위치를 강제로 제어하는 것은 매우 중요하다. ROS2 환경에서는 uXRCE-DDS 브리지를 통해 MAVLink 커맨더와 직접 상호작용한다.

// ROS2 기반 비상 정지 명령 퍼블리싱 예시(C++)
#include <px4_msgs/msg/vehicle_command.hpp>

// ... 초기화 및 퍼블리셔 선언부 생략 ...

px4_msgs::msg::VehicleCommand msg;
msg.param1 = 0.0; // 0.0: Kill Switch 활성화, 1.0: Flight Termination
msg.command = px4_msgs::msg::VehicleCommand::VEHICLE_CMD_DO_FLIGHTTERMINATION;
msg.target_system = 1;
msg.target_component = 1;
msg.source_system = 255;
msg.from_external = true;
msg.timestamp = node->get_clock()->now().nanoseconds() / 1000;

vehicle_command_publisher_->publish(msg);

이와 같이 ROS2 노드에서 발행된 제어 명령은 vehicle_command uORB 토픽으로 매핑되고, 커맨더(Commander) 내부 루프에서 킬 스위치 이벤트로 변환된다.

6. 소결

가역적 제어 차단이라는 특성은 기체가 위험한 상황에서 긴급히 추력을 상실해야 하지만, 고도가 확보되어 있거나 혹은 소프트웨어적 결함(Glitch)으로 인해 일시적 외란이 인가된 경우에 다시 조종권을 회복할 수 있는 공학적 여지를 남겨둔다. 따라서 PX4 개발자 또는 시스템 통합(SI) 엔지니어는 킬 스위치가 본질적으로 시스템의 모든 상태를 초기화하는 것이 아니라, 액추에이터 출력을 물리적으로 억제함으로써 상태 머신에 강력한 일시적 외란을 거는 고신뢰성 제어 기동임을 정확히 이해해야 한다. 향후 EASA SORA나 ASTM 표준을 충족하는 UAM(Urban Air Mobility) 급 시스템 설계 시 이러한 가역/비가역 메커니즘의 차이는 인증(Certification) 획득의 핵심 요소로 작용하게 된다.
