52.4.3.1 피에조 부저(Piezo Buzzer)의 펄스 폭 변조(PWM) 주파수 변조를 통한 인간 청각 인지도 극대화 경고음 생성 알고리즘

본 절에서는 PX4-Autopilot(기준 펌웨어 버전: v1.14.0)에서 비상 상황 시 운용자의 즉각적인 대응을 유도하기 위해 피에조 부저(Piezo Buzzer)를 활용하는 청각 피드백 알고리즘을 분석한다. 특히, 인간의 청각 인지도(Human Auditory Cognitive Recognition)를 극대화하기 위하여 펄스 폭 변조(PWM, Pulse Width Modulation)의 주파수 및 듀티 사이클(Duty Cycle)을 동적으로 변조(Modulation)하는 내부 메커니즘과 소스 코드 코어 로직을 중점적으로 다룬다.

1. 인간 청각 인지도와 알람 신호 설계 원리

인간의 청각 시스템은 대략 2kHz ~ 5kHz 대역축에서 가장 높은 민감도를 나타낸다. 항공 전자(Avionics) 시스템의 비상 알람은 주변의 로터(Rotor) 소음, 바람 소리 등 광대역 백색 잡음(Broadband White Noise)을 뚫고 전달되어야 하므로, 단일 주파수 파형(Single-tone)보다는 인간의 주의를 끄는 비대칭적 다중 주파수(Multi-frequency)의 교차 패턴이 효과적이다.
PX4는 이를 구현하기 위해 단순한 GPIO On/Off 토글이 아닌, 실시간 워크 큐(Work Queue)에 기반한 하드웨어 타이머 PWM 변조 방식을 채택하고 있다. 이는 부저의 구동 전압을 제어하는 듀티 사이클 비율과 음계를 결정하는 주파수(Frequency)를 매우 짧은 밀리초(ms) 단위로 가변시켜, 소위 ‘사이렌(Siren)’ 파형이나 ‘치프(Chirp)’ 파형을 생성한다.

2. PWM 주파수 변조 기반 튠(Tune) 제어 아키텍처

PX4의 디바이스 드라이버 로직은 tune_control 모듈을 통해 관리된다. 비상 상황(예: 낙하산 펼침, 배터리 임계치 도달 등)이 트리거되면 배열 형태로 사전에 스크립팅된 튠 문자열(Tune String)이 해석(Parsing)되어 PWM 펄스로 변환된다.

위의 아키텍처에서 볼 수 있듯, 튠 컨트롤 태스크는 uORB 메시지를 구독하여 즉각적으로 하드웨어 인터럽트 사이클을 구성한다. TUNE_ID_NOTIFY_CRITICAL_ERROR 등과 같은 사전 정의된 ID는 고유의 패턴을 가지고 있으며, 이 패턴들은 기존의 멜로디 재생을 강제 종료(Preemption)하고 최고 우선순위로 재생된다.

3. 소스 코드 구현: 튠 파서(Tune Parser)와 타이머 제어

PX4 소스 트리의 src/lib/tunes/tune_definition.cpp 및 하드웨어 특정 드라이버(예: src/drivers/stm32/tone_alarm)를 분석해보면, 부저로 전달되는 주파수가 수학적 알고리즘으로 계산되는 것을 확인할 수 있다. 문자로 이루어진 안시 튠 스크립트(예: “MBT200L8O3cdcdcdcd”)는 각각의 알파벳과 숫자 조합에 따라 타이머의 Auto-reload Register(ARR)와 Capture/Compare Register(CCR) 값을 동적으로 변경한다.

// 경보음 생성을 위한 주파수 계산 및 PWM 설정 (유사 코드)
void ToneAlarm::set_note(unsigned frequency, unsigned duration_ms) {
    if (frequency == 0) {
        // 쉼표: PWM 출력 비활성화
        pwm_disable();
    } else {
        // 주파수를 타이머 클록 카운트로 변환
        uint32_t timer_clock = get_timer_clock(); 
        uint32_t period = timer_clock / frequency;
        
        // 듀티 사이클 50% 설정으로 구동 전력 최적화
        uint32_t pulse_width = period / 2;
        
        // 하드웨어 레지스터 갱신
        up_pwm_servo_set(channel, pulse_width, period);
    }
    // 스케줄러를 통해 지정된 지속 시간(duration_ms) 후 다음 음표 실행
    schedule_delayed_work(&_work, MSEC2TICK(duration_ms));
}

특히 비상 상황에서는 기본 옥타브(Octave)를 초과하는 고음역대(약 4kHz 부근)의 두 주파수를 짧은 간격(duration_ms가 50~100ms)으로 스위칭하여, 인간의 뇌간(Brainstem) 청각 인지 반응을 자극하는 패턴을 구동한다.

4. Ardupilot의 알람 스케줄링과의 비교

Ardupilot은 AP_Notify 모듈의 내부 클래스들을 사용하여 피에조 부저를 구동한다. 대체로 일정한 스케줄러 루프 내에서 On/Off를 반복하는 형태로 동작하기 때문에, 복잡한 주파수 천이(Frequency Shifting)나 복잡한 음악적 문자열 파싱을 통한 패턴화에는 상대적으로 보수적인 입장을 취한다.

반면, PX4-Autopilot은 독립된 백그라운드 워크 큐(Work Queue)를 기반으로 작동하는 전용 파서를 두어, CPU 메인 루프에 부하를 주지 않으면서도 수 밀리초 단위의 미세한 주파수 변화와 복합적인 멜로디 구현이 가능하다. 이러한 워크 큐 기반 PWM 타이머 변조는, 비행 제어 및 모터 출력(Actuator output) 루프를 전혀 방해하지 않는다는 점에서 실시간 스케줄링 관점의 설계 우수성을 방증한다.

5. 지상 관제 시스템(QGroundControl) MAVLink 연동 제어

운용자는 필요에 따라 QGroundControl 측에서 MAVLink 프로토콜의 PLAY_TUNE (MAV_CMD_PLAY_TUNE_V2) 명령을 기체에 전송하여 사용자 정의 경고음을 송출할 수 있다.
이를 통해 GCS 개발자는 자율 에이전트(Autonomous Agent)가 특정 미션 웨이포인트(Waypoint)에 도달했거나 임계 상황에 당면했을 때, 시스템 내장 부저를 프로그래밍 방식으로 조작하여 주변 사람들에게 시운전 위험성이나 기체 접근을 경고할 수 있다. 이처럼 단순한 경보음 이상을 넘어선 ’동적 펄스 변조 기반의 청각 피드백 시스템’은 무인기 운영의 안전과 사용자 인터페이스 설계의 한 축을 튼튼하게 지지하고 있다.
