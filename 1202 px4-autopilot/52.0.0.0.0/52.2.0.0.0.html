52.2. 비상 정지(Kill Switch) 하드웨어 인터럽트 및 비행 동역학적 억제 메커니즘

최신 PX4-Autopilot(펌웨어 버전 1.14 기준)이 탑재된 무인항공기에서 비상 정지(Kill Switch) 기능은 시스템 오류나 물리적 위협이 발생했을 때 기체의 동력을 즉각적으로 차단하여 2차 피해를 막는 가장 원초적이고 직관적인 안전 임계(Safety-Critical) 메커니즘이다. 본 절에서는 QGroundControl(버전 4.3 기준) 및 조종기(RC) 스위치를 통한 킬 스위치 발동부터, 비행 제어기(FC) 하드웨어 로우 레벨에서의 인터럽트 라우팅, 그리고 모드가 발동된 후 발생하는 기체의 동역학적 억제 상태에 이르기까지 전 주기에 걸친 공학적 아키텍처를 분석한다.

1. 하드웨어 인터럽트 라우팅 및 신호 캡처 프로세스

물리적인 무선 조종기(ex: SBUS, CRSF, PPM 통신 프로토콜을 사용하는 수신기)에서 조종사가 킬 스위치를 토글(Toggle)하면 수신기는 직렬 통신 스트림 패킷을 변경하여 비행 제어기(Pixhawk 계열 등)의 UART 범용 비동기 송수신기 포트로 전송한다.

NuttX RTOS 커널 계층에서 수신 포트의 하드웨어 수신(RX) 핀은 외부 인터럽트(IRQ)를 발생시키고 인터럽트 서비스 루틴(ISR)을 즉각 호출한다.
센서 및 하드웨어 통신을 처리하는 rc_update 또는 직렬 드라이버 백그라운드 스레드는 인터럽트를 감지하고 프레임 오류율(Frame Error Rate, FER)을 검사한 후 input_rc라는 이름의 uORB 메시지로 퍼블리싱한다. 물리적 스위치의 접점 불량으로 인한 채널 튀는 현상(Bouncing)을 방지하기 위해 소프트웨어 디바운싱(Debouncing) 필터가 입력단에 적용되어 극단적인 노이즈를 걸러낸다.

2. 소스 코드 기반 제어 로직 아키텍처 (Source Code Analysis)

최상위 두뇌 역할을 하는 PX4의 commander 태스크 모듈은 uORB 버스를 통해 구독 중인 조종기 채널 상태 변화를 폴링(Polling) 감지한다. 소스 코드 상에서 킬 스위치 매핑 파라미터(RC_MAP_KILL_SW)에 할당된 채널 값이 지정된 쓰레시홀드(주로 PWM 1500us 이상의 스위치 ON 상태)를 초과하게 되면, 즉시 kill_switch_is_on() 플래그가 참(true)으로 평가된다.

이 플래그가 참이 되면 상태 머신은 비행 모드에 상관없이 강제적인 권한을 발동한다. 비행 제어기 보조(IO) 프로세서와 통신하여 actuator_armed uORB 토픽 내부의 내부 변수(armed 및 force_failsafe)를 갱신하게 되며, 구동기 믹서(Mixer) 또는 제어 할당기(Control Allocator)는 모터(ESC)로 나가는 PWM 출력을 0, 혹은 설정된 disarmed 최소 PWM 신호(주로 900us)로 동기화 하강시킨다. 이 모든 과정은 지연 스케줄링 없이 선점형(Preemptive)으로 처리된다.

3. 설정 및 구성 (Configuration) 방안

  • 조종기 스위치 할당: 하드웨어 조종기의 2단 스위치를 킬 스위치로 할당하고 QGroundControl에서 파라미터 RC_MAP_KILL_SW 를 해당 채널로 매핑한다.
  • 안전 릴레이 차단 해제 설계: 벤치마킹이나 소프트웨어 인 루프 테스트(SITL) 시 강제 정지 장치를 소프트웨어적으로 점퍼 패싱하기 위해 회로 차단기 역할을 하는 CBRK_KILL 파라미터를 비활성화 설정할 수도 있다.

4. 비행 동역학적 억제 (Flight Dynamics Suppression) 제어 분석

킬 스위치가 발동하여 구동계 출력이 단절되면 순간적으로 기체는 제어 권한을 상실한다.

  • 멀티로터 기체: 모터 출력이 0으로 소멸하기 때문에 로터 추력이 사라져 중력 방향으로 자유 낙하(Free Fall) 역학 상태에 돌입한다. PID 제어 관점에서 mc_att_control (자세 제어 루프)의 적분기(Integrator)가 누적값 팽창(Windup)되는 것을 방지하기 위해 로직 내부적으로 적분항을 즉시 소거(Reset)한다.
  • 고정익/VTOL 기체: 주 동력이 끊기면 양항비(Glide Ratio)에 의존하는 활공 상태로 진입하지만 트림(Trim) 모드가 아닌 액추에이터 고정 상태이므로 극심한 실속(Stall) 및 슬립(Slip) 동역학에 노출되어 기체 밸런스가 통제 궤도를 이탈할 가능성이 존재한다.

5. MAVLink, ROS2 기반 제어 및 Ardupilot 아키텍처 비교

5.1 GCS 및 지능형 에이전트 관제

지상 관제 시스템 QGroundControl은 GUI 버튼을 시각적으로 활성화하여 물리 채널과 동일한 비상 제어 명령을 발생시킬 수 있다. MAVLink 프로토콜의 MAV_CMD_DO_FLIGHTTERMINATION 메시지를 전송하거나 긴급 DISARM 명령 패킷을 비동기 송신하여 Commander가 킬 스위치 루틴을 활성화하도록 한다.
자율 에이전트 비행 제어가 탑재된 ROS2 시스템에서는 비전 센서 기반 장애물 감지가 심각한 에러로 판명될 때, uXRCE-DDS 퍼블리셔(Publisher) 노드 측에서 오프보드 제어 중단을 선언하고 vehicle_command uORB 토픽 매핑을 통해 소프트웨어 단위의 억제를 동기식으로 추진하는 코드를 개발할 수 있다.

5.2 Ardupilot 대비 구조적 차이점

Ardupilot (ChibiOS 통합 구조) 시스템에서는 킬 스위치를 ‘Emergency Stop’ 혹은 E-Stop 모터 인터락(Interlock)이라는 이름으로 처리한다. Ardupilot의 메인 스케줄링 루프에서는 타이머 플래그 및 전역 구조체(Global Struct) 플래그를 변이시켜 E-Stop 발동을 인지하고 모터 출력을 컷오프하는 방식으로 접근한다.
이와 대조적으로 PX4-Autopilot은 철저히 uORB 토픽 스트리밍의 비동기적 아키텍처를 따른다. input_rc로부터 상태 변화를 구독(Subscribe)한 commander가 출력을 제어하는 여러 모듈(px4io, pwm_out, dshot)에 메시지 알림을 분산 전사시키는 다대다(N:N) 라우팅 메커니즘을 띠고 있으며, 이 접근법은 하드웨어 종속성을 완화하고 시스템 확장성 면에서 압도적으로 견고한 소프트웨어 공학적 토대를 제공한다.
