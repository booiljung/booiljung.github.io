52.7.1.1. 소프트웨어 인 더 루프(SITL) 환경에서 가상 센서 데이터에 백색 잡음(White Noise) 및 바이어스(Bias)를 강제 주입하여 칼만 필터 발산을 유도하는 스트레스 테스트 기법

현대의 무인항공기(UAV)는 주로 확장 칼만 필터(EKF, Extended Kalman Filter)를 사용하여 여러 센서의 데이터를 융합하고 기체의 자세와 위치를 추정(State Estimation)한다. 그러나 급격한 환경 변화나 센서 결함으로 인해 입력 데이터의 오차가 EKF의 설계된 혁신량(Innovation) 한계를 지속적으로 초과할 경우, 필터는 발산(Divergence)하게 되며 기체는 통제 불능 상태에 빠질 수 있다. 본 절에서는 PX4-Autopilot v1.14.0과 QGroundControl v4.3.0의 SITL(Software-In-The-Loop) 환경을 이용하여 가상 센서 데이터에 의도적으로 잡음과 바이어스를 투입해 EKF2 모듈의 발산을 유도하는 스트레스 테스트(Stress Test) 기법에 대해 상세히 다룬다.

1. 필터 발산을 위한 센서 오차 모델 (Sensor Error Model for Filter Divergence)

SITL 및 Gazebo 시스템 내에서 가상 센서 물리량 z(t)를 모사할 때, 이상적인 측정값 x(t)에 의도적으로 백색 잡음(White Noise, \eta(t))과 시간 종속적인 바이어스 드리프트(Bias Drift, \beta(t))를 합성하여 오차 주입 모델을 구성한다.

수식으로는 다음과 같이 표현된다.
z(t) = x(t) + \beta(t) + \eta(t)
여기서 \eta(t) \sim \mathcal{N}(0, \sigma^2)는 가우스 분포를 따르는 백색 잡음이고, 바이어스 \beta(t)는 특정 순간 t_{fault} 이후부터 선형적 혹은 계단형(Step-wise)으로 증가하도록 모델링한다.

EKF2 발산 유도의 원리 (Divergence Mechanism)

PX4의 상태 추정기(EKF2)는 각 센서 데이터가 들어올 때마다 혁신량 오차(Innovation Error)를 계산하고, 이것이 미리 설정된 임계치(Innovation Gate)를 초과하면 해당 측정값을 신뢰하지 않고 버린다(Reject). 그러나 바이어스가 서서히 증가하여 임계치 언저리에 장기간 머물거나, 과도한 백색 잡음으로 인해 상태 공분산 행렬(State Covariance Matrix)이 팽창(Inflation)하면 EKF2는 내부적으로 발산 판정을 내리고 센서 재설정(Sensor Reset) 또는 다른 센서로의 페일오버(Failover)를 시도하게 된다.

이러한 상황을 극한으로 몰고 가 모든 백업 센서(Backup Sensors)조차 무력화시킬 때 commander가 시스템 붕괴를 어떻게 방어하는지를 관찰하는 것이 이 테스트의 핵심 목적이다.

SITL 및 ROS2 환경을 활용한 결함 주입 실습

PX4-Autopilot의 SITL 환경 상에서 ROS2(Micro XRCE-DDS) 미들웨어를 연동하면, GCS나 시뮬레이터 내부 코드를 직접 수정하지 않고도 동적인 센서 노이즈 주입이 가능하다.

ROS2 노드를 통한 토픽 변조(Topic Manipulation) 예시

로깅된 가상 센서 데이터 토픽을 구독(Subscribe)한 후, 노이즈 수식을 적용하여 다시 원래의 PX4 토픽으로 덮어쓰거나 퍼블리싱(Publishing)하는 파이프라인(Pipeline)을 구축한다.

# ROS2 Python 노드 기반의 GPS 센서 바이어스 주입 예시
import rclpy
from rclpy.node import Node
from px4_msgs.msg import SensorGps
import numpy as np

class GPSNoiseInjector(Node):
    def __init__(self):
        super().__init__('gps_noise_injector')
        # 정상적인 GPS 토픽 구독 및 노이즈 주입 후 Publish 구조
        self.subscription = self.create_subscription(SensorGps, '/fmu/out/sensor_gps', self.listener_callback, 10)
        self.publisher_ = self.create_publisher(SensorGps, '/fmu/in/sensor_gps_injected', 10)
        self.bias = 0.0

    def listener_callback(self, msg):
        # 1초당 0.0001도의 위도 바이어스 누적 및 가우스 노이즈 추가
        self.bias += 0.0001
        noise = np.random.normal(0, 0.00005)
        
        msg.latitude_deg += int((self.bias + noise) * 1e7)
        self.publisher_.publish(msg)

스트레스 테스트 결과 분석 및 학술적 의미

가상 센서에 과도한 잡음을 주입하는 기법은 EKF의 강건성(Robustness)을 평가하는 대표적인 방법론이다. 관련 논문인 “Robustness Analysis of Extended Kalman Filter for UAV Navigation under Sensor Faults” 등에서도 입증되었듯, 백색 잡음이 임계치를 넘어설 때 EKF가 서서히 발산하는 패턴과 바이어스가 계단 단계로 입력되었을 때 튀는 양상은 전혀 다르다.

PX4의 모듈식 아키텍처 상에서는 EKF2가 발산하더라도, 비상 제어 시스템이 즉각 비행 모드를 전환성(Mode Transition, 예: Position Hold \rightarrow Altitude Hold \rightarrow Land)으로 강등시켜 연쇄적인 하드웨어 훼손을 방어하는 메커니즘을 관찰하고 조정할 수 있다. 이 테스트를 통과한 펌웨어 설정값(Parameters)만이 실비행 환경에서 예상치 못한 극한의 전자기장 간섭(EMI)이나 진동에도 기체의 생존성을 보장할 수 있다.
