52.2.1.1. 직렬 통신(UART) 기반 수신기 패킷의 프레임 오류율(FER) 모니터링 및 스위치 채널 디바운싱(Debouncing) 필터 설계

무인항공기의 킬 스위치(Kill Switch)와 같은 안전 임계(Safety-Critical) 데이터는 물리적 환경의 전파 간섭(RF Interference), 하드웨어 배선 불량, 반송파 감쇄 현상 등 다양한 오류 주입(Error Injection) 요인으로부터 강건하게 보호되어야 한다. 최신 PX4-Autopilot 아키텍처는 직렬 통신 기반으로 수신되는 조종기(RC) 패킷의 무결성을 수학적으로 보증하기 위하여 정교한 프레임 오류율(FER) 모니터링 기법과 릴레이 스위치 오작동을 막는 소프트웨어 디바운싱(Debouncing) 필터를 탑재하고 있다.

1. 직렬 통신(UART) 수신기 패킷 검증 및 FER 기초 연산

대다수의 고성능 조종기 프로토콜(SBUS, CRSF 등)은 16개 이상의 채널 데이터를 압축한 페이로드 바이트 배열과, 프레임의 종단을 알리며 무결성을 검증하는 CRC(Cyclic Redundancy Check) 혹은 시그니처 비트를 내포한다. FC 버퍼로 유입되는 데이터 스트림은 UART 컨트롤러를 거치면서 인터럽트를 발생시키며, PX4 시스템의 인터페이스 수신 데몬은 다음과 같은 기준으로 유효 프레임인지 판별한다.

  • 패리티(Parity) 및 런트 프레임(Runt Frame) 기각: 프레임 길이가 지정된 규격(ex. SBUS의 경우 25바이트)에 미치지 못하거나 바이트 패리티 에러가 통보된 경우 해당 프레임을 폐기한다.
  • CRC 기반 페이로드 무결성 평가: CRSF와 같은 최신 양방향 프로토콜에서는 생성 다항식(Generator Polynomial)을 적용하여 원시 송신 데이터의 오염 여부를 계산하며, 해시가 일치하지 않을 경우 악성 프레임으로 규정하고 버린다.

전체 유입 패킷 대비 이러한 기각 패킷의 비율을 수학적으로 산출한 지표가 프레임 오류율(Frame Error Rate, FER)이다.

2. 시계열 데이터 기반의 이동 평균 FER(Frame Error Rate) 필터링

순간적인 전파 노이즈로 1~2 프레임이 유실된 것을 즉각 통신 두절(Fail-Safe)로 판단하면 비행 안정성이 크게 훼손된다. 따라서 PX4 내부 로직(rc_update 등)은 지수 가중 이동 평균(Exponentially Weighted Moving Average, EWMA) 방식의 저역 통과 필터(Low-Pass Filter)를 사용하여 시계열 FER 지표의 평활화(Smoothing)를 진행한다.

식으로 표현하면 현재 시점 t의 평균 오류율 F_{avg}(t)는 다음과 같다.
F_{avg}(t) = \alpha \cdot F_{current} + (1 - \alpha) \cdot F_{avg}(t-1)
여기에 사용된 F_{current}는 현재 윈도우에서 판단된 에러 패킷의 유무(0 또는 1) 혹은 단위 시간 당 손실률이며, \alpha는 과거 데이터 대비 최신 오류의 가중치를 결정하는 스무딩 팩터 상수이다. 이 평균 오류율 값은 uORB의 input_rc 버스 내 데이터 구조체에 담겨 상위 commander 모듈에 지속 퍼블리싱 되며, 임계값(예: 30% 이상 지속)을 돌파하면 GCS로 경고가 발생하거나 RTL(Return to Launch)이 트리거된다.

3. 스위치 채널의 파편화 방지를 위한 디바운싱(Debouncing) 아키텍처

아날로그 조종기의 내부 물리 스위치 접점이 기계적으로 붙었다 떨어지는 찰나의 순간에는 10~50 밀리초(ms) 단위로 ON/OFF 상태가 요동치는 채널 바운싱(Bouncing) 혹은 글리치(Glitch) 현상이 동반된다. 비상 정지 명령인 킬 스위치가 이런 노이즈로 인해 오인 발동되거나, 반대로 발동 중 순간적으로 해제되는 현상을 막기 위하여 PX4는 이산 시간 기반의 디바운스 필터를 구현한다.

디바운스 필터는 상태 전이(State Transition) 이전에 히스테리시스(Hysteresis) 마진과 시간 적분(Time Integration) 대기열을 요구한다. 즉, 특정 킬 스위치 채널의 PWM 값이 쓰레시홀드(예: 1500us를 넘어 1900us에 도달)를 초과했다고 즉시 모터를 차단하는 것이 아니라, 해당 상태를 나타내는 ’유효한 프레임’이 연속적으로 N회 이상 혹은 지정된 누적 시간 T_{debounce} 이상 변함없이 유지되었음을 평가(isValid() 논리 충족 여부)한 뒤에만 최종 actuator_armed 전환 논리를 트리거한다.

4. Ardupilot 아키텍처와의 신호 무결성 처리 철학 비교

디바운싱 및 패킷 필터링 구조에 있어서, Ardupilot과 PX4는 뚜렷한 아키텍처 설계 차별점을 갖는다.

Ardupilot은 메인 스케줄러를 순회하면서 루프당 설정값(SCHED_LOOP_RATE)에 동기화되는 방식으로 채널의 디바운싱 계산 타이머를 동작시키는 모놀리식 메커니즘을 주로 활용한다. 이 방식은 루프 틱(Tick) 단위의 스케줄링 속도에 직접 의존하게 되는 특징을 보인다.
반면 PX4의 경우 uORB 토픽 배포 전 단인 독립적인 수신 드라이버 데몬 단에서 FER 컷오프 연산과 채널 디바운스 로직을 온전히 전담한다. 하위 계층에서 정제된 깨끗한 데이터만이 uORB 미들웨어로 주입되기 때문에, 최상위 Commander 로직은 복잡한 상태 천이 검증 시 잡음에 대한 연산 자원을 낭비하지 않고 비상 판단의 시간 지연(Latency)을 사전에 차단하는 강력한 비동기식 소프트웨어 모듈화를 구성한다.

이상과 같이 채널 디바운싱 필터 설계와 수학적 FER 모니터링은 비록 최하단 계층(Low Level)에 숨어 있는 로직이지만, ROS2 오프보드(Offboard)와 조종기의 복합 운용 환경 시 돌발적으로 나타날 수 있는 기계적 오류 신호를 격리시키고 확정적인 비정상 상태만을 골라내는 통제 시스템 아키텍처의 파수꾼 역할을 수행한다.
