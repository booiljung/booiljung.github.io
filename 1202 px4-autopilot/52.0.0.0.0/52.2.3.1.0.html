52.2.3.1. 펄스 폭 변조(PWM) 타이머 레지스터의 강제 초기화 및 제어 주기(Control Loop Rate) 독립적 영점(Zero) 출력 강제

무인항공기(UAV)의 제어 체계에서 비상 정지(Kill Switch) 상황이 도래하여 구동기를 물리적으로 차단해야 할 때, 가장 기저(Base) 수준의 통제력 발휘는 메인보드 마이크로컨트롤러(MCU) 내부에 탑재된 실리콘 하드웨어 타이머 모듈을 직접 강제(Overwrite) 제어하는 것이다. 무인항공기 하드웨어(예: Pixhawk FMUv5/v6x 등)의 두뇌 칩셋으로 광범위하게 사용되는 STM32 아키텍처 환경에서, BLDC 모터 구동용 펄스 폭 변조(PWM) 신호는 고성능 타이머(Advanced-control Timers, 예: TIM1, TIM8)와 자원이 풍부한 범용 타이머(General-purpose Timers) 핀의 출력 비교(Output Compare) 기능을 통해 독립적으로 생성된다. 본 절에서는 PX4-Autopilot의 저수준(Low-level) 하드웨어 드라이버 계층이 극단적인 비상 상황에서 어떻게 운영체제(RTOS) 메인 비행 루프 주기(Control Loop Rate)의 스케줄에 구속되지 않고 즉각적으로 타이머 레지스터를 초기화하여, 안전이 판위된 영점(Zero-throttle) 펄스로 강제 치환해 내는지 그 하드웨어적 메커니즘을 상세히 분석한다.

1. 하드웨어 타이머 레지스터의 체계와 PWM 파동 자동 생성 원리

임베디드 시스템에서 모터 전기 변속기(ESC)로 전달되는 수백 헤르츠(Hz) 이상의 초당 PWM 신호 파형은, 소프트웨어 커널의 메인 루프 스레드가 매 순간 CPU 점유 시간을 할애하며 비트 뱅잉(Bit-banging)으로 인위적인 파형을 만들어내는 구조가 아니다. 이들은 한 번 초기 환경 세팅(주기 파라미터 및 펄스 점유율)을 완료해 두면 이후에는 칩셋의 물리적 타이머가 백그라운드에서 알아서 주기적으로 전압 파형을 자동 토글(Toggle)하는 비동기 DMA/하드웨어 인터럽트 처리에 온전히 의존한다.

이를 위해 STM32의 타이머 구조체는 다음과 같은 핵심 레지스터를 매칭한다.

  • ARR (Auto-Reload Register): 타이머 카운터가 맥시멈으로 카운팅을 수행할 상한 임계값을 결정하여, 전체 PWM 파형의 기본 주기(Frequency Base)를 설정한다. (통상적으로 400Hz 빠른 제어 주기를 사용하는 경우 2.5밀리초(ms) 물리 시간에 매칭되는 스케일 틱(Tick)을 담는다.)
  • CCR (Capture/Compare Register): 순차적으로 오르는 카운터 값과 하드웨어적으로 비교(Compare)되어, 실제 신호가 High 전압(3.3V) 펄스로 유지될 시간 폭(Duty Cycle Pulse Width)을 결정한다. 모터가 뿜어낼 실질 추력을 즉각적으로 제어하는 심장부 변수이다.

일반적인 고속 비행 상태에서 PX4 코어의 믹서 매트릭스(Mixer Matrix) 모듈은 초당 400회의 주파수(400Hz Control Loop) 속도로 연산된 다축 제어 할당량을 바탕으로, 하단의 각 채널 CCR 레지스터 섀도우 맵에 다음 출력을 통보(Update 예약) 해둔다. 그러면 타이머가 오버플로우(Overflow)되어 다음 주기가 막 시작되는 순간에 하드웨어 트리거(Update Event)가 발생하며, 이 예약된 버퍼 값이 메인 동작 레지스터에 조용하고 매끄럽게 흡수(Shadow Transfer)된다.

2. 제어 주기(Control Loop Rate) 종속성에 따른 지연 위험성과 탈피 설계

안전을 최우선으로 담보해야 할 비상 정지 통제 명령 시스템 아키텍처에서 가장 극도로 배척해야 할 병목은 이른바 제어 주기 연동성(Coupled Synchrony)으로 일컬어지는 타임 딜레이(Time Delay) 지연 현상이다.

동기식 통제 설계의 치명적인 시나리오를 가정해 보자. 만약 커널 내 메인 루프 스레드가 과도하게 밀집된 센서 데이터 융합 계산, 또는 예측 불가능한 부동소수점 예외(Floating-point Exception 트랩)로 말미암아 지정된 스케줄 처리 기한 라인을 3~5 밀리초 남짓 초과(Deadline Miss)하는 랙이 걸릴 경우, 이 스레드의 하위에 속해있던 믹서 모듈 자체의 계산까지도 덩달아 지연 처리된다. 그 결과로 비상 정지를 알리는 이른바 \text{PWM} = 900\mu s 커맨드가 하드웨어 레지스터 쪽으로 제때 배달되지 못하고 메모리에 묶인다. 즉, 비상 정지 조치의 실제 물리적 발현이 K번째 타임스탬프 스핀에서 K+1번째 스핀으로 강제로 연기(Postponed)되며, 그 찰나의 지연 시간 동안 드론 모터는 가장 마지막에 통신 제어를 받았던 엄청난 고속 회전 관성 펄스(\text{PWM}_{prev}) 값으로 끊임없이 기계적 회전을 영속한다.

비상 하강 및 완전 정지가 요구되는 위기 국면을 한시라도 타파하기 위해 PX4의 맨바닥 계층, 이른바 IO 드라이버 계층(NuttX 커널 스페이스)은 소프트웨어 비상 정지 플래그 신호를 직통으로 감지하는 그 찰나, 기존의 육중한 메인 비행 제어 루프(Flight Control Loop)의 정형적인 분기 계산 흐름 구조에서 지체 없이 구조적으로 탈주해 버린다.

3. 타이머 채널 레지스터의 원자적(Atomic) 강제 초기화 로직 구현

이러한 물리 단자를 조향하는 PX4 플라이트 펌웨어의 극 최하단 소형 계층 (예: src/drivers/pwm_out 류의 드라이버 모듈) 소스 코드상에는, 생명의 안전을 최종적으로 담보하는 임계 보호 함수들(예컨대 up_pwm_servo_disarm() C++ 루틴)이 고정 배치되어 있다. 이 함수는 외부 요인에 의해 비상 통제권 지시가 시스템 런타임(Runtime)에 발동하면 지체 없는 응답성을 가지기 위해 아래와 같은 원자성 파괴적(Atomic and Destructive) 파라미터 초기화 과정을 단독으로 결행한다.

3.1 인터럽트 스위치 마스킹(Interrupt Masking)을 통한 원자성 담보 보장

모든 드라이버 타이머의 포트 비교 제어 채널(CCR) 레지스터 구조체를 영점 컷오프 전압(Zero-Thrust 출력 베이스에 매칭되는 Failsafe PWM 값. 일반적으로 900 µs 주변값으로 셋업)으로 완전히 재구조화하기 단 1 나노초 전, CPU의 프라이머리 마스크 레지스터(Primary Mask Register) 프레임을 침범하여 일체의 외부/내부 선점형 언밸런스 인터럽트를 차단(irqsave 함수 호출)한다.
이는 비상 차단 파라미터를 메모리에 우겨넣는 아주 짧은 수 마이크로초의 간격 동안에도, 다른 비동기적 하드웨어 인터럽트 루틴이나 스케줄러가 절대 난입하지 못하도록 보호함으로써, 각 채널별 레지스터 메모리 무결성이 찢어지거나 덜 덮어써지는(Data Tearing) 아티팩트 현상을 근거 차단하기 위한 필수 군사적 하드웨어 제어 행위이다.

3.2 섀도우 레지스터 강제 업데이트 이벤트의 폭주 (Update Generation/EGR Triggering)

일반적으로 타이머 마이크로칩 설계는 High 레벨 맥파 펄스가 포트를 타고 방출하고 있는 중간 도중에 외부 지시로 CCR 메모리 값을 함부로 급변시킬 경우, 듀티비 파형 자체가 잘려나가거나(Glitch Spike) 위상 벡터가 통째로 뒤틀려 전기적으로 모터를 태워먹는 교차 문제를 막기 위해 ‘프리로드(Preload) 보호 버퍼’ 기능을 활용하여 다음 타이머 주기가 올 때까지 섀도우 버퍼 이면에 대기하며 숨겨둔다.
하지만 비상 파괴 본연의 상황은 우아한 파형의 미시적 일그러짐 따위를 배려할 여유가 일절 없다. 물리적인 추력을 유발하여 기체를 가속하는 위험천만한 에너지 펄스 그 자체를 지금, 당장, 마이크로초 내에 메스로 자르듯이 끊어버려야만 생존율이 담보된다.
따라서 PX4 최하층 저수준 드라이버 루틴은 액티브 채널의 CCR 체인 레지스터 메모리 포진값에 영점(Zero) 펄스 마이크로초 상수 변수를 폭력적으로 찔러 넣은 후, 이벤트 생성 레지스터(예컨대 STM32 데이터 시트의 TIMx_EGR 구조 중 UG 비트)를 소프트웨서적으로 곧장 강제 토글(Toggle)하여 대기 상태 중인 오프 트랙 신호 상태값을 외부 I/O 핀으로 단지 내리 꽂아(Flush-out) 버린다.

4. 시스템 강건성 설계 및 엔지니어링 패러다임 결론

이러한 가장 원초적인 타이머 콤퍼넌트 단위체 레지스터 메모리를 상대로 벌이는 초단기적이고 강제적인 핀 단위 방식의 폭력적 덮어쓰기(Hardware Overwrite) 로직 철학은, 최상단의 인공지능형 부동소수점 믹서 알고리즘과 신경망 비정형 자율 항법 제어 파이프라인이 아무리 복잡 다난하게 서로 얽혀 타임오프(Timeoff) 및 응답 랙(Lag)이 생길 지언정 그 소프트웨어적 지연 현상과는 완전히 독립적이며 고립적으로 작용(Runs Independently)함을 보장한다.

안전 임계라는 수식어가 대변하는 고지능 무인 비상 정지 체계는, 궁극적으로 수학 방정식의 소프트웨어 시뮬레이션 영역만으로 구축되는 공간이 아니다. 이 공간은, 무기질 메탈 핀(Metal Pin) 전극과 MOSFET 트랜지스터(Transistor) 전자 게이트가 통리적으로 만나는 임베디드 실리콘 공학 기저의 바닥 영역에서 가장 최후 통첩(Last Resort)의 하드웨어적 지시 신뢰성을 확보한 뒤 솟아올라야 한다. 따라서, 제어 소프트웨어 주기를 미련 없이 우회하며 실리콘 하드웨어 타이머 레지스터 풀을 통째로 일괄 초기화 조치 해버리는 PX4-Autopilot의 깊고 투박한 내부 아키텍처 제어 구조는, 그들이 표방하는 엄격하고 파괴적이기까지 한 생존 안전성 철학 본질이 오직 펌웨어 오픈 소스 코드 라인 구석구석에 강철처럼 어김없이 구체화되어 뿌리 내려져 있음을 그 무엇보다 극명하게 방증하는 거울이다.
