52.4.3.2 고휘도 RGB 발광 다이오드(LED)의 시각적 명시성 확보를 위한 듀티 사이클(Duty Cycle) 비대칭 점멸 패턴 동적 할당

본 절에서는 PX4-Autopilot(기준 펌웨어 버전: v1.14.0)에서 비상(Critical) 또는 예외 상황 시 기체의 상태를 외부 관찰자에게 직관적으로 전파하기 위한 고휘도 RGB 발광 다이오드(LED) 제어 알고리즘을 분석한다. 단순한 색상 변화를 넘어, 인간의 시각 인지 심리학(Visual Cognitive Psychology)에 기반한 비대칭 듀티 사이클(Asymmetric Duty Cycle) 점멸 패턴을 동적으로 할당하는 과정과 내부 소프트웨어 아키텍처를 상세히 기술한다.

1. 시각적 명시성(Visual Conspicuity)과 비대칭 점멸 패턴

무인기는 야외 환경의 강한 직사광선(Direct Sunlight) 아래에서 운용되는 경우가 많기 때문에, 단순히 LED 칩에 전력을 지속적으로 인가하는 상태(Solid On)로는 색상의 가시성이 극도로 제한된다. 따라서, 인간 시각 세포가 급격한 명암 대비에 민감하게 반응한다는 특성을 활용하여 특정 주파수(Frequency)에서의 점멸(Blinking) 패턴을 사용한다.

특히 치명적 오류(Critical Failure) 시점에는 단순히 50:50 듀티 사이클로 점멸하는 대칭 패턴(Symmetric Pattern)을 지양하고, 발광 시간(On-time)을 짧게 하되 오프 시간(Off-time) 대비 매우 짧고 강한 여러 번의 연쇄 발광(Strobe)을 발생시키는 비대칭 듀티 사이클 기법을 사용한다. 이 ‘스트로보스코프(Stroboscope)’ 형태의 광학 시그니처는 뇌에 강력한 경고 자극을 주어 시각적 명시성(Visual Conspicuity)을 극대화한다.

2. LED 제어 아키텍처 및 uORB 통신 흐름

PX4 시스템 내에서 LED 하드웨어를 직접 제어하는 모듈은 각 보드 고유의 드라이버(예: I2C 기반 커스텀 LED 컨트롤러, TCA62724 등)에 의존하며, 이 드라이버들은 독립적인 백그라운드 태스크나 워크 큐(Work Queue) 위에서 동작한다. 상태 변화를 감지한 Commander 모듈은 led_control uORB 토픽을 통해 점멸의 색상(Color)과 모드(Mode)를 명령한다.

    class Commander {
        +bool _status_critical
        +publish_led_control()
    }
    class led_control_s {
        <<uORB Message>>

    }

    Commander --> led_control_s : Publish
    led_control_s --> LEDController : Subscribe
    LEDController --> HardwareLayer : Execute Duty Cycle Update

LED 컨트롤러 모듈은 구독한 메시지의 priority 필드를 비교하여, 현재 점멸 중인 덜 중요한 상태(예: GPS 3D Fix 확보 전 황색 점멸 등)를 선점(Preemption)하고 최고 우선순위인 비상 적색 스트로브(Emergency Red Strobe) 모드로 즉각 전환한다.

3. 소스 코드 구현: 동적 듀티 사이클 생성 로직

PX4의 src/lib/led/led.cpp 및 관련된 드라이버 소스 코드를 살펴보면, 각 모드(Mode)에 대하여 고정된 타이머 주기가 배열 리스트 형태로 구현되어 있음을 알 수 있다. 이 중 MODE_BLINK_FAST 등과 같은 비상 모드는 비대칭 비율을 나타낸다.

// LED 점멸 상태 머신 및 듀티 사이클 제어 (유사 코드)
void LedController::run() {
    uint32_t now = hrt_absolute_time();

    if (_led_state.mode == led_control_s::MODE_BLINK_FAST) {
        // 비대칭 듀티 사이클: 100ms On, 300ms Off
        const uint32_t on_duration = 100_ms;
        const uint32_t off_duration = 300_ms;
        const uint32_t total_period = on_duration + off_duration;
        
        uint32_t phase = (now - _start_time) % total_period;

        if (phase < on_duration) {
            set_hardware_color(_led_state.color); // 최대 전류 인가
        } else {
            set_hardware_color(COLOR_OFF);        // 완전 소등
        }
    }
    // 다음 상태 천이를 위한 스케줄링
    schedule_delayed_work(&_work, TICK_INTERVAL);
}

위의 예제처럼 100ms On / 300ms Off와 같은 비대칭 듀티 사이클은 픽셀에 공급되는 평균 소비 전력(RMS Power)을 낮추어 고휘도 투광으로 인한 LED 다이의 발열 손상을 방지하는 동시에 방전율을 고려하는 배터리 임계 상황에서 전력을 아낄 수 있다는 부가적인 장점이 존재한다.

4. Ardupilot LED 매핑 아키텍처와의 비교

Ardupilot은 전통적으로 Toshiba LED 등 특정 외부 LED 모듈에 대한 전담 드라이버를 통해 미리 정해진 상태 기반 LED 패턴 엔진(AP_Notify)을 구동하며, LED의 구체적 듀티 사이클보다는 “어떤 상태인가?“에 초점을 맞춰 색상을 관리하는 데 치중한다.

반면, PX4-Autopilot은 led_control 토픽을 통해 색상(Color), 모드(Mode, 점멸 속도 및 정적 켜짐 상태), 깜빡임 횟수(num_blinks)까지 매개변수화(Parameterize)하여 메시지에 담아 전송한다. 이러한 설계는 비상 제어 모듈이 복잡한 소프트웨어 라이브러리를 직접 건드리지 않고도 요청 지향적인(Request-oriented) 비대칭 점멸 패턴을 시스템의 어느 위치에서든 동기적으로 할당할 수 있는 극적인 형태의 유연성을 제공한다.

5. 비상 관제 요약 및 결론

낙하산 사출 및 프롭락(Kill Switch)이 걸린 기체의 추락 현장을 지상 요원(Ground Personnel)이 육안으로 빠르게 포착하기 위해서 고휘도 RGB LED의 비대칭 스트로보 제어는 핵심적이다. 특히 지상 관제 시스템(QGroundControl)과 연동하여, LED 패턴에 해당하는 MAVLink SYS_STATUS가 발송될 때 관제 화면 내 UI 역시 LED와 동일한 주파수로 깜빡이는 시각 효과를 적용한다면 인간-기계 인터페이스(HMI)의 인지 일관성을 비약적으로 높여 사고 조치를 신속하게 이룰 수 있다.
