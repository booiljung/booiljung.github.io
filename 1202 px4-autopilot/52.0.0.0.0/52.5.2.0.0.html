52.5.2 제어 할당기(Control Allocator) 모듈 내 모터 오버라이드(Override) 연산 파이프라인 우회

1. 개요 및 설계 철학

PX4-Autopilot의 최신 아키텍처(펌웨어 버전 v1.14 이상)에서는 기존의 정적 믹서(Static Mixer) 시스템을 대체하여 동적 **제어 할당기(Control Allocator)**가 도입되었다. 비상 상황(Kill Switch 활성화 또는 비행 종료) 발생 시, 비행 제어기(Flight Controller) 루프에서 계산된 가상 제어 명령(토크 및 추력)을 무시하고 즉각적으로 모든 모터와 액추에이터의 출력을 차단(Disarm 또는 PWM 최소값 출력)해야 한다. 이를 위해 제어 할당기 모듈 내부에는 정상적인 연산 파이프라인을 우회(Bypass)하는 모터 오버라이드(Motor Override) 메커니즘이 설계되어 있다.

2. 연산 파이프라인 및 우회 메커니즘

정상 비행 상태에서는 자세 제어기(Attitude Controller)가 계산한 3축 토크(Roll, Pitch, Yaw Torque)와 3축 추력(Thrust) 명령이 vehicle_torque_setpoint 및 vehicle_thrust_setpoint uORB 메시지로 퍼블리시된다. 제어 할당기는 이를 구독하여 기하학적 형상(기체 프레임 구조)에 따른 행렬 연산을 수행, 각 개별 액추에이터의 목표 신호 크기(actuator_motors, actuator_servos)로 맵핑(Mapping)한다.

하지만 비상 정지 상태에서는 이 행렬 연산 자체가 시간 지연을 유발하거나, 수치적 불안정성으로 인해 예기치 못한 출력 변동을 일으킬 위험이 있다. 따라서 제어 할당기 메인 루프에서는 uORB 메시지인 actuator_armed 내의 manual_lockdown 또는 force_failsafe 플래그를 가장 먼저 평가(Evaluation)한다.

3. Ardupilot 대비 아키텍처 차이

Ardupilot의 경우 AP_Motors 라이브러리가 모터 믹싱과 출력 제한을 동시에 담당한다. 비상(E-Stop) 명령이 하달되면 AP_Motors::set_interlock() 및 output_min() 함수를 호출하여 모터 출력을 즉각 하한선으로 제한한다.

반면 PX4-Autopilot은 독립된 제어 할당 스택(ControlAllocation C++ 클래스)을 별도의 프로세스/스레드로 분리하여 운영 체제(NuttX RTOS) 차원에서의 모듈성을 극대화했다. PX4에서는 오버라이드가 단순히 최하단 드라이버에서만 일어나는 것이 아니라, 제어 할당 모듈 내부에서 할당 행렬(Allocation Matrix) 연산 전에 가로채어 연산량 낭비를 막고 즉시 disarmed 상태의 기본값(Default PWM)을 하위 드라이버(예: pwm_out, dshot) 버퍼에 기록한다.

4. 소스 코드 분석 (control_allocator.cpp)

PX4의 src/modules/control_allocator/ControlAllocator.cpp 파일 내부의 주요 흐름은 다음과 같다.

  1. 상태 감지: 루프 갱신 함수인 Run() 내에서 _armed_sub.update()를 호출하여 최신 플래그를 가져온다.
  2. 조건부 분기: armed.manual_lockdown 필드가 true인 경우, ControlAllocation::set_effectivenes_matrix()를 통해 계산된 할당 행렬을 적용하는 연산을 시작하지 않고 곧바로 _actuator_motors_pub.publish()에 0 또는 초기화된 출력을 밀어넣는다.
  3. 가속화된 파이프라인 처리: 부동 소수점 행렬 곱셈을 스킵하므로 시스템 부하가 일시적으로 감소하며, 이는 크래시(Crash) 직후 발생할 수 있는 시스템 오버헤드나 센서 노이즈로 인한 스케줄링 지연에서도 가장 확실한 반응 속도를 보장한다.

5. 요약

제어 할당기 모듈 내 모터 오버라이드 연산 파이프라인 우회 기술은 PX4-Autopilot의 비상 제어 시스템에 있어 “Fail-Fast(빠른 실패)” 설계 철학을 반영한 대표적인 사례이다. 복잡한 다변수 수학적 행렬 연산을 원천 차단하고 구조적 하단 제어 레벨로 즉시 최소화된 출력을 브로드캐스트하여 추가적인 에러 전파를 방지한다.
