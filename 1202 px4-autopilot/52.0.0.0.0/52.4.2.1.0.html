52.4.2.1. 3축 가속도계 데이터의 동적 노이즈 저역 통과 필터링 후 직교 좌표계 가속도 벡터 스칼라 크기의 중력 영점 수렴(자유 낙하) 검출 조건식

1. 개요 (Introduction)

무인항공기(UAV)의 항법 및 제어 시스템이 직면하는 심각한 위험 인자 중 하나는 모터 클러스터 고장에 의한 강하, 즉 자유 낙하(Free-fall) 현상이다. PX4-Autopilot은 이 순간적인 G-포스(G-force) 반전을 식별하기 위해 3축 가속도계(3-axis Accelerometer) 센서의 로우 레벨(Raw Level) 측정치를 활용하며, 복잡하게 혼입된 고주파 진동(High-frequency Vibration)을 전처리한 다음 수학적 스칼라 크기를 판별하는 정량적인 조건식을 사용한다.

2. 동적 노이즈(Dynamic Noise) 억제 및 저역 통과 필터링 (Low-Pass Filtering)

비행 중인 드론, 특히 멀티로터의 프레임에는 구동체(모터 및 프로펠러)에 의한 고주파 진동 노이즈가 강하게 유도된다. 관성 측정 장치(IMU)는 센서 대역폭(Bandwidth) 내에서 이러한 진동을 민감하게 포착하므로, 단순히 센서에서 출력되는 원형(Raw) 데이터에 임계값을 적용하면 거짓 양성(False Positive) 에러가 속출하게 된다.
이를 억제하고자 PX4의 센서 버스 파이프라인(Sensor Bus Pipeline)에서는 데이터 스트림에 디지털 **저역 통과 필터(Low-Pass Filter, LPF)**를 적용한다.
일반적인 1차 저역 통과 필터 이산화(Discretization) 수식은 다음과 같다.
y_n = \alpha x_n + (1 - \alpha) y_{n-1}
여기서 x_n은 현재의 가속도계 샘플(Raw Data), y_{n-1}은 이전 스텝의 필터링 된 값이며, \alpha는 차단 주파수(Cutoff Frequency)와 샘플링 레이트에 의해 결정되는 가중치(Weight) 파라미터이다. 이 과정을 통해 모터 공진 주파수 대역이 소거되고 기체 순수 운동 가속도만 남게 된다. (모듈 설정 시 수학적 기호 표기에 유의하라.)

3. 가속도 벡터 스칼라 크기 산출 및 중력 영점 수렴 (Vector Scalar Magnitude and Null Convergence)

3차원 직교 좌표계(XYZ Cartesian Coordinate)에서 필터링을 거친 가속도 벡터 \mathbf{a} = [a_x, a_y, a_z]^T에 대한 스칼라 크기(Magnitude)는 다음과 같이 유클리드 노름(Euclidean Norm)으로 계산된다.
a_{mag} = \sqrt{a_x^2 + a_y^2 + a_z^2}
정상적으로 호버링 중이거나 등속 비행 중인 기체는 센서 프레임의 기준에서 볼 때 중력 원심력과 수직항력의 관계로 인해 a_{mag} \approx 9.81 \text{ m/s}^2 (1G)의 평균 스칼라 값을 나타낸다. 그러나 모터 출력이 0에 가까워지고 기체가 자유 낙하를 시작하게 되면, 센서 칩 내부의 미세 지지 스프링에 가해지던 관성력이 사라져 a_{mag}는 급격히 0 \text{ m/s}^2 (0G)인 **중력 영점(Gravity Null)**으로 수렴(Convergence)하게 된다.

3. PX4 소스 코드의 자유 낙하 검출 조건식 (Failure Detection Condition)

PX4의 Failure Detector 서브모듈은 자유 낙하를 확인하기 위해 다음과 같은 프로그래밍적 조건식을 만족하는지 지속 평가한다.

stateDiagram-v2
    [*] --> Read_IMU: 3축 가속도계 토픽 구독 (uORB)
    Read_IMU --> Apply_LPF: 진동 노이즈 저역 통과 필터 적용
    Apply_LPF --> Calc_Magnitude: a_mag = sqrt(ax^2 + ay^2 + az^2)
    
    state Decision_Tree {
        Calc_Magnitude --> Compare: a_mag < G_NULL_THRESH ?
        Compare --> Timer_Accumulate: 조건 충족 시 타임 윈도우(Time Window) 누적
        Timer_Accumulate --> Confirmed: 누적 시간이 MIN_FALL_TIME 초과 시 추락 확정
    }
    
    Confirmed --> Publish_Failure: vehicle_status 비상 플래그 게시
    Publish_Failure --> [*]

이 로직의 핵심은 \vert a_{mag} \vert < Threshold 가 아주 잠깐 발생했다고 해서 추락으로 단정 짓지 않는다는 점이다. 일시적인 공기 역학적 드롭(Drop)과 구분하기 위해 일정 기간(Time Hysteresis) 조건을 부가하여 신뢰성을 높였다.

4. Ardupilot 대비 노이즈 필터링 및 조건 차이 (Comparison with Ardupilot)

Ardupilot 또한 EKF3의 고도 변화율과 Crash Check를 활용하여 추락을 감지하지만, INS_ACCEL_FILTER 등 하위 덴핑(Damping) 필터의 파라미터를 사용자가 상대적으로 직관적으로 만질 수 있도록 개방해 놓았다. Ardupilot은 \vert a_{Z} \vert 축의 역전 및 낙하 현상에 조금 더 가중치를 두는 반면, PX4-Autopilot은 자율 비행(Offboard) 관제와 다중 센서 퓨전 안정성을 목적으로 구조화된 FailureDetector 객체 내부에서 3차원 스칼라 에너지 변화량을 포괄적으로 계산하여 통합한다는 점에 차별화가 된다.

5. 결론 (Conclusion)

3축 가속도계의 측정 노이즈를 시계열적으로 평활화(Smoothing)한 후 산출하는 스칼라 에너지의 영점 수렴 판별식은 기구학적 측면에서 가장 명확하게 기체의 자유 낙하를 수학적으로 증명할 수 있는 수단이다. PX4-Autopilot 환경에서는 이러한 알고리즘 최적화를 통해 치명적인 에러 플래그 남발(False Alarm)을 억제하면서 생존율을 보호하고, 지상으로의 2차 피해를 막는 역할을 성공적으로 수행한다.
