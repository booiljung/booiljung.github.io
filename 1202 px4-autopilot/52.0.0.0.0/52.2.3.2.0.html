52.2.3.2 디지털 통신 프로토콜(DShot)의 비동기적 제어 정지 프레임 전송 및 직류 링크(DC Link) 과전압 방지(Anti-Spike) 로직

초소형 무인항공기(UAV) 및 다중 로터(Multirotor) 시스템의 제어 주기(Control Loop)가 고속화됨에 따라, 비행 제어기(Flight Controller, FC)와 전자 변속기(Electronic Speed Controller, ESC) 간의 통신 지연을 최소화하기 위해 디지털 통신 프로토콜인 DShot(Digital Shot)이 널리 채택되었다. 비상 정지(Kill Switch) 상황이 발생할 경우, DShot 프로토콜은 아날로그 방식(PWM)이 지닌 갱신 주기(Refresh Rate)의 한계를 극복하고 매우 즉각적이며 비동기적인 제어 정지 명령을 하달할 수 있다. 본 절에서는 비상 정지 시 DShot 프로토콜을 통한 프레임 전송 로직의 물리적/소프트웨어적 메커니즘과, 모터의 급격한 정지에 수반되는 역기전력(Back-EMF)으로 인한 직류 링크(DC Link) 과전압 방지(Anti-Spike) 기술을 심도 있게 분석한다.

1. DShot 프로토콜의 비동기적 제어 정지 프레임(Asynchronous Control Stop Frame) 전송 기법

DShot 프로토콜은 16비트(16-bit)의 디지털 데이터 패킷을 사용하여 ESC에 직접적인 스로틀(Throttle) 값과 제어 명령을 전달한다. 패킷은 11비트의 스로틀/명령 데이터, 1비트의 텔레메트리(Telemetry) 요구 비트, 그리고 4비트의 결함 검사용 CRC(Cyclic Redundancy Check)로 구성된다. 예컨대 11비트 값 0은 제어 강제 회수 및 모터 정지(Disarm/Zero Throttle)를, 값 1~47은 특수 명령(디렉션 변경 텔레메트리 요구 등)을, 48~2047 구간은 실제 스로틀 퍼센티지로 매핑된다.

1.1 동기적 갱신(Synchronous Update) 대 비동기적 개입(Asynchronous Preemption)

일반적인 비행 상태에서 PX4-Autopilot은 자이로스코프(Gyroscope) 필터링 및 자세 제어(Attitude Control) 루프 주기에 맞추어 동기적으로 DShot 프레임을 전송한다. 그러나 비상 정지가 발동되면 시스템은 메인 루프 주기와 무관하게 즉각적으로 조치해야 한다. PX4-Autopilot(펌웨어 v1.13 이상 기준)에서는 마브링크(MAVLink) COMMAND_LONG 수신이나 하드웨어 원격 제어기(RC) 스위치를 통한 강제 종료(Kill) 인터럽트 감지 즉시, 스케줄링 대기 중이던 DMA(Direct Memory Access) 혹은 타이머 기반 DShot 데이터 버퍼를 비운다. 그 후 영입력(Zero Throttle, DShot 패킷 값 ‘0’) 명령 패킷을 가장 높은 시스템 우선순위로 비동기 송출(Asynchronous Preemption)하게 된다.

    participant Timer as 하드웨어 타이머/DMA
    participant ESC as 전자 변속기 (DShot ESC)
    participant Motor as BLDC 모터

    FMU->>Timer: 일반 DShot 프레임 (Throttle 1500 매핑)
    Timer->>ESC: 패킷 전송 (CRC 4-bit 포함)
    ESC->>Motor: 3상 전류 출력 유지
    Note over FMU: [비상 인터럽트 발생] (Kill Switch)
    FMU-->>FMU: 제어기 루프 정지 및 하위 액추에이터 상태 비활성화
    FMU->>Timer: DShot 출력 버퍼 덮어쓰기 (Data = 0)
    FMU->>Timer: 비동기 트리거(Asynchronous DMA Trigger)
    Timer->>ESC: 제어 정지 프레임 전송 (Value: 0, 패킷 우선 개입)
    ESC->>Motor: 능동 제동 (Active Braking) 인가 / 출력 차단

이러한 비동기식 프레임 주입 구조는 실시간 운영체제(NuttX RTOS) 단위에서 즉각적인 콘텍스트 스위칭(Context Switching) 및 하드웨어 가로채기를 통해 구현되므로, 메인 루프 타이머 틱(Timer Tick)에 얽매이지 않는 극한의 빠른 차단 성능을 보장한다.

2. 직류 링크(DC Link) 과전압 방지(Anti-Spike) 및 능동 제동 메커니즘

비상 정지 명령 프레임을 수신한 현대적인 브러시리스 직류(BLDC) 모터용 ESC는 대부분 능동 제동(Active Braking 또는 Damped Light 모드) 기술을 채택하고 있다. 능동 제동은 모터의 관성 회전을 급격히 감속시키기 위해 3상 브리지(Bridge) 회로의 MOSFET을 제어하여 코일을 단락(Short)시키고, 모터 자체를 발전기(Generator) 삼아 운동 에너지를 전기 에너지로 역변환시켜 소산하는 방식이다.

2.1 역기전력(Back-EMF) 발생과 직류 링크 전압 상승 상승 현상

비행 중 최대 추력 근방에서 비상 정지 스위치가 인가되면, 고속 회전하던 프로펠러 메커니즘의 높은 운동 에너지가 즉각 역기전력(Back-EMF) 형태로 변환된다. 생성된 전류는 ESC 내부 브리지의 프리휠링 다이오드(Freewheeling Diode)나 MOSFET 내부 경로를 타고 전원 방향으로 역류(Regeneration) 조치된다.
이는 곧 전원 분배 보드(Power Distribution Board, PDB) 및 배터리가 직렬 결합된 시스템 전체의 메인 버스 전압(직류 링크 전압, DC Link Voltage)을 비정상적으로 스파이크(Spike) 시키는 원인이 된다.

특히 배터리의 내부 저항(Equivalent Series Resistance, ESR)이 높을 경우 이 역류 에너지를 완전히 흡수하지 못하게 된다. 이로 인해 유도된 급격한 스파이크는 순간적으로 회로 상의 시스템 허용 전압(예: 커패시터 정격 한계, BEC 컨버터 최대 허용 입력 한계 등)을 초과 돌파할 위험성이 매우 높다. 최악의 경우 온보드 5V/3.3V 전압 레귤레이터가 관통 파괴를 일으켜 비행 제어기의 마이크로컨트롤러 유닛(MCU)이나 주변 센서가 일시에 소손될 수 있다.

2.2 하드웨어 및 소프트웨어 단위의 과전압 억제(Anti-Spike) 완화 기법

이러한 파괴적 볼티지 스파이크(Voltage Spike) 현상을 제어하기 위해 시스템 레벨에서 하드웨어 및 소프트웨어적인 억제 로직이 상호 보완적으로 작동한다.

  1. 소프트웨어 및 DShot 텔레메트리 피드백 기반 제한 논리 (Soft-Limiting)
    ESC가 DShot 텔레메트리 라인을 통해 메인 버스의 실시간 전압, 소모 전류, 그리고 모터 회전 속도(eRPM) 정보를 PX4 제어기 측으로 반환한다. 전압의 시간에 따른 상승률(\frac{dV}{dt})이 급격하거나, 피크 전압이 미리 설정된 임계치(Threshold) 한계선에 근접할 경우, 일부 지능형 ESC(AM32, BLHeli_32 기반 고급형 등)는 자체 펌웨어 단에서 역기전력을 저감하기 위해 능동 제동 토크(Braking Torque)를 짧게 이완시키는 소프트 댐핑(Soft Dumping) 또는 능동 프리휠링 제어를 수행한다. 이는 급정지 중에도 메인 회로 소손을 막는 스마트 안전 핀 역할을 한다.

  2. 하드웨어 디커플링 및 스너버(Snubber) 캐패시터 뱅크 설계
    ESC의 전원 라인 접점에는 본질적으로 외부 역기전력 방지 및 고주파 차단을 위해 매우 낮은 내부 저항(Low ESR) 특성을 가지는 전해 커패시터(Electrolytic Capacitor)들이 장착된다. 스파이크 유입 시, 이 커패시터 뱅크가 일시적 버퍼(Damping) 역할을 수행한다. 이때 직류 링크의 과도 전압 방정식은 대략적으로 다음과 같다.

    C \frac{dV_{\text{DC}}}{dt} = I_{\text{Regen}} - I_{\text{Load}}

    여기서 I_{\text{Regen}}은 비상 정지 시 모터로부터 역류해 들어오는 스파이크 전류 강도이고, C는 디커플링 커패시터 뱅크의 총 용량이다. 전압 피크를 안전 구역으로 제한하려면 충분히 여유 있는 단면적의 C 설계 및 TVS(Transient Voltage Suppressor) 보호 다이오드가 시스템 통합 차원에서 직렬/병렬로 요구된다.

3. PX4 통합 및 아키텍처적 고려사항 (System Integration & Architecture)

PX4 파라미터(Parameter) 시스템 내에서는 비상 정지와 관련된 여러 안전 기준 및 통신 레이트 파라미터가 정의된다. 예컨대 기체 동역학적 특성에 따라 통신 속도 프로파일을 선택하는 최적화가 필수적이다.

  • DSHOT_CONFIG: DShot 통신 보드 레이트 속도를 설정한다 (예: DShot150, 300, 600 등). 속도가 높을수록(비트 시간폭 단축) 인터럽트 발생 즉시 “제로 값(Value 0)” 프레임 조성이 완료되는 데 걸리는 절대 시간이 밀리초 단위 이하 수준으로 축소되므로, 비동기 응답 특성이 더욱 예민하고 우수해진다.

3.1 Ardupilot 통신 계층과의 아키텍처 비교

Ardupilot 환경 역시 DShot 프로토콜을 광범위하게 지원하지만, 비상 정지(Motor Interlock / eStop) 인터럽트를 처리하는 스케줄링 로직 구조에는 객체 지향적 차이가 존재한다.
Ardupilot의 스케줄러 계층은 전반적으로 직렬화(Serialization)된 구동 명령 큐에 어느 정도 의존하여 메인 루프 주기와 동반 갱신되는 특성이 상대적으로 빈번하다.
반면 PX4-Autopilot은 uORB (Micro Object Request Broker) 통신 토폴로지 내에서 구동계 제어 토픽인 actuator_controls (또는 최신 프레임워크의 vehicle_command) 시스템 아키텍처가 완전히 모듈화되어 분리 구동된다. 하드웨어 인터럽트가 uORB 버스에 브로드캐스트되는 순간, 모터 출력 모듈(output_pwm 이나 DShot 드라이버 데몬) 자체의 독립된 콜백(Callback) 스레드가 직접 반응해 다이렉트 메모리 쓰기(Direct Register Write)를 강행하는 구조적 이점이 있다. 이는 RTOS 기반의 스레드 풀(Thread Pool) 이점으로 비동기 특성을 극대화시켜 극히 공격적인 지연 감축(Latency Reduction)을 도모한다.

4. 요약

DShot 기반 디지털 통신 구동 시스템은 비상 정지 상황에서 기존 PWM 방식에 내재된 클럭 주기 위상차를 무시하고 텔레메트리/제어 패킷을 비동기적으로 강제 삽입(Asynchronous Preemption)할 수 있어 대단히 신속히 회전자를 정지시킨다.
그러나 이와 같이 급격하게 진행되는 능동 제동 매커니즘은 막대한 역기전력(Back-EMF)을 필연적으로 동반하며, 전원 계통망의 메인 직류 링크(DC Link) 과전압(Voltage Spike) 파괴를 유발할 소지가 크다. 결과적으로 신뢰성 있는 무인항공기 구축을 위해서는, 소프트웨어 단에서의 즉각적 통신 정지 로직 및 DShot 피드백 제어와 스너버 커패시터, TVS 소자 등을 포괄하는 하드웨어적 과전압 방지(Anti-Spike) 설계 전략이 필연적으로 동반되어야 한다.
