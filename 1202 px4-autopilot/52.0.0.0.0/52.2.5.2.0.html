52.2.5.2. 구동기 재가동 시 관성 모멘트에 의한 돌입 전류(Inrush Current) 억제와 비례-적분-미분(PID) 제어기 적분기 와인드업(Integrator Windup) 초기화

비상 정지(Kill Switch) 해제 명령이 수달되어 비행 제어기(Flight Controller)가 자세 복구를 결심한 시점부터, 실제 물리적인 추력이 발생하기까지는 구동기(Actuator) 역학과 제어 알고리즘의 초기화라는 심각한 진입 장벽이 존재한다. 특히 자유 낙하 중이던 로터를 재가동할 때 발생하는 전기적 부하 현상과, 정지 구간 동안 누적된 제어 오차를 다루는 방식은 기체의 생존을 결정짓는 핵심 파라미터이다. 이 절에서는 PX4-Autopilot의 구동기 돌입 전류(Inrush Current) 억제 메커니즘과 비례-적분-미분(PID) 제어기의 적분항 와인드업(Integrator Windup) 방지 및 초기화 아키텍처를 상세히 다룬다.

1. 관성 모멘트에 의한 돌입 전류(Inrush Current) 역학

공중 비상 정지 상태에서 멀티로터의 프로펠러는 관성 모멘트(Moment of Inertia, J)에 의해 정지해 있거나, 낙하 시 발생하는 하향 기류에 의해 역방향 수동 회전(Windmilling) 상태에 놓이게 된다. 이러한 상태에서 전자 변속기(ESC, Electronic Speed Controller)에 갑작스럽게 높은 역률의 PWM(Pulse Width Modulation) 또는 DShot 제어 신호가 인가될 경우, 모터 코일은 고정자를 강제로 회전시키기 위해 막대한 토크 행렬을 요구하며 이로 인해 전기적 돌입 전류(Inrush Current) 현상이 발생한다.

1.1 돌입 전류의 치명적 결과

배터리에서 모터로 순간적으로 공급되는 전류 I(t)는 모터의 권선 저항 R 과 역기전력(Back EMF, V_b)에 의해 다음과 같이 모델링할 수 있다.
I(t) = \frac{V_{in} - V_b(t)}{R}
정지 상태 혹은 역회전 상태에서는 역기전력 V_b 가 0이거나 반대 부호를 가지므로 \vert I(t) \vert 가 배터리와 ESC의 허용 한계(Peak Current Rating)를 쉽게 초과하게 된다. 이는 하드웨어 OCP(Over Current Protection)를 유발하여 ESC의 영구적인 컷오프(Cut-off)를 초래하거나, 전원 분배 보드(PDB)의 전압 강하(Voltage Sag)를 일으켜 비행 제어기의 브라운아웃(Brownout) 재부팅을 유발할 수 있다.

1.2 PX4의 Spool-up 억제 알고리즘

PX4(v1.14 기준)는 구동기 오버라이드 해제 시 COM_SPOOLUP_TIME 파라미터를 기반으로 선형 보간(Linear Interpolation)된 램프(Ramp) 함수를 ESC에 인가한다. 즉, 목표 자세를 복구하기 위해 높은 출력이 연산되더라도 최하단 제어 믹서(Mixer) 또는 Control Allocation 모듈은 지정된 Spool-up 시간(예: 0.5초) 동안 출력을 서서히 증가시켜 돌입 전류를 부품의 정격 허용치 이내로 억제한다.

반면 ArduPilot의 경우 MOT_SPOOL_TIME을 통해 유사한 기능을 제공하나, PX4는 CBRK_FLIGHTTERM과 결합한 상태에서 비행 종료 로직 해제 후 즉각적인 PWM 스텝 응답을 어느 정도 허용할지 세밀한 내부 가중치를 동적으로 조율하는 mc_att_control 로직이 차별화된다.

2. 적분기 와인드업(Integrator Windup)의 수학적 원리 및 위험성

비례-적분-미분(PID) 프레임워크에서 제어 입력 \mathbf{u}(t) 는 오차 \mathbf{e}(t) 에 기반하여 다음과 같이 연산된다.
\mathbf{u}(t) = K_p \mathbf{e}(t) + K_i \int_{0}^{t} \mathbf{e}(\tau) d\tau + K_d \frac{d\mathbf{e}(t)}{dt}
비상 정지 구간에서는 실제 모터 출력이 0이므로, 기체는 목표 자세 \mathbf{q}_{sp} 를 추종하지 못하고 자유 낙하한다. 따라서 오차 \mathbf{e}(t) 는 해소되지 채 큰 값을 유지하게 된다. 문제는 비행 제어기의 소프트웨어 루프(Loop)가 계속 연산되고 있다면 적분항(\int \mathbf{e}(\tau) d\tau)에 이 막대한 오차가 지속적으로 누적(Windup)된다는 점이다.

비상 정지가 해제되는 순간, 누적된 적분항은 K_i 이득과 결합되어 한계를 초과하는 극한의 포화명령(Saturated Command)을 출력한다. 이 오버슛(Overshoot)은 기체가 한 방향으로 극심하게 뒤집어지게 만들어 치명적인 궤적 이탈을 야기한다.

3. PX4의 능동적 적분기 초기화(Filter Reset) 아키텍처

PX4는 비상 정지가 발동된 상태에서 PID 제어 루프 내부의 상태 변수를 능동적으로 관리하는 Anti-windup 메커니즘을 가동한다.

  1. 상태 감지 기반 동결(Freeze): VehicleStatus의 arming_state가 ARMED이지만 kill_switch 플래그가 참(True)인 구간(공중 정지 구간)에서는 내부적으로 자세 제어기의 적분항(Rate Controller Integral) 연산을 동결(Freeze)한다.
  2. 해제 시 0 초기화(Zero-initialization): 비상 정지 플래그가 해제됨과 동시에 상태 머신은 Rate PID 객체의 reset_integral() 메서드를 호출하여 누적 적분 변수를 강제로 0으로 덮어쓴다.
  3. P-Term과 D-Term 위주의 초기 응답 유도: 초기 상태 복구 구간에서는 오직 비례 제어(Proportional)와 미분 제어(Derivative) 항만이 개입하여 강한 토크로 자세를 원상 복구하며, 오차가 일정 범위 안으로 좁혀진 이후에야 제한적인 적분(Integral) 누적을 허용한다.
