52.5.2.2. 수동 잠금(Manual Lockdown) 플래그 감지 시 제어 할당기의 출력 버퍼 초기화 및 액추에이터 제어 무효화 함수 호출 체인

1. 개요

PX4-Autopilot의 운영체제(RTOS) 커널 상에서 비상 정지(Kill Switch) 명령어 또는 코어의 안전 임계(Safety Critical) 조건이 활성화되면, 비행 제어 스택은 최상위 계층부터 하드웨어 I/O 드라이버 계층까지 즉각적인 연산 중단을 요구한다. 이 중 가장 핵심적인 단계가 제어 할당기(Control Allocator)에서의 수동 잠금(Manual Lockdown) 처리 메커니즘이다.

이 장에서는 커맨더(Commander) 모듈에서 발행한 manual_lockdown 플래그가 제어 할당기에 도달했을 때, 기존의 비행 동역학 연산이 어떻게 무효화되고 최종 액추에이터 출력 버퍼가 어떻게 초기화되는지에 대한 내부 C++ 함수 호출 체인(Function Call Chain)을 상세히 다룬다. (PX4 펌웨어 버전 v1.14 이상 기준)

2. 수동 잠금(Manual Lockdown) 플래그의 의미

actuator_armed uORB 메시지는 기체의 무장 상태(Armed State)를 다루는 가장 중요한 데이터 구조체이다. 이 구조체 내부에 정의된 변수 중 manual_lockdown은 물리적 비상 정지 스위치가 켜졌거나 MAVLink를 통한 비행 종료(Flight Termination) 커맨드가 수신되었을 때 true로 설정된다.
이 속성이 true가 되면 기체는 현재 허공에 있든 지상에 있든 모든 모터 구동을 물리적으로 차단(Disarm)해야 한다.

3. 제어 할당기의 함수 호출 체인 (Function Call Chain)

제어 할당기(ControlAllocator.cpp)는 매 주기마다 uORB 메시지를 업데이트하며 상태를 폴링(Polling)한다. manual_lockdown이 활성화되었을 때의 코어 실행 흐름은 다음과 같은 연속적인 함수 호출로 이루어진다.

  1. ControlAllocator::Run(): 메인 스레드의 런 루프. 토픽 갱신 여부를 확인한다.
  2. _armed_sub.update(&_armed): 커맨더 모듈이 브로드캐스트한 가장 최신의 무장 상태 메타데이터를 복사한다.
  3. if (_armed.manual_lockdown) 분기: 루프 상단에서 즉시 평가된다. 이 블록에 진입하면 기존 믹싱 행렬 연산을 담당하는 내부 메서드 인보크(Invoke)를 건너뛴다.
  4. Allocation::set_disarmed() / 버퍼 초기화 코드: 동적 제어 할당기 라이브러리 내부 배열이나, 브로드캐스트하기 위한 로컬 객체(Output Buffer)의 메모리를 0이나 지정된 Disarmed PWM 값(예: 900μs, DShot 0)으로 일괄 채운다(Memset 또는 반복문 덧쓰기).
  5. _actuator_motors_pub.publish(): 믹싱 연산을 거치지 않고 오직 0 또는 최소 신호 파라미터로만 채워진 결과값을 드라이버 측으로 발행(Publish)한다.

4. Ardupilot의 호출 체인과의 비교

PX4-Autopilot은 독립적인 스레드와 uORB 버스 구조를 가지므로, 커맨더(Commander) 스레드와 제어 할당기 스레드가 비동기적으로 분리(Decoupling)되어 있다. 따라서 위와 같이 퍼블리셔-서브스크라이버 모델로 호출 체인이 성립한다.

반면 Ardupilot은 메인 스케줄러 내에서 순차적 콜백(Sequential Callback)을 보장하는 모놀리식(Monolithic) 호출 스택을 지향한다. Copter::fast_loop() 내에서 모터 라이브러리 motors->output() 함수가 직접 호출되며, 비상 정지 상태인 경우 motors 객체 내부의 armed 플래그 및 interlock 상태에 따라 가장 바깥 단의 클래스 메서드 내에서 즉시 hal.rcout->write() 호출 값을 최소값으로 강제 설정한다.
즉, PX4는 인터프로세스 메시징을 통한 **데이터 지향적 무효화(Data-oriented Invalidation)**를, Ardupilot은 클래스 상속과 동기화된 루프 호출을 통한 **절차 지향적 무효화(Procedural Invalidation)**를 채택하고 있다.

5. 설계 안정성 분석

제어 할당기 내부에서 출력 배열 버퍼를 직접 초기화하고 우회(Bypass)하는 코드는 치명적 오류를 막기 위해 포인터 연산을 최소화하고 안전한 정적 배열 접근(Static Array Access)만을 수행하도록 설계되었다. 이러한 호출 체인은 예기치 못한 메모리 참조 오류(Segmentation Fault)를 배제하여 시스템의 수동 잠금 명령이 어떤 소프트웨어적 악조건 하에서도 100% 하드웨어 드라이버로 전파되도록 보장한다.
