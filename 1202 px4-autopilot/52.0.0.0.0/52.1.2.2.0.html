52.1.2.2 비행 종료(Flight Termination)의 비가역적 영구 잠금 특성과 유한 상태 기계(Finite State Machine) 내 종단 흡수 상태(Absorbing State) 모델링

본 절에서는 유한 상태 기계(Finite State Machine, FSM) 이론을 바탕으로, PX4-Autopilot의 최고 수준 안전 기동(Safety Maneuver)인 비행 종료(Flight Termination) 메커니즘을 분석한다. 이 기능은 가역적인 비상 정지(Kill Switch)와 근본적으로 다른 비가역적 영구 잠금(Irreversible Permanent Lockdown) 특성을 가지며, 제어 이론의 관점에서 종단 흡수 상태(Absorbing State)로 모델링된다.

1. 비가역적 영구 잠금(Irreversible Permanent Lockdown)의 특성

비행 종료(Flight Termination) 모듈은 기체의 회복이 물리적, 공학적으로 불가능하다고 판단되거나 치명적인 결함(예: 로터 축 절단, 심각한 조종면 이탈)이 발생했을 때 작동한다. 비가역적(Irreversible)이라는 의미는 다음과 같다.

  1. 액추에이터 출력 영구 차단: 비상 정지가 단순히 PWM 신호를 하한치로 억제하는 수준이라면, 비행 종료는 모터뿐만 아니라 기체를 구동하는 모든 주 제어 표면의 동작 권한을 OS 수준(NuttX 커널)에서 회수한다.
  2. 재시동(Re-arming) 원천 봉쇄: 비행 종료가 한 번 트리거되면, 관제 시스템(QGroundControl 등)에서 시동 해제(Disarm) 및 재시동 명령을 보내더라도 이를 거부한다. 기체가 완전히 전원 재부팅(Power Cycle)을 거치지 않는 한 시스템은 록다운(Lockdown) 상태를 유지한다.
  3. 능동적 조치(Active Measures): 외부 안전 페이로드, 특히 낙하산(Parachute)을 전개하기 위해 별도의 릴레이나 PWM 채널을 강제로 개방(Open)시킨다.

2. 유한 상태 기계(FSM) 내 종단 흡수 상태(Absorbing State) 모델링

마르코프 연쇄(Markov Chain) 및 유한 상태 기계 이론에서 흡수 상태(Absorbing State)란 한 번 진입하면 외부에서 어떠한 이벤트(외란이나 제어 명령 등)가 발생하더라도 결코 다른 상태로 전이하지 못하고 자체 상태로 루프백(Loopback)하는 상태를 말한다. PX4의 커맨더(Commander) 내부 아키텍처는 비행 종료를 바로 이 흡수 상태로 취급한다.

2.1 상호 절연된 상태 천이도 (Isolated State Transition)

비가역성을 설명하기 위해 다음의 Mermaid 다이어그램을 살펴보자.

stateDiagram-v2
    [*] --> Standby
    Standby --> Armed : Arming Cmd
    
    state Flight_Operations {
        Normal_Flight --> Kill_Switch : Kill Switch ON
        Kill_Switch --> Normal_Flight : Kill Switch OFF
    }
    Armed --> Flight_Operations
    
    Flight_Operations --> Flight_Termination : Critical Failure / Manual Terminate
    Standby --> Flight_Termination : Pre-arm Critical Failure
    
    state Flight_Termination {
        Lockdown_Active
    }
    
    %% 비가역적 루프백(Absorbing Property)
    Flight_Termination --> Flight_Termination : REJECT_ALL_CMD

위 다이어그램에서 알 수 있듯, Flight_Termination 상태로 들어가는 방향의 트랜지션(Transition) 화살표는 존재하지만, 해당 상태에서 다른 상태(Normal_Flight, Standby 등)로 빠져나오는 경로는 존재하지 않는다. 모든 명령은 거부(REJECT)되고 자체 상태로 다시 루프백된다.

3. PX4 소스 코드 수준의 구현 메커니즘 (v1.14.0 기준)

PX4 펌웨어에서 비행 종료 로직은 크게 세 군데에서 처리된다. Commander.cpp 안의 상태 머신 코어, 유도 규칙(Nav State) 머신 코어, 그리고 IO 보드(px4io) 단의 페일세이프 록다운 시스템이다.

MAVLink를 통한 활성화는 MAV_CMD_DO_FLIGHTTERMINATION 메시지의 파라미터 1을 1.0으로 인가함으로써 수행된다. 내부적으로 일어나는 과정은 아래와 같다.

  1. 플래그 영구 소작(Flag Burn-in): vehicle_status_s 구조체 내의 비행 종료 관련 플래그가 세팅되며, 이를 검증하는 루틴(actuator_armed_s.force_failsafe 등)이 동작한다. 이 플래그가 한 번 참(True)으로 설정되면, 코드는 부팅 시까지 플래그를 지우지 않는 하드 코딩된 래치(Latch) 로직을 거친다.
  2. 안전 페이로드 발동 플래그: 낙하산 트리거가 구성된 경우(CBRK_FLIGHTTERM 파라미터 등과 관련), vehicle_command를 통해 낙하산 전개 채널을 최대치로 고정(PWM 2000 부근)시킨다.
  3. 제어 할당기 셧다운: Control Allocator(제어 할당기)는 모터 출력을 0으로 맞출 뿐 아니라, 더 이상 vehicle_torque_setpoint나 vehicle_thrust_setpoint 토픽을 구독(Subscription)하지 않는다. 즉 데이터 파이프라인(Data Pipeline) 계층 자체가 닫힌다.

4. Ardupilot의 Crash Check 및 E-Stop 기능과의 심층 비교

PX4의 엄격한 흡수 상태 로직을 Ardupilot 메커니즘과 비교하면 접근 방식의 차이를 알 수 있다.

  • Ardupilot (Crash Check & E-Stop): Ardupilot은 내부 Crash Detector가 기체의 비정상적인 추락(가속도계 충격 및 자세 반전)을 감지하면 모터를 Disarm한다. 또한 E-Stop(Emergency Stop) 기능은 설정에 따라 물리적인 록다운을 유도할 수 있으나, Ardupilot은 전통적으로 조종사의 수동 제어권 회복 자유도를 넓게 잡는 철학을 갖고 있어, 특정 상황에서는 지상 통제(Mission Planner) 설정을 우회하여 재시동을 강제할 수 있는 백도어가 존재할 여지가 있다.
  • PX4-Autopilot (Flight Termination): 철저히 상용 항공기(Commercial Aviation) 유도 시스템의 표준을 따른다. FSM 상호 모듈의 독립성을 해치지 않기 위해 비행 종료 상태에 진입한 차량 인스턴스는 더 이상 “비행 제어 대상“이 아닌 “사고 후 상태 수습 객체(Post-accident Payload)“로 논리적 지위를 격하(Downgrade)당한다.

5. ROS2 미들웨어에서의 Flight Termination 핸들링

ROS2 환경(uXRCE-DDS)에서 자율 비행 시스템(Autonomous Agent)이 시스템의 파국적 치명 결함을 감지했을 때 프로그램적으로 종단 흡수 상태를 유발할 수 있다.

#include <px4_msgs/msg/vehicle_command.hpp>

// 자율 에이전트 결함 감지 노드 내 비행 종료 트리거 함수
void terminate_flight() {
    px4_msgs::msg::VehicleCommand msg;
    // Parameter 1을 1.0으로 주어 영구적 비행 종료를 명시 (0.0은 Kill Switch)
    msg.param1 = 1.0f; 
    msg.command = px4_msgs::msg::VehicleCommand::VEHICLE_CMD_DO_FLIGHTTERMINATION;
    msg.target_system = 1;
    msg.target_component = 1;
    msg.source_system = 255;
    msg.from_external = true;
    msg.timestamp = node->get_clock()->now().nanoseconds() / 1000;

    vehicle_command_publisher_->publish(msg);
    RCLCPP_FATAL(node->get_logger(), "CRITICAL FAILURE: Initiated irreversible flight termination.");
}

해당 코드가 실행되어 록다운 구동이 완료되면, 이후 노드에서 어떠한 OffboardControlMode 메시지를 퍼블리싱하더라도 PX4 커맨더는 이를 무시하고 MAVLink ACK(NACK 처리)로 응답한다. 자율주행 알고리즘 개발자는 이러한 비가역적 특성을 이해하고 시스템 재시동 프로세스가 단순한 ROS2 노드 리셋이 아닌 PX4 하드웨어 전원 재부팅을 요함을 고려하여 아키텍처를 설계해야 한다.

6. 결론

비행 종료(Flight Termination) 메커니즘의 영구 잠금 특성은 기체의 시스템 상태를 흡수 상태(Absorbing State)로 전이 시킴으로써 예측 불가능한 추가 동작(고장 난 시스템의 임의 기동)을 원천 차단하는 가장 높은 수준의 결함 허용(Fault-Tolerant) 종단 처리 로직이다. 이는 고신뢰성이 요구되는 UAM 및 도심지 다중 군집 드론(Swarm) 환경에서 시스템 파급 효과를 제한하기 위한 필수적인 공학적 기반 계층으로 동작한다.
