52.4.2. 관성 측정 장치(IMU) 기반 추락 인지 및 다중 센서 퓨전 임계값 검출 메커니즘

1. 개요 (Introduction)

무인항공기(UAV) 운영 중 모터 고장, 프로펠러 파손, 혹은 제어 표면의 결함이 발생할 경우 기체는 급격히 통제력을 잃고 추락하게 된다. PX4-Autopilot은 치명적인 사고로 이어지기 전에 이러한 비정상적 비행 동역학을 신속하게 인지하고 비상 복구 절차(낙하산 사출 등)를 가동하기 위해 관성 측정 장치(IMU) 및 다중 센서 퓨전을 활용한 추락 인지 알고리즘(Failure Detection Algorithm)을 탑재하고 있다.

2. 관성 측정 장치(IMU) 기반 추락 인지 (IMU-based Fall Detection)

추락의 1차적인 징후는 가속도계(Accelerometer)와 자이로스코프(Gyroscope)가 측정하는 물리량의 급격한 이상에서 나타난다.

  • 초과 각도 및 각속도 (Attitude and Angular Rate Anomaly): 정상적인 제어 한계를 벗어나는 극단적인 피치(Pitch) 또는 롤(Roll) 역전 현상이 감지되거나, 각속도가 설정 기준치 이상으로 오랜 시간 누적될 경우 추락으로 판단한다.
  • 자유 낙하 (Free-fall): 기체에 작용하는 Z축 가속도가 중력 상쇄를 잃고 0 \text{ m/s}^2에 근접하는 자유 낙하 현상이 일시적으로 발생하는 빈도를 감지한다.

3. 다중 센서 퓨전 임계값 검출 메커니즘 (Multi-Sensor Fusion Threshold Detection Mechanism)

순수 IMU 데이터에만 의존할 경우 격렬한 아크로 모드(Acro Mode) 조작이나 외부 돌풍(Gust)에 의한 일시적인 노이즈를 추락으로 오인할 가능성(False Positive)이 존재한다. 이를 방지하기 위해 PX4는 EKF2(Extended Kalman Filter)의 상태 추정치를 바탕으로 다중 센서 퓨전을 수행한다.

  • 고도 하강 속도 검증: 기압계(Barometer) 및 GPS로 융합된 수직 속도(Vertical Velocity)가 정상적인 하강 임계값을 현저히 초과하는가?
  • 상태 추정치 발산 (Estimator Innovation): EKF2 필터 내부의 혁신(Innovation) 오차가 급증하여 위치 추정이나 자세 추정이 한계를 넘었는가 파악한다. \vert속도 차이\vert, \vert위치 차이\vert 등의 임계값을 복합적으로 계산하여 신뢰도 지표를 낮춘다.

4. PX4 소스 코드 및 아키텍처 분석 (PX4 Source Code and Architecture)

PX4에서 추락 감지를 담당하는 핵심 모듈은 src/modules/commander/failure_detector/FailureDetector.cpp 이다.

추락 판단은 단순히 순간적인 값에 반응하는 것이 아니라, 임계값을 초과하는 상태가 사전에 정의된 기간(예: 300 ms) 동안 연속적으로 유지되는지 평가하는 저역 통과 특성의 시간 지연(Time Hysteresis) 필터링을 거친다.

5. Ardupilot 대비 추락 검출 차이점 (Comparison with Ardupilot)

Ardupilot은 Crash Check 로직이라는 이름으로 추락 감지를 구현하며, 조종자의 조종 입력과 기체의 실제 반응 간의 오차(Control Error)를 중심으로 평가한다. FS_CRASH_CHECK 파라미터가 이를 총괄한다.
반면 PX4-Autopilot의 Failure Detector는 Commander 서브모듈로서 독립적으로 분리되어 있으며, 센서 퓨전 층의 데이터 유효성과 자세 임계값을 조금 더 명시적이고 분산적으로 평가한다. uORB의 vehicle_status 메시지를 통해 시스템 단위로 치명적 오류를 브로드캐스트하는 아키텍처적 차이가 있다.

6. QGroundControl 파라미터 구성 (Configuration in QGC)

QGroundControl (v4.3 이상) 환경에서 추락 감지 감도 및 동작 유무를 세밀하게 제어할 수 있다.

  • FD_FAIL_P: 최대 피치(Pitch) 각도 초과에 대한 추락 인지 임계값 (예: 60도)
  • FD_FAIL_R: 최대 롤(Roll) 각도 초과에 대한 추락 인지 임계값 (예: 60도)
  • CBRK_FLIGHTTERM: 비행 종료 시스템 및 추락 자동 인지 기능을 억제(Circuit Breaker)하는 파라미터. (실비행 시에는 이 값을 해제해야 추락 감지 기능이 켜진다.)

7. 결론 (Conclusion)

PX4-Autopilot의 IMU 기반 추락 인지 및 다중 센서 퓨전 임계값 검출 메커니즘은 단순한 운동학적 한계 초과를 넘어 상태 추정기의 무결성(Integrity) 지표까지 융합하여 결론을 내린다. 이는 돌풍이나 아크로바틱 비행으로 인한 오작동 확률을 최소화하고, 진정한 치명적 고장에만 신뢰성 있게 비상 프로토콜을 트리거할 수 있게 해주는 핵심 자율 방어 기술이다.
