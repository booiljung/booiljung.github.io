52.3.2.1 원격 측정(Telemetry) 통신 심장박동(Heartbeat) 패킷 누락 주기와 링크 대역폭 가용성을 결합한 시상수(Time Constant) 도출

무인항공기의 제어권이 완전히 상실되는 데이터 링크 단절(Data Link Loss)은 페일세이프(Failsafe) 발동의 가장 빈번한 원인이다. PX4-Autopilot(기준 펌웨어 버전: v1.14.0)은 단순한 타이머 기반의 통신 두절 판단을 넘어서, 시스템의 가용 통신 대역폭(Bandwidth) 포화 상태와 MAVLink 심장박동(Heartbeat) 패킷의 통계적 누락 주기를 결합하여 동적인 필터 시상수(Time Constant, \tau)를 도출한다. 본 절에서는 이러한 통신 단절 판단 아키텍처의 수학적 도출 과정과 내부 소스 코드 연산을 다룬다.

1. 개요 및 MAVLink 심장박동(Heartbeat) 통신 특성

지상 관제 시스템(QGroundControl 등)과 PX4 간의 통신은 주로 라디오 텔레메트리(SiK Radio, Microhard 등)를 통해 이루어진다. 정상적인 상태에서 MAVLink의 HEARTBEAT 메시지는 1Hz(초당 1회) 주기로 양방향 교환된다.
그러나 전파 간섭이나 장애물에 의한 신호 감쇠(Attenuation)가 발생하거나, 대용량 임무(Mission) 파일 전송 및 로그 다운로드(ULog)로 인해 제한된 대역폭(일반적으로 57600bps 또는 115200bps)이 포화(Saturation)될 경우, HEARTBEAT 패킷의 누락(Packet Loss)이나 지연이 빈발하게 된다.

PX4의 커맨더(Commander) 모듈은 이러한 일시적인 대역폭 큐(Queue) 포화로 인한 지연을 실제적인 “물리적 신호 유실“과 구분해야 한다. 즉, 일시적으로 대역폭이 100% 가동 중이어서 Heartbeat가 밀린 것인지, 아니면 통신권 밖으로 기체가 이탈한 것인지를 판단하기 위해 동적인 시상수를 연산한다.

2. 링크 대역폭 가용성을 결합한 시상수 연산 모델

데이터 링크 손실을 감지하는 타이머의 마감 시간(Deadline)은 고정값 COM_DL_LOSS_T 파라미터(기본 10초)에서 출발하지만, 내부적으로 필터링된 유효 유예 시간(Effective Grace Period) T_{eff}는 링크 가용성 지표와 합산되어 재계산된다.

2.1 수학적 모델

텔레메트리 채널의 전송 버퍼 점유율(Buffer Occupancy)을 \rho, Heartbeat 패킷 누락률 계수를 \alpha라 할 때, 필터에 적용되는 동적 시상수 \tau는 다음과 같이 도출된다.

변수 / 수식설명
\rho = \frac{\text{Bytes \ in \ Transmit \ Buffer}}{\text{Total \ Buffer \ Size}}현재 MAVLink 전송 버퍼의 포화 상태 (0 \le \rho \le 1)
\alpha_k = \lambda \cdot \alpha_{k-1} + (1 - \lambda) \cdot \text{Loss}_{k}이전 패킷 누락 이력을 포함한 1차 저역통과필터된 누락률 (\lambda는 필터 계수)
\tau = f(\rho) \cdot \tau_{base}포화도 기반 시상수. \rho \to 1일수록 대역폭 포화로 간주하여 \tau를 비례 증가
T_{eff} = COM\_DL\_LOSS\_T + \Delta T(\tau, \alpha)최종적으로 통신 단절을 선언하기 전의 유효 대기 시간

대역폭이 극단적으로 부족한 환경(\rho \approx 1)에서는 Heartbeat 누락이 신호 유실 때문이 아니라 단순 큐 지연(Queue Delay)일 확률이 높기 때문에 시스템은 \tau를 증가시켜 섣부른 RTL(Return To Launch) 전환을 억제한다.

C++ 로직 상에서 mavlink_main.cpp는 송수신 클래스 내부에서 telemetry_status_s 구조체의 필드인 heartbeats 카운터와 rxerrors를 계산한다. 커맨더 모듈 내의 파라미터 옵션에 따라, 텍스트 콘솔 대용량 출력이 활성화된 경우(예: MAV_BROADCAST)에는 명시적으로 데이터 링크 타임아웃 감시 로직에 하드코딩된 여유 계수(Margin)를 곱해주기도 한다.

4. QGroundControl (v4.3 기준) 및 Ardupilot 대비 관제 차이점

  • QGroundControl 인터페이스 연동: QGroundControl 상단의 원격 측정 신호 게이지(Link Quality 게이지)는 MAVLink의 RADIO_STATUS 메시지와 더불어 Heartbeat 수신 주기를 결합하여 표시된다. PX4에서 대역폭 부족으로 인해 Heartbeat가 지연되면 QGC 내부적으로도 수신 타임아웃을 평가하나, 기체의 동적 시상수 연산 덕분에 불필요하게 [COMM_LOST] 음성 경고(TTS)가 남발되는 빈도를 극적으로 낮춘다.
  • Ardupilot 설계 분기 (FS_GCS_ENABL):
    Ardupilot의 경우 FS_GCS_ENABL 파라미터를 통해 고정된 타임아웃(통상 1.5초 ~ 5초) 타이머를 직접 참조하는 정적 접근 방식을 사용한다. 이는 직관적이지만 통신 대역폭이 900MHz 망환 환경 등에서 임시 포화되었을 때 빈번하게 Failsafe 모드로 전환되었다가 복귀하는 ‘탁구(Ping-Pong)’ 거동을 유발할 수 있다. PX4의 동적 시상수 아키텍처는 이러한 엣지 케이스(Edge Case)를 방어하는 데 있어 보다 산업용 무인기에 적합한 필터링 모델이다.

5. 결론

무인항공기가 장거리 또는 비가시권(BVLOS) 비행 임무를 수행할 때, 가장 중요한 것은 오탐지(False Positive) 방지이다. PX4-Autopilot은 원격 측정 통신 끊김의 원인을 단순히 “Heartbeat 수신 불량” 하나로 치부하지 않고, 전송 버퍼의 상태(대역폭 점유율)와 결합된 시상수(Time Constant) 도출 방정식을 도입하였다. 이는 통신 링크의 물리적 열화와 논리적 대역폭 포화를 효과적으로 구분하게 하여, 안정적인 지상 관제 시스템(GCS) 연동과 불필요한 비상 모드 진입을 방지하는 우수한 제어 통제 철학을 반영하고 있다.
