28.2.4.1.1. flag_control_manual_enabled, flag_control_auto_enabled 플래그 파싱 및 검증

vehicle_control_mode.msg를 통해 상부 Commander 데몬으로부터 제어 권한 위임장을 수령한 Flight Mode Manager (이하 FMM)는, 자신에게 하달된 이 플래그(Flag)들이 현재 드론의 처참한 하드웨어 센서 현실과 진짜로 부합하는지 냉정하게 교차 검증(Cross-validation)하는 파싱 단계를 거친다.
특히 가장 근본이 되는 두 가지 권한 플래그인 **수동 입력 활성화(flag_control_manual_enabled)**와 **자동 입력 활성화(flag_control_auto_enabled)**의 검열 로직은 드론의 생존과 직결된다.

1. flag_control_manual_enabled (수동 조종 권한)의 모순 필터링

이 플래그는 조종사가 들고 있는 RC 조종기나 지상 관제소(QGC)의 가상 조이스틱 입력값을 궤적 계산의 피연산자로 사용할 수 있도록 허락하는 ’수동 제어 면허증’이다.

  • Commander의 허가: 조종사가 Stabilized(안정화) 모드나 Position(위치) 모드를 켜면, Commander는 당연히 조종기 스틱을 읽어야 하므로 이 플래그를 true로 세팅해 FMM에게 내려보낸다.
  • FMM의 현실 검증 (Sanity Check):
    FMM은 이 권한을 받자마자, 별도로 자신이 구독하고 있는 manual_control_setpoint (수동 스틱 데이터 토픽)의 유통기한(Timestamp 타임아웃)을 확인한다.
    만약 상부에서는 “조종기 입력을 받아라!“라고 true를 줬는데, 막상 FMM이 조종기 수신 데이터 버퍼를 열어보니 통신이 끊어져 0.5초가 넘도록 새 스틱 데이터가 안 들어오고 있는 상태(RC Loss 상황)라면 어떻게 될까?
  • 권한의 자진 반납(Override):
    FMM은 상부의 지시(true)를 하극상으로 무시해 버리고, 자신의 메모리 안에서만 이 플래그를 치명적인 에러 플래그와 결합하여 **강제로 false로 마스킹(Masking)**해버린다.
    이러한 자체 파싱 검증 덕분에, RC 수신기가 공중에서 타버린 절체절명의 수동 비행 상황에서도 FMM 하위의 위치/자세 제어기들은 조종기 스틱 데이터(0,0,0)를 유효한 입력으로 착각하고 기체를 땅으로 추락시키는 끔찍한 오판을 피할 수 있게 된다.

2. flag_control_auto_enabled (자동 임무 권한)의 아키텍처적 우선순위

이 플래그는 반대로 컴패니언 컴퓨터나 내부 미션 플래너가 찍어주는 웨이포인트(Waypoint) 좌표들을 궤적 계산기로 삼아도 좋다는 ’자동 주행 라이센스’다.

  • 상호 배타적 원칙(Mutually Exclusive):
    일반적인 매뉴얼-오버라이드(Manual-Override) 철학을 따르는 PX4에서는, 하드코딩된 예외(예: 반자율 주행인 Position 모드에서의 스틱 입력)를 제외하고는 원칙적으로 수동 입력 플래그와 자동 입력 플래그가 동시에 true로 순수하게 맞물려 작동하는 것을 극도로 경계한다.
  • Offboard 모드에서의 검증:
    특히 ROS2를 연동하여 컴패니언 컴퓨터가 비전을 통해 조종하는 Offboard 모드의 경우, Commander는 flag_control_auto_enabled를 true로 준다. 이때 FMM은 외부 컴퓨터로부터 2Hz(0.5초) 주기로 안정적인 제어 신호 패킷(Offboard Control Mode)이 도달하고 있는지를 이 플래그 파싱 단계에서 깐깐하게 검열한다. 만약 라즈베리 파이 연결 케이블이 빠져 오프보드 핑(Ping)이 끊어지면, FMM은 즉시 오토 제어 플래그를 자신의 스코프 내에서 false로 찢어버리고, Failsafe 로직으로 튕겨 넘겨 기체가 마지막 속도 벡터를 유지하다가 어디로 날아갈지 모르는(Fly-away) 비극을 차단한다.

3. C++ 비트마스킹(Bitmasking)을 통한 압축적 구조

FMM의 소스 코드를 뜯어보면, 이러한 권한 파싱 및 마스킹 오버라이드 연산이 여러 줄의 지저분한 if-else 중첩문으로 작성되어 있지 않다. 실시간 OS 제어 데몬의 품격에 맞게 깔끔한 비트 연산(Bitwise Operation)이나 논리곱(AND) 오퍼레이터로 우아하게 결합되어 있다.

// FMM 내부의 플래그 파싱 논리곱 검증 로직의 예시적 형태
bool valid_manual_control = _vehicle_control_mode.flag_control_manual_enabled && 
                            !_manual_control_setpoint_is_stale;

bool valid_auto_control = _vehicle_control_mode.flag_control_auto_enabled &&
                          _is_offboard_link_alive;

이렇게 파싱 및 자체 팩트 체크를 통해 한 번 걸러진(Filtered) 순수도 100%의 진짜 제어 권한 불리언(Boolean) 값들만이 최종적으로 자식 객체(FlightTask)들의 켤레 함수 속으로 들어가는 자격 요건을 갖추게 되며, 이것이 조종사의 목숨을 담보하는 비행 펌웨어가 데이터를 대하는 가장 안전한 철학이다.
