28.3.4.1.2. NaN(Not a Number) 처리를 통한 특정 축(Axis) 제어 무시(Ignore) 매커니즘 구현

trajectory_setpoint.msg 구조체가 위치, 속도, 가속도 등 모든 동역학 변수를 품고 있다고 해서, 모든 비행 모드가 이 변수들을 100% 꽉꽉 채워야 하는 것은 아니다.

자율 미션 비행(FlightTaskAuto 트리)은 모든 축의 제어권이 필요하지만, 고도 유지 모드(FlightTaskManualAltitude)에서는 드론의 수직(Z) 고도만 자동 제어될 뿐 수평(X, Y) 움직임은 조종사의 스틱 값(속도 명령)에 전적으로 의존한다. 즉, 수평 X, Y 축에 대한 ‘절대 위치’ 셋포인트는 아예 존재하지 않아야 한다.

이러한 **부분 제어권(Partial Control)**을 하위 제어기에게 우아하게 전달하기 위해, PX4는 IEEE 754 부동소수점 표준의 특수 값인 **NaN (Not a Number)**을 적극적으로 착취(?)하는 영리한 아키텍처를 구현했다.

1. Boolean 플래그의 낭비 제거

만약 NaN을 쓰지 않았다면, 셋포인트 메시지 구조체 내부는 다음과 같이 끔찍해졌을 것이다.

// 비효율적인 구형 아키텍처 예시 (PX4는 이렇게 하지 않음)
float position[3];
bool ignore_pos_x;
bool ignore_pos_y;
bool ignore_pos_z;
float velocity[3];
bool ignore_vel_x... // 끔찍한 불리언 오버헤드의 향연

PX4 설계자들은 수십 개의 무의미한 boolean 무시 플래그(Ignore Flag)들을 메시지에 얹어 통신 대역폭을 낭비하는 대신, “유효하지 않은 축의 float 변수 자체에 NaN을 밀어 넣자“는 깔끔한 합의를 도출해 냈다.

2. 기저 클래스의 Wipe-out과 선택적 오버라이트

이 매커니즘은 FlightTask 최상단 기저 클래스에서부터 체계적으로 작동한다.

  • 초기화 (Initialization): 기저 클래스는 루프가 시작될 때마다 _trajectory_setpoint 버퍼의 모든 실숫값 필드에 C++의 NAN 매크로 값을 부어버려 완전한 무(無)의 상태로 되돌린다.
  • 선택적 기입 (Selective Overwrite): 자식 태스크 로직이 실행되면서, 자신이 책임을 질 축(Axis)의 데이터 포인터에만 유의미한 float 실수 숫자를 덮어쓴다. 예를 들어 FlightTaskManualAltitude는 position[2](Z축 고도)와 velocity[0], velocity[1](수평 속도)에만 숫자를 기입하고 나머지는 NAN 그대로 방치한다.

3. 제어기(Controller)의 PX4_ISFINITE 필터링

이렇게 구멍이 송송 뚫린 채로 패킹된 메시지가 mc_pos_control 같은 하위 제어기에 도착하면, 제어기 내부의 수학 필터들은 숫자를 다루기 전에 반드시 PX4_ISFINITE() 매크로 함수를 거친다.

// 하위 제어기 (mc_pos_control) 내부의 수신 처리 로직 예시
if (PX4_ISFINITE(setpoint.position[0])) {
    // X축 위치 제어 PID 루프 가동
    _vel_sp(0) = _pos_ctrl.calc_velocity_setpoint(setpoint.position[0]);
} else if (PX4_ISFINITE(setpoint.velocity[0])) {
    // X축 위치 제어가 NAN이므로 무시하고, 즉시 X축 속도 제어로 강등(Bypass)
    _vel_sp(0) = setpoint.velocity[0];
} else {
    // 둘 다 NAN이면 해당 축 모터 출력을 0으로 셧다운
    _vel_sp(0) = 0.0f; 
}

이처럼 NaN을 논리적 트리거(Trigger)로 사용하는 이 영리한 트릭 덕분에, FMM은 어떠한 복잡한 이종(Hybrid) 비행 모드—예를 들어 X축은 자율 비행, Y축은 수동 비행, Z축은 레이더 고도 유지 같은 괴랄한 조합—라도 하위 제어기 로직을 한 줄도 뜯어고치지 않은 채 단일 trajectory_setpoint 메시지 하나로 완벽하게 통제할 수 있게 된 것이다.
