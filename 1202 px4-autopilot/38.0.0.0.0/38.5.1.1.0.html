38.5.1.1 상향링크(Uplink) 제어 지연 곡선과 하향링크(Downlink) 관측 지연 곡선 교차점에서의 상태 불일치 벡터 공간 분석

38.5.1절에서 정성적으로 다룬 2차원 시공간(Spacetime) 다이어그램의 경쟁 상태를, 보다 엄밀하게 제어 공학의 상태 공간(State Space) 모델로 수식화해 보자. 이 기하학적 모델링은 MAVSDK나 현대적인 UI 프레임워크가 어떤 수학적 근거로 낡은 텔레메트리 데이터를 고의적으로 무시(Masking)하는지 완벽하게 증명해 준다.

1. 상태 벡터(State Vector)의 정의

먼저, 시스템의 핵심을 구성하는 노드들의 상태를 기하학적 벡터로 정의한다. 편의상 비행 모드(M)와 시동 상태(A)를 원소로 갖는 2차원 열 벡터로 한정한다.

  • 시점 t에서 기체 내부에 존재하는 절대적 진성 상태(True State):
    \vec{S}_{uav}(t) = \begin{bmatrix} M_{uav}(t) \\ A_{uav}(t) \end{bmatrix}
  • 시점 t에서 GCS 메모리에 캐싱되어 있는 섀도우 상태(Shadow State):
    \vec{S}_{gcs}(t) = \begin{bmatrix} M_{gcs}(t) \\ A_{gcs}(t) \end{bmatrix}

2. 관측 지연과 제어 지연 곡선의 만남

분산 네트워크 환경에서 두 벡터는 결코 동일한 시점 t에 완벽히 일치할 수 없다. 오직 하향링크(\Delta t_{down})와 상향링크(\Delta t_{up})의 전파 지연상수만큼 서로 어긋난 위상(Phase)을 가진 채 쫓고 쫓기는 관계를 맺는다.

  1. 하향링크 관측 지연 곡선 (Downlink Observation Delay):
    아무런 조작을 가하지 않은 평시 상태에서, GCS가 현재 화면에 띄우고 있는 섀도우 상태 벡터는 사실 과거의 진성 상태가 복사된 것이다.
    \vec{S}_{gcs}(t) = \vec{S}_{uav}(t - \Delta t_{down})

  2. 상향링크 제어 지연 곡선 (Uplink Control Delay):
    시점 t_0에 운용자가 GCS를 통해 상태를 강제로 바꾸려는 제어 명령 벡터 \vec{C}를 발사하면, 이 명령은 \Delta t_{up} 만큼의 공간을 비행한 뒤 기체의 진성 상태를 전이(Transition)시킨다.
    \vec{S}_{uav}(t_0 + \Delta t_{up}) = f(\vec{S}_{uav}(t_0 + \Delta t_{up}), \vec{C})

3. 맹목 구간(Blind Transition Window) 내의 오차 벡터 공간

문제의 핵심은 운용자가 버튼을 누른 t_0 시점부터, 명령이 기체에 닿고 기체가 이를 승인하여 다시 GCS로 COMMAND_ACK 패킷이 도달하는 시점, 즉 왕복 지연 시간(\Delta t_{up} + \Delta t_{down}) 동안 벌어지는 일이다.

만약 두 노드 간의 상태 불일치 벡터(Mismatch Vector)를 \vec{E}(t) = \vec{S}_{gcs}(t) - \vec{S}_{uav}(t) 로 정의할 때, 이 왕복 지연 시간, 곧 맹목 구간(Blind Transition Window) 내부에서는 기하학적으로 무조건 **오차의 노름(Norm)이 0 초과(|\vec{E}(t)| > 0)**인 불일치 공간이 형성됨을 피할 길이 없다.

4. PX4 아키텍처의 상태 고정 필터(Lock-out Filter)

따라서 PX4를 비롯한 현대적인 GCS-UAV 통신 아키텍처는 이 불가피한 불일치 공간 안에서 UI가 미친 듯이 진동(Ping-Pong)하는 것을 막기 위해, 상위 계층 애플리케이션에 일종의 **방향성 필터(Directional Filter)**를 구축한다.

명령 \vec{C}가 발사되는 시점 t_0에 GCS 내부 로직은 is_transitioning 이라는 논리 락(Lock)을 건다. 이 락이 걸려있는 동안 GCS는 \vec{S}_{gcs}(t) = \vec{S}_{uav}(t - \Delta t_{down}) 이라는 하향링크 관측 공식을 완전히 무시하고, 화면의 UI를 강제로 자신이 목표로 한 제어 벡터 \vec{C} 쪽에 붙잡아 둔다.

시간이 흘러 비행 제어기가 실제로 궤도를 틀고 COMMAND_ACK를 보내어 맹목 구간이 종료(t = t_0 + \Delta t_{up} + \Delta t_{down})되면 그제야 논리 락을 해제한다.
이러한 기하학적인 지연 곡선 교차에 대한 근본적 고찰 없이는, 수백 km 밖의 비가시권 통신망에서 화면의 게이지와 모드 버튼들이 오작동하는 섬뜩한 ’유령 현상’을 코드 레벨에서 결단코 잠재울 수 없다.
