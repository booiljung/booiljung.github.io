38.4.3 패킷 손실률(Packet Loss Rate)의 추계학적(Stochastic) 정량화 모델

38.4.2절의 적응형 타임아웃 알고리즘은 무선 링크가 “완전히 숨통이 끊어져 묵음이 된 상태“를 영리하게 감지해 내지만, 텔레메트리 트래픽 분석에 있어서 치명적인 맹점이 하나 존재한다.

만약 채널 용량 붕괴나 간섭으로 인해 GCS에서 보낸 데이터의 90%가 허공에 증발하고 있는데도, 모뎀이 필사적으로 10번에 1번꼴로 HEARTBEAT 패킷만은 어떻게든 GCS에 밀어 넣는 데 성공하고 있다면 어떨까?
이 경우 GCS의 워치독 타이머는 절대 임계치를 넘지 않으므로 연결이 살아있다고 착각한다. 하지만 실질적으로는 기체 제어를 위한 웨이포인트 업로드나 위치 정보(GLOBAL_POSITION_INT)는 거의 전달되지 않는 **‘좀비(Zombie) 구간’**에 빠지게 된다.

이러한 숨통만 겨우 붙어있는 좀비 상태를 타개하기 위해, 제어 시스템은 시간(Latency) 차원을 넘어 데이터의 밀도(Density) 차원을 검증하는 **패킷 손실률(Packet Loss Rate, PLR)**의 추계학적 평가 모델을 가동해야 한다.

1. MAVLink 시퀀스(Sequence) 번호 기반의 손실 평가 메커니즘

MAVLink 프로토콜의 표준 텔레메트리 스트림은 ACK/NACK 확인 응답을 받지 않는 Fire-and-Forget 방식(UDP와 유사)을 기본으로 한다. 따라서 송신 측(UAV)은 자기가 보낸 데이터 중 몇 퍼센트가 GCS에 도착했는지 알 길이 없다. 오직 데이터를 받아먹는 수신 측(GCS)만이 누락된 조각을 세어볼 수 있다.

이를 가능하게 하는 것이 모든 MAVLink v1 및 v2 헤더에 1바이트(8비트) 크기로 박혀 있는 **시퀀스 번호(seq)**이다. 송신 측은 MAVLink 패킷을 하나 만들어 보낼 때마다 이 값을 0에서 255까지 1씩 순차적으로 증가(Increment)시킨다.

수신 측은 직전 패킷의 시퀀스 번호 S_{n-1}과 지금 막 도착한 패킷의 시퀀스 번호 S_n을 비교하여, 그 사이에 허공으로 사라진 패킷의 연속된 개수(Gap) \Delta_{loss}를 수학적으로 계산해 낸다. 이때 8비트 정수 오버플로우(255에서 0으로 랩어라운드되는 현상)를 고려한 모듈로 연산이 필요하다.

\Delta_{loss} = (S_n - S_{n-1} - 1) \pmod{256}

  • 만약 패킷이 완벽히 순서대로 도착했다면: S_n = S_{n-1} + 1 이므로, \Delta_{loss} = 0 이 된다.
  • 만약 시퀀스 41번 다음 패킷이 45번이라면: \Delta_{loss} = (45 - 41 - 1) \pmod{256} = 3 개의 패킷이 소실되었음을 즉각 인지한다.

추계학적 손실률(PLR) 산출 및 방어 로직 연계

이러한 불연속성을 기반으로, 시스템은 특정 시간 윈도우(Time Window, 예: 최근 5초 또는 최근 100개 패킷 구간) 동안의 통계적 패킷 손실률(PLR)을 산출한다.

\text{PLR} = \frac{\sum \Delta_{loss}}{\sum \Delta_{loss} + \text{Packets\_Received}} \times 100 (\%)

양대 펌웨어는 이 PLR 지표를 실전에서 다르게 응용한다.

  • Ardupilot: 이 PLR 수치를 텔레메트리 라디오의 RADIO_STATUS 오류율 지표와 결합하여 GCS 화면(Mission Planner 등)에 직접적으로 띄워 운용자에게 강력한 경고를 제공한다. 또한, 파라미터 설정을 통해 평균 PLR이 특정 임계치(예: 80%)를 넘으면 즉각 RTL과 같은 페일세이프를 물리적으로 발동시키는 맹렬한(Aggressive) 컷오프 기준으로 삼을 수 있다.
  • PX4: 이 손실률을 단순히 기계를 죽이는 비상 스위치로만 쓰지 않는다. PX4 생태계는 PLR의 악화를 ’통신망 건강 악화 계수’로 받아들여, 통신 채널에 보내는 백그라운드 텔레메트리 스트림(센서 디버그 로그, 고속 자세 파라미터 등)의 발행 주기를 능동적으로 스스로 낮추어 통신망의 체증을 풀어주는 율속(Rate Limiting/Traffic Shaping) 알고리즘의 척도로서 추계학적 활용을 꾀한다.
