38.2.4 시스템 건전성(System Health) 비트맵(Bitmap) 인코딩 분석

비행 제어기(PX4-Autopilot)는 수십 개의 센서와 액추에이터, 보조 마이크로컨트롤러가 얽혀 있는 복잡한 생태계다. GCS(QGroundControl)가 기체의 전반적인 “건강 상태“를 판단하기 위해서는 HEARTBEAT 메시지에 포함된 거시적 상태 정보뿐만 아니라, 하드웨어 컴포넌트 레벨의 미시적 진단 정보가 필수적이다.

MAVLink 프로토콜은 기체의 거시적 생명 주기(Lifecycle)를 HEARTBEAT 내의 system_status 필드로 보고하고, 각 컴포넌트의 상세한 결함 여부를 SYS_STATUS (메시지 ID 1) 패킷 내의 3가지 거대한 32비트 비트맵(Bitmap) 변수를 통해 인코딩하여 전송한다.

1. HEARTBEAT의 거시적 생명 주기: system_status (8-bit)

HEARTBEAT 페이로드의 system_status 필드는 기체 전체의 상태를 단일 열거형(MAV_STATE) 정수 값으로 정의한다. 이는 비트맵이 아닌 스칼라 값이며, 주요 상태는 다음과 같다.

  • MAV_STATE_UNINIT (0): 부팅 초기, 시스템 미초기화 상태.
  • MAV_STATE_BOOT (1): 운영체제(NuttX) 및 드라이버 로딩 중.
  • MAV_STATE_CALIBRATING (2): 센서 캘리브레이션 수행 중 (가속도계, 나침반 등).
  • MAV_STATE_STANDBY (3): 시동(Arm) 대기 중. 지상에 안전하게 착륙한 비무장 상태.
  • MAV_STATE_ACTIVE (4): 모터가 활성화된 비행(또는 구동) 상태.
  • MAV_STATE_CRITICAL (5): 중요 컴포넌트 결함 발생. (RTL 비상 복구 트리거)
  • MAV_STATE_EMERGENCY (6): 통제 불능. 비상 엔진 정지(Kill Switch) 임박 상태.
  • MAV_STATE_POWEROFF (7): 전원 차단 진행.

이 플랫(Flat)한 8비트 정수 하나만으로도 GCS는 현재 비행기가 안전한지, 비상사태에 돌입했는지를 색상 코드(초록/노랑/빨강)로 단순화하여 사용자에게 즉각 보고할 수 있다.

2. SYS_STATUS의 미시적 하드웨어 비트맵 구조체

기체가 MAV_STATE_CRITICAL에 빠졌을 때, 정확히 “어떤 부품“이 고장 났는지를 알려면 SYS_STATUS 패킷의 비트맵을 해독해야 한다. MAVLink 설계자는 32비트 정수 3개를 나란히 배치하여 MAV_SYS_STATUS_SENSOR라는 비트 마스크 규칙을 적용했다.

  1. onboard_control_sensors_present: 하드웨어(I2C/SPI 버스)적으로 인식된 센서 목록
  2. onboard_control_sensors_enabled: 비행 제어 스펙상 활성화된(사용 중인) 센서 목록
  3. onboard_control_sensors_health: 현재 에러 없이 정상적으로 데이터를 퍼블리시 중인 센서 목록

각 비트(Bit) 슬롯은 고유한 하드웨어를 지칭한다.

  • Bit 0 (1): 3축 자이로스코프(Gyroscope)
  • Bit 1 (2): 3축 가속도계(Accelerometer)
  • Bit 2 (4): 3축 지자기 센서(Magnetometer)
  • Bit 3 (8): 절대압 기압계(Absolute Pressure)
  • Bit 5 (32): 글로벌 포지셔닝 시스템(GPS)
  • Bit 10 (1024): 레이저 기반 거리 측정기(LiDAR/Rangefinder)

2.1 비트 반전(~ 기호) 및 논리곱/합을 이용한 상태 토글(Toggle) 로직

PX4 내부의 commander 시스템이 GPS 연결 유실을 감지했다고 가정해 보자. 기존의 health 비트맵 변수에서 GPS에 해당하는 5번 비트(32)만 정밀하게 0으로 끄려면(Clear), **비트 반전(Bitwise NOT, ~)**과 논리곱 연산을 결합해야 한다.

// GPS의 건강 상태 비트 끄기 (결함 발생)
health_bitmap &= ~(MAV_SYS_STATUS_SENSOR_GPS); 
// 만약 다른 센서가 정상이고 GPS만 고장 났다면 5번 비트만 0으로 소거됨

반대로 재부팅 중 연결된 LiDAR 센서가 파라미터 활성화 승인을 받았다면, enabled 변수에 논리합으로 비트를 1로 켜준다(Set).

// LiDAR 센서 활성화 비트 켜기
enabled_bitmap |= MAV_SYS_STATUS_SENSOR_LASER_POSITION;

2.2 GCS의 결함 감지 불-대수(Boolean Algebra) 연산 판단

이제 GCS 애플리케이션은 드론으로부터 이 비트맵 3개를 수신하여 불 대수 연산을 통해 논리적 모순점을 찾는다.

만약 어떤 컴포넌트 X가 활성화되어야 하는 상태(enabled & X)인데, 정작 건강 상태 비트맵에서는 꺼져 있다면(!(health & X)), 이는 명백한 하드웨어 장애(Hardware Failure)를 의미한다.

bool is_gps_failed = (enabled_bitmap & MAV_SYS_STATUS_SENSOR_GPS) && 
                    !(health_bitmap  & MAV_SYS_STATUS_SENSOR_GPS);

if(is_gps_failed) {
    ui->showAlarm("GPS Signal Lost!");
}

이와 같은 32비트의 폭넓은 비트맵 체계와 불-대수 파서(Parser)의 결합은, PX4나 Ardupilot이 I2C 배선의 물리적 단선부터 칼만 필터(EKF)의 수학적 발산 현상에 이르기까지 모든 컴포넌트의 결함 사유를 O(1)의 상수 시간 안에 GCS로 직렬화(Serialize)하여 보고할 수 있게 하는 통신 아키텍처의 위대한 기반이 된다.
