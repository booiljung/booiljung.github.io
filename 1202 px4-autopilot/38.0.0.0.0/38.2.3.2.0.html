38.2.3.2 상위 비트(MSB) 및 하위 비트(LSB) 시프트(Shift) 연산을 활용한 메인/서브 모드 디코딩(Decoding) 알고리즘

앞선 38.2.3.1절에서 우리는 비트 마스킹(&)을 통해 custom_mode 데이터 뭉치로부터 특정 8비트 구역을 안전하게 격리(Isolation)하는 데 성공했다.

그러나 이 격리된 값(예: main_mode를 나타내는 0x00030000)은 그 자체로 의미를 갖지 못한다. 수치적으로 이 값은 10진수로 196,608이라는 거대한 스칼라(Scalar) 값이며, C++의 switch-case 구문에서 case 3:을 찾아 매핑해야 하는 로직 입장에서는 완전히 쓸모없는 쓰레기 값에 불과하다.

따라서 격리된 비트 열을 오른쪽 끝인 최하위 비트(Least Significant Bit, LSB) 위치까지 쭉 밀어 내려 정상적인 0 \sim 255 범위의 uint8_t 값으로 복원(Downscaling)해야 한다. 이 수학적 이동을 C/C++에서는 우측 시프트(Right Shift, >>) 연산자로 구현한다.

1. 우측 시프트(>>): GCS에서의 디코딩(Decoding) 로직

QGroundControl(GCS)이 PX4의 HEARTBEAT 패킷을 해석하는 백엔드(Vehicle.cc) 소스 코드를 해부해 보면 다음과 같은 전형적인 디코딩 파이프라인이 나타난다.

1.1 Main Mode 디코딩 수식

main_mode는 원본 32비트 정수에서 16번 비트 위치에 안착해 있다. 따라서 마스킹이 끝난 값을 오른쪽으로 16칸(16비트) 밀어버려야 한다.

// 1. 마스킹: (custom_mode & 0x00FF0000)
// 2. 시프트: >> 16
uint8_t main_mode = (custom_mode & 0x00FF0000) >> 16;

만약 원본 값이 0x04030000이었다면, 마스킹 결과는 0x00030000이 되고, 이를 16칸 우측으로 시프트하면 놀랍게도 정확히 0x00000003 (10진수 3)이라는 순수한 메인 모드 식별자가 추출된다.

1.2 Sub Mode 디코딩 수식

sub_mode는 24번 비트라는 더 높은 위치(MSB 부근)에 자리 잡고 있다.

uint8_t sub_mode = (custom_mode & 0xFF000000) >> 24;

원본 값 0x04030000에 0xFF000000 마스크를 씌우면 0x04000000이 남고, 이를 다시 우측으로 24칸 시프트(>> 24)하면 완벽한 0x00000004 (10진수 4)가 산출된다.

2. 좌측 시프트(<<): PX4 Commander 모듈의 인코딩(Encoding) 로직

역으로, 비행 제어기(PX4) 내부의 commander 모듈이 현재 기체의 비행 모드 구조체를 MAVLink 망으로 쏘아 보낼 때는 반대 방향의 수학적 거푸집(Template) 조립 작업이 필요하다.

main_mode 값 3과 sub_mode 값 4를 하나의 uint32_t 상자에 구겨 넣기 위해 좌측 시프트(Left Shift, <<) 연산을 수행한다. 3은 좌측으로 16칸 올려 보내고, 4는 좌측으로 24칸 올려 보낸 뒤 논리합(|)으로 묶는다.

// PX4 Commander 소스 코드 압축 원리
uint32_t custom_mode = 0;
custom_mode |= ((uint32_t)main_mode << 16);
custom_mode |= ((uint32_t)sub_mode << 24);
// 결과적으로 custom_mode 변수에는 0x04030000 이 들어간다.

2.1 Ardupilot과의 관제 철학(GCS) 분기점 판단

이렇듯 PX4는 custom_mode 하나에 4바이트를 쪼개어 쓰는 지독하게 정교한 다차원 구조를 취하고 있다. 반면 Ardupilot은 이 32비트 변수 전체를 단일 플랫 Enum(예: 15 == GUIDED)으로 간주하여 시프트 연산을 낭비하지 않는다.

따라서 QGroundControl 같은 통합 지상 관제 시스템은 HEARTBEAT가 수신되었을 때 가장 먼저 오토파일럿의 태생 정보(autopilot 필드, 예: MAV_AUTOPILOT_PX4 혹은 MAV_AUTOPILOT_ARDUPILOTMEGA)를 확인한다.
그리고 펌웨어가 PX4로 확인된 경우에만 위에서 설명한 ((custom & mask) >> shift) 계열의 디코딩 파이프라인으로 트래픽을 분기(Branching)시키고, Ardupilot일 경우엔 파싱 없이 곧바로 스위치-케이스(Switch-case) 뷰어로 넘긴다. 이는 GCS가 이기종 시스템을 단일 인터페이스로 수용하는 가장 결정적인 다형성(Polymorphism)의 기술적 현장이다.
