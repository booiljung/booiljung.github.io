38.1.1.1 비행 상태 전이 행렬(State Transition Matrix)의 대수학적 구조와 고유 공간(Eigenspace) 제약

비행 제어 시스템의 이산 사건 시스템(Discrete Event System)은 연속 시스템의 상태 공간 방정식(State-Space Equation, \dot{x} = Ax + Bu)과 유사하게 모델링될 수 있다. 연속 시스템에서 시스템 행렬 A가 동역학적 특성을 결정짓는다면, 이산 논리 도메인에서는 ’비행 상태 전이 행렬(State Transition Matrix)’이 기체의 행동 가능 범위를 규정한다. PX4-Autopilot의 FSM 체계는 이러한 행렬을 수학적으로 매우 희소하게(Sparse) 유지함으로써 예측 불가능성을 차단하는, 제어 공학의 고유 공간(Eigenspace) 제약과 매우 닮은 아키텍처를 취한다.

1. 상태 전이 행렬의 논리 대수학적(Logical Algebraic) 표현

PX4의 commander에서 다루는 비행 상태 전이는 본질적으로 불리언 행렬(Boolean Matrix)로 표현될 수 있다. 총 N개의 유효한 기체 상태(주 상태, 무장 상태, 오류 비상 상태의 벡터 표기) 집합을 S = \{s_1, s_2, \dots, s_N\}이라 할 때, 상태 전이 행렬 T는 N \times N 차원의 논리 행렬이 된다.

행렬의 각 원소 T_{ij}는 현재 상태 s_i에서 상태 s_j로의 전이를 촉발시키는 전제 조건 및 외부 이벤트의 유효성 검증 함수(Validation Function)로 정의된다.
T_{ij} = f_{validation}(s_i, \text{event}) \in \{0, 1\}

코드 레벨(PX4-Autopilot/src/modules/commander/state_machine_helper.cpp 내부)에서 이 행렬은 명시적인 2차원 배열이 아닌, 스위치-케이스(Switch-case) 및 다단계 조건문을 통해 논리적으로 구축된다.

  • T_{ij} = 1: 모든 필수 조건식(Pre-flight Check, 현재 배터리 상태, 고도 제약 등)이 정상(True)일 때만 성립되는 합법적 전이(Legal Transition).
  • T_{ij} = 0: 그 외의 모든 경우 (예외 없음).

불변 부분 공간(Invariant Subspace)과 고유 공간(Eigenspace) 제약의 유추

선형대수학에서 고유 공간(Eigenspace)은 시스템에 어떠한 선형 변환을 가했을 때 그 방향이 변하지 않고 특정 부분 공간 내에 귀속되는 벡터들의 집합을 의미한다.

PX4-Autopilot의 전이 모델에 이 개념을 은유적으로 적용하면, “허용된 비행 상태의 집합” 자체가 일종의 변치 않는 허용된 공간(Admissible Eigenspace)으로 작용한다. 외부에서 GCS를 통해 어떠한 악의적이거나 잘못된 상태 변경 이벤트(변환 연산자, Operator)가 시스템으로 주입되더라도, PX4의 엄격한 상태 전이 행렬은 허용되지 않은 투영(Projection)을 무효화(T_{ij} = 0)한다.

즉, 기체가 ’오류 없이 물리적으로 제어 가능한 상태’라는 고유 공간을 이탈하는 것을 대수학적 제약으로 원천 봉쇄하는 것이다. GCS가 비행 중인 기체에게 ARMING_STATE_STANDBY(비행 중 시동 강제 해제) 명령을 전달하더라도, IN_AIR 환경 변수가 참(True)인 상황에서 전이 행렬의 해당 원소값은 0이므로 기체는 명령을 수용하지 않고 원래의 궤도 공간을 사수한다.

행렬의 희소성(Sparsity)과 시스템의 안정성

PX4의 상태 전이 행렬 T는 지극히 희소 행렬(Sparse Matrix)이다. 주 대각선(Self-transition)과 인접한 논리적 계층(예: STANDBY \rightarrow ARMED, ARMED \rightarrow IN_AIR_TAKEOFF) 등 극소수의 원소만이 전이 가능(1) 영역으로 설정되어 있다.

Ardupilot과의 철학적 대조: 밀집된 행렬 구조

Ardupilot의 경우, 조종사의 권한을 최대한 연장하기 위해 상태 전이 행렬이 PX4 대비 상대적으로 조밀한(Dense) 성향을 갖는다. 예컨대 GPS가 물리적 신호를 잃어 일시적인 불량 상태가 되었을 때, PX4는 전이 행렬 규칙에 따라 즉각적이고 확정적으로 데드레코닝 기반의 Failsafe 동작 공간으로 전이시키는 반면, Ardupilot의 일부 구현은 내부 EKF 추정 오차가 임계치에 도달할 때까지 지연시키거나, 조종사가 수동 비행으로 강제 복귀시키는 경로(추가적인 T_{ij} = 1)를 더 많이 열어둔다.

이러한 행렬의 구조적 치밀함(Sparsity vs Density)은 곧 시스템의 ‘기계적 증명 가능성(Mechanical Provability)’ 여부를 결정짓는다. QGroundControl은 PX4의 이러한 강력한 내부 수리적 제약 모델을 신뢰하기 때문에, GCS 차원에서도 PX4의 내부 상태 행렬과 동일한 제약 조건을 UI(User Interface) 레이어에 디지털 트윈 형태로 매핑하여, 사용자가 애초에 불가능한 명령 버튼을 누를 수 없도록 상호 배제(Mutual Exclusion) 로직을 연동하게 된다.
