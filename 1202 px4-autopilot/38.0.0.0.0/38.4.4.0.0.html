38.4.4 다중 통신 링크(Multi-link) 혼합망의 트래픽 라우팅 로직

지금까지 우리는 단일 RF 링크 내부에서 발생하는 지연과 손실을 어떻게 수학적으로 정량화할 것인가에 집중했다. 그러나 현대의 기업용 BVLOS(비가시권) 드론이나 관제 아키텍처는 결코 단일 장애점(Single Point of Failure)에 기체의 운명을 맡기지 않는다.

실제 상용 기체들은 대개 이종(Heterogeneous) 다중 통신망을 탑재한다.

  1. 단거리 고대역폭: 2.4GHz / 5.8GHz Wi-Fi (이륙장 근거리용, 영상 전송용)
  2. 중거리 저대역폭: 900MHz / 433MHz SiK 텔레메트리 (수 km 반경의 든든한 1차 제어망)
  3. 글로벌 비가시권: LTE / 5G 셀룰러 네트워크 (또는 이리듐 위성 통신망)

이렇게 2~3개의 통신 파이프라인이 동시에 기체와 GCS를 연결하고 있을 때, MAVLink 통신 프로토콜은 단순한 직렬(Serial) 디스패처를 넘어선 복잡한 **라우터(Router) 및 멀티플렉서(Multiplexer)**의 역할을 수행해야 한다.

1. 맹목적 브로드캐스팅(Broadcasting)의 재앙

만약 다중 링크 환경에서 프로토콜 설계자가 가장 단순한 해결책—“기체가 생성하는 모든 MAVLink 패킷을 3개의 포트(Wi-Fi, 900MHz, LTE)에 동시에 복사해서 쏜다”—을 선택한다면 어떻게 될까?

생존성은 극대화될지 모르나, 시스템은 즉시 마비된다. 고해상도 비디오 스트림이나 고속 센서 로그가 초당 100Kbytes씩 협대역 900MHz 모뎀의 버퍼로 쑤셔 박혀 앞서 38.4.3.1절에서 설명한 대규모 ‘혼잡 윈도우 붕괴’ 현상을 일으킬 것이다. 또한, 비싼 셀룰러 종량제 데이터 캡(Data Cap)을 쓸데없는 중복 텔레메트리로 낭비하게 된다.

2. MAVLink 라우팅 시스템과 동적 절체(Dynamic Failover)

이러한 병목과 낭비를 막기 위해 PX4의 MAVLink 모듈(mavlink_receiver.cpp, mavlink_main.cpp)과 QGroundControl의 통신 아키텍처는 스마트한 **링크별 트래픽 라우팅(Traffic Routing)**과 동적 절체(Dynamic Failover) 기능을 수행한다.

  1. 링크별 특성 할당 (Link Profile Allocation):
    특정 MAVLink 인스턴스는 자신이 연결된 하드웨어 포트의 특성을 파악한다. Wi-Fi 연동 포트는 MAV_1_RATE를 고속(High-rate) 스트리밍 권한으로 열어두지만, 텔레메트리 포트는 저속(Low-rate) 모드로 고정하여 대용량 패킷 큐잉 자체를 거부한다.

  2. 상태 기반 주/부 전환 (Primary/Secondary Switch):
    기체가 이륙 후 멀어지면서 1차망인 Wi-Fi 링크의 확률론적 건전성(앞장 참고) 점수가 급락하고 가드 타임을 초과하여 타임아웃이 발생하면, QGC와 PX4는 즉각 ‘주 제어망(Primary Control Link)’ 타이틀을 2차망인 900MHz RF나 3차망인 LTE로 지연 없이 스위칭(Failover)해야 한다.

이 스위칭 결정이 우물쭈물 지연되면 멀티 링크를 달아놓고도 데드락(Deadlock)에 빠져 기체가 추락하게 된다. 따라서 시스템은 소프트웨어적인 패킷 수신 간격(Timeout)에만 의존하지 않고, 물리 계층 하드웨어 안테나가 직접 보고하는 ’수신 신호 강도(RSSI)’의 쇠락 추세를 예측하여 선제적으로 라우팅을 우회해야 한다.

그 구체적인 예측 필터링 기법이 다음 절(38.4.4.1 칼만 필터 기반 RSSI 평활화)에서 전개될 핵심 주제다.
