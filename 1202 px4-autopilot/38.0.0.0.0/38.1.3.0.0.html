38.1.3 제어권(Control Authority) 전이의 상호 배제적(Mutually Exclusive) 논리 회로

PX4-Autopilot의 역동적인 비행 환경에서 기체는 조종기의 수동 조작(RC Stick Inputs), 오프보드(Offboard) 보조 컴퓨터의 궤적 명령, GCS의 MAVLink 제어, 그리고 내부 임무 계획기(Mission Planner) 등 다수의 출처(Source)로부터 제어 명령을 받는다. 이처럼 다중 에이전트가 단일한 물리적 액추에이터(모터 및 서보)를 통제하려 할 때, 제어권(Control Authority)을 명확하고 배타적으로 설정하지 않으면 각 제어기의 출력이 충돌하여 치명적인 비행 불안정성을 초래하게 된다.

본 절에서는 PX4 아키텍처가 논리 회로 수준의 상호 배제(Mutual Exclusion) 사상을 통해 어떻게 단일 제어권 통제를 달성하는지 그 개념체계를 분석한다.

1. 상호 배제적 다중화기(Multiplexer)로서의 비행 모드(Flight Mode)

PX4의 모드 제어 체계는 전자 회로의 MUX(Multiplexer)와 본질적으로 동일한 역할을 수행한다. 현재 기체가 어떤 상태(State)에 있느냐에 따라 오직 1개의 상위 제어기만이 목표 설정값(Setpoint)을 하위 제어기로 전달할 수 있도록 강제하는 것이다.

가동 중인 제어기 모듈들은 항상 백그라운드에서 각자의 목표 궤적을 연산하고 있을지언정, 최종적으로 조향 제어기(Position/Attitude Controller)에 입력으로 들어가는 uORB 메시지(예: trajectory_setpoint, vehicle_attitude_setpoint)의 출처는 단 하나로 고정된다.

예를 들어:

  • 기체가 MISSION 상태일 때: navigator 모듈만이 trajectory_setpoint를 독점 발행한다.
  • 기체가 MANUAL 상태일 때: 조종기 입력 매퍼(manual_control_setpoint)만이 vehicle_attitude_setpoint로 변환되어 전달된다.
  • 기체가 OFFBOARD 상태일 때: MAVLink를 통해 외부 보조 컴퓨터(Companion Computer)에서 전송되는 세트포인트만이 승인된다.

이렇게 **상호 배제적(Mutually Exclusive)**으로 설계된 제어권 할당 파이프라인은 복수의 제어기가 모터 출력을 ’혼합(Blending)’하거나 덮어쓰는 행위를 원천적으로 차단한다. 이를 통해 제어 공학의 폐루프(Closed-loop) 시스템 응답이 항상 단일한 선형/비선형 동역학 모델 내에서 수학적으로 예측 가능해진다.

2. 제어권 전환 시의 과도 응답(Transient Response) 문제

A 제어기(예: MISSION)에서 B 제어기(예: POSITION)로 상호 배제적 스위치가 넘어가게 되면, 제어권의 주체가 순간적으로 달라진다. 이때 A 제어기가 출력하던 세트포인트와 B 제어기가 새롭게 생성한 세트포인트 간에 물리적 단차(Jump)가 발생할 수 있다.

제어 공학에서는 이러한 미분 불가능한 불연속점(Discontinuity)이 액추에이터에 입력될 때 발생하는 갑작스러운 충격을 ‘과도 응답(Transient Response) 충격’ 또는 피킹(Peaking) 현상이라고 부른다.

PX4의 commander와 제어 계층은 상응하는 ‘제어권 전이(Control Handoff)’ 메커니즘을 통해 충격을 완화한다.
예를 들어 자율 비행 모드에서 수동 비행 모드로 튕겨 나올 때, PX4 제어 루프는 전환되는 시점의 현재 기체 물리량(위치, 속도, 자세각)을 새로운 B 제어기의 초기 조건(Initial Condition)으로 삼고, 세트포인트를 부드럽게 스무딩(Smoothing)하는 슬루 레이트(Slew Rate) 제한 기법을 적용하여 물리적 비행 궤적이 튀어 오르는 것을 억제한다.

3. Ardupilot의 조종성(Flyability) 매핑과의 철학적 궤적 비교

Ardupilot은 전통적으로 RC 항공기 기반의 ’조종사 개입(Pilot Intervention)’을 극대화하는 방향으로 발전해 왔다. 따라서 특정 자율 비행 모드(예: Loiter 모드) 중에도 조종사가 조종기 스틱을 치면, 자율 제어기의 위치 산출값과 조종사의 스틱 입력값을 적절히 ’혼합(Mixing)’하거나 임시로 제어권을 비례적(Proportional)으로 넘겨주는 연성 전이(Soft-transition) 구조를 허용한다.

이에 반해 PX4-Autopilot은 명시적이고 엄격한 상호 배제를 추구한다. 다점 제어기 혼합을 허용할 경우 시스템의 관측 가능성(Observability) 모델이 복잡해지며 수학적 궤적 증명이 난해해지기 때문이다. PX4에서는 조종사가 스틱을 움직이더라도, 결국 그 스틱 값은 속도 세트포인트(velocity_setpoint)로 해석되어 ’단일 제어기(Position Controller)’의 입력 변수로 삽입될 뿐, 병렬적인 제어권 주체로 인정되지 않는다.

이러한 통제 아키텍처는 이어지는 하위 절에서 언급할 궤적 추종 제어기(Trajectory Tracking)와 직접 구동(Direct Actuator Control) 사이의 ’제어권 역전 방지 모델’과 ’이력 현상 필터’를 통한 채터링 억제 설계 원리로 한층 더 구체화된다.
