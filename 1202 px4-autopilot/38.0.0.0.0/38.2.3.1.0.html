38.2.3.1 비트 마스킹(Bit Masking) 및 논리곱(AND) 연산을 통한 다차원 상태 플래그(Multi-dimensional State Flags) 병렬 추출 행렬

MAVLink HEARTBEAT의 base_mode에 속한 단일 비트 토글(Toggle)과 달리, 32비트(4바이트) 크기인 custom_mode는 PX4-Autopilot 환경에서 내부적으로 고도화된 다차원(Multi-dimensional) 데이터 구조를 갖는다.

평면적인(Flat) 열거형(Enum) 숫자를 그대로 집어넣는 Ardupilot과 대비하여, PX4는 이 32비트 변수를 다시 4개의 8비트(1바이트) 구역으로 분할하여 구조체 패킹(Struct Packing)과 유사하게 래핑(Wrapping)한다.

  • 바이트 1 (Bits 0-7): reserved (스펙 예비용)
  • 바이트 2 (Bits 8-15): reserved (스펙 예비용)
  • 바이트 3 (Bits 16-23): main_mode (메인 비행 모드. 예: Manual, Auto)
  • 바이트 4 (Bits 24-31): sub_mode (서브 비행 모드. 예: Auto 내의 Mission, RTL, Takeoff 등)

1. 비트 마스킹(Bit Masking)을 통한 데이터 분리(Isolation)의 수학적 모델

GCS(QGroundControl 등)가 수신한 32비트 정수 하나에서 main_mode와 sub_mode라는 완전히 이질적인 두 가지 차원의 파라미터를 동시에, 그리고 독립적으로 끄집어내기 위해서는 비트 마스킹(Bit Masking) 기법이 원용된다.

비트 마스크는 선형대수학의 대각 행렬(Diagonal Matrix) 곱셈 필터를 적용하는 것과 수학적으로 완벽히 동일하다. 원본 데이터 중 남기고 싶은 비트의 위치에는 1을, 지우고 싶은 위치에는 0을 배치하여 논리곱(AND, &) 연산을 수행(Projection)하는 원리이다.

1.1 Main Mode 추출 필터링

main_mode는 16번 비트부터 23번 비트까지 8자리를 차지한다. 따라서 이 구역만 1로 채워진 필터를 16진수로 표현하면 0x00FF0000이 된다.
원본 custom_mode 정수에 & 0x00FF0000 연산을 가열하면, 나머지 24개 비트는 CPU 내 산술논리연산장치(ALU)에 의해 한 클럭(Clock) 사이클 만에 0으로 소거(Clear)되고 오직 main_mode의 비트 패턴만 살아남게 된다.

1.2 Sub Mode 추출 필터링

마찬가지로 sub_mode는 24번부터 31번 비트까지 최상단 8비트 구역을 차지한다. 이를 추출하는 마스크는 0xFF000000이다.
동일하게 custom_mode & 0xFF000000 논리곱 연산을 수행하면 하위 24비트의 데이터(메인 모드의 간섭 포함)가 완벽하게 배제된다.

2. ALU 병렬 처리와 O(1) 시간 복잡도 역량

이러한 논리곱 마스킹 방식이 갖는 가장 위대한 공학적 성취는, 이 복잡한 다차원 플래그 분리 연산이 문자열 파싱(String Parsing)이나 배열 순회, 중첩 조건문(Nested If) 없이 단일 어셈블리(Assembly) 명령어로 처리된다는 점이다.

CPU의 레지스터 폭(32-bit 단위) 범위 내에서 각각의 비트 쌍에 대한 AND 게이트 연산이 동시에 병렬(Parallel)로 수행되므로, 알고리즘의 시간 복잡도는 무조건 상수 시간인 O(1)을 달성한다. 이는 수십 대의 군집 드론(Swarm) 환경에서 초당 수천 개의 HEARTBEAT를 실시간으로 모니터링해야 하는 GCS 코어 스레드의 부하를 비약적으로 경감시키는 설계 패턴이다.

다만 여기까지 마스킹을 마친 데이터(예: 0x00030000)는 그 자체로는 ’3’이라는 의미 있는 C/C++ Integer 스칼라(Scalar) 값으로 직접 사용할 수 없다. 빈자리(0비트)가 하단에 붙어있어 그 값이 수만 배로 증폭되어 보이기 때문이다. 이 고유한 숫자 패턴을 0 \sim 255 범위 안의 정상적인 uint8_t 값으로 강하(Downscaling)시키기 위한 시프트(Shift) 연산의 원리가 다음 38.2.3.2절로 이어진다.
