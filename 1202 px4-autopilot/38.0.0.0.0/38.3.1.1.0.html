38.3.1.1 전송 지연(Transmission Latency)의 라플라스 변환(Laplace Transform) 전달 함수 분석 및 위상 지연(Phase Lag) 평가

분산 제어 환경에서 발생하는 지연 시간 \tau가 GCS 기반 폐루프(Closed-loop) 제어 시스템에 미치는 치명적인 영향을 수학적으로 증명하기 위해서는, 시간 영역(Time Domain, t)의 오차 모델을 주파수 영역(Frequency Domain, s)으로 넘기는 **라플라스 변환(Laplace Transform)**이 수반되어야 한다.

시간 영역에서 기체 상태 X_{uav}(t)와 GCS 수신 상태 X_{gcs}(t)의 관계는 순수한 시간 이동(Time Shift)인 X_{gcs}(t) = X_{uav}(t - \tau)로 기술된다. 이를 라플라스 변환하면 시간 지연은 지수 함수(Exponential Function) 형태로 맵핑된다.

1. 텔레메트리 지연의 전달 함수(Transfer Function) 모델링

입력을 X_{uav}(s), 출력을 X_{gcs}(s)로 두었을 때, 텔레메트리 네트워크 자체가 가지는 전달 함수 G_{delay}(s)는 다음과 같이 정의된다.

G_{delay}(s) = \frac{X_{gcs}(s)}{X_{uav}(s)} = e^{-\tau s}

순수 지연 요소(Pure Delay Element)인 e^{-\tau s}는 제어 공학에서 가장 가장 까다롭고 위험한 비선형 요소 중 하나다. 그 파괴력을 분석하기 위해 복소 변수 s에 j\omega를 대입하여 주파수 응답(Frequency Response) 특성을 보데 선도(Bode Plot) 관점에서 평가해 보자.

G_{delay}(j\omega) = e^{-j\omega\tau}

1.1 크기(Magnitude) 스펙트럼의 기만성

위 전달 함수의 크기(게인)를 구해보면 \vert G_{delay}(j\omega) \vert = \vert e^{-j\omega\tau} \vert = 1이 된다.
이는 무선 텔레메트리 지연 현상이 원본 데이터의 ‘크기’ 자체를 왜곡시키지는 않음을 의미한다. 기체가 실제로 10m/s로 날고 있다면, \tau 시간 뒤 GCS 화면에도 정확히 10m/s라는 텍스트가 렌더링 된다. 즉, 사용자 UI 화면 관점에서는 아무런 문제가 없어 보이는 완벽한 기만(Deception)을 달성한다.

1.2 위상 지연(Phase Lag)의 파괴성

그러나 위상(Phase) 각도를 구해보면 이야기가 완전히 달라진다.

\phi(\omega) = \angle e^{-j\omega\tau} = -\omega \tau \text{ (라디안)}

위상 지연은 지연 시간 \tau와 기체 기동 주파수 \omega의 곱에 정확히 비례하여 선형적으로 무한히 감소한다.
드론이 고기동을 수행하여 주파수 \omega가 높아질수록, 위상은 마이너스 무한대(-\infty)를 향해 곤두박질친다. 일반적으로 폐루프 시스템이 안정성을 유지하려면 최소 30^\circ \sim 60^\circ의 위상 여유(Phase Margin)가 확보되어야 한다.
만약 GCS가 100ms(\tau = 0.1s)의 무선 지연을 가진 채로 10Hz(\omega \approx 62.8 rad/s)의 주파수로 기체를 롤/피치 제어하려 한다면, 오직 통신 지연만으로 -6.28 라디안(\approx -360^\circ)의 폭력적인 위상 깎임 현상이 발생한다. 이는 시스템을 즉각적인 발산(Oscillation)과 추락(Crash)으로 몰고 간다.

Padé 근사화(Padé Approximation)와 비최소 위상(Non-minimum Phase) 시스템 특성

제어 공학자들은 무한 차원을 갖는 비선형 요소 e^{-\tau s}를 선형 행렬 부등식(LMI)이나 근궤적법(Root Locus)으로 쉽게 해석하기 위해 1차 파데 근사(1st-order Padé Approximation)를 즐겨 사용한다.

e^{-\tau s} \approx \frac{1 - \frac{\tau}{2}s}{1 + \frac{\tau}{2}s}

이 근사식의 분자를 보면 s = \frac{2}{\tau} 위치에 우반평면(Right-Half Plane, RHP) 영점(Zero)이 형성됨을 알 수 있다. RHP 영점은 시스템에 제어 입력을 인가했을 때 목표 방향과 반대로 움찔거리는 역응답(Inverse Response)을 유발하는 악명 높은 비최소 위상(Non-minimum Phase) 특성의 주범이다.

2. PX4 아키텍처의 설계 결론: 제어 주체의 분리

라플라스 변환 기반의 이 냉혹한 수학적 사실 때문에, PX4와 Ardupilot을 막론하고 무인항공기 생태계는 텔레메트리(\tau \gg 0)를 거치는 GCS를 더 이상 ’실시간 제어기’로 취급하지 않는다.

GCS는 오직 주파수 \omega \approx 0 (DC에 가까운 정적 상태)인 Waypoint 전송, 비행 모드 변경 등의 외곽 루프(Outer-Loop) 관리 작업만을 담당토록 아키텍처가 철저히 분리(Decoupling)되었다. 높은 \omega를 요구하는 자세 제어(Attitude)나 고속 속도 제어(Velocity) 루프는 반드시 기체 내부의 NuttX RTOS 내부 스레드나, 물리적으로 직결되어 \tau \approx 0인 동반 컴퓨터(Companion Computer, ROS 2 환경)에서만 연산되도록 강제되는 것이다.
