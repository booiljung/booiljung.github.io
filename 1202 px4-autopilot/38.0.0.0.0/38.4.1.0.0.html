38.4.1 섀넌-하틀리 정리(Shannon-Hartley Theorem) 기반 채널 용량(Channel Capacity) 제약 분석

확률론적 건전성(Connection Health)을 논하기 위해서는, 무선 데이터 링크가 근본적으로 안고 있는 물리 계층(Physical Layer)의 절대적 한계를 먼저 수학적으로 정의해야 한다. 아무리 GCS와 비행 제어기(FC)의 소프트웨어가 고도화되어 있더라도, 통신 이론의 대원칙인 **섀넌-하틀리 정리(Shannon-Hartley Theorem)**를 벗어날 수는 없기 때문이다.

이 정리는 특정 대역폭(Bandwidth)과 노이즈(Noise) 환경이 주어졌을 때, 해당 무선 채널을 통해 오류 없이 전송할 수 있는 **최대 정보 전송률(Maximum Information Rate, 즉 채널 용량)**을 규정한다.

1. 섀넌-하틀리 방정식과 UAS 텔레메트리 환경

수식은 다음과 같이 정의된다.

C = B \log_2\left(1 + \frac{S}{N}\right)

  • C: 채널 용량 (Channel Capacity, 이론적 최대 전송속도, 비트/초, bps)
  • B: 채널의 대역폭 (Bandwidth, 헤르츠, Hz)
  • S: 수신된 신호의 평균 전력 (Received Signal Power, 와트, W)
  • N: 백색 가우시안 잡음의 전력 (Noise Power, 와트, W)
  • S/N: 신호 대 잡음비 (SNR, Signal-to-Noise Ratio)

이 방정식이 드론(UAS) 제어 공학에서 시사하는 바는 명확하면서도 가혹하다.
드론이 활용하는 장거리 텔레메트리(예: RFD900, SiK Radio, 433MHz 계열) 모뎀들은 비행 거리(Range)를 극대화하기 위해 대역폭 B를 매우 좁게(Narrowband) 설계한다.

기체가 GCS로부터 수 킬로미터 멀어짐에 따라 역제곱 법칙(Inverse-square Law)에 의해 수신 신호 전력 S는 급격히 감소한다. 반면, 도심지 비행의 경우 무수히 쏟아지는 전자기파와 와이파이, 셀룰러 망으로 인해 배경 잡음 전력 N은 일정하거나 오히려 치솟는다.

결과적으로 기체가 멀어질수록 신호 대 잡음비(S/N)는 곤두박질치게 되며, 섀넌-하틀리 방정식에 따라 전체 채널 용량 C가 극도로 수축해 버린다.

병목(Bottleneck) 현상과 MAVLink 패킷 손실의 근본 원인

PX4나 Ardupilot이 파라미터(MAV_1_RATE, SR1_POSITION 등)에 의해 요구하는 초당 MAVLink 데이터 전송 요구량(Required Data Rate)을 R이라고 하자.

  1. 안정 구역 (R < C): 기체가 가까이 있어 SNR이 높을 때는 채널 용량 C가 요구량 R보다 크므로, 모든 HEARTBEAT와 ATTITUDE 패킷이 온전히 GCS에 당도한다.
  2. 한계 돌파 및 붕괴 (R \ge C): 기체가 섀넌의 물리적 한계점 너머로 진입하여 채널 용량 C가 요구 데이터 량 R보다 작아지는 순간, 텔레메트리 하드웨어의 전송 버퍼는 순식간에 포화(Saturation)상태에 이른다.

버퍼가 오버플로우되면 모뎀은 물리적으로 통신을 지연시키거나(Latency 급증), 패킷을 임의로 폐기(Drop)하기 시작한다. 바로 이때부터 GCS 관제 화면의 드론 객체가 뚝뚝 끊기며 움직이기 시작하고, HEARTBEAT가 누락되며, 통계적 패킷 손실률이 기하급수적으로 치솟는 것이다.

하드웨어 FEC와 적응형 대역폭 제어

이러한 물리적 한계를 방어하기 위해 시스템은 두 가지 조치를 취한다.

  1. 순방향 오류 정정(FEC, Forward Error Correction): 모뎀 자체가 수축된 채널 용량 C의 일부를 할당하여 리드-솔로몬(Reed-Solomon) 부호 등의 오류 복구 데이터를 추가 전송한다. 이는 신뢰성을 높이지만 실효 데이터 전송률을 더욱 갉아먹는 트레이드오프(Trade-off)를 갖는다.
  2. 동적 스트리밍 억제: 모뎀이 GCS로 쏘아주는 RADIO_STATUS MAVLink 비컨 메시지를 분석하여 링크 상태가 열화됨을 감지하면, GCS나 FC 내부에서 우선순위가 낮은 센서 데이터(예: 디버깅용 파라미터나 고속 진동 로그)의 스트리밍 주파수를 스스로 낮춰 R을 C 아래로 끌어내리는 생존 전략을 취하게 된다.
