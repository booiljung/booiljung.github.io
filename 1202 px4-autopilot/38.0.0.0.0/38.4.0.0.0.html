38.4 확률론적 통신 링크 건전성(Connection Health) 평가 로직

앞선 38.3절에서 우리는 지상 관제 시스템(GCS)이 어떻게 불완전한 텔레메트리 데이터를 모아 기체의 디지털 트윈(Digital Twin)을 빚어내는지 살펴보았다. 그러나 이 모든 추정 알고리즘의 대전제는 “GCS와 기체를 연결하는 무선 다리(Data Link)가 붕괴되지 않았다“는 믿음이다.

이 무선 링크의 생존 여부를 판단하는 과정을 통신 링크 건전성(Connection Health) 평가라고 한다. 무인기 관제 아키텍처 설계에서 가장 까다로운 난제 중 하나가 바로 이 ’건전성’을 어떻게 수학적으로 정량화할 것인가 하는 문제다.

1. RF 무선 환경의 고유한 적대성(Hostility)

기업용 유선 LAN 통신이나 광케이블 네트워크 환경이라면, 패킷 하나가 도달하지 않았을 때 “연결이 끊겼다(Disconnected)“고 단정 짓는 이분법적 논리(Boolean Logic)가 통용될지 모른다.

그러나 PX4나 Ardupilot이 작동하는 900MHz, 433MHz 텔레메트리 또는 2.4GHz Wi-Fi/RF 환경은 본질적으로 적대적(Hostile)이다. 무선 전파는 건물이나 지형지물에 부딪혀 다중 경로 페이딩(Multipath Fading)을 일으키고, 도심지의 무수한 전자기 간섭(EMI)에 의해 찢겨 나가며, 거리가 멀어짐에 따라 자유 공간 경로 손실(FSPL, Free Space Path Loss)을 겪는다.

이러한 전파의 바다에서는 초당 수십 개의 MAVLink 패킷 중 서너 개가 조용히 증발하는 현상이 일상적으로 일어난다.

2. 이분법적 단절 판별의 재앙과 오탐지(False Positive)

만약 GCS 네트워크 라우터나 PX4의 커맨더(commander.cpp) 모듈이 순진하게 제1세대 결정론적 로직—“패킷이 1초 동안 안 오면 연결이 끊긴 것이다”—을 사용한다면 어떤 일이 벌어질까?

비행 중 안테나 각도가 살짝 틀어져 1.5초간 패킷이 지연될 때마다, 시스템은 패닉에 빠져 “데이터 링크 상실(Data Link Loss)” 비상사태를 선포하고 즉각 복귀(RTL, Return to Launch) 모드를 트리거할 것이다. 이러한 높은 빈도의 **오탐지(False Positive)**는 임무 수행을 불가능하게 만들고, 멀쩡한 기체를 불필요한 비상 기동으로 몰아넣어 배터리를 고갈시키는 재앙을 초래한다.

3. 확률론적(Stochastic) 건강 상태 평가의 도입

바로 이 지점에서 현대적인 비행 제어 및 관제 소프트웨어는 확률론적(Stochastic) 접근법을 채택한다. 단순히 “연결되었다/끊겼다“의 디지털 상태가 아니라, 링크가 감당할 수 있는 한계 용량을 초과했는지, 패킷의 손실 배분(Variance)이 통계적 임계치를 넘었는지 등 연속적인 ‘건강(Health)’ 점수를 산출해 내는 것이다.

이를 위해 GCS와 기체 양측의 통신 모듈은 다음과 같은 핵심 지표들을 수집하고 수학적 필터를 거친다.

  1. 채널 대역폭 포화도 (Bandwidth Saturation)
  2. 패킷 도달 지연 시간의 분산 (Latency Variance & Jitter)
  3. 통계적 패킷 손실률 (Packet Loss Rate)
  4. 수신 신호 강도 (RSSI, Received Signal Strength Indicator)

이번 38.4절에서는 단 1회의 MAVLink HEARTBEAT 메시지 손실만으로 기체가 추락하는 우를 범하지 않기 위해, PX4 개발진과 통신 공학자들이 로직 내부에 어떠한 확률론적 방어 기제들을 겹겹이 쌓아두었는지 그 심연을 파헤쳐 본다.
