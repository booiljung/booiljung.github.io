38.2.1.2 X.25 다항식(Polynomial) 기반 순환 중복 검사(Cyclic Redundancy Check, CRC-16-CCITT) 해싱 및 무결성 검증 로직

무인항공기가 운용되는 무선 주파수(RF) 환경(가시권 텔레메트리, WiFi, LTE 등)은 전자기 간섭(EMI), 다중 경로 페이딩(Multipath Fading), 도플러 효과 등으로 인해 데이터 패킷의 비트 플립(Bit-flip) 현상이 빈번하게 발생한다. 앞서 다룬 HEARTBEAT 메시지의 custom_mode 필드 중 단 1비트만 역전되더라도, GCS는 기체가 “안전한 대기 모드“에서 “고속 자율 비행 모드“로 전환되었다고 치명적으로 오판할 수 있다.

MAVLink는 이러한 무선 링크의 태생적 불안정성을 극복하기 위해 패킷 프레임워크의 꼬리(Tail)에 강력한 2바이트 수학적 해시(Hash)인 CRC(Cyclic Redundancy Check)를 부착한다.

1. CRC-16-CCITT (X.25 다항식)의 수학적 성질

MAVLink가 채택한 해싱 알고리즘은 ITU-T에서 권고한 CRC-16-CCITT 규격이다. 이 알고리즘은 데이터를 거대한 이진 다항식(Polynomial)으로 간주하고, 이를 특정한 생성 다항식(Generator Polynomial)으로 나누었을 때 남는 ’나머지(Remainder)’를 체크섬(Checksum)으로 사용한다.

MAVLink에서 사용하는 다항식의 수학적 표현식은 다음과 같다.
P(x) = x^{16} + x^{12} + x^5 + 1
이를 16진수로 표현하면 0x1021이 된다. 이 특정 다항식(X.25 규격)은 통신 공학적으로 최적화되어 있어 다음과 같은 에러 탐지율을 수학적으로 보장한다.

  • 16비트 이하의 모든 연속적인 버스트 에러(Burst Error) 100% 탐지
  • 모든 홀수 개의 비트 에러 100% 탐지
  • 무작위 2비트 에러 연쇄 100% 탐지
  • 16비트를 초과하는 무작위 에러에 대해서도 99.998% 이상의 탐지율

송신 측(드론)은 MAVLink 헤더와 9바이트의 HEARTBEAT 페이로드를 바이트 단위로 시프트(Shift)하며 이 다항식 나눗셈의 누산기(Accumulator)를 돌리고, 최종 산출된 16비트 나머지를 패킷 맨 끝에 2바이트 체커로 덧붙여 전송한다.

MAVLink의 독창적 보안: CRC_EXTRA 바이트 주입

하지만 MAVLink의 무결성 검증은 단순한 RF 노이즈 필터링에서 끝나지 않는다. 분산 오픈소스 프로젝트 특성상 가장 흔하게 일어나는 대참사는 **“다이얼렉트 버전 불일치(Dialect Version Mismatch)”**이다.

예를 들어 드론은 2024년 버전의 HEARTBEAT(10바이트)를 펌웨어에 구워서 날고 있는데, GCS 노트북은 2020년 버전의 QGroundControl(HEARTBEAT가 9바이트라고 믿음)을 돌리고 있을 수 있다. 이 경우 바이트 배열이 밀려 완전히 엉뚱한 값으로 해석되는 치명타가 발생한다.

MAVLink는 이 의미론적(Semantic) 충돌을 막기 위해 CRC_EXTRA라는 독창적인 시드(Seed) 값을 창안했다.
코드 생성기(pymavlink)는 XML 메시지 정의서(이름, 길이, 필드 순서 등) 전체를 텍스트 해싱하여 각 메시지 ID마다 고유한 1바이트의 ’구조 변화 지문(Fingerprint)’인 CRC_EXTRA를 생성한다.

동작 시퀀스는 다음과 같다.

  1. 송신 측(드론)은 데이터 페이로드를 CRC 다항식 연산으로 해싱한다.
  2. 마지막 연산 직전에, **패킷 안에는 존재하지도 않는 가상의 1바이트(CRC_EXTRA)**를 메모리 누산기에 몰래 추가로 밀어 넣어 최종 16비트 CRC를 추출한다.
  3. 수신 측(GCS) 패킷 파서(mavlink_parse_char())는 바이트를 입력받으며 자체적으로 CRC를 계산한다.
  4. GCS 파서 역시 자신이 하드코딩으로 알고 있는 해당 메시지 ID의 CRC_EXTRA 바이트를 마지막 피연산자로 삽입한다.
  5. GCS가 계산한 CRC와 드론이 보낸 2바이트 CRC가 일치하는지 비교한다.

무결점 필터링과 드롭(Drop) 정책

수신 측에서 이 비교 결과가 일치(==)한다면 통신 공학적으로 두 가지 거대한 명제가 동시에 참(True)으로 증명된다.

  1. 물리적 무결성: 공중 무선 링크를 타고 오는 동안 단 1비트의 RF 노이즈 변형도 없었다.
  2. 논리적 무결성: 송신자와 수신자가 100% 동일한 필드 구조의 XML 호환성 계보(Dialect Version)를 공유하고 있다.

하나라도 어긋나 계산된 CRC가 틀어지면, PX4와 Ardupilot의 MAVLink 수신 스테이트 머신은 해당 패킷을 상위 어플리케이션(Commander 모듈 등)으로 올리지 않고 그 즉시 **조용히 폐기(Silently Drop)**한다. 이 수학적 철벽 덕분에 비행 제어기 코어는 오직 온전하게 신뢰할 수 있는 데이터만으로 상태 전이를 판단하게 된다.
