38.3.3.1 섀도우 객체(Shadow Object) 디자인 패턴을 활용한 기체 상태 추정치(State Estimate) 갱신 파이프라인

분산 제어 아키텍처에서 고지연(High-latency) 네트워크 너머에 있는 원격 엔티티(Drone)의 상태를 관리하기 위해 소프트웨어 엔지니어링에서는 섀도우 객체(Shadow Object) 디자인 패턴을 차용한다.

QGroundControl(QGC) 내부의 Vehicle 클래스, 혹은 PX4 공식 라이브러리인 MAVSDK의 Telemetry 플러그인 모듈이 바로 이 섀도우 객체의 완벽한 구현체다. 이 객체는 하늘에 떠 있는 기체가 뿜어내는 수많은 MAVLink 파편(Byte Stream)들을 수집하여, 백엔드 메모리 공간 내에 ’단일 진실 공급원(Single Source of Truth)’으로서의 기체 상태를 빚어낸다.

1. 상태 갱신(State Update) 파이프라인의 5단계

물리적 기체의 상태 변화가 GCS 내부의 섀도우 객체로 전이되는 과정은 엄격한 멀티스레딩(Multi-threading) 파이프라인을 거친다.

  1. 물리 계층 수신 (Reception): 수십 밀리초의 무선 지연(\tau)을 뚫고 도착한 UDP 또는 직렬(Serial) 바이트 스트림이 QGC의 LinkManager 백그라운드 I/O 스레드에 도착한다.
  2. 역직렬화 (Deserialization): MAVLinkProtocol 모듈이 바이트를 파싱(Parsing)하여 C++ 구조체(예: mavlink_attitude_t)로 디코딩한다.
  3. 객체 라우팅 (Routing): 패킷의 sysid(System ID) 속성을 검사하여, 다중 기체 구동 환경(Swarm) 중 자신과 매칭되는 Vehicle 섀도우 객체 인스턴스로 이벤트를 라우팅한다.
  4. 상태 돌연변이 (State Mutation): 섀도우 객체가 내부 캐시 메모리(예: _roll, _latitude)를 새로운 수신 값으로 덮어쓴다(Overwrite). 이 과정에서 교착 상태(Deadlock)를 방지하기 위해 정교한 뮤텍스(Mutex) 락 제어가 수반된다.
  5. UI 동기화 (Notification): 값이 변경되었음을 알리는 이벤트 신호(Qt Signal 방출)를 발생시킨다.

2. 순서 역전(Out-of-Order) 방지와 단조 시간(Monotonic Time) 검증

이 파이프라인에서 가장 치명적인 함정은 무선 UDP 통신망이 패킷의 도착 순서를 보장하지 않는다는 점이다. 드론이 급격히 기수를 돌릴 때, 100ms 전에 보낸 패킷이 공중에서 지연되다가 10ms 전에 보낸 패킷보다 늦게 GCS에 도착할 수 있다.

만약 섀도우 객체가 도착하는 족족 무비판적으로 캐시를 덮어쓴다면, GCS 화면 속 드론은 순간적으로 과거의 자세로 되돌아가는 치명적인 상태 역행(State Regression) 현상을 겪게 된다.

이를 방어하기 위해 PX4 섀도우 객체 파이프라인 내의 State Mutation(상태 돌연변이) 단계에서는 반드시 타임스탬프 검증 로직이 개입한다.
수신된 모든 MAVLink 패킷에 포함된 time_boot_ms(기체 부팅 후 경과 시간)를 추출하여, 현재 섀도우 객체가 캐싱하고 있는 타임스탬프보다 엄밀히 클(Strictly Greater) 경우에만 갱신을 허가한다. 이를 통해 분산 시스템 내에서 시간의 단조 증가(Monotonic Increase)를 강제하고 추정 상태의 인과율(Causality)을 보존한다.

3. MAVSDK 메커니즘 (QGC와의 철학적 차이)

QGroundControl이 Qt 프레임워크의 Signal/Slot 기능을 이용하여 UI 스레드로 큐 연결(Queued Connection)을 넘기는 방식(주로 인간 조종사를 위한 시각적 렌더링 목적)을 취한다면, 자율 에이전트(Autonomous Agent) 개발에 주로 쓰이는 **MAVSDK (C++)**는 다른 방식의 섀도우 파이프라인을 구축한다.

MAVSDK의 섀도우 객체(Telemetry 시스템)는 C++11 표준의 std::promise와 std::future, 그리고 콜백(Callback) 레지스트리를 활용한다. 즉, 개발자가 subscribe_attitude() 함수에 람다(Lambda) 콜백을 등록해 두면, 백그라운드 스레드에서 패킷 파이프라인 처리가 끝나는 즉시 해당 람다 함수가 비동기적으로(Asynchronously) 깨어나 자율 비행 알고리즘을 구동시키는 고성능 논블로킹(Non-blocking) 패턴을 채택하고 있다. 두 시스템 모두 분산 시스템의 섀도우 객체 패턴을 극도로 고도화한 형태라 할 수 있다.
