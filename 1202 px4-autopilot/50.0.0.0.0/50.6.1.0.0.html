50.6.1. 무선 원격 측정 링크(Telemetry) 대역폭을 고려한 기체의 운동학적 데이터 실시간 다운링크 스케줄링 및 보간(Interpolation) 렌더링

지상 관제 시스템(GCS)인 QGroundControl(QGC) 화면에 기체의 3차원 자세 모델(Attitude Indicator)과 지도 상의 궤적(Trajectory)이 부드럽게 표시되기 위해서는, 비행 제어기(Flight Controller, FC)에서 산출되는 고주파수(High frequency) 역학 데이터가 GCS로 끊김 없이 전달되어야 한다. 그러나 주로 사용되는 VHF/UHF 대역의 무선 원격 측정 링크(Telemetry Link, 예: SiK Radio, Microhard 등)는 물리적 대역폭(Bandwidth) 제한과 전파 간섭으로 인해 고속의 데이터를 원형 그대로 전송하는 데 한계가 있다.

본 절에서는 제한된 대역폭 환경에서 PX4-Autopilot이 기체의 주요 운동학적 데이터(속도, 자세, 위치)를 MAVLink 프로토콜을 통해 어떻게 스케줄링(Scheduling)하여 다운링크(Downlink)하는지 분석하고, 수신된 저주파수(Low frequency) 데이터를 QGroundControl 프론트엔드에서 어떻게 보간(Interpolation)하여 부드럽게 렌더링(Rendering)하는지 심층적으로 다룬다.

1. MAVLink 데이터 스트림 스케줄링(Scheduling) 아키텍처

비행 제어기 내부의 에스티메이터(EKF2 등)는 기체의 자세(Attitude)와 위치(Position)를 최소 250Hz 이상의 속도로 추정한다. 이를 57600 bps 속도의 텔레메트리 모뎀으로 모두 전송하면 링크 포화(Link Saturation)가 발생하여 중요 명령 제어 패킷이 유실될 위험이 높다.
PX4의 mavlink 모듈은 가용 대역폭에 따라 텔레메트리 스트림의 송신 주기를 그룹별로 스케줄링하는 레이트 리미터(Rate Limiter) 및 토큰 버킷(Token Bucket) 알고리즘 기반의 스트리밍 아키텍처를 구현하고 있다.

  • 메시지 스트림 그룹핑(Stream Grouping): PX4는 MAVLink 메시지들을 중요도와 특성에 따라 파티셔닝한다. 예를 들어, ATTITUDE(자세), LOCAL_POSITION_NED(국지 위치), GLOBAL_POSITION_INT(전역 위치) 등의 메시지는 시각적 렌더링과 밀접하여 상대적으로 높은 주파수(예: 10~30Hz)가 할당되는 반면, SYS_STATUS나 RC_CHANNELS는 더 낮은 주파수(예: 1~2Hz)로 할당된다.
  • 동적 대역폭 할당(Dynamic Bandwidth Allocation): PX4 파라미터 중 MAV_0_MODE 및 MAV_0_RATE 설정에 따라 지정된 최대 전송률을 초과하지 않도록 내부 큐(Queue) 스케줄러가 각 토픽의 송신 주파수를 런타임에 동적으로 조절(Throttling)한다.

2. QGroundControl 수신 단의 객체 바인딩 및 주파수 한계

GCS 백엔드에서 통신 링크 관리자(LinkManager)를 통해 수신된 ATTITUDE 패킷(Roll, Pitch, Yaw 데이터 포함)과 GLOBAL_POSITION_INT 패킷은 즉시 Vehicle C++ 객체의 roll, pitch, heading, latitude, longitude 속성(Property)으로 맵핑된다.
사용자의 모니터나 태블릿 PC의 디스플레이 주사율은 일반적으로 60Hz(초당 60프레임)이다. 그러나 텔레메트리로부터 전달받는 갱신 주기는 10~15Hz 남짓이므로, 단순히 데이터가 들어오는 즉시 화면을 갱신(Update)하게 되면 3D 기체 모델이 뚝뚝 끊기면서 회전하는 스터터링(Stuttering) 현상이나 계단 현상(Stepping)이 발생한다.

3. UI 렌더링을 위한 보간(Interpolation) 및 평활화(Smoothing) 기법

QGroundControl은 백엔드에서 올라오는 이산적인 저주파수 텔레메트리 데이터를 프론트엔드의 부드러운 60fps 렌더링 요소로 변환하기 위해 추가적인 필터링 및 선형 보간 알고리즘을 프론트엔드 뷰 모델(View Model) 또는 Qt 특유의 애니메이션 프레임워크에 위임한다.

3.1 객체 자세(Attitude)의 구면 선형 보간(Slerp) 또는 지수 평활화

QML 계층에 위치한 가상 인공 수평기(Artificial Horizon) 위젯은 activeVehicle.roll 속성이 바뀔 때마다 값을 곧바로 직결(Binding)하는 방식 대신, 내부 애니메이션 엔진인 Behavior on 혹은 NumberAnimation을 활용한다.

  • 이동 평균 선형 보간(Linear Interpolation): 현재 각도 ( \theta_t ) 와 이전 각도 ( \theta_{t-1} ) 의 차이를 Qt 프레임워크의 애니메이션 타이머가 정해진 지속 시간(Duration, 예: 100ms) 동안 수 십 단계로 잘게 쪼개어(Ease-out/Ease-in-out 결합) 점진적으로 UI 엘리먼트의 회전각(rotation)에 반영한다.
  • 이를 통해 10Hz 텔레메트리로 수신된 데이터도 시각적으로는 초당 60프레임 수준의 부드러움을 가진 시계열 데이터처럼 소비자에 제공된다.

3.2 위치(Position) 외삽(Extrapolation) 및 경로 렌더링

단순한 1차원 게이지 필터링 뿐 아니라 지도 상의 위치 이동의 경우, GLOBAL_POSITION_INT 메시지가 함께 전송하는 기체의 속도 벡터(( v_x, v_y, v_z ))를 활용한 외삽(Extrapolation) 필터가 고려된다.

  • Dead Reckoning 기법 반영: 다음 MAVLink 위치 패킷이 도달할 때까지 남는 공백 시간(수십~수백 ms) 동안, GCS 내부에서는 마지막으로 수신된 속도 방향을 스칼라적으로 곱해 지도상의 포인트 요소를 가상으로 전진시킨다.
  • 결과적으로 대역폭이 좁은 915MHz 텔레메트리 라디오를 쓰는 상황에서도 조종자는 항공기가 부드러운 궤적을 그리며 타겟 웨이포인트(Waypoint)로 날아가는 유체역학적 렌더링 궤적을 경험할 수 있다.

4. 고대역폭 네트워크(Wi-Fi / LTE / 5G) 연동 시의 최적화

최근의 GCS 관제 환경은 기존의 협대역(Narrowband) 텔레메트리를 벗어나 보조 컴퓨터(Companion Computer)와 연동된 모바일 통신(LTE/5G)이나 와이파이 라우팅 연결을 사용하는 경우가 늘어나고 있다.
이러한 고대역폭 IP 기반 네트워크를 인지하면, 실무적으로는 MAVLink REQUEST_DATA_STREAM 명령이나 GCS 설정을 통해 자세 제어 스트림(Stream Rate) 파라미터를 강제로 50Hz 이상으로 끌어올릴 수 있다. 이 경우 QGroundControl 측에서는 인위적인 보간 알고리즘의 비중을 낮추고, 원시 데이터 구동(Raw Data Driven) 빈도를 상향시켜 조종 지연 시간(End-to-end Latency)을 극도로 낮추는 방식(퍼스트-피규어 반응 시간 단축)으로 시스템을 동적으로 최적화(Tuning)하게 된다.

5. 결론

원격지에 존재하는 비행체의 운동 역학 데이터를 화면에 렌더링하는 과정은 단순 파싱(Parsing)에 그치지 않는다. PX4 내부 모듈의 네트워크 대역폭 한계를 고려한 동적 스트림 스케줄링 알고리즘과 짝을 맞추어, QGroundControl 프론트엔드가 자체적으로 구동하는 저간격 시계열 뷰 데이터의 평활화 및 보간(Interpolation) 렌더링 아키텍처는 제한적 통신 자원 아래서 관제사의 UI 사용성(Usability)과 상황 인지(Situational Awareness) 능력을 극대화하는 핵심 원리로 작용한다.
