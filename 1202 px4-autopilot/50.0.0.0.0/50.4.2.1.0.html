50.4.2.1. 위상 1: 공중 체공(In-air) 비행 상태 유지 분기

1. 개요

다단계 계층적 유한 상태 기계(HSM) 모델에서 첫 번째이자 기본이 되는 위상(Phase 1)은 공중 체공(In-air) 상태이다. PX4-Autopilot(펌웨어 버전 1.14 이상 기준)의 land_detector 모듈이 부팅 직후 이륙(Takeoff)을 인지하는 순간부터, 기체가 다시 목적지나 홈 고도 주변의 지면에 물리적으로 다가가 충격을 받기 전까지 유지되는 가장 길고 핵심적인 상태 분기이다.

이 위상에서는 기체의 모든 물리적 비행 제어 루프(위치 제어기, 자세 제어기, 각속도 제어기)가 100%의 제어 권한(Control Authority)을 행사하며, 이 상태가 굳건히 유지되어야만 외부 풍압이나 난기류 속에서도 안정적인 궤도 비행과 웨이포인트(Waypoint) 추종이 보장된다. 역으로 말해, 기체가 지상 100m 상공을 비행 중일 때 센서 융합의 일시적 오류로 인해 이 ‘In-air’ 상태가 ‘Ground Contact’ 기반의 하위 상태로 붕괴한다면 모터 시동 해제(Disarming)라는 치명타로 직결될 수 있으므로, 이 분기의 유지 조건은 가장 방어적(Defensive)인 형태를 띤다.

2. In-air 위상의 논리적 유지 조건 (Maintenance Variables)

land_detector 소스 코드를 바탕으로 할 때, 상태 머신이 In-air 위상에 머무르기 위해서는 하위 착륙 판정 조건들(Ground Contact Conditions) 중 적어도 하나 이상이 명확하게 **거짓(False)**임이 증명되어야 한다. 주로 다음과 같은 물리량 중 하나라도 충족되면 기체는 자유비행 상태를 지속한다.

  1. 유효 하강/상승 속도의 존재 (Significant Vertical Velocity): EKF2 추정기의 로컬 Z축 수직 속도(Vertical Velocity) 놈(Norm)이 0에 수렴하지 않고 일정 한계치 이상(예: 0.5m/s 이상)으로 동적 범위를 지니고 있을 때.
  2. 높은 추력 지령 요구 (High Thrust Setpoint): 기체를 부양하기 위해 제어기가 연산하는 출력값(Thrust Setpoint)이 최솟값(MPC_THR_MIN 연계)을 상회하는 상태일 때. 호버링(Hovering) 중이거나 상승 비행 중인 기체는 당연히 이 상태에 해당한다.
  3. 수평 가속 및 각속도 변화율의 불일치 (Kinematics Divergence): 기체가 수평 방향으로 이동 중이거나 피치/롤 축으로 동적인 각속도 변화를 보이고 있을 때.

3. 제어기 내부 적분기(Integrator) 활성화 및 포화(Saturation) 관리

In-air 위상의 핵심 공학적 특징은 비행 제어기(Flight Controller) 내부 PID 루프의 **적분기(Integrator)**가 완전히 활성화(Active)되어 동작한다는 점이다. 기체가 바람에 휩쓸려 고도를 잃거나 위치 피드백 오차가 지속해서 발생할 때, PID 제어기의 I(Integral) 항은 오차를 지속해서 누적(Accumulation)하여 대응 추력을 끌어올린다.

만약 In-air 위상에서 다른 착륙 위상으로 오탐지되어 적분기가 강제로 멈추거나 초기화(Anti-windup 개입)된다면 기체는 심한 고도 침하(Altitude Drop)를 겪게 된다. 따라서 In-air 상태 분기는 적분기가 자유롭게 외란 관측 및 보상을 수행할 수 있도록 수학적 자유도를 보장하는 방어 메커니즘을 제공한다.

4. 지상 관제 시스템(QGroundControl)과의 MAVLink 상태 동기화

In-air 상태에 있는 동안 PX4-Autopilot은 QGroundControl (v4.3 이상) 등의 GCS 시스템에 MAVLink 프로토콜의 EXTENDED_SYS_STATE 메시지를 주기적으로 브로드캐스팅(약 1Hz)한다. 이때, 메시지 필드 중 landed_state 항목은 MAV_LANDED_STATE_IN_AIR 값으로 인코딩된다.

이를 수신한 QGroundControl 프론트엔드(QML 아키텍처)는 화면 좌측 상단의 기체 텔레메트리(Telemetry) 아이콘을 ‘비행 중(Flying)’ 형태의 사용자 위젯(User Widget)으로 렌더링(Rendering)한다. 관제사는 이 아이콘의 명시적 표출을 통해 MAVLink 데이터 링크 계층(Data Link Layer)이 건강하며 현재 자율 비행 상태가 시스템적으로 이상이 없음을 시각적으로 확신할 수 있게 된다.

5. 외란 기인 오탐지(False Positive) 방어 기제: 왜 In-air 분기를 사수해야 하는가?

강한 열기둥(Thermal Updraft)을 지나가거나 돌풍(Wind Gust)에 의해 기복이 심한 환경에서 기체는 일시적으로 수직 속도가 0에 수렴하는 등, 언뜻 보기에 지면에 닿은 것과 비슷한 감각적 데이터를 마주칠 때가 있다. 이때 In-air 분기는 다단계 계층적 유한 상태 기계(HSM)의 최상단에서 블랙홀처럼 롤백(Roll-back) 조건을 감시한다.

어느 하나라도 센서 데이터의 모순(Discrepancy) - 예를 들어 수직 속도는 0인데 가속도계 센서의 노이즈 분산(Variance)이 임계치 이상으로 흔들리는 경우 - 이 발견되면, 내부 검증 로직은 false를 뱉어내고 어떠한 지연 여과(Time Hysteresis) 없이 즉시 기체의 상태를 이 In-air 분기로 환원시킨다. 이는 공학적으로 기체가 “안전이 완전히 보장되지 않으면 체공 상태로 간주하라(Fail-Safe to In-air)“는 무인기 설계 시스템의 대전제를 이데아처럼 반영한 것이다.
