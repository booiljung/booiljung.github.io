50.6.2.1. EXTENDED_SYS_STATE MAVLink 메시지 내 landed_state 열거형 필드의 패킷 역직렬화 및 프론트엔드 바인딩 변수 맵핑

지상 관제 시스템(GCS)인 QGroundControl(QGC)이 비행 제어기(PX4)의 착륙 여부를 확정 짓는 데이터의 최소 단위는 MAVLink 프로토콜의 EXTENDED_SYS_STATE (메시지 ID: 245) 패킷 내에 포함된 단일 바이트 필드인 landed_state이다. 무선 통신 링크를 통해 직렬화(Serialization)되어 전송된 이 바이트 스트림은 QGC 백엔드에서 정밀한 역직렬화(Deserialization) 과정을 거쳐 C++ 이넘(Enum)으로 변환되고, 최종적으로 프론트엔드인 QML의 선언적 UI 변수와 결합(Binding)된다.

본 절에서는 MAVLink 통신 프로토콜 수준에서 landed_state 데이터가 디코딩되는 과정부터 QGroundControl 프론트엔드의 화면 단에 이르는 변수 맵핑 아키텍처를 소스 코드 레벨에서 구체적으로 분석한다. (QGroundControl v4.2+ 및 C++11 규격 기준)

1. EXTENDED_SYS_STATE 메시지와 landed_state 열거형의 패킷 페이로드 구조

PX4-Autopilot은 MAVLINK_MSG_ID_EXTENDED_SYS_STATE 코드를 통해 시스템의 확장된 상태 정보를 주기적(보통 1~2Hz)으로 송신한다. 이 메시지의 페이로드(Payload)는 VTOL(수직 이착륙기)의 상태 정보와 착륙 상태 정보를 포함한다. 이 중 착륙과 직접적인 연관이 있는 필드는 무부호 8비트 정수형(uint8_t)인 landed_state이다.

MAVLink 프로토콜 정의(Definition)에 따르면 MAV_LANDED_STATE 열거형은 다음과 같은 이산적인 상태 값을 지닌다.

  • MAV_LANDED_STATE_UNDEFINED (0): 상태 알 수 없음 (초기화 직후)
  • MAV_LANDED_STATE_ON_GROUND (1): 지면에 완전 착륙 상태 (Landed)
  • MAV_LANDED_STATE_IN_AIR (2): 공중 비행 상태 (Flying)
  • MAV_LANDED_STATE_TAKEOFF (3), MAV_LANDED_STATE_LANDING (4) 등 과도기적(Transitional) 상태

이 필드는 네트워크 바이트 순서(Network Byte Order, 리틀 엔디안)로 패킹(Packing)되어 GCS 서버 소켓에 도달한다.

2. QGroundControl 백엔드의 패킷 역직렬화(Deserialization)

통신 채널 관리 모듈인 LinkManager가 시리얼 포트나 UDP 소켓에서 바이트 배열을 추출하면, MAVLinkProtocol::receiveBytes() 함수 안에서 MAVLink 파서(Parser)를 구동하여 시작 바이트(STX), 길이(LEN), 시퀀스(SEQ) 및 검사합(Checksum)을 검증한다. 메시지가 유효하다고 판명되면 Vehicle::_mavlinkMessageReceived 콜백이 호출된다.

여기서 구조체로의 역직렬화를 위해 C-언어 자동 생성 헤더(MAVLink C Library) 기술이 사용된다.

// src/Vehicle/Vehicle.cc의 패킷 디코딩 로직의 간략화된 형태
void Vehicle::_handleExtendedSysState(const mavlink_message_t& message)
{
    mavlink_extended_sys_state_t extendedState;
    // 직렬화된 바이트 배열(message)에서 C-구조체(extendedState)로 값 복사 및 역직렬화
    mavlink_msg_extended_sys_state_decode(&message, &extendedState);

    // 열거형 추출
    uint8_t landedState = extendedState.landed_state;

    // 객체 지향적 상태 전이 처리
    if (_landed_state != landedState) {
        _landed_state = landedState;
        
        bool isFlying = (_landed_state == MAV_LANDED_STATE_IN_AIR);
        bool isLanded = (_landed_state == MAV_LANDED_STATE_ON_GROUND);

        emit flyingChanged(isFlying);
        emit landedChanged(isLanded);
    }
}

위의 디코딩 함수 mavlink_msg_extended_sys_state_decode()는 내부적으로 _MAV_RETURN_uint8_t 매크로를 사용하여 페이로드 버퍼의 특정 오프셋(Offset) 지점에 존재하는 1바이트를 읽어온다. 이 과정은 데이터 정렬(Data Alignment) 및 엔디안(Endianness) 불일치 문제를 MAVLink 라이브러리 레벨에서 완벽하게 추상화하여 보장한다.

3. 프론트엔드 바인딩 변수 맵핑 아키텍처 (Mermaid 도표)

역직렬화되어 C++ 멤버 변수로 변환된 착륙 상태는 Qt 메타 객체 시스템의 Q_PROPERTY를 통해 QML 엔진(V8/Declarative Engine)에 전역 혹은 단일 컨텍스트 변수(Context Variable) 형태로 맵핑된다.

4. QML 프레임워크의 선언적(Declarative) 상태 렌더링 연동

QGroundControl UI의 디자인을 담당하는 .qml 파일들(예: FlightDisplayView.qml, VehicleSummary.qml)은 activeVehicle이라는 이름으로 주입(Inject)된 C++ Vehicle 포인터를 직접 참조한다.
이 참조는 단순한 C++ 포인터 접근을 넘어선, 프레임워크 차원의 반응적 동기화(Reactive Synchronization)를 유도한다.

// QML UI 컴포넌트 내부에서의 바인딩 변수 활용 예시
Rectangle {
    id: flightStatusBadge
    color: activeVehicle.flying ? "green" : (activeVehicle.landed ? "orange" : "gray")

    Text {
        text: activeVehicle.flying ? qsTr("In Air") : (activeVehicle.landed ? qsTr("Landed") : qsTr("Unknown"))
    }
}

위의 QML 스니펫(Snippet)에서 activeVehicle.flying이나 activeVehicle.landed 변수를 호출하는 문법은 정적 값이 아니다. C++ 백엔드(Vehicle 클래스)에서 landedChanged 시그널이 발생할 때마다, QML 평가 엔진(Evaluator)은 상기 구문이 포함된 노드(Rectangle, Text 위젯)의 속성을 더티 플래그(Dirty Flag) 처리하여, 즉시 백엔드의 새 값을 폴링(Polling)해 오고 그에 맞추어 DOM 파싱 트리를 새로이 렌더링한다.

5. 결론

QGroundControl 내 landed_state 처리는 임베디드 통신의 하위 계층(데이터 직렬화)부터 최상위 UI 계층(반응형 바인딩 매핑)까지 이어지는 교과서적인 통신-소프트웨어 수직 통합(Vertical Integration) 아키텍처를 보여준다.
하위 1바이트에 불과한 원시 패킷의 정보는 mavlink_msg_extended_sys_state_decode() 함수를 통해 플랫폼 독립적인 스택 변수로 일차 역직렬화(Deserialization) 된다. 이 값은 다시금 Vehicle 객체의 C++ 상태 머신 논리판단을 거치고, 최종적으로 Qt의 메타 프로퍼티 시스템과 QML 바인딩 엔진의 결속을 통해 “기체 아이콘의 변경” 또는 “메뉴 표시“라는 직접적인 프론트엔드 시각 렌더링으로 표출되며, 관제사의 안전한 이착륙 통제 판단에 기여하게 된다.
