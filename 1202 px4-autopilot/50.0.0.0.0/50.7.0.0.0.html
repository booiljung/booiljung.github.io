50.7 착륙 중 센서 고장 진단 체계 및 환경 외란 감지와 안전 보장(Failsafe) 다중화 제어 로직

무인항공기(UAV)의 비행 단계 중 착륙(Landing) 단계는 지면 효과(Ground Effect), 돌풍(Gust), 예기치 못한 지형지물 등 환경적 외란이 극대화되는 가장 취약한 구간이다. PX4-Autopilot은 이 결정적 순간에 단일 센서의 오차나 고장이 치명적인 추락이나 전복(Flip-over)으로 이어지지 않도록, 정교한 **고장 진단 체계(Failure Detection)**와 **안전 보장 다중화 제어 로직(Failsafe Multiplexing Logic)**을 가동한다. 본 절에서는 PX4 아키텍처 내에서 EKF2(Extended Kalman Filter 2)와 Commander 모듈이 착륙 중 발생할 수 있는 센서 이상 및 환경적 외란을 어떻게 진단하고 대처하는지 분석한다. (적용 기준: PX4 v1.14 이상)

1. 센서 다중화(Sensor Multiplexing) 및 동적 가중치 조정

PX4는 비행의 신뢰성을 확보하기 위해 IMU, 기압계(Barometer), 자력계(Magnetometer), GPS 등 핵심 센서를 이중화 혹은 삼중화하여 탑재하는 것을 권장한다. 착륙 중에는 지면 효과로 인해 기압계 센서 퓨전에 일시적인 이상 튀어오름(Spike)이 발생할 수 있으며, 이로 인해 고착 오차(Bias)가 커지게 된다.

1.1 EKF2의 혁신(Innovation) 기반 고장 진단 체계

PX4 내부의 주력 추정기(Estimator)인 EKF2는 각 센서 데이터가 입력될 때 파생되는 ‘혁신(Innovation, 관측치와 예측치의 차이)’ 값을 실시간으로 모니터링한다. 특정 센서의 혁신 분산(Variance)이 설정된 임계치(Test Ratio > 1.0)를 초과하게 되면, EKF2 모듈은 해당 센서를 즉각 배제(Reject)하거나 가중치를 대폭 낮춘다.
착륙 중 자주 발생하는 시나리오는 다음과 같다.

  • 기압계 지면 효과(Baro Ground Effect): 기체가 지면 반발력을 받을 때 동압이 증가하여 기압계 고도가 실제보다 낮게 추정되는 현상.
  • 다중화 대처: EKF2는 이 구간에서 라이다(LiDAR) 거리 센서(Distance Sensor)나 GPS 수직 속도 데이터의 퓨전 가중치를 높임으로써(Primary Sensor 전환) 기압계 외란을 상쇄(Override)한다. Ardupilot 대비 PX4는 EKF2 프레임워크 내에서 센서의 동적 스왑(Dynamic Swap) 동작이 매우 민첩하게 설계되어 있다.

2. 환경 외란(Wind/Gust) 감지 및 대응 제어

강한 바람이나 돌풍 상황에서의 착륙은 동역학적으로 기체의 자세(Attitude)를 불안정하게 만든다. PX4 멀티로터 위치 제어기(FlightTaskAutoMapper, PositionControl)는 이러한 환경적 외란을 관측기(Observer)를 통해 추정하고 능동적으로 상쇄한다.

2.1 항력 모델 기반 외란 추정 (Drag Model Wind Estimation)

최신 PX4 버전에서는 별도의 피토관(Pitot Tube) 없이도 멀티로터의 차체 공기역학 모델(Aerodynamic Drag Model)과 IMU의 가속도 데이터를 결합하여 X/Y 평면상의 바람 벡터(Wind Vector)를 실시간으로 추정(Estimation)한다. 이 때 도출된 외란 성분은 위치 제어 루프의 피드포워드(Feed-forward) 항으로 작용하여 기체가 강풍 속에서도 수직 착륙 궤적(Vertical Descent Trajectory)을 벗어나지 않도록 강건하게 제어한다.

2.2 터치다운(Touchdown) 시점의 전복(Flip-over) 방지 로직

경사면이나 거친 지형에 접지하는 순간, 한쪽 랜딩 기어(Landing Gear)만 먼저 닿을 경우 기체는 롤(Roll) 또는 피치(Pitch) 축으로 급격한 회전 모멘트를 받는다. PX4 제어기는 다음의 Failsafe 로직을 가동하여 전복을 방지한다.

  • 접지 순간의 수직 가속도(Z-axis Jerk)를 감지하면, 즉각적으로 위치 제어(Position Control)의 적분기(Integrator/I-term) 누적을 동결(Anti-windup)시킨다.
  • 자세 제어기(Attitude Controller)의 게인(Gain) 대역폭을 축소하여, 기체가 수평 복원을 위해 과도하게 모터를 뻗는 급발진 제어 입력을 제한한다.

3. 착륙 감지 중 Failsafe 상태 머신(State Machine)

만약 착륙 중 센서 퓨전이 완전히 붕괴되는 치명적(Critical) 결함이 발생할 경우, Commander 모듈 내의 안전 보장 상태 전이(State Transition) 메커니즘이 작동하여 기법을 전환한다.

3.1 GCS 개입 무결성(Loss of Nav) 로직

GPS 신호가 빌딩 숲에서 난반사되거나 EKF2 발산(Divergence)으로 인해 정상적인 위치 제어(Position Control)가 불가능해지는 ‘Loss of Navigation’ 상황이 발생할 경우, PX4의 Failsafe 머신은 다음 중 하나의 동작을 취한다.

  1. 조종자에게 수동 개입을 요구하며, GCS(QGroundControl) 화면에 붉은색 알람 배너 발송 (CRITICAL: EKF Navigation Lost).
  2. 수동 개입이 불가할 경우(데이터 통신 단절 등), 제어 모드를 수직 속도 제어가 가능한 안전 하강(Altitude Mode Descent) 또는 최소한의 수평 유지가 가능한 무작정 하강(Descend) 모드로 강제 강등(Fallback)시켜 추락 에너지를 최소화한다.

5. 결론 및 실무 제언

PX4의 착륙 안전 모듈은 단순히 모터의 출력을 줄여 내려앉게 하는 고전적인 로봇 제어 수준을 넘어서, 확률론적 센서 퓨전(EKF2) 기반의 고장 진단과 비행 동역학 기반의 외란 상쇄(Wind Estimation) 로직이 긴밀히 통합되어 있다.
비행 안정성을 높이려는 실무자나 연구진은 단순히 QGC의 파라미터(예: 착륙 하강 속도 조절)를 변경하는 것에 그칠 것이 아니라, 기체의 하드웨어 레이아웃 설계 시 기압계가 로터 하강 기류(Propeller Downwash)의 영향을 받지 않도록 폼(Foam) 스폰지 패딩을 보강하고, 라이다 센서를 보조로 장착하여 지면 효과 구간에서의 EKF2 혁신(Innovation) 임계치 발산을 하드웨어적으로 예방하는 방어적 설계가 동반되어야 한다.
