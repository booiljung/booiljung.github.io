50.5.3.2. 지자기 센서(Magnetometer) 및 관성 센서 바이어스(Bias) 재교정을 위한 내부 상태 추정기(Estimator)의 오프셋 공분산 행렬(Covariance Matrix) 재설정

무인항공기의 자율 비행 신뢰성은 센서 측정값을 융합하여 기체의 현재 자세와 위치를 추정하는 상태 추정기(Estimator)의 성능에 크게 좌우된다. 비행 중에는 모터 전류로 인한 전자기장 간섭, 온도 변화에 따른 센서 드리프트(Drift) 등으로 인해 관성 측정 장치(IMU)와 지자기 센서(Magnetometer)의 바이어스(Bias)가 지속적으로 변동한다. 기체가 지면에 접지하여 착륙 상태를 확정지었을 때, 다음 비행의 안전을 담보하기 위해서는 추정기가 점유하고 있는 오프셋 공분산 행렬(Covariance Matrix)을 전략적으로 재설정(Reset)하고 바이어스를 초기화하는 작업이 수반되어야 한다. 본 절에서는 PX4-Autopilot의 확장 칼만 필터(EKF2) 내부 모듈을 중심으로 이러한 재설정 메커니즘을 심도 있게 분석한다.

1. 개요 및 상태 추정기(Estimator) 하부 구조 이해

PX4-Autopilot은 주로 ECL(Estimator Control Library) 기반의 EKF2(Extended Kalman Filter 2) 알고리즘을 기본 추정기로 사용한다.

EKF2는 기체의 동역학 모델(State)과 센서 모델(Measurement) 간의 공분산을 추적하여 최적의 상태를 계산한다. 기체가 장시간 비행하면 센서 자체의 열적 변형이나 모터 등 고전압 케이블에서 발생하는 역기전력(EMF)으로 인해, 지자기 센서와 자이로스코프(Gyroscope)의 참값(True value) 대비 오프셋, 즉 바이어스가 누적된다.

기체가 착륙(Landed)하여 로터 회전이 정지(Spool-down)되면, 기체는 각속도가 0 이고 가속도가 중력 측정 상태(-9.81 m/s^2)로 수렴하는 ‘정적 상태(Static State)’ 피드백에 진입한다. PX4 EKF2 모듈은 이 안정적인 정지 구간을 활용하여 비행 중 불확실성(Uncertainty)이 커진 공분산 행렬의 주요 대각 성분(Diagonal Elements) 중 센서 바이어스 항을 리셋하고, 향상된 영점(Zero) 교정을 실시간으로 백그라운드에서 암묵적으로 수행한다.

2. PX4 아키텍처 및 내부 공분산 행렬 초기화 로직 (Source Code Level)

EKF2 모듈은 vehicle_land_detected 상태와 비행 중(In-Air) 상태 플래그를 지속적으로 모니터링한다. 착륙이 확인되면, 추정기는 센서 입력의 분산(Variance) 한계치를 정적 환경 기반으로 타이트(Tight)하게 조이고, 이전에 누적된 바이어스 공분산을 재설정한다.

2.1 관성 센서 (자이로 및 가속도계) 바이어스 리셋

자이로스코프의 바이어스는 비행 중에도 칼만 필터의 상태 벡터(State Vector)에 포함되어 추정된다. (소스 코드 분석 지점: src/modules/ekf2/ 하위 EKF/control_imu_bias.cpp 등)
비행이 종료되고 disarmed 및 landed 상태가 충족되면, 알고리즘은 현재 상태 벡터 프레임에 기록되어 있는 바이어스 값을 고정(Freeze)하거나 새로운 0점 기준으로 공분산 행렬을 클램핑(Clamping)하여, 다음 시동(Arming) 시 필터가 노이즈로 인해 발산하지 않도록 조치한다.

2.2 지자기 센서 (Magnetometer) 가간섭성 분리 및 공분산 재설정

지자기 센서는 지구 자기장을 측정하여 기체의 헤딩(Heading, Yaw)을 추정하는 데 핵심적인 역할을 한다. 모터 출력에 의한 로컬 자기장 외란이 사라진 착륙 직후 시점은 자기장을 재평가하기에 이상적이다.
EKF2 내부에서는 착륙 감지 시 지자기 센서에 할당된 상태 필터 공분산 부분 행렬(Sub-matrix)의 불확실성을 초기화하여, 다음 이륙 시 필터가 지자기 센서의 값을 즉각적으로, 그리고 높은 신뢰도 수치(Weight)로 수용하도록 상태 전이를 유도한다.

/* 상태 공분산 행렬 재설정 논리 의사 코드 (Pseudo-code) 예시 */
void Ekf::resetSensorBiasesAndCovariances()
{
    // 기체가 안정적으로 지면에 착륙해 있음을 보장하는 플래그 확인
    if (_control_status.flags.in_air == false) {
        
        // 1. 자이로스코프 편향 상태 공분산(Covariance) 강제 P 행렬 축소 계수 할당
        resetGyroBiasCovariance(Initial_Gyro_Bias_Uncertainty);
        
        // 2. 가속도계 바이어스 잔여 오차 영점화 및 불확실성 리셋
        resetAccelBiasCovariance();
        
        // 3. 지방향(Earth Magnetic Field) 및 기체 축(Body Magnetic Field) 자기장 공분산 튜닝
        resetMagStateCovariance();
    }
}

3. 착륙 전후 바이어스 및 공분산 상태 전이 다이어그램 (Mermaid)

비행 역학에 따른 EKF2 로직의 센서 바이어스 융합 전략 및 공분산 행렬(Matrix P)의 상태 전이를 아래와 같이 다이어그램으로 나타낼 수 있다.

    state In_Air_Estimation {
        Normal_Flight --> High_Current_Interference: 모터 출력 상승
        High_Current_Interference --> Covariance_Growth: 지자기/관성 바이어스 불확실성(Covariance) 증가
        Covariance_Growth --> Dynamic_Bias_Learning: 칼만 필터(EKF)의 비행 중 바이어스 추정
    }
    
    Standby --> In_Air_Estimation: Takeoff
    In_Air_Estimation --> Landing_Phase: Land Command or Altitude 0
    
    state Post_Landing_Reset_Sequence {
        Landing_Phase --> Ground_Static_Check: vehicle_land_detected
        Ground_Static_Check --> Reset_Covariance_Matrix: 1. 공분산 행렬(Matrix P) 대각 성분 리셋
        Reset_Covariance_Matrix --> Update_Bias_Estimate: 2. 착륙 상태의 정적 데이터로 바이어스(Bias) 덮어쓰기
        Update_Bias_Estimate --> Publish_Estimator_Status: 3. estimator_status uORB 토픽 발행 
    }
    
    Post_Landing_Reset_Sequence --> Standby: Ready for Next Flight

4. Ardupilot (EKF3) 통합 메커니즘과의 차이점 분석

상태 추정 라이브러리 설계 관점에서 PX4(ECL/EKF2)와 Ardupilot(AP_NavEKF3)는 유사한 칼만 필터 이론을 밑바탕에 두지만, 바이어스를 리셋하고 저장하는 운용 정책에 차별성을 가진다.

  • Ardupilot (EKF3): Ardupilot은 특정 비행 후 산출된 센서 교정 상태(특히 Compass/Mag 바이어스)를 인플라이트(In-flight) 중 자동 학습(Auto-learning, 예: COMPASS_LEARN 파라미터)하고 비행 종료(Disarm) 시 이를 영구 매개변수 저장소(EEPROM/SD)에 자동 저장(Save)하는 기능을 강력하게 지원한다. 따라서 다음번 부팅 시 기존 비행에서 체득한 바이어스 값을 바로 로드한다.
  • PX4-Autopilot (EKF2): PX4는 자동 저장(In-flight calibration save)에 대해 다소 보수적인 입장을 취한다. 비행 중 학습된 상태 추정기의 동적 바이어스 오프셋을 착륙 시 초기화(Reset)하여 현재 위치와 온도에 맞춘 정적 오프셋으로 복원하는 데 집중한다. 즉, 환경 변화에 대한 강건성을 고려하여 새로운 이륙 시 다시 안정적인 초기 상태(Initial State)에서 출발(Warm start)하도록 공분산 행렬의 구조적 리셋에 더 치중하는 아키텍처를 보유한다.

5. MAVLink 데이터 버스 및 QGroundControl 연동 관제 관점

GCS 관제상 지자기 센서 또는 위치 추정 필터가 공분산 리셋 불량으로 인해 발산(Diverge)할 경우, QGroundControl은 MAVLink 시스템을 통해 조종자에게 경고를 시각화해야 한다.

  1. SYS_STATUS 패킷 검사: 내장된 SYS_STATUS 메시지는 기판 상의 개별 센서 헬스 비트(Health Bit)를 제공한다. 만약 바이어스 리셋에 연쇄적인 오류가 발생하여 공분산이 수렴하지 못하면, 해당 비트 플래그가 비정상 상태를 알린다.
  2. Pre-flight Check Failure 알람: 추정기 리셋 후 estimator_status의 혁신량(Innovation) 및 테스트 비율(Test Ratio) 인덱스가 스레숄드(Threshold)를 넘게 되면, QGC 비행 화면에 붉은색의 “Pre-flight Check: Magnetometer Inconsistent” 또는 “High Gyro Bias” 알람을 표시한다. 이는 조종사로 하여금 수동 캘리브레이션 탭 패널을 점검하도록 유도하는 직접적인 연동 트리거로 작동한다.

6. ROS 2 (uXRCE-DDS) 자율 관제 시스템으로의 통합 방안

PX4-ROS2 아키텍처를 기반으로 자율 주행 다중 에이전트(Swarm) 관제탑을 구성하는 경우, 센서의 바이어스 상태를 백엔드 노드에서 지속적으로 추적하는 알고리즘을 구축할 수 있다.

Micro XRCE-DDS 미들웨어 브릿지를 통해 estimator_status 및 estimator_states 토픽을 높은 빈도로 서브스크라이브 한다. ROS 2 노드 내에 별도의 ‘센서 건전성 감시(Health Monitor)’ 콜백을 만들고, 기체가 착륙한 직후 estimator_status.mag_test_ratio 와 estimator_states.states[Covariance_Index] 계열 배열의 값을 모니터링한다. 이를 통해 기체가 다음 임무 배치(Mission Deployment)를 수행할 자격(Validity)이 되는지 관제 서버 컴퓨터에서 중앙 판단형으로 결정하는 고등 분석 모델에 적용될 수 있다.
