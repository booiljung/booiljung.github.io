50.3.2.2. 접지 직전 기수 올림(Flare) 기동을 통한 수직 하강 운동 에너지의 공력 감쇠(Aerodynamic Damping) 및 지면 접근 속도 최소화

1. 개요: 기수 올림(Flare) 기동의 비행 역학적 의미

고정익 무인기 착륙의 가장 마무리에 해당하는 플레어(Flare) 기동은, 기체가 지면에 접지하기 직전에 의도적으로 기수(Nose)를 들어 올려 비행체의 날개 받음각(Angle of Attack, AoA)을 순간적으로 급증시키는 조작을 뜻한다.

활공 경로(Glide Slope)를 따라 비교적 빠른 속도로 강하하던 기체가 물리적 감속 없이 그대로 지면과 충돌하게 되면, 랜딩 기어(Landing Gear)의 통과 하중을 초과하여 기체 구조물의 심각한 파손이 일어날 확률이 높다. 따라서 접지 고도에 다다랐을 때 엘리베이터(Elevator)를 통해 기수를 피치 업(Pitch-up) 하면 날개 위아래의 양력이 순간적으로 압축 증가하여, 수직 하강 속도(Sink Rate)에 강력한 제동(Braking)을 거는 공력 감쇠(Aerodynamic Damping) 쿠션 현상이 발생한다. 더불어 증가한 형상 항력(Drag)은 기체의 수평 전진 에너지마저 안전하게 소모시킴으로써 최종적인 지면 접근 충격량을 완전히 최소화하게 된다.

2. PX4-Autopilot의 플레어(Flare) 제어 메커니즘과 상태 머신

최신 PX4-Autopilot(Firmware v1.14 기준) 시스템의 고정익 위치 제어 모듈은, 하향 레이저 거리 센서(Distance Sensor) 혹은 EKF2 융합 고도 상태망을 초당 수십 회씩 모니터링하다가 설정된 고도 임계치를 하향 돌파하면 즉시 관제 상태 머신을 플레어 트리거(Flare Trigger) 스테이트로 전이시킨다.

플레어 모드 진입과 동시에 TECS(총 에너지 제어 시스템) 기반의 표준 활공 경로 추종 로직은 일시 정지(Suspend) 처리되며, 기체는 극단적인 수직 운동 에너지를 대기 마찰과 양력 피크 점으로 치환하기 위해 독자적인 착륙 최후반 제어 알고리즘 파이프라인으로 넘어가게 된다.

3. 소스 코드(Source Code) 분석 및 궤적 오버라이드 구조

PX4 시스템 C++ 프레임워크 내부에서 착륙의 정점인 플레어 기동을 전담하는 코드는 주로 FixedwingPositionControl.cpp 와 엔진 출력 피치 관리스택을 병합한 tecs.cpp 모듈 안에 구성되어 있다.

3.1 FixedwingPositionControl::control_landing_flare() 함수 메커니즘

동역학 플레어 상태 임계점 플래그(Threshold Flag)가 켜지게 되면, 메인 제어 루프는 더 이상 웨이포인트(Mission Item)를 향한 3차원 활공각 오차 연산을 진행하지 않는다. 대신 이 국소 함수 내부에서 기체의 수직 하강 속도 편차를 집중적으로 파싱하고, 수직 가속도를 매우 매끄럽게 0 에 수렴시키는 닫힌 루프(Closed-loop) 제어를 시작한다.

3.2 감쇠율 제약과 피치 오버라이드(Pitch Override) 알고리즘

TECS 코프세서(Co-processor)는 상위로부터 내려온 플레어 모드 신호(_tecs_is_in_flare)가 수신되면, 스로틀 구동 채널에 하드 클리핑(Hard Clipping) 제약을 즉각 걸어 구동 모터 출력을 멈추거나 아이들(Idle) 최저치로 깎아버린다.
동시에 엘리베이터 피치축 목표(Pitch Setpoint)를 고정익 날개가 치명적인 실속(Stall)에 빠지지 않는 선에서 사전 계산된 최대 피치업 계수(Max Flare Pitch)로 선형 보간(Linear Interpolation)하여 공격적으로 들어올린다. 이를 통해 하강 궤적의 끝단을 일직선이 아닌 부드러운 포물선 점근성(Asymptotic Curve)으로 만들어 완전한 공력 충격 흡수를 도출한다.

4. 제어 조율을 위한 핵심 파라미터(Parameter) 세팅 가이드

QGroundControl(GCS)을 통해 사용 기체의 플레어 기동을 날개 단면적 특성(Airfoil)에 맞게 튜닝하는 일은 착륙의 무사 귀환 생존율에 직접적으로 직결된다.

  • FW_LND_FLALT: 플레어 상태망으로 전이하기 위해 시스템이 스캐닝하는 최초의 발동 지상 고도(Altitude, 단위: m). 하단 측정 센서의 딜레이나 지형의 불규칙성을 고려하여 너무 늦지도 빠르지도 않게 설정해야 한다(통상 보유 날개폭(Wingspan)의 0.5배에서 1.0배 수준이 이상적이다).
  • FW_LND_FL_SINK: 플레어 상태에 들어갔을 때, 기체가 펌웨어 단위에서 물리적으로 억제해야 하는 지면 터치 목표 수직 하강 속도(Terminal Sink Rate).
  • FW_LND_FL_PMIN / FW_LND_FL_PMAX: 플레어 기동 수행 시 제어기가 허락하는 기체 물리적 피치 각도의 하한가와 상한가 범위 제한선. 맞바람(Headwind)이 심하거나 페이로드가 무거울 경우 이 상한선 수치를 너무 공격적으로 높게 세팅하면, 갑작스런 고도 상승 역전 현상인 오버슈트/발루닝(Ballooning)이 발생하여 공중에서 동력을 상실한 채 곤두박질치는 최악의 추락 리스크가 있다.

5. 오픈소스 Ardupilot(Plane) 제어 생태계 모델과의 관제 비교

Ardupilot 무인기 통제 생태계 그룹 역시 고정익 착륙에 있어 Flare 라는 동일 명칭의 스테이트(State)를 확고하게 활용하며 제어 우선순위를 높게 부여하고 있으나, 이를 발동하고 피드백을 제어하는 타이밍 로직 뎁스(Depth)에서 약간의 다른 시스템 철학을 취하고 있다.

  • 동적 트리거 및 시간 기반 보간 예측 모델: PX4-Autopilot은 명시적인 고도 수치 및 하강 변위 목표(Velocity target)를 중심으로 한 공학적 PID 오차 추종 제어 모델을 단단하게 고수하는 것에 반해, Ardupilot Plane 모듈은 LAND_FLARE_SEC 라는 고유의 시간 도메인(Time-based) 파라미터를 갖추고 있다. 다시 말해 강하 속도 제어와 고도 제어를 나누어 연산하지 않고, ’지면 충돌까지 남은 시간’을 선제적으로 실시간 산출하여 플레어 셋업으로 전이하는 타임 윈도우(Time Window) 예측 모델링을 융합 적용하는 특징이 훨씬 더 강하다.
  • 하향 거리 센서 부재 시의 페일세이프 폴백(Fallback): 레이저 소나 및 정밀 고도 스캐닝 센서(Lidar 등)가 부재한 악조건 상황에서, PX4는 종종 EKF 융합 기압계(Barometer) 기준 오차 누적으로 인해 플레어를 땅 속으로 파고 들어가 발동시키거나 허공에서 너무 일찍 해버리는 드리프트 리스크(Drift Risk)에 상대적으로 취약점을 보일 수 있다. 반면 Ardupilot은 TECS의 피치업 감쇠 상한을 조절하여 무리한 플레어 기동을 배제하고 단순 활공 감쇠 만으로 비상 접지하는 베어본(Barebone) 착륙 로직이 조금 더 터프(Tough)하고 방어적으로 마련되어 있다.

6. 결론 요약

엔진을 달고 비행하는 묵직한 고정익 자율 에이전트 무인기가 수행하는 착륙의 대미, 기수 올림(Flare)은 엄청난 중력 가속도를 거스르며 기체가 가진 마지막 전진 운동 속도 벡터의 흐름마저 억제하여 허공에서 마찰 흡수력을 소화시키는 항공역학의 스포일러(Spoiler) 에어브레이크와 같다. PX4-Autopilot은 레이더 등 보조 센서의 정밀 고도 추정력을 십분 발휘하여, 미리 약속된 임계 고도(FW_LND_FLALT) 통과 순간 모든 동력 스로틀을 잠그고 피치를 급격히 밀어 올리는 강제 오버라이드(Override) 알고리즘으로 이 극적인 찰나를 통제한다. 관제 시스템 오퍼레이터는 보유 기체의 총 중량과 익면 항력 특성을 미리 관찰하여 파라미터 내 최대 한계를 실차 반복 튜닝(HITL 혹은 필드 테스트) 함으로써, 마치 거대한 새가 날갯짓을 펴고 사뿐히 내려앉듯 활주로를 쓸고 지나가는 최상주의 소프트 터치다운 궤적 설계를 완성할 책임이 있다.
