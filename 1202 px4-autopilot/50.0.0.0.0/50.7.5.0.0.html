50.7.5. 관제사 개입을 통한 수동 착륙 취소(Abort) 및 복행(Go-around) 제어권 강제 회수 시그널 처리 시퀀스

1. 개요 및 관제사 개입(Manual Override)의 필요성

무인항공기(UAS)의 완전 자동 착륙(Automatic Landing) 궤적 상에서 지상에 예기치 못한 동적 장애물이 난입하거나, 극심한 공기역학적 국지 돌풍(Microburst 등), 혹은 지상 관제 시스템(GCS) 운영자의 명시적 육안 판단에 의해 하강을 멈춰야 하는 비상 국면이 발생할 수 있다. 이때 기체를 지면에 충돌시키지 않고 즉각적으로 하강 궤적을 철회 한 뒤, 이탈하여 다시 재상승하는 일련의 비행 기동을 착륙 취소(Abort) 및 **복행(Go-around)**이라고 통칭한다.
PX4-Autopilot 아키텍처는 Mission 수행 중이나 Return 국면에서의 Land 모드 진입 시, 외부 조종기(RC, Radio Controller) 조작이나 지상 관제 GCS 명령으로 기존의 자율 제어 루프를 즉각적으로 차단하고 제어권을 강제 회수(Force Takeover)할 수 있는 계층화된 시그널 처리 시퀀스를 제공한다.

2. PX4와 Ardupilot의 개입 철학 및 구성(Configuration) 비교

조종사 개입(Stick Override)에 따른 자동 모드 취소 처리에 있어 시스템 간 구동 철학의 차이가 존재한다.

  • Ardupilot의 경우: 여러 모드 변경 시 조종기의 스틱을 특정 임계치 이상 작동시키는 것(Stick Mixing 또는 Override)에 대한 반응성을 파라미터로 세밀하게 쪼개어 제어한다 (예: RC 혼합율 조정). 즉, 자동 임무 중이라도 스틱 조작을 통해 궤적을 “밀어내는(Nudge)” 미세한 스틱 믹싱 방식이 자주 활용된다.
  • PX4-Autopilot의 경우: 상태 머신(State Machine) 전환이 보다 명시적으로 이루어진다. COM_RC_OVERRIDE 파라미터를 활성화하여 조종사의 조종 스틱(Pitch, Roll, Yaw, Throttle) 입력이 임계값을 상회하면, 기체는 즉시 현재 수행 중이던 자동(Auto) 모드를 전면 취소하고 위치 제어(Position) 또는 고도 제어(Altitude) 모드로 시스템의 중앙 상태를 강제 스위칭(Hard Switching)한다. 이를 통해 조종사가 기체 동역학의 지배 권한을 완전히 직관적으로 환수할 수 있도록 보장한다.

3. MAVLink 프로토콜 및 수동 명령 제어 시그널 플로우

지상 관제측(GCS)에서 MAVLink를 매개로 복행 명령을 하달할 경우의 처리 흐름을 살펴보면, MAVLink 프로토콜 내에 정의된 특정 명령어 셋(Command Set)이 활용된다.

  1. 시그널 송신: GCS는 데이터 링크를 통해 착륙 취소 의미를 내포한 COMMAND_LONG 메시지를 발행한다. 주로 MAV_CMD_DO_GO_AROUND (또는 비행 모드 변경 명령 MAV_CMD_DO_SET_MODE를 통한 강제 Position 모드 전이)명령이 활용될 수 있다.
  2. uORB 파싱 및 전달: 펌웨어 내부의 mavlink_receiver 객체가 해당 데이터를 역직렬화(Deserialize)한 뒤, vehicle_command uORB 토픽으로 메시지를 발행(Publish)한다.
  3. Commander 모듈의 상태 머신 전이: commander 데몬이 이를 구독(Subscribe)하고 검증(Validation)한 뒤, vehicle_status 토픽을 업데이트하여 내부 nav_state를 NAVIGATION_STATE_POSCTRL 등으로 강제 전이시킨다.
  4. Navigator 동작 중지: 기존 착륙 궤적을 연산하던 navigator 태스크는 유효성을 잃고 제어 루프 연산 주도권을 FlightTaskManualPosition 기반의 수동 위치 제어 모듈에 이양한다.

4. 제어기 스위칭에 따른 과도 현상(Transient) 억제 및 부드러운 인계(Bumpless Transfer)

착륙을 위해 하방으로 속도가 걸려 있던(Negative Z-velocity) 기체가 조종사 개입으로 인해 갑작스런 고도 유지 혹은 상승(Go-around) 명령을 받게 되면, 위치 제어기 및 자세 제어기의 적분기(Integrator) 누적 값과 새로운 목표치(Setpoint) 간의 커다란 괴리가 발생하게 된다.
PX4 소프트웨어 아키텍처 내부의 제어 할당기(Control Allocator) 및 FlightTaskTransition 관련 모듈들은 다음과 같은 기법을 적용하여 이 과도 응답 지연을 방지한다.

  • Setpoint Reset (Bumpless Transfer): 스위칭이 일어나는 최초 타임스텝에서 제어기는 새로운 위치 추종 기준점(Setpoint)을 현재 기체의 실측 위치 계측 값(Estimate)과 동일하게 선언하여 급격한 위상차 발생(Position Error)을 초기화한다.
  • 추력(Thrust) 포화 방지 및 Spool-up Time 고려: 하강 중 유휴(Idle) 수준에 근접한 모터 회전수 관성을 즉각적으로 극복하기 위해, 내부 궤적 생성기는 상승 속도를 이산적으로 계단식(Step) 인가하는 대신, Jerk 한곗값이 적용된 궤적(S-curve)으로 지령을 부드럽게 재배열한다. 이는 고출력 인가 시 발생할 수 있는 피치/롤 방향의 일시적 자세 흔들림(Wobble)을 원천 차단한다.

5. QGroundControl (GCS) 기반 관제 UI 아키텍처

QGroundControl(이하 QGC) 소스 코드 내부 프론트엔드(QML) 컴포넌트는 직관적인 복행 메뉴를 조종사에게 시각화하여 제공한다.

  • Fly View 인터페이스 결합: 착륙 절차에 돌입 시, 하단 슬라이더 위젯 창에는 비상 정지(Kill)와는 구분되는 ‘착륙 취소(Cancel Land)’ 혹은 ‘고도 유지(Hold)’ 버튼 패널이 팝업된다. FlightDisplayView.qml 기반의 시그널-슬롯 액션에 의해 트리거된다.
  • 단일 터치 기반 회수: 사용자가 슬라이더를 스와이프하면, C++ 클래스 단의 Vehicle::changeFlightMode 메서드를 호출, 해당 차량 모델에 대해 즉각 Hold 또는 Position 모드를 MAVLink 메시지로 발행하여 기동 안정성을 확보한다.

6. 자율 에이전트 비행을 위한 ROS2 (uXRCE-DDS) 관제 기술 연동

현대적인 무인 비행 환경에서는 사람 뿐만 아니라 보조 임무 컴퓨터(Companion Computer) 내부의 자율 에이전트가 복행 여부를 독자적으로 결정하기도 한다.

  • 컴퓨터 비전 기반 객체 인식 연동(Vision Integration): 기체의 하방 카메라 또는 분해능이 높은 LiDAR 센서 포인트 클라우드를 ROS2의 nav2 스택이나 OctoMap이 추론하여 원래의 착륙 지점(Landing Pad) 상에 장애물이 진입했는지를 실시간 검출해 낸다.
  • ROS2 Offboard 전이 메시지 퍼블리싱: 예측 충돌 시간(TTC, Time to Collision)이 임계치 미만으로 하락할 경우, 에이전트는 즉각 uXRCE-DDS 클라이언트 채널을 통해 VehicleCommand 메시지(모드 스위칭) 또는 TrajectorySetpoint 메시지(Offboard 기반 복행 상승 궤적 지령)를 퍼블리시한다. 이는 전통적인 MAVLink 기반 텔레메트리 전송 지연 한계를 마이크로초 단위로 극복하는 강력한 예외 처리 다중화(Redundancy) 기술로 대두되고 있다.
