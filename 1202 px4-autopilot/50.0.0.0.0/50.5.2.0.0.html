50.5.2 추진 시스템 동력 차단(Spool-down) 및 물리적 비행 제어권 회수(Disarm) 프로세스

본 절에서는 PX4-Autopilot(기준 펌웨어 버전 1.14 이상) 환경에서 지면 감지 모듈(land_detector)에 의해 최종 착륙(Landed) 상태가 확정된 후 이어지는 추진 시스템 동력 차단(Spool-down) 시퀀스와 기계적 제어권 회수(Disarm) 프로세스를 원리적으로 분석한다. 시동(Arming)의 역과정에 해당하는 이 과정을 올바르게 이해하여, GCS 상에서 관련 파라미터(COM_DISARM_LAND)를 최적화하고 지상에서의 로터 회전 사고를 방지하라.

1. 개요

기체가 성공적으로 착륙 조건을 충족했다고 판단하더라도 모터를 즉시 정지시킬 경우, 아직 남아있는 프로펠러의 관성(Inertia)으로 인해 비대칭 스러스트가 발생하여 기체의 뒤집힘(Flip-over)을 초래할 수 있다. PX4-Autopilot은 이를 방지하기 위해 일정 시간 동안 모터 구동 속도를 공회전(Idle) 수준에 머무르게 하거나 서서히 감소시키는 스풀-다운(Spool-down) 과정을 거치며, 그 후 논리적으로 PWM 제어 출력을 완전히 차단하는 제어권 회수(Disarming) 단계로 진입하게 된다.

2. 자동 시동 해제 타이머(Auto Disarm Timer)

착륙이 확정되면 커맨더(commander) 모듈은 내부적으로 자동 디스암(Auto-Disarm) 타이머를 가동한다. 이는 COM_DISARM_LAND 파라미터로 관리되며 시동 상태를 유지하는 유예 시간을 의미한다.

  • 파라미터: COM_DISARM_LAND
  • 역할: 착륙 확정 직후부터 실제 Disarm 명령을 실행하기까지 대기하는 지연 시간(단위: 초). 기본값은 2초이다.
  • 메커니즘: 지면 접촉 후 Landed 브로드캐스트를 수신하면, 커맨더 데몬의 루프는 매 틱(Tick)마다 이 유예 시간 경과 여부를 평가한다.
  • 이 대기 시간 동안 기체는 여전히 Armed 상태이며 모터가 최소 공회전(Idle) 임계치 속도로 회전한다. 이는 조종자가 착륙 직후 변심하여 이륙을 원할 경우 빠르게 재이륙(Takeoff)하기 위한 목적도 포함된다.

3. 동력 차단(Spool-down) 및 Disarm 천이 과정

비행 제어 소프트웨어 내부에서는 ESC(Electronic Speed Controller)와 믹서(Mixer) 또는 제어 할당 모델(Control Allocation) 체계 사이에서 구체적인 스풀-다운 이벤트가 전개된다.

  1. 상태 천이 준비: 착륙 판단 후, 위치 제어기(mc_pos_control)는 더 이상 고도 유지를 위한 추가 추력을 요구하지 않고 추력 세트포인트를 최소 임계값으로 고정한다.
  2. 타이머 만료: COM_DISARM_LAND에서 지정한 대기 시간이 완료되면, 커맨더(Commander) 모듈은 actuator_armed uORB 토픽의 armed 플래그를 false로 변경하여 발행(Publish)한다.
  3. PWM 펄스의 차단: 제어 할당기(Control Allocator) 또는 IO 펌웨어(예: PX4IO)는 armed == false 상태를 감지하는 즉시 구동 모터로 전달되던 PWM 신호를 PWM_DISARMED 파라미터(일반적으로 900us 근처) 값으로 하향 갱신한다. 이로써 추진 시스템의 물리적 동력이 완전 차단된다.

4. 프로세스 아키텍처 다이어그램 (Mermaid)

다음은 물리적 제어권 회수 과정을 도식화한 시퀀스 다이어그램이다.

    participant Commander as Commander Module
    participant Allocator as Control Allocator

    activate Commander
    Commander->>Commander: 타이머 시작 (COM_DISARM_LAND 대기)
    Commander->>Allocator: 최소 공회전(Idle) 추력 유지 명령
    
    Note over ESC: 프로펠러 회전 관성 소산 (Spool-down 단계)
    
    Commander->>Commander: 타이머 만료
    Commander->>Allocator: orb_publish(actuator_armed, false)
    deactivate Commander
    
    activate Allocator
    Allocator->>ESC: PWM_DISARMED/PWM_MIN 신호 전송
    Note over ESC: 모터 정지 (Disarmed)
    deactivate Allocator

5. Ardupilot 대비 설정 차이

Ardupilot 환경에서는 착륙 판별 기준(LANDING_DETECTOR)이 작동한 직후 DISARM_DELAY 등의 파라미터에 의존하여 시동을 종료한다. 구조적으로 Ardupilot은 착륙 모드 클래스 안쪽에서 motors->set_spool_state(...) 함수를 통해 SPOOL_SHUTDOWN 상태를 직접 설정하는 반면, PX4-Autopilot은 독립된 commander의 상태 머신 전이를 통해 actuator_armed uORB 토픽 하나로 통합 관리된다.
기반 아키텍처 관점에서 PX4는 미들웨어(uORB) 중심의 발행/구독 구조를 유지함으로써, 만일 별도의 오프보드(Offboard) 컴퓨터(보조 로봇 등)가 제어권을 인계받아야 할 때 상태 변이를 일관성 있게 인지할 수 있는 확장성을 자랑한다.

6. 결론

착륙 후 동력 차단 및 물리적 제어권 회수(Disarm) 프로세스는 COM_DISARM_LAND 타이머를 중심으로 추진 시스템의 스풀-다운 시간과 잔여 관성 충격을 해결하도록 설계되어 있다. 지상의 안전을 보장하기 위해서는 커맨더 모듈이 토픽 파라미터(actuator_armed == false)를 전환하고 PWM 출력단이 이에 호응하여 동작하는 메커니즘을 명확히 이해하고, 기체의 영점 질량이 크고 프롭 지름이 큰 기체일수록 적절한 유예 시간을 설정하여 스풀-다운이 원만히 이뤄질 수 있도록 대응하라.
