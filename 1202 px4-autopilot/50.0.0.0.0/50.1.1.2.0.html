50.1.1.2. 지정 좌표계 유도 착륙(Guided Land)과 제자리 수직 하강 착륙(RTL Land)의 공간 궤적 생성 메커니즘 차이

무인항공기(UAV)가 자율 착륙을 수행할 때, 기체가 위치한 지면의 성격이나 임무의 목적에 따라 착륙의 공간적 기하학(Spatial Geometry)은 크게 두 가지 궤적 생성(Trajectory Generation) 패턴으로 나뉜다. 하나는 현재의 위치에서 즉각적으로 고도를 낮추는 **제자리 수직 하강 착륙(Return To Launch(RTL) Land 혹은 Copter Land)**이고, 다른 하나는 오프보드(Offboard) 알고리즘이나 글로벌 좌표 구속 조건을 갖는 **지정 좌표계 유도 착륙(Guided Land)**이다. 본 소절에서는 PX4-Autopilot의 내부 궤적 생성기(Trajectory Generator)가 이 두 가지 착륙 방식을 어떻게 다르게 산출하고 통제하는지 동역학적 관점에서 분석한다.

1. 제자리 수직 하강 착륙 (RTL Land / In-place Land)

제자리 수직 하강 착륙은 임무 비행 중 배터리 부족, 데이터 링크 단절(Data Link Loss) 또는 QGroundControl(QGC)에서의 긴급 착륙 버튼 클릭 시 주로 발동되는 가장 직관적인 형태의 하강이다.

1.1 궤적 생성 원리

이 모드에서 PX4의 Navigator 모듈은 XY 평면(Horizontal Plane) 상의 위치 이동을 억제(Hold)하고, Z축(Vertical Axis) 방향의 세트포인트(Setpoint)만을 하향 갱신한다. 궤적 생성기는 저크(Jerk, 가속도의 미분값)를 제한하는 스무딩(Smoothing) 프로파일을 적용하여, 로터의 급격한 RPM 변화를 막고 기체의 자세 안정성을 극대화한다.

수직 연산 프로세스는 다음과 같다:

  1. XY축 위치 제어 루프는 기체의 현재 로컬 좌표(Local Position)를 추종 대상으로 고정한다 (vehicle_local_position_setpoint의 VX, VY = 0).
  2. Z축 하강 속도(Descent Rate)는 매개변수(Parameter) MPC_LAND_SPEED1 (고고도 하강 속도)에서 시작하여, 지면에 근접함에 따라 MPC_LAND_SPEED2 (저고도 강하 속도)로 선형적 혹은 단계적으로 감속된다.

2. 지정 좌표계 유도 착륙 (Guided / Precision Land)

지정 좌표계 유도 착륙은 기체의 현재 위치가 아닌 **글로벌 좌표(위도, 경도) 또는 비전 기반 상대 좌표(IR 비컨, ArUco 마커 등)**를 타겟으로 하여 이동하면서 하강하는 복합 궤적 생성(Complex Trajectory Generation) 방식이다. 동반 컴퓨터(Companion Computer)에서 ROS2 MAVROS 또는 uXRCE-DDS를 통해 특정 착륙 패드 위로 유도(Guidance)할 때 주로 사용된다.

2.1 궤적 생성 원리

지정 좌표계 착륙은 단순히 XY축과 Z축을 독립적으로 줄이는 것이 아니라, 공간 상의 3차원 유도 선(Line-of-Sight, LOS)을 추종하는 궤적 스케줄링을 요구한다.

  1. 지정점 접근 (Approach Phase): 기체는 MPC_XY_CRUISE 속도를 유지하며 착륙 지점 상공으로 접근한다. 이 과정에서 고도 점진적 하강(Glide Slope)을 동반할 수 있다.
  2. 동적 궤적 재계산 (Dynamic Re-planning): 센서(예: VIO, 정밀 짐벌 카메라)로부터 착륙 지점의 변동 사항이 uORB 토픽 landing_target_pose로 인입되면, 위치 제어기(FlightTaskAutoMapper)는 궤적을 실시간으로 보정(Correction)한다.
  3. 나선 혹은 경사 하강 (Spiral/Slant Descent): 고정익(Fixed-wing)이 아닌 멀티로터(Multirotor)라 할지라도, 목표 지점과의 거리 허용 오차(Acceptance Radius)에 도달하기 전까지는 수평 및 수직 궤적 연산이 병렬로 실행된다.

3.1 Ardupilot과의 아키텍처 차이점

Ardupilot은 전통적으로 정밀 착륙(Precision Land) 기능이 별도의 라이브러리(AP_LandingGear, AC_PrecLand) 단위로 구성되어 있어, Copter 모드 변경(LAND 모드 내에서 옵션 핑거링)으로 처리된다.

반면, PX4-Autopilot은 Navigator 모듈 내에서 궤적을 계획하는 FlightTask 자체가 교체되는 방식(Polymorphism)을 취한다. 즉, 수직 하강일 때는 FlightTaskAuto의 추종을, 비전 기반 유도 착륙일 때는 FlightTaskAutoMapper나 FlightTaskAutoLine으로 궤적 합성 알고리즘 객체 통째로 다형적 변환이 이루어지므로 제어계의 응답성이 더욱 명료하고 유지보수가 용이한 장점을 지닌다.
