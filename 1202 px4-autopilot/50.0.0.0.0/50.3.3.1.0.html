50.3.3.1 고정익 전진 비행 모드에서 다중 로터 수직 비행 모드로의 양력-추력 변환(Blended Control) 구간 내 시스템 동역학 모델 변경

수직 이착륙기(VTOL)가 순항(Cruise)을 마치고 목적지 상공에서 착륙을 위해 다중 로터(Multicopter, MC) 모드로 전환할 때, 물리적 양력(Lift)의 발생 주체는 고정익(Fixed-wing, FW)의 주익(Main Wing)에서 멀티로터의 추진 모터(Thrust Motor)로 변경된다. 이 과정은 순간적으로 일어날 수 없으며, 두 가지 시스템 동역학 모델이 동시에 작용하는 양력-추력 변환(Blended Control) 구간을 필연적으로 동반한다. 본 절에서는 PX4-Autopilot(Firmware v1.14 기준)이 블렌디드 제어 구간 동안 시스템 동역학 모델을 어떻게 가변적으로 통합 및 제어하는지 분석한다.

1. 양력-추력 변환(Blended Control) 구간의 시스템 동역학적 특징

고정익 전진 비행 시, 기체에 작용하는 양력 L은 대기 속도(Airspeed) V의 제곱에 비례한다 (L = \frac{1}{2} \rho V^2 S C_L). 반면, 수직 비행 모드에서는 전진 속도가 거의 0에 수렴하므로 양력은 전적으로 다중 로터의 수직 추력 T_{MC}에 의존한다. 천이(Transition) 구간, 즉 백 트랜지션(Back-Transition) 단계에서는 속도가 점진적으로 감속되므로 주익에서 발생하는 양력 L이 점차 감소하며 동력학적 불평형 상태가 유발된다.

이 불평형을 방지하기 위해 PX4 제어 시스템은 감속에 따른 양력 손실분(Loss of Lift)을 멀티로터의 추력 T_{MC}로 보상(Compensation)하는 제어 모델 변경을 수행한다. 동역학 수식으로 간략히 표현하면, 동체에 작용하는 수직 방향의 총 중력 대응 힘 F_z는 다음과 같이 두 제어 기법 간의 가중치(Weight) 함수 \alpha(t)의 영향을 받는다.

F_z = \alpha(t) \cdot T_{MC} + (1 - \alpha(t)) \cdot L_{FW}(\vert V_{air}\vert)

이때 \alpha(t)는 0에서 1로 부드럽게 증가하는 융합 가중 계수(Blending Weight Coefficient)이며, 시간에 따른 단순 선형 보간이나 현재 기체의 대기 속도에 연동된 피드백 보간을 통해 결정된다.

2. PX4 제어 아키텍처 내 가중치 융합(Weight Blending) 모델

PX4 내 vtol_att_control 모듈은 고정익(FW) 자세 제어기의 출력과 멀티로터(MC) 자세 제어기의 출력을 어떻게 병합할지 연산한다. 소스 코드 상에서 mc_roll_weight, mc_pitch_weight, mc_yaw_weight, 그리고 mc_throttle_weight 등의 변수가 이러한 역할을 담당한다.

예를 들어, 백 트랜지션이 시작되면 멀티로터 모터가 시동(Spin-up)을 걸어 지정된 최소 활성화 스로틀에 도달하고, 대기 속도에 비례하여(또는 지정된 천이 지속 시간 VT_B_TRANS_DUR을 기준으로) 다중 로터 자세 제어부의 제어 권한(Control Authority)이 서서히 100%로 증가한다.

graph TD
    A[Back-Transition Command Received] --> B[Spin-up MC Motors]
    B --> C{Airspeed & Time Monitor}
    C -->|Blending Phase| D[MC Weight = alpha]
    C -->|Blending Phase| E[FW Weight = 1 - alpha]
    D --> F[Actuator Control Mixer]
    E --> F
    F --> G[Vehicle Dynamics: Lift + Thrust Transfer]
    G --> H{Airspeed < Blend Airspeed?}
    H -->|Yes| I[Complete Transition: MC Mode Activated]
    H -->|No| C

이와 같은 융합 방식은 하드웨어 종속성을 제거한다는 장점이 있다. Ardupilot의 QuadPlane 모듈이 모터 출력(PWM) 단에서 직접적인 스케일링을 수행하는 경향이 있는 반면, PX4는 제어기의 uORB 메시지(예: vehicle_torque_setpoint, vehicle_thrust_setpoint) 레벨에서 제어 명령값을 적정 비율로 블렌딩하므로, 틸트로터(Tiltrotor) 등 다양한 형태의 VTOL에 유연하게 대응할 수 있는 아키텍처적 우위를 지닌다.

3. 동역학 모델 변경에 따른 피치(Pitch) 및 스로틀(Throttle) 제어 변화

블렌디드 구간에서 가장 극적인 모델 변경은 종방향(Longitudinal) 제어인 피치 축 및 고도(Z축) 스로틀에서 일어난다.

  1. 고도 제어 주체의 변경: 고정익에서 고도를 유지하기 위해서는 주로 피치 각도(Pitch Angle)를 올리거나 전진 추력을 높인다. 그러나 백 트랜지션 시작 시 전진용 푸셔(Pusher) 모터는 추력을 줄이거나 역추력(Reverse Thrust)으로 브레이킹을 수행하므로, 고도 제어의 주체는 피치 제어기에서 멀티로터의 Z축 스로틀 제어기로 넘어간다.
  2. 피치 자세의 목적 변환: 기존에는 양력과 고도를 조작하던 피치 각도가, 이제는 공력 감속(Aerodynamic Braking)을 극대화하기 위한 항력 유발 수단(Airbrake)으로 모델이 변경된다. 즉, 목표 피치 각도(Pitch Setpoint)가 고도 유지가 아닌 감속(Deceleration)을 위해 수동적으로 제한된 범위(예: VT_B_TRANS_PIT) 내에서 양의 각도(Nose-up)로 강제 할당된다.

4. GCS 관점과 시뮬레이션(SITL/HITL) 연동 시의 고려사항

이러한 동역학 모델 교지 구간은 관제 측면(QGroundControl 및 ROS2 연동)에서 볼 때, 기체의 응답성이 일시적으로 낮아지거나 비선형성이 커지는 구간이다. GCS에서는 이 짧은 시간(보통 3~10초 내외) 동안 기체의 vehicle_status에서 nav_state가 천이 상태임을 명확히 시각화하여 조종자에게 안내한다.

ROS2 기반 자율 제어 에이전트 개발 시, Offboard 모드 상에서 트랜지션을 강제할 때는 속도 지령(Velocity Setpoint)이 아닌 위치 지령(Position Setpoint) 혹은 고도 유지 기반 하에서 조심스럽게 마이크로 트랜지션(Micro-transition)을 설계해야 한다. 융합 가중치 \alpha(t)가 1.0 에 도달하지 않은 상태에서 무리한 회전(Yawing) 및 강한 피치 유도 명령을 전송하면, 양쪽 제어기의 출력이 상충하여 자세 상실(Attitude Loss)을 초래할 위험이 높다.

5. 요약

블렌디드 컨트롤(Blended Control) 구간의 핵심은 고정익의 유체역학적 양형 모델에서 다중 로터의 뉴턴 역학적(작용-반작용) 추력 모델로의 이음새 없는 부드러운 전환이다. PX4는 vtol_att_control 내부의 가중치 융합 알고리즘을 통해 이 동역학적 간극을 메우며, 이는 속도 및 시간 변수에 의존적인 결정론적(Deterministic) 제어로 구현되어 있어 다양한 임무 시나리오(Mission Scenario) 및 센서 융합에 안정적인 플랫폼을 제공한다.
