50.5 착륙 상태 확정 이후의 제어 시스템(FC) 내부 루프 초기화 및 논리적 종료 시퀀스

본 절에서는 PX4-Autopilot(기준 펌웨어 버전 1.14 이상)에서 지면 감지기(Land Detector)에 의해 기체의 착륙 상태(Landed State)가 최종적으로 확정된 직후, 비행 제어 시스템(Flight Controller, FC) 내부에서 연쇄적으로 발생하는 제어 루프 초기화와 논리적 비행 종료 시퀀스에 대해 상세히 분석한다.

1. 개요

기체가 지면에 완전히 접촉하여 ‘착륙(Landed)’ 상태로 판별되면, 제어 시스템은 단순히 모터를 정지시키는 물리적 과정(Disarm) 이전에 다양한 내부 루프의 상태 변수들을 안전하게 초기화(Reset)해야 한다. 이러한 초기화 및 해제 과정이 생략될 경우, 다음 비행을 위한 이륙(Takeoff) 시 지면에서 불안정한 제어 출력(예: 모터의 비정상적 가속)이 발생하거나, 지상 상태에서의 오작동이 유발될 수 있다. 따라서 비행 상태 머신(Flight State Machine)의 논리적 전환과 PID 제어기의 적분항 초기화(Integral Reset) 메커니즘을 명확히 이해하라.

2. 네비게이션 상태(Navigation State)의 논리적 전환

착륙이 확정되면 가장 먼저 uORB 메시지인 vehicle_land_detected 토픽의 landed 플래그가 true로 브로드캐스팅(Broadcasting)된다. 이 메시지를 수신하는 핵심 모듈 중 하나인 커맨더(commander) 데몬은 기체의 주 상태(Main State) 및 네비게이션 상태를 변경한다.

  • 상태 전이(State Transition): 비행 중이던 모드(예: AUTO_RTL, AUTO_MISSION, POSCTL 등)에서 착륙 상태 로직에 맞추어 네비게이션 상태가 종료 수순에 들어간다.
  • 제어 모드 변경: 기체는 수직/수평 위치 제어 루프의 출력을 점진적으로 차단하며, 위치 제어(Position Control) 모드에서 이탈 허용 상태를 관장하는 vehicle_control_mode 토픽의 플래그를 업데이트한다.

3. 제어 루프(Control Loop) 내부 변수 초기화 시퀀스

착륙 후 논리적 종료의 핵심은 제어기가 가지고 있는 이전 비행의 기억(History)을 지우는 것이다. 주 제어기인 멀티로터 위치 제어기(mc_pos_control)와 자세 제어기(mc_att_control) 내부에서는 다음과 같은 수학적 초기화가 수행된다.

3.1 PID 제어기 적분항 안티-와인드업(Anti-Windup) 초기화

비행 중 바람이나 비대칭한 무게 중심을 보상하기 위해 PID 제어기의 적분항(Integral Term)에는 상당한 에러 값이 누적된다.
기체가 지면에 닿은 상태에서 위치 또는 자세 오차가 발생할 경우, 적분항이 지속적으로 증가(Windup)하여 모터 출력이 튀는 현상을 방지해야 한다.

  • 착륙 신호가 접수되면, 제어기는 즉시 모든 적분항 I_{err}를 강제로 0으로 리셋(Reset)한다.
  • 수식적 관점에서 적분기 제어 출력 u_I(t) = K_I \int_{0}^{t} e(\tau) d\tau는 t_{land} 시점에 초기조건 u_I(t_{land}) = 0으로 재설정된다.

3.2 목표값(Setpoint) 억제 및 동기화

착륙 후에도 GCS(QGroundControl 등)나 센서 조종기로부터 지속적인 목표값(Setpoint)이 인가될 수 있다.

  • 위치/속도 목표값 추종기(Trajectory Generator)는 목표 속도(Target Velocity)를 제로 벡터(Zero Vector, [0, 0, 0]^T)로 강제 할당한다.
  • 자세 목표값 리셋: 롤(Roll), 피치(Pitch) 목표각을 수평 기준(0^\circ)으로 맞추고, 요(Yaw)는 현재 기체의 실제 헤딩(Heading) 값으로 동기화하여 불필요한 비틀림 토크를 차단한다.

4. 논리적 종료 시퀀스 다이어그램 (Mermaid)

다음은 착륙 확정 직후 발생하는 일련의 내부 시스템 논리 전개 과정을 나타낸 다이어그램이다.

sequenceDiagram
    participant LD as Land Detector
    participant Comm as Commander
    participant PosCtl as Position Controller
    participant AttCtl as Attitude Controller
    
    LD->>Comm: vehicle_land_detected (landed=true)
    Comm->>Comm: 네비게이션 상태 확인 및 착륙 확정 판별

    PosCtl->>PosCtl: 속도 목표값(Target Velocity) 0으로 초기화
    PosCtl->>PosCtl: PID 위치 적분항(I-term) 0 리셋
    PosCtl->>AttCtl: 추력(Thrust) 명령 최하단 억제
    deactivate PosCtl
    
    activate AttCtl
    AttCtl->>AttCtl: 목표 자세 보정 (Roll/Pitch=0, Yaw=Current)
    AttCtl->>AttCtl: PID 자세/각속도 적분항(I-term) 0 리셋
    deactivate AttCtl
    
    Comm->>Comm: Disarm 시퀀스 타이머 시작

5. Ardupilot 대비 설정 및 아키텍처 차이

Ardupilot은 착륙 모드(LAND Mode) 진입 후 지면 감지 시 상태 기계 전반에서 각 비행 모드 파일(예: mode_land.cpp)에 구현된 개별적인 로직에 따라 스로틀이 억제되고 적분기 누적이 차단된다.
반면, PX4-Autopilot은 uORB 메시징 퍼블리시/서브스크라이브 패턴을 철저히 따른다. land_detector가 독립적인 데몬(Daemon)으로 동작하며 생성한 메시지를 commander, mc_pos_control, mc_att_control 모듈이 각각 비동기적으로 구독하여(Subscribe), 각자의 루프 내에서 병렬적으로 적분항 리셋 및 논리적 종료 과정을 수행하는 보다 분산된 미들웨어 아키텍처를 가지고 있다.

6. 결론

착륙 상태 확정 이후의 제어 시스템 초기화 및 논리적 종료 시퀀스는 기체가 비행 상태에서 지상 대기 상태로 이행하기 위한 필수적인 안전 장치(Safety Mechanism)다. 커맨더 모듈을 주축으로 이루어지는 제어 모드 변경과 위치/자세 제어기에서의 목표값 동기화 및 PID 적분항 리셋 과정을 명확히 이해하라. 이를 통해, 개발자는 새로운 센서를 추가하거나 독자적인 비행 제어 모듈을 개발할 때 예기치 못한 제어 발산이나 지상에서의 모터 오작동을 예방할 수 있다.
