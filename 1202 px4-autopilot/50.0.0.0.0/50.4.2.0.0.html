50.4.2. 다단계 계층적 유한 상태 기계(Hierarchical State Machine) 기반의 지면 감지 위상 추론 모델

1. 개요

최신 형상의 PX4-Autopilot(펌웨어 버전 1.14 및 그 이상)은 자율 착륙 및 이륙 시나리오에서 극도의 시스템 신뢰성을 보장하기 위해 지면 접촉 감지 프로세스를 단일 이산 변수(명제)가 아닌 시계열적이고 계층적인 상태 위상(Phase) 추론 모델로 변환하여 설계하였다. 외부 풍압(Wind Gust) 혹은 로터 페이딩(Rotor Fading) 등과 같은 불규칙적인 노이즈 요인 속에서 단순히 기압계나 수직 속도계 데이터를 바탕으로 ’착륙 상태(Landed)’를 예단하는 것은 비행 제어기(Flight Controller)의 안전성을 심각하게 위협한다.

따라서 land_detector 모듈은 기체가 허공을 나는 상태(‘In-air’)에서 완전히 지면에 안착하여 제어 권한을 양도하는 상태(‘Landed’)로 전이하기까지를 여러 개의 세부 위상(Phases)으로 쪼갠 다단계 계층적 유한 상태 기계(Hierarchical State Machine, HSM) 구조를 채택하고 있다. 이러한 계층 구조는 각 관문(Gate)을 통과할 때마다 더 엄격한 휴리스틱 파라미터를 요구하며, 조건이 단 한 순간이라도 어긋날 경우 즉시 최하단(In-air)으로 복귀시킴으로써 치명적 오탐지(False Positive) 파국을 원천 봉쇄한다.

2. 지면 감지 위상의 정의 및 통계 검증 계층 전개

상호 배타적이면서 계층적으로 배열된 지면 감지 위상은 크게 다음 4개의 정상 단계(Normal Phases)와 1개의 예외 단계(Exceptional Phase)로 구성된다. (멀티로터 기준)

2.1 위상 1: 공중 체공 (In-air)

기체가 비행 제어 및 항법 알고리즘 주관하에 허공을 날고 있는 통상 상태이다. 착륙 모드(Land Mode)가 수행 중이더라도 기체가 하강하며 속도를 지니고 있다면 이 위상에 머무르며, 제어기는 위치 제어기(Position Controller)의 Z-Axis 출력을 최대 성능으로 유지한다.

2.2 위상 2: 지면 접촉 (Ground Contact)

착륙 장치(Landing Gear)가 물리적으로 지면과 닿기 시작하여 일시적인 마찰이나 반발력을 경험하는 예비 위상이다. 상태 머신은 1) 수직 속도의 임계 수렴, 2) 추력 지령(Thrust Setpoint)의 최소화 등 첫 번째 물리적 한계점(Boundary)을 넘었을 때 플래그를 참(True)으로 전이시킨다. 이 기간에도 기동 제어기(Attitude/Position Controller)는 일부 권한을 유지하며, 바운싱(Bouncing)이 일어나는 시기다.

2.3 위상 3: 부분 착륙 대기 (Maybe Landed)

Ground Contact 상태가 일정 시간(약 0.25초 이상) 소실되지 않고 일관되게 유지될 때 도달하는 상위 위상이다. 상태 머신은 외부 외란이 없으며 기체가 수평으로 슬립(Sliding)하지 않는다고 확신하기 위해 수평 속도 분산과 각속도 노이즈 검증을 덧붙여 수행한다. 이 상태에 돌입하면 내부 제어 루프의 적분기(Integrator) 동작이 제한되어 와인드업(Wind-up)에 의한 역행 플립(Flip) 현상을 사전 저지한다.

2.4 위상 4: 완전 착륙 확정 (Landed)

가장 엄격한 최종 위상이다. Maybe Landed 조건이 내부 시스템 파라미터(LNDMC_TRIG_TIME 등)에 명시된 시간 동안 단 한 번의 끊어짐 없이 유지되었을 때 비로소 진입한다. 상태 머신은 이 위상에 도달하면 즉각 시동 해제 대기 플래그(Ready for Auto-disarm)를 활성화하며, QGroundControl(QGC) 같은 지상 관제 시스템으로 EXTENDED_SYS_STATE 메시지를 발송하여 관제 화면상 기동 상태 아이콘을 갱신시킨다.

2.5 예외 위상: 자유 낙하 (Freefall)

추력 손실이나 프로펠러 파손 등으로 인해 기체가 1G (중력가속도)에 근접하는 하방 가속도를 연속적으로 경험할 때 전이되는 특수 상태이다. 이 계층으로 빠진 경우, PX4-Autopilot은 즉각적으로 동력 회로를 차단하거나 자동 낙하산(Parachute Release) 전개 루틴을 호출한다.

3. 계층적 유한 상태 기계(HSM) 상태 전이(State Transition) 다이어그램

계층적 상태 머신은 각 상위 모드가 하위 모드의 참(True) 상태를 대원칙으로 삼는 상속적 의존(Inherited Dependency) 다이어그램으로 도식화된다.

    [*] --> In_Air
    
    state Landing_Sequence {
        In_Air --> Ground_Contact : 수직속도 제로화 & 추력 최솟값 도달
        
        Ground_Contact --> Maybe_Landed : Ground_Contact 시간 유지 & 각속도 노이즈 한계 이내
        Maybe_Landed --> Landed : Maybe_Landed 상태 타임아웃 도달
        
        Ground_Contact --> In_Air : 데이터 이탈 (False)
        Maybe_Landed --> In_Air : 기체 흔들림 발생 (False)
        Landed --> In_Air : 조종사 스틱 스로틀링(Takeoff)
    }
    
    In_Air --> Freefall : 1G 임계 가속도 감지
    Freefall --> [*] : Crash / Parachute

4. 계층적 역방향 전이(Backward Transition) 메커니즘 해석

다단계 계층 설계의 핵심적인 공학적 이점은 역방향 전이(Backward Transition), 즉 롤백(Roll-back)의 즉각성에 있다. 기체가 계단을 오르듯 Ground Contact 단계를 거쳐 Maybe Landed를 통과하는 중이라도, 단 한 순간(순간의 이노베이션 스파이크 혹은 융기된 험지에 부딪혀 튕겨 나가는 현상)만 조건식 연산의 곱(AND 연산) 결과가 False로 치환되면, 상태 머신은 직전 단계로 유보하는 것이 아니라 즉시 In-air 단계 최상위 루트 코드로 폴백(Fall-back)한다.
이러한 즉각적인 상태 붕괴 로직(State Demotion Logic)은 기체가 살짝 지면에 부딪혔다가 다시 떠오르는 스키핑(Skipping) 현상 발생 시 모터 전원이 강제로 꺼지는 최악의 대형 크래시 사고를 막아내는 가장 우수한 시스템 단위의 보험 체계이다.

5. 결론 및 공학 설계 관점의 평가

서브시스템을 다단계 계층적 유한 상태 기계(HSM)로 분할하여 추론 모델을 수립하는 PX4-Autopilot의 아키텍처는 제어기의 안정화 및 지면 효과 관찰 구간을 시간적, 역학적 차원으로 완벽히 격리하는 최첨단 무인기 설계 패러다임이다.
비행 제어 로직 설계 엔지니어는 커스텀 GCS나 ROS2를 활용한 백엔드 인터페이스 구축 시, 기체가 단순히 ’비행 중’이냐 ’착륙했느냐’의 단순 이분법을 넘어 vehicle_land_detected 메시지 내의 세부 계층 플래그(Ground Contact, Maybe Landed)를 스트림으로 수집함으로써 기체의 지면 충전 강성 및 진동 거동을 고해상도로 프로파일링할 수 있다. 이는 향후 군집 비행(Swarm) 환경이나 항공모함(Moving Platform) 동적 착륙 시 추후 보상 알고리즘 설계의 귀중한 템플릿(Template)이 된다.
