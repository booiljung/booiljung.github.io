50.1.1.1. 임무 제어기(Mission Controller)와 유도 제어기(Guidance Controller) 간의 유한 상태 기계(FSM) 제어 권한 위임 토폴로지

PX4-Autopilot의 아키텍처에서 ’자율 비행(Auto Flight)’은 단일 모듈 내의 절차적 스크립트가 아니라, 다수의 유한 상태 기계(FSM, Finite State Machine) 객체들이 트리 구조로 상호작용하는 복합 다이내믹스(Dynamics)를 띈다. 자율 착륙(Auto Land) 모드가 발동될 때, 최상위 의사 결정을 담당하는 **임무 제어기(Mission Controller)**와 실제 경로의 기하학적 형상을 책임지는 유도 제어기(Guidance Controller, Navigator 내부에 위치) 사이에는 명확한 권한 위임(Delegation of Authority) 인터페이스가 존재한다. 본 절에서는 착륙 명령이 하달되는 순간 양자 간에 발생하는 FSM의 토폴로지(Topology) 기반 통제권 이양 과정을 분석한다.

1. 제어 권한의 분리: What vs How

PX4 비행 제어 스택에서 임무 제어기와 유도 제어기의 역할은 “무엇을 할 것인가(What to do)“와 “어떻게 할 것인가(How to do)“로 엄격히 분리된다.

  • 임무 제어기 (Mission Controller/Commander): 사용자의 MAVLink 명령어 수신 혹은 사전 계획된 웨이포인트(Waypoint) 리스트의 해석을 담당한다. 특정 시점에 “지금 착륙하라“는 상태적 결단을 내린다.
  • 유도 제어기 (Guidance Controller/Navigator): 커맨더가 하달한 “착륙“이라는 추상적인 미션을 넘겨받아, 이를 물리적으로 수행하기 위한 하강 속도(Descent Rate)와 3차원 위치 세트포인트(Setpoint)를 실시간 100Hz 단위로 계산한다.

1.1 Ardupilot의 단일 모놀리식(Monolithic) 제어와의 차이

Ardupilot은 AP_Mission 라이브러리가 존재하지만, 특정 비행 상태(예: Land)에 진입하면 Copter::land_run() 같은 주 제어 루프가 웨이포인트 관리와 유도 계산을 동시에 수행하는 경향이 짙다. 반면, PX4-Autopilot은 Commander (임무 결심) 모듈과 Navigator (유도 산출) 모듈이 서로 다른 프로세스로 구동되며, uORB(Micro Object Request Broker) 기반의 느슨한 결합(Loose Coupling)으로 엮여 있다. 이는 특정 유도 로직에 버그가 발생하더라도 임무 제어 로직 자체가 다운되는 페이탈 에러(Fatal Error)를 막아주는 강력한 구조적 방어선으로 작용한다.

2. 상태 권한 위임 토폴로지 (FSM Delegation Topology)

자율 착륙 모드 진입 시 발생하는 권한 위임은 FSM의 마스터-슬레이브(Master-Slave) 전환 패러다임으로 설명될 수 있다. QGroundControl(QGC) 등으로부터 착륙 명령을 받으면 다음의 과정이 진행된다.

  1. 상태 천이 판정: Commander는 내부 FSM을 갱신하여 기체의 메인 모드를 main_state_s::MAIN_STATE_AUTO_LAND로 전환하고, 이를 uORB의 vehicle_status 토픽으로 전파한다.
  2. 권한 위임: Navigator 프로세스의 메인 루틴은 vehicle_status의 변경을 감지한다. 기존 수행 중이던 임무(예: Waypoint 비행)의 FSM 실행 코드를 보류(Suspend)하고, 내부 서브 모듈인 LandItem (착륙 유도 클래스)에 실행 권한을 위임(Delegate)한다.
  3. 유도 제어 점유: LandItem (혹은 관련 FlightTask 모듈)은 자체적인 내부 FSM 서브스테이트(예: 강하 진입, 일정한 속도로 수직 강하, 지면 탐색 감속)를 가동하며 실질적인 기체 하강 궤적을 빚어낸다.

이를 제어 흐름 다이어그램으로 나타내면 아래와 같다.

3. 권한 위임의 동역학적 이점

이러한 토폴로지 구성은 시스템의 신뢰성을 근본적으로 향상시킨다. 예를 들어 착륙 중에 유도 제어기가 강풍이나 센서의 노이즈로 인해 목표 하강 속도를 유지하지 못하더라도, 상위 체계인 임무 제어기는 이와 무관하게 건재하다. 따라서 배터리 전압이 급감계선(Critical Level) 미만으로 떨어지는 상황이 발생 시, 임무 제어기는 유도 제어기의 권한을 강제로 회수하고 즉시 모터를 정지시키는 하드 디스암(Hard Disarm) 명령을 FSM 우선도(Priority) 기반으로 실행할 수 있다.

결과적으로 PX4-Autopilot의 FSM 토폴로지는 ’선 결심, 후 유도’라는 유연한 객체 연쇄 구조를 채택함으로써 자율 착륙의 안전성을 고도로 보장하는 기반을 제공한다.
