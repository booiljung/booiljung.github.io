50.3.4 이종 센서 데이터 융합(Sensor Fusion)을 이용한 지면 근접 정밀 고도 추정 칼만 필터(EKF) 설계

기체가 착륙을 위해 지면에 근접하는 단계(일반적으로 지상 고도 5~10 미터 이내)에서는 대기압의 난기류(Turbulence)나 지면 효과(Ground Effect)로 인해 기압계(Barometer)의 오차가 급격히 증가하며, 글로벌 포지셔닝 시스템(GPS)의 고도 (AMSL) 데이터 또한 고유의 정밀도 한계(대체로 ±1~3m 내외)를 지닌다. 부드럽고 안전한 착륙을 보장하기 위해 제어기(Controller)는 Z축 스로틀 제어에 사용할 신뢰성 높은 상대 고도(AGL, Above Ground Level)를 필요로 한다.
본 절에서는 PX4-Autopilot의 코어 추정기 모델인 EKF2(Extended Kalman Filter 2)가 어떤 방식으로 이종 센서(IMU, 기압계, GPS, 거리 측정 센서)의 데이터를 융합(Sensor Fusion)하여 지면 근접 환경에서 정밀한 고도를 추정해 내는지 그 아키텍처와 알고리즘 설계를 분석한다.

1. 정밀 고도 추정의 아키텍처 이해

PX4-Autopilot(Firmware v1.14 기준) 내부의 비행 상태 추정 핵심 모듈인 ekf2는 내장된 ECL(Estimation and Control Library)를 사용하여 24개 이상의 상태(States)를 지속적으로 업데이트하는 거대한 확장 칼만 필터 구조를 갖춘다. 고도 및 지면 형태 추정(Terrain Estimation)을 위해 EKF2 모듈은 다종의 uORB 토픽을 입력받아 이노베이션(Innovation) 오차를 비교하고 칼만 이득(Kalman Gain)을 조율하여 최종 로컬 Z축 좌표 지령을 갱신한다.

graph TD
    A[IMU Accelerometer Z] -->|Predict Step| EKF2(Extended Kalman Filter)
    B[Barometer] -->|Update Z_obs| C{Height Source Selector}
    D[GPS Altitude] -->|Update Z_obs| C
    E[Distance Sensor: LiDAR/Radar] -->|Update Terrain_obs| C
    
    C -->|Observation Data| EKF2
    EKF2 -->|Local Position| F(Control: mc_pos_control)
    EKF2 -->|Terrain Estimate| G(Terrain Height Model)

이 모델에서 IMU의 수직 가속도 값은 시스템의 매우 빠른 동역학 상태(Predict Step)를 추정하는 베이스 라인 역할을 하지만 이중 적분 과정에서 드리프트 오차(Drift Error)가 발산한다. 이를 억제하기 위해 기압계나 GPS 등의 느리지만 절대적인 계측값으로 측정 업데이트(Update Step)를 수행하며, 지면과 근접하는 순간부터는 광학 거리 측정기(LiDAR)나 전파 고도계(Radar)와 같은 거리 측정 센서(Distance Sensor)의 계측값이 주축을 이룬다.

2. 거리 측정 센서 기반의 EKF2 Range Aid 로직

지면 근접 시 EKF가 단순히 고도 센서를 Distance Sensor로 1:1 스위칭하는 것은 돌발적인 잡음(Spike)에 기체가 요동치는 원인이 된다. PX4의 EKF2는 이를 방지하고자 ’거리 측정기 보조(Range Aid)’라는 개념을 사용한다. PX4 소스 코드 src/modules/ekf2/ 내의 updateRangeData() 및 거리 처리 로직은 아래와 같이 동작한다.

  1. 지면 추정 분리 (Terrain Estimator Layer): EKF는 기체의 기준 절대 고도 추정기(Primary Height Estimator)와 별개로 지면의 고도(Terrain Height)를 역산출하는 별도의 상태(State)를 추적한다. 거리 측정기가 주는 측정값은 (기체 고도 - 지면 고도)의 관계를 지니므로, 이 관계식을 활용해 지면의 굴곡을 추정한다.
  2. Range Aid 제어 조건 검증 (Heuristics): EKF2_RNG_CTRL 파라미터를 통해 언제 거리 센서 데이터를 융합할지 결정한다. 수평 속도가 지정된 EKF2_RNG_A_VMAX 이하로 감소하고, 기체가 안정적인 수평에 가까울 때(통상 착륙 직전) 이노베이션 루프가 거리 측정값을 고도 추정 업데이트 스텝에 강한 칼만 이득으로 개입시킨다.
  3. 노이즈 공분산 최적 차단(Outlier Rejection): 거리 측정 센서는 풀, 나무, 바위 등의 거친 반사면에 의해 다중 반사 노이즈가 발생하기 쉽다. EKF2는 파라미터 EKF2_RNG_NOISE를 이용해 공분산 모델(Covariance Model)을 유지하며, 혁신 값(Innovation, 추정치와 계측치의 차이) 오차가 EKF2_RNG_GATE 를 초과하면 센서 값 수신을 ‘Stuck’ 처리하거나 일시적으로 융합에서 배제하여 데이터 파괴를 미연에 방지한다.

3. Ardupilot의 고도 센서 융합 시스템과의 차이

Ardupilot 역시 최신 프레임워크에서 EKF3(Extended Kalman Filter 3)를 채택하고 있어 기본 이념은 대동소이하다. 그러나 Ardupilot은 서피스 트래킹(Surface Tracking)이라는 명목 하에, 기압계의 드리프트를 Rangefinder를 통해 직접 스케일링 보정하는(예: RNGFND_GAIN) 휴리스틱을 더 깊이 결합하는 방식을 취한다.

반면 PX4-Autopilot은 철저한 수학적 융합 체계(Kalman update matrix)를 유지하는 것을 선호한다. 다시 말해, 센서를 단독의 추정 소스(Primary Source, EKF2_HGT_REF)로 강제 할당하기보다는, 각 센서의 이노베이션 분산 치수(Innovation Variance)가 임계값을 넘지 않는 한 병렬로 확률 융합 모델을 전개하여 기체가 지면의 복잡한 경사각에 직면하더라도 추정기가 발산(Divergence)하지 않도록 보호하는 수학적 견고성(Robustness)을 우선한다.

4. 모드(Mode) 전환 및 ROS2(uXRCE-DDS) 활용 시 주의사항

거리 센서 데이터의 융합 품질이 좋아질수록 Offboard 모드나 Auto.Land 모드 시 고도 좌표의 제약(Constraints)은 훨씬 예민하게 거동한다.
특히 자율 에이전트(Companion Computer)가 uXRCE-DDS 미들웨어를 통해 ROS2로 직접 거리를 계측하고 distance_sensor uORB 메시지로 PX4에 MAVLink 대신 네이티브하게 퍼블리싱(Publishing)하는 시스템 아키텍처를 도입할 경우, 타임스탬프(Timestamp) 오차로 인한 지연(Latency)이 EKF의 이노베이션 거부(Innovation Rejections) 플래그를 유발할 수 있다. QGroundControl(v4.3 이상)의 MAVLink Inspector나 ULog 분석(Flight Review)을 통해 estimator_status 토픽 내부의 hgt_test_ratio 변수를 필히 모니터링하여 센서 융합의 건강 상태(Health)를 점검해야 한다.

5. 요약

이종 센서 융합 기반의 EKF 설계는 불확실성이 지배적인 실물 환경 속에서, 대기 수식 모델과 레이저 역학 모델이라는 이종 영역의 파라미터를 통일된 좌표계로 안전하게 편입시키는 중추 기술이다. 이러한 관제 융합 기술이 뒷받침되기 때문에 착륙(Land) 모드 등의 복합 상태 관리가 신뢰성 있게 유지될 수 있다.
