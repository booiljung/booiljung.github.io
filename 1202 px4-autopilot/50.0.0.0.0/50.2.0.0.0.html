50.2. PX4 내부 실시간 미들웨어(uORB)의 착륙 명령 파싱 및 중앙 상태 전이(State Transition) 메커니즘

1. 개요 (Introduction)

무인항공기 제어 소프트웨어에서 지상 관제 시스템(GCS)으로부터 전달된 고수준(High-level)의 물리적 명령을 기체의 저수준(Low-level) 액추에이터 동작으로 변환하는 과정은 매우 복잡한 데이터 파이프라인을 요구한다. PX4-Autopilot(펌웨어 버전 v1.14 기준)은 이러한 복잡성을 해소하고 모듈 간의 결합도를 낮추기 위하여 uORB(Micro Object Request Broker)라는 독자적인 실시간 비동기 퍼블리시-서브스크라이브(Publish-Subscribe) 미들웨어를 채택하고 있다.
본 장에서는 MAVLink 통신망을 통해 유입된 착륙(Land) 명령이 uORB 미들웨어를 거치면서 어떠한 방식으로 파싱(Parsing)되며, 최종적으로 PX4의 핵심인 커맨더(Commander) 모듈에 의해 중앙 상태 전이(State Transition) 메커니즘을 어떻게 발동시키는지 컴퓨터 공학적 관점에서 심도 있게 분석한다.

2. uORB 실시간 미들웨어 아키텍처 개관

PX4의 소프트웨어 설계 철학은 철저히 모듈화(Modularity)를 지향한다. 통신 모듈, 내비게이션 모듈, 자세 제어 모듈 등 각 태스크 모듈들은 메모리에 직접 접근하여 데이터를 교환하지 않으며, 오직 uORB 이벤트 버스를 통해서만 상태 정보를 주고받는다.

  • 설계 구조: uORB는 C/C++ 구조체 기반의 토픽(Topic)을 메시지 포맷으로 정의하며, 각 토픽은 고유의 ORB ID를 갖는다.
  • 통신 방식: 데이터를 생산하는 모듈은 해당 토픽에 데이터를 발행(Publish)하고, 이를 필요로 하는 모듈은 토픽을 구독(Subscribe)한다. uORB 내부의 이벤트 폴링(Event Polling) 기법은 새로운 메시지가 도착했을 때만 수신 스레드를 깨워(Wake-up) 자원 소모를 최소화한다.

3. 착륙 명령(MAV_CMD_NAV_LAND)의 파싱 파이프라인

QGroundControl(v4.3 기준)에서 착륙 명령이 하달되어 텔레메트리를 통해 기체에 도달하면, 이는 직렬화된 바이트 스트림 형태로 mavlink_receiver 백그라운드 태스크에 수신된다. 이 모듈은 다음과 같은 데이터 파싱 과정을 수행한다.

  1. 메시지 언패킹 (Unpacking): MAVLink 프레임의 헤더와 페이로드를 검증한 후, COMMAND_LONG 데이터 패킷 구조체로 변환한다.
  2. uORB 토픽 매핑 (Topic Mapping): MAVLink 명령을 해석하여 PX4 내부의 규격화된 vehicle_command_s C++ 구조체 인터페이스로 매핑한다. 이때 command 필드에는 MAVLink의 상수인 MAV_CMD_NAV_LAND (21)가 그대로 삽입되며, 파라미터(위도, 경도, 고도 등) 구조 배열 속성 또한 플로팅 포인트(Floating-point) 포맷으로 전사된다.
  3. 발행 (Publishing): 포맷 변환이 이뤄진 vehicle_command 인스턴스를 uORB 버스망에 비동기적으로(Publish) 밀어 넣는다.

4. 커맨더(Commander) 모듈의 중앙 상태 전이 메커니즘

mavlink_receiver가 버스망에 올려놓은 vehicle_command 토픽의 가장 핵심적인 구독자(Subscriber)는 펌웨어의 두뇌 역할을 수임하는 커맨더(Commander) 모듈이다. 착륙 명령에 따른 상태 전이는 이하의 엄격한 유한 상태 기계(Finite State Machine, FSM) 검증 루틴을 관통해야만 성립된다.

4.1 조건부 제약 검사 (Pre-condition Evaluation)

단순 착륙 명령이라 할지라도 기체가 무장(Armed) 완료 상태인지, 공중에 체공 중(In-air)인지, 센서(IMU, GPS) 칼만 필터(EKF2)의 위치 수렴성이 착륙 추종 궤적을 렌더링하기에 충분한지 등을 종합적으로 심사한다. 조건에 부합하지 않는다면 앞선 장에서 서술된 바와 같이 MAV_RESULT_TEMPORARILY_REJECTED 패킷을 GCS로 송달한다.

4.2 오토 랜드(Auto Land) 상태로의 전이

상태 조건이 전부 패스(Pass)되면 커맨더는 전역 상태 변수인 nav_state (Navigation State)의 열거형 값을 NAVIGATION_STATE_AUTO_LAND로 변경(Transition)하게 된다.
이러한 상태 기계의 전이가 발생하면, 커맨더는 다시금 이 변경된 시스템 상태를 담아 vehicle_status uORB 토픽으로 네트워크에 발행한다.

4.3 하위 내비게이션 모듈 연쇄 작동

  • Navigator 모듈: vehicle_status를 주시하고 있던 Navigator 모듈은 기체가 착륙 모드로 전이되었음을 포착하고, 착륙 지점의 3차원 위치 데이터를 바탕으로 하강 궤적(Descent Trajectory) 웨이포인트(Waypoint) 셋을 동적 생성하여 버스망에 올린다.
  • 비행 태스크 (Flight Tasks): 위치 제어 모듈(Position Controller) 내부의 착륙 전문 비행 클래스 코드가 Navigator가 생성한 고도 목표치(Set-point)를 구독하여 지속적인 하강 감속 제어를 수행한다.

6. 결론

PX4의 uORB 실시간 미들웨어 아키텍처는 통신 레이어(MAVLink Receiver)와 의사결정 레이어(Commander), 제어 레이어(Controller) 간의 결합성(Coupling)을 비약적으로 낮춤으로써 시스템 안정성을 극대화하였다. 특히 단일 착륙 명령 하나가 uORB 토픽의 “파싱-발행-구독-상태 전이“라는 일련의 객체지향적 체인을 거치며 하강 동작으로 발현되는 과정은, 다목적 모듈형 로보틱스 프레임워크 설계의 모범 답안으로 평가받는다.
ROS 2에서 사용되는 DDS(Data Distribution Service) 미들웨어의 경량화 버전격인 이 uORB 구조의 안정성은, 오늘날 다양한 학술 컨퍼런스에서 드론의 제어 루프 지연성(Control Loop Latency)을 방어해내는 최적화 메커니즘으로써 널리 소개되고 연구되고 있다.
