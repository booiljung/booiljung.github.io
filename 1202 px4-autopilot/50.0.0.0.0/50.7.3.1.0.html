50.7.3.1. 절대 위치 제어(Position Control) 모드에서 상대 고도 유지 및 자세 제어(Altitude/Attitude Control) 모드로의 자동 모드 강등(Degradation) 논리 회로 활성화

글로벌 항법 위성 시스템(GNSS)의 유실은 무인항공기의 자율 비행 중 발생하는 가장 흔하면서도 치명적인 장애 요소 중 하나이다. PX4-Autopilot은 이와 같은 센서 융합 시스템의 결함을 감지했을 때, 비행 상태를 즉시 포기하거나 제어 불능에 빠뜨리지 않고 안전성을 최대한 유지하도록 설계된 형태의 내부 보호 기작을 지닌다. 본 절에서는 가장 대표적인 비상 안전 조치인 ‘절대 위치 제어(Position Control) 모드에서 상대 고도 유지(Altitude Control) 및 자세 제어(Attitude Control) 모드로의 자동 모드 강등(Degradation) 논리 회로’ 메커니즘을 상세히 다룬다.

1. 모드 강등(Mode Degradation)의 기본 개념과 철학

비행 모드의 강등이란 기체의 내비게이션 상태(Navigation State)를 높은 수준의 제어 권한에서 더 낮은 수준의 제어 권한으로 자동 하향시키는 것을 뜻한다.

위치 제어(Position Control) 모드는 X, Y축(수평 위치)과 Z축(고도)을 모두 통제하여 공간 상의 특정 3차원 좌표(Setpoint)에 기체를 완전히 고정시키는 가장 진보된 형태의 제어기다. 위치 제어 폐루프(Closed-loop)를 정상적으로 구동하기 위해서는 EKF2(Extended Kalman Filter)가 출력하는 고정밀의 전역 또는 국소 위치 융합 데이터(vehicle_local_position)가 필수적이다. 만약 GNSS 및 비전 융합 센서의 품질이 나빠져 EKF 혁신(Innovation) 편차가 급증하면, 위치 정보는 더 이상 유효하지 않게(position_valid = false) 된다.

이때 시스템은 수평 좌표축에 대한 폐루프 제어를 스스로 포기하되, 기압계(Barometer)나 레이저 고도계(LiDAR)를 통한 Z축 기반의 수직 고도 유지는 계속 살려 두기 위해 고도 제어(Altitude Control) 모드로 1차 강등된다. 만일 1차 강등 이후 고도 데이터 통계마저 신뢰할 수 없다면, 최후의 수단으로 모터 출력과 롤(Roll)/피치(Pitch) 각도만 통제하는 수동 자세 제어(Attitude/Stabilized Control) 모드로 2차 강등된다.

2. PX4-Autopilot 내 Mode Degradation 아키텍처 및 논리 회로

모드 강등 논리는 주로 PX4의 중앙 상태 머신(State Machine)인 커맨더(commander) 모듈과 비행 작업 관리자(FlightTaskManager)에 의해 물리적으로 구현되어 있다.

  1. 상태 평가 (Condition Assessment): EKF2 모듈은 각 센서의 잔차(Residual)와 혁신 분산(Innovation Variance)을 모니터링하여 지속적으로 개별 센서의 건전성을 판단하고, vehicle_local_position uORB 토픽 내에 Boolean 형태의 플래그(xy_valid, z_valid, v_xy_valid 등)를 주기적으로 발행(Publish)한다.
  2. 커맨더 모니터링 (Commander Polling): 총괄 비행 관장 스레드인 Commander 애플리케이션은 초당 수십 차례 이 uORB 플래그를 읽어 들이며 시스템이 요구하는 비행 모드의 런타임 제약(Runtime Constraints)을 지속 만족하는지 검사한다.
  3. 폴백(Fallback) 회로 활성화: 위치 유효성이 결여되어 xy_valid = false 인데, 기체의 현재 제어 모드가 AUTO.MISSION 이나 POSCTRL 형태라면 논리 회로는 이를 ’조건 불일치(Constraint Violation)’로 인식하고 폴백을 구성한다.
  4. 모드 스위칭: 커맨더는 즉시 main_state_transition() 함수를 호출하여 활성 모드를 ALTCTRL(고도 제어 기동) 등으로 변경하라는 지시를 포함한 vehicle_status 메시지를 시스템 버스(uORB)에 분배한다.

3. ArduPilot과의 처리 방식 비교

ArduPilot(ArduCopter 등) 또한 EKF 기반의 Failsafe 메커니즘을 동일하게 보유하고 있으나, 모드 강등 상태를 관장하는 시스템적 시각 아키텍처에 차이가 존재한다.

  • PX4-Autopilot: 철저하게 중앙 집중화된 통제적 논리가 돋보인다. 하위 제어기(Position Controller)는 커맨더(Commander)로부터 하향식(Top-down)으로 전달받은 nav_state에 온전히 복종한다. 제어기가 스스로 자신의 제어 모드를 변경하지 않으며, 전역적인 시스템 상태 머신이 강등을 엄격히 조율한다. 이를 통해 모듈 간 결합도를 낮추고 안전 보장의 일관성을 챙긴다.
  • ArduPilot: 주 비행 루프(Flight Loop) 내부와 EKF가 유기적으로 단단히 얽혀 있다. EKF Failsafe가 발동하면, Copter의 주 제어 루프가 직접 개입하여 FS_EKF_ACTION 파라미터 변수에 따라 Land(착륙), AltHold(고도 유지) 방식 등으로 기체 행동 기준을 즉각 변환(Override)한다. PX4에 비해 논리적 구분이 상대적으로 직관적이고 유연하나, 객체 지향적 시스템 분리도(Decoupling) 측면에서는 PX4 대비 종속성이 강하다.

4. QGroundControl (GCS) 구조 설계 및 MAVLink 관제

설계된 코어 논리 회로가 활성화되어 기체의 모드가 강등될 경우, 지상 통제(GCS) 측 조종사의 화면(UI) 상에서 즉각 시각적, 청각적 알림이 이루어져야 사고를 예방할 수 있다. QGroundControl의 소스코드(v4.3 이상 기준)에서는 내부 Vehicle.h/cpp 인스턴스가 MAVLink HEARTBEAT 메시지의 custom_mode 필드를 역직렬화(Decode)하여 현재 UI 상단 툴바에 비행 모드를 표기하고 있다.

기체 내부의 커맨더가 모드 강등을 실행함과 동시에 통신망으로는 MAVLink의 STATUSTEXT (메시지 ID: 253) 패킷을 통해 “Failsafe: Position estimate lost” 페이로드를 브로드캐스트한다. QGroundControl 내 UASMessageHandler 클래스는 텍스트 메시지의 심각도(Severity) 플래그가 긴급(CRITICAL/EMERGENCY) 레벨로 지정된 이와 같은 패킷을 수신하면, 빨간색 알림 배너를 메인 지도 화면 최상단에 강제로 출력시킨다.

자율 에이전트(Autonomous Agent) 운용 시, ROS2 네트워크 상의 자동화 관제 스크립트는 uXRCE-DDS 퍼블리셔(Publisher)를 통해 ROS 영역으로 브리지(Bridge)된 /fmu/out/vehicle_status 토픽을 실시간으로 구독(Subscribe)하여 상태를 감시한다.
만일 nav_state 필드가 NAVIGATION_STATE_POSCTRL에서 NAVIGATION_STATE_ALTCTRL 단위로 지상 관제 명령과 무관하게 자율 전이되었음을 감지하면, 에이전트 노드는 이를 즉각 비상 사태로 간주하고 아래와 같은 예외 처리 명령(MAVLink Control Method) 시퀀스를 하달할 수 있다.

  • 주변 시각 센서(Odometry) 등 대체 파이프라인의 보조 측위 데이터 주입 활성화.
  • SET_MODE MAVLink 명령 패킷 조립을 이용한 수동 원격 통제(Manual Hand-over) 모드 강제 전환 타진.

5. 소스 코드 분석: 강등 논리 구현

이러한 모드 강등 논리 회로는 PX4-Autopilot 펌웨어 소스 트리 중 /src/modules/commander/state_machine_helper.cpp 내부 유효성 함수들에 밀집되어 구현되어 있다.

다음은 위치 추정 유효성 검사 및 하향식 강등 논리 회로를 매우 단순화하여 개념적으로 나타낸 로직 예시이다.

bool set_nav_state(vehicle_status_s *status, const vehicle_local_position_s *local_pos)
{
    // 위치 상태 제약 평가 (Position Constraint Evaluation)
    bool requires_xy = (status->nav_state == vehicle_status_s::NAVIGATION_STATE_POSCTRL ||
                        status->nav_state == vehicle_status_s::NAVIGATION_STATE_AUTO_MISSION);

    if (requires_xy && !local_pos->xy_valid) {
        // XY 유효성 실패: 1차 감쇠 강등 논리 회로 (고도 제어 단일 모드로 전환 시도)
        if (local_pos->z_valid) {
            status->nav_state = vehicle_status_s::NAVIGATION_STATE_ALTCTRL;
            mavlink_log_critical(&mavlink_log_pub, "Position estimate lost, degraded to Altitude mode");
            return true; // 강등 성공
        } 
        // Z(고도) 유효성마저 동반 실패한 경우: 2차 강등 논리 회로 (수동 자세 제어 모드로 급격한 Fallback)
        else {
            status->nav_state = vehicle_status_s::NAVIGATION_STATE_STAB; // Stabilized (자세 제어 전용)
            mavlink_log_critical(&mavlink_log_pub, "Altitude estimate lost, degraded to Stabilized mode");
            return true; // 최하단 강등 성공
        }
    }
    return false; // 강등 불필요 (양호)
}

제시된 예제와 같이 펌웨어는 가장 이상적인 다차원 위치 폐루프 모델에서부터 단계별로 하나씩의 제약 조건 제어(Constraints Control)를 해제시키며 하드웨어 및 임무의 생존성을 보장한다.

결론적으로 PX4-Autopilot의 자동 모드 위치 제어 강등 회로는 센서 데이터의 물리적 소실을 소프트웨어적으로 유연하게 흡수해내는 방어적 아키텍처이다. GNSS 측위 솔루션이 마비되는 최악의 비상 항법 환경 속에서도 최소한의 Z축 고도 제어나 기체 자세를 안정적으로 유지시켜, 관성 항법 오차 범위를 지연하고 조종사 및 지상 관제망 시스템이 비상 회복(Recovery) 조치를 유도할 수 있는 공력적 골든 타임(Golden Time)을 제공하는 핵심 중추의 역할을 담당한다.
