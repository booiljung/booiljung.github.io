50.5.2.2. 조종자 실수 방지 및 시스템 안전성을 위한 자동 시동 해제(Auto-Disarm) 트리거 작동 타이머 만료 조건 검사

무인항공기 시스템(UAS)의 운용 중 발생할 수 있는 가장 치명적인 사고 중 하나는 뜻하지 않은 프로펠러 회전으로 인한 인명 피해나 기체 손상이다. 따라서 착륙을 완료하거나 비행 대기 중인 상태에서 일정 시간 이상 유의미한 제어 입력이 없을 경우, 시스템이 스스로 동력을 차단하는 자동 시동 해제(Auto-Disarm) 안전 메커니즘을 구비하는 것은 필수적이다. 본 절에서는 조종자 실수(Pilot Error)를 방지하고 시스템 안전성을 보증하기 위해 작동 타이머(Timer) 만료 조건을 검사하여 자동으로 시동을 해제하는 구조를 분석하고 설명한다. (PX4 v1.14 및 QGroundControl v4.3 기준)

1. 개요 및 시스템 목적

PX4-Autopilot은 기체의 상태(Vehicle State)를 안전하게 관리하기 위한 커맨더(Commander) 모듈을 포함하며, 여기에는 자동 시동 해제를 위한 타이머 메커니즘이 내장되어 있다. 타이머 기반의 시동 해제 로직은 다음의 두 가지 주요 상황에서 시스템을 보호한다.

  1. 이륙 전 대기 지연(Pre-flight Timeout): 기체에 시동(Armed)을 걸었으나, 조종자의 부주의, 임무(Mission) 준비 지연 혹은 통신 문제 등으로 인해 설정된 시간 내에 이륙(Takeoff)하지 않았을 경우.
  2. 착륙 후 접지 대기(Post-landing Timeout): 기체가 지면에 닿아 착륙(Landed) 상태로 진입한 이후, 추가적인 이동 명령이 발생하지 않는 상태에서 로터의 회전을 멈추어 인접 인원 및 장비를 보호하는 경우.

이 메커니즘은 단순한 타이머에 국한되지 않으며, 기체의 동역학적 상태(예: 지면과의 접촉 유무, 스로틀 입력 상태)와 함께 작동하여 오작동으로 인한 추락을 미연에 방지한다.

2. 타이머 기반 자동 시동 해제를 위한 관리 파라미터(Parameter) 설정

PX4-Autopilot은 사용자가 운용 목적에 맞게 시동 해제 타이머를 조정할 수 있도록 전역 파라미터(Global Parameters)를 제공한다. 주요 파라미터는 COM_DISARM_PRFLT와 COM_DISARM_LAND이다.

  • COM_DISARM_PRFLT: 시동을 건 후 이륙하지 않은 채로 대기할 수 있는 최대 시간(초)을 의미한다. 기본값은 보통 10초로 설정된다. 이 시간이 만료되면 기체는 자동으로 DISARMED 상태로 전환된다. 값을 0으로 설정하면 이 기능이 비활성화된다.
  • COM_DISARM_LAND: 기체가 착륙(Landed) 상태로 인식된 시점부터 시스템을 시동 해제 상태로 전이하기까지 대기하는 시간(초)이다. 착륙 순간 일시적인 튕김 현상(Bouncing)으로 인해 시동이 잠겨버리는 것을 막기 위한 버퍼 타임으로 기능하기도 한다. 기본값은 보통 2초로 설정된다.

이러한 파라미터들은 시스템 시작 시 Commander 모듈 내부의 변수로 로드(Load)되며, 내부 시스템 태스크 틱(Tick)마다 누적되는 절대 시간(Absolute Time)과 비교되어 조건 검사가 이루어진다.

3. 소프트웨어 아키텍처 및 커맨더(Commander) 내부 만료 조건 검사 체계

3.1 시간 기반 조건 검증 소스 코드 분석

PX4 소스 코드 계층(/src/modules/commander/Commander.cpp 및 관련된 상태 머신 루틴)을 분석해 보면, 루프 내부에서 고주파수로 기체의 상태를 갱신하며 시간 조건을 검사한다. 시스템 시계( System Clock, hrt: High Resolution Timer)를 이용해 이륙 후 경과한 물리적 시간과 파라미터로 명시된 설정 시간을 상시 비교하는 회로가 작동한다.

  • 조건 검사식 예시 개요:
    타이머 동작은 특정 기준점(armed_time 또는 landed_time)에서 현재 시간(hrt_absolute_time())을 뺀 값이 설정 파라미터(e.g., COM_DISARM_LAND * 1000000)를 초과하는지 여부를 검사하는 일련의 논리식으로 구현된다.
/* 유사 코드 형태의 논리 표현식 (PX4 Commander.cpp 기준) */
const hrt_abstime time_now = hrt_absolute_time();
if (vehicle_status.arming_state == vehicle_status_s::ARMING_STATE_ARMED) {
    if (land_detector.landed) {
        // 착륙 이후 COM_DISARM_LAND 대기 조건 만료 검사
        if ((time_now - time_when_landed) > (COM_DISARM_LAND * 1000000ULL)) {
            // 조건 만료시 시동 해제 명령 하달
            arm_auth_check();
            disarm_vehicle();
        }
    } else if (!vehicle_status.in_air) {
        // 이륙 전 COM_DISARM_PRFLT 대기 조건 만료 검사
        if ((time_now - time_when_armed) > (COM_DISARM_PRFLT * 1000000ULL)) {
            disarm_vehicle();
        }
    }
}

3.2 상태 전이 및 로직 다이어그램 (Mermaid)

자동 시동 해제의 조건 분기 및 타이머 스케줄링 흐름을 아래 모델을 통해 명확히 이해할 수 있다.

stateDiagram-v2
    [*] --> Standby: 전원 인가 및 부팅 완료
    Standby --> Armed: 조종자 시동 명령 입력 (MAVLink or RC)
    
    state Armed {
        [*] --> Grounded_PreFlight
        
        Grounded_PreFlight --> Airborne: 기체 이륙 조건 달성 및 In-Air 감지
        Grounded_PreFlight --> Disarming_Process: COM_DISARM_PRFLT 시간 초과
        
        Airborne --> Grounded_Landed: 지면 도달 및 Landed 감지 완료
        
        Grounded_Landed --> Airborne: 재이륙 트리거 (바운싱 또는 재이륙 명령)
        Grounded_Landed --> Disarming_Process: COM_DISARM_LAND 시간 초과
    }
    
    Disarming_Process --> Disarmed: 연산 변수 초기화 및 동력 차단 완료
    Disarmed --> Standby: 상태 초기화 완료

타이머 동작 동안 조종기 스로틀 입력이 기준 스레숄드(Threshold) 이상으로 증가하거나 활성 제어 커맨드가 접수되면 타이머는 리셋(Reset)되어 의도적인 비행의 연속성을 확보하게 된다.

4. Ardupilot 대비 자동 시동 해제 판정 메커니즘의 차이

PX4-Autopilot과 Ardupilot 모두 조종자 보호를 위한 자동 시동 해제 기능을 지원하지만, 파라미터 체계와 판정 접근 방식에서 차이가 발생한다.

  • 파라미터 체계의 직관성 분리: Ardupilot의 경우 DISARM_DELAY 단일 또는 연관된 계열 파라미터를 사용하여 시동 후 및 착륙 후 자동 해제 대기시간을 조작하는 경향이 강한 반면, PX4는 COM_DISARM_PRFLT(이륙 전)와 COM_DISARM_LAND(비행 후 착륙) 파라미터로 상황을 명확히 분리하여, 운영자가 사전(Pre) 비행과 사후(Post) 비행의 안전 철학을 독립적으로 설계하도록 유도한다.
  • 지상 감시(Land Detector) 의존도 차이: PX4는 상태 머신 기반의 엄격한 유한 상태 기계(FSM) 원형을 따라, 착륙 감지기(Land Detector)가 발행하는 vehicle_land_detected uORB 토픽의 불리언(Boolean) 값 의존도가 상대적으로 높다. 따라서 센서 노이즈가 높은 환경 하에서 바운싱 효과가 발생할 때 타이머가 초기화되는 조건이 매우 보수적으로 설계되어 있다.

5. MAVLink 제어 및 QGroundControl (QGC) 통신 연동 아키텍처

기체 내 시스템 컴포넌트 간에는 uORB 메시지로 시동 해제 상태가 전파되나, 최종적으로 사용자와 상호작용하기 위해서는 MAVLink 패킷으로 직렬화(Serialization)되어 QGroundControl 등의 지상 관제 시스템에 도달해야 한다.

5.1 HEARTBEAT 메시지를 통한 상태 브로드캐스팅

시동 해제 트리거가 만료되어 disarm_vehicle() 루틴이 활성화되면, MAVLink 통신 모듈은 기체 상태의 변화를 HEARTBEAT (Message ID: 0) 패킷의 base_mode 마스킹을 통해 송신한다. base_mode 필드의 MAV_MODE_FLAG_SAFETY_ARMED 비트가 0으로 전환되면 GCS는 기체의 시동이 해제되었음을 판단한다. 더하여, 구체적인 상태 변화 알림을 위해 STATUSTEXT 메시지가 전송되어 사용자에게 텍스트 경고창(예: “Auto-disarmed by terrain timeout”)을 전시한다.

5.2 QGroundControl에서의 처리 소스 구조 시각화

QGC 애플리케이션 프레임워크 내에서 기체의 무장 상태(Arming state)를 관리하는 C++ 백엔드 모듈은 Vehicle 객체(Vehicle.cc / Vehicle.h) 내부의 _armed 프라퍼티(Property)로 연결되어 있다. 이 상태가 변동할 때마다 Qt 프레임워크의 시그널-슬롯(Signal-Slot) 메커니즘을 통해 QML 프론트엔드 UI 컴포넌트로 이벤트가 방출(Emit)된다.

// QGroundControl (v4.3) MAVLink Heartbeat 파싱 (예시 구조)
void Vehicle::_handleHeartbeat(const mavlink_message_t& message)
{
    mavlink_heartbeat_t heartbeat;
    mavlink_msg_heartbeat_decode(&message, &heartbeat);

    bool newArmed = heartbeat.base_mode & MAV_MODE_FLAG_SAFETY_ARMED;
    if (_armed != newArmed) {
        _armed = newArmed;
        emit armedChanged(_armed);
        // 상태 전이에 따른 음성 피드백 로직 트리거
        if (!_armed) {
            qgcApp()->toolbox()->audioOutput()->say(tr("Disarmed"));
        }
    }
}

이와 같은 연동은 하드웨어와 GCS 사이의 강력한 논리적 계층 구조를 보여주며, 자동 시동 해제 발생 시 운영자가 실시간으로 기체 통제 권한의 회수 여부를 인지하도록 보장한다.

6. ROS2 기반 관제 연동 아키텍처 (uXRCE-DDS)

로보틱스 어플리케이션 환경을 위해 PX4-Autopilot을 ROS2와 연동(uXRCE-DDS 사용)할 때, 외부 Companion Computer는 VehicleStatus 토픽을 서브스크라이빙(Subscribing)하여 자동 시동 해제 이벤트를 능동적으로 관측 및 대응할 수 있다.

군집 비행이나 완전 자율화 로직에서는 시동이 예상치 못하게 해제될 경우, 상위 레이어의 경로 계획기(Path Planner)가 무한 루프나 오류 상태(Fault State)에 빠지지 않도록 빠른 상태 인식이 중요하다.

  • ROS2 노드 측 구현 예시: VehicleStatus의 arming_state 필드가 ARMING_STATE_DISARMED로 전환되었으나 임무 스케줄(Mission Schedule)이 남아있을 경우, 이는 에지 케이스(Edge Case)이므로 복구 로직이 호출되거나 GCS망을 통해 비상 에러 코드를 보고해야 한다.

이를 통해 자동 시동 해제 타이머 안전장치가 단일 기체의 하위 레벨 안전뿐만 아니라 복합 다중 로봇 관제 환경(Multi-agent Control Environment)에서도 상위 응용 계층의 신뢰도를 높여주는 중요한 관제 접점이 된다.

7. 결론

조종자 실수 방지 및 시스템 안전성을 위한 자동 시동 해제 (Auto-Disarm) 조건 검사 메커니즘은 표면적으로 단순한 타이머에 불과해 보일 수 있으나, 시스템적 관점에서 보면 (1) 비행 계획과 센서 피드백, (2) 조종자의 행위 연속성 이탈, (3) 물리적 로터 제어 회로의 독립적 보장 이라는 세 가지의 정교한 관제 조건이 융합된 안전장치이다. 개발자 및 오퍼레이터는 PX4의 해당 타이머 파라미터를 운용 시나리오별로 세밀히 조정하고, uORB와 MAVLink 간의 상태 동기화 기법을 완벽히 이해해야 잠재적인 자동화 사고를 방어할 수 있다.
