62.4.3 다항식 궤적 생성기(Polynomial Trajectory Generator) 연동: TrajectorySetpoint 메시지 스케줄링

1. 개요 (Overview)

PX4-Autopilot(Firmware v1.14 기준)의 오프보드(Offboard) 제어 모드에서 자율 에이전트의 부드럽고 연속적인 비행을 구현하기 위해서는, 단순한 목표 지점(Waypoint) 하달을 넘어 시간에 따른 위치, 속도, 가속도의 프로파일을 정의하는 궤적 생성기(Trajectory Generator)의 연동이 필수적이다. 특히 다항식(Polynomial) 기반의 궤적 생성 알고리즘은 로보틱스 분야에서 저크(Jerk)를 최소화하여 모터와 구조물에 가해지는 물리적 부하를 줄이는 데 널리 활용된다. 본 절에서는 ROS 2 환경에서 생성된 다항식 궤적 데이터를 PX4의 TrajectorySetpoint uORB 메시지로 변환하여 스케줄링하고 퍼블리싱(Publishing)하는 아키텍처와 구현 방법을 심층적으로 분석한다.

2. 다항식 궤적 생성의 수학적 배경 (Mathematical Background of Polynomial Trajectory)

다항식 궤적 생성은 시작점과 목표점 사이의 이동을 기구학적 구속 조건(운동학적 제약, Kinematic Constraints) 내에서 최적화하는 기법이다. 통상적으로 5차 다항식(Quintic Polynomial) 모델을 가장 널리 사용하며, 이는 위치, 속도, 가속도의 시작 조건 및 종결 조건을 모두 원활히 만족시킬 수 있기 때문이다.

경로 계획 알고리즘의 최적화 기법에 대한 상세한 수학적 전개는 원형 논문인 “Minimum Snap Trajectory Generation and Control for Quadrotors”(Mellinger and Kumar, 2011)를 참조하라.

3. TrajectorySetpoint 메시지 구조 및 역할 (Structure and Role of TrajectorySetpoint Message)

PX4 비행 제어기(Flight Controller)의 위치 제어 모듈(Position Controller Module)은 입력된 목표 궤적을 추종한다. ROS 2 인터페이스(uXRCE-DDS)를 통해 외부(Offboard)에서 목표 궤적을 주입할 때 사용되는 핵심 메시지가 TrajectorySetpoint이다.

4. 아키텍처 파이프라인 및 메시지 스케줄링 (Architecture Pipeline and Message Scheduling)

부드러운 비행 제어를 보장하려면 다항식 궤적 생성기가 지정된 주기(일반적으로 50Hz ~ 100Hz)에 맞추어 TrajectorySetpoint 메시지를 일정하게 퍼블리싱해야 한다.

4.1 타임스탬프 스케줄링 (Timestamp Scheduling)

메시지를 스케줄링할 때 가장 중요한 요소는 시계열의 무결성이다. 생성된 궤적이 FCU 측의 제어 루프와 정확히 맞물리도록 하려면, ROS 2의 시스템 시간(System Time)과 하드웨어 추상화 계층(Hardware Abstraction Layer)에서 생성되는 PX4 기반 타임스탬프를 동기화(Time Synchronization)해야 한다.

5. 구현 예시 (Implementation Example)

다음은 C++ 기반의 ROS 2 노드에서 5차 다항식 궤적의 한 점(Point)을 생성하여 퍼블리싱하는 간략한 소스 코드 구조이다.

#include <px4_msgs/msg/trajectory_setpoint.hpp>
#include <rclcpp/rclcpp.hpp>
#include <cmath>

class TrajectoryPublisher : public rclcpp::Node {
public:
    TrajectoryPublisher() : Node("trajectory_publisher"), time_sec_(0.0) {
        publisher_ = this->create_publisher<px4_msgs::msg::TrajectorySetpoint>(
            "/fmu/in/trajectory_setpoint", 10);
        
        // 50Hz 타이머 콜백 스케줄링
        timer_ = this->create_wall_timer(
            std::chrono::milliseconds(20),
            std::bind(&TrajectoryPublisher::publish_setpoint, this));
    }

private:
    void publish_setpoint() {
        px4_msgs::msg::TrajectorySetpoint msg{};
        msg.timestamp = this->get_clock()->now().nanoseconds() / 1000;
        
        // 시간 기반 다항식 궤적 산출 (예: 반경 2m의 원 궤적)
        double radius = 2.0;
        double omega = 0.5; // 각속도
        
        msg.position[0] = radius * std::cos(omega * time_sec_); // X
        msg.position[1] = radius * std::sin(omega * time_sec_); // Y
        msg.position[2] = -5.0; // Z (5미터 이륙 고도 유지, 평면 추종)
        
        msg.velocity[0] = -radius * omega * std::sin(omega * time_sec_);
        msg.velocity[1] = radius * omega * std::cos(omega * time_sec_);
        msg.velocity[2] = 0.0;
        
        msg.yaw = std::atan2(msg.velocity[1], msg.velocity[0]);
        
        publisher_->publish(msg);
        time_sec_ += 0.02; // 20ms 증가
    }

    rclcpp::Publisher<px4_msgs::msg::TrajectorySetpoint>::SharedPtr publisher_;
    rclcpp::TimerBase::SharedPtr timer_;
    double time_sec_;
};

이와 같은 궤적 추종 루틴이 안정적으로 실행되어야 제어기가 연속적인 오차 연산(Error Computation)을 수행하고, 다중 플랫폼 간의 관제 시스템(QGroundControl v4.3 이상 권장)에서도 궤적 편차가 최소화된 안정적 비행 시퀀스를 시각화할 수 있다.
