62.3.3.1. 비트 논리곱(Bitwise AND) 마스킹 연산을 이용한 무장(Arming) 및 모드(Mode) 플래그 단일화 추출

무인항공기(UAV)와 지상 관제 시스템(GCS) 혹은 컴패니언 컴퓨터 간의 통신은 무선 주파수(RF) 대역폭의 제약을 강하게 받는다. 따라서 PX4-Autopilot 및 MAVLink 프로토콜은 기체의 다양한 이산 상태(조종기 연결 상태, 무장 상태, 커스텀 비행 모드 등)를 여러 개의 1바이트(Byte) 독립 변수로 전송하는 대신, 하나의 정수형 변수 내 각 비트(Bit)에 상태 유무(1 또는 0)를 매핑하는 비트필드(Bitfield) 방식을 널리 채택하고 있다.

본 섹션에서는 수신된 비트필드 데이터에서 비트 논리곱(Bitwise AND) 연산과 비트 마스킹(Masking) 기법을 활용하여 지상 관제 및 자율 비행 로직에 필요한 단일 플래그(예: Arming 상태)를 정밀하게 추출해내는 아키텍처에 대해 심층 분석한다.

1. 비트필드(Bitfield)와 플래그 마스크(Flag Mask)의 기본 구조

정보 통신의 효율을 높이기 위해, MAVLink의 HEARTBEAT 메시지에 포함된 base_mode 필드나 시스템의 특정 센서 헬스(Health) 플래그 집합은 8비트 혹은 32비트 정수형으로 인코딩된다. 예를 들어, base_mode는 8개의 비트로 구성되어 있으며 각각의 자릿수가 서로 다른 의미를 내포한다.

  • 128 (0x80, 0b10000000): 무장(ARMED) 상채. 기체 모터가 회전할 수 있음을 나타냄.
  • 64 (0x40, 0b01000000): 수동 개입(MANUAL) 가능성.
  • 32 (0x20, 0b00100000): 하드웨어 인더 루프(HIL) 시뮬레이션 적용 여부.
  • 16 (0x10, 0b00010000): 자율 비행(AUTO) 모드 활성화.

이렇게 중첩된 변수를 if (base_mode == 128) 형태로 평가하면, 다른 플래그(예: 수동 가능성이 켜져 있어 총합이 192가 된 경우)가 함께 활성화되어 있을 때 원하는 무장 상태를 정상적으로 판별할 수 없다. 이를 극복하기 위해 비트 마스킹 기법이 강제된다.

2. 비트 논리곱(Bitwise AND) 추출의 수학적 원리

C/C++를 비롯한 대부분의 프로그래밍 언어에서 비트 논리곱 연산자(&)는 두 이진수의 각 자리(Bit)를 비교하여 두 비트가 모두 참(1)일 때만 결과 비트를 1로 설정한다.

예를 들어, 현재 기체가 무장(Armed) 상태이면서 동시에 자율 비행(Auto) 모드에 진입해 있다고 가정하자. 기체의 상태 변수 값은 128 + 16 = 144 (0b10010000)이다.
우리가 원하는 “무장 상태” 단일 마스크는 128 (0b10000000)이므로 연산식은 다음과 같다.

    0b10010000 (수신된 현재 상태값)
&   0b10000000 (무장 검증용 마스크: MAV_MODE_FLAG_SAFETY_ARMED)
--------------------------------------
    0b10000000 (결과값 = 128)

결과값이 0이 아니라는 점(> 0)을 통해 현재 무장 상태가 명확하게 true임을 컴퓨터에 인지시킬 수 있다.

3. 마스킹 아키텍처 파이프라인 (Mermaid 설계도)

수신된 통합 상태 데이터 스트림에서 비트 연산을 통하여 개별적인 논리(Boolean) 액터(Actor)로 파생하는 파이프라인의 구조는 다음과 같다. MAVLink 데이터나 uORB 비트 열거형 데이터를 수신한 자율 제어 노드는 필드 파서(Parser)를 바탕으로 조건부 제어 블록으로 신호를 분주(Demultiplexing)한다.

4. C++ 기반 추출 구현 예시 (MAVLink / ROS 2 융합 환경)

GCS용 백엔드 애플리케이션이나 ROS 2 오프보드 노드 내부에서, 수신된 1바이트 압축 모드 상태를 개별 bool 변수로 단일화하여 구조화(Struct)하는 C++ 코드를 설계할 수 있다.

#include <iostream>
#include <cstdint>

// MAVLink 모드 플래그 상수 (헤더 파일에 기정의된 매크로를 가정)
constexpr uint8_t MAV_MODE_FLAG_SAFETY_ARMED = 128; // 0x80
constexpr uint8_t MAV_MODE_FLAG_MANUAL_INPUT_ENABLED = 64; // 0x40
constexpr uint8_t MAV_MODE_FLAG_AUTO_ENABLED = 16; // 0x10

// 시스템 상태를 보관하는 추상화 구조체
struct ExtractedVehicleState {
    bool is_armed;
    bool is_manual_allowed;
    bool is_auto_mission;
};

// 원시 플래그 변수에서 개별 상태를 파싱하는 함수
ExtractedVehicleState parse_vehicle_base_mode(uint8_t raw_base_mode) {
    ExtractedVehicleState state;

    // Bitwise AND 연산 및 단일화 평가
    state.is_armed = (raw_base_mode & MAV_MODE_FLAG_SAFETY_ARMED) != 0;
    state.is_manual_allowed = (raw_base_mode & MAV_MODE_FLAG_MANUAL_INPUT_ENABLED) != 0;
    state.is_auto_mission = (raw_base_mode & MAV_MODE_FLAG_AUTO_ENABLED) != 0;

    return state;
}

void evaluation_example(uint8_t received_data) {
    ExtractedVehicleState current_state = parse_vehicle_base_mode(received_data);

    if (current_state.is_armed && current_state.is_auto_mission) {
        std::cout << "[System] 기체가 무장되었으며 자율 임무 수행 중입니다." << std::endl;
        /* 해당 상태에 특화된 PID 제어 혹은 경로 추종 알고리즘 호출 */
    } else {
        std::cout << "[System] 상태 대기 중이거나 관리자 통제권 하에 있습니다." << std::endl;
    }
}

이와 같은 마스킹 연산 레이어는 원시(Raw) 데이터를 시스템 개발자가 읽기 쉬운 부울(Boolean) 대수로 변환하여 인지 부하를 덜어주는 역할을 한다. 더불어 메모리 상에서 단일 비트 명령 연산으로 처리되므로 딜레이가 발생하지 않으며, 임베디드 코어의 자원을 보호하는 아키텍처 최적화의 핵심 기반이 된다.
