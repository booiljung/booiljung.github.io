62.4.2 제어 권한 독점 및 유지를 위한 OffboardControlMode 메시지의 논리적 무결성 주입

자율 에이전트(Autonomous Agent)가 PX4-Autopilot 기반 기체의 제어 권한을 외부(Offboard)에서 독점적으로 유지하기 위해서는, 기체 내부의 다중 제어 루프(위치, 속도, 자세 등) 중 어떠한 계층에명령을 내릴 것인지 명확히 선언해야 한다. PX4는 이러한 제어 권한과 의도를 관리하기 위해 OffboardControlMode라는 핵심 uORB 메시지를 사용한다. 이 메시지는 비행 제어기(FCU) 내비게이션 스택에 논리적인 제어 형태를 주입하여, 충돌하는 다중 명령 채널 간에 무결성을 보장하는 역할을 수행한다.

1. OffboardControlMode 메시지의 아키텍처 역할

오프보드 모델에서는 외부 애플리케이션(예: ROS 2 탑재 컴패니언 컴퓨터)이 비행 제어기를 완전히 신뢰하고 하위 제어기(PID 제어기 등)를 구동할 수 있도록 제어 루프를 직접 지정한다. OffboardControlMode 토픽은 단순히 “오프보드 모드“라는 추상화된 상태를 유지하는 것을 넘어, 구체적으로 어떤 물리량(Physical Quantity)을 제어할 것인지를 비트 단위(Boolean Array)의 플래그로 활성화(Enable)하는 메시지이다.

ArduPilot의 GUIDED 모드에서는 MAVLink 메시지(예: SET_POSITION_TARGET_LOCAL_NED) 내부에 포함된 16비트 비트마스크(Bitmask)인 type_mask 필드를 사용하여 무시할 필드를 지정(Ignore Type)하는 네거티브(Negative) 마스킹 방식을 사용한다. 반면, PX4의 OffboardControlMode는 각각의 제어 목표를 개별적인 불리언(Boolean) 변수로 할당하여 직접적으로 포지티브(Positive) 활성화를 명시한다는 구조적 차이가 있다.

2. 논리적 무결성 주입 매커니즘

외부에서 제어 권한을 획득하고 이를 유지하기 위해서는, OffboardControlMode 메시지 내부의 트리거들을 상호 배타적이고 논리적으로 모순되지 않게 설정(무결성 주입)하여야 한다.

2.1 메시지 내부 구조 (Boolean Payload)

ROS 2 (Micro XRCE-DDS) 환경에서 px4_msgs::msg::OffboardControlMode의 필드 구성은 다음과 같다.

제어 스택 플래그 (Boolean)설명 (Description)필요 하위 설정치 (Setpoint Topic)
position3차원 위치(Position) 제어 루프 활성화 유무TrajectorySetpoint
velocity3차원 속도(Velocity) 제어 루프 활성화 유무TrajectorySetpoint
acceleration3차원 가속도(Acceleration) 제어 루프 활성화 유무TrajectorySetpoint
attitude3축 자세(Roll/Pitch/Yaw) 제어 루프 활성화 유무VehicleAttitudeSetpoint
body_rate3축 각속도(Body Rate) 제어 루프 활성화 유무VehicleRatesSetpoint
direct_actuator모터(Actuator) 출력 PWM 직접 제어 유무ActuatorMotors / ActuatorServos

외부 노드(Node)가 위치 제어를 수행하려 할 때, position과 attitude를 동시에 true로 설정하면 상호 참조 오류가 발생하여 비행 제어기는 내부 캐스케이드(Cascade) 제어 구조의 계층 위반으로 이를 기각할 수 있다. 따라서 논리적 무결성을 유지하기 위해서는 목표로 하는 최상위 계층 하나만을 활성화하거나, 동기화가 보장된 하위 계층(예: 위치와 속도의 융합)에 대해서만 전략적으로 다중 활성화를 주입해야 한다.

3. 타임스탬프(Timestamp) 동기화 및 심박수(Heartbeat) 대체

명령 할당기(Command Allocator)에서 OffboardControlMode 메시지를 생성할 때 가장 필수적인 필드는 논리 플래그뿐 아니라 timestamp이다.

  • timestamp: PX4 시스템 클럭 기준의 마이크로초(Microsecond) 타임스탬프. ROS 2 환경에서는 px4_ros_com과 px-micro-dds-agent를 통해 연동되며 기체 타임스탬프와 미들웨어 간의 동기화가 이루어진다.

이 메시지 자체가 PX4 오프보드 타이머(Watchdog Timer)를 갱신시키는 “생명 유지(Keep-alive)” 신호, 즉 심박동(Heartbeat) 역할을 겸한다. 일반적인 MAVLink 관제 환경에서의 Heartbeat와 달리, 오프보드 제어 상태에서는 이 메시지가 **2Hz 이상(일반적으로 50Hz)**의 빈도로 입력되지 않는다면, 즉시 제어 권한 무결성이 상실된 것으로 간주되어 페일세이프(Failsafe) 로직으로 강제 전환된다.

4. 결론

독점적인 오프보드 비행 제어를 확보하기 위해서는, 제어의 의도(Intention)를 단일화하여 OffboardControlMode 객체에 논리적 모순점 없이 주입해야 한다. 이는 ArduPilot의 마스크 비트 연산 방식에 비해 보다 직관적이고 철저한 검증을 가능하게 하며, 외부 관제 시스템이나 엣지 컴퓨팅(Edge Computing) 보드가 비행 제어기의 내비게이션 아키텍처에 안정적으로 개입할 수 있게 해주는 필수 인프라스트럭처이다.
