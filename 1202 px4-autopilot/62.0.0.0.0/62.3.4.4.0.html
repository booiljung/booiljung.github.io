62.3.4.4 사원수(Quaternion) 대수계를 활용한 강체(Rigid Body) 회전 상태 표현 및 켤레 사원수(Conjugate Quaternion)를 이용한 역변환 연산

PX4-Autopilot과 같은 최신 무인항공기(UAV) 비행 제어 시스템에서는 기체의 3차원 자세(Attitude)를 표현하기 위해 짐벌락(Gimbal Lock) 현상과 삼각함수 연산 부하라는 치명적인 단점을 지닌 오일러 각(Euler Angles) 대신, 사원수(Quaternion) 대수계를 표준 강체(Rigid Body) 회전 상태 표현 규격으로 일괄 채택하고 있다. 본 절에서는 센서 데이터(VehicleOdometry 등) 동기화 과정에서 빈번히 활용되는 사원수의 수학적 특성과, 켤레 사원수(Conjugate Quaternion)를 이용한 초고속 역변환(Inverse Transformation) 연산 과정에 대하여 상세히 고찰한다.

1. 사원수(Quaternion) 대수계를 활용한 회전 표현

사원수는 스칼라(Scalar) 성분 하나와 3차원 벡터(Vector) 성분으로 이루어진 4차원 복소수 확장 개념이다. PX4의 내부 아키텍처 및 ROS2 nav_msgs/Odometry 메시지 내에서 사원수 q는 주로 다음과 같이 대수적으로 정의된다.

q = q_0 + q_1 \mathbf{i} + q_2 \mathbf{j} + q_3 \mathbf{k} \quad (\text{또는 } q = [q_w, q_x, q_y, q_z]^T)

강체가 임의의 기준 좌표계로부터 단위 벡터 \mathbf{u} = (u_x, u_y, u_z)를 회전의 기준 축으로 하여 \theta 각도만큼 회전한 상태(Rotation State)를 나타낼 때, 이를 표현하는 단위 사원수(Unit Quaternion)는 다음과 같이 공식화된다.

q_w = \cos\left(\frac{\theta}{2}\right)
\mathbf{q}_v = [q_x, q_y, q_z]^T = \mathbf{u} \sin\left(\frac{\theta}{2}\right)

이러한 단위 사원수는 스칼라와 벡터의 결합 연산을 통해 기체의 연속적인 회전을 전통적인 3 \times 3 회전 행렬 곱셈보다 훨씬 적은 컴퓨팅 연산량으로 처리할 수 있게 해주며, 오일러 각과 같이 물리적인 특정 각도(예: \pm 90^\circ의 피치)에서 자유도(Degree of Freedom)를 상실하는 짐벌락(Gimbal Lock) 문제로부터 비행 시스템을 완전히 해방시킨다.

2. 켤레 사원수(Conjugate Quaternion)와 역변환 연산

동반 컴퓨터(ROS2)와 비행 제어기(PX4) 간에 정밀한 실시간 타임스탬프가 연계된 VehicleOdometry 데이터를 교환할 때, 좌표계 프레임간(NED \leftrightarrow ENU) 혹은 동적 센서 프레임 간의 역방향 기준 회전 변환이 필수적으로 수반된다.

사원수의 가장 강력한 수학적 장점 중 하나는 기체 회전의 역변환(Inverse Rotation)을 구할 때, 역행렬(Inverse Matrix)을 구하기 위한 복잡한 나눗셈 및 행렬식 계산을 진행할 필요 없이 단순히 벡터 성분의 부호만 반대로 뒤집는 켤레 사원수(Conjugate Quaternion, q^* ) 를 취하는 것만으로 계산이 완결된다는 점이다.

q^* = q_0 - q_1 \mathbf{i} - q_2 \mathbf{j} - q_3 \mathbf{k} \quad (\text{또는 } q^* = [q_w, -q_x, -q_y, -q_z]^T)

회전 연산자로서 임의의 3차원 공간 벡터 v를 사원수 q로 정방향 회전시킨 결과를 v'라고 하면, 이 회전은 대수적으로 v' = q \otimes v \otimes q^* 와 같이 표현된다. (여기서 \otimes는 사원수의 비가환적 곱셈 연산자이다.)
반대로 획득한 회전된 프레임 벡터 v'를 원래의 베이스 기준계로 복구(역변환)하고자 할 때에는, 기존 회전 연산에 쓰인 단위 사원수의 켤레 사원수 q^*를 적용하여 v = q^* \otimes v' \otimes (q^*)^* 즉, 앞뒤를 반전시킨 v = q^* \otimes v' \otimes q 의 형태로 대입하여 곱셈을 전개하기만 하면 완벽한 좌표 역변환이 이루어진다.

3. 회전 연산 파이프라인 아키텍처 (Mermaid)

다음은 PX4 EKF2 추정기 내부 로직에서 외부 ROS2 노드로 강체 회전 상태가 전달될 때, 사원수와 켤레 사원수를 활용하여 데이터 프레임이 어떻게 상호 변환되는지를 입체적으로 보여주는 연산 흐름도이다.

graph TD
    A[PX4 EKF2 / IMU Sensor<br/>Raw Quaternion <br/> q = qw, qx, qy, qz] -->|Transmits q| B(Frame Transformer Node)
    
    B -->|Need Inverse Rotation| C[Calculate Conjugate<br/>q* = qw, -qx, -qy, -qz]
    C -->|Construct Rotation| D[Quaternion Multiplication<br/>v_original = q* x v_rotated x q]
    
    D --> E{Reference Frame Shift<br/>e.g., NED to ENU}
    B -->|Direct Forward Rotation| F[Quaternion Multiplication<br/>v_rotated = q x v_original x q*]
    F --> E
    
    E -->|Publish Msg| G[ROS2 Control Stack<br/>Odometry Subsystem]

위의 시스템 파이프라인 다이어그램에서 통찰할 수 있듯이, 켤레 사원수를 활용한 역변환 연산 기법은 회전 상태 방정식의 연산 복잡도(Computational Complexity)를 비약적으로 낮추어 메모리 및 CPU 자원의 소모를 절감한다. 이는 다양한 센서 정보를 결합하는 통합 센서 퓨전 과정에서 마이크로초 단위의 엄격하고 타이트한 타이밍 제어를 요구하는 고주파 오도메트리 데이터 동기화 알고리즘에 고도로 최적화된 수학적 도구라 할 수 있다. 애플리케이션 개발자는 PX4 펌웨어의 mathlib 폴더 내장 라이브러리 또는 ROS2의 범용 tf2 라이브러리를 통하여 완전히 최적화된 켤레 사원수 기반 연산 루틴을 손쉽게 호출하여 사용할 수 있다.
