62.4.3.3 요(Yaw) 회전 각도 스칼라 및 회전 각속도 스칼라의 유효 범위 포화(Saturation) 방지 클리핑(Clipping) 알고리즘

본 절에서는 무인항공기(UAV)의 방향 제어에 핵심적인 요(Yaw) 회전 각도 스칼라(Scalar) 및 회전 각속도 스칼라 값의 유효 범위 포화(Saturation) 방지를 위한 클리핑(Clipping) 알고리즘에 대해 다룬다. 특히 PX4-Autopilot(펌웨어 버전 1.14 이상)과 ROS 2(Micro XRCE-DDS) 환경에서 TrajectorySetpoint 메시지를 퍼블리싱(Publishing)할 때 발생할 수 있는 각도 랩어라운드(Wrap-around) 문제와 제어기 포화 현상을 방지하는 수학적 기법 및 구현 방법을 심층적으로 분석한다.

1. 요(Yaw) 제어에서의 물리적 한계와 클리핑의 필요성

멀티로터 비행체의 요(Yaw) 제어는 기체의 헤딩(Heading) 방향을 결정하는 핵심 요소이다. 오프보드(Offboard) 환경에서 기체 외부의 에이전트(ROS 2 노드 등)가 목표 요 각도(Yaw Angle, \psi) 또는 요 각속도(Yaw Rate, \dot{\psi})를 하달할 때, 유효 범위를 벗어난 값이 입력되면 비행 제어기(Flight Controller) 내부의 PID(Proportional-Integral-Derivative) 제어기가 예기치 않은 동작을 수행할 수 있다.

  • 각도 랩어라운드(Angle Wrap-around): 요 각도는 통상적으로 -\pi \le \psi \le \pi 라디안(Radian) 또는 -180^\circ \le \psi \le 180^\circ 범위를 갖는다. 이 범위를 초과하는 목표 각도가 입력되면, 최단 경로가 아닌 반대 방향으로 회전하거나, 오차(Error) 값이 급증하여 적분기 와인드업(Integrator Windup) 현상이 발생할 수 있다.
  • 포화(Saturation): 모터의 물리적 회전 속도 한계로 인해, 기체가 도달할 수 있는 최대 요 각속도(Maximum Yaw Rate)는 제한된다. 한계치를 넘는 과도한 \dot{\psi} 값이 입력될 경우, 제어 입력이 시스템 한계에 도달하는 포화 현상이 발생하여 비행체의 자세 안정성을 심각하게 훼손할 수 있다.

따라서 전송 전에 입력 스칼라 값을 올바른 범위 내로 강제하는 **클리핑 알고리즘(Clipping Algorithm)**이 반드시 적용되어야 한다.

2. 클리핑 알고리즘의 수학적 원리 (Mathematical Principles)

2.1 요 각도(Yaw Angle) 정규화 (Normalization)

방위각 \psi가 -\pi와 \pi 사이의 유효 범위를 유지하도록 하는 정규화 함수 wrap\_pi(\psi)는 다음과 같이 정의된다.

wrap\_pi(\psi) = \psi - 2\pi \left\lfloor \frac{\psi + \pi}{2\pi} \right\rfloor

여기서 \lfloor \cdot \rfloor는 바닥 함수(Floor Function)를 의미한다. 이 연산을 통해 임의의 요 각도 입력은 항상 주치(Principal Value)인 [-\pi, \pi) 내로 매핑된다.

2.2 요 각속도(Yaw Rate) 클리핑 (Clipping)

스칼라 변수 x(여기서는 \dot{\psi})를 상한 상한값(Upper Bound) max와 하한값(Lower Bound) min 사이로 제한하는 클리핑 함수 clip(x, min, max)는 다음과 같다.

clip(x, min, max) = \max(min, \min(x, max))

만약 절대값 형태의 제어 한계 \vert \dot{\psi}_{max} \vert (수식 내 \vert 사용)가 주어졌다면, 클리핑 연산은 다음과 같이 간략화할 수 있다.

\dot{\psi}_{clipped} = \max(-\dot{\psi}_{max}, \min(\dot{\psi}_{target}, \dot{\psi}_{max}))

3. PX4 및 ROS 2 아키텍처에서의 데이터 흐름

외부 제어 컴퓨터(Companion Computer)에서 실행되는 ROS 2 노드는 TrajectorySetpoint 메시지에 목표 요 각도(yaw) 또는 요 각속도(yawspeed)를 채워 퍼블리싱한다. 이 값은 Micro XRCE-DDS 에이전트와 클라이언트를 거쳐 PX4의 uORB 토픽으로 전달되며, FlightModeManager를 통해 mc_att_control(멀티로터 자세 제어 모듈)로 라우팅된다.

4. ROS 2 C++ 구현 예제 (Implementation Example)

다음은 C++14 이상의 문법을 이용하여 ROS 2 제어 노드에서 px4_msgs::msg::TrajectorySetpoint 퍼블리싱 전, 요 각도 정규화 및 각속도 클리핑을 수행하는 코드 예시이다.

#include <cmath>
#include <algorithm>
#include <px4_msgs/msg/trajectory_setpoint.hpp>

class YawControlAllocator {
private:
    const float MAX_YAW_RATE = 1.0f; // 최대 요 각속도 (rad/s), 기체 사양에 맞게 조정

    // [-pi, pi] 범위 내로 요 각도 정규화 (랩어라운드 처리)
    float wrap_pi(float angle) {
        float wrapped = std::fmod(angle + M_PI, 2.0 * M_PI);
        if (wrapped < 0) {
            wrapped += 2.0 * M_PI;
        }
        return wrapped - M_PI;
    }

    // 값의 스칼라 범위 클리핑
    float clip(float value, float min_val, float max_val) {
        return std::max(min_val, std::min(value, max_val));
    }

public:
    void publish_trajectory_setpoint(float target_yaw, float target_yaw_rate) {
        px4_msgs::msg::TrajectorySetpoint msg{};
        
        // 1. 요(Yaw) 각도 랩어라운드 매핑
        msg.yaw = wrap_pi(target_yaw);
        
        // 2. 요(Yaw) 각속도 포화 방지 클리핑
        msg.yawspeed = clip(target_yaw_rate, -MAX_YAW_RATE, MAX_YAW_RATE);
        
        // 추가: X, Y, Z 위치 제어 변수 할당 (이 문서 범위 외)
        // ... 생략 ...

        // ROS 2 Publisher를 통한 메시지 전송
        // trajectory_setpoint_publisher_->publish(msg);
    }
};

위의 함수 wrap_pi는 부동소수점 오차(Floating-point Error)를 방지하면서도 O(1)의 시간 복잡도(Time Complexity)로 입력을 정규화한다. 이를 통해 급격한 스텝 입력(Step Input)이나 누적되는 센서 노이즈가 유입되더라도 제어 명령이 안전한 범위를 유지하도록 보장한다.

5. 결론 및 참고 문헌

요(Yaw) 축의 동작 범위 포화 방기 클리핑 및 정규화 체계는 단순한 데이터 전처리가 아닌 기체의 동역학적 한계(Kinematic Constraints)를 극복하기 위한 필수적인 이산 제어(Discrete Control) 과정이다. 안전한 오프보드 제어를 달성하기 위해서는 이러한 보호 메커니즘 메트릭이 발행자(Publisher) 노드에 견고하게 내장되어야 한다.
