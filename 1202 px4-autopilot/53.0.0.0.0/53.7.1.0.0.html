53.7.1 이기종 무선 네트워크(RF, LTE/5G 등) 간의 무중단 페일오버(Seamless Failover) 아키텍처

무인항공기(UAV) 운영 환경에서 데이터 링크의 신뢰성은 기체의 안전과 임무 완수에 직결된다. 단일 통신 링크에 의존하는 시스템은 전파 방해(Jamming), 지형적 장애물, 간섭 등으로 인해 연결이 단절될 치명적 위험(Single Point of Failure)을 안고 있다. 이를 극복하기 위해 최신 관제 시스템은 텔레메트리 무선 주파수(RF, Radio Frequency) 라디오와 이동통신망(LTE/5G) 등을 병행하여 사용하는 이기종 무선 네트워크(Heterogeneous Wireless Networks) 환경을 채택한다. 본 절에서는 PX4-Autopilot(v1.14 이상 기준)과 QGroundControl(v4.3 이상 기준) 환경에서 이기종 링크 간 무중단 페일오버(Seamless Failover)를 달성하기 위한 시스템 아키텍처와 제어 기법을 분석한다.

1. 이기종 통신 네트워크의 계층적 구조(Hierarchical Structure)

이기종 통신 네트워크 시스템은 물리적으로 다른 두 개 이상의 통신 매체를 논리적으로 단일 채널인 것처럼 추상화(Abstraction)하여 지상 관제 시스템(GCS)과 PX4 기체 시스템 간의 지속적인 MAVLink 데이터 스트림을 보장한다.

1.1 하드웨어 및 네트워크 링크 구성 요소

  1. RF 텔레메트리(Telemetry) 라디오:
  • 433MHz, 900MHz, 2.4GHz 등의 주파수 대역을 사용하는 점대점(Point-to-Point) 근거리/중거리 통신 링크이다.
  • 송수신 지연(Latency)이 짧고 예측 가능성이 높으나, 대역폭(Bandwidth)이 좁고(일반적으로 수십 kbps) 가시거리(LOS, Line of Sight)에 크게 제한을 받는다.
  1. 이동 통신망(LTE/5G):
  • 셀룰러 기지국을 통한 IP 기반 광역 네트워크(WAN) 링크이다.
  • 가시거리 외(BVLOS, Beyond Visual Line of Sight) 비행이 가능하고 대용량 비디오 스트리밍을 지원하지만, 기지국 핸드오버 지연, 트래픽 폭주 등에 따른 네트워크 지터(Jitter)와 지연 시간이 발생할 수 있다.

이와 같이 각기 다른 물리적 특성을 갖는 링크를 결합하여 하나가 단절되더라도 다른 링크를 통해 MAVLink 트래픽이 끊김 없이 라우팅되도록 설계해야 한다.

2. 무중단 페일오버 아키텍처 설계 (Architecture Design)

QGroundControl(GCS)과 PX4가 이종 네트워크 간의 페일오버를 수행하는 아키텍처는 MAVLink 라우팅 계층의 동적 관리(Dynamic Management)를 핵심 메커니즘으로 삼는다.

2.1 MAVLink 기반 다중 링크 관리 체계

QGroundControl은 백엔드 코어(C++)의 LinkManager 클래스와 Vehicle 인스턴스 내부의 링크 추론 논리를 통해 다수의 연결 채널을 동시에 관리한다. PX4 기체 역시 동반 컴퓨터(Companion Computer)에 설치된 MAVLink 라우터(MAVLink Router)를 통해 다중 물리 채널로 MAVLink 패킷을 중계(Relay)할 수 있다.

2.2 Ardupilot 대비 PX4 시스템의 차이점

PX4 아키텍처는 일반적으로 픽스호크 분산 보드 내에서의 MAVLink 통신 채널을 여러 포트(예: TELEM1, TELEM2)에 즉시 할당하고 동시에 독립적인 MAVLink 인스턴스를 실행하는 아키텍처(mavlink start 모듈 명령어)를 선호한다. 동반 컴퓨터를 도입할 때는 MAVLink Router나 Micro XRCE-DDS(ROS2 연동 시)를 활용하여 네트워크 믹싱(Mixing)을 수행한다.
Ardupilot 역시 여러 직렬 포트(Serial Port) 인터페이스를 통해 MAVLink 스트림을 독립적으로 뿜어내지만, PX4의 경우 mavlink 모듈에서 데이터 전송의 우선순위(Priority) 큐 체계를 UORB 토픽별로 세밀하게 관리할 수 있어 링크 대역폭에 따른 맞춤형 채널 구성(예: LTE는 비디오+고해상도 텔레메트리, RF는 필수 제어 패킷만 전송)을 구현하기 용이하다.

3. 상태 기반 페일오버 판별 알고리즘 (Failover Detection Algorithm)

이기종 링크 간 페일오버를 달성하려면, 현재 사용 중인 링크의 품질 저하 상태를 빠르게 판단하고 백업 링크로 송신 경로를 전환해야 한다. 단순히 통신이 끊어질 때까지 기다리는 하드 페일오버(Hard Failover)가 아닌, 사전 인지를 통한 무중단 페일오버(Seamless Failover / Soft Failover)를 위해서는 연결 품질 측정 메커니즘이 수반되어야 한다.

3.1 RSSI 및 패킷 손실률(Packet Loss Rate) 분석

통신 링크의 상태를 판단하는 주요 지표로는 송수신 신호 강도 표시(RSSI), 드롭된 MAVLink 패킷 수 추이를 활용한다. MAVLink 프로토콜의 RADIO_STATUS 메시지는 전용 RF 라디오 장치가 송출하는 하드웨어 레이어의 상태 정보를 포함하므로 적극적으로 모니터링할 수 있다.
LTE 채널의 경우 IP망 종단 간 핑(Ping) 딜레이나 MAVLink HEARTBEAT 메시지의 수신 주기를 기반으로 통신 활성화를 결정한다.

각 통신 채널의 우선순위를 부여할 수 있으며, MAVLink 통신 아키텍처에서는 수신된 패킷 번호(Seq)를 바탕으로 데이터 손실률(Dropped Packets)을 실시간 산출한다. 채널 손실 비용(Cost Function) 맵핑 시 다음 식을 참조할 수 있다.

C = w_1 \cdot L_{loss} + w_2 \cdot T_{delay} + w_3 \cdot \left( \frac{1}{\text{RSSI} - \text{Noise}} \right)

(상기 식에서 L_{loss}는 기간당 패킷 손실률, T_{delay}는 패킷 전송 지연 시간, w_x는 각 가중치를 나타낸다. 참고로 테이블에서 표현 시 \vert 기호를 사용해 척도를 구분한다.)

3.2 QGroundControl 소스 코드 내 동적 채널 결정 흐름

QGC는 여러 개의 LinkInterface 하위 클래스(예: SerialLink, UDPLink) 인스턴스로부터 들어오는 패킷을 단일 장치 객체(Vehicle 클래스 내 _addMAVLinkPacket 함수)에서 처리한다. GCS 내에서 특정 명령을 기체로 송신할 때 어떤 링크 인터페이스를 사용하여 송출(Vehicle::sendMessageOnLink())할 것인지에 대한 폴백(Fallback) 관리가 핵심이다.

우선순위 링크 관리 객체는 메인 링크가 타임아웃 됨과 동시에 예비 링크 배열에서 유효한 활성 연결을 찾아 송출 라우팅 주소를 전환한다. MAVLink 데이터 패킷은 사실상 상태 정보이므로 링크 변경으로 인해 일시적인 패킷 순서(Sequence) 역전 현상이 생길 수 있으나, MAVLink 파서는 최신 상태만 수리(Accept)하도록 설계되어 있어 무리 없는 페일오버를 지원한다.

4. 자율 에이전트 및 ROS2 연동 시 고려사항

ROS2 미들웨어를 도입할 경우, 마이크로(Micro) XRCE-DDS 기반 통신에서도 이기종 네트워크 구조를 고려해야 한다. px4_ros_com과 XRCE-DDS Agent 구성 시, UDP 기반 통신은 본질적으로 비연결성(Connectionless)이므로 통신 단절 시 하위 소켓 채널을 동적으로 재바인딩해주는 네트워크 매니저가 중간에 삽입되어야 완전한 무중단 서비스를 제공받을 수 있다.

따라서 동반 컴퓨터에 ZeroTier나 Tailscale과 같은 가상 사설망(VPN) 환경을 구축하고, IP 레이어에서 물리적인 LTE 모뎀 경로와 RF(Point-to-Point IP망 터널링 브릿지 등 활용)의 MPTCP(Multipath TCP) 혹은 멀티링크 중첩 구조를 네트워크 레벨에서 은닉시켜 구현하는 방식도 최신 자율 에이전트 시스템에서 적극 도입되는 기법이다. 이는 QGC나 ROS2 노드를 수정하지 않고 운영체제의 라우팅 테이블 레벨에서 무중단 페일오버 아키텍처를 달성할 수 있는 강점을 지닌다.

5. 요약 및 구현 시 주의사항

이기종 네트워크 기반 무중단 페일오버 시스템의 구현 요소는 다음과 같다.

  • 다중 인터페이스 활성화: PX4 내부에서 두 개 이상의 텔레메트리 포트(MAVLink 인스턴스)를 동시에 활성화하거나, 동반 컴퓨터의 라우팅 소프트웨어를 운용하라.
  • 비대칭 트래픽 제어: RF 같은 저대역폭 구간에는 MAVLink 속도(-r 옵션 설정)를 낮추고 필수 제어 메시지만 전송하며, 고대역폭 네트워크 구간에는 카메라 이미지, 포인트 클라우드 등 ROS2 토픽이나 비디오 스트리밍을 라우팅하도록 규칙을 제정하라.
  • 의사 결정 지연 최적화: 너무 짧은 확인 주기(Timeout)는 링크 플래핑(Link Flapping: 경로가 계속 바뀌는 현상)을 유발하며, 매우 긴 딜레이는 페일세이프(Failsafe)의 Return To Launch (RTL) 트리거 시점과 충돌을 야기할 수 있으므로 상황에 맞게 타임아웃 변수를 정교하게 튜닝해야 한다.

이기종 망 환경의 페일오버 설계는 물리 채널의 한계를 소프트웨어 공학 기술로 극복하려는 시도이며, 상용급 대형 무인기나 원격 관제가 불가피한 도심항공모빌리티(UAM) 실증의 핵심 요구사항임을 명심하라.
