53.7.1.1 다중 링크 품질 지표를 상태 변수로 갖는 마르코프 결정 과정(Markov Decision Process) 기반의 최적 라우팅 전환 정책

무인항공기(UAV)의 이기종 다중 통신 환경에서 단순한 임계값(Threshold) 기반의 링크 전환 로직은 빈번한 핑퐁(Ping-pong) 현상과 네트워크 지연을 초래할 수 있다. QGroundControl(GCS)과 PX4-Autopilot(v1.14 이상) 간의 향상된 통신 무중단 페일오버 처리를 위해서는 실시간 링크 파라미터를 기반으로 미래 상태를 예측하여 최적의 전환 시점을 결정하는 확률적(Probabilistic) 접근이 요구된다. 본 절에서는 다중 링크 품질 지표를 상태 변수로 정의하고 마르코프 결정 과정(MDP, Markov Decision Process)을 적용하여 링크 라우팅 전환 정책을 최적화하는 기법을 구조화하여 설명한다.

1. 마르코프 결정 과정(MDP) 적용 배경

기존 Ardupilot 기반 시스템이나 초기 GCS는 주로 MAVLink의 HEARTBEAT 손실 타임아웃이나 RSSI 단일 지표가 특정 기준치 아래로 떨어지면 즉시 백업 채널로 전환하는 휴리스틱 방식을 취했다. 반면 MDP 기반의 라우팅 전환 모델은 여러 개의 품질 지표(Packet Loss, RTT, Jitter, RSSI 등)의 동적 변화를 고려하여 보상(Reward)을 극대화(즉, 통신 지연 및 단절 최소화)하는 ’행동(Action)’을 맵핑한다.

이를 통해 다음과 같은 이점을 얻을 수 있다.

  • 예측 기반(Predictive) 전환: 완전한 통신 두절 전에 네트워크 열화를 인지하고 사전 전환 수행.
  • 불필요한 스위칭 오버헤드 방지: 스위칭(Switching Cost) 패널티를 부여하여 잦은 전환을 억제.

2. 상태 공간(State Space) 및 행동 공간(Action Space) 정의

MDP 모델은 M = \langle S, A, P, R, \gamma \rangle 투플(Tuple)로 정의된다.

2.1 상태 변수 집합 (S)

기체가 인지하는 MAVLink 링크와 전송 계층의 상태 지표를 활용하여 상태 공간 S를 구성한다. 특정 시점 t에서의 시스템 상태 s_t \in S는 아래와 같은 다중 링크 품질 지표 벡터로 표현된다.

상태 변수물리적 의미로깅 및 참조(uORB / MAVLink)비고
x_1RSSI 및 잡음비(SNR)RADIO_STATUS 메시지의 rssi, noise 필드물리 계층
x_2패킷 손실률(PLR)수신된 패킷 Sequence의 \vert N_{expected} - N_{received} \vert / N_{expected} 계산MAVLink 전송 계층
x_3왕복 지연 시간(RTT)TIMESYNC 메시지 \vert T_{rx} - T_{tx} \vert 연산 결과네트워크 혼잡도 지표
x_4현재 사용 중인 링크L_{RF} 또는 L_{LTE}행동 결정 패널티 요소

상태 공간의 복잡도를 줄이기 위해 센서 노드 데이터 분석 단계를 거쳐 K-Means 클러스터링 등으로 상태를 이산화(Discretization)하는 기법이 주로 활용된다.

2.2 행동 집합 (A)

에이전트(GCS 또는 PX4 라우터)가 특정 상태에서 취할 수 있는 라우팅 행동 a_t \in A는 다음과 같다.

  • a_1: 유지(Stay) - 현재 활성화된 데이터 링크(예: RF 텔레메트리)를 계속 사용한다.
  • a_2: 전환(Switch) - 예비 데이터 링크(예: LTE/5G 인터페이스)로 라우팅 포인트를 변경한다.

3. 전이 확률(Transition Probability) 및 보상 함수(Reward Function) 설계

3.1 상태 전이 확률 행렬 (P)

특정 상태 s에서 행동 a를 취했을 때 다음 상태 s'로 변화할 확률 함수 P(s' \vert s, a)는 비행 채널의 환경 모델링을 통해 결정된다. 실제 PX4-Autopilot의 로깅 파일(ULog)을 파싱하여, 특정 고도/위치에서 텔레메트리 RTT와 RSSI가 어떻게 변동하는지에 대한 과거 경험적 데이터(Empirical Data)를 기반으로 마르코프 전이 행렬을 추정한다.

3.2 보상 함수 설계 (R)

라우팅 전환 행동에 대한 보상 R(s, a)은 통신 성능 극대화 방형으로 설계된다.

R(s_t, a_t) = w_1 \cdot f_{T}(RTT) + w_2 \cdot f_{L}(PLR) - C_{switch} \cdot \delta(a_t)

  • f_T, f_L: RTT와 패킷 손실이 작을수록 높은 양수를 반환하는 유틸리티 함수
  • C_{switch}: 세션 재연결, 소켓 바인딩 등에 드는 스위칭 오버헤드 패널티 비용 상수
  • \delta(a_t): 행동이 ’전환(Switch)’일 경우 1, ’유지’일 경우 0을 반환하는 디랙 델타(Dirac Delta) 형태의 식

이 보상 함수를 통해 벨만 최적 방정식(Bellman Optimality Equation)을 풀고 각 상태별 행동 가치 함수(Q-Value)를 도출해 내는 정책 반복(Policy Iteration) 혹은 Q-러닝(Q-Learning)을 QGroundControl C++ 클래스에 플러그인(Plugin) 형태로 연동할 수 있다.

4. QGroundControl 및 ROS2 노드 구현 아키텍처

PX4 본체의 uORB 메시지 처리 부담을 줄이기 위해, MDP 기반 최적화 연산은 주로 동반 컴퓨터(Companion Computer)나 지상 관제 시스템(QGC) 단의 백엔드 쓰레드에서 수행된다.

    CC->>QGC: 중계 (via LTE & RF)

        QGC->>QGC: 3. MDP Policy 탐색 (Lookup Table / NN)
        alt Action == Switch
            QGC->>QGC: 4. 활성 소켓 교체 비용 적용 (Switching)

4.1 ROS2 환경에서의 오프로딩(Offloading)

ROS2 (Micro XRCE-DDS) 인프라를 활용할 경우, px4_ros_com 패키지 단에서 센서 데이터를 퍼블리시(Publish)하는 QoS(Quality of Service) 정책 자체에 이 MDP 논리를 연동할 수 있다.
예를 들어 Best Effort QoS 모델로 동작할 때, 상태 지표가 임계 수준의 열화 구간에 진입(MDP 기대 보상 급락)하면 ROS2 네비게이션 스택의 제어 명령이 누락되는 사태를 막기 위해 동반 컴퓨터 내부의 QoS 설정 관리자가 즉각적으로 데이터 파이프라인(Network Interface)을 전환하게 유도하라. 이는 고전적인 임계값 알고리즘 기반인 Ardupilot의 Link Management 방식을 뛰어넘는 자율 군집 드론망 제어공학의 핵심 선행 기법이다. (관련 논문: “Markov Decision Process-based Optimal Routing for Heterogeneous Unmanned Aerial Vehicle Networks” 등의 학술 문헌에서 주로 다루는 개념 적용임)

5. 요약

마르코프 결정 과정 기반의 통신 라우팅 정책은 PX4-Autopilot의 다중 연결성(Multi-connectivity) 한계를 동적으로 해결하는 인공지능형 관제 기술이다.

  • 통신 파라미터(RTT, RSSI, PLR)를 실시간 퓨전하여 단일 상태 변수로 정의하라.
  • 단순히 신호가 약해졌다고 통신 단절로 간주하지 말고, 전환(Switching) 이후의 보상 값을 예측하여 잦은 핑퐁 라우팅을 억제하라.
  • 이러한 고급 예외 처리 정책은 실시간 보장성이 필수인 ROS2 Offboard 자율 비행 관제에서 기체의 생존성과 직결된다는 점을 설계에 반드시 반영해야 한다.
