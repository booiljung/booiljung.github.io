53.2.2 동적 임계값(Dynamic Threshold) 산출을 위한 적응형 디지털 필터(Adaptive Digital Filter) 설계

1. 개요 및 필요성

PX4-Autopilot과 지상 관제 시스템(GCS, Ground Control Station) 간의 통신은 주로 MAVLink 프로토콜을 기반으로 구성된다. 운영 환경에서 무선 링크(예: 텔레메트리 라디오, Wi-Fi, LTE 등)를 통과하는 패킷은 네트워크 혼잡이나 물리적 간섭에 의해 전송 지연(Latency)과 패킷 손실(Packet Loss)이 불가피하게 발생한다. 일반적으로 통신 단절(Data Link Loss)을 감지하기 위해 고정된 타임아웃(Static Timeout) 값을 사용하지만, 이는 네트워크 지연이 일시적으로 커지는 지터(Jitter) 상황에서 페일세이프(Failsafe) 오작동을 유발할 수 있다.

이러한 문제를 해결하기 위해, QGroundControl(QGC) 등 GCS 측에서는 기체로부터 수신되는 하트비트(HEARTBEAT) 메시지의 도달 시간 간격(Inter-Arrival Time, IAT)을 동적으로 학습하고 평가해야 한다. 이에 따라 일시적 상태 변동과 영구적 통신 단절을 구별해 내는 동적 임계값(Dynamic Threshold) 산출 로직이 요구되며, 이를 위해 적응형 디지털 필터(Adaptive Digital Filter)를 설계하여 적용한다.

2. 하트비트 지연 모델과 적응형 필터 설계

하트비트가 수신되는 즉각적인 도달 시간 간격을 확률 변수 t_k라고 정의해 보자. 네트워크의 상태는 시간에 따라 서서히 변화하므로 통신 지연의 평균 기댓값 \mu와 편차 \sigma을 누적하여 추정해야 한다. 동적인 임계값 T_{dynamic}(k)는 일반적으로 다음과 같이 산출된다.

T_{dynamic}(k) = \hat{\mu}(k) + \alpha \cdot \hat{\sigma}(k)

여기서 \hat{\mu}(k)와 \hat{\sigma}(k)는 시간 k에서의 추정치이며, \alpha는 허용 오차 범위를 결정하는 가중치 팩터이다. 계산 자원의 한계를 지닌 임베디드 시스템 및 GCS 백엔드 스레드 아키텍처에서는 메모리를 적게 차지하는 무한 임펄스 응답(IIR, Infinite Impulse Response) 기반의 필터 구조가 주로 채택된다.

2.1. 적응형 칼만 필터(Adaptive Kalman Filter)의 활용

디지털 통신 신호 처리에 있어 단순한 평활화(Smoothing) 필터의 한계를 극복하기 위해, 상태 궤적의 불확실성을 고려하는 적응형 칼만 필터가 적용될 수 있다. 예컨대 관련 학술 논문인 “Adaptive Kalman Filtering for Time-Varying Network Delay Estimation in Unmanned Aerial Systems“의 방법론과 같이, 일시적인 스파이크(Spike)에 의해 임계값이 급격히 발산하는 것을 막고자 혁신(Innovation) 항의 가중치를 조절하는 논리를 취한다. 즉, 수신 간격이 평소 예측치를 심하게 벗어날 경우 해당 데이터의 신뢰도를 일시적으로 낮춰 측정 잡음 공분산(Measurement Noise Covariance)을 동적으로 갱신한다.

3. GCS 처리 파이프라인 아키텍처

다음 구조도는 GCS 환경에서 수신된 패킷 기록을 바탕으로 필터가 동작하여 실시간으로 통신 상태를 모니터링하는 논리를 보여준다.

flowchart TD
    A[MAVLink Receiver] -->|HEARTBEAT| B(Time-stamp Recorder)
    B -->|Calculate IAT| C{Spike Detection filter}
    
    C -->|Normal Latency| D[Adaptive Digital Filter]
    C -->|Abnormal Spike| E[Outlier Rejector / Clamping]
    
    D -->|Update Estimated μ, σ| F(Dynamic Threshold Calculator)
    E --> F
    
    F -->|Threshold T_d| G[Link Status Monitor]
    G -->|IAT > T_d| H[Trigger Link Loss FSM Event]

이 흐름에 따르면, 지시된 임계값을 초과하는 간격이 발생할 때만 통신 단절 유한 상태 기계(FSM, Finite State Machine)를 동작시켜 신뢰도 높은 예외 처리(Exception Handling)를 달성한다.

4. PX4-Autopilot 및 Ardupilot의 차이

기존의 Ardupilot 기반 시스템이나 Mission Planner의 경우 파라미터화된 강력한 정적 타임아웃 설정에 주로 의존한다. 시스템의 예측 가능성을 높이는 데에는 유리하지만, 복잡한 통신 인터페이스를 지닌 다중 에이전트 시스템(Swarm) 환경이나 LTE/5G 같은 셀룰러망 도입 시 불필요한 Return to Launch(RTL)의 잦은 유발원이 될 수 있다.

반면, 현대 관제망과 활발히 결합되는 PX4-Autopilot 및 QGroundControl 환경은 보다 유연한 아키텍처를 지향한다. QGroundControl(QGC v4.x 이상 기준)의 소스 코드 트리에서 LinkManager와 Vehicle 통신 계층을 살펴보면, 백그라운드 타이머 이벤트와 이벤트 루프에서 MAVLink 메시지의 유실을 관찰하는 구조적 기반이 마련되어 있다. 여기에 본 문서에서 다루어진 적응형 필터 모델을 적용하면, 오탐지(False Positive) 비율을 획기적으로 낮출 수 있는 맞춤형 안전 로직을 GCS에 주입할 수 있다.

5. 요약 및 적용 시 주의사항

동적 임계값 산출을 위한 적응형 디지털 필터 아키텍처를 실제 구현할 때는 다음의 특성을 주의하여 조정(Tuning)하라.

  1. 초기 수렴 속도(Convergence Rate): 비행 시작 시점의 네트워크 상태에 필터가 신속하게 적응하도록 초기화 루틴을 구성하라.
  2. 안전 한계값(Clamp Limits)의 지정: \hat{\mu}(k)가 극단적으로 낮아지거나 높아지는 현상을 막기 위해, 최소 보장 임계값(예: 1.5초)과 최대 허용 임계값(예: 10초)의 상하한선을 반드시 씌워라.
  3. 이중화 통신망(Redundancy) 고려: 텔레메트리와 보조 통신(LTE 등) 양쪽의 개별적인 측정치를 관리하도록 인스턴스를 분리하여 디지털 필터를 운용하라.

이러한 제어 공학적 필터 파이프라인은 자율 무인기 시스템의 외부 조종 및 관제 네트워크 복원력(Resilience)을 보장하는 기초 뼈대가 된다.
