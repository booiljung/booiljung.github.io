53.2.5.1. 수신 신호 강도 지시자(RSSI) 및 신호 대 잡음비(SNR)의 고속 푸리에 변환(FFT) 기반 저주파 대역 필터링

무인항공기 시스템(UAS)에서 지상 관제 시스템(Ground Control Station, GCS)과의 신뢰성 있는 통신은 비행 안전성과 임무 수행 능력을 결정짓는 핵심 요소이다. 통신 단절(Data Link Loss) 상황을 조기에 감지하고 적절한 예외 처리(Exception Handling)를 수행하려면 수신 신호 강도 지시자(Received Signal Strength Indicator, RSSI)와 신호 대 잡음비(Signal-to-Noise Ratio, SNR)의 정확한 추정이 필수적이다. 본 절에서는 고속 푸리에 변환(Fast Fourier Transform, FFT) 기반의 저주파 대역 필터링(Low-Pass Filtering) 기법을 활용하여 순간적인 간섭 노이즈를 제거하고 유효한 통신 상태를 추정하는 알고리즘 및 시스템 아키텍처를 분석한다.

1. 무선 주파수(RF) 신호 품질 지표의 특성과 한계

드론과 GCS 간의 MAVLink 통신은 주로 433MHz, 900MHz 또는 2.4GHz 대역의 무선 주파수(Radio Frequency, RF) 채널을 이용한다. 환경적 요인(예: 다중 경로 페이딩(Multipath Fading), 전파 간섭(Interference), 기체 자세 변화에 따른 안테나 방사 패턴의 변동)으로 인해 RSSI 및 SNR 값은 심각한 비선형성과 고주파 잡음(High-Frequency Noise) 특성을 지닌다.

단순한 이동 평균(Moving Average) 필터나 단순 임계값 판별 로직은 이러한 고주파 잡음 환경에서 통신 단절 오탐지(False Positive) 또는 지연 감지(Latency) 문제를 야기할 수 있다. 오탐지는 불필요한 페일세이프(Failsafe) 발동(예: RTL(Return to Launch) 트리거)을 초래하여 임무 연속성을 훼손한다. 따라서 신호의 주파수 스펙트럼(Spectrum)을 분석하여 노이즈 성분을 분리하는 접근이 요구된다.

2. 고속 푸리에 변환(FFT) 기반 저주파 대역 필터링 시스템 구조

시간 영역(Time Domain)의 RSSI/SNR 시계열 데이터를 주파수 영역(Frequency Domain)으로 변환함으로써, 기체의 움직임이나 느린 거리 변화에 의한 기저 주파수(Base Frequency)와 순간적 간섭에 의한 고주파 잡음을 효과적으로 분리할 수 있다.

  1. 시간 영역 데이터 윈도잉(Windowing): 이산 시간(Discrete-Time) 프레임 내의 MAVLink RADIO_STATUS 메시지 배열을 버퍼링한다.
  2. FFT 연산: X(k) = \sum_{n=0}^{N-1} x(n) e^{-j 2\pi k n / N} 식을 적용하여 스펙트럼 성분 X(k)를 추출한다.
  3. 저주파 통과 필터(Low-Pass Filter) 적용: 차단 주파수(Cut-off Frequency, f_c) 이상의 고주파수 대역 성분을 감쇠(Attenuation)시킨다.
  4. 역 고속 푸리에 변환(IFFT): 필터링된 주파수 스펙트럼을 다시 시간 영역의 평활화(Smoothed)된 유효 RSSI/SNR 값으로 복원한다.
graph TD
    A[MAVLink RADIO_STATUS 메시지 수신] --> B[RSSI / SNR 데이터 버퍼 스택 적재]
    B --> C[윈도잉 함수 적용 Hamming/Hanning]
    C --> D[고속 푸리에 변환 FFT]
    D --> E[주파수 영역 분석 스펙트럼 매핑]
    E --> F[저주파 대역 필터링 Cut-off 주파수 이상 감쇠]
    F --> G[역 고속 푸리에 변환 IFFT]
    G --> H[평활화된 신호 기반 통신 단절 위험도 산출]

이러한 신호 처리 체인은 통신 장치 자체의 잡음 특성과 지형적 다중 경로 페이딩을 상쇄하여 보다 견고한 텔레메트리 연결 상태 추정(Telemetry Link State Estimation)을 가능하게 한다.

3. PX4-Autopilot 텔레메트리 처리 소스 코드 및 아키텍처

PX4-Autopilot(펌웨어 v1.14.x 기준) 아키텍처 내부에서 텔레메트리 상태는 telemetry_status uORB(Micro Object Request Broker) 토픽을 통해 관리된다. MAVLink 데이터 패킷이 수신되면 MAVLink 수신 모듈이 파싱을 통해 RADIO_STATUS 메시지 정보를 uORB 시스템에 퍼블리시(Publish)한다.

3.1 소스 코드 레벨 분석

src/modules/mavlink/mavlink_receiver.cpp 모듈 주변에서는 하드웨어에서 전달받은 txbuf, noise, rssi 값을 처리한다.
보통 PX4 시스템은 메모리와 CPU 자원의 한계(NuttX RTOS 환경의 제약)로 인해 비행 제어기 보드(Flight Controller, FC) 단에서 직접 고부하의 FFT 연산을 수행하기보다는 단순화된 IIR(Infinite Impulse Response) 필터를 통한 저항적 필터링을 채택하기도 한다. 하지만 GCS(예: QGroundControl) 또는 온보드 컴패니언 컴퓨터(Companion Computer, ROS2 노드 기반) 측에서는 충분한 컴퓨팅 파워를 통해 FFT 기반 정밀 필터링을 구현할 수 있다.

컴패니언 컴퓨터 상에서 ROS2 노드를 이용한 필터링 구현 시 아키텍처는 다음과 같다:

  1. uXRCE-DDS 통신 브리지: PX4의 telemetry_status uORB 토픽 데이터를 ROS2로 동기화한다.
  2. ROS2 QoS(Quality of Service) 설정: 센서 데이터 스트리밍에 적합한 Best-Effort 구독 모델을 활용한다.
  3. Python/C++ 기반 FFT 처리 노드: numpy.fft 또는 C++ 정밀 수학 라이브러리(e.g., FFTW)를 사용하여 수신 데이터를 실시간으로 변환 및 복원 후 동적 임계값과 비교 판별을 수행한다.

4. Ardupilot과 PX4-Autopilot 통신 신호 관제 패러다임 비교

Ardupilot의 통신 신호 처리 패러다임과 비교했을 때, PX4는 보다 엄격한 상태 머신(State Machine) 기반 통신 단절 감지(Data Link Loss Detection) 전환 로직을 제공한다.

  • Ardupilot: 시스템 파라미터(예: FS_GCS_ENAB) 기반으로 타임아웃 지연 시간(보통 1.5~2초)이 초과되면 즉시 GCS 페일세이프(Failsafe)를 발동시키는 결정론적(Deterministic) 방식의 성향이 강하다. 신호의 강도(RSSI) 자체의 변동성보다는 ’마지막 프로토콜 패킷 수신 시간’에 높은 가중치를 두어 연산 자원을 아낀다.
  • PX4-Autopilot: 커맨더(Commander) 모듈 내부에서 하트비트(Heartbeat) 타임아웃 지표와 함께 telemetry_status 내의 상태를 유기적으로 모델링하여 반영한다. GCS나 컴패니언 컴퓨터에서 보정된 신호 품질 지표(Signal Quality Metric)를 적극적으로 활용할 수 있는 다중 구조 확장에 유리한 마이크로 서비스 기반(uORB) 퍼블리시-서브스크라이브 패턴을 채택하고 있다. 이는 FFT와 같은 고강도 신호 처리에 의해 판별된 “절대적인 링크 손실” 플래그를 외부에서 주입하기에 매우 유연한 장점을 지닌다.

5. QGroundControl 및 설정(Configuration) 적용 방식

QGroundControl(v4.3.x 이상 기준)의 사용자 인터페이스(UI) 프론트엔드에서는 수신된 텔레메트리 라디오 모듈의 신호 감도가 퍼센티지(%) 혹은 dBm 단위로 실시간 표출된다. 고주파 잡음에 의해 이 수치가 불규칙하게 요동치는(Fluctuation) 현상을 시각적으로 방지하고자, GCS 소스코드 내부(Vehicle/Vehicle.cc 및 통신 매니저 클래스 단위)에서는 기초적인 데이터 평균 처리(Averaging) 메커니즘이 내장되어 있다.

이를 더 확장하여 GCS 시스템 단에 FFT 기반 저감 필터 적용 C++ 코드를 플러그인 형태로 추가 구현하면, 시각적 피드백의 인지적 안정성을 극대화하고 GCS 주도적인(GCS-Led) 능동형 연결 복구 체계를 세울 수 있다. 아키텍처 설계적인 측면에서 QGroundControl의 LinkManager 클래스 내에 필터 파이프라인(Filter Pipeline) 처리를 인라인(Inline)으로 배치하도록 아키텍처를 구성하라.

6. 수학적 모델링 요소

고주파 신호를 깎아내는 단순한 임계값 판단 시, 발생할 수 있는 노이즈 왜곡을 수학적으로 방지하기 위해 다음 형태의 판별 기준 모델을 적용한다. (기호 \vert 는 절댓값 등을 명시함)

척도 (Metric)설명판단 식 (FFT 후 IFFT 복원값 \tilde{s}(t) 기준)
순간 간섭 임계값 산출허용 가능한 최대 신호 요동량 도출\mathit{Deviation} = \vert \tilde{s}(t) - \tilde{s}(t-1) \vert < \mathit{Threshold}_{intf}
통신 단절 확정 조건주어진 역치 시간 윈도우 내 평균 신호 손실량 계산\frac{1}{T} \int_{0}^{T} \tilde{s}(t) dt < \mathit{Threshold}_{loss}
무선 연결 품질 인덱스전체 대역 신호 에너지 대비 저주파 대역 에너지 비율Q_{index} = \frac{\int_{0}^{f_c} \vert X(f) \vert^2 df}{\int_{0}^{\infty} \vert X(f) \vert^2 df}

학술 논문 “Analysis of FFT-based Filtering Algorithms for Unmanned Aerial Vehicle Signal Processing(무인항공기 신호 처리를 위한 FFT 기반 필터링 알고리즘 분석)” 등에 따르면, 위와 같이 신호 대 잡음비를 시간 영역을 넘어 스펙트럼 도메인(Spectrum Domain) 단계에서 처리할 경우 페일세이프 오작동률(False Positive Rate)이 현격하게 감소하는 최적화 효과가 존재한다고 서술하고 있다.

7. 결론

수신 신호 강도 지시자(RSSI) 및 신호 대 잡음비(SNR)에 대한 고속 푸리에 변환(FFT) 기반 저주파 대역 필터링 기법은 PX4-Autopilot 시스템이 복잡한 전파 간섭 환경에서 비행할 때 통신 안정성을 보장하기 위한 강력하고 고도화된 소프트웨어적 접근법이다. 물리 계층(하드웨어) 측면에서는 안테나와 라디오 통신 모듈이 단순 물리적 신호를 캡처하지만, 체계 통합적 정보 이론적(Information-Theoretic) 관점에서는 시스템 소프트웨어의 보완적 FFT 처리 과정을 거침으로써 매우 유의미하고 안정적인 거리-신호 감쇄 모델을 추출할 수 있다.

궁극적으로 이러한 신호 처리 아키텍처는 MAVLink 기반 통신 및 ROS2 기반 컴패니언 컴퓨터 연동 환경의 신뢰성을 극대화하여 완전한 자율 에이전트 다중 비행 시스템(Autonomous Agent Swarm Flight System)으로 진일보하기 위한 핵심 통신 인프라 기술이다.
