53.2.2.2 시계열 네트워크 지연 데이터의 리아프노프 안정성(Lyapunov Stability) 기반 임계값 발산 방지 기법

1. 지연 모델의 발산 문제와 안정성 이론의 도입 필요성

QGroundControl(GCS) 단이나 PX4-Autopilot의 백엔드에서 통신 단절(Data Link Loss)을 정밀하게 감지하기 위해 동적 임계값 규칙(Dynamic Thresholding Rule)을 적용할 때, 가장 심각하게 고려해야 하는 취약점 중 하나가 바로 “상태의 발산(State Divergence)“이다. 극단적인 무선 주파수(RF) 간섭이나 셀룰러 망 연결 불량으로 인해 MAVLink 수신 지연 데이터가 겉잡을 수 없이 커지게 될 때, 적응형 필터가 이 비정상적인 지터를 “정상 상태의 변화”로 오인하여 계속해서 학습해버리면 추정된 임계값 역시 무한히 커지게 된다. 결과적으로 페일세이프(Failsafe) 트리거 타임아웃이 30초나 1분 등 비정상적인 수치로 늘어나 기체의 보호 메커니즘 자체가 실질적으로 무력화되는 망각 오류(Forgetfulness Fault)를 초래한다.

이를 수학적 및 제어 이론적으로 원천 방지하기 위해 비선형 제어(Non-linear Control) 동역학의 근간을 이루는 리아프노프 안정성(Lyapunov Stability) 이론을 도입해야 한다. 이는 네트워크 지연 오차(Delay Error) 상태가 특정 평형점(Equilibrium Point) 주변에 제한적으로 머물며 안정적으로 수렴함을 보장하는 적응형 갱신 규칙(Adaptive Update Law)을 설계하는 과정이다.

2. 리아프노프 예비 함수(Lyapunov Candidate Function)의 모델링

동적 임계값을 추정하는 알고리즘 자체를 하나의 제어 대상 시스템(Plant)으로 간주해 보자. 현재 시간 단계 k에서의 시계열 도달 간격을 x_k 라 하고, 모델이 산출한 동적 임계값을 T_k, 그리고 현재 네트워크 상태에 부합하는 이상적인(Ideal) 타임아웃 임계값을 T^*라 정의할 때, 시스템의 추정 오차는 e_k = T_k - T^* 로 정의된다.

이 알고리즘이 안정된 상태를 유지하는지 판별하기 위해, 양의 정부호성(Positive Definite) 특성을 지니는 리아프노프 함수 V(e_k)를 다음과 같은 오차의 제곱 형태로 모델링한다.

V(e_k) = \frac{1}{2} e_k^2

리아프노프 안정성 이론 제2법칙에 의거하여, 필터가 시계 탐색 단계를 거치며 임계값 추정 오차가 무한대로 발산하지 않고 유계(Bounded) 내지 0으로 수렴하게 하려면 에너지 함수의 변화율이 음의 반정부호성(Negative Semi-definite)을 반드시 만족해야 한다.

\Delta V(e_k) = V(e_{k+1}) - V(e_k) \le 0

위 부등식을 제약 조건(Constraint)으로 두고 식을 전개하면, 추정 임계값 T_{k+1}을 업데이트할 때 측정 오차가 시스템 게인(Gain)에 반영되는 규칙의 수학적 상한선(Upper Bound)을 결정할 수 있다.

3. 임계값 발산 방지를 위한 강건한 제어 법칙 (Update Law) 구현

앞서 증명된 리아프노프 기반 안정성 제약 조건(Lyapunov-based Stability Constraint)은 실제 GCS의 C++ 백엔드 소스 층위에서 오차의 누수(Leakage) 계수와 포화 한계(Saturation Limit)의 형태로 직관적으로 투영된다.

3.1 \sigma-수정 (Sigma-Modification) 및 붕괴(Leakage) 인자의 결합

추정되는 지연 변동폭이나 임계값 T_k 자체가 내부 적분기(Integrator) 구조에 의해 꾸준히 커지는 와인드업(Wind-up) 현상을 방지하기 위해 0 < \gamma < 1 범위의 서서히 붕괴하는 누수율(Leakage Rate)을 피드백 루프에 도입한다. 이는 추정 오차가 무한한 상태 공간으로 벗어나지 않도록 일종의 인공적인 스프링 복원력(Restoring Force)을 시스템에 부여하는 역할을 한다. 적응 제어 공학에서 널리 알려진 \sigma-수정 기법의 산물이다.

3.2 포화 함수(Saturation Function)를 통한 클램프 한계 설정

이론적 궤적을 만족시킬 수 없는 극단적 측정 외란(Disturbance) 유입 시, 그 값이 내부 상태 벡터를 훼손하기 전에 강제로 값을 깎아내는 Saturator 블록을 거치도록 알고리즘을 설계한다.

// QGroundControl 통신 안정화 백엔드 설계에서의 개념적 안전성 검증 로직 구현
double calculate_lyapunov_stable_threshold(double current_iat, double prev_threshold, double max_safe_limit) {
    // 1. 측정 오퍼레이션과 리아프노프 이득(Gain K) 반영
    double estimation_err = current_iat - prev_threshold;
    double K = adaptive_gain_from_lyapunov(estimation_err); 
    double tentative_threshold = prev_threshold + K * estimation_err;

    // 2. 누수 인자 결합으로 인한 적분 팽창 억제 (Sigma-Modification 개념)
    tentative_threshold = LEAKAGE_FACTOR * tentative_threshold;

    // 3. 발산 상한(Upper Boundary)을 통한 Saturation / Clamping
    if (tentative_threshold > max_safe_limit) {
        tentative_threshold = max_safe_limit;
    }
    
    // 4. 최소한도 임계값 유지 보증 (최소 Failsafe 기준 보호)
    return std::max(MIN_SAFE_LIMIT, tentative_threshold);
}

4. Ardupilot 대비 PX4 생태계의 차별성 설계 관점 향상

보수적인 파라미터화의 Ardupilot 환경이나 단순한 MAVLink 구조에서는 하드코딩된 단일 상수값(1.5\text{s} 내지 3\text{s} 등)을 Failsafe 기준점으로 취급하는 것에 안주하여, 동적 변수화가 잉태할 수 있는 발산 리스크를 원천적으로 우회해왔다.

하지만 ROS2 구조(Micro-XRCE-DDS)와 결합되거나 복잡한 위성(SATCOM), 인터넷 릴레이 다중 서버 망을 거치는 현대의 군집형 PX4-Autopilot 환경에서는 통신 주기의 비결정성(Non-determinism)이 매우 높기 때문에 단일 상수만으로는 효과적인 관제가 불가능하다. 여기서 “적응성(Adaptability)“의 도입은 필연적이지만 역으로 자율적으로 변하는 파라미터가 시스템의 방어 기제를 파괴할 잠재적 취약점을 안고 통제 망에 침투한다. 이러한 아키텍처적 위기 상황에서, 리아프노프 안정성을 기반으로 한 임계값 계산 파이프라인(Thresholding Pipeline) 설계는 측정 데이터가 무한대로 폭증하는 최악의 조건 속에서도 Failsafe FSM의 신뢰도(Reliability)를 엄밀한 제어 이론적 잣대로 사전에 보증(Guarantee)하는 돌파구를 제시한다.

5. 결론 및 고찰

동적 임계값을 차용하는 자율 무인기 관제망(Autonomous UAV Ground Network)을 아키텍처화 할 때, 측정된 시계열 MAVLink 도달 간격에 단순히 소프트웨어적인 이동 평균이나 칼만 필터 함수를 라이브러리로 적용하는 수준에서 그쳐서는 안 된다. FSM 상태 공간을 이동시키는 이 추정 메커니즘 자체가 닫힌 제어 루프(Closed-loop) 생태계 안에서 결코 무한대로 발산하지 않는다는 리아프노프 기반의 수학적 안전성을 확보하고, 그 증명 모델이 소스코드 내의 누수율(Leakage Rate)과 포화 한계(Saturation Limit)의 형태로 견고하게 반영되어야만 비할 데 없는 강건성(Robustness)을 갖춘 GCS를 배포할 수 있을 것이다.
