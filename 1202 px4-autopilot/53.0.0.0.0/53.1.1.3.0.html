53.1.1.3 상향 링크(Uplink) 제어 신호 소실에 따른 개루프(Open-Loop) 상태 전이와 적분기 와인드업(Integrator Windup) 누적 오차 발산 현상

무인항공기(UAV)의 비행 제어 시스템(FCS, Flight Control System)에서 지상 관제 시스템(GCS, Ground Control Station) 또는 조종기(Radio Control)로부터 수신되는 상향 링크(Uplink) 제어 신호는 기체의 상태를 참조(Reference)하는 중요한 입력 값이다. 이러한 제어 신호가 소실되면, 폐루프(Closed-Loop) 기반의 제어 구조가 개루프(Open-Loop) 상태로 전환되며, PID 제어기의 적분기 와인드업(Integrator Windup) 현상이 발생하여 오차가 발산할 위험이 존재한다.

1. 개루프(Open-Loop) 상태 전이의 제어 공학적 의미

정상적인 비행 상태에서는 GCS나 조종기의 입력과 기체 센서(IMU, GPS 등)의 피드백을 통해 오차(Error)를 계산하고, 이를 보상하는 폐루프 제어가 동작한다. 하지만 상향 링크 신호가 단절되면, 조종 명령이라는 목표값(Setpoint)이 업데이트되지 않고 이전 상태를 유지하거나 최솟값으로 초기화된다.

이때 피드백 루프는 계속 작동하더라도, 유효한 목표값이 없으므로 제어기는 불명확한 상태를 추종하게 되어 사실상 개루프(Open-Loop) 특성을 보이게 된다. 특히 위치 제어(Position Control) 모드에서 이러한 현상이 발생하면, 바람이나 외부 교란(Disturbance)에 의해 기체가 드리프트(Drift)하는 것을 제어기가 정상적인 목표값의 변경으로 착각하거나 제대로 보상하지 못하는 상황이 발생한다.

2. 적분기 와인드업(Integrator Windup) 현상 메커니즘

PID 제어기에서 적분 제어(Integral Control) 항은 과거의 누적 오차를 반영하여 정상 상태 오차(Steady-State Error)를 제거하는 역할을 한다. 수식으로는 다음과 같이 표현된다.

I(t) = K_i \int_{0}^{t} e(\tau) d\tau

여기서 K_i는 적분 이득(Integral Gain)이고, e(\tau)는 시간 \tau에서의 오차이다.

상향 링크 단절로 인해 제어 목표값이 비정상적인 값으로 고정되거나, 기체가 외부 교란에 의해 밀려나면 반대 방향을 가리키는 오차 e(t)가 지속적으로 발생한다. 액추에이터(모터)가 이미 물리적인 최대 한계 출력(Saturation)에 도달하여 더 이상 자세나 위치를 교정할 수 없는 상태임에도 불구하고, 적분기는 이 오차를 지속적으로 누적하게 된다. 이를 적분기 와인드업이라고 부른다.

와인드업 현상이 발생하면 적분항 I(t)가 과도하게 큰 값을 가지게 되어, 이후 통신이 복구되거나 페일세이프(Failsafe) 모드로 진입하더라도 누적된 거대한 오차 값으로 인해 심하게 오버슈트(Overshoot)하거나 회복 불가능한 불안정한 발산 거동을 보이게 된다.

PX4-Autopilot의 와인드업 방지 아키텍처

PX4-Autopilot은 이러한 현상을 방지하기 위해 안티 와인드업(Anti-Windup) 기법과 상태 머신(State Machine) 전환 로직을 강력하게 적용하고 있다.

1. 동적 적분기 클램핑(Dynamic Integrator Clamping)

PX4의 비행 제어 모듈(예: mc_pos_control, mc_att_control) 내부에서는 제어기의 출력이 액추에이터의 한계에 도달했을 때 적분기의 누적을 일시 중지하는 동적 클램핑(Clamping) 기법을 사용한다. 액추에이터가 포화(Saturation) 상태에 있고, 현재 오차의 부호가 포화된 출력의 방향과 같을 때 적분 연산을 즉각적으로 정지시킨다.

2. 제어 모드 전환 시 적분기 초기화(Integrator Reset)

Uplink 소실이 감지되어 기체의 모드가 수동 제어(Manual Control) 등에서 자율 복귀(RTL, Return to Launch)나 홀드(Hold) 등의 페일세이프 모드로 자동 전환될 때, PX4는 제어기의 적분항 값을 0으로 초기화하거나 새로운 안전 상태에 맞춰 재설정(Reset)한다. 이를 통해 통신 단절 전까지 비정상적으로 누적되었던 오차가 새로운 모드의 제어 루프에 영향을 주는 것을 사전에 차단한다.

stateDiagram-v2
    [*] --> Normal_Flight : Uplink Active (Command Received)
    
    state Normal_Flight {
        [*] --> PID_Active
        PID_Active --> Error_Calculation
        Error_Calculation --> Actuator_Output
    }
    
    Normal_Flight --> Link_Lost : Uplink Timeout (Data Link Loss)
    
    state Link_Lost {
        [*] --> Anti_Windup_Triggered
        Anti_Windup_Triggered --> Integrator_Clamped : Stop Error Accumulation
        Integrator_Clamped --> Failsafe_Mode_Evaluation
    }
    
    Link_Lost --> RTL_Mode : Failsafe Triggered
    
    state RTL_Mode {
        [*] --> Integrator_Reset : Reset Accumulated I-Term
        Integrator_Reset --> Autonomous_Return
    }

Ardupilot 대비 차이점

Ardupilot은 전통적으로 Leaky Integrator(누설 적분기) 방식을 일부 사용하거나 I_MAX 파라미터를 통해 적분기의 최대값을 절대적인 수치로 강하게 제한하는 방식이 자주 쓰인다. 반면, PX4는 제어 루틴의 ControlMath::setZeroIfPtr 같은 유틸리티 함수나 혼합기(Mixer/Allocation) 레벨에서의 포화 상태 피드백을 받아, 제어 루프 내에서 가변적(Dynamic)으로 클램핑 한계를 조절하는 데 더 최적화된 아키텍처를 보여준다. 두 시스템 모두 진보된 안티-와인드업 구조를 가지나, PX4의 모듈식 독립 구조는 상태 머신 전환(Commander 모듈)과 제어 모듈(Controller) 간의 적분기 제어권 이양이 조금 더 세분화되고 객체 지향적으로 이루어지는 특징이 있다.

시사점 및 결론

상향 링크의 소실은 단순히 기체에 명령을 내릴 수 없는 조종 불능 상태를 넘어, 제어기 내부의 오차가 비정상적으로 증폭되는 치명적인 상태를 유발할 수 있다. PX4-Autopilot과 같은 현대적인 다중 로터 비행 제어 시스템은 통신 타임아웃을 감지하는 즉시 안티 와인드업 로직을 가동하여 물리적 포화를 방어하고, 비상 모드로의 전이 시 적분기를 적절히 초기화함으로써 이러한 누적 오차의 발산을 제어 공학적으로 강력하게 억제한다.
