53.3.1.2 워치독 타이머(Watchdog Timer) 인터럽트 발생에 따른 실시간 운영체제(NuttX RTOS) 컨텍스트 스위칭(Context Switching) 우선순위 할당

본 절에서는 PX4-Autopilot(펌웨어 버전 1.14 기준)이 탑재된 시스템에서 워치독 타이머(Watchdog Timer) 인터럽트가 발생했을 때, 하부 실시간 운영체제인 NuttX RTOS 상에서 어떠한 방식으로 컨텍스트 스위칭(Context Switching)이 일어나는지, 그리고 스레드(Thread) 우선순위 할당(Priority Allocation)은 어떻게 이루어지는지 상세히 분석한다. 비행 제어기(Flight Controller)의 안전은 최우선 과제이며, 시스템 멈춤(Hang)이나 무한 루프(Infinite Loop) 발생 시 이를 신속하게 감지하여 복구하거나 안전하게 시스템을 정지시키는 메커니즘이 필수적이다.

1. 워치독 타이머(Watchdog Timer) 개요 및 시스템적 의미

워치독 타이머는 하드웨어 혹은 소프트웨어적으로 구현된 타이머로, 시스템이 정상적으로 동작하고 있을 때는 주기적으로 타이머를 초기화(Kick 또는 Feed)하여 시간이 만료되는 것을 방지한다. 만약 메인 제어 루프가 멈추거나 특정 태스크(Task)가 교착 상태(Deadlock)에 빠져 타이머를 리셋하지 못하면, 워치독 타이머는 만료(Expiration)되어 시스템에 비마스커블 인터럽트(NMI, Non-Maskable Interrupt)나 최고 우선순위 하드웨어 인터럽트를 발생시킨다.

PX4-Autopilot에서는 비행 안전성을 보장하기 위해 각 주요 모듈(예: commander, sensors, ekf2 등)의 동작 주기를 감시한다. 특히, 하드웨어 타이머와 결합된 하위 레벨의 워치독 아키텍처는 시스템 프로세서가 제어 불능 상태에 도달했을 때 마지막 비상 복구 수단으로 작용한다.

2. NuttX RTOS의 인터럽트 서비스 루틴(ISR) 및 컨텍스트 스위칭 프로세스

워치독 타이머 인터럽트가 발생하면, 기초 운영체제인 NuttX RTOS는 즉각적으로 현재 진행 중인 스레드의 실행을 중단하고 인터럽트 서비스 루틴(ISR, Interrupt Service Routine)으로 제어권을 이전한다. 이 과정을 컨텍스트 스위칭(Context Switching)이라고 부른다.

  1. 상태 레지스터 백업(State Register Backup): 현재 실행 중인 태스크의 프로그램 카운터(Program Counter), 스택 포인터(Stack Pointer) 및 주요 레지스터(Register) 값들을 해당 태스크의 스택(Stack) 또는 TCB(Task Control Block)에 저장한다.
  2. ISR 핸들러 호출(ISR Handler Invocation): 워치독 인터럽트에 매핑된 벡터 테이블(Vector Table)을 참조하여 하드웨어 워치독 오류 핸들러(Fault Handler)를 실행한다.
  3. 페일세이프 트리거(Failsafe Trigger) 및 스케줄링 큐(Scheduling Queue) 재구성: 워치독 ISR은 단순 로그 기록에 그칠 수도 있으나, 심각도에 따라 시스템의 리셋을 예약하거나 최고 우선순위의 페일세이프 스레드(Failsafe Thread)를 대기열(Ready Queue)의 최상단에 삽입한다.
  4. 우선순위 기반 복귀(Priority-Based Return): ISR 종료 시, 스케줄러는 다시 실행할 태스크를 선정한다. 이때 일반적인 제어 스레드보다 워치독 복구 및 페일세이프 처리를 담당하는 스레드의 우선순위가 높기 때문에, 즉시 해당 스레드로 컨텍스트 스위칭이 발생한다.

3. 워치독 예외 발생에 따른 우선순위 할당(Priority Allocation) 및 스케줄링 전략

NuttX RTOS는 선점형(Preemptive) 스케줄링 방식을 사용한다. 256단계(0~255)의 우선순위를 지원하며, 번호가 높을수록 높은 우선순위를 의미한다.

  • 인터럽트 레벨(Interrupt Level): 워치독 타이머 인터럽트는 일반적인 센서 데이터 수신(SPI/I2C) 인터럽트보다 높은 최상위 하드웨어 레벨 인터럽트로 설정된다. 이는 다른 모든 인터럽트를 중단(Preempt)시키고 즉각 실행됨을 의미한다.
  • 페일세이프 스레드 우선순위(Failsafe Thread Priority): 워치독 인터럽트에 의해 깨어나는(Awaken) 오류 처리 또는 안전 종료 워크 큐(Work Queue) 태스크는 NuttX 체계 내에서 가장 높은 우선순위(예: SCHED_PRIORITY_MAX, 보통 255)에 가깝게 임시 상향(Priority Boosting)되거나 처음부터 매우 높게 할당되어 있다.
  • 우선순위 역전 방지(Priority Inversion Prevention): 시스템이 세마포어(Semaphore)나 뮤텍스(Mutex) 대기로 인해 워치독 스레드가 자원을 얻지 못하는 것을 방지하기 위해, 우선순위 상속(Priority Inheritance) 메커니즘이 동작하거나 워치독 핸들러 내에서는 블로킹(Blocking) API 사용을 엄격히 금지한다.

이러한 철학은 ChibiOS를 사용하는 Ardupilot과 궤를 같이하나, NuttX 환경 하에서 POSIX 표준에 입각한 멀티 스레드 아키텍처를 유연하게 활용한다는 점에서 PX4만의 독특한 강건성을 가진다.

4. 시스템 동작 시퀀스 도식화

다음은 제어 루프가 지연되어 워치독 인터럽트가 발생하고, NuttX RTOS가 컨텍스트 스위칭을 통해 비상 로직을 처리하는 과정을 나타낸 시퀀스 다이어그램(Sequence Diagram)이다.

sequenceDiagram
    participant MainLoop as Main Control Loop (Thread)
    participant WDT as Hardware Watchdog Timer
    participant RTOS as NuttX RTOS (Scheduler)
    participant ISR as Watchdog ISR
    participant Failsafe as Failsafe/Recovery Thread

    MainLoop->>WDT: 주기적 타이머 리셋 (Kick)
    Note over MainLoop, WDT: 정상 상태 유지
    MainLoop-->>MainLoop: 무한 루프 또는 데드락 발생! (지연)
    Note over WDT: 설정된 타임아웃 초과 (Timeout)
    WDT->>RTOS: 하드웨어 인터럽트 발생 (NMI)
    RTOS->>RTOS: 현재 MainLoop 컨텍스트 임시 저장 (TCB)
    RTOS->>ISR: 인터럽트 서비스 루틴 실행
    ISR->>Failsafe: 페일세이프 스레드 준비 (Ready Queue 삽입)
    ISR-->>RTOS: ISR 완료 및 리턴
    RTOS->>RTOS: 스케줄러 재호출 (Priority Check)
    Note over RTOS: Failsafe 우선순위가 최고 수준으로 평가됨
    RTOS->>Failsafe: 컨텍스트 복구 및 Failsafe 동작 개시
    Failsafe->>Failsafe: 시스템 안전 정지, 모터 컷오프 또는 재부팅 수행

5. 소스 코드 레벨에서의 우선순위 제어 분석

PX4의 소스 코드 구조에서 시스템 전반의 태스크 생존 여부를 모니터링하는 컴포넌트는 주로 src/systemcmds/watchdog/ 및 하부 드라이버 계층에 구현되어 있다. (특정 보드(Board) 패키지의 아키텍처별로 상이할 수 있다.)

NuttX 내부 구조(nuttx/sched/)의 up_switchcontext() 함수나 인터럽트 진입점 코드는 다음과 같은 원리로 동작한다.

  1. 하드웨어 타이머 레지스터에서 인터럽트 플래그 세팅.
  2. 어셈블리 레벨의 트랩 핸들러(Trap Handler) 호출로 모든 레지스터 상태를 프로세스 스택 푸시.
  3. PX4 보드별 워치독 타이머 콜백에서 치명적 에러(Fatal Error) 판단 시 스케줄러의 문맥 스위치 강제 호출.

이와 같은 정교한 실시간 운영체제 컨텍스트 스위칭과 우선순위 할당은 통신 단절(Data Link Loss) 감지나 센서 결함 등의 상위 레벨 페일세이프보다 더 근본적인, CPU 및 메모리 수준의 보호막(Safeguard)을 제공하여 현대 무인항공기 시스템(UAS)의 궁극적인 신뢰성을 확보한다.
