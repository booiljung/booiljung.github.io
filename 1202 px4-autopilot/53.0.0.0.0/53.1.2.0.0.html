53.1.2 개방형 시스템 간 상호 연결(OSI) 7계층 아키텍처 기반 계층별 통신 단절 분류 체계

무인항공기(UAV) 지상 관제 시스템(GCS, Ground Control Station)과 비행 제어 시스템(FCS, Flight Control System) 간의 통신 단절(Data Link Loss) 현상을 분석하고 예외 처리를 수행하기 위해서는, 단절의 원인이 시스템의 어느 계층에서 발생했는지를 명확히 규명해야 한다. 개방형 시스템 간 상호 연결(OSI, Open Systems Interconnection) 7계층 아키텍처는 통신 시스템을 모듈화하여 이해하는 표준 프레임워크로, 무인기의 통신 단절 현상을 진단하는 데 매우 효과적인 기준을 제공한다.

1. OSI 7계층 기반 통신 단절의 구조적 이해

PX4-Autopilot과 QGroundControl (또는 사용자 정의 GCS) 사이의 통신은 텔레메트리 라디오(Telemetry Radio), Wi-Fi, LTE 등 다양한 매체를 통해 이루어진다. 이들은 모두 OSI 참조 모델의 특정 계층 스택을 지나며 동작한다. 통신 장애가 발생했을 때 물리적인 칩셋의 문제인지, 라우팅의 문제인지, 혹은 응용 프로그램의 타임아웃 오류인지 판단하는 분류 기준은 다음과 같다.

1.1 물리 계층 (Physical Layer, L1)

물리 계층은 하드웨어와 전파 매체를 직접적으로 다룬다. 여기서 발생하는 단절은 아날로그 신호 자체의 소실을 의미한다.

  • 원인: 송수신 안테나의 파손, RF(Radio Frequency) 신호의 간섭 및 감쇠(Attenuation), 전원 공급 불량으로 인한 모뎀 꺼짐 현상. 반송파 감지 불량(Carrier Sense Failure).
  • PX4 및 GCS에서의 관찰: 라디오 모뎀의 링크 LED가 소등되거나, 스펙트럼 분석기에서 신호가 사라진 것으로 관찰된다. 소프트웨어적으로는 하위 디바이스 드라이버 레벨에서 Hardware I/O Error로 나타난다.

1.2 데이터 링크 계층 (Data Link Layer, L2)

노드 간의 프레임 전송을 다루며, 다중 접속 환경에서의 충돌 비중이 높다.

  • 원인: MAC(Media Access Control) 프레임 단위의 충돌(Collision), 프레임 체크 시퀀스(FCS, Frame Check Sequence) 오류로 인한 패킷 드랍. 군집 비행(Swarm) 중 타 기체와의 채널 점유 실패.
  • PX4 및 GCS에서의 관찰: 물리적 신호는 식별되나 유효한 패킷으로 재조립되지 못하므로, 라디오 펌웨어(SiK 텔레메트리 등)에서 높은 Packet Error Rate(PER)로 집계된다.

1.3 네트워크 계층 (Network Layer, L3)

IP 통신 환경(Wi-Fi, LTE 등)에서 다른 네트워크를 거칠 때의 경로 선택을 다룬다. (Serial 통신 기반의 기본 텔레메트리는 이 계층과 L4의 영향을 덜 받지만, UDP/TCP 환경에서는 중요하게 작용한다.)

  • 원인: IP 할당 실패, 라우팅 테이블 오류, 망 릴레이(Relay) 서버의 다운, Network Address Translation(NAT) 장비의 포트 매핑 해제.
  • PX4 및 GCS에서의 관찰: Ping(ICMP) 테스트 시 Destination Host Unreachable 메시지가 나타난다.

1.4 전송 계층 (Transport Layer, L4)

단말 간의 신뢰성 또는 비신뢰성 데이터 전송(TCP/UDP)을 담당한다.

  • 원인: UDP 패킷 유실률의 급증으로 인한 버퍼 언더런(Underrun). TCP 혼잡 제어(Congestion Control) 알고리즘 오동작, 소켓(Socket) 연결 강제 종료(Connection Reset), EAGAIN 또는 포트 바인딩 실패.
  • PX4 및 GCS에서의 관찰: MAVLink 라우터나 GCS 소켓 레벨에서 수신 버퍼가 갱신되지 않아 이벤트를 발생시키지 못한다.

1.5 세션 계층 (Session Layer, L5)

통신 세션의 수립, 유지, 동기화를 관리한다.

  • 원인: GCS와 기체간의 논리적 통신 세션 수립 실패, 장기 미사용 포트의 세션 만료, VPN(Virtual Private Network) 환경에서의 터널 유지 갱신 실패.
  • PX4 및 GCS에서의 관찰: 소켓은 열려 있으나, 상위 어플리케이션(QGC) 내부의 차량(Vehicle) 관리 매니저가 유효한 세션으로 인정하지 않는다.

1.6 표현 계층 (Presentation Layer, L6)

데이터의 포맷, 인코딩 변환 작업과 암호화/복호화를 담당한다.

  • 원인: MAVLink v1/v2 패킷 인코딩 및 디코딩 불일치. 보안 통신(Security Link) 환경에서의 암호화 키 불일치 및 인증서 만료로 인한 패킷 파싱 거부(Parsing Rejection).
  • PX4 및 GCS에서의 관찰: 패킷은 정상적으로 도달하나 데이터 스트림에 대한 CRC(Cyclic Redundancy Check) 검사에서 지속적으로 실패하여, 찌그러진(Garbage) 데이터로 버려진다.

1.7 응용 계층 (Application Layer, L7)

소프트웨어 응용 프로그램과 사용자 인터페이스에서 최종적인 서비스가 제공되는 영역이다.

  • 원인: MAVLink Heartbeat 메시지 타임아웃, GCS 내부 통신 스레드(Thread)의 데드락(Deadlock) 또는 기아(Starvation) 상태, PX4 commander 모듈의 프로세스 크래시. MAVSDK 내 ConnectionLost 예외.
  • PX4 및 GCS에서의 관찰: 하위 계층에 아무런 문제가 없더라도, 특정 시간(예: 1~3초) 동안 유효한 MAVLink Heartbeat 메시지를 수신하지 못하면 어플리케이션은 “통신 단절(Link Lost)” 사용자 인터페이스(UI) 팝업을 띄우고 페일세이프 동작을 유발한다.

2. 아키텍처 분류 체계를 활용한 통신 단절 진단 워크플로우

위의 계층적 특징을 기반으로, 시스템 개발자는 통신 단절 감지 시 아래의 계층적 진단 구조를 가져야 한다.

3. 시사점

실제 현장에서 발생하는 무인기의 “통신 단절(Data Link Loss)“은 흔히 전파가 끊어졌다는 1차원적인 인식으로 다뤄지기 쉽다. 그러나 PX4-Autopilot 환경과 같이 복잡성 높은 소프트웨어 스택과 MAVLink/ROS2 등의 상위 프로토콜을 사용하는 시스템에서는, 단절 현상이 물리적 계층(L1)은 물론 프로토콜 파싱(L6)이나 어플리케이션 스케줄링 장애(L7) 등 다방면에서 발생할 수 있다. OSI 7계층 아키텍처에 입각한 원인 분류와 접근은 GCS 개발자와 펌웨어 엔지니어가 시스템 예외 처리 로직을 구조적으로 설계하고 디버깅하는 데 필수적인 방법론이다.
