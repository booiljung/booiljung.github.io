53.1.1 무인기 비행 제어 시스템(FCS) 내 폐루프(Closed-Loop) 제어 상실의 공학적 메커니즘

무인항공기(UAV)의 비행 제어 시스템(FCS: Flight Control System)은 다중 루프(Multi-loop) 피드백 제어 구조를 가진다. 일반적으로 안쪽 루프(Inner Loop)는 기체의 각속도 및 자세(Attitude)를 안정화하고, 바깥쪽 루프(Outer Loop)는 고도, 위치, 속도를 제어한다. 지상 관제 시스템(GCS)이나 보조 컴퓨터(Companion Computer)와 연결된 데이터 링크(Data Link)는 이 제어 구조의 최외곽에서 웨이포인트(Waypoint)나 궤적(Trajectory) 형태의 기준 지령(Reference Command)을 제공하는 핵심 요소다. 통신 단절(Data Link Loss)은 바로 이 최외곽 제어 루프의 단절을 의미하며, 이는 시스템 전체의 안정성에 심각한 공학적 파급 효과를 미친다.

1. 폐루프 제어에서의 기준 지령(Reference Signal) 상실

제어 공학적으로 폐루프 시스템의 오차(Error) e(t)는 기준 지령 r(t)와 출력 응답 y(t)의 차이로 정의된다.

e(t) = r(t) - y(t)

정상적인 데이터 링크 상태에서 r(t)는 GCS나 MAVLink 프로토콜을 통한 외부 에이전트(예: ROS2 네트워크 노드)에 의해 지속적으로 갱신된다. 그러나 통신이 단절되면 r(t)의 업데이트가 중단된다. 디지털 제어기에서 이러한 현상은 영차 유지(Zero-Order Hold, ZOH) 효과로 나타나며, 내부 제어 시스템은 마지막으로 수신된 r(t_0) 값을 무기한 유지하게 된다.

1.1 속도 및 가속도 제어 루프에서의 발산(Divergence) 위험

만약 오프보드(Offboard) 모드에서 기체가 특정 방향으로 5m/s의 속도 지령 또는 가속도 지령을 추종하던 중 통신이 유실된다고 가정하자.
ZOH 특성에 의해 FCS는 계속해서 5m/s의 속도(혹은 가속도)를 달성하려 시도할 것이다. 외부 제어기가 새로운 위치 설정점을 주어 기체를 정지시켜야 하지만, 그 루프가 끊어졌으므로 기체는 장애물과 충돌하거나 비행 구역을 이탈할 때까지 해당 속도를 유지하며 궤적 발산(Trajectory Divergence)을 일으킨다.

2. MAVLink 제어 루프 체인과 타임아웃(Timeout) 방어 기재

이러한 폐루프 제어 상실의 치명성을 방지하기 위해 PX4-Autopilot은 데이터 링크 자체를 ’가상의 센서 혹은 상태 제어기’로 취급하여 타임아웃 방어 기재를 내장하고 있다.

graph TD
    GCS[GCS / ROS2 Agent] -->|MAVLink Reference Commands| MAV_RCV[MAVLink Receiver Module]
    MAV_RCV -->|uORB: trajectory_setpoint| Commander[Commander FSM]

    Commander -->|Timeout| Failsafe[Failsafe Trigger: Hold / RTL]
    Control_Allocation --> Actuators[Actuators / Motors]

PX4의 commander 모듈은 GCS로부터 수신되는 HEARTBEAT 메시지나 ROS2 브리지(uXRCE-DDS)를 통한 OffboardControlMode 메시지의 타임스탬프를 상시 감시한다.
통신 지연이 파라미터(예: COM_DL_LOSS_T 또는 COM_OBL_LOSS_T)에 설정된 임계값을 초과하면, PX4는 현행 r(t) 추종 루프를 의도적으로 파기(Break)하고, 안전이 보장된 내부 내비게이터(Navigator) 기반의 새로운 폐루프(예: 제자리 정지(Hold) 또는 시작점 복귀(RTL))로 제어권을 강제 이관한다.

3. Ardupilot 대비 PX4-Autopilot의 관제 철학 차이

PX4와 Ardupilot 모두 통신 단절로 인한 제어 상실을 방어하는 메커니즘을 갖추고 있으나, 구조적 초점에 미세한 차이가 있다.

  • PX4-Autopilot: 다중 에이전트 및 ROS2를 활용한 분산 제어 시스템(Distributed Control System) 연동에 특화되어 있다. 따라서 오프보드 제어 루프 상실(COM_OBL_LOSS_T)과 일반적인 GCS 원격 측정 상실(COM_DL_LOSS_T)을 엄격히 분리하여 상태 머신(State Machine)을 운용한다. 고속으로 갱신되어야 하는 ROS2 제어 루프가 끊어졌을 때 더 빠르고 민감하게(보통 1초 미만) 반응하도록 설계되어 있다.
  • Ardupilot: FS_GCS_ENABL 파라미터를 통해 GCS 안전 장치를 설정하며, 상대적으로 장거리 매핑(Mapping)이나 안정적인 순항 임무 중 발생하는 간헐적 통신 지연에 대해 다소 보수적이고 통합적인(단일화된) Failsafe 천이 모델을 제공한다.

결과적으로 무인기 비행 제어 시스템에서 통신 단절은 단순한 ’연결 상태 표시등의 꺼짐’이 아니라, 기체의 궤적을 결정하는 가장 큰 피드백 루프의 절단을 의미한다. 따라서 GCS 엔지니어 및 ROS2 기반 자율 제어 개발자는 통신 갱신 주기와 PX4의 Failsafe 타임아웃 파라미터를 면밀히 튜닝하여 제어 상실로 인한 궤도 발산을 원천 차단해야 한다.
