53.1.1.1 라플라스 변환(Laplace Transform) 기반 순수 시간 지연 요소(Pure Time Delay Element) 증가가 제어계 위상 여유(Phase Margin)에 미치는 영향

UAS(Unmanned Aerial System)의 데이터 링크 환경에서 통신이 지연되거나 패킷 손실(Packet Loss)이 발생하여 재전송이 일어나는 상황은 제어 공학 관점에서 제어 루프 내 ‘순수 시간 지연(Pure Time Delay)’ 요소가 유입됨을 의미한다. 지상 관제 시스템(GCS) 또는 외부의 ROS2 기반 자율 에이전트가 오프보드(Offboard) 모드에서 기체를 제어할 때, 이러한 통신 지연은 시스템의 동적 안정성에 치명적인 영향을 미칠 수 있다. 본 절에서는 연속 시간(Continuous-Time) 시스템의 라플라스 변환(Laplace Transform) 모델을 통해 통신 지연이 위상 여유(Phase Margin)에 미치는 영향을 역학적으로 분석한다.

1. 순수 시간 지연(Pure Time Delay)의 라플라스 변환 모델

어떤 신호 x(t)가 물리적 통신 매개체(RF 링크 등)를 거치며 시간 T_d 만큼 지연되어 수신되는 신호를 x(t - T_d)라 하자.
이 지연된 신호의 라플라스 변환(Laplace Transform)은 시간 전이 정리(Time Shifting Theorem)에 의해 다음과 같이 표현된다.

\mathcal{L} \{ x(t - T_d) \} = e^{-s T_d} X(s)

여기서 지연 요소의 전달 함수(Transfer Function)는 G_{delay}(s) = e^{-s T_d} 가 된다. 이 요소는 신호의 크기나 형태를 왜곡하지 않고 단지 시간 축에서 평행 이동시킬 뿐이므로 ’순수 시간 지연(Pure Time Delay)’이라고 부른다.

2. 주파수 응답(Frequency Response) 및 위상 지연(Phase Lag)

시간 지연이 폐루프 안정성에 미치는 영향을 주파수 영역(Frequency Domain)에서 확인하기 위해 s = j\omega 를 대입하여 주파수 응답을 구하면 다음과 같다.

G_{delay}(j\omega) = e^{-j\omega T_d}

이 복소 함수의 크기(Magnitude)와 위상(Phase)은 오일러 공식(Euler’s Formula)에 의해 다음 특성을 갖는다.

  1. 크기(Magnitude): \vert e^{-j\omega T_d} \vert = 1 (즉, 모든 주파수 대역에서 0 dB로 크기 감쇠는 없다)
  2. 위상(Phase): \angle e^{-j\omega T_d} = -\omega T_d [radian]

크기는 변화가 없으나, 통과하는 주파수 \omega가 높아질수록 또는 지연 타임 T_d가 길어질수록 주파수에 비례하여 선형적으로 막대한 형태의 **위상 지연(Phase Lag)**이 발생하게 된다.

3. 위상 여유(Phase Margin)의 잠식 및 불안정성 역학

안정적인 무인기 비행 제어를 위해 폐루프 시스템의 개루프 전달 함수(Open-Loop Transfer Function) L(j\omega)는 일반적으로 수십 도(예: 45^\circ \sim 60^\circ) 이상의 위상 여유(Phase Margin, PM)를 확보하도록 PID 또는 LQR 형태의 제어기법을 통해 튜닝(Tuning)된다.
위상 여유는 이득 교차 주파수(Gain Crossover Frequency, \omega_{cp})에서 시스템의 위상과 나이퀴스트 안정도 판별을 위한 불안정 임계점인 -180^\circ (-\pi \text{ rad}) 간의 거리이다.

시간 지연 요소 e^{-s T_d} 가 삽입될 경우 새로운 위상 여유 PM_{new}는 다음과 같이 기존 위상 여유 PM_{old}에서 통신 지연으로 인한 위상 손실을 뺀 값이 된다. (도를 단위로 표기할 때)

PM_{new} = PM_{old} - \left( \omega_{cp} \times T_d \times \frac{180^\circ}{\pi} \right)

3.1 제어 발산(Divergence)의 예시

만약 기체의 위치 제어 대역폭(\omega_{cp})이 10 rad/s (약 1.6 Hz)로 다소 빠르고 공격적으로 튜닝되어 있고 기존 위상 여유가 60^\circ라고 가정하자.
GCS나 ROS2 자율 에이전트와의 간헐적 통신 지연 T_d가 0.2초(200ms) 발생하게 된다면, 잃어버리는 위상 각도는 10 \times 0.2 \times (180 / 3.14) \approx 114.6^\circ 에 달한다.
이렇게 되면 PM_{new} = 60^\circ - 114.6^\circ = -54.6^\circ 가 되어 시스템은 즉시 한계 사이클(Limit Cycle)을 넘어서 발산 진동(Unstable Oscillation) 구간으로 진입하고, 기체는 극심한 요동(Oscillation)과 궤적 이탈을 겪으며 추락하게 된다.

4. PX4 및 Ardupilot 체계에서의 시사점

통신 단절이나 극심한 지연은 곧 폐루프 시스템 내 T_d가 무한대로 수렴해가는 과정이며, 그 이전 단계의 과도 상태에서도 위상 여유는 음수(Negative) 영역으로 붕괴된다.

이러한 제어 공학적 근거로 인해 PX4-Autopilot과 Ardupilot는 다음과 같이 보수적인 통신 타임아웃 메커니즘을 적용한다.

  1. 타임아웃 파라미터(COM_OBL_LOSS_T, FS_GCS_ENABL)의 존재 가치: GCS 통신 지연이 길어지기 전에 능동적으로 외부 제어기(명령어 추종 루프)의 체계를 절단(Cut-off)하고, 기체 내부 센서에 의존하는 Failsafe 로컬 제어 루프로 천이하여 시스템 붕괴를 사전에 차단한다.
  2. MAVLink Offboard 모드 제약: PX4에서 ROS2 기반 오프보드 제어 시 초당 주기적 메시지(최소 2Hz 이상) 수신을 강제하는 이유는, 고대역폭을 요구하는 외부 궤도 발생기(Trajectory Generator) 연동 시 찰나의 시간 지연(T_d 증가) 조차도 기체 자세 및 궤도 발산이라는 심대한 위상 잠식을 유발하기 때문이다.
