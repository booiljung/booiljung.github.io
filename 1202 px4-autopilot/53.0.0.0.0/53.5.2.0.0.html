53.5.2 추측 항법(Dead Reckoning) 기반 단절 이후 기체 예상 궤적 벡터 시각화

지상 관제 시스템(GCS, Ground Control Station)과 무인기(UAV) 간의 통신 단절(Data Link Loss)은 예측 불가능한 환경 요인이나 하드웨어 결함으로 인해 빈번하게 발생할 수 있는 치명적인 상황이다. 이러한 상황에서 기체의 현재 위치 정보를 실시간으로 수신하지 못하면, 관제사는 기체의 거동을 파악할 수 없어 2차 사고의 위험에 노출된다. 본 장에서는 PX4-Autopilot(v1.14 기준)과 QGroundControl(v4.3 기준)을 기반으로, MAVLink 통신 단절 직후 마지막으로 수신된 상태 데이터를 활용하여 기체의 예상 궤적(Estimated Trajectory)을 추측 항법(Dead Reckoning)을 통해 계산하고, 이를 GCS 상에 시각화(Visualization)하는 아키텍처 및 구현 방식에 대해 심도 있게 다룬다.

1. 추측 항법(Dead Reckoning)의 개념과 시스템적 필요성

1.1 추측 항법(Dead Reckoning) 정의

추측 항법(Dead Reckoning)이란, 이동체의 과거 위치(Position), 속도(Velocity), 가속도(Acceleration), 자세(Attitude) 및 진행 방향(Heading) 정보를 바탕으로, 시간 변화에 따른 현재 물리적 이동 궤적을 미적분학적 모델링을 통해 추정하는 기법이다. 통상적으로 GPS와 같은 글로벌 포지셔닝 시스템의 신호가 끊기거나 지상 통신이 단절되었을 때, 최후의 알려진 상태(Last Known State)로부터 기체의 미래 위치를 산출하는 데 활용된다.

1.2 MAVLink 통신 단절 상황에서의 활용

GCS는 기체로부터 주기적으로 GLOBAL_POSITION_INT(전역 위치), LOCAL_POSITION_NED(지역 위치), HIGHRES_IMU(고해상도 관성 측정치) 및 ATTITUDE(자세) 메시지를 수신한다. 통신이 단절되면 GCS는 더 이상 상태를 갱신할 수 없게 되며, 이때 QGroundControl은 내장된 추측 항법 모듈을 활성화하여 지도(Map) 인터페이스 상에 기체의 예상 경로 벡터(Expected Path Vector)를 오버레이(Overlay) 렌더링한다. 이를 통해 관제사는 통신 복구(Re-link)를 위한 기체 수색 범위를 좁히거나, 안전 장치(Failsafe) 작동 전후의 기체 궤적을 직관적으로 파악할 수 있다.

2. 예상 궤적 추정 알고리즘 및 수학적 모델링

PX4-Autopilot의 확장 칼만 필터(EKF2, Extended Kalman Filter 2)는 기체 내부에서 정밀한 추정을 수행하지만, 통신이 단절된 GCS 측에서는 EKF2의 내부 공분산 데이터에 접근할 수 없다. 따라서 GCS는 자체적인 동역학 선형 모델(Linear Kinematic Model)을 사용하여 궤적을 추정한다.

2.1 등가속도 운동 모델(Constant Acceleration Model)

가장 보편적으로 GCS에서 사용하는 추측 항법 모델은 3차원 공간(NED 프레임)에서의 등가속도 운동 모델이다. 다음과 같은 수식으로 예측 위치 P(t)를 산출한다.

P(t) = P(t_0) + V(t_0) \Delta t + \frac{1}{2} A(t_0) \Delta t^2

여기서:

  • P(t_0): 마지막 통신 시점의 기체 위치 행렬
  • V(t_0): 마지막 수신된 기체 속도 벡터
  • A(t_0): 외력을 고려한 순 가속도 벡터
  • \Delta t: 단절 이후 경과 시간(Time since loss)

다만, 시간이 경과함에 따라 속도와 가속도의 미세한 오차가 적분되면서 시스템의 불확실성(Uncertainty) 반경이 기하급수적으로 커지는 한계(Drift Error)를 갖는다. 이를 보완하기 위해 GCS는 경과 시간에 비례하여 원근감 있게 반경이 확장되는 원뿔(Cone) 형태의 예상 지역을 지도에 렌더링한다.

3. QGroundControl 프론트엔드 및 백엔드 궤적 렌더링 아키텍처

QGroundControl에서의 시각화는 백엔드(C++)의 로직 연산과 프론트엔드(QML)의 그래픽 렌더링(Rendering) 분산 구조를 따른다.

3.1 백엔드(C++): 상태 추정 및 데이터 갱신

Vehicle 클래스와 통신 매니저 모듈은 MAVLink Link Loss를 감지하는 즉시 TrajectoryPredictor (가칭 사용자 정의 모듈 혹은 확장 모듈) 인스턴스를 실행한다. 이 클래스는 10Hz의 주기로 타임스탬프를 갱신하며 위도, 경도, 고도의 예상 좌표값을 산출한 후 시그널(Signal)을 발생시켜 프론트엔드로 전달한다.

3.2 프론트엔드(QML): FlightMap 컴포넌트 시각화 오버레이

QML 측의 FlightMap은 백엔드에서 생성된 일련의 QGeoCoordinate 리스트를 MapPolyline 또는 MapPolygon 객체로 변환하여 렌더링한다.

  • 예상 궤적 선(Trajectory Line): 단선 점선(Dotted Line)과 채도(Saturation)를 낮춘 색상(예: 반투명 노란색)을 사용하여, 실제 수신된 궤적(Solid Line)과 시각적으로 명확히 구분되도록 HMI(휴먼 머신 인터페이스) 관점에서 설계된다.
  • 불확실성 영역(Uncertainty Area): 경과 시간에 비례하여 넓어지는 영역을 MapCircle 컴포넌트나 불투명도(Opacity) 그라데이션이 적용된 이미지 오버레이로 구현하여 가시성을 확보한다.

3.3 시각화 데이터 플로우 (Mermaid 시퀀스 다이어그램)

4. ArduPilot (Mission Planner) 시스템과의 관제 아키텍처 비교

통신 단절 시 예상 궤적 시각화 처리에 있어서 PX4-Autopilot(QGroundControl)과 ArduPilot(Mission Planner 기준)은 지향하는 관점과 처리 방식에서 유의미한 차이를 보인다.

비교 항목PX4-Autopilot + QGroundControlArduPilot + Mission Planner
추측 항법 중심 관점모바일 및 태블릿 환경(Touch 기반)에 최적화된 QML 기반의 유연한 그래픽 오버레이 채택Windows Forms 기반으로 보수적이고 정적이지만 방대한 원시 텍스트 및 로그 시각화에 강점
불확실성 표현점선 궤적 및 다각형 오버레이를 통해 시각적으로 즉각적인 HMI(인지 공학적) 경고 제공통신 단절 아이콘 점멸 우선 및 마지막 위치 고정에 치중하여 예상 궤적 연산의 UI 개입 최소화
백엔드 리소스크로스 플랫폼을 위해 C++ 내 비동기 타이머를 활용하여 배터리 효율과 렌더링 부하 최소화PC의 CPU 자원을 충분히 활용한 로컬 GCS 로그 기반 사후 추적 분석에 더 적합

이러한 차이는 QGroundControl이 통신 단절과 같은 위급 상황에서 조종사(운용자)가 화면을 보고 직관적으로 기체의 현재 위치를 예상하고 즉각적인 대처를 할 수 있도록 인지 부하(Cognitive Load)를 줄이는 데 목적이 있음을 시사한다. 반면, ArduPilot의 Mission Planner는 사후 원인 분석 및 로그 복구 등 종합적인 데이터 시스템 유지보수에 더 큰 강점을 지닌다.

5. 기능 개발 시의 주의점 (ROS2 환경 연동 포함)

독자적인 GCS 환경 구축이나 분산관제 구조를 위해 ROS2 Micro XRCE-DDS 에이전트를 연동할 경우, GCS 내부의 추측 항법 시스템만으로는 부족할 수 있다.

  • 오프보드(Offboard) 모드로 자율 비행(Autonomous Flight) 중일 때는, 기체가 통신 단절을 인지하기 전까지 기존 경로를 유지하려는 관성을 가지므로, GCS는 기체 내부에 프로그래밍된 페일세이프(Failsafe) 궤적(예: Return, Hold)을 추측 항법 수학 모델에 사전에 반영(Feedback)해야 한다.
  • 단순히 등속도/등가속도 모델만 사용하면 기체가 페일세이프 모드로 진입하며 회항(Return) 경로로 선회할 때, GCS 지도상의 예상 궤적은 계속 직진하는 것으로 잘못 표기되어 관제사에게 혼란을 줄 수 있다.
  • 따라서 진보된 GCS의 추측 항법 모듈은 PX4 측의 페일세이프 파라미터(예: NAV_RCL_ACT, NAV_DLL_ACT)를 동기화하여, 단절 발생 n초 후 자동 회항 프로시저가 시작될 것까지 시뮬레이션하여 궤적을 렌더링하는 상태 기반(State-based) 추정을 적용해야 한다.

6. 요약

추측 항법(Dead Reckoning) 기반 기체 예상 궤적 렌더링 기술은 통신 단절이라는 최악의 상황 속에서 관제사에게 최소한의 정보망을 유지해주어 심리적 안정감을 제공함과 동시에, 빠른 수색 및 대처를 가능케 하는 핵심 HMI 로직이다. PX4-Autopilot의 상세한 텔레메트리 데이터를 GCS 내 C++ 백엔드에서 정밀하게 선형 예측 모델로 변환하고 시각화하는 과정은 무인항공기 관제의 신뢰성과 안전성을 보장하는 중요한 토대가 된다.
