1. 군집성 패킷 손실(Burst Packet Loss) 특성의 정의

실제 운용되는 무인항공기의 무선 통신 환경(예: 900MHz 텔레메트리, Wi-Fi, 5G 기반 LTE 통신망 등)에서 발생하는 MAVLink 패킷의 유실은 결코 통계학적으로 완벽히 독립적이고 균일한(Uniformly Distributed) 양상으로 나타나지 않는다. 비행체가 전파 방해 지역 구조물 뒤로 들어가거나, 셀 타워 핸드오버(Handover) 과정에 돌입할 때에는 필연적으로 수 십 개의 패킷이 연달아 소실되는 군집성 패킷 손실(Burst Packet Loss) 양상을 전개한다.

이때 단순한 선형적 손실률(Packet Loss Rate)만 관측하는 고전적 방식은, 패킷이 드문드문 빠지는 파편적 백색 잡음(White Noise) 환경과 통신 채널 자체가 먹통이 되어가는 구조적 장애(Physical Obstacle / Blockage) 환경을 제대로 분별해 내지 못한다. 따라서 GCS(QGroundControl)나 기체의 페일세이프(Failsafe) 제어기가 현재 통신 채널이 일시적 혼잡에 시달리는 것인지, 또는 영구적 단절(Blackout) 사태로 접어들었는지를 수학적으로 추정하기 위해 은닉 마르코프 모델(HMM, Hidden Markov Model) 이론이 각광받으며 채택된다.

2. Gilbert-Elliott (GE) 채널 모델 기반의 통신계 HMM 설계

디지털 통신 공학에서 군집성 에러를 명확히 모델링하기 위해 널리 활용되는 구조가 바로 HMM의 2-상태(Two-state) 특수 형태인 Gilbert-Elliott(GE) 모델이다. 무선 데이터 링크 스트림 속에 존재하는 실제 물리적 전파 채널의 보이지 않는(Hidden) 내부 상태를 다음과 같이 철저히 두 가지 은닉 상태(Hidden States)로 도식화한다.

  1. 상태 G (Good): 패킷 손실 확률이 매우 낮거나 0에 가까운, 안정성을 띤 정상 통신 궤적.
  2. 상태 B (Bad): 고강도 전파 간섭이나 물리적 차폐로 인해 패킷 손실 확률이 극도로 높으며, 한 번 진입하면 쉽게 벗어나지 못하는 군집성 에러 궤적.

그리고 이 모델 안에서 QGroundControl이나 무인기가 실제 눈으로 목격(Observation)할 수 있는 데이터는 오직 수신되는 MAVLink 패킷 시퀀스의 성공(1) 혹은 유실(0) 이분법뿐이다. GE 확률 모델은 시스템 상태 변화를 제어하기 위해 다음 네 가지의 핵심적인 상태 전이 확률(State Transition Probabilities) 파라미터로 구축된다.

  • P_{GB}: Good 상태에서 Bad 상태로 악화될 전이 확률
  • P_{BG}: Bad 상태에서 Good 상태로 회복될 전이 확률
  • P_{GG} = 1 - P_{GB} (Good 상태 잔류 확률)
  • P_{BB} = 1 - P_{BG} (Bad 군집 에러 상태 잔류 확률)

3. QGC 및 기체 백엔드 단의 베이지안 상태 추론 알고리즘

수신기에 도달하는 실제 패킷 시퀀스 관측 확률 열(Observation Sequence)을 O = \{o_1, o_2, ..., o_k\} 라 가정해 보자. 통제 시스템의 궁극적인 임무는, 현재 직면한 채널 상태가 극심한 군집적 파괴 구간인 B (Bad 상태)에 깊숙이 머무르고 있을 사후 확률(Posterior Probability) P(State=B | O) 를 즉각 도출하는 것이다. PX4-Autopilot의 FSM 백엔드 층위에서는 포워드(Forward) 알고리즘이나 비터비(Viterbi) 알고리즘의 무거운 연산망 대신, 단순화된 베이지안 평활 필터(Bayesian Smoothing Filter) 형태로 C++ 클래스 내에 이를 이식할 수 있다.

// HMM 기반 링크 상태 추정 파이프라인의 C++ 논리적 골격
double p_good = 1.0;
double p_bad = 0.0;

void update_hmm_based_channel_state(bool packet_received) {
    // 1. Prediction: 이전 상태들의 전이 확률을 통한 사전 확률 예상 도출
    double prior_good = p_good * PROB_GG + p_bad * PROB_BG;
    double prior_bad  = p_good * PROB_GB + p_bad * PROB_BB;
    
    // 2. Observation Model: 기하학적 분포 특성을 포함한 관측 확률 업데이트
    //    Good 상태에서 패킷 수신 성공률은 매우 높고, Bad 에서는 매우 낮음
    double prob_obs_given_good = packet_received ? 0.99 : 0.01;
    double prob_obs_given_bad  = packet_received ? 0.10 : 0.90;
    
    p_good = prior_good * prob_obs_given_good;
    p_bad  = prior_bad * prob_obs_given_bad;
    
    // 3. Normalization: 확률 범위를 0~1사이로 정규화 수렴
    double normalization_sum = p_good + p_bad;
    p_good /= normalization_sum;
    p_bad  /= normalization_sum;
}

위의 연쇄적 필터링을 거듭하면서, 은닉된 채널 훼손 확률 p_bad 가 특정 임계수준(예: 0.85)을 돌파하여 지속될 시점에 한해서만 일상적 간섭을 넘어선 “군집성 망각 상태(Burst Failure)“로 규명하고, Failsafe Escalation 로직의 방아쇠(Trigering)를 당기도록 허가한다.

4. Ardupilot 관제 패러다임 대비 PX4의 학술적 강건성 고도화 방향

Ardupilot 통제 생태계는 역사적으로 RADIO_STATUS 정보나 단순히 수신단을 통과하지 못한 패킷 수의 Lost Count 총합에 의지하여 정적인 판별 기준을 채용해 왔다. 그러나 저러한 기초적 구조 하에서는 \approx 10\%의 분산된 패킷 유실을 동반하는 환경과, \approx 10\%의 빈도지만 한 번 막힐 때 20개의 패킷이 무더기로 사장되는 장애 환경을 FC(Flight Controller) 상에서 지능적으로 구별할 겨를이 없어 치명적인 하드웨어 덤프 현상이나 불필요한 복귀 장치(Return To Launch, RTL) 발동을 남발하게 되었다.

반면에 현재 PX4 생태계가 고려하는 HMM 기반 상태 추론 모델은 오직 극단적이고 치명적인 “기하급수적 단절 패턴(Burst Pattern)“만이 임무 존폐에 위협이 되는 상황을 사전에 선별해 내는 최상위 필터링 역량을 가능케 한다. 즉, 데이터 링크의 연속적 스트림에 완벽하게 의존해야 하는 군집 비행 통제기(Swarm Controller)나 라우팅 추적 플랫폼이 언제 외부 개입(Operator intervention)을 포기하고 호버링(Hovering) 모드로 후퇴해야 할지 결정짓는, 고도로 정밀한 자율 역학(Autonomous Dynamics)의 수학적 권위와 신뢰성을 획득하는 근거이다.

5. 결론 및 공학적 함의 요약

결론적으로, 은닉 마르코프 모델(HMM) 기반의 통신 오류 상태 파이프라인의 설계 적용은 MAVLink 프로토콜상에서 송출되는 0과 1의 불완전한 꼬리표 데이터 한 조각을 단서 삼더라도, 보이지 않는 RF 물리 계층 환경의 거대한 열화 국면 전체를 정확하게 역추적(Back-tracking)해내는 놀라운 성과이다. 무인항공시스템이 직면한 공간적 및 전자기적 변동성을 깊이 추론해 냄으로써, GCS와 비행 객체 상단에 깔린 예외 처리(Exception Handling) 로직은 조건-반사(Reflex) 작용의 프레임을 깨뜨리고 마침내 진정한 의미의 “예측적 연계 회피 체계(Predictive Risk Evasion System)“의 단계로 거듭나게 된다.
