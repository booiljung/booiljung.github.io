53.2.2.1 지수 가중 이동 평균(Exponential Moving Average) 필터의 시정수(Time Constant) 및 평활화 계수(Smoothing Factor) 최적화

1. 개요 및 설계 변수 정의

GCS(QGroundControl)나 무인 기체의 페일세이프(Failsafe) 제어기에서 통신 단절(Link Loss)을 감지하는 동적 임계값 로직은 메모리 버퍼 요구량이 적고 계산 비용이 매우 저렴한 지수 가중 이동 평균(EMA, Exponential Moving Average) 필터를 주로 활용한다. EMA 필터는 제한된 마이크로컨트롤러(MCU) 리소스를 지닌 PX4-Autopilot의 펌웨어 환경과, 실시간으로 수십 개의 데이터 스트림을 비동기 처리해내야 하는 관제 애플리케이션의 백엔드에서 공통적으로 채택할 수 있는 이상적인 저주파 통과 필터(LPF) 방식이다.

이 필터 파이프라인(Filter Pipeline)의 신뢰성은 MAVLink 패킷 도달 간격의 측정치인 평균 주기(Mean) 및 편차(Deviation)를 얼마나 정밀하게 추정하는가에 달려있다. 이때 핵심적인 조율 파라미터로 작용하는 것이 측정 모델의 시정수(Time Constant) \tau와 평활화 계수(Smoothing Factor) \alpha이다.

2. 평활화 계수(\alpha)와 시정수(\tau)의 상관 관계 모델

연속적으로 수신되는 하트비트 데이터의 도달 간격을 x_k라 할 때, EMA 필터의 표준 점화식은 다음과 같이 정의된다.

y_k = \alpha \cdot x_k + (1 - \alpha) \cdot y_{k-1}

식에서 y_k는 새롭게 갱신된 평활화 평균값이고, \alpha (단, 0 < \alpha < 1)는 최근 측정된 패킷 지연치 x_k가 현재 평균 상태에 기여하는 비중을 뜻한다. 이를 연속 시간 영역(Continuous-Time Domain)의 1차 저주파 통과 필터의 시정수 \tau, 그리고 메시지 샘플링 주기 \Delta t와의 상관관계 구조로 맵핑(Mapping)하면 아래의 관계식을 얻을 수 있다.

\alpha = \frac{\Delta t}{\tau + \Delta t}

  • \alpha가 1에 가까울 때(작은 \tau 환경): 새로운 네트워크 지터(Jitter)나 혼잡에 의한 편차 데이터가 즉시 평균에 편입된다. 단절 감지의 반응성은 가장 빠르지만, 일시적인 지연 스파이크(Spike)에 필터가 과도하게 반응하여 Ardupilot의 고정 타임아웃 오류처럼 빈번하게 오탐지(False Positive)를 발생시킬 우려가 크다.
  • \alpha가 0에 가까울 때(큰 \tau 환경): 과거 데이터에 대한 필터의 관성이 강하게 작용하여 곡선이 매우 부드러워진다(Smoothing 극대화). 외란에는 강하지만, 실제 진정한 통신 단절이 발생했을 때 안전 장치를 촉발(Trigger)하기까지의 응답 지연(Response Latency)이 길어지는 치명적인 부작용이 있다.

그러므로 \alpha는 무선 주파수(RF) 네트워크 채널 구성이나 텔레메트리 대역폭 차이에 따라 신중하게 도출 및 실증(Validation) 되어야 한다.

3. PX4 및 QGC 기반 \alpha 튜닝 시스템 아키텍처

관제 환경은 하나의 고정된 평활화 상수를 텔레메트리, Wi-Fi, 5G 기반 LTE 망 등 이질적인 매체에 동일하게 적용하는 전통적 관행을 탈피해야 한다. 최신 QGroundControl의 통신 모델 동적 상태 감지를 최적화하기 위해 다음과 같은 제어 로직 설계를 도입할 수 있다.

3.1 이상치 거부(Outlier Rejection) 기반 동적 평활화 유도

순간적인 무선 노이즈 환경의 교란 탓에 혁신 항(현재 IAT와 평활화된 IAT의 편차)이 과도하게 커지는 경우, 일시적으로 \alpha를 낮춤으로써 임계값이 급격히 발산하는 것을 막는다. 관제 소프트웨어 계층(C++ 백엔드)에서의 개념적 적용 로직은 다음과 같이 구현될 수 있다.

// 1. 선행 지연 평균치와 현 시간 수신 간격 간의 혁신(Innovation) 도출
double innovation = std::abs(current_iat - ema_avg);

// 2. 오차 범위 기반 평활화 계수 동적 스위칭 (Dynamic Alpha Switching)
double current_alpha = BASE_ALPHA;
if (innovation > SPIKE_TOLERANCE_THRESHOLD) {
    // 거대한 네트워크 교란 스파이크 발견 - 과거 추세 유지에 집중
    current_alpha = BASE_ALPHA * 0.1;
}

// 3. 지수 가중 이동 평균 점화식 갱신
ema_avg = current_alpha * current_iat + (1.0 - current_alpha) * ema_avg;

3.2 이중화 통신 링크의 독립적인 시정수 유지

다중 에이전트 라우팅이나 이중화 통신 채널(Multi-link)을 동시에 다루는 아키텍처에서는 각 네트워크의 지터 특성이 다르다.

  • 장거리 텔레메트리망(예: 900MHz 계열): 셀 방식 대비 데이터 유실 구조가 비교적 일정하다. 지터가 낮으므로 반송파 간섭에 의한 오결정을 피하도록 \alpha를 적절한 저역 통과 구조로 세팅한다.
  • IP망 통신(LTE/5G): 기지국 간 핸드오버(Handover)나 네트워크 폭주 시 순간적인 지연 시간이 초 단위까지 커졌다가 정상으로 급복구되는 특징이 있다. 이럴 때 예외 처리 트랜지션이 동작하지 않도록, IP 기반 소켓 LinkManager 인스턴스에서는 더 작은 베이스 \alpha와 매우 강력한 디스파이킹(De-spiking) 로직 동기화가 요구된다.

4. 요약 결론

지수 가중 이동 평균(EMA) 필터의 시정수 \tau와 평활화 계수 \alpha의 튜닝 문제는 결과적으로 무인항공체의 이상 증세 반응 속도(Responsiveness)와 통신 간섭 속 노이즈 내성(Noise Immunity) 간의 트레이드오프(Trade-off) 밸런스를 확립하는 공학적 과정이다. 향상된 PX4-Autopilot 및 이를 제어하는 최신의 지상 관제 시스템 모델에서는 무선 매체의 채널 성격, 임무의 보안성 수준, 그리고 페일세이프 트리거 비용을 종합적으로 고려하여 동적 평활화를 실험적, 그리고 통계학적으로 검증해야 한다.
