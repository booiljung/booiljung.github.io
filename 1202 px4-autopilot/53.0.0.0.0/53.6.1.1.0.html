## 0.1 MAVLink 컴포넌트 식별자(Component ID) 충돌 방지를 위한 의사 난수(Pseudorandom Number) 생성 기반 세션 토큰 발행

다수 무인기 제어(Swarm Control) 환경이나 여러 대의 지상 관제 시스템(GCS)이 동일한 로컬 네트워크(LAN) 혹은 RF 대역폭을 공유하는 다중 에이전트 환경에서 통신이 단절되었다가 복구될 경우, 가장 빈번하게 발생하는 문제는 네트워크 노드 간의 식별자 충돌(Identifier Clash)이다. MAVLink 프로토콜은 통신에 참여하는 각 노드를 8비트의 System ID(1~255)와 8비트의 Component ID(1~255)로 구분한다. 본 절에서는 통신 복구 시 QGroundControl과 PX4-Autopilot이 식별자 충돌을 회피하고 고유한 세션을 수립하기 위해 사용하는 의사 난수 기반의 세션 토큰 발행 아키텍처를 분석한다.

0.1.1 MAVLink 식별자 충돌의 위험성

기본적으로 PX4-Autopilot은 System ID = 1, Component ID = 1 (MAV_COMP_ID_AUTOPILOT1)을 사용하며, QGroundControl은 System ID = 255, Component ID = 190 (MAV_COMP_ID_MISSIONPLANNER)을 기본값으로 사용한다.

네트워크 상에 GCS가 한 대뿐이라면 기본값을 사용해도 문제가 없지만, 다음과 같은 통신 복구 시나리오에서는 치명적인 충돌이 발생할 수 있다.

  • 기존 GCS(A)가 통신이 끊어진 사이, 보조 GCS(B)가 동일한 System ID와 Component ID로 통신을 장악한 경우.
  • 이후 통신이 복구된 GCS(A)가 다시 패킷을 브로드캐스트하기 시작하면, 기체(PX4)는 GCS(A)와 GCS(B)에서 오는 명령을 동일한 발신자의 것으로 오인하여 시퀀스 번호(Sequence Number) 오류나 제어 혼선을 겪게 된다.

0.1.2 의사 난수 생성(PRNG) 기반의 동적 Component ID 할당

이러한 레이스 컨디션(Race Condition)과 식별자 충돌을 방지하기 위해, 고급 GCS 라우팅 아키텍처에서는 고정된 식별자를 버리고 의사 난수 생성기(Pseudorandom Number Generator, PRNG)를 활용한 동적 식별자 할당 기법을 채택한다. 이는 마치 TCP/IP 소켓 통신에서 동적 포트(Ephemeral Port)를 할당하는 것과 유사한 개념이다.

  1. 난수 기반 토큰 생성: QGroundControl(또는 관제 소프트웨어)은 시작 시 운영 체제의 엔트로피(예: /dev/urandom 또는 CPU 타임스탬프)를 시드(Seed)로 삼아 PRNG를 초기화한다.
  2. 동적 Component ID 매핑: 산출된 난수를 해싱(Hashing)하여 가용한 Component ID 풀(Pool) 내에서 충돌 확률이 극히 희박한 고유 ID를 GCS의 서브 컴포넌트(예: 카메라 매니저, 미션 플래너 등)에 동적으로 부여한다.
  3. 세션 검증 시그니처: GCS는 통신 복구 시 SYS_STATUS나 HEARTBEAT 패킷을 전송할 때, 할당받은 동적 ID를 발신자 주소로 명시함으로써, 드론이 기존의 난입한 GCS(B)와 복구된 GCS(A)를 완전히 다른 엔티티(Entity)로 인식하게 만든다.

0.1.3 통신 복구 시의 세션 토큰 교환 메커니즘 (MAVLink 2.0 환경)

MAVLink 2.0 프로토콜 레벨에서는 보안 서명(Security Signing) 기능과 결합하여, 단순한 Component ID를 넘어선 암호화된 세션 토큰(Session Token)을 활용하기도 한다. 통신 복구 핸드셰이크 시 플로우는 다음과 같이 전개된다.

[도표 설명] 통신 복구 시나리오에서 시스템/컴포넌트 ID 불일치를 통해 다중 관제권이 충돌하는 상황. 의사 난수를 기반으로 동적 ID를 할당해둔 덕분에, PX4 기체는 서로 다른 두 GCS 컨트롤러를 분리하여 인식할 수 있다.

만약 두 GCS가 모두 하드코딩된 정적 ID(255/190)를 사용했다면, 기체 내의 MAVLink 수신 버퍼(Receiver Buffer)는 시퀀스 카운터가 비정상적으로 뒤섞이는 현상(Packet Dropping / Sequence Misalignment)을 겪으며 치명적인 예외를 발생시킬 것이다.

0.1.4 설계 시 고려사항: 충돌 해결 도메인 (Collision Resolution Domain)

비동기 분산 메시징 시스템에서 의사 난수 기반 할당을 도입할 때는 다음과 같은 제어 공학 원칙이 요구된다.

  • 생일 문제(Birthday Paradox) 고려: GCS 컴포넌트 ID로 할당 가능한 범위가 8비트로 제한되어 있으므로(일부 예약된 ID 제외), 네트워크 내 활성 노드 수가 많아지면 난수를 사용하더라도 해시 충돌(Hash Collision)이 발생할 확률이 기하급수적으로 증가한다.
  • 초기 상태 조율(Initial State Negotiation): 통신이 처음 복구된 GCS는 타겟 드론에게 현재 네트워크 세그먼트에서 어느 MAVLink ID들이 점유(Occupied)되어 있는지 질의(MAV_CMD_REQUEST_MESSAGE)하고, 충돌이 감지되면 자신의 난수상태를 재생성(Reseeding)하여 새로운 토큰 ID로 다시 세션을 수립하는 적응형 로직(Adaptive Logic)을 내장해야 한다.

이와 같은 난수 생성 기반의 세션 식별 메커니즘은, 현대 전장이나 복잡한 스마트 시티(Smart City) 공역 내에서 다수의 UAV와 GCS가 동적으로 이합집산하는 애드혹(Ad-hoc) 네트워크 환경의 통신 단절 및 복구 시나리오에서, 시스템의 결함 허용(Fault Tolerance)과 신뢰성을 극대화하는 중추적 역할을 수행한다.
