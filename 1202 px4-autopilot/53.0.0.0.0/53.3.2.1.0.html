53.3.2.1 제어 주파수 대역폭과 임계 파라미터 간의 나이퀴스트 주파수(Nyquist Frequency) 매핑 관계성

본 절에서는 무인항공기(UAV)의 비행 제어 시스템에서 통신 단절을 판별하는 임계 파라미터(예: COM_DL_LOSS_T)와 지상 관제 시스템(GCS) 사이의 제어 주파수 대역폭(Control Frequency Bandwidth) 간의 관계를 신호 처리의 기본 원리인 나이퀴스트-섀넌 샘플링 정리(Nyquist-Shannon Sampling Theorem)를 통해 제어 공학적 시각에서 분석한다.

1. MAVLink 데이터 링크에서의 나이퀴스트 정리 적용

디지털 제어 시스템에서 아날로그 세계의 연속적인 동작 상태를 디지털 신호로 완벽하게 재구성하거나 제어하기 위해서는 대상 시스템의 최대 주파수 대역폭의 최소 2배 이상의 속도로 샘플링(Sampling) 및 제어 명령 송신이 이루어져야 한다. 이를 나이퀴스트 주파수(Nyquist Frequency)라고 한다.

수식으로 표현하면, 기체의 제어 대역폭을 f_{BW}라고 할 때, GCS와 기체 간의 연속적인 원격 제어(Teleoperation)나 상태 추정(State Estimation)을 유지하기 위한 최소 통신 주파수(MAVLink 메시지 전송률) f_{telemetry}는 다음과 같다.

f_{telemetry} \geq 2 \cdot f_{BW}

  • 제어 주파수 대역폭 (f_{BW}): 멀티로터의 경우 자세(Attitude) 제어 루프는 약 250Hz~400Hz, 위치(Position) 제어 루프는 50Hz 전후의 대역폭을 가진다. 하지만 GCS 통신 레벨에서의 고수준(High-level) 상태 모니터링 및 경로 제어 대역폭은 보통 1~5Hz 수준이다.
  • 통신 주기 (T_{telemetry}): T_{telemetry} = 1 / f_{telemetry} 이며, 통신 링크에서 Heartbeat나 제어 명령 패킷이 도착해야 하는 필수적인 주기이다.

2. 임계 파라미터(COM_DL_LOSS_T)와의 매핑 관계

PX4-Autopilot에서 데이터 링크 단절을 정의하는 파라미터 COM_DL_LOSS_T는 제어 주기 관점에서의 “버퍼(Buffer)” 또는 “최대 허용 데드 타임(Maximum Allowable Dead Time)“을 의미한다.

나이퀴스트 정리를 우회하여 해석하면, 통신 주기가 나이퀴스트 조건 2 \cdot f_{BW}를 만족하지 못하게 되는 순간부터 제어기의 엘리어싱(Aliasing)이 발생하며, 이는 GCS가 기체의 상태를 잘못 파악하거나 제어 지연으로 인한 불안정성(Instability)을 유발한다는 것을 의미한다.

따라서 COM_DL_LOSS_T는 단순히 “메시지가 안 들어오는 시간“이 아니라, 기체가 해당 지연 시간 동안 개루프 제어(Open-loop Control) 또는 내부 자율 제어 추측 항법(Dead Reckoning)으로 버틸 수 있는 제어 대역폭의 한계치로 매핑되어야 한다.

  • 관계식 측면에서 COM_DL_LOSS_T는 다음 정리를 만족하도록 보수적으로 설정된다.
    COM\_DL\_LOSS\_T \gg T_{telemetry}
    COM\_DL\_LOSS\_T \leq T_{diverge}
    (여기서 T_{diverge}는 기체가 GCS의 보정 없이 비행 한계나 안전 경로를 이탈하는 데 걸리는 시간이다.)

3. 통신 지연에 따른 제어 성능 및 엘리어싱(Aliasing) 효과 도식화

통신 주기가 나이퀴스트 조건을 위반했을 때 GCS 화면에 나타나는 상태의 위상 지연(Phase Lag)과 데이터 유실(Data Loss)이 시스템의 임계 파라미터를 어떻게 촉발하는지 도식화한 내용이다.

    G --> H[Telemetry Delay Accumulation]

    I -- No --> J[GCS Warning: High Latency]
    I -- Yes --> K[Trigger PX4 Failsafe Sequence]

4. 제어 모드에 따른 임계 파라미터의 동적 설계 (Ardupilot 비교 포함)

비행 모드(Flight Mode)에 따라 요구되는 제어 주파수 대역폭은 극명하게 다르다.

  1. 매뉴얼/수동 조종 모드 (Manual/Acro Mode):
    조종사의 RC/조이스틱 입력은 가장 높은 대역폭을 요구한다. 따라서 수동 제어 링크(RC Loss Timeout)의 경우 0.5초 정도로 매우 짧은 임계값을 가진 실시간 처리를 요한다. (PX4의 COM_RC_IN_MODE, COM_RC_LOSS_T 등 적용)
  2. 오프보드 제어 모드 (Offboard Mode):
    컴패니언 컴퓨터(Companion Computer)가 직접 위치 및 속도 명령을 전달하는 경우 제어 대역폭이 높다(최소 2Hz 이상). 따라서 통신 단절 한계치(COM_OF_LOSS_T)는 GCS보다 민감하게(예: 0.5초 ~ 1초) 설정되어야 제어 발산(Divergence)을 막을 수 있다.
  3. 지상 관제 시스템 자동 비행 모드 (Mission/Auto Mode):
    자율 비행 중에는 GCS 통신망의 역할이 상태 수신 및 상위 수준 개입으로 한정된다. 즉, 제어 대역폭 f_{BW}가 0(Zero)에 가깝게 낮아지므로 나이퀴스트 한계 주기 역시 길어진다. 이로 인해 임무 도중의 COM_DL_LOSS_T는 10초 이상의 긴 값으로 설정하여 시스템 강건성을 확보하는 설계가 공학적으로 정당화된다.

Ardupilot 환경에서도 FS_GCS_ENABLE 및 FS_OPTIONS 파라미터를 통해 Mission 모드 중에는 GCS 페일세이프 트리거를 유보(Continue with Mission in Auto Mode)하는 로직을 제공하며, 이는 위상적 제어 대역폭 변동 원리와 완전히 동일한 궤를 같이한다.

5. 소스 코드 레벨 매핑 구현

PX4-Autopilot의 내부 펌웨어에서는 이 관계성이 명시적인 수식으로 코드화되어 있지는 않으나, 각 모드별 Commander 루프 속도(Loop Rate) 설계 시 암묵적으로 나이퀴스트 정리가 적용되어 있다.

  • Offboard 제어 모듈에서는 setpoint_triplet이나 trajectory_setpoint 토픽을 수신할 때마다 타임스탬프를 검사하며, 설정된 샘플링 윈도우(COM_OF_LOSS_T) 내에 N개 이상의 패킷이 들어오지 않으면 적정 주기가 깨진 것으로 간주하여 위치 제어를 포기(Fallback)하는 타이머 구조체 코드가 존재한다.
  • 반면 GCS의 telemetry_status 검사는 백그라운드 태스크에서 여유롭게(약 1~2Hz 주기) 모니터링하여 CPU 컨텍스트 부하를 줄인다.

결론적으로, PX4의 COM_DL_LOSS_T와 같은 통신 단절 임계 파라미터는 단순한 소프트웨어 타이머를 넘어, 드론의 폐루프(Closed-loop) 제어 주파수 특성과 비행 생존 시간 한계를 수학적으로 연결짓는 필수적인 “연결 고리(Mapping Link)“로서 역할을 수행한다.
