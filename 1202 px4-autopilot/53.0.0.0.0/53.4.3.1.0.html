53.4.3.1. 발송 대기 중인 유향 비순환 그래프(DAG) 형태의 종속 명령어 집합 무효화 처리

자율 비행을 위한 지상 관제 시스템(GCS) 설계에서, 비행 제어를 위한 MAVLink 명령어들은 단순한 순차적 나열이 아닌, 명령어 간의 인과적 종속성(Causal Dependency)을 가지는 경우가 많다. 복잡한 임무 수행을 위해서는 특정 조건이 만족되었을 때만 다음 명령이 유효해지는 논리 체계가 필요한데, 이를 수학적으로 모델링하기 위해 명령어들의 종속 관계를 유향 비순환 그래프(Directed Acyclic Graph, DAG) 형태로 추상화할 수 있다.

통신 단절(Data Link Loss) 이벤트가 발생했을 때 이 DAG 트리 구조를 올바르게 무효화(Invalidation)하지 않으면, 통신이 잠시 회복된 찰나의 순간에 이전의 유효기간이 지난 명령들이 연쇄적으로 기체(PX4-Autopilot)로 쏟아져 들어가 심각한 오작동이나 예기치 않은 모드 전이를 유발할 수 있다.

1. 종속 명령어 집합의 DAG 추상화 모델링

예를 들어, GCS에서 오프보드(Offboard) 모드 진입 후 기체를 특정 위치로 이동시키는 스크립트 기반 매크로를 실행한다고 가정하자. 이 절차는 다음과 같은 명령의 DAG 흐름으로 구성될 수 있다.

  1. MAV_CMD_DO_SET_MODE (Offboard 모드 전환)
  2. SET_POSITION_TARGET_LOCAL_NED (목표 위치 A 설정)
  3. MAV_CMD_CONDITION_YAW (목표를 향한 요각 정렬)
  4. MAV_CMD_DO_MOUNT_CONTROL (카메라 짐벌 트리거)

위 명령어 집합에서 2번 명령어는 1번 명령어가 성공적으로 반영(ACK 수신)되어 기체가 Offboard 모드임을 보장할 때만 의미가 있다. 4번 명령어 역시 기체가 지정된 요각 구속 조건을 만족한 뒤에 수행되어야 한다. 이들은 각각의 노드(명령)와 방향성 간선(종속 조건)으로 묶인 DAG 트리를 형성하며, 하나의 Root 명령부터 Leaf 명령까지 일관된 트랜잭션(Transaction) 원자성을 보장해야 한다.

2. 통신 장애 시 DAG 기반 상태 무효화 로직

통신 단절이 감지되었을 때, 송신 대기열(Command Queue)에 남아있는 DAG 구조의 제어 명령들을 맹목적으로 유지하는 것은 매우 위험하다. GCS의 상태 머신은 타임아웃을 감지하는 즉시, 완료되지 않은 DAG 트리의 정점(Vertex)을 탐색하여 연쇄적인 초기화 체인(Invalidation Chain) 가동해야 한다.

  1. 단절 시점의 종속 상태 평가: 타임아웃 이벤트가 발생하면, GCS 메모리 내에 유지되는 미결(Pending) 상태의 명령어 노드를 식별한다.
  2. 역방향 트리 순회(Reverse Tree Traversal): 미결 노드에 의존성을 가지는(즉, 미결 노드로부터 나가는 간선을 가지는) 모든 하위 후손(Descendant) 노드들을 재귀적으로 추적한다.
  3. 상태 레지스터리 파기(Tearing Down): 추적된 모든 노드를 대기열에서 삭제하고, 각 명령에 연결된 비동기 콜백(Callback)이나 Promise 객체들을 ‘거부(Rejected)’ 또는 ‘취소됨(Cancelled)’ 예외 상태로 강제 전이시킨다.

이를 통해 무인항공기 관제 애플리케이션은 무의미한 재전송을 차단하고, MAVSDK 모듈 또는 QGroundControl의 내부 플러그인 컴포넌트들이 정상적인 예외 처리를 거쳐 유휴 상태(Idle)로 안전하게 회귀할 수 있게 돕는다.

3. Ardupilot과 PX4 제어 철학의 차이에 따른 무효화 기법 차이

Ardupilot은 전통적으로 지상국에서 웨이포인트를 배열 형태로 업로드한 후 AUTO 모드로 일괄 위임하는 방식이 주류를 이루었기 때문에, GCS 내부에서 실시간 제어 명령 간의 미세한 DAG 종속성을 관리할 필요성이 분산되어 있었다.

반면, 로스토픽(ROS Topic)이나 ROS2의 uXRCE-DDS 미들웨어를 통해 실시간 오프보드 제어를 수행하는 PX4-Autopilot 아키텍처에서는, GCS나 임무 컴퓨터가 매초 수십 헤르츠(Hz) 빈도로 제어 궤적을 갱신하고, 상태 조건을 반영하여 다음 행동을 지시하는 마이크로 매니지먼트가 훨씬 일반적이다. 이로 인해 PX4 기반의 고도화된 GCS 개발 시에는 텔레메트리 단절 시점의 명령어 스냅샷을 분석하고, 해당 DAG 덩어리를 통째로 파기(Flush)한 뒤, 시스템이 안전망(예: RTL 또는 Loiter) 모드로 진입하기를 기대하는 것이 올바른 설계 방향이다. 통신이 복구되었을 때는 이전의 DAG를 이어서 실행하는 것이 아니라, 새로운 현재 상태의 기하학적 백터 공간을 다시 파악하여 새로운 DAG 트리를 구성하고 발행해야만 제어 공학적 무결성(Integrity)을 유지할 수 있다.
