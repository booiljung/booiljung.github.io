53.1 통신 단절(Data Link Loss)의 시스템적 정의와 제어 공학적 영향 평가

현대의 무인항공기 시스템(UAS)은 기체 내부의 독립적인 자동 조종 장치(Autopilot)와 지상 관제 시스템(GCS), 혹은 ROS2 기반의 자율 에이전트(Autonomous Agent)가 유기적으로 통신하며 동작하는 분산 제어 시스템(Distributed Control System)의 특성을 띤다. 이러한 구조에서 ’통신 단절(Data Link Loss)’은 단순한 정보 표출의 누락을 넘어 기체의 생존성과 임무 수행 능력을 위협하는 중대한 시스템 결함으로 정의된다.

1. 통신 단절의 시스템적(Systemic) 정의

통신 단절은 기체의 텔레메트리(Telemetry) 송신 데이터와 GCS의 제어 및 상태 갱신(Update) 명령을 주고받는 RF(Radio Frequency) 또는 네트워크 경로의 연속성이 주어진 임계 시간(TimeoutThreshold) 이상 상실된 상태를 가리킨다.

PX4-Autopilot 및 Ardupilot 체계에서는 이러한 통신 연속성을 판별하는 주된 지표로 MAVLink(Micro Air Vehicle Link)의 HEARTBEAT 메시지를 사용한다.
PX4-Autopilot을 기준으로 한 통신 단절(Data Link Loss) 상태의 수학적/시스템적 정의는 다음과 같다.

T_{current} - T_{last\_heartbeat} > \text{COM\_DL\_LOSS\_T}

여기서 T_{current}는 비행 제어기(FCS: Flight Control System)의 현재 시각, T_{last\_heartbeat}는 GCS로부터 마지막으로 수신된 유효한 HEARTBEAT 메시지의 타임스탬프, COM_DL_LOSS_T는 통신 단절 발생 억제 및 허용에 관한 파라미터 임계값(PX4의 경우 기본적으로 10초)이다.

2. 제어 공학적 관점에서의 영향 평가

비행 제어 시스템(FCS)은 본질적으로 다양한 피드백 루프(Feedback Loop)를 가진 다변수 폐루프 제어(Closed-Loop Control) 시스템이다. 외부 관제 컴퓨터(GCS 또는 ROS2 에이전트)가 기체를 제어할 때, 관제 대상 시스템(Plant) 전체는 더욱 큰 스케일의 외부 폐루프(Outer Loop)를 형성하게 된다. 통신 단절은 바로 이 거시적 지령(Reference Command) 루프의 단선을 의미한다.

2.1 외부 설정점(Setpoint) 업데이트의 정지 및 영차 유지(Zero-Order Hold) 문제

Offboard 모드나 Guided 모드 체계에서 기체는 GCS나 외부 에이전트로부터 초당 일정 횟수 이상(예: ROS2 비행 시 최소 2Hz 이상)의 위치, 속도, 자세 제어 설정점을 지속적으로 요구한다.
통신이 단절될 경우, 내부 제어기(Inner Loop Controller)는 마지막으로 수신된 명령 수치를 그대로 유지(Zero-Order Hold)하며 이를 추종하려 시도한다. 만약 기체가 특정 방향으로 일정한 가속도 명령을 받고 있던 순간 통신이 유실된다면, 기체는 해당 속력을 물리적 한계점까지 지속적으로 증가시키는 등 치명적인 발산 상태(Divergence)에 빠질 위험이 크다.

2.2 시간 지연(Time Delay) 및 위상 여유(Phase Margin)의 축소

간헐적 통신 장애(Intermittent Link Loss) 등으로 인해 텔레메트리와 명령 사이에 큰 지연(Latency)이 발생하면, 이는 제어 모델의 라플라스 영역에서 e^{-s T_d} (여기서 T_d는 지연 시간)의 순수 시간 지연 요소로 작용한다.
순수 시간 지연은 시스템의 위상 여유(Phase Margin)를 \omega \cdot T_d 만큼 급격히 갉아먹으며 궤적 제어 루프를 불안정하게 만들어 궁극적으로 한계 사이클(Limit Cycle) 발생 및 진동(Oscillation)을 유발할 수 있다.

3. 시스템 아키텍처 및 설정 파라미터에서의 조치 메커니즘

3.1 PX4-Autopilot의 아키텍처적 대응

PX4는 이러한 제어 공학적 불안정성을 극복하기 위해 통신 단절을 인지하는 즉시 제어 모드를 동결(Zero-Order Hold 상태 지속 방지)하고 내부 루프만으로 자율적인 Failsafe 상태 머신(State Machine)으로 강제 진입시킨다.

  • RTL(Return To Launch): 마지막으로 설정된 홈(Home) 위치나 집결지(Rally Point)로 내부 내비게이터(Navigator)를 활성화시켜 귀환.
  • Loiter (체공): 현재 위치에서 제자리 비행하며 통신 재연결을 대기. 해당 상태에서도 복구되지 않으면 배터리 잔량을 평가하여 자동 착륙(Land) 모드로 천이.

3.2 Ardupilot과의 차별점

  • PX4-Autopilot: Offboard 제어(컴퓨터 비전, MAVSDK, ROS2)에 특화된 Failsafe 타임아웃 파라미터를 갖춘다(COM_OBL_LOSS_T). 이는 일반 GCS 통신 장애(COM_DL_LOSS_T)보다 타임아웃이 일반적으로 훨씬 짧게(기본 0.5~1초) 설정되어, 고속 기동 중 외부 에이전트 통신 유실로 인한 심각한 궤도 이탈을 적극적으로 방지한다.
  • Ardupilot: FS_GCS_ENABL 파라미터를 통해 GCS 통신 장애 대비를 활성화하며, 기본 타임아웃은 보통 1.5초(설정에 따라 가변적)이다. Ardupilot는 EKF3 기반의 상태 추정을 바탕으로 더 보수적인 GCS 유실 대비 자동 경로 천이 동작을 지원한다.

결론적으로, 통신 단절은 단순한 연결 해제 정보가 아니라 능동적인 제어명령 갱신 중단(Command Starvation) 상태이므로, 이에 직면한 무인기 시스템은 반드시 제어권을 내부 자동 비행 FSM(Finite State Machine)으로 신속히 넘겨 보수적이고 안전한 생존 기동(Surviving Maneuver)을 최우선으로 실행해야 한다.
