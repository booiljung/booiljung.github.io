53.1.3.1 상·하향 링크 간의 다중 경로 페이딩(Multipath Fading) 상호 연관성 부재(Lack of Cross-Correlation) 분석

1. 개요 및 배경

무인항공기(UAV) 운영 환경, 특히 저고도 점대점(Point-to-Point) 통신에서는 지형, 건물 구조물, 해수면 등에 의해 반사되거나 회절된 통신 전파가 여러 경로로 도달하는 다중 경로 페이딩(Multipath Fading)이 빈번하게 일어난다. 무선 채널의 동적인 특성으로 인해 GCS에서 기체로 향하는 상향 링크(Uplink)와 기체에서 GCS로 내려오는 하향 링크(Downlink)가 경험하는 페이딩 환경은 근본적으로 다르다.

이 절에서는 다중 경로 페이딩 환경에서 상·하향 링크 간의 채널 상호 연관성이 부재(Lack of Cross-Correlation)하는 원인을 식별하고, 이러한 비상관성이 PX4-Autopilot의 관제 시스템에 미치는 영향 및 QGroundControl(QGC) 내 평가 로직의 한계와 대응 구조를 분석한다.

2. 상호 연관성 부재의 원인 분석

드론과 GCS 간 통신 시스템에서 상·하향 링크 주파수는 시분할 이중화(TDD, Time Division Duplexing) 또는 주파수 분할 이중화(FDD, Frequency Division Duplexing) 방식으로 운용된다.

  • FDD 환경의 주파수 이격(Frequency Separation): 송수신 주파수가 채널의 상관 대역폭(Coherence Bandwidth) 이상으로 떨어져 있을 때, 두 통신 대역이 겪는 페이딩 패턴은 완전히 독립적으로 변화한다.
  • TDD 환경의 시간 변화(Time Variance): 동일 주파수를 사용하더라도 송수신 시점의 차이가 채널의 상관 시간(Coherence Time)을 초과하게 되면(기체의 고속 이동에 따른 도플러 편이(Doppler Shift) 효과 등) 채널 응답이 급격히 변동한다.
  • 안테나 방사 패턴 및 위치 차이: 기체에 장착된 평면 안테나의 다이폴 방사 특성과 GCS의 지향성 패치 안테나의 공간적 차이 역시 송수신 이득의 비대칭성을 야기한다.

이러한 요인들 때문에, “하향 링크(텔레메트리) 상태가 양호하다고 해서 상향 링크(제어 명령) 상태도 양호할 것“이라는 가정은 성립하지 않는다.

3. PX4 관제 시스템 및 MAVLink 통신에 미치는 영향

상호 연관성 부재는 GCS 운영자 및 자율 비행 관제 프로세스에 명확한 불확실성(Uncertainty)을 초래한다.

3.1 GCS의 RSSI(수신 신호 강도) 평가 한계

QGroundControl은 기체로부터 수신되는 RADIO_STATUS (Message ID: #109) 메시지와 로컬 수신기의 RSSI 값을 결합하여 텔레메트리 링크 품질을 표출한다. 그러나 하향 링크 데이터 스트림 기반의 RSSI 지수로는 상향 링크의 패킷 드롭률 추정이 불가능하다.

graph TD
    A[GCS Transmits COMMAND_LONG] -->|Uplink Fade Area| B(Command Dropped)
    C[Vehicle Transmits SYS_STATUS] -->|Downlink Clear Area| D[GCS Receives Telemetry]
    D --> E[GCS UI Shows 90% Signal Strength]
    E -.-> |Illusion of Control| A
    B -.-> |Vehicle does not switch mode| F[State Machine Mismatch]

위 Mermaid 도표와 같이 GCS UI에서는 신호 강도가 90% 인 것으로 표출됨에도 불구하고, 기체는 상향 링크 페이딩 영역에 진입하여 MAVLink 제어 명령 패킷을 잃어버리는 일루전(Illusion) 상황이 발생하게 된다.

3.2 PX4-Autopilot 소스 코드 내 MAVLink Receiver 수신 평가

PX4 펌웨어의 mavlink_receiver.cpp 에서는 MAVLink 메시지의 시퀀스 누락을 탐지하여 링크 에러를 산출한다. 하지만 다중 경로 페이딩에 기인한 일시적인 상향 링크 단절 상황에서는 PX4 측 수신 로그에만 통신 불량(Packet Drop)이 마킹되고, 이 정보가 역으로 GCS에 즉각 반영되지 않아 동기화 단절을 겪게 된다.

// src/modules/mavlink/mavlink_receiver.cpp
void MavlinkReceiver::handle_message(mavlink_message_t *msg)
{
    // 이전 등록된 시퀀스 넘버와 비교 (페이딩 구간 내 패킷 손실 감지)
    if (_last_sequence_num != 0 && (msg->seq - _last_sequence_num) > 1) {
        _dropped_packets += (msg->seq - _last_sequence_num - 1);
        // 드롭된 패킷 누적은 PX4 기체의 자체 판단용으로 사용됨
    }
    _last_sequence_num = msg->seq;
}

Ardupilot 대비 차이: Ardupilot 역시 시퀀스 감사를 기반으로 수신 품질을 산출하나, PX4-Autopilot은 이 드롭율 정보를 telemetry_status uORB 토픽에 발행하여 내부 FSM(Failsafe State Machine)에서 보다 밀접하게 가중치 평가 수단으로 활용한다.

4. QGroundControl 관제 대응 알고리즘

하향 링크의 양호함이 상향 링크의 생존성을 보장하지 않기 때문에, QGroundControl은 이 비상관성을 보완하기 위한 응용 계층의 상태 기반 제어(State-based Control) 논리에 의존한다.

  1. 상태 피드백(State Feedback) 필수화:
    QGC 내 Vehicle.cc는 COMMAND_LONG 셋업 후, 하향 링크에서 주기적으로 송출되는 HEARTBEAT이나 SYS_STATUS의 비행 모드(Mode) 바이트가 실제로 변경되었는지를 대조(Verify)한다. 상향 링크가 상실되어 명령이 드롭되었다면, GCS 측 재전송(Retry) 로직이 즉각 액션을 다시 요청한다.
  2. MAVLink ACK 타임아웃 큐잉 체계:
    QGC 백엔드에서는 특정 주요 트랜잭션(예: Arming, Mission Upload) 시 로컬 타이머 큐를 생성하여 다중 경로 페이딩의 상관 시간(Coherence Time, 약 수백 ms) 초과 타이밍에 2차 패킷을 밀어 넣도록 시간 다양성(Time Diversity) 기법과 유사한 재전송 시간차 로직을 적용한다.

5. 결론 및 공학적 시사점

서로 독립적인 다중 경로 페이딩을 겪는 비대칭 채널 환경은 PX4-Autopilot 관제 신뢰도 저하의 숨은 원인이 된다. 관제 애플리케이션이나 자율 에이전트를 개발할 때, RSSI 지표에 전적으로 의존하는 것은 위험하다. 무인기 시스템 설계자는 MAVLink의 ACK/NACK 트랜잭션 피드백과 동기화 타임아웃(Synchronization Timeout)을 응용 로직상에 엄격히 캡슐화해야 하며, ROS2 미들웨어(uXRCE-DDS) 적용 시 제어 토픽에 대해 Best Effort가 아닌 Reliable QoS 프로토콜을 반드시 지정하여 상·하향 링크의 상호 연관성 부재를 프로토콜 계층에서 상쇄하라.
