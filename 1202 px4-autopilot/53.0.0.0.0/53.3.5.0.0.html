53.3.5. 기구학적 제약(Kinematic Constraints) 기반 최적 귀환 궤적(Optimal Return Trajectory) 생성

통신 단절(Data Link Loss) 혹은 치명적인 배터리 부족 상황이 발생하면 무인항공기(UAV)는 안전을 확보하기 위해 홈 위치(Home Position)나 랠리 포인트(Rally Point)로 귀환을 시도한다. 이때 기체가 단순히 직선으로 이동하는 것은 물리적으로 불가능하거나 바람직하지 않을 수 있다. 비행체의 형태(멀티로터, 고정익, VTOL)에 따라 지닐 수밖에 없는 물리적인 한계값들을 **기구학적 제약(Kinematic Constraints)**이라 한다. PX4-Autopilot은 이러한 기구학적 제약을 고려하여 에너지를 최소화하고 물리적 부하를 줄이면서 도달 가능성을 극대화하는 **최적 귀환 궤적(Optimal Return Trajectory)**을 생성한다. 본 절에서는 최신 PX4-Autopilot(v1.14 이상)의 소스 코드를 바탕으로 이 궤적 생성 알고리즘과 아키텍처에 대해 상세히 분석한다.

1. 기구학적 제약(Kinematic Constraints)의 공학적 정의

비행 제어 및 경로 계획(Path Planning) 분야에서 기구학적 제약이란, 물체의 운동 속도, 가속도, 회전율과 같이 시간에 따른 위치 변화율에 대한 물리적인 상한과 하한을 의미한다.

  • 멀티로터(Multicopter):
  • 최대 상하 속도(Max Ascent/Descent Rate), 최대 수평 이동 속도, 저크(Jerk, 가속도의 미분값) 한계가 있다.
  • 최대 틸트각(Tilt Angle) 설정에 의한 측면 가속도(Lateral Acceleration)의 한계를 준수해야 한다.
  • 고정익(Fixed-wing):
  • 최소 비행 속도인 실속 속도(Stall Speed), 선회 시 요구되는 최소 회전 곡률 반경(Minimum Radius of Curvature) 제약을 극복해야 한다.
  • 최대 뱅크각(Max Bank Angle)과 피치각(Pitch Angle)을 제한해야 한다.
  • VTOL(Vertical Take-Off and Landing):
  • 호버(Hover) 모드와 전진 비행(Forward Flight) 모드 간의 천이(Transition) 구간에서의 공기역학적 속도 대역 한계를 고려해야 한다.

PX4 모델에서는 이러한 제약조건들을 MPC_* (Multicopter Position Control), FW_* (Fixed-wing Control) 파라미터를 통해 사용자가 설정할 수 있도록 시스템화되어 있다.

2. PX4-Autopilot의 최적 귀환 궤적(Optimal Return Trajectory) 생성 아키텍처

PX4에서 귀환 모드(Return Mode)가 활성화되면, 상태 머신을 관장하는 Navigator 내의 RTL(Return To Launch) 모듈이 작동을 시작한다. 최근에는 경로 추종 가이던스(Path Following Guidance)를 위해 NPFG (Non-linear Path Following Guidance) 알고리즘이 적용되고 있다.

2.1 Ardupilot과의 궤적 처리 아키텍처 비교

  • PX4-Autopilot: 귀환을 하나의 비행 작업(Flight Task)인 FlightTaskReturn 클래스로 분리하여 관리한다. 귀환의 각 단계(상승, 순항, 하강, 착륙)에서 위치와 속도의 궤적을 3차 베지어 곡선(Bézier Curve)이나 저크 제한(Jerk-limited) 궤적 생성기(Trajectory Generator)를 통해 부드러운 위치 제어 목표(Setpoint)로 산출한다.
  • Ardupilot: ModeRTL 객체를 통해 L1 내비게이션 컨트롤러(L1 Navigation Controller)를 활용해 홈 위치를 목표점으로 삼아 이동한다. 이동 중에 웨이포인트(Waypoint) 추종 알고리즘을 그대로 가져다 쓰면서 홈 주변에서 궤도를 원형으로 그리는 배회(Loiter) 처리에 초점이 맞춰져 있다.

두 펌웨어 모두 안전한 귀환을 지원하나, 최신의 PX4는 수리적 최적화 기반으로 기체의 흔들림을 없애기 위해 저크와 가속도를 억제한 수식 기반의 궤적 프로파일링(Profiling)에 특화된 경향을 띤다.

2.2 궤적 플래닝 상태 머신(Trajectory Planning State Machine) 설계

귀환 명령 시퀀스는 기체의 현재 고도 및 설정에 따라 고도를 상승시킨 후 목적지로 수평 이동시키는 3차원 플래닝으로 진행된다. 다음은 귀환 궤적 생성의 생명 주기를 표현한 소프트웨어 아키텍처 다이어그램이다.

stateDiagram-v2
    direction TB
    [*] --> RTL_TRIGGERED : 통신 단절 감지(Data Link Loss)

    state RTL_TRIGGERED {
        [*] --> InitialClimb
        InitialClimb --> ReturnCruise : 고도가 RTL_RETURN_ALT 이상 확보됨
        ReturnCruise --> LoiterDown : 목적지 상공 도달
        LoiterDown --> Landing : RTL_DESCEND_ALT에서 하강 착륙 준비 완료
        Landing --> [*]
    }

    InitialClimb : 최대 수직 가속도 한계(Jerk-limited) 이내로 부드러운 상승 궤적 산출
    ReturnCruise : 최적 비행 속도(Optimal Cruise Speed) 기반 직선 혹은 스플라인 경로 생성
    LoiterDown : 나선형(Spiral) 하강을 통한 운동에너지 감소 궤적 경로 추종
    Landing : 지면 포착(Land Detector)까지 최저 속력 하강 비행

3. 기구학적 기반 궤적 최적화 제어 알고리즘 및 소스 코드의 이해

귀환 궤적이 단순히 최단 직선 경로(L_2 Norm 기반)만 따르게 되면 요구되는 특정 뱅크각이나 추력이 모터의 기계적 한계를 넘어서 과부하가 걸리거나 기체가 실속(Stall)에 빠질 수 있다. 이를 방지하는 최적화 추종 로직의 수리적 모델은 다음과 같다.

3.1 저크 제한 기반 스무딩 제어 로직(Jerk-Limited Smoothing Logic)

위치를 최적화하는 데 있어 저크(Jerk)의 크기를 최소화하는 방법론인 최소 저크 궤적(Minimum Jerk Trajectory) 생성법을 차용한다. 논문 “Minimum Snap Trajectory Generation and Control for Quadrotors” 에 설명된 이론처럼, 포지션 p(t) 가 주어졌을 때 모터 부하를 최소화하는 비용 함수(Cost Function) J 를 산출한다.

비용 함수 최적화 공식 수식 블록
J = \int_{0}^{T} \left\vert \frac{d^3 p(t)}{dt^3} \right\vert^2 dt

PX4 내부의 FlightTaskReturn은 실시간으로 이 적분 값을 수치적으로 계산할 여유가 없기 때문에, 휴리스틱 기반의 사다리꼴 가속도 프로파일(Trapezoidal Acceleration Profile)을 사용하여 물리적 제약 내에서의 위치 목표값 집합 유닛인 vehicle_local_position_setpoint 을 생성한다.

3.2 고정익/VTOL 경로 추종에서의 궤적 굽힘(Corner Path Bending) 설계

고정익은 반드시 직진성이 있어야 기체의 양력(Lift Force)을 발생시킨다. 따라서 귀환 시 코너를 도는 궤적에서 두빈스 경로(Dubins Path) 알고리즘을 차용하여 최소 선회 반경 제약을 지킨다. 이를 처리하는 MAVLink 메시지는 POSITION_TARGET_LOCAL_NED이며, 해당 메시지는 가이던스 모듈을 거쳐 내부 제어기가 안정적으로 목적지를 따라가게 설계되어 있다.

4. QGroundControl 연동 및 ROS2 기반 자율 에이전트 관제 제어

4.1 QGC 관련 Failsafe 구성 및 동작 상태 검증

QGroundControl(버전 4.3.x)에서 Failsafe 탭을 통해 RTL 설정 파라미터에 접근하여 GCS 관제 요원이 궤적 제약을 제어할 수 있다. 기구학적 제약에 직접 영향을 주기 위해 사용자가 조절할 수 있는 핵심 제어 파라미터는 다음과 같다.

  • RTL_RETURN_ALT: 홈으로 복귀하기 전 상승할 최소 안전 고도 값이다.
  • RTL_DESCEND_ALT: 홈 상공에서 배회(Loiter)하며 강하를 수행할 전환 고도 값이다.
  • RTL_CONE_ANG: 기체가 목적지에 다가갈수록 고도를 부드럽게 낮추도록 만드는 안전 원뿔 각도(Cone Angle) 제약조건이다.

4.2 MAVLink 및 ROS2를 활용한 궤적 시뮬레이션 제어

GCS 애플리케이션 개발자는 GCS 혹은 외부 컴패니언 컴퓨터(Companion Computer)에서 기체를 인위적으로 귀환시키고자 할 때 COMMAND_LONG 메시지를 이용해 MAV_CMD_NAV_RETURN_TO_LAUNCH 커맨드를 전송한다. 기체측에 전송된 명령은 내부의 유한 상태 기계를 전환시키며 귀환 절차를 발동시킨다. 이때, GCS 애플리케이션에서는 기체의 복귀 진행 상황을 파악하기 위해 NAV_CONTROLLER_OUTPUT 메시지 내의 wp_dist (웨이포인트까지의 거리)와 target_bearing을 수신하여 사용자 인터페이스에 실시간으로 표시한다.

ROS2 uXRCE-DDS 브릿지 구조에서는 이러한 귀환 경로를 실시간으로 모니터링하기 위해 VehicleTrajectoryWaypoint 및 VehicleLocalPositionSetpoint 메시지를 서브스크라이빙 시켜 귀환 중 기체가 그리는 3차원 궤적들을 외부 시각화 노드로 전송한다. 멀티 에이전트나 자율 회피 성능을 향상시키기 위한 Offboard 설정에서는, 자체 개발된 ROS2 경로 계획(Path Planning) 컴퓨터 비전 노드 측에서 이 수학적 제약을 함께 연산하여 안전한 우회로를 토픽으로 푸시해주는 방법이 권장된다.
