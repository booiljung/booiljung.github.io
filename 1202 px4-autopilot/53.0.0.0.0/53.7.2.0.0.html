53.7.2 링크 전환 시 전송 계층(Transport Layer) 세션 연속성 확보 방안

이기종 네트워크(예: 텔레메트리 라디오와 LTE/5G) 간에 물리적인 링크 전환(Link Handover)이 발생할 때, 기존 확립된 통신 세션이 단절되지 않고 유지되도록 보장하는 것은 무인항공기 관제의 핵심 성능 지표 중 하나이다. MAVLink 자체가 프로토콜 계층 내에서 시퀀스(Sequence Number)를 관리하긴 하지만, 그 하위의 전송 계층(Transport Layer) 연결이 끊어지면 지상 관제 시스템(GCS)과 PX4-Autopilot 간의 재접속 시퀀스로 인해 긴 레이턴시(Latency)가 발생할 수 있다. 본 절에서는 이러한 링크 전환 상황에서 전송 계층의 세션 연속성을 확보하는 아키텍처적 방안을 다룬다.

1. 전송 계층 단절이 초래하는 문제점

데이터 링크 통신 단절 시 하위 소켓 연결이 해제되면, 다음과 같은 제어 공학적 문제가 비행 시스템에 발생한다.

  1. 핸드셰이크 지연(Handshake Delay): TCP 기반 링크를 운용할 경우(예: 고해상도 이미지 전송) 재접속 시 3-Way Handshaking 과정에 의해 추가적인 지연 시간이 발생한다.
  2. MAVLink 스트림 재협상(Stream Renegotiation): QGroundControl(QGC)과 PX4 간의 하위 소켓 바인딩이 새로 이루어지면, QGC는 기체의 세부 파라미터나 임무(Mission) 리스트를 처음부터 다시 다운로드하려고 시도한다. 이는 네트워크 대역폭을 급격히 고갈시키며 기체 통제 불능 시간을 연장한다.
  3. ROS2 DDS 노드 이탈: Micro XRCE-DDS 클라이언트(Client)와 에이전트(Agent) 간의 UDP 세션 타임아웃이 발생하면 로스(ROS2) 네트워크 토픽 브로커에서 해당 기체 노드가 오프라인 처리되어 자율 비행(Offboard) 제어 명령이 전달되지 않는다.

2. UDP 하트비트(Heartbeat) 기반 논리적 세션 유지 메커니즘

PX4와 QGroundControl(v4.3 이상)은 MAVLink 프로토콜의 특수성을 고려하여, 비동기(UDP) 세션 위에서 논리적 상태(Logical State)를 관리하는 방식을 주로 채택한다. UDP는 본질적으로 비연결성(Connectionless) 지향 프로토콜이므로, IP 주소나 포트가 일시적으로 변경되더라도 QGC 내부의 LinkManager는 새로운 패킷을 기존 Vehicle 객체와 매핑할 수 있다.

    Note over MR, QGC: Link A 환경 악화 (Switch to B)
    MR--)QGC: [Link B: LTE] 새로운 IP로 UDP 전송
    QGC->>QGC: 패킷의 System ID 확인
    Note over QGC: 기존 Vehicle 세션과 일치 판별
    QGC-->>PX4: 응답 패킷을 새로운 도착지 IP로 갱신하여 송출

QGC는 들어오는 모든 MAVLink 패킷 디코딩 과정에서 System ID와 Component ID를 식별한다. 하위 TCP 소켓이 끊어지거나 UDP 출발지 포트가 달라지더라도 식별자(ID)가 동일하고 시퀀스가 이어질 경우, 전송 계층의 단절 여부와 무관하게 MAVLink 세션을 연속된 것으로 간주(Session Continuation)하도록 설계되어 있다. 이는 세션 지향 라우팅에 얽매이는 Ardupilot의 일부 구형 GCS 환경(예: 고정 소켓 바인딩을 강제하는 스크립트) 대비 유연한 장점이다.

3. 고급 전송 계층 세션 보존 기술

단순 UDP 매핑을 넘어 VPN이나 ROS2 토픽 수준의 강력한 세션 연속성을 확보하기 위해 다음과 같은 전송 계층 보존 기법이 사용된다.

3.1 MPTCP(Multipath TCP)를 통한 전송망 융합

운영체제(OS) 수준에서 2개 인터페이스(예: eth0, wlan0)를 단일 TCP 소켓에 매핑하는 MPTCP(Multipath TCP) 기술을 기반으로 터널링을 구성하면 완벽한 무중단(Hitless) 페일오버가 구현된다. TCP 서브플로우(Subflow) 중 하나가 물리적으로 끊겨도 다른 서브플로우가 즉각적으로 소켓 번호를 승계하므로 애플리케이션(QGC, PX4 프로세스)은 임의의 단절을 인지하지 못한다.

3.2 고정 IP 할당을 위한 모바일 VPN(Virtual Private Network)

이동 중인 기체의 LTE 모뎀은 기지국(Cell)을 이동할 때마다 통신사로부터 새로운 동적 사설 IP를 할당받는다. 이는 라우팅 테이블 변경을 동반하며 전송 계층 단절을 유발한다. 이를 회피하기 위해 Tailscale 혹은 WireGuard와 같은 VPN 데몬을 기체 탑재 컴퓨터(Companion Computer)에 적용하라.

  1. 가상 인터페이스(Virtual Interface): tun0 가상 이더넷 어댑터가 생성되며, 변하지 않는 고유 사설망 IP 대역(10.x.x.x)을 할당받는다.
  2. 세션 유지 원리: 물리 인터넷망 IP 주소가 자주 변하더라도 UDP 데이터그램 위에서 패킷 캡슐화(Encapsulation)를 처리하므로, 실질적인 MAVLink 통신 애플리케이션 포트는 일관성 있는 tun0 가상 IP와만 통신하여 전송 계층 세션 단절 상황을 원천적으로 막는다.

4. PX4 및 ROS2 오프보드 제어 시 주의사항

ROS2 미들웨어 스택과 연동할 때 Micro XRCE-DDS 에이전트단의 UDPTransport 레이어 생존 점검은 필수적이다.

  • 클라이언트 타임아웃 튜닝: 세션 단절 후 재접속 시까지 허용되는 타임아웃 마진(Timeout Margin) 파라미터를 물리 링크 스위치오버 소요 시간보다 충분히 길게 확보하라. 기한 내 응답 실패 시 DDS 브로커가 세션을 파괴하여 자율(Offboard) 커맨드 송수신이 무효화될 수 있다.
  • Heartbeat 주파수 상향 조정: 이종망 링크 스위치오버 직후, 새로운 연결 경로 인지를 앞당기기 위해 일시적으로 HEARTBEAT 스트림 메시징 주기를 상향 조절(Burst 방식 송출)하도록 기체 상태 머신 메커니즘을 튜닝하는 전략이 필요하다.

5. 요약

링크 전환 상황에서 전송 계층 단절로 인한 레이턴시를 억제하는 솔루션은 비행 기체의 조종 권한 이탈을 방지하는 필수 체계이다.

  • QGC 내부의 MAVLink System ID 기반 비동기 UDP 라우팅 시스템 장점을 적용해 복잡한 세션 재생성 로직을 배제하라.
  • 이종망 간 매끄러운 핸드오버 체계를 위해 가상 인터페이스(VPN) 터널링으로 IP 대역을 고정하고 물리적인 소켓 절단 위험을 회피하라.
  • 위 설계 패턴은 GCS 레벨의 파라미터 재요청(Re-request) 트래픽 스톰을 방어하여 모바일 통신(LTE/5G) 실외 환경에서 운용 신뢰도를 높일 수 있다.
