46.4.2.1. 유클리드 놈(Euclidean Norm) 연산 기반 3차원 위치 오차 벡터(Position Error Vector) 스칼라 추출 및 센서 노이즈에 따른 정밀도 한계 분석

단일 목적지 이동(Goto) 제어 시 네비게이터(Navigator)가 기체의 목표 도달 여부를 판별하기 위해서는, 현재 위치와 목표 위치 간의 기하학적 차이를 정량적인 단일 스칼라(Scalar) 수치로 변환해야 한다. 다차원 공간에서의 다이내믹스(Dynamics)를 수학적으로 해석하기 위해 PX4-Autopilot은 유클리드 놈(Euclidean Norm) 연산을 기반으로 오차 벡터의 크기를 추출한다 (기반 펌웨어 버전: PX4-Autopilot v1.14 기준). 본 절에서는 이 위치 오차 벡터의 계산 방식과 센서 노이즈가 유발하는 제어 정밀도 한계를 분석한다.

1. 차원 위치 오차 벡터(Position Error Vector)의 계산

지상 관제 시스템(GCS)으로부터 전달받은 목표 좌표는 로컬 NED(North-East-Down) 직교 좌표계(Cartesian Coordinate System) 상의 목표 위치 벡터 \mathbf{P}_{target} = (x_{sp}, y_{sp}, z_{sp})^T로 표현된다. 기체의 확장 칼만 필터(EKF2)가 추정한 현재 위치 벡터 \mathbf{P}_{current} = (x, y, z)^T가 주어질 때, 위치 오차 벡터 \mathbf{E}_{pos}는 두 벡터의 차이로 정의된다.

\mathbf{E}_{pos} = \mathbf{P}_{current} - \mathbf{P}_{target} = \begin{bmatrix} x - x_{sp} \\ y - y_{sp} \\ z - z_{sp} \end{bmatrix} = \begin{bmatrix} e_x \\ e_y \\ e_z \end{bmatrix}

1.1 수평 오차 스칼라 추출: 유클리드 놈(Euclidean Norm)

PX4 도달 판별 알고리즘은 수직 축(Z)과 수평 평면(XY)을 분리하여 해석한다. 수평면 상의 오차 벡터 스칼라, 즉 두 점 사이의 최단 직선 거리는 L2-Norm(Euclidean Norm)을 사용하여 계산된다.

e_{xy} = \|\mathbf{E}_{xy}\|_2 = \sqrt{e_x^2 + e_y^2}

이 e_{xy} 스칼라 값이 사용자가 정의한 수용 반경 NAV_ACC_RAD 보다 작은지 여부가 2차원 평면에서의 1차적인 도달 조건이 된다. 내부 소스코드(예: mathlib 라이브러리의 Vector2f 객체) 상에서는 루트(sqrt) 연산을 매 사이클마다 수행하는 것이 임베디드 CPU 부하를 유발할 수 있으므로, 최적화를 위해 제곱합(Sum of Squares) 공간에서 곧바로 비교(e_x^2 + e_y^2 < R^2)하는 기법이 함께 병행되기도 한다.

2. 센서 노이즈(Sensor Noise) 특성에 따른 정밀도 한계(Precision Limit)

제어 이론적으로 유클리드 놈 수식이 0으로 수렴(e_{xy} \to 0)하는 것이 이상적이나, 실제 시스템에서는 **센서 노이즈(Sensor Noise)**와 **상태 추정기(Estimator) 드리프트(Drift)**가 근본적인 정밀도의 하한선(Lower Bound)을 강제한다.

기체의 현재 위치 \mathbf{P}_{current}는 EKF2 모듈이 제공하는 추정값(Estimate)이다. 일반적인 멀티로터 시스템은 GPS/GNSS 모듈의 삼변측량 오차와 IMU(관성 측정 장치)의 열 잡음(Thermal Noise) 등으로 인해 일정한 오차 공분산(Covariance)을 지닌다.
수학적으로 \mathbf{P}_{current}는 실 위치 \mathbf{P}_{true}와 가우시안 백색 잡음(Gaussian White Noise) \mathbf{w} \sim \mathcal{N}(0, \Sigma)의 합으로 나타낼 수 있다.

\|\mathbf{E}_{xy}\|_{measured} = \|\mathbf{P}_{true} + \mathbf{w}_{xy} - \mathbf{P}_{target}\|_2

2.1 노이즈 증폭(Noise Amplification)과 수용 반경(Acceptance Radius)의 상관관계

만약 기체가 실제로 목표 수평 좌표에 완벽히 정지해 있다고 가정하면(\mathbf{P}_{true} = \mathbf{P}_{target}), 오차 수식은 다음과 같이 변모한다.

\|\mathbf{E}_{xy}\|_{measured} = \|\mathbf{w}_{xy}\|_2

노이즈 벡터 \mathbf{w}_x, \mathbf{w}_y가 각각 독립적이고 분산이 \sigma^2인 정규 분포를 따른다면, 위치 오차의 놈(Norm)인 \|\mathbf{w}_{xy}\|_2는 **레일리 분포(Rayleigh Distribution)**를 띠게 된다.
이는 기체가 완벽하게 목적지에 도달하여 움직이지 않더라도, EKF2 추정 위치의 요동침으로 인해 위치 제어기 입장에서는 항상 일정한 스칼라 거리가 남아있는 것처럼 인식된다는 것을 뜻한다.

graph LR
    A[Ideal Position: 0m Error] -->|Sensor Noise <br/> (GPS, IMU)| B(Estimated Error: Rayleigh Distribution)
    B -->|Constant False Error| C{If NAV_ACC_RAD < Noise Covariance}
    C -- Yes --> D[Reach condition never met <br/> (Infinite Loitering)]
    C -- No --> E[Robust Reach Declared]

따라서 NAV_ACC_RAD는 노이즈 공분산 최댓값(3\sigma)보다는 무조건 크게 설정해야 하며, RTK GPS를 장착하여 \sigma가 밀리미터 단위로 극히 낮아지지 않는 이상 센서 노이즈 한계점 이하의 도달 정밀도를 추구하는 판별 조건식 설계는 시스템 불안정성만 가중시킬 뿐이다.

3. ArduPilot의 센서 노이즈 대처 방식 비교

ArduPilot의 웨이포인트 알고리즘(예: AC_WPNav) 역시 유클리드 놈 기반으로 수평 반경(WPNAV_RADIUS)을 산출하여 거리를 측정한다.

다만, 위치 추정(EKF)의 노이즈 처리 관점에서 약간의 철학적 차이가 존재한다. ArduPilot은 S-Curve의 궤도 생성 초기 단계부터 EKF의 속도(Velocity) 추정 벡터 위주로 유도(Guidance)를 강하게 보조하여, 위치 노이즈가 즉각적으로 위치 오차 행렬로 전파되는 현상을 속도 필터 대역 폭 내에서 상당 부분 상쇄(Damping)시키는 성향이 있다.
반면 PX4-Autopilot은 네비게이터-커맨더 계층 간의 철저한 객체 지향적 책임을 중시하므로, 네비게이터의 상태 전이 조건식 안에서는 위치 놈(Norm) 자체의 스칼라 비교를 직관적으로 수행하고, 그 이전 EKF2의 칼만 이득(Kalman Gain) 튜닝 파라미터(예: EKF2_GPS_POS_X)를 통해 상태 추정단에서 스스로 노이즈를 컷오프(Cut-off)하도록 유도한다는 점에서 차이가 있다.
