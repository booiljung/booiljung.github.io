46.3.4.2. FlightTaskAuto 모듈의 3차 에르미트 스플라인(Cubic Hermite Spline) 연속 시간 궤적 생성 메커니즘

자율 비행(Auto) 모드에서 기체가 여러 경유점(Waypoints)을 통과하거나, 특정 목적지로 이동(Reposition/Goto)할 때 가장 중요한 과제 중 하나는 비행 궤적의 평활성(Smoothness)을 확보하는 것이다. 단순히 점과 점을 잇는 선형(Linear) 궤적을 추종할 경우, 경유점에서 목표 속도의 방향이 급격히 꺾이면서 물리적으로 불가능한 무한대의 가속도 요구량이 발생하게 된다. PX4-Autopilot의 FlightTaskAuto 연관 모듈 및 궤적 라이브러리(Trajectory Library)는 이러한 문제를 해결하기 위해 3차 에르미트 스플라인(Cubic Hermite Spline) 수학 모델을 도입하여 연속 시간 궤적(Continuous-time Trajectory)을 생성한다 (기반 펌웨어 버전: PX4-Autopilot v1.14 기준).

1. 연속 궤적 확장의 필요성

선형 궤적(Linear Trajectory) 상에서 기체는 각 웨이포인트 노드에서 완전히 정지(Velocity = 0)한 후 다음 목적지를 향해 다시 가속해야 한다. 이는 비행 시간의 낭비를 초래할 뿐만 아니라, 에너지 효율(배터리 소모) 측면에서도 매우 불리하다.
반면 스플라인 보간법(Spline Interpolation)을 사용하면, 기체가 경유점을 통과할 때 정지하지 않고 사전에 계산된 곡선을 따라 비행할 수 있어 부드러운 카메라 짐벌 움직임과 높은 비행 효율성을 보장한다.

2. 차 에르미트 스플라인(Cubic Hermite Spline) 수학적 모델링

3차 에르미트 스플라인은 제어점(Control Points)의 위치(Position)뿐만 아니라 해당 점에서의 접선 벡터(Tangent Vector, 즉 속도)를 함께 지정하여 3차 다항식 곡선을 정의하는 기법이다. 매개변수 t \in [0, 1]에 대하여, 두 점 \mathbf{p}_0와 \mathbf{p}_1 사이의 궤적 \mathbf{p}(t)는 다음과 같이 4개의 에르미트 기저 함수(Basis Functions)와 위치/속도 벡터의 선형 결합으로 표현된다.

\mathbf{p}(t) = h_{00}(t)\mathbf{p}_0 + h_{10}(t)\mathbf{m}_0 + h_{01}(t)\mathbf{p}_1 + h_{11}(t)\mathbf{m}_1

여기서:

  • \mathbf{p}_0, \mathbf{p}_1: 시작점과 끝점의 3차원 위치 좌표
  • \mathbf{m}_0, \mathbf{m}_1: 시작점과 끝점에서의 속도 벡터(Tangent)
  • h_{ij}(t): 매개변수 t에 대한 3차 다항식 (Hermite Basis)

PX4의 궤적 생성기는 이전, 현재, 다음 목적지(position_setpoint_triplet)의 좌표 관계를 분석하여, 경유점 \mathbf{p}_1을 통과할 때의 부드러운 접선 벡터 \mathbf{m}_1을 휴리스틱(Heuristic) 또는 인접 벡터의 평균 기울기로 산출한다. 이렇게 산출된 \mathbf{m}_i를 바탕으로 스플라인 곡선이 실시간으로 연산된다.

3. FlightTaskAuto 모듈의 내부 구현 메커니즘

PX4 소스 코드 내에서 스플라인 궤적은 주로 FlightTaskAutoSmooth (또는 과거 버전의 관련 자식 클래스) 및 수학 라이브러리 내의 베지에/에르미트 곡선 유틸리티를 통해 구현된다.

  1. 시간 매개변수화(Time Parameterization): 기체의 목표 순항 속도(Cruise Speed, MPC_XY_CRUISE 등)와 두 점 사이의 거리를 기반으로 소요 시간 T를 추정한다. 스플라인 매개변수 t는 현재 진행 시간 경과를 T로 나눈 비율로 매핑된다.
  2. 접선(Tangent) 제약 조건: 교차 지점에서 최대 가속도(MPC_ACC_HOR_MAX) 또는 저크 제약조건을 침해하지 않도록, 너무 좁은 회전 반경이나 과도한 속도 요구가 발생할 경우 \mathbf{m}_i의 크기를 제한(Scaling down)한다.
  3. 셋포인트 연산(Setpoint Generation): 위치 제어 제어 주기(~250Hz)마다 식에 현재 경과 시간 t를 대입하여 국소적(Local) 목표 위치, 속도, 가속도를 vehicle_local_position_setpoint uORB 토픽에 채워 퍼블리싱(Publishing)한다. 가속도의 경우 3차 다항식을 두 번 미분한 1차 함수(선형) 형태로 계산되어 피드포워드(Feedforward) 제어에 입력된다.

4. ArduPilot의 S-Curve와의 설계 차이

ArduPilot은 비교적 최근에 웨이포인트 네비게이션을 위해 SCurve 라이브러리를 전면 도입하였다. ArduPilot의 S-Curve 방식은 기하학적 곡선 생성과 저크-제한(Jerk-limited) 속도 프로파일을 단일 라이브러리 내에서 강력하게 결합하여 처리한다. 즉, 시작부터 끝까지 전체 궤적의 3차 미분(Jerk) 요소를 해석적으로 풀어서 C^2(가속도 연속성)까지 완벽히 보장한다.

반면, PX4-Autopilot은 FlightTaskAuto 구조를 통해 상대적으로 모듈화된 접근을 취한다. 에르미트 스플라인은 주로 공간적 궤적(Spatial Path)의 C^1(속도 연속성) 평활화에 집중하며, 시간적 속도 프로파일의 저크 제약은 별도의 VelocitySmoothing 등의 로직을 혼합하여 달성한다. 이는 구현의 독립성을 보장하고, 필요한 경우 새로운 수학적 스플라인 모델(예: B-Spline)을 새로운 Task로 쉽게 탈부착할 수 있게 하는 아키텍처적 유연함을 지니고 있다.
