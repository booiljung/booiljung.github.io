## 0.1 공간 이동 제어 수행을 위한 비선형 상태 추정(Nonlinear State Estimation) 시스템 선행 조건

PX4-Autopilot의 자율 비행 상태 기계(State Machine)에서 단일 목적지 이동(Goto/Reposition)과 같은 공간 이동 제어를 안정적으로 수행하기 위해서는, 기체의 현재 상태를 정확하게 파악하는 것이 필수적이다. 이를 위해 PX4는 확장 칼만 필터(Extended Kalman Filter, EKF2) 알고리즘을 활용하는 비선형 상태 추정(Nonlinear State Estimation) 시스템을 운용한다. EKF2는 다양한 센서(IMU, GNSS, 마그네토미터, 기압계 등)로부터 획득한 데이터를 융합하여 기체의 위치, 속도, 자세(Position, Velocity, Attitude) 추정치를 실시간으로 계산한다. 공간 이동 제어 명령을 수락하기 전, 시스템은 EKF2 추정값의 확률론적 유효성과 수렴성을 반드시 검증해야 한다.

0.1.1 비선형 상태 추정 시스템 개요 및 EKF2 아키텍처

PX4의 Estimator 모듈(ECL/EKF2)은 관성 항법 장치(INS)의 적분 오차(Drift)를 위성 항법 시스템(GNSS) 및 기타 외장 센서의 관측치로 보정하는 구조를 가진다.

  • Ardupilot 대비 차이: Ardupilot의 EKF3 코어는 다중 EKF 인스턴스를 병렬로 실행하여 코어 간 비교를 통해 결함 허용성(Failover)을 강화하는 방식(Lane switching)을 채택한 반면, PX4-Autopilot(Firmware v1.14 기준)의 EKF2 단일 모듈 내에서 혁신 시퀀스(Innovation Sequence)와 데이터 융합 품질 지표를 정밀하게 평가하여 자체적인 상태 유효성(Validity)을 결정하는 구조에 더욱 최적화되어 있다. PX4는 상태 추정 무결성이 깨졌을 때 빠른 모드 전환(예: Position Mode에서 Altitude Mode로 재조정)을 트리거하는 인터로크(Interlock) 메커니즘을 내장하고 있다.

0.1.2 공간 이동 제어의 핵심 선행 조건

단일 목적지 이동(Goto) 제어기로 진입하기 위해서는 다음의 EKF2 상태 추정 선행 조건들이 만족되어야 한다.

  • 전역 위치(Global Position) 및 지역 위치(Local Position) 유효성 확보: GNSS 신호를 기반으로 측정된 기체의 위도, 경도, 고도 데이터가 지속적으로 안정적이어야 한다. EKF2는 파라미터 EKF2_REQ_GPS_H로 설정된 수평 정확도 요구 사항을 충족해야 하며, vehicle_local_position 및 vehicle_global_position uORB 토픽을 통해 위치 유효성 플래그(xy_valid, z_valid, v_xy_valid, v_z_valid)를 발행하며, 이 값들이 모두 참(True) 상태를 유지해야 한다.
  • 공분산 대각 성분(Covariance Diagonal Elements)의 수렴: EKF2 내 상태 공분산 행렬의 대각 성분은 각 추정 변수(위치, 속도 등)의 오차 불확실성을 나타낸다. 시스템은 이 불확실성이 사전 정의된 임계값(Threshold) 이하로 수렴했는지 확인한다. 수렴되지 않은 상태에서 공간 이동을 시도할 경우 기체 궤적의 발산(Divergence) 위험이 다분하다.
  • 센서 퓨전 혁신(Innovation) 검증: 필터의 예측값과 실제 측정 센서 관측값 사이의 차이인 이노베이션(Innovation) 오차가 일관성(Consistency) 측정 임계값을 초과하지 않아야 한다. 이를 통하여 다중 경로 오차(Multipath Error) 또는 마그네토미터 환경 교란(Magnetic Interference)을 기각(Reject)한다.
  • 지역 접평면(Local Tangent Plane) 좌표계 원점 초기화: 기체가 시스템 이륙을 감지한 시점이나 EKF2가 3차원 위치(Global Position)를 확정한 시점에, NED(North-East-Down) 방식을 따르는 지역 좌표계의 원점(Origin) 좌표가 명시적으로 선언되어야 한다. 이는 상대적 거리 기준의 유도 제어(Guidance Control)를 도출하는 기준 정보이다.

0.1.4 지상 관제 시스템(GCS) 측면의 공학적 고려

QGroundControl(QGC, v4.3 이상 권장)과 같은 지상 관제 시스템 단계에서는 기체에서 하향 전송되는 MAVLink 패킷인 ESTIMATOR_STATUS 프로토콜 메시지 내 상태 비트플래그(Flags)를 디코딩하여 EKF2 필터의 추정 건전성을 인터페이스 화면에 시각화해 준다. 만일 센서 융합 불확실성이 높은 경우라면 QGC 상단 중앙의 시스템 표시줄에 “GPS Estimate Error” 혹은 “Position Control Denied” 등의 시스템 수준의 경고 시그널을 출력하게 된다. 따라서, 개발자가 독자 기종용으로 사용자 정의 GCS 애플리케이션 프런트엔드를 설계 및 프로그래밍할 때에도, 무인기의 ESTIMATOR_STATUS 속성 상태를 연속적으로 폴링(Polling)하고 파싱하여, 자율 Goto 명령 파라미터 전달 가능 여부를 사전 검사(Pre-check)하는 관제 설계 기법을 반드시 적용해야 한다.
