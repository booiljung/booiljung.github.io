46.5.3.1 이동 궤적 추종 중 무선 주파수(RF) 원격 측정(Telemetry) 데이터 링크 단절(Data Link Loss) 감지 시 Failsafe 우선순위 인터럽트(Priority Interrupt) 처리

무인항공기(UAV)가 단일 목적지 이동(Goto/Reposition) 비행을 수행하는 도중, 지상 관제 시스템(GCS)과의 무선 주파수(RF) 원격 측정(Telemetry) 통신이 단절되는 상황(Data Link Loss)은 자율 비행 시스템에서 가장 빈번하게 발생하는 중대 결함 중 하나이다. 비행 제어기(Flight Controller)는 통신 두절을 감지하는 즉시, 기존에 수행 중이던 궤적 추종(Trajectory Tracking) 임무를 중단하고 시스템에 사전 정의된 안전 계층으로 제어권을 강제 위임(Override)해야 한다.

본 절에서는 PX4-Autopilot(펌웨어 v1.14.0 기준)이 데이터 링크 단절을 감지하는 통신 타임아웃(Timeout) 메커니즘과, 이를 기반으로 Commander 모듈에서 발생하는 Failsafe 우선순위 인터럽트(Priority Interrupt) 처리 로직의 소프트웨어 구조를 심층적으로 분석하라.

1. 데이터 링크 단절(Data Link Loss) 감지 메커니즘

PX4는 GCS와의 통신 상태를 지속적으로 모니터링하기 위해 MAVLink 네트워크의 심박수(Heartbeat) 메시지에 의존한다. 데이터 링크 단절 감지는 단순히 신호의 물리적 세기(RSSI)가 아닌, 논리적 MAVLink 데이터 수신율을 기준으로 작동한다.

1.1 Heartbeat 타임아웃 평가 (Heartbeat Timeout Evaluation)

PX4의 mavlink_receiver 모듈은 GCS(QGroundControl v4.3.0 기준) 시스템 ID로부터 1Hz 주기로 수신되는 HEARTBEAT (메시지 ID: 0) 패킷의 타임스탬프를 감시한다.

  • 컴패니언 컴퓨터(Companion Computer): ROS2 (uXRCE-DDS) 기반 Offboard 연결의 경우, 텔레메트리 포트(Telemetry Port) 또는 이더넷(Ethernet) 링크 상의 활동량이 평가 대상이 된다.
  • 파라미터 NAV_DLL_ACT: 데이터 링크가 단절되었을 때 취할 Failsafe 행동(비활성화, Hold, RTL, Land 등)을 정의한다.
  • 파라미터 COM_DL_LOSS_T: GCS로부터 데이터 패킷(MAVLink 메시지)을 수신하지 못한 지 얼마의 시간(기본값: 10초)이 경과했을 때 Data Link Loss Failsafe를 발동시킬 것인지를 정의하는 타임아웃 임계값이다.

2. Failsafe 우선순위 인터럽트(Priority Interrupt) 동작 설계

데이터 링크 단절 타임아웃 임계값을 초과(Threshold Exceeded)하게 되면, PX4 Commander 모듈은 현재 진행 중인 내비게이션 상태(Navigation State)를 선점(Preempt)하고 Failsafe 모드로 상태 머신을 전환한다.

2.1 인터럽트 발생 및 처리 파이프라인

Goto 궤적 추종은 Navigator 모듈 내부에서 처리되는 ’일반 우선순위(Normal Priority)’의 임무이지만, Data Link Loss Failsafe는 커맨더 레벨에서 발동되는 ‘고안전성 우선순위(High-Safety Priority)’ 인터럽트이다.

  1. 상태 감지(State Detection): mavlink_receiver 노드가 타임아웃을 감지하면 uORB telemetry_status 토픽의 상태 깃발(Status Flag)을 갱신한다.
  2. 에러 격상(Error Escalation): Commander는 telemetry_status 토픽을 구독(Subscribe)하여 data_link_lost 불리언(Boolean) 값을 확인하고 시스템 Failsafe 조건을 평가한다.
  3. 궤적 제어권 회수(Trajectory Control Handover): Goto 명령을 추종하던 위치 제어 루프 로직에 인터럽트(Interrupt)를 걸어 현재 위치를 목표 위치(Setpoint)로 일치시키며 제자리 비행(Loiter/Hold)을 하거나, 곧바로 RTL(Return To Launch) 상태로 파이프라인을 전환한다.

3. Ardupilot의 대응 기법과의 제어 아키텍처 비교

데이터 링크 상실 대응 부분에서 양대 펌웨어는 GCS 연결성 상실(GCS Failsafe)을 파라미터 구조와 복구 논리에서 다르게 다룬다.

비교 요소PX4-AutopilotArdupilot (Copter)
감지 소스(Detection Source)MAVLink를 수신하는 모든 포트(TELEM1, TELEM2 등)의 활동 내역(Activity)과 Heartbeat를 통합 평가하며, Companion Computer(ROS2) 데이터 링크까지 단일 telemetry_status uORB 토픽으로 묶어 처리한다.RC 수신기 기반의 FS_THR_ENABLE (Radio Failsafe)와 MAVLink 트래픽 기반의 FS_GCS_ENABLE (GCS Failsafe)을 뚜렷이 구분하여 독립된 인터럽트 플래그로 취급한다.
타임아웃 및 복구(Recovery) 논리RF 링크 단절 후 Failsafe가 발동된 뒤 다시 통신이 복구되더라도, 사용자가 명확하게 모드 변경을 명시(Mode Switch Toggle)하거나 텔레메트리로 명령을 보내지 않으면 기존의 Goto 비행으로 자동 복귀하지 않고 Failsafe 궤적(RTL 시퀀스 등)을 유지한다. (의도 검증 철학)파라미터 FS_GCS_ENABLE 설정값에 따라, “Continue with Mission in Auto Mode“와 같이 통신 두절을 무시하고 기존 임무를 유지하는 선택지를 매우 쉽게 활성화할 수 있다.

PX4의 인터럽트 처리 방식은 “안전이 담보되기 전까지 시스템은 가장 보수적인 안전 궤적을 밟아야 한다“는 강건한 상태 기계(State Machine) 설계 패러다임을 보여준다.

4. GCS(QGroundControl) 및 맞춤형 앱의 대응 소프트웨어 공학 패턴

GCS 측면에서도 원격지의 기체가 Data Link Loss Failsafe에 진입했음을 판단하고 로컬 UI 상태를 업데이트해야 한다.

  1. GCS 타임아웃 감지: QGroundControl 본체(Vehicle Manager)는 기체로부터 오는 Heartbeat 패킷을 수신하며, 패킷이 도착하지 않는 시간이 내장 타임아웃 설정값을 초과하면 통신 연결 단절(Communication Lost) 이벤트를 발생시켜 비디오 스트림과 계기판 통신을 비활성화한다.
  2. 복구 후(Post-Recovery) 제어권 이양: 만일 비행체 측에서 Failsafe(예: RTL)로 복귀하던 중 통신이 일시 재결합(Reconnected) 된 경우, QGroundControl은 COMMAND_LONG 폼의 MAV_CMD_NAV_RETURN_TO_LAUNCH 중단 명령이나 새로운 오프보드(Offboard) 재진입 명령(VEHICLE_CMD_DO_SET_MODE)을 발행해야 원래 수행하던 미션이나 Goto 알고리즘을 재생시킬 수 있다.

5. 결론

Goto 및 실시간 궤적 제어 중의 데이터 링크 단절(Data Link Loss)은 자율 비행의 신뢰성을 시험하는 중대한 단절점이다. PX4-Autopilot은 uORB 기반의 고속 상태 감시 파이프라인을 통해 임무 제어기(Navigator)가 생성하는 일반 궤적을 강제 선점(Priority Interrupt)하고 즉각적인 비상 복구 절차로 이행한다. 시스템 설계자와 GCS SW 엔지니어는 기체와 GCS 간의 상호 타임아웃 윈도우(Timeout Window)를 일치시키고, 자율 에이전트 운용 중 우발적 연결 해제가 발생할 때를 대비한 기체의 동적 리셋(Dynamic Reset) 프로토콜을 면밀하게 설계해야만 예측 불가능한 도심 및 특수 환경에서의 비상 상황을 원천적으로 제어할 수 있다.
