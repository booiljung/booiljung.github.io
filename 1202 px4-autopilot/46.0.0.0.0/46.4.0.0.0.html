46.4. 도달 판별(Arrival Determination) 알고리즘의 제어공학적 설계 및 기하학적 해석학

단일 목적지 이동(Goto/Reposition) 또는 자동 비행 임무(Mission) 중, 기체가 목표로 설정된 웨이포인트(Waypoint)에 도착했는지를 시스템이 판별하는 논리는 드론의 자율성을 결정짓는 가장 핵심적인 기능 중 하나이다. PX4-Autopilot은 단순히 특정한 3차원 점(Point)에 도달했는지 여부를 참/거짓(Boolean)으로 판별하지 않으며, 제어공학과 3차원 기하학이 결합된 다차원 도달 판별(Arrival Determination) 알고리즘을 사용하여 수렴 상태를 결정한다 (기반 펌웨어 버전: PX4-Autopilot v1.14 기준). 본 장에서는 도달 판별의 본질적인 제어공학적 의미와 이를 구현하는 기하학적 해석 방법에 대해 살펴본다.

1. 도달 판별(Arrival Determination)의 제어공학적 정의

관성(Inertia)을 지닌 비행 기체는 외부의 바람이나 센서 노이즈오차 등으로 인해 математически 완벽하게 3차원 유클리드 공간상의 단일 점(Point) 좌표 (x, y, z)에 안착할 수 없다. 제어공학적 관점에서 도달 판별이란 기체의 위치 제어기(Position Controller)가 산출하는 현재 위치 상태값이 목표 위치와의 오차(Error) 한계 내로 충분히 수렴(Convergence)하였는가를 결정하는 시스템 식별(System Identification) 과정이다.

PX4-Autopilot은 시스템의 강건성(Robustness)을 위해 기하학적 ‘도달(Reach)’ 여부와 동역학적 ‘지연(Stay)’ 여부를 함께 평가한다.

1.1 기하학적 수용 영역(Acceptance Region) 모델링

목표 지점에 도달했는지를 판단하는 기준선, 즉 수용 반경(Acceptance Radius)은 3차원 공간에서 다음과 같이 기하학적 볼륨(Volume)으로 해석된다.

  • 수평면(XY-Plane) 수용 영역: 목표 좌표를 중심으로 하는 2차원 원(Circle)의 형태를 갖는다. 수용 반경 NAV_ACC_RAD 설정값에 따라 반경의 크기가 결정된다.
  • 수직축(Z-Axis) 수용 영역: 고도 오차가 지정된 수직 수용 범위(일반적으로 NAV_MC_ALT_RAD) 내에 들어왔는지를 독립적으로 판단하는 1차원 선형(Linear) 구간이다.

따라서 기체가 목표에 도달했다는 것은, 이 2차원 원 영역과 1차원 선형 구간이 직교(Orthogonal)하여 만들어내는 원기둥(Cylinder) 형태의 3차원 기하학적 교집합 볼륨 안으로 기체의 추정 위치(Estimated Position)가 진입했음을 의미한다.

2. 속도 및 동적 조건을 결합한 하이브리드 판별 방식

PX4-Autopilot은 위치뿐만 아니라 기체의 운동 에너지(Kinetic Energy), 즉 속도(Velocity)까지 도달 판별의 인자로 활용한다.

만약 기체가 빠른 속도로 이동하다가 수용 반경 내에 진입했다 하더라도, 기체의 관성으로 인해 곧바로 반경을 벗어날(Overshoot) 확률이 높다. 이러한 상태를 섣불리 ’도달 완료(Arrival)’로 판별하게 되면, 네비게이터(Navigator)는 다음 웨이포인트로 즉시 목표를 변경하게 되어 궤적이 크게 왜곡되는 현상이 나타난다.

이를 방지하기 위해 PX4는 다음의 조건을 추가적으로 확인한다.

  1. 가상 반경(Virtual Radius) 확장: 기체의 현재 속도 벡터를 목표 지점 방향으로 투영(Projection)하여, 속도가 빠를 경우 수용 반경을 동적으로 약간 더 넓게 해석하는 전방향(Look-ahead) 판별법을 사용한다.
  2. 시간 지연(Time Delay) 기반 검증: 수용 반경 안에 진입한 후, 위치가 안정화(Stabilization)될 때까지 설정된 지연 시간(Time to stay) 동안 궤도 내에 머물러 있는지를 평가한다.

3. ArduPilot의 도달 판별 메커니즘과의 비교

이기종인 ArduPilot(Copter)의 도달 판별 로직 또한 지정된 반경(일반적으로 WPNAV_RADIUS) 내로 기체가 진입했는지를 확인하는 유사한 기하학적 원통형 모델을 사용한다.

다만 PX4-Autopilot은 mc_pos_control 내부의 FlightTask 아키텍처와 결합되어, 가속도 및 저크(Jerk)의 제한 사항이 물리적으로 만족되는 과정 자체를 도달의 선행 조건으로 매끄럽게 연결한다는 점에 차이가 있다. ArduPilot이 주로 S-Curve 알고리즘 내부에서 궤적 상의 진행률(Progress) 100% 달성 여부로 판단하는 성향이 강하다면, PX4는 유클리드 공간 상의 위치 오차 행렬 연산과 네비게이터의 상태 전이 루프를 논리적으로 명확하게 분리하여 제어 루프의 격리성(Isolation)을 높였다는 특징이 있다.
