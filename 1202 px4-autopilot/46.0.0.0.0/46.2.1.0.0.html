46.2.1. MAV_CMD_DO_REPOSITION 제어 명령 패킷의 페이로드(Payload) 비트 필드(Bitfield) 구조 및 제어 변수 공학적 매핑

본 절에서는 MAVLink 프로토콜에서 단일 목적지 이동을 지시하는 핵심 명령인 MAV_CMD_DO_REPOSITION (Command ID: 192)의 페이로드 구조를 분석하고, 각 매개변수(Parameter)가 PX4 기체 내부의 제어 변수로 어떻게 매핑(Mapping)되는지 공학적 관점에서 심층적으로 다룬다.

1. MAV_CMD_DO_REPOSITION 페이로드 데이터 아키텍처

COMMAND_LONG 혹은 COMMAND_INT 메시지에 탑재되는 MAV_CMD_DO_REPOSITION 명령은 총 7개의 실수형(Float) 매개변수(param1 ~ param7)를 갖는다. 각각의 매개변수는 목표 위치, 이동 속도, 기수각(Heading Angle), 그리고 제어 특성을 정의하는 비트 필드(Bitfield) 옵션 등을 포함한다.

명령어의 구문적(Syntactic) 규격은 다음과 같다.

매개변수(Param)자료형설명(Description)PX4 내부 매핑 변수명
Param 1Float이동 속도(Translation Speed) [m/s]. -1 입력 시 기본 속도 사용.velocity_limit
Param 2Float비트 필드(Bitfield) 옵션 플래그.reposition_flags
Param 3Float예약됨(Reserved). 사용하지 않음(0).N/A
Param 4Float목적지에서의 요(Yaw)/기수각(Heading) [rad]. NaN 입력 시 현재 각도 유지.yaw_setpoint

2. 공학적 제어 변수 매핑(Engineering Variable Mapping) 메커니즘

GCS(QGroundControl)가 전송한 MAVLink 패킷은 PX4의 mavlink_receiver 모듈에서 파싱(Parsing)되어 uORB 미들웨어의 vehicle_command 구조체로 역직렬화(Deserialization)된다.

2.1 이동 속도(Translation Speed) 매핑

Param 1에 입력된 속도는 제어기 내부의 상한선 속도인 MPC_XY_CRUISE 및 MPC_Z_V_AUTO_UP/MPC_Z_V_AUTO_DN 파라미터와 비교 검증을 거친 후, position_setpoint_triplet의 cruise_speed 속성으로 매핑된다. Ardupilot의 WPNAV_SPEED와 유사한 역할을 수행하지만, PX4는 가속도 지터(Jerk)를 방지하기 위해 로우패스 필터(Low-pass Filter) 및 궤적 생성기(Trajectory Generator)에 해당 제한값을 통합시켜 연속적인 다항식(Polynomial) 궤적을 산출한다.

2.2 기수각(Heading Angle) 매핑

Param 4의 요(Yaw) 각도는 드론이 목적지에 도달한 후 유지해야 할 혹은 이동 중에 지향할 방향을 의미한다. PX4는 NaN 값이 수신될 경우 기존 방향(Current Heading)을 유지하거나 경로 접선 방향(Tangent Vector)을 지향하도록 자동 설정 파라미터(MPC_YAW_MODE)에 따라 유연하게 결정한다. QGroundControl은 기본적으로 목적지를 바라보며 이동하는 설정을 제공한다.

2.3 비트 필드(Bitfield) 옵션 검증

Param 2는 향후 확장성(Extensibility)을 위해 남겨진 비트 필드 플래그다. 특정 비트를 0 또는 1로 토글하여 동작 모드를 결정하는데, 예를 들어 지형 추종(Terrain Following) 모드 강제 활성화 혹은 속도 제한 우회 등의 기능이 고려될 수 있다. 제어 로직 내부에서는 비트 마스킹(Bit Masking) 기법을 사용하여 플래그를 해석한다.

4. 제어 계층(Control Tier)에서의 우선순위 및 예외 처리

MAV_CMD_DO_REPOSITION은 단일 인터럽트(Interrupt) 명령이므로, 펌웨어가 기존의 Mission 모드 혹은 Position 모드로 비행 중일 때 수신되면 기존 궤적(Trajectory) 계산을 즉시 중단(Abort)하고 새로운 Setpoint를 주입(Injection)한다. 이때 매핑된 속도 변수나 좌표 변수가 기체의 한계 성능(Dynamic Limits)을 초과할 경우, Commander 모듈 내의 안전 검사 모듈(Pre-flight Check & Failsafe)이 MAV_RESULT_DENIED를 GCS로 회신하고 기존 비행 상태를 유지한다. Ardupilot의 Guided 모드 명령 전환과 궤를 같이하나 제어 탈취 및 우선순위 양도의 아키텍처 투명성 측면에서 PX4의 State Machine이 더 직관적인 구조를 지닌다.
