46.1.1.1 오프보드(Offboard) 궤적 추종 제어기와의 폐루프(Closed-Loop) 응답 특성 비교

지상 관제 시스템(GCS) 또는 외부 컴패니언 컴퓨터(Companion Computer)가 무인 항공기(UAV)의 위치를 제어하는 방식은 크게 지정된 단일 목표점으로의 유도(단일 목적지 이동, Reposition)와 연속적인 궤적 좌표의 실시간 주입(오프보드 제어, Offboard Control) 두 가지 양상으로 대별된다. 본 절에서는 이 두 방식이 PX4-Autopilot(v1.14 기준) 내부의 위치 제어기(Position Controller)와 결합할 때 나타나는 폐루프(Closed-Loop) 응답 특성과 제어공학적 메커니즘의 차이를 비교 분석한다.

1. 제어 루프 아키텍처 및 궤적 생성의 주체

가장 결정적인 차이는 시간에 따른 **궤적 생성(Trajectory Generation)**의 주체가 온보드(Onboard, 비행 제어기 내부)에 있는지, 오프보드(Offboard, 외부 컴퓨터)에 있는지에 기인한다.

아래 블록 선도(Block Diagram)는 두 제어 모드가 위치 제어 루프를 어떻게 형성하는지 직관적으로 보여준다.

    subgraph "Reposition Mode (단일 목적지 이동)"

        Nav -- position_setpoint_triplet --> TrajGen_ON[온보드 Trajectory Generator]
        TrajGen_ON -- 부드러운 궤적<br/>(100Hz 이상) --> PosCtrl_R[Position Controller]
        PosCtrl_R -- 속도/자세 제어량 --> Plant_R[기체 동역학]
        Plant_R -- 상태 피드백 --> PosCtrl_R
    end

    subgraph "Offboard Mode (오프보드 모드)"

        SET_POS -- 연속적인 목표점 --> PosCtrl_O[Position Controller]
        PosCtrl_O -- 속도/자세 제어량 --> Plant_O[기체 동역학]
        Plant_O -- 상태 피드백 --> PosCtrl_O
    end
  • 단일 목적지 이동(Reposition): GCS가 MAVLink 트랜잭션을 통해 목표 위치를 1회 전송하면, 파라미터로 설정된 최대 가속도(MPC_ACC_HOR)와 최대 저크(MPC_JERK_AUTO) 제약 조건을 바탕으로 PX4 내부의 내비게이터(Navigator)와 궤적 생성기가 스스로 비행 프로파일(S-Curve 등)을 생성하여 위치 제어기로 전달한다.
  • 오프보드 표적 추종(Offboard): ROS 2와 같은 외부 프레임워크가 실시간으로 10Hz ~ 50Hz 단위의 궤적(위치, 속도, 가속도 벡터)을 지속적으로 기체에 스트리밍한다. PX4 내부의 궤적 생성 레이어는 이를 우회(Bypass)하거나 최소한의 보간(Interpolation)만을 수행한 후 직접 위치 제어 루프로 진입시킨다.

2. 폐루프 응답 특성(Response Characteristics) 비교

이러한 아키텍처의 차이는 곧 제어 시스템의 응답 특성과 안정성 여유(Stability Margin)에 중대한 영향을 미친다.

2.1 스텝 응답(Step Response) 및 과도 상태(Transient State) 특성

  • Reposition: 멀리 떨어진 거리를 단일 스텝(Step Input)으로 인가하더라도, 내부 궤적 생성기가 가감속 모델을 엄격하게 통제하므로 오버슛(Overshoot)이나 진동 없이 매우 부드러운 감쇠비(Damping Ratio)를 보인다. 외부 통신 단절이 발생하더라도 생성된 궤적이 기체 내부에 존재하므로 목적지까지 안전하게 도달하고 정지(Hovering)한다.
  • Offboard: 외부 컴퓨터가 위치 단계(Step) 명령을 불연속적으로 인가할 경우, 궤적 보정이 없으면 위치 제어기에 막대한 초기 오차 신호가 유입되어 공격적인 피치/롤(Pitch/Roll) 피드백 개입을 야기한다. 따라서 오프보드 제어를 사용할 경우 외부에서 선형 이차 레귤레이터(LQR)나 모델 예측 제어(MPC)와 같은 진보된 로직을 활용해 충분히 매끄러운 궤적을 실시간으로 추종시켜야만 기체가 발산하지 않는다.

2.2 시지연(Time Delay)에 따른 동특성 확보

오프보드 제어는 제어 루프의 일부 경로가 통신 텔레메트리(UART, TCP/UDP 브릿지 등)를 경유하므로, 필연적으로 통신 지연시간(Latency)과 주기의 지터(Jitter)를 수반한다. 고속으로 비행하는 동적 객체를 추종할 때 통신 지연은 위상 여유(Phase Margin)를 갉아먹어 폐루프를 불안정하게 조장할 수 있다.

반면 Reposition 제어는 단 일회의 통신 후 모든 폐루프 계산이 온보드 RTOS(NuttX) 내에서 결정론적(Deterministic)으로 100Hz ~ 250Hz 단위로 연산되므로 제어 주파수 대역폭(Bandwidth) 내외의 외란에 대해 훨씬 강건(Robust)한 지연 특성을 지닌다.

3. MAVLink 및 프로토콜 차이 비교

두 기능의 사상적 차이는 MAVLink 프로토콜의 설계에도 반영되어 있다.

  • 단일 목적지 이동은 MAV_CMD_DO_REPOSITION와 같은 커맨드(Command) 인터페이스를 사용하여 수락(ACK)/거부(NACK) 트랜잭션의 신뢰성 검증을 거친다.
  • 오프보드 궤적 추종은 SET_POSITION_TARGET_LOCAL_NED와 같은 상태량 세트포인트 스트림(Setpoint Stream) 메시지로 구성되어 속도감을 중시하며, ACK를 기다리지 않는다(Fire-and-forget).

4. ArduPilot과의 비교 고찰

ArduPilot의 GUIDED 모드는 외부 제어를 담당함에 있어 PX4의 Offboard와 Reposition 모드의 성격이 통합된 형태를 취한다. 즉, GCS가 단일 웨이포인트를 쏘아 보내면 내부 웨이포인트 이동기(WPNav)가 이를 처리하기도 하고, 동시에 고주파 주기로 위치/속도를 주입하면 부드럽게 추종하기도 한다.

이렇게 하나로 병합된 설계는 운용의 직관성을 높이는 장점이 있으나, PX4처럼 ’내부 임무(Mission) 수준의 자율 이동(Reposition)’과 ’외부 알고리즘에 의한 완전 종속 제어(Offboard)’의 권한 추상화 벽(Abstraction Layer)이 엄격하게 분리되지 않아, 임무 도중 컴패니언 컴퓨터로 제어를 넘기고 다시 회수하는 복합 시스템 아키텍처를 구성할 때 상태 머신 처리의 모호성을 발생시킬 수 있다.
