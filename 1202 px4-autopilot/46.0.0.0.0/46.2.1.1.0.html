46.2.1.1. 이동 속도(Translation Speed) 및 기수각(Heading Angle) 제약 조건의 실수형(Floating Point) 매개변수화(Parameterization)

단일 목적지 제어(Goto/Reposition) 명령 시, 이동 속도와 기수각은 비행체의 동역학적 한계 및 탑재 장비(지향성 안테나, 카메라 등)의 시야각 유지를 위해 정밀하게 제어되어야 하는 필수 요소다. 본 절에서는 MAV_CMD_DO_REPOSITION 등에서 전달되는 이동 속도와 기수각 파라미터가 PX4 내부에서 실수형(Floating Point) 매개변수화(Parameterization)를 거치며 어떻게 제약 조건(Constraints)을 통과하는지 설명한다.

1. 실수형 매개변수화(Floating Point Parameterization)의 엔지니어링적 의미

MAVLink 프로토콜은 32비트 IEEE 754 단정밀도 부동소수점(Single-precision Float)을 사용하여 연속적인 물리량을 전송한다. 정수 혹은 열거형(Enum)이 아닌 실수형 파라미터를 사용하는 이유는 기체의 가속도 프로파일(Jerk-limited Trajectory)과 부드럽게 결합되는 연속 함수(Continuous Function) 생성 시, 해상도(Resolution)의 손실 없이 값을 입력받기 위함이다.

또한 IEEE 754 표준에서 지원하는 특수 값(예: NaN, Not-a-Number)을 플래그 용도로 오버로딩(Overloading)하여, “제한 없음” 혹은 “현재 상태 유지“와 같은 특별한 제어 철학을 구현한다.

2. 이동 속도(Translation Speed)의 제약 조건 및 정규화 메커니즘

수신된 이동 속도 타겟 값(V_{cmd})은 PX4의 위치 제어 모듈(Position Controller)에서 다음과 같은 계층적 제약 조건 검사 프로세스를 거친다.

  1. 디폴트 값 판별(Default Value Detection):
    GCS 시스템이 속도를 명시하지 않고 디폴트 파라미터(일반적으로 -1.0 혹은 NaN)를 전송한 경우, PX4는 하드코딩된 상수 대신 시스템 파라미터인 MPC_XY_CRUISE (수평 비행 순항 속도) 및 MPC_Z_V_AUTO_UP/DN (수직 상승/하강 속도) 값을 자동으로 매개변수로 차용한다.
  2. 상한/하한 클리핑(Bounding / Saturation):
    유효한 V_{cmd}가 입력된 경우, 이 값이 기체의 물리적 한계치(MPC_XY_VEL_MAX 등)를 초과하는지 검사한다. 만약 최대 설정 범위를 벗어난다면 Saturation 함수를 통해 안전한 최댓값으로 강제 절삭(Clipping)된다. Ardupilot의 WPNAV_SPEED 상한선 적용 룰과 철학이 유사하나, PX4는 다차원(수평, 수직) 각각에 대해 엄격하게 분해하여 속도 포락선(Velocity Envelope)을 재계산한다.

3. 기수각(Heading Angle)의 제약 조건 및 비선형 래핑(Wrapping)

요(Yaw) 방향 제어는 드론의 항법 및 카메라 페이로드 지향에 결정적인 역할을 수행하므로, 회전 운동의 주기성(Periodicity)을 고려한 특수한 제약 조건이 요구된다.

3.1 각도 래핑(Angle Wrapping)

수신되는 목적지 기수각(\psi_{cmd})은 라디안(Radian) 단위의 Float 값이다. 로터리 운동학(Rotary Kinematics) 특성상 제어기는 도달해야 할 각도가 최단 거리가 되도록 [-\pi, \pi] 구간으로 각도를 래핑(Wrapping)한다.
\psi_{wrapped} = \text{mod}(\psi_{cmd} + \pi, 2\pi) - \pi

3.2. 상태 보존 플래그(NaN)

헤딩 파라미터에 NaN이 수신되는 경우, 비행 모듈의 제어 상태 머신은 기존 헤딩 유지(Maintain Current Heading) 상태로 진입한다. 이는 QGroundControl 관제 시 지도 상에서 단순히 지점 이동을 클릭하였을 때, 불필요한 제자리 회전(Pirouette) 거동을 억제하기 위한 핵심 예외 처리 방식이다.

만약 MPC_YAW_MODE가 자동 지향(Auto Heading)으로 설정되어 있다면, NaN 수신 시에도 기체는 다음 웨이포인트(Waypoint)를 향하는 방향 벡터(Velocity Vector)의 각도를 \text{atan2}(V_y, V_x)로 스스로 계산하여 매개변수에 동적으로 주입한다.

4. 매개변수 제약 검사 데이터 플로우 (Data Flow Diagram)

다음은 수신된 Float 변수들이 각 제약 필터를 거치는 메커니즘을 표현한 알고리즘 순서도다.

flowchart TD
    Start([실수형 Param 수신<br/>Speed & Yaw]) --> SpdCheck{Speed == -1<br/>or NaN?}
    
    SpdCheck -- Yes --> SpdDef[MPC_XY_CRUISE 파라미터 차용]
    SpdCheck -- No --> SpdClip[MPC_XY_VEL_MAX 기준<br/>Saturation Clipping]
    
    SpdDef --> YawCheck{Yaw == NaN?}
    SpdClip --> YawCheck
    
    YawCheck -- Yes --> YawMode{MPC_YAW_MODE<br/>확인}
    YawCheck -- No --> YawWrap[-π ~ π 범위로<br/>Angle Wrapping 진행]
    
    YawMode -- 경로 지향 --> YawVel[궤적 접선 벡터 기반<br/>atan2(Vy, Vx) 산출]
    YawMode -- 고정 유지 --> YawKeep[현재 기수각 상태 유지]
    
    YawWrap --> Out([최종 Trajectory <br/>Setpoint 생성기 입력])
    YawVel --> Out
    YawKeep --> Out

결과적으로, PX4의 Float 매개변수화 메커니즘은 단순한 값 전달을 넘어, IEEE 754 특수값을 비행 제어 논리의 신호기(Semaphore)로 사용하고 시스템 안정성과 제어 동역학을 고려한 실시간 제약 검사(Constraint Check)를 수행하는 시스템 공학적 최적화의 우수성을 보여준다.
