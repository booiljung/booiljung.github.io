46.5.2.1. 지속적인 바람(Constant Wind) 외란 하에서 정상 상태 위치 오차 잔존으로 인한 유한 상태 기계(FSM) 무한 루프(Infinite Looping) 교착 상태(Deadlock) 수학적 분석

무인항공기(UAV)가 특정 목적지로 이동하는 임무(Mission) 또는 단일 목적지 이동(Goto) 명령을 수행할 때, 가장 빈번하게 조우하는 환경적 외란(Disturbance) 요소는 바람(Wind)이다. 특히 일정한 방향과 세기로 지속해서 부는 지속적인 바람(Constant Wind)은 위치 제어 루프(Position Control Loop)에서 영(Zero)이 아닌 정상 상태 오차(Steady-State Error)를 잔존시키는 주요 원인으로 작용한다. PX4-Autopilot의 내비게이터(Navigator) 유한 상태 기계(Finite State Machine, FSM)는 목적지 반경(Acceptance Radius) 진입 여부를 기반으로 다음 상태로 전이(State Transition)하기 때문에, 이 오차가 반경보다 클 경우 시스템은 목적지에 도달하지 못한 것으로 판단하여 동일한 이동 상태에 머무르는 무한 루프(Infinite Looping) 방식의 교착 상태(Deadlock)에 빠질 수 있다.

이 섹션에서는 제어공학적(Control Engineering) 관점에서 이러한 현상이 발생하는 원인을 수학적으로 모델링하고 분석하라.

1. 정상 상태 위치 오차(Steady-State Position Error)의 수학적 모델링

멀티로터의 위치 제어 시스템은 일반적으로 위치, 속도, 자세 제어기가 직렬로 연결된 캐스케이드(Cascade) P-PID 제어 구조를 갖는다. PX4-Autopilot의 비행 제어 스택에서 수평 위치 제어기(Horizontal Position Controller)의 동역학적 특성은 간략화된 1차원 선형 시스템으로 근사화할 수 있다.

지속적인 바람에 의한 항력(Drag Force) 외란을 D_{wind}라고 할 때, 멀티로터의 병진 운동(Translational Motion) 방정식은 뉴턴 제2법칙에 의해 다음과 같이 표현된다.

m \ddot{x} = F_{thrust} \sin(\theta) - D_{wind}

비행 제어 시스템이 목표 위치 x_{setpoint}로 이동하기 위해 비례-적분-미분(PID) 위치 제어기를 적용한다고 가정할 때, 제어 입력 u(t)(즉, 목표 자세각 \theta_{setpoint})는 위치 오차 e(t) = x_{setpoint} - x(t)에 대해 다음과 같이 전개된다.

u(t) = K_p e(t) + K_i \int e(t) dt + K_d \dot{e}(t)

외부 공간에 불어오는 바람이 일정(Constant)하다고 가정하면, 시간에 따른 시스템의 과도 응답(Transient Response)이 모두 소멸한 후 시스템은 정상 상태(Steady-State)에 도달한다. 즉, 시간 t \to \infty에서 가속도 \ddot{x} \to 0이고 속도 \dot{x} \to 0이 된다.

만약 위치 제어기에 적분기(Integrator) 이득(K_i)이 존재하지 않는 순수 P(비례) 제어기일 경우, 지속적인 외란 D_{wind}를 보상하기 위한 0이 아닌 제어 신호 u_{SS}가 지속적으로 요구된다.
따라서, 정상 상태에서는 다음과 같은 수학적 관계가 성립한다.

K_p e_{SS} = u_{SS} \propto D_{wind}

식에서 유도할 수 있듯 정상 상태 위치 오차 e_{SS}는 D_{wind} / K_p에 비례하여 잔존하게 된다. P-PID 구조 하에서 속도 제어기에는 적분기(I-term)가 적용되어 있지만 위치 제어루프에서 적분 제어를 부분적으로 사용하거나 적분 누적 한계(Anti-Windup Limit)로 인해 적분기가 포화(Saturation) 상태에 머물게 되면, 온전한 오차 소거를 이루지 못하고 \vert e_{SS} \vert > 0 인 결과에 직면하게 된다.

2. 내비게이터(Navigator) 유한 상태 기계(FSM)의 교착 상태(Deadlock) 원리 분석

PX4-Autopilot의 내비게이터 모듈은 임무(Mission)나 단일 목적지 이동(Goto) 비행 중, 현재 위치와 목표 웨이포인트(Waypoint) 간의 유클리드 거리(Euclidean Distance)를 주기적으로 계산한다. 계산된 거리가 사전에 정의된 허용 반경(Acceptance Radius, 통상 NAV_ACC_RAD 파라미터에 의해 지정됨) 이내로 들어오면 FSM은 비로소 도달(Reached) 이벤트로 판단하고 다음 임무 항목으로 상태를 전이한다.

목적지 도달 판별 알고리즘의 핵심 조건은 아래와 같다.

\left\Vert \mathbf{P}_{current} - \mathbf{P}_{setpoint} \right\Vert_{2} < R_{acceptance}

정상 상태 위치 오차 벡터의 크기 \vert e_{SS} \vert가 외란의 크기로 증가하여 허용 반경 R_{acceptance}보다 커지게 되는 상황, 즉

\Vert e_{SS} \Vert_{2} > R_{acceptance}

의 조건이 성립한다면, 제어기가 물리적인 바람을 이기고 더 이상 목적지에 가까워지지 못하는 평형(Equilibrium) 상태에 머물게 된다.

이때, 기체의 제어기(Controller)는 현재 위치를 유지하고 있으나, 상위 수준 시스템인 내비게이터 FSM은 아래의 다이어그램처럼 무한 루프 상태에 돌입하게 된다.

stateDiagram-v2
    [*] --> MovingToWaypoint: Goto Command Received
    
    state MovingToWaypoint {
        direction LR
        Calculate_Distance: 유클리드 기하학적 거리 계산
        Check_Radius: Acceptance Radius 비교
        
        Calculate_Distance --> Check_Radius
        Check_Radius --> Calculate_Distance: 거리 > NAV_ACC_RAD (정상 상태 오차 잔존)
    }
    
    MovingToWaypoint --> WaypointReached: 거리 <= NAV_ACC_RAD
    WaypointReached --> [*]: 다음 임무로 전환

위의 FSM 전이도에서 명확히 드러나듯, 기동 중인 에이전트가 지속적인 바람의 저항력과 시스템의 추력이 완벽하게 평형을 이루면서 목표 지점 바깥에 위치(\Vert e_{SS} \Vert_2 > R_{acceptance})하게 되면, FSM의 상태는 Check_Radius에서 영구적으로 실패(False) 조건을 받아 결코 WaypointReached 상태로 빠져나가지 못한다.

3. QGroundControl (QGC) 관점에서의 현상 관측과 시사점

교착 상태에 직면한 무인기는 QGroundControl(GCS) 인터페이스 상에서 다음과 같이 관측된다. 기체의 실시간 궤적이 목표지점 근처에서 특정한 오프셋(Offset)을 둔 채 정지 비행(Hovering) 시스템으로 고착된 것처럼 지도(Map) 스크린 상에 나타난다. 이 때 GCS 상단의 활성 비행 모드 알림 컴포넌트는 여전히 비행이 진행 중(예: Mission 또는 Hold로 전환되지 않음)인 것으로 계속 표출되며, MAVLink의 MISSION_ITEM_REACHED 메시지 패킷이 GCS로 전혀 수신되지 않는다.

이는 자율 비행(Autonomous Flight) 신뢰성과 안전성에 직결되는 중대한 고장 메커니즘으로 취급될 수 있다. 특히 군집 비행(Swarm) 중이거나 후속 타임 윈도우 스케줄링(Time Window Scheduling)이 엄격하게 짜인 임무인 경우, 이 교착 상태가 전체 임무의 연속적인 시스템 장애 전파(Fault Propagation)를 유발할 위험을 내포한다.

결과적으로 PX4-Autopilot 및 GCS 설계 관점에서, 이러한 영구적 무한 루프(Infinite Looping) 문제를 원천적으로 방지하기 위해 다음과 같은 제어 공학적 결함을 식별하고 완화(Mitigation)해야 할 당위성을 제시한다.

  • 바람 추정기(Wind Estimator)를 통한 피드포워드(Feed-forward) 보상 기법 도입의 필요성 확인하라.
  • 안티 와인드업(Anti-Windup)이 적용된 강건한 적분 제어기의 이득(Gain) 설정 가이드라인 (예: K_{i} 이득 증강 전개)을 점검하라.
  • 도달 실패 시간을 판별하는 강제 도달 인정(Forced Arrival Acceptance) 또는 타임아웃(Timeout) 릴레이 트리거(Relay Trigger) 메커니즘의 도입 기법을 구상하라.
