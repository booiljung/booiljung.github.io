46.4.4.3. 목표 지점 인근에서의 잔류 선속도(Residual Linear Velocity) 지수적 감쇠(Exponential Decay) 모델 및 정적 호버링(Static Hovering) 상태 전이 임계값(Threshold) 판별 논리

다기체 역학(Multi-body Dynamics) 시스템인 드론은 목표 지점(Waypoint 또는 Goto Target) 반경 내에 진입하더라도 관성과 공기 저항, 그리고 위치 제어기의 미세한 피드백 오차로 인해 잔류 선속도(Residual Linear Velocity)를 지니게 된다. PX4-Autopilot(v1.14 기준)은 기체가 목표점 반경 안에 진입하면 궤적 제어 단계를 마무리하고, 이 잔류 속도를 지수적으로 감쇠(Exponential Decay)시켜 완벽한 정적 호버링(Static Hovering) 상태로 부드럽게 전이(Transition)시키는 고도화된 소프트웨어 논리를 포함하고 있다.

1. 잔류 선속도(Residual Linear Velocity) 지수적 감쇠 모델

오버슈트를 방지하기 위한 제동(Braking) 단계를 거친 기체는 수용 반경(NAV_ACC_RAD) 내부에 진입할 때 0에 가까운 미세한 선속도를 가지게 된다. 이때 선형 감속을 계속 적용하거나 일반적인 위치 제어 게인(Position Control Gain)을 그대로 사용할 경우, 목표점 주위를 미세하게 맴도는 진동(Oscillation) 현상이나 한계 주기(Limit Cycle) 오류가 발생할 수 있다.

이를 극복하기 위해 PX4는 반경 내부에서 목표 속도(Velocity Setpoint, \vec{v}_{sp})를 지수 감쇠 함수 형태로 강제하여 0에 점근수렴(Asymptotic Convergence)하도록 유도한다.

미소 시간 \Delta t 동안의 속도 감쇠 모델은 다음과 같이 이산 시간(Discrete-time) 방정식으로 표현된다.

\vec{v}_{sp}(k+1) = \vec{v}_{sp}(k) \cdot e^{-\lambda \cdot \Delta t}

여기서 감쇠 상수 \lambda는 기체의 반응성(MPC_XY_VEL_P 등)과 연계되어 결정되며, 오차가 줄어들수록 제어력이 부드럽게 풀리게(Relaxation) 만들어 기체가 매우 안정적으로 정지 상태에 도달하도록 한다.

2. 정적 호버링(Static Hovering) 상태 전이 임계값(Threshold) 판별

수용 반경 안에서 속도가 지수적으로 감쇠됨에 따라, 시스템은 언제 비행 임무를 구체적으로 완료(“Mission Item Reached”)했다고 선언할지 엄격한 임계값(Threshold) 평가를 수행한다. PX4의 내비게이터(Navigator) 모듈은 다음 두 가지 조건의 논리곱(AND)을 통해 진정한 도달을 판별한다.

  1. 위치 임계값(Position Threshold): \vert \vec{P}_{target} - \vec{P}_{current} \vert < R_{accept}
  2. 속도 임계값(Velocity Threshold): \vert \vec{V}_{current} \vert < V_{threshold}

여기서 V_{threshold}는 기체가 정지했다고 간주할 수 있는 최소 선속도 한계선이다. 속도 임계값을 확인하지 않으면 기체가 지나치게 빠른 속도로 반경을 스쳐 지나가는(“Fly-by”) 순간에도 도달로 오판하는 심각한 플라이어웨이(Flyaway) 현상이 야기될 수 있다.

상태 전이 메커니즘

위 논리곱 조건이 충족되면 PX4는 내부 상태 머신(State Machine) 상에서 제어 루프를 다음과 같이 전이시킨다.

graph TD
    A[Waypoint Tracking Phase] -->|Enter Acceptance Radius| B[Exponential Velocity Decay Phase]
    B --> C{Is |Velocity| < Threshold?}
    C -- No --> B
    C -- Yes --> D[Trigger 'Arrival' / Mission Reached]
    D --> E[Transition to Hold Mode / Static Hovering]
    E --> F[Nullify XY Velocity Setpoint to exactly 0.0]

도달 판별이 완료되면 조향(Heading/Yaw)이 목표 각도를 완전히 정렬할 때까지 대기하거나, 다음 이동 명령이 들어올 때까지 위치 홀드(Auto-Hold) 상태로 고정(Clamp)된다.

3. Ardupilot과의 제어 논리 비교

  • PX4-Autopilot: 위치 추종 업무(FlightTaskAuto)가 끝나면 명시적으로 잔류 속도를 감쇠시키는 상태 체인이 존재하며, 도달 완료 조건을 위치 반경 및 속도 노름(Norm) 두 가지 관점에서 매우 타이트하게 평가한다.
  • Ardupilot: 전통적으로 WPNAV_RADIUS 영역 내에 기체가 도달하면 다음 웨이포인트(Next Waypoint)로 즉시 궤적을 전환하거나, 도착 후 대세(Hold)를 유지하기 위해 L1 컨트롤러 내에 자체적인 감속 필터를 가동한다. 최신 S-Curve 시스템 도입 이후에는 반경 내부에서의 속도 궤적 평활화 기능이 PX4의 지수 감쇠 모델과 유사한 역할을 수행하고 있다.

4. 소스 코드 구현체 분석

잔류 속도 감쇠와 호버링 상태 전이는 PX4 소스 트리의 여러 단계에서 협력하여 이루어진다.

  • 도달 판별 로직: src/modules/navigator/navigation.cpp의 mission_item_reached 함수 부근에서 속도 및 위치 기반의 최종 임계값을 판별한다.
  • 감쇠 제어 적용: src/modules/flight_mode_manager/tasks/Auto/FlightTaskAuto.cpp 내부에서 상태가 전이되기 직전, 타겟 속도를 점진적으로 줄이는 로직이 구동된다.
  • 이 구역에서 오차를 수렴시키는 알고리즘은 센서 노이즈(Sensor Noise, GPS Position Drift)로 인해 발생하는 속도의 요동(Chattering)을 무시하기 위해 저역통과필터(Low Pass Filter)를 동반하여 판별 로직의 강건성(Robustness)을 확보한다.

5. 결론

목표 지점 인근에서의 지수적 잔류 속도 감쇠 및 동적 임계값 판별 논리는 PX4-Autopilot이 정밀 택배 배송(Drone Delivery), 초정밀 측량비행, 무인기 도킹 스테이션(Docking Station) 자율 착륙 등 극한의 위치 제어 품질이 요구되는 산업 분야에 쓰일 수 있는 원동력이다. 이 알고리즘을 통해 기체는 목표점에 도달할 시점에 오버슈트나 진동 없이 우아하고(Elegant) 단호한 정적 호버링(Static Hovering)을 시전할 수 있다.
