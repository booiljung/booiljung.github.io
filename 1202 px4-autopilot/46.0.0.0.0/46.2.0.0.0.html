46.2 MAVLink 통신 프로토콜 기반 Goto/Reposition 제어 명령 해석 및 기구학적 좌표계 변환 메커니즘

본 절에서는 지상 관제 시스템(GCS; Ground Control Station) 혹은 외부 컴패니언 컴퓨터(Companion Computer)로부터 전달되는 Goto 및 Reposition 등 단일 목적지 이동 제어 명령이 PX4-Autopilot(기준 버전: v1.14.0) 내부에서 어떠한 MAVLink 프로토콜 절차를 거쳐 수신되는지, 그리고 이를 제어기(Controller)가 이해할 수 있는 형태의 로컬 좌표계(Local Coordinate Frame)로 변환하는 기구학적(Kinematic) 메커니즘에 대해 심층적으로 다룬다.

1. MAVLink Goto/Reposition 제어 명령 수신 아키텍처

무인항공기 시스템의 이동 제어는 통상적으로 MAVLink 2.0 프로토콜 규격에 정의된 COMMAND_LONG 혹은 COMMAND_INT 메시지 패킷 내부의 명령 ID(Command ID)를 통해 이루어진다. 단일 목적지 이동을 지시할 때 가장 범용적으로 활용되는 명령은 MAV_CMD_DO_REPOSITION이다.

해당 명령을 수신한 PX4-Autopilot의 내부 수신 모듈(mavlink_receiver_main)은 패킷의 페이로드(Payload)를 분석하여 기하학적 파라미터(위도, 경도, 고도 및 변경할 기수각 등)를 추출한다. 추출된 데이터는 PX4 운영체제 내부의 메시지 중심 미들웨어인 uORB(Micro Object Request Broker) 프로토콜을 관통하여 vehicle_command 토픽(Topic) 형태로 발행(Publish)된다.

다음은 MAVLink 프로토콜을 통해 수신된 제어 명령이 내부 모듈로 전달되는 파이프라인(Pipeline)을 보여준다.

2. 참조 좌표계(Reference Frame)의 정의와 기구학적 상관관계

지구상에서 무인항공기가 3차원 공간을 비행하기 위해 제어 명령을 정밀하게 수행하려면 다양한 좌표계 간의 기하학적 관계를 정의해야 한다. 단일 목적지 이동 제어에서는 주로 **글로벌 좌표계(Global Coordinate System)**와 로컬 좌표계(Local Coordinate System) 두 가지 프레임이 사용된다.

  1. WGS84 글로벌 좌표계(World Geodetic System 1984):
    GCS 애플리케이션(QGroundControl 등) 사용자 인터페이스 기반 맵에서 선택한 지점 정보는 일반적으로 위도(Latitude), 경도(Longitude), 평균 해수면 기준 고도(AMSL; Above Mean Sea Level)의 구면 좌표 기반 좌표계로 전달된다.
  2. 로컬 NED 직교 좌표계(Local North-East-Down Frame):
    PX4의 위치 제어기(Position Controller)와 추정기(Estimator, EKF2 등)는 자코비안 행렬(Jacobian Matrix) 연산 및 선형 대수적 미분/적분의 편의성을 확보하기 위해 로컬 직교 좌표계를 기준으로 제어 입력을 생성한다. 원점(Origin)은 드론이 시동(Arming) 혹은 이륙(Takeoff)한 지점 체계로 설정된다. 북(North) 축이 X, 동(East) 축이 Y, 하단 지면 방향(Down)이 Z축이 된다.

명령 전달 과정에서 GCS는 WGS84 좌표계로 목적지 정보를 전송하지만, 비행 제어를 실질적으로 수행하는 위치 제어(Position Control) 모듈은 로컬 NED 좌표 단위의 Setpoint(설정값/추종값)를 요구하므로 필수 원활한 좌표 변환(Coordinate Transformation) 동작이 요구된다.

3. 기구학적 좌표계 평면 투영 변환(Map Projection Transformation)

구면 위에 정의된 글로벌 WGS84 3차원 측지 좌표(Geodetic Coordinates) P(lat, lon, alt)를 로컬 직교 좌표계 P(x, y, z)_{NED}로 변환하기 위해, PX4 내부에서는 map_projection이라는 투영 라이브러리를 사용한다. 이는 국소 지역 내에서 지구 표면을 2차원 평면(Euclidean Plane)으로 근사하여 삼각 함수를 통해 X와 Y 오프셋을 산출하는 방식이다.

이러한 지도 투영 변환(Map Projection) 메커니즘을 상세히 분석하면 다음의 연산 단계로 나눌 수 있다.

  1. 기준점(Reference Point) 설정: EKF(Extended Kalman Filter)를 통해 글로벌 위치의 유효성 검증을 완료한 후, 초기 로컬 원점(Local Origin Point) 계산 목적으로 홈 기준점의 글로벌 좌표 Ref(lat_0, lon_0, alt_0)를 저장한다.
  2. 이격 거리(Displacement) 투영 연산: 입력된 타겟 위도/경도와 기준 위도/경도의 라디안(Radian) 차이를 이용하여, 구면 삼각법(Spherical Trigonometry)에 기반한 하버사인 변형 혹은 테일러 급수로 근사화된 맵 매핑 함수에 적용한다. 이를 통해 북동(North-East) 축 평면 상에서의 오차 단위 벡터 오프셋 거리(미터(m)) 계산 결과를 획득한다.
    PX4 펌웨어 내 lib/geo/geo.cpp의 map_projection_project() 함수 참조
  3. 고도 변환(Altitude Transformation): 글로벌 고도(AMSL) alt와 로컬 고도 alt_0의 차이를 부호 반전하여(NED 프레임에서는 지면으로 갈수록 Positive) Down 축인 Z 좌표를 산출한다.
    Z_{NED} = -(alt_{target} - alt_{reference})

4. 제어 변수의 동적 전달 메커니즘 요약

좌표계 변환을 거쳐 얻어진 로컬 NED 타겟 설정값(Target Setpoint)은 비행 환경적 제약 사항과 드론의 최대 속도(Maximum Velocity), 가속도(Acceleration), 혼합 제어 한계(Jerk Limit) 등을 고려하여 경로 추종 알고리즘(Trajectory Path Builder) 내로 입력된다. 이 메커니즘을 통해 GCS의 MAVLink 제스처 입력은 최종적으로 position_setpoint_triplet 토픽이 되어 기체를 우아하고 부드럽게 지정된 목표 지점으로 유도하는 매개변수로 기능하게 된다.

결론적으로 PX4의 Goto 명령 처리 구조는, GCS와의 유연한 추상화 통신을 지원하는 원시 패킷 수신 계층과, 이를 엄격한 로컬 자코비안 로직과 제어 공학 방정식에 결합시켜 주는 기구학적 변환 중간 미들웨어 레이어 설계가 탁월하게 추상화되어 분리된 우수한 사례라 할 수 있다.
