46.4.2.3. 이동체 동역학을 고려한 속도 벡터(Velocity Vector) 내적(Inner Product) 기반의 가상 수용 반경(Virtual Acceptance Radius) 확장 이론

자율 비행 기반의 무인항공기(UAV) 제어 시스템에서 단순한 유클리드 기하학적 거리(Euclidean Distance)만을 기준으로 목적지 도달(Arrival)을 판별하는 것은 다중 웨이포인트(Waypoint) 비행 시 여러 가지 관성적 한계를 노출한다. 특히 고속 순항(High-speed Cruise) 중에는 기체의 운동 에너지(Kinetic Energy)로 인해 지정된 수용 반경(Acceptance Radius)을 순식간에 지나치거나, 선회 반경이 과도하게 커지는 현상이 발생한다.

이러한 문제를 해결하기 위해 PX4-Autopilot 아키텍처(펌웨어 버전 1.14 기준)의 내비게이터(Navigator) 시스템에서는 이동체의 기구학적(Kinematic) 상태를 실시간으로 반영하여 도달 영역을 논리적으로 확장하는 기법을 사용한다. 본 절에서는 속도 벡터(Velocity Vector)의 내적(Inner Product) 연산을 활용하여 기존의 고정된 기하학적 수용 반경을 확장하는 가상 수용 반경(Virtual Acceptance Radius) 확장 이론에 대해 제어공학적 시각으로 심층 분석한다.

1. 정적 수용 반경의 동역학적 한계

드론이 단일 목적지 이동(Goto) 임무를 수행할 때 초기 제어 알고리즘은 오직 현재 위치 좌표(\mathbf{p}_{current})와 목표 좌표(\mathbf{p}_{target}) 사이의 유클리드 거리 ||\mathbf{p}_{target} - \mathbf{p}_{current}|| 에만 의존했다.

이 경우 만약 기체가 10 m/s 이상의 고속으로 이동 중이라면, 설정된 수용 반경(예: NAV_ACC_RAD = 2.0m) 영역 내에 머무르는 시간은 고작 0.4초 이하에 불과하다. 이 찰나의 순간 동안 GPS 센서 데이터가 스킵(Skipped)되거나 EKF(Extended Kalman Filter) 융합 주기가 어긋날 경우, 비행 제어 시스템은 목적지 도달을 인지하지 못한 채 목표점을 지나쳐버리는 위치 오버슈트(Position Overshoot) 현상을 겪게 된다.

이를 방지하기 위해 단순히 정적인 반경을 무한정 늘리는 것은 정밀한 비행 궤적 생성을 불가능하게 만든다. 따라서 기체의 ’현재 진행 방향(Heading)’과 ’이동 속도(Speed)’를 적극적으로 고려하여 도달 허용 공간을 동적으로 유연화하는 알고리즘이 필수적이다.

2. 속도 벡터 내적(Inner Product)을 이용한 가상 도달 선언 기법

고속 비행 중 도달 판정을 보다 실시간적이고 유연하게 만들기 위해, 최신 제어 알고리즘은 기하학적 평면 개념을 차용한다. 핵심 아이디어는 목표점을 지나 기하학적으로 수직인 가상의 통과 평면(Virtual Passing Plane)을 세우고, 기체가 이 평면을 지나는 순간을 도달 트리거(Arrival Trigger)로 활용하는 것이다.

2.1 벡터 기하학적 수식화

기체의 현재 3차원 위치 벡터를 \mathbf{p}, 목표점(Waypoint)의 위치 벡터를 \mathbf{w}, 이전 목표점(Previous Waypoint)의 위치 벡터를 \mathbf{w}_{prev}라 가정하라.

  1. 목표 지향 벡터(Target Direction Vector, \mathbf{d}): 이전 웨이포인트에서 현재 목표점을 향하는 단위 방향 벡터(Unit Direction Vector)를 계산한다.
    \mathbf{d} = \frac{\mathbf{w} - \mathbf{w}_{prev}}{||\mathbf{w} - \mathbf{w}_{prev}||}
  2. 현재 위치 오차 벡터(Position Error Vector, \mathbf{e}): 목표점과 현재 위치 간의 오차 벡터를 정의한다.
    \mathbf{e} = \mathbf{w} - \mathbf{p}
  3. 내적(Dot Product)을 통한 투영(Projection): 오차 벡터 \mathbf{e} 와 목표 지향 벡터 \mathbf{d} 의 내적 공간(Inner Product Space)을 분석한다.
    s = \mathbf{e} \cdot \mathbf{d}

여기서 얻어진 스칼라 값 s는 기체가 목표점까지 남은 실질적인 직선상의 거리를 의미한다.
만약 s \le 0 가 되는 순간, 이는 수학적으로 기체 위치 벡터 \mathbf{p}가 목표점을 수직으로 가로지르는 무한한 평면(통과 평면)을 넘어섰음을 시사한다. 이 논리를 바탕으로, 기존의 원형(혹은 구형) 수용 반경을 통과 평면 개념으로 확장하는 것을 ‘가상 수용 반경(Virtual Acceptance Radius)’ 기법이라 일컫는다.

3. 이동체 동역학(Kinematics)을 반영한 선행 도달 판별(Early Arrival)

앞서 설명한 단순 투영법에 기체의 속도(Velocity) 벡터 \mathbf{v} 와 하드웨어의 감속 능력을 나타내는 시스템 지연(System Lag) 시정수 \tau를 추가로 도입하라.

목표점 도달 시 자연스러운 선회(Coordinated Turn) 혹은 감속 정지를 위해, 관성이 큰 기체일수록 실측 거리가 도달하기 전에 선제적으로 ’도달 선언’을 내려주어야 한다.

3.1. 관성 기반 선제 예측 방정식

기체의 현재 이동 속도 벡터를 \mathbf{v} = (v_x, v_y, v_z) 라 할 때, 기체의 짧은 미래 예측 위치 \mathbf{p}_{pred} 는 1차 동역학 전방 오일러 근사(Forward Euler Approximation)에 의해 다음과 같이 나타낼 수 있다.
\mathbf{p}_{pred} = \mathbf{p} + \tau \mathbf{v}

이 예측 위치를 내적 기반 통과 판별 식에 다시 대입하라.
s_{pred} = (\mathbf{w} - \mathbf{p}_{pred}) \cdot \mathbf{d}
s_{pred} = (\mathbf{w} - \mathbf{p} - \tau \mathbf{v}) \cdot \mathbf{d}

이 공식에 따르면, 기체의 속도 벡터 \mathbf{v} 가 목표 지향 벡터 \mathbf{d} 와 같은 방향(내적값이 양수)일수록, 그리고 속도가 빠를수록 항 \tau \mathbf{v} \cdot \mathbf{d} 가 커져서 s_{pred} \le 0 조건을 더욱 일찍 달성하게 된다.

제어공학 관점에서 볼 때, 이 방식은 비례-미분(PD) 제어기의 미분(Derivative) 선행 작용과 유사한 효과를 내어 오버슈트를 극적으로 억제한다. 비행 제어기(Flight Controller)의 궤적 생성기(Trajectory Generator)는 기체가 실제 물리적 반경에 도달하기 전이라도 동역학적으로 충분히 빠른 속도를 지니고 있다면 선제적인(Proactive) 상태 전이 명령을 내려 연속적인 곡선 궤적(Continuous Curve Trajectory)을 형성할 수 있도록 보장한다.

3. Ardupilot 관제 대비 PX4 아키텍처의 우수성

Ardupilot은 이와 유사한 개념을 L1 내비게이션 제어기(L1 Navigation Controller) 내의 리드 인 로직(Lead-in Logic)을 통해 처리한다. 하지만 Ardupilot은 주로 속도 프로파일의 감속도를 미리 역산하는 스플라인(Spline) 추종에 강하게 종속되는 경향이 있다.

반면 PX4-Autopilot 아키텍처는 궤적 추종기(Tracker)와 별개로, 임무 내비게이터(Navigator) 모듈 레벨에서 벡터 내적 기반의 기하학적 교집합 계산 로직을 최상위 도달 임계 조건으로 완전히 독립시켜 구현해두었다 (GitHub 소스코드의 position_setpoint_triplet_s 토픽 평가부 참조).

이러한 분리 설계(Decoupled Design)는 외부 지상 관제 시스템(GCS)인 QGroundControl 측에서 복잡한 임무 계획 곡선을 사전에 그려주지 않아도, 비행 제어 펌웨어 모듈 스스로가 실시간 비행 상태에 맞춰 가장 역학적으로 최적화된 가상 반경을 자체 생성(Self-generating) 해낸다는 뛰어난 소프트웨어 공학적 우수성을 갖는다.
