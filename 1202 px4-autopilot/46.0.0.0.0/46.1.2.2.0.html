## 0.1 이노베이션 시퀀스(Innovation Sequence) 기반 센서 퓨전(Sensor Fusion) 데이터의 확률론적 유효성(Probabilistic Validity) 평가

무인기가 단일 목적지 이동(Goto/Reposition) 명령을 수행하기 위해서는 기체의 항법 시스템이 외부 환경의 외란을 견디며 강건한(Robust) 상태 추정치를 유지해야 한다. PX4-Autopilot의 확장 칼만 필터(EKF2)는 다중 센서 데이터를 병합하는 과정에서 불량 데이터(Outlier)를 걸러내어 전체 추정 알고리즘의 붕괴를 예방하는 보안 기제를 갖추고 있다. 그 핵심이 바로 이노베이션 시퀀스(Innovation Sequence)를 기반으로 한 센서 퓨전 데이터의 확률론적 유효성 평가 메커니즘이다.

0.1.1 이노베이션(Innovation)과 테스트 비율(Test Ratio)의 공학적 정의

비선형 상태 추정 이론에서 ’이노베이션(y 혹은 \tilde{z})’이란, 필터 메커니즘이 이전 상태를 기반으로 예측한 이상적인 측정치(Predicted Measurement)와 실제 센서 하드웨어가 읽어들인 관측치(Actual Measurement) 간의 잔차(Residual, 차이점)를 일컫는다. 다시 말해, 센서가 보내온 새로운 데이터가 필터의 기존 예상과 비교해 얼마나 “얼마나 놀라운가(Surprising)“를 나타내는 척도이다.

PX4 EKF2 아키텍처는 단순히 잔차의 크기만으로 데이터를 배제하지 않고, 그 잔차를 “이노베이션 공분산(Innovation Covariance)“으로 나눈 지표인 테스트 비율(Innovation Variance Test Ratio) 을 계산해 데이터의 수용 여부를 가늠한다.

  • 테스트 비율 공식 개념: \text{Test Ratio} = \frac{\text{Innovation}^2}{\text{Gate}^2 \times \text{Innovation Variance}}
  • 테스트 비율이 1.0 미만이면: 센서 측정치가 확률적 오차 범위(Gate) 안에 들어왔다고 간주되어 상태 업데이트(State Update)에 정상 융합(Fusion)된다. (통상 0.3 미만이면 매우 우수한 신호 품질로 간주)
  • 테스트 비율이 1.0 이상이면: 측정치가 일관성을 벗어난 이상치(Outlier)로 판별되어 필터 융합에서 즉시 기각(Reject)된다.

0.1.2 EKF2의 주요 테스트 비율 지표 및 파라미터 게이트(Gate) 튜닝

PX4 시스템은 각 측정 센서 도메인 별로 개별적인 테스트 비율을 추적하며, 이를 조절하는 게이트(Gate) 파라미터를 제공한다. 단일 목적지 이동(Goto) 체계의 선행 조건이 성립하려면 아래의 핵심 테스트 비율들이 지속적으로 1.0 미만을 유지해야만 한다.

  • pos_test_ratio (GPS 수평 위치 테스트 비율): GPS 위성 신호의 다중 경로 오차(Multipath Error)나 심각한 전파 간섭 시 급증한다. 연관 게이트 파라미터는 EKF2_GPS_P_GATE이다.
  • vel_test_ratio (GPS 속도 테스트 비율): GPS 속도 데이터의 튀는 현상(Spike)이나 IMU(관성 측정 장치)의 클리핑(Clipping), 앨리어싱(Aliasing) 문제가 발생할 때 상호 불일치로 인해 수치가 상승한다. 관련 게이트 파라미터는 EKF2_GPS_V_GATE이다.
  • hgt_test_ratio (고도 테스트 비율): 기압계 혹은 거리 측정 센서(Rangefinder) 데이터의 이노베이션이다.
  • mag_test_ratio (자기장 테스트 비율): 모터 전원선에 의한 전자기 간섭 또는 외부 철골 구조물에 의한 나침반(Compass) 교란이 발생하면 수치가 치솟아 데이터가 기각된다. (관련 게이트: EKF2_MAG_GATE)

0.1.3 센서 데이터 기각 시의 EKF2 복구 로직과 제어권 이관

만일 EKF2가 특정 센서(예: GPS)의 pos_test_ratio가 1.0을 초과하여 데이터를 연속적으로 기각하는 사태가 발생하면, 시스템은 다음과 같이 대응한다.

  1. 임시 기각 (Temporary Rejection): 초기 기각 시점에는 필터가 오로지 IMU 적분만을 활용하여 데드 레코닝(Dead Reckoning) 비행을 일정 시간(수 초 단상) 유지하도록 시도한다.
  2. 공분산 인플레이션 (Covariance Inflation): 불일치가 장기화될 경우 시스템은 자신의 예측치 안정성에 대한 자체 신뢰도(공분산 역수)를 점진적으로 낮춘다. 이를 통해 이노베이션 게이트(Gate)를 논리적으로 넓혀주어, 다시 GPS 데이터가 비집고 들어올 수 있도록 필터를 개방하는 복구(Recovery) 절차를 가동한다.
  3. 유효성 상실 및 타임아웃 전이: 그럼에도 복구되지 않으면 10초 내외의 시간 만료(Timeout) 로직이 발동하여 전역 위치 유효성 플래그(global_position_valid)를 거짓(False)으로 내린다. 즉각적으로 위치 제어 불능 상태가 되며, 진행 중이던 Goto 임무는 중단되고 안전 장치(Failsafe) 체제인 고도 유지(Altitude Hold) 등급이나 위치 기반 자동 착륙 등의 비상 모드로 강제 스위칭된다.

0.1.5 지상 관제 시스템 및 진단 도구를 통한 실무적 관측

실제 GCS 인프라에서 자율 에이전트 비행 관리 기술을 개발할 때는 MAVLink의 ESTIMATOR_STATUS 속에서 pos_horiz_accuracy와 각종 플래그를 읽어 들이는 한편, 근본적인 원인 규명을 위하여 비행 데이터 로깅(Logging) 분석에 크게 의존한다. PX4의 ‘Flight Review’ 도구 체계(비행 직후 ULog 파싱 결과를 띄우는 웹 서비스 또는 MAVExplorer 도구)를 활용하여, 각 센서들의 이노베이션 테스트 비율 그래프(ekf2_innovations)가 어느 지점에서 1.0의 천장을 “치는(Flat-lining)” 증상을 발현했는지 트러블슈팅(Troubleshooting)을 수행하는 과정이 펌웨어 및 관제 연동 솔루션 개발자의 필수 덕목으로 꼽힌다.
