46.5.1.1 3차원 다각형 지오펜스(Polygonal Geofence) 기하학적 경계면과 예상 비행 궤적 간의 광선 교차 판별(Ray-Casting Intersection) 알고리즘

1. 개요 (Introduction)

비행 금지 구역(No-Fly Zone)이나 특정 임무 구역을 설정할 때, 단순한 원형 반경이 아닌 복잡한 3차원 다각형 형태의 지오펜스(Polygonal Geofence)가 필수적으로 요구된다. 특정 건물, 사유지, 통제 공역 등을 정밀하게 회피하기 위함이다. PX4-Autopilot은 기체의 현재 위치에서 목표 지점(Goto Setpoint)으로 향하는 예상 비행 궤적(동역학적 궤적의 선분 근사)이 3차원 다각형 경계면을 관통하는지를 실시간으로 판별하기 위해 광선 교차 판별(Ray-Casting Intersection) 알고리즘을 사용한다.

2. 차원 다각형 지오펜스의 기하학적 모델링 (Geometric Modeling of 3D Polygonal Geofences)

QGroundControl을 통해 GCS에서 다각형 펜스를 생성하면, MAVLink 프로토콜의 FENCE_POINT 메시지를 통해 꼭짓점(Vertices)의 WGS84 위경도 좌표 배열이 비행 제어기로 전송된다. PX4는 연산 효율성을 위해 이를 다음과 같이 모델링한다.

  1. 국소 직교 좌표계 변환 (Local Cartesian Projection): 위도와 경도(Lat/Lon)로 구성된 전역 좌표를 EKF 근원점(Origin)을 기준으로 하는 국소 NED(North-East-Down) 직교 좌표계로 사영(Projection)한다.
  2. 2.5D 형상 모델링: 완전한 임의의 3D 다면체가 아닌, 수평면 상의 다각형(Polygon) 기둥에 수직적인 상한/하한 고도(Roof and Floor)를 결합한 형태로 기하학적 볼륨(Volume)을 정의한다.

3. 광선 교차 판별 알고리즘 원리 (Principles of Ray-Casting Intersection)

PX4 비행 스택 내에서 예상 비행 궤적과 다각형 경계 간의 충돌 여부를 검사하는 핵심 로직은 다음과 같이 계산된다.

3.1 수평면 제약 검사 (Horizontal Plane Polygon Check)

기체의 현재 위치 \mathbf{P}_0 = (x_0, y_0)와 목표 지점 \mathbf{P}_1 = (x_1, y_1)를 잇는 선분(선속, Ray)과 다각형을 이루는 각 변(Edge) 간의 2차원 선분 교차(Line Segment Intersection)를 검사한다.

  • 두 선분 \overline{\mathbf{P}_0 \mathbf{P}_1} 와 펜스의 한 변 벡터 \overline{\mathbf{V}_i \mathbf{V}_{i+1}} 가 주어질 때, 벡터의 외적(Cross Product)을 활용하여 교차 여부를 판별한다.
  • 또한 Point-in-Polygon (Even-Odd Rule) 알고리즘을 사용하여 목표 지점 \mathbf{P}_1 자체가 다각형의 내부(Inclusion)에 있는지 외부(Exclusion)에 있는지를 빔을 가상으로 발사(Ray-Casting)하여 짝수/홀수 번 교차점 개수로 판별한다.

3.2 수직 고도 제약 검사 (Vertical Altitude Check)

수평적 교차가 없다 하더라도 기체의 수직 궤적이 설정된 최대/최소 고도(Z_{max}, Z_{min}) 평면을 관통하는지 검사한다. NED 좌표계에서 목표 고도 Z_{setpoint} 가 범위 밖으로 벗어나는 경우, 궤적은 해당 경계 평면의 교차점에서 멈추도록(클리핑) 처리된다.

4. PX4 소스 코드 레벨에서의 구현 분석

PX4 펌웨어 모듈 아키텍처 상 이 알고리즘은 주로 src/modules/systemlib/polygon.cpp 와 src/modules/flight_mode_manager/tasks/Utility/Geofence.cpp 라이브러리에 구현되어 있다.

  • 다각형 펜스 꼭짓점 리스트를 순회하며 선분들의 교차를 벡터 외적으로 구하는 최적화된 C++ 코드가 실행된다.
  • 부동소수점 오차(Floating-point precision issues)를 방지하기 위해 충분한 기하학적 마진(Epsilon)을 두어 교차점을 계산하도록 설계되어 있다.
  • 이러한 검사는 고주파 제어 루프 내부가 아닌 Navigator 또는 Flight Task의 궤적 생성 주기(약 10~50Hz)에서 호출되므로 RTOS 태스크 스케줄링에 오버헤드를 주지 않도록 가볍게 설계된다.

5. Ardupilot 대비 알고리즘 적용 차이

  • PX4의 접근: 궤적을 계획(Trajectory Planning)하는 단계에서 기하학적 교차점(Intersection Point)을 정확히 산출한 뒤, 이 교차점 직전에 부드럽게 멈추도록 저크(Jerk) 기반 궤적 제어기 세트포인트를 동적으로 클리핑(Clipping)하는 예측적 방식(Predictive Clipping)을 주로 사용한다.
  • Ardupilot의 접근: AP_Limit 및 AC_Avoid 라이브러리에서 다각형 지오펜스를 처리할 때, 기체의 물리적인 ’거리 여유도(Margin)’를 감시하다가 위반 상태에 근접하면 센서 회피 논리와 유사하게 가상의 척력(Repulsive Force) 혹은 제동 벡터를 가하여 부딪히듯 튕겨내는 동역학적 반응(Reactive Braking) 방식의 색채가 짙다.

결론적으로 PX4의 Ray-Casting 기반 다각형 지오펜스 교차 판별 알고리즘은 궤적 기하학을 명시적으로 해석하여 QGroundControl이나 MAVSDK를 통한 복잡한 임무의 안전성을 수학적으로 담보하는 필수 불결한 제어 소프트웨어 구성요소이다.
