46.4.4 기구학적 제약 조건(Kinematic Constraints)을 내포한 동적 도달 판별 및 위치 오버슈트(Overshoot) 제어 알고리즘

무인항공기(UAV)의 자동 비행(Auto Mode) 및 이동(Goto) 명령 수행 시 목표 지점 도달 판별(Arrival Determination)은 기체의 기구학적 제약 조건(Kinematic Constraints)을 필수적으로 고려해야 하는 정밀 제어의 영역이다. PX4-Autopilot(v1.14 기준)은 단순히 현재 위치와 목표 지점 사이의 기하학적 유클리드 거리(Euclidean Distance)만을 측정하는 것을 넘어, 기체의 현재 속도, 가속도 한계, 저크(Jerk) 제한 등의 운동학적 상태를 종합하여 동적 도달 판별 알고리즘을 적용한다. 이를 통해 목표 지점 근처에서의 위치 오버슈트(Position Overshoot)를 최소화하고 안정적인 호버링(Hovering) 또는 다음 임무로의 매끄러운 전이(Transition)를 보장한다.

1. 기구학적 제약 조건(Kinematic Constraints)의 이해

멀티로터 기체 시스템은 관성을 가지고 있으므로 순간적으로 속도를 0으로 만들거나 무한대의 가속도를 낼 수 없다. 위치 제어기(Position Controller)는 이러한 물리적 한계를 소프트웨어적으로 제약(Constraints)하여 제어 명령을 생성한다.

  • 최대 가속도 및 감속도(Maximum Acceleration/Deceleration): 파라미터 MPC_ACC_HOR_MAX 및 MPC_JERK_AUTO 등을 통해 물리적으로 불가능한 급기동을 방지한다.
  • 예측 제동 거리(Predictive Braking Distance): 기체의 현재 선속도(Linear Velocity)와 최대 감속도를 기반으로, 기체가 완전히 정지하기까지 필요한 최소 거리를 주기적으로 계산한다.

PX4의 비행 작업 로직(FlightTaskAuto 모듈 등)은 이러한 제약 조건을 궤적 생성기(Trajectory Generator)에 통합하여, 가속도 및 감속도 한계 내에서 목표 지점 도달을 판단한다.

2. 동적 도달 판별 알고리즘 구조

기존의 정적 수용 반경(Static Acceptance Radius) 기반 도달 판별 알고리즘은 단순히 반경 내 진입 여부만 확인하므로 관성에 의한 오버슈트가 빈번하게 발생할 위험이 있다. 이를 보완하기 위해 PX4는 위치 오차와 선속도의 노름(Norm)을 융합한 도달 평가 기준을 적용한다.

목표 지점 P_{target}과 기체의 현재 위치 P_{current}, 현재 속도 V_{current}가 주어졌을 때, 기구학적 한계를 고려한 수용 조건은 다음과 같은 형태를 띤다.

\vert P_{target} - P_{current} \vert \le R_{acc} + f(V_{current}, A_{max})

여기서 R_{acc}는 기본 수용 반경 파라미터(NAV_ACC_RAD)를 나타내고, f(V_{current}, A_{max})는 현재 속도와 물리적 감속 능력을 바탕으로 동적으로 계산되는 추가 허용 거리(Braking Margin)이다. 이 함수는 기체가 목표 반경에 도달했을 뿐만 아니라 멈출 수 있는 기동 범위 안에 들어왔는지를 판별하게 해 준다.

S-Curve(Jerk-limited) 궤적 프로파일의 적용

목표 지점으로 접근하는 과정에서 잔류 속도를 부드럽게 감쇠시키기 위해, 제어기는 저크(Jerk, 가속도의 미분값)를 제한하는 S-커브 궤적 프로파일(S-Curve Trajectory Profile)을 추종한다. 이 과정에서 피드포워드(Feedforward) 제어기가 위치 오버슈트를 강력하게 억제하는 상호 보완적 역할을 수행한다.

graph TD
    A[Start: Goto Command Received] --> B[Generate S-Curve Trajectory]
    B --> C[Compute Current Kinematics: Pos, Vel, Acc]
    C --> D{Is Dist < NAV_ACC_RAD + Braking Dist?}
    D -- No --> E[Continue S-Curve Tracking]
    E --> C
    D -- Yes --> F[Check Residual Linear Velocity]
    F --> G{Is Velocity < Threshold?}
    G -- No --> H[Apply Exponential Deceleration]
    H --> C
    G -- Yes --> I[Trigger 'Mission Item Reached' / Transfer to Hold]

3. 위치 오버슈트(Overshoot) 제어 및 방어 기법

위치 오버슈트 방지는 단순히 위치 제어(P-Control)의 이득 값을 낮추는 것으로만 해결되지 않는다. 제어 이득을 무분별하게 낮추면 도달 시간이 길어지는 지연(Tracking Lag) 문제가 발생하므로 기구학적 제약 조건 안에서만 이득 제어를 수행해야 한다.

  1. Feedforward 제어 출력 억제:
    목표점 도달이 임박할 경우 제동 거리 예측 모델이 작동하면서, 속도 제어 루프(Velocity Control Loop)로 입력되는 목표 속도 참조값(Velocity Reference)을 능동적으로 축소시킨다. 이는 PI 제어기의 적분기 와인드업(Integrator Windup) 방지 로직과 결합하여 감속 구간의 오버슈트 유발 요인을 근본적으로 제거한다.
  2. 에너지 감쇠 논리(Energy Decay Logic):
    목표점의 기본 반경 내에 기체가 진입하게 되면, 병진 운동 에너지(Translational Kinetic Energy)를 감쇠시키는 모드로 알고리즘이 전환된다. 이때 속도 제어기는 제로 위치 오차(Zero Position Error)로의 점진적 수렴 대신 영속도(Zero Velocity) 추종을 더 높은 우선순위로 두도록 내부 가중치가 변동된다.

4. Ardupilot 시스템과의 제어 아키텍처 비교

Ardupilot은 전통적으로 정밀한 오버슈트 제어를 위해 L1 내비게이션(L1 Navigation Controller) 및 WPNAV 관련 파라미터(WPNAV_ACCEL, WPNAV_SPEED)를 활용하는 반면, 최신 버전으로 올수록 S-Curve를 전면적으로 도입하여 위치 제어 성능을 고도화하고 있다. 관련된 연구 서적인 “Design and Implementation of a Trajectory Generator for Multicopters” 구조를 많이 따르고 있다.

  • PX4-Autopilot: FlightTask 및 PositionControl 등 분리형 아키텍처 내부에서 Jerk-limited 궤적 제어기와 다중 캐스케이드(Cascade) PID 제어를 결합하여 각 기동 단계마다 독립적인 제약 조건을 실시간으로 계산한다 (MPC_JERK_AUTO, MPC_ACC_HOR).
  • Ardupilot: 위치 제어기 내에서 SCurve 객체 모듈을 직접 호출하여 목표 위치까지의 1차원 이산 시간(Discrete-time) 이동 거리를 투영(Projection)하는 방식을 취한다.
  • 양쪽 공히 기구학적 제약 조건을 만족하면서 위치 오버슈트를 줄이는 방향으로 발전했으나, Ardupilot이 파라미터 구성을 통한 사용자의 포괄적 튜닝(Comprehensive Tuning) 접근이 더 용이한 편이라면, PX4는 내부 상태 머신(State Machine)에 의한 물리 모델 기반 예측의 의존성이 더 강하다.

5. 소스 코드 분석 (FlightTaskAuto 모듈)

PX4의 C++ 소스 코드에서는 목표 지점의 도달 상태 머신을 평가할 때 이러한 제동 거리가 직접 계산된다.

  • 모듈 위치: src/modules/flight_mode_manager/tasks/Auto/FlightTaskAuto.cpp 내부.
  • 기체의 현재 위치 인덱스와 목적지 인덱스를 판별하는 함수 영역에서, _target_acceptance_radius 변수는 속도와 가속도에 의한 제동 능력에 따라 가변적으로 업데이트됨을 분석할 수 있다. 오버슈트를 방지하기 위해 계산된 제동 거리가 현재 목표점까지의 남은 거리보다 큰 경우 즉각적으로 감속 명령이 하달되는 제어 논리가 반영되어 있다.

6. 결론

기구학적 제약 조건을 포함하는 동적 도달 판별 및 오버슈트 방어 알고리즘은 PX4-Autopilot의 완전 자율 비행 신뢰도를 결정짓는 핵심 제어 기술이다. 이를 통하여 기체는 목표 위치에 빠르고 정확하게 도달할 뿐만 아니라, 관성에 의한 흔들림과 제어 루프의 불안정성을 억제하며 부드러운 호버링 상태로 무결점 전환을 달성할 수 있다.
