46.5.2 대기 환경적 외란(Wind Disturbance)에 의한 제어 루프 정상 상태 오차(Steady-State Error) 대응 및 적분기 포화(Integrator Windup) 방지 기법

1. 개요 (Introduction)

비행 중인 무인기가 한 방향으로 지속적으로 부는 강한 바람(대기 환경적 외란, Wind Disturbance)에 노출될 경우, 단순히 비례 제어(Proportional Control)만으로는 기체가 목표 위치(Setpoint)에서 밀려나 일정 거리에서 머무는 정상 상태 오차(Steady-State Error) 가 발생한다. 이를 제거하기 위해 적분 제어(Integral Control)가 필수적이나, 구동기 한계를 넘어서는 오차가 장기간 누적될 경우 적분기 포화(Integrator Windup) 라는 치명적인 제어 불안정성을 초래한다. PX4-Autopilot은 이러한 기상 악조건 속에서도 안정적인 호버링과 임무 비행을 수행하기 위해 고도화된 Anti-Windup 기법을 다중 제어 루프(Multi-loop Control) 전반에 걸쳐 적용하고 있다.

2. 바람 외란과 정상 상태 오차 (Wind Disturbance and Steady-State Error)

바람 벡터 W가 기체에 일정하게 작용하면, 기체 역학(Dynamics)에 일정한 외력(Constant Disturbance)이 더해진다.
위치 및 속도 제어기(Velocity Controller)에서 P-게인만 존재할 경우, 오차(Error) e(t)에 비례하는 제어 입력 출력이 바람이 밀어내는 힘과 완전히 균형(평형 상태)을 이루는 지점에서 오차가 더 이상 줄어들지 않고 고착화된다.
정상 상태 오차를 0으로 이끌기 위해서는 시간에 따른 오차의 누적합인 적분항(I-term, \int e(t) dt)이 활성화되어, 바람에 맞서 기울어지는 추가적인 피치/롤 각도를 지속적으로 인가해야 한다.

3. 적분기 포화 현상(Integrator Windup)의 발생 기전

적분 제어기는 작고 지속적인 오차를 성공적으로 제거하지만, 태풍급 돌풍이나 장애물 충돌 등으로 기체가 물리적으로 목표 지점에 도달할 수 없는 상황(Actuator Saturation)에 직면하면 문제가 발생한다.

  1. 기체가 낼 수 있는 최대 추력 및 기울기 한계(Saturation Limit)에 도달했음에도 목표 위치 오차가 좁혀지지 않는다.
  2. 제어기의 I-term은 이 오차를 상쇄하기 위해 수학적으로 무한히 값을 일방향으로 계속 누적(Windup)시킨다.
  3. 이후 바람이 갑자기 멈추거나 기체의 제어 권한이 회복되는 순간, 과도하게 쌓여버린 I-term 방출로 인해 기체가 반대 방향으로 급격히 튕겨 나가는 치명적인 오버슈트(Overshoot) 및 진동이 발생한다.

4. PX4의 적분기 포화 방지(Anti-Windup) 설계 전략

PX4는 이러한 치명적 현상을 방지하기 위해 위치/속도 제어기(mc_pos_control)와 자세/각속도 제어기(mc_att_control) 계층마다 엄격한 Clamping(클램핑) 기반의 Anti-Windup 로직을 설계하였다.

4.1 조건부 클램핑 (Conditional Clamping)

가장 핵심적인 기법으로, 제어기의 총 출력(Total Output)이 포화(Saturation) 한계에 도달했고, 현재 계산된 오차(e(t))의 부호가 적분기를 더 크게 만드는 (즉 포화를 가중시키는) 방향이라면 적분기의 스윕(Integration) 연산을 즉각 일시 정지(Freeze)한다. 반대로 오차가 줄어들어 출력을 한계치 안쪽으로 돌려놓는 방향이라면 적분을 지속하여 빠르게 정상화시킨다.

4.2 제어 모드 전환 시의 초기화 (Reset on Mode Switch)

조종자가 수동 모드(Manual, Stabilized)에서 자동 모드(Position, Mission)로 전환하는 순간, 이전 비행 상태에서 축적된 무의미한 I-term은 새로운 위치 제어에 악영향을 미친다. PX4는 상태 머신(Commander)에서 제어 모드가 변경될 때 모든 하위 PID 제어기의 적분항을 0으로 초기화(Reset)하거나, 끊김 없는 부드러운 전환(Bumpless Transfer)을 위해 현재 상태에 맞춰 I-term을 재조정한다.

4.3 피드포워드 계인(Feedforward)을 통한 I-term 부담 경감

EKF2가 추정한 수평 풍속 데이터(Wind Estimator)를 속도 제어기의 피드포워드(Feedforward) 텀에 직접 더해주어 바람을 보상해버리면, I-term이 반응하기도 전에 제어기출력이 조절된다. 이는 적분값이 커질 일 자체를 사전에 차단하는 매우 강력한 구조적 처방이다.

5. Ardupilot 대비 Anti-Windup 로직 차이점

  • PX4의 클램핑 기반 억제: PX4는 제어기의 출력 한계를 수학적으로 추적(Tracking)하여 조건부로 적분을 정지(Clamping)시키는 현대 제어 공학적 방법론에 철저히 입각해 있다. 소스 코드(PID.cpp) 상에서도 부동소수점 연산 최적화와 함께 상하한 범위 초과 여부를 분기문으로 직관적으로 검사한다.
  • Ardupilot의 Leaky I-term 및 휴리스틱 제한: Ardupilot은 역사적인 튜닝 과정에서 고안된 I-term Max 값(예: IMAX 파라미터) 하드 제한 방식을 오랫동안 채택해 왔으며, 스로틀이나 제어 막대(Stick) 입력이 전혀 없는 특정 이완 상태에서는 I-term이 서서히 0으로 줄어드는 누수 적분기(Leaky Integrator) 로직을 부분적으로 혼용하여 실무적인 강건성을 확보하는 철학을 가진다.

결론적으로 PX4의 정상 상태 오차 대응과 Anti-Windup 메커니즘은 단순한 PID 제어를 넘어서, 상태 추정기(EKF2)의 바람 정보 융합 및 제어 한계 상태의 명시적 조건을 감시하는 상호 연동형 복합 제어기로서 악천후 속 무인기의 신뢰성을 크게 향상시킨다.
