46.1.1 자율 비행 상태 기계(State Machine) 내 단일 목적지 이동의 공학적 위상 및 동적 임무 할당(Dynamic Task Allocation)

군집 비행, 재난 정찰, 동적 표적 추적과 같은 고도화된 무인 항공기(UAV) 운용 환경에서는 이전에 계획된 정적(Static) 임무를 수행하는 도중에도 예측 불가능한 이벤트에 즉각적으로 대응해야 한다. 본 절에서는 이러한 상황에서 요구되는 단일 목적지 이동(Goto/Reposition) 기능이 PX4-Autopilot(v1.14 기준)의 자율 비행 상태 기계(State Machine) 내에서 어떠한 공학적 위상을 가지며, 동적 임무 할당(Dynamic Task Allocation) 관점에서 어떻게 처리되는지를 논구한다.

1. 자율 비행 상태 기계 내 Reposition의 구조적 위상

PX4 시스템의 상태 기계는 커맨더(Commander) 모듈에 의해 최고 수준(High-level)의 모드 전이(Mode Transition)가 통제되며, 자동 비행에 관련된 세부 궤적 생성 및 논리는 내비게이터(Navigator) 인스턴스의 하위 상태로 위임된다.

명령된 자동 비행(Auto Flight) 모드는 내부적으로 여러 가지 세부 상태(Sub-state)를 가지는데, 여기서 Reposition 임무는 일반적인 Mission, RTL(Return To Launch), Loiter 등과 동등한 위상을 지니는 최상위 내비게이션 노드로 취급된다.

stateDiagram-v2
    [*] --> Auto_Flight_Mode
    
    state Auto_Flight_Mode {
        direction TB
        Mission --> Loiter : 임무 일시정지 (Pause)
        Loiter --> Mission : 임무 재개 (Resume)
        Mission --> Reposition : MAV_CMD_DO_REPOSITION 수신
        Reposition --> Loiter : 목적지 도달 시 (Reached)
        Loiter --> RTL : 복귀 명령 수신
    }
    
    Auto_Flight_Mode --> Manual_Mode : 조종기 개입 (RC Override)
    Manual_Mode --> [*]

위의 상태 다이어그램에서 알 수 있듯이, 단일 목적지 이동 명령은 기체가 어떠한 자동 비행 상태에 있더라도 즉각적인 상태 전이(State Transition)를 유발하는 강력한 인터럽트 구조를 띠고 있다. Reposition이 완료되어 설정된 수용 반경(Acceptance Radius)에 도달하면, 기체는 임무 모드로 복귀하지 않고 현재 위치에서 대기(Loiter / Hold) 상태로 전환하여 다음 명령을 기다리게 된다.

2. 동적 임무 할당(Dynamic Task Allocation)의 제어 논리

로봇 공학에서 동적 임무 할당은 로봇이 현재 수행 중인 작업을 중단하고, 비용 함수(Cost Function)나 우선순위(Priority)에 따라 새롭게 발생한 작업을 선언점령(Preemption)하여 수행하는 일련의 과정을 의미한다.

PX4 펌웨어 관점에서, 지상 관제 시스템(GCS; QGroundControl)이나 컴패니언 컴퓨터가 MAV_CMD_DO_REPOSITION 메시지를 발행하는 순간, 커맨더 시스템 내의 동적 할당 스케줄러가 다음과 같은 메커니즘을 구동한다.

  1. 컨텍스트 스위칭(Context Switching): 현재 수행 중이던 GCS 기반의 자율 임무(예: Waypoint 4번으로 이동 중)나 대기(Loiter) 궤적 제어 루프를 중지한다.
  2. 현재 상태 보존(State Preservation): 기체가 Mission 중이었다면, 현재 진행 중이던 임무 시퀀스 번호(Sequence Number)를 메모리에 보존하여, 향후 임무 재개(Mission Resume) 명령이 떨어졌을 때 이탈한 웨이포인트부터 다시 진행할 수 있도록 논리적 연결 고리를 유지한다.
  3. 목표 재정의 및 궤적 트리플릿(Triplet) 병합: Reposition 명령 파라미터(위도, 경도, 고도, 헤딩 및 속도)를 파싱하여 position_setpoint_triplet uORB 토픽의 “현재(Current)” 설정값으로 강제 주입(Injection)한다. 이때, 이전(Previous) 설정값은 명령을 수신한 시점의 기체 현재 위치로 자동 지정됨으로써, 궤적 생성기(Trajectory Generator)가 불연속적인 위치 점프 없이 부드러운 위치 제어를 수행할 수 있도록 한다.

3. ArduPilot과의 생태학적 설계 철학 비교

단일 목적지 이동의 상태 기계 통제 방식에서 ArduPilot과 PX4는 뚜렷한 설계 철학의 차이를 공유한다.

ArduPilot은 상태 분할이 수평적 모드 변경에 집중되어 있다. 임무 비행 중 동적으로 특정 위치로 기체를 이동시키려면 시스템을 AUTO 모드에서 GUIDED 모드로 전환한 후, MISSION_ITEM 기반의 Guided 궤적 표적 전송 절차를 거치는 것이 일반적이다. 만일 이동 후 다시 원래 임무로 복귀하기 원한다면 재차 AUTO 모드로 모드를 전환해야 하므로, 복수 회의 MAVLink 트랜잭션 수립이 불가피하다.

반면, PX4는 Auto 모드군(Mode Group) 내에서 Mission, Loiter, RTL, Reposition을 모두 수용하는 계층형 상태 머신(Hierarchical State Machine) 구조를 취한다. 따라서 PX4 펌웨어에서는 비행 모드 변경 인스트럭션 없이 단 하나의 커맨드(Command) 트랜잭션 수신만으로 내비게이터 모듈 내부에서 상태 천이를 확정짓고 즉각적인 동적 할당을 실행한다. 이 구조는 MAVLink 통신 대역폭을 절약하고 지연 시간(Latency)을 최소화한다는 측면에서 제어공학적 이점을 갖는다.

4. ROS 2 환경을 활용한 선제적 임무 할당 예시

현대의 고도화된 관제 시스템에서는 장애물 회피나 비전 인식 기능이 주로 ROS 2 생태계를 기반으로 동작한다. 이 때 uXRCE-DDS 미들웨어를 통해 실시간으로 발행되는 센서 퓨전 데이터를 토대로 특정 객체를 발견할 경우, 선행하던 임무를 우회하고 해당 객체로 기체를 이동시키는 제어 로직을 구현할 수 있다.

아래는 C++ 기반의 ROS 2 노드에서 동적 이벤트를 감지하고 즉각적인 Reposition 상태로 전환하는 내부 로직 흐름의 의사 코드(Pseudo-code)이다.

void on_target_detected(const vision_msgs::msg::Detection2D::SharedPtr msg) {
    if (msg->results[0].score > CONFIDENCE_THRESHOLD) {
        // 객체의 전역 좌표(위/경도 변환된) 획득
        double target_lat = transform_to_lat(msg->bbox);
        double target_lon = transform_to_lon(msg->bbox);
        
        // 동적 임무 할당을 위한 VehicleCommand 생성
        px4_msgs::msg::VehicleCommand cmd{};
        cmd.timestamp = this->get_clock()->now().nanoseconds() / 1000;
        cmd.command = px4_msgs::msg::VehicleCommand::VEHICLE_CMD_DO_REPOSITION;
        cmd.param5 = target_lat;
        cmd.param6 = target_lon;
        cmd.param7 = CURRENT_CRUISE_ALTITUDE; // 고도 유지
        
        // 현재 상태 인터럽트 파이프라인 수행
        RCLCPP_INFO(this->get_logger(), "동적 임무 할당: 표적위치로 Reposition 실행");
        command_publisher_->publish(cmd);
    }
}

이 코드는 기체의 기존 상태 머신 스케줄을 무효화(Override)시키고, 신호 처리 지연 없이 가장 높은 우선순위로 위치 제어 레이어에 개입하는 동적 임무 할당 메커니즘을 명시적으로 보여준다. 이러한 선제적 반응 구조는 자율 에이전트 다기종 복합 임무 수행체계(Multi-agent Complex Mission Framework) 제어에 있어 필수불가결한 설계라 할 수 있다.
