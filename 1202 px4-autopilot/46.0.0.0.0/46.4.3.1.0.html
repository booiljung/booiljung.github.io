46.4.3.1. 쿼터니언(Quaternion) 대수 기반 목표 기수각(Target Heading)과 현재 기수각 간의 3차원 특수 직교군(SO(3)) 회전 오차(Rotation Error) 최소화 궤적 검증

무인항공기(UAV)의 3차원 자세(Attitude)를 컴퓨터 시스템 내에서 표현하고 제어할 때, 오일러 각(Euler Angle)은 인간에게 직관적이라는 장점이 있으나 짐벌 락(Gimbal Lock)과 같은 수학적 특이점(Singularity) 문제를 근본적으로 안고 있다. 이러한 이유로 전 세계 표준 비행 제어 소프트웨어인 PX4-Autopilot(버전 1.14 기준)은 내부적인 모든 3차원 회전 연산과 추정(Estimation) 과정에 쿼터니언(Quaternion) 대수를 도입하여 3차원 특수 직교군(Special Orthogonal Group, SO(3))의 회전 오차(Rotation Error)를 도출하고 최소화한다. 본 절에서는 쿼터니언 연산을 이용해 드론의 기수각(Heading) 오차를 구하고, 이를 최소화하는 궤적 추종의 제어공학적 메커니즘을 상세히 검증한다.

1. 3차원 회전의 수학적 표현: 쿼터니언(Quaternion) 대수

쿼터니언은 해밀턴(William Rowan Hamilton)이 고안한 4차원 복소수 확장 체계로, 스칼라 실수부 q_0 와 3차원 벡터 허수부 (q_1, q_2, q_3) 로 구성된다.
\mathbf{q} = q_0 + q_1 \mathbf{i} + q_2 \mathbf{j} + q_3 \mathbf{k}
또는 스칼라와 벡터의 결합 형태인 \mathbf{q} = [q_0, \vec{q}]^T 로 표기할 수 있다.

특정 축 \vec{u} (단위 벡터)를 중심으로 \theta 만큼 회전하는 동작을 표현하는 단위 쿼터니언(Unit Quaternion)은 다음과 같이 정의된다.
\mathbf{q}(\theta, \vec{u}) = \left[ \cos\left(\frac{\theta}{2}\right), \vec{u} \sin\left(\frac{\theta}{2}\right) \right]^T

단위 쿼터니언의 크기(Norm)는 항상 1이며(||\mathbf{q}|| = 1), 이는 3차원 회전군 SO(3)의 공간을 4차원 단위 구면(Unit Sphere) S^3 로 매핑(Mapping)하는 완벽한 수단이 된다. PX4의 자세 제어기(Attitude Controller) 소스코드(mc_att_control 모듈 등)에서 기체의 자세를 표현하는 vehicle_attitude 토픽의 데이터 구조체는 모두 이 4원수 배열 형태로 송수신된다.

2. 쿼터니언 기반 회전 오차(Rotation Error) 추출 기법

드론의 현재 자세를 \mathbf{q}_{current}, 단일 목적지(Goto) 이동을 위해 기수가 바라보아야 할 목표 자세(주로 요 각도가 정렬된 자세)를 \mathbf{q}_{target} 이라 하자. 두 자세 간의 각도 차이, 즉 회전 오차 쿼터니언 \mathbf{q}_{error} 를 구하기 위해서는 일반 대수학의 뺄셈이 아닌 쿼터니언 곱셈(Quaternion Multiplication, \otimes)과 켤레(Conjugate, \mathbf{q}^*) 연산이 수행되어야 한다.

2.1 회전 오차 연산식

현재 자세에서 목표 자세로 가기 위해 추가로 요구되는 회전인 \mathbf{q}_{error} 는 다음의 방정식으로 유도된다.
\mathbf{q}_{target} = \mathbf{q}_{error} \otimes \mathbf{q}_{current}
양변의 우측에 \mathbf{q}_{current} 의 역원(Inverse)이자 켤레인 \mathbf{q}_{current}^* 를 곱해주면, 회전 오차 쿼터니언을 손쉽게 도출할 수 있다.
\mathbf{q}_{error} = \mathbf{q}_{target} \otimes \mathbf{q}_{current}^*

이 과정을 거치면 \mathbf{q}_{error} = [q_{e0}, q_{e1}, q_{e2}, q_{e3}]^T 가 도출된다. 여기서 허수부 벡터 성분 \vec{q}_e = [q_{e1}, q_{e2}, q_{e3}]^T 가 바로 3차원 공간상의 회전 오차 축구성을 나타내며, 유클리드 스칼라 곱칙을 적용하여 요축(Yaw/Heading) 좌표 오차 성분인 \Delta \psi 의 크기를 수학적 특이점 없이 직접 추출해 낼 수 있다.

3. 제어공학적 회전 오차 최소화 원리 및 궤적 검증

추출된 회전 오차 \mathbf{q}_{error} 는 기체가 아무리 복잡하게 뒤틀려 있더라도, 어떤 축을 중심으로 얼마만큼 회전해야 목표 자세에 완벽하게 일치하는지 나타내는 최적의 최단 회전 경로(Shortest Rotation Path)를 내포하고 있다.

3.1 비선형 회전 오차 제어 법칙(Control Law)

PX4 펌웨어의 외곽 비례(P) 각도 제어기(Angle Controller) 루프는 이 쿼터니언 허수부 \vec{q}_e 를 입력 피드백으로 삼아 다음과 같은 시스템 비례 제어 출력법칙 식을 구성한다.
\vec{\omega}_{cmd} = \mathbf{K}_p \, \text{sgn}(q_{e0}) \, \vec{q}_e
여기서 \vec{\omega}_{cmd} 는 3차원 각속도(Angular Velocity) 명령 벡터이며, \mathbf{K}_p 는 QGroundControl을 통해 튜닝하는 자세 비례 이득(Attitude P-Gain) 대각행렬이다. 스칼라부 q_{e0} 의 부호 함수 \text{sgn}(q_{e0}) 를 곱하는 이유는 쿼터니언 SO(3) 매핑의 중복성(Dual Covering, 즉 \mathbf{q} 와 -\mathbf{q} 가 동일한 물리적 자세를 의미함)으로 인해 드론이 불필요하게 360도 역회전하며 먼 길을 돌아가는 현상을 방지하기 위한 제어공학적 최단 경로 안정화 기법이다.

예시(Example): 드론이 시속 15m/s로 남쪽(180^\circ)을 향해 비행하다가 급격히 동쪽(90^\circ)의 새로운 Waypoint로 단일 목적지 전환(Goto) 명령을 갱신받았다고 가정하라.
만일 오일러 각을 사용한다면 피치(Pitch) 각이 크게 꺾여 있는 동적 상태에서 요(Yaw) 각도를 180^\circ 에서 90^\circ 로 단순 감산하여 제어하려 할 때 두 축 간의 간섭(Cross-coupling)으로 인해 기체가 순간적으로 크게 흔들리거나 제어 불능에 빠질 수 있다. 반면 단위 쿼터니언 곱셈 방식을 적용하면 기체의 기울어짐 속에서도 현재의 3차원 공간 속 최단 회전축과 각도를 즉각 찾아내어, \mathbf{K}_p 이득을 통해 흔들림 없이 부드럽고 우아한 회복 궤적(Recovery Trajectory)을 생성해 낸다.

4. Ardupilot과 PX4의 쿼터니언 연산 적용 방식 비교

Ardupilot 아키텍처와 PX4-Autopilot 모두 짐벌 락(Gimbal Lock) 방지를 목적으로 코어 자세 추정 시스템과 제어 루프 내부의 상태 공간 변수(State Space Variables)로 쿼터니언을 채택하고 있다는 점은 동일하다.

단, PX4는 uORB 메시지 스택을 타고 흐르는 최하위 토픽(vehicle_attitude, vehicle_attitude_setpoint)부터 상위의 각 통합 제어 모듈까지 쿼터니언 체계(matrix::Quatf)를 일원화하여 광범위하게 사용한다. 반면 Ardupilot은 특정 제어 레이어나 로깅(DataFlash) 시스템, 사용자 편의를 위한 텔레메트리 레이어 등지에서 오일러 각(Euler Angle, Roll, Pitch, Yaw) 기반의 변환 산출물을 더욱 적극적으로 혼용하는 설계적 특징이 있다. PX4 방식은 내부 변환 리소스 소모를 줄이고 SO(3) 공간 본연의 기하학적 연속성(Geometric Continuity)을 극대화한다는 점에서 순수 현대 제어공학적 아키텍처의 정수라 할 수 있다.
