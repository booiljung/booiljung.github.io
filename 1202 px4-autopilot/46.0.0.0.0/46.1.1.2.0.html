46.1.1.2 다중 웨이포인트(Waypoint) 기반 전역 경로 계획(Global Path Planning)과 지역 재배치(Local Repositioning)의 상태 전이(State Transition) 차이점

무인 항공기(UAV)의 자동 비행을 설계함에 있어 목표 지점을 지시하는 방법은 다수의 웨이포인트(Waypoint)로 구성된 전역 경로 계획(Global Path Planning, 임무 모드)과 단일 목적지로 즉각 이동하는 지역 재배치(Local Repositioning, Reposition 모드)로 양분된다. 본 절에서는 두 제어 방식이 PX4-Autopilot(v1.14 기준)의 상태 기계(State Machine) 내에서 어떠한 상태 전이(State Transition) 차이를 유발하며, 시스템 자원 및 제어 공학 측면에서 어떤 파급을 가지는지 명확히 규명한다.

1. 데이터 영속성(Data Persistence) 및 버스 처리 구조

근본적인 차이는 비행경로 데이터의 영속성과 처리 공간에 있다.

  • 다중 웨이포인트 (임무 지향, Mission): MAVLink의 마이크로 트랜잭션 프로토콜(Micro-transaction protocol)을 통해 QGroundControl(QGC) 등으로부터 전송되며, PX4 펌웨어 내의 dataman (Data Manager) 데몬을 통해 SD 카드나 내부 비휘발성 저장소에 기록된다. 전역 경로로 인식되므로 기체를 재부팅하더라도 임무 정보가 유지되는 영속성이 있다.
  • 지역 재배치 (즉각 반응, Reposition): MAVLink 명령(MAV_CMD_DO_REPOSITION) 또는 ROS 2 VehicleCommand로 수신되는 즉시 휘발성 uORB 메모리 버스에만 등재된 후 내비게이터(Navigator)에 의해 1회 소비(Consume)된다.

2. 상태 전이 모델(State Transition Model)의 비교

제어 아키텍처 관점에서 두 모드의 상태 전이는 극명한 차이를 갖는다. 이 차이는 기체가 목적지에 다다랐을 때(Reached)의 다음 행동 양식을 결정한다.

stateDiagram-v2
    state "Global Path Planning (Mission)" as Mission_Mode {
        direction LR
        Seq_N --> Seq_N_plus_1 : N번 WP 수용 반경 통과
        Seq_N_plus_1 --> Seq_N_plus_2 : N+1번 WP 수용 반경 통과
    }

    state "Local Reposition (Goto)" as Reposition_Mode {
        direction LR
        Idle --> Moving : DO_REPOSITION 수신
        Moving --> Auto_Loiter : 목적지 도달 및 정지
    }

점진적(Incremental) 상태 전이를 거치는 임무 모드와 달리, 지역 재배치는 인터럽트 주도형(Interrupt-driven) 1회성 상태 천이 구조를 따른다.

  • Mission 전이: N번째 웨이포인트의 수용 반경(NAV_ACC_RAD)에 도달하면, 기체는 정지하지 않고 내부 시퀀스 카운터(Sequence Counter)를 N+1로 증가시킨 후 즉각 다음 궤적을 갱신한다.
  • Reposition 전이: 단일 목표에 도착함을 판정하는 순간(속도가 0에 수렴할 때), 상태 기계는 Reposition 모드를 논리적으로 소멸시키고 기체를 제자리 비행 상태인 Auto Loiter (또는 Hold) 모드로 자동 전이시킨다(Auto-transition).

3. 내비게이터(Navigator)의 궤적 트리플릿(Triplet) 운용 로직 차이

PX4 내비게이터 내부에서 궤적 쌍을 생성하는 position_setpoint_triplet 메시지는 [Previous, Current, Next] 노드의 정보를 지닌다. 이 트리플릿의 구성 방식이 두 비행 동역학의 턴(Turn) 거동을 결정짓는다.

  1. 다중 웨이포인트 비행: Next 셋포인트가 명확하게 존재한다. 궤적 생성기(Trajectory Generator)는 기체가 Current 웨이포인트를 단순 통과(Pass-through)할 것임을 사전에 알 수 있기 때문에, 코너링(Cornering) 구간에서 매끄러운 곡선 궤적 베지어 곡선(Bézier Curve)이나 엘아르 큐빅 궤적 연산을 수행하여 에너지를 절약하고 속도를 유지한다.
  2. 단일 목적지 재배치 비행: Next 셋포인트가 비어있거나 대기(Loiter)로 고정되어 있다. 따라서 궤적 생성기는 해당 Current 웨이포인트를 비행의 최종점(Dead-end)으로 인식하며, 감가속 모델(Jerk-limited model)에 기반해 거리가 가까워질수록 속도를 선형적/비선형적으로 0까지 줄여나가는 강제 제동(Braking) 동작을 수행한다.

4. ROS 2(uXRCE-DDS) 기반 애플리케이션 통합 아키텍처 조언

컴패니언 컴퓨터를 활용해 자율 에이전트 경로를 계획할 때는 두 모드의 제어 주체를 명확히 이해해야 한다.

만일 ROS 2의 글로벌 플래너(Global Planner) 알고리즘이 10개 이상의 꺾인 경로를 미리 연산해 내었다고 가정해 보자. 이를 PX4 시스템에서 구현할 때, MAVLink 통신을 에뮬레이트하여 MISSION_ITEM을 다중 등재시키는 것보다 오프보드(Offboard) 모드에서 TrajectorySetpoint를 스트리밍하는 것이 훨씬 합리적이다.

하지만 플래너가 단방향으로 장애물을 한 번만 피하면 되는 상황이거나(단기적 회피), 인간-군집 인터페이스(Swarm UI)에서 운용자가 터치스크린으로 기체 대열을 이동시키는 상황이라면, 앞서 설명한 지역 재배치(Reposition) 호출이 가장 안전하고 확실하다. 이는 dataman에 영구적인 I/O 부하를 주지 않으며, 도달 후 자동으로 안전한 로이터(Loiter) 상태로 진입하는 메커니즘을 펌웨어 수준에서 보장해주기 때문이다.

5. ArduPilot의 분기 처리 방식과의 비교

ArduPilot의 경우 다중 웨이포인트를 처리하는 AUTO 모드와 단일 좌표로 이동하는 기능(과거 GUIDED 모드 계통)이 상태 머신 상에서 거의 수평적으로 분리되어 발전해 왔다. 반면 PX4 펌웨어는 임무(Mission) 상태 중 언제라도 Reposition 모드로 부드러운 상태 우회(Bypass)가 가능하며, Reposition 동작 후에도 이전 상태 레지스터에 기록된 컨텍스트를 활용하여 Mission을 즉시 재개(Resume)할 수 있는 계층화된 재귀적 구조(Recursive Hierarchical Structure)를 보유하고 있어 돌발 임무 대처 능력이 더욱 유연하게 설계되어 있다고 볼 수 있다.
