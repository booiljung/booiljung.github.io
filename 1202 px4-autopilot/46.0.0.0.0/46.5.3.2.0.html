46.5.3.2 수동 조종기 입력(RC Override) 감지 시 폐루프 위치 제어(Closed-Loop Position Control) 해제 및 비례 제어권 전환(Proportional Control Handover), 목표치 상태 초기화(State Reset) 로직

PX4-Autopilot 기반의 무인항공기 시스템(UAS)은 자율 임무 수행 중 조종자의 명시적인 개입이 필요할 때를 대비하여 강건한 수동 조종기 입력 수용 메커니즘을 갖추고 있다. 본 절에서는 자동 비행 모드(Mission, Hold, Return 등) 운용 중 수동 조종기 입력(RC Override)이 감지되었을 때, 기존의 폐루프 위치 제어(Closed-Loop Position Control)가 해제되고 조종자의 스틱 입력에 비례하는 제어권 전환(Proportional Control Handover)이 이루어지는 과정, 그리고 이 과정에서 필수적으로 요구되는 제어기의 목표치 상태 초기화(State Reset) 로직에 대해 제어공학적 관점 및 소프트웨어 아키텍처 관점에서 심층적으로 서술한다. (해당 로직은 PX4-Autopilot 펌웨어 v1.14 및 QGroundControl v4.3.x 이상을 기준으로 설명한다.)

1. 수동 조종기 입력(RC Override) 감지 메커니즘

PX4는 비행 모드 관리자(Commander Module)와 개별 제어기 태스크 간의 데이터 통신을 위하여 마이크로 객체 요청 브로커(uORB) 시스템을 사용한다. 자동 비행 상태에서 조종기의 스틱 입력은 주기적으로 폴링(Polling)되며, ManualControlSetpoint uORB 메시지로 퍼블리싱된다. 스틱의 변위량이 사전 설정된 데드존(Deadzone) 및 임계값 파리미터(예: COM_RC_OVERRIDE)를 초과하게 될 경우, 커맨더 로직은 이를 조종자의 명시적 궤적 수정 의도 또는 긴급 개입 조치로 결정한다.

Ardupilot 아키텍처에서는 개입 판별을 위하여 루프 처리에 타이머를 이용하는 방식(RC_OVERRIDE_TIME)을 주로 채택하는 반면, PX4는 uORB 메시지 구독 메커니즘에 의존하는 이벤트 기반의 상태 전이(State Transition) 아키텍처를 취하여 태스크 스레드의 반응 시간을 비약적으로 단축시킨다.

2. 폐루프 위치 제어(Closed-Loop Position Control) 해제

기체가 자동 궤적을 추종하거나 공중 정지(Hovering)를 수행할 때 위치 제어기(Position Controller)는 내부적으로 폐루프 피드백 제어 시스템을 가동 중이다. 이때 조종자가 개입하게 되면 기계적 스틱의 연속적인 명령과 내부 궤적 생성기의 목표치가 스티어링 신호에서 상호 충돌(Conflict)을 일으킨다.

이를 방지하기 위하여 입력 개입이 확인된 즉시 PX4의 제어 관리기(Flight Task Manager)는 기존의 자동 내비게이션 태스크(예: FlightTaskAuto)를 중단하고, 조종기 비례 입력을 수용할 수 있는 수동 태스크(예: FlightTaskManualPosition)로 전환함으로써 기존의 강체 폐루프 제어 루트를 동적으로 우회(Bypass)시킨다.

3. 비례 제어권 전환(Proportional Control Handover)

폐루프 위치 제어가 해제된 직후, 조종기의 물리기구적 스틱 변위량은 기체의 속도 목표치(Velocity Setpoint) 혹은 조향 목표치(Attitude Setpoint)로 선형 변환(Linear Mapping)되는 비례 제어권 전환(Proportional Control Handover)을 거친다.
수동 개입 상황에서 기체의 동특성은 조종자의 인지 반응과 기체의 추종 성능 간 상호작용 속도로 좌우된다. 따라서 입력된 스틱 값은 PID 제어기 중에서도 비례 이득(Proportional Gain) 항목으로 환원되어 위치 변화율(속도) 통제 모델에 직접 인가되며, 이는 직관적인 “인간 루프 개입(Human-in-the-Loop Intervention)“을 가능케 한다. 조종자가 스틱을 중앙(Center)으로 다시 놓는 순간 기체는 즉각 새로운 위치 기준점(Set-point)을 형성하고 다시금 위치 고정(Hold) 모드를 활성화하는 매끄러운 제어권 반환(Bumpless Transfer)을 수행한다.

4. 제어기 목표치 상태 초기화(State Reset) 로직의 필수성

제어 시스템이 특정 모드에서 다른 모드로 돌연 전이할 때 가장 대표적인 제어 불안정 요소는 적분기 포화(Integrator Windup) 현상과 내부 상태 변수의 불일치이다.

만약 자동 비행 상태에서 환경적 외란(Wind Disturbance)에 저항하기 위해 PID 제어기의 적분 오차가 크게 축적된 상황이라고 가정하자. 이때 조종자가 개입하면서 위치 제어가 속도 제어로 전환될 때 기존의 적분 오차(Error)를 즉시 방전 시키지 않거나 초기화(Reset)하지 않으면, 제어 출력단에 거대한 잔류 보상 신호가 인가되어 기체가 예기치 못한 급기동(Sudden Jerk)을 일으킨다.
수학적으로 표현하면 제어 입력 u(t)의 부분 중 적분 텀인 K_i \int_{0}^{t} e(\tau) d\tau이 모드 전환 직전의 \Delta e에 의해 이미 포화된 상태이므로 전이와 동시에 시스템이 튀어 오르는(Spike) 계기가 된다.

PX4에서는 이러한 현상을 막기 위해 제어기가 비례 모드로 전환되거나 스틱이 다시 중앙으로 되돌아오는 시점에 reset() 라이브러리 함수가 호출된다. 해당 로직은 제어기 내부의 적분 텀을 0으로 갱신하며, 새로이 입력된 현재의 위치와 속도 측정값을 필터링된 상태 공간식의 새로운 초기 조건(Initial Condition)으로 강제 정렬(State Re-initialization)시켜 안정적인 트랜지션을 모장한다.

6. 결언

수동 조종기 입력 감지를 통한 비례 제어권 전환 아키텍처는 무인항공기 관제에서 가장 본질적인 안전 조치 중 하나이다. PX4-Autopilot은 이 과정에서 uORB 기반의 명확한 상태 전이와, 적분기 포화 현상을 억제하기 위한 수학적 목표치 내부 상태 초기화(State Reset) 로직을 결합하여 조종자와 기체 간의 안전하고 예측 가능한 상호작용을 보장한다.
