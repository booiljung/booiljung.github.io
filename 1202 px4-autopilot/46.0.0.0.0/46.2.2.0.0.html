46.2.2. 동적 타겟 할당 메시지(SET_POSITION_TARGET_GLOBAL_INT, SET_POSITION_TARGET_LOCAL_NED)의 통신 대역폭 최적화 및 제어 계층(Control Tier) 우선순위 할당

본 절에서는 무인항공기가 외부 컴퓨터 혹은 지상 관제 시스템(GCS)으로부터 고빈도(High Frequency) 타겟 셋포인트(Setpoint)를 지속적으로 입력받을 때 사용되는 동적 할당 메시지인 SET_POSITION_TARGET_GLOBAL_INT 및 SET_POSITION_TARGET_LOCAL_NED의 통신 대역폭 제어 최적화 기법과, 이 명령이 PX4 내부 제어 계층(Control Tier) 우선순위를 어떻게 점유하는지에 대한 소프트웨어 공학적 할당 메커니즘을 상세히 다룬다.

1. 동적 타겟 할당 메시지와 비트 마스킹(Bit Masking) 기반 대역폭 최적화

일회성 명령인 MAV_CMD_DO_REPOSITION과 달리, SET_POSITION_TARGET_* 형태의 MAVLink 메시지는 장애물 회피(Obstacle Avoidance) 시스템, 타겟 추적 알고리즘, 자율 주행 컴퓨터가 기체의 Offboard 모드 비행을 실시간으로 통제하기 위해 지속적인 스트림으로 방출(Streaming)된다. 이러한 통신은 무선 데이터 링크(Telemetry)의 가용 대역폭(Bandwidth)을 상당 수준 점유할 가능성이 있다.

PX4-Autopilot은 이 문제를 해결하기 위해 타입 마스크(type_mask) 역논리(Inverse Logic) 방식을 채택하였다. 이 마스크는 16비트로 구성되며, 제어에 사용하지 않을 센서/물리량 데이터 비트를 ’1’로 설정하여 데이터를 무시(Ignore)하도록 지시한다.

  • type_mask를 활용하면 X, Y, Z 위치를 고정하고 속도 제어만 수행하거나, 속도를 제한하고 위치 셋포인트만 업데이트하는 등 다중 혼합 제어(Mixed Control)를 구현할 수 있다.
  • 통상적으로 비활성화된 항목의 페이로드 Float 값에는 NaN(Not a Number) 혹은 0을 삽입하더라도 수신 모듈에서 이를 필터링하여 불필요한 uORB 직렬화/역직렬화 오버헤드와 연산 주기를 아낄 수 있다.
  • 이는 제한된 텔레메트리 대역폭 내에서 50Hz 이상의 고주파 타겟 스트리밍을 구현할 때 통신 간섭을 최소화하는 결정적 역할을 수행한다.

2. 참조 좌표계(Reference Frame)별 페이로드 특성

목적에 따라 글로벌 및 로컬 두 가지 좌표 기반 메시지가 분기되어 사용된다.

  1. SET_POSITION_TARGET_GLOBAL_INT 메시지:
  • GCS와의 통신 또는 WGS84 위도/경도 체계가 필요한 장거리 경로 추종 비행에서 사용된다.
  • 위도와 경도는 IEEE 754 부동소수점 오차 누적을 방지하기 위해 10^7 상수로 스케일링된 32비트 정수(Int32) 포맷으로 전송되어 글로벌 통신 정밀도를 극대화한다.
  1. SET_POSITION_TARGET_LOCAL_NED 메시지:
  • VIO (Visual Inertial Odometry) 센서 환경이나 ROS2 기반 컴패니언 컴퓨터가 오프보드(Offboard) 직교 좌표 제어를 수행할 때 주로 사용된다.
  • 위치 제어기가 로컬 평면 좌표(North-East-Down)를 그대로 수용할 수 있어 기구학적 구면 투영 연산 과정(Map Projection Latency)을 생략하고 제어기 계층에 제어 입력을 직접 주입(Direct Injection)하는 강점이 있다.

3. 오프보드 모드(Offboard Mode) 제어 계층의 우선순위 점유 아키텍처

PX4의 다중 제어권(Multi-Control Authority) 모드 아키텍처에서 SET_POSITION_TARGET 메시지는 주로 Offboard 모드 상태에서 궤적 추종을 지시하며, 시스템 내 다른 비행 자동화 모듈(예: Position, Mission, Auto)들 간에 제어 계층(Control Tier) 우선권 논리 블록을 점유하기 위해 엄격한 검증을 통과해야 한다.

  1. 지속성 검증(Heartbeat & Timeout Valid Check):
    기체가 Offboard 모드로 전환되기 위해서는, 전환 명령 수신 이전 최소 0.5초(500ms) 이상, 그리고 2Hz 이상의 균일한 빈도로 SET_POSITION_TARGET 메시지가 지속 입력되는 사전 스트리밍 검증(Pre-streaming Validation)을 마쳐야 한다. 전환 이후 통신 지연이나 유실로 인해 지정된 큐 타임아웃(Timeout)이 발생하면 PX4 Commander 모듈은 즉시 오프보드 우선순위를 박탈하고, 기체 안전을 도모하는 Position, Hold, 또는 RTL(Return to Launch) 형태의 안심 모드(Failsafe Mode)로 제어권을 비상 하강(Fallback)시킨다.
  2. 계층 트리를 통한 우선순위(Tier Priority) 탈취:
    Offboard Setpoint 토픽(trajectory_setpoint)이 활성화된 상태라면, 내부적으로 생성된 RC 스틱 기반의 manual_control_setpoint 데이터 흐름이나 Navigator 기반의 GCS position_setpoint보다 우선하여 하단 위치 및 자세 제어 루프의 최상위 멀티플렉서 경로를 독점(Preempt)한다.
  3. 이기종 펌웨어 기반 시스템 비교 (Ardupilot 대비 차이점):
    Ardupilot의 경우 Guided Mode 상태에서 GCS가 보내는 단일 점대점 이동 명령(Goto) 큐브 블록과 외부 컴퓨터의 지속적 셋포인트 스트리밍 명령 제어 파이프라인이 하나의 모드로 혼재되어 자유로운 접근 및 양방향 제어를 허가하는 반면, PX4는 Offboard 모드라는 명시적 외부 컴퓨터 제어 격리 상태(Explicit External Control Isolation State) 인터페이스를 지향한다. 이를 통해 스트리밍 시스템의 텔레메트리 대역폭 통신 무결성 검증 프로세스 실패 시 즉시 제어 우선순위를 회수해 기체를 보호하도록 State Machine 체계가 설계 상 확고히 분리되어 있다.

4. 제어 계층 및 마스킹 인터페이스 처리 흐름도 (Data Handling Flow)

다음은 GCS 및 외부 컴패니언 장치에서 송신한 동적 목적지 데이터가 어떻게 역논리 타입 마스크 및 타임아웃 계층을 거쳐 궁극적인 제어 통제권을 획득하는지를 시각화한 컴포넌트 내부 다이어그램이다.

flowchart TD
    Comp([컴패니언 장치 / GCS<br/>SET_POSITION_TARGET 송신]) --> MAVRx[mavlink_receiver 모듈]
    
    MAVRx --> Validate{타입 마스크 파서<br/>(Ignore 비트 판독)}
    Validate -- Ignore 활성화 (1) --> Skip[해당 축 셋포인트 갱신 스킵]
    Validate -- Apply 활성화 (0) --> Process[토픽 역직렬화 및 내부 버퍼 주입]
    
    Process --> Pub[orb_publish<br/>(trajectory_setpoint 토픽 발행)]
    
    Pub --> FSM{Commander 상태 머신<br/>Offboard 유지조건 타임아웃 검사}
    
    FSM -- 통과 (Timeout 미발생) --> OffboardCtrl[Offboard 제어 모듈이<br/>위치 제어 1순위 독점 점유]
    FSM -- 기각 (Timeout 발생) --> Fallback[Failsafe Trigger 실행:<br/>Position/Hold 모드 강제 전환]
    
    OffboardCtrl --> PosCtrl[(Position Controller 코어 계층)]
    Fallback --> PosCtrl

결론적으로, SET_POSITION_TARGET를 필두로 한 스트리밍 방식 궤적명령 처리 시스템은 통신 계층의 비트 오버헤드와 연산 랙 타임(Lag Time)을 제거해 주는 마스킹 레지스터(Masking Register) 기법의 영리함과, 0.5초 룰(Rule)에 입각하여 시스템 권한을 통제하는 능동적 제어 박탈(Active Preemption) 알고리즘이 유기적으로 조합된 수준 높은 안전 공학 설계(Safety Engineering Design) 철학을 내포하고 있다.
