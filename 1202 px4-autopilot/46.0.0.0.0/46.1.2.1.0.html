## 0.1 확장 칼만 필터(EKF2) 기반 전역 위치 추정(Global Position Estimation) 공분산 행렬(Covariance Matrix) 대각 성분(Diagonal Elements) 수렴성 검증

비행 제어 컴퓨터(Flight Controller) 상에서 자율 비행(Autonomous Flight) 모드 중 하나인 단일 목적지 이동(Goto) 제어를 수행하려면, 기체가 파악하고 있는 현재 시스템의 전역 위치(Global Position)가 높은 신뢰 수준을 확보해야 한다. PX4-Autopilot의 기본 상태 추정기인 EKF2(Extended Kalman Filter 2)는 추정 상태 변수(State Variables)의 확률론적 불확실성을 나타내기 위해 상태 공분산 행렬(State Covariance Matrix)을 실시간으로 관리한다. 이 중 대각 성분(Diagonal Elements)은 각 추정 변수들이 가지는 분산(Variance)을 의미하며, 전역 위치 추정의 건전성을 판단하는 핵심 지표로 작용한다.

0.1.1 공분산 행렬 대각 성분의 수리적 의미와 EPH/EPV 수렴 조건

EKF2가 갱신하는 공분산 행렬 P의 주대각선(Main Diagonal) 원소들은 각각의 편미분 벡터 성분에 대한 오차 분산(\sigma^2)을 수치적으로 나타낸다. 예를 들어, 전역 좌표계 기준 북쪽(North) 및 동쪽(East) 방향의 식별 오차 분산인 P_{N,N}, P_{E,E} 값들은 추정 수평 오차(Estimated Position Horizontal, EPH)를 도출하는 뿌리가 되며, 수직(Down) 방향 오차 분산 P_{D,D}는 추정 수직 오차(Estimated Position Vertical, EPV)를 산출하는 기준이 된다.

시스템이 위치 제어 모드(Position Mode) 및 Goto 명령 체계를 수락하기 위해서는 EPH와 EPV 수치, 즉 공분산의 대각 성분이 사전에 정의된 파라미터 임계값 이하로 “수렴(Convergence)“해야 한다.

  • EKF2_REQ_EPH: 수평 위치 추정이 공간 이동을 수행하기에 충분한 정밀도(통상 3.0 ~ 5.0m 내외)로 수렴했는지 검증하는 파라미터
  • EKF2_REQ_EPV: 수직 위치 추정 정밀도(통상 5.0 ~ 8.0m 내외) 판별 파라미터

초기 GPS/GNSS 수신 시 노이즈가 심하거나 기체에 심한 진동이 유입되는 상황에서는 예측 모델과 측정값의 편차가 커서 분산이 발산(Divergence)하거나 높은 상수 값에 머무르게 되며, 이는 전역 위치 추정 유효성(Global Position Validity) 획득 실패로 이어진다.

지표명EKF2 내부 파라미터 연결성공분산 대각 성분 수식적 기원QGC 경고 기준 임계값 예시
수평 상태 분산EKF2_REQ_EPH\sqrt{P_{N,N} + P_{E,E}}> 5.0 m
수직 상태 분산EKF2_REQ_EPV\sqrt{P_{D,D}}> 8.0 m

(표 내의 1차원 분산 합성은 서적 시스템상의 EPH 연산의 직관적 이해를 돕기 위한 간략화 명세이다.)

0.1.2 EKF2 노이즈 파라미터와 수렴성의 역학 관계

정상적인 위성 항법 시스템(GNSS) 관측 환경임에도 공분산 대각 성분이 수렴하지 않을 경우 측정 시스템의 노이즈 처리 파라미터 튜닝이 필수적이다.

  • EKF2_GPS_P_NOISE (위치 노이즈) 및 EKF2_GPS_V_NOISE (속도 노이즈): 센서 하드웨어 데이터시트(Datasheet)나 외부 환경 특성에 맞게 설정해야 한다. 만일 설정된 노이즈 수치가 실제 센서 노이즈보다 너무 작게 최적화(Optimistic Tuning) 되었다면, 필터는 실제 시스템의 부정확성을 무시한 채 공분산을 거짓으로 수렴시킬 위험이 있다. 이는 Goto 지령 수행 도중 비정상적인 위치 보정 움직임인 ‘글리치(Glitch)’ 현상을 야기할 우려가 존재한다. 반대로 노이즈 파라미터가 비대하게 산정(Pessimistic Tuning)되었다면 오차 분산의 수렴 속도가 현저히 저하되어, 파일럿이 요구 시간 내에 자율 비행 모드 권한을 이양받지 못하게 된다.

0.1.3 Ardupilot EKF3 대비 PX4 시스템의 위치 수렴 평가 철학

  • Ardupilot EKF3: 복수의 EKF 레인(Lane)을 코어 기반으로 동시 구동하고 각 코어 별 독립적인 공분산 행렬을 연산 추적한다. 각 코어에서 도출된 EPH/EPV 중 가장 최적의 상태 무결성을 보이는 레인으로 스위칭하여 Goto 제어 및 웨이포인트(Waypoint) 비행 시 시스템 파국을 방지하는 결함 보상 중심의 설계 사상을 띤다.
  • PX4 EKF2: PX4는 단일 코어 EKF 엔진 환경을 주력으로 구동하며, 공분산 수렴성(Covariance Convergence)과 자체 내부 테스트 비율(Test Ratio, 예: pos_test_ratio) 지표를 더욱 엄격하게 묶어서 단일 척도 판정을 수행한다. PX4의 커맨더(Commander) 모듈은 EKF2 필터가 물리적 수렴 상태에 다다라 ESTIMATOR_STATUS 플래그를 정상치로 응답하기 전까지는 MAVLink 측의 MAV_CMD_DO_REPOSITION 이동 명령이나 POSCTL 등 특정 비행 모드 진입 요청을 즉시 기각하는 강력한 인터로킹(Interlocking) 구조를 통해 보수적으로 안전성을 보장한다.

0.1.4 상태 수렴 모니터링 논리 및 정보 파이프라인 구조

QGroundControl에서의 시각 정보 전송 및 EKF2 내부 타이밍에 따른 공분산 수렴 모니터링은 다음의 논리 파이프라인으로 구현된다.

sequenceDiagram
    participant GNSS as GNSS Sensor
    participant EKF2 as PX4 EKF2 Core
    participant CMD as Commander (PX4)
    participant QGC as QGroundControl

    GNSS->>EKF2: Read Lat/Lon/Alt & Accuracy Match
    activate EKF2
    EKF2->>EKF2: Compute Covariance Matrix (P)
    EKF2->>EKF2: Extract Diagonal Elements (P_{N,N}, P_{E,E}, P_{D,D})
    EKF2->>EKF2: Calculate EPH and EPV
    
    alt EPH < EKF2_REQ_EPH & EPV < EKF2_REQ_EPV
        EKF2->>CMD: Global Position Valid (xy_valid=true)
        EKF2->>QGC: ESTIMATOR_STATUS (Flags: GPS OK)
    else EPH or EPV Exceeds Threshold
        EKF2->>CMD: Global Position Invalid
        EKF2->>QGC: ESTIMATOR_STATUS (Flags: GPS Error)
        QGC->>QGC: Display "Position Control Denied"
    end
    deactivate EKF2

상기에 서술된 평가 메커니즘을 통하여, 전역 위치 추정 오차 분산 벡터가 무인기 시스템 설계 단계에서 요구하는 궤적 추종 오차(Tracking Error) 허용 한계 내로 안전하게 진입하여야만, MAVLink 프로토콜을 통하여 관제 모듈(GCS) 또는 보조 컴퓨터 체계(Companion Computer, ROS2 연동기 등)에서 발령하는 Goto(목적지 지정 이동 제어) 명령 처리의 제반 선행 요건이 완성된다.
