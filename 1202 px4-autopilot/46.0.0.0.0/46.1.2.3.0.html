## 0.1 지역 접평면 좌표계(Local Tangent Plane Coordinate System) 원점(Origin) 초기화 및 기준점(Home Position) 참조 프레임 무결성(Integrity) 확인 메커니즘

단일 목적지 이동(Goto) 또는 상대적 위치 제어(Reposition) 명령은 무인기 시스템 내부 수학적 모델에서 평면 기하학 기반의 벡터 연산으로 처리된다. 이를 위해 지구의 곡률을 무시할 수 있는 좁은 영역의 비행 공간을 데카르트 좌표계(Cartesian Coordinate System)로 근사하는 **지역 접평면(Local Tangent Plane, LTP)**이 필수적이다. PX4-Autopilot은 이 LTP를 구성하기 위하여 NED (North-East-Down) 좌표 기준 프레임을 활용하며, 공간 이동 제어를 발동하기 전에 해당 좌표계의 원점(Origin) 초기화 및 기준점(Home Position) 무결성을 엄격하게 검증한다.

0.1.1 EKF2 지역 접평면(LTP) 원점의 공학적 정의

PX4의 기본 상태 추정기인 EKF2(Extended Kalman Filter 2)는 글로벌 센서(GNSS, GPS) 혹은 외부 비전 시스템(VIO/mocap)으로부터 최초의 유효한 절대 위치 데이터를 수신하여 융합(Fusion)을 시작하는 정확한 순간에 시스템 내부적인 **지역 위치 원점(Local Position Origin, EKF Origin)**을 생성한다.

  • 이 지점이 지역 좌표 [X, Y, Z] = [0, 0, 0]이 되며, MAVLink 상의 LOCAL_POSITION_NED 메시지는 이 원점을 기준으로 한 미터(m) 단위의 상대적 변위를 파싱하여 전송한다.
  • EKF2 원점은 비행 중 센서 이상이나 필터 리셋이 발생하지 않는 한 통상적으로 고정(Fixed)된다. PX4 펌웨어 버전 1.12.3 이후부터는 관제 시스템(QGroundControl)이 MAVLink 명령인 SET_GPS_GLOBAL_ORIGIN을 통해 이 원점 위치를 명시적으로 재설정하거나 덮어쓸 수 있는 API 인터페이스를 지원한다.

0.1.2 기준점(Home Position)의 차이 및 역할

종종 EKF2 원점과 혼동되는 개념이 **기준점(Home Position)**이다. 두 참조 프레임은 체계적으로 분리되어 동작한다.

  • 기준점(Home Position): 위도, 경도, 고도(Lat, Lon, Alt)로 정의되며, 조종사가 기체에 시동(Arming)을 거는 순간의 위치로 동적 할당되는 것이 기본 설정이다. 이 좌표 체계는 주로 임무 종료, 통신 두절(Failsafe) 시 자동 복귀(Return To Launch, RTL)를 위한 전역 지리 좌표(Global Geospatial Coordinate)로 활용된다. MAVLink의 HOME_POSITION 메시지를 통해 관제되며, MAV_CMD_DO_SET_HOME 명령으로 비행 중에도 갱신할 수 있다.
  • Goto 명령과의 연계: MAVLink를 통한 원격 Goto 명령(MAV_CMD_DO_REPOSITION 등)이 전역 좌표(위도, 경도)로 하달되더라도, PX4의 위치 제어 모듈(Position Controller Flight Task)은 현재 초기화된 단일 EKF2 지역 접평면 기준(LTP Reference) 프레임으로 타겟 좌표를 역변환(Reprojection)하여 기구학적 항법 유도를 수행한다.

0.1.3 참조 프레임 무결성(Integrity) 및 선행 조건 검증 메커니즘

공간 이동 제어 기능(Goto/Reposition)이 승인되기 위한 펌웨어(커맨더-Commander 모듈) 레벨의 검증은 다음과 같은 절차로 진행된다.

  1. uORB 메시지 타임스탬프 검증: 상태 추정기(EKF2)가 발행하는 vehicle_local_position 토픽 내부에 명시된 ref_timestamp 속성이 0보다 큰 유효한 값(시스템 부팅 이후 경과 마이크로초)을 가지고 있는지 판단하여, LTP 좌표계가 메모리에 실증적으로 확립되었는가를 확인한다.
  2. 투영 기준 무결성 검증: NED 좌표를 WGS84 위경도로 상호 투영(Projection)하기 위한 기준 위경도 정보인 ref_lat, ref_lon 변수가 정상적으로 등록되어 있는지 확인한다.
  3. Local Valid Flag 검증: 지역 변위 제어를 허가하기 전, 최종적으로 EKF2 내 공분산 및 속도 혁신 임계값이 수렴하여 xy_valid = true 및 z_valid = true 플래그 상태가 일치하는 순간 인터로크(Interlock)를 해제하고 Position Mode 제어 루프를 개통한다.

0.1.5 Ardupilot 플랫폼 대비 GCS 관제 차이

Ardupilot의 경우 EKF 참조 원점(EKF Origin)과 시스템 기준점(Home Position)의 연동성이 비교적 밀접하게 결합되어 있어, 초기 이륙 환경에서 GPS 스푸핑(Spoofing) 등의 문제로 기준 교란이 올 경우 체계 복구가 까다로울 수 있다.
반면 PX4는 vehicle_local_position uORB 아키텍처를 매개체로 하여 필터링 추정 지점(EKF Origin)과 지리적 귀환점(Home Position)을 명백히 디커플링(Decoupling)하였다. 따라서 GCS 어플리케이션(QGroundControl 등) 개발 시, 무인기 화면에 그려질 무인기의 상대 위치(Local Target Mode)는 MAVLink LOCAL_POSITION_NED를 활용하고, 단순 복귀 점은 HOME_POSITION 토픽을 활용하여 서로 분리된 계층에서 렌더링해야 함을 유의하여야 한다.
