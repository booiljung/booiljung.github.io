46.3.4.3. 불연속 제어 입력 방지를 위한 저역통과 필터(Low-pass Filter) 기반의 위치 및 속도 궤적 평활화(Trajectory Smoothing) 기법

단일 목적지 이동(Goto/Reposition)이나 조종기의 직접 개입 상황에서, 지상 관제 시스템(GCS) 또는 조종자가 하달하는 제어 입력(Setpoint)은 본질적으로 불연속적인 특성(예: 계단 함수 형태의 입력)을 지니는 경우가 많다. 이와 같은 불연속 제어 명령(Step Input)이 비례-적분-미분(PID) 제어 루프에 여과 없이 전달되면, 미분항(D-term)에서의 스파이크(Spike) 현상과 액추에이터의 순간적 포화(Saturation)가 발생하여 기체의 기계적 피로도 증가 및 자세 불안정을 초래하게 된다. PX4-Autopilot은 이를 억제하기 위해 제어 전위 단계에서 저역통과 필터(Low-pass Filter, LPF) 및 속도 제한기(Slew Rate Limiter)를 결합한 평활화(Trajectory Smoothing) 기법을 기본적으로 적용하고 있다 (기반 펌웨어 버전: PX4-Autopilot v1.14 기준).

1. 불연속 제어 입력이 야기하는 문제점

일반적으로 고수준의 요구 속도(Velocity Setpoint)가 한 순간에 0 m/s에서 5 m/s로 급변할 경우, 이상적인 가속도는 무한대에 가깝다.

  • 액추에이터(Actuator) 포화: 모터와 ESC가 물리적으로 추종할 수 없는 속도 변화를 요구받아 제어 대역폭(Control Bandwidth)을 초과한다.
  • 고주파 노이즈 증폭: 통신 지연시간(Latency)이나 위치 추정(EKF)의 노이즈가 섞인 셋포인트가 그대로 반영되어 진동(Oscillation)을 유발한다.

2. 저역통과 필터(Low-pass Filter)의 수학적 원리 및 PX4 구현

소프트웨어적으로 셋포인트 평활화에 가장 널리 사용되는 방식은 1차 무한 임펄스 응답(IIR: Infinite Impulse Response) 필터, 일명 알파 필터(Alpha Filter)이다. 이산 시간(Discrete Time) 환경에서의 기본 방정식은 다음과 같다.

y[k] = \alpha \cdot x[k] + (1 - \alpha) \cdot y[k-1]
v_{sp\_filtered} = \alpha \cdot v_{sp\_raw} + (1 - \alpha) \cdot v_{sp\_prev}

여기서:

  • v_{sp\_raw}: 새롭게 수신된 불연속적 목표 속도
  • v_{sp\_filtered}: 필터링되어 부드러워진 목표 속도
  • \alpha: 차단 주파수(Cut-off Frequency, f_c)와 샘플링 시간(\Delta t)에 의해 결정되는 필터 계수 (0 \le \alpha \le 1)

PX4 소스 코드의 Mathlib 계층 내 AlphaFilter 클래스를 기반으로, 기체의 위치 제어기(mc_pos_control)는 조이스틱 입력(FlightTaskManual)이나 GCS에서 온 Offboard 명령을 파라미터(예: MPC_JERK_MAX, MPC_ACC_HOR)에 맞추어 평활화한다.

2.1 Slew Rate Limiter와의 하이브리드 적용

단순한 LPF만 사용하면 목표 지점 도달 시 지연(Phase Lag)이 길어지는 단점이 존재한다. 이를 보완하기 위해 PX4는 속도의 미분값인 가속도의 최대치 제약(Acceleration limit) 또는 저크 제약(Jerk limit) 알고리즘을 함께 사용한다.

이러한 하이브리드 파이프라인을 거치게 되면, 목표 속도는 초기에 최대 설정 가속도(MPC_ACC_HOR_MAX 등) 이내에서 선형적으로 증가하고 코너 부분은 저역통과 필터에 의해 둥글게(Smooth) 처리되어, 완벽한 “S-Curve” 형태에 수렴하는 연속 시간 궤적이 완성된다.

3. ArduPilot 대비 평활화 철학(Smoothing Philosophy)의 차이

ArduPilot의 멀티로터 제어기(Copter) 역시 입력 평활화의 중요성을 깊이 인식하여 Input Shaping 기술을 적용하고 있다. ArduPilot에서는 ATC_INPUT_TC (Attitude Control Input Time Constant) 등의 파라미터를 통해 1차 저역통과 필터의 지연 시간을 직관적인 초(Seconds) 단위로 튜닝할 수 있도록 제공하며, 웨이포인트 네비게이션 시 WPNAV_ACCEL 규칙에 종속된 엄격한 가속도 리미터를 통과시킨다.

반면, PX4-Autopilot은 FlightTask 아키텍처 내부의 궤적 생성기(Trajectory Generator) 모듈이 평활화의 주도권을 갖는다. 수동 조작용인 FlightTaskManual에서는 스틱 파형에 LPF가 강력하게 개입되는 반면, 자율 비행인 FlightTaskAuto에서는 3차 에르미트 스플라인 알고리즘 자체가 이미 C^1 (속도) 계층의 연속성을 보장하므로 LPF의 역할이 보조적인 백업 수단(고주파 진동 억제)으로 축소된다. 이는 각 비행 상황(모드)별로 최적화된 수학 모델을 동적으로 적용하고자 하는 PX4 소프트웨어 구상 철학을 대변한다.
