44.7.5. 이륙 공간의 다각형 지오펜스(Polygonal Geofence) 상한 고도 제한 및 가상 경계 충돌 예측을 위한 3차원 광선 교차(Ray Intersection) 알고리즘 방어 로직

무인항공기(UAV)의 독립적이고 자율적인 비행 환경에서 안전을 보장하는 핵심 시스템 중 하나는 지오펜스(Geofence)이다. 특히 도심지 비행이나 정밀 측량이 요구되는 산업 현장에서는 단순한 원통형 경계(Cylindrical Boundary)를 넘어, 비행 불가능 구역을 정밀하게 회피하기 위한 다각형 지오펜스(Polygonal Geofence) 설정이 필수적이다. 본 절에서는 PX4-Autopilot(펌웨어 v1.14.0 기준)이 이륙 단계에서 다각형 지오펜스의 상한 고도 제한과 측면 가상 경계 충돌을 어떻게 사전에 예측하고 방어하는지, 그리고 그 근간이 되는 3차원 광선 교차(3D Ray Intersection) 알고리즘의 수학적 원리와 소프트웨어 아키텍처를 상세히 분석한다.

1. 다각형 지오펜스 시스템의 구조와 MAVLink 프로토콜

지오펜스는 사전 정의된 3차원 공간 내에 무인항공기가 이탈하지 못하도록 가상의 울타리를 설정하는 기능이다. 다각형 지오펜스는 여러 개의 위도(Latitude)와 경도(Longitude) 정점(Vertex)을 연결하여 2차원 평면의 다각형(Polygon)을 구성하고, 상한 고도(Max Altitude) 및 하한 고도(Min Altitude) 제한을 두어 결과적으로 3차원 각기둥(Prism) 형태의 비행 가능 영역(Inclusion zone) 또는 비행 금지 영역(Exclusion zone)을 형성한다.

QGroundControl(v4.3.0 이상)과 같은 GCS(지상 관제 시스템)는 사용자가 지도상에서 다각형을 그리면, 이를 MAVLink 프로토콜을 통해 비행 제어기(Flight Controller, FC)에 전달한다. 이때 MISSION_ITEM_INT 메시지 포맷을 활용하며, MAV_CMD_NAV_FENCE_POLYGON_VERTEX_INCLUSION 또는 MAV_CMD_NAV_FENCE_POLYGON_VERTEX_EXCLUSION 형태의 커맨드를 사용하여 정점 정보가 순차적으로 전송된다.

1.1 주요 설정 파라미터(Parameters)

PX4 내에서 다각형 지오펜스와 상한 고도를 제어하는 주요 파라미터는 다음과 같다.

  • GF_ACTION: 지오펜스 위반 시 수행할 방어 동작(RTL, 착륙, 비행 종료(Flight Termination) 등)을 정의한다.
  • GF_MAX_VER_DIST: 이륙 지점(Home Position)으로부터 최대 허용 상한 고도를 설정한다.
  • GF_MAX_HOR_DIST: 원통형 최대 반경 제한을 설정한다 (다각형 펜스와 복합적으로 작용 가능).
  • MIS_TAKEOFF_ALT: 이륙 시 도달해야 하는 목표 고도를 의미하며, 이 고도가 지오펜스의 상한을 넘지 않도록 검증 과정이 수행된다.

2. 가상 경계 충돌 예측을 위한 알고리즘 원리

이륙 명령이 인가되었을 때, PX4는 현재 기체의 위치에서 목표 이륙 고도까지의 궤적(Trajectory)을 하나의 선분(Line segment) 혹은 광선(Ray)으로 모델링한다. 기체가 이 선분을 따라 상승할 때 지오펜스 경계면을 돌파하는지(Intersection) 판단함으로써 선제적인 방어 로직을 가동한다.

2.1 포인트 포함 여부 판별 (Point-In-Polygon, PIP)

우선 기체의 현재 위치 및 이륙 목표 지점이 2차원 다각형 내부에 존재하는지 확인해야 한다. 시스템은 래 캐스트(Ray Casting) 알고리즘의 2차원 형태인 Winding Number 또는 Crossing Number(회전 수 또는 교차 수) 알고리즘을 사용한다.
어떤 점이 다각형 내부에 있는지 판단하기 위해 점으로부터 무한대의 반직선을 그어, 다각형의 변과 교차하는 횟수가 홀수이면 내부, 짝수이면 외부로 판별한다.

2.2 3차원 광선 교차(Ray Intersection) 알고리즘 모델링

보다 능동적인 충돌 방지를 위해서는 3차원 벡터 기반의 광선 교차 검사가 필요하다. 이륙 궤적은 다음과 같은 광선 방정식(Ray Equation)으로 표현될 수 있다.

\mathbf{P}(t) = \mathbf{O} + t\mathbf{D}

여기서:

  • \mathbf{O}: 광선의 시작점, 즉 이륙 직전의 기체 3차원 위치 좌표
  • \mathbf{D}: 이륙 방향의 단위 방향 벡터(Direction Vector) (수직 상승의 경우, 주로 +Z 방향, NED 좌표계 기준으로는 -Z 방향)
  • t: 스칼라 매개변수 (시간이나 거리에 비례, t \geq 0)

다각형 지오펜스의 측면 및 상단/하단 경계면(평면, Plane)은 다음과 같은 평면 방정식으로 표현된다.

\mathbf{N} \cdot (\mathbf{P} - \mathbf{P_0}) = 0

여기서:

  • \mathbf{N}: 평면의 법선 벡터(Normal Vector)
  • \mathbf{P_0}: 평면 위에 존재하는 임의의 한 점

광선이 평면과 교차하는 점을 찾기 위해 \mathbf{P}(t)를 평면 방정식에 대입하면 다음과 같이 t를 구할 수 있다.

t = \frac{\mathbf{N} \cdot (\mathbf{P_0} - \mathbf{O})}{\mathbf{N} \cdot \mathbf{D}}

해당 t 값이 0 \leq t \leq \vert\vert \text{목표 지점} - \mathbf{O} \vert\vert 범위에 속한다면, 기체의 이륙 궤적이 지오펜스의 평면 경계와 교차(충돌)함을 의미한다. (※ 테이블 내부 수식 표현을 위해 \vert 대신 \vert 사용)

전산 기하학(Computational Geometry)에서 널리 쓰이는 뮐러-트룸보어 광선-다각형 교차 알고리즘(Möller-Trumbore ray-triangle intersection algorithm, Fast, Minimum Storage Ray/Triangle Intersection)을 응용하면 행렬식(Determinant) 연산을 기반으로 매우 빠른 속도로 교차 여부를 판별할 수 있다.

3. PX4-Autopilot 내부 방어 및 격리 로직 (Source Code Level)

PX4의 소스 코드에서 지오펜스 위반 검증은 주로 Navigator 모듈과 Geofence 클래스 내부에서 이루어진다. 이륙 명령(Takeoff)이 수신되면 목표 위치 위도, 경도, 상한 고도를 기반으로 Geofence::check(const struct vehicle_global_position_s &global_position ...)와 같은 검증 함수가 호출된다.

  1. 사전 검사 (Pre-flight Validation):
    이륙 목표 고도(Target Altitude)가 지오펜스 다각형 영역의 내부에 존재하는지 1차로 확인한다. 목표 고도가 GF_MAX_VER_DIST를 초과하거나, 다각형의 경계를 벗어나면 즉각적으로 NACK(Negative Acknowledge)를 반환하고 이륙을 거부한다.
  2. 비행 중 실시간 모니터링 (Runtime Verification):
    멀티콥터의 위치 제어 모듈(mc_pos_control) 내부에서는 매 제어 주기마다 유효 목표 지점(Setpoint)이 지오펜스를 이탈했는지 감시한다.
    만약 환경 외란이나 제어기 한계로 인해 기체의 실시간 예측 궤적이 경계를 돌파하는 상황이 감지되면, Geofence 클래스는 방어 기동인 GF_ACTION (예: Loiter, RTL, Terminate) 이벤트 플래그를 발생시켜 State Machine을 강제 전이(Transition)시킨다.
  3. 목표 클램핑(Setpoint Clamping):
    이륙 중 측풍 등에 의해 밀려나는 경우, 목표 위치를 다각형 내부로 유지하기 위해 벡터 프로젝션(Vector Projection)을 사용하여 제어기의 목표 지점(Setpoints)을 지오펜스 경계면 내부로 제한(Clamping)시키는 방어 로직을 수행한다.

4. Ardupilot의 지오펜스 처리 체계와의 비교 분석

다각형 지오펜스 및 고도 제한 로직에 있어서 통칭 ’펜스 라이브러리(AC_Fence)’를 운영하는 Ardupilot 프레임워크와 PX4는 설계 사상의 차이가 존재한다.

  1. 지오펜스의 수학적 모델링 차이:
  • Ardupilot (Copter/Plane): 과거 원통형 경계와 다각형 경계를 별도로 처리하다가 최신 커밋에서 복합 펜스 구조를 도입하였다. 하지만 충돌 회피 기동 전략에 있어 다각형 변을 벽(Wall)처럼 인식하여 AC_Avoid 라이브러리를 통해 속도 벡터(Velocity Vector)를 반대 방향으로 줄이는 형태의 반발력(Repulsion) 기반 알고리즘을 혼합하여 적극 사용한다.
  • PX4-Autopilot: 주로 목표 지점(Setpoints)을 경계선 안쪽으로 자르는(Clipping/Clamping) 명시적인 방식과, 엄격하게 Winding Number 등을 적용하여 위반 시 Navigator 레벨의 높은 추상화 단계에서 예외 행동(RTL 등)으로 State 자체를 변경해버리는 결정론적(Deterministic) 방식을 선호한다.
  1. 이륙 단계(Takeoff) 검증 주도권:
  • PX4의 이륙 명령 처리 시 지오펜스 교차 예측 알고리즘은 비행의 시작부터 종료까지 Navigator가 전역적 감시자 (Global Sentinel) 역할을 하면서 State 전이를 관장한다. 반면, Ardupilot은 이륙 전 Pre-Arm Check 단계에서 고도 제한치를 먼저 강력하게 체킹한 후, Copter 모드의 궤적 튜닝 파라미터(예: WPNAV_SPEED_UP)에 의존하는 경향이 있다.

5. 결론

다각형 지오펜스와 상한 고도를 준수하기 위한 3차원 광선 교차 방어 로직은 단순 비인가 구역 진입 차단을 넘어서 비행 시스템 전체의 신뢰성을 담보하는 핵심적인 수학적·소프트웨어적 메커니즘이다.
PX4 개발자 및 GCS 엔지니어는 시스템 구축 시 이륙 교착 상태 해결뿐만 아니라, GF_MAX_HOR_DIST와 다각형 정점 간의 기하학적 교차 검토 비용을 최소화해야 한다. 즉, RTOS 환경의 자원 제약하에 뮐러-트룸보어(Möller-Trumbore) 교차 등 효율화된 렌더링/물리엔진 기반의 벡터 수학적 기법을 시스템 아키텍처에 연동함으로써, 돌발적인 센서 오차에 대해서도 기체의 안전 장치(Failsafe)와 제어 천이를 온전히 보장할 수 있을 것이다.
