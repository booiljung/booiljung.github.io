44.6.2. 상태 추정기(Estimator) 피드백 텔레메트리를 이용한 목표 고도 추종 오차(Tracking Error)의 실시간 편차 제어 로직

1. 개요

PX4-Autopilot의 이륙 완료 판별 알고리즘은 단방향적인 타이머 기반 통제가 아닌, 상태 추정기(Estimator)로부터 유입되는 정밀한 피드백 텔레메트리를 기반으로 동작하는 폐루프 제어(Closed-loop Control)의 핵심 영역이다. 비행 제어기(Flight Controller)는 기체가 상승함에 따라 생성되는 동적 목표 고도 궤적(Setpoint Trajectory)과, 센서 융합을 통해 산출된 실제 기체 고도 간의 차이인 추종 오차(Tracking Error)를 실시간으로 모니터링한다. 본 절에서는 확장 칼만 필터(EKF2, Estimation and Control Library) 시스템이 제공하는 텔레메트리를 활용하여, 목표 고도 추종 오차를 어떻게 제한 범위 내로 편차 제어하는지 심층적으로 논의한다.

2. EKF2 추정기의 이륙 고도 피드백 메커니즘

PX4-Autopilot(Firmware v1.14 기준)의 기본 에스테메이터인 EKF2 모듈은 관성 측정 장치(IMU)의 고주파 가속도 적분값과 기압계(Barometer), GNSS, 혹은 비전 센서(Visual Odometry)와 같은 저주파 위치 참조 센서의 관측값을 상태 융합(State Fusion) 알고리즘을 통해 결합한다.

  • 안정적 NED 프레임 구축: EKF2는 이륙 대기 상태(t=0)에서 로컬 좌표계(Local Position NED Frame)의 원점을 확립하며, 여기서 기체의 Z축(Down 축방향, 상승 시 음수 값을 가짐) 위치 Z_{est} 와 수직 속도 V_{Z,est} 를 고정밀로 추정해낸다.
  • 텔레메트리 발행(Publish): 추정된 Z_{est} 와 V_{Z,est} 는 vehicle_local_position uORB 토픽을 통해 100Hz 안팎의 고속 주기로 위치 제어기(Position Controller) 모듈에 지속적으로 피드백된다.

3. 위치 제어기 내부의 추종 오차(Tracking Error) 제어 루프

이륙(Takeoff) 명령이 개시되면, PX4 내의 궤적 생성기(Trajectory Generator)는 한순간에 최종 목표 고도(예: 3.0m)를 직접적으로 인가하지 않는다. 대신, 지정된 이륙 상승 속도 제약조건(MPC_TKO_SPEED) 산하에서 부드러운 램프(Ramp) 혹은 S-Curve 프로파일을 따르는 이상적인 Z축 목표 궤적 Z_{sp}(t) 을 실시간으로 생성한다.

flowchart LR
    A[궤적 생성기: Z_sp(t), Vz_sp(t)] -->|+ / -| B((합산기))
    C[EKF2 추정기: Z_est(t), Vz_est(t)] -->|- / -| B
    B -->|e_z(t) 위치 추종 오차| D[위치 제어 루프: P-Controller]
    D -->|Target V_z 보정 입력| E[속도 제어 루프: PID-Controller]
    E -->|Thrust Setpoint| F[믹서 및 모터 출력]
    F -->|물리적 상승| G[기체 동역학]
    G -->|센서 측정| C
  • 실시간 편차 산출식: 매 제어 프레임마다, 위치 제어기는 수직 위치 추종 오차 e_z(t) 를 다음과 같이 계산한다.
    e_z(t) = Z_{sp}(t) - Z_{est}(t)
  • 제어 보상(Compensation): 이 위치 편차 e_z(t) 는 P 제어기 이득(Gain)인 MPC_Z_P 파라미터 계수와 비례 증폭되어 하위 속도 제어기의 입력값(Velocity Setpoint) 수정을 유도한다. 만약 e_z(t) 가 커지면(기체의 실제 상승이 목표 궤적을 제때 뒤따라가지 못하면), 제어기는 수직 속도 요구치를 상향 조정하여 기체가 이상적인 궤적에 신속히 복귀(Tracking catch-up)하도록 모터 추력을 증가시킨다.

4. 이륙 완료 판별을 위한 편차 검증의 동기화

목표를 향한 비행 중 최종 위치(Final Altitude Setpoint)에 기체가 접근하면(Z_{sp}(t) = Z_{target}), 단순히 목표 고도값을 산술적으로 초과했는지만을 따지지 않고, 편차 제어 루프의 수렴성(Convergence)에 대한 복합 다이내믹스 검증을 실시한다.

  1. 위치 추종 편차 수용 범위(Tolerance): e_z(t) 가 파라미터에 지정된 수직 편차 마진(Vertical Acceptance Radius)보다 작은가? 즉, \vert e_z(t) \vert \le \epsilon_p 가 성립하는지 확인한다.
  2. 속도 추종 편차 감쇠 확인: 위치 오차가 수용 범위에 들어왔더라도 기체가 상승 관성에 의해 빠르게 치솟고 있다면 오버슈트(Overshoot) 발생 위험이 높다. 따라서 수직 속도 편차 e_v(t) = V_{Z,sp}(t) - V_{Z,est}(t) 를 조사하여 실제 수직 속도가 영(0)에 근접했는지를 영구속 판별(Zero-velocity logic) 알고리즘을 통해 추가 검사한다.
  3. 글리치(Glitch) 시간 보호: EKF2 내 기압계의 난기류 유발 압력 변화나 GPS 간섭으로 일시적인 추정 스파이크(Spike)가 발생하면 e_z(t) 데이터가 순간적으로 진동할 수 있다. PX4-Autopilot은 이 편차 수렴 상태가 특정 안정화 검증 시간 타임아웃 윈도우(Timeout Buffer Window) 동안 연속 유지될 때만 최종적으로 Takeoff Reached 플래그를 참(True)으로 확정 선언한다.

5. Ardupilot의 고도 제어 비교

Ardupilot 역시 동일하게 확장 칼만 필터(EKF3)를 기반으로 고도 피드백을 수신하나, 이륙 편차 보상 로직의 설계 철학 구조에는 미세한 접근 방식 차이가 존재한다.
Ardupilot은 이륙 전용 제어기 단계에서 최소 이륙 추력 리미트(Takeoff Throttle)를 명시적으로 규정하여 초기 이륙 시 I 적분항 거동을 의도적으로 빠르게 변조, 오차를 묵시하고 신속히 기단을 지면 충돌 난기류(Ground Effect) 범위 밖으로 이탈시키는 데 주안점을 둔다.
반면 PX4는 범용적인 위치-속도 캐스케이드(Cascaded) 제어 구조를 최초 이륙 시점부터 끝까지 일관되게 적용하되, 정밀한 Trajectory 추종 오차 e_z(t) 중심의 연속 상승 곡선을 통해 역학적 제어 엄밀성에 무게를 싣는 편이다.

6. 결론

PX4-Autopilot의 이륙 완료 판별 과정은 EKF2 상태 추정기로부터 도출된 고해상도 텔레메트리와 다단계 제어 루프의 치밀한 편차 분석 체계가 결합된 고차원적 루틴이다. 지속적으로 추종 오차 e_z(t)를 산출하여 편차를 필터링하고 동적 오버슈트를 제어하는 설계 덕분에, PX4 기체 시스템은 외부의 환경적 외란이나 탑재 페이로드 변동에도 불구하고 높은 수준의 이륙 제어 신뢰도(Reliability) 및 궤적 안정성을 변함없이 보장할 수 있다.
