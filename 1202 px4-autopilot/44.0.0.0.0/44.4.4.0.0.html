44.4.4 추력 제어(Spool up) 및 모터 구동기(Mixer) 할당 프로파일

1. 개요 및 배경

PX4-Autopilot에서 멀티로터의 제어 시스템이 비행을 결정하고 최종 제어 명령(Control Allocation)을 실제 물리적인 회전 구동 부품으로 에스컬레이트하기 직전 단계를 관장하는 것이 바로 믹서(Mixer) 아키텍처이다. 시동(Arming) 직후부터 기체가 완전히 공중으로 부양하는 이륙(Takeoff) 과도기 상태까지, 상위 자동 비행 시스템이나 조종사의 추력 명령이 모터에 즉각적으로 100% 인가되면 기체는 매우 불안정한 롤(Roll)/피치(Pitch) 트위스트나 급격한 요(Yaw) 스핀 역학을 필연적으로 겪게 된다.

이를 선제 방어하기 위해 PX4는 스풀업(Spool-up) 이라는 독립적인 추력 제어 상태 기계(State Machine)를 도입하였다. 스풀업 로직은 모터 구동계가 이륙 명령과 함께 안전하고 부드럽게 목표 회전수에 도달할 수 있도록 추력 상승 제한(Ramping) 곡선을 강압적으로 부여하며, 이 제한된 추력은 모터 구동기(Mixer / Control Allocator) 모듈을 통과하며 기체의 기하학적 형상(멀티로터의 모터 배치, 쿼드/헥사 구조 등)에 최적화되게 각 모터의 PWM, DShot 신호로 역산 분배된다.

2. 모터 스풀업(Spool-up) 상태 기계 (State Machine) 구조 역공학

PX4의 모터 스풀업 내부 제어 로직은 이륙 초기의 비행 안정성을 극한으로 확보하기 위해 구동기 출력을 시간에 따라 4단계로 세밀하게 구획하고 통제한다. 이 상태 머신의 상태 변이는 시간에 의존하는 엄격한 사이클을 따른다.

stateDiagram-v2
    [*] --> STATE_OFF : Disarmed 상태 (시동 해제)
    STATE_OFF --> STATE_IDLE : Arming 명령 수신 및 안전 절차 통과
    STATE_IDLE --> STATE_SPOOLING_UP : 최소 추력 이상의 Takeoff 명령 인가
    STATE_SPOOLING_UP --> STATE_ON : 파라미터 정의 램프(Ramp) 완료 시간 도달
    STATE_ON --> STATE_OFF : Disarming 명령 수신 (착륙 완료)
  1. STATE_OFF: 기체의 시동이 해제된 안전 모드로 모터 ESC 측에 어떠한 PWM 출력도 나가지 않거나(혹은 DShot 디스암 커맨드), 최소 강하 신호만 유지되는 완전 물리적 격리 상태이다.
  2. STATE_IDLE: 시동(Arm)이 성공적으로 걸린 직후의 상태. 기체 모델 파라미터 COM_MOT_STARTS나 MOT_SLEW_TIME 설정에 따라 모터가 가장 낮은 공회전(Idle) 속도로 회전하여, 유체 저항을 뚫고 즉각적인 추력을 발생시킬 태세를 갖춘다.
  3. STATE_SPOOLING_UP: 스로틀(Throttle) 셋포인트가 부상할 정도로 올라가거나 상위 모듈에서 자동 이륙(Auto Takeoff) 제어 명령이 하달되었을 때 진입하는 과도기(Transient) 상태. 파라미터 COM_SPOOLUP_TIME에 정의된 시간 제약(예: 1.0초) 동안 최대 추력 한계 허용치를 0에서 기준치까지 선형적으로(Linear Ramp) 증가시킨다.
  4. STATE_ON: 지정된 스풀업 시간이 온전히 완료되어, 속도 제어기(PID) 루프가 논리적으로 요구하는 전체 추력을 모터가 제약 없이 100% 수행토록 열려있는 비행(In-air) 정상 작동 모드이다.

3. 제어 할당 (Control Allocation) 모듈 및 핵심 믹서(Mixer) 수리 아키텍처

근래의 PX4 v1.14 아키텍처부터는 전통적으로 SD카드에서 인터프리트하던 정적 믹서 파일(.main.mix, .aux.mix) 기반 체계를 탈피하고, C++ 행렬 계산 기반의 동적 제어 할당 코어 모듈(src/modules/control_allocator)로 고도화 전환되었다. 속도 및 무브먼트 제어기(PID)에서 역공학 된 3축 토크 지령(roll, pitch, yaw)과 Z축 승각 추력(thrust)은 이 제어 할당기를 거치며, 기체 동역학 매트릭스에 투영되어 각 모터 ESC의 개별 RPM 지령(PWM / DShot 텔레메트리)으로 정밀 변환된다.

3.1 믹서 매트릭스 선형 변환 (Matrix Mathematical Evaluation)

전형적인 쿼드콥터(Quadrotor-X 배치)를 기준으로 4개의 모터 출력벡터 \mathbf{u} = [u_1, u_2, u_3, u_4]^T는 정교하게 계산된 기하학적 믹스 행렬 \mathbf{M}과 3차원 토크/추력 명령벡터 \mathbf{c} = [\tau_x, \tau_y, \tau_z, f_z]^T의 선형 사상(Linear Mapping)을 통해 다음과 같이 산출된다.

\mathbf{u} = \mathbf{M} \, \mathbf{c}

이 고속 연산 과정 중 특정 모터의 출력 지령값이 100%(최대 구동 능력)를 초과하거나 0%(최소 가동 기준치) 미만 지대로 산출되는 이상 역학 범위가 발생할 수 있다. 이때 기체의 공간 상 자세 유지 우선순위(Attitude Priority)를 결사적으로 방어하기 위해 절대 추력(Thrust) 자체를 능동적으로 스케일다운 시켜(Thrust Loss 감수) 여유 토크(Torque Margin)를 확보해내는 안티-새추레이션 관리(Saturation Management) 가 제어 할당 알고리즘 내에서 동시 수행된다.

5. 결론 및 공학적 시사점 총평

추력 제어 스풀업과 모터 구동기 믹스 할당 아키텍처는 PX4-Autopilot 내부에 떠도는 가상 제어 신호(수학적 벡터 텐서)를 실제 자연 물리계(모터 RPM 및 공기 역학적 부양력)로 안전하고 효율적으로 치환하는 절대적인 병목(Bottleneck)점이다. 이륙 초기 모터/로터 관성 극복을 통제하는 스풀업 램핑 기법은 기계적 과도기 충격을 대폭 감소시키며 기구학적 수명을 늘린다. 더불어 진보된 동적 믹서 시스템 알고리즘은 극한의 항력이나 모터 1개 결함과 같아 불가피하게 시스템 요구량이 모터 물리적 한계를 초과하는 극한 상황에서도, 기체의 완전 조종 불능 추락 상태(Loss of Control)를 가용한 모든 자원을 끌어모아 막고 절대적인 코어 자세 안정성을 보장해 주는 방벽 역할을 수행한다.
