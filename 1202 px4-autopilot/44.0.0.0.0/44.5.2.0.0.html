44.5.2 지면 접촉 상태(ground_contact, maybe_landed, landed)의 슈미트 트리거(Schmitt Trigger) 기반 이력 현상(Hysteresis) 임계치 모델링

무인항공기(UAV)의 이착륙 과정은 로터에 의한 와류(Vortex)와 지면 효과(Ground Effect)로 인해 센서 데이터에 극심한 기계적 진동과 공기역학적 노이즈를 동반한다. 이러한 잡음 속에서 기체가 지면에 완전히 착륙했는지 판별하기 위해, PX4-Autopilot(Firmware v1.14 기준)의 Land Detector 모듈은 단순한 불리언(Boolean) 기반의 1차원 판별 로직을 탈피하고, 3단계 계층적 상태 구조(ground_contact, maybe_landed, landed)와 슈미트 트리거(Schmitt Trigger)를 응용한 시간 기반 이력 현상(Hysteresis) 필터를 도입하였다. 본 장에서는 이 모델의 수학적·논리적 아키텍처를 분석한다.

1. 3단계 계층적 지면 접촉 추정 모델 (3-Stage Estimation Model)

단일 임계치를 적용할 경우 기체가 살짝 바운싱(Bouncing)될 때마다 상태가 빈번하게 토글(Toggle)되는 채터링(Chattering) 현상이 발생한다. PX4는 상태의 전환을 3단계로 분리하여 각 판별 조건과 요구 유지 시간을 다르게 설정한다.

  1. 지면 접촉 (Ground Contact, ground_contact):
  • 의미: 기체의 랜딩 기어가 물리적으로 지면에 터치다운(Touchdown)했음을 1차적으로 감지하는 가장 민감한 상태이다.
  • 조건: 수직 속도(V_z)가 0 근처(LNDMC_Z_VEL_MAX 미만)에 도달하고, Z축 목표 설정 추력(Thrust Setpoint)이 유의미하게 하락했을 때 활성화된다.
  1. 착륙 가능성 (Maybe Landed, maybe_landed):
  • 의미: ground_contact가 트리거된 이후, 비행을 재개하려는 조종사의 조작(예: 스로틀 급상승)이 없음을 필터링하는 과도기(Intermediate) 상태이다.
  • 조건: ground_contact 상태가 해제되지 않고, 수평 방향의 이동(V_{xy})이나 거센 회전(Rotation)이 감지되지 않은 상태가 임정 시간 이상 유지될 때 켜진다.
  1. 최종 착륙 확정 (Landed, landed):
  • 의미: 기체가 완전히 지면에 도달하여 모터를 정지(Disarming)해도 안전한 최종 확정 상태이다.
  • 조건: maybe_landed 상태가 지정된 히스테리시스 타이머(Hysteresis Timer)를 완전히 소진할 만큼 장해 없이 유지될 때 최종적으로 펌웨어에 공표(Publish)된다.

2. 슈미트 트리거(Schmitt Trigger) 및 시간 이력 현상(Hysteresis)의 공학적 모델링

전자 공학에서의 슈미트 트리거 소자 메커니즘을 소프트웨어적으로 차용하여, 상승 에지(Rising Edge)와 하강 에지(Falling Edge)의 판별 임계치를 다르게 설계하고(비대칭성), 여기에 타이머(Timer) 기반 지연 검증을 추가한 것이 PX4 히스테리시스 필터이다.

    In_Air --> Ground_Contact : Vz < 임계치 & Thrust 하락 감지

    Ground_Contact --> Maybe_Landed : Ground_Contact 연속 유지 (Hysteresis 시간 초과)

    Landed --> In_Air : 높은 스로틀 & 이륙 의지 인가 및 가속도 감지

2.1 Hysteresis 필터의 핵심 변수

객체 지향적으로 분리된 Hysteresis 클래스 객체가 이 상태 전이 타이머를 전담한다.

  • set_hysteresis_time_from(false, true): In_Air (false) 에서 Landed (true) 로 전이되기까지 요구되는 지연 시간이다. 보통 착륙 확정을 위해 엄격하고 보수적인 시간(예: 수백 밀리초 ~ 1초)이 요구된다.
  • set_hysteresis_time_from(true, false): 역으로 지면을 이탈하여 비행(In_Air)으로 판별할 때 요구되는 시간이다. 기체가 튀어오를 경우 신속하게 자세 제어 적분기(Integrator)를 복구해야 하므로 착륙 판별 시간보다 훨씬 짧은 임계치로 세팅된다.

3.1 PX4와 Ardupilot 간 아키텍처적 관점의 차이

  • Ardupilot (Firmware v4.4): Ardupilot은 착륙 점검(Land Check) 기능을 커맨더 스케줄러 계층에 병합하여 10hz나 400hz 등 메인 루틴 안에서 속도 및 추력 조건을 다중 카운터(Counter) 변수(land_detector_count)로 증가시키며 판단(LAND_DETECTOR_TRIGGER_SEC 참조)한다. 플래그 관리에 가깝다.
  • PX4-Autopilot: 상태 간의 결합도를 낮추기 위해 별도 쓰레드의 Land Detector 모듈 내에서 Hysteresis라는 독립 유틸리티 클래스를 주입(Injection)하여 사용한다. 시간 만료(Timeout) 로직을 RTOS 고해상도 타임스탬프(hrt_absolute_time)에 의존하여 구동하므로 타이밍 관리가 수학적으로 대단히 정교하다.

4. GCS(QGroundControl)와의 MAVLink 상호 작용

PX4 내부에서 슈미트 트리거 로직을 통과하여 최종적으로 landed = true 상태로 수렴하게 되면, 해당 결과는 uORB 메시지 구조체인 vehicle_land_detected 토픽으로 출판(Publish)된다.

  • 커맨더 모듈은 이를 즉각 수신하여 특정 모드(예: 자동 복귀 후 착륙 등)에 따라 모터를 완전히 끄는 Auto-Disarm 절차를 시작한다.
  • MAVLink의 EXTENDED_SYS_STATE 브로드캐스팅을 통해 관제 측 QGroundControl(QGC, v4.3 기준)에 전달되며, 이를 통해 관제 화면 상의 상태 아이콘이 “In_Air“에서 실시간으로 갱신된다. 만일 센서 튜닝 값이 잘못되어 landed 상태로 넘어가지 않는다면(예컨대 바닥의 호버링 이펙트를 해소하지 못해 Vz가 임계치 내에 못 들어감), 드론은 착륙 중임에도 무기한 모터를 끄지 못하는 불상사로 이어질 수 있으므로 임계치 튜닝은 매우 치명적인 과정이다.

5. 결론

최신 펌웨어의 PX4-Autopilot은 단순한 값 비교 판별에서 벗어나, 데이터 노이즈와 채터링을 소프트웨어 수준에서 여과하는 전자공학적 슈미트 트리거 메커니즘과 Hysteresis 필터를 적용하였다. ground_contact부터 landed에 이르는 예측-검증-확인의 3단계 파이프라인 덕분에, 불규칙한 비포장 지면이나 선박 위 착륙 시에도 오동작 없이 기체 파손을 방지한다. 이 구조는 ROS2를 기반으로 자율 이착륙 에이전트를 개발하거나 특수 거리 측정 스캐너 센서를 융합할 때, 상위 제어기가 신뢰할 수 있는 단일한 착륙 진실치(Ground Truth)를 제공해준다는 의의를 갖는다.
