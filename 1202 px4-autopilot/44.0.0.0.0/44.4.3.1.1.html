44.4.3.1.1 뱅뱅 제어(Bang-Bang Control) 원리를 응용한 3차 미분(Jerk) 구속 조건 기반 시간 최적화(Time-Optimal) 상승 프로파일

1. 개요 및 배경

PX4-Autopilot(기준 펌웨어 v1.14 이상 및 QGroundControl v4.3+)에서 멀티로터의 이륙(Takeoff) 및 고도 상승 제어는 단순한 목표 고도 도달을 넘어, 기계적 진동(Mechanical Resonance)을 최소화하고 승차감(Ride Comfort)과 유사한 비행 안정성을 확보하는 방향으로 진화해 왔다. 이 과정에서 핵심적인 역할을 하는 것이 유한 상태 기반의 궤적 생성기(Trajectory Generator)이다. 특히 지정된 목표 고도까지 가장 빠르게 도달하면서도(시간 최적화, Time-Optimal), 모터와 프레임에 가해지는 급격한 추력 변화를 방지하기 위해 3차 미분인 저크(Jerk, \frac{da}{dt})에 구속 조건(Constraint)을 부여한다.

이러한 저크 제한(Jerk-limited) 궤적 생성은 고전적인 뱅뱅 제어(Bang-Bang Control) 이론의 확장된 형태로 모델링된다. 뱅뱅 제어는 제어 입력을 양의 최대값과 음의 최대값 사이에서 스위칭하여 최단 시간 내에 시스템을 목표 상태로 천이시키는 기법이다. PX4에서는 이를 가속도(Acceleration)가 아닌 저크(Jerk) 상태 공간에 적용하여 부드러운 S-커브(S-curve) 속도 프로파일을 생성한다.

2. 뱅뱅 제어(Bang-Bang Control)와 저크 구속의 수학적 모델링

이륙 상승 모션에서 고도 z(t), 속도 v(t), 가속도 a(t), 저크 j(t) 사이의 관계는 연속적인 미적분 관계로 정의된다.

z(t) = \int v(t) dt, \quad v(t) = \int a(t) dt, \quad a(t) = \int j(t) dt

순수 뱅뱅 제어를 가속도 a(t)에 직접 적용하면 가속도가 불연속적으로 변하여 무한대의 저크가 발생하며, 이는 프레임에 심각한 충격(Impulse)을 준다. 따라서 PX4는 저크 j(t)를 물리적 제어 입력으로 설정하고 다음과 같은 시스템 구속 조건을 부여한다.

-J_{max} \le j(t) \le J_{max}
-A_{max} \le a(t) \le A_{max}
-V_{max} \le v(t) \le V_{max}

목표 도달 시간을 단축하기 위한 시간 최적화 상승 프로파일에서 저크 j(t)는 최댓값(+J_{max}), 0, 최솟값(-J_{max}) 중 하나의 값을 가지는 “Bang-Zero-Bang” 형태의 제어 시퀀스로 도출된다.

2.1 7구간 상승 트래젝토리 (7-Phase Trajectory)

위와 같은 구속 조건 하에서의 시간 최적화 프로파일은 최대 7개의 위상(Phase)으로 나뉜다. 이륙 상승 과정은 다음과 같이 모형화된다.

graph TD
    P1["Phase 1: 가속도 증가<br>(j = +Jmax)"] --> P2["Phase 2: 최대 가속도 도달 및 유지<br>(j = 0, a = Amax)"]
    P2 --> P3["Phase 3: 가속도 감소<br>(j = -Jmax)"]
    P3 --> P4["Phase 4: 최대 속도 도달 및 유지<br>(j = 0, a = 0, v = Vmax)"]
    P4 --> P5["Phase 5: 감속도 증가 (가속도의 음수화)<br>(j = -Jmax)"]
    P5 --> P6["Phase 6: 최대 감속도 도달 및 유지<br>(j = 0, a = -Amax)"]
    P6 --> P7["Phase 7: 감속도 감소<br>(j = +Jmax, a -> 0, v -> 0)"]
    P7 --> End["목표 고도 정밀 도달 (Hovering)"]

위 7구간 프로파일을 통해 속도 프로파일은 자연스러운 S자 곡선을 그리게 되어 이륙 제어 시 안정성이 극대화된다.

3. PX4-Autopilot 소스 코드 내 구현 아키텍처

PX4에서는 이러한 시간 최적화 궤적을 FlightTaskAutoTakeoff 클래스와 VelocitySmoothing 기능 모듈을 조합하여 구현한다.

3.1 VelocitySmoothing 및 Trajectory 라이브러리 분석

소스코드 레벨에서 3차 미분 구속 기반 프로파일 생성은 주로 src/lib/mathlib/math/filter/VelocitySmoothing.cpp 및 독립적인 Trajectory 유틸리티에서 이루어진다.

핵심 알고리즘은 현재 상태벡터(위치, 속도, 가속도)에서 목표 상태까지 다항식 형태의 상태 천이를 계산하고, 제어 주기(제어 루프 비율인 250Hz, 4ms 등) 단위로 기댓값(Setpoint)을 부드럽게 생성한다.

  • 파라미터 연동 메커니즘:
  • MPC_JERK_MAX: 최대 저크(Jerk)를 한계값으로 설정한다. 값이 작을수록 움직임은 극적으로 부드러워지지만 역설적으로 목표 고도 до달 시간은 길어진다.
  • MPC_ACC_UP_MAX: 상승 시 허용되는 최대 가속도.
  • MPC_Z_VEL_MAX_UP: 최대 수직(상승) 속도.

이러한 파라미터들은 이륙(Takeoff) 비행 모드는 물론 GCS(QGroundControl) 상에서 사용자가 미세하게 조정할 수 있도록 노출되어 있다.

4. Ardupilot 대비 관제 시스템 및 제어 철학의 차이

Ardupilot(Copter-4.3 이상 버전에 도입된 SCurve) 역시 이러한 7구간 프로파일(S-Curve)과 동일한 형태의 궤적 생성기를 사용한다. 하지만 궤적 생성기와 비행 모드 사이의 결합도(Coupling) 측면에서 유의미한 구조적 차이를 갖는다.

궤적 생성 알고리즘 위치FlightTask 체제의 독립 모듈 (VelocitySmoothing)항법 컨트롤러 코어 객체 내로 편입 (AC_PosControl / AC_AttitudeControl)
튜닝 파라미터 구조MPC_JERK_MAX, MPC_JERK_AUTO 등 제어 루프 독립적 설계WPNAV_ACCEL_Z, WPNAV_JERK 등 웨이포인트(Waypoint) 변수에 강하게 종속됨
시스템 유연성(Flexibility)uORB 버스로 trajectory_setpoint 토픽을 발행하여 모듈 격리가 우수객체 지향의 클래스 다형성을 유지하며 내부 로직에 타이트하게 결합

PX4 시스템은 uORB(Micro Object Request Broker) 인터페이스를 기반으로 모듈성을 극대화하였다. 이로 인해 임무(Mission) 모드 전용 궤적 생성기든 자율 회피 시스템이든, 서로 간섭 없이 시간 최적화 제어기를 통해 하위 자세 제어부에 일관된 궤적 명령을 전송할 수 있다.

5. 결론 및 공학적 시사점

뱅뱅 제어 원리를 3차 미분 상태 공간으로 확장한 저크 제한 시간 최적화 상승 프로파일은 현대적인 PX4-Autopilot 시스템에서 안정적 자율 비행의 중추적인 역할을 담당한다. 이는 멀티로터의 상승 과도 응답(Transient Response)에서 발생하는 진동을 대폭 감소시켜, Cascade 구조로 된 내부 위치 제어기(Position Controller)와 자세 제어기(Attitude Controller)에 입력되는 왜곡 요소를 최소화한다. 결론적으로 이 알고리즘은 관제 시스템의 목표 고도 명령에 대해 시스템이 견고(Robust)하고 신속하게 대응할 수 있도록 이론적, 실무적 완성도를 부여한다.
