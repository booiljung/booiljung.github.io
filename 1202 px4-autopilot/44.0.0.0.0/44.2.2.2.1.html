44.2.2.2.1. 오일러 각(Euler Angle) 입력값의 쿼터니언(Quaternion) 변환 및 짐벌 락(Gimbal Lock) 특이점 회피 알고리즘

1. 개요 (Introduction)

무인항공기의 3차원 공간 내 자세(Attitude)를 제어하고 표현하는 방법론은 비행 제어기(Flight Controller) 설계의 핵심이다. PX4-Autopilot(기준 펌웨어 버전: v1.14.0)을 비롯한 현대적 드론 제어 시스템은 직관적인 사용자 인터페이스를 위해 지상 관제 시스템(예: QGroundControl v4.3)에서는 롤(Roll), 피치(Pitch), 요(Yaw)라는 오일러 각(Euler Angle)을 입력받거나 표시하지만, 내부 연산 및 통신 과정에서는 반드시 쿼터니언(Quaternion, 사원수)으로 변환하여 처리한다. 본 절에서는 오일러 각 기반 자세 제어의 본질적인 한계인 짐벌 락(Gimbal Lock) 현상의 수학적 원인과, 이를 회피하기 위한 PX4-Autopilot 내부의 쿼터니언 변환 알고리즘 및 소프트웨어 아키텍처를 심도 있게 분석한다.

2. 짐벌 락(Gimbal Lock) 특이점 현상의 기하학적 이해

오일러 각은 3개의 직교하는 회전 축(일반적으로 항공 역학에서는 Z-Y-X 순서의 Tait-Bryan 각을 의미)을 순차적으로 회전시켜 물체의 자세를 정의한다. 그러나 이 순차적 회전 방식은 기체의 피치각(Pitch angle)이 90°(수직 상승) 또는 -90°(수직 하강)에 도달할 때, 첫 번째 회전 축(Yaw)과 세 번째 회전 축(Roll)이 기하학적으로 평행하게 정렬되어 하나의 회전 자유도(Degree of Freedom)를 상실하는 특이점(Singularity) 현상을 유발한다. 이 현상을 짐벌 락(Gimbal Lock)이라 부른다.

만약 비행 제어기가 오일러 각만을 의존하여 제어 루프를 구성할 경우, 고정익(Fixed-wing)이나 VTOL 기체가 급격한 기동을 수행하거나 상승 궤적을 그릴 때 특이점 부근에서 자세 추정 연산이 무한대(NaN)로 발산하여 제어 불능 상태에 빠지게 된다. 과거 초창기 ArduPilot이나 단순한 제어 루프에서는 이 범위를 소프트웨어적으로 제한하여 회피했으나, PX4-Autopilot은 전역적인 비특이성(Non-singularity)을 갖는 쿼터니언(Quaternion) 체계를 코어 아키텍처에 내재화하여 이 문제를 근본적으로 해결했다.

3. 쿼터니언(Quaternion) 회전 연산의 수학적 모델

해밀턴(W. R. Hamilton)이 고안한 쿼터니언은 4차원 복소수 체계를 이용하여 3차원 공간의 회전을 콤팩트하게 표현한다. 쿼터니언 q는 하나의 실수부와 세 개의 허수부로 구성된다.

q = q_0 + q_1 \mathbf{i} + q_2 \mathbf{j} + q_3 \mathbf{k}
또는 스칼라(Scalar) 성분 w와 벡터(Vector) 성분 \mathbf{v} = (x, y, z)의 결합인 q = (w, x, y, z) 로 표현할 수 있다.
임의의 회전 축 \mathbf{u} = (u_x, u_y, u_z) (단위 벡터)를 기준으로 \theta 만큼 회전하는 회전 변환은 다음과 같은 쿼터니언으로 정의된다.

q = \cos\left(\frac{\theta}{2}\right) + \sin\left(\frac{\theta}{2}\right)(u_x\mathbf{i} + u_y\mathbf{j} + u_z\mathbf{k})

이 모델은 삼각함수의 분모가 0이 되는 지점이 존재하지 않으므로 전 자세 구간(All-attitude)에서 기하학적 연속성을 보장한다.

4. PX4 소스코드 레벨의 오일러-쿼터니언 변환 및 회피 로직 (Source Code Analysis)

PX4-Autopilot은 matrix라는 자체 수학 라이브러리를 통해 이 변환 알고리즘을 최적화된 C++ 템플릿 형태로 구현하고 있다 (src/lib/matrix/matrix/math.hpp 참조). QGroundControl GCS나 ROS2 Offboard 모드로부터 오일러 각 타겟 메시지(예: MAV_CMD_DO_SET_ATTITUDE)를 수신하면, 수신 모듈 및 제어기 진입부에서 즉각적으로 쿼터니언 인스턴스로 자료값을 파싱한다.

4.1 변환 알고리즘 (Conversion Algorithm)

주어진 Roll(\phi), Pitch(\theta), Yaw(\psi)에 대해 PX4 펌웨어 내부는 각 절반 각도(\frac{\phi}{2}, \frac{\theta}{2}, \frac{\psi}{2})의 사인(sine)과 코사인(cosine)을 계산하고 이를 행렬 연산 대신 대수적으로 결합한다.

// PX4 matrix 라이브러리 방식의 단순화된 구현 예시
void EulerToQuaternion(float roll, float pitch, float yaw, float q[4]) {
    float cy = cos(yaw * 0.5f);
    float sy = sin(yaw * 0.5f);
    float cp = cos(pitch * 0.5f);
    float sp = sin(pitch * 0.5f);
    float cr = cos(roll * 0.5f);
    float sr = sin(roll * 0.5f);

    q[0] = cr * cp * cy + sr * sp * sy; // w 성분
    q[1] = sr * cp * cy - cr * sp * sy; // x 성분
    q[2] = cr * sp * cy + sr * cp * sy; // y 성분
    q[3] = cr * cp * sy - sr * sp * cy; // z 성분
}

4.2 제어 분해능 향상과 계산 효율성 (Computational Efficiency)

오일러 각 기반의 회전 행렬(Rotation Matrix)이 9개의 요소와 잦은 삼각함수 연산을 요구하는 반면, 쿼터니언 결합(Multiplication) 연산은 순수 대수 연산만으로 두 자세의 오차를 매우 빠르게 연산한다. 자세 제어기(mc_att_control) 내부에서 현재 기체 자세 쿼터니언과 목표 자세 쿼터니언의 에러 쿼터니언 q_{error} = q_{target} \otimes q_{current}^{-1} 을 구하는 과정은 삼각함수 없이 덧셈, 뺄셈, 곱셈 파이프라인으로 처리되며 이로 인해 NuttX RTOS 기반의 CPU 스케줄링 자원을 대폭 절감한다.

5. MAVLink 프로토콜 및 ROS2 생태계에서의 강제성 (MAVLink and ROS2 Strictness)

초기 드론 통신 프로토콜들은 대규모 데이터 전송량을 줄이고 디버깅을 용이하게 하기 위해 오일러 각 메시지를 주로 활용했다. 하지만 학계의 연구(“Attitude Representation for Drone Control,” IEEE Transactions, 2018 등)와 실제 비행 데이터 로깅(Log analysis) 시스템(ULog) 분석 결과가 누적됨에 따라, PX4 개발진은 특이점 회피를 절대적인 기준으로 삼기 시작했다.
현재 ROS2 uXRCE-DDS 브릿지를 이용해 에이전트를 개발할 때, 토픽 간 교환되는 VehicleAttitude.msg 타입 데이터는 철저하게 단위 쿼터니언 배열로 강제 규정된다. GCS 응용프로그램 개발자는 사용자 UI 레벨에서만 오일러 각 변환을 수행하고, 백엔드 통신 및 로우 레벨 API 호출에는 변환된 쿼터니언 구조체(Quatf)를 주입해야 한다.

6. 결론 (Conclusion)

PX4-Autopilot에서 짐벌 락(Gimbal Lock) 특이점 회피는 단순히 예외 경우를 차단하기 위한 우회(Bypass) 수단이 아니라, 코어 수학 라이브러리와 자세 제어 루프를 아우르는 근본적인 수학적 스키마타(Mathematical Schemata) 설계다. QGroundControl 및 ROS2 융합 환경 개발 시, 개발자는 입력 단계에서의 오일러-쿼터니언 매핑 구조와 계산의 가벼움(Lightweight)을 이해해야만 복잡한 군집 비행(Swarm)이나 아크로바틱(Acrobatic) 기동 제어기의 정밀성을 십분 발휘할 수 있을 것이다.
