44.3.1.3.1. 쌍선형 보간법(Bilinear Interpolation)을 이용한 이산적(Discrete) 지형 그리드 데이터의 공간 연속성 확보

1. 개요 및 필요성

비행 중인 무인항공기(UAS)의 지형 추종(Terrain Following)을 위해서는 실시간으로 기체 바로 아래의 지표면 절대 고도(AMSL)를 정확히 알아야 한다. 이전 절에서 설명한 바와 같이, PX4-Autopilot(Firmware 버전 v1.14 기준)은 MAVLink 통신을 통해 QGroundControl(QGC)로부터 지형의 수치 표고 모델(DEM: Digital Elevation Model) 타일 데이터를 수신한다.

그러나 이 DEM 데이터는 연속된 3차원 곡면이 아니라, 특정한 위도(Latitude)와 경도(Longitude) 간격으로 샘플링된 이산적(Discrete)인 그리드 포인트(Grid Point) 의 집합이다. 드론이 이 그리드 포인트 정확히 위를 비행하는 순간은 거의 없으며, 대부분의 비행 궤적은 4개의 인접한 그리드 포인트 사이의 임의의 공간을 가로지르게 된다. 만약 드론이 가장 가까운 그리드의 고도값만 단편적으로 채택(Nearest-Neighbor)하게 되면, 그리드를 넘어갈 때마다 고도 설정점(Altitude Setpoint)이 계단식(Step-function)으로 급변하게 되어, 위치 제어기(Position Controller)에 과도한 제어 오차를 유발하고 피치(Pitch)/스로틀(Throttle)의 급격한 출렁임(Oscillation)을 발생시킨다.

이를 방지하고 이산적 데이터를 부드러운 연속 공간(Continuous Space)으로 변환하기 위해, PX4 시스템은 쌍선형 보간법(Bilinear Interpolation) 알고리즘을 적용하여 현재 위치의 정확한 지형 고도를 추정(Estimation)한다.

2. 쌍선형 보간법(Bilinear Interpolation)의 수학적 모델링

쌍선형 보간법은 2차원 공간(위도와 경도 바탕의 평면)에서 1차원 선형 보간(Linear Interpolation)을 두 번 연속 수행하여, 네 개의 픽셀(주변 그리드 꼭짓점) 사이의 임의의 점에 대한 값을 합성하는 기법이다.

기체의 현재 위치가 (x, y)이고, 이 위치를 둘러싸고 있는 DEM 데이터의 4개의 기준점(Grid Points)을 각각 Q_{11}, Q_{12}, Q_{21}, Q_{22} 라고 정의하자.

  • Q_{11} = (x_1, y_1)에서의 지형 고도 h_{11}
  • Q_{12} = (x_1, y_2)에서의 지형 고도 h_{12}
  • Q_{21} = (x_2, y_1)에서의 지형 고도 h_{21}
  • Q_{22} = (x_2, y_2)에서의 지형 고도 h_{22}

이때 x_1 \le x \le x_2 이고 y_1 \le y \le y_2 이다.

2.1 1단계: X축(위도 또는 경도 방향) 선형 보간

먼저 x축 방향으로 두 개의 중간점 R_1과 R_2의 고도를 산출한다.
h(R_1) = \frac{x_2 - x}{x_2 - x_1} h_{11} + \frac{x - x_1}{x_2 - x_1} h_{21}
h(R_2) = \frac{x_2 - x}{x_2 - x_1} h_{12} + \frac{x - x_1}{x_2 - x_1} h_{22}

2.2 2단계: Y축 방향 최종 선형 보간

구해진 R_1(x, y_1)과 R_2(x, y_2)를 바탕으로 y축 방향으로 한 번 더 보간하여 기체의 최종 좌표 (x, y)에서의 지형 고도 h(x, y)를 도출한다.
h(x, y) = \frac{y_2 - y}{y_2 - y_1} h(R_1) + \frac{y - y_1}{y_2 - y_1} h(R_2)

3. PX4-Autopilot 아키텍처 내 적용 및 소스 코드 동작 원리

PX4 소스 코드의 Terrain 모듈(Estimator/Terrain 폴더 내)에서는 이산된 지형 고도 정보인 TERRAIN_DATA 메시지를 수신하여 캐시에 저장해 두고 있다. 기상 정보나 궤적(Trajectory) 추종 모드가 활성화되어 지형 고도를 요구할 때, 내부 함수는 아래의 워크플로우를 따른다.

graph TD
    A[Current Global Position\nLat, Lon from EKF2] --> B{Find 4 Nearest DEM Grid Points}
    B -->|Fetch| C(h11, h12, h21, h22 from Terrain Cache)
    C --> D[Calculate Normalized Distance\nFraction x, y]
    D --> E[Compute Bilinear Interpolation Equation]
    E --> F[Publish to uORB\n'vehicle_local_position.vxy_terrain']
    F --> G[Flight Task Controller\nGenerates Smooth Altitude Setpoint]
  1. 그리드 인덱싱(Grid Indexing): EKF2가 추정한 고정밀 위도/경도를 바탕으로 현재 위치가 포함된 4방향의 타일 인덱스(Tile Index)를 찾는다.
  2. 분수 거릿값(Fractional Distance) 계산: 기체가 두 그리드 좌표 간 어디쯤 위치해 있는지 비율(Fraction, 0.0 \sim 1.0)을 산출한다.
  3. 고도 도출(Altitude Calculation): 쌍선형 보간 수학식에 대입하여 연속성을 가지는 Z_{terrain} 값을 도출한다.
  4. 저주파 필터링(Low-pass Filtering): 혹시 모를 타일 가장자리의 미세한 노이즈를 방어하기 위해 약한 위치 제어 필터를 거쳐 vehicle_local_position uORB 토픽에 발행(Publishing)한다.

4. 제어 포화(Actuator Saturation) 방어 효과

단순 근접 이웃(Nearest-Neighbor) 보간을 사용했다면, 그리드 타일의 경계를 지날 때마다 Z_{terrain}이 순간적으로 1 \sim 5m 이상 점프(Jump)하는 현상이 일어날 수 있다. PID 제어기로 구성된 고도 제어 루프(Altitude Control Loop)는 이 급격한 오차(Error)에 대해 미분 제어(D-term)와 비례 제어(P-term)를 크게 발생시켜, 순간적으로 모터(ESC)에 스로틀(Throttle) 100%를 명령하는 제어 포화(Actuator Saturation) 를 야기할 수 있다.

쌍선형 보간법은 이러한 계단식 입력(Step Input)을 부드러운 경사로(Ramp Input)로 변환해 준다. 결과적으로 드론이 산비탈을 올라갈 때 제동력과 추력을 급격하게 허비하지 않고 매끄러운 상승 곡선을 그리게 만들어 줌으로써 배터리 소모를 줄이고 비행 안정성을 확보하는 핵심 설계 철학을 보여준다.

5. PX4와 Ardupilot 의 비교 차이

  • 연산 정밀도 및 타입: PX4 펌웨어는 실시간 임베디드 운영체제(NuttX RTOS) 상에서 보간법 코어 로직 수행 시 부동소수점(float형) 고속 연산(FPU 칩셋 활용)을 통해 이 과정을 매 타임 스텝(250Hz 주기의 제어 루프 내부)마다 막힘없이 처리하도록 설계되었다.
  • Ardupilot의 보간 철학: Ardupilot 또한 동일한 쌍선형 보간법 철학을 사용하고 있으나, 지형 정보 추종의 보간(Interpolation) 연산을 EKF 깊숙한 단위 통합보다 AP_Terrain 라이브러리로 분리하여 L1 내비게이션(L1 Navigation Controller)에 전달하는 구조적 특징을 갖는다. 기본적 수학적 원리는 같으나 위치 제어기(PosControl)가 해당 추세를 받아들이는 스플라인(Spline) 추종 민감도는 두 오토파일럿(Auto-pilot) 간 미묘한 주행 궤적 차이를 만들어낸다.

요약하자면, 이산적인 DEM 데이터를 쌍선형 보간법으로 처리하는 메커니즘은 단순한 수학적 기법을 넘어 무인항공기의 궤적 추종 능력과 전력 효율, 모터 제어 안정성을 비약적으로 높이는 PX4-Autopilot 비행 제어기(Flight Controller)의 숨겨진 코어 기술이다.
