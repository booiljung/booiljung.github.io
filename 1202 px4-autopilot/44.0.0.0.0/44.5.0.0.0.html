44.5 Land Detector 모듈의 지면 이탈 판별 알고리즘 및 신호 처리(Signal Processing) 모델

멀티로터 및 고정익 무인항공기(UAV)의 이륙(Takeoff)과 착륙(Landing) 단계에서 기체가 지면(Ground)과 접촉하고 있는지, 혹은 완전히 지면을 이탈하여 체공(In-air) 상태인지 안정적으로 판별하는 것은 자율 비행(Autonomous Flight) 제어 기술의 핵심 필수 요구사항이다. 최신 PX4-Autopilot(Firmware v1.14 이상 기준)은 이를 위해 독립적인 데몬 프로세스인 Land Detector (land_detector) 모듈을 운영한다. 본 장에서는 Land Detector 모듈이 다양한 센서의 신호를 처리(Signal Processing)하여 지면 이탈 여부를 판별하는 알고리즘과 내부 로직, 그리고 Ardupilot 및 QGroundControl 환경에서의 관제 차이점을 학술적 관점에서 상세히 기술한다.

1. 지면 이탈 및 착륙 판별의 제어 공학적 필요성

시동(Arming) 직후 이륙을 시작하거나 착륙을 완료하여 모터를 정지(Disarming)하는 일련의 과정에서, 기체의 로터 추력(Thrust)과 지면 반발력(Ground Reaction Force) 간의 상호 작용으로 인해 심한 진동과 외란이 발생한다.
만약 비행 제어기(Flight Controller)가 지면 이탈 상태(In-air)를 너무 일찍 혹은 늦게 오판할 경우, 위치 제어기(Position Controller)의 I-항(Integral Term) 누적으로 인해 기체가 갑작스럽게 튀어 오르거나 이륙 직후 전복되는 사고(Ground Resonance or Tip-over)가 발생할 수 있다.
따라서 PX4 시스템은 단일 센서에 의존하지 않고, 관성 센서(IMU), 기압계(Barometer), 거리 측정기(Rangefinder) 등의 다중 데이터를 융합(Sensor Fusion)하여 강건한(Robust) 지면 이탈 판별 알고리즘을 수행한다.

2. 신호 처리(Signal Processing) 관점의 다중 센서 입력 모델

Land Detector 모듈은 실시간으로 uORB(Micro Object Request Broker) 메시지를 구독(Subscribe)하여 다층적인 저역 통과 필터(Low-pass Filter, LPF) 및 슈미트 트리거(Schmitt Trigger)를 통과시킴으로써 노이즈를 억제한다.

주된 판단 기준 파라미터는 다음과 같다.

  • 수직 속도 (Vertical Velocity, \dot{z}): 기압계와 GPS, 가속도계 데이터를 융합한 EKF(Extended Kalman Filter) 출력값(vehicle_local_position.vz).
  • 수직 가속도 (Vertical Acceleration, \ddot{z}): 기체 Z축 방향의 비가선형 가속도 성분.
  • 스로틀 명령 (Hover Throttle & Thrust Cmd): 현재 모터로 인가되는 Z축 추력 비율 명령(vehicle_local_position_setpoint).
  • 자세 제어 오차 (Attitude Error): 지면에 닿은 상태에서의 비대칭적인 저항으로 인한 롤/피치 자세 각도 오차 한계선.

2.1 저역 통과 필터(LPF) 기반 진동 제거

지면에 바운싱(Bouncing)될 때 발생하는 순간적인 충격 센서 데이터는 지면 이탈(In-air) 오판을 유발한다. 따라서 수직 속도와 가속도 데이터는 내부적으로 시상수(Time Constant)를 가지는 1차 저역 통과 필터(1st-order Low-pass Filter)를 거쳐 고주파 임펄스 노이즈가 스무딩(Smoothing) 처리된다.

3. Land Detector 상태 머신(State Machine) 아키텍처

PX4의 Land Detector는 단순히 ‘착륙’ 혹은 ’비행’의 이분법적 상태를 넘어 시간 이력(Hysteresis) 기반의 유한 상태 기계(FSM, Finite State Machine)를 가진다.

stateDiagram-v2
    [*] --> Ground_Contact : (시동 직후 지면 접촉 추정)
    Ground_Contact --> Maybe_Landed : 수직 / 수평 변위 감지 안됨 & 추력 임계치 이하
    Maybe_Landed --> Landed : 시간(Time Threshold) 충족 및 상태 유지
    Landed --> In_air : 추력 급증 & Z축 상승 속도 감지 (지면 이탈)
    In_air --> Ground_Contact : 하강 속도 0 근접 & 추력 하락 감지 (착륙 감지)
    In_air --> Freefall : 무중력 상태 (가속도 영점) 감지

위 전이(Transition) 과정에서 지면 이탈(In-air) 상황으로 판별되는 조건식은 대략 다음과 같은 논리 다항식으로 요약된다.

  1. 기체의 수직 위치 변동이 일정 임계치(예: LNDMC_Z_VEL_MAX)를 초과하거나,
  2. Z축 추력 명령어 합산값이 공중 호버링 스로틀(Hovering Throttle, MPC_THR_HOVER) 기준에 근접 및 초과할 경우.

4. PX4 파라미터 튜닝 및 Ardupilot 대비 아키텍처 비교표

기체 특성(무게, 이착륙 쿠션 등)에 맞게 튜닝하는 설정 파라미터는 안정적인 제어에 필수적이다.

파라미터명 (PX4 v1.14)기능 설명 및 신호 처리 관점의 역할
LNDMC_Z_VEL_MAX이 값을 초과하는 수직 속도(상하)가 감지되면 즉각 In_air 상태로 판별하기 위한 한계치 속도.
LNDMC_XY_VEL_MAX지면에서 미끄러짐을 배제하기 위해 허용되는 수평 속도의 최대 노이즈 폭 한계.
LNDMC_ALT_MAX착륙 시 기압계의 드리프트를 보조 필터링하기 위해 허용하는 최대 고도 이탈값 한계.

4.1 Ardupilot과의 관제 차이 (PX4 vs Ardupilot)

  • Ardupilot (Firmware v4.4): Ardupilot은 착륙 판별 로직(Crash check 및 Land validation)을 Copter::update_land_detector()와 같은 메인 비행 컨트롤 스레드 루프 안쪽에 무겁게 결합하여 처리하는 경향이 강하다. LAND_SPEED와 DISARM_DELAY 타이머 플래그를 조합하여 모터를 정지시킨다.
  • PX4-Autopilot: PX4는 uORB 아키텍처 철학에 맞게, Land Detector를 데몬(Daemon) 모듈인 별도 작업자 스레드(Worker Thread)로 완벽히 분리(src/modules/land_detector)하였다. 위치 제어기나 커맨더는 vehicle_land_detected 토픽 상태만을 비동기적으로 수신(Subscribe)받아 처리하므로 전체 비행 제어 루프의 실시간성(Real-time Constraints) 보장에 더 유리하다.

5. 소스코드 구현 로직 분석 (Source Code Analysis)

지면 이탈 및 착륙 판별의 핵심 로직은 src/modules/land_detector/MulticopterLandDetector.cpp의 내부 _get_landed_state() 함수를 통해 구현된다. (코드는 버전마다 상이할 수 있으나 핵심 제어 흐름은 아래와 같다)

// 멀티로터 지면 이탈 / 접촉 판별 임시 예제 (논리적 흐름)
bool MulticopterLandDetector::_get_ground_contact_state()
{
    // 1. Z축 제어 추력이 충분히 낮은가? (이륙 의지 없음)
    bool thrust_low = (_vehicle_local_position_setpoint.thrust[2] < _param_lndmc_thr_range.get());
    
    // 2. 수평 및 수직 가속도/속도가 임계치 이하(안정적인 상태)인가?
    bool hit_ground = (fabsf(_vehicle_local_position.vz) < _param_lndmc_z_vel_max.get());
    
    // 3. 필터링된 히스토리 상태를 바탕으로 최종 판별
    return thrust_low && hit_ground;
}

이러한 연산은 대략 50Hz (20ms 주기)의 빈도로 갱신되며, 로직 내부적으로 노이즈에의한 채터링(Chattering) 방지용 시간 이력 함수(Hysteresis)가 적용되어 true 상태가 최소 수백 밀리초 이상 연속 유지되어야만 실제 uORB 메시지로 확정 발송된다.

6. MAVLink 통신 및 QGroundControl 관제 차별성

Land Detector에서 판별한 기체 상태(Landed / In-air)는 커맨더(Commander) 모듈을 거쳐 MAVLink 프로토콜의 EXTENDED_SYS_STATE 메시지에 담겨 주기적으로 지상 관제 시스템(GCS)으로 전송된다.

  • MAVLink 필드: landed_state 필드는 MAV_LANDED_STATE_ON_GROUND, MAV_LANDED_STATE_IN_AIR, MAV_LANDED_STATE_TAKEOFF, MAV_LANDED_STATE_LANDING 의 상태 열거형 값을 가진다.
  • QGroundControl 구현 고려사항: QGroundControl 모바일 UI (QML)상에서는 위 상태 코드에 따라 비행 표시 HUD(Head-Up Display)의 고도계 색상이나 음성(Voice) 안내(“Takeoff detected”, “Landing detected”) 로직이 촉발된다. QGC 내부의 차량(Vehicle) 관리 매니저는 EXTENDED_SYS_STATE를 파싱하여 내부 상태 객체로 반영하고, 만일 이륙(Takeoff) 명령 전송 후 timeout 시간 내에 IN_AIR 상태가 확인되지 않으면 NACK을 띄우고 GCS 관제사에게 경고(Alarm)를 발생시킨다.

7. 결론

PX4-Autopilot의 Land Detector 모듈은 단순한 플래그 처리가 아니라, 드론 시스템 제어 공학의 최전선에서 동작하는 복합적인 센서 퓨전 및 신호 필터링(Signal Filtering) 모듈이다. 진동 노이즈 환경에서도 강건한 지면 이탈 판별 알고리즘을 수행하며, 수직 속도와 추력 한계치의 정밀한 튜닝은 드론의 안전한 자율 이착륙 성패를 좌우한다. 사용자 지정 GCS 애플리케이션 및 ROS2 미들웨어 환경을 구축하는 연구자라면 반드시 이 vehicle_land_detected 데이터 스트림의 지연 특성과 판별 주기를 깊이 고찰해야 할 것이다.
