44.3. 목표 고도(Target Altitude) 설정의 측지학적 모델링 및 제어 입력 기준

비행 제어 시스템에서 무인항공기(UAV)의 이륙(Takeoff), 경유지 이동(Waypoint Navigation), 모드 변경 시 가장 빈번하게 참조되는 파라미터 중 하나는 기체에 지시된 목표 고도(Target Altitude)이다. 단일 축의 1차원 수치로 보일 수 있으나, 전역 공간(Global Space) 네비게이션을 수행하는 PX4-Autopilot 아키텍처 내부에서는 이 대상 고도를 3차원 측지학(Geodesy) 맥락에서 해석하고 모델링하는 매우 복잡하고 정밀한 수학적 변환 과정이 진행된다.

본 절에서는 단방향 MAVLink 프로토콜을 통해 수신된 고도 명령 패킷이 PX4 내부에서 어떠한 측지학적 프레임(Geodetic Reference Frame)을 기준으로 변환되며, 위치 제어기(Position Controller)의 최종 제어 목표(Setpoint) 입력으로 인가되기까지의 역학적 모델링 및 기준 체계 모델을 분석하라. 본 내용은 최신 안정화 버전인 PX4 펌웨어 v1.14 구조를 기준으로 기술한다.

1. MAVLink 고도 파라미터의 다중 기준 체계 규격 (Altitude Reference Frames)

지상 관제 시스템(GCS, 예: QGroundControl)에서 PX4 기체로 목표 이동 고도를 설정하여 지시할 때, MAVLink 프로토콜과 시스템은 단순한 0점 기반의 단일한 ‘고도’ 개념을 사용하지 않고 다중 기준 체계(Multiple Reference Systems)를 프레임 형태로 할당하여 지원한다. 제어 스택에서 취급되는 대표적인 고도 프레임은 아래와 같다.

  1. AMSL (Above Mean Sea Level) - 평균 해수면 기준 고도: 지구 중력장 맵인 지오이드(Geoid, 예: EGM96) 표면 모델 상의 전 지구적 0점 해수면을 기준으로 측정한 절대 고도 체계이다. 주로 MAV_FRAME_GLOBAL 등의 전역(Global) 기반 명령 패킷에서 사용된다. 온보드 GNSS(GPS) 수신기 센서에서 출력되는 원시 데이터 초기 측정값은 WGS84 타원체 고도(Ellipsoid Height) 규격이나, PX4 내부 추정기(Estimator) 및 MAVLink는 이를 지오이드 기반의 공통 표준인 AMSL로 변환하여 취급하는 것을 시스템 표준으로 삼는다.
  2. AGL (Above Ground Level) - 이륙 지점 또는 지표면 기준 상대 고도: 홈 포인트(Home Position, 기체가 시동(Arming)을 걸었거나 전원 인가 시 지면에 위치했던 기준 지점)의 고도를 원점 0\text{ m} 로 설정하고, 그 기준 표면으로부터 상대적으로 상승한 높이를 의미한다. MAV_FRAME_GLOBAL_RELATIVE_ALT와 같이 자동 임무(Mission) 비행에서 실무적으로 가장 빈번히 채택되는 직관적인 제어 프레임이다. 지역(Local) 제어 모델인 NED 평면 직교 프레임에서는 -Z(Down축의 반대 방향인 상승 벡터) 형태인 상대적 타겟 고도로 1:1 매핑된다.
  3. Terrain Altitude - 지형 추종(Terrain Following) 고도: 기체 하단에 탑재된 하향 거리 파악 센서(Downward LiDAR, Radar, Sonar) 시스템이나, GCS 관제 프로그램으로부터 통신망 비동기 방식으로 지속 제공받는 디지털 표고 모델(DEM, Digital Elevation Model) 지리 데이터를 기반으로 산출한다. 이는 굴곡진 산맥이나 건물 등 현재 비행 중인 기체 바로 아래의 불규칙한 실제 지형 표면 자체를 실시간 기준(0\text{ m})으로 추종하는 특수 목적의 정밀 비행 제어 지형 고도 체계이다.

2. 목표 고도 변환의 수학적 모델링 프로세스

PX4 시스템의 궤적 생성 모듈인 Navigator 내부 런타임에서, 외부 데이터인 MAVLink를 통해 들어온 타겟 고도 값(h_{cmd})은 다음과 같은 측지학적 수학적 상쇄 모델 변환 수식을 통해 시스템 전체에 일원화된 로컬 NED 제어 고도 목표값(Z_{sp}, Local Z Setpoint)으로 변환(Translation)된다.

2.1 상대 고도(Relative Altitude)의 국지 좌표계(Local NED) 직교 매핑

가장 빈번하게 사용되는 상대 고도 체계인 AGL 기반 이동 지시를 받았을 경우, 기체는 비가휘발성 또는 휘발성 플래그 메모리에 저장된 홈 위치(Home Position)의 절대 수직 위치 H_{home} (\text{AMSL})을 기준 기저 공간으로 둔 채로 연동 계산을 수행한다.

Z_{sp} = -(h_{cmd})

여기서 Local NED 프레임의 수학적 정의는 Z축 역방향이 지구 중심부(Down)를 향하므로, 기체의 위쪽(Up) 방향의 상대 타겟 상승 고도인 양수 수치값(+h_{cmd})을 단순히 부호가 반전된 음수로 직관적 변환하는 단일 스케일 맵핑 투영을 수행한다.

2.2 절대 고도(AMSL) 명령의 내부 참조점 기반 정역학적 오프셋 상쇄

목표 고도 명령이 절대 고도 프레임 규격(h_{AMSL\_cmd})으로 입력되었을 경우, PX4 내부 네비게이터는 이 절대적인 거대 수치(예: 2000.5\text{ m})를 그대로 위치 제어기의 미분 방정식 변수 항으로 무리하게 밀어넣지 않는다. 대신, 현재 활성화된 로컬 직교 원점을 기준으로 다음과 같은 환산 및 추적 보정 상쇄 과정을 항상 거친다.

Z_{sp} = -(h_{AMSL\_cmd} - H_{local\_origin\_AMSL})

여기서 H_{local\_origin\_AMSL}은 EKF2(Extended Kalman Filter 2) 상태 추정 모듈이 센서 퓨전을 통한 칼만 필터 초기화(Initialization) 과정 중에서 융합 및 수렴시켜 확정한 국지 프레임(Local Frame) 원점 자체의 절대(AMSL) 고도이다. 이 설계 아키텍처를 통해 지구 곡률과 방대한 광역 지형 편차 수치가 정해진 국지 기준 원점 안에서 정역학적 오프셋 상쇄(Static Offset Cancellation) 효과를 누리게 된다. 이는 제어 루프 시스템이 수천 단위의 거대한 숫자를 내부에서 지속적으로 연산함으로 인해 발생하는 부동소수점 정밀도 하락 오차(Floating-point Precision Loss)와 양자화 오류를 원천 차단하여, 센티미터 단위의 오차 없이 매우 정밀하게 타겟 위치를 추적할 수 있도록 하는 최우선 수학 모델링 기법이다.

3. 고도 제어 입력으로의 변환 아키텍처 다이어그램 및 시각화

지시된 목표 고도가 어떠한 추상화된 프레임 구조로 유입되든, 최종적으로 시스템 스택 구조 내에서는 속도 미분 제어기능이 코어단에 내포된 다중 피드백 루프(Cascaded Feedback Loop) 구조의 위치 제어기에 오차 평가 함수(Error Evaluation Function) 형태로 정밀도 있게 인가된다. 이를 객체 모델링한 데이터 처리 파이프라인 흐름 구조도를 다음과 같이 명확히 시각화하여 분석하라.

이 다이어그램 아키텍처는 최상위의 측지학적 추상 레이어의 고도(Geodetic Altitude)가 어떠한 정변환 산출 과정을 거쳐 물리적 속도 벡터(Velocity Vector) 및 인가 추력 전기 신호(Thrust PWM)라는 최하위 동역학 변수로 점진적으로 파고들어(Cascade) 분해 전송되는지를 기술적으로 가장 자명하게 나타낸다.

4. 실무 제언 (Conclusion and Engineering Implications)

서드 파티 GCS 플러그인 또는 온보드(Onboard) 동반 컴퓨터(Companion Computer, 예: ROS2 기반 비전 경로 노드) 모듈을 활용하여 무인기의 목표 고도를 동적으로 지시하거나 제어하고자 할 때, 로보틱스 엔지니어는 기체 이상 행동 방지를 위해 다음의 설계 변환 매핑 원칙을 절대적으로 준수하라.

  1. 상대 고도(Relative) 체계의 최우선 활용 원칙: 절대 고도 체계 제어 명령(AMSL) 방식은 지오이드 굴곡 테이블 룩업(Geoid lookup) 과정에서 GNSS 다중경로(Multipath) 오차나 통신 기기의 수렴 부적절에 의한 초기화 오차(H_{local\_origin\_AMSL} 결정 오류) 지연 문제가 생길 확률이 농후하다. 이러한 경우 실제 고도는 정상이나 무인기 스스로가 10\sim30 \text{ m} 가량이 어긋났다고 판단하여 지면에 강하게 충돌하거나 솟아오르는 치명적인 이륙 위치 편차를 발생시킬 수 있다. 따라서 GCS 애플리케이션 개발 시, 전역 임무 프로파일의 3차원 궤적(Trajectory) 모델을 수립할 때 특수 지형 연동 상황(험준한 산맥 지형 간섭, 골짜기 포물선 비행 등)이 아니라면 항상 시스템의 기준 원점을 잡는 MAV_FRAME_GLOBAL_RELATIVE_ALT를 기준 제어 데이터 프레임 디폴트(Default) 규격으로 일관되게 확보하라.
  2. 동적 오프셋 캘리브레이션 유의점: PX4는 EKF2의 정밀한 지속 추정을 위해 위치가 유지되는 상태라면 로컬 원점(H_{local\_origin\_AMSL})을 비행 초기에 지속적으로 갱신(Continual Update)할 권한을 갖는다. ROS2 미들웨어나 통신 브리지 인터페이스인 MAVSDK를 통해 스크립트를 작성하여 제어할 경우, 하드코딩된 단 한 번의 상대 고도 측정치에 절대 의존하지 말고 드론에서 지속적으로 송출되는 고주파수 상태 VEHICLE_LOCAL_POSITION 등의 uORB/MAVLink 구조체 스트림을 서브스크라이빙(Subscribing)하여 동적인 피드백 제어 루프 주기를 유지해야 안전장치(Failsafe) 록인 현상을 회피할 수 있다.
