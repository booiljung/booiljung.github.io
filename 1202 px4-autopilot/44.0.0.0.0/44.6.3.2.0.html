44.6.3.2. 1차 지연 시스템(First-order Lag System) 응답 모델 기반 수직 속도 감쇠(Damping) 비율의 시상수(Time Constant) 분석 및 체류 시간(Dwell Time) 확인

자율 제어 시스템에서 기체가 목표 고도에 기하학적으로 진입했다고 해서 즉시 상태 전환(State Transition)을 수행하는 것은 제어 공학적으로 매우 위험한 접근이다. 이는 센서의 백색 잡음(White Noise)이나 위치 제어기의 오버슈트(Overshoot)로 인해 일시적으로 오차 허용 구역 내에 진입했다가 다시 이탈할 가능성이 존재하기 때문이다. PX4-Autopilot은 이를 방지하기 위해 위치 오차뿐만 아니라 **수직 속도(Vertical Velocity)의 감쇠(Damping)**와 특정 시간 동안의 **체류 시간(Dwell Time)**을 함께 검증한다. 본 절에서는 멀티로터 시스템의 고도 제어가 1차 지연 시스템(First-order Lag System) 특성을 가짐을 분석하고, 수직 속도가 어떻게 감쇠하며 체류 시간이 계산되는지를 수학적으로 규명한다.

1. 멀티로터 고도 제어의 1차 지연 시스템 모델

멀티로터의 Z축 제어기(PX4 모듈 내 mc_pos_control)는 위치 오차를 입력으로 받아 속도 지령(Velocity Setpoint)을 생성하고, 속도 제어 루프를 통해 최종 추력(Thrust)을 제어하는 캐스케이드(Cascaded) 제어 구조를 갖는다. 비행 기체가 목표 고도 z_{ref} 에 점근적으로 접근할 때, 위치 제어기의 P-Gain (K_p)에 의해 유도되는 폐루프(Closed-loop) 시스템의 수직 거동은 1차 지연 시스템으로 근사화될 수 있다.

수직 고도 위치 z(t) 제어의 미분 방정식은 속도 제어 응답이 기구학적 한계를 초과하지 않는다는 가정하에 다음과 같이 나타낼 수 있다.
\dot{z}(t) + K_p z(t) = K_p z_{ref}

이때 1차 지연 시스템의 핵심 파라미터인 시상수(Time Constant) \tau 는 \frac{1}{K_p} 로 정의된다. 시상수는 기체가 수직 속도를 줄이며 목표 고도에 안착하기까지의 감쇠(Damping) 특성을 결정환다.

기체의 수직 위치 거동응답은 다음과 같다.
z(t) = z_{ref} + (z_0 - z_{ref}) e^{-\frac{t}{\tau}}

이에 따른 수직 속도 V_z(t) = \dot{z}(t) 의 감쇠 응답(Damping Response)은 시상수 \tau의 역수에 비례하여 지수적으로 감소한다.
V_z(t) = -\frac{z_0 - z_{ref}}{\tau} e^{-\frac{t}{\tau}}

즉, PX4의 내비게이터(Navigator)는 유클리드 노름 상으로 고도 반경에 도달했음을 확인한 후, 수직 속도 V_z 가 시스템 노이즈 마진(Noise Margin) 이하로 완전히 감쇠하여 V_z \approx 0 에 도달하는지 검증하는 동역학적 판단 거친다.

2. 체류 시간(Dwell Time)의 설정과 제어적 의미

수직 속도의 지수적 감쇠는 수학적으로 t \to \infty 일 때 완전히 0에 수렴하므로, 실무적으로는 허용 가능한 최소 속도 임계값(Velocity Threshold) 이하로 떨어지는 일정 시간을 정의해야 한다. 이것이 바로 **체류 시간(Dwell Time)**이다.

기체가 오차 허용 구역(Acceptance Volume) 내로 진입한 시점을 t_{entry} 라고 할 때, 다음 상태로 안정적으로 넘어가기 위해서는 t_{entry} 로부터 특정 지연 시간 T_{dwell} 만큼 그 상태를 온전히 유지해야 한다.

T_{dwell} \ge n \cdot \tau \quad (\text{단, } n \text{은 } 3 \sim 5 \text{ 범위의 튜닝 상수})

일반적인 1차 지연 시스템에서 3\tau 는 약 95%, 5\tau 는 약 99.3%의 수렴을 의미하므로, 이 시간만큼 대기하면 오버슈트로 허용 반경을 다시 이탈할 확률이 극히 낮아진다.

2. PX4 내비게이터(Navigator)의 속도 및 시간 검증 로직 흐름

PX4-Autopilot의 내부 아키텍처에서 이 체류 시간 검증과 수직 속도 감쇠 검증은 상태 머신 타이머(Timer)를 통해 이루어진다.

stateDiagram-v2
    state "고도 상승 중 (Climbing)" as Climb
    state "허용 구역 진입 (Acceptance Volume Entered)" as Entered
    state "체류 시간 대기 (Dwell Timer Active)" as Dwelling
    state "고도 도달 인가 (Altitude Reached)" as Reached

    Climb --> Entered : ||e_xyz|| <= R_accept
    Entered --> Dwelling : t_entry 기록, 타이머 시작
    Dwelling --> Climb : ||e_xyz|| > R_accept (오버슈트 발생)
    Dwelling --> Dwelling : V_z 감쇠율 모니터링
    Dwelling --> Reached : (t - t_entry) >= T_dwell AND |V_z| <= V_thresh
    Reached --> [*] : Hold / Mission 상태로 FSM 전환

위 다이어그램에서 보듯 체류 시간 카운트 도중 위치 오차가 커지면(허용 구역 이탈) 타이머는 리셋(Reset)되며 기체는 다시 고도 추종 루프로 돌아간다.

4. 지상 관제 시스템(GCS) UI 관점에서의 교훈

지상 관제 시스템인 QGroundControl(QGC, v4.3.x) 또는 자체 개발 GCS를 설계할 때, 고도 텔레메트리(altitude) 정보만을 보고 선반영하여 조종사에게 “이륙 완료” 상태를 알리면 위험한 UI 불일치(Mismatch)가 발생한다. 펌웨어는 1차 지연 응답에 의한 체류 시간(T_{dwell})이 지나고 수직 속도(V_z)가 감쇠될 때까지는 여전히 이륙(Takeoff) 모드를 유지하고 있기 때문이다. 따라서 GCS의 프론트엔드 비동기 렌더링에서는 펌웨어가 전송하는 MAVLink 메시지 중 최종 상태(Vehicle Status / Extended Sys State)의 변경 확인 시그널만을 유일한 소스로 간주하여 UI 동기화를 수행해야 한다.
