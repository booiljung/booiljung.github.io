44.3.1.3. 수치 표고 모델(DEM) 데이터 기반 지형 추종(Terrain Following) 고도 기준점 설정

1. 개요: 지형 추종(Terrain Following) 제어의 필요성

무인항공기 시스템(UAS)이 평탄한 지형을 비행할 때는 이륙 지점의 고도를 기준(HOME 가상 평면)으로 삼는 상대 고도(Relative Altitude) 제어만으로도 충돌 회피와 목적지 도달이 가능하다. 하지만 산악지대, 계곡, 혹은 경사면과 같이 고저차가 심한 지형을 비행하는 경우, 단순히 이륙 지점을 기준으로 비행하면 산비탈에 충돌하거나 지면과 지나치게 멀어져 임무 장비(예: 고해상도 카메라, 라이다 등)의 데이터 품질이 저하되는 현상이 발생할 수 있다.

이를 방지하기 위해 드론이 지표면의 굴곡을 따라 일정한 상대 고도를 유지하며 비행하는 기술을 지형 추종(Terrain Following) 이라고 한다. PX4-Autopilot(Firmware 버전 v1.14 기준)은 하방 거리 측정 센서(LiDAR 등)를 사용할 뿐만 아니라, 장거리 비행 시 센서의 한계를 극복하기 위해 수치 표고 모델(DEM: Digital Elevation Model) 을 활용하여 전역적인 고도 기준점을 동적으로 설정하는 고도화된 아키텍처를 지원한다.

2. 수치 표고 모델(DEM)의 구조와 데이터 소스

수치 표고 모델(DEM)은 지구 표면의 지형 높이를 일정한 격자(Grid) 간격으로 샘플링하여 3차원 공간 좌표로 표현한 디지털 데이터베이스이다. 지형 지물(건축물, 식생 등)을 제외하고 순수한 흙과 바위의 표면 높이만을 나타내는 수치 지형 모델(DTM: Digital Terrain Model)의 일종으로 주로 사용된다.

일반적으로 QGroundControl(버전 v4.3.0 이상)과 같은 지상 관제 시스템(GCS)은 인터넷을 통해 SRTM(Shuttle Radar Topography Mission) 모델이나 Mapbox, AGI STK Terrain Server 등의 외부 DEM 공급자로부터 지형 데이터를 가져온다.

  • SRTM 해상도: 전 세계적으로 1 아크초(Arc-second, 약 30m 해상도) 또는 3 아크초(약 90m 해상도) 타일을 사용한다.
  • GCS는 사용자가 비행 임무(Mission)를 계획하는 영역에 해당하는 폴리곤(Polygon) 내의 모든 DEM 타일을 다운로드하여 내부적으로 캐싱(Caching)한다.

3. PX4와 QGroundControl 간의 DEM 데이터 스트리밍 아키텍처

비행 제어 컴퓨터(Flight Controller)의 메모리와 저장 공간은 제한적이므로, 전 세계의 고해상도 DEM 데이터를 내부에 보관할 수 없다. 따라서 PX4-Autopilot은 비행 중 필요한 영역의 지형 데이터만을 GCS에 요청하는 동적 요청/응답(Dynamic Request/Response) 아키텍처를 사용한다.

이 통신은 MAVLink 프로토콜의 TERRAIN_REQUEST, TERRAIN_DATA, TERRAIN_REPORT 메시지를 통해 이루어진다.

sequenceDiagram
    participant PX4 as PX4-Autopilot (Terrain Module)
    participant MAVLink as MAVLink Interface
    participant QGC as QGroundControl (GCS)
    
    PX4->>MAVLink: 예상 비행 경로의 그리드(Grid) 정보 계산
    MAVLink->>QGC: TERRAIN_REQUEST (Lat, Lon, Grid Spacing) 전송
    Note over QGC: 로컬 DEM 캐시 또는<br/>온라인 데이터베이스 조회
    QGC->>MAVLink: TERRAIN_DATA (16개의 고도 값 배열) 전송
    MAVLink->>PX4: uORB 'vehicle_trajectory_waypoint' 등 업데이트
    PX4->>MAVLink: TERRAIN_REPORT 주기적 발행 (현재 지형 고도)
  1. 지형 데이터 요청(TERRAIN_REQUEST): PX4 내부의 Terrain Estimator 모듈은 현재 위치 또는 향후 도달해야 할 웨이포인트(Waypoint) 좌표가 내부 캐시에 없는 경우, MAVLink 네트워크를 통해 GCS에 타일 데이터를 요청한다.
  2. 지형 데이터 응답(TERRAIN_DATA): QGC는 요청받은 4x4 크기(총 16개 지점)의 지형 고도 배열을 패킷에 담아 응답한다.
  3. uORB 퍼블리싱: 수신된 데이터는 PX4 내부의 uORB 메시징 버스에서 vehicle_local_position과 융합되어, 위치 제어기(Position Controller)가 접근할 수 있는 형태의 평형 기준점으로 변환된다.

4. 위치 제어기(Position Controller)의 지형 추종 알고리즘 (Firmware 소스 코드 이해)

PX4의 다중 루프(Cascaded Loop) 위치 제어 시스템 속에서, 목표 고도 점(Setpoint)의 생성 방식이 지형 추종 모드에 의해 크게 달라진다.

수학적으로, t 시간에서 기체의 고도 설정점 Z_{sp}(t)는 아래와 같이 계산된다.

Z_{sp}(t) = Z_{terrain}(Lat(t), Lon(t)) + h_{target}

  • Z_{terrain}: 현재 위도 및 경도 좌표에서 DEM 데이터로 추정된 절대 해발 고도(AMSL Height).
  • h_{target}: 사용자가 지정하여 유지하고자 하는 지면 대비 목표 상대 고도(Target AGL, Above Ground Level).

이륙 및 비행 제어 루프에서의 활용

일반적인 이륙(Takeoff) 명령이 HOME 위치의 기압 고도를 0으로 간주하는 것과 달리, 지형 추종이 활성화된 미션(Mission) 혹은 위치 제어 모드에서는 EKF2에서 제공하는 전역 좌표계 위경도를 바탕으로 Terrain 모듈이 지속적으로 Z_{terrain}을 보간(Interpolation)하여 위치 제어기에 피드포워드(Feedforward) 값을 인가한다. 이를 통해 산등성이를 넘을 때 드론은 상승 피치(Pitch)와 추력(Thrust)을 증가시키는 제어 포화 방어 로직을 역동적으로 구동하게 된다.

5. 센서 융합의 상호 보완성: DEM과 LiDAR

거리 측정 센서(LiDAR) 하단 장착만으로 지형 추종을 구현하는 데에는 한계가 있다.

  • LiDAR는 최대 측정 거리(보통 수 십 미터)가 제한되어 있으며, 울창한 숲(Canopy)을 비행할 경우 나무 꼭대기를 지면으로 오인하여 드론이 급상승하는 팝업(Pop-up) 현상을 유발할 위험이 높다.
  • PX4는 EKF2_TERR_MASK 등을 통해 장기적인 지형 추세(Low-frequency trend) 는 DEM 데이터를 기반으로 추종하고, 단기적이고 근접한 장애물 회피(High-frequency trend) 는 LiDAR로 보완하는 혼합 방식을 권장힌다. (특정 조건 하에서 두 데이터를 융합하는 Weighting 파라미터가 적용된다.)

6. PX4와 Ardupilot 간 동작 원리(Architecture) 비교 차이

PX4와 Ardupilot(Mission Planner 기반)은 모두 DEM 데이터에 의한 지형 추종 기능을 훌륭히 지원하지만, 내부 시스템 아키텍처 관점에서는 차이를 보인다.

  • PX4-Autopilot: PX4는 상대적으로 ’타겟 모션 제너레이터(Target Motion Generator)’와 실시간 제어 루프 간의 매끄러운 궤적 생성력에 집중한다. 비행 궤적이 지형 데이터를 기반으로 스무딩(Smoothing)되며, MAVLink 스트리밍 시 통신 끊김(Data Link Loss)이 발생하면 지형 정보에 기반한 비행 모드가 즉각 유지보수(Hold) 모드로 격하되는 등 보수적인 안전 장치(Failsafe)를 선호한다.
  • Ardupilot: Ardupilot은 GCS상에서 지형 데이터를 훨씬 선제적으로 펌웨어 SD 카드에 삽입(Injection)하는 것을 권장하며, GCS 연결이 끊기더라도 SD 카드의 /TERRAIN 폴더를 참조해 독자적으로 자율 비행을 계속 진행하는 성향이 강하다. 이는 Ardupilot이 보다 임무 연속성에 중점을 둔 관제 철학을 가지고 있기 때문이다.

이러한 DEM 기반의 지형 추종 아키텍처는 농업용 드론 방제, 3D 매핑(Mapping), 정찰용 무인항공기의 필수 요소로 작용하며, 고속으로 변화하는 지형 환경에서도 기체의 물리적 충돌 한계(Limit)를 지정하는 제어의 근간이 된다.
