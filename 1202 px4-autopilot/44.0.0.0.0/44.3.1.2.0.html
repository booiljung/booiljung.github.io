44.3.1.2 측고 공식(Hypsometric Equation)을 적용한 기압-고도 변환 및 정역학적 평형(Hydrostatic Equilibrium) 상태의 열역학적 편차(Drift) 보정

1. 개요 및 설계 철학

PX4-Autopilot(펌웨어 v1.14 기준)을 비롯한 무인항공기 시스템(UAS)에서 비행 고도의 정밀한 측정과 제어는 자율 비행의 필수 요소이다. PX4는 고도 상태 추정(State Estimation)을 위해 기압계(Barometer), GPS, IMU, 그리고 광학식 거리 측정 센서(Range Finder) 데이터를 EKF2(Extended Kalman Filter 2) 아키텍처를 통해 융합(Sensor Fusion)한다.

이러한 센서 융합 중에서도 기압계는 대기압 데이터를 기반으로 고도를 추정하는 핵심 센서이다. 기압을 측정하여 고도를 역산하기 위해서는 측고 공식(Hypsometric Equation) 을 적용해야 하며, 이 공식은 대기가 정역학적 평형(Hydrostatic Equilibrium) 상태에 있다는 이상적인 가정을 바탕으로 성립된다. 그러나 실제 비행 환경에서는 센서 내부의 발열, 급격한 온도 강하, 그리고 기류의 변화로 인해 기압 데이터와 실제 고도 간에 피할 수 없는 열역학적 편차(Thermodynamic Drift) 가 발생한다. 본 절에서는 기압과 고도 간의 수학적 변환 원리를 이해하고, PX4 시스템이 이러한 편차를 실시간으로 어떻게 보정하는지 심층적으로 탐구한다.

2. 정역학적 평형 (Hydrostatic Equilibrium) 구조

대기 모델의 수학적 접근은 공기가 정역학적 평형을 유지한다는 대전제에서 출발한다. 정역학적 평형이란 대기의 수직 방향으로 압력 기울기에 의한 힘(Pressure Gradient Force)과 중력(Gravity)이 균형을 이루어 공기 덩어리에 가속도가 발생하지 않는 상태를 의미한다.

정역학 방정식(Hydrostatic Equation)은 다음과 같이 정의된다.

\frac{dP}{dz} = -\rho g

여기서:

  • P: 대기압 (Pressure)
  • z: 고도 (Altitude)
  • \rho: 공기의 밀도 (Density)
  • g: 중력 가속도 (Gravity Acceleration, 상수로 취급 9.80665 m/s^2)

이 수식은 위로 올라갈수록(고도가 증가할수록) 기압이 감소함을 명확하게 보여준다.

3. 측고 공식 (Hypsometric Equation)을 통한 고도 변환

이상기체 상태 방정식(Ideal Gas Law)인 P = \rho R T (여기서 R은 기체 상수, T는 절대 온도)를 정역학 방정식에 대입하여 \rho를 소거하면 다음과 같은 미분 형태가 도출된다.

\frac{dP}{P} = -\frac{g}{RT} dz

이 미분 방정식을 기준면(기준 기압 P_0, 기준 고도 z_0)에서 측정 고도(측정 기압 P, 목표 고도 z)까지 적분하면, 기압으로부터 두 지점 간의 고도 차이를 구하는 측고 공식(Hypsometric Equation) 이 유도된다.

z - z_0 = \frac{R \cdot \bar{T}}{g} \ln \left( \frac{P_0}{P} \right)

  • \bar{T}: 두 고도 계층 간의 가상 평균 온도(Virtual Temperature)
  • P_0: 기준면의 기압(통상적으로 이륙 시점의 기압 또는 표준 해수면 기압 QNH)

PX4-Autopilot의 소스 코드 내 센서 모듈(Sensor Module)에서는 초기 이륙 시점의 기초 기압(Reference Pressure)과 온도를 샘플링하여, 입력되는 기압 데이터를 위 공식을 통해 z(상대 고도)로 연속적으로 변환한다.

4. 열역학적 편차 (Thermodynamic Drift) 발생 원인

이론적인 측고 공식은 특정 시점의 온도가 일정하고, 대기가 완전히 정역학적 평형 상태라는 매우 제한적인 가정을 기반으로 한다. 실제 환경에서는 여러 요인에 의해 심각한 고도 편차(Drift) 가 유발된다.

  1. 상대성 센서 발열 인자: 기판(Pixhawk 등)에 전원이 인가되면 프로세서 및 전원 모듈에서 발생하는 열로 인해 기압계 센서 자체의 측정 온도가 상승한다. 외부 대기 온도는 일정하더라도 내부 센서 온도의 왜곡은 공식 내 T 값을 왜곡하여 점진적인 고도 상승 혹은 하락으로 잘못 인식되는 현상(Drift)을 유발한다.
  2. 외기 온도 감률(Lapse Rate): 기체가 고도를 높임에 따라 대기 온도는 표준 대기 모델 기준 1000m 당 약 6.5^\circ\text{C}씩 하강한다. 이러한 온도구배(Temperature Gradient)를 실시간으로 완벽히 측정하여 수식에 반영하는 것은 단일 기압계의 한계 상 불가능하다.
  3. 동적 압력(Dynamic Pressure) 간섭: 기체의 빠른 이동 기류나 멀티로터의 프로펠러 후류(Propwash)가 기압계의 정압 포트(Static Port)에 국소적인 기압 변화를 일으킨다. 이를 베르누이 효과(Bernoulli Effect)에 의한 정압 하강이라고 하며, 이는 기체가 실제보다 더 높은 고도에 있다고 착각하게 만든다.

5. PX4-Autopilot의 열역학적 편차 보정 아키텍처

PX4는 이러한 오류 모델을 소프트웨어 아키텍처 수준에서 두 가지 단계별(Cascaded) 방식으로 보정한다. 바로 오프라인 온도 캘리브레이션과 EKF2 온라인 상태 추정(Online State Estimation) 이다.

5.1 오프라인 기압계 온도 보상 (Offline Barometer Temperature Compensation)

열에 의한 센서 기계적 변형이나 전자적 오차를 줄이기 위해, 시스템은 공장 초기화 혹은 유지보수 단계에서 열 챔버(Thermal Chamber) 혹은 자체 발열 사이클을 이용하여 보정을 수행한다.

  • T 온도에 따른 압력 변화 \Delta P(T)를 기록하여, 3차 또는 5차 다항식(Polynomial curve)으로 커브 피팅(Curve-fitting)을 수행한다.
  • 산출된 다항식의 계수는 파라미터 트리(Parameter Tree) 내의 CAL_BARO* 관련 파라미터 트리에 비휘발성으로 저장된다.
  • 작동 중 센서 파이프라인(Sensor Pipeline)은 로우 데이터(Raw Data)를 받을 때 가장 먼저 파라미터를 읽어들여 1차 온도 보상을 완료한 뒤에 uORB 토픽(Topic)으로 통신망에 퍼블리싱(Publishing)한다.

5.2 EKF2 관측 모델을 통한 온라인 실시간 융합 보정

기압 센서 단일 데이터로는 물리적 대기의 기압 배치 변화(예: 비행 중 날씨 변화로 인한 저기압 전선 통과 등)나 기상 편차를 예측하고 방어할 수 없다. 이를 위해 PX4의 내부 상태 추정기인 EKF2(Extended Kalman Filter 2) 는 기압을 직접적인 고도 값(Absolute Truth)으로 다루지 않고, 동적 바이어스(Bias)가 껴 있는 상태 모델(State Model)로 다룬다.

EKF2 알고리즘 내에서 기압 측정 모델의 상태 벡터에는 기압 바이어스 상태(Barometer Bias State) 가 할당된다.

  • 필터는 IMU 가속도계의 고주파 동특성과 GPS 및 기압계의 저주파 응답특성을 융합한다.
  • GPS나 Optical Flow의 고도 데이터가 가용할 때, 시스템은 기압 고도의 점진적 변화(Drift)인 ‘이노베이션(Innovation, 예측치와 측정치의 차이)’ 잔차를 지속 모니터링한다.
  • 만약 GPS 고도는 변함이 없는데 기압 고도만 점점 내려간다면(기압이 상승), EKF2 내부 파라미터(EKF2_BARO_NOISE, EKF2_BARO_GATE) 설정에 따라 기압 바이어스 추정치를 지속적으로 업데이트 하여 관측치의 드리프트를 실시간 보상한다.

6. Ardupilot과 PX4 시스템 간 보정 설계 철학 차이

Ardupilot 역시 Extended Kalman Filter(현재 EKF3 구조)를 통해 위와 같은 모델링을 지원한다. 하지만 구체적인 파라미터 구조와 관제 차이에서 다른 철학을 지닌다.

  • 파라미터 중심 vs EKF 중심: Ardupilot은 GND_TEMP나 GND_ABS_PRESS 같은 기준점 파라미터를 사용자가 직접 명시적으로 제어할 여지를 많이 열어두며, GCS 상(Mission Planner)에서 즉각적인 기압 보정을 명령하도록 유도하는 경향이 있다.
  • 자율 융합(Autonomous Fusion) 기반: 반면, PX4는 하드웨어 추상화 계층 위에서 EKF2가 블랙박스 형태로 기압 편차를 통합하여 추정하도록 아키텍처링 되어 있다. 개발자는 EKF2_HGT_REF 파라미터를 통해 고도의 1차 참조 소스(기압계, GPS, 거리측정기 등)만을 선택하며, 세부 바이어스 조작보다 센서 간 가중치 신뢰도(Sensor Noise Parameters) 조작을 통해 간접적으로 EKF2 모델 가중치를 조정하도록 시스템 설계가 강제되어 있다.

이는 PX4가 보다 자동화되고 상태 추정기의 수학적 건전성(Mathematical Robustness)에 더욱 의존하는 현대적인 로보틱스 프레임워크 설계 패턴을 따름을 보여주는 대목이다.
