44.1.2.1.1. 관측 갱신(Observation Update) 과정의 혁신(Innovation) 시퀀스에 대한 마할라노비스 거리(Mahalanobis Distance) 기반 일관성 확인

PX4-Autopilot의 확장 칼만 필터(Extended Kalman Filter, EKF2) 모듈은 다중 센서 생태계에서 들어오는 관측 데이터(Observation Data)를 융합하여 기체의 상태 추정(State Estimation)을 수행한다. 이 때, 무인기(UAV)의 신뢰성 있는 자율 비행을 보장하기 위해 QGroundControl(이하 QGC, 기준 테스트 버전 v4.3.0)과 PX4 펌웨어(기준 테스트 버전 v1.14.0)는 외부 센서 데이터(GPS, 기압계, 지자기 센서, 비전 센서 등)가 사전에 구축된 관성항법장치(Inertial Navigation System, INS) 기반의 예측 모델에 통합되기 적합한지 판단하는 엄격한 일관성 검증(Consistency Check) 과정을 거친다.

이를 위해 PX4 시스템은 다차원 데이터의 통계적 오차 한계를 정밀하게 평가할 수 있는 마할라노비스 거리(Mahalanobis Distance)를 활용하여 혁신(Innovation) 테스트를 수행한다. 이 검증 로직은 이륙 가능 여부를 GCS(Ground Control Station) 인 QGC에서 최종 승인하는 비행 전 점검(Pre-flight Check)의 핵심 판별 지표가 된다.

1. 혁신(Innovation)의 개념과 마할라노비스 거리(Mahalanobis Distance)

칼만 필터 이론에서 ‘혁신(Innovation)’ 혹은 ’잔차(Residual)’는 센서로부터 물리적으로 획득한 실제 측정값(Measurement)과, 필터가 이전 상태 및 동역학 모델을 기반으로 수학적으로 예측한 측정값(Predicted Measurement) 간의 차이를 의미한다.

만약 단순한 유클리디안 거리(Euclidean Distance)를 이용하여 이러한 잔차의 크기를 평가할 경우, 개별 센서가 가지는 고유의 노이즈(Noise) 특성과 축(Axis) 간의 상관관계(Correlation) 오차를 전혀 반영하지 못하게 된다. 따라서 PX4-Autopilot의 EKF2는 다음 수식과 같이 혁신 공분산 행렬(Innovation Covariance Matrix) 레이어를 거쳐 잔차를 통계적으로 정규화하는 마할라노비스 거리를 채택한다.

혁신 벡터(Innovation Vector)를 y, 혁신 공분산 행렬(Innovation Covariance Matrix)을 S라고 정의할 때, 마할라노비스 거리의 제곱(D^2)은 다음과 같이 계산된다.

D^2 = y^T S^{-1} y

여기서 1차원 데이터(예: 기압계 기반 단일 고도 센서치)를 융합하는 경우, 상관관계가 배제되므로 위 식은 단순화되어 y^2 / s 형태로 계산된다. PX4 EKF2 모듈은 산출된 D^2 값이 사전에 시스템 파라미터로 정의된 개별 센서의 게이트 임계값(Gate Size, 예를 들어 EKF2_GPS_GATE, EKF2_MAG_GATE 등)보다 작을 때만 해당 측정치를 ’일관성이 있다(Consistent)’고 판정하며 필터의 관측 갱신(Observation Update)을 허가한다.

2. PX4-Autopilot EKF2 모듈의 일관성 확인 아키텍처

PX4 펌웨어 소스 코드(src/modules/ekf2) 내부 구조를 면밀히 분석하면, GPS, 기압계(Baro), 나침반(Mag) 등의 센서 데이터를 융합하기 직전에 fuse()라는 구체적 연산 함수군(예: fuseVelPosHeight(), fuseMag())이 호출됨을 확인할 수 있다. 이 함수 단계에서 마할라노비스 거리를 연산하며, 통칭 게이트 테스트(Gate Test)라 불리는 해당 검증 로직을 최우선적으로 실행한다.

이 아키텍처를 시각화하면 다음과 같다.

graph TD;
    A[센서 관측치 수신: GPS/Mag/Baro 등] --> B[상태 예측 모델 결과값과 비교하여 혁신 벡터 y 산출];
    B --> C[시스템 노이즈 모델 기반 혁신 공분산 S 연산];
    C --> D[마할라노비스 거리 산출 및 Test Ratio 계산];
    D --> E{Test Ratio < 1.0 (일관성 통과 여부)?};
    E -- Pass (수락) --> F[칼만 이득(Kalman Gain) 적용 및 EKF 상태 갱신];
    E -- Fail (거절) --> G[해당 관측치 기각 처리 및 타임아웃/에러 플래그 누적];
    F --> H[Estimator Status 텔레메트리 퍼블리싱 (uORB)];
    G --> H;

센서 측정 오류가 발생할 경우, PX4의 비행 제어 철학은 극히 보수적이다. Ardupilot의 EKF3(Extended Kalman Filter 3) 모듈의 경우 복수의 인스턴스 코어 간 합의(Consensus)를 통해 복구 및 절체(Failover)를 시도하는 다중 백업 특성이 강한 반면, PX4는 주로 단일 활성 코어의 오염원 격리(Isolation)와 신속한 리셋(Reset) 및 센서 융합 거부 메커니즘에 의존하는 경향이 있다. 특정 센서에 대해 마할라노비스 테스트 실패(Innovation Consistency Failure)가 임계 시간 이상 지속되면, EKF2는 해당 센서 데이터를 신뢰할 수 없는 데이터로 완전히 배제하고, 가용한 다른 대체 센서(예: GPS 장애 시 Optical Flow)로 전환(Failover)하거나, 상태를 재설정(State Reset)한다.

3. QGroundControl (GCS) 관제 측면의 시동 전제 조건 검증 지표

기체의 비행 전 점검(Pre-flight Check) 시, GCS인 QGroundControl(QGC)은 MAVLink 프로토콜의 ESTIMATOR_STATUS 서브 메시지 패킷을 해독하여 사용자에게 기체 상태를 모니터링해 준다. MAVLink 패킷 내부의 test_ratio 필드들은 PX4 EKF2 모듈에서 수행한 게이트 테스트의 마진율(Margin Ratio)을 부동 소수점 형태로 나타낸다. 여기서 test_ratio 연산 결과가 1.0에 도달한다는 것은 마할라노비스 거리가 파라미터로 설정된 시스템 컷오프(Cut-off) 게이트 크기에 정확히 도달했음을 의미한다.

자율 비행을 위한 주요 측정 지표는 다음과 같다.

  • vel_test_ratio : 속도 측정 잔차의 일관성 상태
  • pos_test_ratio : 위치 측정 잔차의 일관성 상태
  • mag_test_ratio : 지자기 측정 중심의 헤딩(Heading) 잔차 일관성 상태
  • hgt_test_ratio : 고도 센서 기반 측정 잔차 일관성 상태

예를 들어 도심 지역이나 건물 근처에서 GPS 다중 경로 간섭(Multipath Interference, 위성 신호가 건물에 반사되어 수신되는 현상)이 발생하여 위치 오차가 급증할 경우, EKF2에서 계산한 pos_test_ratio는 1.0 이상으로 급격히 증가하게 된다. 이 지표가 임계치를 초과하면 붉은색 경고(Red Alarm)가 트리거되며, 관성항법 모델의 추정치를 심각하게 벗어났음을 나타낸다.

이 경우 QGroundControl의 관제 로직은 펌웨어로부터 전송된 SYS_STATUS의 상태 플래그를 확인하여, 사용자 UI 구성 화면 상의 비행 전 점검 항목에 “Preflight Fail: High GPS Error” 또는 “EKF primary is not ready” 와 같은 직관적인 오류 메시지를 표기하고 이륙 승인(Arming)을 원천적으로 거부(Arming Denied)한다.

결과적으로, 이 마할라노비스 거리 기반의 혁신 테스트 프로세스는 손상되거나 오염된 외부 관측값이 코어 비행 제어 루프에 유입되는 것을 차단하는 자율 비행 시스템의 1차 방호벽이다. 이는 치명적인 제어 발산(Control Divergence)을 막고 자율 비행의 근간이 되는 물리적 안전성을 보장하는 핵심 기술적 장치로 기능한다.
