44.1.2.2.1 등가 직렬 저항(ESR) 추정치 기반의 부하 인가 시 동적 전압 강하(Dynamic Voltage Drop) 예측 모델

무인항공기(UAV)의 이륙(Takeoff) 절차는 비행 전체 구간 중 가장 급격하게 추력이 요구되는 구간이다. 이에 따라 모터 및 변속기(ESC)로 향하는 전류(Current) 소모량이 기하급수적으로 증가하며, 배터리의 성능과 환경 요인에 따라 전압(Voltage)이 순간적으로 곤두박질치는 동적 전압 강하(Dynamic Voltage Drop) 현상이 유발된다. 이러한 상태를 사전에 추정하지 못하면, 이륙 순간 비행 제어기(Flight Controller, FC)의 최소 동작 전압 이하로 배터리 전압이 떨어져 시스템이 강제로 셧다운(Brownout)되거나 추력 부족으로 추락하는 심각한 사고를 초래할 수 있다.

따라서 PX4-Autopilot은 이륙 전 비행 전 점검(Pre-flight Checks) 단계에서 배터리의 등가 직렬 저항(Equivalent Series Resistance, ESR) 추정치를 바탕으로 부하 인가 시 발생할 동적 전압 강하 폭을 미리 계산하고, 이륙 가용 전력을 판단하는 비선형 동역학 모델(Non-linear Dynamics Model) 알고리즘을 수행한다.

1. 등가 직렬 저항(ESR)과 테브난 배터리 모델(Thevenin Battery Model)의 이해

배터리의 전기적 동특성을 수학적으로 분석하기 위해, 무인항공기 제어 소프트웨어는 일반적으로 테브난 등가 회로(Thevenin Equivalent Circuit) 모델 등을 차용한다. 실제 배터리는 단순히 일정한 전압을 내는 이상적인 전압원이 아니라, 내부에 직렬로 연결된 저항(Internal Resistance/ESR)을 포함하는 비이상적인 전압원으로 해석해야 한다. 단순화된 1차 저항 모델을 수식으로 표현하면 다음과 같다.

V_{load}(t) = V_{resting} - \{ I_{load}(t) \times R_{ESR} \}

  • V_{load}(t): 임의의 시점 t에서 측정되는 부하 인가 시의 터미널 전압 (Terminal Voltage)
  • V_{resting}: 무부하 상태일 때의 유휴 전압 (Resting Voltage, Open Circuit Voltage)
  • I_{load}(t): 시스템(특히 모터 및 페이로드)이 배터리로부터 인출하는 소모 전류량
  • R_{ESR}: 배터리의 등가 직렬 저항 (내부 저항)

비행 제어기가 모터에 100A의 전류 인출을 명령하고, 배터리의 R_{ESR} 값이 0.015 \Omega (15m \Omega)이라고 가정하자. 이때 옴의 법칙(Ohm’s Law)에 따라 100A \times 0.015\Omega = 1.5V 만큼의 전압 강하가 발생한다. 만약 V_{resting}이 14.8V (4S 배터리 규격)라면, 무인기가 전출력 이륙을 시도하는 순간 배터리 전압은 13.3V까지 급속하게 하락하게 된다. 시스템은 예측된 13.3V 값이 페일세이프(Failsafe) 임계값(BAT_CRIT_THR)보다 높은지, 관측오차 수준에서 여유 폭이 있는지를 평가하게 된다.

2. PX4-Autopilot의 동적 전압 강하 예측 아키텍처

PX4-Autopilot(펌웨어 버전 v1.14 기준)의 battery_status 모듈 및 AnalogBattery 클래스는 주기적으로 배터리 전압과 인출 전류를 아날로그-디지털 변환기(ADC)를 통해 획득하고 필터링한다.

  1. 상태 관측(State Observation): 시동(Arming) 전 유휴 상태에서 수집된 초기 전압을 V_{resting}의 근사치로 활용하거나, 전압 필터(Low-pass Filter)를 거쳐 전류 인출이 없는 상태의 기준 전압을 확인한다.
  2. ESR 추정 및 반영 (ESR Estimation): 사용자는 QGroundControl을 통해 BAT_R_INTERNAL (배터리 팩 내부 저항값 파라미터)에 값을 입력할 수 있다. 만약 별도로 설정되지 않으면 기본적으로 -1.0 (비활성화) 상태이며, 내부적으로 전력 추정 시 보수적인 하드코딩 모델을 준용하거나 전압 강하 보정을 제한한다.
  3. uORB 데이터 파이프라인: 추정 및 보상된 배터리 데이터는 battery_status.msg 구조체(uORB 토픽)에 담겨 발행(Publishing)된다. 이 데이터 구조체 내부에는 측정 전압(voltage_filtered_v)과 전류(current_filtered_a), 그리고 부하 보정이 들어간 유휴 전압 추정치(voltage_resting_estimate_v)가 포함된다.

2.1 예시: 소스 코드 레벨 구조 이해 (src/modules/battery_status)

PX4 내부의 C++ 소스 코드에서는 저항값 보완을 통해 무부하 기준 전압을 동적으로 산출하는 로직이 적용되어 있다.

// PX4-Autopilot : src/lib/battery/battery.cpp 내 예시 (논리적 추상화)
void Battery::updateBatteryStatus()
{
    // 필터링된 현재 전압 및 전류 로드
    float voltage_v = _voltage_filter.getState();
    float current_a = _current_filter.getState();
    
    // 내부 저항(ESR) 적용 유무 판별
    float resistance = _params.r_internal; 
    
    // Resting Voltage (유휴 전압) 추정: 현재 측정 전압에 전압 하강분(I * R)을 합산하여 
    // 실제 셀(Cell)의 물리적 잔여 개방 전압을 예측한다.
    float voltage_resting_estimate_v = voltage_v;
    if (resistance > 0.0f && current_a > 0.0f) {
        voltage_resting_estimate_v += (current_a * resistance);
    }
    
    // 이후 이륙 가능 여부는 이 Resting Voltage와 예측 부하 전류를 
    // 활용하여 Failsafe 로직에 전달된다.
}

Pre-flight Checks(비행 전 점검)를 총괄하는 커맨더(Commander) 모듈은 voltage_resting_estimate_v에서 예상 최대 이륙 전류(Maximum Takeoff Current)를 빼서 계산된 최소 예상 전압(Predicted Minimal Voltage)이 BAT_LOW_THR 이하가 될 것으로 추정되면 시스템 이륙 경고를 발생시키며 Arming 및 이륙(Takeoff) 명령을 거절(Reject)한다.

3. 작동 원리 다이어그램 (Mermaid)

다음은 GCS로부터 이륙 명령(Takeoff Command)이 하달될 때 PX4 내부에서 등가 직렬 저항 기반 동적 전압 예측을 통해 안전성을 검증하는 과정이다.

sequenceDiagram
    participant GCS as QGroundControl (v4.3+)
    participant Cmd as Commander (PX4)
    participant Bat as Battery_Status (PX4)
    participant Est as Dynamics Estimator
    
    GCS->>Cmd: COMMAND_LONG (MAV_CMD_NAV_TAKEOFF)
    activate Cmd
    Cmd->>Bat: uORB: 배터리 상태 검사 요청
    activate Bat
    Bat->>Est: 측정된 전압, 전류 및 BAT_R_INTERNAL 파라미터 전달
    Est-->>Bat: Resting Voltage (유휴 전압 추정치) 반환
    Bat-->>Cmd: 예상 최소 전압 및 배터리 잔량(%) 도출 데이터 반환
    deactivate Bat
    
    alt 예측 최소 전압 > BAT_CRIT_THR
        Cmd->>GCS: COMMAND_ACK (MAV_RESULT_ACCEPTED)
        Cmd->>Cmd: 상태 전환 (TAKEOFF 실행)
    else 예측 최소 전압 <= BAT_CRIT_THR
        Cmd->>GCS: COMMAND_ACK (MAV_RESULT_DENIED)
        Note right of GCS: "이륙 부하 시 전압 강하가<br>임계값을 초과할 것으로 예상됨"
        Cmd->>Cmd: 이륙 절차 중단 및 안전 모드 유지
    end
    deactivate Cmd

4. PX4-Autopilot과 Ardupilot 간 아키텍처 및 철학 비교

Ardupilot 및 Mission Planner 환경과 비교할 때, 배터리 동적 변화에 대응하는 양측의 관제 및 제어 철학에는 미세한 차이가 존재한다.

  1. 상태 추정 연속성:

    • PX4-Autopilot: battery_status uORB 메시지 및 관련 모듈은 상태 머신(State Machine)에 따라 배터리 상태를 전역적으로 평가한다. 단순 전압이 아닌 V_{resting} 추정 체계를 전반적인 Failsafe 트리거(Trigger) 기준으로 삼으려는 경향이 강하며, 급격한 스로틀 레버(Throttle Lever) 상승에도 단기적 전압 딥(Voltage Dip)으로 인해 오판행위를 방지하기 위한 필터링 및 융합 모델 적용을 선호한다.
    • Ardupilot: 배터리에 BATT_ARM_VOLT를 두고 시동(Arm) 시점에서 통과해야 할 직관적인 전압 하한선을 설정한다. Ardupilot에서도 BATT_RESIST 파라미터를 통해 내부 저항 보상이 이루어지나, 시스템의 반응 구조상 “배터리 로우 상태 전환 시점의 타이머 딜레이“를 통해 순간 전압 강하를 예외 처리하는 형태가 많이 사용된다.

  2. QGroundControl 관제 차이:

    • QGroundControl UI는 PX4 연결 시 MAVLink BATTERY_STATUS 다이얼로그를 통해 전압 강하 보정 기능을 그래픽적으로 지원한다. 즉, 단순히 실시간 측정 전압 표시를 넘어 비행 중 지속적으로 계산된 남은 비행 시간(Estimated Flight Time Remaining)의 변동성을 최소화하는 방향성으로 정보를 제공한다.

5. 파라미터 설정 및 관리를 통한 시스템 최적화

등가 직렬 저항(ESR) 모델이 정밀하게 개입하기 위해 사용자는 반드시 사용 중인 배터리의 특성을 파악하여 QGroundControl을 통해 다음 파라미터를 보정해야 한다.

  • BAT_R_INTERNAL: 배터리 내부 총 저항(단위: \Omega). 실제 방전 테스트기(Discharger) 혹은 충전기(Charger) 상에서 측정된 각 셀 내부 저항 값의 합으로 기입해야 한다.
  • BAT_LOW_THR & BAT_CRIT_THR: 전압 강하 모델의 판단점. 각각 예비 착륙(Return) 혹은 즉각적인 긴급 착륙(Land/Failsafe)이 이행되는 경고 임계값이다. BAT_R_INTERNAL 설정값이 클수록 동적 부하 시 보정 수치가 커지므로, 임계값 조정은 측정값 기반으로 보수적인 세팅이 선행되어야 한다.

결과적으로, 등가 직렬 저항 추정 모델을 이륙 심사 및 관제 시스템에 연동하는 것은 하드웨어 동작이 초래할 극한의 물리적 변동성을 소프트웨어의 상태 머신(State Machine) 내에서 선제적으로 수학화하고 융합하는 무인항공기 안정성의 핵심 기술로 볼 수 있다.
