39.1.1.2. 다중 경로 페이딩(Multipath Fading) 및 간섭에 의한 군집(Burst) 패킷 손실 확률 모델링

무인항공기(UAV)가 도심지, 산악 지형, 또는 복잡한 산업 채굴 현장 등에서 비행할 때, 지상 관제 시스템(GCS)에서 송신된 무선 전파는 기체에 단일한 직선 경로(Line-of-Sight, LoS)로만 도달하지 않는다. 주변 건물, 지표면, 장애물에 의해 전파는 반사(Reflection), 회절(Diffraction), 산란(Scattering) 과정을 거치며 여러 경로를 통해 수신 안테나에 복합적으로 도달한다. 본 절에서는 이러한 다중 경로 페이딩(Multipath Fading) 현상이 무인기 제어망에 미치는 영향을 정보 통신 공학적 관점에서 분석하고, 이로 인해 발생하는 군집(Burst) 패킷 손실 모델을 고찰한다.

1. 다중 경로 페이딩(Multipath Fading) 이론

다중 경로에 의해 수신 안테나에 도달하는 각각의 전파는 이동 거리의 차이로 인해 서로 다른 위상(Phase)과 진폭(Amplitude)을 둔다. 이 파동들이 합쳐질 때, 위상이 일치하면 신호가 증폭(Constructive Interference)되나, 위상이 180도 엇갈리면 신호가 서로 상쇄(Destructive Interference)되어 수신 신호 강도(RSSI)가 급격히 영(Null)에 가깝게 떨어지는 페이딩 현상을 유발한다.

고속으로 기동하는 기체 특성상 이러한 페이딩 현상은 시간에 따라 매우 동적으로 변동(Time-varying Fading)하며, 대표적으로 다음 두 가지 통계적 모델로 설명된다.

  1. 라이시안 페이딩(Rician Fading): 강력한 가시 경로(LoS)가 하나 존재하고, 나머지 다중 경로 신호들이 약하게 존재하는 경우의 채널 모델이다. 드론이 탁 트인 개활지에서 비행할 때 주로 적용된다.
  2. 레이리 페이딩(Rayleigh Fading): 지배적인 가시 경로(LoS)가 없이(Non-Line-of-Sight, NLoS), 모든 신호가 건물이나 지형지물에 반사되어 도달하는 경우이다. 도심지 빌딩숲 비행 시 적용되며, 신호의 심각한 딥(Dip) 현상이 빈번하게 발생한다.

2. 군집(Burst) 패킷 손실 확률 모델링: Gilbert-Elliott 모델

레이리 페이딩이 발생하는 환경이나, 드론이 대형 건물의 뒤로 숨어버리는 섀도잉(Shadowing) 현상이 발생할 때, 데이터 링크의 오류 패턴은 단발성의 무작위 에러(Random Error)가 아닌 **군집 에러(Burst Error)**의 형태를 띤다. 즉, 한 번 패킷 오류가 발생하면 그 다음 패킷도 연속해서 유실될 확률이 매우 높아진다.

이러한 군집 패킷 손실 확률을 정보 이론적으로 모델링하기 위해 널리 사용되는 것이 **길버트-엘리엇 모델(Gilbert-Elliott Model)**이다. 이는 통신 채널을 두 가지 마르코프 상태(Markov State), 즉 통신 상태가 양호한 ’Good State(G)’와 에러율이 높은 ’Bad State(B)’로 정의한다.

  • P_{GG}: 양호한 상태에서 다음 패킷도 양호할 확률
  • P_{GB}: 양호한 상태에서 다음 패킷 시점에 나쁜 상태로 전이될 확률
  • P_{BB}: 나쁜 상태에서 다음 패킷도 유실(연속 에러)될 확률
  • P_{BG}: 나쁜 상태에서 통신이 회복될 확률

이 모델에 따르면, 군집 에러의 평균 지속 시간(Average Burst Length)은 1 / P_{BG}로 계산된다. 도심지 비행 시 P_{BB}는 매우 높은 값을 가지며, 이는 한 번 통신이 끊기면 수 초간 회복되지 않는 “Blackout” 상태가 유지됨을 수학적으로 증명한다.

3. PX4 및 MAVLink 기반 제어망에서의 공학적 대응

군집 형태의 패킷 손실은 GCS가 전송한 제어 명령어(COMMAND_LONG 등)가 수 회 연속으로 묵살됨을 의미하며, 단순한 1~2회의 즉각적인 재전송(Retransmission) 메커니즘으로는 성공하기 어렵다는 공학적 결론을 내린다. 따라서 PX4 시스템과 QGroundControl은 다음과 같은 구조적 대응책을 강구해야 한다.

3.1 지수적 백오프(Exponential Backoff)를 적용한 적응형 재전송

QGroundControl이 명령어를 보내고 COMMAND_ACK를 대기할 때, 타임아웃 주기를 고정된 짧은 시간(예: 0.5초)으로 설정하고 연속해서 요청을 보내는 것은 페이딩 딥(Dip) 구간, 즉 P_{BB}가 우세한 Bad State 구간에서는 무의미하다. 오히려 채널 점유율만 높여 충돌을 유발한다. 따라서 딥(Dip) 구간이 지나갈 때까지 기다리는 적응형 타임아웃이나 재전송 주기를 점진적으로 늘리는 설계가 요구된다.

3.2 포워드 에러 코렉션(FEC, Forward Error Correction)

단일 패킷 유실을 복구하기 위한 네트워크 계층에서의 최적화다. 데이터 스트림에 잉여 비트(Redundant Bit)를 포함시켜 수신단에서 어느 정도의 단발성 오류를 스스로 정정하게 만드는 기법이다. MAVLink 자체는 애플리케이션 프로토콜이므로 FEC 메커니즘을 내장하고 있지 않으나, RF 모뎀 설정(예: Microhard, RFD900)에서 FEC 옵션을 활성화함으로써 군집성이 옅은 얕은 페이딩 구간을 극복할 수 있다.

3.3 명령 원자성 보존 및 상태 레지스터

기체가 Bad State(버스트 손실 상태)에 진입하더라도 오동작하지 않도록 PX4의 내부 상태 머신(State Machine)은 마지막으로 수신한 정상 명령의 맥락을 엄격히 보존한다. 데이터 링크가 복구(P_{BG} 전이)되어 낡은 명령어의 재전송분이 지연 도착하더라도, PX4는 내부 시퀀스 및 상태 레지스터 검증을 통해 최신 제어 명령과의 충돌을 방지하여 시스템 안정을 유지하도록 구조화되어 있다.
