39.6.1.1. 시계열 평활화를 위한 지수 가중 이동 평균(Exponential Weighted Moving Average, EWMA) 필터링 기법

지상 관제 시스템(GCS)과 PX4-Autopilot를 연결하는 무선 텔레메트리 환경에서는, 취득된 왕복 지연 시간(RTT) 데이터 세트가 심한 잡음(Noise)과 단기적 스파이크(Spike)를 포함하는 비정상(Non-stationary) 시계열 특성을 지닌다. 이러한 거친(Raw) RTT 샘플을 재전송 타임아웃(RTO) 계산에 바로 대입하게 되면, 시스템 타이머의 갱신이 지나치게 민감해져 전체 MAVLink 제어 루프의 치명적인 불안정성을 초래한다. 이를 방지하기 위해 최신 관제 아키텍처에서는 메모리 효율이 뛰어나면서도 최근 통신 상태의 가중치를 높게 유지하는 지수 가중 이동 평균(Exponential Weighted Moving Average, EWMA) 필터링 기법을 투입하여 통계 시계열을 매끄럽게 평활화(Smoothing)한다.

1. 단순 이동 평균(SMA) 모델 한계와 EWMA 도입의 공학적 이점

센서 데이터나 네트워크 지표를 평활화하고자 할 때 통상적으로 많이 쓰이는 단순 이동 평균(SMA, Simple Moving Average)은 큐잉된 최근 N개의 샘플에 모두 동일한 파라미터 가중치를 부여하는 산술 방식이다. 그러나 MAVLink 응답성 지연 추정 모듈에 SMA를 단일 적용할 경우 다음과 같은 시스템 설계상의 치명적 한계에 부딪힌다.

  1. 메모리(Memory Footprint) 오버헤드: 길이 N에 달하는 윈도우 크기의 배열 버퍼(Buffer)를 점유하고 유지해야 하므로, 다수의 에이전트를 동시 다발적으로 관제하는(Swarm) GCS C++ 힙(Heap) 환경이나 PX4 내부 시스템에서는 메모리 관리 단가가 급증한다.
  2. 응답성의 위상 지연(Phase Lag): 극단적인 과거 데이터와 무르익은 최근 데이터가 산술적으로 동일한 비중으로 통합되므로, 기체가 산악형 장애물 뒤로 진입하여 네트워크 전파 상태가 급박하게 변동할 때 타임아웃 값이 이를 즉각적으로 추종하지 못하는 지연 현상이 뚜렷하게 도출된다.

반면, EWMA 필터 모형은 1차 저역 통과 필터(1st-order Low-pass Filter)와 해석학적으로 동일한 무한 임펄스 응답(IIR, Infinite Impulse Response) 전달 속성을 내포하며 오직 직전의 상태 변수(State Variable) 단 하나만을 메모리에 보존하면 된다. 아울러 이 모델은 방금 수신한 최신 RTT 샘플에 가장 압도적인 가중치를 부여하고, 과거로 역행할수록 그 영향력을 지수 함수적으로 깎아내리므로 통신 대역 상태의 최신 트렌드를 매우 민첩하게 추종(Tracking)할 수 있는 탁월한 알고리즘적 이점을 관제 시스템에 제공한다.

2. MAVLink RTT 평활화를 위한 EWMA 수학적 메커니즘

QGroundControl(v4.3)의 내부 백엔드 타임아웃 산출 로직에서 맹활약하는 EWMA의 뼈대 수식은 다음과 같이 정립된다. 특정한 i번째 텔레메트리 RTT 계측치(샘플)를 R_i라 약속할 때, 새롭게 평활화된 RTT 예상 척도(Smoothed RTT, SRTT)는 재귀적 형태의 수식으로 갱신된다.

SRTT_i = (1 - \alpha) \cdot SRTT_{i-1} + \alpha \cdot R_i

여기서 \alpha 변수는 **평활화 상수(Smoothing Factor)**로, 0 < \alpha < 1의 도메인을 영위한다.

  • \alpha가 1에 근접할수록: 직전 샘플인 R_i의 반영률이 극대화되므로, 무선 채널의 변화 동향에 매우 민감하게 적응하지만 노이즈(Jitter) 유발에 따른 평활 궤적의 출렁임이 악화된다.
  • \alpha가 0에 근접할수록: 기왕의 누적 관성치인 SRTT_{i-1}에 강박적으로 의존하게 되어 궤적의 평활화 효과는 극대화되나, 신호 세기 저하로 인해 통신망이 급변했을 때 타임아웃 마진을 상향/하향 조정하는 반응 민첩성이 심각하게 저하된다.

일반적으로 고신뢰성을 요하는 공학적 관제 모델에서 TCP 생태계의 기초 혼잡 제어 패러다임을 차용하여 **\alpha = 0.125 \left( = \frac{1}{8} \right)**를 적용하는 것이 사실상의 팩토링 스탠다드로 통용된다. 이러한 계수 설정은 부동소수점 연산이 제한되거나 계산 기회비용이 비싼 마이크로컨트롤러(예: PX4의 NuttX 코어 환경) 내부에서, 나눗셈 대신 빠른 비트 시프트 연산(>> 3)으로 투명하게 치환될 수 있어 인터럽트 락 실행 시간의 무결성을 철저히 고수할 수 있다.

SRTT_i = SRTT_{i-1} + \frac{1}{8} \cdot (R_i - SRTT_{i-1})

3. ROS2 오프로드 제어 환경에서의 분산 적용 사례

PX4 v1.14 버전을 기점으로 Micro XRCE-DDS 구조를 경유하여 ROS2(Robot Operating System 2)의 노드 생태계로 진입하게 되면, 통신 지연 추정의 레이어 위상이 격상된다. 이기종 분산 시스템 간 결합을 밀착 관리하는 DDS 미들웨어 레이어는, 내부적으로 글로벌 DataWriter와 로컬 DataReader 간의 RTPS(Real-Time Publish Subscribe) 심박(Heartbeat) 응답 타임라인을 평가할 때 역시 이와 유사한 수학적 구조의 EWMA 캐싱 모델을 은닉형으로 가동하여 패킷 전송 주기를 리밸런싱(Rebalancing)한다.

ROS2 Offboard 컨트롤 노드 기반의 자율 에이전트(Autonomous Agent) 운용을 구상할 시, 개발자는 C++ 기반의 컨트롤 계층에서 rclcpp::TimerBase를 통해 주기적인 궤적 제어 루프를 생성한다. 이때 GCS(오프보드 컴패니언 컴퓨터)와 기체 간의 링크 레이턴시를 직접 감리하고자 하는 특성화된 에이전트 아키텍처에서는, 어플리케이션 계층 소스 스크립트에 다음과 같은 커스텀 EWMA 필터 객체를 직접 이식할 수 있다.

class LatencyEstimator {
private:
    double srtt_;
    double alpha_;
    bool initialized_;

public:
    LatencyEstimator(double alpha = 0.125) 
        : srtt_(0.0), alpha_(alpha), initialized_(false) {}

    double update(double rtt_sample) {
        if (!initialized_) {
            srtt_ = rtt_sample; // 최초 샘플로 필터 컨버전스 초기화
            initialized_ = true;
        } else {
            // EWMA 수식 전개 부문
            srtt_ = (1.0 - alpha_) * srtt_ + alpha_ * rtt_sample; 
        }
        return srtt_;
    }
};

위 로직 블록은 COMMAND_LONG 릴레이 파이프라인에서 직수신된 COMMAND_ACK 트랜잭션의 도달 소요 시간을 GCS가 직접 샘플 캡처하여, 군집 비행(Swarm) 중 통신이 일시적으로 열화된 특정 개체에 대한 제어 명령어 퍼블리싱 주파수(Publishing Rate)를 동적으로 스로틀링(Throttling)하기 위한 피드백 선행 지표로 유용하게 환원된다.

4. Ardupilot 관제 파이프라인과의 아키텍처 비교 관점

Ardupilot은 본질적으로 광범위하고 다양한 저사양 레거시 하드웨어 변수를 품기 위한 범용 파라미터 방어막(Safety net) 구축을 선호하므로, 네트워크 모델링 시 고정형 상수 타임아웃에 묵직하게 기대어 재전송 전략을 구사하는 비중이 매우 두텁다. 만일 Ardupilot을 운용하는 환경(Mission Planner 등)에서 네트워크의 동적 변화에 순응할 필요가 발생한다면, 해당 튜닝 절차는 유저 폼 인터페이스 상의 파라미터 수동 업데이트를 기반으로 일회적, 계단식 조정을 채택하는 데 그치는 경향이 강하다.

하지만, PX4-Autopilot 생태계의 중추를 겨냥하여 진화하는 QGroundControl의 차량 통신 관리 매니저(Vehicle Link Manager)는 C++ Vehicle 객체의 메인 생명주기(Lifecycle)에 EWMA 기반의 동적 예측 필터를 생체적으로 결속시킨 점이 돋보인다. 이를 통해 어떠한 복잡 다단한 통신 환경 굴곡 요인도 연속적인 시계열 평활화 모델 속으로 자연스레 융화시켜 버리며, 우발적인 일과성 패킷 로스가 관제 큐 전체의 상태 머신(State Machine) 오동작이나 블로킹(Blocking)으로 전이되는 연쇄 반응을 원천적으로 봉쇄하는 최상급의 구조적 매끄러움(Structural Smoothness)을 시현한다.
