39.2.5.1. 재전송 횟수 식별을 위한 단조 증가 함수(Monotonically Increasing Function) 적용

1. 개요

MAVLink 명령어 패킷의 무결성을 수호하는 confirmation 계수기(Counter)는 임의의 난수나 해시(Hash) 프레임을 사용하는 대신, 오직 양의 방향으로만 더해지는 **단조 증가 함수(Monotonically Increasing Function)**의 수학적 성질을 채택하였다.

단조 증가란 시간이 흐름에 따라 변수의 값이 결코 감소하지 않고 순차적으로 증가(혹은 유지)하는 수학적 제약 조건(Constraints)을 의미한다. 본 절에서는 GCS(지상 관제 시스템)가 재전송 루프를 가동할 때 이 단조 증가 로직이 어떻게 적용되는지, 그리고 이 수학적 성질이 무선 통신망에서 빈번하게 발생하는 ’패킷 순서 역전(Out-of-Order Delivery)’이라는 치명적 결함을 어떻게 방어해 내는지 심층적으로 분석한다.

2. 단조 증가 계수기(Counter)의 데이터 파이프라인

GCS(예: QGroundControl) 내부에 구현된 비동기 명령어 대기열(Pending Command Queue)은 특정 드론으로 명령을 전송할 때 다음과 같은 단조 증가 사이클을 밟는다.

  1. 최초 발송 (Transmission T_0):
    임무(Mission) 시작 명령을 내릴 때 confirmation 필드를 0으로 세팅해 발송한다. 동시에 GCS 내부의 타임아웃 타이머(e.g., 500\text{ms})가 돌기 시작한다.
  2. 1차 재전송 (Retransmission T_1):
    타임아웃이 발생할 때까지 ACK가 오지 않으면, GCS는 원본 데이터를 폐기하지 않고 꺼내서 confirmation = 1 로 변경한 뒤 재발송한다.
  3. N차 재전송 (Retransmission T_n):
    마찬가지로 응답이 없으면 confirmation 밸류를 2, 3, 4… 로 단조 증가시키며 쏜다. 최대 허용치(Max Retries, 보통 5 이하)에 도달하면 해당 트랜잭션을 포기(Timeout Error) 처리한다.

여기서 가장 핵심적인 알고리즘 원칙은 **“한 번 시작된 명령 트랜잭션 내에서는 confirmation 값이 절대로 이전 값으로 되돌아가지 않는다(No Rollback)”**는 점이다.

3. 무선망의 패킷 순서 역전(Out-of-Order) 방어 메커니즘

무선 주파수(RF) 기반의 텔레메트리나 다중 홉(Multi-hop) 메시 네트워크 환경에서는, 늦게 출발한 패킷이 빠른 경로를 타서 먼저 출발한 패킷을 앞지르는 현상 즉, **순서 역전(Out-of-Order Delivery)**이 매우 빈번하게 지연 시간(Jitter)으로 작용한다.

만약 명령어 재전송 과정에서 GCS가 보낸 1차 재전송(confirmation=1) 패킷이 어느 중계기 큐에 갇혀 버렸고, 2차 재전송(confirmation=2) 패킷이 기체로 먼저 도착했다고 가정해 보자.

  1. **기체(PX4)**는 confirmation=2 인 패킷을 제일 처음 수신한다. 새로운 명령임을 인지하고 즉각 액추에이터를 가동한 뒤 GCS로 성공(ACK)을 보낸다. (그리고 자신의 캐시에 latest_confirmation = 2를 기록해 둔다.)
  2. 그로부터 0.5초 뒤, 인터넷 혹은 라디오 망에서 길을 잃었던 confirmation=1 패킷이 지각하여 기체에 도달한다.

이때 만약 confirmation 필드가 단순한 플래그(Flag)였거나 랜덤한 난수였다면, 기체는 이게 늦게 온 쓰레기(Stale) 패킷인지 구별할 방도가 없어 또다시 명령을 중복 실행(Duplicate Execution)해 버렸을 것이다.

그러나 단조 증가 함수라는 검증 조건 덕분에 기체의 파서(Parser) 로직은 이 지각 패킷을 완벽히 걸러낸다.
기체는 수신한 패킷의 confirmation=1 값을 자신의 캐시에 저장된 latest_confirmation = 2와 비교(Compare)한다.
조건문 if (incoming_confirmation < cached_confirmation)이 참(True)이 되므로, 기체는 “이것은 유효 기한이 지난(Stale) 과거의 유령 패킷이다“라고 확정 지을 수 있다. 기체는 이 패킷에 대해 제어 계통(State Machine)은 전혀 건드리지 않은 채, 조용히 폐기(Drop) 하거나 단순히 이전 캐시 결과를 담은 ACK만 앵무새처럼 되풀이해 발송하는 것으로 방어망을 전개한다.

4. 시스템 아키텍처적 결론: 시공간적 인과율(Causality)의 부여

결국 MAVLink 설계자들이 확인 필드에 단조 증가 계수기 모델을 도입한 이유는, 신뢰성이 결여된(UDP-like) 기반 네트워크 위상 공간에서 패킷들 간의 **‘시간적 전후 관계(Temporal Causality)’**를 강제하기 위함이다.

비행 제어기(PX4-Autopilot)는 밀리초 단위로 수십 개의 메시지를 쏟아내는 급박한 I/O 환경 속에서도, 오직 단조 증가하는 정수값 하나를 검사하는 O(1) 상수 시간 복잡도의 알고리즘만으로 과거(Stale Packet)와 현재(Valid Packet)를 명확히 갈라치기(Decouple) 할 수 있게 된다.
이처럼 극도의 미니멀리즘(1바이트 정수)을 추구하면서도 분산 시스템(Distributed System)의 가장 큰 난제인 ’순서 역전(Out-of-order)’과 ’중복 실행(Idempotency 문제)’을 동시에 압살하는 이 계수기 모델은, PX4 GCS-UAV 통신 파이프라인 디자인이 어째서 항공 우주급 신뢰성(Aerospace-grade Reliability)을 만족하는지 스스로 입증하고 있다.
