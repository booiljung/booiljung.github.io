39.1.3.1. 무인기 시스템에서의 일관성(Consistency)과 가용성(Availability)의 트레이드오프 설계

지상 관제 시스템(GCS)과 PX4-Autopilot 간의 무선 네트워크가 분할(Partition)되는 데이터 링크 손실(Data Link Loss) 상황은 피할 수 없는 현실이다. 이 때 제어 시스템은 조종자의 통제권이 미치는 ’일관성(Consistency)’을 수호할 것인가, 혹은 드론이 현재 수행 중인 임무의 ’가용성(Availability)’을 유지할 것인가의 중대한 설계 트레이드오프에 직면하게 된다. 무인기(UAV) 운영 환경에서는 비행 목적(가시권 수동 조종, 비가시권 자율 비행 등)에 따라 이상적인 균형점이 달라지므로, PX4는 단일한 잣대가 아닌 비행 모드(Flight Mode)와 사용자 파라미터(Parameter)를 결합하여 설계된 유연한 방어 모델을 제공한다.

1. 강한 일관성(Strong Consistency)과 안전 최우선 모델

강한 일관성 모델은 분산 시스템 내의 모든 노드(GCS와 PX4)가 정확히 동일한 상태 데이터를 바라보아야만 후속 동작을 수행하는 보수적인 접근법이다. 조종자와 기체 간의 즉각적인 피드백이 생명줄인 수동 비행 모드(Manual, Stabilized, Position 등) 혹은 조이스틱 제어 환경에서 필수적으로 요구된다.

1.1 설계 원칙

  • 수동 조종 중 네트워크 분할이 발생하면, 기체의 조종 간(Stick) 입력 상태와 GCS 화면의 텔레메트리(ATTITUDE) 사이의 일관성이 붕괴된다.
  • 이러한 ‘Blind Delay’ 상태에서 기체가 마지막으로 받은 조종 입력을 그대로 유지하며 비행하는 것(가용성 유지)은 치명적인 추락 사고로 직결된다.
  • 따라서 PX4 커맨더(Commander) 모듈은 HEARTBEAT 패킷의 도달이나 조종 입력(Manual Control Setpoint)이 사전에 정의된 타임아웃(Timeout) 한계치(예: 0.5초)를 넘기면 즉각적으로 제어의 가용성을 포기하고 일관성 방어를 위한 Failsafe를 가동한다.

1.2 MAVLink 및 로직 구현

PX4의 커맨더(Commander) 메인 루프는 GCS 또는 유선 리모트 컨트롤러(RC)의 연결 상태 플래그를 주기적으로 평가한다. 통신 두절로 일관성 붕괴가 감지되면 정지(Hold) 명령어를 내부적으로 발행하고, 상황이 지속되면 RTL(Return-To-Launch)이나 비상 착륙 모드로 자동 전환(NAV_DLL_ACT 파라미터 적용)하여 기체를 안전한 상태 공간으로 강제 편입시킨다.

2. 최종적 일관성(Eventual Consistency)과 임무 지속 모델

비가시권(BVLOS) 비행, 무선 주파수 음영 지역(건물 뒤, 지하 등) 투입, 혹은 동반 컴퓨터(Companion Computer)가 탑재된 **자동 비행(Mission, Offboard)**의 경우 설계 방향성이 정반대로 적용된다. 이 환경에서는 네트워크 단절이 자주 일어날 것을 시스템이 사전에 가정하고 있다.

2.1 설계 원칙

  • 데이터 링크가 끊어져 GCS 화면에 실시간 기체 궤적이 나타나지 않더라도(일관성 상실), 기체의 항법 장치(GPS, VIO 등)와 온보드 제어기(PX4 Navigator)가 정상이라면 기체는 자율 임무를 중단해서는 안 된다(가용성 유지).
  • 시스템은 네트워크가 재연결(Re-connection)되었을 때 그동안 누적된 비행 궤적이나 이벤트를 한꺼번에 동기화하는 구조 즉, **최종적 일관성(Eventual Consistency)**을 채택한다. 네트워크 분리 기간 중에는 가용성 극대화에 초점을 맞춘다.

2.2 MAVLink 및 로직 구현

PX4는 자율 모드 시 통신 상실에 따른 반응을 유연하게 제어하기 위한 옵션을 제공한다.

  • NAV_DLL_ACT 매개변수를 ‘Disabled’ (0)로 설정할 경우: 임무(Mission) 모드 도중 GCS와의 텔레메트리가 끊기더라도 비행을 멈추거나 RTL로 강제 전환되지 않고 설정된 웨이포인트(Waypoint)를 끝까지 수행한다.
  • Offboard 모드 대응 (COM_OBL_ACT): 라즈베리 파이나 ROS2 노드 같은 온보드 컴패니언 컴퓨터가 오프보드 제어를 지휘하고 있을 때, 외부 GCS 통신망이 아닌 단거리 TARGET_COMP 간 브릿지(Serial/Ethernet)만 무결하다면 자율 제어 능력을 신뢰하고 가용성을 이어나간다.
