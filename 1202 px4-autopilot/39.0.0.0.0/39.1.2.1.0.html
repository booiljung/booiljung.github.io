39.1.2.1. 텔레메트리 스트림의 마르코프 특성(Markov Property)과 오차 허용성

무인항공기(UAV)와 지상 관제 시스템(GCS) 사이의 데이터 링크에서 가장 많은 대역폭을 차지하는 시계열 데이터(Time-series Data)는 기체의 상태 정보를 담고 있는 텔레메트리(Telemetry) 스트림이다. 센서에서 수집되어 오토파일럿(PX4)의 추정기(Estimator, EKF2)를 거친 롤, 피치, 요(Roll, Pitch, Yaw)와 같은 ATTITUDE 데이터나 위도, 경도, 고도 등의 GLOBAL_POSITION_INT 메시지들이 대표적이다. 본 절에서는 이러한 텔레메트리 데이터 스트림이 지니는 수학적 성질인 ’마르코프 특성(Markov Property)’을 정의하고, 이를 통해 도출되는 통신 시스템의 오차 허용성(Fault Tolerance)과 PX4 내부 설계의 타당성을 입증한다.

1. 텔레메트리 데이터의 마르코프 특성(Markov Property)

확률론(Probability Theory)에서 ’마르코프 특성(Markov Property)’이란, 확률 과정(Stochastic Process)의 미래 상태가 과거의 상태들이나 도달 경로와는 무관하고, 오직 현재의 상태(Present State)에만 완전히 의존한다는 성질을 뜻한다. 즉, ’과거는 현재에 내포되어 있다’는 전제 하에 모형이 구성된다.

무인기의 비행 역학(Flight Dynamics) 기반 상태 공간 모델(State Space Model)은 전형적인 마르코프 결정 과정(Markov Decision Process, MDP)을 따른다. 시점 t에서의 기체 상태 벡터를 \mathbf{x}_t 라 할 때, 시점 t+1의 상태 \mathbf{x}_{t+1}은 이전 궤적 \mathbf{x}_0, \mathbf{x}_1, \dots, \mathbf{x}_{t-1} 이 아닌 오직 \mathbf{x}_t와 해당 시점의 제어 입력 \mathbf{u}_t 에만 의존한다.

지상 관제 시스템(QGroundControl)이 기체의 비행 궤적을 3D 지도에 렌더링하고 계기판(Artificial Horizon)을 스크린에 그리기 위해 필요한 것은 가장 최신의 상태 벡터 \mathbf{x}_t이다. 시간 t-1의 자세 정보(ATTITUDE)가 무선 주파수 환경의 간섭으로 인해 GCS에 수신되지 않고 유실(Drop)되었다고 하더라도, 곧바로 t 시점의 신규 패킷이 도착하면 UI는 완벽하게 그리고 연속적으로 기체의 상태를 표현할 수 있다. 텔레메트리 스트림이 지닌 이와 같은 강력한 마르코프 특성은 과거 데이터의 ’역사성’보다 최신 데이터의 ’현재성’에 절대적인 가치를 부여한다.

2. 마르코프성 기반의 오차 허용성(Fault Tolerance)

데이터가 마르코프 특성을 지닌다는 점은 시스템 레벨에서 놀라운 이점을 제공한다. 통신 채널에 심각한 잡음(Noise)이 개입하여 일정 기간 동안 군집성 패킷 유실(Burst Packet Loss)이 발생하더라도, 시스템은 유실된 과거의 데이터를 복구할 필요를 느끼지 않는다. 즉, 과거 패킷의 **오류 및 유실에 대한 강한 내성(Fault Tolerance)**을 지니게 된다.

2.1 패킷 재전송(Retransmission) 메커니즘의 부재

제어 명령어 통신망에서 오차를 복구하기 위한 핵심 무기는 재전송(Retransmission)이다. 하지만 마르코프 특성을 띠는 텔레메트리 스트림에서 재전송은 득보다 실이 크다.
만약 t-1 시점의 유실된 자세 데이터를 t+1 시점에 재전송받아 화면에 그린다면, 관제 화면 상의 비행기 아이콘이 뒤로 “역주행(Rewind)“하는 심각한 시각적 왜곡(Visual Jitter)이 발생하며 사용자에게 혼동을 주게 된다.

2.2 무선 대역폭 최적화

마르코프 특성에 기반을 둔 오차 허용성은 한정된 무선(RF) 대역폭을 극도로 효율적으로 사용할 수 있게 해준다. GCS나 컴패니언 컴퓨터(ROS2 노드)는 텔레메트리 데이터에 대해 수신 확인(ACK, Acknowledgment)을 회신할 필요가 없다. (이와 대조되는 메커니즘이 앞서 살펴본 COMMAND_LONG / COMMAND_ACK 구조이다.) 이로 인해 수신 측에서 송신 측으로 흘러가는 오버헤드 통신이 삭제되어 망을 경량화할 수 있다.

3. PX4-Autopilot 아키텍처의 반영

PX4 시스템의 MAVLink 처리 및 스트리밍 레이어는 텔레메트리 데이터의 마르코프 특성을 깊이 이해하고 다음과 같이 소프트웨어에 투영하였다.

  1. Ring Buffer 구조의 폐기 정책:
    PX4 내의 특정 모듈(예: mavlink_messages.cpp)이 uORB 토픽(예: vehicle_attitude)을 구독하고 이를 MAVLink를 통해 스트리밍할 때, 전송 버퍼(Tx Buffer)가 꽉 차면 과거의 오래된 메시지는 가차 없이 폐기(Drop)된다. 가장 최신 데이터의 송출 슬롯을 확보하는 방식을 채택하여 최신성을 우선한다.
  2. 동적 주파수 할당(Rate Control):
    데이터의 유용도에 따라 초당 전송 횟수를 조절한다. 자세 제어와 직결된 토픽은 높은 Hz로 전송하여 단일 패킷 유실 시 관제상의 공백(Gap) 범위를 줄이고, 배터리 소모량 같이 변화율이 적은 토픽은 낮은 Hz 배정을 통해 망 부하를 경감한다.

이처럼 무인기의 텔레메트리 전송 파이프라인은 데이터가 갖는 마르코프 특성에 의거하여, 손실 발생에 재전송이나 복구 전략으로 맞서기보다는 ’무시(Ignore)와 최신 데이터 덮어쓰기’라는 단순명료한 시스템 엔지니어링 패러다임을 통해 통신 장애에 훌륭하게 대처하고 있다.
