39.2.5. 확인(Confirmation) 필드의 계수기(Counter) 모델

무인항공기(UAV)를 원격으로 제어하는 개방된 텔레메트리 무선 통신망은 도심지의 Wi-Fi 간섭, 전파 교란(Jamming), 혹은 물리적인 장애물 반사에 의한 신호 감쇄(Multipath Fading) 등으로 인해 필연적이고 일상적인 데이터 패킷의 유실(Packet Loss)을 겪게 된다. 지상 관제 시스템(GCS)이 드론의 자율 비행 알고리즘에 개입하기 위해 하달한 중대한 제어 명령(COMMAND_LONG 또는 COMMAND_INT)이 공중에서 분해되어 사라졌을 때, 시스템을 복구하기 위한 가장 원초적이고 강력한 기법은 오류 복구이론의 뼈대인 ’명령의 재전송(Retransmission)’이다.
그러나 똑같은 제어 프레임이 시간차를 두고 여러 번 중복 수신될 경우, 드론 펌웨어는 이것이 조종자가 새롭게 요구한 2회 통제 지시인지, 아니면 단지 과거에 통신 유실을 우려한 송신자가 바보같이 다시 쏘아 올린 네트워크의 복사본 유령인지 판별해야 하는 깊은 식별의 딜레마에 봉착한다. MAVLink 프로토콜은 이 치명적인 명령 중복(Duplication) 수행 문제를 극복하기 위해 제어 명령어 구조체의 꼬리 부분에 확인(Confirmation) 계수기(Counter) 필드를 단일 바이트 규격으로 견고하게 내장하고 있다. 본 절에서는 이 confirmation 필드의 데이터 상태 모델과 통신 동기화에서의 구체적인 역할을 파헤친다.

1. Confirmation 컴포넌트의 데이터 규격과 존재 이유

COMMAND_LONG 밎 COMMAND_INT 구조체의 헤더 정의서를 뜯어보면, 글로벌 라우팅을 관장하는 target_system과 target_component 쌍 바로 직후에 uint8_t confirmation 변수가 구조체 패치워크의 마침표처럼 자리 잡고 있다.

  • 메모리 타입: 부호 없는 1바이트 정수 (uint8_t)
  • 물리적 표현 범위: 0부터 255 사이의 자연수 공간
  • 베이스라인 초기값: 오직 0 (순수한 최초 송신 시)

분산 운영체제(Distributed OS)에서 패킷의 고유성을 증명하기 위해 타임스탬프(Timestamp)나 128비트 해시(Hash) 암호를 떡칠하는 무거운 관행에서 벗어나, MAVLink는 극한의 밴드위스 최적화를 위해 이 단 1바이트 변수 공간만을 할당했다. 이 필드의 임무는 오로지 **“이 동일한 명령어 세트가 GCS 서버 측의 타임아웃 큐(Queue)에 의해 몇 번이나 연속해서 재전송(Retry) 발사되었는가?”**를 지시하는 순수 상태 계수기(Incremental Counter) 역할만을 우직하게 전담하는 것이다.

2. 송신자(QGC 등 GCS) 측의 Confirmation 카운팅 오토마타

QGroundControl과 같은 지상 관제 소프트웨어가 조종자의 개입이나 임무 시퀀스에 의해 새로운 통제 타스크(예: MAV_CMD_DO_CHANGE_SPEED, 속도 변경)를 생성하여 백그라운드 비행 관제 파이프라인으로 흘려보낼 때, confirmation 필드의 생애 주기는 다음과 같은 유한 상태 오토마타(FSM)를 그리며 팽창한다.

  1. 초도 절대 전송 (Initial Transmission): 통제 스레드가 명령 버퍼를 처음 조립(Assemble)할 때, 메모리 상의 confirmation 값은 변수의 여지없이 0으로 하드코딩되어 모뎀으로 전송된다. 이는 무인기에게 “이 명령은 과거의 그 어떠한 재전송 이력도 품지 않은, 완전히 신선하고 독창적인 최상위 1회차 명령 패킷이다“라는 명시적인 신원 선언이다.
  2. 타임아웃 감시 (Timeout Detection): GCS 전송 모듈은 소켓 통신을 벗어난 패킷 전송 직후 즉각적으로 비동기 타이머를 가동한다. 만약 사전에 약속된 임계 타임아웃 윈도우(PX4의 경우 MAVSDK 기준 보통 명령 스텝당 500ms 내외) 내에 상공의 오토파일럿 보드로부터 “명령을 인지했다“는 확달 메시지(COMMAND_ACK)가 역방향 텔레메트리로 안착하지 않으면, GCS는 보냈던 원본 패킷이 대기권에서 증발했거나 수신 측의 ACK가 지상으로 내려오다 물리적으로 격추된 것으로 보수적으로 판정한다.
  3. 오토-리트라이 (Auto-Retry) 펌핑: 타임아웃 큐(Queue)에 매달린 GCS 재전송 루프는 이전 패킷의 페이로드 데이터(param1 ~ param7, target 등)를 일절 훼손하거나 재생성하지 않고, 오직 꼬리에 달린 confirmation 필드의 정수 값만을 0에서 1로 강제 펌핑(Increment) 시킨 뒤 재차 ब्र로드캐스트로 쏘아 올린다.
    만약 이조차도 실패하면 이 수치는 2, 3으로 선형 증가(Linear Increment)의 궤적을 그리게 되며, 펌웨어에 규정된 최대 재전송 인내 제한 횟수(예: 3회 ~ 5회 임계치)에 도달할 때까지 좀비처럼 이 복구 행위를 끈질기게 반복하게 된다.

3. 수신자(PX4) 측의 방어적 중복 제거(Deduplication) 파이프라인

어리석어 보이는 범용 카운터 confirmation 필드가 진정한 소프트웨어 공학적 기지를 발휘하는 최전선은 바로 수신단, 즉 픽스호크 펌웨어 모듈인 mavlink_receiver의 명령어 파싱 파이프라인 심장부이다.

만약 GCS가 쏜 최초 명령(Confirmation = 0)이 픽스호크 보드에 온전하게 도달하여 명령된 모터나 짐벌을 성공적으로 구동시켰으나, 기체가 임무 완수를 축하하며 지상으로 뱉어낸 응답(COMMAND_ACK) 포로토콜만이 하필 내려가던 중 전파 노이즈에 휩쓸려 허무하게 소실되었다고 극단적인 가정을 전개해 보자. 밑에서 발을 동동 구르던 GCS는 응답 없는 드론을 향해 0.5초 뒤 똑같은 페이로드의 명령(Confirmation = 1)을 앙칼지게 재차 쏘아 올릴 것이다.

이 절대 절명의 순간, PX4의 커맨더 스레드는 다음과 같은 방어적 판단 논리를 거쳐 시스템의 멱등성(Idempotence, 몇 번을 수행해도 결과가 동일한 성질)을 결사적으로 사수해 낸다.

  1. 메모리 스냅샷 캐싱 검사: 수신된 패킷의 핵심 식별자인 MAV_CMD 고유 번호, 세부 파라미터 구성 값들, 그리고 발송 호스트인 GCS의 System ID를 끄집어내어, 직전 마이크로초에 이미 처리 완료하여 레지스터 메모리에 임시 캐싱(Caching)해둔 ’가장 최근에 수용한 명령 스냅샷’과 바이트 단위로 즉각 대조 검사를 실시한다.
  2. 클론 식별(Clone Identification): 만약 검사 결과 새로 들어온 내용물이 직전에 수신한 명령어 트랜잭션과 단일 비트 하나 틀리지 않고 쌍둥이처럼 완벽하게 코드가 일치하는데, 오직 꼬리표인 confirmation 필드의 숫자만이 양수(1 이상의 정수)로 펌핑되어 들어왔다면, 똑똑하게 설계된 파서는 이 패킷이 자신이 놓쳐버린 ’새로운 사용자 지시’가 단연코 아님을 확증한다. 이것은 오로지 지상의 GCS가 잃어버린 자신의 ACK 패킷 부재에 극심한 불안감을 느껴 스팸(Spam)처럼 보낸 빈 껍데기 **‘재전송된 복사본(Duplicated Clone Frame)’**임을 단번에 간파하는 것이다.
  3. 명령 폐기 및 안심 응답(Pacifier) 반환 로직: 이 확증 편향에 따라 PX4 수신기는, 무식하게 과거에 이미 완수했던 비행 모드 변경 시퀀스나 카메라 셔터 트리거 펄스를 두 번 연발하여 기계적 관성을 어지럽히는 멍청한 행위를 일체 차단 무효화(명령어 완전 폐기) 시켜 버린다.
    그 대신, 이 애처로운 재전송 컴레인을 지상 GCS가 거두어들일 수 있도록 안심시키기 위한 유일한 조치로서, 이전에 폐기된 응답 패킷과 동일한 COMMAND_ACK (수행 완료 선언) 패킷 하나만을 억지로 강제 위장 생성시켜 UART 통신망을 통해 지상으로 툭 반환해 주는 고도의 지능적인 백-오프(Back-off) 반사 로직을 구사한다.

결과적으로 단 1바이트 사이즈를 소비하는 이 작디작은 confirmation 카운터 모델은, 분산 스웜(Swarm) 시스템 제어망에 있어 기체를 추락시킬 수도 있는 치명적 잠재 위협인 ‘명령의 중복 트리거 수행(Double-executing)’ 현상과 끝없이 서로 기다리는 ’네트워크 데드락(Deadlock)’의 사슬을 시스템 엔드-투-엔드(End-to-End) 트랜잭션 최앞단에서 가위로 잘라버리는 가장 경제적이고 확실한 비동기 동기화(Asynchronous Synchronization) 보안 키로 작동하는 것이다.
