39.1.3. CAP 정리(CAP Theorem) 관점의 네트워크 분할(Network Partition) 대응 전략

분산 컴퓨팅 및 데이터베이스 시스템을 설계할 때 가장 근본적인 수학적 한계를 제시하는 이론이 바로 **CAP 정리(CAP Theorem, Brewer’s Theorem)**이다. 이는 하나의 분산 시스템이 일관성(Consistency), 가용성(Availability), **분할 내성(Partition Tolerance)**의 세 가지 속성 중 두 가지만을 동시에 완벽하게 충족할 수 있다는 정리이다. 지상 관제 시스템(GCS)과 무인기(PX4-Autopilot)가 무선 데이터 링크를 통해 상호작용하는 체계 역시, 물리적으로 분리된 두 개 이상의 노드가 네트워크를 통해 상태를 동기화하는 ’분산 시스템’이므로 CAP 정리의 철저한 지배를 받는다. 본 절에서는 무인기 제어망에서 네트워크 분할(Network Partition) 상황이 발생했을 때 PX4 시스템이 CAP 정리 관점에서 어떠한 전략적 선택을 내리는지 분석한다.

1. 드론 제어망에서의 CAP 속성 정의

무인기-GCS 기반 분산 시스템에서 CAP 모델의 세 가지 축은 다음과 같이 해석된다.

  • 분할 내성 (Partition Tolerance, P): 드론 통신에 있어 P는 ’선택 사항’이 아닌 **‘필연적 현실’**이다. 무선 주파수 환경은 어떠한 고도화된 안테나 트래킹 장비를 사용하더라도 간섭, 지형지물 섀도잉, 거리 증가에 의해 데이터 링크 단절(Data Link Loss), 즉 네트워크 분할(Partition)이 무조건 발생할 수밖에 없다.
  • 일관성 (Consistency, C): GCS가 인지하고 있는 기체의 비행 모드 및 목적지 상태와 실제 PX4가 수행 중인 현재 상태가 한 치의 오차도 없이 동일하게 일치(Synchronized)해야 함을 의미한다.
  • 가용성 (Availability, A): 기체가 GCS와의 통신 상태와 무관하게 시스템 장애를 일으키지 않고 현재 명령받은 임무(Mission)나 비행 제어를 멈추지 않고 지속할 수 있는 능력을 의미한다.

네트워크 분할(P)이 기정사실화된 무선 무인기 환경에서, 시스템은 데이터 링크가 끊어졌을 때(Partition 발생 시) **일관성(C)**을 지키기 위해 기체의 가용성을 포기할 것인가, 아니면 **가용성(A)**을 유지하기 위해 일관성을 희생할 것인가의 중대한 설계 기로에 서게 된다.

2. CP 시스템 vs AP 시스템 설계의 트레이드오프

PX4-Autopilot은 임무의 성격과 사용자 설정(Parameter)에 따라 CP(Consistency-Partition tolerance) 방식과 AP(Availability-Partition tolerance) 방식 사이를 유연하게 오가도록 설계되어 있다.

2.1 CP 설계 우선 (일관성과 안전성 우선 전략)

통신이 단절(P)되었을 때, GCS(조종자)가 기체의 상태를 통제 및 인지할 수 없는 ‘일관성 붕괴’ 상황을 최고의 위험으로 간주하는 설계이다.

  • 작동 방식: PX4는 데이터 링크 손실 시간(NAV_DLL_TO)이 초과하면 즉각적으로 현재 수행 중인 임무(가용성)를 중단하고 안전 장치(Failsafe) 모드로 강제 전환한다.
  • 대표적 대응: 제자리 비행(Hold), 귀환(RTL/Return to Launch), 또는 비상 착륙(Land).
  • 공학적 의의: 기체가 조종자의 통제 및 시야(Consistency) 밖에서 독자적인 행동을 하는 것을 원천 차단하여 물리적 사고를 예방한다. MAVLink 커맨드의 안전(Safety) 설계 철학은 기본적으로 CP 시스템을 지향한다.

2.2 AP 설계 우선 (가용성과 미션 수행 지속 전략)

데이터 링크가 단절되더라도 기체 내부에 탑재된 내비게이터(Navigator) 모듈이나 컴패니언 컴퓨터(ROS2 기반 비전 시스템)의 자율 무결성을 신뢰하고 기존 임무를 밀어붙이는 설계이다.

  • 작동 방식: 데이터 링크 상실 시에도 Failsafe 모드로 진입하는 것을 무시(Disabled)하도록 파라미터(NAV_DLL_ACT = 0)를 설정한다.
  • 대표적 대응: 오프보드(Offboard) 모드에서 온보드 에이전트 주도 하에 목적지까지 계속 비행하거나, 사전에 입력된 복잡한 지도 매핑 미션(Mission)을 통신 없이도 끝까지 완수한다.
  • 공학적 의의: 지하 동굴 탐사, 해저 탐사, 또는 극심한 전파 방해(Jamming)가 예상되는 군사 작전 등 애초에 지속적인 통신(C) 유지가 불가능한 작전 환경에서 기체의 생존성 및 가용성(A)을 극대화한다.

3. 네트워크 힐링(Healing)과 상태 동기화

일시적인 네트워크 분할(P)이 끝난 후 통신이 재개되었을 때, PX4와 QGroundControl(GCS)은 붕괴된 일관성(C)을 재구축하기 위해 신속한 복구 절차를 거행한다.

GCS는 통신 복구 직후 기체의 현재 비행 모드 여부를 평가하고, 이전에 전송했으나 응답(ACK)을 받지 못해 Pending 상태에 머물러 있던 COMMAND_LONG 큐를 비우거나 재평가한다. PX4의 경우, 통신 단절 중에 이미 RTL과 같은 자동 안전 모드로 전환된 상태라면 GCS가 아무리 이전의 조종(Manual) 좌표를 브로드캐스트하더라도 기체는 하드웨어 스위치나 명시적인 비행 모드 변경(COMMAND)이 들어오기 전까지는 Failsafe 상태를 능동적으로 빠져나가지 않음으로써, 사후 사고(Post-partition crash)를 방지한다.

결론적으로 PX4 시스템은 CAP 정리의 한계를 정면으로 인식하고, 파라미터 기반의 커스터마이제이션을 통해 작전 반경과 목적(Safety vs Autonomy)에 맞게 안전한 네트워크 분할 대응 전략을 수립할 수 있도록 고도의 융통성 있는 FSM(Finite State Machine)을 제공하고 있다.
