39.7.2. PX4 ULog 데이터 기반 운영체제 수준 스케줄링 지연(Scheduling Latency) 분석

1. 개요

GCS 텔레메트리 로그 분석을 통해 “네트워크 패킷 자체는 기체로 잘 넘어갔으나, 기체가 대답(ACK)을 비정상적으로 늦게 주거나 씹어버렸다“는 결론에 도달했다면, 이제 수술대는 지상 통제소에서 공중의 드론 내부로 옮겨져야 한다.
드론 내부의 블랙박스인 ULog는 단순한 센서 데이터의 모음이 아니다. 수십 개의 독립적인 스레드(모듈)들이 NuttX RTOS 위에서 CPU 제어권을 두고 다투는 처절한 흔적(Trace)이 밀리초(ms) 단위의 타임스탬프와 함께 새겨져 있는 하드코어 시스템 로그다.

명령어 하나가 텔레메트리 직렬 포트(Serial Port) 인터럽트를 통해 들어와서, uORB 파이프라인을 타고 Commander 모듈에 도달한 뒤, 다시 응답으로 반사되어 나가기까지 걸리는 시간을 **운영체제 수준의 스케줄링 지연(Scheduling Latency)**이라고 부른다. 본 절에서는 비행 제어 소프트웨어 내부에서 발생하는 IPC(Inter-Process Communication) 병목과 스레드 타이밍 결함을 ULog 데이터를 통해 어떻게 발라내는지 디버깅 기법을 소개한다.
