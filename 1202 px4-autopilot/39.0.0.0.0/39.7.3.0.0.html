39.7.3. 물리 및 전송 계층의 패킷 드롭 검출 실무

1. 개요

앞서 GCS의 논리적 큐 상태(State Mismatch)와 PX4 운영체제의 스케줄링 딜레이(Scheduling Latency)를 점검했다. 하지만 이 두 소프트웨어적인 두뇌 영역이 모두 정상임에도 불구하고 명령어가 기체에 도달하지 못한다면, 범인은 명백히 두 시스템을 잇는 허공, 즉 **물리(PHY) 및 전송(Transport) 계층의 패킷 유실(Packet Drop)**이다.

2.4GHz Wi-Fi든, 900MHz 텔레메트리 라디오든, 무선 통신망은 전파 간섭(Interference), 안테나 지향성 상실, 다중 경로 페이딩(Multipath Fading) 등 물리적 제약을 피할 수 없다. 눈에 보이지 않는 무선 전파의 결함을 소프트웨어 엔지니어가 증명하기 위해서는 패킷 수준(Packet-level)의 차가운 데이터 스니핑(Sniffing) 결과물이 필요하다. 본 절에서는 QGroundControl(QGC) 내장 분석기와 Wireshark를 동원하여, 허공에서 갈기갈기 찢겨나간 MAVLink 바이트 스트림의 시체를 부검하는 실무적인 프로토콜 디버깅 기법을 살펴본다.
