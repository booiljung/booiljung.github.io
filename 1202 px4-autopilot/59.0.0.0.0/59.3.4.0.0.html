59.3.4 통신 대역폭 포화(Bandwidth Saturation) 방지를 위한 텔레메트리 스트림의 적응형 전송률 제어(Adaptive Rate Control) 휴리스틱

1. 개요: 텔레메트리 데이터 폭주와 대역폭 포화의 위협

무인항공기(UAV) 지상 관제 시스템(GCS, Ground Control Station)은 비행 제어 장치(FCU, Flight Control Unit)로부터 기체의 자세(Attitude), 위치(Position), 배터리 상태, 센서 원시 데이터(Raw Sensor Data) 등 수많은 텔레메트리 지표를 실시간으로 수신한다. 그러나 433MHz/915MHz 대역의 무선 주파수(RF) 모뎀이나 저속 위성 통신 링크와 같이 대역폭(Bandwidth)이 극히 제한적인 반이중(Half-Duplex) 통신 환경에서, 드론이 생산해 내는 모든 MAVLink 메시지를 최대 주기로 송출할 경우 대역폭 포화(Bandwidth Saturation) 및 병목 현상이 즉각적으로 발생한다.

대역폭 포화 현상이 발생하면 송신 버퍼(Tx Buffer) 초과로 인한 패킷 드롭아웃(Packet Dropout)이 급증하고, GCS 컨트롤 커맨드의 응답 지연(Latency Spike)을 유발하여 치명적인 조종 불능 상태에 빠질 수 있다. 따라서 GCS 고신뢰성 소프트웨어 아키텍처에서는 통신 링크의 물리적 가용 대역폭을 동적으로 추정하고 스트림 송신 주기를 스스로 조절하는 적응형 전송률 제어(Adaptive Rate Control) 휴리스틱 알고리즘 구현이 필수적 구비 조건이 된다.

참고: 이 문서의 기술적 지침은 PX4 펌웨어 v1.14 및 QGroundControl v4.3.0 통신 알고리즘 설계에 기조를 두고 있다.

2. PX4-Autopilot의 스트림 제어 구조 (MAVLink Stream Rate)

Ardupilot 통신 체계와 유사하게, PX4-Autopilot 역시 MAVLink 메시지를 범주(Category)별로 묶어 서로 다른 주기(Hz)로 송출하는 기능을 제공한다. PX4의 내부 mavlink 모듈은 GCS로부터 MAV_CMD_SET_MESSAGE_INTERVAL 커맨드 혹은 REQUEST_DATA_STREAM 메시지를 수신했을 때, 요청된 Hz(주파수)에 맞추어 uORB 토픽 데이터를 폴링 및 직렬화하여 채널로 방출한다.

다만, GCS 개발자가 기체의 상태를 고해상도 그래픽으로 렌더링하고자 모든 센서 원시 스트림(RAW_IMU, HIGHRES_IMU)을 무비판적으로 50Hz 이상으로 요청하게 되면 네트워크 큐가 붕괴한다.

3. 적응형 전송률 제어(Adaptive Rate Control) 휴리스틱 설계론

신뢰성 있는 GCS 애플리케이션 개발을 위한 적응형 전송률 제어 알고리즘은 수신율 계산, 트래픽 감쇠 검사, 그리고 동적 레이트 할당이라는 3단계 휴리스틱 체인을 통해 완성된다.

3.1 무선 링크 품질(Link Quality) 및 패킷 손실률(Packet Loss Rate) 추정

MAVLink 헤더 구조에는 seq (Sequence Number, 0~255) 필드가 포함되어 있다. GCS는 수신되는 모든 패킷의 일련번호를 검사하여 패킷이 중간에 유실되었는지를 판별한다. 만약 패킷이 건너뛰는 현상(current_seq != (last_seq + 1) % 256)이 집중적으로 발생한다면, 이는 수신 단말 고장이 아닌 물리적 대역폭 큐의 오버플로우(Overflow)로 간주할 수 있다.

// 수신된 MAVLink 패킷 손실률 모니터링 코드 개념
float calculate_packet_loss(uint8_t current_seq, uint8_t &last_seq, uint32_t &total_packets, uint32_t &lost_packets) {
    total_packets++;
    uint8_t expected_seq = (last_seq + 1) % 256;
    
    if (current_seq != expected_seq) {
        // 손실 발생 (계산 편의를 위해 순서 뒤바뀜이 아닌 단순 누락으로 판단)
        uint8_t lost_count = (current_seq > last_seq) ? (current_seq - last_seq - 1) : (256 - last_seq + current_seq - 1);
        lost_packets += lost_count;
        total_packets += lost_count;
    }
    last_seq = current_seq;
    
    return (float)lost_packets / total_packets;
}

3.2 대역폭 혼잡 회피 휴리스틱 (Congestion Avoidance Heuristic)

전통적인 TCP의 혼잡 제어(AIMD: Additive Increase Multiplicative Decrease) 알고리즘 논리를 차용하여, MAVLink 네트워크 포화를 제어할 수 있다.

  1. 포화 징후 임계점 (Saturation Threshold): 패킷 유실률이 특정 임계값(예: 5% 이상)을 넘어서거나, GCS 측에서 타임스탬프를 비교하여 왕복 지연 시간(Round Trip Time, RTT)이 임계치(예: 500ms 이상)로 치솟을 때 포화 우려 상태로 판정한다.
  2. 다중 감소 (Multiplicative Decrease): 포화 징후가 감지되면, GCS의 Rate Controller는 기체 측에 중요도가 낮은 데이터 계층(예: 디버그 스트림, 센서 원시 데이터)의 전송률을 대폭(절반 수준) 낮추거나 전송을 끄는 명령(MAV_CMD_SET_MESSAGE_INTERVAL 값을 큰 수로 할당 혹은 -1로 배정)을 즉시 발송한다.
  3. 가산 증가 (Additive Increase): 일정 시간(예: 10초) 동안 패킷 손실이 거의 발생하지 않는다면, 대역폭 여유가 있는 것으로 추정하고 데이터 레이트를 소폭(예: 1Hz씩) 상향시켜 영상 및 짐벌(Gimbal) 제어 데이터 수신의 부드러움을 복구한다.

3.3 필수 데이터의 방어적 전송 (Defensive Transmission of Critical Data)

적응형 대역폭 감쇠가 일어날 때, 무조건적으로 모든 메시지의 주기를 줄이면 안 된다. 시스템 상태 동기화를 위해 절대 포기할 수 없는 핵심 데이터 스트림 집합(Critical Data Set)은 하한선(Lower Bound)을 두어 보호해야 한다.

  • 최우선 보호 채널 (최소 1Hz 보장): HEARTBEAT, SYS_STATUS, BATTERY_STATUS, GLOBAL_POSITION_INT
  • 하향 조정 대상 채널 (가변 허용): ATTITUDE, VFR_HUD, RC_CHANNELS, SERVO_OUTPUT_RAW

4. GCS 개발 실무 적용 지침

자체 GCS를 개발하며 QGroundControl 수준 이상의 관제 신뢰도를 확보하기 위해서는, 초기 연결 수립 단계에서 통신 모뎀의 물리적 스펙(예: 시리얼의 경우 Baud Rate)을 기준으로 삼아 **“이론상 최대 바이트 처리량(Theoretical Max Throughput)”**을 미리 산출해 두는 것이 바람직하다.
초당 처리 가능 바이트(Bps) = (Baud Rate * 0.8) / 10 (스타트/스톱/패리티 비트 오버헤드 감안)

실제 수신되는 초당 트래픽 누적량이 이론상 최대 처리량의 80% 근사치에 도달할 때, 선제적으로 적응형 감쇠 알고리즘을 개입시키는 안전 마진(Safety Margin) 정책을 채택하라. 이러한 동적 유량 제어(Flow Control) 기법은 하드웨어 한계를 넘어서는 극한의 장거리 비행(Beyond Visual Line of Sight, BVLOS) 테스트에서 통신 두절(Radio Loss)로 인한 기체 추락이나 통제권 상실을 방어할 최후의 소프트웨어 보루로 기능할 것이다.
