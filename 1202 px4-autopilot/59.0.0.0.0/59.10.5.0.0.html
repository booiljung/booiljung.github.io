59.10.5 관제 링크 단절 시간(Link Loss Timeout)의 누적 확률 분포 함수(CDF) 분석을 통한 페일세이프(Failsafe) 발동 임계치 산출 및 하트비트 오프셋 보정 필터링

무인항공기(UAV)의 비가시권(BVLOS; Beyond Visual Line of Sight) 비행 및 장거리 임무 수행에 있어서 지상 관제 시스템(GCS)과의 무선 데이터 링크(Datalink) 연속성은 임무 완수와 기체 생존을 결정짓는 핵심 요소다. PX4-Autopilot은 GCS와 기체 간의 통신이 단절되었을 때 이를 감지하고 스스로 귀환(RTL; Return to Launch)하거나 착륙하는 페일세이프(Failsafe) 방어 체계를 갖추고 있다. 본 절에서는 단순한 정적 타임아웃(Static Timeout) 방식의 한계를 극복하기 위해 통신 지연의 누적 확률 분포 함수(CDF; Cumulative Distribution Function)를 분석하여 수학적으로 최적화된 페일세이프 발동 임계치를 산출하는 엔진 기법과, 하트비트 수신 간극을 보정하는 오프셋 필터링(Offset Filtering) 알고리즘을 심도 있게 다룬다.

1. 정적(Static) 링크 단절 임계치의 한계와 확률론적 접근의 필요성

일반적으로 PX4 펌웨어의 COM_DL_LOSS_T 파라미터나 Ardupilot의 FS_GCS_ENABL 연관 타임아웃 시간은 기본적으로 10초(Seconds) 내외의 정적 상수로 하드코딩되어 평가된다.

그러나 LTE/5G 상용망이나 장거리 RF 통신 환경은 다음과 같은 이유로 극심한 비선형적 지연(Latency Jitter)을 유발한다.

  • 기지국 핸드오버(Handover): 셀룰러 통신 기반 관제 시 기지국을 전환하는 수 초 동안 일시적인 블랙아웃이 발생.
  • 다중 경로 페이딩(Multipath Fading): 도심지 비행 시 구조물로 인한 전파 굴절 및 간섭.
  • 패킷 재전송(Retransmission): 링크 품질 저하로 인한 통신 프로토콜 수준의 재전송 대기.

만약 환경적 요인에 의해 MAVLink HEARTBEAT 패킷이 일시적으로 11초 지연되었다고 가정할 때, 타임아웃이 10초로 고정되어 있다면 기체는 통신이 완전히 붕괴되지 않았음에도 불구하고 불필요한 페일세이프(RTL 등)를 발동하여 임무를 강제 중단하게 된다. 반대로 너무 긴 타임아웃을 설정하면 진짜 통신이 두절되었을 때 기체가 장애물로 돌진할 위험이 커진다.

2. 하트비트 도달 시간의 누적 확률 분포 함수(CDF) 분석

최적의 타임아웃 임계점(Threshold)을 산출하기 위해서는 비행 전 테스트나 과거의 비행 로그(ULog, tlog)로부터 MAVLink HEARTBEAT 패킷 간의 도달 간격 시간(Inter-arrival Time, \Delta t) 데이터를 수집하고 통계적으로 분석해야 한다.

\Delta t_{i} = t_{i} - t_{i-1}

이때, \Delta t의 확률 밀도 함수(PDF; Probability Density Function)를 바탕으로 누적 확률 분포 함수(CDF)를 도출한다. CDF 그래프 상에서 P(\Delta t \le T_{timeout}) 의 값이 목표 신뢰도(예: 99.9%)를 만족하는 수치 T_{timeout}을 최적의 페일세이프 임계치로 정의할 수 있다.

  • 예시 환경 분석: 도심지 스웜(Swarm) 비행 데이터 분석 결과, 평균 하트비트 도달 시간은 1초이나 핸드오버 현상으로 인해 롱테일(Long-tail) 분포가 나타났다고 가정하자. CDF 분석 결과 \Delta t가 5초 이내일 확률이 95%, 8초 이내일 확률이 99.9%라면 임무의 크리티컬 수준에 따라 타임아웃 수치를 8.5초로 통계적으로 확정짓는 식이다. 이는 경험에 의존한 ’찍기’식 튜닝을 배제하는 시스템 공학적 설계 기법이다.

3. 하트비트 오프셋 보정 필터링(Offset Filtering) 아키텍처

누적 통계치 기반의 정적 설계뿐만 아니라, **실시간 동적 필터링(Dynamic Filtering)**을 관제 시스템에 도입하여 링크의 불안정한 정도를 실시간으로 계측할 수 있다.

네트워크 지터(Jitter)가 극심할 때는 하트비트가 무작위한 뭉치 단위로 쏟아져 들어오거나 긴 공백을 두며 불규칙하게 도달한다. GCS나 동반 컴퓨터(Companion Computer)에서는 칼만 필터(Kalman Filter) 혹은 이동 평균 필터(Moving Average Filter)를 활용하여 하트비트 도달율(Arrival Rate)의 ’오프셋(Offset)’을 실시간 추정한다.

필터 구동 시퀀스 아키텍처

flowchart TD
    A[GCS 또는 비행 제어기 통신 모듈] -->|MAVLink HEARTBEAT 수신| B[도달 스탬프($t_i$) 기록]
    B --> C{이전 패킷과의 간격 $\Delta t$ 계산}
    C --> D[지터 필터 / 이동 평균 큐버퍼 진입]
    D --> E[통계적 분산(Variance) 및 평균 갱신]
    E --> F{동적 타임아웃 임계치 $T_{dynamic}$ 산출}
    
    C --> G{현 시점 체공시간 - 마지막 수신시간 > $T_{dynamic}$ ?}
    G -->|No| H[정상 상태 유지 (통신 양호)]
    G -->|Yes| I[통신 단절 1차 경고 (Warning)]
    I -->|절대적 최대 임계치(COM_DL_LOSS_T) 추가 초과| J[하드웨어 Failsafe (RTL/Land) 최종 발동]

필터링 로직을 통해 링크 품질이 일시적으로 나빠지는 구간에서는 페일세이프 타임아웃 임계치를 유동적으로 수 초 늘려 불필요한 임무 포기를 방지하고, 네트워크 환경이 매우 쾌적한 상황에서는 임계치를 타이트하게 줄여 예기치 않은 완벽한 단절 시 기체가 더욱 민첩하게 비상 모드로 돌입하도록 튜닝할 수 있다.

4. PX4-Autopilot과 Ardupilot 간의 아키텍처 및 철학 차이

관제 단절(Datalink Loss) 페일세이프를 다루는 로직에 있어서 두 시스템은 모두 MAVLink의 심장 박동(HEARTBEAT, 메세지 ID 0)을 근간으로 삼으나 세부 철학에서 결을 달리한다.

PX4-Autopilot (동적 유연성 지향)

  • 파라미터 기반의 복합 타이머 구조를 선호한다. 일반 텔레메트리 데이터 무선 단절을 의미하는 COM_DL_LOSS_T 파라미터와, 강력한 실시간 오프보드 제어(Offboard Control / 동반 컴퓨터 연동) 시의 단절 타임아웃을 의미하는 COM_OBL_RC_ACT를 철저히 분리하여 운영한다.
  • PX4는 모드 머신(Mode Machine / Commander) 내부 구조가 모듈화되어 있어 외부 ROS2/MAVSDK 에이전트 측에서 GCS를 대신해 통계적(CDF 기반) 페일세이프 주입 로직을 짜넣기가 상대적으로 수월하다.

Ardupilot (직관적 보수주의)

  • 전통적으로 FS_GCS_ENABL 활성화 플래그 아래에서 보수적이고 선형적인 단절 처리를 우선시한다. Ardupilot의 철학은 텔레메트리 연결이 소실되면 조종사의 주관적 개입이 완벽히 차단된 상태이므로, ’복잡한 유예 로직 없이 최대한 빠르게 기체를 회수(RTL)하거나 그 자리에 고정 대기(Hold)시켜 리스크를 소멸시킨다’는 공리주의에 입각해 있다. 따라서 동적인 오프셋 허용보다는 직관적이고 칼같은 타임아웃 차단을 선호하며 산업 현장에서 강한 신뢰를 받아왔다.

5. 결론

무인 시스템(UAS)의 생존망을 결정짓는 페일세이프 임계치(Threshold)의 결정은 단순한 감에 의존한 상수 기입을 넘어서야 한다. 누적 확률 분포 함수(CDF)를 활용한 통신 로그 데이터의 수학적 통계 분석과, 하트비트 도달율 변동을 억제하는 실시간 오프셋 필터링 기술은 지연이 심한 LTE/5G 관제 및 도심형 항공 모빌리티(UAM) 실증 환경에서 그 진가를 발휘한다. 정교하게 산출된 타임아웃 곡선은 돌발적인 비행 임무 강제 종료 트러블을 극한으로 억제하며, 실제 위급 상황에서는 일말의 오차 없이 기체의 페일세이프 보호 체계를 가동하는 지휘 계통의 초석이 될 것이다.
