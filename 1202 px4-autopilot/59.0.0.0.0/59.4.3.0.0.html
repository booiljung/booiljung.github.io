59.4.3. 국소 평면 좌표계(NED/ENU)와 구면 좌표계 간의 비선형 아핀 변환(Non-linear Affine Transformation) 및 투영(Projection) 수학적 모델링

지상 관제 시스템(GCS)과 PX4-Autopilot은 내부적으로 서로 다른 차원과 단위를 갖는 좌표계를 다룬다. GCS 사용자 인터페이스에 표시되는 지도는 위경도(Latitude, Longitude) 기반의 구면 좌표계(Spherical Coordinate)인 반면, 무인기(UAV) 내부의 항법 필터(EKF2)나 속도 제어기(Velocity Controller) 등은 미터(meter) 단위의 직교 좌표계인 국소 평면 좌표계(Local Tangent Plane: NED 또는 ENU)를 기준으로 제어 루프를 고속 연산한다. 따라서 이 두 좌표계를 자유롭게 상호 변환하며, 센서 및 지도 데이터를 융합하기 위한 정밀한 비선형 타원체-평면 변환 수학 모델링의 이해는 GCS 자율 관제 시스템 개발의 핵심 공학적 역량이다. (기준 펌웨어: PX4-Autopilot v1.14 이상, 관제 시스템 기준: QGroundControl v4.3 이상)

1. 국소 평면 좌표계(Local Tangent Plane)의 정의

가상의 수학적 타원체 모델인 WGS84의 특정 지점(일반적으로 이륙 지점, Home Position)을 원점(Origin, 0,0,0)으로 설정하고, 그 지점의 접평면(Tangent Plane)을 생성한 좌표계이다.

  1. NED (North-East-Down): 북쪽(X), 동쪽(Y), 아래(Z)를 향하는 좌표계다. 항공우주 공학의 전통적인 관습을 따르는 PX4-Autopilot 제어기의 표준이다. DOWN은 중력 방향(Gravity Vector)과 일치한다.
  2. ENU (East-North-Up): 동쪽(X), 북쪽(Y), 위쪽(Z)을 향하는 좌표계다. 모바일 로보틱스와 ROS/ROS2 플랫폼의 표준이다. MAVROS나 uXRCE-DDS 브리지를 통해 PX4와 통신하는 외부 GCS/에이전트는 직수신한 NED 좌표를 수학적 변환(부호 반전 및 축 교환)을 거쳐 ENU로 치환해야 한다.

2. 구면 좌표계(WGS84) \rightarrow 국소 평면 좌표계(NED) 변환 (Forward Projection)

MAVLink를 통해 GCS에 수신되는 글로벌 좌표 GLOBAL_POSITION_INT (위도 \phi, 경도 \lambda, 고도 h)를 홈(Home) 기준의 미터 단위 NED 직교 좌표 (x, y, z)로 변환하는 동향 추정 로직은 비선형 함수를 수반한다.

2.1 기준점 (Home Position) 설정

기준점 \phi_0, \lambda_0, h_0가 설정되어야 투영 기준 평면이 확립된다.

2.2 곡률 반경 계산

지구는 완전한 구가 아닌 타원체(Oblate Spheroid)이므로 위도에 따라 곡률 반경이 변한다. WGS84 타원체의 장반경(Equatorial Radius, a)은 6,378,137m, 편평률을 통해 구한 이심률의 제곱(Square of Excentricity, e^2)은 약 0.00669437999이다. 특정 위도 \phi_0에서의 자오선 곡률 반경(Radius of curvature in the meridian, M)과 횡단 곡률 반경(Radius of curvature in the prime vertical, N)은 다음과 같다.

W = \sqrt{1 - e^2 \sin^2 \phi_0}
M = \frac{a (1 - e^2)}{W^3}
N = \frac{a}{W}

2.3 선형 근사를 통한 아핀 변환 (Small Angle Approximation)

반경 50km 이내의 단거리 미션에서는 삼각함수를 단순화한 비선형 아핀 맵핑을 통해 고속 계산이 가능하다. 주어진 \phi, \lambda의 편차를 라디안 단위 \Delta \phi, \Delta \lambda로 취하면, NED 평면 좌표 (X, Y, Z)는 차분 근사(Differential Approximation)에 의해 다음과 같이 도출된다.

X (\text{North}) = (M + h_0) \Delta \phi
Y (\text{East}) = (N + h_0) \cos(\phi_0) \Delta \lambda
Z (\text{Down}) = h_0 - h

이 수학적 변환은 행렬식(Jacobian)으로 표현될 수 있으며, GCS 백엔드의 10Hz 이상의 궤적 렌더링 갱신 시 O(1)의 시간 복잡도로 변환을 처리할 수 있다.

3. 국소 평면 좌표계(NED) \rightarrow 구면 좌표계(WGS84) 변환 (Inverse Projection)

GCS에서 마우스로 클릭하거나 VIO 기반 컴퓨터 비전 추론 모델이 임의의 상대 거리 \Delta x, \Delta y를 이동하라고 제어 명령을 내릴 때, 이 NED 좌표를 타원체의 Lat/Lon 글로벌 좌표로 맵핑(역투영)하여 MISSION_ITEM을 업로드해야 한다. 상기 방정식의 역함수를 도출한다.

\Delta \phi = \frac{X}{M + h_0}
\Delta \lambda = \frac{Y}{(N + h_0) \cos(\phi_0)}
\phi = \phi_0 + \Delta \phi
\lambda = \lambda_0 + \Delta \lambda
h = h_0 - Z

3.1 지구 중심-지구 고정 좌표계 (ECEF) 기반 정밀 변환

항속 거리가 100km를 초과하는 고정익(Fixed-Wing)의 경우 평면 근사 범위를 벗어나 지구 곡률 오차가 기하급수적으로 커진다. 이때는 Geodetic \rightarrow ECEF \rightarrow NED 의 2단계 엄밀 변환(Rigorous Transformation) 수학 모델을 거쳐야 한다. ECEF(Earth-Centered, Earth-Fixed)는 지구 중심을 원점으로 하는 3차원 직교 좌표계이다.

ECEF 로의 변환식:
X_{ecef} = (N + h) \cos\phi \cos\lambda
Y_{ecef} = (N + h) \cos\phi \sin\lambda
Z_{ecef} = [N(1 - e^2) + h] \sin\phi

이러한 로테이션 연산은 회전 행렬(Rotation Matrix, R)과 평행 이동 베이스라인 벡터 결합을 통해 다차원 행렬 대수(Matrix Algebra) 연산으로 GCS 내부에서 구현되어야 한다.

4. GCS 애플리케이션 아키텍처 적용 사례와 C++ 코드 최적화

상용 엔진인 QGroundControl은 이와 같은 지단 표기 및 수론 변환 알고리즘을 수행할 때 QGeoCoordinate 클래스 등의 정적 헬퍼 함수를 제공한다. 독자적인 GCS 개발 시 직접 C++로 연산을 작성할 때의 모범적인 함수 헤더는 다음과 같다.

// 비선형 구면 투영 클래스 싱글톤 구조
class GeoProjection {
public:
    // WGS84 상수를 정의
    static constexpr double WGS84_A = 6378137.0;
    static constexpr double WGS84_E_SQ = 0.00669437999;
    
    // 초기 기준점(Home) 세팅 시 M, N 캐싱 처리 (최적화)
    void initOrigin(double lat0, double lon0, double alt0);
    
    // Forward Transformation
    Eigen::Vector3d geodeticToNed(double lat, double lon, double alt);
    
    // Inverse Transformation
    Eigen::Vector3d nedToGeodetic(double N, double E, double D);
    
private:
    double ref_lat_rad, ref_lon_rad, ref_alt;
    double M_cache, N_cache;
};

여기서 삼각함수 연산 비용(Cost)을 최소화하기 위해 Reference 포인트가 변경될 때마다 M, N 및 \cos(\phi_0)를 캐시(DB 또는 메모리)에 저장하는 것은 고속 연장(예: 드론 100대 동시 관제)을 위한 주요 최적화 기법이다.

5. 결론

국소 평면(NED)과 구면 측지계(WGS84) 간의 양방향 투영 및 아핀 변환은 무인항공기 GCS의 자율성(Autonomy)을 담보하는 필수적인 기초 수학 알고리즘이다. 근사식을 활용한 단순 평면 투영부터 ECEF를 경유하는 광역 엄밀 변환까지의 수학적 모델링을 GCS 응용 계층 내에 강결합함으로써 외부 ROS2 컴포넌트나 항법 궤적 알고리즘(Motion Planning)이 지도 상에서 오차 없이 정확하게 작동하도록 보장할 수 있다.
