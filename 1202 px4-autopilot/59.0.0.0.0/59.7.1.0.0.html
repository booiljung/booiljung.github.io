59.7.1 힉-하이먼 법칙(Hick-Hyman Law)을 적용한 비상 상황 시 조종자 의사결정 소요 시간 최소화 UI 컴포넌트 배치 모델링

무인항공기(UAV) 지상 관제 시스템(GCS)은 정상 비행 시 다채로운 원격 모니터링 정보를 관제사에게 제공하지만, 기체 이상이나 외부 환경 요인으로 인한 비상 상황(Failsafe) 발생 시에는 관제사의 신속하고 정확한 의사결정을 유도할 수 있도록 화면의 정보량과 선택지를 극적으로 통제해야 한다. 이러한 인간-기계 상호작용(HMI) 최적화의 대표적인 인지 심리학적 모델이 바로 힉-하이먼 법칙(Hick-Hyman Law)이다. 본 절에서는 이 수학적 법칙을 PX4-Autopilot 기반의 GCS 비상 응답 UI(User Interface) 설계에 어떻게 적용할 수 있는지 상세히 분석한다.

1. 힉-하이먼 법칙의 개요와 수학적 모델

힉-하이먼 법칙은 사용자가 주어지는 자극(선택지)에 대해 결정을 내리는 데 걸리는 반응 시간(Reaction Time)이 선택지의 개수에 따라 대수적(Logarithmic)으로 증가한다는 인지 공학 원리이다. 이를 수식으로 나타내면 다음과 같다.

RT = a + b \log_2(n + 1)

  • RT (Reaction Time): 관제사가 비상 상황을 인지하고 조작(예: 버튼 클릭)하기까지 소요되는 총 반응 시간이다.
  • n: 관제사에게 주어진 대안적 선택지의 개수이다.
  • a: 기초적인 인지 및 지각 시간 상수(선택지가 0개일 때의 반응 시간)이다.
  • b: 처리해야 할 정보량 1비트당 추가되는 시간 상수이다.

이 수식은 관제 화면에 표시되는 버튼이나 행동 선택지(n)가 많아질수록 인간의 의사결정 소요 시간(RT)이 비선형적으로 증가함을 의미한다. 임무 크리티컬(Mission-critical)한 비행 관제에서 RT의 지연은 기체의 추락이나 2차 사고로 직결될 수 있다.

2. PX4-Autopilot GCS 생태계에서의 문제점과 적용 방식

PX4의 상태 머신(State Machine)은 예외 상황 발생 시 다양한 복구 모드(Return, Land, Kill, Hold, Loiter 등)를 지원한다. 표준적으로 사용되는 QGroundControl(QGC)과 같은 범용 인터페이스는 비상 경고가 발생했을 때 여러 개의 텍스트 경고창과 함께 다양한 대처 버튼을 동시에 나열하는 경향이 있다. 이는 힉-하이먼 법칙에 따르면 조종자에게 탐색 로드를 부과하여 치명적인 지연을 낳는다.

2.1 맥락 인식형(Context-Aware) 메뉴링을 통한 옵션(n) 최소화

힉-하이먼 법칙을 활용한 모델링의 핵심은 선택지의 절대적인 숫자 n을 상황에 맞게 자동으로 축소(Pruning)하는 것이다.

  • 고도에 따른 옵션 축소: 기체가 고도 5m 이하에서 GPS 신호를 상실(GPS Glitch)했을 경우, 시스템은 ‘Return(복귀)’ 명령어를 숨기고 화면에 오직 ’Hold(고도 유지)’와 ’Land(착륙)’라는 2개의 극단적 선택지만을 거대한 컴포넌트로 제시해야 한다. (n = 2)
  • 동력異常(Motor Failure) 시의 옵션: 모터가 정지하여 추력이 상실된 상황에서는 어떠한 비행 경로 명령도 소용이 없다. 이때는 오로지 ’Kill Switch(비행 강제 종료 및 낙하산 전개)’와 같은 단일 버튼(n=1)만을 화면의 중앙에 노출시켜 의사결정 시간(RT)을 a + b에 수렴하도록 설계해야 한다.

2.2 계층적 정보 캡슐화 (Hierarchical Encapsulation)

모든 기능을 한 화면에 나타내는 플랫(Flat)한 UI 구성은 n을 불필요하게 증가시킨다. 힉-하이먼 모델 기반 설계에서는 비상시 당장 필요한 MAVLink 명령(예: MAV_CMD_DO_FLIGHTTERMINATION 또는 MAV_CMD_NAV_LAND)을 최상위 레이어에 즉각 노출시키고, 세부적인 파라미터 재설정(Parameter Configuration)이나 웨이포인트(Waypoint) 재탐색과 같은 복잡한 요소들은 평상시 메뉴 하위로 캡슐화하여 격리해야 한다.

3. 의사결정 소요 시간 최소화 UI 컴포넌트 배치 (Layout Modeling)

선택지의 수를 줄이는 것 외에도 화면 구성의 논리적 배치(Layouting)는 반응 시간의 상수 a 및 b 값을 낮추는 데 기여한다.

  1. 시선 추적(Eye Tracking) 기반 중앙 집중형 배치: 경고 발생 시 화면 구석에 팝업을 띄우는 것이 아니라, 1차 비행 계기(PFD)의 중앙, 즉 관제사의 시선이 가장 오래 머무는 위치(Center Fixation Point)에 비상 오버레이(Emergency Overlay)를 반투명으로 전개하여 탐색 시간(Visual Search Time)을 제거한다.
  2. 색상 및 패턴(Visual Weight)의 활용: 경고 수준이 가장 높은 옵션(예: RTH 버튼)의 명도 대비비(Contrast Ratio)와 폰트 크기를 비정상적으로 키우고 붉은색 계열을 사용하여, 조종자가 여러 버튼 중 하나를 고르는 것이 아니라 ’제시된 버튼의 승인(Confirmation)’만 수행하도록 뇌의 인지 구조를 유도한다.

4. 결론

GCS에서 비상 상황 대처 능력은 관제사의 개인적 역량뿐만 아니라, HMI 인터페이스가 제공하는 정보 제공 방식에 크게 의존한다. 힉-하이먼 법칙에 수학적 근거를 둔 UI 컴포넌트 배치 모델링은 화면상에 노출되는 의사결정 옵션(n)을 동적으로 최소화하고 가장 안전하고 확률 높은 PX4-Autopilot 제어 명령(Return, Land 등)만을 시각적으로 두드러지게 배치함으로써, 혼란스러운 비상 상황에서도 조종자의 반응 속도를 한계치까지 단축시킬 수 있는 필수적인 인지 공학 설계론이다.
