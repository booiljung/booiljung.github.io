59.10. 시스템 보안(Security), 암호학적 통제 및 내결함성(Fault Tolerance) 공학 설계

1. 개요

지상 관제 시스템(Ground Control Station, GCS)과 무인항공기(UAV) 간의 통신 구간은 본질적으로 개방된 무선 채널(Wireless Channel)을 사용하므로, 악의적인 공격자(Attacker)에 의한 도청(Eavesdropping), 메시지 위변조(Tampering), 재전송 공격(Replay Attack), 그리고 서비스 거부(Denial of Service, DoS) 공격에 취약하다.
상업 공역 및 국방 분야에 운용되는 고신뢰성 드론은 통신 및 제어 루프의 마비를 방지하기 위한 체계적인 **시스템 보안(System Security)**과 불의의 하드웨어/네트워크 결함에도 임무를 지속하거나 안전하게 복구할 수 있는 **내결함성(Fault Tolerance)**을 소프트웨어 아키텍처 관점에서 확보해야 한다.

본 장에서는 PX4-Autopilot 및 MAVLink 기반 통신 생태계에서 적용 가능한 암호학적 통제(Cryptographic Control) 메커니즘을 분석하고, 독자적인 GCS 설계 시 고려해야 할 내결함성 공학(Fault Tolerant Engineering)의 핵심 설계 패턴을 다룬다.

기준 시스템 버전 정보

  • PX4-Autopilot: v1.14.x
  • QGroundControl: v4.3.x 이상

2. GCS 기반 드론 시스템 보안 레이어(Security Layer) 아키텍처

드론의 보안 체계는 단일 계층만으로는 보장될 수 없으며, 방어 종심(Defense-in-Depth) 원칙에 따라 크게 세 가지 레이어로 구성된다.

2.1 물리 및 하드웨어 보안 (Hardware Security)

  • 보안 부트로더(Secure Bootloader): 비인가된 펌웨어 플래싱(Flashing)을 방지하기 위해 Bootloader 단계에서 RSA 또는 ECDSA(Elliptic Curve Digital Signature Algorithm) 서명을 검증한다. PX4는 하드웨어 암호화 가속기를 탑재한 픽스호크(Pixhawk) 시리즈의 특정 모델(예: Pixhawk 5X/6X)에서 보안 부팅 모드를 부분적으로 지원할 수 있는 구조를 가진다.
  • 물리적 포트 통제: 비행 중 불필요한 직렬 포트(Serial Port)의 쉘(Shell) 접근을 차단하고, 디버그(Debug) 인터페이스(JTAG/SWD)를 퓨즈 비트(Fuse Bit)를 통해 물리적으로 잠금 처리한다.

2.2 네트워크 통신 보안 (Network Communication Security)

기존 MAVLink 1.0은 단순 체크섬(Checksum)만을 제공하여 사이버 공격에 완전히 무방비 상태였으나, MAVLink 2.0 규격은 보안을 근본적으로 보완하는 서명(Signature) 필드를 채택했다.

  • MAVLink 2.0 메시지 서명(Message Signature):
    메시지 끝에 13바이트의 Signature 블록을 추가한다. 내부적으로는 암호화 해시 함수(SHA-256 상위 48비트)를 기반으로 하여 비밀키(Secret Key)와 타임스탬프(Timestamp) 단위로 서명을 생성한다.
  • 기밀성(Confidentiality) 한계: MAVLink 2.0 기본 서명은 ’접근 통제 및 무결성’만을 보장하며, 데이터 페이로드 자체는 평문(Plaintext)으로 노출된다. 전체 기밀성을 위해서는 Wi-Fi WPA3나 IP VPM, VPN/IPsec과 같은 하위 트랜스포트 레이어의 암호화 캡슐화가 필요하다.
  • 재전송 공격 방어(Replay Attack Prevention): 타임스탬프 기반 체계를 사용하여, 과거의 패킷을 캡처해 다시 전송하는 스푸핑 시도를 원천적으로 무효화한다.

3. GCS 파이프라인의 암호학적 통제 및 내결함성 (Fault Tolerance) 설계

독자적인 GCS는 예상치 못한 공격이나 자연적인 통신 두절, 기체 예외 동작 상태에서 전체 운용 스택이 연쇄적인 붕괴(Cascading Failure)를 일으키지 않도록 내결함성 패턴을 가져야 한다.

3.1 암호학적 키 교환 및 관리 체계

GCS와 기체 간의 MAVLink 2.0 비밀키는 안전하게 교환되어야 한다. 주로 SETUP_SIGNING (#256) 메시지를 통해 초기 바인딩 시 대칭키를 설정하게 되는데, QGroundControl 소스 코드 내에서는 MAVLinkProtocol::setupSigning() 루틴을 통해 로컬 설정 정보에 기반하여 기체에 서명 키를 비대칭/혹은 유선 직렬 환경 하에서 주입하도록 가이드한다. (무선 구간에서 평문으로 키를 교환하는 것은 지양해야 한다.)

3.2 고신뢰성 GCS 구조를 위한 핵심 패턴 (Mermaid 시각화)

비정상적 상태에 대응하기 위한 상태 격리(State Isolation)와 워치독(Watchdog), 예비 회선(Redundant Link)의 구성 아키텍처를 아래 도표로 나타낸다.

3.3 내결함성 확보를 위한 3가지 시스템 원칙

  1. 듀얼 링크 기반 다중화(Redundancy): Telemetry 주파수 밴드(예: 433MHz/900MHz)와 셀룰러(LTE/5G) 망을 동시에 연결하여 MAVLink 데이터를 GCS 단의 커넥션 매니저(인터페이스 기반 LinkManager)에서 병합(Multi-link Merging) 처리한다. 하나의 데이터 링크가 재밍(Jamming)을 받더라도 보조 링크로 명령이 지속된다.
  2. 소프트웨어적 하트비트 워치독(Heartbeat Watchdog): MAVLink #0 HEARTBEAT 메시지의 수신 간격을 모니터링한다. GCS 엔진 내의 타이머 스레드(Timer Thread)는 지정된 타임아웃(Timeout, 보통 3-5초) 초과 시, 사용자에게 시각/청각적 경고를 발동하며 ‘데이터 링크 유실(Data Link Loss)’ 매크로를 실행하여 오퍼레이터가 신속히 복귀(RTL) 또는 비상착륙 명령을 고려하도록 예비 동작을 지원한다.
  3. 가비지 커맨드 거부(Garbage Command Rejection): 암호화 통제가 물리적으로 뚫렸다 할지라도, GCS 내부에서 조종사 입력값의 물리적 타당성(예: 고도 10,000m 목표 설정)을 검증하여 파서를 통과하지 못하게 하는 위생 확인(Sanitize Check) 기능을 도입한다.

4. 결론

시스템 보안과 암호학적 통제는 단순한 부가 기능이 아닌 상업용 무인기 시스템의 본질적 요구사항이다. PX4와 MAVLink 2.0이 기본 제공하는 서명 체계를 바탕으로, GCS 애플리케이션 레벨에서 추가적인 VPN/IPsec 암호화 계층과 워치독 및 다중화를 통한 내결함성(Fault Tolerance) 설계를 구현하는 것은 현대 항공 소프트웨어 공학의 교과서적인 접근 방식이다. 이는 향상된 QGroundControl 변형 빌드 및 독자 시스템 모두에 철저히 적용되어야 한다.
