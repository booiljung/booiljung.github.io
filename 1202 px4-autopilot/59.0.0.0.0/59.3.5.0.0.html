59.3.5 패킷 드롭아웃(Packet Dropout) 보상을 위한 영차 유지(Zero-Order Hold) 및 선형 외삽법(Linear Extrapolation) 기반 상태 추정 보정

1. 개요: 텔레메트리 패킷 드롭아웃의 필연성과 시각적 끊김 현상

지상 관제 시스템(GCS, Ground Control Station)과 비행 제어 장치(FCU) 사이의 무선 데이터 링크는 전자기적 간섭, 지형지물에 의한 다중 경로 페이딩(Multipath Fading), 혹은 안테나 방사 패턴의 사각지대 진입 등 물리적 요인에 의해 빈번한 패킷 유실(Packet Dropout)을 수반한다.

패킷 드롭아웃이 발생하면, GCS 화면 상에 렌더링되는 기체의 3차원 자세(Attitude) 위젯이나 2차원 지도 상의 궤적이 뚝뚝 끊어지며 순간 이동하는 이른바 스터터링(Stuttering) 혹은 제터(Jitter) 현상이 나타난다. 특히, PX4-Autopilot의 고속 비행(예: 고정익 모드) 중 1~2초의 데이터 단절은 지도상 수십 미터의 공백을 의미한다. 이를 소프트웨어적으로 은닉하고 부드러운 사용자 경험(UX)과 안정적인 관제 상황 인식을 제공하기 위해서는 텔레메트리 데이터의 누락 구간을 수학적으로 메우는 상태 추정 보정(State Estimation Compensation) 기법이 요구된다.

참고: 이 항목에 기술된 추정 보정 논리는 QGroundControl v4.3.0의 Vehicle 클래스 좌표 렌더링 방식 및 PX4 v1.14 텔레메트리 송출 주기를 바탕으로 설계된 자체 GCS 개발 지침이다.

2. 상태 보정의 기본: 영차 유지(Zero-Order Hold, ZOH) 기법

가장 단순하면서도 보편적으로 사용되는 데이터 프레임 보간 방식은 영차 유지(Zero-Order Hold, ZOH) 기법이다. 이는 이산 신호 처리(Discrete Signal Processing)의 기초적인 개념으로, 새로운 데이터(MAVLink 패킷)가 도착하기 전까지 이전에 수신된 가장 마지막 데이터를 그대로 유지(Hold)하는 방식이다.

  • 장점: 구현이 지극히 직관적이며, 데이터가 누락되었을 때 GCS 애플리케이션이 NaN(Not a Number) 혹은 Null 값을 참조하여 크래시(Crash)되는 현상을 원천 차단한다.
  • 한계점: 위치나 속도 데이터가 누락되는 동안 기체가 정지해 있는 것으로 표현된다. 드론이 고속으로 이동 중일 때 ZOH를 적용하면, 새로운 패킷이 도착한 순간 기체 아이콘이 목적지를 향해 비정상적으로 ’순간이동(Teleport)’하는 시각적 왜곡을 발생시킨다.

3. 선형 외삽법(Linear Extrapolation) 기반의 궤적 예측

ZOH가 가지는 불연속성의 한계를 극복하기 위해, 제어 공학에서 널리 쓰이는 동역학적 예측 모델인 선형 외삽법(Linear Extrapolation) 알고리즘을 GCS 렌더링 루프에 결합한다.

선형 외삽법은 마지막으로 성공적으로 수신한 위치(Position) 데이터뿐만 아니라 속도(Velocity) 혹은 가속도(Acceleration) 데이터를 함께 활용하여 데이터가 갱신되지 않는 시간(\Delta t) 동안 기체의 미래 궤적을 예측하여 렌더링하는 기법이다. MAVLink의 GLOBAL_POSITION_INT 메세지는 위도, 경도뿐만 아니라 V_x, V_y, V_z (속도 벡터)를 포함하므로, 이를 GCS 단의 예측 추정기(Estimator)에 손쉽게 대입할 수 있다.

3.1 선형 외삽법 수리 모델

2차원 횡축(NED 좌표계의 North/East) 기준으로 패킷 누락 시간 \Delta t 경과 후 예측된 위치 P_{pred}(t + \Delta t)는 다음과 같은 1차 선형 운동 방정식으로 계산된다.

P_{pred}(t + \Delta t) = P(t) + V(t) \cdot \Delta t

여기서:

  • P(t): 마지막 통신 성공 시점의 기체 위치
  • V(t): 마지막 통신 성공 시점의 기체 속도 벡터
  • \Delta t: 마지막 수신 이후 경과된 시간 (GCS 렌더링 루프의 Delta Time 누적)
// GCS 렌더링 엔진 내부의 선형 외삽법 로직 개념 코드
void Vehicle::updatePositionDisplay(double delta_time) {
    time_since_last_packet += delta_time;
    
    // 타임아웃(Timeout) 징후 감지: 패킷 지연이 0.5초를 초과할 경우 외삽 시작
    if (time_since_last_packet > 0.5 && time_since_last_packet < 5.0) {
        // NED 좌표계 기반 선형 예측
        estimated_lat += (last_velocity_x * delta_time) * LAT_TO_METER_CONSTANT;
        estimated_lon += (last_velocity_y * delta_time) * LON_TO_METER_CONSTANT;
        
        moveGcsVehicleIcon(estimated_lat, estimated_lon);
    } else if (time_since_last_packet >= 5.0) {
        // 데이터 유실이 장기화될 경우 외삽 중단 (추정 오차 폭주 방지)
        enterDataLinkLossMode();
    }
}

4. 커스텀 GCS 적용을 위한 하이브리드 보상 전략 (Hybrid Compensation Strategy)

최고 수준의 시각적 안정성을 요하는 상업용 GCS 애플리케이션을 개발할 때는 ZOH와 선형 외삽법을 단독으로 쓰기보다는, 상황에 따라 알고리즘을 전환하는 하이브리드(Hybrid) 보상 모델을 도입해야 한다.

  1. 단기 누락 (0.1 ~ 0.5초): ZOH를 적용하여 컴퓨팅 리소스를 아끼고, 과도한 예측 연산을 억제한다.
  2. 중기 누락 (0.5 ~ 3.0초): 선형 외삽법을 가동하여 기체의 예상 진행 경로를 애니메이션 처리(Dead Reckoning)한다. 이때 UI 상에 궤적이 반투명하게 변하거나 꼬리표(Trail) 색상을 변경하여 사용자에게 현재 위치가 ’추정치’임을 직관적으로 명시해야 한다.
  3. 장기 누락 (3.0초 초과): 외삽법의 오차가 기하급수적으로 누적되므로 추정을 즉각 중지(Clamp)하고 마지막 예측 지점에 기체를 ZOH로 정지시킨 뒤, 붉은색 경고(Link Loss)를 시각화해야 한다.

이러한 단계적 외삽 결합 기법은 통신 대역폭이 불규칙적으로 요동치는 열악한 현장에서 관제사가 시각적 혼란에 빠지는 것을 막고, 기체의 운동학적(Kinematic) 연속성을 모니터 스크린에 온전히 보존하는 핵심 관제 엔지니어링 기술이다.
