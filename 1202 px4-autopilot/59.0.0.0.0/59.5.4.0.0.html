59.5.4 에이스타(A*) 휴리스틱 탐색 알고리즘 기반의 3차원 장애물 회피 경로 탐색 및 그래프 이론(Graph Theory) 적용

도심 항공 모빌리티(UAM) 기체나 지형이 험준한 산악 지역을 비행하는 무인 항공기(UAV)에게 3차원 공간상의 장애물 회피(Obstacle Avoidance) 비행은 생존과 직결된 핵심 기술이다. 단순히 2차원 평면에서의 우회 비행을 넘어서서 시간, 고도, 에너지를 통합적으로 고려하는 3차원 모션 플래닝을 위해서는, 공간을 이산화(Discretization)하고 그래프(Graph) 형태로 변환하여 최적의 노드(Node)를 찾아가는 탐색 이론이 필수적으로 요구된다.

본 장에서는 그래프 이론(Graph Theory)을 기반으로 한 대표적인 경로 탐색 알고리즘인 에이스타(A*) 알고리즘의 수학적 원리를 고찰하고, 이를 활용하여 PX4-Autopilot 및 ROS2 환경에서 3차원 최적 회피 경로를 설계하는 소프트웨어 아키텍처를 상세히 분석한다.

1. 3차원 공간의 그래프 이산화 모델링: 복셀(Voxel)과 옥트리(Octree)

연속적인 3차원 비행 공간 상에서 탐색 공간(Search Space)을 식별하기 위해, 무배수 공간을 기하학적 그리드로 나누는 작업이 선행되어야 한다.

그래프 이론 관점에서 비행가능한 한 지점은 노드(Node)가 되며, 인접한 위치 간의 비행 궤적은 엣지(Edge)가 된다. 3차원 모델링에 널리 사용되는 방식은 다음과 같다.

  • 복셀(Voxel) 그리드: 공간을 동일한 크기의 정육면체(Voxel)로 강제 분할하는 기법.
  • 옥트리(Octree): 장애물이 존재하는 영역만 재귀적으로 세밀하게 분할하여 메모리 최적화를 유도하는 트리(Tree) 계층 구조형 모델링.

이러한 지도 정보(Map Data)는 주로 동반 컴퓨터(Companion Computer)에 설치된 ROS2 상의 OctoMap 패키지를 통해 생성 및 유지보수 된다.

2. 에이스타(A*) 알고리즘과 휴리스틱(Heuristic) 함수 설계

에이스타(A*, A-star) 알고리즘은 너비 우선 탐색(BFS)과 다익스트라(Dijkstra) 알고리즘의 장점을 결합하여 탐색 노드 수를 획기적으로 줄이도록 고안된 최단 경로 탐색 알고리즘이다.

특정 노드 n을 방문할 때의 총 예상 비용 f(n)은 아래의 수식으로 정의된다.

f(n) = g(n) + h(n)

  • g(n) : 시작 노드로부터 현재 탐색 중인 노드 n까지의 실제 누적 비행 경로 비용 (Actual Cost).
  • h(n) : 현재 탐색 중인 노드 n에서 목표 웨이포인트(Target Waypoint)까지의 예상 잔여 거리 비용, 즉 휴리스틱(Heuristic) 함수.

휴리스틱 h(n)을 어떻게 설계하느냐에 따라 탐색 속도와 정확도가 크게 달라진다. 3차원 비행에 주로 적용하는 휴리스틱 거리는 다음과 같다.

  • 맨해튼 거리(Manhattan Distance): 축에 평행하게 비행할 경우(멀티로터의 직교 축 이동).
  • 유클리디안 거리(Euclidean Distance): 대각선 방향 및 최단 직선거리로의 움직일 경우. 실제 UAV 제어 환경에서 주로 채택.

3. PX4-Autopilot 장애물 회피 알고리즘 연동 아키텍처

PX4 프레임워크 단독만으로는 고성능의 A* 등 복잡한 3차원 탐색 알고리즘을 수행하기에 하드웨어(NuttX RTOS 상의 Pixhawk) 연산력이 부족할 수 있다. 따라서 PX4 체계에서는 주로 Offboard 모드와 동반 컴퓨터(Companion Computer, ROS2 탑재) 간의 MAVLink(OBSTACLE_DISTANCE, TRAJECTORY_REPRESENTATION_WAYPOINTS 메시지 등) 연동을 활용한다.

3.1 로컬 플래너(Local Planner) 연동 매커니즘

  1. 센서 입력: Lidar, Depth Camera(VIO) 등 구동. 동반 컴퓨터 내 OctoMap에 포인트 클라우드 적재 및 장애물 위치 인식.
  2. 경로 탐색: 동반 컴퓨터 플래너 노드에서 A* 기반 목적지까지의 무충돌 그래프(Graph) 경로 계산.
  3. 가이던스 송신: 동반 컴퓨터가 PX4의 Offboard 모드 노드로 짧은 분절(Segment) 형태의 안전 웨이포인트나 속도 지령(SET_POSITION_TARGET_LOCAL_NED)을 10~50Hz로 스트리밍.
  4. PX4 제어: PX4 내부 Position Controller가 위치 지령 추종.

4. Ardupilot의 장애물 회피 방식 대비 철학적 차이

두 오픈소스 시스템 모두 강력한 장애물 회피 시스템(OAS, Obstacle Avoidance System)을 갖추고 있다.

  • Ardupilot: 내부 펌웨어 상에 BendyRuler 규칙 기반 로컬 플래너 및 Dijkstra 기반 단순 회피를 구현해 둠으로써 동반 컴퓨터의 개입 없이도 LiDAR나 Proximity 센서의 데이터를 받아 간단한 회피(특히 2D 평면에서의 우회)를 자체적으로 수행하는 것을 적극 지원한다.
  • PX4-Autopilot: 비행 제어기(FCU) 본연의 역할(핵심 안정성 제어 및 추정)을 중시하며, 메모리 집약적인 A* 알고리즘 기동 방안은 ROS2 미들웨어(uXRCE-DDS)와 연결된 외부 에이전트 쪽이 담당할 것을 설계 지향점(Best Practice)으로 둔다.

5. 지상 관제 시스템(QGroundControl) 에서의 통합 모니터링 전략

사용자 중심의 관제를 위해서는, 동반 컴퓨터 내부의 옥트리 데이터 중 핵심되는 정보가 QGroundControl로 MAVLink 경유되어 표출될 수 있어야 한다.
QGC 4.3 이상에서는 기체 프로필과 연계하여 3D 환경을 일부 시각화하거나, 적어도 OBSTACLE_DISTANCE 메시지를 받아 충돌 임박 방향에 방사형 레이더 알럿(Radar Alert) UI를 표출할 수 있도록 아키텍처가 설계되어 있다.

독자적인 GCS를 개발한다면 그래프 탐색 과정에서 생성된 f(n) 값이 일정한 문턱(Threshold, 예: ‘도달 불가’ 상태의 막힌 골목 패널티)에 도달할 경우 타임아웃 경보를 발생시키고 안전 고도 대기(Hold & Climb) 모드로 즉각 전환시키는 강건한 트랜잭션 에러 처리 철학을 HMI에 반영해야 한다.
