59.4.5 다각형 내부 판별을 위한 광선 교차 알고리즘(Ray-Casting Algorithm) 및 R-트리(R-Tree) 공간 인덱싱 기반의 실시간 지오펜스(Geofence) 연산

1. 개요 및 지오펜스의 필요성

무인항공기(UAV)의 비행 영역을 안전하게 제한하고 법적/운영적 비행 금지 구역(No-Fly Zone, NFZ) 침범을 예방하기 위해 지오펜스(Geofence) 시스템은 자율 비행의 핵심 안전망으로 작용한다. 지오펜스는 가상의 다각형(Polygon) 형태인 공간적 3차원 벽 또는 평면 영역을 구성하며, 기체의 현재 위치가 해당 다각형 영역 내외부 어디에 위치하는지 실시간으로 판별해야 한다.

현대의 PX4-Autopilot(펌웨어 v1.14 이상 권장) 및 독자적인 지상 관제 시스템(GCS)에서는 복잡한 형상의 다각형 내부 판별을 수학적으로 해결하기 위한 **광선 교차 알고리즘(Ray-Casting Algorithm)**과, 수백 개 이상의 다중 다각형(Multi-polygon) 연산을 실시간으로 처리하기 위한 R-트리(R-Tree) 공간 인덱싱(Spatial Indexing) 기법을 활용한다. 본 장에서는 자율 비행 관제 시스템의 궤적 및 위치가 지오펜스를 교차하는지 극도의 저지연(Low-latency) 성능으로 산출해내는 알고리즘 아키텍처와 이를 구현해내는 객체 지향적 과정에 대해 심층적으로 분석한다.

2. 광선 교차 알고리즘(Ray-Casting Algorithm) 이론

공간 해석 분야에서 ‘점 인 폴리곤(Point-in-Polygon, PIP)’ 문제를 해결하기 위한 가장 보편적이고 견고한 지오메트리 접근법은 광선 교차 알고리즘이다.

2.1 수학적 원리 (Mathematical Principle)

기체의 현재 좌표 위치 P(x, y) (WGS84 구면 좌표계를 국소 NED 또는 ENU 투영 면으로 변환한 평면 벡터)에서 시작하여 특정 방향(예: X축 정방향)으로 무한히 이어지는 반직선 형태의 가상 광선(Ray)을 투사(Casting)한다고 가정한다.

이 광선이 정의된 다각형의 모든 경계 선분(Edge)과 교차하는 횟수를 시도(Count)하여 다음과 같은 위상수학(Topology) 규칙에 기반하여 판별한다:

  • 교차 횟수가 홀수(Odd) 이면 해당 점 P는 폐쇄된 다각형 내부(Inside) 에 존재한다.
  • 교차 횟수가 짝수(Even) 이면 점 P는 폐쇄 다각형 외부(Outside) 에 존재한다.

2.2 PX4-Autopilot의 지오펜스 구현 차별성 (Ardupilot 비교)

과거 Ardupilot 기반의 소규모 지오펜스 응용 사례의 경우, 연산 부하를 줄이기 위해 단순한 원형 반경(Cylinder 모델) 기반의 판별이나 제한된 수량의 파라미터 기반 선형 판별을 혼합하여 연산하는 방식이 주력이었다.

반면, 현대의 PX4 비행 제어기 및 유연한 MAVLink 마이크로 서비스 통신 아키텍처 환경에서는 GCS(QGroundControl v4.3+ 및 독자 앱)를 기반으로 고도화된 복잡 금지 구역(Complex Inclusion/Exclusion Polygon) 데이터를 FENCE_POINT (MAVLink MSG ID 160) 계열의 프로토콜 트랜잭션을 통해 시스템 내부의 지오펜스 매니저(Geofence Manager) 모듈 측으로 양방향 동기화한다.
PX4 내부 소스 코드 환경(src/modules/navigator/geofence.cpp 참조)은 제한된 NuttX RTOS 스케줄링 환경 하에서도, WGS84 구면 좌표계의 표면 거리 왜곡률 오차를 평면 상에서 허용 범위 내로 빠르게 재배치한 뒤 O(N) 복잡도의 고속 광선 교차 연산을 실시하여 외부 침범 여부를 매우 안정적으로 감시한다.

3. R-트리(R-Tree) 기반 공간 인덱싱을 통한 병합 최적화 전략

도심 항공 교통(UAM)이나 대규모 국책 인프라 주변의 비행 제한 구역을 고려할 시, 수십 내지 수백 개의 개별 다각형으로 구성된 공간 지리 데이터를 다루게 된다. 이 때, 100Hz 이상의 내비게이션 상태 추정 주기마다 매번 전체 다각형의 각 선분에 대하여 투사 교차 연산을 수행하는 것은 시스템의 심대한 CPU 오버헤드를 유발한다. 따라서 이를 해결하기 위한 선행 가속체로서 R-트리(R-Tree) 공간 인덱싱 기법이 도입된다.

3.1 R-트리 운용 구조 체계 (Tree Data Structure)

R-트리는 각 다각형 또는 지오펜스 복합 객체를 물리적으로 완전히 감쌀 수 있는 최소 경계 직사각형(Minimum Bounding Rectangle, MBR) 단위로 그룹화 및 분할하여 계층적인 탐색 트리(Tree)를 형성하는 공간 데이터 구조이다.

  1. MBR 질의 통과: 기체의 현재 2차원 좌표 P 위치가 질의 노드로 입력되며 트리의 루트(Root) 노드부터 탐색이 개시된다.
  2. 배제 루틴 수행(Pruning 방어): 점 P가 특정 가지(Branch) MBR 공간의 임계 영역 외부라면, 해당 노드의 자식 트리 전체(광활한 영역에 흩어진 수많은 복잡 다각형 집합)를 수학적으로 단번에 배제(Prunned-out)할 수 있다.
  3. 리프 분석: 트리 탐색 복잡도를 O(\log N) 단위로 혁신적으로 떨어뜨린 이후, 충돌 위험성이 존재하는 최하단 리프 노드(단일 다각형) 그룹에 한해서만 값비싼 광선 투사 교차 연산을 실시하게 된다.

3.2 지오펜스 공간 질의 파이프라인 아키텍처 설계도 (Mermaid)

4. 자율 비행(ROS2) 시스템 동역학 연동 및 시스템 통합 보안

4.1 ROS2 (Micro XRCE-DDS) 기반 궤적 간섭(Trajectory Interference) 예방 패턴

독자적으로 제작된 GCS 시스템 상의 단일 시각적 모니터링 체계를 넘어, 자율 임무 에이전트를 자동 제어하는 고성능 컴퓨팅 모듈(Companion Computer) 환경에서는 마이크로소프트 ROS2 운영체제의 다이나믹 지오펜싱(Dynamic Geofencing) 연산이 주도된다.
차량의 동역학적 상태 데이터를 실시간 Micro XRCE-DDS 토픽 미들웨어로 구독(Subscribe) 중인 ROS2 백본의 경로 계획(Path Planning) 노드는 미래의 궤적을 계획할 시 앞선 알고리즘 체계를 응용한다. 즉, 예측 최적화된 단위 스플라인(Spline) 경로 선분과 지오펜스 다각형 선분 상호간의 직접 교차 검출(Line-Segment Intersection) 논리를 선제적으로 확인하여, 충돌 불능 회피(Obstacle Avoidance) 로직에 수리적 타당성을 제공한다.

4.2 C++ 코드 내장 광선 교차 검출 핵심 루틴 설계의 (개념적 예제)

다음 소스 코드는 GCS의 백엔드 지오 공간 처리 엔진, 혹은 ROS2 내장형 맞춤형 관제 계층 내 RayCasting 함수를 모듈화 시킨 예제이다.

// 점 P가 주어진 폴리곤 영역(polygon vector) 내부에 존재하는지 수학적으로 판별하는 루틴 (C++17)
struct Point2D { double x, y; };

bool isPointInPolygon(const Point2D& p, const std::vector<Point2D>& polygon) {
    bool isInside = false;
    size_t n = polygon.size();
    
    // 폐쇄형 다각형의 모든 연속된 선분 모서리(i 번째 정점과 j 번째 정점)에 대해 반복 탐색
    for (size_t i = 0, j = n - 1; i < n; j = i++) {
        // 1. 선분의 양쪽 끝 정점의 Y축 범위 고저(Height) 사이에 탐색점 P의 Y좌표가 스캐닝되는지 점검
        if (((polygon[i].y > p.y) != (polygon[j].y > p.y))) {
            // 2. 광선투사 검사: 해당 선분 방정식의 무한 연장선 위에서 점 P가 투사될 X 좌표의 실제 교차점을 수리적 산출
            double intersectX = (polygon[j].x - polygon[i].x) * (p.y - polygon[i].y) 
                                / (polygon[j].y - polygon[i].y) + polygon[i].x;
            
            // 3. 도출된 교차점의 X 좌표가 실제 탐색점 P의 X 좌표축 위치보다 우측에 위치할 경우(광선 통과 검증) 교차 상태 토글 전환
            if (p.x < intersectX) {
                isInside = !isInside;
            }
        }
    }
    return isInside; // True(내부 침범 혹은 속함), False(외부 안전구역) 반환
}

본 챕터의 주 논의 대상인 MBR 노드 계층별 선여과(Pruning) 필터링 트리 메커니즘과 미시적 선분별 광선 교차 해석 판별 알고리즘의 유기적인 하드웨어적/소프트웨어적 결합은, PX4 기반 드론 시스템의 MAVLink 인터페이스망과 GCS 단말 처리부 전 영역에 이중화(Redundancy) 보호기능으로 배치 운영됨으로써 자율 체계 관제 능의 내결함성(Fault Tolerance) 공학 한계를 대폭 끌어올릴 수 있는 핵심 기반 구조가 된다.
