59.4.2. 세계 측지계(WGS84) 타원체 모델과 지구 중력장 모델(EGM96) 기반의 지오이드(Geoid) 고도 편차 보정 알고리즘

지상 관제 시스템(GCS)에서 드론의 고도를 정확하게 3차원 지도에 렌더링하고 비행 임무(Mission)를 계획할 때 가장 큰 혼란과 사고를 유발하는 요소 중 하나는 ’고도(Altitude)의 기준점’이다. GPS 수신기가 제공하는 고도와 기압계가 추정하는 고도, 그리고 사람이 체감하는 해수면 기준 고도는 서로 다른 물리적·수학적 모델을 바탕으로 산출되기 때문이다. PX4-Autopilot 기반의 고신뢰성 GCS를 개발하기 위해서는 WGS84 타원체 모델(Ellipsoid Model)과 EGM96 지구 중력장 모델(Earth Gravitational Model 1996)의 관계를 명확히 이해하고, 이들 간의 고도 편차(Geoid Undulation)를 소프트웨어적으로 자동 보정하는 알고리즘을 구현해야 한다. (기준 펌웨어: PX4-Autopilot v1.14 이상, 관제 시스템 기준: QGroundControl v4.3 이상)

1. 고도의 세 가지 주요 정의

항공기 시스템에서 쓰이는 고도는 측정 방식과 기준면에 따라 크게 3가지로 분류된다.

  1. 타원체고 (Ellipsoidal Height, h): GPS 모듈(GNSS receiver)이 직접 획득하는 원시 고도이다. 지구를 수학적으로 완벽하게 매끄러운 타원체(WGS84 Reference Ellipsoid)로 가정하고, 이 껍질로부터 기체까지의 수직 거리를 의미한다.
  2. 지오이드고 (Geoid Height, H 또는 AMSL): 인간이 일상적으로 사용하는 ‘평균 해수면(Mean Sea Level)’ 기준의 고도이다. 지구 표면의 중력 포텐셜이 동일한 가상의 표면(Geoid)을 기준으로 측정한 고도이다.
  3. 오르토메트릭고 (Orthometric Height): 엄밀한 지형 표면(Topography)부터의 상대 고도인 AGL(Above Ground Level)과는 다르지만, AMSL과 거의 유사한 의미로 사용된다.

문제는 지구 내부의 질량 분포가 불균일하여 실제 중력이 작용하는 표면(해수면, Geoid)이 타원체(WGS84) 공간 상에서 최대 -100m에서 +85m까지 굴곡(Undulation)을 가진다는 점이다.

2. 타원체고와 지오이드고의 수학적 관계

기체(UAV)의 타원체고(h), 지오이드고(H), 그리고 타원체와 지오이드 표면 사이의 편차를 의미하는 지오이드 파고(Geoid Undulation, N) 사이에는 다음과 같은 선형 관계가 성립한다.

h = H + N
H = h - N

  • h: GPS가 출력하는 타원체고
  • H: GCS와 조종사가 확인해야 할 실제 평균 해수면 고도 (AMSL)
  • N: EGM96 모델에서 도출되는 해당 위/경도에서의 편차 값

대부분의 상용 등급 u-blox GPS 모듈은 NMEA 문장을 출력할 때 내부적으로 EGM96 보정을 거친 AMSL 값을 GGA 문장에 포함하여 전송한다. 그러나 PX4-Autopilot의 커맨더(Commander) 또는 MAVLink의 GLOBAL_POSITION_INT를 통해 GCS로 전송되는 alt 필드의 값이 AMSL 기준인지 타원체 기준인지, EKF2(Extended Kalman Filter)의 주 고도 소스(Primary Altitude Source) 파라미터가 기압계(BARO)인지 GPS인지에 따라 참조 프레임이 달라지므로 GCS는 이를 수동으로 캘리브레이션할 수 있어야 한다.

3. EGM96 지구 중력장 모델 보정 알고리즘

GCS에서 웨이포인트를 생성할 때, 개발자는 마우스로 클릭된 스크린의 위치(Lat, Lon) 고도를 AMSL 단위로 입력받아 MAVLink MISSION_ITEM_INT (Message ID: 73) 메시지로 전송한다. 만약 에이전트 측 EKF2가 GPS 타원체고만 신뢰하도록 설정된 특수 환경이라면(혹은 VIO 융합 시), GCS가 N을 계산해 보정한 뒤 명령을 내려야 지면에 충돌방지 또는 비정상적인 고도 펌핑(Pumping) 현상을 방지할 수 있다.

3.1 EGM96 그리드(Grid) 보간 알고리즘 (Interpolation)

수학적 모델인 EGM96은 전 지구를 15분 단위(또는 그보다 정밀한 해상도)의 격자(Grid)로 나눈 룩업 테이블(LUT, Look-Up Table) 파일 형식으로 제공된다(일반적으로 수 MB의 바이너리 파일). GCS는 특정 위도 \phi, 경도 \lambda에서의 편차 N을 계산하기 위해 파일에서 근접한 4개의 꼭짓점을 읽어와 이중 선형 보간법(Bilinear Interpolation) 을 수행한다.

  1. 격자 좌표 식별: 주어진 (\phi, \lambda)를 둘러싸고 있는 4개의 그리드 포인트 (x_1, y_1), (x_1, y_2), (x_2, y_1), (x_2, y_2)를 식별한다.

  2. 이중 선형 보간 수행:
    먼저 X축(경도) 방향으로 두 점의 N 값을 보간하고, 그 결과를 다시 Y축(위도) 방향으로 보간하여 최종 N(\phi, \lambda)을 구한다.

    N \approx \frac{1}{(x_2-x_1)(y_2-y_1)} \begin{bmatrix} x_2-x & x-x_1 \end{bmatrix} \begin{bmatrix} N_{11} & N_{12} \\ N_{21} & N_{22} \end{bmatrix} \begin{bmatrix} y_2-y \\ y-y_1 \end{bmatrix}

  3. 고도 보정 적용: 도출된 N 값을 H = h - N 식에 대입하여 실제 AMSL을 추정한다.

3.2 GCS 내 EGM96 구현을 위한 C++ 코드 모범 사례

QGroundControl과 같은 성숙한 GCS는 위 로직을 내장 라이브러리(예: GeographicLib)를 통해 처리한다. 커스텀 GCS를 위한 C++ 구현은 다음과 같이 파일 I/O 대기시간(Latency)을 최소화하도록 메모리 캐싱을 포함해야 한다.

#include <GeographicLib/Geoid.hpp>
#include <iostream>

class AltitudeCompensator {
private:
    GeographicLib::Geoid* egm96;
    
public:
    // 생성자에서 EGM96 바이너리 모델을 메모리로 로드 (경로: /usr/share/GeographicLib/geoids/)
    AltitudeCompensator() {
        try {
            egm96 = new GeographicLib::Geoid("egm96-5"); // 5분각 해상도 모델
        } catch (const std::exception& e) {
            std::cerr << "최초 지오이드 모델 파일 로딩 실패: " << e.what() << '\n';
        }
    }

    ~AltitudeCompensator() {
        delete egm96;
    }

    // 타원체고(Ellipsoidal Height)를 AMSL(Mean Sea Level)로 변환
    double convertEllipsoidToAMSL(double lat, double lon, double ellipsoidal_h) {
        // 내부 함수가 Bilinear Interpolation을 수행하여 편차 N을 반환
        double N = egm96->operator()(lat, lon);
        return ellipsoidal_h - N; 
    }
};

4. 실무 관제(GCS)에서의 지오이드 문제 해결 전략

GCS 개발자가 지오이드 편차를 이해하지 못하면, 특정 지역(예: 지오이드 편차가 +30m인 지역)에서 드론에 50m 이륙 명령을 내렸을 때 실제 지면 위 고도가 20m 또는 80m가 되는 심각한 충돌(Crash) 시나리오가 발생할 수 있다.

  1. 상대 고도(Relative Altitude)의 적극적 활용: 절대 고도(AMSL)의 불확실성을 피하기 위해 MAVLink 프로토콜의 Home Position 설정 시점에서의 고도를 0으로 설정하는 AGL(지표면 기준 추정 고도) 및 MAV_FRAME_GLOBAL_RELATIVE_ALT (FRAME=3)를 GCS의 주된 임무 명령 프레임으로 사용하는 권장 패턴이 있다.
  2. QGroundControl의 접근 방식 일치: QGC는 일반적으로 무인기에 미션(Mission) 파일을 전송할 때, Relative Altitude 프레임을 디폴트로 선택한다. 이는 지형 표고 데이터 확보 및 지오이드 보정 연계 실패에서 오는 치명적인 항법 오차를 우회하기 위한 시스템 공학적 방어 기법이다.

5. 결론

PX4-Autopilot 기반의 무인기 항법 데이터를 정확하게 삼차원 공간에 위치시키기 위해 WGS84와 EGM96 모델의 차이를 반영한 지오이드 고도 편차 보정 알고리즘 구현은 필수 불가결하다. GCS 애플리케이션에 이중 선형 보간법을 활용한 자체 편차 룩업 모듈이나 GeographicLib를 밀착 통합하여 MAVLink 텔레메트리로 전송되는 고도의 절대적인 무결성(Integrity)을 보장해야 한다.
