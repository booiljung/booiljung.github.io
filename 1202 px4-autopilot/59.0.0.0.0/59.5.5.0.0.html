59.5.5 사진 측량 및 정사영상 취득을 위한 잔디깎기(Lawnmower) 패턴 및 중복도(Overlap) 기반 격자 피복(Grid Coverage) 알고리즘

무인 항공기(UAV) 산업에서 가장 핵심적이고 상업적인 응용 분야 중 하나는 지표면의 형상을 고해상도로 기록하는 사진 측량(Photogrammetry)과 이를 바탕으로 한 정사영상(Orthomosaic) 및 3D 지형 모델 취득이다. 특정 대상 영역(Target Polygon)을 누락 없이, 그리고 후처리 소프트웨어(Pix4D, WebODM 등)가 픽셀 단위의 특징점(Feature Point)을 매칭할 수 있도록 비행 경로를 설계하는 과정을 커버리지 경로 계획(CPP, Coverage Path Planning)이라고 한다.

본 장에서는 CPP 알고리즘 중 가장 널리 쓰이는 잔디깎기(Lawnmower) 패턴의 수학적 설계 원리를 알아보고, 카메라 렌즈의 스펙과 중복도(Overlap)가 궤적 생성에 미치는 영향을 분석한다. 또한 QGroundControl(QGC) 내부의 C++ 아키텍처가 이를 어떻게 처리하여 PX4-Autopilot으로 MAVLink 임무(Mission)를 전달하는지 상세히 기술한다.

1. GSD(Ground Sample Distance)와 광학계 특성 이해

정밀한 사진 측량 임무를 계획하기 위해서는 카메라 렌즈가 지표면을 바라볼 때 한 픽셀이 담아내는 지표면의 실제 길이인 GSD(Ground Sample Distance, 단위: cm/px)를 최우선으로 결정해야 한다.

GSD는 기체의 비행 고도(H), 카메라 센서의 너비(S_w), 초점 거리(f), 이미지 픽셀 해상도 가로(I_w)에 의해 결정된다.

GSD = \frac{H \cdot S_w}{f \cdot I_w}

GSD 값이 작을수록 해상도가 높은 정사영상을 산출할 수 있으나, 더 낮은 고도에서 더 촘촘한 간격으로 비행해야 하므로 배터리 소모와 임무 시간이 증가한다.

2. 종방향 오버랩(Front Overlap)과 횡방향 오버랩(Side Overlap)

사진 측량 프로그램이 여러 장의 사진에서 동일점(Tie Point)을 찾아 파노라마처럼 엮기 위해서는, 인접한 사진끼리 겹치는 면적이 필요하다. 이를 중복도(Overlap)라고 하며, 보통 70~80%로 설정한다.

  1. 종방향 오버랩(Front Overlap): 기체가 진행 방향으로 날아가며 연속으로 찍는 사진 간의 겹침 비율. 이는 카메라 촬영 주기(Trigger Interval) 혹은 촬영 거리 간격에 영향을 미친다.
  2. 횡방향 오버랩(Side Overlap): 한 왕복 경로를 끝내고 옆 차선(Transect)으로 이동했을 때, 옆 차선에서 찍힌 사진과의 겹침 비율. 이는 평행선 사이의 **차선 간격(Line Spacing)**에 영향을 미친다.

QGC 또는 GCS 내부 알고리즘은 사용자가 요구한 GSD와 중복도 퍼센티지를 입력받아 실제 지상에서의 카메라 풋프린트(Footprint) 크기를 구하고, 이로부터 기체가 움직여야 하는 정확한 횡방향 간격을 계산해 낸다.

3. 잔디깎기(Lawnmower) 패턴 격자 피복 알고리즘 설계

CPP를 위한 잔디깎기 알고리즘, 즉 스윕 라인 알고리즘(Sweep Line Algorithm)은 사용자가 지도에 그린 다각형(Polygon)을 평행한 선분들로 채우는 기하학 알고리즘이다.

  1. 기준축 설정: 설정된 비행 각도(Angle)에 맞춰 다각형의 꼭짓점 좌표계를 회전시킨다. 바람의 방향을 고려하여 에너지 소모를 최소화하는 방향으로 설정하는 것이 일반적이다.
  2. 평행선 생성: 횡방향 오버랩 계산식에서 도출된 ’차선 간격’만큼 띄워가며 스윕 라인(Sweep Line)을 긋는다.
  3. 교차점(Intersection) 추출: 스윕 라인과 다각형 변(Edge)이 만나는 교차점을 찾는다.
  4. 웨이포인트(Waypoint) 생성: 이 교차점들을 지그재그 패턴으로 연결하여 연속적인 MAVLink MISSION_ITEM 배열을 생성한다.

4. QGroundControl (QGC)의 측량(Survey) 아키텍처

QGC 소스 코드(v4.3 기준) 내에서 이러한 그리드 커버리지(Grid Coverage) 로직은 프론트엔드 QML 지도 컴포넌트와 백엔드 C++ 클래스의 융합으로 동작한다.

  • src/MissionManager/TransectStyleComplexItem.cc: 잔디깎기 형태의 왕복 차선(Transect) 비행을 일반화한 추상 클래스이다.
  • src/MissionManager/SurveyComplexItem.cc: 카메라 풋프린트, 오버랩, 비행 각도를 기반으로 교차점 수학(Geometry Math)을 연산하고 임무 항목 리스트를 생성하는 실질적인 비즈니스 로직을 포함한다.
  • 사용자가 QGC 지도에서 꼭짓점을 드래그하면, 즉시 백엔드에서 격자 교차점 배열을 재계산하여 QML 뷰포트에 실시간으로 비행 경로 선분(Polyline)을 렌더링한다.

5. 지상 관제 시스템간의 철학 비교 및 PX4 트리거 동기화

  • Ardupilot(Mission Planner) 체계: 전통적으로 서베이 임무 생성 시, 카메라 파라미터 툴이 매우 노출되어 있어 전문 측량사가 렌즈 왜곡률 옵션까지 제어하기 편하게 만들어져 있다.
  • QGroundControl / PX4 체계: 직관적인 모바일 지향의 사용자 경험(UX)을 추구하므로, 복잡한 파라미터는 Camera Calc 위젯에 숨겨두고, GSD나 고도 중심의 슬라이더 조작으로도 안전한 잔디깎기 궤적이 즉시 산출되도록 설계되어 있다.

측량 비행 도중의 사진 촬영은 보통 PX4 펌웨어 측에 거리 기반 트리거 명령인 MAV_CMD_DO_SET_CAM_TRIGG_DIST 메시지를 전달하여 이루어진다. PX4 내부 모듈(e.g., camera_trigger 모듈)은 기체의 GPS 위치가 지정된 거리만큼 변할 때마다 하드웨어 핀(PWM/GPIO)이나 MAVLink IMAGE_START_CAPTURE를 통해 실제 카메라의 셔터를 작동시킨다. 이는 기체의 속도가 바람에 의해 요동치더라도, 일정한 종방향 오버랩을 강제하여 무결점 측량 데이터를 획득하게 해주는 핵심 시스템이다.

6. 결론

잔디깎기 패턴과 중복도 기반의 격자 피복 알고리즘은 비행 제어 및 광학 기하학(Optical Geometry), 컴퓨터 그래픽스(Polygon Intersection) 이론이 융합된 기능이다. 완성도 높은 독자 GCS 애플리케이션을 개발하기 위해서는 GSD 상관관계를 완벽하게 코드로 구현해 내어야 하며, 이를 PX4의 카메라 트리거 시스템과 유기적으로 통신하도록 MAVLink 트랜잭션을 설계해야 한다. 이를 통해 사용자는 직관적인 조작만으로 복잡한 지형의 무결점 3D 데이터를 취득할 수 있다.
