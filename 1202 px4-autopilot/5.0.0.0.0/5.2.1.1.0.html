5.2.1.1. PX4: NuttX POSIX 표준 기반의 진정한 다중 스레드(Multi-thread) 및 모듈화

무인항공기(UAS) 소프트웨어에서 “동시에 여러 일을 처리한다“는 것은 단순한 비유가 아니라 기체의 생존을 직결하는 공학적 조건이다. PX4-Autopilot은 이 동시성(Concurrency) 문제를 해결하기 위해, 펌웨어를 하나의 거대한 함수들의 덩어리(Monolithic)로 짜는 전통적인 방식을 철저히 배격하고 NuttX RTOS 기반의 POSIX(Portable Operating System Interface) 다중 스레드(Multi-thread) 아키텍처를 전면 도입하였다.

0.1 POSIX 호환성이 부여하는 리눅스(Linux) 경험

POSIX는 유닉스 계열 운영체제 간에 애플리케이션의 이식성(Portability)을 보장하기 위한 IEEE 표준 인터페이스다. PX4 개발팀이 임베디드 시대를 호령하던 가벼운 FreeRTOS 등을 제쳐두고 굳이 메모리를 더 차지하는 NuttX를 핵심 커널로 채택한 근본적인 이유는, NuttX가 완벽에 가까운 POSIX 호환성을 제공하기 때문이다.

  • pthreads 기반 스레드 통제: PX4 내의 각 제어기(Controller) 모듈이나 센서 드라이버는 리눅스 서버에서 백그라운드 데몬(Daemon)을 띄우듯, pthread_create()와 같은 친숙한 C/C++ 표준 API를 사용하여 독립적인 프로세스나 스레드로 구동된다.
  • 파일 시스템 중심의 디바이스 제어: POSIX의 철학인 “모든 것은 파일이다(Everything is a file)“가 PX4에도 동일하게 적용된다. PX4 개발자가 GPS나 UART 포트에 접근할 때 복잡한 레지스터 맵을 뒤지는 것이 아니라, /dev/ttyS1 경로를 open()하고 read()/write()하는 방식으로 리눅스의 문법을 그대로 사용한다.

이러한 POSIX 표준화는 수많은 PC 기반 우분투(Ubuntu) 개발자들이 단 하루 만에 임베디드 비행 제어기 코드를 작성하게 만드는 엄청난 진입 장벽 파괴 효과를 낳았다.

0.2 진정한 다중 스레드(Multi-thread) 스케줄링의 위력

Ardupilot이 단일 거대 루프 안에서 내부 스케줄러가 시간을 쪼개 쓰는 협력적 방식에 의존한다면, PX4는 커널 레벨의 선점형(Preemptive) 다중 스레딩을 구사한다.

위 아키텍처 다이어그램에서 보듯, 수십 개의 PX4 모듈은 각각 고유의 우선순위(Priority)를 갖춘 스레드로 존재한다.
SD 카드에 로그를 기록하는 SD Logger 태스크가 병목을 일으켜 CPU를 물고 늘어지더라도 걱정할 필요가 없다. 1 밀리초(ms) 단위의 커널 틱(Tick)이 발생하는 순간, 하드웨어 타이머의 도움을 받은 스케줄러는 로거 태스크를 강제로 재우고(Preempt), 최우선 순위인 센서 스레드(Sensors)나 자세 추정 알고리즘(EKF2)을 즉각적으로 깨운다(Wake-up).

이 진정한 다중 스레딩 아키텍처 덕분에 PX4는 어떤 부가적인 통신이나 페이로드 로직이 돌고 있든 말든 무관하게, 가장 중요한 비행 역학 제어 주기가 1 마이크로초(us)의 오차도 없이 결정론적(Deterministic)으로 실행되는 하드 리얼타임(Hard Real-time) 환경을 담보할 수 있게 되었다.

0.3 모듈형 아키텍처(Modular Architecture)의 이점

NuttX 위의 다중 스레드 환경은 PX4의 코드를 완벽한 스탠드얼론(Standalone) 모듈들의 집합체로 격상시켰다.

전역 변수(Global Variable) 등을 통한 스파게티식 상호 참조를 원천 차단하고 src/modules 경로 하위에 철저히 분산된 각각의 모듈들은, 오직 uORB(유오알비)라는 퍼블리시-서브스크라이브(Pub/Sub) IPC 채널을 통해서만 메시지를 주고받는다.
결과적으로 새로운 옵티컬 플로우(Optical Flow) 회피 알고리즘을 개발하려는 엔지니어는, 코어 시스템의 수백만 줄의 C++ 코드를 분석할 필요 없이, 그저 빈 C++ 파일을 만들어 uORB 토픽(예: vehicle_local_position_s)을 구독(Subscribe)하고 계산하여 새 토픽을 발행(Publish)하는 독립 스레드 모듈 하나만 컴파일하여 얹어주면 끝나는, 극한의 추상화와 유연성을 얻게 되었다.
