5.3.2. 센서 오류에 대한 페일오버(Failover) 및 복원 알고리즘

수백 미터 상공을 고속으로 날아다니는 무인항공기에게 센서 오류는 단순한 ’데이터 누락’이 아니라 물리적인 ’추락(Crash)’을 의미합니다. 대기권의 급격한 난기류, 자기장을 교란하는 거대한 철탑, 그리고 적대적인 전파 재밍(Jamming) 환경과 같은 복합적인 물리적 위협 속에서 비행 제어기(Flight Controller)의 상태 추정기(EKF)는 끊임없이 혹사당합니다.

본 장에서는 센서 시스템의 치명적인 결함이 감지되었을 때, EKF가 어떻게 기체를 포기하지 않고 차선책으로 안전하게 갈아타는지, 그리고 위협이 지나고 센서가 다시 정상화되었을 때 어떻게 원래의 상태로 부드럽게 복구(Recovery)하는지에 대한 전반적인 페일오버(Failover) 및 복원(Reset/Restore) 아키텍처를 다룹니다.

0.1 하드 페일세이프(Hard Failsafe)와 점진적 강등(Graceful Degradation)의 차이

일반적으로 사람들은 ’센서가 고장 났다’고 하면 낙하산이 펴지거나 모터가 꺼지는 비상사태(Emergency Failsafe)를 떠올리기 쉽습니다. 하지만 현대의 상태 추정기(PX4 ECL/EKF2 및 Ardupilot EKF3)는 단일 센서의 오류만으로 기체를 즉시 추락시키거나 임무를 포기하지 않도록 설계되어 있습니다. 이를 **점진적 성능 강등(Graceful Degradation)**이라고 부릅니다.

  • 1단계 (정상 융합): GPS + 기압계 + 나침반 + IMU가 모두 정상적으로 24차원 EKF에 융합되며, 오차 공분산(Covariance) 수치가 매우 낮고 안정적입니다.
  • 2단계 (오류 감지 및 페일오버): 갑작스러운 건물 사이 비행으로 GPS 위성 개수(Satellites count)가 4개 이하로 떨어지거나 HDOP(수평 오차) 수치가 폭등합니다. EKF 모듈은 내부 혁신량(Innovation) 한계치(Threshold) 검사를 통해 GPS 데이터를 “독(Poisonous Data)“으로 규정하고, 융합 큐(Fusion Queue)에서 즉시 퇴출시킵니다.
    이때 기체는 추락하는 대신, 센서 의존도를 IMU(관성항법장치)의 자유 적분과 기압계, 또는 보조 비전 센서만으로 순간적으로 넘기는 페일오버(Failover) 스위칭을 단행합니다.
  • 3단계 (제어 루프의 모드 킥아웃): 만약 GPS가 끊겨 위치와 속도를 도저히 알 수 없게 되면, 커맨더(Commander) 모듈은 현재 비행 모드인 Position 모드(위치 제어)를 강제로 해제해 버리고, 조종사의 스틱 입력을 자세(Attitude)나 고도(Altitude) 제어로 직접 연결하는 하위 모드로 강등(Kick-out)시킵니다.

0.2 센서 타임아웃(Timeout)과 혁신량(Innovation) 감시 체계

페일오버가 발동되는 트리거 팩터(Trigger Factor)는 크게 두 가지입니다.

  1. 물리적 데이터 단절 (Timeout): uORB 톱니바퀴에서 센서 모듈의 퍼블리시(Publish) 틱이 아예 멈춘 경우입니다. 예를 들어 EKF2_GPS_CHECK 파라미터에 의해 지정된 시간(보통 1~2초) 동안 GPS 메시지가 갱신되지 않으면 센서 전원 단절/케이블 단선으로 판정하고 페일오버를 수행합니다.
  2. 수학적 융합 거절 (Innovation Gate Rejection): 데이터는 꾸준히 들어오고 있지만 그 안의 값이 미쳐 날뛰는 경우입니다. EKF 모듈은 각 센서의 혁신 테스트 비율(Innovation Test Ratio)을 매 틱마다 계산합니다. 이 값이 1.0을 초과한다는 것은 “네가 보낸 데이터가 물리학의 뉴턴 운동 역학 법칙을 근본적으로 위배하고 있다“는 수학적 증명이므로, 해당 센서의 데이터 연산(Update)을 지속해서 건너뛰며(Skip) 퓨전을 거부합니다.

0.3 복원(Restoration/State Reset) 메커니즘: 평온을 되찾기까지

센서 장애는 영구적일 수도 있지만, 드론이 전파 음영 지역(다리 밑, 깊은 계곡)을 통과하는 단 몇 초 동안만 발생하는 일시적 교란일 수도 있습니다. 페일오버 이후 다시 센서가 맑은 신호를 보내기 시작한다면, EKF는 기체의 위치를 다시 올바른 궤도 위로 돌려놓아야 합니다. 이를 **인플라이트 리셋(In-flight State Reset)**이라고 부릅니다.

그러나 센서가 다시 돌아왔다고 해서 과거의 오류 난 궤적을 억지로 수렴시키려 하면 칼만 필터의 제어 이득(Gain)이 폭발하여 모터가 튀어 버립니다. (비명 현상)

복원 단계에서 일어나는 PX4 EKF 내부의 안전 조치는 다음과 같습니다.

  1. 검역(Quarantine): GPS 신호가 다시 들어와 수평 오차가 양호해져도, EKF는 즉시 데이터를 융합하지 않습니다. 펌웨어에 할당된 딜레이 파라미터(수 초간의 대기 시간) 동안 데이터가 안정적으로 지속되는지 검역 단계를 거칩니다.
  2. 상태 행렬 초기화(State Vector Override): 검역이 끝난 신뢰할 수 있는 센서 좌표(예: 위/경도)를 이용해, 그동안 나 홀로 누적되어 오차가 커졌던 IMU 기반 추정 좌표를 아예 덮어써 버립니다(Reset).
  3. 공분산 확장 방지: 값이 순간적으로 튀었기 때문에 위치 공분산 성분을 강제로 리셋하여 제어기가 이 순간적인 오차 점프(Jump)를 보고 놀라서 모터를 급격히 틀어버리지 않도록 완충 작용(Bumpless Transfer / Reset Velocity Limit)을 취합니다.

이러한 정교한 차단과 복원의 춤(Dance)은 드론이 악천후와 전파 교란 속에서도 끈질기게 목숨을 부지할 수 있도록 돕는 자율 생존 본능 그 자체입니다. 이어지는 하위 절에서는 드론 비행 중 가장 빈번하게 발생하는 ’비행 위치 소실(GPS 교란)’과 ‘방위각 소실(나침반 교란)’ 상황에서 구체적으로 어떤 전환 로직이 작동하는지 더 세밀한 수준에서 탐구하겠습니다.
