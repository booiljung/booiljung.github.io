5.3.1.1.2. 단일 코어 환경에서의 연산량 최소화 및 예측(Prediction)/업데이트(Update) 분리

본디 24차원의 대형 자코비안 행렬(Jacobian Matrix)을 곱하고 역행렬(Inverse Matrix)을 구해야만 하는 EKF(확장 칼만 필터) 알고리즘은 스마트폰에 탑재되는 AP(Application Processor) 정도는 되어야 원활히 돌아가는 무거운 수학적 괴물입니다.
그러나 비행 제어기 하드웨어는 주로 클럭이 100MHz~400MHz 대에 불과한 단일 코어(Single Core) 마이크로컨트롤러, 즉 ARM Cortex-M4 또는 M7 칩셋에 근간을 두고 있습니다.

이러한 열악한 임베디드 단일 코어 환경에서, PX4의 ECL/EKF2 라이브러리가 어떻게 실시간성(Real-time)을 잃지 않고 400Hz의 빈도로 이 잔혹한 수식을 소화해 내는지, 그 극한의 연산량 최소화 아키텍처를 해부합니다.

0.1 예측(Prediction)과 업데이트(Update)의 비대칭적 분리 아키텍처

EKF 알고리즘 연산량의 대부분은 관측 모델과 공분산 행렬을 곱하고 융합할 때 발생하는 O(n^3) 공간의 계산 복잡도에서 기인합니다. 만약 IMU 데이터가 들어오는 매 400Hz 틱(Tick)마다 이 전체 공분산을 계산하려 든다면, Pixhawk의 CPU 점유율은 300%를 초과하여 즉시 스케줄링 오류가 발생할 것입니다.

따라서 PX4 EKF2는 연산의 사이클을 철저하게 ’가볍고 빠른 단계’와 ’무겁지만 가끔 하는 단계’로 비대칭 분리(Asymmetric Separation) 하였습니다.

  • 상태 예측(State Prediction) 루프 - 초고속 처리 (예: 250Hz ~ 400Hz)
  • 센서 중 가장 빠른 IMU(자이로스코프, 가속도계)의 델타 각도와 델타 속도를 입력받습니다.
  • 여기서는 무거운 행렬 곱셈을 뺍니다. 뉴턴의 운동 방정식과 단순한 회전 행렬 적분 등 **가벼운 차분 방정식(Differential Equation)**만 수행하여 현재의 자세와 위치를 한 발짝 전진시킵니다.
  • 단일 코어의 CPU 자원을 극히 적게 소모합니다.
  • 측정 업데이트(Measurement Update) 루프 - 저주파 처리 (예: 10Hz ~ 50Hz)
  • GPS, 기압계, 지자기 센서 등 주파수가 느린 센서 데이터가 도착할 때만 이벤트 트리거 형태로 비동기 진입합니다.
  • 이 시점에 비로소 타임스탬프 링 버퍼에서 과거로 타임 점프를 수행하여, 그동안 미뤄두었던 오차 공분산(Covariance Matrix) 보정과 칼만 이득(Kalman Gain) 융합 연산을 무겁게(Heavy) 수행해오고 현재로 돌아옵니다.
  • 이 무거운 계산을 1초에 10~20번꼴로 크게 늦추어 스케줄러를 분산시켰기에, 단일 코어 MCU가 숨을 쉴 수 있습니다.

0.2 순차적 자코비안 연산(Sequential Innovation Updates)의 수학적 마법

업데이트 주파수를 낮추었더라도, 24x24 크기의 칼만 필터 오차 공분산 행렬(P)의 역행렬을 단 한 번이라도 통째로 구하려 들면 그 거대한 연산 스파이크(Spike)에 의해 메인 제어 통신(PWM) 출력이 순간 지연될 수 있습니다.

PX4의 EKF2 원작자인 Paul Riseborough는 여기서 순차적 업데이트(Sequential Update) 메커니즘을 동원합니다.

만약 3차원 위치 데이터를 한꺼번에 융합하려면 3 \times 3 행렬의 역행렬을 풀어야 합니다. 하지만, EKF2는 X축, Y축, Z축의 위치 오차가 서로 수학적으로 독립적(Uncorrelated)이라고 강제로 선언(가정)합니다.
이러면 3 \times 3 역행렬 연산 1번 대신, 행렬이 아닌 단순 스칼라(Scalar) 나눗셈 연산 3번으로 분해할 수 있습니다.

// EKF2의 속도 퓨전(Velocity Fusion) 파트 개념적 코드
// 행렬 연산 대신 각 축별(NED)로 순차 융합(Sequential Fusion)을 수행함
for (int i=0; i<3; i++) {
    // 1축 스칼라 칼만 이득(Kalman Gain) 연산 (역행렬 불필요)
    float innovation_variance = _P(k,k) + R; 
    float K = _P_row / innovation_variance; // 단순 나눗셈(Scalar division)
    
    // 1축 상태 벡터 업데이트
    _state += K * innovation_error[i];
    
    // 1축 공분산 행렬 보정
    _P -= K * _P_row;
}

이 엄청난 대수적(Algebraic) 꼼수를 통해, 역행렬 연산을 펌웨어 연산 코드에서 거의 완전히 제거해 버렸습니다.

0.3 코드 단위 최적화 (Symmetric Matrix 등)

또한 코어 연산을 쥐어짜기 위해 공분산 행렬(P)의 성질인 대칭 행렬(Symmetric Matrix) 구조를 극도로 활용합니다. 즉, 행렬의 중앙 대각선을 기준으로 위아래 삼각 영역의 숫자가 똑같다는 수학적 진리를 활용하여, EKF2의 C++ 코드는 아예 나머지 절반의 행렬 계산 자체를 생략해 버립니다. 연산량이 또다시 절반으로 줄어드는 효과를 냅니다.

그리고 이 모든 치열한 수학 변환의 산물은, 수천 줄의 복잡한 C++ 코드가 사람이 직접 타이핑한 스파게티가 되지 않도록 SymPy(순수 파이썬 대수 기호 라이브러리) 스크립트를 통해 사전에 최적화된 C++ 코드로 자동 컴파일(Auto-Generation) 되어 펌웨어 트리에 이식됩니다. 단일 코어 한계를 박살 내며 우아하게 이착륙을 해내는 PX4 드론 궤적의 뒤에는 이토록 지독한 연산량 절감 철학이 숨어 있습니다.
