5.3.2.1. GPS 재밍/스푸핑 환경에서의 관성 항법(INS) 의존 및 Optical Flow 전환 로직 차이

글로벌 위치 파악 장치(GPS/GNSS)는 무인항공기 자율 비행의 근간이지만, 우주 공간에서 2만 km 이상을 날아오는 극도로 미약한 전파 대역(L1/L2)을 사용하기 때문에 의도적인 전자전(Jamming)이나 악의적인 위치 기만(Spoofing) 공격에 가장 취약한 아킬레스건이기도 합니다.

만약 드론이 자동 비행 임무(Auto Mission) 중에 적대적 전파로 인해 자신의 위치가 수km 밖으로 뛰어올랐다고 착각하게 된다면 어떻게 될까요? 본 절에서는 최악의 GPS 환경에서도 살아남기 위한 PX4와 Ardupilot의 단계론적 방어 기제와 비전 센서 대체(Fallback) 로직의 구조적 차이점을 심층 분석합니다.

0.1 스푸핑 감지 및 추측 항법(Dead Reckoning)으로의 1차 도피

GPS 스푸핑 기기는 서서히 좌표를 조작하여 드론을 납치하려고 시도합니다. 그러나 EKF(상태 추정기)는 이를 속절없이 당하지만은 않습니다.

내부의 가속도계(Accelerometer)와 자이로스코프(Gyroscope)를 적분하여 얻어지는 관성 항법 시스템(INS) 기반의 단기 궤적은 짧은 시간 동안 외부 환경의 개입 없이 물리학적으로 절대적인 참값을 가집니다. EKF는 다음과 같이 GPS의 조작을 감지합니다.

  1. 동역학 모순 감지: INS 적분 결과로는 기체가 초당 1m로 전진 중인데, 갑자기 도착한 GPS 데이터가 초당 50m로 움직였다고 보고합니다.
  2. 혁신량 폭발(Innovation Gate Rejection): 위치 혁신 지표(Position Innovation)가 EKF에 설정된 5 표준편차(5-sigma) 이상의 임계점을 강하게 돌파합니다.
  3. GPS 퇴출: 센서 장애 관리자는 이 GPS 신호가 “물리학적으로 말이 안 되는 오염 신호“라고 판정하고 버퍼 융합 리스트에서 즉각 차단합니다.
  4. 관성 항법 의존(Dead Reckoning) 모드 돌입: GPS가 차단된 순간, 제어기는 오직 내부 관성 센서(IMU) 데이터만을 적분하며 눈 감고 허공을 날아가는 (Dead Reckoning) 상태로 돌입합니다.

하지만 저가형 MEMS IMU의 적분 오차 한계상, 수 초(~10초)만 지나도 드리프트(Drift) 누적으로 인해 수평 위치가 무한정 쏠려버리는 치명적인 결점을 안고 있습니다.

0.2 Optical Flow 및 외부 비전(EV)으로의 2차 페일오버 (PX4 vs Ardupilot)

단순한 관성 적분의 시간제한(수 초)이 끝나기 전에, 드론은 어떻게든 다른 대안 닻(Anchor)을 바닥에 내려야 합니다. 이때 기체 하단을 바라보는 광학 흐름(Optical Flow) 센서나 ROS2 컴퓨터 비전 기반의 VIO(Visual Inertial Odometry) 센서로 제어권이 이양됩니다. 양대 펌웨어는 이 구조에서 설계 사상의 차이를 보입니다.

0.2.1 PX4 (ECL/EKF2): 단일 행렬 내의 융합 가중치(Fusion Weight) 스위칭

단일 인스턴스 철학을 유지하는 PX4는 수평 위치(Horizontal Position)를 구성하는 지표(Index)를 하나로 둔 채 센서 소스(Source)만 갈아끼우는 방식을 택합니다.

  • 평소에는 EKF2_AID_MASK 에 의해 GPS 융합만 활성화되어 있습니다.
  • GPS 신호가 Innovation 검사를 통과하지 못해 잘려나가면, EKF2의 상태 매니저는 즉각 내부에 대기 중이던 Optical Flow(속도) 또는 VIO(위치) 퓨전 함수 계층(Fusion Wrapper)을 깨웁니다.
  • 융합 방정식의 변화: 칼만 필터의 측정 업데이트(Update) 방정식에서 H 매트릭스(관측 행렬)의 참조를 GPS 좌표 체계에서 EV(External Vision)의 Local NED 좌표계 차원으로 동적으로 재매핑(Remapping) 해버립니다. 즉, 두뇌는 동일하되 눈을 감고 대신 귀와 촉각을 열어 융합을 지속합니다.

0.2.2 Ardupilot (EKF3): 코어 자체의 교체 (Lane Switch)

다중 인스턴스 중심의 Ardupilot은 특정 센서를 껐다 켰다 하는 복잡한 행렬 전환 연산을 기피하고, 아예 통째로 대체 코어(Lane)로 넘겨버립니다.

  • 이러한 시나리오를 대비해 조종자는 부팅 시점부터 EKF3 Core 0(GPS 전용)과 EKF3 Core 1(Optical Flow 전용)을 동시에 가동해 놓습니다.
  • GPS 재밍이 발생하면 Core 0의 건강 점수는 최악으로 떨어집니다. 셀렉터(Selector)는 단 한 순간의 망설임 없이 비행 제어 루프의 스위치를 꺼내서 아무런 GPS 오염을 받지 않고 조용히 Optical Flow 로만 위치를 유지해 오던 **Core 1으로 물리적 전환(Switching)**을 해버립니다.
  • 행렬을 뜯어고칠 필요가 없으므로 연산상 매우 안정적이나, 각 코어 간의 좌표계(Origin) 원점이 미세하게 틀어져 있을 경우 부드러운 전환(Bumpless Transfer) 오프셋 알고리즘의 최적화가 필수적입니다.

0.3 최종 방어선: 제어 모드의 강등 (Mode Fallback)

만약 기체에 Optical Flow도 없고 VIO도 달리지 않은 순수 GPS 드론이라면 어떻게 될까요?
추측 항법(Dead Reckoning)의 한계 시간이 초과하는 순간 EKF의 위치 분산(Position Variance) 지표가 기하급수적으로 발산합니다.

이때 커맨더(Commander) 트리 상단에서는 Failsafe 조건이 발동하게 됩니다.

  • Position Mode (위치 고정) 불가 판정: 더 이상 제자리에 가만히 있을 수 없습니다.
  • 운영자의 사전 설정(COM_POS_FS_DELAY 등)에 따라 시스템은 강제로 브레이크를 걸고 기압계 기반의 Altitude Hold (고도 유지) 모드로 킥아웃(Kick-out) 되며, 조종사의 스틱을 받아 수동 비행을 유도하거나 심각할 경우 제자리 수직 착륙(Land Mode) 절차로 최종 돌입하게 됩니다.
