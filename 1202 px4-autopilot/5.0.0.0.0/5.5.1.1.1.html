5.5.1.1.1. 내비게이션 상태(Nav State)와 비행 모드(Flight Mode)의 논리적 분리

일반적으로 상용 드론 사용자들은 조종기 스위치를 딸깍거리는 동작 자체를 ’모드 전환’이라고 부릅니다. 하지만 PX4-Autopilot의 아키텍처 내부를 뜯어보면, 사용자의 의도(Intent)인 ’비행 모드(Flight/Custom Mode)’와 시스템이 실제로 집행하고 있는 물리적 상태인 ’내비게이션 상태(Nav State)’가 철저하게 이중화되어 분리되어 있음을 알 수 있습니다.

본 절에서는 이러한 변수 분리 철학이 PX4의 안전성과 모듈 간 결합도(Coupling)를 어떻게 낮추는지 심층적으로 분석합니다.

0.1 설계의 이중화: “원한다고 다 들어줄 수는 없다”

PX4의 철학은 조종사를 절대적으로 신뢰하지 않는 데서 출발합니다.

  • 비행 모드 (Flight Mode / Custom Mode): 이는 조종기가 매핑한 스위치 채널 조작이나, QGroundControl(GCS)에서 보내온 MAVLink SET_MODE 명령 메시지입니다. 즉, “내가 지금부터 이렇게 날고 싶다“는 **사용자의 요구사항(Request)**일 뿐입니다.
  • 내비게이션 상태 (Nav State): vehicle_status uORB 메모리 토픽에 기록되는 이 값은, 커맨더(Commander)가 모든 조건을 검열한 끝에 시스템에 하달한 **최종 승인 상태(Approved State)**입니다.

조종사가 스위치를 쳐서 ’Position 모드’를 요구하더라도, 커맨더가 현재 GPS 위성 개수가 부족하다고 판단하면 이 요구(Flight Mode 변경 요청)를 즉각 기각(Reject)합니다. 스위치는 Position 방향으로 돌아가 있더라도, 실제 시스템의 Nav State는 여전히 Altitude 혹은 Manual 상태에 머물게 되는 논리적 디커플링(Decoupling)이 발생합니다.

0.2 비상상황 시의 오토 펠백(Auto-Fallback) 메커니즘

이 이중 구조의 진가는 비행 도중 센서 사망(Sensor Failure) 사태가 벌어졌을 때 빛을 발합니다.

기체가 정상적으로 Position 모드(Nav State = POSITION)로 순항하고 있다가 강한 전파 방해를 받아 GPS 연결이 끊어졌다고 가정해 보겠습니다.
Ardupilot과 같은 일부 시스템에서는 해당 모드 클래스 내부 로직이 붕괴하며 즉각 롤/피치 제어권을 반환해야 하는 급박한 상황에 처할 수 있습니다.

반면 PX4의 커맨더는 백그라운드에서 매틱(Tick)마다 이 헬스 징후를 감시하고 있다가, GPS 유효성 플래그가 꺼지는 순간 즉시 상태 트리(State Tree)의 하위 단계인 ALTITUDE(고도 유지) 혹은 최후의 수단인 DESCEND(강하)로 내부 Nav State를 강제로 덮어씌워버립니다(Fallback).
사용자의 조종기 스위치는 여전히 Position을 가리키고 있지만, 시스템은 이미 자체적인 수학적 헬스 체크 지표에 따라 안전한 하위 Nav State로 비상 탈출을 완료한 것입니다.

0.3 제어 모듈 관점에서의 완전한 맹목성(Blindness) 부여

이러한 상태 관리의 추상화(Abstraction)는 제어기 코드를 매우 간결하게 만들어 줍니다.
PX4의 위치 제어 모듈(mc_pos_control) 내부에는 “조종기 스위치가 1번 위치인가 2번 위치인가?” 같은 코드가 단 한 줄도 존재하지 않습니다.

제어기 모듈들은 오직 글로벌 버스(uORB)에 떠다니는 vehicle_status.nav_state 변수값 단 하나만 쳐다봅니다.
지금 이 Nav State가 POSITION이 되면 위치 제어 루프를 켜서 궤적을 쫓아가고, AUTO_MISSION이 되면 또 열심히 궤적을 쫓아갈 뿐입니다. 이 내비게이션 상태 명령이 조종사가 내린 것인지, 아니면 커맨더가 GPS 에러로 인해 비상 강등(Fallback) 시켜버린 결과인지 제어기 입장에서는 알 필요조차 없습니다.

이처럼 “사용자 인터페이스(조종기/GCS)“와 “실제 물리 제어 루프” 사이의 끈을 가위로 잘라버리고, 오직 커맨더(Commander)라는 거대한 검열관의 도장이 찍힌 Nav State 문서 하나만으로 시스템 전반을 통제하는 것.
이것이 바로 미 국방성이나 복잡한 자율 비행 로보틱스 생태계가 PX4의 아키텍처를 안전성의 상징으로 꼽아 도입하는 가장 핵심적인 논리 무결성(Logical Integrity)의 결정체입니다.
