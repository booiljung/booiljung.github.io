5.5.1.1.2. 모드 전환 전제 조건(Pre-flight Check) 검증의 중앙 집중화

현대의 무인항공기는 단지 모터가 돌아가는 것만으로는 비행할 수 없습니다. 컴파스의 자기장 간섭부터 EKF2의 속도 공분산(Variance), 배터리의 내부 저항 상태까지 수십 개의 하드웨어/소프트웨어적 지표가 모두 ‘정상(Healthy)’ 판정을 받아야만 안전하게 날아오를 수 있습니다.

PX4 아키텍처는 이러한 방대한 검증 절차를 각 비행 모드나 센서 드라이버 단위로 분산시키지 않고, 커맨더(Commander) 내부의 **PreFlightCheck라는 단일 클래스(Class)로 강력하게 중앙 집중화(Centralization)**시켰습니다.

0.1 PreFlightCheck.cpp의 포괄적 진단 리스트

PX4의 소스 코드 트리에서 src/modules/commander/PreFlightCheck.cpp 모듈은 말 그대로 드론의 이륙과 모드 전환을 승인하는 ‘검문소’ 역할을 수행합니다. 이 검문소에서는 크게 4가지 카테고리의 헬스 체크(Health Check)를 동시에 수행합니다.

  1. 센서 무결성 평가: 가속도계, 자이로스코프, 지자기 센서, 기압계의 캘리브레이션 상태를 확인합니다. 더불어 두 개 이상의 중복(Redundant) 센서가 장착된 경우, 센서 간의 데이터 불일치(Inconsistency)가 허용 오차를 넘지 않는지 검열합니다.
  2. 상태 추정기(Estimator/EKF2) 신뢰성: 센서가 멀쩡하더라도 이를 융합하는 EKF 알고리즘이 발산(Divergence)하고 있다면 추락합니다. 따라서 수평 위치, 수직 속도, 요(Yaw) 각도의 추정 오차(Variance) 척도가 특정 임계치 이하인지 꼼꼼히 검증합니다.
  3. 전력 시스템(Power System) 안정성: 배터리 전압이 너무 낮거나, USB 케이블만 꽂혀 있어 모터를 돌릴 전력이 부족한 상태인지 진단합니다.
  4. 안전망 및 통신(Failsafe & RC/GCS): RC 조종기 신호가 유효한지, 지상관제소(GCS)와의 Heartbeat 통신이 살아있는지, 물리적 안전 스위치(Safety Switch)가 해제되었는지 파악합니다.

0.2 수문장 로직: 조건 미달 시 암(Arming) 및 모드 전환 거부

조종사가 시동(Arming) 스위치를 당기거나, 고도 유지 모드에서 위치 제어 모드(Position)로 진입하려 할 때, 커맨더는 즉시 이 PreFlightCheck 검사표를 꺼내 듭니다.

만약 Position 모드 진입을 요구받았는데 GPS 신호가 충분하지 않아 EKF2의 글로벌 포지션 추정치가 불안정하다면(수평 위치 오차 초과), 커맨더는 이 전환 요청을 단호하게 거부(Reject)합니다. 그리고 다음과 같은 MAVLink 에러 메시지를 QGroundControl 화면에 강제로 띄우고 부저(Buzzer)를 울립니다.
"REJECT POSITION CONTROL: High GPS HDOP or No Global Position"

이처럼 모든 센서 에러와 상태 검열 권한이 단일 파일에 응집되어 있기 때문에, PX4 시스템에서는 “특정 비행 모드에서는 GPS 에러를 무시하고 이륙해버리는” 식의 아키텍처적 결함이 구조적으로 발생할 수 없습니다.

0.3 유연성의 확보: 파라미터(COM_*)를 통한 예외 처리

중앙 집중화된 검열 시스템은 매우 안전하지만, 가끔은 실내 비행이나 연구 목적의 테스트 비행 시 치명적인 제약(Bottleneck)이 되기도 합니다.

이를 극복하기 위해 PX4는 소스 코드를 수정하지 않고도 검열 수위를 낮출 수 있도록 COM_ARM_* 접두사가 붙은 수많은 튜닝 파라미터를 제공합니다.

  • COM_ARM_WO_GPS: GPS 없이도 수평 및 고도 제어 모드 진입을 허용(Allow arming without GPS).
  • COM_ARM_MAG_STR: 지자기 센서의 자기장 간섭 검사 임계치 완화.
  • COM_ARM_CHK_BAT: 배터리 검열 로직 무시.

조종사나 개발자는 펌웨어를 재빌드하는 대신 통제된 파라미터 값 변경만으로 중앙 수문장의 검사 기준표를 특정 임무에 맞게 동적으로 편집할 수 있습니다.
이처럼 **“규칙의 집행은 무조건 중앙에서 단일화하되, 그 규칙의 엄격함은 외부 파라미터 설정으로 유연하게 넘겨주는 것”**이 PX4가 엔터프라이즈급 안정성을 확보하는 아키텍처 설계의 정수입니다.
