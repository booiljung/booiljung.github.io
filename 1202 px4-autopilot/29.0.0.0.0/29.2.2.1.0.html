29.2.2.1. 선형 보간법(Linear Interpolation)을 이용한 원시 통신 신호의 유효 제어 범위 실수형 정규화 코드 (apply_calibration() 함수)

본 소절에서는 ManualControl 클래스의 가장 기저에 위치한 핵심 수학 연산 유틸리티인 apply_calibration() 함수의 C/C++ 소스 레벨 구현을 해부한다. 이 함수는 1000 \sim 2000\mu s 대역표를 가지는 정수(Integer) 형태의 원시 PWM(Pulse Width Modulation) 시간 길이 값을, PX4 제어기가 직접 취급할 수 있는 무차원(Dimensionless)의 정교한 -1.0 \sim +1.0 부동 소수점(Floating-point) 형식으로 프로젝션(Projection)하는 선형 보간(Linear Interpolation) 캘리브레이션 알고리즘을 담고 있다.

1. 다원화된 하드웨어 규격을 통합하는 선형 스케일링(Linear Scaling)의 수학적 당위성

시장에 존재하는 수많은 조종기 제조사(Futaba, FrSky, TBS, Spektrum 등)는 제각기 다른 짐벌(Gimbal) 가변 저항의 전기적 가동 범위(Travel Range)를 가진다. 어떤 송신기의 최하단 스로틀 값은 988\mu s 로 계측되는 반면, 다른 송신기는 1015\mu s 에서 제한되기도 한다. 만약 이러한 제조사별 편차(Variance)를 보정 없이 제어기 행렬(Control Matrix)에 직접 입력할 경우, 조종자가 스틱을 완전히 끝까지 밀었음에도 기체의 기구학적 최대 출력이 100%를 달성하지 못하거나(Under-shoot), 혹은 중앙에 두었음에도 기체가 한쪽으로 미세하게 쏠려 흐르는 치명적 기하학적 인지 부조화가 발생한다.

PX4-Autopilot은 이 막대한 하드웨어 파편화 문제를 해결하기 위해, 각 기체마다 사용자 캘리브레이션 프로세스(QGroundControl 활용)를 거쳐 도출된 세 가지 마일스톤(Milestone) 파라미터 변수 무리를 플래시 메모리에 시스템 오프셋으로 각인해 둔다.

  • RCx_MIN: 조종 스틱 물리적 최하단(Minimum)의 마이크로초 값 (예: 1000\mu s)
  • RCx_MAX: 조종 스틱 물리적 최상단(Maximum)의 마이크로초 값 (예: 2000\mu s)
  • RCx_TRIM: 손을 놓았을 때 스프링에 의해 복원된 기계적 중앙(Center)의 마이크로초 값 (예: 1500\mu s)

apply_calibration() 함수는 이 세 가지 절대 지표를 활용하여, 시시각각 변동하며 유입되는 현재 스틱 위치값(raw_value)을 비율적(Ratio)인 실수 공간으로 맵핑하는 1차 함수(Linear Function) 방정식의 해를 구하는 커널 모듈이다.

2. apply_calibration() 함수의 C++ 소스 아키텍처 및 구간 분할(Piecewise) 보간 로직

전기적 잡음으로 인해 원시 값이 최소/최대 임계치를 관통해 들어오는 이상 현상(Out-of-bound)을 방어하고, 논리적 영점(Trim)을 기준으로 좌우(상하) 대칭적인 거동을 철저히 보장하기 위해 코드는 단일 수식이 아닌 중앙 트림을 기점으로 쪼개진 2-세그먼트 구간 분할 선형 보간(Piecewise Linear Interpolation) 기법을 채택하고 있다.

// src/lib/mathlib/math/filter/LinearInterpolation.cpp 혹은 ManualControl 내부 구현체의 개념적 소스 분석
float apply_calibration(int16_t raw_value, int16_t value_min, int16_t value_center, int16_t value_max, uint16_t deadzone)
{
    float result = 0.0f;

    // 1. 역전파 방어를 위한 Min/Max 범위 내 강제 클램핑(Clamping)
    raw_value = math::constrain(raw_value, value_min, value_max);

    // 2. 데드존(Deadzone) 필터링 기법: 중앙 트림 주변의 기계적 잡음 평탄화 역학
    if (abs(raw_value - value_center) <= deadzone) {
        return 0.0f; // 중심 잔떨림을 완벽한 양자 제로(0.0) 상태로 흡수시켜 의도치 않은 출력 축적 방어
    }

    // 3. 구간 분할 선형 보간(Piecewise Linear Interpolation) 연산
    // 현재 들어온 원시 값(raw_value)의 위치가 중앙 트림을 기준으로 어느 사분면에 존재하는가?
    if (raw_value > value_center) {
        // 물리 스틱이 상단(오른쪽)으로 꺾인 상태: 0.0 ~ +1.0 구간 맵핑
        // 분모: 데드존의 경계선부터 최고점(Max)까지의 순수 유효 행정 거리(Effective Travel Limit)
        // 분자: 현재 값이 데드존 경계선을 벗어난 순수 이동 변위량
        int16_t travel_limit = value_max - (value_center + deadzone);
        int16_t travel_actual = raw_value - (value_center + deadzone);
        
        result = (float)travel_actual / (float)travel_limit; // 비율 도출
        
    } else if (raw_value < value_center) {
        // 물리 스틱이 하단(왼쪽)으로 꺾인 상태: -1.0 ~ 0.0 구간 역방향 맵핑
        int16_t travel_limit = (value_center - deadzone) - value_min;
        int16_t travel_actual = (value_center - deadzone) - raw_value;
        
        result = -(float)travel_actual / (float)travel_limit; // 음수 비율 도출
    }

    // 최종 결과 반환 (이론적으로 엄격히 -1.0f 와 1.0f 사이에 존재함)
    return math::constrain(result, -1.0f, 1.0f);
}

위 C++ 로직은 단순한 나누기 연산을 넘어선 공학적 방어 기재의 집약체이다. 첫째로, math::constrain 함수를 전진 배치하여 온도 변화나 배터리 전압 강하로 인해 조종기 저항 값이 한계치를 초과 송출하더라도 연산 결괏값이 기체 한계인 \pm1.0을 돌파하여 기구학 모듈 내 오버플로우(Overflow)를 치명적으로 유발하는 사태를 수학적으로 사전 색출한다.

둘째로, 보간 연산의 사상 분모를 단순히 value_max - value_center 로 취하지 않고, 반드시 데드존(deadzone) 길이를 감산한 ‘유효 행정 거리(Effective Travel Limit)’ 로 정의한다. 만약 이 보정식을 생략한다면 스틱이 데드존을 간신히 탈출하는 바로 그 찰나의 지점에서 출력값이 0.0이 아닌 약 0.05 따위의 값으로 계단식 도약(Step-jump)을 일으키게 되는데, 이러한 불연속성(Discontinuity)은 PID 제어기의 미분(D) 항을 크게 진동시키는 불안 요인으로 직결된다. 연속성과 미분 가능성을 중시하는 PX4 커널 수학 라이브러리의 치밀한 계산 철학이 여기에 고스란히 담겨 있다.
