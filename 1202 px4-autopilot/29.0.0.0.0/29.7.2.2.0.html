29.7.2.2. 자유 낙하(Freefall) 및 역방향(Inverted) 기동 감지: 중력 가속도 벡터 변화를 감지하여 일시적으로 적분 제어항 누적을 멈추는 동적 제어 이득(Gain) 스케줄링 구현부

무인항공기(UAV)의 자세 및 위치 제어 시스템은 “기체가 정상적인 중력장 위에서 부력(Thrust)을 발휘하고 있다“는 대전제를 바탕으로 PID 선형 피드백 제어기를 작동시킨다. 그러나 모터 고장, 조종사의 급격한 스로틀 차단, 혹은 강한 하강 기류로 인해 기체가 자유 낙하(Freefall) 상태에 빠지거나, 물리적 외력에 의해 기체가 완전히 뒤집히는 역방향(Inverted) 자세가 되면 제어기의 기본 가설이 무너진다.

이러한 상태에서 제어 오차(Error)를 보상하기 위해 적분기(Integrator)가 계속 동작하게 되면, 모터가 복구되었을 때 감당할 수 없는 수준의 누적 에러로 인해 기체가 발산(Windup)하게 된다. 본 장에서는 PX4-Autopilot의 센서 퓨전 및 제어 인프라가 중력 가속도 벡터를 어떻게 모니터링하여 자유 낙하 상태를 인지하고, PID 제어기의 동적 이득(Gain)을 스케줄링하는지 코어 소스 코드를 통해 분석한다.

1. 중력 가속도 벡터 변화 감지 아키텍처

자유 낙하 궤적 하에서 기체의 IMU(Inertial Measurement Unit) 가속도계가 측정하는 벡터의 크기(Norm)는 중력 상수(9.81m/s^2)에서 영(0m/s^2, 무중력)으로 수렴한다. PX4의 EKF2(Extended Kalman Filter) 및 sensor_combined 처리를 담당하는 모듈은 이 스칼라 값을 지속적으로 산출한다.

a_{norm} = \sqrt{a_x^2 + a_y^2 + a_z^2}

자유 낙하 판별기(Freefall Detector)는 단순히 한 주기(Tick)의 a_{norm}이 작다고 해서 자유 낙하로 판별하지 않는다. 내부 링 버퍼(Ring Buffer)와 타이머(Timer)를 두어, 특정 임계값(COM_Z_FREEFALL 파라미터 등) 미만의 가속도 벡터가 일정 시간 이상 지속될 때만 vehicle_status uORB 메시지 내부에 자유 낙하 플래그(Freefall Flag)를 활성화시킨다.

    A[IMU 가속도계 센서 데이터 스트림] --> B{a_norm 계산 및 LPF 필터링}

    C -- No --> E[타이머 리셋 및 정상 상태 유지]
    D --> F{지속 시간 > 판단 임계 시간?}
    F -- Yes --> G[Freefall Flag 설정 (vehicle_status 발행)]
    F -- No --> B

2. 역방향(Inverted) 자세에서의 벡터 부호 감지

기체가 피치나 롤 기준 \pm 90^\circ를 초과하여 뒤집힐 경우, 기체 좌표계(Body Frame) 기준 Z축 센서가 계측하는 중력 벡터의 부호가 역전된다.

안정화(Stabilized) 모드와 같이 순방향 중력을 전제로 하는 제어 모드에서 역방향 자세가 인지되면, 추력을 높일수록 기체는 지상으로 곤두박질치게 된다. 따라서 제어 코어는 쿼터니언 회전 행렬의 Z축 코사인 성분(비행체 하단부를 향하는 벡터)이 음수(-)로 떨어지는 순간을 캐치하여, 고도 제어기(Altitude Controller)가 스로틀(PWM 출력)을 강제로 최소화(Idle)하도록 개입한다.

3. 적분항 누적 동결 (Anti-Windup Mechanism) 구동부

자유 낙하나 역방향 상태가 시스템에 의해 인지되었다면, PX4-Autopilot의 src/lib/pid 라이브러리와 자세 및 속도 제어기(mc_att_control, mc_pos_control)는 오차 복구를 일시적으로 포기하고 시스템이 다시 정상 중력장에 복귀할 때까지 대기해야 한다. 여기서 핵심이 **비선형 동적 제어 스케줄링(Anti-Windup)**이다.

// src/lib/pid/PID.cpp 내의 적분기 동작 분기문 의사 코드
void PID::setIntegration(bool enabled) {
    _integration_enabled = enabled;
}

// src/modules/mc_att_control/AttitudeControl.cpp 컨트롤러 평가 단계
if (vehicle_status.freefall || vehicle_in_inverted_state) {
    // 자유 낙하 상태이거나 기체가 뒤집혀 정상적인 스러스트 제어가 무의미할 때
    rate_control.setIntegration(false); // PID I항 누적 정지 (Freeze)
} else {
    // 정상 비행 영역으로 들어오면 다시 적분기 활성화
    rate_control.setIntegration(true);
}

이 코드는 매우 직관적이고 강력하다. 제어 오차(error = SP - PV)가 막대하게 발생하더라도 제어기가 이를 I항에 우겨넣지 못하도록 원천 차단하는 것이다. Ardupilot 시스템 역시 Anti-Windup 루틴이 백그라운드에서 동작하지만, PX4는 각 모듈 단위의 컨트롤러 클래스가 인스턴스화 될 때 상태 메시지(vehicle_status)를 개별적으로 구독(Subscribe)하여 자율적으로 적분기를 온오프(On/Off)하는 분산형 안전 아키텍처를 지닌다. 이는 특정 제어기 객체(예: 외장 페이로드 김벌 컨트롤러)는 낙하 중에도 적분 제어를 계속 유지하고, 비행 코어만 정지시키는 식의 고도화된 스케줄링을 가능하게 한다.

4. 제어 이득(Gain) 스케줄링을 통한 시스템의 탄성 회복

기체가 자유 낙하에서 회복되어 다시 프로펠러가 공기를 채기 시작할 때(Recovery Phase), 갑작스럽게 적분기가 활성화되면 제어 출력이 튀는 현상(Bumpless Transfer 실패)이 나타날 수 있다.

PX4-Autopilot은 이 회복 단계에서 PID의 P(비례)와 D(미분) 게인을 비행 동역학 압력 플래그에 따라 한시적으로 스케일링(Scaling)하는 동적 이득 스케줄링(Dynamic Gain Scheduling) 기법을 함께 적용한다. 이는 스로틀이 회복되는 0.5초 이내의 순간에만 감쇠비(Damping Ratio)를 인위적으로 높여 오버슈트(Overshoot)를 억제하는 고도의 응용 소프트웨어 디버깅-설계 패턴에 해당한다.

결과적으로 다차원적인 가속도 벡터 검증 체계와 uORB 플래그 전이, 그리고 저수준(Low-level) PID 클래스 단위의 적분항 동결 로직은 PX4가 물리적 불가능 구간에서도 논리적 제어망을 잃지 않게끔 하는 가장 정교하고 확실한 엣지 케이스(Edge Case) 방어기제이다.
