29.7.2. 극단적 자세 및 동역학적 특이점 한계 상황 처리

비행 제어 시스템(Flight Control System) 설계에 있어 가장 까다로운 요구사항 중 하나는 기체가 정상적인 호버링(Hovering)이나 크루즈(Cruise) 비행 궤적을 벗어나, 동역학적 특이점(Singularity)이나 한계 상황(Edge Conditions)에 돌입했을 때 제어 무결성을 잃지 않도록 방어하는 것이다. 조종사의 격렬한 수동 제어 조작, 급격한 돌풍에 의한 뒤집힘, 혹은 기계적 결함으로 인한 일시적인 자유 낙하(Freefall) 상태 등은 선형 제어기(Linear Controller)의 가설 모델을 붕괴시킨다.

본 절에서는 PX4-Autopilot 아키텍처가 수학적 특이점을 어떻게 우회하고, 비선형적인 물리적 극단 상태를 소스 코드 레벨에서 어떻게 인지하여 치명적인 발산(Divergence)을 차단하는지 살펴본다.

1. 짐벌 락(Gimbal Lock)과 쿼터니언 기반 동역학 특이점 회피

오일러 각(Euler Angles: \phi, \theta, \psi)을 이용하여 기체의 자세를 추정하고 제어하는 구형 시스템이나 베이직 멀티위(MultiWii) 레거시 코드는, 피치(Pitch, \theta) 각도가 \pm 90^\circ에 근접할 때 롤(Roll)과 요(Yaw) 축의 구분이 사라지는 수학적 특이점(Singularity), 즉 **짐벌 락(Gimbal Lock)**에 빠지게 된다.

코리올리 힘(Coriolis Force)이나 자이로스코픽 수식 전개 시, 오일러 역학 분모에 \cos(\theta)가 포함되기 때문에 분모가 영(0)으로 수렴하면서 미분 값이 발산(Infinity)하는 치명적인 소프트웨어 결함이 발생한다.

이를 차단하기 위해 PX4의 자세 제어기(mc_att_control) 및 EKF2 추정기(Estimator)는 모든 내부 상태 연산과 행렬 스케줄링을 **쿼터니언(Quaternion)**으로 전면 교체하여 수행한다.

\mathbf{q} = q_0 + q_1i + q_2j + q_3k

수동 제어 시 조종기가 롤 및 피치 스틱의 변위를 요구 각도(Euler Angle Setpoint)로 전달하더라도, PX4는 내부적으로 이를 곧바로 쿼터니언 배열로 변환(Euler to Quaternion)하여 에러 벡터를 산출한다.

2. 자유 낙하(Freefall) 감지 및 비정상 자세에서의 역추력 제어 방지

수동 비행 시 모터를 정지 수준으로 내려버리거나 돌풍에 의해 기체가 전복 될 경우, 기체는 순간적으로 **무중력 상태(Zero-G)**와 유사한 자유 낙하 상태를 겪는다.

2.1 가속도계 기반 자유 낙하 인지 커널

PX4의 센서 퓨전 레이어(EKF) 및 제어 스택은 가속도계(Accelerometer)의 3축 벡터 스칼라 합(\vert\vert \mathbf{a} \vert\vert)이 중력 가속도(9.81m/s^2)보다 현저히 낮아질증폭을 검사하여 자유 낙하 상태 플래그를 변환한다.

이 상태에서 자세 제어 루프의 PID 적분항(Integral Term)이 계속 동작한다면 허공에서 제어 이득이 비정상적으로 누적되는 와인드업(Windup) 현상이 발생하므로, PX4는 낙하 플래그가 참(True)일 때 자세 루프의 적분기 스케줄러를 동결(Freeze)한다.

2.2 인버티드(Inverted) 기동 한계 클리핑

안정화(Stabilized) 모드 등 수직 추력이 항상 아래 방향으로 전제된 모드에서, 피치나 롤 각도가 90^\circ를 초과하여 기체가 뒤집히기 시작하면 수직 추력은 오히려 기체를 지면으로 가속시키는 역할을 하게 된다.

일반 배터리 드론의 경우 90^\circ 이상의 목표 자세가 인가되지 못하도록 매뉴얼 커맨드 전처리 모듈 안에서 입력 세트포인트를 하드 리밋(Hard Limit)으로 클리핑한다.

3. 소프트웨어적 한계 도달 시의 성능 방어(Anti-Windup 및 Saturation) 아키텍처

극단적 상황에서 모터(Actuator)가 최대 허용 대역(Maximum PWM)에 도달해버린 액추에이터 포화(Saturation) 상태에서는 제어기가 아무리 제어량을 높여도 기체는 반응할 수 없다.

// src/lib/pid/PID.cpp 내의 적분기 안티 와인드업 분기
if (actuator_is_saturated && (error * integral_state > 0.0f)) {
    // 적분치 증가 방향과 에러 방향이 포화를 더 심화시키는 조건일 때
    // 적분 수치 업데이트 중지 (Freeze)
    _integral = _integral_previous;
} else {
    // 포화를 완화하는 방향이거나 미포화 상태일 때
    _integral += error * dt;
}

제어기가 한계 상황을 인식하지 못하고 적분항을 과도하게 키워버린다면, 기체가 다시 정상 자세로 회복되었을 때 누적된 제어값으로 인해 반대 방향으로 튕겨 나가는 오버슈트(Overshoot)가 필연적으로 발생한다. 위와 같은 소프트웨어적 안전 분기 모듈은 기체가 비선형적 엣지 케이스 영역에서 다시 선형 영역으로 되돌아올 때 아무런 충격(Bumpless Transfer) 없이 제어권을 확보할 수 있게 하는 PX4-Autopilot 아키텍처의 핵심 무결성(Integrity) 로직이다. Ardupilot 시스템 역시 Anti-Windup 적용에 철저하나, PX4는 Control Allocation(믹서) 모듈에서 액추에이터 포화 플래그를 정밀하게 추출하여 실시간으로 Rate Controller에 피드백한다는 점에서 더욱 계층적이고 세밀한 통제를 구현하고 있다.
