29.2.4.3. 통신 소스 혼용 모드에서의 입력 스위칭 지연(Hysteresis) 및 플로팅(Floating) 현상 방지 코드

본 소절에서는 COM_RC_IN_MODE=2 (RC/Joystick Fallback) 같이 이기종 제어 소스가 혼용되는 다중화(Multiplexing) 환경에서, 신호의 경계 조건(Boundary Condition) 부근에서 발생하는 핑퐁(Ping-pong) 식 스위칭을 억제하는 지연(Hysteresis) 알고리즘과, 제어권 전환 시 스틱 갭(Gap)으로 인해 기체가 널뛰는 플로팅(Floating) 현상을 소프트웨어적으로 어떻게 부드럽게 방어(Damping)하는지 심층 분석한다.

1. 신호 한계선에서의 핑퐁(Ping-pong) 스위칭과 기구학적 불안정성

조종자가 물리적 RC 조종기(Primary Source)를 이용해 1km 밖에서 비행 중이라고 가정해 보자. 기체가 전파 음영 지역에 걸쳐 RF 신호의 수신 강도(RSSI)가 임계점(Threshold) 부근에서 요동치게 되면, 수신기는 0.1초 단위로 ’신호 끊김(Lost)’과 ‘신호 회복(Recovered)’ 상태를 반복하게 된다.

만약 PX4의 ManualControlSelector에 어떠한 방어 로직이 없다면 다음과 같은 치명적 스위칭 루프가 발생한다.

  1. (T=0.0s) RC 신호 끊김 \rightarrow 2차 소스인 GCS 조이스틱으로 즉시 제어권 인계 (Fallback 적용)
  2. (T=0.1s) RC 신호 찰나의 회복 \rightarrow 1차 소스인 RC로 즉시 통제권 강제 회수 (Take-over)
  3. (T=0.2s) RC 신호 다시 끊김 \rightarrow 조이스틱으로 재인계

문제는 물리 RC 스틱은 전진(Pitch 50\%) 상태에 있고, 백업용 GCS 조이스틱은 중앙(Pitch 0\%)에 놓여 있을 때 발생한다. 초당 10번씩 타각 명령이 50 \leftrightarrow 0 을 오가며 널뛰기(Chattering)를 하게 되고, 이는 드론을 공중에서 심하게 요동치게 만들어 결국 추락에 이르게 한다.

2. 스위칭 방어벽: 히스테리시스(Hysteresis)와 상태 천이(State Transition) 잠금

이러한 고주파 스위칭 진동(Oscillation)을 억제하기 위해 PX4 커널은 히스테리시스(Hysteresis, 이력 현상) 타이머를 도입하여 제어 모드의 상태 전이(State Transition)에 일종의 ‘끈적임(Viscosity)’ 혹은 관성(Inertia)을 부여한다.

// src/modules/manual_control/ManualControlSelector.h & .cpp 의 히스테리시스 개념적 구조
class ManualControlSelector {
private:
    // 히스테리시스 타이머: 특정 상태가 충분히 '숙성(Maturation)'되었는지 검증
    hrt_abstime _rc_lost_timestamp{0};
    hrt_abstime _rc_recovered_timestamp{0};
    
    // 파라미터로 정의된 승인 대기 시간
    static constexpr uint64_t RC_RECOVERY_DEBOUNCE_US = 500_ms; // 0.5초
};

// ... update() 로직 내부 ...
void ManualControlSelector::update(uint64_t now) {
    bool rc_currently_valid = _rc_in.is_valid(now);
    
    // RC 신호가 회복되었더라도 즉시 제어권을 넘기지 않음 (Debounce / Hysteresis)
    if (rc_currently_valid) {
        if (_rc_recovered_timestamp == 0) {
            // 회복 시작점 마킹
            _rc_recovered_timestamp = now; 
        } else if (now - _rc_recovered_timestamp > RC_RECOVERY_DEBOUNCE_US) {
            // 0.5초 이상 단 한 번의 끊김 없이 연속적으로 유효해야만 진짜 회복(True Recovery)으로 인정
            _active_source = DataSource::RC; 
        }
    } else {
        // 단 한 프레임이라도 끊기면 회복 타이머 즉각 초기화 (가혹한 검증)
        _rc_recovered_timestamp = 0; 
        
        // 조이스틱 폴백(Fallback) 로직 실행...
    }
}

위의 C++ 최적화 패턴인 데바운스(Debounce) 알고리즘은 인터럽트 기반의 노이즈 필터링에서 흔히 쓰이는 기법이다. RC 신호가 잠시 돌아왔다 하더라도, 비행 제어기(FC)는 최소 500ms(파라미터 기반 가변적 설정 가능) 동안 그 신호가 한 번의 흠결도 없이 안정적으로 유지됨을 증명해 내야만 1차 제어권(Primary Control)을 다시 내어준다. 이 히스테리시스 갭(Gap)이 버퍼 쿠션으로 작용하여 핑퐁 스위칭 체인이 원천 분쇄되며, 동역학계는 치명적인 임펄스 충격에서 해방된다.

3. 조종권 인계 시의 스틱 오프셋(Offset) 매칭과 Slew Rate 필터링 기법

신호 스위칭 떨림은 잡았다 하더라도, RC 컨트롤러(Pitch 50\%)에서 GCS 조이스틱(Pitch 0\%)으로 권한이 뚝 떨어지듯(Drop) 넘어가는 순간 기체는 50\% \rightarrow 0\% 의 계단 함수(Step Function) 성격의 물리적 타격(Jerk)을 고스란히 받게 된다.

이를 부드럽게 녹여내기 위해(Smoothing) 상위 서브시스템(주로 Flight Mode Manager 나 Position Controller의 궤적 제어기 단계)은 지수적 감쇠(Exponential Decay) 혹은 비율 제한기(Slew Rate Limiter, Slew) 필터를 결합하여 스위칭 갭을 소프트웨어적으로 블렌딩(Blending)한다.

직전 프레임까지 RC 소스가 지시하던 궤적 값(X_{\text{old}})과 현재 조이스틱이 지시하는 궤적 값(X_{\text{new}}) 사이를, 마치 페이드 아웃/페이드 인(Fade-out/Fade-in) 애니메이션처럼 일정 밀리초(ms)에 걸쳐 보간(Interpolation) 램프(Ramp)를 그려 이어 붙이는 방식이다. 이것은 하드웨어 센서나 매체가 무엇으로 변하든, 가장 말단의 믹서(Mixer)와 모터에 인가되는 PWM 목표 제어량 곡선은 수학적으로 연속(Continuous)이어야 하며 절대 불연속 분절(Discontinuity)이 있어서는 안 된다는 항공 우주 소프트웨어(Aerospace Software)의 가장 엄격한 대전제를 충족시키는 안전 장치이다.

이로써 어떠한 혼용 제어 상황에서도 PX4의 시스템 커널은 우아함(Gracefulness)과 연속성(Continuity)을 쟁취해 낸다.
