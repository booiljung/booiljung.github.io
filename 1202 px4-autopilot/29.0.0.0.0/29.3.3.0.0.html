29.3.3. 매뉴얼 모드 안전 장치 및 EKF 독립성 검증

본 절에서는 단순한 제어 파이프라인 우회(Bypass)를 넘어, 매뉴얼 모드(Manual Mode)가 PX4 시스템 최악의 재난 상황인 센서 퓨전 발산(Sensor Fusion Divergence) 상황에서 조종자와 기체를 구원하는 최후의 보루(Last Resort)로서 기능하기 위한 아키텍처적 안전 장치(Fail-safe)와 확장 칼만 필터(EKF) 독립성(Independence)을 심층 검증한다.

1. 상태 추정기(EKF2) 발산의 공학적 치명성

PX4의 자율 비행(Auto) 시스템과 위치 제어(Position Control) 루프는 전적으로 ekf2 모듈이 발행(Publish)하는 vehicle_local_position 및 vehicle_attitude 토픽에 의지한다. EKF는 가속도계, 자이로스코프, 지자기 센서, GPS, 기압계 데이터를 거대한 오차 공분산 행렬 속에 융합하여 기체의 현재 상태를 추정(Estimation)한다.

그러나 센서 데이터는 현실 세계의 물리적 잡음(Noise)에 극도로 취약하다.

  • 강한 태양 흑점 폭발이나 터널 레이저 반사로 인한 GPS 다중 경로(Multipath) 에러
  • 고압선 통과 시 발생하는 기체 지자기 센서의 자기장 교란(Magnetic Interference)
  • 핏토관(Pitot tube) 동결로 인한 대기속도(Airspeed) 상실

이러한 교란이 임계치(Innovation Gate)를 뚫고 EKF 상태 변수를 오염시키기 시작하면, EKF 연산은 발산(Diverge)하게 되며, FC는 실제로는 수평으로 잘 날고 있는 기체를 “좌측으로 90^{\circ} 기울어져 추락 중“이라고 착각하게 된다. 이때 자율 비행 피드백 제어기는 기체를 살리겠다는 명목 하에 반대쪽으로 에일러론을 끝까지 꺾어버려, 멀쩡한 기체를 오히려 땅으로 내동댕이치는 이른바 CFIT(Controlled Flight Into Terrain) 현상을 유발한다.

2. 매뉴얼 모드의 EKF 상태 변수 무의존성(Zero-dependency)

EKF가 붕괴되는 순간, 조종자가 기체를 살릴 수 있는 유일한 탈출구는 즉각적으로 비행 모드 스위치를 ‘매뉴얼(Manual)’ 로 전환하는 것뿐이다.

이 전환이 성공하기 위해서는, 그리고 전환 후 기체가 조종자의 뜻대로 움직이기 위해서는 매뉴얼 모드의 제어 루프 레이어가 철저히 ‘EKF 독립적(EKF-Independent)’ 이어야 한다는 소프트웨어 설계 원칙이 필수적으로 요구된다.

앞선 29.3.2절에서 살펴보았듯, 고정익 다이렉트 패스스루의 경우 제어 파이프라인(FlightTaskManual) 내부에 EKF 추정값 참조 코드가 완전히 소거되어 있다. _quat_state(현재 EKF 자세 쿼터니언)나 _rate_state(현재 필터링된 각속도) 변수들은 조건문 if (is_manual) 블록 바깥에 격리되어 있어, EKF가 아무리 NaN(Not a Number)이나 무한대(\infty)의 쓰레기 값을 내뱉고 있더라도 믹서로 향하는 스틱 입력 버퍼(_actuator_controls.control)는 단 1\text{bit} 도 오염되지 않는다.

3. 커맨더(Commander) 모듈의 매뉴얼 전환 강제 승인(Force-Grant) 로직

EKF 결함 시 단순히 제어 루프만 분리되어 있다고 해서 기체가 안전해지는 것은 아니다. 상태 머신을 통제하는 Commander가 고장 난 상태(Unhealthy)의 센서를 핑계로 조종자의 매뉴얼 모드 전환 요청을 거부(Deny)해버리면 아무 소용이 없기 때문이다.

이 때문에 state_machine_helper.cpp 내부의 전이 조건 검열기(Transition Validator)는 다수의 비행 모드에 대해 “GPS 유효성(GPS valid)”, “로컬 포지션 유효성(Local position valid)”, “자세 유효성(Attitude valid)” 등의 강한 true 조건을 요구(return TRANSITION_DENIED)하는 반면, MAIN_STATE_MANUAL 전환 에 대해서만큼은 유일하게 이러한 까다로운 EKF 상태 점검 플래그들을 전부 면제(Exempt)해 준다.

// 상태 머신의 EKF 독립적 승인 철학 (개념화)
if (request == MAIN_STATE_MANUAL) {
    if (status.rc_signal_lost) {
        return TRANSITION_DENIED; // RC 통신 자체의 두절 외에는 거부 명분이 없음
    }
    // EKF가 죽었든(GPS 손실), IMU 자세가 발산했든 묻지도 따지지도 않고 전환 승인
    return TRANSITION_CHANGED; 
}

이러한 ‘묻지 마 승인’ 아키텍처는 “시스템이 미쳐 날뛰기 시작하면, 그 시스템의 판단력을 박탈하고 시스템 바깥에 있는 인간의 눈(Human Visual Sensor)과 손(Manual Sticks)을 유일한 지표로 삼는다“는 유인 항공 시대부터 내려온 항공 철학을 오픈소스 C++ 펌웨어의 분기문 레벨에서 가장 충실히 계승하고 있는 모습이다. 이어지는 장에서는 EKF 센서 발산 시 구체적인 상태 머신의 유지 조건과, 엔진 시동(Arming) 시 스로틀 단에서 이루어지는 하위 안전 프로토콜에 대해 검토할 것이다.
