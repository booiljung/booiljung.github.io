29.1.2.2. 처리단: manual_control_setpoint_s 구조체의 축별 정규화 필드 및 보조 스위치 제어 신호 맵핑

본 소절에서는 최하단 input_rc 토픽 버스로부터 포집된 하드웨어 원시 채널 배열이 manual_control 데몬(Daemon) 모듈의 연산 코어를 거쳐 기구학적 수학적 의미를 동적으로 부여받은 manual_control_setpoint_s 구조체로 스케일링(Scaling) 및 엄격히 정규화(Normalization)되는 데이터 변이 과정을 심층 분석한다. 본 구조체는 조종자(Human Pilot)의 뇌에서 발원한 비행 의도(Flight Intent)가 순수 공학적 스칼라 역학 계수로 트랜스파일(Transpile)된 최초의 시스템 기구학적 명세서(Specification) 격 역할을 묵묵히 수행한다.

1. manual_control_setpoint_s 구조체의 수학적 정규화 패러다임

manual_control_setpoint_s 토픽 아키텍처는 특정 서드파티 RC 송신기의 저열한 하드웨어적 제조 한계(예: 짐벌 요크 최대 타각 시 2000\mu s에 미달하는 1850\mu s 까지만 ADC 전압 펄스가 출력되는 하드웨어 백래시 현상)나 상이한 라디오 통신 프로토콜의 채널 배치 차이를 완전하게 시스템으로부터 은닉(Hide)하는 소프트웨어 중간 계층 미들웨어(Middleware Layer) 인터페이스의 꽃이다.

// msg/manual_control_setpoint.msg (수학적 개념 직관을 위한 핵심 필드 요약 배열)
uint64 timestamp        # 마이크로초 단위의 버스 발행 시간 및 동기화 인덱스
uint64 timestamp_sample # 원시 input_rc가 수신기로부터 취득된 오리지널 시점의 타임스탬프 (지연 계측용)

float32 roll            # [-1.0, 1.0] 범위로 정밀 정규화된 롤 축 스칼라 (통상 Mode 2 기준 우측 스틱 수평)
float32 pitch           # [-1.0, 1.0] 범위로 정규화된 피치 축 스칼라 (통상 Mode 2 기준 우측 스틱 수직)
float32 yaw             # [-1.0, 1.0] 범위로 정규화된 요 축 스칼라 (좌측 스틱 수평 회전)
float32 throttle        # [-1.0, 1.0] 또는 [0.0, 1.0] 범위로 정규화된 스로틀 추력 인자 (좌측 스틱 수직)

int8 aux1               # [-1, 0, 1] 구간의 이산 범위 보조 스위치 1 (예: Flight Mode 가드 트리거 등)
int8 aux2               # [-1, 0, 1] 구간의 이산 범위 보조 스위치 2
// ... (서브 시스템 제어를 위해 통상 aux6까지 선형 확장 지원)

위 구조체 명세에서 설계 공학적으로 가장 주요한 패러다임의 핵심은 바로 항공기의 주 조종 4축인 roll, pitch, yaw, throttle 인자 스트림이 무조건 부동 소수점(Floating Point) 형식의 완벽무결한 [-1.0, 1.0] 유닛 폼(Unit Form) 스칼라 영역으로 캘리브레이트 변환 채택된다는 절대 원칙에 기인한다. 이러한 타이트한 스케일링 디자인 설계 모델이 주는 거대한 이점은, 수신 버퍼 하위에 종속 연결된 수많은 최신 커스텀 FlightTask 컨테이너들이 조종기의 물리적 브랜드 팩터 종류를 아예 백지 상태로 무시(의식하지 않음)하고, 단지 소프트웨어 버스로부터 데이터 0.5라는 무차원 스칼라 값을 인가받았을 때 “파라미터로 지정된 기체 비행 동역학 프로파일 최대 역량 한계치(예: MPC_ROLL_MAX)의 정확히 50% 각속도 역량을 인출하여 인가한다“는 극도로 일관된 제네릭(Generic)한 스칼라 증폭 행렬 연산만 일방향으로 수행하도록 소프트웨어 책임을 엄격히 계층 분리할 수 있다는 공학적 미덕이다. 결과적으로 이는 멀티로터나 고정익 등 이기종 비행 컨트롤러의 튜닝 파라미터(Parameter) 관리가 비약적으로 안정화되고 통일화 단순해지는 거대한 기반이 된다.

2. 채널 매트릭스 맵핑(Channel Mapping) 및 데드밴드(Deadzone) 산출 소스 로직 해부

manual_control 애플리케이션 소스 트리 데몬의 코어 C++ 연산부 루프에서는 단순한 1차원적 분수 비율식(Ratio) 변환 차원을 격하게 뛰어넘는 복합적인 인간공학적 수학 비선형 증폭 보정이 쉼 없이 개입한다.

  1. 동적 파라미터 기반 채널 바인딩(Parameter-based Dynamic Channel Binding):
    펌웨어 파라미터 메모리 존 내부에는 RC_MAP_ROLL, RC_MAP_PITCH 등의 가상 맵핑 치환 파라미터가 펌웨어 상시 등록되어 있다. 오퍼레이터(조종자)가 QGroundControl 등의 GCS RC 설정 채널 화면에서 각 보유 기체의 스틱 축을 물리적으로 흔들어 매칭을 할당 저장하면 이 백그라운드 파라미터 번호가 비휘발성 저장소에 갱신된다. manual_control 데몬의 이벤트 폴링 루프는 매 틱 이 파라미터 인덱스를 딥 레퍼런스 input_rc.values[] 배열의 주소 포인터 키(Key)로 실시간 역산(Reverse Calculation) 추출하여 아무 의미가 없던 하드웨어 원시 물리 바이트를 논리적 항공 기동 축으로 직결 매칭시킨다.
  2. 보정수차 캘리브레이션 트리밍(Trimming) 및 스케일링 계수 환산:
    고가 라디오 컨트롤러 조차도 스틱 짐벌(Gimbal) 내부 텐션 링의 물리적 금속 스프링 장력 편차나 포텐쇼미터 마모 오차로 인하여, 조종자가 스틱 손을 뗀 중앙(Center) 자유 상태에 놓았을 때 소프트웨어적으로 정확히 1500\mu s가 출력되지 않는 데드 오프셋(Offset)이 반드시 필연적으로 존재한다. 인하우스 커널 알고리즘은 RC1_MIN, RC1_TRIM, RC1_MAX 라는 철저한 3점 독립 캘리브레이션 프로파일 파라미터를 메모리로 동원하여 호출하고, 중앙 스틱 텐션 좌표(Trim)를 기점으로 그 이하의 하프 대역과 이상의 상승 밴드 대역을 상하 논리 분리 파티셔닝한 뒤 각각 완전히 독립적인 비율 계수 편차를 지닌 기울기 선형 보간기(Linear Interpolator) 수학 트랙 스코어를 태운다. 이를 통해 비툴어진 물리적 기성품 하드웨어 한계를 완벽한 수학적 부동소수점 도메인 상의 극점 0.0(절대 중앙점)으로 록인 강제 수렴 트림(Trim)시킨다.
  3. 데드존 노이즈(Deadzone Noise) 필터 억제 수학망:
    허공에 조용히 가만히 둔 파일럿 스틱 미세 주변의 지속적인 ADC(아날로그-디지털 컨버터) 양자화 노이즈 전압 플럭추에이션 파동이 하위 PID 루프 미분기 스파크를 타고 모터 및 기체의 프레임 미세 고주파 떨림(Jitter)으로 무한 증폭 확산 전송되는 현상을 공학적으로 막기 위해, PX4는 RC1_DZ (Deadzone) 파라미터 마스킹 대역을 스틱 중앙 수학적 트림 영점 주변에 방파제처럼 선형 형성시킨다. 이 설정된 노이즈 파동 구간 게이지 내에 위치한 미세한 input_rc 펄스 증감 폭은 manual_control_setpoint 스칼라 수축 산출 마지막 단계에서 C++ 선형 대수학적 판단문에 의해 하드 강압적인 0.0 스칼라 상수치로 포스 치환(Zeroing)되어 불필요한 노이즈 캐스케이드(Cascade) 전파 파이프라인을 원천 단절시킨다.

3. 보조 스위치(Auxiliary Switch)의 트리스테이트(Tri-state) 맵핑 역학과 하위 라우팅 분기

RC 송신기의 어깨 라인에 배치된 다채널 토글 스위치(Toggle Switch)들 역시 메인 아날로그 조종 스틱과 완벽히 동일한 물리 베이스 프레임을 타고 input_rc 정수 배열로 연속 수신된다. manual_control_setpoint_s 출력단에서는 복잡다단한 시스템 구동을 위해 이 원시 스위칭 신호를 단순무식한 이진 판별(Binary ON/OFF) 레벨이 아닌, int8 메모리 자료형 포맷 아키텍처 위에서의 정교한 트리스테이트(Tri-state: -1, 0, 1) 로직 트리로 양자 맵핑(Mapping)한다.

일반적인 3단 다축 토글 스위치(예: Flight Mode 전환 트리거 로터리 스위치 등) 펄스를 맵핑 변환할 때, 수집된 펄스 폭 약 1000\mu s 레인지 하한 구간은 프로그래머블 상수 -1, 1500\mu s 미들 구간은 0, 2000\mu s 어퍼 구간은 양수 1로 확정 이산화(Discretization)하는 소프트웨어 폴링 스루 임계치 타겟팅 알고리즘이 촘촘하게 적용 및 치환 동작된다. 파이프라인 종단 부를 감시하는 상위 상태 체계 컨트롤러인 커맨더(Commander) 또는 랜딩 기어(Landing Gear), 고정익 플랩(Flaps) 구동 이펙터 믹서들은 바로 이 -1, 0, 1로 깔끔하게 소팅 정리된 보조 신호 변수 배열 맵을 직접 리드 다중 Switch/Case 제어문으로 하드 분기 폴링 채용하여 안전한 런다운 모드 체인지를 즉시 집행한다. 이처럼 더러운 변동성을 지닌 보편적인 모든 물리적 외부 전기 라디오 신호 인터페이스를 소프트웨어 커널 진입 단계에서만 임시 차용하고 내부적으로는 철저히 정제된 논리적 데이터 상수로 치환해 내는 manual_control 모듈의 극강의 방어 프로세스 덕택에, PX4-Autopilot의 두뇌는 하드웨어 인풋 기기의 생태계 종속성 제약 사슬에서 완벽히 해방된다.
