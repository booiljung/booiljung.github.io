29.1.2. 수동 모드 uORB 메시지 파이프라인 데이터 흐름도 (엔드투엔드)

본 소절에서는 조종자(Human Pilot)의 손끝에서 발생한 조이스틱의 물리적 변위가 기체의 최종 로터(Rotor) 회전수 명령으로 역산되어 변환되기까지, PX4-Autopilot 커널 내부를 관통하는 ’수동 제어 모드 엔드투엔드(End-to-End) 데이터 파이프라인’을 uORB 메시지의 생명 주기(Lifecycle) 및 변이 관점에서 도식화하고 서술한다.

1. uORB 기반 데이터 스트림 파이프라인 개요

PX4의 모듈 간 데이터 버스 통신은 공유 메모리(Shared Memory) 기반의 고속 발행/구독(Publish/Subscribe) 미들웨어인 uORB를 통해 전적으로 이루어진다. 수동 조종 제어 루프 아키텍처 역시 이 비동기 철학을 엄격히 따르며, 통상적으로 100Hz ~ 250Hz 물리 대역의 고주파 틱(Tick)으로 아래와 같은 단계별 핵심 제어 메시지들이 순차적으로 생성되고 릴레이(Relay)된다.

2. 각 파이프라인 계층 단위별 메시지 변형 역학

상단의 시퀀스 다이어그램에서 관찰할 수 있듯, 조종기의 아날로그 전기적 입력 신호는 소프트웨어 파이프라인의 하위단 제어기로 흘러내려갈수록 추상적인 데이터에서 점차 물리적이고 구체적인 제어 기구학적 의미 수납체로 구체화된다.

  1. 원시 신호(Raw Signal) 단계 (input_rc):
    하드웨어 추상화 드라이버(HAL) 계층이 UART나 SPI 시리얼 통신 버스로부터 수신한 데이터를 1차 디코딩(Decoding)한 직후의 날것 상태이다. 펄스폭변조(PWM) 프로토콜 기준 1000\mu s ~ 2000\mu s 범위의 마이크로초 단위 값 자체나 최신 고속 프로토콜의 11비트(Bit) 정수 변위 정보를 갖는다. 이 시점의 메시지 페이로드를 단면 분석해 보면, 익명화된 채널 번호(Channel Number) 배열만 존재할 뿐, 해당 채널 값이 기동 방향이 롤(Roll)인지 피치(Pitch) 방향인지는 소프트웨어적으로 전혀 명시(Tagging)되지 않은 단순 데이터 덩어리에 불과하다.

  2. 기구학적 정규화(Kinematics Normalization) 단계 (manual_control_setpoint):
    manual_control 데몬이 구동되며 펌웨어에 이식된 RC_MAP_* 파라미터 맵핑 조합을 실시간으로 참조하여, 의미가 상실된 단순 배열 채널의 개별 값들에 롤, 피치, 요, 스로틀(Throttle)이라는 3차원 기구학적 역할을 비로소 부여한다. 더불어 송신기의 제조사 라인업(Futaba, FrSky 등)이나 조이스틱 내 짐벌 센서의 하드웨어 캘리브레이션 치우침 오차 범위에 구애받지 않고 언제나 동작이 일관성을 지니도록, 모든 인풋 값을 완벽한 -1.0 (하한 수축)에서 +1.0 (상한 팽창) 사이의 정밀한 부동 소수점(float) 데이터로 선형 스케일링(Scaling) 조작한다. 인간의 조종기 스틱 주변 떨림 노이즈를 상쇄하는 데드밴드(Deadband) 처리나 EXPO 회귀 연산(비선형 스틱 응답성 곡선)이 일차적으로 소프트웨어 연산 적용되는 중요 구간이기도 하다.

  3. 제어 절대 지령화(Setpoint Generation) 단계 (vehicle_attitude_setpoint / vehicle_rates_setpoint):
    데이터가 FlightModeManager 내부 비행 태스크(Flight Task) 객체 메모리들로 유입되면서 이 논리적 정규화 값(-1.0 \sim 1.0)은 기체의 실제 물리적 타깃 제어량으로 극적 증폭(Amplification)된다. 예컨대 스틱 최대 입력 기울기(+1.0)는 파라미터 MPC_ROLL_MAX 값 인자와 직접 곱해져 물리적 기체가 달성해야할 타깃 롤 회전 각도 +45^\circ(내부적으론 라디안 Radian 변환)라는 물리적 절대 지령(Absolute Command) 값으로 환골탈태한다.

  4. 제어 동역학 출력(Control Dynamics Output) 단계 (vehicle_torque_setpoint 등):
    이후 파이프라인의 최하단에 포진한 PID 지령 프로세스 컴포넌트(mc_att_control, mc_rate_control)들이 순연히 개입하여, 실제 EKF 추정 동역학(Dynamics) 아웃풋과 앞서 하달받은 타깃 지령(Setpoint)의 동시간적 상태 잔차 타임 오차(Error)를 산출해 내어 고전 제어 공학의 비례-적분-미분 연산 처리를 실시간 거친다. 그 거대한 매트릭스 연산 결과물로 에어프레임 본체 로터스 구조가 공간에 내뿜어야 할 물리적 3차원 회전 토크량(\tau_x, \tau_y, \tau_z) 벡터와 Z축 절대 추력값(F_z) 스칼라가 도출된다. 마지막으로 이 값들은 비행기 기체 형상별 제어 분배 행렬인 최종 믹서 행렬 매트릭스(Mixer Matrix, Control Allocator) 컴포넌트를 통과하며, 쿼드 살포 드론 기준으로 4개의 독립된 모터 RPM 커맨드(PWM 듀티비 구동 폭 또는 DShot 디지털 명령 틱 프레임)로 완전히 개별 확정 분해되어 기체의 궤도를 지배하게 된다.
