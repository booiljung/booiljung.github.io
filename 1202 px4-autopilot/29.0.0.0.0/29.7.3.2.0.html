29.7.3.2. 주파수 응답성 보드 플롯(Bode Plot) 분석을 위한 로그 추출: 목표 입력과 실제 센서 출력 데이터 매칭을 통해 수동 제어 루프의 대역폭(Bandwidth) 및 위상 여유(Phase Margin)를 분석하는 방법론

무인항공기(UAV)의 자세 제어기(Attitude Controller)를 시간 영역(Time Domain)에서 단순히 오버슈트(Overshoot)나 정착 시간(Settling Time)만으로 평가하는 것은 한계가 뚜렷하다. 제어 루프가 외부 외란이나 고주파 노이즈에 얼마나 강건한지(Robustness), 그리고 시스템이 발산하기 전까지 어느 정도의 여유를 가지고 있는지를 정밀하게 규명하려면 **주파수 영역(Frequency Domain)**으로의 변환이 필수적이다.

본 절에서는 PX4-Autopilot의 고해상도 ULog 데이터를 기반으로 주파수 응답성 보드 플롯(Bode Plot)을 도출해내어 기체의 **대역폭(Bandwidth)**과 **위상 여유(Phase Margin)**를 객관적으로 산출해내는 소프트웨어적 방법론을 분석한다.

1. 시스템 식별(System Identification)과 쳐프(Chirp) 신호 추출

보드 플롯을 그리기 위해서는 시스템에 의도적으로 다양한 주파수 대역을 스윕(Sweep)하는 입력 신호를 가해야 한다. 이를 주파수 쳐프(Frequency Chirp) 시그널링이라 부른다. PX4에서는 SYS_AUTOSTART 및 식별 모듈을 활성화하여 기체가 호버링하는 동안 Roll/Pitch 축에 인위적인 사인파(Sine wave) 쳐프 입력을 발생시킬 수 있다.

이때 쳐프 입력과 그에 상응하는 기체의 동역학적 반응을 매칭하기 위해 추출해야 하는 핵심 ULog 토픽은 다음과 같다.

  • 목표 입력 신호(Input): vehicle_rates_setpoint
  • 실제 센서 응답(Output): vehicle_angular_velocity (자이로스코프 원시 또는 노이즈가 1차 컷오프된 데이터)

2. 대역폭(Bandwidth) 및 위상 여유(Phase Margin) 분석 파이프라인

ULog에서 추출한 두 시계열(Time-series) 데이터는 Python 기반의 scipy.signal.tfestimate 혹은 PX4 Flight Review의 내장 System Identification 루틴을 통해 푸리에 변환(FFT)되어 선형 시불변(LTI) 시스템의 전달 함수(Transfer Function) H(j\omega) 로 변환된다.

H(j\omega) = \frac{Y(j\omega)}{U(j\omega)}

여기서 U(j\omega)는 입력 쳐프(Setpoint), Y(j\omega)는 센서 출력 응답(Rate)의 주파수 스펙트럼이다. 이 결과는 크게 Magnitude(크기) 플롯과 Phase(위상) 플롯으로 나타난다.

2.1 대역폭(Bandwidth) 추정 (Magnitude Plot)

크기 플롯에서 0dB 근처를 맴돌던 게인(Gain)이 고주파 대역으로 진입하면서 감쇠하기 시작하여 -3dB 지점을 통과하는 주파수를 시스템 컷오프 주파수(Cut-off Frequency) 혹은 대역폭(Bandwidth)으로 정의한다.

  • 대역폭이 넓을수록(예: 20Hz \sim 40Hz 이상) 기체는 조종사의 빠른 스틱 이동이나 고주파 외란에 민첩하게 반응한다.
  • PX4 레이싱 기체들의 롤(Roll) 축 대역폭은 30Hz 이상을 목표로 튜닝하지만, 대형 상업용 기체는 구동계 지연으로 인해 5Hz \sim 15Hz 에서 차단 주파수가 형성된다.

2.2 위상 여유(Phase Margin) 분석 (Phase Plot)

기계 시스템의 한계로 인해 주파수가 높아질수록 실제 기체의 반응 위상은 늦어지게(Phase Lag) 된다.
위상 여유는 크기 게인이 0dB(교차 주파수, Crossover Frequency)일 때, 위상 각도가 발산 한계선인 -180^\circ 에 도달하기까지 남은 여유 각도를 나타낸다.

\text{Phase Margin} = \angle H(j\omega_{crossover}) - (-180^\circ)

  • 일반적으로 안정적인 비행(오버슈트 억제 및 진동 방지)을 위해 최소 45^\circ \sim 60^\circ 의 위상 여유 확보가 권장된다.

3. Ardupilot의 시스템 식별 기법(System ID)과의 아키텍처 비교

주파수 제어 검증 체계에서 Ardupilot 시스템은 펌웨어 루프 내에 실시간 주파수 분석기를 탑재하는 System ID 모드를 지원하여 MAVLink를 통해 실시간으로 쳐프 상태를 추종하는 고유한 아키텍처를 발전시켜 왔다. 이는 현장에서 즉각적인 분석을 요구하는 엔지니어에게 유리하다.

반면, PX4-Autopilot은 MCU(Microcontroller Unit)의 연산 리소스를 비행 제어 루프 본연의 결정성(Determinism) 유지에 온전히 집중시키고자 한다. 따라서 주파수 도메인 변환 연산은 철저히 오프라인 로깅 생태계(ULog 분석 툴체인)로 위임(Off-loading)한다. ULog가 고주파 레이트(최대 1kHz 근접)로 덤프한 데이터를 후처리로 푸리에 변환함으로써, 기체의 스펙트럼 밀도(Spectral Density)나 노이즈 공진 주파수(Resonant Frequency)까지 복합적으로 분석할 수 있는 해상도 이점을 챙긴 것이다.

이러한 로깅 기반 오프라인 분석 파이프라인은 복잡한 다축 드론뿐만 아니라 회전익(Helicopter) 및 기하구조상 비대칭을 가지는 VTOL(수직 이착륙기)의 자세 제어 불안정성을 트러블슈팅하는 데 가장 강력한 과학적 근거가 된다.
