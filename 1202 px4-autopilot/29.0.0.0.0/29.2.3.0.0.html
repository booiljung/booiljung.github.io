29.2.3. 조종감(Feel) 최적화를 위한 비선형 디지털 신호 처리(DSP) 알고리즘 (src/lib/mathlib)

본 소절에서는 PX4-Autopilot 커널이 단순한 선형(Linear) 비례 제어를 지양하고, 실제 인간 파일럿의 인체공학적 조종감(Feel)을 극대화하기 위해 src/lib/mathlib 라이브러리에 내장시킨 비선형 디지털 신호 처리(Non-linear DSP) 알고리즘 아키텍처를 심층 분석한다. 스틱의 물리적 입력값을 그대로 액추에이터 목표값으로 환산하는 평면적인 제어 방식에서 탈피하여, 엑스포넨셜 곡선(Exponential Curve)과 동적 저주파 통과 필터(LPF) 등을 융합함으로써 조종자의 미세한 의도를 어떻게 수학적으로 증폭시키거나 억제(Damping)하는지 공학적으로 살펴본다.

1. 선형 맵핑(Linear Mapping)의 한계와 비선형(Non-linear) 조종감에 대한 기구학적 요구

지금까지 분석한 29.2.2절의 선형 정규화(Linear Normalization) 과정은 조종기 스틱의 물리적 행정 거리를 기계 논리 영역인 -1.0 \sim 1.0 부동 소수점 공간으로 정확히 프로젝션(Projection)하는 것에 불과하다. 만약 이 선형화된 1차 함수 결괏값을 그대로 쿼드콥터의 롤/피치 각속도 목표값(Angular Rate Setpoint)으로 인가한다면, 파일럿은 기체를 조종하는 데 극심한 이질감을 느끼게 된다.

기구학적으로, 호버링(Hovering) 중인 드론을 안정되게 유지하기 위해서는 중심 영점(Trim) 부근에서 아주 미세하고 부드러운 타각 역량(Micro-control)이 필수적이다. 그러나 스틱을 극단으로 밀어 아크로배틱 플립(Acrobatic Flip)을 수행해야 할 때는 폭발적인 각속도 전이율(High Roll-rate)이 필요하다. 만약 입력 대비 출력이 y = ax 형태의 고정된 기울기(Linear)를 갖는다면, 중앙부에서는 기체가 너무 민감하게 반응하여 조종 피로도가 치솟고, 극단부에서는 회전력이 부족해지는 기하학적 딜레마에 빠진다.

이를 소프트웨어로 해결하기 위해 도입된 것이 바로 엑스포(Expo, 엑스포넨셜 커브)와 슈퍼 레이트(Super Rate, 곡선 가속도 모델) 등 다항식(Polynomial) 기반의 비선형 곡선 피팅(Curve Fitting) 알고리즘이다.

2. 엑스포(Expo) 곡선 피팅을 위한 3차 다항식(Cubic Polynomial) 매핑 모델 분석

PX4의 mathlib 은 이러한 비례-지수 합성 맵핑을 하드웨어 FPU(부동 소수점 연산 장치)에서 가장 우아하고 빠르게 연산할 수 있도록 단순 무식한 거듭제곱 연산(pow())이나 exp() 함수 대신 정교하게 설계된 3차 다항식 블렌딩(Cubic Polynomial Blending) 모델을 채택하고 있다.

// src/lib/mathlib/math/filter/Expo.hpp 의 개념화된 비선형 매핑 커널
float math::expo(float value, float expo_value)
{
    // value: -1.0 ~ 1.0 로 정규화된 선형 입력 스틱 값
    // expo_value: 0.0 ~ 1.0 범위의 비선형 가중치 파라미터 (1.0일수록 곡선이 깊어짐)
    
    // 3차 다항식 곡선: x^3 
    float value_cubed = value * value * value; 

    // 선형 함수(y=x)와 3차 함수(y=x^3) 사이의 덧셈 블렌딩(Blending) 혼합
    // 공식: Out = (1.0 - expo) * value + expo * (value^3)
    return (1.0f - expo_value) * value + expo_value * value_cubed;
}

이 고전적이면서도 경량화된 수식 모델 (1 - e)*x + e*x^3 은 항공 역학 제어 분야의 표준이다.

  • 파라미터 expo_value 가 0.0 일 경우: 식은 순수한 y = x 의 완전한 1차 선형 함수가 된다.
  • 파라미터 expo_value 가 1.0 일 경우: 식은 순수한 y = x^3 의 완전한 3차 곡선이 된다. 변곡점인 (0,0) 부근에서는 기울기가 극도로 완만해져 스틱을 꽤 많이 밀어도 기체가 둔감하게 반응하며, 반대로 끝단(Ends)인 \pm 1.0 근처에서는 기울기가 가파르게 수직 상승하여 폭발적인 출력을 뿜어낸다.

부동 소수점 거듭제곱 API(pow(x, 3.0))를 호출하는 대신 단순히 변수를 세 번 연쇄 곱셈(Chain Multiplication)하는 코드 구조(value * value * value)는 수학적 정확도를 잃지 않으면서도 수 밀리초(ms) 단위의 런타임 제어 루프 컨텍스트 내에서 연산 오버헤드를 O(1) 사이클 차원으로 억압하는 임베디드 코어 최적화의 극치이다.

3. 고주파 잡음(Jitter) 상쇄를 위한 1차 IIR 저주파 통과 필터(LPF) 이산화 모델

수동 모드 체감 품질을 결정짓는 두 번째 축은 짐벌 가변 저항(Potentiometer) 소자 특유의 고주파성 아날로그 지터(Jitter) 노이즈를 부드럽게 감쇄시키는 타이밍 필터링(Timing Filtering) 기법이다. 조종자가 스틱을 가만히 쥐고 있더라도 시스템엔 수백 헤르츠의 노이즈 파동이 실려오며, 이 미세한 진동이 수식 모델을 거쳐 하부 PID 액추에이터 루프의 연산 미분항(D-Term)으로 직행할 경우 드론의 모터에서 가혹한 로보틱 떨림(Oscillation)과 심각한 역기전력을 발생시킨다.

PX4 mathlib 은 이 스틱 노이즈 분산(Variance)을 억눌러 마우스 커서의 스무딩(Smoothing) 같은 매끄러운 궤적을 창출하기 위해 시계열 처리형 1차 IIR(Infinite Impulse Response) 저주파 통과 필터(LPF) 를 적용한다.

// src/lib/mathlib/math/filter/LowPassFilter.cpp 의 IIR 필터 개념
void LowPassFilter::update(float new_input, float dt)
{
    // dt: 루프 타임 (Delta Time), 통상 0.01s (100Hz)
    // T: 사용자가 설정한 필터 시정수(Time Constant), 차단 주파수(Cutoff Frequency)의 역수
    
    // 이산화(Discretization) 계수 Alpha(가중치) 도출 공식
    float alpha = dt / (T + dt); 
    
    // IIR 필터 점화식 (재귀 연산)
    // 신규 출력 = (Alpha * 현재 입력값) + ((1 - Alpha) * 직전 누적 출력값)
    _filter_state = alpha * new_input + (1.0f - alpha) * _filter_state;
}

이 IIR 필터 모델은 신호 처리 공학의 연속 시간 차단 방정식 T\frac{dy}{dt} + y = u(t) 을 오일러 전방 차분법(Forward Euler Discretization)으로 이산화한 전형적인 지수 평활법(Exponential Smoothing) 수식이다. 계수 \alpha 가 가중하는 비율에 따라, 새롭게 유입되는 뾰족한 고주파 노이즈 스파이크(Spike)는 무시당하고 오직 시정수 지표 T 에 필터링된 점진적이고 거시적인 저주파 궤적 파도만이 _filter_state 에 축적되어 부팅된다.

이 LPF 계층이 엑스포넨셜 곡선 맵핑 로직 뒤편에 탠덤(Tandem)으로 직렬 결합됨으로써, PX4의 ManualControl 모듈은 마치 진공 상태를 유영하는 듯한 기계적 매끄러움과, 극단적 플립 기동 시의 폭발적 타격감을 시스템의 런타임 제약 지연(Latency) 없이 하드 리얼타임(Hard Real-time) 환경에서 동시 달성하는 찬란한 비행 감각(Flight Feel)을 완성하게 된다.
