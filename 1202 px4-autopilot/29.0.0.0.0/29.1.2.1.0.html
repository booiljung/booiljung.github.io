29.1.2.1. 입력단: input_rc_s 구조체의 원시 타임스탬프 처리 및 다채널 배열 신호 수신 로직

본 소절에서는 PX4-Autopilot 수동 비행 데이터 파이프라인의 물리적 관문(Gateway)이자 최말단 입력 자료구조인 input_rc_s 구조체의 소스 코드 내부 명세(Specification)를 심층 해부하고, 송수신기(Transceiver) 하드웨어 레이어에서 포집된 타임스탬프(Timestamp) 타임라인과 다채널 데이터 배열이 uORB ми들웨어를 통해 연산 코어로 수신 및 처리되는 커널 로직을 조명한다.

1. input_rc_s 구조체 설계와 하드웨어 추상화(Hardware Abstraction) 패러다임

PX4 소스 코드 트리의 msg/input_rc.msg 텍스트 정의 파일(빌드 시 input_rc_s C++ 헤더 파일로 자동 트랜스파일 및 오토제네레이션 됨)은 특정 기성 드론 하드웨어 제조사의 폐쇄형 프로토콜(예: Futaba S.BUS, Spektrum DSMX, TBS Crossfire 등) 규격에 소프트웨어적으로 종속되지 않는 범용적(Generic) 형태의 데이터 명세와 아키텍처를 보유하고 있다.

// msg/input_rc.msg (개념 해석을 위한 주요 필드 발췌)
uint64 timestamp             # 구조체가 발행된 시스템 부팅 후 마이크로초(us) 시간
uint64 timestamp_last_signal # 마지막으로 '유효한' 무선 프레임이 수신된 시스템 시간 (us)
uint8 channel_count          # 현재 송신기 하드웨어로부터 디코딩된 유효 채널의 총 개수
int32 rssi                   # 수신 신호 강도 지시자(Received Signal Strength Indicator) (-100 ~ 0, 혹은 0 ~ 255 척도)
bool rc_failsafe             # 하드웨어 칩셋 수신기(Decoder) 자체 레벨에서의 페일세이프 활성화 판별 플래그
bool rc_lost                 # 소프트웨어 타임아웃 혹은 하드웨어 판정에 의한 무선 링크 신호 소실(Loss) 추론 플래그
uint16[18] values            # 물리적 무선 채널 데이터 배열 (일반적으로 1000 ~ 2000 범위 펄스 폭 매칭, 다중 채널 지원)

본 런타임 이진 페이로드 구조체 내에서 양적으로 가장 절대적인 비중을 차지하는 values 배열 컬렉션은 수신된 RC 채널 신호의 ’원시 물리 치환량(Raw Magnitude)’을 배열형 변수로 그대로 메모리에 연속 매핑해 둔다. 구형 아날로그 직렬 PWM 핀 수신기 보드를 운용할 경우에는 각 채널 포트별 1000\mu s ~ 2000\mu s 사이의 변위 펄스 폭 정수치 데이터가 가감 없이 그대로 담기며, 최신의 고속 디지털 직렬(Digital Serial) 수신기(예: CRSF)를 시스템에 연동할 경우 하드웨어 수신 모듈(src/drivers/rc_input) 뎁스(Depth) 내부의 프레임 파서(Parser) 서브루틴이 해당 프로토콜 통신 스펙에 맞추어 비트 시프트 연산과 헥스(Hex) 구문 수치값을 정확히 디코딩 파싱(Parsing)한 직후 구획된 이 공통 16비트 메모리 배열 변수에 순차적으로 밀어 채워 넣는 구조로 전가된다.

2. 절대 시간 타임스탬프(Timestamp)에 기반한 강건한 상태 유효성(Validity) 검증 로직

공중 기동을 통제하는 무선 RC 입력 버퍼 데이터에서 기구학적 채널 스틱 편차 변위값 못지않게 항공기 생존에 치명적으로 위중한 비중을 가지는 백본(Backbone) 데이터 페이로드(Payload)는 다름 아닌 시스템 시간 좌표(Timestamp) 이다. input_rc_s 객체에는 이 데이터 패킷 구조체가 uORB 공유 메모리 체제 버스 상에 최신으로 릴리즈(Issue)된 시점을 통고하는 timestamp 인뎁스 외에도, 무선 통신 주파수 신뢰도 공백 현상 결여 시점을 역추산하기 위한 마커 스탬프인 timestamp_last_signal 변수 릴레이가 추가 메모리로 엄격히 파티셔닝(Partitioning) 할당되어 있다.

이러한 이중 타임스탬프 스파이크(Spike) 변수들의 주체적인 생성 목적은 커맨더(Commander) 최고 관리자 모듈 또는 모드 매니저의 제어 루프가 다운타임(Downtime), 스레드 지연 타임아웃(Timeout) 록업 방어 및 조종자 신호 기아(Starvation) 단절 위급 상태를 스스로 감지 도출해 내는 데 있다.
src/modules/manual_control 데몬 루틴이나 상위 FSM(유한 상태 기계)은 매 루프 틱마다 시스템 델타 시간차 방정식 T_{diff} = (T_{current} - T_{last\_signal}) 로직을 구동하여 마이크로초 윈도우 단위로 추적 실시간 연산한다.

  • 신호 단절 논리 판정: 만약 비행체 공중 배터리 방전이나 전파 음영지대 진입, 송신 전원 차단 연결이 끊어지거나 수신 안테나가 전리층 또는 지형에 차폐되어 델타 타임 T_{diff} 수치가 펌웨어 시스템 구동 파라미터 보틀넥으로 사전에 엄밀히 정의된 커스텀 임계 수용 한계 시간(예: COM_RC_LOSS_T 파라미터 룰셋 기준, 디폴트 약 500,000\mu s = 0.5초 경과) 허용치를 초과(Exceeding)할 경우, PX4 메인 시스템은 조종자의 휴먼 신호가 소실 및 브로큰(Broken) 된 것(rc_lost == true 플래그 활성 캐시)으로 즉각 엄정히 논리적 간주 상태에 진입한다.
  • 하드웨어 확정급 Failsafe 감별: 나아가 드론 조종 수신기 모듈 하드웨어 자체가 내장 지원하는 수신 단절 전용 물리 페일세이프(Failsafe) 플래그 1비트가 전기적 하이(High) 신호로 켜져 송출될 경우(rc_failsafe == true 락인), 시스템 코어는 이는 단순한 패킷 지연 타임아웃 핑계 기류 지연과는 질적으로 근본이 다른, 무선 하드웨어 레이어 관점 단에서의 완벽하고 확정적인 연결 비상 절단(Severe Link Broken) 상황 선언이므로 상위 모듈 매니저가 이 구조체의 최상위 경고 플래그를 읽어낸 뒤 지체 없이 이륙 원점 회빙 RTL(Return-to-Launch), 비행 강제 종료 패러슈트(Flight Termination) 메커니즘을 핫 점화 트리거(Hot Triggering) 시키도록 구동한다.

3. 원시 제어 배열 데이터 처리 아키텍처의 구조적 특징: 의미론적 비분리성(Semantic Non-separation) 패러다임

본 1차 데이터 게이트 진입단 수용 객체인 input_rc_s 구조체의 C++ 데이터 조립 및 처리 소스 코드상 논리 아키텍처 로직에서 공학적으로 가장 유의미하게 눈여겨 숙지해야 할 철학적, 아키텍처적 특이점은 바로 철저하게 고립된 의미론적 비분리성(Semantic Non-separation) 이다.

이 메모리 구조체 내부 인덱스를 할당받은 values[0] 배열 요소 데이터 덩어리가 기체 물리의 롤(Roll) 공력 조종 축을 대변하는지, 아니면 단순한 페이로드 짐벌 카메라의 상하 틸트(Gimbal Tilt) 조도 조종 레버인지는 물리 포트를 담당하는 src/drivers/rc_input 디바이스 드라이버 로우(Low) 레벨 계층에서는 아예 코딩 관심사 밖의 범주이다. 드라이버는 오로지 자신에게 주어진 하드웨어 임무, 즉 외부 무선 포트로 수집 및 포집된 아날로그/디지털 트랜지스터 전기적 신호의 증폭된 정수값 배열을 있는 그대로 아무런 조작 왜곡 훼손 없이 복사 퍼블리싱(Publishing)한다는 현대 객체 지향 공학의 절대 규칙 중 하나인 단일 책임 원칙(SRP, Single Responsibility Principle) 사상만을 엄격하고 우직하게 고수 및 구현한다.

이렇게 추상적 순수 데이터로 수집 처리된 물리적 원시 바이트(Byte) 배열 패키팅 덩어리는 드라이버 종속의 굴레를 탈피한 채 비로소 다음 파이프라인 정제 단계인 manual_control 독립 데몬 시스템 모듈 프로세스로 안전하게 직렬 토스(Toss)되며, 이 상위 2차 논리 처리 모듈 코어 파이프 내부의 RC 파라미터 매치업 맵핑 연산 스케줄(예: MC\_ROLL_MAP = 0 인덱스 주소 링킹 등) 게이트를 무사히 통과한 뒤에야 비로소 비행 제어 지령 믹서에 요구되는 절대적 기구학적 ’방향성의 고유 의미 및 권한’을 온전하게 물리적으로 취득 및 할당받게 된다. 이는 지구 상에 난립하는 다양한 이기종 서드파티 하드웨어 무선 수신기 채널 프로토콜의 스펙 스파게티 복잡성을 정밀한 PX4 시스템 소프트웨어 코어 제어 커널 스택(Core Control Stack) 영역으로부터 완전히 차단 벽결리 격리 추상화(Hardware Protocol Abstraction)시켜 덤프 결합도(Coupling Factor)를 극한까지 낮추는 세계 최고 수준 파일럿 아키텍처적 구조적 우위를 장엄하게 달성하게 하는 원동력이 된다.
