29.2.3.2. 엑스포넨셜 커브 피팅: 중앙부 미세 제어를 위한 3차 다항식 모델(Cubic Polynomial Model) 매핑 소스 (math::expo())

본 소절에서는 src/lib/mathlib 의 정수(精髓)라 할 수 있는 math::expo() 함수의 최적화된 C++ 내부 구현을 해부한다. 이 모듈은 선형적인(Linear) 조종기 입력 궤적을 인간의 아날로그적 감각에 부합하는 비선형(Non-linear) 곡선으로 휨 가공(Warping)하는 역할을 담당한다. 특히, 고비용의 지수 함수(exp())를 배제하고 하드웨어 FPU에 극도로 친화적인 3차 다항식(Cubic Polynomial) 블렌딩 기법을 채택하여, 트림(Trim) 중앙부의 해상도를 극한으로 팽창시키고 끝단(Ends)의 추력비를 기하급수적으로 폭발시키는 역학적 모델링을 증명한다.

1. Expo(지수) 매핑의 항공 역학적(Aerodynamic) 목적과 인체공학적 당위성

멀티로터나 3D 아크로배틱 고정익 기체는 질량 중심(CG)을 축으로 회전할 때 폭발적인 기동성(Agility)을 확보하기 위해 매우 높은 최대 각속도(Max Angular Rate, 예: 1000^\circ/s)로 셋팅되는 경우가 잦다. 선형 제어(Linear Control) 환경 하에서, 스틱을 50% 기울였을 때 기체가 500^\circ/s 로 맹렬히 회전한다면 인간의 반사 신경으로는 중앙부 근처에서 정밀한 지향(Aiming)이나 부드러운 호버링(Hovering)을 통제하는 것이 구조적으로 불가능해진다.

조종자는 결국 두 가지 상충되는(Trade-off) 요구사항을 지니게 된다.

  1. 정밀도(Precision) 구간: 스틱 중앙 \pm20\% 영역에서는 스틱을 많이 움직여도 기체가 아주 미세하게만 둔감 반응하여 정밀한 위치 고수(Position Hold)가 가능해야 한다.
  2. 반응성(Responsiveness) 구간: 반대로 스틱을 80\% \sim 100\% 로 끝까지 거칠게 밀어붙일 때는 둔감했던 기울기가 가파른 절벽처럼 솟구쳐올라 순간적인 회피 기동이나 플립(Flip)을 압도적인 속도로 구사해야 한다.

이러한 S자 형상(혹은 기함수 형태의 곡선)의 전달 함수(Transfer Function)를 구현하는 것을 RC 업계 표준 용어로 ’Expo(Exponential)를 먹인다’고 관습적으로 칭한다.

2. 계산 복잡도(Computational Complexity) 회피를 위한 대수학적 3차 블렌딩(Cubic Blending) 설계

수학적으로 진정한 지수 함수(e.g., y = x \cdot e^{a(x-1)})를 펌웨어 코어에서 연산하는 것은 math.h 라이브러리의 무거운 테일러 급수(Taylor Series) 전개를 유발하여 마이크로 컨트롤러의 명령어 사이클을 치명적으로 갉아먹는다(알고리즘 복잡도 급증 및 FPU 스톨링 위험).

따라서 PX4 mathlib 설계자들은 진성 지수 함수 곡선과 오차율이 극히 적으면서도 대수(Algebraic) 곱셈 단 몇 번만으로 결과를 산출해 낼 수 있는 영리한 3차 다항식 혼합(Cubic Polynomial Blending) 추정 모델을 채택하였다.

// src/lib/mathlib/math/filter/Expo.hpp (혹은 동등한 math 네임스페이스 내 Expo 커널 로직)

namespace math {

/**
 * 선형 제어 공간을 비선형 지수 공간으로 맵핑하는 3차 다항식 피팅 함수
 * 
 * @param value 정규화된 선형 조종 스틱 물리량 (-1.0f ~ 1.0f)
 * @param expo 곡선의 곡률 깊이를 결정하는 비비례 가중치 (0.0f = 완전 선형, 1.0f = 완벽한 3차 곡선)
 * @return float 정규화 한계를 돌파하지 않는 휜(Warped) 제어값 (-1.0f ~ 1.0f)
 */
inline float expo(float value, float expo)
{
    // 선형 입력값 'value'는 이미 29.2.2.1절을 통해 데드존 및 트림 스케일링을 완벽히 마친 무결점 상태임
    
    // 3차항 도출: powf(value, 3.0f) 와 같은 고비용 API 절대 호출 금지
    // 캐시 L1 친화적인 단순 연속 실수 곱셈(FMA 지시어 최적화 가능성) 2회로 3제곱 공간(Volume) 연산 완료
    float value_cubed = value * value * value; 
    
    // 블렌딩(Blending) 연산: 선형 항(x)과 3차 항(x^3) 사이의 비율적 줄다리기
    // expo(가중치) 파라미터가 0.0이면 value만 남고, 1.0이면 value_cubed만 남음
    return (1.0f - expo) * value + expo * value_cubed;
}

} // namespace math

이 알고리즘의 우아함은 그 간결성(Simplicity)에만 머물지 않는다. f(x) = (1-E)x + Ex^3 수식(여기서 E 는 Expo 파라미터)은 x = 0 일 때 정확히 0을 통과폭(Intercept)으로 가지며, x = 1 일 때도 정확히 1을 출력하고, x = -1 일 때 반전 대칭 기합성(Odd Function)에 의해 무조건 -1 로 회귀하는 완벽한 고정점(Fixed-point Boundary) 사상(Mapping) 역학을 보장한다. 즉, 아무리 Expo 값을 극단적으로 높이더라도 조종 스틱의 최종 최대 도달량(Max Rates)은 절대 \pm1.0f를 초과하거나 모자람 없이 100\%로 정확히 매칭된다는 수학적 안정성을 스스로 내포하고 있다.

3. 부동 소수점 곱아드(FMA) 명령어 계층 최적화 추론

컴파일러(GCC 기반 크로스 컴파일러)가 -O2 나 -O3 런타임 최적화 플래그를 달고 위 C++ 코드를 빌드할 때, 마지막 줄의 리턴값인 (1.0f - expo) * value + expo * value_cubed 연산은 타겟 칩셋(예: Pixhawk 시리즈의 STM32 계열 Cortex-M7 부동 소수점 아키텍처)의 하드웨어 FMA (Fused Multiply-Add) 명령어로 단일압축(Condensation)될 가능성이 অত্যন্ত 농후하다.

FMA는 A \times B + C 형태의 연산을 레지스터 이동 없이 한 번의 클럭 사이클 내에서 중간 라운딩 오차(Rounding Error) 없이 일원화하여 처리하는 초고속 기계어 인스트럭션이다. 3차식을 활용한 PX4의 커브 피팅 방식은 분기문(Branch)이 철저히 배제되어 있을 뿐만 아니라 FMA 벡터화에 가장 이상적인 수식 골격을 제공함으로써, 수십 개의 다중 입력 채널에 대해 1000Hz (주기 1ms)를 상회하는 미친 듯이 타이트(Tight)한 제어 루프 속에서도, CPU가 다른 더 무거운 임무(EKF2 추정 등)에 매진할 수 있도록 자원의 병목 현상(Bottleneck)을 완벽하게 해갈해 주는 소프트웨어 공학의 숨은 보석 같은 알고리즘이다.
