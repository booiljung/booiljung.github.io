29.3.2.2. 메인 제어 루프 내 수동 제어 활성화 플래그 검사 시 PID 피드백 제어 연산 생략(Bypass) 코드 분석

본 소절에서는 고정익 자세 제어기(fw_att_control)의 메인 실행 스레드 내에서, uORB 기반의 모드 제어 플래그(vehicle_control_mode_s) 조합을 해석하여 피드백 제어(Feedback Control) 연산 블록을 통째로 생략(Bypass)하는 세부 조건문과 그에 따른 제어 공학적 와인드업(Windup) 방지 기법을 심층 분석한다.

1. vehicle_control_mode 비트 필드 플래그의 상태 판별 로직

PX4 아키텍처에서 개별 제어기는 자신의 현재 임무를 독립적으로 판단하지 않는다. 상위 궤적 지휘자인 Commander가 퍼블리싱하는 vehicle_control_mode_s 토픽의 불리언(Boolean) 플래그 조합만이 유일한 법전이다. FixedwingAttitudeControl의 메인 루프(Run()) 시작점에서는 이 플래그들을 추출하여 현재 시스템이 순수 수동 제어 상태인지 감별한다.

// src/modules/fw_att_control/FixedwingAttitudeControl.cpp 의 제어 루프 도입부
const bool is_manual = _vcontrol_mode.flag_control_manual_enabled && 
                      !_vcontrol_mode.flag_control_attitude_enabled &&
                      !_vcontrol_mode.flag_control_rates_enabled;

이 논리곱(AND) 연산은 매우 엄격한 3중 조건을 갖는다.

  1. flag_control_manual_enabled == true: 조종자의 RC 또는 MAVLink 수동 입력이 유효하게 들어오고 있는가?
  2. flag_control_attitude_enabled == false: 외부 각도(Attitude) 추종 제어기가 반드시 꺼져(Off) 있어야 한다.
  3. flag_control_rates_enabled == false: 내부 각속도(Angular Rate) 추종 제어기마저 꺼져 있어야 한다.

즉, 오토파일럿(Autopilot)의 어떠한 개입도 철저히 배제된, 기체의 안정화 루프가 완전히 해제된 상태(Un-stabilized)임을 증명하는 수학적 논리식이다.

2. 블록 생략(Bypass) 및 제어권 이양 구조

위의 is_manual 플래그가 참(True)으로 판별되면, 메인 루틴은 수백여 줄에 달하는 자율 비행 추종 로직을 완전히 뛰어넘어(Jump) 다이렉트 패스스루 공간으로 진입한다.

if (is_manual) {
    // 1. [순수 수동 제어 블록]
    // 29.3.2.1절에서 살펴본 FlightTaskManual 이 구동되는 구역이다.
    // 여기서는 어떠한 피드백(Feedback) 연산도 일어나지 않는다.
    
    // ...FlightTaskManual을 통한 _actuator_controls 버퍼 갱신...
    
} else if (_vcontrol_mode.flag_control_attitude_enabled) {
    // 2. [자세 제어(Attitude Control) 블록]
    // Stabilized, Mission 등에서 타겟 오일러 각을 추종하여 목표 각속도를 도출
    _att_ctrl.update(_quat_state, _att_sp);
    _rate_sp = _att_ctrl.get_rate_setpoint();
    
} else if (_vcontrol_mode.flag_control_rates_enabled) {
    // 3. [각속도 제어(Acro) 블록]
    // 자세 제어가 꺼져 있더라도, 조종기가 지시한 목표 각속도를 유지하려는 부분적 피드백
    // _rate_sp 는 조종기 스틱 데이터로부터 바로 계산됨
}

if (_vcontrol_mode.flag_control_rates_enabled) {
    // [최종 각속도 PID 제어 루프 연산]
    // 2번이나 3번 블록을 거쳐 내려온 목표 각속도(_rate_sp)와 자이로 속도(_rate_state) 간의 오차 보정
    _actuator_controls = _rate_ctrl.update(_rate_state, _rate_sp);
}

이 제어 프레임워크의 if-else 분기 트리가 시사하는 바는 명확하다. is_manual 구역으로 빠져나간 데이터 흐름은 하단의 _actuator_controls = _rate_ctrl.update(...) 라는 거대한 PID 피드백 컨베이어 벨트를 타지 않는다. 제어 행렬곱과 에러 공분산 연산이 철저히 도려내어지면서(Bypassed), FlightTaskManual이 작성해 둔 원시(Raw) 액추에이터 버퍼가 흠집 없이 하류(Downstream)로 플러시(Flush)되는 것이다.

3. 적분 누적(Integral Windup) 방지 및 안전한 모드 복귀 메커니즘

만약 다이렉트 패스스루 상태에서 아무런 조치 없이 방치한다면 심각한 공학적 결함이 발생한다. 기체가 조종자의 손에 이끌려 심하게 롤(Roll)을 도는 동안, 백그라운드 환경에서는 현재 기체의 각도(45^{\circ})와 목표 각도(0^{\circ} 기본값)의 엄청난 차이(Error)가 초당 수백 번 연산되어 PID 제어기의 적분항(I-term) 메모리 뱅크에 끝없이 적립(Windup)되는 현상이다.

이후 조종자가 다시 자율 비행(Auto) 스위치를 켰을 때(즉, if(is_manual) 블록을 빠져나오는 순간), 꾹꾹 눌러 담겨있던 적분 에러가 P와 D 제어 파라미터를 압도하며 폭발적으로 터져 나와 서보 모터를 한계점(End-loop)까지 극단적으로 후려치는 ‘스냅백(Snap-back)’ 응답을 야기할 수 있다.

이를 사전에 분쇄하기 위해 PX4는 is_manual 구역 처음에 반드시 다음의 한 줄을 추가한다.

_rate_ctrl.reset();

이 reset() 인터페이스는 내부 PID 컨트롤러 객체의 모든 상태 변수(적분기, 미분용 저주파 필터 버퍼 이력 등)를 0 으로 초기화(Flush)한다. 매뉴얼 동작 구간 내내 자율 주행의 뇌를 백지상태로 유지시킴으로써, 언제 어느 때고 조종권이 오토파일럿으로 다시 인계되었을 때 가장 부드럽고 차분한 초기 상태(Initial State)에서 제어가 재개되도록 유도하는 필수 불가결한 안전망(Safeguard) 구조인 것이다.
