29.3. 매뉴얼(Manual) 모드 제어 바이패스(Bypass) 로직 분석

본 절에서는 PX4-Autopilot의 수동 제어(Manual Control) 파이프라인에서 가장 극단적이자 원초적인 형태인 직접 제어 바이패스(Direct Control Bypass) 로직의 소프트웨어적 아키텍처를 분석한다. 조종기의 스틱 입력이 복잡한 PID 피드백(Feedback) 제어기나 비행 태스크(Flight Task)의 궤적 생성기를 거치지 않고, 시스템의 최하단인 믹서(Mixer)와 액추에이터(Actuator) 모터 출력단으로 직행(Passthrough)하는 시스템적 경로와 그 한계 조건을 조명한다.

1. 기상천외한 우회로: 바이패스(Bypass) 아키텍처의 존재 이유

동시대의 비행 제어 시스템(FCS)에서 ’자동화’의 반대말은 ’수동 제어’이지만, 공학적 관점에서 철저한 수동 제어 모드는 컴퓨터의 연산 개입을 0\% 로 억제하는 것을 뜻한다. 조종자가 RC 스틱의 롤(Roll)을 50% 기울이면, 에일러론(Aileron) 서보 모터가 정확히 50%만큼 움직이는 가장 원초적인 1:1 선형 사상(Linear Mapping)이 바로 그것이다.

PX4 커널 내에서 이러한 제어 루프의 기저 생략(Bypass)은 다음과 같은 공학적 이점과 목적을 포괄한다.

  • 안전 장치의 최후 보루(Failsafe Backstop): 확장 칼만 필터(EKF2)나 IMU 등 코어 센서가 완전히 고장나 발산(Divergence)하는 상황에서도, 외부의 자세 추정값(Attitude Estimate)에 의존하지 않고 기체를 살려 귀환시킬 수 있는 퓨어(Pure) 아날로그 제어 경로를 오픈해 둔다.
  • 공기 역학적 테스트베드(Testbed): 고정익(Fixed-wing) 항공기의 최초 처녀 비행(Maiden Flight) 시, PID 제어기의 튜닝이 빗나가 기체가 발진(Oscillation)할 우려가 있을 때 순수 기구학적 특성만을 평가하기 위해 사용된다.
  • 최소 지연(Minimum Latency): 연산 파이프라인(Rate Controller 등)을 우회하므로 시스템 전체의 위상 지연(Phase Lag)이 최소화되며, 가장 즉각적인 응답성을 보장한다.

2. 기체 동역학 유형별 바이패스 적용의 비대칭성(Asymmetry)

결론부터 말하자면, PX4 커널은 모든 기체 템플릿(Airframe)에 대해 평등하게 바이패스 로직을 허용하지 않는다. 이는 항공기의 선천적인(Inherent) 기구학적 동적 안정성 유무에 철저히 기인한다.

  1. 고정익(Fixed-wing) 및 로버(Rover): 이들은 전진 비행 시 꼬리 날개에 의한 공기역학적 복원력이 존재하거나, 지면과 접촉하여 마찰에 의해 지탱되는 구조적 안정성을 지니고 있다. 따라서 PID 제어기가 꺼진 상태로 조종자가 날개(Control Surface)를 직접 제어하더라도 물리적인 체공 및 주행 유지가 가능하다. 펌웨어는 이들에 대해 완벽한 바이패스를 승인한다.
  2. 멀티로터(Multirotor): 쿼드콥터와 같은 다림발이 기체는 선천적으로 동태적 불안정상태(Dynamically Unstable)에 있다. 초당 수백 번 연산되는 자이로(Gyro) 기반 각속도 피드백 루프 없이는 인간의 반사 신경만으로는 절대로 체공할 수 없다. 따라서 멀티로터의 Manual 모드는 사실상 순수 바이패스가 원천 차단된, 각속도 제어 이상의 보정이 개입하는 의사 수동(Pseudo-manual) 상태로 설계될 수밖에 없는 렌즈적 한계가 존재한다.

3. 하위 구조 분석을 위한 스코핑(Scoping) 요약

이어지는 29.3.1절부터 29.3.3절까지의 파이프라인 해부에서는, PX4의 Commander 상태 머신이 어떻게 멀티로터의 순수 수동 제어를 강제로 차단하고 대안 모드(Fallback)로 라우팅하는지 그 보안 체계를 살펴본다(commander.cpp).

더불어 fw_att_control 로 대변되는 고정익 시스템에서 manual_control_setpoint 가 복잡한 비행 태스크(FlightTask)를 어떻게 우회(Bypass)하여 믹서의 actuator_controls 로 직격하게 되는지에 대한 하위 포인터 전달 및 메모리 할당 구조를 심층적으로 묘사한다. 마지막으로, EKF 센서 추정에 치명적 결함이 도래했을 때 시스템이 최후의 수단으로 매뉴얼 모드 제어권을 사수하는 소프트웨어학적 메커니즘을 증명할 것이다.
