29.2.2.3. 스틱 방향 역전 비트 연산 및 부호 반전 알고리즘 로직

본 소절에서는 ManualControl 모듈의 정규화 파이프라인(Normalization Pipeline) 말단에 위치한 스틱 방향 역전(Reversal, Invert) 메커니즘을 상세히 다룬다. 사용자의 조종기 하드웨어 배선 상태나 기체의 기구학적 요구사항에 따라 특정 제어 축(예: Pitch 스틱을 밀었을 때 값이 감소해야 하는 경우)의 극성(Polarity)을 소프트웨어적으로 뒤집어야 할 때, PX4 커널이 파라미터 비트마스크(Bitmask)를 활용하여 어떻게 O(1)의 시간 복잡도로 무차원(Dimensionless) 실수 값의 부호를 반전(Sign Inversion)시키는지 그 C++ 연산 논리를 집중 분석한다.

1. 조종기 축 역전(Reversal)의 기하학적 필요성과 파라미터 추상화

항공 공학적 관습에 따르면, 엘리베이터(Pitch) 스틱을 앞으로 밀면 기수가 숙여지며(Nose Down) 전진하는 것이 일반적이다. 이 때 PX4 내부의 기구학적 제어 좌표계(e.g., FRD - Forward, Right, Down)를 준수하려면 피치 정규화(Normalized Pitch) 제어값(Setpoint)은 통상 음수(Negative) 측으로 이동해야 한다. 반면, 스로틀(Throttle) 스틱을 위로 올릴 때는 프로펠러의 추력을 증가시켜야 하므로 양수(Positive) 기댓값을 갖는다.

그러나 물리 조종기(Transmitter) 내부의 가변 저항(Potentiometer) 단자 배선이나 센서 지향에 따라, 스틱을 위로 밀었을 때 뿜어져 나오는 원시 PWM 시간 값(Microseconds)이 증가(1500 \rightarrow 2000)할 수도 있고 반대로 감소(1500 \rightarrow 1000)할 수도 있는 극심한 하드웨어 파편화가 존재한다.

이러한 물리 짐벌의 전기적 편향성(Directional Bias) 극복을 위해, PX4 시스템은 플래시 메모리에 RCx_REV (여기서 x는 스틱 채널 인덱스 1~18) 파라미터를 마련해 둔다. 기본적으로 이 파라미터는 1.0f 상태를 유지하지만, QGroundControl을 통해 역전 체크박스를 활성화하면 시스템은 이 값을 -1.0f 로 전환하여 저장한다.

2. 부호 반전 스칼라 곱연산(Scalar Multiplication) 및 논리 마스킹 로직

앞선 29.2.2.1절 및 29.2.2.2절에서 다룬 apply_calibration() 함수를 완전히 빠져나온 직후, 정규화된 값(예: N_{out} \in [-1.0, 1.0])은 마지막 교정 단계인 역전(Reversal) 필터를 거친다. PX4 커널 개발진은 장황하고 값비싼 조건 분기문(If-else Branching)을 배제하고, 수학적 행렬 곱이나 대수학적 부호 치환식을 사용하여 CPU 연산 파이프라인의 캐시 미스(Cache Miss)를 예방한다.

// src/modules/manual_control/ManualControl.cpp의 비례/역전 스케일링 개념 발췌
// _param_rc_rev 파라미터 배열은 오직 1.0f 또는 -1.0f 두 가지 값만을 내포함

void ManualControl::process_rc_input() {
    float roll_norm = apply_calibration(raw_roll, min_roll, center_roll, max_roll, deadzone);
    
    // 파라미터 비례 스칼라 곱을 이용한 부호 치환(Sign Flipping)
    roll_norm = roll_norm * _param_rc_rev[map_roll].get(); 
    
    // 역전 이후 혹시라도 발생할 수 있는 산술 한계 돌파 방지를 위한 2차 램프(Clamp) 방어선
    _manual_control_setpoint_pub.roll = math::constrain(roll_norm, -1.0f, 1.0f);
}

위의 C++ 런타임 코드는 파라미터 시스템이 부동 소수점(float) 1.0 또는 -1.0 을 반환하도록 모델링하여, 단순 곱셈 연산(Multiplication Instruction) 단 1회만으로 조종기 스틱의 물리적 전파 방향을 완벽히 반전(Invert)시켜 버리는 직관적이면서도 고속화된 수학 논리(Algebraic Logic)를 보여준다. 분기문(if (rev == true) val = -val;)을 피함으로써 최신 ARM Cortex-M7 마이크로프로세서의 부동 소수점 연산 유닛(FPU) 성능을 끝까지 착취(Exploit)하는 하부 계층 튜닝의 진가를 엿볼 수 있다.

3. 스위치 채널의 이산(Discrete) 논리 역전 비트 연산 체계

연속적(Continuous) 조종 스틱이 아닌, 2단/3단 토글 스위치(Auxiliary Switches)의 채널 맵핑 로직에서는 실수형 연산이 아닌 정수형 이진 트리거 로직 최적화가 가미된다. 앞서 보았듯 스위치는 내부적으로 -1 (하단), 0 (중앙), 1 (상단) 의 int8_t 시그니처 맵을 갖는다.

스위치 채널의 역전이 요청되었을 때, ManualControl 모듈은 내부 3단 필터의 평가 결괏값을 비트 조작(Bitwise Manipulation)이나 단항 마이너스 연산자(Unary Minus)를 통해 뒤집는다.

// 보조 스위치 채널(AUX) 부호 반전 개념 C++ 코드
int8_t switch_state = evaluate_switch_threshold(raw_value); // -1, 0, 1 중 하나 반환

bool is_reversed = _param_rc_rev[aux_channel_index].get() < 0.0f;
if (is_reversed) {
    switch_state = -switch_state; // 정수형 부호 비트 반전 (2의 보수 연산)
}

이러한 전역적인 부호 뒤집기 알고리즘의 보편적 적용을 통해, PX4 비행 제어기 코어(Flight Control Core)는 더 이상 하드웨어 송신기가 뿜어내는 기계적 특이성(Quirks)이나 조립 배선의 물리적 모순을 염려하지 않는다. ManualControl 클래스라는 하나의 거대한 차폐벽(Abstraction Barrier)이 온갖 지저분한 물리 신호를 완전히 소화(Digestion)해 주었기 때문에, 이 장벽 너머에 포진한 상층부의 핵심 커맨더(Commander) 모듈이나 자세 제어기(Attitude Controller)들은 수동 비행 조작이 언제나 이상적인 수학적 규범(Mathematical Standard, -1.0 \sim 1.0)과 올바른 위상(Phase)으로 도달한다는 것을 조건 없이 신뢰하고 순수 기구학적 연산에만 시스템 역량을 100% 매진할 수 있게 된다.
