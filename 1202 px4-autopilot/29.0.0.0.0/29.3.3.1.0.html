29.3.3.1. 확장 칼만 필터(EKF) 추정 상태 발산 시 상태 머신이 매뉴얼 모드 제어권을 최우선으로 유지하는 구조적 이유

본 소절에서는 확장 칼만 필터(EKF2)가 치명적인 수학적 발산(Divergence)을 일으켜 기체가 이른바 ‘비행 불능(Un-navigable)’ 상태에 빠졌을 때, 수많은 자동화된 페일세이프(Failsafe) 로직들을 뚫고 매뉴얼 모드(Manual Mode) 가 어떻게 시스템 제어권의 최상단 포식자(Apex Predator) 지위를 탈환하는지 그 상태 머신(State Machine)의 계층적 구조를 분석한다.

1. EKF 붕괴 시 페일세이프(Failsafe)의 연쇄 강등(Degradation) 폭포

보통의 비행 상태(예: Mission 모드)에서 GPS 다중 경로 에러나 지자기 교란으로 EKF의 local_position 혁신(Innovation) 오차가 임계값을 초과하면, PX4의 페일세이프 매니저는 즉각적으로 생존을 위한 프로토콜을 가동한다.

  1. RTL (복귀) 시도: 기체는 가장 먼저 이륙 지점으로 돌아가려 한다. 하지만 RTL은 글로벌 좌표계(GPS)가 필수적이다. EKF가 위치 추정을 포기했으므로 이 시도는 즉각 기각(Denied)된다.
  2. Altitude (고도 유지) 강등: 위치를 모른다면 기압계를 이용해 고도라도 유지하며 표류하려 한다. 허나 기압계마저 결빙되었거나 EKF의 수직 계측마저 무너졌다면 이 역시 기각된다.
  3. Stabilized (자세 유지) 강등: 고도마저 모른다면 자이로스코프에 의존해 수평(Attitude)만 맞춘 채 천천히 하강(Descend)하려 시도한다. 그러나 EKF 쿼터니언(Quaternion) 자체가 발산했다면 기체는 뒤집힌 채로 자신이 수평이라고 착각하며 땅으로 곤두박질치는 치명적 궤도(CFIT)를 그린다.
  4. Flight Termination (비행 종료): 모든 상태 추정이 불가능하다고 판단되면, 최악의 경우 모터를 정지시키고 기체를 추락시키는 극단적인 셧다운이 발생할 수 있다.

오토파일럿(Auto-pilot)이라는 맹인(Blind) 소프트웨어에게 목숨을 맡겨둔다면 위와 같은 비극적인 상태 강등(Degradation Waterfall)의 끝은 추락뿐이다.

2. 매뉴얼 탈출(Manual Override)을 위한 상태 머신의 맹점(Blind Spot) 개방

이 참담한 강등 사슬을 끊어낼 수 있는 유일한 ’데우스 엑스 마키나(Deus Ex Machina)’는 조종자의 매뉴얼 스위치 토글(Toggle) 행위이다. Commander 모듈은 조종자의 수동 개입 신호(flag_control_manual_enabled)가 포착되는 즉시, 진행 중이던 모든 페일세이프 강등 시퀀스를 일시 중지(Override)하고 매뉴얼 라우팅 판별식으로 제어권을 강제 점프(Jump)시킨다.

// src/modules/commander/state_machine_helper.cpp 의 내부 평가 로직
transition_result_t set_main_state(vehicle_status_s &status, bool &changed)
{
    // ... 페일세이프 인터럽트 로직 ...

    case commander_state_s::MAIN_STATE_MANUAL:
        // [EKF 면책 특권구역]
        if (status.rc_signal_lost) {
            return TRANSITION_DENIED; // 조종기와 통신이 안 되면 당연히 수동 제어 불가
        }
        
        // EKF의 상태를 증명하는 아래의 엄격한 유효성(Validity) 검사들이 문맥에서 완전히 누락되어 있음!
        // - status.condition_local_position_valid
        // - status.condition_global_position_valid
        // - status.condition_attitude_valid
        
        if (status.is_rotary_wing) {
            status.nav_state = vehicle_status_s::NAVIGATION_STATE_STAB; 
        } else {
            status.nav_state = vehicle_status_s::NAVIGATION_STATE_MANUAL;
        }
        
        changed = true;
        return TRANSITION_CHANGED; // 조건 없는 무사 통과(Free-pass)
}

코드에서 목격할 수 있듯, MAIN_STATE_MANUAL 전환을 심사하는 분기문 내부에는 EKF의 건강 상태(condition_*_valid)를 묻는 조건검사(if문)가 설계 단계에서부터 의도적으로 통째로 누락(Omission)되어 있다.

3. 구조적 권한 위임의 철학: 소프트웨어의 백기 투항(White Flag)

이러한 상태 머신의 ‘면책 특권(Immunity)’ 뷰는 극도로 치밀하게 계산된 로직이다. 소프트웨어가 자신의 센서 융합 풀(EKF)이 발산하여 현재 우주 공간의 어디에 있는지, 기체가 얼마나 기울어져 있는지 전혀 알지 못하는(Unaware) 상황에 처했을 때, 오토파일럿이 섣부른 판단으로 모터를 제어하려 드는 것은 재앙을 증폭시킬 뿐이다.

따라서 PX4 펌웨어는 상태 추정(State Estimation)에 대해 완전한 불확정성(Uncertainty)에 빠졌을 때, 자신(FC)의 추정값보다 지상에서 기체를 육안으로 바라보고 있는 조종자의 뇌(Human Control Center)와 손가락(Sticks)의 판단을 구조적으로 상위에 위치시킨다. 조종자가 매뉴얼 스위치를 치는 순간, FC는 EKF의 발산 에러를 무시하고 믹서(Mixer)로 통하는 직통 회로를 열어줌으로써(Direct Passthrough), 최후의 복원 기회를 아날로그의 물리적 세계로 완벽히 되돌려주는 것이다.
