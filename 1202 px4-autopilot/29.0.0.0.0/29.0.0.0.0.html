Chapter 29. 수동 비행 모드(Manual, Stabilized, Acro) 소스 코드 분석

무인항공기(UAV)의 가용 비행 모드 중 ‘수동 비행 모드(Manual Flight Modes)’ 계열은 조종자(Human Pilot)의 이산적(Discrete) 혹은 아날로그적(Analog) 입력 스틱 신호를 기체의 물리적 운동학 변수(목표 자세, 목표 각속도 등)로 병합하는 가장 근본적인 제어 루프를 구성한다. 최신 PX4-Autopilot 아키텍처 상에서 이 모드들은 더 이상 RC 수신기의 PWM(Pulse Width Modulation) 신호를 하드웨어 모터로 직접 바이패스(Bypass)하는 원초적인 형태가 아니며, 내부에 정밀하게 튜닝된 다중 루프(Multi-loop) 캐스케이드(Cascade) PID 제어기를 반드시 경유하는 동적 인터페이스 계층으로 동작한다.

본 장에서는 PX4 비행 제어 스택의 중추인 수동 모드(Manual), 안정화 모드(Stabilized), 그리고 아크로 모드(Acro) 가 펌웨어 소스 트리(src/modules/) 내에서 어떠한 이벤트 드리븐(Event-driven) 아키텍처로 구현되어 있는지를 심층적으로 해부한다. 외부의 무선 조종기(Remote Controller) 물리 입력단에서 출발하여, 시스템의 척추인 uORB 메시징 파이프라인을 관통하고, 파이널 믹서(Mixer) 모듈과 액추에이터 제어 명령(Actuator Control Command) 분배에 도달하기까지의 전방위적 데이터 흐름을 C++ 클래스 레벨에서 정밀하게 추적할 것이다.

1. 주요 분석 목표와 전개 방향

  1. 비행 제어 스택 라우팅 아키텍처 구조화: FlightModeManager 모듈이 수행하는 스레드 폴링(Polling) 로직과 런타임 내 동적인 비행 태스크(Flight Task) 할당 및 자원 분배 구조를 수학적 상태 머신(State Machine) 관점에서 파악한다.
  2. uORB 파이프라인의 엔드투엔드(End-to-End) 데이터 정규화: 조종기의 원시(Raw) 다채널 신호가 축별로 독립된 정규화 신호(Normalized Setpoint) 구조체로 변환되어 위치/자세 제어기로 위임(Delegation)되는 처리 채널 맵핑 로직을 검증한다.
  3. 동역학적 제어 차이점 증명: 기체의 평형을 자가 복원하는 Stabilized 모드의 절대 자세 각도 지령 구조와, 드론 레이싱 등에 주로 활용되는 Acro 모드의 각속도 지령 제어 분기점 간의 C++ 소스 코드 레벨 연산 차이를 조명한다.

PX4의 소스 코드 구조는 Ardupilot과 대비하여 독립 프로세스 기반의 모듈성(Modularity)을 극한으로 추구하고 있어, 상호 모듈 간의 메모리 간섭(Memory Interference)을 철저히 락-프리(Lock-free)로 차단하고 있다. 본 장의 아키텍처 추적 과정은 제어공학자나 오픈소스 소프트웨어 개발자가 드론의 조종감(Stick Feel)을 미세 객체 지향적으로 튜닝하거나, 시스템 코어에 독자적인 커스텀 수동 모드를 이식할 때 반드시 준수해야 하는 NuttX RTOS 기반의 스케줄링 가이드라인을 제공할 것이다.
