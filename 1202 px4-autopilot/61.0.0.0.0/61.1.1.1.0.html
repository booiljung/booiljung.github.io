61.1.1.1. 발행-구독(Publish-Subscribe) 모델의 비순환 방향 그래프(DAG) 기반 상태 전이 메커니즘

자율 비행 에이전트(Autonomous Agent) 운용을 위한 고도화된 오프보드(Offboard) 제어 아키텍처에서는 다수의 병렬 데이터 스트림(Data Stream) 무결성을 유지하는 것이 절대적이다. 최근 PX4-Autopilot(v1.14 기준)은 ROS2 기반의 오프보드 연산 환경과 결합하기 위하여 Micro XRCE-DDS 미들웨어 브리지를 핵심 인프라로 채택하였다. 이러한 이종 소프트웨어 미들웨어 간의 통신은 퍼블리시-서브스크라이브(Publish-Subscribe) 객체 지향 모델을 기반으로 하며, 엄밀한 의미에서 비순환 방향 그래프(Directed Acyclic Graph, DAG) 형태의 철저한 상태 전이(State Transition) 메커니즘을 내포한다.

1. 퍼블리시-서브스크라이브 네트워크 트리의 수리적 모델링 (Architecture Understanding)

ROS2 노드(Node) 아키텍처와 PX4 펌웨어 내부의 uORB 메시지 토픽(Topic) 간 교류 네트워크는 수학적으로 유향 그래프(Directed Graph) G = (V, E)로 추상화되어 모델링될 수 있다. 여기서 정점 집합 V는 실제 메모리 공간 내에서 데이터를 생산하거나 소비하는 에이전트 추론 노드 및 제어 모듈 주체들을 나타내며, 간선 집합 E는 데이터 분배 규칙에 의해 형성된 특정 주제(Topic)에 대한 직렬화된 페이로드 데이터 흐름을 의미한다.

데이터 루프(Data Loop) 현상으로 인한 제어 오차의 양의 피드백(Positive Feedback) 증폭 발산을 구조적으로 방지하기 위하여, 이 논리적 연결 그래프 G는 어떠한 폐경로(Closed Path) 즉 닫힌 루프도 절대로 허용하지 않는 비순환 방향 그래프(DAG, Directed Acyclic Graph) 위상 상태 조건을 강제로 충족시키도록 설계되었다.

본 위상 수학론적(Topological) 소프트웨어 설계 패턴에서는 데이터 패킷 상태의 역류(Backflow) 현상을 근본적으로 차단하기 위해 세심한 노력이 들어간다. PX4 메인 모듈에서 외부로 센서 데이터를 내보내는 송출 간선(VehicleOdometry)과, 오프보드 알고리즘 노드가 독자적으로 산출하여 외부에서 PX4 내부 제어기로 유입되는 지시 명령 유입 간선(TrajectorySetpoint)은 서로 교차되지 않는 완전히 분할된 네임스페이스(Namespace) 경로를 구축하여 강력한 방향성 채널을 유지한다.

2. 비동기 상태 전이(State Transition) 메커니즘을 통한 동역학 제어 분리

DDS 미들웨어가 도입한 이러한 DAG 기반 퍼블리시-서브스크라이브 패턴은 자율주행기기 내의 상태 머신(State Machine) 동기화의 본질을 혁신적으로 변화시켰다.
과거 지상 관제 모델 기반 하에서는 컴패니언 컴퓨터가 PX4의 내부 제어 루프 아키텍처 속력을 정확하게 동기 추종(Synchronous Tracking)하며, 순차적으로 MAVLink의 상태 폴링(Polling) 요청과 응답 절차 이행을 전담해야 하는 강한 락킹(Locking) 절차적 제약이 존재했다.

하지만 최신 PX4의 uXRCE-DDS 에이전트 체계 하에서는 상태 개입의 패러다임이 비동기 이벤트 전달 방식으로 변화한다. 오프보드 측의 범용 ROS2 비전 및 궤적 계획(Path Planning) 컴퓨터 액터들은 단지 “발행자(Publisher)” 역할로서 미래의 목표 궤적 상태 행렬 값을 DAG 흐름 망에 제한 없이 즉발적으로 밀어 넣기(Push/Publish)만 하면 된다. 반대로 PX4 내부의 고주파수 다중 로터 속도 제어기(Rate Controller)는 자신만의 독립적인 마이크로 타임 틱(Tick)에서 이를 단순히 최신값만 “구독(Subscribe)“하여 자신의 내부 동역학(Dynamics) 로직에 맞게 모터 PWM 가중치를 보정 조작한다. 요컨대, 메인 컴퓨팅 노드 간의 상태(State) 결합도가 완벽하게 시간 도메인에서 분리(Decoupling)되는 소프트웨어 아키텍처적 성과를 거두게 된 것이다.

3. Ardupilot의 중재자 기반 상태 라우팅 구조와의 차별성

이러한 DAG 구조는 GCS 관제 기술의 관점에서 Ardupilot 시스템 모델이 갖는 구조적 복잡도와 엄청난 철학적 괴리이자 차이를 형성한다. Ardupilot 환경은 다중 오프보드 분산 에이전트 처리를 관장하기 위해 주로 MAVProxy와 같은 MAVLink 기반 중재자 프로세스 호스트(Mediator Process Host)를 두어 허브 앤 스포크(Hub-and-Spoke) 형태로 전체 패킷을 라우팅(Routing)시킨다.

  • Ardupilot 통신 에코 (Echo) 취약성: 단일 중재자(Mediator) 라우터에 데이터 병목 누적이 우려되며, 상태 천이 전파 정보의 흐름 규칙이 전방향성을 지니거나 단방향 스패닝 트리(Spanning Tree) 형태를 엄격하게 지켜내기 어렵기 때문에, 애플리케이션 개발 시 잘못된 MAVLink 라우팅 아이디(System ID, Component ID)를 설정할 경우 양방향 통신 에코(Echo Protocol Loop)에 의한 무한 루프 과부하 장애가 도출되기 십상이다.
  • PX4-Autopilot DAG 기반 무결성 인프라: uXRCE-DDS 에이전트와 PX4 간의 하위 토픽 매칭(Topic Matching) 알고리즘은 오직 철저하고 편향된 퍼블리시-서브스크라이브 페어링(Pairing) 논리에 종속된다. 데이터의 생산자(Writer)와 소비자(Reader) 간 메시지 조작 및 접근 권한이, 소스 코드 빌드 및 커널 프로비저닝 단계 전부터 논리 계층의 DAG에 의해 엄격하게 제약 및 구속되기에, 통신 반사 및 에코에 의한 자율 에이전트 제어 루프 붕괴 현상이 원천적으로 차단된다.

4. 통신 무결성 및 시스템 제어공학 관점의 종합 지견 (Control Understanding)

최신 드론 융합 논문 “Directed Acyclic Graph Topology in Cyber-Physical Systems“에서 빈번하게 연구되고 다루어지는 바와 같이, 이러한 DAG 기반 분산 미들웨어 네트워크 토폴로지는 실시간 커널(RTOS)상의 태스크 체인(Task Chain) 호출 절차의 완전한 복잡도 선형화(Linearization)를 이룩하는 데 막대하게 기여한다.

C++ 소스 코드 관점에서 PX4 내부 데이터 브리지 애플리케이션인 uxrce_dds_client는 DAG 전송 망의 싱크(Sink) 정점 역할 동작 리스너와 소스(Source) 정점 역할 발송 스트림을 철저히 분산 격리된 세퍼레이트 스레드(Separate Thread) 컨텍스트로 구동 관리한다. 이는 외부 요인으로 오프보드 컴퓨터 운영체제 시스템이 과부하에 봉착하거나 비가역적인 통신 단절(Data Link Loss) 안전 불감 상태에 직면하더라도, 역방향인 기체 내부 센서 자체의 데이터 송출 스트림의 독립성 생존을 침해하지 않도록 보호벽 샌드박스 역할을 해준다. 이를 통해 남은 코어 제어 모듈이 독립된 안전 장치(Failsafe) 비상 복귀 로직을 오작동 없이 정상 동작시킬 수 있는 고도로 견고한 관제 생태계 신뢰성이 확보된다.
