61.2.1.1. 표준 리눅스 스케줄러(CFS)의 선점 지연(Preemption Latency) 한계 분석 및 완전 선점형(PREEMPT_RT) 커널 패치 적용의 위상 수학적 이점

1. 개요

PX4-Autopilot과 연동되는 GCS(지상 관제 시스템) 또는 동반 컴퓨터(Companion Computer) 내부의 ROS2 기반 자율 에이전트는, 센서 데이터 수집과 궤적 제어 루프를 명확한 데드라인(Deadline) 내에 완료해야 한다. 그러나 우분투(Ubuntu) 등 범용 리눅스는 처리량(Throughput)을 극대화하기 위해 완전 공정 스케줄러(Completely Fair Scheduler, CFS)를 기본 메커니즘으로 채택하고 있어 본질적인 선점 지연(Preemption Latency)의 한계를 내포하고 있다. 본 절에서는 위상 수학(Topology) 및 그래프 이론(Graph Theory) 관점에서 CFS의 한계를 분석하고, 완전 선점형 리눅스 커널 패치(PREEMPT_RT)의 적용이 최악 허용 지연(Worst-Case Execution Time, WCET) 보장에 어떠한 근원적 이점을 제공하는지 탐구한다.

2. 표준 리눅스 스케줄러(CFS)의 위상적 한계와 선점 지연 현상

표준 리눅스 커널의 CFS는 각 태스크의 가상 실행 시간(Virtual Runtime, vruntime)을 기준으로 적-흑 트리(Red-Black Tree)라는 위상적 자료 구조를 유지하며, 스케줄링 복잡도 O(\log N)을 달성한다. 이 구조는 평균적인 스케줄링에서는 높은 공정성을 달성하지만, 다음과 같은 선점 지연을 불가피하게 수반한다.

2.1 스핀락(Spinlock)에 의한 커널 공간 내 비선점 구간 형성

범용 커널에서는 한 스레드가 커널 공간 자원을 수정 중일 때 보호 메커니즘으로 스핀락을 사용한다. 스핀락이 획득된 상태에서 코드는 ’비선점 구간(Non-preemptible Section)’에 진입하며, 이 시점에 매우 긴급한 RT(Real-Time) 제어 태스크가 스케줄러 큐에 삽입되더라도, 락이 해제될 때까지 선점은 유보된다.

2.2 우선순위 역전(Priority Inversion)의 그래프 이론적 악순환

다수의 노드(ROS2 애플리케이션)가 공유 자원(예: 직렬 포트 락 등)을 차례로 점유하는 상황을 유향 그래프(Directed Graph) 상의 의존성(Dependency)으로 표시할 수 있다. 하위 우선순위 태스크(Low)가 자원을 소유한 상태에서 중간 우선순위 태스크(Medium)가 CPU를 선점해버리면, 정작 최우선순위 태스크(High)는 영구적으로 대기 상태에 빠질 수 있다(무제한의 선점 지연 발생). 이러한 구조적 약점은 논문 “Priority Inheritance Protocols: An Approach to Real-Time Synchronization” (1990) 에서 수학적으로 증명된 바 있다.

3. PREEMPT_RT 패치의 위상 수학적 이점 (Topological Advantages)

완전 선점형 패치인 PREEMPT_RT는 리눅스 커널의 전반적인 아키텍처를 하드 리얼타임(Hard Real-Time) 요구사항에 가깝도록 대대적으로 재구성한다. 그중에서도 핵심적인 변화는 범용 커널의 스핀락 대다수를 ’실시간 뮤텍스(rt_mutex)’로 대체하는 것이다.

3.1 그래프 사이클의 붕괴: 우선순위 상속(Priority Inheritance)

rt_mutex는 우선순위 역전 현상을 막기 위해 ’우선순위 상속 프로토콜(Priority Inheritance Protocol)’을 기본으로 제공한다. 자원을 점유 중인 하위 태스크 T_{low}는 자신을 대기하는 최상위 태스크 T_{high}의 우선순위를 일시적으로 상속받는다.
수학적으로 이는 스케줄링 대기열 그래프 상에서 대기 사이클(Wait Cycle) 자체를 절단하는 위상적 이점(Topological Benefit)을 갖는다. 우선순위 상승 덕분에 중간 우선순위 T_{medium} 태스크들의 진입이 차단되어, T_{high}가 겪을 수 있는 차단 시간(Blocking Time)은 명시적인 유한 상수(Finite Bound)로 수렴하게 된다.

3.2 선점 가능 구간(Preemptible Section)의 위상적 확대

스핀락이 rt_mutex로 대체됨에 따라, 인터럽트 컨텍스트가 아닌 커널 컨텍스트의 대부분 영역이 선점 가능한(Preemptible) 상태로 변경된다. 위상적으로 볼 때 시스템 전체의 상태 전이 다이어그램(State Transition Diagram) 상에서 ‘선점 불가(Non-interruptible)’ 계곡(Valley) 영역이 최소한의 임계 영역(Critical Section)으로 좁혀져, PX4-ROS2 간 주기적 통신 스레드의 지연시간 편차(Jitter)를 수십 마이크로초 수준으로 보정할 수 있다.

4. 정리

CFS 스케줄러 자체는 시스템 전체의 데이터 처리량 극대화에 있어서는 수학적으로 최적화된 자료 구조를 따르나, Micro XRCE-DDS 클라이언트의 1kHz 퍼블리시(Publish) 루프 보장에는 심각한 위상적 취약점(무한대에 가까운 선점 지연 가능성)을 갖는다. PREEMPT_RT 패치는 운영체제 내 비선점 구간의 위상학적 부피(Volume)를 축소하고, 우선순위 상속을 통하여 대기 그래프 구조를 예측 가능한 결정론적 집합(Deterministic Set)으로 전환함으로써, ROS2 오프보드 제어 환경의 시스템 응답성을 공학적으로 보장하는 기반을 마련해 준다.
