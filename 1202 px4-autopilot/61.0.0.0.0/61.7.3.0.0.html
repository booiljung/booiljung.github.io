61.7.3. 서비스 품질(QoS) 다차원 프로파일의 동적 튜닝 및 파레토 최적화(Pareto Optimization)

현대의 자율 무인항공기(UAV) 체계에서 PX4-Autopilot(Firmware Version 1.14 이상)과 ROS2 컴패니언 컴퓨터, 그리고 지상 관제 시스템(GCS)을 아우르는 통신 브리지는 단일한 데이터 파이프라인이 아니다. 고대역폭을 요구하는 고해상도 비디오 스트림(Video Stream), 극저지연(Ultra-Low Latency)을 요구하는 오프보드 제어 명령(Offboard Control), 그리고 절대적 도달(Absolute Reliability)을 요구하는 미션 웨이포인트(Waypoint) 업로드가 좁은 무선 데이터 링크망 내에서 치열하게 경합(Contention)을 벌인다.

uXRCE-DDS 및 ROS2의 Data Distribution Service(DDS) 미들웨어는 이러한 이기종 트래픽 간의 우선순위와 통신 보장 수준을 세밀하게 조율하기 위해 다차원적인 서비스 품질(QoS, Quality of Service) 정책을 제공한다. 본 절에서는 통신 장애 환경에서 생존성을 극대화하기 위한 QoS 파라미터 프로파일링 기법과, 지연 시간(Latency)과 신뢰성(Reliability)이라는 두 상충 곡선 사이에서 파레토 최적화(Pareto Optimization) 지점을 동적으로 탐색하는 방법론을 고찰한다.

1. DDS QoS 다차원 파라미터 프로파일링

DDS 미들웨어의 QoS 정책은 데이터 송수신자의 계약(Contract)이다. 에이전트(Agent)와 클라이언트(Client) 간의 EDP(Endpoint Discovery Phase) 매칭 과정에서 양측의 QoS 설정이 수학적으로 호환 불가능(Incompatible)할 경우 연결 자체가 거부된다.

1.1 신뢰성(Reliability) 프로파일: Best-Effort 대 Reliable

데이터 유실을 대하는 프로토콜의 튜닝 포인트이다.

  • Best-Effort (최선 노력): 네트워크 혼잡 등으로 패킷이 유실되더라도 재전송(Retransmission)을 요청하지 않고 가차 없이 누락시킨다. 과거의 데이터를 수집단에서 기다릴 필요가 없는 sensor_combined (IMU), vehicle_odometry 등 최소 지연(Minimum Latency)이 생명인 고주기(High-rate) 센서 스트림에 강제 적용된다.
  • Reliable (신뢰성 보장): 수신단에서 ACK(Acknowledgement)를 받지 못하거나 NACK(Negative ACK)를 수신하면 잃어버린 패킷을 강제로 재전송한다. 자율 비행의 생사 여탈권을 쥔 vehicle_command (이륙, 모드 변경)나 궤적 데이터, 파라미터 교환 프로파일에 반드시 적용해야 하는 설정이다.

1.2 내구성(Durability) 프로파일: Volatile 대 Transient Local

데이터(토픽)가 발간된 직후의 생명 주기를 제정한다.

  • Volatile (휘발성): 구독자(Subscriber)가 늦게 네트워크에 접속(Late-joiner)했을 경우 과거의 데이터는 구해주지 않는다. 실시간 제어 데이터에 주로 쓰인다.
  • Transient Local (근거리 영속성): 퍼블리셔(Publisher)의 캐시(Cache/History) 메모리에 데이터를 보존하고 있다가 늦게 나타난 GCS 연결 시 과거 파라미터 상태를 일시적으로 쏟아내어 상태 머신(State Machine)을 즉각 동기화시킨다.

1.3 이력(History) 및 데드라인(Deadline) 제약

버퍼링 정책이다. KEEP_LAST 와 depth (예: 10) 조합은 최신 10개의 패킷만을 큐(Queue)에 들고 있으며 오래된 것부터 폐기(Drop)되도록 설계하여 통신 대기열 적체 현상(Queue Saturation)을 막는다. Deadline 정책은 센서 데이터가 일정 시간(T_{dead}) 이상 갱신되지 않으면 콜백(Callback)을 발생시켜 Failsafe 트리거를 강제하는 강력한 시간 방비책이다.

2. 지연 시간(Latency) vs 신뢰성(Reliability)의 파레토 최적화(Pareto Optimization)

한정된 대역폭 하에서 신뢰성 보장 프로파일(Reliable)을 전역적으로 남발할 경우, 재전송 폭풍(Retransmission Storm)이 발생하여 지연 시간이 수십 초로 팽창되는 참사를 맞이한다. 이 두 지표는 트레이드오프(Trade-off) 관계에 놓여 있으며, 완벽한 통신 시스템은 존재하지 않는다.

2.1 파레토 경계(Pareto Frontier)와 무선 링크 열화 튜닝

파레토 최적화란, (신뢰성을 희생하지 않고서는 지연 시간을 개선할 수 없는) 다목적 최적화의 한계 곡선을 찾는 동역학적 수학 과정이다.

  • 비행체가 이륙하여 가시권(VLOS) 내 수십 미터 거리에 있을 때(신호 강도 RSSI 최상): 대역폭이 풍부하므로, 경로 제어 주기를 50\text{Hz} 수준으로 올리고 파레토 셋(Pareto Set)은 [저지연, 적정 신뢰성] 지점에 위치한다.
  • 비행체가 수 킬로미터 벗어나 비가시권(BVLOS)에 접어들며 간섭이 폭증할 때(RSSI 열화): DDS의 동적 QoS 프로파일 조정(Dynamic QoS Tuning) 알고리즘이 개입하여 토픽 퍼블리시 속도를 5\text{Hz} 로 강제 스로틀링(Throttling)하고, 제어 명령은 철저히 [고신뢰성, 지연 수용] 파레토 극단으로 정책을 이동시켜야 한다.

3. PX4와 커스텀 QoS 정책 구현 아키텍처 결론

로보틱스 미들웨어 관점에서 QoS 튜닝은 단순한 옵션(Option) 조작이 아닌, 통신 단절(Data Link Loss) 및 EMI 공격에 맞서는 능동적 지휘 텍틱스(Tactics)다.

PX4 개발자는 px4_ros_com 패키지 내부 브리지 생성을 담당하는 클라이언트-에이전트 데몬을 구성할 때, 단순 디폴트(rmw_qos_profile_default)에 의존할 것이 아니라, 각 센서 프로파일의 통계적 허용도(Statistical Tolerance)를 정밀 역산한 수십 종류의 다차원 QoS 메타 프로파일(XML 형태 혹은 C++ 헤더 데피니션)을 직조해야 한다. 신호 열화가 감지되는 그 순간, 유연하게 파레토 최적화 곡선을 따라 QoS 세팅을 하향 또는 상향 이동시킬 수 있는 동적 튜닝 파이프라인의 완성도는 장거리 자율 비행의 신뢰도를 판가름하는 가장 날카로운 척도이다.
