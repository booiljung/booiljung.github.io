61.5.2.2. 송수신 병목 완화를 위한 직접 메모리 접근(DMA) 컨트롤러 병렬 전송 구성 및 링 버퍼(Ring Buffer) 오버플로우 한계점 도달 시 하드웨어 흐름 제어(RTS/CTS) 개입 로직

현대의 PX4-Autopilot 시스템(Firmware Version 1.14 이상)은 고밀도 센서 데이터(VIO, LiDAR 등)와 군집 비행(Swarm) 제어 명령을 실시간으로 처리하기 위해 직렬 통신 대역폭을 921,600 bps 이상의 초고속으로 끌어올렸다. 이러한 광대역 데이터 스트림을 MAVLink나 uXRCE-DDS 미들웨어가 단일 보드 컴퓨터(SBC)와 교환할 때, 기존의 바이트 단위 인터럽트(Interrupt-per-byte) 방식은 CPU의 콘텍스트 스위칭(Context Switching) 오버헤드를 극단적으로 가중시켜 비행 제어 루프의 실시간성을 훼손한다.

이러한 송수신 병목(Bottleneck) 사태를 근본적으로 완화하기 위해, 최신 Pixhawk 계열(Cortex-M7 기반)의 하드웨어 추상화 계층(HAL)은 직접 메모리 접근(DMA, Direct Memory Access) 컨트롤러를 통한 병렬 데이터 전송 아키텍처를 도입하였다. 아울러 극단적인 혼잡 상황에서 소프트웨어 링 버퍼(Ring Buffer)가 한계점에 도달했을 때 데이터 조각화(Fragmentation)를 막기 위한 하드웨어 흐름 제어(Hardware Flow Control)의 자동 개입 메커니즘이 적용된다. 본 절에서는 이 두 가지 핵심 하드웨어 융합 기술의 원리를 심도 있게 고찰한다.

1. 직접 메모리 접근(DMA) 컨트롤러 기반 병렬 전송 아키텍처

DMA는 CPU의 개입 없이 메모리와 주변장치(Peripheral, 예: UART 레지스터) 간에 데이터를 직접 교환하는 독립적인 하드웨어 서브시스템이다. PX4 HAL에서 스케줄링 간섭을 배제하고 대용량 px4_msgs 직렬화 페이로드를 고속으로 송수신하기 위해 DMA는 필수불가결하다.

1.1 송신(Tx) DMA 로직 추상화

애플리케이션 계층에서 write() 시스템 콜을 통해 대규모 버퍼를 직렬 드라이버로 전달하면, HAL 계층은 다음의 시퀀스로 DMA를 구동한다.

  1. 버퍼 매핑: 송신할 데이터가 적재된 소프트웨어 버퍼의 시작 메모리 주소(SRAM)와 전송할 바이트 수를 DMA 컨트롤러의 레지스터에 설정한다.
  2. 비동기 전송 개시: DMA 구동이 시작되면, CPU는 즉각 해당 제어 스레드로 복귀하여 EKF2(Extended Kalman Filter 2) 등 다른 비행 로직을 수행한다.
  3. 병렬 시프트(Shift): DMA 컨트롤러는 버스 매트릭스(Bus Matrix)의 여유 사이클을 도둑질(Cycle Stealing)하여, UART의 전송 데이터 레지스터(TDR)가 빌 때마다 1바이트씩 백그라운드에서 하드웨어적으로 복사한다.
  4. 전송 완료 인터럽트(TC, Transfer Complete): 지정된 바이트 수가 모두 전송되면 DMA가 CPU에 단 한 번의 인터럽트를 발생시켜 송신 스레드를 깨운다. (1000바이트 전송 시 1000번의 인터럽트가 1번으로 격감)

1.2 수신(Rx) DMA 및 링 버퍼(Ring Buffer) 결합 메커니즘

수신의 경우, 데이터가 언제 얼마나 들어올지 예측할 수 없으므로 무한 순환 구조인 링 버퍼(Ring Buffer/Circular Buffer)와 DMA를 정교하게 결합해야 한다.

  • DMA는 UART 수신 레지스터(RDR)에 데이터가 도착하는 즉시 링 버퍼의 현재 쓰기 포인터(Write Pointer) 위치로 바이트를 직접 기록한다.
  • DMA 하프 전송(Half-Transfer) 및 전송 완료(Transfer Complete) 인터럽트, 그리고 UART의 수신 선로 유휴 감지(IDLE Line Detection) 인터럽트를 조합하여 CPU에 데이터가 유효하게 누적되었음을 알린다. 이로써 CPU는 안전하게 링 버퍼의 읽기 포인터(Read Pointer)를 증가시키며 데이터를 애플리케이션 계층으로 퍼올린다.

2. 링 버퍼 오버플로우 한계점과 RTS/CTS 하드웨어 흐름 제어 개입

고속 DMA 전송이라 하더라도, 컴패니언 컴퓨터(ROS2 노드) 측에서 일시적인 스케줄링 지연(Load Spike) 등으로 데이터를 미처 빼가지 못하면, 한정된 용량의 수신 링 버퍼(수 킬로바이트 수준)는 순식간에 차오른다. 버퍼 풀(Buffer Full) 상태에서 새로운 데이터가 덮어씌워지는 오버플로우(Overflow)는 패킷 프레이밍 구조를 파괴하여 Segmentation Fault나 심각한 위치 추정 오차를 유발한다.

이를 하드웨어 차원에서 원천 차단하는 기술이 CRTSCTS 플래그로 활성화되는 즉각적인 버퍼 흐름 제어(Hardware Flow Control)이다.

2.1 RTS(Ready To Send) 및 CTS(Clear To Send)의 트리거(Trigger) 로직

이 메커니즘은 텔레메트리 핀에 추가로 할당된 2개의 제어 라인(RTS, CTS)을 이용한 교차 통신 비동기 제어 기법이다. (송신 측의 RTS가 수신 측의 CTS로 연결됨)

  1. 정상 수신 상태: 기체의 수신 링 버퍼가 여유(예: 50\% 미만 점유)가 있을 때, UART 하드웨어는 RTS 핀을 논리적 ‘Low(활성화)’ 상태로 유지하여 상대방에게 “계속 데이터를 보내라“는 전기적 신호를 보낸다.
  2. 한계점(High Watermark) 도달: 백그라운드 DMA 전송이나 ISR에 의해 링 버퍼의 쓰기 포인터가 읽기 포인터를 거의 추격하게 되어 임계치(예: 버퍼 용량의 80\% 이상)를 초과하게 되면, UART 컨트롤러 하드웨어가 스스로 이를 감지하고(또는 최소한의 ISR 개입으로) RTS 핀의 전압을 논리적 ’High(비활성)’로 스윙(Swing) 시킨다.
  3. 송신측 하드웨어 정지: 컴패니언 컴퓨터 측의 UART 컨트롤러는 자신의 CTS 핀 전압이 High로 변경된 것을 감지하는 즉시, 진행 중이던 DMA 송신을 바이트 경계에서 정확히 일시 정지(Pause)시킨다. 소프트웨어의 판단이 개입되기 전인 마이크로초 단위의 즉결 처분이다.
  4. 복구(Drain) 및 재개: CPU가 밀린 로직을 처리하여 링 버퍼 안의 데이터를 충분히 읽어내고 점유율이 여유 임계치(Low Watermark, 예: 20\%) 밑으로 떨어지면, 기체의 하드웨어는 다시 RTS 핀을 ’Low’로 리셋한다. 이에 따라 지연되었던 상대측의 송신 DMA가 즉각 멈춘 지점부터 재개된다.
