61.7.4.2. 시스템 초기화 매니저(Systemd)의 타이머 인터럽트(Timer Interrupt)를 활용한 에이전트 데몬 프로세스 교착 상태(Deadlock) 탐지 워치독(Watchdog) 타이머 설정 및 고립된 자원 강제 회수(SIGKILL) 기반 자동 재시작 로직

PX4-Autopilot(Firmware Version 1.14 이상)과 연동되는 ROS2 컴패니언 컴퓨터(예: Raspberry Pi 4, Jetson Nano) 환경에서, uXRCE-DDS 에이전트(Agent) 데몬은 UART 또는 UDP 포트를 점유하고 양방향 트래픽을 중계하는 미들웨어의 중추이다. 그러나 이 커널 레벨의 소켓 통신 프로세스는 불안정한 무선 텔레메트리 환경의 극한 노이즈나 직렬 포트(Serial Port) 버퍼 슬립(Slip) 현상에 노출될 때, 무한 대기(Infinite Wait)나 교착 상태(Deadlock)에 빠질 위험성을 영구적으로 내포하고 있다.

데몬이 교착 상태에 빠지면 CPU 점유율은 0\% 나 100\% 에 멈춰있지만, 데이터 라우팅은 전면 마비되어 결과적으로 FCU(Flight Control Unit)와의 오프보드 제어 연결이 끊기는 치명적인 통신 단절(Data Link Loss)을 야기한다. 본 절에서는 리눅스 커널의 시스템 초기화 매니저인 Systemd를 활용하여, 에이전트 프로세스의 심박수(Heartbeat)를 타이머 인터럽트(Timer Interrupt) 기반으로 하드웨어 수준에서 감시(Watchdog)하고, 응답이 끊긴 고립 프로세스를 SIGKILL 시그널로 처형한 뒤 고아 자원(Orphaned Resources)을 강제 회수하여 자동 재시작하는 무결점 복구 아키텍처를 해설한다.

1. 에이전트 데몬 교착 상태(Deadlock)의 시스템 병리학

Micro-XRCE-DDS-Agent가 먹통(Halt)이 되는 원인은 단순히 코드가 멈추는 것이 아니라, 커널 자원을 점유한 채로 인터럽트를 반환하지 않는 상태 증후군이다.

1.1 TTY 직렬 포트 홀딩(Holding)과 세마포어(Semaphore) 고갈

직렬 연결(UART) 기반의 에이전트 구동 시, 통신 케이블의 물리적인 단선-재접속(Glitch)이 발생하면 리눅스 커널의 ttyUSB0 장치 드라이버 파일 디스크립터(File Descriptor) 상태가 꼬이면서 read() 함수 블록에서 프로세스가 영구적으로 대기 상태(D-State: Uninterruptible Sleep)에 빠질 수 있다. 이 상태에서는 노드가 외부의 부드러운 종료 요청(SIGTERM)조차 수신하지 못하며, 재실행하려 해도 “Device or resource busy” 에러를 뿜으며 포트 접근을 거부한다.

2. Systemd 워치독(Watchdog) 타이머 인터럽트 설계

이러한 불치병에 걸린 프로세스를 밖에서 살려내기 위해서는 리눅스 시스템 배포판의 PID 1번인 systemd 의 강력한 타이머 기반 감시 메커니즘을 동원해야 한다.

2.1 하드웨어 의존적 WatchdogSec 프로비저닝

Systemd 서비스 유닛 파일(.service) 설정 내에 WatchdogSec 지시어(Directive)를 선언하여 타이머 인터럽트를 설정한다.

# /etc/systemd/system/uxrce_dds_agent.service
[Unit]
Description=Micro-XRCE-DDS Agent Daemon for PX4
After=network.target

[Service]
ExecStart=/usr/local/bin/MicroXRCEAgent serial --dev /dev/ttyUSB0 -b 921600
Restart=always
RestartSec=2
# -------- Watchdog Configuration --------
WatchdogSec=3s
StartLimitIntervalSec=60
StartLimitBurst=5
# ----------------------------------------

[Install]
WantedBy=multi-user.target
  • 메커니즘 원리: 위와 같이 설정되면, Systemd는 3\text{초}(WatchdogSec=3s)짜리 알람시계를 켠다. uXRCE-DDS 에이전트 데몬은 정상 구동되는 동안 리눅스 내장 함수인 sd_notify("WATCHDOG=1") 를 호출하여 자신이 살아있음(Heartbeat)을 Systemd 커널에 주기적으로 보고해야 한다. (명시적 지원 기능이 없을 경우 스니퍼(Sniffer) 래퍼 스크립트를 통해 PING을 대리 발송하도록 구현해야 한다.)

3. SIGKILL 기반의 고립 자원 강제 회수 및 영점 재시작(Cold Restart)

만약 에이전트에서 데드락이 발생하여 3\text{초} 동안 sd_notify 응답이 오지 않으면, Systemd의 타이머 인터럽트가 트리거된다.

3.1 은혜로운 종료(SIGTERM)의 포기와 자비 없는 처형(SIGKILL)

교착 상태에 빠진 프로세스에 “부디 자원을 정리하고 종료해 주십시오“라고 요청하는 SIGTERM (Signal 15)는 전혀 소용이 없다. Systemd 워치독은 즉각적으로 커널 단에서 가장 강력한 파괴 명령인 SIGKILL (Signal 9)을 방열(Fire)한다.

stateDiagram-v2
    [*] --> Running : Systemctl Start
    Running --> Heartbeat : ttyUSB0 I/O Active
    Heartbeat --> Running : sd_notify("WATCHDOG=1") 
    
    Heartbeat --> Deadlock : EMI / Serial Port Glitch
    Deadlock --> Watchdog_Timer_Tick : No Pulse for 3 sec
    
    Watchdog_Timer_Tick --> SIGKILL : Timer Expired!
    
    state SIGKILL {
        direction LR
        A[Kernel Terminates Process] --> B[Force Unbind /dev/ttyUSB0]
        B --> C[Reclaim Leaked RAM]
    }
    
    SIGKILL --> Restart : Restart=always Policy
    Restart --> Running : Clean Cold Re-spawn
  • 격리된 자원의 해방(Reclamation): SIGKILL은 해당 프로세스가 점유하고 있던 힙(Heap) 메모리와, 엉켜버린 커널 파일 디스크립터(/dev/ttyUSB0) 바인딩을 OS 커널 차원에서 무자비하게 박탈하고 회수한다.
  • 자동 재시작(Restart): 자원 강제 수거가 완료된 후 2\text{초}(RestartSec=2)의 쿨다운을 거쳐 에이전트 데몬을 처음부터 다시 깔끔하게 기동시킨다.

결과적으로 FCU(PX4) 입장에서는 컴패니언 컴퓨터가 약 5\text{초} 동안 응답이 없다가 다시 온전한 상태로 접속하는 것으로 체감되며, 시스템 전체의 마비를 회피한다. 물리 계층의 잡음과 소프트웨어의 비결정성이 매초 충돌하는 아수라장 속에서, 커널 타이머 인터럽트와 SIGKILL 폭격 기반의 자가 치유(Self-Healing) 아키텍처는 마지막 순간 제어 권한을 사수해 내는 컴패니언 컴퓨터의 가장 가혹하고도 든든한 면역 체계이다.
