61.5.3.1. 단일 보드 컴퓨터(SBC) 내 시뮬레이션 환경(SITL) 통신 지연 최소화를 위한 로컬 루프백(Loopback) 커널 라우팅 테이블 완전 우회 기법

PX4-Autopilot의 아키텍처는 하드웨어 추상화가 고도로 발달하여, 실제 비행 제어기 하드웨어 없이도 호스트 리눅스(Linux) 머신이나 단일 보드 컴퓨터(SBC, Single Board Computer)의 운영체제 위에서 펌웨어 로직을 그대로 구동할 수 있는 소프트웨어 인 더 루프(SITL, Software-In-The-Loop) 환경을 완벽히 지원한다(Firmware Version 1.14 이상 기준).

이러한 SITL 환경에서 PX4 데몬(Daemon), ROS2 기반의 경로 계획(Path Planning) 노드, 그리고 Gazebo와 같은 물리 시뮬레이터(Physics Simulator)는 모두 동일한 컴퓨터 내에서 독립적인 프로세스(Process) 샌드박스로 실행되며, 이들 간의 데이터 교환(예: VIO 오도메트리 수신 및 Actuator 제어 명령 송신)은 가상 네트워크 환경인 루프백 인터페이스(Loopback Interface, lo)를 통해 이루어진다.

그러나 단순 로컬 환경이라 할지라도, 운영체제의 표준 네트워크 스택(Network Stack)을 곧이곧대로 통과하는 것은 커널(Kernel) 단의 스케줄링 오버헤드와 메모리 복사 지연을 수반한다. 본 절에서는 SITL 시뮬레이션의 타이밍 정밀성을 실제 하드웨어 수준(Real-time)으로 극한까지 모사하기 위해, 표준 커널 라우팅 테이블(Routing Table)을 우회(Bypass)하여 통신 지연을 최소화하는 리눅스 네트워킹 심화 기법을 다룬다.

1. SITL 환경의 로컬 루프백 프로세스 간 통신(IPC) 병목 원리

SITL에서 PX4 프로세스와 ROS2 미들웨어(Fast DDS 경유) 간의 UDP 패킷 교환은 통상적으로 목적지 IP를 127.0.0.1 (로컬호스트)로 설정하여 진행된다. 표면적으로는 네트워크 케이블을 타지 않아 빠를 것처럼 보이나, 호스트 리눅스 커널의 관점에서는 다음과 같은 심층적인 패킷 횡단(Traversal) 페널티를 여전히 지불해야만 한다.

  1. 사용자 공간 \rightarrow 커널 공간 컨텍스트 스위칭: sendto() 시스템 콜 호출 시 메시지가 유저 영역 메모리 버퍼에서 커널 버퍼(sk_buff)로 복사(Copy)된다.
  2. TCP/IP 프로토콜 스택 처리: 물리적인 랜카드(NIC)로 나가지 않을 뿐, IP 헤더 검사, UDP 포트 매핑, 넷필터(Netfilter/iptables) 방화벽 룰 체크 등의 3계층(L3) / 4계층(L4) 연산이 그대로 수행된다.
  3. 가상 라우팅 테이블 탐색(Virtual Routing Forwarding): 패킷의 목적지가 127.0.0.1임을 확인하고 커널 라우팅 테이블을 조회하여 데이터를 루프백 가상 장치 드라이버(lo)의 수신 큐(Rx Queue)로 회귀(Loop-back)시킨다.
  4. 커널 공간 \rightarrow 사용자 공간 복사 반환: 수신 측 ROS2 노드의 recvfrom() 호출에 응답하여, 커널 버퍼의 데이터가 다시 수신 측 유저 버퍼로 복사되며 프로세스를 깨운다.

이러한 “유저-커널-유저” 왕복의 스루풋 파이프라인에서 수백 헤르츠(Hz) 빈도로 발산되는 ROS2 제어 토픽(e.g., px4_msgs::msg::OffboardControlMode)은 리눅스 스케줄러의 타임 슬라이스(Time Slice) 경쟁에 의해 비선형적인 지터(Jitter)를 겪게 된다.

2. 커널 라우팅 테이블 완전 우회 기법 (eBPF / AF_XDP)

최고 수준의 제어 지연 안정성(Deterministic Latency)을 추구하는 최첨단 자율 에이전트 GCS/SBC 연구 환경에서는, 위와 같은 낡은 네트워크 스택 패러다임을 타파하기 위해 최신 리눅스 커널(Version 5.x 이상)의 eBPF(Extended Berkeley Packet Filter) 기반 패킷 바이패스(Bypass) 기술을 도입한다.

2.1 로컬 통신 패스트 경로(Fast-Path) 개척 메커니즘

  1. 소켓 어피니티(Socket Affinity) 및 공유 메모리(Shared Memory)
    DDS 설정 파일(XML Profiles)에서 로컬호스트 간의 탐색(Discovery) 및 전송 트랜스포트(Transport)를 UDP 오버 루프백(UDP over Loopback) 방식에서 1순위로 공유 메모리 Транс포트(SHM, Shared Memory Transport)를 강제 사용하도록 제약한다. 이 경우 TCP/IP 데몬을 아예 타지 않고 RAM 수준의 원자(Atomic) 버퍼 복사만으로 끝난다.
  2. AF_XDP (eXpress Data Path) 소켓 적용
    만일 DDS 파이프라인이나 MAVLink 포워딩 레거시 코드 때문에 반드시 소켓(Socket) API를 거쳐야만 한다면, 커널 넷필터 계층 하단에서 동작하는 AF_XDP 소켓을 활용한다. 이는 패킷이 sk_buff 객체로 할당되기도 전인 RX 큐 최하단 단계에서 목적지 포트를 스누핑(Snooping)하여, 데이터를 커널 라우팅 스택에 넘기지 않고 수신 측 프로세스의 유저 맵(Memory Map)으로 다이렉트 복사(Zero-Copy)하는 극한의 치트(Cheat) 경로이다.

3. PX4-SITL 환경 프로비저닝 튜닝 (튜토리얼 및 적용)

SBC나 고성능 서버 환경에서 PX4의 SITL을 최적의 시간 도메인 지터 방어 성능으로 끌어올리기 위해서는 컴패니언 컴퓨터 리눅스 튜닝이 병행되어야 한다.

  1. 루프백 장치 인터럽트 병합 비활성화:
    ethtool -K lo rx-checksumming off tx-checksumming off
    가상 네트워크 장치(lo)에서 불필요한 체크섬(Checksum) 가속 오프로딩을 소프트웨어적으로 비활성화하여 마이크로초 단위의 직렬화 패널티를 깎아낸다.
  2. RT 프리엠팁(Preempt-RT) 커널과의 결합:
    ROS2와 PX4 SITL이 구동되는 우분투(Ubuntu) 커널에 RT 패치(Real-Time Patch)를 가하여, 네트워크 소켓 소프트웨어 인터럽트(SoftIRQ) 처리가 사용자 레벨 제어 프로세스(Flight Control Thread)의 스케줄링을 침범하지 못하도록 강제 우선순위 매핑을 적용해야 한다.

결론적으로, 아무리 가상의 127.0.0.1 샌드박스 내부라 하더라도 데이터가 리눅스 표준 라우팅 테이블을 거치는 순간 타이밍 비결정적(Non-deterministic) 오버헤드가 발생한다. eBPF 소켓 프로그래밍 또는 엄격한 공유 메모리(SHM) 트랜스포트 구성을 통해 네트워크 커널 스택을 완전히 우회(Bypass)하는 것만이, 물리적 공력(Aerodynamics)과 제어 주파수의 미세한 상관관계를 실험실에서 완벽하게 모사할 수 있는 프로토타이핑(Prototyping)의 첩경이다.
