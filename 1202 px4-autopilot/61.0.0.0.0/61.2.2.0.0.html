61.2.2. ROS2 코어 시스템 아키텍처 및 무결성 기반 의존성 패키지 설치

1. 개요

오프보드(Offboard) 제어를 통하여 PX4-Autopilot과 이종 시스템 간의 자율 비행 알고리즘을 연동하기 위해서는, 로봇 운영체제(Robot Operating System 2, ROS2) 미들웨어가 제공하는 분산 컴퓨팅 계층이 필수적이다. PX4는 자체적으로 Micro XRCE-DDS 에이전트를 통해 통신하므로, 동반 컴퓨터(Companion Computer) 측에서는 ROS2 시스템 아키텍처를 정확히 이해하고 무결성이 검증된 의존성 패키지를 설치하여 데이터 신뢰성을 확보해야 한다. 본 절에서는 ROS2 코어 시스템의 계층적 아키텍처를 분석하고, 우분투(Ubuntu, 예: 22.04 LTS 커널 기반) 환경에서 GPG 키 기반의 무결성 검증을 거쳐 필수 의존성 패키지를 안전하게 프로비저닝(Provisioning)하는 메커니즘을 상세히 다룬다.

2. ROS2 코어 시스템 아키텍처 구조

ROS2는 ROS1의 단일 마스터(Single-Master) 통신 구조가 갖는 단일 장애점(Single Point of Failure) 한계를 극복하기 위해, DDS(Data Distribution Service) 기반의 탈중앙화된 코어 아키텍처를 채택하였다. 이 아키텍처는 아래와 같이 다층적인 추상화 계층(Abstraction Layer)으로 구성된다.

  1. 사용자 애플리케이션 계층(User Application): VIO(Visual Inertial Odometry), 경로 계획(Path Planning) 등의 사용자 알고리즘이 동작하는 계층이다.
  2. 클라이언트 라이브러리 계층(rclcpp, rclpy): 사용자가 C++나 Python을 사용하여 노드(Node)를 작성할 수 있도록 제공되는 ভাষা 종속적(Language-Specific) API 계층이다.
  3. ROS 클라이언트 라이브러리 코어(rcl, ROS Client Library): 언어에 독립적인 ROS2의 핵심 로직(노드 생명주기 관리, 파라미터 제어 등)을 C 언어로 구현한 공통 계층이다.
  4. 미들웨어 추상화 계층(rmw, ROS Middleware Interface): 다양한 제조사의 DDS 구현체(Fast DDS, Cyclone DDS 등)를 하나의 표준화된 인터페이스로 추상화하여, rcl 계층이 하위 네트워크 통신 기술의 세부 구현에 종속되지 않도록 격리(Decoupling)한다.

PX4-Autopilot의 내장된 Micro XRCE-DDS 클라이언트는 DDS의 경량화 버전 프로토콜을 사용하므로, 브릿지 역할을 하는 에이전트(Agent)가 수신한 데이터를 RTPS(Real-Time Publish-Subscribe) 프로토콜을 통해 위 그림의 하위 DDS 계층과 상호작용하게 된다.

3. 무결성 기반 의존성 패키지 설치

항공 우주 및 드론 제어 시스템의 특성상, 미들웨어를 구성하는 패키지가 위변조되거나 호환되지 않는 버전이 섞여 설치되는 현상(Dependency Hell)을 철저히 차단해야 한다. 이를 보장하기 위해 APT 저장소의 서명 검증(Cryptographic Verification)과 rosdep 도구를 통한 무결성 기반 설치 절차가 강제된다.

3.1 GPG 키 서명 확인 및 검증된 저장소 등록

운영체제에 패키지를 제공하는 서버가 공식 ROS2 릴리스 서버임을 암호학적으로 증명하기 위해, 리파지토리(Repository)의 공개 키(GPG Key)를 시스템의 키체인에 등록해야 한다. 공격자에 의한 Man-in-the-Middle(MITM) 패키지 변조를 막는 1차 방어선이다.

# 필수 시스템 유틸리티 리포지토리 설정
sudo apt update && sudo apt install -y curl gnupg lsb-release

# ROS2 공식 GPG 암호화 키 다운로드 및 키 링 적용
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

# 아키텍처 무결성에 맞춘 공식 미러 저장소 리스트 추가
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(source /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

3.2 베이스 패키지 설치 및 초기화

ROS2의 릴리스 버전(예: ros-humble-desktop)을 설치한 뒤, PX4 통신 브릿지인 px4_ros_com과 메시지 타입인 px4_msgs를 빌드하는 과정에서 시스템 내 의존성이 충돌하지 않도록 rosdep을 통해 루트(Root) 무결성을 검증한다.

sudo apt update
sudo apt install -y ros-humble-desktop python3-colcon-common-extensions python3-rosdep

# rosdep 초기화 (의존성 패키지 인덱스의 무결성 업데이트)
sudo rosdep init
rosdep update

3.3 미들웨어 구현체(DDS) 선택적 프로비저닝

PX4 커뮤니티에서는 Fast DDS 외에도 고가용성 멀티캐스트 네트워크 환경에 최적화된 Eclipse Cyclone DDS의 사용을 지원한다. GCS와 다중 에이전트(Swarm) 간 네트워킹 구조에 따라 추가적인 DDS 구현체를 설치 및 등록할 수 있다.

sudo apt install -y ros-humble-rmw-cyclonedds-cpp
# 런타임에 DDS 구현체를 명시적으로 지정하여 스택 일관성 유지
export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp

4. 정리

ROS2의 다층적 코어 아키텍처는 노드 개발자가 하위 네트워크 프로토콜의 파편화에 신경 쓰지 않고 PX4-Autopilot 제어 로직 구현에 집중할 수 있는 추상화를 제공한다. 또한, 암호학적 서명이 적용된 패키지 관리 체계를 통해 설치된 미들웨어 스택의 무결성을 엄격하게 유지함으로써, 비행 중 발생할 수 있는 코어 덤프(Core Dump)나 런타임 예외 장애를 예방하는 근원적인 토대를 제공한다. 이어지는 장에서는 이렇게 구축된 아키텍처 위에서 메타 빌드 시스템(colcon)을 활용하여 PX4-ROS2 브릿지를 샌드박스(Sandbox) 내에서 안전하게 병렬 컴파일하는 방법에 대해 논의한다.
