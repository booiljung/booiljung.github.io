61.2.2.2. 공개키 기반 구조(PKI)의 비대칭 암호화 서명 검증을 통한 공급망 공격(Supply Chain Attack) 원천 차단 및 패키지 무결성 확보 절차

1. 개요

현대의 무인항공기(UAV) 및 자율 에이전트 개발은 PX4-Autopilot 및 ROS2 생태계와 같은 방대한 오픈소스 소프트웨어(OSS) 의존성에 기반한다. 이로 인해 최근 사이버 보안 영역에서 급증하고 있는 ’공급망 공격(Supply Chain Attack)’의 위협에 고스란히 노출되어 있다. 악의적인 공격자가 데비안(Debian) 저장소 미러 서버를 탈취하거나 통신 패킷을 변조(Man-in-the-Middle, MITM)하여 조작된 ROS2 패키지나 PX4 통신 브릿지 라이브러리를 동반 컴퓨터(Companion Computer)에 주입할 경우, 드론의 제어권 탈취나 치명적인 비행 사고로 이어질 수 있다. 본 절에서는 공개키 기반 구조(Public Key Infrastructure, PKI)와 비대칭 암호화(Asymmetric Cryptography) 수학 모델을 활용하여, 데비안 패키지 관리 도구(APT)가 어떻게 이러한 공급망 공격을 원천적으로 차단하고 컴파일 시점의 패키지 무결성을 확증하는지 분석한다.

2. 비대칭 암호화 구조와 패키지 서명 메커니즘

공급망 공격 방어의 핵심은 다운로드된 패키지의 출처(Authenticity)와 위변조 여부(Integrity)를 수학적으로 증명하는 데 있다. 리눅스의 APT 패키징 시스템은 GnuPG(GNU Privacy Guard) 체계를 기반으로 다음과 같은 과정의 비대칭 암호화 서명을 수행한다.

  1. 저장소 측 서명(Repository Signing): ROS2 공식 유지보수 주체(예: OSRF)는 보안이 유지되는 빌드 팜(Build Farm)에서 모든 .deb 패키지들의 SHA-256 체크섬(Checksum) 해시 테이블을 생성하여 Release 파일에 기록한다. 이후 유지보수 주체의 개인키(Private Key) 를 사용하여 이 Release 파일을 암호화(서명, Signature)한다.
  2. 클라이언트 측 검증(Client Verification): 사용자의 우분투 기기(동반 컴퓨터)는 구성 단계에서 미리 획득한 ROS2 유지보수 주체의 공개키(Public Key) 를 이용하여 다운로드된 Release.gpg (혹은 InRelease) 서명의 유효성을 복호화하여 검증한다. 서명 검증이 성공하면, 그 안에 기록된 개별 .deb 파일들의 SHA-256 해시값과 실제 다운로드된 파일의 해시값을 대조하여 파일 수준의 조작 여부를 최종 판별한다.

3. 격리된 키 링(Keyring) 구성을 통한 무결성 확보 실무

전통적으로 리눅스 사용자는 apt-key add 명령을 사용하여 저장소 키를 전역(Global) trusted.gpg 키 링에 추가하였다. 그러나 이 경우, A 저장소의 키가 B 저장소의 패키지 서명까지 신뢰(Trust)하게 만드는 보안 누수(Cross-signing vulnerability)가 발생할 수 있다.
이러한 취약점을 해결하기 위해 최근의 최적화된 패키지 무결성 확보 절차는 signed-by 플래그를 사용하여 특정 저장소와 특정 GPG 공개키를 일대일(1:1)로 매핑하는 구조를 취한다.

3.1 ROS2 및 PX4 의존성 구축 시 안전한 키 관리 스니펫 예시

# 1. ROS2 공식 GPG 공개키를 별도의 격리된(isolated) 폴더에 ASCII Armor 해제 후 저장
curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key | \
  gpg --dearmor | sudo tee /usr/share/keyrings/ros-archive-keyring.gpg > /dev/null

# 2. 전역 시스템 트러스트(Global Trust)를 피하기 위해 signed-by 지시자 명시
#    -> 오직 ROS2 아카이브 서버에서 다운로드된 패키지만 해당 키로 증명되도록 한정함
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 3. APT 데이터베이스 업데이트 및 무결성 기반의 패키지 설치
sudo apt update
sudo apt install -y ros-humble-desktop

4. 정리

통신 네트워크를 통한 패키지 수급 체계에서 암호학적 서명이 없다면, 해커가 미러 사이트의 DNS를 스푸핑(Spoofing)하여 악성 프로세스가 삽입된 rclcpp나 Fast-DDS 라이브러리를 동반 컴퓨터에 설치하게 끔 우회할 수 있다. PKI 기반의 비대칭 암호화 아키텍처는 이를 차단함으로써, PX4-Autopilot과 상위 관제 알고리즘 간에 이루어지는 MAVLink 통신 및 uORB 메시지 인터페이스가 순수한(Untampered) 원본 소스코드의 의도대로만 동작하도록 보장하는 소프트웨어적 방탄조끼(Bulletproof vest) 역할을 수행한다.
