61.6.3.1. 이동 평균(Moving Average) 필터를 적용한 단위 시간당 데이터 프레임 누적 합산 기반의 유효 처리량(Effective Throughput)과 물리적 한계 채널 용량 간의 점유율 역산 모델

PX4-Autopilot(Firmware Version 1.14 이상)과 GCS(또는 ROS2 컴패니언 컴퓨터)를 연결하는 원격 측정(Telemetry) 링크는 이더넷(Ethernet), Wi-Fi, 혹은 57.6\text{kbps} 수준의 저대역폭 직렬 무선 모뎀(Serial Radio Modem) 등 다양한 물리 계층(Physical Layer)을 활용한다. 제한된 무선 자원 환경에서 uXRCE-DDS 또는 MAVLink 메시지 라우팅이 원활히 이루어지는지를 검증하기 위해서는, 현재 송수신되고 있는 데이터량이 전체 채널 용량(Channel Capacity)의 어느 정도를 점유하고 있는지 정량적으로 파악하는 네트워크 프로파일링(Profiling)이 필수적이다.

본 절에서는 찰나의 버스트 트래픽(Burst Traffic)으로 인한 왜곡을 방지하기 위해 이동 평균(Moving Average) 필터를 도입하고, 단위 시간당 도달한 데이터 프레임 바이트를 누적 합산하여 실질적인 “유효 처리량(Effective Throughput)“을 도출해낸 뒤, 이론적 물리 한계 대비 네트워크 점유율(Network Occupancy Rate)을 역산하는 수학적 모델링 과정을 분석한다.

1. 물리적 채널 용량(Channel Capacity) 한계 모델링

네트워크 점유율을 역산하기 위해서는 근본이 되는 분모, 즉 물리 매체가 단위 시간당 전송할 수 있는 최대 바이트(Capacity)를 규정해야 한다.

1.1 직렬(UART) 무선 텔레메트리 환경의 물리 한계 레이트

라디오 모뎀(예: SiK Radio)의 통신 보드 레이트(Baud Rate)가 B = 57600\text{bps} (bits per second)로 설정되어 있다고 가정하자. 범용 비동기 송수신기(UART) 규격은 보통 1바이트(8비트) 페이로드를 전송하기 위해 시작 비트(Start bit) 1개와 정지 비트(Stop bit) 1개를 덧붙여 총 10비트를 소모한다(8-N-1 설정).

따라서 이 채널의 초당 최대 전송 가능한 원시 바이트 수(Raw Channel Capacity, C_{physical})는 다음과 같이 산정된다.

C_{physical} = \frac{57600 \text{ bits/s}}{10 \text{ bits/byte}} = 5760 \text{ Bytes/s}

미들웨어 계층에서는 이 5760\text{ B/s}가 넘어서는 트래픽을 송출하려 할 경우, 남은 패킷들은 운영체제의 송신 버퍼(Tx Buffer)에 체류하다 큐 포화(Queue Overflow)로 강제 폐기(Drop) 운명을 맞이하게 된다.

2. 단위 시간 프레임 누적 합산 및 유효 처리량(Effective Throughput) 산출

이제 분자에 해당하는 실제 데이터 유량(Data Rate)을 구해야 한다. ROS2 서브스크라이버 노드(Subscriber Node) 또는 QGroundControl의 수신 콜백(Callback) 단에서 도달한 메시지의 페이로드 크기(Payload Size)와 프로토콜 헤더 오버헤드(Header Overhead)를 모두 합산한다.

2.1 이동 평균(Moving Average) 필터를 통한 버스트 평활화(Gated Smoothing)

멀티콥터 비행 중에는 특정 이벤트(경로 웨이포인트 업로드, 파라미터 동기화 등) 발생 시 수백 밀리초 단위로 막대한 버스트(Burst) 트래픽이 쏟아져 나온다. 이를 순수하게 1초(1Hz) 단위로 끊어서 측정하면, 어떤 초에는 0 근처였다가 다음 초에는 한없이 치솟는 극심한 노이즈 도표가 그려져 시스템의 전반적인 건강도(Health)를 판단하기 어렵다.

이를 방해하기 위해 윈도우 크기(Window Size)가 N이고 샘플링 주기 \Delta t를 갖는 단순 이동 평균(SMA, Simple Moving Average) 또는 지수 가중 이동 평균(EWMA) 모델을 적용하여 초당 바이트 처리량 T_{eff}(k)를 도출한다.

T_{eff}(k) = \frac{1}{N} \sum_{i=0}^{N-1} \left( \frac{ \sum_{j \in Frame(k-i)} B_j }{ \Delta t } \right)

여기서,

  • B_j: j번째 수신된 데이터 프레임의 전체 크기 (페이로드 + MAVLink/DDS 헤더 길이).
  • \sum B_j: 짧은 관측 시간 \Delta t (예: 0.1\text{s}) 동안 쏟아져 들어온 바이트의 총합.
  • N: 평활화 윈도우 스텝 수 (예: 10스텝).

이 이동 평균 파이프라인을 거치면 순간적인 버스트 픽(Peak)이 부드럽게 감쇠하여, 현재 시스템이 요구하는 기저 전송률(Baseline Data Rate)의 추계 현황을 매끄럽게 모니터링할 수 있다.

2. 네트워크 점유율(Occupancy Rate) 역산 및 대역폭 포화 한계 감시

최종적으로 시스템 엔지니어는 산출된 유효 처리량 T_{eff} 와 앞서 구한 물리 한계 용량 C_{physical} 의 비(Ratio)를 역산하여 현재 네트워크의 혼잡도를 0 ~ 100\% 범위의 백분율로 산정해낸다.

O_{rate}(\%) = \left( \frac{T_{eff}}{C_{physical}} \right) \times 100

3.1 점유율 지표를 통한 자율 비행 아키텍처 건전성 평가

  • O_{rate} < 50\% (건전 상태): 대역폭 마진(Margin)이 임계치 이상 확보된 상태로, GCS에서 대용량의 비행 미션 스크립트를 밀어 넣거나 파라미터를 요청하더라도 센서 토픽의 지연(Latency)이 크게 저해되지 않는다.
  • 60\% \le O_{rate} < 85\% (혼잡 경고 상태): 배경 트래픽(오도메트리 수신)만으로도 채널 한계의 절반을 넘어간 상태다. 패킷 손실(Drop) 빈도가 상승하며, DDS QoS 프로파일 조정(Rate Limit 10\text{Hz} \rightarrow 5\text{Hz})을 통한 셰이핑(Traffic Shaping) 조치가 요구된다.
  • O_{rate} \ge 90\% (포화 및 제어 붕괴): 대기열이 가득 차 전파 지연 비대칭성(Jitter)이 폭발적으로 지수 상승한다. 타임아웃 오류 및 Failsafe 발동의 징후로서, 즉각적인 전송률 저감 스크립트 실행이 불가피하다.

이처럼 하위 물리 계층의 이론적 대역폭을 역산하고 소프트웨어 계층의 누적 수신량을 필터링하여 대조하는 행위는, 비선형적으로 요동치는 무선 스웜(Swarm) 통신 환경 속에서 제어 루프를 사수하기 위한 ‘맥박(Heartbeat) 진단’ 메커니즘이라 할 수 있다.
