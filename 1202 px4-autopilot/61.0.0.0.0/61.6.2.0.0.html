61.6.2. 복합 센서 융합 데이터의 시계열 주파수 응답 및 나이퀴스트 적합성 검증

PX4-Autopilot(Firmware Version 1.14 이상 탑재)의 확장 칼만 필터(EKF2, Extended Kalman Filter)는 관성 측정 장치(IMU: 가속도계, 자이로스코프), 기압계(Barometer), 자기나침반(Magnetometer), 그리고 외부 오도메트리(VIO, Visual Inertial Odometry) 등 다양한 위상(Phase)과 대역폭(Bandwidth)을 가진 센서들의 데이터를 융합(Fusion)하여 최적의 상태 추정치를 도출한다.

이러한 고도화된 멀티 센서 퓨전 시스템에서 가장 치명적인 오류는 소프트웨어 버그가 아닌, 이산 시간(Discrete-Time) 샘플링 이론의 근간을 위배할 때 발생하는 ’에일리어싱(Aliasing, 계단 현상 및 저주파 왜곡)’이다. 기체의 강력한 회전 전동기(Motor)와 프로펠러가 만들어내는 고주파 진동(High-frequency Vibration)이 센서의 측정 대역폭을 넘어서는 순간, 이 진동 노이즈는 기만적인 저주파 신호로 둔갑하여 제어 루프를 피드백 발산(Feedback Divergence)으로 몰고 간다.

본 절에서는 복합 센서 스트림의 시계열 데이터를 주파수 영역(Frequency Domain)으로 변환하여 시스템의 기계적 진동 응답 특성을 프로파일링하고, 디지털 신호 처리의 대원칙인 나이퀴스트 적합성(Nyquist Fitness)을 수학적으로 검증하는 방법론을 고찰한다.

1. 나이퀴스트-섀넌 표본화 정리(Nyquist-Shannon Sampling Theorem)와 PX4 적용

아날로그 세계의 기체 동역학적 물리량을 디지털 펌웨어(NuttX RTOS)로 온전히 가져오기 위한 유일한 수학적 보증 수표는 나이퀴스트-섀넌 표본화 정리이다.
본 정리에 따르면, 기저 대역 신호(Baseband Signal)가 포함하고 있는 최고 유효 주파수 성분이 f_{max}일 때, 이를 정보의 손실 없이 완벽하게 복원하기 위한 최소 샘플링 주파수 f_s는 다음과 같아야 한다.

f_s > 2 \times f_{max}

여기서 f_{nyq} = \frac{f_s}{2} 를 나이퀴스트 주파수(Nyquist Frequency)라고 부른다.

1.1 IMU 샘플링과 진동(Vibration) 스펙트럼의 충돌

PX4의 최고속 제어 루프인 rate controller는 주로 400\text{Hz} ~ 800\text{Hz}의 속도로 동작하며, 이를 뒷받침하기 위해 IMU 센서(IC)는 내부적으로 1\text{kHz} ~ 4\text{kHz} 이상의 극초고속(Oversampling)으로 아날로그 데이터를 컨버팅(ADC)한다.

  • 만약 쿼드콥터의 프로펠러 회전 속도가 10,000\text{RPM}이라면, 기본 기계적 진동 주파수는 대략 166\text{Hz}가 되며 그 고조파(Harmonics)는 수백 \text{Hz}에 달한다.
  • 만일 IMU 로직이 전력 절감을 위해 250\text{Hz}로 샘플링(f_s)을 수행한다면, 나이퀴스트 한계 주파수는 125\text{Hz}가 된다.
  • 이때 166\text{Hz}의 모터 진동 신호는 나이퀴스트 한계를 초과하여 스펙트럼이 접히는 현상(Folding)을 유발하며, 제어기 입장에서는 이를 84\text{Hz} (250 - 166)의 완전히 엉뚱한(허위의) 기체 요동으로 착각하여 추력(Thrust)을 급격히 변조시키고 결국 기체를 추락시킨다.

2. PX4 시계열 스펙트럼 밀도(PSD) 프로파일링 분석 프로세스

에일리어싱과 센서 주파수 응답 한계를 검증하기 위해, 비행 데이터 로깅(ULog 파일)을 활용한 고속 퓨리에 변환(FFT, Fast Fourier Transform) 전력 스펙트럼 밀도(PSD, Power Spectral Density) 분석이 필수적이다.

2.1 비행 로그 수집 및 FFT 해독

  1. 에일리어싱 방지용 Raw 로깅 활성화: SDLOG_PROFILE 파라미터에서 ‘High rate (Raw IMU)’ 프로필을 활성화하여 기체의 하드웨어 원시 샘플링 레이트와 동일한 속도(예: 1\text{kHz})로 가속도(sensor_accel) 및 자이로(sensor_gyro) 데이터를 마이크로 SD 카드에 기록한다.
  2. 이산 퓨리에 변환(DFT/FFT): Flight Review 툴이나 Python의 scipy.signal.welch 함수를 이용하여 시계열(Time-Domain) 배열 데이터를 주파수-에너지 영역으로 변환한다.
  3. 피크(Peak) 주파수 식별: 생성된 스펙트럼 도표 상에서 노이즈 에너지가 집중된 고주파 대역(통상 모터 회전수와 프로펠러 날개 수의 곱에 해당하는 대역, Blade Passage Frequency)을 식별한다.

3. 에일리어싱 회피를 위한 아키텍처 수준의 필터링 기법

스펙트럼 분석 결과 치명적인 고주파 노이즈가 확인되었고, 하드웨어 댐퍼(Damper) 설치 등의 물리적 튜닝이 한계에 달했다면, 소프트웨어 아키텍처 수준에서 신호를 다듬어야 한다. PX4는 IMU 파이프라인에 정교한 캐스케이드(Cascade) 필터를 내장하고 있다.

3.1 하드웨어 저역 통과 필터 (Hardware LPF)

센서 칩셋 자체(예: Bosch BMI088, InvenSense ICM-20689) 레지스터에서 지원하는 아날로그 안티-에일리어싱 필터(Anti-aliasing Filter)이다. PX4는 부팅 시 이 레지스터를 조작하여 ADC 전에 f_{nyq} 이상의 아날로그 잡음을 1차적으로 소거한다.

3.2 노치 필터 (Notch Filter) 연동

FFT 분석을 통해 모터 진동의 중심 주파수(Center Frequency)가 166\text{Hz}로 확고히 식별되었다면, IMU_GYRO_NF_FREQ 파라미터를 해당 값에 맞추어 활성화한다. 노치 필터는 전체 제어 대역폭을 깎아먹는 저역 통과 필터(LPF)와 달리, 오직 해당 특정 “바늘 같은(Notch)” 주파수 대역의 에너지만을 -40\text{dB} 급으로 강력하게 도려내는(Killing) 고정밀 외과 수술적 필터링을 수행한다. 최신 PX4 펌웨어는 ESC 텔레메트리 값을 피드백 받아 모터 RPM에 따라 노치 주파수를 실시간으로 이동시키는 ‘동적 노치 필터(Dynamic Notch Filter)’ 알고리즘도 지원한다.

결론적으로, 다중 센서 융합 시스템에서 나이퀴스트 적합성 검증은 단순한 주파수 수학을 넘어선다. 이는 로봇 기구가 생산하는 고주파 파괴 에너지가 제어 루프의 저주파 평온함을 침범하지 못하도록 차단하는, 물리계와 디지털계 사이의 댐(Dam)을 견고히 쌓는 가장 중대한 방어 공학적 절차이다.
