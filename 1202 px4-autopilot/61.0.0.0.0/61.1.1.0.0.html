61.1.1. 분산 제어 시스템 내 데이터 분배 서비스(DDS)의 논리적 토폴로지

무인항공기 시스템(UAS)의 제어 규모가 단일 기체 단위에서 군집(Swarm) 및 복합 다중 에이전트(Multi-Agent) 시스템으로 확장됨에 따라, 분산 제어 시스템의 통신망 설계는 매우 치명적인 학술적 문제로 대두되었다. PX4-Autopilot 운영체제는 ROS2 플랫폼과의 완전한 이종 결합을 이루기 위하여 데이터 분배 서비스(Data Distribution Service, DDS) 기반의 논리적 토폴로지(Logical Topology)를 기저 네트워크 모델로 확립하였다. 본 절에서는 이러한 DDS가 오프보드 제어 체계 내에서 형성하는 위상학적 구조에 대해 분석한다.

1. DDS의 퍼블리시-서브스크라이브 데이터 스페이스 토폴로지 (Architecture Understanding)

전통적인 네트워크 모델에서 흔히 쓰이는 클라이언트-서버(Client-Server) 아키텍처는 제어 데이터의 중앙 집중 병목(Centralized Bottleneck) 현상을 야기한다. 이를 극복하기 위해 DDS는 데이터 중심적(Data-Centric) 퍼블리시-서브스크라이브(Publish-Subscribe) 토폴로지를 구축한다.

이 토폴로지의 근간은 **글로벌 데이터 스페이스(Global Data Space)**라는 논리적인 공유 메모리 개념의 추상화 공간이다. 각 네트워크 노드(Node)들은 데이터의 물리적 주소가 아닌 ’주제(Topic)’라는 메타데이터 속성을 기준으로 연결된다.

본 다이어그램이 나타내는 바와 같이, 생산자(Publisher)는 데이터를 목표 노드로 직접 송신하지 않으며 오직 해당 토픽 채널 공간에 상태를 전개할 뿐이다. 소비자(Subscriber) 역시 데이터의 출처 IP나 MAC 주소를 전혀 알 필요 없이 오로지 원하는 토픽의 생명주기(Lifecycle) 만을 구독한다. 이는 소프트웨어 모듈 간의 시공간적 결합도(Spatio-Temporal Coupling)를 획기적으로 감축시키는 핵심 논리 토폴로지라 할 수 있다.

2. 비순환 방향 그래프 (DAG) 기반의 동적 피어링 (Dynamic Peering)

DDS 논리망에서 발생되는 신호 흐름 토폴로지는 참여자(Participant)들이 런타임에 동적으로 진입(Discovery) 및 탈퇴(Leave)할 수 있는 동적 피어 투 피어(Dynamic Peer-to-Peer) 구조를 갖는다.
네트워크 상의 제어 메시지는 닫힌 루프(Closed Loop)에 의한 양의 피드백 진동(Positive Feedback Oscillation)을 방지하기 위해 엄격히 **비순환 방향 그래프(Directed Acyclic Graph, DAG)**의 트리 논리 형태를 띠며 라우팅된다.
특히 PX4 펌웨어 단에서 오프보드로 전달되는 물리량(예: 가속도, 각속도, 짐벌 상태 등)은 단반향 엣지(Edge)의 특성을 지니며, 역방향 반환 채널은 위치 제어 세트포인트(TrajectorySetpoint)와 같이 완전히 독립적인 제 2의 토픽 트리를 구성하여 무한 루프 파장을 구조적으로 물리친다.

3. Ardupilot MAVLink 릴레이 통신과의 토폴로지 우수성 비교

MAVLink 기반 점대점(Point-to-Point) 통신에 근간을 두는 Ardupilot 시스템 모델과 비교 시, DDS 토폴로지의 아키텍처적 우위성은 명확히 드러난다.

  • MAVLink 기반 계층적 릴레이 방식 (Ardupilot): 복수의 오프보드 비전(Vision) 센서 노드가 단일 비행 제어기 하드웨어에 배포되는 상황에서, Ardupilot은 다중 접속을 중재하기 위하여 MAVProxy 혹은 MAVROS와 같은 중앙 집중식 릴레이 라우터 서버 계층을 하나 더 구비해야 한다. 이는 논리적 스타(Star) 토폴로지 구성을 강제하여, 중앙 노드 사멸 시 전체 트래픽이 마비되는 단일 장애점(Single Point of Failure, SPOF) 위협에 노출된다.
  • DDS 네이티브 다이렉트 매칭 (PX4-Autopilot): 반면 PX4 시스템의 uXRCE-DDS 클라이언트는 MAVLink라는 전통적 허브를 생략하고 곧바로 오프보드의 DDS 논리망 브로드캐스트 멀티캐스트 그룹에 분산 편입된다. 엣지 디바이스 노드가 N개 늘어나더라도, 중앙 허브의 CPU 페이로드 병목 척도를 증가시키지 않는다. 이는 확장에 한계가 없는 스케일 아웃(Scale-out) 확장성 관점에서 DDS만의 절대적인 성능적 쾌거이다.

4. 자율 제어 시스템 관제 기술 적용 시사점 (Control Understanding)

다수의 에이전트 인공지능이 병렬로 실행되는 차세대 무인항공 플랫폼(GCS 및 자율 에이전트 복합체)에서 이러한 데이터 평면 분배 논리 토폴로지가 창출하는 레이턴시 효율은 산술적 한계를 타파해 준다.
예컨대, 오프보드 컴퓨터 내부에서 광학 흐름(Optical Flow) 노드와 객체 추적(Object Tracking) 노드가 동시에 카메라 고속 이미지 토픽과 PX4의 VehicleOdometry 토픽을 요구한다고 가정하자. MAVLink 단일 목적지 기반 토폴로지에서는 똑같은 페이로드를 채널 수만큼 반복해서 직렬화 및 물리 계층 전송을 감행해야 하므로 대역폭 낭비가 치명적이다.
그러나 DDS 서브스크립션 다이렉트 매칭 공간 상에서는 논리적 멀티캐스트(Multicast Topology) 트리를 OS 레벨 도메인 소켓을 통해 단 한 번만 생성한 뒤 동일 캐시 라인 상에서 여러 노드가 공유 참조하게 함으로써, 인터네트워크 트래픽 오버헤드를 0(Zero) 단위로 수렴시킬 수 있게 된다.
