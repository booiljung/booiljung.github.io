61.1.1.2 실시간 발행-구독 프로토콜(RTPS)의 동적 종단점 발견(Dynamic Endpoint Discovery) 메커니즘

분산 제어 시스템과 다중 에이전트 로보틱스 환경에서 로봇 운영체제(ROS 2)와 PX4-Autopilot을 원활하게 연동하기 위해서는 데이터 통신 계층의 자동화된 노드 인식 기술이 필수적이다. PX4-Autopilot(펌웨어 v1.14 이상 기준)은 Micro XRCE-DDS를 통해 외부 ROS 2 네트워크와 통합되며, 이 기반에는 데이터 분배 서비스(DDS, Data Distribution Service)의 핵심 유선 프로토콜인 실시간 발행-구독 프로토콜(RTPS, Real-Time Publish-Subscribe)이 존재한다. 본 절에서는 RTPS의 핵심 기능 중 하나인 동적 종단점 발견(Dynamic Endpoint Discovery) 메커니즘의 원리와 PX4 통신 아키텍처 내 작동 방식을 심층적으로 분석한다.

1. RTPS 동적 발견 메커니즘의 개요

DDS 표준에 정의된 동적 종단점 발견 메커니즘은 시스템에 참여하는 각 노드(Participant)가 상호 간의 존재를 자동으로 인식하고, 발행자(Publisher)와 구독자(Subscriber) 간의 데이터 형(Data Type) 및 서비스 품질(QoS, Quality of Service) 매칭을 수행하는 일련의 자율적 프로토콜 체계이다. 중앙 서버에 의존하지 않고 피어 투 피어(Peer-to-Peer) 기반으로 노드를 검색하므로, 비행 중인 드론이나 임시로 네트워크에 진입하는 지상 관제 시스템(GCS, 예: QGroundControl v4.3 이상) 환경에서 높은 유연성과 내결함성(Fault Tolerance)을 보장한다.

RTPS 프로토콜의 발견 메커니즘은 크게 두 가지 하위 프로토콜로 나뉜다.

  1. 참여자 발견 프로토콜(PDP, Participant Discovery Protocol): 네트워크 상의 도메인 참여자(Domain Participant)를 서로 알리기 위해 사용된다. 주로 단순 참여자 발견 프로토콜(SPDP, Simple Participant Discovery Protocol)이 구현되어 멀티캐스트(Multicast)를 통해 참여자 정보를 주기적으로 브로드캐스팅한다.
  2. 종단점 발견 프로토콜(SEDP, Endpoint Discovery Protocol): SPDP를 통해 서로를 인식한 참여자 간에 유니캐스트(Unicast)를 이용하여 구체적인 발행자 및 구독자의 정보(토픽명, 데이터 타입, QoS 매개변수 등)를 교환하는 프로토콜이다.

2. 동적 종단점 발견(Endpoint Discovery)의 단계적 프로세스

동적 발견 과정은 다음과 같은 순서로 진행된다. 데이터의 일관성 및 실시간성을 확보하기 위해 일련의 악수(Handshake) 과정을 거치게 된다.

sequenceDiagram
    participant PX4_Agent as Micro XRCE-DDS Agent (ROS 2 Domain)
    participant ROS2_Node as ROS 2 Node (GCS / Companion CV)
    
    note over PX4_Agent, ROS2_Node: 1단계: 참여자 발견 (SPDP)
    PX4_Agent->>ROS2_Node: Multicast SPDP (Participant Data)
    ROS2_Node->>PX4_Agent: Multicast SPDP (Participant Data)
    
    note over PX4_Agent, ROS2_Node: 2단계: 양방향 신뢰 채널 형성
    PX4_Agent->>ROS2_Node: Unicast (Participant Proxy 활성화)
    
    note over PX4_Agent, ROS2_Node: 3단계: 종단점 발견 (SEDP)
    PX4_Agent->>ROS2_Node: SEDP (Publisher info: Topic, Type, QoS)
    ROS2_Node->>PX4_Agent: SEDP (Subscriber info: Topic, Type, QoS)
    
    note over PX4_Agent, ROS2_Node: 4단계: 토픽 매칭 및 데이터 스트림
    ROS2_Node-->>PX4_Agent: 매칭 완료 로직 수행
    PX4_Agent->>ROS2_Node: RTPS User Data 전송 (센서, 상태 등)

위의 도표에서 나타나듯, 발견 과정은 다중 캐스트 기반의 초기 인식을 거쳐, 유니캐스트 기반의 세밀한 매개변수 교환으로 전환된다. QoS 매개변수(예: Reliability, Durability)가 양립(Compatible)할 경우에만 종단점 매칭이 성립되며, 이후 본격적인 데이터의 발행 및 구독이 시작된다.

3. PX4와 Micro XRCE-DDS 환경에서의 발견 역학

PX4 환경에서는 드론 내부의 자원 제약(메모리 및 CPU 처리량)으로 인해 완전한 형태의 RTPS 스택을 탑재하기 어렵다. 따라서 PX4는 Micro XRCE-DDS 클라이언트(Client)로 동작하고, 연산용 보드(Companion Computer) 또는 GCS 하드웨어에서 동작하는 Micro XRCE-DDS 에이전트(Agent)가 완전한 RTPS 논리 노드로서 기능한다.

3.1 에이전트 위임 모델(Agent Delegation Model)

PX4 내부의 uORB 토픽들은 Micro XRCE-DDS 클라이언트를 통해 에이전트로 전달된다. 이때 에이전트는 DDS 네트워크 상에서 논리적 도메인 참여자(Logical Domain Participant)로 행동하며, 네트워크의 다른 ROS 2 노드들과 RTPS 동적 발견 프로토콜(SPDP/SEDP)을 수행한다. 즉, 무거운 SPDP 멀티캐스트 트래픽과 SEDP 매칭 오버헤드를 리소스가 충분한 호스트 CPU 영역으로 위임(Offload)함으로써, 비행 제어기(NuttX RTOS)의 실시간 제어 스케줄링 간섭을 최소화하는 하드웨어-소프트웨어 공동 설계(Co-design) 원리를 실현한다.

3.2 QoS 불일치와 발견 실패 사례

동적 발견 메커니즘에서 가장 빈번하게 발생하는 문제는 양 종단 간의 QoS 매개변수 불일치로 인한 SEDP 매칭 실패 현상이다.

  • 예시: PX4 에이전트 측에서 센서 데이터(예: /fmu/out/sensor_combined)를 BEST_EFFORT 신뢰성(Reliability)으로 발행하고 있으나, 수신 측 ROS 2 노드가 RELIABLE 신뢰성을 요구하는 경우 SEDP 단계에서 매칭이 거부되어 데이터 교환이 시작되지 않는다.
  • 이러한 매칭 파편화를 방지하기 위해 ROS 2 상속 아키텍처에서는 XML 기반 프로파일을 활용하여 발행-구독 노드 간의 QoS 프로토콜 협상 규칙을 사전에 엄격히 정의해야 한다.

4. 논문 및 아키텍처 출처 참조

본 기능적인 아키텍처는 국제 OMG(Object Management Group)의 “Data Distribution Service (DDS) Interoperability Wire Protocol Specification (DDSI-RTPS)” 문서에 기반하며, 통신 부하에 관련된 세부 최적화 기법에 관해서는 드론 시스템 제어 공학의 최신 연구 문헌을 적극 반영하고 있다. 자원 제약적 환경하에서 RTPS의 확장성에 관한 논의는 “Performance Evaluation of DDS and MQTT for Resource-Constrained Devices“와 같은 선행 연구에서 그 효율성을 수학적으로 모델링하여 제시한 바 있다.

5. 결론 및 실무적 고려사항

PX4 시스템의 확장에 있어 RTPS의 동적 종단점 발견 메커니즘은 정적인 네트워크 설정 없이도 런타임에 다수의 기체(Swarm)와 지상 서버 간의 유기적 통로 설정을 가능하게 한다. 그러나 스웜 드론 모델 혹은 대규모 네트워크로 노드 개수가 팽창할 경우, 각 노드에서 주기적으로 발생하는 SPDP 브로드캐스트 트래픽이 기하급수적으로 증가하여 네트워크 매체의 대역폭을 잠식하는 ‘발견 폭풍(Discovery Storm)’ 문제가 발생할 수 있다. 이에 대비하여 공학 실무에서는 ROS 2의 초기 피어(Initial Peer) 기능을 통한 멀티캐스트 제어나, 빠른 로컬 네트워크 브릿지를 구성하는 하이브리드 토폴로지 구성을 채택해야 한다.
