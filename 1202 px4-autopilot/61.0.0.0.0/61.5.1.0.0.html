61.5.1. 전송 매체별 섀넌-하틀리 정리(Shannon-Hartley Theorem) 기반 채널 용량 및 스루풋 모델

분산 자율 비행(Distributed Autonomous Flight) 생태계에서 PX4-Autopilot과 로컬 컴패니언 컴퓨터(Local Companion Computer, 예: ROS2 노드) 간의 하드웨어 레벨 통신 링크는 잡음(Noise)과 간섭(Interference)으로부터 자유로울 수 없다. 항공기의 파워트레인에서 발생하는 전자기 간섭(EMI)과 비행 제어 유닛(FCU)의 열화상 잡음(Thermal Noise)은 통신 채널의 아날로그 특성을 지속적으로 훼손시킨다. 이러한 잡음 기반의 물리적 제약 속에서 디지털 데이터의 신뢰성 있는 전송을 보장하기 위해, 정보이론의 핵심인 섀넌-하틀리 정리(Shannon-Hartley Theorem)를 도입하여 채널 용량(Channel Capacity)을 이론적으로 규명하고, 이를 기반으로 최적화된 스루풋(Throughput) 모델을 설계하는 과정이 요구된다.

본 절에서는 섀넌-하틀리 정리를 바탕으로 PX4-Autopilot(Firmware Version 1.14 이상)의 다양한 전송 매체별 통신 용량을 정량화하고, 잡음 환경에서의 한계 스루풋 달성을 위한 이론적 토대를 분석한다.

1. 섀넌-하틀리 정리의 теорети적 배경과 직렬 채널 접근

정보이론(Information Theory)을 주창한 클로드 섀넌(Claude Shannon)의 정리에 따르면, 부가 백색 가우시안 잡음(AWGN, Additive White Gaussian Noise)이 존재하는 임의의 아날로그 채널을 통해 오류 없이 전송할 수 있는 최대 디지털 정보의 전송률(Data Rate)은 다음 공식으로 결정된다.

C = B \log_2 \left( 1 + \frac{S}{N} \right)

여기서,

  • C : 채널 용량(Channel Capacity), 즉 초당 전송 가능한 최대 비트 수 (bits per second, bps)
  • B : 매체의 통과 대역폭(Bandwidth) (Hz)
  • S : 평균 수신 신호 전력(Signal Power) (Watts)
  • N : 채널 내의 가우시안 잡음 전력(Noise Power) (Watts)
  • S/N : 신호 대 잡음비 (SNR, Signal-to-Noise Ratio)

이 수식은 “전송 매체의 물리적 대역폭이 넓을수록, 그리고 잡음 대비 신호 전력이 강력할수록 더 많은 양의 디지털 데이터를 높은 보드 레이트(Baudrate)로 전송할 수 있다“는 공학적 진리를 증명한다.

1.1 유선 UART 직렬 링크에서의 한계 역산

보편적으로 Pixhawk 4 또는 Pixhawk 6C 관제 시스템 외부 단자(TELEM1, TELEM2)에서 뻗어나오는 범용 비동기 송수신(UART) 라인은 기저 대역(Baseband) 주파수 특성을 가진다. UART 자체는 변조(Modulation)를 거치지 않고 디지털 구형파(Square Wave)를 3.3V 또는 5V 레벨에서 그대로 전송하지만, 전선의 기생 정전용량(Parasitic Capacitance) 문제로 인해 물리 패스 자체에는 대역폭 한계 B가 명확히 존재한다.

예를 들어, UART 케이블의 컷오프 주파수(Cut-off Frequency) 대역폭 B가 약 1\text{MHz}이고 통상적인 드론 내 EMI 간섭 환경에서 신호 대 잡음비(SNR)가 약 10\text{dB} (즉, \frac{S}{N} \approx 10)라면,

C = 1,000,000 \times \log_2 (1 + 10) \approx 1,000,000 \times 3.459 = 3.459 \text{ Mbps}

이러한 물리적 한계치 C \approx 3.45\text{ Mbps} 하에서, 우리가 흔히 설정하는 최고 보드 레이트인 921,600\text{ bps} (또는 1,500,000\text{ bps})는 섀넌 한계(Shannon Limit) 내에 안전하게 포진해 있음을 수학적으로 입증할 수 있다. 반면, SNR이 극도로 열악해지는 환경(예: 고전력 BLDC 모터 선축과 UART 선이 밀착된 상태)에서는 위 공식에 의해 한계 채널 용량 C가 급격하게 저하되어, 921,600\text{ bps} 전송 시 비트 오류율(Bit Error Rate, BER)이 기하급수적으로 폭증하게 됨을 예측할 수 있다.

2. 매체별 채널 특성 파라미터화(Parameterization)

최신의 분산 비행 제어 시스템은 통상 단일 매체에 국한되지 않고 다목적 인터페이스를 사용한다. 각각의 매체 특성에 따른 프로비저닝 차이를 인지해야 한다.

2.1 이더넷(Ethernet, 100BASE-T1 등)

PX4 스택 내에서 UDP/IP 처리를 위해 사용되는 이더넷(예: Pixhawk 5X 이상의 Autocore 환경)은 기본적으로 100\text{ MHz} 이상의 넉넉한 대역폭 B와, 차동 신호 쌍(Differential Signal Pair) 전송방식에 힘입어 매우 우수한 SNR을 확보하고 있다. 따라서 섀넌-하틀리 공식 적용 시 수십에서 수백 \text{Mbps} 수준의 용량을 보장하므로, 실질적으로 uORB 메시지 트래픽(대역폭 수 \text{Mbps} 이내) 송수신 시 링크 측면의 물리적 병목은 거의 제로(Zero)에 수렴한다.

2.2 USB/CAN Bus 링크에서의 시간 및 다채널 분할

CAN(Controller Area Network) 버스, 특히 UAVCAN/DroneCAN이나 ROS2 기반 전송 환경에서의 대역폭 B는 고정(1Mbps 또는 8Mbps(CAN-FD))되지만, 버스에 물려있는 여러 노드(센서 파드, 모터 컨트롤러 등)들이 이 대역폭을 시분할 다중 접속(TDMA 또는 CSMA/BA) 형태로 공유한다. 이때 각 노드가 겪는 유효(Effective) 대역폭은 B_{eff} = B / (\text{활성 노드의 수})로 급감하므로, 섀넌 한계 도달 이전에 매체 접근 지연(MAC Delay)이 스루풋(Throughput) 저하를 일으키는 주요인으로 작용한다.

결과적으로, 통신 채널에 대한 용량 할당(Provisioning)은 다음과 같은 모델 흐름으로 귀결된다.

    A[매체 통과 대역폭 B 측정] --> B{노이즈 환경 실측력 SNR 추론}
    B --> C["섀넌 통신 용량 한계 (Capacity) 계산"]
    C --> D[요구 전송 레이트 (Baudrate)]

    E -->|Yes| F[비트 오류율(BER) 안정권]

    F --> H[오버헤드 팩터 적용 유효 스루풋 역산]

3. 스루풋(Throughput) 역산 및 안정적 프로비저닝 전략

기체 제조사나 시스템 통합 엔지니어 모델 설계 시, 이론상 채널 용량 C를 100\% 맹신하여 실제 스루풋 상한으로 설정하는 것은 금물이다. Ardupilot과 PX4-Autopilot를 불문하고 운영체제의 인터럽트 패널티 대역 및 프로토콜 래핑 패딩(Padding)이 존재하므로, 통상 안전 스루풋 계수(Safety Factor) \alpha (일반적으로 0.4 \sim 0.6)를 곱하여 유효 설정 가능 스루풋 마진(C_{effective} = \alpha \times C)을 정의한다.

이를 기반으로, “이 통신 라인에서 최대 몇 개의 uORB 토픽(초당 100Hz 기준)을 ROS2로 포워딩할 수 있는가?“를 실무적으로 사전 검증하는 모델이 성립된다. 이와 같은 수학적이며 선제적인 진단이야말로 분산 멀티로터의 자율화를 뒷받침하는 차세대 GCS 관제 기술의 핵심이다.
