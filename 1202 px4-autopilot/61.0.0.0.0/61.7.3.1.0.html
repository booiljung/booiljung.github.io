61.7.3.1. 최선 노력(Best-Effort) 전송 방식의 제어 주기 지연 시간 감소 특성과 신뢰성(Reliable) 보장 방식의 확인 응답(ACK) 라운드 트립(Round-Trip) 지연 간의 제어 성능 파레토 최적 곡선 도출

PX4-Autopilot(Firmware Version 1.14 이상)과 ROS2를 결합한 오프보드 제어(Offboard Control) 아키텍처에서, 통신 미들웨어(uXRCE-DDS)의 서비스 품질(QoS) 프로파일 결정은 비행체의 동역학적 궤적 추종 성능을 좌우하는 가장 치명적인 인자(Factor) 중 하나이다. 특히 데이터 도달의 무결성을 보장하는 신뢰성(Reliable) 방식과 지연 시간의 최소화를 추구하는 최선 노력(Best-Effort) 방식은 물리적 한계를 가진 무선 링크망(Radio Link)에서 불가피한 상충 관계(Trade-off)를 형성한다.

본 절에서는 통신 오류 확률(Packet Error Rate, PER)이 존재하는 비선형 무선 환경에서, Best-Effort의 극단적인 저지연 특성과 Reliable 방식의 ACK/NACK 라운드 트립(Round-Trip) 지연 누적 간의 역학을 수학적으로 모델링하여, 폐루프 제어(Closed-Loop Control) 성능을 극대화할 수 있는 파레토 최적 곡선(Pareto Optimal Curve) 도출 기법을 심층 분석한다.

1. 신뢰성 프로파일에 따른 전송 지연 시간(Latency) 역학 모델

무선 채널의 패킷 에러 제어(Error Control) 방식에 따라, ROS2 노드에서 PX4 비행 제어기(FCU)로 하달되는 제어 명령(TrajectorySetpoint 등)의 유효 지연 시간 T_{eff} 는 완전히 다른 수학적 거동을 보인다.

1.1 최선 노력(Best-Effort) 방식의 지연 특성

Best-Effort QoS 하에서 송신단은 데이터 프레임을 매체에 뿌려놓고(Fire-and-Forget) 뒤돌아보지 않는다. 수신단 역시 에러가 난 패킷은 쿨하게 버리고(Drop) 다음 패킷을 기다릴 뿐, 재전송을 요구하지 않는다.

  • 유효 지연 시간(T_{BE}): 순수한 편도 전송 지연 시간(One-way Delay)인 T_{prop} + T_{proc} (전파 지연 + 파싱 지연) 에 수렴한다.
  • 맹점(Blind Spot): 네트워크 에러율(P_e > 0)이 높아 연속적으로 3개 이상의 패킷이 버려지면, 제어기 입장에서는 다음 명령을 받을 때까지 과거의 낡은 제어값(Zero-Order Hold)을 유지해야 하므로 제어 오차(Tracking Error)가 발산하기 시작한다.

1.2 신뢰성(Reliable) 보장 방식의 라운드 트립 팽창 현상

Reliable 방식(MAVLink의 재전송 큐 또는 RTPS의 ACK/NACK 매커니즘)에서는 수신단이 제대로 받았다는 ACK를 보낼 때까지, 혹은 Timeout 발생 시 송신단이 끝까지 재전송을 욱여넣는다.

지연 시간 수학적 기대값 E[T_{REL}] 은 에러 확률 P_e 와 RTT(Round-Trip Time, T_{RTT} = 2 \times T_{prop})의 무한 등비급수로 전개된다.

E[T_{REL}] = T_{prop} + \sum_{k=1}^{\infty} P_e^k (T_{RTT} + T_{timeout})

  • 치명적 한계(Fatal Flaw): P_e 가 10\% 를 넘어서는 순간 꼬리 지연(Tail Latency)이 지수적(Exponential)으로 폭주한다. 오래된 센서 데이터를 재전송받아봐야, \Delta t 단위로 현행 동역학을 적분해야 하는 EKF2(Extended Kalman Filter) 입장에서는 “시간 여행을 한 쓰레기 데이터“일 뿐이므로 이노베이션 리젝션(Innovation Rejection)을 내뿜고 안정성을 잃게 된다.

2. 제어 성능 지표와 파레토 최적 곡선(Pareto Optimal Curve) 도출

네트워크 엔지니어링에서의 ‘파레토 최적(Pareto Optimal)’ 상태란, “A 지표(제어 주기)를 훼손하지 않고서는 B 지표(제어 궤적 오차율)를 조금도 개선할 수 없는 한계 곡선” 상에 존재하는 시스템 세팅 점을 의미한다.

2.1 다목적 최적화 공간(Multi-Objective Optimization Space) 매핑

x축을 유효 제어 지연 시간(E[T_{eff}]), y축을 궤적 추종 오차 반경(RMSE, Root Mean Square Error)라 정의하고, 네트워크 상태(P_e)에 따라 시뮬레이션 데이터를 플로팅(Plotting)한다.

  1. 저잡음, 근거리 환경 (P_e < 1\%)
    대부분의 전송이 1회 차에 성공하므로 Reliable과 Best-Effort의 지연 시간 차이가 거의 없다. 그러나 1\% 의 누락마저 완벽히 틀어막아 주는 Reliable 세팅이 제어 오차를 더 줄여주므로, 파레토 최적점은 Reliable 쪽에 머문다.
  2. 고잡음, 간섭 폭주 환경 (P_e > 10\%)
    네트워크 상태가 나빠지면 Reliable의 E[T_{REL}] 이 수백 \text{ms} 단위로 늘어나며, 지연에 극도로 예민한 PID Rate 제어 루프를 산산조각 낸다. 이 시점부터는 패킷의 10\% 를 손실하더라도 남아있는 90\% 의 최신 패킷을 지연 없이(T_{BE}) 수혈하는 Best-Effort 세팅만이 기체가 땅에 처박히는 것(Crash)을 막을 수 있는 유일한 파레토 최적 해(Optimal Solution)가 된다.

2.2 하이브리드 토픽 기반 적응형(Adaptive) QoS 파인 튜닝

PX4-ROS2 아키텍트가 도달해야 할 최종 완성형은 이 파레토 곡선을 런타임 중에 횡단하는 시스템 구축이다.

  • 고속 센서(IMU, Odometry) 토픽은 망의 상태와 무관하게 무조건 항구적인 Best-Effort 파레토 극단에 바인딩한다.
  • 임무 파일과 글로벌 파라미터는 시간 지연에 아랑곳하지 않고 완벽 무결성이 요구되므로 Transient Local + Reliable 프로파일에 영구 바인딩한다.
  • 오프보드 제어 명령(TrajectorySetpoint) 은 통상적으로 50\text{Hz} 의 Best-Effort로 구동하다가, 기체가 거친 바람(Gust)과 통신 음영 지역에 휩싸여 패킷 드롭이 극심해질 경우 즉각 ROS2 노드에서 스로틀링(Throttling)을 걸어 발간 주기를 10\text{Hz} 로 낮추되 QoS를 Reliable 로 동적 업그레이드하여, ’제어 주기 감소’와 ‘명령 신뢰성’ 사이의 새로운 교착 파레토 점으로 피신하는 방어 로직을 설계해야 한다.

이와 같이 무선 통신 계층의 불확실성을 역산하여 제어 공학의 오차율 한계선과 맞바꾸는 파레토 최적 곡선의 도출 작업은, 맹목적인 ’빠른 통신’을 넘어서서 악조건 속에서도 제어 위상 여유(Phase Margin)를 방어해 내는 가장 강건한 튜닝 철학이다.
