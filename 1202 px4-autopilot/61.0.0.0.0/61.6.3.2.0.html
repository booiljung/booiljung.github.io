61.6.3.2. 데이터 패킷 도달 시간의 확률 밀도 함수(PDF) 편차(Jitter) 한계 측정을 통한 폐루프 제어기(Closed-Loop Controller)의 위상 여유(Phase Margin) 훼손 및 불안정성 유발 임계점 분석

PX4-Autopilot(Firmware Version 1.14 이상)에 기반한 자율 로보틱스 시스템에서 ROS2와 결합된 오프보드(Offboard) 폐루프 제어(Closed-Loop Control)는 본질적으로 센서 수집 \rightarrow 네트워크 전송 \rightarrow 제어 연산 \rightarrow 명령 회신의 피드백 파이프라인을 형성한다. 이 과정에서 이더넷 송수신 버퍼나 Wi-Fi의 반송파 감지 다중 접속(CSMA/CA) 매체 제어 기술은 필연적으로 패킷 도달 시간의 비결정성(Non-determinism)을 야기한다. 평균 지연 시간(Average Latency)이 제로에 수렴하더라도, 이 도달 시간의 요동(Fluctuation), 즉 지터(Jitter)가 임계치를 넘어서면 제어 공학의 핵심 안성성 지표인 위상 여유(Phase Margin)를 급격히 갉아먹는다.

본 절에서는 데이터 패킷 도달 시간의 확률 밀도 함수(PDF, Probability Density Function)를 도출하여 네트워크 지터의 확률적 분포를 정량화하고, 이 지터 한계가 폐루프 제어기의 주파수 응답에 미치는 위상 훼손 동역학 및 물리적 제어 불안정성(Instability)을 유발하는 수학적 임계점(Critical Point)을 분석한다.

1. 패킷 도달 시간(Arrival Time)의 통계적 프로파일링

오프보드 제어 환경에서 지터를 객관적으로 계측하기 위해서는, ROS2 컴패니언 컴퓨터 상에서 수신되는 px4_msgs::msg::SensorCombined 토픽의 네트워크 도달 시간 간격(\Delta t_{rx}) 데이터를 수집하여 통계 모델을 구성해야 한다.

1.1 지터(Jitter) 확률 밀도 함수(PDF) 매핑

이론적으로 완벽한 네트워크 환경이라면 \Delta t_{rx}는 송신 주기인 T_s (250\text{Hz}의 경우 0.004\text{s})라는 단일 값에 수렴하는 임펄스 함수(Dirac Delta Function) 형태를 띤다. 그러나 현실적인 DDS 네트워크 환경에서 \Delta t_{rx}의 히스토그램을 작성하면, 일반적으로 감마 분포(Gamma Distribution)나 꼬리가 긴 로그 정규 분포(Log-Normal Distribution) 형태를 나타낸다.

P(\Delta t) = \frac{1}{\sigma \sqrt{2\pi}} \exp \left( -\frac{(\Delta t - \mu)^2}{2\sigma^2} \right) \quad (\text{가우시안 유사 근사 시})

  • 평균(\mu): 이상적인 송신 주기 T_s 에 근접한다.
  • 표준 편차(\sigma): 이 값이 바로 통계적 지터 폭(Jitter Magnitude) 이다.
  • 최대 지연(\Delta t_{max}): PDF의 꼬리(Tail) 끝부분에 해당하는 극단적 이상치(Outlier)로, 통상 99.9\% 백분위수(Percentile) 지표로 측정되며 시스템 불안정성의 주범이 된다.

2. 지터에 의한 위상 여유(Phase Margin) 훼손 동역학

제어 공학 관점에서 지터는 고정된 시스템 지연 시간(Constant Time Delay)보다 훨씬 악의적으로 작동한다. 폐루프 시스템 내에서 지연 시간 T_d 는 주파수 영역에서 다음과 같은 위상 지연(Phase Lag) \phi_{delay} 를 유발한다.

\phi_{delay}(\omega) = -\omega T_d

1.2 시간 가변적 지연(Time-Varying Delay)과 위상 여유의 잠식

지터 환경에서 지연 시간 T_d 는 상수가 아닌 확률 변수 T_d(t) \sim \mu \pm \sigma 가 된다.
위상 여유(Phase Margin, PM)는 시스템 개루프(Open-Loop) 전달 함수의 이득(Gain)이 0\text{dB} 범위를 교차하는 주파수(\omega_c, 교차 주파수)에서 기체가 발산(Phase = -180^\circ)하기까지 남은 각도의 여유분이다. (안정적인 비행을 위해서는 보통 PM > 45^\circ 에서 60^\circ 가 요구된다.)

  • 만일 최대 버스트 지터 \Delta t_{max} (예: 20\text{ms})가 발생하면, 교차 주파수(\omega_c = 30\text{rad/s})에서의 순간적인 위상 지연은 \Delta \phi = 30 \times 0.02 = 0.6\text{rad} \approx 34.4^\circ 에 달한다.
  • 기존 기체의 기계적 위상 여유가 50^\circ 였다면, 이 순간 위상 여유는 50^\circ - 34.4^\circ = 15.6^\circ 로 급전직하한다.

결과적으로 지터 편차가 일시적으로 확산되는 순간, 기체는 바람이나 작은 스틱(Stick) 조작과 같은 외란에 추종하지 못하고 강하게 떤다(Oscillate).

2. 불안정성 유발 임계점(Critical Point) 및 아키텍처 방어책

설계자는 PDF 분석을 통해 산출된 최대 지터 폭(3\sigma 또는 99.9\% 한계)이 시스템을 불안정성(PM \le 0^\circ)으로 몰고 가는 임계 주파수 대역을 사전에 확정해야 한다.

2.1 붕괴 임계점(Instability Threshold) 산출

통상적으로 제어기 한계 지터 \sigma_{max} 는 다음 조건식을 초과해서는 안 된다.

\sigma_{max} < \frac{PM_{design}}{\omega_c} - T_{fixed\_delay}

즉, 기체의 설계상 위상 여유(PM_{design})가 좁고 제어 주파수 대역폭(\omega_c)이 넓은 민첩한(Agile) 레이싱 드론일수록 허용 가능한 네트워크 지터 폭(\sigma_{max})은 극단적으로 작아진다.

3.2 오버샘플링(Oversampling)과 수신 버퍼 재생(Playback Buffer) 방어

이러한 한계를 방비하기 위해 ROS2 측 오프보드 제어기 아키텍처는 다음과 같은 기술을 통합해야 한다.

  1. DDS 트래픽 분리: 대용량 점대점 데이터(카메라 이미지 등)와 제어 루프 데이터(uORB 텔레메트리)를 물리적으로 분리된 도메인 ID 혹은 별개의 이더넷 스위치 큐(QoS)로 격리하여 롱테일(Long-tail) 패킷 지연을 절단(Truncate)한다.
  2. 지터 버퍼(Jitter Buffer): 수신단에서 즉각적인 콜백 처리를 보류하고 극소량의 시간 공간(예: 2\text{ms}) 동안 버퍼링한 뒤, 로컬 실시간 타이머(Real-Time Timer)에 맞춰 일정한 T_s 간격으로 재송출(Playback)하여 이산 적분 루프의 시간 델타(\Delta t)를 강제로 상수로 수렴시킨다. 이는 지연 시간 면에서는 손해를 보지만(+2\text{ms} 패널티), 지터를 완전히 소멸(Zero Jitter)시킴으로써 위상 여유의 변동성을 제압하는 고전적이고 확실한 신호 처리 기법이다.
