61.7.3.2. 후기 진입 가입자 노드(Late-Joiner Subscriber)의 초기 상태 벡터 동기화를 위한 링 버퍼(Ring Buffer) 형태의 이력(History) 및 내구성(Durability) 메모리 큐 크기의 지수 보상(Exponential Backoff) 동적 할당 알고리즘

PX4-Autopilot(Firmware Version 1.14 이상)과 다수의 ROS2 노드로 구성된 분산 제어 시스템(Distributed Control System)에서, 모든 노드가 전원을 켜자마자 동시에 네트워크에 완벽히 접속하는 이상적인 시나리오는 존재하지 않는다. 지상 관제 시스템(GCS) 애플리케이션이 비행 도중 재시작되거나, 특정 컴퓨터 비전(Computer Vision) 노드가 부팅 지연으로 뒤늦게 uXRCE-DDS 네트워크 망에 합류하는 상황은 일상적으로 발생한다.

이러한 후기 진입 가입자(Late-Joiner Subscriber) 노드가 시스템에 배속되었을 때, 이 노드가 “현재 기체의 비행 모드는 무엇인가?”, “기체에 설정된 궤적 웨이포인트(Waypoint) 맵은 어떠한가?“와 같은 핵심 ’초기 상태 벡터(Initial State Vector)’를 즉각적으로 동기화받지 못하면 치명적인 오작동을 유발한다. 본 절에서는 이런 현상을 방지하기 위해 퍼블리셔(Publisher) 단에서 운용하는 이력(History) 및 내구성(Durability) QoS 프로파일의 링 버퍼(Ring Buffer) 설계와, 자원 제약 환경에서 이 큐(Queue)의 크기를 동적으로 최적화하는 지수 보상(Exponential Backoff) 동적 할당 알고리즘을 분석한다.

1. 내구성(Durability) QoS와 링 버퍼(Ring Buffer) 캐싱 아키텍처

DDS 미들웨어는 데이터의 ’발행 시점’과 가입자의 ’구독 시점’이 어긋나는 시간적 비동기화 영역을 극복하기 위해 내구성(Durability) 인프라를 제공한다.

1.1 Transient Local 프로파일과 State Vector 보존

기본값인 Volatile 프로파일은 퍼블리셔가 데이터를 방사(Emit)한 직후 수신 확인 여부와 상관없이 메시지를 휘발시킨다. 이와 대조적으로 Transient Local 프로파일을 설정하면, 송신 노드의 로컬 메모리에 일정한 이력(History)을 보존하는 캐시(Cache) 영역이 활성화된다.
후기 진입 노드(Late-Joiner)가 DDS 매칭 과정(Endpoint Discovery)을 마치는 즉시, 퍼블리셔는 이 캐시에 담겨 있던 과거 메시지들을 일제히 방출(Burst)하여 가입자의 상태 벡터를 현재 시간으로 순식간에 동기화(Sync)시킨다.

1.2 링 버퍼(Ring Buffer) 형태의 KEEP_LAST 관리 기법

FCU와 같은 RAM 자원이 지극히 고갈된 환경에서 무한정 이력을 쌓아둘 수는 없다(OOM 발생). 따라서 송신 큐는 꼬리가 머리를 물고 도는 고정 크기(Fixed-size)의 링 버퍼 형태로 구현되며, 이를 KEEP_LAST History QoS 속성이라 한다.

만일 Depth(N)가 10으로 설정되어 있다면, 11번째 vehicle_status 메시지가 발간되는 순간 가장 오래된 1번째 메시지를 메모리에서 덮어씌운다(Overwrite). 늦게 진입한 노드는 이 링 버퍼에 존재하는 가장 신선한 N개의 맥락(Context) 데이터만을 전달받게 된다.

2. 큐 크기(Queue Depth)의 지수 보상(Exponential Backoff) 동적 할당 알고리즘

그러나 파라미터(Parameter) 동기화나 임무(Mission) 업로드와 같이 “단 하나의 데이터도 누실되어서는 안 되는(KEEP_ALL)” 거대 상태 벡터를 다룰 때는 링 버퍼의 크기(Depth)를 사전에 고정하기 어렵다. 통신이 극도로 단절된 무선 환경에서는 버퍼가 순식간에 차오르게 되지만, FCU의 한정된 힙(Heap) 메모리를 맹목적으로 소진시킬 수는 없다.

이를 소프트웨어적으로 조율하는 지수 보상 백오프(Exponential Backoff) 기반의 동적 버퍼 할당 알고리즘이 도입된다.

2.1 지수적 성장(Exponential Growth)과 한계 클램핑(Clamping)

메시지 재전송 대기열(Wait-Queue)이나 이력 큐에 적체된 프레임 수가 임계점(Threshold)을 넘기려 판단되면, 알고리즘은 버퍼 슬롯을 지수적으로 늘려 공간을 긴급 대출(Lending)받는다. 그러나 일정 횟수 이상(예: 통신 단절 5\text{초} 초과) 실패가 반복되면 메모리 고갈 방지를 위해 상한선(Max Bound)에서 하드 클램핑(Clamp)을 걸고 지수 보상 타이머를 발동시켜 낡은 메시지를 강제로 증발시킨다.

2.2 비행 안정성(Flight Stability)과의 결속

이 알고리즘은 비행 제어기가 통신을 복구하기 위해 “얼마나 메모리 자원 감수성에 베팅할 것인가“를 동적으로 결정(Negotiation)하는 로직이다. Late-Joiner 노드(예: 딥러닝 보조 제어기)의 동기화를 위해 무리하게 버퍼를 유지하다가는 메인 스레드가 OOM으로 자살하는 파국을 맞는다. 지수 보상 알고리즘은 통신 모듈 스스로가 적절한 시점에 데이터 동기화 포기를 선언하게 함으로써, 전체 항공기계 시스템의 물리적 생존성(Survival)을 우위에 두는 고도의 소프트웨어 타협(Compromise) 설계이다.
