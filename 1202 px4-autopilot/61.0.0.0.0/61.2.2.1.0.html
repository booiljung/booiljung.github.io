61.2.2.1. 데비안 패키지 관리 도구(APT)의 위상 정렬(Topological Sort)을 이용한 동적 링크 라이브러리(.so) 의존성 트리 병합 및 충돌 회피 알고리즘

1. 개요

우분투(Ubuntu) 환경에서 ROS2와 PX4-Autopilot의 오프보드 제어 스택을 구축할 때, 사용자는 필연적으로 수백 개의 오픈소스 라이브러리와 ROS 패키지들을 데비안 패키지 관리 도구(APT, Advanced Package Tool)를 통해 설치하게 된다. 이때 rclcpp, px4_msgs, Fast-DDS 등의 상위 패키지들은 각기 다른 하위 공유 라이브러리(.so, Shared Object)를 요구하며, 의존성 관계가 복잡하게 얽히게 된다. 본 절에서는 APT가 설치해야 할 동적 링크 라이브러리 패키지들의 순서를 어떻게 수학적으로 결정하는지, 그리고 버전 충돌 방지를 위해 의존성 트리를 কিভাবে 병합하고 위상 정렬(Topological Sort) 알고리즘을 적용하는지 심층적으로 추적한다.

2. 패키지 의존성의 방향성 비순환 그래프(DAG) 모델링

APT 시스템에서 데비안 패키지 간의 “A가 B를 필요로 한다(A Depends on B)“라는 관계는 방향성 비순환 그래프(Directed Acyclic Graph, DAG)로 완벽하게 모델링된다. 여기서 각 정점(Node)은 .deb 패키지(예: libfastrtps.so)를 나타내고, 간선(Edge)은 의존성(Depends, Pre-Depends) 조건을 의미한다.

만약 의존성 그래프 내에 순환(Cycle, 예: A가 B를 요구하고 B가 다시 A를 요구하는 상태)이 존재한다면 시스템은 닭과 달걀의 논리적 모순에 빠지게 된다. 따라서 APT의 의존성 해석기(Dependency Resolver)는 데비안 리포지토리 메타데이터를 파싱하여 철저한 비순환 구조를 검증하고, 모순 없는 단일 컴파일 및 설치 파이프라인 그래프를 구축한다.

3. 위상 정렬(Topological Sort)을 통한 설치 순서 결정

위상 정렬은 유향 그래프의 모든 정점을 방향성에 위배되지 않도록 선형(Linear)으로 수열화하는 수학적 알고리즘이다. APT는 패키지 설치 시 pre-depends 및 depends 필드를 분석하여, 다른 어떤 패키지도 의존하지 않는 최말단 기초 라이브러리(예: libc6, libgcc_s1)부터 상위 메타 패키지(예: ros-humble-desktop) 방향으로 트리를 역순회(Reverse Traversal)하며 설치 계획을 수립한다.

대체로 APT 엔진 내부에서는 타잔(Tarjan) 알고리즘이나 칸(Kahn)의 알고리즘에 기초한 깊이 우선 탐색(DFS, Depth-First Search) 변형 기법을 사용하여 위상 순서를 추출한다. 이 과정은 하위 라이브러리(.so)가 파일 시스템에 복사 및 ldconfig 캐시에 등록된 이후에만 상위 패키지의 구성 스크립트(postinst)가 실행되도록 보장한다.

위의 위상 정렬 트리에서 보듯, libc6 \rightarrow libfastcdr \rightarrow rmw_fastrtps_cpp \rightarrow rclcpp 순으로 단방향 의존성 설치 스트림이 형성되며, 이 순서가 보장되지 않으면 설치 도중 링커(ld) 오류나 실행 시점의 심볼 누락(Unresolved Symbol) 오류가 발생한다.

4. 동적 링크 라이브러리(.so) 의존성 트리 병합 및 충돌 회피 알고리즘

ROS2 생태계와 PX4 관련 도구들은 수백 개의 개별 개발자들이 유지보수하기 때문에, 각각 요구하는 기반 C++ 라이브러리의 버전이 다를 수 있다. 이때 APT는 ’충돌 회피 알고리즘(Conflict Resolution Algorithm)’을 통해 트리 병합 시 발생하는 모순을 해소한다.

4.1 버전 고정(Version Pinning) 및 심볼 충돌 방지

데비안 패키지 네이밍 컨벤션에서는 동일한 기반 라이브러리라도 메이저 버전이 달라 애플리케이션 이진 인터페이스(ABI, Application Binary Interface)가 호환되지 않을 경우 패키지 이름을 분리한다(예: libpoco-dev와 libpoco-dev-62). APT는 위상 트리 병합 단계에서 서로 호환되지 않는 Conflicts/Replaces 태그가 발견되면 백트래킹(Backtracking) 알고리즘을 수행하여 두 패키지가 동시에 설치되지 않는 대안(Alternative) 설치 경로를 탐색한다.

4.2 가상 패키지(Virtual Package)와 프로바이더(Provider) 해석

일부 하위 구조체는 ros-humble-rmw-implementation과 같은 가상 의존성을 요구한다. 이 때 APT 탐색 트리는 제공자(Provider, 예를 들어 Fast-DDS나 Cyclone DDS) 리스트 중 시스템 정책, 우선순위 매핑, 또는 사용자 지정 선호도에 따라 가장 높은 점수(Score)를 가진 노드를 선택하여 병합함으로써, 동적 라이브러리 의존성 체인의 단절을 예방한다.

5. 정리

데비안 APT 시스템은 단순한 아카이브 압축 해제 도구가 아니다. ROS2와 PX4가 요구하는 거대한 소프트웨어 생태계가 운영체제 위에서 단 하나의 의존성 균열(.so Missing) 없이 견고히 작동할 수 있는 것은, 내부에 구현된 수학적 방향성 비순환 그래프 모델링과 정교한 위상 정렬 알고리즘 덕분이다. 이러한 의존성 충돌 회피 논리를 통해 시스템 오류를 예방하는 과정은 오프보드 제어용 데이터 계층을 무결하게 컴파일하기 위한 본질적 뼈대이다.
