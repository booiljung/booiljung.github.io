36.7.4. 고급 암호화 표준(AES)의 갈루아/카운터 모드(Galois/Counter Mode, GCM)를 활용한 페이로드 데이터의 인증된 암호화(AEAD)

1. 개요 및 인증된 암호화(AEAD)의 등장 배경 (Introduction)

앞서 다룬 HMAC-SHA256가 데이터의 위변조를 판지하는 무결성(Integrity) 검증의 최전선이라면, MAVLink 프로토콜을 통과하는 텔레메트리와 비행 명령 자체를 아무도 알아볼 수 없도록 완전히 차폐(Masking)하는 기밀성(Confidentiality)의 최고봉은 범용 대칭키 알고리즘인 **고급 암호화 표준(Advanced Encryption Standard, AES)**이다.

과거에는 암호화(기밀성 보장) 작업과 해시 서명(무결성 보장) 작업을 별도의 모듈에서 순차적으로 두 번 구동하는 Encrypt-then-MAC 접근법이 사용되었으나, 이는 자원이 열악한 임베디드 드론(MCU) 환경에서 막대한 오버헤드를 유발했다. 이를 해결하기 위해 현대 보안 아키텍처는 한 번의 파이프라인(Operation) 안에서 암호화와 무결성 검증을 동시에 달성하는 인증된 암호화(Authenticated Encryption with Associated Data, AEAD) 철학을 주창하였으며, PX4-Autopilot은 이 개념의 가장 완벽한 수학적 구현체인 **AES-GCM (Galois/Counter Mode)**을 MAVLink 2.0 보안 사양으로 전격 수용하였다.

2. 갈루아/카운터 모드(GCM)의 병렬 처리 아키텍처 (GCM Architecture)

AES 알고리즘은 128비트(16\text{ bytes}) 단위의 블록 암호화(Block Cipher)이다. 이 블록들을 어떤 흐름으로 엮어서 전체 MAVLink 페이로드를 암호화할 것인가를 결정하는 것이 ’운영 모드(Mode of Operation)’이다.

  • 과거 널리 쓰이던 CBC(Cipher Block Chaining) 모드는 바로 직전 블록의 암호화 결과물이 다음 블록의 입력으로 종속되는 직렬(Sequential) 구조를 갖는다. 이는 어느 한 블록 연산에 딜레이가 걸리면 전체 통신이 멎어버리는, 레이턴시(Latency)에 극도로 취약한 치명적인 약점을 지녔다.
  • 카운터(Counter, CTR) 모드의 승리: 반면 AES-GCM의 근간이 되는 CTR 모드는 1씩 증가하는 논리적 카운터 값을 AES 알고리즘(암호화)에 통과시켜 난수 스트림(Key Stream)을 뽑아낸 뒤, 이를 MAVLink 평문 블록과 단순 XOR 연산하여 암호문을 생성한다. 이 과정은 각 블록 병렬로 완전히 분리되어 있다. 덕분에 멀티 코어가 탑재된 최신 컴패니언 컴퓨터나 DMA(Direct Memory Access) 보조를 받는 비행 제어기에서는 다중 스레드를 통한 100% 병렬 가속 연산이 가능하여, 암호화에 소모되는 지연 시간을 극단적으로 0에 수렴하게 만든다.

3. 부가 데이터(Associated Data)와 비복호화 포워딩의 미학

AEAD 구조 내 GCM의 핵심은 카운터 모드를 통한 데이터 암호화뿐만 아니라, 갈루아 필드(GF(2^{128})) 위에서의 다항식 곡선 곱셈 연산을 통해 고속의 식별태그(GMAC, Galois Message Authentication Code)를 동시 산출한다는 점이다.

MAVLink 보안 설계에서 가장 주의 깊게 보아야 할 점은 **부가 데이터(Associated Data, AD)**의 개념이다. 암호화를 거치면 페이로드 안의 내용물은 읽을 수 없지만, 중간에 위치한 MAVLink 미들웨어 라우터 장비들은 이 패킷을 어디로 포워딩(Routing)해야 할지 알아야 하므로 패킷의 **헤더(Header: Target SysID, CompID, Message ID 등)**는 암호화하지 않고 평문으로 둔다.

AES-GCM은 기밀성이 불필요한 이 평문 헤더(Header)들을 ‘부가 데이터(AD)’ 공간으로 격리 수용한 뒤 페이로드와 합쳐 전체 무결성 서명(GMAC)을 계산해 낸다. 결과적으로 릴레이 서버나 라우팅 드론은 자신이 알지 못하는 암호를 복호화(Decrypt)하느라 CPU를 낭비할 필요 없이 목적지만 보고 즉시 패킷을 패스(Pass) 시켜 버리는 예술적인 MAVLink 분산 처리 효율성을 획득한다.

4. PX4 암호화 코프로세서 (Hardware Crypto Coprocessor) 통합

순수 소프트웨어(Software-only)만으로 갈루아 필드의 행렬 곱셈 연산을 수행할 경우 임베디드 보드에서는 여전히 오버헤드가 발생한다. PX4 생태계를 겨냥해 출시된 최신의 비행 제어용 MCU(예: STMicroelectronics STM32H7, 픽스호크 6 시리즈 등)에는 이 AES-GCM 연산을 마이크로초 단위 하드웨어 레이어에서 전담하는 **크립토 코프로세서(Crypto Coprocessor)**가 실리콘 칩 레벨에서 각인되어 있다.
NuttX 커널과 PX4 mavlink_secure 모듈은 이 하드웨어 액셀러레이터의 레지스터를 직접 DMA 맵핑하여 펌웨어가 암호화 연산 병목에 발목을 잡혀 EKF 추력 제어를 방해하는 코너 케이스(Corner Case)를 시스템 레벨에서 영구적으로 소멸시켰다.

6. 결론 (Conclusion)

보안(Security)과 속도(Latency)는 태생적으로 소프트웨어 공학의 안티테제(Antithesis) 관계이나, 분초를 다투는 드론 자율 관제 환경에서는 어느 하나 포기할 수 없는 양쪽 날개이다. PX4-Autopilot가 타겟하고 있는 MAVLink 2.0 AEAD 체계, 즉 고급 암호화 표준의 갈루아 카운터 모드(AES-GCM) 통합은 이 딜레마를 완벽히 종식시킨다.
블록 독립적인 카운터 모드를 통한 딜레이 없는 다중 병렬 암호화, 불필요한 복호화를 배제하게 허락한 부가 데이터(AD) 인증 설계, 그리고 실리콘 코프로세서와의 환상적인 하드웨어 조화는 외부의 도청 침투망을 모두 휴지조각으로 만들어 버리면서도 비행 동역학의 순수성을 전혀 타협하지 않는 궁극의 사이버-물리(Cyber-Physical) 암호 해법을 완성한다.
