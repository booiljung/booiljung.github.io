36.2.4. 다중 GCS 간 제어권 이양(Handover) 최적화를 위한 마르코프 결정 과정(Markov Decision Process, MDP) 천이 모델

1. 개요 (Introduction)
초장거리(BVLOS, Beyond Visual Line of Sight) 비행 임무나 국경을 넘나드는 대규모 물류 배송 시, 단일 지상 관제 시스템(GCS) 안테나의 물리적 전파 도달 범위로는 무인기(UAV)의 전 비행 코스를 커버할 수 없다. 기체가 첫 번째 로컬 관제소(GCS-A)의 통신 커버리지 경계를 벗어날 무렵, 비행 제어권(Control Authority) 및 텔레메트리 릴레이 세션을 즉시 다음 구역의 관제소(GCS-B)로 이양(Handover)해야만 끊김 없는 고신뢰성 임무 수행이 가능하다. 그러나 드론의 이양 타이밍은 시시각각 변하는 무선 채널의 감쇠, 기동 속도, 인접 네트워크 부하 등 수많은 불확실성(Uncertainty)의 지배를 받는다. 본 장에서는 이러한 확률론적(Stochastic) 핸드오버 결정을 최적화하기 위해, 인공지능 강화학습의 수학적 기저인 마르코프 결정 과정(Markov Decision Process, MDP) 천이 모델을 다중 GCS 관제 토폴로지에 적용하는 알고리즘적 접근법을 살펴본다.

2. GCS 제어권 이양(Handover)의 불확실성 요인과 수학적 한계
무인 항공기 이동 통신 네트워크(Aeronautical Ad-hoc Network)에서 MAVLink 세션을 다른 지상국 포트로 스위칭하는 작업은 지상 스마트폰의 셀룰러 핸드오버보다 훨씬 까다롭다.

  • 핑퐁 효과 (Ping-pong Effect): 두 GCS의 경계면 부근에서 전파 세기(RSSI)가 요동칠 경우, 드론의 라우팅 매니저는 GCS-A와 GCS-B 사이에서 제어권 이양(Handover) 핑퐁을 무한 반복하게 되어 치명적인 MAVLink 시퀀스(Sequence) 붕괴와 시스템 부하를 유발한다.
  • 불확실성의 딜레마: 이양(Handover) 시기를 너무 일찍 잡으면 타깃 GCS의 신호가 미약하여 치명적인 패킷 드롭아웃(Packet Dropout)이 생기고, 너무 늦게 잡으면 현재 통신이 먼저 단절되어 비상 모드(Return to Launch, RTL) 발동의 위험에 노출된다.

이러한 결정 딜레마는 과거의 단순한 If-Else 분기문(예: “RSSI가 -90dBm 이하일 때 이양”)만으로는 결코 완벽히 방어될 수 없으며, 미래의 상태 변화 확률까지 고려하는 통계적 최적화 체계의 도입이 요구된다.

3. 마르코프 결정 과정(MDP) 기반 토폴로지 천이 모델 구축
문제 해결을 위해, 제어권 이양 의사 결정 메커니즘을 튜플(Tuple) 구조인 MDP = \langle \mathcal{S}, \mathcal{A}, \mathcal{P}, \mathcal{R}, \gamma \rangle 로 추상화 모델링한다.

  • 상태 공간 (State Space, \mathcal{S}): 기체의 현재 무선 환경 상태 집합이다. GCS-A와 GCS-B로부터 수신하는 MAVLink 패킷 손실률(Loss Rate), 실시간 링크 신호 강도(RSSI), 기체의 비행 벡터(Velocity & Heading), 그리고 현재 제어권을 소유한 GCS의 식별자(SYSID) 등이 벡터 스페이스로 정의된다.
  • 행동 공간 (Action Space, \mathcal{A}): 매 타임 스텝 t 마다 무인기의 네트워크 제어기가 취할 수 있는 선택지로, a_0: “현재 GCS 유지(Stay)”, a_1: “타깃 GCS로 즉각 핸드오버 실행(Handover)“의 이산적 결정 집합이다.
  • 상태 천이 확률 (Transition Probability, \mathcal{P}): 현재 상태 s 에서 행동 a 를 취했을 때 다음 상태 s' 로 전이될 확률 행렬 \mathcal{P}(s'|s,a) 이다. 이는 이동 매질의 레일리 수치(Rayleigh Fading) 등 물리적 무선 채널 모델링을 기반으로 확률론적 추이(Markov Property)를 그린다.
  • 보상 함수 (Reward Function, \mathcal{R}): 안정적인 텔레메트리 대역폭(QoS) 보장 시 양(+)의 보상을 부여하고, 불필요한 핸드오버 반복(Ping-pong) 실행이나 제어 패킷 탈락(Drop) 발생 시 강력한 페널티(음의 보상)를 부과하는 최적화 수학의 목적 함수이다.
  • 할인율 (Discount Factor, \gamma): 미래의 보상을 현재 가치로 치환하는 계수(0 \le \gamma < 1)이다.

4. 벨만 방정식(Bellman Equation)을 통한 최적 이양 정책(Optimal Policy) 도출
MDP 모델이 수립되면 GCS 네트워크 서버 또는 라우팅 데몬은 기체에게 가장 높은 생존율과 대역폭을 보장하는 최적 행동 결정 규칙(Policy, \pi^*)을 산출해 내야 한다.
주어진 상태 s 에서 기체가 얻을 수 있는 장기적인 기대 누적 보상의 최댓값 V^*(s) 는 동적 계획법(Dynamic Programming)의 근간인 다음 벨만 방정식(Bellman Optimality Equation)에 의해 재귀적으로 정의되며 풀린다.

V^*(s) = \max_a \left( \mathcal{R}(s, a) + \gamma \sum_{s'} \mathcal{P}(s' | s, a) \, V^*(s') \right)

이 수식 연산을 (가치 반복 혹은 정책 반복 알고리즘을 통해) 사전에 오프라인 연산하여 테이블 화하거나 강화학습(Q-Learning, DQN) 워커 스레드로 올려둔다면, PX4 모델의 Companion Computer는 현재 무선 환경(RSSI, 패킷 손실률 패턴)을 실시간 룩업(Lookup)하여 가장 수학적으로 안전한 순간에만 타깃 GCS로 연결을 이관(Handover Action)하는 “스마트 핸드오버“를 성취해 낼 수 있다.

5. 다중 GCS MAVLink 라우팅 시스템 내 MDP 통합 아키텍처 (PX4 및 QGroundControl 관점)

  • 동적 SYSID 스위칭 메커니즘:
    현실적인 PX4/MAVSDK 연동 환경에서는 다중 GCS 인프라 구축 시 각 QGC 서버가 고유의 SYSID(예: 255, 254)를 점유한다. MDP 판독 위원회 데몬(Decision Daemon)이 “핸드오버 승인” 상태 행동을 결정하면, 기체의 통신 레이어(MAVLink Router 등)는 기존 GCS로 향하던 다운링크 목적지 포워딩 룰을 즉각 삭제하고 새로운 GCS의 IP 포트와 SYSID로 변경된 타깃 궤적을 갱신하는 핫스왑(Hot-swap) 라우팅을 밀어 넣는다.
  • Ardupilot 생태계의 브로드캐스트적 한계 지양:
    전통적으로 Ardupilot 통신에서는 다중 접속 시 무논리적 브로드캐스팅 네트워크에 의존하여 텔레메트리를 다수의 GCS에 무조건 살포하는 수동적 라우팅 철학에 기대는 경우가 빈번했다. 반면 MDP 기반의 PX4 중심 분산 핸드오버 토폴로지는 확률 모델에 의해 철저하게 계산된 ’단일 독점 제어권’만을 유니캐스트(Unicast)로 보장하게 되므로, 전역 GCS 망의 통신 트래픽 스톰(Storm)을 원천적으로 증발시키며 통제권 탈취나 혼란 사고의 가능성을 100% 논리 차단한다.

6. 결론 (Conclusion)
광역 스케일의 GCS 인프라 네트워크에서 다수의 관제탑(GCS)이 제어권을 넘겨주는 행위는, 드론 생명줄의 스위치를 끄고 켜는 가장 아슬아슬하고 극단적인 무선 네트워크 분기점이다. 고전 임계값 기반(Threshold-based) 정책이 야기하는 빈번한 연결 실패와 이양 핑퐁 무한루프 결함을 해소하기 위해, 미래의 통신 상태 천이와 보상을 추계학적으로 평가하는 마르코프 결정 과정(MDP) 수학 모델의 이식은 군집 제어 관제망 아키텍처에 경이로운 지능적 최적화를 부여한다. 시스템 개발자는 이러한 고도의 비행-통신 결합 강화학습 철학을 관제 백엔드 라우터 데몬과 연계 설계함으로써 궁극적으로 지구 반대편을 넘나드는 드론 에이전트의 끊임없는 장거리 임무 생태계를 완성할 수 있다.
