1. 개요 및 비밀 키 분배(Key Distribution)의 근본적 딜레마 (Introduction)

MAVLink 2.0 메시지 인증 코드(MAC)나 데이터 암호화(Encryption)가 수학적 무결성과 기밀성을 완벽에 가깝게 성취한다 할지라도, 그 전제 조건은 **“GCS와 PX4 드론 양측이 사전에 동일한 대칭 비밀 키(Symmetric Secret Key)를 공유하고 있어야 한다”**는 점이다.

초기 시스템 설정 환경에서 USB 케이블을 통해 유선으로 비밀 키를 주입하는 방식(Pre-Shared Key, PSK)은 안전할 수 있으나, 전장에서 새로운 드론을 예비 편성하거나 다수의 임시 GCS 모바일 단말(태블릿 등)이 네트워크에 동적으로 참여하는 애드혹(Ad-hoc) 무선 환경에서는 통하지 않는다. 누구나 전파를 스니핑(Sniffing)할 수 있는 개방된 열린 무선 채널(RF Channel) 상에서, 어떻게 중간자에게 들키지 않고 양측이 동일한 32바이트 암호 키를 생성 및 공유할 것인가? 이 수학적 딜레마를 해결하기 위해 PX4-Autopilot 보안 아키텍처는 **타원 곡선 계열의 비대칭 암호화 기술(ECC)**을 호출한다.

2. RSA의 한계와 타원 곡선 암호(ECC)의 효율성 도입 (Efficiency of ECC)

비대칭 키(Asymmetric Key, 공개키/개인키 스택) 교환의 대명사인 RSA(Rivest-Shamir-Adleman) 알고리즘은 극히 거대한 소인수 분해의 난해함에 의존하므로, 안전성을 보장하기 위해 통상 2048비트에서 4096비트에 이르는 거대한 키 길이가 요구된다.
문제는 임베디드 실시간 운영체제(NuttX)로 구동되는 드론의 MCU 자원 생태계이다. 배터리가 제한되고 연산 주기가 촉박한 비행 제어 스레드 환경에서, 4096비트의 거대한 RSA 지수승 연산을 1초에도 여러 번 돌리는 것은 심각한 CPU 점유 오버헤드와 비행 지터(Jitter)를 수반한다.

그 대안이 된 것이 바로 **타원 곡선 암호(Elliptic Curve Cryptography, ECC)**이다. ECC는 타원 곡선(y^2 = x^3 + ax + b) 군(Group) 상의 점(Point) 덧셈과 스칼라 곱셈 역전의 어려움(이산 대수 문제, Discrete Logarithm Problem)에 의존한다. ECC는 불과 256비트의 키 길이만으로도 RSA 3072비트와 필적하는 아찔한 보안 강도를 달성해 내며, 이 극단적인 경량성 트레이드오프 덕분에 MAVLink 스택 내부의 오버헤드 없이 실시간 키 교환 로직을 펌웨어 내에 이식하는 것이 가능해졌다.

3. 타원 곡선 디피-헬만(ECDH) 기반의 동적 키 교환 메커니즘 (ECDH Protocol)

PX4와 QGroundControl 간의 MAVLink ECDH 안전한 키 교환 파이프라인은 아래와 같은 극적인 수학적 절차를 거친다. (예: secp256r1와 같은 표준 타원 곡선 사용)

  1. 개인 키 생성: 드론(Node A)과 GCS(Node B)는 각자 내부 난수 생성기를 돌려 자신만이 아는 256비트 스칼라 값, 즉 개인 키(d_A, d_B)를 비밀리에 생성한다.
  2. 공개 키 도출 및 전송: 각 노드는 타원 곡선 상의 공통 베이스 포인트(G)를 자신의 개인 키만큼 스칼라 곱연산(Q = d \times G)하여, 좌표 형태의 공개 키(Q_A, Q_B)를 도출한다. 이후 ఈ 공개 키 Q_A와 Q_B를 암호화되지 않은 라디오 주파수로 브로드캐스트하여 서로 교환한다. (해커가 이 패킷을 훔쳐보아도 타원 곡선의 이산 대수 역산이 불가하므로 개인 키 d를 알아낼 수 없다.)
  3. 공유 비밀(Shared Secret) 동시 도출: 드론은 수신된 GCS의 공개 키(Q_B)에 자신의 개인 키(d_A)를 다시 곱하고(S = d_A \times Q_B = d_A \times d_B \times G), GCS 역시 드론의 공개 키(Q_A)에 자신의 개인 키(d_B)를 곱한다(S = d_B \times Q_A = d_B \times d_A \times G).
  4. 교환 법칙에 의해 양측은 무선 채널을 통해 비밀 키를 직접 전송한 적이 단 한 번도 없음에도 불구하고, 완전히 동일한 타원 곡선 좌표(S)를 동시에 도출하는 마법 같은 결과를 얻게 된다. 이 x좌표 값을 해싱 파이프라인을 거쳐 대칭형 마스터 키(Secret Key)로 격상시킨다.

4. 포워드 시크리시(Forward Secrecy) 및 동적 세션 관리

이렇게 ECDH를 통해 교환된 비밀 키는 비행 임무(Session) 단위를 커버하는 세션 키(Session Key)로만 활용되며, 영구 보존되지 않고 RAM 상에만 등재된 뒤 비행이 끝나면 파기된다.
이를 보안 학계에서는 **순방향 비밀성 혹은 전방향 안전성(Forward Secrecy)**이라 칭한다.
가상의 최악 시나리오로, 며칠 뒤 해커가 드론 하드웨어를 분해하여 롬 칩셋을 추출하더라도, 과거에 비행 제어 및 텔레메트리 암호화에 쓰였던 일회성 세션 키는 이미 증발했으므로 해커는 자신이 며칠 전 가로막고 저장해 둔 과거 비행 암호화 데이터를 영원히 복호화해 낼 수 없다.

6. 결론 (Conclusion)

안전한 연결성이 화두가 되는 복합 무인기 통제 환경에서, 비밀 키를 유선으로 일일이 이식하는 아날로그적 셋업 방식은 작전의 치명적 병목으로 작용한다. PX4 생태계는 통상 대형 IT 인프라에서나 구동되던 비대칭 통신망의 정수, 타원 곡선 암호(ECC)를 비행 제어기 보드의 임베디드 리소스 수준에 걸맞게 압축하고 최적화하여 MAVLink 2.0에 이식하였다.

타원 곡선 디피-헬만(ECDH) 교환 알고리즘은 중간의 적들이 모든 통계를 듣고 있는 개활지 위에서도, 기체와 관제 시스템 단 둘만의 난공불락의 비밀 암호망을 창조해 낸다. 실시간 오버헤드를 줄이기 위한 극단적 256비트 스칼라 최적화와 포워드 시크리시(Forward Secrecy)의 통합은, PX4 소프트웨어 아키텍처가 단순한 항공 역학 제어 엔진 덩어리에서, 현대 암호학(Modern Cryptography)의 정수가 살아 숨 쉬는 하늘 위의 방탄 보안 서버(Secure Server)로 변모했음을 상징하는 강력한 기술적 반증이다.
