36.6.3. 브로드캐스트 스톰(Broadcast Storm) 억제를 위한 그래프 순환 탐지(Cycle Detection) 깊이 우선 탐색(DFS) 알고리즘

1. 개요 및 브로드캐스트 스톰(Broadcast Storm) 방어의 필요성 (Introduction)

단일 지상 관제 시스템(GCS)과 여러 대의 드론이 뒤얽힌 메쉬 네트워크(Mesh Network), 혹은 다중 홉(Multi-Hop) 토폴로지 환경에서 특정 장비가 예기치 않게 다른 장비를 가리키거나 전역 시스템 수신을 의미하는 브로드캐스트 패킷(target_system = 0, target_component = 0)을 라우팅할 때, 최악의 네트워크 재앙이 발생할 수 있다.

이 재앙은 노드 1에서 출발한 패킷이 노드 2, 노드 3을 거쳐 다시 노드 1으로 유입되고, 노드 1은 이를 다시 노드 2로 재조준하여 발사하는 무한 순환 루프(Infinite Routing Loop) 형태를 띤다. 이로 인해 트래픽이 기하급수적으로 증폭되어 드론의 UART 대역폭과 CPU 자원을 1초 이내에 100% 고갈시키는 현상을 **브로드캐스트 스톰(Broadcast Storm)**이라 명명한다. PX4-Autopilot 아키텍처는 유선 스위치망의 스패닝 트리 프로토콜(Spanning Tree Protocol) 철학을 이어받아, 그래프 이론(Graph Theory)에 근거한 순환 탐지(Cycle Detection) 알고리즘을 내부 라우팅 테이블 및 필터링 정책에 융합하였다.

2. 그래프 간선 모델링 및 백-에지(Back-Edge) 현상 논의 (Graph Modeling)

다중 홉 MAVLink 망은 노드(Node, 각 드론/GCS)와 간선(Edge, 라디오 링크)으로 이루어진 공간 무방향 그래프(Spatial Undirected Graph)로 정의할 수 있다. 브로드캐스트 스톰이 일어나려면 이 그래프 내에 반드시 폐루프(Closed Loop), 즉 순환(Cycle)이 형성되어 있어야만 한다.

  • 스웜(Swarm) 비행 중 Node A와 Node B, C가 삼각형 모양으로 링크가 스위칭될 때, 브로드캐스트 속성의 STATUSTEXT 패킷이 A \to B \to C \to A 로 귀환하는 라우팅 백-에지(Back-Edge) 현상이 성립된다.
  • Ardupilot의 중앙 집중형 라우터는 이러한 루프를 단일 객체 기반 락(Lock)으로 관리하는 데 유리하지만, PX4는 분산된 다중 mavlink_main 인스턴스 간 통신을 uORB가 브릿징하므로 개별 노드가 자체적으로 스티어링(Steering) 필터를 수행해야 하는 고도화된 스웜 억제 로직이 요구된다.

3. 깊이 우선 탐색(DFS)을 응용한 순환 탐지(Cycle Detection) 알고리즘

이론적으로 그래프 내 순환을 감지하기 위한 가장 효율적이고 절대적인 수학 모델은 타잔 알고리즘(Tarjan’s Algorithm) 혹은 기본 깊이 우선 탐색(Depth-First Search, DFS)이다. 이 알고리즘은 각 MAVLink 노드를 순회할 때, 논리적 상태를 3가지의 색상(White, Gray, Black)으로 마킹(Coloring)하는 방식을 차용한다.

  1. White (미탐색 노드): MAVLink 네트워크 상에서 아직 현재의 브로드캐스트 데이터 시그니처가 거쳐가지 않은 노드 집합.
  2. Gray (탐색 중 노드 / Call Stack에 잔류): 패킷을 수신하고 현재 하위 노드로 전파를 시도 중인 노드.
  3. Black (탐색 완료 노드): 모든 하위 포워딩 작업이 끝난 노드.

만약 패킷이 포워딩 계층을 거치다가 **현재 Gray 상태로 마킹된 노드를 다시 방문(Visit)**하게 된다면, 이는 그래프 내에 순환(Cycle) 경로가 존재한다는 완벽한 수학적 증빙이 된다. PX4 라우터는 이 순간 즉시 패킷의 포워딩 재귀를 끊고 해당 데이터를 Drop(폐기) 조치하여 스톰의 증폭 스파크를 원천 차단해 버린다.

4. PX4 구현 차원의 중복 방어(Deduplication) 및 시그니처 윈도우

실제 실시간(RTOS) 환경의 극단적 트래픽 속도 하에서 전체 그래프를 DFS로 순회 검증하는 것은 컴퓨팅 비용이 크다. 따라서 PX4-Autopilot은 DFS의 캐싱(Caching) 원리를 시그니처 메모이제이션(Signature Memoization) 체제로 변환하여 사용한다.

  • mavlink_receiver 스레드는 최근 통과한 MAVLink 패킷들의 출발지(System ID), 시퀀스 번호(Sequence Number), 그리고 메시지 ID(Message ID) 해시를 짧은 생명 주기(Short TTL)를 가지는 링 버퍼 윈도우(Sliding Window)에 기록한다.
  • 이는 DFS 방문 배열(Visited Array)의 역할을 대신한다. 새로운 브로드캐스트 패킷이 도착했을 때, 이 시그니처 해시 윈도우 안에 동일한 흔적이 존재한다면(Gray/Black 상태 마주침), 내부의 스톰 방어 상태 머신은 이를 즉각 백-에지 순환으로 오인하여 가차 없이 차단 처리한다.

6. 결론 (Conclusion)

MAVLink 프로토콜의 유연한 확장은 필연적으로 복잡한 네트워크 토폴로지가 뿜어내는 브로드캐스트 스톰이라는 날카로운 양날의 검을 동반하게 되었다. PX4-Autopilot는 이러한 복합 스웜(Swarm) 채널 통신망의 무결성을 보호하고자, 컴퓨터 공학 전산망 이론의 핵심인 깊이 우선 탐색(DFS) 알고리즘과 순환 탐지 이론을 라우터의 핵심 미들웨어 계층에 융합하였다.
이는 각 드론(Node)이 자신이 이 거대한 다중 홉 인프라 속에서 언제 무한 루프의 희생양이 될 수 있는지를 마이크로초(Microsecond) 단위로 자가 진단하게 해 주며, 가장 극단적이고 혼탁한 전파 간섭 환경 아래서도 PX4 편대가 붕괴 없이 통신 대열을 이어나가는 압도적인 자율 보안(Autonomous Security) 면역 쳬계로 자리매김한다.
