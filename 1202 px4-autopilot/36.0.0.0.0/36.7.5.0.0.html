36.7.5. 단조 증가(Monotonic) 카운터와 슬라이딩 윈도우(Sliding Window) 알고리즘을 결합한 재전송 공격(Replay Attack) 방어 상태 머신

1. 개요 및 시공간적 유일성(Uniqueness)의 부재 (Introduction)

앞 절에서 분석한 위협 모델링 중 가장 해소하기 까다로운 적대적 행위는, 암호화된 정상 패킷 전체를 통째로 녹음해 두었다가 나중에 다시 발사하여 드론을 무력화시키는 재전송 공격(Replay Attack)이다.

아무리 AES-GCM으로 내용이 가려져 있고 HMAC-SHA256으로 서명되었다 한들, 패킷 뭉치 자체가 ’수학적으로 완벽한 진본의 복사본’이라면 수신단의 암호화 해독 로직은 이를 정상 패킷으로 판정해 버리는 논리적 모순에 부딪힌다.
이를 영구적으로 단절해 내기 위해 PX4-Autopilot의 MAVLink 2.0 보안 아키텍처는 매 패킷마다 우주에서 단 한 번만 존재하는 **시공간의 일회성(Nonce)**을 부여해야 했고, 그 해결책으로 단조 증가 타임스탬프 카운터와 슬라이딩 윈도우 체계를 상호 결합(Coupling)하여 구현하였다.

2. 단조 증가(Monotonic) 타임스탬프 카운터의 역전 불가성

방해 전파를 이겨내기 위해 MAVLink 2.0 보안 프레임의 헤더에는 48\text{ bits}(6\text{ bytes}) 크기의 거대한 논리적 타임스탬프(Timestamp)가 필수적으로 편입된다.
이 타임스탬프는 절대적인 글로벌 표준 시간(GPS Time 등)일 수도 있지만, 핵심은 오직 앞으로만 전진하며 어떤 예외라도 절대 과거의 값으로 되감기지 않는 단조 증가(Monotonic Non-decreasing) 속성을 엄격하게 지켜야 한다는 점에 있다.

GCS가 “비행 모드 변경” 이라는 똑같은 명령 내용을 백 번 보내더라도, 전송할 때마다 이 단조 증가 카운터 값이 헤더에 새롭게 갱신된다. 그로 인해 패킷 헤더와 페이로드를 통틀어 생성되는 최종 암호학적 서명(MAC) 해시값은 송신할 때마다 눈사태 효과(Avalanche Effect)를 통해 100번 모두 판이하게 다른 값을 지니게 된다. 공격자가 어제 가로챈 패킷을 오늘 재전송하면, 수신 드론은 패킷 내 타임스탬프가 이미 한참 과거의 카운터에 머물러 있음을 식별하고 즉결 사형(Drop) 시켜 버린다.

3. 라디오 딜레이 딜레마와 슬라이딩 윈도우(Sliding Window) 허용 알고리즘

단순히 “내가 방금 전에 받은 타임스탬프보다 값이 커야만 통과시킨다“는 엄격한 무결점 포맷을 임베디드 코어에 덮어씌울 경우, 지독한 부작용인 **절벽 효과(Cliff Effect)**가 연출된다.

실제 무선 주파수망(RF Network)은 전파 간섭이나 다중 홉(Multi-Hop) 라우팅을 거치며 패킷 도착 순서가 뒤집히는 역전 현상(Out-of-Order)이 비일비재하게 발생한다. 가장 따끈따끈한 미래의 타임스탬프 번호(예: T=100)를 수신했다고 해서, 잠시 느린 채널을 타고 뒤늦게 도착한 과거의 합법적 명령 패킷(T=98)을 무조건 Replay 공격으로 간주하고 폐기해 버리면 조종사의 치명적 임무 제어가 손실된다.
이 딜레마를 수학적으로 조율하기 위해 PX4는 수신 모듈에 **슬라이딩 윈도우(Sliding Window)**라는 한시적 묵인 공간을 창조한다.

예컨대 수신기가 인식한 최고 수위 타임스탬프(High Watermark) 기준 반경 수 초 이내의 ‘과거’ 패킷 수신 영역을 합법적 윈도우 범주로 허용해 주는 메커니즘이다.

4. 윈도우 내 중복 검사 비트 필드(Bitfield Map) 상태 머신

윈도우 구간(지연된 근과거 시간대)이라고 해서 무작정 수락할 수는 없으므로, MAVLink 수신기는 윈도우 내 도착 여부를 관리하는 **비트 필드 맵(Bitfield Map)**을 RAM 상에 동적으로 전개한다.

  • 미래(Future) 타임스탬프 유입: 가장 최신의 타임스탬프가 들어오면, 슬라이딩 윈도우의 최고 수위(Watermark)를 앞으로 전진(Slide) 시킨다.
  • 윈도우 내 과거(Windowed Past) 타임스탬프 유입: 윈도우 범위 안쪽으로 늦게 도착한 패킷이라면 비트맵 배열을 조회한다. “이미 0으로 체크된 빈자리인가?“를 확인하여, 한 번도 받지 않은 패킷이면 수락하고 해당 비트를 1로 잠근다. 만약 공격자가 보낸 재전송 패킷이라 이미 비트가 1로 세팅되어 있다면 단호하게 기각(Deny)한다.
  • 윈도우 밖 과거(Outdated Past) 타임스탬프 유입: 슬라이딩 윈도우의 후방 한계선조차 넘지 못할 정도로 심하게 낡은 패킷(예: 어제 녹음해 둔 공격 패킷)은 메모리를 뒤질 가치도 없이 하드웨어 인터럽트 선에서 전파를 버린다.

6. 결론 (Conclusion)

보안 프로토콜 설계에서 시간에 대한 맹신은 독약과 같다. 단조 증가(Monotonic) 서비스타임스탬프가 그리는 곧은 궤적은 재전송 위협 시나리오를 효과적으로 끊어내지만, 변화무쌍한 물리적 RF 라디오 파동의 도착 역행 현상 앞에서는 비행 컨트롤의 단절이라는 한계를 드러냈다.

선형성(Linearity)을 강제하는 타임스탬프와 비선형적 전파 지연을 품어내는 슬라이딩 윈도우(Sliding Window)의 이질적인 조합은, 극도의 편집증적 방어망 속에서도 현실 세계 통신망의 물리적 어그러짐을 유연하게 수용하는 우아한 공학적 해법이다. 이러한 동적 메모리 비트 필드 검증 머신의 투입을 통해, PX4 펌웨어는 해커가 시간을 얼리고 되감아 치명적인 오작동을 주입하려는 모든 사이버 공습 시도를 논리적으로 파괴하는 군사 전략 무기로 진화하였다.
