36.4. 전송 계층(Transport Layer) 혼잡 제어 및 순방향 오류 정정(FEC)

비행 중인 무인항공기(UAV)와 지상 관제 시스템(GCS) 사이의 물리 계층(PHY)이 추계학적 페이딩과 간섭을 견디며 기본 연결을 제공한다면, 그 위에 위치하는 전송 계층(Transport Layer)은 불안정한 무선 링크 위에서 데이터의 흐름을 통제하고 신뢰성을 보장하는 중추적인 역할을 담당한다. 특히 MAVLink 프로토콜을 사용하는 PX4-Autopilot 아키텍처에서는 지연 시간에 민감한 제어 명령과 대용량 원격 측정(Telemetry) 데이터가 혼재되어 전송되므로, 혼잡 제어(Congestion Control)와 순방향 오류 정정(Forward Error Correction, FEC) 기법의 융합 설계가 비행 안전의 핵심이 된다.

1. 전송 계층에서의 무인기 통신 문제 정의

현대의 스마트 드론 통합 시스템(예: ROS2 보조 컴퓨터(Companion Computer)가 탑재된 기체)은 텔레메트리 라디오뿐만 아니라 Wi-Fi, LTE 등 IP 기반의 네트워크를 활용한다. 이 때 지연 시간 최소화를 위해 주로 비연결성 프로토콜인 UDP(User Datagram Protocol)를 사용한다.

그러나 UDP는 자체적인 전송 속도 조절 기능이나 오류 복구 기능이 없다. 만약 GCS(예: QGroundControl)나 온보드 컴퓨터가 물리 계층의 송신 가능 대역폭을 초과하여 고해상도 비디오나 고주파 IMU 센서 데이터를 밀어넣게 되면, 모뎀의 송신 버퍼가 가득 차는 버퍼블로트(Bufferbloat) 현상이 발생한다. 이는 조종사의 긴급 수동 제어(RC Override)나 비상 정지(Kill Switch) 명령의 전달을 지연시켜 치명적인 사고로 이어질 수 있다.

2. 드론 네트워크의 혼잡 제어(Congestion Control)

유선 인터넷에서 사용되는 TCP의 혼잡 제어 알고리즘(예: TCP Reno, CUBIC 등)은 패킷 손실을 혼잡의 결과로 간주하여 송신 속도율(Window Size)을 절반으로 급감시킨다. 그러나 무인기 환경에서는 네트워크 혼잡이 아닌 물리 계층의 페이딩 때문에 패킷이 드롭되는 경우가 대다수이므로, 표준 TCP 혼잡 제어를 그대로 적용하면 통신 효율이 극단적으로 저하되는 한계가 있다.

2.1 PX4 및 GCS단에서의 애플리케이션 기반 흐름 제어

이를 해결하기 위해 PX4-Autopilot 생태계는 전송 계층의 혼잡 제어를 MAVLink 애플리케이션 레벨의 피드백 제어로 대체 내지 보완한다.

  1. 동적 속도 조절(Dynamic Rate Adjustment): QGroundControl(QGC)은 수신되는 RADIO_STATUS 메시지를 통해 현재 무선 링크의 여유 대역폭(Tx Buf 비어있음 정도 등)을 감시한다. 대역폭 포화 징후가 보이면, QGC는 REQUEST_DATA_STREAM 또는 SET_MESSAGE_INTERVAL 메시지를 PX4 제어기에 보내어 비핵심 텔레메트리(예: 배터리 상태, GPS 위성 상세 정보)의 전송 주파수(Hz)를 능동적으로 하향 조정한다.
  2. 보조 컴퓨터에서의 트래픽 셰이핑(Traffic Shaping): ROS2 기반 시스템에서는 Micro XRCE-DDS 에이전트가 토큰 버킷(Token Bucket) 또는 누수 버킷(Leaky Bucket) 알고리즘을 사용하여, 이더넷을 통해 쏟아져 들어오는 대역폭을 무선 모뎀의 한계에 맞추어 평활화(Smoothing)함으로써 하드웨어 버퍼 버스트(Burst)를 방지한다.

3. 순방향 오류 정정(Forward Error Correction, FEC)

비가시권(BVLOS) 비행 등 물리적 수신 감도가 불량한계 근처에 다다르면 패킷 내 일부 비트(Bit)가 깨지는 현상이 속출한다. 일반적인 유선 네트워크의 재전송 기법인 ARQ(Automatic Repeat reQuest)는 수신자가 패킷 오류(NACK)를 보고하고 송신자가 다시 보내주기를 기다려야 한다. 그러나 장거리 무인기 통신의 긴 왕복 지연 시간(RTT) 상황에서 ARQ에 의존하면 실시간 제어 루프가 붕괴된다.

FEC는 송신 측에서 원본 데이터에 수학적인 패리티(Parity) 비트 배열을 덧붙여 전송하는 기법이다. 수신 측은 패킷에 약간의 비트 오류가 발생하더라도, 수신된 중복 데이터를 수학적으로 계산하여 깨진 원본 비트를 스스로 복원(Correction)해 낸다. 즉, 재전송을 요청할 필요가 없으므로 지연 시간(Latency)이 획기적으로 낮아지고 결정론적(Deterministic) 통신이 가능해진다.

3.1 무인기 모뎀 및 미들웨어에서의 FEC 구현

  • 리드-솔로몬 부호(Reed-Solomon Codes): 바이트 단위의 군집 에러(Burst Error) 수정에 강력하다. 과거부터 마이크로웨이브 텔레메트리 장비의 펌웨어 내부 물리/전송 계층에서 가장 널리 쓰였다.
  • LDPC(Low-Density Parity-Check) 및 터보 코드(Turbo Code): 섀논 한계(Shannon Limit)에 근접하는 우수한 기법으로, 최신 고속 광대역 SDR 모뎀 등에 적용되어 노이즈 마진이 거의 없는 극악의 상태에서도 MAVLink 데이터 패킷을 무결하게 살려낸다.

4. 요약

전송 계층의 혼잡 제어와 순방향 오류 정정(FEC) 기능의 통합 설계는 불안정한 물리적 RF 채널을 ’예측 가능한(Predictable) 통신 파이프라인’으로 탈바꿈시킨다. PX4-Autopilot 아키텍트를 설계할 때는, 사용하는 통신 장비가 하드웨어 레벨에서 어떠한 FEC 체계를 지원하는지 확인해야 하며, 동시에 상위 GCS 소프트웨어와 온보드 라우팅 노드 간의 MAVLink 대역폭 혼잡 회피 메커니즘이 적절히 작동하도록 QoS(Quality of Service) 및 전송률 매개변수를 교정해야만 완벽한 BVLOS 임무 신뢰성을 확보할 수 있다.
