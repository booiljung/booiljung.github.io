36.2.1. 다중 기체-관제망의 방향성 비순환 그래프(Directed Acyclic Graph, DAG) 모델링

1. 개요 (Introduction)
다수의 무인기(UAV)가 투입되는 광역 관제 임무나 통신 인프라가 파괴된 재난 지역에서의 탐색 구조 임무에서는 지상 관제 시스템(GCS)과 모든 기체 간의 직접적인 통신(LOS, Line of Sight) 보장이 물리적으로 불가능하다. 이러한 한계를 극복하기 위해 다중 기체 네트워크는 필연적으로 기체들이 서로의 통신 패킷을 중계(Relay)하는 멀티홉(Multi-hop) 통신망 형태로 진화하게 된다. 본 장에서는 이러한 복잡한 다중 기체-관제망 패킷 릴레이 사슬 구조 안에서 발생할 수 있는 치명적인 데이터 무한 루프(Routing Loop) 현상을 수학적으로 모델링하여 규명하고, 이를 방어하기 위한 최전선 통신 아키텍처로서 방향성 비순환 그래프(DAG, Directed Acyclic Graph) 이론을 도입하여 MAVLink 데이터 흐름을 최적화하는 통신 공학적 토폴로지 설계 방안을 논의한다.

2. 다중 무인기 망에서의 라우팅 루프(Routing Loop) 발생 원리
특정 네트워크 노드 세트 \mathcal{V} = \{GCS, UAV_1, UAV_2, UAV_3\} 상에서, GCS가 전방의 UAV_3를 일방향으로 제어하고자 할 때 중간 기체 UAV_1과 UAV_2가 릴레이 중계 역할을 맡는 위상을 가정하자.
만약 UAV_1이 패킷을 수신한 후 다음 라우팅 홉으로 UAV_2를 지정하여 전달헀는데, 이때 UAV_2의 동적 라우팅 프로토콜 테이블이 수렴(Convergence) 문제로 임시 붕괴되거나 물리적 신호 방사 각도의 왜곡으로 인해 타겟 UAV_3를 찾지 못하고 되레 다시 UAV_1에게 패킷을 토스하는 상황(Bounce)이 발생할 수 있다.
결과적으로 해당 MAVLink 패킷은 [UAV_1 \rightarrow UAV_2 \rightarrow UAV_1 \rightarrow \dots] 의 무한 궤도(Cycle)를 돌며 공중에서 좀비(Zombie) 패킷화된다. 이 상태가 지속되면 동일한 C2 명령어 패킷이 통신 채널의 소중한 대역폭을 모두 집어삼키는 브로드캐스트 스톰(Broadcast Storm) 계열의 라우팅 붕괴가 야기된다.

3. 방향성 비순환 그래프(DAG, Directed Acyclic Graph) 모델의 수학적 정의
DAG는 그래프 이론(Graph Theory)에서 유방향 간선(Directed Edge)을 가지면서, 동시에 그 어떤 노드에서도 출발하여 다시 자기 자신으로 되돌아오는 순환 결로(Cycle)를 허용하지 않는 엄격한 위상 구조 체계를 일컫는다.
시스템 \mathcal{G} = (\mathcal{V}, \mathcal{E}) 내부의 임의의 노드 \mathcal{V}_i 에 대하여 시퀀스 \mathcal{V}_i \rightarrow \mathcal{V}_j \rightarrow \dots \rightarrow \mathcal{V}_k 가 존재할 때, 종착 노드 \mathcal{V}_k 에서 최초 노드 \mathcal{V}_i 로 이어지는 간선 e_{ki} 는 존재할 수 없다는 수학 법칙(Acyclic)을 철저히 규정한다. 다중 에이전트 관제망에 이 DAG 구조를 적용할 경우, 패킷의 릴레이 방향성은 논리적으로 완벽한 트리(Tree) 또는 폭포수(Waterfall) 흐름을 타게 되므로 정보의 무한 루프 핑퐁(Ping-pong) 악순환이 원천적으로 증명 불가(방어)해진다.

4. MAVLink 라우팅 내 구조적 DAG 위상 강제화 요소
PX4 생태계 내부의 MAVLink 로직은 애당초 군집 네트워킹을 염두에 두고 강력한 라우팅 보호 규약을 자체 프레이밍에 품고 있다. 이러한 요소들이 훌륭하게 결합되어 논리적인 DAG 위상 통신망을 수립한다.

  • 발신자(System ID) 및 수신자(Target System ID) 주소 체계의 직관성:
    MAVLink 패킷 내부의 포워딩 엔진은 패킷이 거쳐 가는 물리 채널(Serial, UDP 등) 브릿지 역할을 수행할 때, 절대 패킷이 원래 유입된(Input) 인터페이스 채널로 동일 패킷을 무의미하게 재전송(Reflect)하지 않는 ‘스플릿 호라이즌(Split Horizon)’ 원칙을 엄수하여 백워드(Backward) 루프 간선을 잘라낸다.
  • 포워딩 홉(TTL, Time-To-Live)의 부재와 컴포넌트 라우팅 제약:
    흥미롭게도 TCP/IP 세계의 IPv4 헤더에는 패킷의 좀비화를 막기 위한 TTL(Time To Live) 필드가 존재하여 라우터를 거칠 때마다 1씩 차감하지만, 경량화를 극대화한 MAVLink v1/v2 기본 프레임에는 TTL 필드가 존재하지 않는다. 이는 자칫 무한 루프 발생 시 방어벽이 전무해질 수 있다는 치명적 약점이 된다. 따라서 GCS 및 Companion Computer 단의 MAVLink 라우터 프로세스(예: MAVRouter, MAVLink-Routerd)는 라우팅 정책 테이블에서 철저하게 DAG 모델을 사전 정의(Hard-coding 또는 계층적 주도 프로토콜 적용)하여 사이클(Cycle)의 위상적 생성을 코어 단위 알고리즘으로 차단하는 것이 필수불가결하다.

5. 군집 제어 및 GCS(QGroundControl) 활용 위상 최적화 (Topological Optimization)

  • 위상 정렬 (Topological Sorting):
    다수의 릴레이 기체가 활약하는 선형적 임무(예: 산불 라인 예찰을 따라 징검다리 식으로 배치된 드론망) 수행 시, GCS는 비행 전(Pre-flight) 계획 단계나 실행 중 노드 간의 위상을 정렬(Sorting)하여 통신 계층의 부모-자식 관계를 DAG로 형성할 수 있다. 이는 GCS \rightarrow 리더 기체(Leader) \rightarrow 중계 기체(Relay) \rightarrow 위성 기체(Follower) 순으로 엄격한 하향식(Top-Down) 명령 체계를 구축함으로써 상위 노드를 향한 역류성 트래픽 생성을 억제한다.
  • Ardupilot 생태계 대비 간결한 중계 구축:
    전통적 Ardupilot 관제 환경에서는 RFD900 등의 하드웨어 모듈 자체의 비동기 메시(Mesh) 기능에 의존하는 경향이 짙어, 복잡한 통신 오류 발생 시 소프트웨어 단의 위상 추적이 까다로웠다. 반면 최근 PX4-Autopilot 및 ROS2 DDS 브릿지 중심의 현대 관제 연동 프레임에서는 GCS 백엔드의 소프트웨어 라우팅 그래프 맵이 매우 직관적으로 스케줄링되므로, DAG의 이론적 장점을 응용 계층(Application Layer) 소프트웨어에 고스란히 이식하여 네트워크의 결정론적(Deterministic) 통신 시간과 상태 수렴 보장을 획득하기 수월하다.

6. 결론 (Conclusion)
광활하고 통신이 단절된 공역에서 다중 무인기 망이 서로를 매개로 P2P 기반 메시 릴레이 통신을 수행할 때, 무작위한 홉(Hop) 연결은 체계의 통신 루프 대혼란(Broadcast Storm)을 부를 수 있는 양날의 검이다. 데이터 방향성이 정해져 있고 순환 경로가 없는 방향성 비순환 그래프(DAG) 알고리즘을 MAVLink 라우팅 포워딩 철학 모델의 근간으로 강력히 적용하면, GCS의 지휘 통제 패킷과 기체의 텔레메트리 스트림은 한치의 교착 순환 없이 각자의 목표물을 향해 구조적으로 최적의 단방향(Unidirectional Workflow) 강물을 타며 빠르고 안정적으로 이동할 수 있다.
