36.3.1. 샤논-하틀리 정리(Shannon-Hartley Theorem) 기반 고잡음 UAV 환경에서의 최대 채널 용량(Channel Capacity) 도출

무인항공기(UAV)와 지상 관제 시스템(GCS) 사이의 제어 및 원격 전송(Telemetry) 통신은 대역폭이 제한적이고 전기적 잡음이 매우 심한 환경에서 이루어진다. 특히 PX4-Autopilot을 구동하는 하드웨어(예: Pixhawk 시리즈) 주변에는 전자 변속기(ESC), 고전류 배터리 케이블, 강력한 브러시리스 모터 등 전자기 간섭(EMI)을 유발하는 요소들이 밀집해 있다. 본 절에서는 통신 이론의 가장 뼈대가 되는 샤논-하틀리 정리(Shannon-Hartley Theorem)를 바탕으로, 고잡음 드론 환경에서 물리적으로 전송 가능한 최대 채널 용량을 도출하고, 이것이 MAVLink 데이터 스트리밍에 미치는 한계를 학술적으로 조명한다.

1. 샤논-하틀리 정리의 수학적 정의

샤논-하틀리 정리는 열잡음(Thermal Noise)과 무작위적인 백색 잡음(White Noise)이 존재하는 가우시안 채널에서, 오류 없이 전송할 수 있는 최대의 정보 전송률(Maximum Information Rate)인 채널 용량(Channel Capacity)을 제시한다.

이 정리는 다음과 같은 수식으로 표현된다.

C = B \log_2 \left( 1 + \frac{S}{N} \right)

여기서 각 변수의 의미는 다음과 같다.

  • C: 채널 용량 (Channel Capacity, 단위: bps 또는 bits per second)
  • B: 채널의 대역폭 (Bandwidth, 단위: Hz)
  • S: 수신 신호의 평균 전력 (Average Signal Power, 단위: Watt)
  • N: 평균 가우시안 잡음 전력 (Average Noise Power, 단위: Watt)
  • S/N: 신호 대 잡음비 (Signal-to-Noise Ratio, 일반적인 SNR)

이 정리에 따르면 상한선 C를 초과하여 데이터를 전송하고자 할 경우, 어떠한 첨단 오류 정정 부호(Forward Error Correction, FEC)를 사용하더라도 수신 측에서 에러 없는(Error-free) 데이터를 복원하는 것은 수학적으로 불가능하다.

2. 고잡음 UAV 통신 환경의 특수성

일반적인 지상 이동통신이나 Wi-Fi 환경과 비교할 때, PX4 드론 시스템의 무선 통신 환경은 다음과 같은 특수한 제약에 직면한다.

  1. 자체 유발 간섭 (Self-Interference): 비행 제어기(Flight Controller) 인접부의 전원 분배 보드(Power Distribution Board, PDB)와 모터에서 발생하는 스위칭 노이즈는 주로 저주파 대역뿐만 아니라 고주파(RF) 통신 대역까지 광대역 잡음(N)을 증가시킨다.
  2. 안테나 성능 저하: 공기역학적 한계 및 기체 크기 제약으로 인해 고이득(High-gain) 안테나 탑재가 불가능하여 수신 신호 전력(S)이 제한된다. 반사파에 의한 다중 경로 페이딩 현상 탓에 수신 전력의 급격한 딥(Dip)이 수시로 발생한다.
  3. 제한된 주파수 대역폭 (B): 대부분의 장거리 텔레메트리 모듈(예: 433MHz 또는 915MHz 대역의 SiK Radio 호환 모듈)은 ISM(Industrial, Scientific, and Medical) 대역을 파편적으로 사용하므로 할당되는 B가 극히 좁다.

2.1 최대 채널 용량 계산 예시

PX4 기체와 사용하는 915MHz SiK 텔레메트리를 예로 들어보자. 무선 채널의 주어진 대역폭이 B = 100 \text{ kHz}이고, 원거리 비행 시 모터 노이즈 및 감쇠로 인해 수신단에서의 SNR이 10 \text{ dB} (즉, \frac{S}{N} = 10)라고 가정한다.

이 경우 샤논-하틀리 정리에 의한 최대 채널 용량은 다음과 같이 계산된다.

C = 100,000 \times \log_2 (1 + 10) \approx 100,000 \times 3.459 = 345,900 \text{ bps} \ (\approx 346 \text{ kbps})

이 346 \text{ kbps}라는 수치는 어떠한 오버헤드나 물리적 한계를 이상적으로 극복했을 때의 이론적인 한계치이다. 실제 모듈에서는 변조(Modulation) 방식의 한계, 프로토콜 오버헤드, 보호 대역(Guard Band) 등으로 인해 실제 전송률(Throughput)은 이론적 채널 용량의 30\% \sim 50\% 수준에 머무르게 된다.

2. MAVLink 데이터 스케줄링への 영향

앞서 도출한 물리 계층의 최대 채널 용량 한계는 PX4-Autopilot 소프트웨어의 통신 아키텍처, 특히 MAVLink 스트리밍 로직에 직접적인 제약을 부과한다.

QGroundControl(QGC) 등 GCS 소프트웨어는 사용자에게 부드러운 인공 수평의(Attitude Indicator)를 보여주고 실시간 기체 상태를 감시하기 위해 ATTITUDE, HIGHRES_IMU, LOCAL_POSITION_NED 등의 수많은 MAVLink 토픽을 요구한다. 만약 PX4 내부의 MAVLink 모듈 통신 설정(예: 파라미터 MAV_0_RATE)이 물리적 채널 용량 C를 초과하는 전송률을 요구하도록 구성된다면 지상 관제 시스템은 필연적으로 극심한 패킷 드롭(Packet Drop)을 경험하게 된다.

2.1 동적 대역폭 할당 및 비율 제어 전략

이러한 물리적 한계를 극복하기 위하여 PX4-Autopilot은 다음과 같은 기법을 적용한다.

  • 메시지 중요도 기반 QoS 처리: 조종 제어 명령이나 Heartbeat 등 필수 메시지에는 높은 우선순위를 할당하여, 대역폭 포화 시 잔여 노이즈 용량 내 우선 송신되도록 구성한다.
  • 상태 독립적 동적 샘플링: QGC에서는 연결된 통신 링크 특성(USB, WiFi, Radio 등)을 자체적으로 파악하고, 저대역폭 환경일 경우 상태 데이터 수신 빈도(요구 Hz)를 낮추는 방향으로 PX4에 REQUEST_DATA_STREAM 또는 SET_MESSAGE_INTERVAL 메시지를 자동 전송한다.

3. 결론

샤논-하틀리 정리는 고잡음, 저전력 특성을 지니는 무인항공기 원격 제어 통신 환경의 절대적인 물리적 경계선을 정의한다. PX4 기반 시스템 구축에 있어 개발자는 사용하는 트랜시버의 대역폭과 운용 환경의 예측 SNR을 바탕으로 최대 가용 대역폭을 수치적으로 산출(Estimation)해야 한다. 이를 기반으로 MAV_ 파라미터 그룹의 데이터 전송률과 QGroundControl의 텔레메트리 요구 빈도를 최적화하는 전략은 비가시권 장거리 비행(BVLOS)에서 기체 통신 안정성을 담보하는 최우선 과제이다.
