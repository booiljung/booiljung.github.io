36.2.2. 모바일 애드혹 네트워크(MANET) 라우팅: 주도적(Proactive) 프로토콜과 반응적(Reactive) 프로토콜의 위상 수학적 특성 비교

1. 개요 (Introduction)
다수의 PX4-Autopilot 장착 무인기가 사전 인프라 없이 독립적인 군집(Swarm)을 이루어 통신하는 환경은 컴퓨터 네트워킹 이론에서 모바일 애드혹 네트워크(MANET, Mobile Ad-hoc Network)로 고스란히 치환된다. MANET 환경 내 노드(Node)들은 시속 수십 km 이상으로 비행하며 3차원 공간을 끊임없이 이동하므로, 그래프(Graph) 간선의 가중치와 연결 상태가 극심하게 변동하는 시변 위상(Time-Varying Topology)을 지닌다. 지상 관제 시스템(GCS)에서 MAVLink 명령 패킷을 타깃 기체로 릴레이(Relay)하기 위해서는 이러한 역동적 위상 구조를 신속하게 포착하는 라우팅 프로토콜이 필수적이다. 본 장에서는 MANET 통신의 두 가지 핵심 사상인 주도적(Proactive) 프로토콜과 반응적(Reactive) 프로토콜의 수학적 위상 관리 특성을 비교 분석하고, 각각의 철학이 군집 제어 대역폭 및 배터리 자원 효율에 미치는 영향을 학술적으로 조명한다.

2. 주도적(Proactive) 라우팅 프로토콜의 토폴로지 관리 특성
주도적 라우팅 프로토콜(예: OLSR, DSDV)은 네트워크 내의 모든 노드 \mathcal{V}_i 가 자신을 포함한 전체 토폴로지 지도를 항상 실시간으로 유지하려는 사상을 갖는다.

  • 위상 수학적 메커니즘:
    각 드론 기체는 주기적(Periodical)으로 ‘HELLO’ 패킷이나 링크 상태(Link State) 정보(TC 메시지 등)를 브로드캐스팅(Broadcasting)하여 이웃 노드의 생존을 확인한다. 이를 통해 수집된 국소적 위상 변화 데이터는 전체 네트워크로 범람(Flooding) 전파되며, 모든 노드는 벨만-포드(Bellman-Ford) 또는 다익스트라(Dijkstra) 알고리즘을 연산하여 전역 라우팅 테이블(Global Routing Table)을 최신화한다.
  • 통신 제어(GCS) 관점에서의 장단점:
    GCS에서 특정 UAV_x 로 패킷을 송신하고자 할 때, 최적 경로 비용(Cost)이 이미 최신 테이블 내에 연산되어 캐싱되어 있으므로 탐색 지연(Initial Delay)이 0에 수렴한다는 압도적인 장점이 있다.
    그러나 극심한 수학적 페널티가 존재한다. 드론 군집의 기동이 잦거나 노드 수가 N 개로 증가할 경우, 위상 업데이트를 위한 제어 트래픽(Control Overhead) 오버헤드는 \mathcal{O}(N^2) 에 가깝게 비선형 폭증한다. 기체들이 제어 명령보다 경로 최신화 패킷을 주고받는 데 무선 대역폭(Bandwidth)과 프로세서 전력을 모두 탕진해버리는 치명적인 ‘네트워크 마비’ 사태가 빈벌하게 발생할 위험이 내포되어 있다.

3. 반응적(Reactive) 라우팅 프로토콜의 온디맨드(On-Demand) 경로 탐색
반응적 라우팅 프로토콜(예: AODV, DSR)은 테이블을 유지하는 강박을 버리고, 오직 데이터를 전송해야 할 수요(Demand)가 발생한 그 순간에만 탐색 파동을 발산하여 경로 위상을 수립하는 실용주의적 사상을 띤다.

  • 위상 탐색(Route Discovery) 메커니즘:
    GCS(출발지)가 타깃 드론(목적지)으로 패킷 구동을 명하면, GCS 노드는 경로 요청(RREQ, Route Request) 브로드캐스트 패킷을 인접 홉에 터뜨린다. 이 탐색파는 네트워크라는 거미줄(Graph) 위를 퍼져나가 목적지 드론에 도달하며, 목적지는 자신이 받은 최초(최단) 경로를 역추적(Reverse Path)하는 경로 응답(RREP, Route Reply) 패킷을 회신하여 간선 \mathcal{E} 의 경로 궤적을 논리적으로 고정(Locking)시킨다.
  • 제어 오버헤드 \leftrightarrow 시동 지연율(Latency) 트레이드오프:
    평상시에는 위상 관리용 핑(Ping) 트래픽이 전혀 발생하지 않으므로, 제한적인 무인기 편대의 900MHz 텔레메트리 라디오 대역폭을 낭비하지 않는 놀라운 효용성을 자랑한다.
    하지만 거대한 약점 또한 자명하다. 수학적 그래프 탐색 연산이 실제 데이터 전송 ’직전’에 수반되어야 하므로, 최초의 제어 명령 패킷(C2 Command) 세션이 성립하기까지 목적지까지의 물리 통신 거리에 비례하는 시동 지연 시간(Route Discovery Latency) 병목이 가중된다. 긴박한 비상 침로 변경 등을 요하는 하드 리얼타임(Hard Real-time) 제어 명령에서 이 지연은 아찔한 한계점으로 작용할 수 있다.

4. PX4 빌트인 MAVLink 라우터의 태생적 반응성(Reactivity) 체계
PX4-Autopilot 생태계 기반 GCS 환경에서는 거대한 외부 라우팅 데몬(Batman-adv 등)을 OS 레벨에 덧대지 않는 한, 네이티브 MAVLink 통신 스택 자체의 고정 경로 규칙과 미드레벨(Mid-level) 메시 철학을 우선적으로 맞이하게 된다.

  • 온디맨드 헤더 릴레이 구조:
    MAVLink의 포워딩 철학은 본질상 가장 가벼운 형태의 반응적(Reactive) 행동과 테이블 캐싱(Caching)을 묘하게 결합하고 있다. GCS는 패킷 헤더의 target_system 아이디로 노드를 지목하며, 중간 중계 기체(Companion MCU)는 자신이 수신했던 이전 송신자의 SYSID 포트 매핑 기록(역방향 테이블)을 바탕으로 동적 릴레이를 결정한다.
  • 트래픽 기반의 경로 테이블 생명주기 관리:
    PX4의 제어 메시지는 주기적으로 쏘아올리는 Heartbeat 패킷 자체를 일종의 가벼운 Proactive 신호망으로 대체 활용한다. 이는 MAVLink 대역폭을 좀먹는 대규모 전역 라우팅 연산(Bellman-Ford 등)을 과감히 포기(생략)하는 대신, GCS와 기체 간 필수 유지되는 State Machine Heartbeat 스트림을 라우팅 프로토콜의 위상 유지 생명줄(Keep-alive)로 융합하여 설계 복잡도를 최소화하고 오버헤드를 경감시킨 MAVLink 생태계만의 고전적이지만 세련된 통신 우회 모델이다.

5. 결론 (Conclusion)
제한적인 텔레메트리 대역폭과 극심한 배터리/프로세서 제약을 견뎌야 하는 다중 에이전트 GCS 관제 환경において, MANET에 대한 라우팅 설계 도입은 결코 일방향 최적값으로 귀결되지 않는다. 주도적(Proactive) 프로토콜은 최초 제어 지연율 0이라는 달콤한 파라미터를 보장하지만 군집 규모가 커질수록 네트워크 혼잡 폭동을 각오해야 하며, 반응적(Reactive) 프로토콜은 압도적인 대역폭 절약을 약속하지만 초기 지연율이라는 담성을 넘어야 한다. 실전 PX4 시스템 통합 엔지니어는 MAVLink 고유의 Heartbeat 생체 리듬과 온디맨드 릴레이 기술을 근간으로 삼되, 운용 환경의 특이성(예: 정찰형 드론은 Reactive, 호위형 밀집 군집은 국소적 Proactive 블록)에 따라 하이브리드(ZRP 계열) 위상 수학적 라우팅 프로토콜을 통신 미들웨어(ROS2 DDS 브릿지 등) 위에 유연하게 얹어냄으로써 궁극적인 관제 네트워크 건전성(Network Health)을 확보해야 한다.
