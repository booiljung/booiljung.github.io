36.5. 실시간 운영체제(RTOS) 내 통신 스케줄링 및 하드웨어 추상화(HAL)

1. 개요 및 설계 철학 (Introduction and Design Philosophy)

현대의 무인항공기(UAV)와 지상 관제 시스템(Ground Control Station, GCS) 간의 통신은 매우 엄격한 실시간성(Real-time Constraints)을 요구한다. 최신 PX4-Autopilot(v1.14 기준)은 NuttX 실시간 운영체제(RTOS) 위에서 구동되며, 고속 비행 제어 루프와 지연 시간에 민감한 MAVLink 텔레메트리(Telemetry) 스트림을 동시에 처리해야 한다. 이를 달성하기 위해 PX4는 정교한 통신 스케줄링(Communication Scheduling) 모델과 하드웨어 추상화 계층(Hardware Abstraction Layer, HAL)을 접목하여 아키텍처를 구성한다.

하드웨어 추상화(HAL)는 다양한 마이크로컨트롤러(MCU) 및 프로세서 아키텍처(예: STM32, NXP 등)의 레지스터(Register) 및 주변장치(Peripherals) 제어 코드를 표준화된 인터페이스로 감싸는 계층이다. 스케줄링 기법과 HAL이 융합됨에 따라, PX4 개발자는 하드웨어의 미시적인 차이에 얽매이지 않고 고차원의 통신 로직(예: 동적 대역폭 할당 및 우선순위 큐 모델링)에 집중할 수 있다.

2. NuttX RTOS 기반 통신 스케줄링(Communication Scheduling) 아키텍처

PX4 내부의 mavlink 모듈은 비행 제어(Flight Control) 스레드와 통신 스레드를 분리하여 구동하는 데 중점을 둔다. 수많은 uORB(Micro Object Request Broker) 토픽을 직렬화(Serialization)하여 UART 하드웨어 버퍼에 밀어 넣어야 하기 때문에, 스레드의 우선순위 설정이 시스템 전반의 안정성에 지대한 영향을 미친다.

2.1 워크 큐(Work Queue) 와 통신 스레드 분리

NuttX는 고성능 다중 스레딩을 지원하며, PX4는 효율적인 컨텍스트 스위칭(Context Switching)을 위해 워크 큐 기반의 스케줄링을 사용한다. mavlink_main.cpp 소스 코드를 분석해 보면, MAVLink 수신(Receiver) 및 송신(Transmitter) 인스턴스는 별도의 독립된 스레드로 작동한다.

  • mavlink_rcv (수신 스레드): GCS(예: QGroundControl v4.3)로부터 들어오는 명령(Command)과 심장박동(Heartbeat) 패킷을 파싱한다. 가장 빠른 반응을 위해 인터럽트 서비스 루틴(ISR) 직후에 데이터를 읽어 들인다.
  • mavlink_tx (송신 스레드): uORB 토픽의 변경 사항을 구독(Subscribe)하고 스트리밍 주기에 맞추어 버퍼를 할당한 뒤 UART를 통해 GCS로 전송한다.

2.2 비율 단조 스케줄링(Rate Monotonic Scheduling, RMS)의 응용

통신 태스크가 물리적인 제어 루프(예: px4io 또는 muorb 등)의 CPU 자원을 점유하는 것을 막기 위해, PX4는 우선순위 역전(Priority Inversion) 방지를 고려하여 스레드 우선순위를 할당한다. 통신 스레드는 자세 제어(Attitude Control)나 EKF(Extended Kalman Filter) 태스크에 비해 상대적으로 낮은 스케줄링 우선순위(Low Priority)를 부여받는다. 그러나 MAVLink 메시지의 중요도(예: 비상 정지 명령)에 따라 내부 큐(Queue)의 우선순위를 동적으로 인터리빙(Interleaving)하는 스케줄링 기법이 적용된다.

3. 통신 하드웨어 추상화 계층(HAL)과 DMA 기반 비동기 I/O

비행 제어기의 두뇌에 해당하는 CPU는 통신 패킷 송수신을 처리하면서도 자세 제어 루프를 방해해서는 안 된다. 이를 해결하기 위해 HAL은 직접 메모리 접근(Direct Memory Access, DMA)을 기반으로 I/O를 추상화한다.

3.1 분산-수집(Scatter-Gather) DMA 트랜잭션

NuttX의 직렬 포트 드라이버(예: stm32_serial.c)는 시스템 버스의 부하를 줄이기 위해 DMA 인터페이스와 깊게 결합되어 있다.
블록 다이어그램으로 나타낸 하드웨어 추상화 구조는 다음과 같다:

3.2 블로킹(Blocking) 대비 비동기 논블로킹(Non-Blocking) 모델링

통신 태스크가 GCS와의 연결 품질 저하(예: RF 모뎀의 하드웨어 FIFO 가득 참 상황)로 인해 블로킹되는 것을 방지하기 위해 파일 디스크립터(File Descriptor)는 O_NONBLOCK 플래그로 관리된다. poll() 또는 select() 함수 호출을 추상화하여, 하드웨어가 데이터를 수용할 준비가 되었을 때에만 CPU를 할당하여 패킷 큐를 방출(Flush)한다.

4. Ardupilot (ChibiOS) 대비 PX4 (NuttX) 의 아키텍처 차이점 비교 (Architectural Differences against Ardupilot)

Ardupilot 통신 스택은 전통적으로 HAL 계층(AP_HAL) 내에 통신 드라이버를 집중시켜 놓았다. Ardupilot의 AP_HAL_ChibiOS 계층은 하드웨어 인터럽트와 시리얼 버퍼 관리를 강력하게 결합하여 직관성을 높인 반면, 플랫폼 독립성에 일부 희생을 따랐다.

반대로 PX4는 엄격한 **POSIX 컴플라이언스(POSIX Compliance)**를 지향한다. NuttX가 제공하는 VFS(Virtual File System) 모델(예: /dev/ttyS1)을 그대로 사용하여, 하드웨어 장치를 리눅스 환경과 동일하게 파일처럼 다룬다.

  • Ardupilot (AP_HAL): C++ 기반의 강결합 다형성(Polymorphism)을 활용하여 AP_HAL::UARTDriver 인터페이스를 제공한다.
  • PX4 (NuttX HAL): C 기반의 운영체제 표준 POSIX API (open, read, write, ioctl)를 준수하며 하위 단의 BSP(Board Support Package)가 HAL 역할을 전담한다.
    이러한 차이 덕분에 PX4 시스템은 디버깅 시 리눅스와 유사한 쉘(Shell) 명령어(예: NSH 환경에서의 cat /dev/ttyS0 모니터링)를 활용할 수 있는 강력한 유지보수 이점을 제공한다.

5. QGroundControl (QGC) 와의 스트리밍 대역폭 제어 (Bandwidth Control)

실제 임무 비행 중에는 RF 통신의 대역폭 한계로 인하여 다량의 MAVLink 트래픽이 손실(Drop)될 위험이 존재한다. 이를 제어하기 위해 시스템은 다음과 같은 스케줄링 튜닝 파라미터를 제공한다.

  • SER_TEL1_BAUD: 통신 하드웨어의 보드레이트(Baudrate)를 결정짓는 파라미터를 의미한다. (예: 57600 bps)
  • MAV_1_RATE: 해당 시리얼 포트 전체의 최대 허용 대역폭(Bytes per Second) 제한값 지정 파라미터이다.

QGroundControl(v4.3 기준)은 접속 직후 REQUEST_DATA_STREAM 메시지를 PX4-Autopilot로 송신하여 동적 주파수 할당을 요청한다. 이 명령을 수신한 PX4의 MAVLink 모듈은 내부 스케줄러(Scheduler)를 깨워 발행할 각 메시지 간격(Interval)을 재계산한다. 이 과정에서 버퍼 오버런(Buffer Overrun)을 방지하기 위해 토큰 버킷(Token Bucket) 기반의 병목 회피 트래픽 셰이핑(Traffic Shaping) 알고리즘이 일부 적용되며, 만일 CPU 사용률이 비정상적으로 높아지면 고주파수(예: IMU 50Hz) 데이터 스트림을 자동으로 억제하여 시스템 안정성을 확보한다.

6. 결론 (Conclusion)

PX4-Autopilot에서의 RTOS 기반 통신 스케줄링과 HAL 아키텍처는 유연한 모듈화와 실시간 제어의 타협점(Trade-off)을 절묘하게 포착한 뛰어난 설계 패턴이다. NuttX VFS와 DMA의 조화를 통해 데이터 버스와 프로세서 간의 경합을 최소화하며, GCS로 향하는 MAVLink 메시지를 시스템의 부하 상황에 맞추어 지능적으로 분배한다. 이는 결과적으로 예기치 못한 네트워크 환경 변화나 하드웨어 장애 상황에서도 드론의 코어 제어 루프를 항상 절대적으로 보호하는 훌륭한 안전망(Failsafe Foundation)을 제공한다.
