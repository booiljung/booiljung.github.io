Chapter 12. GPS 및 글로벌 포지셔닝 시스템 연동

무인항공기(UAV)의 신뢰성 있는 자율 비행을 가능케 하는 가장 핵심적인 기술은 기체의 절대적인 지구 좌표를 파악하는 글로벌 포지셔닝 시스템(Global Positioning System) 연동이다. PX4-Autopilot은 단순한 위치 수신을 넘어, 다양한 위성 항법 시스템(GNSS: Global Navigation Satellite System)으로부터 유입되는 다중 데이터를 정밀하게 처리하고, 이를 내부 상태 추정기(Estimator)와 결합하여 고도의 위치 제어 능력을 구현한다.

본 장에서는 PX4-Autopilot 환경에서의 GPS 운용 원리부터 하위 드라이버 아키텍처, 그리고 최신 RTK(Real-Time Kinematic) 기술의 적용에 이르기까지 위성 측위 시스템 전반을 심도 있게 분석한다.

1. 개요 및 주요 특성

PX4는 GPS, GLONASS, Galileo, BeiDou 등 현대 무인기 측위에 사용되는 대부분의 위성 시스템을 지원한다. 특히 PX4의 설계 철학은 센서 데이터의 ’무결성(Integrity)’과 ’리던던시(Redundancy)’를 최우선으로 하며, 이는 Ardupilot과 비교했을 때 더욱 모듈화된 프로세스로 나타난다.

1.1 위치 추정 아키텍처의 핵심

PX4 내에서 GPS 데이터는 독립적인 드라이버 모듈인 src/drivers/gps를 통해 수집된다. 수집된 데이터는 uORB 메시징 버스를 통해 전파되며, 최종적으로 EKF2(Extended Kalman Filter) 알고리즘에 의해 가속도계(Accelerometer), 자이로스코프(Gyroscope), 기압계(Barometer) 데이터와 융합된다.

1.2 PX4와 Ardupilot의 위성 항법 제어 철학 차이

Ardupilot이 AP_GPS 라이브러리를 통해 라이브러리 수준의 폴링 워커(Polling Worker) 방식을 선호한다면, PX4-Autopilot은 NuttX RTOS 상에서 별도의 독립 프로세스(Process) 혹은 워크 큐(Work Queue) 타스크로 GPS 드라이버를 실행한다. 이러한 아키텍처적 차이는 다중 GPS(Dual GNSS) 구성 시 개별 센서 타임아웃 처리에 대한 유연성을 제공하며, CPU 스케줄링 측면에서 더욱 명확한 우선순위 관리를 가능케 한다.

2. 주요 연동 기술 및 프로토콜

PX4는 단순 위치 정보뿐만 아니라 위성 항법 장치의 기술적 한계를 극복하기 위한 다양한 부가 기능을 포함한다.

  1. 다중 프로토콜 지원: NMEA 0183와 같은 표준 ASCII 프로토콜부터 u-blox UBX, Trimble, Ashtech 등의 고효율 바이너리 프로토콜을 자동 감지(Auto-detection)한다.
  2. RTK(Real-Time Kinematic) 연동: m단위 수준의 오차를 cm단위로 줄이기 위해 QGroundControl과 연동된 RTK 베이스 스테이션 데이터를 MAVLink를 통해 기체로 주입(Injection)한다.
  3. 이중화(Redundancy): 두 개 이상의 GPS 모듈을 장착할 경우, PX4는 각 모듈의 신뢰도(hAcc, vAcc 지표 등)를 실시간으로 평가하여 최적의 데이터를 선택하거나 블렌딩(Blending)한다.

3. 설정 및 구성 가이드 (PX4 v1.14+ 기준)

PX4 파라미터 설정을 통해 GPS 동작을 제어할 수 있다. QGroundControl의 ‘Parameters’ 탭에서 다음 주요 변수들을 확인하라.

  • GPS_1_CONFIG: 첫 번째 GPS 포트의 물리적 연결 규격(Serial, CAN 등)을 설정한다.
  • SENS_GPS_MASK: 다중 GPS 구성 시 주 센서를 선택하거나 데이터 선택 알고리즘을 정의한다.
  • EKF2_GPS_POS_X/Y/Z: 기체의 무게 중심(CG)으로부터 GPS 안테나 위상 중심까지의 거리(Offset)를 보정하는 레버 암(Lever Arm) 파라미터이다.
