NextCloud Hub 일반 사용자 안내서

1. NextCloud Hub 시작하기

1.1 NextCloud Hub 소개

NextCloud Hub는 단순한 파일 동기화 및 공유 소프트웨어를 넘어, 사용자가 자신의 데이터에 대한 완전한 통제권, 즉 데이터 주권(Data Sovereignty)을 확보할 수 있도록 설계된 통합 협업 플랫폼이다.1 이는 개인이 자택에서 무료 서버를 운영하는 것부터 대규모 기업이 엔터프라이즈 구독을 통해 전문적인 지원을 받는 것에 이르기까지 모든 규모의 사용자에게 적용된다.3

플랫폼의 핵심은 여러 애플리케이션이 유기적으로 통합되어 있다는 점에 있다. 주요 구성 요소는 다음과 같다 5:

  • Files: 안전한 파일 동기화, 공유 및 관리를 위한 핵심 기능이다.

  • Talk: 채팅, 화상 회의, 화면 공유를 통한 실시간 소통 및 협업 도구이다.

  • Groupware: Calendar(캘린더), Contacts(주소록), Mail(메일)을 통합하여 일정 관리와 커뮤니케이션을 지원한다.

  • Office: 웹 브라우저에서 직접 문서, 스프레드시트, 프레젠테이션을 실시간으로 공동 편집할 수 있는 온라인 오피스 스위트이다.

  • Assistant: 로컬 AI 비서로, 콘텐츠 생성, 데이터 관련 질문 답변, 이메일 요약 등 다양한 지능형 작업을 수행한다.5

  • Flow: 반복적인 작업을 자동화하고 비즈니스 프로세스를 최적화하는 워크플로우 자동화 도구이다.5

이러한 구성 요소들은 서로 긴밀하게 연동되어 사용자의 생산성을 극대화한다. 예를 들어, Files에 저장된 문서를 Office에서 공동으로 편집하고, 그 과정에서 Talk를 통해 화상 회의를 진행하며, 관련된 작업은 Deck에서 추적하고, 회의 일정은 Calendar에 등록하는 일련의 작업 흐름이 하나의 플랫폼 안에서 매끄럽게 이루어진다.6

NextCloud의 근본 철학은 ’100% 오픈 소스’와 ’데이터 주권’에 기반한다.6 사용자는 자신의 데이터를 사내 서버(On-Premises)나 신뢰할 수 있는 클라우드 제공업체에 직접 보관할 수 있다. 이를 통해 제3자 서비스로 인한 데이터 유출 위험을 원천적으로 차단하고, 유럽의 GDPR과 같은 엄격한 개인정보 보호 규정을 준수할 수 있다.2

1.2 웹 인터페이스 접속 및 로그인

NextCloud Hub의 모든 기능은 웹 브라우저를 통해 접근할 수 있다. 최상의 사용자 경험을 위해서는 Google Chrome/Chromium, Mozilla Firefox, Apple Safari, Microsoft Edge와 같은 최신 버전의 웹 브라우저 사용이 권장된다.4 특히, NextCloud Talk의 화상 회의 및 화면 공유 기능을 온전히 사용하기 위해서는 Firefox 52 이상 또는 Chrome/Chromium 49 이상의 버전이 필요하다.4 Microsoft Internet Explorer는 지원되지 않는다.4

로그인 절차는 다음과 같다:

  1. 웹 브라우저의 주소창에 관리자로부터 전달받은 NextCloud 서버 URL(예: https://cloud.example.com)을 입력한다.4

  2. 로그인 화면이 나타나면 할당된 사용자 이름과 비밀번호를 입력하고 로그인 버튼을 클릭한다.8

  3. 계정에 2단계 인증(Two-Factor Authentication, 2FA)이 활성화된 경우, 비밀번호를 입력한 후 추가 인증 화면이 나타난다. 스마트폰의 TOTP(시간 기반 일회용 비밀번호) 인증 앱(예: Google Authenticator, Aegis)을 열고, 화면에 표시되는 6자리 인증 코드를 입력하여 로그인을 완료한다.10

1.3 대시보드 이해하기

로그인 후 가장 먼저 접하게 되는 대시보드는 사용자의 하루 업무를 시작하는 개인화된 정보 허브 역할을 한다.12 대시보드는 다양한 위젯을 통해 흩어져 있는 중요 정보를 한눈에 파악할 수 있도록 구성되어 있다.

기본적으로 표시되는 위젯은 서버 관리자의 설정에 따라 다를 수 있으나, 일반적으로 다음과 같은 정보를 제공한다 12:

  • 파일 (Files): 최근에 수정되거나 업로드된 파일을 표시하여 빠르게 접근할 수 있도록 돕는다.

  • 활동 (Activity): 새로운 파일 공유, 댓글, 캘린더 초대, Talk 통화 요청 등 사용자와 관련된 모든 최신 활동 내역을 시간순으로 보여준다.13

  • 캘린더 (Calendar): 예정된 약속이나 회의 일정을 표시하여 중요한 스케줄을 놓치지 않도록 한다.

  • Talk: 최근에 수신된 채팅 메시지나 자신을 언급(@)한 메시지를 보여준다.14

  • 상태 (Status): 현재 사용자의 상태 메시지를 표시하거나 변경할 수 있다.

사용자는 자신의 필요에 맞게 대시보드를 꾸밀 수 있다. 각 위젯의 위치를 드래그 앤 드롭으로 자유롭게 변경할 수 있으며, 배경화면을 원하는 이미지로 설정하여 개인화된 작업 공간을 만들 수 있다.13

1.4 기본 인터페이스 탐색

NextCloud 웹 인터페이스는 일관되고 직관적인 구조를 가지고 있어 사용자가 쉽게 기능을 탐색하고 사용할 수 있도록 설계되었다. 주요 구성 요소는 다음과 같다 8:

  • (1) 앱 선택 메뉴 (Apps Selection Menu): 화면 좌측 상단에 위치한다. 아이콘 형태로 Files, Dashboard, Photos, Activity 등 설치된 모든 핵심 앱이 나열되어 있으며, 아이콘을 클릭하여 해당 앱으로 즉시 이동할 수 있다.

  • (2) 앱 정보 필드 (Apps Information field): 왼쪽 사이드바에 위치하며, 현재 선택된 앱에 특화된 필터와 컨텍스트 메뉴를 제공한다. 예를 들어, Files 앱에서는 ‘모든 파일’, ‘즐겨찾기’, ’나와 공유됨’과 같은 필터를 통해 파일 목록을 빠르게 정렬할 수 있다.9 이 영역은 어떤 앱을 선택했는지에 따라 동적으로 내용이 변경되어, 사용자가 항상 현재 작업에 필요한 도구를 쉽게 찾을 수 있도록 돕는다.

  • (3) 애플리케이션 뷰 (Application View): 인터페이스의 가장 넓은 중앙 영역이다. 이곳에 파일 목록, 캘린더 일정, Talk 대화창 등 선택된 앱의 핵심 콘텐츠가 표시된다.8

  • (4) 탐색 바 (Navigation Bar): 애플리케이션 뷰 바로 위에 위치한다. 파일 시스템의 폴더를 탐색할 때 현재 위치를 ‘경로(breadcrumbs)’ 형태로 보여주어 상위 폴더로 쉽게 이동할 수 있게 한다.9

  • (6) 검색 필드 (Search field): 화면 우측 상단의 돋보기 아이콘을 클릭하면 활성화된다. 이 검색창은 현재 보고 있는 앱뿐만 아니라 NextCloud 전체의 데이터를 대상으로 통합 검색을 수행하여 관련 파일, 대화, 일정 등을 찾아준다.9

  • (9) 개인 설정 메뉴 (Settings menu): 화면 우측 상단의 프로필 사진이나 이니셜 아이콘을 클릭하면 드롭다운 메뉴가 열린다. 이 메뉴를 통해 ‘설정’, ‘도움말’, ‘로그아웃’ 등의 기능에 접근할 수 있으며, 데스크톱 및 모바일 클라이언트 다운로드 링크도 제공된다.4

이러한 인터페이스 설계는 NextCloud가 단순한 애플리케이션의 집합이 아닌, 확장 가능한 ’플랫폼’임을 보여준다. 사용자는 새로운 앱을 설치하더라도 기존에 익숙한 ’좌측 컨텍스트, 중앙 콘텐츠, 상단 탐색’이라는 일관된 구조 안에서 직관적으로 기능을 습득하고 사용할 수 있다. 이는 앱 간의 전환 비용을 줄이고 플랫폼 전체의 사용성을 높이는 핵심적인 설계 원칙이다.

2. 파일 관리 및 동기화 (NextCloud Files)

2.1 파일 및 폴더 기본 관리

NextCloud Files는 웹 인터페이스를 통해 직관적으로 파일과 폴더를 관리할 수 있는 환경을 제공한다.

  • 생성 및 업로드:

  • 새로운 파일이나 폴더를 생성하려면, 파일 목록 상단의 탐색 바에 있는 + 새로 만들기 버튼을 클릭한다. 드롭다운 메뉴에서 ‘폴더’, ’텍스트 파일’을 선택하여 생성하거나, ’파일 업로드’를 선택하여 로컬 컴퓨터의 파일을 서버로 전송할 수 있다.4

  • 더 간편한 방법으로, 컴퓨터의 파일 탐색기에서 원하는 파일이나 폴더를 웹 브라우저의 파일 목록 영역으로 직접 끌어다 놓아(드래그 앤 드롭) 업로드할 수 있다. 단, 폴더 전체를 드래그 앤 드롭으로 업로드하는 기능은 Chrome, Chromium 기반 브라우저에서 가장 안정적으로 지원된다.15

  • 파일 제어 및 관리:

  • 각 파일이나 폴더 항목 위에 마우스 커서를 올리면 다양한 관리 아이콘이 나타난다.

  • 오버플로우 메뉴 (...): 파일명 오른쪽에 있는 점 세 개 아이콘을 클릭하면 추가 작업 메뉴가 열린다. 이 메뉴를 통해 ‘이름 바꾸기’, ‘이동 또는 복사’, ‘다운로드’, ‘삭제’ 등의 작업을 수행할 수 있다.17

  • 세부 정보: 오버플로우 메뉴에서 ’세부 정보’를 선택하거나 공유 아이콘 옆의 아이콘을 클릭하면 화면 오른쪽에 상세 정보 사이드바가 나타난다. 이 사이드바에서는 ‘활동’ 로그, ‘댓글’, ‘공유’ 상태, ‘버전’ 이력 등 해당 파일에 대한 모든 메타데이터를 확인하고 관리할 수 있다.10

  • 다운로드:

  • 단일 파일을 다운로드하려면 파일명을 클릭하거나 오버플로우 메뉴에서 ’다운로드’를 선택한다.

  • 여러 개의 파일이나 폴더를 한 번에 다운로드하려면, 각 항목 앞의 체크박스를 선택한다. 그러면 화면 상단에 ‘작업’ 메뉴가 나타나며, 여기서 ’다운로드’를 클릭하면 선택된 모든 항목이 하나의 ZIP 압축 파일로 묶여 다운로드된다. 만약 ‘다운로드’ 버튼이 보이지 않는다면, 서버 관리자가 해당 기능을 비활성화한 것이다.17

2.2 파일 정리 및 검색

수많은 파일을 효율적으로 관리하기 위해 NextCloud는 즐겨찾기, 태그, 강력한 검색 기능을 제공한다.

  • 즐겨찾기: 자주 사용하거나 중요한 파일 및 폴더는 파일명 왼쪽의 별 모양 아이콘을 클릭하여 ’즐겨찾기’로 지정할 수 있다.17 즐겨찾기로 지정된 모든 항목은 왼쪽 사이드바의 ‘즐겨찾기’ 필터를 클릭하여 한 번에 모아볼 수 있다.4

  • 태그 지정: 파일을 체계적으로 분류하기 위해 태그를 사용할 수 있다. 파일의 ‘세부 정보’ 뷰에서 원하는 태그 이름을 입력하고 Enter 키를 누르면 태그가 추가된다. 여러 개의 태그를 지정할 수 있으며, 이 태그들은 시스템 전체 사용자에게 공유된다.17 왼쪽 사이드바의 ‘태그’ 목록에서 특정 태그를 클릭하면 해당 태그가 지정된 모든 파일과 폴더를 필터링하여 볼 수 있다.10

NextCloud에서 ’태그’는 사용자가 파일을 정리하는 단순한 분류표를 넘어선다. 이것은 관리자가 시스템 전반의 데이터 정책을 자동화하는 핵심적인 메커니즘으로 작동한다. 예를 들어, 관리자는 ’기밀’이라는 태그가 붙은 파일은 특정 그룹의 사용자만 접근할 수 있도록 ‘파일 접근 제어(File Access Control)’ 규칙을 설정할 수 있다.20 또한, ‘보관’ 태그가 붙은 파일은 7년이 지나면 자동으로 삭제되도록 ‘보존(Retention)’ 정책을 적용할 수도 있다.21 사용자가 특정 폴더에 파일을 업로드하는 행위는 관리자가 설정한 규칙에 따라 자동으로 특정 태그를 파일에 부여하고, 이 태그는 다시 파일의 접근 권한이나 보존 기한을 결정하는 연쇄적인 워크플로우를 촉발시킨다.23 따라서 사용자는 파일을 태그하는 행위가 단순한 정리를 넘어, 해당 데이터의 보안 등급과 생명 주기를 결정하는 중요한 작업일 수 있음을 인지해야 한다.

  • 통합 검색: 화면 우측 상단의 돋보기 아이콘을 클릭하여 통합 검색 기능을 사용할 수 있다. 이 검색 기능은 파일명뿐만 아니라 파일의 내용(서버에 전체 텍스트 검색 앱이 설치된 경우), 댓글, 태그 등 다양한 정보를 기반으로 검색을 수행한다.13 이를 통해 사용자는 파일의 정확한 위치나 이름을 기억하지 못하더라도 키워드를 통해 원하는 정보를 신속하게 찾을 수 있다.

2.3 파일 공유

NextCloud의 강력한 공유 기능은 협업의 핵심이다. 내부 사용자는 물론 외부 파트너와도 안전하고 유연하게 파일을 공유할 수 있다.

  • 내부 공유:

  • 파일이나 폴더를 내부 사용자와 공유하려면, 해당 항목의 공유 아이콘을 클릭한다.

  • 오른쪽에 나타나는 공유 사이드바의 입력란에 공유할 사용자, 그룹, 또는 서클(Circles)의 이름을 입력하여 선택한다.24

  • 공유 대상을 선택한 후, ‘편집 허용’, ‘다시 공유 허용’, ‘삭제 허용’ 등 세부적인 권한을 체크박스를 통해 제어할 수 있다.24

  • 공개 링크 공유:

  • NextCloud 계정이 없는 외부 사용자와 파일을 공유해야 할 경우, ‘공유 링크’ 기능을 사용한다. 공유 사이드바에서 + 공유 링크 만들기 버튼을 클릭하면 고유한 URL이 생성된다.24

  • 단순히 링크를 생성하는 것을 넘어, 다양한 옵션을 통해 보안 수준과 협업 방식을 정밀하게 제어할 수 있다. 이는 NextCloud가 데이터 보안을 최우선으로 고려하며 협업 기능을 설계했음을 보여주는 중요한 특징이다. 각 옵션의 정확한 기능과 사용 사례를 이해하는 것은 안전한 외부 협업을 위해 필수적이다.

2.4 버전 관리 및 복원

NextCloud는 파일이 수정될 때마다 이전 버전을 자동으로 저장하여, 실수로 내용을 잘못 수정하거나 삭제했을 때 이전 상태로 쉽게 되돌릴 수 있는 버전 관리 기능을 제공한다.4

  • 버전 확인 및 복원:

  • 파일의 ‘세부 정보’ 사이드바에서 ‘버전’ 탭을 선택하면 해당 파일의 모든 변경 이력이 시간 순서대로 나열된다.4

  • 특정 버전의 시간 표시(타임스탬프)를 클릭하면 해당 버전의 파일을 다운로드할 수 있다.

  • 특정 버전을 현재 파일로 되돌리려면, 해당 버전 옆의 되돌리기(원형 화살표) 아이콘을 클릭한다.27

  • 버전 저장 규칙 및 정책:

  • 새로운 버전은 파일에 마지막 변경이 있은 후 2분 이상 경과했을 때 생성된다.4

  • 저장 공간을 효율적으로 사용하기 위해, 오래된 버전은 정해진 패턴에 따라 자동으로 삭제된다. 예를 들어, 첫 24시간 동안은 매시간 버전을 보관하고, 이후 30일까지는 매일, 그 이후에는 매주 하나의 버전을 보관하는 방식으로 오래될수록 보관 빈도가 줄어든다.4

  • 버전 관리에 사용되는 총 저장 공간은 사용자에게 할당된 전체 용량(Quota)의 50%를 초과하지 않도록 자동으로 관리된다. 만약 버전 파일들의 크기가 이 한도를 넘어서면, 가장 오래된 버전부터 순차적으로 삭제된다.27

2.5 휴지통 및 파일 복구

삭제된 파일은 즉시 영구적으로 사라지지 않고, 먼저 휴지통으로 이동하여 사용자가 복구할 기회를 제공한다.29

  • 휴지통 접근 및 파일 복원:

  • 왼쪽 사이드바 하단의 ‘삭제된 파일’ 메뉴를 클릭하면 휴지통으로 이동할 수 있다.29

  • 휴지통 목록에서 복원하고 싶은 파일이나 폴더를 찾아 ‘복원’ 버튼을 클릭하면 원래 위치로 되돌아간다.

  • 파일을 완전히 삭제하고 싶다면 ‘영구 삭제’ 버튼을 클릭한다.29

  • 휴지통 저장 공간 정책:

  • 휴지통에 있는 파일들은 사용자의 저장 공간 할당량(Quota)에 포함되지 않는다.29

  • 휴지통은 사용자에게 남은 여유 공간의 최대 50%까지만 사용하도록 제한된다. 만약 삭제된 파일들의 총량이 이 한도를 초과하면, 가장 오래전에 삭제된 파일부터 자동으로 영구 삭제되어 공간을 확보한다.29

  • 기본적으로 파일은 휴지통에서 30일간 보관되며, 이 기간이 지나면 영구적으로 삭제될 수 있다. 이 보관 기간은 서버 관리자가 변경할 수 있다.29

3. 소통 및 협업 (NextCloud Talk)

3.1 대화 시작하기

NextCloud Talk는 모든 실시간 소통이 ’대화(Conversation)’라는 단위 안에서 이루어지는 구조를 가진다. 대화에는 두 가지 기본 유형이 있다.4

  • 1:1 대화: 특정 사용자 한 명과 진행하는 비공개 대화이다. 이 대화방에는 다른 참여자를 추가하거나 외부 공유 링크를 생성할 수 없다. 새로운 1:1 대화를 시작하려면, Talk 인터페이스의 검색창에서 상대방의 이름을 검색한 후 목록에서 선택하면 즉시 대화방이 생성된다.32

  • 그룹 대화: 세 명 이상의 사용자가 참여하는 대화이다. 그룹 대화는 공개 링크를 생성하여 NextCloud 계정이 없는 외부 손님도 화상 회의에 참여하도록 초대할 수 있다.32

  • 새로운 그룹 대화를 생성하려면, 검색창 옆의 + 버튼을 클릭하고 ’새 대화 만들기’를 선택한다.4

  • 나타나는 설정 창에서 대화의 이름과 설명을 입력하고, 공개 대화 여부 등을 설정한 후, 참여할 사용자들을 추가하여 대화를 생성한다.34

3.2 화상 회의

모든 Talk 대화방은 텍스트 채팅뿐만 아니라 고품질의 화상 회의 기능을 기본적으로 포함한다.

  • 시작 및 참여:

  • 대화방 상단에 있는 ‘통화 시작(Start call)’ 버튼을 클릭하면 즉시 화상 회의가 시작되며, 대화에 참여한 모든 구성원에게 통화 시작 알림이 전송된다.32

  • 다른 사용자가 이미 통화를 시작한 경우, 이 버튼은 ’통화 참여(Join call)’라는 녹색 버튼으로 바뀌며, 이를 클릭하여 진행 중인 회의에 참여할 수 있다.33

  • 통화 중 제어 기능:

  • 마이크 및 비디오: 통화 중 자신의 비디오 화면이나 상단 바에 있는 아이콘을 통해 마이크를 음소거하거나 비디오 송출을 중단할 수 있다. 단축키 M(마이크)과 V(비디오)를 사용하면 더 빠르게 제어할 수 있다.32

  • Push-to-Talk: 스페이스바는 유용한 토글 키로 작동한다. 마이크가 음소거된 상태에서 스페이스바를 누르고 있는 동안 일시적으로 음소거가 해제되어 발언할 수 있으며, 키에서 손을 떼면 다시 음소거된다.33

  • 참가자 관리: 대화방의 중재자(Moderator)는 참가자 목록에서 다른 사용자의 발언 권한을 제어하거나 회의에서 내보낼 수 있다.33

  • 기록: 회의의 효율성을 위해, 중재자는 참가자 목록을 CSV 파일로 다운로드하여 출석 기록 등으로 활용할 수 있다.7

3.3 화면 공유

화상 회의 중 자신의 화면을 다른 참여자들과 공유하여 프레젠테이션이나 데모를 진행할 수 있다.

  • 화면 공유 시작: 통화 중인 자신의 비디오 화면 위에 마우스를 올리면 나타나는 모니터 모양의 ‘화면 공유’ 아이콘을 클릭한다.32

  • 공유 옵션 선택: 화면 공유 아이콘을 클릭하면, 사용 중인 웹 브라우저에 따라 다음과 같은 공유 옵션을 선택할 수 있는 창이 나타난다 32:

  • 전체 화면 (Entire Screen): 자신의 컴퓨터 모니터 화면 전체를 공유한다. 여러 애플리케이션을 전환하며 보여줘야 할 때 유용하다.

  • 애플리케이션 창 (Application Window): 현재 실행 중인 특정 애플리케이션(예: 파워포인트, 포토샵, 코드 에디터)의 창 하나만 선택하여 공유한다. 다른 창은 참여자에게 보이지 않는다.

  • 브라우저 탭 (Browser Tab): 현재 웹 브라우저에서 열려 있는 여러 탭 중 하나만 선택하여 공유한다. 프라이버시를 보호하며 특정 웹 페이지만 보여주고 싶을 때 적합하다.

  • 발표자 화면: 화면을 공유하는 동안에도 발표자의 웹캠 영상은 작은 PIP(Picture-in-Picture) 형태로 참여자들에게 계속 표시되어, 발표자의 표정과 제스처를 함께 전달할 수 있다.38

3.4 대화 중 협업

Talk는 단순한 소통 도구를 넘어, 다른 NextCloud 앱과 연동하여 실시간 협업을 촉진한다.

  • 파일 공유: 컴퓨터의 파일을 대화창으로 드래그 앤 드롭하거나, 입력창 옆의 클립 아이콘을 클릭하여 NextCloud Files에 저장된 파일을 직접 공유할 수 있다. 공유된 파일은 대화에 참여한 모든 구성원(외부 손님 포함)이 즉시 접근하고 다운로드할 수 있다.32

  • 텍스트 서식 및 반응: 메시지 입력 시 마크다운(Markdown) 문법을 사용하여 텍스트를 굵게 하거나 목록을 만드는 등 서식을 적용할 수 있다.14 또한, 이모티콘 선택기를 사용하여 다양한 감정을 표현할 수 있다.32

  • AI 기반 요약: 긴 대화나 회의 내용을 일일이 다시 읽기 어려울 때, NextCloud Assistant의 AI 요약 기능을 사용하면 핵심 논의 사항과 결정 사항을 빠르게 파악할 수 있다.7

  • 실시간 화이트보드: 통화 중에 ‘화이트보드’ 기능을 실행하면, 모든 참여자가 함께 그림을 그리고, 텍스트를 입력하며, 다이어그램을 만드는 등 실시간으로 아이디어를 시각화하고 발전시킬 수 있는 무한한 캔버스가 제공된다.13

4. 그룹웨어 활용 (Calendar, Contacts, Mail)

4.1 캘린더 (Calendar)

NextCloud Calendar는 개인의 일정 관리를 넘어 팀의 협업을 조율하는 중심 허브 역할을 수행한다.

  • 캘린더 관리:

  • 생성 및 편집: 왼쪽 사이드바에서 + 새 캘린더를 클릭하여 ‘업무’, ‘개인’, ‘프로젝트’ 등 용도별로 여러 캘린더를 생성할 수 있다. 각 캘린더의 이름과 색상은 언제든지 변경 가능하다.41

  • 가져오기 및 내보내기: 표준 iCalendar(.ics) 형식을 지원하므로, 다른 캘린더 서비스(Google, Apple 등)의 일정을 파일로 가져오거나, NextCloud 캘린더를 파일로 내보내 백업할 수 있다.42

  • 이벤트(일정) 관리:

  • 생성: 캘린더의 날짜나 시간대를 마우스로 클릭하거나 드래그하는 것만으로 간단하게 새 일정을 생성할 수 있다. 이후 일정 제목, 시간, 장소, 상세 설명 등을 입력한다.41

  • 참석자 초대: ‘참석자’ 필드에 동료의 이름이나 이메일 주소를 입력하여 회의에 초대할 수 있다. 초대된 참석자는 이메일로 알림을 받고 참석 여부를 회신하며, 이 응답은 캘린더에 자동으로 반영된다.42

  • 반복 일정: 매주 반복되는 팀 회의나 매월 진행되는 보고 등, 규칙적인 일정을 ‘반복’ 옵션을 통해 손쉽게 설정할 수 있다.41

  • 공유 및 구독:

  • 캘린더 공유: 자신의 캘린더를 특정 사용자나 그룹과 공유할 수 있다. 공유 시 ‘편집 가능’ 권한을 부여하면, 공유받은 사람도 해당 캘린더에 새로운 일정을 추가하거나 기존 일정을 수정할 수 있다. ’읽기 전용’으로 공유하면 일정 확인만 가능하다.4

  • 외부 캘린더 구독: 다른 서비스에서 제공하는 공개 캘린더의 iCal 구독 주소(URL)를 사용하여 자신의 NextCloud 캘린더에 추가할 수 있다. 구독한 캘린더의 내용은 주기적으로 자동 동기화되어 항상 최신 상태를 유지한다.41

NextCloud Calendar는 단순한 스케줄러가 아니다. 이것은 팀 협업의 시작점이다. 예를 들어, 사용자가 ’프로젝트 기획 회의’라는 새 일정을 생성하면서, 관련 팀원들을 참석자로 초대하고 동시에 ‘Talk 대화 추가’ 기능을 통해 해당 회의를 위한 전용 온라인 회의실을 즉시 생성할 수 있다.44 이처럼 일정 수립(Calendar)이 곧바로 소통 채널 확보(Talk)와 자료 공유 준비(Files)로 이어지는 통합된 워크플로우를 제공함으로써, 사용자는 여러 도구를 오갈 필요 없이 NextCloud 내에서 효율적으로 협업을 시작할 수 있다.

4.2 주소록 (Contacts) 및 메일 (Mail)

  • 주소록 (Contacts): 개인 및 업무 연락처를 체계적으로 관리하는 공간이다. 주소록에 등록된 정보는 NextCloud의 다른 앱과 긴밀하게 연동된다. 예를 들어, Calendar에서 일정을 만들 때 참석자 이름을 입력하면 주소록에서 자동으로 추천해주고, Mail에서 메일을 작성할 때도 수신자 정보를 쉽게 불러올 수 있어 편리하다.21

  • 메일 (Mail): 사용자의 기존 이메일 계정(IMAP/SMTP)을 연동하여 NextCloud 인터페이스 내에서 직접 이메일을 확인하고, 작성하며, 관리할 수 있는 웹메일 클라이언트이다.5 이메일에 첨부된 파일을 클릭 한 번으로 NextCloud Files의 특정 폴더에 저장하거나, 반대로 Files에 있는 대용량 파일을 첨부하는 대신 안전한 공유 링크로 변환하여 발송하는 등 Files 앱과의 유기적인 통합이 큰 장점이다.7

5. 생산성 도구 (Deck, Notes, Office)

5.1 프로젝트 관리 (Deck)

NextCloud Deck은 칸반(Kanban) 보드 방식으로 프로젝트와 작업을 시각적으로 관리할 수 있는 강력한 도구이다.21

  • 보드, 스택, 카드 구조:

  • 보드(Board): 하나의 프로젝트 또는 관리 단위를 의미한다. + 보드 추가를 클릭하여 새로운 보드를 생성하고, 프로젝트 팀원들과 공유하여 협업 공간으로 사용한다.46

  • 스택(Stack): 보드 내의 작업 단계를 나타내는 세로 열이다. 일반적으로 ‘할 일(To-Do)’, ‘진행 중(In Progress)’, ’완료(Done)’와 같은 스택을 만들어 작업의 흐름을 관리한다.46

  • 카드(Card): 개별적인 작업 항목을 의미한다. 각 스택 하단의 + 버튼을 눌러 새로운 카드를 추가하고, 작업이 진행됨에 따라 카드를 다음 스택으로 드래그 앤 드롭하여 상태를 업데이트한다.46

  • 카드 상세 관리:

  • 생성된 카드를 클릭하면 오른쪽에 상세 관리 패널이 나타나며, 다음과 같은 세부 정보를 추가하고 관리할 수 있다 46:

  • 담당자 할당: 카드(작업)를 특정 팀원에게 할당한다. 할당된 사용자는 알림을 받게 된다.

  • 마감일 설정: 작업의 마감 기한을 지정한다.

  • 태그(라벨): ‘긴급’, ‘디자인’, ‘개발’ 등 미리 설정된 태그를 카드에 부착하여 작업을 분류하고 필터링한다.

  • 첨부파일: 작업과 관련된 문서, 이미지 등 NextCloud Files에 있는 파일을 직접 첨부한다.

  • 설명 및 댓글: 마크다운 서식을 지원하는 설명란에 작업의 상세 내용을 기록하고, 댓글 기능을 통해 팀원들과 의견을 나눌 수 있다.

5.2 노트 필기 (Notes)

NextCloud Notes는 군더더기 없는 깔끔한 인터페이스를 통해 사용자가 생각과 아이디어를 빠르게 기록하고 정리할 수 있도록 돕는 디지털 노트 앱이다.21 웹 인터페이스에서 작성된 모든 노트는 마크다운 형식으로 서버에 자동 저장 및 동기화되어, 데스크톱, 스마트폰 등 어떤 기기에서든 동일한 내용에 접근할 수 있다.49

5.3 온라인 오피스 (Office)

NextCloud Office는 Collabora Online 또는 ONLYOFFICE 기술을 기반으로 하는 웹 기반 오피스 스위트로, 별도의 프로그램을 설치할 필요 없이 브라우저 내에서 문서 작업을 할 수 있게 해준다.5

  • 문서 생성 및 편집:

  • Files 앱에서 + 새로 만들기 버튼을 클릭하면 나타나는 메뉴에서 ‘문서(.docx)’, ‘스프레드시트(.xlsx)’, ’프레젠테이션(.pptx)’을 선택하여 새로운 오피스 파일을 생성할 수 있다.39

  • Microsoft Office 형식(DOCX, XLSX, PPTX)과의 높은 호환성을 제공하여, 외부와 문서를 주고받을 때 발생할 수 있는 서식 깨짐 문제를 최소화한다.39

  • 실시간 공동 편집:

  • NextCloud Office의 가장 강력한 기능은 실시간 공동 편집이다. 여러 명의 사용자가 동시에 같은 문서에 접속하여 함께 작업할 수 있다.39

  • 실시간 커서 추적: 각 사용자의 편집 커서가 고유한 색상으로 표시되어, 누가 어느 부분을 수정하고 있는지 직관적으로 파악할 수 있다.39

  • 댓글 및 토론: 문서의 특정 텍스트나 개체를 선택하여 댓글을 남기고, 답글을 통해 의견을 교환하며 문서를 완성해 나갈 수 있다.6

  • 통합 커뮤니케이션: 문서 편집 중 궁금한 점이 생기면, 편집 화면을 벗어나지 않고 사이드바에서 바로 Talk 채팅이나 화상 회의를 시작하여 동료와 즉시 소통할 수 있다.6

6. 데스크톱 및 모바일 클라이언트

6.1 데스크톱 클라이언트

NextCloud 데스크톱 클라이언트는 로컬 컴퓨터의 파일과 서버를 자동으로 동기화하여, 인터넷 연결 여부와 관계없이 항상 최신 파일에 접근할 수 있도록 해주는 필수적인 도구이다.

  • 설치:

  • Windows, macOS, Linux 운영체제에 맞는 클라이언트를 NextCloud 공식 웹사이트의 다운로드 페이지에서 내려받는다.53

  • 다운로드한 설치 파일을 실행하고, 설치 마법사의 지시에 따라 프로그램을 설치한다.54

  • 계정 연결 및 동기화 설정:

  1. 설치 후 클라이언트를 처음 실행하면 계정 설정 마법사가 나타난다. ‘로그인’ 버튼을 클릭한다.55

  2. ‘서버 주소’ 입력란에 자신의 NextCloud 서버 URL(예: https://cloud.example.com)을 입력한다.54

  3. ’다음’을 클릭하면 웹 브라우저가 자동으로 열리며 NextCloud 로그인 페이지가 나타난다. 사용자 이름과 비밀번호를 입력하여 로그인하고, 데스크톱 클라이언트의 접근을 허용해준다.55

  4. 다시 클라이언트 설정 창으로 돌아오면, 동기화 옵션을 설정하는 화면이 나타난다. 여기서 서버의 모든 파일을 로컬 컴퓨터와 동기화할지, 아니면 특정 폴더만 선택하여 동기화할지 결정할 수 있다.55

  5. ‘로컬 폴더’ 경로를 지정한다. 기본적으로 사용자 홈 디렉토리 아래에 ‘Nextcloud’ 폴더가 생성된다.

  6. ‘연결’ 버튼을 누르면 초기 동기화가 시작된다.54 이후 이 폴더에 파일을 추가하거나 수정하면 자동으로 서버에 반영되며, 서버의 변경 사항 또한 이 폴더에 자동으로 반영된다.

6.2 모바일 앱

Android 및 iOS용 모바일 앱을 통해 이동 중에도 파일에 접근하고, 사진을 업로드하며, 동료와 협업할 수 있다.

  • 설치 및 로그인:

  • Android 사용자는 Google Play Store 또는 F-Droid에서, iOS 사용자는 Apple App Store에서 ‘Nextcloud’ 앱을 검색하여 설치한다.4

  • 앱을 실행한 후, 데스크톱 클라이언트와 마찬가지로 서버 주소와 계정 정보를 입력하여 로그인한다.4

  • 주요 기능:

  • 파일 관리: 서버에 저장된 모든 파일과 폴더를 탐색하고, 파일을 열어보거나 다운로드, 업로드, 삭제하는 등 기본적인 파일 관리가 가능하다.13

  • 자동 업로드: 스마트폰으로 촬영한 사진이나 동영상을 자동으로 NextCloud 서버의 지정된 폴더로 업로드하도록 설정할 수 있다. 이는 스마트폰 분실 시에도 소중한 데이터를 안전하게 보관하는 효과적인 방법이다.16

  • 오프라인 접근: 중요한 파일이나 폴더를 ’오프라인에서 사용 가능’으로 설정해두면, 해당 파일이 스마트폰에 다운로드되어 인터넷이 연결되지 않은 환경에서도 열어볼 수 있다.13

  • 공유: 모바일 환경에서도 손쉽게 다른 사용자에게 파일을 공유하거나, 외부 공유를 위한 공개 링크를 생성하고 전송할 수 있다.13

  • 주소록 및 캘린더 동기화 (Android 기준):

  • NextCloud의 주소록과 캘린더를 스마트폰의 기본 앱과 연동하려면, 별도의 동기화 앱인 DAVx⁵를 설치해야 한다.56

  • 먼저 Google Play Store 또는 F-Droid에서 DAVx⁵를 설치한다.

  • 그 다음, NextCloud 모바일 앱을 열고 ‘설정’ 메뉴로 이동하여 ‘캘린더 및 주소록 동기화’ 항목을 탭한다.57

  • 이 과정을 통해 DAVx⁵ 앱이 자동으로 실행되고 NextCloud 계정 정보가 연동된다. DAVx⁵에 필요한 권한을 부여하고 동기화할 주소록과 캘린더를 선택하면 설정이 완료된다. 이후로는 스마트폰의 기본 주소록 앱이나 캘린더 앱에서 NextCloud 데이터를 확인하고 편집할 수 있다.57

7. 개인 설정 및 보안 강화

7.1 프로필 관리

사용자는 자신의 프로필 정보를 직접 관리하고 정보의 공개 범위를 설정할 수 있다.

  • 정보 변경:

  • 화면 우측 상단의 프로필 아이콘을 클릭한 후 드롭다운 메뉴에서 ’설정’을 선택하면 개인 설정 페이지로 이동한다.4

  • 이 페이지에서 ‘전체 이름’, ‘이메일 주소’, ‘프로필 사진’ 등 개인 정보를 수정할 수 있다. 단, 시스템에서 사용자를 식별하는 고유한 ID인 ’사용자 이름(Login Name)’은 변경할 수 없다.4

  • 프로필 공개 범위 설정:

  • 각 정보 항목(전화번호, 주소, 웹사이트 등) 옆에는 자물쇠 아이콘이 있으며, 이를 클릭하여 정보의 공개 범위를 세밀하게 조정할 수 있다.60

  • Private(비공개): 본인만 볼 수 있다.

  • Local(내부): 동일 NextCloud 서버에 로그인한 사용자들만 볼 수 있다.

  • Federated(연합): 내부 사용자 및 관리자가 신뢰하도록 설정한 다른 NextCloud 서버의 사용자도 볼 수 있다.

  • Published(공개): 인터넷에 연결된 누구나 볼 수 있다.61

7.2 인터페이스 설정

  • 언어 설정: 개인 설정 페이지에서 NextCloud 인터페이스에 표시될 언어를 선택할 수 있다. 관리자가 특정 언어를 강제로 설정하지 않은 경우, 사용자는 자신이 가장 편한 언어로 변경할 수 있다.4

  • 로캘(Locale) 설정: 날짜, 시간, 숫자 형식을 결정하는 로캘을 설정할 수 있다. 이는 보통 언어 설정에 따라 자동으로 지정되지만, ’미국식 날짜(월/일/년)’와 ’유럽식 날짜(일/월/년)’처럼 필요에 따라 수동으로 변경할 수 있다.63

7.3 보안 설정

NextCloud는 사용자가 자신의 계정을 안전하게 보호할 수 있도록 강력한 보안 기능을 제공한다.

  • 비밀번호 변경:

  • ‘설정’ 메뉴의 왼쪽 탐색 창에서 ’보안’을 클릭한다.65

  • ‘비밀번호’ 섹션에서 현재 사용 중인 비밀번호를 입력한 후, 새로 사용할 비밀번호를 두 번 입력하여 변경을 완료한다.65

  • 2단계 인증(Two-Factor Authentication, 2FA) 설정:

  • 2FA는 비밀번호 외에 추가적인 인증 단계를 거치게 함으로써 계정 보안을 대폭 강화하는 기능이다.11

  • 설정 절차:

  1. ‘설정 > 보안’ 페이지의 ‘2단계 인증’ 섹션으로 이동한다.

  2. 관리자가 활성화한 인증 방법 중 ’TOTP(시간 기반 일회용 비밀번호)’를 활성화한다.68

  3. 화면에 QR 코드와 비밀 키가 표시된다. 스마트폰에 설치된 인증 앱(Aegis, Google Authenticator 등)을 사용하여 QR 코드를 스캔하거나 비밀 키를 직접 입력한다.11

  4. 인증 앱에 NextCloud 계정이 추가되고 30초마다 갱신되는 6자리 코드가 생성된다. 이 코드를 웹 페이지의 확인란에 입력하여 설정을 완료한다.68

  • 백업 코드의 중요성:

  • 2FA 설정 과정에서 가장 중요한 단계 중 하나는 백업 코드를 생성하고 안전하게 보관하는 것이다.11

  • 만약 스마트폰을 분실하거나 교체하여 인증 앱에 접근할 수 없게 되면, 이 백업 코드가 계정에 다시 로그인할 수 있는 유일한 방법이 된다.

  • ‘백업 코드 생성’ 버튼을 클릭하면 여러 개의 일회용 코드가 제공된다. 이 코드들을 인쇄하여 집이나 사무실 등 스마트폰과 물리적으로 분리된 안전한 장소에 보관해야 한다. 각 코드는 한 번만 사용할 수 있다.10

2단계 인증을 활성화하는 것은 단순히 웹 로그인 절차만 변경하는 것이 아니다. 이는 NextCloud 생태계 전반의 인증 방식에 영향을 미친다. 2FA가 활성화되면, 데스크톱 및 모바일 클라이언트와 같이 2FA 로그인 흐름을 직접 지원하지 않는 애플리케이션들은 더 이상 기존의 사용자 비밀번호만으로는 서버에 접속할 수 없게 된다.11 이 문제를 해결하기 위해 사용자는 ‘설정 > 보안 > 기기 및 세션’ 메뉴에서 각 클라이언트 애플리케이션을 위한 전용 ’앱 비밀번호’를 생성해야 한다.4 따라서, 2FA를 활성화한 후에는 반드시 자신이 사용하는 모든 클라이언트(데스크톱, 모바일, WebDAV 등)의 비밀번호를 새로 생성된 앱 비밀번호로 교체하는 후속 작업이 필요하다. 이 점을 인지하지 못하면, 2FA 활성화 후 갑자기 파일 동기화가 중단되는 문제를 겪을 수 있다.
