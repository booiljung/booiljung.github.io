Chapter 13. 분산 데이터 컴퓨팅: Zenoh-Flow

지금까지 우리는 데이터를 ‘A에서 B로 보내는 기술’, 즉 Zenoh의 본질적인 통신(Routing) 기능에 집착해왔다.
하지만 현대 시스템 아키텍처는 단순한 택배 기사를 원하지 않는다. 방대한 4K 라이다(LiDAR) 영상과 수천 개의 공장 센서 진동수, 그리고 각 에지(Edge) 로봇에서 터져 나오는 기하급수적인 데이터는 목적지로 배달되기 전에 필연적으로 정제되고, 필터링되며, 인공지능(AI) 모델에 의해 추론(Inference)되는 **‘가공의 과정(Computing)’**을 거쳐야만 그 가치를 발휘한다.

만약 이 무거운 모든 데이터의 가공을 전부 무식하게 중앙 클라우드 AWS로 빨아들여서 처리한다면?
대역폭은 찢어질 것이고, 클라우드 요금은 여러분을 파산시킬 것이며, 자율주행 자동차는 지연 시간 탓에 절벽으로 떨어질 것이다.

이 치명적인 병목을 부수기 위해 탄생한 것이 바로 Zenoh-Flow다.
Zenoh-Flow는 데이터가 흘러가는 네트워크의 중간중간, 즉 라우터와 로봇의 두뇌 속 빈 공간에 **‘연산 노드(Computing Node)’**를 강제로 심어 넣는다. 데이터가 흘러가는 길목 자체에서 처리(Processing)가 일어나게 만드는 경이로운 분산형 데이터플로우(Dataflow) 컴퓨팅 프레임워크의 세계로 들어갈 시간이다.
