13.2.1 Zenoh-Flow 전역 및 로컬 아키텍처 모델

Zenoh-Flow 의 아키텍처는 “현미경으로 보느냐, 망원경으로 보느냐” 에 따라 구조가 완전히 다르게 작동한다.

0.1 [인스펙션] 하이브리드 토폴로지의 이중성

1. 로컬 배관망 (단일 머신 내에서의 폭주)
만약 Source, Operator, Sink 가 “기초 C 컴파일 랩” 의 하나의 PC 나 라즈베리 파이 화면 안에서 돈다면, Zenoh-Flow 런타임은 네트워크 계층(TCP 소켓)을 아예 깨우지 않는다. 대신 메모리 스레드 간의 락-프리 큐(Lock-free Queue) 와 링 버퍼(Ring Buffer) 만으로 데이터를 초당 수만 개 단위로 넘긴다. 외부에서 스니핑조차 불가능한 0 오버헤드의 블랙박스 화가 진행된다.

2. 전역 배관망 (클러스터 단위의 스케일 아웃)
당신이 YAML 파이프라인 파일에서 node: Operator 의 위치 속성(mapping) 을 AWS 서버로 바꾸는 순간 마법이 일어난다.
로컬에서 잘 돌던 컨베이어 벨트에 투명한 게이트웨이(Zenoh Router) 가 생성되고, 데이터는 UDP 멀티캐스트 혹은 TCP 를 타고 바다를 건너가 AWS 의 수신 컨베이어 벨트에 안착한다.
개발자가 소켓 프로그래밍, 직렬화, 커넥션 풀을 1줄도 짤 필요 없이, 그저 프레임워크가 아키텍처 크기에 맞춰 “가장 최적화된 통신 모드” 로 자동 기어 변속을 해준다.
