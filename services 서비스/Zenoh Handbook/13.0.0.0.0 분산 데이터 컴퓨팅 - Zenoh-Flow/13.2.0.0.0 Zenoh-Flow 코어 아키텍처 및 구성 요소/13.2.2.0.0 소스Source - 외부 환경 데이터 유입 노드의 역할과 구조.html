13.2.2 소스(Source) - 외부 환경 데이터 유입 노드의 역할과 구조

모든 데이터의 강은 여기서 발원한다. 소스는 파이프라인 안쪽에 있는 노드이지만, 그 눈은 바깥세상(External World)을 향해 있는 가장 중요한 외교관이다.

0.1 [인스펙션] 발원지(Spring) 설계 원칙

1. 폴링(Polling) 대 이벤트-드리븐 (Event-driven)

  • 소스 노드는 크게 두 가지 양태로 동작한다.
  • 타이머를 시곗바늘처럼 달아놓고 1초마다 카메라 드라이버 베어메탈 계층에서 프레임을 긁어오는 주기적 소스(Periodic Source).
  • MQTT 나 외부 디버거 핑이 떨어질 때만 인터럽트처럼 깨어나 벨트에 데이터를 올리는 이벤트 소스(Reactive Source).

2. 백프레셔(Backpressure) 와의 싸움
만들어낸 소스 노드가 초당 100장의 이미지를 컨베이어 벨트에 밀어 넣는데, 다음 처리 노드(Operator) 가 1지능당 10장밖에 처리 못한다면? 벨트는 넘치고 터진다.
훌륭한 Zenoh-Flow 소스 엔진은 뒤에서 “잠깐, 큐 꽉 찼어!” 라는 신호(Backpressure) 가 오면 알아서 데이터를 생성 단계에서 바닥에 버리는 드롭 로직(Drop Strategy) 을 반드시 프로그래밍해야 한다.
