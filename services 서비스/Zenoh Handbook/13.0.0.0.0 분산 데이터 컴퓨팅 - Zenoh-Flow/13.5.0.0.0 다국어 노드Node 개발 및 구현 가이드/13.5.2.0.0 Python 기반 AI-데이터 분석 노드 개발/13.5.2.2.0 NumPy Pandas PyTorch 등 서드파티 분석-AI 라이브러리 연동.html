13.5.2.2 NumPy, Pandas, PyTorch 등 서드파티 분석/AI 라이브러리 연동

이미 센서 퓨전 코드를 Pandas 와 NumPy 로 다 짜놨다. 이걸 어떻게 Zenoh-Flow 로 이식하는가?

0.1 [Runbook] 이기종 메모리 타입 변환 전술

1. 메모리 오버헤드 직시하기
Rust 노드가 쏘아 올린 센서 데이터(Vec<u8>) 는 데몬의 공유 메모리를 타고 날아와 Python 노드의 inputs.recv() 로 떨어진다.
이 때 떨어지는 결과물은 Python 기본 객체인 bytes 이다.
문제는 이것을 PyTorch 나 NumPy 가 이해하는 ndarray 텐서로 먹일 때 발생한다.

import numpy as np
import cv2

def process(self, context, inputs, outputs):
    raw_bytes = inputs["camera_in"].recv()
    
    # [경고!] 이 순간 바이트 복사(Memory Copy) 가 발생한다!
    np_arr = np.frombuffer(raw_bytes, dtype=np.uint8)
    
    # 복사된 메모리를 이미지 차원으로 복원
    img = np_arr.reshape((1080, 1920, 3))
    
    # 모델 추론
    bounding_box = self.yolo(img)
    
    # 추론 결과를 다시 JSON/Bytes 로 구워서 파이프로 전송!
    outputs["box_out"].send(bounding_box.tobytes())

파이썬 환경에서는 네이티브 수준의 완벽한 Zero-Copy 가 사실상 불가능하다. Rust(C/C++) 바인딩 계층에서 파이썬 객체를 직렬화/역직렬화해야 하기 때문이다. 아키텍트는 이런 비용을 무릅쓰고서라도 “로직 구현의 압도적인 단축” 을 위해 파이썬 AI 런타임을 투입하는 기회비용을 재는 구조 설계자다.
