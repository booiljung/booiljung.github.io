13.5.2.1 Python API 개요 및 의존성 격리(가상환경) 통합

데몬은 자체적으로 내장된 포터블 파이썬 가상머신(PyO3) 을 통해 당신의 .py 스크립트를 동적 라이브러리로 띄운다.

0.1 [Runbook] 데몬과 Python 의 아키텍처 결합

1. 클래스 기반의 노드 주입
Python 스크립트 작성 시 데몬이 인식할 수 있는 규격화된 클래스와 데코레이터(register) 를 사용해야 한다.

from zenoh_flow import Operator, register

@register
class MyInferenceOp(Operator):
    def __init__(self, context, configuration):
        # YAML 에 적힌 파라미터가 여기서 생성자로 떨어진다!
        model_path = configuration["model_path"]
        self.model = load_model(model_path)
        
    def process(self, context, inputs, outputs):
        # 1. 포트에서 이미지 꺼내기
        img_bytes = inputs["image_in"].recv()
        
        # 2. 추론
        result = self.model.predict(img_bytes)
        
        # 3. 다음 노드로 전송!
        outputs["result_out"].send(result)

2. 지옥의 의존성(venv) 격리 기법
만약 파이프라인 안의 A 노드는 PyTorch 1.1 을 쓰고 B 노드는 PyTorch 2.0 을 쓴다면? 전역 파이썬 런타임 하나로는 무조건 깨진다.
아키텍트는 YAML 데몬 설계도에서 각 노드마다 “별도의 가상환경 창고” 를 강제 지정해야 한다.

operators:
  - id: "yolo_node"
    uri: "file://modules/my_yolo.py"
    # 데몬아, 이 파이썬 코드를 켤 때는 이 가상환경을 통째로 메모리에 로드해!!
    configuration:
      venv_path: "/opt/venv/yolo_env"

파이썬의 가장 큰 적, 의존성 충돌을 아키텍처 레벨에서 차단하는 완벽한 디커플링이다.
