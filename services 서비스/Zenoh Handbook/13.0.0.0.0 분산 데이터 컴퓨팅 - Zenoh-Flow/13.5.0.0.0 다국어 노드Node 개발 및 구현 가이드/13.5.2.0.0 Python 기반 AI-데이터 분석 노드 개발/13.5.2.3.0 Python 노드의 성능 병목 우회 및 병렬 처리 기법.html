13.5.2.3 Python 노드의 성능 병목 우회 및 병렬 처리 기법

우리가 작성한 파이썬 필터(Operator) 는 데몬 안에서 쓰레드로 돈다. 여기서 파이썬 최악의 악몽, GIL (Global Interpreter Lock) 이 깨어난다.

0.1 [인스펙션] 멀티 코어 감옥 탈출 전술

1. 멀티 스레딩의 붕괴
데몬이 아무리 스레드 10개를 열어서 당신의 Python process() 함수 10개를 콜하려 해도, GIL 때문에 파이썬 엔진은 “한 번에 하나씩만 실행해!” 라고 락을 건다. 파이프라인의 엄청난 대역폭이 여기서 멈춰버린다.

2. 아키텍트의 우회로 설계 (Scale-Out)
이 한계를 깨는 방법은 스레드를 늘리는 것이 아니라, 데몬 안에 가상머신 인스턴스(Node) 자체를 늘려버리는 것이다!

YAML 데몬 설계 도면을 뜯어고친다.

operators:
  - id: "ai_infer_1"
    uri: "file://ai_node.py"
  - id: "ai_infer_2" # 복제본!
    uri: "file://ai_node.py"
  - id: "ai_infer_3" # 복제본!
    uri: "file://ai_node.py"

데이터 분배를 담당하는 허브(Hub) 노드에서 들어오는 카메라 프레임을 로드밸런싱 하듯이 ai_infer_1, 2, 3 포트로 차례대로 분배(Round-Robin) 해준다.
이러면 데몬은 파이썬 가상머신 구조체 3개를 완전히 찢어진 메모리 공간 안에 격리 생성하기 때문에 GIL 의 간섭을 피하며 3 코어를 풀로 쓸 수 있다. 코드 한 줄 건드리지 않고 성능을 300% 펌핑시키는 데이터플로우만의 런북이다.
