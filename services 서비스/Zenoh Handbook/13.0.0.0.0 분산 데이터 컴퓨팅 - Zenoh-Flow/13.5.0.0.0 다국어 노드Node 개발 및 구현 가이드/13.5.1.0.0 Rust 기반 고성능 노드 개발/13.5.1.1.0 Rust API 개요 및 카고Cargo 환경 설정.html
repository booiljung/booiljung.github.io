13.5.1.1 Rust API 개요 및 카고(Cargo) 환경 설정

우리는 평범한 실행 파일(.exe) 이 아닌 데몬(Daemon) 에 꽂힐 동적 라이브러리 플러그인(.so / .dll) 을 굽는다.

0.1 [Runbook] 플러그인 전용 빌드 스펙 강제화

1. Cargo.toml 포맷 파괴

[package]
name = "my_rust_operator"
version = "0.1.0"
edition = "2021"

[lib]
# 이거 안 쓰면 데몬이 당신의 코드를 물지 못한다!
crate-type = ["cdylib"] 

[dependencies]
zenoh-flow-nodes = "0.6.0"
async-trait = "0.1" # 비동기 떡칠을 위한 필수품

2. 데몬 진입점(Entrypoint) 뚫기
Rust 코드를 짜기 전에 데몬이 당신의 코드를 인식할 수 있도록 “매크로 도장” 을 찍어야 한다.

use zenoh_flow_nodes::prelude::*;

// 오퍼레이터 구조체
struct MyOp;

// 이 블록이 당신의 클래스를 시스템에 등록해 주는 마법진이다
export_operator!(register_my_op);

fn register_my_op() -> NodeRegistration {
    // 런타임에 YAML 로부터 파라미터를 받아 객체를 튀겨낸다!
    NodeRegistration::Operator(Box::new(|_context, _configuration| {
        async { Ok(Box::new(MyOp) as Box<dyn Operator>) }
    }))
}

여기까지 세팅했다면, 파이프라인의 C/Rust 하이브리드 엔진에서 돌아갈 튼튼한 금속 뼈대가 완성된 것이다.
