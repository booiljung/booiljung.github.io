13.5.1.3 고주파수 데이터를 위한 메모리 제로 카피(Zero-copy) 최적화 기법

1초에 100번씩 들어오는 레이저(LiDAR) 3D 포인트 클라우드 100MB 를 파이프라인에서 A->B->C 필터로 옮길 때마다 매번 메모리 복사(memcpy) 를 뜬다면 RAM 버그가 터진다.

0.1 [Runbook] 레퍼런스 카운팅(RC)을 구사한 포인터 타격 전술

1. Payload 의 정체
당신이 process() 함수에서 건네받는 데이터는 복사된 이진 덤프가 아니다.
이 녀석은 Arc<Vec<u8>> 와 같은 Atomic Reference Counter 이다.

2. 복사 금지 율법 (Do Not Clone The Heap)

// [절망편] 버퍼를 복사해서 메모리를 박살 내는 주범
let my_data = message.payload().clone(); 

// [런북편] 데이터의 포인터 "주소" 만 슬쩍 빌려오는 천재적 C-Style 기법
let my_data_ref = &message.payload()[..];

받아온 바이트 뷰(View) 의 필요한 부분만 오프셋(Offset) 매핑으로 파싱해서 읽어라.
Zenoh 측에서 메모리를 알아서 살려둘 테니, 당신은 데이터를 수정할 생각 말고 “계산의 결과물” 만 새로 할당하여 다음 벨트에 던져주면 된다. 이것이 진정한 불변(Immutable) 상태 기반의 Zero-copy 스트리밍 예술이다.
