17.1.2 Zenoh 시스템 모니터링의 핵심 지표 (KPI)

수천 개의 지표 중에 단 4가지만 살아서 퇴근길에 당신의 눈을 따라와야 한다.

0.1 [Runbook] 골든 시그널 4대 천왕 추출

1. 라우팅 테이블(Routing Table) 엔트리 수

  • Zenoh 는 구독(Sub) 이 생길 때마다 지도를 그린다.
  • 이 지도가 갑자기 10만 개, 100만 개로 튀어오르는 순간(Spike), 시스템은 룩업(Look-up) 딜레이에 빠져 죽는다. 와일드카드 구독의 남발을 감시하는 핵심이다.

2. Dropped Messages (학살당한 패킷)

  • 가장 치명적인 지표. 라우터가 자기가 보낼 능력이 없어서 바닥에 찢어버린 패킷의 숫자다.
  • 이 숫자가 0 이 아니라면, 당신이 설계한 파이프 두께(Bandwidth) 나 큐 리미트(Queue Limit) 가 명백히 실패했다는 소리다. (16.5장 혼잡 제어 튜닝 필요).

3. In/Out Throughput (대역폭 절대량)

  • “이 라우터가 지금 초당 몇 메가바이트(MB/s) 를 들이마시고, 몇 번의 세션(Session) 을 안고 있는가?”
  • 인프라의 한계 용량(Capacity) 을 계산하여, 라우터를 2대로 스케일 아웃(Scale-out) 할 시점을 알려주는 지표다.

4. 락 경합(Lock Contention) / CPU 스파이크

  • OS 레벨의 지표다. 트래픽은 저조한데 CPU가 100% 라면, 이것은 무조건 암호화(TLS) 버그나 코딩 실수(Lock Deadlock)다.
