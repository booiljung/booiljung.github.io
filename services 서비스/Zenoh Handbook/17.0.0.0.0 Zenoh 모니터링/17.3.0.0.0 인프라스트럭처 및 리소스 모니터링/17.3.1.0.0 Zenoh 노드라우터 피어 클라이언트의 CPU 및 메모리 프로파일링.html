17.3.1 Zenoh 노드(라우터, 피어, 클라이언트)의 CPU 및 메모리 프로파일링

데몬 프로세스(zenohd) 가 서버의 피(RAM) 를 얼마나 빨아먹고 있는지 감시하라.

0.1 [Runbook] OS 레벨 시스템 매트릭 추출 전술

1. node_exporter 탑재
서버(에지/클라우드) 의 기본 상태를 Prometheus (17.4장) 로 쏘기 위해 node_exporter 를 데몬으로 올린다.

# 백그라운드에서 CPU, Memory 수치를 수집하여 9100 포트로 열어두는 표준 툴
./node_exporter &

2. 프로세스(PID) 타겟팅
전체 서버의 CPU 도 중요하지만, 우리는 zenohd 가 범인인지 아닌지 명확히 알고 싶다. process-exporter 를 추가로 연동하여 zenohd 이름을 가진 프로세스만의 CPU 커널(Kernel) 모드와 유저(User) 모드 점유율을 찢어서 발라봐야 한다. 만약 CPU 커널 타임 리소스가 유저 타임 리소스를 압도한다면, 그것은 I/O(소켓 버퍼 튜닝) 나 폴링(Polling) 에서 병목 코드가 돌고 있다는 물리적인 적신호다.
