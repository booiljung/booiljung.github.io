17.4.3 브릿지(ROS2, MQTT, DDS) 및 백엔드 플러그인 전용 지표 수집

이기종 통신의 병목은 항상 변환기(Bridge) 와 스토리지(Backend DB) 에서 발생한다. 이 녀석들의 지표를 놓치면 모니터링은 반쪽짜리가 된다.

0.1 [인스펙션] 이단아들의 심장박동 스크래핑

1. ROS2 Bridge (zenoh-bridge-dds) 텔레메트리
브릿지 역시 독립적인 데몬이므로 자체 REST 포트를 열게 패러미터를 준다.

# 브릿지의 관리용 포트를 8080 으로 뚫어버린다.
zenoh-bridge-dds --rest-http-port 8080

이후 http://localhost:8080/@/router/local/metrics 로 접속하면, DDS 진영에서 받아온 트래픽과 Zenoh 로 내보낸 트래픽의 오차가 얼마나 되는지(Loss) 증명하는 지표들이 쏟아진다.

2. 스토리지 백엔드(RocksDB/InfluxDB) 블로킹 추적
라우터 본체의 프로메테우스 로깅 포트(8000) 를 유심히 살펴보면 zenoh_storage_insert_time_seconds 같은 지표가 있다. 당신의 백엔드(DB) 가 하드디스크 I/O 병목으로 인해 쿼리를 버벅거리는지 실시간으로 고발하는 치명적 팩트 데이터다.
