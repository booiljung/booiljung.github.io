17.4.1 Zenoh-Prometheus 익스포터(Exporter) 아키텍처

프로메테우스는 자기가 직접 데이터를 수집하지 않는다. 누군가 웹 포트(HTTP) 로 “/metrics” 라는 페이지를 열어주면, 거길 정기적으로 긁어가는(Scraping) 구조(Pull 기반) 다.

0.1 [인스펙션] 지표 적출구 개방 전술

Zenoh 코어 라우터 데몬은 기본적으로 친절하게도 프로메테우스 익스포터 엔진을 내장하고 있다. 켤 줄만 알면 된다.

1. 라우터 설정(zenohd.json5) 에 웹 서버 점화
라우터의 설정 뇌관에 플러그인을 하나 꽂는다.

// zenohd.json5
plugins: {
  rest: { // REST API 서버 플러그인 기동
    listen: ["http/0.0.0.0:8000"]
  }
}

2. 프로메테우스 노출 확인 (Scrape Target)
라우터를 켜고 웹 브라우저나 curl 로 http://localhost:8000/@/router/local/metrics 에 들어가 보라.
화면에 zenoh_session_count 12, zenoh_messages_dropped_total 0 같은 형태의 프로메테우스 규격 텍스트가 쏟아진다면, Zenoh 가 자신의 심장 박동(지표) 을 외부인에게 HTTP 포트로 무방비하게 뱉어낼 준비를 마친 것이다.
