17.4.2 퍼블리셔-서브스크라이버 통신 메트릭 수집 설정

라우터뿐만 아니라, 당신이 직접 짠 C++/Rust 로봇 프로그램에서도 성능 지표가 나와야 한다. “이 퍼블리셔 객체 1개가 초당 몇 번의 Put 을 호출하고 있는가?”

0.1 [Runbook] 어플리케이션 계층 지표 삽입 런북

1. Zenoh 세션 내부 측정 옵션(Metrics) 켬
개발자가 Zenoh(Rust) 클라이언트를 연결할 때, “내 기록(Metrics)을 남겨줘” 라고 선언해야 한다.

let mut config = zenoh::Config::default();
// 클라이언트 객체 스스로 자기 자신의 Rx/Tx 지표를 수집하게 만든다.
config.insert_json5("metrics/enabled", "true"); 
let session = zenoh::open(config).res().await.unwrap();

2. 커스텀 지표(Custom Metrics) 강제 발행
기본 제공되는 rx_bytes 외에도, 당신의 프로그램의 비즈니스 로직 지표(예: ai_inference_delay) 도 프로메테우스에 태워야 한다. 가장 쉬운 꼼수는 Zenoh 애플리케이션 내에 별도의 prometheus 크레이트(Crate) 와 hyper (웹 서버) 를 하나 파서 /metrics 포트를 임의로 열어버린 뒤 이 값들을 병렬로 노출하는 것이다. (망 전체가 한곳을 가리키는 Pull 기반 철학 유지).
