17.2.1 Admin Space(@/router/)의 이해와 활용

Zenoh 라우터는 일반 사용자의 데이터 토픽(robot/sensor) 외에도, 자기 자신의 내장 장기(상태) 를 숨겨진 은밀한 백도어 토픽 공간에 실시간으로 업데이트하고 있다. 이 공간이 바로 Admin Space (@/*) 다.

0.1 [인스펙션] 라우터 뇌파 투시 전술

라우터의 설정과 상태를 보려면 환경 설정 파일이나 남이 만든 문서를 찾을 게 아니라, 라우터에게 직접 물어보면 된다.

1. @/router 공간의 강제 리딩(Sub)
이 숨겨진 토픽을 구독(Subscribe) 하는 순간, 라우터 안에 어떤 사용자가 접속해 있고 버전이 뭔지 쏟아진다.

# 내 컴퓨터의 로컬 라우터에게 "너의 모든 설정과 상태를 내놔라" 고 명령한다.
z_get "@/**"

2. 런타임 설정의 핫-리로드(Hot-Reload) 주입
이 Admin Space 가 무서운 점은, 읽기(Get) 만 되는 게 아니라 쓰기(Put) 도 지원한다는 점이다. 라우터를 끄지 않고도 런타임에 설정을 바꿀 수 있다.

# 라우터를 재부팅하지 않고, 즉시 백엔드 스토리지 로깅 레벨을 debug로 변경!
z_put "@/router/local/config/logging/level" "debug"

에지(Edge) 인프라에서 라우터를 껐다 켜는 것 자체가 서비스 마비(Outage) 다. Admin Space 의 제어권을 장악하는 자가 무중단(Zero-downtime) 관제의 승리자가 된다.
