17.2.4 라우터 및 피어 연결 상태와 세션 확인

연결된 놈 중 누가 피어(Peer) 이고 누가 클라이언트(Client) 인지, 그리고 이놈이 지난 1분 동안 무슨 개짓거리(Traffic) 를 했는지 낱낱이 파헤친다.

0.1 [Runbook] 세션 텔레메트리(Telemetry) 추출 전술

Admin Space 에는 “현재 이 세션이 얼마나 많은 패킷을 쐈는지“에 대한 엑셀 표가 실시간으로 돌아가고 있다.

1. 세션의 정체와 권한 파악법
라우터에 접속된 세션 목록을 까보면 그놈의 신분이 튀어나온다.

z_get "@/router/local/sessions/*"
# 출력 예시: [Session: xxx, WhatAmI: Client, Locators: tcp/10.0.0.5:45677]

2. 네트워크 도둑(Traffic Hog) 색출
특정 로봇 하나가 라이다 센서의 배칭(Batching) 을 풀어서, 라우터 대역폭 전체를 갉아먹는 트래픽 빌런이 됐다고 치자.

# 각 세션(접속자) 들의 RX(수신), TX(송신) 바이트 카운터를 실시간 덤프
curl http://localhost:8000/@/router/local/sessions/*/metrics

여기서 다른 놈들은 rx_bytes: 1540 인데 혼자 rx_bytes: 9999999999 를 찍고 있는 세션 ID 를 찾아라. 그 ID 에 물린 IP 범인이 바로 당신의 시스템을 지금 마비시키고 있는 좀비(Zombie) 프로세스다.
