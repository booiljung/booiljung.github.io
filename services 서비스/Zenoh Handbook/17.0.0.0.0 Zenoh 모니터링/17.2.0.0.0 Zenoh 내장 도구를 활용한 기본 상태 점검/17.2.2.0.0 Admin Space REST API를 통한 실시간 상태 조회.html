17.2.2 Admin Space REST API를 통한 실시간 상태 조회

Zenoh CLI(z_get) 가 없거나 설치할 수 없는 원격지의 웹 브라우저, 나아가서 중앙 관제탑의 TypeScript 백엔드 서버에서 라우터 상태를 긁어오려면 무거운 Zenoh 라이브러리 연동보다 HTTP 가 훨씬 가볍다.

0.1 [Runbook] HTTP 브리지(REST API) 관제 전술

Zenoh 라우터를 기동할 때 터져 나오는 기본 관리용 REST API 포트(8000번) 를 적극적으로 악용하라.

1. 라우터 설정(Config) 덤프 추출
웹 브라우저 창이나 curl 로 라우터에 질문을 던진다.

# 라우터의 핵심 설정(Admin) 들을 JSON 포맷으로 떨어뜨려라.
curl http://localhost:8000/@/router/local/config

2. 구독 맵(Routing Table) 훔쳐보기
사내 메신저 슬랙(Slack) 의 봇(Bot) 에게 “지금 라우터에 물려있는 토픽 리스트 좀 가져와“라고 명령하기 가장 좋은 아키텍처다.

# 현재 라우터가 중계하고 있는 모든 세션들의 목록을 JSON 으로 뽑아낸다.
curl http://localhost:8000/@/router/local/sessions

이 REST API 자체를 사내 인트라넷 모니터링 페이지(React/Vue) 와 다이렉트로 연동해 버리면, 그것 자체가 훌륭한 1차 경고(Alert) 대시보드로 작동한다.
