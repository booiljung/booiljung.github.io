17.8.1 Prometheus Alertmanager 경고 규칙(Alerting Rules) 작성

프로메테우스 데몬은 바보처럼 데이터만 수집할 뿐, 뇌가 없다. 이 옆에 Alertmanager 라는 사냥개를 붙여주고 언제 짖어야 할지 규칙(Rule) 을 적어 주어야 한다.

0.1 [인스펙션] 한계점 타격(Trigger) 룰 생성 전술

1. 임계치 폭발 알람 (Throughput Limit)
네트워크 대역폭이 1GB 를 초과하려 할 때 짖게 만든다.

# prometheus_rules.yml
groups:
- name: zenoh_network_alerts
  rules:
  - alert: HighPacketDropRate
    expr: rate(zenoh_messages_dropped_total[5m]) > 100
    for: 1m
    labels:
      severity: critical
    annotations:
      summary: "Zenoh Router 5분 평균 패킷 학살 발생!"
      description: "{{ $labels.instance }} 라우터가 초당 100개 이상의 패킷을 버리고 있습니다. 즉시 버퍼 튜닝 바랍니다."

2. 지연 시간(Latency) SLA 위반 타격
99% 의 지연시간(P99 Latency) 이 100ms 를 초과하는 순간 짖게 만든다.

  - alert: LatencySpike
    expr: histogram_quantile(0.99, rate(zenoh_latency_bucket[1m])) > 0.1
    for: 2m
    labels:
      severity: warning

여기서 가장 중요한 속성은 for: 1m 이다. 1초 살짝 튀었다고 알람을 울리면 양치기 소년이 되어 나중엔 아무도 안 본다. 반드시 1분 이상 “지속“될 때만 알람이 울리도록 노이즈 필터링 장치를 우겨 넣어야 한다.
