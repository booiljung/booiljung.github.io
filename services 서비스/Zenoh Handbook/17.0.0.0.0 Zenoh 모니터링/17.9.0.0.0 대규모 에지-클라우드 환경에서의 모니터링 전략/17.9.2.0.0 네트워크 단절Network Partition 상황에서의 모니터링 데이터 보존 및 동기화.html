17.9.2 네트워크 단절(Network Partition) 상황에서의 모니터링 데이터 보존 및 동기화

로봇이 지하 주차장에 들어갔다. 10분 동안 5G 망이 끊어졌다.
일반적인 관제탑 도구들은 “로봇이 보이지 않음(Down)” 이라고만 표기하고 그 시간에 무슨 일이 있었는지 데이터를 그냥 허공에 날려버린다(Loss).

0.1 [Runbook] 데이터 블랙박스(Store and Forward) 전술

사각지대(Offline) 에서의 데이터를 살려내는 에지 옵저버빌리티 기법이다.

1. Remote Write 기반의 밀어내기 (Push)
표준 프로메테우스 규격인 Pull(가져오기) 은 망이 끊기면 무용지물이다.
로봇(Edge) 에 설치된 수집기를 Remote Write 모드로 강제 설정하라.
망이 끊긴 동안 로봇은 수집된 CPU 사용량과 Zenoh 메트릭을 자기 디스크 백업 폴더(WAL - Write Ahead Log) 에 미친 듯이 박아둔다.

2. 재연결 시 역류(Backfill) 전송
10분 뒤, 로봇이 지상으로 올라와 5G 에 재연결된다.
이 순간 로봇의 OTel Collector 나 vmagent (VictoriaMetrics Agent) 가 디스크에 쌓아둔 “끊긴 10분 치의 모니터링 파일” 을 통째로 클라우드로 발사한다. 11분 뒤 관제탑의 그라파나(Grafana) 차트는, 끊겼던 10분 구간의 하얀 이빨(공백) 을 과거 시점에서 완벽하게 메워버리는 사후 복원(Backfilling) 을 달성하게 된다.
