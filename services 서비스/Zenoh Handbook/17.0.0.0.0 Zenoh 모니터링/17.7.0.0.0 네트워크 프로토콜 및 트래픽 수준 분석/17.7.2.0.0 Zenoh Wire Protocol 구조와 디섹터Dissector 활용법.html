17.7.2 Zenoh Wire Protocol 구조와 디섹터(Dissector) 활용법

17.7.1 에서 잡은 PCAP 파일은 Data 부분에 그냥 Payload: 1a 2f 4c... 같은 알아볼 수 없는 쓰레기값만 보여준다. Wireshark 는 HTTP 같은 건 잘 읽지만, 커스텀 규격인 Zenoh 패킷은 해독하지 못하기 때문이다.

0.1 [Runbook] 프로토콜 해독기(Dissector) 장착 전술

1. LUA 디섹터 주입
Zenoh 공식 GitHub 에서 제공하는 zenoh.lua Wireshark 플러그인 파일을 다운로드하여 Wireshark 의 플러그인 폴더에 심는다. (Wireshark 재시작 필요).

2. 1바이트 뼈대(Wire Protocol) 해부
해독기를 키포 Wireshark 에서 아무 패킷이나 하나 눌러보면, Payload 항목이 마법처럼 펼쳐지며 뼈대가 드러난다.

  • Message Type: 이게 Put (데이터 쏘기) 인지 Declare (나 구독할래) 인지 식별된다.
  • Key Expression: 누가 어떤 토픽(예: robot/video) 으로 쏘았는지 평문으로 보인다.
  • Attachment/Timestamp: 이 데이터가 정확히 언제 발생했는지 증거가 나온다.

이 해부학을 통해 특정 토픽(robot/useless/log) 이 1초에 100만 번 보내지면서 전체 7447 대역폭의 90%를 갉아먹는 트래픽의 주범(Hog) 임을 정확하게 색출해 낼 수 있다.
