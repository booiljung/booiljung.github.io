17.7.1 Wireshark를 이용한 Zenoh 프로토콜 패킷 캡처 (PCAP)

공장에서 “로봇이 미쳤어요, 데이터를 가끔 빼먹습니다!” 라고 울부짖을 때 애플리케이션 로그에는 정상이 찍혀있다면? 답은 공기 중으로 쏘아 올린 전파(Wire) 에 있다.

0.1 [인스펙션] 덤프 오디오(Dump Audio) 채취 전술

에지 컴퓨터(로봇) 에 Wireshark UI 를 깔 수는 없다. CLI 환경에서 소리소문없이 패킷을 훔쳐야 한다.

1. 대규모 덤프(PCAP) 커맨드
로봇의 터미널(SSH) 에 들어가서 tcpdump 로 7447 포트의 통신을 낚아챈다.

# eth0 인터페이스에서 라우터(7447 포트) 로 오가는 패킷 1만 개를 잡아 파일로 저장
sudo tcpdump -i eth0 port 7447 -w robot_zenoh_drop_investigation.pcap -c 10000

2. PCAP 파일의 사후 부검
저장된 .pcap 파일을 개발자의 노트북으로 복사(scp) 해 온 뒤 Wireshark 로 연다.
이 단계에서는 단순히 “패킷이 오가느냐” 와 “TCP 레벨의 재전송(Retransmission) 이 있느냐” 만을 확인한다. 만약 검정색과 빨간색으로 범벅된 TCP Retrasmission 바다가 펼쳐진다면, Zenoh 코드가 아니라 로봇의 와이파이(Wi-Fi) 안테나가 고장난 것이다. (하드웨어 교체).
