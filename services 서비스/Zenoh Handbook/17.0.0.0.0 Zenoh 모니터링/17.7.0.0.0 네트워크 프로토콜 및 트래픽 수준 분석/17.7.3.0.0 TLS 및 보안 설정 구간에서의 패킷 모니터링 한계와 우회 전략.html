17.7.3 TLS 및 보안 설정 구간에서의 패킷 모니터링 한계와 우회 전략

문제는, 앞서 15장 규정대로 이 통신에 TLS(암호화) 를 발라버렸다면?
Wireshark 의 디섹터(lua) 따위는 통하지 않는다. 눈앞에 보이는 건 완전한 노이즈 덩어리(Encrypted Application Data) 뿐이다.

0.1 [인스펙션] 크립토 락픽(Lock-pick) 전술

개발 환경에 한해, 이 암호를 까기 위해 마스터 키를 잠시 Wireshark 에 넘겨주는 우회로를 뚫는다.

1. 세션 키 덤프(SSLKEYLOGFILE) 유출
클라이언트 코드나 라우터가 TLS 핸드셰이크를 할 때 생성하는 “세션 비밀키” 를 텍스트 파일로 몰래 빼돌리도록 OS 환경변수를 설정한다.

# Zenoh 구동 시, TLS 암호 해독용 마스터 키를 저 파일에 계속 기록해라!
export SSLKEYLOGFILE=/tmp/zenoh_keys.log
./zenohd

2. Wireshark 복호화 엔진 연동
나중에 PCAP 파일을 Wireshark 에서 열 때, Edit -> Preferences -> Protocols -> TLS 메뉴에서 (Pre)-Master-Secret log filename 항목에 아까 저장한 /tmp/zenoh_keys.log 파일을 물려준다.
적용 즉시 암호화되어 있던 패킷들의 봉인이 풀리면서, 평문이었을 때처럼 Put 명령어와 토픽 문자열이 화면에 쏟아지는 기적을 볼 수 있다. (프로덕션 환경에서는 이 행위 자체가 치명적 보안 위반이므로 즉각 삭제해야 한다).
