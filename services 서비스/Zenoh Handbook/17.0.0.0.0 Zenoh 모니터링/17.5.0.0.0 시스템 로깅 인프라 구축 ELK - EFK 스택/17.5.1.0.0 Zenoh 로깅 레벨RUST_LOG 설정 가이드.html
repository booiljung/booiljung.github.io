17.5.1 Zenoh 로깅 레벨(RUST_LOG) 설정 가이드

Zenoh 코어는 Rust 로 짜여 있다. 그리고 Rust 진영의 가장 강력하고 무지비한 로깅 엔진인 tracing 과 env_logger 규격을 따르고 있다.

0.1 [인스펙션] 노이즈 캔슬링(Noise-canceling) 런북

아무 설정 없이 켜면 화면에 쓸데없는 정보(INFO) 가 너무 많이 나와 디스크를 꽉 채워버린다.

1. RUST_LOG 의 기본 타겟팅
운영체제의 환경 변수(RUST_LOG) 하나만으로 Zenoh 의 입을 열고 닫을 수 있다.

# 프로덕션(Production) 표준 세팅: 전체는 경고(WARN)만 띄우되, zenoh 본체 모듈만 INFO를 띄워라.
export RUST_LOG="warn,zenoh=info"
./zenohd

2. 트러블슈팅을 위한 극단적 엑스레이 (Trace)
개발 중 패킷 하나가 어디서 드랍되는지 찾고 싶을 때는 코어 깊숙한 곳의 네트워크 모듈만 미친 듯이 파고들어야 한다.

# 디버깅 세팅: zenoh의 네트워크(net) 모듈 중 프레이밍(framing) 파트의 모든 바이트 흐름(trace)을 까발려라.
export RUST_LOG="warn,zenoh::net::framing=trace"

이렇게 범위를 좁히지 않고 무작정 RUST_LOG=trace 를 주면 1초 만에 기가바이트(GB) 단위의 로그가 터져 나와 터미널이 마비되고 컴퓨터가 죽어버리니 주의하라.
