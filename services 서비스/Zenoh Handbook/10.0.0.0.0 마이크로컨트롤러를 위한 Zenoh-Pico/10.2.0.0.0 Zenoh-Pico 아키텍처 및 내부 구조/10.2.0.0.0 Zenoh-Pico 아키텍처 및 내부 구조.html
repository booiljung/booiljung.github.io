10.2 Zenoh-Pico 아키텍처 및 내부 구조

어떻게 8KB 짜리 램에서 거대한 멀티캐스트 라우팅 엔진이 돌아가는가? 비결은 “아무것도 하지 않는 것” 이다.

Zenoh-Pico 의 소스코드를 열어보면, 화려한 스레드 스케줄러(Thread Scheduler)도, 데이터 베이스도, 캐시(Cache) 시스템도 없다. 오직 단일 스레드(Single Thread)에서 돌아가는 원시적인 상태 머신(State Machine)과, 바이트를 오른쪽 왼쪽으로 옮기는 연속된 Memcpy 의 향연만이 존재한다.

이 챕터는 극한의 환경에서 동작하는 Zenoh-Pico 내부 톱니바퀴의 구동 원리를 낱낱이 분해하여 보여준다. 이 원리를 제대로 꿰뚫지 못한 채 API만 대충 호출한다면, 당신의 로봇은 통신을 시작하자마자 커널 크래시(Kernel Crash)를 내며 멈춰 설 것이다.
