Chapter 5. Rust에서의 Zenoh

클라우드 네이티브에서 엣지 컴퓨팅, 그리고 로보틱스에 이르기까지 마이크로 단위의 지연 시간(Latency)과 메가바이트 급의 처리량(Throughput)을 동시 다발적으로 확보해야 하는 시대가 도래했다. 이러한 전쟁터에서 가비지 컬렉터(GC)에 의존하여 런타임 스파이크(Spike)를 발생시키는 언어나, 개발자의 실수로 런타임 메모리 에러(Segfault)를 뱉는 언어는 더 이상 환영받지 못한다.

이 장에서는 “안전하면서도 C/C++ 이상의 속도를 낸다“는 철학을 가진 시스템 프로그래밍 언어 Rust를 무기로, 차세대 분산형 미들웨어 프로토콜인 Zenoh를 완벽하게 지휘하는 법을 살펴본다.

단순히 “이 함수를 호출하면 데이터가 전송됩니다“라는 수준의 매뉴얼이 아니다. 비동기 런타임(Tokio) 모델과의 결합, 심장 박동(Liveliness)과 QoS 제어, 메모리 복사 비용을 ’제로(Zero)’로 깎아내는 공유 메모리 전술(Shared Memory), 그리고 GitHub Actions와 연동되는 자동화 통합 테스트망 구축까지. 프로덕션 환경의 백엔드 시스템을 100% 무결점 상태로 타겟 디바이스에 올리기 위한 철저한 실전 배치 가이드(Runbook & Redbook)를 제공한다.
