0.1 [Runbook] 데이터 혈관 조영술(Angiography)

1. Node-to-Node(구간별) 분절 테스트
전체 파이프라인이 100ms 가 걸린다면, 중간 관절들을 끊어보고 테스트한다.

  • 로봇 앱 \leftrightarrow 로봇 라우터 (로컬 루프백): 유도된 지연시간 0.1ms 이내. 여기서 10ms 가 찍힌다면 Zenoh 설정이 잘못된 게 아니라 로봇의 C++ / Python 로직 스레드(Thread) 가 잠자는 것이다(CPU 스케줄링 병목).
  • 로봇 라우터 \leftrightarrow 클라우드 라우터 (WAN): ping 명령어로는 모른다. 프로토콜 단위에서의 통신 지연을 봐야 한다. 대역폭 지연 곱(BDP) 이 터지고 있거나, 기지국 문제다.
  • 클라우드 라우터 \leftrightarrow 스토리지 (Disk I/O): I/O 대기를 봐라(iostat). 데이터베이스가 락(Lock)에 걸려 라우팅이 막혀있다.

2. 블로킹(Blocking) vs 비블로킹(Non-blocking)
네트워크 계층(레이어 3, 4) 은 미친 듯이 빠른데 애플리케이션 계층(레이어 7) 이 미치도록 느린 경우가 90%다.
수신단 콜백(Callback) 함수 안에서 SQL 커밋이나 로그(printf) 를 적지 마라. 콜백 함수가 막혀있는 동안, 소켓 버퍼에 차있는 후속 데이터 패킷들은 영원히 대기 상태에 빠지고 큐가 붕괴한다.
