16.1.3 Zenoh 성능 측정을 위한 벤치마킹 도구(zenoh-bench) 활용

내가 직접 만든 파이썬 수신 스크립트로 성능을 재지 마라. 파이썬의 태생적 한계(GIL 블로킹 이슈) 때문에, 네트워크가 늦는 건지 파이썬이 늦는 건지 알 길이 없다. 무조건 프로토콜 한계점 테스트는 C/Rust 기반의 순정 네이티브 도구로 찍어라.

0.1 [Runbook] z_perf 및 zenoh-bench를 통한 한계 돌파 측정

우선 터미널 2개를 연다.

1. 대역폭(Throughput) 테스트: 우주 방어 측정
“내 망이 1초에 기가바이트를 견딜 수 있는가?”

# 터미널 1 (수신단)
z_sub_perf -e "tcp/127.0.0.1:7447"

# 터미널 2 (송신단: 1024바이트짜리 더미 데이터를 초당 죽을만큼 보냄)
z_pub_perf -e "tcp/127.0.0.1:7447" -s 1024

마지막에 화면에 [Throughput] 1.2 GB/s, 1,200,000 msgs/s 라는 스펙이 찍힌다. 이것이 당신이 구축한 인프라와 OS 환경의 “이론상 우주 최강 한계값“이다.

2. 지연 시간(Latency / Ping-Pong) 테스트: 날 끝의 예리함 측정
“왕복에 몇 마이크로(Micro)초가 걸리는가?”

# 터미널 1 (응답자)
z_ping -e "tcp/MY_ROUTER_IP:7447"

# 터미널 2 (질의자)
z_pong -e "tcp/MY_ROUTER_IP:7447"

나오는 Ping: 124us (마이크로초) 라는 값. 이것보다 느린 서비스 코드가 있다면, 네트워크 문제가 아니라 전적으로 100% 로봇 애플리케이션 코드를 잘못 짠 것이다. 이 벤치마킹 값을 면죄부(Baseline) 로 활용해라.
