16.2.4 CPU 선호도(Affinity) 및 NUMA(Non-Uniform Memory Access) 아키텍처 인지 배치

서버 하나에 CPU 가 2개 꽂혀있고, 램(RAM) 도 두 바구니에 나뉘어(NUMA) 있다치자.
네트워크 카드(NIC) 는 0번 CPU 쪽에 꽂혀 있는데, 정작 Zenoh 라우터 프로세스는 1번 CPU 위에서 돌고 있다면? 패킷 하나를 처리할 때마다 데이터가 메인보드 중앙(QPI 통신망) 을 건너가며 치명적인 병목 지연(NUMA Miss) 이 발생한다.

0.1 [Runbook] 물리적 코어 바인딩(Pinning) 전술

“일하는 자의 발 밑에 자재를 두라.”

1. 인터럽트 핀셋팅 (SMP Affinity)
랜 카드가 외치는 통보망을 오직 0번, 1번 물리 코어로 몰아넣는다.

echo 3 > /proc/irq/<NIC_IRQ_NUMBER>/smp_affinity

2. 데몬 프로세스 격리 수용 (Taskset / NUMA Control)
Zenoh 라우터 바이너리(zenohd) 를 기동할 때, 랜 카드가 꽂혀 있는 바로 그 NUMA 노드에만 독박으로 가둬버린다.

# NUMA 0번 존에서만 zenohd 를 실행하고 램을 당겨써라!
numactl --cpunodebind=0 --membind=0 zenohd -c config.json5

이렇게 물리적으로 CPU 와 패킷과 메모리 버퍼를 한 구역에 밀어 넣고 문을 잠가버리면, 캐시 적중률(Cache Hit Ratio) 이 99%에 육박하게 되어 마이크로초 단위의 성능 최적화가 완성된다.
