16.3 트랜스포트 계층(Transport Layer) 최적화

OS 라는 거대한 항구를 정비했다면, 이제 배를 수리할 차례다. 어떻게 실어나를 것인가? (TCP, UDP, QUIC)

Zenoh 는 세상에 존재하는 거의 모든 트랜스포트(Transport) 프로토콜을 설정 한 줄(tcp/, udp/, quic/) 로 바꿀 수 있는 오버스펙(Over-spec) 의 결정체다.
“어차피 다 된다면 무조건 TCP(안전함)를 쓰면 되는 것 아닌가?”
바로 이 안일한 아키텍처적 타협이 지연 시간을 폭발시키고 기지국을 과부하에 빠뜨리는 주범이 된다. 당신의 애플리케이션 데이터가 가진 질량(크기) 과 속성에 맞는 완벽한 로켓(Transport Scheme) 을 매칭하는 런북을 완성하라.
