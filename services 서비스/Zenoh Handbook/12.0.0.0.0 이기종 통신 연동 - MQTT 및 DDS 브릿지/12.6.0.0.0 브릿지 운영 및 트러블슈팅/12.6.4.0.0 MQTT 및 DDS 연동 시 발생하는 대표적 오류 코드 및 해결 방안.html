12.6.4 MQTT 및 DDS 연동 시 발생하는 대표적 오류 코드 및 해결 방안

경험 많은 아키텍트들의 눈물과 밤샘이 누적된 장애 해결의 핵심 바이블이다.

0.1 [인스펙션] 크로스 매핑 오류 부검 리스트

1. [ERROR] Unable to convert payload from MQTT to DDS

  • 증상: MQTT 에서 보낸 센서값이 ROS2 화면에 안 뜬다.
  • 원인: MQTT 는 문자열("{temp:20}") 을 보냈는데, 수신 측 ROS2 앱은 C++ 의 Float32 메모리 이진 덤프를 기다리고 있다. 데이터 타입(Type) 불일치로 브릿지가 번역을 포기한 것이다.
  • 해결책: 12.5.4 장의 런북을 참고하여, 중간에 JSON 을 CDR 규격으로 파싱해 주는 마이크로 파이썬 스크립트를 하나 띄워서 포맷을 일치시켜라. 브릿지는 마법사가 아니다.

2. [FATAL] Failed to initialize DDS participant

  • 증상: 로봇 안에서 브릿지를 켜는 순간 바로 꺼져버린다.
  • 원인: 브릿지와 ROS2 코어가 리눅스 커널의 공유 메모리(SHM) 세그먼트를 서로 잡아먹으려다 충돌한 것이다. (보통 CycloneDDS 환경의 버그).
  • 해결책: export CYCLONEDDS_URI=... 설정을 주어 브릿지의 DDS 통신을 공유메모리(SHM) 방식이 아닌 순수 UDP 네트워크 모드로 강제 격하시켜라.

3. 브릿지는 죽지 않았는데 데이터가 한 방향으로만 안 흐른다!

  • 증상: 로봇 상태는 클라우드로 잘 올라가는데, 클라우드 관제 대시보드에서 내리는 조작 명령이 로봇으로 들어가지 않는다. (절반의 성공)
  • 원인: 십중팔구 브릿지의 Allow / Deny (화이트리스트) 설정을 한쪽 방향만 열어 두었기 때문이다. zenoh_to_dds 정책과 dds_to_zenoh 정책이 서로 독립적으로 구성되어 있다는 기초를 망각한 실수다. 설정 파일의 forwarding 항목을 다시 읽어라.
