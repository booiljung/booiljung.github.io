12.6.2 브릿지 프로세스의 CPU 및 메모리 사용률 최적화 기법

브릿지는 통신 트래픽의 모든 칼바람을 온몸으로 맞는 최전방 방어막이다. 세팅이 틀어지면 브릿지가 로봇 컴퓨터의 RAM 의 절반을 집어삼키는 괴물이 된다.

0.1 [Runbook] 메모리 다이어트 및 시스템 자원 결발 전술

1. DDS 브릿지 메모리 폭주 (OOM) 억제

  • 증상: ROS2 가 떠 있는 로봇 안에 켠 브릿지가 점점 메모리를 갉아먹다가 OOM(Out of Memory) 으로 강제 종료된다.
  • 주원인: 로봇이 대용량 데이터(카메라, 레이저 등) 를 TRANSIENT_LOCAL (내구성 옵션 켜짐) 로 쏠 경우 브릿지가 이걸 Zenoh 캐시에 우겨넣으려다 램이 터진다.
  • 타개책: 브릿지 옵션 중 QoS 매핑을 무조건 VOLATILE (버림) 로 강제 강등시켜라.

2. CPU 코어 락온 (Taskset)
브릿지의 패킷 해독 속률을 최상으로 끌어올리려면 운영체제가 브릿지 스레드를 이 코어 저 코어로 넘기며 일어나는 캐시 미스(Cache Miss) 를 차단해야 한다.

# 리눅스의 1번과 2번 CPU 코어에만 브릿지 데몬을 강제 결속시킨다!
sudo taskset -c 1,2 zenoh-bridge-dds -c my_config.json5

이 한 줄로 브릿지의 Jitter(지연 시간 흔들림 폭) 가 30% 이상 극적으로 안정되는 튜닝의 마법을 겪게 될 것이다.
