12.3.8 TLS/SSL을 활용한 안전한 MQTT 브릿지 통신 구성

공장 외부 인터넷(WAN) 을 거쳐 AWS IoT 브로커와 연결하려면, ID/PW 만으론 부족하고 패킷 자체가 암호화(MQTTS) 되어야 한다. 포트번호 또한 1883 이 아니라 8883 (TLS) 가 국룰이다.

0.1 [Runbook] 암호화 프로토콜 핸드셰이크 전술

1. TLS 엔드포인트 타게팅
브릿지를 켤 때 tcp/ 접두사가 아니라 tls/ 를 사용하여 암호화 스택을 깨운다.

{
  plugins: {
    mqtt: {
      // 엔드포인트를 tls 로 바꾸고 포트번호도 8883(표준) 으로 지정!
      broker: "tls/aws-iot-broker.com:8883",
      
      // AWS 가 묻는다 
      // "네가 등록된 단말기인지 너의 인증서를 제출해라!" (Client Cert Mutual TLS)
      tls: {
        client_cert: "/opt/certs/device_001.pem",
        client_key: "/opt/certs/device_001.private.key",
        
        // AWS 서버가 진짜인지 내가 검증하는 루트 인증서 목록
        root_ca_certs: ["/opt/certs/AmazonRootCA1.pem"]
      }
    }
  }
}

이렇게 세팅된 브릿지는 그 보안 깐깐하기로 소문난 AWS IoT Core 까지 완벽하게 패스하여, Zenoh 에서 일어나는 로봇 움직임을 안전하게 글로벌 클라우드의 S3 스토리지 파이프라인(MQTT) 안으로 꽂아 넣게 된다.
