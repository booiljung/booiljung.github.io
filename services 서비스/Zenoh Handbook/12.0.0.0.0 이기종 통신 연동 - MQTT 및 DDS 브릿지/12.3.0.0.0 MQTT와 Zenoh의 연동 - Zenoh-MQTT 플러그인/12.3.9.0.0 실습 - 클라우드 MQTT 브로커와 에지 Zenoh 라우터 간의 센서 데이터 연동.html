12.3.9 실습: 클라우드 MQTT 브로커와 에지 Zenoh 라우터 간의 센서 데이터 연동

마지막 최종 부검 실습이다. 내 회사 방구석에 떠 있는 Zenoh 데스크톱이 있고, 전 세계 누구나 쓸 수 있는 퍼블릭(Public) MQTT 브로커(Eclipse Mosquitto) 가 있다. 이 둘을 구멍 뚫어 기적을 만든다.

0.1 [Runbook] 세상의 끝과 끝 연결하기

1. 내 데스크톱 (Zenoh 수신 준비)

# 내 PC 에서 Zenoh 로 "mqtt/sensor" 토픽 대기 (아직 아무것도 안옴)
z_sub -k "mqtt/sensor/#"

2. 브릿지 투입 (스파이 파견)
나의 PC 백그라운드에서, 퍼블릭 테스트용 MQTT 브로커 서버(test.mosquitto.org) 로 스파이를 위장 송출한다.

# 외부 브로커에 빨대를 꽂고, 
# 겹치지 않게 "mqtt" 라는 네임스페이스 폴더 안에 다 담아와라!
zenoh-bridge-mqtt -b tcp/test.mosquitto.org:1883 --scope "mqtt"

3. 원격지 센서(발신) 흉내 내기
스마트폰이나 다른 컴퓨터에서, 오직 표준 MQTT 프로그램만 켜서 공용 브로커에 토픽을 쏜다.

# 일반 MQTT 클라이언트 도구 사용 (Zenoh 와 무관하게 발송)
mosquitto_pub -h test.mosquitto.org -t "sensor/temp" -m "35.5 C"

[결과 분석]
당신의 z_sub 가 켜져 있던 데스크톱 (1번 창) 에 35.5 C 가 찍혔는가?
이 마법은 이렇다: 외부 IoT 기기 -> (MQTT 규격) -> 공용 브로커 -> 브릿지 스니핑 -> (Zenoh 규격 변환) -> mqtt 네임스페이스 덮어쓰기 -> 당신의 로컬 z_sub 도착.
이로써 거대한 MQTT 대륙이 당신의 작은 Zenoh 식탁 위로 병합 완료되었다.
