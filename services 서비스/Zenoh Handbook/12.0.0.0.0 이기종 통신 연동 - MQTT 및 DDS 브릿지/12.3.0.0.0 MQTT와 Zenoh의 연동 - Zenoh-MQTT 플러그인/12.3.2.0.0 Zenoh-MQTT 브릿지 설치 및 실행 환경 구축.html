12.3.2 Zenoh-MQTT 브릿지 설치 및 실행 환경 구축

가장 고전적이고 안정적인 브릿지 플러그인이다.

0.1 [Runbook] MQTT 플러그인 핫-로드 전술

1. 바이너리 획득

# Ubuntu / Debian 환경
sudo apt install zenoh-plugin-mqtt

2. 브릿지 단독 실행 모드 (Stand-alone)
Zenoh 본체(라우터) 와 별개로 브릿지 코드만 독립적인 프로세스로 켤 때 쓴다.

# 로컬(127.0.0.1)에 떠 있는 MQTT 브로커(포트 1883) 에 빨대를 꽂는다!
zenoh-bridge-mqtt -b tcp/localhost:1883

3. 라우터 기생 모드 (Plugin Injection)
가장 권장하는 엔터프라이즈 전술이다. zenohd (라우터) 를 켤 때 MQTT 브릿지 라이브러리를 통째로 몸속에 박아 넣은 채로 기동한다.

  • zenohd 의 설정 파일(zenoh.json5) 에 plugins 항목을 연다.
// zenoh.json5
{
  plugins: {
    mqtt: {
      // 로컬 MQTT 브로커와 통신!
      broker: "tcp/127.0.0.1:1883" 
    }
  }
}

이제 터미널에서 zenohd --config zenoh.json5 를 실행하면, 순식간에 MQTT 브로커의 모든 트래픽이 Zenoh 글로벌 망으로 흡수되기 시작한다.
