Chapter 12. 이기종 통신 연동: MQTT 및 DDS 브릿지

모든 시스템이 단 하나의 언어로, 단 하나의 프로토콜로 대화하는 세상은 이상에 불과하다.
현실의 시스템은 파편화되어 있다. 에지 디바이스와 IoT 센서는 수력 발전소부터 스마트 홈까지 수십 년 전부터 MQTT(Message Queuing Telemetry Transport)의 가벼운 지배를 받고 있으며, 로보틱스와 방위 산업 현장은 DDS(Data Distribution Service)라는 무겁고 신뢰성 높은 장벽 뒤에 숨어 있다.

이 거대한 단절을 메우기 위해 우리는 그동안 ’미들웨어들의 미들웨어’를 덕지덕지 이어 붙이는 어리석음을 범해왔다.
하지만 Zenoh는 단순히 또 하나의 프로토콜로 남기를 거부한다.
Zenoh는 자신의 극단적 라우팅 유연성을 무기로, DDS와 MQTT라는 양극단의 괴물들을 자신의 생태계 안으로 흡수하여 완벽히 통제하는 **‘이기종 프로토콜의 대통합 허브(Universal Communication Hub)’**로 군림한다.

이 12장에서는 단절된 에지와 클라우드, 구세대 프로토콜과 차세대 미들웨어를 이어주는 마법의 톱니바퀴, **Zenoh 브릿지 플러그인(Bridge Plugin)**의 내장을 해부한다. MQTT 브로커와 Zenoh 라우터를 강제로 융합시키는 법, DDS 네트워크를 광역망으로 끌어올리는 아키텍처, 그리고 이 모든 것을 묶어 데이터 무한 루프의 재앙 없이 엔드투엔드(End-to-End) 파이프라인으로 제련하는 궁극의 연동술을 파헤칠 것이다.
