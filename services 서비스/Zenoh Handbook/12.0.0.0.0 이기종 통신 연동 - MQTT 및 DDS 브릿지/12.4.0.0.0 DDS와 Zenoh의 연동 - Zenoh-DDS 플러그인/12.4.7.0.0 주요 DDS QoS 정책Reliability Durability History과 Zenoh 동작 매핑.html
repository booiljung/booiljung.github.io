12.4.7 주요 DDS QoS 정책(Reliability, Durability, History)과 Zenoh 동작 매핑

DDS 에는 수만 가지 종류의 변태적인 QoS 조합이 존재한다. 브릿지가 이 복잡한 정책을 잘못 번역하면 통신이 얼어붙는다.

0.1 [Runbook] QoS 타협과 가지치기 (Pruning) 전술

가장 치명적인 3대 QoS 매핑 룰을 점검한다.

1. Reliability (신뢰성)

  • DDS 의 RELIABLE -> Zenoh 브릿지는 이를 감지하고 Zenoh 내부 터널의 Reliable 옵션을 켠다.
  • DDS 의 BEST_EFFORT -> Zenoh 패킷 유실을 허용하고 무조건 최신 데이터를 밀어 넣는다. (카메라나 라이다 토픽에 무조건 이 QoS 를 써야 시스템이 살 수 있다).

2. Durability (내구성 - 과거 보존)

  • DDS TRANSIENT_LOCAL (나중에 접속한 놈에게 과거 데이터를 보여줌) -> 브릿지는 Zenoh 엔진에 내장된 인-메모리 캐시(Cache) 를 켜서 일시적으로 데이터를 쥐고 있는 상태로 변환된다.
  • [위험 경고] 대용량 카메라 영상에 이 옵션이 켜져 있으면, 브릿지를 켜자마자 컴퓨터의 RAM 이 폭발한다! 반드시 VOLATILE (휘발성) 로 설정하라.

3. Liveliness (생존 확인)
DDS 의 피곤한 심장 박동(Heartbeat) 정책. 브릿지는 이것을 적당선에서 무시하거나 Zenoh 세션이 살아있는 한 계속 살아있는 것으로 퉁(?) 치고 넘어가도록 유연하게 설계되어 있어, WAN 통신에서의 트래픽 오버헤드를 극적으로 줄여준다.
