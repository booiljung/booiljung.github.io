12.4.9 실습: 기존 ROS2/DDS 로봇 네트워크를 Zenoh 인터넷 망으로 확장하기

방화벽 밖에 있는 로봇(Domain 0)에 떠 있는 /chatter 토픽을, AWS 관제 서버를 뚫고 통과하여 내 집 노트북 Rviz 화면으로 직결시키는 최종 스나이핑 예제다.

0.1 [Runbook] VPN 제로 인터넷 터널링 전술

1. [클라우드 백본] 인터넷상에 라우터 세우기
AWS EC2 를 한 대 띄우고 오직 공유기 역할만 할 라우터를 켠다. (IP: 198.51.100.1)

# AWS 터미널
zenohd

2. [지구 한편의 로봇] DDS 발송 준비 및 브릿지 연결
서울에 있는 로봇(Raspberry Pi) 이 DDS 망 안에 /chatter 를 쏘고 있다.
이 데이터를 낚아채 AWS 로 올려버린다.

# 로봇 터미널 (Scope 옵션을 줘서 이름표를 바꾼다)
zenoh-bridge-dds -e tcp/198.51.100.1:7447 --scope "robotA"

3. [지구 반대편 내 노트북] 브릿지 연결 및 데이터 수신
집에 있는 내 PC 에서는 이 로봇의 데이터를 다시 Rviz나 DDS 콘솔로 던져줘야 한다.

# 노트북 터미널 (브릿지)
# 클라우드에서 내려오는 Zenoh 패킷을 낚아채서 
# 내 방 안의 DDS 로 뿌려준다!
zenoh-bridge-dds -e tcp/198.51.100.1:7447

[작전 성공 증명]
내 방 노트북에서 ROS2 의 기본 커맨드를 친다.
ros2 topic echo /robotA/chatter
짜잔! 서울 로봇의 DDS 데이터가 직렬화 오버헤드 없이, 어떤 오픈 VPN 이나 포트 포워딩 작업도 없이 내 방 데스크톱의 터미널에 송곳처럼 꽂힌다. 이것이 이 책이 증명하고자 했던 “이기종 글로벌 분산 통신망의 물리적 타파” 이다.
