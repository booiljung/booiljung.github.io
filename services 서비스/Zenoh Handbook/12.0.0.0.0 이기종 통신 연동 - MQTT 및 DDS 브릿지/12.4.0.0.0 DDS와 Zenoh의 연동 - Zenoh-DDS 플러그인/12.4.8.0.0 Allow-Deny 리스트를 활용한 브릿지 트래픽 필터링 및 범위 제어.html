12.4.8 Allow/Deny 리스트를 활용한 브릿지 트래픽 필터링 및 범위 제어

DDS 망 전체를 Zenoh 다리(Bridge)로 퍼 올리는 짓은 곧 네트워크 자살이다.
/rosout 에러 로그나 거대한 Map 데이터를 막아내고 오직 센서 통계값만 외부로 빼내야 한다.

0.1 [Runbook] 데이터 방화벽(Tollgate) 설정 전술

설정 파일 config.json5 의 위력을 보여주는 화이트리스트(Whitelist) 런북.

// dds_bridge.json5
{
  // 1. DDS망에서 Zenoh (외부 클라우드)로 나가는 데이터 통제
  dds_to_zenoh: {
    // 디폴트값: 아무것도 밖으로 나갈 수 없다. 방화벽 셧다운!
    forwarding: "deny",
    
    // 예외 조항 (허락된 놈들만 나간다)
    allow: {
      topics: [
        "/robot_status", 
        "/battery_level"
      ],
      // 특정 이상한 타입(Type) 이 들어오면 토픽명 상관없이 다 드랍!
      // (예: 대용량 이미지 타입 외부 유출 1차 방화벽)
      types: [ "std_msgs::msg::dds_::Float32_" ]
    }
  },

  // 2. 외부(클라우드)에서 내 로봇장비(DDS) 로 들어오는 데이터 통제
  zenoh_to_dds: {
    forwarding: "deny",
    allow: {
      topics: [
        "/cmd_vel" // 오직 주행 명령만 외부 해킹(?) 허용
      ]
    }
  }
}

실행: zenoh-bridge-dds -c dds_bridge.json5
이 스크립트는 거대한 공장 안에서 오직 허가된 텔레메트리(원격 통계) 데이터만 LTE 망의 좁은 구멍을 통과하게 만드는 기업형 엣지 톨게이트 세팅의 정수다.
