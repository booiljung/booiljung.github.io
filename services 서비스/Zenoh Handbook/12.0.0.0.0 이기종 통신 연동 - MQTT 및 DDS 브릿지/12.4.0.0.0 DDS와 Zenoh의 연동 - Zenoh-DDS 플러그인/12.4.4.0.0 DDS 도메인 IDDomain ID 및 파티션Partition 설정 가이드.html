12.4.4 DDS 도메인 ID(Domain ID) 및 파티션(Partition) 설정 가이드

로봇 공장에 두 대의 로봇이 똑같이 Domain ID: 0 을 쓰고 있다면 조이스틱을 꺾을 때 두 대가 같이 움직이는 대참사가 벌어진다. 이를 막기 위해 로봇별로 도메인 번호를 나눈다. 브릿지도 이 룰을 준수해야 한다.

0.1 [Runbook] 도메인 타게팅(Targeting) 훔치기 전술

1. 도메인 조준경 설정 옵션
로봇 A가 Domain ID: 41 안에서 돌고 있다면, 브릿지를 켤 때 이 방(Room) 에 맞춰서 들어가야 한다. 안 그러면 엉뚱한 로봇의 데이터를 퍼 올리게 된다.

# `-d` 옵션을 주어 41번 도메인의 트래픽만 잡아먹어라!
zenoh-bridge-dds -d 41 -e tcp/aws-router.com:7447

2. 멀티 도메인(Multi-Domain) 동시 연결은 불가능하다!

  • 경고: 브릿지 프로세스 하나당 오직 한 개의 도메인 아이디만 락온(Lock-on) 할 수 있다.
  • 만약 한 컴퓨터에서 41번과 42번 도메인을 모두 클라우드로 올리고 싶다면, zenoh-bridge-dds -d 41 프로세스와 -d 42 프로세스를 따로 2개 띄우는 것 만이 유일한 해법이다. (포트 충돌 주의).
