15.4.5 시스템 재시작 없는 동적 ACL 업데이트 및 핫 리로드(Hot Reload)

직원 한 명이 퇴사했다. 그의 권한(deny) 을 막으려고 라우터를 껐다 켜면(Restart), 그 라우터 밑에 붙어있던 드론 1,000 대의 조종 스틱이 1초간 마비되어 추락할 것이다. 보안을 위해 물리적 시스템 가용성을 버티면 안 된다.

0.1 [Runbook] 무정지 ACL 업데이트 전술

Zenoh 라우터의 관리자(Admin) 공간을 통과하는 라이브 업데이트 런북이다.

1. 관리자 토픽에 수술용 메스 꽂기
Zenoh는 자기 자신의 설정을 자기 자신의 토픽(Key Expression) 으로 관리하는 기묘한 재귀적(Recursive) 프로토콜이다.
특정 라우터(예: ID가 1234 인 놈) 의 ACL 설정은 그 라우터의 @/router/1234/plugin/access_control/config 라는 특수한 관리자용 토픽으로 매핑되어 있다.

2. 핫-리로드 API 호출 (z_put)
아래와 같이 새로운 JSON 설정 문자열을 만들어서, 다른 컴퓨터에 앉은 관리자가 Put 을 쏴서 라우터의 머릿속에 던져 넣는다.

# REST API (HTTP PUT) 를 사용하는 런북
curl -X PUT http://admin:admin_password@<ROUTER_IP>:8000/@/router/local/plugin/access_control/config \
     -H "Content-Type: application/json" \
     -d '{
           "users": { "drone_01": "worker", "bad_guy_01": "hacker" },
           "roles": { "hacker": { "deny": ["**"] } }
         }'

이 패킷이 라우터에 꽂히는 즉시, 데몬 프로세스는 재부팅 없이(No Restart) 메모리 위의 ACL 트리를 갈아 끼운다(Hot-Swap). 통신선이 1나노초도 끊기지 않은 채, bad_guy_01 유저가 쏜 다음번 패킷부터는 즉시 소켓에서 필터링당하는 진정한 로보틱스-네이티브 방어 시스템이 구현된다.
