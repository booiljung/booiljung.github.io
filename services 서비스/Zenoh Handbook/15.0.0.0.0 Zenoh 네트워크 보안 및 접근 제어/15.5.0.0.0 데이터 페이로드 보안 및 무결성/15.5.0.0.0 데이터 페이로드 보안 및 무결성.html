15.5 데이터 페이로드 보안 및 무결성

아무리 단단한 철갑선(TLS 터널, mTLS 인증) 이라 하더라도, 결국 항구(라우터 관리자) 에 도착하면 화물 컨테이너(Data Payload) 는 벌거벗겨진다.
스파이가 아예 AWS 인프라(라우터 데몬) 의 최고 권한(Root) 을 장악했다면 라우터를 지나가는 모든 평문(Plaintext) 은 털린 거나 다름없다.

이 장에서는 네트워크 터널(인프라) 따위에 내 기밀을 절대로 의존하지 않는 방어 체계, 즉 엔드-투-엔드(End-to-End, E2E) 애플리케이션 단독 암호화의 지옥불 런북을 완성한다. 라우터조차 속여 넘기는 궁극의 영-지식(Zero-Knowledge) 아키텍처다.
