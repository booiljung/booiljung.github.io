15.5.2 메시지 서명(Message Signing)과 위변조 방지

내용을 숨길 필요는 없는데, 누가 중간에서(MitM) 값을 바꿔치기 하는 짓거리는 절대 용납 못 할 때 쓰는 전술이다.

0.1 [Runbook] 불변성(Immutability) 해싱 전술

자율주행 자동차가 앞차의 ‘브레이크 강도’(10) 를 받았다 치자. 누군가 중간에서 이 값을 0 로 바꾸면 살인 무기가 된다. 수신자는 이 패킷의 값이 원래 앞차가 만든 10 이 맞는지 수학적으로 추궁해야 한다.

1. 디지털 서명 (Digital Signature) 동봉
로봇이 Put 할 공간(Payload) 의 앞단 일부(Header) 를 잘라서, 실제 데이터의 SHA-256 해시값과 자신의 비공개 키로 만든 서명(Signature) 을 끼워 넣는다.

  • Payload 구조 = [Signature (256 Byte)] + [실제 온도값 (4 Byte)]

2. 구독자의 검증망 (Verification)
구독자(관제 서버) 가 이 데이터를 들었다.

  • 서버는 로봇의 공개 키(Public Key) 를 들고 있다.
  • 아무나 위조할 수 없는 앞단의 256바이트 서명을 로봇의 공개 키로 박살내 본다.
  • 그 안에서 튀어나온 원본 다이제스트(해쉬값) 와, 그 뒤에 달려온 온도값(4 Byte) 의 해쉬값을 매칭해 본다. 일치하면 순결한 데이터, 단 한 비트라도 다르면 오는 도중 오염된 좀비 데이터(Corrupted) 다. 즉결 가비지 컬렉터(Drop) 로 전송하라.
