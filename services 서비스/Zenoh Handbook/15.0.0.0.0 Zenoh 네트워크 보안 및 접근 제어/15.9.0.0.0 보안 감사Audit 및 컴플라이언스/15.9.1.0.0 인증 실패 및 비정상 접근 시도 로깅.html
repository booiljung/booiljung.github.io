15.9.1 인증 실패 및 비정상 접근 시도 로깅

시스템이 누군가의 접근을 차단했다면 잘한 일이지만, 차단 로그를 남기지 않았다면 당신은 눈을 감고 맞고 있는 것과 같다.

0.1 [Runbook] 침투 지표(IoC, Indicator of Compromise) 확보 전술

Zenoh 코어 라우터를 구동할 때, 로깅(Logging) 레벨의 정밀한 조준이 필요하다.

1. 보안 레벨(Security Level) 로깅의 필터링
단순히 RUST_LOG=info 로 켜면 에지 로봇 1만 대가 1초마다 보내는 “연결됨/종료됨” 로그에 하드디스크가 3시간 만에 타버린다. 오직 “인증 모듈“과 “ACL” 에 대한 필터만을 핀셋으로 집어내야 한다.

# Auth(인증) 와 ACL 모듈에서 발생하는 WARN 급 이상의 에러만 로그 파일로 떨어뜨려라!
export RUST_LOG="zenoh::net::auth=warn,zenoh::plugin::acl=warn"
zenohd -c config.json5 > /var/log/zenoh_security.log 2>&1

2. 로그 분석 (Fail2Ban 연동)
이 로그 파일에는 [WARN] Auth failed for user 'admin_fake' from IP 198.51.100.99 라는 텍스트가 쌓인다.
이 텍스트 파일을 리눅스의 Fail2Ban 데몬이 실시간으로 읽게 하여, 1분 안에 로그인 실패가 5번 이상 찍히는 즉시 OS 단위(iptables) 에서 그 IP 를 1시간 동안 완전 블록(Drop) 시켜버리는 2중 방어망을 구축해야 한다.
