15.6.2 리스너(Listener) 바인딩 제한 및 네트워크 인터페이스 격리

컴퓨터에 랜 카드가 2개(인터넷망 하나, 로봇 내부 내부망 하나) 꽂혀 있다. 라우터를 켰을 때 어느 랜 카드로 통신을 열어둘 것인가?

0.1 [인스펙션] 0.0.0.0의 대학살과 인페이스(Interface) 고립 전술

가장 초보적인 실수는 listen: ["tcp/0.0.0.0:7447"] 이라고 적는 것이다.
0.0.0.0 은 “이 컴퓨터에 달린 모른 랜 카드의 7447 포트를 전부 다 열어라” 라는 뜻이다. 로봇 내부 부품끼리만 은밀하게 대화하려던 통신선이, 바깥 인터넷까지 싸그리 노출되는 참상이 벌어진다.

1. 로컬 호스트(Localhost) 감금
관제 센터의 백엔드용 라우터나, 로봇 내부 컴포넌트 간의 브릿지 용도로만 라우터를 띄운다면, 타겟 반경을 하드디스크 밖으로 벗어나지 못하게 묶어라.

// 오직 내 컴퓨터 안에 있는 프로세스들끼리만 접속 가능! 
// 바깥 세상(네트워크) 에서는 이 포트가 아예 닫힌 것으로 보인다.
listen: ["tcp/127.0.0.1:7447"]

2. 특정 랜 카드(NIC) 강제 타겟팅 지정
에지 컴퓨터에 5G 모뎀(외부망, 10.0.x.x) 과 라이다 센서 직결망(내부 랜, 192.168.1.1) 이 동시에 존재한다면?
외부에서 해커가 5G망을 뚫고 들어와 라이다 제어 포트를 조작하지 못하도록, 제어 라우터는 오직 격리된 내부 랜 카드의 IP 로만 소켓을 뚫어라.

// 라이다망(내부망) 만 이 라우터를 볼 수 있다.
listen: ["tcp/192.168.1.1:7447"]

OS 의 방화벽(iptables)조차 건드릴 필요 없이, Zenoh 레벨에서 인터페이스 바인딩을 잘라버리는 가장 우아하고 무자비한 네트워크 격리법이다.
