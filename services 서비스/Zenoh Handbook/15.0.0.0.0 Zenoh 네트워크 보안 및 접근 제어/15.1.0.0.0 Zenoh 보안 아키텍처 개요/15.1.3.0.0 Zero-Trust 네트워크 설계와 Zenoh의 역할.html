15.1.3 Zero-Trust 네트워크 설계와 Zenoh의 역할

“Zero-Trust (아무 것도 믿지 마라)”
기존 방화벽 개념은 성벽 스위치와 같다. “사내 IP 10.0.X.X 대역은 착한 놈들이니 묻지도 따지지도 말고 포트를 다 열어줘라”
이것이 클라우드와 엣지가 뒤섞인 현대 로보틱스 망을 가장 빨리 망하게 하는 지름길이다.

0.1 [인스펙션] 기만없는 파편화 인증의 역학

1. IP의 무가치성 선언
로봇은 5G를 타고 터널 안에 들어가면 IP가 동적으로 계속 바뀐다. IP 주소가 바뀌는 상황에서 “IP 기반 방화벽“은 아무런 통제 장치가 되지 못한다.
Zenoh는 접속하는 자의 IP를 믿지 않는다. 오직 “그 자가 가져온 암호화폐(인증서 키 Pair)” 만을 보고 판단(Authentication)한다.

2. 노드 레벨의 미시적(Micro) 방화벽
중앙(클라우드 입구)에 거대한 방화벽 1개를 두는 짓을 버려라.
Zenoh에서는 1,000대의 라즈베리 파이 단말기마다 떠 있는 1,000개의 데몬 스레드 각각이 자체적인 ACL 인증 엔진을 소유하도록 설계되어 있다. 이웃한 로봇 2대가 직통(P2P)으로 연결될 때조차 상호 간의 TLS 인증서 검문을 수행한다. 거대한 방화벽 1개가 아니라, 1,000개의 미세 바리케이드가 망 전체에 깔려 있는 궁극의 분산 격리(Distributed Isolation) 망이다.
