15.1.4 Zenoh 보안 도메인 및 격리(Isolation) 개념

회사에 A 부서(공장 제어) 와 B 부서(인사부 화상 회의) 가 섞여 있다 치자.
망이 연결되어 있다는 이유만으로 A 부서 직원이 실수로 B 부서의 패킷을 들여다볼 가능성을 아예 0%로 만들 수는 없는가?

0.1 [Runbook] 도메인 아이디(Domain ID) 파티셔닝 전술

Zenoh는 이것을 물리적으로 가위질해 버릴 수 있다. 바로 도메인 아이디(Domain ID) 메타데이터를 통해서다.

1. 완전한 데이터 절연(Galvanic Isolation)
Zenoh 프로토콜을 설정할 때, domain 이라는 숫자를 박아 넣을 수 있다.

// 부서 A 라우터
{
  domain: 100 // 로보틱스 전용 도메인
}

// 부서 B 라우터
{
  domain: 200 // 비디오 스트리밍 전용 도메인
}

이 규칙이 켜지는 순간, 도메인 100 에 속한 라우터와 클라이언트가 날린 Put 혹은 Get 패킷은 다른 도메인에게 절대 전달되지 않는다.
토픽 이름(Key Expression)이 똑같다 하더라도 상관없다. 밑단의 프로토콜 엔진 자체가 도메인 번호가 다르면 그 통신 소켓을 마치 “다른 물리적 네트워크 랜 케이블“이 꽂힌 것처럼 완벽하게 격리(Isolate)하고 폐기한다.

도메인 레벨 파티셔닝은 복잡한 접근 제어(ACL) 설정 없이도 가장 원시적이고 철저하게 데이터 군락을 독립시키는, 보안 아키텍트의 무책임하지만 가장 강력한 칼날이다.
