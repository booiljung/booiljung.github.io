15.7.3 MQTT 및 DDS 브릿지 연결 시의 보안 경계(Security Perimeter) 설정

공장 벽면에 붙은 수백 대의 구형 MQTT 센서들. 이 녀석들은 ID/PW 를 다 까발린 평문으로 1883 포트로 데이터를 쏜다.

0.1 [인스펙션] DMZ (Demilitarized Zone) 억류 전술

브릿지를 클라우드 서버나 관제망(Core) 에 깊숙히 설치하지 마라.
오염된 레거시 프로토콜이 Zenoh 와 맞닿는 “브릿지 프로세스” 자체를 가장 최전방(Edge) 지뢰밭에 배치해야 한다.

1. 경계망(Perimeter) 전진 배치
MQTT 브로커 프로그램(Mosquitto) 이 깔려있는 바로 그 “공장 허브 PC” 안에 zenoh-bridge-mqtt 를 설치하라.

  • 센서 \rightarrow 브로커 \rightarrow 브릿지: 오직 공장 건물 내부의 스위칭 허브 구간만 “위험한” 평문으로 둔다.
  • 브릿지 \rightarrow Zenoh 라우터: 브릿지가 데이터를 물고 밖으로 나가는 순간부터는 철저하게 Zenoh 의 룰(TLS, ACL, 인증서) 에 지배당하도록 터널을 뚫어버린다.

2. 브릿지 전용 격리 계정 생성
zenoh-bridge-mqtt 도 결국 Zenoh 라우터 입장에서는 단순한 클라이언트(Client) 다.
브릿지가 쓰는 설정 파일에 username: legacy_mqtt_bridge 라고 적어두고, 라우터 ACL 에서는 legacy_mqtt_bridge 계정이 절대로 다른 핵심 장비(드론, 코어 백엔드) 의 토픽을 훔쳐보지 못하도록, 오직 자기가 수집한 센서 토픽(factory/mqtt_zone/**) 에만 Put 을 쏠 수 있게 창살 없는 감옥을 설계해야 한다.
외부인(브릿지) 이 흘리는 피를 아군 네트워크의 심장부에 묻히지 마라.
