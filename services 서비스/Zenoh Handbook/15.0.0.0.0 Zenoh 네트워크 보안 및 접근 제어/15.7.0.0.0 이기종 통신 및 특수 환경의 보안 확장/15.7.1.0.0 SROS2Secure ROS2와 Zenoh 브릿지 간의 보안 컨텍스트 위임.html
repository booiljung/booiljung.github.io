15.7.1 SROS2(Secure ROS2)와 Zenoh 브릿지 간의 보안 컨텍스트 위임

ROS2는 자체적으로 SROS2(Secure ROS2) 라는 거대한 보안 아키텍처를 가지고 있다. 이 노드들이 뿜어내는 데이터를 Zenoh 브릿지가 훔쳐서 클라우드로 던지려 할 때 벌어지는 피아식별 교착 상태의 해결책이다.

0.1 [인스펙션] 양면 거울(Two-Way Mirror) 브릿징 전술

브릿지(zenoh-bridge-dds) 프로세스는 ROS2 입장에서는 ’가짜 로봇’이고, 클라우드 Zenoh 라우터 입장에서는 ’가장 말단 센서’다. 양쪽 모두에게 신분증을 내밀어야 한다.

1. SROS2 참전(Enclave) 자격 획득
브릿지를 켤 때, ROS2 보안 공간(Enclave) 에 접근할 수 있는 DDS 인증서 더미를 쥐어줘야 한다.

# 브릿지가 SROS2 망의 데이터를 탈취할 수 있도록 환경변수 강제 주입
export ROS_SECURITY_ENABLE=true
export ROS_SECURITY_STRATEGY=Enforce
export ROS_SECURITY_KEYSTORE=/path/to/my_bridge_keystore

# SROS2 환경변수를 머금은 채 브릿지 기동!
zenoh-bridge-dds -e "tls/cloud-router:7447"

2. Zenoh 터널(TLS) 로의 환승
브릿지가 SROS2 망에서 데이터를 성공적으로 훔쳐냈다면, 이제 이것을 클라우드로 던지기 위해 Zenoh의 TLS 옷으로 갈아입어야 한다.
이는 zenoh-bridge-dds 커맨드의 -e 옵션 끝에 붙은 접속 주소를 반드시 tls/스킴으로 처리하고, 브릿지 자체의 Zenoh 설정 파일에도 클라이언트 인증서(mTLS) 를 박아 넣음으로써 완성된다. 결과적으로 데이터는 SROS2(로컬) \rightarrow 브릿지(암호 해제 및 재포장) \rightarrow Zenoh TLS(클라우드) 라는 물리적으로 단절된 두 개의 방탄 터널을 통과하게 된다.
