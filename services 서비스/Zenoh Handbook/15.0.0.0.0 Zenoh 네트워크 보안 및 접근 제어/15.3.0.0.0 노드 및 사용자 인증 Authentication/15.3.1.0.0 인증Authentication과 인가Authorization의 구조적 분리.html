15.3.1 인증(Authentication)과 인가(Authorization)의 구조적 분리

신분증을 확인하는 데스크와, “비밀금고” 출입 열쇠를 주는 데스크는 완벽하게 쪼개져야 한다.

0.1 [인스펙션] 쌍두마차 격리 통제 전술

Zenoh 코어는 이 두 가지를 단 하나의 시스템 플러그인에서 처리하지 않고 2개의 계층(Layer)으로 찢어 놓았다.

1. 1차 관문: 인증 (AuthN, Authentication)

  • “너는 누구인가?”
  • 가장 밖에서 작동한다. 접속자가 보낸 패킷이 암호(Password) 건 인증서(TLS Certificate) 건 간에, 그 값이 1비트라도 틀리면 즉각 TCP/QUIC 소켓 연결 자체를 드랍(Tear-down)시킨다.
  • 연결 단계(Session Establish)에서만 1회성으로 벌어지는 검문이다.

2. 2차 관문: 인가 (AuthZ, Authorization / ACL)

  • “들어온 네 녀석이 어느 서랍을 열 수 있는가?”
  • 1차 관문을 통과한 클라이언트에게 “사용자 이름(Username)“이라는 딱지표(Token)를 부여한다.
  • 이후부터 이 클라이언트가 z_put 혹은 z_get 패킷을 날릴 때마다, 라우터 내부의 ACL 플러그인이 패킷 1개 1개 단위로 딱지표와 토픽 이름을 대조해 낸다(Access Control).

인증이 무너지면 시스템 전체가 뚫리지만, 인가가 무너지면 특정 데이터 베이스 한 개만 뚫리는 수준의 방어선 설계다. 아키텍트는 인증을 가장 무자비하게, 인가를 가장 촘촘하게 짜야 한다.
