15.0 Zenoh 네트워크 보안 및 접근 제어

가장 완벽하게 연결된 네트워크(Fully Connected Network)란, 해커에게 가장 넓은 침투 경로(Attack Surface)가 열려있다는 뜻과 같다.
인터넷 위에서 로봇과 서버를 투명하게 소통시키는 혁신적인 프로토콜인 Zenoh. 바로 그 유연함 때문에 아키텍트에 의한 철저하고도 파멸적인 수준의 방패(Shield)가 필요하다.

이 15장에서는 “데이터를 어떻게 보낼 것인가“라는 질문을 완전히 버리고, **“누가 감히 이 데이터를 보내고 요구하는가?”**에 대한 검문과 살육(Drop)의 런북(Runbook)을 시작한다. 암호화 트래픽 설계부터 인증, 그리고 가차 없는 ACL 통제벽을 치는 방법을 통해, 당신의 Zenoh 망을 접근 불가능한 요새(Fortress)로 격상시켜라.
