2.7.5 공유 메모리(Shared Memory) 트랜스포트를 이용한 노드 간(IPC) 통신 가속화

인공지능 로보틱스를 구동하는 자율주행 프로세스와 이를 화면에 모니터링해 주는 UI 프로세스가 비록 동일한 한 대의 리눅스 컴퓨터(Host OS) 안에서 구동 중임에도 불구하고, 과거 이 둘을 이어주려면 강제적으로 가상의 이더넷 소켓(Localhost TCP 등)을 열어 패킷을 그 허공의 운영체제(OS) 네트워크 스택으로 밀어 넣었다가 다시 받아내는 수고스러운 이중 복사(Double-copying Context Switch) 과정이 필요했다.

이는 시스템 코어의 CPU 자원을 극심하게 갉아먹는 비효율의 온상이었으나, 놀랍게도 이 불필요한 비용의 해결책마저 Zenoh의 ‘트랜스포트 계층(Transport Layer)’ 추상화에 완전히 담겨있다.

로컬 노드 통신의 혁신적 지름길인 ‘공유 메모리(Shared Memory)’ 모듈 기술이다. Zenoh를 구동하는 두 개의 어플리케이션(예컨대 파이썬 스크립트 기반 피어와 C++ 기반의 라우터)이 물리적으로 한 컴퓨터 내에서 마주치는 순간, 이들은 서로가 뿜어대는 스카우트 비콘을 똑똑하게 감지해 낸 뒤 “어라, 우리 둘이 같은 컴퓨터 메모리 블록, 동일 호스트 안에 살고 있네!“라는 사실을 깨닫는다.

그 즉시 TCP 소켓 같은 멍청하고 우회적인 통로의 스위치는 꺼버리고, 두 프로세스만이 공통 영역으로 쓰는 거대한 램(RAM)의 빈 공간에 직접 포인터 주소를 심어버리는 이른바 ‘제로-카피(Zero-copy) IPC(Inter-Process Communication)’ 트랜스포트로 모드를 즉각 강등(Fallback) 혹은 전환해 버린다.

수 기가바이트(GB)에 달하는 딥러닝 영상 처리를 한 컴퓨터 안의 여러 노드가 나눌 때조차, OS가 그 거대한 데이터를 복사하느라 단말기를 버리고 발버둥 치는 비통한 상황은 이제 영원히 작별하고, 램의 포인터 주머니만을 단 몇 바이트로 넘겨주는 진정한 광속(Zero Latency) IPC 체계를 이루어 낸 것이다.
