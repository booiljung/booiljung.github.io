2.7.3 직렬 통신(Serial), BLE 등 비표준 네트워크 계층 연동 원리

인터넷에 물린 거대한 백본이 우아한 TCP/UDP의 안락함 속에서 혜택을 본다면, 초소형 마이크로컨트롤러 단이 배치된 지독한 산업 현장은 전혀 다른 세계다. 라즈베리 파이부터 아두이노(Arduino)에 이르기까지, 와이파이나 이더넷 칩조차 달 여력이 없어 구리선 3가닥짜리 낡고 얇은 UART 시리얼(Serial) 통신에 몸을 기대거나, 저전력 블루투스(BLE, Bluetooth Low Energy), 심지어는 CAN Bus 통신 파이프 등에 데이터를 밀어 넣고 살아야 하는 처절한 환경에 놓이곤 한다.

과거에는 이런 하부 물리단의 센서를 클라우드망에 합류시키기 위해, 반드시 양측을 통역해 주는 더럽고 복잡한 특수 브릿지(Bridge) 게이트웨이 코드를 수백 줄로 제작해야 했다.

그러나 Zenoh는 앞장에서 깎아놓은 미니멀 와이어 구조의 경이로움 덕분에 물리 계층(Physical Layer)이 구리 선 한 가닥이든 블루투스 안테나든 완벽하게 무던해진다. 개발자는 코드 내에서 자신의 트랜스포트 계층(Transport)을 serial://dev/ttyUSB0라고 한 줄만 선언해 바꿔 주면 끝이다.

이 선언 하나로, 아무리 열악한 대역폭(예컨대 초당 9600bps 조차 간당간당한)을 쥐어짜는 환경일지라도 마법의 ‘배치(Batching)와 비동기 와이어 프로토콜 압축’ 로직이 스스로 스케일 타운(Scale-down) 되어 그 협소한 구리선 사이로 우아하게 프레임을 전개시킨다. 이것이 이더넷 케이블과 블루투스 전파가 단 한 치의 어설픔 없이 통일된 언어로 합류되는 진정한 ’비표준 망 포용 원리’이다.
