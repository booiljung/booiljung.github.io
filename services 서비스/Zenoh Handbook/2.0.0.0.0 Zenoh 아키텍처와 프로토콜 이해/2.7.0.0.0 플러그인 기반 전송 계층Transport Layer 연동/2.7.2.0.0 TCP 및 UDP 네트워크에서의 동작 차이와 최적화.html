2.7.2 TCP 및 UDP 네트워크에서의 동작 차이와 최적화

가장 대중적이고 핵심적인 대역인 TCP와 UDP 환경 위에서도 Zenoh는 각 프로토콜이 가진 철학과 특성에 맞추어 와이어 프레이밍 전략을 완벽히 다르게 채택하고 최적화의 한계선까지 몰아붙인다.

TCP(Transmission Control Protocol) 기반에서는 상대방의 수신 상태가 보장되고 바이트 스트림이 순차적으로 날아가는 안정적인 고속도로다. 따라서 Zenoh는 여기서 불필요한 에러 체크(Checksum)나 손실된 데이터 재요청 같은 로직을 전부 꺼버리고(Offload), 최대한 많은 데이터 조각들을 한 블록에 길게 밀어 넣는 데 총력을 다한다. 이때의 핵심 최적화 요소는 바로 무지막지한 배치(Batching)를 통해 네트워크 I/O 빈도를 낮춰버리는 스루풋(Throughput, 처리량) 극대화다.

반대로 UDP(User Datagram Protocol) 환경은 신호등조차 없는 비포장 오프로드다. 중간에 드론이 터널을 지나며 패킷 수천 개를 우수수 떨어뜨려도 누구도 책임지지 않는다. 이 혹독한 도로에서 Zenoh는 즉발적이고 지연 없는 초단기 전송의 강점을 백분 살리는 동시에 자신이 직접 프레임 속 재전송(Retransmission) 플래그를 켜서 잃어버린 데이터를 수면 위로 끌어올린다. 또한, UDP의 특권 중 하나인 ‘멀티캐스트(Multicast)’ 도구를 동적 노드 탐색(Discovery) 과정에 완벽히 활용하여 한 번의 울림만으로 반경 내의 수많은 네트워크 센서를 즉각 기상시킨다.

이토록 영리하게 스위칭되는 전송 계층 지능이, 한쪽은 철저하게 대역폭을 쥐어짜는 물류 트럭(TCP)으로, 다른 한쪽은 초저지연을 무기로 날아다니는 경비행기(UDP)의 진면목을 자유자재로 뽑아 쓸 수 있도록 관장해 준다.
