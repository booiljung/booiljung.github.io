2.7 플러그인 기반 전송 계층(Transport Layer) 연동

지금까지 우리는 Zenoh가 만들어낸 데이터 중심(Data-Centric) 철학과 라우팅의 마법에 대해 논했다. 이것이 인터넷의 고속도로를 설계하는 “길 찾기“였다면, 이번 장에서는 그 길을 실제로 달리는 자동차 바퀴, 즉 전송 계층(Transport Layer)에 대해 파고든다.

데이터가 아무리 멋진 이름표(키)를 달고 완벽한 라우터 지도를 가졌다 하더라도, 최종적으로 물리적인 비트가 상대방에게 넘어가기 위해서는 TCP, UDP와 같은 밑단 전송 프로토콜의 힘을 빌려야만 한다. 하지만 통신 환경은 극도로 이질적이다. 어떤 시스템은 방화벽으로 꽉 막힌 거대 클라우드 웹 브라우저 안에서 돌아가고, 어떤 센서는 인터넷망 근처에도 가지 못한 채 시리얼(Serial) 선 하나에 의지하거나, 심지어 두 프로그램이 컴퓨터 한 대 안에서 공유 메모리만을 써야 할지도 모른다.

Zenoh는 이 기상천외한 물리 계층의 다양성을 정면으로 돌파하기 위해, 네트워크의 밑바닥 바퀴를 언제든 갈아 끼울 수 있는 ’멀티 트랜스포트(Multi-transport) 아키텍처’라는 완벽한 플러그인 모듈 방식을 고안해 냈다. 이 장에서는 Zenoh가 어떻게 전 세계에 존재하는 모든 통신 배선 위로 무결점 수용력을 가져가며 완벽한 이식성을 담보했는지 하나씩 살펴본다.
