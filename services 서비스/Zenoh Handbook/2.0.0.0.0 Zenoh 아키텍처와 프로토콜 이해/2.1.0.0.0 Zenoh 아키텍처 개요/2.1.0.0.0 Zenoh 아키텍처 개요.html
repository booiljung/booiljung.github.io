2.1 Zenoh 아키텍처 개요

“네트워크가 느리다“는 변명은 더 이상 허용되지 않는다. 현대의 분산 시스템 아키텍트는 하드웨어의 물리적 한계를 탓하기 전에 자신들이 깔아 놓은 논리적 통신망(Middleware)의 병목을 의심해야 한다.

Zenoh(제노)는 단순히 기존의 REST나 MQTT를 대체하는 ’빠른 택배 기사’가 아니다. 이것은 인터넷 프로토콜(IP)의 낡은 관성을 완전히 깨부수고, “누구(Who/Where)에게 보낼 것인가” 대신 “어떤 데이터(What)를 다룰 것인가“에 집착하는 네임드 데이터 네트워킹(NDN) 사상을 산업계에 최초로 실용화시킨 거대한 ’전역 데이터 분배 플랫폼(Global Data Space)’이다.

이 절에서는 기존의 통신 패러다임이 왜 수십만 대의 군집 로봇 제어나 스마트 시티의 백본망에서 처참하게 붕괴하는지 그 구조적 저주(Curse of Coupling)를 해부하고, 공간(Space)과 시간(Time)을 완벽하게 분리해 낸 Zenoh의 근본적인 아키텍처 사상을 철저한 실무 런북(Runbook) 관점에서 검증한다. 설계의 근원을 이해하지 못하고 복사-붙여넣기 한 코드는 결국 재앙으로 튀어 오른다.
