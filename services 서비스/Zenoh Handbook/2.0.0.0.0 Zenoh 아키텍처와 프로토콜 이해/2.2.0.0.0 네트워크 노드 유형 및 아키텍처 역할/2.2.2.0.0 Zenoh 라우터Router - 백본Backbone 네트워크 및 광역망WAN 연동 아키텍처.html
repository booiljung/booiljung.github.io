2.2.2 Zenoh 라우터(Router) - 백본(Backbone) 네트워크 및 광역망(WAN) 연동 아키텍처

라우터(Router)는 피어(Peer)의 강화 진화형(Super Node)이자 인프라를 지탱하는 척추(Backbone)다.
애플리케이션(파이썬, C++ 앱)단과 결합되어 구동되는 피어와 달리, 오직 “통신 패킷 스위칭과 데이터 캐싱 기능” 한 우물만 파는 독립적인 백그라운드 데몬 프로세스(zenohd 바이너리)다.

0.1 라우터의 핵심 책무

  1. 플러그인 확장 포털: 스토리지 백업(Storage Manager), REST API 어댑터, WebServer 지원 등의 거구적 아키텍처는 오직 ‘라우터’ 모드에만 탑재할 수 있다.
  2. 글로벌 터널(Tunnel) 개통: 클라이언트가 뿜어낸 데이터를 TCP/QUIC 롱-폴링 커넥션을 통해 해외의 AWS/GCP 라우터 인스턴스까지 실어 나른다.

0.2 [Runbook] 라우터 클러스터 이중화 및 브레인 스플릿 방어

데이터센터에 라우터를 1대만 두는 것은 바보나 하는 짓이다. 도커(Docker)로 라우터를 3대 띄워 결속(Mesh/Clique)시켜 고가용성(HA)을 달성한다.

// [zenohd.json5] 메인 라우터 A의 위상 설정 (B와 C를 바라봄)
mode: "router",
connect: {
  endpoints: [
    "tcp/router-B.local:7447",
    "tcp/router-C.local:7447"
  ]
}

[아키텍처 경고] 브레인 스플릿 (Split-Brain) 조율:
A, B, C가 엮여서 동작하다가 C가 인터넷 단절로 끊겼다가 다시 들어온다. 그 사이 A와 B에 저장된 “로봇 온도 캐시“와 독립된 C에 저장된 “로봇 온도 캐시“가 다르다면?
Zenoh 라우터는 합의 알고리즘(Gossip)과 HLC(Hybrid Logical Clock; 벡터 시계열) 타임스탬프를 대조하여, 나중에 다시 결합했을 때 ’어느 쪽의 이벤트가 진짜 최신 본인가’를 수학적으로 검증해 동기화(Sync)하는 무결점 클러스터 힐링(Healing) 병기가 내장되어 있다. 당신은 단순히 저 connect 설정만 맵핑해주면 된다.
