2.2 네트워크 노드 유형 및 아키텍처 역할

Zenoh 네트워크는 단일 형태의 데몬으로만 굴러가지 않는다. “마이크로컨트롤러(MCU)부터 데이터센터 클라우드 환경까지 아우르겠다(Continuum)“는 철학을 달성하기 위해, Zenoh는 디바이스의 하드웨어 체급과 네트워크 관여도에 따라 각기 다른 ’직업’을 부여한다.

이 직업은 하드코딩된 것이 아니며, 실행 시점에 파라미터(CLI Args)나 JSON 설정 파일을 통해 동적으로 부여할 수 있다.
어떤 노드를 뼈대(Router)로 삼을 것이고, 어떤 노드를 손발(Client)로 삼으며, 무정지 상태를 위해 어떤 노드들을 엮어(Peer) 춤추게 할 것인지 인프라 오케스트레이션(Orchestration)의 승패가 바로 이 노드 타입 분배 설계에서 결정 난다.

본 절에서는 Router, Peer, Client의 명확한 구조적 차이와 트러블슈팅 포인트, 그리고 이들이 방화벽 너머 상대를 찾기 위해 날려대는 스카우팅(Scouting)의 원천 기술을 심층 분석한다.
