2.3.4 네트워크 파티셔닝(Partitioning) 대응 및 동적 재구성(Dynamic Reconfiguration)

전쟁터나 해저 혹은 거대 스마트 팩토리 한복판에서, 포크레인이 팩토리를 가로지르는 메인 광케이블을 끊어버렸다 치자. 어제까지만 해도 하나로 평화롭게 통신하던 1만 대의 로봇망이, 이제 강제 물리력에 의해 A 구역(5천 대, 클라우드 연결 살아있음)과 B 구역(5천 대, 클라우드 연결 끊겨 고립됨)으로 산산조각 났다.

0.1 스플릿 브레인(Split-brain) 상태의 자생적 진화

이것을 네트워크 파티셔닝(Network Partitioning)이라 한다. 기존 아키텍처 기반의 B 구역 로봇들은 브로커가 사라졌기에 즉시 전멸(Halt)한다.
그러나 Zenoh의 Peer와 Router들은 물리적 케이블이 끊기는 그 즉시, 당황하지 않고 지도를 다시 그린다(Dynamic Reconfiguration).

  1. B 구역의 라우터와 피어들은 “아, 클라우드로 가는 길목 Node가 증발했군.“이라고 인지한다.
  2. 하지만 B 구역 내부에 있는 자기들끼리의 로컬 지도는 여전히 유효하다.
  3. 결과적으로 B 구역 5천 대의 로봇끼리 조향, 충돌 방지 제어를 주고받는 Pub/Sub은 1초(라우팅 재계산 틱)의 지연 이후 완벽하게 정상 가동을 이어간다.

0.2 [Runbook] 데이터 정합성 충돌의 사후 봉합 (Merge)

광케이블 수리팀이 도착해 선을 다시 이었다. A 망과 B 망이 하나로 융합되는 드라마틱한 순간, 악몽이 시작될 수 있다.
파티션이 쪼개진 동안 B 구역에 있던 스토리지 라우터에 쌓인 과거 데이터(/robot1/status=broken)와 A 구역에서 뻘짓하며 저장된 과거 데이터가 하나의 스토리지로 통합될 때 충돌(Conflict)이 일어나는 현상이다.

HLC(Hybrid Logical Clocks) 기반 충돌 제압:
Zenoh는 이 분단 사태를 철저히 예견하여 모든 패킷 원형에 HLC (Hybrid Logical Clock)라는 ’물리+논리 시간’의 조합 도장을 찍어 보낸다.
네트워크가 결합하여 2개의 과거 캐시가 충돌할 때, 타임스탬프만 대조하는 것이 아니라 “논리적 인과관계 벡터(Logical Vector Clock)“를 비교(Resolution)하여 한 치의 오차도 없이 어떤 것이 최신 사건인지 자동으로 병합(Merge)해 낸다. 완벽한 분산 힐링 시스템이다.
