2.3 네트워크 토폴로지와 라우팅 메커니즘

수백 개의 디바이스가 뭉치기 시작하면 그 관계망(Topology)은 더 이상 단순한 선(Line)이 아니라 생태계를 닮아간다. 일반적인 네트워크 프로토콜(TCP, HTTP)은 트리형이나 성형(Star) 같은 고정된 뼈대에 애플리케이션 구동을 강제로 끼워 맞추려고 한다.

그러나 숲길이 막히면 샛길로 트래픽을 돌려야만 하는 진보된 로보틱스 관제망이나 실시간 IoT 시스템에서는 이러한 딱딱한 라우팅 방식은 자살 행위에 가깝다. Zenoh는 망의 형태를 런타임에 구부려 버리고(Plasticity), 망 속성에 맞춰 데이터를 어떻게, 어느 길로 포워딩할지 매 비트(Bit) 최적의 판단을 내리는 지능형 최적 경로 탐색 알고리즘을 소유하고 있다.

이 절에서는 망이 끊어져도 살아남는 기적(Split & Healing)을 구현하는 P2P Mesh 병합 기법과, 데이터의 이름(Name)을 묻고 따지지도 않고 경로를 트는 네임 기반(Name-based) 라우팅의 원리를 해부한다. 당신 조직의 인프라 도면이 다이내믹한 생물체로 거듭나는 시발점이다.
