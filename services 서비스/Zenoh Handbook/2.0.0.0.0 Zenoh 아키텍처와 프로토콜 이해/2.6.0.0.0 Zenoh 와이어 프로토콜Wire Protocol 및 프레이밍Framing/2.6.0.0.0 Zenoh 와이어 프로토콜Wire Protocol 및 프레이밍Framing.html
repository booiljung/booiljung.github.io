2.6 Zenoh 와이어 프로토콜(Wire Protocol) 및 프레이밍(Framing)

아키텍처의 철학이 아무리 눈부시다 한들, 실제로 네트워크의 심해를 타고 넘어가는 데이터 패킷의 생김새, 즉 와이어 프로토콜(Wire Protocol)이 뚱뚱하고 무겁다면 그 미들웨어는 결코 산업의 혁명을 주도할 수 없다.

우리가 앞서 1장에서 그토록 찬양했던 ‘최소한의 오버헤드(Minimal Overhead)’ 원칙이 눈앞에서 기적처럼 살아 움직이는 공간이 바로 이 극단적으로 다이어트 된 와이어 프로토콜 계층이다. Zenoh의 창시자들은 인터넷 통신의 수십 개가 넘는 규약들을 참고하면서도, 무선 센서의 1바이트조차 아껴야 하는 처절한 현실을 반영해 바이트 스트림의 패키징 구조를 뼛속부터 재설계했다.

이 절에서는, 어떻게 Zenoh가 단 4~5바이트 수준의 헤더만으로 “이게 무슨 데이터고 누구에게로 가야 하는지“를 명시하는 마법의 프레이밍(Framing) 구조를 만들어냈는지 바이트 단위로 현미경 분석을 시도한다. 아울러 MTU(Maximum Transmission Unit) 한계에 걸리는 거대한 비디오 영상 스트림을 우아하게 잘라 보내고 다시 붙이는 단편화(Fragmentation)와, 자잘한 패킷 1,000개를 한 상자에 몰아넣고 배송 트럭을 한 번만 돌리는 배치(Batching) 처리 기술을 통해 네트워크 트래픽 효율이 우주 끝까지 극대화되는 마술을 경험하게 될 것이다.
