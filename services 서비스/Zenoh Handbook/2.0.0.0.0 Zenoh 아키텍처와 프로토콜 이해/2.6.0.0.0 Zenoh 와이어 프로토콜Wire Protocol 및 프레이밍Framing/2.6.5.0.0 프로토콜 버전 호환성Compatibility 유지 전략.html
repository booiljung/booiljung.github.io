2.6.5 프로토콜 버전 호환성(Compatibility) 유지 전략

프로토콜의 생애 주기를 마주하는 아키텍트들의 가장 큰 공포는 버전을 올릴 때마다 기존에 깔아두었던 모든 인프라를 한꺼번에 갈아엎어야 하는 ‘호환성 단절(Compatibility Break)’ 현상이다. 로봇의 두뇌는 어제 릴리즈된 최신 프로토콜 버전을 채택했는데, 공장 바닥에 깔려있는 센서들이 구형 프로토콜로 남아 통신 언쟁을 벌이다가 패킷을 뱉어내는 끔찍한 현상이 바로 이것이다.

Zenoh의 창시자들은 이런 대참사를 기술적으로 해결하기 위해, 와이어 프로토콜 헤더의 4바이트 중 극히 소중한 한 톨의 비트 영역을 오로지 ‘메이저-마이너 버전 협상 테이블’ 공간으로 양보하고 할당했다.

이는 서로 생면부지의 피어(Peer) 노드들이 동적 노드 탐색(Discovery) 과정에서 마주쳤을 때, 상대방이 옛날 구버전의 Zenoh를 구동하고 있다면 거기에 맞는 옛날 와이어 포맷으로 자동으로 스위칭하여(Fallback) 말을 다르게 걸어주는 놀라운 다운그레이드-호환 기술을 선사한다. 그 반대의 경우라도 상위 버전의 노드가 똑똑하게 스펙을 강등시켜 패키징을 맞춰준다.

이로 인하여 Zenoh는 버저닝 제약을 두려워하지 않는 놀라운 레거시(Legacy) 병합 아키텍처 생태계, 말하자면 ’낡은 피를 버리지 않고 포용하는 완벽한 수혈 능력(Forward & Backward Compatibility)’이라는 기적으로 완성되었다.
