2.6.3 배치(Batching) 처리를 통한 네트워크 효율 극대화 방안

만약 앞 절의 ’단편화’가 엄청난 대용량 영상을 여러 대의 트럭에 강제로 조각으로 분리해 실어 보내는 고통이라면, 그 극단적인 반대의 고민을 풀어야 할 때도 닥친다. 1초에 무려 1만 개의 온도 변화를 감지해 아주 짧은 10~20바이트씩을 송출하는 고주파수(High-frequency) 센서가 뿜어대는 수만 개의 모래알 패킷들이다.

모든 작은 알맹이마다 일일이 통신 엔진의 와이어 포장(FramingHeader)을 두르고 와이파이 안테나로 전파 스위칭을 켠 뒤 날려대는 짓을 1,000번 반복하는 것은 그야말로 엄청난 대역폭의 시스템 호출(System Call) CPU 자원 살인과 배터리 폭식을 재촉하는 행위다.

이를 방지하기 위해 Zenoh에는 기막힌 발상의 배치(Batching) 즉, 묶음 전송 처리 로직이 통로 입구에 마련되어 있다.

Zenoh 엔진 내에서는 개발자가 보낼 데이터를 바로 네트워크에 쓰지 않고, 극히 미세한 밀리초(milliseconds) 단위의 찰나 동안 임시 화물 상자(배치 버퍼, Batch Buffer)에 데이터를 켜켜이 눌러 담는 기다림의 미학을 구사한다. 그리고 이 상자의 MTU 덩치가 가득 차오르면(혹은 배치 타이머가 종료되는 순간), 10~1,000개에 이르는 그 무수한 알맹이들을 오직 라우팅 ‘와이어 프레임(Wire Frame)’ 하나에 묶어 버석거리게 포장하여 단 번의 네트워크 I/O만으로 발송한다.

결과적으로 엄청나게 무거웠을 시스템 인터럽트 스위치(Context Switching)가 완벽한 침묵 속으로 잠시 숨죽이며 전체 무선 파이프라인의 엄청난 대역폭 절감 효율(Throughput Maximization)이라는 폭발적인 성능 향상을 이끌어 냈다.
