2.6.2 메시지 단편화(Fragmentation)와 재조립(Reassembly) 메커니즘

수 기가바이트(GB)에 달하는 머신러닝 영상 모델이라든가 고해상도의 압축되지 않은 카메라 스트림을 보내고 싶을 때, 네트워크 세상에는 그 어떤 슈퍼컴퓨터도 깰 수 없는 물리적 파이프라인의 넓이 제한선이 법(Rule)처럼 존재한다. 우리는 이를 통상 최대 전송 단위, 즉 MTU(Maximum Transmission Unit)라고 부른다(전통적인 이더넷에서는 1500바이트 남짓).

이 거대한 코끼리 덩어리가 통신선의 작은 파이프를 통과하기 위해서는 필연적으로 조각을 내고 자르는 도마 위의 수술 과정이 필요하다. 이 기술이 이른바 송신측의 고수준 단편화(Fragmentation)와, 이를 다시 수신측에서 바느질하여 이어 붙이는 데이터 재조립(Reassembly) 메커니즘이다.

Zenoh는 이 단편화를 외부 타사 프레임워크나 TCP 전송 계층의 하위 단에 전적으로 버려두지 않고, 자신의 코어 프로토콜 안에 직접 구현하여 네트워크의 한계를 초극했다.

조각난 메시지의 페이로드(Payload) 비트마다 자신의 분할 번호인 파편 식별 번호(Header Flag 속)가 순서대로 아로새겨진다. 수신측의 라우터 혹은 구독자 피어는 바다를 떠내려오는 수천 개의 조각들을 실시간으로 임시 캐시(메모리 테이블)에 담아둔 채 자신이 맞춰야 할 퍼즐 번호를 세기 시작한다.

퍼즐이 완벽히 이어 붙여 무결성이 확인되는 단 그 찰나의 순간에만 상위 어플리케이션으로 그 거대한 덩어리(Complete Blob)를 결합하여 넘겨준다. 이로써 외부 네트워크에서 소실되어 버린 중간중간의 조각으로 인해 앱 전체가 에러의 늪(Corrupted Frame Error)에 빠져 허우적대는 일이 완벽하게 원천 보장되고 처리된다.
