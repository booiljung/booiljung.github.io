2.5 통신 패턴의 추상화 및 다중 패러다임

보편적으로 하나의 네크워크 프로토콜은 단 하나의 통신 사상만을 강요한다. REST는 동기적인 “질문-답변“만을, MQTT는 일방적인 “퍼뜨리기“만을 지원한다.
그러나 자율주행과 거대 클라우드 팩토리가 융합된 IT 생태계에서는 초당 수십 편의 카메라 영상(Push) 속에 간헐적인 데이터베이스 조회(Pull) 쿼리가 마치 양자 얽힘처럼 공존해야만 시스템이 돌아간다.

Zenoh가 “미들웨어(Middleware)의 궁극적 스위스 아미 나이프“로 추앙받는 이유는 이 상반된 사상(Paradigm)들을 단 하나의 프로토콜 문법 안으로 이질감 없이 녹여 통합시켰기 때문이다.
이 절에서는 무지성 이벤트 남발과 폭력적인 랙(Lag)을 제어하기 위해 Pub/Sub 모델이 Zenoh 내에서 어떻게 통제되는지와 더불어, 분산 쿼리의 꽃이라 할 수 있는 Query Consolidation(다중 응답 병합), 그리고 노드의 죽음을 예견하여 방어하는 Liveliness(유언장) 토큰의 실전 매뉴얼을 파헤친다.
