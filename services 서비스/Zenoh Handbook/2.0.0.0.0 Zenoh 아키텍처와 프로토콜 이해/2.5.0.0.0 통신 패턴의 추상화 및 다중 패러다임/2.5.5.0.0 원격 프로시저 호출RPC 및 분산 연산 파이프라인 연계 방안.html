2.5.5 원격 프로시저 호출(RPC) 및 분산 연산 파이프라인 연계 방안

Zenoh가 아무리 데이터 중심(Data-centric)이라고 해도, 때로는 로봇에게 명시적으로 “브레이크를 잡아라!“라든가 “데이터베이스의 1번 테이블을 지워라!” 같이 액션(Action)을 지시하고 그 결과를 동기적으로 받아야 할 때가 있다.
이러한 RPC(Remote Procedure Call)의 욕구를 해소하기 위해 Zenoh는 일반적인 데이터 조회가 아닌 함수 호출(Function Invocation) 개념을 Queryable에 결합시켰다.

0.1 Queryable을 이용한 RPC 흉내 내기 (RPC-over-Zenoh)

Zenoh 네트워크에서는 서버 포트를 열고 클라이언트를 기다리는 식의 gRPC 서버를 만들 필요가 없다.
대신 에지의 로봇 프로그램이 특정 Key 값에 대해 “누군가 이 키를 물어보면 내 함수를 실행시켜서 결과를 던져주겠다!” 라고 선언해 두면 그것이 곧 RPC 서버가 된다.

[Runbook: 에지 카메라 해상도 제어 RPC 구현]

# [RPC 서버(수신) 영역 - 에지 라즈베리 파이]
# 'robot/camera/set_resolution' 라는 경로로 누군가 Get 쿼리를 던지고 
# 파라미터(value)를 주면 이 콜백 함수가 실행된다.

def on_resolution_change_request(query):
    # 쿼리의 파라미터 파싱
    target_res = query.parameters() 
    
    # 1. 실제 하드웨어 제어 로직 수행
    change_hardware_resolution(target_res) 
    
    # 2. 로직이 끝났음을 호출자에게 통보(응답)
    query.reply(query.key_expr, f"Resolution successfully changed to {target_res}")

# 해당 키에 대해 나를 "부를 수 있는 함수"로 네트워크에 퍼블리싱한다.
session.declare_queryable("robot/camera/set_resolution", on_resolution_change_request)

# [RPC 클라이언트(송신) 영역 - 클라우드 관제 대시보드]
# 대시보드 관리자가 해상도 1080p 버튼을 누름 
# (마치 REST API의 PUT/POST처럼 동기적으로 실행 결과를 기대함)

replies = session.get("robot/camera/set_resolution?1080p", timeout=3.0)
for reply in replies:
    # 에지 로봇이 뱉어낸 "성공" 메시지를 화면에 띄운다.
    alert(reply.sample.payload)

0.2 파이프라인 연계와 gRPC 한계 돌파

이 구조의 파괴적인 장점은, robot/camera/set_resolution 쿼리를 던졌을 때 해당 쿼리어블(함수)을 켜둔 로봇이 10대라면 10대의 로봇 해상도가 동시에 1080p로 바뀌고, 동시에 10대의 [성공] 리포트가 클라우드로 병합(Consolidation)되어 날아온다는 것이다.

단 한 줄의 커넥션 없이, 단 한 줄의 클라우드 API 게이트웨이(Gateway) 설정 없이 ’1:N 멀티캐스트 원격 함수 제어’가 완벽하게 성립된다. 군집 로봇 제어나 거대 스마트 빌딩 조명 제어의 백업 파이프라인으로 이보다 거침없는 구조는 지구상에 존재하지 않는다.
