2.5.1 출판-구독(Publish-Subscribe) 패턴 - 비동기 데이터 푸시(Push)

비동기 메시징의 상징인 Pub/Sub 패턴의 진정한 가치는 “내가 쏘는 데이터를 누가 받는지 나는 몰라도 된다“는 완벽한 익명성(Anonymity)과 디커플링(Decoupling)에 있다.
100Hz로 라이다 페이로드를 밀어내는 로봇 퍼블리셔가, 구독자가 0명이든 10,000명이든 동일하게 1번의 Send 연산만 수행할 뿐이라는 점이 이 패턴의 핵심이다. (나머지 분배 연산은 라우터가 수행한다)

0.1 [Runbook] Pub/Sub 설계 시의 대역폭 붕괴(OOM) 방어 요강

센서 주기가 1ms인 데이터를 네트워크에 뿌리는데, 구독 프로그램(Python 등)의 파싱(Parsing) 속도가 100ms라면? 단 1초 만에 파이썬 프로세스의 운영체제 TCP 수신 버퍼 큐(Rx Queue)가 꽉 차버린다. 이때 퍼블리셔마저 블로킹(Blocking)을 먹고 로봇 관제 전체가 Freeze 되는 대재앙이 일어난다.

[트러블슈팅 1] Congestion Control (혼잡 제어) 강제화
망이 느리거나 버퍼가 찰 때 퍼블리셔의 행동 원칙을 규정하라.

  • Block: [안티 패턴] 구독자가 느리면 퍼블리셔도 멈춰서 기다린다. 로봇 팔 제어 망에서는 자살 행위다.
  • Drop: [권장] 수신 측이 못 따라오면 최신 데이터 1개만 남기고 과거 버퍼에 쌓인 데이터는 무자비하게 날려버린다(Lossy). 비전 스트림이나 텔레메트리 데이터의 기본값이다.

[트러블슈팅 2] Express 모드 (초저지연 우회 차선)
일반적인 Pub/Sub은 라우팅 신뢰성 검사(Reliability)를 거친다. 하지만 속도가 목숨보다 중요한 군집 주행 회피 컨트롤 데이터라면? 안전벨트를 풀고 Express 모드로 바꿔야 한다.

// 일반적인 큐잉을 생략하고 MAC 계층을 향해 데이터를 냅다 다이렉트로 던진다.
// 패킷이 유실될 확률은 높아지지만, 레이턴시(ns)는 로컬 변수 대입 수준으로 짧아진다.
session.put("robot/brake", "ENGAGE").express().res().await.unwrap();

“정확하게 도착하는 1초 전의 브레이크 신호보다, 좀 유실되더라도 0.001초 전에 도착하는 최신 브레이크 신호 하나가 차를 살린다.” 이것이 에지 네트워크 Pub/Sub 튜닝의 실존주의적 원칙이다.
