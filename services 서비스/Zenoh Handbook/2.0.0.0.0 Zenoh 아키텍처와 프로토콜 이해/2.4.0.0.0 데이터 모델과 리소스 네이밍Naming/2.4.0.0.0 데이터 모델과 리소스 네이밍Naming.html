2.4 데이터 모델과 리소스 네이밍(Naming)

Zenoh 통신망을 지배하려면 인프라 서버 조작 능력 이전에 “이름(Name)“을 지배해야만 한다. 앞선 장에서 강조했듯 Zenoh는 철저한 ’네임드 데이터 네트워킹(NDN)’이다. 당신 조직의 개발자들이 토픽(Topic)이나 리소스의 이름을 체계 없이, 그때그때 생각나는 대로 중구난방 짓는 순간 라우터의 트라이(Trie) 엔진은 혼란에 빠지고 네트워크 보안 장벽(ACL)은 형해화 된다.

이 절에서는 단순히 “URL처럼 슬래시(/)를 쓰면 된다“는 기초 수준을 넘어, 데이터를 글로벌 캐시에 효율적으로 보관(Storage)하고, 무제한에 가까운 필터링(Wildcard) 기능을 악용없이 활용하여 “망 자체가 분산 SQL 질의어처럼 기능하도록” 돕는 고도의 엔터프라이즈 네이밍(Naming) 런북 가이드를 제공한다.
