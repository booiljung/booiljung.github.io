2.4.1 키-값(Key-Value) 기반의 글로벌 데이터 공간(Global Data Space) 추상화

REST API 서버에 쿼리를 날리려면 먼저 그 서버의 도메인(e.g., api.factory.com/v1/sensors)을 입력해야 한다. 반면 Zenoh는 글로벌 데이터 백본 전체를 거대한 하나의 해시맵(HashMap) 형태의 공간으로 가상화(Virtualization)시켜 놓았다. 이것이 바로 글로벌 데이터 공간(Global Data Space)이다.

0.1 통신망을 하나의 “분산 변수(Global Variable)” 처럼 다루기

C++ 개발자가 메모리에 배열 var x = 10;을 선언하고 필요할 때 print(x);로 읽어가는 과정과, Zenoh 클라이언트가 유럽의 로봇에 데이터를 올리고 한국에서 뽑아보는 과정이 ’논리적으로 100% 동일’하다.

// [TypeScript 런북: 변수처럼 다루는 통신망]

// 1. Write: 유럽의 센서가 데이터를 글로벌 공간이라는 가상 해시맵에 무심히 던져 넣는다.
await session.put("world/europe/sensor/temp", 25.5);

// 2. Read: 한국의 관제 앱이 (호스트 주소 없이) 글로벌 해시맵의 Key만 조회한다.
const replies = await session.get("world/europe/sensor/temp");
const temp = replies[0].sample.value; // 25.5

이 엄청난 추상화는 개발자를 분산 시스템의 그물에서 해방시킨다. 이 “세계구급 해시맵“을 구현하기 위해 Zenoh 데몬들은 물밑에서 Gossip 프로토콜로 키 목록을 동기화하고, 최적화된 트리를 통해 요청값을 대리(Proxy)로 찾아다 주는 완벽한 짐꾼 파이프라인으로 작동하고 있는 것이다.
