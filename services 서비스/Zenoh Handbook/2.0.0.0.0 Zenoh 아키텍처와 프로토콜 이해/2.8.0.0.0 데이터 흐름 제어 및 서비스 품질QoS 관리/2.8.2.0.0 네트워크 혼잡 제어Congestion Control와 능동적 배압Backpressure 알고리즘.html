2.8.2 네트워크 혼잡 제어(Congestion Control)와 능동적 배압(Backpressure) 알고리즘

만일 고성능 라이다 100만 화소 센서가, 데이터를 아주 빠르고 완벽한 신뢰 채널을 통해 초당 수십 기가바이트씩 라우터 통로로 밀어 넣는 데 이웃의 무선(Wi-Fi) 전송 처리 속도가 병목에 걸려 고작 1초에 10메가바이트 남짓밖에 처리해 내지 못한다면 무슨 일이 벌어질까? 라우터 칩셋의 비좁은 메모리 큐(Queue)는 금세 용량이 초과되어 터져버리기 직전(OOM: Out-Of-Memory)이 될 것이고, 결국 라우터는 다운되며 그 이후의 데이터 파이프는 전부 불타버리게 된다. 이를 통신 공학에서는 네트워크 혼잡 고립(Network Congestion Failure)이라고 일컫는다.

그 수많은 미들웨어 프레임워크가 처참히 무너져 내린 이 정체 구간을 돌파하기 위해 Zenoh는 댐에서 수위를 조절하듯 스스로 수문을 여닫는, 이른바 ‘액티브 백프레셔(Active Backpressure, 적극적 배압 조절)’ 알고리즘이라는 기막힌 엔진 제어 기법을 도입했다.

    participant 데이터발행자_센서
    participant Zenoh라우터_버퍼큐
    participant 느린_수신자_원격DB

    데이터발행자_센서->>Zenoh라우터_버퍼큐: (초당 1GB 폭격!) 데이터 퍼블리시
    Zenoh라우터_버퍼큐--x느린_수신자_원격DB: (병목 발생) "천천히 줘, 나 못 받아!"
    Note over Zenoh라우터_버퍼큐: 라우터 내부의 버퍼 큐가 가득 참(95%)
    Zenoh라우터_버퍼큐->>데이터발행자_센서: 🚨 배압 발생! "잠깐 멈춰!" (TCP Window 0 수축 통보)
    데이터발행자_센서-->>데이터발행자_센서: 앱에서 발행 차단 대기 (Blocking/Drop 결정)

이 경이로운 배압 제어 기술은 다음과 같이 작동한다. 하위 말단의 라우터가 앞단의 병목으로 큐(Queue) 메모리가 가득 차올라 숨이 턱밑까지 차는 상황을 감지하면, 백프레셔 신호를 즉각적으로 이전 노드(네트워크 상위 방향, 발행자 쪽)로 강하게 역류(Back-pressure)시킨다! 그 신호를 받은 직전 라우터나 혹은 데이터의 최초 퍼블리셔(센서)는 자신의 퍼블리시 함수 호출을 즉시 블로킹(Blocking) 시키거나, 사전에 지정된 QoS 등급이 무시형(Drop)이라면 발행하려던 패킷을 허공으로 날려 스스로 숨통을 틔운다.

네트워크 전체의 파이프라인 수압 상황을 스스로 뒤로 전파함으로써 중앙 통제 없이도 시스템 자체의 마비(Crash)를 자생적으로 막아내는 혁신! 이것이 Zenoh가 데이터센터에서부터 마이크로컨트롤러까지를 관통하는 무결점 생존망 설계로 존경받게 하는 강력한 엔진의 튜닝 비결이다.
