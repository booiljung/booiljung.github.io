2.8.5 시간 동기화 문제 해결 및 분산 타임스탬프(Timestamp) 활용 방안

분산된 장비들(로봇, 컴퓨터 등)이 모여 하나의 군집(Swarm)을 이루게 되면 가장 치명적으로 부딪히는 이슈가 바로 물리적 기기들 간의 ‘시간(Time) 고립 및 오차’ 현상이다. 로봇 A의 내장 시계는 오후 1시 정각을 가리키는데, 네트워크 파티셔닝으로 인해 한 시간 동안 분리되어 있던 로봇 B의 시계는 배터리 방전 문제로 12시 59분에 멈춰 있다면 둘 사이는 어떻게 시공간을 맞춰 진실을 겨룰 것인가?

만일 인터넷 규약 중 가장 흔하게 쓰는 전통적인 NTP(Network Time Protocol) 인터넷 시계 서버를 이용한다 한들, 폐쇄망 무선 조종이나 해저 드론 통신 등에선 외부 안테나가 없어 완전히 무력화된다. 나아가 앞 절에서 언급했던 “동일한 여러 로봇에게 뿌린 분산 질의(Query)에 대한 병합(Consolidation)“을 Zenoh 라우터가 수행하려 할 때, 두 로봇의 시계가 어긋나 있다면 과연 어느 것을 최신(Latest Data) 값이라 신뢰하고 하나로 뭉쳐 전달할 것인가!

이를 무결하게 타파하기 위한 최상위 설계 철학으로 Zenoh는 HLC(Hybrid Logical Clocks, 혼합 논리 클럭)라는 놀라운 분산 타임스탬프 매커니즘을 전면에 포용했다.

journey
    title HLC(혼합 논리 클럭) 기반의 시계 결합 마법
    section (로봇 A 시계: 1초, 하이브리드=3)
      로봇 A가 데이터 전송(3): 5: 발행
    section (로봇 B 시계: 0초 (과거 오차), 하이브리드=1)
      로봇 B가 데이터 수신: 5: 도착
    section 로봇 B의 논리시계 진화(Sync!)
      "아, A가 나보다 논리적으로 3이네. 내 물리 시간 무시하고 4로 즉각 업데이트한다!": 5: 타임 점프!

HLC는 단순히 기계 기판의 물리적 시간 장치가 아니라, 네트워크를 타고 “나 여기까지 몇 번 주고받았어“라는 논리적인 횟수 카운터 비트(Logical Counter Bit)를 시간과 교배(Hybrid) 시켜 탄생시킨 마법의 시계다.

로봇 A가 B에게 통신할 때, B의 하드웨어 시계 고장이 생겼다 한들 이 HLC 카운터 꼬리표를 맞교환하는 과정을 통해서 전 세계를 구성하는 수십만 대의 노드들이 인터넷 시간 서버가 없어도 서로 “네가 먼저, 내가 나중!“이라는 선후 인과 관계의 시간 흐름을 자생적으로 동기화할 수 있다. 그 어떤 통신망보다 절대적인 인과성(Causality)을 무결점 상태로 보장하며 흐름에 대한 데이터 순서의 지배자를 선언하게 된 것이다.
