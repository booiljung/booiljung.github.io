Chapter 8. TypeScript에서의 Zenoh

클라우드 네이티브(Cloud-native) 백엔드부터 브라우저 렌더링을 담당하는 프론트엔드 환경에 이르기까지 전 방위적인 생태계를 구축한 TypeScript(및 JavaScript)는 현대 웹과 엔터프라이즈 시스템 구축을 위한 사실상의(De-facto) 표준이다. 그러나 V8 엔진의 싱글 스레드 이벤트 루프(Single-threaded Event Loop) 모델은 전통적으로 초고주파수의 텔레메트리(Telemetry) 데이터 스트리밍이나 저지연(Low Latency) 통신망 구축에 근본적인 설계상 한계를 지녀왔다.

Zenoh는 이러한 한계를 극복하기 위해 Rust 코어의 강력한 성능을 네이티브 바인딩 및 웹어셈블리(WebAssembly)로 포팅하여, TypeScript 개발자들이 Node.js와 웹 브라우저 양방향에서 완벽한 비동기 분산 네트워킹을 구현할 수 있도록 zenoh-ts 패키지를 제공한다. 본 장(Chapter 8)에서는 전통적인 HTTP 폴링(Polling)이나 상태 저장형(Stateful) WebSocket의 구조적 결함을 타파하고, 진정한 의미의 서버리스 실시간 데이터 버스(Serverless Real-time Data Bus)를 구축하는 전술을 제시한다.

2. 고성능 비동기 스트리밍과 메모리 관리 패턴

엄격한 가비지 컬렉터(Garbage Collector)의 통제를 받는 TypeScript 환경에서 로보틱스급 센서 데이터를 유실 없이 처리하기란 대단히 까다롭다. 이를 해결하기 위해 async/await 및 비동기 이터레이터(AsyncIterator)를 기반으로 한 무한 이벤트 스트림 루프의 구현 기법을 소개한다(8.7). 나아가 대량 수신 시 발생하는 메모리 점유의 폭증을 억제하기 위한 백프레셔(Backpressure) 패턴과 스로틀링(Throttling) 제어 로직을 통합한다.

더불어 JSON 퍼서(Parser)의 연산 오버헤드를 대체할 수 있는 Protobuf, FlatBuffers 등 구조화된 바이너리 전송 방식(8.8)과 고속 Uint8Array 파싱 기법(8.4)을 다룬다.

3. 프론트엔드 최적화 및 마이크로서비스 아키텍처 연동

프론트엔드 렌더링에 있어서 컴포넌트의 생명주기(Lifecycle)와 구독(Subscribe) 이벤트가 일치하지 않으면 심각한 메모리 누수를 촉발한다. 본 장의 후반부에서는 React 환경에서 커스텀 훅(Custom Hook) 패턴을 창안하여 UI 프레임워크와 Zenoh의 상태를 안전하게 결합하는 로봇 텔레메트리 대시보드 구축 모범 사례를 제공한다(8.9).

또한, 백엔드 영토에서는 NestJS 혹은 Express와 같은 웹 서비스 플랫폼 위에 하이브리드 이벤트 드리븐(Event-driven) 아키텍처를 세우고(8.10), 기존의 HTTP RESTful API를 Zenoh 분산 쿼리(Query)망과 브릿징(Bridging)하는 체계적인 연동 방안을 모색할 것이다. 이를 통해 시스템 아키텍트들은 레거시 웹 환경을 가장 신뢰할 만한 데이터 중심의 실시간 분산 시스템으로 끌어올릴 수 있는 완벽한 청사진을 획득하게 된다.
