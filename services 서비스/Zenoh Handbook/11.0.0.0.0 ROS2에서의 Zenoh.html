Chapter 11. ROS2에서의 Zenoh

로봇 운영체제(ROS2, Robot Operating System 2)는 자율주행, 제조, 해양, 우주 항공 등 현대 지능형 로보틱스 시스템을 구축하는 사실상의(De-facto) 글로벌 표준 컴퓨팅 프레임워크로 자리매김했다. 하지만 ROS2의 기반 통신 미들웨어로 채택된 DDS(Data Distribution Service)는 로컬 네트워크(LAN) 환경에서의 우수한 성능에도 불구하고, 대형 멀티캐스트(Multicast) 트래픽 발생, 이기종 네트워크(WAN/5G) 확장 한계, 그리고 마이크로컨트롤러 환경 적용의 어려움이라는 구조적 한계를 안고 있다.

차세대 분산 라우터 네트워크인 Zenoh는 데이터 중심(Data-centric) 아키텍처 철학을 바탕으로 DDS의 맹점을 보완하는 데 그치지 않고, 로봇 내부의 노드 통신부터 글로벌 클라우드 관제 시스템(Fleet Management)까지 단일 프로토콜로 통합할 수 있는 혁신적인 통신 백본을 제공한다. 본 장(Chapter 11)에서는 ROS2 생태계에 Zenoh를 이식하고 통합하는 최적의 설계 원리 및 실전 가이드를 전개한다.

1. ROS2 통신의 구조적 난제와 Zenoh의 브릿징 아키텍처

로보틱스 개발자는 단일 로봇 단위의 통신을 넘어 다중 로봇 운영(Swarm Robotics) 환경을 직면하게 되며, 이때 발생하는 통신 프로토콜 간의 이질성을 극복해야 한다.

본 장은 Zenoh를 ROS2에 통합하는 두 가지 핵심 메커니즘을 해부한다. 첫째는 기존 DDS 시스템을 훼손하지 않으면서 국지적 네트워크를 WAN으로 브릿징(Bridging)하여 연장하는 zenoh-bridge-dds(11.3) 접근법이다. 둘째는 기존 ROS2의 RMW(ROS Middleware) 계층 자체를 대체하여 멀티캐스트 없는 고속 전송을 네이티브(Native)하게 구현하는 rmw_zenoh(11.4) 심층 분석이다. 두 아키텍처 중 어떤 상황에서 어느 방식을 선택해야 하는지 정량적인 기준과 성능 지표(11.7)를 제시한다.

2. QoS, 보안 시스템, 고대역폭 센서 무결성

LiDAR의 대역폭 융단폭격(Point Cloud)이나 고해상도 카메라 스트리밍은 언제나 네트워크 패킷 드롭(Packet Drop)을 유발하는 원인이다.
이를 극복하기 위해 패킷 조각화(Fragmentation)와 커스텀 시리얼라이제이션(Serialization) 기법을 소개하고(11.6), QoS(Quality of Service) 규칙이 Zenoh의 배압(Backpressure) 알고리즘과 어떻게 치밀하게 매핑되는지 증명한다.

더 나아가 SROS2의 암호화 체계가 Zenoh의 권한/접근 제어망(Access Control List, ACL)과 연동되는 절차(11.8)를 살펴보고, 브라우저 단에서 rosbridge라는 무거운 스택 없이 순수 TypeScript 기반 통신으로 ROS 토픽을 조작하는 직접 연결(Direct Integration) 기법(11.9)까지 확장함으로써 로보틱스 시스템 아키텍처의 한계를 거침없이 파괴하는 강력한 비전을 공유할 것이다.
