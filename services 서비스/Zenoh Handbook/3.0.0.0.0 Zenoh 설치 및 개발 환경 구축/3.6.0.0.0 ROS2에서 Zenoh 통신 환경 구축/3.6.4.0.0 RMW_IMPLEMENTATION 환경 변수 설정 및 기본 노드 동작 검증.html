3.6.4 RMW_IMPLEMENTATION 환경 변수 설정 및 기본 노드 동작 검증

수술 대에 올려 RMW Zenoh 인공 심장을 박아 넣었어도, 로봇의 뇌(운영체제)는 여전히 습관적으로 기존 DDS 심장을 뛰게 하려고 시도한다. 개발자가 OS에 대고 “앞으로 우리 통신의 근육은 모두 Zenoh-RMW 만을 쓸 것이다!” 라고 선언(Declaration)해 주지 않으면 이 패키지는 투명 인간 꼴을 면치 못한다.

0.1 [Runbook] 영구적인 RMW 교체 선언

특정 터미널에서만 치고 마는 것이 아니라, 로봇의 시동 스크립트(.bashrc 혹은 도커 ENTRYPOINT)에 전역 주사를 놓아야 한다.

# 1. 방금 우리가 빌드한(혹은 설치한) 패키지의 아티팩트 링크 활성화
source ~/zenoh_ws/install/setup.bash

# 2. [가장 중요한 치트키] RMW 타겟 변환 강제
export RMW_IMPLEMENTATION=rmw_zenoh_cpp

# 주의사항: 위 명령어를 ~/.bashrc의 최하단에 박아넣어서 부팅시마다 
# 로봇의 모든 노드가 DDS 대신 Zenoh를 물고 기동하도록 영구 봉합시켜라.
echo "export RMW_IMPLEMENTATION=rmw_zenoh_cpp" >> ~/.bashrc

0.2 [검증 런북] 데스매치 테스트 - ROS2 노드 간 교신 정합성

정말 DDS가 죽고 Zenoh 위로 트래픽이 흐르는지 두 눈으로 확인해야 한다.

터미널 A (Talker 모드 구동)

ros2 run demo_nodes_cpp talker

터미널 B (Listener 모드 구동)

ros2 run demo_nodes_cpp listener

[아키텍처 인스펙션] 너희 정말 Zenoh 쓰는 거 맞아?
화면에 텍스트가 잘 오간다고 안심하지 마라. htop 이나 Wireshark를 띄워라.
만약 터미널에서 ros2 topic echo /chatter 를 쳤을 때, Wireshark에서 239.255.0.1 같은 DDS 특유의 미친 듯한 RTPS 멀티캐스트 폭풍 패킷이 잡힌다면 당신의 RMW_IMPLEMENTATION 변수는 먹지 않은 것이다 (실패!).
반대로 조용히 유니캐스트 TCP/UDP 세션 몇 가닥만 깔끔하게 떨어지거나 Zenoh 헤더 플래그가 보인다면, 축하한다. 이제 당신의 로봇 편대는 디스커버리 오버헤드로 인한 와이파이 폭파 위협에서 완전히 영구 면제(Exempt)되었다.
