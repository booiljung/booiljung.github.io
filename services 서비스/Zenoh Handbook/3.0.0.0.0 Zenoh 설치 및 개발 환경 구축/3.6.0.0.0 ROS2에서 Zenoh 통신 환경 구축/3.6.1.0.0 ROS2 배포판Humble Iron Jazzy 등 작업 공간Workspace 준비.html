3.6.1 ROS2 배포판(Humble, Iron, Jazzy 등) 작업 공간(Workspace) 준비

Zenoh 플러그인이나 RMW는 C/C++ 및 Rust의 소스코드로 엮어지므로, 단순히 apt install로 깔고 끝나는 애플리케이션이 아니다. ROS2의 최상위 빌드 시스템인 colcon 워크스페이스 내에서 타겟 로봇에 맞춰 교차 빌드(Cross-build)를 준비해야 한다.

0.1 [Runbook] ROS2 버전 매트릭스 및 호환성 방어

ROS2의 배포판(Distro)마다 내부적으로 묶여 있는 Python 버전과 FastDDS/CycloneDDS의 버전이 다르므로, Zenoh 브릿지 빌드 시 타겟 ROS2 소스를 정확히 지칭(Source)해야 컴파일러가 심볼을 찾을 수 있다.

환경 고정 런북 (ex. Ubuntu 22.04 LTS / ROS2 Humble 기준)

# 1. 쉘을 열 때마다 다른 ROS 버전이 섞이지 않도록 환경변수 록인(Lock-in)
source /opt/ros/humble/setup.bash

# 2. 로봇의 뇌를 구성할 독립된 작업 공간(Workspace) 창설
mkdir -p ~/zenoh_ws/src
cd ~/zenoh_ws/src

# 3. 필수 빌드 종속성 일괄 설치 (ROS2 C++ 컴파일 도구 및 패키지 매니저)
sudo apt update
sudo apt install -y build-essential cmake git python3-colcon-common-extensions python3-vcstool

0.2 [트러블슈팅] 의존성 트리 붕괴 방어

colcon build를 치면 언제나 “어떤 패키지가 누락되었다“는 절망적인 에러가 뜬다. 이를 막기 위해 작업 공간의 모든 종속성을 ROS의 데비안 패키지 관리자인 rosdep으로 런타임 검진하는 절차를 습관화하라.

cd ~/zenoh_ws
# 내가 src 폴더에 다운받은 패키지들이 요구하는 라이브러리를 OS에 강제로 깔게 한다!
rosdep update
rosdep install --from-paths src --ignore-src --rosdistro humble -y

이 기본 영양토(Workspace)가 완벽하게 다져지지 않은 상태에서 Zenoh 플러그인 소스를 다운로드하는 것은 건물 기초 없이 시멘트만 들이붓는 격이다.
