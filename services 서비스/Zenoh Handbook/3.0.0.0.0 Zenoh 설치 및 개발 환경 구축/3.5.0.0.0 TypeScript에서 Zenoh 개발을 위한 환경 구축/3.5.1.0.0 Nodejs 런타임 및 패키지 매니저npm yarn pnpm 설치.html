3.5.1 Node.js 런타임 및 패키지 매니저(npm, yarn, pnpm) 설치

“우분투 저장소 apt install nodejs“를 사용하는 짓은 버그의 무덤을 파는 행위다. OS 기본 레지스트리에 있는 Node.js는 수년 전의 구형 버전일 확률이 높으며, 최신 Zenoh-TS가 의존하는 N-API(Node API) 네이티브 바이너리 결합(Rust Binding) 시 호환성 에러(ERR_NAPI_INVALID_COMPAT)를 무자비하게 뿜어낸다.

0.1 [Runbook] NVM을 통한 최신 런타임 격리 설치

인프라 팀은 Node.js의 버전을 NVM(Node Version Manager)으로 통제해야 한다.

# 1. NVM 백그라운드 설치 (루트 권한 불필요)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc

# 2. Zenoh N-API 네이티브 컴파일 안정성이 검증된 Node.js v20 이상(LTS) 강제 설치
nvm install 20
nvm use 20

# 3. 팀원 간 패키지 일원화를 위한 프로젝트 강제 (선택사항)
echo "20" > .nvmrc

0.2 [트러블슈팅] 의존성 호일링(Hoisting)과 pnpm

yarn 이나 npm 거대 모노레포(Monorepo) 환경에서 Zenoh를 받아오면 가끔 네이티브 모듈 경로 유실 사태가 벌어진다.
현대 클라우드 백엔드 환경에서는 pnpm 사용을 강력히 권장한다.

corepack enable
pnpm init

pnpm 시스템은 node_modules 내부의 심볼릭 링크(Symlink) 트리 구조를 엄격하게 잡기 때문에, 추후 Zenoh JS 바인딩이 C++ 네이티브 node-gyp를 엮을 때 참조 경로를 놓치는 재앙을 원천 방어한다.
