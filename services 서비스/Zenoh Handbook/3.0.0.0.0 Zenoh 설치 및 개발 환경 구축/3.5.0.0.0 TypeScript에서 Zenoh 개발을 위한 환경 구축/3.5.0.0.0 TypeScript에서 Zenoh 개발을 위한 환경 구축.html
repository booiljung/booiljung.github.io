3.5 TypeScript에서 Zenoh 개발을 위한 환경 구축

백엔드 라우터 단에서 아무리 나노초(ns) 단위로 트래픽을 최적화한들, 최종 사용자가 바라보는 관제 웹 대시보드(Frontend)가 먹통이거나 옛날 데이터를 렌더링하고 있다면 그 인프라는 실패한 것이다.
Zenoh를 최신 클라우드 생태계와 딥 뉴럴넷 관제망에 녹여내기 위해선 TypeScript/JavaScript 바인딩이 필수적이다.

문제는 JS 생태계 특유의 단일 쓰레드(Single-Thread) 비동기 이벤트 루프 병목과, CommonJS/ESM 간의 지독한 모듈 파편화다. 본 장에서는 프론트엔드와 백엔드(Node.js)를 아우르는 @eclipse-zenoh/zenoh-ts 패키지의 운영 수준(Production-Ready) 환경 세팅과 함정 회피 런북을 전개한다.
