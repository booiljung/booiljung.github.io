3.5.2 Zenoh TypeScript API(zenoh-ts) 패키지 설치

@eclipse-zenoh/zenoh-ts 는 단순한 자바스크립트 쪼가리가 아니다. 내부적으로는 Rust로 직조된 Zenoh 코어 엔진을 N-API(Node.js 네이티브 애드온)로 감싼 무거운 “네이티브 런타임 바이너리“다. 따라서 운영체제(Mac/Linux/Win)에 따라 설치 시 다운받아지는 코어(*.node 파일)가 달라진다는 점을 명심하라.

0.1 [Runbook] 설치 및 아키텍처 호환성 검증

1. 패키지 설치

# NPM이나 로컬 저장소에서 젠오 SDK 설치 
pnpm add @eclipse-zenoh/zenoh-ts

2. [경고] 크로스 플랫폼 도커(Docker) 배포 시 붕괴 참사
당신이 M1/M2 맥북(ARM64)에서 코드를 짜고 node_modules 폴더까지 통째로 압축해서 K8s(x86_64) 클러스터로 던져 올렸다고 치자.
배포 후 node dist/main.js 를 실행하는 순간 아래와 같은 끔찍한 데스 에러에 직면한다.

Error: cannot open shared object file: .../zenoh.linux-x64-gnu.node: invalid ELF header

[아키텍처 대응 방침]
네이티브 모듈 의존성 라이브러리(zenoh-ts)는 절대 OS를 가로질러 복사(Copy)하면 안 된다.
도커(Docker) Dockerfile 안에서 K8s와 운영 환경(Linux x64)과 동일한 OS 베이스를 두고, 컨테이너 내에서 pnpm install을 현지 다시치기(Re-install) 하여 네이티브 바이너리가 리눅스 x64용으로 정확히 꽂히도록 파이프라인(CI)을 설계해야 한다. 이거 하나 놓치면 금요일 밤 야근이 확정된다.
