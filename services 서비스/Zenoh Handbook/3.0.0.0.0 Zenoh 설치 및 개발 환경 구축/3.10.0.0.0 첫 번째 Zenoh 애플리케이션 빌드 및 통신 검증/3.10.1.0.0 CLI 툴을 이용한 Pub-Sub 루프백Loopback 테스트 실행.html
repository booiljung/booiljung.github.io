3.10.1 CLI 툴을 이용한 Pub-Sub 루프백(Loopback) 테스트 실행

방금 갓 깔아 둔 Zenoh 패키지가 tcpdump 레벨에서 똑바로 작동하는지, 복잡한 코딩 없이 zenoh-cli 바이너리 단 두줄로 검증하는 인프라 최소 무결성 테스트다.
로봇 개발자가 “네트워크망이 이상해요!“라고 당신(백엔드 인프라 팀)에게 클레임을 걸었을 때, 가장 먼저 이 명령어 방패로 당신 잘못이 아님을 증명(Alibi)할 수 있어야 한다.

0.1 [Runbook] CLI를 통한 멀티캐스트 & 브로커(Client-Router) 접속 테스트

터미널 창을 좌우로 두 개 분할 선점하라.

테스트 A: P2P(멀티캐스트) 망 생존 여부 타진

  1. (터미널 1/Sub): zenoh-plugin-sub --mode peer -k "demo/test"
  • 이 순간, 공유기 전체에 UDP 224.0.0.225 스카우팅 신호를 쏜다.
  1. (터미널 2/Pub): zenoh-plugin-pub --mode peer -k "demo/test" -v "Hello Zenoh"
  2. [성공 판정]: 터미널 1에 [demo/test] : Hello Zenoh 가 찍힌다.
  • 실패 시 조치: 도커 안이거나 공유기 포트(UPnP/IGMP)가 차단된 것이다. 3.2.x 챕터의 방화벽 설정으로 돌아가라.

테스트 B: 정적 IP(라우터 경유) 릴레이 생존 타진

  1. 로컬에 라우터를 하나 백그라운드로 켠다. zenohd &
  2. (터미널 1/Sub): 이제는 peer 비콘을 포기하고 라우터에 대고 얌전히 접속한다.
    zenoh-plugin-sub --mode client -e tcp/localhost:7447 -k "demo/test"
  3. (터미널 2/Pub):
    zenoh-plugin-pub --mode client -e tcp/localhost:7447 -k "demo/test" -v "Hello Server"
  4. [성공 판정]: 터미널 1에 정상 도달 확인.
  • 실패 시 조치: zenohd가 7447 포트 점유를 못 했거나 방화벽 인바운드가 막힌 것이다.

이 zenoh-plugin-pub/sub 툴은 심지어 초당 throughput과 페이로드 사이즈를 테스트할 수 있는 매개변수(-p, -s)를 제공하므로, 통신 스트레스 테스트의 핵(Core)으로 활용하라.
