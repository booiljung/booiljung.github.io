3.10.3 다음 챕터를 위한 기본 템플릿 프로젝트 생성

당신이 만약 사내 개발팀의 인프라 혹은 데브옵스(DevOps) 담당자라면, 방금 성공한 빈 껍데기 코드를 그저 “잘 되네” 하고 휴지통에 버려선 안 된다.
이 껍데기가 바로 내일부터 수십 명의 개발자가 Clone 해가서 비즈니스 로직을 붙여야 할 ‘사내 표준 Zenoh 스캐폴딩(Scaffolding) 템플릿’ 이기 때문이다.

0.1 [Runbook] 사내 공용 모노레포(Monorepo) 보일러플레이트 규격화

다음 장(Chapter 4 이상)의 본격적인 하드코어 프로그래밍을 위해, 바탕화면에 zenoh-standard-fleet 이라는 템플릿 디렉터리를 만들고 아래 구조로 형상 관리(Git)를 고정하라.

zenoh-standard-fleet/
├── infra/                  # [인프라 통제 구역]
│   ├── docker-compose.yml  # 라우터(zenohd) 및 InfluxDB 캐시 통합 컨테이너
│   └── zenohd.json5        # 사내 공용 라우팅 ACL 및 스토리지 튜닝 설정 파일
├── edge-robot-rust/        # [에지 디바이스 개발 구역]
│   ├── Cargo.toml          # 버전이 고정된 zenoh = "0.10.1-rc"
│   └── src/
│       └── main.rs         # 센서 퍼블리셔/Queryable 콜백 기본 템플릿
├── cloud-dash-ts/          # [클라우드 백엔드 구역]
│   ├── package.json        # ESLint 및 ESM("type": "module") 설정 완료
│   ├── tsconfig.json       # Top-level Await 및 ESM 설정 
│   └── src/
│       └── index.ts        # Consolidation 파라미터가 들어간 기본 Get/Sub 쿼리 템플릿
└── README.md               # 3장의 방화벽 개방 룰 및 실행 런북(명령어)가 적힌 가이드
