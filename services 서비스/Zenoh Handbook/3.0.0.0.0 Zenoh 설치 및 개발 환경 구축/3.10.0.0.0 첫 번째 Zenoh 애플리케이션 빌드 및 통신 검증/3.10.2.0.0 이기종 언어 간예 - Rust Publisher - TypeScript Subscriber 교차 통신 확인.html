3.10.2 이기종 언어(Rust Publisher - TypeScript Subscriber) 교차 통신 확인

CLI 도구로 검증을 마쳤다면, 이제 이 책의 지향점인 Edge(Rust) to Cloud(TypeScript) 의 이기종 파이프라인(Heterogeneous Pipeline)을 코드로 박아넣을 차례다.

0.1 [Runbook] 이기종 통합 데스매치 테스트

1. 에지 애플리케이션 (Rust Publisher)
센서의 역할을 영기할 Rust 코드를 작성한다.

cargo new edge_sensor && cd edge_sensor
cargo add zenoh tokio --features tokio/full

// src/main.rs
use zenoh::config::Config;
use std::time::Duration;

#[tokio::main]
async fn main() {
    // 로컬 망의 Peer 혹은 라우터를 자율 스카우팅
    let session = zenoh::open(Config::default()).res().await.unwrap();
    let publisher = session.declare_publisher("robot/1/battery").res().await.unwrap();

    loop {
        // 끊임없이 ZBytes(Untyped) 형태로 베어링 데이터를 난사!
        publisher.put("85%").res().await.unwrap();
        println!("Sent battery status.");
        tokio::time::sleep(Duration::from_secs(1)).await;
    }
}

2. 클라우드 관제 대시보드 (TypeScript Subscriber)
웹소켓 서버의 역할을 대리할 TypeScript 측 모듈을 작성한다.

# npm/pnpm 초기화 
npm init -y
npm i @eclipse-zenoh/zenoh-ts typescript ts-node

// index.ts
import * as zenoh from '@eclipse-zenoh/zenoh-ts';

async function main() {
    console.log("Connecting to Zenoh Global Data Space...");
    const session = await zenoh.open();

    // 이기종의 파편화된 메모리 주소(Rust -> C -> V8 JS)를 관통하고 도착한 데이터의 파싱!
    await session.declareSubscriber("robot/1/battery", (sample) => {
        const value = sample.payload.toString('utf-8');
        console.log(`[Alert] 로봇 1번 배터리 최신 캐시: ${value}`);
    });
}
main();

[아키텍처 인스펙션]
각각의 터미널에서 cargo run과 ts-node index.ts를 실행하라.
1초마다 Rust가 쏘아 올린 "85%"라는 5바이트짜리 문자열이, 통신 큐 버퍼와 ABI(Application Binary Interface) 장벽을 박살 내고 V8 자바스크립트 엔진의 힙(Heap) 메모리 안으로 다이렉트로 투하되는 광경이다. 변환 서버(Proxy) 따위는 존재하지 않는다.
