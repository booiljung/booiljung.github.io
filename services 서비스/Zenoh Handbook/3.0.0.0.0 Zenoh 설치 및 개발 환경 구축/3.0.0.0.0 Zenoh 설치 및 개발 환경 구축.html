Chapter 3. Zenoh 설치 및 개발 환경 구축

이론적으로 완벽한 아키텍처를 이해하는 것과, 실제로 내 개발 환경에서 라우터를 띄우고 코드를 작성하여 두 대의 컴퓨터 간 데이터를 주고받는 것은 완전히 다른 차원의 이야기다. Chapter 2에서 우리는 Zenoh(제노)가 어떻게 와이어(Wire)를 설계하고, 라우팅을 수행하며, 공간과 시간을 추상화해 내는지 그 깊은 코어 철학을 파헤쳤다. 이제는 그 이론을 바탕으로 실제 코드와 인프라의 세계로 진입할 차례이다.

본 장, Chapter 3에서는 막연했던 Zenoh 생태계를 독자의 로컬 머신(Windows, macOS, Linux) 위에 견고하게 뿌리내리는 과정을 다룬다.
Zenoh의 진정한 강력함은 단순히 특정 언어 하나에 종속되지 않고 시스템 프로그래밍 언어의 최강자인 Rust와 C/C++부터, 프론트엔드 및 백엔드 개발자의 무기인 TypeScript와 Python, 그리고 거대한 로봇 생태계의 표준인 ROS 2에 이르기까지 모든 언어와 플랫폼을 하나로 엮어내는 광활한 바인딩(Binding) 지원에 있다.

이 장을 통해 독자들은 다음의 목표를 달성하게 될 것이다.

  1. 코어 인프라 구축: 글로벌 라우팅의 심장인 zenohd(Zenoh Router) 데몬과 네트워크 테스트를 위한 강력한 도구인 핵심 CLI(Command Line Interface)들을 설치하고 가동한다.
  2. 멀티 랭귀지(Multi-language) 셋업: Rust, C/C++, TypeScript, Python 등 각기 다른 프로그래밍 언어별로 최적화된 Zenoh 클라이언트 라이브러리를 설치하고 프로젝트 빌드 환경을 세팅한다.
  3. 로보틱스 토대 마련: Zenoh-DDS 브릿지를 통해 ROS 2 시스템이 Zenoh 네트워크에 합류할 수 있는 길을 뚫어낸다.
  4. 관측 가능성(Observability) 확보: 단순히 데이터를 쏘는 것을 넘어, 네트워크 내부에서 무슨 일이 벌어지는지 눈으로 확인할 수 있도록 모니터링 및 시각화 인프라를 구축한다.

이어지는 각 절을 순서대로 따라오거나, 자신이 주로 사용하는 개발 언어와 환경에 맞춰 발췌독을 진행해도 무방하다. 모든 환경 구축이 완료되면, 3장의 마지막에서 각기 다른 언어로 짜인 애플리케이션들이 하나의 단일한 Zenoh 데이터 버스(Data Bus) 위에서 완벽히 대화하는 첫 번째 경이로운 통신 검증을 경험하게 될 것이다.
