3.7.2 시계열 데이터 수집을 위한 Prometheus 서버 연동 설정

자, 라우터가 8000번 포트로 내뱉는 저 수만 줄의 매트릭스 텍스트는 이 모니터링 생태계 서사시의 딱 절반이다. 그저 브라우저로 접속해 눈으로 보기만 할 뿐, 아무리 많은 트래픽 저하 정보가 방출되더라도 시간이 지나면 그 가치는 허공으로 소실(Drop)되고 만다.

우리는 네트워크 아키텍트로써 이 소실될 데이터를 절대적 가치로 만들기 위해 위대한 감시용 창고, **프로메테우스(Prometheus)**의 절대 권력을 모셔와 연동의 퍼즐을 끼워 넣어야 한다.

프로메테우스의 철학은 내가 수천 대의 로봇망과 서버 클러스터를 다 거느린다 한들, 그들이 먼저 자신에게 보고(Push)하게 내버려두지 않고, 본인이 수 초에 한 번씩 감시 대상 병사들의 입에 귀를 대고 “지금 데이터 뭐야?“라고 강제로 빨아들이는(Scraping) 그 무지비한 풀(Pull) 전능적 권력 체계에 있다.

journey
    title 라우터 생명선과 프로메테우스 융합 작전
    section 1단계: 프로메테우스 덤프 서버 기상 
      "Docker Image나 설치 패키지를 내려받아": 5: 서버 기동
    section 2단계: 크롤러(Scraper) 연동 스크립트 도화지
      `prometheus.yml` 에 수술의 메스 대기: 5: 텍스트에디터
    section 3단계: Zenoh 타겟 저격 삽입!
      "가서 라우터의 8000포트를 매 5초마다 미친듯이 긁어와!": 5: 영구수집 
      "수개월치 데이터가 덤프 디스크에 영구 적재!": 5: 빅데이터화

여러분의 프로메테우스 관제 서버 안에 자리 잡고 있는 심장, 즉 prometheus.yml 파일에서 scrape_configs: 섹션의 밑에 단 4줄의 암살자 좌표를 더하라.

  - job_name: 'zenoh_core_router' # (내가 이름붙인 이 로봇 클러스터의 명칭)
    scrape_interval: 2s # 2초마다 심박수를 빨아들이는 강행군 
    static_configs:
      - targets: ['내_라우터_IP_혹은_도메인:8000']

이 연동이 끝난 채 프로메테우스 데몬을 재기동하는 순간, Zenoh 플러그인이 아무리 우스꽝스러운 기계어 형태의 매트릭스 텍스트로 중얼거려도 프로메테우스의 괴물 엔진은 그 포맷(OpenMetrics 표준)을 완벽하게 파싱(Parsing)해 자신의 메모리 큐를 관통시켜 시계열 분산 디스크인 TSDB에 차곡차곡 무수히 겹겹이 쌓아 올린다.

당신의 라우터가 뻗어버리는 대참사가 났을 때, 전날 밤 4시 2분에 큐가 얼마나 펌핑(Pumping) 되었는지를 영원토록 돌려볼 수 있는 신들린 관제 백업 저장소 파이프라인이 마침내 개통된 것이다.
