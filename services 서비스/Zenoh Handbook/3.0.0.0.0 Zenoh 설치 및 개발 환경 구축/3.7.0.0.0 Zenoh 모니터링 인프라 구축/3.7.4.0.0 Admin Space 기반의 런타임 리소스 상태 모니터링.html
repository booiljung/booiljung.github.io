3.7.4 Admin Space 기반의 런타임 리소스 상태 모니터링

거대한 프로메테우스 인프라를 지어놓고 대시보드를 켰다고 해서 끝이 아니다. 만약 엔지니어인 당신이 낯선 원격 라우터 한 대에 접속하여, 번거로운 외부 설정 하나 없이 즉석에서 “저 라우터와 지금 몇 대의 피어(Peer) 노드들이 연결되어 있고, 그놈들은 지금 무슨 토픽을 퍼블리시 중이며, 라우터가 배포하고 있는 핵심 환경 변수는 무엇일까?“를 날것 그대로 까발려 보고 싶다면 어떡해야 할까?

마법 같게도 Zenoh 코어에는 이 모든 은밀한 내부 상태값과 수치들을 특권층(Admin)이 언제든 네트워크로 훔쳐보고 제어할 수 있도록 개방해 둔 신성한 공간, 이른바 **어드민 스페이스(Admin Space)**라는 신기루 같은 네임스페이스(Namespace) 구역이 기본 예약되어 있다.

이 공간은 모든 라우터가 메모리에 자기만의 시스템 현황을 키-값(Key-Value) 구조의 살아있는 데이터로 등록해 둔 금고다. 보통의 데이터 키 이름들(예: sensor/temp)이 오가는 평범한 퍼블릭망과 격리되어 가장 권위 있는 슬래시(@) 심볼 밑으로 숨어 있다. 이 세계에 접근하려면, 그저 터미널에서 zenoh-cli의 검색 권력을 휘두르기만 하면 수 문이 열린다.

# 내 라우터 엔진에 연결된 수백 대의 모든 피어(기계 친구들) 리스트를 모조리 내놓아 실시간 덤프 명령!
z_get -k '@*/router/*/peer/*'

당신이 이 쿼리(Query) 한 줄을 허공에 날려 보내면, 즉시 라우터망은 이 신성불가침의 공간의 봉인을 풀고, 자신에게 다가와 꽂혀 있는 수많은 피어들의 UUID, 아이피 주소 스펙, 그리고 연결 상태가 기록된 가장 내밀한 상태 텍스트 덩어리를 엄청난 속도로 터미널로 복사(Dump)해버린다.

여기서 더 나아가 “어떤 노드가 어떤 데이터를 지금 뱉어내고 있는지”, 즉 라우팅 테이블(Routing Table)을 까짓것 직접 엿보거나(@*/router/*/session/*/subscriber/*), 심지어 어드민 권한으로 이 공간에 값(Value)을 써넣어(z_put) 운영 중인 라우터의 설정을 재기동 없이 라이브(Live)로 뜯어고쳐 버리는 경악스러운 튜닝(Runtime Reconfiguration)의 기적마저 이 어드민 스페이스 안에서는 허락된다.

엔지니어는 이 신의 영역을 활용하여 제어 불능에 빠진 라우터의 멱살을 잡고 심문하여, 그 단단한 껍질 속 생태계의 모든 기밀을 티끌 한 점 남김없이 모니터링의 도마 위로 꺼낼 수 있다!
