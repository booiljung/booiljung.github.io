3.7 Zenoh 모니터링 인프라 구축

당신의 개발팀이 만일 세계 최고의 속도를 자랑하는 Zenoh 통신 엔진을 라우터에 성공적으로 실어 서버에 올렸다고 치자. 수천 대의 로봇 1만 개의 센서가 데이터를 뿜어내며 라우터를 마구 두드리기 시작하는데, 정작 당신의 관제 시스템은 눈을 가린 채로 라우터가 지금 CPU를 얼마나 먹고 있는지, 큐(Queue) 대기열에서 통로가 막혀 불쌍한 데이터들이 소실(Drop)되고 있지는 않은지 단 1비트도 대답할 수 없다면? 그것은 혁신이 아니다, 단지 불량 폭탄을 네트워크에 덩그러니 얹어놓은 미친 짓에 가깝다.

운영(Ops) 환경의 생명선은 무결점 철학과 그에 준하는 ’투명한 감시탑’이다.

이번 절에서는, 그저 침묵하며 패킷을 실어 나르던 무명(無名)의 Zenoh 플랫폼을 완벽하게 해부하고 실시간으로 스펙트럼 수치를 스캐닝(Scanning)할 수 있는 중앙 통제 인프라(Telemetry Pipeline)를 건설하는 가장 정석적인 여정을 밟아본다. 우리는 개발 도구가 아니라 진정한 현대 클라우드 옵저버빌리티(Observability)의 거인, ’프로메테우스(Prometheus)’를 전장에 끌고 올 것이다.

가장 기초적인 상태 플러그인(Metrics) 배양법부터, 시계열 데이터(Time-Series Data)를 영구 저장해주는 외부 시스템 연동, 나쁜 패킷 병목 구간을 잡아내는 레이턴시(Latency) 진단 스캐너 배치 전략, 그리고 프로토콜 그 자체의 심장을 훔쳐볼 수 있게 해주는 경이로운 Zenoh 고유의 특권—‘어드민 스페이스(Admin Space)’—를 활짝 열어젖히는 런타임 제어술까지, 어두운 백그라운드 프로세스에서 동작하는 Zenoh 서버의 맥박을 여러분의 콘솔창 속 모니터링 심박기로 빼곡히 찍어 올리는 정석을 전수한다.
