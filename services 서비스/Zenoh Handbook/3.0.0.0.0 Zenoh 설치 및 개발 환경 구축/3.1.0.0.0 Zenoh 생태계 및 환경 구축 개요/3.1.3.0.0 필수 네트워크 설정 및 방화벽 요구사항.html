3.1.3 필수 네트워크 설정 및 방화벽 요구사항

Zenoh 데몬을 켰는데 연결이 안 되는 이유의 99%는 당신 회사의 방화벽과 IPTables 세팅 탓이다. Zenoh는 기본적으로 아래 규약된 2개의 핵심 관문을 통과해야만 숨을 실수 있다. 보안팀에 인바운드/아웃바운드 개방 RFC를 날릴 때 이 문서를 복사해서 제출하라.

0.1 [Runbook] Zenoh 통신 방화벽 개방 포트 넘버

디폴트 설정 기준, 반드시 아래 프로토콜과 포트가 열려있어야 한다.

1. 유니캐스트 (Unicast) - 실제 데이터의 송수신 및 라우터 연결 통로

  • TCP 포트 7447: (필수) 클라이언트가 라우터를 향해 -e tcp/xxx 로 접속하기 위한 메인 고속도로.
  • UDP 포트 7447: (선택/권장) TCP의 트래픽 체증을 회피하는 QUIC 프로토콜의 통로. 클라우드 간 고속 비디오 스트리밍을 원한다면 UDP 7447도 세트로 열어야 한다.

2. 멀티캐스트 (Multicast) - Peer 간 자율 디스커버리 (스카우팅)

  • UDP 포트 7447 & 멀티캐스트 IP 224.0.0.225:
    P2P 망을 구성할 때, 동일 로컬/VLAN 망 내에서 서로를 수색하는 레이더용 패킷이 날아다니는 주소다.
    만약 K8s 내에서 Calico 등의 CNI(컨테이너 네트워크 패브릭)를 쓴다면, IGMP 스누핑(Snooping)에 의해 저 IP의 패킷이 막히지 않도록 인프라 엔지니어가 CNI 설정을 해제(Allow)해줘야 한다.

0.2 방화벽 트러블슈팅 런북 커맨드

어딘가 막힌 것 같다면 무지성으로 앱을 껐다 켰다 하지 말고, 즉각 tcpdump나 nc 커맨드로 포트 결착 여부를 증명하라.

# 1. 클라우드 라우터(서버)가 해당 포트를 열고 듣고 있는지(Listen) 확인
sudo netstat -tulpn | grep 7447
# 출력 예시: tcp6       0      0 :::7447          :::*           LISTEN      1234/zenohd

# 2. 클라이언트 PC(에지 로봇)에서 클라우드 라우터 측으로 7447 포트가 찔리는지 테스트
# "succeeded!"가 안뜨면 당신 앱의 문제가 아니라 그냥 망이 단절된 것이다. 방화벽 팀에 연락하라.
nc -zv 13.124.x.x 7447
