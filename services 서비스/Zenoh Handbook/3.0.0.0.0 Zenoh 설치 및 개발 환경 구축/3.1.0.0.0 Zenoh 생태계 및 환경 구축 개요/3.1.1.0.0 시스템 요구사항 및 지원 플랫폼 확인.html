3.1.1 시스템 요구사항 및 지원 플랫폼 확인

Zenoh는 본질적으로 크로스 플랫폼(Cross-platform)을 지향하지만, 하드웨어 사양에 따라 사용할 수 있는 바이너리의 ’격’이 다르다. 당신의 장비가 K8s 서버인지, 64KB 램을 가진 아두이노인지에 따라 빌드 전략을 다르게 취해야 한다.

0.2 컴파일 아키텍처 타겟 주의사항

만약 M1/M2/M3 맥북(ARM64)에서 빌드한 도커 이미지를 AWS의 x86_64 인스턴스에 밀어 넣으려 한다면, 크로스 컴파일(Cross-compile) 에러 혹은 exec format error라는 치명적 런타임 오류가 발생한다.
Zenoh 핵심 엔진 자체가 Rust로 하드웨어 최적화 컴파일을 수행하기 때문에, CI/CD 구축 시 반드시 타겟 아키텍처(x86_64-unknown-linux-gnu 또는 aarch64-unknown-linux-gnu)를 Docker Buildx나 Rust Target 명령어를 통해 명확히 크로스 컴파일하여 분리 리포지토리에 저장하라.
