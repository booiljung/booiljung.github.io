3.1 Zenoh 생태계 및 환경 구축 개요

분산 시스템의 붕괴는 런타임 코드가 아니라 패키지 설치와 환경 구축 단계부터 이미 싹트기 시작한다.
“제 PC에서는 잘 되는데요?” 라는 변명은 Zenoh의 세계에서 용납되지 않는다. 로봇 제어보드(MCU)부터 개발자의 MacBook, 그리고 클라우드의 Linux 컨테이너까지 완벽히 동일한 프로토콜로 묶여야 하는 에지-클라우드 컨티뉴엄(Continuum) 환경에서는 ’타겟 별 정확한 툴체인 종속성’이 아키텍처의 알파이자 오메가다.

이 장에서는 단순한 apt-get install 수준의 튜토리얼을 넘어, **‘운영 수준(Production-grade)’**의 안정성을 담보하기 위해 각 운영체제와 언어(Rust, C/C++, TypeScript, ROS2)별로 어떤 옵션을 켜고 어떤 방화벽을 뚫어야만 Zenoh 데몬이 방해 없이 구동될 수 있는지 그 근본적인 런북을 제시한다.

당신이 만약 AWS 방화벽에서 UDP 7447 포트를 열어두지 않고 이 장을 넘긴다면, 당신의 피어(Peer)들은 영원히 서로를 찾지 못하는 저주에 빠지게 될 것이다.
