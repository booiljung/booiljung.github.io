3.2.2 윈도우(Windows) 환경에서의 설치 및 환경 변수 구성

IoT 통신의 끝단에는 종종 파나소닉(Panasonic) 터프북이나 공장 라인의 PLC 터미널 제어용 Windows PC가 엮어 들어가기 마련이다. 안타깝게도 Windows는 리눅스처럼 깔끔한 패키지 매니저 인프라가 미흡하기에, 수동 바이너리 이식(Portable) 방식과 환경 변수 수동 매핑(Path)의 기초 런북을 철저히 따라야 한다.

0.1 단계: 바이너리 다운로드 및 무결성 파종

GUI 설치 마법사(Installer) 따위는 기대하지 마라.

  1. GitHub의 Eclipse Zenoh Releases 페이지 접속.
  2. x86_64-pc-windows-msvc.zip 배포본 다운로드.
  3. 압축을 풀고 바이너리(zenohd.exe, zenoh-plugin-pub.exe 등)를 운영체제가 훼손되지 않을 공간(예: C:\zenoh)으로 이동시킨다.

0.2 단계: 글로벌 PATH 환경 변수 주입 강제

이 단계를 빼먹고 개발 중인 C# 폴더 안에 zenohd.exe를 복붙해서 쓰는 짓을 반복하다간, 버전 파편화로 인해 DLL 충돌 버그의 무덤을 파게 될 것이다.

[명령 프롬프트(Admin) 자동화 스크립트 런북]

@echo off
REM Zenoh 바이너리 폴더를 영구적인 시스템 PATH에 때려박는 명령
setx PATH "%PATH%;C:\zenoh" /M
echo Zenoh is now globally accessible!

0.3 단계: Windows 방화벽 인바운드 예외 타공 (Hole Punching)

윈도우 디펜더 방화벽(Windows Defender Firewall)은 공장 Wi-Fi 대역에서 다른 노트북이 zenohd.exe의 7447 포트로 들어오는 것을 99%의 확률로 적대적 패킷으로 간주하여 파기(Drop)해 버린다.

[PowerShell 방화벽 강제 개방 런북]
관리자 권한 PowerShell을 열고 당신의 라우터가 숨을 쉴 허파를 강제로 찢어발겨 연다.

# TCP 7447 라우팅 포트 영구 개방
New-NetFirewallRule -DisplayName "Zenoh Router TCP" -Direction Inbound -LocalPort 7447 -Protocol TCP -Action Allow

# UDP 7447 멀티캐스트 (Scouting용) 및 QUIC 개방
New-NetFirewallRule -DisplayName "Zenoh Router UDP" -Direction Inbound -LocalPort 7447 -Protocol UDP -Action Allow

개발을 하다가 “이상하네 내 로컬 피어에선 붙는데, 옆자리 책임님 PC에선 접속이 안 돼요“라는 탄식이 터져 나올 때 가장 먼저 실행해야 할 치트키 커맨드다.
