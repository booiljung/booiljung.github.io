3.2.1 리눅스(Linux) 환경에서의 설치 및 시스템 데몬(systemd) 등록

우분투(Ubuntu)나 데비안(Debian) 서버를 클라우드 관제 라우터로 사용할 때, 매번 SSH에 접속해 수동으로 zenohd를 켜는 것은 폭탄을 안고 있는 것과 같다. 여기서는 apt를 통한 글로벌 바이너리 설치부터, 죽어도 자력으로 다시 일어나는 Systemd 불사조 세팅까지의 런북을 전개한다.

0.1 패키지 저장소 및 바이너리 설치

Eclipse 재단의 공식 apt 리포지토리를 시스템에 편입시켜야 추후 자동 보안 업데이트가 가능하다.

# 1. 서명 키와 리포지토리 소스 등록
echo "deb [trusted=yes] https://download.eclipse.org/zenoh/debian-repo/ /" | sudo tee -a /etc/apt/sources.list.d/zenoh.list

# 2. apt 캐시 업데이트 및 라우터 패키지 설치
sudo apt update
sudo apt install zenoh

0.2 [Runbook] Systemd 데몬 서비스 등록

공장 현장에서는 정전으로 서버가 꺼졌다 다시 켜졌을 때, 로봇 개발자가 붙기 전에도 라우터 허브가 먼저 살아나 있어야 한다.

1. 설정 파일 분리 배치:
설정 파일은 어플리케이션 디렉토리가 아닌 격리된 vhost 통제 구역에 두어라.

sudo mkdir -p /etc/zenoh
sudo nano /etc/zenoh/zenohd.json5
# 내용: mode: "router", 등 당신의 라우팅 설정 입력

2. 서비스 유닛 파일 작성:

sudo nano /etc/systemd/system/zenohd.service

[Unit]
Description=Eclipse Zenoh Router Daemon
After=network-online.target
Wants=network-online.target

[Service]
# 보안: root 대신 zenoh 전용 유저를 만들어 할당하는 것이 베스트다.
User=root
# 핵심: 바이너리 실행 경로 및 설정파일 파라미터 강제 먹이기
ExecStart=/usr/bin/zenohd -c /etc/zenoh/zenohd.json5
# 죽으면 5초 뒤에 백오프(Backoff) 무한 예초기 재시동!
Restart=always
RestartSec=5

# OOM이나 로그 폭주시의 시스템 제한 (옵션)
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target

3. 서비스 활성화 및 생사 확인 (Health Check)

sudo systemctl daemon-reload
# 부팅 시 자동 시작 등록
sudo systemctl enable zenohd
# 당장 데몬 가동
sudo systemctl start zenohd

# 제대로 살아서 로그를 뿜고 있는지 검증 
sudo journalctl -u zenohd -f

시스템 관리자는 이 세팅을 Ansible이나 Chef 플레이북으로 말아서 신규 에지 서버가 입고될 때마다 원크릭으로 배포하는 형상 관리를 이룩해야 한다.
