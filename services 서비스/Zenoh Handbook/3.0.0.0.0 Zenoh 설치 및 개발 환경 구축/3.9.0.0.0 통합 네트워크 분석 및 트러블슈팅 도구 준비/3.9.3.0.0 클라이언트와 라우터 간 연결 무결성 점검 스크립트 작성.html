3.9.3 클라이언트와 라우터 간 연결 무결성 점검 스크립트 작성

단순한 테스트용 단발성 관제를 넘어, 기업의 수만 대 로컬망(Edge Devices Cluster) 생태계 지휘관이라면 단순히 눈으로 와이어샤크나 그라파나 대시보드(Grafana Dashboard)의 신선도를 관찰하는 것에 만족해선 안 된다. 가장 전지전능한 인프라 관리자는 자신의 자리를 비우더라도 알아서 현장을 감시하고 자동으로 알람을 치는 로보틱스 오퍼레이터(Operator) 봇(Bot) 스크립트를 남긴다.

어드민 스페이스를 심문하는 ‘배시(Bash) + jq’ 자동화 기법
가장 우아하고 순수한 연결 무결성 감시 스크립트는 3.7.4절에서 익혔던 Zenoh 라우터 속의 특권 API인 Admin Space 상태 덤프 통로를 찌르고, 그 텍스트 데이터를 무지비한 유닉스 명령 도구인 jq(JSON 스위트 정렬 및 해체기)로 요리하여 1초 이내에 결함을 뽑아내는 콤보(Combo)로 완성된다.

#!/bin/bash
# (무결성 진단 데몬 스크립트: Check_zenoh_health.sh)

ROUTER_ADMIN_URL="http://192.168.1.100:8000/@/router/local/peer/"
EXPECTED_PEER_COUNT=150

echo "🔍 [Zenoh Health Analyzer] 라우터 피어 연결 상태 무결성 검증 착수..."

# 1. 100번 서버 라우터의 Admin Space를 HTTP GET (REST 뷰어) 방식으로 찌르고 가져온다!
RAW_JSON=$(curl -s $ROUTER_ADMIN_URL)

# 2. jq 파서를 거쳐, 순수하게 연결 살아있는 Peer들의 "개수(Count)"만을 도출해낸다.
CONNECTED_PEERS=$(echo $RAW_JSON | jq '. | length')

if [ "$CONNECTED_PEERS" -lt "$EXPECTED_PEER_COUNT" ]; then
    echo "🚨 [CRITICAL ALERT] 맙소사! 노드 탈락 현상 발견! 예상: $EXPECTED_PEER_COUNT, 현재 생존: $CONNECTED_PEERS"
    # (선택) 여기에 Slack이나 PagerDuty 웹훅 콜을 쏴서 잠자는 인프라 담당자 핸드폰을 깨운다!
    exit 1
else
    echo "✅ [ALL GREEN] 완벽합니다! 150대의 에지 로봇이 Zenoh 코어 라우터와 100% 무결점으로 결속되어 순항 중입니다."
    exit 0
fi

이 고전적이나 절대 지지 않는 감시 스크립트 파이프라인을 여러분의 AWS 이중화(EC2) 서버나 로컬 CronJob 데몬 껍데기 안에 이식하라. 이 스크립트가 매 10초마다 스스로 깨어나 당신의 150대 드론 중 한 대라도 숲속 기지국의 무선망 전파 영역(Coverage)을 이탈해 Zenoh 파이프를 놓치는 순간, 즉시 슬랙 알림과 휴대폰 문자 메시지를 날리며 전 무장 생태계 라우팅 파이프라인의 안전망으로 봉사할 것이다.
