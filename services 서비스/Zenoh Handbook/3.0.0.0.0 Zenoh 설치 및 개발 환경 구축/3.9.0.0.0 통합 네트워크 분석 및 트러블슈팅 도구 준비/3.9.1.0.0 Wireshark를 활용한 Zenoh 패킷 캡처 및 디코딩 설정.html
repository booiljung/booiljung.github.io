3.9.1 Wireshark를 활용한 Zenoh 패킷 캡처 및 디코딩 설정

인프라의 최하단, 케이블에 진짜로 전파가 흐르는지 증명하는 가장 투명한 도구는 무엇이든 데이터의 무덤까지 파헤치는 **와이어샤크(Wireshark)**다. 그런데 여러분은 앞서 2장에서 깨달았듯이, Zenoh 프로토콜의 와이어 프레이밍(Wire Framing) 엔진은 역사상 가장 완벽하게 오버헤드(Overhead)를 다이어트한 고작 4~5바이트 수준의 무식할 만큼 짧고 압축된 정수값 덩어리였다! 일반적인 패킷 스캐너는 이 녀석이 Zenoh의 선언인지 쓰레기 UDP 찌꺼기인지 전혀 구분하지 못하고 화면에 당황스러운 Data 덤프 텍스트만을 띄우게 된다.

이 끔찍한 가독성 지옥을 타파하고, 이 미세한 4바이트가 ’Subcribe 쿼리’인지, ’Liveliness Drop 토큰’인지 와이어샤크가 스스로 완벽하게 색깔을 덧입히며 인간이 읽을 수 있게 번역(Dissect)하도록, 이른바 루아(Lua) 스크립트 플러그인 주입 수술을 단행해야만 한다.

  1. Zenoh 공식 Dissector (해부기) 스크립트 발췌
    Eclipse의 흉포한 천재들은 이미 깃허브 레포지토리 어둠 속에 이 zenoh.lua라는 와이어샤크용 특수 돋보기 플러그인을 공개해 두었다. 여러분은 리눅스라면 ~/.local/lib/wireshark/plugins/ 혹은 윈도우라면 %APPDATA%\Wireshark\plugins\ 같은 각자의 은밀한 디렉터리에 이 루아 스크립트를 다운로드해 툭 던져 넣기만 하면 된다.

  2. Wireshark의 화려한 각성과 7447 포트 선언
    이 루아 파일이 적용된 채 와이어샤크를 재시작하고 udp.port == 7447 혹은 tcp.port == 7447 (Zenoh의 전세계 공통 대동맥 포트다!)라는 표시 필터(Display Filter)를 상단 검색바에 살포시 입력하고 엔터를 치는 순간!

journey
    title 맹인 와이어샤크의 광명 회복!
    section 이종의 쓰레기 패킷 덩어리
      [1a 0f ff ...] 이게 대체 뭐야?: 1: 절망의 HEX
    section zenoh.lua 와이어샤크 플러그인의 결합 
      포트 7447번 패킷을 만남!: 5: 낚아챔
    section 다채로운 파싱과 영광스런 디섹션(Dissection)
      이건 `PUBLISH` 모드, `factory/temp` 키네! 와우!: 5: 환골탈태

놀랍지 않은가? 아까 전까지만 해도 그저 이해 불능의 까만 16진수 숫자에 불과했던 페이로드들이, 파란색과 노란색의 형형색색 줄이 처짐과 동시에 마법처럼 MsgType_PUBLISH, RoutingKey_ID=5와 같이 사람의 언어로 가장 완벽하고 직관적인 해부가 펼쳐진다! 당신의 와이어샤크 창은 이제 가장 강력하고 무결한 엑스레이(X-Ray) 머신으로 탈바꿈, 그 어떤 이상한 로봇 펌웨어들의 패킷 폭주와 오류마저 0.1초 단위의 원시(Primitive) 레벨에서 증명해 내는 마법의 전장 감시경이 될 것이다.
