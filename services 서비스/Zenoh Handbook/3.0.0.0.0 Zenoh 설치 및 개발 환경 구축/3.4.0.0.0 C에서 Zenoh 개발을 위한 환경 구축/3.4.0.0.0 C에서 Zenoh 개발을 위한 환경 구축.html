3.4 C에서 Zenoh 개발을 위한 환경 구축

클라우드는 Rust나 Python으로 어떻게든 치장할 수 있지만, 산업 현장의 끝단(Edge)에 뒹구는 수많은 PLC 컨트롤러, 모터 제어 드라이버, 구형 임베디드 펌웨어 플랫폼들은 여전히 순수 C언어 문법이라는 철옹성 아래 놓여 있다.
Rust로 짜인 Zenoh의 미친 속도를 낡은 C 컴파일러(GCC) 생태계로 이식하기 위해선, **zenoh-c**라는 FFI(Foreign Function Interface) 브릿지 계통을 완벽하게 컴파일하고 링킹(Linking)시키는 인프라 지식이 요구된다.

이 절에서는 Makefile 하나 다룰 줄 모르는 초보적 접근을 배제하고, CMake 기반의 크로스 컴파일 파이프라인(X86_64 -> ARM64) 구축 및 메모리 누수 방어(Valgrind)를 위한 극악무도한 C 환경 세팅 런북을 전개한다.
