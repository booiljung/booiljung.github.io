3.4.1 C/C++ 컴파일러(GCC, Clang, MSVC) 요구사항 및 설정

Rust로 작성된 엔진을 불러다 쓰기 위해서는 C 코드가 읽고 엮일 수 있는(Linking) 표준 헤더(.h)와 동적/정적 라이브러리(.so, .a, .dll) 파일들이 필요하다. 이들을 문제 없이 소화하려면 베이스 C 컴파일러의 세팅이 최신 표준(C99 이상)을 타당하게 지원해야 한다.

0.1 [Runbook] 운영체제별 빌드 도구 무결성 준비

개발자의 PC 혹은 CI/CD 빌드 인스턴스에 다음 컴포넌트들을 의무 장착해야 한다.

  1. Linux (Ubuntu/Debian 팩토리 서버):
  • GCC 및 빌드 도구 세트: sudo apt install build-essential cmake pkg-config
  • 주의: 너무 오래된 Ubuntu 16.04 같은 환경의 구형 GCC는 심볼 링킹에서 비명을 지를 수 있으므로 최소 GCC 7.0 이상을 확립하라.
  1. Windows (터프북 현장 장비 개발):
  • MSVC (Visual Studio Build Tools) 2019 이상 필수 설치.
  • 윈도우에서는 정적 링킹(.lib)을 걸 때 MSVC 고유의 런타임 라이브러리(MD vs MT) 옵션 충돌이 1순위 장애 요인이다. Zenoh 컴파일 포맷과 당신의 솔루션(*.sln) 런타임 포맷을 반드시 일치시켜라.
  1. macOS (개발용 장비):
  • Apple Clang 활성화: xcode-select --install
  • M1 계열에서 homebrew로 받은 Intel용 라이브러리를 끌고 와 합치려고 하면 Mach-O Linker Error로 컴파일러가 크래시된다. 타겟 아키텍처에 병적인 집착을 보이지 않으면 빌드는 터진다.

[안티 패턴 분쇄기] zenoh-c는 사실상 Rust 코드를 C 형태로 포장(Wrapper)한 것에 불과하다. C 컴파일러가 아무리 뛰어나도, 그 이면에 있는 Rust 빌드 툴체인(rustc, cargo)이 장착되어 있지 않으면 껍데기뿐인 라이브러리 소스를 컴파일할 방도가 없다. 반드시 앞선 3.3.1 장의 Rustup 설치 런북을 이 서버에도 동일하게 적용하고 넘어가야 함을 명심하라.
