3.3.1 Rust 공식 툴체인(rustup, cargo) 설치

“우분투 패키지 매니저(apt install rustc)로 Rust를 깐다“는 것은 가장 한심한 안티 패턴이다. 리눅스 배포판이 제공하는 범용 컴파일러는 업데이트 주기가 1~2년 단위로 느려 터져서, 0.x.x 단위로 릴리즈 버전이 격변하는 최신 Zenoh 라이브러리를 빌드하는 즉시 Syntax Error 와 Edition 2021 Mismatch 지옥에 빠지게 된다.

0.1 [Runbook] Rustup 기반의 격리된 툴체인 파종

반드시 공식 인스톨러 rustup을 통해 유저 스페이스(User Space) 안에 Rust 컴파일러를 격리 설치해야 한다.

# 1. rustup 스크립트 실행 (안전망: 배포판의 오래된 rustc와 충돌하지 않음)
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 2. 셸 환경변수 즉시 반영
source $HOME/.cargo/env

# 3. 버전 무결성 검증 (여기서 최소 1.70 이상이 떠야 Zenoh 빌드가 가능하다)
rustc --version

[인프라 강제 규칙: Toolchain 고정]
팀원 A의 랩탑과 팀원 B의 랩탑 Rust 버전이 다르면 매크로 컴파일 결과가 달라지는 재앙이 벌어진다. 프로젝트 최상단 디렉토리에 rust-toolchain.toml 파일을 강제로 박아 넣어라.

# 프로젝트 루트 / rust-toolchain.toml
[toolchain]
channel = "1.74.0" # 이 프로젝트를 git clone 한 모든 팀원은 이 버전으로 강제 통일됨!
components = ["rustfmt", "clippy"]

이 룰이 지켜지지 않는 팀의 CI/CD 파이프라인은 어느 날 갑자기 의존성 충돌로 비명을 지르며 멈추게 될 것이다.
