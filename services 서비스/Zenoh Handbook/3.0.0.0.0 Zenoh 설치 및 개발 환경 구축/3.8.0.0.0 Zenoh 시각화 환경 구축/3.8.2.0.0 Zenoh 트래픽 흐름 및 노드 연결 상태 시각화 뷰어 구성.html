3.8.2 Zenoh 트래픽 흐름 및 노드 연결 상태 시각화 뷰어 구성

그라파나(Grafana)의 대시보드가 라우터의 심박수(CPU/RAM/Throughput 등)라는 생명 유지 장치의 숫자를 그려냈다면, 이제 더 고차원적이고 직관적인 두 번째 뷰어가 시급해진다. 수만 대의 센서들(피어) 중, 대체 “어느 드론이 방금 서버망에 들어와서, 대체 누구한테 어떤 무거운 데이터(Traffic)를 던졌는지“를 실시간 통제실의 지도(Map)처럼 내려다보아야만 한다.

과거 DDS 미들웨어를 관제하느라 온갖 C++ 전용 뷰어 프로그램이나 무거운 자바 클라이언트를 설치해 보며 비참하게 좌절을 맛봤던 기억을 지워라. Zenoh는 인터넷 시대에 가장 최적화된, 이름마저 직관적인 ‘zenoh-plugin-webserver’ 혹은 Eclipse 재단이 배포하는 수많은 파생 브라우저 웹 기반 대시보드 도구들을 통해 당신의 모니터를 장악한다.

가장 빠르고 강력한 우회로이자 정석은 앞서 3.7.4절에서 설명했던 특권 공간—‘Admin Space(@/router/*)’—의 마법을 다시 한번 웹 화면(Web)으로 렌더링(Rendering) 시키는 전용 도구들의 조합이다.

수많은 오픈소스 엔지니어들이 GitHub에 올려놓은 ‘zenoh-dashboard’ 혹은 ‘zenoh-viewer’ 류의 리액트(React) 기반 프론트엔드 프로젝트를 클론(Clone)하여 여러분의 로컬 터미널에서 띄워(가동해) 주기만 하면 된다. 이 자바스크립트 UI 창은 백그라운드에서 교묘하게 3.7절의 Admin Space API를 호출하며, “수만 대의 로봇 중 방금 접속이 끊긴 피어(Peer) 노드“를 빨간 점으로 깜빡이게 하고, 초당 패킷 쓰루풋(Throughput)이 엄청난 병목 구간 선형 파이프는 굵고 진한 초록색 화살표 애니메이션 선회로 화면상에서 직관적인 지점별 동맥경화를 입증해 준다.

단순한 숫자나 로그(Log)가 아니라, 당신의 이 거대한 자율주행 라우터 관제망이 거미줄처럼 펼쳐지는 거시적인 전장, 완벽한 오버뷰(Overview) 생태계의 사령탑을 드디어 지휘하게 된 것이다.
