3.8 Zenoh 시각화 환경 구축

만약 우리가 앞 절에서 건설한 프로메테우스(Prometheus) 백엔드와 관리자(Admin) 터미널 공간의 덤프 텍스트 뭉치만으로 만족한다면, 그것은 눈을 뜨고도 까만 텍스트 파일만 뒤지며 서버의 불통을 감시하는 원시적인(Primitive) 해커의 수준에 머무르는 것이다.

진정한 시스템 운영팀(SRE, Site Reliability Engineering)은 절대 수만 대의 로봇이 뿜어내는 데이터를 숫자의 늪 속에서 허우적대며 찾지 않는다. 1초에 1기가바이트(GB)씩 쏟아져 들어오는 트래픽의 병발, 갑자기 라우터가 죽어가며 1분 전부터 치솟는 큐 사이즈(Queue Size), 그리고 수백 대의 로봇 궤적이 어떻게 서로 피어(Peer)를 맺어 물리적 거미줄(Topology)을 완성했는지, 이 모든 절망과 축복의 현장들을 가장 눈부시고 화려한 대시보드(Dashboard) 위 시각적인 예술(Visual Art)로 스크린에 띄워야만 한다.

본 절에서는 그 무미건조한 텍스트 데이터에 색(Color)과 선(Line)을 부여하여 거대한 워룸(War Room)의 관제 모니터로 승격시키는 시각화 렌더링(Rendering) 도구들의 장엄한 융합 과정을 집도한다. 프로메테우스의 단짝이자 시각화의 황제인 그라파나(Grafana) 패널 공사부터, 피어와 피어가 그물망처럼 맞닿는 그래프 노드(Graph Node) 매핑, 그리고 흐르는 실제 패킷 데이터 알맹이를 살포시 가로채어 그 생김새를 디스플레이 장치에 수놓아 버리는 이른바 페이로드 트레이싱(Tracing)의 현장에 이르기까지, 모든 데이터의 박동을 가장 화려한 맥박 그래픽으로 지휘하는 궁극의 시각화 아키텍처를 안내한다.
