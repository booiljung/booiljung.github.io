3.8.1 Grafana 대시보드 설치 및 Prometheus 데이터소스 연결

우리가 앞서 구축해둔 프로메테우스(Prometheus)가 데이터를 어둠 속 하드디스크에 쌓아두는 냉철한 회계사라면, **그라파나(Grafana)**는 그 숫자들을 세상 밖으로 끄집어내 화려한 그래프의 불꽃으로 쏘아 올리는 가장 위대한 시각화 렌더러(Renderer) 엔진이다.

이 두 도구를 당신의 인프라에서 결합(Coupling)하는 수술은 현대 옵저버빌리티(Observability) 생태계의 가장 기본기이자 정석이 된 지 오래이므로, 도커(Docker) 기반이라면 너무나도 우아하고 자비 없이 명령어 두어 줄로 허무하게 끝날 것이다.

여러분의 쉘 프롬프트에 docker run -d -p 3000:3000 grafana/grafana를 타이핑하여 그라파나라는 텅 빈 도화지 엔진을 가동시킨다. 브라우저로 localhost:3000에 접속해 관리자 권한으로 로그인한 뒤, ‘Configuration(환경설정)’ 탭 속 가장 성스러운 ‘Data Sources’ 창에 진입하라. 그리고 “나침반 방향은 내가 아까 컴파일해 설치해둔 프로메테우스 서버 주소(예: http://localhost:9090)야!” 라고 주입하고 Save & Test 초록(Green) 선언 버튼을 타격하면, 그라파나는 지체 없이 프로메테우스가 거느린 Zenoh의 수만 줄 빅데이터 스키마와 완벽한 호흡의 빨대(Pipe)를 꽂아버린다.

이제 빈 캔버스 메인 화면으로 돌아와, 프로메테우스 전용 쿼리 언어(PromQL)인 zenoh_router_cpu_usage 따위의 문구를 한 줄 딱 기입하게 되면! 텅 비어있던 그래프 프레임이 마침내 그토록 경망스럽고 빠르게 요동치는 당신의 최신 Zenoh 코어 라우터의 1초 단위 심박동수를 실시간 아날로그 게이지 바늘이나 구불구불한 주파수 파형 곡선 그래픽 덩어리로 물들여 완벽한 살아있는 관제 룸을 완성해내게 된다.
