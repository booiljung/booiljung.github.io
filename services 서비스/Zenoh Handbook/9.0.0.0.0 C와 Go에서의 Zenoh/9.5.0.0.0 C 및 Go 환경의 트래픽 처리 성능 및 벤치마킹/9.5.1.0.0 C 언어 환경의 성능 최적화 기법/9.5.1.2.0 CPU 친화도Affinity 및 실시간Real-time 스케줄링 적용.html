9.5.1.2 CPU 친화도(Affinity) 및 실시간(Real-time) 스케줄링 적용

Linux 커널은 매우 공평하다. 모든 스레드에 찔끔찔끔 CPU 시간을 공평하게 나눠주려다 보니 캐시 메모리(L1/L2)가 자꾸 리셋(Thrashing)된다.
우리는 리눅스에게 말해야 한다. “이 Zenoh 통신 스레드는 무조건 3번 코어에 묶고, 우선순위를 강제로 최상위로 올려라!”

0.1 [Runbook] POSIX 코어 격리(Core Isolation) 전술

로봇 제어용 메인 구동부를 만들 때 반드시 적용해야 할 설정이다.

#ifndef _GNU_SOURCE
#define _GNU_SOURCE
#endif
#include <pthread.h>
#include <sched.h>
#include <stdio.h>

void optimize_thread_for_robotics(pthread_t thread_id, int core_num) {
    // 1. CPU 코어 강제 할당 (Affinity)
    cpu_set_t cpuset;
    CPU_ZERO(&cpuset);
    CPU_SET(core_num, &cpuset); // 0,1,2,3 번 코어 중 지정

    int rc = pthread_setaffinity_np(thread_id, sizeof(cpu_set_t), &cpuset);
    if (rc != 0) {
        printf("!! 코어 바인딩 실패\n");
    }

    // 2. 리눅스 실시간(Real-Time) 스케줄링 강제 세팅 (FIFO 방식)
    // 일반 스레드 따위는 다 집어던지고 이 스레드만 최우선으로 돌리겠다는 선언
    struct sched_param param;
    param.sched_priority = 99; // 1~99 (99가 최고 존엄)
    
    // 이 명령어는 Sudo 권한이 없으면 실패한다! 
    rc = pthread_setschedparam(thread_id, SCHED_FIFO, &param);
    if (rc != 0) {
        printf("!! RT 스케줄링 설정 실패 (루트 권한 필요!)\n");
    }
}

로봇 프로그램이 시작하자마자 메인 스레드나 Zenoh의 수신 스핀 스레드에 이 함수를 쏴버리면, OS 단의 컨텍스트 스위칭에 의해 패킷이 지연되거나 모터 제어가 0.05초씩 튀는(Jitter) 현상을 원천 차단할 수 있다. (단, 이 설정을 남용하면 리눅스 OS 자체가 먹통이 되어 마우스조차 움직이지 않을 수 있다.)
