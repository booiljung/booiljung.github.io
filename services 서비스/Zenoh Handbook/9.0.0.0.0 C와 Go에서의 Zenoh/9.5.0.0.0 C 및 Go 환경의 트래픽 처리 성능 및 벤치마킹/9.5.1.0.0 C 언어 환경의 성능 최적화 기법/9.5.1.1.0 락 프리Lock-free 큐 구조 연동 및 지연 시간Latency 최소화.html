9.5.1.1 락 프리(Lock-free) 큐 구조 연동 및 지연 시간(Latency) 최소화

앞선 9.2.4 장에서 Mutex로 전역 변수를 보호했다고 하더라도, 초당 수십만 번 락(Lock)을 쥐었다 폈다 하면 커널 단위의 컨텍스트 스위칭 오버헤드로 인해 시스템이 정지된 듯한 지연(Jitter)이 발생한다.

0.1 [Runbook] 락-프리 링 버퍼(Ring Buffer) 전술

다수의 Subscriber 스레드(생산자)가 단방향으로 데이터를 써넣고, 하나의 메인 로봇 스레드(소비자)가 데이터를 집어 읽는 이상적인 모델(MPSC: Multi-Producer Single-Consumer)을 구현한다.

#include <stdatomic.h> // C11 아토믹 연산
#include <zenoh.h>
#include <string.h>

#define RING_SIZE 1024
typedef struct {
    char data[32]; // 페이로드 보관함
} Packet;

Packet ring_buffer[RING_SIZE];
atomic_size_t ring_head = 0; // 쓰는 곳
atomic_size_t ring_tail = 0; // 읽을 곳

// [Zenoh 콜백: 여러 스레드에서 무작위로 호출됨]
void lockfree_callback(const z_sample_t* sample, void* ctx) {
    // 1. 머리를 원자적(Atomic)으로 1칸 전진시킨다 (락이 없음!)
    // 이 연산은 CPU 레벨에서 처리되므로 지연이 0.000001초 미만이다.
    size_t current_head = atomic_fetch_add(&ring_head, 1) % RING_SIZE;
    
    // 2. 가로챈 버퍼 자리에 내 데이터를 우겨넣는다.
    // (여기서 페이로드 크기가 32바이트를 안 넘는다고 가정)
    memcpy(ring_buffer[current_head].data, sample->payload.start, sample->payload.len);
    ring_buffer[current_head].data[sample->payload.len] = '\0';
}

// [메인 로봇 제어 스레드: 1명만 무한히 순회함]
void* robot_control_loop(void* arg) {
    while(1) {
        // 머리가 꼬리보다 크면 = 새 데이터가 들어왔다는 뜻
        size_t h = atomic_load(&ring_head);
        size_t t = atomic_load(&ring_tail);
        
        while (t < h) {
            size_t idx = t % RING_SIZE;
            printf("Lock-free 소비: %s\n", ring_buffer[idx].data);
            t++;
        }
        atomic_store(&ring_tail, t); // 꼬리 위치 갱신
        
        // (적절히 sleep 하여 CPU 코어 독점 방지)
        usleep(100); 
    }
}

위의 아토믹(atomic_fetch_add) 링 버퍼는 Pthread Mutex를 쓰는 방식 대비 최소 100배에서 500배 빠른 처리 속도를 자랑한다. 로보틱스의 C 개발자라면 숨 쉬듯 짜낼 수 있어야 하는 가장 강력한 자료 구조다.
