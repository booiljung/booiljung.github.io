9. C와 Go에서의 Zenoh

“세상에서 가장 가볍고 미개한 장비부터, 세상에서 가장 빠르게 클러스터링되는 서버까지.”

로보틱스와 IoT, 클라우드를 관통하는 이기종(Heterogeneous) 통신망에서 Rust 하나만으로 세상을 통일할 수는 없다.
1메가바이트의 램(RAM)조차 낭비할 수 없는 마이크로컨트롤러(MCU)와 안드로이드 NDK 환경에서는 태초의 언어 C (C89/C99) 가 여전히 군림하고 있으며, Kubernetes와 Docker로 떡칠된 클라우드 네이티브 백엔드에서는 동시성(Concurrency)의 제왕 Go (Golang) 가 왕좌를 차지하고 있다.

이 장은 극도로 대조적인 이 두 세계(C의 ’포인터 수동 조작’과 Go의 ‘고루틴 채널’) 안으로, 어떻게 Zenoh 라우터가 완벽한 네이티브 성능으로 비집고 들어가는지 파헤치는 런북(Runbook) 이다.
메모리 생태계가 정반대인 두 언어를 하나로 이어붙여, 엣단의 펌웨어와 클라우드의 백엔드를 0.1ms 딜레이로 꿰매는 ’인프라 용접(Welding)’을 시작한다.
