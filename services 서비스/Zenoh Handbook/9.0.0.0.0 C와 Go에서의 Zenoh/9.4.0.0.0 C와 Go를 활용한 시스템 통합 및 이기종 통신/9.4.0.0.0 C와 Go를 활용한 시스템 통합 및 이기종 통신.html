9.4 C와 Go를 활용한 시스템 통합 및 이기종 통신

공장 컨베이어 벨트에 부착된 센서(C)에서 뿜어져 나온 배터리 상태 값이, 클라우드의 관제 서버(Go) 구조체 프로퍼티 안에 이쁘게 담기기까지의 여정은 험난하다.
C 개발자는 char buf[256] 에다 메모리를 대충 복사(memcpy)해서 던져버리고, Go 개발자는 날아온 []byte 를 자기가 선언해 둔 struct 에 끼워 넣으려고 json.Unmarshal 을 돌리다 서버가 터지는 파편화(Fragmentation) 현상이 발생하기 때문이다.

이 장은 C의 날것(Raw) 데이터와 Go의 엄격한 타입(Typed) 데이터가 네트워크 케이블을 타고 넘어갈 때, 서로의 메모리를 오염시키지 않고 깔끔하게 소화/분기시키는 이기종 직렬화(Cross-Language Serialization) 의 끝판왕 런북이다.
