9.4.2.2 데이터 타입 크기 및 호환성 변환 가이드

이기종 통신의 가장 큰 적은 언어마다 “말이 다르다” 는 것이다.
C 코드의 int 와 long 은 32비트 칩이냐 64비트 칩이냐에 따라 사이즈가 고무줄처럼 늘어났다 줄어들었다 한다. 이걸 받은 Go 언어의 int 역시 64비트 변수다. 서로 통신을 하는 순간 4바이트가 비거나 꼬여서 숫자가 깨진다.

0.1 [Runbook] 데이터 타입의 하드코어 고정(Pinning) 전술

이기종 Zenoh 통신을 염두에 둔 C 헤더 파일에는 절대로 기본 int 나 long 을 선언해서는 안 된다.

// ❌ 절대 금지 (이기종 재앙의 씨앗)
typedef struct {
    int id;           // 16비트일 수도, 32비트일 수도 있음
    long timestamp;   // 32비트일 수도, 64비트일 수도 있음
} CBadStruct;

// ✅ 베스트 프랙티스 (C99 stdint.h 사용 필수)
#include <stdint.h>
typedef struct {
    uint32_t id;         // 무조건 4바이트 양수 
    int64_t timestamp;   // 무조건 8바이트 정수 (UNIX Timestamp)
} CGoodStruct;

이렇게 하면 Go 백엔드 구조체에서도 변수형을 int32 나 int64 등으로 1:1 완벽하게 핀 포인트로 맞출 수(Mapping) 있게 된다.
로봇부터 클라우드까지 흐르는 데이터는 처음부터 끝까지 Float, Int, String 이 아니라 Float32, Int64, UTF-8 Array 로 철저하게 기계적으로 검증된 상태여야 한다.
