9.2.1.3 임베디드 리눅스 및 RTOS를 위한 교차 컴파일(Cross-compilation) 설정

당신의 노트북은 Intel CPU(x86_64)다. 하지만 당신이 만든 zenoh-c 통신 프로그램이 돌아가야 할 곳은 라즈베리 파이(ARM64)나, NVIDIA Jetson Nano(AArch64)다.
노트북에서 일반 gcc 로 컴파일한 실행 파일을 저질 로봇에 던져 넣으면 아예 실행 규격이 달라서 켜지지도 않는다. 타겟 컴퓨터를 위한 옷(컴파일)을 내 컴퓨터에서 미리 만들어주는 기술, 즉 교차 컴파일(Cross-compile) 이 필요하다.

0.1 [Runbook] ARM64 타겟 크로스 컴파일 파이프라인

전제조건: 만약 ARM용 빌드라면, zenoh-c 패키지 역시 zenoh-c-linux-aarch64 버전을 다운받아 libs_arm/ 폴더에 세팅해 둬야 한다. (x86용 .so 를 ARM 링커에 물리면 터진다)

1. 리눅스 교차 툴체인 설치 (Ubuntu 기준)

# ARM64(AArch64) 코드를 생성할 수 있는 십자창(Cross-compiler) 설치
sudo apt install gcc-aarch64-linux-gnu g++-aarch64-linux-gnu

2. CMake 툴체인 파일 작성 (arm64-toolchain.cmake)
CMake가 내 컴퓨터의 gcc가 아니라 방금 설치한 aarch64-gcc 를 쓰도록 강제하는 저격 파일이다.

# 커널 타겟 선언
set(CMAKE_SYSTEM_NAME Linux)
set(CMAKE_SYSTEM_PROCESSOR aarch64)

# 무기(컴파일러) 교체!
set(CMAKE_C_COMPILER aarch64-linux-gnu-gcc)
set(CMAKE_CXX_COMPILER aarch64-linux-gnu-g++)

# 타겟 환경의 루트 디렉토리 무시 설정 방어
set(CMAKE_FIND_ROOT_PATH_MODE_PROGRAM NEVER)
set(CMAKE_FIND_ROOT_PATH_MODE_LIBRARY ONLY)
set(CMAKE_FIND_ROOT_PATH_MODE_INCLUDE ONLY)

3. 빌드 개시!
터미널에서 CMake 툴체인 옵션을 주어 빌드한다.

mkdir build_arm && cd build_arm
# 내가 만든 툴체인 파일을 지목!
cmake -DCMAKE_TOOLCHAIN_FILE=../arm64-toolchain.cmake ..
make

완성된 robot_node 실행 파일은 내 노트북에서는 영원히 실행할 수 없다. scp 명령어를 통해 라즈베리파이로 복사한 뒤 실행해야만 진정한 엣지 컴퓨팅망의 톱니바퀴가 돌아가기 시작한다.
