9.2.1.1 CMake 및 다양한 빌드 시스템(Make, Ninja)과의 연동

요즘 C/C++ 프로젝트를 맨땅에 Makefile부터 짜는 엔지니어는 드물다. 크로스 플랫폼 빌드 시스템의 표준인 CMake 를 기반으로 zenoh-c 패키지를 우아하게 빨아들이는(FindPackage) 로직을 전개한다.

0.1 [Runbook] CMake 기반 Zenoh 결속 전술

zenoh-c 의 릴리즈 압축 파일(예: zenoh-c-0.10.x-linux-x86_64.zip)을 다운로드하여 프로젝트 폴더 내 libs/zenoh 경로에 압축을 푼다고 가정한다.

1. 폴더 구조 설계

my_robot_project/
├── CMakeLists.txt
├── src/
│   └── main.c
└── libs/
    └── zenoh/
        ├── include/   (zenoh.h 존재)
        └── lib/       (libzenohc.so 혹은 libzenohc.a 존재)

2. CMakeLists.txt 작성의 정석

cmake_minimum_required(VERSION 3.10)
project(MyRobotComm C CXX)

set(CMAKE_C_STANDARD 99) # Zenoh-C는 C99 이상 권장

# 1. 헤더 파일(include) 탐색 경로 추가
include_directories(${CMAKE_SOURCE_DIR}/libs/zenoh/include)

# 2. 링커 경로 지정 (라이브러리 덩어리가 어디 있는가?)
link_directories(${CMAKE_SOURCE_DIR}/libs/zenoh/lib)

# 3. 내 소스코드 컴파일 타겟 지정
add_executable(robot_node src/main.c)

# 4. Zenoh 동적 라이브러리(zenohc) 링킹
# (참고: 리눅스에서는 추가적으로 pthread와 m(수학) 라이브러리가 필요할 수 있다)
target_link_libraries(robot_node zenohc pthread m)

이 CMake 문법은 make 를 돌리든, Ninja (초고속 병렬 빌드)로 돌리든 동일하게 컴파일러(GCC/Clang) 옵션으로 변환되어, 당신의 main.c 에 선언된 #include <zenoh.h> 를 런타임에 완벽하게 묶어(Link)준다.
