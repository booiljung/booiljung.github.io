9.2.3.3 Queryable 및 Querier를 통한 RPC 방식의 요청-응답 구현

분산 쿼리와 응답 전술을 C 언어로 구현한다.
C 개발자에게 “비동기 요청-다중 응답(Multi-Reply)” 이란 매우 낯선 개념이다. HTTP 처럼 하나 보내면 무조건 하나가 온다고 가정하면 시스템이 꼬인다. 한 번의 쿼리에 여러 데이터가 돌아오면, C 언어에서는 어떻게 그 목록들을 주워야 할까?

0.1 [Runbook] 지연 분산 쿼리(Query) 격발 전술

핵심은 응답 큐(FIFO Queue) 를 만들어서 그 안에다가 돌아오는 대답들을 차곡차곡 쌓는(push) 것이다.

1. 쿼리 서버 (대답하는 놈)

#include <zenoh.h>

void my_query_handler(const z_query_t* query, void* context) {
    // 누군가 질의를 쏘면 이 함수가 깨어난다.
    printf("[수신] 누군가 데이터를 원함.\n");
    
    // 대답할 키 경로를 원래 질문자의 타겟 패스로 다시 세팅
    z_owned_keyexpr_t reply_key = z_keyexpr_clone(z_query_keyexpr(query));
    
    z_query_reply_options_t opts = z_query_reply_options_default();
    
    // 쿼무새처럼 그대로 대답 발사!
    z_query_reply(query, z_loan(reply_key), (const uint8_t*)"My Battery is 75%", 17, &opts);
    
    z_drop(z_move(reply_key));
}

// 메인문에서: z_declare_queryable() 로 이 핸들러를 바인드해둔다.

2. 쿼리 클라이언트 (질문하고 수거하는 놈)

void fetch_remote_data(z_owned_session_t* session) {
    z_owned_keyexpr_t target_key = z_keyexpr_new("robot/*/battery");
    z_get_options_t opts = z_get_options_default();
    
    // [핵심] C 언어는 for await 나 Promise.all 이 없다. 
    // 대신 Zenoh 가 던져주는 대답을 순차적으로 담을 FIFO 파이프라인(z_owned_reply_channel_t)을 딴다.
    z_owned_reply_channel_t channel = zc_reply_fifo_new(16); // 16개 한도 큐

    // 타격 개시! 대답들은 아까 만든 channel 통로로 속속들이 들어오기 시작한다.
    z_get(z_loan(*session), z_loan(target_key), "", z_move(channel.send), &opts);

    // [회수 루프]
    z_owned_reply_t reply = z_reply_null();
    
    // 채널 파이프 뚜껑을 열고 빨아들인다! 반환값이 유효할 때까지 뺑뺑이
    for (z_call(channel.recv, &reply); z_reply_check(&reply); z_call(channel.recv, &reply)) {
        
        if (z_reply_is_ok(&reply)) {
            z_sample_t sample = z_reply_ok(&reply);
            printf("[건져냄] 데이터: %.*s\n", (int)sample.payload.len, sample.payload.start);
        } else {
            printf("[건져냄] 통신 파이프 에러 상태 응답!\n");
        }
        
        z_drop(z_move(reply)); // 방금 읽은 응답 파기 (필수)
    }

    // 채널 완전히 파괴
    z_drop(z_move(channel));
    z_drop(z_move(target_key));
}

이 코드는 C 언어에서 “비동기 멀티캐스트의 흐름 제어(Flow Control)“를 구현하는 가장 완벽한 패턴이다. FIFO 큐(z_owned_reply_channel_t)가 일종의 댐 역할을 해주며, 앞쪽의 z_get 은 백그라운드 스레드에서 댐에 물을 퍼 붓고, 메인 루프인 for 문에서는 내가 원하는 템포로 물(데이터)을 한 바가지씩 퍼다 쓰는 것이다.
