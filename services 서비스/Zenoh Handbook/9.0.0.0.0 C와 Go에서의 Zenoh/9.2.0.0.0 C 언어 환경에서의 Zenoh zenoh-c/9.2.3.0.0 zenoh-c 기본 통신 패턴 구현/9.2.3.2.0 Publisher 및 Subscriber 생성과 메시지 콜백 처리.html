9.2.3.2 Publisher 및 Subscriber 생성과 메시지 콜백 처리

“보내는 자” 와 “듣는 자” 를 모두 C 언어로 붙여본다.
C 언어 콜백(Callback) 매커니즘 특성상, “어떤 객체(구조체)에 소속된 쓰레드에서 이 함수가 불리는가?“를 놓치면 공유 메모리 변수를 조작하다 레이스 컨디션(Race Condition)으로 시스템이 박살 난다.

0.1 [Runbook] 데이터 버스 송수신 전술

수신측 (콜백 구역)

#include <zenoh.h>
#include <stdio.h>

// 이 함수는 메인 스레드가 아니라 백그라운드 "Zenoh Rust 엔진 스레드" 에서 비동기적으로 불린다!
void motor_command_callback(const z_sample_t* sample, void* my_context) {
    // 1. Key 와 매칭
    z_owned_str_t keystr = z_keyexpr_to_string(sample->keyexpr);
    
    // 2. 바디 데이터 열람 (제로 카피!)
    printf("[명령 수신] 경로: %s, 바이트길이: %zu\n", z_loan(keystr), sample->payload.len);
    
    // 3. (옵션) Context 를 포인터로 받아서 메인 스택 변수를 조작할 수도 있다.
    // int* motor_speed = (int*)my_context;
    
    z_drop(z_move(keystr)); // 문자열을 새로 팠으니 버려준다.
}

메인 구동부

int run_dual_node(z_owned_session_t* session) {
    // 1. 퍼블리셔(송신 무기) 장착
    z_owned_keyexpr_t pub_key = z_keyexpr_new("robot/status");
    z_owned_publisher_t pub = z_declare_publisher(z_loan(*session), z_loan(pub_key), NULL);
    
    // 2. 서브스크라이버(수신기) 장착
    z_owned_keyexpr_t sub_key = z_keyexpr_new("robot/motor/cmd");
    // 클로저(Closure) 구조체를 만들어 콜백 함수를 박아 넣는다.
    z_owned_closure_sample_t callback;
    z_closure(&callback, motor_command_callback, NULL, NULL); // context 는 비워둠
    
    z_owned_subscriber_t sub = z_declare_subscriber(z_loan(*session), z_loan(sub_key), z_move(callback), NULL);
    
    // 3. 송신 반복문 (로봇 메인 루프)
    printf(">> 양방향 노드 기동 완료. 1초에 한 번씩 상태를 쏩니다.\n");
    for (int i=0; i<100; i++) {
        char buf[64];
        int len = snprintf(buf, sizeof(buf), "Battery: %d", 100 - i);
        
        // C 언어 특유의 Raw Byte Array 통짜 발사!
        z_publisher_put(z_loan(pub), (const uint8_t*)buf, len, NULL);
        
        // (unistd.h) sleep 1초
        sleep(1); 
    }
    
    // 4. 해제 (RAII가 없으므로 무조건 수동 청소)
    z_undeclare_subscriber(z_move(sub));
    z_undeclare_publisher(z_move(pub));
    z_drop(z_move(pub_key));
    z_drop(z_move(sub_key));
    return 0;
}

이 패턴에서 가장 치명적인 위험은, motor_command_callback 안에서 오랜 시간 루프를 돌거나 sleep 을 해버리면, Zenoh 의 Rust 이벤트 분배 스퍼드(Dispatcher) 가 막혀버려서 다른 네트워킹 이벤트까지 통째로 지연(Delay)된다는 사실이다. C 콜백 안에서는 무조건 변수 값만 갱신(혹은 Lock-free 큐에 밀어 넣기)하고 1ms 안에 함수 밖으로 뛰쳐나와야 한다.
