9.2.3.1 세션(Session) 생성, 구성 설정 및 종료

모든 Zenoh C 어플리케이션은 세션을 열고 닫는 것에서 시작해 끝난다.

0.1 [Runbook] 클라이언트 모드 고정 및 세션 훅 전술

MCU나 로봇 펌웨어는 십중팔구 “라우터(Router)“가 아니라 “클라이언트(Client)” 역할을 수행해야 한다. 자신이 무선 망의 중계기가 되어버리면 배터리와 메인 CPU가 폭발적으로 소모되기 때문이다.

#include <zenoh.h>
#include <stdio.h>
#include <stdlib.h>

int boot_robot_comm() {
    // 1. 디폴트 설정 가져오기 (소유권 획득)
    z_owned_config_t config = z_config_default();

    // 2. [강제] 나는 절대 라우터가 아니라 클라이언트다!
    // z_config_insert() 를 통해 Rust 내부 JSON 설정 트리를 비틀어버린다.
    // 주의: 문자열 값을 넣을 때 내부적으로 JSON 파서가 돌기 때문에 
    // "client" 문자열 자체가 따옴표로 감싸진 '"client"' 여야 한다.
    zc_config_insert_json(z_loan(config), Z_CONFIG_MODE_KEY, "\"client\"");
    
    // (선택) 특정 WiFi 공유기나 사내 중앙 라우터에 IP를 박고 싶을 때
    zc_config_insert_json(z_loan(config), Z_CONFIG_CONNECT_KEY, "[\"tcp/192.168.0.5:7447\"]");

    printf(">> Zenoh 엔진 시동 개시...\n");

    // 3. 세션 점화! 
    // z_move(config) 를 통해 내가 깎아둔 설정값의 소유권을 세션에게 넘긴다.
    // 이 시점 이후 'config' 변수는 C코드에서 쓰면 Segfault 가 난다.
    z_owned_session_t session = z_open(z_move(config));

    if (!z_session_check(&session)) {
        printf("!! 세션 개시 실패. 라우터 강제 탐색을 끕니다.\n");
        return -1;
    }

    printf(">> 스웜 접속 완료!\n");

    // 4. 로봇 비즈니스 로직 수행
    // ... do something ...

    // 5. 우아한 종료 (소켓 파기 및 메모리 회수)
    z_close(z_move(session));
    return 0;
}

이 패턴에서 명심할 점은, z_config_t 객체 안에 IP를 쑤셔넣는 함수 이름이 버전에 따라 zc_config_insert_json 으로 미세하게 바뀔 수 있다는 점이다. 공식 Github 레포의 헤더 파일(/include/zenoh.h)을 뜯어보며 C-macro 의 이름을 확인하는 습관을 들여야 한다.
