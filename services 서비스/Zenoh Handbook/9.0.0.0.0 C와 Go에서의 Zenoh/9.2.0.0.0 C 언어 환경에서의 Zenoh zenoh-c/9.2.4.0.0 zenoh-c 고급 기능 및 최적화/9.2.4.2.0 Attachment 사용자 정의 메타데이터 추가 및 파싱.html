9.2.4.2 Attachment 사용자 정의 메타데이터 추가 및 파싱

본문 데이터(Payload) 외에, “이 데이터는 중요도가 1번이다”, “이 데이터는 2번 압축 포맷이다” 같은 편지봉투(Header)를 붙여야 할 때가 있다.
본문 바이트 배열의 맨 앞부분을 뜯어내서(Parsing) 직접 헤더를 만들지 말고, Zenoh 의 기본 스펙인 Attachment 에 우아하게 붙여 보내라.

0.1 [Runbook] 데이터 꼬리표 부착(Attachment) 전술

송신부 (서류에 견출지 붙이기)

// 무기명 빈 설정 객체 생성
z_publisher_put_options_t options = z_publisher_put_options_default();

// 첨부파일(Attachment) 바이트 정의
const uint8_t* meta = (const uint8_t*)"FORMAT=JPEG;PRIORITY=9";

// C API 특유의 Bytes 생성 패턴
z_owned_bytes_t attachment_bytes = z_bytes_new(meta, 22);

// 옵션 객체에 쑤셔넣음 (이 때도 z_move 로 소유권을 넘김!)
options.attachment = z_move(attachment_bytes);

// 편지지에 페이로드와 견출지(옵션)를 함께 담아 발송
z_publisher_put(z_loan(pub), payload_addr, payload_len, &options);
