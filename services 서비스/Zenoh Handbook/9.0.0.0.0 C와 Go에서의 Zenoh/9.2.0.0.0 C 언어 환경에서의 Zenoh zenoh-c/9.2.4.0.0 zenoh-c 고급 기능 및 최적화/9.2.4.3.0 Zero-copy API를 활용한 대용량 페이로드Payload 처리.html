9.2.4.3 Zero-copy API를 활용한 대용량 페이로드(Payload) 처리

memcpy() 함수는 프로그래머에게 마약이다. 편하지만, 메모리 복사 시간이 수 밀리초씩 깎여나가면 CPU가 불탄다. C 언어의 궁극기는 “OS가 잡아준 메모리 물리 주소값을 그대로 상대방 프로그램에 쑤셔넣는 것” 이다.

0.1 [Runbook] 메모리 수거 위임(Deallocation Delegation) 전술

Zenoh 의 put 은 보통 안전을 위해 버퍼를 “복사“해서 자기들만의 엔진 큐에 담아버린다.
“복사하지 말고 내 주소값을 바로 읽어가라“고 하려면 명시적 Zero-Copy API를 써야 한다.

#include <zenoh.h>
#include <stdlib.h>

// [함수 정의] 내 메모리를 대신 버려주는 청소부 
void my_custom_free(uint8_t *ptr, size_t len, void *ctx) {
    printf("Zero-copy 통신 완료! 내 메모리 %p 청소 개시.\n", ptr);
    free(ptr);
}

void zero_copy_shoot(z_owned_session_t* session, z_owned_publisher_t* pub) {
    // 1. 방대한 1GB 짜리 가상 메모리 객체를 내가 만들었다.
    size_t HUGE_SIZE = 1024 * 1024 * 1024;
    uint8_t* huge_map = (uint8_t*)malloc(HUGE_SIZE);
    
    // (데이터 쓰기 작렬)
    huge_map[0] = 0xFF; // ...
    
    // 2. 옵션 초기화
    z_publisher_put_options_t options = z_publisher_put_options_default();
    
    // 3. 대망의 Zero copy 바이트 생성!
    // 내 메모리 주소($huge_map)와, 이게 다 쓰이고 나면 누가 이걸 치워버릴지(my_custom_free)
    // 그 책임 자체를 Zenoh(Rust 엔진) 에게 던져버린다.
    z_owned_bytes_t zcb = z_bytes_new_alloc(huge_map, HUGE_SIZE, my_custom_free, NULL);
    
    // 4. 발사 
    // z_move 로 보냈지만, 내부적으론 아무런 1GB짜리 memcpy 가 일어나지 않았다! (O(1) 통신 속도)
    z_publisher_put(z_loan(*pub), z_loan(zcb), HUGE_SIZE, &options);
    
    // 5. 메모리 파기
    z_drop(z_move(zcb)); 
    
    // 주의! 이 밑에서 огром_map 포인터를 건드리면 Use-After-Free 터짐!!!
    // 엄청나게 빠른 대신 인간의 실수를 용서하지 않는다.
}

이것이 C로 짠 카메라 캡처 보드가 FHD 60FPS 영상을 멀티캐스트로 흩뿌리면서도 라즈베리 파이의 CPU 사용량이 단 5% 에 머무는 핵심 기술이다.
