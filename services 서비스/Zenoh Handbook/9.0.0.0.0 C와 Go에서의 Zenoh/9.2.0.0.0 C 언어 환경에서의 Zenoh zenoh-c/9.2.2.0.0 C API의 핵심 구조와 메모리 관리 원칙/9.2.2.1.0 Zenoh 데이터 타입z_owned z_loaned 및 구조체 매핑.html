9.2.2.1 Zenoh 데이터 타입(z_owned, z_loaned) 및 구조체 매핑

Rust 언어의 핵심 철학인 소유권(Ownership)과 빌림(Borrowing) 개념이 zenoh-c 에 그대로 강림했다. C 언어 문법에는 이런 것이 없기 때문에, 구조체의 접두어(Prefix)로 이를 강제한다.

0.1 [인스펙션] 생살여탈권의 분배

  1. z_owned_xxx_t (소유권자 - 내꺼다!)
  • 개발자가 이 객체를 얻었다면, 영구적인 메모리 해제 의무를 갖는다.
  • 예: z_owned_session_t (내가 연 세션), z_owned_publisher_t (내가 등록한 발행자)
  • 철칙: 이걸 사용하고 나서 z_drop() 이나 z_close() 를 안 부르면 즉각적인 메모리 릭이 발생한다.
  1. z_loaned_xxx_t (대출자 - 빌린 거다!)
  • Zenoh(Rust 엔진)가 당신의 C 코드에 “잠깐 구경만 해라” 하고 주소값을 던져준 상태다.
  • 예: Subscriber 콜백 안으로 들어오는 z_sample_t 의 내부 데이터들.
  • 철칙: 절대! 절대로 여기다가 z_drop() 이나 free() 를 때려서는 안 된다. 당신의 것이 아니다. 콜백 함수가 끝나는 순간 Rust 엔진이 알아서 수거해 간다.

0.2 [Runbook] 데이터 타입 매칭의 실전 예

#include <zenoh.h>
#include <stdio.h>

void type_matching_example() {
    // 1. 소유권(Owner) 생성 타임 
    // z_config_default() 가 힙에 공간을 파서 당신에게 줘버림
    z_owned_config_t config = z_config_default();
    
    // session_drop() 이라는 수거 함수를 통해 소유권 박탈
    z_owned_session_t session = z_open(z_move(config));

    // 2. 대출권(Loan) 빌리기 타임
    // z_keyexpr_new()는 소유권(owned)을 만들지만,
    // z_loan() 매크로를 통해 대출권 껍데기만 살짝 씌워 넘겨줄 수 있다.
    z_owned_keyexpr_t my_key = z_keyexpr_new("robot/status");
    z_publisher_put_options_t options = z_publisher_put_options_default();

    // 3. 발사할 때 소유권을 넘겨준다.
    // [중요] C API에서는 구조체를 함수의 파라미터로 "던져줄 때" 
    // z_move(my_key) 를 통해 주소값과 권한을 완전히 줘버리는 Rust-like 패턴이 쓰인다.
    z_session_put(&session, z_loan(my_key), (const uint8_t *)"hello", 5, &options);

    // 4. 세션 해제 (자동으로 휘하의 리소스 파괴됨)
    z_close(z_move(session));
}

빌려 쓰는 놈(loaned)과 영구 소유하는 놈(owned)을 눈으로 5초 안에 식별하지 못하면 zenoh-c 코딩은 Core Dump(Segfault) 의 연속이 될 것이다.
