9.2.2.2 수동 메모리 할당 및 해제 메커니즘, 메모리 누수 방지

C 개발자들의 숙명인 “청소부 역할“이다. zenoh-c 는 z_drop() 이라는 공용 해제(Free) 매크로를 제공한다.
하지만 언제 써야 할까? 그리고 에러가 나서 return 이 중간에 일어나는 방어 코드(Guard Clause)가 있을 땐 어떻게 이빨 빠진 메모리를 해제해야 할까?

0.1 [Runbook] 방어적 z_drop() 전술 (GOTO 스윕 기법)

리눅스 커널 소스코드에서 가장 사랑받는 에러 처리 및 일괄 메모리 수거 기법이다.

#include <zenoh.h>
#include <stdio.h>

int safe_publisher_task() {
    z_owned_config_t config = z_config_default();
    z_owned_session_t session;
    z_owned_publisher_t pub;
    z_owned_keyexpr_t key_expr;

    int ret_code = 0; // 성공/실패 여부

    // [1단계] 세션 열기
    session = z_open(z_move(config));
    if (!z_session_check(&session)) {
        printf("세션 부팅 실패\n");
        // 아직 아무것도 (pub 등) 생성 안됐으므로 바로 탈출!
        return -1; 
    }

    // [2단계] 키 표현식(자원) 할당
    key_expr = z_keyexpr_new("robot/battery");
    // 주의: new 가 실패했을수도 있으니 반드시 체크
    if (!z_keyexpr_check(&key_expr)) {
        printf("키 생성 실패\n");
        ret_code = -1;
        // ⭐ 이 시점부터 에러가 나면, '지금까지 만든 것들' 을 청소해야 함!
        goto cleanup_session; 
    }

    // [3단계] 퍼블리셔(자원) 할당
    pub = z_declare_publisher(z_loan(session), z_loan(key_expr), NULL);
    if (!z_publisher_check(&pub)) {
        printf("퍼블리셔 생성 실패\n");
        ret_code = -1;
        goto cleanup_key; 
    }

    // [4단계] 정상 비즈니스 로직 수행
    z_publisher_put(z_loan(pub), (const uint8_t*)"100", 3, NULL);
    printf("명령 하달 성공\n");


// === [메모리 수거 청소 구역 (Teardown)] ===
// 스택의 역순으로 할당 해제를 수행한다.
cleanup_pub:
    z_undeclare_publisher(z_move(pub)); // 퍼블리셔 파괴
cleanup_key:
    z_drop(z_move(key_expr));         // 키 표현식 파괴
cleanup_session:
    z_close(z_move(session));         // 최종 세션 파괴

    return ret_code;
}

최신 C++ 의 RAII (Resource Acquisition Is Initialization) 가 없는 C 언어에서는, 이 GOTO 를 활용한 낙수 효과 청소법만이 초당 천번 호출되는 백그라운드 함수에서 “0%의 메모리 누수“를 달성하는 최강의 건축술이다.
