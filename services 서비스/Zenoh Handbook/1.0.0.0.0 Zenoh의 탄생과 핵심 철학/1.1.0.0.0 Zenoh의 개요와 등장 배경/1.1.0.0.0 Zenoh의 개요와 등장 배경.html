1.1 Zenoh의 개요와 등장 배경

클라우드 컴퓨팅과 사물인터넷(IoT)의 폭발적 성장은 네트워크 인프라에 전례 없는 압력을 가하고 있다. 전통적인 클라이언트-서버 모델이나 중앙 집중식 브로커(MQTT) 기반의 아키텍처는 수백억 개의 디바이스가 뿜어내는 ’데이터 쓰나미’를 감당하기엔 한계에 달했다.

Zenoh(제노)는 바로 이러한 시대적 요구(데이터의 폭증, 초저지연의 필요성, 이기종 네트워크 통신)에 응답하기 위해 Eclipse Foundation에서 잉태된 차세대 데이터 분산 네트워크(Data Distribution Network, DDN) 프로토콜이다. “가장 가벼운 곳(마이크로컨트롤러)부터 가장 거대한 곳(클라우드 데이터센터)까지 단일한 언어(프로토콜)로 소통하게 하라“는 철학 아래 탄생했다.

이 절에서는 단순히 개념을 훑는 데 그치지 않고, 엔지니어가 왜 당장 레거시(MQTT, DDS)를 걷어내고 Zenoh를 도입해야 하는지, 실제 현업 시스템이 직면한 아키텍처적 붕괴 시나리오와 이를 구원하는 Zenoh의 등장 배경을 런북(Runbook) 관점에서 실전적으로 해부한다.
