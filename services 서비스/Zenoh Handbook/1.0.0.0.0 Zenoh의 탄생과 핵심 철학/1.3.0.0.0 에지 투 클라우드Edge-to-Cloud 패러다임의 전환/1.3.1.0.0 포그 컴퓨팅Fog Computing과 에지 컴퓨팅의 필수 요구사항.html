1.3.1 포그 컴퓨팅(Fog Computing)과 에지 컴퓨팅의 필수 요구사항

데이터센터(구름, Cloud)와 지표면(에지 단말기) 사이의 대기를 형성하는 중간 레이어를 ’안개(Fog)’라 부른다. 공장 한구석에 놓인 워크스테이션, 5G 기지국의 MEC 컴퓨팅 노드가 이에 해당한다.

0.1 거대 지연 버퍼와 네트워크 백프레셔(Backpressure)

로봇 단말기(Edge)가 수집한 데이터를 클라우드에 밀어 넣으려 하는데 통신망이 잠시 끊겼다. 초소형 에지 단말기는 RAM이 수 메가바이트밖에 안 되기 때문에 1초 분량의 데이타도 저장하지 못해 터져버린다(OOM).
이때 빵빵한 SSD를 장착한 중간의 ’포그(Fog) 라우터’가 필요한 것이다.

Zenoh 라우터(Fog)의 인프라적 역할 부여 시나리오:
포그 계층에 Zenoh 라우터를 띄운 뒤 storage_manager 스토리지 플러그인을 결합하라.

// 포그 라우터의 zenohd.json5 내 볼륨 설정
plugins: {
  storage_manager: {
    volumes: {
      rocksdb: {
        dir: "/var/lib/zenoh/fog-buffer" // 통신망이 끊기면 안지우고 SSD에 무한 저장 
      }
    }
  }
}

통신망이 단절되면 포그 라우터는 로봇 단말(Edge)로부터 데이터를 무자비하게 빨아들여 자기 로컬 SSD에 저장(Back-pressure 완충)해두고 대신 답장을 쏴준다. 그리고 인터넷망(Cloud)이 부활하면 그동안 쌓은 데이터를 천천히 클라우드로 포워딩(Shipping)하는 지능적인 데이터 방파제 역할을 수행한다. 포그 계층은 단순한 스위치가 아니라 물리적 데이터 캐비닛이어야 한다.
