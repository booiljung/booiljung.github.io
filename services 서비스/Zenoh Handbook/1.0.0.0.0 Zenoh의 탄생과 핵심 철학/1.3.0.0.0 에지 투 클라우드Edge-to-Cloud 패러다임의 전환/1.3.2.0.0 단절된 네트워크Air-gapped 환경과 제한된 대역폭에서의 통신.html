1.3.2 단절된 네트워크(Air-gapped) 환경과 제한된 대역폭에서의 통신

잠수함, 자율 잠수정(AUV), 깊은 광산 기지, 혹은 철저히 망을 분리해 둔 1급 기밀 국방 보안구역 등 인터넷 핑이 도달할 수 없는 ‘Air-gapped’ 생태계는 일반 IT 클라우드 상식을 철저하게 붕괴시킨다.

0.1 치명적 문제: DNS 의존성과 고정 서버 인프라 부재

어둠의 격리망 안에서 단말기들이 서로의 IP를 찾을 방법이 없다. DNS 서버도 고장 났거나 애초에 존재하지 않는다.

런북: Zenoh의 Gossip 디스커버리를 통한 자생적 복원
Zenoh는 이런 환경에서도 무 브로커(Broker-less) 자생 기능을 지원한다.
장비들을 같은 허브/스위치 망에 물리거나 무선 Ad-hoc 망에 엮어 놓기만 하면 된다.

  1. 장비 A가 스카우팅(Scouting) 멀티캐스트 패킷 발사.
  2. 장비 B와 C가 그 소리를 듣고 서로의 라우팅 체계를 교환.
  3. 곧바로 그들끼리 작은 독립적 로컬 클라우드(Mesh Network)를 구축.

0.2 제한된 대역폭 (협대역통신망, LPWAN)의 극복

수 센티미터 두께의 쇠창살 너머로 통과하는 LoRaWAN망(수백 bit 지연 전송)에서 1KB 패킷 사이즈는 생사를 오가는 짐이다.

Zenoh는 이 구간의 연결 속성을 감지하면 트랜스포트 레이어를 Serial(직렬) 프로토콜 모드나 초경량 UDP 모드로 튜닝한 채(예: z_put -e serial/dev/ttyUSB0), 심지어 자신의 Zenoh 헤더 껍질조차 깎아내는 하드코어 최적화 스펙(v0.6 이후 도입된 Minimal Headers)을 발동할 수 있다. Air-gap 구간의 대역폭 한계상황 속에서 Zenoh보다 페이로드를 조밀하게 압축할 수 있는 미들웨어는 세상에 존재하지 않는다고 단언해도 무방하다.
