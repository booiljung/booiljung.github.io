1.3 에지 투 클라우드(Edge-to-Cloud) 패러다임의 전환

’모든 데이터를 클라우드에 쏟아부어 중앙에서 AI를 돌리겠다’는 빅 테크의 야심만만한 꿈은 물리학의 법칙 앞단에서 처참히 박살 났다. 수만 대의 고화질 카메라와 로봇 Lidar가 뿜어내는 수십 기가의 데이터는 전 지구의 통신망을 마비시킬 수준이었다. 결국 뇌의 일부를 클라우드에서 단말기(Edge) 쪽으로 밀어내어 현장에서 데이터를 자체 처리해야 한다는 ‘에지 투 클라우드(Edge-to-Cloud)’ 패러다임이 업계의 생존 법칙으로 등극했다.

하지만 이 아키텍처는 가혹한 기술적 딜레마를 동반한다. 마이크로컨트롤러(MCU)부터 게이트웨이(Fog), 클라우드(Data Center)까지의 스택은 각기 다른 OS, 다른 컴퓨팅 파워, 완전히 다른 네트워크 인프라를 갖고 있다.
기존 환경에서는 MCU용 프로토콜(CoAP/I2C), 게이트웨이용 프로토콜(MQTT), 클라우드용 프로토콜(gRPC, Kafka)을 따로 두어 다리가 세 번 끊기는 기저귀(Adapter) 아키텍처를 덧댔다.

본 절에서는 이러한 거대한 파이프라인의 조각난 경계를 단일 언어로 매끄럽게 봉합하는 Zenoh만의 “Continuum(연속성)” 비전과 단절된 망(Air-gapped) 환경에서의 비상 탈출 전략을 실제 런북(Runbook) 아키텍처 예시와 함께 정립한다.
