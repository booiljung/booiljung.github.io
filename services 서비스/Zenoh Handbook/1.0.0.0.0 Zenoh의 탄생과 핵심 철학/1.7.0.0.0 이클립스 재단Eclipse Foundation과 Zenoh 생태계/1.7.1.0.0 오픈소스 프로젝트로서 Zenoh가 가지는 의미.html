1.7.1 오픈소스 프로젝트로서 Zenoh가 가지는 의미

만약 Zenoh가 특정 스타트업의 전유물이거나, A사(Apple)나 M사(Microsoft) 같은 거대 단일 기업망에 종속된 스택이었다면, 당신의 공장 인프라 생사여탈권을 그 기업의 라이선스 정책 하나에 저당 잡히는 꼴이 된다.

0.1 플랫폼 중립성(Vendor Neutrality) 보장의 방어막

Zenoh 프로젝트가 Eclipse Foundation 소속으로 등록(Incubated)되었다는 것은 다음을 의미한다.

  1. 소스코드의 영구적인 공개 보장: Eclipse Public License v2.0(EPL-2.0) 및 Apache 2.0 듀얼 라이선스 체제로 어떠한 상용 족쇄나 특허 침해 리포그램(Troll)의 위협 없이 코드 자체를 프로덕션에 자유롭게 가져다 쓸 수 있다.
  2. 거버넌스(Governance) 민주화: 특정 회사의 CEO가 아닌, 위원회 중심의 집단 지성 합의체로 프로토콜의 스펙이 정의된다.

0.2 [Runbook] 기술 벤더 락인(Vendor Lock-in) 회피 전략 선언

당신의 팀이 자체 플랫폼을 짤 때 팀장에게 올릴 아키텍처 제언(RFC) 문서에 다음과 같이 명시하라.

“당사는 IoT 메시징 중추로 클라우드 공급자 종속적인 AWS IoT Core SDK나 Azure IoT Hub 전용 프로토콜 대신 Eclipse Zenoh를 채택한다. 향후 5년 뒤 클라우드 인프라를 AWS에서 NCP(Naver Cloud) 혹은 자체 On-Premise 망으로 이전(Migration)하더라도, 통신 미들웨어 및 에지 단말기의 펌웨어 코드는 단 1줄도 수정될 필요가 없는 100% 클라우드 독립성을 확보한다.”

이 선언 하나가 스타트업부터 대기업 아키텍트까지 짊어지고 있는 ’인프라 노예화’라는 짐을 혁명적으로 벗어던질 수 있게 해 줄 것이다.
