1.4.4 네임드 데이터 네트워킹(Named Data Networking, NDN) 개념의 실용적 도입

IP 네트워킹 구조의 낡은 관성을 깨부수기 위해 2010년대 학계에서 탄생한 거대한 연구 테마가 바로 ’네임드 데이터 네트워킹(NDN)’이다. “데이터를 감싼 패킷의 헤더에 목적지의 IP가 아니라 데이터의 이름(계층적 URL 형식)을 적어 라우팅하자“는 우아한 발상이었으나, 전 세계 글로벌 라우터 펌웨어를 다 뜯어고쳐야 한다는 현실적 장벽 탓에 수십 년간 논문 속의 이상향으로만 남았다.

Zenoh가 해낸 일은 이 몽상적인 NDN 패러다임을 **“소프트웨어 오버레이 네트워크(Software Overlay Network)”**라는 실용적 캡슐로 감싸서 현재의 구형 IP 라우터들 위에서도 완벽하게 작동하도록 상용화시킨 것이다.

0.1 실용주의적 NDN 트라이 트리기반(Trie-tree) 라우팅

인터넷 라우터가 IP 대역표(CIDR BGP 테이블)를 보고 라우팅을 하듯, Zenoh 데몬(zenohd)의 뇌 속에는 IP가 아닌 거대한 경로 트리(Trie) 명부가 들어있다.

  • Node 1: factory/robot1/temp/motor
  • Node 2: factory/robot1/camera/color
  • Node 3: factory/robot2/temp/battery

클라이언트가 session.get("factory/robot1/**") 이라는 데이터 쿼리를 날린다.
Zenoh 라우터 엔진은 IP 테이블을 뒤지는 것이 아니라 자신의 메모리에 적힌 트리맵(Trie Map)을 검색하여 이 수많은 이름표 중에 정확히 매칭하는 놈(Node 1, Node 2)을 찾아내고 경로를 개방한다.

0.2 학계의 NDN 한계를 복구한 하이브리드 전략

학계의 순수 NDN은 데이터를 부를 때(Interest) 항상 단방향 당기기(Pull) 구조밖에 없어 Lidar 영상 같은 실시간 비디오 초고속 스트리밍 발사(Push)에는 구조적으로 젬병이었다.
Zenoh는 이 순수 NDN의 광신도적 학구열을 과감히 버리고, 철저히 산업계의 요구에 맞춰 하이브리드 교배를 감행했다.

  • Get/Queryable: NDN 본연의 질의-응답(Pull) 계층을 유지하여 캐시 히트율 극대화
  • Pub/Sub (Push): ROS2를 포섭하기 위해 DDS의 고속 밀어내기(Push) 엔진을 강제 결합

그래서 Zenoh는 완벽한 학문적 NDN 시스템이 아니다. 하지만 지구상에 존재하는 그 어떤 상용 프로토콜보다 가장 NDN의 이상향(이름 기반 라우팅)에 근접하면서도 동시에 자율주행 차를 사고 없이 몰 수 있는 폭력적인 스피드를 겸비한, 가장 현실 타협적인 실용적 데이터 분배 선봉장이다.
