1.4 Zenoh의 핵심 철학과 패러다임 혁신

“훌륭한 기술은 시스템 아키텍처를 뒤엎지만, 위대한 기술은 문제의 본질(패러다임) 자체를 재정의한다.”

Zenoh 개발진은 분산 컴퓨팅의 가장 악랄한 골칫거리였던 ’위치 복잡성’과 ‘시간 파편화’ 문제를 해결하고자 근본적인 아키텍처부터 백지상태에서 시작했다. 기존의 IP 네트워킹 구조를 뒤엎고 공간의 독립성과 시간의 비동기화를 동시에 달성한 Zenoh는 데이터 중심형 설계(Data-centric Design)라는 무기를 들고 나타났다.

이 절에서는 단순히 코드 API를 설명하는 것을 넘어, 서버와 클라이언트, 과거와 현재라는 족쇄를 풀어내는 ‘시간과 공간의 디커플링(Decoupling)’ 철학이 실제 런타임 시스템 안에서 어떤 모습으로 발현되는지 증명한다. 개발자가 아닌 오케스트레이터의 관점에서 이 혁신적인 패러다임을 지배하는 법을 터득하라.
