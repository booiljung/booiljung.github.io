1.6 데이터 추상화와 다중 통신 의미론(Semantics)

REST API는 문서(HTML, JSON)를 교환하는 데 최적화되었고, MQTT는 이벤트를 통보하는 푸시(Push)에 최적화되었다. 하지만 현대 로보틱스와 IoT의 혈관에서는 비디오 스트리밍, 제어 명령 파이프, 센서 로그 캐시 질의 등 수많은 목적이 혼재되어 흐른다. 이 수많은 목적의 전선을 각기 다른 프로토콜로 가설하면 소프트웨어 유지보수 비용은 기하급수적으로 폭발한다.

Zenoh가 “혁신적인 접착제“로 불리는 이유는, **Pub/Sub (단방향 이벤트 방송)**의 속도감과 **Query/Reply (양방향 데이터 질의)**의 편의성을 단일한 엔진 내에서 완벽하게 추상화하여 중첩시켰기 때문이다.

이 절에서는 데이터의 이름을 짓는 URL 계층 트리 원리(Key Expression)부터 시작하여, 폭발적인 발사(Push) 메커니즘과 우아한 수집(Pull) 메커니즘이 Zenoh 내부에서 어떻게 모순 없이 결합되어 라우팅되는지 실전 프로그래밍 패턴을 통해 짚어본다. 당신의 네트워크 뼈대를 단순한 통신 파이프에서 거대한 “분산 상태 데이터베이스“로 진화시키는 핵심 도구 모음이다.
