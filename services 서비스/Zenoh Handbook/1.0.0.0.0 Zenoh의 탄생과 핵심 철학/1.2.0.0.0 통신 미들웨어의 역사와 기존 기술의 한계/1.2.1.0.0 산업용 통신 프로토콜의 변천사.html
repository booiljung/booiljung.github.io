1.2.1 산업용 통신 프로토콜의 변천사

현장의 네트워크는 시대정신에 따라 3단계로 변모해 왔다. 당신이 낡은 공장을 현대화하는 솔루션 아키텍트라면, 아래의 퇴적층을 파악하는 것부터가 컨설팅의 시작이다.

0.1 세대: L2 버스(Bus)와 1:1 결합 구조 (Modbus, CAN)

1980년대부터 공장을 지배한 프로토콜이다. IP 네트워크조차 없던 시절, 구리선 2가닥(RS-485) 위에 직접 전기 신호를 쏘아붙였다.

  • 특징: 결정론적(Deterministic) 속도를 보장하고 지독하게 튼튼하다.
  • 파국 런북 시나리오: 자동차의 CAN 버스는 보안 개념 자체가 없어 선 하나만 피복을 벗기고 스니핑 장비를 물리면 브레이크 제어 명령까지 위조할 수 있다. 하드웨어 종속적이라 클라우드와의 접점이 완벽히 차단된다. 결국 C 언어 기반의 거대한 C++ 어댑터를 중앙 서버에 포진시켜야만 겨우 웹과 소통시킬 수 있다.

0.2 세대: 분산화와 유연성 이식 (MQTT, AMQP)

2000년대 후반, 사물인터넷(IoT)이 폭발하며 모든 단말기를 인터넷(TCP/IP) 위에 올리기 위한 대단결이 시작되었다. Pub/Sub 패턴이 왕좌에 올랐다.

  • 특징: 디커플링(디바이스들이 서로를 몰라도 브로커만 알면 된다)의 매직. JSON 페이로드를 채택하며 웹 개발자들의 폭발적인 환호를 받았다.
  • 파국 런북 시나리오: MQTT 브로커(Mosquitto) IP 하나가 죽으면 수만 대의 단말기가 좀비 상태로 변모한다. 클러스터링을 구성하려 해도 이 동기화 자체가 또 다른 레이턴시 오버헤드를 야기하며, 초당 만 건이 넘어가는 데이터 폭주(Flood) 방어 체계가 부실하다.

0.3 세대: 실시간 제어와 강력한 QoS (DDS - Data Distribution Service)

로켓이나 전투기, ROS2 로봇 등 수백억짜리 장비들이 에러 하나로 박살 나는 것을 막기 위해 미 해군 등이 주도하여 만든 극강의 프로토콜 계층이다.

  • 특징: 브로커(서버)가 없다! 완벽한 서버리스 P2P(멀티캐스트) 망을 구성하고 20개가 넘는 세밀한 QoS 정책으로 극한의 안정성을 뽐낸다.
  • 파국 런북 시나리오: 이 무시무시한 P2P 성능은 방화벽이나 외부 인터넷 클라우드로 넘어가는 순간 개복치처럼 죽어버린다. 서브넷을 넘어가기가 지옥 같으며, C++ 컴파일 타임에 수만 줄짜리 XML(IDL) 설정을 들이부어야만 동작하는 극악의 DX(Developer Experience)를 자랑한다.

Zenoh가 노리는 과녁이 바로 이 지점이다. DDS의 무브로커(P2P) 초고성능과 튼튼함을 계승하되, MQTT의 가벼운 인터넷 이식성을 결합하려는 4세대 통합의 시작점이다.
