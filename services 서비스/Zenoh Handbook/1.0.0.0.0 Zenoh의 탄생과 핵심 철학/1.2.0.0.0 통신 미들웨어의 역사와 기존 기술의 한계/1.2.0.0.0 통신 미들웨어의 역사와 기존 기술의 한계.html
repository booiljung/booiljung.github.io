1.2 통신 미들웨어의 역사와 기존 기술의 한계

Zenoh가 하늘에서 갑자기 뚝 떨어진 외계 타자기의 기술은 아니다. 지난 30년간 산업 인프라 엔지니어들이 지독한 네트워크 불안정성과 싸우며 남긴 투쟁의 산물이자, 기존 레거시 프로토콜들의 유전적 한계를 극복하기 위해 다시 빚어낸 진화의 결정체다.

구축하려는 시스템이 단순한 교보재를 넘어 24시간 365일 무정지(Zero-Downtime)로 굴러가는 공장과 자율주행 차에 탑재되는 순간, 우리는 반드시 직면하게 된다. “가벼워서 썼던 기술(MQTT)은 데이터가 많아지면 죽어버리고, 무겁고 강력한 기술(DDS)은 유지보수 비용과 배포 세팅이 너무 복잡해 팀의 숨을 막히게 한다“는 이율배반적 현실을.

이 절에서는 로보틱스와 IoT 씬(Scene)에서 표준으로 자리매김했던 두 거장이자 적폐, MQTT와 DDS의 실제 운영 환경에서의 비참한 한계치(Bottleneck) 현상을 낱낱이 파헤친다. 당신의 서버 에러 로그가 왜 그토록 아우성쳤는지, 과거의 한계를 복기하며 새로운 네트워크 패러다임을 맞이할 준비를 하라.
