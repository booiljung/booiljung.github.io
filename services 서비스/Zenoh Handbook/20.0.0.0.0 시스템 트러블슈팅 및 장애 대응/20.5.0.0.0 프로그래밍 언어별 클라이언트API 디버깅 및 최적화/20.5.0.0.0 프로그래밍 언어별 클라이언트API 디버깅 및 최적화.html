20.5 프로그래밍 언어별 클라이언트(API) 디버깅 및 최적화

Zenoh의 가장 큰 무기 중 하나는 폴리글랏(Polyglot) 정신이다. C 코드로 펌웨어가 박힌 초소형 센서 컨트롤러부터, TypeScript로 화려하게 렌더링되는 클라우드 대시보드 백엔드, 그리고 Python으로 육중하게 돌아가는 AI 분석 머신까지 모두 동일한 통신 버스(Bus)에 우아하게 엮인다.

하지만 네트워크 코어가 완벽할지언정, 통신을 수행하는 각 프로그래밍 언어의 런타임 자체(Garbage Collector, Async Runtime, Event Loop 등)가 지닌 한계와 메모리 관리 방식의 차이로 인해 클라이언트 애플리케이션 단에서 크래시와 타임아웃이 연발된다.
“데이터를 안 보낸 게 아니라, 보낼 데이터 구조체를 C 언어단에서 malloc 하다가 세그멘테이션 폴트로 뻗었다“든가, “TypeScript의 단일 스레드 이벤트 루프가 1억 개의 JSON을 파싱하느라 블로킹되어 Zenoh 콜백을 처리하지 못했다“는 식의 장애가 빈번하다.

이 절에서는 거시적인 네트워크 망에서 눈을 돌려, 각 애플리케이션의 뼈대를 이루는 언어별 생태계(Rust, C/C++, TypeScript, Python, Go 단)의 취약점을 직접 타격하는 로컬 디버깅 및 코드 튜닝 기술을 집대성한다. 당신이 선택한 언어의 약점을 명확히 인지하고, Zenoh API를 가장 완벽하게 직조하는 코드 장인(Craftsman)으로 도약하라.
