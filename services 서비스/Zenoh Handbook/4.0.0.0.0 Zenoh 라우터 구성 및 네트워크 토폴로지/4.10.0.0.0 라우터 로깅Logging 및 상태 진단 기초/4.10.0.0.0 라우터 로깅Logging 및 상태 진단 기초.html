4.10 라우터 로깅(Logging) 및 상태 진단 기초

로봇이 오작동하거나 대시보드 데이터 갱신이 멈췄을 때, 어리석은 자는 클라이언트 애플리케이션 코드를 뜯어보고, 현명한 자는 네트워크 백본이 내뿜는 진단 패킷(Log)을 들여다본다.

C 언어로 작성되었던 과거의 불친절한 데몬들과 달리, Rust 기반의 Zenoh 코어 라우터는 심장 구석구석을 실시간으로 들여다볼 수 있는 무시무시한 프로파일링 로깅 능력을 내장하고 있다.
로봇 엔지니어가 “라우터가 제 메시지를 버리는 것 같아요!“라고 항의할 때, 인프라 엔지니어는 억울함을 호소할 게 아니라 이 장에서 배울 로깅 레벨 트레이싱(Tracing) 과 Admin API 캐치를 통해 몇 초 몇 밀리초에 그 패킷이 드랍(Drop)되었는지 증거로 제시할 줄 알아야 한다.
