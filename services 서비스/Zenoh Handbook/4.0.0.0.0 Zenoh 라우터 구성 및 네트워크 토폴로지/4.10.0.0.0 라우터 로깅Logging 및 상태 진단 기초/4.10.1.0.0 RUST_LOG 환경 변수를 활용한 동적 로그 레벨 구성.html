4.10.1 RUST_LOG 환경 변수를 활용한 동적 로그 레벨 구성

수많은 데몬 데브옵스 도구 중 RUST_LOG 환경 변수는 Rust 생태계의 성배(Holy Grail)와 같다. zenohd.json5 설정 파일을 건드리지 않고도 라우터의 주둥이(Stdout) 크기를 자유자재로 지휘할 수 있다.

0.1 [Runbook] 라우터 상태 청진기 대기

1. 평시 운영 모드 (Info/Warn Level)
라우터가 부팅 정보나 심각한 오류만 뱉어낸다. K8s Pod 로그에 쓰레기 텍스트가 쌓이는 걸 막는다.

# 기본값이다. 별도 지정이 없으면 적용된다.
RUST_LOG=info zenohd

2. 모듈 핀포인트 로깅 (특정 기능만 현미경 관찰)
전체 디버그를 켜면 터미널이 글자로 폭발해서 아무것도 읽을 수 없다. 만약 “라우팅(Routing) 테이블 동기화” 쪽에 병목이 의심된다면, 그쪽 모듈만 콕 찝어 디버그를 켠다.

# zenoh 코어의 라우팅 모듈만 DEBUG로 투시하고, 나머지는 info로 입을 막는다.
RUST_LOG="info,zenoh::routing=debug" zenohd

3. 스카우팅 장애 원인 집중 해부
“로봇이 자꾸 라우터를 못 찾는다!“는 민원이 접수되었을 때 치는 치트키 커맨드.

# 멀티캐스트 네트워크 관련 패킷 드랍만 찾아낸다.
RUST_LOG="info,zenoh::net::scouting=trace" zenohd

이 로깅 변수는 로봇(클라이언트 단말) 프로그램을 띄울 때(cargo run)도 동일하게 적용되므로, 통신이 막힌 양 끝 단(End-to-end)의 터미널에 이 변수를 걸고 쌍방향 대조를 해라.
