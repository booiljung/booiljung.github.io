4.7 에지(Edge)와 클라우드(Cloud) 간의 통신 최적화

에지(공장, 로봇, 자율주행차)와 클라우드(AWS, GCP 데이터센터) 사이를 가로막는 것은 물리적인 거리(Latency)뿐만이 아니다. 그 사이에는 통신사 망의 무자비한 네트워크 주소 변환(NAT), 기업 보안팀의 방화벽(Firewall), 그리고 대역폭 종량제라는 서늘한 비용 청구서가 도사리고 있다.

초보자는 어떻게든 핑(Ping)만 가면 통신이 성공했다고 믿지만, 진정한 설계자는 TCP 혼잡 제어가 5G 망에서 어떻게 요동치는지, 끊어진 세션이 NAT 테이블에서 만료되기 전에 어떻게 숨통(Keep-Alive)을 틔워 놓는지를 설계해야 한다. 본 절에서는 WAN(Wide Area Network)이라는 거칠고 변덕스러운 바다 위에서 라우터를 띄우고 보호하는 “지속적 연결 유지 전술“을 런북으로 기술한다.
