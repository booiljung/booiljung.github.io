4.7.3 클라우드 인스턴스(AWS, GCP, Azure) 간의 Zenoh 브릿징

자율주행 아키텍처가 전 세계 규모로 팽창하면, “미국 자동차(GCP US) ➔ 미국 관제 클라우드 ➔ 한국 관제 클라우드(AWS Asia)“처럼 클라우드-투-클라우드(C2C) 데이터 라우팅이 필수적이다.
이 거대한 해저 케이블 위에 Zenoh 백본망을 깔아 Multi-Region Cluster를 구축하는 방안이다.

0.1 [Runbook] 대륙 간 백본 듀얼 커넥트 아키텍처

한 네트워크가 무너져도 다른 리전(Region)에서 즉각 백업(Failover)이 일어나도록, 라우터끼리의 1:N 거미줄 결속(Bridging)을 도면화해야 한다.

1. AWS Asia(메인) 라우터 실행

{
  id: "AWS_BACKBONE_ASIA"
  // 여기서 뻗어나가는 대신, 다른 라우터의 접근을 허용
  listen: { endpoints: ["quic/0.0.0.0:7447"] }
}

2. GCP US(에지 격) 라우터 실행
비용이 비싸고 드랍율이 높은 국제 구간 패킷 전송을 위해 무조건 TCP 대신 QUIC을 주력 화살로 꺼내 든다.

{
  id: "GCP_BACKBONE_US",
  connect: {
    endpoints: [
      // 1순위: 메인 아시아 데이터센터 연결
      "quic/aws-asia-main.myapp.com:7447",
      // 2순위: 메인 망 단절시 유럽 데이터센터로 자동 우회
      "quic/aws-euro-fallback.myapp.com:7447"
    ]
  }
}

[라우팅 선언(Declarations) 병합 효과]
GCP 라우터 아래에 아무리 많은 미국 차량들이 달라붙어도, GCP 라우터가 이 트래픽을 단일한 ZBytes 청크(덩어리)로 묶어(Consolidation) 해저 케이블로 쏘아 보낸다. BGP(Border Gateway Protocol) 못지않은 애플리케이션 레벨의 L7 클러스터링이 단 2줄의 connect 코드로 완성되는 것이다. 이것이 Zenoh의 가장 마법적인 아키텍처 추상화다.
