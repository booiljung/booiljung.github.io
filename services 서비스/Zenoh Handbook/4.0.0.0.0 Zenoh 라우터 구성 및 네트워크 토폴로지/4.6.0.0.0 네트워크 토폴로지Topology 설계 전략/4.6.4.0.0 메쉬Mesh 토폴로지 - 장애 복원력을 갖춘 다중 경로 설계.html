4.6.4 메쉬(Mesh) 토폴로지 - 장애 복원력을 갖춘 다중 경로 설계

트리(Tree) 토폴로지의 유일한 약점은, 가지를 담당하는 에지 라우터가 죽으면 그 밑에 매달린 수백 대의 로봇 통신망 전체가 고립(Isolate)된다는 것이다.
이를 막기 위해 거미줄처럼 이중, 삼중의 여분 선(Redundant Links)을 꼬아놓는 형태를 메쉬(Mesh)라 부른다.

0.1 [Runbook] 다중 연결(Multi-connect) 무한 루프 회피 런북

클라이언트 하나가 라우터 A, 라우터 B에 동시에 화살표를 꽂아 두었다고 치자.(zenohd -e tcp/A -e tcp/B)
로봇이 “헬로“를 던지면 A, B 두 라우터가 동시에 패킷을 복제하여 클라우드로 쏜다?
아니다. Zenoh 라우터 엔진 내부에는 패킷이 뱅글뱅글 맴도는 브로드캐스트 스톰(Storm)을 막기 위한 지능형 브레인(Spanning Tree)이 작동한다.

[운영계 Mesh 연결 치트키] 무지성 이중화 코딩

클라이언트 앱을 기동할 때 그냥 배열로 복수 개의 타겟을 담아라.

// 로봇 클라이언트 설정파일
{
  connect: {
    // A라우터가 폭발하면 자동으로 B라우터로 파이프라인 우회(Failover)
    endpoints: [
      "tcp/192.168.10.10:7447", // 기본 Main 라우터
      "tcp/192.168.10.20:7447"  // 백업 Fallback 라우터
    ]
  }
}

에지 라우터들끼리도 링(Ring) 구조로 connect 배열을 물고 물리게 만들면, 굴착기가 지나가다 공장 LAN 케이블 하나를 끊어 먹어도(Link Down), Zenoh 코어가 눈 깜짝할 새(10ms 이내)에 우회 경로(Alternative Path)를 찾아 패킷을 전송하는 완벽한 자가 치유(Self-Healing) 그물망이 형성된다.
