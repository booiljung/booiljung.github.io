4.6.3 트리(Tree) 토폴로지 - 계층적 데이터 병합 및 확산

데이터센터가 중앙에, 각 공장의 생산 라인별로 게이트웨이 PC가, 그리고 그 아래 센서 노드들이 주렁주렁 매달리는 전형적인 계층적(Hierarchical) 모델이다.
Zenoh가 전 세계 어떤 프로토콜보다 압도적으로 뛰어난 역량을 뽐내는 무대가 바로 이 트리 아키텍처다.

0.1 [Runbook] 트리 기반 지능형 데이터 융합(Consolidation) 전술

하위 로봇들이 모두 중앙 클라우드로 데이터를 맹목적으로 퍼올리면(Push), 국가 간 백본 요금(Data Egress Cost)만 수백만 원이 청구된다.

1. 파이프를 자르는 Queryable 전술

  • 상위 클라우드 라우터에서 “데이터베이스에 초당 한 번씩 밀어 넣는 행위“를 끊어버려라.
  • 트리 가지(Branch)에 해당하는 에지 라우터(edge_router_1)에만 Storage 플러그인을 붙여서, 로봇들의 라이브 데이터를 지역 캐싱(Local Caching) 하게 한다.

2. 필요할 때만 당겨 올리는(Pull) 나무 타기
클라우드 대시보드 관리자가 GET /factory/1/robot/*/battery 쿼리를 날린다.

  1. 클라우드 최상단 라우터는 이 질문을 하위 edge_router_1 로만 전달(Forward)한다. (다른 공장 라우터로는 트래픽을 아예 주지 않는다!)
  2. edge_router_1은 자기가 캐싱해 둔 10대 로봇의 데이터를 1개의 ZBytes 배열 덩어리로 병합(Consolidation) 한 뒤, 상위 클라우드로 배달한다.

이 트리 설계 원칙에 따르면, 평상시 클라우드로 올라가는 Idle 트래픽이 0 Byte가 되는 마법을 달성할 수 있다. 라우터를 계급화(계층화) 시키는 것, 이것이 데이터 대역폭 최적화의 알파이자 오메가다.
