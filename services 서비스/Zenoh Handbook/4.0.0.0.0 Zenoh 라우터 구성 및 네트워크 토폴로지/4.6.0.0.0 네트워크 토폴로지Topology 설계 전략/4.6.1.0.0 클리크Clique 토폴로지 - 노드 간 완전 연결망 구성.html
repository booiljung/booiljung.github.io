4.6.1 클리크(Clique) 토폴로지 - 노드 간 완전 연결망 구성

클리크(Clique)는 수학적 그래프 이론에서 “모든 정점이 서로 다이렉트로 연결된 완전 그래프(Full Mesh)“를 뜻한다. 중간에 릴레이를 거치는 허브(라우터) 없이, 피어(Peer) 10대가 있다면 무려 45개의 거미줄 같은 직접 링크(TCP/UDP 세션)가 뚫려버리는 궁극의 탈중앙화 형태다.

0.1 [Runbook] 클리크 모드의 득과 독

[최고의 응답성 (Latency)]
로봇 A가 B에게 비상 정지 신호를 쏘려 할 때 거치는 홉(Hop) 카운트가 완벽한 0이다. 통신이 OS 커널에서 곧장 상대방 커널로 내리꽂힌다. 로봇 팔의 협동 제어(Co-arm)처럼 1ms 미만의 지연이 요구되는 폐쇄망에서만 예외적으로 쓴다.

[안티 패턴: 연결 폭증(Conn Explosion) 주의보]

  • 제약: 노드 수가 50개를 넘어가는 순간, 각 노드는 49개의 활성 소켓 방어를 유지하느라 파일 디스크립터(FD) 낭비와 Keep-Alive 헬스체크 트래픽 폭발로 인해 CPU 코어가 100%를 친다.
  • 아키텍처 런북: 노드 수가 적고 고정된 특수망(자동차 내부 CAN/이더넷 통합망 등)에만 쓴다. 이때 모든 노드의 모드는 zenoh-cli --mode peer 로 동작해야 하며 라우터(zenohd)는 배제된다.
