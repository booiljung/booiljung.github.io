4.6.5 동적 환경(로보틱스, 드론)에서의 유동적 토폴로지 변경 대응

드론 군집이 하늘을 날며 대형을 수시로 흩트리고 다시 뭉친다. 어제까진 A 드론이 클러스터 헤드(Head) 역할을 하며 주변 친구들을 통신 중계해 줬는데, 오늘 배터리가 떨어져서 그 역할을 이탈했다.
기존 브로커 시스템이라면 중앙 관제소가 일일이 “앞으로는 B 드론을 향해서 쏴라!” 라고 동적 재할당 명령(Configuration Update)을 내려줘야 했을 것이다.

0.1 [Runbook] Ad-hoc 망에서의 유동적 적응 스크립트

Zenoh는 이 기이한 환경(Mobile Ad-hoc Network)을 위해 ’피어 투 라우터 융합형’의 극단적 모드를 지원한다.

[생존 본능 모드: Peer + Scouting]
드론 단말기의 코드를 Client 모드로 박지 마라. 군집 비행 같은 초가변적 환경에서는 오직 Peer 모드 만이 스스로 살아남는다.
코드 하나 고칠 필요 없이 skouting 기능만 살려둔다.

let mut config = Config::default();
// 나는 평민(Peer)이지만 반경 100m 주변 와이파이 망을 이잡듯이 뒤지겠다.
config.insert_json5("scouting/multicast/enabled", "true").unwrap();
let session = zenoh::open(config).res().await.unwrap();

[자가 치유 시나리오 발생]

  1. 10대의 드론이 무질서하게(Mesh) 얽혀 통신한다.
  2. 5대 그룹(A)과 5대 그룹(B)이 산봉우리 양옆으로 갈라져 통신이 단절(Network Partition)된다.
  3. Zenoh는 당황하지 않고 A 내부만의 독립적인 라우팅 테이블(DHT) 재편을 수행하여 자기들 안에서 충돌 없이 논리망을 분할 유지한다.
  4. 산봉우리를 다시 넘어 만나는 순간, 10대가 쏘아대는 멀티캐스트 비콘이 교차하며 0.1초 만에 두 망(A+B)의 해시 테이블이 하나로 병합되고 끊겼던 통신 링크가 기적처럼 부활한다.

네트워크 파티셔닝(Partitioning)이라는 컴퓨터 공학의 난제를 Zenoh는 피어 모드(Peer)의 자율 디스커버리 짬짜미만으로 우아하게 해결해 버렸다.
