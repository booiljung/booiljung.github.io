4.4 엔드포인트(Endpoint) 및 트랜스포트(Transport) 구성

Zenoh는 데이터 페이로드(ZBytes)를 어떻게 포장할지에 대한 논리적 껍데기일 뿐이다. 실제 데이터가 날아가는 물리적 도로는 밑바닥의 트랜스포트 계층(Transport Layer) 이 담당한다.

초보자들은 디폴트 설정인 TCP 단일 회선만을 쓰며 “Zenoh가 왜 이리 끊기냐“고 투덜댄다. 하지만 1,000대의 센서 노드가 쏟아내는 지연을 막으려면 TCP, UDP, QUIC, WebSocket을 목적에 맞게 병렬 조합(Multiplexing)하는 네트워크 설계자의 시야가 필요하다.

로컬 망 안에서는 저지연 UDP를 타다가, 클라우드로 올라갈 때는 보안이 걸린 QUIC으로 갈아타고, 관제실 대시보드는 WebSocket으로 빨아들이는 다장르 오케스트레이션이 바로 이 엔드포인트 세팅에서 갈린다.
