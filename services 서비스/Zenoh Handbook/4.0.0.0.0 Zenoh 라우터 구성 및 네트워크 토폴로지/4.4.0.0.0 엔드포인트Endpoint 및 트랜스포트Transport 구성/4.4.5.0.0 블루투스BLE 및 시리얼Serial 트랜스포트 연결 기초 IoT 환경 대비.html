4.4.5 블루투스(BLE) 및 시리얼(Serial) 트랜스포트 연결 기초 (IoT 환경 대비)

TCP/IP 네트워크 칩셋조차 달 수 없는 극단적인 임베디드 장비(Arduino, 낡은 PLC 계측기)나 BLE만 달린 웨어러블 장비를 백본 라우터에 우겨넣으려면 어떻게 해야 할까?
이 영역은 IP 체계를 벗어난 L2 무선 밴드 및 물리적 시리얼 통신선(RS232/485)으로 진입한다.

0.1 [Runbook] Serial/BLE 브릿지 프로토콜 매핑

현재 Zenoh 메인 데몬은 TCP/UDP/QUIC을 주력으로 다루며, Serial/BLE 백엔드를 다루려면 플러그인 또는 C(zenoh-pico) 코어 래핑이 필요하다.

1. USB 시리얼(Serial) 통신 런북 (Linux/Mac)
장비가 /dev/ttyUSB0 로 붙었다면, 라우터 측에서 시리얼 채널을 파이프라인으로 연다.

// [실험적 기능 주의]
{
  listen: {
    // 115200 Baudrate 규격 시리얼 포트를 Zenoh의 L4 트랜스포트처럼 승격시킴
    endpoints: ["serial//dev/ttyUSB0?baud=115200"]
  }
}

2. 에지 마이크로컨트롤러(zenoh-pico) 직결
센서에서 C언어로 zenoh-pico 트랜스포트를 짤 때, IP 스택 대신 UART나 블루투스 L2CAP 프로토콜로 직렬화 바이트를 던지게 만든다.

(이 부분은 10장 ‘Zenoh-Pico’ 마이크로컨트롤러 파트에서 완벽한 C 코드 펌웨어 플래싱 레벨업으로 연계되므로, 아키텍트 입장에서는 “IP망이 끊겨도 물리선 브릿지로 망을 통합시킬 수 있다“는 설계적 가능성만 인지하고 넘어가면 된다)
