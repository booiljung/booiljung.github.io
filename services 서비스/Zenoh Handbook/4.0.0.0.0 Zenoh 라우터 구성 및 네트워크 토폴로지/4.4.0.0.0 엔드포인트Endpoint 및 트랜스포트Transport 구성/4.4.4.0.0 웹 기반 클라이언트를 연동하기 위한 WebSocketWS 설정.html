4.4.4 웹 기반 클라이언트를 연동하기 위한 WebSocket(WS) 설정

Chrome 브라우저에서 돌아가는 Vue.js/React 관제 화면은 웹 브라우저 샌드박스 정책으로 인해 Native TCP나 UDP 소켓을 절대 못 연다. “관제 화면 구축” 프로젝트가 떴다면, 라우터 측에서 프론트엔드가 비벼볼 수 있는 WebSocket(WS) 문을 개방해 줘야 한다.

0.1 [Runbook] 라우터 웹소켓 개방

[zenohd.json5 세팅]
단순히 127.0.0.1로 열면 Vue.js 클라이언트(개발자 맥북)가 리눅스 서버 라우터에 접속을 못 한다. 철저하게 0.0.0.0 으로 포트를 열어 제껴라.

{
  listen: {
    // 내부 로봇들은 TCP 7447로 들어오고,
    // 외부 브라우저(대시보드)들은 웹소켓 8000으로 들어와 통신망을 합친다.
    endpoints: ["tcp/0.0.0.0:7447", "ws/0.0.0.0:8000"]
  }
}

[프론트엔드 Typescript 런북]
관제 웹 프론트(Vue.js) 측에서는 아래와 같이 접속 파라미터를 강제한다.

import * as zenoh from '@eclipse-zenoh/zenoh-ts';

// [주의] 브라우저 환경에서는 반드시 "ws://" 프로토콜을 백엔드로 지어라.
const conf = zenoh.Config.empty();
conf.insertJson5("connect/endpoints", `["ws://13.124.x.x:8000"]`);

console.log("웹 브라우저에서 Zenoh 생태계로 진입 시도...");
const session = await zenoh.open(conf);

이제 웹 프론트엔드 개발자도 그 지긋지긋한 HTTP Polling이나 Socket.IO 서버 개발에서 해방되어, 로봇이 TCP로 쏘는 데이터를 “브라우저 상에서” 다이렉트로 Pub/Sub 캐치하는 무서운 직결 망이 완성된다.
