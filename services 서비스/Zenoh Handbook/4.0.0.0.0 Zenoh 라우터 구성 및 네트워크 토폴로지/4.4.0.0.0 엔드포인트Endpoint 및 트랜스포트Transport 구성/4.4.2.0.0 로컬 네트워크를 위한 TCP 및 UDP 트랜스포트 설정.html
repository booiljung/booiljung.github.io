4.4.2 로컬 네트워크를 위한 TCP 및 UDP 트랜스포트 설정

공장이나 사무실처럼 NAT(Network Address Translation) 구간을 넘지 않는 Local Area Network(LAN) 안에서 가장 견고하게 트래픽을 처리하는 런북이다.

0.1 [Runbook] TCP/UDP 혼용 하이브리드 바인딩

하나의 라우터가 온도 센서처럼 안 잃어버려야 할 데이터는 TCP 포트로 받고, 라이다(LiDAR)처럼 던져야 하는 데이터는 UDP 포트로 받는 양방향 인터페이스 개방 전술이다.

// zenohd.json5
{
  listen: {
    endpoints: [
      "tcp/0.0.0.0:7447", // 제어/상태 트래픽 수신용 신뢰성 채널
      "udp/0.0.0.0:7448"  // 미디어/센서 트래픽 수신용 초저지연 채널
    ]
  }
}

[클라이언트 프로그래밍 런북] 강제 포트 매핑:
단말을 켤 때, 반드시 데이터 속성에 따라 포트를 지정해야 한다.

// 일반적인 명령/상태 제어 클라이언트 (TCP)
let config = Config::from_env().unwrap(); // 기본 TCP 7447 탐색
let tcp_session = zenoh::open(config).res().await.unwrap();

// 무지막지한 비디오 스트림 전송 전용 클라이언트 (UDP 강제 배정)
let mut udp_config = Config::default();
udp_config.insert_json5("connect/endpoints", "[\"udp/192.168.1.100:7448\"]").unwrap();
let udp_session = zenoh::open(udp_config).res().await.unwrap();

“이 데이터가 유실되었을 때 로봇이 멈추는가?” 라는 질문에 “No“라면 과감하게 클라이언트 연결을 UDP로 빼서 TCP 포트의 소켓 파열을 예방하라.
