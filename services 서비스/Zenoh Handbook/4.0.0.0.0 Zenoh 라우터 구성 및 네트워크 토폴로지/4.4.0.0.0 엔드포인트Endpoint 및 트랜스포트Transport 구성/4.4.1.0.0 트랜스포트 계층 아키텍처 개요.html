4.4.1 트랜스포트 계층 아키텍처 개요

네트워크 대역폭을 설계할 때 Zenoh의 트랜스포트 레이어는 다형성(Polymorphism)을 가진다. 동일한 Publish 코드를 실행시켜도, 바닥에 깔린 트랜스포트가 무엇이냐에 따라 패킷의 포장 방식(MTU 사이즈, 핸드쉐이크 오버헤드)이 극명하게 달라진다.

0.1 연결형 보안 프로토콜 (QUIC, TLS/TCP)

  • 특징: 중간에 패킷이 유실되면 커널이나 런타임이 재전송을 보장한다. 암호화가 기본이다.
  • Runbook 타겟: 클라우드 EC2와 로컬 공장 엣지 라우터를 연결하는 장거리 백본(Backbone).

0.2 비연결성 날것 프로토콜 (UDP, Shm)

  • 특징: 핸드쉐이크 없이 그냥 데이터를 IP 주소창에 쑤셔 넣고 끝낸다(Fire & Forget).
  • Runbook 타겟: 한 공장 내에서 실시간 로봇 비전 이미지(H.264)를 관제 PC로 쏠 때. 유실되어도 다음 프레임이 바로 날아오므로 재전송 대기(Head-of-Line Blocking)가 없는 UDP가 무조건 유리하다. (나아가서 같은 서버 내통신이면 Share Memory(Shm) 커널 바이패스를 태워라)
