4.2.3 백그라운드 데몬(Daemon) 및 시스템 서비스(Systemd) 등록

앞선 3.2.1 장에서 기초 패키지 설치를 다뤘지만, 여기서는 순수 아키텍트 관점에서의 리눅스 systemd 통제 철학을 깊게 판다.
공장 현장에서 KVM 가상머신이 뻗었다가 재부팅되었을 때 “라우터 왜 안 켜졌어?” 라는 비명을 듣기 싫다면 무조건 아래의 서비스 유닛(Unit) 텍스트를 인프라에 발라(Bake) 두어야 한다.

0.1 [Runbook] Production 환경 데몬 유닛(Unit) 작성의 정석

단순히 껐다 켜는 것이 아니라, 강력한 리소스 격리(Cgroups) 를 걸어서 라우터 프로세스가 미쳐 날뛰어 리눅스 전체를 죽이는 걸 막아야 한다.

sudo nano /etc/systemd/system/zenohd.service 파일에 다음 무기를 주입한다.

[Unit]
Description=Zenoh Backbone Router Daemon
# 네트워크가 확실히 살아난 뒤에야 이 데몬을 올리라는 순서 보장 방벽
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=zenoh
Group=zenoh

# 핵심 1: 절대 권한 파일 포인터. 실행 위치 지정.
ExecStart=/usr/bin/zenohd -c /etc/zenoh/zenohd.json5

# 핵심 2: 오토 서바이버(Auto-Survivor) 옵션
Restart=on-failure
RestartSec=3

# 핵심 3: 커널 자원 제한 (Cgroups Limits)
# 좀비 소켓 커넥션이 폭증해도 최대 파일 식별자(File Descriptors)를 늘려 수용 능력을 극대화한다.
LimitNOFILE=1048576
# [안전장치] 만약 라우터가 캐싱을 과도하게 해서 서버 RAM의 80% 이상을 먹으면 OS가 강제로 라우터를 죽이게 한다(재부팅되게).
MemoryMax=8G

[Install]
WantedBy=multi-user.target

이 코드를 ansible-playbook에 담아 수십 대의 에지 게이트웨이(Edge Gateway)에 뿌리고, systemctl enable --now zenohd 로 숨통을 트이게 하라. 그것이 진정한 백엔드 운영이다.
