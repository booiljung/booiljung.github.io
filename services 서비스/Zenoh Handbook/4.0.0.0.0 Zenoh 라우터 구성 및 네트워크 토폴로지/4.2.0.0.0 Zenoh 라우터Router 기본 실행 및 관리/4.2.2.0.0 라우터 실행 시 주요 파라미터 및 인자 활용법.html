4.2.2 라우터 실행 시 주요 파라미터 및 인자 활용법

CLI에 플래그(Flag)를 다는 건 빠르지만, 명령어가 길어지면 휴먼 에러를 유발하고 재현성이 떨어진다.
따라서 복잡한 포트포워딩과 플러그인 셋업은 전부 json5 포맷의 설정 파일(Config)로 몰아넣고, CLI에서는 딱 하나 설정 파일의 주소만 인자로 넘기는 것이 인프라 엔지니어링의 정석이다.

0.1 핵심 CLI 인자 런북

1. 설정 파일 매핑 (-c)

# /etc 템플릿에 박아둔 JSON5 설정 파일을 기반으로 영혼을 주입한다
zenohd -c /etc/zenoh/production.json5

2. 다른 라우터와의 브릿지 연결 선언 (-e, Connect to)
이 라우터가 최고 조상(Root)이 아니라면, 기동하면서 상위 클라우드에 뻗치게 만들어야 한다.

# 내가 켜짐과 동시에 AWS에 있는 도쿄 리전(Tokyo Region) 마스터 라우터와 백본 파이프를 튼다.
zenohd -e tcp/aws-master.com:7447

3. 도메인 / 세그먼트 아이디 분리 (-i, Router ID)
로컬 망에 A팀, B팀 라우터가 공존할 때 라우터 혼선을 막기 위한 이름표(ID)를 찍는다.

# 16진수 아이디 부여
zenohd --id A0B1C2

0.2 환경변수 덮어쓰기 (Environment Overrides)

운영 스크립트를 변경할 수 없는 도커(Docker) 환경에서는, json5 파일을 수정하는 대신 환경변수(ZENOH_xxx)로 인자를 강제로 내리찍는 방법이 최고다. Docker의 섭리를 따르라.

# JSON5의 "transport.multicast.ipv4_address" 설정을 쉘 변수로 덮어쓰기
export ZENOH_CONNECT="tcp/remote-router:7447"
export ZENOH_LISTEN="tcp/0.0.0.0:7447"
zenohd
