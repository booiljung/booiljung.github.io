4.2.1 CLI(Command Line Interface)를 통한 라우터 단일 실행

새로운 장비를 납품할 때나 오만가지 네트워크 환경(TCP, QUIC, UDP)이 꼬였을 때, 가장 먼저 빼들어야 할 메스(스캘펠)는 설정 파일 파싱 없이 오직 CLI 플래그만으로 기동하는 단일 인스턴스 띄우기다.

0.1 [Runbook] CLI 치트키 - 망의 상태를 증명하는 1회성 라우터 가동

1. 로컬 개발자 전용 (기본 실행)

# 그냥 켜면 로컬에서 7447 포트를 잡고 스스로를 라우터로 선언한다.
zenohd

2. 특수 포트 바인딩 (다른 프로세스와 충돌 시)

  • 로컬의 7447 포트를 이미 ROS2가 물고 있거나 브릿지가 차지했다면 강제로 비껴간다.
# TCP 8000번, UDP 8000번(스카우팅)에 귀를 대도록 바인딩 포인트를 강제로 지정
zenohd -l tcp/0.0.0.0:8000

3. 스카우팅(Multicast) 강제 차단

  • “내가 띄우는 이 라우터는 다른 옆 팀의 로봇들이 자동으로 스카우팅해서 붙지 못하게 만들어라!” (보안 및 트래픽 폭주 방지)
# --no-multicast-scout 옵션으로 자동 연결의 문을 걸어 잠근다.
# 오직 이 IP를 명시적으로 알고 있는(-e tcp/...) 클라이언트만 들이게 된다.
zenohd --no-multicast-scout

-l tcp/127.0.0.1:7447 로 켜면 이 라우터는 오직 서버 자기 자신 안에서만 쓸 수 있다. 클라이언트 PC가 절대 외부에서 치고 들어오지 못한다. 외부 로봇들을 받아주려면 반드시 -l tcp/0.0.0.0:7447 (자신의 모든 NIC 개방) 상태로 구동해야 함을 망 각하지 마라.
