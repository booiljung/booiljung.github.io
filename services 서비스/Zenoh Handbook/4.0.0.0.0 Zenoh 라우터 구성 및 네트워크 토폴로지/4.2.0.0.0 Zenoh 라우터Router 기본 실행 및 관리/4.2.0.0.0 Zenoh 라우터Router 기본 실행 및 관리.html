4.2 Zenoh 라우터(Router) 기본 실행 및 관리

마스터 노드에서 라우터를 실행시키는 것은 분산 시스템의 척추를 세우는 것과 같다. 이 척추가 구부러지거나(잘못된 설정) 약하면(자원 제약) 시스템 전체가 속앓이를 한다.

“터미널에서 zenohd 치면 켜지는 거 아니야?” 라고 묻는다면 그건 취미용 홈 IoT(스마트 전구) 수준의 발상이다.
기업용 상용망(Production)에서는 서버 데몬이 어떤 네트워크 인터페이스를 물고 있는지, 메모리 누수나 크래시가 났을 때 리눅스 커널 수준에서 어떻게 백오프(Backoff)하고 자가 치유(Auto-Restart)할 것인지, 도커 환경에서는 네트워크 브릿지를 어떻게 탈출할 것인지 철저한 런북이 필요하다.

본 절에서는 단순 CLI 구동부터 Systemd 데몬 등록, 그리고 쿠버네티스(k8s)의 초석이 되는 Docker 컨테이너 패키징까지 라우터 엔진의 고삐를 쥐는 방법을 서술한다.
