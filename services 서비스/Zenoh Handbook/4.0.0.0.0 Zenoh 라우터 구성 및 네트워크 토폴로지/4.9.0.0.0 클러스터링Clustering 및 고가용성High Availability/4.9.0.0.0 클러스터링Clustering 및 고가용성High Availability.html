4.9 클러스터링(Clustering) 및 고가용성(High Availability)

아무리 잘 짜인 런타임 언어(Rust)와 튜닝된 파라미터가 있다 한들, 클라우드 데이터센터의 전력 케이블이 뽑히거나 AWS 리전(Region) 통째로 다운되는 재앙 앞에서는 단일 라우터(Single Point)는 허무하게 증발한다.

“내 로봇은 클라우드가 죽어도 계속 생산 공정을 유지해야 한다”
이 가혹한 요구사항(SLA 99.999%)을 만족하려면 설계자는 라우터를 단독 데몬이 아닌 2대, 3대로 묶인 운명 공동체(Cluster) 로 격상시켜야 한다.
이 절에서는 라우터를 이중화(Active-Active)하는 방법과, 네트워크가 반갈죽(Split-brain)났을 때의 재난 복구(Disaster Recovery) 아키텍처를 설계하는 데브옵스의 고등 런북을 전수한다.
