4.8.2 REST API 플러그인 설정 및 HTTP 클라이언트 연동

레거시 프론트엔드 개발자나 외부 제휴사는 Zenoh가 뭔지 모른다. 그들에게 “우리 회사의 자율주행 택시 위치를 가져가라“며 Zenoh C++ 라이브러리를 던져주면 욕을 먹는다.
우리는 포용력 있는 백엔더로서, 그들이 친숙하게 쓰는 범용 HTTP REST API 문을 열어줄 의무가 있다.

0.1 [Runbook] HTTP ➔ Zenoh 터널 개방

1. zenohd.json5에서 REST 플러그인 이그니션(Ignition)

{
  plugins: {
    // REST API 프록시 서버를 컨테이너 내 8000번 포트에 기동한다.
    rest: {
      http_port: 8000
    }
  }
}

2. [타 팀 제공용 런북] cURL 호출 명세서
이제 백엔드 엔지니어는 제휴사에게 아래와 같은 REST 명세서만 주면 파이프라인 연동이 끝난다.

# 외부 개발자가 우리의 8000번 포트를 치는 순간...
# [GET] = Zenoh Query 로 번역되어 전 세계 로봇망으로 흩뿌려진다!
curl -X GET http://router-ip:8000/taxi/*/location

# [PUT] = Zenoh Publish 로 번역되어 구독 중인 로봇들에게 꽂힌다!
curl -X PUT -T "stop_flag.json" http://router-ip:8000/taxi/10/cmd

# [POST] (SSE) = Server-Sent Events로 구독 파이프라인을 브라우저에 연다!
curl -X POST http://router-ip:8000/taxi/10/status

이 갓(God)-플러그인 하나로 인해 당신은 “중간 API 웹 패칭 서버“를 Node.js로 따로 짜야 하는 1달짜리 공수에서 해방된다.
