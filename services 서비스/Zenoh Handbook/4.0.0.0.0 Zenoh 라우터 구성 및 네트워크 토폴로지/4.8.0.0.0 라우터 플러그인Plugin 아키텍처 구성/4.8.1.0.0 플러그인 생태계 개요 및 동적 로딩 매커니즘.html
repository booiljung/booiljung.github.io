4.8.1 플러그인 생태계 개요 및 동적 로딩 매커니즘

플러그인은 독립된 실행 파일(Binary)이 아니라, 리눅스에서는 .so, 윈도우에서는 .dll 확장자를 가진 동적 링크 라이브러리(Dynamic Link Library) 파일이다.

0.1 [Runbook] 플러그인 인스펙션 및 장착법

1. 내 라우터가 가진 무기 확인
라우터를 띄운 서버에서 zenohd --help를 쳐서 플러그인 리스트가 잡히는지 확인하라. 빌드할 때나 패키지 매니저로 깔 때 zenoh-plugin-rest, zenoh-plugin-storage-manager 등이 포함되지 않았다면 플러그인 생태계는 작동하지 않는다.

2. 동적 로딩 경로 튜닝
때때로 zenohd는 플러그인 .so 파일의 주소를 못 찾아서 플러그인을 무시한다(Silent load failure). OS 환경변수로 플러그인 이빨이 맞물리게 강제 연결하라.

# Ubuntu 환경 - 플러그인 링킹 에러가 났을 때 경로를 덮어쓰는 치트키
export LD_LIBRARY_PATH=/usr/lib/zenoh:$LD_LIBRARY_PATH
zenohd

[아키텍처 인스펙션] 플러그인은 코어 프로세스의 힙(Heap) 메모리를 공유한다. 만약 잘못 짜인 커스텀 플러그인(C/Rust)이 메모리 누수를 일으키면 고결한 Zenoh 라우터 핵심 코어스레드까지 연대 책임으로 죽어버린다. 서드파티 플러그인을 운영망에 올리기 전에는 반드시 샌드박스에서 누수 테스트를 마쳐라.
