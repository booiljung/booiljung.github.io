4.8.4 커스텀 플러그인 개발 및 적용을 위한 라우터 인터페이스

주요 클라이언트의 요구사항은 언제나 기본 기능을 벗어난다. “Zenoh로 들어온 데이터를 실시간으로 AI 침입 감지 필터에 태웠다가 내보내고 싶다.”
이럴 때는 라우터 위에서 자체 커스텀 로직이 돌 수 있도록 벤더(Vendor) 전용 플러그인을 Rust 모듈로 구워내야 한다.

0.1 [Runbook] 커스텀 플러그인 스캐폴딩 뼈대

완벽한 고성능을 위해 커스텀 플러그인은 반드시 Rust로 작성되어야 한다.

  1. 카고 라이브러리(cdylib) 선언:
  • 이 프로그래머블 코드가 실행 바이너리가 아닌, OS가 링크할 수 있는 라이브러리(.so/.dll)임을 컴파일러에 못 박는다.
# Cargo.toml
[lib]
crate-type = ["cdylib"]

[dependencies]
zenoh = "0.10"
zenoh-plugin-trait = "0.10" # 라우터와 몸통을 도킹하기 위한 특이점!
  1. 메모리 후킹(Hook) 인터페이스 구조:
use zenoh_plugin_trait::{Configuration, Plugin, PluginControl};

// 플러그인 등록 매크로 (zenohd 가 부팅될 때 이 심볼을 찾아 로딩한다)
zenoh_plugin_trait::declare_plugin!(MyCustomAIFilter);

pub struct MyCustomAIFilter;

impl Plugin for MyCustomAIFilter {
    type StartArgs = ();
    // 플러그인 칩셋이 꽃히는 순간 실행되는 콜백!
    fn start(_name: &str, _conf: &Configuration, _args: Self::StartArgs) -> Result<PluginControl, ()> {
        println!("[Alert] AI 커스텀 필터가 라우터 메인 메모리에 탑재되었습니다.");
        
        // 이곳에서 로컬 세션(local.session)을 열고 들어오는 데이터를 조작(Interceptor) 파이프라인으로 연결
        
        Ok(PluginControl {})
    }
}

이 코드를 cargo build --release로 구워낸 libmy_custom_ai_filter.so 파일을 라우터의 $LD_LIBRARY_PATH 디렉터리에 던져주면 나만의 독점적 생태계가 탄생한다.
