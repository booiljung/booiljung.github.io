4.8 라우터 플러그인(Plugin) 아키텍처 구성

Zenoh 라우터는 그 자체로는 데이터를 중계(Relay)만 하는 깡통(Core)에 불과하지만, 플러그인이라는 칩셋을 꽂는 순간 HTTP 웹서버가 되기도 하고 데이터베이스 캐시가 되기도 하는 마이크로커널 아키텍처(Microkernel Architecture) 를 자랑한다.

엔터프라이즈 레벨의 인프라를 구축할 때 90% 이상의 마법은 모두 이 ’플러그인’에서 기인한다. 브라우저에서 curl 명령어 하나로 로봇의 온도를 묻고, 연결이 끊겨도 라우터의 Memory Storage에서 1시간 전의 로봇 좌표를 꺼내오는 모든 작업이 라우터 재컴파일 없이 동적으로 수행된다. 본 장에서는 시스템의 무기고를 열어 가장 강력한 플러그인들의 런북을 해부한다.
