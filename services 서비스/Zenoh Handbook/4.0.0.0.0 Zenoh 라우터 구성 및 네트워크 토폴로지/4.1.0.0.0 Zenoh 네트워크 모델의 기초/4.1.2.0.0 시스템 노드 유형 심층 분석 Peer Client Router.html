4.1.2 시스템 노드 유형 심층 분석 (Peer, Client, Router)

Zenoh 통신망을 지배하는 3대 계급장(Mode)이다. 프로그램 혹은 데몬을 띄울 때 첫 줄에 이걸 무엇으로 적어 냈느냐에 따라 패킷이 흐르는 운명이 결정된다.

0.1 Peer (자유로운 평민)

  • 행동 양식: “내가 리더다. 나를 따르라.” 라우터 같은 상위 기관 통제 없이, 동일(VLAN/WIFI) 네트워크 상에 있는 다른 Peer나 Router들을 UDP 멀티캐스트(스카우팅)로 스스로 찾아내어 1:1 선(Mesh)을 복잡하게 연결한다.
  • [Runbook 유의점]: 로봇끼리의 P2P 응답성(Latency)은 최고조에 달하지만, 50대 이상의 Peer가 한 와이파이 망에 올라오면 서로를 수색하는 스카우팅 트래픽 폭풍이 발생하여 공유기가 타버린다. 국소 지역 망 전용이다.

0.2 Client (순종적인 일꾼)

  • 행동 양식: 스카우팅 능력을 거세당한 모드. 자력으로 다른 노드를 찾지 않으며 오직 내가 바라보라고 지정해 준 상위 Router나 Peer의 IP/포트 1개하고만 TCP 파이프라인(Unicast)을 뚫고 데이터를 상납한다.
  • [Runbook 유의점]: 마이크로컨트롤러나 수천 대의 상용 판매용 IoT 단말은 배터리 소모와 불필요한 트래픽 낭비를 막기 위해 반드시 Client 모드로 박아 넣어야 한다.

0.3 Router (통치자 - zenohd)

  • 행동 양식: 데이터 생산은 1도 하지 않으면서, 쏟아져 들어오는 수많은 Client와 Peer들의 데이터를 받쳐주고 릴레이(Relay) 해주는 전업 우체국이다. 자기들끼리도 백본 연결을 맺어 망을 증식시킨다.
  • [Runbook 유의점]: 오직 인프라/서버망(K8s, Edge PC, Cloud EC2)에만 데몬 형태로 떠 있어야 한다. 이 노드에 메모리/디스크 스토리지 플러그인을 결착하여 데이터센터(Data Center)를 만든다. 클라이언트 단말기 안쪽 코드에 악의적으로 라우터 모드를 엮어 넣으면 전체 망 통신 비용이 파괴된다.
