4.1.3 세션(Session) 설정 및 데이터 평면(Data Plane)의 분리

당신은 물리적인 인터넷 망 위에 가상의 “방(Room)“을 구획할 수 있어야 한다. 로봇 청소기 텔레메트리 망에서 떨어진 데이터가 자율주행 택시 관제망에 흘러 들어가는 꼴을 보고 싶은게 아니라면 말이다. 이때 등장하는 개념이 데이터 평면(Data Plane)의 분리와 세션 아이디다.

0.1 데이터 평면(Data Plane) 분리란?

데이터 평면 구획이란, 물리적으로 1개의 와이파이 망을 공유하더라도 논리적으로 통신망을 완벽하게 절단(Isolate) 하는 기술이다. A 평면의 로봇과 B 평면의 로봇은 서로가 허공에 패킷을 뱉어도 인지하지 못한다.

0.2 [Runbook] zenoh::open(config)를 통한 강제 망 분리 절차

클라이언트 앱이나 라우터를 기동할 때, ID(ID 기반 세그먼트)를 주입하여 담벼락을 친다.

[코드 레벨 적용: Rust 예시]

use zenoh::config::Config;

// 인프라 관리자가 배포한 config.json5 파일이나 코드상에서 
// 네트워크 도메인 ID(예: 1번 공장망 = 1)를 강제 주입
let mut config = Config::default();

// 공유기(Transport)망 위에서 "Hex: 0A" 로 마스킹된 트래픽끼리만 엮이게 강제 분리!
config.insert_json5("scouting/multicast/address", "\"224.0.0.225:7448\"").unwrap();

let session = zenoh::open(config).res().await.unwrap();

[라우터 격리 기동 런북]
만약 서버 1대에 a팀(자율주행), b팀(관제웹) 이 각각 별개의 라우터를 올려 달라고 요청한다면?
포트(Port)와 스카우팅 평면을 완전히 가르는 2개의 설정 파일을 배포하라.

# A팀 라우터: 기본 7447 포트, 스카우팅 포트 7447
zenohd -c a_team_config.json5 

# B팀 라우터: TCP 포트를 7449로, 스카우팅 채널 주소도 분리하여 완벽 격리 
zenohd -c b_team_config.json5

이러한 세그먼테이션(Segmentation) 규칙이 사내 표준 런북으로 제정되어 있지 않으면, 수많은 개발 팀들이 던지는 테스트 토픽("test/hello")들이 운영망 데이터를 오염시키는 참극이 빚어진다.
