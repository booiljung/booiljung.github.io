Chapter 4. Zenoh 라우터 구성 및 네트워크 토폴로지

지금까지 우리는 1장부터 3장까지 기나긴 여정을 거치며 **Zenoh(제노)**라는 차세대 데이터 분배 네트워크의 철학을 이해하고, 바닥부터 코드를 엮어내어 이질적인 시스템들이 서로 첫인사를 나누는 작은 기적(Pub/Sub 루프백)을 실현했다. 이제 옹알이를 넘어, 수백 대의 로봇과 수만 개의 IoT 센서 코어가 광활한 대륙을 가로지르며 하나의 거대한 신경망처럼 벼락같이 데이터를 주고받을 수 있도록, 진정한 의미의 **네트워크 토폴로지(Network Topology)**를 엮어낼 시간이다.

본 장에서는 Zenoh 아키텍처의 심장부이자 교통경찰 역할을 수행하는 **Zenoh 라우터(Router)**의 해부학적 구조를 낱낱이 파헤친다. 단순히 실행 프로그램을 띄우는 법을 넘어서, 라우터가 브로커(Broker)와 어떻게 다른지(4.1), 피어(Peer)와 클라이언트(Client)가 어떻게 라우터와 얽히며 동적 네트워크 망을 자가조립(Self-Assembly)하는지 그 심오한 원리를 탐구할 것이다.

또한 메가바이트 단위의 치밀한 JSON 라우터 설정 파일을 마스터하고(4.3), 물리적 한계를 극복하는 엔드포인트 세팅(4.4), 에지와 클라우드 간의 레이턴시를 깎아내는 통신 최적화 전략(4.7)을 차례대로 마스터하게 된다. 이 장의 마지막 장을 덮을 무렵이면, 거미줄처럼 복잡하게 얽힌 엔터프라이즈급 분산 데이터 백본망(Backbone Network)을 여러분의 손끝에서 설계하고 자유자재로 지휘할 수 있는 아키텍트의 시야를 얻게 될 것이다.
