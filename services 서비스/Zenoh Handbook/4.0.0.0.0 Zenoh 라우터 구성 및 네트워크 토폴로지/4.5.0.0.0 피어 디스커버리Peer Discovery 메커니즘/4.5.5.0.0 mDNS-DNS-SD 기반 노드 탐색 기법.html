4.5.5 mDNS/DNS-SD 기반 노드 탐색 기법

어제 설정한 tcp/192.168.1.15 주소가 오늘 아침 공유기가 재부팅되면서 DHCP에 의해 1.20으로 바뀌어 버렸다?
하드코딩(Static) 조준선 정렬의 끔찍한 부작용은 동적 IP 할당 환경에서 그 추적 링크가 끊어진다는 데 있다.

이때 구원투수로 등판하는 것이 바로 로컬 네트워크 애플 생태계의 마법인 mDNS (Multicast DNS) 와 Bonjour 프로토콜을 차용한 탐색 기법이다. (Zenoh 생태계 내부 혹은 운영체제 단의 avahi-daemon 연동).

0.1 [Runbook] 고정 IP 없는 로컬 망 생존법

로봇 단말기(Raspberry Pi)의 호스트 네임을 명확히(robot_arm1.local) 부여하고, 젠오 연결 설정 지점(Connect config)에 IP 주소 대신 .local 도메인을 박아 넣는다.

# 로봇 1번의 터미널
sudo hostnamectl set-hostname robot1
sudo apt install avahi-daemon   # mDNS 엔진 가동

// 로봇 2번 (연결자) 의 Zenoh 클라이언트 설정
{
  connect: {
    // 192.168 IP 대신 네트워크 허공에 "robot1.local 누구냐!" 라고 찾아내어 연결한다.
    endpoints: ["tcp/robot1.local:7447"]
  }
}

로컬 공장 구축 사업(SI)을 나갔을 때, 고객의 현장 공유기 IP가 어떻게 바뀔지 모르겠다면 이 .local 포워딩 타겟팅을 기본값으로 가져가라. 유지보수 엔지니어의 퇴근 시간을 앞당겨줄 지상 최고의 런북 트릭이다.
