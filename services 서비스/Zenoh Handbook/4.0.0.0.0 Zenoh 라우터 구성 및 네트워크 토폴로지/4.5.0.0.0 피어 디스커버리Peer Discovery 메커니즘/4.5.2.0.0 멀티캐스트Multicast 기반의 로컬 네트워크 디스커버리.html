4.5.2 멀티캐스트(Multicast) 기반의 로컬 네트워크 디스커버리

스카우트 비콘을 도대체 어떤 주소로 쏠 것인가? Zenoh는 기본적으로 IPv4 멀티캐스트 전용 대역폭(기본 224.0.0.225)을 오남용 없이 우아하게 사용한다.
공유기(AP)나 스위치에 물려있는 동등한 L2/L3 서브넷 안에 있다면, IP를 지정하지 않아도 저 주소 하나로 전체 방사(Broadcasting) 효과를 누린다.

0.1 [트러블슈팅] 멀티캐스트 장애와 IGMP 스누핑(Snooping)

가장 많이 접수되는 에러. “로봇 두 대가 같은 스위치에 물려있는데 서로를 못 찾습니다!”

원인 분석 (스위치 레이어 방벽):
기업용 L3 스위치나 Cisco 라우터 장비들은 기본적으로 멀티캐스트 폭풍을 막기 위해 224 대역의 패킷을 고의로 폐기(Drop)하는 IGMP Snooping 기능이 켜져 있다. 혹은 개발자 노트북의 Windows Defender가 UDP 빙의를 막아낸 것이다.

[생존 런북] 강제 뚫기 명령

  1. 네트워크 팀에 증명: tcpdump -i eth0 host 224.0.0.225 명령을 쳐서 로봇이 쏘는 패킷이 스위치를 통과하는지 입증하라.
  2. 소켓 바인딩(Interface Bind) 고립: 노트북에 도커 가상 랜카드, 유선랜, 와이파이가 다 물려 있으면 멀티캐스트가 어디로 나갈지 몰라 길을 잃는다.
    설정 파일에 zenohd.json5 -> scouting -> multicast -> interface : "eth1" 이렇게 로봇청소기와 엮인 순수 물리 랜카드 1개를 명시적으로 지목해야 헤매지 않는다.
