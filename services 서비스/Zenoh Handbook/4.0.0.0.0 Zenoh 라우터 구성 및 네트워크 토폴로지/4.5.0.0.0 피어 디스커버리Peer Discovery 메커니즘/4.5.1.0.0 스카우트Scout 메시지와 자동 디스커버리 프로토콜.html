4.5.1 스카우트(Scout) 메시지와 자동 디스커버리 프로토콜

Zenoh의 노드(특히 Peer와 Router)가 기동될 때 가장 먼저 하는 행동은 자신을 알리기 위해 사방으로 레이더파를 발사하는 것이다. 이를 Scout Message(스카우트 메시지) 라 부른다.

0.1 Scout 메시지의 본질

이 레이더 안에는 “나의 고유 아키텍처 식별자(Router ID)”, “내가 쓰는 프로토콜(TCP/QUIC)”, “나의 현재 IP 및 개방된 포트(7447)” 등 자신의 명함(Locator) 정보가 꽉꽉 담겨 있다.
주변에 있는 다른 Zenoh 노드가 이 비콘(Beacon) 메시지를 수신하면, 그 즉시 명함에 적힌 주소로 다이렉트 TCP/UDP 핸드쉐이크를 시도하여 교신(Session)을 성립시킨다.

0.2 [Runbook] Scout 옵션 튜닝과 침묵(Silence) 강제

“서로를 자동으로 찾는다“는 것은 해커도 우리 망의 Zenoh 데몬을 쉽게 스캔할 수 있음을 의미한다.

[안전한 관제를 위한 스카우트 제어 방벽]

  • 주기적 스카우트 끄기: 기본적으로 연결이 끊어지면 라우터는 3초마다 레이더 파를 쏘며 남을 찾는다. 사내 K8s 환경에서 모든 IP가 고정되어 있다면 이 기능은 대역폭 낭비일 뿐이다. 라우터 설정에서 이 자동 박동을 꺼버려라.
  • 클라이언트 모드(Client Mode)의 본질:
    코드 단에서 zenoh-cli --mode client 로 켜면, 이 녀석은 절대 자신을 알리는 스카우트 패킷을 발사(Publish)하지 않고 쥐죽은 듯 리스닝만 수행한다. 이것이 배터리로 구동되는 에지(Edge) 마이크로 센서가 반드시 취해야 할 디폴트 스탠스다.
