4.5 피어 디스커버리(Peer Discovery) 메커니즘

수천 대의 로봇 떼(Swarm)가 전장에 풀려났을 때, 이들이 어떻게 서로의 존재를 인지하고 통신망(Mesh)을 구축할 것인가?
모든 로봇에 192.168.x.x 와 같은 고정 IP를 할당해 주고, 설정 파일에 1000줄짜리 하드코딩을 박아 넣는 것은 과거의 유물이다.

Zenoh가 제시하는 자동 디스커버리(Auto Discovery) 메커니즘은 IP 체계의 딱딱함을 부수고, 노드가 부팅되는 순간 “내가 여기 있다(I am here)!“라고 선언하며 주변 동료들과 자율적으로 연결되는 생명체적 특징을 지닌다. 하지만 이 마법 같은 자동화는 네트워크 엔지니어의 엄격한 통제가 없으면 스카우팅 브로드캐스트 패킷이 로컬망을 질식시키는 부메랑으로 돌아온다.
본 절에서는 스카우팅(Scouting) 레이더의 작동 원리부터, 도커(Docker) 브릿지나 클라우드망과 같은 제약 환경에서 어떻게 디스커버리를 뚫어내는지에 대한 생존 런북을 서술한다.
