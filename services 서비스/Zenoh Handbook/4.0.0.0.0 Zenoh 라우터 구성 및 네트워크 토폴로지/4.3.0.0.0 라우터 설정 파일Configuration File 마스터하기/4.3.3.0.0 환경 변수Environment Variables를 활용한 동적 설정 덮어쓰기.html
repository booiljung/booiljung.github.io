4.3.3 환경 변수(Environment Variables)를 활용한 동적 설정 덮어쓰기

Docker 이미지나 K8s(Kubernetes) 포드 안에 설정 파일(.json5)을 꽉 박아(Bake) 두었다고 가정하자.
갑자기 부산 공장의 라우터만 7447 포트가 막혀서 7448로 바꿔서 올려야 하는 상황이 터졌다.
이미지(Image)를 새로 빌드해서 굽는 것은 클라우드 철학에 어긋나는 하책이다. 이때 환경 변수 덮어쓰기(Env Override) 라는 최상위 명령 체계가 발동한다.

0.1 [Runbook] 치트키: ZENOH_ 접두사 변환 공식

Zenoh 데몬은 부팅 직전, .json5 파일의 섹션 구조를 정확히 흉내 낸 OS 환경 변수를 감지해 파일 설정을 무자비하게 짓이겨(Override) 버린다.

[경로 치환 공식]
JSON 파일의 키(Key) 계층을 대문자로 바꾸고 사이에 언더바(_)를 넣은 뒤 앞에 ZENOH_를 붙이면 끝이다.

예시 1: 수신 포트(Listen) 변경

  • JSON 구조: { "listen": { "endpoints": ["tcp/0.0.0.0:7448"] } }
  • 환경 변수: ZENOH_LISTEN_ENDPOINTS="tcp/0.0.0.0:7448"

예시 2: 상위 접속 서버 주소(Connect) 변경

  • 환경 변수: ZENOH_CONNECT_ENDPOINTS="tcp/192.168.1.10:7447"

0.2 Docker-Compose 환경 변수 주입의 미학

현장에서 진정 유용한 것은 docker-compose.yml 에서 이 무기를 꺼내 들 때다.

version: "3"
services:
  router-busan:
    image: eclipse/zenoh:latest
    environment:
      # 파일 수정 없이 오직 도커 실행변수 만으로 스카우팅을 꺼버림!
      - ZENOH_SCOUTING_MULTICAST_ENABLED=false
      - ZENOH_LISTEN_ENDPOINTS=["tcp/0.0.0.0:7448"]
    ports:
      - "7448:7448/tcp"

파일은 “조직의 기본 베이스라인“을 잡는 용도로 두고, 기기별/지역별 특수 조건은 철저히 k8s의 Env 객체를 통해 오버라이드하라. 그래야만 “하나의 도커 이미지, 수천 개의 변형 배포“라는 진정한 데브옵스의 가치가 실현된다.
