4.3.1 설정 파일 포맷 비교 (JSON, JSON5, YAML)

“설정 파일 좀 수정해주세요” 라고 했을 때 가장 먼저 부딪히는 딜레마. Zenoh는 3가지의 언어 포맷을 지원하지만 실무 운영진(DevOps)은 어느 것을 쓸까?

0.1 순수 JSON의 저주

  • 특징: 컴퓨터가 읽기는 좋으나 콤마(,) 하나 빼먹으면 다운되고, 가장 치명적인 주석(Comment)을 달 수 없다는 한계를 가진다.
  • [Runbook]: 사람이 직접 편집할 목적으로는 당장 폐기하라. 오직 앱에서 자동 생성되는 설정에만 쓴다.

0.2 YAML의 가독성 환상

  • 특징: 띄어쓰기(Indentation)만으로 계층을 강제하므로 눈으로 보기는 예쁘다.
  • [Runbook]: 복사/붙여넣기를 할 때 스페이스 하나만 엇나가면 전체 구조가 깨지며 원인을 알 수 없는 서버 크래시를 유발한다. k8s에 길들여진 사람이 아니라면 섣불리 건들지 마라.

0.3 [최적화 런북] 만능 열쇠: JSON5

  • 특징: JSON의 구조적 엄격함과 JavaScript의 관대함을 합쳤다.
  • 장점:
  1. // 와 /* */ 를 통한 마이그레이션 변경 이력(주석) 남기기가 가능하다.
  2. 마지막 줄에 남은 콤마(Trailing Comma)를 허용해서 복붙 시 오류를 없애준다.
  3. 따옴표를 생략해도 키(Key) 값으로 인식한다.
  • 인프라 팀 규칙 강제: 사내 모든 Zenoh 라우터 설정 파일은 무조건 .json5 확장자로 형상 관리(Git) 해야 한다.
