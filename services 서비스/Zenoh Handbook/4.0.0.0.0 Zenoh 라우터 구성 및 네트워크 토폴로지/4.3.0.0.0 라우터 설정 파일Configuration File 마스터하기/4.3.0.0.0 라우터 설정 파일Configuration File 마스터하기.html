4.3 라우터 설정 파일(Configuration File) 마스터하기

초보 개발자들은 zenohd를 켜놓고 기능이 동작하지 않으면 무작정 CLI에 덕지덕지 수십 개의 파라미터를 넘겨 강제 주입하려 든다. 이는 Docker, K8s로 대표되는 현대적 IaC(Infrastructure as Code) 생태계에 대한 완벽한 항명이다.

Zenoh 라우터를 현장에 깔 때는 “포트 번호”, “접근 제어(ACL)”, “DB 플러그인 로드 주소”, “멀티캐스트 스카우팅 타임아웃” 수백 가지의 옵션 변수들을 다루어야 한다. 이 방대한 인프라 스펙을 정의하고 관리하는 유일한 정답은 바로 정적 설정 파일(Config) 이다.

본 절에서는 JSON5라는 관대한 문법 아래, Zenoh의 핵심 라우팅 뇌 구조를 어떻게 조각하고 환경변수(ENV) 트릭을 통해 K8s ConfigMap 환경에서 동적으로 치환해 내는지에 대한 실무 런북을 파고든다.
