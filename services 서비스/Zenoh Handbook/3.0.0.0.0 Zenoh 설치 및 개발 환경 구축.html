Chapter 3. Zenoh 설치 및 개발 환경 구축

이론적으로 완벽한 아키텍처를 이해하는 것과, 실제로 개발 환경에서 라우터(Router)를 구동하고 분산 노드 간에 데이터를 주고받는 것은 완전히 다른 차원의 공학적 과제이다. 제2장(Chapter 2)에서 Zenoh의 데이터 중심(Data-centric) 코어 철학과 와이어 프로토콜(Wire Protocol) 메커니즘을 이론적으로 정립했다면, 본 장(Chapter 3)에서는 이를 실제 물리적 인프라와 소프트웨어 생태계에 구축하는 실천적 과정을 체계적으로 다룬다.

Zenoh가 지닌 진정한 가치는 특정한 운영체제나 프로그래밍 언어에 종속되지 않고, 시스템 프로그래밍 언어부터 웹 에코시스템, 그리고 표준 로보틱스 미들웨어에 이르기까지 이기종(Heterogeneous) 환경을 완벽하게 관통하는 범용 바인딩(Binding)을 제공하는 데 있다. 본 장은 이러한 다국어 및 멀티 플랫폼(Multi-platform) 지원 철학을 바탕으로 개발자가 직면할 수 있는 모든 런타임 환경에 대응하는 설치 및 설정 로드맵을 제시한다.

1. 코어 인프라: Zenoh 라우터 및 CLI 도구 구축

분산 신경망의 근간을 이루는 Zenoh 라우터 데몬(zenohd) 시스템은 단일 에지(Edge) 기기에서 수백 대의 클러스터를 아우르는 백본(Backbone) 네트워크에 이르기까지 다양한 스케일로 동작한다. 시스템의 요구사항, 방화벽(Firewall) 정책 등을 점검한 후, 각 OS 환경(Linux, Windows, macOS) 및 격리된 Docker 컨테이너(Container) 환경에서 zenohd를 안정적으로 실행하고 백그라운드 서비스(systemd 등)로 이행하는 방법을 설명한다. 또한, 터미널 상에서 직접 명령어를 통해 데이터를 확인하고 테스트할 수 있는 주요 CLI(Command Line Interface) 도구들의 활용법을 익힌다.

2. 이기종 개발 환경 통합 (Multi-language Setup)

분산 애플리케이션 개발의 핵심은 각기 다른 언어로 작성된 노드들이 단일한 글로벌 데이터 공간(Global Data Space)하에서 어떻게 동일한 언어를 구사하는지에 달려 있다.

본 장은 대표적인 네 가지 언어 생태계와 ROS2 환경 구축을 상세히 제공한다.

  • Rust 환경: 공식 툴체인(cargo)을 통한 의존성 주입과 tokio 기반 비동기(Asynchronous) 런타임 최적화 전략.
  • C/C++ 환경: CMake를 활용한 크로스 플랫폼 빌드와 수동 메모리 관리를 지원하는 도구 세팅.
  • TypeScript 환경: Node.js 기반의 서버 사이드 또는 브라우저 환경에서의 비동기 이벤트 루프 통합.
  • Python 환경: AI 및 데이터 사이언스 생태계를 위한 빠르고 유연한 인터프리터 스크립팅 연동.
  • ROS2 인터페이스: 글로벌 로보틱스 플랫폼과의 통합을 위한 zenoh-bridge-dds 브리징 및 rmw_zenoh 네이티브 통합 가이드.

3. 관측 가능성(Observability) 및 디버깅 인프라

안정적인 통신 환경 구축은 코드의 작성만으로 완성되지 않는다. 분산망 내부에서 발생하는 트래픽 포화, 지연 시간(Latency)의 병목, 파티션 분절(Network Partition) 등의 상태를 외부에서 즉각 탐지할 수 있어야 한다.
이를 위해 Prometheus 연동을 통한 성능 지표(Metrics) 수집과 Grafana 기반 대시보드를 시각적으로 구성하며, Wireshark를 비롯한 네트워크 분석 도구(Network Analyzer)를 결합하여 심층 트러블슈팅(Troubleshooting)을 수행할 수 있는 견고한 관측 가능성(Observability) 생태계를 구축한다.

이 모든 인프라와 툴체인 설정이 완료되면, 본 장의 종착지에서 서로 다른 두 개의 이기종 언어(예: Rust 퍼블리셔와 Python 구독자)가 동일한 단위의 자원에 접근해 성공적으로 통신하는 첫 번째 루프백(Loopback) 통신 검증을 통해 완벽히 구축된 플랫폼의 효용을 직접 체감하게 될 것이다.
