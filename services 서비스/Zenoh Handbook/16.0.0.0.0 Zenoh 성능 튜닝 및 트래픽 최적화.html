Chapter 16. Zenoh 성능 튜닝 및 트래픽 최적화

네트워크 코드를 구축하고 시스템 접속 테스트를 통과했다고 해서 인프라의 설계가 완료된 것은 아니다. 실제 프로덕션(Production) 환경에서는 거대한 데이터 폭풍(Data Storm)이 도래한다. 초당 10만 건 이상의 수많은 상태 메시지가 실시간으로 유입되며, 라이다(LiDAR) 센서 집단이나 고해상도 카메라 노드가 생성하는 기가바이트(Gigabytes) 단위의 데이터 덤프가 제한된 대역폭(5G 또는 위성망 등) 파이프라인 구간을 동시에 통과해야 하는 병목이 발생한다.

단순히 “동작하는” 수준의 아키텍처를 넘어, 시스템 자원의 낭비를 완벽히 배제하는 제로-웨이스트(Zero-Waste) 지향의 초고성능 네트워크 토폴로지로 인프라를 전면 최적화(Optimization) 해야 할 시점이다. 이 16장에서는 네트워크의 물리적 지연 시간(Latency)을 밀리초(ms) 단위에서 마이크로초(µs) 단위로 정교하게 단축하고, 폭증하는 데이터 처리량(Throughput) 한계의 병목 현상을 운영체제(OS)의 하위 커널(Kernel) 및 네트워크 인터페이스 컨트롤러(NIC) 계층 밑단에서부터 근본적으로 해소하는 극한의 하드코어 성능 최적화(Performance Tuning) 런북(Runbook)을 전개한다.
