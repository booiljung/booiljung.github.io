11.1.3 이기종 네트워크(LAN-WAN) 간의 확장성 및 멀티캐스트 제약

“우리 회사 공장(LAN)에 있는 로봇들을 지구 반대편 클라우드(WAN, AWS)에서 관제하고 싶다.”

이 요구안 앞에서 ROS2 레거시 엔지니어들은 머리를 쥐어뜯는다. DDS는 본질적으로 ‘한 방(Local subnet)’ 안에 있는 기기들끼리 대화하도록 만들어진 프로토콜이기 때문이다.

0.1 [인스펙션] WAN 라우팅의 절망편

1. 서브넷 라우팅 장벽
ROS2 노드(Node)들은 라우터(NAT) 장비를 넘어갈 수 없다. 192.168.0.x 대역을 벗어나는 순간 DDS 통신은 증발한다. 서로 다른 빌딩, 심지어 서로 다른 망(5G와 Wi-Fi) 사이의 로봇 군집은 절대 ROS2 통신으로 묶일 수 없었다.

2. 인프라 개조의 한계
이 장벽을 강제로 뚫어내기 위해서 그동안 개발자들은 SROS2(보안) 가상 VPN 터널을 뚫거나, DDS Router 라는 써드파티 장비를 매 서버마다 지저분하게 달아주어야 했다.

진정한 글로벌 스케일 인터넷(WAN) 위에서 로봇들이 마치 한 사무실에 있는 것처럼 대화할 수 있는, 인터넷 백본과 직접적으로 동기화되는 새로운 통신 라우팅 아키텍처가 절실했다.
