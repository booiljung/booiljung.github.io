11.1.2 무선 네트워크(Wi-Fi, 5G) 환경에서의 데이터 유실 및 불안정성 문제

DDS 기반의 ROS2 로봇이 무대 위에서 시연을 하다가 와이파이를 잡지 못하고 멀뚱멀뚱 서 있는 장면은 업계의 흔한 촌극이다.

0.1 [인스펙션] Wi-Fi 환경에서의 DDS 붕괴 원인

1. 대역폭 질식 (Bandwidth Choking)
로봇이 라이다(LiDAR) 포인트 클라우드나 카메라 영상(Image) 등 초당 수십 메가바이트의 데이터를 DDS 코어에 쑤셔 넣기 시작하면, 유선 환경에서는 완벽하게 돌아가던 ROS2 코어가 무선망(Wi-Fi) 위에서는 TCP 재전송의 늪에 빠져 패킷을 분실(Packet Loss)하기 시작하고 통신 큐가 마비된다.

2. 통신 단절(Disconnection) 회복 불능
Wi-Fi 음영지대(Shadow zone)에 로봇이 1초만 들어갔다가 나와도, DDS는 수만 번의 재전송 요청을 처리하느라 CPU 100%를 잡아먹다가 죽어버린다. 한번 끊어진 DDS 세션이 재차 ഡി스커버리(Discovery)를 돌려 연결을 회복하기까지 최소 5초 이상이 걸린다. 자율 주행 자동차에서 이 5초는 곧 대형 사망사고를 의미한다.

결국 진정한 자율주행 엣지(Edge) 로보틱스를 위해서는 무선망의 단절과 핑(Ping)의 파도침을 본질적으로 견뎌내는(Loss-Tolerant) 이방인 계층이 투입되어야 했다.
