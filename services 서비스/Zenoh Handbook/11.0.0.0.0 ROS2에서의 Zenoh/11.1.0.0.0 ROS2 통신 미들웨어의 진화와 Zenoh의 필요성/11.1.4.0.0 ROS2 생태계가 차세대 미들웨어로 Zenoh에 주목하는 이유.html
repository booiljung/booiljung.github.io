11.1.4 ROS2 생태계가 차세대 미들웨어로 Zenoh에 주목하는 이유

Eclipse 재단의 Zenoh 가 등장하자마자 ROS2 핵심 개발자 재단(Open Robotics)은 환호성을 질렀다. DDS의 무거운 족쇄를 끊어내고 로봇을 거대한 클라우드 대륙으로 인도해 줄 이 신성이 완벽한 타이밍에 맞춰 등장했기 때문이다.

0.1 [인스펙션] Zenoh 가 ROS2의 구원자인 4가지 핵심 이유

  1. 디스커버리 폭풍 제로(Zero-Discovery Overheads)
    Zenoh는 라우터(Router) 기반 아키텍처 매핑을 통해 노드끼리 무식하게 인사를 나누지 않는다. 오직 라우터에게만 선언(Declare)하면 전 세계와 통신이 개통된다.

  2. 완벽한 이기종 망 통합 (LAN to WAN Seamless Bridge)
    NAT, 방화벽, 5G망 등 어떠한 더러운 인터넷 망 구조 속에서도 Zenoh 라우터는 알아서 길을 뚫는다. 클라우드 서버와 로봇 노드 간의 통신이 한 방(Subnet) 에 있는 것처럼 매끄럽게 돌아간다.

  3. 마이크로컨트롤러(MCU)부터 데이터센터까지의 포용성
    DDS는 절대 100KB 짜리 센서 보드 위에 얹을 수 없었지만, 앞선 10장에서 보았듯 Zenoh-Pico 는 모든 생태계의 작은 칩까지도 ROS2 시스템 안으로 통합시켜버린다.

  4. 오픈소스와 비-침투성 브릿징 (Non-Intrusive Bridging)
    이러한 거대한 마이그레이션 아키텍처 변경을 겪으면서도, 로봇 펌웨어 짜는 엔지니어는 단 한 줄의 rclcpp 코드도 바꿀 필요가 없다!

이 혁명적인 가치 증명으로 인해 ROS2 최신 배포판(Jazzy 등)에서는 아예 공식 기본 미들웨어(RMW) 중 하나로 Zenoh (rmw_zenoh) 채택 논의가 강력하게 진행되며 글로벌 스탠더드의 대격변을 예고하고 있다.
