9.2 C 언어 환경에서의 Zenoh (zenoh-c)

가비지 컬렉터(Garbage Collector, GC)를 배제하고, 예외 처리(Exception Handling, try-catch) 블록을 지원하지 않으며, 명시적인 메모리 할당(malloc) 및 해제(free)를 강제하는 C 언어(C99 표준)는 현대의 고수준 언어들에 비해 저수준의 제어력을 요구한다.

그러나 이러한 원초적인 메모리 제어(Manual Memory Management)의 제약 조건이야말로 하드웨어 제어의 총아인 로보틱스 도메인(모터 구동, 자율주행 LiDAR 센싱 등)을 가동하는 불변의 동력원이다. 자율주행 차량의 제동 시스템(Braking System) 파이프라인에서 가비지 컬렉션 퍼즈(GC Pause)로 인하여 프로세스 스케줄링이 0.1초 지연되는 순간, 치명적인 물리적 충돌(Collision)이 발생하게 된다. 즉, 시스템의 결정론적(Deterministic) 실행 보장은 타협할 수 없는 절대적 척도이다.

본 장에서는 극한의 연산 속도와 예측 가능한 자원 관리가 요구되는 에지 디바이스(MCU, ECU, RTOS) 계층에서, C 언어 고유의 포인터(Pointer) 제어 및 매크로(Macro) 전개 기법을 활용하여 Zenoh 통신 프로토콜의 멀티캐스트(Multicast) 통신 퍼포먼스를 극대화하는 실전 구현 로직을 설계한다.
C++ 언어가 수반하는 객체 지향 패러다임의 런타임 오버헤드(Runtime Overhead)를 극복하고, 가장 경량화되고 최적화된 C 함수 인터페이스(zenoh-c)의 조합을 통해 초저지연 데이터 통신망(Data Network)을 구축할 시스템 아키텍처를 전개하라.
