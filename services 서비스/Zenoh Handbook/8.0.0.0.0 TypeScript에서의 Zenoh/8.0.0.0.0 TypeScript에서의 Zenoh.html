8. TypeScript에서의 Zenoh

“세상의 절반은 웹 브라우저 위에서 돌아가고, 나머지 절반은 Node.js 서버 위에서 돈다. 이 거대한 제국에 초고속 로보틱스 통신망을 깔아라.”

TypeScript(및 JavaScript)는 인류 역사상 가장 성공한, 그리고 가장 파편화된 언어다. IoT 기기의 작은 버튼부터, 글로벌 기업의 클라우드 백엔드(NestJS), 그리고 사용자의 눈앞에 펼쳐지는 현란한 SPA(React/Vue) 대시보드까지 모든 곳에 침투해 있다.

이 거대한 V8 엔진의 바다 위에 C++ / Rust 기반의 묵직한 분산 라우터 네트워크인 Zenoh를 이식하는 것은 결코 단순한 npm install 로 끝나지 않는다.

이 장은 단순한 API 매뉴얼이 아니다. 웹 개발자가 분산 시스템 아키텍트로 각성하기 위한 야전 교범(Runbook) 이다.
싱글 스레드 이벤트 루프가 1초에 1만 번씩 폭격하는 로봇 센서 데이터를 어떻게 튕겨내지 않고 파싱(Parsing)할 것인가?
React의 까다로운 렌더링 생명주기 안에, 어떻게 하면 WebSocket 세션을 우아하게 가둬서 메모리 누수(OOM)를 방어할 것인가?

이제 REST API의 낡은 폴링(Polling) 지옥을 박살 내고,
전 세계의 센서와 로봇이 브라우저 DOM 위에서 실시간으로 박동하게 만드는 진정한 서버리스 실시간 웹(Serverless Real-time Web) 의 문을 연다.
