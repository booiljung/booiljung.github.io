8.1 TypeScript와 Zenoh의 통합 개요

1990년대의 브라우저 장난감 언어였던 JavaScript는, 이제 Node.js와 TypeScript라는 거대한 갑옷을 입고 클라우드 백엔드와 프론트엔드를 모두 지배하는 황제로 군림하고 있다.
기존의 웹 생태계는 클라이언트가 리퀘스트(Request)를 보내면 서버가 리스폰스(Response)를 주는 수동적인 HTTP(REST) 구조나, 기껏해야 중앙 집중형 브로커에 매달린 WebSocket 통신에 갇혀 있었다.

하지만 만약 웹 브라우저 자체가 거대한 분산 네트워크망의 P2P 노드 중 하나로 직접 편입될 수 있다면 어떨까? 중간에 서버나 데이터베이스를 거치지 않고, 스마트폰의 크롬 브라우저가 공장에 있는 로봇의 휠 베어링 온도를 마이크로초 단위로 직접 뽑아올 수 있다면?

이 장에서는 왜 TypeScript 생태계가 Zenoh를 만나야만 했는지, 이들의 만남이 기존의 REST / GraphQL / Socket.io 생태계를 어떻게 파괴하고 혁신하는지 그 아키텍처의 포문을 연다. 단순한 라이브러리 소개를 넘어, 진정한 ’분산형 풀스택(Full-Stack) 엔지니어’로 거듭나기 위한 런북(Runbook)의 시작이다.
