21.8.2 에지 디바이스 펌웨어 무선 업데이트(OTA) 및 CI/CD 파이프라인

수천 대의 로봇과 센서가 공장 곳곳에 설치된 상태에서 버그를 수정하기 위해 엔지니어가 직접 USB 포트를 꽂고 다니는 것은 불가능하다. Zenoh 통신망을 활용하여 코드 개발(Git)부터 단말 기기까지 펌웨어를 무선 업데이트(Over-The-Air, OTA)하는 파이프라인(CI/CD)을 구축하라.

첫째, GitHub Actions 기반 CI 파이프라인 설계다.
C 엔지니어나 Rust 엔지니어가 에지 코드를 커밋(Commit)하면, CI 빌드 서버가 이를 크로스 컴파일(Cross-compile)하여 ARM 아키텍처용 패키지 혹은 바이너리를 생성한다.

둘째, Zenoh를 통한 바이너리(Firmware) 전파다.
생성된 펌웨어 .bin 파일을 일반 REST 서버가 아닌 클라우드 라우터의 특정 토픽(예: /factory/ota/firmware/v1.2)에 거대한 청크 블록 단위로 퍼블리시하라.

셋째, 에지 노드의 OTA 구독(Subscribe) 및 자가 패치(Self-patch) 로직이다.
각 디바이스의 부트로더 보조 시스템(혹은 운영체제 데몬)은 OTA 토픽을 항시 구독한다. 새로운 펌웨어 메시지가 들어오면 버전을 확인한 뒤 무중단으로 로컬 디스크에 내려받는다. 다운로드가 100% 완료되면 Zenoh 브릿지 연결을 안전하게 종료(drop(session))한 뒤, 시스템을 재부팅(Reboot)하여 새로운 C/Rust 버전 노드로 깨어나게 만든다. 이 모든 과정이 인간의 개입 없이 물 흐르듯 일어나야 한다.
