21.8.3 네트워크 단절 및 복구(Failover) 시나리오에서의 로봇 안전성 검증

AMR이 공장 한가운데 철제 랙 뒤(Dead Zone)로 숨어 클라우드 기반 관제망과의 접속이 끊겼다.

[단절 상황의 로터스(Lotus) 생존 검증]

  1. P2P 대체경로 작동: AMR_01의 코어 연결망이 잘렸을 때, 옆을 지나가던 AMR_02가 스카우팅(Scouting) 레이더에 잡히면 자동으로 AMR_02의 노드를 경유(Routing)하여 관제망에 상태 패킷을 뱉어내는지 확인하라. 이것이 분산 데이터 패브릭의 꽃이다.
  2. 하드 타임아웃 락(Lock): 5초간 모든 통신이 두절(Ping Timeout)된 최악의 상황. 관제탑의 E-Stop 제어 신호마저 못 받는다면 AMR은 그 즉시 자율주행 모터 전원을 차단(Safe-Mode) 해야 한다.
  3. 지능적 재연결(Backoff): 통신망이 복구될 기미가 보일 때 너무 많은 노드가 동시에 connect 를 시도해 라우터가 “연결 폭풍(Connection Storm)“에 질려 뻗어버리는 현상을 막기 위해, 랜덤한 대기 시간(Exponential Backoff) 기반으로 연결을 재시도하도록 튜닝하라.
