21.3.3 Zenoh-Pico를 이용한 초소형 마이크로컨트롤러(MCU) 액추에이터 연동

운영체제(OS)가 없는 베어메탈(Bare-metal)이나 FreeRTOS 기반 마이크로컨트롤러(MCU, 예: ESP32, STM32)에서는 표준 IP 스택이 무겁기 짝이 없다. 이런 한계 상황에서 클라우드 관제 지시를 받고 모터를 구동하거나 밸브를 제어하는 액추에이터 노드를 만들기 위해서는 Zenoh-Pico가 필수적이다.

Zenoh-Pico는 아주 적은 수 킬로바이트(KB)의 RAM만으로도 시리얼통신, Bluetooth, Thread/Zigbee 및 최소한의 UDP/TCP 스택 위에서 Zenoh 프로토콜을 온전히 구현해낸다.

첫째, ESP32 기반 MCU에서 Zenoh-Pico의 Sub 기능을 활성화하여 클라우드에서 내려오는 명령을 기다리는 코드를 작성하라.

#include "z_pico.h"

void command_callback(const z_sample_t *sample, void *arg) {
    printf("Received command: %.*s\\n", sample->payload.len, sample->payload.start);
    // 예: "VALVE_OPEN" 문자열 수신 시 GPIO 핀 High 설정
    // turn_on_relay();
}

void main() {
    z_session_t session;
    z_open(&session, NULL);

    z_owned_closure_sample_t callback = z_closure_sample(command_callback, NULL);
    z_subscriber_t sub;
    z_declare_subscriber(&sub, &session, z_keyexpr("factory/zone_A/actuator/valve_1"), z_move(callback), NULL);

    // FreeRTOS Task Loop
    while (1) {
        z_v_task_delay(10); // 양보 처리
    }
}

둘째, Zenoh-Pico 디바이스들은 직접 클라우드 라우터로 붙는 것보다 직렬 통신(Serial)을 통해 안개 계층(Fog) 라우터(Raspberry Pi 등)에 붙게 설정하라. 공장에 수만 개의 노드가 퍼져 나갈 때, Zenoh-Pico는 상위 라우터라는 플러그(Plug)에 의존하는 스포크(Spoke) 역할을 한다.

이렇게 구현된 마이크로컨트롤러 액추에이터는 TypeScript에서 Zenoh를 사용하는 웹 대시보드 관리자가 클릭 한 번으로 토픽 /factory/zone_A/actuator/valve_1에 “ON” 페이로드를 Put(Publish) 하는 즉시, 밀리초 단위의 지연만으로 즉시 물리적 장비를 제어(Control)하게 만든다.
