21.1.1 스마트 팩토리 및 자율주행 물류 로봇(AMR) 시나리오 정의

현대적인 스마트 팩토리는 정적인 제조 설비뿐만 아니라, 생산 라인을 따라 동적으로 이동하며 물류를 운반하는 자율주행 물류 로봇(AMR)들이 필수적으로 도입된다. 이러한 환경에서는 매우 복잡한 데이터 스트림들이 발생하며 통신적 과제를 만들어 낸다. 이 프로젝트의 핵심 시나리오를 정의하여 설계의 방향성을 명확히 하라.

첫째, 고정된 환경에서 수집되는 초정밀 산업용 IoT(IIoT) 센서 네트워크다. 공장 내 생산 설비들(예: 모터, 프레스기)의 진동, 온도, 전력 소모량 등은 고주파수 단위로 측정되어 기계적 이상이나 예지 보전을 위해 수집된다. 이 데이터들은 밀리초(ms) 단위의 높은 처리량(Throughput)으로 상위 계층으로 라우팅되어야 한다. 이를 위해 C와 구동 환경에 제약이 적은 초소형 디바이스들이 사용된다.

둘째, 동적 환경에서 작동하는 자율주행 물류 로봇(AMR)의 제어 및 관제다. 이 로봇들은 공장 내부를 이동하며 무선 네트워크를 통해 제어된다. 음영 지역을 스칠 때 통신 끊김이나 대역폭 제한이 일어날 수 있다. 로봇에 내장된 LiDAR, 카메라 등 고용량 데이터 센서들을 원격으로 모니터링하면서도 충돌 방지나 이동 경로 업데이트와 같은 중요한 제어 명령들을 지연 없이 주고받기 위해 신뢰성 높은 통신 연결 계층이 필수적이다. 따라서 ROS2에서 Zenoh 프로토콜을 통하여 클라우드 서버와 Seamless한 통신이 유지되어야 한다.

셋째, 글로벌 통합 로봇 관리/관제시스템이다. 여러 대의 로봇과 공장 내 각종 설비들의 센서값을 종합하여 공장 운영자에게 시각적으로 보여주는 대시보드 애플리케이션이다. TypeScript에서 Zenoh를 활용하여, 복잡한 REST API 기반의 폴링(Polling) 방식을 벗어나 실시간 Pub/Sub 통신과 분산 Query 메커니즘을 웹 백엔드 및 모니터링 플랫폼에서 수용해야 한다.

요약하자면, 각 디바이스(센서 노드 중심의 마이크로컨트롤러, ROS2 기반 AMR 모바일 컴퓨팅 유닛, 중앙 클라우드 및 에지 서버)는 독립적인 역할을 지니며, 끊김 없는 정보의 흐름을 보장하기 위해 Zenoh를 통신망의 척추(Backbone)로 삼게 될 것이다.
