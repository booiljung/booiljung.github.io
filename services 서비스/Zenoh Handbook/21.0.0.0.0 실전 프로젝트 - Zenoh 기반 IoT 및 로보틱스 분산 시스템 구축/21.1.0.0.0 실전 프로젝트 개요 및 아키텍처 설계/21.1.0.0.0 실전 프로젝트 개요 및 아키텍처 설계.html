21.1 실전 프로젝트 개요 및 아키텍처 설계

분산 시스템 설계, 데이터 로우팅, QoS(Quality of Service), 보안, 통신 최적화 등 지금까지 다루어 왔던 Zenoh의 핵심 기술과 개념들을 실제 산업 현장의 문제에 적용할 시간이다.

본 장에서는 스마트 팩토리(Smart Factory) 환경에서 다수의 자율주행 물류 로봇(Autonomous Mobile Robot, AMR)과 사물인터넷(IoT) 센서 노드가 유기적으로 동작하는 아키텍처를 그린다. 이러한 환경에서는 수만 개의 센서가 뿜어내는 데이터의 실시간 처리, 이동하는 로봇의 끊김 없는 통신 유지, 클라우드와 에지(Edge) 인프라 간의 동적 데이터 연동이라는 과제가 주어진다. 단일 프로토콜로 에지 센서부터 클라우드 빅데이터 플랫폼까지 이어지는 통신 체계를 Zenoh를 기반으로 구성하라.

이 프로젝트를 완수함으로써 다음과 같은 기술적 목표를 달성한다. 첫째, 마이크로컨트롤러부터 클라우드 백엔드 구축까지 다양한 언어와 플랫폼 간의 상호 운용성을 확인한다. 둘째, ROS2(Robot Operating System 2)의 한계를 극복하는 원거리 오버레이(Overlay) 통신을 구현한다. 셋째, 분산 쿼리(Distributed Query)를 통해 중앙 집중형 데이터베이스의 부하를 줄이고 데이터 중심의 관제 플랫폼(Dashboard)을 완성한다.
