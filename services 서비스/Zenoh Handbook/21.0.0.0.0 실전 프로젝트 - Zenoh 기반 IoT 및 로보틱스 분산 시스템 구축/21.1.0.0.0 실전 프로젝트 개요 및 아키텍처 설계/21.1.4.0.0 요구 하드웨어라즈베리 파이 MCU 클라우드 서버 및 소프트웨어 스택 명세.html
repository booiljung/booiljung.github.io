21.1.4 요구 하드웨어(라즈베리 파이, MCU, 클라우드 서버) 및 소프트웨어 스택 명세

실전 프로젝트를 원활하게 시작하기 위해 필요한 하드웨어 및 소프트웨어 스택을 정의하고 준비하라. 개발 환경은 단일 노트북으로 가상화하여 테스트할 수 있지만, 물리적 인프라 환경에서 검증하는 것을 목표로 한다. 분산 시스템이므로 각 하드웨어 노드별 스택을 체계화한다.

첫째, 초소형 마이크로컨트롤러(MCU) 및 임베디드 장비(Edge Device)이다.

  • 하드웨어: ESP32, STM32, 혹은 간단한 아두이노(Arduino) 보드.
  • 소프트웨어 스택: FreeRTOS나 베어메탈(Bare-metal) 환경. C 환경에서의 Zenoh-Pico를 탑재하라. 이들은 저전력으로 온도 센서나 진동 센서 측정값을 일방향으로 송출(Publish)하는 역할을 수행한다.

둘째, 안개(Fog) 계층을 담당하는 중간 게이트웨이 및 소형 컴퓨터다.

  • 하드웨어: Raspberry Pi 4 모델(혹은 젯슨 나노), ARM 기반 64비트 싱글 보드 컴퓨터. 스마트 팩토리의 섹션 단위 중간 공유기로 동작한다.
  • 소프트웨어 스택: Ubuntu Linux 22.04 LTS. 이곳엔 C++에서 Zenoh 혹은 Rust에서 Zenoh 개발을 통한 1차적 복잡한 예지 보전 모델 및 데이터 필터링 기능이 동작하며, 별도 데몬(Daemon)으로 Zenoh 라우터(zenohd)가 실행된다. ROS2와 인프라의 브릿지 허브 역할도 겸한다.

셋째, 로보틱스 측면의 컴퓨팅 노드(AMR)이다.

  • 하드웨어: 모터 컨트롤러, 라이다 센서 장착형 이동 플랫폼 보드 및 이를 제어할 호스트 PC(Raspberry Pi 혹은 Intel NUC).
  • 소프트웨어 스택: Ubuntu 22.04와 ROS2 Humble 분산 프레임워크 기반. ROS2에서 Zenoh를 활용하기 위해 zenoh-bridge-dds 패키지를 설치하라. Navigation2 및 로컬라이제이션 스택이 동작한다.

넷째, 클라우드 서버 및 관제 계층이다.

  • 하드웨어: AWS EC2, GCP Compute Engine 등의 리눅스 기반 가상 머신이나 데스크탑 서버급 파워. x86_64 아키텍처.
  • 소프트웨어 스택: Docker 컨테이너 오케스트레이션을 활용하라. zenohd의 루트 라우터 역할, 시계열 데이터 저장을 위한 InfluxDB, PostgreSQL, 그리고 TypeScript에서 Zenoh 기반으로 구동되는 Node.js(혹은 Next.js) 백엔드를 구동한다. 클라우드의 자원은 무한대라 가정하고, 트래픽 유입에 따라 확장되는 수평적 설계(Scale-out)를 구비하라.

이 다채로운 소프트웨어 스택 위에서 오직 Zenoh만이 모든 언어와 프로토콜 틈새를 투명하게 메워 데이터의 신경망을 완성해 줄 것이다.
