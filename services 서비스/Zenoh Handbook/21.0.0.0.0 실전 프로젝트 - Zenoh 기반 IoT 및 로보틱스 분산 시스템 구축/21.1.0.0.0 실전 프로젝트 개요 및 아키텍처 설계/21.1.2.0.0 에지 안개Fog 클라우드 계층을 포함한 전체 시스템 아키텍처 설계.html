21.1.2 에지, 안개(Fog), 클라우드 계층을 포함한 전체 시스템 아키텍처 설계

스마트 팩토리 시스템의 안정성과 확장성을 확보하기 위해 컴퓨팅 자원을 에지(Edge), 안개(Fog), 클라우드(Cloud) 3계층으로 분리하여 배치하라. Zenoh 라우터망은 이 세 계층을 관통하는 통신의 연결점 역할을 수행한다. 세 계층 간의 물리적 및 네트워킹 아키텍처를 면밀히 설계해야만 지연 없는 제어가 가능하다.

첫째, 에지 계층(Edge Layer)이다. 센서 데이터를 수집하는 소형 디바이스와 자율주행 로봇들이 위치하는 공간이다. 리소스가 극히 제한된 하드웨어는 Zenoh-Pico(C 언어 기반)를 사용하며, C++에서 Zenoh를 통해 구동되는 IIoT 장비들이 데이터를 퍼블리시한다. ROS2에서 Zenoh 브릿지를 구동하는 내비게이션 자율주행 모듈은 주기적인 배터리 및 위치(Odometry) 정보를 상위 라우터로 전달한다.

둘째, 안개 계층(Fog Layer)이다. 공장 내 서버급 장비 또는 소형 게이트웨이들이 이에 해당한다. 각 구역(Zone)별로 Zenoh 라우터 모듈을 띄워 하위 에지 노드들로부터 쏟아지는 통신 트래픽을 지역적으로 1차 종합(Aggregation)하라. 데이터가 클라우드로 무조건 넘어가기 전에 로컬 제어를 담당하거나, 단기적인 통신 단절을 보상할 수 있는 중간 캐싱 역할을 부여한다.

셋째, 클라우드 계층(Cloud Layer)이다. 가장 성능이 좋은 백엔드 서버와 영속성 데이터베이스가 있다. 다수의 공장(Fog) 라우터들이 글로벌 클라우드에 위치한 마스터 다중 라우터 네트워크(Mesh)로 연결된다. 퍼블리시된 시계열 데이터가 스토리지 백엔드에 통합 기록되며, TypeScript에서 Zenoh 및 GraphQL, 웹소켓 등과 인터페이스하여 최종 사용자(관리자)에게 대시보드로 요약된 정보를 제공한다. 이를 통해 에지와 클라우드 환경에서 끊김 없는 통합 시스템 아키텍처가 전개된다.
