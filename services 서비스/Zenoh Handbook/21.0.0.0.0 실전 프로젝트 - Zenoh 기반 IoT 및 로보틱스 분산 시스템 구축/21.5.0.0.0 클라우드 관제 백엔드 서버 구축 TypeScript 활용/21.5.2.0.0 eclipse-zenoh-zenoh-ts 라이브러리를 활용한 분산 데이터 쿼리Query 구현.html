21.5.2 @eclipse-zenoh/zenoh-ts 라이브러리를 활용한 분산 데이터 쿼리(Query) 구현

REST API에서 GET /sensors/temp를 호출하면 백엔드가 DB를 긁어오는 방식이 일반적이지만, Zenoh에서는 session.get() 이라는 단 한 줄의 Query로 백엔드, 로컬 라우터의 DB 플러그인, 저 멀리 에지의 C 기반 노드까지 모든 저장 매체를 횡단하며 데이터를 당겨올 수 있다.

이것이 TypeScript에서 Zenoh를 사용할 때 경험하는 가장 강력한 무기 중 하나다.

첫째, 백엔드 라디우팅(DB 대신 Zenoh 쿼리 활용)을 구현하라.

import { Session, QueryTarget } from "@eclipse-zenoh/zenoh-ts";

async function fetchRobotBatteries(session: Session) {
    // 모든 amr 로봇의 battery_state 토픽을 일괄 쿼리
    // 이 쿼리는 라우터를 통해 실제 동작 중인 로봇의 Queryable 혹은 InfluxDB 백엔드로 직행한다.
    const replies = await session.get("amr/*/battery_state", { target: QueryTarget.ALL });
    
    let batteryList = [];
    for await (const reply of replies) {
        if (reply.keyExpr) {
            batteryList.push({
                robotId: reply.keyExpr.toString(),
                level: parseFloat(Buffer.from(reply.payload).toString())
            });
        }
    }
    return batteryList;
}

둘째, Queryable 노드의 작성이다. 웹 백엔드 측도 정보의 수집자만이 아니다. 만약 레거시 관제 시스템의 ERP DB에서 ’오늘의 작업 지시서’를 관리한다면, 백엔드는 해당 데이터를 쥐고 있는 Queryable이 되어야 한다.
로봇(Rust)이 아침에 켜지자마자 session.get("erp/today_task/amr_1") 을 날리면, 클라우드의 TypeScript 웹 백엔드의 session.declareQueryable() 콜백이 훅(Hook) 되어 ERP DB를 읽어 역으로 로봇에게 응답(Reply)을 돌려주는 구조를 만들어라.
