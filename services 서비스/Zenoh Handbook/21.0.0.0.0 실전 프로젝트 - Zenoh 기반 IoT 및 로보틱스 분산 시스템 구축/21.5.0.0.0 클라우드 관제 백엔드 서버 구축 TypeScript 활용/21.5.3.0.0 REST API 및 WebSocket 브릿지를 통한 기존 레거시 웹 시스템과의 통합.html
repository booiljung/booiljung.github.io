21.5.3 REST API 및 WebSocket 브릿지를 통한 기존 레거시 웹 시스템과의 통합

모든 기업이 하루아침에 전면적인 Zenoh 네트워크로 시스템을 뜯어고칠 수는 없다. 기존의 레거시(Legacy) ERP, MES, 혹은 오래된 관제 모니터링 시스템은 여전히 HTTP REST API나 일반적인 WebSocket을 통해서만 데이터를 받을 수 있는 경우가 잦다. 우리의 클라우드 백엔드는 이 둘 사이를 매끄럽게 연결하는 브릿지가 되어야 한다.

첫째, Zenoh REST 플러그인의 한계를 넘어서라. zenohd 라우터 자체에도 REST API 플러그인이 제공되지만, 인증 로직과 복잡한 비즈니스 파싱 로직을 덧붙이기엔 무리가 있다. 따라서 TypeScript 기반의 Express 백엔드에서 Zenoh 세션을 쥐고 직접 REST 컨트롤러를 작성하라.

app.post('/api/robot/:id/move', async (req, res) => {
    const robotId = req.params.id;
    const { x, y } = req.body;
    
    try {
        const publisher = zenohSession.declarePublisher(`amr/${robotId}/goal_pose`);
        await publisher.put(JSON.stringify({ x, y }));
        res.status(200).send({ status: "success", msg: "Command published to Zenoh network" });
    } catch(err) {
        res.status(500).send({ status: "error" });
    }
});

둘째, 레거시 시스템을 위한 WebSocket 푸시(Push) 서버 구성이다. 기존의 React/Vue 프론트엔드와 백엔드 간에 Socket.IO 등을 연결해 두고, 백엔드가 Zenoh 망에서 amr/*/odom 을 지속적으로 Subscribe 하다가 데이터가 들어올 때마다 WebSocket을 통해 브라우저로 중계(Broadcast)하라. 이 브릿징을 통해 구형 프론트엔드를 한 줄도 고치지 않고서 현장의 공장 데이터를 실시간으로 밀어 넣어 줄 수 있다.
