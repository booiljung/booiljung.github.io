21.5.4 로봇 작업 할당 및 IoT 센서 원격 설정 제어 로직 구현

수집과 조회를 끝마쳤다면, 이번에는 클라우드 관제 백엔드가 능동적으로 공장 시스템에 개입하는 양방향(Bi-directional) 제어 로직을 작성하라. 중앙 서버는 현장의 상황을 파악한 뒤 개별 로봇에게 목적지를 하달하고, 오류가 발생한 센서 노드의 민감도를 원격으로 변경할 수 있어야 한다.

첫째, 자율주행 물류 로봇(AMR)으로의 작업(Task) 할당 로직이다.
백엔드 서버 내부의 스케줄러가 대기 중인 로봇의 목록(Zenoh 쿼리 이용)을 확인하고 유휴 로봇에게 업무를 푸시(Publish)한다.

async function assignTaskToRobot(robotId: string, taskData: any) {
    const topic = `/amr/${robotId}/task/assign`;
    // 작업 데이터를 JSON으로 변환하여 퍼블리시
    await zenohSession.put(topic, JSON.stringify(taskData));
    console.log(`Task dispatched to ${robotId}`);
}

둘째, 원거리 IoT 디바이스(C/Rust 기반) 설정 제어다.
현장에 있는 진동 센서가 너무 많은 노이즈 데이터를 보내고 있다면, 관리자는 대시보드 버튼을 눌러 센싱 주기를 10ms에서 50ms로 늦출 수 있어야 한다.
이 경우 /factory/zone_A/sensor/vibration/motor_1/config/interval 토픽에 50 이라는 값을 퍼블리시한다. 에지 단의 C 퍼블리서는 이 config 토픽을 동시에 Subscribe하고 있으므로 콜백이 호출되는 즉시 내부의 usleep() 지연값을 동적으로 변경하게 된다.

결국 클라우드의 백엔드 코드 단 몇 줄의 퍼블리시 명령이, Zenoh의 최단 경로 라우팅을 타고 수천 킬로미터 밖 공장 모터 옆에 붙은 초소형 칩의 동작 사이클을 즉각적으로 제어하는 짜릿한 엔드 투 엔드(End-to-End) 제어를 완성한다.
