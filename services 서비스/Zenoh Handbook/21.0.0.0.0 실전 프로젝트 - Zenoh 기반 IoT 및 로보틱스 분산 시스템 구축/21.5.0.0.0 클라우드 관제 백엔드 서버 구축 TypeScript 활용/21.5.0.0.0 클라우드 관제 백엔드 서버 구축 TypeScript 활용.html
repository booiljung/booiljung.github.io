21.5 클라우드 관제 백엔드 서버 구축: TypeScript 활용

수많은 에지 데이터가 클라우드로 집결했다면 이제 브라우저 기반의 대시보드 애플리케이션에 렌더링하고, 사용자의 마우스 클릭이 역으로 거대한 공장을 제어할 수 있게 만들 차례다. 최신 웹 백엔드 생태계의 표준인 Node.js 환경에서 작동하는 TypeScript에서 Zenoh 패키지를 활용하라.

지금까지의 전통적인 사물인터넷(IoT) 백엔드는 HTTP REST 폴링(Polling)이나 중앙 집중식 MQTT 브로커를 섞어 쓰는 스파게티 구조였다면, Zenoh 기반의 생태계에서는 백엔드 서버마저도 하나의 거대한 분산 애플리케이션 노드(Node)에 불과해진다.

본 절에서는 @eclipse-zenoh/zenoh-ts 라이브러리를 통해 공장 라우터로 직결된 클라우드 프로세스를 만들고, Express나 NestJS와 결합된 아키텍처를 설계한다. 분산 쿼리를 적극 활용해 무거운 DB의 역할 중 일부를 에지 라우터들에게 위임(Offloading)하여 클라우드 백엔드의 아키텍처를 혁신하라.
