21.5.1 TypeScript와 Node.js 기반의 관제 웹 서버 아키텍처 설계

스마트 팩토리의 수많은 장비 및 로봇 상태를 모니터링하기 위해 백엔드 구조 역시 비동기 I/O를 적극적으로 관리해야 한다. TypeScript에서 Zenoh 및 Express 프레임워크를 결합하여 웹 서버 아키텍처를 설계하라.

첫째, 웹 서버(Node.js 인스턴스) 자체가 Zenoh 클라이언트 모드로 클라우드 메인 라우터에 세션을 맺도록 구성하라.

import { open, Config, Session } from "@eclipse-zenoh/zenoh-ts";
import express from 'express';

const app = express();
let zenohSession: Session;

async function initZenoh() {
    const config = new Config();
    // 클라우드 라우터(localhost에 데몬으로 띄운 zenohd)에 연결
    config.insertJson5("mode", `"client"`);
    zenohSession = await open(config);
    console.log("Connected to Zenoh Cloud Router");
}

app.listen(3000, async () => {
    await initZenoh();
    console.log("TypeScript Dashboard Backend Running");
});

둘째, 서버리스(Serverless) 트래픽 회피다. 웹 브라우저 클라이언트가 1초에 한 번씩 수만 개의 센서 데이터를 REST API로 묻는 구조를 탈피하라. 그 대신 백엔드는 Zenoh 라우터로부터 Subscribe 콜백을 등록해 놓고, React 기반의 웹 프론트엔드와는 단일 WebSocket으로 연결을 묶어 데이터가 도착할 때마다 즉시 푸시(Push) 해주는 반응형 구조를 채택해야 한다.

셋째, 메모리 리크(Leak) 방지다. TypeScript에서 Zenoh 워커를 생성할 때 동적으로 수많은 로봇들의 토픽 구독(Subscriber) 인스턴스를 만들고 삭제하게 되는데, 로봇 로그오프 시 subscriber.close()를 호출하여 이벤트 루프에 누수가 발생하지 않도록 코드를 철저하게 생명주기(Lifecycle)에 바인딩하라.
