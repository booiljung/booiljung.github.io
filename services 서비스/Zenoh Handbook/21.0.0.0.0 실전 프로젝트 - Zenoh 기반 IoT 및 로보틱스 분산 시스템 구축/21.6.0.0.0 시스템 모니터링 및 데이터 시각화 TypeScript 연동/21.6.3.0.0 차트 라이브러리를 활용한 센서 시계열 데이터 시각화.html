21.6.3 차트 라이브러리를 활용한 센서 시계열 데이터 시각화

AMR의 위치가 2D 캔버스 공간의 점이라면, 공장 에지(Edge)의 수많은 모터와 프레스기가 뿜어내는 데이터는 시간의 축을 따라 흐르는 시계열(Time Series) 데이터로 시각화해야 한다. C 기반의 고주파수 노드에서 수집되어 InfluxDB에 저장된 데이터, 그리고 현재 실시간으로 스트리밍되는 데이터를 Chart.js나 Apache ECharts 같은 라이브러리에 렌더링하라.

첫째, 과거 데이터와 실시간 데이터의 봉합이다. 화면에 차트를 그리기 시작할 때, 클라우드의 TypeScript 웹 백엔드가 Zenoh의 session.get('/factory/zone_A/sensor/temp') 분산 쿼리를 영속성 백엔드(InfluxDB)로 날려 최근 1시간의 과거 이력(History) 배열을 반환받는다. 이를 차트에 일차적으로 그린 후, 이후 들어오는 실시간 데이터는 배열의 뒤에 push() 하고 가장 오래된 데이터를 shift() 하며 애니메이션 효과를 부여하라.

둘째, 주파수 다운샘플링(Down-sampling)이다. 산업용 진동 센서가 1kHz로 퍼블리시한 모든 데이터를 브라우저에 들이부으면 탭이 멈춘다.
TypeScript 백엔드가 데이터를 수신할 때 LTTB(Largest Triangle Three Buckets) 알고리즘이나 단순 평균법을 적용하여 1000개의 점을 10개로 묶어 프론트엔드로 전달하라. 이를 통해 브라우저 차트 라이브러리의 DOM 렌더링 병목 현상을 원천적으로 차단할 수 있다.
