21.6.2 2D/3D Canvas를 이용한 공장 내 AMR 실시간 위치 매핑

ROS2와 Zenoh를 거쳐 올라온 로봇 오도메트리(Odometry - amr/+/odom) 데이터 속 기체의 x, y 좌표 및 방향각(Yaw) 정보를 화면에 그리는 기능은 대시보드에서 가장 시각적 몰입도가 높은 요소다.

정적인 HTML DOM으로 수십 대의 로봇 아이콘을 움직이면 퍼포먼스 제약이 심하다. 이를 회피하기 위해 HTML5 Canvas 2D 혹은 Three.js 기반의 WebGL 3D 캔버스를 활용하라.

첫째, Canvas 2D 기반 매핑이다. 배경에는 공장 레이아웃 지도를 이미지로 깔아두고, 그 위에 로봇의 현재 좌표를 뷰포트(Viewport) 픽셀 좌표로 변환(Transformation Matrix)하여 렌더링해라.

function drawRobots(ctx, robotsList) {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    // (옵션) 배경 지도 렌더링
    
    robotsList.forEach(robot => {
        const px = mapXtoPixel(robot.x);
        const py = mapYtoPixel(robot.y);
        
        ctx.save();
        ctx.translate(px, py);
        ctx.rotate(robot.theta); // 방향 회전
        
        // 화살표 또는 로봇 아이콘 드로잉
        ctx.fillStyle = robot.battery < 20 ? 'red' : 'green';
        ctx.fillRect(-10, -10, 20, 20);
        
        ctx.restore();
    });
}

둘째, 렌더링 루프 분리다. 데이터가 도착할 때마다 그리지 말고 브라우저의 requestAnimationFrame을 사용하여 모니터 주사율(주로 60fps)에 맞춰 부드럽게 화면을 갱신해야 시각화 안정성이 높다. 이 과정은 Zenoh 시각화의 정점이며 ROS의 Rviz2 대신 웹에서 클라우드 관제를 대체하는 강력한 효과를 발휘한다.
