21.6.4 로봇의 현재 위치(Map) 및 라우팅 경로의 웹 브라우저 시각화 구현

AMR에서 생산되는 ROS2의 nav_msgs/Odometry 메시지는 x, y, z 좌표와 쿼터니언(Quaternion) 회전각을 포함한다. 브리지(Bridge)를 통과하여 Node.js를 거쳐 브라우저로 들어온 이 숫자를 2D 평면도에 찍어라.

[Canvas API를 활용한 로봇 드로잉]
공장의 도면 레이아웃 이미지를 백그라운드에 깔고, 그 위에 HTML5 Canvas를 오버레이(Overlay) 한다.

// 렌더 타겟 함수 예시 (React Canvas)
function drawRobot(ctx: CanvasRenderingContext2D, x: number, y: number, yaw: number) {
    ctx.save();
    // 공장 맵의 배율에 맞게 픽셀(Pixel) 좌표로 스케일링 변환
    ctx.translate(x * SCALE_FACTOR, y * SCALE_FACTOR); 
    // 로봇의 회전각(Heading) 반영
    ctx.rotate(yaw);
    
    // 로봇을 나타내는 화살표나 원 그리기
    ctx.fillStyle = "blue";
    ctx.fillRect(-10, -10, 20, 20); 
    ctx.restore();
}

웹 브라우저는 60FPS로 draw_robot 을 무한 호출하며, 전 세계 어디서 접속하든 공장 바닥의 로봇 수십 대가 꿀벌처럼 움직이는 관경을 끊김 없이 감시할 수 있게 된다.
