Chapter 6. C++에서의 Zenoh

자율주행 모빌리티(Autonomous Mobility)의 인지 및 제어 프레임워크, 스마트 팩토리의 고성능 프로그램 가능 논리 제어기(PLC), 그리고 ROS(Robot Operating System) 생태계에 이르기까지, 현대 하드 리얼타임(Hard Real-time) 산업계의 기저에는 여전히 **C++**이라는 견고한 언어가 자리 잡고 있다. 막대한 연산 자원이 필요한 밀리초(ms) 단위의 궤적 계산이나, 고빈도 거래(HFT)와 같이 지연 시간(Latency)에 극도로 민감한 도메인에서 C++은 대체 불가능한 지위를 갖는다.

Zenoh 코어 모듈이 Rust로 완벽히 안전하게 작성되었음에도 불구하고, 이를 기존의 거대한 C/C++ 생태계와 매끄럽게 연결하기 위해 Zenoh는 C/C++ 바인딩을 강력하게 지원한다. 본 장(Chapter 6)에서는 네이티브 레벨의 성능을 훼손하지 않으면서도, C++ 개발자들이 모던 아키텍처에서 요구하는 고도의 분산 네트워킹을 어떻게 이식할 수 있는지 체계적으로 고찰한다.

1. zenoh-cpp 아키텍처와 메모리 모델

Zenoh의 외부 인터페이스(FFI, Foreign Function Interface)는 완벽하게 C 언어 표준(C ABI)을 준수하도록 설계된 zenoh-c로 노출된다. 그리고 이 zenoh-c 계층 위에 C++17 규격을 바탕으로 RAII(Resource Acquisition Is Initialization) 패턴과 지능형 포인터(Smart Pointer)를 결합하여 개발자 생산성을 극대화한 계층이 바로 zenoh-cpp 라이브러리이다.

기존의 C 언어 개발자들이 포인터(Pointer) 제어와 메모리 할당(malloc/free)의 늪에 빠져 메모리 누수(Memory Leak)에 취약했던 반면, zenoh-cpp는 std::shared_ptr 와 std::unique_ptr를 통해 세션(Session), 퍼블리셔(Publisher), 그리고 쿼리어블(Queryable) 객체의 생명주기를 안전하게 추상화한다.

2. C++ 환경에서의 동시성과 제로 카피(Zero-Copy)

C++ 환경에서 Zenoh의 진가는 메모리 복사를 억제하는 고성능 통신 기법에 있다.
데이터 발행(Publish) 및 분산 질의(Distributed Query) 처리 과정에서 발생하는 복사(Copy) 연산을 최소화하기 위해 제로 카피(Zero-Copy) API를 활용하는 전략을 제시한다(6.5). 또한, 센서 데이터처럼 무거운 페이로드(Payload)를 동일 호스트 내에서 전송할 때 병목을 완전히 제거하는 공유 메모리(Shared Memory) 아키텍처 활성화 방안을 다룬다.

이와 더불어 람다(Lambda) 표현식을 콜백(Callback) 메커니즘으로 채택하여 구독(Subscribe) 이벤트를 다룰 때 발생하는 멀티스레딩(Multi-threading) 환경 속 문맥 교환(Context Switch)의 비용을 최소화하고 스레드 안전성(Thread-safety)을 사수하는 방법을 익힌다.

3. CMake 통합 및 C++ 생태계 연동

C++ 프로젝트는 고유의 복잡한 빌드 아키텍처를 수반한다. 본 장은 단순히 소스코드를 설명하는 데 그치지 않고, CMake 기반 프로젝트 파이프라인에 Zenoh를 결합하고, 외부 종속성 관리 도구(vcpkg, Conan)와 통일성 있게 연동하는 크로스 컴파일(Cross-compile) 전략(6.1)을 제시한다.

또한, 하드웨어 제어기를 넘어서 Protocol Buffers (Protobuf)나 FlatBuffers와 같은 범용 직렬화(Serialization) 프레임워크와의 데이터 구조 매핑(6.7)을 시연함으로써 이기종 언어 간의 데이터 포맷 호환성을 증명한다. 장의 후반부(6.9)에서는 다중 센서 융합(Multi-sensor Fusion) 노드라는 실전 미니 프로젝트 구축을 통해 성능 튜닝(Valgrind, AddressSanitizer 활용) 및 로깅 시스템 연동 방법까지 망라하여 다룬다. 이를 통해 C++ 아키텍트들은 레거시 시스템을 차세대 분산 모델로 안전하게 이식(Migration)할 강력한 지침서를 획득하게 될 것이다.
