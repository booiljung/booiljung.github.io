18.7.2 GeoJSON 및 Web GIS(Mapbox, Leaflet) 기반 공간 데이터 연동

구글 지도(Google Maps) 만 띄워놓는다고 끝이 아니다. 우리가 통제하는 공장의 금지 구역(Geofence) 이나 화재 지점 같은 우리만의 공간 빅데이터(GIS) 와 로봇이 오버레이 되어야 한다.

0.1 [Runbook] 지리 공간 결합(Geospatial Binding) 런북

오픈소스 기반인 Leaflet 이나 고성능 패러다임인 Mapbox GL JS 에 Zenoh 를 물린다.

1. GeoJSON 폴리곤 (정적 지형물)
관제사가 지도에 그어놓은 농장 경계선이나 드론 비행 금지선은 GeoJSON 형태의 벡터 그래픽이다. 이것은 정적이므로 초기 페이지 로딩 시 한 번만 그린다.

2. Mapbox Deck.gl 을 활용한 10,000 대 밀집 렌더링
Mapbox 의 가장 무서운 파트너인 Deck.gl 에 Zenoh 웹소켓 구독기를 끼워 넣는다.
Zenoh 에 들어온 수천 대의 드론 좌표([lng, lat, alt]) 배열을 Deck.gl 의 IconLayer 나 ScatterplotLayer 속성에 통으로 꽂아버리면, 브라우저의 GPU(WebGL) 가 미친 듯이 연산을 분산 처리하여 1만 대의 드론이 강남역에 새카맣게 몰려드는 개미 떼 같은 애니메이션을 60 프레임(FPS) 으로 아주 부드럽게 그려낸다. DOM(Div 태그) 으로 지도 마커를 찍는 Leaflet 따위로는 1,000 개만 붙어도 브라우저가 장렬히 폭사한다.
