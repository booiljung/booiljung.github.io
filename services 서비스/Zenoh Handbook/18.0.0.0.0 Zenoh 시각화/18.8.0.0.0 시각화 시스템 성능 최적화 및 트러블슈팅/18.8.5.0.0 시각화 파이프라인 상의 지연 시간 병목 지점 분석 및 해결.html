18.8.5 시각화 파이프라인 상의 지연 시간 병목 지점 분석 및 해결

그래프가 안 나온다. 로봇 탓인가, 네트워크 탓인가, 그라파나 탓인가, 당신 브라우저 탓인가.

0.1 [인스펙션] 시각화 망막 파열 분석 전술

이 병목을 찾아내려면 프론트엔드 개발자도 브라우저를 까고 F12 를 눌러야 한다.

1. 크롬 개발자 도구 (Performance Tab) 해부
브라우저 F12 의 Performance 탭에서 녹화를 누르고 5초간 대시보드를 관찰하라.
노란색 막대(Scripting) 가 화면 전체를 덮고 있으면 백엔드 잘못이 아니라 자바스크립트를 거지같이 짠 것이다. (웹 워커 도입).
반대로 보라색 막대(Rendering / Painting) 가 미친 듯이 치솟아 있다면, 한 번에 너무 많은 점들을 DOM 에 쑤셔 넣느라 브라우저 그래픽 엔진이 터진 것이다. (Canvas 엔진으로 교체 필요).

2. Time-to-Glass (유리 도달 시간) 분석
센서에서 빛이 출발해 내 모니터 유리(Glass) 표면에 닿기까지의 파이프라인 구간을 해부한다.

  • A: 센서 \rightarrow DB (Zenoh Ingestion Time)
  • B: DB \rightarrow 시각화 툴 (Grafana Query Time)
  • C: 화면에 그리기 (Browser Render Time)
    그라파나 하단의 쿼리 인스펙터(Query Inspector) 기능을 켜고 쿼리 시간을 까봐라.
    만약 B구간 쿼리(Query) 속도가 5초가 걸린다면, 아무리 브라우저 프론트엔드 최적화를 해봤자 소용없다. 인프라 DB 로 당장 달려가 타임시리즈 인덱싱(Indexing) 과 하드 디스크 I/O 속도 폭발을 막는 근원적 튜닝(14장 참고) 에 착수해야 한다.
