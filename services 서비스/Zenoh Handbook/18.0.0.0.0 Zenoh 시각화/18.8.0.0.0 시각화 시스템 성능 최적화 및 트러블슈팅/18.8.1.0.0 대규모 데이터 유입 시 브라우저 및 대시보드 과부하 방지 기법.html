18.8.1 대규모 데이터 유입 시 브라우저 및 대시보드 과부하 방지 기법

브라우저의 가비지 컬렉터(Garbage Collector) 는 빅데이터를 혐오한다. 1초마다 수천 개의 객체가 생기고 버려진다면 화면 전체가 1초 단위로 뚝뚝 끊기는 프리징(Freezing) 이 온다.

0.1 [인스펙션] 메모리 누수 방탄(Bulletproof) 전술

이벤트 버블링과 쓰레기 객체 생성의 악순환을 끊어야 한다.

1. 변수 풀링 (Object Pooling)
new 키워드로 배열이나 객체를 계속 생성하지 마라.
Zenoh 에 들어온 패킷 배열을 다룰 때, 길이가 10만인 Array 를 딱 하나만 화면 로딩 시 처음 만들어두어라(미리 할당). 그리고 새 데이터가 들어올 때 무식하게 덮어쓰기(Overwrite) 만 수행해야 가비지 컬렉터가 잠에서 깨지 않고 프레임을 칼같이 지킬 수 있다.

2. 보이지 않으면 렌더링하지 않는다 (DOM Virtualization)
Grafana 가 그려주는 패널이 50개인데, 한 화면에 보이는 건 5개다.
웹 UI 가 화면 밖(Off-screen) 에 스크롤 되어 숨겨진 45개의 차트를 여전히 CPU 로 갱신하고 있다면 당장 옵션을 켜라. (React 의 react-window, Intersection Observer). 눈에 보이지 않는 패널은 Zenoh 의 Subscribe 자체를 Pause 시키거나 차트 업데이트를 동결하는 기능이 반드시 구현되어야 브라우저 과부하를 막을 수 있다.
