18.8.3 백그라운드 워커(Web Worker)를 활용한 시각화 연산 분산 처리

자바스크립트는 태생적으로 싱글 스레드(Single-Thread) 다. 데이터 파싱 연산이 UI 쓰레드를 막아버리면, 사용자가 마우스 클릭을 해도 패널이 열리지 않는 저주에 걸린다.

0.1 [인스펙션] 브라우저 내부의 코어 찢기 전술

HTML UI 의 부드러운 반응 속도를 수호하기 위한 병렬 아키텍처.

1. Zenoh Payload 파싱의 감금
Zenoh 로 날아온 바이너리 페이로드(MsgPack 등) 를 푸는 행위는 극도로 무거운 연산이다.
이 파싱 로직 전체를 worker.js (Web Worker) 안으로 유배 보낸다. 메인 쓰레드는 아무것도 고민하지 않고, 그저 Web Worker 가 예쁘게 해독해서 넘겨준 좌표 평면의 배열(postMessage()) 만 받아다가 즉시 그리기만 하면 마우스는 폭풍 속에서도 끊기지 않는다.

2. 차트 레이아웃 연산 스레드 오프로딩
노드 1,000 대짜리 네트워크 토폴로지를 Cytoscape.js 로 켤 때마다 화면이 3초간 얼어붙는다? 레이아웃 정돈 연산을 워커 안에 가둬야 한다. (예: fcose 의 Web Worker 지원 모드 활성화). 화면은 빙글빙글(Loading Spin) 계속 부드럽게 돌아가면서, 워커가 연산을 마쳤다고 신호를 주면 그때 1ms 만에 노드를 착, 하고 펼치면 된다.
