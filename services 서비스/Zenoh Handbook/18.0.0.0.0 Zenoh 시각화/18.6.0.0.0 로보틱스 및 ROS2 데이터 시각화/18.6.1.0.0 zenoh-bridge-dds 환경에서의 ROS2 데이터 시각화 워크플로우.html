18.6.1 zenoh-bridge-dds 환경에서의 ROS2 데이터 시각화 워크플로우

관제 센터에 앉아있는 당신의 ROS2 시각화 툴(RViz2) 은, 캘리포니아 공장에 있는 로봇의 존재를 모른다. DDS 멀티캐스트(UDP) 는 바다를 건널 수 없기 때문이다.

0.1 [인스펙션] 시각적 웜홀(Wormhole) 개통 전술

zenoh-bridge-dds 는 이 단절된 시야를 이어주는 투명한 유리창이다.

1. 대륙 간 브릿지(Bridge) 파이프라인 결속

  • 공장(로봇 쪽): 로봇이 내뿜는 100개의 DDS 토픽을 zenoh-bridge-dds 가 집어삼킨 뒤, 단 하나의 TCP 소켓으로 압축해 클라우드 라우터로 쏜다.
  • 관제소(나의 PC 쪽): 내 PC 에도 zenoh-bridge-dds 를 켠다. 클라우드 라우터에서 내려온 TCP 데이터를 받아, 다시 내 PC 내부망에 DDS 멀티캐스트로 흩뿌린다. (로컬 호스트 디스커버리)

2. 구독 트리거 (On-Demand Visualizing)
Zenoh 브릿지의 마법은 “게으름(Laziness)” 에 있다.
내가 PC 에서 RViz2 를 켜지 않으면 데이터는 바다를 건너오지 않는다.
하지만 내가 RViz2 를 켜고 /robot/lidar 화면 표시 체크박스를 누르는(Subscribe) 그 순간! ROS2 통신 규약에 의한 Discovery 가 트리거되고, Zenoh 브릿지가 대륙 너머 로봇에게 “야 지금 당장 라이다 주시 시작해!” 라고 명령하여 비로소 대용량 그래픽 데이터가 흐르기 시작하는 완벽한 On-Demand 워크플로우를 보장한다.
