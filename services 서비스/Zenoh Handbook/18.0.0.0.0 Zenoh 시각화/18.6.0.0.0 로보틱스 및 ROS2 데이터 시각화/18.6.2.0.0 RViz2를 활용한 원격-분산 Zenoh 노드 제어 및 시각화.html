18.6.2 RViz2를 활용한 원격/분산 Zenoh 노드 제어 및 시각화

관제탑 노트북에서 RViz2 (ROS2 표준 3D 시각화 도구) 가 무사히 캘리포니아 로봇의 라이다를 잡았다. 그런데 로봇을 원격 조종하려니 지연 시간(Lag) 때문에 차가 벽에 처박힌다.

0.1 [Runbook] RViz2 텔레오퍼레이션(Teleop) 튜닝 런북

단방향 시각화와 양방향 제어(Control) 는 차원이 다른 오버헤드다.

1. QoS 프로필 불일치 해결
로봇은 카메라 영상을 Best-Effort (UDP, 빠르지만 손실 가능) 로 쏘는데, 관제탑 RViz2 의 영상 표시 위젯이 기본값인 Reliable (TCP, 느리지만 완벽함) 로 설정되어 있다면 화면이 영원히 나오지 않는다.
원격 제어 RViz 의 모든 렌더링 패널(Topic) 설정 창에 들어가, QoS 정책을 반드시 Best-Effort 와 Volatile (과거 데이터 안 받음) 로 튜닝하여 브릿지가 에러 없이 패킷을 즉시 버릴 수(Drop) 있도록 허락하라.

2. Teleop 제어 토픽(cmd_vel) 역전송 분리
보는 것(영상) 은 지연되어도 되지만, 조이스틱 제동(Stop) 명령은 0.1초 만에 로봇에 도달해야 한다.
zenoh-bridge-dds 설정(allow 룰) 에서 /cmd_vel 토픽만큼은 반드시 Reliable 브릿지 채널을 거치거나, 우선순위가 높은 프록시 대역(Priority 1) 을 태워 시각화 렌더링 찌꺼기 패킷들에 조향 명령이 파묻히는 참사를 원천 세팅으로 틀어막아야 한다.
