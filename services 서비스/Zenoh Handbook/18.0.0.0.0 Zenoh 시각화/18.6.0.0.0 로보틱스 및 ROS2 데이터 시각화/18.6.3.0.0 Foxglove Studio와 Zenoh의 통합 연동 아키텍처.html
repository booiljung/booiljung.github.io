18.6.3 Foxglove Studio와 Zenoh의 통합 연동 아키텍처

로봇 엔지니어들은 C++ 기반의 RViz2 를 좋아하지만, 웹 브라우저 하나로 접속해야 하는 클라우드 관제 플랫폼에서는 차세대 로보틱스 시각화 표준, 즉 Foxglove Studio 가 절대 권력이다.

0.1 [인스펙션] 진화된 브라우저 관제 센터 가동

1. Foxglove WebSocket 브릿지 통신망
Foxglove 는 기본적으로 웹 기술 베이스다. 로컬 PC 의 DDS 통신에 기생하는 대신, rosbridge_server (또는 foxglove_bridge) 를 로봇단에 켜둔다.
Zenoh 는 이 로봇 내부의 Foxglove 웹소켓 포트(8765) 트래픽을 통째로 릴레이(Relay) 하여, 관제탑 브라우저에 접속한 Foxglove UI 화면으로 토스해버리는 완벽한 TCP 터널 역할을 수행한다.

2. 메세지 규격의 투명성 (Schema Resolution)
Foxglove 의 최대 강점은 데이터를 해석하는 IDL(스키마) 파일이 없어도 동적으로 그래프를 그린다는 것이다.
Zenoh 망을 통해 넘긴 ROS2 메시지 바이너리에는 그 구조(Type) 가 꼬리표처럼 달려 있다. Foxglove 가 이 꼬리표를 읽어 자동으로 “온도 꺾은선그래프” 나 “카메라 이미지” 위젯을 마법처럼 스냅(Snap) 시키는 놀라운 연동성을 체감하라. 더 이상 개발용 PC 에 무거운 ROS2 전체 패키지를 깔 필요가 없다. 브라우저면 충분하다.
