18.6.5 다중 로봇(Multi-Robot) 관제 화면 설계 및 구현

로봇이 100대다. RViz2 화면을 창 100개 띄워놓고 볼 것인가?
다중 로봇 관제탑 (Fleet Management System, FMS) 의 화면은 개별 로봇의 현미경이 아니라, 100대의 로봇 전체를 포괄하는 거시적 위성 지도여야 한다.

0.1 [인스펙션] 전지적 작전 지도(Bird’s Eye View) 전술

우리의 시각화 패러다임이 3D 공간(Room) 레벨에서 국토(Map) 레벨로 확산한다.

1. 네임스페이스(Namespace) 기반 클러스터링 매핑
각 로봇의 좌표 토픽 구조가 fleet/seoul/robot_001/gps 와 같이 네임스페이스로 완벽히 격리되어 있어야 한다.
웹 대시보드의 메인 화면(지도 패널) 은 fleet/**/gps 와일드카드로 100대 로봇의 좌표를 한 번에 다 구독한다. 지도상에는 개별 로봇이 점 하나(Dot) 로 표시되거나 너무 뭉쳐있다면 집단 핀(Cluster Marker) 1개로 합쳐져 큰 물결처럼 움직임을 표현한다.

2. 트래픽의 스파셜(Spatial) 멀티플렉싱 뷰
관제사가 지도에서 에러가 난 “특정 로봇 아이콘(robot_042)” 을 마우스로 클릭하는 순간!
대시보드 프론트엔드는 그 즉시 다른 99대 로봇의 모든 데이터 구독(Subscription) 을 중지하고, fleet/*/robot_042/** 와일드카드의 고해상도 정보(Vitals, LiDAR, Camera) 만을 집중적으로 빨아당기는 “저격용 구독 전환(Targeted Sub Switch)” 아키텍처를 가동해야 한다.
보지 않는 것은 구독하지 않는 우아한 토픽 통제. 이것이 1만 대 로봇 시각화를 브라우저 하나로 통제하는 FMS(Multi-Robot) 관제 프론트엔드의 정통 코어 철학이다.
