18.2.4 토폴로지 시각화를 위한 오픈소스 그래프 도구 연동 예 (D3.js, Cytoscape.js)

모두가 처음부터 WebGL 코드를 짤 수는 없다. 거인들의 어깨, D3.js 와 Cytoscape.js 의 물리 엔진을 훔쳐 써라.

0.1 [Runbook] 엔진 선택의 십자군 전술

당신의 네트워크 사이즈에 맞춰 엔진을 택해야 브라우저가 안 뻗는다.

1. 100 노드 미만: D3.js (예술적 자유도)

  • 장점: 커스터마이징의 끝판왕. 노드 모양을 로봇(SVG) 아이콘으로 만들고, 선(Edge) 위를 패킷 알갱이들이 날아가는 애니메이션까지 땀구멍까지 제어할 수 있다.
  • 단점: 100개가 넘어가는 순간 중력(Force-directed) 알고리즘이 꼬이면서 화면의 프레임이 10fps 이하로 곤두박질친다.

2. 1,000 노드 이상: Cytoscape.js (엔터프라이즈의 무식함)

  • 장점: 바이오(Bio) 유전자 맵핑, 엔터프라이즈 망 관제에 특화된 학술적/엔터프라이즈 엔진이다. 수백 개의 노드가 얽혀도 가장 안정적으로 가지런히 트리/그물망 구조를 펴주는 막강한 레이아웃(Layout) 연산 알고리즘을 보유하고 있다.
  • Cola 나 fCose 레이아웃 플러그인을 끼워서 Zenoh 의 JSON 맵을 때려 부으면, 자기가 알아서 라우터들은 한가운데 뭉쳐주고(Core), 클라이언트들은 바깥 테두리(Edge) 로 튕겨내 주는 기가 막힌 자동 정렬을 수행한다.
