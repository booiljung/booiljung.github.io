18.2.1 네트워크 아키텍처 매핑: 라우터, 피어, 클라이언트 식별

그래프를 그리려면 화면에 찍을 놈이 동그라미(라우터) 인지, 네모(클라이언트) 인지, 세모(피어) 인지 노드의 신분증(Type) 을 발급해주어야 한다.

0.1 [인스펙션] 노드 신분 추출 전술

라우터의 Admin Space(17.2장) 에서 가져온 sessions 정보를 파싱하여 생물학적 분류를 수행한다.

1. 세션의 WHAT AM I 속성 파싱

// GET /@/router/local/sessions 의 응답 예시 일부
[
  { "id": "A1B2", "whatami": "Client", "locators": "tcp/10.0.0.5:7447" },
  { "id": "C3D4", "whatami": "Peer", "locators": "udp/10.0.0.8:7447" }
]

이 JSON 을 파싱하여 자바스크립트 객체의 type 속성에 우겨 넣는다. 라우터는 크고 굵은 원형 노드로 치환하고, 클라이언트는 라우터 밑에 매달린 작은 나뭇잎(Leaf) 모양으로 계층을 물리적으로 분리하는 디자인이 필요하다.

2. 식별 불가능성(Unidentifiable) 배제
간혹 Zenoh 버전이 낮거나 외부 브릿지(MQTT Bridge) 인 경우 whatami 속성이 모호할 때가 있다. 이런 경우 토픽의 패턴(*/bridge/**) 을 훔쳐보아서라도 프론트엔드 엔진에게 강제로 “얘는 외부 시스템이다” 라는 회색(Gray) 또는 파선(Dashed) 노드 스타일을 부여해야 한다.
