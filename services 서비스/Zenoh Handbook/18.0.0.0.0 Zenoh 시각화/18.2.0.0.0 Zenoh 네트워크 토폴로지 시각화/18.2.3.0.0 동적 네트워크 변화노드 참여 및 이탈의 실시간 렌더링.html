18.2.3 동적 네트워크 변화(노드 참여 및 이탈)의 실시간 렌더링

Zenoh 망은 생물이다. 드론이 날아오르면 노드(Node) 가 하나 뿅 하고 나타나 피어(Peer) 에 붙고, 와이파이가 끊어지면 사라진다. 시각화 화면이 이 변화를 10초 늦게 보여주면 이미 그 관제 화면은 죽은 데이터다.

0.1 [인스펙션] 델타(Delta) 렌더링 전술

화면을 지웠다(Clear) 다시 그리는(Redraw) 짓은 브라우저 CPU 를 찢어버린다.

1. 이벤트 드리븐(Event-Driven) 노드 갱신
REST API 로 주기적인 폴링(Polling) 을 하는 대신, Zenoh 웹소켓 플러그인에 subscribe 를 걸어 “새로운 세션이 붙었다(Join)” 또는 “세션이 터졌다(Leave)” 라는 델타 이벤트(Delta Event) 메시지만 받아야 한다.

2. 브라우저 생동감 부여 (Animation)

  • 참여 (Join): 드론이 켜지면 노드가 투명도(Opacity) 0에서 1로 페이드인(Fade-in) 되면서 주변 라우터의 인력(Gravity) 에 의해 화면 밖에서 빨려 들어오듯이 렌더링해야 관제사가 새로운 기기의 출현을 즉각 눈치챈다.
  • 이탈 (Leave): 드론이 터졌다고 노드를 화면에서 즉시 빡! 하고 지워버리면 인지 부조화가 온다. 노드를 붉은색 해골(또는 회색) 로 바꾼 뒤, 10초에 걸쳐 서서히 페이드아웃(Fade-out) 시키며 “방금 이곳에서 누군가 죽었다“는 여운(Shadow) 을 남겨 시각적 인지력을 보존하라.
