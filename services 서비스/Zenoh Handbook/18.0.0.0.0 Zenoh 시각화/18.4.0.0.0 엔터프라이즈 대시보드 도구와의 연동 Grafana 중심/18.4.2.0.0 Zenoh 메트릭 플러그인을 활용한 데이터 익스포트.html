18.4.2 Zenoh 메트릭 플러그인을 활용한 데이터 익스포트

만약 Zenoh 가 기본적으로 내뿜는 메트릭 외에 다른 커스텀 데이터가 보고 싶다면?

0.1 [Runbook] 데이터 착즙기(Juicer) 확장 전술

1. StatsD / Telegraf 브릿지 우회
Zenoh 가 내장 프로메테우스 익스포터를 지원하지만, 사내 인프라 규정상 InfluxDB 나 Datadog 으로 데이터를 밀어 넣어야(Push) 한다면 Telegraf 데몬을 중간에 끼워 넣어야 한다.
Zenoh 가 쓰는 포트(8000번) 를 Telegraf 가 긁어간(Scrape) 뒤, 뒷단에서 Influx 양식으로 변환하여 쏘아주는 데이터 번역기(Translator) 파이프라인을 구축하라.

2. 변수 템플릿(Variable Templating) 필터
라우터가 100 대인데 대시보드를 100 개 그릴 것인가?
그라파나의 상단에 Variable 을 추가하고, PromQL 함수 label_values(zenoh_session_count, instance) 를 적어 넣는다. 그러면 그라파나 화면 위쪽에 100대의 라우터 IP 가 쭉 나오는 드롭다운 박스가 자동 생성된다. 사용자가 IP 를 바꿀 때마다 대시보드 전체의 그래프가 그 라우터의 데이터로 한 번에 착착 바뀌는 “동적 대시보드(Dynamic Dashboard)” 야말로 엔터프라이즈 시각화의 꽃이다.
