18.4.4 주요 성능 지표(Throughput, Latency, CPU/Memory) 패널 구성

가장 크고 화려하게 대시보드 정중앙에 배치해야 할 생명 유지 장치들이다.

0.1 [Runbook] 심박수(Vitals) 모니터 전술

1. 대역폭(Throughput) 차트: 들숨과 날숨

  • 패널: Time series (꺾은선)
  • 초록색 선은 rate(rx_bytes) (수신), 빨간색 선은 rate(tx_bytes) (송신) 으로 그린다. 정상적인 라우터라면 초록색이 들어온 만큼 빨간색이 나가야 한다. 만약 초록색은 치솟는데 빨간색이 바닥을 기고 있다면 그 라우터는 지금 먹기만 하고 내보내지 못하는 변비(Deadlock/Buffer full) 에 걸린 것이다.

2. 지연 시간(Latency) 차트: 아우토반 스피드 건

  • 패널: Heatmap 또는 Time series
  • histogram_quantile(0.99, ...) 쿼리를 이용해 하위 99% 의 속도가 얼마나 걸리는지 방어선을 친다. 평균(0.50) 속도는 아무 쓸모가 없다. 가장 느린 쓰레기 같은 상위 1% 패킷이 얼마나 쳐지는지를 감시하는 것이 엔터프라이즈 관제의 핵심이다.

3. 하드웨어(CPU/Memory) 퓨즈 차트

  • 라우터가 메모리를 80% 이상 먹었을 때 노란색, 95% 이상 먹었을 때 빨간색으로 게이지(Gauge) 색깔이 자동으로 바뀌게 세팅하여 시각적 직관성을 극대화하라.
