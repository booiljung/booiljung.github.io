18.4.1 Prometheus 및 Grafana를 활용한 시각화 아키텍처 설계

그라파나 자신은 데이터베이스가 아니다. 오로지 남의 데이터(Data Source) 를 예쁘게 포장해서 보여주는(View) 프론트엔드 액자일 뿐이다.

0.1 [인스펙션] 전시관(Gallery) 조립 전술

1. 데이터 소스(Data Source) 연동
그라파나를 켜고 Connections -> Data Sources 메뉴에서 17장에서 세팅한 Prometheus 서버의 URL (http://prometheus:9090) 을 꽂아 넣는다. 이 한 번의 연결로 그라파나는 Zenoh 라우터가 뱉어내는 모든 지표를 자신의 붓과 물감으로 쓸 권한을 얻는다.

2. 대시보드 아키텍처 (Layered Design)
대시보드를 막 만들지 마라. 화면을 3개의 층(Layer) 으로 나누어야 한다.

  • Top 층 (경영진 뷰): 현재 네트워크 전체의 총 대역폭, 총 연결된 로봇 수. (거대한 숫자 타일)
  • Middle 층 (운영팀 뷰): 라우터별 CPU 사용량, 패킷 전송 수 꺾은선그래프. (비교 차트)
  • Bottom 층 (엔지니어 뷰): 각 세션별 패킷 Drop 발생량, 메모리 프로파일링 (에러 텍스트).
