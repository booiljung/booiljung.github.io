18.3.1 Pub/Sub 메시지 배포 경로 및 흐름 추적

Put 한 번에 구독자가 10명이라면, 데이터는 1번 쏘일 때 라우터 안에서 10갈래로 복제되어(Multicast) 사방으로 산개한다.

0.1 [인스펙션] 빛의 궤적(Particle Animation) 전술

D3.js 와 WebGL 을 사용한 점 이동(Particle) 시각화 런북이다.

1. 트리거 기반 파티클 사출
어떤 로봇이 robot/cam 토픽으로 데이터를 쏘았다는 이벤트 메트릭이 수집되는 즉시, 시각화 프론트엔드는 지도 위의 “해당 로봇 노드” 에서 출발하는 작은 형광색 공(Particle) 애니메이션 10개를 뱉어낸다.

2. 브랜칭 애니메이션 (Branching)
이 공은 선(Edge) 을 타고 이동하다가 “라우터 A” 에 도착한다. 프론트엔드는 라우터 A 에 현재 물려있는 robot/cam 의 구독자(Session) 목록을 읽고, 그 즉시 라우터 A 에서 또다시 공을 다중 분할시켜 10명의 구독자를 향해 핑(Ping) 을 퍼뜨리듯 흩뿌린다.
수십 개의 데이터가 실시간으로 빛처럼 번져나가는 이 시각적 쾌감은, 특정 라우터에 데이터가 꽂혔는데 거미줄이 가지치기 되지 못하고 사멸하는 “구독 불량(Routing Failure)” 포인트 구간을 육안으로 1초 만에 잡아내는 마법이 된다.
