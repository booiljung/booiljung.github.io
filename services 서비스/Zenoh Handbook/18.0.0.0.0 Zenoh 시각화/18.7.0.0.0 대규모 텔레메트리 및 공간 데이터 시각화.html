18.7 대규모 텔레메트리 및 공간 데이터 시각화

배달 로봇 100대가 강남역 사거리를 횡단하고, 농업용 드론 50대가 거대한 농장을 편대 비행한다.
이 거대한 야전 시스템을 관제하려면, 앞 장의 노드 차원(Topology) 을 넘어서서, 우리가 살고 있는 “물리적인 지구의 좌표계(GPS, GIS)” 위에 Zenoh 의 데이터를 핀셋처럼 꽂아 넣어야만 한다.

이 장에서는 Zenoh 프로토콜을 공간 정보 시스템(GIS) 에 직접 결합하여, 단순한 점의 이동을 넘어선 스마트 팩토리와 디지털 트윈, 드론 군집 비행의 지리적 동태(Geospatial Dynamics) 를 스크린에 융합하는 런북을 설파한다.

1. GPS 기반 이동체(Vehicle, Drone)의 지도 기반 실시간 궤적 시각화

이동체 관제의 핵심은 “현재 어디 있는가” 와 “직전에 어디를 지나왔는가(궤적)” 를 동시에 보는 것이다.

1.0.1 [인스펙션] 모션 트레일(Motion Trail) 렌더링 전술

1. Zenoh Payload 의 정규화 (Lat, Lng)
로봇마다 뱉어내는 GPS 데이터 규격이 다르면 안 된다. 모든 로봇은 Zenoh 의 robot/*/telemetry/gps 토픽에 [위도, 경도, 방향각] 형태의 압축된 바이너리나 JSON 을 초당 5회씩(5Hz) 흩뿌려야 한다.

2. 꼬리(Trail) 버퍼의 프론트엔드 관리
백엔드 DB 에서 로봇의 1시간 치 궤적을 계속 가져오면 망이 못 버틴다.
관제탑 브라우저는 딱 최신 GPS 좌표 하나씩만 Zenoh 로부터 실시간 구독(Subscribe) 받는다. 그리고 브라우저 메모리에 크기가 100 인 Trail Array (꼬리 배열) 를 만들어두고, 새 좌표가 들어올 때마다 앞에 넣고 맨 뒤좌표를 팝(Pop) 해버린다. 그러면 로봇 아이콘 뒤로 100개의 점으로 구성된 투명한 혜성 꼬리(Polyline) 가 지도 위에 자동으로 그려지며 로봇이 꿈틀대는 궤적을 공짜로 렌더링할 수 있다.

2. GeoJSON 및 Web GIS(Mapbox, Leaflet) 기반 공간 데이터 연동

구글 지도(Google Maps) 만 띄워놓는다고 끝이 아니다. 우리가 통제하는 공장의 금지 구역(Geofence) 이나 화재 지점 같은 우리만의 공간 빅데이터(GIS) 와 로봇이 오버레이 되어야 한다.

2.0.1 [Runbook] 지리 공간 결합(Geospatial Binding) 런북

오픈소스 기반인 Leaflet 이나 고성능 패러다임인 Mapbox GL JS 에 Zenoh 를 물린다.

1. GeoJSON 폴리곤 (정적 지형물)
관제사가 지도에 그어놓은 농장 경계선이나 드론 비행 금지선은 GeoJSON 형태의 벡터 그래픽이다. 이것은 정적이므로 초기 페이지 로딩 시 한 번만 그린다.

2. Mapbox Deck.gl 을 활용한 10,000 대 밀집 렌더링
Mapbox 의 가장 무서운 파트너인 Deck.gl 에 Zenoh 웹소켓 구독기를 끼워 넣는다.
Zenoh 에 들어온 수천 대의 드론 좌표([lng, lat, alt]) 배열을 Deck.gl 의 IconLayer 나 ScatterplotLayer 속성에 통으로 꽂아버리면, 브라우저의 GPU(WebGL) 가 미친 듯이 연산을 분산 처리하여 1만 대의 드론이 강남역에 새카맣게 몰려드는 개미 떼 같은 애니메이션을 60 프레임(FPS) 으로 아주 부드럽게 그려낸다. DOM(Div 태그) 으로 지도 마커를 찍는 Leaflet 따위로는 1,000 개만 붙어도 브라우저가 장렬히 폭사한다.

3. 드론 군집(Swarm) 비행 상태 모니터링 UI 구축

드론 군집 비행에서는 “위도, 경도” 만큼이나 “고도(Z축)” 와 “드론들 간의 충돌 회피(간격)” 지표가 화면을 지배한다.

3.0.1 [인스펙션] 3D 파사드(Facade) 관제탑 전술

일반적인 2D 지도를 꺾어서 3차원 기울기(Pitch) 를 줘야만 진정한 관제가 시작된다.

1. Z축 핀(Pin) 렌더링
드론이 하늘로 날아오르면, 지도 위에 붙은 아이콘도 Z축으로 공중에 떠야 한다.
이때 공중에 뜬 드론 아이콘 아래로 반투명한 수직 점선을 그려 바닥(지도 поверхность) 과 연결해주어, 현재 이 드론이 “산 위를 날고 있는지 빌딩 위를 날고 있는지” 지형(Terrain) 과의 원근감을 관제사에게 인지시켜야 한다.

2. 에러 확산(Vibration) 알럿 (군집 이탈)
드론 100 대 중 하나가 모터 이상으로 고도가 떨어지기 시작하면, 그 드론 노드의 아이콘을 빨갛게 색칠할 뿐만 아니라, 아이콘 전체를 미친 듯이 좌우로 흔드는 (CSS Animation: shake) 극단적인 상태 이상 표시를 가동해야 한다. 군집 비행에서 단 1대의 오류는 나머지 99 대를 격추하는 폭발물이 되기 때문이다.

4. 스마트 팩토리 내 IIoT 설비 상태의 디지털 트윈(Digital Twin) 시각화 기초

도요타나 테슬라의 기가 팩토리 관제탑 뒷면에는 팩토리 라인 전체를 본뜬 3D 비디오 게임 같은 화면이 돌아간다. 센서값(숫자) 과 3D 모델(물체) 의 완벽한 동기화, 이것이 디지털 트윈이다.

4.0.1 [Runbook] 공장 메타버스(Metaverse) 바인딩 런북

Three.js 화면 안에 얹어진 로봇팔 3D 모델(GLTF) 관절에 Zenoh 의 데이터 혈관을 꽂아 넣는다.

1. 메타 데이터-3D 뼈대(Bone) 바인딩
웹 디자이너가 공장 설비를 3D 로 렌더링해 뒀다 치자.
컨베이어 벨트에 있는 로봇 팔 3D 객체의 이름을 코드 상에서 model.getObjectByName('Joint_A') 로 찾는다.
그리고 Zenoh 구독 콜백 함수 안에 이 뼈대를 통째로 물려버린다.

// Zenoh에서 관절 각도가 넘어올 때마다 브라우저 속 3D 로봇팔도 꺾인다.
zenohSession.subscribe('factory/robot1/joint_a', (val) => {
    const angle = parseFloat(val.toString());
    robotArmBone.rotation.z = THREE.MathUtils.degToRad(angle);
});

2. IIoT 색상(Color) 지시등 투영
공장 용광로의 온도(Temperature) 센서 값이 1,000도를 돌파했다면?
단순히 숫자만 띄우지 마라. 쓰리디(3D) 모델 내 용광로 매트리얼(Material) 의 발광(Emissive) 강도를 센서 온도에 비례하게 팍팍 올려주어, 용광로 3D 모델 자체가 화면에서 붉은 용암처럼 스스로 불타오르게 시각적 압박감을 주어야 진짜 디지털 트윈이다.
