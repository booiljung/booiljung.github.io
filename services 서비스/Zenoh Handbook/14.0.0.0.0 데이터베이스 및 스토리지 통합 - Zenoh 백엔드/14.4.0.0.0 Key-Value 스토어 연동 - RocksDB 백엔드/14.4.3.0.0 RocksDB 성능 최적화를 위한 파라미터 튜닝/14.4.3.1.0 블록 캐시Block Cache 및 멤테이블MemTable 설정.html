14.4.3.1 블록 캐시(Block Cache) 및 멤테이블(MemTable) 설정

하드디스크 쓰기(Write) 는 죄악이다. 최대한 램(RAM) 에 버티고 모았다가 한 방에 쏴야 한다.

0.1 [Runbook] LSM Tree 강제 펌핑 전술

JSON 볼륨 설정의 하위 파라미터에 RocksDB 전용 튜닝 값을 주입한다.

1. 멤테이블 (MemTable = 쓰기 버퍼)

storages: {
  robot_rocks: {
    // ...
    write_buffer_size: 67108864, // 64MB: RAM에 쓸 데이터가 이만큼 찰 때까지 버텨라!
    max_write_buffer_number: 3   // 그런 통을 3개나 만들어 둬라!
  }
}

진동 센서가 1초에 만 번 Put 을 쏘더라도, RocksDB 는 64MB 가 찰 때까지 하드디스크를 단 한 번도 긁지 않는다. SD 카드 수명을 영원으로 수렴케 하는 궁극의 런북이다.

2. 블록 캐시 (Block Cache = 읽기 버퍼)
Get 쿼리가 미친 듯이 들어올 때 과거 데이터를 디스크에서 매번 꺼내오면 죽는다.

storages: {
  robot_rocks: {
    block_cache_capacity: 134217728 // 128MB: 한번 읽은 건 메모리에 128MB 까지 살려둬라!
  }
}

관제 서버에서 1초마다 대시보드를 갱신해도 엣지(로봇) 의 디스크 I/O 는 절대 흔들리지 않는다.
