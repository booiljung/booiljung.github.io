14.4.2 RocksDB 기반 Zenoh 라우터 구축 및 플러그인 적재

라즈베리 파이건, 거대한 클라우드건 설정 파일 한장이면 로컬 디스크가 거대한 DB 서버로 돌변한다.

0.1 [Runbook] 엔진 점화 전술

1. 폴더 소유권 확보
RocksDB는 파일 하나에 데이터를 다 구겨 넣지 않고 무수한 .sst 조각 파일을 디스크에 뿌린다. 무조건 빈 폴더를 하나 내어줘야 한다.

mkdir -p /var/lib/zenoh/rocksdb_store

2. 라우터 설정(Mount)

storages: {
  robot_rocks: {
    key_expr: "robot/telemetry/**",
    backend: "rocksdb",
    // RocksDB 만의 특수 파라미터 영역!
    dir: "/var/lib/zenoh/rocksdb_store", /* 필수 */
    create_if_missing: true              /* 엔진 자동 초기화 */
  }
}

이제 데몬을 켜고 z_put "robot/telemetry/gps" "LAT/LON" 을 쏘면, /var/lib/zenoh/rocksdb_store 안에 무수한 16진수 바이너리 조각들이 쏟아지기 시작한다. 오직 Zenoh 라우터만이 이진 파일들을 해석할 수 있는 유일한 자물쇠다.
