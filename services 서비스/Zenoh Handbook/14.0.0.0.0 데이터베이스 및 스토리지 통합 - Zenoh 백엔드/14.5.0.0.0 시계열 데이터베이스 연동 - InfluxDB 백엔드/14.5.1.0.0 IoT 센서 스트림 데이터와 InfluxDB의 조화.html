14.5.1 IoT 센서 스트림 데이터와 InfluxDB의 조화

왜 시계열 DB 를 써야만 하는가? RocksDB 나 MySQL 로는 안 되는가?

0.1 [인스펙션] 시간 축의 강제 정렬 전술

1. 일반 KV 스토어의 덮어쓰기 파괴
z_put "robot/temp" "25.0"
이걸 RocksDB 에 백 번 쏘면, 과거의 온도 값 99개는 영영 지워지고 마지막 1개만 남는다 (Overwrite).

2. 시계열 DB 의 선형 누적
똑같은 Put 을 InfluxDB 백엔드에 백 번 쏘면?
["10:01:00", 24.5], ["10:01:01", 25.0] ...
Zenoh 의 플러그인이 데이터가 도달하는 족족, 혹은 센서가 찍어 보낸 HLC 타임스탬프를 끄집어내어 “절대 지워지지 않는 시간의 로그” 로 차곡차곡 쌓아(Append-only) 준다.
로봇이 남긴 모든 궤적이 지워지지 않는 역사책에 기록되는 것이다.
