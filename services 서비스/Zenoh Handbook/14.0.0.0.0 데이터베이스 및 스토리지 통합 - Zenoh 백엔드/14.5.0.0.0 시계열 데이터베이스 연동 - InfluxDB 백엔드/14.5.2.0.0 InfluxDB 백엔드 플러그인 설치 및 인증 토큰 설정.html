14.5.2 InfluxDB 백엔드 플러그인 설치 및 인증 토큰 설정

InfluxDB 서버는 외부에 별도로 띄우고, Zenoh 라우터는 이 서버에 무지막지한 쓰기 API 를 꽂아 넣는다.

0.1 [Runbook] HTTP 기반 스트림 주입기 세팅

1. 플러그인 빌드 및 로드
Zenoh는 내장된 RocksDB와 달리 InfluxDB는 외부 네트워크 데몬이기에 별도의 브릿지가 필요하다.

# 플러그인 설치 (또는 바이너리 다운로드)
cargo install zenoh-backend-influxdb

2. 통신 브릿지 암호화 마운트
라우터를 켤 때 외부 InfluxDB 의 주소와 토큰(Token), 그리고 어느 버킷(Bucket) 에 부을 것인지를 명시해야 한다.

storages: {
  timeseries_db: {
    key_expr: "factory/telemetry/**",
    backend: "influxdb",
    // InfluxDB 연결 스펙
    url: "http://10.0.0.5:8086",
    org: "my_robotics_inc",
    bucket: "sensor_raw_data",
    token: "my-super-secret-auth-token"
  }
}

이제 라우터는 factory/telemetry/** 망을 도청하다가 패킷이 나타나면 곧바로 10.0.0.5 서버의 sensor_raw_data 버킷으로 HTTP POST 융단폭격을 개시한다.
