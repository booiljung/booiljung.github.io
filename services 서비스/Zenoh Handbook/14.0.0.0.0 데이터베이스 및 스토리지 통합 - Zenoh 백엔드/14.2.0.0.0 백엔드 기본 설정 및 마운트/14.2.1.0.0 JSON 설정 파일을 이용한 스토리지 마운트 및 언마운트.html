14.2.1 JSON 설정 파일을 이용한 스토리지 마운트 및 언마운트

라우터를 깨우기 전, zenohd.json5 라는 작전명세서(Configuration)를 짜야 한다.

[1. 플러그인 로드와 마운트 선언]
여러분이 Zenoh 공식 RocksDB 백엔드(zenoh-backend-rocksdb)를 설치했다 치자.

// zenohd.json5
{
  plugins: {
    // 1. 사용할 백엔드 엔진의 이름을 박는다.
    storage_rocksdb: {
      // 2. 이 엔진에 몇 개의 스토리지 볼륨을 할당할 것인가?
      volumes: [
        {
          id: "sensor_db",                 // 스토리지의 고유 식별자
          key_expr: "factory/sensor/**",   // 이 스토리지가 훔쳐먹을(?) 데이터의 범위
          // RocksDB 전용 설정들 (어느 경로에 저장할지)
          dir: "/var/lib/zenoh/sensor_db" 
        }
      ]
    }
  }
}

이 10줄 남짓한 JSON 파일 하나면 끝이다. 여러분은 코드를 컴파일할 필요가 전혀 없다.

[2. 라우터 시동과 마운트(Mount)]

$ zenohd -c zenohd.json5

라우터가 켜지면서 /usr/lib/zenoh/libzenoh_backend_rocksdb.so 동적 라이브러리를 찾아서 메모리에 욱여넣는다. 그 후 /var/lib/zenoh/sensor_db 경로에 로컬 RocksDB 인스턴스를 강제로 마운트시키고, 라우팅 테이블에 factory/sensor/** 로 향하는 모든 Pub 트래픽과 Query 트래픽을 이 스토리지로 리다이렉션(Redirection)하도록 징발한다.

라우터를 끄면 스토리지는 자연스럽게 언마운트(Unmount)되며 디스크는 안전하게 잠든다.
