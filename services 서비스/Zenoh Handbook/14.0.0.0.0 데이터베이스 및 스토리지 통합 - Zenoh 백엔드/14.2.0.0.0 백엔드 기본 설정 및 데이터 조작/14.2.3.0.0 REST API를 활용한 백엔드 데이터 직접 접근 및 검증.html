14.2.3 REST API를 활용한 백엔드 데이터 직접 접근 및 검증

웹 모니터링 대시보드(React/Vue) 를 만드는 개발자에게 “Zenoh C++ SDK 깔아서 z_get 치세요” 라고 할 수는 없다.
Zenoh 라우터는 훌륭한 HTTP 브릿지(REST API) 플러그인을 기본 보유하고 있다.

0.1 [Runbook] 웹 관제망과의 접합 전술

라우터 세팅에서 rest 플러그인을 활성화(zenohd --rest-http-port 8000) 시켰다. 이제 모든 CRUD 는 포스트맨(Postman) 이나 웹 브라우저(curl) 의 HTTP 메서드로 치환된다.

1. Read (HTTP GET)
웹 프론트엔드에서 스토리지에 데이터를 물어본다.

curl -X GET http://<router_ip>:8000/robot/sensors/temp

라우터가 이 HTTP GET 요청을 순수 Zenoh 의 z_get 패킷으로 번역하여 백엔드 DB 에 쿼리하고, 그 응답을 다시 브라우저에 JSON 으로 돌려준다.

2. Create/Update (HTTP PUT)

curl -X PUT http://<router_ip>:8000/robot/sensors/temp -d '27.5'

이 REST API 백도어가 존재하기 때문에, 아키텍트는 분산 DB 의 검증(Validation) 을 위해 굳이 클라이언트 컴파일 코드를 짤 필요 없이, 브라우저 주소창에 치는 것만으로도 수천 킬로미터 떨어져 있는 로봇의 DB 단 무결성을 즉석에서 디버깅해 낼 수 있다.
