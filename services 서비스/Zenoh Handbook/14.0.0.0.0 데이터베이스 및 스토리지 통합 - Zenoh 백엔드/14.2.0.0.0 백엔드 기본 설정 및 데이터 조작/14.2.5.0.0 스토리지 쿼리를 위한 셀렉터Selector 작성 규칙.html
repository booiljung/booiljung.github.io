14.2.5 스토리지 쿼리를 위한 셀렉터(Selector) 작성 규칙

스토리지가 집어먹은 하위 센서값 수백 개를 한 방에 쓸어 올려야(Read) 할 때, 코드를 어떻게 짜야 하는가?

0.1 [Runbook] 와일드카드 전광석화 추출 전술

Zenoh 의 쿼리 셀렉터는 SQL 의 SELECT * FROM tbl 을 대체하는 고속 계층형 슬라이서다.

1. 단일 노드 정밀 타격

z_get "factory/robot_1/sensor/temp"

DB 안에서 정확히 저 키에 해당하는 값 하나를 가져온다. (로컬 데이터 캐싱).

2. 하위 브랜치(자식) 전체 폭격 (*)

z_get "factory/robot_1/sensor/*"

스토리지 백엔드는 temp, humidity, pressure 등 sensor/ 바로 아래에 매달린 1계층 자식들만 뽑아서 복수 개의 Reply 배열로 던져준다.

3. 섭-트리 글로벌 폭격 (**)

z_get "factory/robot_1/**"

스토리지는 robot_1 하위에 숨겨진 센서, 모터, 카메라, 상태 로그 등 수천 개의 계층구조 트리 전체를 풀 스캔(Full-Scan) 쳐서 융단폭격으로 반환한다.
이 기능을 웹 관제(REST API) 에 열어둘 때에는 반드시 디스크 I/O 과부하(DB 성능 마비) 로 이어질 위험이 있으므로, 권한 설정(ACL) 으로 와일드카드 사용을 철저히 막아야 하는 생존 런북이다.
