14.3.3 FS 백엔드의 디렉터리 구조 및 파일 저장 규칙

와일드카드 질의가 들어오면 스토리지 엔진은 리눅스 하드디스크를 어떤 형식으로 긁어오는가?

0.1 [인스펙션] 하드디스크 I/O 매핑 룰

1. URI 기반 폴더 트리 생성기
z_put "factory/zone_1/sensor_a" "24.5"
이 명령 하나는 디스크 상에 factory/zone_1/ 이라는 중첩 폴더를 찍어낸다. 만약 트리가 100 뎁스라면 리눅스 폴더도 100 뎁스가 만들어진다.

2. 덤프 타겟으로서의 텍스트 파일
Zenoh 프로토콜의 Payload 값("24.5") 은 sensor_a 라는 이름의 파일 본문으로 저장된다. 이때 확장자는 따로 붙지 않는다.

3. 와일드카드 폭격 대응 (Tree Traversal)
만약 관제사가 z_get "factory/**" 권한으로 쿼리를 날린다면?
FS 플러그인은 리눅스의 find /var/lib/zenoh/my_storage/factory -type f 명령어를 돌려 모든 파일시스템 트리를 긁어온 뒤, 그걸 몽땅 JSON 배열로 직렬화하여 클라이언트에 던진다.

  • 경고: 매우 직관적이지만, 파일 갯수가 10만 개가 넘어가는 순간 운영체제의 하드디스크 I/O 블로킹 병목에 걸려 DB 로서의 성능을 완전히 상실한다. (이후 설명할 RocksDB 가 필요한 이유다).
