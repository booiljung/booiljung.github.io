14.10.2 백엔드 플러그인 전용 지표(Metrics) 수집 및 시각화 연동

라우터 겉면에서 아무리 패킷 수를 세어봤자, 스토리지 내부의 상태는 보이지 않는다.

0.1 [인스펙션] Prometheus 심장 박동 모니터링 전술

Zenoh 라우터에 내장된 원격 측정(Telemetry) 포트를 개방하여 관제탑으로 모든 스탯을 전송한다.

1. 프로메테우스(Prometheus) 종단점(Endpoint) 해제

// zenohd.json5
plugins: {
  // 모니터링 플러그인 켬!
  prometheus: {
    port: 8001
  }
}

2. 스토리지 심층 지표(Deep Metric) 가시화
기본 라우팅 스탯뿐 아니라 스토리지의 입출력 건수까지 찍혀 나온다.

  • zenohd_storage_put_count_total{volume="robot_rocks"}: 이 볼륨에 몇 건이나 Insert 되었는가?
  • zenohd_storage_get_latency_ms{volume="robot_rocks"}: 쿼리를 때렸을 때 디스크에서 꺼내오는 데 걸린 내부 소요 밀리초 단위 시간.

이 지표를 Grafana 대시보드에 연동해 두면, “어느 순간 쿼리 리턴 속도(Latency)가 1밀리초에서 갑자기 500밀리초로 역주행하기 시작하는 시점“을 포착할 수 있다. 그것이 바로 디스크 조각 모음 기능(Compaction)이 발동하여 시스템 I/O가 정체되는 치명적인 순간이다.
