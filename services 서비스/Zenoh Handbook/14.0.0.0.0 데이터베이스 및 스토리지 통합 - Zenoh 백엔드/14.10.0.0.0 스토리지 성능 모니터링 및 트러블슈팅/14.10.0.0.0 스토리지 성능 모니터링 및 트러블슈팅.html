14.10 스토리지 성능 모니터링 및 트러블슈팅

Zenoh의 통신 오버헤드가 제로(0)에 가깝다 한들, 데이터를 받아먹는 디스크(DB)가 체하면 전체 파이프라인이 즉격으로 박살 난다.

모니터링 없는 스토리지 운영은 눈을 가리고 시속 200km로 달리는 것과 같다. RocksDB가 디스크를 얼마나 긁고 있는지, InfluxDB에 데이터를 밀어 넣는 큐(Queue)가 꽉 차서 터지기 직전인지 알아야 한다.
이 장에서는 백엔드 플러그인이 내뿜는 비명 소리(Metrics)를 수집하여, 엣지(Edge)와 클라우드(Cloud)의 디스크 병목을 조기에 색출하고 멸망을 막아내는 트러블슈팅 런북(Troubleshooting Runbook)을 마스터한다.
