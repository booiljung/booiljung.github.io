14.8.3 플러그인 컴파일 및 공유 라이브러리(.so, .dll) 동적 로딩 매커니즘

Rust 코드를 다 짰다면, 이제 이것을 쇳덩어리(Binary)로 구워서 라우터에 먹여야 한다(Feeding).

0.1 [Runbook] 핫-로드(Hot-Load) 외과 수술 전술

1. 릴리즈(Release) 빌드 사수
로보틱스 런타임에서 디버그(Debug) 모드 플러그인은 성능을 50배 하락시키는 바이러스다. 무조건 릴리즈 모드로 강제 컴파일하라.

cargo build --release

빌드가 끝나면 target/release/ 폴더 아래에 libzenoh_backend_mycustom.so (윈도우는 .dll) 라는 파일이 떨어진다.

2. 라우터 주입 규칙 (Injection)
Zenoh 라우터는 기동될 때 딱 2가지 장소에서만 플러그인을 찾는다.

  • 자신(zenohd)이 켜진 바로 그 폴더.
  • 사용자가 지정한 특별한 라이브러리 경로 폴더(예: ~/.zenoh/lib).

만든 .so 파일을 폴더에 던져놓고 라우터를 켜라.

zenohd -c my_custom_config.json5

라우터가 부팅되면서 “Oh, I found mycustom backend!” 하는 로그를 띄우면 수술 성공이다. 시스템 본체 코드(Zenoh Core)를 단 한 줄도 재컴파일하지 않고도, 커스텀 DB 드라이버를 런타임(Runtime)에 강제 주입해 버린 것이다.
