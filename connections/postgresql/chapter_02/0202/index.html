PostgreSQL 16을 설치하기 위해서는 공식 웹사이트에서 직접 소스 코드를 내려받아 빌드하는 방법과, 운영 체제별 패키지 관리자를 사용하여 손쉽게 설치하는 방법이 존재한다. 설치 과정에 앞서 원하는 방식에 따라 적절한 패키지를 선택하는 것이 중요하며, 운영 환경에 맞춘 안정성과 편의성을 확보하기 위해 각 배포판에서 제공하는 공식 리포지토리를 적극 활용하는 것이 권장된다.

공식 웹사이트 다운로드¶

PostgreSQL 공식 웹사이트에서는 소스 코드와 운영 체제별로 사전에 빌드된 바이너리를 제공한다. 소스 코드를 빌드하면 원하는 옵션을 세밀하게 조정할 수 있지만 컴파일 환경 및 의존 관계를 준비해야 한다. 반면 사전 빌드된 바이너리 패키지는 설치 절차가 간편하고 의존성도 자동으로 처리되는 장점이 있다.

원본 소스 코드는 공식 홈페이지나 Git 저장소에서 구할 수 있다. 최신 버전인 16을 선택하거나, 안정성을 위해 특정 마이너 버전을 명시적으로 지정할 수도 있다. 소스 빌드를 고려한다면 시스템에 C 컴파일러(예: GCC)와 GNU make, 그리고 zlib, OpenSSL 등 필요한 라이브러리가 갖추어져 있어야 한다.

리눅스 환경에서의 패키지 설치¶

리눅스 배포판마다 패키지 관리자가 상이하므로, 공식 PostgreSQL 리포지토리를 추가하거나 배포판에서 제공하는 패키지를 이용한다. 배포판 내장 리포지토리를 사용하면 버전이 다소 늦게 업데이트될 수 있으나 안정성이 검증된 경우가 많다. 최신 기능을 활용하기 위해서는 PostgreSQL 공식 리포지토리를 추가하여 필요한 패키지를 설치하는 것이 바람직하다.

Debian 기반 시스템(Ubuntu 포함)¶

공식 리포지토리를 통해 설치할 때에는 GPG 키를 추가하고 sources.list 또는 별도 리포지토리 파일을 생성한 뒤 apt-get(또는 apt) 명령어를 이용한다. 예시로 Ubuntu 22.04에서 PostgreSQL 16을 설치하기 위해서는 다음 과정을 거친다.

sudo apt-get update
sudo apt-get install wget gnupg lsb-release
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -
echo "deb http://apt.postgresql.org/pub/repos/apt/ $(lsb_release -cs)-pgdg main" | sudo tee /etc/apt/sources.list.d/pgdg.list
sudo apt-get update
sudo apt-get install postgresql-16

설치가 완료되면 PostgreSQL 서버 데몬이 자동으로 실행되며, 포트가 충돌되지 않는 이상 시스템 재부팅 시에도 자동으로 실행된다.

Red Hat 기반 시스템(CentOS, Fedora 등)¶

Red Hat 기반 배포판에서는 기본적으로 yum 또는 dnf 패키지 관리자를 사용한다. PostgreSQL 글로벌 개발 그룹에서 제공하는 공식 리포지토리를 추가한 후 설치할 수 있다. 예시로 CentOS 8 환경에서 PostgreSQL 16을 설치하는 과정은 다음과 같이 진행한다.

sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-8-x86_64/pgdg-redhat-repo-latest.noarch.rpm
sudo dnf -qy module disable postgresql
sudo dnf install -y postgresql16-server postgresql16

설치 후 PostgreSQL 데이터베이스 초기화를 다음과 같이 진행할 수 있다.

sudo /usr/pgsql-16/bin/postgresql-16-setup initdb
sudo systemctl enable postgresql-16
sudo systemctl start postgresql-16

SUSE Linux¶

SUSE의 패키지 관리 툴인 zypper를 통해 설치한다. SUSE Enterprise Linux 또는 openSUSE에서 공식 PostgreSQL 리포지토리를 이용하면 최신 버전을 간편하게 가져올 수 있다. 리포지토리를 추가한 다음 postgresql16 관련 패키지를 zypper로 설치하면 된다.

기타 리눅스 배포판¶

대부분의 리눅스 배포판은 자체 리포지토리나 공식 PostgreSQL 리포지토리를 통해 패키지를 제공한다. 배포판 문서를 참조하여 리포지토리를 추가한 뒤 postgresql-16 패키지를 설치하고 데이터 디렉터리를 초기화한다.

macOS 환경에서의 패키지 설치¶

macOS 환경에서는 Homebrew 또는 MacPorts 등을 통해 PostgreSQL 16을 간편하게 설치할 수 있다. Homebrew를 사용한다면 다음과 같이 명령어를 입력한다.

brew update
brew install postgresql@16

이후 PostgreSQL 서버를 수동으로 실행하거나, macOS의 launchctl을 통해 자동 시작하도록 등록할 수 있다. Homebrew가 제공하는 서비스를 사용하고자 한다면, PostgreSQL을 자동으로 시작하도록 설정이 가능하다.

Windows 환경에서의 패키지 설치¶

Windows 환경에서는 공식 홈페이지에서 제공하는 Windows용 인스톨러를 통해 PostgreSQL 16을 설치할 수 있다. 인스톨러 실행 시 그래픽 마법사가 나타나며, 설치 경로와 데이터 디렉터리, 관리자 계정 암호 등을 설정하게 된다. Visual C++ 재배포 가능 패키지 등을 자동으로 설치해주기 때문에 별도의 컴파일 작업 없이 편리하게 PostgreSQL 16 환경을 구축할 수 있다.

기본 설정이 완료된 뒤에는 PostgreSQL 서비스가 Windows 서비스 목록에 등록되며, 시스템 시작 시 자동으로 구동되도록 설정할 수 있다.

설치 후 검증¶

설치가 끝난 뒤에는 명령줄이나 GUI 클라이언트를 사용하여 정상 동작을 확인할 수 있다. 터미널 환경에서 psql 클라이언트를 실행해 본 뒤 데이터베이스에 접속할 수 있는지 점검하거나, Windows 환경에서는 SQL Shell(psql) 아이콘을 통해 접속 테스트를 시도할 수 있다.

모든 과정을 마친 후에는 설치된 PostgreSQL 16 버전을 실제로 사용하기 전에 필요한 의존 패키지와 호환성이 올바른지 확인하고, 데이터베이스 인코딩이나 locale 설정 등을 세밀하게 조정함으로써 운영 환경에 맞게 최적화하는 것이 바람직하다.
