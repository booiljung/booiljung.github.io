Keycloak은 다양한 언어로 사용자 인터페이스와 이메일 등을 제공할 수 있도록 국제화(i18n) 기능을 지원한다. 이를 통해 로그인 화면, 계정 관리 화면, 이메일 템플릿 등에서 사용자별 선호 언어를 적용하고, 필요에 따라 추가적인 언어팩을 배포할 수 있다. 다음에서는 국제화를 활성화하고 설정하는 전반적인 과정을 엄밀하게 살펴본다.

국제화 설정 개요¶

Keycloak의 국제화 기능은 주로 ‘Realm Settings’의 ‘Login’ 탭을 통해 구성한다. 관리 콘솔에서 국제화를 활성화한 뒤 지원할 언어들을 선택하면, 로그인 페이지 등 Keycloak에서 제공하는 여러 화면에서 다국어가 적용된다. 기본적으로 영문 메시지가 제공되며, 필요하다면 메시지 번들을 수정하거나 새로 만들어 특정 언어의 메시지를 맞춤화할 수도 있다.

국제화 기능 활성화¶

  1. Keycloak 관리자 콘솔에 접속하여 원하는 Realm을 선택한다.
  2. ‘Realm Settings’ 메뉴에서 ‘Login’ 탭을 클릭한다.
  3. ‘Internationalization’ 섹션에서 ‘Enable Internationalization’ 옵션을 활성화한다.
  4. ‘Supported Locales’ 목록에서 해당 Realm이 지원할 언어들을 선택한다.
  5. ‘Default Locale’을 지정하면, 사용자가 별도의 언어를 지정하지 않았을 때 해당 언어가 기본 언어로 사용된다.

설정을 저장하면, 로그인 화면 우측 상단(또는 하단)에서 언어 선택 드롭다운이 노출되어 사용자가 즉시 언어를 바꿀 수 있게 된다.

사용자 인터페이스 국제화¶

Keycloak은 로그인 화면, 계정 관리 콘솔, 이메일 템플릿 등 다양한 사용자 인터페이스 요소에 대해 국제화를 지원한다.

  • 로그인 화면: 사용자가 직접 언어를 선택할 수 있으며, 지원 목록에 추가된 언어만 표시된다.
  • 계정 관리 콘솔: 사용자가 로그인 후 계정 설정 화면으로 이동하면, 브라우저 언어 또는 Realm에서 지정된 기본 언어를 기준으로 화면이 표시된다.
  • 이메일 템플릿: 비밀번호 재설정이나 인증 알림 메일 등이 사용자의 설정 언어에 맞추어 전송된다.

메시지 번들 수정 및 확장¶

Keycloak의 테마 시스템을 활용하면, 기본 제공되는 메시지 번들을 수정하거나 신규 언어에 대한 번들을 추가할 수 있다.

  1. Keycloak 서버의 테마 폴더(themes) 내에 커스텀 테마 디렉터리를 생성한다.
  2. 테마 종류(예: login, email, account)에 맞춰 messages_<locale>.properties 파일을 준비한다.
  3. 필요한 키와 메시지를 정의하고, 원하는 언어로 번역된 텍스트를 작성한다.
  4. Keycloak 관리자 콘솔의 ‘Realm Settings’ → ‘Themes’ 탭에서 커스텀 테마를 지정하면, 그 테마에서 작성한 메시지 파일이 우선적으로 적용된다.

다국어 환경에서의 유의사항¶

  • 브라우저 언어 우선순위: 사용자가 언어 선택을 명시하지 않은 경우, 브라우저 요청 헤더에 포함된 언어 정보를 기준으로 Keycloak이 UI 언어를 결정한다.
  • 테마 캐싱: Keycloak은 테마 리소스를 캐싱한다. 메시지 파일이 수정된 뒤에는 서버 재시작이 필요할 수 있다.
  • 동적 언어 추가: Realm 설정에서 언제든 추가 언어를 활성화할 수 있다. 단, 이미 로그인 세션이 진행 중인 사용자는 바로 반영되지 않을 수 있으므로, 테스트 환경에서 충분히 검증 후 운영 환경에 반영하는 것이 좋다.
  • 일관성 유지: 이메일, 로그인 화면, 계정 관리 콘솔 간에 메시지 키가 불일치하면 언어별 일부 문구만 영문으로 표시될 수 있다. 따라서 메시지 번들을 수정할 때는 각 테마 간 키를 동일하게 유지해야 한다.
  • 국제화를 활성화하려면 관리자 콘솔에서 ‘Enable Internationalization’을 활성화하고 지원할 언어와 기본 언어를 지정한다.
  • 로그인 화면, 계정 콘솔, 이메일 등 Keycloak 전반에 걸쳐 언어가 적용되며, 사용자는 로그인 화면 등에서 직접 언어를 선택할 수 있다.
  • Keycloak 테마 시스템을 통해 메시지 번들을 수정하거나 신규 언어 파일을 추가할 수 있다.
  • 브라우저 언어, 캐싱, 메시지 키 일관성 등에 주의하여 다국어 환경을 안정적으로 운영해야 한다.

국제화 설정은 다양한 사용자에게 편의성을 제공할 수 있는 중요한 기능이므로, 실제 운영 환경에서는 지원할 언어를 신중히 결정하고 테스트를 거쳐 반영하는 것이 바람직하다.
