HTTP란 무엇인가¶

인터넷 기반의 통신 프로토콜¶

HTTP는 웹 상에서 데이터를 주고받기 위한 애플리케이션 계층 프로토콜이다. 월드 와이드 웹(WWW)의 핵심적 요소로서, 클라이언트와 서버 간에 텍스트, 이미지, 동영상 등 다양한 형태의 데이터를 전송하는 데 사용된다. TCP/IP 위에서 동작하며, 주로 포트 80(HTTP)이나 443(HTTPS) 등과 결합되어 작동한다. 단순 텍스트 구조로 통신 내용을 기술하기 때문에 사람이 직접 읽고 이해하기도 용이하다는 특징이 있다.

HTTP는 엄격하고 표준화된 형태의 메시지 교환 규칙을 정의한다. 클라이언트가 특정 리소스를 요청하면, 서버는 그에 대한 적절한 응답을 보내는 형식이다. 이러한 구조를 통해 전 세계 어디서든 브라우저나 각종 애플리케이션을 통해 손쉽게 정보를 교환할 수 있게 된다.

요청과 응답의 구조¶

HTTP는 요청(request)과 응답(response)으로 이뤄진 교환 과정을 통해 동작한다. 클라이언트는 메서드, 경로, 프로토콜 버전을 담은 요청라인과 함께 각종 헤더를 전송하고, 서버는 요청의 내용을 해석하여 결과를 돌려준다. 메시지 형식은 엄격히 정해져 있어 협의된 규칙에 맞춰야 한다.

요청 메시지의 예시는 다음과 같은 형태를 가진다.

GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: MyBrowser/1.0
Accept: text/html

메서드(GET, POST, PUT, DELETE 등)로 무엇을 할지 정의하고, 요청 대상 경로(index.html)를 지정하며, 마지막으로 프로토콜 버전(HTTP/1.1 등)을 명시한다. 헤더들은 요청에 대한 세부 정보를 담는다.

응답 메시지는 상태 줄(status line)과 헤더들, 그리고 선택적인 본문(body)으로 구성된다.

HTTP/1.1 200 OK
Date: Tue, 23 Jan 2025 10:00:00 GMT
Server: ExampleServer/2.0
Content-Type: text/html

<html>
  <head><title>Example</title></head>
  <body>Hello World</body>
</html>

상태 줄에는 응답 코드(200, 404, 500 등)와 메시지가 포함된다. 헤더에는 응답 내용을 어떻게 해석할지에 대한 세부 정보가 담긴다. 본문에는 실제 HTML 문서나 이미지, JSON 등 다양한 형식의 데이터가 들어갈 수 있다.

스테이트리스 특성¶

HTTP는 기본적으로 스테이트리스(stateless)한 프로토콜로 설계되었다. 클라이언트와 서버 간 연결이 끊어지면 이전에 어떤 요청이 있었는지 기억하지 않는 구조다. 같은 클라이언트가 여러 번 요청하더라도 서버는 이를 전혀 별개의 요청으로 취급한다.

이런 특성 때문에 HTTP는 간단하고 유연하지만, 웹 서비스 개발 시에는 사용자 상태를 관리하기 위해 별도의 장치가 필요하게 된다. 쿠키, 세션, 토큰 등이 이러한 상태 관리를 보완하기 위한 도구로 사용된다. 이를 통해 인증이나 장바구니 같은 기능을 구현할 수 있다.

한편, HTTP는 초기 디자인과 달리 지속 연결(persistent connection)을 도입해 연결을 재활용하도록 개선되었다. HTTP/1.1 이후로는 Keep-Alive 헤더를 통해 여러 요청을 하나의 연결로 처리할 수 있게 되어 성능이 향상되었다.

발전 과정¶

HTTP는 태생적으로 간단했지만, 웹의 발전과 함께 요구 사항이 늘어감에 따라 여러 버전이 등장했다. 초기의 HTTP/0.9에서는 단순한 GET 메서드만 지원했고, 헤더 개념도 없었다. 이후 HTTP/1.0과 HTTP/1.1을 통해 헤더와 메서드가 확장되었고, 지속 연결과 같은 성능 개선도 이뤄졌다.

현대에 이르러서는 HTTP/2와 HTTP/3 등 더 나은 속도와 보안, 효율성을 지향하는 프로토콜이 널리 쓰이고 있다. HTTP/2는 멀티플렉싱과 헤더 압축을 도입해 전송 효율을 높였고, HTTP/3는 QUIC 기반으로 더욱 빠른 연결 설정과 전송 방식을 제공한다.

핵심 기능과 의의¶

HTTP는 단순함과 확장성이 공존하는 프로토콜로, 웹 뿐 아니라 다양한 서비스 간 데이터 교환에도 활용되고 있다. RESTful API에서도 근간으로 사용되어 애플리케이션 간 통신을 표준화하고, JSON, XML 등 구조화된 포맷을 교환하는 기반이 된다.

웹의 기초를 이루는 HTTP는 오늘날에도 지속적으로 발전하고 있다. 최신 보안 표준인 TLS를 통해 HTTPS가 기본으로 사용되는 추세이며, 웹 기술 전반이 더욱 복잡하고 대규모화되는 환경에서도 HTTP는 안정적이고 표준화된 통신을 보장한다. 이러한 점에서 HTTP는 단순한 문서 전송 방식 이상의 의미를 지니며, 인터넷 에코시스템을 떠받드는 핵심 프로토콜로 간주된다.
