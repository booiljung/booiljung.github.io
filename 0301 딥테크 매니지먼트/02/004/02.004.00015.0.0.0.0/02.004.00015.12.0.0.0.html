15.12 프로젝트 착수(Kick-off) 단계의 역할 분담과 책임 할당 매트릭스(RACI)

1. 프로젝트 착수 단계의 의의

프로젝트 착수(Kick-off)는 계약 체결 후 프로젝트의 공식적 시작을 선언하고, 프로젝트 팀의 역할·책임·소통 체계를 확립하는 단계이다. PMBOK Guide에 따르면, 프로젝트 착수 단계에서 프로젝트 헌장(Project Charter)이 승인되고, 핵심 이해관계자가 식별되며, 프로젝트 관리자가 공식적으로 임명된다.

딥테크 B2B 프로젝트의 착수 단계에서 기술 담당, 과제 관리자, 그리고 영업 담당의 역할 분담을 명확히 하는 것은 이후 프로젝트 실행의 효율성을 결정짓는 핵심 활동이다.

2. 착수 회의(Kick-off Meeting)의 구성

착수 회의는 내부 착수 회의와 고객 착수 회의로 구분된다. 내부 착수 회의에서 과제 관리자는 프로젝트 계획, 일정, 자원 배분, 그리고 리스크를 팀에 공유한다. 기술 담당은 기술적 접근 방법, 아키텍처 방향, 그리고 기술적 리스크를 설명한다. 영업 담당은 고객의 기대, 상업적 조건, 그리고 고객 조직의 특성을 공유한다.

고객 착수 회의에서 과제 관리자는 프로젝트 관리 체계, 소통 계획, 그리고 보고 체계를 발표한다. 기술 담당은 기술 솔루션의 개요와 구현 접근 방법을 설명한다. 영업 담당은 양사 간의 관계 관리 채널과 에스컬레이션 경로를 확인한다.

3. RACI 매트릭스의 수립

RACI 매트릭스는 프로젝트의 주요 활동에 대한 역할별 책임 수준을 명시적으로 정의하는 도구이다. 프로젝트 착수 시 수립되는 RACI 매트릭스의 주요 항목은 다음과 같다.
