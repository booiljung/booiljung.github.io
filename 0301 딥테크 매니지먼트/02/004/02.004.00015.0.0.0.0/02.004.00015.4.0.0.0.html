15.4 세 직군 간 기능적 교차 영역의 식별과 분류

1. 기능적 교차의 개념

기능적 교차(Functional Intersection)는 둘 이상의 기능 조직이 동일한 업무 영역이나 의사결정에 관여하는 지점을 의미한다. 기술 담당, 과제 관리자, 그리고 영업 담당의 세 직군은 각각 고유한 책임 영역을 보유하면서도, 제품 개발과 시장 전달의 과정에서 불가피하게 책임과 활동이 중첩되는 교차 영역이 존재한다.

Lawrence and Lorsch(1967)의 분화-통합(Differentiation-Integration) 이론에 따르면, 조직의 기능적 분화 정도가 높을수록 기능 간 통합의 필요성이 증대되며, 통합이 효과적으로 이루어지지 않으면 조직의 성과가 저하된다. 기능적 교차 영역의 명확한 식별과 분류는 이러한 통합을 체계적으로 관리하기 위한 전제 조건이다.

2. 교차 영역의 체계적 분류

2.1 기술-영업 교차 영역

기술 담당과 영업 담당 간의 기능적 교차 영역은 다음과 같다.

사전 판매 기술 지원(Pre-sales Technical Support)이다. 고객의 기술적 요구를 분석하고, 자사 기술의 적합성을 평가하며, 기술 시연과 개념 증명(PoC)을 수행하는 영역이다. 기술 담당은 기술적 전문성을, 영업 담당은 고객 관계와 사업적 맥락을 각각 기여한다.

고객 맞춤 요구의 평가(Custom Requirements Assessment)이다. 고객의 맞춤 요구가 기술적으로 실현 가능한지, 그리고 기존 제품 로드맵과 정합적인지를 공동으로 평가하는 영역이다.

기술 사양의 고객 전달(Technical Specification Communication)이다. 제품의 기술 사양을 고객이 이해할 수 있는 형태로 전환하여 전달하는 영역이다. 기술 담당이 사양의 정확성을, 영업 담당이 소통의 적절성을 각각 책임진다.

고객 기술 지원(Customer Technical Support)이다. 제품 도입 이후 고객의 기술적 문제를 해결하는 영역이다. 기술 담당이 문제의 진단과 해결을, 영업 담당이 고객 관계의 유지를 각각 담당한다.

2.2 기술-과제관리 교차 영역

기술 담당과 과제 관리자 간의 기능적 교차 영역은 다음과 같다.

노력 추정(Effort Estimation)이다. 기술 과업의 소요 노력을 추정하는 영역이다. 기술 담당이 기술적 복잡성에 기반한 추정을 수행하고, 과제 관리자가 이를 일정 계획에 통합한다.

리스크 식별과 관리(Risk Identification and Management)이다. 기술적 리스크의 식별은 기술 담당의 전문적 판단에 의존하고, 리스크의 체계적 관리(등록, 추적, 대응)는 과제 관리자의 프로세스에 의하여 수행된다.

범위 변경 관리(Scope Change Management)이다. 기술적 발견이나 요구 변경에 의한 범위 변경의 기술적 영향을 기술 담당이 분석하고, 과제 관리자가 일정·비용에의 영향을 관리한다.

품질 관리(Quality Management)이다. 기술적 품질(코드 품질, 설계 품질, 시험 커버리지)은 기술 담당이 직접 관리하고, 품질 프로세스의 운영(검토 절차, 게이트 기준)은 과제 관리자가 관리한다.

2.3 영업-과제관리 교차 영역

영업 담당과 과제 관리자 간의 기능적 교차 영역은 다음과 같다.

납기 관리(Delivery Management)이다. 영업 담당이 고객에게 약속한 납기와 과제 관리자가 관리하는 프로젝트 일정 간의 정합성을 확보하는 영역이다.

고객 프로젝트의 상태 보고(Project Status Reporting to Customer)이다. 고객에게 프로젝트의 진행 상황을 전달하는 영역이다. 과제 관리자가 내부 상태 정보를 제공하고, 영업 담당이 고객에게 적절한 형태로 전달한다.

우선 고객에 대한 자원 배분(Priority Customer Resource Allocation)이다. 전략적으로 중요한 고객의 프로젝트에 대한 우선적 자원 배분을 영업 담당이 요청하고, 과제 관리자가 전사적 자원 제약 내에서 이를 조율한다.

2.4 삼자 공동 교차 영역

세 직군이 모두 관여하는 교차 영역은 다음과 같다.

제품 출시 관리(Product Launch Management)이다. 제품의 시장 출시에서 기술 담당(제품의 기술적 준비), 과제 관리자(출시 일정과 체크리스트 관리), 그리고 영업 담당(고객 전달과 교육)이 공동으로 참여한다.

개념 증명(PoC) 프로젝트 관리이다. 고객의 개념 증명 요구에 대하여 기술 담당(기술적 구현), 과제 관리자(PoC 일정과 자원), 그리고 영업 담당(고객 기대 관리)이 공동으로 대응한다.

인증·규제 대응이다. 제품의 인증과 규제 준수에서 기술 담당(기술 문서와 시험), 과제 관리자(인증 일정), 그리고 영업 담당(인증 상태의 고객 전달)이 각각의 역할을 수행한다.
