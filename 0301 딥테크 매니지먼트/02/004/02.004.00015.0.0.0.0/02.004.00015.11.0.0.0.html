15.11 과업 지시서(SOW) 및 요구사항 명세서 검토에서의 삼자 협업 구조

1. 과업 지시서(SOW)의 개념과 역할

과업 지시서(Statement of Work, SOW)는 프로젝트에서 수행되어야 할 작업의 범위, 산출물, 일정, 그리고 인수 기준을 명시하는 공식 계약 문서이다. 딥테크 B2B 프로젝트에서 SOW는 고객과 기업 간의 기술적·상업적 합의의 핵심 근거이며, 프로젝트 실행과 분쟁 해결의 법적 기반이 된다.

2. 삼자 협업의 구조

SOW의 작성은 기술 담당, 과제 관리자, 그리고 영업 담당의 삼자 협업을 필요로 한다. 영업 담당은 고객과의 상업적 조건 협상과 고객 기대의 전달을 담당한다. 기술 담당은 기술적 범위의 정확한 정의, 산출물의 기술적 사양 명세, 기술적 인수 기준의 설정을 담당한다. 과제 관리자는 SOW 문서의 전체 구조 관리, 프로젝트 일정과 마일스톤의 수립, 양측의 역할과 책임의 명시를 담당한다.

요구사항 명세서의 검토에서 기술 담당은 기술적 정확성과 실현 가능성을, 과제 관리자는 추정 가능성과 범위의 명확성을, 영업 담당은 고객 기대와의 정합성을 각각 검토한다. SOW의 최종 확정은 삼자의 승인을 거쳐야 하며, 공동 검토 회의와 검토 체크리스트의 운영을 통하여 검토의 완전성을 보장한다.

딥테크 기업에서는 기술적 불확실성의 명시적 반영, 지식재산권 조항의 기술적 검토, 그리고 인증 요건의 반영이 SOW의 특수한 고려 사항이다.
