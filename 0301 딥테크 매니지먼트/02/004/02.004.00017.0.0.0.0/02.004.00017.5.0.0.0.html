17.5 매트릭스 조직 구조에서의 이중 보고 체계와 권한 충돌 관리

1. 매트릭스 조직의 구조

매트릭스 조직(Matrix Organization)은 기능별 조직(Functional Organization)과 프로젝트/제품별 조직(Project/Product Organization)을 결합한 하이브리드 구조이다. Galbraith(1971)에 따르면, 매트릭스 구조는 기능별 전문성의 유지와 프로젝트 중심의 통합을 동시에 추구하기 위한 조직 설계이다.

매트릭스 조직에서 각 구성원은 기능 관리자(Functional Manager)와 프로젝트/제품 관리자(Project/Product Manager)라는 이중 보고 라인(Dual Reporting Line)을 가진다. 기능 관리자는 기술적 전문성의 개발, 경력 관리, 성과 평가, 그리고 자원 배분을 담당한다. 프로젝트 관리자는 프로젝트의 일정, 범위, 품질, 그리고 팀 내 업무 조율을 담당한다.

2. 이중 보고 체계의 유형

Davis and Lawrence(1977)는 매트릭스의 유형을 기능 매트릭스(Functional Matrix, 기능 관리자의 권한이 우세), 균형 매트릭스(Balanced Matrix, 양자의 권한이 동등), 그리고 프로젝트 매트릭스(Project Matrix, 프로젝트 관리자의 권한이 우세)로 분류하였다.

3. 권한 충돌의 관리

매트릭스 조직에서 기능 관리자와 프로젝트 관리자 간의 권한 충돌은 구조적으로 불가피하다. 자원 배분의 충돌(기능 부서의 자원 요구와 프로젝트의 자원 요구 간의 상충), 우선순위의 충돌(기능적 전문성 개발과 프로젝트 납기 간의 상충), 그리고 성과 평가의 충돌(기능 관리자와 프로젝트 관리자가 상이한 성과 기준을 적용)이 대표적이다.

권한 충돌의 관리를 위한 메커니즘은 역할과 권한의 명시적 문서화이다. 기능 관리자와 프로젝트 관리자 각각의 의사결정 영역을 사전에 명확히 정의한다. 정기적 조율 회의이다. 양 관리자가 정기적으로 자원 배분과 우선순위를 조율한다. 에스컬레이션 체계의 확립이다. 양 관리자 간 합의가 이루어지지 않는 경우의 상위 의사결정 경로를 사전에 정의한다. 공동 성과 평가의 운영이다. 양 관리자의 평가를 통합하여 구성원에 대한 균형 잡힌 성과 평가를 수행한다.
