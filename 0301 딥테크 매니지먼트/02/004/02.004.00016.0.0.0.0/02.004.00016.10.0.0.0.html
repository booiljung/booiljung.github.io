16.10 스프린트 계획(Sprint Planning) 회의의 설계와 촉진 절차

1. 스프린트 계획의 목적과 구조

스프린트 계획(Sprint Planning)은 스프린트의 첫 이벤트로서, 스프린트에서 수행할 작업을 계획하는 활동이다. Schwaber and Sutherland(2020)의 스크럼 가이드에 따르면, 스프린트 계획은 세 가지 주제를 다룬다. 이 스프린트가 왜 가치 있는가(Why is this Sprint valuable?), 이 스프린트에서 무엇을 완료할 수 있는가(What can be Done this Sprint?), 그리고 선택된 작업을 어떻게 수행할 것인가(How will the chosen work get done?)이다.

스프린트 계획의 시간 제한(Time-box)은 1개월 스프린트 기준 최대 8시간이며, 짧은 스프린트에서는 비례하여 단축된다.

2. 스크럼 마스터의 촉진 역할

2.1 사전 준비

스크럼 마스터는 스프린트 계획의 효과성을 위하여 다음과 같은 사전 준비를 수행한다. 프러덕트 오너가 백로그 항목을 충분히 정제(Refinement)하였는지 확인한다. 팀의 가용 역량(Capacity)을 사전에 파악한다(휴가, 교육, 다른 업무 등에 의한 감소 고려). 회의 공간, 도구, 그리고 시각화 자료를 준비한다. 이전 스프린트의 속도(Velocity) 데이터를 준비한다.

2.2 촉진 절차

스프린트 목표(Sprint Goal)의 수립 촉진이다. 스크럼 마스터는 프러덕트 오너가 제안하는 스프린트 목표가 명확하고 달성 가능하며 팀에 동기를 부여하는지를 검토하고, 팀 전체의 합의를 도출하도록 촉진한다.

백로그 항목의 선택 촉진이다. 팀이 스프린트 목표에 부합하는 백로그 항목을 선택하는 과정에서, 팀의 역량과 과거 속도를 고려한 현실적 범위 설정을 촉진한다. 과대 약속(Over-commitment)이나 과소 약속(Under-commitment)의 위험을 팀에 인식시킨다.

실행 계획의 수립 촉진이다. 선택된 백로그 항목을 구체적 작업(Task)으로 분해하고, 작업의 의존 관계와 순서를 정의하는 과정을 촉진한다. 이 과정에서 스크럼 마스터는 기술적 세부 사항에 간여하지 않되, 계획 과정의 체계성과 완전성을 보장한다.

시간 관리이다. 스프린트 계획이 시간 제한 내에 완료되도록 시간을 관리하고, 특정 항목에 대한 과도한 토론을 조절한다.

3. 딥테크 기업에서의 스프린트 계획 특수성

딥테크 기업에서 스프린트 계획은 다음과 같은 특수성을 반영하여야 한다. 기술적 불확실성이 높은 항목에 대한 탐색(Spike) 활동의 명시적 포함이다. 하드웨어 의존적 작업의 리드 타임 고려이다. 안전 관련 검증·문서화 활동의 스프린트 계획에의 포함이다. 연구 활동과 개발 활동의 균형 잡힌 포함이다.
