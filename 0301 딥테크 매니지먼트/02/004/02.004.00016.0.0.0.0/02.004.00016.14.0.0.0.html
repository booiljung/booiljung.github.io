16.14 백로그 정제(Backlog Refinement) 세션에서의 스크럼 마스터 역할

1. 백로그 정제의 개념과 목적

백로그 정제(Backlog Refinement, 이전 명칭 Backlog Grooming)는 프러덕트 오너와 개발 팀이 공동으로 참여하여 제품 백로그 항목의 세부 내용을 명확히 하고, 추정치를 갱신하며, 우선순위를 재검토하는 지속적 활동이다. 스크럼 가이드에 따르면, 백로그 정제는 스프린트 역량(Sprint Capacity)의 최대 10% 이내에서 수행되는 것이 권장된다.

2. 스크럼 마스터의 역할

스크럼 마스터는 백로그 정제 세션에서 다음과 같은 촉진적 역할을 수행한다.

세션의 구조화이다. 정제 대상 항목의 선별, 논의 순서, 시간 배분, 그리고 기대 산출물을 사전에 프러덕트 오너와 협의하여 세션의 구조를 설계한다.

논의의 촉진이다. 프러덕트 오너가 백로그 항목의 맥락과 목적을 설명하고, 개발 팀이 기술적 질문과 명확화 요청을 제기하는 양방향 대화를 촉진한다. 특정 항목에 대한 과도한 토론을 조절하고, 논의가 의사결정에 도달하지 못하는 경우 시간 제한적 결론 또는 추가 조사(Spike)의 필요성을 제안한다.

추정 활동의 촉진이다. 플래닝 포커(Planning Poker), T-셔츠 사이징(T-shirt Sizing), 또는 상대적 추정(Relative Estimation) 등의 추정 기법을 운영하고, 추정치의 극단적 차이가 발생한 경우 의견 교환을 촉진하여 합의를 도출한다.

INVEST 기준의 검증이다. 정제된 백로그 항목이 Independent, Negotiable, Valuable, Estimable, Small, Testable의 기준을 충족하는지를 팀이 자체적으로 검증하도록 촉진한다.

인수 기준(Acceptance Criteria)의 명확화 지원이다. 프러덕트 오너가 정의한 인수 기준이 충분히 명확하고 검증 가능한지를 팀이 검토하는 과정을 촉진한다.

딥테크 기업에서 백로그 정제의 특수성은 기술적 불확실성이 높은 항목에 대하여 추정의 어려움이 가중된다는 점이다. 스크럼 마스터는 이러한 항목에 대하여 탐색(Spike)의 선행 필요성을 팀에 인식시키고, 불확실성이 해소된 후 정제를 재수행하는 유연한 접근을 촉진하여야 한다.
