16.16 다기능 팀(Cross-functional Team) 내 역할 갈등 조정 권한

1. 다기능 팀에서의 역할 갈등의 구조

스크럼 팀은 교차 기능적(Cross-functional) 구성을 원칙으로 하며, 스프린트 목표 달성에 필요한 다양한 기능적 역량(소프트웨어 개발, 하드웨어 엔지니어링, 품질 보증, UX 설계 등)을 팀 내부에 보유한다. 이러한 다기능적 구성에서 상이한 기능적 배경을 가진 구성원 간의 역할 갈등은 자연스러운 조직 현상이다.

Jehn(1995)의 갈등 유형 분류에 따르면, 다기능 팀에서 발생하는 갈등은 과업 갈등(Task Conflict), 관계 갈등(Relationship Conflict), 그리고 프로세스 갈등(Process Conflict)으로 구분된다. 적정 수준의 과업 갈등은 의사결정의 질을 향상시킬 수 있으나, 관계 갈등은 팀 성과에 부정적 영향을 미친다.

2. 스크럼 마스터의 갈등 조정 권한

스크럼 마스터는 팀 내 역할 갈등의 조정에서 중재자(Mediator)의 역할을 수행한다. 스크럼 마스터의 갈등 조정 권한은 강제적 결정이 아닌 촉진적 중재에 기반한다.

갈등 당사자 간의 대화를 촉진하여 상호 이해를 도모한다. 갈등의 근본 원인(역할 모호성, 자원 경합, 가치관 차이)을 팀과 함께 분석한다. 역할과 책임의 재명확화를 통하여 구조적 원인을 해소한다. 합의 기반의 해결 방안을 도출하도록 촉진한다. 팀 자체적으로 해소되지 않는 갈등에 대하여 에스컬레이션을 수행한다.

딥테크 기업에서 가장 빈번한 역할 갈등은 하드웨어 엔지니어와 소프트웨어 개발자 간의 기능 배분(Function Allocation) 갈등, 연구자와 개발자 간의 탐색-활용(Exploration-Exploitation) 우선순위 갈등, 그리고 품질 보증 담당과 개발자 간의 품질 기준 갈등이다. 스크럼 마스터는 이러한 갈등의 기술적 맥락을 이해하면서도, 기술적 결정 자체에 간여하지 않고 갈등 해소의 프로세스를 촉진하는 균형을 유지하여야 한다.
