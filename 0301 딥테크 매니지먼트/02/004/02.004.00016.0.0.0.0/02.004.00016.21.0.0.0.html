16.21 규모 확장 애자일 프레임워크(SAFe)에서의 스크럼 마스터 역할

1. SAFe의 구조와 스크럼 마스터

규모 확장 애자일 프레임워크(Scaled Agile Framework, SAFe)는 Leffingwell(2011)이 개발한 대규모 조직에서의 애자일 적용 프레임워크이다. SAFe는 팀 수준(Team Level), 프로그램 수준(Program Level), 대규모 솔루션 수준(Large Solution Level), 그리고 포트폴리오 수준(Portfolio Level)의 4계층 구조를 가진다.

SAFe에서 스크럼 마스터의 역할은 단일 팀 수준의 스크럼과 비교하여 확장된다. 팀 수준에서의 기본적 스크럼 마스터 역할에 더하여, 프로그램 수준에서의 애자일 릴리스 트레인(Agile Release Train, ART) 참여가 추가된다.

2. SAFe에서의 스크럼 마스터 핵심 활동

팀 수준 활동은 기본 스크럼의 스크럼 마스터 역할과 동일하다. 스프린트(SAFe에서는 Iteration) 이벤트의 촉진, 장애물 제거, 팀 코칭이 핵심이다.

프로그램 수준 활동은 프로그램 증분 계획(PI Planning)에의 참여와 촉진이다. PI Planning은 8~12주 단위의 계획 이벤트로, 복수의 팀이 동시에 참석하여 향후 프로그램 증분(Program Increment)의 목표와 계획을 수립한다. 스크럼 마스터는 자신의 팀이 PI Planning에 효과적으로 참여하도록 촉진한다.

스크럼 오브 스크럼즈(Scrum of Scrums)에의 참여이다. 복수 팀 간의 의존성 관리와 장애물 해소를 위한 정기적 조율 회의에 참석한다. 시스템 시연(System Demo)에의 참여이다. 복수 팀의 통합된 증분을 이해관계자에게 시연하는 활동을 지원한다. 검사와 적응(Inspect and Adapt) 워크숍에의 참여이다. 프로그램 수준의 회고 활동에 참여하여 프로세스 개선을 촉진한다.

3. 릴리스 트레인 엔지니어(RTE)와의 관계

SAFe에서 릴리스 트레인 엔지니어(Release Train Engineer, RTE)는 프로그램 수준의 스크럼 마스터에 해당하는 역할이다. RTE는 ART 전체의 프로세스를 촉진하고, 팀 간 의존성을 관리하며, 프로그램 수준의 장애물을 제거한다.

개별 팀의 스크럼 마스터와 RTE의 관계는 팀 수준과 프로그램 수준의 조율 관계이다. 스크럼 마스터는 팀 내부의 장애물을 해결하고, 팀 차원에서 해결 불가능한 프로그램 수준의 장애물을 RTE에게 에스컬레이션한다.

딥테크 기업에서 SAFe의 적용은 복수의 하드웨어-소프트웨어 팀이 동시에 참여하는 대규모 시스템 개발에서 유효하다. 스크럼 마스터는 SAFe의 형식적 프로세스를 준수하면서도 딥테크 기업의 기술적 특수성(긴 리드 타임, 하드웨어 의존성, 규제 요구)을 프레임워크에 적응적으로 반영하여야 한다.
