16.15 장애물(Impediment) 식별, 제거 권한 및 에스컬레이션 절차

1. 장애물의 개념과 유형

장애물(Impediment)은 스크럼 팀이 스프린트 목표를 달성하거나 효과적으로 활동하는 것을 저해하는 모든 요인이다. 장애물의 제거는 스크럼 마스터의 가장 직접적이고 가시적인 책임이다.

장애물의 유형은 팀 내부 장애물(팀 내 소통 부재, 기술적 병목, 역량 부족, 도구 문제), 조직 간 장애물(타 부서와의 의존성, 승인 절차 지연, 자원 경합), 조직 구조적 장애물(비효율적 프로세스, 부적절한 정책, 경직적 조직 구조), 그리고 외부 장애물(외부 공급업체 지연, 규제 기관 응답 지연, 고객측 결정 지연)로 분류된다.

2. 장애물 식별의 메커니즘

스크럼 마스터는 다음 채널을 통하여 장애물을 식별한다. 일일 스크럼에서 개발자가 보고하는 장애물이다. 스프린트 회고에서 도출되는 프로세스적 장애물이다. 스크럼 마스터 자신의 관찰에 의한 식별이다. 팀 구성원과의 일대일 면담(1:1 Meeting)을 통한 식별이다.

식별된 장애물은 장애물 보드(Impediment Board) 또는 장애물 목록(Impediment Log)에 기록하여 투명하게 관리한다.

3. 제거 권한과 에스컬레이션

3.1 직접 제거 가능한 장애물

스크럼 마스터가 자신의 권한 범위 내에서 직접 제거할 수 있는 장애물은 팀 내 소통 문제의 조정, 회의 효율화, 도구 설정 변경, 팀 내 프로세스의 개선 등이다.

3.2 에스컬레이션이 필요한 장애물

스크럼 마스터의 직접 제거 권한을 넘어서는 장애물은 상위 관리자에게 에스컬레이션하여야 한다. 에스컬레이션 절차는 다음과 같이 구조화된다. 장애물의 명확한 기술이다. 장애물의 내용, 팀에 미치는 영향, 그리고 시도된 해결 방안을 문서화한다. 에스컬레이션 경로의 준수이다. 사전에 정의된 에스컬레이션 경로(팀 리드 → 부서장 → CTO/VP → CEO)에 따라 단계적으로 이관한다. 기대 행동의 명시이다. 에스컬레이션 시 상위 관리자에게 기대하는 구체적 행동(결정, 자원 배분, 조직 변경 등)을 명시한다. 추적과 피드백이다. 에스컬레이션된 장애물의 해결 진행 상황을 추적하고, 팀에 피드백한다.

4. 딥테크 기업에서의 장애물 특수성

딥테크 기업에서 빈번하게 발생하는 특수한 장애물 유형은 다음과 같다. 실험 장비·시설의 가용성 부족이다. 하드웨어 부품의 조달 지연이다. 외부 인증 기관의 응답 지연이다. 학제 간 소통 장벽이다. 규제 요건의 불명확성이다. 스크럼 마스터는 이러한 딥테크 특수적 장애물의 성격을 이해하고, 적절한 해결 경로를 식별하여야 한다.
