16.18 스크럼 마스터와 기술 리드(Tech Lead) 간의 기술 의사결정 분담

1. 역할의 본질적 구분

스크럼 마스터와 기술 리드(Tech Lead)는 스크럼 팀 내에서 상호 보완적이나 본질적으로 구분되는 역할이다. 스크럼 마스터는 프로세스의 효과성에 대한 책임을 지며, 기술 리드는 기술적 의사결정의 품질에 대한 책임을 진다.

기술 리드의 핵심 영역은 기술 아키텍처의 설계와 진화, 기술 스택의 선택과 유지, 코드 품질 기준의 설정과 검토, 기술적 문제 해결의 주도, 그리고 팀 구성원에 대한 기술적 멘토링이다. 스크럼 마스터의 핵심 영역은 스크럼 프레임워크의 올바른 실천 촉진, 팀의 자기 관리 역량 개발, 장애물의 제거, 그리고 이해관계자 소통의 촉진이다.

핵심 원칙은 스크럼 마스터가 기술적 의사결정에 직접 간여하지 않으며, 기술 리드가 프로세스 관리에 직접 간여하지 않는다는 것이다. 양 역할은 각자의 전문 영역을 존중하면서 상호 보완적으로 협력한다.

협력이 필요한 교차 영역은 기술 부채 관리의 스프린트 통합(기술 리드가 기술 부채를 식별하고 스크럼 마스터가 이를 백로그 관리 프로세스에 통합하도록 촉진), 추정 과정의 효과성(기술 리드가 기술적 추정을 주도하고 스크럼 마스터가 추정 프로세스의 효율성을 촉진), 그리고 완료 정의(Definition of Done)의 기술적 내용 결정(기술 리드가 기술적 기준을 정의하고 스크럼 마스터가 합의 과정을 촉진)이다.
