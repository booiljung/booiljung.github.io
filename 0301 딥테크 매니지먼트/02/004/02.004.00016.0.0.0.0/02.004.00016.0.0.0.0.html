Chapter 16. 애자일(Agile) 환경에서의 스크럼 마스터(Scrum Master)의 권한

1. 서론

스크럼 마스터(Scrum Master)는 스크럼(Scrum) 프레임워크에서 공식적으로 정의된 세 가지 역할(Product Owner, Scrum Master, Developers) 가운데 하나로, 스크럼의 올바른 이해와 실천을 촉진하는 핵심 역할이다. Schwaber and Sutherland(2020)의 스크럼 가이드에 따르면, 스크럼 마스터는 “스크럼 가이드에 정의된 스크럼의 확립에 책임(Accountable for establishing Scrum)“을 지는 인물이다.

딥테크(Deep Tech) 기업에서 스크럼 마스터의 역할은 순수 소프트웨어 기업과 비교하여 고유한 도전을 수반한다. 하드웨어-소프트웨어 융합 개발, 기초 연구 활동의 스프린트 통합, 안전 관련 프로세스 요구 사항, 그리고 다학제적 팀 구성 등의 특수한 환경에서 스크럼 프레임워크를 효과적으로 적용하기 위하여는 스크럼 마스터의 역할과 권한에 대한 깊은 이해가 필요하다.

2. 스크럼 마스터의 핵심 역할

2.1 서번트 리더(Servant Leader)로서의 역할

스크럼 마스터는 전통적 의미의 관리자(Manager)가 아니라 서번트 리더(Servant Leader)이다. Greenleaf(1970)가 제시한 서번트 리더십 개념에 따르면, 서번트 리더는 구성원의 성장과 자율성을 지원하고, 조직의 장애물을 제거하며, 팀이 목표를 달성할 수 있는 환경을 조성하는 것을 주된 역할로 한다.

스크럼 마스터의 서번트 리더십은 개발 팀에 대하여, 프러덕트 오너에 대하여, 그리고 조직 전체에 대하여 차별적으로 발현된다. 개발 팀에 대하여는 자기 관리(Self-management) 역량의 강화를 코칭하고, 교차 기능적 역량의 개발을 지원하며, 팀의 생산성을 저해하는 장애물을 제거한다. 프러덕트 오너에 대하여는 효과적인 제품 목표 정의와 백로그 관리 기법을 제안하고, 필요시 이해관계자 협력을 촉진한다. 조직 전체에 대하여는 스크럼의 이해와 채택을 주도하고, 경험적(Empirical) 접근법의 도입을 촉진하며, 이해관계자와 스크럼 팀 사이의 장벽을 제거한다.

3. 스크럼 마스터의 권한 구조

3.1 프로세스 권한(Process Authority)

스크럼 마스터는 스크럼 프레임워크의 올바른 적용에 관한 프로세스 권한을 보유한다. 스크럼 이벤트(Sprint Planning, Daily Scrum, Sprint Review, Sprint Retrospective)의 정기적 개최를 보장하고, 각 이벤트가 목적에 부합하게 운영되도록 촉진한다. 스크럼의 핵심 원칙(투명성, 검사, 적응)이 준수되도록 조직 내에서 옹호한다.

3.2 장애물 제거 권한

스크럼 마스터는 팀의 생산성을 저해하는 조직적·프로세스적 장애물을 식별하고 제거하는 권한을 보유한다. 이 권한은 팀 내부의 문제뿐만 아니라 타 부서와의 조율, 조직적 정책의 변경 제안, 그리고 인프라 문제의 해소에까지 미친다.

3.3 코칭 권한

스크럼 마스터는 팀과 개인에 대하여 애자일 원칙과 스크럼 프레임워크에 관한 코칭을 수행할 권한을 보유한다. 이 코칭은 지시(Directive)가 아닌 촉진(Facilitative)의 성격을 가지며, 팀이 스스로 최적의 방법을 발견하도록 안내하는 것을 목표로 한다.

3.4 권한의 한계

스크럼 마스터는 다음과 같은 권한을 보유하지 않는다. 개발 팀에 대한 작업 할당 권한이다. 스크럼 마스터는 팀원에게 특정 작업을 지시하지 않는다. 제품 우선순위 결정 권한이다. 이는 프러덕트 오너의 전담 권한이다. 인사적 권한(채용, 해고, 성과 평가)이다. 스크럼 마스터는 팀의 관리자가 아니므로 인사적 권한을 보유하지 않는다.

4. 딥테크 기업에서의 스크럼 마스터 역할 특수성

딥테크 기업에서 스크럼 마스터의 역할은 다음과 같은 특수한 도전을 수반한다. 하드웨어-소프트웨어 융합 환경에서의 스크럼 적용이다. V-모형과 애자일의 하이브리드 접근을 촉진하고, 하드웨어 개발의 긴 리드 타임을 스프린트 구조에 통합하는 역할이 요구된다. 연구 활동의 스프린트 관리이다. 기초 연구 활동의 비결정적(Non-deterministic) 성격을 스크럼의 반복적·점진적 프레임워크에 적합하게 조정하는 촉진 역할이 요구된다. 다학제적 팀의 소통 촉진이다. 상이한 학문적 배경(물리학, 공학, 컴퓨터 과학 등)을 가진 팀 구성원 간의 소통 장벽을 해소하는 역할이 요구된다. 안전 관련 프로세스의 통합이다. 기능 안전 표준(ISO 26262, DO-178C 등)의 요구 사항을 스크럼 프로세스에 통합하는 역할이 요구된다.

5. 결론

스크럼 마스터의 권한은 전통적 관리 권한과 근본적으로 상이하다. 지시와 통제가 아니라 촉진과 코칭을 통하여 팀의 자기 관리 역량을 강화하고, 조직적 장애물을 제거하며, 스크럼의 올바른 실천을 보장하는 것이 스크럼 마스터 권한의 본질이다. 딥테크 기업에서 스크럼 마스터는 이러한 본질적 역할에 더하여, 기술적으로 복잡하고 학제적으로 다양한 환경에서 스크럼 프레임워크를 적응적으로 적용하는 고유한 역량을 보유하여야 한다.
