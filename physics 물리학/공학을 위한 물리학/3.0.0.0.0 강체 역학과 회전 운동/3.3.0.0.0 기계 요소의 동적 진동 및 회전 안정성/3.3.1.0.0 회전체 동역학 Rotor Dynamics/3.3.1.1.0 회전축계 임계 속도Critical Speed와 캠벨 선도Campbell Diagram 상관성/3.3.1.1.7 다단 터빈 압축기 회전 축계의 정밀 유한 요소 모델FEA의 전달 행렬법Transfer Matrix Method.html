3.3.1.1.7 다단 터빈, 압축기 회전 축계의 정밀 유한 요소 모델(FEA)의 전달 행렬법(Transfer Matrix Method)

지금까지 배운 칠판 쪼가리 공식들은 가운데 커다란 팽이 하나만 꽂혀있는 불쌍한 제프콧축 이었지만, 현실의 500\text{MW} 화력 발전 터빈 기계 길이는 빌딩 6층 높이(20\text{m})에, 박혀있는 압축 디스크 날개 덩어리만 수십 개씩 연결된 거대한 다단 연성체다.
수십 개의 쇳덩어리가 주렁주렁 매달린 축이 미친 듯이 떨리는 20\text{m} 곡률 모습을 도대체 어떻게 계산할까? 컴퓨터가 없던 1944년, 미콜라이 프로홀(N. O. Myklestad) 과 프로흘(M. A. Prohl) 이 손으로 수십 개의 연립행렬식을 풀기 위해 고안해 낸 마법결계선, 전달 행렬법(Transfer Matrix Method, TMM) 이 훗날 유한요소해석(FEA) 의 뿌리가 되어 거대 엔진의 떨림을 모조리 예측해 낸다.

1. 덩어리 자르기: 집중 질량점(Station)과 뼈대(Field) 모델링

터빈 축을 잘 관찰해보면, 무거운 날개(디스크) 부위가 있고, 그 사이를 이어주는 가벼운 파이프 몸체(샤프트) 부위가 있다. 프로흘은 이 20\text{m} 짜리 거대한 기계를 N 개로 무참하게 토막 냈다.

  • 날개 부분은 오직 m, I_p 무게와 자이로만 존재하는 질점(Station) 점(Point) 부품으로 뭉쳐버리고,
  • 질점 사이를 잇는 샤프트는 오직 굽힘 탄성 계수 k(E \cdot I) 와 휨 변위만 관장하는 스프링 뼈대 뼈(Field)** 관으로 단순화시켜(N 개의 뼈-N 개의 관절점) 수학적 소시지 구슬 채찍으로 바꿔버렸다.

2. 상태 벡터 전달(Transfer) 행렬의 도미노 게임

한 지점(Station i) 에서 축이 얼마나 휘어졌고(v), 꺾였고(\theta), 굽힘 모멘트(힘)를 받고(M), 칼부림 전단력을 당하는지(V) 이 네 가지 물리 숫자를 하나의 \mathbf{4\times1} 상태 벡터 열(\text{State \, Vector, } \{Z\}_i) 로 뭉친다.
수학의 거대 도미노 게임이 시작된다!

  • 첫 번째 베어링 지점(1번) 의 상태 벡터 \{Z\}_1 (휘어짐, 꺾임 텐서 등) 에다가 뼈대 전달 행렬([F] 계수) 한 개를 곱(\times) 하면 신기하게도 2번째 디스크의 상태가 뿅! 튀어나온다.
  • 이 2번째 상태에 질점 전달 행렬([P] 자이로/관성 폭파 텐서) 를 또 옆에서 곱하면, 디스크를 통과한 다음 허공의 상태가 튀어나오고!
  • 이렇게 [F] 곱하고 [P] 곱하는 단순 행렬 곱셈을 축 왼쪽 끝(A 베어링단) 에서부터 축 우측 끝(B 베어링단) 까지 컴파일러로 도미노처럼 주루루룩 \mathbf{N} 번 곱하면서 지나가면([T] = [F_N][P_N]\dots[F_1][P_1]), 놀랍게도 좌측 끝단 벡터 \{Z\}_1 이 전체 터빈을 빛의 속도로 꿰뚫어 지나가며 우측 끝단 벡터 \{Z\}_N 결괏값을 단 한 줄기의 전체 행렬방정식으로 토해냈다! (Global Transfer Matrix 도출)
  • 마지막 끝단인 오른쪽 허공에서 굽힘력과 전단력이 0 이 되어야 하는 경계조건(Boundary Condition)을 0으로 풀어버리는 \Omega(각속도) 만 컴퓨터 반복계산(Newton-Raphson)으로 하나 찾으면? 그것이 바로 이 복잡한 다단 터빈의 진짜 임계 공진 파괴 스피드 다.
