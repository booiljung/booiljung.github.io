3.1.2.1.6 쿼터니언 켤레(Conjugate) 역행렬 연산과 스크류(Screw) 운동/축-각도(Axis-Angle) 대입 표현

아무리 4차원 복소수(\mathbf{q} = q_0 + q_1 \mathbf{i} + q_2 \mathbf{j} + q_3 \mathbf{k})가 우주선 컴퓨터를 폭주에서 구원했다 한들, 대체 “이 숫자 네 개가 기계가 X, Y, Z 축 어디를 찌르고 있는지 어떻게 인간이 해석하는가?“라는 벽에 부딪힌다.
수학자 오일러가 제시한 본질적 ‘등가 회전(Equivalent Rotation)’ 정리에 따르면, 강체가 허공에서 아무리 복잡한 각도 3 개(\text{Roll, Pitch, Yaw})를 섞어 비틀어져 돌아갔다 하더라도, 결과적으로 보면 “어디인지 모를 우주의 삐딱한 어떤 하나의 가상 막대기(축 벡터 \vec{e}) 하나를 콱 꽂아놓고, 그냥 그 막대기를 중심으로 각도 \Phi 만큼 빙긋 돌린 것(Axis-Angle)” 과 수학적으로 100\% 똑같이 포개진다(스크류 전개). 쿼터니언의 이 4 개 숫자는 바로 이 신비한 단일 꼬챙이 축(\vec{e})과 그 축의 회전 각도(\Phi)를 자궁처럼 품고 있는 기하 압축기다.

1. 축-각도(Axis-Angle) 매핑: 스크류(Screw) 축의 체화

쿼터니언 4 개 배열 [q_0, q_1, q_2, q_3] 을 조립식으로 찢어서 보면, 그 속엔 인간이 직관적으로 우주에 그릴 수 있는 꼬챙이 벡터 수치가 완벽하게 숨겨져 있다.

  • 스칼라 부위 (q_0): q_0 = \cos(\Phi / 2). 즉, 기체가 그 삐딱한 꼬챙이 축을 중심으로 몇 도 돌아갔느냐(\Phi, 변위) 의 삼각함수.
  • 벡터 부위 (q_1, q_2, q_3): 이 허수 3 개 배열 [q_1\mathbf{i}, q_2\mathbf{j}, q_3\mathbf{k}] 는 바로 q_{1,2,3} = \vec{e} \sin(\Phi / 2) 로, “도대체 그 하나의 꼬챙이 축(\vec{e}) 화살표가 우주 공간 X, Y, Z 어디를 찌르고 향하고 있는 방향인가?” 를 알려주는 3차원 지시봉 벡터다.
  • 이 아름다운 \vec{e} 라는 회전축과 \Phi 라는 각의 융합은, 비행기가 오일러 방정식처럼 조잡하게 \text{Z} 로 돌았다 \text{X} 로 돌았다 3 단계를 밟을 필요 없이, 그냥 최단 거리 기하학적 단 한 번의 단일 회전(Single Rotation) 조향만으로 가장 빠르게 목표 자세를 쟁취하는 우주 최단 시간(Optimal Time) 제어 루트를 프로그래밍해 낸다!

2. 방향의 뒤집기: 쿼터니언 켤레(Conjugate, \mathbf{q}^*)의 경이로운 대수 역전

방향 코사인 행렬(\text{DCM})은 비행기를 반대로 돌리려고(지구 \rightarrow 비행기 \leftarrow 지구 역산) 3 \times 3 매트릭스 전체 원소를 대각선으로 엎어치는 미친 전치(\text{Transpose}) 노가다를 해야 했다.
그렇다면 쿼터니언 세계에선 “지금 각도의 정확히 정반대 뒤집기 역회전(\mathbf{q}^{-1} 역행렬 파동)“을 연산하려면 어떻게 할까?

\mathbf{q}^* (\text{켤레 복소수}) = q_0 - q_1 \mathbf{i} - q_2 \mathbf{j} - q_3 \mathbf{k}

  • 컴퓨터 프로그래머는 \mathbf{q} 의 역행렬 분수 나눗셈 같은 것을 절대 짜지 않는다.
  • 단지 뒤쪽 허수 벡터(q_1, q_2, q_3) 부분의 부호를 플러스(+)에서 마이너스(-) 로 띡 하고 3개 반전시켜 버린 켤레(Conjugate, \mathbf{q}^*) 를 만들면, 기적처럼 그것이 수학적으로 완벽한 회전 복구 역행렬 \mathbf{q}^{-1} 과 완전히 똑같은 역진동 텐서 화살표 기능을 한다!
  • \mathbf{q} \otimes \mathbf{q}^* = 1 \text{ (Identity 불변) }. 이것이 고작 덧셈 뺄셈(-) 부호 뒤집기 몇 번으로 우주선 3D 방향을 뒤집어 역투과하는 실리콘 칩 연산의 극한 최적화 치트키다.

쿼터니언은 인간의 머리로는 눈을 감고 도저히 회전 모습을 상상할 수 없다(0.5 + 0.3i 가 몇 도인지 도저히 직관이 안 됨). 그러나 기계와 CPU 칩 텐서 안에서는 어떠한 나눗셈이나 복잡한 삼각 스윕루프 없이(오직 \text{부호 뒤집기} 와 \text{곱셈}), 우주 3차원 공간의 가장 기괴하고 지독한 멀티 덤블링 난기류 연속 회전을 가장 적은 4개의 바이트(Byte) 배열 숫자 덩어리만으로 압축해 담아내는 궁극의 Axis-Angle \text{(축-각도) 스크류} 지퍼(Zipper) 파일 변환기다.
