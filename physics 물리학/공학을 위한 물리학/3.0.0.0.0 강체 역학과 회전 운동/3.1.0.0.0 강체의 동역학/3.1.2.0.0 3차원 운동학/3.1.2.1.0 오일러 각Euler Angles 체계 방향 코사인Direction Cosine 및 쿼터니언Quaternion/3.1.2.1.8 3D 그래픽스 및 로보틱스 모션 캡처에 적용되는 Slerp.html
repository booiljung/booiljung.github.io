3.1.2.1.8 3D 그래픽스 및 로보틱스 모션 캡처에 적용되는 구면 선형 보간 알고리즘(Slerp)

게임 엔진(Unreal Engine) 화면 속에 A 방향을 쳐다보던 스페이스 마린 캐릭터가 갑자기 총을 B 방향으로 틀었다. 만약 애니메이터 코드가 아날로그 방식인 오일러 각도 \text{Roll, Pitch, Yaw} 를 A 에서 B 까지 숫자로 대충 50\% 선형 보간(Linear Interpolation, Lerp)을 때려서 평균 궤적을 잇는다면?
총기 마우스 무대(Camera Viewport)에서 캐릭터의 팔은 가장 짧은 궤적을 따라 예쁘게 원을 돌지 않고, 미친 듯이 늘어지거나 관절이 90^\circ 꺾이는(Gimbal Lock) 요동을 치며 기괴한 커브로 질주할 것이다(Jerky motion).

이 끔찍한 오일러 중간 보간 요동을 파쇄하고, 4차원 구(Sphere) 표면에서 가장 우아하고 속도가 일정한 최단 거리 대원(Great Circle) 곡선을 긋게 해 주는 궁극의 애니메이션 공학의 성배. 그것이 켄 슈메이커(Ken Shoemake, 1985)가 쿼터니언을 통해 세상에 던진 \text{Slerp (Spherical Linear Interpolation, 구면 선형 보간)} 이다.

1. Lerp(직선)의 기하 파괴 궤적 vs Slerp(구면 대원) 최단 아크망

우선 출발지 자세 암호가 \mathbf{q}_1 단위 쿼터니언(4차원 공 위 표면의 한 점) 방위고, 도착해야 할 목표 기체 궤적이 \mathbf{q}_2 방위각 좌표라고 해보자. 컴퓨터는 0 초부터 1 초 사이 시간 t 에 걸쳐(가령 t=0.5 는 딱 중간) 이 두 개 점 \mathbf{q}_1 \rightarrow \mathbf{q}_2 을 잇는 중간 자세 \mathbf{q}(t) 궤적 배열을 렌더링해야(보간) 한다.

  • \mathbf{Lerp} 오일러 선점: 이 둘 사이를 가장 쉬운 수학인 직선망 \mathbf{q}(t) = (1-t)\mathbf{q}_1 + t\mathbf{q}_2 로 수천 개 중간 점을 찍고 관절을 잇는다. 이 수학 공식은 두 쿼터니언 점(구의 표면) 사이의 “속을 관통하는 지름길 직선 뼈대“를 타고 파고든다.
    결과적으로, 중간 궤적 변위(\mathbf{q}(0.5))가 구면 표면(1\text{ 거리})에서 푹 파인 땅속 내부로 찌그러지며 단위 쿼터니언 1의 진폭(\text{Unit Length 1})이 붕괴된다(비정규화). 이 강제가 뼈대를 줄어들게 하고 렌더링 각속도 \omega 가 처음엔 느리다가 중간에 윙 날아가고 끝에 쾅 멈추는 2 중 가속 징글거림(Distortion)을 초래한다.

  • \mathbf{Slerp} 텐서의 곡선화: 반면 Slerp 공식은, 두 쿼터니언 궤도가 4차원 초구(Hypersphere) 위를 미끄러지는 ’공 표면의 대원 곡선(Arc)’의 지름길 선분만을 고집한다.

\text{Slerp}(\mathbf{q}_1, \mathbf{q}_2, t) = \frac{\sin((1-t)\Omega)}{\sin\Omega} \mathbf{q}_1 + \frac{\sin(t\Omega)}{\sin\Omega} \mathbf{q}_2
(단, \cos\Omega = \mathbf{q}_1 \cdot \mathbf{q}_2 쿼터니언 스칼라 내적, \Omega 는 두 복소 벡터 사이 각)

2. 4D 회전 구 반구체 일정 속도(Constant Velocity)의 기적

저 미친 \text{Slerp} 수식에 시간 t=0 \sim 1.0 (애니메이션 100 프레임 분량)을 집어넣어 미분해 보라. 이 보간 공식은 구(공) 겉껍질 표면을 단면으로 깔끔하게 잘라내듯, 표면 곡선 위를 타고 “단 1의 가속이나 감속도, 덜컹거림도 없이 완벽하게 일정한 원 등속 각속도(Constant Arc Velocity)” 로 유유히 미끄러져 부드럽게 두 부품 관절 좌표를 이어버린다.

\text{Slerp} 를 사용하기 전, 조향 수학망은 로봇 관절 \mathbf{q}_1 과 거울 대칭인 -\mathbf{q}_1 (쌍둥이 대척점) 중 어느 점을 타고 넘어가야, 각도 상 최단 거리(Shortest Path) 180^\circ 이하로 궤적을 치고 나갈 수 있을지(\text{q}_1 \cdot \text{q}_2 내적 스칼라 양수/음수 판별) 최단 궤적 역방향 방파제 점프 연산 알고리즘을 추가한다. 그래야 로봇 팔이 옆으로 돌아도 될 것을 굳이 뒤로 괴상하게 한 바퀴 등 뒤로 더 거꾸로 도는 역기하 삽질 오류(Long-way around Rotation) 경로를 차단할 수 있다. \text{Slerp} 는 게임 프로그래머가 수학의 괴물, 쿼터니언과 손잡은 가장 거룩한 실전 최첨단 선형 렌더링 미적분 기명이다.
