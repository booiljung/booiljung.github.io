3.1.2.2.3 두 개 이상의 상대 회전 좌표를 병합(Composition)한 각속도 절대 벡터 벡터합(Vector Addition) 정리

다소곳하게 멈춰 있는 지구(\text{Earth Frame}) 위를 우주 정거장(\text{ISS Frame})이 거대한 각도로 자전하며 돌고 있고, 그 우주 정거장 외벽 한가운데에 달린 로봇 캠 팔(\text{Camera Frame})이 스스로의 관절을 비틀고 있다.
이때 지구에서 렌즈를 쳐다보았을 때, 캠 렌즈의 “마지막 절대 각속도(Absolute Angular Velocity, \vec{\omega}_{\text{abs}})” 가 도대체 무엇이냐는 문제가 떨어졌다.
선형 속도(\vec{v} = 100\text{km/h} + 50\text{km/h}) 덧셈은 너무나 단순하다. 하지만 우주 3차원 축 안목에서 거친 스크류처럼 파고드는 ’역학 회전 속력(RPM \vec{\omega})’들을, 마치 돈을 세듯 수학적으로 \vec{\omega}_1 + \vec{\omega}_2 라고 무식하게 더해도(Vector Addition) 되는가?
우주 운동학의 대답은 기적에 가깝게도 “예스(Yes), 단, 벡터들의 꼬챙이를 완벽히 똑같은 3\text{D} 좌표계 뼈대에 일치시켜 놓았다면” 이다.

1. 회전 병합망 수식: \vec{\omega}_{global\_to\_cam} = \vec{\omega}_{global\_to\_iss} + \vec{\omega}_{iss\_to\_cam}

오일러 각도를 A \times B \times C 연속으로 비틀 때(Attitude matrix Composition), 우리는 앞 장에서 자세 각도는 절대 \mathbf{A+B+C} 덧셈으로 성립하지 않고 기괴한 9차원 행렬 교차 내적의 역순 행렬 곱셈(\mathbf{C} \times \mathbf{B} \times \mathbf{A}) 지옥을 맛본다는 비가환성(Non-commutativity) 대수를 배웠다.
그러나 정말 소름 끼치는 물리학의 대칭 은혜는, 그렇게 끔찍한 행렬 덧셈이 안 되는 회전 ’거리(각도)’와 다르게, 회전 ‘속도(Angular Velocity)’ 벡터 \vec{\omega} 만큼은 언제나 \text{Vector 선형 합}이 100\% 완벽히 성립(Kinematic Addition Theorem) 한다는 점이다.

\vec{\omega}_{3/0} = \vec{\omega}_{3/2} + \vec{\omega}_{2/1} + \vec{\omega}_{1/0}

  • 어깨의 회전 속도, 팔꿈치의 회전 속도, 손목의 회전 속도를 3차원 우주 X, Y, Z 방향 스칼라 성분별로 정직하게 그냥 더하면(Vector Addition), 그게 손가락 끝에 찍힌 최종 절대 우주 각속도가 튀어나온다.
  • 단, 이 수학적 은총을 누리기 위해서는 절대적 전제 조건 결속(Coordinates Matching) 이 선포되어야 한다. 내 어깨뼈의 x축과, 팔꿈치 뼈의 x축 꼬챙이가 지금 각기 엉뚱한 곳을 향해 틀어져 있다면, 저 숫자들을 무턱대고 더해선 안 된다.
  • 모두 통일된 언어(가령 모두 Global Frame 지구 좌표) 하나로 방향 코사인 매트릭스 변환 \mathbf{C} 텐서를 양동이처럼 끼얹어, 모든 우주 각속도 \vec{\omega} 벡터 화살표들의 등뼈를 ஒரே 한 방향(Base Frame)으로 강제 정렬 매핑시켜 놓은 후에 더해야만 3차원 렌더링이 성립한다!

2. 좌표 변환 분해와 강제 합산(Superposition) 공식

다관절 기계(Manipulator) 로보틱스에서 \mathbf{J} 관절 n 개의 최종 속도를 더하는 사슬망은 이렇게 겹겹이 중첩(Transformation) 분쇄된다.

\vec{\omega}_{n/0}^{(0)} = \vec{\omega}_{n/n-1}^{(0)} + \vec{\omega}_{n-1/n-2}^{(0)} + \dots + \vec{\omega}_{1/0}^{(0)}

그런데, 현실의 3 번 모터 센서는 자기 자신(Local \text{ 3})의 자전축 z_3 로 도는 값(\vec{\omega}_{3/2}^{(3)})을 뱉지, 남의 동네(지구 우주 좌표 (0)) 따위를 모른다. 따라서, 엔지니어는 local 변수 벡터들을 전부 바깥 껍질 매트릭스(\text{Rotation Matrix } \mathbf{R})로 겹겹이 씌워(포장하여) 지구 좌표(0) 언어로 변환(Translate)해 주는 중노동을 더하기식 항마다 물려야 한다.

\vec{\omega}_{\text{Total}}^{(0)} = \mathbf{R}_{0 \leftarrow 1} \, \vec{\omega}_{1}^{(1)} \;+\; \mathbf{R}_{0 \leftarrow 2} \, \vec{\omega}_{2}^{(2)} \;+\; \mathbf{R}_{0 \leftarrow 3} \, \vec{\omega}_{3}^{(3)}

회전(Rotation 각도 모션) 행렬은 3 \times 3 기하학적 곡면의 노예라 교환 법칙이 멸망하지만, 초미세 시간(Time Derivative)으로 자른 속도의 화살표들(\vec{\omega} 극한 벡터 파동) 은 직선 공간을 부드럽게 활공하는 순수 3\text{D} 벡터다. 따라서 언제나 합산, 평행이동, 벡터 투영 치환의 모든 선형 역학 축복 스칼라들을 온전히 흡수할 수 있다. 로보틱스와 다물체 자동차 서스펜션(Multi-body Dynamics) 등 뼈 수백 개가 미친 듯이 연쇄 충돌 이동하는 공간에서 \vec{\omega}_{total} = \sum \vec{\omega}_{local}^0 공리의 무결점 덧셈은 생명선 그 자체다.
