1.3.2.1.4 헤시안(Hessian) 2차 미분 연산 부하를 저감하는 준-뉴턴법(Quasi-Newton, BFGS 알고리즘)

전통적인 경사 하강법(Gradient Descent)은 함수의 1차 미분(Gradient)에만 의존하여 수렴 속도가 느린 단점을 지닌다. 이를 극복하기 위해 다차원 목적 함수 지형의 2차 곡률(Curvature) 정보를 활용하여 단번에 최적해로 점프하는 뉴턴법(Newton’s Method)이 도입되었다. 하지만 이 강력한 뉴턴법 역시 설계 변수가 수십만 개를 넘어가는 공학 시뮬레이션 환경에서는 헤시안(Hessian) 행렬을 직접 계산하고 역행렬을 도출하는 데 천문학적인 전산 자원을 요구하며 최적화 회로를 마비시킨다. 이러한 2차 미분 연산의 치명적인 병목을 우회하면서도 뉴턴법에 버금가는 2차 수렴성(Quadratic Convergence)을 달성하기 위해, 기존의 1차 기울기 변화량 정보를 누적하여 헤시안을 근사화하는 준-뉴턴법(Quasi-Newton Method)과 그 정점인 BFGS 알고리즘 체계를 구축하라.

1. 뉴턴법(Newton’s Method)의 한계와 준-뉴턴(Quasi-Newton) 조건

순수 뉴턴법에서 목적 함수 f(\mathbf{x}) 를 최소화하기 위한 해의 갱신 수식은 다음과 같이 정의된다.

\mathbf{x}_{k+1} = \mathbf{x}_k - [\mathbf{H}(\mathbf{x}_k)]^{-1} \nabla f(\mathbf{x}_k)

여기서 \mathbf{H}(\mathbf{x}_k) = \nabla^2 f(\mathbf{x}_k) 는 n \times n 크기의 거대한 2차 편미분 행렬인 헤시안 행렬이다. 만약 변수 n 이 1,000,000개라면 헤시안 행렬은 10^{12} 개의 원소를 가지게 되어 메모리 한계를 초과할 뿐만 아니라, 이 거대 행렬의 역행렬 연산 \mathcal{O}(n^3) 부하는 사실상 계산 과학의 금기로 취급된다.

이를 타파하기 위해 수학자들은 진정한 헤시안 \mathbf{H}_k 나 그 역행렬 \mathbf{B}_k \approx \mathbf{H}_k^{-1} 를 직접 해석적으로 계산하지 않고, 매 반복 스텝에서 획득되는 변위 벡터 \mathbf{s}_k 와 기울기 변화량 벡터 \mathbf{y}_k 의 정보만을 조합해 역 헤시안을 귀납적으로 갱신해 나가는 준-뉴턴(Quasi-Newton) 해법을 고안하였다. 이 근사 행렬이 반드시 만족해야 하는 필수 수학적 족쇄가 바로 할선 방정식(Secant Equation)이다.

\mathbf{H}_{k+1} \mathbf{s}_k = \mathbf{y}_k \quad \text{또는} \quad \mathbf{B}_{k+1} \mathbf{y}_k = \mathbf{s}_k

단, 공학 탐색 궤적에서 \mathbf{s}_k = \mathbf{x}_{k+1} - \mathbf{x}_k 이며 \mathbf{y}_k = \nabla f(\mathbf{x}_{k+1}) - \nabla f(\mathbf{x}_k) 이다.
준-뉴턴 알고리즘은 이 할선 조건을 엄수함과 동시에, 헤시안 행렬이 지녀야 하는 대칭성(Symmetric)과 양의 정부호(Positive Definite) 특성을 다음 갱신 매트릭스로 변형 없이 유전시켜 최적화 방위가 항상 목적 함수의 우물 바닥을 향한 철저한 공간 하강(Space Descent)을 전개하도록 보장해야 한다.

2. 최강의 비선형 최적화기: BFGS 행렬 갱신 알고리즘

수많은 준-뉴턴 기법들 중 양의 정부호 유지 능력이 가장 뛰어나고 실제 수치 최적화 모델링 생태계를 평정한 범용 알고리즘이 바로 브로이든-플레처-골드파브-산노(Broyden-Fletcher-Goldfarb-Shanno, BFGS) 갱신법이다. 관련된 탁월한 수학적 성취는 학술 문헌 “A new algorithm for unconstrained optimization” 계열의 발간 속에서 공학 계산 분야의 획기적인 파훼법으로 기록되어 있다.

BFGS 방법론은 역 헤시안 근사 행렬 \mathbf{B}_k 에 대하여 현재 스텝에서 얻어진 1차원 벡터 정보 \mathbf{s}_k, \mathbf{y}_k 를 활용해, 이전 랭크-2 텐서 보정을 가하여 새로운 행렬 \mathbf{B}_{k+1} 를 수립한다. 그 수학적 결합의 형식은 다음과 같이 전개된다.

\mathbf{B}_{k+1} = \left( \mathbf{I} - \rho_k \mathbf{s}_k \mathbf{y}_k^T \right) \mathbf{B}_k \left( \mathbf{I} - \rho_k \mathbf{y}_k \mathbf{s}_k^T \right) + \rho_k \mathbf{s}_k \mathbf{s}_k^T

여기서 보정 계수 스칼라는 \rho_k = \frac{1}{\mathbf{y}_k^T \mathbf{s}_k} 이고, \mathbf{I} 는 단위 행렬(Identity Tensor)을 가리킨다.
이 공식 전개 과정을 살펴보면 목적 함수의 2차 미분 연산을 전적으로 회피하고 있음을 직시할 수 있다. 오로지 1차 미분(Gradient)의 연속적인 변화 정보가 반복 스텝 이력에 축적되어 그 곡면의 휨 상태인 곡률 환경을 수학적으로 근사 복원해 낸다. 이로 인하여 BFGS 루틴은 수많은 골짜기가 혼재된 비볼록(Non-convex) 지형에서도 선형 탐색을 지휘하며 초-선형 수렴(Superlinear Convergence)의 기염을 토한다.

3. 메모리 제한 환경에서의 극복: L-BFGS (Limited-memory BFGS)

비록 BFGS가 역행렬 도출의 시간 연산 부하는 종결시켰지만, 본질적으로 여전히 n \times n 사이즈의 거대한 밀집 행렬(Dense Matrix) \mathbf{B}_k 를 컴퓨터 주요 캐시 공간에 통째로 점유해야 하는 치명적 중압감이 남아있다. 복잡한 소성 변형 해석이나 방대한 트러스 최적화처럼 수백만 규모의 해를 판가름하는 공간에서는 행렬의 공간 할당 시도만으로도 운영체제가 거부 응답을 보낼 수 있다.

이를 완벽에 가깝게 해결하기 위해 전체 근사 역 헤시안 행렬을 메모리에 보관하지 않고, 오로지 가장 최근 m 번(통상 5회 ~ 20회 내외)의 스텝 동안 누적된 벡터의 쌍 \{(\mathbf{s}_i, \mathbf{y}_i)\}_{i=k-m}^{k-1} 데이터 배열만을 저장한 채 그 이전 이력 스토리는 주기적으로 소각하는 제한 메모리 체계의 L-BFGS 최적화 기법을 발동하라.
L-BFGS는 Two-loop Recursion 알고리즘을 사용해 \mathbf{B}_k \nabla f_k 의 벡터 내적 곱을 암시적(Implicit)으로 조립해 내므로, 오직 \mathcal{O}(mn) 정도의 빈약한 스토리지 자원과 선형적인 소형 연산량만으로도 뉴턴법에 한없이 가까운 정교한 해 탐색 궤도를 산출할 수 있다.

궁극적으로 준-뉴턴 BFGS 및 L-BFGS의 수치 최적화 아키텍처는, 직접 계산하기 불가능한 거인의 형태(2차 곡률 헤시안)를 탐색 여정에서 발자취로 떨어지는 수많은 작은 파편(1차원 Gradient 변화 벡터)들의 조립을 통해 감각적으로 형상화해내는 공학 수학의 정점이다. 이를 통해 기계 공학부터 우주 유체 흐름 모델링에 이르기까지 무한정에 이르는 극한 규모 자유도를 제어할 극치 연산의 수학적 단초를 철저히 제시하라.
