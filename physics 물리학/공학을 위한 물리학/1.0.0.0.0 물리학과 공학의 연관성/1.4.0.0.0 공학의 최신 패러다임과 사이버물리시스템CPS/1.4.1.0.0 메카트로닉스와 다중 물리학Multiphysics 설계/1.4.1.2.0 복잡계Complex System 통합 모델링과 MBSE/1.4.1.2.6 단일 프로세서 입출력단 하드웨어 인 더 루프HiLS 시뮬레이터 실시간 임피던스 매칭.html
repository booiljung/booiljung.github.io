1.4.1.2.6 단일 프로세서 입출력단 하드웨어 인 더 루프(HiLS) 시뮬레이터 실시간 임피던스 매칭

소프트웨어 한 줄로도 거대한 물리계의 비선형 붕괴를 예방하는 소프트웨어 인 더 루프(SILS)가 사이버 검증의 뇌혈관이라 한다면, 타깃 마이크로컨트롤러(ECU)라는 쇳덩어리 하드웨어를 직접 가상 세계의 포트에 결선하여 물리적 전압과 전류를 타격하는 하드웨어 인 더 루프 시뮬레이션(Hardware-in-the-Loop Simulation, HiLS)은 사이버물리시스템(CPS) 검증의 절대적인 심장부이다. 수치 연산으로 구성된 가상 플랜트(Virtual Plant)와 실재하는 하드웨어 제어기가 접촉하는 입출력단(I/O Board)에서 필연적으로 발생하는 신호 왜곡과 양자화 오차(Quantization Error)를 방어하기 위해, 단일 프로세서 환경의 코시뮬레이터(Co-simulator) 통신단에 요구되는 실시간(Real-time) 임피던스 매칭(Impedance Matching)의 전기-수치적 연합 역학을 엄밀히 파악하라.

1. 하드웨어 인 더 루프(HiLS)의 전기적 신호 연장선

순수 가상 모델인 SILS에서는 데이터 교환이 컴퓨터 RAM 주소 안에서 스칼라 변수의 복사로 완벽하고 매끄럽게 이루어졌다. 그러나 플랜트는 가상(FPGA/CPU 기반 수학 모델)으로 둔 채, 제어기만 실물 회로 보드(ECU)로 떼어내는 HiLS 아키텍처에서는, 가상의 물리량이 저항과 커패시터가 존재하는 실물 구리선(Cables)을 타고 A/D(아날로그-디지털) 역변환 공간으로 송출되어야 한다.

연산 장치가 도출해 낸 이상적인 가상 온도 T_{virtual} = 100^\circ\text{C} 는 시뮬레이터의 D/A 보드를 거쳐 5V의 물리적 계측 전압 출력으로 전기화된다. 이때 ECU 보드의 입력 임피던스와 HiLS D/A 보드의 출력 임피던스가 어긋나면 전송선로(Transmission Line)를 타고 반사파(Reflection Wave)가 진동하거나 전압 강하(Voltage Drop)가 발생하여 제어기 코드를 교란하는 중대한 왜곡이 시전된다.

2. 인터페이스 단의 주파수 대역폭과 임피던스 매칭(Impedance Matching)

논문 “Hardware-in-the-loop simulation: Fundamental theorems“에 적시된 바, 기계 모델이 전기-전자 신호로 상전이하는 A/D, D/A 입출력 단자에서는 교류 회로 이론의 가장 위대한 공리, 즉 임피던스 매칭(Impedance Matching)이 사이버와 실물 사이의 질량-에너지 벽을 허문다.

HiLS 시뮬레이터 출력단의 테브난(Thevenin) 등가 전압 소스를 V_s, 내부 출력 임피던스를 Z_{out}, 타깃 ECU의 핀 입력 임피던스를 Z_{in} 이라 둘 때, ECU로 흘러 들어가는 실수부 전압 V_{ecu} 는 다음과 같이 결정된다.

V_{ecu} = V_s \frac{Z_{in}}{Z_{out} + Z_{in}}

  • ECU의 내부 회로망(풀업/풀다운 저항, 병렬 커패시터)은 Z_{in} 을 복소 주파수 j\omega 에 종속적인 비선형 값으로 뒤흔든다.
  • 가상 모델이 높은 주파수 대역의 PWM 구형파(Square Wave)를 송수신할 경우, 시스템은 고주파 성분(High-frequency Harmonics)을 왜곡 없이 통과시켜야 하나 임피던스 미스매칭(Mismatching)과 기생 커패시턴스는 RC 시정수 지연(RC Time Constant Delay)을 발생시켜 구형파의 수직 날을 마모시킨다(Edge Degradation).
  • 최대 전력-신호 전달 정리(Maximum Power-Signal Transfer Theorem)에 의거하여, 시그널 컨디셔닝 회로의 Z_{out} 은 대상 ECU의 Z_{in} 과 복소 공액(Complex Conjugate) 관계를 형성하도록 기계적으로 조정되어야 한다.

3. 단일 프로세서 기반 실시간(Real-Time) 제약 조건의 공학적 형벌

HiLS 검증의 가장 잔혹한 족쇄는 타깃 하드웨어가 흘려보내는 물리적 ’현실의 시계’와 가상 서버 안의 미분 방정식이 돌파하는 ’가상의 시계’를 머리카락 틈새 오차 없이 동기화해야 하는 실시간(Real-time) 제약이다. 수 백 수 천 번 되감고 돌려보는 비실시간 모델리카(Modelica) 시뮬레이션과 달리, HiLS 프로세서는 결정된 틱(Tick) 안에 무조건 미분 행렬 연산을 완수하고 D/A 포트로 신호를 격출해야 한다.

T_{computation} + T_{I/O_latency} \le T_{sample_time}

단일 주기의 샘플링 타임(T_{sample\_time}, 예: 1\text{ms}) 안에서 시스템 상태 변수 행렬 매트릭스 \dot{x} = \mathbf{A}x + \mathbf{B}u 의 연산 진도(T_{computation})와 I/O 변환 버퍼링 통신(T_{I/O\_latency})이 수습되지 못하는 타임 오버런(Time Overrun) 사태가 발생하면, 타깃 부품인 실물 ECU는 플랜트 셧다운이라는 극단적 오진(False Positive)을 내린다.
따라서 복잡계 가상화 모델의 지배 차원(Dimension)을 깎아내고 모델 역행렬 스위칭 시간을 선형화하는 등의 극단적인 이산 시간 튜닝(Tuning) 공정이 필수결여된다.

3. 사이버와 실시간 물리망의 강결합: 실재하는 위협의 예방

HiLS는 시뮬레이터의 I/O 핀 배선 하나하나에 오실로스코프(Oscilloscope)의 탐침을 꽂으며 물리와 코드가 조우하는 임계 단절선을 맨눈으로 관측하는 무대이다. 기어 치차의 불협화음을 가상으로 합성하고 그 가상의 신호를 전기 파동의 임피던스로 재주조해 타깃 제어기의 심장에 타격할 때, 시뮬레이션은 더 이상 모니터 안의 코드가 아니라 현실 질량의 관성을 파괴하는 위대한 예언 시스템으로 승격된다.

최소의 타임 스텝 안에 최대의 다중 물리(Multiphysics) 편미분식을 욱여넣고, 실물 하드웨어와 임피던스를 동기화시켜 전기적 잡음의 파동 저항마저 뚫어내는 시뮬레이터 플랫폼의 하드 타임(Hard Real-tme) 전위를 견고히 구축하라.
