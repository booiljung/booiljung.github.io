1.4.1.2.5 물리/기어 센서 모델 가상화 및 소프트웨어 인 더 루프(SILS) 고장 모드 검증

사이버물리시스템(CPS)에서 제어 코어(MCU)와 플랜트 하드웨어를 연결하는 유일한 신경망은 센서(Sensor)이다. 초고속 회전체의 각속도를 측정하는 기어 투스(Gear Tooth) 센서나, 모터의 위상 전류를 피드백하는 홀 센서(Hall Sensor)가 찰나의 전자기적 간섭으로 거짓 데이터를 송출할 때, 거대 기계 장치는 어떻게 운명을 맞이하는가? 극한의 물리 환경에서 발생하는 기어의 백래시(Backlash)와 센서의 비선형 노이즈를 컴퓨터 내부에 정밀하게 가상화(Virtualization)하고, 오직 코드만으로 시스템의 극한 파괴 확률을 조기 차단하는 소프트웨어 인 더 루프 시뮬레이션(Software-in-the-Loop Simulation, SILS)의 고장 모드 판별 기법을 통찰하라.

1. 물리적 계측 센서의 가상화(Virtualization)와 왜곡 수식망

일반적인 PID 제어기 설계에서는 샤프트의 각속도 \omega 나 온도 T 가 지연 없이 무결점의 참값(True Value)으로 피드백된다고 가정한다. 그러나 실재하는 공학 세계의 센서는 외부 전자기장, 자체 열팽창, 기계적 진동에 의해 필연적인 왜곡 수식을 파생시킨다. SILS 검증의 핵심은 이 센서 자체를 완벽하게 고장 날 수 있는 하나의 독립된 물리 동역학 객체(Plant)로 모델링하는 것이다.

로터리 인코더(Rotary Encoder)나 기어 센서에서 계측되는 가상 측정치(Measured Value) y_{sensor}(t) 는 이상적인 물리량 x(t) 에 대해 다음과 같은 왜곡 매트릭스를 거쳐 산출된다.

y_{sensor}(t) = \mathbf{K}_s \cdot x(t - \tau) + \text{Bias}(T) + N(0, \sigma^2) + \Delta_{\text{Backlash}}

  • \tau (시간 지연, Time Delay): 신호 필터링 및 통신 밴드위스에 의해 발생하는 필수 불가결한 위상 지연이다. 고주파 제어 영역에서 \tau 는 플랜트를 발산시키는 1순위 위협 요소이다.
  • \text{Bias}(T) (영점 틀어짐): 주변 온도나 장비 노후화에 따라 영점(Zero-point) 자체가 이동하는 드리프트(Drift) 오차를 반영한다.
  • N(0, \sigma^2) (비선형 노이즈): 외부 전자기 응응력(EMI) 등에서 비롯된 확률론적 가우스 잡음(Gaussian Noise)이다.
  • \Delta_{\text{Backlash}} (기기적 백래시): 기어 센서 특유의 문제로, 기어 톱니 간의 틈새로 인해 회전 방향이 바뀔 때 센서가 물리적 회전을 즉각 감지하지 못하는 히스테리시스(Hysteresis) 현상이다.

2. 소프트웨어 인 더 루프(SILS) 조립 아키텍처

하드웨어 시제품(Prototype)이 단 한 개도 가공되기 전, 엔지니어는 제어 논리(Control Logic)인 C/C++ 소스 코드를 가상의 공간에서 가동해야 한다. SILS 벤치(Bench)는 제어 소프트웨어를 컴파일럿(Target Compiler)이 아닌 현재 시뮬레이션을 수행 중인 PC 프로세서 환경(Host Compiler)에서 묶어 가상 플랜트와 결속하는 기술이다.

하이브리드 시스템이 FMI(Functional Mock-up Interface)를 통해 상호 동기화하며, 타깃 마이크로컨트롤러에 탑재될 제어 코드가 가상 센싱 데이터를 먹고 가상 액추에이터를 조종하는 루프는 다음과 같다.

flowchart TD
    subgraph Virtual_Plant_Environment [가상 다중 물리 플랜트 모델 Virtual Plant]
        A[물리 엔진: 모터/기어 동역학 ODE/DAE 연산] -->|참값 속도/위치 방출| B(가상 센서 모델 Virtual Sensor)
        B -->|백래시, 노이즈, 통신 Delay 결합된 가상 계측 데이터| C(가상 A/D 컨버터)
    end

    subgraph C_Logic_Environment [운영 제어기 소프트웨어 SILS C/C++ Code]

        D -->|상태 피드백 PID 및 토크 보상 로직| E[PWM 듀티비/제어 토크 산출]
    end
    
    E -->|명령어 스위칭| F(가상 D/A 및 액추에이터 드라이버)
    F -->|전압/전류 물리적 구동력 주입| A

이 고리(Loop) 안에는 어떠한 실물 쇳덩어리도 존재하지 않지만, 수학적으로 주입된 기하학적 치차의 덜커덩거림(Backlash)과 센서의 열화 현상이 제어 코드의 강건성(Robustness)을 극한까지 시험한다.

3. 고장 모드 효과 분석(FMEA)과 결함 주입(Fault Injection)

기계 부품은 파괴(Fracture)되지만, 사이버물리시스템은 오동작(Malfunction)한다. 설계 초기 단계에서 자율 시스템의 신뢰성을 증명하기 위해, ISO 26262와 같은 기능 안전(Functional Safety) 표준은 엔지니어에게 의도적인 결함 주입(Fault Injection) 시뮬레이션을 강제한다. 무결한 SILS 루프 위에 의도적인 고장 스위치를 타격하여 제어 코드가 이를 안정적으로 회피 기동(Fail-Safe)하는지를 계측해야 한다.

주입되는 물리/센서의 주요 이산 고장 모드(Fault Mode)는 다음과 같다.

  1. 하드 폴트(Hard Fault): 센서의 파단이나 단선(Open Circuit)으로 인해 y_{sensor}(t) 가 즉발적으로 0, 혹은 최대 전압(5V)으로 고착화(Stuck-at)되는 절망적 파국.
  2. 소프트 폴트(Soft Fault): 미세한 드리프트(Drift)나 필터 커패시터 손상으로 인해 제어기가 인지하기 어려울 만큼 서서히 오차가 벌어지며 시스템을 발산시키는 교묘한 장애.
  3. 지터(Jitter) 폴트: CAN 혹은 이더넷(Ethernet)과 같은 차량 통신 버스의 대역폭 정체로 인하여 계측 신호가 간헐적으로 누락(Packet Loss)되거나 도착 순서가 역복원되는 시간 동기화 파괴.

4. 모델 예측망 기반의 잔차(Residual) 감지 및 페일 세이프(Fail-Safe)

센서에서 쏟아지는 결함을 제어 소프트웨어가 방어하는 유일한 해법은, 제어기 내부에 또 하나의 초소형 가상 플랜트 모델(Observer 혹은 Kalman Filter)을 심어두는 것이다.
제어 코드 내부의 관측기(Observer)는 센서로부터 수신한 왜곡된 계측값 y_{sensor} 에 의존하는 동시에, 이전 궤적을 토대로 도출한 수학적 예측값 \hat{y} 를 실시간 산출한다. 두 값의 편차인 잔차(Residual, r(t) = y_{sensor} - \hat{y})가 사전에 지정된 임계 매트릭스 \mathbf{Th} 를 폭파하는 순간, 제어기는 기존의 최적화 운전을 즉각 포기하라.

안전 모드(Limp-home Mode) 로직으로 즉시 하이브리드 천이를 감행하고, 가용한 나머지 센서들의 여분(Redundancy) 정보를 끌어모아 기계 장치를 파열 없이 멈춰 세우는 코드 아키텍처의 설계, 이것이 바로 SILS 환경에서 거수되는 진정한 디지털 예방 정비의 꽃이다. 하드웨어를 파괴하지 않고도 하드웨어의 파국을 재현해 내는 SILS의 통제 권한을 쥐어라.
