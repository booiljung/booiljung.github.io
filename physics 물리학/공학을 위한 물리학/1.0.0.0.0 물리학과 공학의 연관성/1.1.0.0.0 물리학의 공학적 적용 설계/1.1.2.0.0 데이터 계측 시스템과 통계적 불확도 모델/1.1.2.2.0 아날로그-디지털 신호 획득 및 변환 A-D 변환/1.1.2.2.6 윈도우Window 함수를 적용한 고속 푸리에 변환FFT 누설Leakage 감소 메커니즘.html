## 0.1 윈도우(Window) 함수를 적용한 고속 푸리에 변환(FFT) 누설(Leakage) 감소 메커니즘

복잡하게 엉켜있는 타임 도메인(Time Domain)의 진동 데이터를 거시적 주파수 도메인(Frequency Domain)으로 분해하여 해석하기 위해, 공학자들은 고속 푸리에 변환(Fast Fourier Transform, FFT)이라는 압도적인 수학적 텐서를 구동한다. 그러나 현실의 계측 시스템이 획득하는 데이터는 영원히 지속되는 무한 파동이 아니며, 샘플링 시간에 의해 강제로 앞뒤가 잘려나간 유한한 길이의 블록(Block) 단위로 파쇄되어 FFT 엔진에 주입된다. 이 역학적 절단 과정에서 파동의 시작점과 끝점이 완벽히 매끄럽게 맞물리지 않는 단절의 위계가 발생하며, 이로 인해 치명적인 주파수 해석 오류인 스펙트럴 누설(Spectral Leakage)이 터져 나오게 된다.

1. 파형 단절과 퓨리에 누설(Spectral Leakage) 참사
FFT 수학 알고리즘의 대전제는 “주입된 찰나의 시간 블록(예: 1초치 데이터)이 우주 끝까지 무한히 동일하게 반복(Periodic)된다“라고 가정하는 맹목성에 있다.

  • 만약 샘플링된 시간 블록 내에서 파동의 시작 위상(Phase)과 끝 위상의 진폭 궤적이 서로 어긋난 상태(비동기화 단절)라면, FFT 엔진은 이를 매끄러운 정현파가 아니라 ’날카롭게 찢어진 계단형 이음새’를 가진 극단적인 광대역 파동으로 파악해 버린다.
  • 이 어긋난 임펄스성 날이 빚어내는 환영 고주파들이 인접 주파수 대역으로 줄줄 새어나가(Leak), 진짜 뾰족해야 할 주파수 피크(Peak)를 넓게 뭉개버리고 주변의 미세한 스펙트럼 디테일을 노이즈 스모그로 덮어버리는 참사를 누설(Leakage) 이라 규정한다.

2. 윈도우 함수(Window Function)의 수학적 감쇠 렌더링
이 파괴적인 누설 에러망을 진압하기 위해, 타임 도메인의 데이터 블록을 FFT에 욱여넣기 직전, 블록의 시작점과 끝점 부근의 진폭 폭주를 강제로 0 에 수렴하도록 부드럽게 깎아내려 이음새를 억지로 봉합하는 수학적 덮개를 씌운다. 이 벨(Bell) 모양의 필터 텐서를 윈도우 함수(Window Function) 라 명칭한다.

  • 해닝(Hanning) 및 해밍(Hamming) 윈도우: 코사인(Cosine) 곡선의 파형을 응용하여 데이터 블록의 중앙부 진폭은 온전히 보존하되 절단면인 양 끝단을 부드럽게 영(Zero)으로 가라앉히는 가장 대중적인 윈도우다. 노이즈 바닥(Noise floor)을 신속하게 억제하여, 주파수 피크끼리 뭉개지는 것을 막아내는 우수한 선택성을 제공한다.
  • 플랫 탑(Flat-top) 윈도우: 주파수 피크의 진폭(Amplitude) 값을 칼같이 정확히 계측해야 하는 진동 캘리브레이션 렌더링에 배타적으로 사용된다.

따라서, 동역학적 기계 진동이나 복합 음향 렌더링을 계측하는 공학도는 무결한 주파수 밀도(PSD) 획득을 위해, 입력 데이터 파동의 특질에 걸맞은 가장 파괴력이 낮은 윈도우 알고리즘을 하드웨어 프로세싱 단계에서 선제적으로 곱해(Multiply) 넣어야만 한다.
