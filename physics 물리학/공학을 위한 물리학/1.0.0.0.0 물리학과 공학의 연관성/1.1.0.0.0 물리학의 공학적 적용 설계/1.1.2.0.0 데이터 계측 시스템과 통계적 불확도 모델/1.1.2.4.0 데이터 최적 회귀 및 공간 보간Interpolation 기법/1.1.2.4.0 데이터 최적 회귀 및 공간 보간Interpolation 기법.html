1.1.2.4 데이터 최적 회귀 및 공간 보간(Interpolation) 기법

물성 모형화나 센서 네트워킹을 통한 공학적 데이터 취득 과정에서, 엔지니어는 결코 우주처럼 무한하고 연속적인 물리계의 정보를 온전히 획득할 수 없다. 관찰자는 본질적으로 공간적, 시간적으로 파편화된 유한한 이산 데이터(Discrete Data) 점군(Point Cloud)만을 확보하게 된다. 이 불연속적인 데이터의 틈새를 수학적 통찰과 물리학적인 지배 방정식(Governing Equation)에 근거하여 연속적인 곡면이나 함수 스페이스로 이어 붙이고, 내재된 거시적 경향성을 추출하여 미지의 영역을 예측해 내는 기술 연립 체계가 곧 데이터 회귀(Regression) 및 공간 보간(Interpolation) 기법이다. 본 절에서는 이러한 제약된 관측을 무한의 연속 정보계로 격상시키는 해석적 메커니즘을 총체적으로 개관한다.

1. 회귀(Regression)와 보간(Interpolation)의 역학적 구별

이 두 가지 수학적 재구성 기법은 데이터를 처리하는 궤도와 물리학적 전제 조건에서 첨예한 차이를 지닌다.

  • 데이터 최적 회귀(Data Regression): 계측된 관측 데이터에는 백색 잡음(White Noise)과 같은 통계적 무작위 오차가 이미 짙게 배어 있음을 명백히 인정하는 기법이다. 회귀의 철학은 개별 데이터 점을 완벽하게 통과하려는 모순된 강박을 버리고, 전체 데이터의 군집이 가리키는 포괄적이고 본질적인 거시 역학 법칙(예: 물체의 관성에 의한 1차 선형 비례 관계, 냉각 법칙의 지수적 감소)을 찾아내어 시스템의 중심 뼈대 함수 형상을 전역적(Global)으로 최적화(Optimization)하는 데 있다.
  • 공간 보간(Spatial Interpolation): 반면 공학 해석 중 설계된 도메인에 분포한 노드(Node)나 유한요소 메쉬(Mesh) 위의 데이터처럼, 주어진 계측 값이 사실상 오차가 제거된 완벽한 진값(True Value)이라고 극도로 신뢰할 수 있는 구간에서 발동된다. 이 기법은 주어지지 않은 임의의 미지 좌표계의 값을 추정하되, 반드시 기 확보된 데이터 점 지점들을 오차 없이 정확하게 통과(Passing through)하는 유체적 국소(Local) 다차원 곡면이나 수학적 밧줄(스플라인)을 생성해 내는 것을 최우선 목표로 삼는다.

2. 데이터 재구성 기법의 물리학적 필요성

공학 현장에서 파편적인 데이터를 넘어 이산 좌표계 외곽의 정보까지 모델링을 감행해야 하는 당위성은 다방면으로 뻗쳐 있다.

  • 미지 영역의 지배 방정식 도출: 실험실에서 추출한 스프링의 인장 변위 대비 복원력 데이터를 단순한 점의 나열이 아니라 훅의 법칙(Hooke’s Law)이라는 본질적인 선형 방정식 함수, 즉 강성(Stiffness) 행렬 체계로 추상화하기 위해서는 회귀 기술이 필수 불가결하다.
  • 불균일 공간의 유체장 연속 맵 생성: 풍동 실험이나 연소실 내부에서 제한된 몇 개의 온도 및 압력 열전대 센서로 확보된 데이터를 토대로 전체 벽면과 입체의 연속적인 등고선(Contour) 구배를 완성하여 열응력 분포를 전산 모사하려면 고등 공간 보간 역학이 요구된다.
  • 파손 및 잡음 신호계 복원: 악천후나 전파 간섭으로 인해 드론 센서 네트워크나 레이더 전리층 수신 데이터 중 이가 빠진 소실 데이터(Missing Data) 블록을 주변 인접점들의 물리적 연속성(Continuity)과 스펙트럼 밀도 역학에 의지하여 역산 복원해 내기 위해 통달해야 한다.

엔지니어는 파편화된 데이터로 물리 시스템을 구현함에 있어 단순히 기존 선분(Linear Segment)으로 점들을 잇는 보간의 노예가 되어서는 안 된다. 시스템이 1차 선형인 최소자승법(Least Squares)의 회귀에 조응하는지, 노이즈를 척결하는 로버스트(Robust) 역학에 종속되는지, 또는 큐빅 스플라인 곡면을 거쳐 주파수 공간의 푸리에(Fourier) 복원으로 비상해야 하는지 날카롭게 재단하고 조립하는 수학적 선구안을 지녀야 한다. 이 섹션의 전개는 곧, 가려진 미지의 물리 영역을 수학이라는 랜턴으로 완전하게 밝혀내는 투영 역학의 대단원과 같다.
