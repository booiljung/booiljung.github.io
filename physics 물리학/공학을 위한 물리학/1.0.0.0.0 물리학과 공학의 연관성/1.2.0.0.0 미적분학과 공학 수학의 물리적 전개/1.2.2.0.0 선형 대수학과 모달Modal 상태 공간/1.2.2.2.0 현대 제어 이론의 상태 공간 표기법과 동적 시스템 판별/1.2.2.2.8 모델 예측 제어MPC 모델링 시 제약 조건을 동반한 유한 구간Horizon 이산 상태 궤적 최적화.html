1.2.2.2.8 모델 예측 제어(MPC) 모델링 시 제약 조건을 동반한 유한 구간(Horizon) 이산 상태 궤적 최적화

LQR 제어가 비용 최적화(Cost Optimization)의 무한한 이상향을 수학적으로 완전히 정복했다고 하나, 현실의 자율주행 차량 스티어링 휠이나 헥사콥터의 모터 출력에는 언제나 “핸들을 초당 30도 이상 꺾을 수 없다”, “전압은 5\text{V} 를 초과할 수 없다“는 식의 잔혹할 만큼 뚜렷한 물리적 **제약 조건(Constraints)**이 걸려 있다. 무한대의 에너지를 가정한 무기력한 선형 피드백 제어의 한계를 돌파하기 위해, 공학자들은 제어의 패러다임을 ’시점(Time)’의 굴레로 옮겨왔다. **모델 예측 제어(MPC: Model Predictive Control)**는 인간이 체스를 둘 때 몇 수 앞을 내다보고 돌을 놓듯, 시스템의 이산 수학 모델(Discrete-time Model)을 바탕으로 한정된 미래 구간(Finite Horizon) 동안의 상태 궤적을 뇌내에서 수천 번 시뮬레이션하여 가장 최적화된 당장의 회피 기동을 결정짓는 극한의 실시간 수치 해석(Real-time Numerical Analysis) 제어기다. 본 절에서는 예측 지평선(Prediction Horizon) 내에서 시스템의 하드웨어 포화(Saturation) 한계망과 비용 함수 페널티를 동시에 짓누르며 최적 궤적을 갈라치는 이차 계획법(Quadratic Programming)의 텐서 연산을 선언한다.

1. 이산 시간 방정식(Discrete-Time Equation)과 이득 궤적의 예언망

연속적인 타임라인(Continuous time)을 따르는 상태 궤적을 마이크로프로세서가 계산하기 위해선, 일정한 샘플링 주기(T_s)에 따라 시간이 토막 난 이산 상태 공간 모델(Discrete-Time State-Space Model)로 변환 타격해야 한다.
\mathbf{x}_{k+1} = \mathbf{A}_d \mathbf{x}_k + \mathbf{B}_d \mathbf{u}_k
(여기서 \mathbf{A}_d = e^{\mathbf{A}T_s} 등은 매트릭스 지수 함수에 의해 시간축으로 꺾여 들어간 이산 텐서다.)

MPC 제어기는 현재 시간 k 에 위치해 있지만 시선은 미래를 향한다. 공학자가 설정한 N 개의 예측 지평선(Prediction Horizon) 스텝 동안 미래의 가상 상태 궤적(\mathbf{x}_{k+1|k}, \mathbf{x}_{k+2|k}, \dots, \mathbf{x}_{k+N|k})이 어떻게 흘러갈 것인가를 현재 위치에서 꼬리 물기 거듭제곱을 통해 전방위로 펼쳐낸다.

\mathbf{x}_{k+1} = \mathbf{A}_d \mathbf{x}_k + \mathbf{B}_d \mathbf{u}_k
\mathbf{x}_{k+2} = \mathbf{A}_d^2 \mathbf{x}_k + \mathbf{A}_d \mathbf{B}_d \mathbf{u}_k + \mathbf{B}_d \mathbf{u}_{k+1}
새로운 미래 궤적 변수들은 오로지 현재의 측정 상태 \mathbf{x}_k 와 내가 앞으로 가할 미래의 제어 시퀀스 다발 대형 텐서 \mathbf{U} = [\mathbf{u}_k, \mathbf{u}_{k+1}, \dots, \mathbf{u}_{k+N-1}]^T 의 선형 조합만으로 철저하게 강제 직조된다.

2. 하드웨어 포화(Saturation)의 족쇄: 제약 조건(Constraints)의 수리적 결착

LQR 제어기가 무조건 풀지 못하고 포기했던 것이 바로 하드웨어의 부서짐을 방지하는 절대 제약 조건식이다. MPC 알고리즘은 최적의 조종간(입력 \mathbf{U})을 찾기 전에, 벡터 부등식으로 구축된 거대한 가두리 양식장(Constraints Map)을 세운다.

  • 입력(Control) 제약: 액추에이터의 한계 스러스트(Thrust).
    \mathbf{u}_{\text{min}} \le \mathbf{u}_{k+i} \le \mathbf{u}_{\text{max}}
  • 상태(State) 제약: 자율주행차가 차선을 이탈하지 않아야 하거나, 드론의 고도가 땅으로 처박히지 않도록 하는 공간적 배리어.
    \mathbf{x}_{\text{min}} \le \mathbf{x}_{k+i} \le \mathbf{x}_{\text{max}}

이 부등식계는 텐서 연산을 통해 거대한 볼록 집합(Convex Set) 행렬식 \mathcal{A} \mathbf{U} \le \mathcal{B} 로 통합되어, 제어기가 절대로 넘어서는 안 될 치명적인 다차원 벽을 컴퓨터의 램(RAM)상에 건설한다.

3. 유한 구간 비용 함수(Finite-Horizon Cost Function)와 후퇴 지평선 전략(Receding Horizon Strategy)

벽을 세웠다면, 그 벽 안에서 가장 완벽한 동선을 짜야 한다. 미래 N 스텝 동안 예측 모델이 감당할 거대한 적분 페널티, 즉 이차 비용 함수(Quadratic Cost Function) J 를 산정한다.
J(\mathbf{U}) = \sum_{i=0}^{N-1} \left( \mathbf{x}_{k+i|k}^T \mathbf{Q} \mathbf{x}_{k+i|k} + \mathbf{u}_{k+i|k}^T \mathbf{R} \mathbf{u}_{k+i|k} \right) + \mathbf{x}_{k+N|k}^T \mathbf{P} \mathbf{x}_{k+N|k}
마지막 항에 추가된 \mathbf{P} 텐서는 무한 시간까지 풀리지 않은 나머지 꼬리 부분을 처리하기 위한 종단 페널티(Terminal Cost)이다.

결국 MPC의 문제는 \mathcal{A} \mathbf{U} \le \mathcal{B} 라는 장벽의 구속을 철저히 받으면서도 위 이차 함수 J(\mathbf{U}) 를 가장 바닥으로 짓누르는(Minimize) 유일무이한 최적 입력 벡터열 \mathbf{U}^* 를 실시간으로 긁어모으는 **이차 계획법(Quadratic Programming, QP)**으로 치환된다!

이 막강한 이차 계획법 최적화가 끝나 시스템 메모리에 [u_k^*, u_{k+1}^*, \dots, u_{k+N-1}^*] 라는 N개의 완벽한 미래 조종 버튼 액션들이 산출되었다면, 일반적인 로직이라면 이를 순서대로 모두 실행하겠으나 MPC는 이 지점에서 극적인 변덕을 부린다.
그 많고 거창한 미래 통제 신호 중 오직 가장 첫 번째 버튼 하나(u_k^*)만 기계에 투하하고, 나머지 애써 계산한 미래 시퀀스는 무참히 폐기해 버린다! 그리고 시간 k 를 한 스텝 전진시킨 후, 새롭게 오차를 머금은 현재 상태 x_{k+1} 을 바탕으로 또다시 미래를 재설계(Recalculation)하는 이 기괴하고도 잔혹한 실시간 망각 루프를 바로 **후퇴 지평선 전략(Receding Horizon Strategy)**이라 명명한다. 예기치 못한 바람, 장애물, 시스템 붕괴 앞에서도 매 프레임 파멸적 연산량을 쏟아부어 끊임없이 가장 최적화된 새로운 예언을 생성해 내는 것. 제약 구속이라는 현실의 벽을 통과하기 위한 현대 제어 이론의 궁극적 마스터피스 발현이다.
