센서(Sensor) 구성: 카메라, 라이다, IMU 등¶

Isaac Sim에서 센서를 구성하는 과정은 가상 환경에서 로봇의 주변 인식 능력을 향상하고, 나아가 ROS2와의 상호 작용을 통해 실제 로봇 시스템과 유사한 상황을 시뮬레이션하기 위한 핵심이다. 센서는 주로 카메라, 라이다, IMU 등을 사용하며, 각 센서는 물리 엔진과 결합되어 가상 공간에서 실제와 유사한 데이터를 출력한다. 이 장에서는 센서 구성 기초부터 고급 설정까지 폭넓게 다룬다.

Isaac Sim의 Extension 기능을 사용하면 다양한 센서를 손쉽게 추가할 수 있다. 예를 들어 카메라 센서의 경우 뷰포트 기반 렌더링을 이용해 RGB, 심도(Depth), 세그멘테이션 등 여러 종류의 영상 정보를 동시에 획득할 수 있다. 라이다 센서는 가상 레이캐스팅을 통해 점군(Point Cloud) 정보를 생성하고, IMU는 가상 관성 측정 데이터를 제공하여 로봇 동작 제어와 내비게이션 등에 활용할 수 있다.

카메라 센서를 구성할 때 가장 기본적인 모델은 바늘구멍(핀홀) 카메라 모델이다. 이 모델을 통해 내부 파라미터(초점거리, 광학중심 등)와 외부 파라미터(월드 좌표계 상의 위치, 자세)를 정의한다. 시뮬레이션 환경에서 카메라의 기하학적 관계를 수식으로 표현하면 다음과 같다.

\[ \begin{align} z \begin{pmatrix} u \\ v \\ 1 \end{pmatrix} &= \mathbf{K} \begin{pmatrix} x \\ y \\ z \end{pmatrix} \end{align} \]

여기서 \(u, v\)는 이미지 평면 좌표, \(x, y, z\)는 월드(또는 카메라) 좌표계에서의 물리적 좌표, \(\mathbf{K}\)는 내부 파라미터 행렬이며, \(z\)는 스케일 팩터이다. Isaac Sim에서 이 내부 파라미터와 카메라 위치, 회전 등을 적절히 설정해주면, 물리적으로 일관성 있는 이미지를 획득할 수 있다.

라이다 센서는 시뮬레이션 환경 내에서 360도 또는 특정 각도 범위를 레이캐스팅하여, 빔이 충돌하는 지점까지의 거리를 측정함으로써 점군 데이터를 생성한다. 라이다의 일반적인 레인지 방정식을 간단히 표현하면 아래와 같다.

\[ \begin{align} d &= \|\mathbf{p_{hit}} - \mathbf{p_{origin}}\| \end{align} \]

여기서 \(\mathbf{p_{origin}}\)은 라이다 빔이 발사되는 위치, \(\mathbf{p_{hit}}\)는 빔이 충돌한 지점이며, \(d\)는 측정된 거리이다. Isaac Sim에서는 레이캐스팅의 해상도, 스캔 횟수, 각도 범위 등의 파라미터를 조정할 수 있고, 이러한 설정값은 ROS2의 LaserScan 혹은 PointCloud2 메시지 형태로 전달되어 후속 처리에 활용된다.

IMU 센서는 가속도와 각속도(또는 고급 옵션으로는 자기장까지) 데이터를 시뮬레이션한다. Isaac Sim에서 가상 관성을 정의할 수 있으며, 중력, 가속, 회전 등에 대한 시뮬레이션 결과가 ROS2의 IMU 메시지 센서 토픽으로 연결된다. IMU 메시지는 다음과 같이 선형 가속도, 각속도, 자세(쿼터니언) 정보를 담는다.

센서를 Isaac Sim에 구성한 뒤, ROS2와 연동하려면 ROS2 Extension을 활성화하고 센서별 토픽을 지정해야 한다. 예를 들어 카메라 센서의 경우 /camera/rgb, 라이다 센서의 경우 /lidar/points, IMU의 경우 /imu/data 등의 형태로 설정 가능하다. ROS2에서 시뮬레이션 데이터를 구독(subscribe)하는 코드는 다음과 같이 작성할 수 있다.

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import Image, PointCloud2, Imu

class SensorSubscriber(Node):
    def __init__(self):
        super().__init__('sensor_subscriber')
        self.create_subscription(Image, '/camera/rgb', self.image_callback, 10)
        self.create_subscription(PointCloud2, '/lidar/points', self.lidar_callback, 10)
        self.create_subscription(Imu, '/imu/data', self.imu_callback, 10)

    def image_callback(self, msg):
        # 카메라 이미지 처리 코드
        pass

    def lidar_callback(self, msg):
        # 라이다 점군 처리 코드
        pass

    def imu_callback(self, msg):
        # IMU 데이터 처리 코드
        pass

def main(args=None):
    rclpy.init(args=args)
    node = SensorSubscriber()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Isaac Sim에서는 센서 간 상호작용을 확인하거나, ROS2의 TF(Transform) 트리를 활용해 관성기준좌표계와 센서좌표계를 잇는 가상 캘리브레이션 절차도 실험할 수 있다. 예를 들어, 카메라와 라이다를 합성하여 RGB-D 데이터를 구성하거나, IMU와 라이다를 결합해 관성항법 기반 SLAM을 테스트하는 시나리오도 구현 가능하다.

센서 구성을 마친 뒤에는 더욱 고급 활용으로써 멀티 카메라 시스템, 멀티 라이다 병합, IMU 기반 동적 이동 물체 추적 등이 가능하다. 가령, 멀티 카메라의 내·외부 파라미터를 정밀하게 설정하고, ROS2에서 메시지 동기화(sync)를 실행하여 스테레오 매칭이나 3D 재구성을 시도할 수 있다. 나아가 라이다와 IMU를 결합한 SLAM 시스템, 카메라·라이다·IMU가 함께 동작하는 Sensor Fusion 시스템도 Isaac Sim에서 검증할 수 있다.

이처럼 Isaac Sim에서 센서 구성을 제대로 이해하고 설정해두면, 물리적 환경에서 발생할 수 있는 다양한 시나리오를 가상 공간에서 재현할 수 있다. 이는 ROS2를 사용한 자율주행, 로봇암 제어, 산업용 로봇 시뮬레이션 등 폭넓은 분야에서 매우 중요한 기반이 된다.
