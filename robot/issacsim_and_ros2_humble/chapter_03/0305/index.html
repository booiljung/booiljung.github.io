물리 엔진의 기본 이해와 Isaac Sim의 특징¶

Isaac Sim은 NVIDIA Omniverse 기반에서 동작하며, 다양한 물리 엔진 기능을 제공한다. 기본적으로 PhysX가 탑재되어 강체(Rigid Body)부터 복합적인 충돌(Contacts), 조인트(Joints), 유체 시뮬레이션 등 여러 기능을 확장할 수 있다. ROS2 Humble과 연동하면 시뮬레이션 결과를 실시간으로 퍼블리시하거나, ROS2 토픽으로 제어 명령을 전달할 수 있다.

Isaac Sim 내부 물리 엔진의 핵심은 뉴턴 운동법칙에 따라 물체의 움직임을 계산하는 것이다. 가장 간단한 예로서 질량이 \(m\)인 물체에 작용하는 힘 \(\mathbf{F}\)가 있을 때, 가속도 \(\mathbf{a}\)는 다음과 같다.

\[ \begin{align} \mathbf{F} = m \mathbf{a} \\ \end{align} \]

또한 물체가 회전하는 경우, 관성 행렬 \(\mathbf{I}\)와 각가속도 \(\mathbf{\alpha}\)가 주어졌을 때 토크 \(\mathbf{\tau}\)는

\[ \begin{align} \mathbf{\tau} = \mathbf{I} \mathbf{\alpha} \\ \end{align} \]

와 같이 정의된다. Isaac Sim은 이러한 기본 방정식을 물리 엔진 내부에서 수치적 통합(Numerical Integration) 알고리즘을 통해 연산한다.

시뮬레이션 환경과 좌표계 설정¶

Isaac Sim에서의 시뮬레이션 환경을 구성할 때, 좌표계를 어떻게 설정하느냐가 중요하다. 보통 월드(World) 좌표계에서 중력은 \(-\mathbf{z}\) 방향으로 설정된다. 만일 다른 축을 중력 방향으로 하고자 한다면 Isaac Sim의 Physics Settings에서 Gravity 벡터를 재설정한다. 예컨대 지구 표면 근처에서 약 \(9.81 ,\text{m/s}^2\)의 중력가속도를 적용하려면, 월드 좌표계의 \(z\)축 음의 방향으로 이 값을 넣는다.

Isaac Sim에서는 회전 좌표계와 트랜스폼(Transform)도 쉽게 제어할 수 있다. 물리 시뮬레이션에 참여하는 오브젝트의 위치와 회전값은 ROS2 TF(Transform)와 연결할 수도 있는데, 이를 통해 실제 로봇 시스템과 유사한 좌표 관계를 유지한다.

기본 물리 파라미터 설정¶

Isaac Sim은 월드 전체적으로 또는 개별 오브젝트마다 물리 파라미터를 세부 조정할 수 있다. 파라미터 예시로는 Time Step, Sub Steps, Solver Iteration Count, 마찰(Friction), 반발계수(Restitution) 등이 있다. 마찰은 물체 간의 상대 운동을 제어하는 중요한 요소이며, 기본적으로 쿨롱(Coulomb) 마찰 모형에 기초한다. 예를 들어 물체가 지면에 놓였을 때 최대 정지 마찰력 크기는

\[ \begin{align} F_\text{max} = \mu_s N \\ \end{align} \]

으로 나타낼 수 있다. 여기서 \(\mu_s\)는 정지 마찰 계수, \(N\)은 법선 방향의 반작용 힘(예: 중력에 의한 지면 반작용)이다. Isaac Sim에서는 이 마찰 계수를 Static Friction, Dynamic Friction 등으로 구분하여 설정할 수 있다.

시간 통합(Time Integration)은 시뮬레이션 정확도와 실시간성의 균형을 맞추는 핵심 요소다. Time Step이 너무 크면 오브젝트 움직임이 부정확해지고, 너무 작으면 계산량이 급증하여 시뮬레이션 속도가 저하된다. Isaac Sim은 서브 스텝(Sub Steps) 방식을 사용하여 높은 안정도를 유지하는데, 한 프레임당 여러 번의 물리 업데이트를 수행함으로써 충돌이나 빠른 움직임 상황에서도 정확도를 높일 수 있다.

Isaac Sim에서의 물리 설정 GUI와 스크립트 제어¶

Omniverse Isaac Sim에는 기본 Physics 설정을 제어할 수 있는 GUI 인터페이스가 있으며, 이를 통해 손쉽게 중력, 타임스텝, 마찰계수 등을 설정한다. 동시에 Python 스크립트를 사용해 프로그램matically 세팅도 가능하다. 다음은 Isaac Sim 내부의 확장 스크립트로 물리 파라미터를 간단히 변경하는 예시다.

import omni

physics_interface = omni.physx.get_physx_interface()
settings = physics_interface.get_physics_settings()

settings.gravity = (0.0, 0.0, -9.81)
settings.default_static_friction = 0.8
settings.default_dynamic_friction = 0.5
settings.time_steps_per_second = 60
settings.sub_steps_per_frame = 4

이렇게 Python 스크립트를 활용하면 여러 환경에서 일관된 물리 파라미터를 적용할 수 있으며, ROS2와 연동할 때도 자동화 파이프라인을 구축하기 쉽다.

충돌(Contacts) 설정과 솔버(Solver) 튜닝¶

물체가 충돌할 때 접점(Contact) 정보를 어떻게 계산하고, 이를 물리 엔진이 어떤 방식으로 풀어나가는지는 시뮬레이션 정확도에 큰 영향을 준다. Isaac Sim의 PhysX 솔버는 PGS(Projected Gauss-Seidel) 방식 등을 사용하며, 접점 계산 정확도를 높이기 위한 접촉 오프셋(Contact Offset), Rest Offset 등의 매개변수가 있다.

솔버 반복 횟수(Iterations)를 늘리면 접점 해가 더 정확해지는 반면, 계산 비용이 증가한다. 고급 시뮬레이션에서는 조인트를 많이 사용하는 로봇 조인트, 줄(케이블) 시뮬레이션 등의 복합 시스템을 다룰 때 솔버 튜닝이 필수적이다. Isaac Sim에서는 Joint Compliance, Joint Friction 등을 GUI로도 설정할 수 있으며, 이를 통한 조인트 움직임 제어가 가능하다.

고급 물리 기능 활용¶

Isaac Sim은 CPU 기반 물리뿐 아니라 GPU 가속 물리(Fast Soft Body Simulation, Contact Simulation 등)도 지원한다. Cloth(직물), Soft Body, 유체(Fluid) 등의 시뮬레이션은 고급 엔진 플러그인으로 확장 가능하다. 이런 기능들은 ROS2와 함께 복잡한 로봇 상호 작용이나 시뮬레이션 환경을 구축할 때 유용하다.

GPU 가속 기능을 사용하려면 NVIDIA GPU가 필요하며, Isaac Sim의 확장(Extension)에서 해당 기능을 활성화해야 한다. 이렇게 물리를 GPU에서 처리하면 대규모 시뮬레이션 환경에서도 상대적으로 빠른 계산이 가능하다.

아래는 Isaac Sim 내부 구성과 물리 계산 흐름을 단순화한 도식이다.

flowchart LR A[Scene Objects<br>Meshes, Lights] --> B(Physics Engine<br>Omniverse Isaac Sim) B --> C[CPU/GPU Simulation<br>PhysX or Extensions] C --> D[Simulation Results] D --> E[ROS2 Topics / Rendering]

이 도식에서 Scene Objects는 Mesh, Light 등이 포함된 3D 세계를 의미하며, Physics Engine을 통해 CPU 또는 GPU 자원을 활용하여 시뮬레이션을 수행한 뒤, 최종 결과를 화면에 렌더링하거나 ROS2 Topics로 퍼블리시하여 실제 로봇 프로그램과 상호작용할 수 있다.

---와 주의사항

Isaac Sim에서 물리 시뮬레이션을 정확히 설정하기 위해서는 기본 물리 파라미터(Time Step, 마찰, Restitution)를 먼저 조정하고, 로봇이나 오브젝트에 맞춰 솔버 세부 옵션을 튜닝하는 과정이 필수다. 물체의 충돌 경계(Collision Shape)가 실제 형상과 맞지 않으면 충돌 계산이 부정확해지므로, Isaac Sim의 컬라이더(Collider) 생성 시 Mesh Simplification 옵션 등을 통해 물리 충돌 영역을 정교하게 만드는 작업이 필요하다.

ROS2 Humble과 연동 시에는 시뮬레이션이 실시간성을 만족하도록 Time Step을 너무 크게 설정하면 오버슈트(over-shoot)가 발생할 수 있으므로 적절한 서브 스텝(Sub Steps) 조정이 권장된다. 추가로, ROS2의 주기와 Isaac Sim의 물리 업데이트 간격이 어긋나지 않도록 상호간 주기를 맞추는 작업도 중요하다.

이상으로 Isaac Sim에서 물리 시뮬레이션을 설정하는 기본부터 고급 기능까지 개괄하였다. 실제 프로젝트 환경에 따라 파라미터를 세밀하게 튜닝하여, ROS2 로봇과 시뮬레이션 환경이 일관되게 동작하도록 구성하기 바란다.
