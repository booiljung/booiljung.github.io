Isaac Sim에서 Asset 활용 개요¶

Isaac Sim에서 3D 모델(Asset)은 시뮬레이션 환경을 구성하는 핵심 요소다. 기본적으로 Omniverse Nucleus 서버의 라이브러리나 로컬 폴더에 저장된 USD, FBX, OBJ 등의 다양한 3D 형식을 불러올 수 있다. Isaac Sim은 Omniverse의 강력한 USD(Universal Scene Description) 기반 워크플로우를 제공하여, 모델의 기하 구조·재질·물리 속성을 간단하게 관리하고 시뮬레이션에 적용할 수 있다.

3D 모델을 불러올 때 필요한 정보에는 객체의 기하 구조(메시), 컬라이더(collider), 물리 재질(physics material) 등이 포함된다. 특히 물리 시뮬레이션을 수행할 경우에는 질량·마찰 계수 같은 추가 속성도 고려해야 한다. 이러한 속성들은 모두 USD 파일 내에 정의될 수도 있고, Isaac Sim에서 개별적으로 설정할 수도 있다.

Asset 불러오기¶

Isaac Sim을 실행한 뒤, 상단 메뉴나 뷰포트에서 다양한 방식으로 Asset을 불러올 수 있다. 대표적으로 Content 창, Omniverse Nucleus 연결, 로컬 파일 임포트 등이 있다. Omniverse Nucleus 서버를 사용하면 여러 사용자가 공용 스토리지를 통해 손쉽게 협업할 수 있으며, Isaac Sim의 Content 브라우저를 통해 해당 서버에 저장된 Asset을 끌어와서 즉시 활용할 수 있다.

외부 3D 모델을 불러오려면 Isaac Sim의 메뉴에서 File → Import → “USD / FBX / OBJ” 형식을 선택하거나, Content 브라우저에서 원하는 파일을 드래그앤드롭한다. 이때 일부 모델 형식은 USD로 자동 변환이 이뤄지며, 메시뿐만 아니라 계층 구조(계층 트리)도 반영된다.

셸 명령으로 Isaac Sim을 구동한 후 스크립트 형태로 Asset을 불러오고 싶다면 아래와 같이 수행할 수 있다.

./isaac-sim.sh --allow-root --ext folder/to/extension

Python 스크립트 내부에서 매크로나 API를 이용해 USD 파일을 스테이지에 불러올 수도 있다.

from omni.isaac.core import World
from omni.isaac.core.utils.stage import open_stage

world = World()
open_stage("omniverse://localhost/NVIDIA/Assets/Example/Chair.usd")

world.reset()
world.run()

위 예제에서는 open_stage 함수를 통해 원하는 USD 파일을 Isaac Sim 스테이지로 불러온다.

Asset 배치 및 설정¶

Asset을 스테이지로 불러온 뒤에는 위치·회전·크기 등의 좌표계를 설정해주어야 한다. Isaac Sim은 기본적으로 우측 트랜스폼(Transform) 창을 통해 각각의 파라미터를 수치 입력으로 조정할 수 있다.

배치의 정확한 정의를 위해 3차원 공간 상에서의 좌표 변환 과정을 살펴본다. 3차원에서 임의의 점을 \(\mathbf{p}_{i}\)라고 할 때, 해당 점에 대한 변환(회전·병진·확장)은 다음과 같은 4×4 동차 좌표계 행렬 \(\mathbf{T}\)로 표현할 수 있다.

\[ \begin{align} \mathbf{T} = \begin{bmatrix} 1 & 0 & 0 & x \\ 0 & 1 & 0 & y \\ 0 & 0 & 1 & z \\ 0 & 0 & 0 & 1 \end{bmatrix} \end{align} \]

\(x, y, z\)는 병진(translation)을 나타내며, 회전(rotation)을 포함한 일반적인 변환 행렬은 오일러 각이나 쿼터니언으로부터 유도된다. Isaac Sim 상에서 에디터로 이 값들을 조정하면, 해당 Asset이 월드(World) 좌표계에서 어떤 위치와 자세로 놓일지를 결정하게 된다.

에디터로 배치한 후에 물리 속성을 부여하거나, 충돌(콜리전) 형태를 설정하기 위해서는 Property 창에서 Collider 옵션을 활성화하고 재질(Material)을 선택해야 한다. 예컨대 바닥이나 벽과 상호작용해야 하는 객체라면 Rigid Body 속성과 Collider를 세팅하여 물리 시뮬레이션에서 올바른 충돌 검사와 동작을 수행하도록 한다.

고급 사용 방법¶

Asset을 불러온 뒤 물리 시뮬레이션에서 동적으로 생성하거나, 시뮬레이션 도중 위치를 변경해야 하는 경우에는 스크립팅이나 확장(Extension)을 통해 자동화할 수 있다. Isaac Sim은 Python 기반의 확장 포인트를 지원하므로, 프로그래밍 방식으로 여러 Asset을 일괄 배치하거나 프로시저럴(Procedural)하게 생성하는 시나리오를 구축하기 쉽다.

아래 예시는 Isaac Sim에서 Python API로 여러 개의 Asset을 반복적으로 생성하고, 일정 간격으로 배치하는 예시 코드다.

import omni

# Stage 참조
stage = omni.usd.get_context().get_stage()

# 모델 파일 경로
asset_path = "omniverse://localhost/NVIDIA/Assets/Example/Box.usd"

# 여러 개의 Asset을 자동 생성
for i in range(3):
    new_prim_path = f"/World/Box_{i}"
    prim = stage.DefinePrim(new_prim_path, "Xform")
    prim.GetReferences().AddReference(asset_path)

    # 트랜스폼 설정: x축 방향으로 i * 100만큼 이동
    xform_api = omni.usd.get_editor_interface().get_xform_api(prim)
    xform_api.set_translate([float(i*100), 0.0, 0.0])

메시가 복잡하거나 물리 상호작용이 많은 장면에서 충돌 형태를 단순화하거나 LOD(Level of Detail)를 설정하여 퍼포먼스를 높이는 것도 중요하다. Isaac Sim에서 Collider 옵션을 단순 기하(큐브, 캡슐 등)로 설정하면 물리 연산 부하를 줄일 수 있다.

아래와 같은 순서로 3D 모델을 가져와 시뮬레이션에 투입되는 과정을 시각화하면 이해에 도움이 된다. (mermaid 사용 예시)

flowchart TB A[Omniverse Nucleus / Local] --> B[Isaac Sim 스테이지로 Asset 불러오기] B --> C[트랜스폼 및 물리 파라미터 설정] C --> D[시뮬레이션 실행]

위 다이어그램에서 보이듯이 3D 모델은 Omniverse Nucleus나 로컬 폴더 등에서 가져온 뒤, Isaac Sim 스테이지에서 위치·크기·회전·물리 속성 등을 조정하고 시뮬레이션을 구동하는 단계를 거친다. 중간 단계에서 USD 파일의 구조를 직접 편집하거나, 프로그래밍적 스크립트로 배치·생성을 자동화해 대규모 환경을 구성할 수도 있다.

이와 같이 3D 모델(Asset) 불러오기와 배치는 Isaac Sim으로 환경을 구축하는 데 있어 필수적인 과정이다. 기본적인 단계부터 고급 스크립팅 및 물리 설정까지 숙지하면 ROS2와의 연동 시뮬레이션에서도 훨씬 유연하고 빠르게 작업을 진행할 수 있다.
