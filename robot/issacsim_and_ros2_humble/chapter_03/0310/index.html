개요¶

Isaac Sim은 NVIDIA의 Omniverse 플랫폼 위에서 작동하는 강력한 시뮬레이션 환경이다. 물리 기반 시뮬레이션(Physics Simulation), 3D 렌더링, ROS2 연결, AI 모델 연동 등이 유기적으로 결합되어 있어 로봇 개발에 활용하기 매우 적합하다. Isaac Sim은 Omniverse의 확장(Extension) 형태로 제공되는 많은 API를 통해 파이썬(Python) 스크립팅을 지원한다. 이 파이썬 API를 이해하면 Isaac Sim 내부에서 원하는 대로 장면(Scene)을 구성하고, 객체를 동적으로 제어하며, 센서 데이터를 확인하고, 시뮬레이션 파이프라인을 자동화할 수 있다.

Isaac Sim 스크립팅을 익히는 과정은 다음과 같은 흐름으로 생각하면 이해하기 쉽다.

  • Isaac Sim 내부 구조 및 환경 설정
  • Isaac Sim API 및 Extension 구조 이해
  • 기초 예제를 통해 개념 정립
  • 고급 기법(Extension 제작, 복잡한 물리 제어, ROS2와의 연동)

아래에서는 이 흐름을 차근차근 살펴보고, Python 스크립트를 사용해 Isaac Sim을 제어하는 실질적인 방법을 다룬다.

Isaac Sim Python 환경 설정¶

Isaac Sim은 기본적으로 Omniverse Kit 기반에서 동작하며, 별도의 파이썬 인터프리터를 포함한다. Isaac Sim이 설치된 디렉터리 내부에는 자체 Python 환경이 존재하므로 일반 파이썬 환경과 격리되어 있을 수 있다. Isaac Sim 환경 설정 시 다음과 같은 방식으로 Isaac Sim의 파이썬을 사용한다.

# Isaac Sim 폴더 내부에 존재하는 Python 인터프리터로 진입
cd /path/to/isaac_sim
./python.sh

이후 Isaac Sim이 제공하는 옴니(omni) 모듈과 Isaac Sim 전용 라이브러리를 import하여 사용할 수 있다.

Isaac Sim API 구조¶

Isaac Sim에서 제공하는 API는 크게 다음 영역으로 구성된다.

  • Omniverse Kit 기반 모듈: omniverse 엔진 동작에 필요한 옴니(omni) 네임스페이스의 모듈(omni.usd, omni.ui 등).
  • Isaac Sim 전용 모듈: 로봇 및 물리 시뮬레이션, 센서 시뮬레이션, 확장(Extension) API 등(omni.isaac.core, omni.isaac.sensor 등).

이 모듈들을 적절히 import하면 장면을 불러오거나, 새로운 객체를 생성하거나, 물리 엔진 파라미터를 조절하는 등 폭넓은 기능을 수행할 수 있다.

예를 들어, 가장 기초적인 Isaac Sim API 활용 방법은 다음과 같이 장면(Scene) 및 Stage 객체를 가져오는 것이다.

import omni
from omni.isaac.kit import SimulationApp

# Isaac Sim 환경 시작
simulation_app = SimulationApp({"headless": False})

# 현재 Stage (USD 파일이 로드되는 3D 장면) 가져오기
stage = omni.usd.get_context().get_stage()

위 코드는 Isaac Sim의 SimulationApp을 활성화하고, 현재 USD Stage에 접근하여 다양한 오퍼레이션을 수행할 준비를 마친 상태를 만든다.

예제: Python 스크립트 기초¶

Isaac Sim에서 가장 기본적인 동작은 USD(Stage)에 새로운 프리미티브(기하 객체)를 추가하고, 위치와 방향(트랜스폼)을 설정하는 작업이다. 예를 들어, 큐브(Cube)를 추가해보자.

import omni
from omni.isaac.kit import SimulationApp
from pxr import UsdGeom, Gf

simulation_app = SimulationApp({"headless": False})
stage = omni.usd.get_context().get_stage()

# Cube 생성
cube_path = "/World/Cube"
UsdGeom.Cube.Define(stage, cube_path)

# Transform 설정 (위치와 회전, 스케일)
cube_prim = stage.GetPrimAtPath(cube_path)
xform = UsdGeom.Xformable(cube_prim)
xform.ClearXformOpOrder()

# 위치 이동 (Px, Py, Pz)
translate_op = xform.AddTranslateOp()
translate_op.Set(Gf.Vec3f(100, 0, 0))

# 시뮬레이션 종료 시까지 대기
simulation_app.update()
simulation_app.stop()

위 코드에서 UsdGeom.Cube.Define을 통해 "/World/Cube" 경로에 큐브를 새로 정의하고, Xformable 인터페이스를 활용해 위치(Transform)를 설정했다. Isaac Sim 내에서는 모든 객체가 USD(Universal Scene Description) 기반으로 관리되며, prim(primitive)이라는 단위로 구별되어 있다.

좌표계 변환과 행렬 연산¶

Isaac Sim에서 3D 공간 내 객체의 위치와 자세(Pose)를 다룰 때 종종 변환 행렬(Transformation Matrix)을 사용하게 된다. 3차원 공간에서 한 객체의 위치와 회전을 나타내기 위해 아래와 같은 4×4 변환 행렬을 고려할 수 있다.

\[ \begin{align} \mathbf{T} = \begin{bmatrix} \mathbf{R} & \mathbf{p} \\ 0 & 1 \end{bmatrix} \\ \end{align} \]

여기서 \(\mathbf{R}\)은 3×3 회전 행렬, \(\mathbf{p}\)는 3×1 위치 벡터이다. Isaac Sim의 Xform 연산은 내부적으로 이러한 변환 행렬 연산을 처리하여 prim의 위치와 회전을 적용한다. 물리 시뮬레이션(PhysX 등)을 통해 로봇이나 객체가 움직이면, 시뮬레이션 엔진이 실시간으로 변환 행렬을 갱신한다.

물리 시뮬레이션 연동¶

Isaac Sim의 큰 장점 중 하나는 PhysX 기반 물리 엔진을 쉽게 스크립팅으로 접근할 수 있다는 것이다.

새로 생성된 prim에 물리 속성을 부여하려면 RigidBody, Collider, Shape 등을 설정해야 한다. 다음 코드 예시는 간단히 BoxCollider와 RigidBody를 추가하는 방법을 나타낸다.

import omni
from omni.isaac.kit import SimulationApp
from pxr import UsdGeom, Gf, PhysxSchema

simulation_app = SimulationApp({"headless": False})
stage = omni.usd.get_context().get_stage()

cube_path = "/World/DynamicCube"
UsdGeom.Cube.Define(stage, cube_path)

cube_prim = stage.GetPrimAtPath(cube_path)
xform = UsdGeom.Xformable(cube_prim)
xform.ClearXformOpOrder()

translate_op = xform.AddTranslateOp()
translate_op.Set(Gf.Vec3f(0, 0, 1))

# Collider와 RigidBody 추가
PhysxSchema.PhysxRigidBodyAPI.Apply(cube_prim)
PhysxSchema.PhysxCollisionAPI.Apply(cube_prim)
PhysxSchema.PhysxCollisionAPI(cube_prim).CreateShapeTypeAttr("BOX")

simulation_app.update()
simulation_app.stop()

위 코드는 큐브를 월드 좌표의 높이 1m 지점에 위치시키고, 물리 엔진에서 Dynamic RigidBody로 동작하도록 설정했다. ShapeTypeAttr에 "BOX"를 지정해 충돌 형상(Box Collider)을 포함한다.

Isaac Sim 고급 스크립팅: Extension 작성¶

Isaac Sim은 Omniverse Kit 확장(Extension)이라는 개념을 사용한다. 확장은 옴니버스에 기능을 추가하기 위한 모듈이며, 파이썬으로 작성되고 Isaac Sim 실행 시 동적으로 로드된다.

확장을 작성하기 위해서는 Isaac Sim 설치 디렉터리 내의 exts 폴더나 지정된 경로에 확장 폴더를 생성한 뒤, extension.toml과 python 스크립트 파일을 작성한다. extension.toml은 대략 다음과 같은 구조를 가진다.

[package]
title = "MyIsaacExtension"
version = "1.0.0"
description = "Custom Extension for Isaac Sim"

[python]
module = "my_isaac_extension"

[dependencies]
"omni.isaac.core" = {}

이후 my_isaac_extension.py 내부에 확장 초기화, 업데이트, 종료 로직을 작성하여 Isaac Sim UI에 새로운 메뉴나 기능을 추가하고, 시뮬레이션 동작을 재정의할 수 있다.

ROS2와의 연동¶

ROS2 Humble과 Isaac Sim을 함께 사용할 때는 Isaac Sim에서 제공하는 ROS2 브리지(Bridge) 확장이나 직접 파이썬 스크립팅으로 sensor_msgs, geometry_msgs 등을 발행/구독하여 로봇을 제어할 수 있다. 이를 통해 실제 ROS2 네트워크처럼 Isaac Sim 환경에서 가상 로봇의 센서와 액추에이터를 시뮬레이션하고, 동일한 주제(topic)와 서비스(service)를 사용해 테스트를 수행할 수 있다.

ROS2 메시지 변환 시에도 내부적으로 행렬 및 벡터 연산이 자주 활용된다. 예를 들어, ROS2의 geometry_msgs/Pose 메시지를 Isaac Sim 좌표계로 변환하기 위해 변환 행렬 \(\mathbf{T}\)를 고려하고, \(\mathbf{R}\)과 \(\mathbf{p}\)를 분해하여 roll, pitch, yaw를 추출하는 등 수학적 연산이 병행된다.

실습 예제: Python으로 간단한 로봇 조작¶

다음 예제 스크립트는 Isaac Sim 내부에 임의의 6자유도 조인트 로봇을 로드해 조인트를 제어하는 모습을 보여준다.

import omni
from omni.isaac.kit import SimulationApp
from omni.isaac.core import World
from omni.isaac.core.robots import Robot
from omni.isaac.core.utils.extensions import enable_extension

simulation_app = SimulationApp({"headless": False})

enable_extension("omni.isaac.urdf")

world = World(stage_units_in_meters=True)
# 임의의 로봇 URDF 파일 경로
robot = Robot(prim_path="/World/MyRobot", urdf_file="path/to/robot.urdf")
world.add_robot(robot)

world.reset()

# 조인트 위치 설정
desired_joint_positions = [0.0, 1.57, 0.0, -1.57, 0.0, 0.0]
robot.set_joint_positions(desired_joint_positions)

for _ in range(1000):
    world.step(render=True)

simulation_app.stop()

위 코드는 URDF 형식으로 저장된 로봇을 Isaac Sim 월드에 로드하고, 로봇의 특정 조인트 위치를 설정한 뒤 시뮬레이션을 진행한다. omni.isaac.urdf 확장이 활성화되어 있어야 하며, enable_extension("omni.isaac.urdf") 등을 통해 확장 설정을 활성화한다.

Isaac Sim 파이썬 스크립팅은 기본적으로 USD 기반의 객체 관리와 물리 엔진 연동을 이해하고, Omniverse Kit 확장 구조에 익숙해지는 것에서 출발한다. URDF, SDF 등 로봇 디스크립션 형식과 ROS2 네트워크를 연결하여 실제 로봇 개발 환경에 근접한 시뮬레이션 시스템을 구축할 수 있다. 고급 단계에서는 Isaac Sim 확장을 직접 작성하거나, 수학적으로 복잡한 변환 행렬 계산을 통해 다양한 시뮬레이션 제어 로직을 구현할 수 있다.

Isaac Sim과 파이썬 스크립트를 활용하면 반복적인 수작업을 자동화하고, 시뮬레이션 객체(로봇, 센서, 환경)를 동적으로 생성하거나 제어할 수 있으므로, 로봇 연구 및 개발 프로세스를 크게 가속화할 수 있다.
