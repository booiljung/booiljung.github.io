Isaac Sim에서의 모션 이해¶

Isaac Sim은 물리 기반 시뮬레이션 환경을 통해 다양한 로봇 움직임을 정교하게 구현할 수 있도록 지원한다. USD(Universal Scene Description) 구조로 로봇의 각 조인트를 정의하고, PhysX나 Articulation 같은 물리 엔진을 기반으로 시뮬레이션을 진행한다. 이러한 체계에서 조인트의 위치, 속도, 가속도를 제어하는 과정은 시뮬레이션 시간 내에 반복적으로 적용되어 실제 로봇처럼 자연스러운 모션을 만들어 낸다.

로봇 모션을 제어할 때 가장 기본이 되는 방법은 정해진 경로 혹은 특정 궤적을 그대로 추종하도록 하는 것이다. 정적 궤적(Static trajectory) 또는 동적 궤적(Dynamic trajectory)을 생성하여 시뮬레이션 시간에 맞추어 각 조인트의 위치를 순차적으로 업데이트한다. 예컨대 시점 \(t\)에서 조인트 위치 벡터를 \(\mathbf{q}(t)\), 조인트 속도를 \(\dot{\mathbf{q}}(t)\)라 할 때, 시뮬레이터 내부에서 시간 간격 \(\Delta t\)마다 이를 갱신하여 로봇이 올바르게 움직이도록 만든다.

고전적인 운동학 모델에서 전진 운동학(Forward Kinematics, FK)은 각 조인트(조인트)의 변위 \(\mathbf{q}\)로부터 엔드 이펙터의 위치를 구하는 방법이며, 역운동학(Inverse Kinematics, IK)은 목표 지점에서 각 조인트가 가져야 할 각도를 계산하는 방법이다. Isaac Sim 내의 API 또는 OmniGraph 노드로 두 기법을 모두 활용할 수 있다.

Isaac Sim에서의 상호작용 기초¶

Isaac Sim은 로봇과 환경이 서로 충돌하거나 접촉하는 과정을 물리 엔진이 계산하여 시각적, 물리적 상호작용이 구현되도록 한다. 예를 들어 로봇 팔의 엔드 이펙터가 물체를 움켜쥐거나 놓을 때의 행위는 접촉 힘과 마찰력을 기반으로 한 동적 시뮬레이션으로 처리된다. 또 다른 상호작용은 시각적인 요소를 포함한다. 시뮬레이션 내 객체를 선택하거나 드래그하는 사용자 상호작용도 가능하며, 하드웨어 센서 시뮬레이션(Depth 카메라, LiDAR 등)을 통해 로봇이 가상의 환경 정보를 수집하게 할 수도 있다.

오브젝트의 물리 특성(질량, 관성, 마찰 등)을 정의하면, 로봇이 움직일 때 발생하는 충돌이나 접촉 시 강체(Rigid Body) 역학과 조인트의 Articulation 모델이 결합되어 상호작용이 더욱 실감 나게 표현된다.

OmniGraph와 ActionGraph를 활용한 모션 구성¶

Isaac Sim에서는 OmniGraph 또는 ActionGraph를 이용해 로봇 모션과 상호작용에 관한 노드(Node) 기반 흐름을 구성할 수 있다. 각 노드는 특정 기능을 수행하고, 노드 간 연결을 통해 움직임이나 이벤트가 전달된다. 예를 들어 조인트 제어 입력 노드가 로봇의 목표 각도를 산출하고, 해당 값을 물리 시뮬레이션 노드가 받아 실제 조인트를 갱신한다. 충돌 체크 노드에서 충돌 이벤트가 감지되면 후속 상호작용 노드에서 이에 대한 적절한 동작(예: 그리퍼를 닫거나 에러 플래그 설정 등)을 처리하도록 구성할 수 있다.

물리 시뮬레이션 파라미터 설정¶

시뮬레이션 품질과 성능은 물리 파라미터에 의해 크게 좌우된다. Isaac Sim에서 대표적으로 조정할 수 있는 물리 파라미터는 다음과 같은 것들이 있다(하지만 여기서는 목록을 나열하지 않는다). 적절한 시간 스텝(Time step), 서브 스텝(Sub-steps), 마찰 계수, 감쇠 계수 등을 적절히 조정하면, 상호작용이 보다 물리적으로 정확하면서도 안정적인 시뮬레이션을 얻을 수 있다.

시뮬레이터가 내부적으로 사용하는 기본 동역학 방정식을 살펴보면, 강체(또는 강체로 근사된 조인트) 시스템의 운동은 아래와 같은 형태로 나타낼 수 있다.

\[ \begin{align} \mathbf{M}(\mathbf{q}) \ddot{\mathbf{q}} + \mathbf{C}(\mathbf{q}, \dot{\mathbf{q}})\dot{\mathbf{q}} + \mathbf{G}(\mathbf{q}) = \mathbf{\tau}_{\mathrm{ext}} + \mathbf{\tau}_{\mathrm{ctrl}} \\ \end{align} \]

여기서 \(\mathbf{M}(\mathbf{q})\)는 조인트 위치에 따른 질량 행렬이고 \(\mathbf{C}(\mathbf{q}, \dot{\mathbf{q}})\)는 코리올리 및 원심력 항, \(\mathbf{G}(\mathbf{q})\)는 중력 항이다. \(\mathbf{\tau}*{\mathrm{ext}}\)는 외력(충돌, 접촉 등 상호작용에서 발생하는 힘)에 의해 생기는 토크를 의미하고, \(\mathbf{\tau}*{\mathrm{ctrl}}\)은 제어기에서 발생시키는 토크다. Isaac Sim은 이러한 동역학 방정식을 내장된 물리 엔진을 통해 실시간으로 근사 계산한다.

기본 모션 제어 예제¶

Python 스크립트를 활용해 Isaac Sim 환경에서 로봇의 특정 조인트 각도 또는 엔드 이펙터 위치를 목표로 설정해본다. Python API를 사용하면 Isaac Sim 내의 객체나 로봇 조인트를 프로그래밍적으로 제어할 수 있다.

import omni
from omni.isaac.core import World
from omni.isaac.core.objects import DynamicCuboid
from omni.isaac.core.robots import SingleArmManipulator

# Isaac Sim World 생성
world = World(stage_units_in_meters=True)

# 로봇 추가 (예시로 SingleArmManipulator 사용)
robot = world.scene.add(
    SingleArmManipulator(
        prim_path="/World/robot",  # 로봇 USD 경로
        name="my_robot",
    )
)

# 간단한 상자(DynamicCuboid) 추가
box = world.scene.add(
    DynamicCuboid(
        prim_path="/World/box",
        name="my_box",
        position=[0.5, 0.0, 0.5],
        size=[0.1, 0.1, 0.1],
        color=[1.0, 0.0, 0.0]
    )
)

world.reset()

# 예: 로봇의 특정 조인트 각도를 변경하거나 IK를 통해 엔드 이펙터를 특정 위치로 이동
joint_positions = [0.0, -1.57, 1.57, 0.0]  # 예시 각도
robot.set_joint_positions(joint_positions)

for step in range(1000):
    world.step(render=True)

이와 같이 Isaac Sim의 Python API로 로봇 상태를 조작하고, 물리 시뮬레이션이 이를 업데이트하는 방식으로 움직임과 상호작용이 구성된다.

고급: Inverse Kinematics와 Jacobian 활용¶

Isaac Sim 내부에서 역운동학(IK)을 처리할 때, Jacobian 행렬을 활용하면 목표 위치나 자세를 효율적으로 추정할 수 있다. 간단히 말해, 엔드 이펙터의 위치를 \(\mathbf{x}\)라 하고 조인트 벡터를 \(\mathbf{q}\)라 할 때, 다음의 관계가 성립한다.

\[ \begin{align} \mathbf{x} = f(\mathbf{q}) \\ \dot{\mathbf{x}} = \frac{\partial f}{\partial \mathbf{q}} \dot{\mathbf{q}} = \mathbf{J}(\mathbf{q}) \dot{\mathbf{q}} \\ \end{align} \]

여기서 \(\mathbf{J}(\mathbf{q})\)는 Jacobian 행렬이다. 목표 위치 \(\mathbf{x}_{\mathrm{des}}\)에 접근하기 위해 반복 계산되는 \(\Delta \mathbf{q}\)는 간단히 다음과 같은 근사를 통할 수 있다.

\[ \begin{align} \Delta \mathbf{q} \approx \mathbf{J}(\mathbf{q})^+ (\mathbf{x}_{\mathrm{des}} - \mathbf{x}) \\ \end{align} \]

\(\mathbf{J}(\mathbf{q})^+\)는 Jacobian의 의사역행렬이다. Isaac Sim은 이러한 계산을 내부적으로 처리하는 IK 솔버를 제공하기도 하며, 사용자는 Python 스크립트에서 직접 Jacobian 기반 계산을 구현해 세부적인 제어 전략을 적용할 수도 있다.

상호작용 이벤트 파이프라인¶

아래 다이어그램은 Isaac Sim에서 로봇 모션과 상호작용 이벤트가 처리되는 간략한 흐름을 나타낸다.

flowchart TB A[로봇 상태 갱신] --> B[Forward Kinematics 적용] B --> C[Collision Check] C --> D[상호작용 이벤트 처리] D --> E[물리 시뮬레이션 업데이트] E --> A

로봇 상태가 갱신되면 Forward Kinematics로 엔드 이펙터 위치를 계산하고, 충돌 체크를 통해 환경과 상호작용이 일어나는지를 평가한다. 충돌 또는 접촉 이벤트가 감지되면 상호작용 이벤트 처리 단계를 거쳐 물리 시뮬레이션 업데이트가 이뤄지고, 결과가 다시 로봇 상태로 피드백되어 반복된다.

Isaac Sim의 움직임과 상호작용 체계는 단순한 조인트 이동부터 복잡한 역운동학, 충돌 처리, 물리 엔진과 연동된 상호작용까지 유기적으로 결합되어 있다. 정확한 물리 파라미터와 운동학·동역학 모델을 설정하고, OmniGraph나 Python API를 통해 제어 흐름을 구성함으로써 높은 자유도로 로봇을 시뮬레이션하고 다양한 환경 상호작용을 테스트할 수 있다. 이를 바탕으로 ROS2 Humble과 연동하면 실제 로봇 플랫폼과의 네트워크 통신, 센서 피드백, 제어 명령까지 하나의 일관된 파이프라인으로 다룰 수 있어 로봇 개발 효율을 극대화할 수 있다.
