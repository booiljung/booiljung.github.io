Isaac Sim에서 조명 이해하기¶

Isaac Sim은 Omniverse 기반의 고성능 시뮬레이션 환경으로, 다양한 조명 기법을 실시간으로 반영할 수 있다. 물리적으로 정확한 광원(Area Light, Sphere Light, Disk Light 등)을 비롯하여 스팟 조명, 포인트 조명, 디렉셔널 조명 등 여러 유형의 라이트를 지원한다. 이러한 조명은 단순한 시각 효과뿐 아니라 센서 시뮬레이션 결과에도 직접적으로 영향을 주므로, 조명 설정을 제대로 이해하고 활용해야 한다.

조명의 이론적 배경으로 널리 알려진 모델 중 하나는 Phong 반사(Reflection) 모델이다. 물체 표면에 입사하는 빛의 확산(디퓨즈) 성분, 반사(스펙큘러) 성분, 주변(Ambient) 성분 등을 나누어 계산한다. Phong 모델은 다음과 같이 표현할 수 있다.

\[ \begin{align} I = k_d (\mathbf{L} \cdot \mathbf{N}) + k_s (\mathbf{R} \cdot \mathbf{V})^\alpha + k_a \end{align} \]

\ 여기에서 \(\mathbf{L}\)은 정규화된 광원 방향 벡터, \(\mathbf{N}\)은 표면의 법선 벡터, \(\mathbf{R}\)은 반사 벡터, \(\mathbf{V}\)는 정규화된 관찰자(혹은 카메라) 방향 벡터를 의미한다. \(k_d\), \(k_s\), \(k_a\)는 각각 확산 반사 계수, 스펙큘러 반사 계수, 주변광 계수를 나타내고, \(\alpha\)는 하이라이트 정도를 조절하는 지수이다.

Isaac Sim에서 조명 설정 방법¶

Isaac Sim에서 조명을 배치하거나 수정하려면 Scene 창에서 라이트 객체를 추가하고, Light settings 패널에서 원하는 파라미터(밝기, 색온도, 반경, 스팟 각도 등)를 수정한다. Python 확장 기능을 통해 프로그래밍적으로 조명을 설정할 수도 있다. 예시는 다음과 같이 작성할 수 있다.

import omni
from pxr import UsdGeom, Gf

stage = omni.usd.get_context().get_stage()

# 새로운 SphereLight 생성
light_path = "/World/MySphereLight"
sphere_light = UsdGeom.SphereLight.Define(stage, light_path)
sphere_light.CreateIntensityAttr(1000.0)
sphere_light.CreateRadiusAttr(25.0)
sphere_light.CreateColorAttr(Gf.Vec3f(1.0, 0.9, 0.8))  # 약간의 주황색 계열

파이썬 확장에서 다양한 라이트 타입(예: DistantLight, RectLight 등)에 대해 비슷한 방식으로 생성 및 수정이 가능하다.

카메라의 기초 개념¶

시뮬레이션 내에서 카메라는 실제 물리 세계의 카메라 역할을 한다. Isaac Sim에서 카메라는 시점(Render Product)에 대응하며, 배치된 카메라를 통해 시뮬레이션 환경을 렌더링하게 된다. 카메라 매개변수로는 초점 거리(focal length), 시야각(Field of View, FOV), 센서 해상도, 이미지 왜곡 계수 등이 있다.

기본적인 투영 모델은 간단히 다음과 같이 표현할 수 있다.

\[ \begin{align} \mathbf{p}' = \mathbf{K}\,[\mathbf{R}|\mathbf{t}]\;\mathbf{P} \end{align} \]

여기서 \(\mathbf{P}\)는 3차원 공간(\(\mathbb{R}^3\)) 상의 점의 위치, \(\mathbf{p}'\)는 2차원 이미지 좌표, \(\mathbf{R}\)과 \(\mathbf{t}\)는 외부 매개변수(카메라 자세), \(\mathbf{K}\)는 내부 매개변수(초점 거리, 주점(offset) 등)를 나타내는 행렬이다.

Isaac Sim에서 카메라 설정 방법¶

Isaac Sim에서 카메라는 Create → Camera 메뉴를 통해 추가하거나, Scripting(Python) 방식을 통해 생성할 수 있다. 예시는 다음과 같다.

import omni
from pxr import UsdGeom, Gf

stage = omni.usd.get_context().get_stage()

camera_path = "/World/MyCamera"
camera = UsdGeom.Camera.Define(stage, camera_path)

# 화각(FOV) 설정
camera.CreateFocalLengthAttr(24.0)

# 센서 해상도
camera.CreateHorizontalApertureAttr(20.955)  # mm 단위 예시
camera.CreateVerticalApertureAttr(15.290)

# 카메라 위치와 자세 지정
transform = Gf.Matrix4d().SetTranslate(Gf.Vec3d(100, 200, 150))
camera_xform = UsdGeom.XformCommonAPI(camera)
camera_xform.SetTranslate(transform.ExtractTranslation())

고급 설정과 효율적인 조명 기법¶

환경광, 직접광, 간접광을 적절히 혼합해주어야 물체 인식이나 딥러닝 학습용 데이터 수집에서 왜곡 없는 결과를 얻을 수 있다. Isaac Sim에서는 Physically Based Rendering(PBR)을 구현하여 실제 빛의 반사, 굴절, 적분 등을 보다 정교하게 시뮬레이션한다. HDR(High Dynamic Range) 이미지를 사용한 환경 맵(Environment Map)을 지정하거나, Bloom, Tone Mapping, Global Illumination 등의 후처리(Post Processing) 효과를 추가하여 사실감을 높일 수 있다.

조명은 많은 계산 자원을 소모하므로, 필요 이상의 라이트를 생성하거나 광원의 크기(Area Light 등)가 지나치게 큰 경우 성능 문제가 발생할 수 있다. 적절한 라이트 수와 광원 특성을 파악하여 GPU 자원 사용량을 관리하는 것이 중요하다.

카메라 시뮬레이션과 렌즈 왜곡 효과¶

딥러닝 기반의 비전 파이프라인을 검증하거나 센서 데이터셋을 생성할 때, 실제 카메라의 왜곡(distortion)이나 Depth of Field(초점심도), Motion Blur(움직임 블러) 등의 효과를 고려하는 것이 좋다. Isaac Sim은 카메라 모델에 비축동 왜곡 파라미터를 부여하거나, 포스트 프로세싱 스택에서 심도(depth) 기반 효과를 추가로 줄 수 있는 기능을 제공한다.

아래는 단순화한 파이프라인을 나타내는 mermaid 예시다.

graph LR A["환경(Geometry)"] --> B{조명} B --> C[렌더링 파이프라인] C --> D["카메라 시뮬레이션(출력 영상)"]

시뮬레이션 최적화 및 주의사항¶

조명과 카메라 요소가 복잡해질수록 렌더링 시간과 하드웨어 사용량이 증가한다. 실시간 시뮬레이션의 경우 프레임레이트(FPS)를 모니터링하며 조명 세기와 개수를 조정하고, 불필요한 후처리 효과를 최소화해야 한다. 고해상도 이미지가 꼭 필요한 상황이 아니라면 적절한 센서 해상도를 설정하여 GPU 부담을 줄이는 편이 좋다.

ROS2 통합 환경에서 센서 메시지를 퍼블리시할 때, 조명 조건이나 카메라 매개변수가 변경되면 이미지 결과값이 달라지므로, 시뮬레이션 파이프라인에서 이러한 변경이 필요할 경우 꼭 재현성(reproducibility)을 고려해야 한다. 필요한 경우 Isaac Sim과 ROS2 노드 간의 동기화를 사전에 설계하여 로봇 제어 로직과 센서 데이터가 일관되도록 관리한다.

조명(Lighting)과 카메라(Camera)는 Isaac Sim 시뮬레이션 환경에서 시각 인식을 다루는 모든 작업의 핵심 요소다. 기본부터 고급 기법까지 적절히 활용하고, 필요한 경우 ROS2와 유기적으로 연동함으로써 높은 수준의 시뮬레이션 정확도와 재현성을 확보할 수 있다.
