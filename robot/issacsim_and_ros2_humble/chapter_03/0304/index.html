조명의 기본 개념과 Isaac Sim에서의 렌더링 특징¶

Isaac Sim에서 그래픽 환경을 구성할 때 조명을 어떻게 설정하느냐에 따라 객체가 보이는 방식과 전체적인 장면의 분위기가 크게 달라진다. 조명을 적절히 배치하면 시뮬레이션 환경의 사실감과 깊이를 높일 수 있다. Isaac Sim은 물리 기반 렌더링(Physically Based Rendering, PBR)을 지원하며, NVIDIA RTX 기술을 활용해 광선 추적(Ray Tracing)이 가능하다. 따라서 광학적 특성을 정교하게 시뮬레이션하여 사실감 있는 조명 효과를 얻을 수 있다. 이러한 사실감을 극대화하기 위해서는 환경의 형태와 재질, 반사 특성, 조명의 색온도, 세기 등의 요소를 종합적으로 고려해야 한다.

경우에 따라서는 시뮬레이션 성능이 우선이 될 수 있다. 이때는 실시간 모드의 렌더링 품질 설정을 조절하거나 최소한의 광원만 사용하는 전략을 취할 수 있다. 반면 고해상도 결과 이미지를 생성하거나, 광학적으로 엄밀한 분석이 필요한 상황에서는 경로 추적(Path Tracing) 모드를 활성화하고 모든 재질과 광원을 정교하게 설정해야 한다.

조명 유형과 배치 전략¶

Isaac Sim에서 제공되는 주요 조명 유형은 점 광원(Point Light), 스포트라이트(Spot Light), 영역 광원(Rect Light), 돔 라이트(Dome Light) 등이 있다. 점 광원은 빛이 모든 방향으로 고르게 퍼져 나가는 특성을 가지며, 스포트라이트는 특정 방향과 각도 안에서만 빛을 집중적으로 투사한다. 영역 광원은 면적이 있는 광원으로, 부드러운 그림자를 형성하기에 유리하다. 돔 라이트는 HDRI(High Dynamic Range Image)를 이용하여 전방위 배경 조명을 제공한다.

조명을 배치할 때는 물리적으로 가능한 빛의 경로와 반사를 염두에 두어야 한다. 예를 들어 로봇 팔이 작업하는 테이블을 균일하게 밝히고 싶다면, 테이블 상단에 영역 광원을 배치하고 충분한 세기(intensity)를 부여해 그림자가 너무 짙지 않도록 조정할 수 있다. 실제 스튜디오 조명과 유사한 효과를 원한다면, 메인 라이트(Key Light), 보조 라이트(Fill Light), 엣지 라이트(Rim Light) 등으로 역할을 나누어 배치하고 각 조명의 색상과 세기를 미세 조정할 수 있다. 배경과 로봇 본체 간에 명확한 대비를 주고자 할 경우, 뒤쪽에서 조명을 배치해 피사체의 윤곽선을 강조하는 방법도 있다.

Isaac Sim UI를 통한 조명 설정¶

조명을 추가하기 위해서는 Isaac Sim 인터페이스 상단에 위치한 Create 메뉴에서 Light 항목을 찾은 뒤 원하는 조명 유형을 선택하면 된다. 조명을 추가한 후에는 Stage 창에 해당 조명 노드가 생성되며, Properties 창에서 위치, 회전, 세기, 색상 등 세부 속성을 수정할 수 있다. 여러 조명을 동시에 배치할 때는 조명의 우선순위나 색온도가 서로 겹치지 않도록 주의해야 한다. 장면이 너무 밝아지면 일부 물체가 과노출(Overexposure)되어 디테일이 보이지 않을 수 있으니, 적절한 밝기와 노출을 유지하도록 해야 한다.

레이트레이싱 모드(RTX)를 사용할 경우, Properties 창에서 Ray Tracing 관련 파라미터를 조정하거나, Render Settings 창에서 Path Tracing과 Realtime Ray Tracing 중 하나를 선택할 수 있다. 조명을 배치한 뒤에는 카메라 뷰에서 씬을 확인하며 반사나 그림자 등이 자연스럽게 표현되는지 검토하는 작업이 중요하다.

조명 색온도와 광원 스펙트럼 고려¶

시뮬레이션 결과를 실제 카메라 센서나 인간의 눈에 가깝게 표현하고 싶다면, 조명의 색온도(Color Temperature)와 스펙트럼 특성을 고민해볼 수 있다. Isaac Sim에서 조명의 색상 파라미터를 적절히 조정하면 색온도를 대략적으로 흉내낼 수 있다. 예를 들어 낮의 태양광(약 5500K~6500K) 환경을 시뮬레이션한다면, 청백색에 가깝고 충분히 강한 빛을 사용한다. 저녁 무렵의 분위기를 낼 때는 주황색이나 붉은 기운이 감도는 색상을 사용한다. LED 광원을 가정한다면 CRI(Color Rendering Index)나 스펙트럼 분포를 어떻게 모델링할지 고려해야 한다.

물리 기반 재질과의 상호 작용¶

조명은 재질(Material)과 상호 작용하여 실제 광학적 현상을 흉내 내게 된다. 금속 재질을 가진 객체는 반사율이 높고 거울처럼 빛을 반사하는 특성을 보인다. 플라스틱이나 고무 재질은 난반사를 일으키므로 빛이 여러 방향으로 퍼져 나간다. Isaac Sim의 재질 시스템은 이러한 특성을 반영할 수 있으므로, 조명을 배치할 때 재질 속성과의 시너지 효과를 고려하는 것이 좋다. 예를 들어 로봇 팔 끝 부분에 금속 재질이 적용되어 있다면, 광원에 따라 반사광이 지나치게 강해질 수 있으므로 조명 각도나 세기를 조절하여 지나친 하이라이트(Highlight)가 생기지 않도록 한다.

카메라 설정과 시점 구성¶

Isaac Sim에서 카메라는 시뮬레이션 장면을 관찰하기 위한 주요 도구이다. 작업 현장의 상태를 모니터링하거나 사진처럼 장면을 캡처하려면 적절한 카메라 배치가 필수적이다. 카메라는 Create 메뉴의 Camera 항목을 통해 추가할 수 있으며, Stage 창에서 카메라 노드를 선택해 Properties 창에서 초점 거리(Focal Length), 센서 크기(Sensor Size), 종횡비(Aspect Ratio), 조리개(Aperture) 등을 조절할 수 있다. Depth of Field(피사계 심도) 효과가 필요한 경우, 조리개와 초점 거리를 조정하여 피사체의 일부만 선명하게 하고 배경을 흐리게 만들 수 있다.

뷰포트에서의 카메라 제어¶

Isaac Sim의 뷰포트(Viewport) 우측 상단에는 카메라 선택 메뉴가 있어, 현재 선택 중인 카메라를 실시간으로 제어할 수 있다. 시점을 움직이기 위해서는 마우스 드래그와 키보드 입력을 사용할 수 있으며, 카메라의 위치와 방향은 Properties 창에서도 수치로 미세하게 조정 가능하다. 특정 객체를 프레임에 정확히 맞추고 싶다면, 해당 객체를 선택한 뒤 뷰포트 상단의 옵션에서 Frame 기능을 사용할 수 있다. 이렇게 하면 선택된 객체가 카메라 시야의 중심에 위치하게 된다.

여러 카메라 활용과 시뮬레이션 데이터 저장¶

Isaac Sim에서는 여러 개의 카메라를 동시에 배치할 수 있으므로, 다양한 시점에서 동시 녹화나 이미지 캡처를 수행할 수 있다. 시뮬레이션을 통해 연구 목적의 데이터를 수집하려면, 카메라 노드별로 해상도나 화각(Field of View), 왜곡 파라미터를 다르게 설정해 놓고 다양한 조건에서 영상을 얻을 수 있다. 로봇이 물체를 집거나 조립하는 장면을 기록하려면 로봇 근처 상단에 광각 카메라를 두어 전체 상황을 기록하고, 동시에 로봇 팔 말단 장치(End-Effector)에 근접 카메라를 달아 세부 동작을 관찰하도록 구성할 수도 있다.

ROS2 Humble와의 연동을 위한 센서 토픽 구성¶

Isaac Sim에서 카메라 이미지를 ROS2 토픽으로 송출하려면 ROS2 Humble 패키지 설정에서 센서(예: Image) 노드를 구성해야 한다. 일단 카메라를 Stage에 생성한 뒤, 카메라 노드를 ROS Bridge에 연결하면 해당 카메라에서 촬영된 이미지를 ROS2 메시지 형식으로 변환해 퍼블리시할 수 있다. 이때 토픽 이름, 이미지 해상도, 압축 방식 등을 설정할 수 있다. 라이더(LiDAR)나 딥 러닝용 분할(Segmentation) 카메라 등 다른 센서와 함께 사용하면 로봇의 주행, 조작, 인식 알고리즘을 종합적으로 개발할 수 있다.

카메라 왜곡 모델과 렌즈 효과¶

물리적으로 정확한 시뮬레이션을 위해서는 렌즈 왜곡(Distortion) 파라미터가 중요하다. 예를 들어 광각 렌즈를 사용할 경우 배럴 왜곡(Barrel Distortion)이 나타난다. Isaac Sim에서 제공하는 카메라에는 기본적으로 왜곡 모델이 적용되어 있지 않지만, 포스트 프로세싱(Post-processing) 단계에서 왜곡 효과를 구현할 수 있다. 혹은 ROS2 Humble 측에서 camera_info 토픽을 통해 내부 파라미터와 왜곡 파라미터를 전달받아 이를 고려하는 알고리즘을 적용할 수 있다. 시뮬레이션 환경에서 카메라 보정을 수행하는 연습을 하면, 실제 하드웨어 카메라를 사용했을 때 발생하는 왜곡 보정 과정을 미리 검증할 수 있다.

초점 거리, 화각, 센서 크기의 상관관계¶

카메라에서 초점 거리(\(f\)), 화각(\(\theta\)), 센서 크기(\(S\))는 서로 밀접한 상관관계를 갖는다. 단순화하여 가로 화각을 예시로 보면, 다음과 같은 근사식을 사용할 수 있다.

\[ \theta \approx 2 \arctan\left(\frac{S}{2f}\right) \]

여기서 \(\theta\)는 화각, \(S\)는 센서의 가로 길이, ff는 초점 거리다. θ\theta를 변경해 광각 또는 망원 효과를 내고 싶다면, 센서 크기나 초점 거리를 조정해야 한다. Isaac Sim에서는 Properties 창의 Focal Length와 Sensor Size를 수치로 입력하여 미세 조정할 수 있다.

카메라의 자동 노출과 수동 노출¶

Isaac Sim은 카메라 자동 노출(Exposure) 기능을 지원한다. 이 기능을 켜면 장면 밝기에 따라 노출값이 동적으로 변하면서 항상 적절한 밝기를 유지하려고 시도한다. 그러나 이는 실험 결과에 영향을 줄 수 있으므로, 정밀한 광학 측정을 목표로 한다면 수동 노출 모드에서 노출 시간을 고정해야 한다. 수동 노출 모드에서 ISO, 셔터 스피드, 조리개 값 등을 직접 지정하고 빛의 세기와 장면 반사율을 직접 제어하는 방식으로 작업하면, 항상 동일한 노출 조건에서 영상을 얻을 수 있다.

포스트 프로세싱 효과와 조명 합성¶

Isaac Sim은 Bloom, Tone Mapping, Color Correction 등 다양한 포스트 프로세싱 효과를 제공한다. Bloom 효과는 밝은 영역 주위에 빛번짐(Glow)을 추가하여 더욱 극적인 장면을 연출할 수 있고, Tone Mapping은 전체 밝기 분포를 재조정하여 물리 기반 렌더링의 HDR 이미지를 모니터나 영상으로 변환할 때 자연스럽게 보여준다. 조명을 배치하고, 적절한 카메라 노출을 설정한 뒤, 포스트 프로세싱 효과를 추가로 적용하면 실제 촬영 영상에 가까운 결과물을 얻을 수 있다.

Python 스크립트를 이용한 조명 및 카메라 배치 자동화¶

Isaac Sim은 Python API를 통해 조명과 카메라 배치를 프로그래밍 방식으로 제어할 수 있다. 대규모 시뮬레이션을 수행하거나 여러 환경을 자동으로 생성해야 할 때는 Python 스크립트가 유용하다.

아래 예시는 Python 코드를 통해 점 광원을 생성하고 위치, 세기, 색상을 지정한 뒤 카메라를 생성하는 간단한 예시다.

from omni.isaac.core.utils.stage import add_reference_to_stage
from omni.isaac.synthetic_utils import create_prim

# 점 광원 생성
point_light_path = "/World/MyPointLight"
create_prim(
    prim_path=point_light_path,
    prim_type="SphereLight",
    attributes={
        "intensity": 1000.0,
        "color": (1.0, 0.9, 0.8),
        "radius": 0.1,
    }
)

# 카메라 생성
camera_path = "/World/MyCamera"
create_prim(
    prim_path=camera_path,
    prim_type="Camera",
    attributes={
        "focalLength": 35.0,
    }
)

# 카메라 위치와 회전 설정
from pxr import Gf
from omni.isaac.core.utils.prims import set_transform

camera_position = Gf.Vec3d(0.0, -2.0, 1.0)
camera_rotation = Gf.Vec3d(0.0, 0.0, 0.0)
set_transform(camera_path, camera_position, camera_rotation)

이 코드에서 create_prim 함수로 신규 프림(Prim)을 생성하고, set_transform 함수를 통해 카메라의 위치와 회전을 설정한다. SphereLight 타입은 기본적으로 점 광원에 가까운 조명을 나타낸다. intensity 파라미터를 통해 밝기를 조절하고, color 파라미터를 통해 광원의 색을 설정한다. 작업 환경에 맞추어 여러 개의 광원을 생성하고, 필요한 카메라도 여러 개 만들 수 있다.

조명과 카메라의 상호 보완적 활용¶

시뮬레이션 환경에서 조명과 카메라는 서로 깊이 연관되어 있다. 카메라가 인식할 수 있는 장면의 밝기와 색상은 조명 배치에 의해 크게 좌우된다. 동시에, 카메라가 어떻게 노출을 잡고 색 보정을 하느냐에 따라 동일한 조명 조건에서도 최종 영상의 인상이 달라진다. 따라서 로봇 작업 환경을 설계할 때는 먼저 조명 조건을 실험적으로 결정하고, 이후 카메라 파라미터를 상황에 맞춰 조정하는 과정을 거치는 편이 효율적이다. 물체의 테두리가 선명하게 보이게 하려면 강한 역광을 사용하고, 특정 부분만 강조하려면 스포트라이트를 사용하며 배경을 어둡게 하는 식으로 꾸밀 수 있다.

시뮬레이션 성능과 최적화¶

고해상도 텍스처와 복잡한 조명, 여러 대의 카메라가 동시에 활성화된 상황에서는 GPU 자원이 많이 소모될 수 있다. Isaac Sim에서 실시간 추론이나 시뮬레이션 속도를 우선해야 한다면, 조명의 개수를 줄이거나 그림자 해상도를 낮추고, 카메라 해상도도 필요한 최소한의 수준으로 설정해야 한다. Path Tracing 모드 대신 Realtime 모드를 사용하면 프레임레이트를 높일 수 있지만, 광학적 사실성은 다소 떨어질 수 있다. 연구 목적이나 홍보용 고퀄리티 이미지 생성처럼 성능보다는 품질이 중요한 경우에는 Path Tracing 모드에서 충분한 샘플링을 적용하여 노이즈를 최소화하고, 조명 효과를 극대화하는 편이 좋다.

효율적인 시뮬레이션 장면 관리¶

조명과 카메라를 여러 개 배치하면 Stage가 복잡해질 수 있다. Stage 창에서 적절한 네이밍 규칙과 계층 구조(폴더)를 사용해 각 조명과 카메라를 구분해 두면 협업 및 유지보수에 도움이 된다. 예를 들어 TopLight, SideLight, RobotArmCamera, OverheadCamera 등의 명명 방식을 채택하면, 원하는 노드를 즉시 찾아 수정할 수 있다. 또한 Properties 창에서 조명의 On/Off 스위치를 통해 쉽게 장면을 비교해볼 수도 있다.

Isaac Sim에서 조명과 카메라를 설정하는 작업은 시뮬레이션의 사실감과 분석 정확도에 직결되므로 세심한 접근이 필요하다. 물리 기반 렌더링 기법과 RTX 기술을 적극 활용하면 매우 사실적인 장면을 구현할 수 있고, Python API로 자동화하면 대규모 실험 설정도 빠르게 반복 가능하다. 카메라 노출과 조명을 균형 있게 설정하면, ROS2 Humble 환경에서 센서 데이터를 사용하여 로봇 제어나 인공지능 모델을 학습할 때 보다 신뢰성 있는 결과를 얻을 수 있다. 현장의 요구사항과 시뮬레이션 성능 제약을 모두 고려하여, 가장 적절한 조명 및 카메라 구성 방법을 채택하는 것이 바람직하다.
