Isaac Sim 물리 엔진 개요¶

Isaac Sim은 주로 NVIDIA PhysX 엔진을 기반으로 고도의 물리 시뮬레이션을 제공한다. PhysX 엔진은 강체(rigid body), 유연체(soft body), 유체(fluid) 등의 다양한 물리 현상을 고속으로 계산한다. Isaac Sim에서는 PhysX 엔진의 다양한 파라미터를 수정하여 시뮬레이션 대상의 거동을 보다 정밀하게 구현할 수 있다. 이를 통해 로봇 매니퓰레이터, 자율주행 차량, 휴머노이드 등의 동작 특성을 사실적으로 재현하거나, 환경의 물리적 반응을 세밀하게 조절할 수 있다.

물리 파라미터 튜닝을 위해서는 먼저 물리 엔진의 기초를 이해해야 한다. Isaac Sim 내부에서 물리 엔진은 매 순간 객체들의 운동방정식을 풀어 각기 다른 시간 스텝(time step)에서의 위치, 속도, 가속도 등을 업데이트한다. 이때 질량, 마찰, 반발 계수, 조인트 특성 등 여러 파라미터가 물리 시뮬레이션의 품질과 안정성을 결정한다.

기초 물리 파라미터¶

기본적인 물리 파라미터로 질량(mass), 마찰 계수(friction coefficient), 반발 계수(restitution coefficient)가 있다. Isaac Sim에서는 이들 파라미터를 개별 오브젝트나 링크(link)에 대해 설정할 수 있다.

질량은 물체에 작용하는 힘에 대해 가속도가 어떻게 반응하는지를 결정한다. 고전역학에서 가장 기본이 되는 식은 \(\mathbf{F} = m \mathbf{a}\)이다.

마찰 계수는 접촉면에서 발생하는 마찰력을 정의하며, Isaac Sim에서는 주로 동적 마찰(dynamic friction)과 정적 마찰(static friction)을 구분한다. 마찰력은 다음과 같이 표현할 수 있다.

\[ \begin{align} \mathbf{F}_\text{f} = \mu \mathbf{N} \\ \end{align} \]

여기서 \(\mu\)는 마찰 계수, \(\mathbf{N}\)은 법선 방향 반작용(force normal)이다.

반발 계수(restitution)는 충돌 시 물체가 탄성적으로 되튕기는 정도를 나타낸다. Isaac Sim의 PhysX 엔진에서 반발 계수가 1에 가까울수록 완전탄성충돌에 가깝고, 0에 가까울수록 비탄성충돌이 된다.

관성 모멘트와 질량 행렬¶

3차원 상에서 회전 운동은 물체의 질량 분포에 의해 달라지므로, 관성 모멘트(inertia)가 중요하다. 단순 구(球)나 직육면체와 달리 복합 형태의 물체는 질량의 공간적 분포가 복잡하므로, 실제 시뮬레이션에서는 관성 텐서(inertia tensor)를 이용한다. 강체의 운동 방정식은 다음과 같은 행렬 방정식으로 나타낼 수 있다.

\[ \begin{align} \mathbf{M}(\mathbf{q}) \ddot{\mathbf{q}} + \mathbf{C}(\mathbf{q}, \dot{\mathbf{q}})\dot{\mathbf{q}} + \mathbf{G}(\mathbf{q}) = \mathbf{\tau} \\ \end{align} \]

여기서 \(\mathbf{M}(\mathbf{q})\)는 질량 행렬(mass matrix), \(\mathbf{C}(\mathbf{q}, \dot{\mathbf{q}})\)는 코리올리 및 원심력 행렬, \(\mathbf{G}(\mathbf{q})\)는 중력 항, \(\mathbf{\tau}\)는 일반화된 힘 벡터를 의미한다. Isaac Sim에서 관성 모멘트는 주로 각 오브젝트의 물리 속성(Physics Properties) 설정 항목에서 지정 가능하다.

댐핑(Damping)과 안정성¶

시뮬레이션 객체가 진동하거나 불안정하게 튀는 현상을 억제하기 위해 댐핑 계수(damping coefficient)를 적용한다. 로봇의 경우, 조인트에 댐핑을 부여하여 높은 속도의 조인트 회전이 가해질 때 진동이 빠르게 가라앉도록 만들 수 있다. 댐핑은 선형이나 회전에 대해 각각 설정할 수 있으며, Isaac Sim에서는 조인트나 아트리큘레이션(Articulation) 설정에서 댐핑 파라미터를 수정할 수 있다.

댐핑은 운동 방정식에 속도 항과 비례하는 항으로 추가된다. 이를 간단히 나타내면 아래와 같이 나타낼 수 있다.

\[ \begin{align} \mathbf{M} \ddot{\mathbf{x}} + \mathbf{B} \dot{\mathbf{x}} + \mathbf{K}\mathbf{x} = \mathbf{F}_\text{ext} \\ \end{align} \]

여기서 \(\mathbf{B}\)는 댐핑 행렬(damping matrix), \(\mathbf{K}\)는 스프링 강성(stiffness)을 나타낸다. 일반적인 강체 시뮬레이션에서는 \(\mathbf{K}\)가 0이거나 작은 경우가 많으며, 댐핑 계수를 적절히 조정하여 튀는 현상을 방지한다.

조인트 및 조인트(Articulation) 파라미터¶

Isaac Sim에서 로봇 모델은 일반적으로 URDF나 USD 포맷으로 불러온다. 조인트(joint)마다 조인트 한계(joint limit), 마찰, 댐핑, 스프링 계수 등을 설정할 수 있다. 예를 들어 로봇 조인트가 일정 각도를 초과하지 않도록 하려면 joint limit과 restitution 설정을 적절히 조절한다. 조인트 프레임의 마찰이 너무 낮으면 원하는 각도에서 미끄러지듯 움직일 수 있으므로, 조인트 마찰이나 댐핑 계수를 높여 안정적 제어를 구현할 수 있다.

조인트에는 solver iteration count나 allowable error와 같은 내부 솔버 파라미터도 존재한다. 해석학적으로 복잡한 환경에서 조인트가 충돌하거나 강체가 서로 간섭을 일으킬 때, 높은 solver iteration이 요구된다. Isaac Sim에서는 물체가 조인트 범위에 충돌했을 때 정확한 계산을 위해 solver의 iteration 횟수를 높일 수 있으며, 이를 통해 시뮬레이션 품질을 높일 수 있다.

지면과의 상호작용 파라미터¶

자율주행 로봇이나 모바일 로봇 시뮬레이션을 할 때, 지면과 바퀴 사이의 마찰, 지면의 요철, 충돌 경계 등이 주요 인자가 된다. 지면 마찰이 너무 낮으면 바퀴가 헛돌고, 너무 높으면 비현실적인 급정지 현상이 나타난다. Isaac Sim에서는 지면 객체(ground plane)의 물리 파라미터를 직접 수정하거나, 새로 모델링한 지형(terrain)에 맞추어 물리 파라미터를 설정한다.

고급 물리 파라미터 튜닝¶

고급 튜닝에서는 패널티(penalty) 기반 충돌 처리, 서브스텝(substep) 증가, 그리고 아트리큘레이션 내부의 병렬 연산 최적화 등을 다룰 수 있다. Isaac Sim은 시뮬레이션 스텝과 별개로 내부 물리 엔진의 서브스텝 횟수를 조절할 수 있는데, 서브스텝 수를 높이면 물리 계산이 더 자주 이루어져 충돌 정확도가 향상된다. 반면 시뮬레이션 속도는 느려질 수 있으므로, 정확도와 실시간 처리 사이에서 적절한 타협점을 찾아야 한다.

충돌 모델이 복잡하거나 파티클, 유체가 포함된 장면에서는 solver 설정과 substep 설정을 더욱 세밀하게 조정해야 한다. 예를 들어 서브스텝이 지나치게 낮으면 파티클이 관통(penetration)하거나 에너지가 보존되지 않는 문제가 생긴다. Isaac Sim은 GPU 가속 기능을 제공하므로, 파티클이나 유체 시뮬레이션 시 GPU 자원을 적극 활용할 수 있다.

Isaac Sim에서의 물리 파라미터 설정 예시 (Python)¶

Isaac Sim Python API를 사용해 간단한 물리 파라미터를 설정하는 예시이다.

from omni.isaac.kit import SimulationApp
simulation_app = SimulationApp({"headless": False})

import omni
from omni.isaac.core import World
from omni.isaac.core.objects import DynamicCuboid

world = World(stage_units_in_meters=1.0)

cube = DynamicCuboid(
    prim_path="/World/Cube",
    name="cube",
    position=[0.0, 0.0, 1.0],
    size=[0.1, 0.1, 0.1]
)

# 물리 파라미터 변경
# 마찰, 반발 계수 등
cube.set_physics_properties(
    mass=2.0,
    dynamic_friction=0.5,
    static_friction=0.6,
    restitution=0.1
)

world.add_object(cube)
world.reset()
simulation_app.update()
simulation_app.play()

이 예시에서는 큐브 형태의 DynamicCuboid 객체를 생성한 뒤, 질량, 동적 마찰(dynamic friction), 정적 마찰(static friction), 반발 계수(restitution) 등을 설정한다. 이후 시뮬레이션을 실행하면 Cube가 중력, 충돌, 마찰 등의 물리 효과를 반영하여 움직인다.

ROS2 Humble와의 연계¶

Isaac Sim에서 물리 파라미터를 조정한 모델은 ROS2 Humble과 연계하여 움직임을 테스트하거나 센서 데이터를 수집할 수 있다. URDF나 SDF 모델에 설정된 물리 파라미터가 잘 반영되도록, Isaac Sim의 USD 파일과 ROS2 메시지 간 호환성을 유지해야 한다. 이를 위해 ROS2 Humble 통신 브리지(ROS2 Bridge)나 카메라, 라이다 등 센서 플러그인을 Isaac Sim에 추가할 수 있다.

아래는 Isaac Sim에서 ROS2 토픽을 구독하여 이동 명령을 전달받고, 물리 파라미터가 적용된 로봇이 움직이는 과정을 간단하게 나타낸 순서도이다.

flowchart LR A[ROS2 Publisher <cmd_vel>] --> B[Isaac Sim ROS2 Bridge] B --> C["Physics Engine (PhysX)"] C --> D[Robot Articulation] D --> E[Robot Motion in 3D Scene]

ROS2의 토픽, 서비스, 액션 등을 활용하면 Isaac Sim에서 구현한 물리 환경을 외부 제어기나 SLAM 알고리즘 등과 연동할 수 있다. 물리 파라미터가 달라지면 로봇의 동작 성능(가속도, 제동 거리, 회전 반응 등)이 달라지므로, 실제 하드웨어와 유사한 튜닝 값을 찾아내는 것이 중요하다.

--- 및 주의사항

물리 파라미터 튜닝은 시뮬레이션 정확도와 실시간 처리 속도 사이의 균형점을 찾는 과정이다. 질량, 관성 모멘트, 마찰, 댐핑, 반발 계수, solver iteration 수, substep 등을 동시에 고려해야 한다. 또한 Isaac Sim이 제공하는 GPU 가속 기능을 적절히 활용하면 대규모 장면, 복잡한 충돌 처리, 유체 시뮬레이션 등을 원활하게 수행할 수 있다.

ROS2 Humble와 연계할 때는 URDF와 Isaac Sim의 USD 파일 간 물리 파라미터 싱크가 맞는지 주의 깊게 살펴야 한다. 실제 하드웨어 실험과의 오차를 줄이려면, 시뮬레이션과 하드웨어 센서 데이터를 비교·조정하는 과정이 필수적이다. 이를 통해 시뮬레이션에서 개발한 로봇 제어 알고리즘을 실제 환경으로 옮길 때 발생하는 불확실성을 최소화할 수 있다.
