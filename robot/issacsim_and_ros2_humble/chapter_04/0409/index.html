개요¶

Isaac Sim은 NVIDIA Omniverse 환경에서 물리 기반 로보틱스 시뮬레이션을 수행하기 위한 강력한 도구다. 일반적으로 GUI를 통해 시각화하며 각종 센서나 로봇 모델을 직관적으로 제어하고 확인한다. 그러나 대규모 환경에서 테스트를 진행하거나 서버 환경에서 원격으로 Isaac Sim을 구동해야 하는 경우에는 그래픽 인터페이스를 비활성화한 백그라운드(headless) 실행과 서버 모드(server mode)가 필요하다. 이러한 모드는 여러 클라이언트가 동시에 접속하거나 대규모 배치 테스트를 할 수 있도록 유연한 환경을 제공한다.

Isaac Sim 백그라운드 실행의 기초¶

GUI 없이 Isaac Sim을 구동하는 방법은 일반적으로 다음과 같이 실행 스크립트나 도커 컨테이너 내부에서 명령줄 옵션을 추가하여 진행한다. 예시로 로컬 환경에서 Isaac Sim을 백그라운드로 구동하는 명령은 아래와 같다.

./isaac-sim.sh --headless --no-window

위와 같이 실행하면 창이 뜨지 않고 내부의 물리 시뮬레이션과 ROS2, Python API 등을 그대로 사용할 수 있다. 기본적인 요구 사항은 GPU 드라이버가 제대로 설치된 환경이며, 서버 환경에서 원격으로 접속하기 위해서는 추가로 포트가 개방되어 있어야 한다.

Omniverse의 RTX 기능이나 UI가 필요치 않고 순수하게 시뮬레이션 데이터만 수집해야 할 경우 이러한 백그라운드 실행이 유리하다. 로봇 학습(특히 강화학습) 시나리오에서 많은 에피소드를 빠르게 처리하려면 GUI 오버헤드를 제거하는 것이 자원 활용 측면에서 더욱 효율적이다.

Isaac Sim 서버 모드¶

서버 모드는 Isaac Sim의 코어 엔진을 원격으로 구동하고, 다양한 클라이언트(ROS2 노드, Python 스크립트 등)가 네트워크를 통해 해당 시뮬레이션에 접근할 수 있도록 지원한다. 이러한 기능은 중앙 서버에 Isaac Sim을 배포해두고 여러 사용자가 동시에 접속해 시뮬레이션 리소스를 공유하거나, 클라우드 환경에서 Isaac Sim을 컨테이너 기반으로 제공할 때 중요하다.

기본적으로 서버 모드를 사용하려면 headless 옵션을 활용하되, Isaac Sim이 내부적으로 REST API 또는 WebSocket을 통해 통신할 수 있도록 설정해야 한다. 필요에 따라서는 Isaac Sim에서 제공하는 추가 옵션을 통해 접속 포트, 인증 방식 등을 지정해야 한다. 아래는 서버 모드 구동 예시다(단순 예시).

./isaac-sim.sh --headless --no-window --allow-remote-api

실행 후 원격 머신에서 접속할 수 있도록 포트를 확인하거나 방화벽 설정을 조정해야 한다. 서버 모드에서는 자체 제공하는 Omniverse Streaming 기능으로 클라이언트 측에 3D View를 스트리밍하기도 하며, 또는 ROS2의 토픽/서비스/액션 통신을 통해 시뮬레이션 환경에 접근하기도 한다.

스크립트와 연동¶

백그라운드 실행이나 서버 모드 모두 Python 스크립트 및 ROS2 노드와 연동하여 Isaac Sim의 기능을 호출하거나 시뮬레이션 환경을 자동화한다. Isaac Sim은 시뮬레이션을 구동함과 동시에 자체 Python 인터프리터에서 스크립트를 실행할 수 있도록 명령줄 옵션을 제공한다. 예시는 다음과 같다.

./isaac-sim.sh --headless --no-window --python-exec my_script.py

이와 같이 실행하면 GUI 없이 Isaac Sim을 구동하면서 곧바로 Python 스크립트가 실행된다. 스크립트 내부에서는 Omniverse API와 Isaac Sim API를 통해 로봇, 센서, 환경을 설정하고 물리 시뮬레이션을 제어할 수 있다.

Python에서 Isaac Sim을 제어하는 스크립트 예시는 다음과 같이 작성할 수 있다.

import omni
import carb

def main():
    # 스테이지(Stage) 초기화
    stage = omni.usd.get_context().get_stage()

    # 로봇 로드, 환경 설정 등의 작업 수행
    # ...

    # 시뮬레이션 실행
    sim_status = omni.kit.app.get_app().play()
    print("Simulation started")

    # 특정 시간 또는 조건까지 시뮬레이션 진행
    # ...
    # (데이터 수집, 이동, 센서값 확인 등)

    # 종료
    omni.kit.app.get_app().stop()
    print("Simulation stopped")

if __name__ == "__main__":
    main()

고급 활용¶

고급 활용에서는 대규모 배치 시뮬레이션, 컨테이너 기반 클라우드 호스팅, 또는 여러 노드 간 병렬 시뮬레이션을 구성하는 방법이 포함된다. Docker 컨테이너 환경에서 Isaac Sim을 백그라운드로 구동하여 여러 컨테이너를 병렬로 띄우거나, 클러스터 관리 시스템(Kubernetes 등)을 통해 Isaac Sim 컨테이너를 스케줄링하는 방식으로 확장할 수 있다.

Isaac Sim에서 시뮬레이션 상태를 가져와 ROS2 노드에 연동할 때는 점군, 이미지, 조인트 상태 등 다양한 센서 데이터를 Publish하고, 로봇 상태가 업데이트되도록 Subscribe할 수도 있다. 백엔드에서 headless 모드로 Isaac Sim을 구동하고, 프론트엔드에서는 각각의 연구자나 로봇 개발자가 ROS2 Topics, RQt, RViz 또는 Web Client를 통해 접근하는 구조를 구성할 수 있다.

아래는 Isaac Sim 서버 모드에서 주요 처리를 요약한 흐름도다.

flowchart LR A[Isaac Sim 백그라운드 실행] --> B[포트 열기 및 서버 모드 진입] B --> C[명령 및 데이터 통신] C --> D[시뮬레이션 요청/응답] D --> E[결과 전달] E --> F[원격 종료 또는 지속 실행]

시뮬레이션 내 로봇의 위치 \(\mathbf{p}\)와 자세를 4×4 변환 행렬 \(\mathbf{T} \in \mathbb{R}^{4 \times 4}\)로 나타낼 때, 보통 아래와 같은 형태로 표현한다.

\[ \begin{align} \mathbf{T} = \begin{bmatrix} \mathbf{R} & \mathbf{p} \\ \mathbf{0} & 1 \end{bmatrix}\\ \end{align} \]

여기서 \(\mathbf{R}\)은 3×3 회전 행렬이고, \(\mathbf{p}\)는 3×1 변위 벡터다. 이러한 좌표 변환을 활용해 가상 환경에서 로봇의 정밀한 움직임을 시뮬레이션하고, 머니퓰레이션(manipulation) 실험을 진행할 수 있다.

Isaac Sim의 백그라운드 실행 및 서버 모드는 로컬 워크스테이션뿐 아니라 원격 서버, 클라우드 환경에서 Isaac Sim을 유연하고 확장성 있게 운영하기 위한 핵심 기능이다. GUI 오버헤드를 없애 효율적인 배치 학습 및 대규모 실험을 진행하거나, 여러 사용자와 자원을 공유하기 위해 Isaac Sim을 원격 서비스 형태로 제공할 수도 있다. ROS2 Humble과 연동하여 다양한 로보틱스 시나리오를 시뮬레이션하고 자동화함으로써 로봇 개발 프로세스를 크게 향상시킬 수 있다.
