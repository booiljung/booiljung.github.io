Isaac Sim 개요¶

Isaac Sim은 NVIDIA에서 개발한 로보틱스 시뮬레이션 플랫폼으로, 강력한 물리 엔진과 현실감 있는 센서 시뮬레이션 환경을 제공한다. ROS(로보틱스 운영체제)와 결합하여 가상 환경에서 로봇의 동작을 검증할 수 있고, 데이터 수집 과정을 자동화하여 딥러닝을 비롯한 다양한 머신러닝 연구에 활용하기도 좋다. Isaac Sim은 Omniverse 기반으로 만들어졌으며, 고성능 GPU 가속 렌더링과 대규모 3D 씬을 효율적으로 처리할 수 있는 장점을 갖는다.

Isaac Sim을 이용하면 복잡한 로봇의 조인트, 동역학 특성, 센서 데이터 흐름을 실제 환경과 유사한 수준으로 재현할 수 있다. 특히 Lidar, 카메라, IMU와 같은 일반적인 센서뿐만 아니라 GPS, 초음파 센서, Force/Torque 센서 등의 가상 데이터도 생성할 수 있다. 이렇게 생성한 데이터를 ROS2 Humble 환경으로 직접 전달하거나, Isaac ROS 패키지를 활용하여 컴퓨팅 그래프 일부를 Isaac Sim 내에서 동작시키는 전략을 세울 수도 있다.

개발 환경 연동 기본 개념¶

개발 환경 연동은 크게 로컬 머신에서 Isaac Sim을 직접 실행하는 방법, 원격이나 컨테이너(Docker) 상에서 Isaac Sim을 구동하는 방법, 그리고 Isaac Sim에서 제공하는 파이썬 익스텐션(Extension) 기능을 활용하는 방법으로 나눌 수 있다. 어떤 방법을 선택하든 ROS2 Humble의 노드가 Isaac Sim 내부에서 제공하는 가상 센서 토픽을 받고, 다시 Isaac Sim에 명령 토픽을 보내는 과정을 구성할 수 있어야 한다.

Isaac Sim에서 센서를 구성할 때, 센서 정보는 일반적으로 시뮬레이터 내부의 특수한 데이터 포맷으로 만들어진다. 이때 ROS2 토픽으로 변환하기 위해서는 Isaac Sim 내부의 ROS 익스텐션 기능 혹은 Isaac Sim과 별도의 브리지 노드를 이용해야 한다. 따라서 Isaac Sim에서 \(/scan\), \(/camera\), \(/imu\)와 같은 표준 ROS2 센서 토픽을 퍼블리시(publish)하고, 로컬 또는 컨테이너에 있는 ROS2 노드에서 이를 구독(subscribe)하는 구조를 기본으로 생각할 수 있다.

아래는 Isaac Sim이 포함된 전체 개발 환경을 단순화한 예시이다.

flowchart LR A[Isaac Sim] --> B[ROS2 Humble Core] B --> C[로봇 로컬 코드 / AI 모듈] C --> A

Isaac Sim에서 가상 로봇 모델과 센서 정보를 생성하면, ROS2 Humble 코어에서 이를 토픽으로 수신한다. 그 후 로봇 제어 알고리즘 혹은 AI 모듈에서 처리한 명령이 다시 Isaac Sim으로 전달되어, 가상의 로봇 모델이 물리적으로 움직이는 형태로 폐루프 제어가 완성된다.

로컬 개발 환경과 Isaac Sim 연동¶

Isaac Sim과 로컬 개발 환경(ROS2 Humble)이 동일한 물리 호스트에 있을 때, 서로 간의 통신 설정이 비교적 단순해진다. Isaac Sim을 설치한 후, ROS2 Humble 환경 변수가 설정된 터미널에서 시뮬레이터 내부 토픽을 직접 구독하거나, Isaac Sim에서 ROS2 익스텐션을 활성화하여 토픽 브리지 역할을 수행하게 만들 수도 있다. 예를 들어 로컬에서 Isaac Sim의 ROS Bridge를 실행하려면 아래와 비슷한 명령을 사용할 수 있다.

# Isaac Sim 디렉토리로 이동
cd ~/isaac-sim

# Isaac Sim 실행 시, ROS2 브리지 익스텐션 자동 활성화
./isaac-sim.sh --allow-root --ros2-bridge

이렇게 Isaac Sim이 실행된 후, 로컬에서 ROS2 노드를 통해 Isaac Sim이 퍼블리시하는 Lidar 스캔 데이터를 구독하고 싶다면 \(/scan\) 토픽 구독 노드를 예시로 작성할 수 있다.

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import LaserScan

class ScanSubscriber(Node):
    def __init__(self):
        super().__init__('scan_subscriber')
        self.sub = self.create_subscription(
            LaserScan,
            '/scan',
            self.scan_callback,
            10
        )

    def scan_callback(self, msg):
        self.get_logger().info(f'Received scan with angle_min: {msg.angle_min}')

def main(args=None):
    rclpy.init(args=args)
    node = ScanSubscriber()
    rclpy.spin(node)
    rclpy.shutdown()

if __name__ == '__main__':
    main()

이 코드는 Isaac Sim이 가상 센서를 통해 퍼블리시하는 \(/scan\) 토픽을 구독하고, 터미널에 angle_min을 출력한다.

도커(Docker) 활용 전략¶

고성능 GPU 가속이 필요한 Isaac Sim은 일반적으로 호스트 OS에 직접 설치하여 구동하거나, NVIDIA Docker 기반의 컨테이너를 통해 실행할 수 있다. Docker 컨테이너 안에 ROS2 Humble 환경을 구성하고, Isaac Sim 컨테이너를 별도로 구성하여 네트워크 브리지를 통해 통신하는 방법도 가능하다. GPU 자원을 활용해야 하므로, NVIDIA Container Toolkit 등을 이용해 컨테이너 내부에서 GPU를 인식하도록 설정해야 한다.

컨테이너를 이용하면 Isaac Sim 실행 환경과 ROS2 환경을 독립적으로 유지할 수 있고, 여러 버전의 Isaac Sim 및 ROS2를 동시에 실험할 수 있다는 장점이 있다. 다만 컨테이너 상에서 그래픽 렌더링을 사용해야 하므로, X11 포워딩이나 VirtualGL 등을 적절히 설정해야 Isaac Sim의 GUI가 정상적으로 표시된다. 원격 서버에서 Isaac Sim을 구동하는 경우, VNC나 NICE DCV 같은 원격 그래픽 접속 솔루션을 쓰기도 한다.

고급 사용 사례¶

Isaac Sim과 개발 환경을 유기적으로 연동할 때, 물리 시뮬레이션 정확도와 데이터 동기화가 매우 중요해진다. 시뮬레이터 내부의 프레임워크는 일반적으로 60Hz 이상의 렌더링 프레임으로 동작하지만, 물리 엔진이나 ROS2 토픽의 주파수는 다를 수 있다. 정확한 타이밍 동기화를 위해 Isaac Sim 내의 ROS Clock을 활성화하거나, ROS2 시스템에서 시뮬레이터 클록을 받아서 \(/clock\) 토픽으로 처리하는 방법을 쓸 수 있다.

센서 퓨전 알고리즘이나 고차원 제어 알고리즘을 실험할 때는 좌표계를 엄밀하게 정의해두어야 한다. 예를 들어 Isaac Sim의 로봇 베이스 링크를 \(^I\mathbf{T}_B\)와 같이 표현하고, 월드 좌표계를 \(^W\mathbf{T}_I\)로 두면, 전체 변환 관계는 다음과 같은 형태가 될 수 있다.

\[ \begin{align} ^W\mathbf{T}_B = ^W\mathbf{T}_I \times {}^I\mathbf{T}_B \\ \mathbf{x}_B = ({}^W\mathbf{T}_B)^{-1} \mathbf{x}_W \end{align} \]

여기서 \(\mathbf{x}_W \in \mathbb{R}^4\)는 월드 좌표계에서의 위치 벡터를 동차 좌표계 형태로 확장한 것이고, \(\mathbf{x}_B\)는 로봇 베이스 링크 좌표계에서의 위치 벡터를 의미한다. Isaac Sim 내부의 3D 환경에서 실제 로봇 모델이 가진 링크와 조인트 트리가 올바른 좌표계 변환으로 구성되어 있어야 ROS2의 TF(Transform) 시스템과 불일치가 생기지 않는다.

Isaac Sim의 파이썬 익스텐션 기능을 활용하면, 시뮬레이터 내부에서 직접 ROS2 노드를 실행할 수 있고, 렌더링이나 물리 엔진과 더 밀접하게 연동할 수 있다. 예를 들어 Isaac ROS 패키지를 이용해 딥러닝 추론을 시뮬레이터 내부에서 수행한 뒤, 해당 결과를 바로 가상 로봇의 동작에 반영하거나 ROS2 주행 알고리즘 노드로 보내는 방식이다.

고급 사용 사례에서 주의해야 할 점 중 하나는 Isaac Sim의 업데이트 주기가 ROS2 메시지 주기와 달라 과도한 지연(latency)이 발생하거나, 메시지가 누락되는 현상이 나타날 수 있다는 것이다. 이런 문제는 ROS2 QoS 설정, Isaac Sim의 Physics Substeps, 그리고 시뮬레이션 실제 시간 스케일(Simulation Real-Time Scale) 등을 조정하여 완화할 수 있다. 특히 시뮬레이션 시간은 가상 환경에서 빠르게 진행시킬 수도 있고, 실제보다 느리게 진행시킬 수도 있으므로, 데이터셋 생성 시에는 빠른 진행을, 실시간 제어 실험 시에는 실제 시간과 동일하게 맞추는 전략을 사용할 수 있다.

Isaac Sim과 ROS2 Humble 환경을 연동하는 전략은 로컬 환경에서 단순히 Isaac Sim을 직접 실행하는 기본 방식부터 Docker 컨테이너를 활용한 고급 방식까지 다양하다. 물리적 호스트에서 GPU 가속을 지원하도록 설정해야 하며, Isaac Sim 내부의 ROS 익스텐션 기능과 로컬 또는 컨테이너 상의 ROS2 노드 간의 토픽·서비스 연동이 올바르게 설정되어야 한다. 좌표계나 시뮬레이션 시간 같은 기본 개념을 엄밀하게 정의해야 센서 퓨전, SLAM, AI 모델 학습과 같은 복잡한 시나리오를 정확하게 재현할 수 있다. Isaac Sim의 강력한 렌더링, 물리 엔진, ROS2와의 직접적 브리지 기능을 모두 효과적으로 활용한다면, 가상 환경에서 다양한 로봇 연구와 개발을 안전하고 빠르게 진행할 수 있을 것이다.
