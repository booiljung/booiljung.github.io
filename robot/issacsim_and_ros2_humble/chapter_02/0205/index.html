Isaac Sim 개요¶

Isaac Sim은 NVIDIA Omniverse 생태계를 기반으로 하여 로봇, 자율주행, 제조 등 다양한 시뮬레이션 시나리오를 구축하고 검증할 수 있는 고도화된 시뮬레이션 플랫폼이다. Omniverse의 USD(Universal Scene Description) 파일 형식을 바탕으로 복잡한 3D 자산 및 물리 환경을 보다 직관적으로 구성할 수 있다. Isaac Sim을 실행하면 Omniverse 런처 형태의 창이 뜨고, 그 안에서 3D 뷰포트와 함께 여러 패널이 보이게 된다.

Isaac Sim의 기본 인터페이스 구성은 크게 3D 뷰포트, Stage, Property, Console(또는 Log), Script Editor, Timeline, 그리고 다양한 기능 확장을 위한 Extension Manager로 나눌 수 있다. 이 장에서는 Isaac Sim을 실행했을 때 가장 먼저 접하게 되는 주요 인터페이스 구성을 기초부터 고급까지 꼼꼼하게 살펴본다.

3D 뷰포트(3D Viewport)¶

Isaac Sim을 실행하면 중앙에 위치하는 넓은 영역이 3D 뷰포트이다. 여기서 실제 시뮬레이션 장면을 시각적으로 확인할 수 있다. 뷰포트 내에서 마우스 조작(좌클릭, 우클릭, 드래그, 휠 스크롤 등)을 통해 카메라 시점을 이동시키고, 오브젝트의 위치나 회전 등을 직접 확인한다.

Omniverse 뷰포트는 물리 기반 렌더링(PBR, Physically Based Rendering)을 지원하여 실사에 가까운 품질로 장면을 확인할 수 있다. 또한 뷰포트 상단에 있는 툴바(Toolbar)를 통해 선택, 이동, 회전, 확대/축소와 같은 오브젝트 변환 도구를 빠르게 사용할 수 있다.

Stage 패널¶

3D 뷰포트 왼쪽 또는 오른쪽(설정에 따라 위치가 달라질 수 있음)에 자리 잡은 패널이 Stage 패널이다. Omniverse Stage는 USD 기반의 계층적 씬 구조를 확인하고 편집하는 곳으로, 로봇 모델, 센서, 환경 요소, 조명, 카메라 등의 모든 프림(Prim)이 트리(Tree) 형태로 정리되어 있다.

Stage 패널에서 각 프림을 선택하면 그 상세 정보를 Property 패널에서 확인할 수 있다. 프림 이름, 경로, 활성화 여부 등을 한눈에 파악할 수 있고, 드래그 앤 드롭을 통해 프림 간 계층 구조를 쉽게 바꿀 수도 있다.

Property 패널¶

Stage 패널에서 특정 프림을 선택하면, 해당 프림의 속성(Property)이 Property 패널에 나타난다. 프림의 변환(transform), 시각적·물리적 특성(material, collision, mass 등), 추가적인 확장 속성(extension property) 등이 이곳에서 설정된다.

3D 객체의 위치, 회전, 스케일은 기본적으로 변환 행렬 형태로 표현되며, Isaac Sim 내부에서 이를 효율적으로 업데이트하고 관리한다. 예를 들어 3D 공간에서의 변환은 종종 4×4 동차 변환 행렬로 표현되는데, 이를 수식으로 나타내면 다음과 같은 형태가 된다.

T=[Rp01]∈R4×4\begin{align} \mathbf{T} &= \begin{bmatrix} \mathbf{R} & \mathbf{p} \ 0 & 1 \end{bmatrix} \ &\in \mathbb{R}^{4 \times 4} \end{align}

여기서 \(\mathbf{R}\)은 3×3 회전 행렬이며, \(\mathbf{p}\)는 3차원 위치 벡터이다. Property 패널에서 위치와 회전, 스케일을 수치로 입력하면, 내부적으로 위 수식 형태의 변환 행렬이 업데이트되어 3D 뷰포트에 반영된다.

Timeline¶

하단 또는 상단에 위치할 수 있는 Timeline 패널은 시뮬레이션 시간 및 애니메이션 처리를 다룬다. Isaac Sim은 물리 시뮬레이션 환경을 제공하므로, 시간 축에서 시뮬레이션을 재생(play)·정지(pause)·프레임 단위 제어 등을 수행할 수 있다.

Timeline 패널에서 시뮬레이션 재생 속도나 물리 업데이트(Physics Update) 속도를 설정할 수 있으며, 특정 시점에서의 로봇 동작이나 센서 데이터 상태를 확인하는 데 유용하다.

Console(또는 Log) 패널¶

Isaac Sim은 다양한 메시지를 콘솔(또는 Log) 창에 출력한다. 로드된 확장(Extension)이나 플러그인의 동작 상태, 오류 로그, 경고 메시지 등이 표시된다. 특정 행동이 정상적으로 수행되는지, 또는 오류가 발생했는지 등을 실시간으로 확인하기 위해 자주 참조하게 된다.

Console 패널에서 원하는 메시지 레벨(Info, Warning, Error 등)만 필터링하여 볼 수도 있으며, Isaac Sim의 스크립트 실행 결과를 빠르게 확인하는 데 도움이 된다.

Script Editor¶

Isaac Sim은 Python 기반으로 스크립트를 작성하여 시뮬레이션 환경을 자동화하거나 플러그인을 확장할 수 있도록 지원한다. Python API를 활용해 Stage 상의 오브젝트를 생성·삭제하거나 물리 엔진 속성을 조작할 수 있다.

Script Editor 패널을 통해 코드를 작성하고 곧바로 실행할 수 있으며, Isaac Sim 내부 API, Omniverse API 등을 불러와 복합적인 동작을 테스트할 수 있다. 예시로 간단한 Python 코드를 작성해볼 수 있다.

import omni
from omni.isaac.core import World

world = World()
print("새로운 월드가 생성되었다.")

# 원하는 USD 씬을 불러올 수도 있다
world.scene.add_default_ground_plane()

world.reset()
print("시뮬레이션 리셋 완료.")

위 코드는 Script Editor에서 직접 실행할 수 있으며, 실행 후 메시지는 Console 패널에서 확인 가능하다.

Extension Manager¶

Isaac Sim의 기능은 Extension Manager를 통해 활성화하거나 비활성화할 수 있다. 로봇, 센서, 강화학습, 시뮬레이션용 UI 등 다양한 확장이 준비되어 있으며, 필요에 따라 원하는 기능만 골라 사용할 수 있다.

Extension Manager 창에서 확장을 켜거나 끄면 즉시 인터페이스에 반영되어 메뉴가 늘어나거나, 스크립트 API가 추가되는 등의 변화를 확인할 수 있다.

Isaac Sim에서의 고급 3D 변환¶

Isaac Sim을 활용해 로봇의 조인트 운동이나 경로 계획 등의 동작을 시뮬레이션할 때에는 3D 변환이 매우 중요한 역할을 한다. 로봇의 각 조인트(조인트)은 회전 변환을 가지며, 말단 이펙터의 위치를 계산할 때에는 다음과 같은 연쇄 행렬 곱이 이루어진다.

\[ \begin{align} \mathbf{T}_{\text{base} \to \text{end}} &= \mathbf{T}_{\text{base} \to \text{joint1}} \, \mathbf{T}_{\text{joint1} \to \text{joint2}} \, \dots \, \mathbf{T}_{\text{jointN} \to \text{end}} \\ &\in \mathbb{R}^{4 \times 4} \end{align} \]

각 \(\mathbf{T}_{\text{joint(i)} \to \text{joint(i+1)}}\)는 개별 조인트 변위를 나타내는 4×4 동차 좌표계 변환 행렬이다. 이러한 수식을 Isaac Sim 내에서 스크립트로 구현하면, 로봇이 특정 조인트 각도를 가질 때 실제 3D 공간에서 말단 이펙터가 어디에 위치하는지를 정밀하게 계산할 수 있다.

Isaac Sim의 인터페이스는 3D 뷰포트와 USD 기반의 Stage 패널을 중심으로, Property 패널, Timeline, Console, Script Editor, Extension Manager 등의 다양한 요소가 결합되어 있다. 이를 통해 복잡한 로봇 시뮬레이션 환경을 직관적이면서도 유연하게 제어할 수 있다.

본 장에서 소개한 인터페이스의 주요 기능과 구조를 정확히 이해해두면, 이후 ROS2 Humble와 연동하여 동작할 때에도 씬 구성, 물리 설정, 스크립트 작성 등 다양한 작업을 보다 원활하게 진행할 수 있다.
