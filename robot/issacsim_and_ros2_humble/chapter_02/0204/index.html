Isaac Sim 개요¶

Isaac Sim은 NVIDIA Omniverse 플랫폼 상에서 동작하는 강력한 로보틱스 시뮬레이션 환경이다. 물리 엔진, 그래픽 엔진, AI 툴킷 등을 통합적으로 활용하여 정밀한 로봇 시뮬레이션을 수행할 수 있다. ROS2 Humble과의 연동도 공식적으로 지원하므로, 복잡한 로봇 시뮬레이션뿐만 아니라 센서 및 상호작용 테스트에 광범위하게 사용된다.

시스템 요구사항¶

Isaac Sim을 원활하게 구동하려면 충분한 성능의 GPU가 필요하다. 일반적으로 NVIDIA RTX 계열 그래픽카드를 권장하며, CUDA 호환성도 필요하다. CPU 사양은 멀티스레드를 활용할 수 있는 프로세서가 좋고, 메모리는 시뮬레이션 규모에 따라 달라진다. 운영체제는 Ubuntu 20.04 이상(권장) 또는 Windows 10/11(64-bit) 환경에서 주로 사용된다. ROS2 Humble과 연동하려면 Ubuntu 22.04 이상의 환경에서 작업하는 것이 편리하다.

설치 파일 다운로드¶

NVIDIA Omniverse 포털(https://www.nvidia.com/ko-kr/omniverse/)에 접속하여 회원가입을 진행한다. 로그인한 뒤 Isaac Sim을 받을 수 있는 선택지를 찾을 수 있다. Omniverse Launcher를 통한 설치와 독립적인 압축 파일 설치가 모두 가능하다. Omniverse Launcher를 활용하면 GUI 환경에서 Isaac Sim을 손쉽게 설치할 수 있고, 독립적인 패키지를 다운로드 받아 직접 환경을 구성할 수도 있다.

Isaac Sim 설치¶

Omniverse Launcher를 이용하는 방법을 살펴본다.

Omniverse Launcher를 설치한 뒤 실행하면, Isaac Sim 항목을 찾아서 설치 버튼을 누를 수 있다. 설치 후에는 Launcher 내에서 Isaac Sim을 직접 실행할 수 있다. 독립 패키지로 설치하고 싶다면, Isaac Sim 관련 ZIP 파일을 다운로드 받아 다음과 같이 압축을 푼 뒤 경로를 설정한다.

unzip isaac_sim.zip -d ~/isaac-sim

압축을 해제한 뒤, Omniverse용으로 준비된 스크립트를 통해 의존성을 설치한다.

cd ~/isaac-sim
./install_ext_dependencies.sh

Linux에서 Headless 모드 또는 Docker 기반 설치가 필요하다면 NVIDIA GPU와 Docker 세팅이 제대로 구성되어야 한다. Docker CE 설치 후, NVIDIA Docker 툴킷을 다음과 같이 구성한다.

sudo apt-get update
sudo apt-get install \
    ca-certificates \
    curl \
    gnupg \
    lsb-release

curl https://get.docker.com | sudo bash
distribution=$(. /etc/os-release; echo$VERSION_CODENAME)
curl -s -L https://nvidia.github.io/libnvidia-container/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/libnvidia-container/$distribution/libnvidia-container.list | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
sudo apt-get update
sudo apt-get install -y nvidia-docker2
sudo systemctl restart docker

그 다음 Isaac Sim Docker 이미지를 받아서 컨테이너를 구동할 수도 있다. Docker 환경에서 Isaac Sim을 실행하면 호스트 시스템의 설정과 충돌을 최소화하면서 격리된 환경에서 시뮬레이션을 수행할 수 있다.

초기 실행 및 설정¶

설치가 완료되었다면 Isaac Sim을 처음 실행한다. Omniverse Launcher를 통한 실행이 가장 간단하며, 독립 패키지라면 isaac-sim.sh 또는 ./runapp.sh 등을 통해 실행할 수 있다.

초기 구동 시 Omniverse 로그인 화면이 표시될 수 있으니, 정상적으로 로그인된 계정을 사용한다. Isaac Sim이 시작되고 기본 씬이 로딩되면, 물리 시뮬레이션 엔진(PhysX)과 그래픽 엔진(RTX)에 대한 테스트가 가능하다.

ROS2 Humble과의 연동을 위해서는 ROS2 패키지가 설치된 환경에서 Isaac Sim의 ROS Extension을 활성화한다. Isaac Sim 내부의 Extensions 창에서 omni.isaac.ros2_bridge와 관련된 확장 기능을 찾아 체크박스를 활성화한 후, ROS2 Humble 토픽 혹은 액션, 서비스 등을 Isaac Sim 내의 로봇과 연결할 수 있다.

고급 설정 및 활용¶

Isaac Sim은 고급 설정을 통해 더욱 정밀한 로봇 시뮬레이션 환경을 구성할 수 있다. 물리 엔진 파라미터, 재질(Material) 설정, 센서 플러그인 등을 변경하여 실제 환경을 최대한 유사하게 재현할 수 있다. 또한 Lidar, Depth Camera, IMU 등의 가상 센서를 자유롭게 추가 가능하다.

환경 변수와 경로를 맞추는 작업도 중요하다. Isaac Sim을 설치한 후, ROS2 Humble 워크스페이스(~/ros2_ws) 등과 Isaac Sim 확장 디렉터리(~/isaac-sim/exts)가 상호 참조될 수 있도록 PATH나 LD_LIBRARY_PATH를 적절히 설정한다. 예를 들어, $HOME/.bashrc에서 다음과 같이 세팅할 수 있다.

export ISAAC_SIM_PATH=~/isaac-sim
export LD_LIBRARY_PATH=$ISAAC_SIM_PATH/exts:$LD_LIBRARY_PATH
export PATH=$ISAAC_SIM_PATH:$PATH

ROS2 메시지 타입을 Isaac Sim의 USD(Universal Scene Description) 객체와 올바르게 매핑하기 위해서는 URDF, SDF, USD 파일 구조를 확인해야 한다. 로봇의 좌표계 변환은 다음과 같은 형태의 동차 좌표계 변환 행렬을 통해 표현할 수 있다.

\[ \begin{align} \mathbf{T}_{base,link} = \begin{bmatrix} \mathbf{R} & \mathbf{p} \\ 0 & 1 \end{bmatrix} \\ \mathbf{p} \in \mathbb{R}^3,\quad \mathbf{R} \in \mathbb{R}^{3 \times 3} \end{align} \]

Isaac Sim에서는 이런 변환 행렬이 USD 씬 트리 내부에서 노드 간의 상위-하위 관계를 통해 구현된다.

간단한 설치 및 동작 확인 예제 (Python)¶

Isaac Sim 환경이 준비되었으면 Python API를 통해 동작 여부와 ROS2 연동을 간단히 점검할 수 있다. 아래 예제 코드는 Isaac Sim 내부 또는 ROS2 노드 실행 환경에서 시도할 수 있다.

import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class IsaacSimTestNode(Node):
    def __init__(self):
        super().__init__('isaac_sim_test_node')
        self.publisher_ = self.create_publisher(String, 'test_topic', 10)
        self.timer = self.create_timer(1.0, self.timer_callback)

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello Isaac Sim!'
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    node = IsaacSimTestNode()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

이 노드는 ROS2 토픽에 문자열을 계속 발행한다. Isaac Sim 상에서 ROS2 브리지가 설정되어 있으면 $ ros2 topic echo /test_topic 명령어로 해당 메시지를 확인할 수 있다. Isaac Sim의 확장(Extension) 창을 통해 ROS2가 제대로 연결되었는지 점검하자.

설치 프로세스 요약 (mermaid 활용 예시)¶

flowchart TB A[Omniverse Launcher 설치] --> B[Isaac Sim 다운로드] B --> C[설치 스크립트 실행] C --> D[초기 실행 & 설정] D --> E[ROS2 Bridge 활성화]

Isaac Sim의 설치부터 초기 설정 절차는 위의 흐름과 같이 진행된다. 시스템 요구사항 충족 여부를 미리 점검하고, 설치 후에는 물리 및 그래픽 성능 테스트를 시행한다. ROS2 Humble 브리지를 함께 구성한다면, Isaac Sim에서 제공하는 ROS2 Extension을 활성화하고 메시지 교환이 제대로 이뤄지는지 확인해야 한다.

이상으로 Isaac Sim 설치 및 초기 설정 절차에 대해 살펴보았다. Isaac Sim은 Omniverse 에코시스템 위에서 차세대 로보틱스 시뮬레이션 솔루션을 제공하므로, ROS2와 결합하여 활용한다면 더 풍부한 로봇 테스트 및 개발 환경을 구축할 수 있다.
