ROS2 Humble 설치 개요 및 시스템 요구사항¶

개요¶

ROS2 Humble은 다양한 로봇 하드웨어와 소프트웨어를 유연하게 결합하기 위한 미들웨어 프레임워크다. 이전 버전과 비교하면 DDS(데이터 분산 서비스)와 RTOS(실시간 운영체제) 호환성, 보안 강화 등 다양한 개선점이 있으며, 로보틱스 시스템을 구성하는 여러 노드들을 강력하고 안정적으로 연결해 주는 역할을 한다. 본 주제에서는 ROS2 Humble 설치에 앞서 요구되는 OS 정보, 하드웨어 권장사항 등을 다룬다. 이어서 패키지 설치, 소스 코드 빌드, 환경 설정처럼 기초부터 고급까지 아우르는 설치 전략을 간략히 개관한다.

시스템 요구사항¶

ROS2 Humble은 주로 Ubuntu 22.04(LTS)에서 안정적으로 동작한다. 다른 리눅스 배포판(예: Debian, Fedora)이나 Windows, macOS 환경에서도 부분적으로 지원되지만, 공식적으로 가장 원활한 호환성을 보장받으려면 Ubuntu 22.04를 사용하는 것이 좋다. CPU는 멀티스레딩을 지원하는 64비트 프로세서를 권장하며, 메모리는 최소 2GB 이상을 요구한다. 그러나 실제 로봇 개발 환경에서는 여러 패키지를 동시에 실행하게 되므로 8GB 이상의 메모리를 여유 있게 준비하는 편이 안정적이다. 디스크 공간도 ROS2 전체 패키지와 의존성 라이브러리를 설치하기 위해 최소 5GB 정도가 필요하지만, 동영상 캡처나 3D 시뮬레이션 등 추가 활용을 고려하면 20GB 이상 확보하는 것이 무난하다.

GPU 가속이 필요한 시뮬레이션(예: RViz, Gazebo, Isaac Sim 등)을 사용할 경우에는 NVIDIA 그래픽 카드와 호환되는 드라이버 및 CUDA 버전을 맞추어 두어야 한다. 특히 3D 시뮬레이션 환경에서의 성능 향상을 위해서는 최신 드라이버를 유지하고, 노트북보다는 데스크톱 또는 Workstation 급 장비를 준비하는 편이 좋다. 만일 임베디드 장치(NVIDIA Jetson 시리즈 등)에서 활용해야 한다면, ROS2 Humble을 빌드하거나 크로스 컴파일하는 과정을 고려해야 한다.

설치 방식 개요¶

ROS2 Humble을 설치하는 일반적인 방식은 다음과 같다. 첫째, 공인 패키지 저장소(apt 레포지토리)에서 제공되는 바이너리 패키지를 이용해 빠르게 설치할 수 있다. 둘째, 소스 코드를 직접 받아서 빌드함으로써 세밀하게 버전을 관리하거나 커스텀 패키지를 적용할 수 있다. 배포판 설치용 Docker 이미지를 활용하는 방법도 있으며, 별도의 가상환경에서 ROS2 Humble 환경을 격리할 수도 있다.

공인 apt 레포지토리에서 설치하는 가장 단순한 예시는 다음과 같다.

sudo apt update
sudo apt install ros-humble-desktop

설치 후에는 터미널 프로파일(예: ~/.bashrc)에 다음 환경 변수를 추가한다.

source /opt/ros/humble/setup.bash

이제 새로운 터미널을 열거나 위 스크립트를 실행해 두면, ros2 명령어를 바로 사용할 수 있다.

고급 설치 사항¶

ROS2를 보다 세밀하게 다루려면, 소스 코드 빌드와 워크스페이스 설정 과정을 숙지하는 것이 중요하다. 복잡한 프로젝트에서는 여러 패키지를 동시에 수정하고, git 서브모듈 및 의존성 라이브러리를 다양하게 관리하기 때문이다. 예를 들어 여러 버전의 ROS2를 공존시켜야 할 경우, 별도의 워크스페이스를 만들어 colcon 빌드 도구로 패키지들을 구분 지어 관리한다.

소스 빌드의 간략한 과정은 다음과 같다. 먼저 작업용 디렉터리를 만들고 그 아래에서 ros2 소스 코드를 받아 온다. 이후 의존성을 설치한 뒤 빌드하면 된다.

mkdir -p ~/ros2_humble_ws/src
cd ~/ros2_humble_ws/src
git clone https://github.com/ros2/ros2.git -b humble
cd ..
rosdep update
rosdep install --from-paths src --ignore-src -y
colcon build --symlink-install

빌드 후에는 결과물 설치를 위한 스크립트를 워크스페이스에 등록한다.

source ~/ros2_humble_ws/install/setup.bash

만약 Isaac Sim 등 GPU 가속 시뮬레이션과의 연동을 고려한다면, 그래픽 드라이버가 올바르게 설치되어 있어야 하며 CUDA 버전 호환성도 점검이 필요하다. NVIDIA Jetson 시리즈 같은 임베디드 플랫폼에서 소스 빌드를 진행할 경우, 크로스 컴파일 도구체인을 설정하거나 해당 플랫폼용 패키지를 제공해 주는 별도 레포지토리를 활용할 수도 있다.

추가 예시: 좌표변환 행렬¶

ROS2는 좌표계 변환을 주로 TF2 라이브러리로 처리한다. 예를 들어 2D 혹은 3D 공간에서의 임의 좌표계를 다른 좌표계로 옮기는 변환 행렬은 다음과 같이 표현된다. $$ \begin{align} \mathbf{T} &= \begin{bmatrix} \mathbf{R} & \mathbf{p}??라인피드??\ \mathbf{0} & 1 \end{bmatrix} \end{align} $$ 여기서 \(\mathbf{R}\)은 회전 행렬이며, \(\mathbf{p}\)는 평행이동 벡터다. 예를 들어 3차원 공간 \(\mathbb{R}^3\)에서 \(\mathbf{R}\)은 \(3 \times 3\) 행렬, \(\mathbf{p}\)는 3차원 벡터가 된다. 이러한 변환 행렬은 로봇 팔, 이동 로봇, 시뮬레이션 환경에서의 객체 배치 등을 명확하게 정의하는 데 중요한 역할을 한다.

ROS2 Humble 설치 후 이처럼 다양한 좌표 변환 체계를 활용해 본인의 로봇 모델이나 시뮬레이션 환경을 자유롭게 설정할 수 있다. Isaac Sim과 결합하면 하드웨어 수준의 물리 시뮬레이션과 직접 연동하여, 로봇 제어 알고리즘을 정밀하게 검증할 수도 있다.

결론¶

ROS2 Humble을 시작하기 전에는 Ubuntu 22.04 환경이 잘 갖추어져 있어야 하고, 적절한 CPU와 메모리, GPU 드라이버를 확보하는 것이 중요하다. apt를 통한 설치는 간단하지만, 특정 요구사항이 있다면 소스 빌드나 Docker, 크로스 컴파일을 고려할 수 있다. 환경 변수 설정, 워크스페이스 관리, 좌표 변환 등은 ROS2 사용 시 반드시 숙지해야 할 핵심 개념이다. 이러한 준비와 지식을 바탕으로 Isaac Sim, TF2, 다양한 퍼셉션·모션 플래닝 라이브러리를 유기적으로 조합해 로봇 개발을 원활히 진행할 수 있다.
