ROS2 Humble 출시 배경과 주요 개선점¶

ROS2 Humble의 역사적 배경¶

ROS(Robot Operating System)은 오픈 소스 로보틱스 생태계를 이끌어 온 대표적인 프레임워크이다. ROS1 시리즈는 다양한 기능과 패키지를 통해 연구 및 산업용 로봇 개발을 지원해 왔다. 하지만 메시지 전달의 실시간성, 멀티 플랫폼 지원, 네트워크 안정성 등에 대한 요구가 높아짐에 따라 ROS2가 등장했다. ROS2는 DDS(Data Distribution Service)를 기반으로 재설계하여 여러 운영체제와 네트워크 환경에서 안정적이고 확장성 있는 통신을 제공한다.

Humble Hawksbill은 ROS2의 하나의 배포판(distro)으로서 장기 지원 버전(LTS) 특성을 갖추어 오랫동안 유지보수된다. 이전 배포판인 Galactic이나 Foxy에 비해 성능, 안정성, 호환성 측면을 크게 강화했으며, 개발자들이 실제 현장에서 ROS2를 활용하기 용이하도록 다양한 개선사항을 제공한다.

ROS2 Humble의 주요 개선 사항¶

ROS2 Humble은 실시간성 향상, 보안 모델 강화, 노드 간 통신의 안정성 증대 등을 목표로 기능을 세분화하고 개선했다. 멀티 플랫폼 지원 역시 기존의 Ubuntu, Windows, macOS뿐 아니라 다양한 임베디드 플랫폼까지 확장되어 복잡한 로봇 시스템에서의 이식성을 높였다. 또한 RMW(Robot Middleware) 계층에서의 성능 최적화와 함께 DDS 구현체 간 호환성 검증이 더욱 철저히 이루어져 실제 시스템에서 발생하는 통신 지연이나 충돌을 최소화했다.

QoS(품질 보장) 설정이 더욱 직관적으로 바뀌어, 사용자 애플리케이션에 적합한 통신 전략을 세우기 쉬워졌다. 이러한 QoS 확장은 고주파 센서 데이터를 다루는 무인 자율주행이나 협동 로보틱스 영역에서 중요한 역할을 하며, 데이터 손실 없이 높은 주기로 메시지를 주고받을 수 있는 발판을 제공한다.

DDS와의 연동 효율 향상¶

ROS2의 핵심은 DDS를 통한 퍼블리셔-서브스크라이버 모델이다. Humble에서는 RTI Connext, Fast DDS, Cyclone DDS 등 주요 DDS 구현체와의 연동 성능을 최적화하여, 데이터 버스를 통해 주고받는 메시지의 처리 지연을 크게 줄였다. 더불어 DDS 자체의 보안 기능과 ROS2의 보안 계층을 통합적으로 활용할 수 있도록 개선해, TLS나 암호화 키 관리 등 네트워크 레벨 보안을 보다 쉽게 구현할 수 있다.

DDS 토픽의 트래픽을 확인하고 제어하기 위한 툴도 업데이트되어, 복잡한 환경에서의 트러블슈팅이 한층 편리해졌다. 덕분에 개발자는 다양한 센서나 액추에이터가 동시에 구동되는 상황에서도 메시지가 유실되거나 병목이 일어나는 지점을 빠르게 찾아낼 수 있다.

QoS 모델의 확장¶

ROS2에서 QoS는 퍼블리셔와 서브스크라이버를 연결하는 핵심 규칙이다. Humble에서는 QoS 파라미터를 보다 세밀하게 조정할 수 있는 API가 제공되어, 개발자가 통신 지연, 신뢰성, 이력(depth) 설정 등을 자유롭게 튜닝할 수 있다. 네트워크 자원이 부족하거나, 반드시 메시지를 모두 수신해야 하는 환경에서 QoS 파라미터를 최적화하면 시스템 성능과 안정성을 높일 수 있다.

이를 좀 더 수식으로 나타낼 때, QoS에 따른 메시지 지연 시간 \(T_{\mathrm{latency}}\)를 가정해볼 수 있다. 퍼블리셔에서 서브스크라이버로 전송되는 메시지 경로에서의 총 지연 시간은 통신 지연과 콜백 처리 시간을 모두 고려해야 한다고 할 때, $$ \begin{align} T_{\mathrm{latency}} = T_{\mathrm{comm}} + T_{\mathrm{callback}} \end{align} $$ 이때 \(T_{\mathrm{comm}}\)은 DDS 네트워크를 통해 발생하는 전송 지연이며, \(T_{\mathrm{callback}}\)은 ROS2 노드에서 메시지를 수신 후 처리하는 데 걸리는 시간을 말한다. Humble에서는 \(T_{\mathrm{comm}}\)를 단축하기 위해 DDS 구현체에 특화된 최적화가 이루어졌고, 콜백 설계도 노드 간 스레드 운용을 탄력적으로 처리하도록 개선되었다.

Real-time 지원과 성능¶

ROS2는 Real-time 시스템 운용이 가능하도록 설계되었다. Humble 버전에서는 더 높은 수준의 실시간성 구현이 가능하도록 다양한 실험과 최적화가 반영되었다. RT-Preempt 커널 사용 시 콜백 스케줄링을 좀 더 확실하게 관리할 수 있으며, ROS Executor의 구성 방식을 선택하거나 커스텀 Executor를 구현함으로써, 멀티 코어 CPU 환경에서의 실시간 성능을 최대한 끌어낼 수 있다. $$ \begin{align} \mathbf{x}(t + \Delta t) = \mathbf{A} \, \mathbf{x}(t) + \mathbf{B} \, \mathbf{u}(t) ??라인피드??\ \end{align} $$ 위와 같은 로봇 동역학 모델을 다룰 때, 실시간성이 부족하면 \(\mathbf{x}(t)\) 상태 추정이 늦어지고 제어 명령 \(\mathbf{u}(t)\)가 시의적절하게 적용되지 못한다. Humble의 실시간 시스템 지원은 이러한 제어 시스템의 성능 저하를 최소화한다.

최신 툴체인 및 플랫폼 호환성¶

ROS2 Humble은 Ubuntu 22.04, Windows 10/11, macOS 등의 주요 플랫폼에서 안정적으로 동작한다. RMW 구현체와 빌드 시스템(Colcon)을 포함한 전체 툴체인이 업데이트되어, 복잡한 의존성을 자동으로 관리하고, 크로스 컴파일을 비롯한 멀티 플랫폼 배포를 쉽게 할 수 있다. 실제 로봇 하드웨어뿐 아니라 Docker 컨테이너나 가상화 환경에서도 손쉽게 ROS2 환경을 구성할 수 있어, CI/CD 파이프라인과 같은 대규모 개발 프로세스에 용이하다.

고급 예제: ROS2 Humble 노드 구조 분석¶

ROS2 Humble에서 노드 간 통신이 어떻게 구성되는지 간단한 예시를 통해 살펴본다. 퍼블리셔 노드와 서브스크라이버 노드를 구성하고, 서로 다른 QoS 프로파일을 적용해보면 네트워크 상태에 따라 메시지의 전달 및 처리가 어떻게 달라지는지 실험할 수 있다.

아래는 Python 예제 코드로, 간단한 퍼블리셔 노드를 정의한다.

<code>
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimplePublisher(Node):
    def __init__(self):
        super().__init__('simple_publisher')
        self.publisher_ = self.create_publisher(
            String,
            'chatter',
            qos_profile=rclpy.qos.QoSProfile(depth=10)
        )
        self.timer = self.create_timer(0.5, self.timer_callback)
        self.count = 0

    def timer_callback(self):
        msg = String()
        msg.data = f"Hello ROS2 Humble: {self.count}"
        self.publisher_.publish(msg)
        self.get_logger().info(f"Publishing: {msg.data}")
        self.count += 1

def main(args=None):
    rclpy.init(args=args)
    node = SimplePublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
</code>

퍼블리셔 노드는 \(0.5,\mathrm{s}\) 간격으로 문자열 메시지를 발행한다. QoS Profile을 지정해 메시지 보관 버퍼(depth)를 10으로 설정했다. 네트워크 환경이 좋지 않아도, 서브스크라이버는 일시적으로 끊긴 메시지를 나중에 받아볼 수 있다.

다음은 Humble 버전에서 퍼블리셔가 발행하는 토픽 리스트를 확인하는 셸 명령 예시이다.

<shell>
ros2 topic list
</shell>

이처럼 ROS2 Humble에서는 QoS를 상황에 맞게 세부 설정할 수 있으며, DDS 통신의 이점을 쉽게 활용할 수 있다.

결론¶

ROS2 Humble은 이전 버전 대비 큰 폭의 개선점을 제공하며, 장기 지원 버전인 만큼 안정적이고 확장성 높은 로봇 개발 환경을 갖추었다. DDS 기반의 강화된 통신 성능, 다양한 운영체제와 하드웨어 플랫폼에 대한 호환성, 실시간 및 보안 요구 사항 충족 등 실무에서 요구되는 대부분의 요소를 지원한다. 특히 QoS 모델이 확장되고, Executor와 스케줄링 방식이 세분화됨으로써 고급 로봇 시스템에서도 ROS2를 충분히 활용할 수 있게 되었다. Humble 버전을 숙지하고 나면, 더욱 복잡한 로보틱스 환경에서도 신뢰도 높은 애플리케이션을 개발할 수 있을 것이다.
