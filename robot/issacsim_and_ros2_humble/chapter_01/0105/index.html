ROS2 Humble은 이전 버전에서 누적된 여러 개선 사항을 대폭 반영한 장기 지원(LTS) 버전이며, 특히 다양한 로봇 환경에서 안정성과 성능을 강화하기 위한 변화가 많다. 기본적인 동작 체계 변화에서부터 고급 미들웨어 설정에 이르기까지, ROS2 Humble에서 주목해야 할 주요 변경 사항을 엄밀하게 살펴보자.

Ubuntu 22.04(Jammy) 기반의 장기 지원 특성¶

ROS2 Humble은 기본적으로 Ubuntu 22.04(Jammy)를 공식 지원 대상으로 삼는다. Jammy가 오랜 기간 지원되는 LTS(Long Term Support) 배포판인 것처럼, ROS2 Humble 역시 장기 지원 릴리스로 선정되어 장기간 안정적인 패키지 업데이트와 보안 패치를 기대할 수 있다. 이는 실무 환경에서 한번 구축한 로봇 시스템을 오랫동안 유지 보수해야 하는 경우에 매우 큰 이점이 된다.

RMW(Robot Middleware) 및 DDS 관련 변경¶

ROS2의 저수준 통신 계층은 RMW(Robot Middleware) 인터페이스를 통해 다양한 DDS(Data Distribution Service) 구현체를 활용할 수 있다. Humble에서는 일부 DDS 구현체들이 더 최신 버전으로 업데이트되면서 성능과 호환성이 개선되었고, Cyclone DDS나 Fast DDS와 같은 주요 DDS 구현체에서도 안정적인 통신 지연 보장과 메모리 사용 최적화가 이루어졌다. 또한, 특정 RMW 구현체 간 상호 호환성을 유지하면서도, 실시간성(real-time)의 요구 사항을 더 잘 충족할 수 있도록 API 수준에서 세부 수정이 이루어졌다.

QoS(품질 정책) 동작 방식 개선¶

Humble에서 기존의 QoS(Quality of Service) 정책이 더욱 직관적으로 설정될 수 있도록 기본값과 사용법이 다소 변경되었다. 특히 내장된 DDS 계층이 제공하는 신뢰성, 히스토리, 샘플 깊이 등을 보다 명확하게 표시하고, QoS 호환성 오류가 발생했을 때 진단 메시지와 디버깅 정보가 더 풍부해졌다. 이는 노드 간의 통신 품질을 보장하거나 성능 최적화를 위해 세밀하게 파라미터를 튜닝해야 할 때, 사용자가 시행착오를 줄이고 시스템을 빠르게 안정화하도록 도움을 준다.

실행 환경 및 멀티 플랫폼 지원 강화¶

ROS2 Humble에서는 데스크톱뿐만 아니라, 임베디드 보드, ARM 계열 CPU, 애플 실리콘(M1, M2 시리즈) 등 다양한 플랫폼으로 확장할 수 있도록 빌드 시스템과 의존성 관리가 한층 더 유연하게 개선되었다. 예를 들어 Docker 이미지를 통한 다중 플랫폼 환경 구성도 공식적으로 지원이 강화되어, 복잡한 의존성 충돌 없이 컨테이너 단위로 ROS2 Humble 환경을 배포하기가 훨씬 수월해졌다.

실시간성 및 성능 최적화¶

실시간성(real-time)은 로봇 시스템에서 매우 중요한 이슈이다. 이전 버전부터 지속적으로 언급되어 왔던 실시간성 문제를 해결하기 위해, Humble에서는 rclcpp 레벨의 Executor 및 스레드 처리 방식을 최적화하여 스케줄링 지연을 낮추고, 결과적으로 메시지 전송 지연이나 센서-액추에이터 간 통신이 더 안정적으로 이루어지도록 개선했다. 또한 lock-free 데이터 구조와 메모리 풀(memory pool)을 보다 적극적으로 활용함으로써 높은 우선순위 스레드 간 충돌을 줄이고, 상황에 따라 특히 중요한 콜백(callback)이 필요한 시점에 즉각 실행되도록 설계되었다.

노드 컴포지션(Composition) 및 Lifecycle 개선¶

ROS2의 컴포지션 기능은 서로 다른 노드를 하나의 프로세스로 결합해 성능을 높이고 자원을 절감하게 하는 핵심 기술이다. Humble에서는 이러한 노드 컴포지션 과정을 좀 더 직관적으로 지원하기 위해, colcon 빌드와 연동하는 방식과 rclcpp의 컴포넌트 등록 방식을 개선했다. 또한 Lifecycle 노드에서 상태 전이를 시각화하거나 이벤트를 처리할 때 더 명확한 상태 전이 다이어그램을 제어할 수 있게 하여, 복잡한 로보틱스 애플리케이션 개발 시 안정성과 가독성을 높였다.

보안(SROS2) 및 안전성 강화¶

Humble은 SROS2(Security ROS2) 프레임워크를 활용해 네트워크 상에서의 로봇 데이터 전송을 보호하고, 인증, 암호화, 권한 관리를 지원하는 기능을 확대했다. 머신 간 통신에서 암호화가 필수적인 환경(예: 의료 로봇, 산업용 로봇)에서 적용하기 쉬워지도록 GUI나 CLI(Command Line Interface) 도구가 보완되었고, 기본 템플릿이 제공되어 실질적인 보안 구축 난이도를 낮추었다. 또한, 호환성을 유지하면서도 DDS 수준의 보안 정책과 연계가 가능해졌으며, 이를 통해 강력한 미들웨어 보안과 통합된 접근이 가능해졌다.

구축(빌드) 및 의존성 관리 간소화¶

ROS2의 공식 빌드 툴인 colcon이 보다 능동적으로 패키지 의존성을 검사하고, 불필요한 재빌드를 방지하는 기능을 향상시키면서 전체 빌드 시간을 단축했다. Humble에서는 C++17 이상의 표준 사용이 권장되고, 일부 패키지에서는 더 이상 C++14를 공식 지원하지 않는 등의 컴파일러, 링커 옵션 변화가 있다. 따라서 기존 프로젝트를 이전할 때, 코드 호환성 검토가 필수적이다. 파이썬 버전 역시 Ubuntu 22.04의 표준인 Python 3.10 환경과의 호환성이 최우선으로 유지된다.

예시: ROS2 Humble에서의 기본 노드(Python)¶

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimpleTalker(Node):
    def __init__(self):
        super().__init__('simple_talker')
        self.publisher_ = self.create_publisher(String, 'chatter', 10)
        self.timer = self.create_timer(0.5, self.timer_callback)
        self.count = 0

    def timer_callback(self):
        msg = String()
        msg.data = f"Hello ROS2 Humble: {self.count}"
        self.publisher_.publish(msg)
        self.get_logger().info(f"Publishing: {msg.data}")
        self.count += 1

def main():
    rclpy.init()
    node = SimpleTalker()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

위 코드는 ROS2 Humble 환경에서 간단히 동작하는 토커(talker) 노드를 구현한 예시다. 이전 버전의 ROS2에서 사용했던 C++14, Python 3.6 등의 환경과는 달리, Humble은 C++17 이상, Python 3.10 등의 최신 런타임을 사용하므로 이런 부분에 대해 호환성을 확인해야 한다.

로봇 좌표 변환 수식 예시¶

ROS2에서 자주 다루는 좌표 변환(Transform)은 고차원의 회전과 병진(Translation)을 하나의 행렬로 표현한다. 4×4 형태의 동차변환행렬(homogeneous transformation matrix)은 다음과 같이 표현할 수 있다.

\[ \begin{align} \mathbf{T} = \begin{pmatrix} \mathbf{R} & \mathbf{p} \\ 0 & 1 \end{pmatrix} \\ \mathbf{R} \in \mathbb{R}^{3 \times 3},\ \mathbf{p} \in \mathbb{R}^{3} \end{align} \]

여기서 \(\mathbf{R}\)은 회전 행렬, \(\mathbf{p}\)는 병진 벡터를 나타낸다. ROS2 Humble에서도 tf2 패키지 등을 사용하여 이러한 좌표 변환을 효율적으로 다룰 수 있으며, 시스템 성능 최적화와 실시간 동작을 위해 내부 메모리 접근 패턴이나 콜백 구조가 더욱 최적화되었다.

마치며¶

ROS2 Humble은 더욱 안정적이고 강력한 기능들을 제공하면서도, 이전 버전에서 사용되던 다양한 패키지들과 호환성을 최대한 유지하려 노력한 릴리스다. QoS, DDS, 빌드 시스템 등의 내부 구조 개선 덕분에 여러 개발자가 복잡한 로봇 애플리케이션을 빠르게 구축하고, 여기에 Isaac Sim과 같은 시뮬레이션 환경까지 결합하여 고성능 로봇 시뮬레이션을 현실적으로 진행할 수 있다. 향후 ROS2 릴리스와 비교해도 Humble이 갖는 장기 지원 특성, 그리고 더욱 직관적인 통합 환경이 로봇 개발의 안정적인 토대가 될 것으로 기대된다.
