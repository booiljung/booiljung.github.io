일반 개요¶

ROS2 Humble은 장기 지원(LTS) 버전으로 Ubuntu 22.04(Jammy) 환경을 기본적으로 지원한다. 이전 버전인 Foxy, Galactic, Rolling 등에서 축적된 기능적·기술적 개선 사항이 반영되었고, 여러 가지 성능 및 호환성 측면에서도 진일보한 모습을 보인다. 특히 실시간 처리, 보안성, 개발 편의성, QoS 정책, 클라이언트 라이브러리 개선 등 다양한 변화가 존재한다. Isaac Sim과의 연계 관점에서는 ROS2 Humble이 제공하는 개선된 DDS 미들웨어 확장과 다양한 패키지 호환성을 통해 시뮬레이션과 실제 로봇 환경 간 데이터 교환 효율이 더욱 높아진다는 점이 강조된다.

빌드 및 종속성 변화¶

ROS2 Humble은 빌드 도구로서 Colcon을 기본적으로 사용하고, CMake 최소 버전, Python3, 그리고 GCC 11(혹은 Clang 해당 버전)을 표준으로 삼는다. 이는 더 현대화된 컴파일러 기능을 활용하고, C++17 표준을 안정적으로 지원하기 위함이다. Python 바인딩(rclpy)은 CPython 3.10 이상과 함께 작동하며 여러 최적화 및 API 호환성이 개선되었다.

새로운 OS 지원 정책 또한 도입되어 macOS와 Windows에서도 더 원활하게 동작할 수 있도록 크로스 플랫폼 종속성 충돌이 줄어들었다. Isaac Sim과 연동되는 Python API를 사용하는 환경에서는 Python 3.10 버전을 표준으로 삼는 것이 권장된다.

RMW 계층과 DDS 개선¶

ROS2에서 가장 핵심적인 변화 중 하나는 RMW(ROS Middleware Interface) 계층에서의 성능 및 안정성 향상이다. 특히 CycloneDDS와 Fast DDS(rmw_fastrtps)의 개선이 두드러지며, 네트워크 대역폭 활용도와 노드 간 통신 레이턴시가 줄어들었다. Isaac Sim에서 센서 데이터를 송수신하거나 물리 엔진과 로봇 간의 고주파수 트랜스포트를 수행할 때 이러한 DDS의 발전 사항은 매우 중요한 부분이다.

Humble에서는 소켓 통신 알고리즘 개선, RTPS(Real-Time Publish-Subscribe) 구현 최적화, 일부 QoS 정책(Profile, Reliability, History, Depth 등) 사용성 향상이 포함되었다. 특정 DDS 구현체 선택 시 발생하던 QoS 호환성 문제와 로스 문제가 크게 완화되어, 실시간성이나 고속 통신을 요구하는 Isaac Sim의 로보틱스 시뮬레이션에서 더욱 안정적인 퍼포먼스를 기대할 수 있다.

QoS 정책 및 보안성 강화¶

ROS2 Humble에서는 QoS(품질 서비스) 정책이 더욱 세분화되고 구성 방식이 직관적으로 개선되었다. Depth, Reliability, Durability 등 기본 요소 외에도, Deadline, Liveliness, Lifespan 등 고급 옵션의 동작 검증이 보다 체계적으로 이루어진다. 이를 통해 시뮬레이션과 실제 로봇 데이터 스트림 간의 동기화를 보다 세밀하게 제어할 수 있다.

보안 측면에서도 DDS Security 플러그인이 안정적으로 개선되어, 노드별 접근 제어, 데이터 암호화, 권한 인증 등 기능을 더욱 쉽게 설정할 수 있다. Isaac Sim과 연결된 로봇 어플리케이션에서 주고받는 3D 스캔 데이터, 이미지, 제어 데이터 등을 안전하게 보호하기 위한 메커니즘이 확장되었다.

rclcpp, rclpy, 실행 모델 향상¶

ROS2 C++ 클라이언트 라이브러리(rclcpp)와 Python 클라이언트 라이브러리(rclpy)도 여러 면에서 발전되었다. 노드, 콜백 그룹, Executor(실행자) 개념이 유지되는 가운데, Executor 간 통신 충돌을 줄이기 위한 Lock-free 방식, 실시간 스레드 우선순위 조정 방식 등이 실험적으로 도입되었다. 이를 통해 로봇이 고빈도 센서 데이터를 처리하는 시나리오에서 노드 응답성이 향상된다.

Isaac Sim에서 물리 환경 시뮬레이션과 병렬적으로 ROS2 노드를 실행할 때, 다중 스레드 Executor의 활용이 이전 버전 대비 더 안정적이어서 콜백 지연이나 프레임 드롭 등의 현상이 감소된다.

노드 생명주기와 컴포지션¶

Humble에서는 라이프사이클 노드(lifecycle node)의 전이 과정이 더욱 자동화되고 직관적으로 구성된다. 노드가 활성(active), 비활성(inactive) 상태를 전환할 때 발생하는 이벤트를 세밀히 모니터링할 수 있고, Isaac Sim에서 시뮬레이션 상태 변화를 감지하거나 모델 로딩 시점을 제어할 때 유용하다.

컴포지션(Composition) 기능도 한층 강화되어, 다수의 노드를 하나의 프로세스로 묶어 실행하는 시나리오에서 메모리 사용량이 절감되고, 메시지 교환이 프로세스 내부적으로 최적화된다. Isaac Sim에서 카메라 노드, LiDAR 노드, 이동 로봇 제어 노드를 일괄 컴포지션으로 묶을 수 있어, 시뮬레이션 자원 효율성이 향상된다.

수학적 표현 예시¶

ROS2 노드에서 좌표 변환(transform)을 수행하거나 Isaac Sim과 로봇 간의 좌표계를 맞출 때, 2D 또는 3D 공간에서의 변환 행렬을 자주 사용한다. 2D 변환 행렬 예시는 다음과 같이 표현할 수 있다.

\[ \begin{align} \mathbf{T} = \begin{bmatrix} \cos\theta & -\sin\theta & x \,\\ \sin\theta & \cos\theta & y \,\\ 0 & 0 & 1 \end{bmatrix} \end{align} \]

여기서 \(\theta\)는 회전 각도이고 \(\mathbf{p} = (x, y)\)는 평행 이동 벡터다. Isaac Sim에서 출력되는 로컬 좌표계 정보와 실제 로봇의 글로벌 좌표계를 동기화할 때 이러한 변환이 활용된다.

간단한 C++ 코드 예시¶

ROS2 Humble의 rclcpp를 이용해 토픽 퍼블리셔를 생성하는 간단한 예시는 다음과 같다.

#include <rclcpp/rclcpp.hpp>
#include <std_msgs/msg/string.hpp>

class SimplePublisher : public rclcpp::Node
{
public:
  SimplePublisher() : Node("simple_publisher")
  {
    publisher_ = this->create_publisher<std_msgs::msg::String>("chatter", 10);
    timer_ = this->create_wall_timer(
      std::chrono::milliseconds(500),
      [this]() { publishMessage(); }
    );
  }

private:
  void publishMessage()
  {
    auto msg = std_msgs::msg::String();
    msg.data = "Hello from ROS2 Humble!";
    RCLCPP_INFO(this->get_logger(), "Publishing: %s", msg.data.c_str());
    publisher_->publish(msg);
  }

  rclcpp::Publisher<std_msgs::msg::String>::SharedPtr publisher_;
  rclcpp::TimerBase::SharedPtr timer_;
};

int main(int argc, char** argv)
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<SimplePublisher>());
  rclcpp::shutdown();
  return 0;
}

Humble에서는 rclcpp 라이브러리의 코드 사용 패턴은 기존 Foxy, Galactic에서의 방식과 크게 다르지 않지만, 내부 동작 성능과 안정성이 향상되었다. Isaac Sim과 연동 시에는 센서 토픽이나 제어 토픽을 유사한 방식으로 생성하고 구독한다.

간단한 다이어그램 예시¶

Humble 환경에서 Isaac Sim과 ROS2 노드가 어떻게 통신하는지를 단순화한 노드 연결 예시는 다음과 같이 표현할 수 있다.

flowchart LR A[Isaac Sim] ---|DDS| B[ROS2 Humble 노드1] A ---|DDS| C[ROS2 Humble 노드2] B ---|ROS2 Topic| C

Isaac Sim은 ROS2 브리지 혹은 로컬 DDS 구현을 통해 여러 ROS2 노드와 데이터를 주고받는다. Humble의 DDS 확장 개선으로 이러한 통신 경로가 더 짧아지고 안정적이다.

정리¶

ROS2 Humble은 LTS 버전으로서, 빌드 환경, DDS 통신 계층, QoS 및 보안 강화, rclcpp·rclpy 라이브러리, 노드 생명주기, 컴포지션 등 다양한 영역에서 향상된 기능을 제공한다. Isaac Sim에서 시뮬레이션을 통해 개발되는 로봇 응용 프로그램은 이러한 개선사항을 통해 데이터 송수신 안정성과 성능을 더욱 극대화할 수 있다.
