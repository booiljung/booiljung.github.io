개발 환경 및 하드웨어 요구 사항¶

Isaac Sim과 ROS 2 Humble을 활용하려면 하드웨어와 소프트웨어 전반에 걸친 요구 사항을 충분히 고려해야 한다. 일반적으로 Isaac Sim을 원활히 구동하기 위해서는 NVIDIA GPU가 탑재된 환경이 필수적이다. 특히 CUDA 기능을 지원하는 GPU와 적절한 드라이버 버전이 필요하며, GPU 메모리는 가능하면 큰 용량을 갖추는 편이 시뮬레이션 안정성에 유리하다. CPU 역시 멀티 코어 환경을 권장하며, RAM 용량 또한 16GB 이상을 확보해야 복잡한 로보틱스 시뮬레이션을 운영하는 데에 유리하다.

Isaac Sim은 그래픽 및 물리 시뮬레이션을 처리해야 하므로, GPU가 없는 환경에서는 원활한 실행이 어려울 수 있다. Linux 환경에서는 일반적으로 NVIDIA 드라이버가 470 이상(예: 515, 525 등)이 적절하며, Windows 사용자의 경우 NVIDIA Omniverse에서 제시하는 구체적인 버전 가이드를 참조해야 한다.

운영체제 선택 및 설치 조건¶

ROS 2 Humble은 주로 Ubuntu 22.04 LTS 버전을 공식적으로 지원한다. Isaac Sim은 Windows나 Linux 환경 모두에서 동작 가능하지만, 실제 로봇 시스템과의 연동에는 Linux 환경에서의 개발이 선호된다. 운영체제 설치 시 Secure Boot가 활성화되어 있으면 NVIDIA 드라이버 설치나 DKMS 관련 설정에서 어려움이 발생할 수 있으므로, 필요에 따라 Secure Boot를 해제하거나 서명 설정을 수행해야 한다.

Ubuntu를 선택했다면 의존성 패키지 설치와 드라이버 설정을 위해 다음과 같은 형태의 시스템 패키지 업데이트를 수행할 수 있다.

sudo apt update
sudo apt upgrade
sudo apt install build-essential dkms

NVIDIA 드라이버 자동 설치 옵션을 사용하거나, 별도의 PPA를 통해 최신 드라이버를 선택할 수 있다.

Isaac Sim 설치를 위한 준비¶

NVIDIA Omniverse Launcher를 통해 Isaac Sim을 설치할 수 있다. Omniverse Launcher는 GUI 기반으로 Isaac Sim을 내려받고 설치 과정을 자동화해 주지만, CLI 환경에서 설치하려면 별도의 설치 파일을 받아 직접 압축을 풀고, Python 패키지 환경을 구성해야 한다.

Omniverse Launcher를 이용하는 경우, 회원 가입 후 로그인하고 Isaac Sim 버전을 선택하여 설치한다. Isaac Sim을 초기 실행하면 추가 의존 패키지나 드라이버 호환성을 점검한다. Linux 사용자라면 별도의 LIBGL이나 X11 설정이 필요할 수도 있다.

ROS 2 Humble 설치 및 환경 설정¶

ROS 2 Humble 버전은 Ubuntu 22.04 LTS를 공식 지원한다. apt 패키지 관리자와 ROS 2 리포지토리를 이용하여 간편하게 설치할 수 있다. 가장 일반적인 설치 방식은 다음과 같은 형태로 진행된다.

sudo apt update
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update
sudo apt install curl
curl -s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | sudo apt-key add -
sudo sh -c 'echo "deb http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" > /etc/apt/sources.list.d/ros2-latest.list'
sudo apt update
sudo apt install ros-humble-desktop

설치가 완료되면 다음과 같이 환경 설정을 수행한다.

source /opt/ros/humble/setup.bash

이 설정 파일을 매번 입력하지 않도록 하려면 ~/.bashrc 파일에 위 명령을 추가해둔다. 이후 ROS 2 패키지 빌드를 위해서는 colcon이나 CMake가 필요할 수 있으므로 추가 패키지를 설치한다.

sudo apt install python3-colcon-common-extensions

ROS 2 Humble의 동작을 간단히 확인하려면 다음과 같이 기본 예제를 실행해 볼 수 있다.

ros2 run demo_nodes_cpp talker

별도의 터미널을 열어 아래 명령으로 수신 노드가 정상 동작하는지 확인한다.

ros2 run demo_nodes_py listener

화면에 메시지가 정상적으로 출력되면 ROS 2 환경이 성공적으로 구성된 것이다.

Isaac Sim과 ROS 2 패키지 연동 준비¶

Isaac Sim은 내부적으로 ROS 2 패키지들과 연동하기 위한 브리지 기능을 제공한다. Isaac Sim 내부 확장(Extension)을 통해 ROS 2의 토픽과 서비스를 송수신하고, TF 정보를 주고받을 수 있다. 이를 위해서는 Isaac Sim에서 제공하는 ROS Bridge를 활성화하거나, ROS 2용 추가 패키지를 설치해야 한다.

Omniverse Extension 메뉴에서 omni.isaac.ros2_bridge와 같은 ROS 2 확장을 활성화하면 Isaac Sim 환경에서 ROS 2 메시지를 직접 퍼블리시(publish)하거나 서브스크라이브(subscribe)할 수 있다. ROS 2 Humble 버전에 맞추어 메시지 타입이 호환되는지 확인이 필요하다.

개발 툴 및 참고 패키지 설치¶

ROS 2 노드를 작성하거나 Isaac Sim 확장을 수정하기 위해서는 Python이나 C++ 개발 환경이 요구된다. Python 3.10 이상 환경에서 동작을 원활히 유지하려면 pip, setuptools 등이 최신 버전이어야 하며, C++ 개발을 위해서는 GCC 9 이상을 사용하는 것이 바람직하다.

또한 Isaac Sim에서 제공하는 예제 스크립트나 샘플 코드를 수정하려면 Visual Studio Code 같은 에디터를 이용할 수 있다. VSCode 확장을 활용하여 Docker 컨테이너 내부에서 개발하거나, 원격 SSH로 접속한 후 작업할 수도 있다.

GPU 연산 및 CUDA 설정¶

Isaac Sim은 GPU 기반 시뮬레이션을 수행하므로 CUDA 버전 호환성을 확인해야 한다. CUDA 11 이상이 일반적으로 요구되며, 드라이버와의 버전 불일치로 인해 Isaac Sim 실행 시 오류가 발생할 수 있으므로 사전에 다음과 같이 CUDA 버전을 확인한다.

nvcc --version

출력 결과가 현재 사용 중인 드라이버 버전 및 CUDA 버전을 적절히 보여주면, Isaac Sim 및 ROS 2 Humble을 사용할 때 충돌이 일어날 가능성이 낮아진다.

Docker를 통한 환경 관리 (선택 사항)¶

도커(Docker)를 통해 Isaac Sim 및 ROS 2 Humble을 별도의 컨테이너에서 운용할 수도 있다. 이러한 방법은 환경 격리를 통해 시스템 설정을 간소화하고, 종속성 충돌을 줄이는 장점이 있다. NVIDIA GPU를 활용하려면 NVIDIA Container Toolkit 설정이 필요하다.

도커 컨테이너 내부에서 Isaac Sim과 ROS 2를 함께 구동하려면 X11 포워딩 혹은 VirtualGL 같은 추가 설정이 요구될 수 있다. 헤드리스(headless) 모드로 시뮬레이션을 돌린다면 그래픽 인터페이스 없이 Isaac Sim의 물리 엔진 기능만 활용할 수도 있다.

간단한 코드 예제¶

아래는 Python을 이용해 ROS 2 Humble 환경에서 간단한 퍼블리셔를 작성하는 예시다. Isaac Sim과 연동해볼 때 메시지 타이핑이 적절한지 확인하는 용도로 활용할 수 있다.

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class SimplePublisher(Node):
    def __init__(self):
        super().__init__('simple_publisher')
        self.publisher_ = self.create_publisher(String, 'isaac_sim_test', 10)
        self.timer = self.create_timer(1.0, self.publish_message)

    def publish_message(self):
        msg = String()
        msg.data = 'Hello Isaac Sim'
        self.publisher_.publish(msg)
        self.get_logger().info('Publishing: "%s"' % msg.data)

def main(args=None):
    rclpy.init(args=args)
    node = SimplePublisher()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

이 코드를 백그라운드에서 실행해 두고 Isaac Sim 내부의 ROS 2 브리지를 통해 isaac_sim_test 토픽을 서브스크라이브(subscribe)하면 메시지가 정상적으로 수신되는지 확인할 수 있다.

차후 설정 및 주의 사항¶

위 과정들을 마친 뒤에는 Isaac Sim 내부에서 동작하는 ROS 2 브리지 확장과, 실제 로봇 하드웨어 또는 다른 ROS 2 환경과의 통신 테스트를 진행해야 한다. IP나 네트워크 설정, ROS_DOMAIN_ID 충돌 여부 등에 주의하여야 하며, Isaac Sim이 요구하는 Python 라이브러리 버전도 상이할 수 있으므로 가상 환경(virtualenv, conda 등)을 사용하여 관리하는 편이 좋다.

특히 Isaac Sim에서 제공하는 ROS 메시지 타입과 ROS 2 Humble에서 사용하는 메시지 타입 버전이 동일해야 데이터를 정확하게 전송할 수 있다. 만일 메시지 필드가 다르거나 형식이 다른 경우에는 브리지에서 변환 중 오류가 발생할 수 있으니, Isaac Sim 릴리스 노트의 호환 메시지 목록을 반드시 확인해야 한다.

Isaac Sim과 ROS 2 Humble을 함께 사용하기 위해서는 충분한 GPU 리소스, 올바른 드라이버 설정, 그리고 ROS 2 환경 구성이 필수적이다. 운영체제 선택부터 드라이버 버전 호환성, 그리고 Docker 활용 여부까지 사전에 꼼꼼하게 준비해야 시뮬레이션과 실제 로봇 연동에서 문제를 최소화할 수 있다. 또한 Isaac Sim이 내부적으로 제공하는 ROS 2 확장 기능을 제대로 활용하려면 토픽과 서비스, TF 구조를 명확히 이해하고 메시지 타입 호환성을 주기적으로 점검하는 절차가 중요하다. 이러한 준비 과정을 통해 Isaac Sim 환경에서 복잡하고 정교한 로보틱스 시뮬레이션을 편리하게 진행할 수 있다.
