운영 체제 및 하드웨어 요구 사항¶

Ubuntu 22.04 환경에서 Isaac Sim과 ROS 2 Humble을 함께 사용하려면 안정적인 GPU가 장착된 시스템과 최신 그래픽 드라이버가 필요하다. 가능한 한 NVIDIA GPU를 사용하는 것을 권장한다. CPU 코어 수와 RAM 용량이 충분할수록 복잡한 로봇 시뮬레이션이나 물리 엔진 연산 시에도 원활한 작업이 가능하다. Isaac Sim의 물리 시뮬레이션 엔진(PhysX)을 활용하려면 GPU 가속이 중요한 역할을 하므로, CUDA를 지원하는 NVIDIA GPU가 필수적이다.

NVIDIA 드라이버 및 CUDA 설치¶

NVIDIA 드라이버와 CUDA를 먼저 설치하고, 정확한 버전을 확인해야 한다. 드라이버 설치 후 CUDA가 제대로 동작하는지 테스트하는 과정을 거치는 것이 좋다. NVIDIA 공식 문서 또는 패키지 관리자를 통해 설치할 수 있다. 만일 설치된 CUDA 버전이 Isaac Sim의 요구 사항과 호환되지 않으면 에러가 발생할 수 있으므로, 지원 버전을 미리 확인해야 한다.

sudo apt update
sudo apt install nvidia-driver-525

위와 같이 적절한 버전을 선택해 설치한 뒤, CUDA를 설치하고 다음과 같이 버전을 확인한다.

nvcc --version

ROS 2 Humble 설치¶

ROS 2 Humble은 Ubuntu 22.04와 함께 사용하는 것을 기본으로 권장한다. ROS 2 설치를 위해서는 다음처럼 패키지 소스를 추가하고 설치를 진행한다.

sudo apt update && sudo apt install curl gnupg lsb-release
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | sudo apt-key add -
echo "deb [arch=amd64] http://packages.ros.org/ros2/ubuntu $(lsb_release -cs) main" | sudo tee /etc/apt/sources.list.d/ros2-latest.list
sudo apt update
sudo apt install ros-humble-desktop

설치가 완료된 후 ROS 2를 활성화하려면 환경 설정 스크립트를 추가로 로드해야 한다.

source /opt/ros/humble/setup.bash

Isaac Sim 설치¶

Isaac Sim은 NVIDIA의 Omniverse 플랫폼 기반 시뮬레이터로, 독립 실행형 버전과 Docker 버전을 제공한다. 독립 실행형 설치 시, Omniverse 런처나 공식 리포지토리를 통해 다운로드할 수 있다. Docker 버전을 사용하는 경우에는 호스트 머신의 NVIDIA 컨테이너 런타임 설정을 미리 확인해야 한다.

Isaac Sim 내부에서 ROS 2와 상호작용하기 위해서는 ROS 2 브리지 플러그인(ROS2 Bridge Extension)을 활성화해야 한다. 이를 통해 시뮬레이션에서 생성된 센서 데이터나 로봇 상태 정보를 ROS 2 토픽/액션/서비스 형태로 송수신할 수 있다.

Python 환경 설정¶

Isaac Sim과 ROS 2 모두 Python 3 기반 환경을 사용하는 경우가 많다. Isaac Sim 내부에는 자체 파이썬 인터프리터가 포함될 수 있으므로, 외부 Python 환경에서 ROS 2 패키지를 임포트하여 Isaac Sim과 통신할 때 버전 호환성을 주의해야 한다. Python 가상 환경(venv) 또는 Conda 환경을 활용하여 프로젝트별로 의존성을 분리하는 것이 일반적이다.

python3 -m venv isaacsim_ros2_env
source isaacsim_ros2_env/bin/activate
pip install --upgrade pip
pip install setuptools wheel
pip install rosbag2 pycuda ...

고급 설정: 물리 시뮬레이션과 ROS 2 통합¶

Isaac Sim은 물리 시뮬레이션 엔진(PhysX)을 기반으로 로봇이나 물체의 움직임을 사실적으로 표현한다. ROS 2와 통합하여 로봇의 조인트 상태, 센서 데이터(카메라, LiDAR 등), 제어 명령 등을 주고받으려면 올바른 좌표계와 변환 행렬을 이해해야 한다.

시뮬레이션에서 한 좌표계에서 다른 좌표계로 벡터를 변환하기 위해서는 일반적으로 다음과 같은 4×4 동차 변환 행렬을 사용한다.

T=[Rp01]\begin{align} \mathbf T &= \begin{bmatrix} \mathbf R & \mathbf p \ \mathbf 0 & 1 \end{bmatrix} \end{align}

여기서 \(\mathbf R\)은 회전 행렬이고, \(\mathbf p\)는 병진 벡터이며, \(\mathbf 0\)은 1×3 영벡터를 나타낸다. \(\mathbf R \in \mathbb R^{3 \times 3},, \mathbf p \in \mathbb R^{3},, \mathbf T \in \mathbb R^{4 \times 4}\) 라고 할 때, 로봇 팔이나 카메라 등 서로 다른 센서나 액추에이터 간 위치와 자세 정보를 일관성 있게 유지하기 위해 이 변환을 반드시 적용해야 한다.

시뮬레이션에서 센서 데이터를 얻어 ROS 2 노드로 전송하거나, ROS 2 노드에서 제어 명령을 내리는 과정을 한눈에 보면 다음과 같다.

Isaac Sim 내부에서 센서를 구성하고 ROS 2 Humble 노드를 통해 메시지를 송수신함으로써, 가상 환경에서 실제 로봇 시스템과 유사한 테스트를 수행할 수 있다.

예시: Python 코드를 통한 토픽 확인¶

ROS 2 환경에서 Isaac Sim이 발행하는 토픽을 Python 코드로 확인하려면 아래와 같이 rclpy 패키지를 사용한다.

import rclpy
from rclpy.node import Node
from sensor_msgs.msg import LaserScan

class LaserSubscriber(Node):
    def __init__(self):
        super().__init__('laser_subscriber')
        self.subscription = self.create_subscription(
            LaserScan,
            '/scan',  # Isaac Sim에서 설정한 LiDAR 토픽 이름
            self.listener_callback,
            10
        )
        self.subscription

    def listener_callback(self, msg):
        self.get_logger().info(f'Received LaserScan angle_min: {msg.angle_min}')

def main(args=None):
    rclpy.init(args=args)
    node = LaserSubscriber()
    try:
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

이렇게 Isaac Sim에서 발행된 스캔 데이터를 ROS 2 노드가 구독하여 처리할 수 있다. 실험 중에는 Isaac Sim에서 LiDAR 센서를 활성화하고, 해당 토픽이 올바르게 연결되었는지 ROS 2 Humble에서 rclpy나 ros2 topic list 등으로 확인해야 한다.

Isaac Sim과 ROS 2 Humble을 연동하기 위해 필요한 사전 준비는 운영 체제와 드라이버 호환성에서부터 시뮬레이션 소프트웨어와 물리 엔진 이해에 이르기까지 다양하다. 충분히 환경을 구성해 두면 시뮬레이션에서의 실험 결과를 실제 로봇에 쉽게 이식할 수 있고, 시스템 통합 및 디버깅 시간이 단축될 것이다.
