UI와 주요 구성 요소

Isaac Sim(아이작 시뮬레이터)의 UI와 주요 구성 요소는 시뮬레이션 환경을 설정하고 관리하는 데 필요한 다양한 도구와 인터페이스를 제공합니다. 이 섹션에서는 각각의 UI 구성 요소와 주요 역할을 상세히 설명합니다.

Viewport

Viewport는 Isaac Sim에서 가장 중심적인 UI 요소 중 하나로, 시뮬레이션 세계를 실시간으로 시각화하는 창입니다. 이를 통해 사용자는 3D 환경을 확인하고 조작할 수 있습니다.

  • 카메라 컨트롤: Viewport 내에서 카메라 이동, 회전, 줌 기능을 제공합니다. 카메라 위치와 방향을 변경하여 시뮬레이션 환경의 세부 사항을 확인할 수 있습니다.

  • 렌더링 모드

: 뷰포트는 여러 렌더링 모드를 제공합니다. 예를 들어:

  • Wireframe 모드: 객체의 와이어프레임 구조만 보여줌.
  • Shaded 모드: 표면 렌더링과 그림자 표현.
  • Physics Debug 모드: 물리 상호작용을 시각화.

Stage와 Layer

Isaac Sim은 Omniverse의 Stage 및 Layer 시스템을 기반으로 작동합니다.

  • Stage: 시뮬레이션 세계의 최상위 구조로, 모든 객체와 환경이 이 Stage에 포함됩니다. Stage는 USD(Universal Scene Description) 파일로 저장될 수 있으며, 이를 통해 복잡한 장면을 관리할 수 있습니다.
  • Layer: Stage는 여러 Layer로 구성될 수 있으며, 각 Layer는 독립적으로 편집이 가능합니다. 이를 통해 협업 시 충돌을 줄이고 작업을 세분화할 수 있습니다.

Property Panel

Property Panel은 선택된 객체의 속성을 편집할 수 있는 인터페이스입니다.

  • 트랜스폼(Transform)

: 위치, 회전, 크기를 설정합니다.

  • 위치 \mathbf{p}는 \mathbf{p} = [x, y, z]^\top로 표현됩니다.

  • 회전은 쿼터니언 또는 오일러 각도로 설정할 수 있습니다.

  • 재질(Material): 표면의 색상, 반사율, 텍스처를 설정합니다.

  • 물리 속성(Physics Properties): 객체의 질량, 마찰 계수, 강체 여부 등을 정의합니다.

Asset Browser

Asset Browser는 미리 정의된 3D 모델, 텍스처, 재질 등의 자산을 탐색하고 프로젝트에 추가할 수 있는 도구입니다.

  • 검색 기능: 키워드를 입력하여 필요한 자산을 빠르게 검색할 수 있습니다.
  • 드래그 앤 드롭: 자산을 Stage로 직접 끌어와 배치할 수 있습니다.
  • 카테고리 분류: 자산은 범주별로 정리되어 있어, 로봇, 환경, 센서 등의 카테고리에서 선택 가능합니다.

Timeline

Timeline은 시뮬레이션 실행의 시간적 흐름을 제어하는 구성 요소입니다.

  • Play/Pause/Stop: 시뮬레이션을 시작, 일시 정지, 종료할 수 있습니다.
  • 키프레임(Keyframe): 객체의 애니메이션을 설정할 수 있습니다. 예를 들어, 시간 t_1에서의 위치 \mathbf{p}_1와 시간 t_2에서의 위치 \mathbf{p}_2를 지정하면 중간 프레임이 보간됩니다.
  • 루프 제어: 특정 시간 범위를 설정하여 시뮬레이션을 반복 실행할 수 있습니다.

Physics Panel

Physics Panel은 물리 엔진과 상호작용을 설정할 수 있는 UI입니다.

  • 중력(Gravity): 시뮬레이션의 중력 가속도 \mathbf{g}를 설정합니다. 기본값은 \mathbf{g} = [0, -9.8, 0]^\top입니다.
  • 충돌 설정: 객체 간의 충돌을 활성화하거나 비활성화할 수 있습니다.
  • 시뮬레이션 속도: 시뮬레이션 시간의 스케일을 설정하여 실시간보다 빠르거나 느리게 실행할 수 있습니다.

Script Editor

Script Editor는 Python 코드를 작성하고 실행할 수 있는 인터페이스로, 시뮬레이션의 자동화와 커스터마이징을 가능하게 합니다.

  • 스크립트 작성: 사용자는 Python 언어를 사용하여 사용자 정의 기능을 구현할 수 있습니다.
  • API 호출: Isaac Sim의 API를 호출하여 객체 생성, 속성 변경, 시뮬레이션 조작이 가능합니다.
  • 코드 실행: 작성된 스크립트를 실시간으로 실행하여 결과를 확인할 수 있습니다.

Diagnostics Panel

Diagnostics Panel은 시뮬레이션의 성능과 상태를 모니터링하는 데 사용됩니다.

  • 프레임률(Frame Rate): 초당 프레임 수(FPS)를 표시합니다.
  • 메모리 사용량: GPU와 CPU의 메모리 사용량을 실시간으로 확인할 수 있습니다.
  • 에러 로그: 실행 중 발생하는 오류나 경고 메시지를 기록합니다.

Lighting Editor

Lighting Editor는 시뮬레이션 환경의 조명 조건을 제어하고 편집하는 도구입니다.

  • 환경 조명(Environment Lighting): 전체 장면에 영향을 미치는 배경 조명을 설정합니다. 환경 조명은 HDRI 이미지로 정의될 수 있습니다.

  • 점광(Point Light)

: 특정 위치에서 모든 방향으로 빛을 방사하는 광원을 생성합니다.

  • 광원의 위치 \mathbf{p}*\text{light}는 \mathbf{p}*\text{light} = [x, y, z]^\top로 정의됩니다.

  • 스포트라이트(Spotlight): 특정 방향으로 집중된 빛을 생성하는 광원으로, 각도와 거리 등을 설정할 수 있습니다.

  • 태양광(Sunlight): 태양의 방향과 세기를 시뮬레이션하여 자연광 효과를 제공합니다.

Camera Settings Panel

Camera Settings Panel은 뷰포트 내 카메라의 속성을 관리하는 구성 요소입니다.

  • 초점 거리(Focal Length): 카메라의 렌즈 초점 거리를 설정하여 확대/축소 효과를 구현합니다.
  • 시야각(Field of View): 카메라의 시야각을 조정하여 장면에 포함되는 영역을 제어합니다.
  • 깊이 효과(Depth of Field): 특정 초점 거리에서 피사계 심도를 설정하여 실제 카메라와 유사한 효과를 구현합니다.

Action Graph

Action Graph는 노드 기반의 시각적 프로그래밍 인터페이스로, 시뮬레이션의 복잡한 동작을 구성할 수 있습니다.

  • 노드(Node): 각각의 노드는 특정 동작이나 데이터를 처리하는 단위입니다. 예를 들어, 물리 객체의 이동이나 카메라의 회전 동작이 있습니다.
  • 엣지(Edge): 노드 간의 연결을 나타내며, 데이터 흐름이나 이벤트 전달을 표현합니다.
  • 트리거(Trigger): 특정 조건에서 이벤트를 활성화하여 사용자 정의 동작을 실행합니다.

Mermaid 다이어그램으로 Action Graph의 개념을 시각화할 수 있습니다:

graph TD A[Start] --> B[Node 1: 데이터 처리] B --> C[Node 2: 이벤트 발생] C --> D[Node 3: 동작 실행]

Content Browser

Content Browser는 프로젝트 내의 모든 파일과 폴더를 탐색할 수 있는 인터페이스입니다.

  • 파일 관리: 프로젝트에 포함된 USD 파일, 텍스처, 스크립트를 관리합니다.
  • 검색 필터: 확장자별 또는 이름별로 파일을 필터링하여 원하는 파일을 빠르게 찾을 수 있습니다.
  • 외부 파일 가져오기: 외부에서 생성된 3D 모델이나 자산을 프로젝트로 가져올 수 있습니다.

Metrics Panel

Metrics Panel은 시뮬레이션 성능을 심층적으로 분석하는 도구입니다.

  • 렌더링 시간(Render Time): 각 프레임이 렌더링되는 데 소요된 시간을 표시합니다.
  • 물리 연산 시간(Physics Time): 물리 엔진이 계산을 수행하는 데 걸리는 시간을 확인할 수 있습니다.
  • 병목 현상 분석: 성능 병목 현상이 발생하는 위치를 파악하고 최적화 기회를 제공합니다.

OmniGraph

OmniGraph는 Omniverse의 데이터 처리 흐름을 시각화하고 제어하는 고급 인터페이스입니다.

  • 컴퓨트 그래프(Compute Graph): 각 노드는 데이터를 처리하는 작업 단위이며, 노드 간의 연결은 데이터의 흐름을 나타냅니다.
  • 사용자 정의 노드(Custom Node): Python 또는 C++로 새로운 노드를 정의하여 사용자 요구 사항에 맞는 기능을 추가할 수 있습니다.
  • 디버깅 기능: 실행 중 데이터 흐름을 실시간으로 모니터링하여 문제를 식별하고 수정할 수 있습니다.

Extension Manager

Extension Manager는 Isaac Sim 및 Omniverse 플랫폼에서 동작하는 플러그인과 확장을 관리하는 도구입니다.

  • 확장 활성화/비활성화: 사용자가 필요로 하는 기능만 활성화하여 시스템 리소스를 최적화할 수 있습니다.
  • 확장 설치: Omniverse에서 제공하는 기본 확장 외에도 사용자 정의 확장을 추가로 설치할 수 있습니다.
  • 버전 관리: 확장의 버전을 관리하여 호환성을 유지하거나 최신 기능을 사용할 수 있습니다.

Simulation Panel

Simulation Panel은 시뮬레이션의 제어와 상태를 관리하는 중요한 UI 구성 요소입니다.

  • 실행 상태: 현재 시뮬레이션이 실행 중인지, 정지 중인지, 일시 중지 상태인지 나타냅니다.
  • 리셋(Reset): 시뮬레이션 환경을 초기 상태로 되돌릴 수 있습니다.
  • 시뮬레이션 속도 제어: 시뮬레이션 시간을 조절하여 빠르게 진행하거나 세부 사항을 확인하기 위해 느리게 실행할 수 있습니다.

Material Editor

Material Editor는 3D 객체의 표면 특성을 설정하는 도구로, 객체의 시각적 품질을 조정하는 데 사용됩니다.

  • 기본 재질 설정: 기본 색상, 투명도, 반사율 등을 설정합니다.
  • 텍스처 추가: 외부 이미지 파일을 텍스처로 추가하여 객체의 표면 세부 사항을 설정합니다.
  • PBR(Physically Based Rendering): 물리 기반 렌더링을 지원하여 보다 사실적인 재질 표현이 가능합니다.

Layer Manager

Layer Manager는 여러 레이어를 활용하여 복잡한 장면을 구성하고 관리할 수 있도록 도와줍니다.

  • 레이어 잠금: 특정 레이어를 잠금으로 설정하여 의도치 않은 수정이 발생하지 않도록 합니다.
  • 레이어 가시성: 레이어별로 가시성을 설정하여 필요한 요소만 표시할 수 있습니다.
  • 레이어 병합: 여러 레이어를 하나로 병합하여 작업을 단순화할 수 있습니다.

Debugging Tools

Isaac Sim은 디버깅 및 문제 해결을 위한 다양한 도구를 제공합니다.

  • 로그 보기(Log Viewer): 실행 중 발생한 모든 로그 메시지를 실시간으로 확인할 수 있습니다.
  • 물리 디버깅: 물리 상호작용과 충돌을 시각화하여 정확도를 확인합니다.
  • API 호출 추적: 특정 Python 또는 C++ API가 호출된 기록을 추적하여 문제를 분석합니다.

Preferences

Preferences는 사용자 인터페이스와 작업 환경을 개인 맞춤 설정할 수 있는 옵션을 제공합니다.

  • 단축키 설정: 자주 사용하는 기능에 대한 단축키를 정의하거나 수정할 수 있습니다.
  • 테마 변경: 밝은 테마와 어두운 테마 중 선택하여 작업 환경을 개인화할 수 있습니다.
  • 기본 경로 설정: 프로젝트와 자산 파일의 기본 저장 경로를 지정합니다.

USD Composer

USD Composer는 Universal Scene Description(USD) 파일을 작성, 편집 및 시각화하는 고급 도구입니다.

  • 객체 계층 구조 편집: USD 파일 내 객체들의 계층 구조를 수정할 수 있습니다.
  • 속성(Property) 관리: 각 객체의 속성을 추가, 수정 또는 제거합니다.
  • 레퍼런스 추가: 외부 USD 파일을 현재 Stage에 참조하여 장면을 확장합니다.

Task Graph Editor

Task Graph Editor는 시뮬레이션에서 특정 작업을 정의하고 실행 순서를 관리하는 도구입니다.

  • 의존성 설정: 작업 간의 의존 관계를 정의하여 순차적 또는 병렬 실행이 가능하도록 설정합니다.
  • 작업 실행 상태: 각 작업의 실행 상태를 실시간으로 표시합니다.
  • 커스터마이징: 사용자 정의 작업을 추가하여 시뮬레이션의 요구 사항에 맞게 기능을 확장합니다.
