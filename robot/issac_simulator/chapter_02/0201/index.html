시뮬레이터 초기 설정

Isaac Sim 설치 요구 사항 확인

아이작 시뮬레이터를 실행하기 전에 하드웨어 및 소프트웨어 요구 사항을 충족하는지 확인해야 합니다. 다음은 주요 요구 사항입니다.

  • 운영 체제: Ubuntu 20.04 이상 또는 Windows 10/11
  • GPU: NVIDIA GPU (최소 NVIDIA RTX 3060, 권장 RTX 3090 이상)
  • CUDA 및 드라이버: CUDA Toolkit 11.4 이상 및 최신 NVIDIA GPU 드라이버
  • Python: Python 3.8~3.10
  • RAM: 최소 16GB, 권장 32GB 이상
  • 디스크 공간: 최소 30GB의 여유 공간

Isaac Sim 설치

  1. NVIDIA Omniverse Launcher 설치 NVIDIA의 Omniverse Launcher를 통해 Isaac Sim을 설치할 수 있습니다.
  2. NVIDIA 웹사이트에서 Launcher 설치 파일을 다운로드합니다.
  3. 설치 후 로그인하여 "Isaac Sim"을 검색합니다.
  4. 설치 버튼을 클릭하여 다운로드와 설치를 진행합니다.
  5. Standalone 설치 Omniverse Launcher 없이 독립 실행형으로 설치하려면 NVIDIA의 공식 GitHub 저장소에서 관련 바이너리를 다운로드하여 설치할 수 있습니다.

환경 변수 설정

설치 후 환경 변수를 설정해야 합니다. 다음은 Linux를 기준으로 한 환경 변수 설정 예제입니다.

export ISAAC_SIM_PATH=/path/to/isaac-sim
export PATH=$ISAAC_SIM_PATH:$PATH
export PYTHONPATH=$ISAAC_SIM_PATH:$PYTHONPATH

환경 변수는 ~/.bashrc 또는 ~/.zshrc 파일에 추가하여 매번 설정하지 않도록 자동화할 수 있습니다.

GPU 가속 설정

아이작 시뮬레이터는 GPU를 활용하여 고속 물리 시뮬레이션과 렌더링을 제공합니다. GPU 가속을 활성화하려면 다음 사항을 점검합니다.

  1. NVIDIA GPU 드라이버 확인 드라이버 버전이 요구 사항을 충족하는지 확인합니다. Linux에서 확인하려면 다음 명령어를 실행합니다.

bash nvidia-smi

결과 출력에서 CUDA 버전과 GPU 상태를 확인할 수 있습니다.

  1. CUDA Toolkit 설치 적절한 CUDA Toolkit 버전을 설치해야 합니다. 설치 후, CUDA 버전을 확인합니다.

bash nvcc --version

  1. 확인 후 테스트 실행 GPU 가속이 제대로 설정되었는지 테스트하려면 시뮬레이터를 실행하고 성능 모니터링을 통해 GPU 사용률을 확인합니다.

Python 개발 환경 설정

Isaac Sim은 Python API를 통해 제어할 수 있습니다. Python 개발 환경 설정은 다음과 같습니다.

  1. Python 가상 환경 생성 Python 프로젝트의 독립성을 유지하기 위해 가상 환경을 설정합니다.

bash python3 -m venv isaac_env source isaac_env/bin/activate

  1. 필수 Python 패키지 설치 requirements.txt 파일을 통해 필요한 패키지를 설치합니다. 기본 패키지는 다음과 같습니다.

bash pip install -r $ISAAC_SIM_PATH/requirements.txt

시뮬레이터 실행

설정이 완료되면 아이작 시뮬레이터를 실행할 수 있습니다. 실행 명령은 다음과 같습니다.

  1. GUI 모드 실행 GUI 모드를 사용하여 그래픽 인터페이스로 시뮬레이터를 실행합니다.

bash ./isaac-sim.sh

  1. 헤드리스 모드 실행 GUI를 사용하지 않고 서버 환경에서 실행하려면 헤드리스 모드를 활성화합니다.

bash ./isaac-sim.sh --headless

헤드리스 모드는 그래픽이 필요 없는 고속 연산 작업(예: 대규모 학습)에 유리합니다.

파일 디렉토리 구조 이해

아이작 시뮬레이터를 효율적으로 사용하기 위해 디렉토리 구조를 이해해야 합니다. 기본적으로 다음과 같은 디렉토리와 파일들이 포함되어 있습니다.

  1. isaac-sim/ 시뮬레이터의 루트 디렉토리로, 실행 파일과 기본 설정 파일이 위치합니다. 주요 하위 디렉토리는 다음과 같습니다.

  3. apps/: 사전 정의된 애플리케이션 샘플 코드
  4. python/: Python API 및 관련 코드
  5. usd/: Universal Scene Description (USD) 파일 및 리소스
  6. isaac-sim/python/ Python 개발에 필요한 모듈과 라이브러리가 포함되어 있습니다. 주요 디렉토리는 다음과 같습니다.
  7. omni/: Omniverse 관련 Python 모듈
  8. isaac/: Isaac Sim의 주요 API 및 로봇 시뮬레이션 관련 모듈
  9. examples/: 샘플 Python 스크립트
  10. isaac-sim/usd/ 시뮬레이션 환경을 정의하는 USD 파일이 포함되어 있습니다. USD 파일은 3D 씬, 객체, 그리고 물리적 속성을 정의하는 데 사용됩니다.

  1. config/ 디렉토리 시뮬레이터의 초기화 설정 파일들이 포함됩니다. 예를 들어:

  2. app.usda: 기본 애플리케이션 설정 파일

yaml simulation: physics: solver: PhysX time_step: 0.01 rendering: use_gpu: true

USD 씬 파일 로드

USD 파일은 아이작 시뮬레이터의 기본 씬 파일 포맷입니다. 시뮬레이터가 시작될 때 특정 USD 파일을 로드하여 환경을 초기화할 수 있습니다.

  1. USD 파일 경로 지정 실행 시 로드할 USD 파일을 지정하려면 다음 명령을 사용합니다.

bash ./isaac-sim.sh --/app/scene/path="path/to/your_scene.usd"

  1. Python API로 USD 로드 Python 스크립트를 사용하여 시뮬레이션 중에 USD 파일을 동적으로 로드할 수 있습니다.

```python from omni.isaac.core import SimulationApp

sim_app = SimulationApp({"headless": False}) sim_app.open_stage("path/to/your_scene.usd") sim_app.run() ```

물리 엔진 설정

아이작 시뮬레이터는 NVIDIA PhysX를 기본 물리 엔진으로 사용합니다. 물리 엔진 설정은 시뮬레이션의 정확성과 성능에 중요한 영향을 미칩니다.

  1. 타임스텝 설정 물리 시뮬레이션의 시간 간격을 설정합니다. 예를 들어, \(dt = 0.01\)로 설정하면 시뮬레이션이 100Hz로 실행됩니다.

tn+1=tn+dtt_{n+1} = t_n + dt

  1. 중력 설정 시뮬레이션에서 중력은 다음과 같은 기본 벡터로 설정됩니다.

g=[0,0,−9.81]\mathbf{g} = [0, 0, -9.81]

Python 코드로 중력을 변경하려면 다음과 같이 설정할 수 있습니다.

```python from omni.isaac.core.utils.stage import set_gravity

set_gravity([0, 0, -9.8]) ```

플러그인 로드

아이작 시뮬레이터는 다양한 플러그인을 지원하여 확장 가능한 기능을 제공합니다. 플러그인을 로드하는 방법은 다음과 같습니다.

  1. 기본 플러그인 로드 플러그인은 실행 시 자동으로 로드됩니다. 예를 들어, PhysX 플러그인은 config 파일을 통해 활성화됩니다.

  2. 사용자 정의 플러그인 로드 사용자 정의 플러그인을 로드하려면 Python API를 사용합니다.

```python from omni.kit.commands import execute

execute("LoadExtension", extension_id="omni.physx") ```

시뮬레이션 파라미터 조정

시뮬레이션의 성능과 정확도를 최적화하기 위해 다양한 파라미터를 조정할 수 있습니다. 주요 파라미터와 조정 방법은 다음과 같습니다.

시뮬레이션 시간 관리

  1. 시간 스텝(Time Step) 물리 엔진에서 사용할 시간 간격 \(dt\)를 설정합니다. 시간 스텝이 작을수록 시뮬레이션의 정확도가 증가하지만, 계산 비용이 증가합니다. 적절한 시간 스텝은 다음과 같은 관계를 만족해야 합니다.

dt≤1fmaxdt \leq \frac{1}{f_{\text{max}}}

여기서 \(f_{\text{max}}\)는 시뮬레이션에서 발생할 수 있는 최대 힘의 주파수입니다. 일반적으로 \(dt = 0.01\) (100Hz)가 사용됩니다.

Python으로 시간 스텝을 설정하는 예제는 다음과 같습니다.

```python from omni.isaac.core.simulation_context import SimulationContext

sim_context = SimulationContext() sim_context.set_simulation_dt(0.01) # 100Hz ```

  1. 실시간 속도 비율(Real-Time Factor, RTF) 시뮬레이션 속도를 제어하여 실시간과 비교한 비율을 설정합니다. RTF가 1.0이면 시뮬레이션이 실시간으로 실행됩니다.

python sim_context.set_target_real_time_factor(1.0)

물리 엔진 파라미터 설정

  1. Solver Iteration 물리 엔진의 정확도를 높이기 위해 Solver Iteration 값을 조정할 수 있습니다. 이 값이 높을수록 계산 비용이 증가하지만, 정확한 충돌 처리와 제약 조건을 보장합니다.

```python from omni.physx import set_physx_solver_iterations

set_physx_solver_iterations(position_iterations=10, velocity_iterations=4) ```

  1. 접촉 처리(Contact Handling) 물체 간 접촉을 처리하는 파라미터를 조정할 수 있습니다. 예를 들어, 접촉 감지의 정확도를 높이기 위해 접촉 임계값(Contact Offset)을 줄일 수 있습니다.

```python from omni.physx import set_contact_offset

set_contact_offset(0.02) # 2cm ```

렌더링 성능 최적화

  1. 렌더링 해상도 높은 해상도는 더 나은 시각적 품질을 제공하지만 GPU 리소스를 많이 소모합니다. 해상도를 조정하려면 다음과 같이 설정합니다.

```python from omni.kit.viewport.window import get_viewport_window

viewport_window = get_viewport_window() viewport_window.set_resolution(1280, 720) ```

  1. 안티앨리어싱 렌더링 품질을 높이기 위해 안티앨리어싱을 활성화할 수 있습니다. 그러나 성능 저하가 발생할 수 있으므로 GPU 리소스 사용량을 고려해야 합니다.

```python from omni.kit.renderer_settings import set_antialiasing

set_antialiasing(True) ```

시뮬레이션 속성 제어

  1. 중력 값 변경 중력을 사용자 정의 값으로 설정할 수 있습니다. 예를 들어, 달 환경에서의 중력 값은 다음과 같이 설정됩니다.

g=[0,0,−1.62]\mathbf{g} = [0, 0, -1.62]

python set_gravity([0, 0, -1.62])

  1. 세계 축 설정 시뮬레이션의 기본 축을 조정할 수 있습니다. 일반적으로 \(z\) 축이 위로 설정되지만, 필요에 따라 \(y\) 축을 위로 설정할 수도 있습니다.

```python from omni.isaac.core.utils.stage import set_stage_up_axis

set_stage_up_axis("Z") # 또는 "Y" ```

시뮬레이션 상태 저장 및 불러오기

상태 저장(Save State)

시뮬레이션 진행 상태를 저장하려면 USD 파일로 현재 상태를 저장할 수 있습니다. 이를 통해 동일한 환경을 나중에 다시 불러올 수 있습니다.

from omni.isaac.core.simulation_context import SimulationContext

sim_context = SimulationContext()
sim_context.save_stage("path/to/save_state.usd")

상태 불러오기(Load State)

저장된 USD 파일을 불러와 이전 상태에서 시뮬레이션을 재개합니다.

sim_context.open_stage("path/to/save_state.usd")
sim_context.run()

디버깅 및 로그 관리

시뮬레이션 중 발생하는 문제를 디버깅하거나 로그를 관리하기 위해 다음 기능을 활용합니다.

  1. 로깅 활성화 시뮬레이터에서 발생하는 이벤트를 로그 파일로 저장합니다.

```python import logging

logging.basicConfig(level=logging.INFO, filename="simulation.log", filemode="w") logging.info("Simulation started.") ```

  1. 성능 모니터링 시뮬레이션 성능을 실시간으로 확인하려면 성능 모니터를 활성화합니다.

bash ./isaac-sim.sh --enable-perf-monitor

시뮬레이션 상호작용 기능 활성화

아이작 시뮬레이터는 사용자 또는 프로그램이 시뮬레이션에 상호작용할 수 있는 다양한 기능을 제공합니다. 이를 통해 시뮬레이션 중 객체를 조작하거나 특정 이벤트를 트리거할 수 있습니다.

객체 추가 및 삭제

  1. 객체 추가 Python API를 사용하여 시뮬레이션 중 객체를 동적으로 추가할 수 있습니다. 예를 들어, 큐브 객체를 추가하는 코드는 다음과 같습니다.

```python from omni.isaac.core.prims import XFormPrim

cube = XFormPrim("/World/Cube", position=[0, 0, 1], scale=[1, 1, 1]) cube.create() ```

  1. 객체 삭제 특정 객체를 제거하려면 객체의 경로를 지정하고 삭제합니다.

```python from omni.isaac.core.prims import delete_prim

delete_prim("/World/Cube") ```

객체 속성 제어

  1. 위치 및 회전 조작 객체의 위치와 회전은 Python API를 통해 제어할 수 있습니다. 위치는 3D 벡터 \(\mathbf{p}\)로, 회전은 쿼터니언 \(\mathbf{q}\)로 표현됩니다.

p=[x,y,z],q=[w,x,y,z]\mathbf{p} = [x, y, z], \quad \mathbf{q} = [w, x, y, z]

예를 들어, 위치와 회전을 설정하려면 다음과 같이 작성합니다.

python cube.set_world_pose(position=[1, 1, 1], orientation=[1, 0, 0, 0])

  1. 속성 변경 객체의 속성을 동적으로 변경할 수 있습니다. 예를 들어, 물리적 질량을 설정하려면 다음과 같이 설정합니다.

```python from omni.isaac.core.utils.prims import set_prim_mass

set_prim_mass("/World/Cube", mass=5.0) ```

이벤트 트리거

  1. 콜리전 이벤트 두 객체가 충돌했을 때 특정 작업을 수행하는 이벤트를 설정할 수 있습니다. 충돌 이벤트는 다음과 같이 활성화됩니다.

```python from omni.isaac.core.events import CollisionEvent

def on_collision(event): print(f"Collision detected between {event.object1} and {event.object2}")

collision_event = CollisionEvent("/World/Cube", callback=on_collision) collision_event.activate() ```

  1. 시간 기반 이벤트 특정 시간에 이벤트를 트리거하려면 타이머를 설정합니다.

```python from omni.isaac.core.tasks import TimerTask

def time_event(): print("Event triggered at time:", sim_context.get_time())

timer = TimerTask(interval=1.0, callback=time_event) # 매 1초마다 호출 timer.activate() ```

시뮬레이션 데이터 로깅

데이터 기록

시뮬레이션 동안 객체의 상태(예: 위치, 속도)를 기록할 수 있습니다. 데이터는 CSV 파일로 저장하거나 Python 데이터 프레임으로 관리할 수 있습니다.

import csv

log_file = "simulation_data.csv"
with open(log_file, "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["Time", "X", "Y", "Z"])

    def log_data():
        time = sim_context.get_time()
        position = cube.get_world_pose()[0]
        writer.writerow([time, position[0], position[1], position[2]])

    sim_context.add_task(log_data)

데이터 시각화

기록된 데이터를 시각화하려면 Matplotlib을 사용할 수 있습니다. 예를 들어, 객체의 이동 경로를 그래프로 나타냅니다.

import matplotlib.pyplot as plt

time = []
x = []
y = []
z = []

with open(log_file, "r") as file:
    reader = csv.reader(file)
    next(reader)  # 헤더 스킵
    for row in reader:
        time.append(float(row[0]))
        x.append(float(row[1]))
        y.append(float(row[2]))
        z.append(float(row[3]))

plt.plot(time, x, label="X")
plt.plot(time, y, label="Y")
plt.plot(time, z, label="Z")
plt.legend()
plt.xlabel("Time (s)")
plt.ylabel("Position (m)")
plt.title("Object Trajectory")
plt.show()

from omni.isaac.core.simulation_context import SimulationContext

sim_context = SimulationContext()
sim_context.open_stage("path/to/your_scene.usd")

for step in range(1000):  # 1000번 시뮬레이션 스텝 실행

    if step % 100 == 0:
        print(f"Step {step}: Simulation time {sim_context.get_time()}")

네트워크 및 통신 설정

아이작 시뮬레이터는 네트워크 통신을 통해 외부 애플리케이션 또는 다른 시뮬레이션 인스턴스와 데이터를 주고받을 수 있습니다. 이를 활용하여 분산 시뮬레이션, 로봇 제어, 또는 센서 데이터 스트리밍을 구현할 수 있습니다.

ROS2 통합

아이작 시뮬레이터는 ROS2를 기본적으로 지원합니다. ROS2 통합을 통해 로봇 모델을 제어하거나 센서 데이터를 구독(publish/subscribe)할 수 있습니다.

  1. ROS2 확장 활성화 ROS2 통합을 사용하려면 Isaac Sim에서 제공하는 ROS2 확장을 활성화해야 합니다.

```python from omni.isaac.core.utils.extensions import enable_extension

enable_extension("omni.isaac.ros2_bridge") ```

  1. ROS2 노드 실행 시뮬레이터 내에서 ROS2 노드를 실행하고 데이터를 퍼블리시(publish)하거나 구독(subscribe)할 수 있습니다.

```python from omni.isaac.ros2_bridge import Ros2Bridge

ros2_bridge = Ros2Bridge() ros2_bridge.start() ```

  1. 토픽 설정 ROS2 토픽을 통해 시뮬레이션 데이터를 송수신합니다. 예를 들어, 카메라 센서 데이터를 퍼블리시하려면 다음과 같이 설정합니다.

```python from omni.isaac.sensor import Camera

camera = Camera("/World/Camera") camera.publish_to_ros2("/camera/image") ```

TCP/UDP 통신

TCP 또는 UDP를 사용하여 사용자 정의 네트워크 통신을 구현할 수 있습니다.

  1. TCP 서버 설정 TCP를 사용하여 외부 애플리케이션과 통신하는 예제입니다.

```python import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM) server.bind(("localhost", 8080)) server.listen(1) print("Waiting for connection...")

connection, address = server.accept() print("Connected to:", address)

while True: data = connection.recv(1024).decode() if not data: break print("Received:", data) connection.sendall("Acknowledged".encode())

connection.close() server.close() ```

  1. UDP 통신 설정 UDP를 사용하여 비연결 기반 통신을 구현할 수 있습니다.

```python import socket

udp_socket = socket.socket(socket.AF_INET, socket.SOCK_DGRAM) udp_socket.bind(("localhost", 9090))

while True: data, addr = udp_socket.recvfrom(1024) print("Received from", addr, ":", data.decode()) udp_socket.sendto("Acknowledged".encode(), addr) ```

웹소켓 통신

웹 기반 애플리케이션과 통신하려면 WebSocket 프로토콜을 사용할 수 있습니다.

  1. 서버 설정 WebSocket 서버를 Python에서 설정합니다.

```python import asyncio import websockets

async def echo(websocket, path): async for message in websocket: print("Received:", message) await websocket.send("Acknowledged")

start_server = websockets.serve(echo, "localhost", 8765) asyncio.get_event_loop().run_until_complete(start_server) asyncio.get_event_loop().run_forever() ```

  1. 클라이언트 연결 WebSocket 클라이언트를 사용하여 서버와 통신합니다.

```python import asyncio import websockets

async def communicate(): async with websockets.connect("ws://localhost:8765") as websocket: await websocket.send("Hello Server!") response = await websocket.recv() print("Received:", response)

asyncio.get_event_loop().run_until_complete(communicate()) ```
