대규모 행렬 계산의 필요성과 특징¶

제어공학에서는 고차원의 동적 시스템을 다루는 경우가 많다. 상태방정식을 선형화하면 대체로 매우 큰 차원을 가지는 행렬을 구성하게 되고, 이를 효율적이고 정확하게 계산하는 것은 시스템 분석이나 제어기 설계 과정에서 핵심이 된다. 예컨대 수천에서 수백만 차원에 이르는 행렬은 메모리와 연산량의 한계를 초래하기 때문에, 기존의 소규모 행렬 계산과는 다른 전략과 기술이 필요해진다.

대규모 행렬을 다룰 때는 우선 메모리 접근 패턴과 계산 복잡도 문제를 명확히 인식해야 한다. 흔히 사용하는 \(LU\) 분해나 \(QR\) 분해는 계산 복잡도가 높은 편이며, 저장해야 할 행렬 요소의 양도 방대해진다. 따라서 블록 연산을 이용하거나 희소성(sparsity)을 고려하는 기술 등을 통해 효율을 높이기도 한다. 또한 중앙처리장치(CPU)의 다중 코어 환경이나 그래픽처리장치(GPU) 같은 병렬화가 가능한 컴퓨팅 자원을 활용하여 처리 속도를 높이는 방법도 널리 연구되고 있다.

계산 복잡도만큼 중요한 것이 수치적 안정성이다. 대규모 행렬일수록 수많은 부동소수점 연산이 반복되므로, 반올림 오차(round-off error)가 누적되어 결과의 정확도를 떨어뜨리는 현상이 심각하게 나타난다. 실제 계산 시 결과가 지나치게 왜곡되지 않도록 다양한 알горитmic 기법들이 발전해왔고, 각 방법들의 안정성을 정량화하기 위해 고안된 여러 이론적 틀이 존재한다.

직접법과 반복법¶

대규모 선형시스템 \(\mathbf{A}\mathbf{x} = \mathbf{b}\)를 풀 때, 직접법(direct method)과 반복법(iterative method)은 서로 다른 접근 방식을 제공한다. 직접법은 \(\mathbf{A}\)를 \(LU\) 분해나 \(QR\) 분해 등으로 정확히(또는 부분 피벗을 동반하여) 분해한 다음 이를 활용하여 해 \(\mathbf{x}\)를 구한다. 단순한 원리에 비해 매우 큰 차원의 문제에서는 연산량이 폭증하고 수치적 오류가 누적되기 쉬워진다는 단점이 있다. 또한 메모리 사용량이 기하급수적으로 증가하기 때문에 하드웨어적 한계에 부딪히기 쉽다.

반면 반복법은 해를 점진적으로 개선해 나가며, 필요한 정밀도에 이를 때까지 알고리즘을 반복 실행한다. 공액 기울기법(Conjugate Gradient), GMRES(Generalized Minimal Residual), BiCGSTAB 등으로 알려진 다양한 알고리즘이 있다. 이 방법들은 큰 차원 행렬에서도 상대적으로 적은 메모리를 사용하며, 희소 행렬 구조를 적극 활용할 수 있다. 그러나 수렴 속도와 안정성은 문제의 조건수(condition number)나 분해 과정에서의 사전조건화(preconditioning)에 크게 좌우된다.

분해 알고리즘과 블록 연산¶

대규모 행렬 계산에서 중요한 기법 중 하나는 블록(block) 단위로 행렬 연산을 수행하는 것이다. 예를 들어 \(LU\) 분해를 수행할 때 전체 행렬을 작은 블록 행렬로 나눈 뒤, 각각에 대해 연산을 수행하고 이를 재조합하면 캐시 친화적인 메모리 접근이 가능해지고, 병렬 처리에도 유리해진다. 하드웨어의 메모리 계층 구조와 프로세서의 병렬화를 고려하여 고안된 BLAS(Basic Linear Algebra Subprograms), LAPACK(Linear Algebra Package) 같은 라이브러리에서 이러한 블록 알고리즘이 구현되어 있다.

\(QR\) 분해 역시 하우스홀더(Householder) 변환 또는 기븐스(Givens) 회전을 사용할 때, 여러 벡터 연산을 한 번에 처리하기 위해 블록화 기법이 활용된다. 이는 전체적인 연산 횟수를 획기적으로 줄여주지는 않지만, 실제 계산 시 발생하는 캐시 미스(cache miss)를 대폭 줄이고 메모리 대역폭을 보다 효율적으로 사용하게 해준다.

분산 메모리 환경과 병렬화¶

초대형 문제를 풀기 위해서는 단일 CPU와 단일 메모리만으로는 부족한 경우가 많다. 따라서 여러 노드에 행렬을 나누어 저장하는 분산 메모리 환경, 또는 수십~수백 개 코어의 프로세서를 장착한 공유 메모리 환경에서의 병렬화가 요구된다. 분산 메모리 환경에서는 각 노드 사이의 통신 비용이 연산 자체의 비용보다 훨씬 커지는 상황이 벌어지므로, 통신 패턴을 최소화하는 방법이 핵심이다. 예를 들어 스캘라팩(ScaLAPACK)은 메시 패싱 인터페이스(MPI)를 기반으로 대규모 행렬 연산을 수행하도록 설계된 라이브러리이며, 이를 통해 블록 사이즈를 적절히 조절함으로써 노드 간 데이터 전송량을 줄이고 연산 속도를 높인다.

GPU 가속도 대규모 행렬 계산에서 빼놓을 수 없는 주제이다. 고성능의 GPU 상에서 병렬 연산 라이브러리를 사용하는 경우, 수많은 스레드를 통해 행렬의 원소 단위 연산을 동시에 수행할 수 있다. 이때도 여전히 수치적 안정성을 유지하기 위한 기법들이 필요하며, GPU에서 지원하는 부동소수점 정밀도가 CPU와 같지 않을 수 있다는 점도 주의해야 한다.

수치적 안정성과 반올림 오차¶

행렬 연산의 결과가 실제 해와 얼마나 차이를 보이는지 조사하기 위해서는 forward error와 backward error 개념이 활용된다. forward error는 구해진 해 \(\mathbf{x}_\text{computed}\)가 참해 \(\mathbf{x}\)와 얼마나 다른지 직접 비교하는 방식이다. 반면 backward error는 구해진 해가 약간의 오차가 더해진 계수 행렬 \(\mathbf{A} + \Delta \mathbf{A}\)와 우변 벡터 \(\mathbf{b} + \Delta \mathbf{b}\)에 대하여 정확한 해임을 의미한다. 수치적 안정성이 높은 알고리즘은 backward error가 매우 작게 유지되도록 설계된다.

행렬의 조건수(condition number)는 문제의 민감도를 정량화한 지표로서, \(\mathbf{A}\)가 가역 행렬일 때 \(cond(\mathbf{A}) = |\mathbf{A}||\mathbf{A}^{-1}|\)로 정의된다. 조건수가 큰 문제는 작은 반올림 오차에도 결과가 크게 요동칠 수 있는 반면, 조건수가 작은 문제는 오차에 대한 상대적 민감도가 덜하다. 제어공학에서는 모델링 과정에서의 근사나 측정 잡음 등에 의해 이미 교란이 존재하기 때문에, 수치 해석 측면의 오차까지 과도하게 커지지 않도록 문제의 조건수를 낮추거나 적절한 축척(scaling)을 취해주는 방안을 모색한다.

피벗팅과 희소 행렬 처리¶

\(LU\) 분해 과정에서 가장 널리 알려진 것은 부분 피벗팅(partial pivoting)이다. 피벗팅은 분해 과정에서 대각 원소의 크기가 지나치게 작아지는 것을 방지하기 위해 행이나 열을 교환하여 반올림 오차를 줄이려는 기법이다. 피벗팅이 없으면 수치적 불안정성이 크게 증가하여 해의 오차가 폭증할 수 있다.

희소(sparse) 행렬 구조를 가진 문제에서는 피벗팅으로 인해 나타나는 fill-in 현상이 중요해진다. 원래 희소했던 행렬이라도 피벗팅 과정에서 여러 위치의 원소가 0이 아닌 값으로 채워지면, 메모리 사용량이 증가하고 연산 시간이 오래 걸린다. 이러한 문제를 완화하기 위해선 최소 fill-in 전략으로 행렬의 열과 행 순서를 재배치하는 등의 사전처리 기법이 활용된다.

반복적 해법의 수치 정밀도와 안정성¶

크기가 매우 큰 문제에서는 직접법 자체가 부담스러우므로 반복법을 많이 쓴다. 공액 기울기법이나 GMRES와 같은 방법은 희소 행렬 구조를 유지하며 반복적으로 해를 갱신한다. 그러나 반복 횟수가 늘어날수록 반올림 오차가 누적되어 수렴에 악영향을 미칠 수 있다. 따라서 반복 과정에서 주기적으로 잔차(residual)를 재계산하여 정확도를 확인하고, 필요하면 조건수를 개선할 수 있는 사전조건자를 도입한다.

사전조건자(preconditioner)는 \(\mathbf{M}\)이라는 행렬을 새롭게 정의하여 \(\mathbf{M}^{-1}\mathbf{A}\mathbf{x} = \mathbf{M}^{-1}\mathbf{b}\) 형태로 만든 뒤, 새로운 시스템이 더 작은 조건수를 갖도록 유도한다. 사전조건화의 대표적인 예는 incomplete \(LU\) 분해, incomplete Cholesky 분해 등을 꼽을 수 있다. 사전조건자를 구성하는 데 드는 비용과 이를 적용했을 때 반복 횟수가 감소함으로써 절약되는 비용 간의 균형을 맞추는 것이 중요하다.

전진/후진 오차 해석과 재교정 기법¶

선형계 \(\mathbf{A}\mathbf{x} = \mathbf{b}\)를 풀 때 발생하는 오차를 분석하는 한 방법으로 전진 오차(forward error)와 후진 오차(backward error)가 있다. 전진 오차는 실제 해 \(\mathbf{x}\)와 수치 해 \(\mathbf{x}*\text{computed}\)의 차이 \(|\mathbf{x} - \mathbf{x}*\text{computed}|\)를 보는 것이고, 후진 오차는 \(\mathbf{A}\)와 \(\mathbf{b}\)에 약간의 섭동을 가했을 때 정확한 해가 되는가를 보는 방식이다.

가끔은 이미 계산된 해가 매우 부정확할 경우가 존재하는데, 이럴 때 재교정 기법(iterative refinement)을 사용한다. 재교정 기법은 한 번 직접법이나 반복법으로 해를 구한 뒤, 잔차 벡터 \(\mathbf{r} = \mathbf{b} - \mathbf{A}\mathbf{x}_\text{computed}\)를 계산하고, \(\Delta \mathbf{x} = \mathbf{A}^{-1}\mathbf{r}\)를 추정하여 해를 갱신하는 절차를 여러 번 거치는 것이다. 이 방법은 해를 점차 개선하며, 부동소수점 오차에 의해 발생한 편차를 줄이는 효과가 있다.

오차 해석에 유용한 노름¶

오차 해석과 관련된 중심적 개념 중 하나는 노름(norm)이다. \(\mathbf{x}\)가 벡터이고 \(\mathbf{A}\)가 행렬일 때, 다양한 노름이 정의되어 있고 각 노름이 가지는 성질에 따라 오차 해석 방법이 달라진다. 가장 일반적으로 벡터 노름으로는 \(2\)-노름이 많이 쓰이며 \(|\mathbf{x}|_2 = \sqrt{x_1^2 + x_2^2 + \ldots + x_n^2}\)로 정의된다. 행렬 노름도 벡터 노름과 호환되는 방식으로 \(2\)-노름이 정의되며, 특잇값 분해를 통해 행렬 노름을 해석하기가 수월해진다.

수치적 안정성을 논할 때 자주 쓰이는 개념인 조건수도 대개 2-노름을 기반으로 하여 정의한다. 하지만 실제 계산에서는 1-노름이나 무한(norm)에 근접한 \(\infty\)-노름이 더 간단하게 다뤄질 수 있고, 문제의 특성에 따라 다른 노름을 선택해서 쓰기도 한다.

고차원 제어 시스템에서의 응용¶

고차원 시스템을 다루는 현대 제어공학에서는 대규모 행렬을 구성하는 일이 흔하다. 다차원 입력과 출력, 다양한 상태변수 등을 모두 포함하면 행렬 차원이 매우 커질 수 있다. 예를 들어, 수십~수백 개의 센서와 액추에이터가 연결된 산업용 제어 시스템에서는 이를 모두 포함하는 상태공간 모델이 수천 차원에 달하기도 한다.

이러한 대규모 시스템에서 이득(gain) 설계, 안정도 해석, 최적화 제어(예: \(\mathbf{H}_\infty\) 제어) 등을 수행하려면 본질적으로 대규모 선형대수 연산을 빠르고 안정적으로 해낼 수 있어야 한다. 이를 위해 메모리 관리와 병렬화, 희소 구조 활용, 수치적 안정성 확보 등을 종합적으로 고려해야 한다. 잘못된 방식으로 단순히 행렬 연산 라이브러리를 호출하면, 실행 시간이 지나치게 길어지거나 결과가 수치적으로 불안정해질 가능성이 높다.

구현 예시 (Python)¶

직접법이나 반복법을 실제로 구현할 때, 고수준 언어인 Python의 NumPy, SciPy 같은 라이브러리는 많은 편의를 제공한다. 대규모 문제에서 GPU 가속을 쓰거나 분산 처리를 적용하려면 추가적인 라이브러리가 필요할 수도 있다. 아래는 간단한 예시이며, 실제 대규모 문제에서는 더 고급 기법과 병렬화 환경 설정이 필요하다.

import numpy as np
from scipy.sparse import csr_matrix
from scipy.sparse.linalg import spsolve, cg

# 무작위로 큰 희소 행렬과 해벡터, 우변벡터를 생성
size = 10000
A_data = np.random.rand(size*5)  # 비율상 희소를 가정
A_indices = np.random.randint(0, size, size*5)
A_indptr = np.arange(0, size*5+1, 5)
A_sparse = csr_matrix((A_data, A_indices, A_indptr), shape=(size, size))

x_true = np.random.rand(size)
b = A_sparse.dot(x_true)

# 직접법(희소 행렬용 루틴)
x_direct = spsolve(A_sparse, b)

# 반복법(Conjugate Gradient)
x_cg, info = cg(A_sparse, b, tol=1e-8, maxiter=1000)

여기서는 간단히 희소 행렬을 구성하고, spsolve라는 직접법으로 해를 구한 뒤, conjugate gradient 반복법으로도 풀어본다. 실제로는 행렬 생성 과정에서 구조적 희소성을 반영하거나 사전조건자를 적용해 알고리즘의 수렴 성능을 높이는 과정을 거쳐야 한다.

대규모 행렬 계산은 제어공학뿐 아니라 과학기술 전반에 걸쳐 매우 중요한 문제로 부상하고 있다. 문제의 규모가 커질수록 단순 연산 횟수 증대와 더불어 수치적 안정성의 확보가 관건이며, 이를 위해 블록 알고리즘, 희소 행렬 기법, 병렬화, 피벗팅, 사전조건화, 반복적 해법 등의 수단이 종합적으로 활용된다. 알고리즘 선택은 행렬 구조와 하드웨어 환경, 그리고 원하는 정확도와 성능 요구사항에 따라 달라지며, 적절한 기법을 사용해야 수렴 속도가 빨라지고 반올림 오차 누적을 최소화할 수 있다.

현대의 고성능 컴퓨팅 자원을 적극 활용하면 과거에는 불가능하다고 여겨지던 규모의 문제도 충분히 풀 수 있게 되었지만, 하드웨어 스펙을 늘리기만 해서는 여전히 근본적인 수치 불안정성과 반올림 오차 문제를 해결하기 어렵다. 제어공학에서 안정적이고 정확한 모델 분석과 제어기 설계를 위해서는 하드웨어와 소프트웨어, 그리고 이론적 기법을 균형 있게 결합하는 전략이 필요하다. 대규모 행렬 계산과 수치적 안정성 이슈는 앞으로도 계속해서 주요한 연구 주제로 남을 것이며, 더욱 효율적인 알고리즘과 데이터 구조, 그리고 새로운 병렬화 기법이 제시될 전망이다.
