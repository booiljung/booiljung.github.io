희소행렬의 개념과 중요성¶

행렬에서 0이 아닌 원소가 매우 드문 비율로 존재한다면 그러한 행렬을 희소행렬이라고 한다. 즉, 전체 원소 개수에 비해 0이 아닌 원소의 개수가 상대적으로 적을 경우 희소행렬로 간주한다. 제어공학에서는 매우 큰 차원의 시스템을 해석하거나 시뮬레이션할 때, 대규모의 선형방정식을 풀어야 하는 경우가 자주 생긴다. 이때 행렬이 희소 구조를 가진다면 연산 효율을 극적으로 향상시킬 수 있다. 메모리 사용량 또한 크게 절감되어, 대규모 연산을 수행할 때 필수적인 수치해석 기법으로 간주된다.

희소행렬의 표현 형식¶

희소행렬을 효율적으로 다루기 위해서는 0이 아닌 원소와 그 위치 정보만을 별도로 저장하는 기법을 사용한다. 전통적인 밀집행렬(dense matrix) 표현은 모든 원소를 2차원 배열로 저장하지만, 희소행렬은 불필요한 0까지 모두 저장할 필요가 없으므로 다양한 압축 표현 방식을 개발해왔다. 대표적으로는 행 기반 압축(Compressed Row Storage), 열 기반 압축(Compressed Column Storage), 좌표 기반(Coordinate List) 표현 등이 존재한다.

행 기반 압축에서는 행마다 0이 아닌 원소의 열 인덱스를 압축하여 저장하고, 추가로 각 행의 시작 위치를 기록한다. 이러한 정보만 갖추면 전체 행렬에서 0이 아닌 원소가 정확히 어디에 위치하는지 판단할 수 있다. 열 기반 압축은 이와 유사하나 열별로 0이 아닌 원소를 압축하여 저장한다. 좌표 기반 표현은 0이 아닌 각 원소의 (행 인덱스, 열 인덱스, 실제 값)을 묶어서 리스트로 관리한다.

희소행렬을 어떤 방식으로 저장할지는 알고리즘 종류, 연산 패턴, 접근 방식 등에 따라 결정된다. 예를 들어 반복적(Iterative) 선형 시스템 해법에서는 행 벡터와 내적 연산이 빈번하므로 행 기반 압축을 사용하는 것이 일반적이다. 반면에 전치 연산이나 특정 열에 대한 접근이 잦다면 열 기반 압축이 유리할 수도 있다.

희소행렬의 예시¶

희소행렬이 구체적으로 어떤 구조를 가지는지 간단히 살펴본다. 예를 들어 \(3\times 3\) 크기의 행렬에서 0이 아닌 원소가 매우 적은 형태를 생각해보자.

\[ \begin{align} \mathbf{A} = \begin{bmatrix} 0 & 0 & 5 \\ 0 & 2 & 0 \\ 3 & 0 & 0 \end{bmatrix} \end{align} \]

위 행렬은 전체 원소 9개 중에서 0이 아닌 원소가 3개뿐이다. 이러한 구조에서는 굳이 모든 9개의 원소 값을 저장하지 않고, 3개의 값과 해당하는 좌표만 관리하는 방식이 훨씬 효율적이다.

희소행렬 연산의 특징¶

희소행렬 연산에서 가장 중요한 것은 0이 아닌 원소만을 효율적으로 다루어야 한다는 점이다. 행렬-벡터 곱 연산 \(\mathbf{A}\mathbf{x}\)을 예시로 들면, 희소행렬에서는 0이 아닌 원소가 존재하는 위치에서만 곱셈을 수행하면 된다. 즉, 0으로 가득 찬 항은 실제로 연산에 관여하지 않는다. 이는 곧 연산 시간과 메모리 사용이 전반적으로 절약됨을 의미한다.

한편, 희소행렬을 다루는 연산 알고리즘들은 원소 액세스가 불규칙해질 수 있어 캐시 적중률이 떨어질 수 있다. 따라서 적절한 저장 구조를 선택하여 캐시 효율을 높이는 방향으로 알고리즘을 설계해야 한다. 행렬의 크기가 매우 커지면 단순한 2차원 인덱스 접근만으로도 오버헤드가 커질 수 있으므로, 알고리즘 구현 시 데이터 접근 패턴과 메모리 구조를 신중히 고려해야 한다.

희소행렬과 대규모 선형방정식 해법¶

제어공학에서 자주 마주치는 문제 중 하나는 \(\mathbf{A}\mathbf{x} = \mathbf{b}\) 형태의 대규모 선형방정식을 해결하는 일이다. 이러한 문제에서 \(\mathbf{A}\)가 희소 구조를 가지면, 직접해법(direct method)뿐만 아니라 반복해법(iterative method)도 함께 고려한다. 반복해법은 \(\mathbf{A}\)가 대각우세(diagonally dominant)하거나 대칭 양정확(symmetric positive definite) 구조를 가지면 특히 강력하다.

대표적인 반복해법으로는 공액그라디언트(Conjugate Gradient)법, GMRES(Generalized Minimal Residual)법, BiCGSTAB(Bi-Conjugate Gradient Stabilized)법 등이 있다. 반복해법은 초기 벡터를 설정한 뒤, \(\mathbf{A}\mathbf{x} = \mathbf{b}\)를 만족하는 \(\mathbf{x}\)에 수렴할 때까지 반복적으로 해를 업데이트한다. 희소행렬을 사용하면 각 반복 단계에서 수행해야 할 행렬-벡터 곱이 매우 빠르게 계산될 수 있다.

희소행렬의 조건수와 수치 안정성¶

선형방정식 해석에서는 조건수(condition number)가 중요한 역할을 한다. 희소행렬이 대규모라 하더라도 조건수가 지나치게 크면 수치적 불안정성이나 느린 수렴 속도를 겪을 수 있다. 따라서 사전조건화(preconditioning) 기법을 적용하여 효과적으로 조건수를 개선하기도 한다. 사전조건화는 \(\mathbf{A}\)를 직접 바꾸기보다는, \(\mathbf{A}\)와 곱해서 나타나는 새로운 행렬 \(\mathbf{M}^{-1}\mathbf{A}\) 형태를 만들고 여기서 반복 알고리즘을 수행함으로써 수렴 속도를 높이는 방식이다. 희소행렬 환경에서도 사전조건화 행렬 \(\mathbf{M}\) 역시 가능한 한 희소 구조가 유지되도록 설계해야 메모리 사용이 증가하지 않는다.

희소행렬의 응용¶

제어공학에서 희소행렬은 다음과 같은 다양한 상황에 활용된다. 매우 큰 차원의 물리계(예: 유체역학, 전자기장 해석, 구조 해석)에서 유도된 미분방정식을 선형화하면, 시스템 매트릭스가 대부분 0인 형태로 나타난다. 또한 네트워크 이론에서 그래프의 인접 행렬이 큰 경우, 그래프의 노드 간 연결이 제한적이면 희소구조가 된다. 이러한 문제들을 효율적으로 풀어내기 위해서는 희소행렬의 저장 및 연산 최적화가 필수적이다.

크게는 모델 예측 제어(Model Predictive Control) 같은 고차원 최적화 문제를 실제 시간(real-time)으로 풀기 위해서도 희소행렬 기법이 활용된다. 대규모 공정을 실시간으로 제어해야 할 때, 희소 행렬 구조를 가진 최적화 문제를 빠르게 풀어야 하므로 선형대수학과 수치해석 기법, 그리고 병렬 연산 기법이 결합된다.

희소행렬 사용 예시(Octave)¶

희소행렬을 직접 생성하고 연산하는 간단한 예시를 Octave 코드로 살펴본다. Octave에서는 "sparse" 함수를 통해 행렬을 희소 형태로 정의할 수 있다.

% Octave code 예시
% 5x5 희소행렬 생성
A = sparse([
  0   0   0   2   0
  0   3   0   0   0
  0   0   4   0   0
  0   0   0   0   5
  6   0   0   0   0
]);

% 밀집행렬로 변환하여 확인
fullA = full(A)

% 난수 벡터 x와의 곱 연산
x = rand(5,1);
b = A * x;
disp("A * x 연산 결과:");
disp(b);

위 예시에서 sparse 함수를 통해 희소 구조를 가진 \(\mathbf{A}\)를 정의했고, full 함수를 통해 밀집행렬로 변환했을 때 실제 행렬의 내용을 확인할 수 있다. 이렇게 희소행렬로 정의한 \(\mathbf{A}\)를 사용하면, \(\mathbf{A}\mathbf{x}\) 연산에서 0이 아닌 원소에 대해서만 계산을 수행하므로 빠른 연산이 가능하다.

희소행렬 분석과 행렬 분해¶

대규모 제어시스템을 다룰 때는 \(\mathbf{A}\)를 직접 LU 분해나 Cholesky 분해로 처리하는 경우도 흔하다. 만약 \(\mathbf{A}\)가 대칭 양정확 구조라면 Cholesky 분해가 효과적이다. 희소행렬의 분해 과정에서 0이었던 위치에 새로운 0이 아닌 값이 생길 수 있으며, 이를 "fill-in" 현상이라고 한다. fill-in을 최소화하기 위해 행이나 열의 순서를 재배열하는 "reordering" 기법을 쓴다. 예를 들어서 최소 대각선 대역폭(minimum bandwidth)을 달성하기 위해 Cuthill-McKee 알고리즘 같은 것을 적용하면 fill-in을 줄여 메모리와 연산량을 감소시킬 수 있다.

LU 분해 시에도 pivoting 과정에서 fill-in이 발생할 수 있다. pivoting은 수치적 안정성을 개선하지만, 지나친 pivoting으로 인해 희소성이 손상될 수 있다. 그래서 희소행렬 전용 분석 알고리즘은 분해 전 단계에 "심볼릭 팩터라이제이션(symbolic factorization)"을 수행하여 fill-in 패턴을 미리 예측하고, 적절한 pivoting 정책을 세운다.

희소행렬과 제어시스템 시뮬레이션¶

고차원 상태방정식을 시뮬레이션하거나 대규모 피드백 구조를 해석할 때, 반복해서 대규모의 연립방정식을 풀어야 하는 상황이 생긴다. 예를 들어 유한요소법(Finite Element Method)으로 디스크리타이즈(discretize)된 물리계를 주기적으로 시뮬레이션해야 한다고 하면, 각각의 시점마다 희소행렬을 곱하거나 역행렬 근사 연산을 해야 한다. 이 과정을 효율적으로 수행하기 위해서는 희소행렬 구조를 적극적으로 활용해야 한다.

희소행렬 기반 모델링의 한계와 유의사항¶

희소행렬 구조를 활용하면 연산 속도와 메모리 면에서 큰 이점을 얻을 수 있지만, 모델링 단계에서 지나치게 복잡한 상호작용(즉, 0이 아닌 항이 많아지는 경우)이 추가되면 희소성이 약화될 수 있다. 또한 fill-in 현상이 많다면 실제로는 희소 기법이 오히려 불리해질 수 있다. 따라서 모델에 따라 희소 구조가 유지되는지 사전에 분석하고, 적절한 전처리나 행렬 재배열 등을 수행하는 것이 좋다.

희소행렬은 대규모 시스템 해석이 중요한 제어공학에서 필수적인 수치해석 기법이다. 0이 아닌 원소의 위치 정보만을 압축하여 효율적인 메모리 사용과 빠른 연산을 실현할 수 있으며, 반복적 선형해법이나 대규모 최적화 문제 해결에서 핵심적인 역할을 담당한다. 다만 희소행렬 알고리즘을 설계할 때는 fill-in 현상이나 자료구조 선택, 사전조건화, 재배열 기법 등에 대한 종합적인 고려가 필요하다. 이러한 여러 측면을 적절히 조합하면, 대규모 제어시스템을 안정적이고 신속하게 해석하고 시뮬레이션할 수 있다.
