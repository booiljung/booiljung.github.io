Keycloak에서 그룹은 여러 사용자를 하나로 묶어 동일한 권한이나 역할을 부여하기 위한 논리적 단위다. 그룹을 활용하면 조직 내 사용자의 권한 설정과 관리를 체계적으로 수행할 수 있다.

그룹의 개념¶

  • 권한의 중앙 집중화: 그룹을 사용하면 동일한 권한이나 역할을 여러 사용자에게 일관되게 부여할 수 있다.
  • 구성의 유연성: 그룹 간 계층구조를 설정해 상위 그룹의 역할을 하위 그룹이 상속받도록 할 수 있다.
  • 역할과의 연계: 그룹에 직접 역할(Role)을 부여하고, 그룹에 속한 사용자가 자동으로 해당 역할을 상속받도록 할 수 있다.

그룹 생성¶

  1. Keycloak 관리자 콘솔에 로그인한다.
  2. 왼쪽 사이드바에서 Groups 메뉴를 선택한다.
  3. Create group 버튼을 클릭해 그룹을 생성한다.
  4. 그룹 이름을 입력한 후 Save 버튼을 누른다.
  5. 필요한 경우, Attributes 섹션에서 그룹에 대한 속성을 추가하고 Roles 섹션에서 그룹에 역할을 할당할 수 있다.

하위 그룹 설정¶

  • Groups 화면에서 상위 그룹을 클릭한 후, Child Groups 탭에서 New 버튼을 클릭하면 하위 그룹을 생성할 수 있다.
  • 상위 그룹에서 할당한 역할이나 정책은 하위 그룹이 상속받는다.
  • 하위 그룹 자체적으로 필요한 역할을 추가로 설정할 수도 있다.

사용자 그룹 할당¶

  1. Keycloak 관리자 콘솔에서 Users 메뉴를 선택한다.
  2. 특정 사용자를 선택한 후 Groups 탭으로 이동한다.
  3. 사용자를 추가할 그룹을 선택하고 Join 버튼을 누른다.
  4. 여러 그룹에 중복으로 속할 수 있으며, 이를 통해 세분화된 권한 부여가 가능하다.

그룹에 역할 할당¶

  1. Keycloak 관리자 콘솔에서 Groups 메뉴를 선택한다.
  2. 역할을 할당할 그룹을 선택한 후 Role Mappings 탭으로 이동한다.
  3. Available Roles 목록에서 역할을 선택하고 Add selected 버튼을 클릭한다.
  4. 선택된 역할은 해당 그룹에 자동으로 매핑되며, 그룹에 속한 사용자 전체가 공통적으로 해당 역할을 상속받는다.

그룹 관리 시 고려사항¶

  • 그룹 설계: 조직 구조를 반영하여 최상위 그룹 및 하위 그룹의 역할과 상속 관계를 미리 정의하면 향후 권한 유지보수가 용이하다.
  • 역할 중복: 동일한 역할을 상위 그룹과 하위 그룹에 중복으로 부여하면 예상치 못한 충돌이나 과도한 권한이 발생할 수 있으므로 주의가 필요하다.
  • 속성(Attribute) 활용: 그룹에 특정한 속성을 부여하면, 어트리뷰트 기반 인증(ABAC) 시나리오에서 유연하게 활용할 수 있다.

Keycloak에서 그룹을 효율적으로 설계하면, 관리자의 권한 부여 업무가 단순화되고 오류를 줄일 수 있다. 특히 팀 또는 프로젝트별로 사용자 권한이 수시로 변동되는 환경에서는 그룹을 통해 권한을 구조화해 일관된 보안을 확보할 수 있다.
